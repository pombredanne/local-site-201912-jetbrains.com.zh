调试器改进

对 Xamarin 的支持

此版本添加了两项帮助您进行 Xamarin iOS 开发的实用功能。

性能改进

在发布 Rider 2018.3 后，我们收到了许多有关性能的反馈，所以我们重点改进了此方面：

  • 由于我们将 Xamarin Android 支持移动到一个单独的非捆绑插件中，点击 Rider 图标和利用进度条显示初始屏幕之间的时间大大缩短了。 另外，安装程序现在减小了 150 MB。
  • 在之前的一个版本中，为 Rider 二进制文件启动 NGen（原生图像生成器）已损坏。 因此，在 Rider 中打开解决方案时，在 Windows 上需要多等 8 秒，因为运行时也需要运行 JIT 编译进程。 现在，我们重新运行 NGen，在打开每个解决方案时节省 8 秒。
  • 我们还进行了两项热启动优化：在 JavaScript/TypeScript 支持中停止使用 ReSharper 引擎，以及将项目模型的加载设为异步。 因此，热启动现在显著加快（高达 2 倍）。
  • 我们实现了 Early Backend Load，将 ReSharper 语言服务的加载加快 3 秒。 如果“Reopen last solution on startup”设置已启用，您将注意到差别；或者您将快速闪电，并且能够在一秒内在 Welcome 屏幕上点击解决方案。

Razor 支持

Razor 页面上 Web 相关语言（例如 JavaScript 或 CSS）的一切内容已从 ReSharper 引擎移动到 WebStorm 引擎。 这意味着，您将获得对尖端前端技术和框架更出色的支持 - 直接在 Razor 视图中畅享对 Angular、React、Vue.js 和 Meteor 的 Emmet 支持与高级编码辅助。

由于 Razor 视图是最后使用 ReSharper Web 支持的地方，眼下只有 WebStorm 引擎可以分析您的 JavaScript 和 CSS。 由于启动 Rider 时不再加载与 ReSharper 相关的程序集，解决方案加载和 Rider 启动都变得更快。
