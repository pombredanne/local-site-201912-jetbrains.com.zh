ReSharper C++ 2019.2

启动更快

在这个版本中，我们已经完成了在过去几个发布周期中做的工作，现在可以正式宣布在 ReSharper C++ 中实现了新的预处理程序。 事实上，现在有两个 – 一个新的用于索引，一个原本的用于其他部份。 这个新索引在典型解决方案 (如 LLVM 和 Unreal 引擎) 上的索引速度提高了 15-25％。

代码补全

在代码补全中，ReSharper C++ 显示补全列表中每个项目的总结工具提示（由 ReSharper 选项中的 Environment | IntelliSense | Completion Appearance 页面的 Show summary 复选框控制）。 现在，已经用颜色表示选定补全项目的说明，并显示相应项目的文档。

代码提示

编辑器提供了更多代码提示，以帮助提高代码可读性。 在预处理程序指令之后引入了新提示，以及由非 const 引用传递的函数参数，而聚合初始化表达式中的提示现在显示了初始化基类的名称。

更多 C++20 支持

C++ 委员会已近乎准备好签署 C++20 标准。 当然，我们正在努力获得完全合规的支持。 ReSharper C++ 2019.2 支持：

  • 有条件的 explicit，
  • 新的 char8_t 类型，
  • 在 lambda 初始化捕获中的包扩展，
  • 默认的可构造和可分配的无状态 lambda，
  • C++20 聚合规则，
  • consteval 和 constinit 说明符，
  • 结构化绑定声明的 C++20 改进。

输入辅助

改进了多行注释中的输入辅助。 首先，使用单行语法的多行注释可以自动扩展。 其次，启用虚拟空格后，扩展多行注释现在自动插入正确的缩进。

用大括号换行代码块现在更容易 – 只需选择代码块或表达式，然后敲入左大括号。 ReSharper C++ 将替您完成其余工作：

  • 在函数声明和调用括号中使用连续行缩进。
  • 在声明中，偏好在 ")" 前换行。

代码分析

新检查可以帮助您捕获更多代码异味：

  • 不匹配的预处理程序指令。
  • 隐式删除明确默认的特殊成员函数。
  • 最终类中的冗余 final 函数说明符。

用于 C 风格转换的 Use static_cast 修复适用的检查可以批量运行，也可以作为代码清理的一部分。

导航

代码导航操作得到了目标准确的更新：

  • 当一个对象在变量声明和新表达式中默认初始化时，Go to declaration 可以导航到默认构造函数调用。
  • Find usages 在默认初始化中查找默认构造函数的用法。
  • Go to symbol 现在隐藏具有限定名称的符号（例如，类以外的函数定义）。

此外，可以通过 Go to File member 对话框在文件中搜索。 结果列表将包含所有匹配成员的名称，以及搜索字符串在文件中出现的所有位置。

文档

  • 来自 Google Test 的 GTEST_SKIP() 宏。

此外，我们还增加了受支持的框架数量（之前，我们涵盖 Google Test、Boost.Test 和 Catch），并添加了对 doctest 的支持。 这是一种相对较新的轻型框架，非常灵活。 它允许直接在生产代码中编写测试，同时提供了一种方式，让您可以从二进制文件中移除所有与测试相关的内容。

ReSpeller 集成拼写检查

ReSharper C++ 2018.2 提供了开箱即用的拼写检查功能，这得益于新捆绑的 ReSpeller 扩展，此扩展也进行了更新以支持 C++ 语言。 就像代码分析一样，ReSpeller 可以扫描标识符、注释和字符串文字中有无拼写错误和打字错误，并提供快速修复来解决这些问题。

遇到拼写错误时，您可以调用 Alt+Enter 菜单，ReSpeller 将提供多个选项来修复问题：我们可以修复打字错误，或者将此单词添加到 ReSpeller 的字典中。 默认情况下，拼写检查工具包含内置的英语（美国）字典，不过，也可以轻松添加其他语言的字典。

有关如何添加新字典的说明和 ReSpeller 的更多信息，请参阅 ReSharper 与 Rider 中的 ReSpeller 集成拼写检查。

格式化检查

有时，您希望在未遵守格式化规则时能在代码中直接获得通知。 考虑到这一点，ReSharper 代码格式化引擎引入了检查和快速修复来解决格式化程序相关的问题：现在可以观察不一致的缩进、换行、空格和空行，并直接在代码编辑器中修复。

默认情况下，格式化检查为禁用状态，这样就不会影响到在输入时不介意格式化问题的开发者。 转到 ReSharper | Options | Code Inspection | Inspection Severity | C++ | Formatting 可以查看所有可用的检查，并为那些您想要接收通知的检查选择严重性级别。

要了解更多信息，请参阅使用格式化检查保持一致的代码样式。

其他变更

与往常一样，ReSharper C++ 2018.2 也在不同方面进行了许多小幅改进：

  • 执行大小和对齐计算时，代码分析引擎将考虑位域大小、alignas 说明符和 __decltype(align) 特性。
  • 执行聚合初始化时，Parameter Info 可以提供聚合类成员的详细信息。
  • 代码文件中用于重写格式化程序设置的注释可以直接从 Configure Format 对话框中生成。
  • 基于范围的 for 语句可以提供跨越冒号的引用，从而轻松导航到循环使用的 begin() 和 end() 函数。
  • 为静态类字段和静态成员函数添加了不同的颜色设置。
  • Boost.Test 单元测试中支持预期的失败规范。
  • 对 ReSharper 的不同方面进行了许多性能修复，从加快 ClangFormat 支持到缩短解决方案加载时间。 如需了解更多详细信息，请参阅 ReSharper 2018.2 版本专属的性能博文系列。

ReSharper C++ 2018.1

调试步骤筛选器

步骤筛选器是 ReSharper C++ 为调试提供的一系列功能中最新添加的一项。 在您调用 Step Into 时，Visual Studio 调试器通常会分步执行从当前语句调用的所有函数。 不过，一些函数不重要或者已进行妥善测试，您不用分步执行它们。 这正是步骤筛选器的用武之地：借助此功能，您可以指定始终应当由调试器分步执行的函数。

ReSharper C++ 为标准库提供了一个预定义的步骤筛选器集合，但是，您可以随时添加新的筛选器：

  • 您可以在 Tools | Debugging | C++ 选项页面上检查、配置、禁用或启用步骤筛选器，此页面上列出了所有可用的步骤筛选器。
  • 当您单步执行（进入过程）函数并决定未来希望对其进行单步执行（跳过过程）时，则可以使用上下文操作，将此函数快速添加到步骤筛选器列表。 对于模板函数，可以使用单独的上下文操作来仅添加函数的当前实例化或所有实例化。

要了解更多信息，请参阅 ReSharper C++ 中的步骤筛选器。

Includes 分析器

编译时间是大型真实 C++ 项目中最大的问题之一。 ReSharper C++ 2018.1 引入了 Includes 分析器，这是一个新的代码检查工具，可以帮助您测量项目中哪些头文件在代码行方面对整体编译工作负荷的贡献最大。 要使用这一工具，请调用ReSharper | Inspect | Analyze Includes in …中的一个操作，或者从Solution Explorer的上下文菜单中选择Analyze Includes。 ReSharper C++会分析指定范围中的文件并在专用工具窗口中显示报告。

报告将按它们贡献的代码行数列示文件及其传递包含的文件。 随后，您可以通过 Includees 树形视图（子节点对应于在父节点中包含该文件的文件）或 Includers 树形视图（子节点为包含的文件）深入挖掘文件之间的依赖关系。 使用这两个视图时，您应当能够确定贡献最大的头文件，然后尝试消除任何不必要的 #include 指令。

导航改进

Go to File Member 对话框已进行重新设计。 现在，类成员在结果列表中按照它们的包含类分组。 此外，不用搜索，有效文件成员按其声明的顺序排序，而包含结果的列表可以容纳更多条目。 这些变更共同提升了对话框的实用性，可以提供更明确的当前文件结构洞察。

在所有 Go to 对话框和多个其他位置，ReSharper C++ 现在会切断函数和模板参数，尝试缩短过长的符号名称。

当您使用 Recent Files 对话框执行搜索时，Go to File 搜索结果也会追加到结果列表中，位于 Recent File 条目之后。

最后，用于记住 Search Everywhere 对话框中上一个搜索的选项现在默认处于启用状态。 您可以通过 Environment | Search & Navigation | Remember last search 设置恢复成旧行为。

命令行代码清理

ReSharper Command Line Tools 是一系列免费的独立工具，可以在命令行中或作为持续集成流程的一部分使用，现在，借助这些工具，您可以在 C++ 项目中执行代码清理。 在您的解决方案上运行 cleanupcode.x86.exe，它将自动重新格式化您的代码，移除常见冗余项，应用 Clang-Tidy 修复和常见的代码样式修复。 这可以立即修复项目或解决方案中的代码样式违规，并确保一致的基本代码。

同时，InspectCode 工具在 x64 项目上输出的虚假错误在此版本中也得到了修复。 此外，TeamCity 2018.1 还能在 C++ 项目上运行 InspectCode（请参阅 TW-48978 了解详细信息）。

要了解更多信息，请参阅使用 ReSharper 命令行工具清理代码。

代码检查

ReSharper C++ 添加了以下新的内置检查：

  • 当条件为编译时常量表达式时建议使用 if constexpr 语句替换 if 语句的新检查（包含对应的修复和代码清理项）。
  • 用于诊断已删除函数的用例的一组检查。
  • 控制流和用例查看检查已更新，可以在 lambda 主体中使用。

通过更多方式配置检查严重性

在 ReSharper С++ 2018.1 中，.editorconfig 文件现在可用于为任何范围设置检查严重性：整个解决方案、项目或特定文件。 除此之外，还可从项目的 DotSettings 文件读取检查严重性设置。 这意味着项目可以包含不同的启用/禁用检查集或不同的检查严重性。

在 Code Inspection | Settings | General 选项页面上选中 Read settings from EditorConfig and project settings 选项可以启用此功能。

要了解更多信息，请参阅使用 EditorConfig 配置检查严重性。

Clang-Tidy 集成更新

Clang-Tidy 集成获得了多处增强：

  • 绑定的 Clang-Tidy 已更新到版本 6.0，带来了多项新检查。
  • 借助新的检查操作，您可以在当前行按名称禁用特定的 Clang-Tidy 检查。
  • 当 Clang-Tidy 可执行文件崩溃时，从现在开始将不再显示系统错误对话框。

其他改进

ReSharper C++ 的其他变更包括：

  • 现在支持 #include_next 预处理程序指令，可与实现此 GNU 扩展的编译器更好地兼容。
  • C++ 文件中的输入辅助在您按 Enter 时会自动移除尾随空格，以保证您的代码整洁，无冗余空格。
  • 如果您选择表达式并打开 Quick Documentation 窗口，窗口中会显示有关选定表达式的各种信息，例如类型、值和值类别。
  • 新的 Tools | Unit Testing | C++ Tests | Use command-line arguments … 设置会控制 ReSharper C++ 在运行单元测试时是否应使用 Debugging | Local Windows Debugger | Command Arguments 项目属性的值。
  • 补全弹出窗口的性能已得到改进。

ReSharper C++ 2017.3

调试器功能

ReSharper С++ 2017.3 将调试器引入编辑器：局部变量、函数参数、当前行表达式和函数返回值现在会内联显示在相应代码的右侧。 此外，ReSharper C++ 还提供了新的 Go to Breakpoint 操作 (Ctrl+Alt+F9) 和 R# Breakpoints 工具窗口，用于在解决方案中的断点之间快速导航，以及从 Find Results 工具窗口创建断点和跟踪点。

ReSharper C++ 也提供了自己的调试器 DataTips，能够在内部搜索特定值。 要访问 ReSharper DataTips，请点击内联值显示或在 Tools | Debugger ReSharper 选项页面中替换 Visual Studio DataTips。

Clang-Tidy 集成

Clang-Tidy 是一款强大的开源代码分析工具，基于广受欢迎的 clang 编译器。 它提供了广泛的代码检查集和随附的修复。 ReSharper C++ 2017.3 提供与 Clang-Tidy 的无缝连接，这意味着它在您编辑代码时会在后台运行，内置检查的结果将实时显示在代码编辑器中。 ReSharper C++ 还让您能够作为范围操作单独应用 Clang-Tidy 提供的修复，或从 Code Cleanup 应用修复。

除了 Clang-Tidy 检查的诊断，Clang-Tidy 也会给出来自 Clang Static Analyzer 的诊断和 clang 编译器自身提供的警告。 所有这些诊断都有相应的 ReSharper 检查以及可配置的严重性级别，您可以根据自己的需要调整级别。

请注意，Clang-Tidy 当前不支持使用 UTF-16 或 UTF-32 编码的文件。

语言理解

改进 C++ 理解是 2017.3 版本的重点，现在支持一些 C++17 和 C++14 功能：

  • C++17 结构化绑定声明。
  • C++17 内联变量。
  • C++17 constexpr if 语句。
  • 在不使用来自 C++14 的尾随返回类型的情况下推导 auto 函数的返回类型。
  • lambda 表达式的类型检查和返回类型推导，包括对 C++14 泛型 lambda 的支持。

其他的语言相关变更包括：

  • 支持 include_alias pragma 指令。
  • 改进了显式模板实例化的处理。
  • 支持事件处理 Microsoft C++ 扩展。

代码检查和上下文操作

ReSharper C++ 2017.3 提供了一组新的代码检查和随附的修复：

  • 带相同或相等参数的二进制运算符。
  • 从另一个声明或基本函数重定义默认函数参数。
  • 函数声明中参数上的冗余常量说明符。
  • 文档注释中缺少函数参数说明。
  • 成员初始值设定项列表中的冗余成员初始值设定项。
  • 头文件中的非内联变量定义。
  • 未以下划线开头的用户自定义文字后缀。
  • 结构化绑定声明中错误的相关详细诊断。

还添加了一些新的上下文操作：

  • 将成员初始值设定项列表中的成员初始值设定项转换为非静态数据成员初始值设定项。
  • 在类型模板参数声明中的 class 和 typename 关键字之间切换。

代码格式化

现在可以从注释启用/禁用格式化程序，并使用内联 EditorConfig 样式设置对其进行配置。 还添加了一些新的选项，可以实现更灵活的格式化，包括控制以下功能的选项：

  • 多行代码格式化：
    • 为赋值、声明和单行注释对齐列中的代码。
    • 减小缩进链式方法调用中的逗号和成员访问运算符。
  • 统一初始化大括号中逗号前/后的空格和行结尾注释之前的空格。

Go to Declaration 改进

现在可以在模板定义内的依赖名称上使用 Go to Declaration 命令。 ReSharper C++ 无法查找名称引用的确切实体，而是会找到带有匹配名称和签名的所有符号并作为潜在导航目标显示。

而且，借助 Go to Declaration，您现在还可以从函数定义导航到包含非匹配签名的声明（如果找不到匹配的声明）。
