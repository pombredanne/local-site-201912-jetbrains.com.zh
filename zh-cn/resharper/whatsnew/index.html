ReSharper 2019.2

增强的 C# 支持

通过 ReSharper 的增强支持充分利用 C# 8：

  • 支持索引和范文中的最新变化，比如可建索引的和 可设置范围的类型。
  • 上下文操作 Add range check 现在支持按 System.Index/System.Range 建立索引，并且产生正确的检查。
  • 支持非托管通用结构。
  • 两个新检查，‘将 switch 语句转换为 switch 表达式 (Convert switch statement to switch expression)’ 和 ‘将三元表达链 转换为 switch 表达式 (Convert chain of ternary expressions to switch expression)’，以及相应的 ‘转换为 switch 表达式 (Convert to switch expression)’ 快速修复。

应用户要求：上下文操作将连接转换为插值 (Convert concatenation to interpolation) 现在可应用于范围。

启动更快

我们借鉴了 Rider，开始使用基于 MSBuild 的项目模型代替 Visual Studio COM API。 这将有助于优化解决方案加载时间，因为我们已经尽量不使用 COM API， 这会锁定 UI 线程。

此外，我们还更新了性能指南页面。 它新增了 Windows Defender 部分 和所有性能提示的说明，阐明了在应用建议后将要 完成的操作。

导航和查找 用途

在该版本中，Find Usages 开始使用来自全解决方案的错误 分析数据以收集结果。 如果在解决方案中启用 SWEA，Find Usage 显示结果的速度将明显加快。

我们稍微改进了 Navigation，希望您认为 它们有用： 

  • 可以通过“Go to File Member” 对话框在文件中搜索。 结果列表将包含所有匹配成员的名称和 搜索字符串在文件中出现的所有位置。
  • 在 Go To Text 结果窗口中，如果不想在注释字符串中 查找出现情况，您现在可以过滤掉注释行。

调试器数据提示

ReSharper 2019.2 更新了调试器数据提示：

  • 我们新增了几个切换：隐式函数评估和 自动评估属性。
  • 它们遵守 “Enable property evaluation and other implicit function calls” 设置，可以按要求评估属性值。

代码样式和代码格式化程序

我们为 switch 表达式新增格式化选项，以改进对 C# 8.0 的支持。 改进了对 .editorconfig、StyleCop.Settings 和 .clang-format 的 支持，并且默认启用高亮显示和全解决方案错误 分析。 还可以在选项页面中编辑 EditorConfig 属性。

支持 “Per-Monitor DPI Awareness” 模式

ReSharper 旗舰版 2019.2 改进了对 Visual Studio 2019 引入的 “Per-Monitor DPI Awareness” 模式的支持。 我们知道因为 ReSharper 工具窗口曾经存在的渲染问题，某些 ReSharper 用户以前选择禁用 优化像素密度不同的屏幕的渲染 (Optimize rendering for screens with different pixel densities) （通过 Tools | Options | Environment | General）。 请再次启用它，试试 其效果。 请注意，可能仍有一些地方会出现故障。

其他功能

  • 已经改进了参数名称提示的性能并且新增启发。
  • IL Viewer 提供带有关键字说明的新工具提示。
  • ReSharper | Options | Tools | External Sources 下的高级符号选项对话框 现在支持相对路径。
  • 程序集浏览器 (Assembly Explorer) 工具窗口的“属性 (Properties)” 选项卡现在提供“总结 (Summary)” 属性， 以及非托管 (win32) 资源子树。

以下是我们做的代码格式化改进：

  • 在您按下 Backspace 时，取消代码缩进的新输入辅助。
  • 现在可以将所有代码格式化和代码风格设置导入 .editorconfig 文件，并且与不使用 JetBrains 产品的同事共享。

为 Go to Text 移动文件

Go to Text 后的引擎考虑解决方案中的所有 文本文件。

以前，Go to Text 只考虑被 ReSharper 建了索引的文件。 ReSharper 只为包含代码或可以帮助编程的文件建索引，因为在大多数情况下 为其他文件建立全文索引是昂贵的。 因此，所有文件扩展为 “unknown” 的文件都完全超出范围， 意味着不能搜索非代码 文件。

我们升级了引擎！ 现在，显示来自建了索引的文件的结果时， 引擎在后台搜索未建索引的文本文件。 如果以 不同扩展名储存某些数据，比如 *proj、.yml、 和 .csv 时，这很有用。

SDK 项目的引用优化

ReSharper 现在也检查 SDK 项目中未使用的包引用。 使用项目上下文菜单中的 “Optimize References” 来分析和优化这些 引用。 如果项目不含使用给定引用的代码，它将 显示为未使用。 轻松卸载未使用的包，保持项目依赖项树 整洁！

要了解更多信息，请参阅 导出 EditorConfig 代码风格，SDK 项目的优化引用，以及更多 ReSharper 2019.1 更新 。

改进的 Extract Method 重构

Extract Method 重构现在更加开发人员友好。 如果为 引入参数自动生成的名称不是您想要的，只需在重构对话框 中编辑名称，以及引入的方法名称、返回类型和 可见性。

代码编辑器中的提示

现在，代码中有更多地方提供参数名称提示。 首先，可能在属性的 用法中找到它们以显示属性参数的名称。 其次，对于 由 StringFormatMethod 属性标记的方法特殊提示，例如：Console.WriteLine()， 并显示其值将插入该位置的参数的索引。 请 注意，默认禁用 “StringFormatMethod” 提示，但可以在选项中启用。

另外，我们已经为方法声明中的参数引入了新的代码注释 提示。 它们显示了通过 ReSharper 代码分析计算的参数的代码注释 （如果在以下位置指定了一个）：

  • 重写父类中的基方法，或
  • 在父接口中声明的已实现的方法。

如果您在使用之前的ReSharper版本对.NET Core应用程序进行单元测试时遇到问题，请注意，单元测试运行程序现在支持.NET Core项目中的NUnit（1.1和2.0），并且确保您可以在.NET Core 2.0项目中运行MSTest和xUnit.net。

最后但同样重要的是，ReSharper的解决方案生成器现在支持.NET Core项目，可以帮助您节省重新编译应用程序所花的时间。

改进的C# 7.0支持

ReSharper对C# 7.0的支持会随着每个版本的发布而改进。 在这个发布周期中，我们优化了ReSharper处理模式匹配的方式：

  • switch后缀模板现在可以在非密封引用类型的变量上使用。
  • 一个新的上下文操作 - Generate type patterns，可以帮助您轻松地为选定类型生成switch用例。
  • Convert to switch快速修复已完全重新设计，可以将多个if语句转换为包含模式的单个switch语句。

ReSharper现在支持在out变量中使用var（或者根据您的代码样式，使用显式类型）。 此外，还添加了检查和快速修复，用于引入out变量，而不是引入仅用于包含out参数的方法调用的变量。

理解C# 7.1

Visual Studio 2017 15.3引入了C# 7.1，ReSharper已凭借自身的语言功能迎头赶上。 例如，C# 7.1添加了能够使用的default文字，而不是default(T)表达式，根从而据用例上下文推断目标类型。 ReSharper可以识别default文字语法，在使用default(T)时提供检查，并提供快速修复建议以移除冗余类型规范。

ReSharper还通过一个检查支持元组投影初始值设定项，此检查可以在冗余元组值组件名称推断自初始值设定项时显示它们。

最后，ReSharper可以理解async main以及与泛型匹配的模式。

新的IEnumerable检查

ReSharper的一组IEnumerable检查已扩展，以便适应新的场景：

  • Possible multiple enumeration代码检查现在可以考虑从IOrderedEnumerable和ParallelQuery派生的类型。
  • 新的代码检查可以检测从IQueryable到IEnumerable的可能非有意转换。

更多的代码检查和上下文操作

在C# 7.1中，推断了值元组的元素名称，ReSharper会帮助您找到冗余的值元组组件名称。

Introduce auto-property from parameter快速修复已经允许我们引入get-only自动属性和许多选项，例如添加私有setter或使它成为公开可变属性。 现在，Initialize auto-property from parameter上下文操作与快速修复保持一致，可以提供相同的选项。

Null检查偏好设置

有许多方法检查null值，如果您不喜欢ReSharper在生成代码时引入null检查的方式，该怎么办？

在ReSharper 2017.2中，我们添加了新的选项页面 - Null checking，可以在这里配置快速修复、上下文操作和代码生成操作使用的null检查模式的优先级。 您甚至可以创建自己的自定义检查！ 如果您使用Check parameter for null上下文操作，请查看它的新子菜单项Configure null-check pattern，此选项可以帮助您访问这个新页面，而无需在ReSharper的Options对话框中查找。

导航和搜索改进

Go to Everything和其他导航操作现在可以匹配目标查询中以任意顺序包含单词的元素：例如，搜索exactMatching会匹配isMatchingExactly。

另一方面，精确搜索现在可以像搜索引擎一样运行：查找"Collection"会返回Collection，但不会返回FilteredCollection和IterateCollection()，等等。 不过，使用精确搜索筛选复合名称时，您仍然可以使用通配符*和?，根据需要精确地限定变化。

此外，Go to Text现在已集成到Go to Everything（CtrlT）中：如果没有代码匹配您的搜索项，ReSharper将开始搜索文本。

而且，您现在可以使用点分隔符搜索扩展方法：Class.ExtensionMethod。

最后，ReSharper 2017.2引入了新的快捷键，可以在Find Results窗口中探索搜索结果。 按ShiftEnter，而不是数字键盘上的+。

导航到附近的文件

ReSharper 2017.2中的上下文导航已改进：使用Navigate To菜单，您现在可以查看并导航到项目结构中当前文件邻近的文件：浏览与当前文件位于同一个目录级别的文件夹和文件，轻松跳到这些文件或创建新文件。

<inheritdoc/>支持改进

已引入新的检查，用于支持在XML文档中使用<inheritdoc/>。

检测到继承文档的多个候选项时，ReSharper会显示警告并建议快速修复以插入cref参考。 在生成的cref特性中，ReSharper会方便地显示包含所有要引用的可能类型的补全列表。

此外，当派生类重写它的基类的文档时，会显示警告。

新的C#输入辅助

  • ReSharper让您能够快速地使用NotNull和CanBeNull特性标记参数，而无需输入整个特性名称。 在类型名称、参数或成员声明中的名称之后输入!或?会分别插入NotNull或CanBeNull注释。 如果某个名称已经使用NotNull标记，输入!会自动在方法主体的开头处添加null检查。
  • 在=>后面输入{会快速地将表达式转换为花括号中的代码块。
  • 在右大括号前面按Enter会将脱字符号留在同一行，但会向右缩进一次。

TypeScript、JavaScript和JSON支持改进

  • 完全支持 TypeScript 2.3，包括：异步迭代器、可选泛型、无状态 JSX 组件的重载解析、对象文字的上下文 this， 还有 --strict 选项。
  • TypeScript 2.4：支持包含字符串值和混合字符串/数字值的枚举，改进了对根据上下文类型返回进行的泛型推理和泛型上下文签名的支持。
  • 在TypeScript中为映射类型成员改进了Find Usages和Rename。
  • TypeScript中添加了新的Introduce type alias和Inline type alias重构。
  • 在纯JavaScript中提升了代码补全的性能。
  • 改进的JSON支持，包括JSON中的快速文档和对package.json中范围包的支持：现在可以在这些包中使用代码补全、工具提示和快速文档。
  • 文字中的语言注入：注入的XML、C#和JavaScript/TypeScript中的注入路径引用。

Angular 支持

ReSharper 2017.2支持Angular输入/输出别名和特性指令。 现在也支持通过NPM添加的Angular 2组件。

ReSharper提供了代码补全，可以显示快速文档并能够帮助导航到组件声明。 请注意，必须在HTML | JavaScript and Frameworks下的ReSharper选项中配置Support Angular markup in HTML pages。

在Angular 4中，ReSharper 2017.2添加了对*ngIf中; else的支持，以及对*ngIf和*ngFor中变量赋值（如people as person）的支持。

ReSharper 2017.1

Visual Studio 2017 RTM支持

现在终于完全支持Visual Studio 2017的新功能。 您现在可以使用以轻量级模式加载的解决方案。 使用Open Folder选项时，ReSharper会查看文件夹中的所有文件，并支持.xml和.js文件类型。 Move to Folder重构可以在此模式下运行。

ReSharper遵守当前的目标框架上下文。 这是通过编辑器窗口顶部的下拉菜单设置的，并且可以更改设置，例如正在引用的程序集以及构建和预处理程序中定义的符号。

C# 7支持

ReSharper添加了许多新的检查以符合C# 7编译器。 例如，它可以在重写方法或实现接口时检测元组变量名称中的拼写错误。

ReSharper通过一个新的快速修复（Convert a read-only delegate variable into a local function）支持局部函数，还添加了两个新的上下文操作：

  • 将lambda表达式或匿名方法转换为局部函数
  • 将局部函数转换为常规方法

Join null check with assignment快速修复将赋值、null合并运算符和throw相结合，让代码更具可读性。 现有的上下文操作Convert to '?:' operator和.throw后缀模板也支持C# 7 throw表达式。

单元测试

ReSharper的单元测试适用于Visual Studio 2017 RTM中基于project.json和.csproj的项目。 它可以发现并运行MSTest和xUnit测试，并支持代码覆盖率和分析以及以多个框架为目标。 支持以.NET 4x框架为目标的NUnit测试。 还有对MSTest v2的DataRow支持。

请注意，ReSharper目前不支持.NET Core单元测试的连续测试。 此问题将在未来的更新中修复。

EditorConfig支持

EditorConfig支持在默认情况下已启用。 ReSharper理解标准EditorConfig属性和大多数RoslynEditorConfig属性，并提供了一组丰富的自定义EditorConfig属性，用于更精细地配置格式化规则。 这意味着您可以在EditorConfig文件中维护格式规则的整个配置。

File Formatting Info窗口可以显示在所有活动的.editorconfig文件中定义的范围和属性。 此窗口还可以显示缩进设置的来源及其值，以及缩进自动检测的状态。

缩进自动检测和代码样式改进

如果您的大多数团队成员都使用ReSharper，则ReSharper的分层设置功能是维护一致代码样式的绝佳工具。 现在，此功能得到扩展，针对所有支持的语言添加了Tabs and Indents设置。

如果您重新格式化文件中的部分代码，或者在编辑或粘贴时自动格式化代码，ReSharper可以根据该文件中的现有缩进计算和应用缩进。 当您想要为缩进与您的设置不同的现有项目作贡献时，此功能会派上用场。

格式化规则的上下文配置支持С#、C ++、JavaScript和TypeScript。 在需要重新格式化的选定代码块上按AltEnter，然后在操作列表中选择Format Selection | Configure可以查看影响代码的所有格式化规则。

代码清理、新代码样式和格式化选项

ReSharper的Code Cleanup获得了易用性改进。 现在，您可以在Code Cleanup对话框中创建和配置任务特定的自定义分析，只需按CtrlE，C。

ReSharper可以将它的格式化规则应用于您粘贴的任何代码。 默认情况下会设置仅影响缩进的Reindent选项，您可以在Editor Behavior选项页面上更改。

ReSharper为C#类型成员主体引入了新的代码样式设置，允许您在使用带大括号的主体块或使用表达式格式之间进行选择。

最后，我们改进了C#代码格式化引擎并引入了新的格式化选项：

  • 新的换行引擎现在可正确处理对齐
  • 支持逗号开头的换行样式
  • 在“nameof”的括号前和括号内设置空格
  • 为链式二进制表达式选择首选换行样式
  • 单个“case”语句中的可配置换行

导航和搜索

Find Results窗口获得了一个新选项，可以按用例类型对搜索结果进行分组，同时还提供了新筛选器来选择特定用例的实例，并且支持使用引号搜索精确匹配项。

ReSharper的Search & Navigation页面添加了一个新选项，可以从任何地方在Preview选项卡中打开文件。 默认情况下，此选项处于关闭状态。

Go to Everything和Go to Text弹出窗口支持内联命令，可用于对搜索结果进行高级筛选。

Angular 2模板语法

ReSharper理解Angular 2模板语法的以下元素：模板表达式、模板语句、NgFor指令和模板变量。 实现了对Angular @Component修饰器中的template:属性以及@Component中的templateUrl:属性引用的纯HTML的支持。 templateUrl中仅支持相对路径。 对于HTML页面，默认情况下会关闭Angular标记，您可以在ReSharper的Options（Code Editing | HTML | Editor）中启用此标记。

代码完成适用于Angular特性和组件标记，以及[attr.、[style.和[class.。 组件特性获得了对Go to Declaration（F12）命令的支持。 Rename重构适用于组件标记和[class.foo]中的“foo”。

请注意，尚不支持Angular 1。 如果在ReSharper选项中将Support Angular markup in HTML pages设置为“None”，则可将AngularJS插件用于代码补全和实时模板。

TypeScript和JavaScript支持

  • 完全支持TypeScript 2.1：映射和索引类型、对象rest和spread属性、文字类型的更新逻辑、配置继承、无类型导入、组合类型规范化、部分注释签名以及隐式“any”和隐式“any”数组的控制流分析
  • 对TypeScript 2.2的初始支持：“object”类型、JSX spread语法和从签名/元组/交叉/映射类型派生（扩展/实现）
  • 在TypeScript ES6样式的导入中检查和快速修复未使用的导入
  • JavaScript 和 TypeScript 中的后缀 模板
  • 改进了TypeScript和JavaScript中代码补全项的相关性
  • Generate Code菜单获得了一个新选项，可以为TypeScript类生成属性或只读属性。 Generate Overriding Members命令现在会尽可能调用“super”
  • 在TypeScript中导航到实现成员
  • 在TypeScript和JavaScript中导航到函数出口
  • TypeScript中的Rename重构对局部符号的处理速度更快，并获得了一个可以为TypeScript禁用动态搜索的新选项

代码分析

ReSharper的文件状态指示器（错误指示条顶部的小图标）获得了一个新的上下文菜单。 右键点击图标，您不仅可以快速切换ReSharper代码分析，还可以快速切换标识符高亮显示和Visual Studio代码分析（灯泡和波浪线）。 如果当前文件中存在代码问题，您还可以看到用于在最高严重性级别的问题之间导航的命令。

请注意，对于大型文件，ReSharper会自动挂起代码分析。 在这种情况下，您会在状态指示器中看到灰色的暂停图标。
