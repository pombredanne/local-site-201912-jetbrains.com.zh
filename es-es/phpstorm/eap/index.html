Compatibilidad con PHP 7.4

PHP 7.4 is now released, and PhpStorm 2019.3 fully supports all its new features. El IDE también ofrece inspecciones para destacar desusos y arreglos rápidos para actualizar su base de código.

Propiedades por tipos

Hemos añadido compatibilidad con propiedades por tipos en la versión 2019.2. En v2019.3, los arreglos rápidos son capaces de gestionar los casos especiales, y puede añadir declaraciones de tipo rápida y fácilmente pulsando Alt-Enter y seleccionando Add declared type for the field. PhpStorm detectará el tipo automáticamente basándose en el PHPDoc, valor predeterminado o declaración de tipo de argumento por parte del constructor.

Funciones flecha

PhpStorm es totalmente compatible con la nueva sintaxis breve para funciones lambda de una línea en PHP 7.4. El IDE permite completar variables del ámbito externo y, con un solo clic, PhpStorm le llevará a la declaración.

También está disponible un arreglo rápido para convertir funciones lambda de una línea de la sintaxis antigua a la nueva sintaxis breve. Basta con pulsar Alt-Enter en una función anónima y seleccionar Convertir clausura para función flecha.

Separador literal numérico

En PHP 7.4 puede añadir separadores de guiones bajos a un número largo para formatear visualmente y mejorar la legibilidad de su código.

Si ve un número largo, sitúe el signo de intercalación sobre él, pulse Alt+Enter, y seleccione Añadir separadores numéricos. Así añadirá '_' tras cada tres dígitos para números decimales, o cada cuatro dígitos en hex, binario y octal.

Operador ternario anidado

PHP 7.4 desprecia la asociatividad izquierda de los operadores ? : anidados. Esto significa que o no debería usar ternarios anidados en absoluto o, si tiene que hacerlo, debería usar paréntesis para declarar explícitamente el orden de computación.

PhpStorm 2019.3 destacará las expresiones en desuso y le ofrecerá un arreglo rápido para explicar el comportamiento.

Desusos

Esta versión de PHP no solo ha añadido funcionalidades nuevas, sino que también ha dejado en desuso algunas de las antiguas.

PhpStorm 2019.3 cuenta con las inspecciones correspondientes y destacará trozos de código que requieren atención. Cuando sea posible, tendrá la opción de efectuar un arreglo rápido automáticamente en el comportamiento obsoleto.

Retornos covariantes y parámetros contravariantes

PHP 7.4 resuelve múltiples incoherencias, incluida una que afectaba al sistema de tipos. Cuando trabaje con un método heredado de una clase primaria, ahora será posible definir un tipo de retorno más específico, y tipos de parámetros menos específicos para él.

PSR-12

El nuevo estándar de estilo de código PSR-12 se amplía al anterior PSR-2 con normas adicionales y nuevas directrices para algunas de las nuevas funcionalidades PHP. PhpStorm 2019.3 presenta un nuevo estilo de código PSR-12 predefinido, y el IDE sugerirá que lo utilice. También puede pasar a utilizarlo manualmente mediante la acción Set from… en Preferences | Editor | Code Style | PHP.

PhpStorm también sugiere habilitar un conjunto de inspecciones de estilo de código PSR-12 que comprobarán si se han incumplido ciertas reglas.

Puede reformatear el código en el editor pulsando Cmd+Alt+L o ejecutarlo antes de confirmar. También puede reformatear todo el proyecto o carpeta con Code > Code Cleanup...

WSL

El subsistema de Windows para Linux es una capa de compatibilidad para ejecutar ejecutables binarios de Linux de forma nativa en Windows 10. PhpStorm ahora es compatible con el desarrollo en un entorno WSL, es decir, cuando lanza el IDE en una máquina Windows pero el objetivo de su desarrollo es WSL/Linux. Puede especificar un intérprete de PHP remoto en el WSL y utilizarlo para ejecutar scripts, pruebas y comandos de Composer, así como para depurar.

Especifique el intérprete para las configuraciones de ejecución de tests de PHP

Anteriormente, sus configuraciones de ejecución de pruebas solían ejecutarse solo a través del interprete PHP del proyecto predeterminado, lo cual podía resultar poco práctico.

Pero ya no es así. En PhpStorm 2019.3, puede seleccionar cualquier intérprete que tenga configurado y ejecutar sus tests con él: Docker, Vagrant, via SSH, WSL, o cualquier otro. ¡Usted elige!

Barra de herramientas del editor

Cuando abra un archivo .http, ahora verá una barra de herramientas que le dará acceso rápido a todas las acciones habituales:

  • Añadir solicitud: cree solicitudes a partir de plantillas.
  • Convertir desde cURL: cree solicitudes pegando una cURL en el editor.
  • Abrir registro: vea la lista de solicitudes enviadas y abra/compare las respuestas.
  • Ejemplos: explore una colección de prácticas solicitudes HTTP.

Variables dinámicas

Hemos añadido tres variables dinámicas que puede utilizar en las solicitudes:

  • $uuid: genera un nuevo UUID-v4.
  • $timestamp: marca de tiempo Unix actual.
  • $randomInt: un íntegro aleatorio entre 0 y 1000.

Utilice dobles llaves curvas para insertar estas variables cuando lo necesite.

Ejecutar todo

Puede escribir varias solicitudes en un archivo .http. Esto resulta muy práctico si necesita efectuar cadenas de solicitudes en las que una solicitud depende del resultado de las anteriores.

Antes, tenía que ejecutar las solicitudes una a una. Ahora, en PhpStorm 2019.3 ¡puede ejecutarlas todas con un solo clic!

Plegar solicitudes

Si va a enviar una solicitud con un cuerpo largo, por ejemplo, datos de formularios de varias partes, o muchos encabezados, quizá quiera plegarlos para reducir el ruido en el editor.

Puede plegar toda la solicitud dejando solo la primera línea o el cuerpo, haciendo clic en el icono de flecha en el medianil.

A veces, a revisar el registro del historial de versiones, es más sencillo entender la información si la hora se muestra en un formato relativo, como dd/mm/aaaa o mm/dd/aaaa si se encuentra en EE. UU.

Información sobre PhpStorm 2019.2

Typed Properties para PHP 7.4

La versión de PHP 7.4 se espera para finales de noviembre. PhpStorm está empezando a poner en marcha la compatibilidad, para que tenga tiempo de hacer pruebas y de planificar la migración. Probablemente, la funcionalidad más esperada sea la nueva Typed Properties, con la que PhpStorm 2019.2 ya es totalmente compatible. Las nuevas inspecciones resaltarán las infracciones de tipo y podrá actualizar su base de código con el arreglo rápido Add declared type for the field. PhpStorm detectará el tipo de forma automática en función del PHPDoc, el valor predeterminado o la declaración de tipo de un argumento en un constructor.

Encuentre duplicados sobre la marcha

El hecho de que aparezcan los mismos bloques de código en su proyecto puede hacerle perder tiempo al tener que realizar cambios o incluso provocar errores de regresión si olvida actualizar todas las instancias. PhpStorm ayuda a prevenir la creación de duplicados de código en su código resaltándolos sobre la marcha para que los pueda localizar fácilmente. Ahora, si se topa con un duplicado o crea uno en su código, ¡lo sabrá al momento! La nueva inspección se llama Duplicated code fragment y está habilitada de forma predeterminada. Para configurarla, vaya a Preferences | Editor | Inspections | General. Puede modificar la configuración correspondiente en Preferences | Editor | Duplicates. Y, por supuesto, puede ejecutarla en todo el proyecto (o ámbito predefinido) en cualquier momento a través de Code → Run Inspection by Name.

Resaltado y comprobación de expresiones regulares

Las expresiones regulares son una herramienta extremadamente útil para buscar y reemplazar en las cadenas. Ahora con PhpStorm, dispone de inyecciones automáticas de expresiones regulares en PHP. Esto significa que, siempre que su código utilice funciones preg_*, el patrón se resaltará y se comprobará que es correcto. Además, ahora puede probar sus patrones directamente en el IDE. Mueva el signo de intercalación sobre el patrón y presione Alt+Intro para ver la acción rápida Check RegExp.

Ejecute Composer mediante cualquier intérprete remoto de PHP

Previamente, debe tener un intérprete de PHP local para ejecutar Composer. El problema con esto era que el entorno local a menudo no reflejaba el proceso real de producción o no había siquiera un intérprete de PHP local. En PhpStorm 2019.2, puede elegir cualquier intérprete remoto para ejecutar su Composer con Docker, Vagrant, mediante SSH o cualquier otro.

Vaya a Preferences | Languages & Frameworks | PHP | Composer, seleccione Remote Interpreter y elija un intérprete del desplegable, o añada uno nuevo.

Intenciones y arreglos rápidos

Nuevas intenciones en la Herramienta de manipulación de cadenas

PhpStorm viene con varias intenciones Alt+Intro para manipular cadenas. En 2019.2 encontramos una nueva.

Si tiene que introducir un nuevo parámetro en una expresión sprintf, simplemente, selecciónela, ejecute la intención Extract selected string part as format function argument y PhpStorm elegirá la ubicación correcta del parámetro. También funciona para la concatenación.

Convertir una lista de líneas simples a multilínea, y al revés

Según PSR-2, las líneas de más de 80 caracteres deben dividirse en varias líneas de menos de 80 caracteres cada una. De modo que, si la lista de parámetros o el vector han quedado demasiado largos, ahora puede separarlos en un clic usando la nueva intención Alt+Intro Split comma-separated values into multiple lines. La intención contraria también está disponible en caso de que quiera combinar elementos en una única línea.

Intención e inspecciones para declaraciones Switch

Ahora puede añadir ramas más rápidamente copiando las que ya existen y modificándolas. Puede usar Alt+Intro en la declaración de instancia sin cuerpo y elegir Copy ‘switch’ branch. No olvide modificar la que ha copiado, porque PhpStorm le mostrará una advertencia si un bloque Switch tiene una rama duplicada, es decir, el mismo cuerpo, que puede ser tanto redundante como haber estado escrito por error. Además, si en PhpStorm 2019.2 tiene una declaración Switch sin una rama predeterminada en su código, también puede arreglarla rápidamente con Alt+Intro.

Compatibilidad con PHPT

Los archivos .phpt pueden ayudarle a escribir pruebas funcionales de forma simple. Este formato se utiliza para probar el PHP principal. PhpStorm 2019.2 es compatible con archivos .phpt e incluye resaltado de secciones y scripts, inyección automática de lenguajes PHP e INI en las secciones correspondientes, finalización de nombres de secciones y saltos a archivos referenciados en secciones EXTERNAS.

Depuración de plantillas Twig y Blade

La depuración es muy efectiva cuando se enfrenta a un error desagradable o se adentra en una base de código. Pero es una pesadilla examinar el código generado automáticamente de plantillas compiladas. Con PhpStorm y Xdebug, ahora puede llevar a cabo una depuración por pasos en archivos .twig y .blade.php originales sin compilar de estos dos populares motores de plantillas. Aquí encontrará todas las ventajas de la depuración, como ver el contexto, las variables locales y globales, añadir watches, ejecutar paso a paso y muchas otras.

Localización de código muerto

Si trabaja con una base de código heredada y desea hacer limpieza, una buena forma de empezar es eliminando el código muerto. Ahora PhpStorm detecta código redundante y señala las clases, los miembros de clase y las funciones que probablemente nunca se utilizan. La inspección de declaraciones no utilizadas tiene en cuenta usos dinámicos del código, por ejemplo, a través de métodos mágicos. Para comprobar el informe de todo el proyecto, seleccione Code -> Inspect Code….

Refactorización, intenciones y arreglos rápidos

Herramientas de manipulación de cadenas

Hemos incorporado nuevas intenciones para la manipulación de cadenas. Ahora puede convertir una concatenación de cadenas en interpolación, convertir una concatenación o interpolación en una llamada de sprintf o copiar una cadena concatenada al portapapeles, entre otras muchas cosas. Pulse Alt+Intro en una cadena para ver las acciones disponibles.

Eliminación de variables no utilizadas

PhpStorm 2019.1 presenta un nuevo arreglo rápido que le ayudará a deshacerse de las variables no utilizadas que aparecen en las listas de uso de asignaciones o clausuras. El arreglo rápido es lo suficientemente inteligente como para detectar situaciones en las que la expresión de la asignación puede tener efectos secundarios.

Refactorización Mover a clase

Con la nueva refactorización Mover a clase, puede transferir funciones o constantes a la clase a la que pertenezcan lógicamente y PhpStorm buscará y modificará todos los usos por usted. Puede mover varias funciones simultáneamente seleccionándolas en la ventana de herramientas Estructura.

El método puede ser una inspección "estática"

Si una clase tiene un método que no utiliza ninguna referencia de instancia, PhpStorm lo detectará y le ofrecerá un arreglo rápido: pulse Alt+Intro en el método y hágalo estático. Todos los usos se actualizarán automáticamente.

Docker

Reutilización de contenedores

En el caso de intérpretes que se basen en Docker Compose, ahora puede escoger entre docker-compose run o docker-compose exec para ejecutar contenedores. Si tiene un contenedor pesado que no quiere volver a iniciar en cada prueba ejecutada, puede reutilizarla si elige la opción docker-compose exec. O puede utilizar docker-compose run para contenedores ligeros o aquellos que no funcionen en el modo daemon (es decir, que se detienen justo después de iniciarse).

De esta forma se incluyen situaciones en las que le gustaría ver sugerido un determinado conjunto de constantes en vez de algún tipo sencillo como un número entero o una cadena. O si espera que alguna función devuelva una determinada constante. También puede mejorar las sugerencias en PhpStorm para su biblioteca o proyecto si incorpora su propio archivo .phpstorm.meta.php.

Importaciones rediseñadas

Hemos rediseñado las inspecciones y las acciones de intención relacionadas con la importación de espacios de nombres y el uso de FQN. La principal idea subyacente es evitar los calificadores en la medida de lo posible. Ahora, PhpStorm le dejará eliminar simplemente un calificador redundante si es posible o sustituirlo por la declaración de importación de "uso" correspondiente. Asimismo, cuando copie código en un archivo, PhpStorm le pedirá que reutilice un alias existente.

Limpieza de código

En PhpStorm 2019.1, la herramienta de limpieza de código incluye intenciones específicas de PHP: puede optimizar ocurrencias de nombres de clase añadiendo la declaración de uso o eliminando la parte no necesaria. También puede arreglar problemas de estilo de código automáticamente con PHP CS Fixer o PHP_CodeSniffer’s phpcbf. Ejecute una limpieza antes de una confirmación de VCS, o actívela manualmente en cualquier momento a través de Code -> Code Cleanup…..

Tecnologías web

As usual, all the features and improvements of WebStorm are available in PhpStorm, either out of the box or with free plugins available in the repository.

Documentos actualizados y compatibilidad con navegadores

La documentación (F1) de propiedades CSS y etiquetas y atributos de HTML ahora muestra descripciones actualizadas e información acerca de la compatibilidad del navegador desde MDN, así como enlaces a los artículos completos de MDN. Asimismo, para comprobar que todas las propiedades de CSS que utiliza son compatibles con la versión del navegador de destino, puede habilitar una nueva inspección de compatibilidad de navegador en las preferencias.

Soporte mejorado para TypeScript en aplicaciones Vue

PhpStorm ahora aprovecha el servicio de lenguaje TypeScript junto con su propio soporte de TypeScript para el código de TypeScript en archivos .vue. Esto significa que ahora tendrá una comprobación de escritura e información de escritura, podrá utilizar los arreglos rápidos proporcionados por el servicio, y ver todos los errores de TypeScript en el archivo actual en la ventana de la herramienta TypeScript.

Nuevas inspecciones para aplicaciones Angular

Para aplicaciones Angular, PhpStorm añade 17 nuevas inspecciones que le ayudarán a detectar errores específicos de Angular en su aplicación a medida que escribe y le sugerirá arreglos rápidos. Estas inspecciones funcionan en los archivos tanto de TypeScript como de plantilla y comprueban el uso de uniones, directivas, componentes y muchas otras cosas.

Nueva consula depuradora

¡Descubra la nueva y mejorada consola de depuración interactiva en las ventanas de la herramienta de depuración de JavaScript y Node.js! Ahora muestra objetos a través de una vista de árbol y permite aplicar estilos a mensajes de registro con CSS y agruparlos usando console.group() y console.groupEnd(). También puede filtrar para ocultar cualquier tipo de mensaje de registro.

Función de conversión con Promesa a async/await

Puede cambiar automáticamente una función que devuelve una Promesa con llamadas .then() y .catch() a una función async que utiliza la sintaxis async/await. Simplemente pulse Alt-Intro en el nombre de la función y seleccione Convertir a función async. Esto es posible no solo en archivos TypeScript, pero también en JavaScript y Flow.

IDE

Ventana emergente de ubicaciones recientes

En esta nueva ventana emergente de navegación encontrará los puntos de código visitados recientemente presentados en contexto (un par de líneas anteriores y un par de líneas posteriores). Todas las ubicaciones están ordenadas cronológicamente en esta ventana emergente, con la última ubicación visitada en la parte superior. Para invocar la nueva ventana emergente de ubicaciones recientes, pulse Cmd-Mayús-E / Ctrl+Mayús+E. Escriba cualquier texto para buscar en contexto.

Mejoras de VCS

Una nueva casilla de verificación "Uncheck all’" para confirmaciones parciales en Git le permite desmarcar todos los fragmentos de código a la vez y, a continuación, elegir los que desea confirmar. Además, las acciones fixup, squash y cherry-pick ya están disponibles en Git Log.

Temas de interfaz de usuario personalizados

¡Si los temas predeterminados blanco y Darcula no son suficientes para usted, pruebe Dark Purple o cree uno personalizado! El color de cualquier elemento de su IDE, desde los iconos a los botones y flechas, es ahora configurable. ¡Ajuste todo según sus preferencias y guárdelo como un nuevo complemento de tema!

Herramientas para bases de datos

Las herramientas para bases de datos de PhpStorm presentan algunas mejoras útiles gracias a nuestros colegas del equipo de DataGrip:

  • Compatibilidad con nuevas bases de datos: Greenplum, Vertica y Apache Hive
  • La finalización de código es compatible con instrucciones combinadas para CREATE y DROP
  • Compatibilidad con atributos DEFINER en MySQL y MariaDB
  • Compatibilidad con el modo Oracle en MariaDB
  • Ahora puede configurar la carpeta por defecto para un proyecto

Otras mejoras:

  • Nuevas opciones de color para tipos de parámetros primitivos y miembros de clase por visibilidad. Puede configurar el esquema de color en Preferences | Editor | Color Scheme | PHP.
  • Ahora puede habilitar las envolturas suaves en el editor para tipos de archivo específicos. Para ello, abra Preferences/Settings | Editor | General y especifique los tipos de archivo en el campo "Soft-wrap file".
  • Se ha mejorado la configuración de estilo de PHPDoc: clasificación de declaraciones de uso, definición del orden de las etiquetas y personalización del número de espacios. Para encontrar todas las opciones, vaya a Preferences | Editor | Code Style | PHP y busque la pestaña PHPDoc.
