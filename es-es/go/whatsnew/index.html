Novedades de GoLand

GoLand 2019.3 consume menos CPU y presenta un rendimiento más rápido, mejora la compatibilidad con módulos Go, añade un conjunto de arreglos rápidos nuevos y mejora las refactorizaciones. Ya puede llamar a un método de un campo incrustado mediante Evaluate Expression y depurar sus aplicaciones de forma remota utilizando Mozilla rr.

También puede habilitar un Global File Watcher para cada proyecto nuevo, trabajar con Mongo DB ¡y mucho más!

2019.3 29 de noviembre

Rendimiento

Desde el lanzamiento de GoLand 2019.2 este verano, hemos trabajado duro para que GoLand funcione más rápido y consuma menos CPU. Una de las principales mejoras de rendimiento es la reducción de los tiempos de inicialización, cuando el IDE lo prepara todo entre bastidores, muestra la pantalla de bienvenida e indexa un proyecto.

Tenga en cuenta que los resultados pueden variar dependiendo del tamaño del proyecto y de la antigüedad de su dispositivo, RAM y espacio en disco. Seguiremos trabajando en esto, para que inicializar sea todavía más rápido en versiones futuras.

Mejoras en Go Modules

GoLand 2019.3 presenta destacado de sintaxis y navegación para archivos go.mod. Pulse ⌘+B en macOS o Ctrl+B en Windows/Linux en una ruta de importación de dependencias específica y GoLand le llevará a sus fuentes en Project View.

También en Project View, los módulos Go se mostrarán con su ruta de importación completa, en lugar de solo su último fragmento. La versión de cada módulo Go también se mostrará en un color de fuente más suave que la ruta de importación, para que pueda distinguirlos fácilmente.

Actualizaciones del depurador

Método de llamada de un campo incrustado

La evaluación de expresiones del depurador ahora le permite llamar directamente a un método de un campo incrustado.

Depuración remota con Mozilla rr

Muchos servicios de producción escritos en Go se ejecutan en Linux, y Mozilla puede ayudar a investigar errores difíciles de reproducir. La depuración con Mozilla rr es compatible desde GoLand 2018.3, pero solía funcionar únicamente en máquinas de desarrolladores que trabajaban con Linux. Ahora ya se puede arrancar un servidor de depuración en una máquina Linux y depurar la traza de forma remota para macOS y Windows. Para ello, siga los pasos siguientes:

  • Recopile una traza de rr en la máquina remota.
  • Arranque el depurador en la máquina remota del modo siguiente: dlv --headless --api-version 2 -l localhost:2345 replay /path/to/trace/dir /path/to/binary.
  • Conéctese a la máquina remota a través de la configuración de Go Remote.

Tenga en cuenta que Mozilla rr solo está disponible en máquinas Linux.

Nuevos arreglos rápidos

Gestión de parámetros no utilizados

Ahora GoLand destaca el nombre y el tipo del parámetro no utilizado, y puede eliminarse ambos o solo el nombre (si el tipo se utiliza para implementar una especificación de método).

En este último caso, el IDE ofrece dos arreglos rápidos entre los que puede escoger: borrar nombres de parámetros o Rename to_. Lo mismo sucederá con funciones que se utilizan como tipo de función.

Además, ahora también puede quitar todos los parámetros no utilizados de una sola vez. Pulse Alt+Enter y utilice el arreglo rápido Delete all unused parameters.

Creación de un método o función ausentes

GoLand ofrece un arreglo rápido para crear una función o método ausentes cuando se utiliza en un tipo de función. Solo tiene que pulsar Alt+Enter y escoger la opción de Create method/function.

Manejo de los alias de paquete mejorados

GoLand ahora sugiere importar paquetes basados en sus alias utilizados a través de Alt+Intro en cualquier archivo en su proyecto.

Refactorizaciones

Change Signature

Change Signature ahora siempre muestra el tipo de parámetros en las pestañas de Parameters y Result parameters. Marque la nueva casilla de verificación de Reuse types para contraer los tipos de consecuencias en la Signature Preview y el resultado.

Además, encontrará dos nuevas acciones de intención, Reuse signature types, que combina las definiciones de parámetro en una sola declaración, y Expand signature types, que añade tipos a cada parámetro utilizando Alt+Enter.

Extract Variable y Rename

Extract Variable y Rename han mejorado en cuanto a la sugerencia de nombres. Por ejemplo, ahora se sugiere err para expresiones de error de tipo, se sugiere ctx para context.Context, etc.

Comportamiento mejorado de las inspecciones de código

Las inspecciones de código de GoLand ahora están más centradas. Hemos reducido el rango de destacado de las incidencias detectadas en el editor de código, para ayudarle a mostrar los arreglos rápidos más relevantes y las acciones de intención para un trozo de código en particular al invocar el menú de contexto mediante Alt+Enter.

Edición de código

Lexer de literal de runas mejorado

Hemos mejorado el lexer de literal de runas de varios modos:

  • GoLand ofrece un mensaje de error más claro y conciso que explica por qué la expresión es incorrecta.
  • GoLand no destaca un literal de runas válido como error tras editar.
  • El análisis de literales de runas no válidos es más estable.

Resaltado de reasignaciones

Go permite utilizar variables existentes en el lado izquierdo de las declaraciones var breves. Así pues, dos construcciones que parecen iguales pueden tener significados distintos. Para que sea más fácil detectar estos casos, hemos añadido resaltado de sintaxis para estas reasignaciones.

Plegado de código para literales compuestos

GoLand 2019.3 ofrece plegado de código para literales compuestos. Utilice el plegado de código para plegar o expandir el fragmento de código, o pulseCtrl+NumPad Minus/Ctrl+NumPad Plus en Windows o ⌘+NumPad Minus/⌘+NumPad Plus en macOS. Para configurar el comportamiento de plegado de código predeterminado, vaya a Settings/Preferences | Editor | General | Code Folding.

Plantillas activas mejoradas

Las plantillas activas for y forr ahora sugieren nombres de variables relevantes. Por ejemplo, se sugieren nombres como i e index para un índice; s, string, y line son las sugerencias para un valor.

La plantilla activa meth ahora sugiere código para nombres de receptores.

Navegación

Buscar solo en un área seleccionada

Busque solo en un área seleccionada al utilizar Find (⌘+F en macOS y Ctrl+F en Windows/Linux). Basta con seleccionar el trozo de código que necesite en el editor e invocar Find – el IDE efectuará una búsqueda en el área seleccionada. Para cambiar fácilmente a buscar dentro del archivo completo, pulse ⌘+F/ Ctrl+F una segunda vez o cambie la opción In Selection en la barra.

Uso de ‘Find in Path’ con una opción de máscara de archivo

Ahora, al buscar en el diálogo Find in Path utilizando una máscara de archivo, GoLand muestra las extensiones de archivo en los resultados de búsqueda. Así puede distinguir más fácilmente entre archivos con el mismo nombre, pero con diferentes extensiones de archivo.

Habilitar Global File Whatcher para todos los proyectos nuevos

En GoLand 2018.2 añadimos una opción para compartir una configuración de file watcher entre distintos proyectos usando las opciones Project y Global; se denomina Global File Watcher.

Sin embargo, todavía tenía que habilitar este watcher en cada proyecto nuevo. Ahora ya puede habilitarlo de forma predeterminada en todos los proyectos nuevos que cree o abra.

En el menú File, seleccione Other Settings | Settings for new projects, y vaya a Tools | File Watchers, añada un global watcher nuevo y, a continuación, marque la casilla de verificación situada a su lado para habilitarlo en todos los proyectos nuevos.

Tutoriales

A menudo, cuando tiene un proyecto que acabar y unos plazos que cumplir, uno no tiene tiempo de leer todas las páginas de Novedades, notas de la versión y artículos del blog acerca de los productos que utiliza. Por eso hemos añadido tutoriales para que aprenda las funciones de GoLand cuando usted quiera.

Puede encontrar Tutorials en la pantalla de bienvenida, bajo su lista de proyectos. Cuando actualice el IDE a una nueva versión, encontrará una lección que hable de las funcionalidades de esta nueva versión. Como alternativa, puede acceder a los tutoriales a través de la ventana de herramientas correspondiente desde cualquier proyecto.

Más cambios

Ventana de herramientas de jerarquía

Existe una nueva opción para analizar la jerarquía del método del llamado en la ventana de herramientas Hierarchy. Hemos añadido dos botones específicos nuevos que le permiten cambiar entre llamantes y llamados del método seleccionado. Ahora también puede clasificar los elementos de un árbol alfabéticamente, haciendo clic en un botón especial en el menú. Para ver la ventana de herramientas Hierarchy en la lista Tool Windows, tendrá que crear primero una jerarquía y luego ir a Navigate | Call Hierarchy o Navigate | Type Hierarchy.

Cambiar complementos de mapa de tecla y tema instantáneamente

En GoLand 2019.3, puede instalar y desinstalar cualquier complemento de tema y de mapa de teclas sin reiniciar el IDE.

Un acceso directo personalizado para volver a centrarse en el editor

Establezca su acceso directo personalizado para la acción de volver a centrarse en el editor, en lugar de usar ESC. Encontrará la acción Focus Editor en Preferences/Settings | Keymap; haga clic en ella con el botón derecho y seleccione Add Keyboard Shortcut en la lista desplegable.

Barras de desplazamiento más visibles

Hemos añadido una opción para hacer las barras de desplazamiento más visibles. Para ello, seleccione la casilla de verificación Use contrast scrollbars en Preferences/Settings | Appearance & Behavior | Appearance.

Mejoras del control de versiones

Rediseño del diálogo Clonar

El diálogo Clone está situado bajo VCS | Get from Version control. Ahora puede iniciar sesión desde el cuadro de diálogo o, si ya ha iniciado sesión, el IDE le mostrará la lista de todos los repositorios agrupados por cuentas u organizaciones.

Más información acerca de la fusión de conflictos

Ahora es más fácil investigar un conflicto durante una combinación, rebase u operación de cherry pick. Basta con hacer clic en el enlace Show Details para obtener la lista de confirmaciones que dieron lugar al estado de código resultante, y obtener más información acerca de la fuente de los cambios mostrados en el diálogo Merge.

Actualizaciones de bases de datos

Compatiblidad con MongoDB

GoLand 2019.3 ofrece soporte básico para MongoDB. Ahora puede:

  • Visualizar las colecciones y los campos en el explorador de la base de datos.
  • Abrir cualquier colección u observar el resultado de la consulta. También es posible ordenar por columnas o filtrar los valores.
  • Ejecutar consultas y obtener los resultados. Seleccionar una instrucción y ejecutarla pulsando Ctrl/⌘+Enter o utilizando el botón Play en la barra de herramientas.

Búsqueda de error de conexión

Ahora es más fácil explorar los errores de conexión. Cuando falla una conexión, basta con hacer clic en el botón especializado del IDE para ir a los resultados de búsqueda de su error utilizando los motores de búsqueda de Google.

Desarrollo web

Finalización para bibliotecas de CSS en CDN

Si utiliza Bootstrap o cualquier otra biblioteca de CSS y la tiene vinculada desde un CDN en un archivo HTML, ahora dispone de finalización para los nombres de las clases desde dicha biblioteca, sin añadir sus fuentes al proyecto.

Sugerencias de nombre para nuevos símbolos

Al añadir nuevos nombres para las variables y los parámetros en JavaScript o TypeScript, ahora irá obteniendo sugerencias de nombres en función de los nombres de las clases, las interfaces y los tipos de alias definidos en el proyecto, las bibliotecas estándar y las dependencias que utilice.

En el caso de las variables, las funciones, las clases y las interfaces nuevas, también podrá obtener sugerencias de nombres que ya se usaron en el mismo ámbito, pero que todavía no se han definido.

2019.2 25 de julio

Compatibilidad con llamadas a funciones

Examine en detalle lo que sucede en el interior de su expresión en cualquier punto concreto de la ejecución de su programa.

Tenga en cuenta que esta funcionalidad se basa en la característica experimental de compatibilidad con la llamada a función de Delve y Go Runtime.

Si encuentra errores o incidencias con la compatibilidad con las llamadas a funciones, informe de ello a nuestro sistema de seguimiento de incidencias.

Combinación de colores predeterminada y Darcula mejorada

GoLand 2019.2 agrega una variedad al resaltado de código semántico en la combinación de colores predeterminado y Darcula.

Esto le ayudará a comprender mejor su código y seguir su lógica más rápidamente. Distinga fácilmente los nombres de paquete de las variables, las construcciones de código integradas de las suyas propias y las conversiones de tipo de las llamadas a funciones. Ahora los receptores de métodos y funciones/llamadas a funciones tienen también sus propios colores.

Como es habitual, puede cambiar todo a su gusto en Settings/Preferences | Editor | Color Scheme.

Actualizaciones del depurador

Más información sobre variables con el método String()

Ahora el depurador muestra los resultados de las implementaciones simples String(), Error() y DebugString() en el panel de variables. Admite métodos con una única declaración de retorno sin llamadas.

Mejor presentación de tipos especiales

GoLand 2019.2 muestra el valor de Struct derivado de los tipos time.Time, time.Duration y net.IP en un formato de lenguaje natural en el panel de variables del depurador.

Formato predeterminado para enteros

Elija cómo mostrar variables de enteros en el depurador de forma predeterminada: en formato binario, decimal o hexadecimal. Establezca el formato en Settings/Preferences | Build, Execution, Deployment | Debugger | Data Views | Go.

Compatibilidad con funcionalidades de Go 1.13

GoLand 2019.2 admite literales numéricos y enteros con signo como recuentos de desplazamientos. Estos cambios están relacionados con los pasos que estamos dando hacia Go 2. Vaya al blog oficial de Go para obtener más información.

El resaltado de código de GoLand le avisará de cualquier error y le mostrará correctamente el resultado en la Consola.

Lea este artículo para obtener más información sobre las nuevas funcionalidades de Go 1.13 y cómo GoLand 2019.2 es compatible con ellas.

Plantillas de finalización Postfix personalizadas

GoLand 2019.2 le permite crear sus propias plantillas personalizadas de finalización de Postfix.

Navegue hasta Settings/Preferences | Editor | General | Postfix Completion, haga clic en +, seleccione Go, defina su plantilla y seleccione los ámbitos donde se podrá aplicar la plantilla.

Para obtener más información sobre la creación de plantillas personalizadas, consulte este artículo.

Además, puede encontrar una plantilla nueva predefinida de finalización Postfix — .aappend. Antepone una expresión con una llamada a funciones integrada append y asigna su resultado a esta expresión.

Arreglos rápidos basados en Change Signature

Hemos agregado muchos arreglos rápidos basados en la refactorización de Change Signature (Cambiar firma). Por ejemplo, con solo pulsar Alt+Intro, puede inferir automáticamente y agregar un parámetro a la llamada a función o agregar los parámetros de retorno faltantes en una firma.

Refactorización de extracción de método con retornos

La refactorización Extract Method (Extraer Método) ahora gestiona código con declaraciones de retorno. Si el código siempre sale a través de retornos, se extraerá tal como es. Sin embargo, si contiene las rutas de ejecución sin retornos, el método extraído tendrá un indicador booleano adicional como valor de retorno. El indicador se usará para realizar un retorno anticipado en el lado de llamada.

Mejoras de finalización de código

La finalización de código ahora ofrece sugerencias para miembros de receptor aunque no haya receptor. No se recomiendan los nombres de receptor como this o self. Como resultado, los métodos tienen diferentes nombres de receptor. Esta funcionalidad de finalización de código le permite ignorar la diferencia.

Módulos Go más inteligentes e integración dep

GoLand 2019.2 permite la integración de Go Modules/dep automáticamente cuando detecta que un proyecto emplea Go Modules o dep. Anteriormente, deshabilitaba la indexación GOPATH y mostraba solo una notificación sobre la habilitación de la integración de los Go Modules/dep. Nos dimos cuenta de que esto no bastaba porque era fácil perderse o ignorar la notificación y, por tanto, que un proyecto pasara a estado semiconfigurado.

Si esto no le sirve, puede deshabilitar la integración manualmente en Settings/Preferences | Go | Go Modules (vgo) en cualquier momento.

Inspecciones de código

Conversiones de tipo redundantes

Una inspección de código llamada conversiones de tipo redundantes le ayuda a mantener su proyecto más legible y ordenado. Le advierte de las conversiones de tipo redundante que pueden omitirse.

Validación de índices de segmentos en literales

Hemos añadido validación de índices de segmentos en literales compuestas. Esto significa que el editor le advertirá de que su índice debe ser un entero no negativo constante antes de que compile su proyecto.

Edición de código

Compatibilidad con secuencias de escape

La compatibilidad con secuencias de escape en literales de cadena está completamente equipada con:

  • Resaltado de código.
  • Una inspección que le advertirá sobre posibles problemas.
  • Edición inteligente que comprende la ubicación de las secuencias de escape.
  • Corrección ortográfica más precisa.

Los marcadores de posición de formateo están ahora destacados en diferente color para que pueda localizarlos más rápidamente en cadenas largas. Puede personalizar el color en Settings/Preferences | Editor | Color Scheme | Language Defaults | String | Escape Sequence | Valid. O, solo para Go, vaya a Settings/Preferences | Editor | Color Scheme | Go | String | Valid escape.

Acción de rellenado de campos

Ahora es más sencillo rellenar los campos de structs. Primero, debe elegir qué campos desea rellenar en una ventana específica. Después es posible editar los valores generados. Finalmente, esta acción está ahora disponible a través de la finalización de código.

Extensión de selección mejorada

La función Extend Selection ahora extiende el código de manera semánticamente correcta, desde una línea a un bloque de código separado por líneas vacías (en lugar de seleccionar todo el cuerpo de método).

Mejoras de interfaz de usuario

Nuevo aspecto para New | File dialog

El diálogo New | File se ha convertido en una ventana emergente con una lista con todas las opciones posibles para crear un tipo de archivo particular. Anteriormente, tenía que expandir la lista para elegir una alternativa, pero ahora todas las opciones disponibles están visibles de manera inmediata.

Página de complementos mejorada

Ahora GoLand muestra los detalles de los componentes directamente, sin necesidad de abrir las páginas de detalles de complementos uno a uno. Hemos añadido más opciones bajo el icono de rueda dentada para que pueda deshabilitar o habilitar todos los complementos descargados de una vez. El botón Update se sitúa ahora cerca del nombre del complemento en la pestaña Installed. Se ha prescindido de la pestaña Updates puesto que ya no es necesaria.

Ventana de la herramienta de búsqueda ordenada

En la ventana de la herramienta Find, hemos agrupado algunos iconos que se usan poco y hemos eliminado otros por completo.

Cambios generales útiles

Más opciones del seleccionador de ámbitos

Al seleccionador Scope se han añadido ahora nuevos ámbitos como Recently Changed, Recently Viewed Files, Files in Previous Search Result y Selection. También podrá seleccionar los ámbitos Data Sources. Además de esto, hemos añadido la posibilidad de seleccionar un ámbito para la refactorización Rename. Pulse Mayús+F6 dos veces para invocar la ventana de refactorización Rename y seleccione los ámbitos en los que se realizará el cambio de nombre.

Nueva opción para controlar el movimiento del signo de intercalación

El comportamiento predeterminado de la acción Move Caret to Next Word en GoLand 2019.2 ha cambiado: el IDE mueve ahora el signo de intercalación al final de la palabra actual. Sin embargo, puede ajustarlo donde le vaya bien. Vaya a Settings/Preferences/Settings | Editor | General y seleccione dónde debería colocar el IDE el signo de intercalación: al final de la palabra, al principio de la siguiente palabra o incluso entre ambos límites.

Más información en la vista de proyectos

Cambie In-place Descriptions en el menú View para obtener una vista previa del tamaño del archivo y las modificaciones de fecha y hora en Project View.

Mejoras del control de versiones

Confirmación desde Local Changes

GoLand 2019.2 ofrece un método de confirmar directamente desde Local Changes.

Ahora mientras se trabaja en una confirmación, puede explorar el código fuente, ver el historial de un archivo específico, ver el diff para el archivo en la misma área que la confirmación o usar otras funcionalidades del IDE.

Para habilitar esta funcionalidad y confirmar desde Local Changes sin un cuadro de diálogo, vaya a Settings/Preferences | Version Control | Commit Dialog y seleccione la casilla de verificación Commit from the Local Changes without showing a dialog.

Mejoras para archivos .gitignore

GoLand resalta correctamente los archivos ignorados y ofrece sugerencias de finalización de código para el nombre de carpeta y de archivo. Ahora puede añadir rápidamente un archivo a .gitignore mediante el menú contextual desde la pestaña Local Changes de la ventana de herramientas VCS: haga clic derecho sobre él en el grupo de archivos Unversioned y seleccione Add to .gitignore.

Opción para abortar la combinación

Hemos añadido la opción Abort Merge que estará disponible en la ventana emergente Branches cuando haya una combinación en curso. Este es el futuro con el que muchos de nosotros hemos soñado, donde ya no es necesario cambiar al terminal para abortar.

Columnas del registro de VCS personalizables

Haga clic en el icono con forma de ojo para elegir las columnas que se visualizan en la pestaña Log, seleccione el elemento Show Columns desde el menú contextual y elija las columnas que desea ver: Author, Date o Hash.

2019.1 21 de noviembre

Perfiladores Memory, CPU, Mutex y Block

GoLand ha ampliado la compatibilidad de sus herramientas al integrar los perfiladores Memory, CPU, Mutex y Block. Puede comenzar a hacer la generación de perfiles de su aplicación directamente desde el IDE y revisar después los resultados del análisis en una ventana específica de la herramienta (View | Tool Windows | Profiler).

Para ejecutar el generador de perfiles, haga clic en el nuevo botón Run with Profiler y escoja la configuración requerida en la barra de herramientas de navegación. En su defecto, seleccione cualquiera de las nuevas acciones del menú de ejecución, que aparecen al hacer clic en el icono de ejecución en el medianil izquierdo.

Para comprender mejor las posibles incidencias de rendimiento de su código, clasifique los resultados según la métrica disponible para cada tipo de generador de perfiles en concreto; utilice las pestañas Flame Chart, Call Tree y Method List; o navegue incluso hasta el código fuente para inspeccionar las incidencias.

Smart Step Into

La depuración de una cadena de llamadas y las llamadas de métodos anidadas es más sencilla con la nueva acción del depurador, Smart Step Into. Le permite entrar en llamadas arbitrarias en una expresión y saltar automáticamente los métodos que no le interesen.

Por ejemplo, al depurar código como fmt.Printf("Me llamo %s, tengo %d! años", me.FullName(), me.Age()), puede optar por entrar específicamente en Age().

Para invocar Smart Step Into, pulse Mayús+F7 o seleccione Smart Step Into en el menú Run.

Para obtener más información, consulte Depuración en GoLand – Mejoras en 2019.1.

Nuevas refactorizaciones

Extract Interface

La refactorización Extract Interface (Extraer interfaz) le ayuda a crear rápidamente una nueva interfaz basada en el tipo seleccionado. Para invocarla, haga una llamada a Refactor this, pulse Ctrl+Alt+Mayús+T en Windows/Linux o ^T en macOS y escoja Extract Interface en el menú emergente.

El cuadro de diálogo de refactorización muestra una lista de sugerencias de miembros para transferir a la nueva interfaz. Puede especificar un nombre para la interfaz y escoger el archivo, el directorio o el paquete en el que debe ubicarse. Si el tipo ya implementa alguna interfaz, se excluirá automáticamente de la lista.

Para obtener más información, consulte Refactorizaciones en GoLand — Extract Interface.

Cambio de nombre más eficaz

La refactorización Rename también ayuda a cambiar el nombre de los métodos declarados en las interfaces. Al cambiar de nombre dichos métodos, GoLand también le ofrecerá cambiar el nombre de sus implementaciones.

El panel Refactoring Preview en la ventana de la herramienta Find le permite ver a qué afectará la refactorización actual, con usos agrupados por tipo y ubicación en el proyecto.

Nilness Analyzer

Go nos ofrece muchas formas de que el valor cero sea útil, pero cuantas más oportunidades tengamos, más fácil es caer en la trampa.

Nuestro nuevo Nilness Analyzer es un conjunto de inspecciones de código diseñado para detectar inmediatamente si las variables tienen valores cero o no en varios casos, y le alertará sobre posibles errores y problemas.

Para obtener más información, consulte Nilness Analyzer Inspector.
