dotCover的新功能

该页面指导您完成最新的dotCover版本的更新。亮点包括代码覆盖范围突出显示的更多选项，连续测试以获得更快的单元测试反馈，对Visual Studio / JetBrains Rider和.NET Core的支持。

在Windows，macOS和Linux 2019.2上支持Mono

dotCover 2019.2在Windows，macOS和Linux上支持Mono 5.10及更高版本。要在这些操作系统上执行覆盖率分析，必须使用JetBrains Rider或dotCover控制台运行程序。

Linux和macOS 2019.2上的控制台运行器

现在，您不仅可以在Windows上，还可以在macOS和Linux上使用dotCover控制台运行程序。为了在新平台上更加方便，该工具接受Unix样式语法中的命令参数。例如，现在/TargetExecutable=MyApp.exe和--targetexecutable:MyApp.exe都有效。

此外，控制台运行程序还提供了两个新命令来简化.NET Core和Mono应用程序以及单元测试的覆盖范围分析： cover-dotnet和cover-mono 。

在macOS和Linux 2019.1上支持.NET Core

dotCover 2019.1不仅在Windows上而且在macOS和Linux上都完全支持.NET Core 2.0 – 3.0项目。要在这些操作系统上执行覆盖率分析，必须使用JetBrains Rider或dotCover控制台运行程序。

改进的覆盖范围过滤器2018.3

现在，dotCover选项中有两组过滤器：运行时过滤器（在coverage会话期间应用）和结果过滤器（在会话结束后应用于coverage结果）。

覆盖率结果过滤器按文件名获得一个附加的过滤器。

支持多个目标框架2018.3

现在，如果一个项目针对多个框架，那么“ 覆盖树”将分别显示每个框架的结果（每个框架将在树中显示为单独的节点）。代码突出显示还取决于在编辑器中选择的上下文。

Visual Studio 2019预览版1支持2018.3

dotCover 2018.3以及ReSharper Ultimate系列的其他产品现在可以安装到Visual Studio 2019 Preview 1中。

JetBrains Rider集成2018.2

除了Visual Studio，现在dotCover还是Windows上JetBrains Rider IDE的一部分。在此版本中，支持两个主要功能：单元测试的代码覆盖率分析和连续测试。

要了解更多信息，请参阅单元测试范围和连续测试。现在在骑士！

支持dotnet控制台工具2018.2

现在，dotCover命令行运行程序可以用作.NET命令行工具（dotnet.exe）的扩展。例如，您可以通过运行以下命令来运行单元测试的覆盖率分析：
dotnet dotcover测试

此外，“经典” dotCover.exe控制台运行程序也得到了改进。现在，您不应该在analyze命令和cover命令之间选择：始终使用cover 。覆盖率结果将仅取决于--ReportType参数：如果指定了该结果，则将获得某种类型的报告；否则，将保存常规的覆盖率快照。

要了解更多信息，请参阅> dotnetdotсovertest

在任何会议2018.1中进行连续测试

不再需要单独的连续测试会话！可以为任何单元测试会话打开连续测试模式。在dotCover 2017.3及更早版本中，您的连续测试范围受到单个连续测试会话的限制。现在，您可以根据需要拥有任意数量的连续测试范围。

此外，在dotCover 2017.3中，您无法选择在连续测试期间如何自动启动测试。构建或保存解决方案后，dotCover在覆盖范围内开始测试。在2018.1中，不再需要覆盖范围：您可以告诉dotCover仅运行新的和过时的测试（而无需覆盖它们）。如果您已经拥有最新的coverage快照，则可以节省大量时间。

要了解更多信息，请参见更好的连续测试等等！

单元测试覆盖范围窗口2018.1

现在，覆盖率信息显示在单独的“ 单元测试覆盖率”窗口中，而不是每个会话的“ 覆盖率”选项卡。该决定的积极效果是能够在一处查看所有单元测试会话的汇总覆盖率数据。

要了解更多信息，请参见更好的连续测试等等！

改进的控制台运行器工具2018.1

dotCover控制台运行程序具有覆盖任意托管进程和服务（analyst analyze-everything ，analyzer analyze-service ）的能力，并将get coverage snapshot命令发送到控制台工具的运行实例。这极大地简化了在IIS / IIS Express上运行的Web应用程序的覆盖范围。

要了解更多信息，请参阅分析Web应用程序的代码覆盖率 。

.NET Core 2017.3中对NUnit和xUnit的支持

现在，在连续测试模式下，完全支持针对.NET Core的NUnit和xUnit单元测试。

快速评估覆盖范围结果2017.3

Visual Studio编辑器右装订线中的“屏蔽”图标使您可以快速查看总覆盖率以及当前打开文件中的代码失败测试的次数。单击该图标可在未显示的语句中导航。

改进了“显示覆盖率测试” 2017.3

在2017.3之前，在编辑器中使用了Show Covering Tests （ Ctrl + Alt + KT ）之后，弹出窗口会显示一个涵盖测试列表，但没有包含有关这些测试覆盖代码的确切部分的信息：方法或特定语句。现在，弹出窗口会明确地告诉您这一点，甚至会在编辑器中突出显示覆盖的语句。

支持.NET Core MSTest单元测试2017.2

dotCover 2017.2为针对.NET Core的项目中的MSTest单元测试提供了全面的支持，包括：

  • 测试结果的覆盖范围和突出显示，
  • 合并覆盖率数据，
  • 显示覆盖代码中特定位置的测试。

代码覆盖率改进2017.2

dotCover 2017.2在代码覆盖率方面引入了多项改进：

  • 性能提高 。在2017.2之前，在大型解决方案上进行连续测试是一个真正的痛苦 ：虽然测试执行速度很快，但后续的覆盖率分析可能需要几分钟。dotCover 2017.2中解决了该问题：由于采用了新的快照格式，dotCover现在可以更快地处理覆盖率结果。
  • 现在不仅显示特定测试的覆盖率结果，而且还显示测试组，项目，类别等的覆盖率结果。
  • 现在，在查看特定单元测试会话的覆盖率结果时，您可以将它们与快照中的覆盖率数据合并。

异形标记（色盲模式） 2017.2

现在，dotCover的突出显示标记不仅可以用颜色而且可以用形状指示覆盖率和测试结果。

使用彩色背景2017.1突出显示

旧式的高光又回来了，比以前更好了！

由于新的突出显示样式（使用标记）可能并不适合所有用例，因此我们添加了一个选项，可以在标记和彩色背景之间切换或同时显示两者。旧的突出显示样式还支持更新的逻辑，并显示测试覆盖率和测试结果。
