OpenID认证模块

除了内置身份验证模块支持的OpenID提供程序之外，Hub还允许您为几乎所有OpenID提供程序启用身份验证。集线器仅支持使用OpenID 2.0的提供程序。

先决条件

在开始之前，请验证以下要求：

  • 您在集线器中具有“ 创建身份验证模块”和“ 更新身份验证模块”权限。

启用OpenID身份验证

要允许用户使用OpenID身份验证登录集线器，请启用OpenID身份验证模块。

  1. 从更多设置菜单中，选择验证模块 。
  2. 从“ 添加模块”下拉列表中，选择OpenID Provider 。
    • “ 添加模块”对话框打开。
  3. 在“ 添加模块”对话框中，输入以下设置的值：
    领域描述
    名称输入身份验证模块的名称。
    服务器网址输入OpenID提供程序的服务器地址。提供程序定义URL所需的格式。
    有关详细信息，请参阅OpenID提供程序的文档。
  4. 单击创建模块按钮。
    • OpenID身份验证模块已启用。
    • “ 验证模块”页面显示OpenID验证模块的设置。有关此页面上设置的更多信息，请参阅OpenID身份验证模块 。
    • 按钮图像设置中存储的图标将添加到登录对话框窗口中。用户可以单击此图标以使用其OpenID凭据进行身份验证。

设定值

将模块添加到集线器后，将自动配置设置。您不必更改这些设置。要配置用于定义Hub如何使用OpenID凭据对待新用户帐户的选项，请参阅OpenID身份验证模块 。

领域描述
类型显示在集线器中启用了第三方身份验证的应用程序或服务的名称。
名称存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。
认证方式显示模块的当前状态。此状态指示模块当前是启用还是禁用。
审计链接到中心中的“ 审核事件”页面。在这里，您可以查看已应用于此身份验证模块的更改列表。
服务器网址将用于验证登录请求的OpenID提供程序的URL存储在Hub中。

架构图

使用以下设置将使用OpenID提供程序存储的用户数据映射到集线器中的用户帐户。

领域描述
电子邮件存储包含电子邮件地址的位置。
名字存储包含用户名字的位置。
姓存储包含用户姓氏的位置。
全名存储包含用户全名的位置。
头像存储包含用户头像的位置。

用户帐号管理

用户首次使用OpenID帐户登录集线器时，集线器会检查是否存在具有相同电子邮件地址的帐户。如果存在具有相同电子邮件地址的帐户，则会将OpenID凭据添加到现有帐户。

使用以下设置来配置集线器如何管理用户帐户。

领域描述
建立使用者允许为使用OpenID凭据登录的未注册用户创建中心帐户。集线器使用电子邮件地址来确定用户是否具有现有帐户。如果禁用此选项，则只有已经拥有与电子邮件匹配的帐户的用户才能登录到集线器。
自动加入群组使用OpenID凭据登录时将用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。
