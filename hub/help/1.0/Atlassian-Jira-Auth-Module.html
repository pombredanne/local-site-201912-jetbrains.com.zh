Atlassian JIRA Auth模块

该身份验证模块使用户可以使用其JIRA帐户登录到Hub。

  • 先决条件
  • 启用JIRA身份验证
  • 测试连接
  • 设定值
  • 用户帐号管理

先决条件

在开始之前，请验证以下要求：

  • 您在集线器中具有“ 创建身份验证模块”和“ 更新身份验证模块”权限。

启用JIRA身份验证

要允许用户使用其JIRA帐户登录集线器，请启用JIRA身份验证模块。

  1. 从更多设置菜单中，选择验证模块 。
  2. 从“ 添加模块”下拉列表中，选择Atlassian Jira 。
    • “ 添加模块”对话框打开。
  3. 在“ 添加模块”对话框中，输入以下设置的值：
    领域描述
    名称输入身份验证模块的名称。
    服务器网址输入JIRA实例的服务器地址。例如， http://mycompany.com/jira 。
    • JIRA认证模块是自动创建，启用和配置的。
    • “身份验证模块”页面显示JIRA身份验证模块的设置。有关此页面上设置的更多信息，请参阅Atlassian JIRA Auth Module 。
    • 用户可以使用其JIRA凭据登录到Hub。

测试连接

要验证身份验证模块已连接到JIRA实例，请测试连接。

  1. 单击测试登录按钮。
  2. 在“ 测试身份验证”对话框中，输入存储在JIRA实例中的用户的凭据：
    • 在登录字段中，输入domainusername 。
    • 在密码字段中，输入password 。
  3. 单击测试登录按钮。
    • 集线器在JIRA实例中搜索指定的用户帐户。如果找到用户，则显示成功通知。如果出现错误，请检查用户凭据和服务器URL。

设定值

将模块添加到集线器后，将自动配置设置。您不必更改这些设置。要配置用于定义Hub如何使用JIRA凭据对待新用户帐户的选项，请参阅Atlassian JIRA Auth Module 。

领域描述
类型显示在集线器中启用了第三方身份验证的应用程序或服务的名称。
名称存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。
认证方式显示模块的当前状态。此状态指示模块当前是启用还是禁用。
审计链接到中心中的“ 审核事件”页面。在这里，您可以查看应用于此身份验证模块的更改列表。
服务器网址将用于验证登录请求的URL存储在Hub中。
更改密码URI存储用于管理JIRA帐户的更改密码请求的页面的URL。

用户帐号管理

用户首次使用JIRA帐户登录集线器时，集线器会检查是否存在具有相同电子邮件地址的帐户。如果存在具有相同电子邮件地址的帐户，则JIRA凭据将添加到现有帐户中。

使用以下设置来配置集线器如何管理用户帐户。

领域描述
建立使用者为使用JIRA凭据登录的未注册用户创建中心帐户。集线器使用电子邮件地址来确定用户是否具有现有帐户。如果禁用此选项，则只有已经拥有与电子邮件匹配的帐户的用户才能登录到集线器。
自动加入群组使用JIRA凭据登录时将用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。
