创建用户组

除了默认用户组，您还可以根据需要为Hub实例创建任意数量的用户组。

要创建一个新组：

  1. 在管理区域中，单击组 。
  2. 点击工具栏中的+新用户组...按钮。

  3. 在“ 添加组”对话框中，输入名称并选择该组所属的项目。开始输入项目名称以缩小下拉列表。

    请注意这一步！您必须将一个组添加到项目中以便以后使用，例如，将其用作项目中的一组受理人。请仔细阅读以下说明。

  4. 单击创建用户组 。

将创建新组。您将被自动导航到新组的配置文件，在其中您可以配置其参数。

阅读配置用户组设置部分以获取更多详细信息。

为什么需要将组添加到项目

在Hub中，组是项目的资源 。除非组属于项目，否则项目管理员即使拥有足够的权限也无法查看或编辑该组。

例如，假设我们在Hub中有Project A和ProjectB。

用户'John Johnson'在项目A中被授予项目管理员角色。尤其是，他被授予以下权限：

  1. 与项目相关：阅读项目并更新项目
  2. 与组相关：读取用户组，创建用户组，更新用户组

请注意，这些权限仅在项目A中被授予。因此，当“ John Johnson”打开“管理”>“项目”列表时，他将仅看到项目A，而不看到项目B。

默认情况下，我们有“新用户”组，它属于“全球”项目。全局项目是一个虚拟项目，它以“所有用户”组包含系统中所有帐户的相同方式将所有项目合并到系统中。因此，“新用户”组属于Hub实例中的所有项目。

此外，我们为项目的开发人员创建了两个用户组：“团队A”和“团队B”。仅为项目A设置了团队A，为项目B设置了团队B。

因此，要能够查看和更新“团队A”组，该组必须属于项目A（或属于Global项目，因此也属于项目A），否则John Johnson甚至都不会看到“团队A”可供他使用的群组列表。

在我们的案例中，项目A的项目管理员将只能查看和更新或使用（例如，设置为一组受让人）“团队A”和“新用户”组。
