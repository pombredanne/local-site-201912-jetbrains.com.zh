Google身份验证模块

通过此身份验证模块，用户可以使用在Google中管理的电子邮件地址和密码登录集线器。

在集线器中启用Google身份验证时：

  • 您的用户使用其Google帐户使用的凭据登录到Hub。
  • 集线器用户需要记住的帐户和密码更少。
  • 具有Google帐户的新用户可以在Hub中创建自己的帐户。

启用Google身份验证

要允许具有现有Google帐户的用户登录集线器，请启用Google身份验证模块。

此过程分为三个步骤：

  1. 在Hub中生成重定向URI 。当您为Google创建身份验证模块时，集线器会生成一个重定向URI，以与此服务一起使用。该URI标识了每个登录到Google的请求的来源。
  2. 在Google Developers Console中生成一个客户端ID和密钥 。从集线器发送到Google的每个登录请求都包含一个唯一标识符。您存储在身份验证模块中的ID和机密会告诉Google每个登录请求都是经过授权的。
  3. 在Hub中启用身份验证模块 。生成Hub用来向Google进行身份验证的信息后，将值复制到Hub并启用该模块。

在此过程中，您将在Hub和Google Developers Console中生成值。
每个步骤的标题告诉您要遵循的应用程序菜单。

先决条件

在开始之前，请验证以下要求：

  • 您在集线器中具有“ 创建身份验证模块”和“ 更新身份验证模块”权限。
  • 您拥有任何Google产品或服务的有效帐户。

在Hub中生成重定向URI

如果您的Hub实例的基本URL发生更改，请确保更新Google设置中的Redirect URI。例如，将数据迁移到另一个集线器服务或更改代理设置之后。

  1. 从更多设置菜单中，选择验证模块 。
  2. 从添加模块下拉列表中，选择Google 。
    • 验证模块页面显示新的Google验证模块的设置。
    • 集线器会生成一个重定向URI，供您在Google中使用。
  3. 按照页面上的说明复制重定向URI。
  4. 点击链接以访问Google Developers Console。

    此设置要求您将值从Google Developers Console复制到集线器此页面上的输入字段中。
    为了简化设置，请在新的浏览器选项卡或窗口中打开此链接。

在Google Developers Console中生成客户端ID和机密

  1. 登录到您的Google帐户。
  2. 选择或创建一个项目。
  3. 打开菜单，然后选择API Manager 。
  4. 点击左侧的凭据 。
  5. 单击“ 新凭据” ，然后选择“ OAuth客户端ID” 。
    • 将打开“ 创建客户端ID”页面。
  6. 对于“ 应用程序类型” ，选择“ Web应用程序” 。
    • 显示了用于定义客户端ID的其他输入字段。
  7. 在“ 授权的重定向UIR”字段中，粘贴从“集线器”中的“ 身份验证模块”页面复制的重定向URI。
  8. 单击创建按钮。
    • Google会生成设置集线器模块所需的凭据，并将其显示在弹出窗口中。

在集线器中启用身份验证模块

  1. 从Google复制客户端ID，然后将其粘贴到Hub中的“ 客户端ID”输入字段中。
  2. 从Google复制客户端机密，然后将其粘贴到Hub的“ 客户端机密”输入字段中。
  3. 配置身份验证模块的可选设置。有关更多信息，请参阅设置 。
  4. 单击启用模块按钮。
    • Google身份验证模块已启用。
    • 按钮图像设置中存储的图标将添加到登录对话框窗口中。用户可以单击此图标以使用其Google帐户进行身份验证。

设定值

领域描述
类型显示在集线器中启用了第三方身份验证的应用程序或服务的名称。
名称存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。
认证方式显示模块的当前状态。此状态指示模块当前是启用还是禁用。
审计链接到中心中的“ 审核事件”页面。在这里，您可以查看应用于此身份验证模块的更改列表。
服务器网址当用户使用Google帐户登录时，显示集线器向其发送登录请求的服务器的URL。
此字段下方显示的信息可帮助您配置身份验证模块。
客户编号存储Google用于验证登录请求的标识符。当您配置Web应用程序的授权设置并输入授权的重定向URI时，便会在Google Developers Console中生成此值。
客户机密存储用于验证客户端ID的机密或密码。您会在Google Developers Console中生成此值以及客户端ID。
按钮图片显示用于用户单击以使用Google帐户登录集线器的按钮的图像。
建立使用者允许为使用Google帐户登录的未注册用户创建中心帐户。集线器使用电子邮件地址来确定用户是否具有现有帐户。
限制网域将用户帐户的创建限制为具有来自指定域的电子邮件地址的用户。仅当启用“ 创建用户”选项时，此选项才处于活动状态。
自动加入群组使用他们的Google帐户登录时将用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。
