GitHub身份验证模块

该身份验证模块使用户可以使用其GitHub凭据登录到Hub。

启用GitHub身份验证

要允许具有现有GitHub帐户的用户登录到Hub，请启用GitHub身份验证模块。

此过程分为三个步骤：

  1. 在Hub中生成一个回调URL 。当您为GitHub创建身份验证模块时，Hub会生成用于此服务的回调URL。该URL标识每个GitHub登录请求的源。
  2. 在GitHub中生成客户端ID和密钥 。从Hub发送到GitHub的每个登录请求都包含一个唯一的标识符。您存储在身份验证模块中的ID和密码会告诉GitHub每个登录请求均已授权。
  3. 在Hub中启用身份验证模块 。生成Hub用来向GitHub进行身份验证的信息后，将值复制到Hub并启用该模块。

在此过程中，您将在Hub和GitHub中生成值。
每个步骤的标题告诉您要遵循的应用程序菜单。

先决条件

在开始之前，请验证以下要求：

  • 您在集线器中具有“ 创建身份验证模块”和“ 更新身份验证模块”权限。
  • 您有一个已注册的GitHub OAuth应用程序。

在集线器中生成回调URL

如果您的Hub实例的基本URL发生更改，请确保在GitHub设置中更新回调URL。例如，将数据迁移到另一个集线器服务或更改代理设置之后。

  1. 从更多设置菜单中，选择验证模块 。
  2. 从“ 添加模块”下拉列表中，选择GitHub 。
    • Auth Modules页面显示新GitHub身份验证模块的设置。
    • Hub会生成一个回调URL，供您在GitHub中使用。
  3. 按照页面上的说明复制回调URL。
  4. 单击链接以访问GitHub中的“ 新建OAth应用程序”页面。

    此设置要求您将值从GitHub复制到Hub上此页面上的输入字段中。
    为了简化设置，请在新的浏览器选项卡或窗口中打开此链接。

在GitHub中生成客户端ID和密钥

  1. 登录到您的GitHub帐户。
  2. 指定用于注册新OAuth应用程序的常规参数： 名称 ， 主页URL和可选的应用程序描述 。
  3. 在“ 授权”回调URL中 ，粘贴您从“集线器”中“ 身份验证模块”页面复制的回调URL。
  4. 单击注册应用程序按钮。
    • GitHub会生成设置Hub模块所需的凭据，并将其显示在页面顶部。

如果您已经有任何已注册的GitHub应用程序... ...并且不用于连接其他服务，则可以使用其客户端ID和客户端密码：
在边栏中，单击“ 应用程序” ，然后在“ 开发人员应用程序”列表中选择该应用程序 。

在集线器中启用身份验证模块

  1. 从GitHub复制客户端ID，并将其粘贴到Hub的“ 客户端ID”输入字段中。
  2. 从GitHub复制客户端密钥，并将其粘贴到Hub的“ 客户端密钥”输入字段中。
  3. 配置身份验证模块的可选设置。有关更多信息，请参阅设置 。
  4. 单击启用模块按钮。
    • GitHub身份验证模块已启用。
    • 按钮图像设置中存储的图标将添加到登录对话框窗口中。用户可以单击此图标以使用其GitHub帐户进行身份验证。

设定值

领域描述
类型显示在集线器中启用了第三方身份验证的应用程序或服务的名称。
名称存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。
认证方式显示模块的当前状态。此状态指示模块当前是启用还是禁用。
审计链接到中心中的“ 审核事件”页面。在这里，您可以查看应用于此身份验证模块的更改列表。
服务器网址当用户使用GitHub帐户登录时，显示集线器向其发送登录请求的服务器的URL。
此字段下方显示的信息可帮助您配置身份验证模块。
客户编号存储GitHub用于验证登录请求的标识符。当您为应用程序配置授权设置并输入回调URL时，便会在GitHub中生成此值。
客户机密存储用于验证客户端ID的机密或密码。您将在GitHub中生成此值以及客户端ID。
按钮图片显示用于用户单击以使用GitHub帐户登录到Hub的按钮的图像。
建立使用者允许为使用GitHub帐户登录的未注册用户创建中心帐户。集线器使用电子邮件地址来确定用户是否具有现有帐户。
自动加入群组使用GitHub帐户登录时将用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。
