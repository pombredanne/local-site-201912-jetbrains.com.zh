云整合

亚马逊EC2

TeamCity通过将构建代理放置在Amazon EC2上来利用云计算的优势。

TeamCity在其构建计划中考虑虚拟机，并根据加载和构建队列状态自动挂起/恢复虚拟机。

2019.1利用AWS Spot Fleet请求以更高级，更具成本效益的方式在AWS上运行构建。

Microsoft Azure 插件

使用Microsoft Azure云平台？ 将其与TeamCity 集成，并在需要时扩展构建场。不要再担心在发行期间预测构建代理的负载。当您需要更多功能时，TeamCity将自动在Azure上启动代理，并在不再需要它们时将其停止。Azure Artifacts Storage插件提供了使用Azure Blob存储来保留外部构建工件的功能。

Google Cloud 插件

TeamCity提供了与Google Cloud服务的紧密集成。Google Cloud Agents插件允许使用Google Compute Engine按需启动云实例以扩展云构建代理池，还支持使用具有成本效益的可抢占虚拟机。Google Artifact Storage插件提供了使用Google Cloud Storage保留外部构建工件的功能。

VMware vSphere

将TeamCity与VMware vSphere集成，并使用安装在VMware虚拟机上的远程TeamCity代理更有效地处理构建：TeamCity分析构建队列，并使用与已排队的构建匹配的TeamCity代理启动虚拟机。

Kubernetes 插件

Kubernetes是运行Docker容器的一种非常流行的方式。许多团队和组织在生产中配置并使用了Kubernetes集群。使用JetBrains提供的官方TeamCity Kubernetes支持插件，以使用相同的基础架构运行TeamCity构建代理。
