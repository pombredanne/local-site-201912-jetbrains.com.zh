Prise en charge C# améliorée

Profitez de toutes les fonctionnalités de C# 8 grâce à ReSharper :

  • Prise en charge des dernières modifications dans indices et ranges, comme les types indexables et classables.
  • L'action contextuelle Add range check prend désormais en charge l'indexation selon System.Index/System.Range et produit des vérifications correctes.
  • Prise en charge des structures génériques non gérées.
  • Deux nouvelles inspections, « Convert switch statement to switch expression » et « Convert chain of ternary expressions to switch expression », avec un correctif rapide « Convert to switch expression » correspondant.

C'était l'une de vos demandes et nous y avons repondu : l'action contextuelle Convert concatenation to interpolation peut désormais s'appliquer au périmètre.

Démarrage plus rapide

En nous inspirant de Rider, nous avons commencé à utiliser le modèle de projet basé sur MSBuild au lieu de l'API COM de Visual Studio. Cela nous permet d'optimiser les temps de chargement de la solution, car nous avons réduit au strict minimum l'utilisation de l'API COM, qui verrouille le thread de l'interface.

D'autre part, nous avons mis à jour la page Performance Guide. Elle comporte une nouvelle section Windows Defender et des descriptions pour tous les conseils de performance qui permettent de savoir exactement ce qui va se produire une fois la suggestion appliquée.

Navigations et Find Usages

Dans cette version, Find Usages commence à utiliser les données de Solution Wide Error Analysis pour collecter les résultats. Si vous activez SWEA pour votre solution, Find Usages affiche les résultats beaucoup plus rapidement.

Nous avons ajouté deux petites améliorations à la navigation et nous espérons que vous les trouverez utiles : 

  • Vous pouvez faire des recherches dans un fichier directement depuis la boîte de dialogue « Go to File Member ». La liste des résultats inclut les noms de tous les membres restants et de tous les endroits où cette chaîne de recherche s'affiche dans le fichier.
  • La fenêtre de résultats Go To Text permet désormais de filtrer les lignes de commentaires si vous ne souhaitez pas rechercher les occurrences dans des chaînes de commentaires.

Débogueur Data Tips

ReSharper 2019.2 apporte plusieurs mises à jour au débogueur data tips :

  • Nous avons ajouté deux nouveaux choix : implicit function evaluation et auto evaluate property.

Plus de fichiers pour Go to Text

Le moteur derrière Go to Text prend en compte littéralement tous les fichiers texte d'une solution.

Auparavant, Go to Text ne tenait compte que des fichiers déjà indexés par ReSharper. ReSharper n'indexe que les fichiers qui contiennent du code ou qui peuvent vous aider à écrire du code, car l'exécution d'une indexation complète des autres fichiers se révèle onéreuse dans la plupart des cas. Ainsi, tous les fichiers aux extensions "inconnues" étaient donc complètement exclus, ce qui signifie que vous ne pouviez donc pas lancer de recherche dans des fichiers autres que de code.

Nous avons boosté le moteur ! Désormais, pendant l'affichage des résultats des fichiers indexés, le moteur commence déjà à chercher dans les fichiers texte non indexés en arrière-plan. C'est très utile si vous stockez des données dans des fichiers comportant des extensions diverses, comme *proj, .yml et .csv.

Optimiser les références pour les projets SDK

ReSharper contrôle désormais également les références de paquets inutilisées dans les projets SDK. Utilisez “Optimize References” dans le menu contextuel d'un projet pour analyser et optimiser ces références. Si le projet ne contient aucun code qui utilise une référence donnée, cette dernière s'affichera comme inutilisée. Désinstallez facilement les paquets inutilisés pour préserver la propreté de l'arborescence de votre projet !

Pour en savoir plus, reportez-vous à Exporter le style de code EditorConfig, Optimiser les références pour les projets SDK, et plus d'informations sur les mises à jour de ReSharper 2019.1.

Améliorations de la navigation et de la recherche

Go to Everything et les autres actions de navigations sont à présent en mesure de renvoyer des éléments contenant des mots de la requête cible, peu importe l'ordre. Par exemple, une recherche sur exactMatching renverra isMatchingExactly.

D'un autre côté, vous avez maintenant accès à la recherche exacte qui fonctionne comme se doit de fonctionner un moteur de recherche : rechercher "Collection" renverra Collection, mais ne renverra pas FilteredCollection, IterateCollection(), etc. Cependant, quand vous utilisez la recherche exacte pour filtrer les noms composés, vous pouvez toujours utiliser les symboles de remplacement * et ? pour autoriser les variations dont vous avez besoin.

Sinon, Go to text est à présent intégré à Go to Everything (CtrlT) : si aucun code ne correspond à vos critères de recherche, ReSharper va à la place chercher du texte.

De plus, vous pouvez maintenant rechercher des méthodes d'extension en utilisant un point séparateur : Class.ExtensionMethod.

Enfin, ReSharper 2017.2 propose un nouveau raccourci pour explorer les résultats de recherche dans la fenêtre Find Results. Appuyez sur MajEntrée à la place du + du pavé numérique.
