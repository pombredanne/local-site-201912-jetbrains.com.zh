Quoi de neuf dans GoLand

GoLand 2019.3 consomme moins de puissance CPU pour de meilleures performances, améliore la prise en charge de ses modules Go, ajoute un lot de nouveaux correctifs rapides et optimise les refactorisations. Il est possible d'appeler une méthode pour un champ imbriqué à l'aide de Evaluate Expression et de déboguer vos applications à distance grâce à Mozilla rr.

Vous pouvez également activer un observateur de fichier global pour tous les nouveaux projets, travailler avec MongoDB, et bien plus !

2019.3 Nov 29

Performances

Depuis la sortie de GoLand 2019.2 cet été, nous avons travaillé dur pour en accélérer les performances et réduire sa consommation CPU. L'une des principales améliorations de performance a permis de réduire la durée du démarrage, pendant laquelle l'EDI prépare tous ses composants en coulisse, affiche l'écran d'accueil et indexe un projet.

Notez bien que les résultats peuvent varier en fonction de la taille de votre projet ainsi que de l'âge, la mémoire RAM et l'espace disque de votre équipement. Nous allons poursuivre notre travail sur ce front. Vous pouvez vous attendre à un démarrage encore plus rapide dans les futures versions.

Améliorations des modules Go

GoLand 2019.3 permet la mise en évidence des éléments syntaxiques et la navigation dans les fichiers go.mod. Appuyez sur ⌘+B sous macOS ou Ctrl+B sous Windows/Linux sur un chemin d'importation de dépendances spécifiques pour que GoLand vous en affiche les sources dans la vue Project View.

Toujours dans Project View, les modules Go s'affichent avec leur chemin d'importation complet au lieu du dernier segment. La version de chaque module Go sera aussi affichée avec une couleur de police plus atténuée que le chemin d'importation pour permettre de mieux les distinguer.

Mises à jour du débogueur

Appeler une méthode d'un champ imbriqué

Evaluate Expression dans le débogueur vous permet à présent d'appeler directement une méthode dans un champ imbriqué.

Débogage à distance avec Mozilla rr

Beaucoup de services de production écrits en Go s'exécutent sous Linux. Mozilla rr peut aider à enquêter sur les bogues difficiles à reproduire. La prise en charge du débogage dans Mozilla rr remonte à GoLand 2018.3, mais son fonctionnement se limitait jusqu'ici aux machines de développeurs sous Linux. Il est maintenant possible de lancer un serveur de débogage sur une machine Linux et de déboguer la trace à distance depuis macOS ou Windows. Suivez cette procédure :

  • Collectez une trace rr sur la machine distante.
  • Lancez le débogueur sur la machine distante comme ceci : dlv --headless --api-version 2 -l localhost:2345 replay /path/to/trace/dir /path/to/binary.
  • Connectez-vous à la machine distance à l'aide de la configuration Go Remote.

Veuillez noter que Mozilla rr n'est disponible que sur les machines Linux.

Nouveaux correctifs rapides

Gestion des paramètres inutilisés

GoLand met désormais en évidence le nom et le type des paramètres inutilisés s'ils peuvent tous deux être supprimés, ou uniquement leur nom si le type est utilisé pour implémenter la spécification d'une méthode.

Dans ce dernier cas, l'EDI vous propose deux correctifs rapides : Delete parameter names (supprimer les noms de paramètres) ou Rename to_ (renommer en). Le même comportement est fourni pour les fonctions utilisées comme type de fonction

Il est de plus possible à présent de supprimer l'ensemble des paramètres inutilisés d'un coup. Appuyez sur Alt+Enter et utilisez le correctif rapide Delete all unused parameters.

Créer la méthode ou la fonction manquante

GoLand propose un correctif rapide pour créer une fonction ou méthode manquante lorsqu'elle est utilisée dans un type de fonction. Il vous suffit d'appuyer sur Alt+Entrée et de sélectionner Create method/function.

Gestion améliorée des alias de packages

GoLand suggère maintenant d'importer les packages en fonction des alias utilisés via Alt+Enter dans tout fichier de votre projet.

Refactorisations

Signature de changement

Change Signature affiche maintenant toujours le type des paramètres dans les onglets Parameters et Result parameters. Cochez la nouvelle case Reuse types pour réduire l'affichage des types induits dans Signature Preview et le résultat.

Vous trouverez en outre deux nouvelles actions d'intention, Reuse signature types qui fusionne les définitions de paramètres en une seule déclaration, et Expand signature types qui ajoute des types à chaque paramètre à l'aide du raccourci Alt+Entrée.

Extract Variable et Rename

Extract Variable et Rename offrent maintenant de meilleures suggestions de noms. Par exemple, err est suggéré pour les expressions de type error, ctx est suggéré pour context.Context, etc.

Amélioration du comportement des inspections de code

Les inspections de code de GoLand sont désormais plus ciblées. Nous avons réduit la portée de la mise en évidence des problèmes détectés dans l'éditeur de code pour améliorer la pertinence de l'affichage des correctifs rapides et des actions d'intention pour un morceau de code donné lorsque vous invoquez le menu contextuel à l'aide du raccourci Alt+Entrée.

Édition du code

Amélioration de l'analyseur lexical des littéraux Rune

Nous avons amélioré l'analyseur lexical des littéraux de type rune de plusieurs façons :

  • GoLand fournit un message d'erreur plus clair et plus concis pour expliquer pourquoi l'expression n'est pas correcte.
  • GoLand ne signale pas un littéral de rune valide en tant qu'erreur après sa modification.
  • L'analyse des littéraux Rune non valides est plus stable.

Mise en évidence des éléments syntaxiques dans les réaffectations

Go permet d'utiliser des variables existantes sur le côté gauche des déclarations var courtes. Par conséquent, deux constructions similaires peuvent avoir des significations différentes. Pour détecter ces cas plus facilement, nous avons ajouté une mise en évidence de la syntaxe pour les réaffectations de ce genre.

Pliage du code pour les littéraux composites

GoLand 2019.3 permet le pliage de code pour les littéraux composites. Utilisez les commutateurs de pliage de code pour réduire ou étendre le fragment de code, ou appuyez sur Ctrl+Moins (pavé num.)/Ctrl+Plus (pavé num.) sous Windows ou ⌘+Moins (pavé num.)/⌘+Plus (pavé num.) sous macOS. Pour configurer le comportement de pliage du code par défaut, accédez à Settings/Preferences | Editor | General | Code Folding.

Amélioration des modèles dynamiques

Les modèles dynamiques for et forr suggèrent maintenant des noms de variables pertinents. Par exemple, des noms comme i et index sont suggérés pour un index; s, string, et line sont suggérés pour une valeur.

Le live template meth fournit des suggestions de code pour les noms de récepteurs.

Navigation

Recherche limitée à une zone sélectionnée

Limitez votre recherche à une zone sélectionnée lorsque vous utilisez Find (⌘+F sous macOS et Ctrl+F sous Windows/Linux). Sélectionnez simplement le morceau de code dont vous avez besoin dans l'éditeur et invoquez Find. L'EDI lancera alors la recherche dans la zone sélectionnée uniquement. Pour basculer facilement vers une recherche dans le fichier complet, appuyez à nouveau sur ⌘+F/ Ctrl+F, ou décochez l'option In Selection dans la barre d'outils.

Utilisation de 'Find in Path' avec une option de masque de fichiers

Désormais, lorsque vous lancez une recherche dans la boîte de dialogue Find in Path à l'aide d'un masque de fichiers, GoLand affiche les extensions de fichiers dans les résultats de recherche en fonction de ces paramètres. Cela permet de distinguer plus facilement les fichiers ayant le même nom mais des extensions différentes.

Activer un observateur de fichiers global pour tous les nouveaux projets

Dans GoLand 2018.2, nous avions ajouté une option pour partager une configuration d'observateur de fichiers entre plusieurs projets à l'aide des options Project et Global – elle s'appelle Global File Watcher.

Toutefois, vous aviez toujours besoin d'activer cet observateur dans chaque nouveau projet. Il est désormais possible d'activer ce watcher par défaut dans tous les nouveaux projets que vous créez ou ouvrez.

Dans le menu File, sélectionnez Other Settings | Settings for new projects, puis accédez à Tools | File Watchers, ajoutez un nouvel observateur global, puis cochez la case correspondante pour l'activer dans tous les nouveaux projets.

Didacticiels

Souvent, le temps manque pour lire toutes les pages de nouveautés, de notes de version et les articles de blog sur le produit que vous utilisez, alors même que vous avez besoin de finir votre projet dans les temps. C'est pourquoi nous avons ajouté des Didacticiels pour que vous puissiez découvrir les fonctionnalités de GoLand quand vous en aurez le temps.

Vous pouvez trouver des Tutoriels sur l’écran d'accueil, au-dessus de votre liste de projets. Lorsque vous mettrez l'EDI à niveau vers une nouvelle version, vous verrez une leçon abordant les fonctionnalités de la nouvelle version. Vous pouvez également accéder à Tutorials à l'aide de la fenêtre d'outils correspondante depuis n'importe quel projet.

Autres modifications

Fenêtre d'outils Hierarchy

Une nouvelle option permet d'analyser la hiérarchie des méthodes des appelés dans la fenêtre d'outils Hierarchy. Nous avons ajouté deux boutons dédiés qui vous permettent de basculer entre les appelants et les appelés de la méthode sélectionnée. En outre, vous pouvez trier les éléments d'une arborescence par ordre alphabétique, en cliquant sur un bouton spécial dans le menu. Pour afficher la fenêtre d'outils Hierarchy dans la liste Tool Windows, vous avez d'abord besoin de construire une hiérarchie. Accédez ensuite à Navigate | Call Hierarchy or Navigate | Type Hierarchy.

Changez vos extensions de thème et de configuration clavier à la volée

Dans GoLand 2019.3, vous pouvez installer et désinstaller n'importe quelle extension de thème ou de configuration clavier sans redémarrer l'EDI.

Un raccourci personnalisé pour renvoyer le focus vers l'éditeur

Configurez votre raccourci personnalisé pour l'action qui ramène le focus sur l'éditeur, au lieu d'utiliser ESC. Cherchez l'action Focus Editor dans references/Settings | Keymap, effectuez un clic droit dessus, puis sélectionnez Add Keyboard Shortcut dans la liste déroulante.

Amélioration de la visibilité des barres de défilement

Nous avons ajouté une option pour améliorer la visibilité des barres de défilement. Pour ce faire, cochez la case Use contrast scrollbars dans Preferences/Settings | Appearance & Behavior | Appearance.

Améliorations du contrôle de version

Remaniement de la boîte de dialogue Clone

La boîte de dialogue Clone se trouve dans VCS | Get from Version control. Vous pouvez maintenant vous connecter depuis cette boîte de dialogue ou, si vous êtes déjà connecté, obtenir un aperçu instantané des listes de tous les dépôts regroupés par compte ou par organisation.

Plus d'informations sur les conflits de fusion

Il est maintenant plus facile d'enquêter sur un conflit lors d'une opération de fusion, de changement de base ou de sélection précise. Il vous suffit de cliquer sur le lien Show Details pour obtenir la liste des commits qui mènent à l'état du code obtenu, et vous pouvez consulter plus d'informations sur la source des modifications dans la boîte de dialogue Merge.

Mises à jour des bases de données

Prise en charge de MongoDB

GoLand 2019.3 assure une prise en charge de base pour MongoDB. Vous pouvez maintenant :

  • Consultez les collections et les champs dans l'explorateur de base de données.
  • Ouvrez une collection ou observez le résultat de la requête. Il est également possible de trier par colonne ou de filtrer les valeurs.
  • Lancez des requêtes et obtenez-en les résultats. Sélectionnez une instruction et exécutez-la en appuyant sur Ctrl/⌘+Entrée ou à l'aide du bouton Play de la barre d'outils.

Rechercher les erreurs de connexion

Il est maintenant plus facile d'approfondir les erreurs de connexion. En cas d'échec de connexion, vous pouvez simplement cliquer sur le bouton dédié de l'EDI pour accéder aux résultats de recherche concernant votre erreur fournis par le moteur de recherche Google.

Développement web

Saisie automatique pour les bibliothèques CSS sur CDN

Si vous utilisez Bootstrap ou une autre bibliothèque CSS avec un lien provenant d'un CDN dans votre fichier HTML, vous pouvez désormais bénéficier de la saisie automatique pour les noms de classes de cette bibliothèque sans avoir à ajouter ses sources au projet.

Suggestions de noms pour les nouveaux symboles

Lorsque vous ajoutez de nouveaux noms de variables et de paramètres dans des fichiers JavaScript ou TypeScript, vous obtenez maintenant des suggestions de noms en fonction des noms de classes, d'interfaces et d'alias de types définis dans le projet, des bibliothèques standard et des dépendances que vous utilisez.

Pour les nouvelles variables, fonctions, classes et interfaces, vous recevrez également des suggestions pour les noms déjà utilisés dans la même portée mais pas encore définis.

2019.2 25 Juillet

Prise en charge des appels de fonction

Examinez en détail ce qui se passe dans vos expressions quel que soit le stade d'exécution de votre programme.

Remarque : cette fonction repose sur la prise en charge expérimentale des appels de fonction de Delve et de l'exécutable de Go.

Si vous subissez des bogues ou des problèmes en relation avec la prise en charge des appels de fonction, signalez-les sur notre système de suivi de tickets.

Amélioration des palettes de couleurs par défaut et Darcula

GoLand 2019.2 permet de varier le surlignage de la sémantique du code des profils de couleur par défaut et Darcula.

Cela vous aidera à mieux comprendre votre code et à suivre sa logique plus rapidement. Différenciez facilement les noms de paquets de ceux des variables, les constructions de code intégrées des vôtres, et les conversions de type des appels de fonctions. Les récepteurs de méthodes et les fonctions/appels de fonction ont désormais eux aussi leurs propres couleurs.

Comme d'habitude, laissez libre cours à vos envies de personnalisation dans la section Settings/Preferences | Editor | Color Scheme.

Mises à jour du débogueur

Plus d'informations sur les variables avec la méthode String()

Le débogueur affiche désormais les résultats des implémentations de méthode simples String(), Error() et DebugString() dans le volet Variables. Il prend en charge les méthodes avec une déclaration de retour simple, sans appels.

Présentation améliorée des types spéciaux

GoLand 2019.2 affiche la valeur de structure, dérivée depuis les types time.Time, time.Duration et net.IPsous une forme lisible par le développeur, dans le volet Variables du débogueur.

Format par défaut pour les entiers

Choisissez comment afficher les nombres entiers utilisés comme variables dans le débogueur par défaut, à savoir sous forme binaire, décimale ou hexadécimale. Définissez le format dans la section Settings/Preferences | Build, Execution, Deployment | Debugger | Data Views | Go.

Prise en charge des fonctionnalités Go 1.13

GoLand 2019.2 prend en charge les littéraux numériques et les entiers signés en tant que compteurs de décalage. Ces modifications se rapportent à la transition que nous opérons vers Go 2. Consultez le blog officiel de Go pour en savoir plus.

Le surlignage du code de GoLand vous prévient en cas d'erreur et affiche correctement la sortie dans la section Console.

Lisez cet article pour plus d'informations sur les nouvelles fonctionnalités de Go 1.13 et la façon dont GoLand 2019.2 les prend en charge.

Modèles de saisie automatique Postfix personnalisés

GoLand 2019.2 permet de créer vos propres modèles de saisie automatique Postfix personnalisés.

Accédez à Settings/Preferences | Editor | General | Postfix Completion, cliquez sur +, sélectionnez Go, définissez votre modèle et choisissez le périmètre d'application du modèle.

Pour en savoir plus sur les modèles personnalisés, consultez cet article.

Vous trouverez en outre un nouveau modèle prédéfini de saisie automatique Postfix : .aappend. Ce modèle ajoute en préfixe une expression avec un appel de fonction append et l'affecte au résultat de cette expression.

Correctifs rapides basés sur Change Signature

Nous avons ajouté de nombreux correctifs rapides basés sur la refactorisation Change Signature. Par exemple, en appuyant simplement sur Alt+Entrée, vous pouvez automatiquement induire et ajouter un paramètre à l'appel de fonction, ou ajouter les paramètres return manquants dans une signature.

Refactorisation Extract Method avec instructions return

La refactorisation Extract Method traite désormais le code avec des instructions return. Si le code se termine toujours par des instructions return, il est extrait tel quel. Cependant, s'il contient des chemins d'exécution sans instructions return, la méthode extraite aura un indicateur booléen en tant que valeur return. Cet indicateur est utilisé pour exécuter une opération return anticipée côté appel.

Améliorations de la saisie automatique

La saisie automatique inclut désormais des suggestions pour les membres récepteurs, même s'il n'y a pas de récepteur. Les noms de récepteur tels que this ou self sont fortement déconseillés. Par conséquent, les méthodes ont différents noms de récepteur. Cette fonctionnalité de saisie automatique du code vous permet d'ignorer la différence.

Modules Go plus intelligents et intégration dep

GoLand 2019.2 autorise l'intégration automatique des Modules Go/dep lorsqu'il détecte qu'un projet utilise les Modules Go Modules ou dep. Auparavant, il désactivait l'indexation GOPATH et affichait uniquement une notification d'activation de l'intégration Modules Go/dep. Nous avons constaté que ce n'était pas suffisant, car il était facile de rater ou d'ignorer la notification et, par conséquent, de placer un projet dans un état de semi-configuration.

Si cela ne fonctionne pas pour vous, vous pouvez désactiver l'intégration manuellement dans la section Settings/Preferences | Go | Go Modules (vgo) à tout moment.

Inspections de code

Conversions de type redondant

L'inspection de code Redundant type conversions permet d'obtenir un projet plus lisible et plus net. Elle signale les conversions de type redondantes qui peuvent être éliminées.

Validation des index de tranche sous forme de littéraux

Nous avons ajouté la validation des index de tranche dans les littéraux composites ! Cela signifie que l'éditeur prévient que l'index doit être une constante sous forme d'entier non négatif avant de compiler votre projet.

Édition du code

Prise en charge des séquences d'échappement

Prise en charge des séquences d'échappement dans les littéraux de chaîne incluant :

  • Surlignage du code.
  • Inspection signalant les problèmes potentiels.
  • Édition intelligente comprenant l'emplacement des séquences d'échappement.
  • Correction orthographique plus précise.

Les emplacements de mise en forme sont désormais surlignés dans une couleur différente, ce qui permet de les repérer plus facilement dans les chaînes longues. Vous pouvez personnaliser la couleur en cliquant sur Settings/Preferences | Editor | Color Scheme | Language Defaults | String | Escape Sequence | Valid. Ou, pour Go uniquement, cliquez sur Settings/Preferences | Editor | Color Scheme | Go | String | Valid escape.

Action Fill Fields

Il est désormais plus facile de remplir les champs des structs. Tout d'abord, vous pouvez choisir les champs à remplir dans une fenêtre dédiée. Ensuite, les valeurs ainsi générées peuvent à présent être modifiées. Enfin, cette action est désormais disponible depuis la saisie automatique du code.

Méthode Extend Selection améliorée

Extend Selection étend désormais le code de façon correcte du point de vue de la sémantique, d'une ligne à un bloc de code séparé par des lignes vides (au lieu d'avoir à sélectionner tout le corps de la méthode).

Améliorations de l'interface

Aspect remanié de la boîte de dialogue New | File

La boîte de dialogue New | File a été transformée en fenêtre contextuelle regroupant toutes les formes possibles du type de fichier à créer. Auparavant, il fallait développer la liste pour obtenir une solution de remplacement, mais désormais, toutes les options disponibles sont visibles immédiatement.

Page Plugins améliorée

À présent, GoLand affiche directement les informations relatives au plugin, sans avoir à ouvrir les pages de détail du plugin une par une. Nous avons ajouté davantage d'options sous l'icône d'engrenage, pour vous permettre de désactiver ou d'activer toutes les extensions téléchargées à la fois. Le bouton Update est désormais placé près du nom de plugin, dans l'onglet Installed. L'onglet Updates n'existe plus, car il est devenu superflu.

Fenêtre d'outils Find remaniée

Dans la fenêtre d'outils Find, nous avons regroupé les icônes les moins sollicitées et en avons supprimé d'autres.

Modifications générales de convivialité

Davantage d'options dans le sélecteur de portée

Les nouvelles portées, telles que Recently Changed et Recently Viewed Files, Files in Previous Search Result et Selection ont toutes été ajoutées au sélecteur Scope. Vous pouvez également sélectionner les portées de type Data Sources. D'autre part, nous avons ajouté la possibilité de sélectionner une portée pour la refactorisation Rename. Appuyez deux fois sur Maj+F6 pour appeler la fenêtre de refactorisation Rename et choisir la portée du changement de nom.

Nouvelle option permettant de contrôler le déplacement du caret

Le comportement par défaut de l'action Move Caret to Next Word de GoLand 2019.2 a changé : l'EDI déplace désormais le caret vers la fin du mot sélectionné. Cependant, vous pouvez le régler comme bon vous semble. Cliquez sur Settings/Preferences/Settings | Editor | General et sélectionnez l'emplacement où l'EDI doit placer le caret : à la fin du mot, au début du suivant, voire entre les deux.

Supplément d'informations dans la vue Project

Sélectionnez ou désactivez l'option In-place Descriptions dans le menu View pour obtenir un aperçu de la taille du fichier, ainsi que des modifications de la date et de l'heure dans la section Project View.

Améliorations du contrôle de version

Validation depuis la section Local Changes

GoLand 2019.2 permet de procéder à une validation directement depuis la section Local Changes.

Désormais, lorsque vous travaillez sur un commit, vous pouvez parcourir le code source, afficher l'historique d'un fichier spécifique, consulter un fichier diff correspondant à la zone entourant le commit ou utiliser les autres fonctionnalités de l'EDI.

Pour activer cette fonctionnalité et envoyer un commit depuis la section Local Changes sans passer par une boîte de dialogue, allez dans la section Settings/Preferences | Version Control | Commit Dialog et cochez la case Commit from the Local Changes without showing a dialog.

Améliorations concernant les fichiers .gitignore

GoLand surligne correctement tous les fichiers ignorés et fait des suggestions de saisie automatique pour les noms de fichier et de dossier. Vous pouvez maintenant ajouter rapidement un fichier à .gitignore en utilisant le menu contextuel Local Changes de la fenêtre d'outils VCS : cliquez dessus avec le bouton droit dans le groupe de fichiers Unversioned et sélectionnez Add to .gitignore.

Option Abort Merge

Nous avons ajouté l'option Abort Merge qui est disponible dans la fenêtre contextuelle Branches lorsqu'une opération de fusion est en cours. Voici l'avenir dont nombre d'entre nous ont rêvé : il n'est plus nécessaire de basculer vers le terminal pour annuler une action en cours.

Colonnes VCS Log personnalisables

Cliquez sur l’icône représentant un œil pour choisir les colonnes à afficher dans l'onglet Log, sélectionnez l'élément Show Columns dans le menu contextuel et choisissez les colonnes à afficher : Author, Date ou Hash.

Outils

Prise en charge des scripts shell

GoLand 2019.2 prend en charge les scripts shell, ce qui inclut la saisie automatique des mots et des chemins, un aperçu de la documentation rapide et même une fonction de renommage à base de texte.

Mais ce n'est pas tout, nous avons ajouté des intégrations avec Shellcheck et shfmt.

Prise en charge de Kustomize pour Kubernetes

Avec le plugin Kubernetes, GoLand 2019.2 assure la prise en charge de l'édition, la saisie automatique du code pour les clés et les chemins d'accès, de nombreuses inspections et la refactorisation Rename. Vous pouvez également naviguer depuis un chemin d'accès déclaré vers les paquets ou fichiers associés dans la vue de projet des fichiers de ressource destinés à Kustomize.

Navigateur pour le système de fichiers du conteneur Docker

Vous pouvez désormais afficher le système de fichiers du conteneur Docker en cours d'exécution. Le système de fichiers s'affiche dans l'onglet File du nœud Docker, dans la fenêtre d'outils Services.

Développement web

Intentions intelligentes pour JavaScript et TypeScript

Grâce à la nouvelle intention Propagate to destructuring declaration, vous pouvez remplacer une variable supplémentaire par une déstructuration sous certaines conditions. Pour supprimer complètement une déstructuration, utilisez l'action d'intention Replace destructuring with property or index access.

L'EDI vous avertit si l'expression booléenne d'une condition comporte des éléments superflus, et vous propose de la simplifier.

Refactorisation Rename améliorée pour JavaScript

Lorsque vous renommez un symbole dans un fichier JavaScript ou TypeScript, l'EDI regroupe maintenant les utilisations dynamiques et, par défaut, les exclut de la refactorisation. Cela permet de rendre la refactorisation plus précise et vous donne plus de contrôle sur ce qui doit précisément être renommé dans la fenêtre d'outils Refactoring Preview.

Prise en charge améliorée de Vue.js

Vous utilisez Vuetify ou BootstrapVue dans votre application Vue.js ? La saisie automatique du code pour les composants et leurs accessoires, notamment à partir de ces bibliothèques de composants Vue, est maintenant plus précise. Cela a été rendu possible par une nouvelle approche de la manipulation de ces bibliothèques dans l'EDI.

Profileurs de mémoire, de processeur, Mutex, et de bloc

GoLand a étendu sa prise en charge d'outils en intégrant des profileurs de mémoire, de processeur, Mutex et de bloc. Vous pouvez commencer à profiler votre application directement à partir de l'EDI, puis consulter les résultats de l'analyse dans une fenêtre d'outil dédiée (View | Tool Windows | Profiler).

Pour exécuter le profileur, cliquez sur le nouveau bouton Run with Profiler et choisissez la configuration nécessaire dans la barre d'outils de navigation. Vous pouvez également sélectionner l’une des nouvelles actions dans le menu d'exécution, qui s'affiche lorsque vous cliquez sur l’icône d'exécution située dans la gouttière de gauche.

Pour mieux comprendre les problèmes de performances potentiels dans votre code, triez les résultats en fonction des statistiques disponibles pour chaque type de profileur. Utilisez les onglets Flame Chart, Call Tree et Method List, voire accédez au code source pour examiner les problèmes.

Smart Step Into

Le débogage d’une chaîne d’appels et d’appels de méthodes imbriqués est beaucoup plus simple désormais grâce à la nouvelle action de débogage Smart Step Into. Elle vous permet d'intervenir dans des appels arbitraires dans une expression, en passant automatiquement au-dessus des méthodes qui ne vous intéressent pas.

Par exemple, lors du débogage d'un code comme fmt.Printf("Je m'appelle %s, j'ai %d! ans", me.FullName(), me.Age()), vous pouvez choisir de vous arrêter spécifiquement sur Age().

Pour invoquer Smart Step Into, appuyez sur Maj+F7 ou cliquez sur Smart Step Into dans le menu Run.

Pour en savoir plus, reportez-vous à la rubrique Le débogage dans GoLand - Améliorations de la version 2019.1.
