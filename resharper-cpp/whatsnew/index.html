ReSharper C ++ 2019.2

更快的启动时间

在此发行版中，我们已经完成了最后几个发行周期的工作，现在可以正式宣布ReSharper C ++中使用了新的预处理器实现。实际上，现在有两个，一个是较新的用于索引，另一个是旧的。这一新工具在LLVM和Unreal Engine等典型解决方案上的索引编制速度提高了15-25％。

代码完成

在代码完成中， ReSharper C ++为完成列表中的每个项目显示一个摘要工具提示（由ReSharper选项的Environment | IntelliSense | Completion Appearance页面中的Show summary复选框控制）。所选完成项目的说明现在已着色，并显示了相应项目的文档。

代码提示

编辑器具有更多代码提示，有助于提高代码的可读性。在预处理器指令之后以及由非const引用传递的函数参数中引入了新的提示，而聚合初始化表达式中的提示现在显示已初始化基类的名称。

更多C ++ 20支持

C ++ 20标准几乎随时可以由C ++委员会签署。当然，我们正在努力获得完全合规的支持。ReSharper C ++ 2019.2支持以下功能：

  • 新的char8_t类型，
  • 在lambda init-capture中扩展包，
  • 默认的可构造和可分配的无状态Lambda，
  • C ++ 20汇总规则，

  • C ++ 20对结构化绑定声明的改进。

调试步骤过滤器

步骤筛选器是ReSharper C ++提供的用于调试功能的最新功能。当您调用Step Into时 ，Visual Studio调试器通常将逐步进入从当前语句调用的所有函数。但是，有些功能是微不足道的或经过良好测试的，因此您不想涉足其中。这是步进过滤器的用处：使用此功能，您可以指定调试器应始终逐步执行的功能。

ReSharper C ++为标准库提供了预定义的步骤筛选器集合，但是您始终可以添加新的步骤筛选器：

  • 您可以在“ 工具” | “ 工具”上检查，配置，禁用或启用后退过滤器。 调试| C ++选项页，其中列出了所有可用的步骤筛选器。
  • 当您进入某个功能并决定将来要逐步使用该功能时，可以使用上下文操作将该功能快速添加到步骤筛选器列表中。对于模板函数，有单独的上下文操作可仅添加函数的当前实例或其所有实例。

要了解更多信息，请参阅ReSharper C ++中的“步骤筛选器” 。

包括分析仪

在大型现实C ++项目中，编译时间是最大的问题之一。 ReSharper C ++ 2018.1引入了Includes Analyzer ，这是一个新的代码检查工具，可帮助您衡量项目中哪些头文件在代码行方面对总编译工作量的贡献最大。若要使用它，请调用ReSharper |检查在...操作中分析包括，或从解决方案资源管理器的上下文菜单中选择分析包括 。 ReSharper C ++将分析给定范围内的文件，并在专用工具窗口中提供报告。

该报告列出了按文件及其可传递包含的文件的行数排序的文件。然后，您可以使用“ 包含”树视图（其中子节点与在父节点中包含该文件的文件相对应）或“ 包含者”树视图（其中子节点是所包含的文件）来挖掘文件之间的依赖性。使用这两个视图，您应该能够查明贡献最大的头文件，然后尝试摆脱任何不必要的#include指令。

Clang-Tidy整合

Clang-Tidy是基于流行的clang编译器的功能强大的开源代码分析工具。它带有大量的代码检查和随附的修复程序。ReSharper C ++ 2017.3提供了与Clang-Tidy的无缝集成，这意味着它在编辑代码时仅在后台运行，检查结果在代码编辑器中即时显示，并带有内置检查。ReSharper C ++还允许您单独应用Clang-Tidy提供的修补程序，作用域操作或代码清除。

除了通过Clang-Tidy检查进行诊断外，Clang-Tidy还通过Clang静态分析器发出诊断信息，并由clang编译器本身提供警告。它们都具有可配置的严重性级别的相应ReSharper检查，可以根据您的需要进行调整。

请注意，Clang-Tidy目前不支持使用UTF-16或UTF-32编码的文件。
