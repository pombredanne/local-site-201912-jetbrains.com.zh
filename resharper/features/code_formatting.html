代码风格

ReSharper帮助您在整个代码库中保持一致的代码样式。默认样式设置基于广泛接受的约定和最佳实践。但是，如果您的个人偏好或公司标准不同，则可以全局或按解决方案配置它们，并在整个团队中共享设置 。

当ReSharper在代码完成中建议符号名称， 生成新成员 ，应用代码模板并进行重构时 ，将考虑您的代码样式首选项。

通过检查代码可以发现违反代码样式的情况，可以通过快速修复或代码清除来解决。

代码清理

代码清理是一项功能，可帮助您立即消除一个或多个文件，项目或整个解决方案中违反代码风格的情况，所有这些都只需一个快捷键： Ctrl + E，C 。

可以使用您的自定义任务特定配置文件来补充两个默认配置文件- 完全清除 （适用于所有样式样式设置，适用于命名样式 ）和重新格式化代码 （仅适用于格式设置规则 ）。

代码格式化

ReSharper可以根据您的喜好重新格式化任意选定的代码块，当前文件中的所有代码，目录中的所有文件，甚至整个项目或解决方案中的所有文件。ReSharper为所有受支持的语言提供了独特的格式设置选项。重新格式化会影响大括号的布局，空行，换行和换行，各种上下文中的空格，缩进，对齐多个结构以及许多其他选项，您可以在“ ReSharper选项”对话框中进行微调。

通过在需要重新格式化的所选代码上按Alt + Enter ，可以使用格式规则的上下文配置。选择格式选择|在操作列表中进行配置以查看影响代码的所有格式化规则。

您还可以使用代码清除来重新格式化代码。

隐式/显式输入

由于改进了所得代码的可读性，因此使用C＃3.0中引入的隐式类型局部变量（ var关键字）已变得非常流行。默认情况下，ReSharper还鼓励使用var关键字。

但是，您可以更改首选项并选择使用显式类型。

您还可以使用代码清除功能来强制执行隐式/显式键入的首选项。

使用自动属性

自动实现的属性（也称为自动属性）允许以更简单的方式声明没有附加逻辑的类型属性。默认情况下，ReSharper鼓励在可能的情况下使用自动属性，但是此行为是可配置的。

您可以将具有后备字段的属性转换为具有快速修复或代码清除功能的自动属性。

C＃修饰符的样式

类型和类型成员的C＃修饰符可以按任何顺序编写。但是，在整个代码中以类似的方式排列它们是一个好习惯，这可以提高代码的可读性。ReSharper将帮助您定义所需的顺序，查找未排序的修饰语并自动对其进行排序。

此外，ReSharper将帮助您明确使用或不使用private和internal修饰符。

对于在声明或构造函数中分配且仅具有读用法的字段，使用readonly修饰符通常是合理的。默认情况下，如果满足上述条件，ReSharper建议您将字段设置为只读。

修饰符的所有首选项都是可配置的，您可以通过快速修复和代码清除来应用它们。

删除代码冗余（仅C＃）

ReSharper为冗余代码提供了数十种代码检查 。

可以使用范围修复或代码清除来清除大多数冗余，而无需用户交互。

删除代码冗余时，ReSharper会考虑所有其他代码样式设置。如果特定的代码构造被代码样式设置定义为非冗余，则它们将保持不变。

命名方式

ReSharper允许您为不同的语言和符号定义自定义命名样式设置，包括类型，名称空间，接口，参数，方法属性和事件，静态和实例字段等。

对于每种符号，您可以配置以下五个大小写选项之一：前缀和后缀，不同访问权限的变体，保留的缩写以及其他选项。

所有违反命名方式的行为都会在编辑器中突出显示，并且可以轻松修复。

此外，您可以使用“ 查找代码问题”功能在“ 检查结果”窗口中浏览（并快速修复）整个解决方案（或更狭窄范围）中的所有命名风格违规。

命名空间导入样式

为了使代码更具可读性，您可以使用指令删除多余的代码和/或缩短C＃和VB.NET代码文件中的合格引用。此外，导入名称空间的方法也有所不同-您可以using / import指令添加或使用完全限定的类型名称，可以将指令放置在文件顶部或名称空间中，等等。

使用ReSharper，您可以直接在编辑器中快速检测并删除多余的导入，也可以使用代码清除 ，这还可以帮助您按字母顺序对所需指令进行排序并将它们包装在一个区域中。

内置C＃类型的参考样式

C＃类型关键字和.NET Framework（CLR）类型名称对于内置C＃类型可以互换。如果您希望对内置类型名称使用一种或另一种样式，ReSharper可以帮助您保持自己的偏好。

该首选项是可配置的，您还可以通过代码清除来实施它。

C＃属性的样式

C＃中的多个属性可以用不同的方式编写：您可以将多个属性合并在同一方括号部分中，或将多个部分用于不同的属性。ReSharper允许您为多个属性指定样式，并在所需范围内实施此样式：

ReSharper还为属性提供了多种格式设置规则 ，并允许使用代码清理以批处理模式对属性应用所有样式首选项。

可选成员限定词的样式

在C＃中，您可以使用this关键字限定实例成员，而静态成员可以使用类型名或基本类型名进行限定。这些限定词可帮助您消除被局部变量或方法参数隐藏的成员的歧义。在所有其他情况下，这些修饰符是可选的，您可以自行决定是否使用它们。默认情况下，ReSharper将所有可选的成员限定符视为冗余，并建议删除它们。

如果您更喜欢使用可选的限定词，则可以配置相应的代码样式设置。例如，您可能更喜欢使用声明静态类型的类型来限定它们。

可选括号的样式

在许多情况下，表达式中的括号是可选的。有时它们可以帮助您弄清操作的优先顺序-例如if ((a & b) | c) ，但是有时，它们根本没有帮助，只是累加了无效代码-例如var d = a + (b + c); 。ReSharper将帮助您定义可选括号对您有帮助的位置，然后清除多余的括号，或自动为非显而易见的操作优先级添加说明括号。

文件和类型布局（仅C＃）

在C＃代码中，ReSharper可以自动对文件重新排序并在代码中键入成员。成员将根据其种类和访问修饰符进行排序，分组，包装到指定区域等。ReSharper提供了两个默认的布局模式（重新排序规则集）和一个智能的可视化编辑器。
