代码分析

ReSharper在C＃，VB.NET，XAML，XML，ASP.NET，ASP.NET MVC，ASP.NET Core，JavaScript，TypeScript，HTML，CSS，JSON，ResX和Protobuf中提供连续的代码质量分析。甚至在编译之前就可以立即检测到错误和问题。

持续的代码质量分析

ReSharper在设计时对代码进行了2200多次代码检查 ，因此您可以立即查看当前文件甚至整个解决方案是否包含任何错误或问题。

ReSharper在Visual Studio编辑器中突出显示检测到的错误和问题，并使用右侧的标记栏将其可视化。

如果文件包含错误或问题，则标记栏顶部的状态指示器会更改其颜色和图标。右键单击该图标，可以快速切换ReSharper代码分析，标识符突出显示和Visual Studio代码分析，以及在最高严重性级别的问题之间导航。

失误

ReSharper可以在编辑器中立即在编码时立即检测并突出显示错误。

错误以红色字体或下划线突出显示。如果将鼠标指针悬停在错误上，则其描述将显示在工具提示中。

ReSharper不仅能够检测纯C＃或VB.NET代码中的错误，而且还可以检测ASP.NET代码隐藏文件，标记文件中的引用以及ASP.NET MVC调用中的错误。

对于XAML，ReSharper会检查XML语法并熟练地验证XAML语义，包括XAML实体之间的引用。

在构建脚本（NAnt和MSBuild）中，ReSharper分析并突出显示未知的属性，目标，任务和属性。

还可以使用JavaScript，TypeScript，HTML，CSS和XML进行代码检查。

警告事项

除了编译器错误和警告之外，ReSharper还会显示自己的警告，这些警告不会阻止您的代码进行编译，但是仍然可能代表严重的编码效率低下。例如，ReSharper会通知您有关冗余的强制类型转换，不正确的格式字符串，已声明但从未使用过的局部变量等信息。带有相关警告的结构以灰色文本或蓝色卷曲下划线强调。

意见建议

代码建议可提供更深层次的代码分析。建议提供对代码结构的见解，使您将注意力集中在不一定是不好或错误的事情上，但可能对了解有用。建议以绿色卷曲下划线突出显示。

例如，您可能想知道可以使用基本类型声明某个参数，这意味着您的方法仅使用该参数的基本类型的成员。ReSharper还可以建议从具有后备字段的属性创建自动属性，将静态方法调用转换为扩展方法调用，或者在可能时使用'var'关键字。

提示

突出显示严重性的代码的最终和最低程度是提示。与错误，警告和建议不同，在导航到 标记栏中未显示的下一个或上一个突出显示时 ，不会考虑提示，因此不会影响状态指示器的颜色。

提示只是使您注意特定的代码细节，并建议一种改进方法。带有相关提示的代码项以绿色的下划线突出显示。

代码问题快速修复

ReSharper随附1200多个快速修复程序 ，可帮助您立即以所有受支持的语言修复大多数检测到的代码问题。应用快速修复就像在突出显示的代码问题上按Alt + Enter并选择适当的方法来解决问题或改进次优代码一样简单。

解决方案范围内的检查

解决方案范围内的警告和建议可帮助您即时检测未使用的非私有成员。仅当您让ReSharper 分析您的整个解决方案时，这些检查才起作用。查找代码和标记栏上突出显示的问题。具体来说，您会发现：

  • 未使用的非私有声明。
  • 非私有方法的未使用的返回值。
  • 未访问的非私有字段。
  • 非私有成员中未使用的参数。
  • 永远不会调用的抽象或虚拟事件。
  • 未分配的字段。
  • 可以设为内部而非公开的成员和类型。

此外，还提供代码注释 ，如果有意保留某些非私有类型和成员，您可以使用它们来标记它们，以防止ReSharper对其大惊小怪。

禁用和配置代码检查项目

您可以随时更改不同分析项目的严重性，例如，根据您的个人编码风格或公司要求，使ReSharper以错误，警告，代码建议或提示的形式显示特定的检查/通知。
更重要的是，有时您会更好地了解特定代码段就可以了，并且不需要警告，因此您可以仅禁用特定警告。

要禁用或配置代码检查项目，请使用“ 代码检查” | “ 代码检查”。 ReSharper选项的“ 检查严重性”页面。您还可以通过按Alt + Enter调用的操作列表中的“ 检查选项”菜单中配置特定的警告，建议或提示。

在突出显示的代码项之间导航

每个错误，警告或建议都由标记栏上的单独条纹表示。单击该条直接将您导航到包含错误或引起警告/建议的代码行。

您可以通过按Alt + PageDown （向前）和Alt + PageUp （向后）在错误，警告和建议之间导航。您还可以通过按Shift + Alt + PageDown （移至下一个错误）和Shift + Alt + PageUp （移至上一个错误）在错误之间（跳过所有警告和建议）。在状态栏中显示一条描述当前错误，警告或建议的消息。

还有在代码问题之间导航的另一种方法：您可以在特定范围内查找所有代码问题 ，然后在专用工具窗口中进行浏览。

您的自定义代码检查

ReSharper提供了“ 结构搜索和替换”功能来查找与特定模式匹配的代码，并有选择地将其替换为与另一模式匹配的代码。更令人兴奋的是，ReSharper可以持续监视您的搜索模式解决方案，突出显示与它们匹配的代码，并提供快速修复程序来根据您的替换模式替换代码。

从本质上讲，这意味着您可以使用自定义检查扩展ReSharper自己的代码检查集。例如，如果要迁移到较新版本的框架，则可以创建搜索模式以查找其较旧API的用法，并替换模式以引入更新的API。

通过解决方案范围的分析监控错误

ReSharper不仅能够在设计时分析打开的文件，还可以扩展其分析技能，以覆盖整个解决方案。

假设您将成员的可见性从“公开”更改为“内部”，并假定该成员仅在当前程序集中使用。在此阶段，ReSharper在当前文件中未发现任何错误。然后，您打开“解决方案范围分析”，ReSharper会发现出了问题。您跳到解决方案中的下一个错误，您发现有人从当前项目外部使用了此成员。

您要做的就是明确打开“解决方案范围分析”，然后在分析解决方案的代码之后，在专用窗口中查看错误列表。

即使不打开该窗口，您仍然可以使用转到解决方案中的下一个错误/警告 （ Shift + Alt + PageDown ）和转到解决方案中的上一个错误/警告 （ Shift + Alt + F12 ）命令轻松浏览解决方案中的错误 。

通过解决方案范围的分析监控警告

解决方案中的最后一个错误后，红色状态栏指示器将变为绿色。但是，解决方案范围的分析可以帮助您进一步提高解决方案中的代码质量-单击“ 包括警告” 在错误视图中，ReSharper将在整个解决方案中开始监视警告（包括编译器警告以及通过ReSharper的检查以警告严重性级别检测到的警告 ）。

如果解决方案中再次出现错误，警告将被自动隐藏，以便您专注于解决错误。修复所有错误后，将再次显示警告。

通话追踪

过去，尝试跟踪代码中的调用序列可能会导致许多“ 查找结果”窗口和上下文丢失。为了解决此问题，ReSharper可以在单个工具窗口中可视化整个调用序列。这称为“ 呼叫跟踪” ，它使您可以查看和浏览代码中的调用链。

由于支持事件，接口和关闭，ReSharper的呼叫跟踪是对Visual Studio自己的呼叫层次结构的重大改进。

要显示呼叫顺序，请选择ReSharper |检查拨出电话或重新整理|检查来电 ，或使用“ 检查此”快捷功能。

查找代码问题

通过ReSharper，您可以从一个名为“ 检查结果”的工具窗口中找到所有代码质量问题，冗余，语言使用机会以及代码改进建议。

要查找特定范围内的代码问题，请选择ReSharper |。检查解决方案或ReSharper中的 代码问题 |检查当前项目中的代码问题 。或者，在解决方案资源管理器中选择一个节点，右键单击它，然后选择“ 查找代码问题” 。

使用“ 检查结果”窗口可以筛选检测到的问题，按问题类型，严重性，类别和其他属性将结果重新分组，并将结果导出到文件中。

价值追踪

值跟踪为您提供有关程序中数据流的重要信息。您可以在源代码中的任何位置指向变量，参数，字段或属性，并要求ReSharper对其进行检查。然后，您将看到其价值如何在您的程序中流动，流回其源或直接流向消费者。

值跟踪可以帮助您确定某个不正确的值可能如何传递到程序中的给定点，以及下一步该如何传递。这可以帮助您调查可能的NullReferenceExceptions ，不当的行为以及获取不正确值的原因。

要可视化某个值所参与的数据流，请选择ReSharper |检查价值来源或价值 共享|检查值Destination ，或使用“ 检查此”快捷功能。

代码注释

有时，即使像ReSharper一样强大的代码分析引擎也缺少检测某些问题所必需的知识。例如，如果某个方法被设计为永不返回null并且其客户端也进行了相应的设计，则如果有人将方法更改为返回null ，则代码分析将找不到可能的问题。

在这种情况下，还有其他许多情况下， JetBrains.Annotations框架有助于发现无法通过其他方式推断出的问题。在此框架中声明的属性可使ReSharper按您需要的方式分析代码。

要在源代码中使用ReSharper代码注释属性，请安装相应的NuGet软件包 。您可以通过添加using JetBrains.Annotations;指令，ReSharper会帮助您自动获取软件包。
