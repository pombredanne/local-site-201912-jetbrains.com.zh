XAML编辑工具

使用ReSharper，每当与通用Windows平台（UWP），WPF，Silverlight或Windows Phone应用程序一起使用时，您都会获得编辑XAML文件的更多帮助。这包括动态代码质量分析，重构，代码生成和重新排列以及解决方案范围内的快速导航。

代码质量分析

ReSharper的代码质量分析功能可帮助您快速定位和更正XAML文件中的错误。让我们用几个例子看看它是如何工作的。

范例1。XAML语言级别错误

假设您正在一个WPF项目中，该项目只允许使用XAML 2006。如果ReSharper遇到XAML 2009中的通用对象，则会将该对象突出显示为语言错误。如果您在突出显示上按Alt + Enter ，ReSharper将显示一个快速修复程序，建议您声明从System.继承的类型System.集合。通用。改为使用List<string> 。

当您应用快速修复程序时，ReSharper：

  • 创建一个新的.cs文件，在其中声明从List继承的新包装器类型。
  • 在原始XAML文件中，它将删除TypeArguments属性，将对象的类型更改为新的包装器类型，并在必要时插入新的名称空间指令。

示例2不存在的资源

如果您使用的资源尚未声明，则ReSharper错误突出显示不会让您忘记这一点。

当您按Alt + Enter时 ，ReSharper会建议几个可能的位置来创建缺少的资源。

如果尚未声明所生成资源的类型，则没问题：ReSharper将为您提供一个选择，以根据使用情况创建CLR类型或XAML类型（即UserControl ）。选择两个选项之一将创建具有默认实现的文件。

范例3。评论放错位置

在元素中不小心添加注释会导致ReSharper突出显示该注释并提供解决错误的选项。

范例4。冗余类型限定符

如果在样式设置器上定义了TargetType ，则ReSharper会检测类型限定符中的冗余并提供修复程序。

扩展的XAML IntelliSense

您可以在XAML标签中使用ReSharper的代码完成功能。对CamelHumps的支持将帮助您快速添加属性：只需输入一些符号，然后按Ctrl + Space即可显示ReSharper的“ 符号完成”提供的相关建议的列表。

如果您通过按Ctrl + Alt + Space调用Smart Completion ，则ReSharper将根据您在代码中的位置提供其他上下文相关的选项。例如，在事件处理程序声明中调用智能完成将提供生成处理程序方法的功能。

通过在属性内使用智能完成，您可以快速创建新资源（静态或动态），绑定或对静态元素的引用。

ReSharper代码完成的另一个令人兴奋的优点是，它可以显示诸如Background或OpacityMask这样的属性中期望的画笔颜色。

ReSharper还支持XAML中的导入符号完成 。这意味着，您可以从解决方案中引用的程序集中完成类型，但在当前文件中没有相应的名称空间导入指令。

当然，以这种方式使用ReSharper代码完成功能后，所需的名称空间导入指令将自动插入。

或者，如果XAML文件中的类型由于某种原因未导入，则ReSharper将为您提供快速创建名称空间指令的选项。

导航和搜索

ReSharper的“ 导航至”菜单以与C＃代码文件相同的方式处理XAML代码元素。在符号声明或用法上按Alt +`将打开一个菜单，其中包含多个可能的导航目标。

通过“ 转到文件成员”进行的导航在XAML文件中的工作方式与在C＃文件中的工作方式相同，有助于导航到具有特定类型，名称或键的元素。再一次，您不必键入全名-对小写字母CamelHumps的支持使您仅键入wl就可以找到WatchList 。

当您在XAML文件中按Ctrl + Alt + F7组合键时 ，您会得到所有相关文件的列表：例如，代码隐藏文件，当前XAML文件中引用的任何资源或控件中使用的图像。

转到声明 （ F12 ），然后按Ctrl键单击导航在XAML文件中也与在普通C＃代码文件中相同。例如，在按住Ctrl的同时，可以将鼠标悬停在特定类的名称上。ReSharper将在课程名称下划线，单击链接将直接转到课程定义。

同样，与代码文件一样，“ 文件结构”窗口显示XAML文件的结构，使您可以快速导航到要查找的代码元素。

您可以通过按Shift + F12来查找特定XAML元素的用法 ，这将调用“ 查找用法”命令。ReSharper将显示元素正在使用的所有位置。

您还可以按Shift + Alt + F11 ，它将突出显示用法 ，使您可以快速概览当前文件中正在使用特定符号的位置。

生成依赖项属性

创建依赖项属性可能很无聊。这是ReSharper可以解决的另一个问题：其代码模板引擎功能强大，足以让您创建细粒度的代码片段，并通过几次击键将其部署到依赖项属性中。您只需在编辑器中键入活动模板的名称，即可获得一对属性和相应的依赖项属性，并在需要填充的所有占位符中使用智能代码完成功能。

同样，您可以使ReSharper部署附加属性，这些属性比常规依赖项属性还要麻烦。部署实时模板后，您可以传递其所有众多变量，并在适用时使用ReSharper代码完成。

尽管ReSharper暂时没有提供用于依赖项属性和附加属性的实时模板，但是您可以下载并导入它们。

XML和XAML编辑助手

ReSharper支持各种上下文操作 ，这些操作可以简化XML语法的使用，并且可以在XAML中使用。仅举几例，您可以将标签名称替换为其他名称，将属性转换为嵌套元素，或者删除标签并提升其子级。

还有十多个特定于XAML的上下文操作，使您可以添加或删除列或行定义，创建视图和资源，在不同类型的资源引用之间切换，将属性设置器提取为样式等等。

重构XAML代码

由于XAML元素实质上等效于普通类成员，因此某些在C＃中起作用的重构在XAML中也起作用。例如，您只需打开“ 重构此”菜单即可重命名或安全删除资源。

当您浏览XAML中对属性或方法的引用时，“ 重构此菜单”中显示的重构范围与您用C＃或VB.NET代码打开该菜单时所期望的范围相符。

除了C＃和VB.NET重构之外，还有一些特定于XAML的重构：

在XAML中，常见的情况是为多个控件定义内联样式，然后将它们重新定位为样式定义。ReSharper借助“ 提取样式”重构使此操作变得轻而易举。

为该重构而出现的对话框可帮助您确定将属性提取到的资源（您可以选择现有资源或创建新资源），以及要提取的属性。

提取资源和内联资源重构使您几乎可以将任何属性分配（包括标记扩展中的属性分配）移至资源，反之亦然-从资源中提取定义并将其直接拼接到元素中。

代码重新排序

使用ReSharper的代码重新排序功能，您可以在XAML文件中的元素或这些元素中的属性之间移动。这两个操作都是通过按Ctrl + Alt + Shift键和箭头键执行的。

当使用属性，这些组合键让你移动的标签内各地的属性，使左 ， 右箭头键更改属性的位置。但是，如果您按向上和向下键，则您的操作将影响整个标签。

向上和向下键可相对于其他标签移动整个标签。如果您使用Ctrl + Alt + Right 扩展选择范围以包含标记中的所有内容，则还可以按相同程度使用向左和向右键 ，唯一的区别是这些组合也会“删除”您的当前选择，将其放在您遇到的任何标签中。
