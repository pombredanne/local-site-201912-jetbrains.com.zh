导航和搜索

借助ReSharper的导航功能，可以轻松遍历整个解决方案。跳转到代码库中的任何文件，类型或成员，或从特定符号导航至其用法，基本和派生符号或实现。

导航

导航到是用于各种上下文导航的单个快捷方式。导航到以列出当前插入符位置上所有可用的目的地。按Alt +` ，可快速导航至声明，类型声明，基类，继承者或插入符号下符号的用法；接口的接口实现；功能退出功能，以及更多功能。

转到文件成员

要快速导航到当前文件中的特定方法或字段，请使用按Alt + \可用的“转到文件成员”命令。与“转到类型”相同，开始输入符号名称，然后从匹配成员的列表中进行选择。此功能适用于所有受支持语言的文件成员。

转到所有内容/类型

该命令可与Ctrl + T一起使用 ，具有两个不同的功能：

首先出现的转到所有内容，提供对所有可能的目标（类型，符号或文件）的快速导航。建议列表会在您调用命令后立即出现，并且最初包括您最近导航到的文件和代码位置。键入时，列表会根据您的输入进行更新。您可以使用以/开头的特殊内联命令来对搜索结果进行高级过滤。

再次单击快捷方式时出现的“ 转到类型” ，可让您导航到解决方案中的任何类型。

这些和其他“转到”功能也支持通配符： * （星号）表示零个或多个字符；搜索查询周围的双引号将执行完全匹配搜索； ? 完全匹配搜索中的（问号）代表一个字符一次；

转到文本进行文本搜索

通过Ctrl + T，T，T可以使用此导航操作，您可以快速查找并导航到解决方案中包含的源文件和文本文件中的任何文本。与其他导航命令类似，如果有太多匹配项，则可以在数字键盘上按+ ，以在“ 查找结果”窗口中浏览结果。您还可以使用特殊的内联命令来缩小搜索结果的范围： /cp在当前项目中进行搜索， /ext:在具有特定扩展名的文件中进行搜索，或/fm:用于特定文件掩码。

转到实施

使用Ctrl + F12快捷键可以使用此导航功能，使您从基本类型或成员跳转到其任何最终实现，而绕过继承链中的中间步骤。

它的工作方式类似于“ 转到派生符号”，但是当您需要绕过类型层次结构中的一个或多个级别时，可以节省您的时间。观看GIF，以了解不同之处。

行动起来

ReSharper可以帮助您快速查找和应用其任何操作，而无需浏览其菜单（通过action ，我们指的是可用于当前插入符号位置的可执行命令或打开工具窗口的命令）。要查找动作，只需在编辑器中的任意位置按Alt + Enter ，然后开始输入命令名称或相关词即可。

转到公开API

此命令与“ 导航到”快捷方式（ Alt +` ）一起提供，可从一种类型导航到解决方案中的所有位置，在该位置上您可以获取该类型的实例：公共字段和属性，方法返回值，输出参数等。

转到符号

“转到符号”命令按名称在解决方案范围内搜索方法，字段和其他文件成员。通过按Shift + Alt + T可以使用该命令。请注意，所有ReSharper的“转到”命令（包括转到符号）都支持不区分大小写的CamelHumps 。也就是说，你可以找到一个更快的成员通过输入其首字母及任何后续字符，其名称包含-例如，你可以通过输入tfixs找到testFixedSizeBuffer。

前往基地

您可以通过在编辑器中将插入符号放在类型或方法的用法或声明上，然后按Alt + Home ，从而将层次结构导航到基本类型或方法。
顺便说一句，您还可以通过查看方法声明左侧显示的特殊装订线图标来查看方法何时覆盖，实现或隐藏另一个方法。单击该图标可在方法层次结构中向上导航。

前往宣言

要导航到符号的声明，请将插入号放置在任何符号使用位置，然后按F12键，或者按住Ctrl并在符号上单击鼠标左键。ReSharper将跳转到相关源文件中相应类型，方法，字段或局部变量的声明。对于库符号，取决于ReSharper设置，相应的实体将作为元数据视图或反编译的代码显示在Visual Studio的对象浏览器或编辑器中。

在用法之间导航

当插入符已经在符号声明中时，您还可以调用“ 转到声明”命令。如果您只有一种声明和一种用法，则可以使用一种快捷方式在它们之间进行切换。如果您多次使用该符号，则随后的“ 转至声明”命中将使您进一步找到该符号的用法，一次使用一次。在用法之间进行导航是通过类似于查找用法的窗格进行的，该窗格枚举找到的用法，包含在用法之间单击鼠标的其他控件，并根据需要将所有找到的用法刷新到常规的“查找结果”窗口。

转到相关文件

可以通过按Ctrl + Alt + F7组合键使用此功能，将您带到以一种或另一种方式从当前文件引用的任何文件。这对网站和应用程序最有用：例如，它使您能够从Web表单跳转到其主页，用户控件，引用的图像或JavaScript文件。在ASP.NET MVC应用程序中，此功能简化了从视图到控制器以及向后的导航。

要了解有关ReSharper向Web开发人员提供的高级导航和其他功能的更多信息，请参见ASP.NET和ASP.NET MVC工具 。

转到附近的文件

“ 导航到”快捷方式（ Alt +` ）提供了此功能，可帮助您快速浏览当前文件的项目结构。在打开的弹出窗口中，您可以查看当前文件所属的项目，文件夹和文件在同一目录级别，并且可以轻松地跳转到这些文件或创建新文件或文件夹。

导航到库代码

任何解决方案都包含其自身的源代码和对库的引用。我们认为，就像您对源代码一样，能够导航到这些库并在其中找到它会很棒。

一些公司使用调试信息文件（ PDB ）的源服务器功能发布部分源代码。另一方面，Microsoft使用此技术来提供对.NET Framework源代码的访问。如果您有可用的PDB并安装了ReSharper，则可以访问以这种方式存储的源，就好像它们是解决方案的一部分一样。如果您想找出某个.NET Framework类包含的内容，ReSharper也会向您显示。

即使源服务器不可用，也不用担心：ReSharper能够从元数据中重建库代码的结构。

当源服务器可用时，它的工作方式如下：

  1. 您单击Alt +`对库符号，打开导航要与导航选项列表。
  2. 您为库符号选择一个外部导航选项。
  3. ReSharper从源服务器下载PDB文件。
  4. ReSharper会显示所选库符号声明，就像在解决方案中定义的一样。

查找用法

查找用法快速查找代码中任何符号（类型，方法，字段等）的用法。只需将插入号放在要查找其用法的符号上，然后按Shift + F12即可 。如果您有混合语言项目，则可以使用ReSharper在多种语言中找到用法。

搜索结果显示在“ 查找结果”窗口中，并按层次结构进行组织。在此窗口中，您可以使用键盘或鼠标直接导航到任何用法。查找结果的附加功能提供了几种使用搜索结果的方法，包括：

  • 源代码预览
  • 突出显示损坏或未完成的用法
  • 按读写用法，调用用法，属性用法，文档中的用法以及更多条件过滤
  • 按用途类型和类别分组
  • 最近使用情况搜索的标签等。

打开“查找结果”窗口时，可以通过按Ctrl + Alt + PageUp / PageDown （甚至从编辑器中）在用法之间进行导航。

查找高级用法

要指定搜索范围和要查找的用法类型，请按Ctrl + Shift + Alt + F12使用“查找用法高级”功能。

根据上下文，您将能够准确地指定解决方案中的搜索范围。是否在参考图书馆中搜索；是否查找文本出现，实现属性或什至是对所查看符号的动态引用。

档案结构

“文件结构”窗口适用于所有受支持的语言和文件类型，在C＃或VB.NET文件中，您可以查看它们包含的方法，字段，类和区域，还可以直接导航到其声明-只需按Ctrl + Alt + F。

在ASP.NET中，“文件结构”使您可以检查标签，脚本，控件以及当前打开文件的其他结构元素，并直接导航到它们的声明。有关此功能和其他导航功能如何帮助Web开发人员的详细信息，请参见ASP.NET和ASP.NET MVC工具 。

在构建脚本中，“文件结构”使您可以查看当前脚本包含的属性和目标。

该窗口还允许您通过简单的拖放，调用导航操作和重构 ，创建和删除区域来重新排列文件中的项目。

“ 文件结构”窗口与编辑器完全同步。对文件所做的所有更改都会立即反映在文件结构中，反之亦然。

使用Tab进行结构导航

ReSharper允许使用Tab和Shift + Tab键在代码的逻辑部分之间跳转，类似于在Web浏览器中导航的方式。

例如，在编辑for语句时， Tab将选择初始化程序语句，然后选择终止条件，然后选择迭代程序语句。 Shift + Tab将以相反的顺序进行这些选择。

当光标位于行中第一个非空白字符之前时，您仍然可以使用Tab键进行缩进。

IL查看器

使用中间语言查看器，您既可以挖掘第三方库代码，又可以显示与您自己的源代码相对应的IL（请注意，您必须先进行编译）。

要查看IL Viewer的工作原理，请选择ReSharper | Windows |菜单中的IL Viewer 。

堆栈跟踪资源管理器

当您收到外部堆栈跟踪（例如，来自错误报告）时，可以将其复制粘贴到“ 堆栈跟踪资源管理器”中，以导航到异常发生的位置。堆栈跟踪中的行将表示为超链接。要打开堆栈跟踪资源管理器 ，请将堆栈跟踪复制到剪贴板，然后按Ctrl + E，T 。

转到文件

转到文件可将您导航到解决方案中的任何文件。该命令的快捷键是Ctrl + Shift + T ，它支持其他“转到”命令中使用的所有搜索技术和通配符。

转到衍生符号

通过此命令，您可以通过按Alt + End向下导航到派生类型或方法。直接继承者以粗体突出显示。

转到通用替代

此命令与“ 导航到”快捷方式一起提供，显示替换选定的通用类型参数的所有类型的列表，以及使用每个替换的所有代码位置。

转到类型声明

按Ctrl + Shift + F11导航到变量，字段或参数所属的类型的声明。ReSharper将切换到源代码中的声明，或者对于库类型，在对象浏览器中显示相应的符号或显示反编译的代码。

转到下一个成员/标签

您可以通过按Alt + Up快速转到C＃或VB.NET代码文件中的下一个文件成员，或者快速转到XML或XAML文件中的下一个标记。

转到上一个成员/标签

与转到下一个成员类似，此功能可将您带到上一个成员或标签-您所要做的就是按Alt + Down 。

转到用法

您可以通过按Shift + Alt + F12快速导航到很少使用的符号（例如，使用私有字段）。它不会打开整个工具窗口，而是会在弹出窗口中显示用法。

但是，如果事实证明带有符号用法列表的弹出窗口包含太多项目，则只需单击弹出窗口右上角的放大镜图标，即可将用法列表转移到“查找结果”窗口中。

突出显示用法

要突出显示当前文件中某个符号的所有用法，只需将插入号置于任何符号用法上，然后按Shift + Alt + F11 。您还可以突出显示名称空间导入指令甚至表达式的用法。在大文件中，您可以使用右侧的标记栏在突出显示的用法之间导航。
ReSharper可以快速查找并突出显示XAML标记内以及跨语言的符号用法。

结构搜索和替换

使用此功能，您可以创建模式来搜索ReSharper开箱即用无法检测到的代码气味或旧代码，并将其替换为良好的代码。除此之外，您还可以轻松地重复删除可疑代码，可以创建自定义代码检查并设置ReSharper为其提供快速修复 。

建立模式和执行良好实践从未如此简单。公司和团队政策，自定义框架，最喜欢的开源库和工具-结构化的模式能够涵盖所有内容。另外，您可以导入和导出最佳模式以与同事共享或在ReSharper安装之间移动。

类型层次结构

使用ReSharper，您可以在专用窗口中查看某种类型的继承层次结构。该窗口同时显示基本类型和所选类型的继承者，并允许您单击一下即可导航到它们中的任何一个。对于层次结构中的任何节点，您可以在单独的预览窗格中查看所有或仅多态成员。
要打开某个类型的“类型层次结构”窗口，请将插入号放置在该类型名称上，然后按Ctrl + Alt + H或选择ReSharper |。查看|键入层次结构 。

书签

这是一个简单但功能强大的功能：您可以使用单个快捷键放下带编号的标记，然后随时使用另一个快捷键跳回。您最多可以创建10个带编号的书签，以及无限个无编号的书签。书签位置的完整列表显示在单个弹出窗口中，以加快代码点之间的导航。

查看最近的文件

按Ctrl +，将打开一个弹出窗口，其中包含最近打开的文件列表，您可以在其中选择要导航到的任何项目。

查看最近的编辑（并转到最后的编辑位置）

最近编辑的工作方式与最近文件类似，向您显示一个弹出窗口，其中包含您最近修改的文件和符号的列表。如果您当前的编辑会话已经开始了一段时间，请通过键入要重新打开的文件的名称来缩小列表的范围。

或者，使用“转到上一个编辑位置” Ctrl + Shift + Backspace可以将插入号快速定位到更改代码的最新位置，而不会显示弹出窗口。此快捷方式的后续点击将使您更深入地了解编辑历史。

待办事项资源管理器

待办事项资源管理器可帮助您跟踪标记有todo ， note和/或bug标签的所有提醒项，以及引发NotImplementedException代码位置。待办事项资源管理器从解决方案中的所有文件中检索所有此类项目，并将其显示在单个工具窗口中。

如果使用自己的标记在代码注释中做注释，则可以配置自定义模式，To-do Explorer也将找到它。

流程浏览器

“ 进程资源管理器”窗口显示所有当前正在运行的进程，允许浏览其模块并反编译其中的.NET程序集。

您可以选择显示或隐藏本机流程和模块，或打开流程层次结构视图。

筛选列表以查找所需的模块或过程，然后将其程序集添加到Assembly Explorer中以进行进一步调查。
