项目级功能

ReSharper提供了一组功能，这些功能在应用于项目或整个解决方案时非常有用，这与单独的编码助手相反。

这些功能包括ReSharper Build，用于优化MSBuild编译项目的方式；显示项目或类型之间的依赖关系的图，以及查找实现这些依赖关系的特定代码的方式；重构以优化解决方案项目中的参考；以及更多功能。

ReSharper版本

ReSharper Build是一种进程外增量构建工具 ，可以利用多个过程，可视化不同类型的项目构建状态，并针对具有很多依赖性的大型解决方案进行了优化。

ReSharper Build用将启发式方法仅应用于需要更新的项目的系统取代了Visual Studio生成管理。请注意，它不能替代MSBuild，并且您的项目仍可以正常构建。

运行配置

ReSharper提供了使用Run配置在Visual Studio中执行代码的新方法。共有三种运行配置：

  • 项目 —使用此配置类型，您可以运行或调试解决方案中的任何可执行项目。在具有多个可执行项目的大型解决方案中，这可能会有所帮助。
  • 静态方法 -此配置类型使您可以运行或调试任何不带参数的公共静态方法。是否需要单独调试棘手的算法？只需创建一个静态方法，并设置运行配置即可。
  • 可执行文件 -使用此配置类型，您可以运行或调试任何可执行文件。如果选择debug命令，ReSharper将运行可执行文件，然后将Visual Studio附加到其进程中。这种配置类型的一个好用例是开发另一个程序的插件，您必须在其中创建复杂的执行设置。

您也可以执行静态方法而无需创建永久运行配置：只需按Alt + Enter并使用操作列表中的“ 调试”子菜单。

要运行或调试活动配置（默认情况下为Visual Studio启动项目），请按Ctrl + F5或F5 。要管理运行配置，请按Ctrl + Shift + Alt + R或选择ReSharper |。工具|在菜单中运行配置 。

项目依赖图

通过ReSharper，您可以使用解决方案体系结构的直观表示来探索解决方案中的项目依赖项。您可以随时打开架构视图（ ReSharper |架构|显示项目依赖关系图 ）并浏览项目依赖关系，而无需编译任何内容。

在此视图中，您可以执行重构 （例如Adjust Namespaces或Remove Unused References ）， 导航操作和其他有用的命令。此外，您可以保存一个依赖项快照，然后将其与解决方案的当前状态进行比较。

类型依赖图

ReSharper使您可以直观地研究解决方案中类型如何相互依赖。在类型依赖关系图中，您可以添加来自不同项目或已编译程序集的任意数量的类型，并可视化它们之间的不同类型的依赖关系。除了主菜单命令， ReSharper |建筑|显示类型依赖关系图 ，有很多方法可以向图中添加类型：例如，您可以从“ 查找结果”或“ 层次结构”窗口，显示导航结果的弹出窗口，项目依赖关系图，拖放中添加所有类型。其他工具窗口中的其他类型，等等。

与项目依赖关系图相似，类型依赖关系图中也提供了许多导航操作 ，“ 移至文件夹”重构也是如此，该操作在此处可拖放。

装配依赖图

ReSharper还允许您探索已编译程序集之间的依赖关系。您可以从任意数量的装配中创建图，并探索这些装配之间如何相互引用。

您可以在“程序集资源管理器”中在选定的程序集上调用该图，并通过它们之间的引用来探索在程序集之间建立的连接。

NuGet浏览器

使用ReSharper，您可以在NuGet程序包库中搜索类型或名称空间，并安装与您的搜索匹配的程序包-所有这些都无需离开Visual Studio。您可以在NuGet浏览器窗口（ ReSharper | Windows | NuGet浏览器 ）中完成所有这些操作。

您甚至可以直接从编辑器开始搜索：在任何未解析的类型或名称空间上按Alt + Enter ，然后在nuget.org上选择“ 查找”以查找并安装缺少的NuGet软件包。

调整命名空间

某些代码重构功能可以应用于项目，解决方案文件夹或整个解决方案的范围。

例如， Adjust Namespaces重构可帮助您在任何范围内（与整个解决方案一样大）将名称空间同步到文件夹结构。也就是说，如果在某个代码文件中命名空间与项目结构中文件的位置不对应，则命名空间将被重命名以符合文件位置。所有引用修改后的名称空间的using指令都会相应更新。

删除未使用的参考

另一个项目级别的重构，“ 删除未使用的引用” ，可帮助您从项目中检测并删除多余的引用，以清理解决方案。

还有一种方法可以将引用标记为在运行时使用，这在使用依赖项注入框架时很重要。

通过标记所有隐式加载的引用，您可以安全地运行ReSharper的Optimize References和Remove Unused References命令，而不会破坏运行时所需的引用。

查找参考/从属代码

查找相关代码可让您查找依赖于所选项目的代码。 查找引用的代码的方法相反，可以查找在一定范围内遇到的所有传出引用（范围可以像方法一样小，也可以像项目一样大）。

这两个功能的搜索结果都显示在“ 查找结果”窗口中，或者，如果找到了一种用法，则插入标记会移动到相应的位置。

要查找相关代码或引用的代码，请在“解决方案资源管理器”中选择一个节点，或在文本编辑器中将插入符号放置在容器上，然后选择“ ReSharper |查找|查找依赖于模块或ReSharper的 代码 查找|查找范围之外的符号 。

装配浏览器

这个简单的窗口可帮助您打开已编译的程序集，然后使用导航和搜索命令来遍历和浏览它们，就好像您正在使用源代码一样。扩展程序集节点将列出程序集内的名称空间，可以将其进一步扩展为类型和类型成员以及程序集引用。

如果您对某个程序集特别感兴趣，Assembly Explorer将帮助您将其导出到Visual Studio项目，以便您可以浏览源代码，构建，运行和调试它。

在解决方案资源管理器中折叠全部

当在解决方案资源管理器中管理节点状态时，ReSharper可以帮助Visual Studio。在解决方案资源管理器中折叠项目或另一个高级节点时，普通Visual Studio不会折叠其包含的节点。结果，当您以后扩展项目时，您发现它揭示了一个零星的扩展结构。

使用ReSharper，您可以右键单击“解决方案资源管理器”节点，然后在上下文菜单中选择“全部折叠” -所选节点及其任何子节点将被折叠。

全部折叠是ReSharper另一个巧妙功能的完美搭配- 在解决方案资源管理器中定位 。例如，您可以从折叠项目开始，然后继续使用解决方案资源管理器中的“定位”来扩展您真正需要的节点。

在解决方案资源管理器中找到

Shift + Alt + L附带的此便捷导航功能使您可以在解决方案资源管理器中突出显示文本编辑器中当前打开的任何文件的相应节点。ReSharper突出显示该节点并自动滚动“解决方案资源管理器”，让您实际看到突出显示的节点。

当您拥有包含数十到数千个文件的大型解决方案，或者屏幕区域有限而迫使您将“解决方案资源管理器”窗口缩小到极限时，此功能特别有用。此外，该功能与Solution Explorer中的Collapse All非常匹配。

ReSharper设置的分布式存储

ReSharper设置使用分布式存储模型保存，这意味着任何设置或设置组都可以单独保存，ReSharper将在运行时合并并应用所有设置。

该模型的优点包括为不同的解决方案提供不同的设置，轻松共享设置，导入和导出设置等等。这对于共享团队中应该相同的设置尤其有用： 代码样式设置 ， 代码模板 ， 结构搜索和替换模式 。

默认情况下，存在三个设置层：用于本地计算机中的所有解决方案，当前解决方案以及共享解决方案设置。您可以轻松创建，管理和共享新图层，例如用于公司范围内的全局设置。
