代码完成

ReSharper通过更高级的功能扩展了Visual Studio的本机代码完成（IntelliSense）。例如，它根据您的键入来缩小建议的范围，了解lowerCamelHumps的缩写，根据变量和字段的名称和您的命名方式来建议变量和字段名称，自动导入选定的类型和扩展方法，等等。

符号完成

Ctrl + Space代码完成功能可以识别变量和方法，当前可见的类型名称，关键字等。ReSharper会根据您的命名方式在声明标识符时自动建议前缀。

ReSharper甚至为本地范围内的未解析符号提供了补全：即使未声明代码符号，ReSharper也会从其用法中获取其已知成员。

ReSharper的所有代码完成功能（包括符号完成功能）都支持CamelHumps-也就是说，您可以通过仅输入大写字符来完成任何一项。

在ASP.NET文件中，ReSharper还为Web控件属性和事件，数据源，内容占位符，当然还有内联C＃代码提供代码完成。
对于XAML，您可以在资源引用方面获得帮助，而在构建脚本中，ReSharper可以为您提供当前范围内可用的属性，目标，任务和属性名称。

生成完成

但是， Ctrl + Space远不止于此。它还允许您快速添加符号覆盖，部分方法主体和部分类，实现接口成员并生成属性。完成这些项目时，ReSharper将生成并插入方法，构造函数，属性等的整个主体。

智能完成

使用Ctrl + Alt + Space调用的“智能代码完成”功能可以过滤方法，字段或变量的列表，以匹配表达式的预期类型。

例如，当您将Smart Completion用于方法调用参数时，建议值列表仅缩小到所需的参数类型。智能完成还可以建议创建匿名方法，lambda表达式和正则方法，以及用于out参数的局部变量。

智能完成功能在XAML事件处理程序和其他上下文中就像一个魅力。

导入符号完成

导入符号完成是通过Shift + Alt + Space调用的。与“符号完成”（仅完成在当前位置可访问的类型）不同，此命令将显示与指定前缀匹配的所有类型，而不管它们属于哪个名称空间，并在必要时自动添加适当的名称空间导入指令。

在C＃代码文件中，“导入符号完成”在带有扩展名的点之后起作用，并使用伪指令放置必要的内容。导入符号完成在标记名称的XAML标记中也非常有用。还将根据需要插入命名空间导入指令。

双重完成

当完成列表为空或不包含预期的项目时，您始终可以第二次按快捷方式输入任何完成类型。

ReSharper将扩展完成列表，包括受保护的私有成员和内部成员，来自未引用程序集的符号，具有期望数据类型的返回值的调用链，转换为期望数据类型的等等。当然，如果您使用任何建议，则会自动添加缺少的导入和引用。

过滤代码完成

ReSharper允许您通过符号，访问修饰符和其他过滤器来缩小完成建议的范围。您可以在每次调用代码完成功能时修改应用的过滤器集，和/或选择保留过滤器状态。

C＃打字助手

ReSharper还提供了一组C＃键入辅助功能，可在您输入代码时纠正细微的错误类型。

例如，如果您碰巧键入MethodName(.) ，它将被自动替换为MethodName(). ，让您无需处理无效的语法错误即可继续输入呼叫链。

同样，ReSharper将在逐字字符串内插中将输入错误的@$前缀更正为$@ 。
