Performance

Seit der Veröffentlichung von GoLand 2019.2 im Sommer dieses Jahres haben wir intensiv daran gearbeitet, GoLand schneller zu machen und die CPU-Auslastung zu reduzieren. Eine der zentralen Performance-Verbesserungen ist die Reduzierung der Wartezeit beim Start, während die IDE alles unter der Haube vorbereitet, den Begrüßungsbildschirm anzeigt und ein Projekt indiziert.

Bitte beachten Sie, dass die Ergebnisse je nach Projektgröße sowie Alter, RAM-Ausstattung und Speicherplatz Ihres Systems variieren können. Wir arbeiten weiterhin an diesem Thema, sodass Sie in künftigen Versionen noch kürzere Startzeiten erwarten können.

Verbesserungen bei Go-Modulen

GoLand 2019.3 bietet Syntaxhervorhebung und Navigation für go.mod-Dateien. Wenn Sie auf einem Abhängigkeits-Importpfad ⌘+B (macOS) oder Strg+B (Windows/Linux) drücken, navigiert Sie GoLand zu den entsprechenden Quellen in der Projektansicht.

In der Projektansicht werden Go-Module außerdem mit ihrem vollen Importpfad statt nur dem letzten Segment angezeigt. Die Version der Go-Module wird in einer dezenteren Schriftfarbe als der Importpfad angezeigt, damit Sie leichter zwischen den beiden unterscheiden können.

Debugger-Updates

Methode eines eingebetteten Feldes aufrufen

Evaluate Expression im Debugger ermöglicht jetzt das direkte Aufrufen einer Methode eines eingebetteten Feldes.

Remote-Debugging mit Mozilla rr

Viele in Go geschriebene Produktionsservices laufen unter Linux, und Mozilla rr kann dabei helfen, schwer reproduzierbare Fehler zu untersuchen. Das Debugging mit Mozilla rr wird bereits seit GoLand 2018.3 unterstützt, aber bisher funktionierte das nur auf Linux-Entwicklersystemen. Jetzt können Sie einen Debug-Server auf einem Linux-System starten und den Trace per Remote-Debugging unter macOS oder Windows untersuchen. Führen Sie dazu folgende Schritte aus:

  • Erfassen Sie einen rr-Trace auf dem entfernten System.
  • Starten Sie den Debugger auf dem entfernten System wie folgt: dlv --headless --api-version 2 -l localhost:2345 replay /path/to/trace/dir /path/to/binary.
  • Stellen Sie über die Go Remote-Konfiguration eine Verbindung zum entfernten System her.

Bitte beachten Sie, dass Mozilla rr nur auf Linux-Systemen verfügbar ist.

Neue quick-Fixes

Umgang mit nicht genutzten Parametern

Bei ungenutzten Parametern werden in GoLand jetzt der Name und der Typ hervorgehoben, wenn beide entfernt werden können. Wird der Typ zur Implementierung einer Methodenspezifikation verwendet, so wird nur der Name hervorgehoben.

Im letzteren Fall bietet Ihnen die IDE die Wahl zwischen zwei Quick-Fixes: Delete parameter names oder Rename to_. Dasselbe gilt auch für Funktionen, die als Funktionstyp verwendet werden.

Außerdem können Sie jetzt alle ungenutzten Parameter in einem Zug löschen. Drücken Sie Alt+Enter und wählen Sie den Quick-Fix Delete all unused parameters.

Fehlende Methode oder Funktion erstellen

GoLand bietet einen Quick-Fix zum Erstellen einer fehlenden Funktion oder Methode anhand der Verwendung in einem Funktionstyp. Drücken Sie einfach Alt+Enter und wählen Sie Create method/function.

Verbesserte Handhabung von Paketaliasen

GoLang schlägt jetzt Pakete basierend auf den verwendeten Aliase vor, über Alt+Enter in eine beliebige Datei Ihres Projekts zu importieren.

Refaktorierungen

Signatur ändern

Change Signature zeigt jetzt auf den Registerkarten Parameters und Result parameters immer den Typ der Parameter an. Aktivieren Sie das neue Kontrollkästchen Reuse types, um Folgetypen in der Signature Preview und im Ergebnis auszublenden.

Darüber hinaus gibt es zwei neue Intention-Aktionen: Reuse signature types führt Parameterdefinitionen in einer einzigen Deklaration zusammen, und Expand signature types fügt mittels Alt+Enter Typen zu jedem Parameter hinzu.

Umbenennen und Variable extrahieren

Extract Variable und Rename schlagen jetzt bessere Namen vor. err wird zum Beispiel für Ausdrücke vom Typ error vorgeschlagen, ctx wird für context.Context vorgeschlagen usw.

Optimiertes Verhalten von Codeinspektionen

Die Codeinspektionen von GoLand sind jetzt präziser fokussiert. Wir haben den Hervorhebungsumfang bei erkannten Problemen im Editor reduziert, um Ihnen beim Aufrufen des Kontextmenüs über Alt+Enter die relevantesten Quick-Fixes und Intention-Aktionen für den entsprechenden Codeabschnitt vorschlagen zu können.

Codebearbeitung

Verbesserter Lexer für Runenliterale

Wir haben den Runenliteral-Lexer in mehrfacher Hinsicht verbessert:

  • GoLand erklärt jetzt in Fehlermeldungen klarer und prägnanter, warum der Ausdruck falsch ist.
  • GoLand hebt ein gültiges Runenliteral nach der Bearbeitung nicht als Fehler hervor.
  • Das Parsen von ungültigen Runenliteralen ist jetzt stabiler.

Hervorhebung von Neuzuweisungen

Go erlaubt die Verwendung vorhandener Variablen auf der linken Seite von kurzen var-Deklarationen. Somit können zwei Konstrukte, die identisch aussehen, unterschiedliche Bedeutungen haben. Damit Sie solche Fälle leichter erkennen können, haben wir eine Syntaxhervorhebung für solche Neuzuweisungen eingeführt.

Codefaltung für Verbundliterale

GoLand 2019.3 führt die Codefaltung für Verbundliterale ein. Um das Codefragment ein- oder auszublenden, verwenden Sie die Codefaltungs-Umschaltsymbole oder drücken Sie Strg+NumPad Minus/Strg+NumPad Plus auf Windows bzw. ⌘+NumPad Minus/⌘+NumPad Plus auf macOS. Das Standardverhalten für die Codefaltung können Sie unter Settings/Preferences | Editor | General | Code Folding einstellen.

Erweiterte Live-Vorlagen

Die Live-Vorlagen for und forr schlagen jetzt relevante Variablennamen vor. Für Indizes werden zum Beispiel Namen wie i und index vorgeschlagen; für Werte werden die Namen s, string und line empfohlen.

Die Live-Vorlage meth bietet jetzt Codevorschläge für Empfängernamen.

Navigation

Nur im ausgewählten Bereich suchen

Bei Verwendung von Find (⌘+F auf macOS und Ctrl+F auf Windows/Linux) können Sie die Suche auf den ausgewählten Bereich beschränken. Wählen Sie im Editor einfach den gewünschten Codebereich aus und rufen Sie Find auf – die IDE führt die Suche dann nur im ausgewählten Bereich aus. Um die Suche auf die gesamte Datei auszudehnen, drücken Sie ⌘+F/ Ctrl+F ein zweites Mal, oder schalten Sie die Option In Selection in der Suchleiste um.

„Find in Path“ mit Dateimaske

Wenn Sie den Find in Path-Dialog mit einer Dateimaske verwenden, zeigt GoLand jetzt die Dateierweiterungen in den Suchergebnissen an. Dadurch können Sie leichter zwischen Dateien unterscheiden, die den gleichen Namen, aber eine unterschiedliche Erweiterung besitzen.

Globale Dateiüberwachung für alle neuen Projekte

In GoLand 2018.2 haben wir die Möglichkeit eingeführt, eine Dateiüberwachungskonfiguration mithilfe der Optionen Project und Global in mehreren Projekten zu nutzen. Die Funktion heißt Global File Watcher.

Allerdings musste dieser Watcher für jedes neue Projekt einzeln aktiviert werden. Jetzt haben Sie die Möglichkeit, diesen Watcher in allen Projekten, die Sie neu erstellen oder öffnen, standardmäßig zu aktivieren.

Wählen Sie im File-Menü Other Settings | Settings for new projects aus, und rufen Sie dann Tools | File Watchers auf. Erstellen Sie einen neuen globalen Watcher, und aktivieren Sie das Kontrollkästchen daneben, um den Watcher in allen neuen Projekten zu aktivieren.

Lernprogramme

Oft fehlt einem vor lauter Projektstress die Zeit, zu jedem verwendeten Produkt die What's-New-Seiten, Versionshinweise und Blog-Posts durchzulesen. Abhilfe bieten unsere Tutorials, die Ihnen die Möglichkeit geben, die Funktionen von GoLand in Ihrem eigenen Tempo kennenzulernen.

Sie finden die Tutorials auf dem Begrüßungsbildschirm über Ihrer Projektliste. Alternativ können Sie in jedem Projekt über das entsprechende Toolfenster auf die Tutorials zugreifen.

Der Debugger zeigt nun die Ergebnisse einfacher Implementierungen der Methoden String(), Error() und DebugString() im Bereich Variables an. Er unterstützt Methoden mit einer einzelnen return-Anweisung ohne Aufrufe.

GoLand 2019.2 unterstützt Zahlenliterale und vorzeichenbehaftete Integer als Shift-Anzahlen. Diese Änderungen sind Teil der Schritte, die wir in Richtung Go 2 unternehmen. Weitere Informationen finden Sie im offiziellen Go-Blog.

Benutzerdefinierte Vorlagen für Postfix-Vervollständigung

Mit GoLand 2019.2 können Sie Ihre eigenen benutzerdefinierten Vorlagen für die Postfix-Vervollständigung erstellen.

Navigieren Sie zu Settings/Preferences | Editor | General | Postfix Completion, klicken Sie auf +, wählen Sie Go aus, definieren Sie Ihre Vorlage und wählen Sie die Bereiche aus, in denen die Vorlage anwendbar sein soll.

Weitere Informationen über das Erstellen benutzerdefinierter Vorlagen finden Sie in diesem Artikel.

Darüber hinaus gibt es mit .aappend eine neue vordefinierte Postfix-Vervollständigungsvorlage. Einem Ausdruck wird ein integrierter Funktionsaufruf append vorangestellt, und dessen Ergebnis wird diesem Ausdruck zugewiesen.

Quick-Fixes auf Grundlage von Change Signature

Wir haben viele Quick-Fixes auf Grundlage der Refaktorierung Change Signature hinzugefügt. Beispielsweise können Sie durch einfaches Drücken von Alt+Enter automatisch einen Parameter ableiten und dem Funktionsaufruf hinzufügen oder fehlende Rückgabeparameter in eine Signatur einfügen.

Refaktorierung Extract Method mit Rückgaben

Die Refaktorierung Extract Method kann jetzt mit Codeabschnitten umgehen, die return-Anweisungen enthalten. Wenn der Code immer über Rückgaben beendet wird, wird er so extrahiert, wie er ist. Wenn er jedoch Ausführungspfade ohne Rückgaben enthält, hat die extrahierte Methode ein zusätzliches Boolesches Flag als Rückgabewert. Das Flag wird verwendet, um eine frühe Rückgabe auf der Seite des Aufrufs durchzuführen.

Verbesserungen der Codevervollständigung

Die Codevervollständigung bietet jetzt Vorschläge für Receiver-Member, auch wenn kein Receiver vorhanden ist. Von Receiver-Namen wie this oder self wird abgeraten. Das führt dazu, dass Methoden unterschiedliche Receiver-Namen haben. Mit dieser Codevervollständigungsfunktion können Sie den Unterschied ruhig ignorieren.

Schlauere Go-Module und Dep-Integration

GoLand 2019.2 aktiviert die Go Modules- oder Dep-Integration automatisch, wenn es erkennt, dass Go Modules oder Dep in einem Projekt verwendet werden. Früher wurde die GOPATH-Indizierung deaktiviert und nur eine Benachrichtigung über das Aktivieren der Go Modules- oder Dep-Integration angezeigt. Wir haben jedoch festgestellt, dass dies nicht ausreicht, da es leicht passieren kann, dass die Benachrichtigung übersehen oder ignoriert wird und Projekte so in einen halbkonfigurierten Zustand versetzt werden können.

Wenn dies für Ihre Zwecke nicht geeignet ist, können Sie die Integration jederzeit unter Settings/Preferences | Go | Go Modules (vgo) manuell deaktivieren.

Codeinspektionen

Redundante Typkonvertierungen

Eine Codeinspektion mit der Bezeichnung Redundant type conversions hilft dabei, ein Projekt lesbarer und übersichtlicher zu gestalten. Damit wird vor redundanten Typkonvertierungen gewarnt, die weggelassen werden können.

Validierung von Segmentindizes in Literalen

Wir haben eine Validierung von Segmentindizes in zusammengesetzten Literalen hinzugefügt! Das bedeutet, dass Sie vom Editor gewarnt werden, dass Ihr Index eine nicht negative ganzzahlige Konstante sein muss, bevor Sie Ihr Projekt kompilieren.

Speicher-, CPU-, Mutex- und Block-Profiler

Durch die Integration von Speicher-, CPU-, Mutex- und Block-Profilern haben wir die Tools-Unterstützung von GoLand erheblich erweitert. Sie können das Profiling Ihrer Anwendung direkt aus der IDE heraus starten und die Analyseergebnisse in einem separaten Toolfenster überprüfen (View | Tool Windows | Profiler).

Um den Profiler auszuführen, klicken Sie auf die neue Schaltfläche Run with Profiler [Mit Profiler ausführen] und wählen Sie in der Navigationsleiste die gewünschte Konfiguration aus. Alternativ wählen Sie eine der neuen Aktionen im Run-Menü aus, das beim Anklicken des Run-Symbols in der linken Randleiste geöffnet wird.

Um einen besseren Einblick in die möglichen Performance-Probleme in Ihrem Code zu erhalten, sortieren Sie die Ergebnisse nach den verfügbaren Kriterien für jeden einzelnen Profilertyp, verwenden Sie die Registerkarten Flame Chart [Flammendiagramm], Call Tree [Aufrufstruktur] und Method List [Methodenliste] oder navigieren Sie zum Quellcode, um die Probleme im Detail zu untersuchen.

Smart Step Into

Die neue Debugger-Aktion Smart Step Into [Intelligenter Einzelschritt] erleichtert das Debuggen von Aufrufketten und verschachtelten Methodenaufrufen. Sie können innerhalb eines Ausdrucks in beliebige Aufrufe hineinspringen. Methoden, die Sie nicht interessieren, werden dabei automatisch übersprungen.

Zum Beispiel können Sie beim Debuggen von fmt.Printf("Ich heiße %s und bin %d Jahre alt!", me.FullName(), me.Age()), zielgenau in Age() hineinspringen.

Um Smart Step Into aufzurufen, drücken Sie Umschalt+F7 oder wählen Sie Smart Step Into im Run-Menü aus.

Weitere Informationen finden Sie unter Debugging in GoLand – Verbesserungen in 2019.1.

Neue Refaktorierungen

Extract Interface

Die Refaktorierung Extract Interface [Schnittstelle extrahieren] hilft Ihnen, unkompliziert eine neue Schnittstelle auf Grundlage des ausgewählten Typs zu erstellen. Um die Refaktorierung aufzurufen, rufen Sie Refactor This [Dies refaktorieren] auf, drücken Sie unter Windows oder Linux Strg+Alt+Umschalt+T oder unter macOS ^T und wählen Sie im Popup-Menü den Eintrag Extract Interface [Schnittstelle extrahieren] aus.

Der Refaktorierungsdialog zeigt eine Vorschlagsliste mit Membern, die zur neuen Schnittstelle verschoben werden können. Sie können einen Namen für die Schnittstelle angeben und die Datei, das Verzeichnis oder das Paket auswählen, in dem die Schnittstelle erstellt werden soll. Wenn der Typ bereits Schnittstellen implementiert, wird er automatisch von der Liste ausgeschlossen.

Weitere Informationen finden Sie unter Refactorings in GoLand – Extract Interface.

Leistungsfähigeres Rename

Die Rename-Refaktorierung hilft Ihnen jetzt auch, in Schnittstellen deklarierte Methoden umzubenennen. Beim Umbenennen solcher Methoden bietet GoLand an, ihre Implementierungen ebenfalls umzubenennen.

Der Bereich Refactoring Preview [Refaktorierungsvorschau] im Toolfenster Find [Finden] zeigt, worauf sich die aktuelle Refaktorierung auswirken wird. Dabei werden die Verwendungen nach Typ und Position im Projekt gruppiert.

Nullwertanalyse

Go bietet viele sinnvolle Möglichkeiten, den Nullwert zu nutzen, aber je mehr Möglichkeiten es gibt, desto leichter ist es auch, in eine Falle zu tappen.

Die Codeinspektionen in unserem brandneuen Nilness Analyzer [Nullwertanalyse] erkennen sofort, ob Variablen in verschiedenen Fällen einen Nullwert haben können, und weisen Sie auf potenzielle Fehler und Probleme hin.

Weitere Informationen finden Sie unter Nilness Analyzer Inspection.
