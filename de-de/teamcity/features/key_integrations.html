Wichtige Integrationen

Docker

Erstklassige Unterstützung für Docker bereits im Lieferumfang:

  • Erstellen Sie Docker-Images als separater Schritt im Build-Prozess.
  • Lassen Sie Build-Dienste von Docker Compose automatisch starten, sobald der Build-Prozess beginnt, und nach Abschluss des Builds beenden.
  • Starten Sie einen Build-Schritt innerhalb eines bestimmten Docker-Containers als Erweiterung anderer Runner (Gradle, Maven, Ant und Befehlszeile).
  • Bereinigen Sie alle vom Build veröffentlichten Images.
  • Stellen Sie eine Verbindung zur Docker Registry her.

Issue-Tracker

Die Integration von TeamCity und JetBrains YouTrack geht in beide Richtungen:

  • TeamCity erkennt Ticket-IDs, die in den Änderungen vermerkt sind, und erstellt einen Link zu YouTrack.
  • YouTrack kann so konfiguriert werden, dass Informationen zu Änderungen aus TeamCity importiert und zu Tickets hinzugefügt werden. YouTrack kann auch verschiedene Befehle auf die Tickets anwenden, wenn diese in den Änderungen in TeamCity erwähnt werden.

TeamCity unterstützt standardmäßig Jira und Bugzilla und kann mithilfe von Plugins problemlos in andere Systeme integriert werden.

Maven

TeamCity bringt zahlreiche Features für native Maven-Unterstützung mit, darunter: automatische Einrichtung der Build-Schritte für Maven in der Buildkonfiguration, Freigabe allgemeiner Maven-Einstellungen für mehrere Projekte und Visualisierung verschiedener Projektdaten, die via POM gesammelt wurden.
TeamCity kann außerdem Builds auslösen, wenn sich ein bestimmtes Maven-Artefakt ändert.

Visual Studio Team Services

Dank der Integration von Visual Studio Team Services können Sie VSTS als Ihre Quellcodeverwaltung in TeamCity einrichten (sowohl Git als auch TFVC werden unterstützt), TeamCity-Builds und den Verlauf der Versionskontrolle mit Work Items verknüpfen und den Build-Status für Commits und Pull-Requests in Ihren Git-Repositories abrufen.

NuGet

Dank umfangreicher Integration von NuGet kann TeamCity NuGet-Pakete auf Agents installieren und aktualisieren sowie Pakete erstellen und in einem NuGet-Feed veröffentlichen.
TeamCity kann auch als NuGet-Feed-Server agieren und die Erstellung von Builds anhand von Änderungen in einem bestimmten NuGet-Paket auslösen.
Ab Version 2018.2 können Sie mit TeamCity mehrere NuGet-Feeds angeben, die von den Builds eines Projekts und dessen Unterprojekten verwendet werden sollen.

VCS-Hosting-Services

  • Verbinden Sie TeamCity mit Ihrem Issue-Tracker bei GitHub und BitBucket, um Informationen zu Tickets automatisch abzurufen und anzuzeigen.
  • Veröffentlichen Sie den Build-Status direkt auf GitHub, GitLab, Bitbucket und Upsource nach Fertigstellung des Builds.
  • Erstellen Sie Projekte direkt aus GitHub, Bitbucket, GitLab Repositories, ohne eine URL anzugeben. TeamCity stellt eine Verbindung zu Ihrem VCS her und listet alle verfügbaren Repositories auf. Sie müssen dann nur noch eins auswählen.
