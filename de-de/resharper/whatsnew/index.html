Mehr Dateien für Go to Text

Die Engine hinter Go to Text berücksichtigt buchstäblich alle Textdateien in einer Lösung.

Bisher wurden in Go to Text nur Dateien berücksichtigt, die bereits von ReSharper indiziert wurden. ReSharper indiziert nur Dateien, die Code enthalten oder beim Schreiben von Code helfen können, da die vollständige Indizierung anderer Dateien in den meisten Fällen zu aufwändig ist. Daher wurden alle Dateien mit "unbekannten" Dateierweiterungen nicht berücksichtigt und konnten nicht durchsucht werden.

Wir haben die Engine jetzt aufgerüstet! Während die Ergebnisse indizierter Dateien angezeigt werden, beginnt nun die Suchmaschine im Hintergrund mit der Suche in nicht indizierten Textdateien. Dies ist sehr hilfreich, wenn Sie manche Daten in Dateien mit unterschiedlichen Erweiterungen speichern, z. B.*proj, .yml und .csv .

Die Unterstützung von ReSharper für C# 7.0 wird mit jeder Version besser. In diesem Release-Zyklus haben wir die Art und Weise, wie ReSharper Mustervergleiche durchführt, überarbeitet:

  • Die Postfix-Vorlage Switch ist jetzt für Variablen mit nicht versiegelten Referenztypen verfügbar.
  • Mit der neuen Kontextaktion Generate type patterns können Sie ganz einfach Switch Cases für ausgewählte Typen generieren.
  • Der Quick-Fix Convert to switch wurde komplett überarbeitet, um mehrere if-Anweisungen in eine einzige switch-Anweisung mit Mustern umzuwandeln.

Für out-Variablen kann ReSharper jetzt die Verwendung von var vorschlagen (oder, abhängig von Ihrem Codestil, die Verwendung expliziter Typen). Außerdem wurden Inspektionen und Quick-Fixes hinzugefügt, um out-Variablen anstelle von Variablen einzuführen, die nur für Methodenaufrufe mit out-Parametern verwendet werden.

Unterstützung von C# 7.1

In Visual Studio 2017 15.3 wurde C# 7.1 eingeführt, und ReSharper hat dessen Sprachfunktionen zügig übernommen. In C# 7.1 wird beispielsweise das default-Literal eingeführt, das anstelle des Ausdrucks default(T) verwendet werden kann. Dabei wird der Zieltyp anhand des Verwendungskontexts ermittelt. ReSharper erkennt die default-Literal-Syntax, stellt eine Inspektion bereit, wenn default(T) verwendet wird, und schlägt einen Quick-Fix zum Entfernen redundanter Typangaben vor.

ReSharper unterstützt auch Tupel-Projektionsinitialisierer mit einer Inspektion, die redundante Tupelwertkomponentennamen aufdeckt, wenn sie vom Initialisierer abgeleitet werden können.

Zu guter Letzt versteht ReSharper async main and Mustervergleiche mit Generics.

Neue IEnumerable-Inspektionen

Die Reihe von IEnumerable-Inspektionen in ReSharper wurde erweitert, um neuen Szenarien Rechnung zu tragen:

  • Die Codeinspektion Possible multiple enumeration kann jetzt Typen berücksichtigen, die aus IOrderedEnumerable oder ParallelQuery abgeleitet wurden.
  • Eine neue Codeinspektion erkennt potenziell unbeabsichtigte Umwandlungen von IQueryable zu IEnumerable.

Weitere Codeinspektionen und Kontextaktionen

In C# 7.1 werden die Elementnamen von Wertetupeln abgeleitet und ReSharper hilft Ihnen, redundante Tupel-Komponentennamen zu erkennen.

Der Quick-Fix Introduce auto-property from parameter ermöglichte bereits, eine nur abrufbare Auto-Eigenschaft mit umfangreichen Optionen einzuführen, wie beispielsweise einen privaten Setter hinzuzufügen oder eine öffentlich änderbare Eigenschaft daraus zu machen. Jetzt wurde die Kontextaktion Initialize auto-property from parameter dem Quick-Fix angeglichen und bietet dieselben umfangreichen Optionen.

Nullüberprüfungs-Einstellungen

Es gibt viele Möglichkeiten, auf null-Werte zu prüfen. Was tun, wenn einem die von ReSharper generierten Nullüberprüfungen nicht gefallen?

In ReSharper 2017.2 haben wir eine neue Optionsseite Null checking eingerichtet, auf der die Priorität von null-Überprüfungen festgelegt werden kann, die von Quick-Fixes, Kontextaktionen und Codegenerierungsaktionen verwendet werden. Sie können sogar Ihre eigenen Prüfungen definieren! Wenn Sie die Kontextaktion Check parameter for null verwenden, achten Sie auf das neue Untermenüelement Configure null-check pattern, mit dem Sie direkt auf diese neue Seite zugreifen können, ohne sich durch die ReSharper-Optionen kämpfen zu müssen.

Verbesserungen bei Navigation und Suche

Go to Everything und andere Navigationsaktionen können jetzt Elemente, die Wörter in der Zielabfrage enthalten, in beliebiger Reihenfolge abgleichen, zum Beispiel trifft eine Suche nach exactMatching auf isMatchingExactly zu.

Andererseits gibt es jetzt mit Exact search eine Suche, die genau so funktioniert, wie man es von einer Suchmaschine erwartet: Wenn Sie nach "Collection" suchen, wird Collection zurückgegeben, aber nicht FilteredCollection, IterateCollection() etc. Wenn die exakte Suche verwendet wird, um zusammengesetzte Namen herauszufiltern, können dennoch die Platzhaltersymbole * und ? verwendet werden, um genau so viel Variation zuzulassen, wie nötig ist.

Go to text ist jetzt in Go to Everything integriert. (Strg+T): Wenn kein Code vorhanden ist, der dem Suchausdruck entspricht, sucht ReSharper stattdessen nach Text.

Außerdem können Sie jetzt mithilfe des Punkttrennzeichens nach Erweiterungsmethoden suchen: Klasse.Erweiterungsmethode.

Zu guter Letzt gibt es in ReSharper 2017.2 ein neues Tastenkürzel zum Durchsuchen der Suchergebnisse im Find Results-Fenster. Drücken Sie Umschalt+Enter anstelle von + auf dem Ziffernblock.

Verbesserte <inheritdoc/>-Unterstützung

Im generierten cref-Attribut zeigt ReSharper die Vervollständigungsliste mit allen möglichen Typen an, auf die verwiesen werden kann.

Neue C#-Eingabeunterstützungen

  • Mit ReSharper können Sie Argumente schnell mit den Attributen NotNull oder CanBeNull markieren, ohne den gesamten Attributnamen eingeben zu müssen. Wenn Sie ! oder ? nach einem Typnamen, einem Namen in einem Parameter oder einer Elementdeklaration eingeben, wird jeweils die Annotation NotNull oder CanBeNull eingefügt. Verbesserungen der automatischen Erkennung von Einrückungen und des Codestils

Die Funktion Layered settings von ReSharper ist ein hervorragendes Tool, um einen einheitlichen Codestil beizubehalten, wenn die meisten Ihrer Teammitglieder ReSharper verwenden. Nun wurde diese Funktion für alle unterstützten Sprachen um Einstellungen für Tabs und Einrückungen erweitert.

Wenn Sie einen Teil des Codes in einer Datei neu formatieren oder wenn der Code beim Bearbeiten oder Einfügen automatisch formatiert wird, kann ReSharper Einrückungen basierend auf den vorhandenen Einrückungen in dieser Datei berechnen und anwenden. Diese Funktion ist nützlich, wenn Sie zu vorhandenen Projekten beitragen möchten, bei denen die Einrückung von Ihren Einstellungen abweicht.

Die kontextbezogene Konfiguration von Formatierungsregeln ist in C#, C++, JavaScript und TypeScript verfügbar. Drücken Sie Alt+Enter über dem ausgewählten Codeblock, den Sie neu formatieren möchten, und wählen Sie dann in der Aktionsliste Format Selection | Configure aus, um alle Formatierungsregeln anzuzeigen, die sich auf den Code auswirken.

Codebereinigung, neue Codestil- und Formatierungsoptionen
