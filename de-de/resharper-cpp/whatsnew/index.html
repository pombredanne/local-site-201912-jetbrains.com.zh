Das ist neu in ReSharper C++

ReSharper C++ 2019.2 umfasst Dutzende Verbesserungen in den Bereichen Codevervollständigung, Navigation und Codeanalyse. Hinzu kommt eine völlig neue Präprozessor-Engine, die jetzt in der Indizierungsphase eingesetzt wird und bei typischen Lösungen eine um 15 bis 25% schnellere Indizierung ermöglicht. Darüber hinaus unterstützt diese Version weitere C++20-Funktionen, um mit den neuesten Sprachmerkmalen Schritt zu halten. Und für Unreal-Engine-4-Projekte gibt es eine ausgefeiltere Unterstützung für RPC- und Reflection-Makros.

ReSharper C++ 2019.2

Schnellere Starts

In dieser Version haben wir fertiggestellt, woran wir in den letzten Releasezyklen gearbeitet haben, und können nun offiziell bekanntgeben, dass in ReSharper C++ eine neue Präprozessor-Implementierung verwendet wird. Tatsächlich gibt es jetzt zwei davon - eine neuere, die für die Indizierung verwendet wird, und eine ältere, die für alles andere verwendet wird. Diese neue Version ermöglicht eine um 15-25% schnellere Indizierung von typischen Lösungen wie LLVM oder Unreal Engine.

Codevervollständigung

Bei der Codevervollständigung zeigt ReSharper C++ für jedes Element der Vervollständigungsliste einen Zusammenfassungs-Tooltip an (gesteuert in den ReSharper-Optionen auf der Seite "Environment | IntelliSense | Completion Appearance" mit dem Kontrollkästchen "Show summary"). Die Beschreibung für das ausgewählte Vervollständigungselement ist jetzt farbig und zeigt die Dokumentation für das entsprechende Element an.

Code-Hinweise

Der Editor verfügt über noch mehr Code-Hinweise, um die Lesbarkeit des Codes zu verbessern. Für die Präprozessordirektiven sowie für Funktionsargumente, die per nicht konstante Referenz übergeben werden, wurden neue Hinweise hinzugefügt. Hinweise in aggregierten Initialisierungsausdrücken zeigen nun die Namen der initialisierten Basisklassen an.

Mehr Unterstützung für C++20

Der C++20-Standard steht kurz vor der Unterzeichnung durch das C++-Komitee. Und natürlich arbeiten wir hart daran, eine vollständige Unterstützung zu implementieren. ReSharper C++ 2019.2 unterstützt Folgendes:

  • konditionales explicit,
  • neuer char8_t-Typ,
  • Paketerweiterung in Lambda Init-Capture,
  • standardmäßig konstruierbare und zuweisbare zustandslose Lambdas,
  • C++20-Regeln für Aggregate,
  • Spezifizierer consteval und constinit,
  • C++20-Verbesserungen für strukturierte Bindungsdeklarationen.

Eingabeunterstützung

Die Eingabeunterstützung für mehrzeilige Kommentare wurde verbessert. Erstens werden mehrzeilige Kommentare mit einzeiliger Syntax automatisch erweitert. Zweitens fügt das Erweitern mehrzeiliger Kommentare jetzt den passenden Einzug ein, wenn „Virtual space“ aktiviert ist.

Das Umbrechen von Codeblöcken mit geschweiften Klammern ist jetzt noch einfacher - wählen Sie einfach einen Codeblock oder einen Ausdruck aus und geben Sie eine öffnende geschweifte Klammer ein. ReSharper C++ erledigt den Rest für Sie:

Formatierer

In ReSharper C++ 2019.2 gibt es eine ganze Reihe neuer Optionen für den Formatierer:

  • Verwenden einer fortlaufenden Zeileneinrückung in Funktionsdeklarations- und Aufrufklammern.
  • Umbruch bevorzugen vor ")" in Deklaration.
  • Umbruch bevorzugen vor ")" in Aufruf.
  • Verwenden einer fortlaufenden Zeileneinrückung in Initialisiererlisten.
  • Ausrichten von Argumenten in Initialisiererliste.
  • Umbruch bevorzugen nach "{" in Initialisiererlisten.
  • Umbruch bevorzugen vor "{" in Initialisiererlisten.

Codeanalyse

Neue Inspektionen stehen zur Verfügung, um mehr Arten von Code-Smell aufzufangen:

  • Präprozessor-Direktive ohne passendes Gegenstück.
  • Eine spezielle Member-Funktion mit explizitem „default“ wird implizit gelöscht.
  • Redundanter Funktionsspezifizierer final in einer finalen Klasse.

Der Fix Use static_cast für die Inspektion C-style cast used kann in großem Umfang und als Teil der Codebereinigung ausgeführt werden.

Navigation

Code-Navigationsaktionen wurden zielgerichtet aktualisiert:

  • Go to declaration kann zum Standardkonstruktor navigieren, der aufgerufen wird, wenn ein Objekt in Variablendeklarationen oder neuen Ausdrücken standardmäßig initialisiert wird.
  • Find usages findet Verwendungen von Standardkonstruktoren in der Standardinitialisierung.
  • Go to symbol verbirgt jetzt Symbole mit qualifizierten Namen (wie zum Beispiel außerhalb der Klasse liegende Funktionsdefinitionen).

Außerdem können Sie direkt über das Dialogfeld Go to File member in einer Datei suchen. Die Ergebnisliste enthält die Namen aller passenden Member und alle Stellen, an denen die Suchzeichenfolge in der Datei auftritt.

Unreal Engine

In der vorherigen Version haben wir begonnen, an einer besseren Unterstützung für UE4-spezifische Remoteprozeduraufrufe zu arbeiten. Jetzt haben wir die Refaktorierungen Rename und Change Signature aktualisiert, damit sie für RPC-Funktionen ordnungsgemäß funktionieren. Für UE4-basierten Code stehen ebenfalls einige neue Codeanalyse-Prüfungen zur Verfügung, insbesondere für die fehlenden oder falsch eingestellten UE4-Reflection-Makros.

Nach dem Hinzufügen der Codevervollständigung für Reflection-Bezeichner in Version 2019.1 haben wir jetzt eine integrierte Dokumentation dafür hinzugefügt. Diese wird im Schnelldoku-Popup (Strg+Q) angezeigt, wenn sich der Textcursor auf einem Bezeichner befindet.

Weitere Features

ReSharper C++ 2019.2 bietet Erstunterstützung für den in Visual Studio 2019 eingeführten Modus „Per-Monitor DPI Awareness“. Wir haben auch die Seite ReSharper | Options | Environment | Performance Guide aktualisiert, die jetzt einen Abschnitt zu Windows Defender sowie Beschreibungen zu allen Performance-Tipps enthält, in denen erläutert wird, was genau getan wird, nachdem ein Vorschlag angewendet wurde.

Gute Neuigkeiten gibt es auch für Plugin-Autoren: C++-Assemblys von ReSharper C++ sind jetzt im ReSharper-SDK enthalten!

ReSharper C++ 2019.1

Optimierung der Leistung

ReSharper C++ 2019.1 bringt viele Leistungsoptimierungen mit, um die Indexierung zu beschleunigen und die Speicherauslastung zu senken. Die Zeit, die sowohl für den Warmstart großer Projektmappen als auch für den Kaltstart benötigt wird, wurde reduziert. Die Optimierungen der Speichernutzung reduzieren sowohl den Speicherverkehr während der ersten Indexierung als auch den anschließenden Speicherverbrauch, wodurch die Reaktionsfähigkeit des Editors verbessert wird. Darüber hinaus bringt die Erweiterung eine umfangreiche Leistungsoptimierung der Navigation im Code sowie der Codevervollständigung mit.

Zusätzlich zu Speicheroptimierung, verbesserten Algorithmen bei der Indexierung von Code sowie schnellerer Serialisierung von Cache gibt es jetzt auch einige Sondereinstellungen für Projekte der Unreal Engine. Dazu gehört das standardmäßige Überspringen von Plug-ins und Drittanbieter-Code während der Indexierung eines Projekts von Unreal Engine 4. Die Optionen, die sich auf die Codeindexierung in Projektmappen von Unreal Engine 4 beziehen, sind auf einer neuen Seite mit Optionen unter „ReSharper | Options | Code Editing | C++ | Unreal Engine“ zusammengefasst.

Unterstützung für die Unreal Engine

Namenskonventionen der Unreal Engine

Benennungsregeln tragen zur Lesbarkeit von Code bei. Es wundert also nicht, dass Epic Games einen Unreal Engine 4 Coding-Styleguide veröffentlicht hat, der sich eingehend mit Namenskonventionen befasst. Epic Games empfiehlt Spielentwicklern für die Unreal Engine die Einhaltung dieses Styleguides. ReSharper C++ 2019.1 berücksichtigt diese Namenskonventionen in allen Aktionen, einschließlich in den Refaktorierungen „Introduce Variable“ und „Rename“, der Codegenerierung von Gettern und Settern und in vielen anderen Bereichen.

ReSharper C++ analysiert außerdem die Benennungen, die Sie in Ihrer Codebasis verwenden. Alle Inkonsistenzen bezüglich Namenskonventionen in UE4 werden nach der Ausführung der Inspektionen Inconsistent UE4 naming hervorgehoben. Wenden Sie den entsprechenden Quick-Fix an, um für ein solches Symbol die komplette „Rename“-Refaktorierung zu starten.

Vervollständigung für Reflexionsspezifizierer

Um Ihnen die Arbeit mit Reflexionsmakros in Unreal Engine zu erleichtern, bietet ReSharper C++ die Vervollständigung für Reflexionsspezifizierer innerhalb von Reflexionsmakros:

Verwendung von Remoteprozeduraufrufen

ReSharper C++ 2019.1 identifiziert RPCs anhand der Schlüsselwörter Client, Server oder NetMulticast in der Deklarationsanweisung UFUNCTION. ReSharper C++ weiß bei solchen Funktionen, dass die Funktion _Implementation generiert werden muss; dasselbe gilt für die Funktion _Validate, wenn WithValidation angegeben wurde. Es kann beide generieren, wenn sie fehlen sollten, oder nur eine, wenn lediglich eine fehlt.

Beim Navigieren zu einer Definition werden beide Funktionen (_Validate und _Implementation) vorgeschlagen. Alternativ stehen diese Optionen im Menü Alt+Enter ebenfalls zur Verfügung, und zwar als „Related UE4 functions“. Mehr erfahren.

Auto-Import berücksichtigt generierte Header

Wenn Sie ein Symbol verwenden, das im aktuellen Bereich nicht verfügbar ist, hebt ReSharper C++ dieses in roter Schrift hervor und schlägt vor, das fehlende #include hinzuzufügen, falls möglich. Wenn das #include in einer Header-Datei fehlt, wird es vor dem .generated.h eingefügt, was das letzte in der Liste eingeschlossener Elemente sein sollte. Das fehlerhaft platzierte #include wird durch die entsprechende Inspektion hervorgehoben.

Zusätzlich dazu schlägt der Auto-Import nicht mehr Symbole aus den Ordnern Private und Classes Ihres UE4-Projekts vor.

C++-Sprache: C++17 und C++20

Das Verständnis von C++ wurde optimiert:

  • Seit C++17 haben wir die Unterstützung für hexadezimale Gleitkommaliterale hinzugefügt.
  • Seit C++20 gibt es grundlegende Unterstützung für Contracts und einen Dreiweg-Vergleichsoperator. Dazu ist präzises Parsing und teilweises Auflösen nötig, um offensichtliche falsch-positive Meldungen im Code zu vermeiden.
  • Die bekannte Vorlagensyntax von C++20 für generische Lambdas wurde hinzugefügt.
  • Die alternative Repräsentation von Operatoren (in Textform) wird jetzt unterstützt.

C-Deklarationen im K&R-Stil

ReSharper C++ kann jetzt C-Deklarationen im K&R-Stil korrekt verarbeiten. Wenn Sie also eine Legacy-Bibliothek in diesem Stil haben oder ihn aus bestimmten Gründen bevorzugen, wird ReSharper C++ ihn jetzt berücksichtigen:

Clang-Tidy

Schon seit einiger Zeit ermöglicht ReSharper eine nahtlose Integration mit Clang-Tidy, einem leistungsfähigen Opensource-Tool zur Codeanalyse, das auf dem Clang-Compiler basiert. In v2019.1 haben wir die integrierte Clang-Tidy-Binärdatei auf Clang 8 aktualisiert, um Ihnen viele neue Codeüberprüfungen und die entsprechenden Fixes zur Verfügung zu stellen.

Die Optionen zu Clang-Tidy sind auf einer neuen Seite mit Optionen unter „Options | Code Editing | C++ | Clang-Tidy“ zusammengefasst. Listen Sie die Prüfungen, die Sie aktivieren bzw. deaktivieren möchten, mithilfe eines einfachen Textformulars auf, geben Sie eine Clang-Tidy config-Datei an oder verwenden Sie anstelle der mitgelieferten Ihre eigene Clang-Tidy-Binärdatei.

Navigation

Um die Reaktionsfähigkeit des Editors zu verbessern, haben wir in diesem Release die Aktionen Search everywhere und Go to type beschleunigt. Die Suchen Go to file member, Go to base class und Go to derived class können schon während der Indexierung verwendet werden.

Die Performance von Go to Everything wurde ebenfalls in vielerlei Hinsicht optimiert, unter anderem in Bezug auf Gesamtleistung und Suche nach Mustern mit :: oder Leerzeichen.

Wenn Sie Go to File Member aufrufen, können Sie nun auch Basismember in die Liste aufnehmen und zu ihnen navigieren.

Namenskonventionen

Wir haben die Seite mit den Einstellungen der C++-Namenskonventionen (ReSharper Options | Code Editing | C++ | Naming Style) überarbeitet. Jetzt stehen Ihnen zwei separate Listen für Spezifizierer und Entitätstypen (als Baumstruktur) zur Verfügung, die Sie kombinieren können. Darüber hinaus können Sie Regeln hinzufügen, die für eine oder mehrere Entitätstypen spezifisch sind.

Um diese Konfiguration noch flexibler zu gestalten, wurden zahlreiche neue Entitätstypen eingeführt:

  • Template-Parameter wurden in Typ- und Nichttyp-Template-Parameter eingeteilt.
  • Parameter wurden in Funktions- und Lambdaparameter unterteilt.
  • Enumeratoren wurden in Enumeratoren mit und ohne Bereichseinschränkung eingeteilt.

Formatierer

Die Formatierungseinstellungen umfassen mehrere neue Optionen und Verbesserungen:

  • Um den Formatter noch flexibler zu machen, haben wir die Formatierungsoptionen anhand der Verwendung von Leerzeichen um Referenzen und Zeiger eingeteilt. Es gibt jetzt zwei Sets an Optionen für jeden Fall: So können Sie z. B. im Rückgabetyp einer Funktion ein Leerzeichen vor einer Referenz angeben, aber dieses vor dem Zeiger weglassen.
  • Der Umbruch vor den Kommas einer Basisklassenliste ist jetzt möglich.
  • Viele der neuen ClangFormat-Optionen werden von ReSharper C++ unterstützt, z. B. CompactNamespaces, AllowShortBlocksOnASingleLine und AllowShortFunctionsOnASingleLine, AccessModifierOffset, BreakBeforeBraces: Custom und viele mehr.
  • In den Einstellungen können Sie jetzt sehen, welche Formatierungsoptionen von einer bestimmten ClangFormat-Option betroffen sind.

Dokumentation

Wenn Sie einen Dokumentationskommentar in Ihrem Code eingeben, wird die Eingabehilfe in jeder neuen Zeile automatisch /// einfügen, sobald Sie die Eingabetaste drücken. Wenn eine Funktion einen XML-Dokumentationskommentar besitzt und ihr Parameter mithilfe der „Rename“-Refaktorierung umbenannt wird, dann wird das entsprechende Tag <param> im Dokumentationskommentar ebenfalls aktualisiert. Und das ist noch nicht alles: Wenn Sie den Parameter manuell umbenennen, hebt ReSharper C++ den Konflikt hervor und schlägt vor, ein neues <param>-Tag hinzuzufügen oder das vorhandene umzubenennen.

Unit-Tests

Die Unterstützung von Unit-Tests wurde um Folgendes erweitert:

  • Testdeklarationen TEMPLATE_PRODUCT_TEST_CASE von Catch 2.7 und
  • das Makro GTEST_SKIP() von Google Test.

Zusätzlich haben wir die Anzahl der unterstützen Frameworks erhöht (bisher hatten wir Google Test, Boost.Test und Catch) und die Unterstützung für doctest hinzugefügt. Dieses relativ neue Framework ist leichtgewichtig und sehr flexibel. Es erlaubt das Schreiben von Tests direkt im Produktionscode und bietet zugleich einen Weg, alle testbezogenen Zeilen aus der Binärdatei zu entfernen.

Refaktorierungen und Code-Tweaks

Die Refaktorierungen von ReSharper haben jetzt ein noch tieferes Verständnis für das moderne C++. Wenn Sie die Refaktorierung Change Signature verwenden, können Sie eine Funktion als constexpr, const oder noexcept deklarieren und die ref-Qualifizierer ändern.

ReSharper C++ 2018.3 bringt außerdem zahlreiche neue Kontextaktionen mit, um Sie bei der Optimierung Ihrer Codebasis zu unterstützen:

  • Es ist möglich, eine Funktionsimplementierung zu generieren und sie in die Zwischenablage zu kopieren, anstatt sie im Dokument einzufügen.
  • Es ist möglich, eine Funktion zu generieren, die enum zu string konvertiert (anpassbar via Live-Template).
  • Es ist möglich, einen regulären Rückgabetyp in einer Funktionsdeklaration zu einem nachstehenden Rückgabetyp zu konvertieren und umgekehrt.
  • Es ist möglich, Template-Argumente einzufügen, die mithilfe der Ableitung von Argumenten des Klassen-Template abgeleitet wurden.

Navigation

Go to declaration funktioniert nun für nicht abhängige decltype-Spezifizierer und Lambdas (mit dem Lambda-Text als Ziel). Das ist sogar während der ersten Projektindexierung möglich, was Ihnen noch mehr Zeit spart!

Search Everywhere, eine universelle Suchaktion, hat einige neue Filter erhalten. So gibt es jetzt z. B. /gf für globale C++-Funktionen, /gv für globale C++-Variablen und /d für Makrodefinitionen.

Aber die beste aller Funktionen ist vermutlich die Navigation zu Signaturen ohne Entsprechung. Wenn z. B. Go to declaration für die geänderte Deklaration einer Memberfunktion aufgerufen wird, navigiert die Aktion zu ihrer Definition mit der alten Signatur.

ReSharper C++ 2018.2

C++/CLI-Erstunterstützung C++/CLI-Erstunterstützung Ultimate

C++/CLI ist ein Satz von Spracherweiterungen für C++ von Microsoft, die eine einfache Interoperabilität zwischen verwaltetem und nativem Code fördern. Mit einer umfassender Unterstützung sowohl für verwaltete Sprachen als auch für C++ ist es nur natürlich, dass ReSharper auch beim Schreiben von Code in C++/CLI hilft.

Angesichts des Umfangs an syntaktischen Neuzugängen und Änderungen an der Semantik von C++ ist die Handhabung von C++/CLI keine einfache Aufgabe. Doch das Warten hat sich gelohnt: ReSharper C++ 2018.2 macht die C++/CLI-Unterstützung öffentlich und aktualisiert viele wichtige Features (darunter Codeinspektionen, Eingabehilfe, Formatierung und Codevervollständigung), um diesen Dialekt zu unterstützen.

Bitte beachten Sie, dass wir weiterhin an der C++/CLI-Unterstützung arbeiten und dass sprachübergreifende Navigation und Refaktorierungen kontinuierlich überarbeitet werden. Da wir immer mehr Bereiche abdecken, sind wir auf Ihr Feedback angewiesen: Lassen Sie uns wissen, was noch fehlt und worauf wir uns konzentrieren sollten.

Die Unterstützung für C++/CLI erfordert ein Abonnement von ReSharper Ultimate und dass das Produkt ReSharper aktiviert ist. Weitere Informationen dazu finden Sie im Artikel „ ReSharper C++ mit C++/CLI-Unterstützung“.

Features von C++17

Ein weiterer Bereich, in den wir viel Arbeit investieren, ist die Unterstützung für C++17. ReSharper C++ 2018.2 versteht jetzt die folgenden neuen Features:

  • Ableitung von Argumenten des Klassen-Template und benutzerdefinierte Ableitungsführungen
  • Fold-Ausdrücke
  • Auto-Nichttyp-Template-Parameter
  • Paketerweiterungen in using-Deklarationen
  • Using-Deklarationen mit mehreren Deklaratoren
  • Garantierte copy-Elision
  • Aggregate mit Basisklassen
  • Unterstützung der Typeigenschaft std::is_aggregate
  • Präprozessormakro __has_include

Alles in allem bedeuten diese Updates, dass die meisten C++17-Features jetzt von ReSharper C++ unterstützt werden. Die zwei größten Punkte, die noch fehlen, sind hexadezimale Gleitkommaliterale und noexcept als Teil des Funktionstyps.

Funktionen von C++20

Der C++20-Standard entwickelt sich immer weiter, und alle wichtigen Compiler übernehmen seine Funktionen. Auch ReSharper C++ hält Schritt mit der Evolution der Sprache und unterstützt zahlreiche Features des kommenden Standards:

  • Koroutinen (gemäß n4760 Coroutines TS).
  • Designierte Initialisierung.
  • Bereichsbasierte for-Anweisungen mit Initialisierer
  • Feature-Test-Makros
  • Standard-Memberinitialisierer für Bitfelder

Integrierte Rechtschreibprüfung mit ReSpeller

ReSharper C++ 2018.2 bietet eine integrierte Rechtschreibprüfung in Form der kürzlich hinzugefügten ReSpeller-Erweiterung, die aktualisiert wurde, um C++ zu unterstützen. Wie auch bei der Codeanalyse sucht ReSpeller nach Rechtschreib- und Tippfehlern in Bezeichnern, Kommentaren und Stringliteralen und bietet Quick-Fixes zum Lösen solcher Probleme.

Wenn Sie einen Rechtschreibfehler finden, können Sie das Menü über Alt+Eingabe aufrufen. ReSpeller bietet Ihnen dann mehrere Optionen: Rechtschreibfehler beheben oder Wort zum Wörterbuch hinzufügen. Standardmäßig besitzt die Rechtschreibprüfung ein integriertes Wörterbuch für Englisch (USA), es können jedoch problemlos weitere Sprachen hinzugefügt werden.

Eine Anleitung zum Hinzufügen eines neuen Wörterbuchs und weiteren Informationen zu ReSpeller finden Sie hier: Integrierte Rechtschreibprüfung mit ReSpeller in ReSharper und Rider.

Format-Inspektionen

Manchmal möchten Sie direkt im Code sehen, wenn Sie eine Formatierungsregel nicht beachtet haben. Die Codeformatierungs-Engine von ReSharper bringt deshalb Inspektionen und Quick-Fixes für Formatprobleme mit: Inkonsistente Einrückungen, Zeilenumbrüche und Leerzeilen können nun direkt im Code-Editor angezeigt und behoben werden.

Standardmäßig sind die Format-Inspektionen deaktiviert, um Ihr Sichtfeld nicht unnötig zu überladen. Rufen Sie bei Bedarf ReSharper | Options | Code Inspection | Inspection Severity | C++ | Formatting auf, um alle verfügbaren Inspektionen anzuzeigen und die Schweregrade der für Sie relevanten Inspektionen auszuwählen.

Weitere Informationen finden Sie unter: Maintaining consistent code style with formatting inspections.

Andere Änderungen

Wie immer bringt ReSharper C++ 2018.2 noch eine ganze Reihe kleinerer Verbesserungen in verschiedenen Bereichen mit:

  • Bei Berechnungen von Größe und Ausrichtung berücksichtigt die Codeanalyse-Engine Bitfeldgrößen, alignas-Spezifizierer und das Attribut __decltype(align).
  • Parameter Info bietet Details zu den Membern der Aggregatklasse während der Aggregatsinitialisierung.
  • Kommentare in der Codedatei, die die Formatierungseinstellungen überschreiben sollen, können direkt über den Dialog Configure Format generiert werden.
  • Bereichsbasierte for-Anweisungen bieten Referenzen via Doppelpunkt, um die Navigation zu den in der Schleife verwendeten Funktionen begin() und end() zu erleichtern.
  • Es gibt jetzt separate Farbeinstellungen für statische Klassenfelder und statische Memberfunktionen.
  • Die Angabe von erwarteten Fehlern wird in Unit-Tests mit Boost.Test unterstützt.
  • Die Performance in vielen Bereichen von ReSharper wurde optimiert – von der Beschleunigung der ClangFormat-Unterstützung bis hin zur Reduzierung der Ladezeiten von Projektmappen. Nähere Informationen finden Sie in unseren Blogbeiträgen zum Thema Performance, die sich mit dem Release von ReSharper 2018.2 befassen.

ReSharper C++ 2018.1

Step-Filter für Debugging

Step-Filter sind der aktuellste Neuzugang zum Feature-Set, das ReSharper C++ für das Debugging bereitstellt. Wenn Sie Step Into aufrufen, wird der Debugger von Visual Studio normalerweise zu allen Funktionen, die von der aktuellen Anweisung aufgerufen werden, hinspringen. Allerdings gibt es häufig Funktionen, die trivial sind oder bereits getestet wurden und die Sie deshalb vorzugsweise überspringen möchten. Hier kommen Step-Filter ins Spiel: Mithilfe dieses Features können Sie Funktionen angeben, die der Debugger immer auslassen soll.

ReSharper C++ bringt eine vorgefertigte Sammlung an Step-Filtern aus der Standardbibliothek mit, aber Sie können natürlich auch weitere hinzufügen:

  • Sie können Step-Filter inspizieren, konfigurieren, aktivieren und deaktivieren. Öffnen Sie dazu die Optionen unter Tools | Debugging | C++, um eine Liste mit allen verfügbaren Step-Filtern zu erhalten.
  • Wenn Sie eine Funktion schrittweise ausführen, die Sie in Zukunft überspringen möchten, können Sie diese Funktion mithilfe einer Kontextaktion schnell zur Liste mit Step-Filtern hinzufügen. Für Template-Funktionen gibt es separate Kontextaktionen, die entweder nur die aktuelle Instanziierung oder alle Instanziierungen der Funktion hinzufügen.

Weitere Informationen dazu finden Sie im Artikel „ Step-Filter in ReSharper C++“.

Includes-Analyzer

Angemessene Kompilierzeiten stellen eine der größten Herausforderungen für große C++-Projekte dar. ReSharper C++ 2018.1 stellt den Includes Analyzer vor, ein neues Tool für Codeinspektionen, mit dem Sie messen können, welche Header-Dateien in Ihrem Projekt in Bezug auf die Menge an Codezeilen die größte Auswirkung auf den Kompiliervorgang haben. Um es zu verwenden, rufen Sie eine der Aktionen ReSharper | Inspect | Analyze Includes in … auf oder wählen Sie Analyze Includes vom Kontextmenü im Projektmappen-Explorer auf. ReSharper C++ wird die Dateien im vorgegebenen Bereich analysieren und in einem speziellen Toolfenster einen Bericht ausgeben.

Der Bericht listet die Dateien nach Anzahl an Codezeilen sortiert auf, die sie zusammen mit den von ihnen transitiv eingeschlossenen Dateien beitragen. Anschließend können Sie die Abhängigkeiten näher betrachten. Verwenden Sie dazu entweder die Strukturansicht Includees, bei der untergeordnete Knoten den Dateien entsprechen, welche die Datei im übergeordneten Knoten einschließen, oder die Strukturansicht Includers, bei der untergeordnete Knoten die eingeschlossenen Dateien darstellen. Mithilfe dieser beiden Ansichten werden Sie in der Lage sein, die Header-Dateien zu identifizieren, die am meisten beitragen, und können dann alle nicht benötigten #include-Direktiven entfernen.

Verbesserungen bei der Navigation

Der Dialog Go to File Member wurde neu gestaltet. Klassenmember werden nun anhand ihrer enthaltenden Klasse in der Ergebnisliste gruppiert. Darüber hinaus werden aktive Dateimember ohne Suche nach der Reihenfolge ihrer Deklarationen sortiert, und die Liste mit dem Bildlauf kann noch weitere Member aufnehmen. Diese Änderungen tragen insgesamt zur Benutzerfreundlichkeit des Dialogs bei, da sie einen besseren Einblick in die Struktur der aktuellen Datei gewähren.

In allen Go to-Dialogen und vielen anderen Bereichen versucht ReSharper C++ nun, zu lange Symbolnamen zu kürzen, indem die Funktions- und Template-Parameter ausgeschnitten werden.

Wenn Sie eine Suche im Dialog Recent Files durchführen, werden an die Liste mit Ergebnissen nach den Elementen von Recent File auch die Suchergebnisse für Go to File angehängt.

Und zu guter Letzt ist die Option zum Speichern der letzten Suche im Dialog Search Everywhere jetzt standardmäßig aktiviert. Das frühere Verhalten können Sie wieder über die Einstellung Environment | Search & Navigation | Remember last search aktivieren.

Code-Cleanup über die Befehlszeile

Die ReSharper Command Line Tools sind eine kostenlose Sammlung an Tools, die über die Befehlszeile oder als Teil der kontinuierlichen Integration genutzt werden können. Diese Tools erlauben Ihnen, den Code in Ihren C++-Projekten aufzuräumen. Starten Sie cleanupcode.x86.exe in Ihrer Projektmappe, um Ihren Code automatisch neu zu formatieren, geläufige Redundanzen zu löschen sowie Clang-Tidy Fix-Its und häufige Codestil-Fixes anzuwenden. Auf diese Weise eliminieren Sie Verstöße gegen den Codestil in einem Projekt oder einer Projektmappe und gewährleisten eine einheitliche Codebasis.

In diesem Release wurden außerdem Scheinfehler behoben, die das Tool InspectCode für x64-Projekte ausgab. Zusätzlich dazu kann TeamCity 2018.1 InspectCode für C++-Projekte ausführen (weitere Informationen unter TW-48978).

Weitere Informationen dazu finden Sie im Artikel Code-Cleanup mit den Befehlszeilentools von ReSharper.

Codeinspektionen

ReSharper C++ bietet jetzt folgende neue integrierte Inspektionen:

  • Eine neue Inspektion (mit entsprechendem Fix und einer Code-Cleanup-Option), die vorschlägt, if-Anweisungen mir if constexpr-Anweisungen zu ersetzen, wenn die Bedingung ein konstanter Ausdruck mit Auswertung zur Kompilierzeit ist.
  • Eine Reihe an Inspektionen, mit denen die Verwendungen gelöschter Funktionen gefunden werden.
  • Die Inspektionen für Kontrollfluss und Überprüfung von Verwendungen wurden überarbeitet und funktionieren nun auch innerhalb vom Lambdas.

Weitere Methoden, um Inspektionsschweregrade zu konfigurieren

In ReSharper С++ 2018.1 können .editorconfig-Dateien jetzt dazu verwendet werden, um die Schweregrade von Inspektionen für beliebige Bereiche festzulegen: für die gesamte Projektmappe, ein Projekt oder eine bestimmte Datei. Darüber hinaus können die Einstellungen der Schweregrade aus der DotSettings-Datei des Projekts gelesen werden. Das bedeutet, dass verschiedene Projekte unterschiedliche Sets an aktivierten/deaktivierten Inspektionen oder verschiedenen Schweregraden von Inspektionen haben können.

Aktivieren Sie die Option Read settings from EditorConfig and project settings auf der Seite Code Inspection | Settings | General, um diese Funktion zu aktivieren.

Weitere Informationen dazu finden Sie im Artikel Konfiguration der Schweregrade von Inspektionen mit EditorConfig.

Updates der Clang-Tidy-Integration

Die Clang-Tidy-Integration wurde in vielerlei Hinsicht verbessert:

  • Die mitgelieferten Clang-Tidy-Dateien wurden auf Version 6.0 aktualisiert und bieten jetzt mehrere neue Überprüfungen.
  • Eine neue Inspektionsaktion ermöglicht Ihnen, in der aktuellen Zeile eine bestimmte Clang-Tidy-Prüfung anhand ihres Namens zu deaktivieren.
  • Wenn die ausführbare Datei von Clang-Tidy abstürzt, werden keine Systemfehlerdialoge mehr angezeigt.

Andere Verbesserungen

Zu anderen Änderungen an ReSharper C++ gehören:

  • Die Präprozessordirektive #include_next wird jetzt unterstützt, um die Kompatibilität mit Compilern mit GNU-Erweiterung sicherzustellen.
  • Die Eingabehilfe in C++-Dateien löscht automatisch nachstehende Leerzeichen nach Eingabe, um Ihren Code von überflüssigen Leerzeichen freizuhalten.
  • Wenn Sie einen Ausdruck auswählen und das Fenster Quick Documentation öffnen, sehen Sie verschiedene Informationen zum gewählten Ausdruck wie Typ, Wert und Wertkategorie.
  • Die neue Einstellung Tools | Unit Testing | C++ Tests | Use command-line arguments … bestimmt, ob ReSharper C++ beim Ausführen von Unit-Tests den Wert der Projekteigenschaft Debugging | Local Windows Debugger | Command Arguments verwenden soll.
  • Die Performance des Pop-ups mit den Vorschlägen zur Vervollständigung wurde verbessert.

ReSharper C++ 2017.3

Debugger-Funktionen

ReSharper С++ 2017.3 bringt den Debugger zum Editor: Lokale Variablen, Funktionsparameter, Ausdrücke der aktuellen Zeile und Funktionsrückgabewerte werden jetzt inline neben dem entsprechenden Code angezeigt. Zusätzlich dazu bringt ReSharper C++ die neue Aktion Go to Breakpoint (Strg+Alt+F9) und das Toolfenster R# Breakpoints mit, um schnell zwischen Haltepunkten in der Projektmappe zu navigieren, und bietet jetzt die Möglichkeit, Haltepunkte und Ablaufverfolgungspunkte über das Toolfenster Find Results zu erstellen.

ReSharper C++ bietet außerdem seine eigenen Debugger-DataTips, die nach einem bestimmten Wert durchsucht werden können. Um auf die ReSharper-DataTips zuzugreifen, klicken Sie auf die Inline-Wertanzeige oder ersetzen Sie die Visual Studio-DataTips in den ReSharper-Optionen unter Tools | Debugger.

Clang-Tidy-Integration

Clang-Tidy ist ein leistungsfähiges Opensource-Tool zur Codeanalyse, das auf dem beliebten Clang-Compiler basiert. Das Tool bringt ein umfangreiches Set an Codeprüfungen und dazugehörigen Fixes mit. ReSharper C++ 2017.3 integriert Clang-Tidy nahtlos. Das bedeutet, dass das Tool stets im Hintergrund ausgeführt wird, während Sie Ihren Code bearbeiten, und dass die Inspektionsergebnisse im Code-Editor zusammen mit den integrierten Inspektionen in Echtzeit angezeigt werden. ReSharper C++ ermöglicht Ihnen außerdem, Clang-Tidy-Fixes individuell, als Bereichsaktionen oder aus dem Code-Cleanup anzuwenden.

Zusätzlich zu den Diagnosedaten der Clang-Tidy-Prüfungen stellt Clang-Tidy auch Diagnosedaten von Clang Static Analyzer sowie Warnungen des Clang-Compilers bereit. Zur Diagnose gehören entsprechende ReSharper-Inspektionen mit konfigurierbaren Schweregraden, die Sie an Ihre Bedürfnisse anpassen können.

Bitte beachten Sie, dass Dateien mit der Kodierung UTF-16 oder UTF-32 momentan nicht von Clang-Tidy unterstützt werden.

Verständnis der Sprache

Der Schwerpunkt des Releases 2017.3 lag auf der Optimierung des Sprachverständnisses für C++. Das Release unterstützt jetzt Features von C++17 und C++14:

  • Strukturierte Bindungsdeklarationen von C++17
  • Inline-Variablen von C++17
  • constexpr if-Anweisungen von C++17
  • Ableitung des Rückgabetyps für auto-Funktionen ohne nachstehenden Rückgabetyp von C++14
  • Typüberprüfung und Ableitung des Rückgabetyps für Lambda-Ausdrücke, einschließlich der Unterstützung für generische Lambdas von C++14

Die übrigen sprachspezifischen Änderungen umfassen:

  • Unterstützung für include_alias pragma-Direktiven.
  • Verbesserte Handhabung expliziter template-Instanziierungen
  • Unterstützung für die Microsoft C++-Erweiterung Ereignisbehandlung

Codeinspektionen und Kontextaktionen

ReSharper C++ 2017.3 bringt ein Set an neuen Codeinspektionen mit entsprechenden Fixes mit:

  • Binärer Operator mit identischen oder gleichen Argumenten
  • Neudefinition eines Standard-Funktionsarguments aus einer anderen Deklaration oder Basisfunktion
  • Überflüssiger const-Spezifizierer für einen Parameter in einer Funktionsdeklaration
  • Fehlende Beschreibung eines Funktionsparameters in einem Dokumentationskommentar.
  • Überflüssiger Memberinitialisierer in der Initialisierungsliste des Members
  • Definition einer Nicht-Inline-Variable in einer Header-Datei.
  • Benutzerdefinierte Literalsuffixe, die nicht mit einem Unterstrich beginnen
  • Detaillierte Diagnose zu Fehlern in strukturierten Bindungsdeklarationen

Darüber hinaus wurde eine Reihe neuer Kontextaktionen hinzugefügt:

  • Konvertierung des Memberinitialisierers in einer Initialisierungsliste für Member zu einem nicht-statischen Datenmemberinitialisierer
  • Wechsel zwischen den Schlüsselwörtern class und typename in einer Deklaration von Typ-Template-Parametern

Codeformatierung

Der Formatter kann nun über die Kommentare aktiviert/deaktiviert und mithilfe von Inline-Einstellungen im EditorConfig-Stil konfiguriert werden. Dazu kommen zahlreiche neue Optionen für flexiblere Formatierung, darunter Optionen zur Steuerung von:

  • Mehrzeiliger Codeformatierung:
    • Ausrichten von Code in Spalten für Zuweisungen, Deklarationen und einzeilige Kommentare
    • Ausrücken von Kommas und Memberzugriffsoperatoren in verketteten Methodenaufrufen
  • Leerzeichen vor/nach Kommas in einheitlichen Initialisierungsklammern und vor Kommentaren am Zeilenende

Verbesserungen an „Go to Declaration“

Der Befehl Go to Declaration ist jetzt für abhängige Namen innerhalb von Template-Definitionen verfügbar. Da es für ReSharper C++ nicht möglich ist, die exakte Entität zu ermitteln, auf die sich der Name bezieht, werden stattdessen alle Symbole mit passenden Namen und Signaturen gefunden und als potenzielle Zielpunkte für die Navigation angeboten.

Darüber hinaus können Sie mit Go to Declaration jetzt von einer Funktionsdefinition zu einer Deklaration mit nicht übereinstimmender Signatur navigieren, falls keine passenden Deklarationen gefunden werden.
