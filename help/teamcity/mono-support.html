TeamCity 2019.1帮助

单声道支持

Mono框架是在Windows和基于Unix的平台上运行.NET应用程序的替代框架。
有关更多信息，请参考Mono官方网站 。

支持的构建运行器

TeamCity支持在Mono框架以及.NET Framework下使用NAnt和MSBuild运行器运行.NET构建。（MSBuild作为Mono中的xbuild）。

当代理上安装了Mono时， NuGet运行程序支持Linux和macOS。请注意，仅支持Mono 4.4.2+上的NuGet CLI 3.2+。

Mono还支持测试报告任务 。

单声道平台检测

启动构建代理时，它将自动检测到Mono安装。

在每个平台上，Mono检测与NAnt one兼容。看到NAnt.exe.config用于在NAnt上进行框架检测。

代理属性

在代理端自动检测到Mono时，将设置以下属性：

  • 单 -路径单可执行文件（单声道JIT）

  • MonoVersion —单声道版本

  • MonoX.Z-设置为MONO_ROOT/lib/mono/X.Z如果存在

  • MonoX.Z_x64 —设置为MONO_ROOT/lib/mono/X.Z如果存在并且Mono架构是x64

  • MonoX.Z_x86 —设置为MONO_ROOT/lib/mono/X.Z如果存在并且Mono架构是x86

如果无法自动检测到Mono安装（例如，您已将Mono框架安装到自定义目录中），则可以通过在代理配置文件中手动设置它们来使这些属性可用于构建运行器 。

Windows细节

Windows下自动检测Mono框架具有以下特点：

  1. 从中读取Mono版本HKLM\SOFTWARE\Novell\Mono\DefaultCLR 。

  2. 框架路径提取自HKLM\SOFTWARE\Novell\Mono\%MonoVersion% 。

  3. 通过分析检测平台架构mono.exe 。

Mac OS X的详细信息

  1. 该框架是从以下位置自动检测到的/Library/Frameworks/Mono.framework/Versions 。

  2. 选择了最高版本。

  3. 框架路径是从中提取的/Library/Frameworks/Mono.framework/Versions/%MonoVersion%/lib/mono 。

  4. 该平台架构已固定为x86，因为Mono官方版本仅支持X86。

自定义Linux / Unix细节

在Unix下自动检测Mono框架具有以下特点：

  1. 单声道版本从中读取pkg-config --modversion mono 。

  2. 框架路径是从中提取的pkg-config --variable=prefix mono和pkg-config --variable=libdir mono 。

  3. 通过分析PREFIX/bin/mono可执行文件。您可以通过添加以下命令来强制从自定义位置检测Mono： PREFIX/bin目录到的开头PATH和更新PKG_CONFIG_PATH （在pkg-config(1) ）与PREFIX/lib/pkgconfig.
