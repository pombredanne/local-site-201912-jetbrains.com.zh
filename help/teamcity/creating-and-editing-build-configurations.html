创建和编辑构建配置

该页面详细介绍了通过TeamCity Web UI创建构建配置的过程。其他选项包括REST API和基于Kotlin语言在DSL中使用TeamCity项目配置。

在此页：

  • 手动创建新的构建配置
  • 从URL创建新的构建配置
  • 创建指向GitHub.com存储库的新构建配置
  • 创建指向Bitbucket Cloud的新构建配置
  • 创建构建配置模板
  • 从模板创建构建配置
  • 订购构建配置
  • 配置设定
  • 构建配置设置中的操作

要创建构建配置，请打开项目的“ 常规设置” ，然后在“ 构建配置”部分下单击“ 创建构建配置 ”。

TeamCity提供了几种方法来为项目创建新的构建配置 ：

您可以创建一个新的构建配置

  • 手动地
  • 指向存储库URL

  • 指向GitHub存储库

  • 指向Bitbucket存储库

  • 创建一个生成配置模板 ，然后基于该模板创建一个生成配置 。

创建构建配置后，您可以：

  • 改变顺序
  • 编辑他们的设置

  • 通过“ 动作”菜单控制它们

手动创建新的构建配置

  1. 在“ 管理”>“项目”页面上，在列表中选择所需的项目。（或者，使用“ 项目”弹出窗口打开项目，然后单击右侧的“ 编辑项目设置”链接）。将打开“ 项目设置”页面。

  2. 在“项目设置”页上的“ 构建配置”部分，单击“ 创建构建配置”。

  3. 在“创建构建配置”对话框中，指定构建配置的名称，ID和（可选）描述，然后单击创建 。

  4. 继续创建其他设置：
    • 创建/编辑VCS根目录并指定VCS特定的设置

    • 在“ 构建步骤”页面上，配置通过自动检测发现的构建步骤。通过从下拉列表中选择所需的构建运行器来手动创建它们。点击保存 。您可以在一个构建配置中根据需要添加任意多个构建步骤。请注意，它们将顺序执行。最后，构建获得合并状态，并且输出进入相同的构建日志。如果某个步骤失败，则其余步骤是否执行，取决于它们的步骤执行策略 。

    • 此外，配置构建触发选项 ， 依赖项 ， 属性和变量以及代理要求 。

从URL创建新的构建配置

您可以使用VCS URL创建构建配置：

  1. 在“ 管理”>“项目”页面上，在列表中选择所需的项目。（或者，使用“ 项目”弹出窗口打开项目，然后单击右侧的“ 编辑项目设置”链接）。将打开“ 项目设置”页面。

  2. 在“项目设置”页面上的“ 构建配置”部分，单击“ 从URL创建构建配置” 。

  3. 在“创建构建配置”对话框中，指定VCS存储库URL，如果需要身份验证，则指定VCS凭据。点击创建 。

  4. TeamCity将建议构建配置名称，并将为您配置其余设置。

    • 它将确定VCS存储库的类型并创建VCS根目录 。

    • TeamCity将尝试自动检测构建步骤：Ant，NAnt，Gradle，Maven，MSBuild，Visual Studio解决方案文件，PowerShell，Xcode项目文件，Rake和IntelliJ IDEA项目。如果未找到，则必须[手动配置构建步骤]（http：//配置构建步骤）。

    • 接下来，TeamCity UI将显示带有提示的建议图标，以创建构建触发器 ， 失败条件和构建功能 。根据构建配置设置，它可能会建议一些其他配置选项。

  5. 创建构建配置后，您可以运行构建和/或调整设置。

创建指向GitHub.com存储库的新构建配置

  1. 单击创建构建配置按钮，然后选择指向GitHub.com存储库 。

    • 如果您没有配置GitHub连接，那么您将被重定向到Connections页面。为建立连接这里描述 ，然后按照下面的步骤。

    • 如果您已配置GitHub连接，请执行以下步骤。

  2. 在“ 从GitHub创建构建配置”页面上，选择一个存储库。TeamCity将验证存储库连接。如果连接已验证，则将打开新页面。

  3. TeamCity将显示项目并生成配置名称。如果需要，请修改名称，然后单击继续 。

  4. TeamCity将添加VCS构建触发器，并尝试自动检测构建步骤：Ant，NAnt，Gradle，Maven，MSBuild，Visual Studio解决方案文件，PowerShell，Xcode项目文件，Rake和IntelliJ IDEA项目。在“ 自动检测到的构建步骤” 页面上，选择要在构建配置中使用的步骤。单击使用选定的 。如果未找到任何步骤，则必须手动配置构建步骤 。

  5. 您的项目和构建配置已配置。单击运行开始构建。根据构建配置设置，TeamCity可以建议一些其他配置选项。查看建议的设置并配置所需的。

创建指向Bitbucket Cloud的新构建配置

  1. 单击“ 创建构建配置”按钮，然后选择“ 指向Bitbucket Cloud存储库” 。

  • 如果您没有配置Bitbucket连接，您将被重定向到“ 连接”页面。为建立连接这里描述 ，然后按照下面的步骤。

  • 如果您配置了Bitbucket连接，请执行以下步骤。

  1. 在“从Bitbucket Cloud创建构建配置”页面上，选择一个存储库。TeamCity将验证存储库连接。如果连接已验证，则将打开新页面。

  2. TeamCity将显示项目并生成配置名称。如果需要，请修改名称，然后单击继续 。

  3. TeamCity将添加VCS构建触发器，并尝试自动检测构建步骤：Ant，NAnt，Gradle，Maven，MSBuild，Visual Studio解决方案文件，PowerShell，Xcode项目文件，Rake和IntelliJ IDEA项目。
    在“ 自动检测到的构建步骤” 页面上，选择要在构建配置中使用的步骤。单击使用选定的 。如果未找到任何步骤，则必须手动配置构建步骤 。

  4. 您的项目和构建配置已配置。单击运行开始构建。根据构建配置设置，TeamCity可以建议一些其他配置选项。查看建议的设置并配置所需的。

创建构建配置模板

创建新的构建配置模板类似于创建新的配置：

  1. 关于行政|在“项目”页面上，在列表中选择所需的项目。（或者，使用“ 项目”弹出窗口打开项目，然后单击右侧的“ 编辑项目设置”链接）。将打开“ 项目设置”页面。

  2. 在“项目设置”页面上的“ 构建配置模板”部分，单击“ 创建模板”按钮，然后继续配置常规设置 ， VCS设置和构建步骤 。

或者，您可以从现有的构建配置中创建构建配置模板：

  1. 打开现有构建配置设置页面，点击操作在屏幕的右上角，然后选择提取模板选项。

  2. 指定所需的设置，然后单击创建 。

从模板创建构建配置

要基于模板创建构建配置：

  1. 关于行政|在“项目”页面上，在列表中选择所需的项目。（或者，使用“ 项目”弹出窗口打开项目，然后单击右侧的“ 编辑项目设置”链接）。将打开“ 项目设置”页面。

  2. 在“项目设置”页面上的“ 构建配置模板”部分中，找到所需的模板，然后单击其名称或使用“ 编辑” 。

  3. 点击屏幕右上角的操作 ，然后选择创建构建配置 。

  4. 指定新配置的必需设置。

模板中指定的设置无法在从该模板创建的配置中进行编辑。但是，其中一些可以在构建配置中重新定义 。请注意，修改模板本身的设置将影响基于该模板的所有配置 。

或者，您可以根据以下模板创建构建配置：

  1. 关于行政|项目页面，从列表中选择所需的项目。（或者，使用“ 项目”弹出窗口打开项目，然后单击右侧的“ 编辑项目设置”链接）。将打开“ 项目设置”页面。

  2. 在“项目设置”页上的“ 构建配置”部分，单击“ 创建构建配置”。

  3. 在配置设置页面上，使用“ 基于模板”下拉菜单，然后为您的构建配置选择一个模板。

订购构建配置

默认情况下，您可以在按字母顺序列出的“ 项目概述”页面上查看项目的所有构建配置。管理员可以自定义默认顺序 。
