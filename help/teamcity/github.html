TeamCity 2019.1帮助

的GitHub

从TeamCity 10.0开始 ，TeamCity与GitHub集成，并且与GitHub问题跟踪器的集成可以单独设置，也可以作为TeamCity与GitHub源代码托管服务集成的一部分。

如果您在TeamCity 10.0之前使用TeamCity-GitHub 第三方插件 ，则可以安全地删除它：内置的TeamCity集成将检测到与GitHub问题跟踪器的现有连接并自动获取您的设置。

在设置与GitHub的集成时，除了存储库URL和其他常规设置之外，您还需要配置身份验证并指定问题ID模式。

认证方式

TeamCity允许您选择是要匿名连接到GitHub，还是要通过用户名/密码或访问令牌进行身份验证。

将字符串转换为问题的链接

您还需要指定哪些字符串应被视为对跟踪器中问题的引用。对于GitHub，您需要使用regex语法，例如， #(\d+) 。

TeamCity将解决VCS注释中提到的问题编号，并将在Web UI中显示此问题的链接（例如，在构建结果页面的“ 更改”页面上，“ 问题”选项卡上）。
