建立亚军

生成运行器是TeamCity的一部分，它允许与特定的生成工具（Ant，MSBuild，命令行等）集成。在构建配置中，构建运行器定义如何运行构建并报告其结果。

每个构建运行器都有两个部分：

  • 通过Web UI配置的服务器端设置

  • 在代理上执行构建的代理端部分

TeamCity与以下跑步者捆绑在一起：

  • .NET CLI（dotnet）
  • .NET Process Runner
  • 蚂蚁

  • 命令行 –运行任意命令行

  • 部署者
    • 容器部署器
    • FTP上传
    • SMB上传
    • SSH上传
    • SSH执行
  • 重复查找器（ReSharper）
  • 重复查找器（Java）
  • FxCop
  • 摇篮

  • 检查（IntelliJ IDEA） –一组IntelliJ IDEA检查

  • 检查（ReSharper） –一组ReSharper检查

  • IntelliJ IDEA项目及其早期版本： Ipr（已弃用）

  • 马文
  • MSBuild
  • 规格书
  • 南特
  • NuGet相关的跑步者
  • NUnit
  • 电源外壳
  • 耙
  • 简单构建工具（Scala）

  • Visual Studio（sln） – Microsoft Visual Studio 2005/2008/2010/2012/2013/2015解决方案

  • Visual Studio 2003 – Microsoft Visual Studio 2003解决方案

  • Visual Studio测试 （带有集成的MSTest）

  • Xcode项目

从技术上讲，构建运行器是作为插件实现的。

可以在“ 创建/编辑构建配置”页面的“ 配置构建步骤”部分中配置构建运行器 。
