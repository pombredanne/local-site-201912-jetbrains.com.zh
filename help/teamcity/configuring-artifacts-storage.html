配置工件存储

项目设置| “工件存储”选项卡显示在此项目中配置的工件存储以及从父级继承的存储。

默认情况下，将显示内置的TeamCity工件存储并将其标记为活动状态。您可以使用相应的链接激活其他存储。

在此页：

  • 内置文物存储
  • 外部文物存储

      • 权限
    • 其他外部工件存储插件

内置文物存储

TeamCity将由构建产生的工件存储在TeamCity服务器可访问的文件系统上。默认位置是< TeamCity数据目录 > / system / artifacts，但是可以重新定义此位置。

外部文物存储

TeamCity提供了一个可插入的API，以为TeamCity构建工件启用外部存储。可以作为TeamCity的外部插件来实现对不同存储设备的支持： 外部存储设备实施指南中提供了详细信息。

请注意，当启用了工件的外部存储时，TeamCity 内部工件（包括构建日志）仍将发布到TeamCity服务器，并存储在内置工件存储中的TeamCity数据目录中。

关于工件映射的元数据也是如此，它将被发布到TeamCity数据目录的工件目录中。从备份还原时，请确保将其还原以使外部工件插件正常工作。

Amazon S3支持

自TeamCity 2018.1起，TeamCity与Amazon S3 Artifact Storage插件捆绑在一起，该插件允许将构建工件存储在Amazon S3存储桶中。

在项目级别可以用AWS S3替换TeamCity内置工件存储。配置S3工件存储后，它将：

  • 允许从S3上传，下载和删除工件

  • 处理工件依赖项的解析以及工件的清理

  • 显示位于TeamCity Web UI外部的工件

在AWS S3存储桶中启用外部工件存储

  1. 导航到项目设置| “工件存储”选项卡。内置的TeamCity工件存储默认情况下显示，并标记为活动状态。

  2. 点击添加新存储。选择S3存储作为存储类型（前提是未安装其他外部存储插件）。

  3. 提供一个可选的存储名称。

  4. 选择AWS环境并提供所需的设置（如果有）。

  5. 提供您的AWS安全凭证。

  6. 指定一个现有的S3存储桶以存储工件。

  7. 保存设置。

  8. 配置的S3存储将出现在“工件存储”页面上。使用相应的链接激活它。

现在，此项目的构建及其子项目和构建配置所产生的新工件将存储在指定的AWS S3存储桶中。

权限

启用“使用预签名URL进行上传”选项后，TeamCity服务器上提供的AWS凭证或IAM角色应具有权限： DeleteObject, ListAllMyBuckets, GetBucketLocation, GetObject, ListBucket, PutObject 。

禁用“使用预签名URL进行上传”选项时：

  • TeamCity服务器上提供的AWS凭证或IAM角色应具有以下权限：DeleteObject, ListAllMyBuckets, GetBucketLocation, GetObject

  • 应该指定两个AWS凭证并具有ListBucket, PutObject权限，或所有TeamCity代理上的IAM角色都应具有以下权限：ListBucket, PutObject
