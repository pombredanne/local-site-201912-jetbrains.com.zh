TeamCity 2019.1帮助

Visual Studio（sln）

此页面包含有关构建Microsoft Visual Studio 2005-2017的Visual Studio（sln）构建运行器以及自TeamCity 2019.1以来的 Microsoft Visual Studio 2019解决方案文件的参考信息。若要生成Microsoft Visual Studio 2003解决方案文件，请使用Visual Studio 2003运行程序 。

Visual Studio（sln）生成运行器要求在生成代理上安装正确版本的Microsoft Visual Studio。

常规构建运行器选项

选项

描述

解决方案文件路径

指定相对于“ 构建签出目录”要构建的解决方案的路径。例如：

vs-addin \ addin \ addin.sln

工作目录

指定构建工作目录 。 （可选的）

视觉工作室

选择Visual Studio版本。

目标

指定特定于先前选择的Visual Studio版本的Microsoft Visual Studio目标。可能的选项是Build ， Rebuild ， Clean ， Publish或根据您的需要将这些目标组合在一起。多个目标之间用空格隔开。

组态

指定要构建的Microsoft Visual Studio解决方案配置的名称（可选）。

平台

指定解决方案的平台。您可以将该字段保留为空白，TeamCity将从解决方案设置（可选）中获取此信息。

命令行参数

指定要传递给构建运行程序的其他命令行参数。建议您在“ 配置构建参数”页面上定义它们，而不是显式指定这些参数。
