部署构建配置

TeamCity提供构建配置的部署类型。执行部署到某个环境的构建配置可以用这种类型标记：这些通常是对快照具有快照或工件依赖关系的构建配置，它们将部署其结果。

您可以创建构建配置，并使用“ 常规设置”选项卡将其类型设置为“ 部署” 。类型更改不会影响构建配置功能，但会提供一些方便的功能，以区分部署构建和其他构建。
您可以随时将类型更改回常规 。

您可以禁用构建链中某些部分之间的修订同步 ，以使部署设置更加灵活。

将配置标记为Deployment之后 ，TeamCity将以以下方式更改行为：

  • 此配置的“ 运行”按钮标题更改为Deploy 。

  • 如果在构建配置中有一个构建用作依赖项，则在运行构建后，其构建结果页面将显示“ 部署”部分，使您可以快速部署构建。

  • 更改详细信息页面（您可以在其中查看哪些配置受更改影响以及已通过此更改执行了哪些内部版本的页面）具有“ 部署”选项卡，该选项卡显示了首次配置此更改的部署配置中的内部版本。

  • 对于部署配置，TeamCity始终显示最新启动的内部版本，无论其包含的更改如何。与常规的构建配置不同，将显示具有最新更改的构建。

  • 将构建配置类型设置为Deployment时 ，将自动更改一些设置以反映最佳实践：将“限制同时运行的构建的数量”设置为“ 1”，并且“允许触发个人构建”选项被关闭。
