为VMware vSphere和vCenter设置TeamCity

TeamCity vSphere集成允许在VMware vSphere和vCenter安装中使用TeamCity 代理云功能 。它需要使用您的VMware vSphere / vCenter帐户配置TeamCity，然后根据排队的构建，使用TeamCity代理按需处理虚拟机的自动创建/启动/停止/删除。

该功能通过与TeamCity捆绑在一起的插件实现。

在此页：

  • 要求
  • 特征
  • 用法
    • 有关配置代理程序云配置文件的注意事项

要求

集成需要通过API对vSphere / vCenter进行写访问，这在许可版本中可用；免费版本的API支持仅限于只读访问。

特征

TeamCity vSphere集成使您能够：

  • 选择行为类型：

    • 启动/停止现有虚拟机

    • 或克隆虚拟机或模板，在克隆变为空闲时将其删除（构建完成或经过空闲超时后，具体取决于配置文件设置）

  • 选择一个虚拟机快照以启动

  • 指定克隆文件夹和计算机将分配到的资源池

  • 配置启动实例的最大数量

用法

设置TeamCity-VMware vSphere代理云集成需要执行以下步骤：

  1. 创建将在其中运行构建的虚拟机。有关创建虚拟机的详细信息，请参考VMware vSphere网站。

  2. 确保已安装VMware Tools或Open VM Tools。请参阅VMware文档 。

  3. 如本节所述安装TeamCity代理，配置和测试计算机。

    • 如果您希望TeamCity根据需要启动/停止此计算机或对其进行克隆，请继续在TeamCity服务器上配置VMware 云配置文件 。修改配置文件后，TeamCity立即检测到更改，并在代理完成当前构建后强制关闭在这些更改之前启动的代理。

    • 如果要创建此计算机的模板并将其克隆，请参考VMware vSphere网站以获取有关创建模板的详细信息，然后继续在TeamCity服务器上配置VMware 云配置文件 。确保指定有效的vCenter SDK URL 。

配置TeamCity构建代理时，请确保在构建代理属性中指定有效的TeamCity服务器URL。

有关配置代理程序云配置文件的注意事项

  • 您可以限制所有图像的实例数量和/或设置每个图像的限制。

  • 可以为云代理指定唯一的主机名：添加映像时，请从相应字段中选择自定义规范。该选项可用于Windows和Linux虚拟机。

  • 在不同的云配置文件中使用同一映像时，为避免可能的冲突，在TeamCity中配置云配置文件时可以使用自定义代理映像名称。这对于代理的命名模式也很有用。指定自定义代理映像名称后，将从该映像克隆的云代理实例的名称将基于该名称。
