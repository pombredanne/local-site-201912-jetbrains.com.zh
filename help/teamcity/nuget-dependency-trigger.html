TeamCity 2019.1帮助

NuGet依赖触发器

如果在NuGet存储库中检测到NuGet软件包更新，则NuGet依赖关系触发器允许开始新的构建。

要求和限制

对于在Windows上运行的TeamCity服务器，需要.NET 4.0 。

对于在Linux上运行的TeamCity服务器，据报道NuGet依赖项触发器将具有以下限制 ：

  • 不支持按软件包版本规格过滤

  • 仅支持HTTP包源

  • 使用NuGet feed版本1.0，因此可能会出现区分大小写的问题

  • 当前在Linux上的触发器实现可能会增加服务器负载

  • 身份验证问题可能会发生

配置NuGet依赖触发器

  1. 从NuGet.exe下拉列表中选择要使用的NuGet版本（如果您已经预先安装了NuGet ），或者指定一个自定义路径来NuGet.exe ;

  2. 指定NuGet软件包的来源，如果与nuget.org ;

  3. 指定凭据以访问NuGet feed（如果需要）

  4. 输入软件包ID以检查更新。

  5. （可选）您可以指定要检查的软件包版本范围 。如果未指定，TeamCity将检查最新版本。

如果通过选择相应的复选框检测到预发行包版本，也可以选择触发构建。请注意，只有NuGet 1.8版或更高版本才支持此功能。
