将TeamCity与问题跟踪器集成

TeamCity可以与您的问题跟踪器集成在一起，以提供您的开发项目的全面视图。TeamCity检测到版本控制更改的注释中提到的问题，并将其转换为指向TeamCity Web UI中问题跟踪器的链接。

集成是在项目级别配置的：需要项目管理员权限。如果您在TeamCity和问题跟踪器服务器上都有多个项目，或者对不同的项目使用不同的问题跟踪器，则可以配置集成。

对项目启用集成也对其所有子项目启用。如果子项目中的配置设置不同，则其设置优先于项目的设置。

专门支持问题跟踪器

TeamCity支持JIRA ， Bugzilla ， YouTrack以及TeamCity 10.0 GitHub ， Bitbucket Cloud和TFS跟踪器。“ 受支持的平台和环境”页面列出了受支持的版本。

配置集成后，TeamCity自动将VCS提交注释中提到的问题ID（JIRA中的问题密钥，TFS中的工作项ID）转换为指向相应问题的链接，并且基本问题详细信息显示在TeamCity Web UI中将鼠标悬停在问题ID旁边的图标上时（例如，在构建结果的“ 更改”选项卡上）。

也可以在构建结果的“ 问题”选项卡上查看构建中修复的问题 。您可以将列表过滤到特定的构建范围，并查看注释中提到的问题及其状态。

有关使用问题跟踪程序集成的建议

要从问题跟踪器集成中获得最大收益，请执行以下操作：

  • 在提交对版本控制的更改时，请始终在提交的注释中提及与修复相关的问题ID（问题密钥） 。

  • 解决问题后即可解决（解决的时间并不重要）。

  • 使用构建配置的问题日志来获取与构建相关的问题；启用“仅显示已解决的问题”选项，以仅显示内部版本中已解决的问题。

启用问题跟踪程序集成

要求

TeamCity服务器使用提供的凭据检索有关问题的信息，然后将其显示给TeamCity用户。

这有几个含义：

  • TeamCity服务器需要直接访问问题跟踪器。（此外，TeamCity还不支持代理到问题跟踪器的连接）。

  • 在与问题跟踪器的连接中配置的用户必须具有足够的权限才能查看TeamCity中可能提到的问题。另外，TeamCity用户将能够在TeamCity中查看已配置用户有权访问的所有问题的问题详细信息。

配置连接

要启用集成，您需要在“ 项目设置” |“ 项目设置” |“问题跟踪器”上创建连接。 问题跟踪器页面。

以下描述的设置对于所有问题跟踪器都是通用的：

连接类型

从列表中选择问题跟踪器的类型。

显示名称

符号名称，将在问题跟踪器的TeamCity UI中显示。

服务器URL（存储库URL）

问题跟踪网址

用户名/密码（身份验证）

登录到问题跟踪器的证书（如果需要授权）。

相应的部分提供了其他身份验证信息或/和有关如何指定TeamCity识别并转换为跟踪器问题链接的字符串的详细信息：

将TeamCity与其他问题跟踪器集成

要将TeamCity与其他问题跟踪器集成，请将TeamCity配置为将更改注释中提及的任何问题跟踪器问题ID转换为链接。有关详细信息，请参见在注释中映射外部链接 。

还可以通过自定义问题跟踪器集成插件添加对问题跟踪器的专用支持。
