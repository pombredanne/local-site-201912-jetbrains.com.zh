出于生产目的 ，我们建议在单独的计算机上设置TeamCity服务器和代理。

您还可以使用TeamCity Server和TeamCity Build Agent Docker映像。

下载TeamCity

下载免费的TeamCity专业版，功能齐全的TeamCity与3个构建代理捆绑在一起，限制为100个构建配置（对于TeamCity 2017.2之前的版本，限制为20个）。
评估之后，您可以切换到企业版 ： 许可策略提供了详细信息。定价可从JetBrains网站上获得 。

早期版本可在“ 先前版本的下载”页面上找到。

安装TeamCity

在Windows上

运行下载的.exe文件并按照TeamCity安装向导的说明进行操作。TeamCity Web服务器和一个构建代理将安装在同一台计算机上。

在安装期间，您可以配置：

  • 将安装TeamCity的TeamCity主目录

  • TeamCity服务器和代理是否将作为Windows服务运行

  • 端口：

    • 服务器端口： 80是默认端口，其他应用程序（例如，Skype）已经可以使用它。如果服务器端口已在使用中，请对其进行更改。在下面的示例中，我们将端口设置为8111 。

    • 代理端口： 9090是来自服务器的传入连接的默认设置。如果此端口已在使用中，则会要求您通过设置ownPort属性设置为其他值。

如果将TeamCity服务器安装为Windows服务，请按照启动和停止服务的常规过程进行操作。

否则，要同时启动/停止TeamCity服务器和一个默认代理，请使用runAll脚本，在/bin目录：

  • 要启动服务器和默认代理，请使用

    runAll.bat开始

  • 要停止服务器和默认代理，请使用

    runAll.bat停止

如果您未更改默认端口（ 80 ）在安装过程中，可以在安装了TeamCity服务器的同一台计算机上运行的Web浏览器中，通过http：// localhost访问TeamCity Web UI。否则，使用http://localhost: （在我们的示例中为http：// localhost：8111 ）。

在Linux和OS X上

  1. 确保已安装Java 8（例如，使用AdoptOpenJDK的 OpenJDK 8）。
    打开命令行终端，然后运行以下命令：

    Java版本

  2. 确保JAVA_HOME环境变量指向Java安装目录。在命令行终端中运行以下命令：

    回声$ JAVA_HOME

  3. 使用TeamCity.tar.gz存档以手动安装与Tomcat Servlet容器捆绑在一起的TeamCity。
    解压缩档案。例如在Linux下使用

    tar xfz TeamCity .tar.gz

    该归档文件也可以用于Windows上的安装。

  4. TeamCity Web服务器和一个构建代理将安装在同一台计算机上。
    请注意 ，出于生产目的 ，建议在单独的计算机上设置TeamCity服务器和代理。

要同时启动/停止TeamCity服务器和一个默认代理 ，请使用runAll脚本，在/bin目录。

  • 要启动服务器和默认代理，请使用

    runAll.sh开始

  • 要停止服务器和默认代理，请使用

    runAll.sh停止

首次启动TeamCity

在第一个TeamCity上开始：

  1. 查看TeamCity Data Directory的位置 ，所有配置信息都存储在该位置 。单击继续 。

  2. TeamCity将构建历史记录，用户，构建结果和一些运行时数据存储在SQL数据库中，并允许您选择数据库类型。
    现在，保留默认的内部数据库。单击继续 。

    TeamCity需要一些时间来配置必要的组件。

  3. 在下一个屏幕上，接受许可协议以继续启动。点击继续 。

  4. TeamCity显示“ 创建管理员帐户”页面。指定管理员凭据，然后单击创建帐户 。

您已登录TeamCity：现在，您可以配置用户设置以及创建和运行第一个构建 。
