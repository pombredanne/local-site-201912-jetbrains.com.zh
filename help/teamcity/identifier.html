ID是赋予TeamCity实体（ 项目 ， 构建配置 ， 模板 ， VCS根目录等）的标识符。

每个实体都有两个标识符：

  • 外部ID

  • 通用唯一ID或UUID

外部编号

所谓的外部标识符在TeamCity Web UI中配置（例如，项目ID），并且在整个服务器上所有相同类型的对象中必须是唯一的。构建配置和模板共享相同的ID空间。

ID只能包含字母数字字符和下划线（_）-最多80个字符-且应以拉丁字母开头。

使用外部ID

使用外部ID：

  • 在Web界面的URL（包括RSS feeds ， NuGet feeds ）中，例如https://teamcity.jetbrains.com/project.html?projectId=TeamCityPluginsByJetBrains

  • 在里面dep.和vcsRoot.参数参考

  • 在REST API中，并构建用于通过TeamCity自动执行操作的脚本（例如，通过直接URL或Ivy下载工件）

  • 在配置文件中存储项目设置并在<TeamCity Data Directory>/config

  • 在文件名和目录名下< TeamCity Data Directory >/system （例如，构建工件存储）

分配外部ID

默认情况下，TeamCity通过将对象的名称转换为与ID要求相匹配的名称并为之加上父项目ID来自动为该对象建议ID。可以手动修改ID。

建议保留自动生成的ID不变，除非有特殊考虑要对其进行修改。

如果考虑在多个TeamCity服务器安装之间移动项目，则最好确保所有ID在全局上都是唯一的。

在更改项目或构建配置的ID时，所有相关的URL（包括Web UI，工件下载链接， RSS feed和REST API）都会更改。如果在脚本中将包含旧ID的任何URL标记为书签或进行了硬编码，则它们将停止运行并需要更新。ID更改时，TeamCity数据目录下相应命名的目录（包括存储设置和工件的目录）被重命名，这可能会花费一些时间。

要重置ID以匹配所有项目，VCS根目录，构建配置和模板的默认方案，请使用上级项目 “ 管理”页面上的“ 批量编辑ID”操作。要在修改后或更改现有对象名称后使用自动生成的ID，可以使用“ 重新生成ID”操作来重新生成ID。

复制项目时，TeamCity会自动为所有子元素分配新的ID。可以在“ 复制”对话框中预览和更改ID。移动对象时，将保留其ID，并且您可能希望使用“ 重新生成ID”操作使ID反映新的放置位置。

通用唯一ID

TeamCity项目，构建配置和VCS根目录具有UUID，它是自动生成的全局唯一ID。 UUID存储在相应的实体XML配置文件中，该文件位于< TeamCity Data Directory >/config目录。这些UUID绝对不能手动编辑。通过将文件放置到数据目录中创建新实体时，该实体应没有uuid属性。TeamCity将自动生成UUID，并将其保留在文件中。
