P4连接设置

选项

描述

港口

指定Perforce服务器地址。格式是host:port 。对于特定环境，可以在下面的工作区选项中为任何类型的签出指定P4Host。

流

单击此单选按钮以指定现有的Perforce流。TeamCity将使用此流来准备基于流的工作区，并将使用来自该工作区的客户端映射。

TeamCity支持根仓库中更深的目录结构：深度为//DEPOTNAME/1/2/n可以在此字段中指定。
在TeamCity 2017.2之前 ，TeamCity支持在仓库名称下方一级存储的流：格式为//streamdepot/streamname 。

支持参数 。为了StreamAtChange选项，请使用标签签出字段。

绩效影响

当此选项与服务器模式下的结帐一起使用时，服务器端的内部TeamCity源缓存将被禁用，这可能会使clean checkout的性能恶化。同样，使用此选项，快照依赖项构建不会重复使用 。（ TW-41898-在TeamCity 2017.1中修复）

结帐规则限制

当Perforce的流与使用VCS检出模式 ，简单的结算规则喜欢. => sub/directory支持。排除结帐规则，多个包含规则或类似规则aaa=>bbb不支持。

启用功能分支支持（实验性） –除了默认复选框之外，选中此复选框以指定要监视其更改的分支流。输入/编辑分支规范，以换行符分隔的规则集。语法是+|-:stream_name （带有可选*占位符）。

在这里， 管道符号|代表OR命令，如正则表达式中一样：use +用于包括， 或 -排除。

任务流限制

当任务流用于功能分支时，TeamCity可能会丢失任务流中的某些更改，直到做出修改提交为止，这意味着直到对任务流做出“实际”提交之前，才会检测到来自父流的合并提交（ TW -44765 ）。

客户

单击此单选按钮直接指定客户端工作空间名称。工作区必须已经由Perforce客户端应用程序（例如P4V或P4Win）创建。仅使用来自已配置的客户端工作空间的映射规则。客户端名称将被忽略。

绩效影响

当将此选项与VCS Checkout模式一起使用时，服务器端的内部TeamCity源缓存将被禁用，这可能会使clean checkout的性能恶化。同样，使用此选项，快照依赖项构建不会重复使用 。（ TW-41898-在TeamCity 2017.1中修复）

客户端映射

单击此单选按钮以指定软件仓库到客户端计算机的映射。如果选择了“ 客户端映射” ，TeamCity将根据运行构建的构建代理的OS /平台来处理文件分隔符。为了执行特定的行分隔符为所有生成代理，使用客户端或流与LineEnd Perforce中指定的选项，而不是客户端映射 。或者，您可以添加代理要求以仅在特定平台上运行构建。

在映射中使用team-city-agent而不是客户端名称。

例：

// depot / MPS / ... // team-city-agent / ...// depot / MPS / lib / tools /...// team-city-agent / tools / ...

在TeamCity 10.0之前的版本中 ，编辑Perforce VCS根目录的客户端映射会导致在下次构建之前进行Clean Checkout 。提供了一种解决方法 。

现在，在以下情况下， 不对代理方结帐强制执行对客户端映射更改的“ 清理结帐 ”：

  • 当使用Perforce客户端名称时，更改客户端的Perforce客户端映射不会导致干净的签出

  • 当使用Perforce流时，在保持相同流根的同时更改流名称不会导致干净签出

如果更改了直接客户端映射，则除非执行以下操作，否则将强制执行干净签出： teamcity.perforce.enable-no-clean-checkout 内部属性在服务器上设置。

基于票证的身份验证

选中此选项以启用基于票证的身份验证。默认情况下启用此选项，并且不显示。

结帐规则中不区分大小写

请注意，TeamCity中的Perforce支持将签出规则视为区分大小写。通过在“ 编辑签出规则”表单中添加以下注释，可以在“ 版本控制设置”页面上启用基于Perforce的构建配置的不区分大小写： ##teamcity ignore-case 。

检出代理设置

当使用代理端检出时 ，TeamCity为每个检出目录 / VCS root创建一个Perforce工作区。这些工作空间会在必要时自动创建，并在闲置一段时间后自动删除。可以自定义TeamCity生成的名称：添加teamcity.perforce.workspace.prefix “ 参数”页面上的配置参数，值中带有前缀。

工作区选项

如果需要，您可以在此处为p4 client命令设置以下选项： Options ， SubmitOptions和LineEnd 。
对于特定环境，可以在此处为任何类型的签出指定P4Host。

创建非流工作区

（ 自TeamCity 2018.2开始 ）仅当在“ 连接设置 ”中选择“ 流 ”时，此选项才可用。启用该选项以能够基于流规范使用非流工作区进行签出。这允许使用签出规则，但无法提交到构建中的流。

运行“ p4 clean”进行清理

启用此选项可在构建之前清除多余文件中的工作空间（自p4 2014.1起）。启用后， p4 clean命令将在之前运行p4 sync command ，除非p4 sync -f要么p4 sync -p用来。请参阅命令参考 。

跳过清单更新

启用此选项以不同步跟踪Perforce服务器上的文件（始终将所有文件传输到代理， p4 sync -p ）。

额外的同步选项

指定其他p4 sync选项，例如--parallel 。请参阅命令参考 。

Perforce工作区参数

通过在代理上签出，TeamCity提供环境变量来描述在签出过程中创建的Perforce工作区。
如果将多个Perforce VCS根目录用于签出，则会为第一个VCS根目录创建变量。变量是：

  • P4USER-与vcsroot..user 参数

  • P4PORT-与vcsroot..port 参数

  • P4CLIENT-代理上生成的P4工作区的名称这些变量可用于在签出后执行自定义p4命令。

详细信息： TeamCity中的Perforce工作区处理

Perforce代理设置

要允许将Perforce代理与代理端结帐一起使用 ，请指定env.TEAMCITY_P4PORT 构建代理上的环境变量，代理将将此值作为P4PORT值。

其他设定

构建代理上的P4路径

指定Perforce命令行客户端的路径： p4.exe文件。

该字段仅在代理方端进行代理方结帐（ 在TeamCity 2017.2.2之前，此路径同时用于服务器端结帐和代理方结帐）。
对于服务器，p4二进制文件应该存在于TeamCity服务器的PATH中，或者可以通过teamcity.perforce.customP4Path 内部财产 。

要恢复旧的行为， teamcity.perforce.p4PathOnServerWhitelist internal属性可用于指定允许的p4路径的分号分隔列表。

标签/更改列表进行同步

如果您需要签出的不是带有最新版本的资源，而是带有特定Perforce标签（有选择的更改）的签出资源，则可以在此处指定此标签。例如，这对于生成里程碑/发行版本或复制版本可能很有用。如果该字段保留为空白，则最新的更改列表将用于同步。

如果使用符号标签，建议使用代理方结帐 。通过标签上的服务器端检出，TeamCity将执行完整检出。

字符集

选择客户端计算机上使用的字符集。

支持UTF-16编码

如果您将UTF-16文件存储为utf16项目中的Perforce文件类型 。

如果您使用服务器端检出并且具有以下文件，则可能需要启用此选项： utf16 库中的Perforce文件类型 。启用此标志，以使检出文件位于UTF-16编码。否则，此类文件可能会转换为UTF-8结帐时。

如果您存储UTF-16文件作为binary Perforce文件类型 ，将始终按“原样”检出它们，不执行任何转换。
