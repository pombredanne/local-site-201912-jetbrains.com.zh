TeamCity 2019.1帮助

认证模块

TeamCity中有两种类型的身份验证模块 ：

  • 凭据身份验证模块使用登录页面上指定的登录名/密码对对用户进行身份验证。

  • HTTP身份验证模块使用来自特定HTTP请求的某些信息对用户进行身份验证。您可以同时启用多个凭据身份验证模块和多个HTTP身份验证模块 。

尝试通过登录页面登录时，TeamCity会按照指定的顺序询问所有可用的凭据身份验证模块 ，而第一个可以验证用户身份的模块将对其进行身份验证。对于任何HTTP请求，如果还没有经过身份验证的用户，TeamCity将按照指定的顺序询问所有已启用的HTTP身份验证模块 ，并且第一个可以对用户进行身份验证的模块将对其进行身份验证（如果没有HTTP身份验证模块可以对用户进行身份验证，指定HTTP请求的用户，TeamCity会将用户重定向到登录页面）。

TeamCity支持以下凭据身份验证模块 ：

  • 内置 （跨平台）：用户及其密码由TeamCity维护。新用户由TeamCity管理员添加（在“管理”区域中），或者如果管理员允许首次登录时进行用户注册，则他们可以自行注册。

  • Microsoft Windows域 （跨平台）：可以登录到运行TeamCity服务器的计算机的所有NT域用户，也可以使用相同的凭据登录到TeamCity。即，要登录TeamCity，用户应提供域和用户名（ DOMAIN \ username ）及其域密码。

  • LDAP服务器 （跨平台）：身份验证是通过使用登录表单中输入的凭据直接登录LDAP来执行的。

  • 从TeamCity 2019.1开始， 基于令牌的身份验证 （跨平台）：通过TeamCity维护的个人访问令牌进行身份验证。这样，既可以使用登录表单时使用登录名/访问令牌（而非登录名/密码）进行身份验证，又可以基于令牌的HTTP身份验证。

支持以下HTTP身份验证模块 ：

  • 基本HTTP （跨平台）：允许访问某些Web服务器页面并通过各种脚本执行操作。

  • NTLM HTTP （仅适用于Windows服务器）：允许使用NTLM HTTP协议登录。根据客户端的Web浏览器和操作系统的不同，可以提供无需手动输入用户凭据即可登录的功能。

有关特定身份验证模块的配置，请参阅配置身份验证设置 。另请参阅通过HTTP访问服务器页面，以获取有关使用基于令牌的身份验证或基本HTTP身份 验证从脚本访问服务器的详细信息。
