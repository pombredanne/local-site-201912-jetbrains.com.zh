代码覆盖率

代码覆盖率是许多度量标准，用于度量单元测试如何覆盖您的代码。TeamCity开箱即用地支持以下覆盖引擎：

  • Java，请参阅配置Java代码覆盖率

  • IntelliJ IDEA涵盖范围（捆绑）

  • EMMA开源工具包（捆绑）

  • JaCoCo开源（捆绑）

  • .NET ：请参阅配置.NET代码覆盖率

  • JetBrains dotCover （捆绑）

  • NCover

  • PartCover

要从其他覆盖率工具导入报告，请参阅相关说明 。

有关在TeamCity中导入coverage结果的信息，请参阅本节 。

要获得在TeamCity中显示的受支持工具的代码覆盖率信息，您需要在Build Runner设置页面的专用部分中对其进行配置。以下构建运行程序包括代码覆盖率支持：

  • 蚂蚁
  • IntelliJ IDEA项目
  • 马文
  • MSBuild
  • 南特
  • NUnit
  • 规格书
  • .NET Process Runner
  • Ipr（已弃用）

请注意，当前Maven2运行程序支持IntelliJ IDEA和JaCoCo覆盖引擎。
可以在“ 使用生成结果”的“ 概述”选项卡上查看代码覆盖率的结果 ；详细的报告显示在专用的“使用生成结果”选项卡上。

构建配置的“ 统计图表”选项卡上也提供了代码覆盖率图表 。

有关配置代码覆盖率的详细信息，请参阅专用页面： 配置Java代码覆盖率 ， 配置.NET代码覆盖率 。
