Visual Studio 2003

Visual Studio 2003 Build Runner支持构建Microsoft Visual Studio 2003 .NET项目。要生成Microsoft Visual Studio 2005-2017项目，请参阅Visual Studio（sln）生成运行器。

Visual Studio 2003生成运行器使用NAnt而不是MS Visual Studio 2003来执行生成。结果，要求该代理安装.NET Framework 1.1，但是在某些情况下，可能需要.NET Framework SDK 1.1。此NAnt解决方案任务的行为可能与MS Visual Studio 2003不同。有关详细信息，请参见NAnt文档 。
要使用此运行程序，您需要配置NAnt运行程序 。

选项

描述

解决方案文件路径

要构建的解决方案的路径是相对于build checkout目录的 。例如：

vs-addin \ addin \ addin.sln

工作目录

指定构建工作目录 。

组态

指定要构建的解决方案配置的名称。

项目输出

这组选项使您可以使用解决方案中定义的默认输出，或指定自己的输出路径。

所有项目的输出目录

如果选中了“ 覆盖项目输出”选项 ，则此选项可用。指定将放置编译目标的目录。

通过地图解析网址

如果要将URL项目路径映射到物理项目路径，请单击此单选按钮。如果选择此选项，请在“ 键入URL的映射”字段中指定映射。

输入网址的地图

单击此链接，然后在文本区域中指定所需的地图。使用以下格式：

http：// localhost：8111 = myProjectPath / myProject

哪里

  • http：// localhost：8111是将项目上传到的主机

  • myProjectPath/myProject是项目根

通过WebDAV解析URL

如果您希望通过WebDav解析URL，请单击此单选按钮。

确保所有必要的文件都已正确更新。构建代理可能不会隐式更新来自VCS的信息。

MS Visual Studio参考路径

如果要自动将MS Visual Studio的引用路径包括到生成中，请选中此选项。

南特家

指定要运行构建配置的构建的NAnt可执行文件的路径。相对于build checkout目录的路径可以是绝对路径；您也可以使用环境变量。

命令行参数

指定以下任何其他参数NAnt.exe 。

TeamCity会自动将所有已定义的系统属性传递给NAnt，因此您无需在此处通过指定所有属性-D选项。您可以在构建配置设置的“ 构建参数”部分创建必要的属性。

运行NUnit测试

指定.NET程序集，将要运行的NUnit测试存储在其中。多个条目以逗号分隔；启用了NAnt通配符的使用。在以下示例中，TeamCity将在所有项目目录中搜索测试程序集并运行这些测试。

** \ *。dll

所有这些通配符都是相对于包含解决方案文件的路径指定的。

不要针对以下内容运行NUnit测试

指定应从找到的程序集列表中排除的.NET程序集进行测试。多个条目以逗号分隔；启用了NAnt通配符的使用。在以下示例中，TeamCity将忽略此目录中指定的测试。

** \ obj \ ** \ *。dll

所有这些通配符都是相对于包含解决方案文件的路径指定的。

减少测试失败的反馈时间

使用以下选项指示TeamCity先进行一些测试。

首先运行最近失败的测试
