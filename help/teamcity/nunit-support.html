NUnit支持

在TeamCity中设置NUnit测试报告有两种最常用的方法：

  • 使用.NET CLI构建运行器

  • 使用NUnit构建运行器

除此之外，您还可以尝试其他方法，或者使用TeamCity VSTest Adapter在其他任何运行程序（如PowerShell或Command Line ）中运行测试。

框架兼容性

NUnit构建运行器仅支持.NET Framework 。要为.NET Core项目（和.NET Framework项目4.0或更高版本）运行测试，请改用.NET CLI（dotnet）构建运行程序和test命令。

替代方法

如果不适用使用NUnit构建运行程序，则TeamCity提供以下方法在TeamCity中配置NUnit测试报告：

  • TeamCity支持NAnt Build Runner的标准NUnit 。

  • TeamCity提供了用于MSBuild的NUnit，并通过MSBuild社区任务支持了用于MSBuild的NUnit 。

  • TeamCity提供了自己的NUnit测试启动器，可以在MSBuild构建脚本中配置它，也可以从命令行启动它。

  • NUnit的TeamCity插件可用于在NUnit级别上打开报告，而无需修改构建过程。

  • 捆绑的XML Test Reporting插件允许将任何XML报告导入TeamCity。在这种情况下，并非总是能够实时跟踪结果。
    您可以将XML报告处理构建功能添加到构建配置中，或使用以下服务消息： ##teamcity[importData type='sometype' path=''] 。了解更多信息： XML报表处理 ， 与TeamCity建立脚本交互 。

  • TeamCity允许通过服务消息手动配置测试报告。
