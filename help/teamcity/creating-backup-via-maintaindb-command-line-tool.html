通过maintainDB命令行工具创建备份

TeamCity .tar.gz和.exe发行版提供了maintainDB.bat|sh实用程序位于< TeamCity Home >/bin目录。使用此命令行工具，您可以备份服务器数据， 还原服务器数据以及在不同数据库之间迁移 。您也可以使用Web UI创建数据备份。

备份之前

在备份数据之前，建议关闭TeamCity服务器以将所有内部版本都包含在备份中。如果在TeamCity服务器启动时启动了备份过程，则备份中将不包含正在运行和排队的内部版本。

备份文件的位置和格式

备份文件的默认目录是\backup 。

如果没有特别说明， -A选项，TeamCity将从以下位置读取TeamCity数据目录路径： TEAMCITY_DATA_PATH环境变量或默认路径（ $HOME\.Buildserver ） 将会被使用。

备份文件名的默认格式为TeamCity_Backup_.zip ;的后缀添加到YYYYMMDD_HHMMSS格式。

使用maintainDB实用程序执行TeamCity数据备份

本节介绍了一些maintainDB选项。有关所有可用选项的完整列表，请运行maintainDB从命令行不带参数。

要创建数据备份文件 ，请从命令行开始maintainDB实用程序backup命令：

maintenanceDB。[cmd | sh]备份

TeamCity数据备份有一些限制 。默认情况下，如果您运行maintainDB该实用程序没有可选参数，仅备份数据库，服务器设置，项目和构建配置，插件和补充数据（设置历史记录，触发器状态，插件数据等），而忽略构建日志和个人更改。

配置备份范围

要配置要包含在备份文件中的数据范围，请使用以下选项：

  • -C或--include-config-包括内部配置设置

  • -D或--include-database-包含数据库

  • -L或--include-build-logs-包括构建日志

  • -P或--include-personal-changes-包括个人更改

  • -U或--include-supplementary-data-包括补充（插件）数据指定上述选项的不同组合，您可以控制备份文件的内容。例如，要使用所有受支持的数据类型创建备份，请运行

维护数据库备份-C -D -L -P -U

用于数据备份的maintainDB使用示例

要创建具有自定义名称的备份文件 ，请运行带有-F要么--backup-file选项并指定所需的备份文件名，不带扩展名：

maintenanceDB.cmd backup -F 或maintenanceDB.cmd backup --backup-file

执行上面的命令将在默认备份目录中创建一个具有指定名称的新zip文件。

要将时间戳后缀添加到自定义文件名 ，请添加-M要么--timestamp选项：

维护DB.cmd备份-F -M或维护DB.cmd备份-F-时间戳

要在自定义目录中创建备份文件 ，请运行带有-F选项：

维护数据库备份-F 或维护数据库备份--data-dir

maintenanceDB启动选项

如果通过以下方式自定义TeamCity服务器启动选项TEAMCITY_SERVER_OPTS/TEAMCITY_SERVER_MEM_OPTS环境变量或使用自定义JDK安装来运行服务器，则可能需要运行maintainDB添加了相关选项的脚本TEAMCITY_MAINTAINDB_OPTS/TEAMCITY_MAINTAINDB_MEM_OPTS环境变量，并在与TeamCity服务器相同的环境下运行脚本，以便使用相同的JVM。
