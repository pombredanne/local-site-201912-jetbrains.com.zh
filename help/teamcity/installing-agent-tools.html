安装代理工具

在TeamCity中， 代理工具 （即一组文件/二进制分发）是一种插件类型，没有任何类加载到运行时中。代理工具仅用于将二进制文件分发到代理。

TeamCity允许您在服务器上安装/删除其他工具，并分发它们以按需构建代理。
在具有许多构建代理的环境中，您可以集中分发配置文件（例如，如果您要将自定义配置文件/库分发给需要它的所有代理），或一次删除代理上的工具。

政府| “工具”页面提供了统一的界面，以设置适当的插件要使用的工具。您可以安装工具的不同版本和/或更改默认版本。这些工具将自动分发以构建请求代理的代理，并在相关的运行器中使用。

可以通过“ 管理” | “ 管理”来管理以下类型的工具。工具页面：

  • IntelliJ IDEA的捆绑版本设置为默认的IntelliJ Inspections and Duplicates Engine 。

  • 将捆绑版本设置为默认值的JetBrains dotCover命令行工具 。用于收集.NET项目的代码覆盖率。

  • JetBrains ReSharper命令行工具 ：默认情况下，这些工具与TeamCity捆绑在一起，并由Inspections（.NET），Duplicates Finder（.NET）构建运行器使用，以运行代码分析。

  • Maven ：显示多个捆绑版本，默认设置为3.0.5。

  • NuGet特定生成步骤和NuGet依赖关系触发器中使用的NuGet.exe 。NuGet软件包（ .nupkg文件）与tools/NuGet.exe支持里面的文件。

  • NUnit 3 ：可以安装不同的版本，并且可以设置/更改默认版本。

  • Sysinternals handle.exe用于确定在Windows代理的checkout目录中保存文件的进程。

  • 使用代理推送将TeamCity代理从Windows服务器安装到Windows主机时，需要Sysinternals psexec.exe 。

  • 您也可以将自己的工具作为.zip存档上传：“ 插件包装”页面上介绍了该工具插件的结构。TeamCity将使用zip文件的名称作为所有代理上的工具名称。该zip文件将自动在代理上解压缩到具有相同名称的目录。
    安装第一个自定义工具后，页面上将出现“ Zip存档”部分。在本节中，您可以查看所有工具的用法，删除工具或安装新工具。

TeamCity将已安装的工具放入< TeamCity数据目录 > /plugins/.tools中，并监视此文件夹的内容。

从TeamCity 2019.1开始 ，代理仅下载其运行的版本所需的工具。这样，代理可以在服务器升级后立即运行不需要的构建，而无需像以前一样一次下载所有可用工具。
要根据构建配置中的要求设置已安装的工具，请通过%teamcity.tool.%属性位于构建配置设置或参数 （位于支持%parameter%格式）。
在服务器上安装工具时，构建代理会重新启动并获取该工具的ID，而无需下载它。代理仅在启动第一个要求它的构建时才下载该工具。这样可以确保代理仅下载必需和兼容的工具。
一旦下载，这些工具将存储在代理中，因此构建无需花费时间来再次下载它们。

要检查该工具是否出现在代理上，请查找teamcity.tool. 代理在TeamCity Web UI中报告的配置参数 。

如果您在TeamCity Web UI中删除工具，则每个安装了此工具的代理都会检测到此情况，删除该工具的副本并重新启动。
