TeamCity 2019.1帮助

可视SourceSafe

从TeamCity 2018.1开始 ，Visual SourceSafe插件不再与TeamCity捆绑在一起，可以单独下载 。

注意事项和限制

  • TeamCity支持Microsoft Visual SourceSafe 6.0和2005（仅英文版 ）。

  • 仅当TeamCity服务器安装在运行Windows®操作系统的计算机上时，Microsoft Visual SourceSafe才起作用。

  • 确保TeamCity服务器进程由有权访问VSS数据库的用户运行。

TeamCity在Visual SourceSafe中具有以下限制：

  • 共享（非分支）文件无法检出。

  • VSS 6.0中未显示有关文件和目录的添加和删除操作的注释。所有此类操作将具有“无注释”，而不是真正的VSS注释。（此限制是由VSS API施加的，这使得无法检索性能可接受的注释）。

  • VSS签入的时间戳由客户端计算机上的本地时间驱动。因此，如果客户端和构建服务器上的时间不同步，TeamCity将无法正确订购签入。时区也存在问题，但是如果配置正确，它已在VSS客户端版本2005中得到解决。为避免这些问题，请遵循以下建议：

    • 通过时间服务器同步客户端计算机： http : //support.microsoft.com/kb/131715/EN-US 。

    • 设置VSS数据库的时区：启动VSS管理>工具>选项>时区，然后选择一个。

    • 使用VSS 2005。
