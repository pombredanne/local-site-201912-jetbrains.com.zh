分析插件

IntelliJ IDEA项目设置

选项

描述

项目路径

使用此字段指定项目文件的路径（ .ipr ）或项目目录的路径（包含以下内容的项目的根目录.idea夹）。该构建跑步者需要此信息来了解项目的结构。

指定的路径应相对于checkout目录。

检测全局库和基于模块的JDK *.iml档案

如果选中此选项，则保存时将自动扫描所有模块文件以获取对全局库和模块JDK的引用。这有助于您确保所有引用都将得到正确解析。

选择此选项时，打开和保存构建运行器设置的过程可能会很耗时，因为它涉及加载和解析所有项目模块文件。

检查/修复项目

单击以重新分析项目并直接从IDEA项目中导入构建设置，例如JDK列表。

如果您在IntelliJ IDEA中更新项目设置-添加新的JDK，库，请不要忘记通过单击Check / Reparse Project来更新构建运行器设置。

工作目录

如果构建工作目录与构建签出目录不同，请输入路径。

可选，指定是否不同于签出目录。

未解决的项目模块和路径变量

当IntelliJ IDEA模块文件（ .iml ）从IPR文件引用：

  • 找不到

  • 允许您输入IPR文件中使用的路径变量的值

要刷新此部分中的值，请单击Check / Reparse Project 。

选项

描述

如果项目文件包含路径宏（在IntelliJ IDEA的“设置”对话框的“路径变量”对话框中定义），则会出现此字段。在“ 将值设置为”字段中 ，指定要在不同构建代理上使用的项目资源的路径。

项目JDK

本节提供了在项目中检测到的JDK的列表。

选项

描述

JDK主页

使用此字段可以为项目指定JDK主页。

与Ipr运行程序一起构建时，将使用此JDK编译相应的IDEA模块的源。对于“ 检查和重复查找器”内部版本，此JDK将在内部用于解析项目中使用的Java API。
要运行构建过程本身，请在JAVA_HOME将使用环境变量。

JDK Jar文件模式

点击此链接可打开一个文本区域，您可以在其中定义模板。 .jar项目JDK的文件。使用Ant规则定义.jar文件模式。
缺省值用于Linux和Windows操作系统：

jre / lib / *。jar

对于Mac OS X，使用以下几行：

lib / *。jar ../classes/*.jar

IDEA首页

如果您的项目使用IDEA JDK，请指定IDEA主目录的位置

IDEA Jar文件模式

点击此链接可打开一个文本区域，您可以在其中定义模板。 .jar IDEA JDK的文件。

您可以在定义值时使用对外部属性的引用，例如%system.idea_home%要么%env.JDK_1_3% 。这将增加对相应属性的要求 。

Java参数

选项

描述

JDK

选择一个JDK。 本节详细介绍了可用的选项。默认是JAVA_HOME环境变量或代理程序自己的Java。

JDK主页路径

上面选择时，此选项可用。 使用此字段可以指定用于运行构建的自定义JDK的路径。如果该字段为空，则从JAVA_HOME代理计算机上的环境变量，或来自env.JAVA_HOME在构建代理配置文件中指定的属性（ buildAgent.properties ）。如果未指定这些值，TeamCity将使用构建代理进程本身的Java主页。

JVM命令行参数

您可以指定此类JVM命令行参数，例如最大堆大小或启用远程调试的参数。这些值由用于运行构建的JVM传递。
例：

-Xmx512m -Xms256m

伪像

选项

描述

要构建的工件

在此指定在IntelliJ IDEA项目中配置的要构建的工件的名称。

运行配置

选项

描述

运行配置以执行

在此处指定在项目中配置为在TeamCity构建中执行的IntelliJ IDEA运行配置的名称。支持的配置类型为：JUnit，TestNG和应用程序。请注意，应共享此处指定的运行配置（通过IntelliJ IDEA “运行/调试配置”对话框中的 “共享”复选框），并签入版本控件。

支持在IDEA运行配置的启动之前列表中指定的Ant和Build Artifacts任务。

测试参数

  • 要了解有关先运行最近失败的测试以及先 运行新的和修改的测试选项的更多信息，请参阅“先运行风险组测试”页面。

  • 仅运行受影响的测试（基于依赖项）选项将考虑构建更改。启用此选项后，运行程序将计算受当前构建更改影响的模块，并将仅执行直接或间接依赖于受影响模块的那些运行配置。

代码覆盖率

指定代码覆盖率选项，有关详细信息，请参阅IntelliJ IDEA代码覆盖率页面。
