TeamCity 2019.1帮助

上载SSL证书

通过HTTPS或SSL协议建立连接时，可以上传TeamCity认为受信任的SSL证书。这些可以是自签名证书，也可以是由不知名的证书颁发机构（CA）签名的证书。

将可信证书添加到TeamCity服务器

受信任的证书存储是整个服务器的全局存储，并影响所有服务器项目。

添加受信任的证书

  1. 导航到“根项目”的“ 管理”区域，然后在侧栏中选择“ SSL / HTTPS证书”菜单项

  2. 单击上载证书 ，指定证书名称，然后选择一种受支持格式的证书文件：PEM，DER或PKCS＃7。

  3. 保存您的更改。

向TeamCity代理交付证书

所有上载的证书将自动发送给所有TeamCity代理。

但是，有时可能不需要或自动将证书分发给所有代理。然后，您可以将证书放入所需的代理中，以手动将其添加到所需的代理中< TeamCity Agent Home >/conf/trustedCertificates文件夹（每个证书一个文件，文本格式为上述支持的格式之一的证书）。

在以下情况下这很有用：

  • 如果用户使用不受信任的证书运行TeamCity服务器，则需要将服务器证书放入代理程序的此文件夹中以建立代理程序-服务器连接

  • 如果用户认为服务器和代理之间的网络连接不安全，并且不想传输敏感信息。
