创建您的第一个项目

在TeamCity中，默认包含TeamCity中的所有其他项目。要创建项目，请单击右上角的“ 管理”链接，然后单击“ 创建项目” 。系统显示“ 创建项目”页面。

创建项目有几种选择：

  • 从存储库URL （默认）

  • 从现有连接到外部存储库：

    • GitHub.com （请参见下面的示例）

    • 比特桶云

    • GitLab CE / EE和GitLab.com

    • Azure DevOps或以前的Visual Studio Team Services

  • 手动地

从存储库URL创建项目

这是创建第一个版本的最快方法。

  1. 在“ 创建项目”页面上，单击“ 从存储库URL” ，然后将项目 存储库的URL粘贴到“ 存储库URL”字段中。VCS URL格式中列出了受支持的URL格式 。
    如果需要，请指定您的存储库凭据。

  2. 单击继续，然后按照向导进行操作。
    TeamCity会为您完成其余工作：它将识别VCS存储库的类型，测试连接并自动配置VCS存储库设置，并建议项目和构建配置名称。

  3. 单击继续 。
    TeamCity将扫描您的VCS存储库并自动检测您的构建步骤。

  4. 选中步骤框，然后在构建配置中使用它们。

    所选的构建步骤将添加到构建配置中。

恭喜你！您已经配置了包含一个构建步骤的第一个构建。现在，您可以运行构建并根据需要调整其设置 。

创建一个指向GitHub.com存储库的项目

如果要创建的项目的父项目已通过现有连接与GitHub集成， 则会显示快速选项From GitHub.com 。单击它可从预定义的GitHub连接URL 创建项目 ，类似于从存储库URL创建项目 。

  1. 在“ 创建项目”页面上，单击“ 来自存储库URL” ，然后单击“ 存储库URL”字段旁边的GitHub图标。
    请注意，如果为父项目预定义了与GitHub的一个或多个连接 ，TeamCity建议选择其中一个连接以从中创建一个项目。否则，请继续执行步骤2。

  2. 将打开带有“ 添加连接”对话框的“ 连接”页面。它提供了在GitHub服务中注册TeamCity应用程序时要使用的参数。
    单击注册TeamCity链接。

  3. GitHub页面打开。
    您需要在GitHub中将TeamCity注册为OAuth应用程序 。在您的GitHub帐户中执行以下步骤：

    • 登录到您的GitHub帐户。在“ 注册新的OAuth应用程序”页面上，指定TeamCity提供的名称（和可选描述），主页URL和回调URL。使用图标复制所需的参数。

    • 单击注册应用程序 。

    • 向下滚动并单击更新应用程序 。
      该页面将更新为TeamCity应用程序的客户端ID和客户端机密信息。

  4. 继续在TeamCity中配置连接：在“ 添加连接”页面上，指定客户端ID和客户端密码。

  5. 保存设置。

  6. 接下来，您需要在VCS中授权TeamCity：转到“项目”管理，单击“ 创建项目” ，选择“ 来自GitHub.com” ，然后单击“ 登录GitHub” ：

  7. 在打开的页面上，授权TeamCity应用程序。
    授权的应用程序将被授予对私有存储库的完全控制权，并且在GitHub中具有“ 写入存储库”挂钩权限。
    连接已配置：您可以继续在TeamCity中创建项目。将列出用户可用的所有存储库。开始输入以过滤列表并选择所需的存储库：

    TeamCity将验证存储库连接。如果连接已通过验证，则将打开“ 创建项目”页面。TeamCity将显示项目并生成配置名称。如果需要，请修改名称，然后单击继续 。

  8. TeamCity将扫描您的VCS存储库并自动检测您的构建步骤（可能需要一些时间）。选中步骤框，然后在构建配置中使用它们。

  9. 所选的构建步骤将添加到构建配置中。

恭喜你！您已经配置了GitHub连接，并且第一个构建包含一个构建步骤。现在，您可以运行构建并根据需要调整其设置 。

配置了连接后，可以在UI的多个地方（可以指定存储库URL的地方）激活一个小的GitHub图标（ 从URL创建项目，从URL构建配置，从URL创建VCS根目录，从 Git VCS根目录或GitHub问题时）当前项目及其所有子项目的跟踪器），可以更轻松地在TeamCity中创建这些实体。

手动创建项目

如果自动检测设置不适合您，则可以手动创建项目。

  1. 点击右上角的管理链接，然后点击创建项目并选择手动 。指定项目的名称， ID （自动生成，可修改）和可选描述。
    点击创建 ：

  2. 创建项目后，TeamCity会提示使用构建配置来填充它。单击创建构建配置 。
    可以如下所述自动创建配置（类似于创建项目）或手动创建。如果选择手动执行，请指定构建配置名称， ID （自动生成，可修改）和可选描述。

  3. TeamCity提供了创建和附加新的VCS根 目录的权限 ：为了能够创建内部版本，TeamCity必须知道源代码所在的位置，而VCS根目录是VCS设置的集合（源代码，登录名，密码等的路径）设置），以定义TeamCity如何与版本控制（SCM）系统通信以监视更改并获取构建源。每个构建配置必须至少附加一个VCS根目录。可以自动或手动创建VCS根目录。
    要手动创建它，请从下拉菜单中选择一种VCS（在下面的示例中为Git ），指定所需的信息（名称和URL），测试连接，然后单击创建 。

    创建VCS根目录后，可以指示TeamCity从签出中排除某些目录，或将某些路径（复制目录及其所有内容）映射到构建代理上与默认目录不同的位置。这可以通过结帐规则来完成。
    您还可以指定是否希望TeamCity签出代理或服务器上的源。请注意， 并非所有VCS都支持代理端签出，并且如果要使用它，则需要在至少一个代理上安装一个版本控制客户端。
    创建VCS根目录后，构建配置设置将显示在左侧。

  4. 现在，您可以通过选择左侧的相应设置来配置构建步骤 。您可以指示TeamCity在扫描存储库后自动检测构建步骤，或者按照本示例中的说明手动配置构建步骤。
    单击添加构建步骤，然后从下拉菜单中选择构建运行器。

  5. 填写必填字段并保存构建步骤。

如果项目驻留在多个版本控制系统中，则可以根据需要将任意数量的VCS Root附加到该项目。例如，如果将项目的一部分存储在Perforce中，而其余部分存储在Git中，则需要创建并附加两个VCS根目录：一个用于Perforce，另一个用于Git。了解有关配置VCS根的更多信息 。

恭喜你！您已经配置了包含一个构建步骤的第一个构建。现在，您可以运行构建并根据需要调整其设置 。

运行您的第一个版本

您的构建配置当前有一个构建步骤，现在您可以通过单击TeamCity Web UI右上角的“ 运行”来启动第一个构建：

您将被重定向到构建结果页面，您可以在其中查看构建进度并在构建完成时查看其结果。您还可以从此页面访问构建配置设置，并根据需要进行编辑：

您可以根据需要添加任意多个步骤，并根据需要对其重新排序。您可以手动添加步骤，也可以要求TeamCity自动检测它们。TeamCity还将建议设置，例如触发器，故障条件和构建功能。根据构建配置设置，它可能会提示一些其他选项。按照建议并添加设置以配置构建。
运行第一个版本后，您始终可以调整设置。

内部编号格式

TeamCity中的每个内部版本都有一个内部版本号，该内部版本号是一个字符串标识符，根据您的内部配置配置的“ 常规设置”页面上指定的模式组成（可通过单击“ 显示高级选项”链接获得该字段）。您可以在此处保留默认值，在这种情况下，构建号格式将由TeamCity维护，并将在每次新的构建开始时解析为下一个整数值。有关更多详细信息，请参见我们文档的专用部分 。
