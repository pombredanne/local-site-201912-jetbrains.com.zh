TeamCity 2019.1帮助

建立标记

构建标签是可以帮助您进行以下操作的标签：

  • 整理构建历史

  • 快速导航到带有特定标签的版本

  • 搜索带有特定标签的构建

  • 使用特定标签在构建中创建工件依赖项

您可以通过输入多个用空格，逗号，分号等分隔的标签，使用“ 编辑标签”对话框为单个版本分配任意数量的标签，例如“ EAP”或“发行”。

单击标签可滤除历史记录中的所有构建：仅显示标记有标签的构建。此外，您可以使用搜索字段来搜索带有特定标签的内部版本。

标记构建：

TeamCity Web UI提供以下标记特定构建的方法：

  • 使用构建配置主页 ：

    • 转到“ 概述”或“ 历史记录”选项卡

    • 转到构建历史记录表

    • 将标签列用于所需的构建并添加/修改构建标签

  • 将构建结果页面用于特定构建：

    • 单击操作按钮

    • 选择标签并添加/修改构建标签

  • 使用排队的构建页面：

    • 单击操作按钮

    • 选择标签并添加/修改构建标签

  • 使用“ 运行自定义构建”对话框

    • 转到“ 注释和标签”标签并添加/修改构建标签

  • 使用“ 固定” /“取消固定”构建对话框，除了固定构建外，还可以在其中标记构建。对于具有快照依赖关系的构建，有一个选项可以固定和标记该构建及其快照依赖关系
