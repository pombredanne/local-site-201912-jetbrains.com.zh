TeamCity加载项2019.2帮助

本地更改窗口

ReSharper | TeamCity |远程运行（本地更改）
Ctrl+Shift+Alt+T D

此窗口显示在TeamCity加载项选项中启用的源代码管理系统中未提交的更改。使用此窗口，可以在提交更改之前启动个人构建以测试您的更改。

工具栏控件

控制

名称

描述

配置个人构建

允许从所选更改开始个人构建 。

收集本地更改

更新窗口内容，以反映您在TeamCity加载项选项中启用的源代码管理系统中的本地更改。

全部展开 / 全部折叠

展开/折叠当前选项卡中的所有节点。

全部 选中 / 全部取消

选择/取消选择所有修改的文件。

按当前解决方案过滤

当未按下此按钮时，窗口将显示TeamCity加载项选项中配置的VCS的所有更改。
按下此按钮时，窗口仅显示解决方案中已配置的VCS的本地更改。
请注意，此选项仅影响此窗口中显示的本地未提交的更改。请不要将此选项与按解决方案过滤服务器更改混淆。

使用工作区版本预览差异 Ctrl+P

允许您在预览窗格中打开所选文件中更改的差异，然后将窗格放在窗口的右侧或底部。
要启用差异视图，您需要在选项（ ReSharper |选项| TeamCity |外部差异查看器）中配置外部差异查看器 。
请注意，仅Perforce和Subversion VCS支持差异预览。Team Foundation Server不支持统一差异。
