手动更改构建状态

在此页：

  • 总览
  • 将构建标记为成功
  • 将构建标记为失败
  • 权限

总览

具有适当权限的用户可以手动更改构建的状态，即使构建失败或成功（发行TW-2529 ）。

相应的操作在“ 构建结果”页面上的“操作”菜单中可用。

将构建标记为成功

您可能希望使构建成功完成：

  • 使用“ 构建失败”条件时 ，请更改最后一个成功的构建锚，即，如果上一个构建由于度量标准的值不正确而失败，并且此新值有效，则可以将该构建标记为成功的锚。

  • 允许使用在Artifact Dependencies中具有良好工件的不正确失败的构建。

  • 对于正在运行的Personal Build ，您可以将当前失败标记为不相关，以允许通过预测试的提交（如果用户有权执行此操作）。

“标记为成功”操作不适用于无法启动的构建 。

将构建标记为失败

在以下情况下，您可能希望将构建标记为失败：

  • 该版本存在一些问题，并没有影响最终版本的状态 。

  • 构建存在一个已知问题，您的质量检查小组应忽略该问题。

  • 您错误地将构建标记为手动成功。

权限

默认情况下，更改构建状态的权限被授予Project Administrator 。
