Ordering Projects and Build Configurations

By default, TeamCity displays projects, their subprojects, build configurations, and templates in the alphabetical order.

Project administrators can apply custom ordering to subprojects and build configurations on the Project Settings page for the parent project and use it as the default order. To enable reordering, click Reorder above the list of subprojects or build configurations.

Individual users can still manually tweak the display order using the up-down buttons of the Configure Visible Projects pop-up menu on the Projects page.
