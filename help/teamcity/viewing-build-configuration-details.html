查看构建配置详细信息

Build Configuration Home页面提供了配置详细信息，并使您能够：

  • 使用“ 运行”按钮运行自定义版本

  • 使用动作菜单
    • 暂停触发器

    • 检查未完成的更改

    • 强制执行干净结帐

    • 分配调查
  • 编辑配置设置

构建配置详细信息分为几个选项卡，其数量可能会根据您的服务器或项目配置而有所不同，例如， 依赖项 ， TeamCity与其他工具的集成等 。

默认情况下，以下选项卡可用：

  • 总览
  • 历史
  • 变更记录
  • 统计
  • 兼容代理商
  • 待更改
  • 设定值

总览

提供以下信息：

  • 待处理的更改也作为单独的标签列出，请参见下面的详细信息

  • 构建配置的当前状态 ，以及（如果适用）：

    • 排队构建的数量

    • 对于正在运行的构建–带有“ 停止”选项的进度详细信息以终止构建

    • 对于失败的构建 –数字和代理 ，依此类推

  • “ 构建历史记录”部分列出了当前构建配置的构建

历史

在单独的页面上显示“ 构建历史记录” ，并允许按构建代理过滤构建， 标记构建并按标签过滤构建（如果可用）。

变更记录

默认情况下，列出在过去14个工作日内完成的构建中的更改。使用显示全部链接查看完整的更改日志。

该页面显示更改日志及其提交到当前构建配置使用的所有VCS存储库的受监视分支以及当前配置的依赖项和从属配置使用的存储库的提交图。

统计

将收集的统计数据显示为可视图表 。

兼容代理商

列出所有授权的代理商。满足代理要求的代理被列为兼容。对于每个不兼容的代理，提供了原因。
首先列出属于与当前项目关联的池的代理。

待更改

在单独的页面上列出等待包含在下一个版本中的更改 。

设定值

在单独的页面上列出当前的构建配置设置 。
