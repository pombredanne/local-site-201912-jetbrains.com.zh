配置TeamCity服务器启动属性

可以通过在TeamCity服务器启动时传递的设置选项来自定义TeamCity行为的各个方面。这些选项分为两类：影响Java虚拟机（JVM）和影响TeamCity行为。

除非获得TeamCity支持团队的建议，或者您对自己的工作有信心，否则无需指定任何选项。

在此页：

  • TeamCity内部属性
    • JVM选项
    • 标准TeamCity启动脚本

TeamCity内部属性

TeamCity具有内部配置属性，这些属性会影响内部逻辑的各个方面。这些通常用于调试，更改内部常量或启用实验行为。除非TeamCity支持人员要求，否则请勿更改内部属性。

如果您自定义了内部属性，请在转至TeamCity支持时注意这一点。

服务器管理员可以在TeamCity Web UI中查看和编辑内部属性：转到“ 管理” | “ 管理”。服务器管理|在诊断页面上，选择内部属性标签，然后点击编辑内部属性 。
许多属性不需要重新启动服务器，但有些则需要。当需要重新启动时，我们通常会特别注意这一点。

属性存储在< TeamCity Data Directory >/config/internal.properties文件。该文件是Java 属性文件 。创建文件并添加必需的属性=在单独的行上。

另一种但过时的添加内部属性的方法是将其作为-D= JVM选项（请参阅下面的部分 ）。

JVM选项

如果您需要将其他JVM选项传递给TeamCity服务器（例如， -D 报告问题中提到的选项或任何非-D像-X... ），该方法将取决于服务器的运行方式。
如果您正在使用.war分发，请使用Web应用程序服务器的手册。
在所有其他情况下，请参阅下一部分。

有关内存设置的一般说明，请参阅设置TeamCity Server的内存设置 。

您将需要重新启动服务器以使更改生效。

标准TeamCity启动脚本

如果您使用runAll要么teamcity-server脚本或作为Windows服务，您需要通过传递到TeamCity服务器进程的OS 环境变量来设置选项：

  • TEAMCITY_SERVER_MEM_OPTS –服务器JVM内存选项（例如， -Xmx750m ）

  • TEAMCITY_SERVER_OPTS –其他服务器JVM选项（例如， -Dteamcity.git.fetch.separate.process=false ）

确保为其帐户用于运行TeamCity的用户或作为全局环境变量设置了环境变量。环境更改后，您可能需要重新引导计算机才能使更改生效。
