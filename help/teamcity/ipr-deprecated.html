Ipr赛跑者弃用

由于不推荐使用TeamCity 6.0 Ipr运行程序，而推荐使用另一种实现方法的IntelliJ IDEA项目运行程序 。在以下主要TeamCity版本之一中，所有具有Ipr运行程序的构建配置将自动转换为IntelliJ IDEA项目运行程序。由于运行程序在特定配置中的功能可能有所不同，因此强烈建议您将当前基于Ipr运行程序的配置更改为新运行程序，并在最终禁用Ipr运行程序之前检查您的设置。对于所有新创建的项目，也请使用IntelliJ IDEA项目运行器，如果您有任何问题，请告知我们。

除了所支持的IntelliJ IDEA项目功能的范围有所不同之外，运行者在运行测试和覆盖范围方面也有所不同。
即：

  • IntelliJ IDEA项目运行程序不支持EMMA覆盖范围。如果您使用的是EMMA，建议您迁移到IntelliJ IDEA Coverage引擎

  • 在IntelliJ IDEA项目运行程序中，JUnit测试是通过IntelliJ IDEA共享运行配置而不是Ant的运行配置启动的 Ipr亚军中的任务。

以下是建议的步骤，以执行从Ipr到IntelliJ IDEA项目运行器的迁移：

  1. 如果您现有的Ipr运行程序已配置了JUnit测试运行程序设置 ，则将该部分的所有设置备份到例如文本文件中。

  2. 如果您配置了代码覆盖率设置 ，请同时保存这些设置。（另请参阅相关问题 ）

  3. 将运行器类型更改为IntelliJ IDEA Project。除JUnit和代码覆盖率选项外，所有设置都将迁移。

  4. 要恢复JUnit测试，您将需要在IntelliJ IDEA中创建一个共享的运行配置，并将相应的文件提交到版本控件中。然后，可以在要执行的运行配置区域中指定运行配置的名称。

  5. 对于覆盖率，请使用保存的设置重新配置代码覆盖率选项。

检测全局库和基于模块的JDK *.iml档案

选择此选项时，打开和保存构建运行器设置的过程可能会很耗时，因为它涉及加载和解析所有项目模块文件。

附加的前/后处理（蚂蚁）

选项

描述

在构建之前运行

在适当的字段中，输入要在开始构建之前运行的Ant脚本和目标（可选）。Ant文件的路径应相对于项目根目录。

构建后运行

在适当的字段中，输入要在构建完成后运行的Ant脚本和目标（可选）。Ant文件的路径应相对于项目根目录。

JUnit测试运行器设置

JUnit测试设置映射到JUnit任务的属性。有关详细信息，请参阅Apache文档 。

选项

描述

测试图案

单击类型测试图案链接，然后在文本区域中指定所需的测试图案。
这些模式用于生成参数batchtest JUnit任务部分。每个模式都会生成include要么exclude部分。这些模式还用于组成测试运行的类路径。模式中提到的每个模块都将其类路径添加到整个类路径中。
每个模式应放在单独的行上，并具有以下格式：

[-] moduleName：[testFileNamePattern]

哪里：

  • [-] ：如果模式以减号开头，则相应的文件将从构建过程中排除。

  • moduleName ：此名称可以包含通配符。

  • [testFileNamePattern] ：的默认值testFileNamePattern是**/*Test.java ，即所有目录中所有以Test.java结尾的文件。您可以将Ant语法用于文件模式。下面的示例包括以“ test”结尾的模块中的所有测试文件，并从包含“ ui”子软件包的程序包中排除所有文件：

* test：** / * Test.java-*：** / ui / ** / *。java

搜索测试

在IDEA项目中，用户可以将源代码文件夹标记为sources要么test根。此下拉列表允许您指定目录以查找测试：

  • 遍及所有项目资源 ：在两个项目中均进行测试sources和test IDEA项目的文件夹。

  • 仅在测试源中 ：浏览标记为仅测试根目录的文件夹。

测试中的类路径

默认情况下，整个类路径由用于获取测试的模块的所有类路径组成。以下两个选项定义是使用默认的类路径，还是从指定的模块中获取它。

覆盖测试中的类路径

如果选中此选项，则可以从单个明确指定的模块定义测试类路径。

从中使用JDK和classpath的模块名称

如果选中了“ 在测试中覆盖类路径 ”选项，则必须指定模块，在该模块中指定要用于测试的类路径。

JUnit分叉模式

从组合框中选择所需的分叉模式：

  • 不要叉 ：叉已禁用。

  • 每个测试的Fork ：启用fork，并且每个测试类在单独的JVM中运行

  • 一次Fork ：启用fork，并且所有测试类都在单个JVM中运行

每个测试的新类加载器实例

如果要为每个测试用例实例化新的类加载器，请选中此选项。仅当选择了不分叉选项时，此选项才可用。

包括Ant运行时

选中此选项以添加运行JUnit测试所需的Ant类。如果启用了分叉模式（ 每个测试分叉或一次分叉），则此选项可用。

JVM可执行文件

指定将用于调用JVM的命令。如果启用了分叉模式（ 每个测试分叉或一次分叉），则此选项可用。

停止建立错误

如果您希望在测试运行期间发生错误时停止构建，请选中此选项。

JUnit的JVM命令行参数

指定要传递给JUnit任务的JVM参数。

测试工作目录

指定测试工作目录的路径。

测试超时

指定经过的时间（以毫秒为单位），此后将取消测试。如果选择了不分叉选项，则将忽略此值。

减少测试失败的反馈时间

使用以下两个选项来指示TeamCity先进行一些测试。

测试重新排序的工作方式如下：TeamCity提供应首先运行的测试（测试类），之后，当JUnit任务启动时，它将检查是否包括这些测试。如果至少包含一个测试，TeamCity将生成一个仅包含所包含测试的新文件集，并在所有其他文件集之前对其进行处理。它还修补其他文件集，以排除添加到自动生成的文件集的测试。之后，JUnit将启动并照常运行。

首先运行最近失败的测试

如果选中，首先，TeamCity将运行在先前完成或正在运行的构建中失败的测试以及失败率较高的测试（所谓的闪烁测试）

首先运行新的和修改的测试

如果选中，在进行任何其他测试之前，TeamCity将运行在正在运行的内部版本中包含的更改列表中添加或修改的测试。

如果同时启用这两个选项，则新的和修改的测试组的测试将具有更高的优先级，并将首先执行。

详细蚂蚁

如果生成的JUnit任务必须以蚂蚁语生成详细输出，请选中此选项。
