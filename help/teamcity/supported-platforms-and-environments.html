平台（操作系统）

TeamCity服务器

TeamCity服务器的核心功能与平台无关。见考虑在选择服务器平台。

TeamCity服务器是一个Web应用程序，可在功能强大的J2EE Servlet容器中运行。

该服务器需要运行Java SE JRE安装。请参阅有关如何在服务器上安装Java的注释 。

由于TeamCity的2018.2，OpenJDK的8（或1.8）通过AdoptOpenJDK被包括在Windows .exe TeamCity发行版（以前，Oracle Java与TeamCity Windows发行版捆绑在一起）。将TeamCity升级到2018.2。后，捆绑版本的JRE的用户将自动切换到OpenJDK。

支持的Java版本是OpenJDK和Oracle Java 8（32或64位），版本8u16（或1.8.0_16）或更高版本。我们建议使用最新的JDK 8版本。
建议将64位Java用于不重要的生产用途。

请注意.war发行将被终止，强烈建议使用.tar.gz发行版，其中已捆绑了Tomcat Web服务器。除非绝对必要，否则不建议自定义Tomcat设置。如果仍要使用.war发行版，请注意：

  • 仅在Apache Tomcat 8.5+下对TeamCity进行了测试，不支持早期的Tomcat版本。

  • TeamCity旨在支持基于Apache Jasper的J2EE Servlet 3.0+和JSP 2.2+容器。

  • 如果由于APR中的问题而在Tomcat中启用了Apache Portable Runtime，则TeamCity可能无法正常工作。

通常， 支持Windows，Linux和macOS的所有最新版本 。如果您发现与任何操作系统的兼容性问题，请确保告知我们 。

TeamCity服务器在以下操作系统上经过测试：

  • Linux（Ubuntu，Debian，RedHat，SUSE等）

  • 苹果系统

  • Windows 7 / 7x64

  • Windows Server 2008、2012、2016、2019年

  • Windows Server 2016的服务器核心安装

  • Tomcat 8.5+ Web应用程序服务器下的Windows 10。

据报道在以下方面没有已知问题地起作用：

  • Windows 7+

  • Windows Server 2008 R2

  • 的Solaris

  • FreeBSD

  • IBM z / OS

  • HP-UX

请注意，不支持Windows XP / XP x64。

建立代理

TeamCity代理是一个独立的Java应用程序。

构建代理要求运行Java SE JRE安装。请参阅有关如何在代理上配置Java的注释 。

由于TeamCity的2018.2，OpenJDK的8（或1.8）通过AdoptOpenJDK被包括在Windows .exe TeamCity发行版（以前，Oracle Java与TeamCity Windows发行版捆绑在一起）。将TeamCity升级到2018.2。后，捆绑版本的JRE的用户将自动切换到OpenJDK。

支持的Java版本是OpenJDK和Oracle Java 8-10。我们建议使用最新的JDK 8（或1.8）版本。不支持在Java 1.6和1.7下运行代理。

TeamCity代理已在以下操作系统上经过测试：

  • 的Linux

  • 苹果系统

  • Windows 7 / 7x64

  • Windows 10

  • Windows Server 2003 / 2008、2012、2016、2019年

  • Windows Server 2016的服务器核心安装

据报道适用于：

  • Windows XP / XP x64

  • Windows 2000（仅交互式模式）

  • 的Solaris

  • FreeBSD

  • IBM z / OS

  • HP-UX

Windows托盘通知程序

Windows 7 / 7x64 / 10，具有Internet Explorer支持的版本之一。

网页浏览器

TeamCity Web Interface大多与W3C兼容，因此几乎所有现代浏览器都可以与TeamCity很好地配合使用。以下浏览器已经过专门测试，并报告在以下最新版本中可以正常工作：

  • 谷歌浏览器

  • 火狐浏览器

  • Mac下的Safari

  • 微软边缘

  • Microsoft Internet Explorer 11+。自TeamCity 2018.1.4起停止对Internet Explorer 10的支持。

  • 歌剧15+

建立跑步者

TeamCity支持广泛的构建工具，使Java和.NET软件团队均可构建其项目。

受支持的Java构建运行器

  • Ant 1.6-1.10 TeamCity 2018.2与Ant 1.9.11捆绑在一起。

  • Maven版本2.0.x，2.x，3.x（在TeamCity发行时为已知）。支持Java 1.5及更高版本。TeamCity与Maven 2.2.1、3.0.5、3.1.1、3.2.5、3.3.9和3.5.4捆绑在一起。

  • IntelliJ IDEA项目运行器 （需要Java 8）

  • Gradle （需要Gradle 0.9-rc-1或更高版本）

  • 基于IntelliJ IDEA的Java检查和Java复制 （需要Java 8）

受支持的.NET平台构建跑步者

  • MSBuild 。需要在构建代理上安装.NET Framework或Mono。还支持Microsoft Build Tools 2013、2015、2017和2019。

  • NAnt版本0.85-0.91 alpha 2。需要在构建代理上安装.NET Framework或Mono。

  • 微软的Visual Studio解决方案 （ 2003年 - 2015年，2017年，2019年）。需要在生成代理上安装相应版本的Microsoft Visual Studio。

  • FxCop 。需要在构建代理上安装FxCop。

  • 基于ReSharper命令行工具的 C＃和VB.NET代码重复查找器 。支持的语言是C＃到4.0版本以及VB.NET 8.0-10.0版本。需要在构建代理上安装.NET Framework 4.6.1+。

  • 基于ReSharper命令行工具的 .NET检查 。需要在构建代理上安装.NET Framework 4.6.1+。

  • .NET Process Runner，用于运行任何.NET应用程序（需要在构建代理上安装的.NET）

  • Windows，Linux macOS下的NuGet运行程序。需要在代理上安装NuGet.exe命令行工具。Windows下支持的NuGet版本为1.4+。

    • Windows：2.8.6之前的NuGet版本要求在构建代理上安装.NET Framework 4.0+

    • Windows：NuGet 2.8.6和更高版本需要.NET 4.5

    • Linux和macOS：需要在代理上安装Mono 4.4.2+和NuGet CLI 3.2+

  • .NET CLI（dotnet） 。需要在构建代理上安装.NET Core SDK 。

其他跑步者

  • 耙

  • 命令行运行程序，用于使用Shell脚本运行任何构建过程

  • 支持PowerShell版本1.0到5.0。

  • 支持 Xcode 3-8版本（需要在构建代理上安装Xcode）

测试框架

  • JUnit 3.8.1 +，4.x

  • 支持NUnit 2.2.10、2.4.x，2.5.x，2.6.x，3.0.x（专用构建运行器）。

  • TestNG 5.3+

  • Visual Studio Tests运行程序支持MSTest 8.x-12.x，14.x，15.x，19.x和VSTest。需要在生成代理上安装适当的Microsoft Visual Studio版本或Visual Studio测试代理。

  • MSpec （需要在构建代理上安装的MSpec）

版本控制系统

  • Git （用于自动git gc支持需要在服务器上安装Git客户端才能执行Git克隆的维护，建议使用最新版本）

  • Subversion （服务器版本1.4-1.9及更高版本，只要协议向后兼容即可）。

  • Perforce （需要安装在TeamCity服务器上的Perforce客户端）。检查兼容性问题 。

  • 支持Team Foundation Server 2005、2008、2010、2012、2013、2015、2017。

  • Mercurial （要求在服务器上安装Mercurial“ hg”客户端v1.5.2 +）

  • CVS

  • SourceGear Vault 6和7（需要安装在TeamCity服务器上的Vault命令行客户端库）

  • Borland StarTeam 6及更高版本（必须在TeamCity服务器上安装StarTeam客户端应用程序）

  • IBM Rational ClearCase ，Base和UCM模式（需要在TeamCity服务器上安装和配置的ClearCase客户端）

  • Microsoft Visual SourceSafe 6和2005（需要安装在TeamCity服务器上的SourceSafe客户端，仅在Windows平台上可用）

要支持其他VCS，请检查可用的外部插件 。

IDE整合

TeamCity为以下IDE提供了生产力插件：

  • Eclipse ：支持Eclipse 3.8和4.2-4.6版本。Eclipse必须在JDK 1.5+下运行

  • IntelliJ平台插件 ：与IntelliJ IDEA 15.0.x-2018.3.x兼容（最终版和社区版）;以及基于同一平台版本的其他IDE，包括JetBrains RubyMine 6.3 +，JetBrains PyCharm 3.1 +，JetBrains PhpStorm / WebStorm 7.1 +，AppCode 2.1+。查看有关兼容性的更多信息 。

  • ReSharper Ultimate随附的TeamCity Visual Studio 加载项支持Microsoft Visual Studio 2010、2012、2013、2015、2017。必须安装.NET Framework。
