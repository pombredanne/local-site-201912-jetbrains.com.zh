附加功能

Microsoft Visual Studio的TeamCity加载项提供以下功能：

  • 远程运行的TFS，Subversion和Perforce公司（为水银和Git远程运行看科远程运行触发 ）

  • TFS，Subversion和Perforce的预测试（延迟）提交

  • 从TeamCity服务器（请参阅更多信息 ） 中将JetBrains dotCover覆盖率分析数据提取到MS Visual Studio（需要在Visual Studio中安装受支持版本的 dotCover）

  • 在“ 我的更改”工具窗口中查看最近提交的更改和个人构建及其构建状态

  • 从TeamCity Web UI在MS Visual Studio中打开构建失败详细信息

  • 查看构建失败测试的详细信息

  • 通过ReSharper测试运行程序在本地TeamCity构建中重新运行测试失败

  • 从IDE导航到构建结果网页

  • 重新应用在“远程运行”或“经过预先测试的提交”中发送的更改到工作目录

有关详细说明，请参阅TeamCity外接程序联机帮助 。

若要通过使用Web UI中的“在IDE中打开”操作来导航到MS Visual Studio中失败的测试，请确保.pdb NUnit / MSTest单元测试中涉及的程序集的文件生成在当前的Visual Studio项目中已打开。

安装加载项

  1. 在开始加载项安装（初始或升级）之前，请关闭Visual Studio的所有正在运行的实例。

  2. 导航到Visual Studio加载项的下载页面：

    • 点击TeamCity网络用户界面右上角用户名旁边的箭头，然后选择我的设置和工具 。

    • 在右侧的TeamCity工具部分中，单击Visual Studio加载项下载链接。

TeamCity Visual Studio加载项作为ReSharper Ultimate产品捆绑包的一部分提供。安装后，TeamCity加载项将在Visual Studio的RESHARPER菜单下提供。

安装程序将删除捆绑前的产品版本：9.0之前的TeamCity和ReSharper版本，3.0之前的dotCover，6.0之前的dotTrace。ReSharper Ultimate不支持Visual Studio版本2005和2008。

从2005年到2013年，用于Visual Studio版本的TeamCity VS外接程序的旧版本与ReSharper 9.0，dotCover 3.0和dotTrace 6.0之前的JetBrains .NET工具兼容，因为TeamCity 10.0不可用。
