FTP上传

FTP上传允许将文件/目录部署到FTP服务器。

在配置构建步骤中介绍了所有运行程序的通用设置。此页面详细介绍了FTP上传设置。

以下字段支持参数引用 ：百分号之间的任何文本（ % ）被TeamCity视为对财产的引用。为防止TeamCity将百分号中的文本作为属性引用，请使用两个百分号对其进行转义：例如，如果要传递“ %Y%m%d%H%M%S在构建中，将其更改为%%Y%%m%%d%%H%%M%%S ”。

选项

描述

部署目标

目标主机

指定一个FTP服务器（usehostname或IP地址）和一个远程目录（相对于FTP用户的主目录）。

使用绝对*nix路径，使用%2F作为正斜杠。例如：

ftp://hostname.com/hostname.com:34445/subdir127.0.0.1/%2Fetc/

安全连接

在不安全（FTP）和安全连接（FTPS，FTPES）之间选择。

部署凭证

身份验证方法

选择任一匿名 （将提交用户名anonymous和一个空格作为密码）或用户名/密码 （用于自定义凭据）

FTP模式

FTP模式

选择被动或主动模式

传输模式

可选的。选择要强制执行的FTP传输模式：ASCII或二进制FTP传输模式（如果自动检测到的模式导致文件传输中断）

部署来源

来源途径

将部署源指定为文件/目录路径的换行符或逗号分隔列表。

该字段支持Ant样式的通配符模式 （例如， dir/**/*.zip ）。
您也可以使用以下命令指定要创建的目标目录file => directory模式（例如， *.zip => winFiles,unix/distro.tgz => linuxFiles将创建winFiles和linuxFiles目录，并将声明的文件分别放入其中）。
