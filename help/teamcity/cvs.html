此处介绍了常见的VCS根属性。

本页包含CVC特定字段的说明以及设置VCS根目录时可用的选项。根据所选的访问方式，页面显示不同的字段，可帮助您轻松定义CVS设置 ：

CVS根

选项

描述

模块名称

指定CVS管理的模块的名称。

CVS根

使用这些字段选择访问方法，指向用户名，CVS服务器主机和存储库。例如： :pserver:user@host.name.org:/repository/path 。
对于本地连接，仅应使用CVS存储库的路径。
TeamCity支持以下连接方法（如下所述）：

  • 服务器
  • 分机
  • ssh
  • 本地

结帐选项

选项

描述

结帐HEAD版本

从分支结帐

按标签结帐

定义CVS填写和更新工作目录的方式

安静期

由于CVS中没有原子提交，因此使用此设置，您可以指示TeamCity在检测到上一次更改以来经过指定的时间段之前，不要进行（检测）更改。当TeamCity Web UI中一次提交显示为两个不同的更改时，它可以避免出现这种情况。

PServer协议设置

选项

描述

CVS密码

如果要通过输入密码访问CVS存储库，请单击此单选按钮。

密码文件路径

单击此单选按钮以指定到.cvspass文件。

连接超时

指定连接超时。

扩展协议设置

选项

描述

外部rsh的路径

指定用于连接到存储库的rsh程序的路径。

私钥文件的路径

指定包含用户身份验证信息的文件的路径。

附加参数

输入将传递给程序的所有必需的rsh参数。多个参数用空格分隔。

服务器命令

键入服务器命令。server命令是使CVS客户端在服务器模式下工作的命令。默认值为server 。

高级选项

选项

描述

使用GZIP压缩

选中此选项可将gzip压缩应用于CVS服务器和CVS客户端之间传递的数据。

将所有环境变量发送到CVS服务器

选中此选项可将环境变量发送到服务器，以与某些服务器兼容。

历史命令支持

选中此选项可使用服务器上的历史记录文件搜索新提交的更改。启用此选项可提高CVS支持性能。默认情况下，不选中该选项，因为并非每个CVS服务器都支持保留历史记录日志文件。
