TeamCity提供了许多构建参数 ，这些参数可随时用于构建配置的设置或构建脚本中。

预定义的构建参数可以源自多个范围：

  • 服务器构建属性 –由TeamCity在特定构建范围内的服务器端生成的参数。此类属性的一个示例是内部版本号。

  • 代理属性 –代理在与服务器连接时提供的参数。这些参数并非特定于任何构建并表征代理环境（例如，.NET框架的路径）。这些主要用于代理要求 。

  • 代理构建属性 –在开始构建之前，在特定构建范围内的代理端提供的参数。例如，具有已更改文件列表的文件路径。
    所有这些参数最终都传递给构建。

还有一种特殊的服务器端构建参数，可在定义其他参数时在引用中使用，但这些参数不会传递到构建中。有关此类属性的列表，请参见下面的配置参数 。

在定义支持参数的文本值时，可以在TeamCity Web UI中获取最新的参数列表： 文本字段右侧的图标，或输入“ %在文本字段中。

分支相关参数

当TeamCity在配置分支规范的构建配置中开始构建时，它将为每个构建添加一个分支标签。此逻辑分支名称也可用作配置参数：

teamcity.build.branch

为了区分在默认分支和非默认分支上启动的构建，自7.1.5开始有一个附加的布尔配置参数可用于区分以下情况：

teamcity.build.branch.is_default = true | false

对于Git＆Mercurial，TeamCity提供了附加参数，以及在构建开始时已知的VCS分支名称。请注意，这些可能与根据配置的分支规范的逻辑分支名称不同。可通过具有以下名称的配置参数使用此VCS分支：

teamcity.build.vcs.branch。

哪里是VCS根ID，如“ 配置VCS根”页面中所述。

仅当环境变量不存在于环境变量中时才进行定义：如果启动的代理程序已经设置了与Java相关的环境变量，则不会重新定义它们。

定义与Java相关的环境变量

对于Java的每个主要版本V ，可以定义以下变量：

  • JDK_1V
  • JDK_1V_x64
  • JRE_1V
  • JRE_1V_x64

在找到JDK时定义JDK变量，在找到JRE时定义JRE变量，但没有找到JDK。
_x64变量仅指向64位Java。没有_x64后缀的变量可能指向32位或64位安装，但是32位安装是首选。
如果找到具有相同主要版本和相同位数但具有不同次要版本/更新的多个安装，则选择最新版本。

此外，还定义了以下变量：

  • JAVA_HOME –对于最新的JDK安装（但最好使用32位）

  • JDK_HOME –与JAVA_HOME

  • JRE_HOME –对于最新的JRE或JDK安装（但最好是32位安装），即使找到了JDK也已定义。

的JRE_HOME和JDK_HOME变量可能指向不同的安装；例如，如果JRE 1.7和JDK 1.6但未安装JDK 1.7 – JRE_HOME将指向JRE 1.7并JDK_HOME将指向JDK 1.6。

所有变量都指向Java主目录，而不是二进制文件。例如，如果要执行javac 1.6版，则可以使用以下路径：

在TeamCity构建配置中：

％env.JDK_16％/ bin / javac

在Windows bat / cmd文件中：

％JDK_16％bin \ javac

在unix shell脚本中：

$ JDK_16 / bin / javac
