MSBuild

本页包含有关MSBuild构建运行器字段的参考信息。

在此页：

  • 常规构建运行器选项
  • 代码覆盖率

MSBuild运行程序需要在构建代理上安装.NET Framework或Mono。支持Microsoft Build Tools 2013-2019。

在设置构建配置以将MSBuild用作构建运行器之前，请确保您将XML构建项目文件与MSBuild运行器一起使用。

要生成Microsoft Visual Studio解决方案文件，可以使用Visual Studio（sln）生成运行器 。

常规构建运行器选项

选项

描述

构建文件路径

指定相对于build checkout目录要构建的解决方案的路径。例如， vs-addin\addin\addin.sln 。

工作目录

可选的。如果构建工作目录与构建签出目录不同，请指定其路径。

MSBuild版本

选择MSBuild版本：.NET Framework，Mono xbuild或Microsoft Build Tools。

MSBuild ToolsVersion

在此处指定将用于编译的工具的版本（等同于/toolsversion:命令行参数）。

MSBuild支持编译到较早的版本，因此您可以将MSBuild版本设置为4.6.1，并将MSBuild ToolsVersion设置为2.0，以在从.NET Framework 4.6.1运行MSBuild时生成.NET 2.0程序集。有关更多信息，请参阅本文。

运行平台

从下拉列表中选择x64计算机上所需的执行模式。

目标

目标是用于项目目的的任意脚本。输入目标，以空格分隔。通过单击字段旁边的图标，可以在Web UI中查看可用的目标，然后选中相应的框来添加可用的目标。

命令行参数

指定以下任何其他参数MSBuild.exe.

减少测试失败的反馈时间

使用此选项指示TeamCity运行在先前版本中失败的测试，然后再进行其他测试。

代码覆盖率

要了解有关配置代码覆盖率选项的信息，请参阅“ 配置.NET代码覆盖率”页面。

实施说明

MSBuild运行器生成一个包含用户脚本的MSBuild脚本。此脚本用于添加TeamCity提供的MSBuild任务。您的MSBuild脚本将包含在任务中。如果指定了Visual Studio解决方案文件，则将从任务中调用该文件。要禁用它，请设置teamcity.msbuild.generateWrappingScript至false 。
