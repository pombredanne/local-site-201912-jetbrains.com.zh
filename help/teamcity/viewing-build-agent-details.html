查看构建代理详细信息

要查看有关代理的状态和信息，请单击其名称或导航到“ 代理”页面，在已连接，断开连接或授权的代理列表中找到该代理，然后单击其名称。

TeamCity为每个连接的代理提供以下信息：

  • 代理商摘要
    • 代理重启
  • 建立历史
  • 兼容配置
  • 建立跑步者
  • 代理日志
  • 座席参数

代理商摘要

  • 状态 ： 了解有关座席状态的更多信息 。

  • 详细资料 ：

    • 代理商名称

    • 代理主机IP地址

    • TeamCity服务器用于连接到代理的端口

    • 代理程序和服务器之间用于数据传输的通信协议

    • 代理操作系统

    • CPU等级：捆绑的CPU基准测试的结果。请注意，基准测试结果可能取决于代理使用的JVM版本和选项。例如， -server JVM选项对结果有重大影响。CPU基准测试还影响服务器在代理之间分发内部版本的方式。如果无法为代理计算构建持续时间估算值（该代理的历史记录中没有构建），TeamCity将根据CPU基准值选择最快的代理。

    • 代理所属的池

    • 代理启动器版本

  • 如果代理程序上有正在运行的内部版本 ，则页面将显示有关内部版本的信息以及构建结果的链接。

  • 杂项 ：此部分提供了以下选项：
    • 清洁代理上的来源

    • 打开远程桌面：如果在代理上安装并注册了RDP客户端，则适用于Windows操作系统上运行的代理。

    • 重新启动代理计算机：可供具有重新启动构建代理计算机权限的用户使用。单击链接并确认重新启动操作。默认情况下，TeamCity代理将等待，直到当前构建完成。取消选中该复选框，然后单击“重新启动”以立即重新启动代理。
      重新启动命令的其他配置是可能的。请参阅代理重新启动 。

    • 在代理上转储线程

代理重启

代理重新启动是通过执行特定于操作系统的命令来执行的。在某些情况下，该命令可能需要特定于OS环境的自定义。如果默认的重新启动命令失败，则可能需要其他配置。

要调整代理重新启动，请添加teamcity.agent.reboot.command代理程序配置参数buildagent.properties文件，其中包含需要重新引导时执行的命令。配置示例：

teamcity.agent.reboot.command = sudo关闭-r 60

要么

teamcity.agent.reboot.command =关闭-r -t 60 -c“ TeamCity Agent重新启动命令” -f
