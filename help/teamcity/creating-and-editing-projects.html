本节详细介绍通过TeamCity Web UI创建项目。其他选项包括REST API和基于Kotlin语言在DSL中使用TeamCity项目配置。

建立专案

要创建项目，请使用右上角的“ 管理”链接，然后单击“ 创建项目” 。系统显示“ 创建项目”页面。

创建项目有几种选择：

  • 从存储库URL
  • 来自GitHub.com
  • 从Bitbucket Cloud
  • 从GitLab

  • 来自Azure DevOps Services或以前的Visual Studio Team Services

  • 手动地

请注意，默认情况下只有两个选项可用： 从存储库URL和手动 。如果在Root项目（或要创建的项目的父项目）中配置了与某些VCS托管服务的连接 ，则相应的选项将变为可用，因此您可以使用现有的VCS连接规范来创建项目。

要创建子项目，请转到父项目的“ 项目设置”页面，并使用可用选项之一，类似于创建项目。

创建指向存储库URL的项目

1。在“ 创建项目”页面上，选择从存储库URL创建项目。

2。指定项目设置：

该字段旁边的图标代表TeamCity支持的VCS托管服务。如果单击活动的（突出显示的）图标，则可以选择现有的连接规范。如果单击非活动图标，您将被重定向到“ 添加连接”表单。

3。单击继续 。TeamCity将为您配置其余设置。

  • 它将确定VCS储存库的类型，自动配置VCS储存库设置，并建议项目和构建配置名称：

  • 项目，构建配置和VCS根目录将自动创建

  • TeamCity将添加VCS构建触发器。

根据构建配置设置，TeamCity可以建议一些其他配置选项。查看设置列表末尾的建议并配置所需的建议。

创建指向GitHub.com存储库的项目

  1. 在“ 创建项目”页面上，选择从GitHub.com创建一个项目。

  2. 选择一个存储库。TeamCity将验证存储库连接。如果连接经过验证，将打开新页面。

  3. TeamCity将显示项目并生成配置名称。如果需要，请修改名称，然后单击继续 。

  4. TeamCity将添加VCS构建触发器，并尝试自动检测构建步骤：Ant，NAnt，Gradle，Maven，MSBuild，Visual Studio解决方案文件，PowerShell，Xcode项目文件，Rake和IntelliJ IDEA项目。
    在“ 自动检测到的构建步骤”页面上，选择要在构建配置中使用的检测到的步骤。单击使用选定的 。
    如果未找到任何步骤，则必须手动配置构建步骤 。

  5. 您的项目和构建配置已配置。单击运行开始构建。根据构建配置设置，TeamCity可以建议一些其他配置选项。查看设置列表末尾的建议并配置所需的建议。

3。点击创建 。创建一个空项目。

要配置现有项目，请在列表中选择所需的项目，然后单击右侧的“ 编辑项目设置”链接。

4。为项目创建构建配置 （选择构建设置， 配置VCS设置 ，然后选择构建运行器 ）。

管理项目

默认情况下，您可以在按字母顺序列出的“ 项目”页面上查看所有可用的项目和子项目。管理员可以自定义默认顺序 。

要复制，移动，删除或存档项目，请使用“ 项目设置”页面右上角的“ 操作”菜单或“ 更多”按钮父项目设置页面上项目旁边的项目。这些选项不适用于Root项目。

复制项目

使用“ 项目设置”页面右上角的“ 动作”菜单中的相应项目或“ 更多”按钮父项目设置页面上项目旁边的项目。

项目管理员可以复制项目并将其移至另一个项目。

副本将复制原始项目的所有设置， 子项目 ， 构建配置和模板 ，但不会保留与构建相关的数据。使用空的构建历史记录创建副本，并且没有统计信息 。

您可以将项目复制到相同或另一个父项目中。

复制时，TeamCity自动为副本分配一个新名称和ID 。也可以手动更改名称和ID。
选择“ 复制与项目相关的用户，代理和其他设置”选项可确保所有受影响的用户和代理的复制和原始项目的所有设置（如通知规则或代理的兼容性）都完全相同。

您还可以选择复制内部配置的内部版本号。

在专业模式下运行TeamCity时，如果复制后服务器上的构建配置数将超过限制（自TeamCity 2017.2起为100个构建配置，而在较早版本中为20个，则除非您购买了其他Build，否则不会为项目显示“ 复制”选项。代理许可证）。

搬迁项目

在移动项目之前，请考虑以下事项：

  • TeamCity根据每个项目分配用户角色，这意味着移动项目可能会导致更改新项目中用户权限的范围 （可以添加新权限或可以删除现有权限）

  • 项目移动后，包含SSH密钥的Git VCS根目录可能无法使用。

要移动项目，请使用“ 项目设置”页面右上角的“ 动作”菜单中的相应项目或“ 更多”按钮父项目设置页面上项目旁边的项目。

移动项目时，TeamCity会保留其所有设置， 子项目 ， 构建配置 / 模板 ，关联数据以及构建历史 。

归档项目

使用“ 项目设置”页面右上角的“ 动作”菜单中的相应项目或“ 更多”按钮父项目设置页面上项目旁边的项目。请参阅专用页面 。

批量修改ID

执行此操作时必须小心。修改ID将更改与项目相关的所有URL。强烈建议在脚本中加书签或硬编码的任何URL中更新ID。磁盘上的相应配置和工件目录名也将更改，这可能需要一些时间。

  1. 使用“ 项目设置”页面右上角的“ 动作”菜单中的相应项目或“ 更多”按钮父项目设置页面上项目旁边的项目。

  2. 显示当前项目和构建配置ID 。您可以修改或重置所有子项目，VCS根目录，构建配置和模板的ID。单击重新生成以自动获取新ID或手动编辑它们。

  3. 点击提交 。

暂停/激活触发器

您可以暂停项目的所有或选定构建配置的触发器 。使用“ 项目设置”页面右上角的“ 动作”菜单中的相应项目或“ 更多”按钮父项目设置页面上项目旁边的项目。

出口项目

您可以导出项目的配置文件及其子项，以将其移动到其他TeamCity服务器。使用“ 项目设置”页面右上角的“ 动作”菜单中的相应项目或“ 更多”按钮父项目设置页面上项目旁边的项目。

删除专案

使用“ 项目设置”页面右上角的“ 动作”菜单中的相应项目或“ 更多”按钮父项目设置页面上项目旁边的项目。

删除项目时，TeamCity将删除其.xml配置文件。删除后，项目将移至< TeamCity数据目录 > / config / _trash /。ProjectID.projectN目录。在下次清除构建历史记录期间，已删除项目的数据库中存储的所有与项目相关的数据（构建历史记录，工件等）之前，都有一个可配置的超时（默认为5天）。您可以在运行清理之前还原已删除的项目。

< TeamCity数据目录 > / config / _trash /目录不会自动清除，如果您确定不需要删除的项目，可以手动将其清空。

如果您尝试从其他项目中删除具有相关生成配置的项目，TeamCity会警告您。如果继续删除，则依赖项将不再起作用。
