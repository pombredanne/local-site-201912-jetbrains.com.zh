.NET Process Runner

.NET Process Runner可以在选定的.NET Framework版本和平台下运行任何.NET程序集，并且可以选择覆盖.NET代码。您可以使用它来运行xUnit，Gallio或其他.NET测试，而这些测试没有专用的构建运行器。

在此页：

  • .NET Process Runner设置
  • 代码覆盖率

运行程序需要在TeamCity代理上安装.NET Framework。

.NET Process Runner设置

选项

描述

路径

指定.NET可执行文件的路径（例如，xUnit控制台的路径）

命令行参数

提供以换行符或空格分隔的命令行参数，以传递给可执行文件。

工作目录

如果构建工作目录与Build Checkout目录不同，则指定它。

.NET运行时

从“ 平台”下拉列表中，在x64计算机上选择所需的执行模式。支持的值为：自动（MSIL）（默认），x86和x64。
从“ 版本”下拉列表中，选择所需的.NET Framework版本。

如果您具有MSIL .NET 2.0 / 3.5可执行文件，TeamCity可以强制其在任何所需的运行时下执行：x86或x64，以及.NET 2.0或.NET 4.0运行时。

代码覆盖率

如果需要， 添加代码覆盖率 。

请注意，您无需编写任何其他构建脚本。
