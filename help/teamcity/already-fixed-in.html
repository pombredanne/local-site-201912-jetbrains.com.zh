对于某些测试失败，TeamCity可以显示“已修复”构建。

这是该初始失败的测试成功的构建，并且是在初始测试失败的构建之后运行的（对于相同的Build Configuration ）。

“之后”在这里意味着

  • 测试成功的版本比初始失败的版本具有更新的更改

  • 如果更改相同，则在失败的版本之后运行较新的版本

因此，如果您运行History Build ，TeamCity不会将其视为“已修复”的候选对象，以解决以后的构建中的测试失败（根据更改）。

名称相同的测试将被视为相同。如果在内部版本中有多个具有相同名称的测试，TeamCity将对具有相同名称的测试的订单号进行计数并予以考虑。

请注意， 自TeamCity 9.0起 ，TeamCity计数测试Hajipur 9.0 EAP1（内部版本31423）发行说明的方式 。
