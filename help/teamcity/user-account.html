User Account

User account is a combination of username and password that allows TeamCity users to log in to the server and use its features. User accounts can be created manually, or automatically upon log in depending on used authentication scheme (refer to Authentication Modules and LDAP Integration sections for more details).

Each user account:

  • Has an associated role that ensures access to all or specific TeamCity features through corresponding permissions. Learn more about Role and Permission.

  • Belongs to at least one user group. Learn more about user group.

In addition to logged in users, there is a special user account for non-registered users called Guest User, that allows monitoring TeamCity projects without authorization. By default, guest login is disabled. Learn more on the Guest User page.
