处理构建结果

在TeamCity中，构建会经历几种状态：

  • 在发生某些事件时，构建触发器会将构建添加到队列中，其中构建保持等待自由代理。

  • 构建从代理开始，并执行所有已配置的构建步骤。

  • 构建完成，并成为此构建配置的构建历史的一部分。

在TeamCity中，有关特定构建的所有信息（无论是已排队，正在运行还是已完成）都累积在构建结果页面上。单击内部版本号或内部状态链接可以访问该页面。

除了提供构建信息之外，此页面还使您能够：

  • 使用“ 运行”按钮运行自定义版本

  • 使用“ 操作”菜单执行以下操作：

    • 将构建添加到收藏夹

    • 添加评论

    • 固定构建
    • 标记构建

    • 更改构建地位，标志着构建的失败或成功

    • 标记此构建源

    • 删除构建

  • 编辑配置设置

构建细节

可以从构建配置主页访问构建结果页面，也可以从TeamCity Web UI的不同位置访问构建结果页面，其中，构建编号或构建状态显示为链接。某些数据仅在构建完成后才可访问，某些详细信息（例如Change ， Build parameters和Dependencies）也适用于在队列中等待构建的情况。

该页面上可用的构建信息将在以下各节中进行描述。

根据在构建步骤中启用的构建运行程序，页面上的选项卡数和“ 概述”选项卡上的信息可能会有所不同。

构建概述

“ 概述”选项卡显示有关构建的常规信息，例如构建持续时间，使用的代理，触发器，依赖项等。如果将构建放入队列，则选项卡将显示构建在队列中的位置，构建应该开始的时间等。

如果构建正在运行，则选项卡将显示构建进度。您还可以使用“ 概述”选项卡上的相应链接或“ 操作”按钮下拉菜单中的相应选项来停止正在运行的构建。

如果同时运行同一构建配置的另一个构建，则“ 概述”选项卡上会出现一个小窗口，其中包含以下信息：

  • 该构建的构建结果链接到构建结果页面

  • 更改链接带有该版本中包含的更改的下拉列表

  • 内部版本号，开始时间以及指向其运行代理的链接。

如果某个构建可能正在挂起，则相应的通知将显示在“ 概述”选项卡的顶部。在这种情况下，TeamCity提供了一个链接，用于在单独的框架中查看正在运行的构建的进程树以及每个Java或.NET进程的线程转储。如果该进程不是Java或.NET，则将检索其命令行。以下平台支持查看线程转储的可能性：

  • Windows，带有JDK 1.3和更高版本

  • 使用jstack实用程序的Windows（带有JDK 1.6-1.8）

  • 非Windows，使用jstack实用程序的JDK 1.5-1.8

选项卡上的信息可能会有所不同，具体取决于启用的构建运行器。如果进行了配置，您将看到“ 代码覆盖率摘要”以及指向完整报告或/的链接，以及在代码中找到的重复项的数量，以及打开“ 重复项”选项卡的链接，依此类推。有关代码覆盖率和“ 重复项”选项卡的详细信息，请参阅以下各节。

如果构建中存在问题，则“ 概述”选项卡也会显示它们。

如果构建的测试失败，则可以在构建结果页面的“ 概述”选项卡上查看它们。

测验

此版本中成功或失败的测试（如果有）显示在“ 概述”选项卡上。

对于每个失败的测试，您可以查看其堆栈跟踪，预期值与实际值之间的差异，跳转到测试历史记录 ， 指派团队成员调查其失败 ， 在IDE中打开测试和/或立即开始修复它。

要查看与构建相关的所有测试，请使用专用的“ 测试”选项卡。了解更多 。

变化

在更改选项卡中，您可以：

  • 查看版本中包含的所有更改及其版本控制中的相应修订

    • 查看当前构建所依赖的构建中包含的更改：如果当前构建配置具有工件依赖关系，并且与当前配置的先前构建中下载的工件相比，当前构建中下载的工件已发生更改，则工件依赖关系changes节点出现，显示用于从中下载工件依赖项的版本以及该版本中包含的更改。

  • 标记构建源 （TeamCity 8.1之前）

  • 配置构建配置的VCS设置 （如果您具有足够的权限）。

对于此页面上的每次更改，您都可以：

  • 探索细节变化

  • 查看启用了“显示快照依赖项的更改”选项（ 自TeamCity 2017.1开始）后，查看更改来自哪个依赖项生成或使用快照依赖项进行生成。悬停在图标，显示从属版本号；单击该链接可打开从属版本的hanges选项卡。

  • 通过单击更改的文件链接导航到“ 更改详细信息”

  • 通过此更改触发自定义构建

  • 下载补丁

  • 将补丁下载到您的IDE

  • 如果由管理员配置，请在外部变更查看器中查看变更 。

更改选项卡为更改列表提供了高级筛选功能。启用“ 显示图形 ”会将更改显示为对与此构建相关的VCS根目录的提交的图形。

该图显示在列表的左侧，可让您以可变的详细程度获取更改视图。您可以：

  • 查看在此版本中已更改的VCS根目录，每个根目录均以条形表示。

  • 导航到图节点以显示VCS根修订版号。

  • 单击栏以选择单个VCS根目录。与该根目录无关的更改将显示为灰色。

  • 如果存储库的分支之间存在合并，则该图将显示它们。要折叠条，请导航至其较暗的区域，然后单击以隐藏合并历史记录。虚线表示该栏是可扩展的。

  • 如果您的VCS根目录具有子存储库（在更改列表中标记为S），请导航到父存储库中的节点，以查看此修订版本在父存储库中引用了子存储库中的哪些提交。

您可以通过选中显示文件框来选择查看修改后的文件。单击文件名将打开差异查看器。

测试持续时间图

测试持续时间图（请参见上面的屏幕快照）对于比较在此构建配置的构建上运行各个测试所花费的时间非常有用。

测试持续时间结果仅适用于当前在构建历史记录中的构建。清理完构建后 ，该数据将不再可用。

您可以在“测试持续时间”图上执行以下操作：

  • 通过清除“ 显示失败”选项，筛选出未通过测试的内部版本。

  • 通过选择“ 平均值”选项来计算每日平均值。

  • 单击图形上绘制的点，以跳至具有相应构建结果的页面。

  • 在图形上点的工具提示中查看构建摘要，然后导航到相应的“ 构建结果”页面。

  • 通过选择或清除特定代理的代理或单击“ 全部”或“ 无”链接以选择或清除所有代理来过滤信息。

构建日志

对于每个构建，您可以查看和下载其构建日志。有关TeamCity中构建日志的更多信息，请参见此处 。

参量

特定构建使用的所有系统属性和环境变量都在构建结果页面的“ 参数”选项卡上列出。了解有关构建参数的更多信息 。

“ 报告的统计值”页面显示为构建报告的统计值 ，并在单击“ 查看趋势”图标时显示每个值的统计图表。 。

依存关系

如果完成的构建具有工件和/或快照依赖关系，则“ 依赖关系”选项卡将显示在构建结果页面上。在这里，您可以探索其工件和/或源用于创建此构建的构建（下载的工件），以及使用当前构建的工件和/或源的构建（已交付工件）。此外，您可以查看构建的间接依赖关系。也就是说，例如，如果版本A依赖于版本B，而版本B依赖于版本C和D，则这些版本C和D是版本A的间接依赖项。

相关问题

如果您有与问题追踪集成配置，并且如果在所包含的变化或为构建自己的意见的评论中提到的至少一个问题，你会看到在问题涉及到当前的构建问题清单标签。

如果您需要查看与构建配置有关的所有问题，而不仅是与特定构建有关的问题 ，则可以导航到构建配置主页上的“ 问题日志”选项卡，您可以在其中查看映射到注释的所有问题或过滤列出特定的构建范围。

建立文物

如果构建产生了工件 ，则它们都将显示在专用的“ 工件”选项卡上。

代码覆盖率结果

如果在构建运行器中配置了代码覆盖率，则会在构建结果页面上显示一个带有完整HTML代码覆盖率报告的专用选项卡。

通过单击“ 覆盖范围细分”部分中的链接，您可以向下钻取以显示不同范围的统计信息，例如，命名空间，程序集，方法和源代码。

代码检查结果

如果已配置，则“ 代码检查”构建步骤的结果将显示在“ 代码检查”选项卡上。使用左窗格浏览检查结果；过滤后的检查显示在右窗格中。

  • 如果您对警告不感兴趣，请从“ 总计”更改为“ 错误”选项。

  • 使用范围过滤器将视图限制为特定目录。这使开发人员可以更轻松地管理感兴趣的特定代码。

  • 使用范围过滤器下的检查树视图以显示特定检查的结果。

  • 请注意，TeamCity显示包含该问题的源代码行号。单击它可以在您的IDE中跳转代码。

重复项

如果您的构建配置将Duplicates构建运行器作为构建步骤之一，则您将在构建结果中看到Duplicates选项卡。

该选项卡包括：

  • 找到重复项的列表。使用仅新选项，您可以仅显示最新版本中出现的重复项。

  • 包含这些重复项的文件列表。使用左右箭头按钮在选项卡下部的相应窗格中显示选定的重复项。

  • 两个窗格包含包含重复项的文件片段的源代码。

  • 左上角的作用域筛选器列出了包含重复项的特定目录。这种过滤使开发人员更容易管理感兴趣的代码。

Maven构建信息

对于每个Maven构建，TeamCity代理都会收集Maven特定的构建详细信息，构建完成后，这些详细信息将显示在构建结果的Maven Build Info选项卡上。
