Visual C构建问题

如果您在构建代理上运行Visual C ++构建时遇到任何问题，可以尝试按以下步骤依次解决这些问题：

这些步骤中的任何一个都可以解决您的问题。随时留下您的经验反馈。

  • 确保不使用映射的网络驱动器。

  • 确保构建用户具有访问必要网络路径的足够权限

  • 以与build相同的用户身份登录到build agent计算机，然后尝试运行以下命令：

    msbuild.exe / p：配置：发布/ t：重建

  • Build Agent服务在具有本地管理特权的用户下运行

  • 确保在生成代理上安装了Microsoft Visual Studio

  • 您必须一次在生成用户下启动Visual Studio 2005或Visual Studio 2008

  • 如果出现错误生成cmd.exe ，则应将以下几行准确地放入列表中Tools -> Options -> Projects and Solutions -> VC++ Directories ：

    -$（SystemRoot）\ System32-$（SystemRoot）-$（SystemRoot）\ System32 \ wbem

  • 您需要从添加所有环境变量...\Microsoft Visual Studio 9.0\VC\vcvarsall.bat环境或构建代理配置

  • 尝试通过命令行使用devenv.exe而不是Visual Studio（sln）构建运行器

  • 确保到源的所有路径都不包含空格

  • 组VCBuildUserEnvironment=true在跑步者属性中

  • 指定VCBuildAdditionalOptions有价值的财产/useenv在生成配置设置中指示msbuild添加/useenv生成的vcbuild进程的命令行参数。
