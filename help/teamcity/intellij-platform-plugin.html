IntelliJ平台插件

TeamCity插件为基于IntelliJ平台的IDE（包括JetBrains IntelliJ IDEA，RubyMine，PyCharm，PhpStorm / WebStorm，AppCode和Rider）提供TeamCity集成。只有JetBrains与IDE捆绑在一起的VCS集成才支持远程运行/预先测试的提交功能。请参阅单独的页面以获取受支持的版本列表。

相关博客文章中提供了一个用法示例。

特征

TeamCity集成提供以下功能：

  • 远程运行和预测试（延迟）提交

  • 自定义个人构建的参数

  • 远程调试

  • 审查代码重复的可能性

  • 分析远程代码检查的结果

  • 监视特定项目和构建配置的状态以及提交给项目代码库的更改的状态

  • 查看失败的测试并使用突出显示的堆栈跟踪和当前项目文件名构建日志

  • 开始调查失败的构建

  • 将构建配置问题或测试失败的调查表分配给其他团队成员

  • 查看应该检查的构建故障，并在问题解决后放弃调查

  • 将快速修复应用于远程代码分析的结果：有问题的代码可以在编辑器中突出显示，并且您可以在工具窗口中使用项目检查结果的完整报告

  • 下载并仅查看自上次构建以来出现的新检查结果

  • 在专用工具窗口中处理服务器端代码重复搜索的结果

  • 访问服务器端代码覆盖率信息并可视化单元测试覆盖的代码部分

  • 在构建结果窗格的单独选项卡中查看构建编译错误，并导航至源代码

  • 使用JUnit或TestNG从IntelliJ IDEA插件重新运行失败的测试

  • 从更改详细信息网页打开补丁（要使此功能正常工作，您需要安装IDEA X）

安装TeamCity插件

TeamCity IDE插件版本必须与其连接到的TeamCity服务器的版本相对应。通常不支持连接到具有不同版本的TeamCity服务器。

从插件存储库安装插件

插件存储库包含一个TeamCity插件（来自最近发布的版本之一）。您可以从存储库中安装插件（例如，从IntelliJ IDEA设置>插件），然后输入本地TeamCity服务器的地址，并让插件将自身更新为与服务器相对应的版本。

要为IntelliJ平台IDE安装TeamCity插件 ：

  1. 在IDE中，打开“ 设置”对话框。为此，请按Ctrl + Alt + S或选择“ 文件” > “设置...”。 （在MacOS上为Apple > 设置... ）。

  2. 打开插件部分。

  3. 在“ 插件”部分中，搜索“ TeamCity”或单击“ 安装JetBrains插件...”以查看可用插件的列表。

  4. 选择TeamCity集成，单击“ 安装”按钮。

  5. 重新启动IDE。

  6. 使用TeamCity菜单从插件登录到TeamCity服务器。

  7. 在TeamCity菜单中调用Update命令以安装与服务器版本匹配的插件版本，然后重新启动IDE。

手动安装插件

可以从TeamCity Web UI的“ 我的设置和工具”页面上的TeamCity工具区域下载IntelliJ平台的插件。

要安装TeamCity插件 ：

  1. 在TeamCity网络用户界面的右上角，点击用户名旁边的箭头，然后选择我的设置和工具 。

  2. 在右侧的TeamCity Tools区域中找到IntelliJ Platform Plugin部分，单击下载链接，然后保存存档。

  3. 在IDE中，打开“ 设置”对话框。为此，请按Ctrl + Alt + S或选择“ 文件” > “设置...”。 （在MacOS上为Apple > 设置... ）。

  4. 打开插件部分。

  5. 在“ 插件”部分中，单击“ 从磁盘安装插件...” 。

  6. 在“选择插件文件”对话框中，选择下载的存档，单击“ 确定”按钮。

  7. 确保“插件列表中的TeamCity集成”旁边的复选框已启用。

  8. 重新启动IDE 。有关如何使用TeamCity插件的所有其他信息，请参阅IDE帮助系统 。

配置基于IntelliJ IDEA平台的IDE以检查插件更新

  1. 在IntelliJ IDEA中，打开设置|更新 。

  2. 将http：// /update/idea-plugins.xml添加到列表中。

  3. 将“检查更新”设置为“每日”。

  4. 按“ 应用” ，然后按“ 立即检查” 。
