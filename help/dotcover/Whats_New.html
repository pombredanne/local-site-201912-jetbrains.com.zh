dotCover 2019.2帮助

dotCover的新功能

dotCover 2019.2

  • 在Windows，macOS和Linux上支持Mono。
    dotCover 2019.2在Windows，macOS和Linux上支持Mono 5.10及更高版本。要在这些操作系统上执行覆盖率分析，必须使用JetBrains Rider或dotCover控制台运行程序 。请注意，在macOS和Linux上，仅支持64位进程。

  • Linux和macOS上的控制台运行程序。
    现在，您不仅可以在Windows上，还可以在macOS和Linux上使用dotCover控制台运行程序。为了在新平台上更加方便，该工具接受Unix样式语法中的命令参数。例如，现在，两者/TargetExecutable=MyApp.exe和--targetexecutable:MyApp.exe是有效的。

  • 控制台运行程序改进。
    控制台运行程序获得了两个新命令，以简化对.NET Core和Mono应用程序以及单元测试的覆盖范围分析： cover-dotnet和cover-mono 。

  • 覆盖率为0％和.NET Core单元测试的问题已修复。不再需要dotnet dotCover测试变通办法，但如果发现这种运行覆盖率分析的方式更加方便，则仍可以使用它。

dotCover 2019.1

  • 在macOS和Linux上支持.NET Core单元测试。要在这些操作系统上执行代码覆盖率分析，必须使用JetBrains Rider 。

  • Visual Studio的远程覆盖不再可用。该功能已从dotCover中删除。

dotCover 2018.3

  • 改进的覆盖率过滤器 。现在，dotCover选项中有两类过滤器：运行时过滤器（在coverage会话期间应用的过滤器）和结果过滤器（应用于coverage结果的过滤器）。后者以文件名过滤器扩展。

  • 支持多个目标框架 。现在，如果一个项目针对多个框架，那么“ 覆盖树”将分别显示每个框架的结果（每个框架将在树中显示为单独的节点）。代码突出显示还取决于在编辑器中选择的上下文。

dotCover 2018.2

  • JetBrains骑士集成 。除了Visual Studio，现在dotCover还是JetBrains Rider IDE的一部分。在此版本中，支持两个主要功能：单元测试的代码覆盖率分析和连续测试。

  • 支持dotnet控制台工具 。现在，dotCover命令行运行程序可以用作.NET命令行工具（dotnet.exe）的扩展。例如，您可以通过运行以下命令来运行单元测试的覆盖率分析： dotnet dotCover test

  • 现在，在使用dotCover.exe控制台运行程序时，您应该始终喜欢cover指挥analyze命令。覆盖率结果仅取决于--ReportType参数：如果指定，您将获得某种类型的报告；否则，将保存常规的覆盖率快照。

dotCover 2018.1

  • 不再需要单独的连续测试会话-可以为任何单元测试会话打开连续测试模式。

  • 新的连续测试自动启动模式 。

  • 现在，覆盖率信息显示在单独的“ 单元测试覆盖率”窗口中，而不是每个会话的“ 覆盖率”选项卡。在一个地方查看所有单元测试会话的汇总覆盖率数据。

  • dotCover控制台运行程序能够覆盖任意托管流程和服务analyze-everything ， analyze-service并将get coverage snapshot命令发送到控制台工具的运行实例。这极大地简化了在IIS / IIS Express上运行的Web应用程序的覆盖范围 。

dotCover 2017.3

  • .NET Core在NUnit和xUnit的连续测试模式下测试覆盖范围支持。

  • 快速评估覆盖率结果 。Visual Studio编辑器右装订线中的“屏蔽”图标使您可以快速查看总覆盖率和失败测试的数量。单击该图标可在未显示的语句中导航。

  • 改进了“显示覆盖声明” 。现在，在编辑器中使用“ 显示覆盖测试 ”后，弹出窗口将告诉您这些测试覆盖了代码的确切部分：类，方法或特定语句。

dotCover 2017.2

  • .NET Core测试了对MSTest的覆盖支持。

  • 性能提高 。在2017.2之前，对大型解决方案进行连续测试确实令人头疼 ：尽管测试执行得非常快，但可能要花几分钟才能完成后续的覆盖会话。问题在dotCover 2017.2中不再存在：由于采用了新的快照格式，dotCover可以更快地处理coverage结果。

  • 现在不仅显示特定测试的覆盖率结果，而且还显示测试组，项目，类别等的覆盖率结果。

  • 色盲标记 。现在，标记器不仅可以用颜色而且可以用形状指示覆盖率和测试结果。

  • 从快照添加覆盖率数据 。现在，当查看特定单元测试会话的 coverage结果时，您可以将它们与快照中的coverage数据合并。

dotCover 2017.1

  • 使用彩色背景突出显示 。由于新的突出显示样式（使用标记）可能并不适合所有用例，因此我们添加了一个选项，可以在标记和彩色背景之间切换或同时显示两者。旧的突出显示样式还支持更新的逻辑，并显示测试覆盖率和测试结果。

dotCover 2016.3

  • 新代码覆盖范围突出显示 。您可以直接在代码编辑器中查看测试结果。

  • 更灵活的覆盖率过滤器 。

  • 通过排除与覆盖率分析无关的子进程，可以从控制台运行程序中筛选进程以减少不必要的开销。

  • 更方便地调试失败的测试 。您可以设置一个断点并运行调试。

dotCover 2016.2

  • “ 单元测试会话”窗口和“连续测试会话”窗口中的执行日志。

  • 可以使用“ 隐藏自动属性”选项从覆盖范围分析中排除自动属性 。

  • 通过排除与覆盖率分析无关的子进程，可以从控制台运行程序中筛选进程以减少不必要的开销。

dotCover 2016.1

  • 连续测试中支持覆盖过滤器 。
