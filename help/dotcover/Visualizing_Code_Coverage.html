可视化代码覆盖率

ReSharper_ToggleHighlighting

为了可视化覆盖率和测试结果数据，dotCover可以在Visual Studio编辑器中突出显示当前快照中的语句。

在编辑器中可视化代码覆盖率

  1. 在窗口中打开要分析的快照。 覆盖率结果浏览器窗口或“ 单元测试覆盖率”窗口中。

  2. 点击突出显示代码 在打开当前快照的窗口的工具栏上。

  3. dotCover将突出显示当前coverage快照中包含的所有程序集中的代码（请参见左装订线）：

  4. 要禁用突出显示当前快照， 按Ctrl+Alt+K H或选择 ReSharper |封面从主菜单切换代码突出显示

如果您在 “覆盖率结果浏览器”窗口或“ 单元测试覆盖率”窗口中的突出显示不会自动切换。要切换特定快照的突出显示，请单击突出显示代码 在打开当前快照的窗口的工具栏上。

突出显示如何工作

为了指示代码语句的状态，dotCover在Visual Studio的左装订线中使用彩色标记：

要配置dotCover用于突出显示标记的颜色，请打开“ 工具” |“工具”。选项...|环境|字体和颜色，并找到以“ dotCover”开头的条目。

如果您希望突出显示仅显示代码覆盖率（没有在dotCover 2016.2和更早版本中的测试结果），请清除ReSharper | |中的 “ 使用突出显示以显示单元测试结果”选项。 选项|封面编辑 。

记号笔

描述

|

一行测试中至少有一条语句未包含在单元测试中。

一行中的所有语句都包含在单元测试中。所有测试均通过。

|

一行中的所有语句都包含在单元测试中。至少一项测试失败。

请注意，如果在dotCover的突出显示选项中启用了使用形状标记（色盲模式）设置，标记不仅会用颜色而且还会用形状指示覆盖率和测试结果。

更改代码后，测试结果立即过时。在这种情况下，标记保持其颜色，但颜色变浅。标记的工具提示还显示有关先前状态的信息：

如果一行代码由具有不同覆盖率结果的多个语句组成，则dotCover将仅考虑“最差”结果来突出显示该行。例如，如果对第一个语句的测试通过了，但是对第二个语句的测试通过了，则dotCover将用红色突出显示该行。或者，如果一个语句被某些失败的测试所覆盖，而另一条语句根本未被覆盖，则标记将为灰色：

如果您希望在整行代码都突出显示时使用“旧式”突出显示（如dotCover 2016.2及更早版本），则可以通过ReSharper |选项| dotCover |编辑器使用|突出显示代码覆盖率线背景 。

导航到涵盖测试

要继续探索代码覆盖率，您可以导航到覆盖率测试和/或在覆盖率树中找到代码项 。

可视化针对多个框架的项目中的代码覆盖率

如果项目针对多个框架，则dotCover会独立显示每个框架的覆盖范围结果。因此， 覆盖树将每个框架显示为一个单独的节点。代码突出显示还取决于您在编辑器中选择的上下文。
