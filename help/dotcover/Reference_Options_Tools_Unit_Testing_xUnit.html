dotCover 2019.2 Help

xUnit.net

ReSharper | Options | Tools | Unit Testing | Test Frameworks | xUnit.net

In this page of dotCover options, you can adjust unit testing settings related to xUnit.net tests.

Item

Description

Enable xUnit.net support

By default, dotCover adds action indicators allowing you to execute xUnit.net tests right from the editor and automatically discovers xUnit.net tests in your solution. Clear this check box to disable this behavior.

xUnit.net config file

With the Auto-detect option will find your xUnit.net configuration file in any of standard locations. If your configuration file is in a custom location, select Specific and specify the path to the configuration file. To learn about the format of the configuration files, see configuring xUnit.net with JSON and configuring xUnit.net with XML

dotCover's settings are saved in settings layers, which enable you to have different settings for different solutions and to share your settings with your team. To learn more, see Managing and Sharing dotCover Settings.
