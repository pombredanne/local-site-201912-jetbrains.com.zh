导航至覆盖测试

ReSharper |封面显示覆盖测试
Ctrl+Alt+K T
ReSharper_ShowTestsCoveringEditorContext

dotCover提供了一种快捷方式来检测哪些单元测试覆盖了代码中的特定语句。在Visual Studio编辑器中，您可以调用一个弹出列表，其中列出了涵盖的测试，并直接导航至这些测试中的任何一个。此外，从此弹出窗口中，您可以立即运行它们或将其添加到现有的单元测试会话中。

dotCover从当前coverage快照中获取关于Coverage单元测试的信息。因此，在某些情况下，此信息可能不可用，并且dotCover无法执行以下导航：

  • 没有打开任何coverage快照 。在这种情况下，主菜单和上下文菜单中的相应命令将显示为灰色。

  • 当前的陈述中没有任何单元测试 。在这种情况下，命令处于活动状态，但弹出窗口将显示以下消息： 没有测试覆盖此语句。

  • 当前快照没有每项测试数据 。在这种情况下，命令处于活动状态，但是弹出窗口将显示以下消息： 当前快照不包含每次测试的信息。

显然，对于未包含在单元测试中的代码，这种导航是不可能的。因此，您可以先突出显示覆盖的代码 ，然后从突出显示为覆盖的代码中导航。

导航至覆盖测试

  1. 在编辑器中，将插入符号放在代码语句处。

  2. 请执行以下任一操作：

    • 按Ctrl+Alt+K T或选择 ReSharper |封面从主菜单显示覆盖测试 。

    • 如果启用了覆盖突出显示 ，请将光标停留在特定的突出显示标记上，直到出现工具提示，然后单击工具提示。

  3. 在出现的弹出窗口中，您将看到覆盖该语句的单元测试列表。

  4. 双击所需的单元测试以在编辑器中将其打开。

您可以用来浏览代码覆盖率的另一个导航目标是在覆盖树中定位代码项 。
