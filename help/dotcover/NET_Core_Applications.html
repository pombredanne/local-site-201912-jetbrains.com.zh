借助dotCover，您可以运行.NET Core应用程序的覆盖率分析：具有.exe可执行文件的独立应用程序部署或仅具有.dll文件的依赖于框架的部署。

您选择的应用程序或过程必须具有PDB信息。否则，dotCover将无法记录承保范围信息。有关更多信息，请参见在何处搜索PDB信息。

为独立应用程序启动覆盖运行

  1. 请执行以下任一操作：

    • 在Visual Studio中， 按Ctrl+Alt+K K或选择 ReSharper |封面从主菜单覆盖应用程序...

    • 在dotCover独立应用程序中，按Ctrl+Alt+K K或选择文件|菜单中的Cover Application...。

  2. 在出现的“ 覆盖配置”对话框中，选择“ .NET Core应用程序” 。

  3. 在“ 应用程序”字段中，指定应用程序可执行文件或库文件的路径。

  4. （可选）选中“ 高级”复选框以配置其他设置：

    • 如果您需要传递任何命令行参数，请在“ 参数”字段中输入它们，或单击省略号按钮，然后选择一个文件附加到参数字符串中。

    • 如果要为应用程序使用其他工作目录，则可以在“ 工作目录”字段中指定它。

    • 如果要dotCover分析子进程的覆盖范围（如果有），请选中“ 覆盖子进程”复选框。在“ 排除”中 ，指定一个不能用分号分隔的进程名称列表。请注意*支持通配符。例如，如果您添加service*掩码，所有以该名称开头的进程service探查器将忽略它。

    • 单击“ 编辑符号搜索策略”以指定在何处搜索PDB信息 。

    • 单击编辑过滤器 查看现有并添加新的覆盖率过滤器。请注意，使用此按钮添加的过滤器仅应用于当前会话。

  5. 点击运行 。 dotCover将打开Coverage Controller对话框 ，然后运行所选的应用程序。

  6. 如果选中了“ 从开始收集性能分析数据”复选框，则覆盖率运行将立即开始。否则，dotCover会一直等到您在Coverage Controller对话框中单击“ 开始 ”。开始coverage运行时，您可以记录代码coverage 。
