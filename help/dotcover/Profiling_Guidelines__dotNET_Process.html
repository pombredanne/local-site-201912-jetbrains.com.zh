如果需要分析以自定义方式启动的.NET进程的覆盖范围 ，则可以先启动Coverage Controller ，然后再启动.NET进程。然后dotCover将自动附加到流程中，并在流程执行时记录覆盖率分析。

您选择的应用程序或过程必须具有PDB信息。否则，dotCover将无法记录承保范围信息。有关更多信息，请参见在何处搜索PDB信息。

分析任意.NET进程的覆盖范围

  1. 请执行以下任一操作：

    • 在Visual Studio中， 按Ctrl+Alt+K K或选择 ReSharper |封面从主菜单覆盖应用程序...

    • 在dotCover独立应用程序中，按Ctrl+Alt+K K或选择文件|菜单中的Cover Application...。

  2. 在出现的“ 覆盖配置”对话框中，选择“ .NET Process” 。

  3. （可选）选中“ 高级”复选框以配置其他设置：

    • 如果不能保证在单击“运行”后立即启动涵盖的进程，请在“ 仅涵盖指定的.NET进程 ”中设置进程筛选器：

      • 只能有“ include”掩码。因此，添加的任何筛选器掩码都意味着“配置与该掩码匹配的所有内容”。

      • 您必须指定进程名称，包括.exe延期。

      • 星号*支持通配符。

      例如，对所有以下列结尾的过程进行覆盖率分析service ，添加*service.exe过滤。

    • 单击“ 编辑符号搜索策略”以指定在何处搜索PDB信息 。

    • 单击编辑过滤器 查看现有并添加新的覆盖率过滤器。请注意，使用此按钮添加的过滤器仅应用于当前会话。

  4. 点击运行 。 dotCover将打开Coverage Controller对话框 ，然后

    等待第一个.NET进程启动。

  5. 启动您的.NET进程。
    一旦dotCover检测到与指定过滤器匹配的过程， Coverage Controller对话框中的控件就会启用，您可以开始coverage运行。

  6. 如果选中了“ 从开始收集性能分析数据”复选框，则覆盖率运行将立即开始。否则，dotCover会一直等到您在Coverage Controller对话框中单击“ 开始 ”。开始coverage运行时，您可以记录代码coverage 。
