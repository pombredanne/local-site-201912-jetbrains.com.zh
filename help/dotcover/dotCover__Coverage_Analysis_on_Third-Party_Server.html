dotCover 2019.2帮助

在第三方CI服务器上设置覆盖范围

您可以将dotCover与任何持续集成（CI）服务器一起使用，以将测试覆盖率报告添加到.NET应用程序构建的工件中。 dotCover支持多种输出格式。例如，您可以设置构建脚本以获取HTML输出并发布它，或者获取XML输出并针对您的特定需求应用进一步的XSLT转换。

dotCover Console Runner可以在没有Visual Studio的情况下工作。

在CI服务器上设置覆盖率分析

  1. 从JetBrains网站下载dotCover命令行工具包 。

  2. 将存档复制到CI服务器。

  3. 配置您的构建脚本，以在生成所有程序集后运行控制台运行程序可执行文件（ dotCover.exe） 。例如，您可以运行cover命令： dotCover cover config.xml

  4. 编辑配置文件（例如，在上面的示例中： config.xml ），以便控制台运行程序可以启动所需的文件使用必要的参数测试运行程序（MSTest，NUnit，xUnit，MSpec等），并创建所需的输出。有关更多信息，请从命令行查看控制台运行器配置和覆盖率分析
