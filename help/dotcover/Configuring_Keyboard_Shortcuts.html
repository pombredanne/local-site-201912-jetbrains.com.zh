配置键盘快捷键

对于Visual Studio集成，dotCover提供了两个默认的键盘快捷方式方案 。

  • Visual Studio此方案旨在最大程度地减少与Visual Studio自己的键盘快捷键的冲突。

  • ReSharper 2.x / IntelliJ IDEA此方案与IntelliJ IDEA及其派生的Web开发IDE中使用的快捷方式共享大多数。它在所有JetBrains开发环境中提供了以键盘为中心的通用体验。

无论选择哪种方案，都可以在以后随时更改单个键盘快捷键。

更改快捷方式和快捷方式

在安装dotCover之后首次启动Visual Studio时，将打开“ 键盘方案”对话框，您可以在其中选择默认的快捷方式方案。

您可以稍后使用“ 环境” | “ 环境”中的相应选择器来更改选定的方案。 dotCover选项的“ 键盘和菜单”页面。

当您更改先前选择的方案时，由先前方案设置的键绑定不会被取消。要删除旧的键绑定和/或还原被IntelliJ IDEA方案覆盖的Visual Studio键绑定，请打开Visual Studio选项（“ 工具” |“选项” ），转到“ 环境” |“ 选项” 。键盘 ，然后单击重置 。

无论当前选择哪种方案，您都可以始终在Visual Studio选项中配置单个键盘快捷键。

配置单个键盘快捷键

  1. 在主菜单中，选择“ 工具” |“工具”。选项 。

  2. 展开环境节点，然后单击键盘 。

  3. 使用快捷方式表查找命令的别名，您将为其分配新的快捷方式。

  4. 要找到所需的命令，您还可以执行以下操作之一：

    • 在“ 显示包含命令”字段中键入要查找的命令的名称或名称的一部分。可用命令的列表会缩小，您可以轻松找到所需的命令。

    • 浏览可用命令列表，向下滚动到以以下名称开头的命令名称ReSharper 。
      键盘选项卡中的命令列表获得焦点后，请按Ctrl+r跳到初始的项目r 。

  5. 如果存在任何快捷方式，则所选命令的快捷键组合将显示在“ 所选命令的快捷方式”下拉列表中。

  6. 您可以删除现有的组合键和/或添加新的组合键。

    • 在“ 所选命令的快捷方式”下拉列表中选择组合，然后单击“ 删除” 。

    • 可以在不同区域中使用相同的快捷键组合，因此您应该指定区域。使用“ 使用新的快捷方式”下拉列表指定区域。然后将插入符号放置在“ 按快捷键”文本框中，然后按所需的组合键。点击分配 。

  7. 单击确定 。

解决键盘快捷键冲突

默认快捷方式 （特别是ReSharper 2.x / IntelliJ IDEA方式）中使用了某些键盘快捷方式，它们可能已经映射到其他Visual Studio命令。在这种情况下，dotCover不会静默替换现有的Visual Studio键盘快捷键。一旦使用冲突的快捷方式（即Visual Studio和dotCover都使用的快捷方式） ，就会提示您使用dotCover快捷方式冲突对话框解决此冲突：

选择三个可用选项之一：

  • 使用dotCover命令 ：将冲突的快捷方式绑定到dotCover命令。

  • 使用Visual Studio命令 ：解决冲突以支持Visual Studio命令。

  • 配置dotCover命令 ：在Visual Studio“ 选项”对话框中打开“ 键盘”选项卡，以帮助您自定义当前的键盘方案。

无论选择哪种选项，都可以通过选择应用到所有dotCover快捷方式来将选择应用于所有其他冲突的dotCover快捷方式 。
