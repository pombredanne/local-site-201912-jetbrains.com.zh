dotCover允许您对具有PDB信息的.NET应用程序和进程进行覆盖率分析。您可以分析以下类型的应用程序和过程的覆盖范围：

  • Windows服务
  • WCF服务
  • WinRT应用程序
  • .NET进程

查找符号文件（PDB）

查找目标二进制文件的符号文件（PDB）对于计算覆盖率至关重要。如果您涵盖单元测试或涵盖启动项目 ，则dotCover可以使用当前解决方案的结构轻松找到符号文件。

默认情况下，dotCover在以下位置搜索符号文件：

  • 在二进制文件所在的同一目录中；

  • 在二进制文件中指定的调试目录中；

  • 在指定的所有目录中_NT_SYMBOL_PATH环境变量和注册表中；

如有必要，您可以指定其他位置来查找符号文件。为此，在Coverage Configuration对话框中选择Advanced ，然后单击Edit Symbol Search Policy 。

在打开的“ 符号搜索策略”对话框中，可以添加以下方式来搜索符号文件：

  • 在搜索路径字段中， 提供用分号分隔的路径列表以搜索符号文件。每个路径可以是目录路径，也可以是符号服务器路径（例如srv * C：\ LocalSymbols * http：// symbolserver：33417 / ）。

  • 选择访问符号服务器以允许在“ 搜索路径”字段或“ 搜索路径”字段中指定的dotCover访问符号服务器。 _NT_SYMBOL_PATH环境变量。

您也可以禁用某些默认的搜索符号文件的方式。
