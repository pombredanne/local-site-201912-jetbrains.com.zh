使用dotCover，您可以在IIS Express服务器上对Web应用程序运行覆盖率分析。 dotCover将为您启动指定的服务器，并选择启动带有目标应用程序页面的Web浏览器。

您选择的应用程序或过程必须具有PDB信息。否则，dotCover将无法记录承保范围信息。有关更多信息，请参见在何处搜索PDB信息。

若要开始在IIS Express服务器上运行的coverage

  1. 请执行以下任一操作：

    • 在Visual Studio中， 按Ctrl+Alt+K K或选择 ReSharper |封面从主菜单覆盖应用程序...

    • 在dotCover独立应用程序中，按Ctrl+Alt+K K或选择文件|菜单中的Cover Application...。

  2. 在出现的“ 覆盖配置”对话框中，选择“ IIS Express” 。

  3. 要指定目标Web应用程序，请执行以下操作之一：

    • 如果您的Web应用程序已部署在IIS Express服务器上，请选择“ 配置文件” ，然后在“ 站点”列表中选择一个站点名称。

    • 如果您的Web应用程序未部署在IIS Express服务器上，请选择“ 应用程序路径” ，然后在“ 应用程序路径”字段中，指定Web应用程序的物理路径。 dotCover将在开始coverage运行之前部署您的应用程序。

  4. 如果要使用应用程序启动Web浏览器，请选中“ 打开URL”复选框，然后键入Web应用程序的URL。如有必要，请更改应使用的浏览器。

  5. 在右侧面板中的IIS Express应用程序中 ：

    • 如果要配置的Web应用在默认的IIS Express配置文件中进行了描述applicationhost.config ，选择“ 配置文件” ，然后在“ 站点”中选择要配置的网站。

      请注意，配置文件的位置取决于您使用的IDE：

      • JetBrains骑士： （solutionDir）\。idea \ config \ applicationhost.config

      • Microsoft Visual Studio 2015和更高版本： （solutionDir）\。vs \ config \ applicationhost.config

      • Microsoft Visual Studio 2013和更早版本： ％USERPROFILE％Documents \ IISExpress \ config \ applicationhost.config

      如果您不想使用配置文件，则可以选择“ 应用程序路径”，并指定包含配置文件应用程序的目录的完整路径。

    • 如果要自动打开应用程序URL，则在分析开始后，选择“ 打开URL”并指定应用程序URL。使用浏览器中的参数可以选择必须在其中打开URL的浏览器。

    • （可选）如果选择“ 高级” ，则可以指定自定义IIS Express和运行时设置：

      • 在“ 服务器”中 ，选择一个IIS Express实例，

      • 在运行时版本 ：.NET Framework版本中，

      • 在“ 端口”中 ：HTTP端口，或者，如果您希望dotCover自动选择端口，请选择“ 使用随机端口”。

    • 单击“ 编辑符号搜索策略”以指定在何处搜索PDB信息 。

    • 单击编辑过滤器 查看现有并添加新的覆盖率过滤器。请注意，使用此按钮添加的过滤器仅应用于当前会话。

  6. 点击运行 。 dotCover将打开Coverage Controller对话框 ，然后

    在IIS Express服务器上启动指定的Web应用程序。

    然后，您需要在浏览器中导航到您的应用程序网页或按Ctrl+F5刷新页面。
    如果选中了“运行浏览器”复选框，则dotCover将启动所选的Web浏览器或在指定的URL打开一个新的网页选项卡。

  7. 如果选中了“ 从开始收集性能分析数据”复选框，则覆盖率运行将立即开始。否则，dotCover会一直等到您在Coverage Controller对话框中单击“ 开始 ”。开始coverage运行时，您可以记录代码coverage 。
