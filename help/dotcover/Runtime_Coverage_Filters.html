在Visual Studio中：
ReSharper |封面编辑覆盖率筛选器...

在dotCover独立应用程序中：
档案|编辑覆盖率筛选器...

dotCover在覆盖会话期间应用运行时过滤器。使用这些过滤器可以从分析中排除不必要的代码，并减少覆盖会话时间。基于名称空间，项目，类型，类型成员或属性来设置运行时过滤器。

还有两种方法可以从覆盖率分析中排除不必要的代码：

  • 在运行外部应用程序的覆盖率分析时，请添加临时筛选器。

  • 按类别禁用单元测试。为此，请在ReSharper |上的“ 不要从类别运行测试”字段中指定要禁用的类别 。选项|工具| dotCover选项的“ 单元测试”页面。

由于在运行时中应用了过滤器，所以过滤后的项目不包括在coverage快照中 ，也没有显示在 覆盖率结果浏览器窗口或“ 单元测试覆盖率”窗口中。

查看和修改运行时过滤器

覆盖率筛选器有两个作用域：

  • 所有解决方案 -适用于计算机中所有解决方案的全局筛选器。

  • 当前解决方案 -适用于当前解决方案的每个解决方案过滤器。您可以与团队共享这些过滤器；有关详细信息，请参见共享覆盖率过滤器 。

您可以在“ 覆盖率过滤器”对话框中查看和编辑覆盖率过滤器 。要打开此对话框，请执行以下任一操作：

  • 在Visual Studio中， 按Ctrl+Alt+K F或选择 ReSharper |封面从主菜单编辑覆盖率过滤器...

  • 在dotCover独立应用程序中，按Ctrl+Alt+K F或选择文件|在菜单中编辑覆盖率过滤器...。

打开此对话框时，可以在两个范围中看到过滤器。如果未打开任何解决方案，则对话框仅显示全局过滤器。

默认情况下，dotCover提供两个全局过滤器，以排除代码项。 ExcludeFromCodeCoverageAttribute和GeneratedCodeAttribute属性。如有必要，您可以启用/禁用，修改或删除它。

查看和修改当前的运行时覆盖范围过滤器

  1. 打开“ 覆盖范围过滤器”对话框 。

  2. 使用分组依据选择器可以更改过滤器的分组-按作用域或种类。

  3. 要启用或禁用过滤器，请使用其旁边的复选框。

  4. 要删除过滤器，请在列表中选择它，然后单击“ 删除”。

  5. 要编辑过滤器，请在列表中选择它，然后单击“ 编辑” 。在出现的“ 编辑覆盖率过滤器”对话框中，按照下面添加新过滤器的说明进行必要的更改。

  6. 单击保存以保存并应用您的修改（如果有）。

在进行覆盖率分析时（例如，启动和未完成应用程序覆盖率运行时或在进行单元测试的覆盖率分析时），您无法编辑覆盖率过滤器。

添加新的运行时过滤器

添加新的运行时覆盖率过滤器

  1. 在“ 覆盖范围过滤器”对话框中 ，单击“ 添加过滤器” 。

  2. 在出现的“ 添加覆盖率过滤器”对话框中，选择过滤器规则的类型：

    • 分析代码中的 -将指定的代码项包括在代码覆盖率分析中，并排除所有其他代码项。

    • 不要在 -中分析代码，以从代码覆盖率分析中排除指定的代码项。

    • 不要分析标记有属性的代码 -从代码覆盖率分析中排除具有指定属性的代码项。

  3. 如果您选择了前两个选项之一，请指定覆盖率分析中要包括或排除的程序集，类和/或方法。代码项彼此独立过滤。例如，如果您指定一个方法并为程序集和类保留星号（*），以从所有程序集和所有类中排除具有指定名称的方法。

    在过滤器中指定代码项名称时，可以使用星号通配符：“ *”（星号）表示零个或多个字符。
    例如，您可以使用*.Tests通配符，将从代码分析中排除所有测试项目。

  4. 如果选择了最后一个选项，请指定将用于从覆盖率分析中排除符号的属性名称。（可选）指定声明此属性的程序集名称。

  5. 在“ 将过滤器应用于”部分中，指定过滤器的范围：“所有解决方案”或“当前解决方案”。

  6. 单击确定以完成添加过滤器。

  7. 在“ 覆盖率过滤器”对话框中，单击“ 保存”以保存并应用您的修改。

您还可以直接从Coverage树添加新的运行时筛选器：打开所需节点的上下文菜单，然后选择Exclude and Create Runtime Coverage Filter 。

共享运行时覆盖范围过滤器

解决方案范围的过滤器保存在解决方案团队共享的设置层中，因此可以在从事此解决方案的其他开发人员之间共享。

实际上，共享解决方案范围的过滤器所需要做的就是放置[SolutionName] .sln。版本控制系统（VCS）下解决方案文件夹中的DotSettings文件。一旦其他团队成员从VCS获得此文件，过滤器也将应用到他们的计算机上。

请注意，集成到Visual Studio中的其他JetBrains产品可能会将其解决方案范围的团队共享设置存储在同一文件中。因此，如果将此文件放在VCS下，请记住，其他产品的设置也可能会共享。有关更多信息，请参阅管理和共享dotCover设置 。

从应用程序覆盖范围分析中排除系统和非解决方案程序集

对应用程序执行覆盖率分析时 ，“ 过滤”选项中会为非解决方案和系统程序集提供其他过滤器：

  • （推荐）如果要仅分析解决方案中程序集的覆盖范围，请选择排除系统程序集和排除非解决方案程序集 。这将使覆盖率分析更快。

  • 如果要一次分析多个解决方案的装配的覆盖范围，请清除“ 排除非解决方案装配” 。

  • 如果出于某种原因要分析系统组合件中的覆盖率，请清除“ 排除系统组合件” 。
