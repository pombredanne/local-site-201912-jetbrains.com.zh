dotCover 2019.2帮助

NUnit

ReSharper |选项|工具|单元测试|测试框架NUnit

在dotCover选项的此页面中，您可以调整与NUnit测试相关的单元测试设置。

项目

描述

启用NUnit支持

默认情况下，dotCover添加操作指示器，使您可以直接从编辑器执行NUnit测试，并自动在解决方案中发现NUnit测试 。清除对应于特定NUnit版本的复选框以禁用此行为。
根据这些设置，dotCover将使用捆绑的NUnit测试运行程序：v。2.6.4和/或v。3.6.1。

自定义NUnit安装
使用指定的自定义NUnit

如果要使用另一个NUnit版本，请选中此复选框，然后在相应的字段中输入本地NUnit安装的路径。

笔记：

  • dotCover 2019.2支持NUnit 3.10.1及更高版本。

  • 如果此自定义安装中有任何NUnit插件 ，它们也将被加载。

NUnit 2
加载NUnit插件

如果上面选择了对NUnit 2x的支持，并且您需要将此捆绑版本使用NUnit 插件 ，请将插件放入dotCover安装目录的NUnit2 \ addins子文件夹（为方便起见显示），然后选择如何加载它们。

NUnit 3
运行测试时，设置环境。当前目录到：

从NUnit 3开始，NUnit框架未设置当前目录（请参阅nunit tracker中的此答案 ）。 dotCover默认将当前目录设置为temp文件夹，但是您可以更改它。

dotCover的设置保存在设置层中，使您可以针对不同的解决方案进行不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享dotCover设置 。
