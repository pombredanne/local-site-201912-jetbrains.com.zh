当您启动外部应用程序的coverage运行时，此对话框允许选择应用程序类型并配置相关的首选项。根据所选的应用程序类型，对话框中的控件允许配置不同的选项。

例如，当我们选择“ 使用IIS的Web应用程序”时 ，对话框如下所示：

无论选择哪种应用程序类型，“ 覆盖率选项”部分都包含以下控件：

项目

描述

立即开始

选择此复选框以在dotCover控制下启动目标应用程序后立即开始记录覆盖率

高级选项

编辑过滤器

点击这个按钮查看现有并添加新的覆盖率过滤器。请注意，使用此按钮添加的过滤器仅应用于当前会话。

编辑符号搜索策略

单击此按钮以打开“ 编辑符号搜索策略”对话框，并指定在何处搜索PDB信息 。

应用程序类型： 独立应用程序

项目

描述

应用

指定目标应用程序的可执行文件的路径。

高级选项

争论

可选的。如果需要，请指定命令行参数。

工作目录

可选的。指定工作目录。如果未指定工作目录，它将自动设置为包含应用程序可执行文件的路径。

涵盖子流程

可选的。如果要分析子进程的覆盖范围，请选中此复选框。

应用程序类型： .NET Core应用程序

项目

描述

应用

指定已分析应用程序的可执行文件（.exe）或库文件（.dll）的路径。

高级选项

争论

可选的。如果需要，请指定命令行参数。

工作目录

可选的。指定工作目录。如果未指定工作目录，它将自动设置为包含应用程序可执行文件的路径。

涵盖子流程

可选的。如果要分析子进程的覆盖范围，请选中此复选框。

应用程序类型： WinRT应用程序

项目

描述

配套

选择目标应用程序。

应用程序类型： WCF服务

项目

描述

库路径

指定应部署到WCF服务主机的WCF服务库的路径。

配置路径

指定WCF服务库的配置文件的路径。如果库配置文件位于同一目录中，则dotCover会自动检测到它。

高级选项

WCF主机

选择可用的WCF服务主机之一。
WCF服务主机允许您托管和测试WCF服务库。有关更多信息，请参见WCF服务主机 。

客户路径

可选的。指定应与WCF服务主机一起启动的客户端应用程序的路径。

客户论点

可选的。为客户端应用程序指定命令行参数。

应用程序类型： IIS服务器 （IIS） 上的Web应用程序

项目

描述

开启网址

可选的。如果要显式指定在IIS Server启动后应打开的URL和Web浏览器，请选中此复选框。

应用程序类型： IIS Express Server （IIS Express） 上的Web应用程序

项目

描述

配置文件

可选的。使用此字段可以显式指定IIS Express配置文件。一旦指定了有效的配置文件，可用网站的列表就会相应更新。

现场

如果您的Web应用程序已经部署，请选择相应的网站。

申请路径

如果未部署Web应用程序，请指定应用程序文件夹的物理路径。

开启网址

可选的。如果要显式指定在IIS Server启动后应打开的URL和Web浏览器，请选中此复选框。

高级选项

服务器

选择用于部署Web应用程序的IIS Express服务器。请注意，应该安装IIS Express。

运行版本

可选的。使用此字段选择部署应用程序时将使用的.NET运行时版本。

港口

可选的。使用此字段可以指定在部署应用程序时将使用的Web服务器端口。

应用程序类型： Windows服务

项目

描述

服务

从已安装的服务列表中选择一个目标.NET服务。

高级选项

争论

可选的。如有必要，请指定命令行参数。

启动依赖服务

可选的。如果要启动依赖于目标服务的服务，请选中此复选框。

应用程序类型： .NET进程

项目

描述

仅涵盖指定的.NET进程

可选的。指定要由dotCover分析的以分号分隔的进程列表。
