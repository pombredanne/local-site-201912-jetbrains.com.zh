典型的分析工作流程

您在dotMemory中的工作流程可能会有所不同，具体取决于您要分析的问题。但是，通常，它将始终包括以下步骤：

  1. 运行dotMemory性能分析会话（dotMemory测量应用程序不同方面的时间段）。这也将运行您配置的应用程序。

  2. 像平常一样使用您的应用程序一段时间。如果您发现某个内存问题，请在应用程序中重现该问题。

  3. 获取内存快照（托管堆的即时映像）。

  4. 有关内存使用情况的其他信息，您可能还会获得更多快照（例如，在重现问题之前和之后）。 dotMemory将使您能够比较这些快照并了解您的应用程序如何随着时间使用内存。

    在适当的时间拍摄快照对于成功分析某些应用程序可能至关重要。手动执行并不能保证结果。要在准确的时间拍摄快照，可以使用dotMemory API 。

  5. 分析快照以确定问题的原因。

例

在现实生活中会是什么样子？例如，您的应用存在内存泄漏。您的任务是确定导致泄漏的对象以及这些对象的来源（精确调用）。只需遵循工作流程：

  1. 运行分析会话。

  2. 在使用该应用程序时，请进行内存快照。

  3. 与该应用程序一起工作一段时间，以便泄漏可以更明显地显示出来。

  4. 再次拍摄内存快照。

  5. 使用许多dotMemory视图，分析快照并确定：

    • 哪种类型的对象会随着时间增长？

    • 是什么阻止了这些对象的收集？

    • 什么方法导致泄漏？

如果您曾经与其他内存分析器一起工作过，您会发现dotMemory不会脱颖而出，其工作流程与其他工作流程非常相似。真正使dotMemory与众不同的是您分析内存快照的方式。有关详细信息，请参见下一部分。
