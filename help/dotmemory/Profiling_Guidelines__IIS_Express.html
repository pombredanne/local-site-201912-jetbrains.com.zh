使用IIS Express的Web应用程序

要分析在IIS Express上运行的Web应用程序，应指定以下选项：

  • 配置文件
    使用默认配置文件运行IIS Express。默认文件位置取决于您使用的IDE：

    • JetBrains骑士： (solutionDir)\.idea\config\applicationhost.config

    • Microsoft Visual Studio 2015及更高版本： (solutionDir)\.vs\config\applicationhost.config

    • Microsoft Visual Studio 2013及更早版本： \Documents\IISExpress\config\applicationhost.config

    如果选择此选项，还必须指定要运行的Web 站点的配置文件。

  • 申请路径
    不要使用配置文件，而是运行另一个应用程序。在此处指定应用程序的完整路径。

  • 开启网址
    调用Web应用程序的URL。

  • 在浏览器中
    应该运行该应用程序的浏览器。默认选择系统浏览器。

  • 使用环境变量
    使您可以使用特定的环境变量运行配置文件的应用程序。您可以使用选项下方的列表指定任意数量的变量。请注意，必须在新行上指定每个变量。

如果选择“ 高级” ，则可以额外指定以下参数：

  • 服务器
    应当运行应用程序的IIS Express版本。根据您的操作系统和已安装的.NET Framework版本，此列表将产生不同的选项。

  • 运行版本
    .NET Framework版本，必须运行配置文件的应用程序。

  • 港口
    服务器使用的HTTP端口。

  • 使用随机端口
    如果打开，dotMemory将尝试自动检测HTTP端口。
