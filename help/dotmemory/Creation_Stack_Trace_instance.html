创建堆栈跟踪

创建堆栈跟踪视图显示了负责创建所选实例的堆栈跟踪。

使用此视图可以回答问题“哪个函数创建了对象？“该视图类似于用于分析对象集的Back Traces视图。

显示的调用从直接创建实例的调用开始，一直到堆栈中的第一个调用。

呼叫列表包括以下几列：

名称

描述

功能

函数名称。

命名空间

调用所属的名称空间。

选择对象进行进一步分析

在“ 创建堆栈跟踪”视图中，可以选择以下主题以进行进一步分析：

  • 由整个对象集专有保留（控制）的对象 。

选择分析对象集专有保留的对象

  • 点击 打开此设置按钮保留的对象 。

    此后，“ 专有保留的对象”主题将添加到“ 分析路径”，并且所需对象的列表将显示在“ 按类型分组”视图中。
