Starting Profiling Session

By default, your starting point in dotMemory is the Home page. Here you can start new profiling sessions or open the recent ones.

To start profiling of an application

  1. Choose the New Session | Local tab.

  2. In the central panel in Profile Application, select the type of application you are going to profile.

  3. In the right panel, configure profiling settings as described in Configuring Profiling Process.

  4. Click Run.

To start profiling of an already running application

  1. Choose the New Session | Local tab.

  2. Do one of the following:

    • In the central panel in the list of processes, select the .NET process you are going to profile.
      If a process you are looking for is missing, click Refresh to update the list of processes and/or click Show All Processes to grant dotMemory administrative permissions (this will show more processes running in the system).

    • Drag the Attach icon onto the application you want to attach to.

    1. Only processes running on .NET Framework 4.0 (or later) or .NET Core 3.0 Preview 7 (or later) are displayed and available to attach.
    2. dotMemory is unable to collect creation stack traces and traffic information when profiling attached processes.
    3. It is not possible to attach to Universal Windows Platform (UWP) applications.

  3. In the right panel, configure profiling settings as described in Configuring Profiling Process.

  4. Click Run.
