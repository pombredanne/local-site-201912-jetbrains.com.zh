Group by Creation Stack Trace

The Group by Creation Stack Trace views show the stack trace responsible for creating the selected object set. Use these views to answer the question "What function creates the object?"
These views could also be extremely helpful in determining functions that perform excessive allocations.

Same function can be called from different parts of an application; therefore, the function can appear multiple times in the list of calls.

The stack trace data can be grouped in three ways:

  • Call Tree: a list of calls shown starting from the first call in the stack.

  • Call Tree (Icicle Chart): a graphical representation of the call tree.

  • Back Traces: a list of calls shown starting from the last call in the stack.

To apply a grouping, click a particular button in the list of views.
