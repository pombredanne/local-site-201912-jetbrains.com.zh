Analyzing Profiling Results

After you collect enough profiling data, you can proceed to determining the cause of memory issues (the analysis step). As mentioned in the First Look at the User Interface section, memory analysis assumes continuous choosing of subjects for the analysis (certain object sets or instances) and examining these subjects from different perspectives - views. This chapter contains detailed descriptions of dotMemory views.
