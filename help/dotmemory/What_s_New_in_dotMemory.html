dotMemory 2019.2帮助

什么是新的

dotMemory 2019.2

  • dotMemory控制台分析器改进。
    dotMemory 2019.2停止支持远程分析。为了给您提供足够的替代产品，我们显着改进了dotMemory控制台分析器：

    1. 控制台分析器几乎等于独立的dotMemory。您可以使用控制台工具获取以下内容的快照：

      • .NET Core应用程序，

      • IIS Express托管的应用程序，

      • Windows服务

      • UWP应用程序，

      • WCF服务库，

      • 或运行概要分析会话后启动的任何.NET应用程序。

    2. 现在，控制台分析器可以在可能后立即拍摄内存快照。如果您想拍摄基准快照并将其用于比较，此功能可能会很有用。

  • 按需快照处理。
    您可以开始使用快照，而无需等待数据完全处理完。

  • .NET Core 3.0完全支持。
    dotMemory现在可以附加到.NET Core 3.0 Preview 7进程。

  • 改进了对Visual Studio中ASP.NET Core应用程序的支持：

    • 正确支持不同的托管模式，

    • 启动概要分析会话时，会考虑到浏览器设置和“ 启用SSL” ，

    • 支持ASP.NET Core 3.0 Preview 7，

    • 和其他改进...

  • 分析API可以脱离概要分析过程。

  • 不再支持Silverlight和WebDev应用程序。

dotMemory 2019.1

  • .NET Core 3.0应用程序支持。

  • 新的分析API 。我们已经对概要分析API进行了重大更新：

    • 我们已经删除了状态机。现在，无论控制器状态如何，您都可以调用任何API方法。

    • 我们减少了可用方法的数量，从而简化了API的使用。

    • 现在，我们以NuGet包的形式分发API。

  • 改进了对类型声明的导航 。现在，“导航到类型声明”操作不仅可以导航到Visual Studio，还可以导航到JetBrains Rider。

dotMemory 2018.3

  • 改进了按条件获取快照的功能 。性能分析控制器获得了拍摄快照的新条件： 如果总内存使用量超过X MB，则获取快照 。

  • 改进的时间线图。 单击该图以获取特定时间点的准确内存数据。

  • 重做的IIS Express配置文件设置 。现在，配置IIS Express上托管的Web应用程序的默认方法是为dotMemory提供一个applicationhost.config文件。

dotMemory 2018.2

  • 运行配置分析 。 Visual Studio中集成的dotMemory获得了更多的性能分析功能。现在，您不仅可以分析启动项目，还可以分析任何“运行配置”-启动项目，任意可执行文件，甚至任何静态方法。

  • 改进的时间线图 ：

    • 现在，该时间表适用于所有类型的应用程序，包括.NET Core，ASP.NET Core，IIS托管的Web应用程序以及其他。

    • 时间线包含自GC图表以来的LOH中的新分配 。该图显示了对大型对象堆的即时内存分配，这对于检测大型对象的过多分配非常有帮助。

  • 现在，可终结对象检查不仅适用于通过dotMemory捕获的快照，而且还适用于原始内存转储。另外，现在，可终结对象被视为快照/转储中存在的对象，因此您可以打开它们并进行更彻底的检查。

  • 两种新的根类型： F-Reachable队列/ Finalization Queue 。

  • dotMemory控制台运行程序允许您为拍摄的快照设置名称。

dotMemory 2018.1

  • 改进的用户界面。2018.1版本带来了多个次要的用户界面改进。最重要的更改是重新设计的“ 分析主题”面板。现在，导航分析路径要容易得多。

dotMemory 2017.3

  • 改进搜索 。现在，在对象集视图中搜索特定对象时，可以通过使用特殊符号来使搜索更高效。

dotMemory 2017.2

  • 导入内存转储 。现在，您可以导入Windows原始内存转储（通常，您可以使用任务管理器或Process Explorer工具获得它们），并使用dotMemory的所有功能对其进行分析。

  • 改进了“按主导者分组”的观点 。重新整理了 “ 按统治者分组”视图，以使您更容易理解应用程序中的关键对象。现在将主控树与主控森伯斯特图表合并。

  • 改进的用户界面。dotMemory用户界面经过了稍微的重新设计，可以更轻松地浏览分析主题和视图。这项工作的另一个好处是增加了视野。

dotMemory 2017.1

  • 使用拖放将探查器附加到正在运行的应用程序 。

  • 命令行探查器 。 dotMemory 2017.1包含dotMemory.exe工具，可让您从命令行进行概要分析。当您需要自动执行收集内存快照的过程时，该工具非常有用。

dotMemory 2016.3

  • 终结对象检查 。 dotMemory检查您的应用程序是否有最终确定的对象以及排队等待最终确定的对象。

  • 现成的.NET Core应用程序配置文件 。

dotMemory 2016.2

  • 比较来自不同会话的快照 。 dotMemory允许您比较存储在不同工作空间中的内存快照：即在不同的分析会话中收集的快照。

dotMemory 2016.1

  • 浏览查看历史记录 。

dotMemory 10

  • 导航到Visual Studio中的类型声明 。

  • dotMemory主视图。
    dotMemory收到的主视图类似于最新版本的dotTrace中的可用视图。
