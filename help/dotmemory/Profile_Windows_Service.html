配置文件Windows服务

剖析.NET Windows服务

  1. 作为独立应用程序运行dotMemory 。dotMemory窗口将打开。

  2. 在左侧面板中，选择Local 。

  3. 在Profile Application的中间面板中，选择Windows Service 。

  4. 在右侧面板的Windows Service中 ：

    • 在“ 服务”中 ，选择要配置的服务。

    • （可选）如果您的服务需要命令行参数，请选择Advanced并指定Arguments 。

    • （可选）如果您的服务需要启动其他服务，请选择高级和启动依赖服务 。

    有关Windows服务选项的更多详细信息，请参阅Windows服务部分。

  5. *这将使您看到分配了特定对象的方法。

    如果您希望dotMemory在分析会话开始后立即开始收集堆栈跟踪分配数据*，请选择右侧面板中的立即开始收集分配数据选项。
    有关探查器选项的更多详细信息，请参阅探查器选项部分。

  6. （可选）如果要使用API（而不是Controller窗口）控制分析会话（拍摄快照，结束会话等），请选择通过API控制分析 。有关如何使用API控制分析的更多详细信息，请参阅通过API控制分析会话 。

  7. 点击运行 。

  8. 拍摄快照。有关如何控制性能分析会话的更多详细信息，请参阅“控制性能分析会话” （如果您手动控制会话）或“ 通过API控制性能分析会话” （如果您使用API控制性能分析）。

  9. 收集数据后，使用“ 控制器”窗口的“ 分离”按钮分离探查器。如果使用API控制配置文件，则代码必须提供分离功能。

  10. 分析收集的快照 。
