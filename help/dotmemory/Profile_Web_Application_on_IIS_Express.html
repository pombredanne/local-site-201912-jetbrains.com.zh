IIS Express中的配置文件Web应用程序

剖析IIS Express中托管的Web应用程序

  1. 关闭所有正在运行的IIS Express实例。

  2. 作为独立应用程序运行dotMemory 。dotMemory窗口将打开。

  3. 在左侧面板中，选择Local 。

  4. 在Profile Application的中间面板中，选择IIS Express 。

  5. 在右侧面板中的IIS Express应用程序中 ：

    • 如果要配置的Web应用在默认的IIS Express配置文件中进行了描述applicationhost.config ，选择“ 配置文件” ，然后在“ 站点”中选择要配置的网站。

      请注意，配置文件的位置取决于您使用的IDE：

      • JetBrains骑士： (solutionDir)\.idea\config\applicationhost.config

      • Microsoft Visual Studio 2015及更高版本： (solutionDir)\.vs\config\applicationhost.config

      • Microsoft Visual Studio 2013及更早版本： \Documents\IISExpress\config\applicationhost.config

      如果您不想使用配置文件，则可以选择“ 应用程序路径”，并指定包含配置文件应用程序的目录的完整路径。

    • 如果要自动打开应用程序URL，则在分析开始后，选择“ 打开URL”并指定应用程序URL。使用浏览器中的参数可以选择必须在其中打开URL的浏览器。

    • （可选）如果选择“ 高级” ，则可以指定自定义IIS Express和运行时设置：

      • 在“ 服务器”中 ，选择一个IIS Express实例，

      • 在运行时版本 ：.NET Framework版本中，

      • 在Port中 ：一个HTTP端口，或者如果您希望dotMemory自动选择端口，请选择Use random port。

  6. *这将使您看到分配了特定对象的方法。

    如果您希望dotMemory在分析会话开始后立即开始收集堆栈跟踪分配数据*，请选择右侧面板中的“ 从开始处收集内存分配和流量数据”选项。
    有关探查器选项的更多详细信息，请参阅探查器选项部分。

  7. （可选）如果要使用API（而不是Controller窗口）控制分析会话（拍摄快照，结束会话等），请选择通过API控制分析 。有关如何使用API控制分析的更多详细信息，请参阅通过API控制分析会话 。

  8. 点击运行 。

  9. 拍摄快照。有关如何控制性能分析会话的更多详细信息，请参阅“控制性能分析会话” （如果您手动控制会话）或“ 通过API控制性能分析会话” （如果您使用API控制性能分析）。

  10. 收集数据后，关闭配置文件的应用程序或使用“ 控制器”窗口的“ 分离”按钮分离配置文件 。如果使用API控制配置文件，则必须由代码执行分离。

  11. 分析收集的快照 。
