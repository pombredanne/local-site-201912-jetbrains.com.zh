配置文件运行过程

剖析已经运行的进程

  1. 作为独立应用程序运行dotMemory 。dotMemory窗口将打开。

  2. 在左侧面板中，选择附加 。

  3. 在中间面板中，选择要分析的.NET进程。仅显示在.NET Framework 4.0或更高版本上运行的进程，并且可以附加这些进程。

    如果缺少您要查找的进程，请单击“ 刷新”以更新进程列表，和/或单击“ 显示所有进程”以授予dotMemory管理权限（这将显示系统中正在运行的更多进程）。

  4. （可选）如果要使用API（而不是“ 控制器”窗口）控制分析会话（拍摄快照，结束会话等），请在右侧面板中选择“ 通过API控制分析” 。有关如何使用API控制分析的更多详细信息，请参阅通过API控制分析会话 。

  5. 点击运行 。

  6. 拍摄快照。有关如何控制性能分析会话的更多详细信息，请参阅“控制性能分析会话” （如果您手动控制会话）或“ 通过API控制性能分析会话” （如果您使用API控制性能分析）。

  7. 收集数据后，关闭配置文件的应用程序或使用“ 控制器”窗口的“ 分离”按钮分离配置文件 。如果使用API控制配置文件，则必须由代码执行分离。

  8. 分析收集的快照 。
