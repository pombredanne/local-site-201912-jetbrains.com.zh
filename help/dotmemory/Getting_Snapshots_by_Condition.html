按条件获取快照

有时，您可能只在发生特定情况时才对快照感兴趣。当应用程序的内存消耗急剧增加时，最流行的情况是拍摄快照。由于手动执行此操作（使用“ 获取快照”按钮）非常棘手，因此dotMemory允许您自动执行此过程。您所需要做的只是设置一个特定条件，条件发生后将自动收集快照。当前，支持以下条件：

  • 内存消耗以MB为单位增加。

  • 内存消耗增加了％。

  • 以分钟为单位的时间段（定期拍摄快照）。

要更全面地控制快照拍摄条件（例如，在程序的特定执行点拍摄快照），请使用dotMemory API 。

*总内存（即使清除了“ 显示非托管内存”复选框，也包括非托管内存 ）。

启用按条件收集快照

  1. 分析会话开始后，单击分析控件中的“ 按条件获取快照”按钮。

  2. 在“ 自动获取快照”部分中选择所需条件，并为其指定特定值：

    • 如果内存消耗增加 [X] MB ，则当内存消耗*超过基线X MB时自动获取快照。第一个基准是单击“ 按条件获取快照”窗口中的“ 开始”按钮时的内存消耗。条件发生后，dotMemory将获得快照，并且此时的内存消耗将成为新的基准。

    • 如果内存消耗增加 [X] ％ ，则在内存消耗超过基准X％时自动获取快照。第一个基准是单击“ 按条件获取快照”窗口中的“ 开始”按钮时的内存消耗。条件发生后，dotMemory将获得快照，并且此时的内存消耗将成为新的基准。

    • 如果内存消耗超过 [X] MB-当总内存消耗（绝对值）超过X MB时自动获取快照。

    • 每 [X] 分钟 -每X分钟自动获取一个快照。

  3. 如果您希望dotMemory拍摄不超过X个快照，请选中“ 将最大快照数量限制为 [X]”复选框。如果磁盘空间有限，此选项可能很有用。

  4. 点击开始 。

禁用按条件收集快照

  1. 单击分析控件中的“ 按条件获取快照”按钮。

  2. 点击停止 。
