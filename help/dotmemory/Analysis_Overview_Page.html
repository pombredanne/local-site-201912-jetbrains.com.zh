分析概述

收集概要分析数据后，dotMemory会在“ 分析概述”页面中打开一个新选项卡。

您可以一次对同一数据执行许多分析。要将当前分析复制到新选项卡，请单击在dotMemory窗口的右上角，或使用Ctrl+T组合。
要开始新的分析，请使用Ctrl+Alt+T组合。

“ 分析概述”页面为您提供了有关在会话期间收集的数据的信息：

时间线
时间线图显示了您的应用在性能分析过程中如何消耗内存。时间轴由四个图表组成，分别显示Gen0，Gen1，Gen2堆和大型对象堆的即时大小。您还可以通过选择显示非托管内存参数来添加显示非托管内存大小的图表。要查看特定时间点的确切内存堆值，请单击此时间点的时间线图。
收集的快照

此区域包含有关概要分析期间收集的所有快照的简短报告。每个报告都允许您：

  • 获取有关您的应用请求的内存总量 *的详细信息。的合计值（灰色条）组成：

    • 非托管内存：分配给托管堆之外且不受垃圾收集器管理的内存。通常，这是.NET CLR，动态库，图形缓冲区等所需的内存。无法在事件探查器中分析这部分内存。
      如果清除显示非托管内存复选框，则将从总值中减去非托管内存的大小。

    • .NET，总计 （带轮廓的灰色条）：应用程序使用的托管堆中的内存量，包括分配的对象之间的可用空间。有关更多详细信息，请参见下图。

    • .NET，已使用 （深灰色栏）：应用程序使用的托管堆中的内存量，不包括可用空间。这是.NET允许您使用的唯一内存部分。

    *此大小等于Windows Task Manager的Commit大小 ：进程请求的内存量。 dotMemory使用PROCESS_MEMORY_COUNTERS.PagefileUsage获取此值。

  • 通过单击快照名称在“ 快照”视图中打开快照。

  • 使用重新命名快照按钮。

  • 打开堆中的所有对象进行分析（使用[objects size] objects链接）。在这种情况下，“所有对象”对象集将在“ 按类型分组”视图中打开。

  • 打开标记的对象进行分析（使用“ 标记的对象”链接）。“标记的对象”对象集将在“ 按类型分组”视图中打开。在标记实例一节中了解有关标记对象的更多信息。

  • 从分析会话开始到获取快照的那一刻，查看应用程序生成的内存流量（使用“ 内存流量”链接）。了解更多关于在部分分析流量分析流量 。


快照比较区
要比较两个快照，请将它们拖放到“ 比较”区域，或单击“ 添加到每个快照的比较 ”。了解更多关于如何比较在部分快照比较快照 。
