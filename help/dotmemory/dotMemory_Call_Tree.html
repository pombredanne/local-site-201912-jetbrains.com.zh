呼叫树

在“ 调用树 ”中，调用列表从堆栈中的第一个调用开始，一直到直接创建对象集的那个调用。

呼叫列表包括以下几列：

名称

描述

功能

函数名称。

字节数

调用分配的对象的大小。

物件计数

调用分配的对象数。

子树中的字节

所有基础调用分配的对象的大小。

对象在子树中计数

所有基础调用分配的对象数。

命名空间

调用所属的名称空间。

按名称查找功能

按名称查找功能

  1. 点击 查找功能按钮或使用Ctrl+F捷径。将出现“ 查找功能”弹出窗口。

  2. 开始在字段中键入字符。键入时，项目列表会变窄，建议与输入的子字符串匹配的函数名称。

请注意，您可以使用CamelHumps轻松查找某些功能。例如，找到RenderDataDrawingContext功能，您只能输入“ RDDC”。CamelHumps是不区分大小写的功能，因此“ rddc”和“ RDDC”之间没有区别。

选择对象进行进一步分析

调用树不允许您打开在分析会话之前分配的对象或在会话期间跳过的对象。要打开此类对象，请使用Back Traces 。

如果多次调用一个函数，则某些调用将在分配列表中单独显示。为了方便起见，您不仅可以选择特定调用创建的对象，还可以选择所有函数调用创建的对象。

更详细地，您可以选择以下主题进行进一步分析：

  • 在特定功能实例中分配的对象 ;

  • 在功能子树中分配的对象 ;

  • 在函数的所有实例中分配的对象 。

  • 由整个对象集专有保留（控制）的对象 。

选择由特定功能实例分配的对象（精确调用）

  • 请执行以下任一操作：

    • 双击相应的功能。

    • 右键单击该函数，然后选择“ 打开由该函数实例分配的对象” 。

    此后，将在函数[function_name]主题的实例中创建实例并将其添加到“ 分析路径”，并且所需对象的列表将显示在“ 按类型分组”视图中。

选择在函数子树中分配的对象（函数实例及其所有基础调用）

  • 右键单击该函数，然后选择“ 打开此子树中所有函数分配的对象” 。

    此后， 功能[function_name]主题的Created in子树将添加到Analysis Path，并且所需对象的列表将显示在Group by Types视图中。

选择在函数的所有实例中分配的对象

  • 右键单击该函数，然后选择打开该函数的所有实例分配的对象 。

    此后，将在函数[Function_name]的所有实例中创建的对象将添加到“ 分析路径”，并且所需对象的列表将显示在“ 按类型分组”视图中。

选择分析对象集专有保留的对象

  • 点击 打开此设置按钮保留的对象 。

    此后，“ 专有保留的对象”主题将添加到“ 分析路径”，并且所需对象的列表将显示在“ 按类型分组”视图中。
