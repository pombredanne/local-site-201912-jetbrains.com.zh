Profile .NET Core应用程序

剖析.NET Core应用程序

  1. 作为独立应用程序运行dotMemory 。dotMemory窗口将打开。

  2. 在左侧面板中，选择Local 。

  3. 在Profile Application的中间面板中，选择.NET Core Application 。

  4. 在右侧窗格中的.NET Core应用程序中 ：

    • 在“ 应用程序”中 ，指定应用程序可执行文件（如果是独立部署）或库文件（如果它是与框架相关的部署）的路径。

    • （可选）如果您的应用程序需要命令行参数，请选择Advanced并指定Arguments 。

    • （可选）如果您的应用程序创建了多个进程，并且您想对特定进程进行概要分析，请选择高级和概要文件子进程 。如果需要，请按照“ 过滤子进程”部分中的说明添加进程过滤器。

  5. *这将使您看到分配了特定对象的方法。

    如果您希望dotMemory在分析会话开始后立即开始收集堆栈跟踪分配数据*，请选择右侧面板中的“ 从开始处收集内存分配和流量数据”选项。
    有关探查器选项的更多详细信息，请参阅探查器选项部分。

  6. （可选）如果要使用API（而不是Controller窗口）控制分析会话（拍摄快照，结束会话等），请选择通过API控制分析 。有关如何使用API控制分析的更多详细信息，请参阅通过API控制分析会话 。

  7. 点击运行 。

  8. 拍摄快照。有关如何控制性能分析会话的更多详细信息，请参阅“控制性能分析会话” （如果您手动控制会话）或“ 通过API控制性能分析会话” （如果您使用API控制性能分析）。

  9. 收集数据后，关闭配置文件的应用程序或使用“ 控制器”窗口的“ 分离”按钮分离配置文件 。如果使用API控制配置文件，则必须由代码执行分离。

  10. 分析收集的快照 。
