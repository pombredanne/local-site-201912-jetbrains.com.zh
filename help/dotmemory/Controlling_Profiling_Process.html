控制性能分析会话

运行性能分析会话后，dotMemory将打开新的“ 分析”选项卡并启动配置文件的应用程序。

要控制配置过程，请使用页面顶部的以下按钮 ：

  • Force GC ：强制在应用程序中进行完全垃圾收集。

  • 终止进程 ： 终止当前选择的应用程序进程。

    1。请谨慎使用Kill Process ：无条件终止进程可能会导致您的应用使用的数据丢失。通常，您应该以常规方式关闭已配置的应用程序。
    2。由于技术限制，Silverlight进程无法正确终止。强烈建议您不要使用Kill Process杀死它们，因为这可能会导致意外的结果。

  • 杀死所有进程* ：杀死应用程序的所有进程。

  • 分离** ：将分析器与应用程序分离，但保持应用程序运行。

  • 收集分配 ：启用和禁用收集内存分配数据。
    使用此选项可以减少事件探查器的开销，并仅在需要时才收集分配数据（例如，仅在使用要探查的应用程序功能时）。请注意，如果在性能分析期间至少关闭一次收集分配，则内存流量数据将不完整。在分析内存流量中了解更多信息。

  • 按条件获取快照 ：打开“ 按条件获取快照”窗口。使用此窗口，您可以将dotMemory配置为在发生特定情况时自动获取快照。在“ 按条件获取快照”中了解更多信息。

  • 获取快照 ：获取在进程列表中选择的进程的内存快照 *。快照已添加到收集的快照区域 。获取快照不会中断配置过程，允许您获取另一个快照（例如，可以与以后的第一个快照进行比较）。完成捕获快照后，您可以继续分析分析结果 。

*仅当您分析运行多个进程的应用程序时，才会显示进程列表和“ 杀死所有进程”按钮。

**仅当您将探查器连接到已经运行的应用程序时，“ 分离”操作才可用。

分析多进程应用程序

如果您的独立或.NET Core应用程序运行多个进程，则可以获取特定进程的内存快照。

获取特定过程的快照

  1. 使用启用的“ 个人档案”子进程 分析选项开始分析。

  2. 分析会话开始后，从下拉列表中选择所需的过程 。

  3. 单击获取快照 。

查看实时分析数据

时间线图显示您的应用如何实时消耗内存。时间轴由许多图表组成，这些图表显示：

  • 非托管内存 ：瞬间大小非托管内存 。要添加此图表，请选中“ 显示非托管内存”复选框。 。

  • 堆生成0、1、2， 大对象堆（LOH） ：Gen0，Gen1，Gen2堆和大对象堆的即时大小。

  • 从GC开始在LOH中分配 ：自上次垃圾回收以来已分配给Large Object Heap的对象的即时大小（在每个GC之后，图表从零重新开始）。使用此图可以了解何时发生LOH分配（在应用程序启动时，在某些工作中等）以及它们的强度（例如，应用程序可以具有一些不改变LOH大小的重要LOH内存流量）。

在单独的窗口中显示分析控件

您可以通过单击将分析控件分离到单独的“ 分析控制器”窗口。 在按钮 。控制器窗口始终显示在其他窗口的顶部，这在只有一个显示器的台式机上特别有用。
