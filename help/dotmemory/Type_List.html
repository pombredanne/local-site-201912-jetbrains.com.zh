按类型分组

“ 按类型分组”视图显示了所选对象集中的对象列表。为了方便起见，所有对象均按其类型分组：列表中的每一行代表某种类型。集合中存在的所有该类型的对象均计入此行。您可以使用“ 按类型分组”视图来标识消耗过多内存的对象或根本不应该在内存中的对象。

该列表包括以下几列：

名称

描述

类型

输入名称。

物件计数

相同类型的对象数。

字节数

对象的整体浅层大小（以字节为单位）。

最小保留字节

所有排他保留的对象的总大小（以字节为单位）。如果删除某个类型的所有对象，这是释放多少字节的较低估计。

将列表中的对象分组

为了方便起见，您可以通过四种不同方式对列表进行分组：

  • 普通清单
    对象显示在普通列表中。这是识别具有最高内存使用率的对象的最简单方法。

  • 按名称空间分组
    对象按其名称空间分组。当您想集中精力确定自己班级中的问题时，这种分组非常方便。

  • 按组装分组
    对象按照它们来自的程序集进行分组。这种分组方式也可以用于将您自己的类与系统类分开。

  • 按接口分组
    对象按它们实现的接口分组。如果一个对象实现了多个接口，它将显示在每个实现的接口下。

选择对象进行进一步分析

在“ 按类型分组”视图中，可以选择以下主题以进行进一步分析：

  • 某种类型的对象 ;

  • 由整个对象集专有保留（控制）的对象 。

该列表支持多选。要选择多个项目，请在按住所需项目的同时单击所需项目。 Ctrl键。要选择多个相邻项目，请在按住第一个的同时单击第一个，然后单击最后一个Shift键。这将允许分析一组Selected对象 。

选择特定类型的对象

  • 请执行以下任一操作：

    • 双击列表中的类型。

    • 右键单击类型，然后选择打开此对象集 。

    之后，将所选对象集添加到“ 分析路径” ，您可以使用其他对象集视图来更详细地分析对象。

选择分析对象集专有保留的对象

  • 点击 打开此设置按钮保留的对象 。

    此后，“ 专有保留的对象”主题将添加到“ 分析路径”，并且所需对象的列表将显示在“ 按类型分组”视图中。
