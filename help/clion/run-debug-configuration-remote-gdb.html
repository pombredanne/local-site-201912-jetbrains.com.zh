Run/Debug Configuration: GDB Remote Debug

Use this page to configure Run/Debug options for remote GDB debug.

Click here for the description of the options that are common for all run/debug configurations.

Remote GDB configuration settings

Item

Description

GDB

This field suggests Bundled GDB multiarch by default. To specify a custom GDB, provide the path by clicking the button and choosing the directory that contains the GDB executable.

'target remote' args

In this field, specify the medium to carry the debugging packets (serial line, or an IP network using TCP or UDP). Find here more details on connecting to a Remote Target.

Symbol file

If you have connected to the remote GDB server and started the debug session, but your program doesn't stop when reaching a breakpoint, check up that symbol file and path mappings are specified correctly.

In this field, specify the location of debug symbol file. Type the path or click the button and choose it from the dialog box that opens.

Sysroot

In this field, specify the location of sysroot. Type the path or click the button and choose it from the dialog box that opens.

Path mappings

In this pane, in the Remote column specify paths on target machine (like path to project sources etc) which should be mapped to Local paths on host (specified in the Local column).
