远程GDB服务器

本地操作系统 ：macOS / Linux / Windows

远程操作系统 ：任何支持的gdbserver和SSH

所需的工具 ： gdbserver的上tagret

二进制文件同步 ：自动

客户端调试器 ：工具链的GDB

项目格式 ：CMake

远程GDB服务器配置在本地构建CMake目标，将二进制文件上传到远程计算机，在gdbserver下启动它，并连接CLion调试器。

创建一个远程GDB服务器配置

  1. 去运行|编辑配置 ，单击 ，然后从模板列表中选择“ Remote GDB Server ”。

  2. 指定以下设置：

    • 目标和可执行

      选择要构建的目标和要远程上传的可执行文件。

    • 证书

      请点击并提供用于访问远程计算机的凭据：

      CLion会将此连接实体添加到“ 设置” /“首选项” |“服务器”中的服务器访问配置列表中。构建，执行，部署|部署 。

    • 上传可执行文件

      每次启动调试会话（ 始终 ）或二进制文件中的更改（ 仅更新 ）时，都可以上传可执行文件。您也可以选择跳过上传（ 无 ）。

    • 上载路径

      指定将放置可执行文件的远程文件夹。

    • “目标远程”参数

      提供介质来携带调试数据包（串行线或使用TCP或UDP的IP网络）。有关连接到远程目标的更多详细信息，请参见gdb文档 。

    • GDB服务器

      设置gdbserver在远程计算机上的位置。默认路径将自动填充。

    • GDB服务器参数

      指定将传递给gdbserver的参数：连接端口，可执行路径和其他参数（如果需要）。

      默认情况下，此字段包含您在“目标远程”参数中提供的端口以及上载路径后的路径，后跟二进制名称。

默认情况下，此配置使用当前选定的CMake配置文件构建项目，并从“ 生成路径”文件夹中获取二进制文件。

如果您的设置需要交叉编译器工具链，请确保在启动会话之前创建它，配置相应的“ 调试”配置文件并切换到该配置文件。

启动远程调试会话

  1. 在macOS和Linux上，确保在工具链中选择了GDB 。

  2. 在代码中放置断点后，选择新创建的Remote GDB Server配置，然后按 （ Shift+F9 ）。

    • 您可以在“ 消息”工具窗口中找到构建日志：

    • 要监视上传过程，请切换到“ 文件传输”窗口：

    • 调试工具窗口的控制台选项卡显示GDB / gdbserver连接的状态：

  3. 您可以照常调试程序：检查变量，逐步执行，求值表达式，并使用所有其他调试功能。
