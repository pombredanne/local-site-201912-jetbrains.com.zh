配置项目和IDE设置

在CLion中，您可以在两个级别上配置设置-项目级别和IDE级别。

项目级设置

项目级别设置仅应用于当前项目。这些设置与其他项目文件一起存储在.idea目录中。

如果您的项目受版本控制，建议您将具有项目特定设置的XML文件（项目文件夹内的.idea文件夹）存储在版本控制下，但工作区 .xml和task.xml除外 ，它们存储用户特定的设置。

您可以在GitHub上参考JetBrains.gitignore来查看应忽略的内容。

访问项目级别的设置

  • 选择文件|设置 Windows和Linux，或者克利翁| macOS的首选项 （ Ctrl+Alt+S ）。

    在打开的“设置/首选项”对话框中，项目级别的设置标记为 。

IDE级别（全局）设置

IDE级别 （全局）设置将应用于所有新创建的项目。IDE级别设置的列表包含较少的选项，但是它使您能够更改编辑器设置，创建自定义范围，配置检查，设置VCS特定的设置，等等。

如果要在现有项目之间共享IDE设置，则可以使用“ 设置存储库”或“ 设置同步”插件。您还可以将设置导出到ZIP存档中 ，以后再导入到其他IDE实例中。但是，请注意，如果您已经使用“ 设置存储库”或“ 同步插件 ” ，则设置导入可能无法正常进行。

访问IDE级别的设置

  • 从主菜单中，选择文件| Windows和Linux的新项目的设置 ，或“ 文件” |“设置”。 macOS 新项目的首选项 。
