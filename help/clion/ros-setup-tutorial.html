ROS设置教程

ROS （机器人操作系统）是为机器人应用程序设计的一组库和工具。它主要针对C ++和Python开发，并使用基于CMake和Python脚本的catkin构建系统。ROS软件发行版适用于Linux，也适用于处于实验模式的macOS。

您可以将CLion用作ROS项目的IDE。本教程介绍了如何设置工作流程，并提供了一个创建ROS软件包并在CLion中使用它的示例。请注意，此安装过程已在Ubuntu上进行了测试。

在CLion中，还有一些可用于ROS开发的第三方插件： ROS-Robot操作系统和Hatchery 。

注意 ：CLion团队不维护或不支持第三方插件。

在源环境中启动CLion

需要告知CLion特定于ROS的环境变量。通过运行来为工作空间提供源时，将在当前shell中检索这些变量。

源./devel/setup.bash
为CLion提供ROS环境变量的最简单方法是从同一shell启动IDE 。在获取工作空间后，您可以在同一终端中键入：
sh PATH_TO_CLION / bin / clion.sh

在CLion中打开ROS项目

在源环境中启动CLion后，请执行以下操作之一以打开ROS项目：

  • 单击文件|打开并选择ROS工作区的src目录中的CMakeLists.txt ，然后选择“ 以项目形式打开” 。

  • 单击文件|从Sources中新建CMake Project，然后选择工作区的src目录以从中导入项目。

CMake工具窗口将显示项目加载的状态：

设置catkin工作区的构建路径

默认情况下，CLion将构建输出放置在自动创建的cmake-build-debug或cmake-build-release目录中。对于ROS开发，这意味着您将在CLion和运行控制台中拥有两个不同的版本catkin_make 。

要在IDE和控制台上进行单一构建 ，您需要将CLion构建路径设置为catkin工作区目录。为此，转到文件|设定值Ctrl+Alt+S |构建，执行，部署| CMake和更改两个字段：

  • 在“ 生成路径”中 ，设置“ workspace_folder / build” 。

  • 在CMake选项中 ，添加-DCATKIN_DEVEL_PREFIX:PATH=workspace_folder/devel 。

使用启动文件

您可以在CLion中将ROS节点作为常规应用程序运行和调试 。

启动文件不能直接执行，但是可以使用XML语法突出显示和完成来编辑它们， 并将调试器附加到正在运行的节点上。

将调试器附加到正在运行的节点

  1. 从命令行运行.launch文件。例如

    roslaunch roscpp_tutorials talker_listener.launch

  2. 通话运行|从主菜单附加到处理或按Ctrl+Alt+F5 。

示例：创建一个基本的ROS节点，在CLion中编辑并运行它

在此示例中，我们将创建一个简单的ROS包，编辑源代码，然后从CLion运行该节点。我们将在Ubuntu 18.04上使用ROS Melodic，并使用ROS Wiki上的“ 编写简单发布者和订阅者（C ++）”教程中的基本发布者节点。

创建一个基本的ROS包

  1. 创建和构建ROS工作空间：

    mkdir -p ros_workspace / src cd ros_workspace catkin_make

  2. 在工作空间中，创建一个名为my_package的包：

    cd src catkin_create_pkg my_package roscpp rospy std_msgs

启动CLion

  1. 来源工作空间：

    光盘../ ../ ../源./devel/setup.bash

  2. 并在同一终端中启动CLion：

    sh /opt/clion-2018.2/bin/clion.sh

在CLion中将包作为项目打开

  • 在CLion中，转到“ 文件|打开并选择位于package文件夹内的CMakeLists.txt文件，然后选择将其作为项目打开：

添加源文件并编辑CMakeLists.txt

  1. 将新的源文件添加到项目：在“项目”树中的src上单击鼠标右键，选择“ 新建” |“新建”。 C / C ++源文件，并将其命名为my_package.cpp 。

  2. 将整个talker.cpp文件从发布者和订阅者教程复制到my_package.cpp中 。

  3. 新创建的源添加到的CMakeLists.txt和链接其对柳絮库：

    add_executable（my_package src / my_package.cpp）target_link_libraries（my_package $ {catkin_LIBRARIES}）
    之后， 重新加载 CMake项目，并在“运行/调试”配置列表中注意my_package ：
