配置数据库连接

记住密码

您可以选择以下选项来存储密码：

  • 从不 ：每次与数据库建立连接时都会提示输入密码。

  • 在重启之前 ：仅保存当前CLion运行的密码。如果退出CLion并再次打开它，则必须再次提供密码。

  • 对于会话 ：仅在与数据库的当前连接会话中保存密码（直到您断开与数据库的连接）。您可以通过按Disconnect图标断开连接 （ ） 要么Ctrl+F2 。

  • 永远 ：密码保存在CLion存储中。下次打开CLion时无需提供密码。

关闭数据库连接

CLion自动连接到数据库。具有打开的数据库连接的数据源的名称在“ 数据库”工具窗口中以粗体显示。

  • 要关闭数据库连接，请选择一个数据源并单击“ 断开连接”按钮在工具栏上。或者，选择一个数据源，然后按Ctrl+F2 。

为数据源分配颜色

为了更好地区分生产数据库和测试数据库，可以为数据源分配颜色。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要为其分配颜色的数据源。

  3. 点击选择颜色图标（ ），然后选择一种颜色。

  4. 单击确定 。

使用JDBC驱动程序

使用JDBC驱动程序创建与数据库的连接

如果在数据源列表中找不到数据库供应商的名称，请下载数据库管理系统（DBMS）的JDBC驱动程序，然后在CLion中创建连接。使用JDBC驱动程序，您可以连接到DBMS并开始工作。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择驱动程序和数据源 。

  3. 单击用户驱动程序链接。

  4. 在驱动程序文件窗格中，单击添加图标，然后选择自定义JAR 。

  5. 导航到JDBC驱动程序的JAR文件，选择它，然后单击OK 。

  6. 在“ 类”字段中，指定要用于驱动程序的值。

  7. 点击应用 。

  8. 返回到创建的数据源连接。

  9. 指定数据库连接详细信息。或者，将JDBC URL粘贴在URL字段中。

    要删除密码，请右键单击“ 密码”字段，然后选择“ 设置为空” 。

  10. 要确保成功连接到数据源，请单击“ 测试连接” 。

如果您连接到不在支持的数据源列表中的供应商，则CLion将JDBC元数据用于数据库对象检索（自省）和通用SQL方言。JDBC元数据的自省意味着某些特定的数据库对象将不会出现在数据库树视图中。例如，用户定义的类型（Apache Cassandra），宏（Apache Hive），扩展（PostgreSQL）。使用通用SQL方言，您将具有基本的代码补全功能，例如SELECT * FROM ，但代码完成将不包含在内省期间未检索到的对象。

将JDBC驱动程序添加到现有连接

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 在“ 数据源和驱动程序”对话框中，选择要在其中更改驱动程序的数据源，然后单击数据源设置中的“ 驱动程序”链接。

  3. 点击提供的驱动程序条目，然后点击删除 （ ）。

    要还原更改，请单击窗口右上方的“ 重置” 。或者，点击添加按钮（ ），然后选择提供的驱动程序| |

  4. 在驱动程序文件窗格中，单击添加图标（ ），然后选择“ 自定义JAR” 。

  5. 导航到JDBC驱动程序的JAR文件，选择它，然后单击OK 。

  6. 在“ 类”字段中，指定要用于驱动程序的值。

  7. 点击应用 。

向用户驱动程序添加图标

您可以更改用户驱动程序的默认图标。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标（ ），然后选择您创建的驱动程序。要创建用户驱动程序，请参阅将JDBC驱动程序添加到现有连接 。

  2. 单击选项选项卡，向下滚动到其他部分。

  3. 从“ 图标”列表中，选择要分配给驱动程序的图标。

  4. 单击确定 。

配置连接选项

启用单连接模式

单连接模式意味着数据源和所有控制台都使用一个且相同的连接。此模式允许您查看数据库树中的临时对象，或在不同的控制台中使用同一事务。有关控制台的更多信息，请阅读Database Console 。

应用单连接模式时，必须关闭所有现有的打开的连接。如果您打开了连接，CLion将显示一条通知。要关闭所选数据源的打开的连接，请单击是 。要关闭所有已修改数据源的打开的连接，请对所有单击是 。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要启用单连接模式的数据源。

  3. 单击选项选项卡。

  4. 选择单一连接模式 。

  5. 点击应用 。

启用单一数据库模式

当您连接到数据源时，CLion可以检索并显示该数据源具有的所有数据库。但是在某些情况下（例如，使用PgBouncer的某些设置），您可以或被允许仅使用特定的数据库。启用单数据库模式后 ，您在数据库树视图中仅看到在连接设置中指定的数据库，而不是从数据源接收到的所有数据库。

考虑对PostgreSQL，Azure SQL数据库，Greenplum，Amazon Redshift使用此设置。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要启用单连接模式的数据源。

  3. 单击选项选项卡。

  4. 选择单一数据库模式 。

  5. 点击应用 。

启用连接的只读模式

  1. 导航到文件|数据源 Shift+Enter 。

  2. 在“ 数据源”列表中，选择必要的数据源。

  3. 单击选项选项卡。

  4. 选择只读复选框。

保持连接畅通

通过在指定时间段后运行保持活动查询，可以保持与数据库的连接处于活动状态。您可以在驱动程序设置中为不支持的数据库定义自定义查询。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要修改的数据源。

  3. 在“ 选项”选项卡上，选中“每N秒运行一次保持活动查询”复选框，其中N是几秒钟后，CLion再次运行保持活动查询。

要为驱动程序设置自定义保持活动查询，请在“ 驱动程序”列表中选择所需的驱动程序 。单击选项选项卡。在保持活动查询字段中，指定要用作保持活动查询的查询。

在指定时间内断开与数据库的连接

您可以指定以秒为单位的时间，在此之后CLion终止连接。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要修改的数据源，然后单击选项选项卡。

  3. 在“ 选项”选项卡上，选中“ N秒后自动断开连接”复选框，其中N CLion终止连接后的秒数。

设置会话的时区

当您开始与数据库进行交互时，您将开始数据库会话。您可以分配该会话将在其中运行的时区。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要修改的数据源，然后单击选项选项卡。

  3. 在“ 时区”字段中，开始输入要使用的时区。

  4. 应用设置，然后单击“ 确定” 。

建立连接时运行预定义的SQL查询

每次建立连接时，您都可以运行预定义的SQL查询。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要修改的数据源。

  3. 在“ 选项”选项卡上的“ 启动脚本”字段中，指定计划在与数据库的连接上运行的SQL查询。

如果清除“ 单一连接模式”复选框，则每个新控制台都会创建一个新连接。

获取更多字段以进行MongoDB内省

您可以在数据库浏览器中观察集合和字段。CLion从每个集合中获取前10个文档，以获取有关字段的信息。您可以通过指定fetch_documents_for_metainfo数据源属性对话框的“ 高级”选项卡上的JDBC参数。

  1. 在数据库工具窗口（ 视图|工具窗口|数据库 ）中，右键单击MongoDB数据源，然后选择属性 。

  2. 单击高级选项卡。

  3. 在参数表中，向下滚动直到看到fetch_documents_for_metainfo 。

  4. 双击值单元格的fetch_documents_for_metainfo参数。

  5. 指定所需的值。

配置SSH和SSL

以下过程描述了适合大多数数据库的SSL配置。对于某些数据库，您需要使用另一种方法来成功建立连接。您可以在DataGrip文档中看到Cassandra和Heroku Postgres的配置示例。

使用SSL连接到数据库

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要在其中更改连接设置的数据源配置文件。

  3. 单击“ SSH / SSL”选项卡，然后选择“ 使用SSL”复选框。

  4. 在“ CA文件”字段中，导航到CA证书文件（例如， ca.pem ）。

  5. 在“ 客户端证书文件”字段中，导航到客户端证书文件（例如， client-cert.pem ）。

  6. 在“ 客户端密钥文件”字段中，导航到客户端密钥文件（例如， client-key.pem ）。

  7. 要确保成功连接到数据源，请单击“ 测试连接” 。

建议使用PEM证书。

使用自签名证书，在某些情况下使用由受信任的根实体发布的证书，当您使用最新的JDBC驱动程序版本时，可能会遇到错误。如果您的Java密钥库不接受证书链，则SSL连接可能会失败。作为临时解决方案，请尝试降级JDBC驱动程序（例如，对于MySQL连接器，您需要切换到5.1.40版本。）

使用SSH连接到数据库

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要在其中更改连接设置的数据源配置文件。

  3. 单击“ SSH / SSL”选项卡，然后选择“ 使用SSH隧道”复选框。

  4. 在“ 代理主机” ，“ 代理用户” ，“ 代理密码 ”和“ 端口”字段中，指定连接详细信息。

  5. 从“身份验证类型”列表中，选择一种身份验证方法：

    • 密码 ：使用密码访问主机。指定密码，然后选择“ 保存密码”复选框以将密码保存在CLion中。

    • 密钥对（OpenSSH或PuTTY） ：对密钥对使用SSH身份验证 。 要应用此身份验证方法，您必须在客户端计算机上具有私钥，而在远程服务器上必须具有公钥。CLion支持使用OpenSSH实用程序生成的私钥。

      指定存储私钥的文件的路径，然后在相应的字段中键入密码（如果有）。要让CLion记住密码，请选中“ 保存密码”复选框。

    • OpenSSH配置和身份验证代理 ：使用由凭证帮助程序（例如Windows上的Pageant或Mac和Linux上的ssh-agent ）管理的SSH密钥。

  6. 要确保成功连接到数据源，请单击“ 测试连接” 。

使用PuTTY创建SSH隧道（Windows）

  1. 下载并运行最新版本的PuTTY SSH和Telnet客户端（从https://www.putty.org/下载客户端）。

  2. 在“ PuTTY配置”对话框中，导航到“ 连接” |“连接”。 SSH |验证 。

  3. 在用于认证的私钥文件字段中，指定私钥文件的路径，然后单击打开 。

  4. 在命令行窗口中，指定用于SSH隧道的用户名，然后按Enter 。不要关闭命令行窗口。

  5. 在“数据库”窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标在工具栏上。

  6. 选择要在其中更改连接设置的数据源配置文件。

  7. 单击“ SSH / SSL”选项卡，然后选择“ 使用SSH隧道”复选框。

  8. 从“ 身份 验证类型”列表中，选择“ OpenSSH配置和身份验证代理” 。

  9. 在“ 代理主机” ，“ 代理用户 ”和“ 端口”字段中，指定连接详细信息。

  10. 要确保成功连接到数据源，请单击“ 测试连接” 。

使用Pageant创建SSH隧道（Windows）

Pageant是用于PuTTY，PSCP，PSFTP和Plink的SSH身份验证代理。Pageant会存储您的私钥，并且只要它正在运行，它就会为PuTTY或其他工具（如CLion）提供解锁的私钥。您可以在Windows任务栏中找到Pageant图标。

  1. 下载Pageant的最新版本（从https://www.putty.org/下载客户端）。

  2. 在Windows任务栏中，右键单击Pageant图标，然后选择Add Key 。

  3. 在“ 选择私钥文件”对话框中，导航到私钥文件（PPK文件），然后单击“ 打开” 。

  4. （可选）输入私钥密码，然后按Enter 。

  5. 在“数据库”窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标在工具栏上。

  6. 选择要在其中更改连接设置的数据源配置文件。

  7. 单击“ SSH / SSL”选项卡，然后选择“ 使用SSH隧道”复选框。

  8. 从“ 身份 验证类型”列表中，选择“ OpenSSH配置和身份验证代理” 。

  9. 在“ 代理主机” ，“ 代理用户 ”和“ 端口”字段中，指定连接详细信息。

  10. 要确保成功连接到数据源，请单击“ 测试连接” 。
