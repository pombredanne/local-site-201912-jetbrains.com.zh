CMake安装

如果您的工作流程包括项目安装，则可能要使用生成安装规则的CMake install命令。此命令有多个签名，例如TARGETS形式：

安装（TARGETS目标...[EXPORT导出名称] [[ARCHIVE | LIBRARY | RUNTIME | OBJECTS | FRAMEWORK | Bundle | PRIVATE_HEADER | PUBLIC_HEADER | RESOURCE] [DESTINATION dir] [PERMISSIONS权限...][配置[调试|发布| ...][组件] [可选] [EXCLUDE_FROM_ALL] [NAMELINK_ONLY | NAMELINK_SKIP]] [...][包括目的地[dir ...]]）

配置并运行安装

  1. 将安装命令放入您的CMakeLists.txt文件。

  2. 通过install命令的[DESTINATION dir]字段设置安装路径。在这里，您有两个选择：

    • 在完整路径上加一个斜杠或驱动器号。

    • 如果在“ 设置/首选项” | “ 设置” |“首选项” |“ CMake选项”部分提供，则使用相对路径，该相对路径将被解释为相对于CMAKE_INSTALL_PREFIX变量的值。 生成，执行，部署对话框：

    • 要仅运行安装，请致电Build |。从主菜单安装 ：

      CMake install会调用构建目标，因此您无需单独调用Build操作。

    • 如果您希望安装在启动之前自动进行，请打开“ 运行” | “启动”。在“编辑配置”对话框中，选择所需的“运行/调试”配置，然后按在启动前部分中，然后从下拉列表中选择安装 ：

      为了防止两次构建目标，请在添加Install之后排除Build步骤。

      然后调用“运行”或“ 调试”进行此配置。

    结果，将构建您在install命令中指定的目标，并将其放入所需的目录中。
