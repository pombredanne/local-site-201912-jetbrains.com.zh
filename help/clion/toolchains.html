工具链

在设置工具链首选项之前，请确保以下所有软件包均已下载并安装。

请参阅《 安装指南》和《 CLion快速入门指南》 。

档案|设置|构建，执行，部署| Windows和Linux的工具链
CLion |首选项|构建，执行，部署| macOS的工具链

创建工具链时，请将该页面用作参考。另请参阅CMake项目的工具链 。

工具链列表

您可以在此处创建或删除工具链并修改现有的工具链：

-使用按钮创建新的工具链配置。
-使用此按钮删除选定的工具链。
-使用此按钮在列表中向上移动所选工具链。
-使用此按钮在列表中向下移动选定的工具链。

工具链名称和类型

项目

描述

名称

指定工具链名称。

类型（Linux和macOS）

从下拉列表中选择：

  • 系统 -不用于远程开发的工具链的默认值。

  • 远程主机 -为“ 远程开发”选择此选项，并在“ 凭据”字段中提供远程主机凭据。

环境（Windows）

从下拉列表中选择首选环境：

项目

描述

最小GW

选择此项以使用MinGW。在文本字段中，指定到MinGW主目录的路径，或使用CLion检测到的路径。

西格温

选择此项以使用Cygwin。在文本字段中，指定Cygwin主页的路径，或使用CLion检测到的路径。

视觉工作室

选择此项以使用Microsoft Visual C ++编译器。

在文本字段中，指定Microsoft Visual Studio主页的路径（例如C：\ Program Files（x86）\ Microsoft Visual Studio 14 .0 ）。为此，请使用以下方法之一：

  • 键入所需目录的完整路径。

  • 点击按钮，然后从打开的对话框中选择路径。

在“ 架构” ，“ 平台 ”和“ 版本”字段中，指定架构（x86，amd64，x86_arm，amd64_arm等），平台类型（存储，uwp，onecore或将其留空）和版本（Windows SDK名称或将其保留）空白）。

WSL

选择此项以使用WSL工具集。从下拉列表中选择自动检测到的WSL分发路径。

凭证 -此字段表示将用于所选WSL分发的SSH凭证。点击按钮配置凭据，例如用户名，密码和端口。用户名和密码应与您在WSL设置过程中输入的名称和密码相同。

远程主机

选择此项进行“ 远程开发” 。在“ 凭据”字段中，提供远程主机凭据。
