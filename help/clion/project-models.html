项目格式

CLion包含一组智能功能，其中许多功能取决于项目级别的信息，例如包含的文件，编译器标志和链接器选项。这些详细信息使CLion能够正确执行解析/解析，从而以无错误且方便的方式突出显示，自动完成，检查和浏览代码。

CLion不为您的代码提供其自己的项目模型，但支持以下格式：

  • CMake的

  • Gradle （仅新的 Gradle C / C ++项目）

  • JSON编译数据库

此外，还有一个可用于CLion的第三方Bazel插件 。

您也可以在CLion中创建，打开和编辑非项目文件或文件夹。在这种情况下，CLion的功能将受到限制，尽管仍然足以阅读和初步研究代码。

另外，您可以设置自定义构建目标 ，这些目标调用外部工具来执行构建和清理，并为此类目标创建自定义运行/调试配置 。由于定制目标独立于任何特定的构建系统，因此它们可以例如为编译数据库项目提供构建和运行的功能。

要在CLion中使用Makefile项目 ，请结合使用编译数据库和文件监视程序来监视Makefile中的更改。您可以使用自定义构建目标和自定义运行/调试配置来扩展此类项目。

下表显示了每种项目格式如何支持项目级， 远程和动态分析功能（为简便起见，将编译数据库称为CompDB ）。该表不包含远程GDB / gdbserver ，因为它在CLion中的实现完全独立于任何项目模型。

对于Google Sanitizers ，某些项目格式带有黄色复选标记 。这意味着，即使您可以配置消毒剂并在CLion中获得结果，输出仍将以纯文本格式记录到控制台。消毒剂输出的可视化仅适用于CMake项目。另请注意，某些功能标记为支持可能无法立即使用，并且在开始使用它们之前，您需要执行其他配置步骤。
