CLion 2019.3帮助

连接到数据库

要向数据库发出查询，必须创建数据源连接。数据源是数据的位置，可以是服务器，CSV或DDL文件。数据源包括名称和连接设置，具体取决于数据源的类型。

亚马逊Redshift

  1. 在您的Redshift仪表板中，创建一个Redshift集群。

    有关Amazon Redshift集群的更多信息，请阅读Amazon Redshift入门。

  2. 在Redshift集群的设置中，复制JDBC URL 。

  3. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  4. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择Amazon Redshift 。

  5. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。或者，您可以为数据源指定用户驱动程序。有关用户驱动程序的更多信息，请参阅将用户驱动程序添加到现有连接 。

  6. 将JDBC URL从Redshift集群设置粘贴到CLion中的URL字段。

  7. 在用户和密码字段中，指定您的Redshift凭据。

  8. 要确保成功连接到数据源，请单击“ 测试连接” 。

阿帕奇德比

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择Apache Derby 。

  3. 从“ 驱动程序”列表中，选择要使用的JDBC驱动程序的类型。对于远程连接，请选择Apache Derby（远程） 。对于嵌入式数据库，选择“ Apache Derby（嵌入式）” 。

  4. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。或者，您可以为数据源指定用户驱动程序。有关用户驱动程序的更多信息，请参阅将用户驱动程序添加到现有连接 。

  5. 指定数据库连接详细信息。或者，将JDBC URL粘贴在URL字段中。

    要删除密码，请右键单击“ 密码”字段，然后选择“ 设置为空” 。

  6. 要确保成功连接到数据源，请单击“ 测试连接” 。

Azure SQL数据库

Microsoft Azure支持PostgreSQL，MySQL，MariaDB和其他数据库管理系统（DBMS）。如果要连接到这些数据源，请创建与这些DMBS对应的数据源连接。在CLion中，这些连接已经包含必需的JDBC驱动程序。因此，如果要连接到PostgreSQL的Azure数据库 ，请创建PostgreSQL连接 。

  1. 在您的Microsoft Azure帐户中，创建一个SQL数据源。有关Azure SQL数据库的更多信息，请阅读Azure SQL数据库文档。

  2. 在CLion中，导航到“ 文件|数据源 Ctrl+Shift+Alt+S 。

  3. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择“ Azure SQL数据库” 。

  4. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。

  5. 指定数据库连接详细信息。或者，将JDBC URL粘贴在URL字段中。

    要删除密码，请右键单击“ 密码”字段，然后选择“ 设置为空” 。

    若要使用Azure Active Directory身份验证 ，请选择“ 使用Windows域身份验证” 。若要使用SQL身份验证，请清除该复选框，然后在“ 用户”和“ 密码”字段中指定Microsoft Azure凭据。

    注意 ：“ 使用Windows域身份验证”复选框仅在Windows上可用。

  6. 要确保成功连接到数据源，请单击“ 测试连接” 。

MongoDB

在MongoDB中，要进行身份验证，您必须提供用户名，密码以及与此用户名关联的身份验证数据库。如果身份验证数据库与您要连接的数据库不同，请使用authSource URL中的参数。如果未指定身份验证数据库，则MongoDB将使用admin默认情况下是数据库。

例如，您要连接到local数据库，但您的身份验证数据库是admin 。在这种情况下，连接URL将如下所示： mongodb://localhost:27017/local?authSource=admin 。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择MongoDB 。

  3. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。或者，您可以为数据源指定用户驱动程序。有关用户驱动程序的更多信息，请参阅将用户驱动程序添加到现有连接 。

  4. 指定数据库连接详细信息：登录名，密码和要连接的数据库。要设置空密码，请右键单击“ 密码”字段，然后选择“ 设置为空” 。

  5. 要确保成功连接到数据源，请单击“ 测试连接” 。

    • 服务名称 ：使用Oracle实例（或多个实例）的别名。

  5. 指定数据库连接详细信息。或者，将JDBC URL粘贴在URL字段中。

    要删除密码，请右键单击“ 密码”字段，然后选择“ 设置为空” 。

    要删除密码，请右键单击“ 密码”字段，然后选择“ 设置为空” 。

  5. 要确保成功连接到数据源，请单击“ 测试连接” 。

其他

当前，CLion支持前面提到的供应商。该支持包括增强代码完成功能和更好地检索数据库对象（自省）等功能。如果您的供应商不在列表中，您仍然可以使用JDBC驱动程序连接到数据库。

使用JDBC驱动程序创建与数据库的连接

如果在数据源列表中找不到数据库供应商的名称，请下载数据库管理系统（DBMS）的JDBC驱动程序，然后在CLion中创建连接。使用JDBC驱动程序，您可以连接到DBMS并开始工作。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择驱动程序和数据源 。

  3. 单击用户驱动程序链接。

  4. 在驱动程序文件窗格中，单击添加图标，然后选择自定义JAR 。

  5. 导航到JDBC驱动程序的JAR文件，选择它，然后单击OK 。

  6. 在“ 类”字段中，指定要用于驱动程序的值。

  7. 点击应用 。

  8. 返回到创建的数据源连接。

  9. 指定数据库连接详细信息。或者，将JDBC URL粘贴在URL字段中。

    要删除密码，请右键单击“ 密码”字段，然后选择“ 设置为空” 。

  10. 要确保成功连接到数据源，请单击“ 测试连接” 。

DDL数据源

包含数据定义语言语句（DDL语句）的SQL文件可以用作数据源。因此，您可以在编辑器中引用此类文件中定义的所有表，列和其他对象。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择DDL数据源 。

  3. 在“ DDL文件”窗格中，单击“ 添加”按钮 。

  4. 在文件浏览器中，导航到SQL文件或带有DDL语句的文件，然后单击打开 。

  5. 点击应用 。

  6. 在“ SQL方言”对话框中，选择要用于添加的SQL文件的方言。

  7. 应用设置，然后单击“ 确定” 。

您可以使用包含DDL语句的SQL文件附加目录，并将其从“ 文件”工具窗口拖到“ 数据库”工具窗口。
