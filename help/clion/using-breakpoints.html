断点

断点是源代码标记，可让您在特定点挂起程序执行并检查其行为。

设置后，断点将保留在您的项目中，直到您将其明确删除为止（ 临时行断点除外）。如果具有断点的文件在外部进行了修改（例如，通过VCS更新或在外部编辑器中进行了更改），并且行号已更改，则断点将相应地移动。请注意，进行此类更改后，CLion必须正在运行，否则它们将被忽略。

行断点

可以在可执行代码行上设置行断点。在具有此类断点的行之前，线程执行被挂起，并且CLion在该线程的堆栈上显示堆栈帧。

当调试会话尚未开始时，所有行断点都以相同的方式标记：

在调试会话期间，CLion会检测断点状态并相应地更改标记：

  • GDB或LLDB调试器使用提供的调试符号成功解析了行断点。这样的断点可以在执行过程中命中：

  • 行断点无效，这意味着它不能被GDB或LLDB解决，也永远不会被击中。当断点位于可执行代码之外或缺少某些调试符号时，可能会发生这种情况。CLion可以准确地检测到这种情况并即时更新图标（例如，当您加载适当的调试符号时，状态将更改）。

设置行断点

  1. 将插入符号放在要设置断点的可执行代码行中。

  2. 单击该行旁边的左装订线或按Ctrl+F8 。

    要设置临时断点，请按Ctrl+Shift+Alt+F8 。选中后立即将其从您的项目中删除。

删除行断点

  • 单击左侧装订线中的断点图标。

为避免意外删除断点并丢失其参数，您可以选择通过以下方式删除断点：将其拖动到编辑器中，或单击鼠标中键： 在 设置/首选项对话框Ctrl+Alt+S转到构建，执行，部署|调试器，然后选择“ 拖动到编辑器”或使用鼠标中键单击 。单击断点将随后切换其状态（启用/禁用）。

异常断点

抛出指定的异常时会触发异常断点。它们全局适用于异常条件，并且不需要特定的源代码引用。

设置异常断点

  1. 按Ctrl+Shift+F8或选择运行|从主菜单查看断点 。

  2. 在“ 断点”对话框中，按Alt+Insert或点击 ，然后选择C / C ++异常断点，Python异常断点或JavaScript异常断点 。

  3. 在“ 输入异常类”对话框中，从库或项目中指定一个异常类。

删除异常断点

  1. 按Ctrl+Shift+F8或选择运行|从主菜单查看断点 。

  2. 在“ 断点”对话框中，选择要删除的断点，然后单击“确定”。 或按Alt+Delete 。

符号断点

当执行特定功能或方法时，符号断点将停止程序的执行。

创建一个符号断点

  1. 按Ctrl+Shift+F8或选择运行|从主菜单查看断点 。

  2. 在“ 断点”对话框中，按Alt+Insert或点击 ，然后选择符号断点 。

  3. 指定符号名称，然后选择是要在所有模块中还是仅在特定模块中命中此断点。

删除符号断点

  1. 按Ctrl+Shift+F8或选择运行|从主菜单查看断点 。

  2. 在“ 断点”对话框中，在“ 符号断点”下选择要删除的断点 ，然后单击“确定”。 或按N/A 。

断点属性

根据断点类型，您可以配置其他属性，例如：

  • 达到断点时要执行的操作

  • 挂起策略，用于定义在遇到断点时是否必须挂起应用程序

  • 对其他断点的依赖

  • 定义何时必须命中断点的条件

要编辑断点属性，请按Ctrl+Shift+F8 ，或在编辑器装订线中右键单击断点。

描述

已启用

清除以临时禁用断点而不将其从项目中删除。在调试过程中将跳过禁用的断点。

暂停

选择此选项可在遇到断点时暂停程序执行。如果您需要在不中断程序的情况下获取日志信息或在特定点计算表达式，则挂起应用程序很有用。如果您需要创建一个主断点，该主断点将在命中时触发相关断点，请选择不要在该断点处暂停程序。

选择暂停策略：

  • 全部 ：所有线程将被挂起

  • 线程 ：仅包含该断点的线程将被挂起。如果希望将“ 线程”策略用作默认策略，请单击“ 设为默认值”按钮。

如果将程序的策略设置为All ，则该程序不会在断点处挂起，并且在命中该断点时会执行一些步进操作。

条件

选择以指定达到断点的条件。条件是任何布尔表达式，其结果为true要么false 。

此表达式必须在设置断点的行上有效，并且每次击中断点时都会对其求值。如果评估结果是true ，执行选定的动作。在此字段中，您可以输入多行条件表达式。

对于异常断点，请使用this引用当前异常，例如this instanceof IncompatibleClassChangeError 。

日志记录

选择是否要将以下事件记录到控制台：

  • “断点命中”消息 ：命中断点时，控制台输出中将显示一条日志消息。

  • 堆栈跟踪 ：命中时，断点的堆栈跟踪将被打印到控制台。

    如果要在不中断程序执行的情况下检查导致此点的路径，这将很有用。

评估并记录

选择此选项可在命中断点时对表达式求值，并在控制台输出中显示结果。

一经删除

选择该选项后，将其从项目中删除即可。

禁用直到遇到断点

选择将触发当前断点的断点。在达到该断点之前，将禁用当前断点。您也可以选择是否要再次禁用它，或者一旦点击它就将其保持启用状态。

生产力提示

快速访问最常用的设置

右键单击编辑器装订线中的断点以快速禁用或挂起它，或设置条件。

断点意图

您可以通过意图操作来访问最常见的断点操作和过滤器（ Alt+Enter ）。

组断点

例如，如果需要为特定问题标记断点，则可以将断点分组。

在“ 断点”对话框中Ctrl+Shift+F8 ，选择要放置在组中的断点，然后选择“ 移至组| /从上下文菜单中新建 。

移动断点

要移动断点，请将其拖动到目标行。

复制断点

要复制断点，请按Ctrl并将其拖动到目标行。

禁用断点

要临时禁用断点而不将其从项目中删除，请按住Alt键，然后单击装订线中的断点图标。

快速搜索

如果项目中有很多断点，则可以向它们添加简短描述以轻松搜索它们：在“ 断点”对话框中右键单击一个断点Ctrl+Shift+F8并从上下文菜单中选择“ 编辑描述 ”。现在，当您开始输入断点名称时，它便成为焦点。

跳转到源

要从“ 断点”对话框跳转到设置了所选断点的代码行，请按F4 。
