节点解释器

在这个领域里， 指定要使用的Node.js解释器。 如果选择项目别名，CLion将自动从Node.js和NPM页面上的Node解释器字段中使用项目默认解释器 。

在大多数情况下，CLion会检测项目的默认解释器并填写字段本身。您还可以选择其他已配置的本地解释器或单击并配置一个新的。

打字稿

从这个清单中选择要使用的TypeScript版本（CLion显示当前选择的版本）。

  • 默认情况下，文件夹用于从项目的node_modules的打字稿包。

  • 捆绑：选择此选项可使用CLion捆绑的打字稿包，而无需尝试查找另一个。

  • 选择：选择此选项以使用自定义打字稿包，而不是与CLion捆绑在一起的包。在打开的对话框中选择相关软件包的路径。

TypeScript语言服务

根据最新规范，选中此复选框可从TypeScript语言服务获得本机支持。结果是：

  • 语法和错误突出显示基于TypeScript语言服务的注释。

  • 完成列表既包含来自TypeScript语言服务的建议，也包含由CLion计算的建议。

  • TypeScript代码被编译为JavaScript。

也适用于没有tsconfig.json的项目

选中此复选框后，TypeScript语言服务还将处理不包含tsconfig.json配置文件的项目。在这种情况下，默认范围是整个项目。

重新编译更改

  • 选中此复选框后，对TypeScript文件的任何更改都会使编译器“唤醒”。

  • 清除此复选框后，编译器将忽略对TypeScript文件的更改。要重新激活编译器，请打开TypeScript工具窗口 （“ 视图” |“工具窗口” |“ TypeScript” ），单击在工具栏上，然后从列表中选择编译范围：

    如果尚未打开TypeScript工具窗口，并且“ 视图”菜单中没有该窗口，请选择“ 帮助” |“帮助”。查找Action ，然后从列表中查找并启动TypeScript Compile All操作。

编译范围

从此列表中，选择在单击“ 编译”并在TypeScript工具窗口中选择“ 全部 编译”时编译器将在其中工作的范围。可用的选项有：

  • 所有地点：下面列出的所有范围。

  • 项目文件：项目根目录中的所有文件。

  • 临时和控制台：位于“ 项目”工具窗口中的临时和控制台目录中的所有文件。

  • 打开文件：编辑器中当前打开的所有文件。

  • 当前文件：在活动编辑器选项卡中打开的文件。

或者，单击并在打开的“ 范围”对话框中配置自定义范围 。

有关范围的更多详细信息，请参见“ 范围对话框” 。

选件

在这个领域里， 指定在找不到tsconfig.json文件时传递给TypeScript语言服务的命令行选项。请参阅TSC参数上可接受的选项列表。请注意-w要么--watch （ 观看输入文件 ）选项无关紧要。

您可以使用TypeScript语言服务的建议来增强JavaScript文件中的完成功能。为此，添加'allowJS' : true到您的jsconfig.json或tsconfig.json文件。从代码完成和配置JavaScript库中了解更多信息。

默认情况下，“ TypeScript语言服务”复选框处于选中状态。
