CLion 2019.3帮助

上传和下载文件

CLion提供了以下将项目文件和文件夹上载到部署服务器的主要方法：

  • 手动 ，随时通过菜单命令。

  • 自动 ，每次一个文件被更新，或启动调试会话之前，或者在提交给你的版本控制系统。

对于下载文件和文件夹，CLion仅支持手动模式。

CLion在“ 文件传输”工具窗口（“ 视图” |“工具窗口” |“文件传输” ）中显示日志。

手动上传文件或文件夹

  • 在“ 项目”工具窗口中，右键单击文件或文件夹，然后选择“ 部署” |“文件” 。从上下文菜单上载到 ，然后从列表中选择目标部署服务器或服务器组。

    如果指定了默认服务器或服务器组，则还可以选择上载到 。

在远程会话之前上传文件

  1. 创建与远程工具链关联的运行/调试配置。

  2. 在启动前区域中，单击然后从列表中选择“将文件上传到远程主机 ”。

    如果该区域是折叠的，则单击扩大它。

  3. 在打开的“ 上载到远程主机”对话框中：

    • 从服务器列表中，选择目标部署服务器或服务器组。

    • 在“ 本地要上载的根目录”区域中，创建要上载的根目录列表。指定根目录下的所有文件夹和文件将递归上传。

上载本地更改的文件

  1. 切换到 的“本地更改”标签 版本控制窗口Alt+9查看本地更改的文件。

  2. 右键单击文件，然后选择“ 部署” |“文件” 。从上下文菜单上载到 ，然后从列表中选择目标部署服务器或服务器组。如果指定了默认服务器或服务器组组，则还可以选择上载到 。

有关详细信息，请参见在版本控制下管理文件 。

与VCS储存库同步后上传文件

  1. 通过按来将本地文件的内容与VCS存储库同步Ctrl+T或选择VCS | |从主菜单更新 。

  2. 切换到更新信息的标签 版本控制窗口Alt+9 。

  3. 右键单击文件，然后选择“ 部署” |“文件” 。从上下文菜单上载到 ，然后从列表中选择目标部署服务器或服务器组。如果指定了默认服务器或服务器组，则还可以选择上载到 。

提交后立即上传签入的文件

  1. 开始检查您的更改。

  2. 在“ 提交后”区域中，从“将文件上传到”列表中选择目标服务器或服务器组。选择现有配置之一或创建新配置：单击并在打开的“ 部署”对话框中配置对相关服务器的访问权限或设置服务器组 。

  3. 若要将来自动应用您的选择，请选中“ 始终使用选定的服务器或服务器组”复选框。

配置将更改后的文件自动上传到默认服务器或服务器组

CLion认为本地文件在自动或手动保存后立即更改（ File | Save All或Ctrl+S ），请参阅使用源代码 。更改的文件只能自动上载到默认部署服务器 。

  1. 通过执行以下操作之一打开“ 选项”对话框：

    • 在主菜单中，选择“ 工具” |“工具”。部署|选项 。

    • 在 设置/首选项对话框Ctrl+Alt+S ，转到“ 构建，执行，部署” |部署|选项 。

  2. 从“将更改的文件自动上传到默认服务器”列表中，选择您希望CLion何时上传更改的文件：

    • 要上传任何手动或自动保存的文件，请选择始终 。

    • 要仅上传手动保存的文件，请选择进行明确保存操作 。

    • 要禁止自动上传，请选择从不 。

下载文件或文件夹

  • 在“ 远程主机”工具窗口中，选择所需的文件或文件夹，然后从所选内容的上下文菜单中选择“ 从此处下载 ”。

从默认部署服务器下载文件

  • 在主菜单中，选择“ 工具” |“工具”。部署|从下载 。

    CLion将提示您覆盖本地文件（如果有）。
