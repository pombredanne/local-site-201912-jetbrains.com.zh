Google测试支持

在我们的单元测试教程的Google测试部分中了解框架的详细信息。

将Google Test添加到您的项目

  1. 从官方存储库下载Google Test，然后将googletest-master的内容提取到项目中的空文件夹中（例如， Google_tests / lib ）。

    或者，将Google Test克隆为git子模块，或使用CMake下载它（以下说明在后一种情况下将不适用）。

  2. 在Google_tests文件夹中创建一个CMakeLists.txt文件：在项目树中右键单击它，然后选择New | CMakeLists.txt 。

    自定义以下行并将其添加到脚本中：

    ＃'Google_test'是子项目名称project（Google_tests）＃'lib'是具有Google Test源的文件夹add_subdirectory（lib）include_directories（$ {gtest_SOURCE_DIR} / include $ {gtest_SOURCE_DIR}）＃'Google_Tests_run'是目标名称＃' test1.cpp tests2.cpp'是具有测试add_executable（Google_Tests_run test1.cpp tests2.cpp）的源文件target_link_libraries（Google_Tests_run gtest gtest_main）

  3. 在您的根 CMakeLists.txt脚本中，添加add_subdirectory(Google_tests)命令结束，然后重新加载项目。

  4. 编写测试时，请确保添加#include "gtest/gtest.h"在每个带有测试代码的.cpp文件的开头。

Google Test运行/调试配置

尽管Google测试提供了main()条目，并且您可以将测试作为常规应用程序运行，我们建议您使用专用的Google Test运行/调试配置。它包括与测试相关的设置，使您可以从内置的测试运行器中受益，如果您以常规程序运行测试，则无法使用该功能。

探索结果

运行测试时，结果（和过程）显示在“测试运行器”窗口中。该窗口包括：

  • 进度条 ，显示到目前为止已执行的测试百分比，

  • 所有正在运行的测试及其状态和持续时间的树状视图 ，

  • 测试的输出流，

  • 带有重新运行选项的工具栏失败测试，出口或打开自动保存的先前结果 ，按字母顺序对测试进行排序轻松找到特定的测试，或按持续时间对其进行排序了解哪个测试比其他测试运行时间更长。
