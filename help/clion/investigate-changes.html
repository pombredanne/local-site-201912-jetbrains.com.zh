调查变化

在CLion中，您可以追溯项目中的所有更改。这可以帮助您找到任何更改的作者 ，查看文件版本或提交之间的差异，并在必要时安全地回滚和撤消更改。

回顾项目历史

您可以查看对与指定过滤器匹配的项目源所做的所有更改。要查看项目历史记录，请打开 的日志标签 版本控制窗口Alt+9 。它显示了提交给所有分支机构和远程存储库的所有更改：

在多存储库项目中，左侧的彩色条纹指示所选提交所属的根（每个根都用自己的颜色标记）。将鼠标指针悬停在彩色条纹上以调用显示根路径的提示：

浏览和搜索项目历史记录

  • 通过输入完整的提交名称或消息或其片段，修订号或正则表达式来搜索提交列表。

  • 按分支或收藏夹分支 ，用户，日期和文件夹（或多根项目的根目录和文件夹）过滤提交。

  • 点击转到哈希/分支/标签 工具栏上的图标或按Ctrl+F并指定提交哈希， 标记或您要跳转到的分支的名称（您将被带到该分支中的最新提交）。

  • 单击箭头跳到长分支中的下一个提交：

  • 按Left和Right跳转到父/子提交的键。如果您要提交不同的存储库，并且在“ 日志”视图中混合了多个分支，则此功能特别有用。

  • 通过将焦点切换到搜索字段Ctrl+L 。

  • 为避免前后设置过滤器，请单击在工具栏上打开一个与您的过滤器匹配的新标签。

  • 要自定义日期格式，请转至设置/首选项|外观和行为系统设置|日期格式 。

有关项目“ 日志”视图的更多详细信息，请参见“ 日志”选项卡 。

查看特定版本的项目快照

CLion使您可以查看选定修订版本的项目状态。

  1. 打开 版本控制窗口Alt+9并切换到“ 存储库”选项卡。

  2. 选择一个提交，然后从上下文菜单中选择“ 修订时浏览版本库” 。

“ 存储库”工具窗口将打开，其中包含所选版本的项目快照。

查看两次提交之间的差异

CLion允许您检查两次提交之间修改了哪些文件，而不必浏览两次提交之间的更改。

  • 在“ 日志”视图中选择任意两个提交，然后从上下文菜单中选择“ 比较版本 ”。

    将打开在选定提交之间修改的文件的列表。您可以通过以下方式查看任何文件的差异或按Ctrl+D 。

查看文件历史记录

您可以查看对特定文件所做的所有更改，并找到每个修订版中所做的确切修改。

  1. 在任何视图中选择所需的文件（在项目工具窗口工具窗口中的编辑器中，在 的“本地更改”标签 版本控制窗口Alt+9 ， 等等）。

  2. 选择Git |从VCS主菜单或所选内容的上下文菜单显示历史记录 。历史记录选项卡已添加到 “版本控制”窗口显示所选文件的历史记录，并允许您查看和比较其修订版本。

  3. 要确定特定修订中引入了哪些更改，请在列表中选择它。在面板的右侧，差异立即显示。

  4. 要在专用差异查看器中查看整个文件的差异， 在列表中选择它，然后按Ctrl+D或点击工具栏上的按钮。差异查看器将打开，显示此修订中已更改的内容。

查看目录的历史记录

除了查看整个项目或特定文件的历史记录之外，您还可以检查在特定文件夹中进行了哪些更改。

  1. 在“项目”工具窗口中选择一个目录或多个目录，然后选择“ Git”。从上下文菜单中显示历史记录 。

  2. 新标签页已添加到 版本控制窗口Alt+9显示由选定文件夹过滤的提交。

查看本地文件版本与提交文件版本之间的差异

您可以检查提交的文件修订与其本地版本有何不同：

  1. 打开 版本控制窗口Alt+9并切换到“ 存储库”选项卡。

  2. 选择您感兴趣的提交，然后在右窗格中选择文件。

  3. 点击工具栏上的按钮。

审查更改如何合并

克利翁允许您查看如何变化合并从一个分支到另一个 ，以及如何准确冲突（如果有的话）是解决在合并过程中：

  • 在“ 日志”视图中，选择您感兴趣的合并提交。

    • 如果在合并过程中未检测到冲突并解决了冲突，CLion将在“ 更改的文件”窗格中显示相应的消息，并建议您查看源于两个父级的更改：

      从节点之一中选择所需的文件，然后单击“ 显示差异” 工具栏上的图标或按Ctrl+D 。差异查看器将显示两个面板的差异，使您可以将当前版本与选定的父版本进行比较。

    • 如果在合并过程中发生冲突，“ 更改的文件”窗格将向您显示合并了冲突的文件的列表。

      选择所需的文件，然后单击显示差异 工具栏上的图标或按Ctrl+D 。差异查看器将显示一个三面板差异，使您可以将当前版本与其父版本进行比较，并查看如何准确解决冲突。

找到代码作者（注释/责备）

您可以使用VCS注释 （对应于git-blame ）来找出谁对文件进行了哪些更改。带注释的视图显示每行代码的详细信息：

在当前版本中修改的行的注释以粗体和星号标记。

您可以从注释视图跳转：

  • 到Log中的相应提交：使用Select in Git log context-menu命令

  • 到https://github.com上的相应提交：使用“ 在GitHub上打开”上下文菜单命令

启用注释

  • 在编辑器或“ 差异查看器”中右键单击装订线，然后从上下文菜单中选择“ 注释 ”。

    您可以为“ 注释”命令分配自定义快捷方式： 在 设置/首选项对话框Ctrl+Alt+S选择左侧的Keymap ，然后转到“ 版本控制| Git |注释 。

配置注释中显示的信息量

您可以选择要在注释视图中查看的信息量。

  • 右键单击注释装订线，选择“ 查看”，然后选择要查看的信息类型，包括此更改的发起版本，日期，不同格式的作者姓名以及提交编号。

    您还可以在颜色下设置突出显示。

配置注释选项

  • 右键单击注释装订线，然后从上下文菜单中选择“ 选项 ”：

    • 忽略空格 ：空格将被忽略（git blame -w ）。这意味着注释将指向先前的有意义的提交。

    • 检测文件内的移动：当提交在同一文件内移动或复制行时，此类更改将被忽略（git blame -M ）。这意味着注释将指向先前的有意义的提交。

    • 检测跨文件移动：当一次提交移动或从同一提交中修改的其他文件复制行时，此类更改将被忽略（git blame -C ）。这意味着注释将指向先前的有意义的提交。

自定义日期格式

  1. 在 设置/首选项对话框Ctrl+Alt+S前往外观与行为|系统设置|日期格式 。

  2. 单击“ VCS批注”旁边的“ 日期时间模式”字段，然后指定要用于VCS批注的日期格式。请参阅模式参考 。

注释先前的版本

CLion使您不仅可以注释当前文件版本，还可以注释其先前版本。注释装订线的上下文菜单中提供以下选项：

  • 批注修订 ：如果要在提交特定更改后检查文件的外观，此选项很有用。

  • 注释以前的修订版 ：如果您发现自己在特定行中的最后更改无意义的情况下（例如，如果所有更改都是代码格式的），则此选项很有用。在这种情况下，您可以检查文件的先前版本。

您还可以从“ 历史记录”视图中注释特定文件。在“ 历史记录”选项卡中，选择要查看的文件版本，右键单击相应的行，然后从上下文菜单中选择“ 注释 ”。
