Valgrind Memcheck

Valgrind Memcheck是一个工具，可以帮助识别C和C ++代码中的内存错误。它可以检测各种问题，例如内存泄漏，无效的内存访问，不正确的内存释放以及使用未定义的值。

CLion中的Valgrind Memcheck可通过WSL在多个平台上运行：Linux，macOS和Windows（请参阅WSL上的Valgrind ）。

配置Valgrind

  1. 在您的系统上安装 Valgrind。成功安装后，CLion通过搜索系统中定义的标准位置来自动检测Valgrind二进制文件PATH变量。

    对于WSL，您需要在WSL实例上安装Valgrind（ sudo apt-get install valgrind ），并为CLion提供可执行文件的路径（请参阅下一步）。

    如果Valgrind位于非标准位置，则可以将CLion明确指向它：转到“设置” /“首选项” |“设置”。构建，执行，部署|动态分析工具| Valgrind并在Valgrind可执行文件字段中指定路径。

  2. 在同一对话框中，指定所需的分析选项 。

  3. （可选）配置禁止文件列表。例如，您可能想添加一个非默认的抑制文件以关闭某些特定库的Valgrind Memcheck检查。例如，此处的典型情况是禁止使用OpenSSL或Boost。

在WSL上配置Valgrind

要通过WSL在Windows上使用Valgrind，请提供Valgrind的“子系统”路径，默认情况下为/ usr / bin / valgrind ，而不是Valgrind二进制文件的实际Windows位置。但是，在选择连接到WSL工具链的CMake概要文件之前，此路径将无效。为此，请执行以下一项操作：

  • 将WSL工具链设置为默认值。这样，它将自动连接到默认的CMake配置文件：

  • 创建一个单独的CMake配置文件，将其连接到WSL工具链，然后在配置切换器中选择此配置文件：

运行目标

要使用Valgrind Memcheck运行目标（应用程序或测试），请选择所需的“运行/调试”配置，然后执行以下操作之一：

  • 在主菜单上，选择“运行”。使用Valgrind Memcheck选项运行“ your_target_name” 。

  • 单击导航栏中的专用图标：

  • 点击装订线图标，然后从菜单中选择使用Valgrind Memcheck选项运行'your_target_name' ：

探索结果

  • “ 运行”工具窗口中 Valgrind选项卡的左侧窗格代表Valgrind Memcheck发现的所有问题的信息。每个堆栈帧都显示指令指针，有问题的函数名称以及（如果可能）问题所在的源文件：

  • 选择感兴趣的堆栈跟踪。在右侧窗格中，您可以查看目标堆栈跟踪的完整堆栈框架描述以及相应的代码预览（如果可能）。按F4跳回到源代码。

  • 右键单击所选项目以将其从当前结果日志中排除，或者通过向抑制文件添加相应的记录来抑制有问题的功能。

  • 要导入结果，请选择运行|。从主菜单导入Valgrind XML结果 ，然后选择要从中导入的XML文件。
