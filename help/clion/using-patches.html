使用补丁

无需提交本地更改，您可以将它们放在.patch文件中，然后可以将其应用于源，通过电子邮件发送等。使用补丁程序是共享更改而不将其签入VCS存储库的便捷机制。

根据未提交的更改创建补丁

  1. 打开 版本控制窗口Alt+9并切换到“ 本地更改”标签。

  2. 选择要基于其创建补丁的文件或更改列表，然后从上下文菜单中选择“ 从本地更改创建补丁” 。

    您还可以调用“提交更改”对话框 ，选择要包含在补丁中的更改，单击“ 提交”按钮旁边的箭头，然后选择“ 创建补丁” 。

  3. 在打开的对话框中，确保已选择要包含在补丁中的所有更改，输入提交注释（可选），然后单击创建补丁 。

  4. 在“ 补丁文件设置”对话框中，根据需要修改默认补丁文件位置，然后单击“ 确定” 。

如果您不需要将修补程序保存到文件中，并且想要例如通过电子邮件或通过Messenger发送，则可以在菜单栏中右键单击文件。 的“本地更改”标签 版本控制窗口Alt+9并从上下文菜单中选择“ 复制为修补程序到剪贴板 ”。

从整个提交创建补丁

  1. 在“ 日志”视图中找到要创建补丁的提交，然后从上下文菜单中选择“ 创建补丁 ”。

  2. 在“ 补丁文件设置”对话框中，根据需要修改默认补丁文件位置，然后单击“ 确定” 。

从文件创建补丁

  1. 在任何视图中选择所需的文件（在项目工具窗口工具窗口中的编辑器中，在 的“本地更改”标签 版本控制窗口Alt+9 ， 等等）。

  2. 选择Git |从VCS主菜单或所选内容的上下文菜单显示历史记录 。历史记录选项卡已添加到 “版本控制”窗口显示所选文件的历史记录，并允许您查看和比较其修订版本。

  3. 右键单击修订，然后从上下文菜单中选择“ 创建补丁 ”，或单击“ 创建补丁”图标在工具栏上。

套用补丁

  1. 选择VCS |从主菜单应用补丁 。

  2. 在打开的“ 应用补丁”对话框中，指定要应用的.patch文件的路径。

    您可以将文件或电子邮件附件拖到编辑器中的任何位置。

  3. 如有必要，请单击并选择“ 映射基础目录”以指定一个目录，该目录将相对于补丁文件中的文件名进行解释。您可以将基本目录映射到单个文件，目录或选择。

  4. 如果在创建补丁后对源代码进行了编辑，则可能会发生冲突。要检查是否可以成功应用补丁，请单击“ 显示差异”。 Ctrl+D 。如果存在冲突，相应的行将以红色突出显示。

  5. 如果要将更改应用于与补丁程序中指定的文件不同的位置中存储的文件，可以通过单击以删除开头的目录并选择“ 删除所有主要目录” 。

  6. 从列表中选择要在其中添加补丁的现有更改列表，或者在“ 名称”字段中指定新更改列表的名称 ，然后（可选）输入对此更改列表的注释。

  7. 如果要使此更改列表处于活动状态，请选择“ 设置为活动”选项。

  8. 如果您希望CLion在停用时保留与新变更列表关联的任务的上下文，并在变更列表变为活动状态时还原该上下文，请选择“ 跟踪上下文”选项（有关详细信息，请参阅任务和上下文 ）。

  9. 如果要在应用修补程序之前将修补程序移到临时存储（机架），请单击导入到机架 （有关详细信息，请参见“ 搁置和取消搁置更改 ”）。否则，请单击确定 。

您还可以通过选择VCS |复制补丁文件内容并应用它。从主菜单从剪贴板应用修补程序 。例如，当您通过电子邮件收到补丁程序并且不想保存它时，这很方便。对于Git格式的补丁程序，CLion提取提交消息和作者，并自动填充“ 提交更改”对话框中的相应字段。
