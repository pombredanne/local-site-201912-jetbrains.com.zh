GDB远程调试

本地操作系统 ：macOS / Linux / Windows

远程操作系统 ：任何支持的gdbserver

所需的工具 ： gdbserver的上tagret

二进制和符号文件同步 ：手动

客户端调试器 ：捆绑的多体系结构GDB / WSL GDB / Cygwin GDB /自定义GDB

通过GDB远程调试配置，您可以运行由任何构建系统构建的应用程序。唯一的要求是调试符号应可用于二进制文件和库。

目标环境也没有限制 ：您的程序可以在任何操作系统（包括基于Linux的嵌入式系统，例如Raspbian OS ），云平台上，或例如在Docker容器内的gdbserver下运行。您可以连接到任何符合远程gdbserver协议的GDB存根 -例如，使用Qemu调试OS内核或使用OpenOCD调试闪存固件。

CLion捆绑的GDB在默认情况下用作客户端调试器，它具有多体系结构支持，因此适用于各种Linux / Windows / macOS和嵌入式情况下的远程跨平台调试。您可以在下面找到所有受支持目标的完整列表：

捆绑的GDB支持的远程目标

创建一个GDB远程调试配置

去运行|编辑配置 ，单击 ，然后从模板列表中选择GDB远程调试 。

指定以下设置：

  • “目标远程”参数

    在这里，您需要提供媒介来携带调试数据包（串行线路或使用TCP或UDP的IP网络）。有关连接到远程目标的更多详细信息，请参见gdb文档 。

  • 符号文件

    在主机上运行的GDB需要访问目标上运行的可执行文件的调试信息。为此，它要求主机上的符号文件是目标上可执行文件的非剥离副本。但是，将此字段保留为空也可能效果很好，因为最新版本的GDB可以自动从GDB服务器传输符号。

  • Sysroot

    在此字段中，指定包含符号文件和相应子目录中目标库副本的本地目录。

  • 路径映射

    使用此窗格可提供目标计算机上的路径（“ 远程”列），以将其映射到主机上的本地路径（“ 本地”列）。

启动远程调试会话

  1. 首先，在gdbserver下的远程目标上启动应用程序 。为此，您可以使用内置的SSH终端（使用适当的凭据运行“ 工具” |“启动SSH会话 ”）：

    gdbserver在入口点挂起程序，并等待调试器连接到它。

  2. 然后切换到新创建的远程配置并开始调试会议。CLion的调试器连接到远程进程，并使用提供的调试信息使您像在本地运行代码一样检查代码（例如，设置断点，逐步执行并检查变量）：

    通常，您的程序应在IDE设置的断点处停止。如果没有，请仔细检查是否在运行/调试配置设置中正确指定了符号文件和路径映射 。

调试共享库

  • 要调试共享库，您需要在〜/ .gdbinit脚本中设置solib-search-paths变量。但是，默认情况下，此命令将在连接到远程目标之前在调试器启动时执行（请参阅相应的问题 ）。解决方法是，可以使用GDB挂钩 ：

    定义目标hookpost-remote set solib-search-path /path/to/my.so:/path/to/sysroot:/path/to/vendorlibs break main＃如果您还需要调试会话在开始时暂停

    这样，GDB将执行set solib-search-path每次连接远程目标时在挂钩中指定。
