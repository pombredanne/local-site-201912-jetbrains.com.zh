附加到流程

“ 附加到进程”功能使您可以调试在CLion中开发的项目，但是（由于某些原因）不能直接从IDE启动。

对于C / C ++应用程序，您可以使用Windows上的GDB，macOS上的LLDB以及Linux上的GDB和LLDB附加到进程。如果适用，还可以使用其他语言的调试器（例如Python调试器或Swift LLDB）。

Ubuntu用户的先决条件

如果在Ubuntu（或可能在其他Linux发行版）上使用CLion，则在第一次尝试附加到进程时可能会出现错误消息ptrace：不允许的操作 。在这种情况下，请执行以下任一操作：

  • 要暂时禁用此限制，请输入命令：

    回声0 |须藤tee / proc / sys / kernel / yama / ptrace_scope

  • 要永久禁用此限制，请打开/etc/sysctl.d/10-ptrace.conf文件进行编辑并更改该行kernel.yama.ptrace_scope = 1至kernel.yama.ptrace_scope = 0 。
    要立即应用更改，请输入以上命令echo 0 | sudo tee /proc/sys/kernel/yama/ptrace_scope 。或者，运行sudo service procps restart或重新启动系统。

您可以在此处找到更多详细信息。

附加到流程

  1. 从主菜单中，选择运行|附加到处理或按Ctrl+Alt+F5 。

  2. 从可用进程列表中，选择所需的进程。

    通过键入名称或PID的前几个字母来简化搜索。如果有几个调试器可用于选定的进程，则可以在弹出菜单中选择它们：
