CMake项目的工具链

对于CLion中的CMake项目，工具链是构建和运行应用程序所需的所有必要工具的集合：工作环境，CMake可执行文件， make和编译器以及调试器。

您始终可以为一个项目拥有多个工具链，并在需要时在它们之间轻松切换。

当您开始使用CLion时，您已经有了默认的工具链。尽管可以在开发中使用它，但您可能还需要根据项目需要调整工具集（例如，更改工作环境或切换到其他编译器）。

创建一个新的工具链

  1. 转到设置/首选项|构建，执行，部署|工具链并单击添加一个新的工具链。

  2. 在Windows上，选择您的工作环境 ：MinGW，Cygwin，Visual Studio， WSL或远程主机 。有关如何在Windows上配置CLion的更多详细信息，请参阅我们的教程 。

  3. 在Linux或macOS上，选择工具链的类型-用于本地项目的系统或用于完全远程模式的 远程主机 。

  4. 选择首选的CMake安装，可以选择捆绑版本还是自定义版本。

  5. 您在上一步中设置的CMake可执行文件尝试检测make实用程序和C / C ++编译器。这些安装不是默认安装，但CMake会根据当前系统上安装的所有软件包进行检测。

    如果您更喜欢使用自定义的make和编译器而不是检测到的编译器 ， 请使用Make ， C Compiler和C ++ Compiler字段提供路径。

  6. 最后，指定Debugger 。在Windows上，可以在捆绑的GDB或自定义的GDB中进行选择。在macOS和Linux上，CLion还包括LLDB调试器（macOS的默认设置）。请参阅当前支持的版本 。

  7. CLion会验证您的选择，并在缺少任何工具或软件包时通知您。

  8. 配置完成后，单击“ 确定”保存。现在，在创建CMake配置文件时，您还有一个工具链可以使用。
