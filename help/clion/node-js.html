节点解释器

在这个领域里， 指定当前项目的默认Node.js解释器。每次您选择CLion时，CLion都会自动使用它Project创建运行/调试配置或配置依赖于Node.js的工具（例如Prettier或ESLint）时， Node Interpreter列表中的别名。

从列表中选择配置的解释器，或单击然后在打开的对话框中配置一个新对话框，如配置本地Node.js解释器中所述 。如果选择node ，则使用系统Node.js版本 。

版

此只读字段显示Node.js的当前版本。

包装经理

在这个领域里， 选择当前项目的程序包管理器（npm或Yarn）。

CLion使用npm和yarn这些管理器的当前系统路径的别名。要使用自定义安装，请单击“ 选择” ，然后选择相关软件包管理器的安装文件夹，有关详细信息，请参阅为项目配置软件包管理器 。

默认情况下，CLion建议使用npm。但是，如果您打开带有yarn.lock文件的项目，并且计算机上安装了Yarn，则CLion会自动将此项目的程序包管理器更改为Yarn。您还可以将Yarn设置为默认的CLion程序包管理器 。

配套

软件包区域显示计算机上当前安装的所有NPM相关软件包的列表。

  • 软件包：此只读字段显示软件包的名称，与在命令行模式下安装软件包时应引用的名称完全相同。

  • 版本：此只读字段显示计算机上安装的软件包的版本。

  • 最新：此只读字段显示软件包的最新发行版本。如果软件包不是最新的，则标记为 。
