快速教程：在macOS上配置CLion

本教程将指导您完成在macOS上安装，更新和初始配置CLion的过程。

安装程序

在macOS上开始CLion安装之前，请确保您的计算机满足硬件要求 ，并且macOS的版本为10.9.4+。

您始终可以在同一操作系统上安装多个CLion实例，包括发行版和EAP版本。

手动安装和补丁更新

  1. 下载适用于macOS的CLion的最新版本或EAP版本（早期版本可在“ CLion早期版本”页面上找到）。

  2. 打开下载的CLion-*。dmg软件包，然后将CLion拖到Applications文件夹中。

  3. 打开“ 应用程序”文件夹，然后启动CLion安装程序。

  4. 如果出现安全警告，请同意打开应用程序：

  5. 手动安装CLion时，默认情况下会自动执行更新检查（每次IDE准备更新时， 状态栏中都会显示一条消息）。要配置更新过程，请转到首选项|外观和行为系统设置|更新并选择更新频道：

    单击立即检查以立即执行检查。如果根据所选频道有可用的更新，您将看到如下对话框：

或者，您可以通过Homebrew软件包管理器安装CLion： brew cask install clion 。但是，此选项是非官方的，因为CLion团队不参与其维护或支持。

通过工具箱安装和更新

工具箱是一个控制面板，使您可以从单个访问点管理所有JetBrains开发人员工具，包括CLion以及您的项目。它使您能够维护不同版本的CLion，安装更新并在需要时回滚它们。工具箱会记住您的JetBrains帐户，并在安装和注册新工具时使用它自动登录。

  1. 下载工具箱并启动安装文件。

  2. 安装完成后，接受JetBrains隐私策略并登录到您的JetBrains帐户。

  3. 现在，您可以选择要安装的CLion版本：

  4. 工具箱显示已安装版本的列表：

必备工具

CLion需要与C和C ++编译器以及make实用程序一起提供。这些工具可能已预先安装在您的系统上：在“首选项” |“检查”中检查它。 构建，执行，部署|工具链 -编译器和进行检测应成功执行。

如果您的系统没有有效的编译器和make安装，则最简单的解决方案是安装Xcode命令行开发人员工具 。

  1. 运行以下命令：

    xcode-select-安装

  2. 当提示您安装命令行开发人员工具时，单击“ 安装”按钮：

    您也可以选择安装Xcode的完整软件包，尽管对于CLion而言不是必需的。

使用Xcode命令行工具，默认情况下会安装Clang编译器。要检查编译器的存在及其版本，请运行clang --version 。

命令行工具可能不会随系统或Xcode更新而自动更新。这可能会导致错误消息，例如在CLion中加载项目期间无效的活动开发人员路径 。要解决此问题，请运行相同xcode-select --install命令，工具将相应更新。

请注意，您可以根据项目需要使用多个编译器，请参见切换编译器 。

配置工具链

现在，您需要配置要使用的工具链，这意味着选择CMake可执行文件， make和C / C ++编译器的位置以及调试器。导航到首选项|构建，执行，部署|工具链并编辑默认工具链，或单击添加一个新的。

CMake，make和编译器

在CMake字段中，指定要使用的CMake二进制文件。您可以坚持使用捆绑的CMake，也可以使用自定义的CMake可执行文件（请参阅“ 软件要求”中的最低支持版本）。

所选择的CMake的尝试检测编译器和make考虑在系统上安装的软件包。如果检测成功，将自动填充Make ， C Compiler和C ++ Compiler字段：

如果CMake找不到合适的工具，则检测到编译器和make会失败（例如，如果没有Xcode开发人员工具而单独安装它们，则可能会发生这种情况）。在这种情况下，您需要手动提供实际路径。

调试器

适用于macOS的CLion随附了捆绑的LLDB v 9.0（默认调试器）和GDB v 8.0.1。您还可以切换到自定义GDB（支持的版本为7.8.x-8.1.x）。在“首选项” |“选择”中选择调试器。 构建，执行，部署|工具链 ：

在macOS上同时使用捆绑GDB或自定义GDB时，可能会出现问题。要改善此行为，请在注册表中启用cidr.debugger.gdb.workaround.macOS.startupWithShell选项（从主菜单中选择“ 帮助” |“查找操作” ，然后键入Registry ）：

GDB和Clang的组合暗示了在macOS上调试项目的某些限制。有关详细信息和解决方法，请参见调试：macOS特定调整 。

请注意，启用此选项并不等同于设置set startup-with-shell off在您的.gdbinit脚本中。
