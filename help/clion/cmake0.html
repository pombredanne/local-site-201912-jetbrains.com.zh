编辑时自动重新加载CMake项目

如果选中此复选框，则CLion在编辑CMake文件时会自动重新加载项目。

如果未选中此复选框，则在编辑CMake文件后，会显示一条横幅，您可以在其中选择手动重新加载当前项目，或启用自动重新加载，从而导致选中此复选框：

但是，如果进行了外部更改（例如，版本控制更新），则CMake项目仍然会重新加载。

个人资料

个人资料清单

在此字段中，选择要生成的CMake配置文件 。您可以编辑或删除现有配置文件或创建一个新的配置文件。

  • 点击按钮将配置文件添加到列表中。默认情况下，配置文件是根据当前选定的构建类型和工具链命名的。

  • 点击按钮从列表中删除选定的配置文件。

  • 点击按钮创建所选配置文件的副本。

请注意 ，CMake配置文件列表确定“ 运行/调试配置”对话框的配置文件列表和解析上下文列表。

名称

该字段代表所选配置文件的名称。

构建类型

从此下拉列表中选择所需的构建类型 。默认类型为Debug 。

工具链

从此下拉列表中，选择所需的工具链。该列表由您在“ 工具链设置”对话框中创建的工具链组成 。如果未选择任何内容，则使用默认 工具链 。

CMake选项

在此文本字段中，指定要传递给CMake命令的其他CMake选项 。选项应以空格分隔。
您也可以按Shift+Enter或点击在编辑器窗口中键入所需的选项，每个选项都换行。

环境

请点击调用“ 自定义变量”窗格并在此处选择要传递给CMake的“生成”阶段的环境变量。

  • 使用 ， ， 和按钮来管理自定义变量列表。

  • 使用显示链接查看系统环境变量及其值的列表。

产生路径

在此处指定生成的CMake文件的位置。这可以是绝对的（例如， c：\ CLion Projects \ My Project \ CMake Files \ cmake-build-debug，也可以是相对于当前项目根目录的路径（例如， cmake-build-debug ）：

  • 在此文本字段中，输入路径或单击浏览按钮以从“选择路径”对话框中选择所需的路径。

  • 不执行任何操作以将默认位置用于生成的文件。

此设置保存在.idea / workspace.xml中 ，不应共享。

注意：如果选择项目根目录的子目录作为生成输出，则IDE将该子目录的文件标记为从项目中排除，并将其视为非项目文件。诸如“ 导航到文件”之类的导航操作不适用于这些文件。

构建选项

在此文本字段中，指定要传递给CMake使用的构建工具的选项。在CMake文档中找到有关可用构建选项的更多信息。

您也可以点击在编辑器窗口中键入所需的选项。

如果未指定任何内容，则使用默认设置。注意 ，默认设置取决于所选的环境。例如，如果选择了make generator，则默认值为-j ，而对于Microsoft Visual C ++，该字段为空。
