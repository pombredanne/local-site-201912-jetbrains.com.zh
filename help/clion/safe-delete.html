安全删除

通过安全删除重构，您可以安全地从源代码中删除文件。此重构可防止您不受控制地删除在源代码中具有用法的声明。

为了确保删除是安全的，CLion会查找要删除的文件的用法。如果找到了这样的用法，则可以在实际删除符号之前浏览它们并在代码中进行必要的更正。

  1. 选择要删除的项目。

  2. 请执行以下任一操作：

    • 按Alt+Delete 。

    • 选择重构|从主菜单或上下文菜单中安全删除 。

    • 选择重构|从主菜单重构 Ctrl+Shift+Alt+T ，然后选择安全删除 。

  3. 在“ 安全删除”对话框中 ，选择必要的选项，然后单击“ 确定” ：

  4. 如果重构可能不安全，则会打开“ 检测到的使用情况”对话框。

    • 查看用法。 单击此按钮以查看要在代码中使用的项目。结果，“ 查找工具”窗口打开。

      分析您的代码并进行必要的更正。然后单击“ 执行重构” 。（如果要从头开始重新运行重构，请点击重新运行安全删除 。CLion将再次检查重构是否安全。）

    • 仍然删除。 单击此按钮可以删除该项目，而无需查看其用法。

安全删除文件和CMake

当删除文件，如果这些文件中的CMake命令，克利翁执行安全引用删除：文件和引用删除，CMake的重新加载（如果自动重已开启）。

以下命令支持对文件及其在CMakeLists.txt中的引用进行安全删除：

  • add_executable() ;

  • add_library() ;

  • set() ;

  • add_custom_target_command() ;

  • fltk_wrap_ui() ;

  • qt_wrap_cpp() ;

  • qt_wrap_ui() ：

  • set_property() ;

  • set_source_files_properties() ;

  • try_compile() 。

如果您要删除的文件是CMake命令的唯一参数（例如， add_executable(foobar foo.cpp) ），则删除后此命令可能会变得不正确。
