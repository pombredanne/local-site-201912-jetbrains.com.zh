任务和背景

当您处理项目时，可以将您的工作组织在需要完成的较小任务中。

这些可以是您自己设置的任务。在CLion中，您可以将大量工作分解为较小的任务，并将它们与变更列表相关联。

或者这些可以是来自问题跟踪器的任务。例如，您可以直接从CLion处理分配给您的任务和错误。为此，请连接IDE和您的跟踪器帐户。

将IDE连接到您的跟踪器

  1. 在“ 设置/首选项”对话框中Ctrl+Alt+S ，选择工具|任务服务器 。

  2. 请点击然后从列表中选择必要的问题跟踪器。

  3. 输入连接详细信息。请注意，设置因问题跟踪器而异。

    通常，您必须指定服务器URL和连接凭证： Username和Password 。

    在某些情况下，您可以在密码字段中输入API令牌而不是密码。

  4. 选择“ 共享URL”选项以允许团队其他成员访问服务器。

  5. 如果要通过代理服务器访问服务器，请单击“ 代理设置” 。您可以在“ HTTP代理”部分中找到有关代理设置的更多信息。

  6. 在“ 提交消息”选项卡上，可以启用为更改列表添加提交消息并配置消息模板的功能。

  7. 在“ 服务器配置”选项卡上，配置用于连接到问题跟踪器的高级参数 。

    此选项卡仅适用于某些跟踪器（例如，对于不支持开箱即用的跟踪器 ）。

    此选项卡上的字段中提供了代码完成功能。

服务器配置选项卡参数

项目

描述

登录网址

认证资源。IDE每次从服务器检索问题列表之前都会向该资源发送请求，例如： {serverUrl} / rest / user / login？login = {用户名}＆password = {密码} 。

如果已选中“ 常规”选项卡上的“ 使用HTTP身份验证”复选框，则该字段将被禁用。

任务列表URL

用于从服务器检索问题列表的资源，例如： {serverUrl} / rest / api / 2 / search 。

单一任务网址

用于通过ID检索有关问题的详细信息的资源，例如： {serverUrl} / rest / api / 2 / issue / {id} 。

该字段是可选的，除非您选中“ 单独请求中的每个任务”复选框。

GET或POST

选择必要的HTTP请求类型。

每个任务在单独的请求中

启用此选项使IDE可以发送多个请求，以便首先检索具有其ID的问题列表，然后使用“ 单个任务URL”字段中指定的资源分别获取有关每个问题的详细信息。

此选项适用于具有受限REST API的问题跟踪器，这些API无法在单个响应中发送所有必需的信息。

回应类型

选择问题跟踪器响应的格式：用于XPath的XML ，用于JSONPath的 JSON或用于正则表达式的Text 。

选择器表

选择器允许您指定要从服务器响应中检索有关问题的信息。

任务

服务器响应中问题列表的路径。此字段是必填字段。

ID

服务器响应中问题ID的路径。此字段是必填字段。

摘要

服务器响应中问题标题的路径。此字段是必填字段。

有关如何配置这些参数的示例，请参考Configure Generic Task Server教程。

指定其他集成选项

  1. 在“ 设置/首选项”对话框中Ctrl+Alt+S ，选择工具|任务 。

  2. 配置必要的选项：

    • 变更列表 名称格式 ：当您打开或创建新任务时，CLion 会提示您创建与此任务关联的新变更列表 。在此字段中，您可以指定一个模板，该模板将用于为新变更列表生成名称。

      请点击从列表中选择占位符。

    • 功能分支名称格式 ：创建或打开新任务时，CLion会提示您创建一个新的功能分支。在此字段中，您可以配置用于生成新功能分支名称的模板。

      请点击从列表中选择占位符。

      使用小写字母和替换空格为选项来配置提示的功能分支名称。

      如果您的IDE与问题跟踪器集成在一起，则这些设置很有用。例如， DSGN-0001添加新图标任务名称将转换为dsgn-0001add-new-icon功能分支名称。

      如果生成的名称对您不起作用，则始终可以在创建功能分支名称之前对其进行修改。

    • 任务历史记录长度 ：CLion存储的任务数。

    • 在提交时保存上下文 ：每次提交更改时，CLion都会创建一个新的关闭的本地任务，该任务将保留您使用过的文件，书签和断点。这样，您可以在将来的任何时候快速还原与该任务关联的所有选项卡。

    • 启用问题缓存 ：优化CLion与问题跟踪器之间的同步。如果您使用“缓慢”的问题跟踪系统，则特别推荐同步。

      CLion缓存从跟踪器加载的问题列表，并重复更新它们。您可以指定应缓存多少个问题，以及CLion应该多久更新一次有关它们的信息。

任务

在CLion中，有两种类型的任务：

  • 从问题跟踪器加载到CLion的任务。这些是跟踪程序任务跟踪程序任务与问题跟踪程序中的相应问题链接。这使您可以直接从CLion监视和更新它们。

  • 最初在CLion中创建的任务。这些是本地任务 。本地任务与问题跟踪器无关。

如果创建了至少一个任务，则在工具栏上将出现一个名为“ 任务组合”的列表。

打开跟踪器任务

将跟踪器任务连接到问题跟踪器后，会将跟踪器任务加载到CLion。

  1. 在主菜单中，选择工具|任务和上下文|公开任务 Shift+Alt+N ，或单击工具栏上的任务组合。

  2. 从列表中选择必要的任务。

  3. 在“ 打开任务”对话框中，您可以更新问题状态。

    如果要关闭编辑器中当前打开的所有选项卡，请选中“ 清除当前上下文”复选框。

  4. 在“ VCS操作”部分中，您可以创建一个新的变更列表，选择要向其贡献的现有分支，或创建一个新分支。

    您还可以搁置当前更改 ，以便稍后返回。

创建本地任务

在CLion中，您可以创建不是源自问题跟踪器的本地任务。

  1. 在主菜单中，选择工具|任务和上下文|打开任务 。您也可以使用任务组合键或按Shift+Alt+N 。

  2. 在输入任务名称对话框中，选择创建新任务 。

  3. 输入新任务的名称。

    如果要关闭编辑器中当前打开的所有选项卡，请选中“ 清除当前上下文”复选框。

  4. 在“ VCS操作”部分中，您可以创建一个新的变更列表 ，选择您要贡献的现有分支，或创建一个新分支。

    您还可以搁置当前更改 ，以便稍后返回。

查看任务说明

选择要切换到的任务时，任务列表仅显示任务ID。这些信息并不总是足够的，因为它既没有反映导致问题的步骤，也没有反映相关的讨论。

  1. 转到工具|任务和上下文|打开任务 。

  2. 打开必要的任务，然后按Ctrl+Q在CLion中打开任务描述，或Shift+Alt+B在浏览器中查看描述。

或者，转到工具|任务和上下文 ，然后单击在浏览器中 显示“ 任务ID ”说明或打开“ 任务ID ” 。

查看已关闭的任务

如果整个项目或受影响的目录都在版本控制下，则关闭的本地任务是与变更列表不关联的任务。

已关闭的跟踪器任务是在问题跟踪器中具有关闭状态的任务。

  1. 单击任务组合，然后单击“ 打开任务” 。

  2. 选择包括关闭的任务复选框，或按Shift+Alt+N 。

关闭任务

  • 转到工具|任务和上下文 ，然后单击关闭活动任务 。

这将关闭CLion中的当前上下文。选择必要的复选框以提交更改，并选择合并所创建的分支。对于跟踪器任务，您还可以更改其状态。新状态将传播到您的问题跟踪器。

删除任务

如果不需要任务显示在CLion中，则可以将其从任务列表中删除。

  1. 单击主工具栏上的任务组合。

  2. 选择一个或多个要删除的任务。

    使用Shift （用于相邻项目）或Ctrl （对于非相邻项目）键以进行多项选择。

  3. 点击右箭头按钮，然后选择删除 。

删除跟踪器任务时，请将其从IDE中删除。它们将保留在您的问题跟踪器中。在这种情况下，本地任务将被完全删除，因为它们未连接到您的问题跟踪器。

时间跟踪

确保在“ 设置/首选项” | “ 时间”中启用了“ 时间跟踪”插件。插件窗口。

使用CLion，您可以跟踪在编辑器中处理任务所花费的时间。对于本地任务，如果您想知道在处理项目时需要多少时间来完成一项任务，此信息可能会有所帮助。

对于跟踪器任务，如果您的问题跟踪器配置要求您记录花费在任务上的时间，则此选项很有用。在这种情况下，您可以将时间日志从CLion发送到跟踪器。

启用时间跟踪

  1. 在设置/首选项对话框中，选择工具|任务时间跟踪并启用启用时间跟踪选项。

  2. （可选）更改挂起延迟值。

    在这里，您可以指定必须多少时间保持不活动状态才能将任务视为挂起状态。

要重新打开“ 时间跟踪”工具窗口，请选择“ 视图” |“视图”。工具窗口|时间跟踪 。

追踪时间

  • 请点击用于自动时间记录。

    要进行手动时间跟踪，请单击 。要停止手动时间跟踪，请单击 。

将时间日志发送到跟踪器

  1. 单击将工作项发布到错误跟踪器 在时间跟踪工具窗口中。

  2. 指定要记录的时间间隔，并在必要时添加注释。单击确定 。

语境

上下文是在编辑器中打开的一组书签，断点和选项卡。上下文链接到任务，但是您可以使用上下文而无需将它们与特定任务相关联。

具有单独的上下文使您可以处理几件事，并且可以在它们之间切换而无需混合更改。

您可以在“ 收藏夹”工具窗口中查看书签和断点的列表。

保存上下文

  1. 在主菜单中，选择工具|任务和上下文|保存上下文 。

  2. 在“ 保存上下文”对话框中，输入上下文的名称，然后单击“ 确定” 。

将项目添加到现有上下文

  1. 在主菜单中，选择工具|任务和上下文|加载上下文 。

  2. 添加必要的项目（或书签断点），或打开必要的文件，并保存上下文： 工具|任务和上下文|保存上下文 。

在上下文之间切换

在任务之间切换时，IDE会自动切换相关上下文。但是，如果您使用的上下文未与任务关联，则可以手动在它们之间切换。

  1. 在主菜单中，选择工具|任务和上下文|加载上下文 。

  2. 在“ 加载上下文”弹出窗口中，从列表中选择必要的上下文。

    或者，单击向右箭头，然后选择“ 加载” 。

清除上下文

  • 要清除当前上下文而不加载另一个上下文，请选择“ 工具” |“其他”。任务和上下文|从主菜单中清除上下文 ，或按Shift+Alt+X 。

删除上下文

任务完成后，或者如果不再需要上下文，则可以将其删除。

  1. 在主菜单中，选择工具|任务和上下文|加载上下文 。

  2. 在“ 加载上下文”弹出窗口中，单击向右箭头，然后选择“ 删除” 。
