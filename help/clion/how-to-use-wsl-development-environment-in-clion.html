WSL

WSL （ WSL 2 ）-Linux的Windows子系统-是一个兼容性层，用于在Windows 10上本地运行Linux二进制可执行文件。当前，它支持几种Linux发行版 ，例如Ubuntu，OpenSUSE和SLES。

通过为您的项目设置WSL工具链 ，您可以使用CMake和Linux编译器进行构建，并在WSL上运行/调试，而无需在Windows计算机上运行CLion。

配置WSL

  1. 从Microsoft Store下载并安装WSL发行版（例如，Ubuntu）。

    对于此步骤，请确保将Windows 10与最新的“秋季创作者更新”一起使用（最低版本1709，内部版本16299.15）。有关说明，请参阅官方指南“ 安装Linux的Windows子系统” 。

    要使用WSL 2 ，您的Windows版本应为10 build 18917或更高版本。请按照以下说明切换发行版。

    请注意，CLion不支持旧版WSL ，您可能在将系统升级到Windows 10的内部版本16299.15或更高版本之前已经安装了该版本。在这种情况下，您需要更新WSL发行版。

  2. 运行Ubuntu。

    在Ubuntu的第一顿午餐中，系统可能会提示您启用Windows可选功能。在这种情况下，您需要：

    • 以管理员身份打开Windows Power Shell并运行

      Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux

    • 重启你的电脑。

  3. 设置WSL Ubuntu环境：

    • 安装cmake，gcc或/和clang（以及可选的build-essentials软件包），如下所示：

      须藤apt-get install cmake gcc clang gdb build-essential

    • 配置并运行打开的ssh-server。

      您可以手动或使用我们的脚本来设置配置（ 请注意 ，如果您的Linux发行版不同于Ubuntu，则该脚本可能需要调整）。要下载并运行脚本，请使用以下命令行：

      wget https://raw.githubusercontent.com/JetBrains/clion-wsl/master/ubuntu_setup_env.sh && bash ubuntu_setup_env.sh

  4. 接下来，检查ssh连接：

    ssh用户名@localhost -p2222

    端口号2222对我们的配置脚本有效。如果您不使用我们的脚本，则端口号可以不同。

如果已将先前配置的WSL 升级到WSL 2 ，则需要将应用程序绑定到0.0.0.0代替127.0.0.1 （请参阅WSL 1和WSL 2之间的用户体验更改 ）：

  • 1）打开/ etc / ssh / sshd_config为sudo 。

  • 2）在ListenAddress 127.0.0.1线，更换127.0.0.1与0.0.0.0 。

  • 3）保存更改并运行sudo service ssh restart 。

为您的项目配置WSL工具链

  1. 在CLion中，转到“设置/首选项” |“设置”。构建，执行，部署|工具链并单击创建一个新的工具链。从环境列表中选择WSL 。CLion会自动检测已安装的发行版，或者您可以手动设置路径。

    提供远程凭据 （如果发生意外错误，请尝试故障排除说明）：

    如果您正确指定了ubuntu.exe的路径，但CLion仍然找不到可执行文件，则可能是Windows 10版本1803中修复的WSL问题 。您需要将系统更新到版本1803或更高版本，或者启用wsl.close.process.input键入注册表（转到“ 帮助” |“查找操作” ，键入“ 注册表” ，然后按键名进行搜索）。

  2. 现在开始使用工具链，请执行以下一项操作：

    • 将WSL工具链设置为默认值 。这样，它将自动连接到默认的CMake配置文件。

    • 创建一个单独的CMake配置文件 ，将其连接到WSL工具链，然后在配置切换器中选择此配置文件：

为了在WSL上运行Valgrind ，请确保选择与WSL工具链关联的CMake配置文件。有关详细信息，请参见此说明 。

故障排除

  • 确保在“设置” /“首选项” |“设置”中启用了两个捆绑的插件，“ FTP / SFTP连接”和“ 远程主机数据访问 ”。插件 。

  • 如果在创建工具链时收到此WSL分发错误的凭据无效 ，请在Windows注册表编辑器中检查HKEY_CURRENT_USER \ Software \ Microsoft \ Windows \ CurrentVersion \ Lxss {ubuntu_id} \ BasePath的值（要访问它，请启动regedit从“开始”菜单中）。如果值包含非ASCII字符，请使用本期中描述的解决方法。

  • 由于IntelliJ平台问题 ，使用区分大小写的WSL文件系统和不区分大小写的Windows文件系统存在问题。作为一种解决方法，您可以执行以下操作：

    • 转到帮助|编辑自定义属性...菜单选项并指定

      idea.case.sensitive.fs = true

    • 使用文件|使缓存无效并重新启动IDE 。无效的缓存和重新启动菜单选项。

    如果无法从“欢迎”屏幕打开“ 编辑自定义属性”弹出窗口，请打开任何项目，然后从“ 帮助” |“编辑”中编辑属性。编辑自定义属性…主菜单选项。
