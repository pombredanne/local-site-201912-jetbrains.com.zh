为CLion开发插件

当您错过CLion中的某些功能时，可以考虑编写自己的插件。本文旨在帮助您掌握IntelliJ Platform插件开发的基础知识，并指导您完成实现CLion插件的第一步。

基本

语言 ：Java或基于Java的语言 （例如Kotlin，请参阅Kotlin for Plugin Developers ）

IDE ：IntelliJ IDEA社区/ Ultimate

工作流程 ：

  • 基于Gradle-建议用于新插件

  • DevKit-现有插件支持

可用的API ：IntelliJ平台API

任何CLion插件都是IntelliJ Platform的扩展，因此通常它是一个特定的Java项目。IntelliJ插件开发有两种受支持的工作流程：将gradle-intellij-plugin与Gradle作为构建系统一起使用（推荐的方式），或者与IntelliJ IDEA自己的构建系统一起使用DevKit 。

作为用于插件开发的IDE，请使用IntelliJ IDEA 社区或Ultimate版本。

除了IntelliJ IDEA本身之外，您还需要目标CLion发行版。使用基于Gradle的方法，构建脚本可以自动下载它。如果使用DevKit，请确保手动安装CLion。

平台开放API

CLion并未为其自己的公共API提供保证的稳定性和详细说明，但是您可以使用Platform Open API的所有功能。探索Platform API的有效方法是从官方存储库中获取并检查IntelliJ IDEA社区版的源代码。

IntelliJ平台的API可以在发行版之间进行更改，因此会影响插件功能。IntelliJ平台和插件中的“ 不兼容的更改”中列出了此类更改。请注意，CLion内部SDK可能随时更改，恕不另行通知。

要获得有关CLion插件API使用方面的帮助，请随时在社区论坛上提问。

扩展名

IntelliJ平台为插件提供了扩展和扩展点 ，以添加新功能或自定义现有功能。插件应声明其实现了一个或多个扩展或扩展点在plugin.xml文件中。要获取可用扩展名和扩展点的列表，请浏览Community Edition源文件此目录中的* Extension / * ExtentionPoints文件。

一些常见的扩展点如下：

  • 操作 -菜单和工具栏项目。

  • 代码检查 -静态代码分析。

  • 意图 -编辑器中可用的特定于上下文的动作。

  • 代码完成 。

插件依赖

您的插件可能取决于其他插件的类。在这种情况下，插件可以是捆绑的，也可以是第三方的，甚至可以是您自己的。有关如何表达依赖性的说明，请参阅插件依赖性 。

您的插件应指定将与哪些产品兼容（所有基于IntelliJ的IDE，仅CLion或某些子集）。您可以通过使用在plugin.xml中标记（请参阅IntelliJ产品的插件兼容性 ）。

在所有基于IntelliJ的产品中都可以使用某些模块，而在CLion中专门可以使用以下模块：

  • com.intellij.modules.cidr.lang -C / C ++语言支持（此模块在AppCode和Android Studio中也可用）；

  • com.intellij.modules.cidr.debugger -本机调试器支持（也在AppCode和Android Studio中）；

  • com.intellij.modules.clion -核心CLion功能。

Plugin.xml文件

plugin.xml文件是一个关键的插件描述符 ，其中包含插件元数据：名称，描述，作者，变更日志以及已实现功能的列表。

IntelliJ插件的plugin.xml内容的一些示例： Rust支持 ， IdeaVim和.gitignore 。

必须输入名称，标识，描述和供应商的值，才能在插件数据库中正确注册您的插件 。如上所述，每个depends参数都是插件依赖的模块的ID。 Depends参数列表隐式定义了兼容产品。

CLion插件入门

进一步地，我们将重点关注基于Gradle的方法，将其作为所有新插件项目的推荐方法。使用DevKIt中介绍了现有插件仍支持的DevKit工作流程。

首先， 安装 IntelliJ IDEA Community或Ultimate。在开始之前，请检查IntelliJ IDEA实例中是否同时启用了两个捆绑的插件Gradle和DevKit （转到“设置/首选项” |“插件” |“已安装”，并按名称搜索插件）。

进一步的步骤

上面的示例非常基础，它仅使用常见的IntelliJ功能，而没有CLion细节。要进行更深入的研究，请浏览SDK文档的“ 插件结构”部分， IntelliJ SDK文档的常见问题解答和社区论坛 。

如果您有兴趣开发用于自定义语言支持的插件，请参阅Platform SDK文档中的专用部分 ，并查看Antlr的插件开发说明 。PSI Viewer插件可以帮助您熟悉程序结构接口PSI 。

另请参阅有关测试插件的指导文档。使用Gradle和DSL发布发布插件中讨论了发布插件的主题。
