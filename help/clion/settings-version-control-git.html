Git可执行文件的路径

在此字段中，指定Git可执行文件的路径。输入路径或单击浏览 并在打开的对话框中指定路径。

测试

单击此按钮以验证Git可执行文件的路径。

仅为当前项目设置此路径

如果要为不同的项目使用不同的路径，并且不想将此设置全局应用，请选择此选项。

在所有根上执行分支操作

仅当您具有多根项目时，即在单个项目中有多个Git存储库时，此选项才可用。

如果要分支操作（例如， checkout ， merge ，等等。）同步应用于项目中的所有存储库。

自动选择樱桃

当您选择一个特定的提交时，将显示“ 提交更改”对话框。如果选择了“ 自动选择樱桃提交”选项，则在单击“樱桃选择”按钮时以静默方式提交选定的提交 ，而不显示“ 提交更改”对话框。

在选择推送到受保护分支的提交时，添加“从樱桃中摘取”后缀

如果要从受保护的分支中选择提交时保留原始提交的引用，请选择此选项。默认情况下，选择更改时不添加任何后缀。

警告是否要提交CRLF行分隔符

选择此选项以启用智能处理LF和CRLF行分隔符。CLion将分析您的配置，警告您是否要将CRLF提交到存储库，并建议更改core.autocrlf设置为true要么input取决于您的操作系统。

请注意，此设置不适用于已设置任何相关Git属性的文件 。在这种情况下，CLion假定您清楚地了解自己在做什么，并从分析中排除了此类文件。

如果取消选择此选项，则必须使用“ 差异查看器”对话框手动解决行尾的问题。

提交分离的HEAD或重新设置基准时发出警告

如果您希望CLion在从分离的磁头上执行提交或在基于基准的情况下显示警告，请选择此选项，因为这可能会导致问题和代码丢失。

明确检查遥控器上的传入提交

如果启用此选项，CLion将检查是否存在尚未提取到本地存储库的待处理传入提交，并将在“ 分支”弹出窗口中标记此类分支。

选择您要CLion查询远程设备以检查传入提交的方式：

  • 自动 ：如果使用HTTP或Git协议访问远程服务器，CLion将在后台检查更新。如果使用SSH，则不会执行此检查，以免外部身份验证应用程序意外弹出。

  • 始终 ：即使使用SSH访问遥控器，CLion也会在后台检查更新。

  • 永不 ：CLion将不向远程服务器查询传入的提交，并且“ 分支”弹出窗口中将显示警告，允许您手动运行检查。

更新方式

使用此列表选择将本地存储库与远程存储同步的策略。当push操作被拒绝（如果启用了“ 如果拒绝当前分支的推送则自动更新”选项），或者在您调用更新项目操作时被拒绝 。提供以下选项：

  • 合并 ：选择此选项可在更新期间执行合并 。这相当于跑步git fetch然后git merge ， 要么git pull --no-rebase 。

  • 调整基线 ：选择此选项来执行衍合在更新过程中。这相当于跑步git fetch然后git rebase ， 要么git pull --rebase （所有本地提交都将放在更新后的上游头文件的顶部）。

  • 分支默认值 ：如果要对不同的分支应用不同的更新策略，请选择此选项。您可以为分支中的每个分支指定默认更新类型branch. .git / config配置文件部分。

如果拒绝当前分支的推送，则自动更新

如果您希望当前分支自动更新，请选中此复选框。 push从当前分支到其跟踪分支的操作被拒绝。

如果取消选择此选项，则由于本地存储库和远程存储未同步而拒绝推送分支时，CLion将显示“ 拒绝推送”对话框。

请注意以下几点：

  • 如果您之前从未见过“ 拒绝推送”对话框，并且最初启用了此复选框，则CLion将通过以下方式静默更新冲突的本地分支： merge操作。

  • 如果您已经遇到“ 拒绝推送”对话框并选择了“ 记住更新方法选择...”选项，则CLion将保存您的最后选择rebase要么merge并将应用它以静默方式更新冲突的本地分支。

    因此，要更改“已记住”设置，请清除复选框，访问“ 拒绝推送”对话框，选择“ 如果推送...拒绝 ，则自动更新”选项，并调用其他更新策略。

显示提交和推送的推送对话框

如果要在单击“ 提交并推送”后的“ 提交更改”对话框中显示“ 推送”对话框 ，请选择此选项。否则，您的更改将自动推送到受影响的存储库。

仅当提交到受保护的分支时才显示“推入”对话框

如果在单击“ 提交”和 “ 提交更改”对话框中的“ 推送”时，要推送到受保护的分支，则只希望显示“ 推送 更改”对话框 。否则，您的更改将自动推送到受影响的存储库。

受保护的分支

如果要禁用对某些分支强制推送更改的功能，请在此处列出它们（这是团队共享参数，存储在.idea / vcs.xml中 ）。

您可以列出用分号分隔的几个分支，或者在输入被视为正则表达式列表时提供分支模式。

使用凭证助手

如果您不想覆盖凭据助手，请选择此选项，这是默认行为。您将能够在Git登录对话框中使用凭据帮助程序进行身份验证。

按路径过滤更新项目信息

如果你不想要得到的所有更改项目信息更新信息标签，当你执行更新 ，您可以通过过滤特定的路径列表。
