CLion 2019.3帮助

运行和调试Node.js

CLion可帮助您运行和调试Node.js应用程序。您可以调试从CLion启动的应用程序，也可以将其附加到已经运行的应用程序。

在你开始前

如管理插件中所述，在“ 插件”页面上安装并启用NodeJS存储库插件。

调试Node.js应用程序

CLion使调试Node.js应用程序更加容易。您可以在JavaScript或TypeScript代码中放置断点，因此不再需要任何断点。 debugger和console.log()陈述。您可以做很多事情来帮助您探索代码并了解错误的位置。在“ 调试工具”窗口中，您可以查看调用堆栈和当前状态的变量，在编辑器中评估表达式，以及单步执行代码。

您可以通过两种方式启动调试会话：

  • 使用Node.js运行/调试配置，与应用程序一起启动调试器。

  • 将调试器附加到已经运行的应用程序。在这种情况下，您的应用程序已经在调试模式下运行，并且CLion附加到正在运行的进程。

    CLion认可--inspect ， --inspect-brk ，现在已弃用--debug标志，以便您可以访问任何应用程序进行调试。

    要调试正在运行的应用程序，请使用“ 附加到Node.js / Chrome”配置。

在计算机上与Node.js应用程序一起启动调试器

  1. 必要时在代码中设置断点 。

  2. 创建一个Node.js运行/调试配置。必要时，CLion可以生成JavaScript调试配置，并与Node.js配置一起自动启动它。

  3. 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的Node.js配置，然后单击在它的旁边。将打开“ 调试工具”窗口 。

  4. 执行将触发带有断点的代码执行的步骤。

  5. 切换到CLion，此时已启用“ 调试工具”窗口的控件。进行调试会话— 逐步执行断点 ，在帧之间切换，即时更改值， 检查已暂停的程序 ， 评估表达式并设置监视 。

调试使用nodemon的Node.js应用程序

CLion内置调试器可以自动重新连接到正在运行的Node.js进程。 这使您可以调试使用nodemon实用程序的 Node.js应用程序，该实用程序会在代码更新时自动重新加载Node.js进程。

要调试此类应用程序，您需要在调试模式下启动它（使用--inspect要么--inspect-brk标记），然后使用启用自动重新连接选项的附加到Node.js / Chrome调试配置进行连接 。

安装nodemon

  • 在嵌入式终端 （ Alt+F12 ），输入npm install --save-dev nodemon要么yarn add nodemon --dev安装nodemon作为开发依赖项。

调试应用程序

  1. 根据需要在代码中设置断点。

  2. 如调试正在运行的Node.js应用程序中所述，创建新的“ 附加到Node.js / Chrome”配置，然后选中“ 自动重新连接”复选框。

    通常，您无需在配置中更改端口9229因为它是调试器正在侦听的默认端口。但是，您可以在调试模式下运行应用程序时再次检查记录的消息中使用了哪个端口。

  3. 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的“ 附加到Node.js / Chrome”配置，然后单击在它的旁边。调试器将在您在CLion中的代码中的断点处停止。

    现在，每次您对代码进行任何更改并保存它们Ctrl+S ，nodemon将自动重新加载应用程序，调试器将自动重新附加到重新启动的进程。
