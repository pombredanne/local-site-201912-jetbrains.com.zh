当前的问题和局限性

  • CLion使用CMake File API ，它最早出现在CMake v3.14中。但是，CLion从CMake v3.15中引入的更新开始支持它，因此，如果您决定从捆绑的CMake中切换，请确保使用版本3.15.x或更高版本 。

  • 当前，非默认生成器（ CPP-17622 ）不支持重新编译单个文件。

  • 对于Xcode和Visual Studio等多配置生成器，CLion将仅使用与CMake配置文件中指定的生成类型相对应的配置。

环境变量

您可以将其他环境变量传递给CMake并通过CMake设置对话框的Environment字段进行构建（导航至Settings / Preferences | Build，Execution，Deployment | CMake ）。

CMake生成和构建的总体有效环境包括：

  1. 家长环境

    要包含父环境，请通过单击打开“ 环境变量”对话框。 或按Shift+Enter ，然后设置包括系统环境变量复选框。您另外指定的值将附加到系统变量中。否则，清除复选框后，您的自定义值将覆盖系统值。

  2. 工具链环境

    例如，在vcvarsall.bat中为MSVC定义的变量，路径变量如mingw/bin ， 和别的。

  3. CMake配置文件环境

    您的自定义变量在“ 环境”字段中指定。
