检查：成员函数可以是静态的

设置/首选项|编辑器检查 -C / C ++，常规

默认严重性： 弱警告

在C ++代码中，可以标记成员函数static当它不使用它或访问任何非静态类数据时，例如：

类Object {static int x;公开：int getX（）{return x; }}

成员函数可以为静态（MFCBS）检查可检测类似的功能int getX()从上面的代码段中，建议使它们static 。将成员函数转变为成员的好处static在可能的情况是，静态函数是在CPU周期和存储器方面更便宜：没有实例创建所需的一个函数调用，并且没有必要按这个到功能堆栈。

当启用基于Clangd的语言引擎并且默认严重性为“ 弱警告”时 ，默认情况下将启用此检查。您可以在“ 检查设置”对话框（ C / C ++类别，“ 常规”节点）中配置其严重性和范围，或禁用检查。

请注意，如果您禁用基于Clangd的引擎，则此检查仅适用于捆绑的（默认） Clang-Tidy二进制文件 。

功能范围

对于包含错误或不完整（红色）代码的功能，禁用MFCBS 。此外，检查不会建议您执行成员职能static什么时候：

  • 它访问一个非静态的类成员或调用一个非静态的成员函数 ，例如：

    S类{int a; int foo（）{return dbl_a（）; } int dbl_a（）{return（a + a）; };

  • 它被标记为virtual ， const或volatile 。

  • 它具有ref限定词，例如void f() &要么void g() && 。

  • 它是构造函数，析构函数或重载运算符 ，包括诸如operator int() 。

  • 这是一个包含相关名称的模板函数。例如，在下面的代码片段中， int foo()包含对T::boo() ，它可能是非静态成员函数，具体取决于特定的模板实例，因此MFCBS无法确定是否int foo()可以标记为static或不：

    struct A {template int foo（）{return T :: boo（）; } int boo（）{return 11; };
