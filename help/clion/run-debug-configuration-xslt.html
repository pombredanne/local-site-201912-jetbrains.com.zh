输入项

使用此区域中的控件来指定要处理的XML文件和要执行的脚本。

  • XSLT脚本文件 -在此文本框中，指定XSLT样式表文件的路径。手动输入路径或单击浏览 然后在打开的“ 选择XSLT文件”对话框中选择所需的文件。

  • 选择XML输入文件 -从此下拉列表中，选择要转换的XML输入文件。该列表包含通过文件关联功能与所选样式表相关联的所有XML文件。要指定不在列表中的文件，请单击“ 浏览”。 然后在打开的“ 选择XML文件”对话框中选择所需的文件。

输出量

使用此区域中的控件来配置脚本输出的处理。

  • 在默认控制台中显示 -选择此选项可在正常运行控制台中显示输出，以及来自XSLT转换器的任何警告和错误消息，以及脚本生成的消息，例如xsl:message 。

  • 在Extra Console选项卡中显示 -选择此选项可使产生的输出显示在XSLT Output选项卡中。

    默认情况下选中此选项。

  • 高亮显示输出为 -选择文件类型以高亮显示输出。

  • 保存到文件 -选择此选项可将输出直接保存到文件。在文本框中，指定目标文件的名称。手动输入文件的路径或单击浏览 然后在打开的“ 选择输出文件”对话框中选择所需的文件。如果键入不存在的文件名，CLion将创建一个文件并将输出保存到该文件。

    • 执行后在编辑器中打开文件 -选中此复选框，可以在脚本成功执行后在编辑器中打开输出文件。

    • 执行后在Web浏览器中打开文件 -选中此复选框，可在脚本成功执行后在配置的Web浏览器中打开带有输出的文件。

    指定的文件将被覆盖，而无需请求确认。

参量

使用此区域中的控件来创建和管理要传递给脚本的参数列表。

  • 加 -单击此按钮创建一个新条目。

  • 去掉 -单击此按钮可从列表中删除所选条目。

  • 名称 -在此文本框中，指定参数的名称。

  • 值 -在此文本框中，指定参数的值。

    该字段是必填字段。没有值的参数不会传递到脚本。默认情况下不分配值。

进阶标签

在此选项卡中，配置运行配置中通常不需要的其他选项。

项目

描述

智能错误处理

  • 如果在执行过程中发生错误，请清除此复选框以使控制台显示完整的错误消息，包括其完整的堆栈跟踪。

  • 选中此复选框可禁止显示staktrace，并使控制台仅显示有关错误的相关信息。

虚拟机参数

在此文本框中，指定要传递给执行XSLT脚本的VM的可选VM参数。这些可以是堆大小，垃圾回收选项，文件编码等。如果VM参数行太长，请单击展开文本框并在其中键入文本。

工作目录

在此文本框中，指定要使用的工作目录。手动输入路径或单击浏览 然后在打开的“ 工作目录”对话框中选择所需的文件夹。

如果在文本框中未指定文件夹，则工作目录将是XSLT脚本文件所在的目录。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在CLion中定义，请添加其定义。有关更多信息，请参见外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 运行文件监视程序 ：选择此选项可使CLion应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 编译TypeScript ：选择运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都会反映在生成的JavaScript文件中。 在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防万一检测到任何错误：

    • 如果选中“检查错误”复选框，则编译器将显示所有错误，并且运行配置将不会启动。

    • 如果清除“ 检查错误”复选框，则编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

Alt+Delete

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
