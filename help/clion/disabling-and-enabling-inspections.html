启用/禁用和禁止检查

如果您认为某些检查报告了您不感兴趣的问题，则可以禁用此类检查。请注意，当您禁用检查时，它会在当前检查配置文件中被禁用；在所有其他配置文件中，它保持启用状态。

有几种禁用/启用检查的方法：

  • 使用“检查设置”页面 -这是管理检查的主要界面；您可以在此处立即看到在所有检查配置文件中启用或禁用了哪些检查。

  • 使用意图动作 -这是直接在编辑器中禁用突出显示的代码问题的方法。

  • 在“检查结果”工具窗口中 -这是一种在分析检查结果时禁用无趣的问题的快速方法。请注意，在这里您只能禁用检查。

禁用或启用检查

  1. 在设置/首选项|编辑器检查对话框Ctrl+Alt+S ，通过展开分组节点或使用搜索字段来找到所需的检查。

  2. 使用检查旁边的复选框以禁用或启用它。

  3. 应用更改并关闭对话框。

在编辑器中禁用对突出显示问题的检查

  • 当您以这种方式禁用检查时，将禁用当前检查配置文件 。

  • 要重新启用以这种方式禁用的检查，请使用上述主要步骤 。

  1. 将尖号设置为突出显示的问题。

  2. 点击灯泡图标或按Alt+Enter显示检查警报和建议列表。

  3. 选择要禁用的检查，然后单击右箭头按钮或仅按右箭头键。

  4. 在子菜单上，单击“ 禁用” 。

从检验结果报告中禁用检验

  • 当您以这种方式禁用检查时，将禁用用于运行检查的检查配置文件 。您可以在“ 检查结果”窗口的选项卡标题中看到它。

  • 要重新启用以这种方式禁用的检查，请使用上述主要步骤 。

  1. 在“ 检查结果工具”窗口中 ，右键单击要禁用的检查。

  2. 在上下文菜单中，选择禁用检查 。

  3. 按下过滤器按钮隐藏禁用的检查警报。

禁止检查

由于某些原因，您可能要部分禁用特定的检查，即忽略某些代码问题，同时继续检测此检查中的其他问题。

例如，CLion认为某些代码“死了”，您可以看到它是正确的。该检查很有帮助，您不想禁用它。但是，您可能以后要使用此代码，而不希望在编辑器中突出显示该代码，也不希望其出现在问题报告中。

为此，CLion允许您取消对特定语句，函数/方法，标记或文件的某些检查。在分析检查结果时，可以在编辑器中使用建议列表进行操作，也可以在“检查结果”工具窗口中进行操作。

让我们总结一下抑制和禁用代码检查之间的区别：

  • 禁止检查时，CLion会插入一个特殊注释，告诉代码分析引擎忽略此检查在特定代码段中发现的问题。

  • 禁用检查时，只需将其关闭即可，因此代码分析引擎只会忽略此检查发现的代码问题。

不能禁止某些代码检查（例如那些检测错误）。

在编辑器中禁止检查

  1. 在编辑器中将光标设置为突出显示的代码问题。

  2. 按Alt+Enter ，或点击灯泡图标扩展建议列表。

  3. 根据问题的不同，您将看到与检查有关的快速修复或“ 检查”选项项。

  4. 使用向上/向下箭头键选择此项，然后按向右箭头键或仅单击向右箭头在此项目旁边。

    按左箭头键，或Escape隐藏建议列表。

  5. 在检查选项列表中，选择所需的抑制操作。在相应的代码段中，将通过特殊注释取消检查。

从“检查结果”工具窗口中取消检查

  1. 运行代码分析后，在“ 检查结果工具”窗口中选择要取消检查的代码问题。

  2. 点击按钮禁止和选择sustension的范围，或者只需右键单击选定的检查。

  3. 选择所需的抑制动作。

    在相应的代码段中，将通过特殊注释取消检查。

更改文件的突出显示级别

默认情况下，CLion突出显示所有检测到的代码问题。您可以为当前使用的文件更改此行为。请点击在状态栏中（右下角），然后选择以下选项之一：

  • 无 ：关闭高光显示。

  • 语法 ：仅突出显示语法问题。

  • 检查 ：（默认）突出显示语法问题和检查发现的问题。

您也可以从主菜单更改突出显示级别。选择，或按Ctrl+Shift+Alt+H 。
