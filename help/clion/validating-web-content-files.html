验证Web内容文件

CLion执行两种不同的有效性检查：

  • 动态验证适用于所有Web内容文件，并在您编辑文件时执行。CLion检查格式是否正确，即检测各种违反语法要求的行为，例如未关闭的标签，错误的结束标签名称，重复的标签，未解析的链接等。在编辑器中突出显示所有遇到的错误。

    但是，这种形式的代码验证是相当软的 ，也就是说，并未考虑所有要求。

  • 全面验证除了结构检查外，还涉及结构验证。完全验证可用于与XSD（XML架构定义）架构相关联或包含数据类型定义（DTD）的文件 。CLion检查XML文件的结构是否符合相应DTD或Schema中定义的结构。

    完整验证的结果以消息视图的形式提供。

选择默认的HTML语言级别

通常，HTML或XHTML文件具有声明，指出文件中源代码中使用的语言级别 。此语言级别用作验证文件内容所依据的标准。如果HTML或XHTML文件没有声明后，将根据默认标准（模式）验证文件的内容。

  1. 在 设置/首选项对话框Ctrl+Alt+S ，单击“ 语言和框架”下的“ 架构和DTD ”，然后单击“ 默认XML架构” 。将打开“ 默认XML模式”页面。

  2. 在“ 默认HTML语言级别”区域中，选择默认架构以验证HTML和XHTML文件而无需宣言。可用的选项有：

    • HTML 4或HTML 5 ：选择以下选项之一，将文件视为HTML 4或HTML 5，并根据其中一种标准进行验证。

    • 其他doctype ：选择此选项可让HTML文件在默认情况下再次验证自定义DTD或架构，并指定要使用的DTD或架构的URL。

      请注意，此字段中提供代码完成：按Ctrl+Space查看建议的网址列表。

  3. 选择XSD（XML架构定义）架构以验证XML文件。可用的选项有：

    • XML Schema 1.1有关详细信息，请参见W3C XML Schema定义语言（XSD）1.1第1部分：结构和W3C XML Schema定义语言（XSD）1.1第2部分：数据类型 。

    • XML Schema 1.0有关详细信息，请参见XML Schema第1部分：结构第二版和XML Schema第2部分：数据类型第二版 。

选择默认架构以验证XML文件

  1. 在 设置/首选项对话框Ctrl+Alt+S ，单击“ 语言和框架”下的“ 架构和DTD ”，然后单击“ 默认XML架构” 。将打开“ 默认XML模式”页面。

  2. 在“ 默认XML模式版本”区域中， 选择XSD（XML架构定义）架构以验证XML文件。可用的选项有：

    • XML Schema 1.1有关详细信息，请参见W3C XML Schema定义语言（XSD）1.1第1部分：结构和W3C XML Schema定义语言（XSD）1.1第2部分：数据类型 。

    • XML Schema 1.0有关详细信息，请参见XML Schema第1部分：结构第二版和XML Schema第2部分：数据类型第二版 。

在XML文件上运行完整验证

  1. 在编辑器中打开所需的XML文件，或仅在“ 项目”工具窗口中选择它。

  2. 从上下文菜单中，选择Validate 。
