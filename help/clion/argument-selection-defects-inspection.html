检查：参数选择缺陷

设置/首选项|编辑器检查 -C / C ++，常规

默认严重性： 警告

当您调用具有相同类型的多个参数的函数时，您可能会意外地交换它们，因此会遇到可能难以调试的错误。Argument Selection Defects检查可以帮助您发现并快速修复此类错误，而无需编译代码（原始启发式算法可以在本文中找到）：

如果基于Clangd的语言引擎已打开，则默认情况下启用此检查。您可以配置其严重性和范围，或在“ 检查设置”对话框中关闭检查 （选择C / C ++ ，“ 常规” ，“ 参数选择缺陷” ）。

请注意，如果您禁用基于Clangd的引擎，则此检查仅适用于捆绑的（默认） Clang-Tidy二进制文件 。

功能范围

参数选择缺陷检查会检测何时以错误的顺序将两个或多个相同类型的参数传递给函数。对于这种情况，检查可以处理任何类型 ，包括用户定义的类型 。此外，检查是耐错字的 ，因此参数不必完全相同。

限制条件

为避免误报，“参数选择缺陷”检查不会在以下情况下警告您：

  • 该函数是递归的

  • 大概交换的参数是可变参数，例如：

    无效f（int left，...）; void b（int left，int right）{f（right，left）; }

  • 函数或参数名称包含以下子字符串： swap ， reverse ， inverse ， flip ， 向后或rotate

  • 参数名称短于三个符号（例如x, y, ab ）

  • 多次调用该函数，并且以不同的顺序传递相同的参数，例如：

    case LANDSCAPE：位图=新位图（宽度，高度）；休息;情况肖像：位图=新位图（高度，宽度）； ...
