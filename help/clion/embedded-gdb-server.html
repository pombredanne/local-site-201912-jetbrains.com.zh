嵌入式GDB服务器

要使用GDB服务器在芯片上进行调试，请使用涵盖以下用例的专用嵌入式GDB服务器运行/调试配置：

  • ST-Link GDB服务器（您可以在GitHub上找到该工具的开源版本）。

  • Segger J-Link GDB服务器。

  • Qemu作为GDB服务器。

  • OpenOCD作为独立的GDB服务器，可以替代OpenOCD下载和运行配置。

  • 符合GDB服务器协议的任何其他板或特定的GDB存根。

当前，此配置仅适用于基于CMake的项目。

必备工具

  1. 工具链

    嵌入式GDB服务器配置需要预先配置跨编译器工具链。

    对于ARM Cortex-M和Cortex-R MCU，请安装GNU ARM工具链 。安装后，确保工具链显示在系统路径中：run arm-none-eabi-gcc从命令行，您的系统应该可以识别此命令。

    对于所有其他平台，请安装特定于平台的工具链，然后在“设置/首选项” |“设置”中设置工具。 构建，执行，部署|工具链或您的CMake脚本中（请查看模板CMakeLists.txt ）。

    另外，您需要提供链接器文件：从芯片供应商处获取它或手动创建它，然后通过CMake设置进行指定。

    如果您正在使用STM32CubeMX项目 ，则安装GNU ARM工具链是项目设置的一部分，并且CubeMX会自动生成链接器文件。

    Windows特定要求：

    • 仅支持MinGW和MinGW-w64环境（ CPP-15034 ）。

    • 对于GNU ARM工具链 ，请使用版本2019-q3或2018-q2（或更早版本）。见问题与2018-Q4。

  2. GDB客户端

    GDB客户端是在CLion一侧运行并连接到远程GDB服务器的调试器。您可以将捆绑的GDB或工具链的调试器用作GDB客户端。

    请注意，LLDB（macOS的默认调试器）不能用作GDB客户端。

    默认情况下，配置使用在当前CMake配置文件的工具链中选择的调试器。您可以在“设置/首选项” | “工具”中选择的工具链的“ 调试器”字段中检查或更改它。 构建，执行，部署| CMake的 。

  3. GDB服务器

    所选的GDB服务器应安装在您的计算机上。无需手动启动它-启动调试会话时，配置将自动运行GDB服务器 。

创建嵌入式GDB服务器配置

去运行|编辑配置 ，单击并选择Embedded GDB Server 。

配置以下设置：

  • “目标远程”参数

    提供介质来携带调试数据包（串行线或使用TCP或UDP的IP网络）。有关连接到远程目标的更多详细信息，请参见gdb文档 。

  • GDB服务器

    指定用作GDB服务器的二进制文件。

  • GDB服务器参数

    提供GDB服务器特定的参数（例如，端口号或板配置文件）。

  • 高级选项：重置命令

    按下时要发送的命令在调试会话期间。

  • 高级选项：启动延迟

    启动GDB服务器之后，CLion在尝试连接之前将等待的时间。

调试嵌入式GDB服务器配置

调试时 在此配置中，CLion执行一系列步骤：

  1. 在指定的环境中启动GDB服务器。

  2. 等待，直到配置的启动延迟时间过去。

  3. 启动GDB客户端并连接到GDB服务器。

  4. 按照配置设置中“ 下载”选项指定的方式上传二进制文件。

  5. 启动调试会话。
