CLion 2019.3帮助

单步执行

CLion为各种调试策略提供了一组步进操作（例如，您是否需要直接转到下一行或输入在那里途中调用的函数）。

步进按钮位于“ 调试”窗口工具栏上。

越过

即使当前行包含函数调用，也将单步跳过当前行并带您到下一行。呼叫被跳过，您直接移至呼叫者的下一行。

  • 点击“ 跳过”按钮或按F8 。

如果跳过的方法中有断点，调试器将在它们处停止。如果需要跳过途中的任何断点，请使用“ 强制跨过” 。

步入

进入被调用函数的代码。

  • 单击单步执行按钮或按F7 。

由于通常可能不需要调试它们，因此某些方法会被Step跳过。此列表可以在“ 构建，执行，部署” |“发布”中进行调整。 调试器步进页面的 设置/首选项对话框Ctrl+Alt+S 。

明智地进入

当一行上有多个方法调用，并且您想具体说明要输入哪种方法时， 智能步入很有用。此功能使您可以选择感兴趣的方法调用。

在CLion中，此功能不适用于C / C ++调试。但是，您可以将其用于其他语言，例如JavaScript和Python。

  1. 从主菜单中，选择运行|智能进入或按Shift+F7 。

  2. 单击方法或使用箭头键选择它，然后按Enter / F7 。

每次在线上有多个方法调用时，都可以配置要使用的智能步进而不是常规步进 。这是在“设置/首选项” |构建，执行，部署|调试器踩 。

走出去

退出当前功能，然后转到调用者的代码。

  • 点击“ 跳出”按钮或按Shift+F8 。

运行到光标

继续执行直到插入符号的位置。

  1. 将插入号放在您要程序暂停的行上。

  2. 单击运行到光标按钮或按Alt+F9 。

另外，您也可以单击运行游标 。为此，请单击装订线中的行号。

您可以配置是否要运行“游标”以单击“设置/首选项” |“行”中的行号。 构建，执行，部署|调试器 。

如果需要跳过途中的任何断点，请使用“ 强制运行至光标” 。

强制进入

步入函数，即使它跳过由常规步骤默认情况下进入 。

  • 单击“ 强制进入”按钮或按Shift+Alt+F7 。

如果要使用的功能的源代码不可用，则可以在专用视图中调试反汇编的代码 。

强制运行到光标

继续执行直到插入符号的位置。途中的所有断点都将被忽略。

  1. 将插入号放在您要程序暂停的行上。

  2. 从主菜单中，选择运行|强制运行至光标或按Ctrl+Alt+F9 。

强制越过

即使当前突出显示的行中包含方法调用，也将单步执行到当前代码行并带您到下一行。如果调用的方法中有断点，则将其忽略。

  • 从主菜单中，选择运行|强制跨步或按Shift+Alt+F8 。
