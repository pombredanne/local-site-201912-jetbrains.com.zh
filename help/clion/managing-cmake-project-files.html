新增档案

  2. 指定新文件的名称，类型和其他选项 。

    对于C ++ Class ， C / C ++ Source和C / C ++ Header文件模板，CLion将提示您将新文件添加到一个或多个现有CMake目标中：

    如果清除“ 添加到目标”复选框，则始终可以手动将新文件添加到目标（例如，在为该文件创建另一个目标时）。有关更多详细信息，请参见快速CMake教程：添加目标 。

  3. 如果在上一步中选择了目标，则新条目将自动添加到CMakeLists.txt中 ：

管理包含的文件

  • 当您将位于项目根目录下的头文件或源文件包含在任何项目文件中时，CLion也会将其视为项目文件。无需在CMakeLists.txt中手动列出此类包含。

    这不适用于位于项目root之外的标准库或第三方库。

添加不包含的文件

当任何项目文件或CMake目标中未包含源文件或头文件时，CLion会通知您：

在这种情况下，您需要手动将文件指定为一个项目。根据文件位置使用以下方法之一：

  • 如果文件位于项目根目录下 ，则使用将该文件添加到目标中add_executable()要么add_library()命令。例如：

    之前

    add_executable（TestProject main.cpp）

    后

    add_executable（TestProject main.cpp new_file.cpp）

    当包含非项目源的目录位于项目根目录下时，您也可以将其标记为库：在“项目”视图中单击鼠标右键，然后选择“将目录标记为|”。从上下文菜单中选择库文件 。

  • 要使用位于项目根目录之外的库文件，您需要指定标题搜索路径，以便CLion可以索引目录并为目录提供完成和导航。 #include声明：

    include_directories（$ {MY_EXTERNAL_SOURCE__DIR}）

    请参阅《 快速CMake教程：添加包含目录》以获取更多信息。
