通过GDB / gdbserver进行远程调试

通过GDB / gdbserver进行远程调试对于在目标上构建昂贵或不可用的情况特别有用：将二进制文件放在目标上并在gdbserver下启动后，您可以从CLion连接到它并获得完整的IDE调试经验，例如您的程序在本地运行。

CLion提供了两种用于远程GDB / gdbserver调试的配置： 远程GDB服务器和GDB远程调试 。

  • 如果您使用CMake项目，并且希望CLion自动生成二进制文件并将其上传到远程计算机，请使用远程GDB服务器 。此配置还将为您启动gdbserver下的程序 ，因此您无需手动执行该操作。

  • 如果您已有可执行文件和符号文件，请使用GDB远程调试 。此配置独立于特定的构建系统或项目格式。请注意，您将需要同步文件并在gdbserver下手动启动程序。
