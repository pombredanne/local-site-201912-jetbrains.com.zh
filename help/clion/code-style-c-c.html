代码样式。C / C ++

档案|设置|编辑器代码样式|适用于Windows和Linux的C / C ++
CLion |首选项|编辑器代码样式|适用于macOS的C / C ++
Ctrl+Alt+S

使用此部分为C / C ++文件配置代码样式选项。更改这些设置时，更改结果将显示在“ 预览”窗格中。

设置为

单击此链接，从显示的列表中选择当前语言默认代码样式的基础。该列表包含以下选项：

项目

描述

语言

选择此选项可从另一种语言继承代码样式设置。从打开的列表中选择源语言。这样，仅采用适用于当前语言的设置。所有其他设置均不受影响。

预定义样式

选择此选项可以使用一种著名的代码样式准则。从列表中选择以下标准之一。

注意：每种代码样式仅影响基本设置的某些项目，其余部分保持不变。

  • 谷歌
  • 虚拟机
  • 本地数据库
  • t
  • GNU
  • 肉酱

以下样式仅适用于括号放置：

  • 奥尔曼牙套
  • 铁匠括号
  • K＆R大括号

制表符和缩进

项目

描述

使用制表符

  • 如果选中此复选框，则使用制表符：

    • 在按Tab键

    • 缩进

    • 用于代码重新格式化

  • 清除复选框后，CLion使用空格而不是制表符。

智能标签

  • 如果选中此复选框，则由代码块嵌套定义的缩进部分由制表符和（如有必要）空格组成，而由对齐方式定义的缩进部分仅由空格组成。

  • 如果清除此复选框，则仅使用选项卡。这意味着将将一组适合指定制表符大小的空格自动替换为一个制表符，这可能会导致中断精细对齐。

如果选中了“ 使用选项卡字符”复选框，则“ 智能选项卡”复选框可用。

标签尺寸

在此字段中，指定选项卡中包含的空格数。

连续缩进

在此字段中，在表达式，方法声明和方法调用中，指定要在数组元素之间插入的空格数（如果选中了“ 使用制表符”复选框，则为制表符 ）。

缩进空行

如果选中此复选框，则CLion将使空行缩进，就好像它们包含一些代码一样。
如果清除此复选框，CLion将删除制表符和空格。

标签缩进

在此字段中，指定要在label语句之前的下一行插入的空格数（如果选中了“ 使用制表符”复选框，则为制表符 ）。

绝对标签缩进

如果选中此复选框，则标签缩进将计为绝对空格数。否则，标签缩进相对于先前的缩进级别进行计数。

缩进lambda

在此字段中，输入lambda表达式的缩进。

缩进平面结构的成员

在此字段中，键入普通结构成员的缩进。

缩进班级成员

在此字段中，输入类成员的缩进。

在类/结构中缩进可见性关键字

在此字段中，输入可见性关键字的缩进（ public ， protected等）。

缩进名称空间成员

在此字段中，输入名称空间成员的缩进。

预处理程序指令缩进

在此字段中，输入预处理程序指令的缩进量。

遵循代码缩进

选中此复选框，以使预处理程序指令的缩进与周围代码行的缩进相等。

空间

使用此选项卡可以指定代码中需要空格的位置。要让CLion自动在某个位置插入空格，请在列表中选中该位置旁边的复选框。结果显示在“ 预览”窗格中。

包装和大括号

在此选项卡中，定制代码样式选项，CLion将在重新格式化源代码时应用这些选项。左侧窗格包含异常列表（ 重新格式化时保留 ），以及各种代码结构（列表，语句，操作，注释等）的放置和对齐选项。右侧窗格显示预览。

对齐优先于缩进选项。

右边距（列）

使用“ 硬包装在”字段可以指定元素右侧所需的页边空白空间。如果选择默认选项，则使用全局设置中右边距的值。

包装打字

使用打字设置环绕到指定编辑的文本是怎样安装在指定的硬包装的 。您可以选择以下选项之一：

  • 默认值 ：在这种情况下，CLion使用全局设置中指定的“ 键入时自动换行”选项。

  • 是 ：在这种情况下，CLion使用“ 右边界”字段中指定的值。

  • 否 ：在这种情况下，此选项已关闭，并且一行可以超出右边界中指定的值。

视觉指南

使用“ 视觉参考线”字段来指定多个右边距。您可以保留默认值，也可以输入空白。如果要指定多个边距，请输入用逗号分隔的数字。

重新格式化时保持

使用复选框配置CLion在重新格式化源代码时将产生的异常。例如，默认情况下，“ 换行符”复选框处于选中状态。如果您的代码包含比标准约定短的行，则可以在重新格式化源代码之前通过禁用换行符复选框来转换它们。

对齐选项

项目

描述

多行时对齐

如果选中此复选框，则代码构造将从下一行的同一列开始。否则，代码构造的位置由当前的缩进级别确定。

多行时对齐括号

如果选中此复选框，则带括号的代码构造将在下一行的左括号的列处开始。否则，代码构造的位置由当前的缩进级别确定。

新行上的“ else”

使用此复选框可将相应的语句或字符移至下一行。

特别待遇

如果选中此复选框， else if语句位于同一行。

除此以外， else if语句移动到相应缩进级别的下一行。

缩进案例分支

如果选中此复选框，则case语句位于相应的缩进级别。除此以外， case语句与switch 。

大括号放置选项

项目

描述

大括号放置样式

使用此列表可以在类声明 ， 函数声明和其他类型的声明中指定左括号的位置。可用的选项有：

  • 行尾 ：选择此选项可将开括号放在声明行的末端。

  • 如果被换行，则换行 ：选择此选项可将开括号放在多行声明行之后的行首。

  • 下一行 ：选择此选项可将开头括号放在声明行之后的行开头。

  • 转移下一行 ：选择此选项可在将声明行转移到相应的缩进高度后，将开括号放在该行。

  • 每次移动下一行 ：选择此选项可在将声明行移动到相应的缩进级别之后将开括号放在该行，并将下一行也移动到下一个缩进级别。

大括号

从此列表中选择括号的引入方法if ， for ， while和do () while陈述。可用的选项有：

    空行

    使用此选项卡可定义重新格式化后希望CLion保留并插入代码中的空白行的位置和数量。对于每种类型的位置，指定要插入的空白行数。结果显示在“ 预览”窗格中。

    项目

    描述

    保留最大空白行

    在此区域中，指定在指定位置重新格式化后要保留的空白行数。

    最小空白行

    在该区域的文本框中，指定要在指定位置显示的空白行数。

    这些设置不会影响第一项之前和最后一项之后的空白行数。

    代码生成选项卡

    项目

    描述

    一般

    • 第一列的行注释 ：如果选中此复选框，则行注释将在第一列对齐；否则，它们将与代码对齐。

    • 第一列的块注释 ：如果选中此复选框，则块注释将在第一列对齐；否则，它们将与代码对齐。

    • 在块注释中缩进新行 ：如果选中此复选框，则将在块代码的下一行缩进；否则，它将与块的第一行对齐。

    • const / volatile修饰词放置 。该设置定义代码生成期间的关键字顺序。

      选择const int x = 0单选按钮，将const / volatile限定符放在类型说明符之前。
      选择int const x = 0单选按钮，将const / volatile限定符放在类型说明符和符号名称之间。

    C ++

    • 字段前缀 ：输入一个前缀，该前缀将在自动生成的字段的名称之前。

    • 字段的后缀：键入一个后缀，该后缀将附加到自动生成的字段的名称中。

    • 吸气剂的前缀：输入一个前缀，该前缀将在自动生成的吸气剂的名称之前。默认名称是get 。

    • setter的前缀：键入将在自动生成的setter的名称之前的前缀。默认名称是set 。

    • 产生！=和==一起 ：选中此复选框可在生成相等运算符时将此设置作为“ 选择字段”对话框的默认设置。

    • 生成>，<=，> =以及< ：选中此复选框，以在生成关系运算符时将此设置作为“ 选择字段”对话框的默认设置。

    • 将“ virtual”属性与“ override”一起插入 ：选中此复选框以允许为生成的继承函数将“ virtual”属性与“ override”说明符一起插入。从C ++ 11标准或更高版本开始，此选项可用。

    • 使用C ++强制转换运算符 ：如果选中，快速修复将在适当时插入C ++强制转换运算符，而不是C样式强制转换。

    文档注释

    • 添加@brief标签 ：选中此复选框以启用@brief在Doxygen样式注释中标记。

    • 在行注释中标记前缀 ：单击@param或\ param单选按钮以选择首选的前缀类型。

    • 在块注释中标记前缀 ：单击@param或\ param单选按钮以选择首选的前缀类型。

    新文件扩展名选项卡

    在此选项卡中，您可以定义“ 创建新的C / C ++源文件”对话框中使用的源/头文件类型。此选项卡还确定了它们在该对话框的“ 类型”字段中出现的顺序：选项卡中的第一对是对话框中类型列表中的第一个扩展名。

    描述

    标头扩展

    文件命名约定

    从下拉列表中选择首选的命名约定。此设置定义创建新类时在“ 新建C ++类”对话框中生成的文件名的大小写。
