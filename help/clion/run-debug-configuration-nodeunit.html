运行/调试配置：NodeUnit

运行编辑配置| |节点单元

仅当在“ 插件”页面上安装并启用了NodeJS存储库插件时，该对话框才可用，有关详细信息，请参阅管理插件 。

自2018年5月起，不推荐使用Nodeunit。有关详细信息，请参见Nodeunit官方网站。

在此对话框中，创建配置以运行Node.js应用程序的单元测试。

在你开始前

  1. 下载并安装Node.js。

  2. 通过以下方式之一安装Nodeunit测试框架：

    • 从https://github.com/caolan/nodeunit下载该框架，并根据官方说明进行安装。

    • 安装nodeunit如npm和Yarn所述，使用Node Package Manager（NPM）来打包 程序 。

  3. 在“设置/首选项”中安装Node.js插件。 如插件中所述的插件 。

NodeUnit特定的配置设置

项目

描述

节点解释器

在这个领域里， 指定要使用的Node.js解释器。 如果选择项目别名，CLion将自动从Node.js和NPM页面上的Node解释器字段中使用项目默认解释器 。

在大多数情况下，CLion会检测项目的默认解释器并填写字段本身。您还可以选择其他已配置的本地解释器或单击并配置一个新的。

工作目录

在此字段中，指定要在其中查找测试的文件夹。这可以是项目根文件夹，也可以是测试文件夹的父目录。手动输入路径或单击浏览 并在打开的对话框中选择位置。

环境变量

 在此字段中，为Node.js可执行文件指定环境变量 （如果适用）。 点击浏览 在字段右侧，并在“ 环境变量”对话框中配置变量列表，将打开：

  • 要定义新变量，请单击并指定变量名称和值。

  • 要放弃变量定义，请在列表中选择它，然后单击 。

  • 准备好后，单击确定 。

变量的定义显示在“ 环境变量”只读字段中，以分号作为分隔符。可接受的变量是：

  • NODE_PATH ： 一种: -前缀为模块搜索路径的目录的列表。

  • NODE_MODULE_CONTEXTS ：设置为1可以在各自的全局上下文中加载模块。

  • NODE_DISABLE_COLORS ：设置为1以禁用REPL中的颜色。

节点单元模块

在此文本框中，指定Nodeunit框架的安装文件夹。手动输入路径或单击然后在打开的对话框中选择文件夹。

跑

从此下拉列表中，选择要执行的测试范围。可用的选项有：

  • 目录中的所有测试文件：选择此选项可运行文件夹中的所有测试文件，并指定该文件夹相对于工作目录的路径。

  • 测试文件：选择此选项可从特定文件执行测试，然后键入该文件相对于工作目录的路径。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

允许并行运行

如果选中此复选框，则可以并行启动运行配置多次，而不必重新运行它。每个运行程序都将在“ 运行工具窗口”或“ 调试工具窗口 ”的自己的选项卡中启动 。

禁用后，每次启动运行/调试配置的新实例时，CLion都会检查同一运行/调试配置的其他实例是否存在，并显示确认对话框。如果在确认对话框中单击“ 确定” ，则将停止运行器的第一个实例，而将其替换。当启动两个相同类型的运行/调试配置实例时，这会占用过多的CPU和内存资源。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在CLion中定义它，请添加其定义。有关更多信息，请参见外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 运行文件监视程序 ：选择此选项可使CLion应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定在其中定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

Alt+Delete

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（这些任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
