CLion 2019.3帮助

当地历史

CLion使用本地历史记录不断跟踪对项目所做的所有更改。与版本控制系统显示特定提交之间的差异不同， 本地历史记录会自动维护所有有意义事件的修订。这样就产生了项目结构和源代码更改的详细时间表，并可以在必要时回滚到任何地方。

请注意，本地历史记录不能替代适当的源代码控制。当您安装新版本的CLion或使缓存无效时，将清除它。它还具有保留期和最大大小，因此不能保证修订能够持久。

查看当地历史

  • 选择VCS |当地历史|从主菜单显示历史记录 。

本地历史记录包含修订列表，并显示它们之间的差异。查看目录的本地历史记录时 ，修订版仅显示目录中更改的文件的名称。特定文件的修订包含对文件内容的更改。

本地历史记录不适用于二进制文件的内容。

还原为特定版本

  • 从列表中选择修订，然后从上下文菜单中调用“ 还原 ”或单击在工具栏上。

这为“ 本地历史记录”添加了新修订。

查看本地历史记录的源代码元素

在某些情况下，您可能不想查看整个文件的“ 本地历史记录 ”，而只希望查看特定的源代码元素：类，其成员（字段和方法）或任何选定的文本片段。

查看课程的本地历史记录

  • 右键单击编辑器中的类名称，指向“ 本地历史记录” ，然后单击“ 显示类的历史记录” 。

查看班级字段的本地历史记录

  • 右键单击编辑器中的字段名称，指向“ 本地历史记录” ，然后单击“ 显示字段的历史记录” 。

查看本地历史记录以获取类方法

  • 在编辑器中右键单击方法名称，指向“ 本地历史记录” ，然后单击“ 显示方法的历史记录” 。

查看本地历史记录以获取源代码片段

  • 在编辑器中右键单击所选文本，指向“ 本地历史记录” ，然后单击“ 显示所选历史记录” 。

将标签添加到本地历史记录

本地历史记录包含带有时间戳的修订，这些修订不容易浏览。根据预定义的事件，某些修订会自动带有标签标记：运行测试，部署应用，提交更改等。您还可以添加自定义标签来标记“ 本地历史记录”中的特定里程碑。

在当前版本中添加标签

  1. 在VCS菜单上，指向“ 本地历史记录” ，然后单击“ 放置标签” 。

  2. 在“ 放置标签”对话框中，键入标签名称，然后单击“ 确定” 。

分享当地历史

本地历史记录不支持共享访问，仅用于个人使用。但是，您可以创建一个补丁文件，其中包含相对于特定修订版本的更改，您可以将其与他人共享。

产生补丁

  • 在“ 本地历史记录”中选择相关的修订，然后在工具栏上单击“ 创建补丁 ”。

查看最近的变化

除了详细的本地历史记录外 ，CLion还提供了所有项目近期主要变更的摘要。

查看最近的变化

  • 在视图菜单上，单击最近的更改 。

您可以选择感兴趣的更改以查看差异并在必要时将其还原。

本地历史记录的保留期

默认情况下，“ 本地历史记录”配置为存储最近5个工作日 （即文件被修改的日期）的修订。

更改本地历史记录的保留期限

  1. 在帮助菜单上，单击查找操作或按Ctrl+Shift+A 。

  2. 查找并打开注册表编辑器。

  3. 修改值localHistory.daysToKeep参数。

  4. 单击“ 关闭”，然后重新启动CLion，以使更改生效。

如果要完全禁用本地历史记录 ，请将保留期限设置为0 。

或者，您可以通过localHistory.daysToKeep通过配置JVM选项将参数作为Java属性。例如，要将保留期限设置为30天，请将以下行添加到JVM选项文件中：

-DlocalHistory.daysToKeep = 30

本地历史记录文件的位置

本地历史记录以二进制文件的形式存储在CLion系统目录的LocalHistory子目录下：

句法
％HOMEPATH％\。 \系统
例
C：\ Users \ JohnS \。CLion2019.3 \ system
句法
〜/库/缓存/
例
〜/库/缓存/CLion2019.3
句法
〜/。 /系统
例
〜/。CLion2019.3 /系统

您可以使用idea.system.path属性更改系统目录的位置。
