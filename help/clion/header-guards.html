标头护卫

标头防护是预处理器指令的一种模式，可以防止标头多次被包含。标头防护将整个代码内容包装到一个#ifndef （ #if !defined ，或其他类似的）块：

#ifndef MY_HEADER_H＃定义MY_HEADER_H // ...＃万一

编辑头文件模板

默认情况下， 文件模板中包含标题保护，这些标题指定新标题的初始内容。例如，如果您决定一次使用#pragma而不是标题防护，则可以编辑这些模板。为此，请转到设置/首选项|编辑器文件和代码模板，然后打开文件选项卡。从列表中选择C头文件或C ++类头 ，然后更改模板内容：

注意文字语法包装#pragma once 。对于文件模板，CLion使用Apache Velocity语言，该语言需要转义C / C ++预处理程序指令才能正确解析它们。

配置标题保护符号

  • 标头警卫符号（ MY_HEADER_H （来自上面的示例）应该是唯一的，因此它通常与文件名有关。要配置标题保护符号的模式，请在“设置” /“首选项” |“设置 ”中打开“ 命名约定”选项卡。 编辑器代码样式| C / C ++并在Header Guard字段中指定模板：

  • 您可以使用各种预定义变量，例如：

    • ${PROJECT_NAME} -当前项目的名称。

    • ${PROJECT_REL_PATH} -相对目标路径。例如，如果项目位于prj目录中，并且目标文件是prj / src / dir / header.h ，则${PROJECT_REL_PATH}将等于src / dir 。

    • ${FILE_NAME} -目标文件名，不带扩展名。

    • ${EXT} -目标文件扩展名。

    • 您可以在文件模板中找到的其他变量，例如${USER}要么${DATE} 。

    有效的标题保护符号可以包含以下字符 ：大写'A-Z'和小写'a-z'字母，下划线符号_ ，数字（但不能以数字开头）和美元符号$ 。请注意，CLion会附加一个INC_以数字开头的符号的前缀，并使用替换无效的符号INC_${UUID}图案。

  • 配置的符号替换${INCLUDE_GUARD}标题模板中的变量。创建新的C / C ++类和头文件时，CLion将应用该模式。另外，如果重命名类或文件，则与当前配置的样式匹配的标题保护将使用新名称进行更新：
