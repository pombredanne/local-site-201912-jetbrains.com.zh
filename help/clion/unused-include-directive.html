检查：未使用的包含指令

设置/首选项|编辑器检查 -C / C ++，未使用的代码

默认严重性： 警告

基于Clangd的引擎

您的代码中包含不必要的头文件会减慢编译速度，并且可能污染全局名称空间，甚至会导致假装冲突。其中包括添加不必要的依赖关系，使您的代码更难于浏览和维护（例如，在代码完成中您会看到很多无关的结果）。

未使用的包含指令检查可检测到#include不需要且可以安全删除的指令：

检查不会警告您有关包含编译错误的源文件中的内容。此外，它仅检查包含以下内容的标头#pragma once或标题防护 -没有文件防护的文件始终标记为已使用。

当基于Clangd的语言引擎打开时，默认情况下启用检查。如果禁用Clangd引擎，则未使用的包含指令仍将起作用，但是在这种情况下可能会产生误报，并且所选策略不会影响检查的行为：当不使用Clangd时，它建议删除最大数量的未使用指令（ 检测非必需选项）。同样，在这种情况下，检查会检查所有标头，无论它们是否具有标头护罩或#pragma once 。

配置检查

  1. 转到设置/首选项|编辑器检查并导航到检查列表中的C / C ++，未使用代码，未使用包括指令 。

  2. 在“ 选项”字段中，选择用于检测和删除未使用的诊断策略，包括：

    • 检测完全未使用 ：最保守的策略，仅检测#include带有未在翻译单元中使用的声明的指令。

    • 检测未直接使用（默认） ：此策略遵循“ 包括您使用的内容”原理（如果您使用标头中的符号或类型，则应包含该标头的原理）并检测#includes带有直接在当前文件中不使用的声明的指令。

    • 不需要检测 ：最积极的策略，可检测到最小的子集#include指令以保持文件可编译，并建议删除所有其余文件。

  3. 如果需要，请设置在头文件中运行检查复选框。

    启用此选项后，检查将跳过伞形标题 ，因为它们中列出的指令始终被视为已使用（ 伞形标题一次包含所有必要的标题，并且不包含任何声明）。
