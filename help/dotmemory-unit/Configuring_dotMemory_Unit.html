dotMemory Unit 3.0帮助

配置dotMemory单元

默认的dotMemory Unit配置允许“开箱即用”使用它。但是，在某些情况下，您可能需要重新定义特定的选项，例如工作空间文件的位置或自动保存的工作空间的总大小限制。这是在以下人员的帮助下完成的DotMemoryUnitAttribute放在程序集，测试类或测试方法之前。

例

[DotMemoryUnit(SavingStrategy = SavingStrategy.OnCheckFail, Directory = @"C:\tmp\Method")] [Test] public void TestMethod1() { ... }

DotMemoryUnitAttribute

DotMemoryUnitAttribute属性

名称类型描述
CollectAllocations bool 定义是否必须收集内存分配数据（内存流量和创建堆栈跟踪数据）。默认， false 。收集分配数据会大大增加分析开销。仅在需要测量内存流量时才使用它。
请注意， AssertTraffic属性可在标记的方法中自动收集分配。您不需要添加[DotMemoryUnit(CollectAllocations = true)]在这种情况下的属性。
Directory string 必须保存工作空间文件的目录的路径。这可以是完整路径，也可以是相对于声明测试方法的程序集的相对路径。请注意，为简单起见，您可以将许多占位符用于特殊目录：
  • -当前用户文件夹，例如C：\ Users \ [用户名]
  • -本地应用程序数据文件夹，例如C：\ Users \ [用户名] \ AppData \ Local
  • -程序数据文件夹，例如C：\ ProgramData
DiskSpaceLimit int 可用于存储工作空间的最大磁盘大小（以字节为单位）。默认值为200 MB。在保存工作空间之前，dotMemory Unit将检查会话中所有先前保存的工作空间的总大小。如果尚未超过限制，则将保存工作空间（即使超过限制）。
FailIfRunWithoutSupport bool 如果false ，全部dotMemory.Check如果在不支持dotMemory Unit的情况下进行测试，则调用将被忽略。
如果true ，测试包含dotMemory.Check如果在不支持dotMemory Unit的情况下运行此测试，则调用将失败。
默认值为true 。
SavingStrategy SavingStrategy
enum
{OnAnyFail, OnCheckFail, Never} 		定义自动保存工作空间的条件：
  • OnAnyFail -在任何测试失败时自动保存工作区。
  • OnCheckFail -自动保存工作区dotMemory.Check通话失败。
  • Never -不要自动保存工作区。
WorkspaceNumberLimit int 测试失败时可以保存的最大工作空间数。预设值为5。
