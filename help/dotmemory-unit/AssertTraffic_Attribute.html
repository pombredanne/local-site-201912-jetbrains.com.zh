简单方案。AssertTraffic属性

对于有关流量的简单断言，您可以使用AssertTraffic属性。它使您可以断言测试分配的对象或内存量少于特定数量。
这种方法的主要局限性是针对整个单元测试并且仅针对分配的对象计算流量。因此，如果要在特定时间间隔检查流量或对收集的对象进行断言，则应使用流量类型。

您可以添加任意数量的AssertTraffic属性，以防您想按多种条件过滤流量。

例

// Assert the test allocates no more than 1000 strings [AssertTraffic(AllocatedObjectsCount = 1000, Types = new[] {typeof(string)})] [Test] public void TestMethod1() { ... }

AssertTraffic属性

*请注意，您的代码可以生成系统类型的对象，例如string ， int ， 等等。所有这些对象都属于System命名空间。

名称类型描述
AllocatedSizeInBytes long 声明已分配对象的大小（以字节为单位）。
AllocatedObjectsCount int 声明分配的对象数。
Types Type[] 按类型选择要声明的对象。
Interfaces Type[] 通过实现的接口选择要声明的对象。
Namespaces string[] 通过名称空间*选择要声明的对象。
支持星号（*）通配符。
