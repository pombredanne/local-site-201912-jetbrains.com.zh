dotMemory Unit 3.0帮助

开始使用

请注意，对于使用ReSharper单元测试运行器的情况，将给出以下步骤。有关如何使用独立dotMemory Unit启动器运行测试的详细信息，请参阅《 使用dotMemory Unit独立启动器》 。

1。将dotMemory单元安装到测试项目中

*此命令将安装最新的稳定dotMemory Unit版本。如果要安装最新的EAP版本，请使用Install-Package JetBrains.DotMemoryUnit -pre命令。

  1. 在Visual Studio中打开测试项目。
  2. 使用菜单工具|打开NuGet控制台。 NuGet软件包管理器|软件包管理器控制台 。
  3. 通过运行命令*安装dotMemory Unit软件包：
    PM> Install-Package JetBrains.DotMemoryUnit

之后，ReSharper的单元测试菜单将在dotMemory Unit下获得其他项Run Unit Tests 。

2。编写内存测试

  • 创建一个新测试或通过添加“内存”声明来修改现有测试。

我们强烈建议您在“ 示例”部分中熟悉最常用的测试示例 。这将使您简要了解应如何编写“内存”测试。

3。运行测试

  • 要在dotMemory Unit支持下运行所有测试，请使用菜单ReSharper |单元测试|在dotMemory Unit下运行所有测试 。
    如果只想运行特定的测试，请在装订线中单击测试标记，然后在dotMemory Unit下选择Run 。请注意，如果使用“运行” （不支持dotMemory Unit）运行测试，则测试将失败。如果您希望测试运行程序仅忽略dotMemory Unit调用，请应用[DotMemoryUnit(FailIfRunWithoutSupport = false)]所需作用域上的属性：方法，类或程序集。

如果测试失败，则dotMemory Unit将自动保存*相应的内存快照，以便您可以在dotMemory事件探查器中对其进行调查。快照的位置显示在失败测试的消息中。

这是默认行为。您可以使用DotMemoryUnit属性更改自动保存策略。因此，您可以在任何测试失败时保存快照，也可以根本不保存快照。
