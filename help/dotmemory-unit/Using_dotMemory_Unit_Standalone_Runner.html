dotMemory Unit 3.0帮助

使用dotMemory Unit独立启动器

如果您使用独立的单元测试运行程序（而不是Visual Studio）运行测试，或者要使内存测试成为持续集成构建的一部分，则需要一个独立的dotMemory Unit启动器-dotMemoryUnit.exe命令行工具。

dotMemoryUnit.exe充当中介程序-它运行特定的独立单元测试运行程序，并为正在运行的测试中的dotMemory Unit调用提供支持。

dotMemory Unit独立启动器作为zip存档分发，可从dotMemory页面上下载。dotMemory Unit NuGet软件包还包含独立的启动器。

如果您的域策略是将Internet上的文件视为不安全，请不要忘记使用文件属性中的“取消阻止”按钮来取消阻止zip存档。

有关可用工具参数的详细信息，请运行：

dotMemoryUnit.exe --help

使用dotMemoryUnit.exe运行测试

在最简单的情况下，您要做的就是指定单元测试运行器的路径及其参数。例如，在下面的示例中，我们要从MainTests.dll运行NUnit测试：

dotMemoryUnit.exe "C:\NUnit 2.6.4\bin\nunit-console.exe" -- "E:\MyProject\bin\Release\MainTests.dll"

工具的输出包含成功和失败测试的数据。例如：

... Tests run: 3, Errors: 1, Failures: 0, Inconclusive: 0, Time: 28.3051788194675 seconds Not run: 0, Invalid: 0, Ignored: 0, Skipped: 0 Errors and Failures: 1) Test Error : MainTests.IntegrationTests.Method2 AssertTrafficException : Allocated memory amount Expected: 50,000,000 But was: 195,344,723 ...

dotMemoryUnit.exe退出代码

默认情况下，如果工具成功完成工作，则其退出代码为0。当您在CI服务器上运行该工具时，这不是很方便，因为您需要知道构建中是否有任何失败的测试。在这种情况下，最好的选择是使dotMemoryUnit.exe返回单元测试运行程序的退出代码。为此，您应该使用--propagate-exit-code论点。例如：

dotMemoryUnit.exe "C:\NUnit 2.6.4\bin\nunit-console.exe" --propagate-exit-code -- "E:\MyProject\bin\Release\MainTests.dll"

注意，目标可执行文件的参数必须始终在命令末尾传递。
