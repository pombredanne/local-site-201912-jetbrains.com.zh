使用对象集

在测试运行期间，垃圾回收器可以收集程序不再需要的对象。要有意阻止对象收集，请使用GC.KeepAlive方法。

对象集是根据特定条件选择的许多对象。为了便于理解，可以将对象集视为某些查询结果的一部分，例如“选择提升为Gen 2的字符串类型的所有对象”或“选择实例A保留在内存中的所有对象”，依此类推。 。一个ObjectSet实例的返回结果是ObjectSet.GetObjects和ObjectSet.GetExclusivelyRetainedObjects方法。

要根据多种条件对数据进行切片，您可以构建GetObjects电话。的ObjectSet type具有两个可以在测试断言中使用的属性： ObjectsCount和SizeInBytes 。

例

例如，以下代码断言没有string和Foo大对象堆中的对象：

dotMemory.Check(memory => { Assert.That(memory.GetObjects(where => where.Type.Is(typeof(string), typeof(Foo))) .GetObjects(where => where.Generation.Is(Generation.LOH)).ObjectsCount, Is.EqualTo(0)); });

dotMemory单元还为对象集查询提供了替代语法。您可以使用== ， &和|逻辑运算符组合多个查询。例如：

Assert.That(memory.GetObjects(where => where.Type == (typeof(string)) & where.Generation == Generation.LOH).ObjectsCount, Is.EqualTo(0));

ObjectSet类

表示内存中的一组对象。

ObjectSet方法

名称描述
GetObjects(Func query): ObjectSet 通过特定条件获取对象的子集。条件由Query应该由lambda表达式返回。传递给lambda的ObjectProperty对象允许创建查询，以按类型，接口和其他参数选择对象。
返回的实例ObjectSet类型。
GetObjects(ObjectSetQuery objectSetQuery): ObjectSet 通过特定条件获取对象的子集。该条件由可重复使用的查询（ ObjectSetQuery类型的实例）定义。有关详细信息，请参见QueryBuilder 。
返回的实例ObjectSet类型。
GetExclusivelyRetainedObjects(): ObjectSet 获取一组由当前对象集独占保留（控制）的对象。
返回的实例ObjectSet类型。
GroupByType: IReadOnlyCollection 返回TypeMemoryInfo类型的对象的集合。每个对象代表源对象集中的特定类型，并携带有关该类型的对象数量及其总大小的信息。

ObjectSet属性

名称类型描述
ObjectsCount int 集合中对象的总数。
SizeInBytes long 集合中对象的总大小（以字节为单位）。
