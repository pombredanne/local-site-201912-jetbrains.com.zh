打开分配数据的收集

默认情况下，探查器不会收集内存分配数据（内存流量和创建堆栈跟踪数据）。这样做是出于性能方面的考虑，因为收集这些数据意味着大量的分析开销。当您的所有测试都在检查堆中是否存在某些对象时，这种行为适合于简单测试。例如，如果您的测试需要分配数据，则它会检查内存流量，那么您应该手动打开内存分配数据的收集。

这可以通过使用DotMemoryUnit属性来完成：

[DotMemoryUnit(CollectAllocations=true)] [Test] public void TestMethod1() { ... // Some user code }

AssertTraffic属性还自动打开分配数据的收集：

[AssertTraffic(AllocatedMemoryAmount = 1024)] [Test] public void TestMethod1() { ... // Some user code }
