比较快照

的SnapshotDifference type允许您比较dotMemory创建的两个内存检查点（快照） 。检查方法并获取有关在快照之间创建了多少个对象以及收集了多少个对象的数据。例如，这可以帮助您确定无效的内存使用情况或内存泄漏。的对象SnapshotDifference类型是由于GetDifference 内存类型的方法*。

*如果您使用其他方法拍摄内存快照，则应使用dotMemoryApi.GetDifference处理内存流量的方法。

例

例如，以下代码断言foo.Bar()创建10个或更少的字符串：

// Create first memory checkpoint var memoryCheckPoint1 = dotMemory.Check(); foo.Bar(); // some user code dotMemory.Check(memory => { // Compare two checkpoints Assert.That(memory.GetDifference(memoryCheckPoint1).GetNewObjects (where => where.Type.Is<string>()).ObjectsCount, Is.LessThan(10)); });

SnapshotDifference类

表示两个内存快照之间的差异。

SnapshotDifference方法

名称描述
GetNewObjects(Func query): ObjectSet 获取一组新对象-在获取内存检查点之间的时间间隔内已创建但尚未收集的对象。
返回ObjectSet类型的实例。
如果query （可选），则将根据此查询中定义的条件过滤结果对象集。
GetDeadObjects(Func query): ObjectSet 获取一组“死”对象-在获取基本内存检查点之前创建的对象，并在获取第二个检查点之前的时间间隔内收集的对象。
返回ObjectSet类型的实例。
如果query （可选），则将根据此查询中定义的条件过滤结果对象集。
GetSurvivedObjects(Origin from = Origin.NewSnapshot): ObjectSet 获取一组幸存的对象-在获取基本内存检查点之前创建的，并且在获取第二个快照之前的时间间隔内未收集的对象。的from Origin类型的参数指定有关幸存对象（较新或较旧的内存检查点）的数据源。
返回ObjectSet类型的实例。
如果query （可选），则将根据此查询中定义的条件过滤结果对象集。
