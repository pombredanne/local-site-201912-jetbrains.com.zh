DataGrip 2019.2帮助

更新数据夹

默认情况下，DataGrip配置为自动检查更新，并在有新版本可用时通知您。更新通常是基于修补程序的 ：它们适用于现有安装，仅要求您重新启动IDE。但是，有时补丁更新不可用，并且必须安装新版本的DataGrip。

工具箱应用

如果您使用Toolbox App安装了DataGrip ，它将建议在有新版本可用时进行更新。如果您希望工具箱应用程序自动更新所有托管工具，请执行以下操作：

  1. 打开工具箱应用程序，然后单击右上角的螺母图标以打开设置。

  2. 选择自动更新所有工具 。

您还可以分别为每个托管IDE实例配置更新设置：

  1. 打开工具箱应用程序，单击所需实例的螺母图标，然后选择设置 。

  2. 选择是否要保留当前的主版本，自动更新实例以及要使用的更新通道：

    • 发布 ：仅更新为建议用于生产的稳定版本。

    • 发行和EAP ：包括对Beta发行版的更新，候选发行版和Early Access Program （EAP）构建，不建议在生产中使用它们，并包括功能预览。

    • 发布，EAP和每夜 ：包括高度不稳定的每夜构建。

独立实例

如果您手动安装DataGrip ，则更新由独立实例本身管理。当有新版本可用时，它将通知您。您可以选择更新当前实例，下载并安装新版本作为单独的实例，推迟通知或完全忽略更新。

您可以在“ 外观和行为” | “ 更新”页面上管理DataGrip更新。 设置/首选项对话框中的系统设置 Ctrl+Alt+S ：

手动检查更新

如果禁用了自动更新检查，请单击“ 立即检查”按钮以查看是否有较新的版本。

或者，您可以从主菜单检查更新：

  • 对于Windows和Linux： 帮助|检查更新

  • 对于macOS： DataGrip |检查更新

选择更新频道

使用更新渠道列表来限制您要使用的发行版类型。

仅当使用稳定版本时，才可以选择频道。对于EAP构建，通道始终设置为Early Access Program 。

  • 抢先体验计划 ：提供所有更新，包括主要版本的EAP版本和次要版本的预览版本。不建议将此通道用于生产开发。

    DataGrip只能更新为次要的Preview版本，而不能更新为主要的EAP版本。例如，您可以更新DataGrip 2017.2.3至2017.2.4 EAP ，但不是2017.3 EAP 。的2017.3 EAP在这种情况下，版本将作为其他实例安装。

    EAP版本可以更新为较新的EAP版本和稳定的DataGrip版本。如果某个时候将EAP版本更新为稳定版本，则原始安装目录的名称不会更改。

  • Beta版本或公共预览 ：包括稳定版本，候选版本和Beta版本。该频道中的某些更新可能包含一些小错误和功能预览。

  • 稳定发行版 ：仅包括建议用于生产的稳定发行版。

查看和管理忽略的更新

如果您出于某种原因决定忽略建议的更新，则以后可以查看忽略的每个更新，并在改变主意后将其重新提供。

  1. 单击查看/编辑忽略的更新链接。“ 忽略的更新”对话框列出了当前被忽略的内部版本号。

  2. 要使某个内部版本可用，请将其从忽略的更新列表中删除。

    如果您从忽略列表中删除了多个更新，则当您检查更新时，只会提供最新的下载。

卡扣包装

如果您将DataGrip作为snap软件包安装 ，它将自动管理更新。每天都会在后台自动更新所有快照。您还可以通过运行以下命令随时手动获取所有快照的最新版本：

$ sudo snap刷新
