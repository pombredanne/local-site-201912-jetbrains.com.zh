如何使用SSL连接到Cassandra

步骤1。创建一个Apache Cassandra连接

  1. 导航到文件|数据源 Ctrl+Alt+S 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择Apache Cassandra 。

  3. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。或者，您可以为数据源指定用户驱动程序。有关用户驱动程序的更多信息，请参阅将用户驱动程序添加到现有连接 。

  4. 在Host ， Keyspace ， User ， Password和Port字段中，指定连接详细信息。

第2步。验证JDBC驱动程序的版本

确认您使用的是1.3版或更高版本的JDBC驱动程序。

  1. 导航到文件|数据源 Ctrl+Alt+S 。

  2. 在驱动程序列表中，选择Apache Cassandra 。

  3. 在“ 驱动程序文件”窗格中，单击“ ver”。 然后选择Latest要么1.3 。

第三步设置虚拟机选项

  1. 导航到文件|数据源 Ctrl+Alt+S 。

  2. 在驱动程序列表中，选择Apache Cassandra 。

  3. 在驱动程序设置中，单击“ 高级”选项卡。

  4. 在“ VM选项”字段中，指定身份验证选项。

    • 服务器身份验证 ：您的客户端验证服务器是否正确且受信任。信任库必须包含服务器的公共密钥或用于对服务器密钥进行签名的CA证书。

      -Djavax.net.ssl.trustStore = -Djavax.net.ssl.trustStorePassword =

    • 客户端身份验证 ：服务器验证尝试连接到服务器的客户端。

      -Djavax.net.ssl.keyStore = -Djavax.net.ssl.keyStorePassword =

有关JSSE系统属性的更多信息，请参见《 Java安全套接字扩展（JSSE）参考指南》 。

第四步。将sslenabled选项添加到JDBC URL

  1. 添加sslenabled=true JDBC URL的选项。例如， jdbc：cassandra：// localhost：9042 /？sslenabled = true 。

  2. 要确保成功连接到数据源，请单击“ 测试连接” 。
