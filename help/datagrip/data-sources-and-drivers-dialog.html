总览

驱动程序是一个集合，其中包括数据库驱动程序文件和用于创建数据源的默认设置。

    项目

    描述

    只读

    设置只读状态。选中复选框以保护数据源免于意外的数据修改。

    如果选中此复选框，则无法在数据编辑器中修改数据。

    如果驱动程序不支持只读状态，则可以在控制台中进行数据修改。

    交易控制

    设置数据库事务的隔离级别以及提交事务的方式。

    • 自动 ：将本地更改提交到数据库服务器时，当前事务将自动提交。

    • 手册 ：提交给数据库服务器的更改会累积在可以提交或回滚的事务中。

    切换架构

    定义DataGrip用于切换模式的模式。

    • 自动 ：自动或手动切换模式。对于PostgreSQL和Amazon Redshift，请使用此选项保存IDE重新启动之间的已配置搜索路径。

    • 手动 ：仅手动切换模式。

    • 禁用 ：不切换模式。

    单连接方式

    启用单连接模式。单连接模式意味着数据源和所有控制台都使用一个且相同的连接。此模式允许您查看数据库树中的临时对象，或在不同的控制台中使用同一事务。有关控制台的更多信息，请阅读数据库控制台 。

    每N秒运行一次保持活动查询

    运行保持活动查询以保持连接活动。您可以在驱动程序设置中为不支持的数据库定义自定义查询。

    N秒后自动断开连接

    在指定的秒数后，从数据库断开连接。

    单数据库模式

    在数据库树视图中，仅显示和启用在连接设置中指定的数据库。

    当您连接到数据源时，DataGrip可以检索并显示该数据源具有的所有数据库。但是在某些情况下（例如，使用PgBouncer的某些设置），您可以或被允许仅使用特定的数据库。在启用了“ 单数据库”模式的数据库树视图中，您只会看到在连接设置中指定的数据库。

    考虑对PostgreSQL，Azure SQL数据库，Greenplum，Amazon Redshift使用此设置。

    启动脚本

    每次建立连接时都运行SQL查询。

    注意 ：如果清除了“ 单一连接模式”复选框，则每个新控制台都会创建一个新连接。

    • 当您在“ 数据源和驱动程序”对话框中单击“ 确定”时。

    如果取消选中“ 自动同步”复选框，则仅当您单击“ 刷新”按钮时，“ 数据库工具”窗口中数据源的视图才会与数据库的实际状态同步。 或按Ctrl+F5 ）。

    编辑过时的DDL时发出警告

    如果缓存的DDL与数据库中的实际DDL不同，则显示通知。

    每次打开源代码时，DataGrip都会显示在索引过程中加载的版本。但是，如果有人从另一个工作站更改了源代码，则索引版本将过时。

    如果您的版本早于数据库中的版本，则会看到以下通知： Cached object differs from the object in the database 。

    使用JDBC元数据进行内省

    切换到基于JDBC的内省器。

    为了检索有关数据库对象（数据库元数据）的信息，DataGrip使用以下内部检查器：

    • 本机自省（某些DBMS可能不可用）。本机内省员使用特定于DBMS的表和视图作为元数据源。它可以检索特定于DBMS的详细信息，并产生更精确的数据库对象图。

    • 一个基于JDBC的内省器（适用于所有DBMS）。基于JDBC的内省器使用JDBC驱动程序提供的元数据。它只能检索有关数据库对象及其属性的标准信息。

    当本机自检失败或不可用时，请考虑使用基于JDBC的intorspector。

    当您的数据库服务器版本早于DataGrip支持的最低版本时，本机自检器可能会失败。

    您可以尝试切换到基于JDBC的自检器，以解决有关从数据库检索数据库结构信息的问题。例如，当数据库工具窗口中未显示数据库或模式级别以下的数据库对象中存在的模式时。

    对未自检的系统目录使用预先自检的对象

    切换系统目录中自省对象的使用。有关自省和系统目录的更多信息，请参阅系统目录中的自省对象 。

    :[-] ，其中：

    注意 ：如果清除了“ 单一连接模式”复选框，则每个新控制台都会创建一个新连接。

    保持活动查询

    保持活动查询，将运行以保持连接活动。

    自动同步

    自动同步选项的默认设置。

    的负载源

    加载所选模式类别的数据库对象的源代码。

    发送申请信息

    连接到数据库服务器时，如果选中此复选框，则DataGrip会发送有关其自身的信息。

    方言

    与相应数据源关联的SQL语言。

    代码风格

    选择要用于使用此驱动程序的数据源的代码样式。有关自定义代码样式的更多信息，请阅读SQL的代码样式 。

    格式化同步源

    同步后，将代码样式应用于数据库对象的源代码。

    高级

    “ 高级”选项卡包括JVM选项，环境变量和可以作为键值对传递给数据库驱动程序的选项。

    要开始在表中编辑值，请双击相应的“ 值”字段。

    要添加一行，请开始编辑最后一行中显示和的值。新行将自动添加到表中。

    项目

    描述

    虚拟机选项

    运行数据库驱动程序的JVM的默认选项。该驱动程序作为JVM中的单独进程启动。

    虚拟机环境

    数据库驱动程序JVM的环境变量。

    例如，可以通过设置Oracle数据库中与编码有关的问题来解决NLS_LANG变数（例如NLS_LANG=Russian_CIS.CL8MSWIN1251 ）。有关更多信息，请参见Oracle NLS_LANG常见问题解答 。

    问题

    如果检测到潜在的问题，那么“ 问题 ”右边会有一个数字。在这种情况下，如果单击“ 问题” ，则会看到问题列表以及用于修复问题的控件。
