范围

范围是项目中文件和目录的过滤子集。启用合并范围视图时，会将搜索和代码检查之类的操作限制在选定的合并范围内。本节介绍如何配置范围和属于这些范围的文件的颜色。

创建一个新的范围

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到外观和行为|范围 。

  2. 请点击 ，从列表中选择本地或共享 ，然后指定范围的名称。

    • 本地范围仅供个人使用，并存储在您的工作区中。默认位置： ％HOMEPATH％\。 \ config \ projects \ \ .idea \ workspace.xml

    • 共享范围是为当前项目定义的，团队成员可以通过VCS访问。默认位置： ％HOMEPATH％\。 \ config \ projects \ \ .idea \ scopes \

您可以稍后使用页面底部的“ 通过VCS共享”复选框更改共享状态。

配置合并范围中的项目列表

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到外观和行为|作用域，然后选择要配置的作用域。

  2. 要为范围定义模式，请执行以下操作之一：

    • 要将文件和目录包括在范围内，请选择文件和目录，然后按“ 包含”或“ 递归包含 （带有子目录）”按钮。要排除文件或目录，请使用排除或递归排除 。DataGrip创建一个表达式并将其显示在Pattern字段中。

    • 使用范围语言语法在模式字段中手动指定模式。

  3. 点击应用 。

通过文件扩展名过滤附件目录中的文件

  1. 导航到文件|设置|外观与行为|作用域 ，单击“ 显示选项菜单”图标（ ），然后选择编辑范围 。

  2. 选择一个范围或创建一个新的范围。

  3. 在模式字段中，输入以下代码： file：*。 ，在哪里是您要用于过滤器的文件扩展名。

  4. 在“ 文件”工具窗口中，单击“ 文件”列表，然后选择您创建的作用域名称。

将文件颜色与范围相关联

可以在DataGrip的用户界面中以不同的颜色突出显示属于不同范围的文件：在导航列表中 ，在编辑器选项卡中，在“ 文件”工具窗口中 。这样可以在大型项目中更快，更轻松地导航。

将文件颜色与范围相关联

  1. 导航到文件|设置|外观与行为|文件颜色 。

  2. 点击添加 。

  3. 在“ 添加颜色标签”对话框中，选择一个范围并为其选择颜色。

  4. 单击确定 。

  5. 定义要使用范围颜色的区域。要定义区域，请选择“ 在编辑器选项卡中 使用”或“ 在项目视图中使用”复选框。

  6. 选择合并范围颜色关联的可访问性级别。要共享颜色关联，请点击共享图标（ ）。

    文件颜色关联的默认位置是％HOMEPATH％\。 \ config \ projects \ \ .idea \ fileColors.xml

  7. 应用更改。

排列范围的顺序

如果某些文件包含在多个作用域中，则作用域的顺序就变得很重要：DataGrip使用最上面作用域的颜色（在“ 作用域设置”页面中显示）突出显示该文件。当然，您可以更改范围的顺序，从而更改结果的突出显示。

排列范围的顺序

  1. 导航到文件|设置|外观与行为|范围 。

  2. 选择您要移动的范围。

  3. 要更改顺序，请点击上移 （ ）或下移图标 。

  4. 点击应用 。

例

按名称包含一个文件

file:/file:databases/create_schema.sql

在目录中包括一组所有文件，不包含子目录

file:/*file:databases/mysql-db/*

在具有子目录的目录中包括一组所有文件

file://*file:databases/mysql-db//*

包含来自特定项目的文件名

file[project_name]:/file[default]:databases/create_schema.sql

可能的逻辑运算符&&·（AND），||·（OR）、!·（不）。

file[project_name]:&&!file:file[default]:databases/*&&!file:databases/create_schema.sql
