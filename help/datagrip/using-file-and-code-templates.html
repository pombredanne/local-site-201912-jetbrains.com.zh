文件和代码模板

文件模板是创建新文件时要生成的默认内容的规范。根据要创建的文件类型，模板会提供该类型的所有文件中应包含的初始代码和格式（根据行业标准，公司政策或其他原因）。

DataGrip为所有受支持的文件类型提供了预定义的模板，在创建新文件时建议使用这些模板（ Ctrl+Alt+Insert ）。

文件模板在 编辑器| DataGrip设置的“ 文件和代码模板”页面Ctrl+Alt+S 。可以在以下两个范围之一中配置此设置页面：

  • 默认作用域控制与整个工作区有关的模板。它们存储在IDE配置目录中的 fileTemplates下。

  • 项目范围控制每个项目的模板。它们存储在.idea / fileTemplates下的项目文件夹中。这些模板可以在团队成员之间共享。

默认情况下，模板列表仅包含DataGrip提供的预定义模板。其中一些是内部的，这意味着它们无法删除或重命名。内部模板的名称以粗体显示。您修改的模板的名称以及您手动创建的自定义模板的名称显示为蓝色。

DataGrip使用几种类型的模板，分为以下选项卡：

  • “ 文件”选项卡包含可用于创建新文件的文件模板。

  • 包含选项卡包含可重用内容的模板，可以将其插入文件模板（例如，标题）。

  • “ 代码”选项卡包含DataGrip用于生成各种构造的代码片段（片段）的内部模板。您可以在此标签上编辑可用的代码段，但不能创建新的代码段。

  • 其他选项卡包含各种应用程序服务器和框架使用的模板。您可以在此选项卡上编辑现有的内置模板，但不能创建新的模板。

以下过程描述了如何创建文件模板。类似的过程可用于创建包含模板 。

创建一个新的文件模板

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择编辑器|文件和代码模板 。

  2. 在文件选项卡上，单击并指定名称，文件扩展名和模板主体。

  3. 应用更改并关闭对话框。

复制现有文件模板

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择编辑器|文件和代码模板 。

  2. 在文件选项卡上，单击并根据需要修改模板的名称，文件扩展名和主体。

  3. 应用更改并关闭对话框。

将文件另存为模板

  1. 在编辑器中打开一个文件。

  2. 选择工具|从菜单中将文件另存为模板 。

  3. 在打开的“ 将文件另存为模板”对话框中，指定新的模板名称并根据需要编辑正文。

  4. 应用更改并关闭对话框。

句法

文件和代码模板是用速度模板语言 （VTL）编写的，它可以使用以下结构：

  • 固定文本（标记，代码，注释等），按原样呈现。

  • 变量 ，将其替换为它们的值。

  • 各种指令，包括#parse ， #set ， #if ， 和别的。

有关更多信息，请参见VTL参考指南 。

以下示例显示了用于在DataGrip中创建用户定义函数的模板：

CREATE FUNCTION`$ {FUNCTION_NAME}`（`$ arg1`，$ arg2`，$ arg3`）返回开始返回结束#parse（“ Copyright.sql”）

创建新的自定义SQL文件时，此模板会生成一个文件，其内容类似于以下内容：

CREATE FUNCTION`functionName`（`arg1`，`arg2`，`arg3`）返回开始返回结束/ ** *版权。由用户创建于2018年10月10日* /
上次修改时间：2019年11月11日
