在哪里报告问题

在Youtrack中报告问题

  1. 登录到您的YouTrack帐户。

  2. 点击创建问题 。

  3. 确保从“ 项目”列表中选择了DataGrip。

  4. 描述您的问题，并分别在“ 描述”和“ 摘要”字段中提供简短摘要。

  5. 如果可能，请附上一些故障排除材料 。

  6. 点击创建问题 。

您可以使用JetBrains帐户登录到Youtrack。JetBrains帐户是与JetBrains产品和服务（ JetBrains帐户 ）的单一交互点。有关JetBrains帐户的更多信息，请参阅JetBrains帐户帮助 。

在JetBrains支持中心提交请求

  1. 登录到JetBrains支持中心 。

  2. 单击主页顶部的“ 提交请求”按钮。

  3. 填写提交申请表。

  4. 如果可能，请附上一些故障排除材料 。

  5. 点击提交 。

复制DataGrip版本和系统信息

  1. 您可以通过以下操作获取产品和系统信息：

    • 使用产品帮助菜单。

      • 对于macOS，请单击DataGrip |。关于DataGrip 。

      • 对于Windows和Linux，单击帮助|。关于 。

    • 单击帮助|寻找行动 Ctrl+Shift+A ，类型About ，然后按Enter 。

  2. 点击复制图标并将其粘贴到您的Youtrack票证，电子邮件和支持请求中。

找到DataGrip日志

单击帮助|在Explorer / Finder中显示日志 。idea.log文件包含有关您的IDE性能的最新日志信息。

找到查询日志

SQL日志包括您曾经在DataGrip中运行过的所有查询。这意味着所有用户查询和所有内部查询（由JDBC驱动程序运行的查询除外）。日志信息存储在database.log中 。database.log文件存储在您的硬盘驱动器上，直到您删除或覆盖该文件为止。当文件大小达到1 MB时，将创建一个具有不同名称的新文件。

要找到database.log ，请单击“ 帮助” |“帮助”。在Explorer | Finder中显示SQL日志 。您可以在DataGrip或文本编辑器中打开database.log 。

配置DataGrip日志设置

为避免编辑log.xml文件本身，DataGrip建议使用一个方便的对话框来更改类别的日志记录级别。该文件位于DataGrip安装的bin目录下。

  1. 单击帮助|调试日志设置 。

  2. 在“ 自定义调试日志配置”对话框中，键入日志类别名称，并用新行分隔。

不建议更改log.xml ，因为它会不时导致补丁问题。

在编辑log.xml之前，请考虑与支持服务联系。一些用户可能不知道要指定的模块名称，而支持服务可以建议模块以进行更好的诊断。

有关IDE日志文件的更多信息，请参阅支持论坛上的查找IDE日志文件 。

报告绩效问题

DataGrip完全冻结并且不响应任何动作？IDE性能比预期的慢，CPU使用率很高，您怀疑内存泄漏吗？若要解决您的问题，JetBrains可能需要内存快照，分析器信息或线程转储。

有关对这些问题进行故障排除的更多信息，请阅读报告性能问题 。
