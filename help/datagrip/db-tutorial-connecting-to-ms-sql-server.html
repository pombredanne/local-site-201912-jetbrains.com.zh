如何连接到MS SQL Server

先决条件

  • Microsoft SQL Server 2017（Express Edition）– 从Microsoft网站下载链接 。

  • Microsoft SQL Server Management Studio 18.2- 从Microsoft网站下载链接 。

  • DataGrip 2019.1及更高版本– 从JetBrains网站下载链接 。

步骤1。配置SQL Server配置管理器

要打开SQL Server配置管理器，请在Windows上打开“ 搜索”对话框，然后键入SQLServerManager14.msc （适用于Microsoft SQL Server 2017）。双击找到的结果。如果您使用其他版本的Microsoft SQL Server，请在更改第二个数字之前.msc ：

  • SQLServerManager13.msc对于SQL Server 2016

  • SQLServerManager12.msc对于SQL Server 2014

  • SQLServerManager11.msc对于SQL Server 2012

运行SQL Server浏览器

SQL Server浏览器侦听传入的请求，并提供有关计算机上Microsoft SQL Server实例的信息。有关SQL Server浏览器的详细信息，请参阅Microsoft文档中的SQL Server Browser 。

如果禁用了SQL Server Browser菜单项，请尝试启用SQL Server代理服务 。

  1. 在“ SQL Server配置管理器”中，单击“ SQL Server服务” 。

  2. 在列出服务器服务的右窗格中，右键单击“ SQL Server浏览器”，然后选择“ 启动” 。

启用SQL Server代理

如果SQL Server浏览器正在运行，则可以跳过此过程。

SQL Server代理是一种Microsoft Windows服务，它执行计划的管理任务。有关该服务的更多信息，请参见Microsoft官方文档中的SQL Server代理 。确保SQL Server代理正在运行并自动启动。更改后，可能需要重新启动。

  1. 导航到控制面板|管理工具|服务 。

  2. 在“ 服务”窗口中，右键单击“ SQL Server Agent（ ）”，然后选择“ 属性” 。对于本教程， 是MSSQLSERVER。

  3. 从启动类型列表中，选择自动 ，然后单击启动 。

启用TCP / IP连接

  1. 在“ SQL Server配置管理器”中，展开“ SQL Server配置” ，然后单击“ MSSQLSERVER的协议” ，其中MSSQLSERVER是Microsoft SQL Server实例的名称。

  2. 在协议名称列表中，右键单击“ TCP / IP”，然后选择“ 属性” 。

  3. 在“ 协议”选项卡上，从“ 启用”列表中选择“ 是” 。

  4. 在“ IP地址”选项卡上，找到工作站使用的接口并查看连接端口。默认情况下，端口为1433。验证其他正在运行的应用程序不使用同一端口。

如果更改了任何设置，请重新启动服务器。在大多数情况下，重新启动可以解决连接问题。

第2步。配置SQL Server Management Studio（SSMS）

创建一个用户

  1. 在SQL Server Management Studio（SSMS）中，导航到“ 安全性” |“安全性”。登录名 。

  2. 右键单击Logins根文件夹，然后选择New Login 。

  3. 在“ 常规”页面上，在“ 登录名”字段中指定一个登录名。

  4. 选择身份验证模式。您可以在以下身份验证模式之间进行选择：

    • Windows身份验证 ：使用您的域登录名和密码。

    • SQL Server身份验证 ：使用自定义登录名和密码。如果选中“ 强制密码安全策略”复选框，则用户必须先更改分配的密码，然后才能使用DataGrip连接到Microsoft SQL Server。否则，请清除“ 执行密码安全策略”复选框。

  5. 单击确定 。

配置用户角色

  1. 右键单击创建的用户配置文件，然后选择属性 。

  2. 在“ 服务器角色”页面上，选择适当的用户角色（例如sysadmin ）。

第三步使用DataGrip连接到Microsoft SQL Server

以下部分描述了Windows ， macOS和Linux上DataGrip的配置。请注意，“ 使用Windows域身份验证”复选框仅在Windows上可用。要在macOS和Linux上配置Windows域认证，请参阅使用Windows域认证连接 。

视窗

通过使用SQL Server身份验证进行连接

  1. 导航到文件|数据源 Ctrl+Alt+S 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择Microsoft SQL Server 。

  3. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。或者，您可以为数据源指定用户驱动程序。有关用户驱动程序的更多信息，请参阅将用户驱动程序添加到现有连接 。

  4. 在“ 主机” ，“ 实例 ”和“ 端口”字段中，指定您的连接详细信息。

  5. 清除“ 使用Windows域身份验证”复选框。

  6. 在“ 用户”和“ 密码”字段中，指定您的凭据

  7. 要确保成功连接到数据源，请单击“ 测试连接” 。

通过使用Windows域身份验证进行连接

  1. 导航到文件|数据源 Ctrl+Alt+S 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择Microsoft SQL Server 。

  3. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。或者，您可以为数据源指定用户驱动程序。有关用户驱动程序的更多信息，请参阅将用户驱动程序添加到现有连接 。

  4. 在“ 主机” ，“ 实例 ”和“ 端口”字段中，指定您的连接详细信息。

  5. 确保已选中“ 使用Windows域身份验证”复选框。

  6. 要确保成功连接到数据源，请单击“ 测试连接” 。

macOS和Linux

通过使用SQL Server身份验证进行连接

  1. 导航到文件|数据源 Ctrl+Alt+S 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择Microsoft SQL Server 。

  3. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。或者，您可以为数据源指定用户驱动程序。有关用户驱动程序的更多信息，请参阅将用户驱动程序添加到现有连接 。

  4. 在“ 主机” ，“ 实例 ”和“ 端口”字段中，指定您的连接详细信息。

  5. 在“ 用户”和“ 密码”字段中，指定您的凭据。

  6. 要确保成功连接到数据源，请单击“ 测试连接” 。

通过使用Windows域身份验证进行连接

  1. 导航到文件|数据源 Ctrl+Alt+S 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择Microsoft SQL Server 。

  3. 从驱动程序列表中，选择Microsoft SQL Server（jTds） 。

  4. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。或者，您可以为数据源指定用户驱动程序。有关用户驱动程序的更多信息，请参阅将用户驱动程序添加到现有连接 。

  5. 在“ 主机” ，“ 实例 ”和“ 端口”字段中，指定您的连接详细信息。

  6. 在“ 用户”和“ 密码”字段中，指定您的域凭据。在“ 用户”字段中，输入不带域前缀的域用户（例如， John.Smith代替DOMAIN\John.Smith ）。

  7. 在URL字符串中，添加以下参数： ;domain=your_domain;instance=your_ms_sql_server_instance;databaseName=database_name; 。考虑以下完整连接字符串的示例： jdbc:jtds:sqlserver://UNIT-670:1433;domain=DEVELOPMENT;instance=MSSQLSERVER;databaseName=guest; 。

  8. 要确保成功连接到数据源，请单击“ 测试连接” 。
