如何在DataGrip中启动GitHub项目

本教程涵盖两个常规用例：

  • 克隆现有的GitHub存储库
  • 在GitHub上共享目录

克隆现有的GitHub存储库

步骤1。安装Git Integration和GitHub插件

  1. 开启设定|插件 。

  2. 单击市场 。

  3. 在“ 在市场中搜索插件”字段中，输入git然后按Enter 。

  4. 单击安装进行Git集成 。

  5. 单击“为GitHub 安装 ”。

  6. 重新启动DataGrip。

第2步。登录到您的GitHub帐户

  1. 在设置中Ctrl+Alt+S ，导航到版本控制| GitHub的

  2. 点击添加帐户 Alt+Insert 。

  3. 在登录名和密码字段中，输入您的GitHub凭据。

  4. 单击登录 。

如果您使用两因素身份验证，请安装GitHub插件。如果登录过程成功，您将看到您的帐户已连接（ Version Control | GitHub ）。

第三步启用版本控制集成

  1. 导航到VCS |版本控制集成 。

  2. 在“ 启用版本控制集成”对话框中，从列表中选择Git 。

  3. 单击确定 。

第四步。克隆GitHub存储库

出于示例目的，我们将使用转储存储库： https : //github.com/DataGrip/dumps 。

  1. 导航到VCS | Git |克隆 。

  2. 在网址字段中，粘贴https://github.com/DataGrip/dumps 。

  3. 在目录字段中，为转储文件指定存储路径。

  4. 单击克隆 。

  5. 在“ 从版本控制签出”通知窗口中，单击“ 否” 。如果单击“ 是” ，DataGrip建议您使用克隆的文件创建一个新项目。

第五步附加带有克隆文件的目录

  1. 右键单击“ 文件”工具窗口（ 视图|工具窗口|文件 ）中的区域，然后选择“ 附加目录” 。

  2. 在文件浏览器中，导航到包含克隆文件的目录，然后单击“ 打开” 。

第六步注册VCS根目录

DataGrip支持基于目录的版本控制模型。这意味着每个项目目录都可以与不同的版本控制系统关联。

  1. 打开版本控制设置 （“ 文件” |“设置” |“版本控制” ）。此页面显示项目目录和与其关联的版本控制系统的列表（如果未添加目录，则该列表仅包含项目根目录）。

  2. 点击添加按钮（ ）。

  3. 在“ 添加VCS目录映射”对话框中，选择“ 目录”选项。输入要与版本控制系统关联的目录的路径，或单击“ 浏览”按钮并选择dumps目录。

  4. 从VCS列表中，选择Git 。

  5. 单击确定 。

在GitHub上共享目录

步骤1。安装Git Integration和GitHub插件

  1. 开启设定|插件 。

  2. 单击市场 。

  3. 在“ 在市场中搜索插件”字段中，输入git然后按Enter 。

  4. 单击安装进行Git集成 。

  5. 单击“为GitHub 安装 ”。

  6. 重新启动DataGrip。

第2步。登录到您的GitHub帐户

  1. 在设置中Ctrl+Alt+S ，导航到版本控制| GitHub的

  2. 点击添加帐户 Alt+Insert 。

  3. 在登录名和密码字段中，输入您的GitHub凭据。

  4. 单击登录 。

第三步启用版本控制集成

  1. 导航到VCS |版本控制集成 。

  2. 在“ 启用版本控制集成”对话框中，从列表中选择Git 。

  3. 单击确定 。

第四步。附加目录

要在GitHub上共享脚本，您需要将计算机上的目录附加到“ 文件”工具窗口。

  1. 右键单击“ 文件”工具窗口（ 视图|工具窗口|文件 ）中的区域，然后选择“ 附加目录” 。

  2. 在文件浏览器中，导航到要附加的目录，然后单击“ 打开” 。

第五步创建本地Git存储库

在我们的机器上创建一个Git存储库，您将与GitHub上的远程存储库链接。

  1. 导航到VCS |导入版本控制|创建Git存储库 。

  2. 在文件浏览器中，导航到要共享的目录（例如， dumps ）。

  3. 点击打开 。

第六步创建一个GitHub存储库

  1. 在GitHub上，点击加号图标，然后选择新建存储库 。

  2. 在“ 存储库名称”字段中，为您的存储库指定一个名称（例如， dumps ）。

  3. （可选）指定其他存储库设置。

  4. 单击创建存储库 。

    结果，创建了GitHub存储库。在“ 代码”选项卡上，您将看到指向您的存储库的链接（例如， https://github.com/JetBrainsUser/dumps.git ）。

步骤7链接本地和远程存储库

  1. 在DataGrip中，导航到VCS | Git |遥控器 。

  2. 在“ Git遥控器”对话框中，单击“ 添加”图标 。

  3. 在“ 定义远程”对话框中，将链接粘贴到您的GitHub存储库。

  4. 应用更改。

步骤8提交本地更改

  1. 在“ 文件”工具窗口（“ 视图” |“工具窗口” |“文件” ）中，右键单击转储文件夹，然后选择“ Git” |“文件” 。提交目录 。

  2. 在文件窗格中选中“ 未解析的文件”复选框。

  3. 在“ 提交消息”字段中，编写一条提交消息。

  4. 按提交 。

步骤9。推动本地变化

  1. 在DataGrip中，导航到VCS | Git |推 。

  2. 在“ 推送提交”对话框中，单击“ 推送” 。
