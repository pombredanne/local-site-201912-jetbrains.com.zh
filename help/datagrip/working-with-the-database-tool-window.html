DataGrip 2019.2帮助

定制数据库树视图

您的数据库树显示在“ 数据库工具”窗口中。该工具窗口使您可以查看和修改数据库中的数据结构，以及执行其他关联的任务。

您可以对每个数据源的“ 数据库”工具窗口中显示的数据库对象进行过滤，分组和排序，并仅显示所需的对象。

刷新数据库状态

如果有人更改了远程数据库数据或视图，则数据库的本地视图可能与数据库的实际状态不同。

要自动同步数据库状态，请在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”按钮并选择要更改的数据源。在选项选项卡上，选择自动同步复选框。

如果清除了自动同步复选框，则仅当您单击同步图标时， 数据库工具窗口（ 视图|工具窗口|数据库 ）中数据源的视图才会与数据库的实际状态同步。 或按Ctrl+Alt+Y 。

从工具栏过滤对象

要选择必要的对象，请使用工具栏上的“ 过滤器”图标 。

使用对象过滤器过滤对象

在“ 数据源和驱动程序”对话框的“ 模式”选项卡上的“ 对象过滤器”字段中指定过滤选项（ ）。

为“ 对象”过滤器字段编写表达式时，请使用以下模式。

:[-] ，其中：

可能是聚合，整理，事件，fdw，ftable，mview，运算符，包，角色，例程，序列，同义词，表，用户，视图，vtable。

是一个正则表达式。要排除某项，请添加- （减去）。有关正则表达式的更多信息，请参见JavaTM 2 Platform Standard Edition 5.0 API Specification中的类模式 。

请考虑以下示例，这些示例演示了过滤器语法：

  • f.*

    仅显示名称以开头的对象f 。

  • table:[gh].*

    显示名称以开头的表格g要么h ，以及所有其他对象类型。

  • view:new_.*||routine:-[ps].*

    显示名称以开头的视图new_ ，其名称以任何字母开头的例程p要么s ，以及所有其他对象类型。

组数据源

若要对数据源进行分组，启用或排序，请单击“ 显示选项”菜单图标在数据库工具窗口的标题栏上，选择所需的选项。

为数据源分配颜色

为了更好地区分生产数据库和测试数据库，可以为数据源分配颜色。

  1. 导航到文件|数据源 Ctrl+Alt+S 。

  2. 选择要为其分配颜色的数据源。

  3. 点击选择颜色图标（ ），然后选择一种颜色。

  4. 单击确定 。
