使用标签标记特定的提交

Git允许您将标记附加到提交以标记项目历史记录中的某些点，以便将来可以引用它们。例如，您可以标记与发行版本相对应的提交，而不是创建分支来捕获发行快照。

为提交分配标签

  1. 在“ 日志”视图中找到要标记的提交。右键单击它，然后从上下文菜单中选择“ 新建标记 ”。

  2. 输入新标签的名称，然后单击“ 确定” 。标签将显示在“ 日志”视图中：

将带注释的标签分配给提交

带注释的标签的元数据包含创建它们的用户的名称，因此它们使您可以检查放置它们的人员。

  1. 从主菜单中，选择VCS |。 Git |标签

  2. 在打开的“ 标签”对话框中，在“ Git Root”下 ，选择要在其中标记提交的本地存储库的路径，然后指定新标记的名称。

  3. 在“ 提交”字段中，指定要标记的提交。您可以输入提交哈希，或使用表达式，例如： ~ （有关详细信息，请参阅Git 提交命名约定）。

  4. 在消息字段中输入一些注释以创建带注释的标签（如果为空，则将创建常规标签）。

  5. 点击创建标签 。

重新分配现有标签

如果您将标签放置在错误的提交上，并且想要重新分配它（例如，指示发布版本的提交），请执行以下操作：

  1. 从主菜单中，选择VCS |。 Git |标记 。

  2. 在“ 标签”对话框的“ 标签名称”字段中，指定要重新分配的现有标签的名称。

  3. 选择强制选项。

  4. 在提交字段中，指定将标签移动到的提交，然后单击创建标签 。

跳转到标记的提交

打开版本控制工具窗口Alt+9并切换到“ 日志”标签。

  1. 单击转到哈希/分支/标签图标在工具栏上，或按Ctrl+F 。

  2. 输入标签名称（ 代码完成会在您键入时提示标签名称），然后按Enter 。

检出带标记的提交

假设您用标签标记了与发行版本相对应的提交，现在您想在该时间点查看项目的快照。您可以通过签出带标记的提交来做到这一点。请执行以下任一操作：

  • 找到您要签出的带标记的提交，右键单击它，然后从上下文菜单中选择“ 签出修订 ”。

  • 调用分支弹出窗口 ，单击“ 签出标签”或“修订”，然后键入标签名称（在键入时，DataGrip提供匹配标签和修订的列表）。

请注意，此操作将导致HEAD分离 ，这意味着您不再位于任何分支中。您可以使用此快照进行检查和实验。但是，如果要在此快照之上提交更改，则需要创建一个branch 。

删除标签

  • 找到带标记的提交，右键单击它，然后选择“ 标记” |“提交” 。从上下文菜单中删除 。
