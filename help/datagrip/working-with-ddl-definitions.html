使用DDL定义

编辑数据库对象的DDL定义

  1. 在数据库工具窗口（ 视图|工具窗口|数据库 ）中，单击一个数据库对象。

  2. 单击源代码编辑器按钮（ ）。或者，按Ctrl+B 。

为数据库对象生成DDL定义

数据定义语言（DDL）定义数据库的结构，包括行，列，表，索引和其他元素。在DataGrip中，可以使用SQL Generator工具窗口来生成数据库对象的数据定义。

在“ 数据库工具”窗口（“ 视图” |“工具窗口” |“数据库” ）中，右键单击数据库对象，然后选择“ SQL脚本” |“数据库” 。 SQL生成器 Ctrl+Alt+G 。

在“ SQL Generator”窗口中，可以配置输出，将其复制到剪贴板 ，将其保存到文件 ，或在数据库控制台中打开它 。

DEFINER子句指定例程执行的安全上下文（访问权限）。在MySQL和MariaDB中，当为过程或函数生成DDL时，请选择Skip`definer = ...`子句以跳过此子句。

将生成的DDL保存在SQL文件中

  1. 在“ 数据库工具”窗口（“ 视图” |“工具窗口” |“数据库” ）中，右键单击数据库对象，然后选择“ SQL脚本” |“数据库” 。 SQL生成器 Ctrl+Alt+G 。

  2. 在“ SQL Generator工具”窗口中，单击“ 文件输出选项”图标（ ）。

  3. 从“ 布局”列表中，选择要使用的方法：

    • 每个对象的文件 ：生成一组SQL文件。

    • 每个对象的文件顺序 ：生成一组编号的SQL文件。

SQL文件是由groovy脚本生成的。要更改脚本，请点击设置图标（ ），然后选择编辑 。

从引用转到其DDL定义

  1. 在“ 设置/首选项”对话框中Ctrl+Alt+S ，导航到数据库|一般

  2. 在“ 导航”窗格中，清除“ 在DDL编辑器上首选数据库视图”复选框。

生成DDL定义到数据库控制台

在“ 数据库工具”窗口（“ 视图” |“工具窗口” |“数据库” ）中，右键单击数据库对象，然后选择“ SQL脚本” |“数据库” 。生成DDL到Console 。

生成剪贴板的DDL定义

在“ 数据库工具”窗口（“ 视图” |“工具窗口” |“数据库” ）中，右键单击数据库对象，然后选择“ SQL脚本” |“数据库” 。生成DDL到剪贴板 。
