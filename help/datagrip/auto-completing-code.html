代码完成

本节介绍了各种上下文感知的代码完成技术，这些技术可帮助您加快编码过程。

基本完成

基本代码完成功能可帮助您完成可见性范围内的语句，数据库对象名称，参数和关键字。当您调用代码完成功能时，DataGrip会分析上下文并建议从当前插入符号位置可以到达的选择（建议还包括Live模板 ）。

非英语键盘布局也可以使用基本完成功能。

代码完成功能可用于自定义文件类型 。但是，DataGrip无法识别此类文件的结构，并且会建议选项，无论它们在当前上下文中是否合适。

调用基本完成

  1. 开始输入名称。

  2. 按Ctrl+Space或选择代码|完成|从主菜单基本 。

当查询中的名称不明确时，DataGrip将显示上下文。

完整的缩写名称

用camelCase编写的数据库对象的全名，带连字符的对象名称或names_with_underscore。要开始完成，请使用名称中的前几个字母。

完整的简短陈述

缩写语句的全名。要开始完成，请使用名称中的前几个字母。

新对象的完整名称

DDL语句中新创建的对象的完整名称（例如，在ALTER子句之后）。

产生别名

生成对象的别名。要显示建议列表，请按Ctrl+Space 。

要在使用自动完成功能时自动为表添加别名，请转至“设置” |“自动”。优先 Ctrl+Alt+S并导航到编辑器|一般|代码完成 。在“ SQL”部分中，选中“ 完成表名时自动添加别名”复选框。此后，每次您自动完成表名时，DataGrip都会为该表生成一个别名。

如果要为数据库对象设置自己的别名，请单击“ 定制别名”表中的“ 添加别名”图标。在“ 表名”列中键入对象名称，在“ 别名”列中键入所需的别名 。

如果无法完成代码，请确保关闭了“ 省电模式”选项（“ 文件” |“省电模式” ）。启用此选项后，DataGrip通过消除后台操作（包括错误突出显示，即时检查和代码完成）来最大程度地减少笔记本电脑的功耗。

智能完成

智能代码完成功能会过滤建议列表，并仅显示适用于当前上下文的类型。

调用智能完成

  1. 开始输入查询。

  2. 按Ctrl+Shift+Space或选择代码|完成|从主菜单中选择SmartType 。

    突出显示了针对当前上下文的最合适建议。

后缀代码完成

后缀代码完成功能可帮助您减少编写代码时向后插入符号的跳跃。您可以根据在点后键入的后缀，表达式的类型及其上下文，将已经键入的表达式转换为其他表达式。

Postfix完成选项显示为基本完成建议列表的一部分。要查看适用于当前上下文的后缀完成的完整列表，请按Ctrl+J 。

  • 要查看可用的后缀模板的完整列表， 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |一般|后缀完成 。

  • 选择Tab ， Space ， 要么Enter扩展后缀模板。

用后缀转换语句

  1. 键入一个表达式，然后在一个点后键入一个后缀。

  2. 按Tab 。

标签和属性的完成

DataGrip自动完成HTML / XHTML，XML / XSL和JSON中标记和属性的名称和值。

标记和属性名称的完成取决于与文件关联的DTD或Schema 。如果没有模式关联，DataGrip将使用文件内容（标记和属性名称及其值）来完成输入。

  1. 按<并开始输入标签名称。DataGrip显示适用于当前上下文的标签名称列表。

    使用ArrowUp和ArrowDown按钮滚动列表。

  2. 按Enter接受列表中的选择。如果您的文件与模式或DTD关联，则DataGrip会根据其自动插入必填属性。

完成提示和技巧

缩小建议列表

您可以通过键入单词的任何部分（甚至是中间位置的字符）或在点分隔符后调用代码完成来缩小建议列表的范围。DataGrip将显示建议，其中包括您在任何位置输入的字符。

这使得不需要使用通配符：

如果使用CamelCase或snake_case名称，请仅输入首字母。DataGrip自动识别并匹配首字母。

接受建议

您可以通过以下方式之一接受列表中的建议：

  • 按Enter或双击列表项以将其插入插入符号的左侧。

  • 按Tab替换插入符号右侧的字符。

  • 使用Ctrl+Shift+Enter使当前的代码构造在语法上正确（平衡括号，添加缺少的花括号和分号，等等）。

您还可以使用特定的键插入所选的完成建议： 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ）选择编辑器|一般|代码完成，然后按空格，点或其他上下文相关的键选项，选择“ 插入所选建议” 。这些键取决于语言，您的上下文等。

虽然此设置可以帮助您节省时间，但是将其打开可能会导致意外插入项目。
