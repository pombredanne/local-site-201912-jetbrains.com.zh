如何连接到SQL Server Express LocalDB

步骤1。检查LocalDB实例

您可以使用DataGrip轻松连接到LocalDB实例。第一步，检查您的LocalDB实例是否已准备好进行连接。找到SqllocalDB.exe并运行SqllocalDB.exe i终端中的命令。您将在服务器上看到可用LocalDB实例的列表。

要创建实例，请运行SqlLocalDB create "DEVELOPMENT" 14.0 -s 。此命令创建一个名为LocalDB的实例DEVELOPMENT使用SQL Server 2017二进制文件并启动实例。如果你省略14.0 ，版本号默认为SqlLocalDB实用程序的版本。

我们要连接到MSSQLLocalDB实例。首先，我们需要通过运行来检查其状态SqllocalDB.exe i MSSQLLocalDB命令。

当前，实例正在运行（请参见State领域）。如果你有Stopped状态，通过运行启动实例SqllocalDB.exe s MSSQLLocalDB在终端。

默认情况下，在几分钟不活动之后，每个LocalDB实例都会停止。您可以通过以下方式更改此行为： user instance timeout T-SQL选项（更多信息，请参见stackoverflow.com ）。从DataGrip 2019.2开始，当IDE连接到LocalDB时，LocalDB实例将自动启动。

第2步。创建Microsoft SQL Server数据源

实例已准备就绪，可以启动DataGrip 。

在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 添加”图标然后选择Microsoft SQL Server 。

第三步配置LocalDB连接

选择驱动程序

从驱动程序列表中，选择Microsoft SQL Server（jTds） 。

选择连接类型

从“ 连接类型”列表中，选择LocalDB 。

选择LocalDB实例

从“ 实例”列表中，选择要连接的实例（例如， MSSQLLocalDB ）。

选择域

打开Advanced标签并找到DOMAIN选项，并将其设置为安装实例的计算机的名称。如果实例在本地计算机上，请使用计算机的名称。

第四步。输入凭据并测试您的连接

如果您不使用Windows域身份验证，请清除Windows域身份验证复选框，然后输入您的访问凭据。请点击Test connection.
