配置Zend调试器

下载并安装Zend Debugger

  1. 下载与您的操作系统相对应的Zend Debugger软件包 。

  2. 在与您的PHP版本相对应的目录中找到ZendDebugger.so （Unix）或ZendDebugger.dll （Windows）文件（例如4.3.x，4.4.x，5.0.x，5.1.x，5.2.x，5.3） .x，5.4.x）。

  3. 将文件复制到Web服务器可以访问的位置的Web服务器中。

启用Zend Debugger与PHP解释器的集成

  1. 在编辑器中打开活动的php.ini文件：

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在“ 语言和框架”下单击“ PHP ” 。

    2. 在打开的PHP页面上，单击 CLI解释器字段旁边。

    3. 在打开的“ CLI解释器”对话框中，“ 配置文件”只读字段显示活动php.ini文件的路径。单击“ 在编辑器中打开” 。

  2. 找到或创建[Zend]部分。

  3. 要加载Zend Debugger扩展，请在的内部添加以下行之一[Zend]部分，具体取决于您的操作系统：

    • Linux和macOS ：

      zend_extension =

    • Windows ：

      zend_extension_ts =

    • Windows非线程安全 ：

      zend_extension =

      Windows非线程安全二进制文件仅与Zend Core 2.0一起使用。

  4. 要允许从PhpStorm访问Zend Debugger，请添加以下行：

    zend_extension = zend_debugger.allow_hosts = 127.0.0.1 zend_debugger.expose_remotely = allowed_hosts zend_debugger.tunnel_min_port = zend_debugger.tunnel_max_port =

    的价值zend_debugger.allow_hosts参数是连接到服务器调试器的计算机的IP。它可能是IP格式的逗号分隔列表X.X.X.X （例如192.168.0.6）。

    对于线程安全的Windows二进制文件，请使用zend_extension_ts parameter代替zend_extension 。

  5. 重新启动您的Web服务器。

  6. 要检查Zend Debugger是否已正确安装和配置，请创建一个包含以下内容的文件：

    在浏览器中打开与文件对应的页面。输出应包含Zend Debugger部分。

将Zend Debugger与PhpStorm集成

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架|的PHP

  2. 检查与所选PHP解释器关联的Zend Debugger安装：

    1. 在“ PHP”页面上，从“ CLI解释器”列表中选择相关的PHP安装，然后单击“确定”。 旁边的领域。该列表显示了PhpStorm中可用的所有PHP安装，请参阅“ 配置本地PHP解释器”和“ 配置远程PHP解释器” 。

    2. 打开的“ CLI解释器”对话框显示以下内容：

      • 所选PHP安装的版本。

      • 与所选PHP安装（Xdebug或Zend Debugger）关联的调试引擎的名称和版本。如果未配置调试器，则PhpStorm显示Debugger: Not installed 。

    在“ 验证调试引擎的配置”中了解有关检查Zend Debugger安装的更多信息。

  3. 定义Zend Debugger行为。单击“ PHP”节点下的“ 调试 ”。在打开的“ 调试”页面上，在“ Zend Debugger”区域中指定以下设置：

    • 在“ 调试端口”字段中，指定PhpStorm通过该端口与工具进行通信的端口。 通过以下命令在php.ini文件中指定的隧道中键入端口号zend_debugger.tunnel_min_port和zend_debugger.tunnel_max_port 。有关详细信息，请参阅http://files.zend.com/help/previous-version/Zend-Server-4-Community-Edition/zenddebugger.html

    • 要使PhpStorm通过“ 调试端口”字段中指定的端口接受来自Zend Debugger引擎的任何传入连接，请选中“ 可以接受外部连接”复选框。

    • 要在浏览器中使用调试器工具栏，请在“ 设置广播端口”字段中指定调试器设置通过其传递到浏览器的端口 。

    • 使用“ 自动检测IDE IP”复选框可以启用和禁用对热IP地址的自动检测。选中此复选框后， PhpStorm检测到所有主机IP地址，这些主机IP地址将通过以下方式发送到Zend Debugger： debug_host参数。所有检测到的IP地址都列在右侧的字段中。使用Vagrant或VirtualBox或其他虚拟化工具时，自动检测IP地址很有用。

      清除复选框以阻止自动检测主机IP地址，并在该字段中明确指定所需的IP地址。

  4. 在“ 外部连接”区域中， 指定您希望PhpStorm如何处理从主机以及通过未注册为部署服务器配置的端口接收的连接。

    • 通过未注册的服务器配置忽略外部连接 ：选中此复选框可使PhpStorm忽略从主机接收的连接以及未注册为部署服务器配置的端口的连接。选中此复选框后，PhpStorm不会尝试自动创建部署服务器配置。

    • 在PHP脚本的第一行中断 ：选中此复选框使调试器与PhpStorm之间建立连接后立即停止（而不是自动运行直到到达第一个断点）。或者，打开运行|从主菜单在PHP脚本选项的第一行中断 。

    • 最高同时连接：使用此旋转框限制可以同时处理的外部连接的数量。

  5. 要阻止来自Z-Ray系统的请求（如果它们过于频繁地调用PhpStorm调试器而使您烦恼），请选中“ 忽略Z-Ray系统请求”复选框。

配置Zend Debugger以在按需模式下使用

PhpStorm 2016.2和更高版本支持按需模式，您可以在其中禁用全局PHP安装的Zend Debugger，仅在调试命令行脚本或需要代码覆盖率报告时才按需自动启用它。这使您的命令行脚本（包括Composer和单元测试）运行得更快。

  1. 为命令行脚本禁用Zend Debugger：

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在“ 语言和框架”下选择“ PHP ” 。

    2. 从PHP可执行文件列表中，选择相关的PHP解释器，然后单击在它的旁边。在打开的“ CLI解释器”对话框中，单击配置文件旁边的“ 在编辑器中打开”链接： 文件。关闭所有对话框，然后切换到打开php.ini文件的选项卡。

    3. 在php.ini文件中，找到[Zend]部分，并通过添加以下内容注释其中的以下几行;在介词中：

      Linux和macOS ：
      zend_extension =
      Windows ：
      zend_extension_ts =

      Windows非线程安全 ：

      zend_extension =

      Windows非线程安全二进制文件仅与Zend Core 2.0一起使用。

    4. 打开“ CLI解释器”对话框，然后单击 PHP可执行文件字段旁边。PhpStorm会通知您未安装调试器：

  2. 要使PhpStorm在必要时激活Zend Debugger，请在“ 其他”区域的“ 调试器扩展”字段中指定其路径。手动输入路径或单击然后在打开的对话框中选择位置。
