反应

React是一个JavaScript库，用于从封装的组件构建复杂的交互式用户界面。从React官方网站上了解有关库的更多信息。

PhpStorm与React集成在一起，可在配置，编辑，整理，运行，调试和维护应用程序方面提供帮助。

在你开始前

  1. 确保您的计算机上装有Node.js。

  2. 确保在“ 插件”页面上启用了JavaScript和TypeScript捆绑的插件 ，有关详细信息，请参阅管理插件 。

创建一个新的React应用程序

推荐的方式开始建立一个新的阵营单页的应用是创建反应的应用程序内包，PhpStorm下载并运行你使用哪个NPX 。因此，您的开发环境已预先配置为使用webpack，Babel，ESLint和其他工具。

当然，您仍然可以自己下载Create React App或创建一个空的PhpStorm项目并在其中安装React。

从React官方网站上了解有关安装React和创建React应用程序的更多信息。

使用create-react-app生成React应用

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择React App 。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 在“ 节点解释器”字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择添加以配置新的解释器。

    3. 从create-react-app列表中，选择npx create-react-app 。

      另外，对于npm 5.1及更早版本，请安装create-react-app通过运行打包自己npm install --g create-react-app在航站楼 Alt+F12 。创建应用程序时，选择create-react-app包已存储。

  4. 可选地：
    指定在项目生成期间要使用的自定义程序包，而不是react-scripts 。这可以是从react-scripts分叉的软件包之一，例如， react-awesome-scripts ， custom-react-scripts ， react-scripts-ts等。

    PhpStorm保证仅使用react-scripts软件包运行和调试Jest测试。

  5. 当您单击Create时 ，PhpStorm会使用所有必需的配置文件生成一个特定于React的项目，并下载所需的依赖项。PhpStorm还使用默认设置创建一个npm start和JavaScript Debug配置，以运行或调试您的应用程序。

或者，打开内置终端并输入：

  1. npx create-react-app 创建一个应用程序。

  2. cd 切换到应用程序文件夹。

  3. npm start启动Node.js服务器。

要下载项目依赖项，请执行以下任一操作：

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install

  • 从项目根目录中package.json文件的上下文菜单中选择Run'npm install' 。

在PhpStorm中开始使用ESLint

  1. 在内置终端 （ View | Tool Windows | Terminal ）中，键入npm install --save-dev eslint和npm install --save-dev eslint-plugin-react 。

  2. 将ESLint配置文件.eslintrc添加到您的项目。

  3. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | JavaScript |代码质量工具| ESLint 。在打开的ESLint页面上，选择自动ESLint配置 。PhpStorm将自动在项目的node_modules文件夹中找到ESLint，然后默认使用.eslintrc配置。

    您还可以手动配置ESLint以使用自定义ESLint软件包和配置。

运行和调试React应用程序

建议开始构建新的React单页应用程序的方法是Create React App 。仅在这种情况下，您的开发环境才被预先配置为使用webpack和Babel。否则，您需要首先配置构建管道 。

运行一个React应用程序

仅适用于使用create-react-app 。

  • 在npm工具窗口 （ View | Tool Windows | npm ）中，双击start任务。

    由于使用了Webpack Hot Module Replacement ，因此，当开发服务器运行时，只要更改任何源文件并保存更新，应用程序就会自动重新加载。

调试React应用程序

仅适用于使用create-react-app 。

  1. 通过在npm工具窗口中双击启动任务，以开发模式启动应用程序。

  2. 等待应用程序被编译并且Webpack开发服务器准备就绪。在http：// localhost：3000 /中打开浏览器以查看该应用程序。

  3. 复制运行应用程序的URL地址（默认为http：// localhost：3000 / ），以后在创建调试配置时将需要此URL。

  4. 创建一个新的JavaScript调试配置：选择Run |编辑配置 ，单击 ，然后从列表中选择JavaScript Debug 。在“ 运行/调试配置：JavaScript调试”对话框中，将保存的URL http：// localhost：3000 /粘贴到URL字段中。保存配置。

  5. 在代码中设置断点，然后通过单击启动调试会话配置列表旁边。

  6. 遇到第一个断点时，切换到“ 调试工具”窗口，然后照常进行： 逐步执行程序 ， 停止并继续执行程序， 在挂起时对其进行检查 ，浏览调用堆栈和变量，设置监视，评估变量， 查看实际HTML DOM等等。

构建一个React应用

如果在现有的PhpStorm项目中安装了React，则需要设置构建过程。从React官方网站了解有关为React应用程序配置构建管道的各种方法。

如果您使用create-react-app创建应用程序，则您的开发环境已经预先配置为使用Webpack和Babel。

测试React应用

您可以在使用create-react-app创建的 React应用程序中运行和调试Jest测试 。在开始之前，请确保将react-scripts包添加到package.json的依赖项对象中。

您可以通过运行/调试配置来运行和调试Jest测试，或者直接在编辑器中或在“ 项目”工具窗口中运行Jest测试，有关详细信息，请参见Jest 。

创建Jest运行/调试配置

  1. 打开“ 运行/调试配置”对话框（主菜单上的“运行” |“编辑配置 ”），单击在左侧窗格中，然后从列表中选择Jest 。“ 运行/调试配置：Jest”对话框打开。

    或者，在“ 项目”工具窗口中选择一个测试文件，然后选择“ 创建” 从上下文菜单中。

  2. 指定要使用的Node解释器以及应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  3. 在“ Jest包”字段中，指定react-scripts包的路径。

  4. 在“ 笑话选项”字段中，键入--env=jsdom 。

运行测试

  1. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右边。

  2. 测试服务器将自动启动，而无需您采取任何步骤。在“ 运行工具”窗口中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

调试测试

  1. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右边。

  2. 在打开的“ 调试工具”窗口中，照常进行： 逐步执行测试 ， 停止并继续执行测试，在挂起时检查测试 ，依此类推。

已知限制

第一次在调试会话期间打开应用程序时，可能会发生以下情况：页面加载时执行的代码中的某些断点未命中。原因是，要在原始源代码中的断点处停止，PhpStorm需要从浏览器获取源映射。但是，浏览器只能在页面完全加载至少一次之后才能传递这些源映射。解决方法是，自己在浏览器中重新加载页面。
