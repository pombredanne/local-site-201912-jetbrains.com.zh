内置Web服务器

PhpStorm具有内置的Web服务器，可用于预览和调试应用程序。该服务器始终处于运行状态，不需要任何手动配置。所有项目文件都以根URL http：// localhost： /内置服务器上提供 ，关于项目结构。

内置服务器只能提供HTML，JavaScript和CSS等静态内容。要将其与PHP文件一起使用，您需要为项目指定一个本地PHP解释器 。配置解释器后，运行PHP应用程序后，PhpStorm将自动启动PHP内置Web服务器并将所有PHP请求重定向到该服务器 。要运行您的PHP应用程序，请在浏览器中打开一个文件，或者创建专用的运行/调试配置并启动它 。

PhpStorm 4.0及更高版本支持与PHP内置Web服务器的集成。

在网络浏览器中打开文件

要打开旨在由网络浏览器呈现的文件（HTML，XML，JSP等），请执行以下任一操作：

  • 按Alt+F2 。

  • 右键单击文件，然后选择“ 在浏览器中打开” 。

  • 在主菜单中，选择查看|在浏览器中打开 。

  • 使用编辑器窗口右上方的浏览器弹出窗口。单击浏览器按钮以打开Web服务器文件URL，或者Shift+Click它打开本地文件的URL。

“ 在浏览器中打开”操作不适用于其他文件类型。但是，您仍然可以使用“ 查找操作”执行它Ctrl+Shift+A 。

如果为此项目定义了一个部署服务器并将其标记为默认服务器，则将从该服务器提供文件。如有必要，您仍然可以通过PhpStorm内置的Web服务器打开页面。为此，请打开所需的浏览器，并使用http：// localhost： /相对于项目结构键入文件的URL。 作为根URL。

有关使用部署服务器的更多详细信息，请参阅“ 使用Web服务器：复制文件”部分。

配置内置的Web服务器

要自定义内置Web服务器的参数， 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 构建，执行，部署” |调试器 。

港口

使用此旋转框可以指定运行内置Web服务器的端口。默认情况下，此端口设置为端口63342 ，PhpStorm通过该端口接受来自服务的连接。您可以将端口号设置为以1024或更高版本开头的任何其他值。

可以接受外部连接

如果选中此复选框，则可以从另一台计算机访问在指定端口上运行的内置服务器上的文件。
如果清除此复选框（默认情况下），则调试器仅侦听本地连接。

允许未签名的请求

 出于安全原因，默认情况下，拒绝PhpStorm外部对内置服务器上的页面的任何请求，并显示以下授权弹出窗口：要访问请求的页面，请单击“ 将授权URL复制到剪贴板” ，然后将生成的令牌粘贴到浏览器的地址栏中。

但是，此行为可能很烦人，例如，如果不可能进行手动干预，则可能会阻止调试会话。要取消显示授权弹出窗口，请选中“ 允许未签名的请求”复选框。

在macOS上使用PHP内置的Web服务器

在macOS上，尝试使用内置服务器时可能会出现php-cgi not found错误。这表明PhpStorm已配置为使用与macOS捆绑在一起的PHP解释器，其中不包含php-cgi模块。

为了能够使用PHP内置的Web服务器，请执行以下操作：

  1. 安装附带的php-cgi模块的备用PHP解释器。最常见的选项如下：

    • PHP Brew软件包
    • Liip PHP-OSX软件包
    • AMP包装

  2. 如配置本地PHP解释器中所述，指定新安装的PHP解释器在PhpStorm中使用。
