Angular中的代码检查

检查

描述

默认严重性

无效的表达式结果类型

报告Angular绑定表达式的错误返回类型。

警告

未定义从Angular模块导出

报告从Angular模块导出的未声明或未导入的组件，指令和管道。

错误

不受约束或模棱两可的模板参考变量

警告有关在使用时未分配给指令的模板引用变量exportAs或分配给多个指令。

错误

组件标签不明确

如果嵌入式模板元素上的组件匹配，则发出警告 ，或者在任何其他元素上匹配多个组件。

错误

Angular CLI添加依赖项

建议使用“ ng add”命令安装依赖项。

“ ng add”将使用包管理器下载它并调用原理图，该原理图可以通过配置更改来更新您的项目，添加其他依赖项（例如polyfills）或脚手架特定于包的初始化代码。

警告

在AOT模式下无法访问的组件成员

从组件模板访问组件的私有成员或受保护成员时发出警告。这种用法会导致AOT编译失败。

 残障人士
对事件的不安全绑定

报告与事件属性或属性的绑定，例如[onclick]要么[attr.onclick]代替(click) 。

警告

无效的输入组件

检查模块的中是否指定了有效的Angular组件bootstrap要么entryComponents属性。

错误

模块中缺少或无效的组件，指令或管道声明

警告未在任何模块中声明或在多个模块中声明的Angular组件，指令或管道。

错误

无效的动画触发器分配

警告有关动画触发器的无效分配。要将动画附加到元素，请使用[@triggerName]="expression"或没有值的属性@triggerName 。

错误

组件模板定义不正确

如果组件没有关联的模板或同时使用两者，则发出警告template和templateUrl属性。

错误

递归导入或导出Angular模块

报告Angular模块之间的循环依赖关系。

错误

标签内的内容

检查用于内容投影的标记中没有任何文本或其他标记。

错误

无效的导入或声明的符号

报告由Angular模块声明，导入或导出的任何符号，这些符号不是模块，组件，指令或管道，或者不能在属性的上下文中使用。

错误

一个元素上有多个结构指令

报告多个结构指令的使用情况（ *ngIf ， *ngFor等）。

错误

选择器缺失或无效

验证组件或指令selector属性。

错误

未定义的绑定

报告元素上未定义的属性，事件或结构指令绑定。

错误

缺少事件处理程序

警告有关事件绑定的事件处理程序语句丢失。

错误

未定义标签

报告标记，这些标记由超出当前范围的组件或指令定义。
