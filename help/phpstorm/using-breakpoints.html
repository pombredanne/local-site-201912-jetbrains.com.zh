断点

断点是源代码标记，可让您在特定点挂起程序执行并检查其行为。

设置后，断点将保留在您的项目中，直到您将其明确删除为止（ 临时行断点除外）。如果具有断点的文件在外部进行了修改（例如，通过VCS更新或在外部编辑器中进行了更改），并且行号已更改，则断点将相应地移动。请注意，进行此类更改后，PhpStorm必须正在运行，否则它们将被忽略。

行断点

可以在可执行代码行上设置行断点。在具有此类断点的行之前，线程执行被挂起，并且PhpStorm在该线程的堆栈上显示堆栈帧。

设置行断点

  1. 将插入符号放在要设置断点的可执行代码行中。

  2. 单击该行旁边的左装订线或按Ctrl+F8 。

    要设置临时断点，请按Ctrl+Shift+Alt+F8 。选中后立即将其从您的项目中删除。

删除行断点

  • 单击左侧装订线中的断点图标。

为避免意外删除断点并丢失其参数，您可以选择通过以下方式删除断点：将其拖动到编辑器中，或单击鼠标中键： 在里面 设置/首选项对话框Ctrl+Alt+S转到构建，执行，部署|调试器，然后选择“ 拖动到编辑器”或使用鼠标中键单击 。单击断点将随后切换其状态（启用/禁用）。

方法断点

方法断点响应于程序进入或退出特定方法而起作用。它们使您可以在方法级别上遵循程序流程，并检查进入和退出条件。

设置方法断点

  • 单击带有方法声明的行旁边的左侧装订线。

或者，执行以下操作：

  • 按Ctrl+Shift+F8或选择运行|从主菜单查看断点 。

  • 在打开的“ 断点”对话框中，按Alt+Insert或点击 ，然后选择PHP方法断点 。

  • 在“ 添加方法断点”对话框中，指定类和方法，或指定要为其添加断点的普通函数。

删除方法断点

  • 单击方法断点图标在左排水沟中。

或者，执行以下操作：

  • 按Ctrl+Shift+F8或选择运行|从主菜单查看断点 。

  • 在“ 断点”对话框中，在“ PHP方法断点”下选择要删除的断点 ，然后按Delete或点击 。

为避免意外删除断点并丢失其参数，您可以选择通过以下方式删除断点：将其拖动到编辑器中，或单击鼠标中键： 在里面 设置/首选项对话框Ctrl+Alt+S转到构建，执行，部署|调试器，然后选择“ 拖动到编辑器”或使用鼠标中键单击 。单击断点将随后切换其状态（启用/禁用）。

异常断点

抛出指定的异常时会触发异常断点。它们全局适用于异常条件，并且不需要特定的源代码引用。

使用PHP异常断点，可以在脚本开始时启动调试器，并在发生指定的错误或异常时中断调试器。PHP异常断点不需要为Xdebug启用即时模式（请参阅“ 即时模式”中的调试 ）。

设置异常断点

  1. 按Ctrl+Shift+F8或选择运行|从主菜单查看断点 。

  2. 在“ 断点”对话框中，按Alt+Insert或点击 ，然后选择PHP Exception Breakpoint或JavaScript Exception Breakpoint 。

  3. 在“ 添加例外断点”对话框中，从库或项目中指定例外类。

删除异常断点

  1. 按Ctrl+Shift+F8或选择运行|从主菜单查看断点 。

  2. 在“ 断点”对话框中，选择要删除的断点，然后单击“确定”。 或按Alt+Delete 。

断点属性

根据断点类型，您可以配置其他属性，例如：

  • 达到断点时要执行的操作

  • 挂起策略，用于定义在遇到断点时是否必须挂起应用程序

  • 对其他断点的依赖

  • 定义何时必须命中断点的条件

要编辑断点属性，请按Ctrl+Shift+F8 ，或在编辑器装订线中右键单击断点。
