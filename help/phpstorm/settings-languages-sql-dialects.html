SQL方言

档案|设置|语言和框架| Windows和Linux的SQL方言
PhpStorm |首选项|语言和框架|适用于macOS的SQL方言

该页面使您可以指定在各种范围内使用的SQL方言（SQL的DBMS特定版本）。

“文件”，“项目”和“全局”是定义单个方言，在项目中附加有SQL文件的文件夹或在IDE中打开的每个SQL文件的范围内要应用的方言的作用域。如果未设置方言，则将应用通用方言。在这种情况下，某些特定于方言的功能将不可用：针对该方言的检查，该方言的代码完成。另外，每次在IDE中打开新文件时，都需要指定适用的数据源。

项目范围覆盖全局范围。这意味着，如果将Global方言设置为MySQL，将Project方言设置为MariaDB，则项目范围之外的所有文件将自动获得MySQL代码帮助。

总体而言，如果您使用单个DBMS（例如PostgreSQL）并且要激活特定于该DBMS的代码帮助，则指定Global方言很有用。否则，如果您有两个使用PostgreSQL和Oracle的项目，则最好为每个项目指定Project方言。

为了使此页面可用，必须启用数据库工具和SQL插件。请参阅管理插件 。

描述

全球SQL方言

您计算机上所有.sql和.ddl文件的SQL语言；可以在较窄的范围内重新定义-在项目级别和/或单个文件和目录。

SQL方言项目

当前项目中所有.sql和.ddl文件的SQL方言。如果指定后，将继承全局SQL方言。

路径/ SQL方言

单个文件和目录的SQL方言-如果不同于全局方言或项目方言。

如果在本节中未指定任何内容，则项目中的所有.sql和.ddl都将继承项目的方言以及项目外部的所有文件-全局方言。

要为文件或目录指定方言，请单击然后在打开的对话框中选择文件或目录。然后点击或“ SQL方言”单元格，然后选择方言。

明确指定的方言以黑色显示。继承的方言（除非关闭对话框）以灰色斜体显示。

方言选项

指定方言时，除了特定的方言外，您还可以选择：

  • 要么 。结果，继承了更高级别的方言。

  • 。 支持SQL关键字，表和列名的完成和突出显示。禁用此选项的语法错误突出显示功能（文件内容在语法上显示为正确）。

例

假设您计算机上的大多数SQL脚本文件都用于PostgreSQL。在当前项目中，您正在开发Oracle的脚本，但是在项目中的目录之一中，有MySQL的脚本。在这种情况下，您需要指定：

  • 全球SQL方言： PostgreSQL

  • SQL项目方言： Oracle

  • ：MySQL
