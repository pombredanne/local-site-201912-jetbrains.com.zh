根据Joomla编码标准检查代码

借助PhpStorm，您可以结合使用PHP_CodeSniffer工具和Joomla编码标准来检测编码标准问题， Joomla Coding Standards为PHP_CodeSniffer提供了一组特定于Joomla的标准。这将确保您的代码干净，一致并且没有一些常见错误。

首先，请使用安装和配置PHP_CodeSniffer中描述的任何技术来安装PHP_CodeSniffer 。可能最简单的方法是使用Composer安装它。

要同时安装两个软件包，请将它们添加到require-dev composer.json部分，然后按照Composer中所述运行Install 。

安装PHP_CodeSniffer

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择squizlabs / php_codesniffer软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

接下来，安装Joomla编码标准 ，它将为PHP_CodeSniffer提供特定于Joomla的标准。

安装Joomla编码标准

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择joomla / coding-standards软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

将安装Joomla编码标准软件包，并自动为PHP_CodeSniffer验证检查选择相应的Joomla标准。如有必要，您可以在“ 编辑器” | “ 编辑器”中进一步自定义检查。 检查页面的 设置/首选项对话框Ctrl+Alt+S 。有关详细信息，请参见将PHP_CodeSniffer配置为PhpStorm检查 。
