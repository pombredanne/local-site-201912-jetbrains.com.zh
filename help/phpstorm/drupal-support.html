PhpStorm 2019.3帮助

Drupal

您可以将PhpStorm用作用于Drupal开发的IDE，包括模块，主题和核心。的支持的版本为6、7和8。

PhpStorm在开发Drupal模块和核心时提供Symfony2和Drupal 8之间的集成。

下载并安装Drupal

从Drupal官方网站下载Drupal并按照Drupal安装说明中的说明进行安装 。

在PhpStorm中设置Drupal项目

选项1：使用Drupal模块创建PhpStorm项目

您可以通过Drupal模块模板创建PhpStorm项目，该项目将根据Drupal要求进行设置和配置。

PhpStorm会根据所选的Drupal版本生成并配置项目存根。对于Drupal 8，将生成一个module_name.info.yml文件。

  1. 打开“新建项目”对话框

    选择文件|新建项目，或在PhpStorm欢迎屏幕上单击创建新项目 。

  2. 选择项目类型和位置

    在左侧窗格中，选择Drupal Module 。

    在右侧窗格的“ 位置”字段中，指定将在其中创建项目的文件夹。

  3. 选择要使用的Drupal安装

    指定Drupal安装的根文件夹，然后选择要使用的Drupal版本，即支持的版本为6、7和8。

    之后，您可以按照更改Drupal设置中所述更改 Drupal安装并重新配置Frameworks页面上的包含路径。

  4. 配置包含路径

    选择设置PHP |包含路径复选框可让Drupal包含为项目自动配置的路径。 退出对话框后，以下路径将添加到PHP页面上的“ 包含路径”列表中： / includes ， / modules和/ sites / all / modules

选项2：导入现有的Drupal模块

当您打开现有的Drupal模块时，PhpStorm会识别出Drupal特定的结构并建议激活Drupal支持。

  1. 打开你的项目

    在“欢迎”屏幕上单击“ 打开 ”，或选择“ 文件” |“选择” 。从主菜单打开 ，然后选择存储您的Drupal模块的文件夹。

    PhpStorm检测到Drupal特定的结构并显示一条通知：

  2. 启用Drupal支持

    单击通知中的启用 。在打开的Drupal模块对话框中，选中Enable Drupal集成复选框，并按照创建带有Drupal模块的项目的方式进行操作：指定Drupal安装的根文件夹，选择要使用的版本，并配置包含路径。

无论是在现有的PhpStorm项目中启用Drupal支持，还是使用Drupal模块创建新项目，PhpStorm都会检查是否为Drupal开发正确配置了开发环境。

任何检测到的不一致都会在“ 事件日志”工具窗口中报告，并作为弹出窗口报告。要应用建议，请单击所报告事件旁边的链接。

将特定于Drupal的文件与PHP文件类型相关联

PhpStorm会将文件识别为php文件，并根据文件类型关联提供代码突出显示。

选项1：使用事件日志

  • 在“ 事件日志”工具窗口中，单击Drupal支持旁边的“ 修复” ：<*。文件扩展名>文件未与PHP文件类型消息关联 。

选项2：使用“文件类型”页面

  • 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到编辑器|文件类型 。

    在打开的“ 文件类型”页面上，在“已注册模式”区域中定义文件掩码。有关详细信息，请参见设置文件类型关联 。

更改Drupal设置

选项1：启用或禁用Drupal集成

  • 在里面 设置/首选项对话框Ctrl+Alt+S ，导航至语言和框架| PHP |框架 。在打开的“ 框架”页面上，选中“ 启用Drupal集成”复选框以激活或停用当前PhpStorm项目中的Drupal。
    要使用其他Drupal安装，请键入相关安装文件夹的路径。如有必要，请更改版本。

选项2：更新包含路径

  • 在里面 设置/首选项对话框Ctrl+Alt+S ，导航至语言和框架| PHP 。

    在打开的PHP页面上，在“ 包含路径”区域中进行所需的更改。

如果禁用了Drupal集成，则会自动排除路径。

在PhpStorm中使用Drupal挂钩

PhpStorm在.module文件中提供了Drupal挂钩的完全本机支持。

  • 使用代码完成功能进行钩子声明

    PhpStorm会为任何钩子调用建立索引，从而使钩子名称在代码完成中可用于创建钩子实现。要完成声明，请开始输入挂钩名称，然后按Ctrl+Space 。

  • 导航到挂钩调用

    • 要从编辑器导航到挂钩调用，请单击天沟中的图标。

    • 如果有多个调用，PhpStorm将显示可用的挂钩调用的列表，您可以选择要跳转到的列表。

      您将导航到使用以下代码调用相关钩子的行module_invoke_all() ， module_invoke() ， 要么drupal_alter() 。

  • 查看挂钩文档

    将插入号放置在钩子的名称处，然后选择“ 查看” |“添加”。快速文档查找或按Ctrl+Q 。

文档摘自core和其他许多模块提供的.api.php文件，仅供参考。

在PhpStorm项目中设置Drupal代码样式

如果将一个项目识别为Drupal模块，或者在现有项目中启用了Drupal集成，或者使用Drupal模块创建新项目，则PhpStorm会自动提供应用预配置的Drupal编码标准（代码样式）的功能。但是，您可以随时在“设置”对话框的“代码样式：PHP”页面上更改或自定义此设置。

在项目中使用预先配置的Drupal代码样式

  • 在事件日志工具窗口中，单击旁边的Drupal的样式格式 设置 可以为这个项目的消息进行设置 。

您还可以在“ 代码样式：PHP”页面上设置预定义的代码样式。 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到编辑器|代码样式| PHP 。然后，单击“ 设置自”，然后选择“ 预定义| Drupal 。

如果适用，请使用页面上的控件自定义代码样式。

根据Drupal编码标准检查代码

使用PhpStorm，您可以将PHP_CodeSniffer工具与Coder结合使用，该工具可检测编码标准问题，而Coder为PHP_CodeSniffer提供了一组Drupal特定的标准。这将确保您的代码干净，一致并且没有一些常见错误。

首先，请使用安装和配置PHP_CodeSniffer中描述的任何技术来安装PHP_CodeSniffer 。可能最简单的方法是使用Composer安装它。

要同时安装两个软件包，请将它们添加到require-dev composer.json部分，然后按照Composer中所述运行Install 。

安装PHP_CodeSniffer

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择squizlabs / php_codesniffer软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

接下来，安装Coder ，它将为PHP_CodeSniffer提供Drupal特定的标准。

安装编码器

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，可能使用搜索字段从“ 可用软件包”列表中选择drupal / coder软件包。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

将安装Coder软件包，并自动为PHP_CodeSniffer验证检查选择相应的Drupal标准。如有必要，您可以在“ 编辑器” | “ 编辑器”中进一步自定义检查。 检查页面的 设置/首选项对话框Ctrl+Alt+S 。有关详细信息，请参见将PHP_CodeSniffer配置为PhpStorm检查 。

从PhpStorm查看Drupal API文档

在PhpStorm编辑器中，选择您感兴趣的符号，然后从上下文菜单中选择“在Drupal API中搜索” 。将打开Drupal API文档 。

使用PhpStorm的Drush命令行工具

PhpStorm与Drush命令行外壳和脚本界面（版本5.8及更高版本）集成。在开始之前，请按照https://github.com/drush-ops/drush中的说明下载并安装Drush。

将Drush配置为命令行工具

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到工具|命令行工具支持 。

  2. 请点击在工具栏上。

  3. 在“ 命令行工具”对话框中，从列表中选择“ 绘制 ”，然后指定其可见性级别（“ 项目”或“ 全局” ）。

  4. 当您单击OK时 ，将打开Drush对话框。

    指定Drush可执行文件的路径。默认位置通常是Windows上的C：/ProgramData/Drush/drush.bat和macOS或Linux上的/ usr / bin / drush 。如果您遵循标准安装过程，请单击“ 确定”以接受预定义的路径。

    如果是自定义安装，请键入Drush可执行文件的路径，然后单击“ 确定” 。

  5. 单击“ 确定”以应用更改并返回“ 命令行工具支持”页面。（可选）单击编辑工具属性，或自定义命令集。有关详细信息，请参见自定义工具 。

运行Drush命令

  • 在主菜单中，选择“ 工具” |“工具”。运行命令或按Ctrl两次。

    在打开的“运行任何内容”窗口中，在 格式。

    命令执行结果显示在“ 运行工具”窗口中 。

终止命令

  • 请点击在运行工具窗口工具栏上。

调试Drush命令

Drupal命令在扩展的控制器类中定义DrushCommands 。要调试命令，至关重要的是调试命令本身，而不是调试定义在其中的控制器类文件。否则，将跳过Drupal引导过程，并且执行将失败。

  1. 在与所选命令对应的控制器类中，在要设置断点的代码行上单击编辑器装订线。

  2. 创建一个运行/调试配置 ，该配置将使用所选命令运行drush工具。在主菜单中，选择运行|编辑配置 ，然后单击然后从列表中选择PHP脚本 。

  3. 在“ 运行/调试配置：PHP脚本”对话框中，提供运行/调试配置参数。

    • 在“ 文件”字段中，提供Drush可执行文件的路径。

    • 在“ 参数”字段中，键入实际的命令及其参数，例如status 。

  4. 在PhpStorm工具栏上，选择创建的运行/调试配置，然后单击 。命令执行将在指定的断点处停止。

在Symfony2中使用Drupal 8

PhpStorm在Drupal版本8和Symfony2之间提供了紧密的集成。通过这种集成，Symfony2组件与Drupal基础架构相连。要利用这种集成：

  1. 安装Drupal Symfony2 Bridge插件

    在里面 设置/首选项对话框Ctrl+Alt+S ，导航到“ 插件”页面 。单击浏览存储库 ，选择插件，然后单击安装 。有关详细信息，请参阅管理插件 。

  2. 启用注释

    要获得高级注释支持，请按照管理插件中所述安装PHP注释插件 。

有什么针对Drupal 8-Symfony2集成的编码帮助？

  • .info.yml文件中包含与Drupal模块有关的信息的主要Drupal特定参数的完成。

  • 适合在Drupal特定的t（）函数中使用的字符串将在整个项目中建立索引并提供完成。

  • 完成url（）和其他Drupal API函数中的相关yml键值，这使搜索正确的值变得更加容易。

  • 通过按导航到yml文件Ctrl+B或选择导航|去声明 。

  • 全面支持yml文件中描述的服务容器 ，包括完成和导航Ctrl+B 。

  • 支持Twig模板引擎 ，该引擎现在是Drupal 8的默认模板引擎，包括完成，导航和识别Drupal功能。另请参阅Drupal 8中的Twig 。
