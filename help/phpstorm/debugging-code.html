PhpStorm 2019.3帮助

调试

PhpStorm提供了用于调试源代码的各种工具：

  • PHP中的断点。

  • HTML和JavaScript的断点。

  • 多个同时调试会话。

  • 可定制的断点属性：条件，通过次数等。

  • 调试器用户界面中的框架 ， 变量和监视视图。

  • 表达式的运行时评估 。

  • 支持Xdebug和Zend Debugger。

本节描述了各种类型的应用程序和PHP框架通用的过程。

开始调试会话

  1. 确定您要使用的调试引擎： Xdebug或Zend Debugger 。选择引擎后，按照配置调试器引擎中所述安装和配置引擎 。

  2. 要调试CoffeeScript ， TypeScript和Dart代码，您需要为其生成源映射 。这将设置原始代码和生成的JavaScript代码中各行之间的对应关系。如果未生成任何源映射，则将无法正确识别和处理您的断点。

  3. 根据您的调试模式，定义运行/调试配置 。

  4. 创建断点 。

  5. 按照PHP调试会话中所述启动调试会话 。或者，按Shift+F9 。

根据需要暂停或恢复会话。

在调试会话期间，您可以评估表达式 ，动态更改值， 检查挂起的程序并设置watch 。

启动调试会话后， 标记“ 调试工具”窗口的图标切换到指示调试过程处于活动状态。

如果要查看所有当前调试的应用程序的列表，请选择“运行” |“运行”。从主菜单显示运行列表 。有关详细信息，请参考查看运行过程部分。

除错

为项目配置运行配置后 ，可以按以下方式以调试模式启动它： Shift+F9 。

在“ 调试”工具窗口中，您可以看到框架和线程及其状态，变量和监视的列表。选择框架时，将看到与所选框架相对应的变量。

调试器会话

运行到光标

  1. 创建一个断点。

  2. 运行调试会话。要运行调试会话，请单击“ 运行应用程序”图标（ ）在装订线区域，然后选择调试 。

要停止在光标位置执行代码而不添加其他断点，请单击“ 运行到光标”图标（ ）或按Alt+F9 。或者，您可以单击装订线区域中的行号。

图标在“调试工具”窗口的工具栏参考中进行了介绍。

配置调试器设置

  • 在里面 设置/首选项对话框Ctrl+Alt+S ，使用“ 构建”，“执行”，“部署”下的页面|调试器和语言与框架| PHP |调试 。
