国际化中的代码检查

检查

描述

默认严重性

非ASCII字符

此检查会警告您意外以错误编码编写的代码。
例如，

  • 名称/字符串/注释中使用的非ASCII字符或

  • 以不同语言编写的标识符（例如，在变量“ my C ollection信C以西里尔文撰写）或

  • 注释或字符串中使用的Unicode符号（例如注释中的长破折号和箭头“ // a → !a — please note ”）

警告

有损编码

此检查会警告您有关当前文档编码无法表示的字符。
例如，当你

  • 在配置为US-ASCII字符集的文档中输入国际字符。保存时某些字符将丢失。

  • 或使用ISO-8859-1一字节字符集加载UTF-8编码的文件。某些字符将显示不正确。

您可以通过更改文件编码来解决此问题，方法是直接在文件中指定编码，例如通过编辑XML文件的XML序言中的encoding =属性，或者配置Settings | Project Settings | File Encodings 。
