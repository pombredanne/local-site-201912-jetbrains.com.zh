分析Zend调试器性能分析数据

启用与Zend Debugger探查器的集成后，PhpStorm将提供探查器快照的直观表示。PhpStorm将打开一个单独的编辑器选项卡，该选项卡具有四个视图，这些视图根据不同的条件显示数据。

使用Zend Debugger， 零配置调试会话中支持概要分析。

要收集和分析配置文件数据，请执行以下常规步骤：

  • 启动零配置Zend Debugger会话
  • 检查分析数据

启动零配置Zend Debugger会话

  1. 生成书签以切换调试器。 这些书签将显示在浏览器的工具栏上。它们提供了对调试器cookie的控制，通过它们您将激活和停用调试器。

    1. 通过执行以下操作之一（取决于浏览器类型），在浏览器中启用“ 书签”工具栏 ：

      • 在Firefox中 ，选择“ 查看|工具栏书签工具栏 。

      • 在Chrome中 ，选择书签|显示书签栏 。

    2. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |调试 。

    3. 在打开的“ 调试”页面上，单击“ 使用调试器”书签，以从您喜欢的浏览器链接启动调试器 。

    4. 在打开的Zend Debugger＆Xdebug书签页面上，检查调试引擎设置，然后单击Generate 。生成列出的与调试相关的动作的书签。

    5. 将生成的链接拖动到浏览器中的书签工具栏。

  2. 切换“ 开始侦听PHP调试连接”按钮在PhpStorm工具栏上，使其更改为 。之后，PhpStorm开始侦听当前项目中使用的调试引擎的端口。调试端口在“ 语言和框架”上的“ PhpStorm”级别上设置。 PHP |调试页面的 设置/首选项对话框Ctrl+Alt+S 。

  3. 在浏览器中打开应用程序的起始页 ，选择“ 启动调试器”书签以从浏览器激活调试引擎，重新加载当前页面（应用程序的起始页），然后返回到PhpStorm。

    建立零配置调试会话可能会失败，没有断点命中，因此脚本不会挂起。如果发生这种情况如果未正确配置路径映射或配置错误，或者尚未设置任何断点。在后一种情况下，启用“ 外部连接”区域中的“ 在PHP脚本中的第一行中断”复选框，或启用“ 运行” |“启用”。主菜单中的“ PHP脚本中的第一行中断”选项也可能会有所帮助。

    要使PhpStorm在脚本未挂起时显示通知，请在“设置”对话框的“ 调试”页面上的“ 高级设置”区域中，选中“ 如果调试会话没有结束就通知它” 。

  4. 在打开的对话框中，选择配置文件的传入连接，然后点击接受 。当您首次接受来自该主机的连接时，“ 从Zend Debugger传入的连接”对话框仅出现一次。

    PhpStorm在单独的编辑器选项卡中显示收集的性能分析数据，其中包含选定的探查器输出文件的名称。

检查分析数据

当您请求累积的性能分析数据时，PhpStorm在单独的编辑器选项卡中打开其可视化演示文稿。该选项卡以实现您当前正在分析的页面的文件命名，并包含多个视图。在视图之间切换以基于各种分析标准来分析概要分析数据。

  • 在“ 执行统计信息”视图中，检查有关每个调用函数的执行指标的摘要信息。

  • 在“ 调用树”视图中，浏览所有被调用函数的执行路径。

  • 要浏览特定功能的执行路径，请在“ 调用树”视图中选择该功能，然后在“ 被调用者”视图中查看其被调用者 。

  • 要探索可能导致调用特定功能的所有路径，请在“ 调用树”视图中选择该功能，然后在“ 调用者”视图中检查其可能的调用者 。
