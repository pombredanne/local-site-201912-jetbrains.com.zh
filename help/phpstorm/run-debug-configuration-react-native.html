运行/调试配置：React Native

运行编辑配置| |反应本机

在此对话框中，创建用于运行React Native应用程序的配置。

在你开始前

可以访问“运行/调试配置：React Native”对话框

  1. 下载并安装Node.js。

  2. 确保您的计算机上有一个React Native软件包，例如react-native-cli 。

反应特定于本机的配置设置

项目

描述

生成并启动应用程序

  • 如果是第一次启动应用程序，或者自上次运行以来已更新其本机代码，请选中“ 构建并启动”复选框。

  • 如果自上次构建以来未对应用程序的本机代码进行任何更改，请清除此复选框。当您开始调试时，PhpStorm等待您在模拟器中打开您的应用程序，并启用了React的远程调试，如React Native官方网站所述。

  • 如果您的应用程序使用Expo ，请清除该复选框，因为此捆绑程序将自行处理。

平台

在此区域中，选择应用程序所针对的平台。 根据您的选择，PhpStorm将使用react-native run-ios或搭配react-native run-android 。

  • 要模拟Android平台，请使用Android虚拟设备 。

  • 要模拟iOS平台，您需要全局安装ios-sim命令行工具 。您可以通过Node Package Manager（npm） （请参阅npm和Yarn ）或通过运行sudo npm install ios-sim -g命令，具体取决于您的操作系统。

争论

在这个领域里， 输入要传递给React Native的参数，例如，通过‑‑simulator旗： ‑‑simulator="iPhone 4s" 。

浏览器进行调试

在调试React Native应用程序时，PhpStorm依赖于Chrome运行时，该运行时由React Native本身使用。您也可以将DevTools与PhpStorm一起使用。当您启动调试会话时，PhpStorm将启动一个新的Chrome实例并附加到该实例。如果您不想看到任何新的Chrome窗口，请使用Chrome无头模式 。

请注意，在PhpStorm中停止内置调试器后，Chrome Headless将继续运行。

捆绑器主机

在此字段中，指定捆绑程序运行的主机，默认值为localhost 。

如果使用的是Expo，则可能需要更改默认的捆绑程序主机，以确保与调试器的成功连接。此连接可能会失败，因为默认情况下，PhpStorm使用本地主机启动调试，而Expo希望使用127.0.0.1或外部IP地址，具体取决于在Metro捆绑器配置的“连接”字段中选择的内容。有关详细信息，请参见调试使用Expo的React Native应用程序 。

捆绑器端口

在此字段中，指定运行捆绑程序的端口。

  • PhpStorm自动选择8081 ，这是React Native捆绑软件的默认设置。从React Native官方网站上了解更多信息。

  • 如果您的应用程序使用Expo，请将端口更改为19001 。

节点解释器

在这个领域里， 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

React Native包

在此字段中，指定通往react-native-cli的路径。

工作目录

指定应用程序的工作目录 。 默认情况下，该字段显示项目根文件夹 。

环境变量

在此字段中，可以选择键入环境变量以执行react-native run-android或者react-native run-ios命令。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

允许并行运行

如果选中此复选框，则可以并行启动运行配置多次，而不必重新运行它。每个运行程序都将在“ 运行工具窗口”或“ 调试工具窗口 ”的自己的选项卡中启动 。

禁用后，每次启动运行/调试配置的新实例时，PhpStorm都会检查同一运行/调试配置的其他实例是否存在，并显示确认对话框。如果在确认对话框中单击“ 确定” ，则将停止运行器的第一个实例，而将其替换。当启动两个相同类型的运行/调试配置实例时，这会占用过多的CPU和内存资源。

工具列

运行/调试配置的树形视图具有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在PhpStorm中定义，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 启动Web浏览器 ：选择此选项以启动浏览器。在打开的对话框中，选择浏览器的类型并提供起始URL。另外，指定是否要使用JavaScript调试器启动浏览器。

  • 运行文件监视程序 ：选择此选项可使PhpStorm应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 运行Phing目标 ：添加此任务以在运行或调试之前执行指定的Phing目标。要指定Phing目标，请单击“ 浏览”。 然后在打开的对话框中选择所需的目标。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

Alt+Delete

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
