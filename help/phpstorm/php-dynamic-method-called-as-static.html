代码检查：动态方法称为静态

报告对动态类方法的静态调用。

对于具有魔术方法的类__callStatic ，可以设置单独的检查严重性和突出显示级别。

在PHP 5.6和更高版本中，不建议动态调用静态方法并生成一个E_DEPRECATED警告。从PHP 7.0开始，静态调用非静态方法会导致$this在方法内部未定义。有关详细信息，请参见OOP基础（php.net） 。

在下面的示例中，方法foo()称为静态，而实际上称为动态。

类A {公共函数foo（）{}} $ a = new A（）; $ a :: foo（）;
