使用WP-CLI命令行工具

要在命令行模式下运行WordPress，您将需要一组命令行工具，您可以通过使用Composer依赖管理器安装wp-cli / wp-cli软件包或下载wp-cli.phar存档来获取这些工具。下载的命令行工具必须按照PHP命令行工具中所述在PhpStorm中注册。

有关在命令行模式下运行该工具的信息，请参见PHP命令行工具 。

在你开始前

  • 确保按照配置本地PHP解释器和配置远程PHP解释器中的说明在PHP页面的 PhpStorm中配置了PHP解释 器 。

  • 确保已安装并启用了PHP和命令行工具支持插件。

    这些插件默认情况下处于激活状态。如果禁用了插件 ，请按照“ 管理插件”中所述在“ 插件”页面上启用它们。

  • 如配置远程PHP解释器中所述，将PHP解释器配置为使用WordPress 。请注意， WordPress需要PHP 5.3或更高版本。

使用Composer依赖关系管理器安装WP-CLI软件包

在你开始前， 确保Composer已安装在计算机上并按照Composer Dependency Manager中所述在当前项目中初始化。

点击创建 。create-project Composer命令将与所选软件包一起调用。结果，将创建Composer项目，其配置和结构取决于所选的软件包。之后，打开创建的PhpStorm项目。

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择wp-cli / wp-cli软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。

    在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

使用WP-CLI作为PhpStorm命令行工具

在开始之前，请从WordPress CLI下载wp-cli.phar 。

将WP-CLI配置为命令行工具

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到工具|命令行工具支持 。

  2. 请点击在工具栏上。

  3. 在“ 命令行工具”对话框中，从列表中选择WP-CLI ，然后指定其可见性级别（ Project或Global ）。

  4. 当您单击确定时 ，将打开WP-CLI对话框。

    选择运行WP-CLI的方式 ：

    • 通过PHAR安装： 选择此选项可通过PHP脚本启动WordPress ，或让PhpStorm在wp-cli.phar存档中检测并启动启动器。

      从“ PHP解释器”列表中选择已配置的本地PHP解释器之一 。有关详细信息，请参见配置本地PHP解释器 。

      在phar的路径字段中，指定wp-cli.phar存档的位置。 手动输入路径或单击然后在打开的对话框中选择所需的位置。

    • 可执行文件（通过Composer安装，依此类推）： 选择此选项可通过可执行文件启动WordPress ，当您使用软件包管理工具（例如Composer）安装WordPress时，该文件将可用。

      在wp.bat的路径字段中， 指定wp.bat或wp可执行文件的位置。如果您使用的是Composer，则默认位置为\ vendor \ wp \ cli \ bin \ wp或\ vendor \ wp \ cli \ bin \ wp.bat 。 手动输入路径或单击然后在打开的对话框中选择所需的位置。

  5. 单击“ 确定”以应用更改并返回“ 命令行工具支持”页面。（可选）单击编辑工具属性，或自定义命令集。有关详细信息，请参见自定义工具 。

  • 在主菜单中，选择“ 工具” |“工具”。运行命令或按Ctrl两次。

    在打开的“运行任何内容”窗口中，在 格式。

    命令执行结果显示在“ 运行工具”窗口中 。

终止命令

  • 请点击在运行工具窗口工具栏上。

调试WP-CLI命令

Wordpress命令在扩展的控制器类中定义WP_CLI_Command ，以及任意函数。要调试命令，至关重要的是调试命令本身，而不是控制器类或定义它的文件。否则，将跳过Wordpress引导过程，并且执行将失败。

  1. 在控制器类或与所选命令对应的文件中，在要设置断点的代码行上单击编辑器装订线。

  2. 创建一个运行/调试配置 ，该配置将使用所选命令运行wp-cli工具。在主菜单中，选择运行|编辑配置 ，然后单击然后从列表中选择PHP脚本 。

  3. 在“ 运行/调试配置：PHP脚本”对话框中，提供运行/调试配置参数。

    • 在“ 文件”字段中，提供wp-cli可执行文件的路径。

    • 在“ 参数”字段中，键入实际的命令及其参数，例如my-command print_year 。

  4. 在PhpStorm工具栏上，选择创建的运行/调试配置，然后单击 。命令执行将在指定的断点处停止。
