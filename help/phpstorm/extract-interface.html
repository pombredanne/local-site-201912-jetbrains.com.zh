提取界面

假设您有一个带有几种用于检索和存储数据的方法的类。类似的功能也很有可能会在其他类中使用。在此类的基础上创建一个接口非常有帮助。

提取接口

  1. 在项目视图或结构视图中选择一个类，或在编辑器中选择其名称。

  2. 在主菜单上或从所选内容的上下文菜单中，选择“ 重构” |“选择” 。提取物|接口 。出现“ 提取接口”对话框。

  3. 在“ 接口名称”字段中， 指定要创建的接口的名称。默认情况下，PhpStorm建议从中提取接口的当前类的名称。请更新名称，否则PhpStorm将报告错误，并且重构将无法开始。

  4. 指定是否要PhpStorm立即应用提取的接口：

    • 要使PhpStorm创建接口并尝试用PhpDoc注释和方法参数中对接口的引用替换对源类的引用，请选择“ 在可能的情况下用接口替换类引用”复选框。建议的更改是强制性显示在“ 查找”工具窗口的“ 重构预览”窗格中的。请注意，如果实例引用接口中未定义的方法或字段，则不建议替换该实例。

    • 要使PhpStorm只是在一个单独的文件中创建一个接口，并将源类标记为新接口的实现，请清除“ 如果可能，用接口替换类引用”复选框。

  5. 指定新接口所属的名称空间。默认情况下，处于选中状态的源类的名称空间。

  6. 在“ 目标目标目录”列表中，指定该文件夹用于存储为新界面创建的专用文件。默认情况下，该字段为只读，并显示与根据PSR0标准接口将属于的名称空间相对应的文件夹。

    要将具有接口的类存储在另一个文件夹中，请单击F2并指定您选择的文件夹。

  7. 在“ 成员形成界面”区域中，指定要包含在界面中的方法。 下表显示了当前类中检测到的所有方法。要将方法移至“ 将成员移至类”中指定的接口，请选中该方法旁边的复选框。

  8. 在PhpDoc区域中， 指定您希望如何处理PhpDoc注释。可用的选项有：

    • 照原样：选择此选项可将文档保留在源类中的位置。

    • 复制：选择此选项可将内联文档复制到提取的界面，而不将其从其当前位置删除。

    • 移动：选择此选项可将内联文档移动到提取的接口，并从其在源类中的当前位置删除。

  9. 准备好后，单击“ 重构” 。如果PhpStorm在“ 查找”工具窗口中显示“ 重构预览 ”，请查看建议的更改。要提取接口并应用建议的更改，请单击“ 执行重构” 。
