模式

创建模式

  1. 右键单击数据源，然后导航到“ 新建|模式 。

  2. 在名称字段中，指定架构的名称。

  3. 点击执行 。

选择默认架构

您可以使用工具栏右上角的列表来选择默认架构或数据库。选择默认架构时，可以在语句中省略该架构或数据库的名称。

点击列出并选择所需的架构。

显示和隐藏模式

您可以选择要在数据库树视图中显示的架构。

  • 在“ 数据库工具”窗口（“ 视图” |“工具窗口” |“数据库” ）中，右键单击数据源，然后导航至“ 数据库工具” |“数据库”。管理显示的架构 。选择或清除要启用或禁用的架构的复选框。按Enter 。

  • 单击数据源名称附近的N of N链接。在架构选择窗口中，选中或清除要启用或禁用的架构的复选框。按Enter 。

  • 要隐藏所有模式，请导航至“ 数据库工具” |“数据库”。隐藏架构 。

比较两个模式

  1. 选择两个架构。

  2. 右键单击所选内容，然后导航到“ 比较” 。

或者，要比较两个模式，请按Ctrl+D 。

设置PostgreSQL和Redshift的模式搜索路径

的search_path PostgreSQL中的环境变量指定了搜索模式的顺序。例如，您设置search_path至z,a,public ，PostgreSQL将在z模式。如果在z模式，PostgreSQL在a模式。

在PostgreSQL和Amazon Redshift中，除非您指定其他搜索路径，否则将使用默认搜索路径（在数据库中设置的路径）。

  1. 点击列表，导航到数据库模式列表（使用箭头图标或按向右箭头键）。

  2. 选择要添加到搜索路径的架构。

    要形成搜索路径，可以使用以下操作：

    • 按Space在搜索路径中添加突出显示的架构，并从搜索路径中删除架构。

    • 按Alt+Up和Alt+Down重新排序搜索路径中的架构。

  3. 要应用更改，请按OK 。

保存IDE重新启动之间的搜索路径

  1. 在数据库工具窗口（ 视图|工具窗口|数据库 ）中，右键单击PostgreSQL或Amazon Redshift数据源，然后选择属性Shift+Enter 。

  2. 单击选项选项卡。

  3. 从“ 切换模式”列表中，选择“ 自动” 。

  4. 单击确定 。

强制刷新架构信息

“ 强制刷新”操作从缓存中清除数据源信息，然后从头开始再次加载它。

在“ 数据库工具”窗口（“ 视图” |“工具窗口” |“数据库” ）中，右键单击数据源，然后选择“ 数据库工具” |“数据库”。强制刷新 。

系统目录中的预先自检对象

自省是一种检查数据源的方法。执行内省时，将检查数据源中的结构信息以检测表，列，函数和其他具有其属性的元素。

系统目录是关系数据库管理系统（DBMS）存储有关表和列，内置函数以及其他架构对象的信息的地方。这些对象用于编码帮助（例如，代码完成）。

系统架构带有闪电图标在模式选择对话框中。如果未选择这些架构，则PhpStorm不会自检它们，也不会在“ 数据库”工具窗口中显示它们。尽管有关模式对象的信息用于编码帮助。这是可能的，因为PhpStorm使用的是关于以前被内省的架构对象的内部数据（预内省的数据）。

不同DBMS中的系统目录示例：

  • PostgreSQL：pg_catalog，information_schema

  • Microsoft SQL Server：INFORMATION_SCHEMA

  • Oracle：SYS，SYSTEM

  • MySQL：information_schema

  • IBM Db2：SYSCAT，SYSFUN，SYSIBM，SYSIBMADM，SYSPROC，SYSPUBLIC，SYSSTAT，SYSTOOLS

内省系统目录中的数据源

默认情况下，PhpStorm使用预先自检的对象作为系统目录。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，右键单击数据源，然后选择“ 属性”。 Ctrl+Alt+S 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 选项”选项卡。

  3. 清除“对非自检的系统目录使用预自检对象”复选框。

  4. 在方案选择窗口中，选择要自省的系统目录。

将预先检查的数据用于所选系统目录

即使禁用数据源对此类对象的使用，仍可以将其用于系统目录。要使用此功能，请清除数据源设置中“ 选项”选项卡上的“对未自检的系统目录使用预自检对象”复选框。

  1. 打开方案选择窗口，清除系统目录的复选框。

  2. 单击系统目录条目。

  3. 单击闪电图标 ，它位于窗口的右上角。
