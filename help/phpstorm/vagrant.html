Ctrl+Alt+S

在此页面上，启用PhpStorm中的Vagrant支持，指定VagrantFile的位置，并处理要在创建虚拟盒（实例）中使用的Vagrant基本盒列表。

在开始使用Vagrant之前，请确保：

  1. Vagrant已下载并安装。

项目

描述

无业游民的可执行文件

指定可执行文件的标准地址：对于Windows是vagrant.bat ，对于Unix和macOS是vagrant 。手动输入路径，或单击浏览按钮，然后在“ 选择无业游民的可执行文件”对话框中找到所需的文件。

实例文件夹

在此处指定任务的目录的标准路径。 vagrant init已执行，并且Vagrantfile初始化并存储。

Vagrantfile是一个配置文件，用于定义所需的实例 （ 虚拟机 ）。该文件包含虚拟IP地址，端口映射和要分配的内存大小。该文件可以指定要共享的文件夹以及应该安装的第三方软件。根据Vagrantfile，您的实例 （ 虚拟机 ）已配置，根据相关的Vagrant基本框进行了配置，并已部署在您的计算机上。Vagrantfile是通过以下方式创建的vagrant init命令。

通过创建实例 （ 虚拟机 ）的调用时。 vagrant up命令或通过工具|无业游民在菜单选项中，PhpStorm在“ 实例文件夹”字段中指定的目录中查找Vagrantfile 。有关更多信息，请参见http://docs.vagrantup.com/v2/vagrantfile/ 。

您可以在任何目录中创建Vagrantfile并将其指定为实例文件夹 。如果该字段为空，则PhpStorm会将项目根目录视为实例文件夹，并在其中查找Vagrantfile 。

提供者

使用此字段可以指定要使用的提供程序 vagrant up命令。如果将此字段留为空白，则使用默认提供程序。

环境变量

单击省略号按钮或按Shift+Enter指定用于配置提供程序行为的shell变量。

框和插件选项卡

盒子

此列表显示了PhpStorm 中可用的预定义的Vagrant基本框 。

每个项目都显示一个Vagrant基本框 ，Vagrant在该框上配置和启动其实例 （ 虚拟机 ）。该列表的条目与命令的输出相对应vagrant box list 。
Alt+Insert

单击此按钮以下载新的基本框。该命令对应于vagrant box add 。默认情况下，PhpStorm建议使用lucid32框的URL

Alt+Delete

单击此按钮删除选定的Vagrant基本框 。该框和嵌套文件实际上已从磁盘上删除。该命令对应于vagrant box remove

外挂程式

使用此表查看和更改可用插件的列表。

Alt+Insert

单击此按钮以安装新的Vagrant插件。

Alt+Delete

单击此按钮删除选定的插件。

单击此按钮以更新所选插件。

使用此按钮可以将许可证附加到所选插件。
