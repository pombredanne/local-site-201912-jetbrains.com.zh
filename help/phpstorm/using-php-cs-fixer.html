PHP CS修复程序

除了内置的编码帮助外，PhpStorm还通过与PHP CS Fixer工具集成来检查源代码，该工具可检测代码中的编码标准问题。

要从PhpStorm而不是命令行使用PHP CS Fixer，您需要在PhpStorm中进行注册并将其配置为PhpStorm 代码检查 。在PhpStorm中安装并启用该工具后，该工具将在任何打开的PHP文件中可用，并且不需要其他步骤即可启动它。文件中的每次更新都会激活即时代码检查，因此很容易消除发现的问题。

PHP CS Fixer即时报告的错误和警告将显示为弹出消息。当该工具以批处理模式运行时，错误和警告将显示在“ 检查结果”工具窗口中。每个消息都有PHP CS Fixer前缀以将其与PhpStorm内部检查区分开。

您可以应用预定义的规则，也可以定义自己的自定义规则集 。

在你开始前

确保已启用PHP插件。 的默认情况下，插件已激活。如果禁用了插件 ，请按照“ 管理插件”中所述在“ 插件”页面上启用它。

安装和配置PHP CS Fixer

PHP CS Fixer脚本可用作本地脚本，与PHP解释器关联的脚本或声明为项目依赖项并通过Composer安装的脚本，这是推荐的推荐方法。

PhpStorm支持PHP CS Fixer 2.8.0及更高版本。

代码质量工具脚本包含对系统范围的PHP引擎的调用。为了能够使用它们，请确保将包含PHP引擎可执行文件的目录添加到系统中path 。

使用Composer安装PHP CS Fixer

在你开始前， 确保Composer已安装在计算机上并按照Composer Dependency Manager中所述在当前项目中初始化。

在通过Composer安装PHP CS Fixer时，PhpStorm会自动下载必要的脚本，在IDE中注册它们，并有选择地启用和配置相应的代码检查 。

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择friendsofphp / php-cs-fixer 软件包 ，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

手动配置PHP CS Fixer

您可以使用手动下载的本地PHP代码质量工具脚本或与PHP解释程序关联的脚本。可以有许多本地和远程PHP解释器，在Settings / Preferences对话框的PHP页面上指定的一个解释器被认为是Project Default 。在配置远程PHP解释器或配置本地PHP解释器中了解有关配置PHP解释器的更多信息。

选择要使用的PHP CS Fixer脚本

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ PHP CS Fixer”区域。在配置列表中， 选择PHP CS Fixer脚本：

    • 要使用与特定的远程PHP解释器关联的脚本，请选择该解释器的名称。

    • 要使用本地脚本，请选择Local 。在这种情况下，无论项目中使用哪种PHP解释器（本地或远程），都将执行本地PHP CS Fixer。请注意，只能有一个PHP CS定影液本地配置，因为PhpStorm运行的脚本（PHP-CS-fixer.bat用于Windows或PHP-CS-定影液用于Linux和MacOS），其中包含的PHP引擎的路径。

    • 要使用与默认项目解释器关联的脚本，即在“设置/首选项”对话框的PHP页面上选择的脚本 ，请选择“默认项目解释器” 。

配置本地PHP CS Fixer脚本

  1. 下载并安装PHP CS Fixer脚本。

    要检查PHP CS Fixer的安装，请切换到安装目录并运行以下命令：

    php-cs-fixer --version

    如果该工具可用，您将收到以下格式的消息：

    Fabien Potencier和Dariusz Ruminski撰写的PHP CS Fixer 2.13.0 Yogi's BBQ

    要根据您自己的自定义编码标准检查代码，请创建它 。将规则以及指向它们的.php_cs或php_cs.dist文件存储在规则集的根目录中。

  2. 在PhpStorm中注册本地PHP CS Fixer脚本：

    • 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

    • 在打开的“ 质量工具”页面上，展开“ PHP CS Fixer”区域，然后单击“确定”。 配置列表旁边。

    • 在打开的“ PHP CS Fixer”对话框中，在“ PHP CS Fixer路径”字段中指定php-cs-fixer.bat或php-cs- fixer PHP CS Fixer可执行文件的位置。手动输入路径或单击然后在打开的对话框中选择相关文件夹。

      要检查到PHP-CS-fixer.bat或php-CS-定影液指定的路径确保PhpStorm和PHP CS定影液，也就是说，该工具可以从PhpStorm和PhpStorm推出将它接收问题报告之间的互动，点击验证按钮。此验证等同于运行php-cs-fixer --version命令。如果验证成功通过，则PhpStorm将显示有关检测到的PHP CS Fixer版本的信息。

配置与PHP解释器关联的PHP CS Fixer脚本

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ PHP CS Fixer”区域，然后单击“确定”。 配置列表旁边。将打开“ PHP CS Fixer”对话框，在左侧窗格中显示所有已配置的PHP CS Fixer脚本的列表，其中一个的类型为“ 本地” ，其他的则以与脚本关联的PHP解释器命名。请点击在工具栏上。

    请注意，当前PhpStorm仅支持PHP CS Fixer 2.8.0及更高版本。

  3. 在打开的“ 通过远程解释器进行的PHP CS Fixer”对话框中，选择要使用关联脚本的远程PHP解释器。如果列表不包含相关的解释器，请单击并在CLI解释器对话框中配置远程解释器 ，如配置远程PHP解释器中所述 。

    当您单击OK时 ，PhpStorm将带您回到PHP CS Fixer对话框，在该对话框中将新的PHP CS Fixer配置添加到列表中，并且右侧窗格显示了所选的远程PHP解释器，以及与之关联的PHP CS Fixer的路径。 ，以及高级PHP CS Fixer选项。

配置高级PHP CS Fixer选项

PhpStorm允许您指定高级PHP CS Fixer选项，从而根据计算机的配置和所使用的规则集来微调PHP CS Fixer进程的行为。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ PHP CS Fixer”区域。在“ 每个文件的最大邮件数”字段中，设置文件要报告的邮件总数的上限。执行检查后，所有超过此限制的消息将被拒绝，PhpStorm将在编辑器选项卡顶部或“ 检查结果工具”窗口中显示警告消息。

  3. 在“ 工具进程超时”字段中，指定您希望PhpStorm等待PHP CS Fixer结果的时间，然后终止进程以防止CPU和内存使用过多。

将PHP CS Fixer配置为PhpStorm检查

手动配置PHP CS Fixer检查

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在编辑器下单击检查 。

  2. 在打开的“ 检查”页面上，展开PHP |。质量工具节点，然后选中PHP CS Fixer验证旁边的复选框。

  3. 在页面的右侧窗格上，使用“ 选项”区域中的控件配置PHP CS Fixer工具：

    1. 从“ 严重性”列表中，选择“ PHP CS Fixer检查”的严重性等级 。选择的值确定PhpStorm对检测到的差异的严重程度并将其显示在检查结果中 。

    2. 从“ 范围”列表中，选择将检查应用程序限制到的范围 。

    3. 要允许危险规则 （即可以更改代码行为的规则），请选中“ 允许危险规则”复选框。默认情况下，不允许使用冒险规则。

    4. 指定要应用的编码标准。

      • 要使用预定义的编码标准之一 ，请选择“ 编码标准”列表，并指定编码样式以对照您的代码。该列表包含在php-cs-fixer主目录结构中声明的所有编码标准。

      • 要根据您自己先前定义的编码标准检查代码，请选择“ 自定义” 。点击浏览 在打开的“ 自定义编码标准”对话框中，在“ 规则集的路径”字段中为您自己的编码标准指定.php_cs或.php_cs.dist文件的路径 。手动输入路径或单击浏览 然后在打开的对话框中选择相关文件夹。

使用Composer配置PHP CS Fixer检查

您可以在composer.json的脚本部分内包含有关默认和自定义PHP CS Fixer规则集的信息。当您安装或更新项目依赖项时，将检测到指定的规则集，并自动启用PHP CS Fixer验证检查。

如果未在其中指定规则集scripts在composer.json部分，PhpStorm将另外检查项目根目录，以使用默认名称.php_cs或.php_cs.dist定位规则集。如果文件存在，它将被自动选择为检查的“ 自定义”规则集。

使用Composer配置PHP CS Fixer检查

  • 在里面scripts在composer.json部分，添加php-cs-fixer将PHP CS Fixer启动命令放入叶子元素之一。

    提供--rules参数表示使用的编码标准。例如，添加以下记录会将编码标准设置为PSR1 ：

    “脚本”：{“ php-cs-fixer”：“ php-cs-fixer --rules = @ PSR1”}

    或者，提供--config参数和规则集文件的路径，这会将编码标准设置为Custom ：

    “脚本”：{“ php-cs-fixer”：“ php-cs-fixer --config =。/.php_cs“}

与团队共享自定义编码风格

  1. 将您的编码标准的根目录放在项目root下 。

  2. 将PHP CS Fixer配置为PhpStorm检查 。

  3. 指定您的编码标准 。

  4. 确保在“ 检查”页面顶部选择了项目配置文件 。此类配置文件保存在特定项目的.idea目录中（例如， $ PROJECT_DIR / .idea / inspectionProfiles ）。有关详细信息，请参见配置配置文件 。

  5. 在“ 设置/首选项”对话框的“ 版本控制”页面上，确保将.idea目录置于版本控制下 。

以批处理模式运行PHP CS Fixer

  1. 通过选择Code | 运行检查 。从主菜单检查代码 。指定检查范围和配置文件。

  2. 在“ 检查结果工具”窗口中查看检查结果。PHP CS Fixer报告的错误和警告以开头PHP CS Fixer区别于PhpStorm内部检查。

从PHP CS Fixer检查中排除文件

当等待PHP CS Fixer响应超过在PHP CS Fixer对话框的“ 工具进程超时”字段中指定的限制时，PhpStorm建议将文件添加到忽略列表 。

管理被忽略的文件

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ PHP CS Fixer”区域，然后单击“ 显示忽略的文件”链接。

    • 要添加文件，请单击并在打开的对话框中找到所需的文件。

    • 要从列表中删除文件并让PHP CS Fixer再次对其进行处理，请选择该文件并单击 。

    • 要从列表中删除所有文件，请单击“ 清理列表”按钮 。
