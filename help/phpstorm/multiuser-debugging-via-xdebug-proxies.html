通过Xdebug代理进行多用户调试

当您启动调试会话时，Xdebug扩展将连接到运行PhpStorm的IP地址，指定为xdebug.remote_host值。PhpStorm接受此连接，并可以通过它与Xdebug进行通信。Xdebug仅支持连接到单个IP地址，由于安全原因，它不自动连接回运行浏览器的IP地址。为了在多用户环境中调试PHP应用程序，Xdebug提供了一个所谓的DBGp代理 。

使用代理时，PHP Xdebug扩展不再直接连接到PhpStorm，而是连接到DBGp代理服务器。然后，团队中的所有开发人员都将连接到该代理。每个开发人员都有一个运行在此代理上的单独调试会话，这使得可以对同一服务器上的相同代码进行多用户调试。

通过Xdebug代理（DBGp）服务器配置多用户调试

要通过Xdebug代理（DBGp）服务器配置多用户调试，请执行以下常规步骤：

  • 下载并安装DBGp代理
  • 确保Xdebug已在Web服务器上安装和配置
  • 配置从PhpStorm对DBGp代理服务器的访问
  • 在浏览器中启动调试会话

下载并安装DBGp代理

为了能够启动调试会话，您需要在服务器上运行DBGp代理，Web服务器本身以及所有开发人员机器都可以访问该代理。我们可以将代理安装在Web服务器上，同一网络中的计算机上，或通过SSH隧道访问的计算机上。

  1. 在Komodo下载页面上 ，您可以找到特定于DBGp代理平台的Python二进制文件。下载所需的程序包并运行pydbgpproxy或pydbgpproxy.exe文件。

  2. 在Web服务器或可以与Web服务器和所有开发人员计算机通信的计算机上启动DBGp代理。DBGp代理可执行文件接受两个参数： -d和-i 。

    这些参数定义侦听来自Web服务器的调试器连接的IP地址和端口，以及侦听开发人员的IP地址和端口。

    例如，侦听回送地址（ 127.0.0.1 ）和端口9001上的调试器连接，并侦听计算机IP地址和端口9000上的开发人员。

    pydbgpproxy -d 127.0.0.1:9001 -i 192.168.99.1:9000

确保Xdebug已在Web服务器上安装和配置

  1. 在编辑器中打开活动的php.ini文件：

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在“ 语言和框架”下单击“ PHP ” 。

    2. 在打开的PHP页面上，单击 CLI解释器字段旁边。

    3. 在打开的“ CLI解释器”对话框中，“ 配置文件”只读字段显示活动php.ini文件的路径。单击“ 在编辑器中打开” 。

  2. 确保至少指定以下设置：

    [xdebug] zend_extension = xdebug_module_goes_here xdebug.remote_enable = 1 xdebug.remote_host = dbgp_proxy_hostname_or_ip xdebug.remote_port = 9001

有关配置Xdebug的详细信息，请参阅配置Xdebug。

配置从PhpStorm对DBGp代理服务器的访问

  1. 在主菜单中，选择“ 工具” |“工具”。 Xdebug代理|注册IDE 。

  2. 在打开的“ Xdebug代理”对话框中，指定IDE密钥值（对于每个开发人员，它应该是唯一的值，因为它标识了调试会话），Xdebug代理服务器的主机和端口。

  3. 单击确定以连接到指定的代理服务器。服务器注册凭据，并确认连接。

    PhpStorm还将通知连接成功。

  4. 要更新现有凭证，请选择“ 工具” |“工具”。 Xdebug代理|在主菜单中进行配置 。在打开的“ Xdebug代理”对话框中，编辑IDE密钥，主机和端口设置。

    要舍弃当前凭证，请选择“ 工具” |“工具”。 Xdebug代理|在主菜单中取消IDE注册 。

在浏览器中启动调试会话

  1. 确保将您正在使用的浏览器调试扩展设置为已注册的IDE密钥：

  2. 在PhpStorm中，通过单击以下任一方法来侦听传入的调试连接在工具栏上，或选择运行|在主菜单中开始侦听PHP调试连接 。这将确保PhpStorm在启动调试会话时做出反应，并自动打开“ 调试工具”窗口 。启动脚本之前，请确保在“设置/首选项”对话框的“ 调试”页面上设置了断点或启用了“ PHP脚本中的第一行中断 ”选项。 Ctrl+Alt+S 。

    通过刷新浏览器中的页面来启动调试会话。

故障排除

远程文件路径未映射到项目中的任何文件路径

在某些情况下，调试器可以连接，但是我们收到错误消息，指示未定义远程文件与项目文件之间的映射。这意味着PhpStorm无法确定哪个本地文件对应于正在调试的文件。

我们可以通过单击Click来设置路径映射并提供必要的路径映射来解决此问题。

另外，我们可以使用配置与Web服务器的同步中概述的技术来配置这些映射。

由于目标机器主动拒绝连接，因此无法建立连接

  • 确保PhpStorm已向DBGp代理注册 ，并且正在侦听传入的调试器连接 。

  • 确保在PhpStorm以及调试器书签或浏览器扩展中 设置了正确的IDE密钥 。

  • 是否有防火墙阻止与DBGp代理的连接？如果是这样，请确保PHP Xdebug扩展以及开发人员机器都可以连接到它。您可以使用http://canyouseeme.org或类似的服务来检查打开的入站端口。

  • 是否有防火墙阻止与IDE的连接？确保可以从DBGp代理计算机连接到PhpStorm。您可以使用http://canyouseeme.org或类似的服务来检查打开的入站端口。

  • DBGp代理应用程序绑定到错误的IP地址。使用0.0.0.0作为地址运行它，以便它绑定到DBGp代理服务器上的任何IP地址。例如，运行pydbgpproxy -i 0.0.0.0:9001 -d 0.0.0.0:9000 。

使用同一台计算机上安装的Xdebug和DBGp代理在PhpStorm中测试多用户调试时，还有一些其他事项需要验证：

  • PhpStorm绑定到错误的IP地址。当DBGp代理在还运行PhpStorm的本地计算机上运行时，IDE可能使用了错误的网络子网。要解决此问题，请选择“ 工具” |“工具”。 DBGp代理|在主菜单中配置... ，并将DBGp代理服务器IP地址设置为0.0.0.0 。然后，尝试使用DBGp代理重新注册PhpStorm。

  • 端口冲突存在。在本地计算机上测试DBGp时，PhpStorm和DBGp都可能希望绑定到同一端口。通过将DBGp配置为使用其他端口或更改“ 语言和框架”上的PhpStorm的Xdebug端口，确保不是这种情况。 PHP | 设置/首选项对话框的调试页面Ctrl+Alt+S 。
