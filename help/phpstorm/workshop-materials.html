PhpStorm研讨会资料

PhpStorm研讨会资料将指导您完成许多PhpStorm功能和生产力工具。它们并未涵盖PhpStorm中的所有可能选项和功能。相反，他们提供了许多实践练习，以增强您作为PHP开发人员的日常工作流程。

这是一个5分钟的截屏视频，概述了研讨会资料：

先决条件

要完成研讨会资料，您需要在系统上安装PhpStorm 2016.1或更高版本（完整版或免费试用版）。许多事情也可以与早期版本一起使用，但是建议在这些材料版本（2016.1或更高版本）中执行所有操作。

git命令行也将非常有用。

练习题

PhpStorm Workshop练习是一个PhpStorm项目，其中每个文件都是一个新练习，其中可能包含代码和完成操作的提示。这里有关于导航，编辑，检查，实时模板，重构，诸如Composer和HTTP客户端之类的工具以及其他领域的练习。

讲习班是自定进度的，这意味着您可以随时随地自行练习。

键盘快捷键

我们将介绍PhpStorm中提供的多个键盘快捷键。其他基于IntelliJ IDEA的IDE使用相同的键盘快捷键，因此，如果您知道如何使用PhpStorm，将知道如何使用WebStorm ， RubyMine ， PyCharm和IntelliJ IDEA ，反之亦然。在线备有备忘单，该备忘单也包含在研讨会下载中。

获取研讨会资料

  1. 在“ PhpStorm欢迎”屏幕上，单击“ 创建新项目” 。

  2. 从左侧列表中选择PhpStorm Workshop Project ，在Location字段中提供项目位置，然后点击Create 。

另外，您可以从GitHub手动克隆PhpStorm Workshop存储库 。

探索项目

该项目包含文件夹和文件，其中包括一个或多个练习。大多数练习都是独立的，但是有些练习是在以前的练习的基础上进行的。

项目根目录还包括其他几个文件：

  • Readme.md包含有关PhpStorm Workshop的其他信息。

  • PhpStorm参考Card.pdf是PhpStorm键映射。最新版本始终可以在PhpStorm网站上找到。

运行Docker

Workshop项目包含一个预先配置的Docker环境。大多数与编辑器无关的练习都需要运行Docker容器。

  1. 为您的操作系统安装Docker 。

  2. 在PhpStorm中在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 构建，执行，部署” | Docker，然后选择如何连接到Docker守护程序。

    根据您的操作系统，执行以下操作：

    • 选择“ TCP套接字” 。

    • 将引擎API URL设置为tcp：// localhost：2375 。

      在Docker for Windows的常规部分的设置中， 在tcp：// localhost：2375上启用没有TLS的Expose守护程序 。

    • 将“ 证书文件夹”字段保留为空。

    如果使用的是Docker Toolbox，请改用以下配置选项：

    • 将引擎API URL设置为https://192.168.99.100:2376

    • 将Certificates文件夹设置为\ .docker \ machine \ machines \ default 。

    • 选择Docker for Mac 。

    • 选择Unix套接字 。

  3. 在编辑器中打开docker-compose.yml并更新XDEBUG_CONFIG变量，其值取决于您的操作系统。这对于Web调试是必需的。

    使用host.docker.internal值，它是指远程主机，即运行Docker的机器。它将自动解析为主机的内部地址，使您可以从容器连接到主机。

    docker-compose.yml文件的相应部分应如下所示：

    XDEBUG_CONFIG：remote_host = host.docker.internal

    使用host.docker.internal值，它是指远程主机，即运行Docker的机器。它将自动解析为主机的内部地址，使您可以从容器连接到主机。

  4. 在相同的docker-compose.yml文件中，取消注释sftp服务取决于您的操作系统。为使部署正常工作，这是必需的。

  5. 请点击在编辑器装订线中启动所有必需的Docker容器。

    或者，打开内置的PhpStorm终端（ Alt+F12 ）并执行docker-compose up命令。
