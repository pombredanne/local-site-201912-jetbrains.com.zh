PhpStorm 2019.3帮助

运行，调试和上载应用程序到PHP的Google App Engine

所描述的功能由Google App Engine支持的PHP插件处理。从2018.1。版本开始，该插件不再维护且与PhpStorm不兼容。

PhpStorm使您可以在将应用程序上载到Google PHP运行时环境之前，在PHP开发服务器上本地运行和调试您的应用程序。根据专用于PHP运行/调试配置的App Engine在本地启动应用程序 。

在这个部分：

使用app.yaml文件

在app.yaml文件中配置了Google App Engine PHP应用程序。如果生成项目存根（请参阅准备为PHP应用程序开发Google App ），则会自动创建app.yaml文件。如果您在现有项目中激活Google App Engine支持，则需要手动创建一个app.yaml文件。无论哪种情况，请确保runtime属性设置为php55 ，因为用于PHP SDK的Google App Engine包含内置的PHP 5.5解释器。

为PHP创建App Engine类型的运行配置

  1. 在主菜单上，选择“运行”。编辑配置 ，单击并从列表中选择App Engine for PHP 。将打开“ 运行/调试配置：适用于PHP的App Engine” 。

  2. 指定运行开发服务器和应用程序的主机（默认为localhost ）。

  3. 指定 PhpStorm将通过它与开发服务器进行通信的端口（默认端口为8080 ）。

  4. （可选）在“ 命令行”区域中， 指定用于在命令行模式下在PHP开发服务器上运行和调试应用程序的设置：

    • 在“ 解释器选项”字段中， 指定要传递给内置PHP解释器的PHP可执行文件的选项，有关详细信息，请参见命令行参数 。

    • 在“ 自定义工作目录”字段中， 指定与源一起位于文件夹外部且通过相对路径引用的文件的位置。手动输入路径或单击浏览 然后在打开的对话框中选择所需的文件夹。

    • 在Yaml文件字段中， 指定要使用的.yaml配置文件。该字段是可选的，当您的应用程序包含多个模块，因此使用多个.yaml配置文件时，请使用此字段。

    • 在环境变量字段中， 在此字段中，指定将环境变量传递到内置服务器。有关详细信息，请参见Apache中的环境变量 。

在PHP开发服务器上运行应用程序

  1. 如上所述， 为PHP运行配置创建一个App Engine 。

  2. 从列表中选择配置，然后单击在工具栏上。

  3. 在“ 运行”工具窗口的控制台中查看和分析应用程序的输出。

  4. 要查看应用程序执行结果，请在http：// localhost：8080打开浏览器。

在PHP开发服务器上调试应用程序

  1. 如上所述， 为PHP运行配置创建一个App Engine 。

  2. 在代码中设置断点，请参阅Breakpoints 。

  3. 从列表中选择配置，然后单击

  4. 在打开的“ 调试工具”窗口中，单步执行程序 ， 停止并继续执行程序 ， 在挂起时对其进行检查 ，等等。

将应用程序上传到Google PHP运行时环境

通过在开发服务器上本地运行和调试应用程序来测试您的应用程序之后，可以将其部署到Google PHP运行时环境中 。

  1. 在主菜单中，选择工具|适用于PHP的Google App Engine |上传App Engine PHP应用程序 。

  2. 访问http：// .appspot.com /上的应用程序。
