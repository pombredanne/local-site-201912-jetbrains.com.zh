变更签名

更改签名重构结合了可以应用于功能签名的几种不同的修改。您可以使用此重构来：

  • 更改函数名称和返回类型

  • 添加，删除和重新排序参数

  • 为参数分配默认值

  • 更改参数名称和类型

  • 通过函数调用层次结构传播新参数

更改功能签名时，PhpStorm会搜索该功能的所有用法，并更新所有可以安全修改以反映更改的调用，实现和替代功能。

仅PHP和JavaScript支持Change Signature重构（请参阅重构JavaScript ）。

您可以从UML类图访问此重构。

默认值和新参数的传播

对于添加到函数中的每个新参数，您可以在“ 默认”字段中指定默认值（或表达式）。

您还可以将已引入的参数传播到调用要更改其签名的函数的函数。重构结果取决于是否指定默认值以及是否使用传播。

传播 ：新参数可以传播到调用您要更改其签名的函数的任何函数。在这种情况下，通常，调用函数的签名会相应更改。但是，这些更改还取决于为新参数设置的默认值。

默认值 ：通常，这是要添加到函数调用中的值。如果新参数未传播到调用函数，则该函数内的调用也将使用此值。如果使用传播，则此值不会影响调用函数中的函数调用。

更改功能签名

  1. 将插入号放置在要重构的函数的名称上。

  2. 按Ctrl+F6 。或者，选择Refactor |从主菜单或上下文菜单中更改签名 。将打开“ 更改签名”对话框。

  3. 在“ 更改签名”对话框中，根据需要对功能签名进行必要的更改：

    • 更改功能名称。要更改名称，请在“ 名称”字段中编辑文本。

    • 通过编辑“ 返回类型”字段的内容来更改函数的返回类型 。

      只能在以下情况下设置函数返回类型 PHP语言版本7.1及更高版本。 您可以在PHP页面上指定PHP语言级别的 设置/首选项对话框Ctrl+Alt+S 。

    • 管理功能参数。要配置参数，请使用表格和“ 参数”区域中的按钮：

      • 要添加新参数，请单击并在相应的表格行中指定新参数的属性。按Ctrl+Space使用代码完成 。

        • 要提供将影响功能签名的参数默认值，请使用“ 参数”字段，如下所示：

          $ parameterName ='parameterValue';

        • 要提供参数的默认值，这将影响函数的调用，请使用Default字段，如下所示： argumentValue 。

        添加参数时，您可能希望将其传播到调用当前函数的函数。

        在PHP上下文中，当从类的构造函数调用Change签名重构时，可以将新参数初始化为类字段。为此，请使用“ 创建和初始化类属性”复选框：

        • 选中此复选框后，新添加的参数将初始化为字段。PhpStorm创建一个与此参数名称相同的字段，并添加具有以下分配的行：

          $ this-> parameterName = $ parameterName;

        • 清除复选框后，将添加参数而不进行初始化。

        新属性是使用默认可见性修饰符创建的，该修饰符在代码样式的“代码生成”选项卡上设置。PHP页面的 设置/首选项对话框Ctrl+Alt+S 。

      • 要删除参数，请选择任意行，然后单击 。

      • 要重新排列参数，请单击和 。例如，如果要在列表中首先放置某个参数，请单击该参数对应的行中的任何单元格，然后单击所需的次数。

      • 要更改参数的类型，名称或默认值，请在参数表中（在“ 类型”和“ 参数”字段中）进行必要的更新。

    • 沿调用当前函数的函数层次结构传播新的方法参数（如果有）。可能有一些函数调用要更改其签名的函数。这些功能又可以被其他功能调用，依此类推。您可以通过调用函数的层次结构传播对当前函数的参数所做的更改，并指定哪些调用函数应该受到影响，哪些不应该受到影响。

      传播新参数：

      1. 单击传播参数按钮 。

      2. 在“ 选择方法以传播新参数”对话框的左侧窗格中，展开必要的节点，然后选中要将新参数传播到的函数旁边的复选框。

        为了帮助您选择必要的功能，对话框右侧部分分别显示了调用函数和正在调用的函数的代码（分别在“ 调用方方法”和“被调用方 方法”窗格中）。

        当您在左侧窗格中的功能之间切换时，右侧窗格中的代码会相应更改。

      3. 单击确定 。

  4. 单击重构 。

  5. 要查看预期的更改并在重构之前进行调整，请单击预览 。
