移动和复制重构

通过移动重构，您可以在项目中移动类，类成员（方法，字段和常量），常规函数和常量，文件和目录。这样，PhpStorm会跟踪这些移动并自动更正源代码中对已移动符号的所有引用。

将文件或目录移动到另一个目录

  1. 在“ 项目”工具窗口中选择文件或目录。

  2. 请执行以下任一操作：

    • 从主菜单中，选择“ 重构|移动 。

    • 从选择的上下文菜单中，选择“ 重构” |“选择” 。移动 。

    • 按F6 。

    打开的对话框取决于所选符号的类型。

  3. 在“ 到目录”字段中，指定要将所选文件或文件夹移动到的文件夹。从列表中选择一个现有文件夹，或键入要创建的父文件夹的完整路径。

  4. 要根据重构结果更新对选定文件或文件夹的引用，请选中“ 搜索引用”复选框。

移动PHP名称空间

PhpStorm假定项目中的名称空间的排列符合PSR0标准，并在移动名称空间时强制您根据此标准保留文件夹结构和名称空间层次结构。

当您指定将名称空间移动到的名称空间时，PhpStorm会自动更新“ 目标目标目录”字段，该字段显示与该名称空间相对应的文件夹的路径。

将具有所有或某些相关类的名称空间移动到另一个名称空间

  1. 在编辑器中，将插入记号放置在要移动的名称空间处，然后选择“ 重构” |“重构”。从主菜单或者选择上下文菜单中移动 。请注意，如果您在“ 项目”工具窗口中选择了与所需名称空间相对应的文件夹，然后选择“ 重构” |“确定”。 Move ，将调用Move Directory重构。

  2. 在打开的“ 移动命名空间”对话框中，指定新的命名空间名称以及与之对应的文件夹。

    • 新命名空间名称 字段显示所选名称空间的标准名称。指定新的名称空间名称。 仅使用反斜杠 \作为名称空间分隔符。

    • 对话框打开时， 目标目标目录 字段显示与当前名称空间相对应的文件夹的路径。

      路径以以下格式显示：

      ...\
      当您指定新的名称空间名称时，路径会自动更新。 但是，如果要将一个名称空间移动到另一个父名称空间，除非您通过在目录页面上将相关文件夹标记为Sources来为名称空间结构指定根文件夹，否则PhpStorm将不会建议正确的文件夹。 的 设置/首选项对话框Ctrl+Alt+S 。有关详细信息，请参见目录 。

      请执行以下任一操作：

      • 接受显示在字段中的预选路径。

      • 从列表中选择另一个路径。它们都是从名称空间根目录或当前目录中评估的，因此可以安全地选择它们。

      • 请点击然后在打开的对话框中选择一个文件夹。

      • 按N/A并编辑预选路径。请记住，这将来可能会导致自动加载出现问题。

  3. 单击“ 重构”，然后在打开的“使用相关名称空间移动文件”对话框中，指定要移动到新名称空间和新文件夹的类和文件。

    对话框的上部窗格列出了与名称空间相关的类和文件的目标名称空间和文件夹。列表中的每个项目都对应一个类/文件。当您将光标移动到某个项目时，底部窗格显示与之相关的文件的内容。

    • 要将类和相应的文件移动到目标名称空间和目标文件夹，请选中名称空间/文件夹旁边的复选框。

    • 要将所有项目添加到列表中或将其从列表中删除，请分别单击“ 全 选”或“ 取消全选” 。

  4. 单击“ 确定”开始重构。

将静态PHP方法，变量或常量移动到另一个类

  1. 在编辑器中打开一个PHP类，将插入符号放置在静态方法，变量或常量上。

  2. 从主菜单或编辑器上下文菜单中，选择“ 重构” |“重构”。移动 。

  3. 在打开的“ 移动静态成员”对话框中，指定以下内容：

    1. 在要移动到另一个类的成员字段中，选中要移动到另一个类的方法，常量和变量旁边的复选框。该列表显示了当前类中检测到的所有静态方法，变量和常量。

    2. 在将成员移到班级字段中， 键入要在其中将在“成员”中选择的成员移动到另一个班级列表的班级的全限定名。PhpStorm为类和名称空间引用提供完成：在键入类或名称空间名称时，将显示带有相关项目的建议列表。您还可以通过按以下按钮获取可用课程的完整列表Ctrl+Space 。

将函数或常量从PHP文件移动到类

  1. 请执行以下任一操作：

    • 要移动单个常量或函数，请在编辑器中打开一个PHP文件，然后将插入号放置在常量或函数上。

    • 要一次移动多个常量或函数，请在编辑器中打开一个PHP文件，然后在“ 结构”工具窗口中选择要移动的常量或函数（ Alt+7 ）或结构视图弹出窗口（ Ctrl+F12 ）。

  2. 从主菜单或编辑器上下文菜单中，选择“ 重构” |“重构”。移动或按F6 。

  3. 根据您的选择， 将打开“ 移动常数”或“ 移动功能”对话框。

    在对话框中，键入要在其中移动所选常量或函数的类的完全限定名称。PhpStorm为类和名称空间引用提供完成：在键入类或名称空间名称时，将显示带有相关项目的建议列表。您还可以通过按以下按钮获取可用课程的完整列表Ctrl+Space 。

将实例PHP方法移至另一个类

Move实例方法重构将一个实例（非静态）方法移动到另一个类。

将实例PHP方法移至另一个类

  1. 在编辑器中，将插入符号放置在实例方法上，然后执行以下任一操作：

    • 从主菜单或上下文菜单中，选择“ 重构|移动 。

    • 按F6 。

  2. 在打开的“ 移动实例方法”对话框中，指定以下内容：

    • 在“ 选择实例表达式”列表中，选择要将实例方法移至的目标类。潜在移动目标的列表包括当前类中方法参数的类和字段的类。

    • 在“ 可见性”区域中，为目标方法选择首选的可见性修改器。

    • 在参数名称中替换对类的$ this引用字段中，键入所需的参数名称。执行重构时，将为要移动的方法添加参数，它将替换所有$this对当前类的引用。

  3. 通过单击“ 重构”或“ 预览”来启动重构。有关更多详细信息，请参见重构代码 。

将类移至另一个名称空间

PhpStorm假定项目中的名称空间的排列符合PSR0标准，并在您在名称空间之间移动类时强制您根据此标准保留文件夹结构和名称空间层次结构。

当您指定将类移动到的名称空间时，PhpStorm会自动更新“ 目标目标目录”字段，该字段显示将存储实现该类的文件的文件夹的路径。

将类移至另一个名称空间

  1. 在编辑器中，将插入符号放在要移动的班级上。

  2. 请执行以下任一操作：

    • 从主菜单中，选择“ 重构|移动 。

    • 从选择的上下文菜单中，选择“ 重构” |“选择” 。移动 。

    • 按F6 。

  3. 在打开的“ 移动类”对话框中，指定将类移动到的名称空间以及实现该类的文件的存储文件夹。

    1. 在将类移动到名称空间字段中，指定名称空间以将选定的类移动到。可以是：

      • 当前名称空间下的新名称空间。

      • 当前父项下的另一个从属命名空间。

      • 另一个父项下的现有名称空间或新名称空间。

      在任何情况下，引用该类的import语句都会相应地更新。

    2. 在“ 目标目标目录”字段中，指定要让PhpStorm将实现该类的文件移动到的文件夹。

      PhpStorm假定项目中的名称空间的排列符合PSR0标准，并在您在名称空间之间移动类时强制您根据此标准保留文件夹结构和名称空间层次结构。

      当您指定将类移动到的名称空间时，PhpStorm会自动更新“ 目标目标目录”字段，该字段显示实现该类的文件夹文件的路径。

      对话框打开时， 字段显示与当前名称空间相对应的文件夹的路径。

      路径以以下格式显示：

      ...\
      当您指定新的名称空间名称时，路径会自动更新。 但是，如果要将一个名称空间移动到另一个父名称空间，除非您通过在目录页面上将相关文件夹标记为Sources来为名称空间结构指定根文件夹，否则PhpStorm将不会建议正确的文件夹。 的 设置/首选项对话框Ctrl+Alt+S 。有关详细信息，请参见目录 。

      请执行以下任一操作：

      • 接受显示在字段中的预选路径。

      • 从列表中选择另一个路径。它们都是从名称空间根目录或当前目录中评估的，因此可以安全地选择它们。

      • 请点击然后在打开的对话框中选择一个文件夹。

      • 按N/A并编辑预选路径。请记住，这将来可能会导致自动加载出现问题。

    3. 选中将类移动到新文件复选框。保留此默认选择。

  4. 通过单击“ 重构”或“ 预览”来启动重构，有关更多详细信息，请参阅重构代码 。

复制重构

复制重构允许您将类，文件或目录及其整个结构从一个目录复制到另一个目录。

复制类，文件或目录

  1. 在其中一个视图中选择所需的项目，或在编辑器中将其打开。

  2. 请执行以下任一操作：

    • 从主菜单或选择上下文菜单中，选择“ 重构” |“选择” 。复制 。

    • 按F5 。

    • 在“ 项目”工具窗口中选择所需的类，然后使用以下命令将其拖动到目标位置Ctrl按下。

  3. 在打开的“ 复制”对话框中，指定新名称和目的地，然后单击“ 确定” 。
