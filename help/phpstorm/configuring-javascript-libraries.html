配置JavaScript库

在PhpStorm中， 库是一个文件或一组文件，其功能和方法除了PhpStorm从您编辑的项目代码中检索到的功能和方法外，还添加到PhpStorm的内部知识中。在项目范围内，默认情况下其库是受写保护的。

PhpStorm仅使用库来增强编码辅助（即代码完成，语法突出显示，导航和文档查找）。请注意，库不是管理项目依赖项的方法。

使用TypeScript社区存根（TypeScript定义文件）

TypeScript社区存根也称为TypeScript定义文件，TypeScript声明文件，DefinitelyTyped存根或仅d.ts文件。

在PhpStorm中，可以配置DefinitelyTyped存根并将其用作库，这在以下情况下特别有用：

  • 为了提高代码的完整性，请解析对于PhpStorm静态分析而言过于复杂的库或框架的符号，并为此类符号添加类型信息。

  • 从测试框架解析全局定义的符号。

配置Node.js核心库

要获取已编译到Node.js二进制文件中的fs ， path ， http和其他核心模块的代码完成和参考解析，您需要将Node.js Core模块源配置为JavaScript库。

配置Node.js Core

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | Node.js和NPM 。

  2. 选中“ Node.js的编码辅助”复选框。

    请注意，Node.js Core库是特定于版本的。因此，如果您在Node.js和NPM页面上更改Node.js的版本，则需要再次选中该复选框。之后，PhpStorm为此新版本创建一个新库。

配置node_modules库

为了为项目依赖项提供代码完成功能，PhpStorm会自动创建一个node_modules库，以便将Node.js模块保留在您的项目中，但不会对其进行检查，从而提高了性能。

在“ 项目”工具窗口中， node_modules也被标记为库：

但是， node_modules库仅包含在dependencies项目package.json文件的对象。PhpStorm不将依赖项的依赖项包含到node_modules库中，而是将其从项目中排除。

配置库的范围

您可能会不时注意到PhpStorm建议无关的完成，例如，客户端代码中的Node.js API。发生这种情况是因为默认情况下，PhpStorm使用一个库来完成整个项目文件夹中的操作。PhpStorm允许您通过配置库的作用域来调整代码完成。

配置库的范围

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | JavaScript |图书馆 。JavaScript。打开的“ 库”页面显示所有已可用库的列表。

  2. 选择所需的库，清除其旁边的Enabled复选框，然后单击Manage Scopes 。JavaScript库。使用范围对话框打开。

  3. 请点击并选择要包含在库范围中的文件或文件夹。PhpStorm使您返回“ JavaScript库使用范围”对话框，其中“ 路径”字段显示选定的文件或文件夹。

  4. 对于每个添加的文件或文件夹，从“ 库”列表中选择要配置的库。

示例：配置HTML和Node.js Core库的范围

在PhpStorm中使用全栈JavaScript应用程序时，您常常会注意到代码完成建议在客户端代码中使用一些Node.js API，在Node.js代码中使用DOM API。发生这种情况是因为默认情况下，在整个项目中都启用了具有DOM API的HTML库和具有Node.js API的Node.js Core库。要摆脱不相关的完成建议，您需要配置这些库的作用域。

配置HTML和Node.js Core库的范围

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到语言和框架下的JavaScript ，然后单击库 。JavaScript。“库”页面打开，显示为当前项目配置的所有库。

  2. 清除HTML和Node.js Core项旁边的Enabled复选框。

  3. 单击管理范围 。“ JavaScript库使用范围”对话框打开。

  4. 要配置HTML库的作用域，请单击 ，选择包含客户端代码的文件夹，然后从“ 库”列表中选择HTML 。

  5. 要配置Node.js Core库的作用域，请单击再次，选择包含服务器端代码的文件夹，然后从“ 库”列表中选择“ Node.js Core ”。

    现在，PhpStorm解析HTML和Node.js Core库中的项目，并仅在这些选定项目文件夹中的文件中建议它们完成。

以相同的方式，您可以配置自动创建的node_modules库的作用域，请参见配置node_modules库 。

配置通过CDN链接添加的库

通过CDN链接引用库.js文件时，该文件可用于运行时，但对PhpStorm不可见。要将对象从此类库.js文件添加到完成列表，请下载文件并将其配置为外部库。

下载资料库

    该库已下载到PhpStorm缓存（但未下载到您的项目中），并出现带有信息消息的弹出窗口：在“ JavaScript库”页面上，将下载的库添加到列表中，并在当前项目的范围内启用它：在“ 项目”工具窗口中，该库显示在“ 外部库”节点下：

更改库的可见性

  • 默认情况下，PhpStorm将下载的库标记为Global ，这意味着您可以在任何其他PhpStorm项目中启用并重新使用它。要更改此默认设置，请在列表中选择下载的库，单击“ 编辑” ，然后在打开的“ 编辑库”对话框中选择“ 项目 ”。

配置自定义第三方JavaScript库

假设您在项目中或计算机上的其他任何地方都有一个JavaScript框架文件，并且您希望PhpStorm将其视为库，而不仅仅是您编辑的项目代码。

配置自定义库

  1. 下载所需的框架文件。

  2. 在里面 设置/首选项对话框Ctrl+Alt+S ，单击“ 语言和框架”下的“ JavaScript ”，然后单击“ 库” 。在JavaScript上。打开的库页面，单击添加 。将打开“ 新库”对话框。

  3. 指定外部JavaScript库的名称，单击 ，然后从列表中选择“ 附加文件”或“ 附加目录 ”。在打开的对话框中，选择包含已下载框架的文件或文件夹。

可选地

  • 默认情况下，该库为Global ，这意味着您可以将其附加到任何其他项目。要禁止重复使用库，请在“ 新建库”对话框中选择“ 项目 ”。

  • 默认情况下，该库在整个当前项目的范围内处于启用状态。您可以按照配置库的范围中所述更改此默认设置。

  • 在“ 文档URL”区域中，指定库或框架的正式文档的路径。按下时，PhpStorm将打开此URL。 Shift+F1在此库中的符号上。

查看与文件关联的库

  • 在编辑器中打开文件，或在“ 项目”视图中选择它，然后单击在状态栏上。PhpStorm将打开一个弹出窗口，其中列出了与当前文件关联的库。要更改列表，请单击合并范围中的库链接，然后在打开的“ 管理合并范围”对话框中编辑合并范围设置。

  • 或者，在编辑器中打开文件，然后从上下文菜单中选择“ 使用JavaScript库 ”。PhpStorm将打开一个列表，其中包含可用的已配置库。与当前文件关联的库带有打勾标记。

    • 要从库的作用域中删除当前文件，请清除该库旁边的复选框。

    • 要将库与当前文件关联，请选中该库旁边的复选框。

删除图书馆

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，单击“ 语言和框架”下的“ JavaScript ”，然后单击“ 库” 。JavaScript。“库”页面打开，显示所有已可用库的列表。

  2. 选择所需的库，然后单击“ 删除” 。
