目录

此字段显示项目目录或项目根目录的路径。

对于启用了Git或Mercurial集成的项目，PhpStorm扫描项目目录以检查是否存在不受IDE控制的Git / Mercurial存储库。如果找到了这样的存储库，它们将在此处列在“ 未注册的根目录”下 ，并标记为灰色。要添加未注册的根，请在列表中选择它，然后单击“ 添加”按钮 。

PhpStorm还会检查注册的根是否有效，换句话说，指定路径中是否存在Git / Mercurial存储库。如果检测到无效的存储库，则将其标记为红色。

VCS

选择指定目录的版本控制系统。

该列表仅显示启用了相应插件的版本控制系统。

单击此按钮以打开“ 版本控制配置”对话框，并更新所选VCS的配置设置。

单击此按钮可将目录映射添加到列表。“ 添加VCS目录映射”对话框将打开，您可以在其中指定所需的目录，为其选择一个VCS，然后在必要时打开“ 版本控制配置”对话框来配置指定的VCS。

单击此按钮以编辑选定的目录映射。将打开“ 编辑VCS目录映射”对话框，您可以在其中更新选定的映射并根据需要配置指定的VCS。

单击此按钮可从列表中删除所选目录映射。

将历史记录限制为

选中此复选框可以指定文件历史记录显示的行数。如果选中此复选框，则启用历史深度字段和旋转框。

显示具有更改后代的目录

如果选中此复选框，则包含更改的目录将带有颜色标记。

颜色可以在编辑器设置的“ 配色方案”页面中配置（文件状态-子代已更改，子代已更改）。

搁置分布式版本控制系统下文件的基本修订。

此选项仅与Git和Mercurial相关 。选中此复选框可自动搁置Git或Mercurial版本控制下的文件的基本修订。

默认情况下，PhpStorm始终“记住”最后的提交哈希。但是，如果由于运行rebase操作而导致自上次提交以来历史记录已更改，则此信息是不够的。在这种情况下，拥有基本修订的副本可能会有所帮助。

过去几天的节目已更改

选中此复选框可以在堆栈跟踪分析和调试期间应用颜色指示文件状态 。在一定时间内已更改的文件名将相应高亮显示。

按范围过滤更新项目信息

如果启用此选项并选择了范围，则属于该范围的文件将在工具栏的“ 更新项目信息”选项卡中以粗体标记。 版本控制窗口Alt+9 。如果单击“ 筛选器”按钮在“ 更新项目信息”选项卡的工具栏中，将按范围过滤文件，即仅显示属于所选范围的文件。

单击“ 管理范围”链接以打开“ 范围设置”对话框并配置范围。

提交邮件右边距（列）

在此字段中，指定可以放入“ 提交更改”对话框的右边距中的符号数。

如果希望在达到最大字符数时将文本转移到下一行，请选择“ 键入时到达右边缘时换行”选项。

在“提交”对话框中显示未版本控制的文件

选择此选项可在“ 提交更改”对话框的“未版本化文件”节点下查看尚未添加到版本控制中的新添加文件。

检查提交消息的拼写

如果要自动检查提交消息的拼写，请选中此复选框。
