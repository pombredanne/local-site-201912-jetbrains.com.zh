使用PHP异常断点进行调试

使用PHP Exception Breakpoints ，您可以在脚本开始时启动调试器，并在您自己的断点处中断，或者在发生错误或给定类型的Exception时中断。PHP异常断点不需要配置Xdebug的通过设置在刚刚在实时模式下工作xdebug.remote_mode至jit ，请参见“ 即时模式”中的调试以获取详细信息。

配置PHP异常断点

  1. 从主菜单中，选择运行|查看断点 ，或按Ctrl+Shift+F8 。

  2. 在打开的“ 断点”对话框中，单击 。

  3. 从列表中，选择PHP Exception Breakpoint 。

  4. 在打开的“ 添加例外断点”对话框中，指定要调试器挂起的错误或例外。

    • 要打破PHP错误条件，请从列表中选择一种标准类型，可用选项为Warning ， Notice或Deprecated 。

    • 或者，指定自定义Exception type 。请注意，由于E_ERROR ， E_PARSE和E_COMPILE_ERROR会中止PHP引擎的执行，因此不会对其进行处理。

  5. 准备就绪后，单击“ 确定”以返回到“ 断点”对话框。

  6. 如Breakpoints中所述配置新的异常断点。
