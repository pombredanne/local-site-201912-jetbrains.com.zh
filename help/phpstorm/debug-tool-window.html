调试工具窗口

查看|工具窗口|除错
Alt+5

当您开始调试时，此工具窗口将变为可用。

它显示调试会话为您的应用程序生成的输出。如果要调试多个应用程序，则每个应用程序的输出将显示在以相应的run / debug配置命名的单独选项卡中。

对于每个应用程序，都有以下嵌套选项卡：

  • 控制台 ：显示系统信息和错误消息，以及应用程序的控制台输入和输出。

  • 调试器 ：此选项卡分为以下几个区域：

    • 镜框

    • 变数

    • 手表

  • Elements ：如果您使用Chrome浏览器进行调试，则会显示。

每个区域都有一个上下文菜单 ，可让您配置其行为并在选项卡之间导航。

每个选项卡和区域都可以隐藏/还原 ，或移动到您选择的位置。

步进工具栏

项目

工具提示和快捷方式

描述

显示执行点
Alt+F10

单击此按钮以突出显示编辑器中的当前执行点，并在“ 框架”窗格中显示相应的堆栈框架。

越过
F8

单击此按钮可以执行程序，直到当前方法或文件的下一行，并跳过当前执行点（如果有）引用的方法。如果当前行是方法中的最后一行，则执行步骤将移至此方法之后立即执行的行。

步入
F7

单击此按钮可使调试器进入当前执行点调用的方法。

强制进入
Shift+Alt+F7

单击此按钮可使调试器进入当前执行点中调用的方法，即使此方法将被跳过。

走出去
Shift+F8

单击此按钮可使调试器跳出当前方法，移到紧随其后的那一行。

游标
Alt+F9

单击此按钮可恢复程序执行并暂停直到执行点到达编辑器中当前光标位置的行。无需断点。实际上，在插入符号处为当前行设置了一个临时断点，一旦程序执行暂停，该断点将被删除。因此，如果插入符号位于已经执行的行上，则由于无法回滚到先前的断点，因此将仅继续执行程序以进一步执行。当您深入到方法序列并需要一次退出多个方法时，此操作特别有用。

如果在将您带到指定的行之前为应该执行的行设置了断点，则调试器将在遇到的第一个断点处暂停。

当您需要在特定的行上使用一种临时断点（在该处不应中断程序执行）时，请使用此操作。

评估表达
Alt+F8

单击此按钮可以评估表达式 。

显示用户定义的常数

单击此按钮以显示/隐藏“ 常数”节点，从而切换检索用户定义的常数。

如果用户定义的常数数量很大，则隐藏“ 常数”节点可以提高步进性能。

显示价值地址

单击此按钮可在“ 调试工具”窗口中切换显示变量或对象的已分配内存地址。变量窗格。

显示空的超全局变量

单击此按钮可在“ 调试工具”窗口中显示所有超全局变量之间切换。变量窗格，或仅非空变量 。

添加方法以跳过列表
