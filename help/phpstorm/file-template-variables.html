文件模板变量

文件模板可以包含变量，这些变量在应用模板时将被其值替换。变量是一个以美元符号开头的字符串$然后是变量名。变量名称可以选择用大括号括起来。例如： $MyVariable和${MyVariable}是同一变量的不同符号。

预定义模板变量

以下预定义变量可以在文件模板中使用 ：

变量

描述

${DATE}

当前系统日期

${DAY}

该月的当前日期

${DS}

美元符号$ 。此变量用于转义美元字符，因此不会将其视为模板变量的前缀。

${FILE_NAME}

新PHP文件的名称

${HOUR}

当前时间

${MINUTE}

当前分钟

${MONTH}

这个月

${MONTH_NAME_FULL}

当月的全名（一月，二月等）

${MONTH_NAME_SHORT}

当前月份名称的前三个字母（一月，二月，依此类推）

${NAME}

新实体的名称（文件，类，接口等）

${ORGANIZATION_NAME}

在项目设置中指定的组织名称（ Ctrl+Shift+Alt+S ）

${NAMESPACE}

类或字段名称空间的全限定名称（不带斜杠）

当从模板创建PHP类时 ，如创建PHP类中所述，PhpStorm将自动解析类名称空间。

${PRODUCT_NAME}

IDE的名称（例如，PhpStorm）

${PROJECT_NAME}

当前项目名称

${TIME}

当前系统时间

${USER}

当前用户的登录名

${YEAR}

今年

PhpStorm为PHP包含模板提供了一组附加变量，这些模板用于定义可重用的代码段 ，例如文件头和PHPDoc注释 ）。

以下其他预定义变量可在PHP包含模板中使用 ：

变量

描述

${CARET}

完成变量编辑后的插入位置。仅在文件创建期间生成并插入PHPDoc注释时应用。通过代码创建PHPDoc注释时|产生| PHPDoc块 ，提供了多种功能或方法选择。在这种情况下，可以为多个类，函数，方法或字段创建文档注释，因此${CARET}变量将被忽略。

${CLASS_NAME}

定义为其生成PHPDoc注释的字段或方法的类的名称

${NAME}

将为其生成PHPDoc注释的类，字段或函数（方法）的名称

${NAMESPACE}

类或字段名称空间的全限定名称（不带斜杠）

${PARAM_DOC}

参数的文档注释。评估为一组@param type name线。如果为其生成注释的函数不包含任何参数，则求值为空字符串。

${STATIC}

被替换为static如果生成PHPDoc注释的函数（方法）或字段是静态的，则为关键字。否则计算为空字符串。

${THROWS_DOC}

文档注释中的例外。评估为一组@throws type线。如果为其生成注释的函数没有引发任何异常，则求值为空字符串。

${TYPE_HINT}

要为其生成PHPDoc注释的函数（方法）的返回值。如果通过函数（方法）的静态分析无法检测到返回类型，则求值为void 。

自定义模板变量

除了预定义的模板变量外，还可以指定自定义变量。如有必要，您可以使用以下命令在模板中定义自定义变量的值#set指示。

例如，如果要使用全名而不是通过预定义变量定义的登录名${USER} ，请使用以下结构：

#set（$ MyName =“ John Smith”）

如果未在模板中定义变量的值，则PhpStorm将要求您在应用模板时指定它。
