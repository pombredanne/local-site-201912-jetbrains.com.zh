PhpStorm 2019.3帮助

自动完成代码和路径

本节介绍了各种上下文感知的代码完成技术，这些技术可帮助您加快编码过程。

基本完成

基本代码完成功能可帮助您完成可见性范围内的类，方法和关键字的名称。当您调用代码完成功能时，PhpStorm会分析上下文并建议从当前插入符号位置可以到达的选择（建议还包括实时模板 ）。

非英语键盘布局也可以使用基本完成功能。

如果将基本代码完成功能应用于字段，参数或变量声明的一部分，则PhpStorm会根据项目类型建议一个可能的名称列表。

第二次调用基本代码完成会显示无法访问的类和成员（可以通过应用意图动作将其公开）。

调用基本完成

  1. 开始输入名称。

  2. 按Ctrl+Space或选择代码|完成|从主菜单基本 。

  3. 如有必要，请按Ctrl+Space第二次（或按Ctrl+Alt+Space ）。

    这显示了无法访问的类和成员以及静态字段和方法。

智能完成

智能代码完成功能会过滤建议列表，并仅显示适用于当前上下文的类型。

调用智能完成

  1. 开始输入。

  2. 按Ctrl+Shift+Space或选择代码|完成|从主菜单中选择SmartType 。

    突出显示了针对当前上下文的最合适建议。

声明完成

您可以使用语句完成来创建语法正确的代码构造。它插入必要的语法元素（括号，花括号和分号），并使您处于可以开始键入下一条语句的位置。

完成方法声明

  • 开始输入方法声明并按Ctrl+Shift+Enter开括号后。

完成代码构造

  • 开始输入代码构造并按Ctrl+Shift+Enter 。

    PhpStorm自动完成构造并添加所需的标点符号。插入标记位于下一个编辑位置。

嬉皮完成

嬉皮补全是一种补全引擎，它可以在可见范围内分析您的文本并根据当前上下文生成建议。它可以帮助您从任何当前打开的文件中补全单词。

将插入符号处的字符串扩展为现有单词

  1. 键入初始字符串，然后执行以下操作之一：

    • 按Alt+/或选择代码|完成|循环展开单词可在插入符号前搜索匹配的单词。

    • 按Shift+Alt+/或选择代码|完成|循环扩展单词（向后）可在插入符号之后和其他打开的文件中搜索匹配的单词。

    出现第一个建议值，并且原型在源代码中突出显示。

  2. 接受建议，或按住Alt键并保持按下/直到找到所需的单词。

后缀代码完成

后缀代码完成功能可帮助您减少编写代码时向后插入符号的跳跃。您可以根据在点后键入的后缀，表达式的类型及其上下文，将已经键入的表达式转换为其他表达式。

Postfix完成选项显示为基本完成建议列表的一部分。要查看适用于当前上下文的后缀完成的完整列表，请按Ctrl+J 。

  • 要查看可用的后缀模板的完整列表， 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|一般|后缀完成 。

    从JavaScript Postfix Templates了解更多信息。

  • 选择Tab ， Space ， 要么Enter扩展后缀模板。

用后缀转换语句

  • 键入一个表达式，然后在一个点后键入一个后缀，例如，.if:

    函数m（arg）{arg.if}

    初始表达式用if声明：

    函数m（arg）{if（arg）{}}

要禁用某些后缀完成模板， 在里面 设置/首选项对话框Ctrl+Alt+S选择编辑器|一般|后缀完成 。

您可以选择Tab ， Space ， 要么Enter扩展后缀模板。

您可以编辑预定义的后缀模板，例如，将长键替换为较短的后缀模板，或扩展适用的表达式类型列表。

创建自定义的后缀模板

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|一般|后缀完成 。

  2. 请点击在工具栏上。在打开的弹出菜单中，选择创建后缀模板所需的语言。

  3. 指定Key ，它是将调用模板的符号的组合，选择新模板将适用的表达式类型，并以以下格式键入目标表达式： $EXPR$ ， 例如， $EXPR$ !== null 。

    加$END$应用模板后，要将插入标记放置在的位置，例如：

    $ EXPR $！== null; $ END $

  4. 要使PhpStorm在调用模板时自动转换最上面的适用表达式，请选中“ 应用于最上面的表达式”复选框。

    否则，如果清除该复选框，则PhpStorm会在您调用模板时提示您选择表达式：

标签和属性的完成

PhpStorm自动完成许多文件类型中标签和属性的名称和值：

  • HTML / XHTML，包括CSS类和JSX内部HTML标记的 完成 。

  • XML / XSL ，包括名称空间的完成。

  • JSON，有关详细信息，请参见编辑package.json 。

标记和属性名称的完成取决于与文件关联的DTD或Schema 。如果没有架构关联，则PhpStorm将使用文件内容（标记和属性名称及其值）来完成输入。

完整的标签名称

  1. 按<并开始输入标签名称。PhpStorm显示适用于当前上下文的标记名称列表。

    使用Up和Down按钮滚动列表。

  2. 按Enter接受列表中的选择。如果您的文件与模式或DTD关联，则PhpStorm会根据文件自动插入必填属性。

配置代码完成设置

配置完成选项

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择编辑器|一般|代码完成 。

    • 要自动显示建议列表 ，请选中“ 键入时显示建议”复选框。如果取消选中该复选框，则必须通过按显式调用代码完成Ctrl+Space基本完成或Ctrl+Shift+Space智能完成。

      您也可以选择只有一种选择时自动插入建议：在中为自动插入单个建议下选择完成类型。

    • 要按字母顺序对建议进行排序 ，而不是按相关性对建议进行排序 ，请选中按字母顺序对建议进行复选框。

      您也可以通过以下方式切换这些模式要么分别在建议列表的右下角。

      仅当列表很长时才会显示排序图标，而对于仅包含少量条目的列表则不会显示。

    • 如果在建议完成选项时希望将单词大小写考虑在内 ，请选择“ 区分大小写”，然后选择是仅对首字母还是对所有字母都匹配大小写。

    • 如果你想为你滚动它的建议列表中每个项目自动显示文档弹出式窗口 中 ，选择选项显示文档弹出式窗口 。在右侧的字段中，指定延迟（以毫秒为单位），之后应显示弹出窗口。

完成路径

这种完成方式可以加快文件和文件夹的选择速度。例如，当您指定外部工具或版本控制引擎的安装文件夹或配置PHP解释器时，通常会调用用于此目的的对话框 。

文件和文件夹路径的完成在macOS上不可用。

  1. 通过在文件菜单上选择打开文件或通过单击浏览打开选择路径对话框 在需要路径的字段旁边。

  2. 确保“ 路径”字段可见。如果该字段是隐藏的，则单击“ 显示路径”链接。

  3. 开始输入路径。PhpStorm提供了一个列表，您可以在其中快速找到所需的目录。

    键入时，建议列表将缩小以仅显示匹配的路径。您也可以按Ctrl+Space显示建议列表。

  4. 从列表中选择相关建议。

完成提示和技巧

缩小建议列表

您可以通过键入单词的任何部分（甚至是中间位置的字符）或在点分隔符或-> 。PhpStorm将显示建议，其中包括您在任何位置输入的字符。

这使得不需要使用通配符：

如果使用CamelCase或snake_case名称，请仅输入首字母。PhpStorm会自动识别并匹配首字母。

接受建议

您可以通过以下方式之一接受列表中的建议：

  • 按Enter或双击列表项以将其插入插入符号的左侧。

  • 按Tab替换插入符号右侧的字符。

  • 使用Ctrl+Shift+Enter使当前的代码构造在语法上正确（平衡括号，添加缺少的花括号和分号，等等）。

您还可以使用特定的键插入所选的完成建议： 在里面 设置/首选项对话框Ctrl+Alt+S选择编辑器|一般|代码完成，然后按空格，点或其他上下文相关的键选项，选择“ 插入所选建议” 。这些键取决于语言，您的上下文等。

虽然此设置可以帮助您节省时间，但是将其打开可能会导致意外插入项目。

查看代码层次结构

从建议列表中选择一个条目后，您可以查看代码层次结构 ：

  • Ctrl+H -视图类型层次

  • Ctrl+Alt+H -查看通话层次。

  • Ctrl+Shift+H -查看方法层次结构。

故障排除

如果无法完成代码，则可能是由于以下原因之一：

  • 省电模式已打开（ 文件|省电模式 ）。开启它可以消除背景操作，包括错误突出显示，即时检查和代码完成，从而将笔记本电脑的功耗降至最低。

  • 您的文件不驻留在内容根目录中，因此它没有获得代码完成所需的类定义和资源。

  • 包含要在完成建议列表中显示的类和函数的文件被标记为纯文本文件。

  • 包含要在完成建议列表中显示的功能的外部库不会作为依赖项或全局库添加 。
