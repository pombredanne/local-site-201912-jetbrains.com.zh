从命令行启动代码检查

除了从主菜单或“ 项目工具窗口”的上下文菜单中运行代码检查之外，您还可以从命令行启动检查器，而无需实际运行PhpStorm。

这样，您可以在开发过程中执行常规代码检查，这对于大型项目尤其重要。检查结果以XML格式存储，您可以随时下载和查看这些结果。

从命令行启动代码检查

  • 在命令行中，运行inspect使用以下语法的命令：

    检查项目路径检查路径输出路径[-vX] [-d子目录路径]

的inspect如果PhpStorm正在运行，则公用程序将失败，因此您必须在执行命令之前退出IDE。

命令选项如下：

选项

描述

inspect

的完整路径inspect.bat（inspect.sh在MacOS和Linux）脚本位于/ bin目录。

project-path

包含要检查的项目的目录的完整路径。

请注意，您需要指定完整路径。相对路径不被接受。

inspection-path

要检查项目的检查配置文件的完整路径。

检查配置文件是一个.xml文件，用于指定已启用的检查及其选项。

概要文件有两种：项目概要文件和IDE概要文件，它们保存在不同的位置。

默认情况下，项目检查配置文件存储在.idea / inspectionProfiles节点下的项目目录中。

IDE配置文件位于以下目录中：

  • Windows： ％HOMEPATH％\。PhpStorm2019.3 \ config \检查

  • macOS： 〜/ Library / Preferences / PhpStorm2019.3 / inspection

  • Linux： 〜/。PhpStorm2019.3 / config /检查

output-path

现有目录的完整路径，该目录将存储带有检查报告的XML文件。

-vX

（可选）输出的详细级别，其中X是0为了安静1 –嘈杂，和2 –额外的噪音。

-d subdirectory-path

（可选）要检查的子目录的完整路径。

例子

句法
\ bin \ inspect.bat项目路径检查路径输出路径-v2 -d子目录路径
例
\ bin \ inspect.bat E：\ MyProject E：\ MyProject \ .idea \ inspectionProfiles \ Project_Default.xml E：\ MyProject \ inspection-results-dir -v2 -d E：\ MyProject \ subdirectory
句法
PhpStorm.app/Contents/bin/inspect.sh项目路径检查路径输出路径-v2 -d子目录路径
例
PhpStorm.app/Contents/bin/inspect.sh〜/ MyProject〜/ MyProject / .idea / inspectionProfiles / Project_Default.xml〜/ MyProject / inspection-results-dir -v2 -d〜/ MyProject /子目录
句法
/bin/inspect.sh项目路径检查路径输出路径-v2 -d子目录路径
例
/bin/inspect.sh〜/ MyProject〜/ MyProject / .idea / inspectionProfiles / Project_Default.xml〜/ MyProject / inspection-results-dir -v2 -d〜/ MyProject /子目录

查看离线检查的结果

执行脱机检查并将检查结果导出为XML格式的目录后，您始终可以下载并查看这些结果。

  1. 打开要对其运行检查的项目。

  2. 在主菜单中，选择代码|查看离线检查结果 。

  3. 在打开的“ 选择路径”对话框中，单击存储带有检查结果的XML文件的目录。

  4. 单击确定 。检查结果显示在“ 检查结果工具”窗口的“ 脱机视图”选项卡中。

或者，您可以在PhpStorm或任何其他文本处理器中打开相关的XML文件，而无需打开检查的项目。
