代码覆盖率

基本

PhpStorm中的代码覆盖率使您能够以较低的运行时开销为代码执行动态的行覆盖率测量。通常，行覆盖率会回答以下问题：“在单元测试仿真过程中是否执行了此行代码？”

测量代码覆盖率可用于PHPUnit测试 ， Mocha测试和Karma测试 。

代码覆盖率结果显示在覆盖工具窗口，在项目视图项目工具窗口 ，并在编辑器。工具窗口显示以下信息：

  • 对于目录：涵盖的类和行的百分比。

  • 对于文件：覆盖行的百分比。

在编辑器中打开文件时，每行都会突出显示其代码覆盖率状态：

  • 仿真期间执行的行标记为绿色。

  • 仿真期间未执行的行标记为红色。

覆盖范围测量结果包括一个覆盖范围套件 。您可以将新模拟的结果与任何现有套件合并。在这种情况下，如果一条线被至少一个模拟覆盖，则该线将被视为覆盖。

每当执行具有代码覆盖率度量的测试或应用程序时，就会生成覆盖率套件。可能有无限数量的覆盖套件。

以代码覆盖率运行

在项目中使用代码覆盖率的一般步骤

  1. 指定您要如何处理coverage结果 。

  2. 如果要测量测试的代码覆盖率，请为目标代码创建测试。

  3. 在所需的运行/调试配置中配置代码覆盖率度量 。

  4. 使用主菜单中的专用命令运行| 覆盖 运行以Coverage运行 ，或单击以Coverage运行按钮 。

  5. 一旦执行覆盖运行，您就可以执行以下操作：

    • 使用各种覆盖套件 。

    • 查看代码覆盖率数据 。
