发送反馈

PhpStorm 2019.3帮助

检查暂停的程序

调试器会话启动后 ，将出现“ 调试工具”窗口，程序将正常运行，直到发生以下情况之一：

  • 断点被击中

  • 您手动暂停程序

之后，该程序将被挂起，允许您检查其当前状态，控制其进一步执行以及在运行时测试各种方案。

如果您不小心关闭了Debug工具窗口，请转到主菜单，然后选择View |工具窗口|调试或按Alt + 5重新打开它。

检查框架

程序的状态由框架表示。当程序挂起时，当前框架堆栈显示在“ 调试”工具窗口的“ 框架”选项卡上。

框架对应于活动的方法或函数调用。它存储被调用方法或函数的局部变量，其参数以及启用表达式求值的代码上下文。

每次调用方法时，都会在堆栈顶部添加一个新框架。方法执行完成后，将其对应的框架从堆栈中删除（以后进先出的方式）。

检查框架可以帮助您了解为什么将特定参数传递给方法以及调用时调用者的状态。

将堆栈复制到剪贴板

  • 要复制当前线程的调用堆栈，请右键单击“ 框架”选项卡上的任意位置，然后选择“ 复制堆栈” 。

检查/更新变量

“ 变量”选项卡显示所选框架/线程中的变量列表。检查变量有助于理解为什么程序以某种方式运行。

注意可变范围和生存期。如果列表中没有变量，则表示该变量超出了当前执行点上当前框架的范围。

每个变量左侧的图标指示其类型。

复制变量

检查变量时，您可能需要复制一个变量名称或值以将其粘贴到其他位置或与另一个变量进行比较 。

  • 要复制变量保存的值，请右键单击该变量，然后选择“ 复制值 Ctrl + C” 。

  • 要复制变量的名称，请右键单击该变量，然后选择复制名称 。

用剪贴板比较变量

当您需要将变量值与其他值进行比较时 ，请使用“ 与剪贴板比较值”选项。例如，当变量包含一个长字符串，并且您需要将其与另一个长字符串进行比较时，这很有用。

  1. 复制您要比较的内容（例如，从文本文件中复制）。

  2. 在“ 变量”选项卡中，右键单击要与之进行比较的变量，然后选择“ 使用剪贴板比较值” 。

  3. 检查打开的差异查看器中的差异。有关如何有效使用Diff Viewer的其他信息，请参阅“ 比较文件和文件夹”主题。

在专用对话框中查看变量

PhpStorm允许您在专用对话框中检查变量。当您需要跟踪某个变量（或其引用所在的对象）并且同时能够在框架和线程之间导航时，此功能很有用。

  • 右键单击“ 变量”选项卡上的变量，然后选择“ 检查” 。

设置变量值

如果需要测试程序在特定条件下的行为或在运行时修复其当前行为，则可以通过设置/更改变量值来实现。

  1. 右键单击“ 变量”选项卡上的变量，然后选择“ 设置值” ，或者选择该变量，然后按F2键。

  2. 输入变量的值，然后按Enter 。

导航到源代码

如果您需要查看声明了某些变量或类的源代码，则可以从“ 变量”选项卡向右移动。

  • 要导航到声明该变量的代码，请右键单击一个变量，然后选择“ 跳转到源代码 F4” 。

  • 要导航到变量类型的类声明，请右键单击一个变量，然后选择“ 跳转到类型源 Shift + F4” 。

    假设您有$my_car = new Car(); 。如果在“ 变量”窗格中选择$my_car ，则选择“ 跳转到源代码”将使您进入$my_car = new Car();同时选择“ 跳转到类型源”将带您进入class Car() 。

评估表达式

PhpStorm允许您在调试会话期间评估表达式，以获得有关程序状态的其他详细信息或在运行时测试各种方案。

在评估表达式时，请注意可变范围和生存期。所有表达式都在当前执行点的上下文中求值。

在编辑器中评估一个简单的表达式

评估表达式的最简单方法是在代码中指向它。尽管这是最快的方法，但不能用于评估方法调用。为了安全起见，这样做可能会产生副作用。

当需要从编辑器快速求值表达式时，请使用此选项。

  1. 指向要评估的表达式。表达式的结果出现在工具提示中。

  2. 如果需要查看子元素，请单击或按Ctrl + F1 。

如果发现有价值的工具提示分散了您的注意力，则可以增加延迟或将其完全禁用。去做这个， 在里面 设置/首选项对话框Ctrl + Alt + S ，转到“ 构建，执行，部署” |“设置”。调试器数据视图，并根据需要设置“ 显示值”工具提示和“ 值”工具提示延迟选项。

在编辑器中评估复杂的表达式

如果要在涉及方法调用的代码中评估表达式，或者要具体确定要评估的表达式部分，请使用“ 快速评估表达式”选项。

仅当程序在达到断点后挂起（不是手动暂停）时，此选项才可用。

  1. 将插入号放在表达式上（以评估最接近的匹配表达式）或选择它的一部分（如果要具体说明要评估的复杂表达式的哪一部分）。

  2. 点击运行|快速评估表达式 Ctrl + Alt + F8 。

如果从表达式调用的方法内部存在断点，则将忽略这些断点。

调用方法时，请确保您意识到它们可能产生的副作用（例如，对外部变量的更改），因为它们可能会改变程序流程或结果。

您可以将“ 快速评估”配置为仅选择一段代码即可工作（不使用菜单/快捷方式）。请谨慎使用此选项，因为启用该选项时可能会意外调用方法。

要配置对代码选择的快速评估 ，请转到“设置/首选项” |“设置”。构建，执行，部署|调试器数据视图 ，并将“ 在代码选择上显示值工具提示”选项设置为首选。

评估任意表达式

评估任意表达式是最灵活的评估选项。只要它在当前框架的上下文中，它就可以评估任何代码。使用它，您可以评估声明，方法调用，循环，匿名类，lambda等。

使用此功能可获得有关程序当前状态的更多信息，并在同一调试会话中测试所有方案。通过减少必须运行的会话数，可以节省大量时间。

仅当程序在达到断点后挂起（不是手动暂停）时，此选项才可用。

  1. 如果要从当前位于您前面的某些表达式或变量开始（例如，在编辑器中或在“ 变量”选项卡上），请选择它。

  2. 点击运行|计算表达式 Alt + F8 。该快捷方式可能在Ubuntu上不起作用（要正确操作，请调整快捷方式配置 ）。

  3. 在“ 评估”对话框中，修改所选的表达式或在“ 表达式”字段中输入一个新表达式 。如果要评估代码片段，请单击Expand Shift + Enter 。

    请记住，在表达式的主体中声明的所有变量在求值之后都会超出范围。

  4. 单击评估 Ctrl + Enter （对于多行模式， 按Ctrl + Enter ）。表达式结果出现在“ 结果”字段中。

    表达式的结果取自return语句。当没有return语句时，结果取自代码的最后一行（它甚至不必是表达式：文字也可以）。如果没有有效的行来取值，则结果undefined 。如果无法评估指定的表达式，则“ 结果”字段将指示原因。

如果从表达式调用的方法内部存在断点，则将忽略这些断点。

调用方法时，请确保您意识到它们可能产生的副作用（例如，对外部变量的更改），因为它们可能会改变程序流程或结果。

“ 评估”对话框是非模态的，因此您可以将焦点切换回编辑器以复制其他变量和表达式。如果需要，您还可以打开多个“ 评估”对话框。

内联查看值

PhpStorm通过在变量用法旁边显示变量的值来简化调试过程。

变量值更改后，将使用新值更新内联视图并更改其颜色。

内联值视图默认情况下处于启用状态。要关闭它， 在里面 设置/首选项对话框Ctrl + Alt + S ，转到“ 构建，执行，部署” |“设置”。调试器数据视图并禁用内联显示值选项。

使用手表

如果要跟踪某些变量或更复杂的表达式的结果，请监视此变量或表达式。当您需要添加未在变量列表中定期显示的内容或固定某些实例变量，从而无需在每个步骤之后都展开树时，此功能很有用。

仅当程序在达到断点后挂起（不是手动暂停）时，此选项才可用。

调用方法时，请确保您意识到它们可能产生的副作用（例如，对外部变量的更改），因为它们可能会改变程序流程或结果。

在所选框架的上下文中评估手表。当它们脱离上下文或无法编译时，无法评估它们。在这种情况下，手表会标有错误图标 。

默认情况下，“ 监视”选项卡是隐藏的，并且监视在“ 变量”选项卡上显示。要隐藏/显示“ 手表”标签，请使用“ 在变量中显示手表”标签按钮在变量或监视选项卡上。

添加手表

  1. 点击新手表 在“ 手表”标签上。

  2. 输入要评估的变量或表达式。在表达式中，只要在本地上下文中，就可以评估方法调用，lambda，声明变量等。

如果要跟踪的变量或表达式已经在您的前面（例如，在代码编辑器中或在“ 变量”选项卡上），则只需选择并将其拖到“ 监视”选项卡即可。对于变量，也可以在“ 变量”选项卡中右键单击它们，然后选择“ 添加到手表” 。

将变量/表达式添加到Watches后 ，它会保留在那里并针对每个步骤进行评估，从而为您提供当前上下文中的结果。

编辑手表

  • 右键单击所需的手表，然后选择编辑 。

复制手表

  1. 选择您要复制的手表。

  2. 点击重复观看 在“ 变量 / 监视”选项卡上，或按Ctrl + D。

更改手表的顺序

为了方便起见，您可以更改手表在“ 变量 / 手表”选项卡上的显示顺序。

  • 使用移动观看向上 / 移动的变量 / 手表标签，或者Ctrl +向上和Ctrl +向下的键盘快捷键观看向下按钮。

删除手表

  • 要删除单个手表，请右键单击它， 然后在“ 变量 / 手表”选项卡上选择“ 删除手表 N / A ”。

  • 要删除所有手表，请右键单击“ 变量 / 手表”选项卡上的任意位置，然后选择“ 删除所有手表” 。

监视允许执行与变量相同的操作。例如，您可以在专用对话框中查看它们，或使用它们导航到源代码 。

手表是您项目的一部分。这意味着您可以停止并重新运行调试会话，而不会丢失它们。

返回当前执行点

检查程序状态涉及浏览代码，并且您通常需要返回到程序被挂起的位置。

请执行以下任一操作：

  • 从主菜单中，选择运行|显示执行点 。

  • 按Alt + F10 。

  • 请点击在调试工具窗口的步进工具栏上。

当前执行点用蓝线表示。此行的代码尚未执行。
