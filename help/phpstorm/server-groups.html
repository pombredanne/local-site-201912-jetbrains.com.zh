将服务器分组

服务器组使您可以将服务器配置分组在一起，并像对待单个实体一样使用它们。如果需要将代码部署到多个服务器，则可以使用一个服务器组，并避免将其分别部署到每个服务器。

创建服务器组

  1. 通过执行以下操作之一打开“ 部署”页面：

    • 在里面 设置/首选项对话框Ctrl+Alt+S选择部署下建立，执行，部署 。

    • 选择工具|部署|从主菜单进行配置 。

  2. 在列出所有现有服务器配置的左侧窗格中，单击“ 添加”。 然后在弹出菜单中选择服务器组 。

  3. 在打开的“ 创建新组”对话框中，键入要创建的组的名称，然后单击“ 确定” 。关闭“ 创建新组”对话框，然后返回到“ 部署”节点的“ 连接”选项卡。

  4. 要在组内创建新的服务器配置，请在左侧窗格中选择该组，然后单击右侧窗格中的“ 添加新服务器”链接或工具栏按钮。然后，按照配置与Web服务器的同步中所述的类型，配置新创建的服务器。

    要将现有服务器配置添加到组中，请将其拖动到组中。要删除服务器配置，请将其拖出组。

  5. 单击用作默认值 在以下情况下，使PhpStorm静默应用当前服务器组：

    • 自动上传更改的文件 。

    • 手动上传和下载文件，而无需选择目标主机。

    • 编辑远程主机上的单个文件 。

  6. 使用“ 对于当前项目”复选框来配置服务器组可见性，这也适用于该组中包含的所有服务器。

    • 选中复选框以将组的使用限制为当前项目。此类组不能在当前项目外部重用。它不会出现在其他项目的可用配置列表中。

    • 清除复选框后，该组将在所有PhpStorm项目中可见。它的设置（包括SFTP服务器的SSH凭据）可以在多个项目中重复使用。

现在可以使用创建的服务器组将文件上传和下载到包含的服务器。
