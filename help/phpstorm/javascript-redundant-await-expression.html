代码检查：冗余的“等待”表达

报告冗余的await用法，例如await await或await a non-promise结果。
选中“为承诺报告”复选框时，此检查还将建议在适用时（在返回语句中，并带有Promise.resolve / reject）删除在承诺之前的“等待”。
尽管通常不需要这样做，但是在这种情况下删除“ await”会给您带来两个问题。
第一个是用try-catch包围代码，而忘记添加“ await”会改变代码的语义，您可能没有注意到这一点。
第二个问题是没有明确的“等待”可能会阻止V8运行时提供异步堆栈跟踪 。
