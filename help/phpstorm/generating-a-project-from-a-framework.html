从框架模板生成项目

PhpStorm可以为您创建新的特定于框架的应用程序，设置它们的结构，并在必要时下载所需的软件包。您也可以使用Yeoman生成器，有关详细信息，请参见Yeoman 。

作曲者专案

您可以使用Composer从现有包中生成项目存根。在这种情况下，将在项目存根创建期间初始化Composer并安装一个软件包。之后，您可以从命令行或通过用户界面使用Composer。

仅当通过配置的本地PHP解释器运行Composer时，才可以创建项目。

  1. 选择文件|新建项目或在欢迎屏幕上单击创建新项目 。

  2. 在左侧窗格中，选择Composer Project 。 在右侧窗格的“ 位置”字段中，指定将在其中创建项目的文件夹。

  3. 选择如何执行Composer命令：

    • 要通过全局安装的composer可执行文件运行命令，请选择“ composer”可执行文件并在字段中指定其位置。

    • 要通过本地composer.phar存档运行命令，请选择composer.phar 。然后，在字段中指定现有composer.phar文件的位置，或选中从getcomposer.org下载composer.phar复选框以下载文件的新实例。 下载的composer.phar文件将保存在项目根文件夹下。

      然后，从PHP解释器列表中选择已配置的本地PHP解释器之一 。有关详细信息，请参考配置本地PHP解释器 。

  4. 在“ 软件包”区域中，从列表中选择软件包，然后从“ 要安装的版本”列表中选择相关的版本。要搜索软件包，请使用过滤软件包搜索字段。

  5. （可选） 在命令行参数字段中， 键入其他命令行参数。

  6. 点击创建 。create-project Composer命令将与所选软件包一起调用。结果，将创建Composer项目，其配置和结构取决于所选的软件包。之后，打开创建的PhpStorm项目。

Drupal模块

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择Drupal Module 。

  3. 在右侧窗格的“ 位置”字段中，指定将在其中创建项目的文件夹。

  4. 指定Drupal安装的根文件夹，然后选择要使用的Drupal版本，即支持的版本为6、7和8。

  5. 选择设置PHP |包含路径复选框可让Drupal包含为项目自动配置的路径。 退出对话框后，以下路径将添加到PHP页面上的“ 包含路径”列表中： / includes ， / modules和/ sites / all / modules

之后，您可以按照更改Drupal设置中所述更改 Drupal安装并重新配置Frameworks页面上的包含路径。

PhpStorm根据所选的Drupal版本生成项目存根。对于Drupal 8，将生成一个module_name.info.yml文件，其中提供了有关该模块的信息。

Joomla！项目

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择Joomla！整合 。

  3. 在右侧窗格的“ 位置”字段中，指定将在其中创建项目的文件夹。

    • Joomla安装路径

    • 项目类型（组件，模块或插件）

    PhpStorm显示气球，其中包含有关丢失的Joomla的通知！代码样式，名称空间等。通过单击相应的链接，可以立即安装这些元素。通知在状态栏和“ 事件”工具窗口中重复。

PhpStorm创建存根Joomla！包含PHP类.php和.xml 清单文件的项目 。

请参阅“ Joomla ”部分！ 和Joomla！网站 。

WordPress插件

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择WordPress插件 。

  3. 在右侧窗格中，指定将在其中创建项目的文件夹。

    在WordPress安装路径字段中， 指定安装WordPress的文件夹。此文件夹应包含wp-admin和wp-includes子目录。单击“ 创建”开始生成项目存根。

  4. 如果新创建的项目不在WordPress安装文件夹中，请将其添加到包含路径列表中，以将其配置为外部库 。

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | PHP 。

    2. 在打开的PHP页面上，将WordPress安装文件夹的路径添加到“ 包含路径”列表中：单击按钮，然后在打开的对话框中指定安装文件夹的路径。 在配置包含路径中了解更多信息。

AngularJS应用

在你开始前：

  • 如管理插件中所述，在“ 插件”页面上安装并启用AngularJS存储库插件。

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择AngularJS 。

  3. 在右侧窗格中， 指定将存储与项目相关的文件的文件夹的路径。

  4. 当您点击创建时 ， PhpStorm基于AngularJS种子项目，使用所有必需的配置文件生成特定于AngularJS的项目结构

  5. 下载包含AngularJS代码以及支持开发和测试的工具的AngularJS依赖项： 在嵌入式终端 （ Alt+F12 ），输入：

    npm install

    在readme.md文件的Install Dependencies部分中了解有关安装依赖项的更多信息。

PhpStorm可以生成一个项目存根，用于使用AngularJS开发应用程序。要生成Angular存根，请使用Angular CLI ，请参阅创建新的Angular应用程序 。

HTML5 Boilerplate，Web Starter Kit，Bootstrap或Foundation应用程序

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，根据您的首选项和应用程序中要实现的功能，选择要使用的模板：

    • HTML5 Boilerplate：选择此选项以使用HTML5 Boilerplate模板。这是一个初始项目模板，可以轻松地适应您的需求。

    • Web Starter Kit：选择此选项以使用Web Starter Kit 。

    • Bootstrap：选择此选项以使用Bootstrap模板，该模板是精心设计的模块化工具包，具有丰富的HTML，CSS和JavaScript支持。

    • Foundation：选择此选项以使用Foundation框架。

  3. 右侧窗格中的控件集取决于所选模板。

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 从版本列表中，选择要使用的模板版本，然后单击创建 。

Node.js Express应用程序

安装Node.js并将其配置为本地Node.js解释器 。

生成Node.js应用程序

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择Node.js Express App 。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. Node解释器，请参阅配置本地Node.js解释器 。

    3. 要使用-npm或Yarn的程序包管理器，请参见npm和Yarn 。

    4. 要使用的Express模板引擎 。从模板引擎列表中，选择以下选项之一：
      • 灰尘
      • EJS
      • 车把
      • Hogan.js

      • 帕格 （ 帕格（玉） ）

      • 枝条
      • 瓦什

      • 静态HTML

    5. 要使用的CSS引擎。从CSS引擎列表中，选择以下选项之一：
      • 纯CSS
      • 触控笔
      • 减

      • 指南针 。

      • 萨斯

  4. 准备好后，点击创建 。

    PhpStorm将启动Express Project Generator工具，该工具将下载所有必需的数据，设置项目结构，并根据您的选择在当前窗口或新窗口中打开项目。

有关详细信息，请参见Node.js。

飞镖应用

在你开始前：

  1. 下载并安装Dart SDK 。有关详细信息，请参见Dart 。

  2. 如管理插件中所述，在“ 插件”页面上安装并启用Dart存储库插件。

生成Dart应用程序

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择Dart 。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 指定Dart SDK的路径。PhpStorm检测并显示Dart版本 。

    准备好后，单击创建 。

如有必要，可以通过单击“ 设置”按钮并配置命令行选项来打开“ 设置”对话框的Dart页面。

流星应用

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择Meteor App 。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 指定Meteor可执行文件的位置（请参阅安装Meteor ）。

    3. 在模板列表中， 选择要生成的样本。要生成基本的项目结构，请选择“ 默认”选项。

    4. 在文件名字段中， 输入将相互生成的.js ， .html和.css文件的名称。仅当从“ 模板”下拉列表中选择“ 默认样品类型”时，此字段才可用。

  4. 当您单击“ 创建”时 ，PhpStorm会生成Meteor应用程序的框架，包括HTML文件，JavaScript文件，CSS文件和带有子文件夹的.meteor文件夹。 PhpStorm自动将用于存储已构建应用程序的.meteor / local文件夹标记为已排除，但仍将其显示在项目树中。

    默认情况下， 排除的文件显示在项目树中。要隐藏.meteor / local文件夹，请单击在“ 项目”工具窗口的工具栏上，删除“ 显示排除的文件”选项旁边的对勾。

PhoneGap / Cordova / Ionic应用程序

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择Cordova App 。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 指定可执行文件phonegap.cmd或cordova.cmd或ionic.cmd的位置 （请参阅安装PhoneGap / Cordova / Ionic ）。

  4. 当您单击Create时 ，PhpStorm会生成具有框架特定结构的PhoneGap / Cordova / Ionic应用程序的框架。

使用create-react-app反应应用程序

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择React App 。

  3. 在右侧窗格中，指定项目文件夹，要使用的Node.js解释器以及create-react-app包的路径。

  4. 可选地：

    指定在项目生成期间要使用的自定义程序包，而不是react-scripts 。这可以是从react-scripts分叉的软件包之一，例如， react-awesome-scripts ， custom-react-scripts ， react-scripts-ts等。

  5. 当您单击Create时 ，PhpStorm会使用所有必需的配置文件生成一个特定于React的项目。

从React官方网站上了解有关安装React和创建React应用程序的更多信息。

带有@ vue / cli的Vue.js应用程序

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择Vue.js。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 在“ 节点解释器”字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择添加以配置新的解释器。

    3. 从Vue CLI列表中，选择npx --package @ vue / cli vue 。

      另外，对于npm 5.1及更早版本，请安装@vue/cli通过运行打包自己npm install --g @vue/cli在航站楼 Alt+F12 。创建应用程序时，选择@vue/cli包已存储。

    4. 从项目模板列表中，选择要使用的Vue.js模板 。

  4. 当您单击Create时 ，PhpStorm会询问有关工具的其他一些问题，这些工具可以添加到项目中，例如linters和testRunner。这些问题取决于您选择的模板。之后，PhpStorm将使用所有必需的配置文件生成特定于Vue.js的项目。
