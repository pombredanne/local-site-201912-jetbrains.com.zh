Laravel是一个免费的开源PHP Web应用程序框架。它建立在几个Symfony组件的基础上，使常见的任务（如身份验证，路由，会话和缓存）更易于实现。

Laravel支持通过Laravel插件和Laravel IDE帮助器提供 。

在你开始前

在开始使用Laravel之前，请确保已安装并启用 Laravel插件。

此外，请确保Composer已安装在您的计算机上并按照Composer Dependency Manager中所述在当前项目中进行了初始化。

执行初始配置

  1. 使用Composer安装Laravel IDE帮助程序生成器 。为此，请打开“ 工具” |“工具”。作曲家|从主菜单管理依赖项 ，然后安装barryvdh / laravel-ide-helper软件包。有关详细信息，请参阅安装特定的软件包 。

  2. 将Laravel IDE帮助程序添加为ServiceProvider进入应用程序。在config / app.php文件中，添加Barryvdh\LaravelIdeHelper\IdeHelperServiceProvider::class在下面providers元件：

    return array（// ...'providers'=> array（// ...// Laravel IDE帮助程序'Barryvdh \ LaravelIdeHelper \ IdeHelperServiceProvider'，），// ...）;

在更改或添加服务，控制器，模型和视图之后，可能必须运行Laravel IDE Helper。或者，在PhpStorm中设置File Watchers ，以在例如composer.json更新时自动重新生成此文件。

您还可以安装Laravel生成器Composer软件包 ，为模型，视图，控制器等添加各种Laravel生成器。

配置刀片模板

添加，修改或删除Blade指令

在PhpStorm中， Blade指令在Blade Page的Directives选项卡上进行管理。 该选项卡列出了所有当前可用的Blade指令，对于具有参数的指令，还会显示前缀和后缀。当您开始时，列表仅包含预定义的指令。您可以编辑这些指令以及创建自定义指令。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| PHP |刀片 。

  2. 在打开的“ 刀片”页面上，切换到“ 指令”选项卡，其中显示了所有当前可用指令的列表。

    • 要定义新指令，请单击并在“ 名称”字段中指定指令的名称。

      如果新指令需要前缀和后缀，请选中“ 具有参数”复选框，然后分别在“ 前缀”和“ 后缀”字段中键入要使用的前缀和后缀 。PhpStorm会自动将前缀和后缀括在左，右括号和引号中，并添加冒号分隔符:因此参数将如下所示：（ “ ： ”） 。

    • 要编辑现有指令，请在列表中选择它，然后更改以下字段中的值。

      要恢复原始定义，请单击 。

    • 要从列表中删除指令，请选择它并单击 。

配置刀片分隔符

PhpStorm可以识别Blade模板，并根据您指定的分隔符为其提供错误突出显示和代码完成。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| PHP |刀片 。

  2. 在打开的“ 刀片”页面上，切换到“ 文本标签” 。 选项卡中的字段显示原始标签 ， 内容标签和转义标签的开始和结束字符。

  3. 这些字段使用符合Blade Templates 5.8的默认值填充。如果使用的是较早版本，则可以指定相关的自定义分隔符，PhpStorm将根据新规则提供编码帮助。

使用PhpStorm的artisan命令行工具

将artisan配置为命令行工具

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到工具|命令行工具支持 。

  2. 请点击在工具栏上。

  3. 在“ 命令行工具”对话框中，从列表中选择“ 基于Symfony控制台的工具 ”，然后指定其可见性级别（ Project或Global ）。

  4. 单击确定时 ，将打开工具设置对话框。

    指定工具别名，提供artisan和PHP可执行文件的路径。

  5. 单击“ 确定”以应用更改并返回“ 命令行工具支持”页面。（可选）单击编辑工具属性，或自定义命令集。有关详细信息，请参见自定义工具 。

您现在可以运行artisan ide-helper:generate命令生成所需的PHPDoc信息。PhpStorm和Laravel插件将使用此信息来提供代码完成和导航。

调试Artisan命令

Laravel命令在扩展的控制器类中定义Command 。要调试命令，至关重要的是调试命令本身，而不是调试定义在其中的控制器类文件。否则，将跳过Laravel引导过程，并且执行将失败。

  1. 在与所选命令对应的控制器类中，在要设置断点的代码行上单击编辑器装订线。

  2. 创建一个运行/调试配置 ，该配置将使用所选命令运行工匠工具。在主菜单中，选择运行|编辑配置 ，然后单击然后从列表中选择PHP脚本 。

  3. 在“ 运行/调试配置：PHP脚本”对话框中，提供运行/调试配置参数。

    • 在“ 文件”字段中，提供artisan可执行文件的路径。

    • 在“ 参数”字段中，键入实际的命令及其参数，例如view:cache 。

  4. 在PhpStorm工具栏上，选择创建的运行/调试配置，然后单击 。命令执行将在指定的断点处停止。
