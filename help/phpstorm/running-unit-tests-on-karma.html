业力

Karma是用于测试客户端JavaScript的工具。 Karma针对在真实浏览器中运行的应用程序执行测试，从而确保测试结果的正确性和可信赖性。PhpStorm与Karma集成在一起，因此您可以从IDE内部运行，调试和监视测试范围。您可以在树形视图中查看测试结果，并从那里轻松导航到测试源。测试状态显示在编辑器中测试的旁边，并带有可以快速运行或调试的选项。

在你开始前

  1. 下载并安装Node.js。

  2. 如管理插件中所述，在“ 插件”页面上安装并启用Karma存储库插件。

安装业力和插件

  • 在嵌入式终端 （ Alt+F12 ），键入以下命令之一：

    • npm install如果Karma和所有必需的插件已在package.json中定义。

    • 要将Karma和所需的插件（例如， karma-jasmine或jasmine-core ）安装为开发依赖项 ：

      npm install --save-dev业力npm install --save-dev

    从Karma官方网站上了解更多信息 。

您还可以按照npm和Yarn中的 说明 ，将karma软件包安装在Node.js和NPM页面上 。

生成业力配置文件

根据交互模式生成的karma.conf.js配置文件运行Karma测试。如果您的项目中已经有karma.conf.js ，则只需跳过此步骤。有关Karma配置的更多详细信息，请参阅Karma官方网站 。

创建业力配置文件

  1. 打开终端，并根据您的操作系统键入以下内容之一，以启动karma.conf.js生成向导：

    • 对于macOS和Linux：
      ./node_modules/karma/bin/karma初始化

    • 对于Windows：
      npm install -g karma-cli
      karma init

  2. 回答向导的问题，指定要使用的测试框架以及要自动捕获的浏览器。

    另请参阅因果文件：模式匹配 。

运行测试

使用PhpStorm，您可以直接在编辑器中快速运行单个Karma测试，也可以创建运行/调试配置来执行部分或全部测试。

从编辑器运行单个测试

  • 请点击要么在左侧装订线中，然后选择“运行” 从列表中。

    借助测试状态图标，您还可以在编辑器中直接查看测试是否通过或失败在排水沟中。

创建业力运行配置

  1. 打开“ 运行/调试配置”对话框（主菜单上的“运行” |“编辑配置 ”），单击在左侧窗格中，然后从列表中选择Karma 。将打开“ 运行/调试配置：因果”对话框。

    或者， 在“ 项目”工具窗口中选择一个测试文件，然后选择“ 创建” 从上下文菜单中。

  2. 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

    （可选）指定要传递给Node.js的特定于Node.js的选项参数和环境变量 。

  3. 指定位置karma软件包以及karma.conf.js的路径。

  4. 指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  5. （可选） 指定要传递给Karma的命令行选项，以覆盖karma.conf.js配置文件中的默认设置。

    例如，要在Headless Chrome中运行或调试测试，请输入--browsers ChromeHeadless在业力选项字段中。有关详细信息，请参见使用无头Chrome浏览器进行自动测试 。

    或者，添加'ChromeHeadless'到browsers karma.conf.js中的选项。

    要查看所有可用的CLI选项，请键入karma start --help在航站楼 Alt+F12 。

通过运行配置运行测试

  1. 从主工具栏上的列表中选择Karma运行/调试配置，然后单击在列表的右边。

  2. Karma测试服务器将自动启动，而无需您采取任何步骤。在“运行”工具窗口的“ 因果服务器”选项卡中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

重新运行失败的测试

  • 在“ 测试运行器”选项卡中 ，单击“ 在工具栏上。PhpStorm将执行在上一个会话期间失败的所有测试。

  • 要重新运行特定的失败测试，请选择“运行”。 在其上下文菜单上。

有关详细信息，请参见重新运行测试 。

导航

使用PhpStorm，您可以在文件和相关测试文件之间跳转，也可以从“ 测试运行器”选项卡中的测试结果跳转到测试。

测试文件应遵循流行的命名约定，例如，具有.test。 ， .spec。或_spec。后缀，并且应位于源文件旁边或测试文件夹中。

  • 在编辑器中打开文件，然后选择“ 转到|测试或转到|从上下文菜单中测试受试者 ，或者只需按Ctrl+Shift+T 。

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源代码 ”。测试文件将在编辑器中打开，并将光标置于测试定义上。

  • 对于失败的测试，PhpStorm从堆栈跟踪将您带到测试的失败行。如果确切的行不在堆栈跟踪中，则将转到测试定义。

调试测试

借助PhpStorm，您可以直接从编辑器快速开始调试单个Karma测试，也可以创建运行/调试配置来调试部分或全部测试。

从编辑器开始调试单个测试

  • 请点击要么在左侧装订线中，然后选择“ 调试” 从列表中。

通过运行/调试配置启动测试调试

  1. 如上所述创建Karma运行/调试配置。

  2. 从主工具栏上的列表中选择Karma运行/调试配置，然后单击在列表的右边。

  3. 在打开的“ 调试工具”窗口中，照常进行： 逐步执行测试 ， 停止并继续执行测试，在挂起时检查测试 ，依此类推。

监控代码覆盖率

有了PhpStorm， 您还可以监视Karma测试覆盖了多少代码。PhpStorm在专用工具窗口中显示此统计信息，并在编辑器中直观地标记覆盖和未覆盖的线。要监视覆盖范围，您需要安装karma-coverage软件包并更新karma.conf.js 。

安装业力覆盖

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install  --save-dev karma-coverage

要将业力覆盖定义添加到配置文件

  1. 在编辑器中打开karma.conf.js 。

  2. 找到reporters定义并添加coverage格式的值列表：

    记者：['进展'，'报道']

  3. 添加一个preprocessors定义并以以下格式指定覆盖范围：

    预处理器：{'** / *。js'：['coverage']}

启动覆盖测试

  1. 如上所述创建Karma运行/调试配置。

  2. 从主工具栏上的列表中选择Karma运行/调试配置，然后单击在列表的右边。

    或者，使用编辑器中的测试图标快速运行特定套件或具有覆盖率的测试。

  3. 在“ 覆盖率”工具窗口中监视代码覆盖率。

    每次运行Karma测试时，实际上都会在磁盘上生成一份覆盖率报告。覆盖率报告的格式可以在配置文件中配置，例如：

    // karma.conf.js module.exports = function（config）{config.set（{...//（可选）配置报告程序coverageReporter：{类型：'html'，目录：'coverage /'} ...}）;};

    下列type可接受的值：

    • html产生一堆带有注释源代码的HTML文件。

    • lcovonly产生一个lcov.info文件。

    • lcov生成HTML + .lcov文件。默认情况下会应用此格式。

    • cobertura生成cobertura-coverage.xml文件，以方便Hudson集成。

    • text-summary生成覆盖范围的紧凑文本摘要，通常覆盖控制台。

    • text产生一个详细的文本表，覆盖所有文件。
