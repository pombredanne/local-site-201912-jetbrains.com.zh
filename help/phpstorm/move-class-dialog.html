当您将插入号放在编辑器中类的名称并调用重构时，对话框将打开。

PhpStorm假定项目中的名称空间的排列符合PSR0标准，并在您在名称空间之间移动类时强制您根据此标准保留文件夹结构和名称空间层次结构。

当您指定将类移动到的名称空间时，PhpStorm会自动更新“ 目标目标目录”字段，该字段显示将存储实现该类的文件的文件夹的路径。

将类移到名称空间

对话框打开时，此字段显示该类所属的名称空间。指定目标名称空间的标准名称。 根据您的用例，可以是：

  • 当前命名空间下的新命名空间，格式为\ 。

  • 当前父项下的另一个从属命名空间，格式为\ 。

  • 另一个父命名空间下的现有或新命名空间，格式为\ 。

仅使用反斜杠 \作为名称空间分隔符。

目标目的地目录

对话框打开时，该字段显示当前存储用于实现要移动的类的文件的文件夹的路径。

路径以以下格式显示：

...\
当您指定将类移动到的名称空间时，路径会自动更新。 但是，如果要将类移动到另一个父名称空间下的不存在的名称空间，除非您通过在目录页面上将相关文件夹标记为Sources来为名称空间结构指定根文件夹，否则PhpStorm将不会建议正确的文件夹。 的 设置/首选项对话框Ctrl+Alt+S 。有关详细信息，请参见目录 。

请执行以下任一操作：

  • 接受显示在字段中的预选路径。

  • 从列表中选择另一个路径。它们都是从名称空间根目录或当前目录中评估的，因此可以安全地选择它们。

  • 请点击然后在打开的对话框中选择一个文件夹。

  • 按N/A并编辑预选路径。请记住，这将来可能会导致自动加载出现问题。

搜索注释和字符串

如果选中此复选框，则PhpStorm将在源代码文件的注释和字符串文字中查找要移动的类的引用。

搜索文字出现

如果选中此复选框，PhpStorm将在文档，HTML，JSP，各种配置文件（例如.yml ）和项目中包含的其他文件中查找要移动的类的引用。
