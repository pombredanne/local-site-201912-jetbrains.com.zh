搜索模板，过滤器和脚本约束

当您为结构化搜索构建模式时，您基本上是在编写脚本。为了简化脚本编写过程，PhpStorm为您提供了一系列预定义的搜索模板 ，您可以将它们用作搜索模式的原型。

有关如何访问现有搜索和替换模板的列表，请参见结构性搜索和替换部分。

有效的搜索或替换模板代表以下受支持的语言构造之一：

  • 表达式，例如"John" . " " . "Doe"

  • 语句或语句序列，例如$people[] = new Person('Trillian');

  • 例如，类指示符class Engine implements IEngine

  • 行或块注释，例如/** Created in PhpStorm */ 。

在搜索模板中，可以使用以下简化：

  • 方法主体可以省略。

  • 在模板和约束字段中使用短类名（而不是标准名称）。

  • 使用class $Class$作为模板也会导致查找匿名类。

  • 注释和文档注释的模板应包含具有正确注释和PHPDoc / JSDoc语法的变量和构造。

每个搜索或替换模板均包含变量$variable_name$您可以向其添加条件（过滤器）以缩小搜索结果的范围。过滤器取决于搜索模板中的变量。

计数过滤器

计数过滤器指定出现的次数。

例如，在class $a$ {public function $b$()}搜索模式$b$变量在“ 计数”过滤器字段中指定最小和最大数字。要设置无限的最大计数，请在过滤器字段中提供一个空值。

PhpStorm添加[0,∞]到变量并搜索指定的数字范围。

参考过滤器

通过引用过滤器，您可以引用变量中的其他一些搜索模板。

引用将始终包含预配置或保存的模式的名称，您可以使用自动完成功能来填写此字段。

例如，您可以定义一个搜索模板来定位类的静态方法，然后在另一个模板中引用它以检测对此类静态方法的错误实例调用。

类型过滤器

“ 类型”过滤器添加了对指定变量期望的值或表达式的类型。

使用以下过滤器值来找到相应类型的变量：

  • string

  • int要么integer

  • double要么float

  • boolean
  • array
  • null

要查找特定类的实例，请提供其FQN（包括前导） \作为类型过滤器值。

例如，对于$b$变量，类型\ExampleClass在类型过滤器字段中。

PhpStorm搜索ExampleClass实例化。

文字过滤器

文本过滤器根据正则表达式或纯文本检查变量。

例如，要检测显式魔术方法调用，请使用$a$->$b$()模板和设置的文本过滤器的$b$变量^__.+$正则表达式。

脚本约束

脚本过滤器将Groovy脚本约束添加到搜索模式。当您搜索某些语言结构时，将使用脚本约束 。

例如，具有指定数量的参数的构造函数，或具有指定的可见性修饰符的成员。

可以从脚本约束中访问模式中使用的所有变量。当向变量添加脚本约束时，PhpStorm会将其与PSI树进行匹配，该变量实际上是PSI树中的一个节点 。

假设您有一个与方法匹配的变量， toString()方法。那么这个变量实际上是PsiMethod节点。正在检索variable.parent会产生一个PsiClass节点，依此类推。 variable.text然后将为您提供方法的整个文本。如果只需要方法的名称，则可以使用variable.name 。

在另一种情况下，结构搜索和替换变量可以匹配某些表达式，例如，对变量的引用， PsiReferenceExpression 。表达式当然没有名称，但是检索表达式的整个文本将为您提供它所引用的变量的名称。

您可以检查以下现有模板中使用的脚本约束的语法：

  • 具有常量参数的样本方法调用

  • 类

  • 具有无参数构造函数的类

  • 不是最终的静态字段

  • 未实现或扩展的接口

  • 读取的字段/变量

  • 具有给定名称模式的字段/变量已更新

__context__对应于完全匹配变量。
