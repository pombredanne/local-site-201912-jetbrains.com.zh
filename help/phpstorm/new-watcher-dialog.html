PhpStorm |首选项|工具| File Watchers-适用于macOS的新Watcher
Ctrl+Alt+S

单击添加时，对话框打开或编辑 在“ 文件监视程序”页面上 。使用该对话框可基于预定义的PhpStorm File Watcher模板创建项目File Watcher或编辑现有项目File Watcher。

每个模板均包含最适合所选工具的设置。因此，在大多数情况下，您只需要指定工具可执行文件的路径即可。

名称

在名称字段中，输入文件监视程序的名称。默认情况下，PhpStorm建议使用选定的预定义模板的名称。

要观看的文件

文件类型

使用此列表可以指定输入文件的预期类型。 File Watcher仅将此类文件视为分析和处理的主题。基于文件类型和文件扩展名之间的关联来识别文件类型 。

默认情况下，该字段显示根据所选预定义模板的文件类型。

范围

使用此列表定义可以将File Watcher应用于的文件范围。

这些文件中的更改将立即或在保存或取消激活框架后调用File Watcher，具体取决于自动保存已编辑文件的状态以触发 Watcher复选框。

从列表中选择一个预定义的范围。对于项目级别的File Watcher，也可以单击在打开的“ 范围”对话框中配置自定义范围。

  • 所有地点：下面列出的所有范围。

  • 项目文件：项目内容根目录中的所有文件（请参阅内容根目录和配置内容根目录 ）。

  • 项目生产文件：项目内容根中的所有文件（测试源除外）。

  • 项目测试文件：项目测试源根目录中的所有文件。

  • 临时和控制台：位于“ 项目”工具窗口中的临时和控制台目录中的所有文件。

  • 打开文件：编辑器中当前打开的所有文件。

  • 当前文件：在活动编辑器选项卡中打开的文件。

VCS范围：仅当您的项目处于版本控制下时，这些范围才可用。

从列表中选择一个预定义的范围。对于项目级别的File Watcher，也可以单击在打开的“ 范围”对话框中配置自定义范围。

  • 所有地点：下面列出的所有范围。

  • 项目文件：项目内容根目录中的所有文件（请参阅内容根目录和配置内容根目录 ）。

  • 项目生产文件：项目内容根中的所有文件（测试源除外）。

  • 项目测试文件：项目测试源根目录中的所有文件。

  • 临时和控制台：位于“ 项目”工具窗口中的临时和控制台目录中的所有文件。

  • 打开文件：编辑器中当前打开的所有文件。

  • 当前文件：在活动编辑器选项卡中打开的文件。

或者，单击并在打开的“ 范围”对话框中配置自定义范围 。

有关详细信息，请参见范围 。

仅跟踪根文件

 根文件是不包含的文件（例如，通过import ）在指定范围内的任何其他文件中 。

  • 选中此复选框后，“文件监视程序”仅针对根文件运行。

  • 清除复选框后，文件监视程序将在调用文件监视程序的文件以及指定范围内递归包含此文件的所有文件中运行。

请注意，“ 作用域”设置将覆盖“ 仅跟踪根文件”复选框设置：如果依赖项在指定作用域之外，则不会对其应用文件监视程序。

例
假设您有两个文件root.scss和another.scss ，其中root.scss导入another.scss 。选中仅跟踪根文件复选框，编辑another.scss仅在root.scss上调用文件监视程序 。否则，如果清除该复选框，则文件监视程序将处理这两个文件。

此选项仅适用于Babel ， Closure Compiler ， Compass ， Jade ， Less ， Sass / SCSS ， Stylus ， UglifyJS和YUI Compressor JS 。

运行变更的工具

在此区域中，配置与工具的交互：指定要使用的可执行文件，要传递给它的参数以及PhpStorm在何处查找工具的输出。

程序

在此字段中，指定工具可执行文件的路径（ .com ， .exe ， .cmd ， .bat或其他取决于特定工具的路径）。

如果要为项目中安装的工具（例如Prettier ）配置全局文件监视程序，则需要使用宏指定其路径。要在路径中使用可用的宏之一，请按“ 插入宏”按钮。例如，使用路径$ ProjectFileDir $ / node_modules / .bin / prettier ，文件监视程序将使用安装在项目的node_module文件夹中的漂亮文件。

  • 要使用程序文件名而不是文件名路径，请将其文件夹的路径添加到系统环境变量PATH。

  • 在Windows上，您可以跳过.com ， .exe ， .cmd或.bat扩展名。

  • 要使用jar档案，请指定其绝对路径。或者，要使用相对路径，请将其父文件夹添加到以下位置的PhpStorm路径变量中： 外观与行为| PhpStorm设置的“ 路径变量”页面Ctrl+Alt+S 。

争论

在此字段中，定义要传递给工具的参数，从而影响其行为。

通常使用宏来指定参数，例如， $FileName$要么$FileNameWithoutExtension$ ，将替换为实际的文件名。

手动键入宏，或单击“ 插入宏”，然后从打开的“ 宏”对话框中的列表中选择相关的模式。

指定参数时，请遵循以下规则：

  • 使用空格作为分隔符。

  • 如果参数包含空格，请将其或整个参数括在双引号中： some" "arg要么"some arg" 。

  • 如果参数包含双引号，请使用反斜杠将其转义： -Dmy.prop=\"quoted_value"\ 。

输出路径进行刷新

在此字段中，告诉PhpStorm应该在哪里搜索工具的输出：结果源代码，源映射，依赖项或文件本身。输出的位置是特定于工具的。

通常使用宏指定输出路径。您可以手动键入它们，或单击“ 插入宏”，然后从打开的对话框的列表中选择相关的宏 。

请注意，更改“ 输出路径以刷新”字段中的值不会使该工具将其输出存储在其他位置。如果仍然需要这样做，请在“ 参数”字段中指定所需的自定义输出位置：键入以冒号作为分隔符的输出路径，并使用宏。

工作目录和环境变量

工作目录

在此字段中，指定将应用工具的目录。 因为该工具始终在文件的上下文中调用，所以默认的工作目录是当前文件的目录。在所有预定义模板中，默认的工作目录通过$FileDir$宏。要指定自定义工作目录，请在字段中键入其路径，或单击并在“ 选择路径”对话框中选择目录，或单击“ 插入宏” ，然后从“宏”对话框的列表中选择所需的宏。

如果将“ 工作目录”字段保留为空，则PhpStorm将使用调用文件监视程序的文件目录。

环境变量

在此字段中，指定该工具需要但在操作系统级别未指定的环境变量。

高级选项

自动保存编辑的文件以触发观察程序

  • 选中此复选框后，PhpStorm会在您编辑文件后立即保存文件，因此File Watcher会立即唤醒。

  • 清除复选框后，文件监视程序将在保存时（ 文件|全部保存 ）或将焦点从PhpStorm中移出（在禁用帧时）启动。

触发观察者进行外部更改

  • 选中此复选框后，文件监视程序会从其作用域唤醒文件，包括在PhpStorm外部所做的更改或在版本控制系统中签出分支后所做的更改。

  • 清除复选框后，仅当从PhpStorm更新其作用域中的文件时，文件监视程序才会启动，并且忽略其他更改，例如从版本控制系统收到的更改。

触发器观察器，无论语法错误如何

  • 选中该复选框后， 无论文件的语法正确性如何，File Watcher都会启动。File Watcher将在更新，保存或禁用框架后启动，具体取决于自动保存已编辑文件以触发 Watcher复选框的状态。

  • 清除此复选框后， File Watcher会忽略语法上无效的文件中的所有触发器，并且仅在无错误的文件中启动。

从标准输出创建输出文件

  • 在大多数情况下，应清除此复选框，因为大多数工具会将其输出直接写入相关文件。

  • 选中此复选框后，PhpStorm会读取本机工具的输出standard output stream (stdout)并从中生成结果文件。生成的文件的名称取自“ 要刷新的输出路径”字段。如果该字段包含多个名称，则使用列表中的第一个名称。

显示控制台

从此列表中，选择希望文件监视程序打开控制台的时间。

  • 始终：使用此选项，控制台始终在工具执行完成后打开。

  • 错误时：使用此选项时，仅在以下情况下控制台在工具执行后打开Exit code与...不同0 。

  • 永不：选择此选项可完全禁止打开控制台。

输出滤波器

在此字段中，指定与工具关联的输出过滤器。 基于这些过滤器，工具输出中的绝对文件路径和行号将转换为超链接。单击这些链接可在编辑器中打开相应的文件。
例如，要显示有用的错误消息，请键入$FILE_PATH$:$LINE$ $MESSAGE$
