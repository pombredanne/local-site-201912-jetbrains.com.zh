教程：移动PHP类

在本教程中，我们将探索如何使用Move和Copy重构在项目中的命名空间之间移动PHP类。

将类移动到当前名称空间下的新名称空间

假设你move_class项目有namespace1命名空间，带有Class1类在Class1.php文件中实现。

让我们创建一个下级命名空间namespace1\ns1并移动Class1对它。

  1. 在编辑器中打开Class1.php ，将插入号放置在Class1 ，然后按F6 。将打开“ 移动类”对话框。

  2. 将类Class1移至名称空间字段显示当前名称空间namespace1 。更改为namespace1\ns1 。

    键入时，“ 目标目标目录”字段将自动从...\move_class\namespace1至...\move_class\namespace1\ns1 。

  3. 按重构 。结果，PhpStorm创建了ns1子文件夹，并将Class1.php移至该文件夹。因此， Class1班级移至namespace1\ns1 ：

在同一个父项下的两个名称空间之间移动类

假设你move_class项目有namespace1具有两个从属命名空间的命名空间： namespace1\ns1和namespace1\ns3 。这些从属名称空间具有Class1和Class3分别。根据PSR0标准 ，这些类分别在存储在... \ move_class \ namespace1 \ ns1和... \ move_class \ namespace1 \ ns3下的Class1.php和Class3.php文件中实现。

让我们继续前进Class3从namespace1\ns3至namespace1\ns1 。

  1. 在编辑器中打开Class3.php ，将插入号放置在Class3 ，然后按F6 。将打开“ 移动类”对话框。

  2. 将类Class3移至名称空间字段显示当前名称空间namespace1\ns3 。更改为namespace1\ns1 。

    键入时，“ 目标目标目录”字段将自动从...\move_class\namespace1\ns3至...\move_class\namespace1\ns1 。

  3. 按重构 。结果，PhpStorm将Class3来自的班级namespace1\ns3至namespace1\ns1 。相应地， Class3.php文件从... \ move_class \ namespace1 \ ns3文件夹移动到... \ move_class \ namespace1 \ ns1文件夹：

将类移动到另一个父名称空间下的现有名称空间

假设你move_class项目有两个父名称空间， namespace1和namespace2 ，具有从属名称空间namespace1\ns1和namespace2\ns2 。的Class2类定义在namespace2\ns2 ，并按照PSR0标准 ，实现它的Class2.php文件存储在... \ move_class \ namespace2 \ ns2中 。

让我们继续前进Class2从namespace2\ns2至namespace1\ns1 。

  1. 在编辑器中打开Class2.php ，将插入符号放置在Class2 ，然后按F6 。将打开“ 移动类”对话框。

  2. 将类Class2移至名称空间字段显示当前名称空间namespace2\ns2 。更改为namespace1\ns1 。

    键入时，“ 目标目标目录”字段将自动从...\move_class\namespace2\ns2至...\move_class\namespace1\ns1 。

  3. 按重构 。结果，PhpStorm将Class2来自的班级namespace2\ns2至namespace1\ns1 。因此， Class2.php从... \ move_class \ namespace2 \ ns2移到... \ move_class \ namespace1 \ ns1 ：

将类移动到直接父命名空间之外的新命名空间

假设你move_class项目有两个父名称空间， namespace1和namespace2 ，带有下级名称空间， namespace1\ns1和namespace2\ns2 。 Class2在中定义namespace2\ns2 ，并根据PSR0标准将实现该文件的Class2.php文件存储在... \ move_class \ namespace2 \ ns2中 。

让我们继续前进Class2从namespace2\ns2到新的名称空间namespace3那还不存在。为了满足PSR0要求，为...创建了一个新的... \ move_class \ namespace3文件夹。 namespace3还应该创建。

在开始重构时，我们应该记住，要在我们类的名称空间的直接父名称空间之外创建新名称空间，因此PhpStorm将不会为它建议合适的文件夹。因此，我们可以手动指定目标文件夹，也可以为命名空间指定一个根文件夹，这将是PhpStorm为新命名空间计算文件夹的起点。

  1. 在编辑器中打开Class2.php ，将插入符号放置在Class2 ，然后按F6 。将打开“ 移动类”对话框。

  2. 将类Class2移到名称空间字段显示当前namespace2\ns2命名空间。更改为namespace3 。结果取决于您是否为命名空间指定了根文件夹。

    • 如果未指定根文件夹，则“ 目标目标目录”字段仍显示到... \ move_class \ namespace2 \ ns2的路径，因为PhpStorm无法确定新命名空间的文件夹路径namespace3被创建：

      要解决该问题，请按N/A并手动输入路径：

      ... \ move_class \ namespace2 \ ns2

    • 或者，通过将... \ move_class \文件夹标记为Sources来指定名称空间的根文件夹，如标记目录中所述 。请注意，这应该在开始重构之前完成，因此，如果您已经打开了“ 移动类”对话框，请首先将其关闭。

      在将... \ move_class \指定为名称空间的根目录之后，PhpStorm将确定新名称空间的文件夹并自动更新“ 目标目标目录”字段：

    • 按重构 。结果，PhpStorm移动了Class2从namespace2\ns2至namespace3 。因此， Class2.php从... \ move_class \ namespace2 \ ns2移到... \ move_class \ namespace3 ：
