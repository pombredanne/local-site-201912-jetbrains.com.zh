PhpStorm 2019.3帮助

从现有本地资源创建项目

当要使用的文件已经在您的计算机上时，您可以通过三种主要方式来设置开发环境：

  • 您的计算机上没有Web服务器，或者现在不打算使用它。源代码在您的计算机上，您只需进行开发即可，可能使用控制台查看输出。在这种情况下，您可以直接打开存储源的文件夹，而无需配置对服务器的访问。但是，您也可以使用“ 新建项目”向导。

  • 您的计算机上有一台Web服务器，源位于服务器文档根目录下（例如/ htdocs ），然后直接在服务器上进行开发。 在这种情况下，您需要使用“ 新建项目”向导来安排项目中的源，指定项目根文件夹 ，并通过就地类型的服务器访问配置来配置对服务器的访问 。

  • Web服务器已安装在您的计算机上，但是源存储在另一个文件夹或已安装的驱动器中。您进行开发，然后与服务器同步。使用“ 新建项目”向导来安排项目中的源，指定项目根文件夹 ，通过本地类型的服务器访问配置来配置对服务器的访问 ，并将项目根文件夹下的文件夹与服务器上的文件夹进行映射。

通过打开存储源的文件夹来创建项目

如果您现在不打算使用Web服务器，则只需打开包含要在PhpStorm项目中排列的文件的文件夹。

  1. 单击“ 欢迎”屏幕上的“ 打开 ”，或选择“ 文件” |“共享”。从主菜单打开 。

  2. 在打开的对话框中 ，选择包含所需源代码的目录。请注意，外部创建的应用程序带有常规目录图标标记 。

  3. 单击确定，然后指定是要在单独的窗口中打开新项目还是要关闭当前项目并重用现有项目。有关详细信息，请参见打开多个项目 。

使用新项目向导在没有服务器的情况下围绕现有源创建项目

在PhpStorm项目中安排源代码的另一种方法是使用“ 新建项目”向导。

  1. 要启动“ 新建项目”向导，请在主菜单上选择“从现有文件中新建项目” 。

  2. 在“ 选择方案”页面上 ，选择“ 源文件在本地目录中，尚未配置Web服务器” ，然后单击“ 下一步” 。

  3. 在“ 选择项目目录”页面上 ，选择存储源的文件夹，然后单击工具栏上的图标，或从选择的上下文菜单中选择“ 项目根” ，以指定文件夹项目根 。PhpStorm用图标。

  4. 配置项目根目录下的文件夹 。或者，您可以稍后在“ 设置”对话框的“ 目录”页面上执行此操作。

  5. 单击完成，然后指定是要在单独的窗口中打开新项目还是要关闭当前项目并重用现有项目。有关详细信息，请参阅“ 打开多个项目 ”部分。

从服务器文档根目录下的现有源创建项目

在这种情况下，您需要使用“ 新建项目”向导来安排项目中的源，指定项目根文件夹 ，并通过就地类型的服务器访问配置来配置对服务器的访问 。在这种情况下，您需要使用“ 新建项目”向导来安排项目中的源，指定项目根文件夹并配置对服务器的访问。

  1. 要启动“ 新建项目”向导，请在主菜单上选择“从现有文件中新建项目” 。

  2. 在“ 选择方案”页面上 ，选择“ Web服务器已本地安装”，源文件位于其文档根目录下 ，然后单击“ 下一步” 。

  3. 在“ 选择项目目录”页面上 ，选择存储源的文件夹，然后单击工具栏上的图标，或从选择的上下文菜单中选择“ 项目根” ，以指定文件夹项目根 。PhpStorm用图标。

  4. 配置项目根目录下的文件夹 。或者，您可以稍后在“ 设置”对话框的“ 目录”页面上执行此操作。单击下一步 。

  5. 在“ 指定本地服务器”页面上 ，指定要使用的服务器访问配置 。有关更多详细信息，请参阅配置与Web服务器的同步 。

    请执行以下任一操作：

    • 选择“ 使用现有服务器”，然后从列表中选择相关的服务器配置。该列表仅包含就地类型标记为的配置 ，因为您的文件已经在服务器文档根目录下。

    • 选择添加新本地服务器 ，单击下一步 ，然后在打开的添加本地服务器页面上定义新的服务器访问配置：

      1. 在“ 名称”文本框中指定服务器配置名称。PhpStorm本身将配置类型设置为In-place 。

      2. 在“ Web服务器URL”字段中，键入与Web服务器配置文件中定义的Web服务器的文档根目录关联的URL地址。该URL地址将是构建应用程序URL地址的起点。

        HTTP和HTTPS协议均受支持。

        例如，默认文档根目录设置为htdocs文件夹，用于访问其中数据的默认URL地址为http：// localhost 。如果更改了默认端口80 ，则必须明确指定端口： http：// localhost： 。

        准备好后，单击“ 下一步” 。

      3. 在“ 指定Web路径”页面上 ，指定URL地址以访问您的项目根文件夹。在“ 项目根目录的Web路径”文本框中，键入相对于服务器配置文件中指定的服务器文档根目录的项目根文件夹的路径。键入时，PhpStorm将动态构建URL地址，并在“ 项目URL”只读字段中显示为链接。要检查URL地址是否正确构建并确保对项目根目录的访问，请单击链接。

      有关更多详细信息，请参阅创建就地服务器配置 。

  6. 单击完成，然后指定是要在单独的窗口中打开新项目还是要关闭当前项目并重用现有项目。有关详细信息，请参阅“ 打开多个项目 ”部分。

从服务器文档根目录之外的源创建项目

Web服务器已安装在您的计算机上，但是源存储在另一个文件夹或已安装的驱动器中。您进行开发，然后与服务器同步。在这种情况下，您需要使用“ 新建项目”向导来安排项目中的源，指定项目根文件夹并配置对服务器的访问。

  1. 要启动“ 新建项目”向导，请在主菜单上选择“从现有文件中新建项目” 。

  2. 在选择方案页面上 ，选择Web服务器已本地安装，源文件位于本地其他位置 ，然后单击下一步 。

  3. 在“ 指定本地路径”页面上 ，指定项目根目录 。指定用于存储源以及将在其中创建项目的文件夹。

  4. 在“ 指定服务器”页面上，指定服务器访问配置以启用与服务器的交互。有关更多详细信息，请参阅配置与Web服务器的同步 。

    请执行以下任一操作：

    • 选择“ 使用现有服务器”，然后从列表中选择相关的服务器配置。该列表包含本地标志着类型的唯一配置 ，因为文件存储在本地计算机上，但是在服务器文档根目录下，因此就地类型的配置不适合。

    • 选择添加新本地服务器 ，单击下一步 ，然后在打开的“ 添加服务器”页面上定义新的服务器访问配置。

      1. 在“ 名称”文本框中指定服务器配置名称。PhpStorm本身将配置类型设置为Local 。

      2. 在“上载/下载项目文件”区域的“ 文件夹”文本框中， 指定服务器配置根目录 。

        服务器配置根目录是服务器上文件树中可通过服务器配置访问的最高文件夹。最简单的方法是使用Web服务器配置文件中定义的Web服务器的文档根目录。但是，您可以在文档根目录下指定任何其他现有文件夹。

      3. 在“ 浏览服务器上的文件”区域的“ Web服务器URL”文本框中， 指定服务器配置根目录的URL地址。该URL地址将是构建应用程序URL地址的起点。根据您对服务器配置根目录的选择，执行下列操作之一：

        • 键入与Web服务器配置文件中定义的与Web服务器的文档根目录关联的URL地址。

        • 以以下格式输入网址/ 。

        单击下一步 。
      有关更多详细信息和示例，请参阅创建本地服务器配置 。
  5. 在“ 选择远程路径”页面上，在服务器上创建一个文件夹，PhpStorm将在该文件夹上上传项目文件。该文件夹在服务器上称为项目根目录 ，它将自动映射到您的项目根目录 。

    该页面显示指定的服务器配置根目录下的文件夹树。选择相关的父文件夹 ，然后在上下文菜单上选择“ 创建文件夹 ”。当新文件夹出现在树中时，选择它并单击工具栏上的图标或在所选内容的上下文菜单上选择“ 项目根 ”。PhpStorm用图标。

    配置项目根目录下的文件夹 。或者，您可以稍后在“ 设置”对话框的“ 目录”页面上执行此操作。单击下一步 。

  6. 在“ 指定Web路径”页面上 ，指定URL地址以访问上一步中指定的服务器上的项目根文件夹 。在“ 项目根目录的Web路径”文本框中，键入相对于服务器配置root的文件夹的路径。实际上， 在服务器文本框中键入项目根目录的相对路径。键入时，PhpStorm将动态构建URL地址，并将其显示为“ 项目URL”只读字段中的链接。

在项目根目录下配置文件夹

PhpStorm可以区分源文件夹， 资源根文件夹和排除文件夹。默认情况下，任何文件夹都被视为源文件夹，除非它与另一种类型相关联。

您可能还需要禁止上传/下载文件和文件夹。 禁止使用源对文件或文件夹进行上载，下载和同步，以确保源得到保护以防止意外更新。应用于非源时，它可以节省系统资源，因为不再需要不必要地复制媒体，缓存或临时文件。

选择要标记的目录，然后执行以下操作之一：

  • 若要使PhpStorm在建立索引，解析，代码完成等过程中忽略所选目录，请单击“ 从索引中排除”工具栏按钮或在所选内容的上下文菜单上选择“ 排除 ”。

  • 要使PhpStorm能够完成所选文件夹下资源的相对路径，请单击“ 资源根”工具栏按钮或在所选内容的上下文菜单上选择“ 资源根 ”。

  • 要禁止所选文件夹的上载，下载和同步，请单击“ 从下载中排除”图标或在所选内容的上下文菜单上选择“ 排除路径 ”。

有关详细信息，请参阅在内容根目录下配置文件夹 和从上载/下载中排除文件和文件夹 。
