创建远程服务器配置

远程服务器是在另一台计算机（远程主机）上运行的服务器。可通过FTP / SFTP / FTPS协议访问服务器上的文件。

要在此设置中配置对服务器的访问，需要指定以下内容：

  1. 连接设置：服务器主机，端口和用户凭据。

  2. 服务器配置根文件夹和访问它的URL地址。

  3. 项目根文件夹 ，将数据从项目根文件夹复制到的服务器上的文件夹以及访问服务器上复制的数据的URL地址之间的对应关系。这种对应称为映射 。

指定服务器配置的名称，类型和可见性

  1. 通过执行以下操作之一打开“ 部署”页面：

    • 在里面 设置/首选项对话框Ctrl+Alt+S选择部署下建立，执行，部署 。

    • 选择工具|部署|从主菜单进行配置 。

  2. 在列出所有现有服务器配置的左侧窗格中，单击“ 添加”。 然后根据您要用来与服务器交换数据的协议选择服务器配置类型。

    • FTP：选择此选项可使PhpStorm通过FTP 文件传输协议访问服务器。

    • SFTP：选择此选项可使PhpStorm通过SFTP文件传输协议访问服务器。

    • FTPS：选择此选项可使PhpStorm通过SSL上的FTP文件传输协议（ FTPS扩展名）访问服务器。

  3. 在打开的“ 创建新服务器”对话框中，键入要创建的服务器的名称，然后单击“ 确定” 。关闭“ 创建新服务器”对话框，然后返回到“ 部署”节点的“ 连接”选项卡。

  4. 单击用作默认值 在以下情况下，使PhpStorm静默应用当前配置：

    • 自动上传更改的文件 。

    • 手动上传和下载文件，而无需选择目标主机。

    • 将本地文件和文件夹与其远程版本进行比较。

    • 编辑远程主机上的单个文件 。

  5. 仅在此项目中使用可见复选框配置服务器访问配置的可见性。

    • 选中复选框以将配置的使用限制为当前项目。这样的配置不能在当前项目外部重用。它不会出现在其他项目的可用配置列表中。 例如，如果在SFTP配置中选中此复选框，则不能使用其SSH凭据来配置远程解释器。

    • 清除该复选框后，该配置在所有PhpStorm项目中均可见。它的设置（包括SFTP服务器的SSH凭据）可以在多个项目中重复使用。

    有关设置解释器的详细信息，请参阅配置远程PHP解释器 ， 通过SFTP的 Node.js和通过SSH的Node.js。

指定在主机上注册期间定义的用户凭证

  1. 指定注册方式：

    • 要以常规模式登录，请在“ 用户名”字段中指定登录名 。

    • 要启用以您的电子邮件地址作为密码对服务器的匿名访问 ，请选中“ 以匿名登录”复选框。

  2. 指定验证服务器的方式：

    • 对于FTP服务器 ，输入密码，然后选择保存密码复选框，以使PhpStorm记住它。

    • 对于SFTP服务器 ，选择对服务器进行身份验证的方式。请执行以下任一操作：

      • 密码 ：使用密码访问主机。指定密码，然后选择“ 保存密码”复选框以将密码保存在PhpStorm中。

      • 密钥对（OpenSSH或PuTTY） ：对密钥对使用SSH身份验证 。 要应用此身份验证方法，您必须在客户端计算机上具有私钥，而在远程服务器上必须具有公钥。PhpStorm支持使用OpenSSH实用程序生成的私钥。

        指定存储私钥的文件的路径，然后在相应的字段中键入密码（如果有）。要使PhpStorm记住该密码短语，请选中“ 保存密码短语”复选框。

      • OpenSSH配置和身份验证代理 ：使用由凭证帮助程序（例如Windows上的Pageant或Mac和Linux上的ssh-agent ）管理的SSH密钥。

      有关使用SSH密钥的详细信息，请参见生成新的SSH密钥并将其添加到ssh-agent教程。

    • 对于FTPS服务器 ，请指定您的用户名和密码。在“ 高级设置”区域中，选择要应用的安全性机制。

      • 选择“ 显式”以应用显式（活动）安全性 。建立连接后，计算机上的FTP客户端会立即向服务器发送命令，以通过默认FTP端口建立安全控制连接。

        默认情况下选择此方法。

      • 选择“ 隐式”以应用隐式（被动）安全性 。在这种情况下，建立与服务器的连接后会自动提供安全性，该服务器指定了用于安全连接的单独端口。

        请注意，不建议使用此方法，不建议使用此方法。

启用与服务器的连接并指定服务器配置根目录

  1. 指定要与其交换数据的FTP / SFTP / FTPS服务器的主机名以及该服务器侦听的端口。

    默认值为：

    • FTP和FTPS为21

    • SFTP为22

  2. 在“ 根路径”字段中， 指定相对于服务器上根文件夹的服务器配置根 。该文件夹将是可通过当前服务器配置访问的文件夹结构中的最高文件夹。

    请执行以下任一操作：

    • 接受默认的/路径，该路径指向服务器上的根文件夹。

    • 手动输入路径。

    • 请点击然后在打开的“选择根路径”对话框中选择所需的文件夹。

    • 单击“ 自动检测”按钮，使PhpStorm检测FTP / SFTP服务器上的用户主文件夹设置，并根据它们设置根路径 。仅在指定凭据后才启用该按钮。

  3. 在“ Web服务器URL”字段中，键入URL地址以访问服务器配置根目录。 服务器配置根目录是通过服务器配置可访问的本地或远程服务器上文件树中的最高文件夹。对于就地服务器，它是项目根。

    HTTP和HTTPS协议均受支持。

  4. 请点击在字段中，以确保可以访问指定的服务器根URL地址并指向正确的网页。

将本地文件夹映射到服务器上的文件夹和URL地址以访问它们

配置映射 ，即设置项目文件夹，服务器上用于将项目文件复制到的文件夹和URL地址之间的对应关系，以访问服务器上复制的数据。最简单的方法是将整个项目根文件夹映射到服务器上的文件夹。如果在“ 选项”对话框中选择了“ 创建空目录”复选框，则在服务器上将重复这种情况下的项目文件夹结构。有关更多详细信息，请参阅配置与Web服务器的同步和自定义上载/下载 。

  1. 切换到“ 映射”选项卡。

  2. 在本地路径字段中， 指定所需本地文件夹的完整路径。在最简单的情况下，它是项目的根目录。

  3. 在“ 部署路径”字段中， 在服务器上指定PhpStorm将在“ 本地路径”字段中指定的文件夹中上传数据的文件夹。输入相对于服务器配置根目录的文件夹路径。

    如果指定名称的文件夹尚不存在，则只要您在“ 选项”对话框中选择了“ 创建空目录”复选框，PhpStorm便会创建该文件夹。有关更多详细信息，请参见自定义上传/下载 。

  4. 在“ Web路径”字段中， 键入相对于服务器配置根目录的服务器上文件夹的路径。实际上，在“ 部署路径”字段中键入您输入的相对路径。

通过配置嵌套映射来重载部署目标

您可以为项目根目录下的特定文件夹配置单独的映射，以使该文件夹的内容与远程主机上的另一个位置同步。

假设您已按照以下方式配置了映射：

本地路径

部署路径

ftp：//.../htdocs/my_project
/ my_folder ftp：//.../htdocs/my_folder
然后，您的项目中的文件将按以下方式上载：

本地路径

部署路径

/file1.php ftp：//.../htdocs/my_project/file1.php
/my_folder/file2.php

ftp：//.../htdocs/my_folder/file2.php
而不是ftp：//.../htdocs/my_project/my_folder/file2.php
