PhpStorm 2019.3帮助

Symfony

PhpStorm为使用Symfony框架开发应用程序提供了编码辅助和导航功能。

Symfony 插件通过Symfony插件提供支持。插件的源代码及其问题跟踪器可在GitHub上找到 。

在你开始前

在开始使用Symfony之前，请确保已安装并启用了Symfony和PHP Annotations插件。

为项目启用Symfony插件

安装Symfony插件后，您需要为项目明确启用它。创建或打开Symfony项目后，PhpStorm将显示一条通知消息，提示您启用该插件。

为项目启用Symfony插件

  1. 请执行以下任一操作：

    • 单击通知消息中的使用自动配置启用Symfony插件链接。

    • 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP | Symfony，然后选中“ 为此项目启用插件”复选框。

  2. 重新启动PhpStorm，以使更改生效。

指定其他设置

根据Symfony项目的结构，您可能必须在Languages＆Frameworks |下更改一些其他设置。 PHP | Symfony 。当使用Symfony应用程序时，该应用程序遵循例如由symfony/framework-standard-edition Composer项目，默认路径和设置将正常工作。

请注意，必须将appDevDebugProjectContainer.xml和appDevUrlGenerator.php放在Symfony的默认路径下，才能使用Symfony插件提供的所有功能。当找不到这些文件时，它将使用回退机制，但是在这种情况下，完成和导航的精度会降低。

使用非默认项目结构时，应在Symfony插件设置中更新以下路径：

  • Translation Root Path ：复制所有翻译的位置的路径。应将其设置为缓存文件夹下的翻译路径，以便插件可以发现所有可能的翻译。

  • 应用目录 ： 应用目录的路径。

  • 网站目录 ：路径到Web目录。

Symfony探查器

为项目启用Symfony插件后，会将其他区域添加到PhpStorm状态栏。在状态栏中单击Symfony ，将打开弹出菜单，其中包含对我们Symfony应用程序提出的最新请求的相关目标。

导航到相关目标

  • 在探查器弹出菜单中单击相应的项目：

    • 调试网址 ：给定请求的Symfony探查器URL，例如/app_dev.php/_profiler/355651 。

    • 路线 ：应用程序中的路线注册

    • 控制器 ：参与创建响应的控制器。

    • 模板 ：与响应相关的任何模板。

Symfony代码样式

PhpStorm提供了专门为Symfony开发量身定制的内置Symfony2 代码样式 。

设置Symfony2代码样式

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到编辑器|代码样式| PHP 。

  2. 单击右上角的Set from ...链接，然后选择Predefined Style |。弹出菜单中的Symfony2选项。

根据Symfony编码标准检查代码

借助PhpStorm，您可以将PHP_CodeSniffer工具与用于为PHP_CodeSniffer提供一组特定于Symfony的标准的Symfony编码标准结合使用，该工具可检测编码标准问题。这将确保您的代码干净，一致并且没有一些常见错误。

首先，请使用安装和配置PHP_CodeSniffer中描述的任何技术来安装PHP_CodeSniffer 。可能最简单的方法是使用Composer安装它。

要同时安装两个软件包，请将它们添加到require-dev composer.json部分，然后按照Composer中所述运行Install 。

安装PHP_CodeSniffer

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择squizlabs / php_codesniffer软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

接下来，安装Symfony编码标准 ，它将为PHP_CodeSniffer提供特定于Symfony的标准。

安装Symfony编码标准

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择escapestudios / symfony2-coding-standard软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

将安装Symfony编码标准软件包，并自动为PHP_CodeSniffer验证检查选择相应的Symfony标准。如有必要，您可以在“ 编辑器” | “ 编辑器”中进一步自定义检查。 检查页面的 设置/首选项对话框Ctrl+Alt+S 。有关详细信息，请参见将PHP_CodeSniffer配置为PhpStorm检查 。

使用PhpStorm的Symfony命令行工具

PhpStorm与Symfony控制台和脚本界面集成。在开始之前，请按照https://symfony.com/doc/current/components/console.html所述下载并安装Symfony Console。

将Symfony配置为命令行工具

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到工具|命令行工具支持 。

  2. 请点击在工具栏上。

  3. 在“ 命令行工具”对话框中，从列表中选择Symfony ，然后指定其可见性级别（ Project或Global ）。

  4. 当您单击OK时 ， Symfony对话框打开。

    提供Symfony控制台可执行文件和PHP解释器的路径，设置Symfony版本，然后单击OK 。

    对于Symfony 1. * ，默认的可执行文件位置是/ data / bin / symfony ，对于Symfony 2是 / app / console ，对于Symfony 3及更高版本是 / bin / console 。

  5. 单击“ 确定”以应用更改并返回“ 命令行工具支持”页面。（可选）单击编辑工具属性，或自定义命令集。有关详细信息，请参见自定义工具 。

运行Symfony命令

  • 在主菜单中，选择“ 工具” |“工具”。运行命令或按Ctrl两次。

    在打开的“运行任何内容”窗口中，在 格式。

    命令执行结果显示在“ 运行工具”窗口中 。

终止命令

  • 请点击在运行工具窗口工具栏上。

调试Symfony命令

Symfony命令在扩展的控制器类中定义Command 。要调试命令，至关重要的是调试命令本身，而不是调试定义在其中的控制器类文件。否则，将跳过Symfony引导过程，并且执行将失败。

  1. 在与所选命令对应的控制器类中，在要设置断点的代码行上单击编辑器装订线。

  2. 创建一个运行/调试配置 ，该配置将使用所选命令运行symfony工具。在主菜单中，选择运行|编辑配置 ，然后单击然后从列表中选择PHP脚本 。

  3. 在“ 运行/调试配置：PHP脚本”对话框中，提供运行/调试配置参数。

    • 在“ 文件”字段中，提供symfony可执行文件的路径。

    • 在“ 参数”字段中，键入实际的命令及其参数，例如app:list-users 。

  4. 在PhpStorm工具栏上，选择创建的运行/调试配置，然后单击 。命令执行将在指定的断点处停止。
