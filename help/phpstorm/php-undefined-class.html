代码检查：未定义的类

产生两种类型的警告：

  • 未定义类 ：在内置库和项目文件中找不到引用类的声明。

  • 多个声明 ：在项目文件中找到多个类定义。这可能导致代码完成，成员解析和类继承分析方面的问题。

使用以下选项来自定义检查：

  • 不要报告多个类声明潜在的问题 ：如果选中，检查将不会在项目文件中报告具有多个定义的类。

  • 忽略 ：如果选中，则检查将不会报告PHPDoc块中引用的未定义类。

在以下示例中， MyClass内置库和项目文件中未定义class。

$ myClass = new MyClass（）;
