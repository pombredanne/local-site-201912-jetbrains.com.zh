运行/调试配置：代码接收

仅当启用PHP和Codeception插件时，此对话框才可用。 这些插件默认情况下处于激活状态。如果禁用了插件 ，请按照“ 管理插件”中所述在“ 插件”页面上启用它们。

使用此对话框可以创建配置，以用于使用Codeception框架在PHP应用程序上运行和调试单元测试。

在你开始前

如使用Codeception进行测试中所述，在计算机上安装和配置Codeception框架。

测试跑者区

在此区域中，指定要启动的测试，并将命令行开关传递给Codeception 。

项目

描述

测试范围

在此区域中，指定单元测试的位置或列出它们的配置文件。

  • 类型 ：选择此选项可启动特定类型的测试。从清单中选择要运行的测试类型： 验收 ， 功能或单元 。选择“ 全部”以启动所有测试，无论在指定为的文件夹中递归检测到的所有测试类型tests在当前配置文件中。

  • 目录 ：选择此选项可启动目录中的所有测试。在目录字段中，指定要在其中搜索测试的目录。手动输入目录的路径或单击浏览 然后在打开的“选择测试目录”对话框中选择所需的目录。

    文件 ：选择此选项可启动特定文件中的所有测试。在“ 文件”字段中，指定要运行测试的文件。

  • 方法 ：选择此选项可启动特定的测试方法或功能。

    1. 在“ 文件”字段中，指定要在其中搜索测试方法或方案的文件。手动输入文件名或单击浏览 然后在打开的树状视图中选择所需的文件。

    2. 在“ 方法”字段中，指定要运行的测试功能或方法。点击浏览 然后从列表中选择所需的功能：

  • 在配置文件中定义 ：选择此选项可使Codeception从专用的.yml配置文件执行测试。

    默认情况下， Codeception使用在“ 测试框架”页面的“ 测试运行器”区域中指定的配置文件。

    • 要使用所有Codeception运行配置文件的默认设置，请清除“ 使用备用配置文件”复选框。

    • 要从自定义配置文件启动方法/功能，请选中“ 使用备用配置文件”复选框，并在其旁边的字段中指定所需YML文件的位置。

    • 要打开Codeception页面并指定要使用的另一个默认配置文件，请单击按钮。

  • 测试运行程序选项 ：在此字段中，指定要传递给Codeception的命令行选项。例如，添加-vvv –colors作为命令行选项会导致彩色输出消息的调试冗长。有关详细信息，请参见Codeception控制台命令：运行 。

命令行区域

在这方面选择一个PHP解释器，并通过指定要传递给PHP可执行文件的选项和参数来自定义其行为。

项目

描述

口译员

该列表包含所有当前配置的本地和远程PHP解释器。从列表中选择一个已配置的PHP解释器，或单击并定义新的解释器，如配置本地PHP解释器和配置远程PHP解释器中所述 。

口译员选项

在此字段中，指定要传递到PHP可执行文件的选项 。它们会覆盖PHP解释器的默认行为，或确保执行其他活动。

如有必要，请单击然后在“ 命令行选项”对话框中键入所需的选项。在新行上键入每个选项。关闭对话框时，它们全部显示在“ 命令行选项”字段中，并以空格作为分隔符。

自定义工作目录

在这个领域里， 指定要从中执行测试的文件夹，即，测试根目录的父文件夹，如在paths codeception.yml部分。

默认情况下，该字段为空，工作目录为项目的根目录。

环境变量

在此字段中，指定将环境变量传递到内置服务器。有关详细信息，请参见Apache中的环境变量 。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在PhpStorm中定义，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 启动Web浏览器 ：选择此选项以启动浏览器。在打开的对话框中，选择浏览器的类型并提供起始URL。另外，指定是否要使用JavaScript调试器启动浏览器。

  • 运行文件监视程序 ：选择此选项可使PhpStorm应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 编译TypeScript ：选择运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都会反映在生成的JavaScript文件中。 在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防万一检测到任何错误：

    • 如果选中“检查错误”复选框，则编译器将显示所有错误，并且运行配置将不会启动。

    • 如果清除“ 检查错误”复选框，则编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 运行Phing目标 ：添加此任务以在运行或调试之前执行指定的Phing目标。要指定Phing目标，请单击“ 浏览”。 然后在打开的对话框中选择所需的目标。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 将文件上传到远程主机 ：选择此选项可以根据默认服务器访问配置将应用程序文件自动上传到服务器 。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

Alt+Delete

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
