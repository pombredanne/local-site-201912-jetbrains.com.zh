故障排除

本节概述了调试会话期间可能遇到的常见问题的解决方案：

没有配置路径映射

在某些情况下，您可能会收到以下错误消息之一： 远程文件路径path / to / script / on / the / server.php未映射到项目中的任何文件路径或脚本路径/ to / script / on / the /server.php在项目外部 。这意味着PhpStorm不确定哪个本地文件对应于指定的远程文件路径，因为您尚未在Debug服务器配置中指定路径映射。

即使Debug Server配置中的路径映射看起来与Deployment配置中的路径映射非常相似，它们也无法重用，因为Deployment配置要求路径是相对的，并且Debug Server配置依赖绝对路径。

单击链接并在打开的对话框中配置路径映射，如创建PHP调试服务器配置中所述 。

没有配置调试服务器

如果在未配置调试服务器的情况下启动调试会话，则在建立连接后，PhpStorm将显示“ 传入连接”对话框 ，其中建议从服务器访问配置 （ 部署配置 ）导入映射。如果选择从部署中导入映射 ，PhpStorm会尝试检测最合适的部署配置，并在“ 部署”列表和“ 预览”区域中预先选择它显示项目文件的绝对路径，该路径对应于根据所选配置的映射当前执行的脚本。

如果PhpStorm未检测到相关配置：

  1. 从列表中选择最合适的配置或单击并在打开的“ 部署”对话框中创建新配置，随即将新配置添加到列表中。

  2. 在“ 部署根目录”字段中，键入服务器根文件夹的绝对路径。

在这种情况下，您还可以选择“ 手动选择本地文件或项目”选项 PhpStorm显示项目树视图，您可以在其中选择项目文件并将当前执行的脚本映射到该文件。您还可以选择并映射整个项目。

要继续使用导入的或手动指定的配置设置进行调试，请点击接受 。

不暂停脚本

建立零配置调试会话可能会失败，没有断点命中，因此脚本不会挂起。如果发生这种情况如果未正确配置路径映射或配置错误，或者尚未设置任何断点。在后一种情况下，启用“ 外部连接”区域中的“ 在PHP脚本中的第一行中断”复选框，或启用“ 运行” |“启用”。主菜单中的“ PHP脚本中的第一行中断”选项也可能会有所帮助。

要使PhpStorm在脚本未挂起时显示通知，请在“设置”对话框的“ 调试”页面上的“ 高级设置”区域中，选中“ 如果调试会话没有结束就通知它” 。
