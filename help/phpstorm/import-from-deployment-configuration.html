该对话框仅在启用PHP和FTP / SFTP连接插件时可用。 这些插件未与PhpStorm捆绑在一起，但可以如管理插件中所述从JetBrains插件存储库中安装它们。启用后，这些插件将在IDE级别可用，也就是说，您可以在所有PhpStorm项目中使用它们。

单击导入按钮时，对话框打开在“ 服务器”页面的工具栏上。

在此对话框中，选择一种配置以访问服务器上的应用程序并使用其中的主机和端口设置。

部署方式

在此下拉列表中， 选择服务器访问配置（部署配置）以从中复制服务器访问设置。该列表包含所有可用的部署配置。要创建新配置，请单击并在打开的“ 部署：连接选项卡”对话框中指定新设置。

在下面的文本框中，指定服务器部署根文件夹的绝对路径，文本框的名称取决于所选择的服务器访问配置的类型： 绝对路径，为FTP / SFTP / FTPS或远程路径 部署根 所安装的文件夹 ，用于局部或安装文件夹 。

该路径将作为前缀添加到Deployment：Connection选项卡上Root Path字段的路径中。

如果不确定此绝对路径，则可以打开“ 远程主机”工具窗口，选择所需的部署配置，将插入符号放置在根文件夹中，然后从上下文菜单中选择“ 复制路径 ”，有关详细信息，请参阅访问Web服务器上的文件。 。或者，请与您的主机提供商联系。

  • 对于“ 本地”或“已安装的文件夹” ，键入从“部署配置对话框中导入”对话框的“ 文件夹”字段中指定的服务器根目录的绝对路径。

  • 对于Inplace Server配置，不需要映射，因为在这种情况下，本地路径和远程路径相同。

预习

面积显示从所选服务器访问配置（部署配置）检索到的主机/端口和路径映射。

选择要使用的部署配置时，“ 服务器上的绝对路径”字段显示映射到所选配置中项目文件和文件夹的相对路径，即相对于部署根目录的文件和文件夹的路径。当您指定部署根目录（FTP / SFTP / FTPS的服务器根目录或已安装的文件夹）的绝对路径时，该字段的内容将自动更新，最后该字段显示服务器上的绝对路径。

指定PhpStorm如何在服务器上的文件及其本地副本之间建立对应关系。根据这些映射，PhpStorm将打开当前处理文件的本地副本。

PHP Debug Server配置中的路径映射看起来与服务器访问 （ 部署配置）中的路径映射非常相似。不幸的是，它们无法重用，因为部署配置使用相对路径，而PHP Debug Servers配置则依赖绝对路径。

  • 选择使用路径映射复选框如果在远程Web服务器上工作，即Web服务器在物理远程主机上，或者Web服务器已安装在计算机上，但您的项目在Web服务器文档根目录之外。如果使用符号链接，则也请选中此复选框。

    分别使用服务器上的路径和文件/目录字段将绝对路径映射到服务器上的文件和文件夹，并将其绝对路径映射到本地文件系统中的项目文件。

    • 文件/目录：此只读字段显示当前项目的文件和文件夹。选择要用作本地副本的文件或文件夹。

    • 服务器上的路径：在此字段中，指定目标服务器上与所选本地文件或文件夹对应的文件或文件夹的绝对路径。手动键入路径或从列表中选择它。

  • 清除使用路径映射复选框如果您在Web服务器上正确工作，则项目根目录位于服务器文档根目录下。在这种情况下，Web服务器上文件的绝对路径和项目中相应文件的绝对路径是相同的。
