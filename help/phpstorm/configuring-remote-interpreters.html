配置远程PHP解释器

术语“ 远程PHP解释器”表示安装在远程主机或虚拟环境中的PHP引擎。术语“ 远程PHP解释器 ”与计算机上安装的本地PHP解释器相反，请参阅“ 配置本地PHP解释器” 。

您可以通过以下任意一种方式访问远程PHP解释器：

通过使用Vagrant配置，您可以访问安装在相应Vagrant实例上的PHP解释器。

在你开始前：

  1. 确保流浪和Oracle的VirtualBox的下载，安装和您的计算机上配置中描述流浪 。

  2. 确保启用了Vagrant捆绑插件。如果插件被禁用，请按照“ 管理插件”中所述激活它。

  3. 确保将以下可执行文件的父文件夹添加到系统PATH变量中：

    • vagrant.bat或Vagrant安装中的vagrant 。这应该由Vagrant安装程序自动完成。

    • Oracle的VirtualBox安装中的VBoxManage.exe或VBoxManage 。

  4. 在要使用的Vagrant实例中配置PHP开发环境。

了解更多关于使用流浪与PhpStorm 流浪 。

在Vagrant实例中配置PHP解释器

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择语言和框架| PHP 。

  2. 在打开的PHP页面上，单击 CLI解释器列表旁边。

  3. 在打开的“ CLI解释器”对话框中，单击在左侧窗格中，然后从弹出菜单中选择“ 来自Docker，Vagrant，VM，远程... ”。

  4. 在打开的“ 配置远程PHP解释器”对话框中，选择“ Vagrant”方法。

  5. 提供连接参数：

    • 指定Vagrant实例文件夹 ，该文件夹指向您要使用的环境。 从技术上讲，它是所需环境的VagrantFile配置文件所在的文件夹。根据此设置，PhpStorm将检测“ Vagrant主机”并将其显示为“ Vagrant主机URL”只读字段中的链接。

      要使用解释器配置，您需要以下路径映射 ： 设置项目文件夹，服务器上用于将项目文件复制到的文件夹和URL地址之间的对应关系，以访问服务器上复制的数据。 PhpStorm从VagrantFile配置文件评估路径映射。

  6. 在PHP解释器路径字段中， 根据所选远程开发环境的配置指定PHP可执行文件的位置。

    • 对于Vagrant实例，默认情况下，PhpStorm建议使用/ usr / bin / php文件夹。

    要指定其他文件夹，请单击然后在打开的对话框中选择相关文件夹。请注意，必须打开PHP主目录才能进行编辑。

    当您单击OK时 ，PhpStorm将检查PHP可执行文件是否实际存储在指定的文件夹中。

    • 如果未找到PHP可执行文件，PhpStorm将显示一条错误消息，询问您是否继续搜索还是保存解释器配置。

    • 如果找到PHP可执行文件，则返回“ CLI解释器”对话框，其中显示安装文件夹和检测到的PHP解释器版本。

配置自定义映射

如果您使用通过SFTP连接可访问的解释器，或者位于Vagrant实例或Docker容器中的解释器，则将从相应的部署配置Vagrantfile或Dockerfile中自动检索映射。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| PHP 。

  2. 从“ 解释器”列表中，选择要为其自定义映射的远程解释器。“ 路径映射”只读字段显示从相应的部署配置Vagrantfile或Dockerfile检索的路径映射。要指定自定义映射，请单击 路径映射字段旁边。

  3. 打开的“ 编辑项目路径映射”对话框 ，显示从部署配置Vagrantfile或Dockerfile检索的路径映射。这些映射是只读的。

    • 要添加自定义映射，请单击并分别在“ 本地路径”和“ 远程路径”字段中指定项目中的路径以及远程运行时环境中的相应路径。手动输入路径或单击然后在打开的对话框中选择相关文件或文件夹。

    • 要删除自定义映射，请在列表中选择它，然后单击 。

在配置的PHP解释器之间即时切换

  1. 按Ctrl+Shift+A然后开始打字Change PHP interpreter 。在建议列表中，选择“ 更改PHP解释器”操作。

    如有必要，您可以直接在建议列表中为该操作分配键盘快捷键，方法是按Alt+Enter ，或稍后进行配置键盘快捷方式中所述的操作 。

  2. 在打开的弹出菜单中，选择已配置的本地或远程 PHP解释器之一。

所选解释器将在PHP页面上设置为默认项目解释器。 的 设置/首选项对话框Ctrl+Alt+S 。这还将影响设置为使用默认项目解释器的运行/调试配置 ， 测试框架和质量工具的配置。
