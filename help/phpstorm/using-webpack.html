PhpStorm 2019.3帮助

Webpack

PhpStorm与webpack模块捆绑器集成。通过支持webpack模块解析并解析别名，此支持可提高JavaScript文件中的编码帮助。对于Webpack版本2和更高版本，PhpStorm提供了Webpack配置文件中选项的代码完成和快速文档查找。

在你开始前

下载并安装Node.js。

在PhpStorm中配置Webpack

  1. 确保webpack列在dependencies要么devDependencies您package.json的对象。

    如果缺少webpack， 打开嵌入式终端 （ Alt+F12 ）并输入：

    npm install --save-dev webpack

  2. 在项目根目录或其他地方创建一个配置文件（ New | JavaScript file ）。从webpack官方网站了解更多信息。

  3. 指定要使用的webpack配置文件。默认情况下，PhpStorm分析项目根目录中的webpack配置文件。

    要使用另一个Webpack配置文件， 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | JavaScript | Webpack并指定要使用的配置文件的位置。

根据对Webpack配置文件的分析，PhpStorm可以了解Webpack配置并在JavaScript文件中提供编码帮助，请参阅下面的解决模块 。

另请参阅webpack官方网站上的入门 。

编辑Webpack配置文件

对于Webpack版本2和更高版本，PhpStorm在webpack.config.js的配置对象中提供代码完成和文档查找。即时提供代码完成。要查看符号的文档，请按Ctrl+Q 。

PhpStorm仅在名称包含以下内容的Webpack配置文件中提供编码帮助： webpack字符串和webpack在package.json中列出。

解决模块

当您打开项目或编辑在“设置/首选项” |“设置”中指定的webpack.config.js时 语言和框架| JavaScript | Webpack ，PhpStorm在后台分析配置，并根据接收到的信息正确了解项目解析根目录和别名 。由于对项目配置的这种理解，PhpStorm为JavaScript文件中的导入和导出符号提供了更精确的代码完成。结果，一切正常，无需您采取任何措施。
