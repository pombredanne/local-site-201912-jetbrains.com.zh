PhpStorm 2019.3帮助

运行和调试TypeScript

使用PhpStorm，您可以运行和调试在Node.js中运行的客户端TypeScript代码和TypeScript代码。

在运行或调试应用程序之前，您需要将TypeScript代码编译为JavaScript。您可以使用内置的TypeScript编译器和其他工具（包括用于与Node.js一起运行TypeScript的ts-node）执行此操作，这些工具可以单独使用，也可以在构建过程中使用。

在编译期间，还可以生成生成源映射 ，以设置TypeScript代码和实际执行的JavaScript代码之间的对应关系。因此，您可以在TypeScript代码中设置断点，使用类型为JavaScript Debug （用于客户端代码）或Node.js的运行/调试配置启动应用程序，然后逐步执行原始的TypeScript代码，这要归功于生成的源映射。

运行客户端TypeScript应用程序

您可以使用TypeScript编写客户端应用程序，按照将TypeScript编译为JavaScript中所述编译代码，然后以与用JavaScript编写的客户端应用程序完全相同的方式运行和调试应用程序。唯一的区别是您可以在TypeScript代码中直接设置断点。

  1. 将TypeScript代码编译为JavaScript 。

  2. 在编辑器中，打开HTML文件，并引用生成的JavaScript文件。此HTML文件不一定必须是实现应用程序起始页面的HTML文件。

  3. 请执行以下任一操作：

    • 选择查看|从主菜单在浏览器中打开或按Alt+F2 。然后从列表中选择所需的浏览器。

    • 将鼠标指针悬停在代码上以显示浏览器图标栏： 单击表示所需浏览器的图标。

调试客户端TypeScript应用程序

通常，您可能想调试运行在外部开发Web服务器上的客户端应用程序，例如由Node.js支持的服务器。如果您的应用程序在内置PhpStorm服务器上 运行 ，请参阅上面的运行客户端TypeScript应用程序 ，您也可以用与在内置服务器上运行的JavaScript相同的方式对其进行调试。

调试在外部Web服务器上运行的TypeScript应用程序

  1. 如配置JavaScript调试器中所述配置内置调试器 。

  2. 在TypeScript代码中配置和设置断点 。

  3. 在开发模式下运行该应用程序。通常你需要跑步npm start为了那个原因。

    最常见的是，在此阶段将TypeScript编译为JavaScript并生成源映射，有关详细信息，请参见将TypeScript编译为JavaScript 。

    开发服务器准备就绪后，在浏览器中复制应用程序运行所在的URL地址-您将需要在运行/调试配置中指定此URL地址。

  4. 选择运行|从主菜单编辑配置 ， 点击在工具栏上，然后从列表中选择JavaScript Debug 。 在打开的“运行/调试配置：JavaScript调试”对话框中，指定应用程序运行所在的URL地址。如上面的步骤3所述，可以从浏览器的地址栏中复制此URL。

  5. 在工具栏上的“ 选择运行/调试配置”列表中选择新创建的配置，然后单击 。在运行配置中指定的URL地址在选定的浏览器中打开，并出现“ 调试工具”窗口 。

  6. 在“ 调试工具”窗口中，照常进行操作： 逐步执行程序 ， 停止并继续执行程序， 在挂起时进行检查 ， 查看实际的HTML DOM等。

使用Node.js运行和调试服务器端TypeScript应用程序

使用PhpStorm，您可以通过Node.js运行配置在Node.js上启动服务器端TypeScript代码。在运行或调试之前，必须将TypeScript代码编译到JavaScript中，如将TypeScript编译为JavaScript中所述 。

使用Node.js运行服务器端TypeScript

  1. 将您的TypeScript代码编译为JavaScript ，有关详细信息，请参见将TypeScript 编译为JavaScript 。

  2. 创建一个Node.js运行/调试配置：

    1. 从主菜单中，选择运行|编辑配置 ，然后在“ 编辑配置”对话框中， 点击在工具栏上，然后从列表中选择Node.js。

    2. 在打开的“运行/调试配置：Node.js”对话框中，指定要使用的Node.js解释器。在“ JavaScript文件”字段中，指定从启动它的应用程序的主文件生成的编译文件。

    从创建Node.js运行/调试配置中了解更多信息 。

  3. 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的Node.js配置，然后单击在它的旁边。

使用Node.js调试服务器端TypeScript

  1. 将您的TypeScript代码编译为JavaScript ，有关详细信息，请参见将TypeScript 编译为JavaScript 。

  2. 必要时在TypeScript代码中设置断点 。

  3. 如上所述创建Node.js运行/调试配置。

  4. 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的Node.js配置，然后单击在它的旁边。将打开“ 调试工具”窗口 。

  5. 执行将触发带有断点的代码执行的步骤。

  6. 切换到PhpStorm，此时已启用“ 调试工具”窗口的控件。

使用ts-node

如果需要使用Node.js运行或调试单个TypeScript文件，则可以使用ts-node而不是按照将TypeScript编译为JavaScript所述编译代码。

安装ts-node

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install --save-dev ts-node

为ts-node创建自定义Node.js运行/调试配置

  1. 从主菜单中，选择运行|编辑配置 ，然后在“ 编辑配置”对话框中， 点击在工具栏上，然后从列表中选择Node.js。将打开“ 运行/调试配置：Node.js”对话框。

  2. 在“ 节点参数”字段中，添加--require ts-node/register 。

  3. 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

  4. 在“ JavaScript文件”字段中，指定要运行或调试的文件。根据您的工作流程，您可以显式地执行此操作，也可以使用宏。

    • 如果要始终启动相同的TypeScript文件，请单击然后在打开的对话框中选择该文件。默认情况下，运行/调试配置获取所选文件的名称。

    • 如果需要启动其他文件，请键入$FilePathRelativeToProjectRoot$ 。使用此宏，PhpStorm将始终在活动的编辑器选项卡中启动文件。

  5. 可选地

    要将任何其他参数传递给ts-node（例如， --project tsconfig.json ），然后将它们添加到“ 应用程序参数”字段中。

  6. 保存配置。

运行一个TypeScript文件

根据您在运行/调试配置中指定TypeScript文件的方式，执行以下操作之一：

  • 如果您明确键入文件名，请在“ 项目”工具窗口中选择文件，或在编辑器中将其打开，然后选择“运行”。 。

    或者，从工具栏上的列表中选择所需的配置，然后单击列表旁边或按Shift+F10 。

  • 如果指定了宏，请在编辑器中打开TypeScript文件，从工具栏上的列表中选择新创建的配置 ，然后单击或按Shift+F10 。

PhpStorm在“ 运行”工具窗口中显示输出。

调试TypeScript文件

  1. 在要调试的TypeScript文件中，根据需要设置断点。

  2. 根据您在运行/调试配置中指定TypeScript文件的方式，执行以下操作之一：

    • 如果您明确键入文件名，请在“ 项目”工具窗口中选择文件，或在编辑器中将其打开，然后选择“ 调试”。 。

      或者，从工具栏上的列表中选择所需的配置，然后单击列表旁边或按Shift+F9 。

    • 如果指定了宏，请在编辑器中打开TypeScript文件，从工具栏上的列表中选择新创建的配置 ，然后单击列表旁边或按Shift+F9 。
