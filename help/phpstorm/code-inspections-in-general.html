报告的使用情况

  • @SuppressWarning注释，或

  • // noinspection行注释，或

  • /** noinspection */ JavaDoc评论

可以安全删除它们，因为它们影响的检查在这种情况下不再适用。

例如：

public class C { // symbol is already private, // but annotation is still around @SuppressWarnings({"WeakerAccess"}) private boolean CONST = true ; void f() { CONST = false ; } }

警告

重复的代码片段

报告来自选定范围的重复代码块：同一文件或整个项目。检查功能具有快速修复功能，可帮助您设置检测到的重复项的大小，导航至重复的代码片段并在工具窗口中进行比较。

检查选项使您可以选择报告的重复片段的范围，并为重复的语言构造设置初始大小。

您还可以在File | File中配置要匿名化的构造。首选项|编辑器重复项 。

项目指纹

收集文件统计信息（例如语言，树深度）以创建项目指纹

注释者

使您可以在批处理代码检查的结果中查看语言注释者报告的问题。

错误

注入参考

检查由IntelliLang插件注入的引用。

错误

有问题的空格

当代码样式配置为仅使用空格时，报告用于缩进的选项卡。当代码样式配置为仅使用制表符时，报告用于缩进的空间。当代码样式配置为使用智能选项卡时，报告用于缩进的空间和用于对齐的选项卡。

 残障人士
结构搜索检查

通过此检查，可以配置要应用于要编辑的文件的“ 结构搜索/结构替换”模板的数量。

所有匹配项都会以您配置的模板名称突出显示。如果配置了“ 结构替换”模式，则相应的替换将显示为快速修复。

 残障人士
TODO评论

在您的代码中报告TODO注释。TODO注释的格式可通过“ 设置” |“配置”来配置。编辑器TODO面板。由于已经提供了TODO注释的语法突出显示功能，因此，预计将在批处理模式下使用该功能。

 残障人士
分行符不一致

此检查检测到的行分隔符与项目默认值不同的文件。例如，您在“设置” |“代码样式” |“行分隔符”中将行分隔符设置为“ \ n”，并且您正在编辑的文件使用“ \ r \ n”作为行分隔符。
