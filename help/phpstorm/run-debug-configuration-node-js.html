在此字段中，键入要传递到NodeJS可执行文件的特定于Node.js的命令行选项。最常见的选项是：

  • 使用--require coffee-script/register在运行期间将CoffeeScript文件即时编译为JavaScript。

    这种模式要求3软件包的一部分register.js文件应该位于项目内部。因此，您需要在本地的Node.js页面上安装coffee-script软件包，如npm和Yarn中所述 。

  • 准备好后，单击确定 。

变量的定义显示在“ 环境变量”只读字段中，以分号作为分隔符。可接受的变量是：

  • NODE_PATH ： 一种: -前缀为模块搜索路径的目录的列表。

  • NODE_DISABLE_COLORS ：设置为1以禁用REPL中的颜色。

Docker容器设置

仅当为项目选择基于Docker的远程解释器时，才会显示此字段。

请点击打开对话框并指定以下设置：

  • 禁用网络 ：选中此复选框可禁用网络。这对应于--net="none" ，这意味着在容器内部，外部网络资源不可用。

  • 网络模式 ：对应于该选项的其他值--net 。

    • bridge是默认值。将为网桥的网络上的容器分配一个IP地址，流量将通过该网桥路由到容器。
      默认情况下，容器可以通过其IP地址进行通信。要通过名称进行通信，必须将它们链接起来。

    • host ：使用容器内主机的网络堆栈。

    • container: ：使用通过其名称或id指定的另一个容器的网络堆栈。

    有关详细信息，请参阅网络设置文档。

  • 链接 ：使用此部分将要创建的容器与其他容器链接。这适用于Network mode = bridge并对应于--link选项。

  • 发布所有端口 ：将所有容器端口公开给主机。这对应于选项--publish-all 。

  • 端口绑定 ： 指定端口绑定列表。类似于使用-p与选项docker run 。

  • 额外主机 ：这对应于--add-host选项。有关详细信息，请参阅管理/ etc / hosts页面。

  • 卷绑定 ：使用此字段可以指定特殊文件夹（ 卷）和运行Docker守护程序的计算机的文件夹之间的绑定。这对应于-v选项。
    有关详细信息，请参见管理容器中的数据。

  • 环境变量 ：使用此字段可以指定环境变量及其值的列表。这对应于-e选项。有关详细信息，请参见页面ENV（环境变量） 。

请点击扩展表格。请点击 ， ， 要么组成列表。

说到设置与某些选项的对应关系（ --net ， --link等），请注意，这些选项来自Docker命令行参数 。

自动配置

选择此复选框可使PhpStorm配置容器设置。在自动配置模式下：

  • PhpStorm将创建一个新映像并在其中安装npm模块。

  • PhpStorm使用新映像运行容器，将项目文件夹绑定到容器中的/ opt / project文件夹以确保更新同步，并将/ opt / project / node_modules映射到OS临时目录。

即使使用自动配置，您仍然需要将运行应用程序的端口与容器的端口绑定。这些暴露的端口在Docker主机的IP地址上可用（默认为192.168.99.100）。在调试Node.js Express应用程序的客户端时，需要这种绑定。在这种情况下，您需要从计算机打开浏览器，并通过应用程序中指定的端口访问容器主机上的应用程序。

PhpStorm将package.json复制到映像中的/ tmp / project_modules文件夹中，运行npm install ，然后将模块复制到容器中的项目文件夹中。因此，在项目中更改package.json将导致重新生成映像。

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在PhpStorm中定义，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 启动Web浏览器 ：选择此选项以启动浏览器。在打开的对话框中，选择浏览器的类型并提供起始URL。另外，指定是否要使用JavaScript调试器启动浏览器。

  • 运行文件监视程序 ：选择此选项可使PhpStorm应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 编译TypeScript ：选择运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都会反映在生成的JavaScript文件中。 在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防万一检测到任何错误：

    • 如果选中“检查错误”复选框，则编译器将显示所有错误，并且运行配置将不会启动。

    • 如果清除“ 检查错误”复选框，则编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 运行Phing目标 ：添加此任务以在运行或调试之前执行指定的Phing目标。要指定Phing目标，请单击“ 浏览”。 然后在打开的对话框中选择所需的目标。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 将文件上传到远程主机 ：选择此选项可以根据默认服务器访问配置将应用程序文件自动上传到服务器 。

  • 运行远程外部工具 ：添加远程SSH外部工具 。
