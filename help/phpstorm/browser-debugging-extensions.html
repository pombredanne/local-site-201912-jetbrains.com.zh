浏览器调试扩展

为了开始调试，您首先需要激活服务器上的调试器引擎。为此，您需要设置一个特殊的GET/POST要么COOKIE参数（有关详细信息，请参见Xdebug和Zend Debugger官方文档）。虽然可以手动完成，但使用浏览器扩展更为方便，它使您只需单击即可启用调试器。

为Chrome配置Xdebug助手

  1. 从Chrome网上应用店安装适用于Chrome的Xdebug帮助程序扩展。

  2. 在PhpStorm中，通过单击以下任一方法来侦听传入的调试连接在工具栏上，或选择运行|开始侦听PHP调试连接

  3. 从浏览器端启动连接。单击浏览器工具栏上的Xdebug帮助器图标，以启动调试，概要分析或跟踪会话：

  4. 通常，不需要其他配置。如有必要，可以通过右键单击Xdebug Helper图标并从上下文菜单中选择“ 选项”来探索其他设置。

为Firefox配置Zend调试器工具栏

  1. 安装Zend Debugger工具栏 。

  2. 在PhpStorm中，通过单击以下任一方法来侦听传入的调试连接在工具栏上，或选择运行|开始侦听PHP调试连接

  3. 在浏览器中，单击工具栏上的Zend Debugger图标，然后选择Settings 。确保已选择“ 自动检测” ，并且“ 广播端口”值与在PhpStorm的“ 调试”页面上为“设置广播端口”设置的值匹配。

  4. 从浏览器端启动连接。单击浏览器工具栏上的Zend Debugger图标以启动调试或概要分析会话：
