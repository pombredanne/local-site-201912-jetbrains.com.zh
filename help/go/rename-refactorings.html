重命名重构

使用重命名重构可以在整个代码中更改符号，文件的名称以及对它们的所有引用。

由于仅会影响有限的范围，因此可以轻松地内联重命名局部变量或私有方法。重命名类型，接口或公共方法可能会影响许多文件。重构之前先预览可能的更改。

重命名在接口中声明的方法时，还可以查看并重命名其实现。重构更改显示在“ 查找”工具窗口中的“ 重构预览”选项卡上。

您无法重命名.idea项目目录，因为GoLand总是从该目录中读取具有该确切名称的项目文件。

调用重命名重构

  1. 在编辑器中，选择要重命名的元素。如果需要重命名文件，请在“ 项目”工具窗口中选择一个。

  2. 按Shift+F6或从主菜单中选择“ 重构|重命名 。

  3. 您可以就地执行重命名重构或按Shift+F6如果您需要指定其他选项，请再次打开“ 重命名”对话框。例如，指定搜索元素出现的位置或重命名其他内容。您还可以指定重构范围。

  4. 在“ 重构预览”对话框中，查看可能的更改，然后单击“ 执行重构” 。

下次调用“ 重命名”重构时，GoLand会记住在“ 重命名”对话框中指定的选项。

重命名接口中的方法

您可以重命名在接口中声明的方法。重命名方法时，GoLand将显示一个上下文菜单，您可以在其中选择重命名方法的实现。

  1. 右键单击界面中的方法，然后选择“ 重构” |“重构”。重命名 。或者，将插入符放置在界面中的方法处，然后按Shift+F6 。

  2. 在上下文菜单中，选择要重命名的内容。要重命名方法和所有实现，请选择重构方法规范和所有实现 。否则，要仅重构所选方法，请选择仅重构当前方法 。

  3. 在“ 更改签名”对话框中，在“ 名称”字段中更改方法名称。

  4. 单击重构 。

    要在重构之前预览更改，请点击预览 。
