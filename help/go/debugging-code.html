调试代码

在调试会话期间 ，您将在连接了调试器的情况下启动程序。调试器的目的是干扰程序执行，并向您提供有关幕后情况的信息。这有助于检测和修复程序中的错误的过程。

您可以通过多种方式运行调试会话，但是为了简单起见，本文档假设您是从GoLand构建和运行项目的。这是最常见的情况，与更先进的技术相比，它的局限性较小。在单独的部分中介绍了附加到进程和调试远程应用程序的过程。

调试之前

  1. 在“设置/首选项”中配置常见的调试属性和行为 。 构建，执行，部署|调试器 。

    如果您不熟悉调试，则可以使用现成的配置。有关每种调试器功能的主题提供了参考，并在适用时说明了相关设置。如果您是高级用户并正在寻找某些特定属性，请参见“ 调试器参考”部分。

  2. 如果要使用自定义配置，请定义运行/调试配置。如果需要在启动前将一些参数传递给程序或需要执行一些特殊的活动，则这是必需的。有关如何设置运行/调试配置的更多信息，请参阅“使用运行/调试配置”部分。在大多数情况下，您不需要使用它来调试不期望参数或有任何特殊要求的简单程序。

一般调试程序

没有用于调试应用程序的千篇一律的过程。根据实际要求，您可能必须以不同的顺序使用不同的操作。本主题提供了一般准则，它们代表典型的调试步骤。各个主题中提供了有关如何以及何时使用特定功能的详细信息。

  1. 定义需要在哪里停止程序。这是使用断点完成的。断点是特殊的标记，代表调试器需要介入并冻结程序状态时的位置和/或条件。已由调试器冻结的程序称为暂停 。

    使用断点的替代方法是在任意时刻手动挂起程序，但是此方法对调试器功能施加了一些限制，并且不允许在何时挂起程序方面有太多的准确性。

  2. 在调试模式下运行程序 。就像定期运行程序一样，您可以同时运行。

  3. 程序挂起后，使用调试器获取有关程序状态及其在运行期间的变化的信息。

    调试器为您提供有关变量值，线程的当前状态，堆中当前存在的对象的细分等信息。它还允许您通过在程序执行过程中抛出异常（例如，检查异常的处理方式）或运行任意代码来在各种条件下测试程序。

    这些工具使您可以在特定时刻检查程序状态，而步进功能使您可以控制程序的逐步执行。通过组合工具，您可以推断出错误的来源，并测试程序的健壮性。

  4. 确定了需要解决的问题后，无需结束会话即可进行操作。为此，GoLand提供了一项功能，允许您即时调整和重新加载代码。
