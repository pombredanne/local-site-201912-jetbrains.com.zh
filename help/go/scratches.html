GoLand 2019.3帮助

暂存文件

有时您可能需要创建临时注释或在项目上下文之外草拟一些代码。您可以使用暂存文件和暂存缓冲区来代替为此目的切换到其他应用程序。

  • 临时文件是功能齐全，可运行且可调试的文件，它们支持语法突出显示，代码完成以及相应文件类型的所有其他功能。例如，在处理一个项目时，您可能会想出一种方法，供以后在另一个项目中使用。您可以使用该方法的草稿创建一个临时文件，该草稿文件不会存储在您的项目目录中，但可以从另一个项目中访问和打开。您可以使用暂存文件来草拟代码构造， HTTP请求 ，JSON文档等。

  • 暂存缓冲区是简单的文本文件，没有任何编码辅助功能。暂存缓冲区可用于简单的任务列表和自己的注释。它们也没有存储在项目目录中，但是可以从另一个项目访问和打开。您最多可以创建五个具有默认名称的暂存缓冲区，这些缓冲区将通过清除内容而轮换使用。

创建一个临时文件

  1. 请执行以下任一操作：

    • 从“ 文件”菜单中，选择“ 新建” ，然后单击“ 暂存文件” 。

    • 从“ 文件”菜单中，选择“ 新建暂存文件” 。

    • 按Ctrl+Shift+Alt+Insert 。

    • 按Ctrl+Shift+A ，开始输入scratch file然后选择相应的操作。

  2. 选择临时文件的语言。相同类型的暂存文件将自动编号，并添加到“ 项目”工具窗口的“暂存和控制台”目录中。

    创建Go scratch文件时，GoLand会自动添加一个main包和一个main()功能。

创建暂存缓冲区

没有用于创建新暂存缓冲区的操作专用菜单项，但是您可以使用“ 查找操作”弹出窗口Ctrl+Shift+A并运行“ 新暂存缓冲区”操作。

另外，您可以按照配置键盘快捷方式中的说明为“ 新建暂存缓冲区”操作创建快捷方式 。

GoLand创建一个名为buffer1.txt的文本文件。您创建的下一个暂存缓冲区名为buffer2.txt ，依此类推，直到buffer5.txt 。当达到该限制时，它将重新创建buffer1.txt并建议清除其内容。如果要确保在有五个缓冲区后不清除暂存缓冲区，可以重命名它。

查看暂存文件和缓冲区的列表

要查看已创建的暂存文件和缓冲区的列表，请打开“ 项目”工具窗口 ，展开“ 暂存器和控制台” ，然后展开“ 暂存器” 。

控制台用于为GoLand中定义为数据源的数据库编写和执行SQL语句。仅当启用了数据库工具和SQL插件时，此功能才可用。有关更多信息，请参见数据库控制台 。

划痕文件和缓冲区存储在IDE配置目录下的划痕 。从使用此配置目录的任何IDE和项目中都可以使用它们。

句法
％HOMEPATH％\。 \ config
例
C：\ Users \ JohnS \。％ide_system_dir％2019.3 \ config
句法
〜/图书馆/首选项/
例
〜/库/首选项/%ide_system_dir%2019.3
句法
〜/。 /配置
例
〜/.% ide_system_dir％2019.3 / config

查看文档以查找划痕

  • 选择任何暂存文件或缓冲区，然后按Ctrl+Q查看带有位置，类型，大小，创建和修改日期的快速文档 。

移动划痕

  1. 在“ 项目”工具窗口的“ 暂存器和控制台”中 ， 单击“ |”。划痕目录，选择临时文件移动。

  2. 按F6然后在打开的对话框中选择目标文件夹。

更改暂存文件或缓冲区的语言

  1. 在“ 项目”工具窗口中 ，展开“ 暂存器和控制台” ，然后展开“ 暂存器” 。

  2. 右键单击暂存文件或缓冲区，然后单击“ 更改语言” 。当前语言在括号中指定。

  3. 选择所需的语言。您可以开始输入语言名称以缩小列表范围。

与其他文件一样，您可以对暂存文件和缓冲区执行以下操作：

  • 使用装订线图标运行和调试代码

  • 使用本地历史记录 ，撤消和重做更改

  • ， 删除 ， 移动和复制

  • 执行剪贴板操作
