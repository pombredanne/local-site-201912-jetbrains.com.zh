创建测试配置

测试目录的配置

  1. 导航至运行|编辑配置 。

  2. 在“ 运行/调试配置”对话框中，单击“ 添加新配置”。 Alt+Insert然后选择Go Test 。

  3. 在目录字段中，指定包含应用程序文件和测试文件的目录的路径（例如， applicationFolder / main.go和applicationFolder / main_test.go ）。

    （可选）此外，您可以指定以下设置：

    • 模式 ：您为运行子测试和子基准所定义的参数。有关子测试和子基准的更多信息，请参阅golang.org上的子测试和子基准 。考虑以下测试功能：

      func MyTestFunc（t *测试。T）{...t。运行（“ ID = 1”，功能（测试）。T）{...}）t。运行（“ ID = 2”，函数（t *测试。T）{...}）t。运行（“ UID = 1”，函数（t *测试。T）{...}）...}

      为了MyTestFunc功能，可以使用以下go test参数：

      码

      描述

      go test -run

      运行所有测试。

      go test -run Test

      运行与Test模式（例如， MyTestFunc ）。

      go test -run MyTestFunc/ID=

      运行属于顶层的子测试MyTestFunc功能和匹配ID=图案。

      go test -run /UID=1

      运行子测试匹配UID=1来自所有顶级测试功能。

      对于模式字段，您仅列出参数（例如， Test ， MyTestFunc/ID= ， /UID=1 ）。

    • 工作目录 ：用于构建的应用程序的目录。如果您具有创建相对文件或目录的任何代码，则它们将相对于此目录。

    • Environment ：您需要运行测试的环境变量。

    • Go工具参数 ： go工具的参数 （例如， -tags ）。

    • 使用所有自定义构建标签 ：在构建过程中应用的所有标签。标签在设置中列出Ctrl+Alt+S在Go下建立标签和供应商 。

    • 程序参数 ：生成的应用程序的参数 。

    • 使用sudo运行 ：为应用程序授予sudo特权。

    • 启动之前：激活工具窗口 ：在启动应用程序之前添加要启动的任务。要添加任务，请单击添加按钮。 Alt+Insert并选择要添加的工具。

  4. 点击应用 。

包的测试配置

如果要为程序包创建运行/调试配置，请确保应用程序文件和测试文件位于程序包或相关Go程序包（模块）的集合中。软件包必须位于go / src目录中（例如go / src / myPackage ）。模块是go.mod文件，其中列出了所有相关的Go软件包和要求。

  1. 导航至运行|编辑配置 。

  2. 在“ 运行/调试配置”对话框中，单击“ 添加新配置”。 Alt+Insert然后选择Go Test 。

  3. 从“ 测试种类”列表中，选择“ 包” 。

  4. 在“ 文件”字段中，输入测试文件的名称（例如main_test.go ）。您可以使用智能完成 Ctrl+Shift+Space在这个领域里。

    （可选）此外，您可以指定以下设置：

    • 模式 ：您为运行子测试和子基准所定义的参数。有关子测试和子基准的更多信息，请参阅golang.org上的子测试和子基准 。考虑以下测试功能：

      func MyTestFunc（t *测试。T）{...t。运行（“ ID = 1”，功能（测试）。T）{...}）t。运行（“ ID = 2”，函数（t *测试。T）{...}）t。运行（“ UID = 1”，函数（t *测试。T）{...}）...}

      为了MyTestFunc功能，可以使用以下go test参数：

      码

      描述

      go test -run ''

      运行所有测试。

      go test -run Test

      运行与Test模式（例如， MyTestFunc ）。

      go test -run MyTestFunc/ID=

      运行属于顶层的子测试MyTestFunc功能和匹配ID=图案。

      go test -run /UID=1

      运行子测试匹配UID=1来自所有顶级测试功能。

      对于模式字段，您仅列出参数（例如， Test ， MyTestFunc/ID= ， /UID=1 ）。

    • 输出目录 ：存储测试结果（如果有）的目录。

    • 工作目录 ：用于构建的应用程序的目录。如果您具有创建相对文件或目录的任何代码，则它们将相对于此目录。

    • Environment ：您需要运行测试的环境变量。

    • Go工具参数 ： go工具的参数 （例如， -tags ）。

    • 使用所有自定义构建标签 ：在构建过程中应用的所有标签。标签在设置中列出Ctrl+Alt+S在Go下建立标签和供应商 。

    • 程序参数 ：生成的应用程序的参数 。

    • 使用sudo运行 ：为应用程序授予sudo特权。

    • 启动之前：激活工具窗口 ：在启动应用程序之前添加要启动的任务。要添加任务，请单击添加按钮Alt+Insert并选择要添加的工具。

  5. 点击应用 。

    • 程序参数 ：生成的应用程序的参数 。
