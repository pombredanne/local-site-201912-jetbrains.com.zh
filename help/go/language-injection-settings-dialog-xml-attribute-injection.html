语言注入设置对话框：XML属性注入

GoLand |首选项|编辑器语言注入| |适用于macOS的XML属性注入

项目

描述

名称

注射的名称。

语言

要注入的语言。

  • ID。 语言ID或名称。

  • 字首。 在相应的字符串值之前要添加的字符序列。

  • 后缀。 在相应的字符串值之后要添加的字符序列。

前缀和后缀是可选的。

XML标签

在此区域中，指定属性受定义的配置影响的XML标记。

  • 本地名称 ：指定没有名称空间前缀的标签名称。使用正则表达式指定多个标签名称name1|name2 ，不区分大小写的名称（ (?i)tagname匹配tagname和TagName ），依此类推。

    不允许使用空格字符，因为它们会影响匹配结果。

  • 命名空间 -在此字段中，指定XML标签的命名空间URI。

这两个字段都是可选的。但是，如果“ 本地名称”字段为空，则该配置将应用于与配置的名称匹配的任何属性，无论其包含XML标记如何。

XML属性

在此区域中，定义XML标签属性，该属性指示带有该属性的标签中包含的文本应被视为所选语言。

  • 本地名称 ：指定没有名称空间前缀的属性名称。使用正则表达式：例如，要匹配HTML事件处理程序属性，请键入on.*在该领域。

    该字段是可选的，除非XML Tag区域中的Local name字段为空。如果未指定属性本地名称，则该配置将应用于封闭标签的所有属性。

  • 命名空间 -在此字段中，指定属性的命名空间URI。

高级

在该区域中，指定其他设置以启用对注射过程的更细粒度控制。

  • 值模式 ：键入一个正则表达式，该表达式确定将语言注入其中的属性值的一部分。通过使用模式的第一个捕获组作为注入目标，您可以将过程配置为仅将语言注入到与特定模式匹配的值或与模式匹配的多个部分中。

    • 单个文件 ： 如果关闭该选项，则与值模式匹配的片段将被分别视为不同的“文件”，例如，从片段编辑器的角度来看。

      如果启用此选项，则所有相应的片段都将合并在一起以形成一个单元或“文件”。

      给定价值模式

      xxx (.+) yyy (.+) zzz

      和片段

      xxx select * yyy from family zzz ，

      select *和from family如果关闭该选项，则被视为两个独立的片段（或“文件”）。如果该选项打开， select * from family被视为一个单位或“文件”。

  • XPath条件 ：在此字段中，指定XPath表达式以更精确地解决注入目标。计算表达式的上下文是属性本身。

    为了使该字段处于活动状态，必须启用XPathView + XSLT支持插件 。
