代码完成

本节介绍了各种上下文感知的代码完成技术，这些技术可帮助您加快编码过程。

基本完成

基本代码完成功能可帮助您完成可见性范围内的类型，接口，方法和关键字的名称。当您调用代码完成功能时，GoLand会分析上下文并建议从当前插入符号位置可以到达的选择（建议还包括Live模板 ）。

非英语键盘布局也可以使用基本完成功能。

代码完成功能可用于自定义文件类型 。但是，GoLand无法识别此类文件的结构，因此会建议选项，无论它们在当前情况下是否合适。

如果将基本代码补全应用于字段，参数或变量声明的一部分，则GoLand会根据项目类型建议可能的名称列表。

调用基本完成

默认情况下，GoLand在您键入时自动显示代码完成弹出窗口。或者，您可以按Ctrl +空格键或选择“代码” |“选择”。完成|从主菜单基本 。

功能的代码完成

调用功能完成

您可以查看并完成适用于特定类型值的函数。

例如，您具有字符串类型的t变量。当您键入t. ，按住Ctrl并按两次Space ，您会看到一个接受string类型作为第一个参数的函数列表。

智能完成

调用智能完成

智能代码完成功能会过滤建议列表，并仅显示适用于当前上下文的类型。要调用智能完成功能，请开始输入代码，然后按Ctrl + Shift + Space 。或者，选择代码|完成|从主菜单中选择SmartType 。以下动画显示了基本完成和智能完成之间的区别。

声明完成

您可以使用语句完成来创建语法正确的代码构造。它插入必要的语法元素（括号，花括号和分号），并使您处于可以开始键入下一条语句的位置。要调用语句完成，请开始键入代码构造，然后按Ctrl + Shift + Enter 。

嬉皮完成

嬉皮补全是一种补全引擎，它可以在可见范围内分析您的文本并根据当前上下文生成建议。它可以帮助您从任何当前打开的文件中补全单词。

将插入符号处的字符串扩展为现有单词

  1. 键入初始字符串，然后执行以下操作之一：

    • 按Alt + /或选择代码|完成|循环展开单词可在插入符号前搜索匹配的单词。

    • 按Shift + Alt + /或选择代码|完成|循环扩展单词（向后）可在插入符号之后和其他打开的文件中搜索匹配的单词。

    出现第一个建议值，并且原型在源代码中突出显示。

  2. 接受建议，或按住Alt键并持续按/直到找到所需的单词。

后缀代码完成

使用后缀模板，您可以根据点后的后缀，表达式的类型及其上下文，将已经键入的表达式转换为其他表达式。GoLand包括一组预定义的修复后完成模板，您也可以将它们用于其他编程语言。您可以复制，启用和禁用这些模板。

Postfix完成选项显示为基本完成建议列表的一部分。要查看适用于当前上下文的后缀补全的完整列表，请按Ctrl + J。

  • 要查看可用的后缀模板的完整列表， 在里面 设置/首选项对话框Ctrl + Alt + S ，转至编辑器|一般|后缀完成 。

  • 选择Tab ， Space或Enter来展开后缀模板。

用后缀转换语句

键入一个表达式，然后在一个点后键入一个后缀，例如， .if:

func m（b bool）{b.if}

初始表达式包含if语句：

要禁用某些后缀完成模板， 在里面 设置/首选项对话框Ctrl + Alt + S选择编辑器|一般|后缀完成 。

您可以选择Tab ， Space或Enter来展开后缀模板。

您可以编辑预定义的后缀模板，例如，将长键替换为较短的后缀模板，或扩展适用的表达式类型列表。

创建自定义的后缀模板

  1. 在里面 设置/首选项对话框Ctrl + Alt + S ，转至编辑器|一般|后缀完成 。

  2. 请点击在工具栏上。

  3. 指定Key ，即将调用模板的符号的组合，选择新模板将适用于的表达式类型，并以以下格式键入目标表达式： $EXPR$ ，例如$EXPR$ != nil 。

    在模板内部，您可以使用两个预定义变量$EXPR$和$END$ ：

    • $EXPR$捕获点之前的表达式。

    • $END$定义扩展模板后光标的位置。

    “ 应用于最上方的表达式”复选框会将模板应用于整个表达式，而忽略范围。因此，您不必每次都选择模板的范围。

标签和属性的完成

GoLand自动完成HTML / XHTML，XML / XSL和JSON中标记和属性的名称和值。

标记和属性名称的完成取决于与文件关联的DTD或Schema 。如果没有模式关联，GoLand将使用文件内容（标记和属性名称及其值）来完成输入。

配置代码完成设置

配置完成选项

  1. 在里面 设置/首选项对话框Ctrl + Alt + S ，选择编辑器|一般|代码完成 。

    • 要自动显示建议列表 ，请选中“ 键入时显示建议”复选框。如果清除该复选框，则必须通过按Ctrl + Space进行基本完成或按Ctrl + Shift + Space进行智能完成来显式调用代码完成。

      您也可以选择只有一种选择时自动插入建议：在中为自动插入单个建议下选择完成类型。

    • 要按字母顺序对建议进行排序 ，而不是按相关性对建议进行排序 ，请选中按字母顺序对建议进行复选框。

      您也可以通过以下方式切换这些模式要么分别在建议列表的右下角。

      仅当列表很长时才会显示排序图标，而对于仅包含少量条目的列表则不会显示。

    • 如果你想为你滚动它的建议列表中每个项目自动显示文档弹出式窗口 中 ，选择选项显示文档弹出式窗口 。在右侧的字段中，指定延迟（以毫秒为单位），之后应显示弹出窗口。

完成提示和技巧

缩小建议列表

您可以通过键入单词的任何部分（甚至是中间位置的字符）或在点分隔符后调用代码完成来缩小建议列表的范围。GoLand将显示建议，其中包括您在任何位置输入的字符。

这使得不需要使用通配符：

如果使用CamelCase或snake_case名称，请仅输入首字母。GoLand自动识别并匹配首字母。

接受建议

您可以通过以下方式之一接受列表中的建议：

  • 按Enter或双击一个列表项，将其插入到插入符号的左侧。

  • 按Tab键以替换插入符号右侧的字符。

  • 使用Ctrl + Shift + Enter可使当前代码结构在语法上正确（平衡括号，添加缺失的花括号和分号，等等）。

您还可以使用特定的键插入所选的完成建议： 在里面 设置/首选项对话框Ctrl + Alt + S选择编辑器|一般|代码完成，然后按空格，点或其他上下文相关的键选项，选择“ 插入所选建议” 。这些键取决于语言，您的上下文等。

虽然此设置可以帮助您节省时间，但是将其打开可能会导致意外插入项目。

从建议列表中选择一个条目后，您可以查看代码层次结构：

  • Ctrl + Shift + H-查看方法层次结构。

填写结构域

创建新结构时，可以自动为其生成字段名称。

  1. 开始输入结构名称，从完成列表中选择它。

  2. 将插入号放在结构的花括号之间。

  3. 按Alt + Enter 。

  4. 在以下选项之间选择：

    • 填写所有字段 ：为属于当前结构的所有字段生成值。

    • 递归地填充所有字段 ： 递归地为属于当前结构类型的所有字段和指向其他结构类型的所有指针生成值。

    • 填写字段 ：显示“ 选择字段”对话框，您可以在其中选择要添加的字段。递归选择以解析所有相关结构中的字段。清除“ 强制多行”复选框以在一行上显示所有字段。在“ 选择字段”对话框中，按Ctrl ，然后单击要添加的字段。

  5. 单击确定 。

故障排除

如果无法完成代码，则可能是由于以下原因之一：

  • 省电模式已打开（ 文件|省电模式 ）。开启它可以消除背景操作，包括错误突出显示，即时检查和代码完成，从而将笔记本电脑的功耗降至最低。

  • 未为您的项目配置SDK。

  • 您的文件不驻留在内容根目录中，因此它没有获得代码完成所需的类定义和资源。

  • 包含要在完成建议列表中显示的类型和功能的文件被标记为纯文本文件。

  • 包含要显示在完成建议列表中的功能的外部库不会作为依赖项或全局库添加 。
