GoLand 2019.3帮助

管理插件

插件扩展了GoLand的核心功能：

  • 提供与版本控制系统，应用程序服务器和其他工具的集成

  • 添加对各种语言和框架的编码帮助支持

  • 通过快捷方式提示，实时预览，文件查看器等提高工作效率

  • 通过编码练习和验证帮助您学习新的编程语言

打开插件设置

  • 在里面 设置/首选项对话框Ctrl+Alt+S ，选择插件 。

在“ 插件”页面上，可以执行以下操作：

  • 使用“ 市场”选项卡从JetBrains插件存储库或自定义插件存储库中浏览和安装插件。

  • 使用“ 已安装”选项卡浏览已安装的插件，启用，禁用，更新或删除它们。禁用不必要的插件可以提高性能。

大多数插件都可以与任何JetBrains产品一起使用。有些只限于商业产品。还有一些插件需要单独的许可证。

如果某个插件依赖于其他插件，GoLand会通知您相关性。如果您的项目依赖某些插件，请将它们添加到所需插件列表中。

如果现有插件未涵盖某些功能，则可以为GoLand创建自己的插件。有关信息，请参阅开发自己的插件 。

默认情况下，GoLand包含许多捆绑的插件。您可以禁用捆绑的插件，但是不能删除它们。您可以从插件存储库或本地存档文件（ZIP或JAR）安装其他插件。

从存储库安装插件

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择插件 。

  2. 在市场中找到插件，然后单击“ 安装” 。

从磁盘安装插件

下载插件档案文件（ZIP或JAR）后，请执行以下操作：

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择插件 。

  2. 在“ 插件”页面上，单击然后单击“ 从磁盘安装插件” 。

  3. 选择插件存档文件，然后单击“ 确定” 。

  4. 单击“ 确定”以应用更改并在出现提示时重新启动IDE。

您也可以将下载的插件存档文件拖放到GoLand Welcome屏幕中。

删除插件

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择插件 。

  2. 打开“ 安装”选项卡，找到要删除的插件。

  3. 请点击 禁用 / 启用按钮旁边，然后从下拉菜单中选择卸载 。

自定义插件存储库

默认情况下，GoLand被配置为使用JetBrains插件存储库中的插件 。这是所有社区插件的托管地，您可以在那里免费托管插件。但是，如果您仅开发供内部使用的插件，则可以为其建立自定义插件存储库。

有关设置自定义插件存储库的信息，请参阅IntelliJ Platform SDK文档 。

设置插件存储库后，将其添加到GoLand：

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择插件 。

  2. 在“ 插件”页面上，单击然后单击管理插件存储库 。

  3. 在“ 自定义插件存储库”对话框中，单击并指定您的存储库URL。它必须指向updatePlugins.xml文件的位置。该文件可以与您的自定义插件位于同一服务器上，也可以位于专用服务器上。

  4. 在“ 自定义插件存储库”对话框中单击“ 确定 ”以保存插件存储库的列表。

  5. 在“ 设置/首选项”对话框中单击“ 确定 ”以应用更改。

要浏览自定义插件存储库，请输入repository:在“ 插件”页面的“ 市场”选项卡中，后跟存储库的URL。例如：

资料库：“ http://plugins.example.com:8080/updatePlugins.xml” myPlugin

或者，您可以使用自定义存储库URL替换默认的JetBrains插件存储库。如果您只希望GoLand提供自定义存储库插件，这将很有帮助。为此，请按照以下说明编辑平台属性或VM选项文件。

用自定义插件替换默认插件存储库

  1. 从主菜单中，选择“ 帮助” | “ 帮助”。编辑自定义属性或帮助|编辑自定义VM选项 。

  2. 添加idea.plugins.host平台属性文件或-Didea.plugins.host VM选项文件中的选项。例如：

    idea.plugins.host =“ http://plugins.example.com:8080/”
    -Didea.plugins.host =“ http://plugins.example.com:8080/”

    要添加多个URL，请用分号将它们分开; 。

    确保没有plugins.jetbrains.com URL。

  3. 重新启动GoLand。

如果将默认插件存储库替换为自定义存储库，则“ 插件”对话框的“ 市场”选项卡上的搜索字段将仅浏览自定义存储库中的插件。

开发自己的插件

您可以使用任何版本的GoLand来开发插件。它提供了开放的API，专用的SDK，模块以及运行/调试配置，以帮助您。

推荐的工作流程是使用Gradle。还支持使用内部GoLand构建系统的旧工作流程。有关更多信息，请参见《 IntelliJ Platform SDK开发人员指南》 。
