从“实例文档”对话框生成架构

工具| XML操作|从XML文件生成XSD架构

在此对话框中，指定用于生成描述所需XML文件结构的XSD（XML架构定义） 架构的选项。

在活动编辑器选项卡中打开XML文档时，菜单项和对话框可用。

项目

描述

实例文档路径

在此文本框中，指定要用作Schema生成基础的XML文档的完整路径。
默认情况下，该字段显示当前文件的完整路径。要使用另一个XML文档，请单击“ 浏览”。 然后在打开的对话框中选择所需的文件。

结果架构文件名

在此文本框中，指定要生成的架构的输出文件的名称。默认情况下，生成的XSD Schema文件将继承所用实例文档的名称，并将具有XSD扩展名。如果为要生成的架构键入另一个名称，请确保扩展名正确。

生成的Schema的默认位置是源XML实例文档所在的目录。要指定其他位置，请单击浏览 然后在打开的对话框中选择所需的路径。

设计类型

使用此下拉列表可以指定如何声明元素和复杂类型。可用的选项有：
  • 局部元素/全局复杂类型
  • 本地元素/类型
  • 全局元素/局部类型

检测简单的内容类型

从此下拉列表中，选择要呈现叶文本的类型，该类型应与使用的文本区分开。可用的选项有：
  • 串
  • 聪明

检测枚举限制

在此文本框中，键入导致架构枚举出现的出现次数。若要取消架构枚举，请指定0。
