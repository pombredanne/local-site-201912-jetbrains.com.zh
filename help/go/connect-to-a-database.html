如何从GoLand运行PostgreSQL查询

要完成本教程，请确保已安装以下软件：

  • Git 2.20.1和更高版本-https : //git-scm.com/。

  • 泊坞窗18.09.0后来- https://www.docker.com/ 。

  • 2018.3的Goland后来- https://www.jetbrains.com/goland/download 。

步骤1。从存储库克隆文件

出于示例目的，我们将使用以下两个存储库：Sakila转储文件（ https://github.com/DataGrip/dumps ）和Docker撰写文件（ https://github.com/JetBrains/docker-env-compose） ）。 Sakila存储库包括生成Sakila数据库结构的脚本和向数据库对象添加数据的脚本。Docker存储库包含用于定义服务，网络和卷的YAML文件。

  1. 单击VCS | Git |克隆 。

  2. 在“ URL”字段中，复制并粘贴以下Web URL：

    • Sakila转储文件： https : //github.com/DataGrip/dumps.git

    • Docker撰写文件： https : //github.com/JetBrains/docker-env-compose.git

  3. 在目录字段中，单击文件夹图标（ ），并为存储库文件指定目录。

第2步。运行一个Docker容器

在本教程中，我们将为Docker运行PostgreSQL容器（来自docker-env-composit存储库）。您可以在docker-compose.yml中查看可用服务的完整列表。服务名称是服务定义的第一行（在JDBC URL之前）。在以下示例中，服务名称为pg11X ，JDBC URL为jdbc：postgresql：// localhost：54332 / guest？user = guest＆password = guest 。

pg11X：＃jdbc：postgresql：// localhost：54332 / guest？user = guest＆password = guest container_name：pg11X端口：-“ 54332：5432”图片：postgres：11-alpine env_file：.env健康检查：测试：“退出0”

  1. 打开命令行并导航至包含Docker YML文件的文件夹（例如， cd ~/database/docker-env-compose/ ）。

  2. 键入以下命令来运行容器： docker-compose up -d --no-recreate pg11X 。

  3. 按Enter 。

要查看所有正在运行的图像，请使用docker ps命令行中的命令。否则，请使用Docker工具窗口（ View | Tool Windows | Docker ）查看和管理Docker容器。

第三步连接到数据源

根据数据库供应商（MySQL，PostgreSQL，Oracle），您需要创建一个相应的数据源连接。在本教程中，我们将创建一个PostgreSQL连接。如果要连接到其他数据库管理系统（DBMS），请参阅“ 连接到数据库” 。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择PostgreSQL 。

  3. 在数据源设置区域的底部，单击“ 下载缺少的驱动程序文件”链接。或者，您可以为数据源指定用户驱动程序。有关用户驱动程序的更多信息，请参阅将用户驱动程序添加到现有连接 。

  4. 在“ URL”字段中，复制并粘贴JDBC URL： jdbc：postgresql：// localhost：54332 / guest？user = guest＆password = guest 。其他DBMS的JDBC URL，请参阅docker-compose.yml 。您可以在文本编辑器中打开docker-compose.yml 。

  5. （可选）在“ 名称”字段中，删除当前名称，然后为连接键入一个新名称（例如PostgreSQL）。

  6. 要确保成功连接到数据源，请单击“ 测试连接” 。

如果看到正在连接单击“测试连接”按钮时，对话框，指定guest作为密码，然后单击确定 。

第四步。将目录附加到项目

要在GoLand中运行项目中的转储文件，请附加包含转储文件的目录。

  1. 单击文件|打开 。

  2. 导航到要附加的文件夹。在本教程中，这个文件夹是转储 ，你克隆在步骤1（的Sakila转储文件）。

  3. 点击打开 。

  4. 在“ 打开项目”对话框中，单击“ 附加” 。

第五步运行转储文件

创建数据库结构

  1. 在“项目”（“ 视图” |“工具窗口” |“项目” ）工具窗口中，导航到postgres-sakila-db树节点。

  2. 展开postgres-sakila-db树节点。

  3. 右键单击postgres-sakila-schema.sql，然后选择Run postgres-sakila-schema .sql 。或者，单击postgres-sakila-schema.sql文件，然后按Ctrl+Shift+F10 。

  4. 在“ 执行目标”对话框中，选择“ PostgreSQL” ，然后单击“ 确定” 。

将样本数据加载到数据库

  1. 在“项目”（“ 视图” |“工具窗口” |“项目” ）工具窗口中，导航到postgres-sakila-db树节点。

  2. 展开postgres-sakila-db树节点。

  3. 右键单击postgres-sakila-insert-data.sql，然后选择Run postgres-sakila-insert-data.sql 。或者，单击postgres-sakila-insert-data.sql文件，然后按Ctrl+Shift+F10 。

  4. 在“ 执行目标”对话框中，选择“ PostgreSQL” ，然后单击“ 确定” 。

第六步从Go代码中获取依赖项

要将查询发布到PostgreSQL，我们将使用sqlx库，为Go标准提供了一组扩展database/sql图书馆。有关该库的更多信息，请阅读GitHub上的sqlx 。

  1. 在GoLand中创建一个Go项目。

  2. 右键单击项目中的根文件夹，然后导航到“ 新建|转到文件 。

  3. 在“ 新建文件”对话框中，键入main在名称字段中，单击确定 。

  4. 复制随附的代码段并将其粘贴到main.go文件中。

  5. 将插入符号放在“ github.com/jmoiron/sqlx”导入行上，按Alt+Enter然后单击go -t github.com/jmoiron/sqlx 。

步骤7运行配置

要运行配置，请单击“ 运行”图标（ ）在主功能附近的装订线中，然后选择运行'go build main.go' 。
