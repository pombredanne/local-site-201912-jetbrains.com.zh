使用Go模块创建项目（vgo）

的go get该命令从Go存储库的master分支下载最新版本的依赖项。在某些情况下，这种方法不起作用。例如，当您有两个项目依赖于第三方库的不同版本。使用Go模块（以前称为vgo），您可以下载和导入每个项目所需的库版本。在blog.golang.org上了解有关Go模块的更多信息。

使用Go模块（vgo）集成创建项目

  1. 选择文件|新品|项目 。

  2. 在“ 新建项目”页面中，选择“ 转到模块（vgo）” 。

  3. 在位置字段中，指定要在其中存储项目的路径。

  4. 在“ GOROOT”字段中，指定Go安装的位置。通常，位置是自动定义的。

    要更改或安装新版本的Go SDK，请单击“ 添加SDK”按钮，然后选择“ 本地”以在硬盘驱动器上选择Go SDK版本，或选择“ 下载”以从官方存储库下载Go SDK。有关安装Go SDK的更多信息，请参阅安装Go SDK 。

  5. 从“ Vgo可执行文件”列表中，选择要用于Go模块的可执行文件。

  6. 在代理列表中，为GOPROXY环境变量。您可以从下拉列表中选择一个选项（ 直接或关闭 ），也可以键入自己的值（例如， https://proxy.golang.org ）。

    直接值代表与VCS的直接连接（ GOPROXY=direct ）。另外，您可以将空值用于直接连接（ GOPROXY="" ）。

    off值表示不允许网络使用。

  7. （可选）要使用供应商模式，请选中供应商模式复选框。有关供应的更多信息，请参见供应商 。

  8. 点击创建 。

使用Go模块

在“ 项目”工具窗口（“ 视图” |“工具窗口” |“项目” ）中，将显示Go模块及其完整的导入路径。每个Go Module的版本均以静音字体显示，以帮助您区分它们。

使用Go模块获取依赖项（vgo）

如果项目错过了依赖，您可以使用Go模块（vgo）获取依赖。

  1. 单击依赖项声明。

  2. 按Alt+Enter并选择同步 。

如果您从go.mod文件中删除依赖关系，GoLand会自动从可用库列表中删除依赖关系。

在项目中启用Go模块（vgo）

一个新的Go模块项目已经启用了Go模块（vgo）。如果您从Github中提取Go模块（vgo）项目，则需要手动启用％project_ = type％。

  1. 打开设置Ctrl+Alt+S并导航到Go |去模块 。

  2. 选中“ 启用Go模块（vgo）集成”复选框，并确保“ Vgo可执行文件”字段指向有效的Project SDK目录。

  3. 单击确定 。

创建依赖关系图

  1. 在项目中启用Go模块（vgo）。

  2. 右键单击项目中的go.mod文件，然后选择Diagrams |显示图 。

go.mod文件列出了项目的依赖关系。

确保您的项目在GOROOT下。

启用供应商模式

启用供应商模式后，仅使用存储在项目中供应商目录下的依赖项软件包来构建应用程序。在这种模式下，Go命令将忽略go.mod中的依赖项描述，并假定供应商目录包含正确的依赖项副本。默认情况下，当您使用Go模块时，在构建过程中将忽略供应商目录。

  1. 在设置中Ctrl+Alt+S的Go modules（vgo）项目中，导航到Go |转到模块（vgo） 。

  2. 选择供应商模式复选框，然后单击确定 。

创建新的Go modules（vgo）项目时，请在“ 新建项目”对话框中选中“ 供应商模式”复选框。

从依赖项导入路径导航到包源文件

  1. 在“ 项目”工具窗口（“ 视图” |“工具窗口” |“项目” ）中，双击go.mod文件。

  2. 单击依赖项导入路径，然后单击“ 导航”。声明或用法 Ctrl+B 。

安装Go SDK

选择Go SDK的本地副本

确保与围棋SDK的文件夹提供的路径包括仓 и 的src的文件夹。

  1. 在“ 新建项目”对话框中，选择“ 转到模块（vgo）” 。

  2. 单击添加SDK按钮，然后选择本地 。

  3. 在文件浏览器中，导航到硬盘驱动器上的SDK版本。

  4. 点击打开 。

下载Go SDK

  1. 在“ 新建项目”对话框中，选择“ 转到模块（vgo）” 。

  2. 点击添加SDK按钮，选择下载 。

  3. 从版本列表中，选择SDK版本。

  4. 在位置字段中，指定SDK的路径。要使用文件浏览器，请单击“ 浏览”图标 。

  5. 单击确定 。
