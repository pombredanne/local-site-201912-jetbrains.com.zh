探索Go核心转储

有时，在调试程序时，您需要检查代码执行流程并了解程序的当前状态。Go核心转储是一个文件，其中包含正在运行的进程的内存转储及其在程序生命周期中的状态。您可以在程序完成执行或仍在运行时调试核心转储。

您只能在Linux机器上创建Go核心转储文件。但是您可以在任何支持GoLand的操作系统上查看转储文件。

在Linux上创建Go核心转储文件

  1. 使用文件在目录中打开一个终端。

  2. 设置ulimit参数unlimited ： ulimit -c unlimited 。

  3. 通过运行来构建程序go build .在终端。的build命令在当前项目文件夹中创建一个二进制文件（例如awesomeProject ）。

  4. 要创建核心转储文件，请运行GOTRACEBACK=crash ./ （例如， GOTRACEBACK=crash ./awesomeProject ）。此命令在当前项目文件夹中创建一个核心文件。

查看转储日志

  1. 导航至运行|打开核心转储 。

  2. 在“ 可执行文件”字段中，指定二进制文件的路径（例如awesomeProject ）。

  3. 在“ 核心转储”字段中，指定到core文件（例如core ）。

  4. 单击确定 。在“ 调试工具”窗口中，选择要检查的框架。

在GoLand中查看Go核心转储

  1. 打开或创建Go文件的Go Build配置。

  2. 在环境字段中，单击文件夹（ ）。

  3. 在“环境变量”对话框中，单击“ 添加”图标（ ）。

  4. 单击名称字段，然后键入GOTRACEBACK 。

  5. 单击值字段，然后键入crash 。

  6. 保存所有更改，然后单击“ 确定” 。

  7. 运行程序Shift+F10 。程序的输出显示在调试器窗口中。
