使用Mozilla rr进行调试

Mozilla rr是可用于记录，重播和调试应用程序的工具。Mozilla rr的主要思想是帮助您捕获不重要的错误。

Mozilla rr记录整个程序的执行情况。这意味着您只能在程序结束执行时调试记录的跟踪。对于服务器和其他长时间运行的应用程序，必须终止正在运行的应用程序（例如，通过从控制台发送SIGTERM信号）。录制后，您可以根据需要在调试器中重播执行次数。在Mozilla rr 官方网站上了解有关Mozilla rr的更多信息。

使用Mozilla rr调试代码

  1. 安装Mozilla rr。有关安装说明，请参阅《 构建和安装》 。

  2. 在GoLand中，设置一个断点。要设置断点，请单击您希望调试器停止执行代码的代码行附近的装订线。有关断点的更多信息，请参见调试代码和断点 。

  3. 点击运行图标（ ），然后选择“ 记录并调试” 。在“ 调试器”工具窗口中，您可以查看代码执行不同阶段的变量，进程和线程的状态。

浏览记录的轨迹

  1. 导航至运行|调试保存的跟踪 。

  2. 在“ 跟踪目录”字段中，指定跟踪目录的路径。

  3. 单击确定 。

  4. 在“ 调试器”工具窗口中，单击“ 恢复程序”图标继续执行程序，或单击下面的“ 倒带”图标向后运行调试会话，直到上一个断点为止。

收集Mozilla rr跟踪

  1. 使用Go构建运行/调试配置来构建可执行文件。要轻松找到可执行文件，请在Go构建配置的Output directory字段中指定当前项目目录。有关创建运行/调试配置的更多信息，请参阅创建运行/调试配置 。

  2. 打开终端并运行以下Mozilla rr命令：rr record

    结果，Mozilla rr跟踪文件出现在以下文件夹中： 〜/ .local / share / rr /

使用Mozilla rr进行远程调试

  1. 在远程计算机上， 收集Mozilla rr跟踪 。

  2. 在远程计算机上，通过打开终端并运行以下命令来启动调试器：dlv --headless --api-version=2 -l localhost:2345 replay /path/to/trace/dir /path/to/binary

  3. 在本地计算机上，创建“ 转到远程”运行/调试配置。在“ 转到远程”配置中，指定远程计算机的IP地址和端口。有关创建运行/调试配置的更多信息，请参阅创建运行/调试配置 。

  4. 在本地计算机上，确保在配置列表中选择了“远程运行/调试”配置。

  5. 在本地计算机上，单击运行|。除错 。或者，按Shift+F9 。
