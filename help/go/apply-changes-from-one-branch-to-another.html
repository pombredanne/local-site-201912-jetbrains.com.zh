将更改从一个分支应用于另一分支

在Git中，有几种方法可以将更改从一个分支集成到另一个分支： 合并分支 ，重新设置分支 ，或将一个分支的单独提交应用于另一个分支（cherry-pick） 。

在GoLand中，所有带有分支的操作都在Git Branches弹出窗口中执行。要调用它，请单击状态栏中的Git小部件或按Alt+`然后选择分支 。

合并分支

假设您已经创建了一个功能分支以执行特定任务，并想在完成并测试功能后将工作结果集成到主代码库中：

将分支合并到master是最常见的方法。

在功能分支中工作时，队友会继续将其工作交还给主人：

当你跑步merge ，来自功能分支的更改将集成到目标分支的HEAD中：

Git创建一个新的提交（M），称为合并提交，该合并提交是通过合并功能分支和主分支（从两个分支分开的点）进行的更改而产生的。

合并分支

  1. 在“ 分支”弹出窗口中，选择要将更改集成到的目标分支，然后从弹出菜单中选择“ 签出”以切换到该分支。

  2. 单击GoLand窗口底部的“ 分支”弹出窗口，选择要合并到目标分支的分支，然后从子菜单中选择“ 合并到当前 ”。

如果您的工作树是干净的（这意味着您没有未提交的更改），并且功能分支和目标分支之间没有冲突，则Git将合并两个分支，并且合并提交将出现在Log中 ：

如果分支与目标分支之间发生冲突，将提示您解决它们（请参阅解决冲突 ）。如果合并后还有未解决的冲突，则“ 合并冲突”节点将出现在合并中的相应更改列表中。 的“本地更改”标签 版本控制窗口Alt+9并提供解决问题的链接。

如果您有将被合并覆盖的本地更改，GoLand将建议执行Smart merge 。如果选择此选项，GoLand将存储未提交的更改，执行合并，然后取消存储更改。

您可以通过从Git Branches弹出窗口中选择Abort操作来取消未完成的合并操作。

将当前分支重新置于另一个分支之上

  1. 在“ 分支”弹出窗口中，选择要将当前分支作为基础的分支。

  2. 从弹出菜单中选择“将当前基准重新设置为已选定 ”。

在当前分支之上重新建立分支

  1. 在“ 分支”弹出窗口中，选择要在当前分支之上重新建立基础的分支。

  2. 从弹出菜单中选择“ 签出并重新设置为当前 ”。

您可以通过从Git Branches弹出窗口的顶部分别选择Abort或Continue操作来取消未完成的变基操作或恢复中断的变基。

Cherry-pick单独提交

有时，您只需要对不同分支应用单个提交，而无需重新定级或合并整个分支。例如，如果您在功能分支中工作，并且希望集成来自两个分支分开之后提交的master的修补程序，这可能很有用。或者，您可能需要将修补程序反向移植到以前的版本分支。您可以通过使用Cherry-pick操作来实现。

樱桃选择操作的状态显示在状态栏中。您始终可以通过在Git Branches弹出窗口中选择Abort Cherry-Pick来中止正在进行的Cherry-Pick 。

将提交应用于另一个分支

  1. 在“ 分支”弹出窗口中，选择要将更改集成到的目标分支，然后从弹出菜单中选择“ 签出”以切换到该分支。

  2. 打开 版本控制窗口Alt+9并切换到“ 存储库”选项卡。

  3. 找到包含您要挑选的更改的提交。

    您可以按分支机构，用户或日期过滤提交。您也可以点击在工具栏上，然后选择突出显示|未选择提交按钮使已经应用到当前分支的提交变灰。如果您知道提交哈希，或者正在寻找带标记的提交，则还可以使用“ 转到哈希/分支/标记”操作（按Ctrl+F在日志视图中，或单击在工具栏上）。

  4. 选择所需的提交。使用“ 提交详细信息”区域中的信息来确保这些是您要转移到另一个分支的更改。

  5. 单击“ 领料”按钮在工具栏上。GoLand将显示带有自动生成的提交消息的“ 提交更改”对话框。如果要在将更改提交到目标分支之前查看更改或什至修改代码，可以在此对话框中提供的差异查看器中进行操作。

  6. 完成后，单击“ 提交”以选择所选更改。

    请注意，如果您单击“ 取消” ，则会创建一个单独的变更列表，其中包含您可以在 的“本地更改”标签 版本控制窗口Alt+9 。您可以查看这些更改，并在以后根据需要提交。

  7. 将更改推送到目标分支。

套用其他变更

想象一下，您已经对要应用到其他分支的文件进行了一些更改，但是这些更改是与其他已修改文件一起提交的。GoLand允许您应用单独的更改，而不是挑选整个提交。

  1. 在“ 分支”弹出窗口中，选择要将更改集成到的目标分支，然后从弹出菜单中选择“ 签出”以切换到该分支。

  2. 打开 版本控制窗口Alt+9并切换到“ 存储库”选项卡。

  3. 找到包含要应用的更改的提交。

    您可以按分支机构，用户或日期过滤提交。您也可以点击在工具栏上，然后选择突出显示|未选择提交按钮使已经应用到当前分支的提交变灰。如果您知道提交哈希，或者正在寻找带标记的提交，则还可以使用“ 转到哈希/分支/标记”操作（按Ctrl+F在日志视图中，或单击在工具栏上）。

  4. 在右侧的“ 提交详细信息”窗格中，选择包含要应用到目标分支的更改的文件，然后从上下文菜单中选择“ 应用选定的更改 ”。

  5. 在打开的对话框中，选择一个现有的变更列表或输入新变更列表的名称，然后单击“ 确定” 。

  6. 提交更改，然后将其推送到目标分支。

应用单独的文件

除了对单个文件应用单独的更改之外，您还可以将整个文件的内容复制到另一个分支。例如，如果要应用的文件在目标分支中不存在，或者在几次提交中对文件进行了更改，这可能会很有用。

  1. 签出将应用更改的分支。

  2. 在“ 分支”弹出窗口中，单击包含要应用的文件的分支，然后选择“ 与当前比较” 。

  3. 切换到“ 文件”选项卡，选择要应用于当前分支的文件，然后从上下文菜单中选择“从分支获取” 。

  4. 提交并推送更改。GoLand会将文件的全部内容复制到当前分支。
