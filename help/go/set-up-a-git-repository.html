设置一个Git仓库

从远程主机（克隆）检出项目

GoLand允许您检出（用Git术语clone ）一个现有存储库，并根据您下载的数据创建一个新项目。

  1. 从主菜单中，选择VCS |。从版本控制获取 ，或者，如果当前未打开任何项目，请在“ 欢迎”屏幕上单击“从版本控制获取” 。

  2. 在“ 从版本控制中获取”对话框中，指定要克隆的远程存储库的URL（可以单击“ 测试”以确保可以建立到远程的连接），或选择左侧的VCS托管服务。如果您已经登录到选定的托管服务，则完成将提示您可以克隆的可用存储库的列表。

  3. 单击克隆 。如果要基于已克隆的源创建GoLand项目，请在确认对话框中单击“ 是 ”。Git根映射将自动设置为项目根目录。

    如果您的项目包含子模块 ，它们也将被克隆并自动注册为项目根目录。

将现有项目置于Git版本控制下

您可以基于现有项目源创建本地Git存储库。

将整个项目与单个Git存储库相关联

  1. 打开要放在Git下的项目。

  2. 从“ VCS操作”弹出窗口中选择“ 启用版本控制集成 ” Alt+`或从VCS主菜单中选择。

  3. 选择Git作为版本控制系统，然后单击“ 确定” 。

  4. 启用VCS集成后，GoLand将询问您是否要通过VCS共享项目设置文件。您可以选择“ 始终添加”以与使用GoLand的其他存储库用户同步项目设置。

    注意，这仅适用于Git和Mercurial。

将项目中的不同目录与不同的Git存储库相关联

  1. 打开要放在Git下的项目。

  2. 从主菜单中，选择VCS |。导入版本控制|创建Git存储库 。

  3. 在打开的对话框中，指定将在其中创建新Git存储库的目录。

    Git不支持外部路径，因此，如果您选择项目根目录之外的目录，请确保要在其中创建存储库的文件夹也包含项目根目录。

  4. 如果要在项目结构中创建多个Git存储库，请对每个目录重复上述步骤。

为项目初始化Git存储库后 ，需要将项目文件添加到存储库。

将文件添加到本地存储库

  1. 打开 版本控制窗口Alt+9并切换到“ 本地更改”标签。

  2. 通过按将任何文件放在版本控制下的“无版本文件” 更改列表中Ctrl+Alt+A或从上下文菜单中选择“ 添加到VCS ”。您可以添加整个更改列表，也可以选择单独的文件。

    您还可以从“项目”工具窗口将文件添加到本地Git存储库中-选择要添加的文件，然后按Ctrl+Alt+A或选择Git |从上下文菜单中添加 。

在项目中启用Git集成后，GoLand建议在Git下添加每个新创建的文件。您可以在 设置/首选项对话框Ctrl+Alt+S在版本控制下|确认 。如果您希望某些文件始终保持未版本控制，则可以忽略它们 。

从版本控制中排除文件（忽略）

有时您可能需要保留某些文件的版本。这些可以是VCS管理文件，实用程序，备份副本等。您可以通过GoLand忽略文件，IDE不会建议将它们添加到Git中，而是将其突出显示为被忽略。

您只能忽略未版本控制的文件，即您在“ 未版本控制的文件” 更改列表中看到的文件 。如果文件已添加到Git但未提交 ，则可以在文件中右键单击该文件。 的“本地更改”标签 版本控制窗口Alt+9然后选择回滚 。

Git允许您在两种配置文件中列出被忽略的文件模式：

  • .git / info / exclude文件。
    此文件中列出的模式仅适用于存储库的本地副本。

    初始化或签出Git存储库时会自动创建此文件。

  • VCS根目录及其子目录中的一个或多个.gitignore文件。
    这些文件被检入存储库，以便整个团队都可以使用其中的忽略模式。因此，这是存储忽略文件模式的最常见的地方。

    如果VCS根目录中没有.gitignore文件，则可以在“项目”窗口中的任何位置单击鼠标右键，然后选择“ 新建” |“新建”。在“ 新建文件”对话框中输入文件并键入.gitignore 。

    要在Windows资源管理器中创建.gitignore文件，请创建一个名为.gitignore的文件。 Windows会自动将其重命名为.gitignore 。

将文件添加到.gitignore或.git / info / exclude

  1. 确定要用于忽略文件的Git配置文件类型 。如有疑问，请使用.gitignore 。

  2. 在目录中找到要忽略的未版本化文件或文件夹 的“本地更改”标签 版本控制窗口Alt+9或在“项目”工具窗口中。这些视图中的文件颜色可帮助您识别文件的状态。

  3. 右键单击所选内容，然后选择Git |。添加到.gitignore或Git |添加到.git / info / exclude 。
    这些视图中的文件颜色可帮助您识别文件的状态。

如果您需要按某种模式或某种类型的文件排除文件，则可以编辑.gitignore要么.git/info/exclude直接归档。请参阅.gitignore模式格式

如果要忽略的文件也显示在 的“本地更改”标签 版本控制窗口Alt+9 ，点击在工具栏上，选择显示忽略的文件 。

检查项目状态

GoLand允许您检查本地工作副本与项目存储库版本相比的状态。它使用特定的颜色让您查看哪些文件已被修改，哪些新文件已添加到VCS，以及哪些文件未被Git跟踪。

打开 版本控制窗口Alt+9并切换到“ 本地更改”标签。

  • 默认更改列表显示自上次与远程存储库同步以来已修改的所有文件（以蓝色突出显示），以及已添加到VCS但尚未提交的所有新文件（以绿色突出显示）。

  • “ 未版本控制的文件 ” 更改列表显示所有已添加到项目中但未被Git跟踪的文件。

如果合并期间存在尚未解决的冲突 ，则“ 合并冲突”节点将显示在相应的更改列表中，并带有一个链接来解决这些冲突 。

有关变更列表的更多信息，请参阅将变更分组为不同的变更列表 。

添加一个远程仓库

为了能够在您的Git项目上进行协作，您需要配置要从中获取数据并在需要共享工作时将其推送到的远程存储库。

如果您已从GitHub 克隆了一个远程Git存储库 ，例如该远程库是自动配置的，则在您要与之同步时（即执行拉或推操作时）不必指定它。Git给您从其克隆的遥控器的默认名称是origin 。

但是，如果您基于本地资源创建了一个Git存储库 ，则需要添加一个远程存储库，以使其他贡献者能够将其更改推送到该存储库，并能够共享您的工作结果。

定义一个遥控器

  1. 在任何Git托管（例如Bitbucket或GitHub）上创建一个空的存储库。

    您可以在GitHub上创建存储库而无需离开GoLand：请参阅在GitHub上共享项目 。

  2. 准备好通过选择VCS | Push提交时调用Push对话框。 Git |从主菜单，或按推 Ctrl+Shift+K 。

  3. 如果您到目前为止尚未添加任何远程，则将显示“ 定义远程”链接，而不是远程名称。单击它添加一个遥控器。

  4. 在打开的对话框中，指定远程名称和将在其上托管的URL，然后单击“ 确定” 。

在某些情况下，您还需要添加另一个远程存储库。例如，如果您克隆了一个没有写访问权限的存储库，并且打算将更改推送到原始项目的自己的分支中，则这可能很有用。另一个常见的情况是，您已经克隆了自己的存储库，该存储库是其他人的项目分支，并且您需要与原始项目同步并从中获取更改。

添加第二个遥控器

  1. 从主菜单中，选择VCS |。 Git |遥控器 。Git Remotes对话框将打开。

  2. 点击添加 工具栏上的按钮或按Alt+Insert 。

  3. 在打开的对话框中，指定远程名称和URL，然后单击“ 确定” 。


要编辑一个遥控器（例如，要更改您克隆的原始项目的名称），请在“ Git遥控器”对话框中选择它，然后单击“ 编辑”。 工具栏上的按钮，或按Enter 。

您也可以通过在“ 推送对话框”中单击其名称来编辑它。

要删除不再有效的存储库，请在Git Remotes对话框中选择它，然后单击删除。 工具栏上的按钮，或按Alt+Delete 。
