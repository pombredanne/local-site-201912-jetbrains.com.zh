代码检查

在GoLand中，有一组代码检查可以在编译项目之前检测并更正项目中的异常代码。IDE可以发现并突出显示各种问题，找到死代码，查找可能的错误，拼写问题，并改善整体代码结构。

检查可以扫描所有项目文件中或仅在特定范围内的代码（例如，仅在生产代码中或在修改的文件中）。

每次检查都有一个严重级别 ，即问题可能影响您的代码的程度。严重性在编辑器中以不同的方式突出显示，因此您可以快速区分关键问题和次要问题。GoLand带有一组预定义的严重性级别，使您可以创建自己的严重性级别。

检查及其设置在配置文件中分组。每个概要文件都包含有关已启用检查的信息，它们分析的文件范围以及严重性级别。

访问所有可用的检查及其设置

  • 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|检查 。

    您也可以点击Ctrl+Shift+Alt+H在状态栏中（编辑器右下角），然后点击配置检查 。

使用筛选检查清单。例如，您可以按严重性或语言过滤检查。

选择默认情况下禁用新检查复选框以禁用来自已安装插件的新检查，因为它们可能会影响检查配置文件的配置。

代码检查示例

要查看可用检查的列表，请打开设置Ctrl+Alt+S并导航到编辑器|检查 。

无法访问的代码

无法访问代码检查将检测无法执行的部分代码。

未处理的错误

未处理的错误代码检查会警告您有关可能返回代码中未处理的错误值的函数或方法。在设置中，您可以更改每个范围的严重性级别，并在必要时排除此检查的功能名称或方法名称。

零度分析仪

报告由于使用不当而导致的问题nil值。分析数据流以确定变量是否具有nil或不nil价值观。例如，使用零度分析仪，您可以捕获以下情况：

  • 方法调用nil可能导致nil指针取消引用的接收器。

  • 索引nil可能引起恐慌的切片。

  • 比较像v == nil如果没有意义v总是nil或不nil 。

  • 使用没有正确错误处理就使用值的运算。例如，在类似f, err := foo() ，GoLand着重强调了潜在的危险操作f什么时候err不处理。

有关nilness的更多信息，请参阅JetBrains关于nilness Analyzer的博客文章。

冗余类型转换

冗余类型转换检查会警告您有关可以省略的冗余类型转换。考虑下面的代码示例。

s：= string（“”）_ = s

代码将字符串文字转换为字符串。这是多余的操作，因为无论如何输出都是字符串。
