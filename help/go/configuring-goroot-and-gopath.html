配置GOROOT和GOPATH

Go工具需要源代码的特定布局。GOROOT和GOPATH是定义此布局的环境变量。

GOROOT是一个变量，用于定义Go SDK的位置。除非计划使用其他Go版本，否则无需更改此变量。

GOPATH是定义工作空间根目录的变量。在GOPATH下，您可以存储代码库以及开发所需的所有文件。的go get工具会将软件包下载到GOPATH中指定的第一个工作场所。如果未设置GOPATH，则默认值为〜/ go （Linux和MacOS）和Windows上的％USERPROFILE％/ go 。GOPATH是工作空间的根，并包含以下文件夹：

  • src / ：源文件的位置（例如.go ， .c ， .g ， .s ）。

  • pkg / ：已编译软件包的位置（例如.a ）。

  • bin / ：Go生成的可执行文件的位置。

根

配置GOROOT

要查看GOROOT的当前值，请打开设置（ Ctrl+Alt+S ）并导航到Go | GOROOT 。单击下拉列表，然后选择Go版本。

如果没有可用的Go版本，请单击添加SDK按钮以下载Go版本或选择Go SDK本地副本的路径 。

选择Go SDK的本地副本

确保与围棋SDK的文件夹提供的路径包括仓 и 的src的文件夹。

  1. 打开设置 （ Ctrl+Alt+S ）并导航到Go | GOROOT 。

  2. 点击添加SDK按钮 ，然后选择Local 。

  3. 在文件浏览器中，导航到硬盘驱动器上的SDK版本。

  4. 点击打开 。

下载Go SDK

  1. 打开设置 （ Ctrl+Alt+S ）并导航到Go | GOROOT 。

  2. 点击添加SDK按钮（ ），然后选择下载 。

  3. 从版本列表中，选择SDK版本。

  4. 在位置字段中，指定SDK的路径。要使用文件浏览器，请单击“ 浏览”图标 。

  5. 单击确定 。

GOPATH

为不同的范围配置GOPATH

您可以为以下范围配置GOPATH：全局，项目和模块。如果您指定了所有三个范围，则GoLand首先选择最具体的范围。

  1. 打开设置 （ Ctrl+Alt+S ）并导航到Go | GOPATH 。

  2. 根据您要应用的范围，选择相应的部分（全局GOPATH，项目GOPATH，模块GOPATH），然后单击添加按钮 。

  3. 导航到要与GOPATH关联的目录。

  4. 单击确定 。
