GoLand 2019.3帮助

CPU分析器

CPU Profiler显示哪些功能消耗了多少CPU时间。此信息可以使您更好地了解应用程序的执行方式以及资源的分配方式。

分析完成后，探查器会在报告中可视化输出数据。

探查器配置

GoLand具有两个预定义的Async Profiler配置：CPU Profiler和内存分配Profiler，您可以在Settings / Preferences |构建，执行，部署| Java Profiler 。

调整了这些配置以提供最准确的结果，这就是为什么它们不需要任何修改的原因。但是，如果您仍然需要更改设置，请参阅GitHub上的Async Profiler了解更多信息。

选择探查器配置

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择构建，执行，部署|转到Profiler 。

  2. 选择一种预定义的CPU或内存配置文件配置，或者通过单击创建一个新的配置 。

  3. 应用更改并关闭对话框。

运行分析

运行CPU分析

打开_test.go文件。在要分析的函数或方法附近，单击“ 运行应用程序”图标在装订线区域中，选择“ 使用'CPU Profiler'运行” 。

解释结果

在Profiler工具窗口中，收集的数据显示在三个选项卡上： Flame Graph ， Call Tree和Method List 。

火焰图

“ 火焰图”选项卡显示您的函数调用以及每个调用执行时间的百分比。每个块代表堆栈中的一个功能（堆栈框架）。在Y轴上，从下往上有一个堆栈深度。X轴显示堆栈配置文件，从CPU消耗最大的功能到消耗最少的功能排序。

阅读火焰图时，请注意最宽的方块。这些块是配置文件中最常显示的功能。

将鼠标悬停在任何块上可以查看详细信息：

有关火焰图的更多信息，请参见火焰图 。

呼叫树

“ 调用树”选项卡表示有关在概要分析期间采样的程序的调用堆栈的信息：

  • 方法名称

  • 总采样时间的百分比（可以切换到父级调用视图）

  • 总样本数

  • 过滤的呼叫数

百分比后面的可选数字表示已过滤的呼叫序列。单击以展开此序列。可以在设置中配置过滤规则。

要配置和过滤“呼叫树”视图，请使用“ 演示设置”按钮（ ）。

方法清单

方法列表收集配置文件数据中的所有方法，并按累计采样时间对它们进行排序。此列表中的每个项目都有一个Back Trace树和一个Merge Callees树。

返回跟踪显示了呼叫者的层次结构。使用此视图可跟踪哪些方法调用了所选方法。合并被调用者尝试汇总由所选方法调用的所有方法。
