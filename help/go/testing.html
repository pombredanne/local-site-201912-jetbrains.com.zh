测试应用

测试可确保您的代码行为正确且符合预期。编写良好的测试使您可以放心地开发自己的代码，并且不会破坏现有代码。

GoLand包括几个可用于运行测试的软件包。另外，您可以为所选软件包应用特定的测试范围。

测试包

出于测试目的，GoLand包括以下软件包：

est

用于运行标准单元测试。有关更多信息go test请参阅软件包测试 。

gocheck

用于扩展以下功能go check并运行更复杂的测试。有关更多信息go check ，请参阅检查 。

高脚凳

用于运行性能测试。有关更多信息gobench ，请参阅包装测试：基准 。

测试范围

根据要测试的范围，可以为文件，目录和程序包创建运行/调试配置。

目录

应用程序文件和测试文件位于同一目录中（例如， applicationFolder / main.go和applicationFolder / main_test.go ）。请参阅GitHub存储库中的 main.go和main_test.go文件。

包

应用程序文件和测试文件位于程序包或相关的Go程序包（模块）的集合中。软件包必须位于go / src目录中（例如go / src / myPackage ）。模块是存储在文件树中的Go软件包的集合，其中Go.mod文件列出了所有相关的Go软件包和需求。请参阅GitHub存储库中 divisionMod目录中的示例文件。

文件

应用程序代码和测试功能在同一* _test.go文件中。请参阅GitHub存储库中的main_file_test.go文件。

有关模块的更多信息，请参见GitHub上的Modules文章 。
