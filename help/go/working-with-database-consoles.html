数据库控制台

数据库控制台是SQL文件，您可以在其中编写和执行SQL语句。与暂存文件不同，控制台会附加到数据源。创建数据源时，将自动创建数据库控制台。如有必要，您可以创建其他控制台。

创建数据库控制台

要创建控制台，请在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中使用以下操作之一。

  • 单击数据源，然后选择“ 文件” |“文件”。新品|控制台 。

  • 右键单击数据源，然后选择新建|控制台 。

  • 单击数据源，然后按Alt+Insert ，然后选择控制台 。

  • 单击数据源，然后按Ctrl+Shift+F10 ，然后选择新建控制台 。

新控制台意味着与数据源的新连接。您只能为数据源使用一个连接，并让所有控制台使用同一连接。在“ 启用单一连接模式”中了解有关单一连接模式的更多信息。

打开控制台

  • 在“ 项目”工具窗口（“ 视图” |“工具窗口” |“项目” ）中，导航到“ 临时和控制台” |“工具”。数据库控制台| ，双击控制台文件。

  • 在数据库工具窗口中，单击跳转到控制台图标（ ），然后选择一个控制台。

  • 单击数据源，然后按F4 ，然后选择所需的控制台。

删除数据库控制台

  1. 在“ 项目”工具窗口（“ 视图” |“工具窗口” |“项目” ）中，导航到“ 临时和控制台” |“工具”。数据库控制台 。

  2. 展开包含控制台的数据源组。

  3. 右键单击控制台文件，然后选择“ 删除” 。

修改控制台设置

  • 打开控制台，然后单击“ 打开数据库设置”图标（ ）。

在控制台中更改SQL方言

右键单击输入窗格的编辑区域，选择“ 更改方言”（ ） ，然后选择所需的方言。

默认情况下，SQL控制台方言由附加到控制台的数据源定义。

禁用语法错误突出显示

右键单击输入窗格的编辑区域，选择“ 更改方言”（ ） ，然后选择 。 支持SQL关键字，表和列名的完成和突出显示。禁用此选项的语法错误高亮显示（“ 编辑器”区域中的语句在语法上显示为正确）。

默认方言的更改将影响使用当前数据源驱动程序的所有控制台和数据源。

重命名数据库控制台

  1. 在“ 项目”工具窗口（“ 视图” |“工具窗口” |“项目” ）中，导航到“ 临时和控制台” |“工具”。数据库控制台 。

  2. 展开包含控制台的数据源组。

  3. 右键单击控制台文件，然后选择“ 重构|重命名 。

将控制台文件复制到新位置

  1. 在“ 项目”工具窗口（“ 视图” |“工具窗口” |“项目” ）中，导航到“ 临时和控制台” |“工具”。数据库控制台 。

  2. 展开包含控制台的数据源组。

  3. 右键单击控制台文件，然后选择“ 重构|复制 。

  4. 在“ 到目录”字段中，为文件指定一个新位置。

数据源目录下的组控制台

  1. 在“ 项目”工具窗口（“ 视图” |“工具窗口” |“项目” ）中，导航到“ 临时和控制台” |“工具”。数据库控制台 。

  2. 右键单击要在其中创建新目录的数据源组，然后选择“ 新建” |“新建”。目录 。

  3. 指定目录的名称。

  4. 选择控制台文件并将其拖到目录中。

  5. 在“ 移动”对话框中，单击“ 确定” 。

查看控制台日志

  1. 打开一个控制台。

  2. 右键单击“ 编辑器”区域，然后选择“ 本地历史记录” |“本地历史记录”。显示历史 。
