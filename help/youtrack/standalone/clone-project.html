克隆项目

使用“ 克隆项目”操作，您可以创建与现有项目相似但不同的项目。通过此操作，您可以创建新项目，而无需从头开始进行配置。创建新项目后，可以独立于原始项目修改其设置。

您只能克隆活动项目。如果要克隆的项目已存档，则可以还原该项目，对其进行克隆，然后将其还原为已存档状态。要了解有关已归档项目的更多信息，请参阅归档项目 。

克隆项目：

需要权限：创建项目，完整阅读项目

  1. 单击标题中的“ 项目”链接以打开“ 项目”列表。

  2. 从要克隆项目的项目的“ 管理”菜单中，选择“ 常规” 。

  3. 单击标题中的“ 克隆项目”按钮。

    • 将原始项目的克隆创建为新项目。

    • 已保存的名称为未分配的搜索会以您的用户帐户作为所有者自动生成。此保存的搜索使用查询project: #{Unassigned} 。在项目中创建新问题时，您将自动订阅通知。

      这是对所有新项目执行的标准操作，与原始项目中的设置无关。

    • 新项目将打开到“ 编辑项目”页面。

  4. 根据需要在新项目中自定义设置。使用此页面上提供的信息作为指南。需要更新的常见设置包括：

    设置

    描述

    一般

    新项目的名称和ID是自动生成的。

    直接在“ 编辑项目”页面的“ 常规”选项卡上更新这些设置的值。

    VCS

    如果原始项目与版本控制系统集成，请更新存储库URL并启用集成。否则，将在新项目中禁用VCS集成。

    您可能还需要提供新的凭据以对存储库进行身份验证。

    在“ 编辑项目”页面的“ VCS”选项卡上更新这些设置的值。

    构建服务器集成

    如果原始项目已与构建服务器集成，请更新映射以将新项目与构建服务器中的相应构建或作业链接，并启用集成。否则，将在新项目中禁用构建服务器集成。

    在“ 编辑项目”页面的“ 构建服务器集成”选项卡上更新这些设置的值。

    邮箱整合

    如果原始项目已与邮箱集成，则有权访问“ 邮箱集成”页面的用户需要编辑邮箱规则以从唯一的收件箱中获取消息并启用集成。否则，将在新项目中禁用邮箱集成。

    敏捷板

    从原始项目中克隆的任何敏捷板的名称都是自动生成的。要查看管理新项目中问题的敏捷板列表，请单击页面标题中的项目名称以访问项目概述页面，并检查侧栏中的敏捷板列表。

项目设定

下表描述了用作克隆操作源的项目中的每个设置如何应用于新项目：

设定值

描述

一般

原始项目中的大多数常规设置都会复制到新项目中。克隆原始项目时，将定义以下设置的值：

  • 名称设置为Clone of 。

  • 项目所有者设置为执行克隆操作的用户。

访问

将直接分配给原始项目中的用户和组的所有角色都授予新项目中的同一组用户和组。Project Admin角色有一个例外，该角色分配给执行克隆操作的用户。

球队

原始项目中团队成员的所有用户和组都将添加到新项目中的项目团队中。执行克隆操作的用户将作为项目所有者添加到团队中。

领域

原始项目中可用的所有自定义字段都将添加到新项目中。这些字段中每个字段中可用的值集也被复制到新项目中。

如果执行克隆操作的用户未包括在“ 受理人”字段的值集中，则该用户将添加到新项目的集合中。

自动将自定义字段附加到新项目的全局字段设置将被忽略。

VCS

将原始项目链接到版本控制系统中的存储库的所有VCS集成都将复制到新项目中。在新项目中，VCS集成已禁用。

通知模板

在原始项目中已自定义的所有通知模板都将复制到新项目中。

构建服务器集成

将原始项目映射到版本控制系统中的内部版本的所有内部版本服务器集成都复制到新项目中。在新项目中禁用了映射。这使您有机会重用原始项目中的大多数设置，而无需添加新的映射。您可以编辑映射以指向构建服务器上的其他构建并启用集成。

时间跟踪

如果在原始项目中启用了时间跟踪，那么在新项目中也会启用该功能。在新项目中设置设置为“ 估计”字段和“花费时间”字段的字段 。原始项目中使用的所有工作项类型都将复制到新项目中。

工作流程

只有附加到原始项目的工作流才附加到新项目。将工作流程自动附加到新项目的全局设置将被忽略。

敏捷板

满足以下条件时，将自动为克隆项目生成敏捷板：

  • 原始项目是敏捷板的“ 项目”设置中存储的唯一项目。存储多个项目的敏捷板将被忽略。

  • “ 可以查看和使用董事会”设置可授予对组，项目团队或基于项目的访问权限。仅对所有者可见的敏捷板将被忽略。

  • 如果克隆项目的用户不是有权查看和使用该板的组的成员，则不会创建该板的副本。

从管理原始项目中问题的电路板上复制新电路板的配置，但以下情况除外：

  • 名称设置为-copy 。

  • 项目设置的值将替换为新项目的名称。

  • 板所有者设置为执行克隆操作的用户。

邮箱集成

引用原始项目的所有邮箱规则都将复制到新项目。邮箱规则在新项目中被禁用。

这使您的管理员有机会重新使用原始规则中的大多数设置，而无需添加新的邮箱规则。有权访问“ 邮箱集成”页面的用户可以简单地编辑邮箱规则，以从唯一的收件箱中提取邮件并启用集成。有关更多信息，请参见定义邮箱规则 。

枢纽项目

如果您的YouTrack Standalone安装与外部集线器服务集成在一起，则会将集线器项目创建为新的YouTrack项目的容器。新的YouTrack项目将作为资源添加到Hub项目。

通常从Hub项目继承的所有设置（例如项目团队和访问方案）都将从新的YouTrack项目复制到Hub项目。

克隆操作中排除的项目

克隆操作不会创建引用原始项目的所有对象和集成设置的副本。下表中列出的项目不包括在克隆操作中：

项目

描述

问题

属于原始项目的问题不会复制到新项目中。

报告书

不会克隆显示原始项目中存在问题的数据的报告。

Zendesk整合

共享有问题的Zendesk票证的设置无法复制到新项目中。与第三方系统共享票证的邀请必须在Zendesk实例中启动。

要了解如何在新项目中设置Zendesk集成，请参阅Zendesk集成 。
