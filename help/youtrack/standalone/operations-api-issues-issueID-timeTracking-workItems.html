具有特定IssueWorkItem的操作

此资源使您可以处理特定问题中的工作项。

资源资源
/ api / issues / {issueID} / timeTracking / workItems / {itemID}

退货实体

IssueWorkItem 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读特定的IssueWorkItem 。

  • POST ： 更新特定的IssueWorkItem 。

  • DELETE ： 删除特定的IssueWorkItem 。

IssueWorkItem属性

代表问题中的工作项目。

下表描述了IssueWorkItem实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

作者

 用户

在工作项中为其分配工作的用户。 Can be null 。

创作者

 用户

将工作项添加到问题的用户。 Read-only 。 Can be null 。

文本

串

工作项描述。 Can be null 。

textPreview

串

解析的描述预览。 Read-only 。

类型

 工作项类型

工作项类型。 Can be null 。

被创造

长

工作项的创建日期。 Can be null 。

更新

长

工作项目的最后更新日期。 Can be null 。

持续时间

 持续时间值

工作项目的持续时间。

日期

长

分配给工作项的日期和时间。存储为UTC中的Unix时间戳。时间部分设置为当前日期的午夜。

问题

 问题

存储工作项所附问题的属性。 Read-only 。

使用Markdown

布尔型

存储注释文本的格式。什么时候true ，则注释将以Markdown格式的文本进行处理。否则，将以YouTrack Wiki格式处理评论。

阅读特定的IssueWorkItem

获取问题的特定工作项。

所需权限

要求添加对问题工作项目的访问权限并具有“读取工作项目”权限。

请求语法

GET / api / issues / {issueID} / timeTracking / workItems / {itemID}？{字段}

{itemID}

工作项的数据库ID。

请求参数

参数

类型

描述

领域

串

应在响应中返回的IssueWorkItem属性的列表。如果未指定任何字段，则仅entityID返回

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/issues/2-35/timeTracking/workItems/115-7?fields=author(id,name),creator(id,name),date,duration(id，分钟，演示文稿），id，名称，类型（id，名称）

{“ date”：1538956800000，“ duration”：{“ id”：“ 120”，“ minutes”：120，“ presentation”：“ 2h”，“ $ type”：“ DurationValue”}，“ type”：{“ name”：“测试”，“ id”：“ 65-1”，“ $ type”：“ WorkItemType”}，“ creator”：{“ name”：“ John Doe”，“ id”：“ 1-2” ，“ $ type”：“用户”}，“作者”：{“名称”：“简·多伊”，“ id”：“ 1-3”，“ $ type”：“用户”}，“ id”：“ 115-7“，” $ type“：” IssueWorkItem“}

更新特定的IssueWorkItem

更新问题的特定工作项。

所需权限

如果用户创建了工作项，则需要“更新工作项”权限；否则，需要“更新不属于自己的工作项”。

请求语法

POST / api / issues / {issueID} / timeTracking / workItems / {itemID}？{字段}

{itemID}

工作项的数据库ID。

请求参数

参数

类型

描述

领域

串

应在响应中返回的IssueWorkItem属性的列表。如果未指定任何字段，则仅entityID返回

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/issues/2-35/timeTracking/workItems/115-7
