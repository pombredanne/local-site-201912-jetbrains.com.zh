处理积压

待办事项是一系列等待规划和实施的问题。换句话说，开发团队可以构建这些未完成的工作来改进产品。团队通常会对待办事项中的问题进行优先排序，以确定在每个发行周期中要解决的问题。

在YouTrack中，敏捷搜索的待办事项列表中显示的问题集合由保存的搜索表示。

创建板时 ，可以选择现有的保存搜索作为板的待办事项，也可以创建一个新的。如果您选择创建一个新的保存的搜索，则会为其分配在板设置中提供的名称。以下格式用于生成搜索查询：

项目： 具有：-{Board } #Unsolved

也就是说，默认积压包含未解决的所有尚未在董事会的所有项目中分配给董事会的问题。

如果要在积压中显示一组不同的问题，则可以编辑保存的搜索或选择另一个已保存的搜索作为董事会的积压。

要了解有关敏捷板上积压的更多信息，请观看此演示：

变更检视模式

积压中有两种视图模式可用于解决问题。

查看模式

描述

树视图

在层次结构列表中显示问题。作为父任务的子任务的问题嵌套在树结构中。

如果与保存的搜索匹配的问题是与搜索条件不匹配的问题的子任务，则父任务显示为灰色背景。

在查看模式之间切换的功能仅限于有权编辑面板设置的组成员。

最适合您的视图因董事会和团队而异。

  • 对于使用功能或用户故事识别带有泳道任务的泳道的木板， 树状视图可让您以保留此层次结构的方式查看和管理积压中的问题。团队可以使用保存的搜索来返回功能列表，并直接在待办事项列表中将问题作为子任务添加。

  • 对于具有基于属性的泳道或根本不使用泳道的板，可以使用“ 列表”视图查看和管理仅与保存的搜索匹配的问题。

优先处理待办事项

您可以直接在董事会上重新排列积压的订单，以反映当前的优先级。您还可以直接在待办事项列表中将子任务添加到父问题。

对待办事项中的问题进行重新排序的功能仅限于有权更新用于待办事项中的问题的已保存搜索的组成员。此选项存储在已保存搜索的设置中。

要优先考虑主板上的待办事项，请执行以下操作：

  1. 点击按钮以查看积压。

  2. 将指针移到问题左上角的重新排序图标上。

  3. 按住鼠标按钮，然后将问题拖到列表中的新位置。

    • 垂直放置问题以更改其在列表中的位置。

    • 水平放置问题以更改其在父/子任务层次结构中的级别。

  4. 对要重新排序的待办事项中的每个问题重复步骤2和3。

您也可以在“ 问题”列表中对待办事项进行优先排序。在“ 问题”列表中，您有更多的查看选项，可让您查看更多详细信息。使用已保存的搜索集作为上下文，手动重新排序的问题也将在积压订单中重新排序。

在“ 问题”列表中对问题进行重新排序的功能仅限于有权更新保存的搜索的组成员。此选项存储在已保存搜索的设置中。

要对“问题”列表中的待办事项进行优先级排序：

  1. 点击按钮以查看积压。

  2. 点击在问题列表上显示链接。

    • 与“待办事项列表”使用的已保存搜索匹配的问题列表显示在“ 问题”列表中。

    • 将保存的搜索设置为搜索上下文。

    如果您未将保存的搜索设置为搜索上下文 ，则在“ 问题”列表中所做的更改不会反映在积压中。

  3. 如果需要，将详细程度设置为“ 树状视图”以在分层列表中显示问题。

  4. 将指针移到问题最左边的重新排序图标上。

  5. 按住鼠标按钮，然后将问题拖到列表中的新位置。

完成后，当在敏捷板上的Backlog中查看时， 积压中的问题会以指定的顺序显示。

将问题添加到积压中

与保存的搜索匹配的问题将自动添加到积压中。这意味着您不一定需要打开敏捷板并查看积压的积压问题。

但是，如果您作为一个团队来计划，整理或完善计划中的待办事项，则可以轻松地将问题添加到待办事项中，而无需离开敏捷委员会。

将问题添加到待办事项中的方法取决于当前的查看模式：

  • 在“ 列表”视图中 ，单击页面页脚中的“ 新问题”按钮。

  • 在树视图中 ，您可以选择将问题作为子任务添加到积压中可见的任何问题。当您单击“ 创建”子任务图标并定义问题时，该问题将作为子任务添加到所选问题中。

    要将问题添加到层次结构的最顶层，请使用页脚中的“ 新问题”按钮。

编辑积压保存的搜索

如果需要修改用于待办事项列表的已保存搜索，则可以在“问题”列表中编辑已保存的搜索 ，也可以直接在董事会的待办事项列表中修改搜索查询。

当您直接在待办事项列表中更新保存的搜索时，您无需在“ 问题 ”列表中浏览保存的搜索列表即可找到链接到您的董事会的保存的搜索。

要修改积压中的搜索查询，请执行以下操作：

需要权限：编辑标签或保存的搜索

  1. 打开敏捷板。

  2. 点击按钮以查看积压。

  3. 单击已保存搜索名称旁边的“ 编辑”链接。仅当您有权编辑所选的已保存搜索时，此链接才可用。

    • 将打开“ 编辑保存的搜索”对话框。

  4. 编辑您要更新的任何设置。

    设置

    描述

    名称

    保存的搜索的名称。

    询问

    保存的搜索查询。

    可以查看和使用

    所选组的所有成员都可以查看保存的搜索结果。这还确定哪些用户可以在积压订单中查看保存的搜索的名称，并在“ 问题”列表中查看链接以显示与保存的搜索匹配的问题 。

    可以编辑

    允许所选组的所有成员编辑已保存搜索的设置。这还确定哪些用户可以选择编辑积压的积压搜索并重新排序列表中的问题。

    固定到侧边栏

    将保存的搜索添加到“ 问题”列表的侧栏中。从“ 问题”列表中，可以选择保存的搜索以显示与保存的搜索匹配的问题的列表。此选项基于每个用户启用。

    订阅内容

    选择哪些事件针对与保存的搜索匹配的问题生成通知。这些设置基于每个用户应用到编辑已保存搜索的用户。

  5. 点击应用更改按钮。

    • 保存的搜索已更新。

    • 待办事项列表中的问题列表将立即刷新。

    • 应用于问题列表的更改将立即提供给当前正在查看待办事项列表的任何用户。

选择另一个保存的搜索作为待办事项

您可以使用任何已保存的搜索作为董事会的待办事项。这包括您自己创建的保存的搜索，或已显示给属于您的成员的组的成员。

选择其他已保存搜索的功能仅限于有权编辑面板设置的组成员。

要选择另一个保存的搜索作为待办事项，请执行以下操作：

  1. 点击按钮以查看积压。

  2. 单击待办事项的名称，然后从下拉列表中选择任何可用的已保存搜索。

    该列表仅包含固定在“ 问题”列表侧栏中的已保存搜索。

    • 待办事项列表中的问题列表已更新，以匹配新保存的搜索。

    • 当前正在查看待办事项的任何用户都可以立即使用新的问题列表。
