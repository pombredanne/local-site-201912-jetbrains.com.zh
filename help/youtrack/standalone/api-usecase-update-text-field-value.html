更新文本字段值

用例：

使用REST API使用以下命令更新存储在自定义字段中的值text类型。

请注意，此示例对YouTrack 2019.1和更高版本有效。在以前的版本中，尽管原理相同，但是查询参数和实体属性可能不同。此外，2019.1中引入了许多功能，例如按名称命名实体，但以前的版本中不支持。

摘要

向以下端点发出POST请求： /api/issues/{issueID}?[fields]

一步步

  1. 要更新问题中的自定义文本字段，请向目标问题发送POST请求。您可以按问题ID或问题实体ID引用问题。

  2. 在里面fields request参数，指定要作为响应返回的实体的属性。例如，您可以请求id和name目标自定义字段的值及其更新值。

  3. 在请求正文中，将JSON传递给name ， $type和value您要更新的文本字段。

    因此，最终的请求是：

    curl -X POST'https://example.myjetbrains.com/youtrack/api/issues/NP-98?fields=customFields(id,name,value(text））'-H'Accept：application / json'\ -H'授权：承载烫发：cm9vdA ==。MjZGZWI =。WB02vjX0cM2ltLTJXUE3VOWHpJYYNx'\ -H'内容类型：application / json'\ -d'{“ customFields”：[{“ name”：“ Customer”，“ $ type”：“ TextIssueCustomField”，“ value”：{“ text” ：“重要客户”}}]}'

  4. 为了确认目标问题文本字段设置为相应的值，让我们检查服务器的响应：

    {“ customFields”：[...{“ value”：{“ text”：“尊贵的客户”，“ $ type”：“ TextFieldValue”}，“ name”：“ Customer”，“ id”：“ 194-4”，“ $ type”：“ TextIssueCustomField “}]，” $ type“：”问题“}
