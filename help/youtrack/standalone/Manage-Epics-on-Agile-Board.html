史诗级敏捷板

本教程描述了您可以使用YouTrack来衡量一个或多个项目中的大规模开发工作。此设置可帮助担任领导角色的用户计划技术路线图并计划主要产品发布。这些准则特定于由遵循Scrum开发框架的团队开发的产品。

在Scrum开发中，工作分为不同的单元。这些单元有助于衡量满足要求所需的工作量。每个开发团队都以自己的方式定义这些单元并使用自己的术语。在本教程中，我们使用以下术语和定义。

术语

定义

史诗

大型或复杂的用户案例-太大而无法在单个冲刺中完成。开发计划进行几次迭代。为了更好地理解和描述史诗的要求，将其分解为更小，更简单的用户故事。许多Scrum团队都避免添加估算史诗，而将总估算用于其每个用户故事。小组不太频繁地通过为史诗分配任意值（例如T恤尺寸）来估算史诗的复杂性。

用户故事

最终用户在工作中想要或需要做的事情的描述。用户故事通常足够小以适合单个冲刺。用户故事的复杂性通常以故事点来衡量。当用户故事结束时，团队应该能够向最终用户交付垂直的功能。

任务

交付用户故事所需的各个工作单元。在sprint计划期间，团队将分析在下一次迭代中要交付的用户案例，并将其细分。可以在实际工作时间或理想的日子里估算任务。

担任领导角色的用户所面临的挑战是找到方法来监控实施史诗的总体进度，而冲刺的大部分重点都放在用户故事及其任务上。

两板一眼

在YouTrack中，每个史诗，用户故事和任务都是不同类型的问题。它们之间的关系由发布链接定义。

  • 任务作为子任务链接到其父用户故事。

  • 用户故事作为子任务链接到其父级史诗。

您可以在YouTrack中创建一个显示所有三个层次结构的敏捷板。

  • 泳道代表史诗。

  • 卡代表用户故事。

  • 卡上的子任务代表任务。

这种类型的董事会向领导者概述了每个史诗和用户故事的进展情况。但是，董事会对于开发团队来说是不切实际的。

开发团队需要计划和执行冲刺，以完成每个用户故事所需的任务。这些任务仅在此板上作为链接可见。团队需要自己的董事会，因此他们可以直接完成每个任务。

您可能会认为，在一个敏捷板上管理所有这些信息会很容易。实际上，使用两个委员会可以帮助处于不同角色的用户专注于对他们最重要的工作。

  • 担任领导职务的用户可以使用该委员会来计划其产品路线图。

  • 开发团队使用单独的董事会来计划和执行冲刺。

本教程向您展示如何为同一组项目配置两种类型的电路板。

从头开始

如果您要开始一个新项目，并且想要按此处所述配置敏捷板，请使用Scrum项目模板 。此模板使用史诗，用户案例和任务的问题类型来配置您的项目。它还会自动创建两个敏捷板-一个用于管理史诗和用户故事，另一个用于管理用户故事和任务。

您只需单击几下即可设置项目，而跳过本教程的其余部分。

管理现有项目

如果要跟踪一个或多个现有项目的史诗，请按照以下步骤配置项目，并为领导团队和开发团队创建敏捷板。

检查你的项目

首先，在敏捷板上检查要管理的所有项目，并确保已正确配置它们。

  • 验证每个项目的“ 类型”字段中使用的值集。如果不存在，则添加值Epic ， User Story和Task 。

  • 要为您的问题增加估计，请检查存储这些值的字段。如果尚不存在，请将其添加到您的项目中。

    领域

    类型

    描述

    故事点

    整数

    存储实现用户故事所需的工作量估计。

    理想的日子

    整数

    存储完成任务所需的估计天数。

    您可以添加其他字段来存储史诗的估计，也可以使用“ 故事点”字段将每个用户故事的总估计添加到史诗中。

建立跟踪史诗的董事会

接下来，设置项目管理委员会。该面板代表了史诗与用户存储之间的关系。板上仅显示这些类型的问题。泳道由史诗定义。用户故事的估算以故事点计。

要创建项目管理委员会：

  1. 单击标题中的“ 敏捷板”链接。

  2. 单击创建按钮，然后选择板 。

  3. 单击Scrum面板按钮。

  4. 输入面板的常规设置：

    • 给董事会起个名字。

    • 在板上添加要管理的项目。

    • 选择为待办事项创建保存的搜索，或选择现有的搜索查询。

    有关此页面上设置的更多信息，请参阅创建敏捷板 。

  5. 单击创建板按钮。

    • 木板被创建并打开到“ 木板设置”页面。

  6. 在“ 列和泳道”选项卡上，更新设置以通过“ 类型”字段中的问题识别泳道并使用Epic 。

  7. 在卡片选项卡上：

    • 将“ 估计字段 ”设置为“故事点” 。

    • 将“ 类型”的默认值设置为“ 用户故事” 。

    如果您在板上管理多个项目，则可以将“ 配色方案”设置为“ 项目”，并为每个项目的卡片分配不同的颜色。

  8. （可选）在“ 图表”选项卡上，将“ 按方法计算的燃尽量”选项设置为“ 估计” 。

    仅当您的团队更新每个用户故事的故事点数以跟踪其进度时，才启用此选项。

  9. 打开sprint下拉列表，然后编辑First sprint 。

    • 设置Sprint名称 。

    • 输入冲刺的目标 。

    • 设置冲刺计划 。项目管理板上的Sprint通常代表一系列迭代，跨越几个月。

  10. 打开待办事项，然后单击“ 编辑”链接以检查已保存的搜索。对于此板，您只想查看史诗和用户故事。要从积压中排除任务，请编辑保存的搜索条件：

    project: has: -{Board } #Unresolved -Task

您的项目管理委员会将如下所示：

使用此板可以监视开发团队的总体进度。

  • 将您的史诗分解为待办事项中的用户故事。

  • 将史诗作为泳道添加到板上。您可以在板上创建新的泳道，也可以从积压中撤出泳道。从待办事项中添加泳道时，您还可以选择将其所有用户故事都移至董事会。

  • 将用户故事作为卡片添加到董事会。

  • 在XL模式下查看每个用户故事的描述和任务。

  • 使用燃尽图跟踪每个用户故事的完成情况并衡量总体进度。

如果您估计用户故事，则此信息也会显示在板上。

  • 当前冲刺的故事点总数显示在工具栏中。

  • 每个史诗的故事点总数显示在每个泳道的右侧。

  • 每个用户故事的当前估算值显示在每个卡上。

为开发团队创建董事会

最后一步是为您的产品开发团队设置一个或多个董事会。项目开发委员会代表了用户故事和任务之间的关系。泳道由用户故事定义。此面板将每个任务显示为卡片。任务的估算是在理想的日子进行的。

设置的这一部分完全取决于您的组织结构。

  • 较大的开发团队可以拥有负责在同一应用程序中开发不同功能的产品所有者。在这种情况下，请为每个功能开发项目分别设置板。更好的是，向每个产品所有者发送指向本指南的链接，并让他们设置和配置自己的电路板。

  • 较小的团队可以专注于单个应用程序的开发，只需要一个敏捷板即可。

创建项目开发板：

  1. 按照之前步骤中的指示进行操作，直到第5步为开发团队创建新的董事会。

  2. 单击创建板按钮。

    • 木板被创建并打开到“ 木板设置”页面。

  3. 在“ 列和泳道”选项卡上，检查Swimlane设置。此设置应该已经通过“ 类型”字段中的问题来标识泳道并使用“ 用户故事” 。

  4. 在卡片选项卡上，将估计字段设置为理想天数 。

  5. （可选）在“ 图表”选项卡上，将“ 按方法计算的燃尽量”选项设置为“ 估计” 。

    仅当您的团队更新每个任务的理想天数以跟踪其进度时，才启用此选项。

  6. 打开sprint下拉列表，然后编辑First sprint 。

    • 设置Sprint名称 。

    • 输入冲刺的目标 。

    • 设置冲刺计划 。

  7. 打开待办事项，然后单击“ 编辑”链接以检查已保存的搜索。对于此面板，您只想查看用户案例和任务。要从积压中排除史诗，请编辑保存的搜索条件：

    project: has: -{Board } #Unresolved -Epic

开发团队的董事会将如下所示：

使用此板可以计划和管理每个冲刺的开发工作。

  • 整理并完善待办事项中的用户案例。

  • 将用户案例作为泳道添加到董事会。您可以在板上创建新的泳道，也可以从积压中撤出泳道。从待办事项中添加泳道时，您还可以选择将其所有任务移至董事会。

  • 将任务作为卡添加到板上。

  • 在XL模式下查看每个任务的描述和子任务。

  • 使用燃尽跟踪在冲刺期间的进度。

如果您估计任务，此信息也会显示在板上。

  • 当前冲刺的理想天总数显示在工具栏中。

  • 每个用户故事的理想天总数显示在每个泳道的右侧。

  • 每个任务的当前估计显示在每个卡上。
