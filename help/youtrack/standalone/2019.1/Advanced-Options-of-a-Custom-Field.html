YouTrack Standalone 2019.1 Help

Global Default Settings

Each of the custom fields in YouTrack has a set of global default settings. These settings define:

  • Whether a field is attached automatically to new projects.

  • Whether a field is private or public when it is attached to a new project.

  • Whether a field is displayed in the Issues list. Custom fields are visible in the Issues list in compact and detailed views.

Auto-attach a Field to New Projects

Auto-attached fields are attached by default to any new project.



Low-level Admin Read

Low-level Admin Write

To attach a field to projects automatically:

  1. From the Core Features section of the Administration menu, select Custom Fields.

  2. Select the Fields List tab.

  3. Select the desired field.

  4. Click the Enable auto-attach button in the toolbar.

Auto-attached fields are marked with an auto-attach badge in the list of available fields.

Set Privacy

A private field is available for viewing and/or editing only to users who have specific permissions in a project: Read Issue Private Fields and Update Issue Private Fields, respectively.

The global privacy setting only affects the initial state for new projects to which the field is attached automatically and existing projects to which the field is attached manually. The privacy setting for exiting projects to which the field is already attached are not affected. A project administrator can override the initial privacy setting at any time.



Low-level Admin Read

Low-level Admin Write

To make a field private by default:

  1. From the Core Features section of the Administration menu, select Custom Fields.

  2. Select the Fields List tab.

  3. Click the name of the field you want make private.

  4. In the toolbar, click Make private button.

For additional information, see Private and Public Issue Fields.

Set Visibility

Values for most custom fields are displayed in compact and detailed views in the Issues list. To reduce visual clutter, you can hide a custom field in the list by default. In this case, users only see the value that is stored in the field when they:

  • Open the issue in single issue view.

  • Override the default field visibility setting in the Issues list.



Low-level Admin Read

Low-level Admin Write

To hide a field from the Issues list by default:

  1. From the Core Features section of the Administration menu, select Custom Fields.

  2. Select the Fields List tab.

  3. Select the desired field.

  4. Click the Hide from Issues list button in the toolbar.
