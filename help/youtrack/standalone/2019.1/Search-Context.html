Search Context

If you want to limit the results of a search query to a specific set of issues, use a search context.

You can use projects, saved searches, and tags as a search context. This means that you can select the scope in which your search request is applied.

Set the search context in the Search Context drop-down list.

Customize the Search Context List

By default, the context is set to Everything. In this context, a search query returns all issues in YouTrack that are available to you.

You can customize the items that are shown search context list. The list includes the projects, tags, and saved searches that are pinned in your sidebar. All you need to do is pin only the projects, tags and saved searches that you use frequently. For more information, see Customize the Sidebar.



The ability to customize the search context is not available in the experimental version of the Issues list. To work with these controls as described here, turn off the experimental UI. For instructions, see Redesigned Issues List.

The following controls are also available in the Search Context box:

  • — Use the selected context as search query. When you click the arrow icon, the search context is entered in the search box as search request.

  • (edit) — Edit the search context. This icon is only visible for searches and tags that are available for editing by you. Click the icon to edit the selected tag or saved search.

When you set the search context, you can reorder issues in the list manually. For more information, see Reorder Issues Manually.

Search Context for Projects

When you set the search context to a project, your search queries are restricted to issues in the selected project. As long as you are using a project for the search context, this project is set as the default value for the Project field in new issues.

In this example, the search context is set to the Customer Support (CS) project.

When you click the New Issue button in the header, the project is assigned to the new issue automatically.
