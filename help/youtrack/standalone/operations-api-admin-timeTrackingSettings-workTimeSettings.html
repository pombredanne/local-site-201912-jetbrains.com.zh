具有特定WorkTimeSettings的操作

此资源使您可以配置系统范围的工作时间设置。

资源资源
/ api / admin / timeTrackingSettings / workTimeSettings

退货实体

WorkTimeSettings 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读特定的WorkTimeSettings 。

  • POST ： 更新特定的WorkTimeSettings 。

WorkTimeSettings属性

工作计划设置。

下表描述了WorkTimeSettings实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

每天分钟

整数

每个工作日的分钟数。例如，对于一天8小时，它将是480。

工作日

整数数组

一周中工作日的索引。周日的索引为0，周一的索引为1，依此类推。

firstDayOfWeek

整数

一周第一天的索引。这取决于服务器的区域设置。 Read-only 。

天数

整数

每周工作天数。 Read-only 。

阅读特定的WorkTimeSettings

阅读系统工作时间设置。

所需权限

需要“读取工作项”权限。

请求语法

GET / api / admin / timeTrackingSettings / workTimeSettings？{字段}

请求参数

参数

类型

描述

领域

串

应在响应中返回的WorkTimeSettings属性列表。如果未指定任何字段，则仅entityID返回

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/timeTrackingSettings/workTimeSettings?fields=id,name,workDays,firstDayOfWeek

样品响应体

{“ workDays”：[0，1，2，3，4，5]，“ firstDayOfWeek”：0，“ id”：“ 64-0”}

更新特定的WorkTimeSettings

更新工作时间设置。

请求语法

POST / api / admin / timeTrackingSettings / workTimeSettings？{字段}

请求参数

参数

类型

描述

领域

串

应在响应中返回的WorkTimeSettings属性列表。如果未指定任何字段，则仅entityID返回

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/timeTrackingSettings/workTimeSettings?fields=id，名称，workDays，firstDayOfWeek，minutesADay

样品申请机构

{“ workDays”：[1,2,3,4,5]，“ firstDayOfWeek”：1，“ minutesADay”：450}

样品响应体

{“ workDays”：[1，2，3，4，5]，“ minutesADay”：450，“ firstDayOfWeek”：0，“ id”：“ 64-0”，“ $ type”：“ WorkTimeSettings”}
