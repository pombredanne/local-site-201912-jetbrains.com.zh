YouTrack Standalone 2019.2帮助

与构建服务器集成

如果您正在从事软件开发项目并使用TeamCity或Jenkins，则有一些有用的集成功能可以减少团队执行的重复工作量。构建服务器集成启用了YouTrack和连接的构建服务器中的有用功能：

  • YouTrack从关联的项目中提取内部版本号。在上次成功的成功构建之后解决的问题会自动分配到下一个成功的构建的内部版本号。此值添加到一个自定义字段，该字段存储每个已解决问题的内部版本号。

    您可以指定是否为提交消息和构建注释中提到的问题分配内部编号，并使用搜索查询为与特定条件匹配的问题分配内部编号。

  • 当开发人员将更新提交到代码存储库时，他们在注释中引用了问题ID。构建服务器集成在更改详细信息中添加了指向YouTrack问题的直接链接。
    注释会自动添加到活动流中针对YouTrack中问题的VCS更改列表。每个VCS更改都包含指向构建服务器中更改详细信息的直接链接。

  • 您的开发人员可以在注释中输入命令，以获取他们在存储库中的提交。当构建服务器组装包含提交消息的构建时，该命令将应用于该命令中引用的所有问题。您可以使用此功能来更新状态或为一个或多个问题添加评论，而无需打开YouTrack。

您可以在“ 编辑项目”页面的“ 构建服务器集成”选项卡上为连接到项目的构建配置选项。

在项目中启用集成后，“ 构建服务器集成”选项卡上将提供以下控件：

控制

描述

添加新的映射

在边栏中打开“ 创建映射”对话框。使用这些设置将您的YouTrack项目映射到已经与您的项目集成的构建服务器中的新构建配置。

跑

从构建服务器中提取新数据。构建链接将添加到构建注释和提交消息中提到的YouTrack问题中。VCS提交消息中的命令将应用于相应的问题。内部版本号将根据映射中定义的选项添加到问题中。在集成设置页面上的相应列中更新了最后处理的构建。

编辑

在侧栏中打开所选构建服务器或映射的设置对话框。

去掉

从YouTrack删除选定的构建服务器或映射。

重启

从所选项目映射中删除最后记录的更改和已处理的内部版本的ID。下次计划更新集成时，YouTrack从构建服务器中提取整个更改和构建历史记录。YouTrack中已经存在的更改和构建不受影响。

禁用

禁用所选服务器或项目中的构建服务器集成。使用此选项可以禁用项目中的集成功能。如果禁用集成，则“ 启用”选项可用。
