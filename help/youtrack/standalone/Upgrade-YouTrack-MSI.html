通过MSI发行版升级

您可以使用MSI发行版本升级任何现有的YouTrack Standalone安装。

以下软件与YouTrack Standalone MSI发行版捆绑在一起：

  • Jetty Servlet引擎和HTTP服务器

  • Java SE运行时环境8 x64

MSI发行版不包括JRE x86。如果要在x86环境中安装YouTrack，则需要下载并安装与您的操作系统兼容的Java Runtime Environment。

MSI安装程序需要.NET Framework 2.0或更高版本。

  • 升级MSI安装
  • 使用MSI发行版升级非MSI安装
  • 故障排除
  • 报告升级安装问题

先决条件

升级之前，请执行以下先决任务：

  • 创建当前数据库的备份 。YouTrack Standalone不提供向前数据库兼容性。在升级过程中，数据库将迁移为与最新版本兼容的格式。这意味着您无法还原到以前的版本，而无法继续使用升级期间处理的数据库。

  • 将当前数据库的备份复制到安全位置，以防万一升级遇到问题并需要回滚到以前的版本。

  • 验证您的YouTrack许可证是否支持升级。要查看许可证限制，请单击“ 管理”菜单中的“ 全局设置”链接。如果您的免费更新期已过，则需要延长订阅时间 。

  • 注意建议的升级路径 。安装的当前版本显示在应用程序中任何页面的底部。

升级MSI安装

如果您以前使用MSI发行版安装了YouTrack Standalone服务，则可以使用MSI发行版升级到最新版本的YouTrack Standalone。

Windows Installer软件包自动处理以下所有升级过程：

  • 停止当前正在运行的服务。

  • 卸载以前的版本。

  • 安装新版本。

  • 重新启动服务。

要升级现有的MSI安装：

  1. 从JetBrains网站下载MSI发行版。

  2. 打开Windows Installer程序包（MSI文件）。

    • 将打开JetBrains YouTrack设置向导 。

  3. 请按照安装向导中的说明进行操作。

    • 设置向导完成后，基于Web的配置向导将在默认浏览器中打开。

      如果主机上未安装Web浏览器，请在另一个工作站上的Web浏览器中打开安装向导提供的URL。

  4. 单击完成按钮以关闭安装向导。

  5. 切换到Web浏览器，然后在基于Web的配置向导中继续进行升级安装。配置向导为您提供以下选项以继续升级：

    • 使用先前安装中检测到的设置进行升级。

    • 选择其他升级源。如果您在另一个位置上有另一个YouTrack Standalone数据库备份，或者想将另一个YouTrack Standalone安装位置用作升级源，请使用此选项。

    • 丢弃检测到的设置并执行全新安装。

  6. 如果选择其他升级源，请验证服务用户帐户JetBrainsYouTrack对向导中显示的所有目录具有完全访问权限。此帐户由安装程序创建以运行Windows服务。如果继续操作而未授予对JetBrainsYouTrack帐户的访问权限，则会遇到错误。

  7. 在“ 确认设置”页面上，确认基本系统设置。

    设置

    描述

    基本网址

    最终用户用来访问YouTrack Standalone的URL。

    应用监听端口

    安装后，您的YouTrack Standalone服务器侦听的端口。

    申请监听地址

    安装后，YouTrack Standalone侦听的IP地址。

    数据目录位置

    YouTrack Standalone存储内容数据的目录。

    备份位置

    YouTrack Standalone存储数据库备份的目录。

    日志位置

    YouTrack Standalone存储日志文件的目录。

    临时位置

    YouTrack Standalone存储特定于产品的临时文件的目录。

    要更改任何这些文件类型的位置，请单击“ 更改”按钮，然后选择一个新位置。

  8. 完成后，点击升级 。

    • 该配置将应用于您的YouTrack Standalone服务器。

      设置完成之前，请勿在浏览器中关闭页面。

    • 当YouTrack Standalone服务器准备就绪时， 仪表板将打开。您的YouTrack Standalone实例已升级并可以使用。

使用MSI发行版升级非MSI安装

您可以升级使用ZIP，Docker映像或JAR发行版安装的任何现有安装，并将安装类型同时更改为MSI。

升级之前，请创建当前数据库的备份 。

要使用MSI发行版升级ZIP或JAR安装，请执行以下操作：

  1. 创建并下载您当前的YouTrack版本的备份。

  2. 停止YouTrack服务。有关具体说明，请参见停止和启动YouTrack 。

  3. 使用操作系统工具来验证YouTrack启动的所有Java进程是否不再运行。如果仍有与YouTrack相关的Java进程仍在运行，请终止未完成的进程树。

  4. 从JetBrains网站下载MSI发行版。

  5. 打开Windows Installer程序包（MSI文件）。

    • 将打开JetBrains YouTrack设置向导 。

  6. 如果安装向导从先前安装的版本中检测到数据和配置文件，请选择推荐的选项以执行全新安装。

    当安装了较早版本的MSI发行版本并随后在服务器上将其卸载时，安装向导将显示此选项。

    设置向导完成后，基于Web的配置向导将在默认浏览器中打开。

    如果主机上未安装Web浏览器，请在另一个工作站上的Web浏览器中打开安装向导提供的URL。

  7. 在配置向导中 ，单击“ 升级” 。

  8. 在“ 选择升级源”页面上，单击“ 选择”按钮，然后选择您创建的备份文件作为升级源。

  9. 验证服务用户帐户JetBrainsYouTrack对向导中显示的所有目录具有完全访问权限。此帐户由安装程序创建以运行Windows服务。如果继续操作而未授予对JetBrainsYouTrack帐户的访问权限，则会遇到错误。

  10. 在“ 确认设置”页面上，确认基本系统设置。

    基本网址

    最终用户用来访问YouTrack Standalone的URL。

故障排除

如果遇到问题并且安装失败，请查看是否符合以下条件之一。

条件

升级失败。YouTrack服务未启动。您现有安装的数据库未升级到最新版本。安装程序会回滚到以前安装的版本。

原因

不确定。

解

  1. 运行启用了详细日志记录的MSI安装程序。在“ 命令提示符”窗口中，输入以下命令：
    msiexec /i youtrack-xxxxx.msi /l*v install.txt

    安装程序将详细的日志数据写入一个名为install.txt并将其保存在当前目录中。

  2. 查看内容install.txt日志文件。故障报告以字符串开头Return value 3 。

  3. 如果日志文件指示服务启动失败，请检查YouTrack日志文件中是否有错误。YouTrack日志文件的默认位置是%programdata%\JetBrains\YouTrack\logs

  4. 如果您可以自己确定并解决问题，请重复升级过程。否则，请在YouTrack项目中创建问题 。

条件

MSI安装程序已成功将安装升级到新版本，但是配置向导无法打开或报告错误。YouTrack服务无法启动。

原因

  • 指定的端口被另一服务占用。

  • 您没有访问权限来更新安装目录中的文件和文件夹。

解

  1. 检查日志文件，特别是service.log ，以了解错误。日志文件的默认位置是%programdata%\JetBrains\YouTrack\logs

  2. 如果您可以识别并解决问题（例如，更改端口号或获得对安装目录的访问权限），请按照以下步骤重复升级过程。

    • 执行新版本的全新安装。

    • 在“ 升级源”页面上，从以前的版本中选择数据库的备份副本。

    否则，请在YouTrack项目中创建问题 。

如果升级失败，您可以还原以前的版本并继续使用YouTrack，直到我们的支持团队解决了该问题。有关说明，请参阅与您的分发类型相关的页面：

  • 恢复Docker安装
  • 还原JAR安装
  • 还原MSI安装
  • 还原ZIP安装

条件

升级失败。YouTrack已安装。显示先前安装的产品代码。先前的安装无法自动升级。

原因

YouTrack先前已使用另一个管理员帐户安装。

解

  1. 登录到早期版本的YouTrack并创建数据库的备份。

  2. 要求执行安装的管理员卸载以前的版本，或者重置管理员帐户凭据并自己卸载以前的版本。您可以使用事件查看器或注册表之类的管理工具来查找用于安装先前版本的用户帐户。

    或者，您可以将先前的安装移动到另一台服务器。

    1. 执行全新安装的YouTrack。选择选项以删除先前的配置。

    2. 在配置向导中，选择先前版本的备份作为升级源。

报告升级安装问题

如果继续遇到升级安装问题：

  1. 在YouTrack项目中创建问题 。

  2. 附加以下文件，以帮助我们调查和解决问题：

    • 压缩内容logs目录并附加ZIP存档。该目录的默认位置是%programdata%\JetBrains\YouTrack\logs 。

      如果此目录的位置在“配置向导”中已更改，则可以在logs-dir的属性%programdata%\JetBrains\YouTrack\conf\internal\bundle.properties文件。

    • 压缩内容conf目录并附加ZIP存档。该目录的默认位置是%programdata%\JetBrains\YouTrack\conf 。

    • 附上install.txt文件。

我们的客户支持工程师将调查问题并与您一起解决问题。
