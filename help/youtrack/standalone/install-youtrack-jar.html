YouTrack Standalone 2019.2帮助

JAR安装

您可以从JetBrains网站上下载YouTrack JAR发行版。

YouTrack JAR的软件包结构类似于ZIP和MSI发行版，并且还包含基于Web的配置向导。

这是跨平台的发行版。您可以使用YouTrack JAR在Linux，macOS和Windows上安装和运行YouTrack。

对于所有操作系统，使用此发行版安装YouTrack的过程都是相似的。

要使用JAR发行版安装YouTrack Standalone：

  1. 从JetBrains网站下载JAR发行版。

  2. 将JAR发行版上传到您的服务器。

  3. 从命令行运行该应用程序 。

  4. 在基于Web的配置向导中配置应用程序。

运行YouTrack JAR

使用以下命令独立运行YouTrack：

java -jar youtrack- .jar [JVM选项] [主机： ] [/ contextPath]

默认情况下设置了以下选项和变量，您无需在命令行中显式设置它们：

-Xmx1024m，-XX：MaxMetaspaceSize = 350m，-ea，-XX：+ HeapDumpOnOutOfMemoryError，-XX：HeapDumpPath = .. \ logs，-XX：ErrorFile = .. \\ logs \ hs_err_pid％p.log，-Dfile.encoding = UTF-8，-Dorg.eclipse.jetty.server。Request.maxFormContentSize = 5000000，-Dorg.eclipse.jetty.server。Request.maxFormKeys = 10000，-Djetbrains.http.request.header.buffer.size = 50000，-Djetbrains.http.response.header.buffer.size = 50000，

默认情况下，您无需在命令行中明确指定任何选项或参数。但是，您可以在命令中为以下变量设置值：

参数

描述

JVM options

启动YouTrack实例时设置的JVM选项。每个JVM选项必须以--J （临时选项）或-J （永久选项）。例如， --J-Xmx2g该命令中指定的值表示在该服务的特定启动时，分配的内存将增加到2Gb。在下次启动时，如果不提供此选项，则分配的内存大小将重置为持久性配置文件中的值。

host

您的YouTrack服务器的监听地址。默认值为0.0.0.0 。

port

YouTrack服务器的HTTP端口号。

contextPath

YouTrack实例的上下文路径。上下文路径是URL路径部分上的前缀。当同一服务器上托管多个Web应用程序时，此路径会将请求路由到正确的Web应用程序。默认情况下，YouTrack服务部署在根目录（“ / '）上下文。

当您设置contextPath ，则可以使用默认路径以外的上下文运行YouTrack。以下是一些针对不同YouTrack服务器的各种URI配置的命令示例：

java -jar youtrack-2019.1.12345.jar java -jar youtrack-2019.1.12345.jar --J-Xmx2g --J-XX：MaxMetaspaceSize = 450M 8080 java -jar youtrack-2019.1.12345.jar --no-browser 8080 / youtrack java -jar youtrack-2019.1.12345.jar 127.0.0.1:8080 java -jar youtrack-2019.1.12345.jar 127.0.0.1:8080/youtrack

启动YouTrack时，该应用程序将在浏览器中打开。如果主机上未安装Web浏览器或不希望浏览器弹出，请使用--no-browser在命令中的jar文件名后面标记：

java -jar youtrack-2019.1.12345.jar --no-browser 8080

有关特定于YouTrack的JVM选项的完整列表，请参阅“ 配置参数” 。

一旦您的YouTrack服务启动并运行，您可能想要设置项目和访问权限，从另一个跟踪器导入问题，或者微调服务器环境。有关更多信息，请检查下面的“ 另请参阅”部分中的主题。
