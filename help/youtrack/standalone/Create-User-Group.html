建立群组

在YouTrack中，组用作项目中的资源 。该组定义了一组用户如何获得对该项目的访问权限。通过向组分配一个或多个角色来定义访问权限。

一个新的组必须链接到至少一个项目。创建后，现有组可以链接到多个项目。

使用组可为执行类似任务且需要非标准访问级别的用户集合提供统一的权限集。要为用户提供标准级别的访问权限，只需将他们添加到项目团队中即可。有关更多信息，请参见管理项目团队 。

除了默认组外，您还可以根据需要为YouTrack实例创建任意数量的组。

要创建一个新组：

需要权限：创建组，阅读Project Basic

  1. 在管理菜单的访问管理部分，选择组 。

  2. 单击工具栏中的“ 新建组”按钮。

  3. 在“ 新建组”对话框中，输入名称并选择该组所属的项目。要缩小项目列表的范围，请开始输入项目名称。

    请注意这一步！您必须将一个组添加到项目中以便以后使用，例如，将其用作项目中的一组受理人。

  4. 单击新建组按钮。

    • 该组已创建，但是它没有任何成员或角色分配。选择以下选项卡之一以执行相应的操作：

      标签

      运作方式

      会员

      将一个或多个用户添加到该组。有关更多信息，请参见添加和删除成员 。

      设定值

      编辑组的基本设置。有关更多信息，请参见“ 编辑基本组设置” 。

      的角色

      在小组的每个成员中授予一个或多个项目的角色。有关更多信息，请参见管理组访问 。

为什么要向项目添加组？

创建组时，必须选择一个项目。但是，组和项目之间的这种关联不会授予该组的成员对该项目的访问权限。要授予访问权限，您必须为组分配项目中的角色。

组和项目之间的链接确定了谁可以管理组。用于管理组的权限均按每个项目分配。这意味着您必须选择一个使用该组作为资源的项目，才能查看和更新组本身。

例如，假设我们在YouTrack中有Project A和ProjectB。

用户'John Johnson'被授予项目A中的默认项目管理员角色。此角色向John授予了项目A中的以下权限（以及其他权限）：

  • 阅读项目基础
  • 阅读项目全文
  • 更新专案
  • 读组
  • 创建组
  • 更新组

John在项目B中没有这些权限。当John打开“ 项目”列表时，他仅看到项目A，而没有看到项目B。

John可以创建，阅读和更新他要用于在项目A中分配角色的组。但是，由于这些权限是按每个项目授予的，因此他看不到仅链接到项目B的组。如果John想要将组添加到项目A中的受让人列表，则他只会看到链接到该项目的组。

作为管理员，您的限制较少。默认情况下，在全局项目中为管理员帐户分配系统管理员角色。这使您能够更新链接到YouTrack中任何项目的组。
