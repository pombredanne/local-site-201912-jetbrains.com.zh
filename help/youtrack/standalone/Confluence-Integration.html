  • Atlassian Confluence Server版本6.1.0和更高版本支持最新版本的YouTrack Integration附加组件。

  • 较旧版本的附件适用于Atlassian Confluence Server 3.5及更高版本，但不包含对Create宏的支持。

本页上的说明与Confluence版本6.1.0有关。

安装和配置加载项

YouTrack Integration插件可在您的Confluence安装中启用集成功能。

要安装附加组件：

  1. 从“ Confluence管理”菜单中，选择“ 附件” 。

  2. 单击页面顶部的“ 查找新加载项”链接。

  3. 在市场上搜索YouTrack Integration加载项并安装它。

有关更多信息，请参考Atlassian Universal Plugin Manager的文档。

接下来，配置插件的设置。

要配置加载项：

  1. 打开“ 管理加载项”页面，然后选择“ YouTrack集成”加载项。

  2. 单击配置按钮。

    • 将打开“ YouTrack集成设置”页面。

  3. 输入以下设置的值：

    设置

    描述

    YouTrack主机REST地址

    输入您的YouTrack安装的基本URL，然后输入/rest/ 。对于YouTrack InCloud实例，您的基本URL包括结尾/youtrack 。例如：

    https://yourcompany.myjetbrains.com/youtrack/rest/

    连续尝试连接的次数

    确定Confluence从YouTrack加载信息的最大连续尝试次数。如果Confluence在没有建立连接的情况下达到此数字，则会显示错误。

    链接库

    覆盖指定为YouTrack主机REST地址的地址，以链接到YouTrack中的问题。

    忽略YouTrack SSL证书检查

    启用后，Confluence在尝试建立连接时不会检查YouTrack服务器的SSL证书。当您的Confluence实例无法识别YouTrack SSL证书时使用。

    记录扩展调试信息

    启用后，附加组件支持的宏会将详细信息写入Confluence日志。

    使用令牌授权

    启用后，将使用永久令牌建立对YouTrack的授权。永久令牌代表生成令牌的用户授权加载项。要了解如何为您的帐户生成永久令牌，请参阅创建永久令牌 。

    Confluence用户只能在该用户可以访问的项目中添加链接，报告和创建问题。为了获得最佳效果，请使用在YouTrack中具有较高访问权限的帐户。

    禁用时，集成设置将显示“ 登录名”和“ 密码”的输入字段。您可以在YouTrack中使用用户帐户的凭据对加载项进行授权。

    我们强烈建议您使用令牌授权，因为它通常比登录名和密码组合更安全。

  4. 单击保存设置按钮。

    • 该加载项已准备就绪，可以使用。

插入问题的动态链接

要将动态链接插入Confluence页面，请使用具有以下格式的Issue宏：{ issue:id= }。链接的可视化表示反映了问题的状态。

  • 未解决问题的链接以普通文本显示。

  • 带有删除线的文本显示了已解决问题的链接。

使用以下设置来定义问题链接：

设置

描述

ID

将问题的ID存储在YouTrack中。

样式

确定问题链接的显示方式。

  • 什么时候short ，仅显示问题ID。

  • 什么时候detailed ，显示问题ID和摘要。

无论采用哪种表示方式，问题ID都会在工具提示中显示关键自定义字段的值和问题属性。

彻底解决

解决问题后，确定问题链接的显示方式。

  • 什么时候all解决问题时，问题摘要（如果可见）和问题ID以删除线文本显示。

  • 什么时候issueId ，在删除线文本中仅显示已解决问题的问题ID。

  • 什么时候none ，解决问题的演示文稿保持不变。

链接文本模板

存储$ issue占位符，用于将链接标识为对问题的引用。此设置无法修改。

使用这些链接可以汇总与Confluence页面上的内容相关的一系列问题。

插入报告

要在YouTrack中插入与查询匹配的问题列表，请使用以下格式的报告宏：{report:project=|query=}

使用以下设置来定义报告：

设置

描述

项目

将报告中的问题限制为单个项目。输入项目ID。

询问

指定报告中显示哪些问题。

reportFields

确定哪些字段和问题属性在表中显示为列。

页面大小

确定报告的单个页面上列出了多少个问题。

numPages

设置可以在报告中单击的页码。

以下示例显示了DOC项目中未解决的问题的列表：{report:project=DOC|query=#Unresolved}

创建问题

要在YouTrack中插入与查询匹配的问题列表，请使用创建宏。链接的可视化表示反映了问题的状态。

  • 未解决问题的链接以普通文本显示。

  • 带有删除线的文本显示了已解决问题的链接。

使用以下设置来定义新问题：

设置

描述

projectId

设置在其中创建新问题的项目。

摘要

设置新问题的摘要。

描述

在新问题上添加描述。

样式

确定问题链接的显示方式。

  • 什么时候short ，仅显示问题ID。

  • 什么时候detailed ，显示问题ID和摘要。

无论采用哪种表示方式，问题ID都会在工具提示中显示关键自定义字段的值和问题属性。

彻底解决

解决问题后，确定问题链接的显示方式。

  • 什么时候all解决问题时，问题摘要（如果可见）和问题ID以删除线文本显示。

  • 什么时候issueId ，在删除线文本中仅显示已解决问题的问题ID。

  • 什么时候none ，解决问题的演示文稿保持不变。

链接文本模板

存储$ issue占位符，用于将链接标识为对问题的引用。此设置无法修改。
