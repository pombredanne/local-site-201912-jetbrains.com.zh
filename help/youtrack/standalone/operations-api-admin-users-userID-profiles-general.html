YouTrack Standalone 2019.2帮助

使用特定GeneralUserProfile的操作

此资源使您可以阅读和更新指定用户的YouTrack配置文件中的常规设置。

资源资源
/ api / admin / users / {userID} / profiles / general

退货实体

GeneralUserProfile 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读特定的GeneralUserProfile 。

  • POST ： 更新特定的GeneralUserProfile 。

GeneralUserProfile属性

代表YouTrack中的用户个人资料。

下表描述了GeneralUserProfile实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

dateFieldFormat

 DateFormatDescriptor

向该用户显示日期的格式。

时区

 时区描述符

用户的时区。

地区

 LocaleDescriptor

用户选择的个人语言环境。

读取特定的GeneralUserProfile

使用用户个人资料的常规设置获取个人资料。

所需权限

需要“读取自我”才能读取自己的个人资料，需要“不拥有个人资料”来读取其他用户的个人资料。

请求语法

GET / api / admin / users / {userID} / profiles / general？{字段}

请求参数

参数

类型

描述

领域

串

应在响应中返回的GeneralUserProfile属性列表。如果未指定任何字段，则仅entityID返回

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/users/1-2/profiles/general?fields=id,dateFieldFormat(id,presentation,pattern,datePattern），时区（id，presentation，offset），语言环境（ID，名称，社区，语言，语言环境）

样品响应体

{“ locale”：{“ name”：“ English”，“ locale”：“ en_US”，“ community”：false，“ id”：“ en_US”，“ language”：“ en”，“ $ type”：“ LocaleDescriptor“}，” timezone“：{” presentation“：”中欧夏令时（UTC / GMT +2小时）“，” id“：”欧洲/柏林“，” offset“：120，” $ type“：” TimeZoneDescriptor“}，” dateFieldFormat“：{” pattern“：” d MMM yyyy HH：mm“，” presentation“：”默认日期格式（2000年12月31日23:59）“，” datePattern“：” d MMM yyyy“， “ id”：“”，“ $ type”：“ DateFormatDescriptor”}，“ id”：“ generalProfile”，“ $ type”：“ GeneralUserProfile”}

更新特定的GeneralUserProfile

更改特定用户配置文件中的设置。

所需权限

需要“更新自我”来更新自己的配置文件，需要“更新自己的配置文件”来更新另一个用户的配置文件。

请求语法

POST / api / admin / users / {userID} / profiles / general？{字段}

请求参数

参数

类型

描述

领域

串

应在响应中返回的GeneralUserProfile属性列表。如果未指定任何字段，则仅entityID返回

样品

让我们更改特定用户的时区。

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/users/1-2/profiles/general?fields=id,timezone(id,presentation,offset）

样品申请机构

{“时区”：{“ id”：“欧洲/莫斯科”}}

样品响应体

{“ timezone”：{“ presentation”：“莫斯科标准时间（UTC / GMT +3小时）”，“ id”：“ Europe / Moscow”，“ offset”：180，“ $ type”：“ TimeZoneDescriptor”}， “ id”：“ generalProfile”，“ $ type”：“ GeneralUserProfile”}
