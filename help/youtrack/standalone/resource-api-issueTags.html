问题标签

该资源使您可以访问和使用YouTrack中的问题标签。

资源资源
/ api / issueTags

退货实体

IssueTag 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读IssueTags列表 。

  • POST ： 添加一个新的IssueTag 。

支持的子资源

  • / api / issueTags / {tagID}： 具有特定IssueTag的操作

  • / api / issueTags / {tagID} /问题： 问题

IssueTag属性

代表问题标签。

扩展WatchFolder

下表描述了IssueTag实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

问题

 一系列问题

将此标签添加到的问题的集合。

颜色

 FieldStyle

标签的颜色设置。

untagOnResolve

布尔型

如果true ，此标签将在问题解决后自动从问题中删除。

所有者

 用户

创建（因此拥有）watch文件夹的用户。 Read-only 。 Can be null 。

visibleFor

 用户组

可以看到此监视文件夹的用户组。如果该文件夹仅对其所有者可见，则此属性为null。 Can be null 。

可更新的

 用户组

可以更新此监视文件夹的用户组。如果只有文件夹的所有者可以更新它，则此属性为null。 Can be null 。

名称

串

问题文件夹的名称。 Can be null 。

阅读IssueTags列表

获取当前用户可见的所有标签。

请求语法

GET / api / issueTags？{字段}＆{$ top}＆{$ skip}

请求参数

参数

类型

描述

领域

串

应在响应中返回的IssueTag属性的列表。如果未指定任何字段，则仅entityID返回

$跳过

整数

可选的 。可让您设置要返回的多个实体，以在返回第一个实体之前跳过。

$ top

整数

可选的 。用于指定响应中返回的最大条目数。

样本请求URI

https://example.myjetbrains.com/youtrack/api/issueTags?fields=name，所有者（登录名，名称），visibleFor（名称，id），问题（idReadable，摘要）＆$ skip = 4＆$ top = 3
