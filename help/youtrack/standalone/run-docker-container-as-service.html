运行Docker容器即服务

Docker团队建议使用跨平台的内置重启策略来将容器作为服务运行。为此， 将您的docker服务配置为在系统启动时启动 ，只需添加参数--restart unless-stopped到docker run启动YouTrack的命令。

但是，当涉及到多个服务（包括YouTrack）的顺序启动时，重新启动策略方法将不适用。您可以改用流程管理器 。

这是在Linux的帮助下如何将YouTrack容器作为服务运行的示例systemd 。

要在具有systemd的Linux上将YouTrack容器作为服务运行：

  1. 创建服务描述符文件/etc/systemd/system/docker.youtrack.service ：

    [Unit]描述= YouTrack Service After = docker.service需求= docker.service [Service] TimeoutStartSec = 0重新启动=总是ExecStartPre =-/ usr / bin / docker exec％n stop ExecStartPre =-// usr / bin / docker rm％ n ExecStartPre = / usr / bin / docker pull jetbrains / youtrack： ExecStart = / usr / bin / docker run --rm --name％n \ -v ：/ opt / youtrack / data \ -v ：/ opt / youtrack / conf \ -v ：/ opt / youtrack / logs \ -v ：/ opt / youtrack / backups \ -p ：8080 \ jetbrains / youtrack： [安装] WantedBy = default.target

  2. 使用以下命令启用在系统启动时启动服务：

    systemctl启用docker.youtrack

您还可以随时分别通过以下命令手动停止和启动服务：

sudo服务docker.youtrack停止sudo服务docker.youtrack开始
