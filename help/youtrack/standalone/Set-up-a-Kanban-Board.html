看板板

在相同位置工作的小型团队可以使用简单的白板来遵循看板方法。如果您的团队是分散的或只是更大的团队，或者无论如何要集成看板板和问题跟踪器，请考虑使用YouTrack Agile板来管理您的项目。

板上的卡片代表您的团队要处理的问题。看板方法的拥护者使用工作项之类的术语，但是在本教程中， 工作项 ， 任务 ， 问题和卡都是术语的同义词。

继续阅读以了解如何在YouTrack中设置和使用看板。要快速浏览，请观看此演示：

配置项目

需要权限：更新项目，读取问题，创建问题，更新问题

在开始配置看板之前，请花一分钟时间选择要在板上管理的项目，并检查其配置是否正确。

看板方法假定您在板上绘制了反映开发过程各个阶段的列。您的过程可以包括很多阶段，所有阶段都可以在板上看到。大多数开发团队使用三到七列。这是一个合理的限制，因为在看板实践中，对董事会的监控至关重要。您拥有的列越多，跟踪起来就越困难。

列通常表示开发阶段，部门名称或您认为是工作阶段的其他任何内容。当您在看板上创建新卡时，该卡代表YouTrack中的问题。问题字段之一存储阶段。默认情况下，开发板使用一个字段将此值存储为state类型。当您在板上移动卡时，该字段将获取已移动到的列的值。

此示例显示了一个状态类型字段，该字段表示项目中问题的开发阶段。该字段不能为空，这意味着在设置此项目时，该项目中创建的所有新问题都将显示在代表该值的列中。

为了获得最佳结果，请为使用状态类型的现有字段编辑一组值。如果您向项目添加新的状态类型字段，请确保删除所有其他状态类型字段。使用多个状态类型字段的项目可能很难使用，并且需要其他设置。有关更多信息，请参见与多个状态类型字段的可能冲突 。

如果要对新项目使用看板板，则可以同时使用看板项目模板创建项目和板。有关更多信息，请参见看板项目模板 。

创建董事会

单击标题中的“ 敏捷板”链接，从“ 创建”菜单中选择“ 板 ”，然后单击“ 看板”按钮。

请遵循以下准则来设置新板：

设置

指导方针

名称

给董事会起个名字。

专案

选择要在板上管理的项目。

可以查看和使用板

从列表中选择项目团队的名称。要查看卡片和泳道并使用面板，项目团队的每个成员还应该有权查看和更新所选项目中的问题。

可以编辑电路板设置

如果您希望项目团队的其他成员有权访问董事会设置，请从列表中选择项目团队的名称。否则，只有您可以更新主板的设置。

积压

如果您已经定义了一个包含产品积压中所有问题的保存搜索，请选择“ 使用现有”选项。否则，选择创建新选项。

保存的搜索

选择要用于板上积压工作的已保存搜索的名称。如果选择了“ 创建新选项”，则此设置将显示在创建木板时分配给新保存的搜索的名称。

新积压的搜索查询格式如下：

project: has: -{Board } #Unresolved

此保存的搜索显示在敏捷板上管理的项目中所有尚未解决的问题，这些问题尚未添加到当前的板上。

询问

筛选积压中显示的问题的搜索查询。如果选择了“ 创建新选项”，则此设置将显示为积压工作创建的已保存搜索的查询。

完成后，单击创建板按钮。“ 板设置”页面将打开“ 常规”选项卡。

选择董事会行为选项

常规选项卡显示您刚定义的创建板的设置。对于使用看板模板创建的模板，默认情况下，“ Sprints”选项为“ Disabled” 。标准看板方法不使用冲刺。您始终可以启用sprint并遵循Scrum / Kanban（Scrumban）混合方法，但这不是本教程的目的。

在修改其他选项卡上的设置之前，您要注意的是“ 董事会行为”选项。

选项

描述

手动分配问题

如果要管理将哪些问题显式添加到板上，请将此默认选项保持启用状态。除非您手动添加它们，否则在板上不会看到项目中创建的新问题。遵循看板方法的团队通常会在董事会最左侧的栏中直接添加新问题。

使用此选项时，请确保积压显示的是主板上当前未显示的问题。如果有人在不使用开发板的情况下在您的项目中创建问题，则可以找到该问题并将其添加到待办事项列表中。

过滤卡以匹配查询

使用此选项可将问题自动添加到板上。任何与搜索查询匹配的问题都将作为卡片添加到板中。直接添加到董事会的新问题会自动继承搜索查询中使用的属性。

如果启用此选项，并将搜索查询字段保留为空，则所选项目中的所有问题都将添加到面板中。每个问题都作为卡片放置在与标识列的字段的值匹配的列中。

定义列和泳道

下一步是配置用于识别板上列和泳道的设置。创建板后，默认情况下，将使用在本教程第一部分中配置的自定义字段来标识列。

在“ 列和Swimlane”选项卡上，可以使用“ 添加列”选项将该字段中的值作为列添加到开发板。

默认情况下，看板板的模板不使用泳道。但是，您可以在面板上添加泳道以添加另一层信息。

例如，您可以激活“ 值”选项并使用“ 优先级”字段来识别泳道。该字段中存储的每个值在面板上均显示为泳道。

设置工作进度限制

进行中（WIP）可以同时指示特定板卡列中可以显示多少张卡。这表示在此开发阶段可以同时管理的任务数。我们建议您的WIP值反映在此工作阶段可以同时执行的实际任务数。

例如，该项目团队只有两名撰稿人。“审阅”阶段的“ 最大在制品”设置为2 。

您可以分别设置每列的最小和最大在制品。

自定义卡演示

卡选项卡为您提供了一些选项，可用于配置看板。

如果您估算完成每个任务所需的时间，则可以在板上显示估算值。使用“ 估计字段”设置选择用于在项目中存储估计的字段。这样，您可以直接在每张卡上显示和更新估算值。

您还可以设置用于突出显示板上卡片的配色方案。对于看板，我们建议您根据看板状态对卡片进行着色。使用看板模板创建项目时，“ 看板状态”字段会自动添加到您的项目中。该字段将问题的当前状态存储为“ 已阻止”或“ 准备拉动” 。“ 看板状态”字段具有默认的配色方案。被阻止的问题为红色。准备解决的问题是绿色的。您可以在“ 编辑项目”页面上为此自定义字段更新配色方案。

当问题从一个阶段移至下一阶段时， 看板默认工作流程会自动设置问题的看板状态。为了防止团队成员在准备好进行下一阶段开发之前就提出问题，请将此工作流程附加到您的项目中。有关更多信息，请参见看板 。

这些状态对于遵循看板方法至关重要。您团队中负责下一步开发的成员应该清楚地知道可以将哪些问题纳入专栏。如果选择在项目中不使用此字段，则建议您使用另一种方法来明显地标记问题。例如，您可以创建一个Ready to pull标签并将其附加到可以移至下一开发阶段的问题上。

如果选择按项目为卡着色，则可以在“ 板设置”页面的“ 卡”选项卡上为每个项目分配一种颜色。

配置图表

图表选项卡使您可以选择要在看板上显示的报告。

对于看板，最有用的选项是累积流程图。该报告可随时显示任务在不同开发阶段的分布。累积流程图可帮助您监视进度，分析开发过程并检测瓶颈。

使用“ 发行过滤器”设置可以限制报告计算中包括哪些卡。在这里，我们使用过滤器从报告中排除次要问题。

与看板委员会合作

完成配置板后，您和您的团队可以使用板来管理您的工作。以下是电路板的一些关键组件，以及使用看板方法使用电路板的一些准则。

将卡添加到板

看板的一种常见做法是将工作项添加到代表打开状态的列中。您可以将问题作为卡直接添加到板上。如果您要在此主板上管理积压的问题，则可以打开待办事项并将问题从积压中拉到主板上。

对于本教程中描述的设置，用于定义列的字段不能为空。在董事会管理的项目中创建的所有问题都直接在董事会上创建。

在主板上移动卡

当您在板上移动卡时，您可以超过列的最大WIP限制。YouTrack通过更改列的背景颜色并指示超出限制来提醒您这种情况。在这里，您看到一列中有五张卡，最大WIP为四。

卡大小和电视模式

板顶部的滑块可设置板上卡片的详细程度。此设置是用户特定的，可以为不同的板单独设置。

一些团队在会议上使用大型监视器或投影仪来显示看板。电视模式以全屏显示面板，并使用深色背景的配色方案。实践证明，这种视觉效果可以提高大型监视器的远距离可视性。

进度指标

进度指示器按阶段显示卡的分配。每个条形图对应板上的一列。

要查看每列中的总牌数，请将指针移到进度指示器中相应的列上。每个泳道中的卡数显示在泳道的右侧。

如果启用了“ 估计字段”设置，则面板将在面板顶部显示所有卡的总估计。呈现方式根据用于估计完成任务所需时间的自定义字段的数据类型而有所不同。

此示例显示了理想日子的估算值。每个泳道中所有卡的总估计值显示在泳道的右侧。

追踪进度

您可以直接在板上打开累积流程图。该图表显示了过去两周按阶段分摊的牌面分布。将指针移到图表上的日期上，以查看报表上显示的任何日期每个阶段的卡数。

在此图上，垂直轴显示每列中的任务数。横轴表示日期时间。

要查看完成工作的速度，请查看Open和Done之间的区域。这个领域代表了发展的所有中间阶段。基本上，减少的幅度越大，您完成工作负载并接近最终发布日期的速度就越快。

而且，中间阶段应该随着时间的推移而缩小。如果它们正在扩展，则可能表明存在瓶颈。这也可能意味着您要为系统添加的工作量超出了当前的容量。
