YouTrack Standalone 2019.2帮助

更新字段值

YouTrack中的每个问题中可用的字段集合是在项目级别定义的。这些字段存储传达基本信息并有助于将问题分类为各种类别的值。每个字段的基本行为由其存储的数据类型定义。例如：

  • 存储日期或日期和时间的字段可让您从日历中选择日期。

  • 存储枚举值的字段使您可以从列表中选择一个或多个值。

  • 存储简单类型（例如字符串和整数）的字段使您可以直接在字段中输入值。

项目管理员还可以调整这些字段的行为。这些设置因项目而异。因此，当您为问题中的不同字段设置和更新值时，您会遇到各种各样的选择。使用此页面上的信息来熟悉这些选项。

补充文字栏位

YouTrack支持将值存储为文本的自定义字段。这些字段存储一串字符串，当问题显示时，这些字符串将用Markdown语法解释。

此类型的字段不会与其他自定义字段一起显示在边栏中。相反，它们显示在问题说明的下方。

与问题摘要和说明一样，只有在创建或编辑模式下打开问题时，才能更改基于文本的字段的值。编辑这些字段时，输入预览会以Markdown格式显示。有关格式选项的完整列表，请参见Markdown语法 。

文本字段中的Markdown与问题说明和评论中的Markdown之间的唯一区别适用于用户提及 。文本字段中的用户提及将替换为用户的全名并设置为链接，但不会像添加到问题描述或评论中那样触发通知。这些提及也被排除在使用“ mentions属性。

否则，使用补充文本字段就像YouTrack中的任何其他自定义字段一样。

  • 要了解如何将字段添加到项目中，请参阅“ 添加自定义字段” 。

  • 要了解有关将价位存储为的字段的更多信息text ，请参阅简单类型 。

必填项

项目管理员可以配置一个字段以要求一个值。必填字段不能存储空值，但未分配默认值。当您报告项目中的问题时，必填字段将在输入字段中显示消息“ 设置值 ”。

如果尝试创建问题而不为必填字段设置值，则会显示一条消息，指出有些字段需要输入。必填字段的输入在字段面板中短划线。

有关更多信息，请参见必填自定义字段 。

条件字段

仅当在另一个自定义字段中选择一个值时，才显示条件字段。最常见的用例是选择针对不同问题类型显示哪些字段。但是，项目管理员可以根据任何枚举字段中的选择添加条件，以显示特定字段，而不仅仅是问题类型。

在以下示例中，存储YouTrack版本和集线器版本的字段的可见性基于在存储产品名称的字段中选择的值。

  • 如果选择了空值，则存储集线器版本和YouTrack版本的字段将被隐藏。

  • 如果产品名称为YouTrack ，则会显示存储YouTrack版本的字段。

  • 如果产品名称为Hub ，则会显示存储Hub版本和YouTrack版本的字段。

有关更多信息，请参见条件自定义字段 。

私人领域

自定义字段的设置决定了它们是私有的还是公共的。将自定义字段标记为私有后，只有在项目中具有“ 读取问题私有字段”或“ 更新问题私有字段”权限的用户才能分别查看或编辑该字段。

当某个字段为公共字段时，在项目中具有“ 读取问题”或“ 更新问题”权限的任何用户都可以查看或编辑该字段。

此功能通过以下方式帮助限制对数据的访问：

  • 它可用于隐藏存储敏感信息（例如个人数据）的字段。

  • 它限制了可以编辑字段中的值的用户数量。例如，它可以让用户查看当前的受让人，但只有经理可以重新分配问题。

如果您无权阅读私有字段，则无法在问题中看到它们。如果您有权查看但不更新私有字段，那么将显示值，但控件将被隐藏。如果您尝试通过单击字段来修改值，则会显示错误。

有关更多信息，请参见私人定制字段 。

工作流驱动的字段

项目管理员可以将工作流程附加到项目中，以针对自定义字段调节从一个值到另一个值的转换。这些过渡由状态机规则进行调节。状态机规则可以应用于任何自定义字段。但是，状态机规则最常见的用例是调节State字段或存储了以下内容的自定义字段的值之间的转换： state类型。

将状态机规则应用于项目时，该字段的下拉列表中显示的选项仅限于状态机规则中为当前状态定义的转换。由状态机规则控制的自定义字段用工作流驱动的字段图标标记。

为状态机规则中的每个值定义的操作将显示在下拉列表中。实际值显示在右侧。

有关更多信息，请参阅状态机规则 。

计算字段

在项目中启用时间跟踪功能后，项目管理员可以选择哪个字段存储问题的累积花费时间。该字段记录为该事件中的所有工作项目报告的总时间。您不能手动更新此值-只能通过添加或编辑工作项。如果尝试手动更新此字段中的值，则会显示错误。

对于作为其他问题的父项链接的问题，YouTrack会自动将估计和花费的时间计算为添加到每个子任务的值的总和。

  • 如果您将工作项添加到父任务，则父任务中的花费时间包括来自添加到父任务的工作项的花费时间及其每个子任务的总花费时间。因此，我们通常建议您避免将工作项添加到父任务中。

  • 如果更改父任务的估计，则父任务及其子任务之间的同步将中断。要恢复同步并自动计算父任务的估计，请将父任务中的估计值设置为其每个子任务的总估计。

有关更多信息，请参见花费的时间计算 。

受让人

每个项目都有一组用户，这些用户在“ 受让人”字段的值集中可用。只要管理员尚未重新定义此字段的行为，新项目中“ 受理人”字段的值集就将包括项目团队的所有成员。

项目管理员可以独立于项目团队管理受让人列表。他们可以删除不应在项目中分配问题的团队成员，也可以添加不属于项目团队的用户。有关更多信息，请参见管理受让人 。

默认的“ 受理人”字段存储一个值。这意味着只能将问题分配给单个用户。但是，管理员可以更新此字段的基本属性以存储多个值。有关详细信息，请参见将问题分配给多个受让人 。

所属领域

将值存储为ownedField具有辅助所有者属性。此属性将字段中的每个值与特定用户相关联。默认的“ 子系统”字段使用此数据类型。

该辅助属性在字段的值列表中不可见。但是，工作流可以访问它。您可以编写工作流程脚本，以对拥有的字段的值的更改做出反应并执行与其所有者相关的操作。例如，如果您在使用“ 子系统分配人”工作流的项目中为此字段设置值，则问题将自动分配给被设置为子系统所有者的用户。

版本字段

将值存储为version具有第二属性的集合。可以为字段中存储的每个值分配一个发布日期，标记为已发布和标记为已存档。

如果某个版本标记为已存档，则它在值列表中不再可见。这意味着您无法选择该值并将其添加到该字段。

要了解有关使用归档值的更多信息，请参见使用归档值 。

遵循Scrum方法的团队通常的做法是将sprint与产品的发行版同步。YouTrack可以通过敏捷板支持这种做法，该板可以将sprint链接到自定义字段的值。

如果您使用由基于版本的开发板构建的敏捷开发板管理项目中的问题，或者配置为将sprint链接到“ 修订版本”字段中存储的值，则可以更新“ 修订版本”字段的值通过在冲刺之间移动问题。冲刺和字段值之间的同步也以其他方式起作用。当您更新问题中“ 修订版本”字段的值时，该问题将自动分配给相应的冲刺。要了解更多信息，请参见使用修订版本链接Sprint 。

建造领域

许多软件开发项目将包含一个或多个将内部版本号存储为build类型。可以为该字段中存储的每个值分配一个组装日期。这主要用于按时间顺序对集合中的值进行排序。

虽然可以手动将值添加到存储内部版本号的字段中，但是大多数都由内部版本服务器集成自动填充。例如，当集成处理的提交消息提到问题时，使用TeamCity集成的项目会为问题分配内部版本号。YouTrack中的内部版本号附加有一个图标，可直接访问TeamCity中的内部版本。

如果您手动将集成已处理的内部版本号分配给某个字段，则TeamCity图标也会添加到该字段中。

有关更多信息，请参阅与Build Server集成 。

向字段添加新值

每个项目可以具有一组不同的值，这些值存储在存储枚举值的自定义字段中。在使用该字段的所有项目中具有“ 更新项目”权限的用户可以将值添加到枚举集中，而无需浏览问题。

对于项目唯一的字段或使用一组独立值的字段，您仅需要当前项目中的“ 更新项目”权限。在这种情况下，您会在现有值列表的底部看到添加新值的选项。

使用存档的值

在大多数存储枚举集中的值的字段中，都有一个选项可以将集中的特定值归档。

  • 自定义字段的下拉列表中没有标记为已归档的值。这意味着您不能直接在用户界面中将字段的值设置为已存档的值。

  • 如果当前分配给该字段的一个或多个值已归档，则可以清除选择以从字段中删除这些值。归档值在下拉列表中显示为带有归档标志。

  • 归档版本仍可用于命令，工作流和REST API。这意味着您可以使用命令来添加或删除归档值或以编程方式更新这些值。要了解有关使用命令更新问题的更多信息，请参阅将命令应用于问题 。
