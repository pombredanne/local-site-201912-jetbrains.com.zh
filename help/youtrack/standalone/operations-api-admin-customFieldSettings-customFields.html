使用特定CustomField的操作

该资源使您可以使用自定义字段。在旧的REST API中，我们称它们为“原型”。

资源资源
/ api / admin / customFieldSettings / customFields / {fieldID}

退货实体

CustomField 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • POST ： 更新特定的CustomField 。

  • DELETE ： 删除特定的CustomField 。

  • GET ： 读取特定的CustomField 。

CustomField属性

代表YouTrack中的自定义字段。

下表描述了CustomField实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

名称

串

自定义字段的名称。 Can be null 。

localizedName

串

如果设置，则此值用于UI中的字段表示。 Can be null 。

fieldType

 字段类型

自定义字段的类型。

isAutoAttached

布尔型

如果true ，那么该字段将自动附加到新项目。

isDisplayedInIssueList

布尔型

如果true ，默认情况下该字段在“问题”列表中可见。

序数

整数

字段编号。

别名

串

逗号分隔的别名列表，可在搜索和命令中使用。 Can be null 。

fieldDefaults

 CustomFieldDefaults

自定义字段的与项目相关的默认设置。对于不同类型的自定义字段，可能包含不同的值。 Read-only 。

hasRunningJob

布尔型

指示此字段是否正在运行后台作业。在这种情况下，某些现场操作可能暂时不可用。 Read-only 。

isUpdateable

布尔型

指示当前用户是否有权更新此字段。 Read-only 。

实例

 ProjectCustomFields的数组

与项目相关的设置。可能包含不同类型的自定义字段的不同值

更新特定的CustomField

使用特定的ID更新自定义字段。

所需权限

需要使用该字段的所有项目中的项目更新权限。如果该字段是自动附加的，则需要“低级管理员写”权限。

请求语法

POST / api / admin / customFieldSettings / customFields / {fieldID}？{字段}

{fieldID}

自定义字段的数据库ID。

请求参数

参数

类型

描述

领域

串

应在响应中返回的CustomField属性列表。如果未指定任何字段，则仅entityID返回
