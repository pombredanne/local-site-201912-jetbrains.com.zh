具有特定IssueTag的操作

该资源使您可以访问和使用YouTrack中的问题标签。

资源资源
/ api / issueTags / {tagID}

退货实体

IssueTag 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读特定的IssueTag 。

  • POST ： 更新特定的IssueTag 。

  • DELETE ： 删除特定的IssueTag 。

IssueTag属性

代表问题标签。

扩展WatchFolder

下表描述了IssueTag实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

问题

 一系列问题

将此标签添加到的问题的集合。

颜色

 FieldStyle

标签的颜色设置。

untagOnResolve

布尔型

如果true ，此标签将在问题解决后自动从问题中删除。

所有者

 用户

创建（因此拥有）watch文件夹的用户。 Read-only 。 Can be null 。

visibleFor

 用户组

可以看到此监视文件夹的用户组。如果该文件夹仅对其所有者可见，则此属性为null。 Can be null 。

可更新的

 用户组

可以更新此监视文件夹的用户组。如果只有文件夹的所有者可以更新它，则此属性为null。 Can be null 。

名称

串

问题文件夹的名称。 Can be null 。

阅读特定的IssueTag

获取具有指定ID的标签的设置。

所需权限

要查看标签，用户应该是标签的作者或共享标签的用户组的一部分。

请求语法

GET / api / issueTags / {tagID}？{字段}

{tagID}

标签的数据库ID。

请求参数

参数

类型

描述

领域

串

应在响应中返回的IssueTag属性的列表。如果未指定任何字段，则仅entityID返回
