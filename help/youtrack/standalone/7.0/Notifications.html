Notifications

You can receive a notification when changes are applied to an issue. There is a set of default notification schemes that are enabled for all users. These notifications are based on subscriptions to changes in issues that match the following saved search criteria:

The administrator of your YouTrack instance enables email and Jabber notifications at the global level. If the administrator disables a notification channel at the global level, you cannot receive notifications through the disabled channel regardless of the personal settings in your user profile.

Saved SearchDescription
Assigned to meYou are notified of changes that are made to issues for which you are set as the Assignee.
Reported by meYou are notified of changes that are made to issues that you reported.
Commented by meYou are notified of changes that are made to issues that you have added a comment to.
Unassigned in [Project Name]You are notified of any unassigned issues that are created in your project. This saved search is generated automatically when you create a project and are set as the project lead. To let other members of your team enable and use this notification scheme, share the saved search. If you delete the saved search, the notification scheme is disabled.

You are also automatically notified of any changes in issues that you are watching. This notification scheme is based on a subscription to changes in issues with the Star tag.

You can customize the default notification schemes and other notification settings on the Filters and Notifications tab of your profile page..

Enable Notifications

YouTrack lets you customize your personal notification scheme. You can enable or disable specific notification channels, and determine the types of changes you want to receive notification for.

To configure your personal notifications:

  1. Open the Profile > Filters and Notifications tab.
  2. Select the channel where you want to receive notifications:
    OptionDescription
    Enable Jabber notificationsSelect to receive notifications over the Jabber protocol. Your administrator configures the notification channel to use a specific Jabber client or service. For more information, see Global Settings.
    Enable email notificationsSelect to receive notifications by email. Select a message format from the drop-drown list. Your administrator configures the notification channel to use a specific SMTP server. For more information, see Global Settings.

    These notification options are enabled by default. Notifications are sent to the Jabber account or email address specified in the General Settings tab of your user profile.

  3. Enable or disable any of the following notification options, based on your personal preference:
    OptionDescription
    Receive notification on my own changesYou are notified any time you apply a change to any issue.
    Receive notification when my issue is noticed by the project teamYou are notified any time a member of the project team views the expanded issue in the Issues list or opens it in full screen mode.
    Receive notifications on @mention in a commentYou are notified any time you are mentioned by username in a comment.
    Receive notifications on changes in duplicate clusterYou are notified when an issue that is part of a duplicate cluster that contains an issue you are watching is updated. In YouTrack, all issues that are connected by "duplicated"-type links, even indirectly, are virtually united in a duplicate cluster. For example, issue A-1 duplicates issue A-2, and issue A-3 duplicates issue A-2. All three of them form a duplicate cluster. If you enable this option, and are the reporter for issue A-1, you are notified about updates to the issue A-3, even though you are not watching the issue directly.
    Auto-watch issues on voteYou are added as a watcher to an issue when you vote for it.
    Auto-watch issues on commentYou are added as a watcher to an issue when you add a comment to it.
    Auto-watch issues on updateYou are added as a watcher to an issue when you change any of its attributes.

Watch Issues With a Specific Tag

When notifications are enabled, you can set up notification triggers to watch issues with specific tags.

The option to receive notification for issues with the Star tag is set by default. This configuration lets you automatically receive notifications for issues you are watching.

To watch issues that are marked with specific tags:

  1. Open the Profile > Filters and Notifications tab.
  2. In the the table with the list of tags for which you receive notifications, select the events you want to be notified about:
    OptionDescription
    on issue updatedYou are notified when an issue with this tag is updated.
    on issue resolvedYou are notified when an issue with this tag is resolved.
    on comment postedYou are notified when a user adds a comment to an issue with this tag.
    on issue votedYou are notified when a user votes for an issue with this tag.
    on issue taggedYou are notified when a user adds this tag to an issue.
    on issue untaggedYou are notified when a user removes this tag from an issue.
  3. To add a tag to the list, click the Add subscription button and select a tag from the drop-down list. Select one or more notification events and click OK.
  4. To disable notifications for a specific tag, deselect all of the notification events associated with the tag.

Watch Issues that Match a Saved Search

When notifications are enabled, you can set up notification triggers to watch issues that match a saved search.

To watch issues that match a saved search:

  1. Create the saved search in the Issues list.
  2. Open the Profile > Filters and Notifications tab.
  3. Click the Add subscription button and select the saved search from the drop-down list.
  4. Select one or more notification events. The following options are available:
    OptionDescription
    on issue createdYou are notified when an issue that matches the saved search criteria is created.
    on issue updatedYou are notified when an issue that matches the saved search criteria is updated.
    on issue resolvedYou are notified when an issue that matches the saved search criteria is resolved.
    on comment postedYou are notified when a user adds a comment to an issue that matches the saved search criteria.
    on issue votedYou are notified when a user votes for an issue that matches the saved search criteria.
  5. Click the OK button.
  6. To disable notifications for a specific saved search, deselect all of the notification events associated with the saved search.
