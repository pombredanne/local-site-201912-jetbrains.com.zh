Introduction to YouTrackHub Profile

Managing Your Profile

All registered users in YouTrack have their own profile. Your profile is where you store information about yourself and manage your personal preferences. To access your profile, click the avatar in the header and select the Profile link.

The Profile page contains four tabs that let you customize your YouTrack experience.

  • General Settings
  • Filters and Notifications
  • User Time Zone and Date
  • TeamCity Integration

General Settings

The General tab displays the attributes that are stored in your user account and your YouTrack preferences.

YouTrack user accounts are managed in the JetBrains Hub service. This service handles login requests and allows access to features based on the permissions that are granted to each user account. The Hub service can either be built into YouTrack or managed externally.

The first item that you see on your YouTrack profile page is a link to your Hub Profile. This link opens your user profile page in Hub, where you can update your profile and contact data. For more information, see Hub Profile.

The following attributes are displayed on the General tab of your profile page.

AttributeDescription
LoginDisplays the username or email address that you use to log in to YouTrack.
Full NameDisplays your full name. This name is shown everywhere your account activity is displayed in YouTrack.
JabberDisplays the account that is used to send you notifications over Jabber. This attribute is only displayed when you have entered a Jabber account in your Hub profile. If Jabber notifications are disabled for your YouTrack server, a message is displayed on the page. To test the connection, click the Send test Jabber message link.
EmailDisplays the address that is used to send you notifications by email. This attribute is only displayed when you have entered an email address in your Hub profile. If email notifications are disabled for your YouTrack server, a message is displayed on the page. To test the connection, click the Send test email link.

The General tab also displays the settings that let you manage your preferences in YouTrack.

SettingDescription
Allow other users to see my online status
  • When enabled, your online status is shown next to your comments in the Comments tab. While you are online, your status is represented by a green dot next to your full name. When you log out of YouTrack, the status indicator is dimmed. This helps other users see when you are logged in to YouTrack and can discuss an issue in the comments thread.
  • When disabled, the status indicator is always dimmed.
Exception stack traces are expanded by defaultDetermines whether stack traces that are added to the description of an issue are expanded or collapsed.
Never suggest applying commands silently to a batch of issuesWhen enabled, the dialog that suggests adding comments silently to multiple issues is suppressed. This option is only supported when you have permission to apply commands silently in the project.
Show fields pane on the leftWhen enabled, the sidebar that displays custom fields in the full page view is displayed to the left of the issue details.
Suggest setting saved searches as a contextWhen enabled, the prompt to set a search query as a context when you select a saved search is suppressed. You can also disable the prompt by clicking the Never suggest link when the prompt is displayed in the issues list.

Filters and Notifications

This tab provides settings that let you manage your notification preferences. The administrator if your YouTrack instance enables email and Jabber notification at the global level. If a notification channel is disabled at the global level, you do not receive notifications, regardless of the personal settings in your user profile. For more information, see Notifications.

SettingDescription
Enable Jabber notificationsActivates notification over Jabber for your user account. Notifications are sent to the Jabber account that is saved in your Hub profile. This option is only supported when Jabber notifications are enabled at the global level for your YouTrack service.
Enable email notificationsActivates notification by email for your user account. Notifications are sent to the email address that is saved in your Hub profile. This option is only supported when email notifications are enabled at the global level for your YouTrack service.
Receive notification on my own changesEnable to receive notification any time you apply a change to any issue.
Receive notification when my issue is noticed by the project teamEnable to receive notification any time a member of the project team views the expanded issue in the issues list or opens it in full screen mode.
Receive notifications on @mention in a commentEnable to receive notification any time you are mentioned by username in a comment.
Receive notifications on changes in duplicate clusterEnable to receive notification any time updates are applied to an issue that is part of a duplicate cluster that contains an issue you are watching.
Auto-watch issues on voteEnable to be added to the list of watchers when you vote for an issue.
Auto-watch issues on commentEnable to be added to the list of watchers when you add a comment to an issue.
Auto-watch issues on updateEnable to be added to the list of watchers when you update an issue.

This tab also includes settings that let you subscribe to updates for issues with specific tags or properties that match a saved search. These settings let you select the conditions that trigger notifications. The following options are available for tags:

SettingDescription
on issue updatedEnable to receive notification when an issue with this tag is updated.
on issue resolvedEnable to receive notification when an issue with this tag is resolved.
on comment postedEnable to receive notification when a user adds a comment to an issue with this tag.
on issue votedEnable to receive notification when a user votes for an issue with this tag.
on issue taggedEnable to receive notification when a user adds this tag to an issue.
on issue untaggedEnable to receive notification when a user removes this tag from an issue.

The visibility of these controls is based on the access permissions that are available to your account and the tags that are stored in YouTrack. If there aren't any tags in YouTrack, these controls are not displayed. For more information about these settings, see Watch Issues with a Specific Tag.

The following settings are available for saved searches:

SettingDescription
on issue updatedEnable to receive notification when a user updates an issue that matches the saved search criteria.
on issue resolvedEnable to receive notification when an issue that matches the saved search criteria is resolved.
on comment postedEnable to receive notification when a user adds a comment to an issue that matches the saved search criteria.
on issue votedEnable to receive notification when a user votes for an issue that matches the saved search criteria.
on issue createdEnable to receive notification when a user creates an issue that matches the saved search criteria.

For more information about these settings, see Watch Issues that Match a Saved Search.

User Time Zone and Date

This tab provides settings that let you manage your personal timezone and date format preferences.

SettingDescription
Time ZoneDetermines which time zone is used to present date values in YouTrack. Click the Try guessing link to automatically detect your time zone.
Date Field FormatDetermines which date format is used to enter and display date values that are stored in custom fields.

TeamCity Integration

This tab provides settings that let you manage notifications that are sent when a build in TeamCity is processed by YouTrack. These options are only supported when your YouTrack service has an active integration with TeamCity.

SettingDescription
Receive notifications when issue is updated by TeamCity integrationEnable to receive notifications when a change is applied to an issue with a command that is entered in a comment in a VCS commit.
Receive notifications on failed TeamCity commandsEnable to receive notifications when a command that is entered in a comment in a VCS commit fails to update an issue.
