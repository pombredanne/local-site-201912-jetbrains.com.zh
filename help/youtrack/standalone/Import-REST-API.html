导入REST API

介绍

REST API导入方法与常见的问题创建和更新方法不同。导入支持

  • 批量更改 。可以同时发布多个问题。

  • 所有问题字段设置 。导入方法允许设置问题创建/更新时间，问题报告者和更新者，项目中的问题编号。

  • 通知关闭 。如果问题是通过导入方法创建的，则其受让人将不会收到其他通知。

导入项目流程

  1. 导入用户 。不要一次放置大量用户。使用合理大小的部分，例如，每个请求一百个用户。

  2. 设置要导入的项目：

    1. 创建项目 。

    2. 将适当的值添加到“状态”，“优先级”，“类型”等默认字段。

  3. 配置自定义字段：

    1. 对于特定的自定义字段，请创建自定义字段原型 。

    2. 定义项目的自定义字段 。

    3. 对于枚举定制字段，也应定义所有捆绑和枚举值 。

  4. 进口问题 。

    1. 受让人自动导入。

  5. 链接：

    1. 如果问题之间存在链接，请设置问题链接类型 。

    2. 导入问题链接 。

  6. 上载导入问题的附件 。
