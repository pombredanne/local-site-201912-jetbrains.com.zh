YouTrack Standalone 2019.2帮助

保护您的安装

YouTrack的开发考虑到安全性。我们非常注意避免遭受各种类型的攻击。我们与第三方合作，这些第三方扫描YouTrack中的漏洞并执行渗透测试。发现安全问题后，我们将努力在下一个主要版本或错误修复版本中解决该问题。

本页上的建议旨在帮助您确保YouTrack数据的安全。这些最佳做法可以帮助您避免导致数据容易被盗，损坏或丢失的陷阱。

记录您的设置

文档是安全计划的重要组成部分。如果您的YouTrack管理人员易手，则新管理员不必通过反复试验来了解如何配置应用程序。当您遵循本指南来保护YouTrack环境时，请记下每个更改并解释为什么按照您的方式进行设置。完成后，收集笔记并将其添加到信息安全策略文档中。

保持最新

为确保您的安装包括最新的安全性增强功能，请确保始终使用最新版本的YouTrack。为确保您没有错过重要更新：

  • 在YouTrack博客上订阅更新，或在Twitter上关注YouTrack。

  • 在“ 全局设置”页面的“ 更新信息”部分中检查更新。

为防止意外泄露，我们通常将可见性限制为JetBrains团队成员的被视为安全问题的问题 。您可能会在发行说明的“ 安全问题”部分中看到问题，但这绝不是任何给定版本中已解决的与安全相关的问题的完整列表。

因此，强烈建议您尽快将安装更新为最新版本。

创造安全的环境

作为基于Web的应用程序，注册用户可以从网络外部连接到YouTrack。暴露于Internet的任何服务器都容易受到黑客的攻击，黑客可以破坏数据，加密数据并将其保存以勒索赎金或窃取您的秘密。我们建议您在受防火墙保护的受信任环境中安装YouTrack。您的防火墙规则在没有任何业务依据的情况下不应允许流量。

请遵循以下准则，以确保安装的安全性：

  • 限制对数据库文件的访问，以防止用户或其他进程修改或删除您的YouTrack数据。

  • 安装操作系统以及与您的安装连接的第三方软件的安全更新。

  • 使用防火墙规则仅将入站网络连接所需的端口列入白名单。

保护您的连接

为了确保网络上的通信安全，您需要将YouTrack安装在提供SSL加密的反向代理服务器之后。然后，您所有的流量都使用HTTPS（基于SSL / TLS的HTTP）进行加密。有关更多信息，请参阅反向代理配置 。

当用户使用不支持SSL / TLS加密的身份验证模块（例如，集线器身份验证模块）登录到YouTrack时，密码将以纯文本格式传输。如果您不使用HTTPS，则数据包嗅探器可以拦截和记录此流量并破坏您的密码。为了保护您的帐户，请使用反向代理通过HTTPS（HTTP over SSL / TLS）保护所有连接的安全。

使用专用服务用户帐户

您应该始终使用专用的用户帐户在服务器上运行YouTrack。

  • 对于MSI安装，在安装过程中，YouTrack会使用名称JetBrainsYouTrack创建一个专用的服务用户帐户。默认情况下，此帐户只能访问YouTrack安装目录和主目录。

    如果更改运行Windows服务的用户帐户，则该帐户也应具有受限的访问权限。有关更多信息，请参见更改运行Windows服务的用户帐户 。

  • 对于ZIP和JAR安装，YouTrack服务由输入命令以启动应用程序的用户运行。这应该是专门为运行YouTrack而创建的用户帐户。在操作系统级别上限制此帐户可用的访问权限，以保护系统的关键部分。该用户应该是非特权用户，只能访问YouTrack安装目录和主目录。

加密您的数据

YouTrack将密码作为盐值存储在数据库中。每个用户的密码都使用不同的随机盐进行哈希处理。使用SHA-256密码哈希函数对加盐的密码进行哈希处理。

但是，YouTrack Standalone安装的数据库未加密。为了确保您的数据安全，我们建议您对运行YouTrack的服务器应用文件系统级或全盘加密。

限制跨域请求

如果您具有从YouTrack域外部请求数据的集成或Web服务，则可以限制对特定来源的访问。YouTrack支持跨域资源共享或CORS。

为了保护您的数据免受恶意攻击，请创建允许访问的受信任来源的固定列表。您可以在“ 全局设置”页面的“ 资源共享”部分中管理此列表。有关更多信息，请参见资源共享 。

使用数字证书

您不仅应该保护通过HTTPS对YouTrack的访问，还应该保护在YouTrack和与其连接的Web服务之间交换的数据。为每个外部服务获取SSL证书，然后将证书导入到YouTrack中。

  • 您应该为与YouTrack集成的任何外部应用程序导入受信任的SSL证书。有关更多信息，请参见SSL证书 。

  • 您还应该具有一个SSL密钥库，当它试图连接到第三方时，该密钥库会将YouTrack标识为客户端。有关更多信息，请参见SSL密钥 。

可以将以下集成配置为使用SSL建立安全连接：

  • 从吉拉进口
  • 邮箱整合
  • TeamCity整合
  • 上游整合
  • VCS集成
  • Zendesk整合
  • LDAP认证模块
  • Atlassian Jira Auth模块

您还应该使用数字证书来保护与SMTP服务器的连接，该服务器从YouTrack发送电子邮件通知。

需要两因素验证

YouTrack支持两因素身份验证（2FA）。YouTrack支持的许多第三方身份验证模块也支持2FA。YouTrack允许您为组织的每个成员要求2FA。用户可以选择使用基于应用程序或基于令牌的2FA保护其帐户。

  • 使用基于应用程序的2FA，用户必须使用外部应用程序来生成验证码，然后在使用密码登录时必须输入该验证码。这增加了一层额外的安全性。即使密码被泄露，没有来自外部应用程序的验证码，恶意用户也无法访问该应用程序。

  • 使用基于令牌的2FA，用户可以将其Hub帐户与硬件设备配对。用户登录时必须拥有此硬件设备。

当您为一个或多个组要求2FA时，这些组的成员可访问的信息将受到附加保护。要了解如何启用此功能，请参阅“ 要求两因素验证” 。

实施密码策略

如果您使用第三方服务进行身份验证并且不需要2FA，则不能保证每个用户都进行设置。或使用强密码。

您所知道的是：

  • 许多用户没有为每个帐户创建唯一的密码。

  • 大多数密码都非常弱，很容易破解。

  • 攻击者只需猜测一个弱密码即可访问您的系统。

通过集线器身份验证模块，可以确保用户创建密码来保护您的数据安全。如果使用此模块来管理登录，我们建议您将密码强度设置为好或非常强 。有关更多信息，请参见设置密码策略 。

需要电子邮件验证

要求用户验证其电子邮件地址可确保您的用户确认其帐户的所有权。当您让用户注册自己的帐户时，这一点尤其重要。

对于使用邮箱集成或通过Zendesk Integration导入的票证从传入电子邮件创建用户帐户的安装，要求电子邮件验证的选项可防止用户注册可用于未经授权访问YouTrack和您的其他应用程序的帐户商业环境。

要求电子邮件验证的选项位于YouTrack的“ 常规设置”页面上。有关更多信息，请参阅全局设置 。

放逐机器人

集线器身份验证模块使用户可以在YouTrack中注册自己的帐户。如果允许自我注册，并且可以从网络外部访问安装，则应保护安装不受注册机器人的侵害。这些机器人可能会消耗资源并声明供人类使用的许可证。

要阻止注册机器人，请在集线器身份验证模块中启用reCAPTCHA 。有关更多信息，请参阅Hub Auth Module 。

节气门尝试登录失败

节流或速率限制有助于保护应用程序免受暴力攻击。集线器身份验证模块具有允许您将速率限制应用于登录名和请求以验证凭据的设置。速率限制适用于每个IP地址。速率限制通过在一系列连续的登录失败后在短时间内阻止新的登录请求来帮助减缓暴力攻击，从而确保密码安全。

这些速率限制适用于来自任何活动身份验证模块的所有登录名。有关更多信息，请参阅通过登录设置限制 。

小心授予权限

YouTrack中的所有操作均由权限方案管理。将权限分配给角色集合，然后将其授予特定项目的用户和组。用户仅具有执行在每个项目中分配的角色所允许的操作的权限。

如果您过于宽容地授予权限，则会使系统面临内部威胁的高风险。一旦您的任何帐户遭到入侵，它也使外部黑客可以访问敏感数据。

我们建议您遵循最小特权原则，仅授予对执行每个用户所需的操作绝对必要的信息和资源的访问权限。从小开始，然后再大-但只有有充分的理由。请遵循以下准则，以防止不必要的访问敏感数据：

  • 向新帐户授予尽可能少的访问权限，然后根据需要添加权限。

  • 不再需要时撤消访问权限。

  • 删除或禁止不必要的用户帐户。

  • 如果在集线器身份验证模块中启用了用户注册，请避免使用“自动加入组”选项。

  • 限制来宾用户帐户可用的权限。

  • 向“ 所有用户”和“ 注册用户”组授予具有非常有限权限的角色。

  • 限制在全局项目中分配角色的用户数量。

  • 尽可能少使用具有系统管理员角色的帐户，并监视其活动。

备份资料

为了防止安装勒索软件并从硬件或软件故障中恢复，请确保您的YouTrack数据库已包含在备份计划中。

  • 为您的YouTrack数据库启用常规备份，并在备份失败时设置通知。有关更多信息，请参见数据库备份 。

  • 通过还原系统来测试备份，以确保该过程正常进行。有关更多信息，请参阅与您的安装类型相关的页面：

    • 恢复Docker安装
    • 还原JAR安装
    • 还原MSI安装
    • 还原ZIP安装

  • 将最新备份的副本保留在本地。

  • 确保您的备份是安全且加密的。

保护默认管理员帐户

在服务器上安装YouTrack Standalone时，可以设置默认管理员帐户的登录名和密码。此帐户有权在YouTrack中执行任何操作。

风险和安全审核发现的最常见问题之一是与特定个人无关的管理员帐户。您可能会想创建一个简单的登录名和密码组合，并与多个管理员共享默认管理员帐户。别。

  • 共享帐户通常用于获得对系统的不受控制的访问。

  • 当您无法将更改与特定个人相关联时，审核事件和日志将变得无用。

  • 管理员帐户具有无限访问权限，应使用尽可能强的密码。

请遵循以下准则来保护您的默认管理员帐户：

  • 初始设置后，为每个管理员创建个人用户帐户，并授予他们所需的访问级别。管理员应使用自己的帐户执行管理任务。

  • 将具有管理访问权限的帐户数限制为需要此访问级别的用户。

  • 要求具有管理员特权的用户使用强密码并定期更改它们。

  • 不再需要管理员访问权限时，将其撤消。

  • 仔细管理默认管理员帐户的所有权。如果负责此帐户的用户离开您的组织或不再对应用程序负责，请选择另一个用户来承担所有权并转移此责任。如果以前的管理员不再需要访问YouTrack：

    • 使用合并用户操作来合并这两个用户帐户。有关更多信息，请参见合并用户帐户 。

    • 从合并的用户帐户中删除以前的管理员的登录凭据。

    如果以前的管理员仍然要求以非管理员角色访问YouTrack：

    • 为以前的管理员创建一个新的用户帐户，并授予对该帐户的适当访问权限。

    • 如上所述，使用“ 合并用户”操作将默认管理员帐户的所有权转移给新管理员。

浏览审核事件并检查安全日志

YouTrack提供的工具可让您监视应用程序中的可疑活动。

  • “ 审核事件”页面提供了YouTrack中记录的每个事件的事件，目标和作者的列表。每次将操作应用于内置集线器服务管理的目标实体时，都会记录这些事件。这包括应用于用户，组，项目，角色，身份验证模块，服务和资源等的更改。您可以过滤列表以搜索特定类型的活动，或将事件下载为JSON文件以进行进一步调查。

  • 被认为与安全相关的事件被打印到专用security.log文件。该文件包含更广泛的事件和目标实体。

  • 登录事件被打印到专用hub-login.log文件。

您可以浏览审核事件和安全日志以检测异常活动或进行故障排除。但是，如果您的组织使用安全信息和事件管理软件，则更好的解决方案是将YouTrack日志文件包括为数据源。这些解决方案使您可以扫描和分析聚合的日志文件，并在检测到活动的可疑模式时生成警报。

请遵循以下准则来保护您的日志文件：

  • 确保只有与工作相关的个人才能访问您存储YouTrack日志文件的数据目录。

  • 在集中式日志服务器上创建YouTrack日志文件的备份副本。

限制对敏感信息的访问

如果您在问题中存储敏感信息，请付出额外的努力以确保此数据的安全性。

  • 使用专用项目来存储包含可用于识别人员的信息的问题。

  • 确保有权访问此项目中的问题的用户了解此信息的敏感性。

  • 将项目访问权限限制为需要查看和使用此信息的用户。

  • 将问题和评论的可见性限制在有权访问项目的用户组中。

对于包含敏感或机密信息的问题，请不要碰任何机会。如果您依靠用户手动设置可见性，那么一定会忘记的。

使用工作流可以根据安全方案设置问题和注释的可见性。您可以创建和附加支持以下常见用例的工作流：

  • 在用于管理敏感信息的项目中创建问题可见性时，会自动设置它。

  • 当为问题分配特定类型或子系统时，更改可见性。

  • 警告为注释设置的可见性与问题可见性不同的用户。

  • 阻止用户更改问题或评论的可见性设置。

有关更多信息，请参见工作流 。

限制匿名访问

默认情况下，所有的YouTrack安装都包含用于来宾的专用用户帐户。在Global项目中，授予该用户默认的Observer角色。如果可以从网络外部访问安装，则知道您服务器地址的任何人都可以使用来宾帐户浏览项目中的问题，而无需登录。

如果您希望限制或阻止匿名访问，则可以使用以下选项：

  • 您可以撤消在全局项目中分配给来宾帐户的角色。然后，您可以选择是否要向来宾帐户授予要公开使用的项目中的观察者角色。我们通常不鼓励授予具有来宾帐户更新权限的角色，因为这些操作不能与个人相关联。

  • 您可以禁止访客帐户。所有用户都必须登录才能查看注册帐户的问题。对每个项目中问题的访问权限取决于按用户或组分配的角色。

有关更多信息，请参阅管理访客用户帐户 。

保护用户隐私

如果您在多租户环境中使用YouTrack或在不同项目中支持客户，请使用“ 严格建议”功能保护用户的隐私。此功能根据授予当前用户的权限来过滤建议自动完成的用户列表。该过滤器适用于问题描述和注释中的搜索查询，命令和@提及。

启用后，您的用户只会看到有权访问同一组项目的其他用户的名称和登录名。有关更多信息，请参见 。
