Administration REST API

Some of the API methods described in this section have been deprecated. In order to manage Users, Groups, Roles and Permissions you should use Hub REST API. This API does not support cookie-based authorization, so existing scripts or other clients should be adapted to Hub OAuth 2.0 auth instead.

All existing YouTrack REST methods support this token-based authorization as well, thus you will not need to maintain both schemes once migrated. We recommend using the permanent token for authorization. See Log in to YouTrack for more information.

To use Administration REST API methods, the user account, which you use to log in to YouTrack, should have Low-level Administration permissions. To check permissions set for a user account, refer to the following page: Administration > Users > <user name> > Access tab.
