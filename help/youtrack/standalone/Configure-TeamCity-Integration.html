YouTrack Standalone 2019.2帮助

与TeamCity集成

请按照本页上的说明，将您的YouTrack项目与TeamCity中的构建配置集成。YouTrack与TeamCity之间的集成可在您的项目中启用以下功能：

  • 从连接的项目中获取内部版本号，并自动更新自定义字段中的值，这些字段存储内部版本号以解决问题。YouTrack中的内部版本号附加有一个图标，可直接访问TeamCity中的内部版本。

  • 将VCS更改记录添加到提到的提交消息中。要了解有关YouTrack问题中VCS更改的更多信息，请参阅查看和编辑VCS更改 。

  • 当TeamCity组装包含提交的构建时，将命令应用于提交消息中的问题。要了解如何格式化提交消息中的命令 ，请参阅《 VCS提交中的应用命令》 。

先决条件

在开始之前，请验证以下要求：

  • 您具有运行8.0或更高版本的有效TeamCity安装。

  • 为您在TeamCity中的用户帐户分配了此集成所需的权限。您需要具有查看项目和所有父项目以及查看用户配置文件的权限。对这些权限的访问基于TeamCity服务器使用的授权模式。

    授权方式

    访问要求

    简单

    这些权限仅对具有管理员级别权限的用户可用。

    每个项目

    这些权限对分配了默认项目管理员角色的用户可用。TeamCity还支持专用的工具集成角色，该角色仅包含此集成所需的权限。可以直接或在组级别将角色分配给您的帐户。

    有关角色和权限的更多信息，请参考TeamCity文档 。

  • 要与TeamCity集成的YouTrack项目使用一个自定义字段，该字段将值存储为构建类型。您可以使用构建中固定的默认自定义字段，也可以使用存储构建类型的任何其他自定义字段。

    如果您不想在问题中添加内部版本号，则可以启用集成而无需将内部版本号映射到YouTrack中的自定义字段。在这种情况下，您的开发人员仍然可以提及问题并在VCS提交消息中应用命令。

  • 如果要向提交和构建注释中提到的问题添加内部版本号，请在TeamCity中启用“ 问题跟踪集成”功能并将其配置为将字符串转换为指向YouTrack中问题的链接。有关更多信息，请参考TeamCity文档 。

连接到TeamCity服务器

要在您的项目中启用集成，YouTrack需要连接到至少一台TeamCity服务器。

  • 如果您在TeamCity中具有足够的权限，则可以自己建立连接。有关详细信息，请参阅先决条件 。

  • 如果您在TeamCity中没有所需的权限级别，则YouTrack管理员可以配置连接。有关详细信息，请参见TeamCity集成 。

  • 如果YouTrack已与TeamCity服务器集成，该服务器将处理包含对项目的引用的构建，请跳过此步骤并为您的项目定义映射 。

要连接到TeamCity服务器：

需要权限：更新项目

用于设置集成的TeamCity帐户在TeamCity中需要以下权限：

  • 查看项目和所有父项目

  • 查看用户资料

  1. 单击标题中的“ 项目”链接以打开“ 项目”列表。

  2. 从项目的“ 管理”菜单中，选择“ 构建服务器集成” 。

  3. 从“ 添加新映射”菜单中，选择“ 添加构建服务器” 。

    • 侧边栏中将打开“ 添加构建服务器”对话框。

  4. 输入以下设置的值：

    设置

    描述

    网址

    输入TeamCity服务器的URL。

    类型

    选择TeamCity 。

    授权类型

    选择要用于设置集成的授权类型。您可以使用令牌，也可以使用登录名和密码登录。

    代币

    输入您的TeamCity用户帐户的应用程序令牌。

    仅当“ 授权类型”设置为“ 令牌”时，才显示此选项。

    登录

    输入TeamCity用户帐户的用户名。

    仅当“ 授权类型”设置为“ 登录名”和“密码”时，才显示此选项。

    密码

    输入TeamCity用户帐户的密码。

    仅当“ 授权类型”设置为“ 登录名”和“密码”时，才显示此选项。

  5. 展开“ 高级设置”部分，并配置以下可选设置：

    设置

    描述

    SSL密钥

    如果TeamCity服务器使用安全连接，并且您已上载SSL客户端密钥以验证您的YouTrack服务，请从下拉列表中选择客户端密钥。有关更多信息，请参见SSL密钥 。

    套接字超时

    设置在断开套接字之前等待TeamCity服务器响应的最长时间。默认值为30秒。

    读取超时

    设置在关闭连接之前等待从TeamCity服务器进行完整数据传输的最长时间。默认值为30秒。

  6. 单击测试连接按钮。

    • YouTrack验证与TeamCity服务器的连接。

  7. 单击添加服务器按钮。

    • TeamCity服务器连接设置已保存。

重复这些步骤，将连接添加到其他TeamCity服务器。

定义项目之间的映射

要在项目中启用TeamCity集成，请将您的YouTrack项目映射到TeamCity中的构建配置。

第一组设置将YouTrack连接到TeamCity服务器上项目的构建配置。默认分支和功能分支的其他选项使您可以定义为每个分支分配了内部编号的问题。您还可以指定命令以应用到YouTrack处理的所有问题。

定义YouTrack和TeamCity之间的映射：

需要权限：更新项目

  1. 在“ 编辑项目”页面的“ 构建服务器集成”选项卡上，在构建服务器集成列表中选择一个TeamCity服务器。

  2. 单击添加新映射按钮。

    • 侧栏中将打开“ 创建映射”对话框。

  3. 输入以下设置的值：

    设置

    描述

    TeamCity服务器

    选择包含要映射到的项目的TeamCity服务器。如果您仅连接到一台TeamCity服务器，则默认设置。

    TeamCity项目

    在TeamCity中选择项目的名称。

    TeamCity建立

    从选定的TeamCity项目中选择一个构建配置。

    工艺变更

    选择YouTrack如何处理TeamCity处理的版本中的VCS更改。提供以下选项：

    • 不附加 -包含对YouTrack问题ID的引用的VCS更改不会添加到所引用问题的活动流中。当您只希望集成将内部版本号添加到自定义字段时，请使用此选项。

    • 附加 -包含对YouTrack问题ID的引用的VCS更改已添加到所引用问题的活动流中。活动流中的条目包含指向包含更改的内部版本的链接。不会为YouTrack命令解析提交消息。

    • 附加和应用命令 -包含对YouTrack问题ID的引用的VCS更改将添加到所引用问题的活动流中。活动流中的条目包含指向包含更改的内部版本的链接。还针对YouTrack命令解析提交消息。

    可见到

    要限制活动流中VCS更改显示的注释和更新的可见性，请从列表中选择一个或多个组。否则，此活动类型对任何有权读取项目中问题的用户都是可见的。

    解析提交问题评论

    启用后，提交消息文本的特定部分将作为注释复制到问题。当您将部分提交消息作为注释复制到问题时，您可以触发@mention通知并将信息公开给无法访问VCS更改的用户。

    此设置不会影响VCS更改中提交消息的显示方式。整个提交消息（包括命令和问题注释）始终显示为活动流中VCS更改记录的一部分。

    您仅应在以下情况下启用此选项：

    • 您希望在提交消息中提及其他用户，并在将文本复制到问题注释时生成通知。

    • 您限制了VCS更改的可见性，并希望使与提交有关的信息作为注释对外部用户可见。

    要了解有关YouTrack如何处理提交消息的更多信息，请参阅《 VCS提交》中的“应用命令” 。

    如果您不想将内部版本号添加到问题，请从步骤9继续。

  4. 单击以展开“ 默认分支的操作”设置。这些设置确定在默认分支中将哪些操作应用于构建。

  5. 在表单的“ 处理”部分中配置选项。这些选项确定集成处理哪些内部版本以及为哪些问题分配内部版本号。

    选项

    描述

    构建失败

    将版本号添加到所有版本的问题中，包括失败的版本。禁用后，只会将成功的构建添加到YouTrack中的问题。

    添加每个版本

    确定将哪些内部版本号添加到“ 内部字段”设置中存储的自定义字段的值集中。即使在构建中没有处理任何YouTrack问题时，也可以启用将构建添加到构建字段中的一组值中。该选项考虑的构建受“ 失败的构建”设置影响。

    相关问题

    将内部版本号添加到提交和内部注释中提到的问题。禁用后，YouTrack会忽略提到的问题。

    将此选项与“ 问题匹配”查询设置一起使用，可以确定为哪些问题分配了内部版本号。

    要使用此选项，您还必须在TeamCity中启用问题跟踪集成功能，并将其配置为将字符串转换为指向YouTrack中问题的链接。有关更多信息，请参阅Team City文档 。

    问题匹配查询

    将内部版本号添加到与特定过滤条件匹配的问题上。标准搜索查询是State: Fixed State: Verified resolved date: ${build.time}..${prev.build.time} 。此查询确保仅考虑自上次成功构建和当前构建以来已解决的问题。

    将此设置与“ 相关问题”选项一起使用，可以确定为哪些问题分配了内部版本号。

  6. 对于“ 构建”字段 ，在项目中选择用于存储TeamCity的构建编号的自定义字段。该选择仅限于存储以下内容的自定义字段： build类型。

  7. 在“ 应用命令”字段中，输入一个可选命令以应用于在YouTrack中分配了内部版本号的所有问题。使用${build}变量以引用TeamCity中的内部版本号。例如： Subsystem Testing Test in build ${build}使用此命令，YouTrack处理的所有问题都分配给“ 测试”子系统，并且“ 在构建中测试”字段设置为内部版本号。

  8. 展开并设置“ 功能分支的动作”设置的选项。这些设置确定在一个或多个功能分支中将哪些操作应用于构建。每个设置都提供与上述默认分支相同的功能。

  9. 单击创建映射按钮。

    • YouTrack项目将映射到TeamCity中的选定项目。

    • TeamCity集成已启用，并可以在您的YouTrack项目中使用。

占位符

您可以在查询和命令设置中使用占位符。这些占位符引用为TeamCity中的构建存储的属性。当YouTrack处理构建时，占位符将替换为所连接版本控制系统中的相应值。可以使用以下占位符：

占位符

描述

$ {build.time}

YouTrack处理的版本的日期和时间。

$ {prev.build.time}

YouTrack处理的版本之前的版本的日期和时间。

$ {build}

YouTrack处理的内部版本号。

匹配VCS提交作者和YouTrack用户

要链接VCS更改并处理提交消息中的命令，YouTrack必须找到与提交消息的作者匹配的用户帐户。如果在提交消息中指定了命令，则会基于在YouTrack中授予用户帐户的访问权限来应用命令。

为了与TeamCity集成，YouTrack会检查对VCS进行更改的用户的TeamCity用户配置文件中存储的电子邮件地址。要识别提交更改的TeamCity用户，应将VCS用户名与TeamCity中的用户配置文件关联。

如果在YouTrack中找不到提交消息的作者，则不会应用该命令。用户名在VCS更改选项卡上以红色文本显示。警告工具提示提供了可用于解决问题的信息。

用户基本上可以做两件事来解决此问题：

  • 为他们在YouTrack和与TeamCity连接的VCS中的帐户使用相同的电子邮件地址。

  • 将其名称（或Bitbucket的昵称）添加到其Hub帐户中的VCS用户名列表中。

    GitHub和GitLab用户经常输入其用户名，而不是在其个人资料中存储为Name的值。如果您仍然无法在将消息提交到GitHub或GitLab存储库的命令中应用命令时遇到麻烦，请检查VCS用户名中的Name 。

    例如，如果传奇软件工程师Linus Torvalds要确保他的GitHub提交已正确链接到他的VCS更改，则他需要将Linus Torvalds添加到YouTrack的VCS用户名列表中，而不是torvalds ，这是他在GitHub中的用户名。
