存档项目

时不时会有一个您不再从事的项目，但又不想立即删除。您认为您可能需要这些数据以供参考，但不想在板上收到该项目的通知或问题。您想要一个类似于删除项目的结果，但是却没有实际删除它并丢失所有数据的结果。

在这种情况下，您可以选择归档项目。

归档项目：

需要权限：更新项目

  1. 在“ 项目”页面上选择一个项目。

  2. 单击存档项目按钮。

    • 在某些条件下（例如， 如果项目用于跨项目敏捷开发板中 ），将要求您确认操作。

    • 所选项目已存档。您不能再在项目中创建问题。

还原专案

与删除项目不同，归档是可逆的操作。您可以根据需要轻松地还原已归档的项目。

还原项目：

需要权限：更新项目

  1. 单击标题中的“ 项目”链接，然后滚动到页面底部。

  2. 展开已归档项目的列表。

  3. 单击已归档项目的名称以打开项目配置文件。

  4. 单击还原项目按钮。

归档项目和敏捷委员会

归档项目后，将不再可以访问其问题。但是，引用归档项目的敏捷板在应用程序中仍然可用。

如果打开引用已归档项目的敏捷板，则该板将显示配置错误。该错误告诉您，板设置中的至少一个项目已归档。如果板设置引用的字段对于已归档项目是唯一的，则这些字段也被列为配置错误。

归档项目的引用不会从存储它们的板上删除。如果项目在以后恢复，则可以在系统中访问其问题，并可以在引用该项目的敏捷板上显示该问题。与归档项目相关的配置错误不再显示。

存档的项目和工作流程

归档项目后，所有附加到它的工作流程都会被停用。但是，附加到活动项目的工作流可以访问任何项目中的问题，即使已将项目存档。例如，使用特定链接类型更新问题的工作流可以在任何项目中返回问题。

如果您的工作流正在将不必要的更新应用于已归档项目中的问题，请修改脚本以检查isArchived目标项目的属性。您可以在警卫队属性和条件语句中引用此属性，以阻止对已归档项目中的问题进行更改。

子任务默认工作流程包含带有条件语句的规则，这些条件语句阻止对已归档项目中的问题进行更新。这是引用isArchived属性：

如果（父母&& parent.project &&！parent.project.isArchived && parent.isReported && parent.isResolved）

存档的项目和合并的用户帐户

合并两个用户帐户时，权限的结果集是两个用户以前每个用户的权限总和。当您归档带有私有字段的项目时，您可能会遇到奇怪的结果。

假设一个用户（让我们称他为“ john.d”）有权查看Project A中的私有字段，而另一个用户（让他称其为“ j.doe”）没有该许可。您存档项目A，然后决定将“ john.d”的用户帐户合并到“ j.doe”。结果，“ j.doe”的用户帐户现在具有查看Project A中私有字段的权限。如果以后还原项目A，则该用户最初没有查看项目中私有字段的权限，现在可以看到它们。

如果两个合并的帐户属于同一个人，这可能没什么大不了的。如果合并两个属于不同用户的帐户，请记住这种可能性。
