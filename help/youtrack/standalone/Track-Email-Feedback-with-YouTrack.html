跟踪电子邮件反馈

您可以使用YouTrack监视和处理反馈。此设置可帮助您：

  • 根据传入的反馈自动创建问题。

  • 将反馈分类，将需要垃圾邮件回复的问题分开。

  • 重新分配发送到反馈渠道的支持请求。

  • 捕获作为反馈发送的建议和功能请求，并将其迁移到您的开发项目中。

  • 确保您及时回应反馈。

本设置指南的目的是创建一个专门的项目，您的营销经理或公共关系团队可以使用该项目来跟踪客户的反馈。本设置指南向您展示了一种配置YouTrack以处理反馈的方法。您可以随时修改和增强此配置以适合您的样式。在本指南的最后，我们提供了一些可供您考虑的替代方法。

此设置的目的是对反馈进行分类并确定需要跟进的问题。发送反馈的客户希望您直接在其发布反馈的渠道中进行回复。当您看到要回答的问题时，您的答复将直接发送到客户的电子邮件收件箱。

工艺概述

要配置和使用YouTrack项目来处理反馈，请按照下列步骤操作：

需要权限：低级管理员写

  1. 在YouTrack中启用电子邮件通知 。

  2. 建立一个反馈项目 。

  3. 配置邮箱集成 。

在YouTrack中启用电子邮件通知

首先，请确保在服务器上启用并正确配置了电子邮件通知。

  • 如果您使用的是YouTrack InCloud，但尚未更改默认设置，请跳过此步骤并继续进行“ 设置反馈项目” 。默认情况下，您的InCloud实例中会启用和配置电子邮件通知。

  • 如果您已经更改了YouTrack InCloud实例中的默认电子邮件通知设置，或者正在使用YouTrack Standalone版本，请验证是否已启用电子邮件通知并已配置到电子邮件服务器的连接。

如果您使用Google邮件服务器发送电子邮件通知，请使用与全局发件人地址和服务台通知相同的电子邮件地址。Google不允许您使用替代电子邮件地址。如果您使用的电子邮件地址与全局设置中的地址不同，则将其用作项目中的“ 发件人”地址 ，则邮件服务器不会传递邮件。

您无需在此处进行任何特殊操作，只需遵循基本的安装说明并测试与邮件服务器的连接即可。有关特定说明，请参阅“ 通知” 。

在YouTrack中启用和配置电子邮件通知后，就可以创建和配置反馈项目了。

设置反馈项目

用于处理反馈的项目的配置应与开发团队用于构建软件的项目的配置不同。对于您的市场营销或公共关系团队，您可以将其精简。

项目的配置特定于此用例，因此我们在此处进行详细描述。

要创建反馈项目：

  1. 单击标题中的“ 项目”链接，然后单击“ 创建项目”按钮。

  2. 输入项目的名称（ Feedback ）和描述。

  3. 选择您要使用的项目类型。任何类型都可以，但如果您选择看板项目，则YouTrack会自动创建一个敏捷板，市场营销或公共关系团队可以用来管理问题。该项目模板还使用较少的默认自定义字段，这使配置更加容易。

  4. 单击创建项目按钮。

    • 将打开新项目的项目概述页面。

  5. 在边栏的“ 管理”部分，单击“ 团队”链接。

  6. 使用“ 添加成员”操作将用户和组添加到项目团队。添加到项目团队的用户和组成员将自动添加到“ 受理人”字段的可能用户列表中。

  7. 选择“ 字段”选项卡，然后在新项目中配置自定义字段：

    • 配置要在项目中使用的自定义字段。对于反馈项目，您可以通过Type ， Priority ， Assignee和State获得 。项目管理员可以根据需要添加更多自定义字段。

      对于“ 优先级”和“ 受让人” ，您可以使用默认字段及其值集。对于类型和状态字段，请考虑自定义这些字段以在反馈项目中使用以下值：

      领域

      价值观

      类型

      评论（正面），评论（负面），问题，功能请求，垃圾邮件

      州

      未回答（默认，未解决），已回答（已解决）

    • 删除所有不相关的默认自定义字段。完成后，反馈项目中的字段列表应类似于下图所示的字段。这些显示了“ 类型”和“ 状态”字段的自定义值集：

  8. 选择工作流程选项卡。

  9. 选择并分离所有不相关的工作流程。例如，如果使用了看板项目模板，则可以分离看板工作流程。它更适合开发团队，而不是市场经理。

  10. 单击附加工作流按钮，然后从下拉列表中选择通知多个未注册用户 。

    • 工作流已附加到项目。

    • require setup标志指示工作流使用项目中未使用的自定义字段。

  11. 将缺少的字段和字段值添加到项目中：

    • 展开工作流程，然后选择第一个工作流程规则。

    • 单击配置项目按钮，将必填字段添加到项目中。

    以下自定义字段已添加到项目中：

    自订栏位

    描述

    所有相关电子邮件

    存储讨论中所有参与者的电子邮件地址。工作流使用此列表将电子邮件消息发送给在YouTrack中不是注册用户的收件人。

    与上一封邮件相关的电子邮件

    仅存储邮箱集成处理的最新消息中包含的电子邮件地址。当YouTrack处理传入电子邮件时，此字段将更新。每次更新问题时，工作流都会将这些电子邮件地址复制到“ 所有相关电子邮件”字段。这样可以确保不会向未包含在最新电子邮件中的收件人发送通知。

  12. 完成后，切换回您的反馈项目中的“ 字段”选项卡。检查项目中的自定义字段列表，以验证所有必填字段均已到位。

配置邮箱集成

接下来，为反馈项目配置邮箱集成。此设置的目标是创建一个处理传入电子邮件反馈的邮箱规则。

请记住，邮箱规则是顺序应用的。与邮箱规则中定义的筛选器匹配的邮件将根据该规则进行处理，并且在后续规则中不考虑进行处理。

此过程描述了此用例所需的基本设置。邮箱规则对话框的“ 后处理”部分中有一些重要设置，可用于自定义此设置。

要配置邮箱集成：

  1. 打开管理>邮箱集成页面。

  2. 设置获取间隔 。有关详细信息，请参阅“ 设置提取电子邮件的时间表” 。

  3. 连接到用于接收电子邮件的邮件服务器。有关详细信息，请参阅连接到邮件服务器 。

  4. 单击添加邮箱规则按钮，然后配置以下设置：

    设置

    描述

    邮箱

    选择您在步骤3中连接到的邮箱。

    项目

    选择您配置为处理反馈的项目。

    维护者

    选择负责监视与此邮箱集成的用户的名称。

  5. 单击以展开“ 过滤器”部分，并输入以下设置的值：

    设置

    描述

    夹

    选择您从中获取邮件服务器上的电子邮件的文件夹。

  6. 单击以展开邮箱规则的“ 报告者”部分。

  7. 选择“ 始终将预定义的用户设置为报告者”选项，然后选择预定义的用户以将其设置为从传入电子邮件消息创建的所有问题的报告者。此选项可确保YouTrack不会为发送支持请求的每个新客户创建用户帐户。

  8. 单击以展开后处理部分。

  9. 在“ 将命令应用于新问题”输入字段中，输入命令以应用到此规则处理的问题。考虑使用设置每个问题的受让人的命令。例如：

    给[用户]

  10. 单击添加规则按钮。

行动中的反馈处理

现在，您已经配置了YouTrack来处理反馈，让我们看看它是如何工作的。这是YouTrack处理电子邮件反馈的方式。

  1. 客户向您用来收集和处理反馈的电子邮件地址提交一条消息。

  2. 邮箱集成从邮件服务器中获取电子邮件，然后根据定义的邮箱规则对其进行处理。YouTrack将电子邮件转换为问题。

  3. 反馈项目的所有者会自动收到有关在反馈项目中创建了新问题的通知。

  4. 如果在邮箱规则中添加了后处理逻辑，则可以将问题自动分配给特定用户。这甚至还可以触发通知，以通知受让人有关新问题的信息。

  5. 项目所有者或受让人审查每个新问题并执行以下操作之一：

    • 该消息包括与不相关的产品或服务的营销相关信息。代理将反馈分类为垃圾邮件 ，并将问题标记为已回答 。不需要采取进一步行动。

    • 客户留下了正面或负面的反馈。代理将相应的类型分配给该问题，并在该问题中写一个响应作为新注释。如果不需要采取其他措施，则代理标记会将问题状态设置为Answered 。

    • 客户建议对您的产品进行改进或请求当前不可用的功能。代理将反馈分类为功能请求，并在问题中将响应写为新注释。然后，代理可以将反馈问题标记为“ 已回答”，也可以将问题移至开发团队用于管理功能请求的项目。

    • 客户提出与支持相关的问题。代理将反馈归类为问题，然后将问题重新分配给客户支持。

增强功能

如果您设法完成基本设置，则可能已经完成了一天的配置。继续-呼吸新鲜空气。当您准备更多内容时，此页面仍会在这里。

我们编写了一些工作流程规则，可用于自动执行反馈处理。默认情况下，该跟踪管理工作流在YouTrack中不可用，但是您可以从自定义工作流存储库中下载它，并将其附加到您的反馈项目中。

这些工作流程规则支持以下功能：

规则

描述

将问题标记为已回答

当开发团队的成员向问题添加评论时，将“ 状态”字段的值设置为“ 已回答” 。这意味着您的团队只需在问题上添加评论即可解决。要使用此工作流程规则，请更改组的名称以匹配存储作为您的反馈团队成员的用户的组。

关闭垃圾邮件问题

当“ 类型”字段的值设置为“ 垃圾邮件”时，将“ 状态”字段的值设置为“ 已回答” 。这为您的团队节省了一键，并允许他们使用简单的命令解决问题Type Spam 。

分配评论问题

如果作者是开发团队的成员，则将问题重新分配给最新评论的作者。这意味着向问题添加评论的用户将自动对其解决方案负责，而不必手动设置受让人。要使用此工作流程规则，请更改组的名称以匹配存储作为您的反馈团队成员的用户的组。

通知有关未答复的反馈

在设定的时间表上向所有未解决问题的受让人发送通知，而忽略标记为垃圾邮件的问题。如果未分配问题，则将通知发送给项目所有者。该消息包括每个未解决问题的链接。

要启用此工作流程：

  1. 从GitHub上的自定义工作流程存储库下载ZIP文件。

  2. 将工作流程导入YouTrack。有关说明，请参阅导入工作流 。

  3. 在内置工作流编辑器中打开工作流规则，然后编辑规则以匹配您的项目配置。有关说明，请参阅“ 编辑工作流程” 。

  4. 将工作流程附加到您的项目。有关说明，请参阅管理多个项目的工作流 。
