与GitHub集成

按照此页面上的指示信息，将您的项目与github.com上托管的VCS存储库或自托管的GitHub Enterprise安装相集成。

GitHub集成启用了YouTrack中的以下功能：

  • 直接从提交注释或请求请求描述中将命令应用于YouTrack问题。有关更多信息，请参见《 VCS提交》中的“应用命令” 。

  • 在YouTrack中跟踪与活动流中特定问题相关的提交。有关更多信息，请参见查看和编辑VCS更改 。

  • 在提交消息中添加指向YouTrack问题的链接。有关更多信息，请参见《 VCS提交中的链接问题》 。

  • 通过将提交哈希值粘贴到问题摘要，描述，注释或补充文本字段中，将链接添加到GitHub存储库。

启用和配置GitHub集成时，您将定义YouTrack项目和GitHub存储库之间的映射。作为项目管理员，您可以基于每个项目设置此集成。

管理员可以为YouTrack中的任何项目启用和管理GitHub集成。有关更多信息，请参阅GitHub Integration 。

先决条件

  • 入站连接可访问YouTrack。具体来说，您需要确保您的网络不会阻止VCS服务器与YouTrack之间的连接。

  • 您用于在GitHub中生成用于授权的个人访问令牌的用户帐户具有存储库的Admin或Owner权限。

如果您要与GitHub Enterprise安装进行集成，并希望与服务器建立安全（HTTPS）连接，则可能需要将GitHub服务器的SSL证书导入YouTrack。

  • 如果您的GitHub Enterprise服务器具有由知名证书颁发机构（CA）签名的有效证书，则JVM供应商可能已将根（CA）证书添加到了证书存储中。您应该能够连接到服务器而无需导入其SSL证书。

  • 如果服务器的证书是自签名的，则需要导入证书和公共密钥以建立安全连接。为了安全起见，仅当YouTrack和GitHub Enterprise服务器都在私有计算机网络上运行时，才使用此选项。只有具有“ 低级管理员读取”和“ 低级管理员写入”权限的用户才能执行此操作。有关详细信息，请参阅SSL证书 。

配置GitHub集成

第一步是在YouTrack中的项目与GitHub中的存储库之间建立连接。要连接GitHub，您需要生成并存储个人访问令牌。该令牌根据授予GitHub帐户的访问权限，向YouTrack授予对存储库的访问权限。

您可以使用单个访问令牌来设置多个集成。如果您还没有个人访问令牌，则可以使用YouTrack的直接链接在此设置过程中生成它。

要连接到GitHub存储库：

需要权限：更新项目

  1. 单击标题中的“ 项目”链接以打开“ 项目”列表。

  2. 从项目的“ 管理”菜单中，选择“ VCS” 。

  3. 单击“ 新建VCS集成”按钮。

    • 将打开“ 新建VCS集成”对话框。

  4. 对于服务器类型 ，选择GitHub 。

  5. 将令牌粘贴到OAuth令牌字段中。

  6. 将您的个人访问令牌粘贴到“ 个人访问令牌”输入字段中。如果您还没有访问令牌，请按照下列步骤操作：

    • 单击生成令牌链接以在GitHub中打开“ 新建个人访问令牌”页面。

    • 输入令牌的描述。

    • 当您使用YouTrack的链接访问“ 新建个人访问令牌”页面时，将为您选择所有必需的范围。如果您直接访问此页面，请选择repo ， read.org和admin:repo_hook 。

    • 单击生成令牌按钮。

    • 将令牌复制到剪贴板。

    • 切换回YouTrack中的“ 新建VCS集成”对话框，并将令牌粘贴到“ 个人访问令牌”字段中。

  7. 单击保存按钮。

    • 您的YouTrack项目已与GitHub中选定的存储库集成。

    • 来自GitHub存储库的引用项目中问题的提交将显示在所引用问题的活动流中。

    • 侧边栏显示用于配置VCS集成的其他设置。

    要了解有关这些设置的更多信息，请参阅集成设置 。

进阶伺服器设定

如果无法使用“ 新建VCS集成”对话框上的基本设置建立与存储库的连接，请单击“ 显示高级服务器设置”链接。

仅在目标服务器尚未与VCS集成时，才需要输入这些设置的值。如果已经与服务器上的单个存储库进行了有效的集成，则可以添加与其他存储库的集成，而无需再次设置这些参数。

使用以下准则来设置这些设置的值：

设置

描述

网址

此设置与GitHub中存储库的连接无关。对于与其他VCS托管服务的集成，此设置有助于标识存储库的路径。

GitHub存储库的路径始终由两个部分组成。“ 存储库URL”设置中这两段之前的所有内容都会自动识别为服务器的路径。

SSL密钥

如果您的服务器环境设置为需要HTTPS身份验证，请选择包含您的YouTrack服务器私钥的密钥库。尝试与GitHub建立连接时，此密钥标识您的YouTrack服务器。此设置仅用于支持内部网络连接所要求的HTTPS身份验证。

该列表仅显示已导入到YouTrack中的SSL密钥。要了解如何生成密钥库文件并将其上传到YouTrack，请参阅SSL密钥 。

整合设定

默认情况下，VCS集成处理由任何分支中的任何用户提交到存储库的更改。在YouTrack中有权访问该问题的任何用户都可以在问题活动流中查看这些更改。

如果您只想处理指定分支中特定用户的更改，或限制YouTrack中VCS更改的可见性，则可以自定义集成设置。使用以下设置来自定义集成：

设置

描述

资料库

显示集成版本控制系统中存储库的路径。

如果需要，可以在设置集成后编辑存储库的位置。有关说明，请参阅“ 编辑存储库设置” 。

提交者

限制更新提交给指定组成员的消息中的命令的问题的能力。来自非选定组成员的用户的VCS更改仍附加到相关问题，但是将忽略其提交中指定的任何命令。

监控分支

存储要监视更改的分支的名称。

  • 使用+包括一个分支。

  • 使用-排除分支。

  • 为了branch name ，请使用分支机构的全限定名称。例如， refs/heads/ 。

  • 使用*作为通配符。该占位符匹配字符串中的一个或多个字符。例如，要包括所有功能分支，请使用：

    +：裁判/负责人/功能/ *

  • 要监视所有分支，请将输入字段留空。

如果在连接到GitHub时输入为存储库URL的地址指向特定分支，则在建立连接时该分支会自动添加到受监视分支列表中。

解析提交问题评论

启用后，提交消息文本的特定部分将作为注释复制到问题。当您将部分提交消息作为注释复制到问题时，您可以触发@mention通知并将信息公开给无法访问VCS更改的用户。

此设置不会影响VCS更改中提交消息的显示方式。整个提交消息（包括命令和问题注释）始终显示为活动流中VCS更改记录的一部分。

您仅应在以下情况下启用此选项：

  • 您希望在提交消息中提及其他用户，并在将文本复制到问题注释时生成通知。

  • 您限制了VCS更改的可见性，并希望使与提交有关的信息作为注释对外部用户可见。

要了解有关YouTrack如何处理提交消息的更多信息，请参阅《 VCS提交》中的“应用命令” 。

VCS更改可见性

将VCS更改的可见性限制为YouTrack中的一个或多个用户组。不受限制时，任何有权阅读该问题的用户都可以看到VCS更改列表。

故障排除

条件 -对VCS提交中的问题的引用未显示为YouTrack中的VCS更改。

原因

解

集成的VCS中的Webhook不存在，被禁用或格式错误。

检查VCS存储库设置中的webhook。确保存在网络钩子并已将其启用。

如果您怀疑Webhook有问题，请在VCS的设置中将其删除或禁用，然后在YouTrack中设置新的VCS集成。

条件 -VCS提交中指定的命令不适用于YouTrack中的问题。

原因

解

提交对存储库所做的更改的用户不是集成设置中Committers组的成员。

将提交者添加到YouTrack中的指定组，或者修改集成设置中的选择。

提交对存储库的更改的用户无权更新所连接项目中的问题。

将这些用户添加到项目团队中，或授予他们在项目中的角色，该角色包括“ 读取问题”和“ 更新问题”权限。

YouTrack找不到与提交消息的作者匹配的用户帐户。

用户必须在YouTrack和GitHub中为其帐户使用相同的电子邮件地址，或者将存储在其GitHub配置文件中作为Name的值添加到其Hub帐户中的VCS用户名列表中。

有关更多信息，请参见匹配提交作者和YouTrack用户 。

条件 -您无法在YouTrack和GitHub Enterprise服务器之间建立连接。查看是否存在以下任何原因。

原因

解

外部服务不可用。

确认GitHub Enterprise服务器正在运行。

连接被防火墙阻止。

打开防火墙中由YouTrack和GitHub Enterprise服务器使用的端口。

代理服务器阻止了与外部服务的连接。

您正在尝试通过错误的端口进行连接。

设置服务器的系统属性，以使YouTrack通过代理服务器连接到其他服务。有关说明，请参阅代理配置 。

GitHub Enterprise服务器需要安全连接。

将GitHub Enterprise服务器的证书导入到YouTrack中。有关说明，请参阅SSL证书 。

您的GitHub Enterprise服务器的SSL证书已过期。

续订并将更新的证书导入到YouTrack中。有关说明，请参阅SSL证书 。
