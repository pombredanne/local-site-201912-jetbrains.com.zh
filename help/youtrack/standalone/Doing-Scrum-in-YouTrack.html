Scrum板

本教程介绍了遵循Scrum开发框架时如何在YouTrack中创建，配置和使用敏捷板。

Scrum简介

Scrum是关于在迭代过程中开发项目的功能。您需要使用较小的预发行版来实现产品的最终发行版，这些预发行版实现了特定功能。这些功能也称为用户故事 。

Scrum中的产品负责人协商最终产品的发布时间表。每个版本必须在截止日期之前。从开发开始到截止日期的时间段被分为较小的时间段，通常每个时间段为两周到一个月。这些有时间限制的工作称为冲刺。每个冲刺都会生成一个可行的产品预发行版本。

用YouTrack开发Scrum

YouTrack中的每个项目都会跟踪不同类型的问题，例如错误报告，功能请求和任务。

您可以通过查看列表中的问题或在敏捷板上显示问题来查看项目问题中的问题。敏捷板上的卡表示列表中显示的相同问题。但是，敏捷开发板可以使您更好地可视化当前项目状态及其进度。该板还包括专门设计用于支持Scrum开发的许多功能。

配置项目

需要权限：更新项目，读取问题，创建问题，更新问题

在开始配置Scrum板之前，请花一分钟时间选择要在板上管理的项目，并检查其配置是否正确。

如果要在新项目中使用Scrum板，则可以与Scrum项目模板同时创建项目和两个敏捷板。有关更多信息，请参见Scrum项目模板 。

Scrum开发的主要敏捷板将功能（或用户故事）显示为板上的泳道。板上的每张卡都代表一项有助于开发相应功能的任务。

您还可以创建第二个板来跟踪最终产品开发的进度。该面板将史诗般的泳道显示为泳道，并带有代表主要功能或用户故事的卡片。

为了支持此配置，您要在Scrum板上管理的所有项目都应将这些值存储在自定义字段中。在这里，我们定制了存储在“ 类型”字段中的一组值。

板上的列代表不同的交付状态。状态也存储在问题字段中。Scrum板使用一个字段将此值存储为state类型。当您在板上移动卡时，该字段将获取已移动到的列的值。

此示例显示了一个状态类型字段，该字段表示项目中的交付状态。该字段不能为空，这意味着在设置此项目时，该项目中创建的所有新问题都将显示在代表该值的列中。

为了获得最佳结果，请为使用状态类型的现有字段编辑一组值。如果您向项目添加新的状态类型字段，请确保删除所有其他状态类型字段。使用多个状态类型字段的项目可能很难使用，并且需要其他设置。有关更多信息，请参见与多个状态类型字段的可能冲突 。

创建董事会

单击标题中的“ 敏捷板”链接，从“ 创建”菜单中选择“ 板 ”，然后单击“ Scrum板”按钮。

请遵循以下准则来设置新板：

设置

指导方针

名称

给董事会起个名字。

专案

选择要在板上管理的项目。

可以查看和使用板

从列表中选择项目团队的名称。要查看卡片和泳道并使用面板，项目团队的每个成员还应该有权查看和更新所选项目中的问题。

此设置支持基于项目的访问的其他选项。选择此选项时，在主板上管理的所有项目中具有读取问题访问权限的任何用户都可以查看和使用它。

可以编辑电路板设置

如果您希望项目团队的其他成员有权访问董事会设置，请从列表中选择项目团队的名称。其他选项使您可以通过其他方式配置对板设置的访问：

  • 选择基于项目的选项时，在主板上管理的任何项目中具有“ 更新项目”访问权限的任何用户都可以更新主板设置。

  • 如果您不希望其他用户更新板设置，请将此选项设置为“ 所有者”，然后自行管理板设置。

积压

如果您已经定义了一个包含产品积压中所有问题的保存搜索，请选择“ 使用现有”选项。否则，选择创建新选项。

新积压的搜索查询格式如下：

project: has: -{Board } #Unresolved

此保存的搜索显示在敏捷板上管理的项目中尚未分配给当前板上的sprint的所有未解决的问题。

保存的搜索

选择要用于板上积压工作的已保存搜索的名称。如果选择了“ 创建新选项”，则此设置将显示在创建木板时分配给新保存的搜索的名称。

询问

筛选积压中显示的问题的搜索查询。如果选择了“ 创建新选项”，则此设置将显示为积压工作创建的已保存搜索的查询。

完成后，单击创建板按钮。“ 板设置”页面将打开“ 常规”选项卡。

选择董事会行为选项

常规选项卡显示您刚定义的创建板的设置。对于使用Scrum板模板创建的板，默认情况下，“ Sprints”选项为“ Enabled” 。启用sprint后，您可以通过三种不同的方式来选择董事会行为 。

这些选项确定如何将问题分配给板上的冲刺。

选项

描述

手动分配问题

如果要管理将哪些问题显式添加到板上，请将此默认选项保持启用状态。除非您手动添加它们，否则在板上不会看到项目中创建的新问题。使用此选项时，您可以通过整理和完善积压工作中的问题来确定所有工作的优先级。

自动向sprint添加新问题

对于Scrum开发，您通常要计划在sprint开头要完成的所有工作。此选项会将您在项目中创建的任何新问题自动添加到开发板上。这种行为违反了弹簧计划的目的，不建议用于Scrum开发。

将sprint链接到自定义字段的值

此选项将sprint与存储在自定义字段中的值同步。启用后，当您更新为链接的自定义字段设置的值时，问题将自动分配给sprint。当您要在每个冲刺中发布以匹配为其开发产品的产品迭代时，请使用此选项。

定义列和泳道

下一步是配置用于识别板上列和泳道的设置。创建板后，默认情况下，将使用在本教程第一部分中配置的自定义字段来标识列。

在“ 列和泳道”选项卡上，可以使用“ 添加列”选项将该字段中的值作为列添加到板中。

默认情况下，Scrum板的模板使用“ 类型”字段来标识泳道。

对于开发板，您将显示功能或用户案例的泳道。板上的每张卡都代表一项有助于完成用户故事的任务。当您创建其他板来跟踪最终产品的开发时，将显示史诗泳道。板上的卡片代表您在每个Sprint期间工作的用户故事。

自定义卡演示

Cards选项卡为您提供了一些选项，可用于配置Scrum板。

如果您估算完成每个任务所需的时间，则可以在板上显示估算值。使用“ 估计字段”设置选择用于在项目中存储估计的字段。这样，您可以直接在每张卡上显示和更新估算值。

您还可以设置用于突出显示板上卡片的配色方案。有两种方法可以为卡片加色。一种简单的方案是根据发行字段（例如Project或Priority）的值为卡片着色。“ 优先级”字段具有默认的配色方案。关键问题以粉色突出显示，正常问题以绿色突出显示。您可以在“ 编辑项目”页面上更新自定义字段的配色方案。

您可以对任何枚举字段使用此方法。

如果按项目为卡着色，则可以在“ 板设置”页面的“ 卡”选项卡上为每个项目分配一种颜色。

第二种方法是根据动态条件为卡片着色，例如临近期限（期限越近，卡片的红色越深）。工作流教程中介绍了这种情况。您可以使用工作流编辑器对业务逻辑进行编程，以自动更改每张卡的颜色。

配置图表

图表选项卡使您可以选择要在Scrum板上显示的报告。

Scrum开发方法使用燃尽图。该报告显示了分配给当前Sprint的所有问题的理想进度与实际进度。燃尽图可以帮助您监视进度，分析开发过程并衡量团队的速度。

使用“ 发行过滤器”设置可以限制报告计算中包括哪些卡。

您可以根据每天解决的问题数量，任务估计中存储的值或作为工作项记录的工作量来计算燃尽量。

与Scrum委员会合作

完成配置板后，您和您的团队可以使用板来管理您的工作。以下是该板的一些关键组件，并提供了一些将板与Scrum开发框架一起使用的指南。

产品积压

Scrum中的关键概念之一是维护产品积压。积压订单中包含交付最终产品的要求的有序列表。通常，待办事项确定在产品中将交付哪些功能以及它们的构建顺序。

您可以直接在面板上打开待办事项。在这里，产品负责人可以将问题拖入分配给冲刺进行开发的顺序。由于积压订单是基于保存的搜索，因此您也可以在“ 问题”列表中打开积压订单。对“ 问题”列表中顺序的更改将实时应用于敏捷板上的待办事项。

计划您的第一个冲刺

收集了要开发的所有功能部件或用户存储并在积压订单中优先考虑了这些问题之后，就可以计划并执行第一个冲刺了。

短跑通常在Scrum团队举行计划活动时开始。在此事件期间，团队确定可以在sprint中完成哪些用户故事。然后，他们确定传递每个用户故事所需的任务，从而创建sprint积压。

您可以使用敏捷板来简化此过程。要创建sprint积压，只需将问题作为子任务添加到积压的用户存储中。

创建Scrum板时，默认情况下会创建计划外的sprint。要安排您的第一个冲刺，请打开“ 冲刺”下拉列表并编辑冲刺。

在“ 编辑冲刺”对话框中，将要在此冲刺中提供的功能描述为冲刺目标 。冲刺目标显示在面板的底部，并且在冲刺过程中对团队可见。

接下来，打开日历并设置冲刺的时间表。

现在，您可以将问题作为泳道添加到sprint中。您可以将问题从待办事项拖到董事会，也可以使用“ 添加泳道来冲刺”选项。此选项使您可以选择是否也要将sprint待办事项中的子任务添加到板中。

每日Scrum

在定义了第一个冲刺之后，团队就可以使用董事会来跟踪他们为当前冲刺所做的努力。Scrum团队的一种常见做法是参加每日的Scrum或站起来。任何阻碍冲刺进度的障碍都可以被识别出来，并作为新的问题或错误记录在板上。

当您的团队完成每项任务时，卡片将从木板的左侧移动到代表最终状态的最右侧的列。

如果团队出于某种原因决定放弃任务，则可以将其从冲刺中删除。您可以打开待办事项并将卡从开发板拖到待办事项。或者，选择多个卡并使用页脚中的选项将选定的卡发送到待办事项列表。

卡大小和电视模式

板顶部的滑块可设置板上卡片的详细程度。此设置是用户特定的，可以为不同的板单独设置。

一些团队在日常混乱期间使用大型监视器或投影仪来显示板。电视模式以全屏显示面板，并使用深色背景的配色方案。实践证明，这种视觉效果可以提高大型监视器的远距离可视性。

进度指标

进度指示器按状态显示卡的分配。每个条形图对应板上的一列。

要查看每列中的总牌数，请将指针移到进度指示器中相应的列上。每个泳道中的卡数显示在泳道的右侧。

如果启用了“ 估计字段”设置，则面板将在面板顶部显示所有卡的总估计。呈现方式根据用于估计完成任务所需时间的自定义字段的数据类型而有所不同。

本示例将估计显示为持续时间。每个泳道中所有卡的总估计值显示在泳道的右侧。

追踪进度

您可以直接在板上打开燃尽图。该图表显示了理想的燃尽量，该燃尽量是根据冲刺开始时的任务数或总估算值计算得出的。剩余工作量行表示分配给冲刺的问题数或未解决问题的总估算。将指针移到图表上的日期上，以比较这些值之间的差异，并查看您是提前还是落后于时间表。

垂直轴表示剩余的工作量。横轴表示日期时间。该图表显示了理想日子的估算值。在选定的时间点，项目团队将提前完成分配给该Sprint的任务。

后续冲刺

当您完成第一个冲刺时，循环将继续。团队通常在sprint结束时进行回顾和回顾，以回顾sprint期间完成的工作并确定下一次迭代的改进。

以下sprint从另一个sprint计划事件开始。在确定了要在下一个sprint中完成的用户素材后 ，请打开“ Sprints”下拉列表，然后选择“ New sprint”选项。

如果您有任何未解决的问题分配给上一个sprint，则可以将其自动移动到新的sprint。

顺其自然。随着您继续开发新功能，产品积压的数量正在减少。如果您按计划进行并按时交付，那么最终将获得最终产品版本，可以发布给客户...

...并开始开发下一个版本！
