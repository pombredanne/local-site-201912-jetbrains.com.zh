自定义字段类型

自定义字段类型是一种分配给该字段中存储的值的格式。此格式确定YouTrack如何解释数据。

支持的类型可以分为两类：

  • 简单类型 -这些类型包括文本，句点，日期，日期和时间，整数和字符串类型。

  • 枚举类型 -这些类型使您可以从预定义列表中选择值。

简单类型

简单类型以特定格式存储单个值。

栏位类型

描述

string

将单个值存储为字符串。

text

存储问题中显示的以Markdown语法解释的字符串。此类型的字段不会与其他自定义字段一起显示在边栏中。相反，它们显示在问题说明的下方。与问题摘要和说明一样，您只能更改基于文本的字段的值，然后在创建或编辑模式下打开问题。

编辑这些字段时，输入预览会以Markdown格式显示。

date

以日期格式存储单个值。用户通过使用日历控件选择日期来设置和更改值。

日期以当前用户在配置文件页面上的日期格式设置中选择的格式显示。

date and time

以日期格式存储单个值。完整的日期和时间显示在自定义字段中。用户使用输入字段设置和更改值。

日期和时间以当前用户在“ 个人资料”页面上的“ 日期格式”设置中选择的格式显示。

period

存储代表一段时间的值。将数据存储为期间类型的字段用于时间跟踪。

integer

将单个值存储为整数。

float

将单个值存储为浮点数。

枚举类型

枚举类型存储一组预定义的值中的一个或多个值。对于这些类型中的许多类型，可以将其他属性分配给集合中的每个值。有关更多信息，请参见特定于值的设置 。

存储一个字段state类型只能存储单个值。您可以配置任何其他枚举类型来存储单个或多个值。

栏位类型

描述

enum

存储预定义的一组值。这种类型的字段的值集是存储为string类型。

group

存储对组的引用。一组可能的值取自YouTrack中的组列表。自定义字段显示组名。可用组列表基于“ 读取组”权限分配。

user

存储对用户帐户的引用。默认的“ 受理人”字段使用此类型。这组可能的值取自YouTrack中的用户列表。自定义字段显示全名。下拉列表还显示用户登录名。可用用户列表基于“ 读取用户”权限分配。

ownedField

存储预定义的一组值。默认的“ 子系统”字段使用此类型。

这种类型类似于enum类型，除了一组值中的每个值都存储一个owner 。此属性将对子系统负责的用户的引用存储为user类型。

state

存储预定义的问题状态集中的问题状态。

这种类型也是enum类型。在此，一组值中的每个值都有一个isResolved属性。此属性指示在此字段中为其分配了相应的值后，是否认为该问题已解决。此值存储为Boolean类型。

有关此类型的更多信息，请参见与多个状态类型字段的可能冲突 。

version

存储预定义版本列表中的版本。

版本集中的每个值都存储以下信息：

  • releaseDate -将发布日期存储为date类型。可以对应于实际或计划的发布日期。

  • released —指示版本是否已发布，并存储为Boolean类型。

  • archived —指示版本是否已存档，存储为Boolean类型。

如果某个版本标记为已存档，则该版本在自定义字段的值列表中不再可见。这意味着您不能选择该值并将其添加到该字段，也不能选择并从字段中删除一个值。

归档版本仍可用于命令，工作流和REST API。这意味着您可以使用命令来添加或删除归档值或以编程方式更新这些值。

build

存储来自一组预定义的内部版本号的内部版本号。

构建集中的每个值都存储一个assembleDate 。此属性存储将所选内部版本组装为日期的日期。 date类型。

与多个状态类型字段的可能冲突

如果您添加多个使用state键入您的项目，您可以创建一些不良结果。在为所有状态类型字段分配了被认为可以解决的值之前，问题不会得到解决。除非您的用户被告知此要求，否则会引起混淆。当他们更新一个状态字段的值来解决问题时，问题的直观表示将保持不变-不会像预期的已解决问题那样用问题ID的删除线文本变暗。

因此，我们强烈建议不要在项目中使用多个状态字段。

如果您没有足够的幸运在您在项目中创建问题之前阅读此警告，那么仍然有希望。以下解决方案可以帮助您获得所需的功能，而不会惹恼您的团队。

  • 最简单的解决方法是将所有额外状态类型字段的所有值标记为已解决。您的用户只需管理单个自定义字段中的值即可确定问题是否已解决。

  • 首选解决方案是将其他状态类型字段替换为使用通用枚举类型的字段。这使解析状态取决于单个自定义字段的值。

    不幸的是，您只能对具有相同数据类型的自定义字段使用“ 替换”功能。使用以下过程创建新的枚举自定义字段，并从项目中删除其他状态类型字段。

要替换其他状态类型字段：

  1. 打开管理>自定义字段页面。

  2. 找到包含额外的状态类型自定义字段的项目。

  3. 单击将字段添加到项目按钮。

  4. 使用设置来定义一个自定义字段，用于存储enum类型。

  5. 将相同的值添加到在新的状态类型自定义字段之一中使用的新的自定义字段。

  6. 完成后，更新存储在项目的自定义字段中的值：

    • 从“ 问题”列表中，输入搜索查询，该查询将在状态类型自定义字段中查找包含特定值的所有问题。从集合中的第一个值开始。例如： In #{Project} Status: Submitted 。

    • 选择搜索查询返回的所有问题。使用键盘快捷键Ctrl + A （ ⌘ + A在macOS上）以选择查询返回的所有问题。

    • 使用命令来更新所有选定的问题，并在新的枚举自定义字段中为它们分配相同的值。例如： Secondary State Submitted 。为了最大程度地减少垃圾邮件，请静默应用命令。

    • 重复此步骤，直到将状态类型自定义字段中存储的所有值添加到新的枚举自定义字段中。

  7. 返回“ 管理>自定义字段”页面，然后从项目中删除额外的状态类型自定义字段。

    • 该字段及其相关值将从项目中删除。

    • 以前存储在此自定义字段中的所有值现在都存储在枚举的自定义字段中。

  8. 重复此过程，直到您替换了项目中所有其他状态类型字段。
