YouTrack Standalone 2019.2帮助

还原JAR安装

要成功地从备份还原数据库，备份文件的版本应与将数据库还原到的YouTrack版本相同或早于该版本。

在开始之前，请确保您具有要还原的数据库的备份副本。请注意备份副本的位置，因此请确保在此过程中选择正确的文件。

如果您没有有效的备份文件，或者您的备份文件已过期，请按照说明进行操作，以还原没有数据库备份的当前版本 。

  • 如果升级失败后回滚到以前的安装版本，请使用在安装之前创建的备份。如果您忽略此先决条件，请在设置为与先前安装版本相对应的“ 备份”位置的目录中找到已归档的备份文件。

  • 如果尝试还原当前安装，请在设置为与当前安装版本相对应的“ 备份”位置的目录中找到已存档的备份文件。

从数据库的备份副本还原安装

要将数据库还原到当前版本或先前版本，您需要删除当前数据库，并将其替换为要还原到的数据库副本。

还原到YouTrack的先前版本时，请使用与要还原的产品版本相对应的备份。要成功还原到以前的版本，此备份文件一定不能用作安装该产品的更高版本的升级源。

要从备份还原数据库：

需要权限：低级管理员写

  1. 停止YouTrack服务。有关详细信息，请参阅停止JAR分发 。

  2. 从数据库位置目录中删除所有内容。默认情况下，数据库位于${user.home}/teamsysdata运行YouTrack服务的用户帐户的文件夹。

    管理员可能已更改了默认数据库位置。有关更多信息，请参见更改数据库位置 。您还可以找到数据库的实际位置，如下所示：

    1. 打开${user.home}/.youtrack/app/internal/conf/installation.xml文件。

    2. 在文件中，找到conf-location定义到的路径的属性conf的子文件夹data目录。因此，您服务的数据库位置是到父级的路径data目录conf-location 。

  3. 将下载的备份文件解压缩到${user.home}/teamsysdata夹。

  4. 确保您的YouTrack服务用户对解压缩的文件具有读/写（“ rw”）权限。

  5. 运行与您的YouTrack数据库版本相对应的JAR文件。

    • YouTrack从备份文件中的数据开始。

在没有数据库备份的情况下还原当前版本

在某些情况下，您将无法从备份还原安装。

  • 您的备份文件丢失或已过期。

  • 服务器上的备份文件已损坏，无法使用。

例如，当服务器崩溃后能够恢复数据，并且在YouTrack中记录的信息不在最新的备份文件中时，您可能还希望遵循此过程。

您可以尝试使用数据库的有效副本来还原安装的当前版本。仅当数据库文件未损坏时，此还原过程才能成功。仅当您拥有从未用于启动该产品更高版本的文件的副本时，才可以还原到YouTrack的早期版本。

要还原数据库而不进行备份：

需要权限：低级管理员写

  1. 找到数据库的最新副本。目录的默认位置在${user.home}/teamsysdata运行YouTrack服务的用户帐户的文件夹。

    请注意，管理员可能已经更改了YouTrack存储这些文件的位置。您也可以使用这些目录的副本，这些副本存储在其他位置。

  2. 停止YouTrack服务。

  3. 从数据库位置目录中删除所有内容。

  4. 将数据库副本移动到${user.home}/teamsysdata夹。

  5. 确保您的YouTrack服务用户对该文件夹具有读/写（“ rw”）权限。

  6. 运行与您的YouTrack数据库版本相对应的JAR文件。

    • YouTrack从复制到数据库位置目录的数据开始。
