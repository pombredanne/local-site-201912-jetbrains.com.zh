从Mantis Bug Tracker导入

按照此页面上的指示信息，使用Python导入脚本从Mantis Bug Tracker （MantisBT）导入问题。此过程需要执行以下步骤：

  1. 设置您的本地环境 。

  2. （可选） 自定义MantisBT数据库中的字段与YouTrack中的issue属性之间的映射 。

  3. 导入您的问题 。

导入细节

如果MantisBT数据库包含对YouTrack中不存在的实体的引用，则会创建它们。您用于运行导入的用户帐户应具有创建项目，问题，用户，甚至更多的权限。我们建议您使用具有系统管理员角色的帐户来执行导入脚本。

专案

如果导入源中引用的项目尚不存在，则会创建一个新项目。

  • 根据MantisBT数据库中的值分配项目名称和ID。

  • 将执行导入脚本的用户设置为项目所有者。

  • 父项目及其子项目被导入到单个项目中。映射文件包含一个参数，可用于在属于子项目的问题中引用父项目。

用户数

几个问题属性，例如reporterName和Assignee存储对用户帐户的引用。通过Login对照现有的用户帐户检查导入中的值。对于与现有用户的登录名不匹配的每个新值，将创建一个新的用户帐户。

  • YouTrack在登录中不支持空格字符。中的空白字符mantis_user.user_name下划线代替。

  • 所有新用户帐户均分配有随机生成的密码。要登录YouTrack，导入的用户必须单击登录页面上的重置密码链接并创建新密码。

导入数据中对用户的引用将映射到YouTrack中的现有帐户。如果找不到匹配项，则会创建一个新的用户帐户。

根据您的订阅计划，为导入的用户分配了许可证。对于超出您的每用户许可限制的用户，导入的用户被标记为禁止用户。

有关更多信息，请参见禁止用户帐户 。

自定义字段

如果导入数据包含YouTrack中不存在的字段，则将创建自定义字段并将其添加到目标项目中。

  • 字段名称和类型是根据映射文件中的定义设置的。

  • 自定义字段的属性设置如下： autoAttached=true ， isPrivate=false ， defaultVisibility=false 。

栏位值

如果源文件包含现有字段的新值，则将它们添加到当前值集中。

设置环境

Python导入脚本支持“导入到YouTrack”。这些脚本基于YouTrack REST API构建。实际上，您实际上不需要使用Python进行任何编程，但是您确实需要安装Python和包含导入脚本的软件包。

设置环境：

  1. 下载并安装Python 。Python客户端库与Python 2.7+兼容。不支持Python 3版本。您可以选择自己喜欢的安装目录。

    macOS，CentOS，Red Hat Enterprise Linux（RHEL）和Ubuntu的最新版本均随附Python 2.7。如果使用任何这些操作系统，请继续下一步。

  2. 为MySQL数据库安装Python库。例如， MySQLdb 。

  3. 安装包含导入脚本的软件包。打开操作系统支持的命令行界面，然后输入以下命令：

    pip install youtrack-scripts

    • 导入脚本安装在您的本地环境中。

    • 的youtrack包含YouTrack REST API的Python客户端库的软件包会自动作为依赖项安装。

    如果您不想全局安装软件包，则只能使用以下命令为您的用户帐户安装软件包：

    pip install --user youtrack-scripts

自定义映射定义

下一步是映射MantisBT数据库中的字段以在YouTrack中发布属性。您可以使用mantis2youtrack脚本以生成示例映射文件。

仅当您在MantisBT中自定义了问题属性时，才需要自定义映射文件。要使用默认的映射定义，请跳过此过程并导入数据 。

生成映射文件：

  1. 打开操作系统支持的命令行界面。

  2. 如有必要，将当前目录更改为Python的安装目录。例如（Windows）：

    cd C:\Python27

  3. 输入以下命令：

    mantis2youtrack -g -m mapping.json

    • 的-g选项告诉脚本生成映射文件。

    • 的-m选项将覆盖映射文件的默认路径和文件名。如果不通过此选项，则文件将在当前目录中生成为mapping.json 。如果要指定其他位置和/或文件名，请使用目标目录的完整路径和/或所需名称为参数指定一个值。

    • 如果命令执行成功，则会在命令行界面中显示一条消息，其中包含生成的文件的位置。

要自定义映射定义：

  1. 打开生成的映射文件。

  2. 更新映射，以确保要从MantisBT数据库导入的所有数据都映射到YouTrack中所需的值。如果文件包含一个或多个不想导入的字段，则可以从映射文件中删除引用。有关默认映射的说明，请参见默认映射 。

  3. 检查是否为charset参数与您的MantisBT数据库的字符编码匹配。MantisBT版本1.1.0x和更高版本使用UTF-8字符编码。

  4. 要确定在导入期间如何处理子项目，请为batch_subprojects参数。

    • 如果True ，则会将子项目自定义字段添加到YouTrack中的目标项目。在属于子项目的问题中，子项目名称被设置为此字段的值。对于父项目中的问题，该值设置为Undefined 。

    • 如果False ，则会忽略父项目和子项目之间的关系。

  5. 保存您的更改并关闭映射文件。

汇入资料

最后一步是运行从MantisBT导入数据的命令。

导入数据：

  1. 打开操作系统支持的命令行界面。

  2. 如有必要，将当前目录更改为Python的安装目录。例如（Windows）：

    cd C:\Python27

  3. 输入以下命令：

    mantis2youtrack -m mapping.json -t token -u login -p password target_url mantis_db_name mantis_db_host mantis_db_port mantis_db_login mantis_db_pass mantis_project mantis_project mantis_project

    将命令行参数替换为如下所述的值：

    参数

    描述

    mapping.json

    与-m选项，以指定自定义映射文件的完整路径和文件名。如果不通过此选项，则使用默认的映射文件。

    token

    当与-t选项，一个永久令牌，用于在YouTrack中验证您的管理员帐户。这是首选的身份验证方法。如果您使用此选项，请跳过-u和-p选项。要了解如何生成永久令牌，请参阅创建永久令牌 。

    或者，您可以使用-T选项，并为包含永久令牌的文件指定完整路径和文件名。

    login

    当与-u选项，YouTrack管理员帐户的登录名。您还必须为-p选项。

    password

    当与-p选项，YouTrack管理员帐户的密码。

    target_url

    目标YouTrack服务器的基本URL。对于YouTrack InCloud实例，您的基本URL包括结尾/youtrack 。例如： https://company.myjetbrains.com/youtrack

    mantis_db

    MantisBT导入的源数据库的名称。

    mantis_host

    MantisBT源数据库的URL。

    mantis_port

    用于访问MantisBT源数据库的端口。如果未特别配置，则默认端口号是3306。

    mantis_login

    登录到MantisBT源数据库的用户名。

    mantis_pass

    用于登录MantisBT源数据库的帐户的密码。

    mantis_project

    要从MantisBT导入的源项目的名称。要导入多个项目，请用逗号分隔项目名称。

    • 该命令执行Python导入脚本。

    • 如果成功，则会在命令行界面中针对每个问题打印以下行：

      已成功导入问题[ ]

  4. 检查您将问题导入的一个或多个项目，并验证数据是否正确显示。如果您对结果不满意，并想重新导入数据：

    • 删除导入期间创建的所有问题。如果将问题导入到新项目中，则只需删除该项目。

    • 编辑您的映射文件。

    • 再次运行导入脚本。

默认映射

Python客户端库包含一个默认的映射文件，用于将问题从MantisBT导入到YouTrack。该文件另存为defaultMantis.py在里面youtrackutils/mantis存储库的子文件夹。

导入脚本引用默认映射文件。如果您创建自定义映射文件，请使用-m导入脚本中的选项，然后如上一节所述指定文件的完整路径和文件名。

自定义字段

您可以在MantisBT和YouTrack中的问题字段之间指定其他映射。要定义其他映射，请将定义添加到field_names字典。默认值defaultMantis.py映射文件包含以下自定义字段的定义：

field_names = { u"severity" : [u"Severity", u"Type"], u"handler" : [u"Assignee"], u"status" : [u"State"], u"resolution" : [u"State"], u"category" : [u"Subsystem"], u"version" : [u"Affected versions"], u"fixed_in_version" : [u"Fix versions"], # DO NOT remove Fix versions form this list, if needed you can add one more field # but DO NOT delete Fix versions u"build" : [u"Fixed in build"], u"os_build" : [u"OS version"], u"subproject" : [u"Subproject"], u"os" : [u"OS"], u"due_date" : [u"Due date"], u"target_version" : [u"Target version"] # it's better to import this fields with version type }

特定字段的字段类型

您还可以使用以下命令修改目标字段类型（YouTrack中的字段类型） field_types字典。对于YouTrack中的默认自定义字段，请使用默认字段类型定义。

在默认映射文件中定义了以下字典：

field_types = { u"Priority" : "enum[1]", u"Type" : "enum[1]", u"State" : "state[1]", u"Fix versions" : "version[*]", u"Affected versions" : "version[*]", u"Assignee" : "user[1]", u"Fixed in build" : "build[1]", u"Subsystem" : "ownedField[1]", u"Subproject" : "ownedField[1]", u"Severity" : "enum[1]", u"Platform" : "string", u"OS" : "string", u"OS version" : "string", u"Reproducibility" : "enum[1]", u"Due date" : "date", u"Target version" : "version[1]" }
