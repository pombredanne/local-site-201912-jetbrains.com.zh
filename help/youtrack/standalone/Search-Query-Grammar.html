搜索查询语法

本页提供有关YouTrack搜索查询语法的BNF描述。

语法

:: = :: = （'or' ）* :: = （'and' ）* :: ='（' ？'）'|术语:: = * :: = | | | | | | | :: = '：' （'，' ）* :: = :: =（'-'？ ）| （'-'？） :: = '..' :: ='＃' :: ='-' :: = :: ='排序依据：' （'，' ）* :: = （'asc'|'desc'）？ :: ='具有：' （'，' ）* :: ='“' '”' :: ='-' :: = :: = | :: = :: ='{' '}'

语法不区分大小写。

有关搜索属性的完整列表，请参阅《 搜索查询参考》 。

例子

询问

描述

"export configuration"

返回包含与摘要，描述或注释中的指定顺序的“导出”和“配置”相匹配的单词格式的问题。

for: me #unresolved

返回分配给当前用户的未解决问题。

for: me unresolved

返回分配给当前用户的问题，这些问题包含与摘要，描述或注释中的“未解决”相匹配的单词格式。

for: mazine

返回分配给用户名为“ mazine”的用户的问题。

mazine

返回包含与“ mazine”匹配的单词形式的问题。

#mazine

返回由用户名“ mazine”创建或分配给用户的问题。

-mazine

返回由任何用户创建或分配给任何用户的问题，但用户名“ mazine”的用户除外。

state: -open, -submitted

返回所有状态为“打开”或“已提交”的问题。

sort by: priority, issue id asc

返回所有问题的列表，这些问题首先按优先级排序，然后按问题ID升序排列。

tag: A, B

返回带有标签“ A”或“ B”的问题的列表。该查询被解析为“ A or B的表达。

tag: A tag: B

返回带有标签“ A”或“ B”的问题的列表。该查询被解析为“ A or B的表达。同一类别中的两个值被视为析取值。

tag: A state: B

返回带有标签“ A”和状态“ B”的问题的列表。该查询被解析为“ A and B的表达。不同类别中的值被视为合取值。

tag: A or tag: B

返回带有标签“ A”或“ B”的问题的列表。该查询被解析为“ A or B的表达。

tag: A and tag: B

返回同时具有标签“ A”和标签“ B”的问题的列表。该查询被解析为“ A and B的表达。
