有特定问题的操作

此资源使您可以处理问题的注释。

资源资源
/ api / issues / {issueID} / comments / {commentID}

退货实体

IssueComment 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读特定的IssueComment 。

  • POST ： 更新特定的IssueComment 。

  • DELETE ： 删除特定的IssueComment 。

IssueComment属性

表示现有问题评论。

下表描述了IssueComment实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

文本

串

评论文本。 Can be null 。

使用Markdown

布尔型

如果为true，则注释文本将解析为Markdown。如果为false，则注释文本将解析为YouTrack Wiki。更改此值不会将标记从一种语法转换为另一种语法。

textPreview

串

使用Wiki / markdown（包括HTML标记）处理后，在UI中显示的注释文本。 Read-only 。

被创造

长

创建注释的时间作为时间戳记。 Read-only 。 Can be null 。

更新

长

上次更新评论的时间为时间戳。 Read-only 。 Can be null 。

作者

 用户

创建评论的用户。 Read-only 。 Can be null 。

问题

 问题

评论所属的问题。 Read-only 。 Can be null 。

附件

 IssueAttachments数组

附加到评论的附件列表。

能见度

 能见度

评论的可见性设置。他们定义允许谁查看评论。 Can be null 。

已删除

布尔型

什么时候true ，则认为该评论已删除，否则为假。

阅读特定的IssueComment

阅读具有特定ID的评论。

所需权限

需要对该问题的读取权限。如果用户是评论的作者，则评论始终可见。除此以外
