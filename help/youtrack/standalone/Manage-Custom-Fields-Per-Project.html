管理自定义字段

作为项目管理员，您可以自定义用于项目中问题的字段。系统管理员可以修改系统中所有项目的自定义字段的属性。有关更多信息，请参见自定义字段 。

要访问和管理项目中用于问题的字段，请在“ 编辑项目”页面中选择“ 字段”选项卡。

添加自定义字段

作为项目管理员，您可以为项目中的问题添加新字段。

将字段添加到项目时，自定义字段可用于系统中的其他项目。具有全局项目管理权限的系统管理员或用户可以在其他项目中重复使用此字段。对于枚举的自定义字段，可以为每个项目独立管理一组可用值。

创建自定义字段时，请确保将要分配的字段类型设置为存储在该字段中的值。对于大多数字段类型，类型是在创建字段时设置的，无法更改。将值从一种数据类型转换为另一种是手动过程，这需要花费时间和精力。有关更多信息，请参见更改字段类型 。

要添加字段：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 单击将字段添加到项目按钮。

    • 侧边栏中将打开“ 新建自定义字段”对话框。

  3. 创建新字段时，字段名称必须唯一。如果要使用的字段已经存在，请按照说明将字段附加到项目中 。

  4. 在自定义字段设置中输入值。有关每个设置的详细说明，请参阅“ 自定义字段设置” 。

  5. 如果该字段是枚举类型，则可以使用以下选项：

    • 选择新设置选项。创建自定义字段后，定义一组值。

    • 从列表中选择一个字段。选择字段中使用的一组值将添加到新字段中。

  6. 单击添加新字段按钮。

    • 自定义字段将添加到您的项目中。

将自定义字段附加到您的项目

如果要在项目中使用现有字段来解决问题，可以将其附加到项目中。

要将字段附加到您的项目：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 单击将字段添加到项目按钮。

    • 在边栏中打开“ 将字段添加到项目”对话框。

  3. 选择使用现有选项卡。

  4. 选择要附加到项目的字段。使用搜索框过滤自定义字段列表。

  5. 单击字段名称旁边的+图标，将其附加到项目。

    • 所选字段的设置显示在侧栏中。

    • 如果在其他项目中使用了该字段，则只有对使用该字段的所有项目具有“ 更新项目”权限时，才可以修改自定义字段的值集。您可以修改项目中字段的属性，而不会影响使用该字段的其他项目。

      如果当前值集与您的项目兼容，则可以使用该字段而无需进行任何更改。

  6. 如果要自定义字段的值集，请单击“ 制作独立副本”按钮。

    • 在您的项目中创建了一组值的副本。

    • 您可以自由编辑值集，而不会影响其他项目中该字段的值集。

私人定制字段

您可以通过将自定义字段设置为私有或公开来管理它们的可用性。将自定义字段标记为私有后，只有在项目中具有“ 读取问题私有字段”或“ 更新问题私有字段”权限的用户才能分别查看或编辑该字段。

当某个字段是公共字段时，在您的项目中具有“ 读取问题”或“ 更新问题”权限的任何用户都可以查看或编辑该字段。

此功能可以通过以下方式帮助您限制对项目中数据的访问：

  • 隐藏存储敏感信息（例如个人数据）的字段。

  • 限制可以编辑字段中的值的用户数。例如，您可以在项目中配置访问权限，以便用户可以查看当前的受让人，但只有经理可以重新分配问题。

要将字段设为私有：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 在字段列表中选择一个或多个要隐藏的自定义字段。

  3. 点击工具栏中的设为私人按钮。

    • 该自定义字段仅对在您的项目中具有“ 读取问题专用字段”权限的用户可见。

    • 只有在项目中具有“ 更新问题专用字段”权限的用户才能更改自定义字段的值。

    • 自定义字段在列表中的字段名称之前显示有键图标。

默认情况下， 优先级和截止日期的自定义字段标记为私有。如果要将这些字段提供给项目中的所有用户使用，请选择这些字段，然后单击工具栏中的“公开”按钮。

必填的自定义字段

您可以根据项目为任何自定义字段配置“ 空值”设置。您为此设置选择的选项确定在项目中报告问题时是否需要该字段。

  • 对于存储简单类型的字段（ string ， date ， period ， integer ， 要么float ），则当“ 空值”选项设置为“不能为空”时，此字段为必填字段。

  • 对于存储枚举类型的字段，当“ 空值”选项设置为“不能为空”并且“ 默认值”选项设置为“ 无值（必填）”时，此字段为必填字段。

当用户报告项目中的问题时，必填字段将在输入字段中显示消息“ 设置值 ”。

在项目中需要字段之前，请考虑以下后果：

  • 如果您需要手动输入同样为私有的字段，那么将阻止没有权限读取和更新私有字段的用户在项目中创建问题。

  • 如果您采用有条件显示的字段并将其配置为必填字段，则只有在满足显示字段的条件时才需要该字段。

  • 仅在报告新问题或用户直接编辑所需字段时才检查字段要求。当用户将问题移到另一个项目时，新项目中所需的字段可以保留为空。

有条件的自定义字段

在每个项目中，只有在另一个自定义字段中选择了一个值时，才可以显示特定的自定义字段。最常见的用例是选择针对不同问题类型显示哪些字段。但是，您可以添加条件以根据任何枚举字段中的选择来显示特定字段，而不仅仅是问题类型。

在为自定义字段设置条件之前，请检查项目中的工作流程。尝试为隐藏在条件后面的自定义字段设置值的工作流会产生错误。

您可以将条件添加到项目中的任何字段。条件自定义字段受到以下限制：

  • 您只能根据存储自定义字段的自定义字段中的选择设置显示自定义字段的条件enum ， state ， ownedField ， version ， 要么build作为单个值。

  • 无法设置链接的依赖项。这意味着您不能使自定义字段的可见性取决于多个条件级别。

在添加用于显示自定义字段的条件之前，请考虑以下后果：

  • 隐藏在条件后面的字段不存储值。这意味着您不能仅使用此功能来隐藏敏感信息。如果要限制对存储个人数据或其他敏感信息的字段的访问，请改用私有字段。有关更多信息，请参见私人和公共发行字段 。

  • 隐藏自定义字段时，其值将被清除。如果将条件添加到包含一个或多个问题中的值的字段，则将在所有不符合条件的问题中清除该值。首先检查要用于创建条件的自定义字段，并确保它在所有问题中都包含所需的值。

    当您要用来隐藏条件自定义字段的值用作默认值时，这尤其重要。为避免丢失数据，请将用于创建条件的字段的值设置为在存储值的所有问题中显示条件字段的值。然后，您可以设置条件并默认在新问题中隐藏该字段。

  • 如果自定义字段隐藏在条件后面，则无法使用工作流程设置该字段的值。如果您使用工作流来管理条件自定义字段中的值，请确保您的工作流不会更新隐藏的字段。您可以自定义工作流程以在应用更改之前检查条件。

设置条件自定义字段：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 从列表中选择要有条件显示的自定义字段。

  3. 展开侧栏中的“ 条件设置”部分。

  4. 从仅在以下情况下显示下拉列表中，选择要用于条件的自定义字段。

  5. 从“ 设置为”下拉列表中，选择一个或多个值。

    • 在属于您的项目的问题中选择此处指定的值之一时，将显示条件字段。

在项目级别更改现场订单

管理员在全局级别设置在“ 问题”列表和“单个问题”视图上显示字段的默认顺序。如果您希望对字段进行不同的排序，则可以为项目设置自定义订单。项目级设置将覆盖全局设置。

更改现场订单：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 通过将自定义字段拖入所需的顺序来设置其顺序。

编辑一组值

您可以编辑字段中使用的一组值。

无论您可以在何处更改字段的值，都可以选择向该集合添加新值。作为项目管理员，您可以将值添加到项目唯一的字段中，也可以使用一组独立的值。全局项目中的系统管理员和具有项目管理员访问权限的用户可以将值添加到所有项目中的集合中。创建后，新值将添加到问题中。此选项在以下位置可见：

  • 问题列表

  • 单期观点

  • 敏捷板

如果该值集与其他项目共享，则可以制作该值的独立副本并编辑项目中字段的值。

要编辑与其他项目共享的一组值：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 选择使用一组值的字段。

    • 所选自定义字段的详细信息对话框在侧栏中打开。

  3. 单击制作独立副本按钮。

  4. 编辑值集。提供以下选项：

    • 单击添加值按钮以将新值添加到列表。

    • 单击“ 从以下位置复制值”按钮，然后选择一个字段以复制所选字段中使用的值。来自所选字段的值将添加到当前值集中。

    • 单击合并方式按钮，然后选择一个字段以将现有值与所选字段中使用的值合并。所选字段中的值与当前值集合并。

对集中的值进行排序

默认情况下，集合中的值按添加到字段的顺序手动排序。手动订购时，将为每个值分配一个序数值，以表示其在列表中的位置。

排序顺序会影响YouTrack中的以下行为：

  • 自定义字段的下拉列表中显示的值以指定的排序顺序显示。例如，默认的优先级字段的值从高到低手动排序。代表最高优先级的值（ Show-stopper ）出现在列表的顶部。

  • 在指定值范围的搜索查询中，排序顺序确定哪些值与搜索请求匹配。例如，包含以下内容的搜索请求Priority: Show-stopper .. Major返回的问题包括Show-stopper ， Critical和Major 。

作为项目管理员，您可以确定项目中使用的字段中的值的排序顺序。系统管理员可以在全局级别上为多个项目中使用的字段设置排序顺序。有关更多信息，请参见对集合中的值进行排序 。

您可以在“ 编辑项目”页面的“ 字段”选项卡上的自定义字段中对值进行排序。排序选项适用于所有存储枚举类型的字段，但不包含存储枚举类型的字段user要么group类型。

  • 对于存储一个user类型，则值将按全名自动按字母顺序排序，当前用户位于列表顶部。

  • 对于存储一个group类型，值按名称的字母顺序排序。

手动排序值

如果要覆盖值在自定义字段中出现的默认顺序，则可以手动使用列表中的值。

要使用手动排序的值：

需要权限：更新项目

  1. 选择一个具有一组可以排序的值的自定义字段。

  2. 如果将自定义字段附加到一个或多个项目，则可以使用以下选项：

    • 编辑所有受影响项目的集 -自定义排序顺序将应用到所有使用此值集的项目中。要使用此选项，您需要具有更新所有受影响项目的权限。

    • 进行独立复制 -自定义排序顺序存储在当前项目中字段的值集中。

    选择适合您目的的选项。

  3. 将值拖到所需的顺序。

    • 自定义订单将自动保存。

按名称对值排序

对于任何支持手动排序的字段，您可以选择按名称自动对值进行排序。值可以按名称升序或降序排序。按名称排序时，添加到集合中的新值将按字母顺序显示。按名称排序有两个选项：

  • 区分大小写的选项按字母顺序从A-Z，然后是a-z排序值。

  • 不区分大小写的选项按字母顺序对值进行排序，而不管第一个字母使用哪种大小写。

当您从手动排序切换到任何其他排序选项时，手动选项将被丢弃。如果您手动恢复为排序值，则必须重新应用自定义顺序。

按名称对值进行排序：

需要权限：更新项目

  1. 选择一个具有一组可以排序的值的自定义字段。

  2. 如果将自定义字段附加到一个或多个项目，则可以使用以下选项：

    • 编辑所有受影响项目的集 -排序顺序将应用到所有使用此值集的项目中。要使用此选项，您需要具有更新所有受影响项目的权限。

    • 进行独立复制 -排序顺序存储在当前项目中该字段的默认值集中。

    选择适合您目的的选项。

  3. 打开排序菜单，然后选择“ 按名称排序”选项（区分大小写或不区分大小写）。

  4. 在“ 更改排序顺序”对话框中确认操作。

    • 如果这些值以前是手动订购的，则手动订购将被丢弃。

    • 集合中的值按名称按字母升序排序。

  5. 如果要按降序对值进行排序，请从下拉列表中选择desc选项。

按版本对值排序

对于存储一个version要么build ，您可以选择按版本对值进行排序。值可以按版本升序或降序排序。使用此选项，数字值将根据语义版本控制规范 （SemVer）自动排序。

此功能最有价值的应用是在搜索查询中。对于许多团队来说，搜索在特定版本或内部版本中已解决的问题的能力有助于为发布发行说明生成准确的列表。

要按版本对值进行排序：

需要权限：更新项目

  1. 选择一个自定义字段来存储version要么build 。

  2. 如果将自定义字段附加到一个或多个项目，则可以使用以下选项：

    • 编辑所有受影响项目的集 -排序顺序将应用到所有使用此值集的项目中。要使用此选项，您需要具有更新所有受影响项目的权限。

    • 进行独立复制 -排序顺序存储在当前项目中该字段的默认值集中。

    选择适合您目的的选项。

  3. 打开排序菜单，然后选择“ 按版本排序”选项。

  4. 在“ 更改排序顺序”对话框中确认操作。

    • 如果这些值以前是手动订购的，则手动订购将被丢弃。

    • 集合中的值按版本升序排序。

  5. 如果要按降序对值进行排序，请从下拉列表中选择desc选项。

克隆现有字段

作为项目管理员，您只能更新拥有“ 更新项目”权限的项目中专用的字段。对于自动附加或在其他项目中使用的自定义字段，您可能会发现自己无法编辑该字段的基本属性。这包括对字段名称的更改，用于确定该字段是存储单个值还是多个值的字段类型的属性以及字段别名。这些基本属性的更改会影响使用该字段的所有项目。

或者，您可以选择克隆字段。克隆字段时，将使用其基本属性创建该字段的副本，并为其分配唯一的名称。然后，您可以编辑字段的基本属性，而不会影响其他项目中原始字段的使用。

“ 克隆”字段操作仅适用于满足以下条件之一的自定义字段：

  • 为该字段启用了自动附加选项。

  • 该字段用于两个或多个项目。

克隆自定义字段：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 从列表中选择要修改的字段。

  3. 如果边栏不可见，请单击标题中的“ 显示详细信息”按钮。

  4. 在边栏中的字段名称旁边，单击“ 克隆”字段图标。

    • 在边栏中打开用于编辑字段属性的表格。

  5. 修改克隆字段的基本属性。请注意， 字段名称必须是唯一的。

  6. 单击保存按钮。

    • 原始字段的用法将替换为项目中的克隆字段。

    • 原始字段中使用的值集用作克隆字段的值集。

    • 原始字段中分配给项目中问题的值将更新为在克隆字段中使用相同的值。

    • 不影响其他项目中原始字段的使用。

替换自定义字段

您可以将项目中使用的自定义字段替换为其他项目中使用的自定义字段。如果存在要在其他项目中使用的自定义字段，而不是在项目中设置的自定义字段，请使用此功能。然后，您可以使用相同的自定义字段，例如，为敏捷板上的所有项目生成报告或显示相同的列和泳道。

不幸的是，您只能对具有相同数据类型的自定义字段使用“ 替换”功能。要将值从一种数据类型转换为另一种数据类型，您需要创建一个具有所需类型的新自定义字段，并针对项目中的所有问题手动更新新字段中的值。有关更多信息，请参见更改字段类型 。

替换自定义字段：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 选择要替换的字段，然后单击“ 替换”按钮。

    • 下拉列表显示了存储相同数据类型的所有可用自定义字段。

  3. 选择要替换现有字段的自定义字段。

    • 自定义字段将替换为项目中选定的自定义字段。

    • 先前的自定义字段使用的值集将添加到项目中替换字段的值集。

    • 原始自定义字段中分配给项目中问题的值将更新为在替换字段中使用相同的值。

    • 在其他项目中用于替换字段的值集保持不变。

变更栏位类型

您可以将大多数存储简单类型的字段转换为另一种存储相似数据的类型。支持以下字段类型转换：

  • 存储一个字段string可以转换为存储text 。您无法转换text字段string 。

  • 存储一个字段date可以转换为存储date and time ，反之亦然。

  • 对于将数据存储为integer ， period ， 要么float ，您可以在这组类型中将数据转换为目标格式。例如，您可以将integer输入到period类型。在这里，您可以选择将存储为整数的值转换为小时数或天数。

对于枚举的字段类型，类型是在创建字段时设置的，无法更改。

您不能基于每个项目更改字段类型。要执行此操作，您需要具有更新使用该字段的所有项目的权限。

要更改字段的类型：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 在列表中选择要更改的字段。

  3. 如果侧边栏是隐藏的，请单击工具栏中的“ 编辑”按钮或单击“ 显示详细信息”按钮。

  4. 从“ 类型”列表中选择所需的类型 。

如果要将现有的自定义字段替换为使用其他类型的字段，则不能使用“ 替换”操作。相反，您需要添加一个自定义字段，该字段将所需类型的值存储到项目中，并从原始字段中手动迁移数据。

要将数据从一个自定义字段迁移到另一个自定义字段：

需要权限：更新项目

  1. 在“ 编辑项目”页面上，选择“ 字段”选项卡。

  2. 单击将字段添加到项目按钮。

  3. 使用设置来定义自定义字段，或选择一个现有字段，以所需类型存储值。

  4. 如果新的自定义字段存储枚举值，则将值添加到现有的自定义字段中使用的新的自定义字段。

  5. 完成后，更新存储在项目的自定义字段中的值：

    • 从“ 问题”列表中，输入搜索查询，该查询在原始自定义字段中查找包含特定值的所有问题。对于枚举类型，请从集合中的第一个值开始。例如： In #{Project} Status: Submitted 。

    • 选择搜索查询返回的所有问题。使用键盘快捷键Ctrl + A （ ⌘ + A在macOS上）以选择查询返回的所有问题。

    • 使用命令来更新所有选定的问题，并在新的自定义字段中为它们分配一个值。例如： Secondary State Submitted 。为了最大程度地减少垃圾邮件，请静默应用命令。

    • 重复此步骤，直到将原始自定义字段中存储的所有数据迁移到新的自定义字段。

  6. 返回“ 管理”>“自定义字段”页面，然后从项目中删除原始的自定义字段。

    • 该字段及其相关值将从项目中删除。

    • 以前存储在原始自定义字段中的所有值现在都存储在新的自定义字段中。
