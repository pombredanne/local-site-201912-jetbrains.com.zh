YouTrack Standalone 2019.2帮助

运行和停止Docker容器

本页介绍如何停止和启动YouTrack Standalone服务器。

以下情况要求您停止YouTrack：

  • 您要更改数据库的位置。

  • 您想要修改YouTrack Standalone服务器的属性或JVM选项。

  • 您要升级YouTrack Standalone安装。

运行YouTrack Docker容器

要运行YouTrack容器，请执行以下命令：

docker run -it --name youtrack-server-instance \ -v ：/ opt / youtrack / data \ -v ：/ opt / youtrack / conf \ -v ：/ opt / youtrack / logs \ -v ：/ opt / youtrack / backups \ -p ：8080 \ jetbrains / youtrack：

命令属性：

  • -it —将YouTrack容器的输入和输出（包括启动日志）附加到终端窗口的标志。请注意，当终端连接到容器输出时，按Ctrl + C会导致容器关闭。使用`Ctrl + PQ`可以将终端从容器输出中分离出来。有关更多详细信息，请参阅官方docker文档 。

  • --name youtrack-server-instance —容器的任意名称。

  • -v :/opt/youtrack/NNN —将主机上特定于YouTrack的“ NNN”目录绑定到各自/opt/youtrack/NNN容器内的目录。

  • -p :8080 —定义端口映射的参数。指示主机监听端口并将所有流量传播到端口8080在docker容器中。

停止YouTrack Docker容器

要正常停止YouTrack服务，请运行以下命令：

docker exec 停止

要正常关闭，您还可以使用标准的` docker kill --signal=SIGTERM `命令。

我们不建议使用标准命令` docker stop `。默认情况下，此命令发送` SIGTERM向Docker容器内的YouTrack进程发出信号，然后等待10秒钟。如果YouTrack在该时间段内未完成关闭，则` SIGKILL `被发送到内核，YouTrack进程被杀死。这可能会导致数据损坏。为避免此结果，请在使用此命令时指定适当的超时值。例如，以下命令为关闭YouTube跟踪服务留出了足够的时间：

码头工人停止-t 60
