代理配置

如果您使用代理服务器来保护内部网络中的客户端，则可能需要配置YouTrack以通过代理服务器与其他应用程序建立HTTP连接。以下集成通过HTTP发送和接收信息，并且必须通过代理服务器进行路由：

  • TeamCity整合
  • 上游整合
  • VCS集成

当您使用连接到第三方服务的身份验证模块时，可能还需要通过代理服务器建立通信。

代理服务器未阻止邮箱集成和从Jira导入。

代理相关的系统属性

使用以下系统属性，使YouTrack通过代理服务器连接：

属性

描述

http.proxyHost

您的代理服务器的主机名。

http.proxyPort

代理服务器侦听的端口号。

https.proxyHost

您的安全代理服务器的主机名。

https.proxyPort

安全代理服务器侦听的端口号。

http.nonProxyHostsYouTrack绕过代理服务器直接访问的主机列表。

  • 用竖线分隔多个主机。

  • 使用星号作为通配符来指定模式。

非代理主机列表可以与HTTP和HTTPS协议一起使用。如果未设置此属性，则所有通信都将通过代理服务器路由。

要了解如何在YouTrack中设置这些属性，请参阅配置JVM选项。

故障排除

如果您使用的是代理服务器，并且YouTrack遇到问题，请查看是否存在以下任何情况。

条件您无法将YouTrack连接到外部服务。例如：

  • 您尝试设置VCS集成并获得bad_request错误。

  • 您尝试设置身份验证模块并获取异常： Authentication failed. java.net.ConnectException: Connection refused 。

原因解

外部服务不可用。

验证外部服务正在运行。

连接被防火墙阻止。

打开防火墙中由YouTrack和连接的服务使用的端口。

代理服务器阻止了与外部服务的连接。

设置服务器的系统属性，以使YouTrack通过代理服务器连接到其他服务。使用这些属性可以指定代理服务器的主机名和端口号。

您正在尝试通过错误的端口进行连接。

检查YouTrack服务器的系统属性，并验证是否正确设置了端口参数。
