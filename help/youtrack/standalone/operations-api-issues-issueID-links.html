使用特定IssueLink的操作

此资源使您可以阅读问题的链接。

资源资源
/ api / issues / {issueID} / links / {linkID}

退货实体

IssueLink 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读特定的IssueLink 。

IssueLink属性

表示特定链接类型的问题链接（例如，“与...相关”）。

下表描述了IssueLink实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

方向

列举

链接方向。支持的值为OUTWARD，INWARD和BOTH。 Read-only 。

linkType

 IssueLinkType

链接类型。 Read-only 。 Can be null 。

问题

 一系列问题

与此类型相关联的问题列表。

修剪的问题

 一系列问题

修剪的问题清单。此属性可用于避免具有许多链接的问题的性能问题。使用$ topLinks和$ skipLinks参数来请求整个列表的特定部分。 Read-only 。

阅读特定的IssueLink

获取问题特定链接的数据。

请求语法

GET / api / issues / {issueID} / links / {linkID}？{字段}

{linkID}

问题链接的数据库ID。

请求参数

参数

类型

描述

领域

响应中应返回的IssueLink属性列表。如果未指定任何字段，则仅entityID返回
