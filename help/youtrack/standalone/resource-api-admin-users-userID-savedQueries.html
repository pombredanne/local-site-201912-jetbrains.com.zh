保存的查询

此资源使您可以获取特定用户创建的已保存搜索的列表。

资源资源
/ api / admin / users / {userID} / savedQueries

退货实体

SavedQuery 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读已保存的查询列表 。

SavedQuery属性

表示已保存的搜索。

扩展WatchFolder

下表描述了SavedQuery实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

询问

串

在此搜索中保存的查询。 Can be null 。

问题

 一系列问题

与此保存的搜索相匹配的问题的集合。 Read-only 。

所有者

 用户

创建（因此拥有）watch文件夹的用户。 Read-only 。 Can be null 。

visibleFor

 用户组

可以看到此监视文件夹的用户组。如果该文件夹仅对其所有者可见，则此属性为null。 Can be null 。

可更新的

 用户组

可以更新此监视文件夹的用户组。如果只有文件夹的所有者可以更新它，则此属性为null。 Can be null 。

名称

串

问题文件夹的名称。 Can be null 。

阅读保存的查询列表

获取此用户创建的所有已保存搜索的列表。

请求语法

GET / api / admin / users / {userID} / savedQueries？{字段}＆{$ top}＆{$ skip}

请求参数

参数

类型

描述

领域

串

应在响应中返回的SavedQuery属性的列表。如果未指定任何字段，则仅entityID返回

$跳过

整数

可选的 。可让您设置要返回的多个实体，以在返回第一个实体之前跳过。

$ top

整数

可选的 。用于指定响应中返回的最大条目数。

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/users/1-2/savedQueries?fields=query,id,name,visibleFor(id,name),isUpdateable,owner(id,name)&$top = 3
