Text Index

You can manage the text indexing settings for your YouTrack server. To access these options, open the Administration menu and select Text Index in the Server Settings section.



Low-level Admin Read

Low-level Admin Write

These settings let you enable stemming (morphological analysis) of the selected languages. When enabled, YouTrack analyzes and identifies stem words and related forms for all of the text that is stored in any issue attribute, including custom fields. Text indexing is supported for the following languages:

Default languages

English, French, German, Russian, Spanish

Additional languages

Arabic, Chinese, Czech, Danish, Dutch, Finnish, Greek, Italian, Japanese, Korean, Latvian, Norwegian, Polish, Portuguese, Swedish, Turkish

Stemming of a language is the basis for the text search in YouTrack. When you enable the morphological analysis of a language, YouTrack finds not only the exact original word, but also its word forms. For example, when you search for the word "board", YouTrack finds issues that contain the words "board", "boards", "boarding", and so on. Each time an issue is created or updated in YouTrack, the issue attributes that store strings of text (the summary, description, and comments) are indexed in each language that is enabled on this page. For more information, see Text Search.

Text indexing is automatically enabled for all default languages. This means that stemming is enabled for all issue attributes that store strings of text in any of the languages that are provided out-of-the-box in YouTrack. YouTrack provides additional support for other languages. This means that you can enable text indexing and optimize your search queries if you write your issues in a language other than the language that is displayed on the user interface. For example, the language setting for your YouTrack server is English, but you create issues in Danish.



Do not enable text indexing for all supported languages at once. When you enable text indexing for fewer languages (especially languages with similar alphabets), the results for text searches in queries is more precise and accurate.

The following controls are available on this page:

Control

Description

Suspend text indexing

Temporarily stops indexing issues. Use this control to suspend text indexing during peak hours when the number of issues to be indexed impairs server performance. You can also suspend text indexing when you want to create a backup of your database, for example, after importing a new project.

When you suspend text indexing, the current status is displayed. Click the Resume button to resume indexing.

Rebuild text index

Rebuilds the text indexing to include the languages that are currently enabled. Use this control to update the text indexing of issue attributes when you enable or disable a language.
