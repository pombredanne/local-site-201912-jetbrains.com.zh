查看问题更新

在单个问题视图中，应用于问题的所有更改都显示在单个供稿中，我们将其称为活动流 。活动流中的过滤器使您可以选择要显示或隐藏的更改类型。

这是一个活动流示例，其中显示注释，问题历史记录和VCS更改：

活动流顶部的工具栏中提供以下控件：

控制

描述

评论

切换活动流中评论的可见性。要了解有关问题评论的更多信息，请参阅添加评论 。

花时间

切换工作项的可见性。仅当在项目中启用了“时间跟踪”功能时，此控件才可见。要了解有关工作项目的更多信息，请参阅添加工作项目 。

工作项目数据在一系列列中显示，这些列显示工作日期，花费的时间，工作类型和注释。

发行历史

切换问题更改历史记录的可见性。此提要包含适用于该问题的所有更新。

VCS变更

切换已链接到问题的VCS更改的可见性。仅当项目与版本控制系统集成或通过与TeamCity集成连接到VCS时，此控件才可见。有关更多信息，请参见查看和编辑VCS更改 。

代码审查

在Upsource中切换与代码审查相关的活动的可见性。仅当为项目配置了与Upsource的集成时，此控件才可见。有关更多信息，请参见监视代码评论 。

要应用过滤器并取消选择所有其他过滤器，请按住Alt键（ Option在macOS上），然后单击要显示的问题活动的过滤器。

活动设置菜单中提供以下选项：

控制

描述

排序：最旧的优先

按时间顺序显示更改。

排序：最新

按时间倒序显示更改。在此模式下，注释输入字段显示在活动流的顶部。

展开问题历史记录项目

启用后，将完整显示在多行上写入的问题历史记录项。否则，将在第一行显示一个控件，以显示每个问题历史记录项目的更多更改。

显示固定链接

显示到具有当前活动类型集的问题的链接。
