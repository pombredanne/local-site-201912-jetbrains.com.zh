创建问题并设置自定义字段

用例：

使用REST API创建新问题并为一个或多个自定义字段设置值。

请注意，此示例对YouTrack 2019.1和更高版本有效。在以前的版本中，尽管原理相同，但是查询参数和实体属性可能不同。此外，2019.1中引入了许多功能，例如按名称命名实体，但以前的版本中不支持。

摘要

要使用自定义字段的设置值来创建问题，您需要初步获取以下参数：

  • 新发行版应属于的项目的实体ID。

  • name和$type您需要设置的自定义字段。

  • name为自定义字段设置的值。

阅读REST API中的自定义字段页面，以获取有关REST API中的自定义字段的更多信息。

一步步

以下过程显示了如何创建一个新发行并为几个发行字段设置值。对于示例，我们决定设置“优先级”和“受让人”字段。

创建问题并为自定义字段设置值

  1. 获取可用项目的列表。

    curl -X GET \'https://example.myjetbrains.com/youtrack/api/admin/projects?fields=id,name,shortName'\ -H'接受：application / json'\ -H'授权：不固定烫发：amFuZS5kb2U =。UkVTVCBBUEk = .wcKuAok8cHmAtzjA6xlc4BrB4hleaX'\ -H'内容类型：application / json'

    对于此类请求，您将从服务器获得以下正文的响应：

    [{“ shortName”：“ GRP”，“ name”：“ GRA Project”，“ id”：“ 0-7”，“ $ type”：“ Project”}，{“ shortName”：“ RAP”，“名称“：” Rest Api项目“，” id“：” 0-2“，” $ type“：” Project“}，{” shortName“：” RP“，” name“：” Rest Project“，” id“： “ 0-6”，“ $ type”：“ Project”}，{“ shortName”：“ SP”，“ name”：“ Sample Project”，“ id”：“ 0-0”，“ $ type”：“ Project“}，{” shortName“：” SNBX“，” name“：”沙盒“，” id“：” 0-3“，” $ type“：” Project“}]

    您还可以通过以下方式过滤项目列表shortName或一个name所需项目。

    通过项目的过滤器提供样品请求name ：

    curl -X GET \'https://example.myjetbrains.com/youtrack/api/admin/projects?fields=id,name,shortName&query=Sample+Project'\ -H'接受：application / json'\ -H'授权：不记名perm：amFuZS5kb2U =。UkVTVCBBUEk = .wcKuAok8cHmAtzjA6xlc4BrB4hleaX'\ -H'内容类型：application / json'

    通过项目的过滤器提供样品请求shortName ：

    curl -X GET \'https://example.myjetbrains.com/youtrack/api/admin/projects?fields=id,name,shortName&query=sp'\ -H'接受：application / json'\ -H'授权：承载烫发：amFuZS5kb2U =。UkVTVCBBUEk = .wcKuAok8cHmAtzjA6xlc4BrB4hleaX'\ -H'内容类型：application / json'

    对于带有query参数，响应主体仅包含匹配项目的数据：

    [{“ shortName”：“ SP”，“ name”：“ Sample Project”，“ id”：“ 0-0”，“ $ type”：“ Project”}]

    在服务器的响应中，找到目标项目的ID。在我们的示例中，所需的项目id是0-0 。

  2. 获取名称和$type您需要设置的自定义字段。您可以将GET请求发送到/api/issues具有查询参数的端点：

    • fields参数以及要返回的问题属性列表。对于我们的情况，我们对自定义字段特别感兴趣。因此，我们可以使用以下字符串：

      fields = project（name），idReadable，customFields（name，$ type，value（name，login））

    • 要将结果缩小到目标项目中的问题，可以使用query参数。例如， query=in:SP缩小了我们特定目标项目的结果。

    • （可选）使用$top查询参数以获取有限数量的问题。

    这是结果请求：

    curl -X GET \'https://example.myjetbrains.com/youtrack/api/issues?fields=idReadable,id,project%28id,name%29,summary,description,customFields%28name,$type,value%28name ，login％29％29＆query = in：SP＆$ top = 1'\ -H'Accept：application / json'\ -H'Content-Type：application / json'\ -H'Authorization：Bearer perm：am9obi5kb2U =。UG9zdG1hbiBKb2huIERvZQ ==。jJe0eYhhkV271j1lCpfknNYOEakNk7'\

    作为响应，服务器返回了以下数据：

    [{“ idReadable”：“ SP-47”，“ project”：{“ name”：“ Sample Project”，“ id”：“ 0-0”，“ $ type”：“ Project”}，“ customFields”： [{“ value”：{“ name”：“ Show-stopper”，“ $ type”：“ EnumBundleElement”}，“ name”：“ Priority”，“ $ type”：“ SingleEnumIssueCustomField”}，{“ value”： {“ name”：“任务”，“ $ type”：“ EnumBundleElement”}，“ name”：“ Type”，“ $ type”：“ SingleEnumIssueCustomField”}，{“ value”：{“ name”：“ Open” ，“ $ type”：“ StateBundleElement”}，“ name”：“ State”，“ $ type”：“ StateIssueCustomField”}，{“ value”：{“ login”：“ jane.doe”，“ name”：“ Jane Doe“，” $ type“：” User“}，” name“：” Assignee“，” $ type“：” SingleUserIssueCustomField“}，{” value“：null，” name“：” Subsystem“，” $ type “：”“ SingleOwnedIssueCustomField”}，{“值”：[]，“名称”：“修复版本”，“ $ type”：“ MultiVersionIssueCustomField”}，{“值”：[]，“名称”：“受影响的版本” ，“ $ type”：“ MultiVersionIssueCustomField”}，{“ value”：null，“ name”：“固定在版本中”，“ $ type”：“ SingleBuildIssueCustomField”}]，“ $ type”：“问题”}]

    从此响应中，您可以看到可用字段及其类型的列表。您还可以在页面上获取问题自定义字段的可用类型的完整列表。

  3. 获取要为目标自定义字段设置的值的名称。通常，仅知道值的名称就足够了-它们在UI中的显示方式。例如，“优先级”字段为“显示停止器”或“专业”，或“状态”字段为“进行中”或“固定”。但是，对于某些字段，例如“修订版本”或“受让人”，可用值列表可能并不明显。在这种情况下，您需要获取用于目标项目中此特定字段的一组值（捆绑）。

  4. 要创建新问题，请将其分配给特定用户，然后将其“优先级”字段设置为Show-stopper ， 寄一个POST请求，该请求的主体包含项目的ID，新问题的摘要以及JSON对象customFields该属性包含：

    • 的name ， $type和value优先级字段。

    • 的name ， $type和value受让人字段。

    虽然description属性不是强制性的，我们也选择指定它。这是结果请求：

    curl -X POST \ https://example.myjetbrains.com/youtrack/api/issues \ -H'Accept：application / json'\ -H'授权：Bearer perm：amFuZS5kb2U =。UkVTVCBBUEk = .wcKuAok8cHmAtzjA6xlc4BrB4hleaX'\ -H'Content-Type：application / json'\ -d'{“ project”：{“ id”：“ 0-0”}，“ summary”：“ REST API允许您创建问题！”，“说明”：“让我们使用YouTrack的REST API创建新问题。”，“ customFields”：[{“ name”：“ Priority”，“ $ type”：“ SingleEnumIssueCustomField “，” value“：{” name“：” Show-stopper“}}，{” name“：” Assignee“，” $ type“：” SingleUserIssueCustomField“，” value“：{” login“：” jane.doe “}}]}'

    在响应中，服务器仅返回创建的问题的实体ID及其$ type，因为我们未指定任何内容fields请求中的参数：

    {“ id”：“ 2-38”，“ $ type”：“问题”}

请注意，您不能同时创建一个新问题并为其添加附件。您只能将文件附加到现有问题。
