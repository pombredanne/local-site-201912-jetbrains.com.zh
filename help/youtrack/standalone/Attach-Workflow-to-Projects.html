管理多个项目的工作流程

作为系统管理员，您可以为YouTrack实例管理所有项目中的工作流程。具有项目管理员角色的用户只能在其有权查看和更新项目的项目中管理工作流。有关更多信息，请参见管理工作流 。

将工作流程附加到项目

要使用工作流程，您需要将其附加到一个或多个项目。从管理菜单中，您可以访问YouTrack中的所有项目。

附加工作流程：

需要权限：更新项目

  1. 在管理菜单的核心功能部分中，选择工作流程 。

  2. 从列表中选择工作流程。

  3. 如果侧边栏处于隐藏状态，请单击显示详细信息按钮。

  4. 在边栏中，单击“ 项目”旁边的+图标，然后选择要在其中应用工作流程的项目。

    • 工作流已附加到所有选定项目。

    • 如果存在不满足工作流程要求的项目，则会显示require setup标志。

require设置标志指示工作流中指定的要求在工作流当前附加的一个或多个项目中不可用。这可能意味着项目不使用必填的自定义字段，自定义字段中的值集中缺少值，或者在YouTrack中未创建问题链接类型。每个需要设置的项目的名称在“ 详细信息”边栏中以红色显示。

在大多数情况下，无需离开工作流程管理页面即可应用这些修复程序。

要解决工作流程中的设置问题：

需要权限：更新项目

  1. 展开列表中的工作流程。

  2. 选择第一个标有“ requires setup”标记的规则，而无需在工作流编辑器中打开模块。含义，选择规则而不单击模块名称的链接。

  3. 如果侧边栏处于隐藏状态，请单击显示详细信息按钮。

  4. 将指针移到以红色文本显示的任何项目上。

    • 工具提示显示激活每个项目中的工作流所需的更改。

  5. 单击“ 应用修订”链接。

    • 将打开“ 应用修订”对话框。

  6. 选择所有必需的修复程序，然后单击“ 应用”按钮。

  7. 对所有需要安装的项目重复步骤5和6。

    • 完成后，会自动从每个规则中删除require setup标志。

    • 刷新页面时，从工作流中删除了require setup标志。

分离和附加工作流程规则

将工作流程附加到项目时，所有规则都将附加并启用。您可以通过分离和附加特定规则来针对每个项目自定义工作流程。

要查看工作流程的规则列表，请单击以展开工作流程。与所有项目分离的规则在“ 用法”列中显示“ 无用法 ”。

要分离项目中的各个规则：

需要权限：更新项目

  1. 在管理菜单的核心功能部分中，选择工作流程 。

  2. 展开工作流程以显示规则列表。

  3. 选择要停用的规则。

    • 所选规则显示在侧栏中。

  4. 单击项目名称旁边的x图标，以在所选项目中分离规则。

当您分离工作流程中的所有规则时，工作流程将自动从项目中分离。

要将单个规则附加到项目中：

需要权限：更新项目

  1. 在管理菜单的核心功能部分中，选择工作流程 。

  2. 展开工作流程以显示规则列表。

  3. 选择您要激活的规则。

  4. 单击+图标，然后从列表中选择一个或多个项目。

分离工作流程

如果您不再想要使用工作流程，则可以将其与一个或多个项目分离。

分离工作流程：

需要权限：更新项目

  1. 在管理菜单的核心功能部分中，选择工作流程 。

  2. 选择要分离的工作流程。

  3. 单击项目名称旁边的x图标，以将工作流程与所选项目分离。

或者，您可以分离工作流程中的所有规则。工作流程会自动从项目中分离。
