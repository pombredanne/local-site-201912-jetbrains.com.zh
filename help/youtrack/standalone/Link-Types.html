发行链接类型

在YouTrack中，您可以链接问题以显示它们之间的关系。当您将一个问题的链接添加到另一个问题时，两个问题都将添加链接。

问题链接类型具有确定其方向的属性。方向确定了问题之间的关系。问题链接类型可以具有以下链接方向之一：

方向

描述

导演

这个方向意味着问题之间存在从属关系。例如，“ 依赖”链接类型指示问题B需要问题A。相反，问题B取决于问题A。在这种情况下，链接类型指示必须首先解决问题A。当您添加到问题A的链接所必需的时， 依赖链接会自动添加到问题B。

无向

这些链接类型仅表明这两个问题是相互关联的。

聚合

这个方向意味着问题可以通过某种方式合并。这些链接表明问题相互重复或属于同一任务。

作为管理员，您可以使用其他名称将其他问题链接类型添加到YouTrack，但是，这些链接的行为取决于它们的方向。使用相同方向设置的所有链接类型的行为都相似。您可以在“ 管理”>“问题链接类型”页面上的YouTrack中查看可用的链接类型列表。

添加新的问题链接类型时，重要的是要了解问题链接类型和问题链接名称之间的区别。链接类型定义了链接行为，并且对用户不可见。要链接问题并搜索链接问题，用户请选择链接名称。链接名称还用于在工作流规则和REST API方法中指定问题链接类型。

  • 链接类型定义链接的行为。用户看不到分配给链接类型的名称。

  • 链接名称是用户在链接问题和搜索链接问题时与之交互的值。您还可以使用链接名称来引用工作流程规则和REST API方法中的链接类型。

添加链接类型时，可以指定链接名称并确定是否定向该链接。

对于定向链接，您可以指定两个名称：

  • 源问题的内部名称 。在向问题添加链接时，将为源问题中的链接分配内部名称。例如，当源问题取决于目标问题时。

  • 目标问题的外部名称 。当您将链接添加到问题时，将为目标问题中的链接分配外部名称。例如，当源问题需要目标问题时 。

要在问题之间创建链接，用户需要基于每个项目具有“ 链接问题”权限。如果用户报告链接问题，请确保正确配置了这些权限。

创建问题链接类型

您可以将自己的链接类型添加到YouTrack，以创建和管理问题之间的依存关系。

要创建问题链接类型：

需要权限：低级管理员写

  1. 在“ 管理”菜单的“ 核心功能”部分中，选择“ 发布链接类型” 。

  2. 点击创建问题链接类型按钮。

  3. 输入以下设置的值：

    设置

    描述

    名称

    为问题链接类型分配一个名称。

    外部名称

    在目标问题中分配一个分配给链接的名称。例如， 由复制 。

    内向名称

    在源问题中分配一个分配给该链接的名称。例如， 重复项 。如果未指定链接类型，则将忽略向内名称-在源和目标问题中，链接均使用向外名称。例如，当您使用无向问题链接类型涉及 ，链接指定的名称为源和目标的问题既涉及到 。

    链接方向

    选择链接方向类型。

  4. 单击保存按钮。

编辑链接类型

您可以修改现有链接类型的设置。修改这些设置时，还可以更改使用此链接类型的链接的行为。具体来说，当您修改“ 链接方向”设置时，已添加到问题中的链接将被修改如下：

您不能修改预定义类型Duplicate和Subtask 。这些链接类型是只读的！

  • 当您将有向链接类型更改为无向链接时，仅保留向外链接名称。现有链接将更新为使用外部名称。

  • 当您将非定向链接类型更改为定向链接时，现有链接名称将更新。新的外部名称将分配给目标问题中的链接。内部名称（如果更改）将分配给源问题中的链接。

要编辑问题链接类型：

需要权限：低级管理员写

  1. 在“ 管理”菜单的“ 核心功能”部分中，选择“ 发布链接类型” 。

  2. 在列表中选择问题链接类型。

  3. 单击编辑图标。

    • 将打开“ 编辑”对话框。

  4. 更新问题链接类型设置。

  5. 单击保存按钮。

删除未使用的链接类型

如果问题链接类型在YouTrack中没有使用，则只能删除它。即使已从所有问题中删除了使用此问题链接类型的所有链接，该链接类型仍保留在问题历史记录中，无法从YouTrack中删除。因此，您只能删除从未添加到任何问题的问题链接类型。

要删除问题链接类型：

需要权限：低级管理员写

  1. 在“ 管理”菜单的“ 核心功能”部分中，选择“ 发布链接类型” 。

  2. 在列表中选择问题链接类型。

  3. 单击删除图标。

    • 打开确认对话框。

  4. 单击删除按钮以确认操作。

    • 问题链接类型已从YouTrack中删除。
