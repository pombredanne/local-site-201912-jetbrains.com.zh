YouTrack Standalone 2019.2帮助

构建服务器集成

构建服务器集成可帮助您减少开发团队执行的重复工作量。YouTrack与构建服务器之间的集成可在两个应用程序中启用有用的功能：

  • YouTrack从关联的项目中提取内部版本号。在上次成功的成功构建之后解决的问题会自动分配到下一个成功的构建的内部版本号。此值添加到一个自定义字段，该字段存储每个已解决问题的内部版本号。YouTrack还在自定义字段中存储的每个内部版本号旁边添加一个图标。用户可以单击此图标来打开包含解决问题的更改的内部版本。

    您可以指定是否为提交消息和构建注释中提到的问题分配内部编号，并使用搜索查询为与特定条件匹配的问题分配内部编号。

  • 当开发人员将更新提交到代码存储库时，可以在注释中引用问题ID。构建服务器集成在更改详细信息中添加了指向YouTrack问题的直接链接。

    注释会自动添加到活动流中针对YouTrack中问题的更改列表。每个VCS更改都包含指向TeamCity中更改详细信息的直接链接。

  • 您的开发人员可以在注释中输入命令以提交对存储库的提交。当构建服务器组装包含提交消息的构建时，该命令将应用于注释中引用的所有问题。开发人员可以使用此功能来更新状态或为一个或多个问题添加评论，而无需打开YouTrack。

构建服务器集成设置

您可以从“ 管理”菜单的“ 集成”部分访问“ 构建服务器集成设置”页面。

此页面显示与YouTrack集成的构建服务器的列表。该表显示了每个集成项目的名称，到所连接构建的链接以及上次处理的构建的编号。

此页面上提供以下控件：

控制

描述

拉取新数据

设置从所有连接的构建服务器中提取新数据的时间间隔。有关更多信息，请参见设置从构建服务器提取数据的时间表 。

添加新的映射

在边栏中打开“ 创建映射”对话框。使用这些设置将YouTrack项目映射到构建配置。

跑

从构建服务器中为所选项目提取新数据。构建链接将添加到构建注释和提交消息中提到的YouTrack问题中。VCS提交消息中的命令将应用于相应的问题。内部版本号将根据映射中定义的选项添加到问题中。在集成设置页面上的相应列中更新了最后处理的构建。

编辑

在侧栏中打开所选构建服务器或映射的设置对话框。

去掉

从YouTrack删除选定的构建服务器或映射。

重启

从所选项目映射中删除最后记录的更改和已处理的内部版本的ID。下次计划更新集成时，YouTrack从构建服务器中提取整个更改和构建历史记录。YouTrack中已经存在的更改和构建不受影响。

禁用

禁用所选构建服务器或项目中的集成。在项目级别使用此选项可基于每个项目禁用集成功能。如果禁用集成，则“ 启用”选项可用。

添加新的构建服务器

在侧边栏中打开一个对话框，可让您定义与新构建服务器的连接。

设置从构建服务器提取数据的时间表

YouTrack根据计划从连接的构建服务器中提取数据。默认间隔为10分钟。您可以将计划更改为预定义的时间间隔，也可以使用cron表达式指定自定义时间间隔。您不能为多个构建服务器服务器独立设置时间表。相同的时间表将应用于与YouTrack集成的所有构建服务器。

指定提取数据的时间间隔：

需要权限：低级管理员写

  1. 在“ 管理”菜单的“ 集成”部分中，选择“ 构建服务器集成” 。

  2. 从“ 提取新数据”下拉菜单中，选择一个预先配置的时间间隔。要指定另一个时间间隔，请选择自定义，然后输入cron表达式。例如，要每天每隔45分钟提取一次新数据，请使用以下表达式：

    0 0/45 * * *？

    有关cron表达式格式的详细信息，请参阅cron触发器教程 。
