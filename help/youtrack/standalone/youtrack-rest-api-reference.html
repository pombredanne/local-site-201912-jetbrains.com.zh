REST API

通过YouTrack REST API，您可以在跟踪器中以编程方式执行各种操作：

  • 从您当前的错误跟踪系统导入问题-以便顺利迁移到YouTrack。

  • 创建，修改和执行其他有问题的操作-因此您可以将YouTrack无缝集成到您的环境中。例如，通过第三方应用程序的自动问题提交。

  • 处理项目，自定义字段和值集，敏捷板，发布链接类型和其他参数。

一般注意事项

REST API返回并使用JSON格式的数据。常规的内容协商规则适用：

  • 使用“ 接受 HTTP请求”标头指示期望的响应数据格式： application/json

  • 将Content-Type HTTP请求标头用于POST和PUT请求： application/json

REST API 始终处于启用状态 。但是，系统管理员可以指定允许哪些站点（来源）使用REST访问YouTrack。有关更多信息，请参见资源共享 。

基于YouTrack REST API的工具和客户端库

在开始将REST API用于自己的开发之前，请看一些使用YouTrack REST API的工具：

  • 适用于Python的YouTrack REST API客户端库使用REST API从其他问题跟踪器导入数据。

  • YouTrack REST API PHP客户端是YouTrack REST API的PHP客户端库。

  • YouTrackSharp是一个用于访问YouTrack REST API的.NET库。该库也可以从NuGet画廊获得 。

  • YouTrack Mobile使用REST API从YouTrack安装中提取数据并将其显示在移动应用程序中。

  • 适用于IntelliJ IDEA和其他JetBrains IDE的YouTrack集成插件使用REST API直接在IDE中显示与问题相关的信息。
