分页

本页介绍了YouTrack的REST API中的分页选项。

默认情况下，所有返回实体集合的资源都会限制返回的实体数。如果您未明确指定要返回的实体数$top查询参数时，服务器最多返回42个第一实体。要获取整个集合，您需要使用分页。

绝大多数资源使用分页$top和$skip查询参数。我们在这里称其为一般分页。

但是，用于与问题相关的活动的资源使用特定的分页方法-游标。

一般分页

当您尝试获取大量元素时，默认情况下，YouTrack仅返回前42个元素。这样做是为了消除服务器过载。要获取元素的完整列表，请使用$skip和$top请求参数：

  • $skip=N让您跳过找到的N个元素，并返回从N + 1开始的元素。

  • $top=M指示服务器返回找到的前M个元素的集合。

结合这两个参数，您可以遍历所需元素的整个集合。例如，您可以遍历包含50个元素的包中的可用问题列表：

https://example.myjetbrains.com/youtrack/api/issues?fields=id,idReadable,summary,description&$skip=0&$top=50 https://example.myjetbrains.com/youtrack/api/issues?fields = id，idReadable，summary，description＆$ skip = 50＆$ top = 50 https://example.myjetbrains.com/youtrack/api/issues?fields=id,idReadable,summary,description&$skip=100&$top=50

或者，您可以打包20个元素，一包地打包一大堆版本：

https://example.myjetbrains.com/youtrack/api/admin/customFieldSettings/bundles/version/42-1/values?$skip=0&$top=20&fields=archived、description、id、name、ordinal、releaseDate，已发布https://example.myjetbrains.com/youtrack/api/admin/customFieldSettings/bundles/version/42-1/values?$skip=20&$top=20&fields=已存档，描述，id，名称，常规，发布日期，已发布https://example.myjetbrains.com/youtrack/api/admin/customFieldSettings/bundles/version/42-1/values?$skip=40&$top=20&fields=已存档，描述，id，名称，常规，发布日期，已发布
