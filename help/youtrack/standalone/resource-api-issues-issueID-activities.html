活动项目

该资源提供对问题活动的访问，并可以通过各种参数对其进行过滤。基本上，此资源使您可以获取特定问题的操作历史记录。

资源资源
/ api / issues / {issueID} /活动

退货实体

ActivityItem 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读ActivityItems列表 。

ActivityItem属性

表示问题或其相关实体中的更改。在用户界面中，您将这些更改视为“活动”流。它显示了该问题的所有更新的供稿：问题历史记录，注释，附件，VCS更改，工作项等。

下表描述了ActivityItem实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

指示返回活动的顺序是从最新到最旧或相反。如果为“ false”，则首先返回与所选过滤器匹配的最早的活动项目。如果为“ true”，则首先返回最新活动。默认情况下，为“ false”。

光标

串

游标的主要应用是活动的分页。活动是一个经常更改的集合，可能在两个请求之间创建新的活动。在这种情况下，使用“ top”和“ skip”参数进行分页的一般方法不起作用。而是使用以下分页方法：

  1. 使用“ activitiesPage”资源之一请求页面。

  2. 从返回的活动页面中选择一个光标。

  3. 将该值作为下一页请求的“光标”参数。

如果在请求中未指定“光标”，则响应页面将根据请求的顺序（正向或反向）从最早的活动或最新的活动开始。

用例：

让我们考虑以下语句作为示例的初始条件：

  • 四个活动项目的集合：A，B，C，D

  • 返回仅包含1个项目的页面的请求：[B]

所提及的页面可以通过以下JSON呈现（请参阅： ActivityCursorPage ）：

{“ activities”：[B]“ cursorBefore”：“ A ^ B” //该值与实数不同，仅用于演示“ cursorAfter”：“ B ^ C” //该值与实数不同并且仅用于演示“ hasBefore”：true“ hasAfter”：true“ reverse”：false}

对以下端点的请求可以接收到该页面：

/ api / activitiesPage？activityId = B＆$ top = 1

游标的实际值是一个复杂字符串。使用的符号“ A ^ B”表示光标指向项目A和B之间的间隙。

为了请求附近的活动页面，我们可以使用接收页面的光标，并请求从光标开始到不同方向的页面。以下组合可能：

  • 请求/api/activitiesPage?$top=100&cursor=A^B&reverse=false返回包含[B，C，D]项目的页面。

  • 请求/api/activitiesPage?$top=100&cursor=A^B&reverse=true返回带有[A]项目的页面。

  • 请求/api/activitiesPage?$top=100&cursor=B^C&reverse=false返回包含[C，D]项目的页面。

  • 请求/api/activitiesPage?$top=100&cursor=B^C&reverse=true返回包含[B，A]项目的页面。

activityId

串

页面中应包含的活动的ID。该活动被分配到页面的中间。
