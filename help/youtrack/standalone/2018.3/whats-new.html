YouTrack Standalone 2018.3 Help

What's New

We deliver updates and improvements to YouTrack on a continuous basis. Here's an overview of changes that are available in YouTrack 2018.3.

New Features

Feature

Description

Availability

Visual Redesign

YouTrack gets a fresh, new look! We've rebuilt every page using components from JetBrains' open-source library for web-based products, Ring UI.

2018.3

Project Overviews

New project overview pages give you a more convenient way to view information related the project. They also serves as a landing page or “home base” for each project.

These pages display the project description and provide direct links to project settings. They also provide a dashboard-like interface where you can add widgets that let you monitor the activity in your projects.

2018.3

Group Logos for Users in Selection Menus

Everywhere you see the option to select a user from a drop-down list, the user avatars are shown with group logos. This includes the selectors for issue and comment visibility, suggestions in @mentions and commands, watchers, and assignees. Add icons to groups to help you quickly distinguish between different sets of users.

2018.3

Delete Issue Condition for On-change Rules in Workflows

A new runOn property for on-change rules lets you specify whether a workflow is triggered when issues are updated or deleted. Use this condition to script rules that prevent accidental deletion of important issues.

2018.3

Supplemental Fields for Text

YouTrack now supports supplemental fields that store data as text, formatted in Markdown. Create dedicated input fields to classify information that goes beyond the issue description.

2018.3

Embedded Content from Google Apps

An extension to the Markdown implementation in YouTrack lets you embed published content from Google apps. Insert documents, spreadsheets, and forms directly into an issue description or comment.

2018.3

Two-factor Authentication

We've built in support for two-factor authentication directly into your Hub account. Pair your Hub account with an identity verification app on your mobile device and add an extra layer of security.

2018.3

Application Passwords

To help you stay connected to third-party services that don't support two-factor authentication, we've also added support for application passwords. Use these randomly generated passwords to give an app or device permission to access your Hub account.

2018.3

Global Password Change Requests

A new action in the Hub authentication module lets an administrator request that all users change the passwords for their Hub accounts. When you believe that a malicious user has obtained unauthorized access, use this feature to keep your installation secure.

2018.3

Jenkins Integration

A new integration with Jenkins is built directly into YouTrack and ready for configuration. Pull build numbers from Jenkins to set the fix version for issues in YouTrack, mention issues in commits, and update issues with commands from commit messages.

2018.3

Updates and Enhancements

Feature

Description

Availability

Search by User Mentions

You can now search for issues that contain @mention references for specific users. Use this search attribute to generate reports and lists that mention your account without having to rely on notifications alone to track these issues.

2018.3.47010

Shortcuts for Issue Links

We've added support for two new keyboard shortcuts in single issue view. These let you expand or collapse the panel for linked issues and open a dialog to add links to target issues.

2018.3.47010

Custom Reply-to Address for Notifications

New options added to the Global Settings and Edit Project pages let you specify dedicated email addresses for handling replies to notifications that are sent by YouTrack.

2018.3.47010

RTL Language Support in New Issue Views

The updated views for new and single issues now support languages that are written from right to left. As a result, installations and instances that have enabled this option are no longer restricted to older issue views.

2018.3.47010

New Notification Templates

The templates that are used to generate notifications for integration-related problems are now available on the Notification Templates page. This means that you can customize the message content at the server and project level.

2018.3.47010

Add Cards to Multiple Sprints

A new option in the Card configuration for agile boards lets you remove the restriction that limits cards to single sprints.

2018.3.47010

Relative Parameter for Last Working Day

We've added a pre-defined parameter that lets you search for date values for the last working day. When you use this parameter in search queries instead of yesterday, you can start your work week with a full schedule of tasks to complete.

2018.3.47010

Redesigned Presentation for Linked Issues

The updated interface for issue links lets you quickly add links to other issues and browse the collection of issues that are linked to the current issue. New controls let you watch, vote, and see how many votes and comments there are in each linked issue.

2018.3

Period Values for Hours per Day

The global time tracking setting for hours per day now stores a period value. This lets you use a value like "7h30m" to define your work day and calculate your spent time accordingly.

2018.3

Custom Order for Favorite Dashboards

You can now apply a custom order to your list of favorite dashboards. Place your dashboards in the desired sequence and page through them in logical order.

2018.3

Find Similar Issues Action

A new action in the issue toolbar lets you open a list of issues that are similar to the current issue in single issue view. This replicates the functionality that was supported on the Similar Issues tab in the old issue view.

2018.3

Flexible First Day of Week

The first day of the week is no longer dependent on the language in your general profile settings. We added an option to your profile that lets you choose which day starts each week in calendar controls.

2018.3

Shared Editing for Reports

A new setting for all reports lets you open your reports to editing by members of a specific group or project team.

2018.3

Explicit Handling for Comments in VCS Commits

We've added an option to the settings for build server and VCS integrations that lets you determine whether commit messages are parsed for comments. When you disable this option in the mapping between your project and the repository, additional commit messages that are generated by the IDE or VCS are shown together with the VCS change and do not generate redundant comments.

2018.3

New REST API

We've published documentation for a partial collection of endpoints that are supported in the current version of our REST API. These endpoints cover the most common methods for updating issues programmatically.

2018.3

Access Interface Updates

We've added a dedicated Access tab to your project profiles. This view gives you better insight into the roles that are assigned to users and groups in your projects beyond the access that is granted to members of your project team. You also have direct access to all of the operations that allow you to revoke access rights on one page.

The updated layout has also been applied to the Roles tabs for users and groups.

2018.3

Throttling Improvements

We've added a list of logins that are currently being tracked for rate limiting. This helps you quickly identify accounts that a malicious user may be using in a brute force attack.

2018.3

Access Token Timeout

A new setting for authentication modules lets you specify the maximum lifetime for access tokens. Use this setting to limit the amount of time a malicious user can access the application.

2018.3

Discontinued Features

Feature

Description

Availability

Old Views for New and Single Issues

The option to use old views for new and single issues has been removed from the user profile settings.

Removed in 2018.3

New Workflows in the YouTrack Workflow Editor

We no longer support new workflows that are written in the external YouTrack Workflow Editor. We will continue to support workflows that were written in this editor through the first YouTrack release in 2019. After that point in time, we will only support workflows in JavaScript.

If you're still working with old workflows, we encourage you to rewrite them in JavaScript as soon as possible. For details, see Converting Old Workflows to JavaScript.

Removed in 2018.3

Old YouTrack REST API

With the initial publication of endpoints for our current REST API, we encourage developers to start migrating from the previous version. Once we have published newer versions for the endpoints that were supported in the previous API, it will be officially deprecated.

If you have an existing application or integration that uses the old API, we encourage you to familiarize yourself with the updated version. If your application uses endpoints for which an equivalent is not yet published in the new API, join our YouTrack Community in Slack. Let us know what endpoints you’re working with and help us identify which features should be published next.
