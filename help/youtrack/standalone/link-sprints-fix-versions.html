YouTrack Standalone 2019.2帮助

将Sprint与修订版本链接

传统Scrum的核心概念之一是每个sprint都会产生一个可行的产品发行版本。这些发行版可以通过连续部署进行中间交付，也可以作为新的主要版本进行收集和交付。

在YouTrack中，我们的敏捷板的第一个版本遵循了该模型。当它在YouTrack版本4中首次发布时，默认情况下，所有sprint都链接到修订版本。在特定冲刺期间修复问题会自动为修复版本设置链接字段的值。当问题被重新安排以用于后续的sprint或被完全放弃时，链接字段中的值也会更新。

自最初发布以来，我们已经放宽了支持其他敏捷框架的限制。我们通过引入将冲刺链接到自定义字段的基于版本的电路板模板来保留原始功能。

JetBrains的几个团队成为YouTrack敏捷开发板的早期采用者。他们将此功能用作其Scrum框架的基础。其他团队已经改编了基于版本的委员会，以显示分配给产品的每个发行版的问题，即使他们没有遵循严格的敏捷方法。

最早的采用者之一是dotMemory团队。这是他们的板在我们的YouTrack安装中的样子：

他们的董事会支持以下流程：

  • 在“ 修订版本”字段中，每个冲刺都喜欢一个值。它们的发布周期与ReSharper的主要版本同步。

  • 对于每个版本，他们都会选择两个或三个主要功能。他们将这些功能的开发分解为较小的任务，并跟踪每个泳道的进度。

  • 在泳道中管理未分类卡片的小功能和错误。

  • 他们使用“ 待办事项”列作为应解决的问题，这些问题应在当前开发版本中解决。

  • 团队中的每个成员一次都会有一个或两个问题。

  • 完成后，用户界面中公开的功能将移至“ 演示”列。在这里，开发人员向团队的其他成员介绍该功能。

    • 如果通过，则转到分支固定或测试 。

    • 如果没有，则返回到Development 。

  • 固定在分支列用于较大或危险的更改。本列适用于在合并到主分支之前在功能分支中经过测试的问题。

  • 发布前一个月，团队检查了sprint中所有未解决的问题。在此版本中无法解决的问题将推迟到下一个冲刺，或者返回到积压中。只需在每个问题中更改“ 修订版本”字段的值即可应用这些更新。

对于遵循Scrum敏捷框架或跟踪分配给产品发行版本的问题的团队而言，这种类型的董事会可能很有用。

要构建这样的板：

  1. 创建您的董事会。

    • 使用基于版本的电路板模板。

    • 选择要在板上管理的项目。

      当您将Sprint链接到修订版本时，通常只处理一个项目。为了正常工作，该项目应该已经使用了名称为Fix version或Fix models的字段。您还可以在基于版本的板上管理多个项目，只要所有项目对“ 修订版本”或“ 修订版本”字段使用相同的一组值即可。

    • 如果您已经保存了积压问题的搜索，请将“ 积压”选项设置为“ 使用现有” ，然后在“ 已保存的搜索”下拉列表中选择它。否则，选择“ 新建” ，然后单击“ 创建板”按钮。

    创建电路板时，将自动将sprint链接到“ 修订版本”或“ 修订版本”字段的选项已启用。当前存储在此字段中的所有值都作为冲刺添加到板中。

  2. （可选）应用过滤器。

    • 通过将sprint链接到自定义字段选项 ，您可以使用搜索查询来过滤板上的卡片。

    • 使用此过滤器仅显示分配给特定子系统或包含特殊标记的问题。

  3. 定义列。

    • 您会立即注意到您的电路板存在配置错误。要修复此问题，请切换到“ 列和泳道”选项卡，然后选择一个字段以标识您的列。最常见的配置是使用“ 状态”字段。

  4. 定义泳道。

    • 大多数软件开发团队都使用问题类型将其工作组织成功能或用户案例。您可以使用这些问题在板上定义泳道并将其子任务作为卡片进行管理。选择“ 问题”选项，选择“ 类型”字段，然后添加用于项目中的父任务的问题类型。

  5. 选择一个图表。

    如果要跟踪进度，请选择反映您如何管理工作负载的图表。

    • 如果您遵循Scrum方法，请使用Burndown图表。

    • 对于看板，请使用“ 累积流量” 。
