Restore Database from a Backup

  • JAR Installation
  • ZIP Installation
  • MSI Installation

To restore your database from a backup successfully, the version of the backup file should be the same or earlier than the version of YouTrack to which you restore this database.

Before you begin, make sure you have a backup copy of the database that you want to restore. Download a backup file (tar.gz|zip) from YouTrack on the Administration > Database Backup page or locate an archived backup file directly in the directory that is set as the Backup location.

JAR Installation

If you installed YouTrack with a JAR distribution, use the following procedure.

To restore a database from a backup:

  1. Stop the YouTrack server.
  2. Remove all of the content from the database location directory. By default, the database is located in the teamsysdata folder in the home directory of a user account, which starts YouTrack. You can change default database location. For more information, see Change Database Location.
  3. Extract the downloaded backup file to your database location.
  4. Make sure YouTrack service user has read/write ("rw") permissions for the unpacked files.
  5. Start YouTrack.

ZIP Installation

If you installed YouTrack from a ZIP distribution, use the following procedure.

To restore your database from a backup:

  1. Stop the YouTrack server.
  2. Download and install a YouTrack 7.0 build that is the same version or later than the version of your database.
  3. In the Configuration Wizard, select the Upgrade option.
  4. On the Select Upgrade Source page, select your backup as the Upgrade Source and click the Next button.
  5. Confirm or change the locations where YouTrack stores product data. When done, click the Next button.
    • YouTrack starts with the data from the backup file.

MSI Installation

If you run YouTrack as a Windows service installed from the MSI distribution, use the following procedure.

To restore your database from a backup:

  1. Uninstall YouTrack 7.0 from the Windows Control Panel.
  2. Make sure that the YouTrack service user (JetBrainYouTrack, by default) has full access to the directory where you saved the database backup file.
  3. Install a YouTrack 7.0 build that is the same version or later than the version of your database. Please note:
    • To troubleshoot errors more effectively, run the MSI installer with verbose logging enabled. In a Command Prompt window, enter the following command:
      msiexec /i youtrack-7.0.xxxxx.msi /l*v install.txt
    • Configuration files from previous installations might be found during the installation. In this case, please use the (Recommended) installation option that is selected by default.
  4. In the Configuration Wizard, select the Upgrade option.
  5. On the Select Upgrade Source page, select your backup as the Upgrade Source and click the Next button.
  6. Confirm or change the locations where YouTrack stores product data. When done, click the Next button.
    • YouTrack starts with the data from the backup file.
