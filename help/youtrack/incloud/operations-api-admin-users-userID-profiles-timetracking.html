使用特定的TimeTrackingUserProfile进行的操作

此资源使您可以在用户的配置文件中配置时间跟踪设置。

资源资源
/ api / admin / users / {userID} / profiles / timetracking

退货实体

TimeTrackingUserProfile 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 读取特定的TimeTrackingUserProfile 。

  • POST ： 更新特定的TimeTrackingUserProfile 。

TimeTrackingUserProfile属性

代表用户个人资料中的时间跟踪设置。

下表描述了TimeTrackingUserProfile实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

periodFormat

 PeriodFieldFormat

格式，其中为用户显示期间值。

读取特定的TimeTrackingUserProfile

获取用户的时间跟踪设置。

所需权限

需要“读取自我”才能读取自己的个人资料，需要“不拥有个人资料”来读取其他用户的个人资料。

请求语法

GET / api / admin / users / {userID} / profiles / timetracking？{字段}

请求参数

参数

类型

描述

领域

串

应在响应中返回的TimeTrackingUserProfile属性列表。如果未指定任何字段，则仅entityID返回

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/users/me/profiles/timetracking?fields=periodFormat(id）

样品响应体

{“ periodFormat”：{“ id”：“ HOURS_MINUTES”，“ $ type”：“ PeriodFieldFormat”}，“ $ type”：“ TimeTrackingUserProfile”}

更新特定的TimeTrackingUserProfile

更新用户的时间跟踪设置。

所需权限

需要“更新自我”来更新自己的配置文件，需要“更新自己的配置文件”来更新另一个用户的配置文件。

请求语法

POST / api / admin / users / {userID} / profiles / timetracking？{字段}

请求参数

参数

类型

描述

领域

串

应在响应中返回的TimeTrackingUserProfile属性列表。如果未指定任何字段，则仅entityID返回

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/users/me/profiles/timetracking?fields=periodFormat(id）

样品申请机构

{“ periodFormat”：{“ id”：“ FULL”}}

样品响应体

{“ periodFormat”：{“ id”：“ FULL”，“ $ type”：“ PeriodFieldFormat”}，“ $ type”：“ TimeTrackingUserProfile”}
