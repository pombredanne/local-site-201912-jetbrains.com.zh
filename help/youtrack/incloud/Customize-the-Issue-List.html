自定义问题列表

问题列表显示与搜索上下文和查询中定义的搜索条件匹配的问题列表。在列表视图中，您可以自定义显示内容以显示对您来说很重要的信息，或隐藏可能使您分心的细节。

目前，实验列表提供了更新版本的问题列表。此处提供的信息适用于该页面的经典版本。要了解实验版本中可用的更改，请参阅重新设计的问题列表 。

设置详细程度

工具栏上有一个选项，可让您设置列表中问题的详细程度。

提供以下选项：

选项

描述

详细视图

每个问题都显示在一个框中。默认情况下显示以下信息：

  • 问题编号。

  • 问题摘要。

  • 附加到问题的标签列表。

  • 报告问题的用户名。

  • 描述的前两到三行。

  • 问题的最后更新日期和日期。当您将指针移到该字段上时，将显示完整的日期和时间以及更新问题的用户的名称。

  • 当前为每个问题字段设置的值。

  • 一个或多个附件的存在由图标。

  • 任何可见度限制都由图标。

  • 问题的票数。如果您可以为该问题投票， 图标已启用。的表示您已经对该问题进行了投票。您可以单击此图标以重新投票。

  • 已添加到问题中的评论数。如果有一个或多个评论，则可以单击图标来显示它们。您也可以在展开的视图中添加新评论。如果没有评论，您可以单击此图标添加评论。

紧凑型视图

该视图显示与详细视图中相同的信息，除了问题说明。

一线检视

每个问题在列表中的一行上显示。默认情况下，该视图被限制为仅显示以下信息：

  • 存储在“ 优先级”字段中的值，以图标表示。

  • 问题编号。

  • 问题摘要。

  • 附加到问题的标签列表。

  • 图标显示一个或多个附件的存在。

  • 任何可见度限制都由图标

  • 问题的最后更新日期和日期。当您将指针移到该字段上时，将显示完整的日期和时间以及更新问题的用户的名称。

树视图

该视图显示与单行视图相同的信息。作为父任务的子任务的问题嵌套在树结构中。如果与当前搜索上下文匹配的问题是当前搜索上下文之外的问题的子任务，则父任务将以较浅的颜色显示。

设置字段可见性

除了详细程度之外，您还可以在列表中设置问题字段的可见性。通过此选项，您可以隐藏对您不重要的问题字段，并减少视觉混乱。

实验版的“ 问题”列表使用一组更新的控件来设置和更改自定义字段的可见性。要了解如何在实验版本中自定义字段可见性，请参见字段可见性设置 。

这些设置仅适用于您自己的帐户，不会影响其他用户的问题字段的可见性。

要自定义列表中显示的问题字段：

  1. 单击可见性设置按钮。

  2. 选择要在列表中显示的字段。

    如果将详细程度设置为单行或树状视图，则只能选择显示的字段是Priority和Tag 。

  3. 取消选择您不想在列表中显示的字段。

  4. 单击保存按钮。

    • 该列表将更新以显示所选字段的值。
