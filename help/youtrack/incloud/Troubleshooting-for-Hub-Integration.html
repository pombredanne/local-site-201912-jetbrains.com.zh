对集线器集成进行故障排除

此页面描述了集线器集成中最常遇到的问题以及解决这些问题的步骤。

连接性问题

集线器集成存在问题的最明显迹象是用户无法登录时。要确定YouTrack是否可以建立与集线器服务的连接，请检查登录表单。

  • 如果与集线器的连接处于活动状态，则会显示集线器登录表单。

  • 如果连接断开，则会显示本机的YouTrack登录表单。

集线器登录表格，连接正常

本机登录表单，未连接到集线器

“ 集线器集成”页面上的“ 连接到集线器”度量标准显示当前连接状态。如果显示以下任何状态消息，则说明YouTrack存在问题，需要支持团队的注意。要解决任何这些问题，请提交支持请求 。

  • 空集线器网址

  • 格式错误的中心网址

  • 中心网址错误

  • YouTrack未在集线器中注册

  • YouTrack已在集线器中注册，但其服务ID和密码不匹配

如果存在以下任一情况，请按照提供的说明进行调查并解决问题。

条件

YouTrack无法建立与集线器服务的连接。连接到集线器的状态显示为无法连接到集线器或连接到集线器超时 。当您尝试登录时，您看到消息权限管理服务（Hub）不可用。 。

原因

集线器服务已关闭或无响应。

解

通过访问内置的集线器服务来验证条件。单击“ 集线器 集成”页面上的集线器URL 。您也可以直接在Web浏览器中访问集线器服务。输入您的YouTrack实例的主机名，然后输入/hub 。例如， https://youryoutrackhostname/hub 。

如果您无法建立连接，请提交支持请求 。

条件

集线器不信任YouTrack。

原因

在大多数情况下，用户已访问连接到集线器的服务列表，并且不信任YouTrack。

解

访问集线器中的“ 服务”页面，并信任YouTrack服务。
  • 对于内置的集线器服务：

    1. 单击中心URL，以直接访问中心服务。

    2. 在“ 管理”菜单的“ 服务器设置”部分中，选择“ 服务” 。

    3. 在“ 服务”页面上选择YouTrack服务。

    4. 单击信任按钮。

  • 对于外部集线器服务：

    1. 打开工具栏中的“ 服务”菜单，然后选择“ 集线器” 。

    2. 在“ 管理”菜单的“ 服务器设置”部分中，选择“ 服务” 。

    3. 在“ 服务”页面上选择YouTrack服务。

    4. 单击信任按钮。

集线器事件同步问题

在正常情况下， 集线器事件同步状态为“ 活动”，正在同步所有内容 。如果状态显示初始化错误或检测到不兼容的集线器服务器 ，请提交支持请求 。

权限管理问题

在正常情况下， 权限管理状态为集线器权限管理 。如果状态显示YouTrack权限管理 ，请提交支持请求 。

损坏的数据映射

数据映射是存储在YouTrack和Hub数据库中的对象之间的虚拟连接。例如，每个用户都有两个用户配置文件。为了使应用程序功能正常运行，必须链接这些配置文件。

诊断例程会检查所有映射的一致性，并针对它们遇到的任何问题提供快速修复。

条件

用户报告一个或多个项目中的访问问题。

原因

有些用户已添加到组或从YouTrack中应用的组或其他更改中删除，或未映射到Hub数据库的其他更改。

解

  1. 单击运行诊断按钮。

  2. 检查页面底部的集成日志中是否有错误。

  3. 如果适用，请应用集成日志中建议的快速修复。
