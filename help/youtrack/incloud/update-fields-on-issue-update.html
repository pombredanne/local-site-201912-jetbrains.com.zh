问题更新的更新字段

工作流程最有用的应用程序之一是自动化问题更新。这种自动化的目的是使YouTrack为您完成肮脏的工作。本页上的用例工作流向您展示了如何根据对问题本身进行的更改来更新问题。

将更改应用于手动更新

您可能从未意识到，每天在YouTrack中使用的部分功能是通过工作流实现的。Subsystem Assignee是一种这样的工作流程，它会自动附加到新项目中。

此工作流程包含一个规则– Set subsystem owner as assignee for unassigned issues –正是标题所说的。这个概念相对简单。许多组织过程暗示您产品中的每个子系统都有负责人或“所有者”。此人是分配给此子系统的每个新问题的默认受让人 。当您为每个子系统指定所有者时，将相应地分配每个新问题。这是相应的工作流程规则：

var实体= require（'@ jetbrains / youtrack-scripting-api / entities'）; var工作流程= require（'@ jetbrains / youtrack-scripting-api / workflow'）; Exports.rule =实体。issue.onChange（{ctx.issue.fields。受让人&& ctx.issue.fields。子系统}，动作：function（ctx）{var issue = ctx.issue;如果（（issue.isReported &&（issue.fields.isChanged（ctx。子系统）|| issue.isChanged（'project'））））|| issue.becomesReported）{issue.fields。受让人= issue.fields。Subsystem.owner; }}，要求：{受让人：{类型：实体。User.fieldType}，子系统：{类型：实体。OwnedField.fieldType}}}）;

YouTrack中的默认工作流程使用workflow.i18n(...)本地化工作流消息的方法。在编写自己的工作流程时，您始终会知道YouTrack安装的语言，因此无需使用此方法。

如果要在默认工作流程中编辑消息，请删除此方法，并将其用法替换为简单的消息文本字符串。有关详细信息，请参阅本地化工作流消息 。

此类更新在很多情况下都很有用。这里只是几个例子：

  • 设置当前用户为受托人时，从打开的问题状态更改为正在进行中 。

  • 添加新评论后，将发布状态从“ 等待回复”更改为“ 打开” 。

  • 标志着一个问题，当修复版本设置为固定 。

  • 当问题状态更改为重复或不完整时，清除修订版本字段。

按设定的时间表应用更改

假设您要提高生产率并在项目中支持Pomodoro技术。有很多方法可以执行此操作：设置物理计时器，使用移动应用程序，启用浏览器扩展...或者您可以仅启用与YouTrack一起分发的Pomodoro Timer工作流程。

要了解有关Pomodoro技术的更多信息，请访问Cirillo Company的网站。

一组其他自定义字段支持此工作流，包括Pomodoro状态和Pomodoro倒计时 。一旦Pomodoro状态更改为Timer的运行时 ， Pomodoro倒数计时器将设置为25。预定规则会立即开始将倒数字段中的值每分钟减小1：

var实体= require（'@ jetbrains / youtrack-scripting-api / entities'）; var工作流程= require（'@ jetbrains / youtrack-scripting-api / workflow'）; Exports.rule =实体。Issue.onSchedule（{标题：workflow.i18n（'启用Pomodoro倒计时'），搜索：'具有：{Pomodoro倒计时} AND Pomodoro倒计时：-0 AND（Pomodoro状态：{计时器正在运行}或Pomodoro状态：{ }）'，cron：'0 * * * *？'，操作：function（ctx）{var issueFields = ctx.issue.fields; issueFields ['Pomodoro countdown']-= 1; }，要求：{'PomodoroCountdown'：{名称：'Pomodoro countdown'，类型：实体。Field.integerType}，'PomodoroState'：{名称：'Pomodoro state'，类型：实体。EnumField.fieldType}}}）;

工作流教程中描述了另一个说明这种类型的自动更新的示例 。
