使用Slack构建集成

通过工作流程API中的http模块 ，您可以使用工作流程编写与Slack的推送式集成脚本。这种类型的集成使您可以将特定问题事件的通知发布到Slack。

JetBrains在Slack App Directory中为YouTrack拥有自己的应用程序 。该集成支持通知，链接预览和问题草稿，而无需任何其他配置。

通过脚本编写自己与工作流的集成，可以选择何时和何处推送通知，但它不支持YouTrack应用程序内置的所有功能。要了解如何安装和使用YouTrack应用，请参阅Slack Integration 。

您不仅限于Slack集成的一种，也可以是另一种。您可以安装YouTrack应用以用于一般用途，但仍可以与工作流程建立自定义集成。

基于工作流的集成使用传入WebHooks启用Slack和YouTrack之间的集成。YouTrack工作流发送带有JSON有效负载的普通HTTP请求，其中包含问题ID和摘要。

此集成仅使您可以将通知发布到Slack。这种类型的集成不允许用户通过向机器人帐户发送直接消息来更新命令问题。

要设置与Slack的集成，请按照以下步骤操作：

  1. 向Slack添加自定义集成
  2. 自定义Slack Notifications工作流程
  3. 在YouTrack中启用Slack Notifications工作流程

此处介绍的基本设置使您可以将通知发布到Slack中的特定频道，以在单个YouTrack项目中发布事件。有关更复杂的集成，请参阅《 多通道和多项目配置》 。

向Slack添加自定义集成

这种类型的Slack集成是在YouTrack中按项目启用的，并在Slack中是为特定通道启用的。

要添加定制集成：

  1. 在Slack中，访问要与YouTrack集成的频道。

  2. 从“ 频道设置”菜单中，选择“ 添加应用” 。

    • “ 应用程序目录”页面将在Web浏览器中打开。

  3. 单击页面标题中的“ 生成 ”。

    • Slack API页面打开。

  4. 单击开始构建按钮。

    • 您的应用页面打开。

  5. 单击创建应用程序按钮。

    • 将打开“ 创建Slack应用程序”对话框。

  6. 填写表格中的字段：

    • 在“ 应用程序名称”字段中，为集成指定名称。

    • 从“ 开发松弛工作区”下拉列表中，选择要与YouTrack集成的通道。

  7. 单击创建应用程序按钮。

    • 新应用将打开“ 基本信息”页面。

  8. 在页面的添加功能部分中，点击传入Webhooks 。

    • “ 传入Webhooks”页面打开。

  9. 将“ 激活传入的Webhooks”选项切换为“ 开” 。

  10. 滚动到页面底部，然后单击将新Webhook添加到工作区按钮。

  11. 从发布到下拉列表中，选择要与YouTrack集成的频道。

  12. 单击授权按钮。

    • WebHook URL已添加到您的工作区。

  13. 单击复制按钮，将Webhook URL复制到剪贴板。

    在以下过程中，将此URL添加到YouTrack中的工作流规则中。

  14. （可选）为您的自定义集成自定义设置：

  15. 单击保存设置按钮。

您可以在“ 基本信息”页面上为您的应用程序自定义其他设置。这些设置会影响将通知发布到您的频道的帐户的外观。

页面的“ 显示信息”部分提供以下设置：

自定义松弛通知工作流程

下一步是编辑默认的Slack Notifications工作流程，以便将通知发送到Slack中的频道。

要自定义Slack Notifications工作流程：

需要权限：更新项目

  1. 在管理菜单的核心功能部分中，选择工作流程 。

  2. 找到您要修改的Slack Notifications工作流程，然后在列表中单击其名称。

    • 工作流程将在“ 工作流程”页面中打开。

  3. 点击左侧菜单中的发送至松弛链接。

    • 将发送到松弛规则在工作流编辑器中打开。

  4. 找到变量声明SLACK_WEBHOOK_URL并将值替换为在上一过程中从自定义集成复制的Webhook URL 。

  5. 单击工作流编辑器中的“ 保存”按钮。

    • 您所做的更改将应用于脚本。

在YouTrack中启用松弛通知工作流程

最后一步是将Slack Notifications工作流附加到YouTrack中的项目。

要启用Slack集成工作流程：

需要权限：更新项目

  1. 打开要与Slack集成的项目。

  2. 单击编辑项目链接。

  3. 选择工作流程选项卡。

  4. 从“ 附加工作流程”下拉列表中，选择“ 松弛通知” 。

    • Slack集成工作流程已附加到您的项目中。

  5. 如果工作流旁边显示需要安装标签，请单击侧栏中的“ 配置项目”按钮以解决所有错误。

    • 该工作流程已启用，可以与您的项目一起使用。

设置完成且工作流已附加到您的项目后，在项目中创建，解决或重新打开问题时，通知会发送到通道。如果要包括对团队重要的其他事件，则可以轻松自定义此通知方案。

在每个通知中，YouTrack问题ID均设置为指向YouTrack中问题的直接链接。

多通道和多项目配置

基本设置使您可以将更新从单个YouTrack项目发布到Slack中的特定频道。但是，您可以为更复杂的集成方案轻松配置此设置。

一个项目，多个渠道

Slack集成工作流的脚本可支持一个YouTrack项目与Slack中一个通道之间的集成。您可以将自定义集成添加到Slack中的不同渠道，并将每个渠道的Webhook URL添加到您的工作流程规则中。

一站式多项目

如果要将通知从多个YouTrack项目发送到Slack中的单个通道，则设置相对简单。

  1. 请按照此处描述的步骤将Slack频道的URL添加到工作流程规则中。

  2. 将此工作流附加到要向Slack报告的所有项目。

单独的项目，单独的渠道

在稍微复杂的设置中，您有多个团队在不同的渠道讨论不同的项目。您希望将通知从特定项目定向到每个团队讨论其问题的渠道。

处理这种情况的最简单方法是为每个项目创建单独的工作流程并相应地自定义规则。

要支持此用例：

  1. 在工作流编辑器中创建一个新的工作流。为工作流命名，以显示其与将应用该项目的项目的关系。

  2. 从Slack Notifications工作流程中复制规则，并将它们作为更改规则添加到新工作流程中。

  3. 更新工作流规则中的Webhook URL，以引用要在此项目中通知的Slack通道。

  4. 将更改保存在工作流编辑器中。

  5. 对每个项目中要通知的每个通道重复步骤1到4。

  6. 将每个工作流程附加到已为其定制的项目。

另一种方法是将条件语句添加到您的工作流程规则。使用条件语句，您可以标识每个问题所属的项目，并将通知发送到Slack中的专用通道。然后，当您想在新项目中启用Slack集成时，可以将相同的工作流程附加到工作流程中引用的所有项目，并更新相同的工作流程规则集。
