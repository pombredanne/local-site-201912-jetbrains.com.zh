管理永久令牌

在YouTrack InCloud中，永久令牌使开发人员可以在脚本和应用程序中使用REST API调用来安全地访问和执行操作，而无需实施OAuth 2.0身份验证流程。永久令牌允许使用授予用户帐户的权限访问服务。

该页面介绍了在用户个人资料中执行的带有永久性令牌的操作。有关使用永久令牌的REST API调用的示例，请参阅“开发者资源”部分中的“ 永久令牌授权”页面。

创建一个永久令牌

要使用授予您的用户帐户的权限以编程方式访问服务，请创建您自己的永久令牌。

除了此处列出的权限之外，您还需要权限才能读取要使用令牌访问的服务。在Hub中管理对已连接服务的访问。如果您使用的是YouTrack InCloud或使用内置Hub服务的YouTrack Standalone安装，则所有用户都可以访问YouTrack服务。

要获取新的永久令牌：

需要权限：更新自我

  1. 打开您的用户个人资料。

  2. 单击链接访问您的中心帐户 。

  3. 选择身份验证选项卡。

  4. 单击新令牌按钮。

  5. 在“ 新永久令牌”对话框中，为新令牌指定名称，并为其指定访问范围。令牌的范围是您可以使用此新令牌访问的服务列表。

    对于具有内置Hub的YouTrack实例，作用域基本上允许您将REST API调用发送到以下端点：

    范围

    网址

    描述

    YouTrack

    		 /youtrack/rest/...
    /youtrack/api/...

    将API请求发送到YouTrack的最终用户部分：问题，标签，命令，敏捷板，仪表板和报告。

    YouTrack管理

    		 /hub/api/...
    /hub/rest/...

    将API请求发送到YouTrack的管理部分：项目和访问管理，服务器设置等。

  6. 单击创建令牌按钮。

    • 显示带有新令牌的对话框窗口。

  7. 复制令牌并将其存储在安全的位置。使用以下两个操作之一来复制令牌：

    • 单击复制令牌按钮。

    • 用指针选择令牌，然后使用操作系统的标准键盘快捷键将当前选择复制到剪贴板。

    此时您必须复制令牌。关闭对话框后，您将无法再次访问令牌。如果意外关闭对话框，则获取令牌的唯一方法是重复该过程并创建一个新令牌。

  8. 复制令牌后，关闭对话框。

    • 新令牌与您的用户帐户相关联，并显示在“ 身份验证”选项卡的列表中。

删除永久令牌

永久令牌没有到期日期。如果您怀疑已验证的服务已被破坏，则可以在个人资料中显式删除此令牌。

要删除永久令牌：

需要权限：更新自我

  1. 打开您的用户个人资料。切换到您的中心帐户。

  2. 打开身份验证选项卡。

  3. 在令牌列表中，选择要撤消的令牌。

  4. 点击垃圾桶按钮。

    • 显示确认对话框。

  5. 单击删除按钮。

    • 选定的永久令牌将被删除并从列表中删除。
