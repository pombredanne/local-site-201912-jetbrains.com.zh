用户捆绑包中的组

该资源使您可以处理添加到特定用户集中的用户组。

资源资源
/ api / admin / customFieldSettings / bundles / user / {bundleID} / groups

退货实体

用户组 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 用户捆绑包中的组 。

  • POST ： 用户捆绑包中的组 。

UserGroup属性

代表一组用户。

下表描述了UserGroup实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

名称

串

组的名称。 Read-only 。 Can be null 。

ringId

串

集线器中组的ID。使用此ID进行集线器中的操作，以及在YouTrack和集线器之间匹配组。 Read-only 。

usersCount

长

组中的用户数。 Read-only 。

图标

串

组图标的URL。 Read-only 。 Can be null 。

allUsersGroup

布尔型

如果该组包含所有用户，则为true，否则为false。 Read-only 。

teamForProject

 项目

将这个小组设置为团队的项目。退货null ，如果没有这样的项目。 Read-only 。 Can be null 。

阅读用户组列表

获取添加到特定用户捆绑包的所有组的列表。

请求语法

GET / api / admin / customFieldSettings / bundles / user / {bundleID} / groups？{字段}＆{$ top}＆{$ skip}

请求参数

参数

类型

描述

领域

串

应在响应中返回的UserGroup属性列表。如果未指定任何字段，则仅entityID返回

$跳过

整数

可选的 。可让您设置要返回的多个实体，以在返回第一个实体之前跳过。

$ top

整数

可选的 。用于指定响应中返回的最大条目数。

添加一个新的用户组

将现有组添加到捆绑中。

所需权限

在使用该捆绑软件的所有项目中都需要“项目更新”权限。如果未在任何字段中使用捆绑软件，则至少在一个项目中需要“更新项目”权限。
