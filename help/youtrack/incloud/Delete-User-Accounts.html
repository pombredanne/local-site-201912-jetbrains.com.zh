删除用户帐号

如果要从系统中删除用户，则可以从YouTrack删除用户。例如，当您有一个用户用问题和评论向应用程序发送垃圾邮件或发布侮辱性或令人反感的内容时，请使用此选项。

对于其他情况，还有删除帐户的替代方法。

  • 如果您出于历史目的要保留问题中对用户的引用，可以禁止该用户帐户 。

  • 如果您有一个拥有多个帐户的用户，并且想要将两个帐户的引用合并为一个用户，则可以合并这些用户帐户 。

  • 如果用户要求被遗忘，则可以匿名化用户的个人数据 。

您也可以通过从系统中删除用户的凭据来简单地阻止访问。有关更多详细信息，请参阅帐户删除的替代方法 。

用户删除的信息

删除用户帐户时，相关的信息位将从系统中完全删除，并且无法恢复。

已删除的用户将从“ 受让人”字段和其他存储以下内容的字段的值集中删除： user输入所有项目。在允许这些字段存储空值的项目中，将删除设置为已删除用户的值。对于存储ownedField类型，在owner属性被删除。

从系统中删除了已删除用户拥有的以下对象：

  • 所有订阅。

  • 个人标签，保存的搜索，报告和敏捷面板（这意味着删除的用户是所有者，没有其他用户有权查看或编辑它们）。

  • 由删除的用户创建的所有仪表板，无论是否共享。

用户提及将被删除的用户的登录名替换为纯文本。

与已删除用户关联的审核事件不会更新。但是，指向已删除用户帐户的链接将重定向到被选择为其替换用户。

已被删除的用户

从YouTrack删除用户帐户时，对可以安全删除的已删除用户的引用将替换为Deleted User 。这适用于：

  • 问题，评论和附件中“ 创建者”和“ 更新者”字段的值。

  • 发行历史记录项。

  • 评论。

  • 工作项目。

  • VCS更改。

  • 共享保存的搜索的所有权。

  • 工作流程的上次修改属性。

  • 维护者，后备默认报告程序和邮箱规则的预定义报告程序。

替换用户

在操作过程中，系统会要求您选择替换用户。替换用户继承了对已删除用户的引用，这些引用保留了对系统实体的访问权限，否则不能为空。这适用于：

  • 问题，评论和附件的可见性限制。

  • 的价值user不能为空的字段。

  • 共享标签，报告和敏捷板的所有权。

  • 项目所有者的分配。

分配给已删除用户帐户的ID将作为别名传输到替换用户。结果，在审计事件中指向已删除用户帐户的链接将重定向到替换用户。

替换用户不继承已删除用户帐户的凭据。已删除用户的凭据将从YouTrack中删除。删除的用户将无法再登录YouTrack。

如果选择来宾帐户作为替换用户，则将进行特殊处理以防止不必要的公开披露。下列引用设置为“ 已删除用户”，而不是用来宾帐户替换对已删除用户的引用：

  • 问题，评论和附件的可见性限制。

  • 共享标签和报告的所有权。

  • 项目所有者的分配。

仔细选择替换用户。

  • 如果删除责任级别相对较高的用户，则理想情况下，应选择具有相似或更高访问级别的替代用户。

  • 仅当您确定要删除的用户不拥有其他用户要执行的工作时，才选择来宾用户作为替换。

如果要替换用户继承对已删除用户的所有引用，请使用合并操作。有关更多信息，请参见合并用户帐户 。

删除用户

您可以在“ 用户”页面上删除多个用户，也可以从其个人资料页面上删除单个用户。此操作无法还原。

删除一个或多个用户：

需要权限：删除用户

  1. 在管理菜单的访问管理部分，选择用户 。

  2. 在列表中选择一个或多个用户。使用过滤器查找与特定属性匹配的用户。

  3. 单击工具栏中的删除按钮。

    • 将打开“ 删除用户”对话框。

  4. 在“ 替换用户”下拉列表中，选择一个用户帐户。

  5. 单击删除用户按钮。

    • 所选用户将从YouTrack中删除。

    • 删除的用户将无法再登录YouTrack。

    • 替换用户继承设置为已删除用户的共享标签，保存的搜索，报告和敏捷板的所有权。

    • 将替换用户设置为分配给已删除用户的项目的项目所有者。

您也可以从用户个人资料中删除单个用户帐户。

要删除单个用户帐户：

需要权限：删除用户

  1. 打开用户个人资料。

  2. 单击删除用户按钮。

    • 将打开“ 删除用户”对话框。

  3. 在“ 替换用户”下拉列表中，选择一个用户帐户。

  4. 单击删除用户按钮。

    • 所选用户将从集线器中删除。

    • 删除的用户无法再登录到集线器或任何连接的服务。

    • 替换用户继承设置为已删除用户的共享标签，保存的搜索，报告和敏捷板的所有权。

    • 将替换用户设置为分配给已删除用户的项目的项目所有者。

帐户删除的替代方法

在许多情况下，删除用户帐户会导致信息丢失。例如，当您删除主动报告问题并添加评论的用户时，您将无法将此信息归因于特定用户。我们建议您仅在确实希望丢失此信息时才删除用户帐户。或者，您可以：

  • 禁止帐户 。

  • 合并到另一个帐户 。

  • 匿名化用户的个人数据 。

在某些情况下，您可能想阻止访问而没有禁止，删除或合并该帐户。为此，请删除用户的登录凭据。

删除用户帐户的登录凭据：

需要权限：更新用户

  1. 在管理菜单的访问管理部分，选择用户 。

  2. 在用户列表中，选择要阻止其登录YouTrack的帐户。

  3. 在用户帐户的“ 登录名”部分中，单击与该用户关联的每组凭据的“ 删除凭据 ”。
