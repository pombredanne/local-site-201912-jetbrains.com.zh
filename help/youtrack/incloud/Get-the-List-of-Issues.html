获取问题列表

GET /问题

获取与一个或多个搜索查询匹配的问题列表。

取得/休息/问题？{过滤器}＆{with}＆{max}＆{after}＆{wikifyDescription}

请求参数：

名称

类型

描述

过滤

串

用于搜索问题的查询。您可以指定多个查询。这些搜索过滤器的结果将在随后的块中返回，每个块列出一个问题。例如，如果您指定：
GET https://example.com/youtrack/rest/issue?filter=for%3A+me+%23Unresolved+%23Exception&filter=for%3A+me+%23Unresolved+%23Show-stopper

在结果xml中，问题的第一块将与搜索查询匹配for:me #Unresolved #Exception然后，所有与搜索查询匹配的问题for:me #Unresolved #Show-stopper 。

该表中描述的所有其他参数（“ with”，“ max”和“ after”）将应用于所有指定的过滤器，并且（如果需要）应在过滤器之后提供。

与

串

结果中应包含的字段列表。例如，要求get /issue?with=comment&with=Priority仅返回结果中这些特定字段的问题列表。其他字段将不包括在结果中xml 。

最高

整数

结果中要包含的最大问题数。如果为空，则返回10期。

此请求返回的最大问题数还受到YouTrack服务的“ 要导出的最大问题数”设置的值的影响。此设置位于“ 全局设置”页面上。例如，如果您使用max=10000 REST API请求中的参数，但“ 要导出的最大问题数”设置限制为5000个问题，响应最多返回5000个问题。

后

整数

获取问题列表之前要跳过的许多问题。例如，如果您指定after=12在请求中，然后在响应中，您将获得与请求匹配的所有问题，但未找到前十二个问题。

wikify描述

布尔值

定义问题描述是以HTML还是纯文本形式返回。

响应参数：

名称

描述

ID

数据库中的问题ID

吉拉德

如果问题是从Jira导入的，则表示它在Jira中具有的ID

projectShortName

问题项目的简称

numberInProject

项目中的发行数量

摘要

问题摘要

描述

问题说明

被创造

创建问题的时间（自1970年1月1日，格林尼治标准时间00:00:00以来的毫秒数，以该日期表示）。

更新

上次更新问题的时间（自1970年1月1日，格林尼治标准时间00:00:00以来的毫秒数，该日期表示）。

updaterName

更新该问题的用户的登录名（即最近的登录名）

解决

如果问题已解决，则会显示时间，最后一次将问题的解决状态设置为该问题的时间（自该日期开始，自1970年1月1日，格林尼治标准时间00:00:00以来的毫秒数）。

报名者

创建问题的用户的登录名

选民名

投票给用户的用户的登录名

commentsCount

发表的评论数

票数

发行票数

allowedGroup

有权阅读此问题的群组；如果未设置组，则表示任何用户都可以访问此问题

评论

代表问题评论（请参阅获取问题评论）

标签

标签，登录用户可以访问

领域

代表问题的任何字段，包括自定义字段（取决于name属性）。字段的数量和类型取决于项目设置。
