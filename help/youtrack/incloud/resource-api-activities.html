活动项目

该资源提供对活动的访问，并可以通过各种参数过滤活动。

资源资源
/ api /活动

退货实体

ActivityItem 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读ActivityItems列表 。

ActivityItem属性

表示问题或其相关实体中的更改。在用户界面中，您将这些更改视为“活动”流。它显示了该问题的所有更新的供稿：问题历史记录，注释，附件，VCS更改，工作项等。

下表描述了ActivityItem实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

类型

领域

 FilterField

过滤器字段。它包含有关目标实体中已修改属性的其他信息。例如，有关链接类型或修改后的自定义字段或捆绑软件类型的信息，等等。 Read-only 。 Can be null 。

获取所有可用活动的列表。

请求语法

GET / api / activities？{字段}＆{$ top}＆{$ skip}＆{categories}＆{reverse}＆{start}＆{end}＆{author}＆{issueQuery}

请求参数

参数

类型

描述

领域

串

响应中应返回的ActivityItem属性列表。如果未指定任何字段，则仅entityID返回

$跳过

整数

可选的 。可让您设置要返回的多个实体，以在返回第一个实体之前跳过。

$ top

整数

可选的 。用于指定响应中返回的最大条目数。

分类

串

强制性的 。参数按类别过滤返回的活动。您必须为每个请求至少指定一个类别。

您可以指定categories以下两种格式的查询参数：

  • categories=IssueCreatedCategory&categories=CommentsCategory
  • categories=IssueCreatedCategory,CommentsCategory

请参阅此表以了解活动项目的可用类别和返回类型之间的映射。

相反

布尔型

指示返回活动的顺序是从最新到最旧或相反。如果为“ false”，则首先返回与所选过滤器匹配的最早的活动项目。如果为“ true”，则首先返回最新活动。默认情况下，为“ false”。

开始

串

活动时间戳记所属时间间隔的开始时间戳记。如果未设置该参数，则将其视为0。

结束

串

活动时间戳记所属时间间隔的结束时间戳记。如果未设置该参数，则将其视为Long。MAX_VALUE。

作者

串

用于过滤作者活动的参数。可以指定当前登录用户的数据库ID或登录名，或Hub ID或“我”。

issueQuery

串

发出搜索查询。阅读有关搜索语法的更多信息。
