私人和公共问题领域

在YouTrack中，您可以将自定义字段标记为私有 。只有具有特定权限的用户才能阅读和/或更新此类字段。隐私由“ 专用字段”属性设置，您可以在创建或编辑自定义字段时进行设置。

对于私有字段，一组额外的权限确定用户是否可以读取或更新该字段。

  • 如果该字段是私有的，则用户必须具有分别读取项目中的Issue Issue私有字段和Update Issue Private字段的权限。

  • 对于公共字段，用户只需要具有“ 读取问题”和“ 更新问题”权限。

  • 问题报告者只需要具有“ 创建问题”权限，即可阅读他们自己报告的问题中的公共字段。

以下用例将更详细地描述此概念。

专用字段用例

假设您的公司已为外部客户端部署了软件安装。您希望客户组织的成员能够登录到YouTrack并创建问题，但只希望他们查看和更新特定字段，例如“ 优先级”和“ 受影响的版本”，以及添加和查看附件。

其他字段，例如State ， Type ， Assignee和Fix版本仅对内部开发人员可见。

要支持此用例：

  1. 为客户端创建一个新项目。

  2. 将您的开发人员添加到项目团队。这将为这些用户授予新项目中的开发者角色，并将他们自动添加到受让人列表中。默认情况下，授予开发人员角色“ 读取问题专用字段”和“ 更新问题专用字段”权限。如果您先前已经编辑了默认角色并更改了权限集，则可以为项目开发人员创建一个新角色，但要确保除所有其他权限外，他们还具有查看和编辑私有字段的权限。

  3. 为客户组织的成员创建一个组，然后为他们分配项目的默认Reporter角色。确保该角色无权 读取“读取问题专用字段”和“ 更新问题专用字段” 。

  4. 现在，将必填字段标记为私有。在这种情况下，这些字段是Type ， Assignee和Fix version ，它们仅供开发人员使用。默认情况下，“ 状态”字段标记为私有。

请注意，此解决方案有一些限制。用户可以读取和/或更新项目中的所有私有自定义字段，也可以不读取任何一个。

例如，如果您希望客户查看但不更新“ 状态”字段，则可以将“ 读取问题专用字段”权限授予“ 报告者”角色。但是，您的客户也可以读取（但不能更新）项目中的其他私有字段。

要为自定义字段管理更复杂的用例，请使用工作流程。有关更多信息，请参见防止不需要的更新 。

将字段设为私有

如果您对所有项目都具有相同的方案（例如，如我们的用例所述，您想限制对所有项目中问题字段的访问），则可以将共享字段设为私有。如果您的项目彼此之间有很大差异，则项目管理员可以独立配置这些设置。有关更多信息，请参见私人定制字段 。

全局隐私设置仅会影响该字段自动附加到的新项目和该字段手动附加到的现有项目的初始状态。退出该字段已附加到的项目的隐私设置不会受到影响。项目管理员可以随时覆盖初始隐私设置。

默认情况下，将字段设为私有：

需要权限：低级管理员写

  1. 在管理菜单的核心功能部分中，选择自定义字段 。

  2. 切换到“ 字段列表”选项卡。

  3. 选择您要设为私有的一个或多个字段。

  4. 点击工具栏中的设为私人按钮。

    • 附加到项目后，更新的字段仅对具有“ 读取问题专用字段”权限的用户可见。

    • 只有具有“ 更新问题专用字段”权限的用户才能更改保留隐私设置的项目中自定义字段的值。

    • 自定义字段在列表中的字段名称之前显示有键图标。

要取消此操作，请选择字段，然后单击工具栏上的“公开”按钮。
