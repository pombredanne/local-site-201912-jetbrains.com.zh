特定用户组的操作

此资源使您可以阅读YouTrack中的用户组和特定用户组的列表。

资源资源
/ api / admin / groups / {groupID}

退货实体

用户组 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 读取特定的用户组 。

UserGroup属性

代表一组用户。

下表描述了UserGroup实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

名称

串

组的名称。 Read-only 。 Can be null 。

ringId

串

集线器中组的ID。使用此ID进行集线器中的操作，以及在YouTrack和集线器之间匹配组。 Read-only 。

usersCount

长

组中的用户数。 Read-only 。

图标

串

组图标的URL。 Read-only 。 Can be null 。

allUsersGroup

布尔型

如果该组包含所有用户，则为true，否则为false。 Read-only 。

teamForProject

 项目

将这个小组设置为团队的项目。退货null ，如果没有这样的项目。 Read-only 。 Can be null 。

读取特定的用户组

读取特定用户组的属性。

所需权限

读取基本信息不需要任何权限，要访问所有数据，需要具有“读取用户组”权限。

请求语法

GET / api / admin / groups / {groupID}？{字段}

{groupID}

YouTrack中用户组的数据库ID。

请求参数

参数

类型

描述

领域

串

应在响应中返回的UserGroup属性列表。如果未指定任何字段，则仅entityID返回

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/groups/3-2?fields=id，名称，ringId，usersCount，teamForProject（名称，简称）

{“ ringId”：“ 034e661c-bfc0-4e43-8efa-acc95ac53fd4”，“ teamForProject”：{“ shortName”：“ SP”，“ name”：“ Sample Project”，“ $ type”：“ Project”}，“ name”：“示例项目团队”，“ usersCount”：3，“ id”：“ 3-2”，“ $ type”：“ UserGroup”}
