降价语法

在YouTrack中，您可以使用Markdown标记语法设置文本格式。您可以设置问题描述，补充文本字段，评论和工作项描述中的文本格式。此类标记是YouTrack Wiki标记的替代方法。

自从它作为公共功能引入以来，Markdown是在YouTrack中格式化文本的推荐选项。YouTrack Wiki标记仍受支持，但随着时间的推移将不建议使用并逐步淘汰。为了在过渡期间为用户提供支持，我们提供了一个选项，可以按用户在Markdown和YouTrack Wiki之间切换。只要您有能力将格式应用于文本输入，此选项就可见。

YouTrack中的Markdown实施遵循带有扩展名的CommonMark规范 。这些扩展支持格式规范中未包括的格式选项，例如清单，删除线文本，表，用户提及和问题链接。

要查看格式选项的完整列表，请单击Markdown链接以打开语法列表，然后单击帮助图标。这将在浏览器中打开“ YouTrack标记参考”页面。

“ YouTrack标记参考”页面提供了一个沙箱。使用沙箱预览格式化的输出。文本格式化为您喜欢的格式后，请复制沙箱的内容并将其粘贴到问题描述或注释中。

可用性

除了问题字段外，YouTrack还支持项目描述 ， Markdown Notes小部件 ， 用户协议 ， 系统范围内的横幅以及登录页面上的自定义消息中的 Markdown 。这些字段中不提供此处描述的许多扩展。例如，仅在问题中支持支持来自Google应用程序的嵌入内容，扩展视频，YouTube视频，问题链接和用户提及的扩展。

为确保您在这些字段中输入的内容的格式正确，请使用输入字段左侧的预览窗格。

字符格式

您可以使用以下Markdown语法将字符格式应用于一系列内联文本。

样式

句法

强大

用两个星号包围文本（ ** ）或两个下划线字符（ __ ）。

重点

带有单星号的环绕文本（ * ）或单个下划线字符（ _ ）。

删除线

用两个波浪号环绕文本（ ~~ ）。

码

用一个或多个反引号将文本引起来（ ` ）来格式化内联代码范围。

组合方式

用两个下划线包围单个下划线字符或两个波浪号。用三个星号包围文本，以突出显示重点。

You should read this, it's **very important**. You _might_ want to read this. ~~Never mind~~ - it wasn't _that_ important. Let's try a few `combinations`: **This text is strong, ~~this text is strong with strikethrough~~, and _this text is formatted with strong emphasis_** ***This text is formatted with strong emphasis too.***

标题

要应用标题样式，请以一到六个数字符号开始该行（ # ）。在数字符号后跟一个空格字符，然后输入标题文本。的数量#标志确定航向。

YouTrack还支持标题级别1和2的替代语法：

  • 对于标题级别1，输入一个或多个=下一行中的字符。

  • 对于标题级别2，输入一个或多个-下一行中的字符。

# Heading 1 ## Heading 2 ### Heading 3 #### Heading 4 ##### Heading 5 ###### Heading 6 Heading 1 - Alternative Syntax ======== Heading 2 - Alternative Syntax --------

段落和换行符

连续的文本行属于同一段落。使用以下准则将内容组织成段落并输入换行符。

  • 要开始新段落，请在文本行之间留空行。

  • 要在段落内开始新行，请在文本行的末尾输入两个或多个尾随空格。如需更明显的替代方法，请输入一个反斜杠（ \ ）。

Even though this text is written on two separate lines, it is parsed as a single paragraph. This paragraph is separated from the previous paragraph by a blank line. This text is parsed as a single paragraph. The two trailing spaces at the end of the previous line start a new line.\ The backslash character also starts a new line.

专题休息

在主题中创建主题栏目。使用以下任何一种方法来添加主题中断：

  • 三个或更多下划线（ ___ ）

  • 三个或更多个减号（ --- ）

  • 三个或更多星号（ *** ）

请注意，主题中断可能会中断段落并可能包含空格字符。缩进空格不能超过三个，并且必须使用相同的非空白字符。

The underscores on the next line create a thematic break below this paragraph. ___ The minus signs below must be separated from this paragraph by a blank line. If not, they are parsed as a level 2 heading. ---- Three or more asterisks also create a thematic break. ****

块引号

使用块引号可以特别注意来自另一个来源的引号。您可以将字符格式应用于带引号的块中的内联文本。

要将文本设置为引号块，请以一个或多个开头>字符。在这些字符后加一个空格，然后输入引号。的数量>符号确定引用块内嵌套的级别。

如果您的报价跨越多个段落，则每个空白行必须以>字符。这样可以确保将整个引号块组合在一起。

> Use block quotes to emulate reply text. > This line is part of the same quote. This line is not formatted and does not belong to the quote block. > This block spans multiple paragraphs. > > The second paragraph is grouped with the previous paragraph in the same quote block. > Character formatting is _also_ supported inside the **quote block**. > Quote blocks can also be nested. >> When you start a new line with additional > characters, >>> it simulates a threaded conversation.

缩进代码块

您可以使用等宽字体设置文本块的格式，以使其更易于识别和读取为代码。

要在Markdown中格式化代码块，请将代码块的每一行缩进至少四个空格。缩进的代码块不能中断段落，因此您必须在紧随其后的缩进的代码块的段落之间插入至少一个空行。输入处理如下：

  • 从代码块的每一行删除了一个缩进级别（四个空格）。

  • 代码块的内容是文字文本，不解析为Markdown。

  • 任何前导空格少于四个的非空白行将结束代码块并开始一个新段落。

Start an indented code block following a paragraph with a blank line and at least four spaces of indentation: This is a code block. Blank lines between indented lines do not end the code block. Here is some HTML:
© 2009—2017 JetBrains · All rights reserved
This line of text is not indented. It ends the code block and starts a new paragraph.

在信息字符串中设置语言时，语言标签显示在代码上方的行中。

堆栈痕迹

栅栏代码块可以格式化为堆栈跟踪。要将代码格式化为堆栈跟踪：

  1. 打开和关闭带有正三个反引号（ ``` ）或波浪号（ ~~~ ）。用与打开代码块相同的字符关闭代码块。

  2. 输入stacktrace在代码栏的开始行中设置语言。

  3. 将堆栈跟踪粘贴到受限制的代码块中。

格式化堆栈跟踪后，会将对源代码和其他问题的引用设置为链接。如果在开发环境中启用了该功能，则可以单击以下链接在IDE中打开目标引用。有关更多信息，请参见在IDE中打开堆栈跟踪 。

The following code block is formatted as a stacktrace. This fenced code block uses tildes, and sets the language as "stacktrace". ~~~stacktrace java.lang.NullPointerException at my.stack.trace(Of.java:123) ~~~

清单

使用以下语法创建列表：

  • 要创建无序列表，请以减号（ - ），星号（ * ）或加号（ + ）。

检查清单

CommonMark规范不支持清单。此功能已通过扩展实现。尽管类似于GitHub Flavored Markdown中的“ 任务”列表项（扩展名） ，但YouTrack的实现略有不同。

使用以下语法创建清单：

  • 就像创建无序列表一样，启动一个清单。在行的开头输入减号（-），星号（*）或加号（+）。这被解释为列表项标记。

  • 在列表项标记后面加上一个或多个空格，左方括号（[），空格字符或小写字母x，然后是右方括号（]）。这被解释为清单项目标记。

    • 如果方括号之间的字符是空格字符，则未选中该复选框。

    • 如果方括号之间的字符是小写字母x，则选中该复选框。

  • 在清单项目标记后输入任意文本字符串。文本与复选框内联显示。

TODO: * [x] This item is marked as complete. The checkbox is checked. - [ ] This item is incomplete. The checkbox is unchecked. + [ ] As with unordered lists, you can start the line with an asterisk, minus sign, or plus sign. - [ ] Nest checklist items with two spaces. - [ ] Checklists support multiple levels of nesting.

桌子

表格是为内容添加结构的绝佳工具。使用以下语法创建表：

  • 要创建列，请使用竖线（ | ）。外栏是可选的。

  • 标题行与表格的其余部分用三个或更多减号分开（ --- ）。

  • 要将列文本向右对齐，请添加冒号（ : ）到分隔符行中减号的右侧。

  • 要将文本置于列的中心，请添加冒号（ : ）到减号的两侧。

请注意，列不必在原始Markdown中完美对齐。您还可以将字符格式添加到表中的文本。

Kitchen Cleanup Rotation | Month | Assignee | Backup | | -------- | -------- | ------ | | January | Dave | Steve | | February | Gregg | Karen | | March | Diane | Jorge | Here's the same text with additional formatting and alternative syntax. + The text in the first column is flush right. + The text in the second column is centered. + The Markdown is stripped down to the minimum syntax that is required to render the table. Month | Assignee | Backup ---:|:---:| --- **January** | Dave | _Steve_ **February** | Gregg | _Karen_ **March** | Diane | _Jorge_

自动链接是在尖括号（）中设置的绝对URI和电子邮件地址（ < > ）。它们被解析为链接，以URL或电子邮件地址作为链接标签。不同于链接 ，让你指定链接文本和工具提示，这句法简单地转换URL或电子邮件地址转换为可点击的链接。

来自Google Apps的嵌入式内容

YouTrack支持图像语法的扩展，使您可以将Google应用中的内容嵌入到问题中。支持的应用程序包括文档，表格，幻灯片和表格。您可以将内容嵌入到支持Markdown的任何字段中，包括问题描述，补充文本字段，注释或工作项。

以下是Google嵌入的内容在YouTrack中的工作方式：

  • 内容必须发布到网络上。这使内容对互联网上的任何人都可见。与组织的其他成员共享文件不会将内容发布到Web。

  • 与上一点类似，在发行历史记录中不会跟踪应用于发布内容的更改。

  • 嵌入式内容不包含在应用程序中打开文件的链接。如果要授予其他用户访问文件的权限，则需要将链接单独粘贴到问题中。

  • 嵌入式内容未在YouTrack中为文本搜索建立索引。

反斜杠转义符

当您拥有要显示为书面形式的被解析为Markdown的字符时，可以使用反斜杠（ \ ）。

  • 非标记字符之前的反斜杠显示为反斜杠字符。

  • 转义字符被视为常规字符。它们在Markdown语法中的通常含义将被忽略。

  • 反斜杠转义不适用于受限制的代码块，内联代码范围或自动链接。
