通知记者批准修复

解决问题后，此工作流会将通知发送给报告问题的用户。

名称

@ jetbrains / youtrack-workflow-notify-reporter-approve-fix

自动连接

没有

模组

通知记者批准修复 （更改规则）
为报告者批准的修订 （状态机规则） 定义“状态”字段的过渡

用例

该工作流程最初是从提交的请求（ JT-7821 ）中获取的。

提交此问题的用户希望确保该问题的解决方案可以由报告该问题的用户批准。

运作方式如下：

  1. 记者制造了一个问题。

  2. 该问题已解决，状态设置为已修复（由TeamCity构建和部署）。

  3. 部署故障单后（在测试/阶段或生产环境中），状态将设置为“ 在测试/生产中待验证” （由开发人员手动或根据TeamCity的某些状态信息自动进行）。

  4. 通知已发送给记者。

  5. 记者在适当的环境中测试该问题，并确定

    • 它已得到解决，并且批准了该问题，并且已经关闭（或状态为“已批准”或类似状态）

    • 它仍然无法正常工作，并以“未获批准”（或项目负责人之类的东西）返回给开发人员。

模组

该工作流程包括两个模块。第一个包含一个发送通知的变化规则，第二个包含管理问题生命周期的状态机规则。

通知记者批准修复

第一条规则通知报告该问题的用户，并将报告者设置为受让人。

更新问题后，此规则将检查状态是否更改为待验证 。如果为true，则：

  • 通知记者。

  • 记者被设置为问题的受让人 。

var实体= require（'@ jetbrains / youtrack-scripting-api / entities'）; var工作流程= require（'@ jetbrains / youtrack-scripting-api / workflow'）; Exports.rule =实体。Issue.onChange（{标题：workflow.i18n（'向通知发送通知以批准修复程序'），警卫：function（ctx）{return ctx.issue.fields.becomes（ctx。状态，ctx。州。待定）; }，动作：function（ctx）{var issue = ctx.issue;问题域。受让人= issue.reporter; issue.reporter.notify（workflow.i18n（'请批准对问题{0}的修复，'issue.id'），workflow.i18n（'您已报告了问题{0}。请验证问题的已应用修复程序并设置适当的状态。'，issue.id），true）; }，要求：{州：{类型：实体。EnumField.fieldType，待定：{名称：“待验证”}}，受让人：{类型：实体。User.fieldType}}}）;

为“状态”字段定义转换，以获取报告者批准的修订

下一条规则定义了支持该用例的问题的生命周期。

该规则为发布状态定义了以下过渡：

  1. 问题始于状态Submitted 。

  2. 从初始状态开始，问题只能转换为以下状态：

    • 在事件（操作） 打开时 ，状态设置为打开 。

  3. 从“ 打开”状态，问题只能转换为以下状态：

    • 在事件（动作） 修复时 ，状态设置为修复 。

  4. 当状态设置为固定时 ，将强制用户在构建字段中设置固定 。

  5. 从固定状态，问题只能转换为以下状态：

    • 在事件（操作） 发送进行验证时 ，状态设置为“ 待验证” 。

  6. 当状态设置为“ 待验证”时 ，将报告者设置为“ 受让人” 。通知记者，并要求其批准此修复程序。

  7. 从待审核状态，问题只能转换为以下状态：

    • 在事件（操作） 批准后 ，状态设置为已验证 。

    • 事件（动作） 重新打开时 ，状态设置为“ 打开” 。

  8. 从打开状态开始，问题只能转换为已修复 。

  9. 从“ 已验证”状态开始，不允许进行进一步的状态转换。
