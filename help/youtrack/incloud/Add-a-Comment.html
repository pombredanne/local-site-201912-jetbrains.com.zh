添加评论

评论是增加问题细节或提出问题以澄清问题的好方法。报告问题后，您可以为问题添加评论。您不能在问题草稿中添加评论。您和其他用户可以添加评论以回答问题，或提供有助于解决问题的其他信息。

在问题列表中添加评论

您可以直接从“ 问题”列表中为问题添加评论。

在“ 问题”列表的实验版本中向问题添加评论的功能仅适用于已经有一个或多个评论的问题。要按此处所述使用这些控件，请关闭实验性UI。有关说明，请参阅重新设计的问题列表 。

要从“问题”列表中添加评论，请执行以下操作：

需要权限：阅读问题，更新问题，创建评论

  1. 单击评论图标。图标旁边的数字显示已添加到问题中的评论数。

    • 如果已经添加了注释，则选定的问题将展开以显示所有注释以及用于添加新注释的输入字段。

    • 如果没有评论，则所选问题将展开以显示用于添加新评论的输入字段。

    如果您在“单行”视图中查看“ 问题”列表，请展开问题，然后单击“注释”图标。

  2. 在输入字段中输入您的评论。您可以使用Wiki标记来格式化文本。通用标记标签的提示显示在输入字段下方。要查看可用标签的列表，请单击更多链接。

  3. 要限制评论的可见性，请从“可见到”下拉列表中选择一个组。有关更多信息，请参见可见性 。

  4. 要添加带有注释的文件，请单击“ 附加”链接。添加注释后，附件将与注释内联显示。

  5. 完成后，单击添加评论按钮。您也可以按Ctrl + Enter在键盘上。

为多个问题添加评论

您可以使用命令同时将相同的注释添加到多个问题。当您要用相同的评论更新一批问题时，请使用此功能。

要对多个问题添加评论：

需要权限：阅读问题，更新问题，创建评论

  1. 在“ 问题”列表中选择要更新的问题 。

  2. 单击工具栏中的“ 添加注释”按钮。

    • 将打开“ 应用命令”对话框。的comment命令已被指定。

    另外，您也可以输入任何字母或按Ctrl + Alt + J打开“ 应用命令”对话框。在这种情况下， 命令输入字段为空。

  3. 在输入字段中输入您的评论。

  4. 要限制评论的可见性，请从“ 可见的组”下拉列表中选择一个组。有关更多信息，请参见可见性 。

  5. 完成后，单击“ 应用”按钮。您也可以按Ctrl + Enter在键盘上。

    • 应用命令，并将注释添加到所有选定的问题。

    • 注释旁边会显示任何将更新应用于问题的命令。有关更多信息，请阅读下面与评论相关的更新部分。

在单期视图中添加评论

在单个问题视图中添加评论非常容易：只需输入评论，然后单击添加评论按钮即可。

要对问题添加评论：

需要权限：阅读问题，更新问题，创建评论

  1. 打开您要添加评论的问题。

  2. 在输入字段中输入您的评论。

  3. 要限制评论的可见性，请从“可见到”列表中选择一个或多个用户或组。有关更多信息，请参见可见性 。

  4. 要添加带有注释的文件，请单击“ 附加文件”链接。添加注释后，附件将与注释内联显示。

  5. 完成后，单击添加评论按钮。您也可以按Ctrl + Enter在键盘上。

从VCS提交生成评论

当您处理与版本控制系统集成的项目中的问题时，可以在将更改提交到存储库时生成注释。以下集成支持此功能：

  • 构建服务器集成 （TeamCity和Jenkins）

  • VCS集成 （GitHub，GitLab和Bitbucket）

VCS或构建服务器集成的设置包括一个用于分析提交以发表问题注释的选项。

  • 启用“ 分析问题注释的提交”选项后，提交消息文本将命令复制到单独的问题注释后开始新的一行。此行为使您可以在注释文本中提及用户，并在处理提交消息时生成通知。它还允许您选择使与提交相关的信息对无权查看VCS更改的外部用户可见。

  • 禁用此选项后，不会从提交消息中生成问题注释。此行为可确保将IDE或VCS生成的其他消息与VCS更改一起显示，并且不会生成冗余注释。例如，消息指示从存储库其他分支中的特定提交中精心挑选的更改。如果在项目中启用了“ 分析提交的问题注释”选项，则这些自动生成的消息将生成单独的注释。

可以基于每个项目独立启用或禁用此功能。由项目管理员确定哪种行为最适合其项目中的问题。

如果启用了此功能，则可以使用以下语法从VCS提交消息生成注释：

[commit message text] #issueID [command 1] [command 2] ... [command n] [issue comment text] ...

要么

[commit message text] ^issueID [command 1] [command 2] ... [command n] [issue comment text] ...

整个提交消息（包括命令和问题注释）始终显示为活动流中VCS更改记录的一部分。

提交消息分为以下几部分：

零件

描述

提交消息文字

第一个字符字符串将添加到VCS提交消息中。此文本不用作对此问题的评论。完整的VCS提交消息显示在活动流的VCS更改列表中。

提交消息文本可以换行并包含空行。

指令

YouTrack会将数字符号（＃）或插入符号（^）识别为问题的标志，然后跟随命令。解析数字符号或插入号后面的字符以匹配问题ID。问题ID后面的任何字符串都将被解析为命令，直到当前行的末尾。

有关更多信息，请参见《 VCS提交》中的“应用命令” 。

发表评论文字

如果为您的集成启用了“ 分析提交问题的注释”选项，则从VCS提交消息中复制命令并将其作为注释发布到问题后，将在新行中添加的文本。否则，该文本仅显示在VCS提交消息中。

问题编号

在提交消息中找到的任何问题ID都将转换为超链接。这包括为命令标记的问题以及在提交消息或问题注释文本中简单提及的问题。

将文件附加到注释

向问题添加评论时，您可以将文件直接附加到评论本身。附加到注释的文件显示在问题的“ 附件”部分中，并且也显示为注释文本正下方的缩略图。

当查看单个问题以及在敏捷板上以查看模式打开卡时，可以在“ 问题”列表中使用将文件附加到注释的选项。使用命令添加注释时，无法附加文件。

通过将文件附加到注释而不是仅将文件附加到问题，使用版本文件的团队可以快速找到最新更新的副本。

要将文件附加到注释：

需要权限：阅读问题，更新问题，创建评论，添加附件

  • 单击输入字段下方的附加文件选项以获取新注释，然后从本地目录中选择一个或多个文件。单击添加评论按钮以添加评论文本并附加文件。

  • 要将文件添加到现有注释中，请单击注释工具栏中的“ 附加文件”图标，然后从本地目录中选择一个或多个文件。

  • 将文件拖到目标注释或新注释的输入字段。单个问题视图的旧版本不支持将文件拖动到现有注释的选项。在旧版本中，您只能将文件拖到评论草稿中。

对于附加到注释的每个文件，您可以访问页面“ 附件”部分中可用的相同操作。

  • 对于具有可见性限制的文件，将指针移到挂锁图标上以显示可以查看文件的用户和组的列表。

  • 单击缩略图以预览图像或下载其他文件类型。

  • 单击下载图标，将文件复制到本地目录。

  • 单击编辑图标以更新文件。

    • 图像在图像编辑器中打开。在这里，您可以将更改直接应用于图像，更改文件名并更新文件的可见性设置。有关更多信息，请参见编辑图像 。

    • 对于其他文件，此操作将打开“ 编辑附件”对话框。在这里，您可以更改文件名并更新可见性设置。有关更多信息，请参见“ 编辑或删除附件” 。

    应用于附加到注释的文件的任何更改都将复制到页面“ 附件”部分中显示的文件副本中。

  • 单击删除图标以从注释中删除文件。确认此操作后，对该文件的引用也会从页面的“ 附件”部分中删除。

评论相关动作

每个问题评论都有一个工具栏。使用此工具栏中的控件可以执行特定于所选注释的操作。

可以使用以下控件：

控制

描述

附加档案

打开一个对话框，用于从本地目录中查找文件。使用此选项可将文件直接附加到注释。有关更多信息，请参见将文件附加到注释 。

回复

将所选注释作者的登录名插入新注释的输入字段。此选项仅适用于其他用户添加的注释。使用此选项可以直接回复输入原始评论的用户。有关更多信息，请参见提及其他用户 。

编辑

打开选定的注释进行编辑。根据您的权限，您可以更新评论文本或更改评论的可见性设置。

删除

从问题中删除所选评论。

固定链接

将指向问题注释的直接超链接复制到剪贴板。

与评论相关的更新

在添加评论后一分钟内应用于问题的所有更改均被视为与该评论相关。在活动流中启用问题历史记录后，这些更改将在评论后立即显示。
