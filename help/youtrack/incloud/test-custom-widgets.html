测试自定义小部件

“ 自定义小部件”页面包含“ 小部件游乐场”页面的链接。小部件游乐场可以让您预览和测试开发中的小部件。

基本上，此页面使您可以在沙盒环境中加载窗口小部件。

在小部件游乐场中加载自定义小部件

要测试您的小部件，您需要将其加载到小部件游乐场。

要将小部件加载到小部件游乐场中：

需要权限：低级管理员写

  1. 首先，将您的窗口小部件上传到开发服务器。

    您可以使用托管静态文件的任何服务器软件，例如webpack 。如果使用此选项，则开发服务器必须提供一个特殊的标头，该标头使小部件游乐场可以访问以加载内容。此标头设置在webpack-config为了webpack-dev-server 。例如：

    devServer：{标头：{'Access-Control-Allow-Origin'：'*'}}，

    您还可以使用npm http-server在本地目录上运行静态服务器 （需要Node.js）。例如，以下命令将全局安装http服务器并提供存储在widget-folder端口目录9002 。

    npm安装http-server -g http-server ./widget-folder --cors -c-1 -p 9002

    然后，您可以将窗口小部件从http://localhost:9002/地址。

    无论使用哪种选项，都需要通过HTTPS提供小部件。有关更多信息，请参见通过安全连接测试小部件 。

  2. 打开小部件游乐场页面。

    • 在“ 管理”菜单的“ 服务器设置”部分中，选择“ 自定义窗口小部件” 。

    • 单击小部件游乐场链接。

  3. 在输入字段中输入自定义窗口小部件的网址。

  4. 单击加载小部件按钮。

    • 小部件已加载并显示在小部件游乐场中。

如果修改代码并将这些更改上传到Web服务器，请使用“ 重新加载小部件”按钮刷新小部件内容并测试更新。

通过安全连接测试小部件

要将自定义窗口小部件加载到窗口小部件游乐场中，用于服务窗口小部件的开发服务器应通过HTTPS可用。这是所有现代Web浏览器的安全性要求。

您的浏览器还将阻止任何尝试与YouTrack InCloud实例建立不安全连接的尝试。

考虑以下选项：

  • 如果您使用webpack ，则可以通过--https用于运行开发服务器的命令中的选项。该服务器将具有自签名证书，因此您可能需要将其添加到本地计算机上的受信任证书列表中。有关更多信息，请参阅https的webpack示例 。

  • 您可以使用ngrok或类似的隧道反向代理，使用有效的SSL证书通过HTTPS将本地服务器公开给Internet。要运行ngrok ，请参阅您操作系统的下载和安装说明。

    以下示例显示了如何运行开发服务器并将其公开用于macOS和Linux的Web：

    ＃运行您的开发服务器，例如：http-server ./widget-folder --cors -c-1 -p 9002＃暴露此端口ngrok http 9002

    ngrok服务会生成一个安全的公共URL，您可以使用该URL来将小部件加载到小部件游乐场页面中。
