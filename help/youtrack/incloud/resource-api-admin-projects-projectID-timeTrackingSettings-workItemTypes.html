工作项目类型

此资源使您可以配置项目的工作项类型。

资源资源
/ api / admin / projects / {projectID} / timeTrackingSettings / workItemTypes

退货实体

WorkItemType 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读WorkItemTypes列表 。

  • POST ： 添加一个新的WorkItemType 。

WorkItemType属性

表示可以分配给工作项目的工作类型。

下表描述了WorkItemType实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

名称

串

工作项类型的名称。 Can be null 。

自动附加

布尔型

如果为true，则在为它启用时间跟踪后，会自动将其添加到项目中。

阅读WorkItemTypes列表

获取项目中使用的所有工作项类型的列表。

请求语法

GET / api / admin / projects / {projectID} / timeTrackingSettings / workItemTypes？{字段}＆{$ top}＆{$ skip}

请求参数

参数

类型

描述

领域

串

应在响应中返回的WorkItemType属性列表。如果未指定任何字段，则仅entityID返回

$跳过

整数

可选的 。可让您设置要返回的多个实体，以在返回第一个实体之前跳过。

$ top

整数

可选的 。用于指定响应中返回的最大条目数。

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/projects/0-0/timeTrackingSettings/workItemTypes?fields=id、name、url&$top=2

样品响应体

[{“名称”：“开发”，“ id”：“ 65-0”，“ $ type”：“ WorkItemType”}，{“ name”：“ Testing”，“ id”：“ 65-1”，“ $ type“：” WorkItemType“}]

添加一个新的WorkItemType

将现有工作项类型添加到项目中。要创建新的工作项类型，请使用系统时间跟踪设置。

必填属性 ： id -工作项类型的数据库实体ID。

请求语法
