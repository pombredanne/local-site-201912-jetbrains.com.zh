YouTrack InCloud 2019.2帮助

詹金斯整合

请按照本页上的说明在YouTrack项目和Jenkins中的作业之间建立连接。YouTrack和Jenkins之间的集成可在两个应用程序中启用有用的功能：

  • 从连接的项目中获取内部版本号，并自动更新自定义字段中的值，这些字段存储内部版本号以解决问题。

  • 将VCS更改记录添加到提到的提交消息中。要了解有关YouTrack问题中VCS更改的更多信息，请参阅查看和编辑VCS更改 。

  • Jenkins组装包含提交的构建时，将命令应用于提交消息中的问题。要了解如何格式化提交消息中的命令 ，请参阅《 VCS提交中的应用命令》 。

您可以将YouTrack与Jenkins 1.651.3（1.x分支）或2.15（2.x分支）或更高版本集成。

先决条件

在开始之前，请验证以下要求：

  • 您具有运行1.651.3版（1.x分支）或2.15版（2.x分支）或更高版本的Jenkins安装。

  • 您的Jenkins帐户对包含您要链接到YouTrack中的项目的作业的视图具有读取权限。

  • 要与Jenkins集成的YouTrack项目使用一个自定义字段，该字段将值存储为构建类型。您可以使用构建中固定的默认自定义字段，也可以使用存储构建类型的任何其他自定义字段。

    如果您不想在问题中添加内部版本号，则可以启用集成而无需将内部版本号映射到YouTrack中的自定义字段。在这种情况下，您的开发人员仍然可以提及问题并在VCS提交消息中应用命令。

  • 要将链接存储库中的VCS用户名映射到YouTrack中的用户帐户，必须将Jenkins中的项目配置为在变更日志中使用提交作者。使用以下导航路径找到此设置： 项目>配置>源代码管理>其他行为>在Changelog中使用提交作者 。

  • 要启用从问题视图中显示的VCS更改到diff的链接，您需要在Jenkins中安装Last Changes插件。

连接到Jenkins服务器

要启用集成，您需要连接到一台或多台Jenkins服务器。该操作也可以由项目管理员基于每个项目执行。有关更多信息，请参阅与Jenkins集成 。

作为管理员，您可以为YouTrack中的任何项目启用Jenkins集成。

要连接到Jenkins服务器：

需要权限：更新项目

  1. 在“ 管理”菜单的“ 集成”部分中，选择“ 构建服务器集成” 。

  2. 在“ 构建服务器集成设置”页面上，单击添加新的构建服务器链接。

    • 侧边栏中将打开“ 添加构建服务器”对话框。

  3. 输入以下设置的值：

    设置

    描述

    网址

    输入Jenkins服务器的URL。

    类型

    选择詹金斯 。

    登录

    输入Jenkins用户帐户的用户名。

    密码

    输入Jenkins用户帐户的密码。

  4. 展开“ 高级设置”部分，并配置以下可选设置：

    设置

    描述

    SSL密钥

    如果Jenkins服务器需要安全连接，并且您已上载SSL客户端密钥以验证您的YouTrack服务，请从下拉列表中选择客户端密钥。有关更多信息，请参见SSL密钥 。

    套接字超时

    设置在断开套接字连接之前等待Jenkins服务器响应的最长时间。默认值为30秒。

    读取超时

    设置在关闭连接之前等待从Jenkins服务器进行完整数据传输的最长时间。默认值为30秒。

  5. 单击测试连接按钮。

    • YouTrack验证与Jenkins服务器的连接。

  6. 单击添加服务器按钮。

    • Jenkins服务器连接设置已保存。

重复这些步骤以将连接添加到其他Jenkins服务器。

定义项目之间的映射

连接到Jenkins服务器时，必须在YouTrack项目和Jenkins作业之间创建映射。作为管理员，您可以自己映射项目，也可以让项目管理员映射自己的项目和作业。有关适合项目管理员的详细说明，请参阅与Jenkins集成 。

第一组设置将YouTrack连接到Jenkins服务器上项目的作业。通过默认分支的其他选项，您可以定义分配了内部版本号的问题。您还可以指定命令以应用到YouTrack处理的所有问题。

定义YouTrack和Jenkins之间的映射：

需要权限：更新项目

  1. 在构建服务器集成列表中选择一个Jenkins服务器。

  2. 单击添加新映射按钮。

    • 侧栏中将打开“ 创建映射”对话框。

  3. 输入以下设置的值：

    设置

    描述

    YouTrack专案

    选择要映射到Jenkins中的项目的YouTrack项目。

    詹金斯服务器

    选择包含要映射到的项目的Jenkins服务器。如果仅连接到一台Jenkins服务器，则默认设置为该服务器。

    詹金斯视图

    选择包含Jenkins项目的视图的名称。

    詹金斯·乔布斯

    在Jenkins项目中选择作业。

    工艺变更

    选择YouTrack如何处理Jenkins处理的版本中的VCS更改。提供以下选项：

    • 不附加 -包含对YouTrack问题ID的引用的VCS更改不会添加到所引用问题的活动流中。当您只希望集成将内部版本号添加到自定义字段时，请使用此选项。

    • 附加 -包含对YouTrack问题ID的引用的VCS更改已添加到所引用问题的活动流中。活动流中的条目包含指向包含更改的内部版本的链接。不会为YouTrack命令解析提交消息。

    • 附加和应用命令 -包含对YouTrack问题ID的引用的VCS更改将添加到所引用问题的活动流中。活动流中的条目包含指向包含更改的内部版本的链接。还针对YouTrack命令解析提交消息。

    可见到

    要限制活动流中VCS更改显示的注释和更新的可见性，请从列表中选择一个或多个组。否则，此活动类型对任何有权读取项目中问题的用户都是可见的。

    解析提交问题评论

    启用后，提交消息文本的特定部分将作为注释复制到问题。当您将部分提交消息作为注释复制到问题时，您可以触发@mention通知并将信息公开给无法访问VCS更改的用户。

    此设置不会影响VCS更改中提交消息的显示方式。整个提交消息（包括命令和问题注释）始终显示为活动流中VCS更改记录的一部分。

    您仅应在以下情况下启用此选项：

    • 您希望在提交消息中提及其他用户，并在将文本复制到问题注释时生成通知。

    • 您限制了VCS更改的可见性，并希望使与提交有关的信息作为注释对外部用户可见。

    要了解有关YouTrack如何处理提交消息的更多信息，请参阅《 VCS提交》中的“应用命令” 。

    如果您不想将内部版本号添加到问题，请从步骤8继续。

  4. 单击以展开“ 默认分支的操作”设置。这些设置确定在默认分支中将哪些操作应用于构建。

  5. 在表单的“ 处理”部分中配置选项。这些选项确定集成处理哪些内部版本以及为哪些问题分配内部版本号。

    选项

    描述

    构建失败

    将版本号添加到所有版本的问题中，包括失败的版本。禁用后，只会将成功的构建添加到YouTrack中的问题。

    添加每个版本

    确定将哪些内部版本号添加到“ 内部字段”设置中存储的自定义字段的值集中。即使在构建中没有处理任何YouTrack问题时，也可以启用将构建添加到构建字段中的一组值中。该选项考虑的构建受“ 失败的构建”设置影响。

    相关问题

    将内部版本号添加到提交和内部注释中提到的问题。禁用后，YouTrack会忽略提到的问题。

    将此选项与“ 问题匹配”查询设置一起使用，可以确定为哪些问题分配了内部版本号。

    问题匹配查询

    将内部版本号添加到与特定过滤条件匹配的问题上。标准搜索查询是State: Fixed State: Verified resolved date: ${build.time}..${prev.build.time} 。此查询确保仅考虑自上次成功构建和当前构建以来已解决的问题。

    将此设置与“ 相关问题”选项一起使用，可以确定为哪些问题分配了内部版本号。

  6. 对于Build字段 ，在项目中选择用于存储Jenkins的内部编号的自定义字段。该选择仅限于存储以下内容的自定义字段： build类型。

  7. 在“ 应用命令”字段中，输入一个可选命令以应用于在YouTrack中分配了内部版本号的所有问题。使用${build}变量以引用詹金斯的内部版本号。例如： Subsystem Testing Test in build ${build}使用此命令，YouTrack处理的所有问题都分配给“ 测试”子系统，并且“ 在构建中测试”字段设置为内部版本号。

  8. 单击创建映射按钮。

    • YouTrack项目已映射到Jenkins中的作业。

    • Jenkins集成已启用，并且可以在选定的YouTrack项目中使用。

占位符

您可以在查询和命令设置中使用占位符。这些占位符引用为Jenkins中的构建存储的属性。当YouTrack处理构建时，占位符将替换为所连接版本控制系统中的相应值。可以使用以下占位符：

占位符

描述

$ {build.time}

YouTrack处理的版本的日期和时间。

$ {prev.build.time}

YouTrack处理的版本之前的版本的日期和时间。

$ {build}

YouTrack处理的内部版本号。
