使用特定CustomFieldDefaults的操作

此资源使您可以阅读和更新项目中自定义字段的默认设置。

资源资源
/ api / admin / customFieldSettings / customFields / {fieldID} / fieldDefaults

退货实体

CustomFieldDefaults 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 读取特定的CustomFieldDefaults 。

  • POST ： 更新特定的CustomFieldDefaults 。

CustomFieldDefaults属性

表示自定义字段的与项目有关的默认设置。在将自定义字段附加到项目时，将应用这些设置。之后，默认设置中的任何更改都不会影响该项目的字段设置。

下表描述了CustomFieldDefaults实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

canBeEmpty

布尔型

指示此字段在问题中是否可以为空值。

emptyFieldText

串

字段为空时显示为占位符的文本。 Can be null 。

isPublic

布尔型

如果为true，则用户需要具有“ 读取问题”和“ 更新问题”权限才能访问此字段。否则，该字段将被视为私有字段，并且需要“ 读取问题私有字段”和“ 更新问题私有字段”权限。

父母

 CustomField

引用自定义字段。 Read-only 。

读取特定的CustomFieldDefaults

获取该字段的默认设置。

所需权限

需要读取访问自定义字段。

请求语法

GET / api / admin / customFieldSettings / customFields / {fieldID} / fieldDefaults？{字段}

请求参数

参数

类型

描述

领域

串

应在响应中返回的CustomFieldDefaults属性列表。如果未指定任何字段，则仅entityID返回

样品

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/customFieldSettings/customFields/58-8/fieldDefaults?fields=canBeEmpty,emptyFieldText,isPublic,parent(id,name）

样品响应体

{“ emptyFieldText”：null，“ canBeEmpty”：true，“ isPublic”：true，“ parent”：{“ name”：“到期日”，“ id”：“ 58-8”，“ $ type”：“ CustomField “}，” $ type“：” CustomFieldDefaults“}

更新特定的CustomFieldDefaults

更新该字段的默认设置。

所需权限

需要使用该字段的所有项目中的项目更新权限。如果该字段是自动附加的，则需要“低级管理员写”权限。

请求语法

POST / api / admin / customFieldSettings / customFields / {fieldID} / fieldDefaults？{字段}

请求参数

参数

类型

描述

领域

串

应在响应中返回的CustomFieldDefaults属性列表。如果未指定任何字段，则仅entityID返回

样品

让我们默认将该字段设为私有。我们还设置它不能为空。

样本请求URI

https://example.myjetbrains.com/youtrack/api/admin/customFieldSettings/customFields/58-8/fieldDefaults?fields=canBeEmpty,emptyFieldText,isPublic,parent(id,name）

样品申请机构

{“ emptyFieldText”：null，“ canBeEmpty”：false，“ isPublic”：false}

样品响应体

{“ emptyFieldText”：null，“ canBeEmpty”：false，“ isPublic”：false，“ parent”：{“ isAutoAttached”：false，“ name”：“到期日”，“ id”：“ 58-8”，“ $ type“：” CustomField“}，” $ type“：” CustomFieldDefaults“}
