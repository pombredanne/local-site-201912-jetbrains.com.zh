删除问题

删除YouTrack中的问题绝非易事。每个问题都有可能包含大量信息，这些信息丢失后将无法轻易恢复。通常，我们建议您尽可能避免删除问题。以下是删除问题的几种替代方法：

  • 如果问题重复了现有问题，请将其作为重复链接到现有问题。在这种情况下，“ 状态”字段的值设置为“ 重复” ，这是一个已解决的状态。未解决问题的搜索结果中排除了该问题，但保留了该问题中包含的所有信息。

  • 如果问题不再重要，并且您不想让任何人浪费时间处理此问题，请为“ 状态”字段选择一个适当的值来解决问题。诸如“无法重现” ， “无法修复” ，“ 未完成 ”和“ 过时”之类的默认状态指示此问题不需要其他工作。这有助于解释为什么在不从系统中删除某个问题之前不应该解决此问题。

  • 如果您在错误的项目中创建了问题，请不要删除它并创建另一个。只需在“ 项目”字段中更新值即可。

删除问题的最佳理由是因为您是错误创建的。如果您确信自己创建的问题不是必需的，则应该能够删除它而不会干扰其他人的工作。只是不要假设此规则适用于其他用户创建的问题。

要删除一个问题：

需要权限：读取问题，删除问题

  1. 在单个问题视图中打开一个问题。

  2. 单击工具栏中的删除图标。

  3. 单击对话框中的删除按钮以确认操作。

    • 该问题已从YouTrack中删除。

    • 该操作无法撤消。

    • 要还原问题，您必须还原包含该问题的备份副本，并将该问题从备份导入到YouTrack数据库。有关更多信息，请参阅还原已删除的问题 。

您还可以使用命令一次删除多个问题。执行此操作时应非常小心，因为您可以选择的问题比“ 问题”列表上显示的问题多，并且可以一次性删除它们。谨慎行事！

要删除多个问题：

需要权限：读取问题，删除问题

  1. 在“ 问题”列表中选择一个或多个问题。

  2. 打开“ 应用命令”对话框：

    • 单击问题工具栏中的“ 命令对话框”图标，然后选择“ 打开命令对话框” 。

    • 按Ctrl + Alt + J （Windows）或Control + Option + J （苹果系统）。

    • 只需开始输入即可。

  3. 在命令输入字段中，输入delete 。

  4. 单击“ 应用”按钮。

  5. 单击对话框中的“ 应用”按钮以确认操作。

    • 问题已从YouTrack中删除。

    • 该操作无法撤消。

    • 要还原这些问题，您必须还原包含问题的备份副本，并将其从备份导入到YouTrack数据库。有关更多信息，请参阅还原已删除的问题 。

恢复已删除的问题

删除问题后，只能从数据库备份中将其还原。

要要求将一个或多个已删除的问题还原到您的YouTrack InCloud实例，请提交支持请求 。我们将尽力找到问题并将其还原到您的实例。但是，我们不能保证已还原的问题包含丢失的所有数据。创建备份后，可能会应用了一些无法恢复的更新。

永久删除问题

为了维护数据库的完整性，将从YouTrack删除的问题保留在数据库中。如果要删除一批问题，可以将其永久删除并减小数据库的大小。

要永久删除问题，您需要从YouTrack删除它们所属的项目。此操作主要用于删除不再使用的旧项目。有关说明，请参阅“ 删除项目” 。

如果要删除较旧的问题，但仍要保持项目打开状态，请使用以下解决方法：

需要权限：创建项目，读取问题，更新问题，删除项目

  1. 创建一个新项目。

  2. 使用搜索查询返回要删除的所有问题。例如，搜索项目中六个月或更早之前已解决的问题。

  3. 使用命令将所有这些问题移至新项目。例如， move to 。

  4. 从YouTrack删除新项目。

限制删除问题权限

在许多问题跟踪器中，删除问题的功能仅限于系统和项目管理员。在YouTrack中，此权限被授予默认的开发者角色。这意味着项目团队的任何成员都可以删除项目中的任何问题。

对于某些组织，此方案可能太宽容。如果您发现用户在不应该删除问题时就删除了它们，而发出一则或两则严厉警告并没有阻止这种行为，请考虑以下一种或多种选择。系统管理员或具有创建角色或更新角色权限的其他用户可以进行这些修改。

  • 从默认的“ 开发人员”角色中删除“ 删除问题”权限。在这种情况下，删除问题的能力仅限于系统和项目管理员。

  • 创建一个不包含“ 删除问题”权限的自定义角色，然后根据每个项目将此角色分配给项目团队。这使项目管理员可以决定是否可以通过删除问题来信任其项目团队的成员。

  • 创建一个仅包含“ 删除问题”权限的自定义角色。然后，您可以按用户授予此权限，以删除特定项目中的问题。
