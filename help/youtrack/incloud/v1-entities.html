要求

“ Requirements”对象有两个目的。首先，它起着安全网的作用。它指定规则必须能够存在，实体才能正常工作。只要不满足一个或多个规则要求，工作流管理UI中就会显示相应的错误。在解决所有问题之前，不会执行该规则。其次，它作为参考。需求中的每个实体都被插入到“ context”对象中，因此您可以从与上下文相关的函数（例如“ action”函数）中引用实体。有两种类型的需求：项目范围和系统范围。项目范围内的要求包含必须附加到使用规则的每个项目的自定义字段的列表，以及每个自定义字段的值集中的必需值。系统范围的要求包含YouTrack中必须可用的其他实体的列表。这包括用户，组，项目，问题，标签，保存的搜索和问题链接类型。

功能


actionFunction

actionFunction（ctx，ctx.issue，ctx.currentUser，ctx.isSilentOperation）

取决于规则类型，不同事件将调用此函数：何时将更改应用于问题（更改规则），何时执行命令（操作规则）或根据预定义的计划（计划的规则）。对于每个相关问题，将分别调用此功能。

参量

名称

类型

描述

 Object

执行上下文。除了下面列出的参数外，上下文还包含您在“ 需求”中描述的对象。

ctx.issue

 Issue

当前的问题。

ctx.currentUser

 User

执行规则的用户。

ctx.isSilentOperation

 Boolean

当为“ true”时，该规则被触发以响应所应用的命令而无须通知。如果为false，则响应于正常应用的命令而触发规则。自2017.4.38771起可用。

例

操作：function（ctx）{console.log（'问题处理'+ ctx.issue.id）; }

 

GuardFunction

guardFunction（ctx，ctx.issue，ctx.currentUser）

调用此函数以确定是否可以将操作函数应用于问题。保护功能用于变更规则，操作规则以及状态机规则的值之间的转换。预定规则也支持保护功能，但是此规则类型包括具有类似用途的search属性。
