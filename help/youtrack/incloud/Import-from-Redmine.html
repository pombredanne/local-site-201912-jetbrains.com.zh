从Redmine导入

按照此页面上的说明，使用Python导入脚本从Redmine导入问题。此过程需要执行以下步骤：

  1. 设置您的本地环境 。

  2. （可选） 自定义Redmine数据库中的字段与YouTrack中的issue属性之间的映射 。

  3. 导入您的问题 。

导入细节

如果Redmine数据库包含对YouTrack中不存在的实体的引用，则会创建它们。您用于运行导入的用户帐户应具有创建项目，问题，用户，甚至更多的权限。我们建议您使用具有系统管理员角色的帐户来执行导入脚本。

专案

Python脚本可让您一次从源项目中导入所有问题。如果该项目尚不存在，则创建一个新项目。

  • 根据Redmine数据库中的值分配项目名称和ID。

  • 将执行导入脚本的用户设置为项目所有者。

用户数

几个问题属性，例如reporterName和Assignee存储对用户帐户的引用。Redmine中的用户帐户可以使用电子邮件地址登录，因此导入脚本首先检查帐户是否登录，然后再检查电子邮件地址。对于每个与YouTrack中现有用户的登录名或电子邮件都不匹配的帐户，将创建一个新的用户帐户。

所有新用户帐户均分配有随机生成的密码。要登录YouTrack，导入的用户必须单击登录页面上的重置密码链接并创建新密码。

导入数据中对用户的引用将映射到YouTrack中的现有帐户。如果找不到匹配项，则会创建一个新的用户帐户。

根据您的订阅计划，为导入的用户分配了许可证。对于超出您的每用户许可限制的用户，导入的用户被标记为禁止用户。

有关更多信息，请参见禁止用户帐户 。

团体

可以将Redmine用户帐户分配给组。对于从Redmine 2.1版及更高版本的导入，这些组将导入到YouTrack。组名称和成员资格来自源数据库。

的角色

在Redmine中，角色分配定义成员在项目中拥有的权限。对于从Redmine 2.2版及更高版本的导入，可以将这些角色导入到YouTrack。角色名称和分配来自源数据库。权限根据映射文件中的定义映射到YouTrack中的权限方案。

自定义字段

如果导入数据包含YouTrack中不存在的字段，则将创建自定义字段并将其添加到目标项目中。

  • 字段名称和类型是根据映射文件中的定义设置的。

  • 自定义字段的属性设置如下： autoAttached=true ， isPrivate=false ， defaultVisibility=false 。

栏位值

如果源文件包含现有字段的新值，则将它们添加到当前值集中。

工作项目

当您从Redmine中的时间跟踪模块导入数据时，存储为“ 花费的时间”的值将转换为在工作项中花费的时间 。YouTrack中“ 花费的时间”字段所使用的值是从添加到导入的工作项中所花费的时间得出的。

Redmine REST API不支持观察者和标签。这些实体不会导入到YouTrack。

设置环境

Python导入脚本支持“导入到YouTrack”。这些脚本基于YouTrack REST API构建。实际上，您实际上不需要使用Python进行任何编程，但是您确实需要安装Python和包含导入脚本的软件包。

设置环境：

  1. 下载并安装Python 。Python客户端库与Python 2.7+兼容。不支持Python 3版本。您可以选择自己喜欢的安装目录。

    macOS，CentOS，Red Hat Enterprise Linux（RHEL）和Ubuntu的最新版本均随附Python 2.7。如果使用任何这些操作系统，请继续下一步。

  2. 安装包含导入脚本的软件包。打开操作系统支持的命令行界面，然后输入以下命令：

    pip install youtrack-scripts

    • 导入脚本安装在您的本地环境中。

    • 的youtrack包含YouTrack REST API的Python客户端库的软件包会自动作为依赖项安装。

    如果您不想全局安装软件包，则只能使用以下命令为您的用户帐户安装软件包：

    pip install --user youtrack-scripts

导入脚本需要Redmine中的REST API支持。

要启用REST API支持：

  1. 在您的Redmine实例中，打开管理>设置>身份验证选项卡。

  2. 选中“ 启用REST Web服务”复选框。

自定义映射定义

下一步是将Redmine数据库中的字段映射到YouTrack中的属性。您可以使用redmine2youtrack脚本以生成示例映射文件。

仅在Redmine中自定义问题属性时，才需要自定义映射文件。要使用默认的映射定义，请跳过此过程并导入数据 。

生成映射文件：

  1. 打开操作系统支持的命令行界面。

  2. 如有必要，将当前目录更改为Python的安装目录。例如（Windows）：

    cd C:\Python27

  3. 输入以下命令：
    redmine2youtrack -g -m mapping.json

    • 的-g选项告诉脚本生成映射文件。

    • 的-m选项将覆盖映射文件的默认路径和文件名。如果不通过此选项，则文件将在当前目录中生成为mapping.json 。如果要指定其他位置和/或文件名，请使用目标目录的完整路径和/或所需名称为参数指定一个值。

    • 如果命令执行成功，则会在命令行界面中显示一条消息，其中包含生成的文件的位置。

要自定义映射定义：

  1. 打开生成的映射文件。

  2. 更新映射，以确保要从Redmine数据库导入的所有数据都映射到YouTrack中所需的值。如果文件包含一个或多个不想导入的字段，则可以从映射文件中删除引用。有关默认映射的说明，请参见默认映射 。

  3. 保存您的更改并关闭映射文件。

汇入资料

最后一步是运行从Redmine导入数据的命令。

导入数据：

  1. 打开操作系统支持的命令行界面。

  2. 如有必要，将当前目录更改为Python的安装目录。例如（Windows）：

    cd C:\Python27

  3. 输入以下命令：

    redmine2youtrack -m mapping.json -t token -u login - p password -a api_key r_user r_pass r_url y_url [project_id ...]

    将命令行参数替换为如下所述的值：

    参数

    描述

    mapping.json

    与-m选项，以指定自定义映射文件的完整路径和文件名。如果不通过此选项，则使用默认的映射文件。

    token

    当与-t选项，一个永久令牌，用于在YouTrack中验证您的管理员帐户。这是首选的身份验证方法。如果您使用此选项，请跳过-u和-p选项。要了解如何生成永久令牌，请参阅创建永久令牌 。

    或者，您可以使用-T选项，并为包含永久令牌的文件指定完整路径和文件名。

    login

    当与-u选项，YouTrack管理员帐户的登录名。您还必须为-p选项。

    password

    当与-p选项，YouTrack管理员帐户的密码。

    api_key

    当与-a选项，用于Redmine中的身份验证的API密钥。您可以在用户帐户页面上找到Redmine API密钥。

    如果您使用-a使用API密钥进行身份验证的选项，请跳过r_user和r_pass参数。

    r_user

    登录到Redmine的用户名。仅在您使用登录名/密码对在Redmine中进行身份验证时指定。

    r_pass

    用于登录Redmine的帐户密码。仅在您使用登录名/密码对在Redmine中进行身份验证时指定。

    r_url

    您的Redmine实例的URL。

    y_url

    目标YouTrack服务器的基本URL。对于YouTrack InCloud实例，您的基本URL包括结尾/youtrack 。例如： https://company.myjetbrains.com/youtrack

    y_user

    YouTrack管理员帐户的登录名。

    y_password

    YouTrack管理员帐户的密码。

    project_id

    您要导入的Redmine项目的ID。

  4. 检查您将问题导入的一个或多个项目，并验证数据是否正确显示。如果您对结果不满意，并想重新导入数据：

    • 删除导入期间创建的所有问题。如果将问题导入到新项目中，则只需删除该项目。

默认映射

Python客户端库包含一个默认映射文件，用于将问题从Redmine导入到YouTrack。该文件另存为mapping.py在里面youtrackutils/redmine存储库的子文件夹。

导入脚本引用默认映射文件。如果您创建自定义映射文件，请使用-m导入脚本中的选项，然后如上一节所述指定文件的完整路径和文件名。

自定义字段

的field_names字典将Redmine中的字段名称映射到YouTrack中的自定义字段。您可以更新本节中的映射，以将数据导入到特定的自定义字段。缺省映射文件包含以下字段的定义：

field_names = { 'id' : 'numberInProject', 'subject' : 'summary', 'author' : 'reporterName', 'status' : 'State', 'priority' : 'Priority', 'created_on' : 'created', 'updated_on' : 'updated', 'tracker' : 'Type', 'assigned_to' : 'Assignee', 'due_date' : 'Due Date', 'estimated_hours' : 'Estimation', 'category' : 'Subsystem', 'fixed_version' : 'Fix versions', 'redmine_id' : 'Redmine ID' }

编辑自定义字段映射时，请注意以下准则：

  • 映射的左侧引用Redmine中的字段名称。这些值对应于Redmine数据库中的字段引用。

  • 映射的右侧引用了YouTrack中的问题属性。基本属性使用在Import REST API中指定的属性名称。要映射到自定义字段，请使用自定义字段名称。

  • 如果您没有为Redmine中的字段指定与YouTrack中的目标字段不匹配的映射，则会创建一个自定义字段，其名称与Redmine中的名称相同。

自定义字段类型

的field_types字典将Redmine中的数据类型映射到YouTrack中的自定义字段类型。如果您已将字段添加到自定义字段的映射中，请在此字典中添加相应的条目，以定义如何将数据存储在YouTrack中。缺省映射文件包含以下字段类型的定义：

field_types = { 'Type' : 'enum[1]', 'State' : 'state[1]', 'Priority' : 'enum[1]', 'Assignee' : 'user[1]', 'Due Date' : 'date', 'Estimation' : 'period', 'Subsystem' : 'ownedField[1]', 'Fix versions' : 'version[*]', 'Redmine ID' : 'integer' }

编辑字段类型映射时，请注意以下准则：

  • 映射的左侧引用了YouTrack中自定义字段的名称。这些值必须与“ field_names字典。

  • 映射的右侧引用了YouTrack中的字段类型。有关有效类型的列表，请参阅Administration REST API 。

  • 如果目标字段已经存在于YouTrack中，则无需在此处添加目标字段。

  • 您不能更改现有字段的数据类型。例如，如果现有的“ 受理人”字段存储单个值（ user[1] ），您不能使用user[*]在映射文件中，并导入多个受让人。您必须首先更新该属性在YouTrack用户界面受让人字段，然后导入数据。

自定义字段的值

的conversion字典将Redmine字段的枚举值映射到自定义字段YouTrack的值。默认的映射文件包含问题State和Priority的映射。

conversion = { 'State': { 'Resolved' : 'Fixed', 'Closed' : 'Fixed', 'Rejected' : "Won't fix" }, 'Priority': { 'High' : 'Major', 'Low' : 'Minor', 'Urgent' : 'Critical', 'Immediate' : 'Show-stopper' } }

权限

的permissions字典将权限从Redmine映射到YouTrack中的权限。在YouTrack中没有相应权限的Redmine权限已被注释掉。

默认映射包括YouTrack中以下权限的定义：

permissions = { 'add_project' : 'CREATE_PROJECT', 'edit_project' : 'UPDATE_PROJECT', 'close_project' : 'DELETE_PROJECT', 'manage_members' : [ 'CREATE_USER', 'READ_USER', 'UPDATE_USER', 'CREATE_USERGROUP', 'READ_USERGROUP', 'UPDATE_USERGROUP' ], 'add_messages' : 'CREATE_COMMENT', 'edit_messages' : 'UPDATE_NOT_OWN_COMMENT', 'edit_own_messages' : 'UPDATE_COMMENT', 'delete_messages' : 'DELETE_NOT_OWN_COMMENT', 'delete_own_messages' : 'DELETE_COMMENT', 'view_issues' : 'READ_ISSUE', 'add_issues' : 'CREATE_ISSUE', 'edit_issues' : 'UPDATE_ISSUE', 'delete_issues' : 'DELETE_ISSUE', 'view_issue_watchers' : 'VIEW_WATCHERS', 'add_issue_watchers' : 'UPDATE_WATCHERS', 'delete_issue_watchers' : 'UPDATE_WATCHERS', 'log_time' : 'UPDATE_WORK_ITEM', 'view_time_entries' : 'READ_WORK_ITEM', 'edit_time_entries' : 'UPDATE_WORK_ITEM', 'edit_own_time_entries' : 'UPDATE_NOT_OWN_WORK_ITEM', }
