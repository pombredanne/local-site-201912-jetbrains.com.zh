订阅用例

通知是在YouTrack中管理问题的重要组成部分。在正确的时间获得正确的通知可以帮助您及时了解最新信息，并确切地知道何时需要采取行动。

常规通知方案包括一个选项，可以在问题描述或评论中用户名提到您的任何时间发送通知。@mention功能通常用于吸引您想参与讨论或解决问题的用户的注意力。您始终可以禁用此设置，并选择退出@mention通知，但是您故意忽略了需要输入内容的其他用户。

当然，您想使您的电子邮件收件箱尽可能避免垃圾邮件。来自YouTrack的过多通知会迅速堆积。在此页面上，我们提供了一些订阅用例，可帮助您仅获取所需的通知。

订阅您关注的问题的更新

默认的通知方案包括对Star标签问题的订阅。基本上，您可以通过此订阅订阅要观看的任何问题的通知。为了充分利用此订阅，您应该确切了解其工作方式。

  • 您可以将Star标记手动添加到要观看的任何问题。此选项在所有问题视图中均可见-从“ 问题”列表，单个问题视图和敏捷板上的查看模式中。

  • 其他用户可以将您添加为观察者。的star和watcher命令将Star标记添加到指定用户的问题中。单个问题视图中的“ 监视者”列表提供了将该命令应用于当前问题的快捷方式。此功能使其他用户可以将您的注意力吸引到问题上，而无需在评论中提及您。

  • 通过常规通知方案中的“ 添加星标”设置，您可以自动将星标添加到问题中。在这里，您可以选择在投票时添加此标签，向其中添加评论或更新任何问题。

  • 您可以从不再需要观看的任何问题中手动删除Star标签。

管理您要观看的问题列表有助于您充分利用此订阅。确保您只关注对您真正重要的问题。要查看您当前正在观看的所有问题，请打开“ 问题”列表，然后在侧栏中选择“ 星号”标签。

既然您已经掌握了如何观察问题的方法，那么让我们仔细看看您的订阅选项。此订阅的默认通知方案发送所有发布事件的通知。如果那是您喜欢的方式，那就顺其自然。但是，您可以通过禁用不太重要的事件来降低噪音。

在这里，我们已经修改了此订阅的默认通知方案。使用此设置，仅在将注释添加到您正在观看的问题或解决该问题时，您才会收到通知。您还可以启用“ 解决时取消标记”选项来删除星形标记，并在解决问题后停止观看。

使用个人标签触发通知

个人标签是一种很好的方法，可让您随时关注阻碍您工作的问题或解决问题后需要采取行动的问题。您可以像添加Star标签一样将这些标签添加到问题中，并应用自定义通知方案。假设您不需要跟踪问题中的每个更改-您只想在问题解决后收到通知，以便您可以开始工作。

这是您的操作方式：

  1. 转到您的个人资料，然后选择“ 标签和保存的搜索”标签。

  2. 单击新建标签或保存的搜索按钮。

  3. 在“ 新建标签”对话框中，配置标签的设置。对于个人标签，请将“ 可以查看并使用”设置为“ 我” 。

  4. 在“ 订阅”部分中，选择“已解决问题”事件。

  5. 单击创建按钮。

做完了现在，您可以将此标签添加到任何在完成后触发后续任务的问题上。解决此标签的任何问题时，您会收到通知。启用“ 解决问题后删除标签”选项，您甚至不必自己清除标签。

通过共享标签关注团队中的活动

我们的许多团队都使用标签临时支持业务逻辑。我们不是在State字段中添加新值，而是添加标签。例如，这些标签可以标记需要UX团队输入的问题，而无需直接重新分配问题。

假设您是该UX小组的成员，并且希望在将此标签添加到问题中时收到通知。在此示例中，我们假设标签是由您的团队负责人创建的，并且已与您的团队共享。

  1. 转到您的个人资料，然后选择“ 通知”标签。

  2. 在“ 订阅”部分中，单击“ 新建订阅”按钮。

  3. 在“ 新建订阅”对话框中，从下拉列表中选择标签。使用输入字段来过滤标签和保存的搜索列表。

  4. 选择标记事件。

  5. 单击保存按钮。

现在，只要您团队中的任何成员将此标签添加到问题中，您都会收到通知。完成设计后，只需删除标签。

获取与保存的搜索匹配的任何问题的更新

当标签不够用时，我们需要更深入地研究。使用保存的搜索，您可以定义触发通知必须满足的非常特定的条件。通过订阅与保存的搜索相匹配的问题，您可以进行广泛的搜索并观看所有内容，或者在没有偏见的情况下进行区分。

您可以创建个人保存的搜索，选择要自己解决的问题，或者与项目团队共享保存的搜索并发出警报。

所有的通知！

  1. 转到您的个人资料，然后选择“ 标签和保存的搜索”标签。

  2. 单击新建标签或保存的搜索按钮。

  3. 在“ 新建标签”对话框中，单击“ 保存的搜索”按钮。

  4. 配置保存的搜索的设置。

  5. 在“ 订阅”部分中，选择所有问题事件。

  6. 单击创建按钮。

仅针对您的错误

假设您希望开发人员收到项目中报告的每个错误的通知。您可以选择所有错误并将其添加为观察者，但这太麻烦了。您希望他们在问题创建后立即关注。这是另一种方式：

  1. 转到您的个人资料，然后选择“ 标签和保存的搜索”标签。

  2. 单击新建标签或保存的搜索按钮。

  3. 在“ 新建标签”对话框中，单击“ 保存的搜索”按钮。

  4. 配置保存的搜索的设置。

    • 对于查询设置，请使用in: Type: bug 。

    • 在“ 可以查看并使用”下拉列表中选择包含开发人员的组。

  5. 保持“ 订阅”选项不变 。你是老板！

  6. 单击创建按钮。

在这里，您有两个选择。您可以向开发人员发送电子邮件，并请他们订阅此保存的搜索。一点信任会走很长一段路。但是，如果您拥有“ 不拥有个人资料”和“ 更新不拥有个人资料”权限，则可以设置订阅并省去麻烦。

  1. 打开您的开发人员之一的用户配置文件。

  2. 选择“ 通知”选项卡。

  3. 在“ 订阅”部分中，单击“ 新建订阅”按钮。

  4. 在“ 新建订阅”对话框中，从下拉列表中选择保存的搜索。使用输入字段来过滤标签和保存的搜索列表。

  5. 选择“ 创建问题”事件。

  6. 单击保存按钮。

  7. 重复此过程，直到所有开发人员都订阅了这些通知。

在那里，您拥有了！当然，您的开发人员可以随时禁用这些通知。他们也可能会讨厌您的顶级管理风格，但是完成任务的一切对吗？

红色警报

因此，您是收件箱零人。您希望收到的通知尽可能少，通常希望尽快删除收到的消息。但是，您负责数据安全。当项目中报告了安全问题时，您需要快速采取行动。次要问题可以等待，但是您需要立即查看具有较高优先级或更高优先级的问题。保存搜索以营救！

  1. 转到您的个人资料，然后选择“ 标签和保存的搜索”标签。

  2. 单击新建标签或保存的搜索按钮。

  3. 在“ 新建标签”对话框中，单击“ 保存的搜索”按钮。

  4. 配置保存的搜索的设置。

    • 对于查询设置，请使用in: Type: {Security Problem} Priority: Major, Critical, Show-stopper 。

  5. 展开订阅选项。

  6. 选择创建的事件。

  7. 单击创建按钮。

现在，您将收到分配给优先级为Major ， Critical或Show-stopper的Security Problem类型的每个问题的通知。通知全天候发送，因此您白天或黑夜都会收到警报。收到通知并意识到问题所在后，您可以将邮件从收件箱移到垃圾箱。
