状态机

静态stateMachine（ruleProperties，ruleProperties.title，ruleProperties.fieldName，ruleProperties.stateFieldName，ruleProperties.states，ruleProperties.defaultMachine，ruleProperties.typeFieldName，ruleProperties.alternativeMachines，ruleProperties.requirements）

创建状态机规则的声明。状态机对自定义字段中的值之间的转换施加了限制。当自定义字段设置为一个值，从一个值更改或从两个特定值转换时，您可以执行操作。此方法返回的对象通常被导出到`rule`属性，否则将不被视为规则。

参量

名称

类型

描述

ruleProperties

 Object

定义规则属性的JSON对象。

ruleProperties.title

 string

规则的可读名称。显示在YouTrack的管理界面中。

ruleProperties.fieldName

 string

由状态机规则管理的字段的名称。声明fieldName或stateFieldName，而不是两者。

ruleProperties.stateFieldName

 string

ruleProperties.fieldName的别名，用于为每个问题类型构建状态机。当同时声明stateFieldName和fieldName时，将引发异常。

ruleProperties.states

 Object

自定义字段的值列表以及它们之间的可能转换。声明状态或defaultStateMachine，而不是两者。

ruleProperties.defaultMachine

 Object

ruleProperties.states的别名，用于为每个问题类型构建状态机。当同时声明defaultMachine和状态时，将引发异常。

ruleProperties.typeFieldName

 string

定义哪个状态机应用于托管字段的字段名称。

ruleProperties.alternativeMachines

 Object

一个对象，其中包含一个或多个适用于不同类型问题的状态机的定义。对象键是ruleProperties.typeFieldName定义的字段的可能值。对象值具有与示例中的“状态”所示相同的结构。当指定ruleProperties.typeFieldName参数时，此参数是必需的。

ruleProperties.requirements

 Requirements

必须存在的一组实体，脚本才能按预期工作。

返回值

类型

描述

Object

规则的对象表示。

例

var实体= require（'@ jetbrains / youtrack-scripting-api / entities'）; Exports.rule =实体。Issue.stateMachine（{标题：“状态机”，fieldName：“状态”，状态：{打开：{初始：true，转换：{开始：{targetState：“进行中”}}}}，“进行中” ：{onEnter：function（ctx）{ctx.issue.fields。受让人= ctx.currentUser; }，过渡：{修订：{targetState：“已固定”}，重新打开：{targetState：“开放”}}}}，已修复：{过渡：{}}}，要求：{受让人：{类型：实体。User.fieldType}}}）;
