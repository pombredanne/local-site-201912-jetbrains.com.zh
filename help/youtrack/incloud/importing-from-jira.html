YouTrack InCloud 2019.2帮助

从吉拉进口

通过YouTrack，您可以连接到Jira实例。通过此集成，您可以执行以下任务：

  • 从Jira导入现有项目，用户，组和组成员。

  • 更新以前导入的项目，以将更改应用于在Jira中进行的问题。自上次导入以来，在Jira项目中应用于问题的新问题和更改已添加到YouTrack。

从具有配置的安全级别的Jira导入的问题的可见性设置被设置为Project Admins组。

此集成还导入存储在Jira实例中的用户帐户。在Jira中为用户创建，更新，添加注释并为其分配问题的用户的相同值将设置为YouTrack中的相应用户帐户。

Jira问题中对用户的引用被映射到YouTrack中的现有帐户。如果找不到匹配项，则会创建一个新的用户帐户。

根据您的订阅计划，为导入的用户分配了许可证。对于超出您的每用户许可限制的用户，导入的用户被标记为禁止用户。

有关更多信息，请参见禁止用户帐户 。

从Jira导入的用户帐户包含来自Jira实例的登录名和密码。要使用户使用其Jira凭据登录到YouTrack，请启用Jira身份验证模块。有关更多信息，请参见Atlassian Jira Auth Module 。

先决条件

将项目从Jira导入到YouTrack之前，请检查以下要求：

  • 您要导入的项目是在Jira版本5或更高版本中创建的。

  • 您的Jira实例允许远程API调用。

  • 您在Jira中的用户帐户具有管理权限。如果使用未分配管理员角色的帐户连接到Jira实例，则无法导入组和相关数据。

连接到Jira实例

连接到Jira实例时，可以将整个项目从Jira导入到YouTrack。导入包括分配给项目的所有问题以及有权访问项目，项目角色，组和组成员身份的用户的用户帐户。

要连接到Jira实例：

需要权限：低级管理员写

  1. 在“ 管理”菜单的“ 集成”部分中，选择“ 从Jira导入” 。

  2. 输入以下设置的值：

    设置

    描述

    基本网址

    输入Jira实例的URL。

    登录

    输入用于登录Jira实例的帐户的用户名。该帐户必须在Jira中具有管理特权，才能导入项目，用户，组和相关数据。

    • 对于Jira安装，请输入用户名。不要输入电子邮件地址。

    • 对于Jira Cloud，您可以使用电子邮件地址作为登录名。

    密码

    输入用于登录Jira实例的帐户的密码。

    对于Jira Cloud，为您的Atlassian帐户创建一个API令牌，然后将其粘贴到“ 密码”设置的输入字段中。

    SSL密钥

    选择一个可选的SSL密钥以与Jira实例建立安全连接。

  3. 单击“ 连接到Jira”按钮。

    • YouTrack使用指定的凭据连接到Jira实例。

    • Jira实例的URL已添加到集成设置页面。

  4. 选择要导入到YouTrack的项目。要导入所有项目，请保留默认的“ 所有可用项目”设置。

    当您的用户帐户对从Jira导入的项目具有有限的访问权限时，将在“ 项目”设置下显示一个下拉菜单。展开此菜单以查看您的访问限制如何影响导入到YouTrack的数据。

    有关更多信息，请参见访问信息 。

  5. 要启用自动导入，请选择“ 连续导入”选项。启用此选项后，YouTrack每10分钟检查一次Jira实例中所选项目中的更改。自上次导入以来添加或更新的问题将在YouTrack中处理和更新。

  6. 单击导入问题按钮。您也可以单击导入单个问题链接以验证与Jira实例的连接。

    • 所选项目将导入到YouTrack。

    • 如果用于连接Jira实例的用户帐户具有足够的权限，则还将导入用户，组和相关数据。

    • 上次导入的日期显示在集成设置页面上。

    • 下载链接使您可以下载导入日志。使用此选项可以查看和调查导入期间发生的错误。

访问信息

要从Jira项目中导入所有数据，您的用户帐户必须具有在YouTrack中更新相应项目及其问题的权限。

当您的用户帐户对从Jira导入的项目具有有限的访问权限时，将在“ 项目”设置下显示一个下拉菜单。

展开此菜单以查看您的访问限制如何影响导入到YouTrack的数据。选择每个项目以查看适用的限制。以下条件会影响如何将数据导入YouTrack：

条件

局限性

创建组以表示Jira角色，但不导入角色演员。

您的帐户无权浏览Jira中的项目。

这些项目的问题在导入期间会被跳过。

您的帐户无权在选定的Jira项目中创建问题。

定制字段的值集仅包含一个或多个Jira问题中使用的值。

您的帐户无权查看所选Jira项目中的选民和观察员。

未导入问题投票者。

您的帐户无权管理所选Jira项目中的观察者列表。

导入选项

初始导入后，可以在“集成设置”页面上使用以下控件：

控制

描述

编辑设置

在编辑模式下打开集成设置页面。使用此选项连接到其他Jira实例或更新登录凭证。

导入更改

立即导入之后，立即导入在选定的Jira项目中所做的任何更改。

清除导入

删除当前的集成设置。附加到导入的问题的Jira实体ID也将被删除。从连接的Jira实例导入的项目，问题，用户和组不受影响。如果您清除导入设置并连接到相同的Jira实例，则会使用新的问题ID将问题重新导入到现有项目中。
