更新问题自定义字段

用例：

使用REST API更新问题自定义字段的值。

请注意，此示例对YouTrack 2019.1和更高版本有效。在以前的版本中，尽管原理相同，但是查询参数和实体属性可能不同。此外，2019.1中引入了许多功能，例如按名称命名实体，但以前的版本中不支持。

摘要

要更新问题自定义字段的值，请执行以下操作：

  1. 使用更新问题请求：

    POST / api / issues / {issueID}？[字段]

  2. 在里面fields request参数指定您要获得响应的字段。例如，字段ID和全值信息。这将帮助您确认所做的更改已应用。

  3. 在请求正文中指定id要么name ， $type并为每个需要更新的问题自定义字段设置json值。

一步步

  1. 要更新问题自定义字段，请发送POST请求目标问题。您可以使用问题ID或问题的实体ID来引用问题。

    POST / api / issues / {issueID}？[字段]

  2. 在里面fields request参数指定您要获得响应的返回实体的属性。例如， id ， name自定义字段及其所有属性的定义value 。这将有助于验证您的更改已应用。

  3. 在请求正文中指定id要么name ， $type并为每个需要更新的问题自定义字段设置json值。

    要形成值json，您需要获取要发布的值（bundleElement）的ID。为了那个原因：

    • 获取相应的项目自定义字段：

      GET / api / admin / projects / {projectID} / customFields / {fieldID}

    • 得到bundle从项目自定义字段的受支持属性中：

      GET / api / admin / projects / {projectID} / customFields / {fieldID}？fields = bundle（值（id，name，$ type），aggregatedUsers（id，name，$ type））
