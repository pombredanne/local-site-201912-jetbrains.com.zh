YouTrack InCloud 2019.2帮助

系统管理员快速入门指南

如果您负责管理新的YouTrack实例，则以下列表可能会有所帮助。这些主题按通常执行每个页面上描述的任务的顺序列出。

设置新的YouTrack服务器

刚开始使用时，您需要确保以所需的方式设置YouTrack，并且需要处理问题和管理项目的人员具有适当的访问级别。使用此列表中的主题指导自己完成此过程。

话题

描述

全局设置

检查您的YouTrack服务器的全局设置。使服务对其他用户可用之前，应配置几个选项。

在“ 全局设置”页面上，您可以：

  • 管理您的YouTrack许可证。

  • 设置默认时区。

  • 为存储日期值的字段设置视觉表示。

  • 更改系统语言。

迁移到YouTrack

如果要迁移到YouTrack，请了解如何从另一个问题跟踪器导入问题。

的角色

YouTrack提供了一些预定义的默认角色来管理用户权限。熟悉这些角色以及为用户分配项目中的角色时为用户启用的访问权限。

默认角色为不同类型的用户提供标准的访问级别。您可以添加和删除默认角色的权限，也可以创建自己的权限。

默认的开发人员角色启用了团队选项。被添加到分配了该角色的项目团队的用户在项目中被授予该角色。默认的项目管理员角色已分配给项目所有者。有关更多信息，请参见管理项目团队 。

团体

检查您的组并验证为其分配了哪些角色。如果要向一组用户分配一组特定的权限，请创建一个组并授予一个或多个角色，以授予对所需权限集的访问权限。请注意任何启用了自动加入选项的组。新用户将自动添加到这些组中，并继承分配给这些组的角色。

从另一个问题跟踪器导入问题时，将自动创建分配给这些问题的项目，以及每个项目团队的组一样。

这些组的成员被分配了默认的Developer角色。

用户数

将用户帐户添加到系统。您可以使用多个选项在YouTrack中创建用户帐户。

  • 手动创建用户帐户。
  • 邀请新用户注册。
  • 让用户注册自己的帐户。
  • 启用使用来自外部身份验证提供程序的凭据登录。

当您让用户注册自己的帐户或使用外部身份验证提供程序登录时，请使用“ 自动加入组”选项自动为用户授予一组特定的权限。

专案

当您从另一个问题跟踪器导入问题时，将自动创建分配给这些问题的项目。如果希望其他用户管理这些项目，请编辑每个项目并选择分配给该项目所有者的用户。

默认情况下，授予“ 项目管理员”角色“ 创建项目”权限。该权限授予用户在全局范围内创建项目的能力。如果要让用户创建和管理自己的项目，只需将这些用户添加到分配了Project Admin角色的组中，或直接在Global项目中将角色授予该用户。

自定义和扩展YouTrack

完成基本设置后，您可以在全局级别上以管理员身份配置一些其他设置和功能。

话题

描述

时间跟踪

默认时间跟踪设置从星期一到星期五使用五个8小时工作日。如果您的组织使用其他计划，则可以修改YouTrack服务器的默认设置。

您还可以将工作项类型添加到系统中。这些工作项类型可用于所有启用了时间跟踪的项目。

项目经理可以针对每个项目激活时间跟踪。

通知模板

如果要让YouTrack发送的消息遵循您的品牌标识，则可以自定义YouTrack通过电子邮件和Jabber发送的消息。

项目经理还可以基于每个项目自定义通知模板。

工作流程

通过脚本化工作流程来自定义和自动化YouTrack处理问题的方式。您可以编写自己的工作流程，或从GitHub中的存储库下载自定义工作流程。

具有低级管理员权限的任何人都可以更新YouTrack系统的工作流程。

整合方式

设置标准集成以将YouTrack与外部服务连接。

  • 同步已导入到YouTrack的项目，以检索在Jira中所做的更改 。

  • 从特定地址获取和处理电子邮件 。发送到邮箱的每封电子邮件都会在YouTrack中转换为问题。

  • 将YouTrack连接到TeamCity服务器 。

  • 将YouTrack连接到GitHub，GitLab或Bitbucket存储库 。

  • 在YouTrack和连接的Zendesk实例之间共享和同步票证 。

在“ 管理”菜单的“ 集成”部分下启用并配置这些集成。

外部整合

几个第三方服务使您可以从外部应用程序连接到YouTrack。如果您是组织中任何这些工具的管理员，则可以将它们与YouTrack集成以提高生产率。

  • 连接到测试管理工具 。PractiTest，TestLink，TestLodge和TestRail都支持与YouTrack集成。

  • 安装一个插件，使您可以插入问题的链接并在Confluence中显示问题列表 。

  • 建立连接并将通知发布到Slack 。
