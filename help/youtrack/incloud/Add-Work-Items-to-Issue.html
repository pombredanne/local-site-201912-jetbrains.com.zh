YouTrack InCloud 2019.2帮助

添加工作项

在项目中启用时间跟踪后，可以将工作项添加到问题中。您可以使用此功能记录在问题上花费的时间并记录所执行工作的类型。

花费时间计算

在每个工作项中输入的时间量将添加到存储以下内容的字段中： period类型。存储此数据的实际字段是在项目级别定义的。该字段记录为该事件中的所有工作项目报告的总时间。您不能手动更新此值-只能通过添加或编辑工作项。

剩余时间和逾期估算

将记录为花费时间的数量与另一个存储的值进行比较period存储估计值的类型字段。存储估算值的字段在进度指示器中显示此计算。不超过原始估算的花费时间以绿色绘制。缺少的一块饼表示剩余的时间量。当您将指针移到进度指示器上时，剩余的时间和估计会显示在工具提示中。

当所花费的时间超过原始估计时，进度指示器将显示为红色圆圈。工具提示显示的花费时间超过了原始估算值。

任务和子任务之间的同步

对于作为其他问题的父项链接的问题，YouTrack会自动将估计和花费的时间计算为添加到每个子任务的值的总和。

  • 如果您将工作项添加到父任务，则父任务中的花费时间包括来自添加到父任务的工作项的花费时间及其每个子任务的总花费时间。因此，我们通常建议您避免将工作项添加到父任务中。

  • 如果更改父任务的估计，则父任务及其子任务之间的同步将中断。要恢复同步并自动计算父任务的估计，请将父任务中的估计值设置为其每个子任务的总估计。

将工作项添加到问题

如果在项目中启用了时间跟踪，则活动流中将显示花费的时间。启用“ 花费的时间”过滤器后，这些事件可见。

要将工作项添加到问题中：

需要权限：读取问题，更新问题，创建工作项

  1. 在单个问题视图中打开问题。

  2. 单击输入字段下方的添加花费的时间链接以发表评论。

    • 注释的输入字段已替换为用于增加花费时间的控件。

  3. 填写表格中的字段。使用以下信息作为指导：

    • 您的用户帐户将自动设置为工作作者。

      如果您在当前项目中具有“ 创建自己的工作项”权限，则可以将另一个用户设置为工作作者。要报告代表另一个用户的时间，请从列表中选择另一个用户。

    • 如果为空，则为工作项目分配当前日期。要更改工作日期，请从日历中选择另一个日期。

    • 花费的时间记录为一系列整数，后跟代表时间单位的字母。例如，您可以输入90m要么1h30m 。根据YouTrack配置文件中的“ 周期”格式设置，周期值会自动转换为时间单位。

    • 工作项类型是可选的。可用工作类型的列表是根据每个项目定义的。

      此值自动设置为您分配给最新工作项的工作项类型。

    • 工作项注释是可选的。

  4. 单击添加花费的时间按钮。

    • 工作项目已添加到问题中。

    • 将花费的时间添加到存储该值的自定义字段。

    • 用于估计总工作量的字段中的进度指示器将更新。

使用命令添加工作项

您可以使用命令来记录问题的工作，而不必打开它并手动输入工作项。您甚至可以将相似的工作项目同时添加到多个问题中。

要使用命令添加工作项：

需要权限：读取问题，更新问题，创建工作项

  1. 在“ 问题”列表中选择一个或多个问题。

  2. 开始输入work打开“ 应用命令”对话框。

  3. 输入以下格式的命令：

    work [work item type] [date] [time period] [description]

    使用以下参数设置工作项的值：

    参数

    条件

    描述

    [工作项目类型]

    可选的

    设置工作项类型。

    [日期]

    可选的

    用格式设置日期yyyy-mm-dd 。如果跳过，则将当前日期分配给工作项目。

    [时间段]

    强制性的

    输入解决该问题所花费的时间。

    [描述]

    可选的

    输入所执行工作的描述。

  4. 输入注释（如果需要）。

  5. 单击“ 应用”按钮。

    • 工作项目已添加到问题中。

    • 将花费的时间添加到存储该值的自定义字段。

    • 用于估计总工作量的字段中的进度指示器将更新。

自动添加工作项

如果您管理项目并希望团队跟踪花费的时间，则可以将项目配置为自动记录工作项。

  1. 在项目中启用和配置时间跟踪。有关更多信息，请参阅启用时间跟踪 。

  2. 将正在进行的工作计时器默认工作流附加到您的项目。有关更多信息，请参见管理工作流 。
