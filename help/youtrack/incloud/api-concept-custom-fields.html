REST API中的自定义字段

此页面描述了YouTrack中自定义字段的层次结构以及CustomField，ProjectCustomField和IssueCustomField实体之间的区别。

CustomField

CustomField是一个定义通用自定义字段的实体，该字段包含基本属性，例如“名称”，“ fieldType”和“ JSON模式”页面上描述的其他属性。请注意，“实例”属性包含不同项目中此字段的设置。

下面的示例演示如何获取自定义字段。默认情况下，YouTrack仅返回$type和id被请求实体的属性。要获取有关自定义字段的更多信息，我们在fields请求的参数。注意$type无论您是否明确指定属性，该属性都会作为响应。

样品要求

curl -X GET \'https://example.myjetbrains.com/youtrack/api/admin/customFieldSettings/customFields?fields=id,name,aliases,instances(id,project(id,name））'\ -H'接受：application / json'\ -H'授权：不记名perm：am9obi5kb2U =。UG9zdG1hbiBKb2huIERvZQ ==。jJe0eYhhkV271j1lCpfknNYOEakNk7'\ -H'Cache-Control：no-cache'\ -H'Content-Type：application / json'

样品响应体

[...{“ id”：“ 58-1”，“ name”：“ Priority”，“ $ type”：“ CustomField”，“ aliases”：null，“ instances”：[{“ id”：“ 92-1”， “ $ type”：“ EnumProjectCustomField”，“ project”：{“ id”：“ 0-0”，“ name”：“ Sample Project”}}，...]}，...{“ id”：“ 58-4”，“ name”：“受让人”，“ $ type”：“ CustomField”，“ aliases”：“用于，分配给”，“实例”：[{“ id”：“ 94-0“，” $ type“：” UserProjectCustomField“，” project“：{” id“：” 0-0“，” name“：” Sample Project“}}，...]}，...]

ProjectCustomField

ProjectCustomField实体包含的设置CustomField （“项目”）在特定项目（“项目”）中的显示（例如，可能值的集合（“捆绑”），字段是否可以为空（“ canBeEmpty”）等）。有关属性的完整列表，请参见JSON模式页面。

样品要求

curl -X GET \'https://example.myjetbrains.com/youtrack/api/admin/projects/0-0/customFields?fields=id,canBeEmpty,emptyFieldText,project(id,name),field(id,name） ）'\ -H'接受：应用程序/ json'\ -H'授权：承载权限：am9obi5kb2U =。UG9zdG1hbiBKb2huIERvZQ ==。jJe0eYhhkV271j1lCpfknNYOEakNk7'\ -H'Cache-Control：no-cache'\ -H'Content-Type：application / json'

IssueCustomField

的IssueCustomField实体包含一个值（“值”属性） ProjectCustomField （“ projectCustomField”）解决特定问题。该值可以是简单的值（字符串，整数，..），到现有实体的链接（到“受让人”字段的特定用户的链接）或值的集合（“受影响的版本”字段）。
