管理冲刺

如果您遵循标准的Scrum开发流程，则会定期将新的sprint添加到敏捷板上。要参加为当前sprint安排的活动，您的项目团队成员可以从标题的下拉列表中选择当前的sprint。

无论您如何顺利地管理流程，都需要进行一两个更改的时机。在创建sprint之后，YouTrack为您提供了多个选项来管理它们：

  • 如果需要延长期限，则可以编辑sprint 。

  • 如果sprint与您当前的工作不再相关，则可以将其存档 。

  • 如果计划的sprint后来被放弃，则可以将其删除 。

请按照本页上的说明进行操作，以使冲刺保持正常状态。

权限

如果sprint未链接到自定义字段的值，则不需要特殊权限。您只需要成为允许查看和使用板并编辑板设置的组的成员即可。将sprint链接到自定义字段中的值的面板需要其他权限。有关详细信息，请参阅在基于版本的Board上管理Sprint 。

编辑Sprint属性

Sprint没有一长串可使用的属性。他们只有一个名字，一个目标和一个时间表。在这些属性中，仅名称是必需的。

但是，这些属性都不是固定的。如果需要在创建sprint之后的任何时间更新这些值中的任何一个，则可以自由地编辑sprint属性。

将自动检测敏捷板当前处于活动状态的Sprint。没有设置可以让您显式选择将哪个sprint指定为当前sprint。

要了解有关YouTrack用于选择当前冲刺的标准的更多信息，请参见“当前冲刺” 。

编辑冲刺：

  1. 单击标题中的“ 敏捷板”链接。

  2. 在下拉菜单中选择您的敏捷板。

  3. 单击冲刺选择菜单。

  4. 单击要存档的Sprint旁边的编辑图标。

    仅当将指针移到列表中的项目上方时，编辑图标才会出现。

    • 将打开“ 编辑冲刺”对话框。

  5. 更新您要更改的属性。您可以编辑Sprint名称，目标和时间表。

  6. 点击应用更改按钮。

    • 您所做的更改将应用于冲刺。

存档冲刺

如果冲刺与您当前的工作不再相关，则可以将其存档。归档的Sprint出现在敏捷板上Sprint选择菜单的单独部分中。

已从命令和建议完成查询中的冲刺列表中删除已归档的冲刺。您仍然可以在搜索查询中引用存档的sprint，但是在下拉列表中不会看到建议的sprint。

存档冲刺：

  1. 单击标题中的“ 敏捷板”链接。

  2. 在下拉菜单中选择您的敏捷板。

  3. 单击冲刺选择菜单。

  4. 单击要存档的Sprint旁边的编辑图标。

    • 将打开“ 编辑冲刺”对话框。

  5. 单击存档按钮。

    • 冲刺已存档。

    • 冲刺将移至冲刺选择菜单中的“ 存档”部分。

    • 选中后， 已存档的单词将显示在sprint选择器中的sprint名称旁边。敏捷板页脚中显示了用于还原或删除sprint的其他选项。

恢复存档的Sprint

可以恢复已存档的所有Sprint。恢复后，将消除适用于存档sprint的副作用。

  • 冲刺再次出现在命令和查询完成的建议中。

  • 在主板上的Sprint选择菜单中，该Sprint与其他活动Sprint分组在一起。

例如，当您决定推迟下一个Sprint的开始日期并且已经存档了当前Sprint时，可以选择恢复Sprint。在这种情况下，您可以还原存档的sprint并更新其属性以延长sprint的期限。

要恢复冲刺：

  1. 单击标题中的“ 敏捷板”链接。

  2. 在下拉菜单中选择您的敏捷板。

  3. 单击冲刺选择菜单。

  4. 请执行以下任一操作：

    • 单击要还原的Sprint旁边的编辑图标，以打开“ 编辑存档的Sprint”对话框，然后单击“ 还原”按钮。

    • 选择存档的Sprint，然后单击敏捷板页脚中的Restore Sprint按钮。

删除冲刺

例如，如果您放弃了与特定冲刺相关的工作，则可以将其删除。删除冲刺将实体从数据库中完全删除。删除后：

  • 冲刺不再适用于命令和搜索查询。

  • 与问题相关联的引用也将被删除。

删除冲刺是一项永久性的操作，无法撤消。仅在绝对确定不需要保留分配了一系列问题的任何记录时，才应删除Sprint。

删除冲刺：

  1. 单击标题中的“ 敏捷板”链接。

  2. 在下拉菜单中选择您的敏捷板。

  3. 单击冲刺选择菜单。

  4. 请执行以下任一操作：

    • 对于活动的或已存档的Sprint ，请单击要删除的 Sprint旁边的编辑图标以打开“ 编辑存档的Sprint”对话框，然后单击“ 删除”按钮。

    • 对于已归档的Sprint，您可以选择已归档的Sprint，然后单击敏捷板页脚中的“ 删除”按钮。

在基于版本的板上管理Sprint

基于版本的电路板配置为将sprint链接到用于存储自定义字段的自定义字段的值version类型。可以应用于敏捷板上的sprint的许多更改会自动与链接的自定义字段中的一组值进行同步。更改Sprint的名称还会更新存储在链接字段中的值的名称。

将以下更改应用于冲刺时，链接字段中的相应值也会更新：

  • 更改sprint名称会更新链接字段中相应值的名称。

  • 当用户存档冲刺时，将在链接的自定义字段中启用对应值的“ 存档”设置。

  • 当用户删除冲刺时，该值将从链接字段的值集中删除。从存储已删除值的所有问题中清除该值。

要更新其他板类型的冲刺，您只需成为允许查看和使用板以及编辑板设置的组的成员即可。在基于版本的板上：

  • 您需要成为董事会的所有者。

  • 您必须具有在使用该字段的所有项目中更新链接字段的值集的权限。

这是因为对sprint的更改将应用于所有使用它的项目中的链接字段中存储的值。如果您的电路板配置为将sprint链接到自定义字段，则这些要求将显示在对话框中，您可以在其中管理sprint。

要了解有关基于版本的电路板的更多信息，请参阅链接Sprint与Fix Versions 。
