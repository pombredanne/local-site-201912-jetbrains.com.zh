Atlassian Jira Auth模块

该身份验证模块使用户可以使用其Jira帐户登录到YouTrack。

启用Jira身份验证

要允许用户使用其Jira帐户登录到YouTrack，请启用Jira身份验证模块。

要启用Jira身份验证：

需要权限：低级管理员写

  1. 在管理菜单的访问管理部分，选择身份验证模块 。

  2. 从“ 添加模块”下拉列表中，选择Atlassian Jira 。

    • 侧边栏中将打开“ 添加模块”对话框。

  3. 在“ 添加模块”对话框中，输入以下设置的值：

    领域

    描述

    名称

    输入身份验证模块的名称。

    服务器网址

    输入Jira实例的服务器地址。例如， https://mycompany.com/jira 。

    • Jira身份验证模块是自动创建，启用和配置的。

    • 认证模块页面显示Jira认证模块的设置。有关此页面上设置的更多信息，请参阅设置 。

    • 用户可以使用其Jira凭据登录到YouTrack。

测试连接

要验证身份验证模块已连接到您的Jira实例，请测试连接。

  1. 单击测试登录按钮。

  2. 在“ 测试身份验证”对话框中，输入存储在您的Jira实例中的用户的凭据：

    • 在登录字段中，输入domainusername 。

    • 在密码字段中，输入password 。

  3. 单击测试登录按钮。

    • YouTrack在Jira实例中搜索指定的用户帐户。如果找到用户，则显示成功通知。如果出现错误，请检查用户凭据和服务器URL。

设定值

将模块添加到YouTrack后，将自动配置设置。您不必更改这些设置。要配置用于定义YouTrack如何使用Jira凭据对待新用户帐户的选项，请参阅其他设置 。

领域

描述

类型

显示在YouTrack中启用了第三方身份验证的应用程序或服务的名称。

名称

存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。

服务器网址

将用于验证登录请求的URL存储在YouTrack中。

更改密码URI

存储用于管理Jira帐户的更改密码请求的页面的URL。

SSL密钥

选择一个SSL密钥以加密YouTrack和Jira服务之间的连接。有关在YouTrack中管理SSL密钥的更多信息，请参阅SSL密钥 。

其他设置

这些设置使您可以管理YouTrack帐户的创建和组成员身份，并减少空闲连接消耗的处理资源的损失。

选项

描述

用户创建

为使用存储在连接的授权服务中的帐户登录的未注册用户创建YouTrack帐户。YouTrack使用电子邮件地址来确定用户是否已有帐户。

自动加入群组

当用户使用存储在连接的授权服务中的帐户登录时，将用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。

连接超时

设置等待建立与授权服务的连接的时间。默认设置为5000毫秒（5秒）。

读取超时

设置等待从授权服务读取和检索用户配置文件数据的时间。默认设置为5000毫秒（5秒）。

审计

链接到YouTrack中的“ 审核事件”页面。在这里，您可以查看已应用于此身份验证模块的更改列表。
