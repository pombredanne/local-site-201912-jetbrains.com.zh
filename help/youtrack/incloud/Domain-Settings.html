域设置

本页描述您的YouTrack InCloud实例的域设置。这些设置在“ 管理”菜单的“ 服务器设置”部分中可用。

设置

描述

基本网址

存储您的YouTrack InCloud实例的基本URL。这是用于在网络浏览器中访问YouTrack的URL。

私钥

存储与您的自定义域的SSL证书对应的私钥。

证明书

存储您的自定义域的SSL证书。要存储信任证书链，请依次输入它们，每个证书都从新行开始。

使用自定义域名

如果您已经注册了一个域，或者计划获得一个域，则可以将YouTrack InCloud实例配置为使用自定义域名。而不是使用yourcompany.myjetbrains.com/youtrack ，您的YouTrack实例位于yourcompany.com/youtrack 。

当您将基本URL更改为使用自定义域时，YouTrack会自动生成SSL证书和相应的私钥。该证书由“ 我们加密”证书颁发机构（CA）签名。通过此证书，可以通过HTTPS安全访问YouTrack InCloud安装。

要为您的YouTrack InCloud实例使用新域名：

需要权限：低级管理员读，低级管理员写

  1. 如果您还没有自定义域，请在任何域名注册机构中注册您的域名。

  2. 在注册商的域设置中，添加一个指向您的YouTrack InCloud实例的CNAME记录。对CNAME使用以下格式： yourcompany.myjetbrains.com 。CNAME记录或规范名称记录是域名系统（DNS）中的一种资源记录，它指定域名是另一个规范域名的别名。例如，一条CNAME记录如下：

    yourcompany.com。CNAME yourcompany.myjetbrains.com。

    可以理解为： yourcompany.com是规范名称（CNAME）的别名yourcompany.myjetbrains.com 。

  3. 在YouTrack中，打开“ 管理”>“域设置”页面。

  4. 在“ 基本URL”输入字段中输入自定义域的URL 。

    基本网址必须包含/youtrack路径，例如https://yourcompany.com/youtrack

  5. 单击保存按钮。

    • 域设置已更新。

    • 大约15到20分钟后，CNAME记录将绑定到您的自定义域。当前绑定状态显示在“ 基本URL”输入字段下方。

    • 在进行CNAME绑定时，您可以继续使用YouTrack实例而不会受到干扰。

将您自己的SSL证书用于自定义域

如果您已经拥有由已知CA签名的SSL证书，并且希望使用它代替“让我们加密”颁发的证书，则可以在“ 域设置”页面上安装自己的证书。

要安装自己的SSL证书，请执行以下操作：

需要权限：低级管理员读，低级管理员写

  1. 以x.509 PEM格式下载SSL证书。

  2. 在YouTrack中，打开“ 管理”>“域设置”页面。

    不要导入证书的SSL证书页上您的自定义域。SSL证书页面存储请求获得对YouTrack服务器的授权访问权的第三方服务器的证书。

  3. 在“ 基本URL”输入字段中输入自定义域的URL 。

    为了安全访问， 基本URL应该以https://并且必须包括/youtrack路径，例如https://myyoutrack.com/youtrack

  4. 将您的SSL私钥和证书粘贴到相应的字段中。密钥和证书必须以PEM格式提供。如果您有多个证书，请依次输入它们，每个证书都从新行开始。

    • 您的自定义域被配置为使用SSL证书通过HTTPS传输数据。
