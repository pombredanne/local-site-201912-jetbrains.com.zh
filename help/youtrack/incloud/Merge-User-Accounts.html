合并用户帐号

YouTrack支持将多个用户帐户合并为一个。合并的结果是，所有角色，组成员身份和分配的问题都将转移到合并的用户帐户中。

如果已将任何合并和删除的用户帐户设置为受理人，记者或评论者，则这些值将替换为合并的用户帐户。

当您将具有较少权限的帐户与具有更多权限的帐户合并时，合并的用户帐户将获得更高级的权限。

当用户首次登录YouTrack时（例如，使用OpenID帐户或自注册帐户），YouTrack会检查是否存在具有相同电子邮件地址的帐户。如果找到现有帐户，则YouTrack会自动将新帐户合并到现有帐户中。有关更多信息，请参见＃自动合并用户帐户 。

有关合并用户帐户的重要说明

我们敦促您记住有关合并帐户的以下要点：

  • 当多个帐户合并在一起时，已删除帐户中的所有凭据都将转移到结果帐户中。这意味着用户仍然可以使用这些凭据（登录名，电子邮件，密码）登录到YouTrack。

  • 如果要阻止合并的用户帐户登录到您的跟踪，则可以使用以下选项：

    • 禁止合并的用户帐户 。

    • 从合并帐户的管理员配置文件中手动删除凭据：

  • 合并帐户的所有角色分配和组成员身份都会转移到结果帐户中。当您将具有较少权限的帐户与具有更多权限的帐户合并时，合并的用户帐户将获得更高级的权限。

  • 如果将任何合并和删除的用户帐户分配给项目中的特定角色，则将这些值替换为结果用户帐户。例如，在YouTrack项目中将用户设置为受让人，记者或评论者时。

手动合并相似的用户帐户

当找到同一用户的两个具有不同凭据的帐户时，可以手动合并它们。

此过程包括两个步骤：

  1. 查找两个或多个可能重复的用户帐户 。

  2. 合并帐目 。

查找重复的帐户：

需要权限：更新用户

  1. 打开“ 用户”页面。

  2. 单击单击查找重复项按钮。

  3. 选择以下选项之一：

    • 具有相同电子邮件和相似名称的用户...

    • 使用相同电子邮件的用户...

    • 用户至少有一些共同点...

    在前两种情况下，要匹配的主要条件是电子邮件地址。搜索结果中只会显示电子邮件地址完全相同的帐户。第一个选项仅查找具有相同电子邮件地址且也具有相似用户名的帐户。

    第三个选项查找具有相似用户名或相似电子邮件地址的所有帐户。搜索结果可能如下所示：

    搜索结果显示为相似用户帐户的可视集。要自动合并集合中的所有相似帐户，请单击“ 全部合并”按钮。否则，您可以选择查看搜索结果并分别合并每组相似的用户。

下一步是合并重复的帐户。

合并相似的用户帐户：

需要权限：更新用户，删除用户

  1. 选择要保留的帐户参数：用户登录名，全名和电子邮件地址。

  2. 如果要禁止合并的用户帐户，请选中“ 禁止”复选框。

  3. 单击合并按钮。

手动合并多个用户帐户

在某些情况下，您可能希望合并多个帐户，即使它们不是重复的也是如此。

手动合并多个帐户：

需要权限：更新用户，删除用户

  1. 在“ 用户”列表中，选择需要合并在一起的帐户，然后单击“ 合并”按钮。

  2. 在显示的对话框中，选择要保留的帐户参数：用户登录名，全名和电子邮件地址。

  3. 如果要禁止合并的用户帐户，请选中“ 禁止”复选框。

  4. 单击合并用户按钮。

自动合并用户帐户

在某些情况下，YouTrack会自动合并用户帐户。当用户创建具有OpenID或LDAP凭据的帐户时，将执行自动合并。如果YouTrack中存在一个帐户，并且该帐户具有与OpenID或LDAP帐户关联的相同电子邮件电子邮件地址，则这些帐户将被合并。

YouTrack不会自动合并由用户或管理员手动创建的帐户。在这种情况下，将有两个具有相同电子邮件地址的独立帐户，管理员可以手动合并它们。

让我们用示例说明自动合并过程的逻辑：

  1. 在YouTrack中，有一个用户登录名“ j.johnson”，电子邮件地址为“ john.johnson@company.com”。此帐户是从另一个错误跟踪器导入的。

  2. 导入帐户后，将为此YouTrack实例启用并配置LDAP集成。用户尝试使用具有相同电子邮件地址的LDAP帐户“ john.johnson”登录到YouTrack。

  3. 登录后，YouTrack会扫描用户群并找到现有帐户。

  4. 新帐户（LDAP中的“ john.johnson”）将自动与现有帐户（“ j.johnson”）合并。由于帐户是自动合并的，因此不会创建单独的用户帐户（'john.johnson'）。LDAP目录服务中用于此帐户的凭据（用户名和密码）已添加到现有帐户。

结果，用户可以使用“ j.johnson”和“ john.johnson”凭据登录。“ j.johnson”的原始帐户显示在可用用户列表中。
