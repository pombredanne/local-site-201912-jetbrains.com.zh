GitLab集成

请按照本页上的说明与gitlab.com ，自托管的GitLab社区版（CE）或自托管的GitLab企业版（EE）服务器上托管的VCS存储库集成。

GitLab集成启用了YouTrack中的以下功能：

  • 直接从提交注释或请求请求描述中将命令应用于YouTrack问题。有关更多信息，请参见《 VCS提交》中的“应用命令” 。

  • 在每个问题的活动流中跟踪与特定问题相关的提交。有关更多信息，请参见查看和编辑VCS更改 。

  • 在提交消息中添加指向YouTrack问题的链接。有关更多信息，请参见《 VCS提交中的链接问题》 。

在GitLab中不完全支持将YouTrack问题ID设置为提交消息中的链接的功能。在GitLab中启用此功能的Custom Issue Tracker服务不支持YouTrack所使用格式的问题URL。所有集成功能都可以在YouTrack端正确处理，但是从GitLab到YouTrack的链接设置不正确。

启用和配置GitLab集成时，您将定义YouTrack项目和GitLab存储库之间的映射。该操作也可以由项目管理员基于每个项目执行。有关更多信息，请参见与GitLab集成 。

作为管理员，您可以为YouTrack中的任何项目启用GitLab集成。

先决条件

  • 入站连接可访问YouTrack。具体来说，您需要确保您的网络不会阻止VCS服务器与YouTrack之间的连接。

  • 您用于连接到GitLab的帐户可以访问您要通过此集成管理的GitLab项目。这是其专用访问令牌用于身份验证的用户帐户。您可以使用在项目中至少具有“ 主”或“ 所有者”访问级别的帐户，也可以使用GitLab管理员帐户。

如果要与GitLab CE或GitLab EE安装集成，并想与服务器建立安全（HTTPS）连接，则可能需要将GitLab服务器的SSL证书导入YouTrack。

  • 如果您的GitLab服务器具有由知名证书颁发机构（CA）签名的有效证书，则JVM供应商可能已将根（CA）证书添加到了证书存储中。您应该能够连接到服务器而无需导入其SSL证书。

  • 如果服务器的证书是自签名的，则需要导入证书和公共密钥以建立安全连接。为了安全起见，仅当YouTrack和GitLab服务器都在专用计算机网络上运行时，才使用此选项。有关说明，请参阅SSL证书 。

在GitLab中启用YouTrack集成

您应该做的第一件事是在GitLab中启用YouTrack集成。这种格式将提交消息中对YouTrack问题的引用设置为指向YouTrack中目标问题的链接。适用以下限制：

  • GitLab 11.9.0及更高版本中支持此功能。

  • 集成仅识别以大写字母开头的问题前缀。无法识别以小写字母或数字开头的问题前缀。

要了解有关此集成的更多信息，请参阅GitLab文档 。

要启用YouTrack集成，请执行以下操作：

  1. 访问要与GitLab集成的项目。

  2. 从设置菜单中，选择集成 。

  3. 在页面的“ 项目服务”部分中，选择“ YouTrack”

  4. 指定以下设置的值：

    设置

    描述

    描述

    输入任何值。使用此设置可以指示您与之集成的YouTrack服务。

    项目网址

    输入您的YouTrack服务的基本URL。

    • 对于YouTrack InCloud实例，您的基本URL包括结尾/youtrack 。

    • 无法通过以项目名称结尾的URL访问YouTrack问题。请勿尝试将项目名称或ID添加到URL。

    问题网址

    输入您的基本网址，然后输入issue/:id 。

  5. 启用活动选项。

  6. 单击测试设置并保存更改按钮。

使用此配置，可以将在YouTrack中被视为问题的任何引用设置为链接。

  • 链接是根据为Issues url设置的模式形成的。

  • 的:id占位符将替换为问题ID，其中包括项目前缀。

这意味着即使您尚未在YouTrack中配置该项目的集成，您也可以在GitLab中将链接添加到YouTrack安装中任何项目中的任何问题。但是，只有在为YouTrack中的项目设置了GitLab集成的问题中，才会显示相关的VCS更改。要启用此集成功能，请按照以下部分中的说明继续进行设置。

连接到GitLab存储库

接下来，您需要在YouTrack中的项目与GitLab中的存储库之间建立连接。要连接GitLab，您需要生成并存储个人访问令牌。该令牌基于授予GitLab帐户的访问权限，向YouTrack授予对存储库的访问权限。

您可以使用单个访问令牌来设置多个集成。如果您还没有个人访问令牌，则可以使用YouTrack的直接链接在此设置过程中生成它。

要连接到GitLab存储库：

需要权限：更新项目

  1. 在YouTrack中打开“ 管理”>“ VCS集成”页面。

  2. 单击“ 新建VCS集成”按钮。

    • 将打开“ 新建VCS集成”对话框。

  3. 从YouTrack项目列表中，选择要与VCS存储库集成的项目的名称。

  4. 对于服务器类型 ，选择GitLab 。

  5. 将指向您的GitLab存储库的URL粘贴到“ 存储库URL”输入字段中。

  6. 将您的个人访问令牌粘贴到“ 个人访问令牌”输入字段中。如果您还没有访问令牌，请按照下列步骤操作：

    • 单击生成令牌链接以在GitLab中打开“ 个人访问令牌”页面。

    • 输入令牌的名称。

    • 设置令牌的范围 。VCS集成要求api ， read_user和read_repository 。

    • 单击创建个人访问令牌按钮。

    • 将令牌复制到剪贴板。

  7. 切换回YouTrack中的“ 新建VCS集成”对话框，并将令牌粘贴到“ 个人访问令牌”字段中。

  8. 单击保存按钮。

    • 您的YouTrack项目已与GitLab中选定的存储库集成。

    • 来自GitLab存储库的引用项目中问题的提交将显示在所引用问题的活动流中。

    • 侧边栏显示用于配置VCS集成的其他设置。

    要了解有关这些设置的更多信息，请参阅集成设置 。

进阶伺服器设定

如果无法使用“ 新建VCS集成”对话框上的基本设置建立与存储库的连接，请单击“ 显示高级服务器设置”链接。

仅在尚未与目标服务器建立VCS集成时，才需要输入这些设置的值。如果已经与服务器上的单个存储库进行了有效的集成，则可以添加与其他存储库的集成，而无需再次设置这些参数。

使用以下准则来设置这些设置的值：

设置

描述

网址

此设置有助于标识GitLab存储库的路径。如果您的GitLab CE或EE服务器的存储库位于https://host:port/SOME_CONTEXT_PATH ，YouTrack无法确定此上下文路径的结束位置以及到存储库的实际路径的开始位置。

要解决此问题，请输入GitLab服务器的基本URL以及指向存储库的上下文路径。例如：https://host:port/context_path

SSL密钥

如果您的服务器环境设置为需要HTTPS身份验证，请选择包含您的YouTrack服务器私钥的密钥库。尝试与GitLab建立连接时，此密钥标识您的YouTrack服务器。此设置仅用于支持内部网络连接所要求的HTTPS身份验证。

该列表仅显示已导入到YouTrack中的SSL密钥。要了解如何生成密钥库文件并将其上传到YouTrack，请参阅SSL密钥 。

整合设定

默认情况下，VCS集成处理由任何分支中的任何用户提交到存储库的更改。在YouTrack中有权访问该问题的任何用户都可以在问题活动流中查看这些更改。

如果您只想处理指定分支中特定用户的更改，或限制YouTrack中VCS更改的可见性，则可以自定义集成设置。使用以下设置来自定义集成：

设置

描述

资料库

显示集成版本控制系统中存储库的路径。

如果需要，可以在设置集成后编辑存储库的位置。有关说明，请参阅“ 编辑存储库设置” 。

YouTrack项目

设置VCS集成处于活动状态的项目。您可以切换集成以更新另一个项目中的问题，而不必添加新的集成。

提交者

限制更新提交给指定组成员的消息中的命令的问题的能力。来自非选定组成员的用户的VCS更改仍附加到相关问题，但是将忽略其提交中指定的任何命令。

监控分支

存储要监视更改的分支的名称。

  • 使用+包括一个分支。

  • 使用-排除分支。

  • 为了branch name ，请使用分支机构的全限定名称。例如， refs/heads/ 。

  • 使用*作为通配符。该占位符匹配字符串中的一个或多个字符。例如，要包括所有功能分支，请使用：

    +：裁判/负责人/功能/ *

  • 要监视所有分支，请将输入字段留空。

如果在连接到GitLab时输入为存储库URL的地址指向特定分支，则在建立连接时该分支会自动添加到受监视分支列表中。

解析提交问题评论

启用后，提交消息文本的特定部分将作为注释复制到问题。当您将部分提交消息作为注释复制到问题时，您可以触发@mention通知并将信息公开给无法访问VCS更改的用户。

此设置不会影响VCS更改中提交消息的显示方式。整个提交消息（包括命令和问题注释）始终显示为活动流中VCS更改记录的一部分。

您仅应在以下情况下启用此选项：

  • 您希望在提交消息中提及其他用户，并在将文本复制到问题注释时生成通知。

  • 您限制了VCS更改的可见性，并希望使与提交有关的信息作为注释对外部用户可见。

要了解有关YouTrack如何处理提交消息的更多信息，请参阅《 VCS提交》中的“应用命令” 。

VCS更改可见性

将VCS更改的可见性限制为YouTrack中的一个或多个用户组。不受限制时，任何有权阅读该问题的用户都可以看到VCS更改列表。

故障排除

条件 -对VCS提交中的问题的引用未显示为YouTrack中的VCS更改。

原因

解

集成的VCS中的Webhook不存在，被禁用或格式错误。

检查VCS存储库设置中的webhook。确保存在网络钩子并已将其启用。

如果您怀疑Webhook有问题，请在VCS的设置中将其删除或禁用，然后在YouTrack中设置新的VCS集成。

条件 -VCS提交中指定的命令不适用于YouTrack中的问题。

原因

解

提交对存储库所做的更改的用户不是集成设置中Committers组的成员。

将提交者添加到YouTrack中的指定组，或者修改集成设置中的选择。

提交对存储库的更改的用户无权更新所连接项目中的问题。

将这些用户添加到项目团队中，或授予他们在项目中的角色，该角色包括“ 读取问题”和“ 更新问题”权限。

YouTrack找不到与提交消息的作者匹配的用户帐户。

用户必须在YouTrack和GitLab中为其帐户使用相同的电子邮件地址，或者将存储在其GitLab配置文件中作为Name的值添加到其Hub帐户中的VCS用户名列表中。

有关更多信息，请参见匹配提交作者和YouTrack用户 。

条件 -您无法连接到GitLab存储库。请按照以下说明解决问题。

原因

解

使用个人访问令牌进行GitLab身份验证的用户无权访问GitLab中的任何项目。

访问您的GitLab项目的成员设置，然后将此用户添加到项目中。您还可以与该用户所属的组共享项目。

该用户不需要特定的访问级别- 来宾访问就足够了。

原因

解

外部服务不可用。

验证您的GitLab服务器正在运行。

连接被防火墙阻止。

打开YouTrack和GitLab服务器使用的防火墙中的端口。

GitLab服务器需要安全连接。

将您的GitLab服务器的证书导入到YouTrack中。有关说明，请参阅SSL证书 。
