有特定问题附件的操作

此资源使您可以处理特定问题中的附件。

资源资源
/ api / issues / {issueID} / attachments / {attachmentID}

退货实体

IssueAttachment 。有关实体属性的描述，请参见“ 支持的字段”部分。

支持的方法

  • GET ： 阅读特定的IssueAttachment 。

  • POST ： 更新特定的IssueAttachment 。

  • DELETE ： 删除特定的IssueAttachment 。

IssueAttachment属性

表示附加到问题或评论的文件。

下表描述了IssueAttachment实体。

  • 要在服务器的响应中接收属性，请在request参数中明确指定它fields 。

  • 要更新属性，请在POST请求的正文中提供它。

领域

类型

描述

名称

串

文件名。 Can be null 。

作者

 用户

将文件附加到问题的用户。 Read-only 。 Can be null 。

被创造

长

创建附件作为时间戳的日期和时间。 Read-only 。

更新

长

附件的最新日期和时间作为时间戳记。 Read-only 。

尺寸

长

附件的大小（以字节为单位）。 Read-only 。

延期

串

定义文件类型的扩展名。 Read-only 。 Can be null 。

字符集

串

文件的字符集。 Read-only 。 Can be null 。

mimeType

串

文件的MIME类型。 Read-only 。 Can be null 。

元数据

串

图像文件的尺寸。对于图像文件，该值为rw=&rh= 。对于非图像文件，该值为空。 Read-only 。 Can be null 。

草案

布尔型

如果为true，则附件尚未发布，否则为false。 Read-only 。

已移除

布尔型

如果为true，则认为已删除附件。 Read-only 。

base64Content

串

使用以下语法表示附件的数据URI：

数据：[ ] [; base64]，

例如：

“ data：image / png; base64，iVBORw0KGgoAAAANSUhEUgAAAAUAAAAFCAYAAACNbyblAAAAHElEQVQI12P4 // 8 / w38GIAXDIBKE0DHxgljNBAAO9TXL0Y4OHwAAAABJRU5ErkJggg ==”
Can be null 。

网址

串

文件的URL。 Read-only 。 Can be null 。

能见度

 能见度

访问附件的设置。 Can be null 。

问题

 问题

文件附加到的问题。 Read-only 。 Can be null 。

评论

 问题评论

文件附加到的注释。退货null ，如果文件直接附加到问题上。 Read-only 。

thumbnailURL

串

附件缩略图的URL。 Read-only 。 Can be null 。

阅读特定的问题附件

阅读该问题的特定附件。

所需权限

  • 要求对附件所属的问题具有读取权限。

  • 如果将附件添加到评论中，则还需要访问该评论。

  • 如果附件可见性仅限于特定的组或用户，则只有该集合和附件作者中的用户才能访问附件。覆盖可见性限制权限允许覆盖这些限制。

请求语法

GET / api / issues / {issueID} / attachments / {attachmentID}？{字段}

{attachmentID}

附件的数据库ID。

请求参数

参数

类型

描述

领域
