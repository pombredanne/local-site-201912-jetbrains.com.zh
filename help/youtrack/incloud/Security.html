YouTrack InCloud 2019.2帮助

安全

YouTrack InCloud是JetBrains设计和使用的托管平台，用于将YouTrack作为服务交付。每个YouTrack InCloud实例实际上位于由Amazon Web Services（AWS）托管的服务器上。

每个服务器的实例数取决于几个参数，例如数据库大小和在线用户数。我们会永久监视每个服务器的负载和活动，以保持均衡的性能。当服务器活动达到一定水平时，我们将关闭该服务器以进行任何新的注册。

  • JetBrains运营团队负责配置，监视和管理InCloud服务器。

  • YouTrack InCloud支持团队为InCloud订户提供技术支持。

我们会全天候监控这些服务器，以确保其可用性和安全性。即使这样，您也可以做很多事情来保护数据。有关更多信息，请参阅保护安装 。

数据存储

我们使用Amazon Web Services （AWS）云作为托管服务提供商。所有数据都存储在Amazon Elastic Block Store（Amazon EBS）上 。每个Amazon EBS卷都会自动在其可用区中复制，以保护您的数据免受组件故障的影响，从而提供高可用性和持久性。Amazon EBS卷可确保一致且低延迟的性能。

传输中的数据加密

该主机上托管的所有实例*.myjetbrains.com域使用HTTPS连接来保护传输中的数据。对于使用自定义域的实例，您可以选择使用自己的CA证书。否则，您的实例将由TLS证书保护，该证书由Let's Encrypt自动生成并签名。让我们加密证书使用SHA-2加密哈希函数来加密传输中的数据。

在2017年，JetBrains停止了对YouTrack InCloud非安全连接的支持。

静态数据加密

存储托管实例信息的数据库已加密。即使在攻击者获得未经授权的访问的情况下，这也可以降低您的数据遭到破坏的可能性。

YouTrack将密码作为盐值存储在数据库中。每个用户的密码都使用不同的随机盐进行哈希处理。使用SHA-256密码哈希函数对加盐的密码进行哈希处理。

数据库本身（包括附件）使用ChaCha20算法加密。ChaCha20的主要实现有几种，包括Google选择ChaCha20替代TLS中的 RC4并将其包含在OpenSSH中 。

将为每个YouTrack InCloud实例分别生成一个唯一的加密密钥。这些键的访问仅限于YouTrack InCloud支持和JetBrains Operations团队。

资质认证

我们在AWS云中运行我们的服务。由于我们无法物理控制服务器，因此我们依赖于AWS进行的第三方认证 。

AWS已获得ISO 27001认证，并已根据支付卡行业（PCI）数据安全标准（DSS）认证为1级服务提供商。他们接受了年度SOC 1审核，并已针对联邦政府系统和中国防部系统的DIACAP 2级成功进行了评估。

资料管理员

JetBrains用于管理YouTrack InCloud的数据的内部应用程序（托管主机）只能由YouTrack InCloud支持和JetBrains运营团队访问。还可以直接在Amazon EC2服务器上访问和管理数据。验证通过单独的SSL密钥执行，并且服务器仅接受来自JetBrains和内部IP地址的传入SSH连接。

人与访问

仅出于维护和支持目的，只有YouTrack Development和JetBrains Operations团队有权访问InCloud服务器和Hosted Master。这些团队仅出于监视应用程序运行状况和执行系统维护的目的，或应客户要求访问Hosted Master和InCloud数据。

YouTrack InCloud旨在仅允许使用适当的凭据访问应用程序数据，这样，在没有明确了解其帐户凭据的情况下，任何客户都无法访问其他客户的数据。客户有责任维护自己的登录信息的安全性。

JetBrains运营团队从我们位于俄罗斯圣彼得堡的运营中心全天候监控YouTrack InCloud服务器（24x7）。根据客户的位置和偏好，我们的服务器托管在欧洲，北美和亚太地区的不同数据中心中。有关我们可用性的概述，请查看YouTrack InCloud服务状态页面。

后备

JetBrains操作团队负责创建和存储备份。备份也存储在Amazon服务器上。我们每天，每周和每月重新同步备份。您可以随时从YouTrack的“ 数据库导出”页面创建和导出自己的备份。有关更多信息，请参见数据库导出 。
