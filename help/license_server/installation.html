浮动许可证服务器帮助

安装

浮动许可证服务器（FLS）是基于Java的应用程序，并且需要Java 8或更早版本。FLS不支持Java 9和更高版本。要检查已安装的Java版本，请运行以下命令：

Java版本

在安装之前，请检查您的环境是否符合要求 。

在具有管理员权限的用户下安装FLS。

安装FLS

  1. 下载ZIP存档 。

    要检查下载的文件，请将其哈希值与SHA-256 checksum进行比较。

  2. 将ZIP软件包的内容提取到要服务器驻留的目录中。在整个文档中，我们将此目录称为FLS安装目录 。

    解压缩许可证服务器安装程序zip -d

  3. 使用命令行更改为所选的安装目录。

  4. 确保端口8080开了。

    默认情况下，FLS在主机上启动0.0.0.0和港口8080 。如果此端口供其他服务使用，请在启动FLS之前更改默认端口 。

  5. 使用以下命令启动FLS：

    ./bin/license-server.sh开始

    如果FLS成功启动，则输出应以以下内容结束：

    JetBrains许可服务正在监听{0.0.0.0:8080} JetBrains许可服务正在运行

    这意味着FLS在默认主机和端口上启动，但是此地址不安全。要对其进行更改，请参阅服务器URL 。

安装FLS

  1. 下载MSI安装程序 。

    要检查下载的文件，请将其哈希值与SHA-256 checksum进行比较。

  2. 启动MSI安装程序。它会在公共安装目录中自动创建目录FLS 。在整个文档中，我们将此目录称为FLS安装目录 。此外，它还会创建JetBrainsLS用户，该用户需要使用JetBrains帐户进行传入和传出连接。

  3. 确保端口80开了。

    默认情况下，FLS在主机上启动127.0.0.1和港口80 。如果此端口供其他服务使用，请在启动FLS之前更改默认端口 。

  4. 当您安装带有MSI分发的FLS时，它将作为Windows服务运行。Windows服务的显示名称是JetBrains License Service 。初始配置后，该应用程序将在计算机的后台运行。

    默认主机和端口不安全。要对其进行更改，请参阅配置主机和端口 。

有两种管理FLS的方法：

  • 要启动，停止和重新启动FLS，请使用服务JetBrains许可服务 。

    可以使用Microsoft管理控制台（MMC）的“ 服务”管理单元控制该服务 。Windows服务的列表通常可在“ 控制面板 ”的“ 管理工具”下找到 ，并且这需要管理员权限。

    或者，您可以使用以下命令从命令行在本地控制Windows服务sc.exe 。还有PowerShell cmdlet可以执行服务管理。

  • 要配置FLS，请使用管理脚本license-service.bat在FLS安装目录中。有关使用脚本的更多信息，请参见《 命令参考》 。

    您可以使用脚本来启动和停止FLS，但是在这种情况下，服务器将在当前用户下运行，并在用户会话结束时停止。

    FLS安装目录中的不同目录下有两个类似的license-server.bat脚本。第一个位于。\ bin下 ，它配置FLS应用程序的MSI捆绑包。第二个位于。\ app \ license-server \ bin下，并配置应用程序本身。本文档中的每个过程都有有关应使用哪种脚本的说明。

要查看用于管理FLS的命令列表，请使用help命令（或没有命令）：

./bin/license-server.sh帮助

有关使用这些命令的更多信息，请参见《 命令参考》 。

如果要从计算机上的任何位置运行管理脚本，可以将其位置添加到$PATH变量。

如果要从计算机上的任何位置运行管理脚本之一，可以将其位置添加到PATH变量。

FLS作为Linux服务

本章不适用于macOS。

您可以创建将在启动时启动FLS的Linux服务。在这种情况下，启动会随着机器的自动启动而发生。

配置systemd

  1. 在/ etc / systemd / system /中创建一个fls.service文件，其内容如下：

    [单位]说明= JetBrains FLS需求= network.target之后= network.target [服务]类型=简单ExecStart = / opt / fls-home / bin / license-server.sh start用户= fls-用户[安装] WantedBy = default.target

    根据需要自定义此文件的内容：

    • 设置User参数为任何现有机器用户的名称。在此配置中，Linux服务不取决于用户状态。

    • 设置ExecStart启动FLS的命令的参数。例如，如果FLS安装目录为/ opt / fls-home ，则将参数值设置为/opt/fls-home/bin/license-server.sh start 。

  2. 使用以下命令重新加载systemd守护程序并启用FLS服务以自动启动：

    systemctl守护进程重新加载systemctl启用fls.service

  3. 使用以下命令运行FLS服务：

    systemctl start fls.service

    使用以下命令检查正在运行的服务：

    systemctl状态fls.service
