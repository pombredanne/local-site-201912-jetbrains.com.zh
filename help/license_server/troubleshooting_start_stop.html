开始/停止问题

本主题为常见的开始和停止问题提供了逐步的解决方案。

未解决的地址

问题

许可证服务器启动时出现未解决的地址错误

*失败的org.eclipse.jetty.server。Server @ 7a9a042：java.net。SocketException：无法解析的地址，原因是：java.nio.channels。UnresolvedAddressException

解

确保已配置的标准主机名可解析为您的IP地址，如服务器URL中所述 。

地址已被使用

问题

* FAILED ServerConnector @ 2bdd8798 {HTTP / 1.1} {0.0.0.0:8080}：java.net。BindException：地址已在使用中

解

检查在同一主机和端口上是否还运行其他软件，然后停止该软件以使主机和端口可供FLS使用。如果不可能，请按照服务器URL中所述重新配置FLS以使用另一个端口。

无法分配请求的地址

问题

* FAILED ServerConnector @ 2243bc0d {HTTP / 1.1} {notexistinghost.jetbrains.com:8080}：java.net。BindException：无法分配请求的地址

解

确保您配置的主机/端口存在并且其地址可用。

当您配置的服务器的主机与安装该服务器的主机的FQDN或IP不匹配时，也可能发生此错误。

检查主机名。

主机名-f

可供配置的主机是主机名和小写的主dns后缀，由点“。”连接。

将获得的主机名与配置文件中指定的主机（放置在安装目录中）进行比较：

./conf/service-config.properties

如果它们不匹配，请调整配置以使它们匹配。

  • 目录

  • %programData%\JetBrains\LicenseService安装有FLS的计算机上的目录%programData%是应用程序数据目录的路径

如果MSI分发软件由服务用户运行JetBrainsLS 。

服务启动类型更改

问题

对于作为Windows服务运行的FLS，FLS启动类型自动更改为禁用

解

尝试使用以下建议来解决问题：

  • 关闭组策略更新，并检查是否有帮助。

  • 检查组策略设置，并检查是否存在关闭特定服务的规则，JetBrains许可服务可能被视为受该规则限制，因此需要更新规则以保持许可服务器启动类型。

  • 设置组策略以启动JetBrains许可服务，并让组策略将其视为保留其启动类型的主体。

如果没有帮助：

  • 使用有关服务使用组策略的说明 。

  • 和指令 ，介绍如何设置组策略。

Windows Server上的服务自动启动

问题

JetBrains许可服务无法在Windows Server操作系统上自动启动。

解

JetBrains许可服务的名称为LicenseService ，其显示名称为JetBrains许可服务 。将“ 自动（延迟启动）”设置为JetBrains许可服务的启动类型 ，以解决此问题。

我完全无法启动

问题

尝试启动FLS软件会导致错误消息，提示它完全无法启动。

正在启动JetBrains许可服务...java.lang。RuntimeException：我完全无法启动。见日志:(

解

将JDK / JRE升级到JRE 1.8或JDK 1.8 。

如果无法将此版本的JDK / JRE设置为默认版本，请使用以下命令指定供FLS使用的JDK / JRE主页：

  1. 使用命令行转到FLS安装目录。

  2. 使用以下命令停止FLS：

    ./bin/license-server.sh停止
  3. ./bin/license-server.sh Java设置

  4. 使用以下命令启动FLS：

    ./bin/license-server.sh开始
