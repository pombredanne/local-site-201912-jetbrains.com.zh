用户限制

如果您需要限制某人对许可证的访问，或者您已对用户进行了优先排序，即使没有可用的许可证，也应获得该许可证，但FLS管理员可以配置用户限制。当用户想要获得许可证时，IDE会向FLS发送请求。将此请求与FLS管理员定义的一组规则进行比较，然后授予或拒绝许可证。

要设置用户限制，请创建access-config.json文件并指定其path 。

access-config.json文件由一个或几个规则列表组成：白名单，黑名单和优先级列表。每个规则都是一组条件，所有条件都应匹配才能触发该规则。根据列表的类型，触发的规则将为接收到的请求授予或拒绝许可证。使用以下参数来配置规则。

规则参数

product

  • 定义要应用此规则的产品名称。该值可以是产品名称或正则表达式。FLS将此参数的值转换为.*.*正则表达式在预处理过程中，这就是将值与整个字符串进行比较的原因。因此，请勿使用字符串开头的符号（ ^ ）和结束符号（ $ ）的字符串。

userName

  • 定义启动IDE的用户。该值可以是纯用户名或正则表达式。您可以通过添加以下命令将其配置到IDE中user.name自定义具有所需属性的选项userName 。配置LDAP集成后，不能使用此参数。

hostName

  • 定义用户的主机名。该值可以是纯主机名或正则表达式。配置LDAP集成后，不能使用此参数。

buildNumber

  • 定义产品内部编号。该值应为正则表达式（可从内部版本＃17768获得 ）。

ip

  • 定义用户的IP地址。该值应为正则表达式。FLS支持IPv4（可从内部版本＃18692获得 ）。配置LDAP集成后，不能使用此参数。

checkLicenseOnly

  • 定义是否仅将规则应用于单个许可证而不检查产品包许可证，请参阅按单个产品许可证过滤 （可从内部版本＃19340获得 ）。

ldap

  • 定义LDAP配置的名称。配置LDAP集成（可从内部版本＃17768获得 ）时，此参数是必需的。

matchCondition

  • 定义LDAP搜索的规则。配置LDAP集成（可从内部版本＃17768获得 ）时，此参数是必需的。

对于指定为正则表达式的值，请使用Kotlin语言中的JVM语法。

黑名单

黑名单定义了禁止谁获得许可证。该列表由许多规则组成，其中每个规则都是一组条件。如果请求符合至少一个规则的所有条件，则FLS拒绝该请求。

具有两个规则的黑名单示例

{“ blacklist”：[{“ product”：“ GoLand”}，{“ product”：“ II”，“ buildNumber”：“ ^ 2017 \\。3. *”，“ userName”：“（username1 | username3 | katya）“，” hostName“：” username.company.net“}]}

在这种情况下，如果符合以下规则之一，则用户将不会获得许可证：

  • 要求的产品是GoLand

  • 该请求符合以下所有条件：

    • 该产品是IntelliJ IDEA

    • 产品版本为2017.3

    • 用户名是username1 ， username3 ， 要么katya

    • 主机名是username.company.net

排除正则表达式的黑名单示例

如果您想限制访问GoLand对于任何用户，除了username3 ， username4和katya并允许他们使用其他产品，请将以下规则添加到黑名单中。

{“ blacklist”：[{“ product”：“ GO”，“ userName”：“（？！username3 | username4 | katya）。*“}]}

该规则将拒绝所有与匹配的用户的请求userName并想要获得一个GO执照。

即使有All Products Pack许可或涵盖受限制产品的任何其他Pack许可，当您的请求与黑名单规则匹配时，您将无法访问。

要了解黑名单和白名单如何同时工作，请一起使用黑名单和白名单 。

白名单

白名单定义了允许谁获得许可证。该列表由许多规则组成，其中每个规则都是一组条件。如果请求符合至少一个规则的所有条件，则FLS授予许可证。创建白名单意味着所有与任何白名单规则都不匹配的请求将被拒绝。

具有一个规则和正则表达式的白名单示例

{“白名单”：[{“产品”：“全部”，“用户名”：“学生[0-9] *”，“ ip”：“ 172 \\。35 \\。152 \\ .. *”} ]}

在这种情况下，如果满足以下所有条件，则用户将获得许可证：

  • 要求的产品包含在All Products Pack执照

  • 用户名与正则表达式匹配，例如， student123

  • 用户的IP范围为172.35.152.0至172.35.152.255

要了解黑名单和白名单如何同时工作，请一起使用黑名单和白名单 。

一起使用黑名单和白名单

如果同时配置了白名单和黑名单，则黑名单的优先级更高。当请求与两个列表中的规则均匹配时，许可请求将被拒绝。

{“ blacklist”：[{“ product”：“ GO”，“ buildNumber”：“ ^ 2018 \\。3. *”，“ userName”：“（username1 | username2）”}]，“白名单”：[{ “ product”：“ GO”，“ userName”：“（username2 | username3 | username4）”}]}

虽然username2被列入白名单中，则该用户将不会获得GoLand 2018.3 ，因为黑名单的优先级更高。但是，如果username2请求其他版本GoLand ，此用户将获得许可证。

优先顺序表

可从版本＃16429获得 。

如果您已为用户划分了优先级，即使没有可用的用户也应有权访问许可证，请配置优先级列表。FLS撤销了他人的许可， last-seen参数是最旧的，并将许可证授予优先级较高的用户。这种选择谁将丢失票证的机制是不可配置的。

优先级列表是一个单独的部分，具有与黑名单和白名单相同的格式和匹配规则机制。它必须至少包含以下参数之一： userName ， hostName ， 要么ip 。其他参数是可选的。

一个用户的优先级列表示例

{“ prioritylist”：[{“ userName”：“ kate”}]}

在这种情况下，如果没有可用的许可证，则FLS授予一个已获得的许可证kate 。如果优先级用户获得了池中的所有现有许可证，则会拒绝优先级用户的另一个请求。

确保优先用户未列入黑名单。在这种情况下，FLS将拒绝其请求。

按单个产品许可证过滤

由于FLS支持JetBrains ToolBox许可模型，因此用户可以获得订阅包许可。例如，ReSharper许可证可以是单个许可证，也可以被所有产品包，ReSharper Ultimate，ReSharper C ++和ReSharper Ultimate + Rider许可证涵盖。如果您希望规则仅适用于单个ReSharper许可证，请启用checkLicenseOnly参数。

CheckLicenseOnly示例

{“ whitelist”：[{“ product”：“ RS0”，“ userName”：“ user2”，“ checkLicenseOnly”：“ true”}]}

在这种情况下， user2只能从单个ReSharper许可中获得ReSharper，但不能作为产品包的一部分。该用户将不会获得ReSharper Ultimate或所有产品包许可证。

该参数可以应用于任何许可证包所涵盖的任何其他产品。

LDAP整合

userName ， hostName和ip 规则参数与LDAP集成不兼容。

轻量级目录访问协议LDAP是与目录服务器进行交互的机制。如果您的组织使用LDAP，则可以配置访问规则以LDAP帐户过滤请求。通过在access-config.json中配置一个单独的块来启用其用法。它具有自己的一组参数。

参数

描述

url

必需参数，用于定义LDAP服务器地址

masterDn

必需参数，用于定义LDAP管理员的帐户

password

必需参数，用于定义管理员帐户的密码

useSSL

可选参数，用于定义是否使用SSL

connectionTO

可选参数，用于定义到LDAP的连接时间（以秒为单位）

responseTO

可选参数，用于定义从LDAP获得响应的时间（以秒为单位）

minPoolSize

可选参数，用于定义最小池大小

maxPoolSize

可选参数，用于定义最大池大小

LDAP配置示例

{“ ldap”：{“ openLdap1”：{“ url”：“ ldap：//”，“ masterDn”：“”，“ password”：“”，“ useSSL”：假，“ connectionTO”：20000，“ responseTO “：30000，” minPoolSize“：1，” maxPoolSize“：5}}，”白名单“：[{”产品“：” IntelliJ IDEA“，” ldap“：” openLdap1“，” matchCondition“：”（＆（objectCategory = user）（uid = $ {userName}））“}]}

在此示例中，第一部分描述了LDAP连接参数。IntelliJ IDEA许可证将仅授予在LDAP记录中具有用户名的用户。在白名单部分中，有一个matchCondition定义LDAP搜索规则的参数。唯一可用的占位符是${userName}这会将许可证请求中的用户名插入搜索中。

确保对FLS的请求中的用户名与LDAP记录中的用户名匹配。

组态

创建access-config.json文件后，在FLS配置文件中指定其路径。我们建议您将此文件放在conf下的FLS安装目录中。

  1. 使用命令行转到FLS安装目录。

  2. 使用以下命令停止FLS：

    ./bin/license-server.sh停止

  3. 将路径设置为access-config.json文件。

    例如，如果它位于conf下的FLS安装目录/ opt / fls-home /中 ，请使用命令行运行以下命令。

    ./bin/license-server.sh configure \ --access.config = file：/opt/fls-home/conf/access-config.json

  4. 使用以下命令启动FLS：

    ./bin/license-server.sh开始

  5. 检查是否正确指定了路径，并且文件本身已被清楚地写入。如果您在创建规则时出错，或者定义了错误的路径，则应该在日志文件中看到类似于以下内容的错误消息。

    无法解析位于/opt/fls-home/conf/access-config.json的配置

    此外，FLS仪表板还会显示以下消息Problem loading access config file以防性能不好。

  1. 使用JetBrains许可服务停止FLS。

    服务Microsoft管理控制台（MMC）（控制面板>管理工具>服务> JetBrains许可服务）

  2. 使用命令行转到FLS安装目录。

  3. 将路径设置为access-config.json文件。

    例如，如果它位于conf下的FLS安装目录C：\ fls-home \中 ，请运行以下命令。

    。\ apps \ license-server \ bin \ license-server.bat配置^ --access.config =文件：\ C：\ fls-home \ conf \ access-config.json

  4. 使用JetBrains许可服务启动FLS。

  5. 检查是否正确指定了路径，并且文件本身已被清楚地写入。如果您在创建规则时出错，或者定义了错误的路径，则应该在日志文件中看到类似于以下内容的错误消息。

    无法解析位于/opt/fls-home/conf/access-config.json的配置
