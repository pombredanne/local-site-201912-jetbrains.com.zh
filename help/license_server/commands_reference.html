浮动许可证服务器帮助

命令参考

管理浮动许可证服务器（FLS）的主要脚本是license-server.sh ，位于FLS安装目录中的/ bin下。要查看该脚本的所有可用命令，请转到FLS安装目录，然后使用以下命令运行该脚本： help命令（或不带命令）：

./bin/license-server.sh帮助

  • 要启动，停止和重新启动FLS，请使用服务JetBrains许可服务 。

    可以使用Microsoft管理控制台（MMC）的“ 服务”管理单元控制该服务 。Windows服务的列表通常可在“ 控制面板 ”的“ 管理工具”下找到 ，并且这需要管理员权限。

    或者，您可以使用以下命令从命令行在本地控制Windows服务sc.exe 。还有PowerShell cmdlet可以执行服务管理。

  • 要配置FLS，请使用管理脚本license-service.bat在FLS安装目录中。有关使用脚本的更多信息，请参见《 命令参考》 。

    您可以使用脚本来启动和停止FLS，但是在这种情况下，服务器将在当前用户下运行，并在用户会话结束时停止。

    FLS安装目录中的不同目录下有两个类似的license-server.bat脚本。第一个位于。\ bin下 ，它配置FLS应用程序的MSI捆绑包。第二个位于。\ app \ license-server \ bin下，并配置应用程序本身。本文档中的每个过程都有有关应使用哪种脚本的说明。

要查看该脚本的所有可用命令，请转到FLS安装目录，然后使用以下命令运行该脚本： help命令（或不带命令）：

。\ bin \ license-server.bat帮助

输出包含带有简短说明的FLS命令列表。有关命令的更多信息，请运行help在该命令上。例如，如果您想要有关configure命令，运行以下命令：

./bin/license-server.sh帮助配置
。\ bin \ license-server.bat帮助配置

命令

描述

help

打印指定命令的用法文本。

start

在后台启动FLS。

  • start soft在后台轻轻启动FLS（在这种情况下，不应用超时）。

run

在当前控制台中运行FLS。

stop

停止FLS。

  • stop soft轻柔地在后台停止FLS（在这种情况下，不应用超时）。

restart

在后台重启FLS。

  • restart soft使用以下命令在后台重启FLS stop soft和start soft 。

rerun

在当前控制台中重新运行FLS。

  • rerun soft使用以下命令在当前控制台中重新运行FLS stop soft和run 。

status

打印JetBrains许可服务的完整状态（如果JetBrains许可服务有效，则退出，代码为0，否则为1）。

  • status short打印简短且可分析的JetBrains许可服务状态。

configure

配置FLS属性和JVM选项。

  • --=要么-- 更改指定属性的值。有关更多信息，请参见配置属性 。

  • -f 从指定的文件读取属性并应用其值。

  • -J将指定的JVM选项添加到运行FLS的JVM使用的选项列表中。有关更多信息，请参见./conf/license-server.jvmoptions 。\ conf \ license-server.jvmoptions文件中的注释。例如，要将最大Java堆大小更改为1024m，请设置-J-Xmx1024m属性。

  • --J指定临时JVM选项。

  • --launcher仅配置启动器，并且不将指定的属性传递给FLS。

dump

提供FLS调试信息。

  • dump threads将FLS线程转储到控制台。

  • dump threads to-file将FLS线程转储到/ logs目录下的文件。

  • dump threads to-file short和...一样dump threads to-file ，但仅将创建文件的绝对路径打印到控制台。

  • dump heap将FLS堆转储到/ logs目录下的文件。

  • dump heap short和...一样dump heap ，但仅将创建文件的绝对路径打印到控制台。

kill

终止JetBrains许可服务过程。

daemon

管理FLS守护程序。运行此命令作为root用户或具有超级用户特权（ sudo ）。

  • daemon configure与configure命令。

  • daemon install默认情况下，安装FLS守护程序以在先前记住的用户或root用户下运行。

  • daemon install --user=要么daemon install --user 记住指定的用户并安装要在此帐户下运行的FLS守护程序。

  • daemon install --force如果守护程序已经存在，请先将其卸载。

  • daemon start加载FLS守护程序并启动它。

  • daemon status与status命令。

  • daemon stop停止并卸载FLS守护程序。

  • daemon restart重新加载并重新启动FLS守护程序。

  • daemon uninstall卸载FLS守护程序。

java

管理用于运行FLS的Java路径。要指定JVM选项，请使用configure命令。

  • java print找到java可执行文件设置为运行FLS并将其打印到控制台。

  • java set 将指定的路径设置为java具有适当版本和属性的运行FLS的可执行文件。

  • java reset重置为FLS显式设置的Java路径。

配置属性

的configure该命令在大多数用于管理FLS的过程中使用。通常，您需要为配置属性之一设置所需的值。

属性

描述

--listen

指定运行FLS的主机（IP地址或域名）。默认情况下是0.0.0.0 。请参阅配置主机和端口 。

--port

指定运行FLS的端口。默认情况下是8080 。请参阅配置主机和端口 。

--context

如果主机具有其他服务，则指定用于访问主机上FLS的自定义URI。请参阅自定义上下文路径 。

--jb.license-server.base-dir

指定FLS数据目录的位置。请参阅配置数据目录 。

--jetty.virtualHosts.names

指定反向代理服务器主机名。请参阅反向代理 。

--access.config

指定用户限制文件access-config.json的路径。请参阅用户限制 。

--reporting.out.of.license.threshold

指定发送有关缺少可用许可证的电子邮件通知的阈值。请参阅“ 缺少可用许可证通知” 。

--reporting.token

配置用于统计访问的密钥。请参阅通过API统计 。

--smtp.server

指定用于发送电子邮件统计信息的邮件服务器的主机名。请参阅通过电子邮件发送统计信息 。

--smtp.server.port

指定用于发送电子邮件统计信息的邮件服务器的端口。请参阅通过电子邮件发送统计信息 。

--stats.recipients

指定谁应该接收电子邮件报告。请参阅通过电子邮件发送统计信息 。

--stats.from

指定电子邮件报告的发件人。请参阅通过电子邮件发送统计信息 。

--smtp.server.username

指定用于访问邮件服务器的用户名。请参阅通过电子邮件发送统计信息 。

--smtp.server.password

指定用于访问邮件服务器的密码。请参阅通过电子邮件发送统计信息 。

下表列出了适用于..bin / license-server.bat的属性，该属性配置FLS应用程序的MSI捆绑包。

属性

描述

--base-url

指定主机（IP地址或域名）和运行FLS的自定义URI。默认情况下是127.0.0.1没有其他URI。请参阅配置主机和端口 。

--listen-port

指定运行FLS的端口。默认情况下是80 。请参阅配置主机和端口 。

--virtual-hosts

指定将请求发送到FLS的主机的名称Host标头验证。请参阅反向代理 。

--virtual-hosts-default

指定默认虚拟主机列表，用于Host标头验证。请参阅反向代理 。
