要了解您拥有的FLS版本以及是否有新的FLS版本，请参阅检查版本 。在升级FLS实例之前，请检查您的环境是否符合要求 。

旧版本（在版本＃1230之前）与当前版本的浮动许可证服务器（FLS）之间有很多差异。从内部版本16429 （于2018年3月19日发布）开始，仅最新的JetBrains产品支持FLS：版本2018.2.1以后（对于.NET工具晚于2018.3）。当IDE连接到不支持的FLS版本时，您将收到以下通知：“检测到过期的许可证服务器”。有关更多信息，请参阅过期的许可证服务器 。要了解最新版本的新功能，请参阅发行说明 。

如果您的FLS版本非常旧（在内部版本＃12431之前），我们建议您安装最新版本之一。在这种情况下，您需要重复配置此新FLS实例的整个过程，并将其链接到您的公司资料。注册后，新的FLS将从JetBrains帐户获取许可证信息。

就地升级可从内部版本＃12431获得 ，类似于新的安装过程。简而言之，您需要使用以前的设置保存配置文件，用新的版本替换旧的FLS程序文件，然后放回配置文件。您需要保存整个conf位于FLS安装目录中的子目录。如果您配置了用户限制 ，请保存access-config.json文件。

我们还建议创建以前版本的备份，以便在新版本出现问题时能够还原它。

升级与新安装类似。您需要重复配置FLS实例的整个过程，并将其链接到公司资料。注册后，新的FLS将从JetBrains帐户获取许可证信息。

  1. 将所有配置文件保存到任何安全目录，并备份当前的FLS安装目录，例如zip存档。

  2. 使用命令行更改为FLS安装目录。

  3. 使用以下命令停止FLS：

    ./bin/license-server.sh停止

  4. 下载ZIP存档并将其解压缩到FLS安装目录（用新文件夹替换旧文件夹和文件）。

  5. 放回保存的配置文件。

  6. 使用以下命令启动FLS：

    ./bin/license-server.sh开始

  7. 确保服务器像升级之前一样向用户授予请求的许可证。这意味着正在使用现有注册。

  3. 运行下载的MSI安装程序并遵循向导步骤。
