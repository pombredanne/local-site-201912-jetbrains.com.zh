配置问题

本主题提供了针对常见配置问题的分步解决方案。

许可证服务器正在运行

问题

尝试通过以下方式配置FLS

./bin/license-server.sh配置\-
。\ bin \ license-server.bat配置^-
导致“ JetBrains许可服务正在运行”，并且不会导致配置数据发生任何变化。

解

有些配置需要停止FLS。

  1. 使用命令行转到FLS安装目录。

  2. 使用以下命令停止FLS：

    ./bin/license-server.sh停止

  3. 尝试再次执行配置命令

  4. 使用以下命令启动FLS：

    ./bin/license-server.sh开始

  1. 使用JetBrains许可服务停止FLS。

    服务Microsoft管理控制台（MMC）（控制面板>管理工具>服务> JetBrains许可服务）

  2. 尝试再次执行配置命令

  3. 使用JetBrains许可服务启动FLS。

该属性未知

问题

[APP-WRAPPER]无法配置JetBrains许可服务：java.util.concurrent。ExecutionException：com.jetbrains.bundle.exceptions。BadConfigurationException：该属性是未知的，无法由configure命令设置

原因

尝试通过以下方式配置FLS

。\ bin \ license-server.bat配置^-

Windows导致“属性未知，无法通过configure命令设置”，并且不会更改配置数据。

解

  1. 使用JetBrains许可服务停止FLS。

    服务Microsoft管理控制台（MMC）（控制面板>管理工具>服务> JetBrains许可服务）

  2. 使用命令行转到FLS安装目录。

  3. 尝试执行配置命令，如下所示：

    。\ apps \ license-server \ bin \ license-server.bat配置^-

  4. 使用JetBrains许可服务启动FLS。

如果这样不利于尝试升级到下载页面上提供的FLS的最新版本。

未使用已配置的代理属性

问题

尽管为FLS连接配置了代理，连接仍然失败。

解

FLS尝试不使用系统代理直接通过HTTPS（端口443）协议与JetBrains联系。如果在安装FLS的环境中需要使用代理进行外部连接，则需要按照代理配置说明中的说明为FLS软件显式配置代理设置。

代理配置可从版本＃8732获得 。如果FLS较旧，则将安装升级到“ 安装”页面上可用的最新版本。

如果版本正确，请检查许可证服务器是否使用了代理。去参观/check-connection （有关更多详细信息，请参见Connection ）。如果没有有关代理的信息，请通过从安装目录运行以下命令来重新启动FLS：

./bin/license-server.sh重新启动

之后尝试访问/check-connection再次检查是否有“可用代理”信息。如果“可用代理”部分中显示了代理信息，但是与JetBrains资源的连接失败，请与您组织的IT部门联系，以确保代理是透明的，并确保服务器通过包含证书在内的所有已传递数据来访问JetBrains资源。由JetBrains资源提供。该服务器应具有的JetBrains资源是www.jetbrains.com和account.jetbrains.com HTTPS（443端口）。

如果版本正确，请检查许可证服务器是否使用了代理。去参观/check-connection （有关更多详细信息，请参见Connection ）。如果没有有关代理的信息，请使用JetBrains许可服务重新启动FLS。

之后尝试访问/check-connection再次检查是否有“可用代理”信息。如果“可用代理”部分中显示了代理信息，但是与JetBrains资源的连接失败，请与您组织的IT部门联系，以确保代理是透明的，并确保服务器通过包含证书在内的所有已传递数据来访问JetBrains资源。由JetBrains资源提供。可用于服务器的JetBrains资源是www.jetbrains.com和account.jetbrains.com HTTPS（443端口）。

活动代理

问题

FLS无法与JetBrains联系，因为它不遵循代理提供的重定向。

解

出于安全原因，要求FLS能够指定连接点本身。如果通过代理建立连接，则要求代理是透明的，并且要求允许服务器访问JetBrains资源，并传递所有数据，包括JetBrains资源提供的证书。

以下配置可能被认为是可能的解决方案：

  • FLS软件安装在可以访问Internet的计算机上，但不能访问组织的内部计算机。

  • 开发人员的计算机可以将计算机与许可证服务器连接，但不能连接到Internet。

描述的配置也称为DMZ（非军事区），其作用是使许可证服务器与JetBrains联系，并同时使开发人员的计算机保持脱机状态。如果无法进行配置，则可以使用脱机使用许可的替代方法-脱机激活代码，适用于隔离环境中安装的产品。可以按照“ 许可和购买常见问题解答”中的说明进行分发。
