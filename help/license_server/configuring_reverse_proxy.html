浮动许可证服务器帮助

反向代理

如果浮动许可证服务器（FLS）在反向代理后面运行，则需要将代理主机名设置为jetty.virtualHosts.names属性。否则，您将收到类似于以下内容的错误，并且将无法访问服务器。

HTTP错误403访问/时出现问题。原因：不允许传递标头“ Host”的值。请与您的服务器管理员联系。由Jetty提供技术：// 9.3.24.v20180605

如果为FLS软件配置的主机与运行该软件的计算机的主机不匹配，则还需要将此主机名设置为属性，以验证请求中Host标头的值。在这种情况下，FLS将使用带有与分配给该属性的主机名匹配的主机头的请求来处理请求。如果Host标头与任何已配置的主机名都不匹配，您将得到相同的错误。

配置中

  1. 使用命令行转到FLS安装目录。

  2. 使用以下命令停止FLS：

    ./bin/license-server.sh停止

  3. 设置值jetty.virtualHosts.names反向代理服务器主机的选项。如果有多个主机，请用逗号分隔它们。例如，如果其值为proxy-server.domain.com和fls.company.net ，运行以下命令：

    ./bin/license-server.sh配置\ --jetty.virtualHosts.names =代理服务器.domain.com，fls.company.net

  4. 使用以下命令启动FLS：

    ./bin/license-server.sh开始

默认情况下，浮动许可证服务器（FLS）不会在对服务器的请求中验证主机标头。要启用主机头验证，您需要配置以下属性之一：

  • --virtual-hosts定义将请求发送到FLS的主机名。

  • --virtual-hosts-default具有三个可能的值：

    • none -没有添加虚拟主机。如果virtual-hosts属性也未设置，则虚拟主机列表为空，并且对主机标头没有任何限制。

    • local -虚拟主机列表包含服务器的所有本地接口，包括已配置的listen地址和服务器主机名。

    • all-known -虚拟主机列表包含服务器的所有网络接口，包括已配置的listen地址和服务器主机名。

如果Host标头与任何已配置的属性值都不匹配，您将收到类似于以下内容的错误：

HTTP错误403访问/时出现问题。原因：不允许传递标头“ Host”的值。请与您的服务器管理员联系。由Jetty提供技术：// 9.3.24.v20180605

配置中

  1. 使用服务Microsoft管理控制台（MMC）停止FLS （控制面板>管理工具>服务> JetBrains许可服务）

  2. 使用具有管理员权限的命令行转到FLS安装目录。

  3. 设定值virtual-hosts配置为使用FLS的主机。如果有多个主机，请用逗号分隔它们。

    例如，如果是proxy-server.domain.com和fls.company.net ，运行以下命令：

    。\ bin \ license-server.bat配置^ --virtual-hosts = proxy-server.domain.com，fls.company.net

  4. 设置virtual-hosts-default的特性达到理想值，如上所述。

    例如，如果所选值是local ，运行以下命令：

    。\ bin \ license-server.bat配置^ --virtual-hosts-default = local

  5. 使用服务Microsoft管理控制台（MMC）启动FLS
