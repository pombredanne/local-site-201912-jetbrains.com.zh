创建项目-常规

  • 项目名称：为您的项目命名并输入。

  • 项目ID：根据指定的项目名称自动创建，但是可以根据需要进行修改。

  • 代码审阅ID模式：一种将在代码审阅ID中使用的命名模式。根据指定的项目名称自动创建，但是可以根据需要进行修改。

VCS

在比较表中查看每个VCS支持的功能 。

选择与您的VCS相对应的选项卡，并按如下所述指定其参数：

吉特

  • Git托管服务提供商：从下拉列表中选择您的托管服务提供商。如果存储库由您组织的服务器托管，请选择Generic 。

  • 存储库URL：示例： https://git.example.com/project要么git://git.example.com/project

  • 身份验证方法：选择首选方法： Anonymous ， Password或SSH并指定凭据以连接到您的VCS

  • 导入请求（合并）请求作为分支：适用于所有GIT托管提供程序（通用除外）。启用它以将拉取请求显示为Upsource中的分支。

  • 同步注释和拉取请求：适用于GitHub —请参阅说明 。

  • 同步注释和合并请求：适用于GitLab —请参见说明 。

水银

  • 资料库URL：格式： hostname:[port]例：http://hg.example.com/project

  • 身份验证方法：选择首选方法： 匿名或密码，然后指定凭据以连接到您的VCS

重要说明：为了让Upsource识别Mercurial存储库，您需要：

  • 在您的上游服务器上安装Mercurial客户端 。

  • 在系统环境变量中指定指向hg可执行文件的PATH。

  • 重新启动源

Perforce

  • 服务器名称：示例：perforce.example.com:1111

  • 身份验证方法：选择首选方法： 匿名或密码，然后指定凭据以连接到您的VCS

  • 工作区映射：

    要包括整个项目，请指定：////...

    要仅包含项目的选定目录，请列出这些目录，然后列出Upsource中的相应目录（由空格分隔）。例：

    // depot / project1 / directory1 /...// upsource / depot / project1 / directory1 / ...// depot / project1 / directory2 /...// upsource / depot / project1 / directory2 / ...

  • 重要：为了让Upsource识别Perforce存储库，您需要：

    1. 在安装了Upsource的计算机上下载并安装Perforce Command-Line Client。

    2. 对于macOS和Linux，在系统环境变量中指定_P4：COMMAND LINE_的PATH（请参阅P4说明 ）。

    3. 重新启动源

颠覆

  • 存储库URL：示例：http://svn.example.com/project/project

  • 身份验证方法：选择首选方法： Anonymous ， Password或SSH并指定凭据以连接到您的VCS

  • 分支：您可以定义一个或多个SVN路径，Upsource应该在其中寻找SVN分支。使用+：或-：之类的规则来包含或排除分支。分支路径是相对于上面指定的存储库URL的。模式可以包含一个*，以匹配目录名称中的任意数量的字符。

    标准SVN布局示例：

    +：树干+：分支/ *

多资料库设定

  • 项目位于多个存储库中：如果将项目划分为几个单独的存储库，则选择此选项，但是构建此项目需要将所有存储库都提取到根目录下的特定位置。映射多个存储库可让您从单个Upsource项目中观察和查看所有存储库中的更改。

    单击添加存储库，然后为每个存储库指定以下设置：

    • 装载路径：在项目内部指定此VCS根目录所在的目录。

    • 映射ID： VCS存储库的唯一ID。它是自动生成的，但是您可以指定自己的。

检查间隔

  • 检查间隔：反映Upsource多久检查一次存储库中的新修订（默认为每300秒一次）。

    如果通常不经常提交到存储库，请考虑将“ 检查间隔”设置为较大的值。

用户数

  • 创建用户组：让Upsource为您创建以下用户组：

    • 代码查看器

    • 开发者

    • 项目管理员

    或将其保留为未选中状态，以便稍后自己创建。

组是用户的集合，其中每个用户具有相同的权限或角色（权限集）。
