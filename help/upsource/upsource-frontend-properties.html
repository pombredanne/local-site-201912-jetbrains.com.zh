上游前端属性

此页面上列出的属性由Upsource Frontend进程使用，可以在配置文件upsource-frontend.jvmoptions中添加或编辑，以更改UI中不可用的Upsource设置。

有关ZIP或Docker安装的信息，请参见相应的分步说明。

特定于上游的属性

下表中特定于上游的属性只能通过使用configure命令或通过手动编辑配置文件upsource-frontend.jvmoptions位于/conf/upsource-frontend目录。

属性

描述

默认值

-Dupsource.github.sync.period.minutes

以分钟为单位指定GitHub同步之间的时间间隔。

20

-Dfrontend.psi.filesize.limit

可以使用代码导航功能的文件的最大大小（以字节为单位）。

100000

-Dupsource.mail.notifications.sending.delay.millis

电子邮件通知间隔，以毫秒为单位（600000 = 10min）。

300000

-Dfrontend.view.filesize.limit=XXXX

要显示的最大文件大小。

2000000

-Dfrontend.diff.filesize.limit

差异显示的最大行数。

1000

-Dfrontend.revision.count.limit

查看/比较分支时要显示的最大修订版本数。

200

-Dfrontend.revision.file.count.limit

查看/比较修订版本时要显示的最大修订版本数。

200

前端默认JVM选项

默认情况下，以下JVM选项为Upsource Frontend进程配置。您可以使用configure命令或通过手动编辑配置文件upsource-frontend.jvmoptions

-ea -XX：+ HeapDumpOnOutOfMemoryError -XX：HeapDumpPath = -XX：ErrorFile = hs_err_pid %% p.log -Dfile.encoding = UTF-8 -Xmx4096m -XX：MaxMetaspaceSize = 512m
