Creating issues for your issue tracker

Code ownership report

This report is displayed at the bottom of the review page if the review includes files that supposedly have a higher chance of having issues and may require more attention from reviewers. The following types of files are listed:

  • The file had been exclusively authored by one person until another person made changes to it recently.
  • The file has no evident person in-charge. It has many (more than 6) authors all of which contribute to it on regular basis.
