Synchronizing your activities in GitHub projects

If GitHub synchronization has been enabled by the administrator, the following user activities will be synchronized between Github and Upsource:

  • Discussion comments posted in GitHub will be shown in Upsource and vice-versa

  • Pull requests are recognized and can be created right from Upsource

  • A branch review will be automatically created on a new pull request

Only open pull requests can be synchronized.

A GitHub icon on the project home page means Upsource recognises you as a GitHub user:

If you are not recognized, click Login to GitHub:

Now you can merge branches, create and close pull requests from Upsource, and the comments you make in Upsource will be posted in GitHub as well.

Pull requests and merges

Creating pull requests and merging from Upsource is only available to contributors registered with a corresponding GitHub project.

When a new branch is created along with a pull request in GitHub, Upsource automatically creates a branch review for it. After reviewing and accepting the changes you can merge the branch without leaving Upsource:

If the branch has conflicts with the base branch and cannot be merged automatically, a corresponding message will be displayed in place of the Merge button:

If the branch has been created without a pull request, you may create a pull request right from Upsource:

  1. Click Create pull request.

  2. In the displayed dialog window, enter your pull request title and select the base branch.

  3. Click Create.

In addition to the pull request, a branch review will be created and the branch revisions included in it.

It's only possible to create a pull request from Upsource on an existing branch, there's no way to do it for a forked repository.
