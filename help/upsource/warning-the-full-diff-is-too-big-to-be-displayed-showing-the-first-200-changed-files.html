上载2019.1帮助

警告：“完整差异太大而无法显示，显示前200个更改的文件”

当您查看或比较包含大量文件的大量修订时，可能会显示此警告。如果必须查看所有更改，则可以提高限制。

解

此更改可能会对UI性能产生负面影响。

要提高ZIP安装中的限制：

  1. 停止上载： /bin/upsource.sh stop

  2. 打开位于/ conf / upsource-frontend目录中的配置文件upsource-frontend.jvmoptions 。如果文件不存在，请创建它。

  3. 将以下行添加到该文件：

    -Dfrontend.revision.file.count.limit = XXXX

    XXXX是要显示的文件数。

  4. 保存并关闭文件。

  5. 启动Upsource 。

在Docker安装中提高限制：

  1. 停止上载： docker exec stop

  2. 打开配置文件upsource-frontend.jvmoptions 。该文件位于/ conf / upsource-frontend /中 。如果文件不存在，请创建它。

  3. 将以下行添加到该文件：

    -Dfrontend.revision.file.count.limit = XXXX

    XXXX是要显示的文件数。

  4. 保存并关闭文件。

  5. 启动上游： docker start 。

上次修改时间：2019年8月9日
