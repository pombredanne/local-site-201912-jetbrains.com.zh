错误：“测试VCS连接失败。加载上传的密钥时出错。无效的私钥”

在Upsource中创建新项目时，您可以选择以下身份验证方法之一来访问VCS：

  • 匿名

  • 密码

  • SSH密钥

如果选择SSH密钥，则还应提供一个私钥和密码短语（如果有）。之后，在测试连接时，您可能会收到以下错误：

测试VCS连接失败。列出远程引用失败：com.jcraft.jsch。JSchException：加载上载的密钥“ the-key”时出错：无效的私钥

原因1

密钥是由最新版本的ssh-keygen采用Upsource不支持的格式。此项以以下内容开头：

----- BEGIN OPENSSH私钥-----

解

通过添加以下内容来生成Upsource支持的格式的密钥： -m pem选择ssh-keygen 。

例如：ssh-keygen -t rsa -b 2048 -m pem -f mykey

正确的密钥应以以下内容开头：

----- BEGIN RSA私钥-----

原因＃2

您的公钥/私钥对可能是通过PuTTYgen生成的。上游不支持PuTTY格式的私钥。

解

将PuTTY格式的密钥转换为OpenSSH格式。为此，请执行以下步骤：

  1. 打开PuttyGen。

  2. 单击文件|加载私钥 。

  3. 转到转化|导出OpenSSH并导出您的私钥。

  4. 将转换后的私钥粘贴到Upsource中。
