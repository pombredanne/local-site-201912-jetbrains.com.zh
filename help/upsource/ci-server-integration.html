CI服务器集成

与CI服务器（连续集成服务器）集成后，Upsource可以在提交特定项目时接收并显示构建状态。

当有人进行提交并且构建系统运行该构建时，其状态将显示在该修订旁边的修订列表中。由于单个提交有时可以启动多个构建，因此该提交可以具有多个构建状态：

从TeamCity接收构建状态

与TeamCity CI服务器集成是通过Commit Status Publisher插件提供的，该插件随附于TeamCity 9.1.7及更高版本。配置它与Upsource一起使用很容易，并且可以在TeamCity端完成。

与TeamCity集成

  1. 确保您对TeamCity中的项目具有管理员访问权限。

  2. 转到设置|构建功能|添加构建功能 。

  3. 从下拉列表中选择“ 提交状态发布者 ”。

  4. 指定以下信息：

    • 发布者：JetBrains Upsource

    • 上游URL：上游服务器的URL

    • 上级项目ID：您的上级项目ID

    • 用户名，密码： 在Upsource中指定的身份验证令牌

      或Upsource项目的管理员凭据。

  5. 保存设置。

有关更多信息，请参阅TeamCity文档。

从其他CI服务器接收构建状态

Upsource带有一个构建状态端点，该端点在http [s]：// your-upsource-server /〜buildStatus /处打开以接收数据作为JSON有效负载。

POST请求应包含以下标头： Content-Type: application/json; charset=UTF-8

如果未提供凭据，则将以来宾权限执行请求。为了确保安全连接，请发送Authorization标头和您的请求：Authorization: Basic login:password

哪里login:password是Upsource中定义的身份验证令牌 。

要代表特定用户发出请求，可以提供用户凭据（用Base64编码）而不是令牌。
