错误：“无法锁定JVM内存（ENOMEM）”

对于某些（所有）修订，上游持续显示“索引编制中”，或者Apache Cassandra数据库会定期重新启动。通过Upsource日志，您会遇到以下错误：

无法锁定JVM内存（ENOMEM）。这可能会导致部分JVM换出，尤其是在启用了mmapped I / O的情况下。增加RLIMIT_MEMLOCK或以root身份运行Cassandra

要么：

打开的文件太多。

解

资源限制不足可能会导致这种错误。为了减轻这种情况，我们建议设置：

  • 最大打开文件数为100000

  • 内存锁定和地址空间限制为无限

  • 进程数至32768

您可以通过在/etc/security/limits.conf文件中添加以下几行来实现：

*-无限锁*-无文件100000 *-nproc 32768 *-无限
