问题跟踪器集成

通过问题跟踪器集成，您可以将Upsource项目（无论是现有项目还是新项目）链接到您的问题跟踪器中的指定项目，从而使用户可以通过Upsource代码审查和讨论来创建问题 。

创建问题时，还可以指定其必填值：问题类型和优先级（适用于JIRA和YouTrack 7.0及更高版本）。

YouTrack

上游资源可以与托管的YouTrack实例集成。不支持与YouTrack InCloud集成。

通过Hub提供与YouTrack的集成-Hub是一项服务，可跨所有与其连接的JetBrains服务和应用程序提供用户授权和用户管理。

要与YouTrack集成，您需要：

  1. 安装并运行 Hub。

  2. 将您的Upsource和YouTrack安装配置为与独立的Hub一起使用。

  3. 转到Hub并合并应该集成的Upsource和YouTrack项目。

  4. 在Upsource的Create（Edit）Project页面上，转到Integration选项卡， 然后从下拉列表中选择YouTrack 。YouTrack集成将自动配置。

吉拉

设置与JIRA的集成：

  1. 在Upsource的Create（Edit）Project页面上，转到Integration选项卡，然后从下拉列表中选择JIRA。

  2. 指定JIRA服务器URL，链接URL（如果与服务器URL不同，则为公共URL）和项目密钥

  3. 选中创建JIRA身份验证模块，以使其他JIRA用户可以使用其JIRA凭据登录到Upsource，并创建将以其自己的名称报告的问题。否则，将从Upsource项目报告的所有问题显示在上述JIRA用户名（项目管理员）下的相应JIRA项目中。

    如果选择此选项，则需要在单击Create（Edit）project之前导航到Hub并创建JIRA Auth Module 。

  4. 选择授权方法并指定凭据。对于JIRA用户帐户（JIRA项目管理员）凭据，仅接受JIRA用户名-不要输入电子邮件地址。

  5. 测试与JIRA服务器的连接。

  6. 选中启用在JIRA中创建新问题 。

  7. 当该评论的状态更新为：时，选项“ 启用JIRA迭代”工作流将更改与某个评论相关的问题的状态：

    • 评论已创建

    • 评论已接受

    • 评论有问题

    • 评论已关闭

    从JIRA项目中的现有问题状态中，为每个审阅状态选择所需的状态，然后将其键入到相应的“ 过渡名称”字段中 。

  8. 要求用户使用其JIRA凭据登录一次Upsource。他们的帐户将被Hub识别并记录。

  9. 授予这些JIRA用户对Upsource项目的访问权限

  10. 您还可以为JIRA用户提供一种使用其Upsource凭据登录Upsource的方法，并且仍然能够以其JIRA名称报告问题。为此，请转到Hub并合并其JIRA和Upsource帐户 。在这种情况下，可以跳过上一步。

在集成设置过程中，将在Upsource中指定的JIRA用户名（项目管理员）下，在相应的JIRA项目中报告源于没有关联JIRA帐户的Upsource用户的所有问题。
