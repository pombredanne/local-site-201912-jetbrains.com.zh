先决条件

在继续安装Upsource之前，请确保...

检查团队的需求和期望

  • 如果您的开发团队正在寻找浏览代码库的不同修订版本而不将其检出到本地计算机，以及讨论和查看在代码库中所做的更改的工具，那么Upsource是适合您的工具。

  • 上游仅识别提交给您的存储库的更改。

  • Upsource可以与任何类型的项目和语言一起使用，但是其静态分析仅适用于使用IntelliJ IDEA，Maven或Gradle作为其构建系统的Java，JavaScript和Kotlin项目。如果将其安装在Upsource服务器计算机上，则还支持PHP和Python。

检查您的硬件和软件

  1. 由于Upsource是本地应用程序，因此您应该具有服务器以将Upsource部署到该服务器，并且该服务器应：

    • 拥有8 GB或更多的RAM

    • 运行以下64位操作系统之一：

      • Windows Vista或更高版本

      • macOS 10.7或更高版本

      • Linux（根据我们的知识，任何64位发行版都可以）

    • 有足够的磁盘空间来容纳您的项目。上游将整个VCS映射到其内部数据库并托管以下内容：

      • 带有提交图和所有文件内容的VCS数据

      • 代码审查数据（包括讨论和时间表数据）

      • 这些项目的IntelliJ IDEA索引，IDEA可以在其中提供代码见解

      因此，实际所需的磁盘空间取决于您的存储库大小，支持IDEA智能的文件数以及您的团队工作流程（每天的提交，审阅和讨论数）。它的大小可能是VCS数据大小的3倍到70倍。

  2. 您的标准主机名应该可以解析为您的IP地址。要检查它：

    • 在Linux或macOS上，运行以下命令：

      ping $（主机名-f）

    • 在Windows上，要获取标准主机名，请运行：

      ipconfig /全部

    然后尝试ping该名称（该名称应类似于mymachine.mydomain.com）

  3. 您的开发团队应使用Git，Mercurial，Subversion或Perforce进行版本控制。

  4. 上游用户应使用现代的Web浏览器。上游支持以下最新版本：

    • 火狐浏览器

    • 铬

    • 苹果浏览器

    • 边缘

    • 歌剧

检查硬件和软件后，可以继续安装和配置Upsource-阅读与您的首选安装类型相对应的说明：

  • ZIP安装

  • Docker安装

  • 云端安装
