当我们询问时如何获取其他详细信息（日志，数据库，HAR文件）

有时我们需要您提供其他详细信息，以便更有效地调查问题。在这种情况下，我们恳请您向我们发送Uplog日志文件，Upsource数据库或将浏览器会话记录到HAR文件中。

所有这些工件都应上传到我们的安全FTP服务器： 匿名用户下的ftp://ftp.intellij.net/.uploads 。任何人都可以将文件上传到此服务器。完成后，请与我们共享档案的名称。

如何获取上游日志文件？

默认情况下，日志文件存储在提取ZIP存档的文件夹的/ logs目录中，即所谓的Upsource主目录。如果在安装过程中更改了默认设置 （“配置”向导中的“ 高级设置”区域），则可以在您设置的目录中找到日志文件。

在哪里可以找到Upsource数据库？

默认情况下，数据库存储在提取ZIP存档的文件夹的/ data目录中。如果在安装过程中更改了默认设置 （“配置”向导中的“ 高级设置”区域），则可以在您设置的目录中找到该数据库。

如何记录浏览器会话？

谷歌浏览器：

  1. 打开谷歌浏览器，然后转到发生问题的页面。

  2. 在Chrome菜单栏中，选择查看|开发人员开发工具 。

  3. 在刚刚出现的工具窗口中选择“ 网络”选项卡。

  4. 按“ 清除”按钮从“ 网络”选项卡中清除所有现有日志。

  5. 选中保留日志框。

  6. 寻找一个圆形的“ 记录”按钮。按下一次开始录制–按钮将变为红色。

  7. 现在，在记录网络请求时，尝试重现您之前遇到的问题。

  8. 重现问题后，右键单击网络请求网格上的任意位置，选择“ 另存为具有内容的HAR” ，然后将文件保存在计算机上。

火狐浏览器

  1. 导航到菜单|开发人员网络

  2. 重新加载页面。

  3. 右键单击表中的任意位置，然后选择全部另存为HAR 。

    自版本41起，Firefox便内置了HAR导出功能，因此您无需安装Firebug和NetExport扩展。

  4. 选择目标文件并将其保存在您的计算机上。

如何获得线程转储

要获取其中一项上游服务的线程转储，请执行以下命令：

\ apps \ \ bin \ .bat | sh将线程转储到文件

示例（对于Linux上的Cassandra服务）：

\ apps \ cassandra \ bin \ cassandra.sh将线程转储到文件

该命令将线程转储存储在logs目录中（实际位置已打印到控制台）。

如何获得堆转储

要获取其中一项上游服务的堆转储，请执行以下命令：

\ apps \ \ bin \ .bat | sh转储堆

示例（对于Linux上的Cassandra服务）：

\ apps \ cassandra \ bin \ cassandra.sh转储堆

该命令将生成的堆转储存储在logs目录中（实际位置已打印到控制台）。
