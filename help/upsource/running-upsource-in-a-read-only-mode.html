Running Upsource in a read-only mode

In a read-only mode Upsource users cannot leave comments and edit reviews, so the review data stored in the Cassandra database remains unchanged. Still, code repositories are kept monitored and updated, and can be viewed by the users.

You can use the read-only mode to minimize server downtime when, for example, moving your Upsource installation to another server, upgrading to a newer version, or carrying out some maintenance tasks.

To enable a read-only mode

  1. Log into Upsource as an administrator.

  2. Click the settings icon on the top right of the screen. The Administration view will open.

  3. Click the Enable read-only mode link.

  4. If you want to inform users why Upsource is in read-only mode, enter your explanation in the text field. A banner with your message will be displayed to the users when the log in.

  5. Click Enable.
