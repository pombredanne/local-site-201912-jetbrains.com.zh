字符集（字符编码）问题：文本显示不正确

提交消息或文件内容中的非拉丁字符显示为正方形或未知字符。

解

Upsource中的默认字符编码为UTF-8，因此，如果您的VCS存储库以不同于UTF-8的编码类型存储提交消息或文件，则某些字符可能无法呈现。

在Upsource中更改字符编码以匹配您的VCS存储库的字符编码。

  1. 停止上传 。

  2. 打开位于以下位置的配置文件upsource-frontend.jvmoptions /conf/upsource-frontend目录。如果该文件不存在，则将文件upsource-frontend.jvmoptions.dist重命名为upsource-frontend.jvmoptions 。

  3. 更改属性的值-Dfile.encoding从UTF-8到所需的字符集。如果该属性没有退出，请将其添加到文件末尾。

    例如： -Dfile.encoding=cp1251 。

  4. 保存并关闭文件。

  5. 启动Upsource 。
