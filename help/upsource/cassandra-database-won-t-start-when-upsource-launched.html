上载2019.1帮助

Upsource启动时，Cassandra数据库将无法启动

除非另有说明，否则此处列出的所有命令都是Windows命令。如果您使用的是Linux或Mac OS服务器，只需更换.bat与.sh 。对于macOS或Linux，请确保以非root用户身份启动Upsource。

原因1：Cassandra数据库的内存不足

解

增加堆大小到Cassandra进程，如下所示：

  1. 停止上载： \bin\upsource.bat stop

  2. 运行以下命令：

    \apps\cassandra\bin\cassandra.bat configure -J-Xmx5000m

  3. 启动上游： \bin\upsource.bat start

原因＃2：冲突的JNA版本

java.lang。UnsatisfiedLinkError：无法获得com.sun.jna类的updateLastError方法。本机

解

在启动Upsource之前禁用JNA-运行以下命令：

/apps/cassandra/bin/cassandra.bat配置-J-Djna.nosys = true
