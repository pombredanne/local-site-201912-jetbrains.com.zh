上载2019.1帮助

上游网页（Web UI）永远不会完成加载

使用IIS反向代理时 ，将建立WebSocket连接，但不会发送或接收任何数据包。

解

该问题是由IIS不支持的WebSocket压缩引起的，应该在每次Upsource启动时将其禁用。

要禁用WebSocket压缩，请停止Upsource，然后使用以下命令启动它：

upsource.bat start --J-Dbundle.websocket.compression.enabled=false

（对于Linux或MacOS，请更换.bat与.sh ）

上次修改时间：2019年8月9日
代码情报不可用 Upsource Web UI中的模糊字体
