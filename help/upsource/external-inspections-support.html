外部检查支持

除了由其内置的IntelliJ IDEA引擎提供的Upsource本机代码检查之外，您还可以使Upsource显示从外部检查运行程序上传的检查结果：与TeamCity和SonarQube捆绑在一起的IntelliJ全局检查和ReSharper检查。

显示TeamCity的检查结果

在TeamCity端，请执行以下步骤：

  1. 设置一个构建配置（如果还没有的话）来运行代码检查。

  2. 在该配置中，定义系统属性system.teamcity.dont.delete.temp.result.dir=true

  3. 如果您正在运行IDEA检查 ，则将以下命令行脚本添加到最后的构建步骤：

    zip results-％env.BUILD_VCS_NUMBER％.zip％system.teamcity.build.tempDir％/ inspection * result / * curl -u -X POST https://upsource.yourcompany.com/~inspections/project-id/%env .BUILD_VCS_NUMBER％/ idea –data-binary @ results-％env.BUILD_VCS_NUMBER％.zip

    如果您正在运行ReSharper检查 ，则将以下命令行脚本添加到最后的构建步骤：

    zip results-％env.BUILD_VCS_NUMBER％.zip％system.teamcity.build.tempDir％/ inspection * result / * curl -u -X POST https://upsource.yourcompany.com/~inspections/project-id/%env .BUILD_VCS_NUMBER％/重新共享–data-binary @ results-％env.BUILD_VCS_NUMBER％.zip

    哪里是Upsource中定义的用于与CI服务器集成的身份验证令牌。在“ 集成”选项卡下的“ 创建/编辑项目”页面上找到它，如果从未定义过，则在此处创建它。

    如果您的CI服务器集成已配置了用户（管理员）凭据，则可以选择使用它们代替令牌。

显示SonarQube的检查结果

为了上传和显示SonarQube检查，Upsource使用了一个专用插件，应将其安装到SonarQube中：

  1. 该插件包含在您的Upsource安装软件包中，位于/sonarqube/target/upsource-sonar-plugin-0.1-SNAPSHOT.jar

  2. 为说明安装插件安装到SonarQube 这里 。

  3. 使用以下额外参数运行声纳扫描仪：

    sonar.upsource.url = sonar.upsource.project = sonar.upsource.revision = sonar.upsource.token = sonar.analysis.mode =问题

    哪里是Upsource中定义的用于与CI服务器集成的身份验证令牌。在“ 集成”选项卡下的“ 创建/编辑项目”页面上找到它，或者如果从未定义过，则在此处创建它。
