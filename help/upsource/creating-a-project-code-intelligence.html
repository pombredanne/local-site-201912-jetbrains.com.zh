创建项目-代码智能

启用代码智能： Upsource将能够在您的存储库中执行静态代码分析，并提供相关功能，包括显示代码问题，“查找用法”，“转到声明”等。

请记住，代码智能适用于每个修订版，并且可能导致索引编制和页面加载性能下降，尤其是在大型项目中。

仅当您在使用Java ， JavaScript ， PHP ， Python或Kotlin的项目上工作时，才考虑使用此选项，因为目前仅对这些语言支持代码分析。如果您正在不使用这些语言的项目中工作，则可以将其禁用。

启用代码智能后，指定以下内容：

运行代码检查： Upsource将检测并可视化编译器和运行时错误以及不太严重的问题，例如错别字或无效代码。有关详细信息，请参见在文件中显示代码检查 。

转换项目模型：指定项目中使用的构建系统。

  • 选择不转换 ：

    1. 如果无论构建系统如何，都使用基于IDEA的IDE来维护代码，并且项目的.idea文件夹是否已提交到存储库中（例如，如果您的项目是使用Ant构建的，但您使用的是基于IntelliJ IDEA的代码IDE，然后选择IntelliJ IDEA 。）

      如果您的团队开发人员将他们的IDE配置为每个人都有不同的.idea文件夹，则仍然建议将大多数.idea文件夹内容存储在版本控制中，仅忽略用户特定的设置，例如.idea / workspace.xml和.idea / shelf 。否则，Upsource将无法为您的项目提供代码洞察力。

    2. 对于任何PHP / JavaScript项目

  • 如果选择Maven作为构建系统：

    • 包含pom.xml的目录：输入父pom.xml文件的路径（相对于项目根目录）。如果该文件位于根目录中，则将该字段保留为空白。如果您的项目模块托管在多个存储库中（例如，每个模块位于其自己的存储库中），则可以在一个Upsource项目中全部配置它们，并将父pom.xml保留在第一个存储库中。

    • Maven设置：如果您的项目需要非标准的Maven设置，请在此处粘贴maven-settings.xml的内容（最大20KB）。

      如果您现有的Maven设置可能与默认设置不同，请确保在此字段中指定它们，否则当在其他用户（例如，作为服务）下运行Upsource时，可能无法识别这些设置。

    • Maven配置文件：在此处指定所需的Maven配置文件。

    • 模型转换系统属性：如果您的项目需要其他Java系统属性进行转换，则可以在此字段中指定它们。

  • 如果选择Gradle作为构建系统：

    • 包含Gradle文件的目录：输入包含build.gradle文件的目录的路径（相对于项目根目录）。如果该文件位于根目录中，则将该字段保留为空白。

    • Gradle属性：在此处指定必需的Gradle属性。

    • Gradle初始化脚本：在此处输入Gradle初始化脚本。

    如果您打算在没有包装的情况下处理Gradle项目：

    1. 在/ conf目录中，创建一个文件upsource.user.properties

    2. 将以下行添加到该文件： gradle.home.directory= 。在指定路径中使用双反斜杠（例如， C:\\Users\\Jet\\Gradle ）

    3. 重新启动源

    要在Android Studio项目中启用代码智能：

    • 在您的Upsource服务器计算机上安装Android SDK（包括所有必需的API版本和工具）。

    • 使用包含Android SDK的目录的路径创建环境变量ANDROID_HOME。

    仅基本的Java和Kotlin代码智能可用。不支持特定于Android Studio的检查。

  • JavaScript版本：如果项目中使用了JavaScript，则指定版本。

    为了对package.json中列出的依赖项启用代码智能，应在Upsource服务器上安装Node.js，npm和Yarn。

  • Python支持：如果项目中使用Python，请指定版本。Python应该安装在Upsource服务器上。

  • PHP支持：如果项目中使用PHP，请指定版本。PHP应该安装在Upsource服务器上。

    如果在项目中使用PHP Composer，则可以扩展代码见解。为了使Upsource能够识别它，主Composer配置文件composer.json应该位于项目存储库的根目录中。
