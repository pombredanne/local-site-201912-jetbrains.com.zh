Migrating from Upsource cluster to a ZIP or Docker installation

Migrating from a cluster to a ZIP or Docker installation can be done using the upgrade procedure. It involves creating a backup of your existing cluster installation, then installing a fresh (ZIP or Docker) version using the backup to restore your data. See Upgrading your Upsource installation for details.

Please note, that Upsource system settings, such as port number or base-URL, cannot be migrated from a cluster installation. The new ZIP or Docker instance will not recognise your previous custom settings and start with the default configuration. To adjust the settings, refer to Advanced Upsource tuning.
