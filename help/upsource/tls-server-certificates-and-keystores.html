TLS服务器证书

要使用内置的TLS保护与您的Upsource服务器的连接，您必须具有服务器证书和私钥或密钥库。

对于生产环境，我们敦促您从受信任的证书颁发机构获取SSL证书。

但是，出于测试和评估目的，您可以使用自签名证书。有几种工具可让您创建SSL密钥和证书。此页面描述了使用OpenSSL工具箱创建自签名服务器证书的过程。

上游接受以下格式的证书数据：

  • 密钥库： JKS ，PKCS#12

  • 证书/私钥： PKCS#1 ，PKCS#8

创建自签名服务器证书

使用OpenSSL创建自签名服务器证书

  1. 生成一个新的2048位RSA密钥：

    openssl genrsa -out Upsource_Server_TLS.pem 2048

  2. 为生成的密钥生成证书请求：

    openssl req -new -key Upsource_Server_TLS.pem -out Upsource_Server_TLS_req.csr

    作为Common Name参数，设置服务器的标准域名（FQDN），即upsource.mydomain.com

    该服务将通过生成的服务器证书在此URL上可用。

  3. 我们建议创建版本v3的证书，以确保与大多数浏览器兼容。要生成这样的证书，您需要初步创建一个配置文件，并在证书生成过程中提供它。创建一个文本配置文件（将其命名为v3.ext ），其内容如下：

    AuthorityKeyIdentifier = keyid，发行人basicConstraints = CA：FALSE keyUsage = digitalSignature，不可否认性，keyEncipherment，dataEncipherment

  4. 生成证书（版本v3）：

    openssl x509 -in Upsource_Server_TLS_req.csr -out Upsource_Server_TLS_cert.pem -req -signkey Upsource_Server_TLS.pem -days 3650 -extfile v3.ext

结果：您具有一个自签名服务器证书Upsource_Server_TLS_cert.pem及其私钥Upsource_Server_TLS.pem 。

现在你可以：

  • 在将当前的Upsource实例切换到HTTP模式时，请使用这些文件。

  • 在安装或升级过程中，在配置向导的“ 确认设置”步骤上载这些文件：

    1. 通过按相应的按钮打开HTTPS>私钥和证书设置。

    2. 上传创建的Upsource_Server_TLS.pem文件作为私钥。

    3. 上传创建的Upsource_Server_TLS_cert.pem作为证书。
