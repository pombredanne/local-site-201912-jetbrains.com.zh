移至文件夹重构

重构|移动...
Ctrl+R, O

此重构可帮助您将一种或多种类型或文件移动到解决方案中任何位置的另一个项目或文件夹。如有必要，重构将为您创建新的目标文件夹。移动的类型所需的所有导入都被固定在新位置。所有不再需要的导入都将从原始文件中删除。

请注意，如果将类型移动到另一个项目，则可能缺少所移动类型中使用的名称空间的项目引用。在这种情况下，您将必须手动添加缺少的参考

要将类型或文件移动到另一个文件夹

  1. 通过以下方式之一选择要移动的类型：

    • 在编辑器中，将插入号设置为类型的名称。

    • 在解决方案资源管理器中选择一种或几种类型和/或文件和/或文件夹。

    • 在“ 结构”窗口中选择一种类型。

  2. 请执行以下任一操作：

    • 按Ctrl+R, O然后选择移至文件夹

    • 按Ctrl+Shift+R然后选择移至文件夹

    • 选择重构|在主菜单中移至文件夹 。

    将打开“ 移至文件夹”对话框。

  3. 在“ 目标文件夹”字段中，指定应将所选项目移动到的文件夹：

    • 开始输入目标文件夹或目标项目的名称，然后使用完成建议将其选中。

    • 例如，如果目标位置无效，则在目标项目中使用了其他语言，或者选择了解决方案文件夹，则出现红色图标。这意味着您需要更改目的地。

    • 如果您需要在所选目标位置创建一个新文件夹，请输入\ [文件夹名称] ，然后点击创建此文件夹 。

  4. 如果您移动了几种类型，则可以选择“ 保留文件夹和文件名的层次结构”，以在重构期间保持原始文件文件夹的结构“不变”，或者选择“ 将类放入单独的文件中”，以将每种类型放入相应的单独文件中名称。

  5. 根据新位置，将自动修复已移动项目中的名称空间导入。如果要手动调整名称空间，请清除“ 修复名称空间”复选框。

  6. 当需要更改私有，受保护和内部可见性限制，以便不破坏现有类型的用法时，请选择“ 允许将内部可见性更改为公共” 。

  7. 默认情况下，所有选定项目的所有类型都将被移动。如有必要，您可以清除要保留在原始位置的类型的复选框。

  8. 要应用重构，请单击下一步 。

  9. 如果未发现冲突，JetBrains Rider将立即执行重构。否则，它会提示您解决冲突 。

    重构完成后，它将在任何地方进行的所有更改（包括其他文件）都注册为一个操作。这样您就可以使用“ 撤消”操作Ctrl+Z只需一次击键即可回滚所有这些更改。

如果您撤消此操作，请注意，已移动的内容将从新创建的文件中删除，但文件本身不会被删除。
