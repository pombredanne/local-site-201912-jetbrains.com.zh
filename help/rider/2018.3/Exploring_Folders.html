Finding Assemblies in Folders

JetBrains Rider allows loading entire folders in the Assembly Explorer view When a folder is loaded, JetBrains Rider processes all its subfolders in hunt for files that it is able to decompile, and displays the folder's hierarchy in the Assembly Explorer.

Since JetBrains Rider processes the selected folder recursively, make sure not to add a folder that may contain too many assemblies (e.g. Program Files). It might take long to load all assemblies in JetBrains Rider.

Alternatively to exploring a folder, you can open a particular assembly file.

To open a folder with .NET assemblies

  • Click Explore Folder on the Assembly Explorer toolbar and then select a target folder.

The folder is added to the Assembly Explorer view. You can explore it by expanding the subfolders and all assembly nodes found within it. Double-click on code symbols to decompile and view the corresponding code in the editor editor.

Any assembly found within folders can be displayed at the root level of the Assembly Explorer. To do so, right-click on an assembly node and choose Show at Root Level in the context menu.
