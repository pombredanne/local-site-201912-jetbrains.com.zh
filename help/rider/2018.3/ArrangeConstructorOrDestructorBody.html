Code Inspection: Use preferred body style (convert to constructor or destructor with preferred body style)

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues

  • change its severity level to make the detected issues less or more noticeable

Expression-bodied functions is a C# 6.0 feature. By default, JetBrains Rider automatically detects C# version based on the associated compiler. However, you can specify the target C# version explicitly — press Alt+Enter on the project in the Solution Explorer and use the Language version selector on the Application page of the Project Properties dialog.

Starting from C# 6.0, you can declare function members using expression bodies, which look very similar to lambda expressions. You can use expression-bodied methods and properties to further simplify the syntax of simple implementations.

If you prefer to stick to either expression-bodied or block-bodied implementations for simple function members, JetBrains Rider lets you configure your preferences separately for different kinds of members and maintain the consistency of your preferences throughout your codebase.

For example, let's select Expression body as the preferred style for constructors and destructors. Accordingly, JetBrains Rider suggests the quick-fix To expression body if a simple constructor is written with a block body:

Suboptimal code

After the quick-fix

public class MyClass { private string _name; public MyClass(string name) { _name = name; } }

public class MyClass { private string _name; public MyClass(string name) => _name = name; }

Reverse transformation is available with Alt+Enter .
