Code Inspection: NUnit. Redundant expected result

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues

  • change its severity level to make the detected issues less or more noticeable

A unit test with the void return type and some ExpectedResult in the TestCase attribute may be a sign of one of the following problems:

  • ExpectedResult is there by mistake, in which case you should simply remove it.

  • The test was originally designed to return a result, in which case you need to rewrite the test method.

  • Expected result and its value is written instead of a data value for the test method parameter (e.g. [TestCase(ExpectedResult = 100)]). In this case, you need to remove the ExpectedResult argument name and only leave the value ([TestCase(100)])
