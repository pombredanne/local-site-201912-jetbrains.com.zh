Code Inspection: Use preferred 'var' style (in deconstruction declarations)

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues

  • change its severity level to make the detected issues less or more noticeable

This code inspection checks violations of your preferences for using var or explicit type in local variable declarations. Specifically, it checks variable declarations of deconstruction declarations.

For variable declarations of other types, JetBrains Rider provides similar code inspections, which you can configure separately.

Using implicitly typed local variables (also known as var keyword) introduced in C# 3.0 has become quite popular as it improves readability in many scenarios. By default, ReSharper also encourages using of var keyword, but preferences of its usage are flexibly configurable â€” for example, you can opt for using explicit types in specific cases or everywhere and JetBrains Rider will help you enforce your preferences.

For more information about configuring and enforcing your preferences of using var or explicit types in local variable declarations, see Code Syntax Style: Implicit/Explicit Typing ('var' Keyword).
