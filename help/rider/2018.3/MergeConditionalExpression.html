Code Inspection: Merge conditional ?: expression into conditional access

Starting from C# 6.0, you can use the null-conditional operator (?.) to test for null before performing a member access.

This inspection suggests that you should use a C# 6.0 feature. By default, JetBrains Rider automatically detects C# version based on the associated compiler. However, you can specify the target C# version explicitly — press Alt+Enter on the project in the Solution Explorer and use the Language version selector on the Application page of the Project Properties dialog.

If you use the conditional ?: (ternary) operator for nullability check when accessing symbol's members, JetBrains Rider suggests to replace it with more elegant ?. operator.

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues

  • change its severity level to make the detected issues less or more noticeable

Here is an example of a quick-fix suggested by this inspection:

Suboptimal code

After the quick-fix

string GetAttr(XElement node, string attrName) { var attrNode = node.Attribute(attrName); return attrNode == null ? null : attrNode.Value; }

string GetAttr(XElement node, string attrName) { var attrNode = node.Attribute(attrName); return attrNode?.Value; }

Reverse transformation is available with Alt+Enter .
