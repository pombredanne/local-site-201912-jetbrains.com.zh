Code Inspection: Add parentheses to avoid non-obvious precedence

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues

  • change its severity level to make the detected issues less or more noticeable

In many cases, parentheses in expressions are optional. Sometimes they help you clarify precedence of operations - e.g. if ((a & b) | c), sometimes, however, they do not help at all and just add up to dead code - e.g. var d = a + (b + c);. JetBrains Rider will help you define where optional parentheses are helpful to you, and then either clean up the redundant parenthesis or automatically add missing ones where necessary.

For more information about configuring and enforcing your preferences for optional parentheses, see Code Syntax Style: Optional Parentheses.
