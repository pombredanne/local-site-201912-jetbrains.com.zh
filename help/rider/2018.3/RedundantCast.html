Code Inspection: Redundant cast

According to C# documentation, explicit conversion (cast) is redundant in the following cases:

  • Conversions from derived classes to base classes.

  • Conversions from smaller to larger integral types.

In these cases the cast is done implicitly by compiler. Needless to say, duplicate casts and casts between the same type are also redundant.

JetBrains Rider suggests removing the explicit cast if no information will be lost in the implicit conversion or if the conversion will always succeed.

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues

  • change its severity level to make the detected issues less or more noticeable

Suboptimal code

After the quick-fix

class Base { } class Derived : Base { } class Program { static void Main() { Derived d = new Derived(); Base b = (Base) d; // ... } }

class Base { } class Derived : Base { } class Program { static void Main() { Derived d = new Derived(); Base b = d; // ... } }
