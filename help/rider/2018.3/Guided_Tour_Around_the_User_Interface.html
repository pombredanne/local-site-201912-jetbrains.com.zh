JetBrains Rider 2018.3 Help

UI Overview

Here you can learn how the JetBrains Rider user interface is organized to help you find your way through your working environment.

This page outlines the default (out-of-the-box) IDE interface layout. Note that plugins you are installing may change the way your IDE looks and behaves, for example there can be extra command buttons or menu items.

When you first run JetBrains Rider or have no solution opened, JetBrains Rider displays the Welcome screen which enables quick access to the major entry points. When you opened a solution, JetBrains Rider displays the main window.

The main window consists of areas, which are shown on the picture below, marked with numeric labels:

  1. Main menu and toolbar — contain commands that affect the entire solution or parts of the project, such as opening, creating a solution, refactoring the code, running and debugging applications, keeping files under version control and more.

    The main toolbar duplicates the main menu's essential commands for quicker access. By default, the main toolbar is hidden. To show it, select View | Toolbar from the main menu.

  2. Navigation bar — a quick alternative to the Explorer window. Use it to navigate across your solution and open files for editing.

    Use View | Navigation Bar to hide or show the navigation bar; press Ctrl+F2 to bring the application focus to the navigation bar.

  3. Status bar  — indicates the status of your solution, the entire IDE, and shows various warning and information messages.

  4. Editor  — here you can read, create, and modify your code.

  5. Tool windows — secondary windows that provide access to solution management, search, running and debugging, integration with version control systems, and other tasks.
