Xamarin

JetBrains Rider支持为Android和iOS创建和使用Xamarin应用程序。尽管当前JetBrains Rider还没有为Xamarin表单提供设计器或预览器，但是您仍然可以受益于C＃ ， VB.NET和其他语言的代码分析 ， 编码帮助和调试功能，以及常规的IDE功能，例如作为集成的VCS客户端 。

如果Xamarin依赖的其他工具（例如，Android SDK，Android Emulator）已正确配置，则您将能够直接从JetBrains Rider IDE生成并运行Xamarin应用程序。

Xamarin SDK

要开发Xamarin应用程序，您需要在计算机上安装Xamarin SDK。有两种不同的Xamarin SDK：适用于iOS / Mac和适用于Android 。

Xamarin SDK包含两个部分：

  • 组件与.NET类型为目标平台。例如，.NET类型代表基本OSX NSObject 。使用这些程序集，IDE和编译器可以解析并生成用户代码。

  • 将.NET项目转换为本机应用程序的工具 ，可以在模拟器或物理设备上部署和执行这些应用程序。例如，使用这些工具，为Android构建.apk软件包。

JetBrains Xamarin SDK

JetBrains Rider可以使用不同的Xamarin SDK，例如Visual Studio中的一个。但是，如果您的计算机上没有Visual Studio，则可以使用由JetBrains Rider团队准备和打包的JetBrains Xamarin SDK。

JetBrains Xamarin SDK是Xamarin GitHub源的自定义版本，具有一些改进和附加代码。

当前，与Visual Studio Xamarin SDK相比，JetBrains Xamarin SDK缺少某些功能，但是它正在不断改进。

JetBrains Xamarin SDK可用于Windows和macOS：

Windows上有两个针对以下平台的JetBrains Xamarin SDK构建：

  • 苹果平台（iOS，Mac，tvOS，watchOS）

    用于iOS开发的JetBrains Xamarin SDK在Windows上提供了有限的功能集。例如，当前它不支持 连接到远程Mac并执行完整的构建/部署 。

    适用于Windows上Apple平台的JetBrains Xamarin SDK以.zip文件（〜60Mb）的形式提供，并安装到JetBrains MsBuild目录中：
    ％LOCALAPPDATA％\ JetBrains \ BuildTools 。

  • 安卓系统

    用于Android开发的JetBrains Xamarin SDK提供了可靠的功能集。但是，当前不支持快速部署 。

    Windows上用于Android平台的JetBrains Xamarin SDK以.zip文件（〜700Mb）的形式提供，并安装到JetBrains MsBuild目录中：
    ％LOCALAPPDATA％\ JetBrains \ BuildTools 。

在macOS上有两个针对以下平台的JetBrains Xamarin SDK构建：

  • 苹果平台（iOS，Mac，tvOS，watchOS）

    在macOS上，Apple平台的JetBrains Xamarin SDK提供的功能集几乎与Visual Studio SDK相同，并且支持所有已知方案。

    用于macOS上的iOS开发的JetBrains Xamarin SDK以.dmg文件（〜700Mb）的形式提供，并安装到：

    • /Library/Frameworks/Xamarin.iOS.framework
    • /库/框架/ Xamarin。Mac框架
    • /图书馆/框架/Mono.framework/外部
  • 安卓系统

    用于Android开发的JetBrains Xamarin SDK提供了可靠的功能集。但是，当前不支持快速部署 。

    在macOS上用于Android开发的JetBrains Xamarin SDK以.dmg文件（〜700Mb）的形式提供，并安装到：

    • /库/框架/ Xamarin。Android.framework
    • /图书馆/框架/Mono.framework/外部

在你开始前

Xamarin旨在在各种不同的平台上执行，因此Xamarin开发依赖于几种不同的工具来构建和运行您的应用程序。

在Windows上，您可以为任何平台开发Xamarin应用程序，但是本地构建和运行/调试仅限于Android设备和仿真器。
如果您使用Visual Studio Xamarin SDK，则可以在iOS和macOS上构建和运行应用程序。为此，请配置可在网络上访问的Mac代理 ，然后连接到它（ 工具| iOS | Xamarin Mac代理 ）。

  1. 通过以下方式之一在计算机上安装Xamarin SDK for iOS。

    • 在Visual Studio中安装Xamarin 。请注意，您可以使用免费的Visual Studio社区。如果已经安装了Visual Studio，则必须向其中添加Xamarin支持 。

    • 在以下位置开始安装Xamarin iOS和Mac： 的环境页面 JetBrains骑士设置Ctrl+Alt+S并按照安装向导进行操作。这样，JetBrains Rider将自动下载并安装适用于iOS和Mac的JetBrains Xamarin SDK 。

    • 或者，您可以从GitHub克隆Xamarin开源存储库，进行构建并将其安装在计算机上。这种方式非常复杂，我们不建议您这样做。

  2. 通过以下方式之一安装Android开发工具：

macOS上的Xcode集成

在macOS上开发Xamarin应用程序时，可以在Xcode中编辑一些资源。

您可以使用.storyboard， 的.xib， 的.plist文件或Xamarin macios项目节点的上下文菜单在Xcode中打开它们。

如果以前从未在Xcode中打开过文件或项目，JetBrains Rider将生成一个Xcode项目，如下所示：

  • 生成了xcodeproj项目规范（类似于csproj的项目描述符，但用于Xcode）

  • 继承的所有用户类型的源文件NSObject （在目标C中生成）（表单，委托，视图，控件等）

  • 复制所有资源（图像，设计器文件）

当项目结构准备就绪时，Xcode将自动启动，您可以使用它来编辑资源。每次Rider获得关注时，它都会查找更改（在现有文件，新文件，已删除文件中进行编辑），并将这些更改集成到.NET项目中。它修改了用户类型的.designer.cs部分（继承自NSObject ）并复制回所有更改的资源。

所有与Xcode相关的事件都打印在Xcode控制台工具窗口中，当您在Xcode中打开资源或项目时会出现该窗口。

运行和调试Xamarin应用程序

创建或打开Xamarin项目时 ，JetBrains Rider会自动为解决方案中的每个Xamarin项目创建运行/调试配置 。

如果要调整应用程序启动和执行的方式，则可以编辑和创建新的运行/调试配置。从IDE启动Xamarin应用程序时，可以使用导航栏上的相应选择器来选择应使用的配置：

调试不是由JetBrains Rider创建的Xamarin项目

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到环境 。

  2. 启用Xamarin Android和Xamarin iOS＆Mac支持。

    如果您使用的是Windows，并且已通过Visual Studio安装了Xamarin SDK，则会自动检测到它。否则，JetBrains Rider将建议安装JetBrains Xamarin SDK 。

    JetBrains Xamarin SDK不能与Visual Studio Xamarin SDK一起安装。

  3. 安装Xamarin SDK后，您可以创建Xamarin特定的运行/调试配置 。

  4. 打开“ 运行/调试配置”对话框：

    • 选择运行|从主菜单编辑配置 。

    • 在导航栏可见（“ 视图” |“外观” |“导航栏” ）下，从运行/调试配置选择器中选择“ 编辑配置 ”。

    • 按Ctrl+Shift+Alt+R ，然后按0或从弹出窗口中选择配置，然后按F4 。

  5. 在打开的“运行/调试配置”对话框中，按Ctrl+N或点击 ，然后选择Xamarin。Android ， Xamarin.iOS或Xamarin。Mac从列表中。

  6. 指定目标项目和其他必要的参数，然后单击确定 。

  7. 使用新创建的配置来运行和调试Xamarin项目。
