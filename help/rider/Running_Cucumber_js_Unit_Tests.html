Cucumber.js

Cucumber.js是用于行为驱动的JavaScript开发的测试框架。Cucumber.js测试以人类可读的Gherkin语言编写，并存储在具有功能扩展名的功能文件中。JetBrains Rider与Cucumber.js集成并识别用Gherkin编写的功能，因此您可以直接在IDE中运行Cucumber.js测试。

在你开始前

  1. 下载并安装Node.js。

  2. 如“ 管理插件”中所述，在“ 插件”页面上安装Cucumber.js和Gherkin存储库插件 。

安装Cucumber.js

  • 在嵌入式终端 （ Ctrl+Alt+1 ），键入以下命令之一：

    • npm install cucumber用于项目中的本地安装。

    • npm install -g cucumber用于全局安装。

    • npm install --save-dev cucumber将Cucumber.js安装为开发依赖项 。

    另请参见Cucumber.js官方网站上的Cucumber.js演示 。

您还可以按照npm和Yarn中的描述，在Node.js和NPM页面上安装黄瓜包。

创建测试定义

使用JetBrains Rider，您可以在JavaScript和TypeScript中编写步骤定义。JetBrains Rider会检测并突出显示缺少定义的步骤，并提出生成它们的快速修复方法。

  1. 将插入号放置在没有定义的步骤处，按Alt+Enter ，然后选择创建步骤定义或创建所有步骤定义 。

    或者，将鼠标悬停在该步骤上，并使用弹出窗口中的链接。

  2. 在打开的对话框中，指定新文件的名称以及用于写入定义的语言。

    如果此类文件已经存在，JetBrains Rider将显示一个弹出窗口，您必须在其中选择一个文件以将定义添加到其中，或选择创建一个新文件。

导航

使用JetBrains Rider，您可以在文件和相关测试文件之间跳转，也可以从“ 测试运行器”选项卡中的测试结果跳转到测试。

  • 在编辑器中打开文件，然后选择“ 转到|测试或转到|从上下文菜单中测试受试者 ，或者只需按N/A 。

  • 要从.feature文件中的步骤跳至步骤定义，请按住Ctrl ，将鼠标悬停在该步骤上，然后在该步骤变成链接时单击它。

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源代码 ”。测试文件将在编辑器中打开，并将光标置于测试定义上。

配置语法突出显示

您可以根据自己的喜好和习惯配置支持Cucumber.js的语法突出显示。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |配色方案|黄瓜

  2. 选择配色方案，接受从默认设置继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。
