Code Inspection: Redundant operand in logical condition expression

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues
  • change its severity level to make the detected issues less or more noticeable

If the first operand in a conditional OR operator evaluates to false, the value of the entire OR expression is determined by the second operand. Therefore, a constant false as the first operand is redundant and JetBrains Rider suggests removing the expression with the redundant operand:

Suboptimal code

After the quick-fix

private static void TestOr(string s) { const bool b = false; bool a = b || s.Contains("."); Console.WriteLine(a); }
private static void TestOr(string s) { const bool b = false; bool a = s.Contains("."); Console.WriteLine(a); }
