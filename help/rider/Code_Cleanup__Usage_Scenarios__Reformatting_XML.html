重新格式化XML文档注释

JetBrains Rider可以根据您的喜好重新格式化XML文档注释 。

请注意，JetBrains Rider不会重新格式化普通注释，例如：
// A plain single-line comment
要么
/* A plain multi-line comment */

要重新格式化XML文档注释，请使用代码清除 。您可以使用默认配置文件“ 默认：完全清理” 运行代码清理 ，也可以使用仅针对特定任务的自定义配置文件运行代码清理 。

使用自定义代码清除配置文件重新格式化XML文档注释

  1. 按Ctrl+Alt+S ，或者从主菜单中选择“ 文件” |“在Windows和Linux或JetBrains公司骑士 设置 | macOS上的首选项 。

  2. 在编辑器上代码样式| C＃| XML文档页面上，配置格式设置，使其符合您的编码惯例或标准。

  3. 转至代码清理设置页面： 编辑|代码清理 。

  4. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 选定的配置文件设置”部分中，选中“ 重新格式化嵌入的XML文档注释”复选框，该复选框可用于C＃，VB.NET和JavaScript。（可选）您可以在此配置文件中启用其他代码清除任务。

  5. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。

  6. 选择您要重新格式化XML文档注释的范围：

    • 将插入符号设置在文件中的任何位置，以将XML文档注释重新格式化为该文件。

    • 在解决方案资源管理器中选择一项或多项，以重新格式化这些节点及其子项下文件中的XML文档注释。

  7. 选择代码|在主菜单中的代码清除 。

  8. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  9. 单击确定 。JetBrains Rider将在选定范围内重新格式化XML文档注释。

代码清除不适用于生成的代码。默认情况下，JetBrains Rider提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 编辑器|检查设置|的“生成的代码”页面 JetBrains骑士设置Ctrl+Alt+S 。
