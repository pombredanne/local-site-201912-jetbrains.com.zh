JetBrains Rider 2019.2帮助

咕unt声

JetBrains Rider与Grunt JavaScript Task Runner集成在一起。 JetBrains Rider解析Gruntfile.js文件，识别任务和目标的定义，在树视图中显示任务和目标，让您在树中的任务或目标及其在Gruntfile.js文件中的定义之间导航，并支持运行和调试任务和目标。

使用JetBrains Rider，您可以在专用的Grunt工具窗口中从任务树运行Grunt任务，或者使用Grunt运行配置 ，或者作为启动前任务。JetBrains Rider在“ 运行”工具窗口中显示执行任务的结果。 工具窗口显示Grunt输出，报告发生的错误，列出未找到的软件包或插件，等等。最后执行的任务的名称显示在工具窗口的标题栏上。

在开始之前，请确保您的计算机上装有Node.js。

安装Grunt

要在JetBrains Rider项目中使用Grunt，您需要两个软件包：

  • 全局安装的grunt-cli软件包（Grunt命令行界面），用于执行Grunt命令。

  • 安装了grunt软件包作为开发依赖关系，以构建项目任务树并在编辑Gruntfile.js或Gruntfile.coffee文件时提供编码帮助。从Grunt官方网站了解有关Gruntfile.js的更多信息。

全局安装grunt-cli

  • 在嵌入式终端 （ Ctrl+Alt+1 ），输入：

    npm install -g grunt-cli

在项目中安装Grunt

  • 在嵌入式终端 （ Ctrl+Alt+1 ），输入：

    npm install grunt --save-dev

您还可以按照npm和Yarn中的说明将软件包安装在Node.js和NPM页面上 。

从任务树运行Grunt任务

JetBrains Rider使您可以从Grunt工具窗口中的任务树轻松快速地运行Grunt任务。JetBrains Rider自动创建一个临时运行配置，您可以在以后保存并在需要时使用。

打开Grunt工具窗口

当您在JetBrains Rider会话期间首次构建任务树时，“ Grunt”工具窗口尚未打开。

  • 在“ 项目”工具窗口中选择所需的Gruntfile.js文件，或在编辑器中将其打开，然后从上下文菜单中选择“ 显示Grunt任务 ”。

从Grunt工具窗口构建任务树

  • 在“ 咕unt”工具窗口中，单击在工具栏上，然后从列表中选择所需的Gruntfile.js文件。默认情况下，JetBrains Rider在项目的根目录中显示Gruntfile.js文件。

  • 如果还有另一个Gruntfile.js文件，请单击“选择Gruntfile.js” ，然后在打开的对话框中选择所需的Gruntfile.js文件。JetBrains Rider会在其标题上添加一个具有所选Gruntfile.js文件路径的新节点，并在新节点下构建一个任务树。

重建一棵树

  • 切换到所需的节点，然后单击在工具栏上。

按名称对树中的任务进行排序

  • 请点击在工具栏上，从菜单中选择“ 排序依据 ”，然后选择“ 名称” 。
    默认情况下，一棵树按照在Gruntfile.js中定义的顺序显示这些任务（选项Definition order ）。

运行任务或目标

  • 双击任务或目标。

  • 在树中选择任务或目标，然后按Enter或选择运行从上下文菜单中。

请注意，从任务树运行任务时， 强制执行和详细模式选项不可用。结果，例如，您无法配置JetBrains Rider忽略警告或提供详细的日志。要使用这些选项，请按照下面的根据运行配置运行和调试任务中所述从运行配置运行任务或目标。

运行默认任务

  • 在树中选择根节点，然后从选择的上下文菜单中选择“运行默认值 ”。

运行多个任务或目标

  • 使用多选模式：按住Shift （用于相邻项目）或Ctrl （对于非相邻项目）键，然后选择所需的任务或目标，然后从所选内容的上下文菜单中选择“运行”或“ 调试 ”。

跳转到任务或目标的定义

  • 在树中选择所需的任务或目标，然后从所选内容的上下文菜单中选择“ 跳转到源 ”。

根据运行配置运行和调试任务

除了使用当您从任务树中运行任务或目标时，JetBrains Rider自动创建的临时运行配置之外，您还可以创建和启动自己的Grunt.js运行配置。

创建一个Grunt.js运行/调试配置

  1. 从主菜单中，选择运行|编辑配置 。

  2. 请点击在工具栏上， 然后从列表中选择Grunt.js 。将打开“ 运行/调试配置：Grunt.js”对话框。

  3. 指定运行配置的名称，要运行的任务（使用空格作为分隔符），定义这些任务的Gruntfile.js文件的位置以及全局安装的grunt-cli软件包的路径。

  4. 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

    （可选）指定要传递给Node.js的特定于Node.js的选项参数和环境变量 。

运行任务

  • 从主工具栏上的列表中选择新创建的运行配置，然后单击在列表旁边。JetBrains Rider在“ 运行”工具窗口中显示任务输出。

调试任务

  1. 创建Grunt.js运行/调试配置如上所述 。

  2. 在编辑器中打开Gruntfile.js文件，并在必要时在其中设置断点。

  3. 要启动调试会话， 从主工具栏上的列表中选择所需的调试配置，然后单击列表旁边或选择运行|除错从主菜单。

  4. 在打开的“ 调试工具”窗口中，如检查挂起的程序和单步执行代码中所述，分析暂停的任务执行，单步执行任务等。

将Grunt任务作为启动前任务运行

  1. 通过选择运行|打开运行/调试配置对话框对话框。在主菜单上， 编辑配置 ，然后从列表中选择所需的配置，或者通过单击来重新创建它并选择相关的运行配置类型。

  2. 在打开的对话框中，单击在“ 启动前”区域中，然后从列表中选择“运行Grunt任务 ”。

  3. 在打开的“ Grunt任务”对话框中，指定在其中定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

自动运行Grunt任务

如果您有定期运行的某些任务或目标，则可以将相应的运行配置添加到启动任务列表中。这些任务将在项目启动时自动执行。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 工具|启动任务 。

  2. 在打开的“ 启动任务”页面上，单击在工具栏上。

  3. 从列表中选择所需的Grunt运行配置。配置已添加到列表中。

    如果项目中没有适用的配置，请单击然后选择编辑配置 。然后在打开的“运行/调试配置：Grunt.js”页面上定义具有所需设置的配置 。保存新配置后，它将自动添加到启动任务列表中。
