代码检查：For循环可以转换为foreach循环

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

当您遍历一个集合而不修改它或访问迭代变量时，可以使用foreach而不是for环。的主要好处foreach循环是它允许创建可以与不同集合类型一起使用的方法。的foreach循环支持任何实现IEnumerable接口（实际上，任何具有GetEnumerator方法）。另外，使用foreach允许使用LINQ查询结果，因为许多LINQ提供程序将查询结果返回为IEnumerable 。

两者之间有轻微的性能差异 for和foreach 。

如果您有一种方法可以迭代List在里面for循环，有一天您可能决定让您的方法与其他集合类型一起使用。然后，您将需要使用foreach能够迭代IEnumerable对象。

在下面的示例中，JetBrains Rider确定可以使用foreach代替for并建议这样做：

快速修复后

public void ViewCategories（List Categories）{for（int i = 0; i < category 。计数; i ++）{控制台。WriteLine（类别[i]）; }}
public void ViewCategories（列出类别）{foreach（类别中的字符串c）{控制台。WriteLine（c）; }}

换句话说，如果您使用一种方法，则可以将其用于更广泛的集合类型foreach因为你可以通过IEnumerable对此。下面是这种方法的简单示例：

public void ViewCategories（IEnumerable mycollection）{foreach（mycollection中的var）{控制台。WriteLine（s）; }}
