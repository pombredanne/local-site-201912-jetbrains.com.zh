JetBrains Rider 2019.2帮助

Webpack

JetBrains Rider与webpack模块捆绑器集成。通过支持webpack模块解析和解析别名，此支持提高了JavaScript文件中的编码帮助。对于Webpack版本2和更高版本，JetBrains Rider提供了代码完成和Webpack配置文件中选项的快速文档查找功能。

在开始之前，请确保您的计算机上装有Node.js。

在JetBrains Rider中配置Webpack

  1. 确保webpack列在dependencies要么devDependencies您package.json的对象。

    如果缺少webpack， 打开嵌入式终端 （ Ctrl+Alt+1 ）并输入：

    npm install --save-dev webpack

  2. 在项目根目录或其他地方创建一个配置文件（ New | JavaScript file ）。从webpack官方网站了解更多信息。

  3. 指定要使用的webpack配置文件。默认情况下，JetBrains Rider分析项目根目录中的webpack配置文件。

    要使用另一个Webpack配置文件， 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到语言和框架| JavaScript | Webpack并指定要使用的配置文件的位置。

根据对Webpack配置文件的分析，JetBrains Rider可以了解Webpack的配置并在JavaScript文件中提供编码帮助，请参阅下面的解决模块 。

另请参阅webpack官方网站上的入门 。

编辑Webpack配置文件

对于Webpack版本2和更高版本，JetBrains Rider在webpack.config.js的配置对象中提供代码完成和文档查找。即时提供代码完成。要查看符号的文档，请按Ctrl+Shift+F1 。

JetBrains Rider仅在名称包含以下内容的Webpack配置文件中提供编码帮助： webpack字符串和webpack在package.json中列出。

解决模块

当您打开项目或编辑在“设置/首选项” |“设置”中指定的webpack.config.js时 语言和框架| JavaScript | Webpack ，JetBrains Rider在后台分析配置，并根据收到的信息正确了解项目解析根目录和别名 。由于对项目配置的了解，JetBrains Rider为JavaScript文件中的导入和导出符号提供了更精确的代码完成。结果，一切正常，无需您采取任何措施。
