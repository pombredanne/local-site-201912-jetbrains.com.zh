工具集和构建

使用基于图层的设置机制可以保存在此页面上可配置的首选项。

JetBrains骑士|首选项|构建，执行，部署|适用于macOS的工具集和构建
Ctrl+Alt+S

在此页面上，您可以配置Rider的构建过程 。

项目

描述

工具集
单声道可执行路径

如果要在macOS上构建.NET项目，则需要指定Mono可执行文件。

.NET Core CLI可执行文件路径

使用此选项可以指定.NET Core项目的生成可执行文件。

使用MSBuild.exe版本

使用此选择器，可以选择用于生成解决方案的MSBuild.exe版本。您可以选择一种自动检测到的MSBuild版本，也可以在非标准位置指定自定义MSBuild可执行文件的路径。

构建引擎
使用ReSharper构建

如果选中此复选框，JetBrains Rider将使用以下命令优化构建过程 ReSharper Build ，它跟踪解决方案中的更改，并且仅重建修改后的项目和必要的依赖项。
如果未选中该复选框，则Rider会将整个解决方案版本委派给MSBuild.exe 。如果您的解决方案包括非MSBuild项目或具有自定义生成步骤的项目，则可能需要禁用此选项。

加载解决方案后运行构建

此自解释选项将独立于所选的构建管理首选项来应用。

构建参数
并行使用多达X个进程

使用此选择器，您可以选择用于构建解决方案的并行进程数。

在构建之前还原NuGet软件包

选中此复选框后，JetBrains Rider将为当前解决方案中的所有项目还原NuGet软件包 。请注意，此选项要求在计算机上安装.NET Framework 4.5或更高版本。如果Build，Execution，Deployment |的相应选项允许，Rider将仅还原NuGet软件包。 NuGet设置页面。

将日志写入输出

选中此复选框后，在其下方选择的日志项目将显示在“构建”窗口中。

写入日志文件

选中此复选框后，构建输出将保存到日志文件。使用此复选框下的控件来选择详细级别和保存日志文件的路径。如果所选的日志目录不存在，则以红色显示。但是，当首次登录到文件时，JetBrains Rider将创建不存在的路径。
请注意，记录到文件并不取决于记录到“生成”窗口。

为跳过的项目调用构建前和构建后事件目标

当JetBrains Rider在构建过程中使用启发式方法跳过某些项目时，此类项目将被完全跳过，包括所有其他构建逻辑。
使用此复选框始终对所有项目执行构建前和构建后事件 。

为跳过的项目调用BeforeBuild和AfterBuild目标

当JetBrains Rider在构建过程中使用启发式方法跳过某些项目时，此类项目将被完全跳过，包括所有其他构建逻辑。
使用此复选框始终执行BeforeBuild和AfterBuild MSBuild目标 。

将Visual Studio参数传递给内部版本

如果您选择将解决方案构建委托给MSBuild（未选择“ 使用ReSharper构建”选项），JetBrains Rider将不会通过$(VisualStudioVersion)和$(BuildingInsideVisualStudio)默认情况下，MSBuild的属性。
如果在项目中使用了这些属性，请选中此复选框以覆盖默认行为。

介绍
状态栏：在状态栏中显示构建进度

如果选中此复选框，则在Visual Studio状态栏中显示生成进度。

任务栏：在Windows任务栏中显示生成进度

如果选中此复选框，则Windows任务栏上的Visual Studio图标将显示生成进度。

工具窗口：构建开始时，在工具窗口中打开构建进度视图

选中此复选框后，JetBrains Rider会弹出“ 生成工具”窗口以可视化生成进度和状态。

构建完成后显示构建结果

该选择器使您可以选择构建消息的级别以显示构建结果 。
