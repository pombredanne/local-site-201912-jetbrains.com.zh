允许在调试模式下进行编辑

此复选框仅影响非.NET语言的文件（例如JavaScript或TypeScript文件）。在.NET语言中，您可以编辑和继续 ，此操作由此页下面的“ 启用编辑并继续”复选框控制。

该复选框默认情况下处于禁用状态，因为在调试模式下编辑代码时，该复选框不再与调试器同步，并且可能导致意外结果。
如果启用了“ 显示允许编辑的提示”，则在接受每次编辑尝试中出现的警告后，您将能够编辑文件。否则，在调试模式下，将禁止编辑非.NET语言的文件。

在模块加载时禁用JIT优化（仅.NET / .NET Core）

此选项（默认情况下处于启用状态）禁止实时（JIT）编译器进行优化。这意味着未优化的代码执行速度较慢，但与源代码匹配。
通常，除非要调试无法在非优化代码中重现的问题，否则需要启用此选项。
如果清除此选项，则JetBrains Rider显示的调试信息可能会受到限制。
还要注意，当您附加到正在运行的进程时 ，将仅对附加后加载的模块禁用JIT优化。

启用外部源调试（仅.NET / .NET Core）

启用对外部代码的自动反编译和调试 。
如果禁用此选项，您仍然可以导航到外部代码并在此处设置断点。但是，调试器将忽略外部代码中的断点，并且您将无法进入外部代码。因此，您的调试经验将受到解决方案中源代码的限制。

即时调试器

这组选项使您可以为系统配置默认的即时（JIT）调试器。您可以看到当前用于32位和64位进程的JIT调试器，并使用这些按钮将JetBrains Rider设置为默认调试器，或恢复先前设置为默认的调试器。

将JetBrains Rider设置为默认的JIT调试器并进行处理时Debugger.Launch ，您将看到一个对话框，可帮助您选择如何启动调试器。（1）如果包含应用程序源代码的解决方案已打开，则可以从已打开的解决方案列表中进行选择。
或者，您可以选择安装JetBrains Rider。在这种情况下，调试器将在一个空的解决方案中启动，并且您将能够调试流程的反编译代码 。另外一个复选框使您可以记住所选的选项（2），在这种情况下，以后每次尝试调试器时，都会自动附加它：

内置服务器

项目

描述

港口

使用此旋转框可以指定运行内置Web服务器的端口。默认情况下，此端口设置为端口63342 ，JetBrains Rider通过该端口接受服务的连接。您可以将端口号设置为以1024或更高版本开头的任何其他值。

可以接受外部连接

如果选中此复选框，则可以从另一台计算机访问在指定端口上运行的内置服务器上的文件。
如果清除此复选框（默认情况下），则调试器仅侦听本地连接。

允许未签名的请求

 出于安全原因，默认情况下，拒绝从JetBrains Rider外部对内置服务器上的页面的任何请求，并显示以下授权弹出窗口：要访问请求的页面，请单击“ 将授权URL复制到剪贴板” ，然后将生成的令牌粘贴到浏览器的地址栏中。

但是，此行为可能很烦人，例如，如果不可能进行手动干预，则可能会阻止调试会话。要取消显示授权弹出窗口，请选中“ 允许未签名的请求”复选框。
