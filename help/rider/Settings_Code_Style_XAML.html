在JetBrains Rider设置的此页面上，您可以在XAML中配置代码样式的各个方面。代码样式首选项在以下选项卡中分组：

  • 命名
  • 格式样式

命名

从2018.3版本开始，JetBrains Rider可以从现有代码中自动学习C＃符号命名规则 。如果要手动配置命名规则，请清除“ 使用自动检测到的规则”复选框 编辑器|代码样式| C＃|的命名页面 JetBrains骑士设置Ctrl+Alt+S 。

在此选项卡上，可以为XAML配置符号命名规则 。

JetBrains Rider可帮助您定义，控制和应用代码中符号的命名方式。命名样式是作为一组规则实现的，每个规则都针对具有一组约束的特定标识符（例如，一条规则可以针对静态私有只读字段）。每个规则可以具有一个或多个关联的样式，这些样式定义后缀，前缀，复合词的大写形式，等等。

JetBrains Rider会考虑这些规则产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。JetBrains Rider还可以帮助您检测和修复违反命名规则的情况。如有必要，可以配置或禁用自动检查命名规则。

格式样式

此选项卡可帮助您配置许多设置，这些设置控制JetBrains Rider如何生成新的XAML代码以及如何重新格式化现有代码 。 您可以调整值并检查页面底部的预览窗格，以查看首选项如何影响代码。

您在此选项卡上配置的选项在以下情况下适用：

  • 当代码在编辑时或粘贴后自动格式化时 （您可以在 编辑器|一般|的打字协助页面 JetBrains骑士设置Ctrl+Alt+S 。

  • 当JetBrains骑士产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。

  • 重新格式化现有代码时 。
