.NET编译器平台（Roslyn）分析器

JetBrains Rider提供了所有支持的语言的2000多种代码检查 ，但如果这还不够，JetBrains Rider允许您另外使用.NET编译器平台（Roslyn）分析器。

这些分析器可能有助于代码分析更深入地了解特定框架，因为许多团队正在编写自己的分析器，以为其构建的框架提供其他工具。例如，xUnit.net员工拥有一套基于Roslyn的分析器，可帮助确保诸如xUnit特定的东西Fact方法没有参数。

Roslyn分析仪支持默认情况下处于禁用状态。您可以启用并配置它 编辑器|检查设置|的罗斯林分析仪页面 JetBrains骑士设置Ctrl+Alt+S 。 启用并保存设置后，Rider将扫描已安装的Roslyn分析仪。来自发现的分析器的所有代码检查都将显示在设置页面上，您将能够以与JetBrains Rider本机检查相同的方式更改严重性级别 。

有两种安装Roslyn分析仪的方法：

  • 如果所需的分析仪以NuGet软件包的形式提供，则可以使用JetBrains Rider的NuGet客户端进行查找和安装。
    从2018.1版本开始，这种方式仅适用于.NET Core项目。

  • 您可以在项目的.csproj文件中引用分析器的.dll ，方法是添加 。

所有代码检查功能（包括快速修复和解决方案范围分析）也可用于从外部分析仪进行检查。

外部分析器也可以作为项目构建的一部分运行。如果分析器的默认严重性为警告或错误，它将在构建输出中显示相应的问题。

这是在.NET Core Web应用程序中安装和使用DisableDateTimeNow分析器的示例。DisableDateTimeNow是一个简单的分析器，用于查找DateTime.Now并建议将其替换为DateTime.UtcNow 。
