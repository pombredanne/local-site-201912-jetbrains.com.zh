JetBrains Rider 2019.2帮助

角度的

Angular是用于构建跨平台应用程序的流行框架。JetBrains Rider为Angular提供支持，并在开发过程的每个步骤中为您提供帮助-从创建新的Angular应用程序，处理组件到调试和测试。

在你开始前

  1. 下载并安装Node.js。

要打开按目录结构组织且没有解决方案文件的应用程序，请按Ctrl+O或选择 档案|打开|从主菜单中打开文件或文件夹... ，然后指定应用程序的根文件夹。

运行和调试Angular应用程序

此外，对于使用Angular CLI创建的应用程序，JetBrains Rider会使用默认设置生成两个运行/调试配置：

  • 一个npm配置 ，默认名称为Angular CLI Server 。此配置运行ng serve启动开发服务器并以开发模式启动应用程序的命令。

  • 默认名称为Angular Application的 JavaScript Debug配置。此配置启动调试会话。

运行Angular应用程序

  1. 对于Angular CLI应用程序，从工具栏上的列表中选择Angular CLI Server运行配置，然后单击在列表旁边。

    或者，运行npm start在航站楼 Ctrl+Alt+1或双击start npm工具窗口 （ 视图|工具窗口| npm ）中的任务。

  2. 等待应用程序被编译并且Webpack开发服务器准备就绪。在http：// localhost：4200 /上打开浏览器以查看该应用程序。

开发服务器运行时，更改任何源文件并保存更新后，应用程序将自动重新加载。

调试Angular应用程序

  1. 在代码中设置断点。

  2. 如上所述 ，以开发模式启动应用程序，然后等待应用程序被编译并且Webpack开发服务器准备就绪。

  3. 要查看该应用程序，请在“ 运行工具”窗口中显示的URL上打开浏览器，默认情况下为http：// localhost：4200 / 。

    如果您正在使用其他端口，或者您的应用不是使用Angular CLI创建的，请复制应用实际运行的URL地址，稍后将在调试配置中指定此URL。

  4. 如果您使用Angular CLI创建了应用程序，并且该应用程序在默认端口上运行4200 ，JetBrains Rider会生成默认的Angular Application配置。

    否则，创建一个新的JavaScript调试配置：去跑|编辑配置 ，单击 ，然后从列表中选择JavaScript Debug 。在“ 运行/调试配置：JavaScript调试”对话框中，将保存的URL粘贴到“ URL”字段中并保存配置。

  5. 从配置列表中，选择自动生成的Angular Application配置或新创建的自定义配置，然后通过单击启动调试会话。 在列表旁边。

  6. 遇到第一个断点时，切换到“ 调试”窗口，然后照常进行： 逐步执行程序 ， 停止并继续执行程序， 在挂起时对其进行检查 ，浏览调用堆栈和变量，设置监视，评估变量， 查看实际的HTML DOM ， 等等。

导航到Angular应用程序

在Angular项目上工作时，您必须在不同的组件文件（例如TypeScript，模板和样式文件）之间跳转。使用JetBrains Rider，您可以使用导航栏 ，“ 项目”视图或“ 转到文件”操作来执行此操作Ctrl+Shift+T 。

您也可以下载Angular CLI QuickSwitch插件并将其安装在计算机上 。

还有一种方法是“ 相关符号”弹出窗口。

  1. 在Angular组件文件中，按Ctrl+Alt+F7 。JetBrains Rider显示一个带有相关文件列表的“相关符号”弹出窗口。在TypeScript组件文件中，弹出窗口还列出了导入到该文件中的所有符号。

  2. 要打开文件，请选择它并按Enter 。或者，使用与每种文件类型关联的数字：

    1. 具有组件类的TypeScript文件

    2. 模板

    3. 测验

    4. 款式

使用ng add将新功能添加到Angular应用程序

在使用Angular CLI 6或更高版本的项目中，可以使用Angular Dependency操作来添加新的库。此操作运行ng add命令，用于安装依赖项并使用特殊的安装脚本更新应用程序。请注意，并非所有库都支持使用ng add 。

  1. 选择文件|从主菜单新建或按Ctrl+N在“ 项目”工具窗口中，然后选择Angular Dependency 。

  2. 从列表中，选择要添加的库。该列表显示了可以肯定安装的库ng add 。要安装不在列表中的软件包，请滚动至其末尾并双击“ 安装以上未列出的软件包”链接，然后在打开的对话框中指定软件包名称。

    可以安装的软件包列表ng add在GitHub上可用 。

如果您通过package.json手动管理依赖项，JetBrains Rider仍会识别支持ng add 。当您将此类软件包添加到package.json时 ，JetBrains Rider建议使用ng add 。

如果已经安装了这种依赖关系，您可能仍想使用ng add 。在package.json文件中，选择包的名称，然后按Alt+Enter ，然后单击ng add重新安装 。

查看特定于Angular的检查列表并进行配置

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |检查 。

  2. 展开“ 角度”节点。

  3. 配置检查配置文件和严重性，禁用和取消预定义的检查，以及创建自定义检查。

使用角度材料设计组件

JetBrains Rider识别Angular Material组件和属性并为其提供编码帮助：

  • 组件完成

  • 属性完成

  • 在组件或属性及其声明之间导航（按F12或选择转到|从上下文菜单中声明 ）。

安装角材料

  • 在嵌入式终端 （ Ctrl+Alt+1 ），输入：

    npm install --save @angular/material

    有关详细信息，请参见Angular Material官方网站上的入门 。

配置语法突出显示

您可以根据自己的喜好和习惯配置Angular-aware语法突出显示。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |配色方案|角度模板 。

  2. 选择配色方案，接受从默认设置继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。
