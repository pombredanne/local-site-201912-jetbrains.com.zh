储存库和传入标签

VCS |显示更改视图-存储库/传入
查看|工具窗口|版本控制-存储库/传入
Alt+9

“ 存储库”和“ 传入”选项卡仅适用于非分布式版本控制系统（JetBrains Rider支持的所有VCS，Git和Mercurial除外）。

“ 存储库”选项卡显示在当前项目内VCS根目录下提交到存储库的更改。接收选项卡显示其他团队成员提交到存储库的更改，尚未在本地检出。这两个选项卡都显示存储在历史记录缓存中的信息。显示的更改列表的数量取决于缓存范围 。

每个选项卡包含以下窗格：

  • “ 更改列表”窗格显示更改列表。

  • “ 更改的文件”窗格显示了在选定的更改列表中已修改并提交的文件的列表。

请注意，如果您同时在服务器和本地工作副本上使用SVN 1.5或更高版本，则“ 存储库”选项卡还具有“ 合并信息”窗格，该窗格在其他两个窗格中配置视图并提供对分支之间的集成的控制。

变更列表窗格

该窗格显示已提交并存储在历史记录缓存中的更改列表。单击更改列表时，受所选提交影响的文件将显示在“ 更改的文件”窗格中。

提交的变更列表通常对应于跟踪系统中的问题。您可以直接在“ 更改列表”窗格中的浏览器中打开此类问题。此功能具有以下先决条件：

  • 错误跟踪系统的模式在“ 问题导航 设置 首选项”对话框中指定。

  • 提交消息中提到了相应的发行号。

配置问题导航后，提交消息中的问题编号将呈现为链接。单击此类链接将带您到问题跟踪器的相应页面。

储存库标签

过滤

使用此列表可以隐藏您不感兴趣的变更列表，并且仅查看满足特定条件的变更列表。

提供以下选项：

  • 用户 ：选择此选项可过滤用户的提交。

  • 结构 ：选择此选项可按目标模块或文件夹过滤提交。

  • 客户端 ：选择此选项可以根据提交的计算机筛选提交。

  • 无 ：选择此选项可关闭过滤并返回默认视图。

两个标签

通过...分组

使用此列表可以按照特定条件对变更列表进行分组。

提供以下选项：

  • 日期 ：选择此选项可按日期对提交进行分组。

  • 用户 ：选择此选项可按用户对提交进行分组。

  • 客户端 ：选择此选项可以根据提交的计算机对提交进行分组。

两个标签

搜索

使用此字段输入搜索模式并找到其提交消息与指定字符串匹配的提交。在您键入时，列表会动态缩小以显示更改列表以及与指定模式匹配的提交消息。

要保存搜索模式，请按Enter 。

要查看最近搜索模式的列表，请点击按钮。

要清除搜索模式列表，请点击按钮。

如果涉及多个项目或根，则可以指定几对分支。
