Code Inspection: Incompatible expected result type or incorrect value

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues
  • change its severity level to make the detected issues less or more noticeable

This inspection notifies you about mismatch of the type of a value that you pass to your test via the ExpectedResult argument and the return type of the test method. For example:

[TestCase(ExpectedResult = "100")] // incorrect: type mismatch [TestCase(ExpectedResult = 100)] // correct public int Test1() { return 100; }

Note that NUnit will try to convert the value supplied via ExpectedResult before using it in the test, and JetBrains Rider takes this conversion into consideration before issuing the warning. Note also, that NUnit 2.x and NUnit 3 have different value conversion logic.
