调试外部代码

借助集成的反编译器，JetBrains Rider可以轻松调试外部库代码，该程序可以即时对外部代码进行反编译并将其注入调试器中。

默认情况下，启用外部源调试。如果要调试器忽略库代码，可以将其禁用 。

完整的.NET Framework和.NET Core支持调试外部代码。

您可以按照与自己的代码相同的常规步骤调试外部库或.NET Framework：

  • 您可以启动调试会话，也可以将调试器附加到任何正在运行的.NET或.NET Core进程。
    如果您附加到正在运行的进程，则即使解决方案中没有相关的源代码，也可以对其进行调试。附带调试器后，使用“随处搜索” Ctrl+T在目标应用程序中查找类型和方法。JetBrains Rider将反编译并在编辑器中显示符号，您将能够在其中设置断点 。

  • 您可以在任何时间点暂停执行，并检查程序逐步执行代码的过程 。
    每次您进入外部实现的代码时，JetBrains Rider都会自动对其进行反编译并显示执行点。

  • 您还可以在反编译的代码中设置断点 。
    要从源代码导航到外部代码中的符号，您可以使用几乎所有的导航和搜索功能 -如果导航目标在库代码中，JetBrains Rider将自动反编译并将其显示在编辑器中。跳转到库代码的最简单方法是Ctrl单击库符号的任何用法或按F12 。

  • 分析堆栈跟踪时 ，您可以双击任何堆栈框架，然后JetBrains Rider将直接导航到代码（如果在您自己的代码库中），或者反编译该堆栈框架的代码并在编辑器中将其打开。

要禁用外部代码调试， 清除启用外部源调试 构建，执行，部署|的调试器页面 JetBrains骑士设置Ctrl+Alt+S 。
如果禁用此选项，您仍然可以导航到外部代码并在此处设置断点。但是，调试器将忽略外部代码中的断点，并且您将无法进入外部代码。因此，您的调试经验将受到解决方案中源代码的限制。

如果您想随时清除反编译缓存，则可以删除％USERPROFILE％\中的数据。Rider2019.2 \ config \ resharper-host \ DecompilerCache 。
