外部工具

您可以将独立的第三方应用程序定义为外部工具，然后从JetBrains Rider运行它们。例如，您可能要运行代码生成器或分析器，预处理器或后处理器，数据库实用程序等。

您可以将项目中的上下文信息作为命令行参数（例如，当前选定或打开的文件，项目源路径等）从命令行传递到外部工具，查看该工具产生的输出，配置为启动运行/调试配置之前的工具等。

可与JetBrains Rider一起使用的外部工具有不同类型：

  • 自定义MSBuild工具可帮助您在当前项目之外运行外部工具自定义MSBuild任务。

  • 本地外部工具是在计算机上本地运行的应用程序。

  • 远程SSH外部工具是通过SSH在远程服务器上执行的。

您可以配置自定义工具 （带有自定义参数的MsBuild或外部可执行文件），以在更改特定文件时自动处理它们。例如，JetBrains Rider使用以下命令从.resx文件生成代码ResXFileCodeGenerator工具（此关联是内置的）。

假设您在编辑.aspx文件时需要生成.aspx.designer.cs文件。为此，您可以使用Redesigner ，一个开源的WebForms发生器。

  1. 要将Redesigner作为可执行文件，请从源代码进行构建，或者从JetBrains网站下载使用.NET Framework 4.6.2的Redesigner.exe构建。

  2. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 工具|自定义工具 。

  3. 请点击并提供所有必要的详细信息：

    • 名称： —标识工具的任意名称。

    • 扩展名： —用工具分隔的文件扩展名列表（不带点）。

    • 路径： -要确保正确指定了路径，请单击... ，然后使用文件选择器对话框。

    • Args： —该工具的命令行参数。要使参数适用于不同的解决方案和项目，请使用对话框中说明的宏。请注意，命令行参数通常对文件路径中的空格敏感，因此请不要忘记用引号将文件路径引起来。命令行参数Redesigner.exe可以是这样的： -r "$PROJECT_FOLDER$" -w "$PROJECT_FOLDER$\bin\$PROJECT_NAME$.dll" "$FILE$"

  4. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。

每次您编辑具有指定扩展名的文件时，都会触发配置的自定义工具。您还可以通过右键单击解决方案资源管理器中的文件，然后从上下文菜单中选择“ 执行[工具名称]”来手动触发它。

如果自定义工具提供了一些输出，则可以在backend.log中找到它（ 帮助|在Explorer / Finder中显示批次 ）。

添加本地外部工具

本示例演示了如何将OptiPNG添加为外部工具，并使用它来从JetBrains Rider中快速优化项目中的图像。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择工具|外部工具 。

  2. 请点击并指定以下设置：

    • 名称 ：将在JetBrains Rider界面（“ 工具”菜单和上下文菜单）中显示的工具的名称。

    • 组 ：工具所属的组的名称。您可以选择一个现有组或键入一个新组的名称。

    • 描述 ：对该工具的有意义的描述。

    • 程序 ：可执行文件的名称。如果包含目录不在您的目录中，请使用绝对路径指定它PATH环境变量。

    • 参数 ：传递给可执行文件的参数，就像您在命令行上指定的一样。

    • 工作目录 ：执行工具的当前工作目录的路径。

    就我们而言OptiPNG将与-o4 $FilePath$论点。您可以使用宏，这些宏可以引用项目名称，当前文件路径，PHP可执行文件的路径等等。单击“ 插入宏”按钮将打开“宏”对话框，其中列出了所有可用的宏及其值。

    使用空格分隔各个参数。对包含空格的参数和路径使用双引号。使用反斜杠可转义作为参数或路径一部分的双引号。例如： -Dmy.prop=\"quoted_value\" "second arg" third" "arg

  3. 单击确定以添加该工具，然后应用更改。

要在选定的类文件映像上运行添加的工具，请执行以下任一操作：

  • 在主菜单中，选择工具|图片|优化PNG 。

  • 在“ 项目”工具窗口中右键单击文件，然后选择“ 工具” |“工具”。图片|从上下文菜单中优化PNG 。

  • 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择“ 键盘映射” ，在“ 外部工具”节点下找到“ 优化PNG”操作，然后为其分配快捷方式。使用快捷方式运行该工具。

添加远程SSH外部工具

远程SSH外部工具的配置与本地外部工具相似，但是还定义了在其上执行远程SSH的远程服务器，并且需要凭据才能通过SSH连接到它。有关使用内置SSh终端的详细信息，请参阅“ 运行SSH终端” 。

本示例演示了如何添加date作为在远程服务器上执行并返回当前日期和时间的远程SSH外部工具。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择工具|远程SSH外部工具 。

  2. 请点击并指定以下设置：

    该对话框提供的设置与添加本地外部工具时的设置相同，但也建议选择远程服务器。这可以是您配置的部署服务器之一，也可以是Vagrant框。默认情况下，每次在部署服务器上运行该工具时，JetBrains Rider都会询问您主机，端口和相关的SSH凭据。

  3. 单击确定以添加该工具，然后应用更改。

运行添加的date工具在远程服务器上，请执行以下任一操作：

  • 在主菜单中，选择工具|远程工具|日期和时间 。

  • 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择Keymap ，然后找到Date and time操作，并为其分配快捷方式。使用快捷方式运行该工具。

指定主机，端口和凭据后，JetBrains Rider将通过SSH连接到服务器并运行date命令，将输出返回到JetBrains Rider中的“运行”工具窗口。

网页浏览器

您可以在自己选择的网络浏览器中预览在编辑器中打开的任何HTML文件。

在网络浏览器中打开文件

要打开旨在由网络浏览器呈现的文件（HTML，XML，JSP等），请执行以下任一操作：

  • 按Alt+F2 。

  • 右键单击文件，然后选择“ 在浏览器中打开” 。

  • 在主菜单中，选择查看|在浏览器中打开 。

  • 使用编辑器窗口右上方的浏览器弹出窗口。单击浏览器按钮以打开Web服务器文件URL，或者Shift+Click它打开本地文件的URL。

“ 在浏览器中打开”操作不适用于其他文件类型。但是，您仍然可以使用“ 查找操作”执行它Ctrl+Shift+A 。

默认情况下，JetBrains Rider支持某些最受欢迎的浏览器，这些浏览器会自动配置（如果可用）：

  • 铬

  • 火狐浏览器

  • 苹果浏览器

  • 歌剧

  • IE浏览器

  • 边缘

查看和配置浏览器列表

  • 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择工具| Web浏览器 。

如果使用标准过程安装了浏览器，则“ 路径”字段中的别名应指向正确的位置。如果没有，请指定相应可执行文件的路径。

列表中浏览器的顺序与它们在弹出菜单，上下文菜单和“ 查看”菜单中的顺序相对应。如果禁用浏览器，它将不会出现在弹出菜单，上下文菜单和“ 查看”菜单中。

添加自定义浏览器

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择工具| Web浏览器 。

  2. 请点击并指定浏览器名称，系列和可执行文件或macOS应用程序的位置。

选择默认浏览器

文件的“ 查看”菜单和上下文菜单中的“ 在浏览器中打开”项列出了所有可用的浏览器和默认浏览器，默认浏览器也用于呈现外部资源。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择工具| Web浏览器 。

  2. 从“ 默认浏览器”列表中，选择以下选项之一：

    • 系统默认值 ：使用操作系统的默认浏览器。

    • 首次列出 ：使用列表中的第一个浏览器。

    • 自定义路径 ：使用不在列表中的浏览器。指定可执行文件的位置。

使用自定义配置文件和设置

您可以为Firefox和Chrome系列浏览器配置自定义配置文件。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择工具| Web浏览器 。

  2. 在列表中选择浏览器，然后单击 。

    • 对于Firefox，请指定profiles.ini文件的路径，然后选择要使用的配置文件。有关更多信息，请参阅Firefox浏览器配置文件 。

    • 对于Chrome浏览器，请选择使用自定义用户数据目录并指定位置的用户数据目录 。

      您还可以指定从JetBrains Rider运行Chrome时要使用的其他命令行选项。有关更多信息，请打开chrome://flags在Chrome地址栏中。
