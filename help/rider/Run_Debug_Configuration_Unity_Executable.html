运行/调试配置：Unity可执行

请不要将运行/调试配置与构建配置混淆。

运行编辑配置| | Unity可执行文件

项目

描述

exe路径：

指定游戏可执行文件的路径。或者，您可以再运行一个Unity Editor实例。为此，您需要指定Unity Editor可执行文件而不是游戏的路径。

程序参数：

启动配置时将传递给可执行文件的参数。
多个参数应使用空格分隔。

工作目录：

工作目录会自动设置为所选目标项目的bin / debug路径，但是您可以根据需要进行更改。

环境变量：

在这里，您可以为目标可执行文件指定自定义环境变量 。

运行时变量：

在启动应用程序时，dotnet和Mono均支持设置自定义运行时参数。例如，Mono支持许多运行时参数以自定义应用程序的运行方式。您可以选择要使用的垃圾收集器（ --gc=boehm要么--gc=sgen ），在macOS上，您可以选择要使用的运行时架构（ --arch=32要么--arch=64 ）， 和更多。

使用单声道运行时如果您使用的是Windows，则可以检查应用程序如何与Mono运行时一起工作。JetBrains Rider将使用在上指定的mono可执行文件 构建，执行，部署|的工具集和构建页面 JetBrains骑士设置Ctrl+Alt+S 。
要使用Mono运行时进行调试，您需要：

  • 单声道4.2+

  • 您项目中的可移植PDB — portable在.csproj文件中，或者使用pdb2mdb生成mdb 。

使用外部控制台：

默认情况下，.NET Core和Mono应用程序的输出显示在“ 运行”或“ 调试”窗口中。使用此复选框可以在外部窗口中显示输出。
此复选框不会影响经典的.NET应用程序类型（例如.NET控制台应用程序）-它们的输出始终显示在外部窗口中。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Ctrl+Shift+Alt+R 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

工具列

运行/调试配置的树形视图有一个工具栏，可帮助您管理解决方案中可用的配置以及调整默认配置模板。

项目

捷径

描述

Ctrl+N

创建运行/调试配置。

Ctrl+R, D

删除所选的运行/调试配置。请注意，您无法删除默认配置。

Ctrl+D

创建所选运行/调试配置的副本。请注意，您将创建默认配置的副本。

仅当您选择临时配置时，才会显示该按钮。单击此按钮将临时配置保存为永久配置。

查看和编辑所选运行/调试配置的模板（即默认设置）。模板显示在“ 模板”节点下，并用于新创建的配置。

/

Alt+Up /Alt+Down

在列表中上下移动选定的运行/调试配置。

列表中的配置顺序定义了顺序，当您选择运行/调试配置时，配置以这些顺序出现。

运行/调试配置的默认模板始终按字母顺序排序。

移至新文件夹/创建新文件夹。您可以通过将运行/调试配置放入文件夹来对它们进行分组。

然后，要将配置移动到文件夹中，在文件夹之间或从文件夹中移出，请使用拖动或和纽扣。

要删除分组，请选择一个文件夹，然后单击 。

单击此按钮以按字母顺序对配置进行排序。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Ctrl+N

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在JetBrains Rider中定义它，请添加其定义。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 构建解决方案 ：选择此选项可在启动配置之前构建解决方案。

  • 运行文件监视程序 ：选择此选项可使JetBrains Rider应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

Ctrl+R, D

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
