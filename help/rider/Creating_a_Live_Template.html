创建实时/环绕声模板

本页介绍.NET语言的模板。Web特定模板中描述了可在HTML，JavaScript，TypeScript等中使用的Web特定模板 。

实时/环绕声模板可用于在现有文件中生成代码，并且（如果存在$SELECTION$模板主体中的参数 ），用于包含模板代码的周围代码片段 ，例如try...catch声明。
每个活动模板都由一个快捷方式 （一个简短的字符串，例如foreach ）标识，您可以在编辑器中直接键入以调用该模板。

实时/环绕声模板可用于多种用途，例如用于创建类，DTO或用于单元测试的框架。

实时/周围模板的主体由纯文本和parameters组成，纯文本将原样显示。想法是将这些结合起来以在调用模板时最大化效率。

创建新的实时/周围模板

  1. 根据所使用的语言，将创建的模板应用到的位置，打开 编辑器|实时模板|的[语言]页面 JetBrains骑士设置Ctrl+Alt+S 。

  2. 点击新模板 。

  3. 在“ 快捷方式”字段中指定模板快捷方式 。从模板创建代码时，它用于快速调用模板 ;它也出现在完成列表中 。在“ 描述”字段中，输入模板的可选描述。

    JetBrains Rider不会将每个代码模板保存在单独的文件中。因此， 快捷方式标识符不必是唯一的。但是，请尝试提供可理解的标识符，以免在应用自定义代码模板时造成混淆。

  4. 要更改模板的默认范围 ，请单击“ 可用性”按钮。在出现的对话框中，使用复选框定义可以在何处应用新模板。您可以扩展某些范围以指定详细信息（例如，文件掩码或语言版本）：

  5. 通过键入或粘贴所需的代码并将其与变量混合来创建模板的主体。
    例如，模板主体可以如下所示：

    public class $ className $ {public $ idType $ $ className $ Id {get;设置；} $ END $}
    如您所见，元素有3种类型：

    • text-在模板主体中以纯文本形式编写的任何内容都将在我们调用模板时按原样显示。

    • 预定义变量 -在我们的示例中， $END$变量告诉JetBrains Rider一旦完成应用模板，我们希望将光标定位在哪里。

    • 用户定义的变量 - 键入变量后（前缀和后缀为$ ），变得可用在里面。您可以按照“ 声明模板变量”部分中的说明配置所有定义的变量。
      在我们的案例中，要在调用时建立类名，我们将$className$$一个变量。同样发生$idType因为我们不知道我们想要的ID是什么类型。终于来了Id属性。因为我们希望它类似于类名，所以我们只使用后缀为“ Id”的相应变量。

  6. 使用“在...中使用”选择器，指定模板是否可用于生成新代码，围绕现有代码或同时使用这两者。如果您要将创建的模板用于周围的代码，请添加$SELECTION$参数，代表用户在调用模板之前选择的文本。

  7. 在助记符字段中，您可以指定[0-9]数字，当您用模板包围代码时将使用该数字：

    • 如果指定了一些助记符数字，则按时，模板将在“ 环绕...”下可用Alt+Enter在选择上。

    • 按下时，指定的助记符数字将出现在模板旁边N/A （或选择 代码在菜单中环绕...。然后，您可以按相应的键快速应用此模板。

  8. 配置模板格式选项：

    • 选择重新格式化复选框以使 JetBrains Rider会根据您的格式首选项设置自动重新格式化扩展的代码片段。

    • 选择“ 缩短合格的引用”复选框以进行应用模板后，JetBrains Rider会自动插入名称空间导入指令或删除多余的限定符。如果未选中此复选框，JetBrains Rider将不会插入任何名称空间导入指令。

  9. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。
