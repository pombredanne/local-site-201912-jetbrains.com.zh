仅基于类型的完成

默认情况下，JetBrains Rider会建议符号的完成方式，无论其类型如何。使用这种方法，在复杂情况下，列表显示了多个完成变体。

为了使完成更加精确，请选中该复选框。完成列表将在很大程度上取决于JetBrains Rider的推论。结果，在推理不佳的情况下，列表可能保持空白。

建议变量名称

如果需要，请选中此复选框 JetBrains Rider在声明新变量时为其建议名称。这些建议基于项目，使用的库以及标准API中定义的类，类型和接口的名称。

默认情况下，清除该复选框。

使用参数名称建议类型注释

选择此复选框可为每个完成建议提供有关其类型的信息。

该复选框仅在选中“ 建议变量名称”复选框时可用。

完成扩展方法主体以进行覆盖

默认， 当您想从父类或接口重写方法并从完成建议列表中选择此方法时，JetBrains Rider会自动添加参数，生成一个super()调用，并在可能的情况下添加类型信息。

清除“ 为替代覆盖完成时扩展方法主体”复选框，以禁止在完成过程中自动生成替代覆盖的方法主体。
