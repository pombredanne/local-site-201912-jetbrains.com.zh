第一步

JetBrains Rider是以键盘为中心的产品。它的大多数操作都具有默认的键盘快捷键，如果需要，您可以为其任何命令分配一个自定义快捷键。

在所有预定义的键盘映射中，您很可能会选择以下之一：

  • Visual Studio / Visual Studio（macOS） -建议以下用户使用此按键图：

    • 没有其他IDE的经验；

    • 有使用普通Visual Studio的经验；

    • 具有使用Visual Studio + ReSharper的经验，并且在ReSharper中更喜欢“ Visual Studio”键盘方案 。

  • ReSharper / ReSharper（macOS） -此键盘映射建议给具有Visual Studio + ReSharper经验并且在ReSharper中偏爱“ ReSharper 2.x / IntelliJ IDEA”键盘方案的用户使用。

  • 默认（IntelliJ IDEA） / Mac OS X 10.5+（IntelliJ IDEA） —对于具有IntelliJ IDEA或基于JetBrains IntelliJ平台的IDE的经验，并且更喜欢使用'Default'/'的用户，建议使用此键盘映射Mac OS X 10.5+”键盘映射。

学习此帮助时，请使用帮助页面顶部的“ 快捷方式”选择器，根据您首选的快捷方式在文本中显示快捷方式。

有关更多信息，请参阅配置键盘快捷键。

访问常用命令

Rider主窗口顶部的导航栏没有显示很多命令，但是最不可或缺的命令在您的指尖：

您可以构建解决方案 ，更改和管理构建配置以及运行/调试配置 ， 启动和停止程序 ， 访问VCS操作以及在任何地方搜索 。

你可以按Ctrl+Shift+A快速查找和执行任何命令 。

导航和搜索

JetBrains Rider提供了很多导航和搜索功能 。让我们熟悉其中的一些。

跳到声明

按Ctrl键并将鼠标悬停在您的代码上。您会看到在焦点处定义的所有其他符号都带有下划线。您可以在按住鼠标左键的同时单击任何符号Ctrl直接导航到其声明的键。如果在当前解决方案中定义了符号，JetBrains Rider将打开相应的文件，并将插入符号带到声明中。如果符号是在已编译的库中定义的，JetBrains Rider将为您反编译该声明。

有关更多信息，请参见声明 。

查找用法

要以相反的方向导航，即找到解决方案中使用该符号的所有位置，请按Shift+F12 。JetBrains Rider将迅速找到并显示该符号的所有用法。有关更多信息，请参见查找用法 。

检查可用的导航操作

另一个方便的导航快捷方式是Alt+` 。当您在任何符号上按下它时，JetBrains Rider将为您显示所有可用的导航选项：

有关更多信息，请参见导航到 。

在您的解决方案中找到任何东西

如果您需要在解决方案中找到任何内容，请按Ctrl+T 。建议列表会在您调用此功能后立即出现，并且最初包括您的近期文件和导航的项目。您可以开始键入以查找类型，符号，文件，最近的编辑，最近的文件和最近查看的方法。有关更多信息，请参见在任何地方搜索 。

在解决方案树中找到当前文件

当导航命令将您带到一个新文件时，您可能想查看它在解决方案资源管理器中的位置。只需按Shift+Alt+L然后解决方案资源管理器将滚动到当前文件并将其突出显示。有关更多信息，请参见在解决方案/组装资源管理器中查找当前文档。

在编辑器中编码

当您在编辑器中工作时，大量的代码编辑助手将近在咫尺。这里有几个。

代码完成（IntelliSense）

所有完成功能均支持CamelHumps-也就是说，您可以通过仅输入大写字符来完成任何一项。

Rider的IntelliSense会自动运行 ，但是在允许输入任何有意义的代码后，您始终可以显式调用JetBrains Rider的代码完成功能。

  • 紧迫Ctrl+Alt+Space调用智能完成功能 ，该功能根据表达式的预期类型提供更智能的建议。

  • 紧迫Shift+Alt+Space调用将显示所有与给定前缀匹配的所有类型的类名完成 ，而不管它们属于什么名称空间，并且还将在必要时向当前文件插入适当的名称空间导入指令。

选择和移动代码块

无论您的插入符号在哪里，请尝试按Ctrl+Alt+Right / Ctrl+Alt+Left 。这些快捷方式使您可以依次选择符号，行或代码块，以便可以轻松选择任何所需的表达方式进行复制，剪切或移动。有关更多信息，请参见在编辑器中选择文本 。

如果需要移动选定的代码块，请按Ctrl+Shift+Alt然后使用箭头键将块移动到任何允许的位置。有关更多信息，请参见重新排列代码元素。

Alt + Enter的力量

通常，您会在编辑器的左侧看到许多不同的动作指示符之一。你可以按Alt+Enter看看JetBrains Rider在当前插入符号位置有什么建议：

以下是几个示例：

  • 如果看到红色灯泡或黄色灯泡图标甚至建议按Alt+Enter因为这些操作指示符告诉您JetBrains Rider检测到错误或其他代码问题，并且知道如何解决。

  • 如果看到锤子图标，除非您想在插入符号处修改代码，否则可以忽略它。如果要进行更改，请按Alt+Enter可能会有所帮助。JetBrains Rider提供了数百种上下文操作 ，这些操作可以例如快速更改符号可见性，添加在集合上迭代的代码等。

  • 您也可以按Alt+Enter如果没有任何行动指标。在这种情况下，您可以开始键入内容以快速找到并执行范围内的所有JetBrains Rider操作。

重构代码

就数量，可用性和应用范围而言，JetBrains Rider的重构集远远超过了Visual Studio提供的重构集。

学习和记住JetBrains Rider提供的几十种重构中的每一种都不容易。但是，您可以按Ctrl+Shift+R在代码中的任何符号上，并查看此处提供哪些重构。

产生程式码

JetBrains Rider提供了许多自动生成样板代码的功能，可以帮助您专注于非平凡的任务。例如，您可以调用不存在的方法，并且JetBrains Rider将根据使用情况（考虑返回类型和参数类型）创建此方法。

生成类型成员

当插入符号在类型声明中的任意位置时，请按Alt+Insert 。在打开的弹出菜单中，您可以选择要为该类型生成的项目。JetBrains Rider可以创建构造函数，属性，重写成员等。有关更多信息，请参见代码生成 。

代码风格很重要

使用JetBrains Rider，您可以控制代码中的大多数样式方面： 命名标准 ， 格式设置规则以及许多其他细微的事情（例如修饰符的顺序或是否使用'var'关键字 ）。

JetBrains Rider代码样式功能的默认值是根据Microsoft准则和许多最佳实践选择的。同时，可以更改代码样式的每个微小方面，以适合您的个人或公司偏好。

要应用代码样式规则，请按Ctrl+E, C 。JetBrains Rider将提示您选择两个默认的代码清除配置文件之一：重新格式化代码或在所选范围内应用多个代码样式规则。
