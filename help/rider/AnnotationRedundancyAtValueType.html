代码检查：可空属性使用，声明具有值或无效类型

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

在下面的代码段中， NotNull注释是没有意义的。

公共无效SetAge（[NotNull] int age）{myAge = age; }
Public Sub SetAge（ 年龄为整数）myAge =年龄结束子

原因是int不可能是null ，因此任何表示它为null或不为null的指示都是多余的，可以删除。具有空返回类型的方法也是如此。
