使用EditorConfig

JetBrains Rider支持以EditorConfig格式定义的代码格式样式 ， 代码语法样式和代码检查严重性级别 。

什么是EditorConfig，JetBrains Rider如何扩展它？

EditorConfig是一种配置文件约定，用于在使用相同代码的团队成员之间以及他们可能使用的不同编辑器和IDE之间定义和维护一致的代码样式。样式保存在名为.editorconfig的类似INI的文件中，其中节名称是文件掩码，节内的属性定义了与该掩码匹配的文件的代码样式。

要在Windows资源管理器中创建.editorconfig文件，请创建一个名为.editorconfig的文件。 ，它将自动重命名为.editorconfig 。

正如EditorConfig约定所建议的那样，JetBrains Rider将应用在当前文件目录及其所有父目录中名为.editorconfig的文件中定义的代码样式，直到到达根文件路径或找到带有以下内容的EditorConfig文件为止： root=true 。例如，在.editorconfig文件中指定的文件掩码*Test.cs也考虑在内。

将代码样式设置导出到EditorConfig

如果要通过EditorConfig共享代码样式设置，则可能需要将已经在JetBrains Rider中配置的样式导出到.editorconfig文件。

要将代码样式设置导出到EditorConfig

  1. 按Ctrl+Alt+S ，或者从主菜单中选择“ 文件” |“在Windows和Linux或JetBrains公司骑士 设置 |在macOS上选择“首选项 ”，然后选择“ 编辑器” |“左侧的代码样式 。

  2. 在EditorConfig部分中，单击Export 。这将打开.editorconfig对话框。

  3. 默认情况下，JetBrains Rider将设置写入当前解决方案的根目录中的.editorconfig文件。如有必要，您可以使用对话框顶部的路径选择器更改目标文件。

  4. 在“ 节（文件掩码）”字段中，指定通配符模式以用于具有已生成属性的节。请注意，如果选择导出标准EditorConfig属性，则会根据语言约定将它们放置在其他部分中。

  5. 要进行更详细的配置，可以展开“ 显示其他选项”，然后选择以下选项：

    • 导出具有默认值的设置
      默认情况下，JetBrains Rider仅保存已更改设置的属性。因此，当ReSharper或Rider读取生成的.editorconfig时，将仅应用这些设置并覆盖其他设置。所有其他设置将采用IDE中指定的值。
      如果启用此选项，JetBrains Rider将所有设置保存为当前状态，无论是否更改。这样，您将获得更严格的.editorconfig ，当被ReSharper或Rider读取时，它将覆盖所有代码样式和格式设置。

    • 删除具有不明确值的现有属性
      一些EditorConfig属性可能对应于多个JetBrains Rider特定的属性，从而可以进行更精确的配置。因此，在某些情况下，更一般的属性会与某些与JetBrains Rider相关的特定属性相吻合，并与其他属性相矛盾，这会使该更一般的属性变得模棱两可。
      由于更详细的属性具有更高的优先级，因此这种情况不成问题，并且无需删除具有歧义值的属性。
      可能要删除具有不明确值的属性的唯一情况是ReSharper和Rider将仅读取项目EditorConfig样式，并且要从.editorconfig文件中删除冗余。

    • 导出标准.editorconfig属性 -是否保存标准.editorconfig属性如果其他编辑器和IDE读取了生成的EditorConfig，则这些属性也将适用。

    • 导出跨编辑器特定于语言的属性 -是否保存.NET-coding-convention属性 。如果生成的EditorConfig被其他编辑器和IDE读取，则这些属性也将适用。

    • 导出JetBrains Rider / ReSharper特定的代码样式属性 -是否保存产品特定的代码格式样式 ， 代码语法样式 。

    • 导出特定于JetBrains Rider / ReSharper的检查严重性 -是否为JetBrains Rider的可配置代码检查保存严重性级别 。

  6. 如果在指定位置存在一个现有的.editorconfig文件，则该位置存在的属性值与JetBrains Rider即将生成的属性值之间可能会发生冲突。这些属性将显示为红色，并且如果继续进行导出，则将解决冲突，而使用新生成的冲突。

  7. 点击导出 。JetBrains Rider将在指定位置创建或更新.editorconfig文件。

标准EditorConfig属性支持说明

JetBrains Rider将应用以下标准EditorConfig属性：

  • indent_size
  • indent_style
  • tab_width
  • max_line_length
  • insert_final_newline
