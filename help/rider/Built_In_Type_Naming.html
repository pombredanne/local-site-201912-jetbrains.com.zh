代码语法样式：内置类型引用

C＃类型关键字和.NET Framework（CLR）类型名称对于内置C＃类型可以互换。如果您喜欢对内置类型名称使用一种或另一种样式，JetBrains Rider可以帮助您保持自己的偏好。

您可以为静态成员访问维护单独的样式（例如，CLR类型名称- var str = String.Empty; ）以及本地变量，成员和参数（例如，关键字- void Foo(string str){ } 。

JetBrains Rider可帮助您在现有代码中强制执行C＃内置类型的首选项，并在您使用首选项时将您的首选项考虑在内产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。

强制执行C＃内置类型的首选项

JetBrains Rider会检查所有内置类型引用是否符合您的偏好 ，如果不符合您的偏好 ，它会突出显示此类引用并建议相应的快速修复或范围修复 。默认JetBrains Rider的首选项表明C＃关键字是首选。

如果更改首选项，JetBrains Rider将帮助您使用.NET Framework类型名称：

内置类型命名设置不适用于生成的代码。默认情况下，JetBrains Rider提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 编辑器|检查设置|的“生成的代码”页面 JetBrains骑士设置Ctrl+Alt+S 。

在批量模式下对C＃内置类型强制执行首选项的另一种方法是代码清除 。您可以使用默认配置文件“ 完全清理”或“ 重新格式化并应用代码样式”之一运行代码清理 ，或者创建和运行仅针对特定任务的自定义配置文件 ，如下所述。

通过自定义代码清除配置文件对C＃内置类型引用应用首选项

  1. 按Ctrl+Alt+S ，或者从主菜单中选择“ 文件” |“在Windows和Linux或JetBrains公司骑士 设置 | macOS上的首选项 。

  2. 转至代码清理设置页面： 编辑|代码清理 。

  3. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 选定的配置文件设置”部分中，选中“ 修复内置类型引用”复选框。（可选）您可以在此配置文件中启用其他代码清除任务。

  4. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。

  5. 选择要在其中执行首选项的范围：

    • 将插入符号设置在文件中的任何位置，以强制您对文件进行首选项。

    • 在解决方案资源管理器中选择一项或多项，以在这些节点及其子项下的文件中强制执行您的首选项。

  6. 选择代码|在主菜单中的代码清除 。

  7. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  8. 单击确定 。JetBrains Rider将在所选范围内强制执行您的首选项。

配置用于引用C＃内置类型的首选项

内置的类型命名首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

为C＃内置类型配置首选项

  1. 去 编辑器|代码样式|的C＃页面 JetBrains骑士设置Ctrl+Alt+S ，然后选择“ 代码样式”标签。

  2. 根据您的编码惯例/标准，修改“ 内置类型命名”类别中的设置。

  3. 右列中的选择器允许您设置代码检查的 严重性级别 ，以检测与您的首选项不同的代码。

  4. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。
