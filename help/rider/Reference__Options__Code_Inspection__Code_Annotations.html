JetBrains注释-NuGet官方包装

单击此链接转到JetBrains注释主页 ，您可以在其中使用NuGet获取注释。

自动传播注释

如果选中此复选框，某些JetBrains Rider功能将自动插入注释属性。例如，“检查参数是否为空” 上下文操作将添加[NotNull]该属性除了生成检查无效性的代码外。

默认注释名称空间

允许您选择JetBrains Rider查找代码注释类型的名称空间。如果您在多个命名空间中具有相同的注释类型，则此选项可能会有所帮助。有关更多信息，请参见源代码中的注释 。

具有代码注释属性的名称空间列表 JetBrains Rider自动检测具有有效注释属性类型的解决方案和引用程序集中的名称空间，并显示这些名称空间的列表（如果有）。

注释属性的默认实现在JetBrains.Annotations名称空间，但注释不能使用此名称空间。您可以将声明放在解决方案中的任何位置-JetBrains Rider会自动检测到它们。但是，如果声明位于不同的名称空间中，则应使用注释属性显式指定名称空间，如下所述。
在某些情况下，您的解决方案可能包含注释属性的几种实现，例如，您的类与JetBrains Rider注释类的名称相同（ CanBeNullAttribute ， NotNullAttribute ，等等），或者您正在使用包含JetBrains Rider注释类的第三方程序集。在这种情况下，您可以选择命名空间，JetBrains Rider应该在该命名空间中寻找适当的批注-属性类集。

注释源代码

在本部分中，您可以复制JetBrains.Annotations剪贴板的命名空间，以便您可以将声明嵌入解决方案中的所需位置。
复制注释之前，可以通过两种方式自定义类：

  • 单击内部 ，将注释类设为内部，即只能在插入它们的模块内访问。

  • 点击条件添加[Conditional("JETBRAINS_ANNOTATIONS")]属性归类。这将使编译器忽略代码中的注释属性，这意味着没有对'JetBrains的二进制引用。生成Annotations.dll程序集。但是，您可以在项目中定义“ JETBRAINS_ANNOTATIONS”条件编译符号，以保留元数据中的属性。
