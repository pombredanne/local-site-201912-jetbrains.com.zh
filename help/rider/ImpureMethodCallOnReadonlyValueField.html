代码检查：值类型的只读字段调用不纯方法

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

考虑以下代码：

类Test {struct MyStruct {public int Value;公共无效IncrementValue（）{值+ = 1; }} readonly MyStruct _readonlyStruct = new MyStruct（）; public void RunTest（）{//警告：值为_readonlyStruct的只读字段调用了不纯方法。IncrementValue（）;安慰。WriteLine（_readonlyStruct。值）; // 0}}

我们最终得到0在输出中，虽然看起来IncrementValue()事先应该做到1 。发生这种情况是因为当我们访问一个readonly值类型的字段，将创建它的副本以防止调用方更改该字段。
_readonlyStruct.IncrementValue()在上面的示例中，创建了一个副本_readonlyStruct ，更改其Value到1 ，然后将副本丢弃。

但是，对于不纯净的方法，我们只会遇到此类问题-会更改对象状态的方法，例如IncrementValue()以上。在类似情况下调用纯方法不是问题，因为无论是否在副本上调用它，它都不会改变其状态，无论如何我们都会从该方法中获得预期的结果。

误报

不幸的是，JetBrains Rider并非总是能够确定该方法是否纯净，因此，这种检查有时会产生误报，但方法带有[Pure] （您可以同时使用两个JetBrains。注解。PureAttribute和系统。诊断。合同。PureAttribute ）始终被认为是纯净的 ，不会产生任何警告。
请注意，.NET Framework和其他常用库中的许多方法都带有注释。 [Pure]通过外部注释 。

解决问题

JetBrains Rider没有提供此检查的任何快速修复 。

如果您打算在代码中更改字段的状态，则可以通过删除readonly修饰符。

如果要使字段保持不变，则应重写代码，以免在字段上调用不纯方法。

如果调用的方法是纯方法，并且警告为假肯定，则可以使用以下方法对其进行注释 [Pure]或禁止检查 。
