在Chrome中调试JavaScript

JetBrains Rider为与Chrome一起使用的客户端 JavaScript代码提供了内置调试器。下面的视频和说明指导您逐步入门该调试器的基本步骤。

在开始之前，请按照配置JavaScript调试器中的说明配置内置调试器 。要使您对HTML，CSS或JavaScript代码所做的更改立即显示在浏览器中而无需重新加载页面，请激活实时编辑功能。有关详细信息，请参见HTML，CSS和JavaScript中的实时编辑 。

调试内置服务器上运行的应用程序

JetBrains Rider具有内置的Web服务器，可用于预览和调试应用程序。该服务器始终处于运行状态，不需要任何手动配置。所有项目文件都以根URL http：// localhost： /内置服务器上提供 ，关于项目结构。

开始调试

  1. 根据需要在JavaScript代码中设置断点 。

  2. 打开引用JavaScript的HTML文件进行调试，或在“ 资源管理器”窗口中选择HTML文件。

  3. 从编辑器或选择的上下文菜单中，选择“ 调试”。 。JetBrains Rider生成调试配置，并通过该配置启动调试会话。该文件在浏览器中打开，并且出现“ 调试工具”窗口 。

    要保存自动生成的配置以备将来使用，请选择“ 保存”。 调试会话结束后，从上下文菜单中单击。

  4. 在“ 调试工具”窗口中，照常进行操作： 单步执行程序 ， 停止并继续执行程序， 在挂起时检查它 ， 查看实际的HTML DOM等

默认情况下，调试会话会在带有自定义Chrome用户数据的新窗口中启动。要以熟悉的外观打开新的Chrome实例，请在JetBrains Rider中将Chrome配置为以用户数据开头，有关详细信息，请参见使用默认的Chrome用户数据启动调试会话 。

调试在外部Web服务器上运行的应用程序

通常，您可能需要调试在外部开发Web服务器上运行的客户端JavaScript，例如由Node.js支持的服务器。

开始调试

  1. 根据需要在JavaScript代码中设置断点 。

  2. 在开发模式下运行该应用程序。通常你需要跑步npm start为了那个原因。开发服务器准备就绪后，在浏览器中复制应用程序运行所在的URL地址-您将需要在运行/调试配置中指定此URL地址。

  3. 创建类型为JavaScript Debug的调试配置：从主菜单中，选择Run | Run。编辑配置 ， 点击在工具栏上，然后从列表中选择JavaScript Debug 。 在打开的“运行/调试配置：JavaScript调试”对话框中，指定应用程序运行所在的URL地址。如上面的步骤2所述，可以从浏览器的地址栏中复制此URL。单击确定以保存配置设置。

  4. 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的配置，然后单击在列表旁边。在运行配置中指定的URL地址在浏览器中打开，并且出现“ 调试工具”窗口 。

  5. 在“ 调试工具”窗口中，照常进行操作： 逐步执行程序 ， 停止并继续执行程序， 在挂起时进行检查 ， 查看实际的HTML DOM等。

有关示例，请参见调试React应用程序和调试Angular应用程序 。

调试异步代码

JetBrains Rider支持调试异步客户端JavaScript代码。 JetBrains Rider可以识别异步代码中的断点，并在这些断点处停止，然后让您进入此类代码。一旦在异步函数中遇到断点，或者您进入异步代码，就会有一个新元素Async call from 被添加到“ 调试器”选项卡的“ 框架”窗格中。JetBrains Rider将显示完整的调用堆栈，包括调用者以及异步操作开始的整个过程。

下图显示了JavaScript调试会话的示例。

调试器在第3行（断点）处停止，然后在第5行（断点）处停止。单击“ 跳入”时 ，调试器将在第5行（在function ），然后移至第6行。

默认情况下，异步调试模式是打开的。要禁用异步堆栈跟踪，请设置js.debugger.async.call.stack.depth在注册表中0 。

调试工

JetBrains Rider支持调试Service Worker和Web Worker 。JetBrains Rider识别每个工作程序中的断点，并在“ 调试”窗口的“ 调试 器”选项卡上的“ 框架”窗格中将其调试数据显示为单独的线程。

请注意，JetBrains Rider只能调试专用的工作程序 ，当前不支持对共享工作程序的调试。

调试工人

  1. 在Workers中设置要调试的断点。

  2. 如果使用Service Workers ，请确保已选中“ 调试器”页面上的“ 允许未签名的请求”复选框。否则，您的服务人员可能在调试会话期间不可用：

  3. 如上文调试外部Web服务器上运行的客户端JavaScript中所述，创建类型为JavaScript Debug的调试配置。

  4. 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的配置，然后单击“ 调试”。 。

    在运行配置中指定的HTML文件在选定的浏览器中打开，并且“ 调试”窗口打开，其中“ 框架”列表显示所有Worker ：

    要检查Worker的数据（变量，监视等），请在列表中选择其线程，然后在“ 变量和监视”窗格中查看其数据。当您选择另一个Worker时 ，窗格的内容将相应地更新。
