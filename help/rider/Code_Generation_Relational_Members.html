JetBrains Rider可帮助您使用选定的字段和属性自动重载关系运算符和相关接口的实现。默认情况下，JetBrains Rider将创建一个实现IComparable ，以及可选的运算符> ， < ， ≥和≤以及IComparable也产生了。

生成关系成员

  1. 在编辑器中，将插入号设置为类型名称，或在要插入关系成员的行的类型内。如果插入符号在类型名称上，则生成的代码将添加到类型声明的开头。

  2. 按Alt+Insert或选择 代码从主菜单生成...。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. 在“ 生成”弹出窗口中，选择“ 关系成员” 。

  4. 在出现的“ 生成”对话框中，您将看到属性和字段的列表。选择要用于类型比较的成员。

    如果您的类型中没有字段/属性，或者您没有选择任何字段/属性，JetBrains Rider会根据您的设置抛出新的NotImplementedException() ，返回默认值或将无法编译的代码放入生成的方法的主体中。

    （可选）在对话框中使用以下控件：

    • 字段可以为null-如果您的类型中包含任何可为空的字段或属性，则显示该字段。默认情况下，此复选框处于选中状态，这意味着JetBrains Rider将为所选字段生成空检查。如果不需要空检查，则可以清除此复选框。

    • 字符串比较 -如果您的类型中有任何字符串字段，则会出现。JetBrains骑士将生成string.Compare(string, string, StringComparison)对于选定的字符串，选择器使您可以选择为StringComparison参数生成的值。

    • CompareTo已经存在 -如果实现CompareTo()已经存在，您可以选择是否：

      • 如果已经存在，请替换该方法。

      • 将新生成的方法与现有方法并排放置。

      • 完全跳过生成新方法。

    • 实施IComparable接口 -选中此复选框以生成IComparable接口的实现IComparable除了界面IComparable ，即生成CompareTo(object)除方法CompareTo(T) 。

  5. 单击“ 确定”完成向导。
