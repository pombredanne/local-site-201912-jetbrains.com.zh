版本控制入门

JetBrains Rider包括一个成熟的VCS客户端，该客户端支持所有主要的版本控制系统。让我们以Git为例，介绍一下VCS的基本功能-我们将为我们的解决方案创建一个本地Git存储库，开始跟踪代码中的更改，然后将其与远程存储库相关联，以便我们的解决方案可供其他贡献者使用。

步骤1。获取Git

如果您曾经在计算机上使用过Git，则可以跳过此步骤-Git已安装，Rider会自动找到它。

否则，请从官方页面（ https://git-scm.com/downloads ）下载Git并运行安装程序。

为确保Rider找到了Git可执行文件，请转到 版本控制|的Git页面 JetBrains骑士设置Ctrl+Alt+S在大多数情况下，您将看到检测到的路径，但是如果您的系统环境是自定义的并且Git路径为空，请手动指定路径。正确的路径后，点击测试 ：

第2步。创建一个Git仓库

要使用Git对解决方案进行版本控制，您需要在解决方案的根目录下创建一个Git存储库。JetBrains骑士可以为您做到这一点。

  1. 选择VCS |从菜单启用版本控制集成 。

  2. 选择Git作为版本控制系统，然后单击“ 确定” 。

  3. 启用VCS集成后，JetBrains Rider将询问您是否要通过VCS共享项目设置文件。您可以选择始终添加以与使用JetBrains Rider的其他存储库用户同步项目设置。

    有关更多信息，请参阅共享基于目录的设置 。

第三步确定要版本的版本

一旦您的解决方案与Git存储库相关联，就可以开始对代码进行版本控制。

  1. 首先要去的是“版本控制”窗口的“ 本地更改”选项卡Alt+9 。您将在Unversioned Files下看到所有项目文件。

  2. 按目录对文件进行分组以可视化目录结构。

  3. 忽略您不想版本的文件和目录。这些通常包括bin和obj项目目录以及具有用户特定设置的 DotSettings.user文件。

    Git允许您在VCS根目录及其子目录的.gitignore文件中列出忽略的文件模式。如果VCS根目录中没有.gitignore文件，则可以将“资源管理器”窗口切换到“文件系统”视图，右键单击根节点，选择“ 添加” |“添加”。在“ 新建文件”对话框中输入文件并键入.gitignore 。

    要在Windows资源管理器中创建.gitignore文件，请创建一个名为.gitignore的文件。 Windows会自动将其重命名为.gitignore 。

    现在，当您在VCS根目录中具有.gitignore时，可以右键单击要忽略的任何未版本控制的文件或文件夹，然后选择“ Git”。添加到.gitignore ：

  4. 建议对.idea目录下的.gitignore文件和项目设置文件进行版本控制（即不要忽略）。

第四步。提交您的更改

未跟踪未版本控制的文件。要在Git中记录新文件的添加，您需要添加文件（或使用Git术语登台），然后将其提交到存储库中。在Rider中，您可以将这两个操作作为一个命令来完成-也就是说，您可以提交未暂存的文件-但我们将通过两个单独的命令来完成，以更好地了解文件状态。

  1. 右键单击要暂存的文件或目录（在本例中为整个“ 未版本控制的文件”节点），然后从上下文菜单中选择“ 添加到VCS ”。

  2. 注意文件颜色的变化。棕色表示未暂存，绿色表示新暂存以进行提交。

  3. 现在，当所有文件都暂存后，单击“ 提交”或按Ctrl+Shift+Alt+K —这将在Git中记录当前解决方案状态的快照，我们将来可以在任何时候返回该快照。

  4. 在打开的对话框中，查看已提交的项目并提供有意义的提交消息。

  5. 在对话框中单击提交 。您将看到没有更多未跟踪的更改-一切都保存在您的Git存储库中。

第五步追踪变更

当您的代码位于VCS下时，您始终可以看到更改的内容和方式。

  1. 进行编辑，您会注意到一个蓝色的装订线标记，通知您该行已更改。

  2. 单击装订线标记，以查看更改之前的不同之处。您可以单击“ 回滚行”以还原编辑。

  3. 在“版本控制”窗口的“ 本地更改”选项卡中，所有跟踪和修改的文件将以蓝色显示。您可以打开差异预览并检查所选文件的更改。

  4. 如果您对更改感到满意，请单击“ 提交”或按Ctrl+Shift+Alt+K 。您可以在“ 提交更改”对话框中再次查看更改 。

  5. 您可以在“版本控制”窗口的“ 日志”选项卡中找到所有提交。

  6. 在历史记录中选择一个提交后，您将看到其详细信息以及此提交中所有文件的更改。您还可以选择Show Diff Ctrl+D从文件的上下文菜单中查看该文件的确切更改。

第六步分布式存储

现在，我们已经在本地设置了Git存储库，这已经使我们能够研究更改的历史记录并恢复到任何已提交状态。

但是要充分利用Git，您的本地存储库应与远程存储库（也称为Origin）相关联。此外，如果本地存储库遭到破坏，这还可以使您的代码及其历史记录保持安全，最重要的是，可以与从事同一项目的其他人员进行协作。

让我们使用GitHub（可以说是最受欢迎的Git托管服务）为我们的示例创建一个远程仓库。

  1. 登录到github.com （如果没有，可以创建一个免费帐户）并切换到“ 存储库”视图。

  2. 单击新建 ，为您的存储库指定任何名称，然后单击创建存储库 。

  3. 创建存储库后，您将看到其URL。单击它旁边的按钮，将其复制到剪贴板。

  4. 回到Rider，选择VCS | Git |从菜单中的遥控器 ，并添加使用您复制的URL GitHub的一个新的遥控器。

  5. 选择VCS | Git |从菜单中推送或按Ctrl+Shift+K 。在打开的对话框中，您可以看到到目前为止已完成的提交的列表。单击对话框中的“ 推送 ”以将其传输到遥控器。

  6. 提供GitHub凭据后 ，您的提交将被推送到我们刚刚在GitHub上创建的远程存储库中。

现在，您可以使用GitHub存储库URL从其他计算机访问您的代码，并与其他人共享您的项目。
