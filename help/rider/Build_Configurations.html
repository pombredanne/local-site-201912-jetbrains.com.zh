构建配置

请不要将运行/调试配置与构建配置混淆。

.NET程序的开发单元是项目，然后将其组织到解决方案中。生成管理遵循相同的原理，因此生成首选项存储在项目配置中 ，然后由解决方案配置进行管理。

项目构建配置

项目配置定义了应如何构建项目，例如，是否优化代码和/或包括调试信息。不同的项目类型允许使用不同的构建属性集。

您可以为一个项目具有多个构建配置，以便可以为不同的平台和目的构建相同的代码。
创建新项目时，通常会从项目模板中添加两个项目配置： Debug （为调试而优化）和Release （为发行代码而生成二进制文件）。如有必要，您可以为任何项目添加自定义构建配置。

在构建项目时（作为构建整个解决方案的一部分或仅作为选定项目的一部分），Rider将使用在当前选定的解决方案配置中为此项目指定的构建配置 。

配置项目构建配置的属性

  1. 在“解决方案”窗口中，右键单击所需的项目，然后选择“ 属性” 。

  2. 在打开的“ 项目属性”对话框中，选择一个项目构建配置。

  3. 根据需要编辑所选配置的构建属性，然后单击“ 确定”以应用您的修改：

解决方案构建配置

解决方案配置定义了构建解决方案中的哪些项目，以及哪些项目配置用于解决方案中的特定项目。
与项目类似，新创建的解决方案包括“ 调试”和“ 发布”配置，您可以通过自定义配置对其进行补充。

您可以使用工具栏右侧的相应控件来切换和管理当前解决方案的构建配置：

管理解决方案构建配置

  1. 在工具栏上的构建配置选择器中，选择“ 编辑解决方案配置 ”。

  2. 在打开的对话框中，展开解决方案配置选择器，然后选择编辑 。

  3. 这将弹出另一个对话框，您可以在其中选择如何管理解决方案构建配置。

  4. 选择所需的操作（添加，删除，编辑或复制）。
    如果创建或编辑解决方案构建配置，则可以为其指定名称和平台。
    管理解决方案构建配置的重要之处在于，默认情况下，Rider会修改（即，添加，删除，编辑或复制）项目构建配置，使其与修改后的解决方案配置的名称和平台相匹配。
    如果您不想修改项目配置，请清除对话框中的相应复选框：

  5. 单击“ 确定”以在每个对话框中应用更改。

为当前解决方案配置设置项目配置

  1. 在工具栏上的构建配置选择器中，选择“ 编辑解决方案配置 ”。

  2. 在打开的对话框中，首先选择要使用的解决方案构建配置。

  3. 使用项目旁边的选择器可以指定在当前解决方案版本中应使用哪个项目版本配置：

    如有必要，请单击“ 编辑”以弹出另一个对话框，您可以在其中添加新的或重复的现有项目构建配置。

  4. 如果要从解决方案构建过程中排除某些项目，只需清除项目旁边“ 构建”列中的复选框即可。
    请注意，您仍然可以构建排除的项目-在“解决方案”视图中右键单击该项目，然后选择“ 构建选定的项目” 。

  5. 单击确定以应用更改。
