JetBrains Rider 2019.2帮助

代码检查：在&&或||中合并顺序检查表达

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

此检查建议您应该使用C＃6.0功能。 默认情况下，JetBrains Rider基于关联的编译器自动检测C＃版本。但是，您可以显式指定目标C＃版本-按Alt+Enter在解决方案资源管理器中的项目上，并使用“ 项目属性”对话框的“ 应用程序”页面上的“ 语言版本”选择器。

如果您进行空比较或类似的检查，例如使用HasValue ，在条件“与”（&&）或条件“或”（||）运算的左侧，然后对该符号或其右侧的成员进行其他检查，JetBrains Rider通常建议通过以下方式简化此运算：合并两个顺序检查。在大多数情况下，C＃6.0中引入的空条件运算符（？。）有助于实现此目的。

以下是JetBrains Rider建议的一些转换示例：

  • if (p == null || p.Arguments == null) > if (p?.Arguments == null)

  • if (t != null && t.Value is TimeSpan) > if (t?.Value is TimeSpan)

  • if (arg != null && arg is IConvertible) > if (arg is IConvertible)
