Code Inspection: Merge sequential checks in && or || expressions

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues
  • change its severity level to make the detected issues less or more noticeable

This inspection suggests that you should use a C# 6.0 feature. By default, JetBrains Rider automatically detects C# version based on the associated compiler. However, you can specify the target C# version explicitly — press Alt+Enter on the project in the Solution Explorer and use the Language version selector on the Application page of the Project Properties dialog.

If you do a null comparison or similar checks, for example use HasValue, in the left part of a conditional-AND (&&) or a conditional-OR (||) operation, and then make some other check with this symbol or its members in the right part, JetBrains Rider often suggests to simplify this operation by merging the two sequential checks. In most cases, the null-conditional operator (?.), introduced in C# 6.0, helps to do this.

Here are some examples of transformations that JetBrains Rider suggests:

  • if (p == null || p.Arguments == null) > if (p?.Arguments == null)

  • if (t != null && t.Value is TimeSpan) > if (t?.Value is TimeSpan)

  • if (arg != null && arg is IConvertible) > if (arg is IConvertible)
