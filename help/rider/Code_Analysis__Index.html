JetBrains Rider 2019.2帮助

代码分析

JetBrains Rider可帮助您分析各种级别的代码，从编辑器中的单个语句开始，一直到整个解决方案的体系结构。

查找代码问题

JetBrains Rider通过在以下应用程序中进行2500多次代码检查来提供静态代码分析（也称为代码检查）。 C＃，VB.NET，XAML，XML，ASP.NET，ASP.NET MVC，Razor，JavaScript，TypeScript，HTML，CSS，ResX和构建脚本代码，检测编译器和运行时错误，甚至在编译前就建议更正和改进。

默认情况下，一旦您在编辑器中打开代码文件，JetBrains Rider便会开始分析代码文件，直到您将其完全编辑为止。通过将所有代码检查连续应用于您的代码，可以无声地执行此设计时检查。JetBrains Rider不仅根据其严重性级别在编辑器中突出显示代码问题，还提供了在代码问题之间导航的命令。

如有必要，您可以运行特定项目或整个解决方案的代码检查，并检查在指定范围内发现的问题列表。

此外，您可以启用“ 解决方案范围分析” ，该功能将检测整个解决方案中的所有错误并扩展当前文件中的代码检查列表（例如，它将找到未使用的公共成员）。

如果需要，您可以使用代码注释自定义JetBrains Rider检查代码的方式。

代码问题快速修复

JetBrains Rider可帮助您自动解决大多数发现的代码问题。您只需要按Alt+Enter当插入符号出现在编辑器中突出显示的代码问题上时，请检查建议的快速修复 。

代码探索

JetBrains Rider还提供了一些功能，这些功能不会自动检测代码问题，而是允许您通过对代码进行更深入的调查来自己发现潜在问题。例如，您可以研究呼叫链并找到特定值的始发地和目的地 。有关这些功能的更多信息，请参见“ 代码探索”部分。
