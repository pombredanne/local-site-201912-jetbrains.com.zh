单元测试窗口

查看|工具窗口|单元测试
Alt+8

通过此窗口，您可以在不同的选项卡中管理所有单元测试活动 。

要搜索测试，请在焦点进入此窗口后立即开始输入测试名称或其CamelHumps缩写。

资源管理器标签

使用此选项卡，您可以浏览和运行/调试整个解决方案中所有受支持框架的单元测试。请注意，仅在构建项目后，项目中的单元测试才会显示在选项卡中。当前打开的文件中的测试会自动更新，创建后的新文件会立即显示在单元测试浏览器中。

单元测试设置

开门 构建，执行，部署|的“单元测试”页面 JetBrains Rider设置，您可以在其中配置与单元测试有关的所有设置。

单元测试会话选项卡

这些标签每次都会出现您可以从当前文档或项目或解决方案中 运行测试 。对于每个新的测试组合（ 单元测试会话 ），将打开一个新选项卡。

这些标签是旨在帮助您同时且彼此独立地运行任意数量的单元测试会话。会话可以由测试的任意组合组成。在调试模式下，一次只能运行一个会话。

单元测试树显示属于会话的测试的结构，您可以对其进行过滤以仅显示通过，失败或忽略的单元测试。您可以通过双击导航到任何单元测试的代码。

使用此按钮可以更改输出窗格的位置（右侧或底部），这使您可以分析测试结果，并从失败的测试的输出导航到引发断言或异常的代码行。

选件

您可以为每个测试会话分别配置这组首选项。

  • 构建策略 -指定在执行测试之前是否需要构建。提供几个选项：

    • 自动 （默认选中）—在执行测试之前，JetBrains Rider将重建与当前测试会话相关并且自上次执行以来已发生更改的所有项目。

    • 整个解决方案 -执行测试之前，JetBrains Rider将重建您解决方案中自上次执行以来发生更改的所有项目。

    • 从不 -执行测试之前，JetBrains Rider将不会生成任何内容。如果选择此选项，则必须确保所有相关项目均已构建。

  • 锁定会话 — 锁定当前测试会话，以防止自动向该会话添加新测试。

  • 平台类型 -允许您明确指定测试运行器的平台体系结构，或选择“ 自动”以使用与项目设置相对应的平台体系结构。

  • 诊断 -单击此按钮可以打开上次执行的测试会话的日志。

标题栏上下文菜单和按钮
