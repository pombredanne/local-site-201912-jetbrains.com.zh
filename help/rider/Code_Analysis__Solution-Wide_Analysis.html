解决方案范围内的分析

JetBrains Rider的解决方案范围分析提供了两个相关但独特的功能：

  • 解决方案范围内的错误/警告监视器，使您能够不断跟踪解决方案中的所有错误/警告：编译器错误/警告以及JetBrains Rider自己的检查中检测到的错误/警告。此功能与设计时代码检查一起使用，并带来两个优点：

    • 您不必打开每个文件来确保解决方案不包含错误/警告。

    • 如果删除公共成员或更改其可见性，当前文件中的代码仍然可以正确使用，但是如果该成员在解决方案的其他地方使用该怎么办？例如，假设您将成员的可见性从public至internal ，假设它仅在当前项目中使用。在此阶段，JetBrains Rider在当前文件中未发现任何错误。然后，您打开解决方案范围的分析 ，实际上确实有错误。您可以跳到解决方案中的下一个错误，并发现有人在您的解决方案中的另一个项目中使用了此成员。

    您可以选择在解决方案范围的分析中仅监视错误，或同时监视错误和警告。

  • 解决方案范围内的代码检查 ，有助于发现只能通过分析整个解决方案（未使用的公共类型和成员，未分配的公共字段，可疑的类型转换等）才能发现的问题。 启用解决方案范围的分析后，以与其他问题相同的方式在打开的文件中突出显示由解决方案范围的检查检测到的问题，并且在特定范围内运行代码检查时，这些问题也会显示在“ 检查结果”窗口 中 。

整个解决方案范围内的分析所构建的所有解决方案文件的详细索引也可用于更快地找到用法 。

启用解决方案范围的分析

整个解决方案的分析可能需要花费一些时间来进行解决方案的初始处理，但是一旦完成，它就会在后台运行并且不会消耗太多资源。如果您的解决方案非常大，并且遇到一些性能问题，则可能需要禁用解决方案范围的分析。

切换解决方案范围的分析

  1. 上 编辑器|的“检查设置”页面 JetBrains骑士设置Ctrl+Alt+S ，请确保已选中启用代码分析复选框，然后点击启用解决方案范围的分析 。（可选）您可以在解决方案范围的分析中启用或禁用警告。

  2. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。

或者，您可以单击状态栏右上角的圆圈指示器，然后使用上下文菜单切换整个解决方案范围的分析。

在大型解决方案中，解决方案范围的分析可能会导致某些性能下降，尤其是在初始分析阶段。但是，有几种方法可以提高解决方案范围分析的性能 。

打开解决方案范围的分析后，将进行以下更改：

  • 状态栏指示器显示分析进度。分析完成后，指示器将变为绿色 （如果未发现错误/警告）， 橙色 （如果在解决方案范围内的分析中启用了警告并且解决方案中有警告）或红色 （如果发现了错误）：

  • 圆圈指示器左侧的小区域被分配来显示包含错误/警告（如果有）的解决方案文件的数量。您可以单击此区域以转到解决方案中的下一个错误/警告。

  • 在“解决方案资源管理器”中，JetBrains Rider检测到错误的文件及其所有父项将用红色下划线突出显示：

  • 在设计时检查代码以及检查特定范围内的代码时，都可以使用针对非私有成员的解决方案范围的代码检查 。

    您可以在以下位置查看JetBrains Rider的可配置代码检查的列表： 编辑器|的检查严重性页面 JetBrains骑士设置Ctrl+Alt+S 。整个解决方案的检查都可以通过非私有可访问性标签来识别。

启用解决方案范围的分析后，您可以使用状态栏指示器监视解决方案中的错误/警告，但是您也可以在“解决方案中的错误”窗口中获得有关解决方案中所有错误/警告的详细报告。

您可以通过将发现的问题复制到剪贴板或导出到文件来共享它们。

查看解决方案中发现的错误列表

  1. 请执行以下任一操作：

    • 按Ctrl+Alt+2 。

    • 选择查看|工具窗口|主菜单中解决方案中的错误 。

    • 单击状态栏右上角的圆圈指示器，然后从列表中选择“解决方案中的错误 ”。

  2. 无论哪种方式，都会打开“ 解决方案中的错误”窗口 ，您可以在其中查看检测到的错误/警告列表，并通过双击以下条目导航到相关代码：

即使不打开此窗口，您仍然可以轻松地通过以下方式导航解决方案中的错误： Shift+Alt+Page Down / Shift+Alt+Page Up ，或仅单击状态栏指示器左侧的错误/警告数量。

作为监视解决方案中的错误/警告的替代方法，您可以在解决方案或更小范围内找到所有问题，然后在“ 检查结果”窗口中查看它们。

解决方案范围分析的性能

首次启用解决方案范围分析时，将进行主要计算。根据解决方案的大小，这些计算可能需要几秒钟到几十分钟的时间。您仍然可以继续工作，但是可能会降低性能。

一旦完成这些耗时的全局计算，将仅根据修改内容执行必要的增量分析。

启用后，解决方案范围的分析也可能会大量使用内存。如果您发现您的系统资源遭受了太多损失，则可以执行以下任一操作：

  • 配置执行代码分析时要跳过的文件，文件掩码和文件夹的列表。
  • 编辑包含生成的代码的文件和区域的列表，以简化此类代码的分析。
  • 不需要时暂停分析，仅在再次需要时恢复分析。
  • 从分析中排除警告。

如果关闭解决方案范围的分析然后决定再次打开它 ，JetBrains Rider将不得不重复一些初始计算，这可能会很费时。为避免这种情况，您可以暂停并恢复解决方案范围的分析。

暂停和恢复解决方案范围的分析

  • 在“ 解决方案中的错误”窗口的工具栏上，单击“ 暂停分析” 或继续分析 。

  • 单击状态栏右上角的圆圈指示器，然后在上下文菜单中选择“ 暂停分析”或“ 继续分析 ”。

在解决方案范围的分析中启用警告时，其工作方式如下：只要解决方案中存在错误，就只会显示错误；如果解决方案中存在错误，则仅显示错误。未解决的警告仅在修复了最后一个错误后才会出现。禁用警告后，一旦纠正了最后一个错误，状态栏指示器就会变为绿色。

在解决方案范围内的分析中包含/排除警告

  • 在“ 解决方案中的错误”窗口的工具栏上，单击“ 包含/排除警告” 。

忽略解决方案中的错误/警告

在解决方案开发的某个阶段，您可能知道太多的错误/警告。JetBrains Rider使您可以灵活地忽略某些错误或包含错误的文件，而不必将其包含在解决方案范围的分析结果中。

这些项目将从“ 解决方案中的错误”窗口中的错误列表中隐藏，并且不会影响状态栏指示器的状态。必要时，您可以停止忽略这些项目以显示它们。

若要进行解决方案范围的分析，请忽略特定的错误/警告

  1. 在“ 解决方案中的错误”窗口中 ，选择一个发现的问题，或者，如果您想忽略特定文件中的所有问题，请选择此文件。

  2. 单击忽略错误/警告。 在工具栏上。

  3. 要查看忽略的错误/警告，请单击“ 显示忽略的错误/警告”。 在工具栏上-您将看到解决方案中的所有错误/警告，被忽略的项目显示为划线。即使您选择在列表中显示忽略的项目，它们也不会影响状态栏指示器。

停止忽略解决方案中的特定错误/警告

  1. 在“ 解决方案中的错误”窗口的工具栏上，单击“ 显示忽略的错误/警告”。 。

  2. 选择一个忽略的项目或文件。此类项目显示为划线。

  3. 单击停止忽略错误/警告。 停止忽略所选项目。
