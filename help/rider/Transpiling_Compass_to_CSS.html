指南针应用程序中的Sass和SCSS

JetBrains Rider与Compass框架集成在一起，因此您可以将Compass应用程序中的Sass或SCSS样式表即时编译为CSS。

在你开始前

  1. 下载并安装Ruby 。

  2. 如管理插件中所述，在“ 插件”页面上安装并启用File Watchers存储库插件。

安装指南针

  • 在嵌入式终端 （ Ctrl+Alt+1 ），输入：

    gem install compass

    Compass安装在存储Ruby可执行文件和gem.bat文件的文件夹中。

从JetBrains Rider中的Compass应用程序开始

如果您还没有Compass应用程序，则可以创建它或将Compass支持添加到现有的JetBrains Rider项目中。无论哪种情况， 都会生成conf.rb配置文件。

创建一个新的指南针应用程序

  • 在嵌入式终端 （ Ctrl+Alt+1 ），输入：

    compass create

将Compass添加到现有的JetBrains Rider项目中

  1. 用要使用Compass的代码打开文件夹（在“欢迎”屏幕上单击“ 打开 ”，或在主菜单上选择“ 文件” |“打开 ”）。

  2. 在嵌入式终端 （ Ctrl+Alt+1 ），输入：

    cd 罗盘初始化

打开计算机上已经存在的应用程序源

  • 单击“ 欢迎”屏幕上的“ 打开 ”，或选择“ 文件” |“共享”。从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中签出应用程序源

  1. 单击“ 欢迎”屏幕上的“ 从版本控制中签出”，或选择“ VCS |从主菜单中的“ 版本控制”中签出 。

  2. 从列表中选择您的版本控制系统。

  3. 在打开的特定于VCS的对话框中，键入您的凭据和存储库以从中检出应用程序源。

在应用程序中激活Compass-aware支持

  1. 打开.sass或.scss文件，点击旁边的红色灯泡到@import 'compass'声明或新闻Alt+Enter ，然后从建议列表中选择“ 配置指南针 ”。

  2. 选中“ 启用Compass支持”复选框，并指定Compass可执行文件和conf.rb配置文件的路径。

或者， 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到语言和框架|样式表|指南针 。

编译Sass和SCSS

要即时编译代码，您需要将编译器配置为JetBrains Rider File Watcher 。

您也可以从命令行使用编译器，或者将其配置为第三方工具，有关详细信息，请参阅配置第三方工具 。

创建文件监视程序

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），点击“ 工具”下的“ 文件监视程序 ”。打开的“ 文件监视程序”页面显示已配置的文件监视程序的列表。

  2. 请点击或按Ctrl+N 。根据要使用的工具，从列表中选择罗盘sass或罗盘scss预定义模板。将打开“ 新建监视程序”对话框。

  3. 在“ 程序”字段中，指定可执行文件的路径：

    • Windows的罗盘

    • Unix和macOS 指南针

  4. 在“ 参数”字段中，根据所使用的操作系统键入以下内容之一：

    • compile $UnixSeparators($ProjectFileDir$)$处理整个目录

    • compile $UnixSeparators($FilePath$)$处理单个文件

    • compile $ProjectFileDir$处理整个目录

    • compile $ProjectFileDir$ $FilePath$处理单个文件

    • compile $ProjectFileDir$处理整个目录

    • compile $ProjectFileDir$ $FilePath$处理单个文件

  5. 按照文件监视程序中所述进行操作。

编译代码

当您打开文件时，JetBrains Rider会检查当前项目中是否有适用的文件监视程序。如果已配置但禁用了此类文件监视程序，JetBrains Rider将显示一个弹出窗口，通知您有关已配置的文件监视程序并建议启用它。

如果在当前项目中配置并启用了适用的文件监视程序，JetBrains Rider会在“ 新建监视程序”对话框中指定的事件后自动启动它。

  • 如果选中了“ 自动保存编辑的文件以触发监视程序”复选框，则在对源代码进行任何更改后都会立即调用文件监视程序。

  • 如果清除了自动保存编辑的文件以触发监视程序复选框，则文件监视程序会在保存时启动（ File | Save All ， Ctrl+Shift+S ），或者当您从JetBrains Rider移动焦点时（关闭框架时）。

JetBrains Rider使用生成的输出创建一个单独的文件。该文件具有源Sass或SCSS文件的名称以及扩展名.css 。在“ 新建监视程序”对话框的“ 刷新的输出路径”字段中定义了生成文件的位置。但是，在项目树中 ，它们显示在源文件下，该文件现在显示为节点。
