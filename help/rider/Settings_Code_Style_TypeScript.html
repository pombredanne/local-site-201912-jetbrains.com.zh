制表符和缩进

使用制表符

  • 如果选中该复选框，则制表符将用于缩进和代码重新格式化。

  • 如果清除该复选框，则JetBrains Rider将使用空格而不是选项卡。

智能标签

压痕由两部分组成。一部分来自嵌套代码块，另一部分由对齐确定。

  • 如果选中此复选框，则嵌套产生的零件将同时包含制表符和空格（如果需要），而由对齐方式定义的零件仅包含空格。

  • 如果清除此复选框，则仅使用选项卡。这意味着在重新格式化一组适合指定制表符大小的空格后，将自动用制表符替换，这可能会导致中断精细对齐。

如果选中了“ 使用选项卡字符”复选框，则“ 智能选项卡”复选框可用。

标签尺寸

在此字段中，指定选项卡中可以容纳的空格数。

缩进

在此字段中，为每个缩进级别指定要插入的空格数（如果选中了“ 使用制表符”复选框，则为制表符 ）。

连续缩进

在此字段中，在表达式，方法声明和方法调用中，指定要在数组的元素之间插入的空格数（如果选中了“ 使用制表符”复选框，则为制表符 ）。

缩进空行

如果选中此复选框，JetBrains Rider会将缩进保留在空行上，就好像它们包含一些代码一样。如果清除此复选框，JetBrains Rider会删除制表符和空白行。

缩进链接方法

在函数声明中，链中的第二个方法和其他方法显示在单独的行上。

  • 选中此复选框后，链中的第二个方法和其他方法将与第一个调用对齐。

  • 清除复选框后，链中的第二个方法和其他方法将与调用它们的对象对齐。

缩进组中所有链接的呼叫

仅当选中缩进链接方法复选框时，此复选框才可用。

空间

使用此选项卡可以指定您希望JetBrains Rider自动在何处插入空格。选中相关位置的描述旁边的复选框，然后在“ 预览”窗格中检查结果。

包装和大括号

在此选项卡中，定制在重新格式化源代码时 JetBrains Rider将应用于各种代码构造的异常，括号放置和对齐选项。在“ 预览”窗格中检查结果。

对齐优先于缩进选项。

硬包装

在此字段中，指定元素右侧所需的空格数。如果您接受默认选项，则使用全局设置中的值。

包装打字

在此字段中，指定如何将编辑的文本适合指定的“ 硬包装”字段。

  • 默认值 ：选择此选项可使用全局设置中的“ 键入时自动换行”值。

  • 是 ：选择此选项可以使用“ 右边距”字段中的值。

  • 否 ：如果选择此选项，则一行可能会超出右边距中指定的值。

视觉指南

在此字段中，指定多个右边距。您可以保留默认值，也可以输入空白。如果要指定多个边距，请输入用逗号分隔的数字。

重新格式化时保持

使用复选框配置JetBrains Rider在重新格式化源代码时将产生的异常。例如，默认情况下，“ 换行符”复选框处于选中状态。如果您的代码包含比标准约定短的行，则可以通过在重新格式化之前禁用“ 换行符”复选框来转换它们。

包装选项

包装样式适用于在左侧窗格中指定的各种代码构造（例如，方法调用参数或赋值语句）。

不要包裹

选择此选项时，不应用特殊的换行样式，嵌套的对齐方式和花括号设置将被忽略。

如果长条包装

选择此选项可使用适当的缩进来包装超出右边距的行。

总是包装

选择此选项可将所有元素包装在列表中，以便每行有一个带有适当缩进的元素。

砍多久

选择此选项可将元素包装在超出右边距的列表中，以便每行有一个具有适当缩进的元素。

对齐选项

多行时对齐

如果选中此复选框，则代码构造将从下一行的同一列开始。否则，代码构造的位置由当前的缩进级别确定。

在下一行

换行时，选中此复选框可将指定的一个或多个字符移动到下一行。

新行上的“ else”

使用此复选框将相应的语句或字符移动到下一行。

之后的新行

选中此复选框可将指定字符后的代码移动到新行。

特别待遇

如果选中此复选框， else if语句位于同一行。

除此以外， else if语句移动到相应缩进级别的下一行。

缩进案例分支

如果选中此复选框，则case语句位于相应的缩进级别。除此以外， case语句与switch 。

对象

从列表中选择如何对齐对象：

  • 不对齐 ：连续行中的属性将不对齐。

  • 在冒号上 ：连续行中的属性将与冒号对齐。

  • 在值上 ：连续行中的属性将与值对齐。

变量声明

选择以下选项之一为等号配置对齐方式：

  • 不对齐 ：等号不对齐。

  • 多行时对齐 ：等号在多行中签名var语句通过插入其他空格来对齐。

  • 分组时对齐 ：等号签到多个var语句通过插入其他空格来对齐。

ES6导入/导出

对齐'from'子句：选中此复选框后，JetBrains Rider对齐import和export ECMAScript 6代码中的语句自动使您的代码更易于阅读和维护。在“ 预览”窗格中比较带有对齐和不带对齐的代码片段的外观。

启用此选项后，JetBrains Rider将即时调整新代码。现有import和export通过按以下命令重新格式化代码后，语句将对齐Ctrl+Alt+Enter ，有关详细信息，请参见“ 重新格式化源代码”部分。

大括号放置选项

大括号放置样式

使用此列表可以在类声明 ， 方法声明 ， 函数声明和其他类型的声明中指定左括号的位置。可用的选项有：

  • 行尾 ：选择此选项可将开括号放在声明行的末端。

  • 如果被换行，则换行 ：选择此选项可将开括号放在多行声明行之后的行首。

  • 下一行 ：选择此选项可将左括号放在声明行之后的行首。

  • 下一行移位 ：选择此选项可在将声明行移位到相应的缩进级别后的行处放置开括号。

  • 每行都移位的下一行 ：选择此选项可在将声明行移动到相应的缩进级别之后的行处放置开括号，并将下一行也移到下一个缩进级别。

大括号

从此列表中选择括号的引入方法if ， for ， while和do () while陈述。可用的选项有：

  • 不强制 ：选择此选项可自动禁止插入括号。

  • 多行时：如果语句占用多行，请选择此选项以自动插入大括号。请注意，JetBrains Rider会分析整个语句中的行数，而不只是分析其条件。

    如果（true）返回false；

    JetBrains Rider将自动插入括号：

    如果（true）{返回false； }

  • 始终 ：选中此复选框后，JetBrains Rider总是自动插入括号。

代码生成

在此选项卡上，为生成的代码配置代码样式。

使用“公共”修饰符

使用此复选框可以在生成的代码中插入或省略公共访问修饰符。
例如，在通过以下方法生成公共方法期间：

类Test {public test（）：void {var x = 1; }}

  • 如果选中此复选框，则公共访问修饰符将自动插入到生成的代码中：

    类Test {public test（）：void {this.extracted（）; } public extract（）{var x = 1; }}

  • 如果清除此复选框，则在代码生成过程中将省略public access修饰符：

    类Test {public test（）：void {this.extracted（）; } extract（）{var x = 1; }}

请参见《 TypeScript语言手册 》的“私有/公共修饰符”一章。

命名约定

在这方面：

  • 配置或接受默认前缀，这些默认前缀将自动添加到生成的字段和属性的名称中。

  • 选择所生成文件名称的样式。例如，当JetBrains Rider建议将类移动到新的自动生成的文件时，该文件的建议名称是根据所选样式从类名称生成的。从保持类名和包含文件符合性中了解更多信息

注释码

在此区域中，为生成的注释配置代码样式。

  • 在JSDoc中包括类型 ：选中此复选框可为以下类型添加类型@param ， @return以及其他标签。请注意，类型信息对于TypeScript是多余的，因为类型是从代码本身中检索的。因此，默认情况下将清除该复选框。

  • 第一列的行注释 ：选中此复选框以在第一列开始行注释。清除复选框后，行注释在代码中对齐。

  • 在注释开始处添加一个空格 ：选中此复选框时，将在行注释字符和注释行的第一个字符之间插入一个空格。

    选择第一行的行注释时，“ 在注释开始处添加空格”复选框不可用。

  • 清除此复选框后，将为每个导入的符号生成一个单独的import语句。

使用相对于tsconfig.json的路径

选中此复选框后，JetBrains Rider使用tsconfig.json文件作为根来计算导入路径。清除此复选框后，JetBrains Rider将计算相对于项目根目录的导入路径。

使用目录导入（节点样式的模块解析）

  • 选中此复选框后， 将根据Node.js模块解析策略生成import语句。

不完全从

在此字段中，指定在自动导入符号期间JetBrains Rider应该跳过的确切路径。相反，JetBrains Rider将寻找替代路径来导入符号。

  2. 在打开的“ 更改模块”对话框中，单击并在“ 添加模块”对话框中指定模块名称。要从列表中删除模块，请选择它并单击 。

对导入的成员进行排序

  • 清除此复选框后，合并的import语句中的成员始终以导入顺序以逗号分隔列出。

按模块对导入进行排序

  • 选中此复选框后，在运行“代码|”时， 导入语句将按模块名称的字母顺序重新排序。优化进口 。

  • 清除此复选框后， 导入语句将始终按生成的顺序显示，并且在运行代码|后不会更改此顺序。优化进口 。

匹配规则

空规则

使用此区域可以创建新的匹配规则或编辑现有规则。您可以从以下过滤器中进行选择：

  • 输入 ：使用此过滤器为规则选择类或方法。

  • 修饰符 ：使用此过滤器可以为规则选择修饰符的类型。
