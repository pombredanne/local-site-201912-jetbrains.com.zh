外部注释

怎么运行的

如果使用的外部库的源不可用，则使用那里的属性指定代码注释似乎不可行。

在这种情况下，您可以使用外部注释 ，它使您可以用JetBrains Rider的代码分析引擎识别的属性来补充已经编译的实体。外部注释使您可以“迷惑”引擎，方法是使引擎看到在编译库时未声明的属性（用于方法，参数和其他声明）。

外部注释在XML文件中指定。加载解决方案时，JetBrains Rider在特定位置查找专门命名的XML文件，并从那里读取注释。这些XML文件的结构类似于XmlDoc。例如，注释方法XmlReader.Create(Stream input)从组装System.Xml .NET Framework 4.0以及NotNull合同，XML文件如下所示：

上述方法用于注释.NET Framework类库以及其他常用库中的大量符号。这些库的外部注释与JetBrains Rider一起安装，以确保更好的代码分析。一些功能（例如，对ASP.NET MVC 5.2的支持）也依赖于外部注释。

可在GitHub上的MIT许可下获得JetBrains外部注释的源代码： https : //github.com/JetBrains/ExternalAnnotations 。欢迎您为该存储库做出贡献或将其用作您自己的外部注释的示例。

创建外部注释

您可以创建外部注释以为任何已编译的程序集指定合同。为了确保JetBrains Rider可以识别带有外部注释的XML文件，该文件应命名为[Assembly。名称]。ExternalAnnotations.xml 。根据要使用带注释的二进制文件的方式，可以在以下位置创建注释文件：

  • 如果要发布一些二进制文件（例如，通过NuGet），请放置批注[Assembly。名称]。带注解的[Assembly.xml所在目录]所在的文件夹中的ExternalAnnotations.xml文件。名称] .dll文件位于。

  • 如果要注释项目中包含的某些二进制文件，请将每个程序集的注释文件放在项目或解决方案文件旁边的ExternalAnnotations文件夹中。

    当您将注释文件放在此文件夹中时，该文件应以程序集名称（即[程序集。名称] .xml 。它也可以位于ExternalAnnotations的子文件夹中，但是该文件夹必须具有程序集名称，即ExternalAnnotations / [Assembly。名称] / [AnyName] .xml

    这样，您还可以将注释保留在VCS下，以便您的团队可以从中受益。

创建自己的注释时，可以将JetBrains Rider注释用作参考。要研究JetBrains Rider外部注释，请查看[JetBrains Rider安装目录] \ ExternalAnnotations目录的子目录中的XML文件。
