代码愿景

Code Vision收集类型和类型成员的各种度量标准，并在声明之前或之后显示此信息。指标的位置，其最大数量和其他首选项可在以下位置配置 编辑器|的代码视觉页面 JetBrains骑士设置Ctrl+Alt+S 。

您可以单击每个指标以显示其他详细信息，或将鼠标悬停在指标上，然后单击“ 更多”以查看隐藏的指标或访问Code Vision设置。

如果要使用键盘访问代码指标详细信息，则可以使用“ 导航到”快捷方式Alt+`所有与导航相关的指标，也可以为“ 显示代码视觉”操作配置自定义快捷方式 。

Code Vision依靠两个子系统来预先计算指标：

  • 解决方案范围内的分析以计算导航指标（ 符号 ， 继承者 ， 返回API的 用法等）。
    如果禁用了解决方案范围的分析，则不会计算这些指标，您可以通过清除“ 禁用解决方案范围的分析时显示不可用的指标”复选框来隐藏它们。 编辑器|的代码视觉页面 JetBrains骑士设置Ctrl+Alt+S 。如果未隐藏，则这些指标将带有问号。但是，您仍然可以单击这些指标以开始所需的计算并获得相应的结果。

  • 集成的VCS客户端以显示代码贡献者。如果为当前解决方案启用了版本控制集成（您可以在“ 版本控制设置”页面上进行检查），则所有支持的VCS客户端（Git，Mercurial，Subversion，TFVC等）都会显示此度量标准。
