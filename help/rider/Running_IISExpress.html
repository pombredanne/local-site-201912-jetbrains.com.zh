运行/调试配置

创建新的ASP.NET Core项目或加载现有的包含带有一个或多个IIS Express配置文件的launchSettings.json文件的项目后，JetBrains Rider将基于这些配置文件自动创建运行/调试配置 。

您可以在“ 运行/调试配置”对话框（ “运行” |“编辑配置” ）中或在工具栏选择器上查看和编辑创建的配置：

对于IIS Express配置文件，运行配置包括一些其他选项：

生成applicationhost.config

为了使用ASP.NET Core应用程序启动IIS Express，JetBrains Rider使用配置文件.idea / config / applicationhost.config ，该文件是使用IIS Express附带的模板创建的。如有可能，还将保留您对此文件进行的自定义。您可能需要禁用此选项以进行大量自定义。

显示IIS Express输出

此选项启用IIS Express进程的输出。默认情况下，它是禁用的（因为输出非常详细），但是启用后，我们可以在运行应用程序时看到更多详细信息：

发送调试请求

启用此选项后，JetBrains Rider将发送初始请求（带有DEBUG HTTP请求方法）到目标ASP.NET Core应用程序。这样可以确保启动应用程序并连接调试器，即使未启动浏览器也是如此。

附加的IIS Express参数

该字段使您可以将其他参数传递给IIS Express主机（例如，启用请求跟踪）。

与其他基于launchSettings的运行配置一样，您可以使用此对话框指定目标框架和其他运行时参数，但是某些IIS Express特定的选项应直接在launchSettings.json文件中进行配置 。

运行和调试

安装并配置完所有内容后，即可运行项目Ctrl+F5或设置断点 （即使在应用程序启动时，例如Configure / ConfigureServices方法），然后调试您的代码Alt+F5 。

生成自签名SSL证书

对于基于launchSettings的运行配置，JetBrains Rider将检查您的计算机上是否设置了ASP.NET Core开发人员证书或IIS Express自签名证书。如果没有，您将看到一条通知，该通知还使您生成一个通知：

可以从该警告或使用“ 设置证书”操作（可以使用“ 查找操作”找到）来设置证书。 Ctrl+Shift+A 。

对于ASP.NET Core，JetBrains Rider将使用dotnet dev-certs https命令以检查证书是否已安装。对于IIS Express，它将检查http.sys证书设置（类似于netsh http show sslcert命令行）。
它还可以验证证书是否受我们的系统信任。这将使开发在我们的开发人员计算机上使用SSL的ASP.NET和ASP.NET Core Web应用程序更加容易。
