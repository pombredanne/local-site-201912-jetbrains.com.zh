在编译代码中导航和搜索

解决方案不仅限于项目的源代码，还包含对解决方案中使用的库的引用。可以访问并有机会从那些库中导航到符号，就像它们是解决方案的一部分一样，这非常方便。JetBrains Rider提供了内置的反编译器和许多功能，可帮助您查看和浏览已编译的代码。

来自已编译程序集的符号的代码表示形式取决于 设置 ，PDB信息的可用性以及所应用的导航命令 。

有两种针对库符号的导航命令：

  • 明确指定在何处以及如何表示库符号的命令：

    • 导航到装配体浏览器
    • 导航到反编译源
    • 导航到元数据视图
    • 导航到符号文件中的源

  • 当目标符号不在解决方案的源代码中时，这些命令可能隐式地导致编译代码。例如，

    • 前往宣言
    • 转到类型声明
    • 转到基本符号

对于第二种命令，您可以指定JetBrains Rider如何表示库符号。您可以在以下位置配置默认导航 工具|的“外部符号”页面 JetBrains骑士设置Ctrl+Alt+S 。

您还可以使用Assembly Explorer视图浏览解决方案之外的任何程序集的源代码。有关更多信息，请参见探索和反汇编程序集 。

如果您想随时清除反编译缓存，则可以删除％USERPROFILE％\中的数据。Rider2019.2 \ config \ resharper-host \ DecompilerCache 。
