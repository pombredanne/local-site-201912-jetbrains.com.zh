代码检查：闭包中访问foreach变量

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

首先，让我们确保您了解什么是闭包 。简而言之，C＃中的闭包是lambda表达式或匿名方法，该方法从外部作用域捕获一些变量。这是最简单的示例：

//一个自包含的lambda。没有关闭。动作printOne =（）=> {控制台。WriteLine（“ one”）; }; //闭包-从外部作用域捕获变量的lambda。字符串myStr =“ one”;操作打印=（）=> {控制台。WriteLine（myStr）; };

在上面的示例中， print将捕获变量myStr （而不是它的值 ），并且只会获得的值 myStr当您调用print() 。

在更复杂的情况下，当在变化的上下文中定义闭包时，它的行为可能不符合预期。

可能发生的情况之一是在foreach使用C＃4.0（Visual Studio 2010）或更早版本编译的语句。

如Eric Lippert的帖子所述， foreach已在C＃5.0（Visual Studio 2012）中进行了更改，以便为每次迭代生成一个新的循环变量，使其可以在闭包中安全使用。

在C＃5.0（Visual Studio 2012）之前，编译器将展开foreach到一个while语句，该变量具有在循环外部定义的迭代变量。因此，如果在闭包中使用此变量，则此闭包的后续调用将始终获得与循环的最后一次迭代相对应的值：

var myActions = new List （）; var myStrings = new List （）{“一个”，“两个”，“三个”}; foreach（myStrings中的var str）{myActions。Add（（）=> {控制台。WriteLine（str）; }）; } myActions [0]（）; //打印“三”

JetBrains Rider检测到此问题并建议通过将loop变量的值复制到定义闭包的范围来解决此问题：

var myActions = new List （）; var myStrings = new List （）{“一个”，“两个”，“三个”}; foreach（myStrings中的var str）{var str1 = str; myActions。Add（（）=> {控制台。WriteLine（str1）; }）; } myActions [0]（）; //打印“一个”

此修复程序确保当您从myActions并获取创建此操作的上下文， str将保存相应迭代的值。
