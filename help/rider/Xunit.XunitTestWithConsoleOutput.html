代码检查：Xunit测试中的控制台输出

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

单元测试的输出通常使用Console.WriteLine 。但是，这可能无法在xUnit.net 2.x上正常工作，因为默认情况下并行化在此处于打开状态。相反，如xUnit.net文档所建议，您应该使用ITestOutputHelper捕获测试输出。

JetBrains Rider会警告您有关以下用途Console.WriteLine里面的事实方法，并建议一个快速修复程序，将这些用法转换为ITestOutputHelper 。

次优代码

快速修复后

公共类XUnitTestClass {[事实]公共无效Test1（）{控制台。WriteLine（“ Hello”）; }}
公共类XUnitTestClass {私有只读ITestOutputHelper _testOutputHelper;公共XUnitTestClass（ITestOutputHelper testOutputHelper）{_testOutputHelper = testOutputHelper; } [事实] public void Test1（）{_testOutputHelper。WriteLine（“ Hello”）; }}
