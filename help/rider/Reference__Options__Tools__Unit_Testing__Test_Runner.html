正在测试的卷影复制程序集

使用此选择器，您可以将参与测试的程序集复制到临时目录。这样，即使相关测试正在运行，您也可以编辑和重建测试程序集。您可以在卷影复制的两个选项之间进行选择：

  • 本机 —启用测试框架的卷影复制机制，该机制通常仅复制程序集。

  • Advanced（高级） —此机制从汇编程序集的目录中复制所有文件和子目录。

在AppDomain中运行测试

此选项适用于xUnit.net和NUnit测试，并使测试运行器在其自己的应用程序域中启动每个测试程序集。在以下情况下可能会派上用场：

  • 您需要对不同的测试程序集使用不同的app.config文件。

  • 您在测试中使用反射。

检查挂起的测试执行

默认情况下，如果测试执行过程没有响应，JetBrains Rider会通知您。
您可能要禁用此选项，如果有一些代码测试完成（SetupFixture TearDown中，类TearDown中） 后，在执行时可能需要很长时间。

环境变量

在此区域中，您可以配置环境变量的“名称/值”对列表，这些变量将传递给单元测试运行器启动的过程及其所有子过程。
这些环境变量将覆盖系统中定义的具有相同名称的变量（如果有）。
根据这些环境变量是否包含敏感信息，您可以将它们保存在“解决方案个人”或“解决方案团队共享”设置层中。
