发布ASP.NET和ASP.NET Core应用程序

发布经典的ASP.NET Web应用程序

要部署经典的ASP.NET项目，请在解决方案资源管理器中右键单击该项目，然后选择“ 高级构建操作|从上下文菜单中打包所选项目 。JetBrains Rider将生成一个.zip文件形式的部署包，然后您可以使用MSDeploy命令行进行部署。

发布ASP.NET Core Web应用程序

JetBrains Rider允许您使用两种运行配置将ASP.NET Core Web应用程序部署到本地或远程服务器：

  • 发布到IIS使用MSBuild和MSDeploy打包和发布我们的应用程序。请注意，此配置仅适用于Windows。

  • 发布到自定义服务器使用MSBuild和JetBrains Rider的内置工具通过FTP和FTPS / SFTP上载我们的应用程序工件。此配置可在Windows，macOS和Linux上使用。

部署ASP.NET Core应用程序

  1. 创建一个新的运行配置 。根据要在其中发布应用程序的服务器，它可以是“ 发布到IIS”类型或“ 发布到自定义服务器”类型（请参见上面的区别）。

  2. 为创建的运行配置命名，并在配置属性中选择ASP.NET项目。

  3. 如果选择“发布到IIS”，则需要一个发布配置文件 。您可以使用某些现有配置文件或创建一个新的配置文件。

  4. 如果选择“发布到自定义服务器”，则将需要服务器配置 。 您可以查看和管理服务器上的列表 构建，执行，部署| JetBrains Rider设置的部署页面Ctrl+Alt+S或单击...按钮。
    请注意，某些服务器（例如Azure Web Apps）需要使用被动模式来上传文件。您可以通过单击服务器配置选项中的“ 高级”选项来设置此（以及更多）选项。

  5. 创建配置后，您可以从工具栏执行部署（或通过按Ctrl+F5选择部署运行配置时）。

发布个人资料

要使用IIS部署ASP.NET Core应用程序，您将需要一个发布配置文件 。发布配置文件以.pubxml文件的形式出现，并包含MSDeploy的必要参数。

您可以在项目中具有多个具有不同设置的发布配置文件。JetBrains Rider附带了一些用于生成.pubxml文件的模板。

Sayed Hashimi说了很多有关MSDeploy选项的示例 。

请注意，JetBrains Rider会使用可配置的文件模板创建发布配置文件 ，您可以在 编辑器|文件模板| JetBrains Rider设置的“ 其他语言”页面Ctrl+Alt+S 。

创建一个新的ASP.NET Core发布配置文件

  1. 请执行以下任一操作：

    • 在解决方案资源管理器中选择一个ASP.NET Core项目，然后按Ctrl+N或右键单击并选择添加 ，然后选择发布XML 。

    • 如果要编辑 “ 发布到IIS”类型的运行配置 ，请单击“ 发布配置文件”字段旁边的+ 。

  2. 在打开的对话框中，为配置文件指定名称并选择其类型：

  3. JetBrains Rider将为您的项目在Properties \ PublishProfiles下将新创建的配置文件添加为.pubxml文件。它还将打开文件进行编辑并突出显示所需的属性，以便您可以为其指定适当的值。

请注意，发布到IIS服务器仅在Windows上有效。可能还需要使用提升的权限来启动JetBrains Rider。当使用针对本地IIS服务器的发布配置文件时，JetBrains Rider会建议这样做。
