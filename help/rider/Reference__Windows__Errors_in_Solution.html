解决方案窗口中的错误

查看|工具窗口|解决方案中的错误
Ctrl+Alt+2

此窗口显示在解决方案范围分析期间发现的解决方案范围错误/警告的列表。

您可以选择在解决方案范围的分析中仅监视错误，或同时监视错误和警告。

工具栏控件

控制

名称

描述

分析解决方案中的错误

打开/关闭整个解决方案范围的分析 。

/

暂停/继续分析

如果关闭解决方案范围的分析然后决定再次打开它 ，JetBrains Rider将不得不重复一些初始计算，这可能会很费时。为避免这种情况，您可以暂停并恢复解决方案范围的分析。

包含/排除警告

使用此按钮可以在解决方案范围内的分析中启用/禁用警告。

在解决方案范围的分析中启用警告时，其工作方式如下：只要解决方案中存在错误，就只会显示错误；如果解决方案中存在错误，则仅显示错误。未解决的警告仅在修复了最后一个错误后才会出现。禁用警告后，一旦纠正了最后一个错误，状态栏指示器就会变为绿色。

忽略错误

从发现的问题列表中排除选定的错误/警告（或选定文件中的所有错误/警告）（如果打开了“ 显示忽略的错误”，则将其删除 ）。有关更多信息，请参阅忽略解决方案中的错误 。

停止忽略错误

显示或以前忽略的错误（如果打开“ 显示忽略的错误”，则删除删除线）。有关更多信息，请参阅忽略解决方案中的错误 。

显示忽略的错误

使被忽略的错误可见，但将其删除。

重新分析有错误的文件

启动重新分析先前发现错误的文件的过程。

重新分析所有

启动重新分析解决方案中所有文件的过程。

全部展开 / 全部折叠
Ctrl+NumPad Plus / Ctrl+NumPad Minus

展开/折叠当前选项卡中的所有节点。

在树中显示框架

启用此选项后，您可以展开每个错误/警告，以查看它适用于哪个选定的.NET框架。

框架选择器

使用此选择器选择应将.NET框架用作错误和警告的来源。

自动滚动到源

按下此按钮后，将根据在窗口中选择的项目滚动编辑器中的源代码。

标题栏上下文菜单和按钮

您可以右键单击窗口标题栏，然后使用上下文菜单配置其查看模式 ，将窗口与其他工具窗口栏关联，或者调整大小并隐藏窗口。

您还可以使用工具栏按钮：

项目

捷径

描述

单击此按钮可以访问上下文菜单命令的子集，这些命令可让您配置窗口的查看模式 。

Shift+Escape

使用此图标或快捷方式隐藏工具窗口。

当与Alt键，单击此图标将隐藏附加到同一工具窗口栏的所有工具窗口 。
