代码语法样式：隐式/显式键入（“ var”关键字）

使用隐式类型的局部变量 （也称为var C＃3.0中引入的关键字（关键字）已变得非常流行，因为它在许多情况下提高了可读性。默认情况下，ReSharper还鼓励使用var关键字，但其使用偏好可以灵活配置 -例如，您可以选择在特定情况下或在任何地方使用显式类型，JetBrains Rider将帮助您实施偏好。

从C＃7.0开始，您可以在解构元组时声明局部变量。若你宁可var在此类声明中，您可以另外配置样式以使用联接或分隔符，例如： var (x, y) = GetTuple();要么(var x, var y) = GetTuple(); 。

C＃7.0的另一个功能是throws ，它还允许使用var确保与可以命名的作用域中的变量没有冲突_ 。JetBrains Rider允许您配置样式以始终使用var并在适当情况下丢弃。

强制使用'var'关键字的首选项

JetBrains Rider会检查所有局部变量是否符合您的偏好 ，如果不符合您的偏好 ，JetBrains Rider会突出显示此类声明并建议相应的快速修复或范围修复 。

'var'关键字用法设置不适用于生成的代码。默认情况下，JetBrains Rider提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 编辑器|检查设置|的“生成的代码”页面 JetBrains骑士设置Ctrl+Alt+S 。

配置使用'var'关键字的首选项

您的“ var”关键字用法首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

配置使用'var'关键字的首选项

  1. 去 编辑器|代码样式|的C＃页面 JetBrains骑士设置Ctrl+Alt+S ，然后选择“ 代码样式”标签。

  2. 根据您的编码惯例/标准， 在声明类别的“ var”用法中修改设置。 您可以为不同类型设置使用'var'或显式类型的不同首选项：

    • 对于内置类型 -适用于C＃内置类型 。

    • 对于简单类型 -适用于不具有通用参数的类型。

    • 其他 -适用于泛型类型和解构声明。

    对于这些首选项中的每一个，您都可以选择使用'var'，显式类型或'var'（当“ explain”时使用） ，这意味着“ var”应仅用于初始化为创建对象，数组和文字或显式类型转换的变量。

  3. 默认情况下，JetBrains Rider将建议多个的联合表示法var例如，在解构声明中 var (x, y) = GetTuple(); 。您可以选择对解构变量更喜欢单独的声明，以选择单独的符号，例如(var x, var y) = GetTuple(); 。

  4. 默认情况下，JetBrains Rider建议使用独立_例如，用于丢弃 (_, _) = (0, 1); 。您可以选择使用“ var”关键字来丢弃，以始终使用var在适当的地方使用丢弃，从而确保与可能命名的范围中的变量没有冲突_ 。

  5. 右列中的选择器允许您设置代码检查的 严重性级别 ，以检测与首选项不同的代码。

  6. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。
