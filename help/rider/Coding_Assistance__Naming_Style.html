命名方式

从2018.3版本开始，JetBrains Rider可以从现有代码中自动学习C＃符号命名规则 。如果要手动配置命名规则，请清除“ 使用自动检测到的规则”复选框 编辑器|代码样式| C＃|的命名页面 JetBrains骑士设置Ctrl+Alt+S 。

JetBrains Rider可帮助您定义，控制和应用代码中符号的命名方式。命名样式是作为一组规则实现的，每个规则都针对具有一组约束的特定标识符（例如，一条规则可以针对静态私有只读字段）。每个规则可以具有一个或多个关联的样式，这些样式定义后缀，前缀，复合词的大写形式，等等。

JetBrains Rider会考虑这些规则产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。JetBrains Rider还可以帮助您检测和修复违反命名规则的情况。如有必要，可以配置或禁用自动检查命名规则。

配置命名规则

开箱即用的JetBrains Rider为每种标识符提供命名规则。这些规则基于Microsoft命名准则 ， .NET Foundation编码准则和最佳实践。
请注意，JetBrains Rider命名规则默认值与上述任何准则之间都没有严格的对应关系。

如果您的个人喜好或公司标准与JetBrains Rider的默认设置不同，则可以灵活地配置命名方式：对于每种标识符，您可以选择大写规则，前缀和后缀，不同访问权限的变体，要保留的缩写，以及更多。

您的命名风格首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

修改默认命名规则

  1. 在Rider设定中Ctrl+Alt+S ，转到编辑器|代码样式| C＃并打开“ 命名”选项卡。

  2. 在左侧列表中选择所需的规则。

  3. 在页面右侧，检查规则的现有样式。

  4. 如果现有样式可以接受，但是您要允许该规则使用其他样式，请点击添加 。当单个规则有多种样式时，如果相应的符号名称至少与这些样式中的一种匹配，则JetBrains Rider不会检测到违反代码样式的情况。否则，JetBrains Rider会检测到违反代码样式的行为，并建议根据默认样式（列表中的第一个样式）重命名符号。要更改默认样式，请使用上移 然后向下移动 纽扣。

  5. 要配置特定样式，请在列表中选择它，并配置前缀，后缀，然后选择一个命名样式。

  6. 要删除不必要的样式，请在列表中选择它，然后单击“ 删除”。 。

  7. 如有必要，您可以禁用与该规则关联的代码风格违规检测 ，从而使JetBrains Rider可以检测其他代码风格违规。为此，请清除“ 启用检查”复选框。

  8. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。

检测并修复违反命名规则的情况

JetBrains Rider使用“不一致命名”代码检查来检测代码中违反命名规则的情况。默认情况下，此检查始终在设计时代码检查期间应用，并在编辑器中突出显示检测到的违规作为警告 。例如，根据默认样式，接口名称应带有“ I”前缀。如果接口名称不符合此规则，JetBrains Rider会突出显示该名称，并将鼠标悬停在该名称上时会通知您有关问题的信息：

如果要查找超出当前文件的命名风格冲突，则可以在项目或整个解决方案中运行“不一致的命名”检查。

查找特定范围内的命名风格违规

  1. 在编辑器中，将插入符号设置为命名不一致的突出显示符号。

  2. 按Alt+Enter或单击插入符号左侧的动作指示器以打开动作列表 。

  3. 选择检验“不一致的命名” |在文件中找到类似的问题，然后选择所需的范围。

  4. 找到的所有违反命名方式的内容都会显示在“ 检查结果”窗口中 。

  5. 您可以双击检测到的问题以导航到相应符号的代码。

对于编辑器中突出显示的大多数命名风格违规，JetBrains Rider建议使用名称一致的快速修复 。你可以按Alt+Enter然后在操作列表中选择“ 重命名为[符合名称] ”。

也可以使用范围内修复功能在当前文件，项目或解决方案中执行自动更正命名样式冲突。

但是，如果具有计算出的一致名称的符号已存在于同一名称空间中，则不建议使用快速修复。您可以使用Rename refactoring修复此类符号的命名 。

配置和禁用自动命名规则检查

如果您不希望JetBrains Rider检查现有代码库中的命名样式，则可以禁用对此负责的代码检查，或者禁用特定规则的自动检查。但是，即使禁用了自动检查，当JetBrains Rider帮助您使用代码完成，生成，重构和模板来生成新代码时，也会遵守所有命名规则。

禁用命名样式的自动检查

  • 上 编辑器|的检查严重性页面 JetBrains骑士设置Ctrl+Alt+S ，开始输入“不一致的命名”，然后清除相应代码检查旁边的复选框。

禁用自动检查特定命名规则

  1. 在Rider设定中Ctrl+Alt+S ，转到编辑器|代码样式| C＃并打开“ 命名”选项卡。

  2. 在左侧列表中选择所需的规则。

  3. 在打开的“ 编辑规则设置”对话框中，清除“ 启用检查”复选框。

  4. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。

使用现有代码中的C＃命名规则

如果您从公共仓库中检出解决方案，或暂时加入建立了不同代码命名风格的团队，则可能要遵循此解决方案中的命名规则，但同时不要更改全局配置的命名规则。

JetBrains Rider将自动处理此类情况-当您打开符号命名与全局设置不同的新解决方案时，它将检测与全局设置不同的命名规则，相应地修改这些规则，然后将修改后的规则保存到“解决方案人员的设置层 （ .sln。解决方案旁边的DotSettings.user文件）。

如果在打开新解决方案时发生这种情况，JetBrains Rider将在编辑器中显示一条通知：

您可以单击撤消以还原这些修改，或单击配置以打开相应的设置页面（ 编辑器|代码样式| C＃|命名 ）。在此设置页面上，根据解决方案代码更改的规则以黄色突出显示。如果要使用JetBrains Rider的默认命名规则或以自己的方式配置命名样式 ，请清除“ 使用自动检测到的规则”复选框。
