调试

调试器是JetBrains Rider不可或缺的一部分，它使您可以检查应用程序的运行时行为，识别有问题的代码并逐步隔离问题的根源。

典型的调试步骤

  1. 为要调试的应用程序定义运行/调试配置 。

  2. 在源代码中设置可能存在问题的断点 。

  3. 开始调试会话 Alt+F5 。

  4. 当程序执行中断时，单步执行代码 ， 评估表达式并设置watch 。

  5. 您也可以编辑代码并继续调试而无需重新启动。该项目将在后台重新编译。当在启用 （ Enable Edit And Continue ）上打勾时，这适用于Windows上针对.NET Framework和.NET Core（WPF除外）的项目 构建，执行，部署|的调试器页面 JetBrains骑士设置Ctrl+Alt+S 。

    在某些情况下，无法修改代码，并且需要重新启动。其中包括更改签名，修改通用功能，任何类型的重命名等等。如果发生这种情况，JetBrains Rider将通过显示工具提示来通知您。

    当前仅在Windows上支持“编辑并继续”，因为其他运行时未实现所需的调试器API

  6. 恢复调试会话F5让程序运行到下一个断点（如果有）。

  7. 要完成调试会话，请按Shift+F5 ，选择运行|停在菜单或单击停止 在调试窗口中 。这会将调试器与应用程序分离，并使应用程序退出（如果您已从JetBrains Rider运行它）。

调试活动的主要部分发生在“ 调试”窗口中 ，一旦开始调试，该窗口便可用。要打开此窗口，请按Alt+5或选择查看|工具窗口|在主菜单中调试 。

您可能还对以下特定于语言和框架的准则感兴趣：

  • 在Chrome中调试JavaScript
  • 在Firefox中调试JavaScript
  • 运行和调试TypeScript
  • 运行和调试Node.js
  • 调试Unity应用程序
