默认严重性

分行符不一致

此检查检测到的行分隔符与项目默认值不同的文件。例如，您在“设置” |“代码样式” |“行分隔符”中将行分隔符设置为“ \ n”，并且您正在编辑的文件使用“ \ r \ n”作为行分隔符。

残障人士

有问题的空格

当代码样式配置为仅使用空格时，报告用于缩进的选项卡。当代码样式配置为仅使用制表符时，报告用于缩进的空间。当代码样式配置为使用智能选项卡时，报告用于缩进的空间和用于对齐的选项卡。

残障人士

TODO评论

在您的代码中报告TODO注释。TODO注释的格式可以通过“ 设置” |“配置”来配置。编辑器TODO面板。由于已经提供了针对TODO注释的语法突出显示功能，因此预计将在批处理模式下大量使用该语法。

残障人士

行长超出代码样式允许的范围

检查会警告您比“代码样式”设置中的右页边距参数长的行。

残障人士

注入参考

检查由IntelliLang插件注入的引用。

冗余抑制

报告的使用情况

  • @SuppressWarning注释，或

  • // noinspection行注释，或

  • /** noinspection */ JavaDoc评论

可以安全删除它们，因为它们影响的检查在这种情况下不再适用。

例如：

public class C { // symbol is already private, // but annotation is still around @SuppressWarnings({"WeakerAccess"}) private boolean CONST = true ; void f() { CONST = false ; } }
