调试服务器和客户端代码

使用JetBrains Rider，您可以通过两种方式调试应用程序的服务器端代码和客户端JavaScript代码：

  • 另外，使用单次运行配置。

  • 同时，使用带有JavaScript Debug配置的复杂Node.js。所有你需要的是配置浏览器的行为并启用调试应用程序的客户端代码。此功能是通过JavaScript Debug运行配置，所以从技术上讲，JetBrains Rider为服务器端和客户端代码创建单独的运行配置，但是您可以在一个专用的Node.js运行配置中指定所有设置。

创建复合的Node.js / JavaScript调试配置

  1. 从主菜单中，选择运行|编辑配置 。

  2. 从列表中，选择Node.js运行配置以与JavaScript Debug配置一起启动。在打开的对话框中，切换到“ 浏览器” /“实时编辑”选项卡。

  3. 选择启动后，在启动调试会话时自动启动浏览器。

  4. 在下面的字段中，输入URL地址以打开应用程序。

  5. 从启动后复选框旁边的列表中选择要使用的浏览器。

    • 要使用系统默认浏览器，请选择默认 。

    • 要使用自定义浏览器，请从列表中选择它。请注意，只有Chrome才完全支持Live Edit 。

    • 要配置浏览器，请单击并在打开的“ Web浏览器”对话框中调整设置。有关更多信息，请参阅配置浏览器 。

  6. 选择“ 使用JavaScript调试器” 。

在Node.js应用程序中启用实时编辑

  1. 如管理插件中所述，在“ 插件”页面上安装LiveEdit插件。

  2. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 构建，执行，部署|调试器实时编辑 。

  3. 选择更改时更新Node.js应用程序 。 指定在编辑器中更改代码与在浏览器中显示此更改之间的时间延迟：接受默认值300 ms或使用相应字段旁边的旋转框指定自定义值。
