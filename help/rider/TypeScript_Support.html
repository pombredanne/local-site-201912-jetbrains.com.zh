JetBrains Rider 2019.2帮助

打字稿

JetBrains Rider支持开发，运行和调试TypeScript源代码。JetBrains Rider可以识别.ts文件，并提供全方位的编码帮助来编辑它们，而无需您执行任何其他步骤。TypeScript文件标有图标。

支持TypeScript的编码帮助包括关键字，标签，变量，参数和函数的完成，错误和语法突出显示，格式设置，大量代码检查和快速修复以及常见的和特定于TypeScript的重构。JetBrains Rider还可以即时将TypeScript验证并编译为JavaScript。

验证TypeScript并将其编译为JavaScript

JetBrains Rider主要根据来自TypeScript语言服务的数据来验证TypeScript代码，该服务还将TypeScript编译为JavaScript。

在TypeScript工具窗口的“ 错误和编译错误”选项卡中报告所有检测到的语法和编译错误。对于每个错误，JetBrains Rider都会提供简要说明和有关发生错误的行号的信息。

自打开工具窗口以来，“ 控制台”选项卡显示TypeScript语言服务的日志。

配置与TypeScript语言服务的集成

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到语言和框架| TypeScript 。将打开“ TypeScript”页面 。

  2. 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

  3. 在TypeScript列表中， 选择要使用的TypeScript版本（JetBrains Rider将显示当前选择的版本）。

    • 默认情况下，文件夹用于从项目的node_modules的打字稿包。

    • 捆绑：选择此选项可使用JetBrains Rider捆绑提供的打字稿包，而无需尝试寻找另一个。

    • 选择：选择此选项以使用自定义打字稿软件包，而不是与JetBrains Rider捆绑在一起的软件包。在打开的对话框中选择相关软件包的路径。

  4. 确保选中“ TypeScript语言服务”复选框。

  5. 使用下面的控件来配置TypeScript语言服务的行为。

  6. 在选项字段中， 指定在找不到tsconfig.json文件时传递给TypeScript语言服务的命令行选项。请参阅TSC参数上可接受的选项列表。请注意-w要么--watch （ 观看输入文件 ）选项无关紧要。

您可以使用TypeScript语言服务的建议来增强JavaScript文件中的完成功能。为此，添加'allowJS' : true到您的jsconfig.json或tsconfig.json文件。从代码完成（IntelliSense）和配置JavaScript库中了解更多信息。

监视语法错误

  • 打开TypeScript工具窗口（“ 视图” |“工具窗口” |“ TypeScript” ），然后切换到“ 错误”选项卡。

    该选项卡列出了TypeScript语言服务检测到的代码中的差异。更改代码后，该列表将动态更新。

    默认情况下，列表仅包含活动编辑器选项卡中文件的错误，并且此文件的完整路径显示在顶部。要显示整个项目中的错误，请按显示项目错误切换按钮在工具栏上。

    错误消息按检测到错误的文件分组。

    要导航至有问题的代码，请双击相应的错误消息或将其选中，然后从上下文菜单中选择“ 跳转到源代码 ”。

监视编译错误

  • 打开TypeScript工具窗口（“ 视图” |“工具窗口” |“ TypeScript” ），然后切换到“ 编译错误”选项卡。

    单击时将打开选项卡然后从列表中选择编译范围：

    默认情况下，“ 全部编译”代表编译整个项目中的所有TypeScript文件 。若要更改此行为，请在TypeScript页面上的“ 编译范围”字段中定义另一个默认范围。

    该选项卡列出了在选定范围内检测到的所有编译错误。该列表不受代码更改的影响，仅在再次手动调用编译时才会更新。

    错误消息按检测到错误的文件分组。 要导航至有问题的代码，请双击相应的错误消息或将其选中，然后从上下文菜单中选择“ 跳转到源代码 ”。

查看参数提示

参数提示显示方法和函数中参数的名称，以使您的代码更易于阅读。 默认情况下，仅对文字或函数表达式的值显示参数提示，而对命名对象则不显示。

配置参数提示

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |一般|外观 。“ 外观”页面打开。

  2. 单击“ 显示参数名称提示”复选框旁边的“ 配置” （默认情况下，该复选框处于选中状态）。

  3. 在打开的“ 配置参数名称提示”对话框中，选中“ 选项”区域中的“ 显示所有参数的名称”复选框。

要隐藏所有类型值的参数提示，请清除“ 显示参数名称提示”复选框。

查看推断的类型信息

要查看对象的推断类型 ，请按住Ctrl然后将鼠标指针悬停在它上面

在TypeScript中使用JavaScript库

在TypeScript中使用JavaScript库时，需要为其安装类型声明 。JetBrains Rider提醒您通过npm安装它们，并相应地更新package.json文件。

安装类型声明

  1. 将插入符号置于警告位置，然后按Alt+Enter 。

  2. 选择建议并按Enter 。

文档查找

通过JetBrains Rider，您可以从项目及其依赖项中获取符号的引用，还可以从外部库中定义的符号以及标准的JavaScript API获得引用，因为TypeScript可以实现所有这些符号。

该文档显示在“ 文档”弹出窗口中，该弹出窗口可帮助您通过超链接导航到相关符号，并提供一个工具栏，用于在已导航的页面之间来回移动。

查看文档中插入符号的符号

  • 按Ctrl+Shift+F1或选择查看|从主菜单快速查找文档 。该符号的引用显示在“ 文档”弹出窗口中。

    对于TypeScript中可用的标准JavaScript方法，JetBrains Rider还会显示指向相应MDN文章的链接。

查看MDN文档中的符号

  • 在“ 文档”窗口中Ctrl+Shift+F1 ，单击MDN链接。

  • 或者，按Shift+F1或选择查看|主菜单中的外部文档 。

JetBrains Rider在默认的JetBrains Rider浏览器中打开MDN文章。

重构代码

JetBrains Rider提供了常见的重构过程（如重命名/移动等）以及特定于TypeScript的重构过程，如更改签名 ， 提取参数 ， 提取变量 。

自动导入TypeScript

JetBrains Rider可以为模块，类，组件以及任何其他导出的TypeScript符号生成导入语句。默认情况下，当您完成TypeScript符号时，JetBrains Rider将添加导入语句。

当您键入代码或粘贴带有尚未导入符号的片段时，JetBrains Rider也可以为该符号生成导入语句。如果只有一个来源可以从中导入符号，JetBrains Rider会静默插入一个导入语句。否则，请使用导入弹出窗口或专用的导入快速修复程序 。

在代码完成时添加导入语句

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |一般|自动导入 。将打开“ 自动导入”页面。

  2. 在“ TypeScript / JavaScript”区域中，选择“ 自动添加TypeScript导入”和“代码完成时”复选框。

要更改导入弹出窗口的背景颜色，请按Ctrl+Alt+S并转到编辑器|配色方案|一般|弹出窗口和提示|问题提示 。

在输入或粘贴代码时添加导入语句

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |一般|自动导入 。将打开“ 自动导入”页面。

  2. 在“ TypeScript / JavaScript”区域中，选中“ 自动添加TypeScript导入”和“即时动态导入”复选框。

配置语法突出显示

您可以根据自己的喜好和习惯配置支持TypeScript的语法突出显示。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |配色方案| TypeScript 。
