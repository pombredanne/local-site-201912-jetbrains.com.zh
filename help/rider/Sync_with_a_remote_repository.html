与远程存储库同步

通过将更改推送到上游可以共享工作结果之前，您需要与远程存储库进行同步，以确保项目的本地副本是最新的。您可以通过以下方式之一执行此操作： 获取更改 ， 提取更改或更新项目 。

获取更改

当您从上游获取更改时，自上次与远程存储库同步以来所做的所有提交的新数据都将下载到本地副本中。此新数据未集成到您的本地文件中，并且更改也未应用于您的代码。

提取的更改存储为远程分支，这使您有机会在将它们与文件合并之前对其进行检查。由于访存不会影响您的本地开发环境，因此这是对远程存储库中所有更改进行更新的安全方法。

  • 要获取更改，请从主菜单中选择VCS |。 Git |抓取 。

拉变更

从远程仓库拉变化为方便快捷方式获取并随后合并的变化。拉动时 ，不仅下载新数据，还将其集成到项目的本地工作副本中。

  1. 从主菜单中，选择VCS |。 Git |拉将打开“ 拉出更改”对话框。

  2. 如果您的项目有多个Git存储库，请从Git Root列表中选择要更新的本地存储库的路径。

  3. Git根目录。

  4. 如果为存储库配置了多个远程服务器，请在“ 远程”列表中选择要从中提取数据的远程服务器的URL。

  5. 选择要从中获取更改的分支，然后合并到当前签出的分支中。

    已预选择跟踪的远程分支。

  6. 从“ 策略”列表中，选择“ 远程”列表中的。

  7. 选择要从中获取更改的分支，然后合并到当前签出的分支中。已预选择跟踪的远程分支。

  8. 从“ 策略”列表中，选择将用于解决合并期间发生的冲突的合并策略 。

  9. 如有必要，请选择以下内容：

    • 不提交 ：如果您不希望JetBrains Rider自动提交合并结果，请选择此选项。在这种情况下，您可以检查它们并在必要时进行调整。

    • 没有快进 ：即使合并被解析为快进（即使仅分支指针已更新），也选择此选项可生成合并提交。

    • Squash commit ：选择此选项可在当前分支之上创建一个提交，而不是合并一个或多个分支。它产生工作树和索引状态，就好像发生了真正的合并一样，但是实际上并没有进行提交或移动HEAD。

    • 添加日志信息 ：如果您希望JetBrains Rider在分支消息中使用合并的实际提交中的单行描述（除了分支名称）来填充日志消息，请选择此选项。

  10. 单击“ 拉取”以从选定的远程存储库中获取并应用更改。

更新你的项目

如果您有多个项目根目录，或者想在每次与远程存储库同步时从所有分支中获取更改，则可能会发现更新项目是一个更方便的选择。

当您执行更新操作时，JetBrains Rider从所有项目根目录和分支中获取更改，并将跟踪的远程分支合并到本地工作副本中（等同于pull ）。

如果您的项目包含子模块 ，并且它们在分支上，它们也将自动更新。

如果子模块处于分离的HEAD状态，JetBrains Rider将调用git submodule update ，它将检出根存储库中引用的提交。这意味着仅在根回购中子模块引用更改或添加了新子模块的情况下，才执行更新。

  1. 从主菜单中，选择VCS |。更新项目或按Ctrl+Alt+W 。将打开“ 更新项目”对话框。

  2. 选择更新类型（此策略将应用于Git版本控制下的所有根）：

    • 合并 ：选择此选项可在更新期间执行合并 。这相当于跑步git fetch然后git merge ， 要么git pull --no-rebase 。

    • 调整基线 ：选择此选项来执行衍合在更新过程中。这相当于跑步git fetch然后git rebase ， 要么git pull --rebase （所有本地提交都将放在更新后的上游头文件的顶部）。

    • 分支默认值 ：如果要对不同的分支应用不同的更新策略，请选择此选项。您可以为分支中的每个分支指定默认更新类型branch. .git / config配置文件部分。

  3. 指定一种在更新前清理工作副本时将用于保存更改的方法，以便在更新完成后可以还原未提交的更改：

    • 使用Stash ：选择此选项可将本地更改保存在git stash中 。如果您需要在JetBrains Rider之外应用带有隐蔽更改的补丁程序，这很有用，因为它们是由Git本身生成的。

    • 使用 货架 ：选择此选项可将本地更改放入货架 。搁置由JetBrains Rider完成，并且搁置更改生成的补丁通常会应用在JetBrains Rider内部。

    如果您选择以后不再显示“ 更新项目”对话框，然后以后再修改默认更新策略， 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ）选择版本控制|确认 ，在调用这些命令时在“ 显示选项”对话框下选择“ 更新 ”，并在下次执行更新时修改更新策略。

更新操作完成后，“ 更新信息”选项卡将添加到“ 版本控制”工具窗口中。它列出了自从上次与远程同步以来的所有提交，并允许您以与“ 日志选项卡 ”中相同的方式查看更改。

如果要查看自上次更新以来已修改的所有文件的完整列表，请将插入标记放置在提交列表中的任何位置，然后按Ctrl+A 。
