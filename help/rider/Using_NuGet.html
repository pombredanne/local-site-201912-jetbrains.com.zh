还原和升级NuGet软件包

Rider提供了两种用于还原NuGet软件包的算法：

  • 正常 -调用标准的NuGet还原机制，该机制可能很慢，尤其是在.NET Core项目上。在主菜单（ 工具| NuGet | NuGet Restore ）的NuGet快速列表中可用。 Shift+Alt+N ，然后单击“还原” 在NuGet窗口工具栏上。

  • 强制 -执行许多试探法以确定应该还原哪些软件包。该算法几乎可以瞬间生效，但是在某些情况下，它无法还原某些软件包。
    我们建议将此算法用于大型项目。在主菜单（ 工具| NuGet | NuGet强制还原 ）或NuGet快速列表中可用。 Shift+Alt+N 。

当您打开解决方案时，Rider会自动应用“ Light”算法来还原丢失的软件包。如有必要，您可以在NuGet设置页面上更改自动恢复行为，也可以通过单击NuGet设置打开该页面。 在NuGet窗口工具栏上。

要升级已安装的软件包，请单击“ 解决方案”中的“升级所有软件包”。 在NuGet窗口工具栏上，然后选择要升级的软件包。
或者，您可以在NuGet窗口的左侧选择已安装的软件包之一，在右侧选择所需的版本，然后将特定项目升级/降级到此软件包版本。

安装或升级/降级NuGet软件包

  1. 如上所述找到所需的软件包，然后在“ 软件包”选项卡的左侧选择它

  2. 在选项卡的右侧，选择要在其中安装所选软件包的项目。如果您的解决方案包含太多项目，则可能要使用工具栏上的范围选择器来显示特定项目并隐藏所有其他项目。

  3. 使用工具栏上的预发布复选框来显示或隐藏预发布包的版本。

  4. 使用右侧选项卡中的“ 版本”选择器选择所需的软件包版本。

  5. 在安装所选软件包之前，您可能需要检查将安装哪些从属软件包。为此，在右侧部分展开“ 框架和依赖关系 ”：

  6. 根据所选软件包的版本以及所选项目中是否安装了所选软件包，您将能够安装，升级或降级该软件包。
    安装新软件包后，您将在编辑器中看到它的readme.txt文件，以便您能够检查安装后是否需要任何其他步骤。
    如果在软件包中实现了配置文件/ XDT转换 ，JetBrains Rider将运行它们来帮助添加所需的条目，例如添加到App.config或Web.config文件中。

私人NuGet资讯提供的凭证提供者

通过JetBrains Rider，您可以对使用不同身份验证方法的私有NuGet源进行身份验证。

用户名/密码组合的身份验证提示

在NuGet窗口的“ 源”选项卡中，您可以编辑或添加源 ，并以用户名/密码组合的形式添加凭据。

私有供稿的URL通常在解决方案文件旁边的NuGet.config文件中与团队成员共享，以便团队中的每个人都可以连接到此类私有供稿。在需要还原软件包的此类解决方案的第一个负载上，JetBrains Rider将提示您输入凭据。

凭据正确无误后，它们将存储在JetBrains Rider的密码存储中，因此您不必在每次访问NuGet feed时都重新输入凭据。密码存储可在以下位置配置 外观和行为系统设置| JetBrains Rider设置的“ 密码”页面Ctrl+Alt+S 。

JetBrains Rider还具有MyGet和Visual Studio Team Services（VSTS）的集成凭据提供程序。

使用身份验证插件

对于使用基于插件的身份验证方法的私有供稿，例如Azure Artifacts提供的私有供稿，您需要更改“ 使用凭据提供程序”首选项 构建，执行，部署| JetBrains Rider设置的NuGet页面Ctrl+Alt+S到NuGet / .NET CLI插件或NuGet / .NET CLI插件，然后集成Rider （如果您使用其他身份验证方法），然后重新启动JetBrains Rider。

启用身份验证插件支持后，将监视％USERPROFILE％\。nuget \ plugins （Windows）和$ HOME / .nuget / plugins （Linux / macOS）路径，以查找适用的凭据提供程序插件，而JetBrains Rider将使用这些插件执行必要的身份验证。

如果要构建自己的凭据提供程序插件，请在GitHub上查看Rider的样板NuGet凭据提供程序插件 。

管理NuGet缓存

NuGet会缓存所有下载的程序包，以在没有Internet连接时使它们可用，并防止同一程序包的多次下载。

使用Rider，您可以查看NuGet缓存的位置，并在NuGet窗口的“ 文件夹”选项卡中清除选定的位置。清除NuGet缓存是安全的，不会影响项目中安装的软件包。
