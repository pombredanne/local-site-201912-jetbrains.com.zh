重新排列代码元素

代码上/下/左/右上移语句
Ctrl+Shift+Alt + Up/Down/Left/Right

重新排列命令的默认快捷方式不适用于远程桌面。解决方法是在上为这些命令分配自定义快捷方式 的“ 键盘映射”页面 JetBrains骑士设置Ctrl+Alt+S 。

JetBrains Rider使您可以快速重新排列代码中的表达式，语句，类型成员和其他元素。

怎么运行的

要重新排列代码，请按Ctrl+Shift+Alt在要移动的代码元素或选择上。然后将键盘箭头之一添加到组合键以调用相应的命令：

要选择逻辑代码块 ，请按Ctrl+Alt+Right一或多次。

上移和下移命令非常简单-它们可以相对于该范围内的其他元素上下移动特定范围内的元素。例如，您可以在文件中上下移动类型，类型中的类型成员，类型成员中或复合语句中的语句等，
如果当前元素在其作用域中是第一个还是最后一个，JetBrains Rider将不允许您使用向上/向下命令将其进一步移动。

向左移动和向右移动命令有两个作用：

  • 重新排列通常单行编写的元素。例如，您可以左右移动属性，函数参数，表达式中的操作数，赋值语句的一部分等。请注意，即使将它们写在多行中，左/右命令也将应用于这些元素。

  • 将语句移动到紧随其后的复合语句或区域内（向右移动）或将语句移动到复合语句或区域外（向左移动）。

尽管JetBrains Rider尝试不允许无效移动，但是在某些情况下，您可能会破坏逻辑或代码的编译。请注意，JetBrains Rider提供了许多重构功能，可帮助您安全地重新布置代码，例如， 将类型移动到外部作用域重构和更改签名重构 。如果在移动代码元素后代码破裂，请按Alt+Enter并检查可用的快速修复 。大多数时候，您会在操作列表中找到正确的重构。

适用案例

JetBrains Rider允许您通过以下方式移动元素：

  • 类型和类型成员。请注意，如果该符号具有XML注释，JetBrains Rider会将它们与该符号一起移动。

    • 在当前文件或包含声明中向上和向下键入

    • 在类型内上下键入成员

    • 左右基本类型

    • 左右输入参数

    • 类型参数上下约束

    • 左右属性

    • 左右声明中的参数和函数用法中的参数（如果对函数声明的参数进行重新排序，JetBrains Rider建议使用Change Signature重构 ，以确保在函数的所有用法中都改变参数。

    • 属性的设置者和获取者

  • 陈述

    • 赋值语句的左右部分

    • 函数内的语句或上下组合语句

    • 从复合语句（左）或紧随其后的复合语句（右）的语句

    • 上下切换语句中的各个部分

    • 从if-else，try-catch语句的一个子句或从一个switch节到另一段的声明

    • catch上下try-catch语句中的子句

    • 上下一个复合语句的右括号（贪婪括号）。要移动闭合括号，请将插入号放置在括号之外，按Ctrl+Shift+Alt然后使用向上和向下箭头在其中包含跟在当前复合语句之后的语句，或者将最后一个语句移至其外部。

    • 打开或关闭类型或名称空间的括号也可以用作“贪婪的括号”。在这种情况下，它将包括/排除嵌套的类型/成员/命名空间。

  • 表达方式

    • 左右表达式中的操作数

    • 集合和数组初始化中的元素左右

    • 查询表达式中的子句上下

    • 左右范围表达式操作数（C＃8）

  • 其他要素

    • 泛型类型/方法声明的类型参数

    • 左右模板字符串中的参数

    • 内插字符串中的值

    • 上下线和c样式注释

    • #region向上和向下，进出另一个#region块

    • #region和#endregion上下指令

    • 在上，下放置插入符号的文本文件中一行

    • 上下选择文本文件
