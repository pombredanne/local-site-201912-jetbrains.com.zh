解决方案范围内的代码检查

大多数JetBrains Rider的代码检查仅需要单个文件的源代码即可检测代码问题。除这些检查外，JetBrains Rider还提供解决方案范围内的代码问题检查，这些检查仅在整个解决方案范围内才可检测到，例如，检测未使用的非私有成员JetBrains Rider需要分析整个解决方案。

为了使解决方案范围的检查有效，您需要以下一项或两项：

  • 启用了简化的全局使用检查— 在关闭解决方案范围的分析时，显示未使用的非私有类型成员 编辑器|的“检查设置”页面 JetBrains骑士设置Ctrl+Alt+S 。

  • 该解决方案范围的分析已启用- 启用解决方案范围的分析 编辑器|的“检查设置”页面 JetBrains骑士设置Ctrl+Alt+S 。

请注意，即使符号在您的解决方案中没有直接使用，并且JetBrains Rider会警告您，但在某些情况下，符号可能会间接使用（例如，通过反射），也可以将它们设计为公共API。在所有这些情况下，您都希望抑制符号的使用检查，并且有几种方法可以这样做：

  • 推荐的方法是用代码注释属性修饰隐式使用的符号。为此，有两个属性： [UsedImplicitly]和[PublicAPI] ，它们在功能上相似，但是可以让您和您的队友了解如何实际使用该符号。

  • 您还可以使用任何自定义属性禁止使用情况检查检查。为此，请使用[MeansImplicitUse]属性标记该属性的定义。

  • 最后，您可以像抑制其他代码检查一样，通过抑制注释或抑制属性来抑制特定的使用检查。

与任何其他检查类似，您也可以禁用或更改任何解决方案范围的检查的严重性级别 。可以直接从Alt+Enter检查突出显示的代码问题上的菜单。或者，您可以禁用/启用或更改检查的严重性级别 编辑器|的检查严重性页面 JetBrains骑士设置Ctrl+Alt+S —整个解决方案的检查均标有“ 非私有可访问性”标签。请注意，您无法为解决方案范围的检查设置“错误”严重性。

启用解决方案范围的分析后，以与其他问题相同的方式在打开的文件中突出显示由解决方案范围的检查检测到的问题，并且在特定范围内运行代码检查时，这些问题也会显示在“ 检查结果”窗口 中 。但是，它们不会出现在“ 解决方案中的错误”窗口中 。

通常，JetBrains Rider为每次解决方案范围的检查提供一个或多个快速修复 。
