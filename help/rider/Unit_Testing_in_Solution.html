在项目或解决方案中使用单元测试

JetBrains Rider的单元测试运行器不允许运行两个或多个具有相同ID的测试。在实践中，这种情况很少发生，但是如果您确实有具有相同ID的测试，它们将在“单元测试”窗口中显示为单个节点，并且仅运行其中一个。
请注意，测试框架以不同的方式识别测试。例如，MSTest使用测试方法FQN，而NUnit使用测试属性的属性作为测试ID。

在解决方案中发现单元测试

对于单元测试管理，JetBrains Rider提供了“ 单元测试”窗口（“ 视图” |“工具”窗口|“单元测试” ）。 使用此窗口，您可以浏览和运行，调试或涵盖整个解决方案中所有受支持框架的单元测试。请注意，来自项目的单元测试仅在构建项目后出现在窗口中。当前打开的文件中的测试会自动更新，创建后的新文件会立即显示在单元测试浏览器中。

单元测试浏览器允许您执行以下操作：

  • 探索解决方案中的测试：浏览所有单元测试，按子字符串搜索测试，按项目，名称空间重新组合单元测试，等等。

  • 通过在视图中双击导航到任何测试或测试类的源代码。

  • 运行，调试或涵盖所选测试。

  • 从所选测试和测试类创建单元测试会话 ，和/或将所选项目添加到当前测试会话。

在项目或解决方案中执行并涵盖单元测试

您可以从单元测试资源管理器或解决方案资源管理器运行，调试或涵盖测试。单元测试资源管理器使您可以仅查看测试和测试类，而在使用其他窗口时，则需要知道哪些项目，文件和类包含测试。

  • 要从Unit Test Explorer执行测试，请选择所需的测试，然后单击运行单元测试 / 调试单元测试 或覆盖单元测试 在工具栏上，或使用相应的快捷方式（ Ctrl+U, R / Ctrl+U, D ）。

  • 要运行，调试或涵盖解决方案中的所有测试，请选择 测试从主菜单中的“ 从解决方案运行所有测试”或按Ctrl+U, L 。

无论选择哪种方式运行，调试或覆盖测试，您都将在“单元测试”窗口中看到执行进度，结果和输出。 如果存在开放的单元测试会话 ，则将已执行的测试添加到该会话中。如果没有测试会话或现有会话被锁定 ，则会创建一个新的测试会话。

如有必要，您始终可以通过按重复执行或最后执行的测试的覆盖率分析Ctrl+U, U或选择 测试在菜单中重复上一个运行 。

您还可以通过按来重新运行失败的测试Ctrl+U, F或选择 测试在菜单中重新运行失败的测试 。
