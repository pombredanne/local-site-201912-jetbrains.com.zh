运行和调试Unity测试

骑士尚不支持Unity播放模式测试。本节中的说明仅适用于编辑模式测试。

骑士通过其“ 单元测试”窗口提供对Unity测试的支持：

  • 从代码和Unity编辑器中发现Unity测试，并自动将其添加到“ 单元测试”窗口的“ 资源管理器”选项卡中。

  • 所有Rider单元测试功能也可用于Unity测试，包括分组，基于测试状态的过滤等。解析测试输出以使堆栈跟踪，类和方法名称成为超链接。

  • 您可以在Unity编辑器或Rider中添加新的Unity测试。可以通过解决方案资源管理器添加单元测试类：在上下文菜单中，选择“ 添加” ，然后选择“ 播放模式测试”或“ 编辑模式测试” 。

  • 单元测试窗口允许您选择要使用的单元测试运行器。例如，可能是：

    • 独立NUnit启动器 ：NUnit测试运行器 。

    • Unity编辑器-编辑模式 ：嵌入在Unity中的单元测试运行器，用于运行编辑模式测试。

  • 您可以在运行常规单元测试时运行和调试编辑模式Unity 测试 。
