JetBrains Rider 2019.2帮助

TypeScript的上下文操作

JetBrains Rider为TypeScript提供以下上下文操作：

名称

描述

更改可见性修改器

更改类和模块成员的可见性

将字段转换为参数属性

介绍从类字段声明为构造函数参数的参数属性

将枚举的'if'转换为'switch'

如果随后使用枚举值在'if'中检查了变量，则可以用开关替换

将Lambda表达式分配的成员转换为函数成员

由lambda表达式分配的类型成员通常可以转换为函数成员。如果您通过“ apply”进行隐式方法调用，则不能保证能正常工作。

将参数属性转换为字段

从构造函数中声明的参数属性中引入字段

创建派生类型

创建实现接口或继承类的类。

从用法创建

根据用法创建类，接口，枚举，模块，成员函数或成员属性
有关更多信息，请参见根据用法生成代码 。

创建不带参数的重载

在没有指定参数的情况下创建重载签名。如果需要，可以将主签名中的后续参数标记为可选。

创建配对访问器

为当前实现的访问器创建配对访问器

命名为嵌套模块

为点名段创建嵌套模块。例如，将带有嵌套模块“ C”的“模块ABC”转换为“模块” AB。

导入类型或类型元素

导入其他地方声明的模块，类，变量，接口，枚举

内联类型别名

内联类型别名，即用相应的类型替换其用法并删除类型别名声明

介绍类型别名

引入了用于选择的新类型别名。

将文件名与类型名匹配

重命名当前文件以匹配其中声明的唯一顶级类型的名称。

将类型移到另一个文件以匹配其名称

将当前类型移动到另一个文件，以便其名称与该类型的名称匹配。

嵌套模块到点名

将嵌套的单个导出模块合并为虚线名称。例如，将带有嵌套导出的“模块C”的“模块AB”转换为“模块ABC”。

用限定符替换别名

用完全限定的名称替换别名

明确指定返回类型

如果可以从返回值派生，则明确指定函数返回类型

明确指定类型

明确指定变量类型以进行赋值声明
