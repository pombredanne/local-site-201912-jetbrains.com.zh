运行/调试配置：.NET启动设置配置文件

请不要将运行/调试配置与构建配置混淆。

运行编辑配置| | .NET启动设置配置文件

此运行/调试配置类型可帮助您运行和调试在launchSettings.json文件中配置了启动设置的 .NET Core / ASP.NET Core应用程序。

什么是launchSettings.json

launchSettings.json （放置在项目的Properties文件夹中）描述了如何启动应用程序-要执行的命令，是否应打开浏览器，应设置哪些环境变量，等等。然后，JetBrains Rider， dotnet run或任何其他工具可以使用此信息来运行或调试应用程序。

本文详细介绍了launchSettings.json中使用的约定 。下面的示例显示了一个带有两个启动配置文件的示例launchSettings.json ： IIS Express和Sample 。

{“ iisSettings”：{“ windowsAuthentication”：false，“ anonymousAuthentication”：true，“ iisExpress”：{“ applicationUrl”：“ http：// localhost：43347”，“ sslPort”：44364}}，“ profiles”：{ “ IIS Express”：{“ commandName”：“ IISExpress”，“ launchBrowser”：true，“ environmentVariables”：{“ ASPNETCORE_ENVIRONMENT”：“ Development”}}，“ Sample”：{“ commandName”：“ Project”，“ launchBrowser “：true，” applicationUrl“：” https：// localhost：5001; http：// localhost：5000“，” environmentVariables“：{” ASPNETCORE_ENVIRONMENT“：” Staging“}}}}

基于launchSettings.json创建运行/调试配置

首次使用launchSettings.json打开项目时，JetBrains Rider将选择可用的启动配置文件并自动创建相应的运行/调试配置 ，然后您可以使用它们来运行和调试项目 。

仅当以前在解决方案中未创建用户定义的运行配置时，JetBrains Rider才会自动生成运行/调试配置。如果存在现有的运行/调试配置，则可以在解决方案资源管理器中右键单击launchSettings.json ，然后选择“ 生成配置”以将项目的启动配置文件导入运行/调试配置。

如果从解决方案中删除所有运行/调试配置，然后重新打开，JetBrains Rider将基于launchSettings.json中的启动配置文件生成新的运行/调试配置。

运行时变量：

在启动应用程序时，dotnet和Mono均支持设置自定义运行时参数。例如，Mono支持许多运行时参数以自定义应用程序的运行方式。您可以选择要使用的垃圾收集器（ --gc=boehm要么--gc=sgen ），在macOS上，您可以选择要使用的运行时架构（ --arch=32要么--arch=64 ）， 和更多。

使用单声道运行时如果您使用的是Windows，则可以检查应用程序如何与Mono运行时一起工作。JetBrains Rider将使用在上指定的mono可执行文件 构建，执行，部署|的工具集和构建页面 JetBrains骑士设置Ctrl+Alt+S 。
要使用Mono运行时进行调试，您需要：

  • 单声道4.2+

  • 您项目中的可移植PDB — portable在.csproj文件中，或者使用pdb2mdb生成mdb 。

使用外部控制台：

默认情况下，.NET Core和Mono应用程序的输出显示在“ 运行”或“ 调试”窗口中。使用此复选框可以在外部窗口中显示输出。
该复选框不会影响经典的.NET应用程序类型（例如.NET控制台应用程序），它们的输出始终显示在外部窗口中。

在启动配置文件中选择IIS Express启动配置文件时，可以使用以下四个选项：选择器。
生成applicationhost.config

为了使用ASP.NET Core应用程序启动IIS Express，JetBrains Rider使用配置文件.idea / config / applicationhost.config ，该文件是使用IIS Express附带的模板创建的。如有可能，还将保留您对此文件进行的自定义。您可能需要禁用此选项以进行大量自定义。

显示IIS Express输出

此选项启用IIS Express进程的输出。默认情况下，它是禁用的（因为输出非常详细），但是启用后，我们可以在运行应用程序时看到更多详细信息：

发送调试请求

启用此选项后，JetBrains Rider将发送初始请求（带有DEBUG HTTP请求方法）到目标ASP.NET Core应用程序。这样可以确保启动应用程序并连接调试器，即使未启动浏览器也是如此。

附加的IIS Express参数

该字段使您可以将其他参数传递给IIS Express主机（例如，启用请求跟踪）。
