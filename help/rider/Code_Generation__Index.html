代码生成

JetBrains Rider提供了多种生成样板代码的方法。例如，您可以使用未声明的代码符号，并根据用途自动生成这些符号，生成类型成员等。

如果在当前文件中启用了设计时代码检查 ，则JetBrains Rider会检测到缺少的成员并建议相应的快速修复程序以实现缺少的成员 ：

上下文动作和代码模板还提供了更多的代码生成方法。

在这个部分：

  • 根据使用情况生成代码
  • 生成空检查例程
  • 生成代码并完成代码
  • 生成GUID
  • 生成类型成员
    • 生成解构函数
    • 产生委派成员
    • 生成处置模式
    • 生成平等比较器
    • 产生平等成员
    • 生成格式化成员
    • 生成部分方法
    • 产生属性
    • 生成关系比较器
    • 产生关系成员
    • 生成类型构造器
    • 实施/覆盖成员

您可以通过两种主要方式控制新代码的生成方式：

  • 配置代码样式的各个方面（例如， 命名样式 ， 格式规则 ）JetBrains Rider在生成代码时将强制执行其要求。

  • 根据您的设置，可以用不同的方式生成新成员的存根。例如，他们可以抛出新的NotImplementedException() ，返回默认值或包含将无法编译的代码。您可以在以下位置配置这些首选项和其他首选项 编辑器|的会员生成页面 JetBrains骑士设置Ctrl+Alt+S 。
