JetBrains Rider 2019.2帮助

在数据库树视图中找不到数据库对象

如果您在架构级别下没有看到任何对象，找不到对象的更改，遇到表损坏或任何其他可视化问题，请尝试以下步骤解决问题。

步骤1。刷新数据库状态

如果有人更改了远程数据库数据或视图，则数据库的本地视图可能与数据库的实际状态不同。

要自动同步数据库状态，请在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”按钮并选择要更改的数据源。在选项选项卡上，选择自动同步复选框。

如果清除了自动同步复选框，则仅当您单击同步图标时， 数据库工具窗口（ 视图|工具窗口|数据库 ）中数据源的视图才会与数据库的实际状态同步。 或按Ctrl+Alt+Y 。

第2步。验证是否已选择必要的架构

您可以选择要在数据库树视图中显示的架构。

  • 在“ 数据库工具”窗口（“ 视图” |“工具窗口” |“数据库” ）中，右键单击数据源，然后导航至“ 数据库工具” |“数据库”。管理显示的架构 。选择或清除要启用或禁用的架构的复选框。按Enter 。

  • 单击数据源名称附近的N of N链接。在架构选择窗口中，选中或清除要启用或禁用的架构的复选框。按Enter 。

  • 要隐藏所有模式，请导航至“ 数据库工具” |“数据库”。隐藏架构 。

第三步强制刷新架构信息

“ 强制刷新”操作从缓存中清除数据源信息，然后从头开始再次加载它。

在“ 数据库工具”窗口（“ 视图” |“工具窗口” |“数据库” ）中，右键单击数据源，然后选择“ 数据库工具” |“数据库”。强制刷新 。

第四步。清除并同步架构缓存

清除JetBrains Rider模式缓存（右键单击数据源，然后选择“ 数据库工具” |“忘记缓存的模式” ）。同步视图（请参阅步骤1。同步数据库状态 ）。

如果步骤1、2和3无法解决您的问题，请通过创建Youtrack票证告知JetBrains您的情况。创建故障单后，请尝试执行步骤4和5作为解决方法。

第五步切换到基于JDBC的内省器

要切换到基于JDBC的自省工具，请单击“ 数据源属性”按钮 ，选择“ 选项”选项卡，然后选择“ 使用JDBC元数据进行内省”复选框。同步视图（请参阅步骤1。同步数据库状态 ）。将此步骤视为临时解决方法。
