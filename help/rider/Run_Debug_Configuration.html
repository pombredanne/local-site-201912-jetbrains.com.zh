使用运行/调试配置

请不要将运行/调试配置与构建配置混淆。

要在JetBrains Rider中运行或调试代码，可以使用许多运行/调试配置。每个运行/调试配置代表一组命名的运行/调试启动属性。当您使用JetBrains Rider执行运行，调试或测试操作时，总是使用其参数基于现有配置之一启动进程。

JetBrains Rider随附了许多运行/调试配置类型，可解决各种运行，调试和测试问题。您可以创建自己的特定类型的运行/调试配置。

每种运行/调试配置类型都有其自己的默认设置。每当创建相应类型的新运行/调试配置时，它都会基于这些默认设置。

临时运行/调试配置

每次您运行一个临时运行/调试配置时，都会自动创建一个public static void编辑器中的方法。

将临时配置另存为永久配置

  • 在“ 运行/调试配置”对话框中选择临时配置，然后单击在工具栏上。

  • 或者，在运行/调试配置选择器中选择临时配置，然后单击保存配置 。

临时配置用半透明图标标记，并且与永久配置的管理方式相同。
默认情况下，每个解决方案允许五个临时运行/调试配置，因此当您创建第六个配置时，将删除第一个创建的配置，依此类推。要更改此限制，请使用“ 运行/调试配置”对话框的“ 模板”页面中的“ 临时配置限制”字段。

运行/调试配置模板

具有默认运行/调试配置设置的模板在“ 模板”节点下的“ 运行/调试配置”对话框中列出。它们表示创建新的运行/调试配置时使用的设置。

您可以为特定配置类型设置默认设置，该设置将适用于以后创建的该类型的任何运行/调试配置。更改默认值不会影响现有的运行/调试配置。

更改运行/调试配置模板中介绍了编辑每个类型的默认配置设置的过程。创建和编辑运行/调试配置中介绍了创建或编辑自定义运行/调试配置的过程。
