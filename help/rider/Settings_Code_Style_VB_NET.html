制表符，缩进，对齐

此选项卡可帮助您指定键入或重新格式化现有代码时，JetBrains Rider如何格式化代码中的缩进 。

缩进设置源

默认情况下，JetBrains Rider对选项卡和缩进使用Visual Studio首选项（ 工具|选项|文本编辑器|所有语言|选项卡和工具|选项|文本编辑器| [语言] |选项卡 ）。但是，如果您希望使用JetBrains Rider的分层设置与您的团队共享格式化程序首选项，则可以清除“ 使用Visual Studio中的缩进样式和大小”复选框，然后配置缩进首选项并将其保存到团队共享的设置层。

如果有.editorconfig文件影响您的解决方案，则此菜单以及其他具有代码样式设置的选项卡上的首选项可能会被EditorConfig样式覆盖。您如果页面上的至少一个首选项被当前文件的EditorConfig样式覆盖，将显示黄色警告，每个被覆盖的首选项也将以黄色突出显示。例如：

多行构造中的缩进和对齐相结合

如果您的喜好要求缩进应使用空格，JetBrains Rider还将使用空格来对齐多个结构-例如，当多行的函数参数被第一个参数对齐时：

无效的foo（int firstParameter，int secondParameter）{}

但是，当您选择对缩进使用制表符时，可能会有不同的方式来对齐多行构造。使用JetBrains Rider，您可以选择在创建新代码并重新格式化现有代码时使用哪种方式。

您可以使用“选项卡用于缩进时如何对齐”选项配置用于对齐多行结构的首选项。

  • 使用空格（建议，看起来与任何制表符大小对齐）
    选择此选项后，JetBrains Rider使用制表符来缩进，并使用空格来对齐：

    推荐使用此选项，因为与第二个和第三个选项对齐的代码在带有不同选项卡大小的编辑器中查看时可能会失去对齐。

  • 仅使用标签（不准确）
    选择此选项后，JetBrains Rider会将选项卡用于缩进和对齐，这可能不会导致精确对齐：

  • 混合制表符和空格以实现最佳填充
    选择此选项后，JetBrains Rider会将选项卡用于缩进，并且对齐方式会添加必要的空间以进行精确对齐。

命名

从2018.3版本开始，JetBrains Rider可以从现有代码中自动学习C＃符号命名规则 。如果要手动配置命名规则，请清除“ 使用自动检测到的规则”复选框 编辑器|代码样式| C＃|的命名页面 JetBrains骑士设置Ctrl+Alt+S 。

在此选项卡上，您可以为VB.NET配置符号命名规则 。

JetBrains Rider可帮助您定义，控制和应用代码中符号的命名方式。命名样式是作为一组规则实现的，每个规则都针对具有一组约束的特定标识符（例如，一条规则可以针对静态私有只读字段）。每个规则可以具有一个或多个关联的样式，这些样式定义后缀，前缀，复合词的大写形式，等等。

JetBrains Rider会考虑这些规则产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。JetBrains Rider还可以帮助您检测和修复违反命名规则的情况。如有必要，可以配置或禁用自动检查命名规则。

代码风格

JetBrains Rider会考虑在此选项卡上可配置的首选项产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。通过使用带有相应设置的代码清除 ，它们也可以应用于现有代码。

带有“ 通知方式”选择器的首选项具有相应的代码检查，这些检查会通知您检查范围内的代码样式的这一方面是否与首选样式不同。使用选择器，您可以配置检查的严重性级别 。

空行

此选项卡使您可以配置JetBrains Rider是否应增加或减少导入指令的名称空间，成员，区域和组周围的空行数。 您可以调整值并检查页面底部的预览窗格，以查看首选项如何影响代码。

“ 保留现有格式”部分中的选项仅在JetBrains Rider 重新格式化现有代码的格式时适用，而“ 空白行”部分包含的选项也会在您键入新代码时生效。

换行和包装

使用此选项卡可配置JetBrains Rider如何在特定语言构造之前/之后添加或删除换行符，以及是否换行超过右页边距（列）指定的长度的长行。 您可以调整值并检查页面底部的预览窗格，以查看首选项如何影响代码。

请注意，名称以“ 保持现有...”开头的首选项允许您保留同一组中其他首选项的现有格式 。

空间

此选项卡用于配置在源代码的某些位置自动插入/删除空格。 您可以调整值并检查页面底部的预览窗格，以查看首选项如何影响代码。

其他

此选项卡可帮助配置许多其他设置，这些设置控制JetBrains Rider如何处理新代码并重新格式化现有代码 。 您可以调整值并检查页面底部的预览窗格，以查看首选项如何影响代码。
