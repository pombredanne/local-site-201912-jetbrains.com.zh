代码产生|关系比较器
Alt+Insert | 关系比较器

JetBrains Rider可帮助您创建一个嵌套的比较器类，该类从Comparer使用选定的字段和属性。除了选择字符串比较算法并生成空检查以外，您还可以选择生成静态属性，该属性将比较器类向使用者公开。

在下面的示例中，此命令用于生成考虑到相关关系的比较器int _birthYear和string _name何时比较对象的字段Person类。

生成关系比较器类

  1. 在编辑器中，将插入号设置为类型名称，或者在要插入关系比较器类的行的类型内。如果插入符号在类型名称上，则生成的代码将添加到类型声明的开头。

  2. 按Alt+Insert或选择 代码从主菜单生成...。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. 在“ 生成”弹出窗口中，选择“ 关系比较器” 。

  4. 在出现的“ 生成”对话框中，您将看到属性和字段的列表。选择要用于类型比较的成员。

    如果您的类型中没有字段/属性，或者您没有选择任何字段/属性，JetBrains Rider会根据您的设置抛出新的NotImplementedException() ，返回默认值，或将无法编译的代码放入生成的方法的主体中。

    （可选）在对话框中使用以下控件：

    • 字段可以为null-如果您的类型中包含任何可为空的字段或属性，则显示该字段。默认情况下，此复选框处于选中状态，这意味着JetBrains Rider将为所选字段生成空检查。如果不需要空检查，则可以清除此复选框。

    • 字符串比较 -如果您的类型中有任何字符串字段，则会出现。JetBrains骑士将生成string.Compare(string, string, StringComparison)对于选定的字符串，选择器使您可以选择为StringComparison参数生成的值。

    • 通过静态属性公开-将比较器类设为私有，并生成将比较器类向使用者公开的静态属性。

    • 比较器名称前缀 -允许您指定将在生成的比较器类的名称中使用的前缀。

  5. 单击“ 确定”完成向导。
