运行/调试配置：适用于Node.js的Spy-js

请不要将运行/调试配置与构建配置混淆。

运行编辑配置| |适用于Node.js的Spy-js

在此对话框中，使用Spy-js工具创建用于跟踪Node.js应用程序的配置。

在你开始前

  1. 下载并安装Node.js。

  2. 如管理插件中所述，在“ 插件”页面上安装NodeJS和Spy-js存储库插件 。

Spy-js特定于Node.js的配置设置

项目

描述

节点解释器

在这个领域里， 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

对于Spy-js，建议您使用Node.js版本0.10.24或更高版本。

要启用跟踪ECMAScript 6脚本，请使用Node.js版本0.11.13或更高版本，并指定--harmony作为Node参数。

节点参数

在此字段中，键入要传递到NodeJS可执行文件的特定于Node.js的命令行选项。 例如，要启用跟踪ECMAScript 6脚本，请指定--harmony作为Node参数。请注意，Node.js必须为0.11.13或更高版本。

工作目录

在此字段中，指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

JavaScript文件

 在此字段中，指定启动它的应用程序主文件的路径（例如，对于Node.js Express应用程序 ，为bin / www ）。

如果要跟踪CoffeeScript，请指定生成的JavaScript文件的路径。可以从外部生成文件，也可以使用文件观察程序进行编译。

应用参数

在此文本框中，键入通过process.argv数组开始时要传递给应用程序的特定于Node.js的参数。

环境变量

 在此字段中，为Node.js可执行文件指定环境变量 （如果适用）。 点击浏览 在字段右侧，并在“ 环境变量”对话框中配置变量列表，将打开：

  • 要定义新变量，请单击并指定变量名称和值。

  • 要放弃变量定义，请在列表中选择它，然后单击 。

  • 准备好后，单击确定 。

变量的定义显示在“ 环境变量”只读字段中，以分号作为分隔符。可接受的变量是：

  • NODE_PATH ： 一种: -前缀为模块搜索路径的目录的列表。

  • NODE_MODULE_CONTEXTS ：设置为1可以在各自的全局上下文中加载模块。

  • NODE_DISABLE_COLORS ：设置为1以禁用REPL中的颜色。

配置文件

从此下拉列表中，选择具有配置设置的文件以应用于跟踪会话。

配置文件是一个扩展名为.js或.conf.js的JavaScript文件，其中包含符合Spy-js配置要求的有效JavaScript代码。如果JetBrains Rider在项目中检测到扩展名为.conf.js的文件，则这些文件将显示在下拉列表中。

手动输入配置文件的路径或单击浏览 然后在打开的对话框中选择位置。指定后，会将配置文件添加到下拉列表中，以便下次从列表中获取（而不是指定路径）。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Ctrl+Shift+Alt+R 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

工具列

运行/调试配置的树形视图有一个工具栏，可帮助您管理解决方案中可用的配置以及调整默认配置模板。

单击此按钮以按字母顺序对配置进行排序。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Ctrl+N

单击此图标添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在JetBrains Rider中定义它，请添加其定义。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 构建解决方案 ：选择此选项可在启动配置之前构建解决方案。

  • 运行文件监视程序 ：选择此选项可使JetBrains Rider应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 编译TypeScript ：选择运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都会反映在生成的JavaScript文件中。 在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防万一检测到任何错误：

    • 如果选中“检查错误”复选框，则编译器将显示所有错误，并且运行配置将不会启动。

    • 如果清除“ 检查错误”复选框，则编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 将文件上传到远程主机 ：选择此选项可以根据默认服务器访问配置将应用程序文件自动上传到服务器 。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

Ctrl+R, D

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（这些任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
