JetBrains Rider 2019.2帮助

代码检查：可以将成员设为静态（共享）（私有可访问性）

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

考虑方法Print下面：

公共类Foo {public void Test（）{Print（“ John”）; } private void Print（字符串名称）{控制台。WriteLine（“您好，{0}！“， 名称）; } }
公共类Foo Public Sub Test（）Print（“ John”）End Sub Private Sub Print（名称为String）控制台。WriteLine（“您好，{0}！“，名称）End Sub End Class

JetBrains Rider建议Print没有实例用法，可以设置为静态。但是有什么意义呢？事实证明，静态成员在特定情况下只能带来很小的性能优势。

这是MSDN文档必须说明的内容：

-不访问实例数据或调用实例方法的成员可以标记为静态（在Visual Basic中为Shared）。将方法标记为静态后，编译器将向这些成员发出非虚拟调用站点。发出非虚拟调用站点将阻止在运行时检查每个调用，以确保当前对象指针为非null。对于性能敏感的代码，这可以实现可衡量的性能提升。在某些情况下，无法访问当前对象实例表示正确性问题。
