代码检查：可以将成员设为静态（共享）（非私有可访问性）

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

这是解决方案范围内的代码检查 。它仅在启用解决方案范围的分析时才有效。

考虑方法Print下面：

公共类Foo {public void Test（）{Print（“ John”）; } public void Print（字符串名称）{控制台。WriteLine（“您好，{0}！“， 名称）; } }
公共类Foo Public Sub Test（）Print（“ John”）End Sub Public Sub Print（名称为String）控制台。WriteLine（“您好，{0}！“，名称）End Sub End Class

如果启用了解决方案范围的检查， JetBrains Rider建议Print没有实例用法，可以设置为静态。但是有什么意义呢？事实证明，静态成员在特定情况下只能带来很小的性能优势。

这是MSDN文档必须说明的内容：

-不访问实例数据或调用实例方法的成员可以标记为静态（在Visual Basic中为Shared）。将方法标记为静态后，编译器将向这些成员发出非虚拟调用站点。发出非虚拟调用站点将阻止在运行时检查每个调用，以确保当前对象指针为非null。对于性能敏感的代码，这可以实现可衡量的性能提升。在某些情况下，无法访问当前对象实例表示正确性问题。

为了使解决方案范围的检查有效，您需要以下一项或两项：

  • 启用了简化的全局使用检查— 在关闭解决方案范围的分析时，显示未使用的非私有类型成员 编辑器|的“检查设置”页面 JetBrains骑士设置Ctrl+Alt+S 。

  • 该解决方案范围的分析已启用- 启用解决方案范围的分析 编辑器|的“检查设置”页面 JetBrains骑士设置Ctrl+Alt+S 。

请注意，即使符号在您的解决方案中没有直接使用，并且JetBrains Rider会警告您，但在某些情况下，符号可能会间接使用（例如，通过反射），也可以将它们设计为公共API。在所有这些情况下，您都希望抑制符号的使用检查，并且有几种方法可以这样做：

  • 推荐的方法是用代码注释属性修饰隐式使用的符号。为此，有两个属性： [UsedImplicitly]和[PublicAPI] ，它们在功能上相似，但是可以让您和您的队友了解如何实际使用该符号。

  • 您还可以使用任何自定义属性禁止使用情况检查检查。为此，请使用[MeansImplicitUse]属性标记该属性的定义。

  • 最后，您可以像抑制其他代码检查一样，通过抑制注释或抑制属性来抑制特定的使用检查。
