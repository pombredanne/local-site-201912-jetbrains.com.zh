查找特定范围内的代码问题

当您在编辑器中进行编码或查看特定文件时， 设计时代码检查将为您提供很大的帮助。但是，JetBrains Rider还允许您在整个解决方案或更窄的范围内运行静态代码分析，并在友好的视图中检查结果。

开始代码检查

在解决方案，项目或自定义范围中检查代码

  1. 决定是否要在报告中包括解决方案范围内的检查结果 （例如，未使用的公共成员）。如果要包括这些结果，请启用解决方案广泛分析 。请注意，启用解决方案范围的分析后，即使您需要检查单个项目，JetBrains Rider也必须分析整个解决方案，因此请做好准备，以免花费更多时间。

  2. 请执行以下任一操作：

    • 在主菜单中，选择 代码检查代码...

    • 您还可以通过选择一次检查来检查所需范围内的代码-例如，您可能想检查是否存在可以转换为自动属性的属性。
      要进行一次检查，请按Ctrl+Shift+Alt+I或选择 代码从主菜单运行“按名称检查... ”，然后开始输入检查的名称：

    • 在“ 解决方案资源管理器”中 ，选择要检查的项目（文件，文件夹，项目，解决方案文件夹），右键单击所选内容，然后在上下文菜单中选择“ 检查代码... ”。

  3. 在打开的对话框中，指定检查代码的范围。

  4. “ 检查结果”窗口将打开，显示检测到的代码问题。

分析检查结果

在“ 检查结果”窗口中，您可以查看发现的代码问题的列表，并通过双击以下条目在编辑器中找到它们：

每次新的代码检查运行都会在窗口中打开一个新选项卡 ，因此将保留以前运行的结果，直到您关闭其选项卡为止。

（可选）您可以打开自动滚动到源 这样一来，只要在工具窗口中找到问题，便可以在编辑器中找到问题。
