节点解释器

在这个领域里， 指定当前项目的默认Node.js解释器。每当您选择以下选项时，JetBrains Rider都会自动使用它Project创建运行/调试配置或配置依赖于Node.js的工具（例如Prettier或Yeoman）时， Node Interpreter列表中的别名。

从列表中或单击时打开的对话框中选择解释器 。如果选择node ，则使用系统Node.js版本 。

在这里，您只能选择或配置本地Node.js解释器，即计算机上安装的Node.js或Windows Subsystem for Linux上的Node.js ，请参阅配置本地Node.js解释器和使用Node.js。有关详细信息，请参见Windows Subsystem for Linux 。

可在“ 运行/调试配置：Node.js”中访问“ 配置Node.js远程解释器”对话框中配置远程解释器 。有关详细信息，请参见使用Docker的 Node.js，通过SFTP的 Node.js，通过SSH的 Node.js以及通过Vagrant的Node.js以获得详细信息。

Node.js的编码帮助

选中此复选框将Node.js Core模块源配置为JavaScript库并将其与您的项目关联。结果，JetBrains Rider为以下人员提供了代码完成，参考解析，验证和调试功能： fs ， path ， http以及编译到Node.js二进制文件中的其他核心模块。

配置完成后，JetBrains Rider 显示有关当前配置版本的信息。

可选地
配置将Node.js Core源视为库的范围：

  1. 单击管理范围 。“ 使用范围”对话框打开。

  2. 单击相关目录，然后从列表中为每个目录选择新配置的Node.js Core库。

配套

软件包区域显示计算机上当前安装的所有NPM相关软件包的列表。

  • 软件包：此只读字段显示软件包的名称，与在命令行模式下安装软件包时应引用的名称完全相同。

  • 版本：此只读字段显示计算机上安装的软件包的版本。

  • 最新：此只读字段显示软件包的最新发行版本。如果软件包不是最新的，则标记为 。

  • 请点击安装新软件包。在打开的“ 可用软件包”对话框中，选择相关的软件包。要全局安装软件包，请选择选项复选框，然后键入-g在选项字段中。全局安装使该软件包在JetBrains Rider级别可用，因此可以在任何JetBrains Rider项目中使用。准备好后，单击“ 安装软件包” 。
