JetBrains Rider代码编辑器中的HTTP客户端

测试Web服务时，可以直接在JetBrains Rider 代码编辑器中创建，编辑和执行HTTP请求。

HTTP请求存储在.http和.rest文件中，并标有图标。

对HTTP文件的支持包括以下功能：

  • 代码突出显示

  • 方法类型和标头字段的代码完成（IntelliSense）

  • 内联文档，用于请求标头字段和doc标签

  • 查看 HTTP请求的结构

  • 请求消息正文中的Web语言中的语言注入

  • 移动重构
  • 实时模板

在开始之前，如有必要，请在“设置/首选项 Ctrl + Alt + S”对话框的“ HTTP代理”页面上配置代理设置 。

通过捆绑的HTTP客户端插件支持测试Web服务 ，默认情况下，该插件在“设置/首选项” |“默认设置”中启用。插件 。

创建HTTP请求文件

您可以从临时文件或HTTP请求类型的物理文件处理HTTP请求 。每个文件可以包含多个请求，并且您可以根据需要创建任意数量的文件。

临时文件可用于在开发过程中测试HTTP请求。临时文件未存储在项目中，因此JetBrains Rider可以修改它们并添加有关请求的其他信息。当从头文件执行HTTP请求时，指向响应输出文件的链接将添加到请求下方和请求历史记录文件顶部。

创建一个HTTP请求暂存文件

  • 按Ctrl + Shift + Alt + Insert并选择HTTP请求 。

  • 切换到“解决方案”窗口中的Scratches视图，右键单击要在其中添加请求文件的文件夹，然后选择“ 添加” |“添加”。上下文菜单中的HTTP请求 。

物理文件可用于记录，测试和验证HTTP请求。物理文件存储在您的项目中，JetBrains Rider不会修改它们。从物理文件执行HTTP请求时，不会修改此文件。带有响应输出文件链接的有关已执行请求的信息将添加到请求历史记录文件的顶部。

创建一个物理HTTP请求文件

  • 右键单击要在其中添加请求文件的文件夹，然后选择“ 添加” |“添加”。上下文菜单中的HTTP请求 。

您可以使用Move refactoring Ctrl + R，O将HTTP请求从头开始移动到物理文件，也可以在物理文件之间移动。

移动HTTP请求

  1. 在编辑器中，将插入符号放置在要移动的请求上，然后执行以下操作之一：

    • 从主菜单或上下文菜单中，选择“ 重构|移动 。

    • 按Ctrl + R，O 。

  2. 在打开的对话框中，单击选择文件或键入要将请求移至的文件的完整路径。请注意，您可以指定不存在的文件的名称，在这种情况下，将自动创建具有提供的名称的新文件。

转换cURL请求

如果使用cURL请求，则可以将它们转换为Editor格式的HTTP请求 。

  • 将cURL请求粘贴到.http文件中，它将即时进行转换。

  • 或者，选择工具| HTTP客户端|从主菜单将cURL转换为HTTP请求 。在打开的对话框中，提供cURL请求。

    单击“ 转换”时 ，将创建一个具有转换后请求的新暂存文件，并在编辑器中将其打开。

使用环境变量

编写HTTP请求时，可以使用环境变量来参数化其元素。例如，您可以使用{{host}}占位符来代替在请求中显式提供主机名。然后，在项目中定义一组环境变量，其中包含所需的主机定义。执行请求时，可以在定义的环境之间进行选择，例如，将请求发送到的主机 ：

在查看请求的结构并在浏览器中打开请求或为其创建运行/调试配置时，所选环境将用作默认环境。

变量的名称可以是任意的，但只能包含字母，数字，下划线符号_或连字符- 。变量可以保存以下请求组件的值：

  • 请求主机，端口和路径

  • 查询参数或值

  • 标头值

  • 直接在请求正文中提供或在外部文件中提供的任意值。

定义环境变量

环境变量在环境文件中定义。

  1. 在项目内部，创建以下文件：

    • rest-client.env.json （或http-client.env.json ）是一个包含公共变量的常规文件，旨在与您的项目一起分发。

    • rest-client.private.env.json （或http-client.private.env.json ）是一个私有文件，可能包含密码，令牌，证书和其他敏感信息。默认情况下，此文件被添加到VCS忽略文件列表中。在http-client.private.env.json文件中指定的变量的值将覆盖环境文件中的值。

  2. 用所需的变量填充创建的文件。

    以下示例http-client.env.json环境文件定义了两个环境： development和production 。附加的http-client.private.env.json文件包含敏感授权数据。

    {“ development”：{“ host”：“ localhost”，“ id-value”：12345，“ username”：“”，“ password”：“”，“ my-var”：“ my-dev-value”} ，“生产”：{“主机”：“ example.com”，“ id值”：6789，“用户名”：“”，“密码”：“”，“ my-var”：“ my-prod-value “}}
    {“开发”：{“用户名”：“开发用户”，“密码”：“开发密码”}，“生产”：{“用户名”：“用户”，“密码”：“密码”}}

    示例HTTP请求如下：

    获取http：// {{host}} / api / json / get？id = {{id-value}}＆key = {{unresolved_var}}授权：基本{{username}} {{password}}内容类型：application / json {“ key”：{{my-var}}}

    当您执行以上请求时，JetBrains Rider在定义的执行环境之间提供一个选择：

    根据您的选择，最终的请求将是以下之一：

    GET http：// localhost / api / json / get？id = 12345＆key = {{unresolved_var}}授权：基本的dev-user dev-password内容类型：application / json {“ key”：“ my-dev-value” }
    GET http://example.com/api/json/get?id=6789&key={{unresolved_var}}授权：基本用户密码内容类型：application / json {“ key”：“ my-prod-value”}

    由于在环境文件中未定义{{unresolved-var}}变量，因此在两种情况下，JetBrains Rider都将发送{{unresolved-var}}文本作为请求的一部分。

您可以通过client.global.get和client.global.set方法在响应处理程序脚本中以编程方式管理变量。

使用响应处理程序脚本

使用响应处理程序脚本，您可以以编程方式对收到的HTTP响应做出反应。通过使用这些脚本，您可以自动处理接收到的数据，并根据您指定的条件对其进行验证。响应处理程序脚本作为HTTP请求文件中请求的一部分提供，并在收到响应后立即执行。要查看响应处理示例，请打开auth-requests或test-responses 请求集合 。

您可以就地或通过引用外部文件将响应处理程序脚本插入请求中。

将脚本插入请求中

  • 要就地插入脚本，请在其前面加上>并将其包含在{% %} ：

    GET host / api / test> {％//响应处理程序脚本...％}

  • 要从外部文件插入脚本，请在其前面加上> ：

    GET host / api / test> scripts / my-script.js

响应处理程序脚本使用JavaScript ECMAScript 5.1编写，并通过捆绑的HTTP Response Handler库处理编码帮助和文档。对于就地脚本，此功能会自动启用。对于外部脚本，您需要手动启用它。

在JavaScript文件中启用HTTP响应处理程序编码帮助

  1. 在编辑器中打开脚本文件。

  2. 在上下文菜单中，选择“ 使用JavaScript库”。 HTTP响应处理程序 。

有关详细信息，请参阅配置JavaScript库 。

HTTP Response Handler库公开了两个用于构成响应处理程序脚本的对象：

  • client对象存储会话元数据，可以在脚本内部对其进行修改。client状态将保留，直到您关闭JetBrains Rider。后续的HTTP请求都可以使用{{variable_name}}来访问client.global保存为variable_name每个变量。

  • response保存有关已接收响应的信息：其内容类型，状态，响应主体等。

要在编辑器中打开HTTP响应处理程序库，请将插入符号放置在库对象上，然后按F12键。

响应处理程序脚本可以包括测试，该测试允许您将HTTP客户端用作测试框架。要创建测试，请调用client.test(testName, function)方法。在测试内部，您可以通过调用client.assert(condition, message)方法来断言一个条件，例如：

GET https://httpbin.org/status/200> {％client.test（“请求成功执行”，function（）{client.assert（response.status === 200，“响应状态不是200”）； }）; ％}

使用HTTP请求运行/调试配置

您可以通过使用HTTP请求类型的运行配置来执行HTTP请求。使用运行配置，您可以随时执行请求，将其添加到“ 服务”工具窗口或复合配置中 。

有关详细信息，请参考使用运行/调试配置 。

创建HTTP请求运行/调试配置

  1. 请执行以下任一操作：

    • 在编辑器中，选择创建从您要为其创建运行/调试配置的请求的上下文菜单中。

    • 或者，选择运行|从主菜单中编辑配置 ，然后单击并从列表中选择HTTP请求 。

  2. 提供运行/调试配置参数：

    • 在环境列表中， 选择一个环境 ，该环境将定义请求中使用的环境变量集。

    • 在文件字段中， 提供HTTP请求文件的路径。您可以手动输入路径，并在输入时使用路径补全Ctrl + Space或单击然后在打开的对话框中选择所需的文件夹。

    • 如果您的请求文件包含多个请求，请在“ 请求”列表中选择要执行的请求的索引。

当您从编辑器执行HTTP请求时 ，JetBrains Rider会自动使用请求参数创建一个临时运行/调试配置。临时运行/调试配置与永久运行/调试配置的工作方式相同。您可以使用“ 运行/调试配置”对话框更改其设置，还可以选择将其保存为永久文件。

保存临时HTTP请求运行/调试配置

    • 在编辑器中，选择“ 保存” 从已执行请求的上下文菜单中。

查看来自Web服务的响应

当您执行HTTP请求时，JetBrains Rider会自动将响应保存到.idea / httpRequests /目录下的单独文件中。您可以查看50个最近存储的响应，并使用请求历史记录导航到相应的文件。如果该请求是从头文件执行的，则其响应输出的链接也会添加到原始请求的下方：

通过响应收到的cookie将自动保存到.idea / httpRequests /目录下的专用http-client.cookies文件中。可以保存的Cookie数量限制为300个 。如果未到到期日期 ，则在对URL的每个后续请求中都会自动包含cookie的名称和值 ，该URL与为cookie指定的域和路径相匹配。

如有必要，可以通过在请求之前添加带有@ no-cookie-jar标记的注释行来防止将接收到的cookie保存到cookie罐中。

查看请求历史记录

JetBrains Rider自动将最近执行的50个请求保存到http-requests-log.http文件中，该文件存储在项目级别的.idea / httpRequests /目录下。使用请求历史记录，您可以快速导航到特定的响应并再次发出任何请求 。如果从请求历史记录再次发出请求，则其执行信息和到响应输出的链接将添加到请求历史记录文件的顶部。

打开请求历史记录

  • 请点击在编辑器的右上角，或选择工具| HTTP客户端|从主菜单显示HTTP请求历史记录 。

如有必要，可以通过在请求之前添加带有@ no-log标记的注释行来防止将请求保存到请求历史记录 。如果请求包含一些敏感数据，并且您不想对其进行记录，这将很有帮助。
