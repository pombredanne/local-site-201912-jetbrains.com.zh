代码检查：具有“ [NotNull]”属性的成员未在构造函数出口处初始化

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

此代码检查警告您，标有[NotNullAttribute]的字段或属性的协定可能无效，因为在创建对象时该成员仍然可以为null。

在下面的示例中，JetBrains Rider警告您， PlaceOfBirth未初始化：

类Person {[NotNull]公共字符串Name {get;组; } [NotNull]公共字符串PlaceOfBirth {get;组; } //警告：具有“ [NotNull]”属性的成员未初始化public Person（[NotNull]字符串名称，字符串placeOfBirth）{名称=名称；安慰。WriteLine（@“名称：{0}，出生地：{1}”，名称，placeOfBirth）； }}

请注意，检查仅检查构造函数代码中成员的用法。因此，如果您使用构造函数中调用的其他方法初始化该成员，JetBrains Rider仍会警告您未初始化的成员。在这种情况下，请通过以下注释取消检查： // ReSharper disable once NotNullMemberIsNotInitialized
