调试窗口

查看|工具窗口|除错
Alt+5

当您开始调试时，此工具窗口将变为可用。

它显示调试会话为您的应用程序生成的输出。如果要调试多个应用程序，则每个应用程序的输出将显示在以相应的run / debug配置命名的单独选项卡中。

对于每个应用程序，都有以下嵌套选项卡：

  • 控制台 ：显示系统信息和错误消息，以及应用程序的控制台输入和输出。

  • 调试器 ：此选项卡分为以下几个区域：

    • 框架 / 螺纹

    • 变数

    • 手表

  • Elements ：如果您使用Chrome浏览器进行调试，则会显示。

  • 调试输出 ：显示您的应用程序发送到调试输出的消息，例如带有Debug.WriteLine()方法。

  • 并行堆栈 ：在图中显示线程及其堆栈框架。

每个区域都有一个上下文菜单 ，可让您配置其行为并在选项卡之间导航。

每个选项卡和区域都可以隐藏/还原 ，或移动到您选择的位置。

步入
F11

单击此按钮可使调试器进入当前执行点调用的方法。

调试器将在从当前行调用的第一个函数的第一条语句处设置执行指针。如果未调用任何函数，则调试器将转到下一条语句。

您还可以检查调用层次结构，而无需调试程序。

默认情况下，JetBrains Rider启用外部源调试 ，这意味着当您进入库代码时，它将自动反编译，并且调试器会将执行点移至该位置。
如果希望调试器忽略库代码，则可以禁用外部源调试- 清除启用外部源调试 构建，执行，部署|的调试器页面 JetBrains骑士设置Ctrl+Alt+S 。

跑到声明
Ctrl+F10

单击此按钮可恢复程序执行并暂停直到执行点到达编辑器中当前光标位置的行。无需断点。实际上，在插入符号处为当前行设置了一个临时断点，一旦程序执行暂停，该断点将被删除。因此，如果插入符号位于已经执行的行上，则由于无法回滚到先前的断点，因此将仅继续执行程序以进一步执行。当您深入到方法序列并需要一次退出多个方法时，此操作特别有用。

如果在将您带到指定的行之前为应该执行的行设置了断点，则调试器将在遇到的第一个断点处暂停。

如果启用了“ 显示浮动调试器操作 ” 构建，执行，部署|的调试器页面 JetBrains骑士设置Ctrl+Alt+S ，您可以将鼠标悬停在目标语句上，然后选择相应的操作：

如果在编辑器中显示了行号，并且启用了单击行号以执行游标运行 构建，执行，部署|的调试器页面 JetBrains骑士设置Ctrl+Alt+S ，您可以通过单击其行号来运行此行。

要在编辑器中 显示行号 ，请选择在以下位置显示行号 编辑器|一般|的外观页面 JetBrains骑士设置Ctrl+Alt+S 。
