在此页面上，您可以配置Rider的NuGet软件包管理器 。

项目

描述

搜索
包括预发行

如果启用此选项，则Rider将搜索并安装软件包的预发行版本。

包括未列出

如果启用此选项，则Rider将搜索并安装服务器上未列出的软件包的版本（例如，在NuGet网站上）。您可能要使用此选项来查看软件包的过时版本。

恢复
允许骑手还原丢失的软件包

此选项定义全局NuGet软件包还原策略。如果您的Internet连接较差，或者您使用的是具有复杂依赖性的大型解决方案（使用默认设置还原软件包实际上会破坏编译），则可能要禁用NuGet软件包还原。

必要时自动还原丢失的软件包

此选项会影响不同的方案，例如，在打开或构建项目时，Rider会自动还原NuGet软件包。顺便说一句，可以在“ 构建”，“执行”，“部署” |“部署” |“设置”上切换“ NuGet自动还原” 。工具集和构建设置页面。
请注意，如果禁用了允许骑手还原丢失的软件包选项，则将禁用自动NuGet软件包还原。

.NET Core还原引擎

默认情况下，JetBrains Rider使用其自己的.NET Core还原引擎实现。您可以改为启用MSBuild引擎，如果在项目中使用Paket依赖管理器 ，则可能需要启用。更重要的是，如果JetBrains Rider在您的解决方案中检测到Paket配置，则此选项将自动设置为MSBuild 。
请注意，MSBuild支持是试验性的，凭据提供程序将无法使用此选项，即，如果您使用私有的NuGet提要，并且在NuGet窗口的“ 源”选项卡上配置了凭据，则MSBuild引擎将无法使用这些凭据。

包装管理
默认包管理格式

使用此选择器选择将使用的NuGet软件包管理格式：

  • packages.config —一种旧格式，使用packages.config文件存储NuGet软件包信息。

  • PackageReference - Visual Studio 2017引入的较新格式，用于将NuGet包信息存储在项目文件中。

默认情况下，JetBrains Rider尝试自动检测应根据项目类型或解决方案中的其他项目使用哪种格式。

凭证提供者
使用凭证提供者

该选择器使您可以为私有NuGet提要选择身份验证方法。

默认的Rider集成提供程序以用户名/密码组合的形式以及MyGet和Visual Studio Team Services（VSTS）的形式支持凭据。

对于使用基于插件的身份验证方法的私人供稿，请选择NuGet / .NET CLI插件或NuGet / .NET CLI插件，然后选择Rider Integrated 。
