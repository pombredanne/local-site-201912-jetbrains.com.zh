代码检查：冗余捕获子句

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

考虑以下代码：

尝试{string s = File。ReadAllText（“ test.txt”）; } catch（ex Exception）{抛出； }

的catch语句可能看起来在做某事，但实际上并非如此：它所做的只是抛出异常（具有相同的堆栈信息），这就是如果根本不编写catch语句会发生的情况。请注意，如果我们写throw ex而不只是throw ，我们再也不能说catch子句是多余的。
