JetBrains Rider 2019.2帮助

运行和调试Node.js

JetBrains Rider可帮助您运行和调试Node.js应用程序。您可以调试从JetBrains Rider启动的应用程序，也可以将其附加到已经运行的应用程序。

在你开始前

如管理插件中所述，在“ 插件”页面上安装NodeJS插件。

运行Node.js应用程序

JetBrains的骑手根据类型的Node.js的运行配置运行的Node.js应用程序。JetBrains Rider还使用此配置在本地调试Node.js应用程序 。

创建Node.js运行/调试配置

  1. 从主菜单中，选择运行|编辑配置 ，然后在“ 编辑配置”对话框中， 点击在工具栏上，然后从列表中选择Node.js。将打开“ 运行/调试配置：Node.js”对话框。

  2. 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

  3. 在“ JavaScript文件”字段中，指定启动它的应用程序主文件的路径（例如，对于Node.js Express应用程序 ，为bin / www ）。

  4. 可选地：

    • 指定自定义Node.js开头的节点参数 。例如，要使您的应用程序可用于调试 ，请根据您的Node.js版本键入调试标志之一： --inspect=适用于6.5和更高版本的Node.js或--debug=适用于早于8的任何Node.js版本。默认调试器端口是5858 。

    • 在“ 应用程序参数”字段中，指定通过process.argv数组开始时要传递给应用程序的特定于Node.js的参数。

运行应用程序

  • 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的Node.js配置，然后单击在它的旁边。应用程序启动，“ 运行”窗口打开，显示应用程序输出。

如果您在应用程序中使用了诸如morgan之类的日志记录工具，并且该工具将日志写入文件，则可以在“ 运行”工具窗口的“ 控制台”选项卡中看到这些日志。

在运行Node.js应用程序时管理日志

  1. 如上所述创建Node.js运行/调试配置，然后转到“ 日志”选项卡。

  2. 请点击 要在控制台字段中显示的日志文件旁边的字段，其中列出了可用的日志文件（如果有）。

  3. 在打开的“ 编辑日志文件别名”对话框中，键入要在日志条目列表中显示的别名，并指定日志文件的位置。选择是要显示此模式覆盖的所有文件还是仅显示最后一个文件。

  4. 单击“ 确定”返回到“ Node.js运行/调试配置”对话框，在该对话框中将新日志文件添加到列表中。选中旁边的“处于活动状态”复选框。要跳过先前的内容，请选中“ 跳过内容”复选框。

  5. 可选地：

    • 要启用将控制台输出保存到日志文件，请选中将控制台输出保存到文件复选框，并指定文件位置。

    • 选择何时显示控制台 。

调试Node.js应用程序

JetBrains Rider使调试Node.js应用程序更加容易。您可以在JavaScript或TypeScript代码中放置断点，因此不再需要任何断点。 debugger和console.log()陈述。您可以做很多事情来帮助您探索代码并了解错误的位置。在“ 调试工具”窗口中，您可以查看调用堆栈和当前状态的变量，在编辑器中评估表达式，以及单步执行代码。

JetBrains Rider支持两种调试模式：

  • 本地调试 ：在此模式下，您的应用程序是从JetBrains Rider启动的，并且正在您的计算机上运行。要对其进行调试，请使用Node.js配置。

  • 调试正在运行的应用程序 ：在这种模式下，您的应用程序已经在调试模式下运行，并且JetBrains Rider附加到正在运行的进程。这种方法使您可以重新启动调试会话，而无需重新启动Node.js服务器。

    JetBrains骑士认识到--inspect现在已弃用--debug标志，以便您可以访问任何应用程序进行调试。

    要调试正在运行的应用程序，请使用“ 附加到Node.js / Chrome”配置。

在本地调试Node.js应用程序

  1. 必要时在Node.js代码中设置断点 。

  2. 如上所述创建Node.js运行/调试配置。如有必要，JetBrains Rider可以生成JavaScript调试配置，并自动将其与Node.js配置一起启动，如调试服务器和客户端代码中所述 。

  3. 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的Node.js配置，然后单击在它的旁边。将打开“ 调试”窗口 。

  4. 执行将触发带有断点的代码执行的步骤。

  5. 切换到JetBrains Rider，此时已启用“ 调试工具”窗口的控件。继续进行调试会话— 逐步执行断点 ，在帧之间切换，即时更改值， 检查已暂停的程序 ， 评估表达式并设置监视 。

调试正在运行的Node.js应用程序

使用JetBrains Rider，您可以使用Chrome调试协议或V8调试协议 （也称为旧协议 ）来调试已经运行的应用程序。

无论哪种情况，都将通过“ 附加到Node.js / Chrome”配置启动调试会话。

使用Chrome调试协议进行调试

使用此协议调试以--inspect旗。此标志与6.3以后的Node.js版本一起使用。

  1. 根据需要设置断点。

  2. 选择运行|从主菜单中编辑配置 ，然后单击在打开的“ 编辑配置”对话框中，然后从列表中选择“ 附加到Node.js / Chrome ”。将打开“ 运行/调试配置：附加到Node.js / Chrome”对话框。

  3. 指定运行目标应用程序的主机以及传递到的端口--inspect启动要连接的Node.js进程时。复制信息消息中的端口号Debugger listening :在终端工具窗口或控制正在运行的应用程序的运行工具窗口中。默认端口为9229。

    （可选）：在“ 本地文件的远程URL”区域中指定项目文件夹的远程路径。如果正在运行的应用程序的根文件夹与您的JetBrains Rider项目根文件夹的名称不同，这将很有帮助。

  4. 在“ 附加到”区域中，选择以--inspect开头的Chrome或Node.js> 6.3 。

  5. 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的“ 附加到Node.js / Chrome”配置，然后单击在它的旁边。将打开“ 调试”窗口 。

  6. 执行将在断点处触发代码的操作。对调试会话的控制权返回给JetBrains Rider。

  7. 切换到JetBrains Rider。在“ 调试工具”窗口中， 逐步浏览断点 ，在帧之间切换，即时更改值， 检查已暂停的程序 ， 评估表达式并设置监视 。

使用V8调试协议进行调试

使用此协议调试以--debug旗。此标志与早于8的Node.js版本一起使用。

  1. 如上所述创建一个Attach to Node.js / Chrome运行/调试配置，并指定传递给该主机和端口--debug 。默认端口为9229。

  2. 确保使用以下参数启动了要调试的应用程序： --debug= 。默认端口是5858 。

  3. 在使用Chrome调试协议的调试会话中进行操作 。

调试使用nodemon的Node.js应用程序

JetBrains Rider内置调试器可以自动重新连接到正在运行的Node.js进程。 这使您可以调试使用nodemon实用程序的 Node.js应用程序，该实用程序会在代码更新时自动重新加载Node.js进程。

要调试此类应用程序，您需要在调试模式下启动它（使用--inspect标记），然后使用启用自动重新连接选项的附加到Node.js / Chrome调试配置进行连接 。

安装nodemon

  • 在嵌入式终端 （ Ctrl+Alt+1 ），输入npm install --save-dev nodemon要么yarn add nodemon --dev安装nodemon作为开发依赖项。

在调试模式下使用nodemon启动应用程序

  • 创建并运行以下内容npm debug脚本：

    debug”：“ nodemon --inspect

    有关详细信息，请参见运行和调试脚本 。

  • 或者，通过inspect通过如上所述的Node.js运行/调试配置进行标记。

调试应用程序

  1. 根据需要在代码中设置断点。

  2. 如调试正在运行的Node.js应用程序中所述，创建新的“ 附加到Node.js / Chrome”配置，然后选中“ 自动重新连接”复选框。

    通常，您无需在配置中更改端口9229因为它是调试器正在侦听的默认端口。但是，您可以在调试模式下运行应用程序时再次检查记录的消息中使用了哪个端口。

  3. 从工具栏上的“ 选择运行/调试配置”列表中选择新创建的“ 附加到Node.js / Chrome”配置，然后单击在它的旁边。调试器将在您在JetBrains Rider中的代码中放置的断点处停止。

    现在，每次您对代码进行任何更改并保存它们Ctrl+Shift+S ，nodemon将自动重新加载应用程序，调试器将自动重新附加到重新启动的进程。
