描述

默认严重性

等于运算符可能会导致类型强制

报告相等运算符的使用情况，这可能会导致意外的类型强制。使用类型安全的等于运算符===和！被认为是一种好习惯。==代替常规的==和！= 。

根据所选的选项，这将突出显示：

  • ==和！的所有用法=运算符。

  • 除与null进行比较外，所有用法。一些代码样式允许使用x == null代替x === null ||。 x ===未定义 。

  • 仅可疑的表达式，例如： ==或！=与0 ， '' ， null ， true ， false或undefined的比较 。

残障人士

类型不匹配

检查被称为函数参数，返回值和分配的表达式的TypeScript是否具有正确的类型。

 错误
“任何”类型上下文中的类型不匹配

在任何类型上下文中检查称为函数参数的TypeScript，返回值，分配的表达式是否为正确的类型。

 弱警告
无法解析的JavaScript变量

检查TypeScript引用的变量和字段是否有效。

 弱警告
验证进口

验证TypeScript import 。

 错误
无法解析的JavaScript函数

检查被TypeScript调用的函数是否有效。

 弱警告
@private和@protected成员可访问性

检查TypeScript 私有成员和受保护成员在当前上下文中是否可见。

 弱警告
进口可以缩短

报告ES6进口，其从部分可以通过导入父目录缩短。

 警告
引用UMD全局

如果当前文件是模块，则报告对UMD全局引用的使用。

 弱警告
字段仅在构造函数中分配，并且可以设为只读

如果仅在构造函数中分配了私有字段，则可以将其设为只读。

 弱警告
缺少增强导入

检查当前文件是否导入了扩充模块。

 信息
显式成员类型

突出显示与当前代码样式不匹配的类型声明。

默认情况下，当可以从上下文中推断类型时，类型声明将突出显示为冗余，例如：

突出显示与当前代码样式不匹配的类型声明。

var pi: number = 3.14

突出显示与当前代码样式不匹配的类型声明。

启用下面的任何选项都会使此设置反转，并建议即使在可以推断类型的地方插入显式类型声明。

 信息
变量类型由类型防护器缩小

突出显示变量用法，其中变量类型由类型防护缩小。请注意，严重性级别不会影响此检查。

tsconfig.json中不包含必需的TypeScript库

检查符号所需的TypeScript库文件是否在tsconfig.json中的“ lib”编译器选项下列出。

 错误
构造函数中的可疑参数分配

当将类字段声明为构造函数参数，然后分配此参数时，警告TypeScript代码中的常见错误。
在这种情况下， 将不分配相应的字段，仅修改本地参数值。

 警告
可以使抽象类构造函数受到保护
