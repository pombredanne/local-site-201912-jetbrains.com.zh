自动完成

JetBrains Rider的自动完成功能会建议类名称，方法，变量，关键字以及当前可见性范围内可用的任何其他符号的名称，包括先前导入的扩展方法。完成列表中还提供了许多其他特定于上下文的建议，例如实时模板 ， 代码生成建议 ， 单元测试类别 。
默认情况下，自动完成建议还包括“ 类名完成”中的项目。如果要更改默认行为，请在 编辑器|一般|的代码完成页 JetBrains骑士设置Ctrl+Alt+S 。建议列表与“ 基本完成 ”列表相同。

开始输入后，将显示自动完成的建议列表。键入时，列表会缩小以仅显示与您输入内容匹配的项目。

默认情况下，自动完成支持CamelHumps ，即您可以键入化合物名称部分的首字母，该名称将出现在建议列表中。

在构建完成建议列表时，JetBrains Rider还会尝试找出您可能会选择的建议，并在列表中预先选择它。使用默认设置，JetBrains Rider会预先选择最佳匹配项，让您按一下即可插入Space以及点，右括号，右括号，分号等（取决于上下文），您可以按照本主题下面的说明更改此默认行为。

配置自动完成

如果显示在您输入时所检查的自动完成建议只出现 编辑器|一般|的代码完成页 JetBrains骑士设置Ctrl+Alt+S 。

要在自动完成列表中启用“硬”选择，请按Enter，点或其他与上下文相关的键选项，选择“ 插入所选建议” 。

如果选择此选项，JetBrains Rider将在自动完成列表中预先选择最佳匹配项，以使其易于插入，此外Enter和Tab ，当您按下时也会插入Space ，以及点，左括号，打开支架，分号，等等（取决于上下文）（在此模式下空间完成可以通过清除所选择的空白插入物的建议...复选框此页面上被禁用）：

如果清除此复选框，JetBrains Rider将通过软选择突出显示最佳匹配项，您可以通过按来插入它Enter要么Tab 。如有必要，还可以通过清除此页面上的“ 输入插入建议...”复选框来另外禁用“ 输入”的完成以进行软选择。
