价值和无效性分析模式

JetBrains Rider可以分析不同模式下的值：

  • 乐观的
    默认情况下，值分析模式是乐观的。在此模式下，JetBrains Rider仅在明确检查该值是否可为空或符号标记有否的情况下警告可能的“空”取消引用。 [CanBeNull]要么[ItemCanBeNull]属性。上面的示例演示了乐观模式。

  • 悲观
    在这种模式下，JetBrains Rider警告所有在未明确检查该值是否可为空的情况下可能会“空”取消引用的情况，除非用符号标记该符号。 [NotNull]要么[ItemNotNull]属性。

要更改值分析模式或禁用分析，请在以下位置配置“ 值分析模式”首选项 编辑器|的“检查设置”页面 JetBrains骑士设置Ctrl+Alt+S 。
