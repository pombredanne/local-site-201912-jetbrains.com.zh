代码检查：未使用分配

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

通过分析代码的控制流，JetBrains Rider能够检测到字段和局部变量的冗余初始化程序。如果在任何执行路径中进行下一个赋值之前未使用您分配的值，JetBrains Rider建议删除冗余初始化程序。

将新值分配给以前从未使用过的参数时，此检查也适用。在这种情况下，JetBrains Rider不建议任何修复，因为尚不清楚该参数是否应替换为局部变量或分配是否错误：

void Bar（string param）{param =“新内容”;安慰。WriteLine（param）; }

冗余分配可以在不同情况下发生。例如，当使用默认值初始化变量时，或者如下图所示，当分配了一些非默认值但从未使用过时。在这里你可以看到myDoc用的新实例初始化XDocument ，但下一行将其分配给a要么b无论如何，因此最初创建的对象不过是垃圾收集器的一项新任务。
