JetBrains Rider 2019.2帮助

打字协助

除了通过在完成列表中提供智能建议来节省时间外，JetBrains Rider还提供其他键入帮助功能，使您可以更快地创建和编辑代码。

自动插入匹配定界符

默认情况下，无论何时键入开头定界符( ， [ ， " ， 要么' ，则会自动插入一个配对的字符（如果适用）。如果您改变主意并按Backspace要删除开始定界符，其结束对也将被删除。如果配对结束定界符已经存在，JetBrains Rider将不会添加另一个定界符。将根据您的格式设置首选项插入匹配的符号。

JetBrains Rider还提供匹配定界符的可配置突出显示 。

关于牙套{} ，JetBrains Rider可以在键入开头或按下时立即提供右括号Enter 。

如有必要，您可以更改默认行为。为此，请转到 编辑器|一般|的打字协助页面 JetBrains骑士设置Ctrl+Alt+S并使用花括号和括号部分中的控件。

校正长度/计数错误

JetBrains Rider防止您因打字错误而绊倒Length / Count数组/集合的属性。一旦您错误地开始输入Count属性以使用数组，JetBrains Rider将允许您从完成列表中选择它并替换为Length属性，您可能要键入。

以类似的方式，它将帮助您致电Count当您开始输入Length财产错误。

您接受填写建议后，就会有正确的属性：

Foo（int [] array，List collection）{if（array。长度==集合。数}

生成相等性并标记枚举类型的检查

当您需要将枚举类型的值与此枚举成员之一进行比较时，只需键入一个点，然后在完成列表中选择所需的枚举成员即可：

JetBrains Rider将为您生成比较：

公共枚举方向{北，东，南，西} void Turn（Direction whereTo）{if（whereTo ==方向。南 }

根据方法参数中的用法创建类型参数

创建泛型方法时，可以通过键入轻松添加类型参数T输入新参数，然后在完成列表中选择相应的项目

JetBrains Rider将在方法声明中添加新的类型参数，并使您到达可以在其声明和用法中键入类型参数名称的位置：

输入名称后，按Enter要么Tab继续打字。

点和分号的智能行为

默认情况下，当您在完成列表中选择一种方法时，JetBrains Rider会自动添加括号并将您的插入号设置在开始输入参数的位置。如果要调用不带参数的方法，并链式调用另一个方法，则可以在插入符号的正确位置键入点，JetBrains Rider会将其移至正确的位置：
myStringBuilder.AppendLine(./*caret*/)变成myStringBuilder.AppendLine()./*caret*/
如果您只是键入一个完整的方法调用，然后决定对另一个方法进行链式调用，则不必移动插入符号，只需在最后一个分号后键入点即可：
myStringBuilder.AppendLine();./*caret*/变成myStringBuilder.AppendLine()./*caret*/;

如果在指定不带前导零的双精度值时使您烦恼，例如DoSomething(.33); ，请清除“ 更正特定于通用语言的拼写错误”复选框 编辑器|一般|的打字协助页面 JetBrains骑士设置Ctrl+Alt+S 。

当插入符号在括号内时，不必将其移到外面以键入最后的分号，只需立即或在最后一个参数之后键入它即可：
myStringBuilder.AppendLine("line";/*caret*/)变成myStringBuilder.AppendLine("line");/*caret*/

更正逐字字符串插值中的前缀

从C＃6.0开始，就可以使相同的字符串既插和逐字只要你加入$和@前缀的正确顺序： $@"some string" 。

使用JetBrains Rider，订单始终是正确的-输入错误@$将自动替换为正确的$@ 。

添加NotNull / CanBeNull批注

如果在项目中启用了代码注释支持，则可以添加[NotNull]和[CanBeNull]注释，以及通过一次按键即可生成空检查：

  • 编写方法签名或成员声明时，键入!要么?直接在类型名称之后获取带有注释的项目[NotNull]要么[CanBeNull] ：

  • 如果符号已经用注释[NotNull]要么[CanBeNull] ，您可以输入?要么!相应地更改现有注释。

  • 如果参数标记有[NotNull]属性，则可以在参数名称或参数类型之后直接设置插入符号，然后按! ：

    私有void Foo（[NotNull] object / *！* / arg / *！* /）
    JetBrains Rider将对此参数生成一个空检查 。您可以在 代码编辑| C＃|的Null Checking页面 JetBrains骑士设置Ctrl+Alt+S 。

默认情况下，键入[NotNull]要么[CanBeNull]已启用。如果要禁用它，请清除“ 注释可为空...”复选框 环境|编辑器的“编辑者行为”页面 JetBrains骑士设置Ctrl+Alt+S 。

在Enter上多行注释

阻止评论

如果编写多行块注释/*....*/ ，您可以输入开头部分/* ，然后写第一行，然后按Enter在行尾— JetBrains Rider将插入结束部分*/ 。

另外， JetBrains Rider将插入星号*按下时，在块注释的每一行的开头Enter 。默认情况下，此行为是启用的，但是您可以通过清除“ 在/ *之后输入”复选框上的“ 插入*”来关闭它 编辑器|一般|的打字协助页面 JetBrains骑士设置Ctrl+Alt+S 。

行注释

在简单的（非XML Doc）行注释中，您还可以按Enter在注释文本的中间，为避免在新行中出现不可编译的文本，JetBrains Rider将自动添加两个斜杠//在文字前面。如果按Enter在注释行的末尾，您将像往常一样开始一个空的新行。

用括号和括号包装代码块

有了JetBrains Rider， 您可以选择一个代码块或一个表达式，然后键入一个开头{或结束}大括号将所选内容放在大括号内，或键入一个开口(或结束)括号将选择内容放在括号内。要启用或禁用此行为，请在键入括号或括号复选框时使用“ 环绕”选项 。 编辑器|一般|的打字协助页面 JetBrains骑士设置Ctrl+Alt+S 。

使用Backspace缩进代码行

当您按Backspace在空白行或插入符号左侧只有空格和制表符的情况下，JetBrains Rider可以将插入符号返回到适当的缩进位置，而不必一次将其移回一个位置。

可使用“ 退格时不显示”选择器来配置此行为 编辑器|一般|的打字协助页面 JetBrains骑士设置Ctrl+Alt+S 。提供以下选项：

  • 禁用 —一次将插入符返回一个位置。

  • 到最接近的缩进位置 —将插入号返回到当前缩进大小的倍数的最接近位置。

  • 到适当的缩进位置 （默认）—根据代码结构将插入号返回到预期的缩进位置。
