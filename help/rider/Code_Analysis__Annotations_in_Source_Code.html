源代码中的注释

受益于JetBrains Rider的代码注释的最简单方法是将注释属性添加到源代码的符号中，并使JetBrains Rider以更高的准确性和洞察力分析您的解决方案。

默认情况下，所有注释属性类都标记有[Conditional("JETBRAINS_ANNOTATIONS")]属性，因此编译器将忽略代码中的属性用法，这意味着没有对'JetBrains的二进制引用。Annotations.dll的程序集将被生成。但是，您可以在项目中定义“ JETBRAINS_ANNOTATIONS”条件编译符号，以保留元数据中的属性。

在项目中启用代码注释支持

要在项目中使用JetBrains Rider代码注释属性，您应该通过以下方式之一引用它们：

  • 推荐的方法是使用以下命令安装NuGet软件包： JetBrains.Annotations部件。
    实际上，您甚至不需要访问NuGet网站即可获得该软件包。只需添加using JetBrains.Annotations;指令，并使用相应的Alt+Enter操作以自动获取软件包。

  • 您可以将项目引用添加到JetBrains。Annotations.dll ，您可以在JetBrains Rider安装目录下的lib \ ReSharperHost下找到该文件 。

在源代码中嵌入代码注释的声明

您可以在上获得代码注释声明的副本。 编辑器|检查设置|的代码注释页面 JetBrains骑士设置Ctrl+Alt+S 。（可选）在复制时，您可以选择将声明设为内部声明和/或添加[Conditional("JETBRAINS_ANNOTATIONS")]声明的属性。

注释属性的默认实现在JetBrains.Annotations名称空间，但注释不能使用此名称空间。您可以将声明放在解决方案中的任何位置-JetBrains Rider会自动检测到它们。但是，如果声明位于不同的名称空间中，则应使用注释属性显式指定名称空间，如下所述。
在某些情况下，您的解决方案可能包含注释属性的几种实现，例如，您的类与JetBrains Rider注释类的名称相同（ CanBeNullAttribute ， NotNullAttribute ，等等），或者您正在使用包含JetBrains Rider注释类的第三方程序集。在这种情况下，您可以选择命名空间，JetBrains Rider应该在该命名空间中寻找适当的批注-属性类集。

  1. 按Ctrl+Alt+S ，或者从主菜单中选择“ 文件” |“在Windows和Linux或JetBrains公司骑士 设置 |在macOS上选择“首选项 ”，然后选择“ 编辑器” |“检查设置|左侧的代码注释 。

  3. 如果列表中有多个名称空间，请在“ 默认注释名称空间”列表中选择JetBrains Rider代码分析引擎应使用的名称空间 。
