检查变量

程序暂停后，您可以在当前选定框架的上下文中检查变量和参数：查看其值并即时对其进行编辑。要更改上下文，请在“ 框架”窗格中选择另一个框架。

您还可以跟踪特定的对象，而不是变量的值和表达式的结果 。

查看变量值

有几种查看变量值的方法。

变量窗格

检查变量值的默认方法是使用“ 调试”窗口的“ 变量”窗格。请注意，如果变量属于复杂类型，则可以在窗格中展开相应的节点以查看该变量包含的值。

有时，您可能会注意到某些变量会更改其颜色。自上次评估以来变量值发生更改时，就会发生这种情况。

您可以通过按以下按钮快速从“ 变量”窗格上的变量导航到源代码中的声明： F4或从上下文菜单中选择“ 跳转到源代码 ”。

内联代码

查看变量值的另一种方法是查看编辑器中的代码。默认情况下，JetBrains Rider在声明这些变量的行的末尾显示变量的值。

请注意，更改后的值也会以不同的颜色显示。

如果出于某种原因要禁用代码内联显示值，请在以下位置禁用“内联显示值 ” 构建，执行，部署|调试器的“数据视图”页面 JetBrains骑士设置Ctrl+Alt+S 。

代码工具提示

当启用显示值工具提示时 构建，执行，部署|调试器 JetBrains Rider设置的“ 数据视图”页面Ctrl+Alt+S ，您可以将鼠标悬停在变量上以查看其值。 如果变量或表达式的值包含子元素，请单击展开节点并显示所有子级。

您还可以使用编辑器tootips来评估表达式 。

函数返回值

“ 变量”窗格将显示在当前上下文中调用的任何方法的返回值。当方法调用的结果未存储在局部变量中时，例如在使用方法链接时，这很有用。

编辑变量值（编辑并继续）

您也可以编辑代码并继续调试而无需重新启动。该项目将在后台重新编译。当在启用 （ Enable Edit And Continue ）上打勾时，这适用于Windows上针对.NET Framework和.NET Core（WPF除外）的项目 构建，执行，部署|的调试器页面 JetBrains骑士设置Ctrl+Alt+S 。

在某些情况下，无法修改代码，并且需要重新启动。其中包括更改签名，修改通用功能，任何类型的重命名等等。如果发生这种情况，JetBrains Rider将通过显示工具提示来通知您。

当前仅在Windows上支持“编辑并继续”，因为其他运行时未实现所需的调试器API

因此，您可以在编辑器中更改代码，也可以在“调试”窗口中编辑变量值。例如，当您要为调试功能重现特定上下文时，这可能很方便。

更改变量值可能会导致结果不一致并中断程序执行。

编辑变量值

  1. 在“ 调试”窗口的“ 变量”窗格中，选择所需的变量。

  2. 请执行以下任一操作：

    • 按F2 。

    • 右键单击变量，然后从上下文菜单中选择“ 设置值... ”。

  3. 指定一个新值，然后按Enter 。

自定义复杂对象的调试器表示

如果一个类型都没有意义ToString()覆盖也不用[DebuggerDisplay]属性，那么在调试器视图中这种类型的对象的表示可能完全没有信息。在Rider中，您可以通过固定选定的成员来自定义此类对象的表示形式。

在调试器视图中自定义对象表示

  1. 在“ 调试”窗口的“ 变量”窗格中，选择所需的变量。

  2. 展开变量，找到要在对象表示中使用的成员，然后单击其图标或右键单击并选择固定到顶部 。

  3. 固定的成员将在列表中上移，其图标将被标记图标替换。 。固定成员的值将用于在调试器视图中形成对象的字符串表示形式。因此，请确保选择具有字符串兼容值的成员。

  4. 解决方案中的所有固定项目均已保存，并且在重新启动后也将可用。您可以在相应节点下的“ 收藏夹”窗口中查看所有这些文件。

  5. 要停止在对象演示文稿中使用成员，请单击其标志图标在“ 变量”窗格中或右键单击并选择“ 取消固定” ，或者在“收藏夹”窗口中找到固定的项目，然后按Delete 。

固定的项目将覆盖对象的呈现，即使它具有ToString()或以[DebuggerDisplay]属性。

在调试器视图中区分对象的另一种方法是按ID跟踪对象 。

比较变量值

在调试期间，您可以将变量值与剪贴板中的值进行比较。这在调试字符串时特别有用。

将变量值与剪贴板进行比较

  1. 在“ 调试”窗口的“ 变量”窗格中，选择所需的变量。

  2. 右键单击该变量，然后从上下文菜单中选择“ 与剪贴板比较值” 。

比较结果将显示在“ 差异”查看器中 。
