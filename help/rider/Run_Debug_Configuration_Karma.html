运行/调试配置：业力

请不要将运行/调试配置与构建配置混淆。

运行编辑配置| |业力

在此对话框中，使用Karma测试运行程序创建用于运行和调试JavaScript单元测试的配置。

在你开始前

  1. 下载并安装Node.js。

  2. 安装噶测试运行中所描述噶 。

业力特定的配置设置

项目

描述

配置文件

在此字段中，指定Karma配置文件的位置。通常，文件具有扩展名.conf.js

业力选项

在这个领域里， 指定要传递给Karma的命令行选项，以覆盖karma.conf.js配置文件中的默认设置。

例如，要在Headless Chrome中运行或调试测试，请输入--browsers ChromeHeadless在业力选项字段中。有关详细信息，请参见使用无头Chrome浏览器进行自动测试 。

要查看所有可用的CLI选项，请键入karma start --help在航站楼 Ctrl+Alt+1 。

节点解释器

在这个领域里， 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

节点选项

在此字段中，键入要传递到Node.js可执行文件的特定于Node.js的命令行选项。

  • 使用--require coffee-script/register在运行期间将CoffeeScript文件即时编译为JavaScript。

    这种模式要求3软件包的一部分register.js文件应该位于项目内部。因此，您需要在本地的Node.js页面上安装coffee-script软件包，如npm和Yarn中所述 。

  • 使用--inspect您将Node.js v7用于Chrome调试协议支持时的参数。否则，默认情况下，调试过程将使用V8调试协议 。

业力包

在此字段中，指定Karma安装目录/ npm / node_modules / karma 。如果您通过Node Package Manager定期安装Karma ，则JetBrains Rider会检测Karma的安装目录。或者，手动键入可执行文件的路径，或单击“ 浏览”。 并在打开的对话框中选择位置。

工作目录

在此字段中，指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

环境变量

在此字段中，可以选择指定用于执行命令的环境变量。

测验

在该区域中，指定要执行的测试。可用的选项有：

  • 所有测试：选择此选项可运行karma.conf.js配置文件中的所有测试。

  • 测试文件：选择此选项可仅运行一个文件中的测试 ，然后在“ 测试文件”字段中指定此文件的路径。

  • 套件：选择此选项可从测试文件运行单个套件，并在“ 测试文件”字段中指定此文件的路径。单击套件名称字段，然后配置要运行的套件列表。要将套件添加到列表中，请单击并输入所需套件的名称。要删除套件，请在列表中选择它，然后单击

  • 测试：选择此选项可运行测试文件中的各个测试，并在“ 测试文件”字段中指定此文件的路径。单击测试名称字段，然后配置要运行的测试列表。要将测试添加到列表中，请单击并输入所需测试的名称。要删除测试，请在列表中选择它，然后单击

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Ctrl+Shift+Alt+R 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

工具列

运行/调试配置的树形视图有一个工具栏，可帮助您管理解决方案中可用的配置以及调整默认配置模板。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Ctrl+N

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在JetBrains Rider中定义它，请添加其定义。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 构建解决方案 ：选择此选项可在启动配置之前构建解决方案。

  • 运行文件监视程序 ：选择此选项可使JetBrains Rider应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

Ctrl+R, D

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
