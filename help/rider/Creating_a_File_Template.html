创建文件模板

本页介绍.NET语言的模板。Web特定模板中描述了可在HTML，JavaScript，TypeScript等中使用的Web特定模板 。

文件模板用于使用预定义的代码（例如类型声明，单元测试等）创建一个或多个新文件。文件模板没有快捷方式，由它们的描述标识。有关详细信息，请参见从模板创建文件 。

文件模板的主体由纯文本和parameters组成，纯文本将原样显示。想法是将这些结合起来以在调用模板时最大化效率。

创建一个新的文件模板

  1. 根据所使用的语言，将创建的模板应用到的位置，打开 编辑器|文件模板|的[语言]页面 JetBrains骑士设置Ctrl+Alt+S 。

  2. 点击新模板 。

  3. 在“ 描述”字段中指定模板名称。

    JetBrains Rider不会将每个代码模板保存在单独的文件中。因此， 描述标识符不必是唯一的。但是，请尝试提供可理解的标识符，以免在应用自定义代码模板时造成混淆。

  4. （可选）在“ 默认文件名”中，指定JetBrains Rider应用模板时建议的名称。您还可以勾选“ 文件名必须是有效的标识符”复选框 JetBrains Rider检查在应用模板时指定的文件名是否作为类型标识符有效。如果您的模板为新类型创建文件，并且使用提供的文件名使用当前文件名（不带扩展名 宏 ）初始化类型名，则此选项很有用。您可以检查预定义的文件模板以了解其实现方式。

  5. 要更改模板的默认范围 ，请单击“ 可用性”按钮。在出现的对话框中，使用复选框定义可以在何处应用新模板。您可以扩展某些范围以指定详细信息（例如，文件掩码或语言版本）：

  6. 通过键入或粘贴所需的代码来创建模板的主体。
    模板代码可以包含纯文本和变量 。 键入变量后（前缀和后缀为$ ），变得可用在里面。按照“ 声明模板变量”部分中的说明配置所有定义的变量。

  7. 配置模板格式选项：

    • 选择重新格式化复选框以使 JetBrains Rider会根据您的格式首选项设置自动重新格式化扩展的代码片段。

    • 选择“ 缩短合格的引用”复选框以进行应用模板后，JetBrains Rider会自动插入名称空间导入指令或删除多余的限定符。如果未选中此复选框，JetBrains Rider将不会插入任何名称空间导入指令。

  8. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。

调用命令以应用文件模板时 ，将显示可用模板列表。我们称其为“快速访问列表”或“快速列表”。您可以编辑此列表以在其中添加您喜欢的模板和/或重新排列其顺序。

编辑快速访问列表

  1. 根据所使用的语言，将创建的模板应用到的位置，打开 编辑器|文件模板|的[语言]页面 JetBrains骑士设置Ctrl+Alt+S 。

  2. 请点击在模板列表的右侧。

  3. 在打开的对话框中，使用箭头控件来配置快速列表中的模板及其顺序。

  4. 请点击 单击保存以应用修改，然后让JetBrains Rider选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参见基于图层的设置 。
