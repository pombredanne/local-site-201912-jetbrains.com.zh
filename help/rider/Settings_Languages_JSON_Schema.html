在此页面上，除了JetBrains Rider提供的系统架构之外，还创建自定义JSON架构的列表。对于每个模式，请配置其范围，即，指定要针对其验证的.json文件。该页面包含两个窗格：“ 模式列表”窗格和“ 模式详细信息”窗格。

模式窗格列表

该页面的中央窗格显示了一个自定义JSON模式的列表，将根据该列表验证当前项目中的.json文件。基于模式，JetBrains Rider检查.json配置文件的结构，报告错误（例如，通知您缺少必填属性），提供代码完成和文档查找。

该窗格每次只显示自定义架构，你以前下载或自己创建，在这两种情况下，自定义模式必须符合JSON模式的标准 ，并且必须在项目的根目录下进行定位。

列表中未显示JetBrains Rider为所有受支持的框架和技术提供的系统架构。

  • 要将架构添加到列表中，请单击在工具栏上，然后在“ 架构详细信息”窗格中配置架构。

  • 要从列表中删除模式，请选择它并单击在窗格的工具栏上。

  • 要配置架构的范围，请选择架构，然后在“ 架构详细信息”窗格中指定要针对该架构进行验证的文件。

模式详细信息窗格

该窗格显示在架构窗格中的列表中选择的模式的细节：所选择的模式的名称，本地上传.json文件实现，或此类文件的URL地址，该模式规范的版本与您的架构符合规定，以及根据架构验证的文件和文件夹的列表。

该列表包含特定文件的名称，整个目录的名称和文件名模式。您无需指定文件和文件夹的完整路径，JetBrains Rider会搜索具有指定名称的文件和文件夹，并且搜索仅限于当前项目。基于该列表，JetBrains Rider在内部创建要验证的文件列表。每个文件仅包含在列表中一次：如果具有指定名称的文件存储在列表中的目录中，或者其名称与模式匹配，则该文件仍仅被验证一次。

  • 要将新文件或文件夹添加到列表中，请单击在窗格的工具栏上，然后在打开的对话框中指定文件，文件夹或文件模式。

  • 要从列表中删除项目，请选择它并单击在窗格的工具栏上。

处理模式范围之间的冲突

当文件，文件夹或模式属于两个或多个模式的范围时，就会发生冲突。JetBrains Rider以两种模式分析范围：

  • 静态分析会检测自定义架构范围内的冲突。如果检测到冲突，JetBrains Rider将在“ 模式详细信息”窗格中显示警告。要查看重叠的范围，请单击显示详细信息链接。JetBrains Rider显示带有消息的弹出窗口，其中列出了冲突的范围和架构：

  • 动态分析检测系统和自定义架构的范围中的冲突。当您打开属于某个范围的文件时，便开始这种类型的分析。如果检测到冲突，JetBrains Rider将在编辑器选项卡的顶部显示警告：

    单击链接以打开“ JSON架构映射”页面，然后编辑冲突的自定义架构的范围。请注意，您无法编辑系统架构的范围。
