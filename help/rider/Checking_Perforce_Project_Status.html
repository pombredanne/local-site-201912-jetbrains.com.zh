Checking Perforce Project Status

Besides indicating the current file status relative to the repository, JetBrains Rider integration with Perforce provides you with the accumulated view of the project files' statuses.

In this section:

  • Viewing the statuses of project files
  • Refreshing file status

Viewing the statuses of project files

  1. Open the required project.

  2. On the main menu, choose VCS | Refresh File Status.

  3. Switch to the Version Control tool window, tab Local.

    The status of each file is indicated by the color in which the path to the file is displayed.

Refreshing file status

JetBrains Rider provides two refresh modes for statuses of files under Perforce control.

  • Standard Refresh takes into consideration only the changes made through the JetBrains Rider integration with Perforce. This improves performance because does not require connecting to the server. However, this approach does not let you know about the changes made outside JetBrains Rider, for example, right through the p4v client application.

  • Force Refresh considers all the changes made to project, both from JetBrains Rider and from any other application, for example, right through p4v client.

  1. Switch to the Version Control tool window, tab Local.

  2. Do one of the following:

    • To run Standard Refresh, click the Refresh toolbar button or press Ctrl+Alt+R.

    • To run Force Refresh, click the Force Refresh toolbar button .
