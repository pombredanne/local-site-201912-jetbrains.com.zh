HttpHandler或WebService中的代码检查

在HttpHandler或WebService中，JetBrains Rider 2019.2提供两种代码检查：3种检查，用于检测语法错误，无法解析的符号，编译器错误等错误（您无法配置或禁用这些检查中的任何一种），以及8 专有代码检查 ，您可以禁用或更改其严重性水平 。
这些代码检查可帮助您在设计时检测所有打开文件中的代码问题，此外，它们还使您可以查找特定范围内的代码问题 。

下面列出了可配置的HttpHandler或WebService检查，按其类别分组。

潜在的代码质量问题（两次检查）

此类检查包括检测关键问题（代码异味）的检查，主要是错误或警告级别。此类别还包括确保本地化协助的检查 。

检查

 ID EditorConfig属性 默认严重性
未访问字段（非私有可访问性） NotAccessedField.Global resharper_not_accessed_field_global_highlighting 建议
路径错误 Asxx.PathError resharper_asxx_path_error_highlighting 警告

符号声明中的冗余（1个检查）

此类别包括代码检查，大多数情况下具有警告严重性级别，可检测空的和未使用的符号声明。

检查

 ID EditorConfig属性 默认严重性
从未使用过类型或成员（非私有可访问性） UnusedMember.Global resharper_unused_member_global_highlighting 建议

拼写问题（5次检查）

这些检查可以检测各种情况下的错别字。

检查

 ID EditorConfig属性 默认严重性
错字评论 CommentTypo resharper_comment_typo_highlighting 建议
错别字 IdentifierTypo resharper_identifier_typo_highlighting 建议
标记属性值中的错字 MarkupAttributeTypo resharper_markup_attribute_typo_highlighting 建议
标记文字中的错字 MarkupTextTypo resharper_markup_text_typo_highlighting 建议
字符串文字中的错字 StringLiteralTypo resharper_string_literal_typo_highlighting 建议
