错误条带和状态指示器

错误条带和状态指示器可让您观察当前文件中设计时代码检查的结果，并在代码问题之间导航。此外，错误条带还为其他JetBrains Rider项目显示特殊标记，例如当您突出显示当前文件中的用法时 。

错误条未显示，状态指示器显示了暂停图标如果：

  • 设计时代码检查已禁用 。

  • 编辑器显示反编译的代码 。

错误条带

误差条是集成到编辑器滚动条中的侧边栏。状态栏可以将以下项目表示为单独的标记：

  • JetBrains Rider的设计时检查发现了代码问题。

  • 待办事项
  • 文件中符号的用法
  • 书签

标记具有不同的颜色，表示相应项目的种类。颜色可能会因各种配色方案而异，但是您可以分别为每种项目配置错误条带颜色 。

条形上标记的位置指示文件中相应项目的相对位置。您可以通过将鼠标指针放在标记上来获得每个项目的弹出提示，还可以通过单击它们直接导航到相关的代码行。

如果与标记对应的代码在可见的编辑器区域之外，并且启用了“ 在滚动条上显示代码镜头悬停”，则 编辑器|一般|的外观页面 JetBrains骑士设置Ctrl+Alt+S ，将鼠标悬停在标记上时，JetBrains Rider将在弹出窗口中显示以下代码片段：

在处理文件时，通常会在错误条中看到许多标记。当您完成代码时，许多这些错误，警告和建议最终都会得到解决。如果您觉得自己的代码已完成，但仍然有任何错误，那么在尝试编译项目之前，探索并解决发现的错误可能会节省时间。

配置错误条的外观

  1. 按Ctrl+Alt+S ，或者从主菜单中选择“ 文件” |“在Windows和Linux或JetBrains公司骑士 设置 |在macOS上选择“首选项 ”，然后选择“ 编辑器” |“配色方案|一般在左边。

  2. 在列表中选择一个项目，然后使用“ 错误条带标记”复选框将其显示或隐藏在错误条带上。使用其旁边的颜色选择器选择颜色以表示错误条上的项目。

  3. 如果不确定如何找到错误条上标记的项目，则可以通过以下方式找到其设置：

    1. 单击错误条纹标记，以便您的插入符号可进入编辑器中的相关项目。

    2. 按Ctrl+Shift+A ，开始输入jump to colors and fonts然后在列表中选择此命令。

    4. 选择此属性后，您将直接进入其设置。

状态指示灯

状态指示器 （错误条顶部的小图标）显示文件中代码检查的状态。将鼠标指针悬停在其上时，指示器将显示当前文件中的错误和警告总数。

状态指示器图标的含义

状态指示器可以具有不同的图标，其含义如下：

只要看到红色标记，就意味着该文件包含错误，这些错误会阻止项目编译或在运行时导致异常，除非在项目属性中启用了将警告视为错误 。

黄色标记表示存在一些警告，但没有错误会阻止项目编译。

绿色标记表示您的代码不包含错误或警告 ，但仍然可能存在严重性级别较低的代码问题（ Recommendation或Hint ）。
