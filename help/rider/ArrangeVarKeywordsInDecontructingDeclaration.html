Code Inspection: Join or separate 'var' in deconstruction declarations

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues
  • change its severity level to make the detected issues less or more noticeable

This inspection checks the code style for using 'var' in declarations.
By default, JetBrains Rider will suggest joined notation for multiple var's in deconstruction declarations, for example var (x, y) = GetTuple();. You can select Prefer separate declarations for deconstructed variables on the Editor | Code Style | C# | Code Style page of JetBrains Rider settings to opt for separate notation, for example (var x, var y) = GetTuple();.
