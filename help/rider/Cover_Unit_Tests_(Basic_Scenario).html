分析单元测试的覆盖范围（基本方案）

使用dotCover的最基本方案是评估测试覆盖了多少代码。基线条件如下。您在Visual Studio中打开了一个解决方案。解决方案中的代码包含一些单元测试。任务是了解涵盖了多少代码。

分析解决方案中单元测试的覆盖范围

  1. 选择查看|工具窗口|从主菜单进行单元测试 。这将打开“ 单元测试资源管理器”工具窗口。在这里，您可以查看解决方案中有多少个测试。

  2. 在“ 单元测试资源管理器”窗口中，选择要分析其覆盖率的测试。如果要分析所有测试，请在树中选择顶级节点。

  3. 单击封面单元测试 。这将运行正常的单元测试会话，但是dotCover将在后台收集coverage数据。测试结果将实时显示在新打开的会话选项卡中。覆盖率结果（测试如何覆盖代码中的特定类和方法）将显示在“ 单元测试覆盖率”工具窗口中。

  4. 通过单击突出显示代码可视化代码覆盖率 。

    默认情况下，覆盖范围和测试结果由编辑器左装订线中的标记显示：如果与当前语句相关的任何测试失败，则标记为红色；如果所有测试通过，则标记为绿色；如果没有测试覆盖此，则标记为灰色。声明。
