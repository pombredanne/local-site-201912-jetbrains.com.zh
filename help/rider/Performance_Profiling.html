.NET代码的性能分析

性能分析的目的是查找应用程序中性能问题的原因。例如，这包括一个简单的“最慢的方法是什么？分析，以及与CPU饥饿，阻塞垃圾收集，磁盘I / O和其他问题有关的性能瓶颈的更复杂分析。

JetBrains Rider与dotTrace Profiler集成在一起，以提供.NET应用程序的性能分析。您可以通过Performance Profiler工具窗口使用Profiler 。

局限性

请注意，JetBrains Rider中并非所有dotTrace功能都可用。因此，集成的探查器仅提供了两个用于分析快照的视图：“ 调用树”和“ 顶部方法” 。要获取整个功能集，您应该在dotTrace的独立版本中打开概要分析快照（仅Windows）。

安装和许可

dotTrace作为Rider的捆绑插件提供，并与Rider一起自动安装，无需执行其他操作。如果出于某种原因要禁用dotTrace，则可以通过Rider的插件设置进行 。

请注意，只有ReSharper Ultimate + Rider或“ 所有产品包”订阅的所有者才能在Rider中使用dotTrace。

您可以使用Rider进行什么配置

  • .NET / .NET Core / Mono独立应用程序
  • ASP.NET/ASP.NET Core Web应用程序（仅IIS Express）
  • 任意.NET / .NET Core进程
  • 运行.NET / .NET Core进程
  • NUnit / xUnit / MSTest单元测试
