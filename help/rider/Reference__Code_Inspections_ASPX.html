JetBrains Rider 2019.2帮助

ASP.NET中的代码检查

在ASP.NET中，JetBrains Rider 2019.2提供两种代码检查：6种检查，以检测错误（例如语法损坏，无法解析的符号，编译器错误等）（您无法配置或禁用这些检查中的任何一种），以及35 专有代码检查 ，您可以禁用或更改其严重性水平 。
这些代码检查可帮助您在设计时检测所有打开文件中的代码问题，此外，它们还使您可以查找特定范围内的代码问题 。

下面列出了可配置的ASP.NET检查，按其类别分组。

通用惯例和规范改进（1次检查）

此类别将检查归类为主要影响代码可读性的中等严重性问题。

检查

具有可选值问题的属性 Asp.TagAttributeWithOptionalValue resharper_asp_tag_attribute_with_optional_value_highlighting 警告

代码中的冗余（3次检查）

此类别中的代码检查查找冗余和无效代码，这些代码会影响代码的可读性和样式，并且可以安全地删除。 某些代码冗余无法自动修复，因此需要在用户输入的情况下以交互方式进行快速修复。但是，可以使用范围内的修复或代码清除来修复大多数冗余，而无需用户交互。

检查

 ID EditorConfig属性 默认严重性
冗余使用指令 RedundantUsingDirective resharper_redundant_using_directive_highlighting 警告
未使用的进口条款 UnusedImportClause resharper_unused_import_clause_highlighting 警告
未使用的寄存器指令 AspUnusedRegisterDirectiveHighlighting resharper_asp_unused_register_directive_highlighting_highlighting 警告
