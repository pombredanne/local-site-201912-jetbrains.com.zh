代码样式和清理

JetBrains Rider提供了许多功能，可让您的代码保持整洁。它们与JetBrains Rider的其他大多数功能紧密地交织在一起，它们可以帮助您生成代码并根据特定的代码样式更改现有的代码库。可以将代码样式（包括命名标准，格式规则，文件布局，文件标题样式以及许多其他微小方面（例如修饰符的顺序或是否使用'var'关键字））配置为非常详细的级别并共享在您的团队中。

JetBrains Rider使用不同的方法来处理代码样式的不同方面。例如：

  • 通过使用代码检查 ，JetBrains Rider可以检测并突出显示与命名标准不一致的符号名称，代码中的冗余等等。

  • 每个突出显示的样式冲突都可以使用quick-fix进行修复 ，或者范围内的修复可以应用于所有这种类型的冲突。

  • 为了使代码在编辑器中保持可读性，未突出显示违反代码格式设置规则以及其他一些较小的不一致之处。可以很容易地解决违反格式规则的问题-选择未格式化的代码，按Alt+Enter然后选择格式选择 。

  • 代码清理有助于在批量模式下为所需范围应用格式设置规则和许多其他代码样式首选项。

  • 当您使用JetBrains Rider时，会考虑到大多数代码样式首选项产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。

  • 共享设置机制允许将代码样式配置保留在VCS下，并自动与团队成员共享。

  • 还可以在.editorconfig文件中配置代码格式样式 ， 代码语法样式和代码检查严重性级别 ，这也允许您共享代码样式设置。

您可以参考本节中的特定主题，以了解JetBrains Rider如何正确处理相应的代码样式方面。
