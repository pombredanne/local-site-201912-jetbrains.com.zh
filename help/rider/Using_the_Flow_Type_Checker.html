JetBrains Rider 2019.2帮助

流

Flow是一个静态类型检查器，将类型注释添加到JavaScript 。JetBrains Rider识别Flow结构并在所有操作系统上为其提供语法突出显示。

在开始之前，请确保您的计算机上装有Node.js。

安装和配置流程

  • 在嵌入式终端 （ Ctrl+Alt+1 ），键入以下命令之一：

    • npm install --global flow-bin全局安装Flow。

    • npm install --save-dev flow-bin将Flow安装为开发依赖项。

从Flow官方网站了解更多信息。

要使JetBrains Rider识别Flow结构，提供正确的语法突出显示，正确报告错误并避免错误的正错误突出显示，将项目中的JavaScript语言级别更改为Flow ，向项目中添加.flowconfig配置文件并提供每个文件用一个检查// @flow在顶部发表评论。

将项目语言级别更改为Flow

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到语言和框架| JavaScript 。JavaScript页面打开。

  2. 从JavaScript语言版本列表中，选择Flow 。

  3. 在流程包或可执行文件字段中， 指定node_modules \ flow-bin node_modules / flow-bin软件包或Flow二进制可执行文件的路径。要使用node_modules \ .bin \ flow node_modules / .bin / flow，请确保将Node.js的路径添加到PATH环境变量。

  4. 在“ 将流程服务器用于：”区域中， 通过选中或清除以下复选框，指定编码帮助的基础：

    • 类型检查： 选中此复选框后，将基于从Flow服务器接收的数据提供语法和错误突出显示。清除复选框后，仅基本的内部JetBrains Rider高亮显示可用。

    • 导航，代码完成和类型提示： 选中此复选框后，参考解析和代码完成的建议列表g既包含从与Flow集成中检索到的建议，又包含由JetBrains Rider计算得出的建议。清除复选框后，仅通过JetBrains Rider计算来解析引用。

    仅在指定Flow可执行文件的路径时，此复选框才可用。

  5. 选中自动保存所有修改的文件复选框以确保连续应用Flow，因为Flow仅在所有其他修改的文件都保存之后才检查当前文件。

启用流程

  1. 要添加.flowconfig ， 打开嵌入式终端 （ Ctrl+Alt+1 ）并输入：

    flow init

    要将.flowconfig添加到特定文件夹，请键入：

    cd
    flow init

  2. 要在文件中启用Flow，请添加一个// @flow在文件顶部添加注释：只需键入flow ， 按Tab ，然后JetBrains Rider会将其扩展为// @flow 。

    如果您仍然没有.flowconfig ，JetBrains Rider会显示工具提示，提示您将配置文件添加到当前文件夹。

    要在当前文件夹和项目根目录之间进行选择，请单击“ 更多操作”或按Alt+Enter 。

监控错误

JetBrains Rider使您可以在专用的Flow工具窗口中查看Flow检测到的错误，将至少一个.flowconfig文件添加到项目后，该窗口将变为可用。要打开工具窗口，请选择“ 视图” | “工具”。工具窗口|从主菜单流 。

流动工具窗口包含两个窗格。

  • “ 错误”窗格显示了在活动编辑器选项卡中打开的文件中检测到的所有差异的列表。在顶部，显示文件的完整路径。

    当您打开“ 流动”工具窗口时，仅显示“ 错误”窗格。

  • 通过单击以下内容对整个项目运行Flow后，“ 项目错误”窗格将显示当前项目中所有文件中检测到的所有差异的列表。 在工具栏上。错误消息按在其中检测到的文件分组。
