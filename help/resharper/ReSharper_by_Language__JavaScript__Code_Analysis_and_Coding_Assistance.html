代码检查

ReSharper的静态代码分析可以检测JavaScript代码中的一百多种不同的错误和问题。

通过对当前文档或在任何指定范围内进行代码检查来执行分析。

要浏览JavaScript可用检查的列表，请打开 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ），然后展开JavaScript节点。

解决全局变量

ReSharper突出显示未定义变量的用法作为警告。但是，在某些情况下这些用法有效。例如，某些变量可以直接在标记中定义。

解决此问题的一种方法是禁用或抑制相应的代码检查。但是，您可以使用// global评论。在这种情况下，检查将保持活动状态，并帮助您查找变量名称中的错别字。

您可以将这些注释放在当前文件的顶部，也可以将单个文件用于项目的所有全局变量。每个变量都需要一个新注释，例如： // global variableName

使ReSharper忽略特定代码

ReSharper允许您以不同的方式忽略特定的文件，文件夹和文件掩码。您可以在以下选项页面上配置忽略的文件：

  • 代码检查|从代码检查中排除文件，文件夹和掩码的设置 。

  • 代码检查|生成的代码可以部分禁用对生成的代码的代码检查 。

  • 环境|搜索和导航以指定“ 按名称搜索”命令应忽略的文件和文件夹。

  • 代码编辑|要添加的第三方代码 C ++，JavaScript，TypeScript，CSS，HTML和JSON文件，文件夹和通配符将被视为“跳过”或“库”。ReSharper将完全忽略“已跳过”的文件，并将“库”文件视为只读文件-为导航而建立索引，但不进行检查 ， 快速修复和重构 。
