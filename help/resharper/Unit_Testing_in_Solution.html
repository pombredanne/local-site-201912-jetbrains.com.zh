在项目或解决方案中使用单元测试

可以使用其他JetBrains .NET产品扩展单元测试帮助：您可以使用dotTrace进行单元测试配置文件，并使用dotCover分析单元测试的代码覆盖率。这些产品也包含在ReSharper Ultimate中 。

ReSharper的单元测试运行程序不允许运行两个或多个具有相同ID的测试。在实践中，这种情况很少发生，但是如果您确实有具有相同ID的测试，它们将在“单元测试”浏览器和“单元测试会话”窗口中显示为单个节点，并且仅运行其中一个。
请注意，测试框架以不同的方式识别测试。例如，MSTest使用测试方法FQN，而NUnit使用测试属性的属性作为测试ID。

在解决方案中发现单元测试

ReSharper将“ 单元测试资源管理器”窗口添加到Visual Studio（ ReSharper |单元测试|单元测试，或ReSharper | Windows |单元测试 ，或Ctrl+Alt+T ）。 使用此窗口，您可以浏览和运行或调试整个解决方案中所有受支持框架的单元测试。请注意，来自项目的单元测试仅在构建项目后出现在窗口中。当前打开的文件中的测试会自动更新，创建后的新文件会立即显示在单元测试浏览器中。

单元测试浏览器允许您执行以下操作：

  • 探索解决方案中的测试：在列表或树视图中浏览所有单元测试，搜索测试并按子字符串过滤，按项目，名称空间， 类别等对单元测试进行重新组合。

  • 通过在视图中双击导航到任何测试或测试类的源代码。

  • 运行或调试选定的测试。

  • 从所选测试和测试类创建单元测试会话 ，和/或将所选项目添加到当前测试会话。

  • 将所有测试从解决方案导出到文本，XML或HTML文件。

在项目或解决方案中运行和调试单元测试

您可以从单元测试资源管理器，解决方案资源管理器或类视图运行或调试测试。单元测试资源管理器使您可以仅查看测试和测试类，而在使用其他窗口时，则需要知道哪些项目，文件和类包含测试。

  • 要从Unit Test Explorer执行测试，请选择所需的测试，然后单击运行单元测试 / 调试单元测试 在工具栏上，或使用相应的快捷方式（ Ctrl+T,R / Ctrl+T,D ）。

  • 要从解决方案资源管理器或类视图运行或调试测试，请选择包含测试的一项或多项（类，文件，文件夹，项目），然后使用 运行单元测试 / 调试单元测试命令，这些命令在主菜单（ ReSharper |单元测试 ），上下文菜单中或通过Ctrl+T,R / Ctrl+T,D相应的快捷方式。

  • 要运行或调试解决方案中的所有测试，请选择 ReSharper |单元测试|从主菜单中的“ 从解决方案运行所有测试”或按Ctrl+T,L 。

无论选择哪种方式运行或调试测试，您都将在“ 单元测试会话”窗口中看到执行进度，结果和输出。 如果存在开放的单元测试会话 ，则将已执行的测试添加到该会话中。如果没有测试会话或现有会话被锁定 ，则会创建一个新的测试会话。

如有必要，您始终可以通过按来重新运行上次执行的测试Ctrl+T,T或选择 ReSharper |单元测试|在菜单中重复上一个运行 。
