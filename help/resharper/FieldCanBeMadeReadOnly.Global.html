代码检查：字段可以设为只读（非私有可访问性）

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

这是解决方案范围内的代码检查 。它仅在启用解决方案范围的分析时才有效。

假设您已决定要Person类，仅通过构造函数进行初始化，但具有公开可用的字段。您继续执行以下操作：

公共类Person {public string Name;公共诠释时代；公共人员（字符串名称，整数年龄）{名称=名称；年龄=年龄； }公开替代字符串ToString（）=> $“ Name：{Name}，Age：{Age}”; }
公共类人公共名称为String公共年龄为Integer Public Sub New（名称为String，年龄为Integer）我。名字=我的名字Age = age End Sub公共重写函数ToString（）as String返回字符串。Format（“名称：{0}，年龄：{1}”，名称，年龄）结束函数结束类

如果启用了解决方案范围的检查，ReSharper将检测到Name和Age字段仅在构造函数中分配，并提供创建附加保护措施的功能-通过标记它们readonly ，我们要确保此类及其用户都不会无意中为这些字段分配新值。

为了使解决方案范围的检查有效，您需要以下一项或两项：

  • 启用了简化的全局使用检查— 在关闭解决方案范围的分析时，显示未使用的非私有类型成员 代码检查|的设置页面 ReSharper选项（ Alt+R,,O ）。

  • 该解决方案范围的分析已启用- 启用解决方案范围的分析 代码检查|的设置页面 ReSharper选项（ Alt+R,,O ）。

请注意，即使符号在您的解决方案中没有直接使用且ReSharper会警告您，在某些情况下，符号可能会间接使用（例如，通过反射），也可能只是设计为公共API。在所有这些情况下，您都希望抑制符号的使用检查，并且有几种方法可以这样做：

  • 推荐的方法是用代码注释属性修饰隐式使用的符号。为此，有两个属性： [UsedImplicitly]和[PublicAPI] ，它们在功能上相似，但是可以让您和您的队友了解如何实际使用该符号。

  • 您还可以使用任何自定义属性禁止使用情况检查检查。为此，请使用[MeansImplicitUse]属性标记该属性的定义。

  • 最后，您可以像抑制其他代码检查一样，通过抑制注释或抑制属性来抑制特定的使用检查。
