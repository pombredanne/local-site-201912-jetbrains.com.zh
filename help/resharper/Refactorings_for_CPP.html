C ++重构

ReSharper C ++提供了C ++支持， ReSharper C ++是一种专用产品，可以单独安装，也可以与ReSharper或ReSharper Ultimate并排安装。

除了重构之外，ReSharper还提供了 Context Actions可用的许多较小的本地代码转换 。

ReSharper为C ++提供了以下重构：

  • 更改签名
  • 提取方法
  • 介绍领域
  • 介绍命名空间别名
  • 介绍变量
  • 内联变量
  • 改名
  • 介绍/内联typedef

执行重构

  1. 将插入符号设置为符号，选择要重构的代码片段，或在工具窗口中选择一个项目。

  2. 请执行以下任一操作：

    • 在主菜单中，选择“ ReSharper |重构 ，然后选择所需的重构。此菜单中可用的重构列表取决于当前上下文。如果ReSharper无法为上下文建议任何重构，则将禁用整个菜单。

    • 在编辑器的“ 文件结构”窗口或其他“ ReSharper”窗口中，右键单击要转换的项目，从上下文菜单中选择“ 重构 ”，然后选择所需的重构。

    • 在主菜单中，选择“ ReSharper |重构|重构此 ，或按Ctrl+Shift+R以显示适用的重构列表，然后选择其中之一。您也可以在选择的上下文菜单中选择“ 重构此 ”。

    • 使用分配给特定重构的默认键盘快捷方式 ，或将自定义快捷方式分配给您喜欢的重构命令。

  3. 如果所选的重构需要用户输入，则重构向导将打开。请注意，向导的对话框不是模态对话框，因此您可以在向导打开时编辑代码。

    要回滚重构操作，向导提供选项“启用撤消，打开所有具有更改的文件以进行编辑” 。如果选择此选项，ReSharper将在新的编辑器选项卡中打开所有修改的文件，并使您能够回滚重构。在这种情况下，您需要自己保存更改。如果未选择此选项，则ReSharper会自动保存已修改的文件，而无需打开它们。

  4. 如果重构操作将导致代码冲突（例如重复的名称，可见性冲突等），则在应用重构之前，向导将在最后一步显示冲突列表。对于某些冲突，向导还可以建议快速修复。有关更多信息，请参见解决重构中的冲突 。

Visual Studio提供了自己的“ 重构”菜单，可从主菜单和上下文菜单中使用。默认情况下，ReSharper隐藏本机Visual Studio菜单，将其替换为自己的菜单。要使Visual Studio菜单可访问，请在“ 环境” | “ 环境”中选择“ 隐藏覆盖的Visual Studio菜单项”复选框。 ReSharper选项的“ 键盘和菜单”页面。

在编辑器中修改代码后，可以立即进行某些重构。有关更多信息，请参见就地重构。

更改签名

更改签名重构允许您对函数的签名进行一个或多个不同的修改。该功能的所有用法，实现和替代都将相应更新。

您还可以使用专用快捷方式调用此重构Ctrl+F6 。

提取方法

通过此重构，您可以基于所选代码片段创建新方法。ReSharper分析选定的语句并检测可以转换为方法参数或代表其返回值的变量。

您还可以使用专用快捷方式调用此重构Ctrl+Alt+M 。

一旦单击“ 下一步” ，就会创建新方法，并将所选的表达式替换为方法调用。

您还可以使用专用快捷方式调用此重构Ctrl+Alt+D 。

介绍命名空间别名

此重构可帮助您为名称空间用法创建名称空间别名，并用别名替换当前选择的用法或当前文档中的所有用法。根据所选用法，在与用法最接近的范围内声明名称空间别名。

要调用此重构，请将插入符号设置为使用命名空间，然后按Ctrl+Shift+R或选择 ReSharper |重构|从菜单中重构… ，然后在“ 重构此”弹出窗口中选择“ 引入名称空间别名 ”。如果在文档中出现了多次使用命名空间的情况，则可以选择是替换当前用法还是替换所有用法。

介绍变量

通过此重构，您可以基于选定的表达式创建新的局部变量，使用该表达式对其进行初始化，最后使用对新引入的变量的引用替换该方法中所有出现的表达式。

您还可以使用专用快捷方式调用此重构Ctrl+Alt+V 。

内联变量

通过这种重构，您可以用其初始化程序替换代码中所有出现的变量。请注意，只有在变量值在初始化后保持不变的情况下，才应应用重构。

您还可以使用专用快捷方式调用此重构Ctrl+Alt+N 。

改名

C ++支持最耗时的重构之一。如果尝试手动修改符号名称，可能会导致很多问题。调用重命名重构时 （也可用于专用F2快捷方式），所有检查均由ReSharper完成。如果不存在冲突，则所有修改都可以顺利进行，或者获得可以手动解决的冲突列表，以确保仅进行必要和适当的更改。

使用此重构重命名类时，ReSharper将自动重命名相应的文件（源文件和标头）。

您也可以在解决方案资源管理器中的文件上调用此重构。为文件提供一个新名称后，ReSharper就会在include中更新其所有用法。

介绍/内联typedef

Introduce typedef重构使您可以快速创建一个typedef选定的数据类型，并用新创建的数据替换选定的数据类型，并可选地将当前文件中所有出现的该数据类型替换typedef 。

内联typedef重构正好相反-它删除了选定的typedef并将其所有用法替换为声明的数据类型。

如果您希望Introduce typedef重构使用类型别名代替typedef ，选中相应的复选框 代码编辑| С++ |的代码样式页面 ReSharper选项（ Alt+R,,O ）。有关更多信息，请参见Microsoft Docs上的别名和typedef（C ++） 。
