外部资源

ReSharper |选项|工具|外部资源

在ReSharper选项的此页面上，您可以定义ReSharper 导航到外部源的方式 。

项目

描述

默认的Visual Studio导航

单击此选项以使用Visual Studio导航。ReSharper到外部源的导航被禁用。

导航到对象浏览器

单击此选项默认情况下导航到“对象浏览器”。

导航到装配体浏览器

单击此选项以使Assembly Explorer窗口成为导航到已编译代码时的默认目标。

导航到源

单击此选项默认情况下启用导航到外部源。ReSharper会根据以下定义的设置导航到“元数据视图”或反编译的代码，或来自符号文件的源。无论选择什么设置，当您明确指定导航目标时，ReSharper都可以导航到任何这些目标。

可用时使用符号文件中的源

选中此复选框可使用调试信息（PDB文件）查找适当的源文件，并使用它们显示和导航代码。

如果选中此复选框，并且可通过调试信息获得源文件，则ReSharper将显示并导航到原始源代码。


如果清除此复选框或源文件不可用，则ReSharper会根据“ 反编译方法”选项反编译代码或显示元数据视图。

您始终可以强制显示反编译的或源代码（如果可以通过PDB文件或从符号服务器获得）。为此，在Assembly Explorer窗口中右键单击一个项目，然后选择Decompiled Sources或Symbol Files中的Sources 。

允许从远程位置下载

选择此复选框以启用从Symbol Server或其他远程位置下载源。如果符号文件在本地和远程均可用，则ReSharper将显示本地符号文件。

您的计算机应连接到Internet，此选项才能起作用。如果使用代理服务器，请检查是否在以下位置正确配置了相应的选项 环境|的“ Web代理设置”页面 ReSharper选项（ Alt+R,,O ）。

高级符号选项

打开“ 高级符号选项”对话框以创建或编辑文件夹替换规则。
如果外部库是用另一台计算机编译的，则PDB文件中的路径可能是相对于该计算机的，因此无法自动解析这些路径。
要解决此问题，您需要为每个路径添加文件夹替换规则。每个文件夹替换规则包含两个路径：

  • 源文件夹 -在PDB文件中指定的文件夹。

  • 目标文件夹 -具有计算机上相关源文件或远程服务器相应URL的文件夹。

反编译方法

选中此复选框可将方法主体反编译为C＃代码。如果未选中此复选框，将显示带有方法签名的元数据视图 。
如果选择了“ 在可用时使用符号文件中的源”并且所需的符号文件可用，则此选项没有任何效果。

默认情况下使用Assembly Explorer而不是Object Browser

如果要在“解决方案资源管理器”中双击引用时要在“ 程序集资源管理器”窗口中而不是“对象浏览器”中打开已编译的程序集，请选中此复选框。无论使用哪种首选项，都可以右键单击引用，然后选择在何处打开它。

在“转到...”结果中显示非公开的已编译项目

如果要在“ 按名称搜索”命令的搜索结果中查看已编译代码中的非公共类型和成员，请选择此选项。
默认情况下，它是禁用的，因为您通常不希望学习此类项目，因为您无法访问它们。

显示支持信息

单击链接以显示有关当前外部源配置的技术详细信息。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
