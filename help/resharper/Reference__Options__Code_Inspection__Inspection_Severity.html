检验等级

ReSharper |选项|代码检查|检验等级

ReSharper选项的此页面按语言和类别列出了所有ReSharper的代码检查 。使用此页面可以更改ReSharper分配给每次检查发现的问题的严重性级别 ，或禁用特定检查。

    请注意，此页面仅列出具有可配置严重性级别的检查。还有数百种检测编译器错误的检查，它们具有固定的错误严重性级别 ，此处未显示。

    要快速查找代码检查项目，请使用页面顶部的搜索框。

    每种语言的代码检查分为以下几类：

    • 潜在的代码质量问题
      此类检查包括检测关键问题（代码异味）的检查，主要是错误或警告级别。此类别还包括确保本地化协助的检查 。

    • 通用惯例和代码改进
      此类别将检查归类为主要影响代码可读性的中等严重性问题。

    • 代码中的冗余
      此类别中的代码检查查找冗余和无效代码，这些代码会影响代码的可读性和样式，并且可以安全地删除。 某些代码冗余无法自动修复，因此需要在用户输入的情况下以交互方式进行快速修复。但是，可以使用范围内的修复或代码清除来修复大多数冗余，而无需用户交互。

    • 语言使用机会
      此类别包括代码检查，大部分带有建议严重性级别，这些通知会在可以使用更高级的语言构造时通知您。这些检查将检测过时的语言版本的语法，并建议使用更多现代语言版本的功能。对于大多数受支持的语言， 可以自动检测语言版本或手动设置 。

    • 代码通知
      此类别将代码检查的严重性级别较小。

    • 代码风格
      此类别中的检查可检测到违反代码语法样式的行为。与大多数其他代码检查相比，这些检查可以检测与代码发布相同的代码构造，也可以不检测，具体取决于在代码上配置的相应代码样式规则。 代码编辑| [语言] |的代码样式页面 ReSharper选项（ Alt+R,,O ）。您还可以使用代码清除修复这些检查检测到的问题。

    • 约束违规
      此类别包括代码检查，大多数情况下具有警告严重性级别，该代码检查可检测与符号属性相关的违规行为，包括ReSharper的代码注释和其他类似问题。

    • 符号声明中的冗余
      此类别包括代码检查，大多数情况下具有警告严重性级别，可检测空的和未使用的符号声明。

    • 编译器警告
      此类检查将在编译之前检测到编译器警告。

    • 拼写问题
      这些检查可以检测各种情况下的错别字。

    • 不可运行的测试
      这些检查可以检测到无法执行的测试。

    • NUnit
      这些检查可以检测与NUnit测试有关的代码问题。

    • Xunit
      这些检查可以检测与xUnit.Net测试相关的代码问题。

    • 格式化
      此类别中的检查可检测代码编码格式问题。

    • lang整齐的支票
      此类检查由Clang-Tidy提供， Clang-Tidy是与ReSharper C ++ 集成的功能强大的开源代码分析工具。

    • 铛
      此类别中的检查对应于与ReSharper C ++ 集成的 Clang编译器警告。

    • lang静态分析仪检查
      此类别中的检查是来自与ReSharper C ++ 集成的 Clang静态分析器的诊断。
      默认情况下，所有静态分析器检查均处于禁用状态，因为启用它们会大大降低Clang-Tidy的速度。

    • 虚幻引擎
      此类别中的检查特定于Unreal Engine项目的C ++代码。

    如果检查的默认严重性级别已更改，您将看到“ 重置为默认” 按钮旁边的按钮，您可以将严重性重置为其默认值。
    同样的按钮也会出现在此类检查所属的类别旁边，单击类别旁边的此按钮会将类别内的所有检查重置为默认严重性级别。

    ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
