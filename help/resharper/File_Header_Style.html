文件头样式

您的文件标题首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

将文件标题用于版权声明和其他标识消息是一种常见的做法。ReSharper允许您配置默认的标题文本，并将其自动插入解决方案的代码文件中。

如果使用文件模板创建配置的文件头，则可以将其插入新文件中。为此，您应该添加一个模板参数 ，并将“文件头” 宏与其关联。

如果要将文件头应用于现有文件，则可以使用代码cleanup来完成 。

配置文件头并将其插入代码文件中并进行代码清理

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择代码编辑|左侧的文件标题文本 。

  2. 在“ 文件头文本”区域中，将要插入的任意文本插入代码文件。您还可以使用允许文件头包含各种信息的宏。要使用宏，请键入它或单击相应的宏，它将被插入到当前插入符号的位置。

  3. （可选）要用区域包围文件头，请在“将文件头包含在具有名称的区域”字段中输入区域名称 。

  4. 转至代码清理设置页面： 代码编辑|代码清理 。

  5. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 所选配置文件设置”部分中，选中“ 更新文件头”复选框。（可选）您可以在此配置文件中启用其他代码清除任务。

  6. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  7. 选择您要更新文件头的范围：

    • 将插入符号设置在文件中的任何位置，以将文件头更新为文件。

    • 在解决方案资源管理器中选择一项或多项，以更新这些节点及其子项下文件的文件头。

  8. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  9. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  10. 点击运行 。ReSharper将更新所选范围中的文件头。

如果要插入文件头而没有打开“ 代码清除”对话框 ，则可以将创建的配置文件绑定到静默清除并通过按运行它Ctrl+Shift+Alt+F 。您还可以创建一个自定义清除配置文件 ，该配置文件将插入文件头与其他代码样式任务结合在一起。
