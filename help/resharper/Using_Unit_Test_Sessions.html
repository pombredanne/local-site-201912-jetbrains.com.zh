使用单元测试会话

ReSharper的单元测试运行程序不允许运行两个或多个具有相同ID的测试。在实践中，这种情况很少发生，但是如果您确实有具有相同ID的测试，它们将在“单元测试”浏览器和“单元测试会话”窗口中显示为单个节点，并且仅运行其中一个。
请注意，测试框架以不同的方式识别测试。例如，MSTest使用测试方法FQN，而NUnit使用测试属性的属性作为测试ID。

ReSharper允许您将针对应用程序特定部分的单元测试分组为单元测试会话。单元测试会话可以包含来自不同项目的不同受支持框架的测试。您可以有多个测试会话，并根据需要分别运行。单个测试/测试类可以包含在几个不同的测试会话中。

如果不同的单元测试会话中的执行设置不同，则测试可能在一个会话中通过而在另一个会话中失败。因此，ReSharper不会在添加了相同测试的其他会话中更新执行结果。同样，在编辑器中为测试显示的执行状态图标对应于上次执行该测试会话的结果。

默认情况下，将为每个解决方案自动保存单元测试会话。即使关闭并重新打开解决方案，ReSharper也会保留它们。如有必要，您可以通过清除保存和恢复单元测试会话复选框来选择不保留单元测试会话。 工具| ReSharper选项的“ 单元测试”页面。

可以使用其他JetBrains .NET产品扩展单元测试帮助：您可以使用dotTrace进行单元测试配置文件，并使用dotCover分析单元测试的代码覆盖率。这些产品也包含在ReSharper Ultimate中 。

访问单元测试会话

单元测试会话管理在“ 单元测试会话”窗口中完成 。 要打开此窗口，请按Ctrl+Alt+R或选择 ReSharper | Windows |从主菜单进行单元测试会话 。每次您从当前文档或项目/解决方案运行或调试单元测试， 创建新的测试会话或将测试添加到 测试会话时 ，该窗口也会自动打开。

所有现有的单元测试会话都显示在“单元测试会话”窗口的选项卡中。

“单元测试会话”窗口还可视化了测试运行器的活动：它显示了执行进度以及已执行测试的状态和输出。每次运行或调试测试时，它们要么添加到当前测试会话中，要么为它们创建一个新的单元测试会话。

要搜索测试，请在焦点进入此窗口后立即开始输入测试名称或其CamelHumps缩写。

创建和关闭单元测试会话

有几种方法可以创建新的测试会话：

  • 只需从当前文档或项目/解决方案中运行或调试单元测试。 如果存在开放的单元测试会话 ，则将已执行的测试添加到该会话中。如果没有测试会话或现有会话被锁定 ，则会创建一个新的测试会话。

  • 在编辑器中，单击测试或测试类旁边的操作指示器 ，或在此项目中设置插入符号，然后按Alt+Enter ，然后在操作列表中选择“ 创建新会话 ”。将创建与此测试/测试类的新会话。

  • 在“ 单元测试资源管理器”窗口中 ，为新会话选择测试/测试类，然后单击“ 创建新会话” 在工具栏上或按Ctrl+T,N 。

  • 在主菜单中，选择 ReSharper |单元测试|创建新会话或按Ctrl+T,N 。如果焦点位于编辑器中或“单元测试资源管理器”窗口中，并且选择了某些测试/测试类，或者插入符号位于测试/测试类中或内部，则将创建带有相应项目的新测试会话。否则，将创建一个空的测试会话。

如果要防止向会话中添加新测试，则可以锁定此会话 。

要关闭单元测试会话，只需关闭“单元测试会话”窗口中的相应选项卡。封闭的会议将不再可用。

在测试会话中添加和删除项目

默认情况下，如果您运行或调试未包含在任何会话中的测试，则这些测试将添加到当前会话中。

如果要防止向会话中添加新测试，则可以锁定此会话 。

还有其他将测试/测试类添加到会话中的方法：

  • 在编辑器中，单击测试或测试类旁边的操作指示器 ，或在此项目中设置插入符号，然后按Alt+Enter ，然后在操作列表中选择“ 附加到[当前会话] ”。如果要将项目添加到其他部分，请单击此命令旁边的小箭头，然后在子菜单中选择任何现有的测试会话：

    您也可以选择 ReSharper |单元测试|在主菜单中将测试追加到会话或按Ctrl+T,A将所选项目添加到会话中。

  • 在“ 单元测试资源管理器”窗口中 ，选择要添加的测试/测试类，然后单击“ 将测试追加到会话” 在工具栏上或按Ctrl+T,A 。如果会话不止一个，请在出现的弹出窗口中选择目标会话。

要从会话中删除测试，请选择要删除的测试或分组项，然后单击“ 删除选定的测试”。 在工具栏上。

重命名测试会话

默认情况下，添加到新测试会话中的第一个测试/类将用作会话的名称。

要重命名会话，请右键单击该会话的选项卡，然后从上下文菜单中选择“ 重命名会话 ”或双击该选项卡，然后在出现的对话框中为该会话指定一个新名称。

锁定和解锁单元测试会话

默认情况下，如果您运行或调试未包含在任何会话中的测试，则这些测试将添加到当前会话中。

如有必要，可以防止将新测试添加到会话中。这样做，在“单元测试会话”窗口中，切换到要锁定的会话的选项卡，单击工具栏上的“ 选项 ”，然后选择“ 锁定会话”。 。

要解锁会话，请重复相同的操作。
