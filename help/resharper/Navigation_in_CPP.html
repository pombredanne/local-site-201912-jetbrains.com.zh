C ++中的导航和搜索

ReSharper C ++提供了C ++支持， ReSharper C ++是一种专用产品，可以单独安装，也可以与ReSharper或ReSharper Ultimate并排安装。

按名称导航

ReSharper可以通过几次按键将您带到任何代码项或文件。导航的范围包括整个解决方案中的符号和文件，以及标准和库以及外部依赖项中的所有符号和文件。

在解决方案中按项目名称查找项目

  1. 要对所有搜索结果使用统一访问，请按Ctrl+N显示一个弹出窗口，您可以在其中开始输入并找到解决方案中与您输入内容匹配的所有内容 。

  2. 如果要将搜索限制为类型（类，接口，结构或枚举），请按Ctrl+N两次。

  3. 如果要通过纯文本匹配查找任何内容 ，请按Ctrl+N三次。

  4. 如果要搜索符号 （类型，方法，属性，字段等），请按Ctrl+Shift+Alt+N 。

  5. 您还可以通过按将搜索限制为％files％ Ctrl+Shift+N 。

  6. ReSharper还可以让您跳到类型并在当前文件中键入成员，方法是按Ctrl+F12 。

  7. 调用以上任何命令后，开始键入目标项目，其CamelHumps缩写和/或通配符。键入内容时，匹配项的列表将缩小，并且最可能的候选项显示在列表顶部附近。

  8. 一旦目标项目出现在列表中，您就可以选择它以在编辑器中将其打开。 要在“ 查找结果”窗口中查看和分析匹配项的列表， 单击在查找结果中显示 ， 按Shift+Enter要么+在数字键盘上。

按Ctrl+N随后在“无处不在搜索”，“转到类型”和“转到文本”命令之间进行迭代。

ReSharper可以记住您在“随处搜索/转到类型” ， “符号” ， “文件 ”和“文本”操作中用于查找内容的最后输入。若要启用或禁用此行为，请使用“ 环境” | “ 记住上一个搜索”复选框。 ReSharper选项的“ 搜索和导航”页面。选择此选项后，ReSharper还将在编辑器中使用您当前的选择作为初始搜索查询。例如，您可以在字符串中选择文件名，然后按Ctrl+Shift+N在您的解决方案中搜索与此名称匹配的文件。

前往申报

使用此命令可从任何符号用法导航到符号声明。 您可以从编辑器，“ 文件结构”窗口和其他工具窗口中调用此命令。
要以相反的方向导航（符号用法），可以在已经在符号声明上使用时调用“ 转到声明”命令，也可以使用“ 转到符号的用法”命令。

要转到符号声明，请按Ctrl+B ，选择 ReSharper |导航|在主菜单中转到“声明” ，或单击按住Ctrl键。

在非依赖项上进行声明工作decltype说明符和lambda（跳到lambda主体）。您可以从声明跳转到签名不匹配的定义（例如，从成员函数的声明更改到带有旧签名的定义），反之亦然。在初始项目索引编制过程中，转到声明也适用。

在不受支持的C ++文件（例如C ++ / CX）中，“转到”声明将自动退回到相应的Visual Studio导航操作。

转到文件成员

使用此命令可以快速导航到当前文档中的特定类，函数或字段。

要导航到文件成员， 按Ctrl+F12或选择 ReSharper |导航|从主菜单转到文件成员… 。

转到符号类型

当“ 转到声明”命令将您导航到声明符号的位置时，“符号类型”命令将导航至当前所选符号类型的声明。例如，如果您有一个变量Foo myVar = new Foo();并在使用时调用此命令myVar ，它将带您进入课程的声明Foo 。如果在函数上调用此命令，它将导航到函数的返回类型。

此命令也适用于typedef。 您可以从编辑器，“ 文件结构”窗口和其他工具窗口中调用此命令。

要导航到符号的类型， 按Ctrl+Shift+T或选择 ReSharper |导航|主菜单中的符号类型 。

转到符号的用法

此命令使您可以从符号的声明或任何其他用法中快速导航到符号的特定用法。 您可以从编辑器，“ 文件结构”窗口和其他工具窗口中调用此命令。相反的导航可用于“ 转到声明”命令。

通过此命令获得的用法列表与“ 查找用法 ”列表相同。但是，如果您正在查找特定用法并且想要在不打开“ 查找结果”窗口的情况下使用该用法，则此命令对于用法数量有限的符号更为方便。

如果您调用此命令并发现用法搜索花费的时间太长，则可以单击在查找结果中显示 ， 按Shift+Enter要么+在数字键盘上-搜索将在“ 查找结果”窗口中的背景上继续。

您可以为发现一种用法的情况配置ReSharper行为。默认情况下，用法显示在弹出窗口中。如果您希望ReSharper在这种情况下跳过单个用法，请在“ 环境” | “ 环境” |“选项”中选择相应的选项（ 转到“用法”：如果只有一个结果，则导航而不显示列表 ）。 ReSharper选项的“ 搜索和导航”页面。

要了解符号的用法， 按Ctrl+Alt+F7或选择 ReSharper |导航|主菜单中符号的用法 。

转到基本符号

该命令使您可以在继承层次结构中导航到当前符号的基本类型或方法。 您可以从编辑器，“ 文件结构”窗口和其他工具窗口中调用此命令。
使用“ 转到派生符号”和“ 转到实现”命令可以进行相反的导航。

要导航至基本符号， 按Ctrl+U或选择 ReSharper |导航|主菜单中的基本符号 。

如果某个方法替代，实现或隐藏了另一个方法，则可以通过出现特殊图标来查看它（例如， ）显示在方法声明的左侧。将鼠标悬停在该图标上可以查看此方法的基本类型；单击该图标可在方法层次结构中向上导航。

前往实施

通过此命令，您可以从基本类型或成员跳到其任何最终实现，而绕过继承链中的中间步骤。换句话说，它的工作方式类似于“ 转到派生符号”，但不显示抽象类。

除了导航之外，您还可以直观地浏览符号的完整继承层次。

您可以从编辑器，“ 文件结构”窗口和其他工具窗口中调用此命令。如有必要，可以使用“ 转到基本符号”命令沿相反方向导航。

要导航到实现， 按Ctrl+Shift+Alt+B或选择 ReSharper |导航|从主菜单转到实施 。

转到派生符号

此命令获取当前符号的所有派生符号的列表，并让您直接沿继承层次结构导航至其中一个。有一个类似的命令Go to Implementation ，它仅列出实现，而在继承链中没有中间步骤。

要导航到派生符号， 按Ctrl+Alt+B或选择 ReSharper |导航|从主菜单派生的符号 。

在标题和源之间切换

对于具有单独的声明和定义的所有符号，ReSharper将显示“ 导航到声明/定义” 动作指示器在左侧。您可以单击此指示器或按Alt+Enter在符号的声明和定义之间快速切换。

您可以隐藏导航到声明/定义 代码编辑器上的动作指示器| C ++ | ReSharper选项的“ 检查”页面。

在头文件和相应的源文件之间有更多的切换方法，反之亦然：

  • 按Alt+O 。

  • 按Ctrl+Shift+G或选择 ReSharper |导航|从主菜单中导航到… ，然后在“ 导航至”弹出窗口中选择“ 切换页眉/源 ”。

  • Ctrl -单击符号名称。

转到主要模板/专业化

如果类或函数模板具有多个专业化，则ReSharper允许您在主要模板/专业化之间切换。

当您的插入符号位于主要模板或某个专业领域时，有两种方法可在各专业领域之间导航：

  • 按Alt+Enter然后选择导航到专业 在行动清单中 。

  • 按Ctrl+Shift+G或选择 ReSharper |导航|从主菜单导航到… ，然后在“ 导航到”弹出窗口中选择“ 专业化 ”。

查找用法

此命令允许查找解决方案中特定符号的所有用法。您可以从符号的声明或任何其他用法开始搜索。 您可以从编辑器，“ 文件结构”窗口和其他工具窗口中调用此命令。

如果移动不止一种，ReSharper可以帮助您在“ 查找结果”窗口中 分析 结果 。

如有必要，您可以更改默认行为，以便在“ 查找结果”窗口中也显示找到的单个用法。要做到这一点，清除转到用法：如果只有一个结果....复选框在环境| ReSharper选项的“ 搜索和导航”页面。

要查找当前符号的用法， 按Alt+F7或选择 ReSharper |查找|从主菜单中找到用法 。

查找高级用法

通过此命令，您可以查找在特定范围内与所选符号某种方式相关的实体的用法（例如，文本出现，动态引用等）。 您可以从编辑器，“ 文件结构”窗口和其他工具窗口中调用此命令。

要查找当前符号和相关实体的用法， 按Shift+Alt+F7或选择 ReSharper |查找|从主菜单中找到“用法高级” 。

突出显示文件中的用法

如果要在当前文档中查找符号的用法，则突出显示所有用法并在它们之间跳转可能比在“ 查找结果”窗口中分析用法更方便。请注意，读写访问用法以不同的颜色突出显示。

要查找当前符号和相关实体的用法， 按Ctrl+Shift+F7或选择 ReSharper |查找|从主菜单中突出显示“文件中的用法” 。

转到包含声明

使用单个快捷键，您可以在编辑器中从其体内的任何位置跳转到类型或函数的声明。

要进入声明， 按Ctrl+[或选择 ReSharper |导航|从主菜单包含声明 。

转到下一个/上一个成员

使用“ 下一个/上一个成员”命令，您可以在代码文件中的成员和嵌套类型之间或标记文件中的标记之间快速导航。

要调用这些命令，请按Alt+Down / Alt+Up或选择ReSharper |导航|主菜单中的下一个/上一个成员 。

转到相关文件

该命令可以将您带到其内容与当前文件某种程度上相关的所有文件。

要转到相关文件， 按Ctrl+Shift+Alt+G或选择 ReSharper |导航|相关文件…从主菜单。

在解决方案资源管理器中找到

此便捷的导航功能使您可以在“解决方案资源管理器”中展开树，自动滚动并突出显示与当前文档相对应的项目。当您拥有庞大的解决方案或有限的屏幕区域（迫使您将“解决方案资源管理器”窗口缩小至极限）时，此功能特别有用。

请注意，如果当前解决方案中未包含在编辑器中打开的文件，则此命令将不起作用。

要转到相关文件，请按Shift+Alt+L或选择ReSharper |工具|在主菜单中的解决方案资源管理器中找到 。

导航到最近的位置

ReSharper会跟踪您的操作和导航历史记录，并提供一些命令，使您可以导航到编辑或刚刚研究过代码的地方。

  • 转到最近的文件 Ctrl+E
  • 转到最近的编辑 Ctrl+Shift+Alt+Backspace
  • 转到上一个编辑 Ctrl+Shift+Backspace

  • 导航后退/前进 （ReSharper的|导航|导航向后/向前 ）

导航文件结构

使用ReSharper，您可以使用“文件结构”窗口浏览当前文档的结构，这大大简化了大型文件的浏览。该窗口与编辑器同步：切换到另一个编辑器选项卡时，该窗口显示相应文件的结构。

类型层次

使用ReSharper，您可以从解决方案和引用的库中研究类型的继承层次结构。您可以同时看到基本类型和所选类型的继承者，并通过单击鼠标导航到它们中的任何一个。有关更多信息，请参见浏览类型层次结构。

包含的层次结构

ReSharper允许您直观地分析单个include指令或文件的include层次结构。

要构建包含的层次结构，请将插入符号设置为#include指令在文件中或在解决方案资源管理器中选择一个文件，然后按Ctrl+Alt+H或选择 ReSharper |检查菜单中的层次结构 。

如果关闭了“ 层次结构”窗口，则可以随时选择“ ReSharper” |“查看”，然后查看并检查以前生成的层次结构。 Windows |主菜单中的层次结构 。
但是，此命令不会生成任何新的层次结构，并且如果由于当前解决方案已打开而尚未生成任何层次结构，则该窗口可能为空。

您可以双击层次结构中的任何文件以在编辑器中将其打开。

默认情况下，每个文件的子节点代表此文件中包含的文件，该文件对应于Includee 工具栏上的“模式”。您可以切换到包含者 模式。在这种模式下，文件的子节点表示当前作用域中包含此文件的文件。

导航到行动

ReSharper可以帮助您快速查找和应用其任何操作，而无需浏览菜单。“ 动作”是指可用于当前插入符号位置的可执行命令或任何全局命令，例如，打开工具窗口的命令。
您也可以找到并导航到选项页面。

要查找并执行ReSharper操作， 按Ctrl+Shift+A或选择 ReSharper |导航|从主菜单转到操作… 。

断点导航

使用ReSharper，您可以快速跳到解决方案中的任何断点/跟踪点，并在专用的工具窗口中研究所有断点，该窗口具有分组，搜索，预览和符号名称的功能。

您可以在“ 查找结果”窗口中查看，切换和启用/禁用断点。

要跳到任何断点，请按Ctrl+Alt+F9或选择 ReSharper |导航|断点…从主菜单
