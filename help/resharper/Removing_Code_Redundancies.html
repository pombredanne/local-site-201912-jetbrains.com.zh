删除代码冗余

上 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ），在不同语言的代码类别中的冗余中 ，ReSharper列出了许多与代码检查相关联的代码冗余。对于大多数这些冗余，ReSharper提供了快速修复 。 某些代码冗余无法自动修复，因此需要在用户输入的情况下以交互方式进行快速修复。但是，可以使用范围内的修复或代码清除来修复大多数冗余，而无需用户交互。

通过在设计时代码检查中突出显示的任何代码冗余，您可以调用“删除冗余代码”快速修复程序，该快速修复程序将删除当前冗余以及文件，项目或解决方案中的所有冗余代码：

在批量模式下删除代码冗余的另一种方法是代码清除 。您可以可以使用默认的默认值：完全清理配置文件运行代码清理 ，或者创建和运行仅针对您的特定任务的自定义配置文件 ，如下所述。

使用自定义代码清除配置文件删除代码冗余

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 转至代码清理设置页面： 代码编辑|代码清理 。

  3. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 选定的配置文件设置”部分中，选中“ 删除代码冗余”复选框。（可选）您可以在此配置文件中启用其他代码清除任务。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  5. 选择您要删除代码冗余的范围：

    • 将插入符号设置在文件中的任何位置，以删除该文件的代码冗余。

    • 在解决方案资源管理器中选择一项或多项，以删除这些节点及其子项下文件中的代码冗余。

  6. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  7. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  8. 点击运行 。ReSharper将删除所选范围内的代码冗余。

代码冗余的首选项不应用于生成的代码。默认情况下，ReSharper提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ）。

如果要删除代码冗余而不打开“ 代码清理”对话框 ，则可以将创建的配置文件绑定到静默清理并通过按以下命令运行它Ctrl+Shift+Alt+F 。您还可以创建一个自定义清除配置文件 ，该配置文件将删除代码冗余与其他代码样式任务结合在一起。
