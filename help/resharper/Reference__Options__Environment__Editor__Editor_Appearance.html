重点介绍
突出显示当前行

选中此复选框可在使用代码时突出显示当前行。

您可以在Visual Studio选项中通过选择“ 工具” |“设置”来设置突出显示的颜色。 选项|环境| ReSharper当前行显示项目的字体和颜色以及调整选项。

重新加载此文档后，对此选项所做的更改将在打开的文档中生效。

突出显示匹配的分隔符

选中此复选框时，当插入符号为：时，任何结束定界符（括号，花括号，括号或引号）都将突出显示：

  • 如果在外侧选择，则紧接在相应的开口定界符的左侧 。

  • 如果您在开口定界符的两侧选择了两侧，则在两侧 。

您也可以使用带高亮显示的单选按钮来选择高亮显示匹配的括号和引号的两种方法之一：用颜色显示或使用轮廓较浅的轮廓。

有关更多信息，请参见突出显示匹配定界符 。

动作指示灯外观

您可以选择操作指示器在编辑器中的显示方式。

  • 在操作栏中显示 （仅在启用操作栏时可用）-所有指示器都显示在操作栏中。

  • 显示得更接近代码 -上下文相关的指示器（ 快速修复 ， 上下文操作等）显示在当前行左侧的文本编辑器区域中，而固定指示器（层次结构导航，单元测试）显示在在操作栏上（如果已启用）或在编辑器的左边缘。

  • 不显示 - 完全不显示上下文相关指示器，而固定指示器则显示在操作栏（如果已启用）或编辑器的左边缘。此选项适用于有经验的用户，他们可以猜测何时按下Alt+Enter调用动作列表 。

天沟痕迹
显示已实施和已覆盖成员的装订线导航标记

当类型或成员在当前解决方案中具有继承者时，您可以通过出现特殊图标（例如， ）显示在声明的左侧。您可以单击图标以展开派生符号列表并进行导航。

为了实时检测成员的继承者并显示相应的装订线标记，ReSharper必须执行一些计算。大型继承层次结构与弱硬件的组合可能导致性能下降。在这种情况下，您可以清除此复选框以避免复杂的计算，从而提高性能。
另一方面，在类型上继承者的即时计算在性能上要便宜得多，因此不被禁用。

如果您选择禁用装订线标记，则始终可以使用显式导航命令- 转到派生符号 Ctrl+Alt+B并执行 Ctrl+Shift+Alt+B -研究派生成员，或以视觉方式探索当前成员的完整继承层次结构。

工具提示

替换Visual Studio工具提示

此选项使您可以将Visual Studio符号工具提示替换为ReSharper工具提示，当您将鼠标悬停在编辑器中的代码符号（快速信息）上时出现。

如果您希望查看本机Visual Studio工具提示或使用Productivity Power Tools扩展，则可能要禁用此选项。

Visual Studio和ReSharper工具提示基本上提供了有关您将光标悬停在其上的符号的类似信息。在不同的Visual Studio版本中，Visual Studio工具提示有所不同。

如果禁用ReSharper 语法突出显示 （通过清除ReSharper选项的“代码检查|设置”页面上的“ 颜色标识符”复选框），还将禁用ReSharper工具提示。

此选项不影响C ++。要替换C ++的Visual Studio工具提示，请在 代码编辑| C ++ |的检查页面 ReSharper选项。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
