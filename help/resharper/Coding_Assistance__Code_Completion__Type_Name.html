导入符号完成

ReSharper |编辑导入符号
Ctrl+Alt+Space
ReSharper_CompleteCodeTypeName

该命令从当前项目中引用的模块中建议类型和扩展方法，而不管其名称空间是否导入当前文件中。每个建议项的名称空间都显示在建议列表的右侧。一旦选择了非导入类型，就会自动添加所需的名称空间导入指令。

您可以连续按几次导入符号完成快捷方式，以更改生成完成建议的方式。有关更多信息，请参见Double Completion 。

默认情况下，C＃和VB.NET中的导入符号完成项目包括在自动和基本完成的建议列表中。如果要更改默认行为，请使用“ 环境” | “ 环境”中的相应控件。 智能感知 ReSharper选项的“ 完成行为”页面。

调用导入符号完成

  1. 将插入符号放置在您要键入代码的位置。

  2. 按Ctrl+Alt+Space 。

  3. 从列表中选择一个建议，然后按Enter或开始键入标识符的首字母或其CamelHumps缩写，以缩小建议列表的范围。

    如有必要，可以通过清除“ 环境” |“ 标识符之间的匹配中间”复选框来禁用CamelHumps 。智能感知 ReSharper选项的“ 完成行为”页面。

您还可以在现有标识符上调用导入符号完成。在这种情况下，有两种方法可以应用所选的完成建议；按Enter插入并保留现有项目或按Tab替换现有项（如果需要，可以在ReSharper选项的Environment | IntelliSense | Completion Characters页面上更改默认快捷方式）。

从导入符号完成中排除项目

您可能希望不建议使用某些符号，例如，如果解决方案中的内容与系统类型相似，请说MyFramework.MyCollections.List ，但实际上并没有使用它。要将此类符号从建议中排除 ，请将其添加到“ 从导入和完成中排除”列表中 代码检查|设置|的自动导入页面 ReSharper选项（ Alt+R,,O ）。 条目的格式为Fully.Qualified.Name ， Fully.Qualified.Name.Prefix* ， 要么*Fully.Qualified.Name.Suffix 。通用类型指定为List`1 。
