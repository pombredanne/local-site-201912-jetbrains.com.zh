剃刀的预定义实时模板

ReSharper |模板资源管理器|实时模板|剃刀

本主题列出了ReSharper 2019.2中Razor的所有预定义实时模板。有关实时模板的更多信息，请参见使用实时模板创建源代码。

模板

细节

ctx

当前文件上下文

范围无处不在

身体

$ CTX $

参量

  • CTX -提供描述当前上下文的项目列表。这包括文件名，包含类型名，名称空间名等（ context ）。

nguid

插入新的GUID

范围无处不在

身体

$ GUID $

参量

  • GUID -生成新的全局唯一标识符（GUID）（ guid ）

创建脚本标签

范围 HTML和类似标签

身体

参量

  • attributeValue -当前html标记属性的建议属性值（ proposalAttributeValue ）

  • END -应用模板后的插入符位置。

t

创建标签

范围 HTML标签，Razor标签

身体

<$ TAG $> $ END $

参量

  • TAG -在当前文档中使用的建议标签名称（ proposalTagName ）

  • END -应用模板后的插入符位置。

tc

创建封闭标签

范围 HTML标签，Razor标签

身体

<$ TAG $ $ END $ />

参量

  • TAG -在当前文档中使用的建议标签名称（ proposalTagName ）

  • END -应用模板后的插入符位置。

a

创建标签属性

范围 HTML和类似属性

身体

$ ATTR $ =“ $ END $”

参量

  • ATTR -建议在当前文档的相同标签中使用的属性名称（ proposalAttributeNameByTag ）

  • END -应用模板后的插入符位置。

@inherits

剃刀继承指令

Scope Razor CSharp的标签

身体

@inherits $ TYPENAME $ $ END $

参量

  • TYPENAME -此时预期的猜测类型（ guessExpectedType ）

  • END -应用模板后的插入符位置。

@model

剃刀型号指令

Scope Razor CSharp的标签

身体

@model $ TYPENAME $ $ END $

参量

  • TYPENAME -在评估变量的位置显示类型完成列表（ completeType ）

  • END -应用模板后的插入符位置。

@section

剃刀区

Scope Razor CSharp的标签

身体

@section $ section $ {$ END $}

参量

  • section -在评估变量的位置显示基本代码完成列表（ 完整 ）

  • END -应用模板后的插入符位置。
