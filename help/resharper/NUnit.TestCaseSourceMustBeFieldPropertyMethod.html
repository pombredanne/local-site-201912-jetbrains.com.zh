代码检查：NUnit。测试用例源必须仅是字段，属性或方法

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

NUnit的TestCaseSource和ValueSource属性要求数据源成员为static 字段，属性或method 。

[TestFixture]公共密封类TestCaseSourceTest {静态IEnumerable _fieldSource = new [] {1,2,3};静态IEnumerable PropertySource => new [] {4，5，6};静态IEnumerable MethodSource（）=> new [] {7，8，9};公共事件UnhandledExceptionEventHandler OnError {添加=>抛出新的NotImplementedException（）;删除=>抛出新的NotImplementedException（）; } [TestCaseSource（nameof（_fieldSource））] //好的[TestCaseSource（nameof（PropertySource））] //好的[TestCaseSource（nameof（MethodSource））] //好的[TestCaseSource（nameof（OnError））] //警告：期望的字段/属性/方法public void Test1（int x）{控制台。WriteLine（x）; }}
