命名方式

从2018.3版本开始，ReSharper可以从现有代码中自动学习C＃符号命名规则 。如果要手动配置命名规则，请清除“ 使用自动检测到的规则”复选框 代码编辑| C＃|的命名样式页面 ReSharper选项（ Alt+R,,O ）。

ReSharper可帮助您为代码中的符号定义，控制和应用命名样式。命名样式是作为一组规则实现的，每个规则都针对具有一组约束的特定标识符（例如，一条规则可以针对静态私有只读字段）。每个规则可以具有一个或多个关联的样式，这些样式定义后缀，前缀，复合词的大写形式，等等。

ReSharper时会考虑这些规则产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。ReSharper还可以帮助您检测和修复违反命名规则的情况。如有必要，可以配置或禁用自动检查命名规则。

配置命名规则

开箱即用，ReSharper为每种类型的标识符提供了命名规则。这些规则基于Microsoft命名准则 ， .NET Foundation编码准则和最佳实践。
请注意，ReSharper命名规则默认值与上述任何准则之间都没有严格的对应关系。

如果您的个人喜好或公司标准与ReSharper的默认设置不同，则可以灵活地配置命名方式：对于每种标识符，您可以选择大写规则，前缀和后缀，不同访问权限的变体，保留的缩写等等。 。

您的命名风格首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

修改默认命名规则

  1. 请执行以下任一操作：

    • 按Alt+Enter在名称不一致的突出显示符号上，然后选择检查“名称不一致” |更改命名规则“ [规则名称]”的设置 。

    • 在主菜单中，选择“ ReSharper |选项|代码编辑| [语言] |命名样式 ，选择所需的规则，然后单击编辑 或双击它。

  2. 在打开的“ 编辑规则设置”对话框中，检查规则的现有样式。

  3. 如果现有样式可以接受，但是您要允许该规则使用其他样式，请点击添加 。当单个规则有多种样式时，如果相应的符号名称至少与这些样式之一匹配，则ReSharper不会检测到代码样式冲突。否则，ReSharper会检测到违反代码样式的行为，并建议根据默认样式（列表中的第一个样式）重命名符号。要更改默认样式，请使用上移 然后向下移动 纽扣。

  4. 要配置特定样式，请在列表中选择它，并配置前缀，后缀，然后选择一个命名样式。

  5. 要删除不必要的样式，请在列表中选择它，然后单击“ 删除”。 。

  6. 如有必要，您可以禁用与该规则关联的代码样式违规检测 ，从而使ReSharper能够检测其他代码样式违规。为此，请清除“ 启用检查”复选框。

  7. 单击设置以关闭“ 编辑规则设置”对话框。

  8. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

如果现有规则未涵盖命名约定的所有方面，则可以创建自定义规则。例如，您可能希望枚举具有大写的驼峰式大写字母和“ E”前缀。

自定义命名样式将覆盖内置样式。如果您的自定义命名样式针对的标识符已经具有内置样式的命名规则，则来自自定义样式的规则将应用于这些标识符。

创建自定义命名规则

  1. 在主菜单中，选择“ ReSharper |选项|代码编辑| C＃|命名样式 ，然后单击高级设置 。

  2. 在打开的“ 高级命名设置”对话框中，单击“ 添加”。 创建自定义命名规则。

  3. 在出现的“ 编辑扩展命名规则”对话框中，配置新规则：在“ 规则描述”字段中为新规则指定名称，在“ 受影响的实体”区域中选择目标标识符，使用“ 访问权限”和“ 静态/非静态”区域进一步限制规则的范围。

  4. 在“ 命名样式”部分中，按照上面的过程中所述，为规则配置一种或多种样式。

  5. 单击“ 确定”关闭“ 编辑扩展命名规则”对话框，然后再次单击“确定”关闭“ 高级命名设置”对话框。

  6. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

检测并修复违反命名规则的情况

ReSharper使用“不一致命名”代码检查来检测代码中是否违反命名规则。默认情况下，此检查始终在设计时代码检查期间应用，并在编辑器中突出显示检测到的违规作为警告 。例如，根据默认样式，接口名称应带有“ I”前缀。如果接口名称与此规则不匹配，则将鼠标悬停在该名称上时，ReSharper会突出显示该名称并通知您有关该问题的信息：

如果要查找超出当前文件的命名风格冲突，则可以在项目或整个解决方案中运行“不一致的命名”检查。

查找特定范围内的命名风格违规

  1. 在编辑器中，将插入符号设置为命名不一致的突出显示符号。

  2. 按Alt+Enter或单击插入符号左侧的动作指示器以打开动作列表 。

  3. 选择检验“不一致的命名” |在文件中找到类似的问题，然后选择所需的范围。

  4. 找到的所有违反命名方式的内容都会显示在“ 检查结果”窗口中 。

  5. 您可以双击检测到的问题以导航到相应符号的代码。

对于编辑器中突出显示的大多数命名风格冲突，ReSharper建议使用名称一致的快速修复程序 。你可以按Alt+Enter然后在操作列表中选择“ 重命名为[符合名称] ”。

也可以使用范围内修复功能在当前文件，项目或解决方案中执行自动更正命名样式冲突。

但是，如果具有计算出的一致名称的符号已存在于同一名称空间中，则不建议使用快速修复。您可以使用Rename refactoring修复此类符号的命名 。

配置和禁用自动命名规则检查

如果您不希望ReSharper在现有代码库中检查命名样式，则可以禁用对此负责的代码检查，或禁用对特定规则的自动检查。但是，即使禁用了自动检查，当ReSharper帮助您使用代码完成，生成，重构和模板来生成新代码时，也会遵守所有命名规则。

禁用自动检查特定命名规则

    • 在主菜单中，选择“ ReSharper |选项|代码编辑| [语言] |命名样式 ，选择所需的规则，然后单击编辑 或双击它。

  2. 在打开的“ 编辑规则设置”对话框中，清除“ 启用检查”复选框。

使用现有代码中的C＃命名规则

如果您从公共仓库中检出解决方案，或暂时加入建立了不同代码命名风格的团队，则可能要遵循此解决方案中的命名规则，但同时不要更改全局配置的命名规则。

ReSharper将自动处理此类情况-当您打开符号命名与全局设置不同的新解决方案时，它将检测与全局设置不同的命名规则，相应地修改这些规则，然后将修改后的规则保存到“解决方案”中。个人”设置层 （ .sln。解决方案旁边的DotSettings.user文件）。

如果在打开新解决方案时发生这种情况，ReSharper将在编辑器中显示一条通知：
