文件和类型布局

ReSharper可以根据不同的模式对C＃文件中的类型进行重新排序和键入成员。模式可以描述许多条件和约束，这些条件和约束在对文件中的项目重新排序和/或将它们包装在指定区域中时进行评估。

您的文件布局首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

布局模式简介

您可以配置多个布局模式以应用于不同的上下文（例如，您可以为类和接口使用不同的模式）。有两种类型的模式：

  • 文件模式 。适用于所有C＃文件。可以安排名称空间，类型和委托；将它们包装在区域中或组织成组，删除区域。类型的内容可以进一步用类型模式排列。请注意，一次只能应用一种文件模式。

  • 键入模式 。仅适用于与模式约束匹配的类型（如果有）。可以安排文件成员，将它们包装到区域中或按组组织，删除区域，排序等等。

在模式中，可以按所需顺序指定匹配项，也可以指定将匹配项包装/分组到的区域/组。

应用当前模式集的算法如下：

  1. 如果存在文件模式，则ReSharper会检查是否应删除文件中的区域，然后会检查模式中的每个匹配项是否与文件中的任何代码项匹配。

  2. 如果存在匹配条目，则它们在文件中的位置将根据文件模式中相应匹配项的位置进行更改。

  3. 如果同一代码项与多个匹配器条目匹配，那么将应用优先级更高或约束更强的匹配器。也就是说，如果有匹配的公共类型和匹配公共静态类型，那么所有的公共静态类型将由图案B匹配模式B模式A.要更改此设置，可以提高模式A的优先级。

  4. 如果存在具有相同约束集的匹配器条目，则根据第一个此类匹配器条目的位置移动匹配项。

  5. 如果指定了区域或组，则将匹配的项目相应地分组或包装在区域中。
    组或区域允许您为其指定优先级。例如，如果某个组/区域的优先级高于其外部的其他匹配器，则ReSharper首先处理该组/区域内部的匹配器，然后再处理其余的匹配器。组区域之间的唯一区别是该区域将匹配的项目包装到#region / #endregion 。

  6. 所有不匹配的内容都将移到匹配项之后。如果需要将不匹配的项目放在其他位置，则可以创建一个没有限制的匹配器，并在模式中为其设置所需的位置。

  7. 此后，将应用类型模式（如果有）。ReSharper会检查每种类型模式是否与文件中的任何类型匹配。如果某个类型有多个匹配模式，则与步骤3和步骤4类似地解决冲突。

  8. 建立类型匹配后，ReSharper会检查是否应删除类型中的区域，然后会检查模式中的每个匹配器条目是否与文件中的任何成员匹配。

  9. 如果存在匹配成员，则它们在类型中的位置会根据相应匹配项在类型样式中的位置而改变。

  10. 区域，组和不匹配成员的处理与步骤5和步骤6相似。

文件和类型布局首选项

开箱即用，ReSharper提供了几种目标类型的模式（例如，用于NUnit测试治具），以及其余C＃类型的默认模式。

默认的成员布局规则基于众多最佳实践，在大多数情况下都可以建议使用。但是，如果您的个人喜好或公司标准与ReSharper的默认设置不同，则可以基于默认设置配置代码重新排序模式，也可以针对特定情况创建新的模式。

对布局模式的所有修改都在 代码编辑| C＃|的“文件布局”页面 ReSharper选项（ Alt+R,,O ）。编辑完成后，单击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

编辑现有的布局图案

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择代码编辑| C＃|左侧的文件布局 。

  2. 点击页面左上角的模式来查看当前集中的所有模式。在此元素的属性中，可以使用“ 静态字段重新排序策略”选择器来定义是否以及如何使用初始化程序对静态字段重新排序。选项包括：

    • 严格 -如果可能会影响程序行为，则不会对静态字段进行重新排序。例如，如果带有初始化器的静态字段引用了同一类的静态成员，则该类的所有静态字段都不会重新排序。

    • 放宽 -禁止对在初始化程序中明确使用其他静态字段以及它们所引用的静态字段的静态字段进行重新排序。

    • 无限制 -始终允许重新排序（不建议）。

    所选行为将应用于所有模式。
    对于带有初始化程序的未重新排序的静态字段，ReSharper保留它们的相对顺序，同时仍将它们与其他声明一起重新排序。

  3. （可选）要加载默认模式集之一，请单击页面右上角的“ 加载 ”，然后选择模式集。

  4. 双击所需的图案，如果位于约束视图中，则将其切换到合成视图（单击图案标题右侧的图标以切换视图）。请注意，文件模式始终显示在合成视图中，也就是说，您不能为其指定任何约束。

  5. 在合成视图中，可以通过以下方式修改模式：

    • 要更改项目的顺序，请将它们拖到图案中。

    • 要删除特定项目，请选择它，然后按Delete 。请注意，如果要删除区域或组，其中的所有项目也将被删除。因此，您可能需要在删除区域/组之前将其拖出。

    • 要添加新项目，请将其从“ 工具箱”区域拖到图案中的所需位置。拖动区域或组以将其用作分组项，或拖动条目以匹配目标代码中的某些内容。您还可以使用Dependency属性匹配器来匹配和排列WPF依赖属性的所有部分。

    • 要配置模式或其中的项目的属性，请选择它并在右下部分中修改其属性：

      • 显示名称 -此属性可用于模式和匹配器条目，仅为了方便起见，应描述与之匹配的内容。

      • 名称 -区域的此属性指定创建区域时如何命名区域。
        您可以使用${0}如果根据“ 分组依据”选项生成区域，则在名称中输入参数以插入区域的序列号。
        您也可以使用命名参数在区域名称中插入分组选项。例如，您可以使用${Access}参数，将在相应区域的名称中替换为“ private”，“ public”等。显然，如果对区域的“分组依据”属性使用相同的值，则此参数有意义。此参数的有效值与“ Group By”属性的值相同，没有空格。也就是说，要在区域名称中插入实现接口的名称，请使用${ImplementsInterface}参数。

      • 删除区域 -文件和类型模式的此属性指定如何处理匹配代码中的现有区域。通过使用默认模式应用此功能，可以将“ 生成的例外”值仅用于删除以前创建的区域。

      • 优先级 -此属性设置匹配器条目的相对优先级。即，如果某个代码段被多个匹配器条目匹配，则根据具有最高优先级的条目的位置来移动它。

      • 排序依据-条目的此属性指定在多个匹配代码项的情况下的排序首选项。一旦选择了除None之外的其他排序首选项，就会出现Then By子句，该子句允许进一步优化排序顺序。

      • 分组依据-区域或组的此属性指定是否根据指定的分组生成区域/组。例如，如果选择“ 种类”作为值，则将为字段，属性和其他类型的成员（如果有）创建多个区域/组。一旦选择了除None以外的分组首选项，就会出现Then By子句，该子句允许进一步分离匹配的代码项。

  6. 要配置定义特定类型模式或匹配项如何匹配代码项的约束，我们切换到约束视图。要打开该视图的类型模式，请单击其标题上的相应图标。要打开此视图以显示匹配项，请双击它。

  7. 在约束视图中，所选项目表示为条件容器。要配置这些条件，请将项目从“ 工具箱”区域拖到已编辑的图案或条目中：

    • 应该先添加逻辑容器中的项目，因为它们定义了如何应用多个条件。这些容器的名称非常清楚地说明了这些容器内部条件之间的关系。如果添加不带逻辑容器的约束，则它们将按And容器分组。

    • “ 约束”组中的项目使您可以定义应该或不应该匹配的项目（如果包装在“ 不”容器中）。并非可以在Name约束的值中使用正则表达式。将项目从该组拖到逻辑容器中，并在右下部分配置其属性。

    • 从“ 修改器”组中拖动项目以指定目标项目应具有或不应当具有的修改器。

重新排序文件和类型成员

要应用文件并在所需范围内键入布局设置，请使用可以使用默认的默认值：完全清理配置文件运行代码清理 ，或者创建和运行仅针对您的特定任务的自定义配置文件 ，如下所述。

使用自定义代码清除配置文件应用文件和类型的布局

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 转至代码清理设置页面： 代码编辑|代码清理 。

  3. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 选定的配置文件设置”部分中，选中“ 应用文件布局”复选框。（可选）您可以在此配置文件中启用其他代码清除任务。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  5. 选择您要应用文件布局的范围：

    • 将插入符号设置在文件中的任何位置，以将文件布局应用于文件。

    • 在解决方案资源管理器中选择一项或多项，以在这些节点及其子项下的文件中应用文件布局。

  6. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  7. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  8. 点击运行 。ReSharper将在选定范围内应用文件布局。

如果要在不打开“ 代码清除”对话框的情况下应用文件布局，则可以将创建的配置文件绑定到静默清除并通过按运行它Ctrl+Shift+Alt+F 。您还可以创建一个自定义清除配置文件 ，该配置文件将应用文件布局与其他代码样式任务结合在一起。
