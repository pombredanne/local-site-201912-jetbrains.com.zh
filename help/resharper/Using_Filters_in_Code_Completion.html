在代码完成中使用过滤器

ReSharper允许您通过符号，访问修饰符等来过滤完成建议。您可以在每次调用代码完成功能时修改应用的过滤器集，和/或选择保留过滤器状态。

默认情况下，ReSharper在完成框底部显示过滤器栏。在此栏中，您可以查看过滤器的状态，然后单击图标以启用或禁用特定的过滤器。

过滤器栏中的过滤器图标取决于当前选择的符号图标方案 。

如有必要，您可以隐藏过滤器栏并在“ 环境” |“配置”中配置其他相关选项。 智能感知 ReSharper选项的“ 完成过滤器”页面。

过滤器的持久状态

默认情况下，每次调用代码完成都会重置您指定的完成过滤器。如有必要，您可以保留过滤器的最后修改状态。为此，请在“ 环境” | “ 环境”中选择“ 持久过滤器 ” 。智能感知 ReSharper选项的“ 完成过滤器”页面。

选中此复选框后，可以选择修改过滤器的默认状态。请注意，这些过滤器状态控件与完成弹出窗口中的过滤器栏同步。

过滤模式

可以以以下两种模式之一启用每个过滤器：包含或排除。

包容模式

如果在此模式下启用了过滤器，则完成列表仅显示与该过滤器匹配的项目。如果在此模式下启用了多个过滤器，则完成列表将显示与所有这些过滤器匹配的项目。

要在此模式下启用过滤器，请在过滤器图标上单击鼠标左键。

在过滤器栏上，此模式下的过滤器以统一的背景突出显示。

独占模式

如果在此模式下启用了过滤器，则完成列表仅显示与该过滤器匹配的项目。如果在此模式下启用了多个过滤器，则完成列表将显示与所有这些过滤器匹配的项目。

要在此模式下启用过滤器，请右键单击过滤器图标。

在过滤器栏上，此模式下的过滤器以边框突出显示。

自定义过滤器

ReSharper允许您定义自定义过滤器，您可以使用这些自定义过滤器按项目的组合件，名称空间和其他参数从完成建议中排除项目。

定义自定义完成过滤器

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择环境|智能感知左侧的完成过滤条件 。

  2. 确保选中“ 启用过滤器”复选框。

  3. 单击页面底部的“ 添加 ”，然后选择“ CLR符号过滤器”

  4. 在打开的“ 自定义过滤器配置”对话框中，指定过滤器详细信息：

    • 为符号指定一个或多个掩码。

    • （可选）选择一种要匹配的符号。

    • 选择是否匹配带有或不带有类型参数的类型。

  5. 单击确定以添加过滤器并关闭对话框。新的过滤器将显示在“ 自定义过滤器”区域中。如有必要，您可以稍后修改过滤器。

  6. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。
