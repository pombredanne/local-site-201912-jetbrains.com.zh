没有调试信息（PDB）的调试模块

使用ReSharper，您可以调试任何已编译的模块，即使它没有调试信息（PDB）。ReSharper将反编译该模块，为其生成PDB，并将该模块的反编译符号自动加载到Visual Studio调试器中。

您可以使用ReSharper导航功能，例如“ 随处搜索” Ctrl+N然后去声明 Ctrl+B在反编译的代码中找到类型和成员，并在那里设置断点。

先决条件

在调试没有PDB的模块之前，请检查以下Visual Studio设置（“ 工具” |“选项” ）：

  • 关于调试|在“符号选项”页面上，确保此目录中的“ 高速缓存”符号字段具有有效的磁盘路径值。

  • 关于调试|在“常规选项”页面上，确保已清除“ 仅启用我的代码”复选框。

但是，即使这两个设置中的一个或两个都不具有预期值，当您尝试从反编译模块中加载符号时，ReSharper也会建议自动更正这些值。在这种情况下，它将添加％LOCALAPPDATA％\ Temp \ SymbolCache作为符号缓存目录。

调试

在调试程序之前或调试开始之后，需要为要调试的每个已编译模块加载符号。您可以通过以下方式之一进行操作：

  • 在调试模式下，打开Visual Studio的“模块”窗口（“ 调试” |“ Windows” |“模块” ），选择几个模块之一，右键单击所选内容，然后选择“ 使用ReSharper Decompiler加载符号” 。

  • 在“ 进程资源管理器”窗口 （ ReSharper | Windows |进程资源管理器 ）中，选择多个托管模块之一或一个或多个进程，然后在上下文菜单中选择“ 启用调试 ”。如果选择流程，则ReSharper将为流程中的所有托管模块生成并加载不存在的PDB。

  • 在已编译的模块中导航到所需的符号，然后在反编译的源代码中设置一个断点。如果尚未为设置断点的模块加载符号，ReSharper将显示一个弹出窗口，您可以在其中启用此模块的调试：

反编译和加载符号可能需要一些时间，因此，以上述方式之一启动该过程后，ReSharper会在“ 项目/ PDB生成状态”窗口中显示进度。

在调试优化的程序集时 ，某些调试器功能不可用，例如，即使符号文件正确，您也不会看到局部变量的值。因此，您可能要禁用优化。

  • 当Visual Studio处于调试模式时，检查程序集是否已优化-打开“ 模块”窗口（“ 调试” |“ Windows” |“模块” ），然后检查“优化”列。

  • 选择Visual Studio选项（“ 工具” |“选项” |“调试” |“常规” ）中的“ 禁止模块加载上的JIT优化”复选框。

  • 要禁用所有NGEN（准JIT前）映像（ * .ni.dll ），可以设置环境变量COMPLUS_ZapDisable=1 。查看这些文章以获取示例和更多信息：
