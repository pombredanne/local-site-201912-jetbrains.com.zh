ReSharper 2019.2帮助

TSLint

ReSharper |选项|工具|网络短绒| TSLint

在ReSharper选项的此页面上，可以启用和配置TSLint linter，以便ReSharper可以使用linter规则在编辑器中验证和修复JavaScript TypeScript代码 。

项目

描述

一般
启用

使您能够启用TSLint验证。启用后，ReSharper将在指定位置使用linter和解决方案中的linter设置来验证.js和.ts文件中的代码。

选择TSLint软件包位置

如果TSLint软件包已安装在您的计算机上并在项目中使用，则ReSharper在大多数情况下会自动找到它。如果找不到，则可以手动指定其位置。无论如何，ReSharper都会显示是否找到了linter软件包。

组态
选择一种TSLint配置方式

在大多数情况下，ReSharper会在其典型位置自动找到TSLint配置文件。如有必要，您可以手动指定配置文件。

选件
Lint JavaScript文件

默认情况下，TSLint仅用于验证TypeScript文件。也可以使用此复选框来验证JavaScript文件。

其他TSLint选项

使用此字段可以传递任何其他用空格分隔的命令行选项 。
请注意，某些选项会影响lint的执行方式，因此无法应用。例如， --fix将被忽略，因为ReSharper提供了自己的方法来应用linter修复程序。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
