TypeScript中的代码样式帮助

ReSharper提供了许多功能来使您的代码保持整洁。它们与ReSharper的其他大多数功能紧密地交织在一起，它们可以帮助您生成代码并根据特定的代码样式更改现有的代码库。可以将代码样式（包括命名标准，格式规则和许多其他微小方面）配置到非常详细的级别，并在整个团队中共享。

您的代码样式首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

与其他语言相似，所有TypeScript代码样式首选项都可以使用“范围内固定”或“代码清除”功能 ，通过一个命令在所需范围内应用。

ReSharper在TypeScript中提供以下代码样式辅助功能：

命名方式

ReSharper可帮助您为代码中的符号定义，控制和应用命名样式。命名样式是作为一组规则实现的，每个规则都针对具有一组约束的特定标识符（例如，一条规则可以针对静态私有只读字段）。每个规则可以具有一个或多个关联的样式，这些样式定义后缀，前缀，复合词的大写形式，等等。

ReSharper时会考虑这些规则产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。ReSharper还可以帮助您检测和修复违反命名规则的情况。如有必要，可以配置或禁用自动检查命名规则。

有关配置和应用命名样式的更多信息，请参见命名样式。

管理和应用代码格式化规则

代码样式的一个重要方面是如何设置代码格式，即如何使用空格和空白行来排列和分隔代码块，是否以及如何使用制表符缩进，是否以及如何包装长行等。 。

ReSharper代码格式化规则集广泛，具有默认配置，该配置考虑了默认的Visual Studio格式化选项以及许多最佳实践。您可以配置格式设置规则的每个细节，并在代码中强制执行规则。这些规则适用于ReSharper 产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。格式设置规则也可以应用于当前选择，当前文件中的现有代码 ，或者适用于整个解决方案的更大范围。

ReSharper使用共享设置机制存储格式设置首选项。您可以在“ 代码编辑” |“选项”下的选项页中配置格式设置规则 。 打字稿|格式样式组。您还可以在Clang-Format文件的EditorConfig中存储和共享格式设置 。

作为浏览选项页面的替代方法，您可以选择一个代码块，然后按Alt+Enter然后选择格式|配置以配置适用于此块的格式设置规则，同时注意该块中的更改。

引号样式

TypeScript规范允许您同时使用单引号（'）和双引号（“）来处理字符串文字。在项目或解决方案中坚持相同的引用样式是一个好主意。为此， 您可以在“ 代码编辑” |“配置”中配置您的首选项以及相应代码检查的严重性级别 。 打字稿| ReSharper选项的“代码样式”页面。ReSharper将检测并突出显示与您的首选项不同的代码。要在现有代码库中强制执行此首选项， 使用默认配置文件“ 完全清理”或“ 重新格式化并应用代码样式”之一运行代码清理 ，或者在选择了相应选项的情况下创建并运行自定义配置文件 。

该首选项与JavaScript的首选项共享，因此，当您以一种语言进行更改时，另一种语言的首选项也会更改。

模块导入样式

从其他模块导入类型有多种方法：您可以添加模块别名或使用完全限定的类型名称。ReSharper可帮助您在所需范围内应用首选的类型导入方法。

应用首选的模块导入样式

  1. 使用“ 代码编辑” |“ 导入类型时 ，更喜欢添加模块别名来限定类型”复选框。 打字稿| ReSharper选项的“代码样式”页面可指定您的首选项。

  2. 使用“完全清除”配置文件在所需范围内运行代码清除 。

如有必要，您可以使用自定义配置文件来应用模块导入样式。为此，请创建一个自定义清除配置文件，然后在TypeScript特定的配置文件设置中选中“ 优化'导入'语句”复选框。

优化参考意见

ReSharper可帮助您检测和删除未使用的参考注释。要优化参考注释，请使用“完全清理”配置文件在所需范围内运行代码清理 ，或创建自定义清理配置文件，然后在TypeScript特定配置文件设置中选中“ 优化参考注释”复选框。

终止声明

如果该语句以换行符结尾或括号中只有一个语句，则JavaScript / TypeScript不需要分号来终止该语句。但是，通常建议在每个语句后使用分号（例如，请参见此StackOverflow线程 ）。ReSharper可以帮助您自动添加分号。

要终止特定文件，项目或解决方案中的语句，请使用“完全清除”配置文件或使用自定义配置文件在所需范围内运行代码清除 ，在JavaScript / TypeScript部分中选择了Terminate语句 。

排列修饰符

如果您希望使用或不使用显式的“公共”修饰符， 您可以在“ 代码编辑” |“配置”中配置您的首选项以及相应代码检查的严重性级别 。 打字稿| ReSharper选项的“代码样式”页面。ReSharper将检测并突出显示与您的首选项不同的代码。要在现有代码库中强制执行此首选项， 使用默认配置文件“ 完全清理”或“ 重新格式化并应用代码样式”之一运行代码清理 ，或者在选择了相应选项的情况下创建并运行自定义配置文件 。

隐式/显式键入

TypeScript规范允许隐式或显式指定类型，并为类型使用'any'关键字。使用ReSharper代码样式帮助，您可以定义是否以及何时应明确指定类型，以及是否允许隐式“ any”。

使用“ 代码编辑” | “ 类型”规范中的控件。 打字稿| ReSharper选项的“代码样式”页面可指定您的首选项。

也可以在项目设置中配置使用隐式“ any”的首选项。如果项目设置中不允许使用隐式“ any”类型，则它们始终会突出显示为错误，而与您的代码样式首选项无关。

要将隐式/显式键入首选项应用到特定文件，项目或解决方案中，请使用“完全清除”配置文件或自定义配置文件在所需范围内运行代码清除，在此配置文件中， 使用代码样式同步类型注释的存在并添加显式“如果需要 ，则在TypeScript特定部分中选中“任何”类型注释复选框。

ReSharper的代码检查也会考虑您的隐式/显式键入首选项。
