使用dotMemory单元监视内存

ReSharper的单元测试运行程序不允许运行两个或多个具有相同ID的测试。在实践中，这种情况很少发生，但是如果您确实有具有相同ID的测试，它们将在“单元测试”浏览器和“单元测试会话”窗口中显示为单个节点，并且仅运行其中一个。
请注意，测试框架以不同的方式识别测试。例如，MSTest使用测试方法FQN，而NUnit使用测试属性的属性作为测试ID。

如果您拥有ReSharper Ultimate许可证，则可以使用JetBrains dotMemory来分析应用程序的内存使用情况。您还可以使用JetBrains dotMemory Unit在单元测试中直接检查代码中是否存在各种内存问题。

启用dotMemory时， 参考dotMemory单元框架 命令与运行单元测试一起出现 / 调试单元测试 编辑器，解决方案资源管理器以及“ 单元测试会话”窗口的工具栏上的命令。您可以使用此命令自动安装JetBrains。DotMemoryUnit NuGet包，并在您的单元测试项目中引用它。

即使没有dotMemory，也可以手动安装JetBrains。DotMemoryUnit NuGet包，并在您的单元测试项目中引用它。

一旦您引用了dotMemory Unit Testing Framework，ReSharper将在dotMemory Unit下添加Run。 （ Ctrl+T,M ）执行其他单元测试操作，这样您就可以使单元测试通过或失败，具体取决于内存流量，内存中特定对象的存在以及其他因素。有关更多信息，请参见单元测试和内存配置文件：它们可以结合使用吗？

您可以通过清除上的dotMemory复选框来禁用dotMemory 环境|的产品和功能页面 ReSharper选项（ Alt+R,,O ）。
