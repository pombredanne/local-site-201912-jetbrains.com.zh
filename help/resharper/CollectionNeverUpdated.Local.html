Code Inspection: Collection is never updated (private accessibility)

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues
  • change its severity level to make the detected issues less or more noticeable

ReSharper analyzes access to collections created in your code and warns you if a collection is never filled or modified even if there is read access to this collection. For example:

public void Bar() { var col = new List<string>(); Console.WriteLine(col[0]); }

All standard collection types are analyzed this way by default. If necessary, you can also apply this analysis to your custom collections that derive from IEnumerable. In this case, you need to tell ReSharper how invocations of collectionâ€™s public methods will affect the contents of the collection. To do so, mark the methods of your collection class with the [CollectionAccessAttribute], which should have one of four flags: None, Read, ModifyExisitingContent, UpdatedContent (the difference between the last two flags is that UpdatedContent means that the marked function can also add new elements to the collection).
