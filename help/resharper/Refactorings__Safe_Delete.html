安全删除重构

ReSharper |重构|安全删除…
Alt+Delete
ReSharper_SafeDelete

如果要删除任何其他符号或项目/装配体引用，请使用此重构来确保解决方案在删除后可以编译。如果没有对已删除项目的引用，或者所有引用都可以安全折叠，则将立即删除该项目。否则，ReSharper将为您显示所有不安全删除的引用，使您可以编辑相应的代码。如果删除符号后还有任何空文件，则重构也可以删除这些文件。

如果只需要删除未使用的引用，请使用“ 删除未使用的引用”重构或“ 优化引用”命令。

安全删除符号

  1. 通过以下方式之一选择符号：

    • 在编辑器中，将插入符号设置为符号名称。

    • 在解决方案资源管理器中选择一个符号。

    • 在“ 文件结构”窗口中选择一个符号。

    • 在“类视图”中选择一个符号。

    • 在对象浏览器中选择一个符号。

    • 在类型依赖关系图中选择一个符号。

  2. 请执行以下任一操作：

    • 按Alt+Delete 。

    • 按Ctrl+Shift+R然后选择“ 安全删除”

    • 右键单击并选择“ 重构”。在快捷菜单中安全删除 。

    • 选择 ReSharper |重构|主菜单中的“ 安全删除” 。

    “ 安全删除”对话框将打开。

  3. 如果该符号具有基本符号或继承者或两者都有，则ReSharper会提示您指定应删除的符号。
    如果是类型，则可以删除该类型及其所有继承者。
    如果是多态方法或属性，则还可以通过选择以下选项之一来删除基本成员：

    • 删除整个层次结构：所选成员，所有基础符号及其所有继承者

    • 删除所选成员及其所有继承者

    • 删除层次结构中的单个成员，即选定的成员

  4. 要应用重构，请单击下一步 。

  5. 如果未找到用法，该符号将立即删除。否则，检测到的冲突将显示在重构向导的下一个陡峭区域中，您将需要手动解决冲突 。

安全删除项目/装配体参考

  1. 在解决方案资源管理器中，选择一个或多个要删除的引用。

  2. 请执行以下任一操作：

    • 按Alt+Delete 。

    • 按Ctrl+Shift+R然后选择“ 安全删除”

    • 右键单击并选择“ 重构”。在快捷菜单中安全删除 。

    • 选择 ReSharper |重构|主菜单中的“ 安全删除” 。

  3. 如果仅选择了未使用的引用，则将立即执行重构。否则，将出现“ 安全删除装配体引用”对话框，其中所有项目/装配体中符号的使用都将被删除。您可以单击每种用法以导航到源代码，然后查看是否可以将其删除。更改代码后，对话框中的用法列表会自动更新。

  4. 对话框中没有更多用法后，可以单击“ 下一步”完成删除。

  5. 如果要删除选定的程序集引用并在以后取消使用 ，请清除仅删除未使用的引用 ，然后单击下一步 。

如果您的项目引用隐式加载的程序集，则可以在启动之前将其从清理中排除

如果您撤消对引用的删除，请注意，源代码中的所有更改都将被还原，但不会添加回引用。

还有两种清理项目和装配体参考的方法。

  • 您可以调用“ 优化引用”命令，该命令将使您分析所有引用并删除未使用的引用。

  • 您可以在项目上或“解决方案资源管理器”中的“ 引用”节点上调用“ 删除未使用的引用”命令。
