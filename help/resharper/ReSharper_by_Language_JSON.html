JSON值辅助工具

ReSharper允许您将某些JSON属性标记为特定的值类型（例如正则表达式，文件路径），以便ReSharper可以应用特定的验证并提供特定的完成建议以匹配JSON文件中的属性。

您可以在“ 代码编辑” |“添加”中为特定属性添加值帮助器。 JSON | ReSharper选项的JSON值帮助器页面。匹配基于文件名掩码和/或架构以及JSON属性的路径（例如/compilerOptions/rootDir ）。

您的JSON值帮助程序首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保留在VCS下，并自动与团队成员共享。

使ReSharper忽略特定代码

ReSharper允许您以不同的方式忽略特定的文件，文件夹和文件掩码。您可以在以下选项页面上配置忽略的文件：

  • 代码检查|从代码检查中排除文件，文件夹和掩码的设置 。

  • 代码检查|生成的代码可以部分禁用对生成的代码的代码检查 。

  • 环境|搜索和导航以指定“ 按名称搜索”命令应忽略的文件和文件夹。

  • 代码编辑|要添加的第三方代码 C ++，JavaScript，TypeScript，CSS，HTML和JSON文件，文件夹和通配符将被视为“跳过”或“库”。ReSharper将完全忽略“已跳过”的文件，并将“库”文件视为只读文件-为导航而建立索引，但不进行检查 ， 快速修复和重构 。
