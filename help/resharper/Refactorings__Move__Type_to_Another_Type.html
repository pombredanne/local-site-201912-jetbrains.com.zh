移至其他类型重构

ReSharper |重构|移动…
F6
ReSharper_Move

此重构可帮助您将嵌套类型和静态成员移动到另一种类型（现有的或新的）。所有参考和用法都会自动更新，包括成员使用其原始类型的其他成员的方案。

要将实例方法移动到另一种类型，请使用Move Instance Method refactoring 。

用命令调用重构

将静态成员或内部类型移动到另一种类型

  1. 通过以下方式之一选择嵌套类型或静态成员：

    • 在编辑器中，将插入符号设置为嵌套类型的名称。

    • 在解决方案资源管理器中选择一个嵌套类型。

    • 在“ 文件结构”窗口中选择一个嵌套类型。

    • 在“类视图”中选择一个嵌套类型。

    • 在对象浏览器中选择一个嵌套类型。

    • 在类型依赖关系图中选择一个嵌套类型。

  2. 请执行以下任一操作：

    • 按F6然后选择“ 移至另一种类型”

    • 按Ctrl+Shift+R然后选择“ 移至另一种类型”

    • 右键单击并选择“ 重构”。在上下文菜单中移至另一种类型 。

    • 选择 ReSharper |重构|在主菜单中移动...。

    将打开“ 移至另一种类型”对话框。

  3. 开始输入目标类型的名称。如果要将项目移动到现有类型，请从下拉列表中选择它，该列表显示解决方案中与您输入匹配的所有类型。

  4. 如有必要，可以将其他嵌套类型和包含类型的静态成员移动到另一种类型。为此，请在“ 要移动的静态成员”区域中选择所需的项目。

  5. （可选）为新类型的mvoed符号指定访问修饰符。

  6. 要应用重构，请单击下一步 。

  7. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。

就地执行重构

您可以通过剪切嵌套类型或静态成员并将其粘贴到其他类型中来应用重构。粘贴项目后，它周围会出现一个灰色边框，并显示一个弹出窗口，通知您重构可用。您可以单击建议或按Alt+Enter调用解决方案范围的重构：
