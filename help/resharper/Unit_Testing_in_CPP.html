C ++中的单元测试协助

ReSharper C ++提供了C ++支持， ReSharper C ++是一种专用产品，可以单独安装，也可以与ReSharper或ReSharper Ultimate并排安装。

ReSharper帮助发现，运行和调试Google C ++测试框架 ， C ++测试框架，Catch和Boost的单元测试。测试框架（v。1.59及更高版本）和doctest单元测试框架都在Visual Studio中提供。

当您调试64位C ++测试二进制文件时，ReSharper无法读取进程输出，并且测试状态不确定。

使用ReSharper，您可以执行单个单元测试，所有测试都在文件，项目或解决方案中。您也可以在测试会话中执行任意数量的测试。

您可以通过清除下面的“ 单元测试”复选框来禁用单元测试辅助功能。 环境|的产品和功能页面 ReSharper选项（ Alt+R,,O ）。

在当前文档中发现测试

ReSharper会在编辑器中立即发现单元测试，并在编辑器中的每个项目旁边添加相应的操作指示符 ：

如果您使用Google Mock框架创建模拟类，那么ReSharper将帮助您生成Google Mock方法 。

  • —在上一次执行期间通过的单元测试。

  • —在最后一次执行期间，单元测试失败。

在当前文档中执行测试

当前文档中有几种运行或调试单元测试的方法。您可以使用操作指示器，主菜单或快捷方式：

  • 要运行或调试单个测试或测试类中的所有测试，请单击其旁边的动作指示器或在测试中设置插入符号，然后按Alt+Enter 。在操作列表中 ，为测试类选择“运行 / 调试 ”。

  • 或者，您可以使用 运行单元测试 / 调试单元测试命令，这些命令在主菜单（ ReSharper |单元测试 ），上下文菜单中或通过Ctrl+T,R / Ctrl+T,D相应的快捷方式。这些命令的作用方式取决于插入符的位置或在编辑器中的选择：

    • 要运行或调试单个测试，请将插入号设置为测试名称，或在编辑器中其声明内的任何位置。

    • 要运行多个测试，请在编辑器中选择所需的测试。

无论选择哪种方式运行或调试测试，您都将在“ 单元测试会话”窗口中看到执行进度，结果和输出。 如果存在开放的单元测试会话 ，则将已执行的测试添加到该会话中。如果没有测试会话或现有会话被锁定 ，则会创建一个新的测试会话。

如有必要，您始终可以通过按来重新运行上次执行的测试Ctrl+T,T或选择 ReSharper |单元测试|在菜单中重复上一个运行 。

在解决方案中发现单元测试

ReSharper将“ 单元测试资源管理器”窗口添加到Visual Studio（ ReSharper |单元测试|单元测试，或ReSharper | Windows |单元测试 ，或Ctrl+Alt+T ）。 使用此窗口，您可以浏览和运行或调试整个解决方案中的单元测试。请注意，来自项目的单元测试仅在构建项目后出现在窗口中。当前打开的文件中的测试会自动更新，创建后的新文件会立即显示在单元测试浏览器中。

单元测试浏览器允许您执行以下操作：

  • 探索解决方案中的测试：在列表或树视图中浏览所有单元测试，搜索测试并按子字符串过滤，按项目，名称空间重新组合单元测试，等等。

  • 通过在视图中双击来导航到任何测试的源代码。

  • 运行或调试选定的测试。

  • 从所选测试创建单元测试会话 ，和/或将所选项目添加到当前测试会话。

  • 将所有测试从解决方案导出到文本，XML或HTML文件。

在项目或解决方案中执行单元测试

您可以从单元测试资源管理器，解决方案资源管理器或类视图运行或调试测试。单元测试资源管理器使您可以仅查看测试，而使用其他窗口则需要知道哪些项目，文件和类包含测试。

  • 要从Unit Test Explorer执行测试，请选择所需的测试，然后单击运行单元测试 / 调试单元测试 在工具栏上，或使用相应的快捷方式（ Ctrl+T,R / Ctrl+T,D ）。

  • 要从解决方案资源管理器或类视图运行或调试测试，请选择包含测试的一项或多项（文件，文件夹，项目），然后使用 运行单元测试 / 调试单元测试命令，这些命令在主菜单（ ReSharper |单元测试 ），上下文菜单中或通过Ctrl+T,R / Ctrl+T,D相应的快捷方式。

  • 要运行或调试解决方案中的所有测试，请选择 ReSharper |单元测试|从主菜单中的“ 从解决方案运行所有测试”或按Ctrl+T,L 。

无论选择哪种方式运行或调试测试，您都将在“ 单元测试会话”窗口中看到执行进度，结果和输出。 如果存在开放的单元测试会话 ，则将已执行的测试添加到该会话中。如果没有测试会话或现有会话被锁定 ，则会创建一个新的测试会话。

如有必要，您始终可以通过按来重新运行上次执行的测试Ctrl+T,T或选择 ReSharper |单元测试|在菜单中重复上一个运行 。

使用单元测试会话

ReSharper允许您将针对应用程序特定部分的单元测试分组为单元测试会话。单元测试会话可以包含来自不同项目的测试。您可以有多个测试会话，并根据需要分别运行。一个测试可以包含在几个不同的测试会话中。

有关与单元测试会话有关的详细说明，请参阅《 使用单元测试会话》 。

执行过程

ReSharper提供了几种执行单元测试的方法。无论选择哪种方式，执行进度，测试结果和输出都将显示在“单元测试会话”窗口中，而覆盖结果将显示在“ 单元测试覆盖率”窗口中。

在单元测试会话中运行测试时，执行进度将显示在“单元测试会话”窗口和“ 停止执行 ”工具栏下的状态栏中。 按钮变为活动状态。您可以单击此按钮来中止测试的执行。您可以同时运行多个单元测试会话。但是，调试测试时，一次只能执行一个测试会话。

如有必要，您可以启用“ 跟踪运行测试” 工具栏上的选项。如果已打开，则测试树中的选择将自动切换到当前正在运行的测试，并且输出面板在执行期间始终显示当前测试的输出。

您还可以在运行测试时启用自动滚动输出 在测试运行时跟踪其输出的选项。

在运行或调试单元测试时，ReSharper使用项目属性中指定的Command和Working Directory配置参数。要访问这些属性，请在项目上单击鼠标右键，然后选择“ 项目属性” |“项目”。配置属性|调试 。

默认情况下，不使用“命令参数”属性（“ 调试” |“本地Windows调试器” |“命令参数” ）。要在测试执行中使用它，请选中相应的复选框 工具|单元测试|测试框架|的C ++测试页面 ReSharper选项（ Alt+R,,O ）。

如果未在项目属性中指定执行设置（例如，在库项目，“打开文件夹”或CMake项目中），则可以在“ 工具” | “ 工具”中定义它们。 单元测试|测试框架| ReSharper选项的“ C ++测试”页面。

配置测试运行器

单元测试执行变量，例如命令，其参数，工作目录和环境变量通常在项目属性（ Project Properties | Configuration Properties | Debugging ）中指定，ReSharper将从那里获取它们。
但是，如果未在此处指定这些属性（例如，在库项目，“打开文件夹”或CMake项目中），则可以在以下位置定义执行设置 工具|单元测试|测试框架|的C ++测试页面 ReSharper选项（ Alt+R,,O ）。

您也可以在该页面上使用“ 合并”环境标志。如果选择它，则ReSharper会将“ 环境”字段中指定的环境变量添加到Visual Studio中的环境变量中。如果未选择，则将仅使用Visual Studio中的环境变量。
