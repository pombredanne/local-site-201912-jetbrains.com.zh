使用驼峰

为以下操作启用CamelHumps ：

  • 扩展/缩小选择 （ Ctrl+W / Ctrl+Shift+W ）

  • 将选择范围扩展到下一个/上一个词 （ Ctrl+Shift+Right,Arrow / Ctrl+Shift+Left,Arrow ）

  • 转到下一个/上一个词 （ Ctrl+Left,Arrow / Ctrl+Right,Arrow ）

  • 从单词开头到插入符号删除文本 Ctrl+Backspace
  • 从插入符号删除单词末尾的文本 Ctrl+Delete

如果选中此复选框，则在使用这些操作时将考虑组成CamelHumped名称的单词。

此复选框对CamelHumps在代码完成（IntelliSense）或“ 导航和搜索”命令中的工作方式没有任何影响。

自动格式化分号

如果选中此复选框，则在键入分号时，ReSharper会将代码格式设置规则应用于该语句。

右括号自动格式化

如果选中此复选框，则在键入代码的右大括号时，ReSharper会将代码格式设置规则应用于代码块。

在粘贴时自动格式化

通过此选择器，您可以配置是否对粘贴的代码应用代码格式化规则 。默认情况下，ReSharper仅在粘贴代码时应用缩进设置，如果您不想非常严格地遵循格式设置规则，这是一个方便的选择。

即使启用了自动格式设置或粘贴时自动缩进，也可以在不重新格式化的情况下粘贴代码：按Ctrl+Z粘贴后立即进行，仅格式化将被撤消。

请注意，默认情况下，Visual Studio对编辑和粘贴的代码应用其自己的格式设置规则。如果您使用的是ReSharper自动格式化功能，建议您禁用Visual Studio自动格式化。您可以在Visual Studio选项（“ 工具” |“选项” ）中进行操作。例如，对于C＃，请清除“ 文本编辑器” | “ 文本编辑器”上的相应复选框。 C＃|代码样式|格式|常规选项页面。

纠正特定于通用语言的拼写错误

如果选中此复选框，则ReSharper会自动修复键入错误的字符- 逐字字符串内插中的@ $ 或括号 内的 点或分号 。
例如，如果您希望指定不带前导零的双精度值，则可能需要清除此复选框。 DoSomething(.33); 。

输入时智能缩进

如果选中此复选框，则每次在编辑器中按Enter时，ReSharper都会考虑影响缩进的所有设置（例如，是否以及如何使用制表符（可在“代码编辑” |“通用格式化程序样式”上配置），计算下一行的缩进。 ），是否对齐链式方法调用（可在“代码编辑| C＃|格式设置|换行和包装”中配置 ），依此类推。
如果清除此复选框，则下一行将根据Visual Studio设置进行对齐。

在输入时插入虚拟缩进

如果启用此选项，则在按时，ReSharper不会插入缩进字符（可以是空格，制表符或两者的组合，具体取决于设置） Enter 。但是，它将根据为此行计算的缩进量设置插入符号。如果您在此行中键入任何内容，则会添加缩进字符；如果按Enter再一次，将不留尾随空格。
如果您希望缩进制表符，与空格对齐，则可能要禁用此选项。

此功能当前依靠Visual Studio插入实际的缩进字符，因此它不支持“缩进制表符，与空格对齐”模式。

在/ *后的Enter上插入*

如果启用此选项， ReSharper将插入星号*按下时，在块注释的每一行的开头Enter 。 即使启用此选项，也可以按Shift+Enter添加没有前导星号的新行。

在选项卡上使用ReSharper格式化程序设置

启用此选项后，ReSharper将使用自动检测到的缩进设置和基于 ReSharper 图层的设置来决定在按下时是在缩进中使用制表符还是空格Tab 。
我们建议您将此选项保持启用状态。

不退格

此选项定义了Backspace当在空白行上按下它或插入符号的左侧仅包含空格和制表符时，可以使用此键。可用的选项有：

  • 禁用 —一次将插入符返回一个位置。

  • 到最接近的缩进位置 —将插入号返回到当前缩进大小的倍数的最接近位置。

  • 到适当的缩进位置 （默认）—根据代码结构将插入号返回到预期的缩进位置。
