C ++中的代码样式帮助

ReSharper C ++提供了C ++支持， ReSharper C ++是一种专用产品，可以单独安装，也可以与ReSharper或ReSharper Ultimate并排安装。

ReSharper提供了许多功能来使您的代码保持整洁。它们与ReSharper的其他大多数功能紧密地交织在一起，它们可以帮助您生成代码并根据特定的代码样式更改现有的代码库。可以将代码样式（包括命名标准，格式规则和许多其他微小方面）配置到非常详细的级别，并在整个团队中共享。

与其他语言类似，所有C ++代码样式首选项都可以使用“范围内固定”或“代码清除”功能 ，通过一个命令在所需范围内应用。

您的代码样式首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

管理和应用代码格式化规则

代码样式的一个重要方面是如何设置代码格式，即如何使用空格和空白行来排列和分隔代码块，是否以及如何使用制表符缩进，是否以及如何包装长行等。 。

ReSharper代码格式化规则集广泛，具有默认配置，该配置考虑了默认的Visual Studio格式化选项以及许多最佳实践。您可以配置格式设置规则的每个细节，并在代码中强制执行规则。这些规则适用于ReSharper 产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。格式设置规则也可以应用于当前选择，当前文件中的现有代码 ，或者适用于整个解决方案的更大范围。

ReSharper使用共享设置机制存储格式设置首选项。您可以在“ 代码编辑” |“选项”下的选项页中配置格式设置规则 。 C ++ |格式样式组。您还可以在Clang-Format文件的EditorConfig中存储和共享格式设置 。

作为浏览选项页面的替代方法，您可以选择一个代码块，然后按Alt+Enter然后选择格式|配置以配置适用于此块的格式设置规则，同时注意该块中的更改。

调整单个格式首选项的另一种方法是从现有代码示例中学习格式规则，该代码示例可以是选定的块，当前文件，一组文件，一个或多个项目或整个解决方案。ReSharper将分析选定的样本并列出与当前设置不同的格式规则。然后，您将能够查看检测到的规则，根据需要对其进行更改，并将其保存为所需的规则。
有关更多信息，请参见使用现有代码中的格式设置规则

命名方式

ReSharper可帮助您为代码中的符号定义，控制和应用命名样式。命名样式是作为一组规则实现的，每个规则都针对具有一组约束的特定标识符。每个规则可以具有一个或多个关联的样式，这些样式定义后缀，前缀，复合词的大写形式，等等。

ReSharper时会考虑这些规则产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。ReSharper还可以帮助您检测和修复违反命名规则的情况。如有必要，可以配置或禁用自动检查命名规则。

上 代码编辑| C ++ |的命名样式页面 ReSharper选项（ Alt+R,,O ）， 您可以将规则集重置为预定义的命名样式方案之一（STL，Google，LLVM，CamelCase，Microsoft）。您可以按原样使用这些方案，也可以根据预定义的规则集配置自己的规则集 。

安排简历限定词

ReSharper帮助您在生成的代码中排列简历限定符 。关于代码编辑C ++ | ReSharper选项的“代码样式”页面上，您可以指定将cv限定词放在类型说明符之前还是之后，以及const和volatile修饰符。

包含指令样式

关于代码编辑C ++ |在ReSharper选项的“代码样式”页面上，您可以在生成的代码的 include指令中指定使用正斜杠还是后斜杠作为路径分隔符。您还可以指定是否在头文件中生成前向声明。

默认指针初始化器样式

您可以指定0 ， nullptr要么NULL作为指针初始化程序的首选初始化程序，例如： const char *foo = nullptr 。您可以在“ 代码编辑” | “ 代码编辑”中更改首选项。 C ++ | ReSharper选项的“代码样式”页面。

文件头样式

将文件标题用于版权声明和其他标识消息是一种常见的做法。ReSharper允许您配置默认的标题文本，并将其自动插入解决方案的代码文件中。

如果使用文件模板创建配置的文件头，则可以将其插入新文件中。为此，您应该添加一个模板参数 ，并将“文件头” 宏与其关联。

如果要将文件头应用于现有文件，则可以使用代码cleanup来完成 。
