代码清理

ReSharper |编辑清理代码
Ctrl+Alt+F
ReSharper_CleanupCode

您也可以使用命令行实用程序运行代码清除。

ReSharper允许您以批量模式应用格式设置和其他代码样式首选项 ，以立即消除一个或多个文件，项目或整个解决方案中违反代码样式的情况。

修正不一致的命名不包括在代码清除中，因为在批量应用时，它可能导致命名冲突。

代码清理配置文件

代码清除可以具有各种配置（配置文件），以根据您的需要在不同情况下应用。现成可用的三个默认配置文件：

  • 内置：重新 格式化仅应用代码格式首选项的代码

  • 内置：重新格式化并应用代码样式 ，可应用代码格式化首选项和代码语法样式 。

  • 内置：完全清除 ，将应用所有可用的清除任务，但更新文件标头除外。

这些配置文件在大多数情况下有帮助，但您无法对其进行配置。如果要使用自定义的清除任务集运行代码清除，则需要创建一个自定义配置文件并在此处指定所需的任务集。

可以与其他开发人员共享代码清除设置（包括自定义配置文件），以确保代码的外观和风格统一（有关详细信息，请参阅管理和共享ReSharper设置 ）。

创建新的自定义清理配置文件

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 转至代码清理设置页面： 代码编辑|代码清理 。

  3. 选择左侧的现有配置文件之一，然后单击页面顶部的“ 重复 ”。ReSharper将使用与原始配置文件中相同的任务集创建一个新的清理配置文件。

  4. 在配置文件列表中选择新创建的配置文件。

  5. 单击页面顶部的配置 。

  6. 所选配置文件的任务在右侧变得可编辑。根据需要配置它们。

  7. 如果要在保存修改之前创建或编辑其他配置文件，请单击页面顶部的“完成”以暂时记住对当前配置文件的修改。

  8. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

当您运行代码清理时，新创建的清理配置文件将在“ 代码清理”对话框中可用。您还可以将此配置文件设置为用于静默清除 。

运行代码清理

ReSharper |编辑清理代码
Ctrl+Alt+F
ReSharper_CleanupCode

ReSharper可以将代码清除应用于任意选择的代码块，当前文件中的所有代码，目录中的所有文件，当前项目甚至是当前解决方案。确保通过选择正确的清理配置文件来选择正确的清理任务集。

代码清除不适用于生成的代码。默认情况下，ReSharper提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ）。

清理选择或整个文件

  1. 选择您要清理代码的范围：

    • 在编辑器中进行选择以清除选择中的代码。

    • 将插入符号设置在文件中的任何位置以清除文件中的代码。

    • 在解决方案资源管理器中选择一个或多个项目，以清除这些项目及其子项目下文件中的代码。

  2. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  3. 在打开的“代码清理”对话框中，选择左侧的清理配置文件。使用右侧的区域检查所选配置文件中包含的任务。

  4. 如果没有任何可用的配置文件适合您，请选择一个现有的配置文件，然后单击“ 复制”以创建新的自定义配置文件 。
    请注意，如果您在此对话框中创建或修改清除配置文件，则将使用智能保存逻辑来保存您的更改。如果需要将修改保存到共享设置层 ，请在“ 代码编辑” |“修改”中进行修改。 ReSharper选项的“代码清理”页面，单击“ 保存到” ，然后选择所需的设置层。

  5. 单击运行以清除所选范围内的代码。

您不仅可以清理单个文件，还可以清理目录，项目或整个解决方案中的多个文件。为此，您应该从Solution Explorer运行代码清理。

清理多个文件，项目或整个解决方案

  1. 在解决方案资源管理器中选择一项或多项（文件，目录，项目或解决方案）。

  2. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  3. 如上所述选择清理配置文件。

  4. 单击运行以清除所选范围内的代码。

静默清理

ReSharper |编辑静默清理代码
Ctrl+Shift+Alt+F
ReSharper_SilentCleanupCode

您也可以在静默模式下运行代码清除-即，不显示“ 代码清除”对话框 。由于在这种情况下，您将无法明确选择配置文件，因此您必须分配一些清理配置文件以用于静默代码清理。

ReSharper不会默认选择静默清除配置文件。这意味着，除非您自己选择一个配置文件，否则无论何时启动静默代码清除，ReSharper都会打开“ 代码清除”对话框 。

分配配置文件以进行静默代码清除

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 转至代码清理设置页面： 代码编辑|代码清理 。

  3. 在左侧选择清除配置文件，然后在选项页面顶部单击“ 设为默认值 ”。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

在静默模式下运行代码清除

  1. 请执行以下任一操作：

    • 要在特定代码文件中运行静默代码清除，请在文本编辑器中将其打开，或在解决方案资源管理器中选择它。

    • 要将静默代码清除应用于一组文件（目录，项目或解决方案），请在解决方案资源管理器中选择相应的项目。

  2. 按Ctrl+Shift+Alt+F或选择 ReSharper |编辑主菜单中的“静默清除代码” 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

删除代码冗余

上 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ）中，在大多数语言的代码类别中，ReSharper会列出50多个与代码检查相关的冗余。其中有些需要特别注意，您不能以批处理模式进行处理。这样，在代码清理中，大约有30个冗余，只要您在清理配置文件中选择“ 删除代码冗余” ，它就可以自动删除 。
具体来说，代码清理：

  • 删除：

    • 冗余的空构造函数。

    • 枚举中的显式整数类型说明符。

    • true中的说明符for陈述。

    • 冗余类型转换。

    • 没有参数的属性中的多余括号。

    • 显式调用不带参数的基类构造函数。

    • 冗余布尔比较。

    • 来自集合初始值设定项的带括号的单个表达式。

    • 冗余的显式委托创建表达式。

    • 来自对象/集合初始化程序的空参数列表。

    • 多余的new关键字。

    • 多余的partial类型声明具有单个部分的类型声明上的修饰符。

    • 多余的unsafe上下文和陈述。

    • 多余的object.ToString()调用（例如，与字符串串联或作为a的参数string.Format()呼叫）。

    • 多余的string.ToCharArray()电话。

    • 参数类型的冗余规范。

    • 匿名方法中的签名，其中方法主体中未使用其参数。

    • 未引用的标签。

    • 冗余字段初始化程序。

    • 匿名类型属性中的冗余显式名称。

    • 冗余的可为空类型，其中值类型可隐式转换为可为空。

  • 转换：

    • 可空类型到短格式Nullable至T? ）。

    • 扩展方法调用的静态方法调用，其中扩展方法作为静态方法被调用。

    • 当可以从初始值设定项推断出数组类型时，以隐式类型化数组创建表达式。

  • 修正：

    • 通过类型的派生成员访问类型的静态成员的情况。
