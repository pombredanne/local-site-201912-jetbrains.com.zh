代码检查：安全的强制转换表达式始终成功，请考虑直接强制转换

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

ReSharper在两种情况下发出此建议：将兼容类型从派生类型转换为基本类型，以及将基本类型转换为派生类型的情况。

为了将引用变量从派生类型安全地转换为基本类型，请使用as运算符 。安全性意味着代码不会引发异常，而是会为您分配值的变量分配null如果无法执行转换。如果类型兼容 ，则使用as总是会成功，因此在这种情况下，显式强制转换就足够了。

在示例中，首先检查类型兼容性（通过使用is操作符），并且仅在类型兼容时才执行强制类型转换。存在检查is使使用as多余的。冗余强制转换可能会降低性能。ReSharper从兼容的派生类型中检测安全的强制类型转换，并用直接强制类型替换此强制类型：

次优代码

快速修复后

class SafeCastDemo {public class Animal {} private class Cat：Animal {} public Animal GetCat（Animal a）{Cat c = new Cat（）;如果（a是Cat）{c = a as Cat; } return c; }}
class SafeCastDemo {public class Animal {} private class Cat：Animal {} public Animal GetCat（Animal a）{Cat c = new Cat（）;如果（a是Cat）{c =（Cat）a; } return c; }}

安全强制转换始终成功的另一种情况是从基本类型转换为派生类型，而根本不需要显式强制转换（编译器将隐式执行该转换）。因此，在进行从基本类型到派生类型的转换时，也会显示删除安全类型转换的快速修复。由于这种转换不需要强制转换，因此下面的代码还调用了另一个快速修复程序，该快速修复程序完全删除了显式强制转换。该示例显示了应用了两个快速修复之后的结果（安全类型强制转换和显式类型强制转换都已删除）。

快速修复后

class SafeCastDemo {class Animal {} class Cat：Animal {} class Program {static void Main（）{Catс= new Cat（）;动物a = c; // ...}}}
