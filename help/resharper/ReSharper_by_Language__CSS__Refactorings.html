CSS重构

CSS样式的重构可帮助您在样式表和HTML文件之间快速移动CSS样式，或即时创建新的样式表并将现有样式移动到那里。您还可以单击几下来重命名CSS类及其用法。

除了重构之外，ReSharper还提供了 Context Actions可用的许多较小的本地代码转换 。

ReSharper为CSS提供了以下重构：

  • 提取样式
  • 移动
  • 改名

执行重构

  1. 将插入符号设置为符号，选择要重构的代码片段，或在工具窗口中选择一个项目。

  2. 请执行以下任一操作：

    • 在主菜单中，选择“ ReSharper |重构 ，然后选择所需的重构。此菜单中可用的重构列表取决于当前上下文。如果ReSharper无法为上下文建议任何重构，则将禁用整个菜单。

    • 在编辑器的“ 文件结构”窗口或其他“ ReSharper”窗口中，右键单击要转换的项目，从上下文菜单中选择“ 重构 ”，然后选择所需的重构。

    • 在主菜单中，选择“ ReSharper |重构|重构此 ，或按Ctrl+Shift+R以显示适用的重构列表，然后选择其中之一。您也可以在选择的上下文菜单中选择“ 重构此 ”。

    • 使用分配给特定重构的默认键盘快捷方式 ，或将自定义快捷方式分配给您喜欢的重构命令。

  3. 如果所选的重构需要用户输入，则重构向导将打开。请注意，向导的对话框不是模态对话框，因此您可以在向导打开时编辑代码。

    要回滚重构操作，向导提供选项“启用撤消，打开所有具有更改的文件以进行编辑” 。如果选择此选项，ReSharper将在新的编辑器选项卡中打开所有修改的文件，并使您能够回滚重构。在这种情况下，您需要自己保存更改。如果未选择此选项，则ReSharper会自动保存已修改的文件，而无需打开它们。

  4. 如果重构操作将导致代码冲突（例如重复的名称，可见性冲突等），则在应用重构之前，向导将在最后一步显示冲突列表。对于某些冲突，向导还可以建议快速修复。有关更多信息，请参见解决重构中的冲突 。

Visual Studio提供了自己的“ 重构”菜单，可从主菜单和上下文菜单中使用。默认情况下，ReSharper隐藏本机Visual Studio菜单，将其替换为自己的菜单。要使Visual Studio菜单可访问，请在“ 环境” | “ 环境”中选择“ 隐藏覆盖的Visual Studio菜单项”复选框。 ReSharper选项的“ 键盘和菜单”页面。

在编辑器中修改代码后，可以立即进行某些重构。有关更多信息，请参见就地重构。

提取样式

通过这种重构，您可以从CSS文件或CSS文件中另一个规则集中定义的现有规则创建CSS规则集。 style HTML中的属性。当插入符位于CSS规则集中的CSS文件或CSS文件中的任意位置时，可以调用此重构。 style标记在HTML文件以及style HTML文件中的属性。在打开的“ 提取样式”对话框中，使用复选框指定要提取的样式声明（或者，可以在调用重构之前选择所需的声明）。如有必要，修改新规则集的选择器。

应用重构后，将在同一文件中创建提取的规则集。如果要从HTML提取内联样式规则style属性，将提取的规则集放置在style标签。如果要将提取的规则集放置在另一个文件中，请稍后使用“ 移动重构”。

移动

通过这种重构，您可以将CSS规则集移至另一个文件。当插入符位于CSS规则集中的CSS文件或CSS文件中的任意位置时，可以调用此重构。 style标记在HTML文件中。在打开的“ 移动”对话框中，指定目标文件或键入新CSS文件的名称，然后单击“ 创建” 。如有必要，请选择插入导入指令以添加@import以及将规则集移动到的文件的URL。

单击“ 下一步”完成重构。

您还可以使用专用快捷方式调用此重构F6 。

改名

CSS中也提供了“ 重命名重构”功能 ，无论您在CSS文件或标记文件中以声明或用法调用重构的位置如何，都可以重命名类选择器或ID选择器。

在打开的“ 重命名”对话框中，为类或ID指定一个新名称，然后单击“ 下一步” 。默认情况下，ReSharper查找重命名项目的所有用法，并将它们显示在重构向导的下一页上。如有必要，您可以清除不想重命名的用法旁边的复选框。

您还可以使用专用快捷方式调用此重构F2 。
