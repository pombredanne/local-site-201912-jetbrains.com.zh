代码检查：NUnit。参数类型不兼容或参数值不正确

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

该检查会通知您有关通过传递给测试的值类型不匹配的信息。 TestCase参数和测试方法中相应参数的类型。例如，在下面的测试方法中，错误非常明显：

[TestCase（“ one”）] public void Test9（int x）{//做某事}

但是有时候，错误可能更难发现。例如

[TestCase（1，2）] public void Test4（int [] values）{//做某事}

在这种情况下，不会自动创建数组，因此参数必须是一个数组，测试才能正常工作：

[TestCase（new [] {1，2}）] public void Test4（int [] values）{//做某事}

请注意，NUnit将尝试转换通过提供的值TestCase在测试中使用它之前，ReSharper会在发出警告之前考虑此转换。还要注意，NUnit 2.x和NUnit 3具有不同的值转换逻辑。
