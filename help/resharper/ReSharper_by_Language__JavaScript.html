ReSharper 2019.2帮助

ReSharper按语言分类：JavaScript

ReSharper完全支持JavaScript 直到ECMAScript 2016，包括实验性功能，例如异步/等待，指数运算符以及对象文字/解构中的休息/传播。 jQuery和JSX语法也受支持。

您可以通过清除以下选项中的JavaScript和TypeScript复选框来禁用JavaScript中的ReSharper功能（.js文件和HTML中嵌入的JavaScript ） 环境|的产品和功能页面 ReSharper选项（ Alt+R,,O ）。

默认情况下， 代码检查和其他ReSharper功能会根据ECMAScript 5标准对JavaScript代码进行分析，该标准得到了普遍支持。如果在项目中使用更高级的JavaScript代码，则可以在“ 代码编辑” |“更改”中更改目标ECMAScript级别。 JavaScript | ReSharper选项的“ 检查”页面。

以下主题对JavaScript的ReSharper功能的描述进行了分组：

  • 代码检查和快速修复
  • 编码协助
  • 导航和搜索
  • 代码样式协助
  • 代码模板
  • 重构
  • 单元测试协助

使ReSharper忽略特定代码

ReSharper允许您以不同的方式忽略特定的文件，文件夹和文件掩码。您可以在以下选项页面上配置忽略的文件：

  • 代码检查|从代码检查中排除文件，文件夹和掩码的设置 。

  • 代码检查|生成的代码可以部分禁用对生成的代码的代码检查 。

  • 环境|搜索和导航以指定“ 按名称搜索”命令应忽略的文件和文件夹。

  • 代码编辑|要添加的第三方代码 C ++，JavaScript，TypeScript，CSS，HTML和JSON文件，文件夹和通配符将被视为“跳过”或“库”。ReSharper将完全忽略“已跳过”的文件，并将“库”文件视为只读文件-为导航而建立索引，但不进行检查 ， 快速修复和重构 。
