ReSharper时会考虑在此页面上可配置的首选项产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。通过使用带有相应设置的代码清除 ，它们也可以应用于现有代码。

引号（继承自JavaScript）

本节中的首选项定义了应将撇号（'）还是引号（“）用作引号。
此首选项与JavaScript的相应首选项共享，即，如果您在此处更改它，则在“ 代码编辑” | “ 代码编辑” |“首选项”上具有相同的首选项。 JavaScript | ReSharper选项的“代码样式”页面也已更改。

修饰符

本节中的首选项定义了是否应明确使用“ public”修饰符。

别名

本节中的首选项定义了如何处理别名。

  • 最好添加模块别名...
    指定是使用非限定类型名称并添加必要的模块别名还是使用完全限定类型名称。

  • 'require'中的相对文件引用样式
    指定是否使用前导./字符。例如， import X = require("file1")要么import X = require("./file1") 。如果选择“ 相对于根文件夹” ，则可以在“ 代码编辑” | “目录”中指定根文件夹的路径。打字稿| ReSharper选项的“ 生成的代码”页面。

型号规格

本节中的首选项定义是否以及何时显式指定类型，以及是否允许隐式“ any”。

也可以在项目设置中配置使用隐式“ any”的首选项。如果项目设置中不允许使用隐式“ any”类型，则它们始终会突出显示为错误，而与您的代码样式首选项无关。
