ReSharper 2019.2帮助

ReSharper按语言分类：C ++

ReSharper C ++提供了C ++支持， ReSharper C ++是一种专用产品，可以单独安装，也可以与ReSharper或ReSharper Ultimate并排安装。C ++支持包括C，C ++ 03，C ++ / CLI和部分C ++ 11，C ++ 14，C ++ 17，C ++ 20。支持ATL，MFC和COM项目。 ReSharper基于平台工具集（项目属性中的General | Platform Toolset ）和/或/ std开关自动检测C ++版本。

您可以通过清除以下位置的ReSharper C ++复选框来禁用C ++文件中的ReSharper功能： 环境|的产品和功能页面 ReSharper选项（ Alt+R,,O ）。

不支持和部分支持的语言构造

不支持的C ++ 11功能：

  • 部分支持Lambda函数

  • constexpr表达式
  • 用户定义的文字

不支持的扩展：

  • C ++ / CX

不支持的MS C ++扩展：

  • C ++组件扩展

  • 标识符，关键字和数字文字内的反斜杠前紧跟新行

  • __identifier关键字
  • 基于指针
  • 本机C ++中的事件处理
  • 解析模板中使用的名称时，所有名称均视为从属名称

部分受支持的MS C ++扩展：

  • __uuidof关键字
  • __pragma关键字

不支持的MS预处理程序扩展：

  • #pragma push_macro / pop_macro指令

使用__RESHARPER__宏禁用ReSharper的代码分析引擎

如果ReSharper的代码分析引擎不支持某些C ++构造或解析不正确，则可以使用以下命令对此代码禁用ReSharper： __RESHARPER__宏，即：

#ifndef __RESHARPER__ //此代码对ReSharper不可见#endif

使ReSharper忽略特定代码

ReSharper允许您以不同的方式忽略特定的文件，文件夹和文件掩码。您可以在以下选项页面上配置忽略的文件：

  • 代码检查|从代码检查中排除文件，文件夹和掩码的设置 。

  • 代码检查|生成的代码可以部分禁用对生成的代码的代码检查 。

  • 环境|搜索和导航以指定“ 按名称搜索”命令应忽略的文件和文件夹。

  • 代码编辑|要添加的第三方代码 C ++，JavaScript，TypeScript，CSS，HTML和JSON文件，文件夹和通配符将被视为“跳过”或“库”。ReSharper将完全忽略“已跳过”的文件，并将“库”文件视为只读文件-为导航而建立索引，但不进行检查 ， 快速修复和重构 。
