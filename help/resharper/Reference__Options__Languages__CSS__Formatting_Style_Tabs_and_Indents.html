制表符和缩进

ReSharper |选项|代码编辑| CSS |格式样式|制表符和缩进

此页面可帮助您指定键入或重新格式化现有代码时，ReSharper应如何在代码中设置缩进格式。

缩进设置源

默认情况下，ReSharper对选项卡和缩进使用Visual Studio首选项（ 工具|选项|文本编辑器|所有语言|选项卡和工具|选项|文本编辑器| [语言] |选项卡 ）。但是，如果您希望使用ReSharper的分层设置与团队共享格式化程序首选项，则可以清除“ 使用Visual Studio中的缩进样式和大小”复选框，然后配置缩进首选项并将其保存到团队共享的设置层。

如果有.editorconfig文件影响您的解决方案，则此页面上的首选项可能会被EditorConfig样式覆盖。您如果页面上的至少一个首选项被当前文件的EditorConfig样式覆盖，将显示黄色警告，每个被覆盖的首选项也将以黄色突出显示。例如：

多行构造中的缩进和对齐相结合

如果您的偏好设置了缩进应使用空格，则ReSharper还将使用空格来对齐多个结构—例如，当多行上的函数参数被第一个参数对齐时：

无效的foo（int firstParameter，int secondParameter）{}

但是，当您选择对缩进使用制表符时，可能会有不同的方式来对齐多行构造。ReSharper使您可以选择在创建新代码和重新格式化现有代码时使用哪种方式。

您可以使用“选项卡用于缩进时如何对齐”选项配置用于对齐多行结构的首选项。

  • 使用空格（建议，看起来与任何制表符大小对齐）
    选择此选项后，ReSharper会使用制表符来缩进，并使用空格来对齐：

    推荐使用此选项，因为与第二个和第三个选项对齐的代码在带有不同选项卡大小的编辑器中查看时可能会失去对齐。

  • 仅使用标签（不准确）
    选择此选项后，ReSharper会同时使用选项卡进行缩进和对齐，这可能不会导致精确对齐：

  • 混合制表符和空格以实现最佳填充
    选中此选项后，ReSharper会将选项卡用于缩进，并且对齐方式将添加必要的空间以进行精确对齐。
