为什么需要步进过滤器？好吧，下面的插图几乎解释了所有内容：

由于某些功能非常简单或经过了很好的测试，因此无需介入。但是问题在于，调试器无法确定哪些功能值得调试，哪些功能不值得，因此Step Into不会错过任何功能。

借助ReSharper，您可以维护步进过滤器-一系列函数掩码，告诉调试器在步进时应跳过什么。默认情况下，该列表包括许多库函数，可以使用您自己的函数掩码对其进行扩展。

调试时，您可以快速向过滤器添加功能：按Alt+Enter在该函数上，它将被添加到过滤器中：

默认情况下，此操作使用智能保存逻辑保存过滤器，但是您也可以使用该操作的子菜单选择应在其中保存过滤器的设置层。

您可能还记得Visual Studio具有一种内置的方式来自定义C ++中的步进行为，这是C ++ Just My Code的一部分 。但是，没有提供用于从IDE配置此功能的用户界面，因此您需要手动编辑.natstepfilter XML配置文件以更改现有规则或添加新规则。此外，由于配置文件是全局的，因此无法添加特定于解决方案的规则。请注意，如果安装了ReSharper C ++，当前Visual Studio对.natstepfilter文件的支持将被禁用。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
