ReSharper 2019.2帮助

代码检查：用CLR类型名称或静态成员访问表达式中的关键字替换内置类型引用

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

C＃类型关键字和.NET Framework（CLR）类型名称对于内置C＃类型可以互换。如果您希望对内置类型名称使用一种或另一种样式，ReSharper可以帮助您保持自己的偏好。

您可以为静态成员访问维护单独的样式（例如，CLR类型名称- var str = String.Empty; ）以及本地变量，成员和参数（例如，关键字- void Foo(string str){ } 。

ReSharper会检查所有内置类型引用是否符合您的偏好 ，如果不符合您的偏好 ，它将突出显示此类引用并建议相应的快速修复或范围内的修复 。默认ReSharper的首选项表示C＃关键字是首选。

有关配置和实施C＃内置类型引用的首选项的更多信息，请参见代码语法样式：内置类型引用 。
