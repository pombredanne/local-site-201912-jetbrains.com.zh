Analyze Code Coverage with dotCover

ReSharper's unit test runner does not allow running two or more tests with the same ID. This rarely happens in practice, but if you do have tests with the same ID, they will appear as a single node in the Unit Tests explorer and Unit Test Sessions window, and only one of them will run.
Note that test frameworks identify tests in different ways. For example, MSTest uses test-methods FQNs but NUnit uses properties of test attributes as test IDs.

If you have ReSharper Ultimate license, you can analyze coverage of unit tests (that is the degree to which the code of your solution is covered by unit tests) with JetBrains dotCover.

When dotCover is enabled, the Cover Unit Tests command appears together with Run Unit Tests / Debug Unit Tests commands. You can use this command to run coverage analysis of unit tests from the editor, from the Solution Explorer, or from unit tests sessions.

You can disable dotCover by clearing the dotCover checkbox on the Environment | Products & Features page of ReSharper options (Alt+R,,O).

To learn more about analyzing coverage results, see dotCover help: Working with coverage results.
