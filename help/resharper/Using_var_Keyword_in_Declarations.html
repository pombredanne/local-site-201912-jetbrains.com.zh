代码语法样式：隐式/显式键入（“ var”关键字）

使用隐式类型的局部变量 （也称为var C＃3.0中引入的关键字（关键字）已变得非常流行，因为它在许多情况下提高了可读性。默认情况下，ReSharper还鼓励使用var关键字，但可以灵活配置其使用偏好-例如，您可以选择在特定情况下或在任何地方使用显式类型，ReSharper将帮助您实施偏好。

从C＃7.0开始，您可以在解构元组时声明局部变量。若你宁可var在此类声明中，您可以另外配置样式以使用联接或分隔符，例如： var (x, y) = GetTuple();要么(var x, var y) = GetTuple(); 。

C＃7.0的另一个功能是throws ，它还允许使用var确保与可以命名的作用域中的变量没有冲突_ 。ReSharper允许您配置样式以始终使用var并在适当情况下丢弃。

强制使用'var'关键字的首选项

ReSharper会检查所有局部变量是否符合您的偏好 ，如果不符合您的偏好 ，ReSharper会突出显示此类声明并建议相应的快速修复或范围修复 。

'var'关键字用法设置不适用于生成的代码。默认情况下，ReSharper提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ）。

强制在批量模式下使用'var'关键字的首选项的另一种选择是代码清除 。您可以使用默认配置文件“ 完全清理”或“ 重新格式化并应用代码样式”之一运行代码清理 ，或者创建和运行仅针对特定任务的自定义配置文件 ，如下所述。

将“ var”关键字与自定义代码清除配置文件一起使用的首选项

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 转至代码清理设置页面： 代码编辑|代码清理 。

  3. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 选定的配置文件设置”部分中，选中“ 强制'var'”关键字使用设置复选框。（可选）您可以在此配置文件中启用其他代码清除任务。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  5. 选择要在其中执行首选项的范围：

    • 将插入符号设置在文件中的任何位置，以强制您对文件进行首选项。

    • 在解决方案资源管理器中选择一项或多项，以在这些节点及其子项下的文件中强制执行您的首选项。

  6. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  7. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  8. 点击运行 。ReSharper将在所选范围内强制执行您的首选项。

如果要在不打开“ 代码清除”对话框的情况下强制执行“ var”关键字使用设置，则可以将创建的配置文件绑定到静默清除并通过按以下命令运行它Ctrl+Shift+Alt+F 。您还可以创建一个自定义清除配置文件 ，该配置文件将应用首选项与其他代码样式任务结合在一起。

配置使用'var'关键字的首选项

您的“ var”关键字用法首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

配置使用'var'关键字的首选项

  1. 去 代码编辑| C＃|的代码样式页面 ReSharper选项（ Alt+R,,O ）。

  2. 根据您的编码惯例/标准， 在声明类别的“ var”用法中修改设置。 您可以为不同类型设置使用'var'或显式类型的不同首选项：

    • 对于内置类型 -适用于C＃内置类型 。

    • 对于简单类型 -适用于不具有通用参数的类型。

    • 其他 -适用于泛型类型和解构声明。

    对于这些首选项中的每一个，您都可以选择使用'var'，显式类型或'var'（当“ explain”时使用） ，这意味着“ var”应仅用于初始化为创建对象，数组和文字或显式类型转换的变量。

  3. 默认情况下，ReSharper将建议多个的联合表示法var例如，在解构声明中 var (x, y) = GetTuple(); 。您可以选择对解构变量更喜欢单独的声明，以选择单独的符号，例如(var x, var y) = GetTuple(); 。

  4. 默认情况下，ReSharper建议使用独立_例如，用于丢弃 (_, _) = (0, 1); 。您可以选择使用“ var”关键字来丢弃，以始终使用var在适当的地方使用丢弃，从而确保与可能命名的范围中的变量没有冲突_ 。

  5. 通过右栏中的“ 使用选择器通知”，您可以设置代码检查的 严重性级别 ，以检测与首选项不同的代码。

  6. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。
