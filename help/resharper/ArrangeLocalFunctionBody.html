Code Inspection: Use preferred body style (convert to local function with preferred body style)

You can configure this inspection:

  • change its severity level to make the detected issues less or more noticeable

Expression-bodied functions is a C# 6.0 feature. By default, ReSharper automatically detects C# version based on the associated compiler. However, you can specify the target C# version explicitly â€” right-click the project in the Solution Explorer, choose Edit project item properties from the context menu and use the C# Language Level selector..

Starting from C# 6.0, you can declare function members using expression bodies, which look very similar to lambda expressions. You can use expression-bodied methods and properties to further simplify the syntax of simple implementations.

If you prefer to stick to either expression-bodied or block-bodied implementations for simple function members, ReSharper lets you configure your preferences separately for different kinds of members and maintain the consistency of your preferences throughout your codebase.

For example, let's select Expression body as the preferred style for local functions. Accordingly, ReSharper suggests the quick-fix To expression body if a simple local function is written with a block body:

Suboptimal code

After the quick-fix

public int LocalFunction() { int Sum(int x, int y) { return x + y; } return Sum(42, 55); }
public int LocalFunction() { int Sum(int x, int y) => x + y; return Sum(42, 55); }

Reverse transformation is available with a context action Alt+Enter.
