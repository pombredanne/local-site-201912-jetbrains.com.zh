待办事项资源管理器可帮助您在解决方案中的待办事项之间导航 。

工具栏控件

控制

名称

描述

全部展开 / 全部折叠

展开/折叠当前选项卡中的所有节点。

上一页 / 下一页
Shift+F8 / F8

导航到上一个/下一个项目，并在源代码中滚动。

显示预览 Ctrl+P

隐藏或显示窗格，该窗格在使用列表指定的位置（在窗口的底部或右侧）显示所选项目的预览。

出口

点击这个按钮以文本格式导出当前在窗口中显示的数据，或使用下拉选择器以XML或HTML格式导出数据。出现的“ 导出数据”对话框将帮助您将数据保存到文件或将其复制到剪贴板。

待办事项设置

开门 工具|的待办事项页面 ReSharper选项，您可以在其中管理待办事项的模式。

油漆项目

切换窗口中待办事项的颜色根据它们的类型（编辑，正常，问题，警告，错误）。颜色可以在“ 环境”中配置。 Visual Studio选项的“ 字体和颜色”页面：

在生成的代码中显示待办事项

允许您在生成的代码中显示或隐藏待办事项。您可以在以下位置配置包含生成的代码的文件和区域的列表： 代码检查|的“生成的代码”页面 ReSharper选项。

过滤

使用此列表选择一个过滤器以仅显示与相应模式有关的项目。默认情况下，提供了五个过滤器：

  • （所有）
  • 托多斯
  • 笔记
  • 虫子
  • 未实现
通过...分组

使用此列表选择将待办事项分组的条件。提供了几个选项：

  • 无 -项目显示为普通列表。

  • 类型 -项目按包含类型进行分组。

  • 命名空间 -项目按命名空间分组，显示为普通列表，不考虑嵌套结构。

  • 命名空间和类型 -项目按名称空间分组，在较低级别按遇到它们的类分组。

  • 目录 -项目按存储它们的目录分组；目录显示为普通列表，不考虑嵌套结构。

  • 文件 -项目按其包含的文件分组。

  • 目录和文件 -项目按存储它们的目录分组，而在较低级别按其包含的文件分组；目录显示为普通列表，不考虑嵌套结构。

  • 项目 -项目按其包含的项目分组。

  • 项目和类型 -项目通过包含项目进行分组，而在较低级别按类型进行分组。

  • 项目和目录 -项目按包含项目的顺序分组，在较低级别按目录分组；在项目中，目录显示为纯列表，不考虑嵌套结构。

  • 项目结构 -根据项目的物理结构将项目分组，并表示所有嵌套关系。
