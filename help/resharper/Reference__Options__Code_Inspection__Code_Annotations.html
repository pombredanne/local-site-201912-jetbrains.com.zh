代码注释

ReSharper |选项|代码检查|代码注释

使用ReSharper选项的此页面，您可以获取代码注释属性类以及进行相关配置。

项目

描述

JetBrains注释-NuGet官方包装

单击此链接转到JetBrains注释主页 ，您可以在其中使用NuGet获取注释。

自动传播注释

如果选中此复选框，则某些ReSharper功能将自动插入注释属性。例如，“检查参数是否为空” 上下文操作将添加[NotNull]该属性除了生成检查无效性的代码外。

默认注释名称空间

允许您选择ReSharper查找代码注释类型的名称空间。如果您在多个命名空间中具有相同的注释类型，则此选项可能会有所帮助。有关更多信息，请参见源代码中的注释 。

具有代码注释属性的名称空间列表 ReSharper会使用有效的注释属性类型自动检测解决方案和引用的程序集中的名称空间，并显示这些名称空间的列表（如果有）。

注释属性的默认实现在JetBrains.Annotations名称空间，但注释不能使用此名称空间。您可以将声明放在解决方案中的任何位置-ReSharper将自动检测它们。但是，如果声明位于不同的名称空间中，则应使用注释属性显式指定名称空间，如下所述。
在某些情况下，您的解决方案可能包含注释属性的几种实现，例如，您的类与ReSharper注释类的名称相同（ CanBeNullAttribute ， NotNullAttribute ，依此类推），或者您正在使用包含ReSharper注释类的第三方程序集。在这种情况下，您可以选择ReSharper应该在其中查找正确的注解-属性类集的名称空间。

注释源代码

在本部分中，您可以复制JetBrains.Annotations剪贴板的命名空间，以便您可以将声明嵌入解决方案中的所需位置。
复制注释之前，可以通过两种方式自定义类：

  • 单击内部 ，将注释类设为内部，即只能在插入它们的模块内访问。

  • 点击条件添加[Conditional("JETBRAINS_ANNOTATIONS")]属性归类。这将使编译器忽略代码中的注释属性，这意味着没有对'JetBrains的二进制引用。生成Annotations.dll程序集。但是，您可以在项目中定义“ JETBRAINS_ANNOTATIONS”条件编译符号，以保留元数据中的属性。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
