生成PDB文件

ReSharper提供了一个命令，用于为任何.NET程序集生成PDB文件（连同源文件）。

在Visual Studio中调试编译的代码时，还可以使用ReShaper反编译器自动加载反编译的符号 。

生成符号文件时，ReSharper会创建与Visual Studio符号缓存相同的目录结构。因此，如果将Visual Studio符号缓存目录设置为输出文件夹，则所有生成的符号将自动提供给Visual Studio调试器。您可以获取或设置缓存目录在Visual Studio中的选项： 调试|符号|在此目录中缓存符号 。

PDB文件的用法示例超出了本主题的范围。要了解有关PDB文件的更多信息，请参见John Robbins的这篇文章 。

为程序集生成PDB文件

  1. 在“程序集资源管理器”窗口中选择所需的程序集。

  2. 请执行以下任一操作：

    • 单击生成PDB 在装配浏览器的工具栏上。

    • 右键单击所选内容，然后在上下文菜单中选择“ 生成PDB ”。

    • 在“ 进程资源管理器”窗口中 ， 右键单击任何.NET进程，然后选择使用ReSharper Decompiler加载符号 。 生成将立即开始到Visual Studio选项（“ 工具” |“选项” |“调试” |“符号” ）中指定的符号缓存目录。如果未指定目录，ReSharper将建议自动将％LOCALAPPDATA％\ Temp \ SymbolCache添加为缓存目录。

  3. 在打开的“ PDB生成”对话框中，指定目标文件夹。（可选）您可以在参考层次结构中选择更多的程序集。如果在选项（ 工具|选项|调试|符号|此目录中的缓存符号）中指定了Visual Studio符号缓存，则可以使用单选按钮选择它

  4. 点击生成 。所选程序集的PDB和源文件在指定的目标文件夹中创建。
    您可以在“ 项目/ PDB生成状态”窗口中查看生成PDB的进度。

如果您想随时清除反编译缓存，则可以删除％LOCALAPPDATA％\ JetBrains \ Shared \ vAny \ DecompilerCache中的数据 。
