代码检查：IEnumerable的可能的多个枚举

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

考虑以下代码片段：

IEnumerable 名称= GetNames（）; foreach（名称中的变量名）控制台。WriteLine（“ Found” +名称）; var allNames = new StringBuilder（）; foreach（名称中的变量名）allNames。追加（名称+“”）;

假如说GetNames()返回一个IEnumerable ，我们实际上是在通过两次枚举此集合两次来做额外的工作foreach陈述。

如果情况变得更糟GetNames()导致数据库查询，在这种情况下，您可能最终在两个数据库中获得不同的值foreach如果其他进程在两次调用之间更改了数据库，则循环。

这种问题很容易解决-通过将序列转换为数组或列表，在变量初始化时强制枚举，例如：

列表名称= GetNames（）。ToList（）;

您的其余代码可以保持不变，因为数组和列表类型都实现了IEnumerable接口。

误报

有时，如果在枚举之前将IEnumerable对象传递给某种方法，则此检查可能会导致误报。例如：

public void Foo（IEnumerable values）{ThrowIfNull（values，nameof（values））; var x =值。ToList（）; //可能对IEnumerable进行多次枚举} public static void ThrowIfNull （T值，字符串名称），其中T：class {//自定义检查null但不枚举}

在这种情况下，ReSharper假定该方法将执行其他枚举，这在大多数情况下是正确的。如果该方法实际上并未枚举IEnumerable对象，则可以使用[NoEnumerationAttribute]标记相应的参数，以防止误报：

public void Foo（IEnumerable values）{ThrowIfNull（values，nameof（values））; var x =值。ToList（）; //没有关于多重枚举的警告}公共静态无效ThrowIfNull （[NoEnumeration] T值，字符串名称）其中T：class {//自定义检查null但不枚举}
