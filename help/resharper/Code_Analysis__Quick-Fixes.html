ReSharper 2019.2帮助

代码问题快速修复

ReSharper帮助您立即修复在设计时发现的大多数代码问题。就像按下一样简单Alt+Enter在突出显示的代码问题上，然后选择解决问题或改进次优代码的适当方法。

应用快速修复

快速修复程序在突出显示的代码问题的左侧以可视方式显示为操作指示符 。当您在代码发布时设置脱字号时，将出现这些指示符。以下指示器表示快速修复：

  • —建议针对错误严重性级别为已检测到的代码问题快速修复 。

  • —建议在NuGet软件包库中找到未解析的类型或名称空间 ，

  • —建议就地重构或可以解决检测到的代码问题的重构。

  • —为检测到的代码问题提供警告 ， 建议或提示严重性级别的快速修复程序 。

  • -建议Visual Studio的代码更正或重构操作。

  • —建议一种可以安全删除的冗余代码快速修复程序 。

查看和应用快速修复

  1. 使用编辑器和标记栏中 突出显示的问题来导航到特定的代码问题。

  2. 将尖号设置在突出显示的问题上。

  3. 按Alt+Enter或单击插入符号左侧的动作指示器以打开动作列表 。

  4. 单击所需的快速修复或使用箭头键将其选中，然后按Enter 。

在大多数情况下，会立即应用快速修复。但是，某些快速修复程序（例如Change All ）需要用户交互才能选择他们如何精确转换代码。在这些情况下，将在编辑器中部署一个热点会话 ，您可以在其中选择建议值之一或在活动输入位置中提供自己的值：

要完成热点会议：

  • 如果ReSharper建议使用当前参数的一些值，请使用Up和Down箭头键浏览建议值列表，或仅键入所需值。

  • 按Tab要么Enter接受该值并移至下一个参数的输入位置。如果这是最后一个参数，则热点会话完成，并且脱字符号移动到为该会话定义的结束位置。

  • 按Shift+Tab将输入焦点移至上一个参数的输入位置。

  • 按Esc退出热点会话。在这种情况下，所有会话参数都将使用默认值初始化。

一些快速修复方法不仅可以应用于当前的插入符号位置，而且可以应用于更大的范围。有关更多信息，请参见在范围内修复 。

可以使用结构搜索和替换模式中定义的自定义代码检查来扩展预定义的快速修复列表。

Visual Studio的快速修复

从2015版开始，Visual Studio附带了自己的代码分析引擎（Roslyn），并提供了自己的灯泡功能以执行快速动作，包括重构和修复错误。

Visual Studio的快速操作通常执行与ReSharper的快速修复 ， 上下文操作或重构相同的修复程序。因此，对于某些错误，Visual Studio和ReSharper可能有两个类似的建议。在下图中，您可以看到两个灯泡都建议删除多余的灯泡using指令：

为避免这种情况，ReSharper在“ 代码检查” | “ 代码检查”中提供了“不显示Visual Studio灯泡”选项。 ReSharper选项的“设置”页面。此选项默认情况下处于启用状态，因此除非禁用它，否则不会看到重复的灯泡。

如果抑制了Visual Studio灯泡，则可以通过选中将Visual Studio灯泡动作合并到ReSharper灯泡复选框来将Visual Studio动作集成到ReSharper的动作列表中 。 如果选择了此选项，ReSharper将为每个Visual Studio的快速操作执行以下操作：

  • 如果存在类似的ReSharper的快速修复或上下文操作 ，则不会显示Visual Studio的快速操作。

  • 如果没有类似的ReSharper动作，则Visual Studio的动作在ReSharper 动作列表中可用。您可以通过特殊图标识别Visual Studio的快速操作 。
