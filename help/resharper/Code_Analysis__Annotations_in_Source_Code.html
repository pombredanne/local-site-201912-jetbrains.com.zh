源代码中的注释

受益于ReSharper的代码注释的最简单方法是将注释属性添加到源代码的符号中，并使ReSharper以更高的准确性和洞察力分析您的解决方案。

默认情况下，所有注释属性类都标记有[Conditional("JETBRAINS_ANNOTATIONS")]属性，因此编译器将忽略代码中的属性用法，这意味着没有对'JetBrains的二进制引用。Annotations.dll的程序集将被生成。但是，您可以在项目中定义“ JETBRAINS_ANNOTATIONS”条件编译符号，以保留元数据中的属性。

在项目中启用代码注释支持

要在项目中使用ReSharper代码注释属性，您应该通过以下方式之一引用它们：

  • 推荐的方法是使用以下命令安装NuGet软件包： JetBrains.Annotations部件。
    实际上，您甚至不需要访问NuGet网站即可获得该软件包。只需添加using JetBrains.Annotations;指令，并使用相应的Alt+Enter操作以自动获取软件包。

  • 您可以将项目引用添加到JetBrains。您可以在ReSharper安装目录中找到Annotations.dll 。

  • 您还可以使用默认值将属性声明嵌入项目中的任何位置JetBrains.Annotations名称空间或任何其他名称空间。

在源代码中嵌入代码注释的声明

您可以在上获得代码注释声明的副本。 代码检查|的代码注释页面 ReSharper选项（ Alt+R,,O ）。（可选）在复制时，您可以选择将声明设为内部声明和/或添加[Conditional("JETBRAINS_ANNOTATIONS")]声明的属性。

注释属性的默认实现在JetBrains.Annotations名称空间，但注释不能使用此名称空间。您可以将声明放在解决方案中的任何位置-ReSharper将自动检测它们。但是，如果声明位于不同的名称空间中，则应使用注释属性显式指定名称空间，如下所述。
在某些情况下，您的解决方案可能包含注释属性的几种实现，例如，您的类与ReSharper注释类的名称相同（ CanBeNullAttribute ， NotNullAttribute ，依此类推），或者您正在使用包含ReSharper注释类的第三方程序集。在这种情况下，您可以选择ReSharper应该在其中查找正确的注解-属性类集的名称空间。

更改代码注释属性的来源

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择代码检查|左侧的代码注释 。

  2. 除了源代码和引用程序集以外的所有名称空间JetBrains.Annotations与注释类（ReSharper只会查找CanBeNullAttribute和NotNullAttribute声明）显示在具有代码注释属性的命名空间列表中。检查包含所需实现的条目。

  3. 如果列表中有多个名称空间，请在“ 默认注释名称空间”列表中选择ReSharper代码分析引擎应使用的名称空间 。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。
