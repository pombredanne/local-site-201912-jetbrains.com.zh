代码检查：冗余固定指针声明

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

固定大小的缓冲区字段是在不安全上下文中可用的结构成员，它们表示C样式内联数组。这些字段主要用于与本机代码互操作。

在C＃7.3存取固定大小缓冲区（该缓冲区可能驻留在可由GC移动的内存中）之前，仅在将缓冲区固定为fixed语句，而保证位于不可移动内存中的固定大小缓冲区可以直接建立索引。

公共不安全的结构MyBufferWrapper {公共固定字节Buffer [4]; public int Foo（）=>缓冲区[0] +缓冲区[1] +缓冲区[2] +缓冲区[3]; // C＃7.3之前的错误public int Bar（）{固定（byte * ptr = Buffer）{return ptr [0] + ptr [1] + ptr [2] + ptr [3]; // 好 } } }

在固定大小的缓冲区仅用于访问其元素的情况下，引入辅助固定指针声明的要求是不合理的，因为除非将缓冲区的地址存放在索引始终安全的地方。

C＃7.3删除了对可移动固定大小的缓冲区编制索引的不必要限制，并使它们的使用更加自然：

公共不安全的结构MyBufferWrapper {公共固定字节Buffer [4]; public int Foo（）=>缓冲区[0] +缓冲区[1] +缓冲区[2] +缓冲区[3]; //可以，因为C＃7.3 public int Bar（）{byte * ptr = Buffer; //错误：获取固定大小缓冲区的地址仍需要固定返回ptr [0] + ptr [1] + ptr [2] + ptr [3]； }}

ReSharper会检测使用fixed语句不是必需的，它提供了一个快速修复程序来删除冗余的固定指针声明。
