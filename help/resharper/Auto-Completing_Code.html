ReSharper 2019.2帮助

代码完成（IntelliSense）

ReSharper通过更高级的功能来补充和扩展Visual Studio的本机代码完成（IntelliSense）。例如，它根据您的键入缩小建议列表，自动导入所选的类型和扩展方法，在完成方法名称时添加括号，根据类型建议变量和字段名称，等等。

为了使您熟悉ReSharper的代码完成功能，以下是一些实现基础知识：

  • 默认情况下，ReSharper用其自身的功能替换了本地Visual Studio的IntelliSense。开始输入新标识符后，将显示“ 自动完成 ”建议列表。当您按默认的Visual Studio IntelliSense快捷方式时，将显示“ 基本完成 ”的建议列表Ctrl+Space 。
    如有必要，您始终可以返回本机Visual Studio的IntelliSense。为此，请在“ 环境”上选择“ Visual Studio ” 。智能感知 ReSharper选项的常规页面。

  • 其他两个完成命令， 智能完成 Ctrl+Shift+Space和导入符号完成 Ctrl+Alt+Space始终可用，而与标准完成命令的所选选项无关，它们始终可用，它们应用高级算法将更多项添加到建议列表中。

  • 所有ReSharper的完成快捷方式都可以连续按几次。在这种情况下，ReSharper会向完成列表添加更多建议。有关更多信息，请参见Double Completion 。

  • 默认情况下，所有ReSharper的完成功能均支持CamelHumps ，即您可以键入复合名称部分的首字母，并且该名称将出现在建议列表中。
    如有必要，可以通过清除“ 环境” |“ 标识符之间的匹配中间”复选框来禁用CamelHumps 。智能感知 ReSharper选项的“ 完成行为”页面。

以下是一些其他一般点，它们显示了ReSharper的代码完成功能与Visual Studio的本机IntelliSense有何不同：

  • 键入时，将根据您的输入缩小完成建议列表的范围。如有必要，您可以通过清除环境|复选框将其关闭。智能感知 ReSharper选项的“ 完成行为”页面。您还可以使用“ 环境” | “ 环境”中的相应选项来指定完成列表的初始大小。 智能感知 ReSharper选项的“ 完成外观”页面。

  • 与继承成员相反，强调了类型的直接成员。

  • 当您在列表中选择建议时，ReSharper会显示其摘自XML文档的摘要。您可以通过清除“ 环境” | “ 显示摘要”复选框来禁用此功能。智能感知 ReSharper选项的“ 完成外观”页面。

  • 默认情况下，方法的所有重载都显示为建议列表中没有签名的单个方法名称。选择后，过载列表（如果有）会显示在弹出窗口中，您可以在其中进行研究。要在弹出菜单中切换签名，您可以按Ctrl+P 。如果要在完成列表中分别查看重载方法的每个签名，请选择“ 环境” |“ 显示成员签名 ” 。智能感知 ReSharper选项的“ 完成外观”页面。

  • 如果适用，Code Completion会建议索引器以及类型成员，并在选择索引器的情况下相应地更正表达式语法。

  • 提供了用于参数和局部变量的自定义图标，以将其与字段区分开。

  • 在现有代码项上使用代码完成功能时，可以通过按插入所选的完成建议Enter或通过按选定的建议替换现有标识符Tab （如果需要，您可以在ReSharper选项的Environment | IntelliSense | Completion Characters页面上更改默认快捷方式）。

  • 在声明字段，方法参数或局部变量时，ReSharper会建议一个可能的名称列表，以考虑当前上下文中的其他名称和命名规则进行选择 。
