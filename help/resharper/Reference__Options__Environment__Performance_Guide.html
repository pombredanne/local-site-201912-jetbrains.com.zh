ReSharper |选项|环境|绩效指南

使用此页面可以获取对性能至关重要的首选项的概述，并快速更改其值。如果首选项的值不是最佳值，则ReSharper会显示一个蜗牛图标在它的旁边。

当您在列表中选择一个首选项时，ReSharper在页面底部显示配置它的路径，当前值和推荐值。如果所选首选项的当前值与建议值不同，则可以单击“ 应用”，并且ReSharper将自动设置建议值。
对于某些首选项-这些首选项带有警告图标 —自动更改其值是不可行的。选择此类首选项后，可以单击页面底部的“ 打开选项 ”并手动进行配置。

页面上的每个项目都有一个下拉选择器，您可以在其中选择如果首选项的值不是最佳值，则ReSharper应该如何反应：

  • 忽略 -ReSharper将不会跟踪此首选项。

  • 在状态栏中显示 —如果首选项的值与建议的值不同，则ReSharper将在Visual Studio状态栏上显示一条通知：

    您始终可以单击通知以选择此选项页面。

  • 静默修复 （某些首选项可能不可用）—如果ReSharper检测到其他值，它将自动为该首选项设置推荐值。

防毒软件

此部分显示在系统中检测到的防病毒软件。

Windows Defender（以及其他防病毒软件）通过不断扫描内容频繁更改的相关高负载进程和文件夹，可能导致Visual Studio和ReSharper出现性能问题。我们建议添加 将devenv.exe ， msbuild.exe ，项目/解决方案文件夹和ReSharper缓存文件夹（默认为％LOCALAPPDATA％\ JetBrains \ Transient和可配置 ）保存到Windows Defender（和其他防病毒软件）的忽略列表中。

ReSharper可以自动检测到devenv.exe已添加到Windows Defender的忽略列表中，并且不会对此进行通知，但是其他防病毒软件则无法实现。因此，如果您的防病毒软件已经忽略了相关的进程和文件夹，请为此项目设置“ 忽略”操作。

不要在垂直滚动条上显示注释

ReSharper 广泛使用右装订线或垂直滚动条显示设计时检查通知 ， 结构搜索和替换结果 ， 待办事项以及文件中符号的用法 。Visual Studio还会在滚动条上显示很多通知，有时Visual Studio和ReSharper会报告相同的错误，该错误被显示两次，并且滚动条上经常出现标记。
如果您更喜欢ReSharper标记，则可以完全或部分禁用Visual Studio批注并通过不呈现它们而获得一些性能上的好处。
或者，您可以在滚动条上禁用ReSharper的标记—转到ReSharper |选项|编辑器编辑器外观，然后选择不显示标记栏 。

不要将Visual Studio操作包括在ReSharper操作列表中

ReSharper可以显示集成在其自己的操作列表中的 Visual Studio灯泡操作。这在性能方面是昂贵的，因为ReSharper必须向Roslyn请求可用的操作，这可能导致CPU和内存使用量增加。
尽管所有对Roslyn的请求都是在后台线程中进行的，但是如果存在任何明显的性能问题，您可能希望退出Visual Studio操作集成。
