ReSharper |帮助|报告错误或提交反馈

您的反馈对于ReSharper开发团队非常重要。因此，除了“ 疑难解答”主题中提到的内容外，其他提供反馈的方法也集成到了ReSharper中：

  • 匿名使用情况统计信息，可帮助我们确定最受欢迎的产品功能，评估平均软件和硬件配置等。默认情况下，此选项处于禁用状态。要启用此选项，请选中“反馈程序”复选框中的“ 匿名参与 ” 环境|的产品反馈页面 ReSharper选项（ Alt+R,,O ）。

    使用情况统计信息是匿名的，并且完全符合GDPR。它是在后台自动发送的，不会影响性能。

  • 直接从产品用户界面报告问题 。这样，您可以在ReSharper的支持门户或带有必要附件的问题跟踪器中快速创建故障单。

  • 直接从产品用户界面向产品团队发送有关ReSharper的一般反馈 。

报告问题

一旦您报告问题，我们将尽力解决。附件中包含的信息将大大有助于我们的调查。

报告问题

  1. 在主菜单中，选择“ ReSharper |帮助|报告错误或提交反馈 。

  2. 在打开的“ 反馈中心”对话框中，切换到“ 问题报告”选项卡。

  3. 在“ 问题类别”列表中，选择以下选项之一：

    • 错误或不正确的行为 -报告错误，可用性问题或您发现不正确的任何行为。在这种情况下，您的报告将直接发送给ReSharper支持团队 。

    • 性能问题 -报告速度下降或冻结。在这种情况下，您的报告将添加到ReSharper Issue Tracker中 。

  4. 在主题字段中，输入报告的标题。

  5. 在“ 描述”文本区域中，您可以解释问题所在，向我们提供问题专案，或者仅描述出现问题时的操作。

  6. 点击附件 将一个或几个文件附加到您的报告。
    请注意，性能快照是提交性能问题所必需的。

  7. 如果您不想在提交报告后丢弃并丢失此信息，请选择“提交后保留文本附件” 。

  8. 默认情况下，问题是匿名提交的，但是如果您想查看所报告问题的进度或提供联系方式，则可以单击更改帐户 。在“ 提交为”对话框中，可以使用以下选项：

    • 登录名/密码 -如果您未启用两步身份验证，请使用此选项在youtrack.jetbrains.com上指定您的YouTrack帐户。

    • 浏览器认证 -使用此选项登录到你的帐户YouTrack youtrack.jetbrains.com用两步认证。

    • 访客 -使用此选项并指定您的姓名和电子邮件，以便我们在需要时可以与您联系。

  9. 如果您的请求已经完成并且可以发送了，请点击Submit 。

发送反馈

它为什么如此重要？

  • 对于我们来说，它使我们知道我们在产品质量和性能方面的地位。这也是了解您在使用产品时面临哪些困难的关键。

  • 对您来说，这是表达您对ReSharper总体印象的最快方法。不要犹豫，与我们分享您的想法！

提交反馈

  1. 在主菜单中，选择“ ReSharper |帮助|报告错误或提交反馈 。

  2. 在打开的“ 反馈中心”对话框中，切换到“ 反馈”选项卡。

  3. 使用此选项卡中的字段可以分隔反馈的各个方面。

  4. 点击附件 将一个或几个文件附加到您的报告。

  5. 如果您不想在提交报告后丢弃并丢失此信息，请选择“提交后保留文本附件” 。

  6. 默认情况下，问题是匿名提交的，但是如果您想查看所报告问题的进度或提供联系方式，则可以单击更改帐户 。在“ 提交为”对话框中，可以使用以下选项：

    • 登录名/密码 -如果您未启用两步身份验证，请使用此选项在youtrack.jetbrains.com上指定您的YouTrack帐户。

    • 浏览器认证 -使用此选项登录到你的帐户YouTrack youtrack.jetbrains.com用两步认证。

    • 访客 -使用此选项并指定您的姓名和电子邮件，以便我们在需要时可以与您联系。

  7. 如果您的请求已经完成并且可以发送了，请点击Submit 。
    您的反馈报告将直接发送给ReSharper支持团队 。

附件

除了文字信息，您还可以向我们发送一些技术数据。您可以在报告中包含许多不同的附件：

  • 当前文件 -在Visual Studio中打开并处于活动状态的文件

  • 文件 -您文件系统中可帮助我们了解您的问题的任何文件

  • 解决方案统计信息 -有关当前解决方案的一些信息，例如项目数，项目类型，文件数和类型，平均源文件大小等。

  • 设置 -具有ReSharper设置的文件

  • 压缩解决方案 -当前解决方案，将打包为.zip软件包

  • 模糊的解决方案结构 -具有空文件和随机名称的当前解决方案的结构

  • 性能快照 -ReSharper通过dotTrace收集的具有性能数据的快照

  • 屏幕截图

    • 屏幕截图 -如果只有一台显示器，则为整个屏幕的屏幕截图

    • 主屏幕的屏幕 截图或所有屏幕的 屏幕 截图 -如果您有多台显示器

默认情况下，每个报告都包含有关当前Visual Studio实例的环境信息：其版本，已安装产品及其版本的列表以及.NET Framework版本的列表。

隐私

所有附件仅对ReSharper团队可见。除此之外，还混淆了其中一些： 解决方案统计信息和解决方案结构 。这意味着，在创建文件，项目和解决方案时，所有文件名都将替换为随机名称，同时删除文件内容以及对外部源和程序集的引用。如果您想向我们发送包含真实源代码的文件和解决方案，请选择当前文件或压缩解决方案 。

如果您对要发送的附件的内容有疑问，请使用上下文菜单打开所有附件，然后查看其中的内容。
