创建文件模板

文件模板用于使用预定义的代码（例如类型声明，单元测试等）创建一个或多个新文件。文件模板没有快捷方式，由它们的描述标识。有关详细信息，请参见从模板 \ 创建文件和创建多文件模板 。

您的新文件模板可方便其他开发人员使用。根据需要，您可以在不同的设置层（包括团队共享层）中创建新的代码模板。如有必要，您可以稍后通过将其复制到团队共享的设置层或将其导出到文件中来 共享 它 。研究ReSharper 存储模板的方式可能也很有用。

文件模板的主体由纯文本和parameters组成，纯文本将原样显示。想法是将这些结合起来以在调用模板时最大化效率。

创建一个新的文件模板

  1. 在主菜单中，选择 ReSharper |工具|模板资源管理器… 。

  2. 在出现的“ 模板资源管理器”窗口中，单击“ 文件模板”选项卡。

  3. （可选）如果要在特定的设置图层中创建模板，请在“ 图层”列表中选择所需的图层 。要了解更多信息，请参阅模板存储 。

  4. 点击新模板 。在模板编辑器中打开新创建的模板 。

  5. 在“ 描述”字段中指定模板名称。

    ReSharper不会将每个代码模板保存在单独的文件中。因此， 描述标识符不必是唯一的。但是，请尝试提供可理解的标识符，以免在应用自定义代码模板时造成混淆。

  6. （可选）在“ 默认文件名”中，指定ReSharper在应用模板时建议的名称。您还可以勾选“ 文件名必须是有效的标识符”复选框 ReSharper检查应用模板时指定的文件名是否作为类型标识符有效。如果您的模板为新类型创建文件，并且使用提供的文件名使用当前文件名（不带扩展名 宏 ）初始化类型名，则此选项很有用。您可以检查预定义的文件模板以了解其实现方式。

  7. 要更改模板的默认范围 ，请单击“ 可用性”超链接。在出现的对话框中，使用复选框定义可以在何处应用新模板。您可以扩展某些范围以指定详细信息（例如，文件掩码或语言版本）：

  8. 通过键入或粘贴所需的代码来创建模板的主体。
    模板代码可以包含纯文本和参数 。 键入参数后（前缀和后缀为$ ），变得可用在文本区域“ 编辑变量”对话框右侧的“ 参数”部分中。按照“ 声明模板参数”部分中的说明配置所有定义的参数。

  9. 配置模板格式选项：

    • 选择重新格式化复选框以使 ReSharper会根据您的格式首选项设置自动重新格式化扩展的代码片段。

    • 选择“ 缩短合格的引用”复选框以进行 ReSharper会自动插入名称空间导入指令或删除应用模板后变得多余的限定符。如果未选中此复选框，则ReSharper将不会插入任何名称空间导入指令。

  10. 要保存模板，请选择“ 文件” | “模板” 。在Visual Studio菜单上保存所选项目或按Ctrl+S 。

调用命令以应用文件模板时 ，将显示可用模板列表。我们称其为“快速访问列表”或“快速列表”。您可以编辑此列表以在其中添加您喜欢的模板和/或重新排列其顺序。

编辑快速访问列表

  1. 在主菜单中，选择 ReSharper |工具|模板资源管理器… 。

  2. 在出现的“ 模板资源管理器”窗口中，单击“ 文件模板”选项卡。

  3. 选择一个范围以查看该范围的模板列表。
    纳入快速访问列表中的模板显示在快捷列表中的部分; “ 不在快速列表中”部分显示了可用于选定范围的其他模板。

  4. （可选）使用模板类别来缩小不在快速访问列表中的可用模板的列表。

  5. 使用拖放操作可将模板添加到快速访问列表中或从快速访问列表中删除模板和/或重新排列其顺序。

您还可以使用“ 选择模板”对话框将模板添加到快速访问列表中。当您应用不在快速访问列表中的文件模板时，将出现此对话框。

创建新模板后，您可能需要按类别组织模板和/或将其添加到快速访问列表中 。
