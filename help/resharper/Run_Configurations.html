使用运行配置

ReSharper |工具|运行配置...
Ctrl+Shift+Alt+R
ReSharper_RunConfigSettings

ReSharper可帮助您创建，管理和执行解决方案的多个运行配置。运行配置使您可以运行或调试项目，静态方法或任何可执行文件。

如果您拥有ReSharper Ultimate许可证，则可以使用JetBrains dotTrace来配置运行配置 。有关更多信息，请参见如何使用ReSharper和dotTrace立即分析代码中的任何方法 。

运行配置弹出窗口

您可以使用“ 运行配置”弹出窗口执行运行配置以及以任何方式管理它们。打开ppopup 按Ctrl+Shift+Alt+R或选择 ReSharper |工具|从主菜单运行配置...。

弹出窗口显示所有现有的运行配置。活动运行配置以粗体突出显示。

在ReSharper Build＆Run中运行配置

您也可以在“ 构建和运行”窗口中执行运行配置。

活动的运行配置显示在窗口工具栏上的组合框中，展开组合框时，将具有与“ 运行配置”弹出窗口中类似的控件。此外，您可以使用工具栏按钮来运行或调试活动配置，而无需重建。

活动运行配置

在所有运行配置中，始终将其设置为“活动”。按下时执行活动配置F5或选择调试|从主菜单开始 。默认情况下存在一个运行配置（VS启动），并且您无法删除它。此配置执行在Visual Studio中配置的启动项目。最初，它被设置为活动配置。

创建新的运行配置后 ，新创建的配置将自动激活。

要将另一个运行配置设置为活动，请在“ 运行配置”弹出窗口中将其选中，然后选择“ 配置” |“配置”。选择 。

重要
ReSharper会覆盖Visual Studio的“ 启动和启动而不调试”命令（可在“ 调试” |“启动/启动而不调试”菜单中或通过F5 / Ctrl+F5快捷方式）以执行ReSharper的活动配置。如果需要返回Visual Studio的配置，请在“ 运行配置”弹出窗口中选择“ VS启动 ” 。

运行配置的类型及其首选项

ReSharper允许您使用三种不同类型的运行配置。每个配置都通过名称进行标识，并且具有一些特定于其类型的属性。

启动项目

使用此配置类型，您可以运行或调试解决方案中的任何可执行项目。对于具有多个可执行项目的大型解决方案，这可以替代在解决方案资源管理器中搜索项目并将要执行的项目设置为StartUp项目的方法。

静态方法

使用此配置类型，您可以运行或调试任何不带参数的公共静态方法。您可以将这种类型的配置用于静态方法，以运行或调试代码库中的任何算法或子系统。如有必要，您可以使用控制台进行交互式执行。

对于这种类型，您需要提供方法的全名（即Project。类。方法 ），以及（可选）工作目录，体系结构，CLR（.NET Framework）版本以及在执行之前要构建的项目。

请注意，您可以使用临时运行配置执行静态方法 。

可执行的

使用此配置类型，您可以运行或调试任何可执行文件。如果选择debug命令，ReSharper将运行可执行文件，然后将Visual Studio附加到相应的进程。此配置类型可帮助您创建复杂的执行设置（例如，如果您正在为另一个程序使用插件）。

对于这种类型，您需要指定目标可执行文件，还可以指定工作目录，命令行参数以及执行之前是构建特定项目还是整个解决方案。

创建和编辑运行配置

创建新运行配置的通用方法是使用“ 运行配置”弹出窗口 Ctrl+Shift+Alt+R 。

创建任何类型的运行配置

  1. 选择添加 在“ 运行配置”弹出窗口中，然后选择要创建的配置类型（可执行文件，静态方法或启动项目）。
    如果要为项目创建运行配置，则可以在解决方案资源管理器中右键单击该项目，然后选择创建运行配置 。

  2. 在打开的对话框中，为配置指定名称和首选项 ，然后单击“ 确定”。

  3. 新创建的配置将保存并变为活动状态 。你可以按F5或单击工具栏上的运行以执行它。

对于静态方法，您还可以在编辑器中快速创建运行配置。

为静态方法创建运行配置

  1. 在没有参数的公共静态方法的声明中设置插入符号。

  2. 按Alt+Enter或单击左侧的操作指示器 。

  3. 在打开的操作列表中，选择“ 调试” 然后选择任何带有省略号的项目（例如Run ... ）

  4. 在打开的对话框中，ReSharper根据所选方法自动填充所有配置首选项。如有必要，您可以更改配置名称和任何其他首选项。

  5. 单击“ 保存并执行”以创建新配置，并以您在步骤3中选择的方式执行（运行/调试，带有/不带有建筑物）。
    或者，您可以单击执行以运行/调试配置而不保存。

如果要创建现有运行配置的副本，请在“ 运行配置”弹出窗口中选择它，然后选择“ 配置” |“配置”。重复 。在打开的对话框中，为复制的配置指定一个新名称，根据需要更改所有首选项，然后单击“ 确定”完成。

要修改现有运行配置的属性，请在“ 运行配置”弹出窗口中选择它，然后选择“ 配置” |“配置”。编辑 ，然后根据需要修改配置的属性。

要删除运行配置，请在“ 运行配置”弹出窗口中选择它，然后选择“ 配置” |“配置”。删除 。

对于静态方法，您可以直接在编辑器中编辑，复制和删除关联的运行配置。您可以通过“修复”操作指示器来识别已运行配置的静态方法在它的左边。单击此指示器或按Alt+Enter然后在“ 配置”项下选择“编辑”，“复制”或“删除”操作。

执行运行配置

执行活动配置

要运行/调试活动配置 ，请按Ctrl+F5 / F5或选择调试|在主菜单中无需调试即可启动/开始 。

如果执行运行或调试特定项目的配置，则ReSharper会自动将此项目设置为StartUp Project。

默认情况下，将重建项目/解决方案。您也可以执行活动配置而无需重建。为此，请按Ctrl+Alt+F5无需建造即可运行或Alt+F5无需构建即可进行调试。

如果您希望在不构建的情况下执行配置，则可以更改默认行为。为此，请在“ 运行配置”弹出窗口中选择“默认情况下不构建 ” Ctrl+Shift+Alt+R 。选择此选项后，使用Ctrl+F5 / F5执行活动配置而无需构建和Ctrl+Alt+F5 / Alt+F5如果要在执行之前重建项目/解决方案。

执行其他配置

您可以使用“ 运行配置”弹出窗口执行任何现有配置Ctrl+Shift+Alt+R 。为此，请在弹出窗口中选择所需的配置，然后选择正确的运行/调试命令。要执行Visual Studio的配置，请在弹出窗口中选择“ VS启动 ”。

为静态方法执行运行配置

ReSharper允许您执行任何不带参数的公共静态方法。您可以按这种方法设置插入符号，然后按Alt+Enter ，选择调试 ，然后选择所需的运行/调试命令。在执行开始之前，您将看到一个对话框，在其中您可以自定义执行参数。单击执行一次执行方法，或者如果你想重用这个配置保存和执行 。

如果静态方法具有关联的运行配置，则ReSharper将显示固定的操作指示器在它的左边。在这种情况下，执行该方法时不会显示对话框。

将调试器附加到正在运行的配置

如果需要调试运行配置，则可以立即以调试模式启动它。但是，如果使用“运行”或“不构建命令运行”来执行运行配置，则以后可以随时将其附加。

您只能附加到以下类型的运行配置： 静态方法 ， 可执行文件 。

如果您正在运行一个或多个运行配置，则可以通过在“ 运行配置”弹出窗口中选择“ 附加到[配置名称]”来附加到运行的进程。 Ctrl+Shift+Alt+R 。
ReSharper将自动将Visual Studio切换到调试模式，并附加到所选运行配置的过程，从而使您可以中断其执行，执行步骤等。

共享运行配置

默认情况下，您创建的所有运行配置都保存在ReSharper 分层设置的“解决方案个人”层中。

如果要与使用同一解决方案的团队成员共享一些运行配置，则可以将这些配置移至“解决方案团队共享”层。为此，请在“ 运行配置”弹出窗口中选择一个配置。 Ctrl+Shift+Alt+R ，然后选择配置|分享 。要停止共享运行配置，请选择“ 配置” |“配置”。取消分享 。将更改推送到共享层（ [SolutionName] .sln。DotSettings文件）插入到您的VCS中，其他用户将您的更改拉出，他们可以轻松使用共享的配置。

请注意，一旦通过VCS更新了“解决方案团队共享”设置层文件，其他团队成员将能够修改共享配置，并且更新的版本将可用。
