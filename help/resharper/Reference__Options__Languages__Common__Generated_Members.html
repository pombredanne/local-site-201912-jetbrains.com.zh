会员代

ReSharper |选项|代码编辑|会员代

ReSharper选项的此页面允许您调整代码生成设置。有关更多信息，请参见代码生成 。

项目

描述

生成的成员默认身体样式

此设置允许您选择ReSharper将在生成的成员主体中包括的内容。

  • 抛出新的NotImplementedException（）
    选择此选项时，ReSharper引发NotImplementedException()在生成的成员体内。

  • 返回默认值
    选择此选项后，ReSharper会根据返回类型在生成的成员中创建默认值return语句。

  • 无法编译的代码
    选择此选项后，ReSharper会在生成的成员的主体中填充阻止编译的符号，从而迫使您提供有意义的实现。

生成的属性样式

此设置定义如何生成用于实现和覆盖成员的属性

  • 自动属性
  • 有后勤领域的财产

  • 具有默认主体的访问器
    默认主体是根据上述“ 生成的成员默认主体样式”设置生成的 。

如有必要，可以在应用相应的代码生成命令时覆盖所有或某些属性的此设置。

生成的文件

此设置定义如何生成XML注释以实现和覆盖成员 ：

  • 无 -不将XML注释添加到生成的成员。

  • 加对于被覆盖的成员 -此标记是从基本成员复制文档的不错选择。尽管该标签是非标准的，但可以被NDoc，Sandcastle和Stylecop之类的工具理解。

  • 从覆盖的成员复制XML文档 -此选项允许将XML文档注释从覆盖的成员复制到生成的覆盖/实现成员。

如有必要，可以在应用相应的代码生成命令时覆盖所有或某些属性的此设置。

其他选择
将支持字段放在属性上方

当ReSharper为字段，具有背景字段的新属性生成属性或将自动属性转换为具有背景字段的属性时，该字段将始终置于属性上方。

包装区域中的生成成员

用描述生成内容的名称对区域内的重写成员和已实现成员进行分组，例如[Based class name的重写]和[base interface name]的实现 。

使用[DebuggerStepThrough]注释属性访问器

选中后，将DebuggerStepThrough属性添加到生成的属性的getter和setter中，以防止调试器进入属性访问器。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
