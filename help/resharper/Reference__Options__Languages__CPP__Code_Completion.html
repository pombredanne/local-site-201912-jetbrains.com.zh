在会员完成时显示免费功能

一种常见的C ++编码实践是将非成员非友函数优先于成员函数。这是增加封装并使类接口尽可能少的好方法。当您键入一个点.或箭->在表达式之后，将在成员函数之后的完成列表中建议接受将表达式作为第一个参数的自由函数：

如果不想在完成建议中使用免费功能，则可以清除此复选框。

显示基本完成的导入项目

当您要使用的符号未与相应的符号导入当前文件时#include ，但它是在标准库或解决方案的其他地方定义的，您可以按Ctrl+Alt+Space在完成建议中快速找到该符号：

您可以使用此选项将导入符号建议包括在自动完成和基本完成的建议列表中。 请注意，这会带来性能上的损失（因为每次ReSharper生成完成列表时，它都必须查看解决方案中的所有符号以匹配它们），因此默认情况下将其禁用。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
