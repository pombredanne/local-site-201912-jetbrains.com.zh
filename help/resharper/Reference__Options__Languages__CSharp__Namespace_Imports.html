命名空间导入

ReSharper |选项|代码编辑| C＃|命名空间导入

该页面包含的选项会影响ReSharper在现有代码库中优化名称空间导入指令的方式。

ReSharper还允许您调整许多影响名称空间导入样式的设置。您可以在代码编辑|查看和更改这些设置。 C＃| ReSharper选项的“代码样式”页面。

项目

描述

如果别名与导入的类型名称不同，请不要使用别名指令删除

如果选中此复选框，则ReSharper永远不会使用伪指令从源代码中删除非平凡的别名，即使它们是多余的也是如此。如果清除它，则ReSharper会删除所有使用指令的冗余。

不应删除的命名空间

在此框中，您可以列出ReSharper不会突出显示为冗余的名称空间，并且在运行代码清理时不会删除它们。

使用“ 添加”按钮将条目添加到列表并指定名称空间。

使用“ 编辑”按钮编辑名称空间条目。

使用删除按钮从列表中删除选定的条目。

应始终导入的命名空间

在此框中，您可以列出ReSharper在运行代码清理时应始终导入的名称空间。

使用“ 添加”按钮将条目添加到列表并指定名称空间。

使用“ 编辑”按钮编辑名称空间条目。

使用删除按钮从列表中删除选定的条目。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
