配置代码检查设置

您还可以使用代码注释自定义ReSharper检查代码的方式。

配置设计时代码检查

默认情况下，ReSharper启用与支持的语言相对应的所有文件中的设计时代码检查 。如有必要，可以将其禁用。无论是否启用了设计时代码检查，您都可以始终在特定范围内运行代码检查 。

配置设计时代码检查

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择代码检查|左侧的设置 。

  2. 使用“ 启用代码分析”复选框来切换设计时代码检查 。

  3. （可选）您可以在此页面上启用或禁用设计时检查功能：

    颜色标识符 此选项使您可以启用或禁用ReSharper 语法突出显示方案。
    如果被选中，语言标识符颜色突出显示在Visual Studio中的选项定义： 工具|选项|环境|字体和颜色 。
    “ 显示项目”列表中提供了ReSharper提供的语法标识符列表，每个名称均以ReSharper前缀开头。

    此选项不影响C ++。要在C ++中切换语法高亮显示，请在 代码编辑| C ++ |的检查页面 ReSharper选项。

    突出显示颜色用法

    启用代码中颜色定义的突出显示。有关更多信息，请参阅颜色辅助 。

    在字符串文字中突出显示特殊字符

    启用高亮显示非普通字符串中正确和不正确的转义序列。例如：

    有关更多信息，请参见正则表达式帮助 。

    突出显示上下文出口

    默认情况下启用此选项，它告诉ReSharper突出显示控制流可以退出当前上下文的所有位置。例如，对于一个方法，它将突出显示该方法的返回类型，所有return ， throw关键字，等等，当您将插入符号设置为这些标识符之一时

    对于循环，它将另外突出显示loop关键字以及所有break此循环中的语句。
    请注意，如果方法在编辑器中不完全可见，则可以在方法名称上调用“ 导航到功能退出”命令以触发另一种突出显示，当插入符号离开方法名称时，该突出显示不会消失。

  4. 如有必要，可以选中“ 启用解决方案范围的分析”复选框以启用“ 解决方案范围的分析” 。

  5. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

从代码检查中排除文件和文件夹

ReSharper允许您配置应从代码检查中排除的文件，文件掩码和文件夹的列表。设计时代码检查和特定范围内的代码检查都将忽略被排除的项目，但是ReSharper仍对它们进行索引，因此您可以导航到或重构从代码检查中排除的符号。

从代码检查中排除文件和文件夹

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择代码检查|左侧的设置 。

  2. 在要跳过的元素部分的左侧，您可以指定代码检查要忽略的文件或文件夹。

  3. 在本部分的右侧，您可以指定掩码（例如* .vb ），这些掩码将从代码检查中排除解决方案中的所有匹配文件。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

您还可以从代码检查中快速排除当前文件，以便ReSharper将此文件添加到排除文件列表中，而无需打开选项。

从代码检查中排除/包括当前文件

  • 按Ctrl+Shift+Alt+8 。

  • 右键单击状态指示器，然后选择“ 暂停/继续分析” 。

如果您打开了未通过代码检查的文件，则可以通过相应的图标来识别它 状态指示灯的 。

从索引中排除冻结的项目

在 代码检查|的设置页面 ReSharper选项（ Alt+R,,O ），您还可以使用“ 忽略的项目”列表来指定ReSharper应该完全忽略的项目。

ReSharper将不会为被忽略的项目建立索引，这意味着这些项目中将没有ReSharper功能，例如代码分析 ， 重构 ， 代码生成 。

该列表的主要目的是通过不对包含第三方代码或冻结代码（不会更改）的项目进行索引和分析，从而提高性能 。

尽管没有分析被忽略项目中的源文件，但ReSharper仍将处理这些项目输出中的编译代码（这在性能上非常便宜）。这意味着，在构建了被忽略的项目之后，ReSharper将正确解析这些项目中符号的用法，您将能够在这些项目中进行导航和搜索 。

因此，如果将一些项目添加到忽略列表后，您遇到诸如“无法解析符号”之类的错误，则需要构建被忽略的项目，ReSharper将能够从编译的工件中解析那些符号。

要将项目添加到忽略列表，请使用项目文件名（例如ProjectOne.csproj）或示例文件 （例如Project * .csproj）来匹配ProjectOne.csproj ， ProjectTwo.csproj等。

部分禁用对生成代码的代码检查

您还可以配置包含生成的代码的文件，文件夹，文件掩码和区域的列表。 对于这些项目，ReSharper仅运行那些检查代码的编译器错误和警告的代码检查 。默认情况下，此列表中包括Visual Studio用于生成代码的某些文件掩码和区域，例如* .designer.cs ，但是您可以根据需要更改默认设置。

使用生成的代码指定文件和区域

  1. 上 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ），请使用“ 添加文件”和“ 添加文件夹”按钮指定包含已生成代码的文件或文件夹。

  2. 在页面的左下方，使用“ 添加”按钮指定蒙版（例如*。Designer.cs ）用于生成的代码文件。

  3. 在页面的右下部分，使用“ 添加”按钮指定包含生成的代码的区域的名称。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

包含生成的代码的项目列表也可以用于禁用代码样式和清除生成的代码。

更改代码检查的严重性级别

每个ReSharper的代码检查都有其自己的默认严重性级别 ，该级别根据其检测到的代码问题的潜在影响来设置。大多数检查具有可配置的严重性级别，您可以更改此级别。

请注意，检测到编译器错误和警告的检查具有相应的严重性级别，无法更改。但是，通过一些编译器警告，您可以使用#pragma指令来压制它们。根据特定警告，在操作列表中查找“ 使用#pragma抑制操作”。

您可以直接在编辑器中更改检查的严重性级别，该检查中发现的代码问题将突出显示。

从编辑器更改检查严重性

  1. 将插入符设置为ReSharper检查中突出显示的代码问题。

  2. 按Alt+Enter或单击插入符号左侧的动作指示器以打开动作列表 。

  3. 在操作清单中，选择检验[检验名称] | |。配置检查严重性 ，然后选择新的严重性级别：

  4. 您的更改将使用智能保存逻辑进行保存。

  5. 如果需要将修改后的严重性级别保存在共享设置层中 ，请单击“ 配置检查严重性”菜单项或按Enter选择它时。在出现的对话框中，选择所需的严重性级别，单击“ 保存到” ，然后选择所需的设置层。

将修改后的严重性级别保存在共享设置层中，或修改多次检查的严重性级别的另一种方法是使用“ ReSharper选项”对话框，如下所述。

通过选项修改代码检查的严重性级别

  1. 上 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ），则可以查看所有可配置的代码检查及其严重性级别。检查按语言分组，然后按类别分组。

  2. 查找并选择要修改其严重性的检查。要查找检查，只需开始输入其名称。

  3. 单击所选条目右侧的列表，然后选择所需的严重性级别：

  4. 如果检查的默认严重性级别已更改，您将看到“ 重置为默认” 按钮旁边的按钮，您可以将严重性重置为其默认值。
    同样的按钮也会出现在此类检查所属的类别旁边，单击类别旁边的此按钮会将类别内的所有检查重置为默认严重性级别。

  5. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

禁用/启用特定代码检查

如果某些代码检查对您而言微不足道或您不感兴趣，则可以禁用此检查，以便在 运行代码检查特定范围时不会在编辑器中突出显示或检测到任何相关问题。您可以在以后的任何时间启用禁用的代码检查。默认情况下，某些检查是禁用的，必要时可以启用它们。

您可以通过以下方式之一禁用任何代码检查：

  • 如果在编辑器中此检查突出显示了代码问题，请将插入符号设置为突出显示的代码，然后按Alt+Enter然后选择检验[检验名称] |配置检查严重性|不显示 。

  • 上 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ），请使用搜索框查找要禁用或启用的检查，然后使用其旁边的复选框。

检查的禁用/启用状态保存在共享设置层中 ，与更改严重性级别完全相同。

使用EditorConfig配置代码检查

从.editorconfig文件读取代码检查设置可能会影响大型解决方案的性能。

如果使用EditorConfig维护项目的代码样式，则还可以从.editorconfig文件配置代码检查。

要从EditorConfig配置代码检查，您必须选择从editorconfig和项目设置读取设置复选框 代码检查|的设置页面 ReSharper选项（ Alt+R,,O ）。

如EditorConfig约定所建议，ReSharper将应用在当前文件目录及其所有父目录中名为.editorconfig的文件中定义的检查设置，直到到达根文件路径或找到带有以下内容的EditorConfig文件为止： root=true 。例如，在.editorconfig文件中指定的文件掩码*Test.cs也考虑在内。

.editorconfig文件中的检查设置与其他属性的配置类似，方法是添加相应的行：
[inspection_editorconfig_property]=[error | warning | suggestion | hint | do_not_show] 。
例如，您可以更改“ 可能的”系统的严重性级别 。NullReferenceException'检查以下行错误 ：
resharper_possible_null_reference_exception_highlighting=error 。

.editorconfig文件中的检查设置具有比在上配置的设置更高的优先级 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ）。

您可以在“ 代码检查索引”部分的页面以及“ EditorConfig属性”的“ 索引”页面上找到每个检查的EditorConfig属性 。 —只需使用浏览器搜索来查找所需检查的属性。

禁止特定范围内的代码检查

忽略特定代码问题的一种方法是禁用相应的代码检查 。在这种情况下，此检查检测到的所有代码问题都会在任何地方被忽略。
有时，您可能需要禁止在特定位置进行特定检查，而在其他地方继续通过此检查检测其他类似问题。
例如，ReSharper认为某些代码“死了”，您可以看到它是真实的。该检查很有帮助，您不想禁用它。但是，您可能需要稍后使用此代码，而不希望在编辑器中突出显示该代码或不希望其出现在检查结果中 。为此，ReSharper允许您取消带有注释或属性的检查。注释对于任意代码段更方便，属性对于抑制整个方法或类型的检查更可取。

抑制特定范围内的代码检查

  1. 将插入符设置为ReSharper检查中突出显示的代码问题。

  2. 按Alt+Enter或单击插入符号左侧的动作指示器以打开动作列表 。

  3. 在操作列表中，选择以下一项：

    • 检验[检验名称] |禁用一次带有注释的选项-此选项插入单个注释ReSharper disable once [inspection id] ，这仅禁止检查首次出现的相应问题。

    • 检验[检验名称] |禁用一次评论|禁用带注释的文件 -此选项插入单个注释- ReSharper disable [inspection id]在文件的开头。此注释禁止检查文件中的所有相应问题。

    • 检验[检验名称] |禁用一次评论|禁用并带注释还原 -此选项在问题前后插入一对注释- ReSharper disable [inspection id]和ReSharper restore [inspection id] 。这些注释禁止检查它们之间的所有相应问题。
      然后，您可以将这些注释移到文件中的其他位置，以便消除这种类型的问题。例如，如果要保留多个未使用的名称空间导入，这对于抑制“冗余名称空间”检查很有用。

    • 检验[检验名称] |禁用一次评论|禁用方法 -此选项将以下属性添加到方法： [SuppressMessage("ReSharper", "[inspection id]")] 。此属性抑制方法中的检查。

    • 检验[检验名称] |禁用一次评论|对类禁用 -此选项将以下属性添加到类： [SuppressMessage("ReSharper", "[inspection id]")] 。此属性抑制整个类的检查。

    • 检验[检验名称] |禁用一次评论|禁用文件中的所有检查 -此选项插入单个注释ReSharper disable All在文件的开头。此注释禁止显示所有检查文件。
      如有必要，您可以插入ReSharper restore All在特定行之后启用代码检查。

您可以在“ 代码检查索引”中找到每个可配置检查的标识符-只需选择一个语言页面，然后使用浏览器搜索来查找所需检查的详细信息。

要取消对类型或方法的所有检查，请添加以下属性： [SuppressMessage("ReSharper", "All")] 。
