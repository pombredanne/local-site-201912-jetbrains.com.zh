代码检查：冗余的“ else”关键字

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

某些if-else条件可以有他们的else子句在不更改语义的情况下被删除。请考虑以下方法：

public int Sign（double d）{if（d> 0.0）return 1;否则返回-1； }

在上面， else语句可以安全删除，因为它的if子句从方法返回。因此，即使没有else ，您将无法越过if子句主体。
