代码检查：泛型类型的静态成员

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

在绝大多数情况下，具有通用类型的静态字段或自动属性是错误的征兆。这样做的原因是，通用类型的静态成员不会在不同的紧密构造类型的实例之间共享。这意味着对于通用类MyGeneric其中有public static string MyProp { get; set; }的值MyGeneric.MyProp和MyGeneric.MyProp具有完全不同的独立价值观。

如果需要在具有不同通用参数的实例之间共享静态字段，请定义一个非通用基类来存储您的静态成员，然后将通用类型设置为从该类型继承。
