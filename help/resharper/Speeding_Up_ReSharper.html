ReSharper 2019.2帮助

加快ReSharper（和Visual Studio）的速度

您可能还对JetBrains Rider （一个独立的跨平台.NET IDE）感兴趣，它几乎包括ReSharper的所有功能。它以64位进程运行，从而减少了发生内存问题的可能性。

Visual Studio中安装了ReSharper的性能问题主要来自两个方面：

  • 您的系统不符合要求 。在这种情况下，我们建议升级系统作为解决问题的第一步。
    我们不断确保ReSharper在现代硬件以及中型和大型解决方案上都能正常运行，而无需进行任何调整。我们相信Visual Studio开发人员正在朝着相同的方向努力。
    通过禁用某些功能来尝试在过时的硬件上加快ReSharper的使用，您剥夺了可以提高开发性能的出色工具。

  • 共享同一32位进程的Visual Studio和ReSharper将您的系统推向极限。通常，据报道这是在大型解决方案上以及将ReSharper安装到Visual Studio v。2015或更高版本时发生的。
    在这种情况下，只要您的系统满足要求，下面的清单将在大多数情况下帮助您解决性能问题。

第一次打开大型解决方案要花费大量时间（最多几分钟），这并不是问题的迹象。ReSharper构建并缓存该解决方案的模型，然后将该模型用于几乎所有功能-不仅用于代码分析 ，而且还用于导航和搜索 ， 代码完成 ， 单元测试等。
随后打开该解决方案将不会导致任何重大延迟，因为索引结果已经缓存在您的硬盘驱动器上。
请注意，由于ReSharper从一开始就处理程序集批注，因此即使您清理ReSharper缓存 ，任何后续的启动都将更快。

先尝试一下

可以通过以下操作消除导致性能问题的最常见原因：

  • 转到ReSharper |选项|环境|性能指南，用于检查并快速修复影响性能的 ReSharper和Visual Studio 首选项 。

  • 如果不使用解决方案范围的分析 ，请禁用它，或考虑在解决方案范围的分析中禁用警告。即使已禁用它，您也可以通过运行整个解决方案的代码检查 ，随时找到解决方案中的所有代码问题。要配置解决方案范围的分析，请转至ReSharper |选项|代码检查|设置 。

  • 在Visual Studio选项中，转到 源代码管理|插件选择 ：为源代码控制插件选择无 。这将关闭Git或其他VCS提供程序，并改善整体性能。

硬件清单

  • 确保没有硬件中断和DPC（通常是由错误的驱动程序和/或虚拟化引起的）。

  • 确保硬盘驱动器没有碎片。

  • 确保页面文件至少为1GB。

  • 确保您至少有15％的可用磁盘空间（MFT碎片/空间不足风险）。

  • 确保至少有4GB可用内存。

  • 将解决方案和ReSharper缓存存储在SSD上会有所帮助，而RAMDisk不会有太大变化。

推荐的系统软件调整

  • 使用Microsoft当前支持的Windows版本

  • 保持Windows更新

  • 使用64位操作系统

  • 添加 将devenv.exe ， msbuild.exe ，项目/解决方案文件夹和ReSharper缓存文件夹（默认为％LOCALAPPDATA％\ JetBrains \ Transient和可配置 ）保存到Windows Defender（和其他防病毒软件）的忽略列表中。

  • 停止不必要的服务和流程

  • 从Windows索引中排除您的项目/解决方案文件夹和ReSharper缓存文件夹

视觉工作室

在开始调整Visual Studio设置之前，请检查是否已安装最新的Visual Studio更新/服务包/热修复程序。

配置Visual Studio首选项

打开Visual Studio选项（“ 工具” |“选项” ），并如下配置首选项：

  • 环境|常规 ：禁用基于客户端性能自动调整视觉体验 ，禁用启用丰富的客户端视觉体验 ，启用（ 如果可用）使用硬件图形加速 。这些调整将减少UI滞后并改善整体性能。

  • 源代码管理|插件选择 ：为源代码控制插件选择无 。这将关闭Git或其他VCS提供程序，并改善整体性能。

  • 环境|启动 ：选择在启动时显示空白环境并禁用下载内容。关闭起始页面和新闻频道可能会节省一些启动时间。

  • 环境|自动恢复 ：禁用保存自动恢复信息 。 尽管在崩溃的情况下不断复制打开文档的当前状态可能很有用，但是在大型解决方案中，它可能会以UI冻结为代价（例如，请参见此处的注释）。因此，我们建议禁用此功能并养成在进行任何重要更改后保存文件的习惯。

  • 文字编辑器|常规 ：禁用跟踪更改 。 启用“跟踪更改”后，Visual Studio将为编辑的行添加装订线高亮显示。如果当前文件中的编辑太多，编辑器可能会变慢，因为经常需要计算这些突出显示。
    尽管这是一项有用的功能，但我们建议禁用它，尤其是在处理大文件时。

  • 文字编辑器|所有语言|滚动条 ：禁用在垂直滚动条上显示注释 。 ReSharper 广泛使用右装订线或垂直滚动条显示设计时检查通知 ， 结构搜索和替换结果 ， 待办事项以及文件中符号的用法 。Visual Studio还会在滚动条上显示很多通知，有时Visual Studio和ReSharper会报告相同的错误，该错误被显示两次，并且滚动条上经常出现标记。
    如果您更喜欢ReSharper标记，则可以完全或部分禁用Visual Studio批注并通过不呈现它们而获得一些性能上的好处。
    或者，您可以在滚动条上禁用ReSharper的标记—转到ReSharper |选项|编辑器编辑器外观，然后选择不显示标记栏 。

  • 文字编辑器|所有语言| CodeLens ：禁用CodeLens。 CodeLens是大型解决方案的几个性能考虑因素之一，因为“ IDE滚动到屏幕上时，基本上会为每种方法执行“查找所有引用”操作” 。因此，如果您发现任何滞后，我们建议禁用它。
    ReSharper提供了CodeLens的某些功能-例如，您始终可以快速获取该符号的所有引用或用法 Alt+F7 。

  • 调试|即时 ：禁用Script的即时调试。这将加快构建和调试速度。

  • Web表单设计器|常规 ：禁用Web窗体设计器。这将减少UI和编辑器的滞后。

  • XAML设计器|常规 ：禁用XAML设计器。 众所周知， Visual XAML设计器是一种非常消耗资源的功能 。因此，如果您可以使用XAML代码而不使用XAML表单的可视表示形式，则强烈建议禁用XAML设计器。

  • 文字编辑器| [语言] |格式化 ：禁用所有自动格式化首选项。如果您使用ReSharper的格式设置帮助 ，这将防止您的代码重新格式化两次并减少编辑器延迟。

  • 文字编辑器| C＃ ：如果您使用“ 文件结构”窗口 ，那么您可能不使用编辑器顶部的导航栏，并且可以在此选项页面上安全地禁用它。

  • 文字编辑器| C / C ++ |高级 ：如果使用的是C / C ++代码，请设置IntelliSense |禁用“语义着色”首选项True 。这将改善大型文件内的打字体验 。

  • 环境|同步设置 ：禁用设置同步。这将改善整体性能。

其他Visual Studio调整

  • 从Visual Studio卸载未使用的程序包和扩展。
    这里值得一提的是，有一些Visual Studio扩展，包括默认情况下启用的扩展，它们提供的功能类似于ReSharper。例如，如果您使用ReSharper的单元测试功能，则可以安全地禁用Visual Studio单元测试模块。

  • 如果您不在某些项目上工作，则可以从Visual Studio卸载它们，并在需要时将它们重新加载回去。在解决方案资源管理器中右键单击项目或解决方案文件夹，然后选择“ 卸载项目”或“ 在解决方案文件夹中卸载项目”，这将提高整体性能。

锐化器

在开始调优ReSharper之前，请检查您是否正在运行最新版本 ，否则请考虑进行更新。

配置ReSharper首选项

ReSharper提供了许多不同的功能，如果需要，您可以在ReSharper选项中禁用其中的大多数功能。因此，这里的一般规则是：如果您有任何性能问题，请禁用不使用的功能。

打开ReSharper选项（ ReSharper | Options ）并按如下方式配置首选项：

  • 代码检查|设置 ：在此选项页上，您可以禁用代码检查的各个方面。您可以从最小的禁用功能开始禁用以下功能：

    1. 在Visual Studio 2015和更高版本中，请确保已选择“不显示Visual Studio 灯泡”，并且清除了将Visual Studio灯泡动作合并到ReSharper灯泡中。 ReSharper可以显示集成在其自己的操作列表中的 Visual Studio灯泡操作。这在性能方面是昂贵的，因为ReSharper必须向Roslyn请求可用的操作，这可能导致CPU和内存使用量增加。
      尽管所有对Roslyn的请求都是在后台线程中进行的，但是如果存在任何明显的性能问题，您可能希望退出Visual Studio操作集成。

    2. 如果不使用解决方案范围的分析 ，请禁用它，或考虑在解决方案范围的分析中禁用警告。即使已禁用它，您也可以通过运行整个解决方案的代码检查 ，随时找到解决方案中的所有代码问题。

    3. 如果您的解决方案中有不需要分析的任何源代码， 请将相应的文件和文件夹添加到忽略列表 。

      您可以通过按以下按钮将当前文件添加到忽略列表中Ctrl+Shift+Alt+8 。再次按下快捷方式将重新启用对此文件的检查。如果要为此操作绑定其他快捷方式，请查找ReSharper_EnableDaemon Visual Studio选项中的命令。

    4. 如果您有包含第三方代码或冻结的代码中的一些项目 ，这是不会改变的，您可以排除索引这些项目 。

    5. 最终步骤是清除“ 启用代码分析”复选框。这将禁用设计时检查 ，但是您仍然可以在需要时在所需的范围内运行代码检查 。

  • 环境|产品和功能 ：在这里您可以禁用不使用的ReSharper功能。例如，清除“ 单元测试”复选框将禁用所有ReSharper单元测试功能并节省一些内存。

  • 环境|常规 ：在“ 存储解决方案在以下位置缓存”选择器中，选择“系统温度”或任何自定义文件夹。如果您的解决方案不在VCS下，则还可以选择“解决方案文件夹”。 建议选择一个自定义文件夹。“用户本地设置文件夹” ％LOCALAPPDATA％的问题在于，当ReSharper写入此文件夹时，可能会触发Visual Studio的目录监视程序，这种情况经常发生。

  • 环境| IntelliSense ：如果键入速度变慢，则可能需要配置它。与代码检查类似，您可以禁用ReSharper IntelliSense的特定部分，也可以将其完全关闭，然后重新使用本机Visual Studio的IntelliSense。您可以从最小的禁用功能开始禁用以下功能：

    1. 环境|智能感知完成外观 ：清除显示成员签名 ， 显示符号类型 ， 显示摘要 。这将简化建筑完工清单。

    2. 环境|智能感知常规 ：关闭特定语言的ReSharper的IntelliSense（ 自定义IntelliSense ）或将其全部关闭（ Visual Studio ）。

  • 环境|编辑器编辑器行为 ：禁用分号的 自动格式设置和右花括号的自动格式设置，以避免在键入时重新格式化代码 ；还禁用正确的通用语言特定拼写错误 ，这将关闭某些键入辅助功能 。清除这些复选框将加快键入速度。
    如果使用C ++，还可以禁用“ 在基本完成情况下显示导入项”首选项-这将简化完成情况列表，并因此减少键入时的可能延迟。

    ReSharper C ++提供了C ++支持， ReSharper C ++是一种专用产品，可以单独安装，也可以与ReSharper或ReSharper Ultimate并排安装。

  • 环境|编辑器编辑器外观 ：禁用突出显示当前行 ， 突出显示匹配的定界符 。这可以减少UI和编辑器滞后的机会。

  • 环境|扩展管理器 ：禁用您不使用的ReSharper扩展。

  • 工具|构建 ：启用ReSharper构建 。ReSharper的增量构建可以大大减少构建时间，尤其是对于大型解决方案。

  • 代码编辑|上下文动作 ：您可以在此处禁用对您没有帮助的上下文动作 。

  • 代码编辑| JavaScript |检查 ：如果您使用JavaScript并遇到一些性能问题，请不要在检查代码时启用“ 分析属性”上下文 。

  • 代码编辑|第三方代码 ：您可以在此处添加 C ++，JavaScript，TypeScript，CSS，HTML和JSON文件，文件夹和通配符将被视为“跳过”或“库”。ReSharper将完全忽略“已跳过”的文件，并将“库”文件视为只读文件-为导航而建立索引，但不进行检查 ， 快速修复和重构 。

  • 代码编辑|语言注入 ：在此页面上，您可以禁用您不感兴趣的自动语言注入 。

不要过度使用复杂的UI控件

  • 关闭未使用的单元测试会话 。

  • 关闭未使用的类型 ， 项目和程序集依赖关系图。

  • 如果启用了解决方案范围的分析 ，则可以关闭“ 解决方案中的错误/警告”窗口，因为无论如何您将在状态栏中看到错误/警告的数量。

其他调整

  • 为了加快扩展代码模板的速度，您可以为使用的模板关闭“ 重新格式化和缩短合格的引用”选项：

  • 不使用时，请关闭“ 进程资源管理器”窗口 。

如果没有帮助

如果您已经尝试了上述所有操作，但是性能仍然很差，则可以暂时禁用ReSharper并检查是否是导致速度降低的原因。要禁用/启用ReSharper，请转到“ 工具” |“启用” 。选项| ReSharper Ultimate ，然后单击立即暂停 / 立即恢复 。

如果暂停ReSharper有助于提高性能，但是您仍然希望偶尔将其用于代码分析 ， 代码清理或重新格式化代码 ，则可能需要快捷方式来快速打开和关闭ReSharper。这是操作方法：转到“ 工具” |选项|环境|键盘并找到ReSharper_ToggleSuspended命令，然后按您选择的快捷方式，然后点击分配 。

已知的性能问题

陈旧的缓存

如果您最近更新了ReSharper并观察到使用早期版本打开的解决方案的性能下降，则可以尝试通过清除ReSharper缓存并删除解决方案.suo文件来加快处理速度。

要清除缓存，请转至ReSharper |选项|环境|常规 ，然后单击清除缓存 。

内置的JavaScript文件

如果您的解决方案中有一些内置的JavaScript文件（例如jquery.js ），则可以通过在以下代码的“ 库代码”部分下列出这些文件来从分析中排除这些文件 代码编辑|的第三方代码页 ReSharper选项（ Alt+R,,O ）。

已知的兼容性问题

其他Visual Studio扩展

已观察到以下产品的主要兼容性问题：

  • DevExpress CodeRush / Refactor Pro（不兼容）

  • Telerik JustCode（不兼容）

  • 全番茄视觉辅助

  • 生产力电动工具

使用以下产品观察到性能下降：

  • 某些版本的StyleCop ReSharper插件

  • Visual Studio的PowerCommands

也有关于Web Essentials的报告，它们在编辑.cshtml文件时导致性能降低。如果您受到此问题的影响，请考虑转到“ 工具” |“其他”。选项| Web Essentials并在Enter上将Auto-format HTML设置为False 。

Mac版Parallels Desktop

如果使用Parallels Desktop在Mac上的Windows虚拟机中运行Visual Studio，则ReSharper IntelliSense列表的呈现速度可能会很慢。

如果在您的设置中发生这种情况，请考虑从“连贯”模式切换到“全屏”模式。有关在两种模式之间切换的指导，请参阅此Parallels知识库条目 。
