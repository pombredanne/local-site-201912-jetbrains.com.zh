重新格式化XML文档注释

ReSharper可以根据您的喜好重新格式化XML文档注释 。

请注意，ReSharper不会重新格式化普通注释，例如：
// A plain single-line comment
要么
/* A plain multi-line comment */

要重新格式化XML文档注释，请使用代码清除 。您可以使用默认配置文件“ 默认：完全清理” 运行代码清理 ，也可以使用仅针对特定任务的自定义配置文件运行代码清理 。

使用自定义代码清除配置文件重新格式化XML文档注释

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 关于代码编辑XML文档注释|在“格式设置样式”页面上，配置格式设置，以使其符合您的编码惯例或标准。

  3. 转至代码清理设置页面： 代码编辑|代码清理 。

  4. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 选定的配置文件设置”部分中，选中“ 重新格式化嵌入的XML文档注释”复选框，该复选框可用于C＃，VB.NET和JavaScript。（可选）您可以在此配置文件中启用其他代码清除任务。

  5. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  6. 选择您要重新格式化XML文档注释的范围：

    • 将插入符号设置在文件中的任何位置，以将XML文档注释重新格式化为该文件。

    • 在解决方案资源管理器中选择一项或多项，以重新格式化这些节点及其子项下文件中的XML文档注释。

  7. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  8. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  9. 点击运行 。ReSharper将在选定范围内重新格式化XML文档注释。

代码清除不适用于生成的代码。默认情况下，ReSharper提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ）。

如果要在不打开“ 代码清除”对话框的情况下重新格式化XML文档注释，则可以将创建的配置文件绑定到静默清除并按来运行Ctrl+Shift+Alt+F 。您还可以创建一个自定义清除配置文件 ，该配置文件将重新格式化XML doc注释与其他代码样式任务结合在一起。
