在运行时获取组件

例子：

  • SampleSolutionComponent.cs
  • SolutionStateTracker.cs

ReSharper使用松耦合设计。ReSharper的组件模型提供了在运行时实时加载和卸载新组件的功能。

通常，作为插件开发人员，您应仅使用Shell和解决方案组件：

  • Shell组件的寿命与ReSharper相同，并标有ShellComponent属性。
  • 解决方案组件与解决方案具有相同的生命周期，因此，一旦加载解决方案就创建它们，并在关闭解决方案时终止它们。解决方案组件应标有SolutionComponent属性。

那么，如何在运行时获取自己的组件或ReSharper组件？让我们看一下最常见的情况。

构造器注入

最简单的方法是使用构造函数注入。您所需要做的就是将所需的组件指定为构造函数参数。ReSharper将自己关心所有依赖项。

例如，您有一个解决方案组件MyClass在创建时应该执行一些操作。为此，它必须获得ActionManager实例。

[SolutionComponent] public class MyClass { public MyClass(Lifetime lifetime, IActionManager actionManager) { actionManager.ExecuteAction()); } }

这里SomeAction是动作的名称。

当前背景

有时，您可以从IDataContext对象。这是真实的，例如，对于行动 ：

protected override void RunAction(IDataContext context, DelegateExecute nextExecute) { var actionManager = context.GetComponent(); ... }

外壳和解决方案组件

另一种方法是通过Shell component-根API点。例如，如果您要进行一些“全局”设置并从插件内部的任何位置访问它们，这将很有帮助。获得Shell例如，您必须使用Instance属性。

[ShellComponent] public class MySettings { public bool AutoRefresh { get; set; } public MySettings() { AutoRefresh = true; } public static MySettings Instance { get { return Shell.Instance.GetComponent(); } } }

因此，您可以从插件的任何位置简单地访问它们：

if (MySettings.Instance.AutoRefresh) ...

解决方案组件也是如此。主要区别在于您应该有一个实现ISolution界面（指当前在Visual Studio中打开的解决方案的界面）：

public void ObtainSomeSolutionComponent(ISolution solution) { var component = solution.GetComponent(); } ... [SolutionComponent] public class SomeSolutionComponent {}
