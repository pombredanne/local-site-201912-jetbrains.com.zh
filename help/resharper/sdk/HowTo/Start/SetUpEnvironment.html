3。搭建环境

下一步是设置环境，以便您可以调试插件。我们建议您有两种不同的Visual Studio配置单元：一种用于插件开发，另一种用于调试。

设置调试插件的环境：

  1. 将ReSharper安装到单独的Visual Studio 配置单元中 。为此，请在ReSharper安装期间，在“ 选项” |“选项”中指定一个配置单元名称。 实验性配置单元 （在ReSharper安装程序中）。在此示例中，我们使用pluginDebug蜂巢名称。
  2. 运行此Visual Studio配置单元。为此，您需要使用/RootSuffix论点。例如，devenv.exe /RootSuffix pluginDebug

  3. 在调试插件之前，必须首先将其安装到用于调试的Visual Studio配置单元中。通过ReSharper执行此操作|扩展经理… 。请注意，您将需要指定带有插件的文件夹作为自定义源。您可以通过ReSharper |选项... |扩展经理|添加 。

    重要！每次对插件进行更改时，都不必重新安装插件。在大多数情况下，仅复制.dll就足够了（请参阅后续步骤）。但是，如果您的更改影响任何集成点 ，例如操作，菜单或其他集成点 ，则必须重新安装插件。

  4. 关闭此Visual Studio实例。现在，您应该配置插件项目，以便在每次运行项目时将生成的.dll复制到“调试” Visual Studio配置单元中。

  5. 将以下行添加到插件项目的.csproj文件中：

    ReSharperPlatformVs14pluginDebug

    这里：

    • Vs14代表Visual Studio版本（2015）。对于Visual Studio 2017，它将是Vs15 。
    • pluginDebug代表Visual Studio配置单元名称。

    这告诉msbuild将生成的MyPlugin.dll文件复制到pluginDebug Visual Studio配置单元（这些行是特殊CopyBuild步骤的属性）。

  6. 现在，您应该配置Visual Studio在运行调试后在所需的配置单元中运行插件。

    在用于插件开发的Visual Studio中，打开项目的属性。

  7. 在“ 调试”中 ，选择“ 启动外部程序”并指定Visual Studio的路径，例如， C:\Program Files (x86)\Microsoft Visual Studio 14.0\Common7\IDE\devenv.exe 。
  8. 在命令行参数中 ，添加/rootSuffix pluginDebug 。运行调试后，这将在pluginDebug配置单元中运行插件。
  9. 尝试运行您的插件项目。如果一切正常，则必须在构建成功后立即运行带有已安装插件的Visual Studio实例。

而已！最后，您有：

  • 插件项目。
  • 用于构建插件包的NuGet配置。
  • 两个Visual Studio配置单元：一个用于开发，一个用于调试。

您准备开始开发您的第一个ReSharper插件！现在，您应该确定要创建的插件类型。以此为基础，选择与ReSharper集成的方式。
