2。为插件创建一个NuGet包

为插件创建可分发的NuGet包：

  1. 下载并安装NuGet ，例如，到c：\ NuGet 。
  2. 创建一个空白的NuGet规范文件（例如， myplugin.nuspec ）并将其放置在插件的项目文件夹中。

  3. 编辑.nuspec文件：

    MyCompany.MyPlugin 0.0.1 Me Me false My first ReSharper plugin id="Wave" version="[8.0]" /> src="bin\Debug\MyPlugin.dll" target="dotFiles\" /> src="bin\Debug\MyPlugin.pdb" target="dotFiles\" />

    这里重要的是：

    • 的id元数据中的参数必须由两部分组成（公司和插件名称），并用. （点）： MyCompany.MyPlugin在我们的例子中。
    • 指定插件目标的ReSharper版本。因此，Wave 8.0代表ReSharper 2017.1，Wave 9.0代表ReSharper 2017.2，依此类推。请注意，我们对所需的ReSharper版本进行了严格限制（通过使用方括号[8.0] ）– ReSharper SDK不向后兼容！

  4. 通过运行以下命令来构建NuGet软件包：

    nuget pack myplugin.nuspec

    有了打包文件后，就可以设置环境了 。
