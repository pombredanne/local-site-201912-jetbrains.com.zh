开始

ReSharper可以通过两种方式扩展：

  • 声明性扩展 ：通过设置文件分发的扩展，例如实时模板或结构搜索和替换模式。它们不需要任何编程，也不是本“操作方法”的主题。
  • 编译的扩展程序或插件 ：已加载到ReSharper进程中并且几乎可以完全访问ReSharper API的已编译程序集。插件能够执行与ReSharper相同的功能-快速修复，上下文操作，重构，代码注入等。

关于ReSharper插件的事实

  • ReSharper插件是已编译的C＃程序集。因此，它只是一个.dll文件。
  • 该插件必须作为NuGet软件包-.nupkg文件分发。
  • 使用ReSharper将插件安装到Visual Studio中。扩展经理...
  • 如果要让所有ReSharper用户都可以使用该插件，则应将其上传到ReSharper存储库 。
  • 插件取决于ReSharper版本。因此，如果发布了新的主要 ReSharper主版本并希望您的插件支持它，则必须为该特定版本重建插件。此外，您还应考虑到每个ReSharper版本可能会引入一些重大更改。因此，您的插件有可能无法与新版本一起使用。

要开始开发插件，您应该：

  1. 创建一个插件项目 。
  2. 为插件创建一个NuGet包 。
  3. 设置用于运行和调试插件的环境 。
