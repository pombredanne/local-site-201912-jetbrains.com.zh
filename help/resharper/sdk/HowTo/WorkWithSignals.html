处理信号

您应该事先知道：

  • 一生

范例（ ？ ）：

  • SignalEmitter.cs
  • SignalListener.cs
  • SolutionStateTracker.cs-使用信号的更复杂的示例

信号是类提供通知的一种方式。信号与C＃事件非常相似，但是有一些差异使它们更适合在ReSharper数据流基础结构中使用。

public class SignalEmitter { public ISignal SomethingHappened { get; } public SignalEmitter(Lifetime lifetime) { SomethingHappened = new Signal(lifetime, "SignalEmitter.SomethingHappened"); } public void MakeItHappen(bool arg) { SomethingHappened.Fire(arg); } } public class SignalListener { public SignalListener(Lifetime lifetime, SignalEmitter signalEmitter) { signalEmitter.SomethingHappened.Advise(lifetime, arg => MessageBox.ShowInfo($"Something happened and it's {arg}")); } }

笔记

  • 信号必须实现ISignal接口。
  • 要将订阅处理程序“订阅”到信号，请使用Advise方法。信号允许任意数量的侦听器。
  • 与C＃事件不同，您不应该担心信号的“取消订阅”以防止内存泄漏。当您通过“订阅”信号时Advise ，您还将度过一生。生命周期终止后，ReSharper将自行处理“取消订阅”。
  • 要触发信号，请使用Fire方法。
  • 信号与IProperty有很多共同点。
  • 当视图触发事件时，信号非常适合MVC概念。
  • 在简单的情况下，当您仅需要一个用于通知目的的信号（没有有效负载）时，可以使用ISimpleSignal接口。
