终生工作

您应该事先知道：

  • 组件模型

本节仅包含有关生存期的简短详细信息。要了解有关寿命的更多信息，请参阅本指南的相应章节 。

生存期是ReSharper API的基本概念之一。这是ReSharper中用于管理对象生命周期的一种模式，例如，处置不再需要的对象。但是我们为什么不使用IDisposable ？长话短说 - IDisposable当您的对象数量相对较少（通常仅存在于方法中）时，它最适合确定性清理。 using关键词）。资源管理管理代码必须在调用类中实现的事实使得IDisposable模式不是很好的可扩展性。生命周期模式从另一方面解决了问题。而不是返回IDisposable知道如何执行清除的对象，需要执行清除的方法接受Lifetime实例，并注册一个执行清理的回调。因此，只有一个实例维护所有清理代码。

让我们看看它是如何工作的。

获得终生

通常，您应该从ReSharper组件模型中获取组件的生存期。与ReSharper API一样，您可以获取Lifetime使用构造函数注入进行输入。

[SolutionComponent] public class MyClass { public MyClass(Lifetime lifetime, ISolutionStateTracker solutionStateTracker) { solutionStateTracker.AfterSolutionOpened.Advise(lifetime, () => {do somehting...}); } }

如MyClass是解决方案的组件，在Visual Studio中关闭解决方案后将终止生命周期。您可以使用生存期实例将此生存期传播到其他组件。请注意，许多ReSharper API方法都接受Lifetime作为争论。即使您可以选择不提供它，也应该始终选择接受生命周期的方法重载。

如何运作？在上面的示例中，我们订阅了AfterSolutionOpened信号使用Advise 。生命周期实例传递给Advise方法意味着该订阅将一直存在，直到生命周期终止（这将在解决方案关闭时发生）。与取消订阅有关的所有例程将由信号本身处理。因此，通常情况下，您会通过Lifetime实例而不是编写自己的清理代码。

运行您自己的清理代码

但是，您总是有机会运行自己的清理代码。为此，您应该使用来注册清理回调AddAction的方法Lifetime类型。

lifetime.AddAction(() => myObject = null);

或者，您甚至可以使用AddBracket接受两个回调的方法-一个立即运行，另一个在生命周期终止后运行。例如，在工具窗口中添加和删除UI元素可以绑定到特定的生存期：

lifetime.AddBracket( () => _containerControl.Controls.Add(_viewControl), () => _containerControl.Controls.Remove(_viewControl));

自定义生命

通常，应该让组件模型为组件创建生存期。但是，有时可能有必要创建自己的短命寿命。在这种情况下，要定义生存期，应使用Lifetimes.Define方法：

var myShortLifetimeDefinition = Lifetimes.Define(lifetime); var myShortLifetime = myShortLifetimeDefinition.Lifetime;

终止您的一生：

myShortLifetimeDefinition.Terminate();

笔记

  • lifetime是父母的一生。一旦终止， myShortLifetime也终止了。
  • Lifetimes.Define创建一个实例LifetimeDefinition类，依次允许您管理生存期，例如终止生存期。
