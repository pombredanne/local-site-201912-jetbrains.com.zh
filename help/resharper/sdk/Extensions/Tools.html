必备工具

您需要以下工具来构建ReSharper扩展：

  • Visual Studio和ReSharper自然而然。
  • NuGet.exe 。ReSharper扩展是NuGet软件包，因此您需要nuget.exe为了从.nuspec文件。

如果您希望构建已编译的扩展（插件），则还需要：

  • ReSharper的“已检查版本” 。这是ReSharper的内部版本，与生产版本相同，不同之处在于它已编译并启用了附加诊断程序。该代码包含断言，这些断言可以在产品开发过程中验证假设，并且在日常使用过程中不需要，但是在开发扩展时非常有用。此外，通过将信息存储在异常中，当引发异常时，经过检查的构建还包括更多的上下文数据。 Exception.Data字典 。可以从此处下载已检查的版本。
  • ReSharper SDK nuget程序包，用于程序集引用和测试 。这些软件包包含项目的程序集引用，以及用于图标创建，新语言开发等的测试基础结构和msbuild任务。更多详细信息，请参见“ NuGet参考”部分。

选购工具

  • 在内部模式下运行时，某些诊断功能可以创建图形和图表，并将启动yEd图形应用程序以查看它们。
  • 当运行将生成的输出文件与预期的“黄金”文件进行比较的测试时，任何差异都可以显示在kdiff3中 。
