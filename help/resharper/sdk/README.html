开源的

本指南是开放源代码，在Apache 2.0下获得许可。源代码（如Markdown） 托管在GitHub上 。请随时提出问题或创建请求请求，以解决不正确或丢失的内容。请参阅“ 贡献”页面以获取有关如何在本地构建的详细信息。

该指南尚在开发中（ReSharper平台具有非常大的功能集和API表面），并且绝不是API的完整参考。目录提供了可用功能的详尽列表，并通过使页面变灰显示缺少的内容。问题是确定优先处理哪些文档的优先级的好方法。请让我们知道我们应该关注什么！

之前的版本

本指南旨在以ReSharper的当前版本为目标，而不是维护与ReSharper Platform早期版本一起使用的文档。但是，架构和实现的大部分内容都非常稳定，并且文档也可能适用于以前的版本。
