守护进程

您可能已经注意到，ReSharper会对代码进行后台分析。实际上，在任何时候，都有几个不同的分析器在不同的阶段运行，所有这些分析器结合在一起可以提供最完整的代码评估。

守护进程

编写插件时，您很可能会实现自己的守护程序阶段-用DaemonStage实现IDaemonStage接口。

该接口非常简单，并且具有两个成员：

  • 的CreateProcess()方法是创建一个或多个守护程序阶段进程的位置。在每个文档的基础上创建一个守护程序阶段过程，以分析文件并提供有关它们的信息。注意IDaemonStageProcess可以产生几种不同的过程，这些过程可以对应于不同的注入的PSI文档。
  • 的NeedsErrorStripe()方法用于确定此守护程序阶段中的分析是否有助于右侧条纹上的标记以及文档错误列表。仅有三个可能的值ErrorStripeRequest可以在此处返回的枚举。

ReSharper还带有实现的具体基类IDaemonStage （例如， CSharpDaemonStageBase ）-这些减少了CreateProcess()创建单个方法IDaemonStageProcess对应于单个PSI文件。

守护进程阶段

守护程序阶段过程也恰好是一个实现IDaemonStageProcess接口。通常，此类通过守护程序阶段传递，以便可以从该变量获取解决方案或当前文件。

的IDaemonStageProcess接口有两个成员：

  • 一个叫做DaemonProcess返回相应的守护进程。通常，您将返回传递给构造函数的值。
  • 一种称为Execute()进行实际分析的地方。

让我们更详细地讨论Execute方法。如果您继承自诸如CSharpDaemonStageBase ， 你的CreateProcess()覆盖已经具有类型的参数ICSharpFile您可以使用的。如果没有，您可以在PsiManagerExtensions获取文件：

var file = process.SourceFile.GetPsiFiles<CSharpLanguage>();

现在，您已经具有要分析的代码结构，您可以在其上旋转一个元素处理器。元素处理器是实现接口的类，例如IRecursiveElementProcessor ，使您可以在特定代码块中遍历所有树节点。对于递归元素处理器，它具有以下方法：

  • InteriorShouldBeProcessed()确定是否应处理提供的ITreeNode的内部。如果返回true，则处理器将向下钻取内部结构。如果返回false，则处理器将在此级别停止。
  • ProcessBeforeInterior()使您可以在进入树节点的后代之前对其进行处理。
  • ProcessAfterInterior()通过树后代后，您可以对其进行处理。

突出显示

现在，递归元素处理器完成的所有工作的最终结果是一组高亮显示 。突出显示本质上是关于必须突出显示的代码（用波浪线下划线）以及标记栏中显示的信息以及可能出现的文档错误的信息。突出显示也恰好是快速修复的要点。

突出显示全部实现IHighlighting界面，并装饰有与其突出显示类型相关的突出显示属性之一。有两种突出显示类型：

  • 的StaticSeverityHighlighting属性用于指示永不消失的突出显示。这意味着，当你设置它，它就会显示出来。
  • 的ConfigurableSeverityHighlighting让用户配置是否以及何时显示此突出显示。因此，它具有上下文操作，可让您暂时禁用它或编辑其设置。

您自己的突出显示类可以提供一些信息。至少可以指定：

  • 突出显示的严重性。这在严重性属性中定义。
  • 在编辑器中将插入符移至突出显示时的突出显示工具提示。
  • 标记栏的工具提示。

因此-返回到守护程序阶段过程-您已调用元素处理器（或多个），并获得了一组突出显示。最后要做的是使用作为参数提供的提交器来提交结果。 Execute()方法：

commiter(new DaemonStageResult(elementProcessor.Highlightings));

就是这样-您的守护程序已准备就绪！

ElementProblemAnalyzer

如果您正在寻找一种更简单的方法来执行分析并将突出显示添加到代码中， ElementProblemAnalyzer类提出了一个替代方案。如果使用此类，则只需具有一个单一类型，而不必同时定义守护程序和守护程序阶段。

简而言之，元素问题分析器是一类：

  • 继承自ElementProblemAnalyzer哪里T是从继承的类型ITreeNode （例如， IExpression ），随后将对整个文件进行分析。
  • 由装饰[ElementProblemAnalyzer]属性。此属性采用的参数指示分析仪突出显示的元素及其使用的突出显示类型。
  • 实现一个称为Run()分析和突出显示的地方。

的Run()为每个类型的元素递归调用该方法T在文件中找到。例如，如果T == IExpression ， Run()将为找到的每个表达式调用方法。方法签名如下：

void Run(T element, ElementProblemAnalyzerData data, IHighlightingConsumer consumer)

参数为：

  • T element -当前正在分析的元素。在继承该类时定义其类型。
  • ElementProblemAnalyzerData data -包含对生态系统某些有用部分的引用，例如控制IDaemonProcess和设置存储。
  • IHighlightingConsumer -此界面用于将突出显示添加到各种元素。

以下是元素问题分析器的示例实现：

[ElementProblemAnalyzer(new[] { typeof(IExpression) }, HighlightingTypes = new[] { typeof(UseOfInt64MaxValueLiteralHighlighting) })] public class UseOfInt64MaxValueLiteralChecker : ElementProblemAnalyzer<IExpression> { protected override void Run(IExpression element, ElementProblemAnalyzerData data, IHighlightingConsumer consumer) { var value = element.ConstantValue; if (!value.IsLong()) return; long number = Convert.ToInt64(value.Value); if (number != Int64.MaxValue) return; consumer.AddHighlighting(new UseOfInt64MaxValueLiteralHighlighting(element), element.GetDocumentRange(), element.GetContainingFile()); } }
