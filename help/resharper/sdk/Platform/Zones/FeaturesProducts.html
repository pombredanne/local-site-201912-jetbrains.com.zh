功能与产品

特征

由于现在组件模型中的所有组件都被分组为区域，因此可以通过从组件模型的容器中删除区域的组件，将区域视为要素，并允许用户禁用该功能。但是，并非所有区域都应该是功能部件，某些区域显然是实现细节，而其他区域是产品不可或缺的，因此不应禁用。

ReSharper支持此功能以用于特定区域定义。选项对话框上的“功能”页面列出了可以禁用的产品和功能。

通过应用ZoneDefinitionConfigurableFeatureAttribute到区域定义。它需要三个设置-功能名称，简短描述和一个标志，用于说明将区域定义视为功能还是产品。

[ZoneDefinition] [ZoneDefinitionConfigurableFeature("CSS Language", "Support for CSS language in .css and HTML based files", false)] public interface ILanguageCssZone : IPsiLanguageZone, IRequire<ILanguageHtmlZone> { }

这些属性仅由UI使用。禁用区域时，区域定义类型的简称将保存到设置中。如果要素区域具有从属区域（即，其他区域取决于要素区域），则那些从属区域（因此要素也将被禁用）。例如，禁用HTML区域也将禁用CSS区域。从属区域定义的简称也被写入设置。

禁用区域需要重新启动Visual Studio，因为所有Visual Studio集成都需要静态注册（例如，程序集注册为MEF导出，动作注册为静态Visual Studio命令）。

产品展示

区域定义可以通过以下方式视为产品true到isInProductSection的参数ZoneDefinitionConfigurableFeatureAttribute 。这里的区别纯粹是化妆品。通过将此参数设置为true ，选项对话框页面将在“产品”部分而不是“功能”部分中显示区域。该标志的目的是允许使用区域的分层特性为用户提供明显的“关闭开关”，以便禁用产品中的所有功能。通过禁用产品区域，所有从属区域也将被禁用。

产品信息

产品还可以通过以下方式向ReSharper平台公开产品信息： ZoneDefinitionProductAttribute 。它只是一个信息属性，用于通知用户已安装产品的名称，负责的公司并提供指向产品网页的链接。

[ZoneDefinition] [ZoneDefinitionProduct(CompanyName = "JetBrains", CompanyNameLegal = "JetBrains s.r.o.", ProductPresentableName = "ReSharper", ProductTechnicalName = "ReSharper", Version = "9.1", ProductUrl = "http://www.jetbrains.com/resharper", ProductIcon = typeof(ReSharperUIThemedIcons.ReSharper16x16))] public class IReSharperProductZone : IZone { }
