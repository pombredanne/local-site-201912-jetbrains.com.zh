主题图标

ReSharper支持在两个轴上设置主题图标-应用程序颜色和源代码符号样式。换句话说，可以根据应用程序的颜色或源代码符号的样式对图标进行主题设置。

应用程序颜色用于匹配宿主应用程序的主题，特别是Visual Studio，并由代表应用程序各部分的图标（工具栏，上下文菜单，选项页）使用。可以使用以下图标集：

  • 颜色 -Visual Studio处于蓝色主题时使用。
  • 灰色 -用于Visual Studio的Light主题。
  • 深灰色 -适用于Visual Studio的深色主题。

ReSharper不会对Visual Studio的主题进行硬编码，而是分析主题颜色以检测高对比度和低对比度的单色主题。如果主题看起来不是单色，则ReSharper将使用“颜色”主题。这有助于ReSharper更好地支持自定义主题。

显示的图标使用源代码符号样式来表示代码元素，例如在代码完成列表，导航窗口或文件结构窗口中。这里有五个选择：

  • Idea -IntelliJ IDEA使用的图标集。
  • 灰色 -与Light Studio主题中的Visual Studio 2012使用的图标匹配。
  • 深灰色 -匹配Visual Studio 2012在“深色”主题中使用的图标。
  • 颜色 -在颜色主题中匹配Visual Studio 2012使用的图标。
  • 旧版Visual Studio-与Visual Studio 2008使用的图标匹配。

默认情况下，这两个主题方面都是自动选择的，这意味着应用程序颜色将匹配环境，而源代码符号将匹配应用程序颜色。但是，用户都可以在选项对话框中覆盖两者。

确定当前主题

当前主题由IconTheme类，可以使用IThemedIconManager.CurrentIconTheme （要么ITheming.CurrentIconTheme ，因为ITheming继承自IThemedIconManger ）。的IconTheme class是一个简单的值类，维护着一个数组IconThemeAspect实例，按优先级顺序排列。

主题方面代表上述两个轴-应用程序颜色和符号样式。应用程序颜色宽高比值由屏幕上的静态值表示GenericIconThemeAspect类：

public static class GenericIconThemeAspect { public static readonly IconThemeAspect Color = new IconThemeAspect(...); public static readonly IconThemeAspect Gray = new IconThemeAspect(...); public static readonly IconThemeAspect DarkGray = new IconThemeAspect(...); }

的PsiSymbolIconThemeAspect类保存源符号样式的值：

public static class PsiSymbolIconThemeAspect { public static readonly IconThemeAspect SymbolsIdea = new IconThemeAspect(...); public static readonly IconThemeAspect SymbolsVs08 = new IconThemeAspect(...); public static readonly IconThemeAspect SymbolsVs11Color = new IconThemeAspect(...); public static readonly IconThemeAspect SymbolsVs11Gray = new IconThemeAspect(...); public static readonly IconThemeAspect SymbolsVs11GrayDark = new IconThemeAspect(...); }

检查时IconTheme.Aspects数组，应将值与中的值进行比较GenericIconThemeAspect和PsiSymbolIconThemeAspect 。

在每个方面中，每个方面可能有多个值IconTheme.Aspects数组。这些是后备值，适用于无法识别或不支持特定方面值的情况。数组按优先级顺序存储，因此应依次检查每个值，并使用第一个识别的图标。

符号样式方面首先列出，因此自定义图标集将首先尝试使用特定的源符号样式，然后使用符号后备，然后使用应用程序颜色和颜色后备。

使用正确的图标主题

检索给定的图标时IconId ， IThemedIconManager通过检查以下内容，将自动使用当前主题IconTheme及其方面。

要确保主题更改时图标已更新，请使用“实时”图像源，或订阅IThemedIconManager.CurrentIconTheme可观察的属性，并从中手动检索新的图像源IThemedIconManager.Icons属性。

如果使用实时图像源，例如IThemedIconManager.Icons[myIconId].LiveImageSource要么IThemedIconManager.Icons[myIconId].LiveGdipBitmapScreenDpi ，那么当主题更改时，observable属性将通知图像更改。

或者，使用WPF ThemedIconViewImage控制或ThemedIcon XAML标记扩展可在主题更改时自动更新图像源。有关更多详细信息，请参见使用图标 。

创建主题识别图标

创建主题感知图标时，每个图像都应通过其适用的主题方面进行标识。公认的应用程序颜色主题方面名称是：

  • 颜色
  • 灰色
  • 深灰色

对于源代码符号：

  • 符号思想
  • 符号VS11颜色
  • 符号VS11灰色
  • 符号Vs11暗灰色
  • 符号Vs08

创建已编译图标部分中列出了如何指定方面名称的详细信息。
