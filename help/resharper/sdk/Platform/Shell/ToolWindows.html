工具视窗

ReSharper的核心能力之一是呈现各种类型的工具窗口。广义上讲，工具窗口有两种类型：一种是预定义其内容结构的工具窗口，另一种是您决定窗口的内容是什么的工具窗口。

通用工具窗口

通用工具窗口通常包含以下内容：

  • 描述符 -一个空的元数据携带类，用于描述工具窗口的详细信息。
  • 注册器 -用于向工具窗口管理器注册工具窗口的组件。
  • 动作 -触发后会弹出工具窗口的动作。
  • 窗口类，如果需要的话。或者，您可以从现有的构建基块中创建UI。

描述符

描述符只是一个从其继承的空类ToolWindowDescriptor并饰以ToolWindowDescriptor属性。所有配置都在属性中进行。这是一个例子：

[ToolWindowDescriptor( ProductNeutralId="KeyboardHelperWindow", Text="Keyboard Helper", Guid="4338B272-3CD7-4DA3-98D2-4AD0025FAB86", Type = ToolWindowType.SingleInstance, VisibilityPersistenceScope = ToolWindowVisibilityPersistenceScope.Global, Icon = typeof(Resources.ServicesThemedIcons.Keyboard), InitialDocking = ToolWindowInitialDocking.Right )] public class KeyboardHelperDescriptor : ToolWindowDescriptor { }

让我们简要回顾一下以上提供的信息：

  • ProductNeutralId-这是工具窗口的标识符。请注意，与Actions不同，此标识符不用于实例化。
  • 文本 -工具窗口标题中显示的文本。
  • Guid-此窗口的唯一标识符。使用nguid实时模板制作一个。
  • 类型 -确定工具窗口是单实例还是多实例。
  • VisibilityPersistenceScope-控制如何记录工具窗口的可见性。工具窗口的可见性可以与特定解决方案相关联，也可以是全局的，即适用于所有解决方案。
  • 图标 -工具窗口中显示的图标的资源标识符的类型。
  • InitialDocking指定显示工具窗口时将停靠在其中的窗口的初始部分。

行动

要创建一个打开工具窗口的动作，只需创建一个继承自ActivateToolWindowHandle ，使用您的描述符类型对其进行参数化（如上所示），并使用ActionHandler属性：

[ActionHandler("MyPlugin.ShowKeyboardHelper")] public class KeyboardHelperAction : ActivateToolWindowActionHandler<KeyboardHelperDescriptor> { }

登记员

注册器是一个组件（可以是外壳程序或解决方案），用于向工具窗口管理器注册工具窗口类。实际的UI创建在这里进行。可以通过两种方式之一创建UI。

第一种选择是简单地使用UserControl （可以是WinForms或WPF）：

[ShellComponent] public class KeyboardHelperRegistrar { private readonly Lifetime lifetime; private readonly ToolWindowClass toolWindowClass; public KeyboardHelperRegistrar(Lifetime lifetime, ToolWindowManager toolWindowManager, IVsUIShell shell, IActionManager actionManager, IShortcutManager shortcutManager, KeyboardHelperDescriptor descriptor) { this.lifetime = lifetime; toolWindowClass = toolWindowManager.Classes[descriptor]; toolWindowClass.RegisterEmptyContent(lifetime, lt => { var window = new KeyboardHelperWindow(shell); return window.BindToLifetime(lt); }); } }

另一种替代方法是在不创建单独的控件类的情况下就地构建UI。为此，您可以使用ReSharper提供的控件（例如RichTextBox ），然后使用相同的BindToLifetime()方法大致相同。
