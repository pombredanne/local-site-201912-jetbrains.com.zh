实施/覆盖成员

ReSharper |编辑生成代码|失踪议员/压倒多数议员
Alt+Insert | 失踪议员/压倒多数议员
ReSharper_GenerateImplementations / ReSharper_GenerateOverrides

对于继承的类型，ReSharper提供了两个相关的代码生成功能：

  • 实现缺少的成员可实现当前类中缺少的任何接口成员或抽象成员。

  • 替代成员的作用与实施缺少成员的作用相同，但是如果适用，它也会覆盖虚拟成员。

通常，您不需要“ 实施缺少的成员”命令，因为ReSharper会突出显示未实现的基本成员为错误的类型，并允许您使用快速修复来生成缺少的成员（ Alt+Enter ）：

根据您的设置，可以用不同的方式生成新成员的存根。例如，他们可以抛出新的NotImplementedException() ，返回默认值或包含将无法编译的代码。您可以在以下位置配置这些首选项和其他首选项 代码编辑|的会员生成页面 ReSharper选项（ Alt+R,,O ）。

实现和/或重写成员

  1. 在要从基本类型实现和/或覆盖成员的类中设置插入符号。

  2. 按Alt+Insert或选择 ReSharper |编辑从主菜单生成代码… 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. 在“ 生成”弹出窗口中，选择“ 缺少成员/替代成员” 。

  4. 在出现的“ 生成”对话框中，选择要实现和/或替代的成员：

    实施缺少成员页面与覆盖成员有两个方面的不同：

    • 它不会显示可覆盖的成员（例如Object此类）。

    • 它不以黑体强调必须执行的成员。

  5. （可选）使用此时应用于所有已实现/重写成员的以下控件（有关配置单个成员的准则，请参阅下一步）：

    • 实施为可让您选择是否将成员实施为：

      • 公众成员

      • 公共虚拟成员

      • 明确将其实施为私人成员

    • 如果已实现的接口中存在仅获取属性，则使属性可变 。默认情况下，ReSharper将生成仅获取实现。使用此复选框可将setter添加到仅获取属性的实现中。

    • 属性为，让您选择是否以以下方式实现属性：

      • 具有默认访问器主体的传统属性。

      • 访问私有字段的传统属性

      • 自动属性

      您可以设置生成属性的默认方式 代码编辑|的会员生成页面 ReSharper选项（ Alt+R,,O ）

  6. 单击完成以完成向导。
    您也可以在“ 代码编辑” |“选项”上单击“ 选项”以查看或修改常见的代码生成首选项。 ReSharper选项的“ 成员生成”页面。
