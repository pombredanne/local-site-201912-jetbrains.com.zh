XAML中的代码检查和快速修复

ReSharper的代码分析的主要功能在XAML中也受支持。您可以在“ 代码分析”部分的相应主题中找到有关这些功能的详细信息。在本节的主要主题中，您还可以找到功能矩阵并检查XAML确切支持的功能。

在本主题中，您可以找到一些在XAML中使用代码分析功能的示例。

通过对当前文档或在任何指定范围内进行代码检查来执行分析。

要浏览XAML可用检查的列表，请打开 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ），然后展开XAML节点。

解决方案范围内的分析

ReSharper不仅分析当前XAML文件中的错误，而且还检查整个解决方案中的所有文件，并在“解决方案中的错误”窗口中显示分析结果。它会考虑文件之间的依赖关系，并在您更改代码时更新结果。

有关更多信息，请参见解决方案范围分析 。
