配置键盘快捷键

可以使用键盘快捷键调用所有ReSharper操作。大多数操作都有默认的快捷方式，您可以直接使用它们。对于其余的操作，可以根据需要分配首选的快捷键。

ReSharper提供了两种默认的键盘快捷方式方案 。

  • Visual Studio此方案旨在最大程度地减少与Visual Studio自己的键盘快捷键的冲突。

  • ReSharper 2.x / IntelliJ IDEA此方案与IntelliJ IDEA及其派生的Web开发IDE中使用的快捷方式共享大多数。它在所有JetBrains开发环境中提供了以键盘为中心的通用体验。

无论选择哪种方案，都可以在以后随时更改单个键盘快捷键。

更改快捷方式和快捷方式

在安装ReSharper之后首次启动Visual Studio时，将打开“ 键盘方案”对话框，您可以在其中选择默认的快捷方式方案。

您可以稍后使用“ 环境” | “ 环境”中的相应选择器来更改选定的方案。 ReSharper选项的“ 键盘和菜单”页面。

当您更改先前选择的方案时，由先前方案设置的键绑定不会被取消。要删除旧的键绑定和/或还原被IntelliJ IDEA方案覆盖的Visual Studio键绑定，请打开Visual Studio选项（“ 工具” |“选项” ），转到“ 环境” |“ 选项” 。键盘 ，然后单击重置 。

无论当前选择哪种方案，您都可以始终在Visual Studio选项中配置单个键盘快捷键。

配置单个键盘快捷键

  1. 选择工具|从菜单中选择选项 ，然后转到环境|键盘页面。

  2. 使用快捷方式表查找命令的别名，您将为其分配新的快捷方式。

    如果某些命令没有默认快捷方式，则可以在相应帮助页面标题下找到其别名。例如， 将“将方法转换为索引器重构” 。

  3. 要找到所需的命令，您还可以执行以下操作之一：

    • 在“ 显示包含命令”字段中键入要查找的命令的名称或名称的一部分。可用命令的列表会缩小，您可以轻松找到所需的命令。

    • 浏览可用命令列表，向下滚动到以以下名称开头的命令名称ReSharper 。
      键盘选项卡中的命令列表获得焦点后，请按Ctrl+r跳到初始的项目r 。

  4. 如果存在任何快捷方式，则所选命令的快捷方式组合将显示在“ 所选命令的快捷方式”列表中。

  5. 您可以删除现有的组合键和/或添加新的组合键。

    • 在“ 所选命令的快捷方式”列表中选择组合，然后单击“ 删除” 。

    • 可以在不同区域中使用相同的快捷键组合，因此您应该指定区域。使用“在以下位置使用新的快捷方式”列表指定区域。然后将插入符号放置在“ 按快捷键”字段中，然后按所需的组合键。点击分配 。

  6. 单击确定 。

解决键盘快捷键冲突

默认快捷方式 （特别是ReSharper 2.x / IntelliJ IDEA方式）中使用了某些键盘快捷方式，它们可能已经映射到其他Visual Studio命令。在这些情况下，ReSharper不会静默替换现有的Visual Studio键盘快捷键。一旦使用了冲突的快捷方式（Visual Studio和ReSharper都使用了该快捷方式） ，就会提示您使用“ ReSharper快捷方式冲突”对话框来解决此冲突：

选择三个可用选项之一：

  • 使用ReSharper命令 ：将冲突的快捷方式绑定到ReSharper命令。

  • 使用Visual Studio命令 ：解决冲突以支持Visual Studio命令。

  • 配置ReSharper命令 ：在Visual Studio“ 选项”对话框中打开“ 键盘”选项卡，以帮助您自定义当前的键盘方案。

无论选择什么选项，都可以通过选择应用到所有ReSharper快捷方式，将选择应用于所有其他冲突的ReSharper快捷方式 。
