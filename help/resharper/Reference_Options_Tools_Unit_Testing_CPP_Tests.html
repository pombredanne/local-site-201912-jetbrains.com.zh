在ReSharper选项的此页面中，您可以调整C ++中单元测试帮助的首选项。

项目

描述

启用Google测试支持

默认情况下，ReSharper添加操作指示器，使您可以直接从编辑器执行Google Test测试，并在解决方案中自动发现Google Test测试 。清除此复选框可禁用此行为。

启用doctest支持

默认情况下，ReSharper添加操作指示器，使您可以直接从编辑器执行doctest测试，并自动在解决方案中发现doctest测试 。清除此复选框可禁用此行为。

启用捕获支持

默认情况下，ReSharper添加操作指示器，使您可以直接从编辑器执行Catch测试，并自动在解决方案中发现Catch测试 。清除此复选框可禁用此行为。

启用Boost。测试支持

默认情况下，ReSharper添加动作指示器，使您可以执行Boost。直接从编辑器进行测试并自动发现Boost。在您的解决方案中测试测试 。清除此复选框可禁用此行为。

在这里，您可以为带有测试的项目指定标准执行设置。这些设定通常在项目属性（ Project Properties | Configuration Properties | Debugging ）中指定，ReSharper将从那里获取它们。
但是，如果未在此处指定这些属性（例如，在库项目，“打开文件夹”或CMake项目中），则可以在此设置页面上定义执行设置。
请注意，如果在项目属性和此页面中都指定了执行设置，则将仅使用Visual Studio项目属性中的设置。但是，您可以选择“ 合并环境”复选框，以将此处指定的环境变量添加到Visual Studio中的环境变量中。

合并环境

如果选中此复选框，则ReSharper将使用Visual Studio和上面“ 环境”字段中的环境变量。
