考虑以下代码：

公共类Foo {私有静态volatile Foo _instance;私有静态只读对象Padlock = new object（）;公共静态Foo GetValue（）{如果（_instance！= null）return _instance;锁定（挂锁）{如果（_instance！= null）return _instance; _instance = new Foo（）; _instance。在里面（）; } return _instance; } private void Init（）{//对象初始化}}

如果我们假设Init()是用于初始化状态的方法Foo ，则上述代码在多线程环境中可能无法按预期运行。

可能存在执行一个线程的情况_instance = new Foo();但尚未执行_instance.Init(); 。如果此时有其他线程调用GetValue() ，该方法将看到_instance不为null，并且未初始化的实例将返回给调用方。

对于上面的代码，有两种方法可以解决此问题。

首先，也是最明显的是重新定位Init()私有构造函数。

第二种是在未检查的变量中执行初始化，然后将其分配给已检查的变量，从而消除了问题。这条路_instance仅在已经初始化时才会变为非null。中的代码lock上面示例中的语句可以重写为：

如果（_instance！= null）return _instance; var temp = new Foo（）;温度在里面（）; _instance = temp;

关于StackOverflow的此答案解释了此模式的其他可能问题，以及原因_instance应该声明volatile 。
