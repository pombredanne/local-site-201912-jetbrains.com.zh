文件头文字

ReSharper |选项|代码编辑|文件头文字

此页面的ReSharper选项允许您指定默认的文件头，当您使用允许插入和更新文件头的配置文件应用代码清除或从模板创建新文件时，ReSharper将作为注释插入到C＃或C ++文件中。

您不必在文件标题文本中包含注释定界符。应用标题文本时，ReSharper会自动为特定语言插入适当的分隔符。

文件标题文字

此字段包含带有可选变量的纯文本，这些可选变量将添加到文件的开头并用注释定界符引起来。

变数

显示在文件标题文本右侧的变量具有不言自明的名称，您可以在标题文本中使用它来使它保持最新状态，而无需手动更改。单击一个变量以将其插入到当前光标位置的标题中。

将文件头包含在名称中

文件头将被包裹的区域的名称。如果将此字段留空，则文件头不会被区域包围。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
