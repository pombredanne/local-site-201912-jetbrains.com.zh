本地化协助

使用C＃和VB.NET代码以及ASP.NET和XAML标记中的资源文件（ .resx ）和资源使用情况时，最熟悉的ReSharper功能可用于解决方案的国际化。

您可以通过清除以下内容的国际化复选框来禁用本地化帮助 环境|的产品和功能页面 ReSharper选项（ Alt+R,,O ）。

ReSharper的代码检查可检测到可以本地化的字符串，资源文件中的问题等。对于大多数问题，它提供了一个或多个快速修复 。

使用ReSharper，您可以从代码文件和资源文件在文化层次结构之间上下导航。ReSharper会提示资源条目覆盖哪些内容以及在何处覆盖它。

有许多与资源相关的重构 ，可大大简化项目的国际化。

本地化帮助的锦上添花是对代码完成（IntelliSense）和快速文档功能的支持。

在这个部分：

  • 本地化检查和快速修复
  • 编码协助本地化
  • 本地化重构
  • 导航
    • 在Resx文件中导航
    • 在代码和Resx文件之间导航
    • 文化之间的导航
    • 查找用法

要在解决方案中查找所有可本地化的字符串，请为相关项目设置Localizable = Yes和Localizable Inspector = Pessimistic ，然后找到任何此类字符串，这些字符串将用下划线突出显示。按Alt+Enter在它上面，然后选择检验[检验名称] |在scope中找到所有此类问题 。
