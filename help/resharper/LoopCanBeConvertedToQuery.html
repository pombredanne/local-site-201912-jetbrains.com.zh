代码检查：循环可以转换为LINQ表达式

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

此检查适用于针对.NET Framework 3.5或更高版本的项目。

当ReSharper确定您要迭代时， IEnumerable与一个for循环，它可以提供将该循环转换为LINQ表达式的功能。例如，以下代码：

int c = 1;为（int i = 0; i <数字。长度; ++ i）c * =数字[i];

可以自动转换为

int c =数字。Aggregate（1，（current，t）=> current * t）;

ReSharper通常足够聪明，可以识别哪些LINQ运算符可以表示循环中定义的操作。例如，如果我们有c += numbers[i]在上面的循环中，ReSharper会将表达式简化为numbers.Sum() 。

逆向转换可与上下文动作一起使用 Alt+Enter 。

但是这种方法的优点是什么？好吧，其中之一是您不必进行与迭代变量有关的记录保存（例外情况是您确实需要迭代变量）。最重要的是，您具有所有LINQ特有的优势 -例如，您可以通过添加以下内容来请求使用并行化： .AsParallel()集合名称后的方法调用。这将指示运行时使用PLINQ（并行LINQ），这可以加快计算速度。
