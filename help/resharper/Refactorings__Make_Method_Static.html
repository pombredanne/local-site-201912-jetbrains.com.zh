使方法/属性静态重构

ReSharper |重构|将方法设为静态…
ReSharper_MakeStatic

通过这种重构，您可以将实例方法或属性转换为静态方法或属性。所有调用，实现和替代都会自动更正。

如果转换后的实例方法使用实例成员，则ReSharper可以帮助您添加表示“ this”的必要参数，以将这些用法应用到其中。如果转换后的实例属性使用实例成员，则无法使用重构。

反向功能在Make Method Non-Static重构中可用。

使方法或属性静态化

  1. 将插入号放在编辑器中的声明或实例方法或属性的用法上，或在“ 文件结构”窗口中选择它。

  2. 请执行以下任一操作：

    • 按Ctrl+Shift+R然后选择“ 设为静态”

    • 右键单击并选择“ 重构”。在上下文菜单中将其设为“静态 ”。

    • 选择 ReSharper |重构|在主菜单中将方法设为静态… 。

    将打开“ 制作静态”对话框。

  3. 如果该方法使用实例成员，则建议保留选项“ 添加'this'作为[类型名称]的参数” 。在这种情况下，ReSharper添加必要的参数以将这些用法应用到。如有必要，您可以不添加参数，也可以将参数应用于某些实例成员用法。在这些情况下，重构后，您将必须手动纠正静态上下文中实例的无效用法。

  4. 要应用重构，请单击下一步 。

  5. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。
