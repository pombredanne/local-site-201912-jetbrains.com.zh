将匿名转换为命名类型重构

ReSharper |重构|转换|匿名到命名类型…
ReSharper_Anonymous2Declared

当您需要本地处理查询结果时，匿名类型非常方便。但是，如果您需要在程序中传递结果，或者有多个查询返回相似的名称/值部分，则可能需要命名类型。

在这种情况下，ReSharper可以将匿名类型的现有用法转换为命名类型并更新用法。如有必要，ReSharper将在整个解决方案中查找并转换类似的匿名类型。在此重构提供的对话框中，您可以自定义创建的类型-指定它应该具有自动属性还是具有后备字段的属性，或者生成相等性和格式化方法替代。

可以将新命名的类创建为当前类型中的嵌套类，也可以创建为当前文件中的顶级类。如有必要，然后可以使用“ 将类型移动到外部作用域”重构和“ 将类型移动到另一个文件”重构来将新类放置在需要的位置。

在下面的示例中，两个类似的匿名类型被转换为BookLIst类，尽管我们仅对其中之一调用重构。在指定范围内找到的所有类型的用法都将相应地修改：

请注意，如果匿名类型初始化程序内部有另一个匿名初始化程序，则无法应用此重构。例如：

var bookList =从书籍中的书籍中选择新的{book。作者，书名=新{本书。标题}};
在这里，您可以将此重构应用于new {book.Title}但不是new {book.Author, title = new {book.Title}} 。

将匿名类型转换为命名类型

  1. 将插入符号放在匿名类型初始化程序或new关键词。

  2. 请执行以下任一操作：

    • 按Ctrl+Shift+R然后选择“ 使用命名类替换匿名类型”

    • 右键单击并选择“ 重构”。在上下文菜单中， 将“匿名类型”替换为“命名类 ”。

    • 选择 ReSharper |重构|转换|在主菜单中与命名类型…匿名 。

    将打开“ 用命名类替换匿名类型”对话框。

  3. 指定新类的名称，并配置以下选项：

    • 范围 ：ReSharper可以在整个解决方案中找到具有兼容对象初始化程序（名称/值部分相似）的匿名类型，或者您可以仅将重构应用于当前方法。

    • Location ：新的命名类型可以在当前类型中创建为嵌套类，或者在当前名称空间中创建为独立类。

    • 如何生成属性 ：ReSharper可以生成读写自动属性，也可以生成具有新类中的后备字段的只读属性。

    • 如何生成方法 ：作为一种选择，您可以选择为创建存根Equals() ， GetHashCode()和ToString()方法在新类中重写。

    • 在“查找结果”中显示处理后的用法 ：如果选中此复选框，则ReSharper会在“ 查找结果”窗口中显示它找到并转换的所有匿名类型。如果您要在整个解决方案中转换相似的匿名类型，则可能需要勾选此复选框。

  4. 要应用重构，请单击下一步 。

  5. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。
