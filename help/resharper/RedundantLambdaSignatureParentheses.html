Code Inspection: Redundant lambda signature parentheses

You can configure this inspection:

  • disable it to ignore all related issues
  • suppress it in any place in the code to ignore specific issues
  • change its severity level to make the detected issues less or more noticeable

If a lambda expression has a single, implicitly typed parameter, parentheses are optional. They are necessary if there are two or more parameters.

ReSharper suggests removing the redundant parentheses to improve code readability:

Suboptimal code

After the quick-fix

Func<string, Type> t = (x) => x.GetType();
Func<string, Type> t = x => x.GetType();
