代码语法样式：命名/位置参数

如果您希望将命名参数用于特定类型的参数，ReSharper可以帮助您实施这种做法。

考虑以下方法调用：

AddCheckbox（true，true，“ AutoFill”，null）;

阅读此代码时，您必须停止研究方法签名，以了解此处发生的情况。 （您可以使用参数信息工具提示或导航到方法声明来完成此操作 ）。使用命名参数，相同的调用变得更加清晰：

AddCheckbox（enabled：true，value：true，label：“ AutoFill”，工具提示：null）;

如果您希望为特定类型指定命名参数，则可以在ReSharper的帮助下强制执行此首选项。

ReSharper可以帮助您在现有代码中安排参数，并在使用时考虑您的偏好产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。

除了在代码中使用命名参数之外，还可以启用镶嵌提示 。

将样式首选项应用于参数

默认情况下，ReSharper建议在所有位置使用位置参数，并且您必须明确指定需要命名参数的参数类型。根据您的喜好，ReSharper 突出显示需要名称的位置参数或应在位置上命名的命名参数，然后建议在范围内进行相应的快速修复或修复 ：

即使您没有参数的样式首选项，也可以随时按Alt+Enter在任何命名或位置参数上，并选择使用相应的上下文操作添加或删除参数名称：

命名/位置参数的代码样式首选项不应用于生成的代码。默认情况下，ReSharper提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ）。

在批量模式下对命名/位置参数强制执行首选项的另一种选择是代码清除 。您可以使用默认配置文件“ 完全清理”或“ 重新格式化并应用代码样式”之一运行代码清理 ，或者创建和运行仅针对特定任务的自定义配置文件 ，如下所述。

使用自定义代码清除配置文件应用属性样式

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 转至代码清理设置页面： 代码编辑|代码清理 。

  3. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 选定的配置文件设置”部分中，选中“ 应用参数样式”复选框。（可选）您可以在此配置文件中启用其他代码清除任务。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  5. 选择要在其中执行首选项的范围：

    • 将插入符号设置在文件中的任何位置，以强制您对文件进行首选项。

    • 在解决方案资源管理器中选择一项或多项，以在这些节点及其子项下的文件中强制执行您的首选项。

  6. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  7. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  8. 点击运行 。ReSharper将在所选范围内强制执行您的首选项。

如果要在不打开“ 代码清除”对话框的情况下排列参数，则可以将默认的“ 重新格式化代码”配置文件绑定到静默清除并通过按以下命令运行它Ctrl+Shift+Alt+F 。您还可以创建一个自定义清除配置文件 ，该配置文件将把自变量与其他代码样式任务结合在一起。

配置命名/位置参数的首选项

您的参数样式首选项使用基于图层的设置机制保存。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

配置参数样式的首选项

  1. 去 代码编辑| C＃|的代码样式页面 ReSharper选项（ Alt+R,,O ）。

  2. 在“ 参数”类别中，选择对特定参数类型使用命名参数还是位置参数。请注意， 文字值是指所有文字（ bool ， int ， double ，依此类推），但字符串除外。字符串首选项在“ 字符串”文字值选择器中定义。
    类别Other适用于所有其他表达式类型，例如条件表达式，空值，二进制，调用或typeof表达

  3. 通过右栏中的“ 使用选择器通知”，您可以设置代码检查的 严重性级别 ，以检测与首选项不同的代码。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

您也可以在编辑器中更改您的首选项，以突出显示相应的问题。

在编辑器中更改命名/位置参数的首选项

  1. 将插入符设置为ReSharper检查中突出显示的代码问题。

  2. 按Alt+Enter或单击插入符号左侧的动作指示器以打开动作列表 。

  3. 在操作列表中，选择“ 检查”“使用首选参数样式...”。配置代码样式 ，然后选择所需的选项：

  4. 您的更改将使用智能保存逻辑进行保存。

  5. 如果需要将修改后的样式首选项保存在共享设置层中 ，请单击“ 配置代码样式”菜单项或按Enter选择它时。ReSharper将打开代码编辑| C＃| ReSharper选项的“代码样式”页面，您可以在其中根据需要修改首选项，单击“ 保存到” ，然后选择所需的设置层。
