描述

包含指令
在include指令中使用正斜杠

使用此复选框可以为include指令选择首选的路径分隔符：正斜杠或反斜杠。

尽可能使用前向声明

如果选择此选项，那么将在头文件中使用正向声明创建新项目。

使用相对于源文件的路径

您可以使用此选择器配置ReSharper在导入具有快速修复或导入符号完成的类型时如何插入源文件的路径。

使用尖括号代替引号

默认情况下，生成的include伪指令添加在引号中，如有必要，您可以使用此选择器指定在尖括号中添加生成的include伪指令的情况。

简历限定词的位置
键入cv限定词

如果选择此选项， const ， volatile和mutable限定符放在类型说明符之前。否则，将它们放在类型说明符之后。

将“ const”放在“ volatile”之前

定义cv限定词的顺序。

初始化
默认指针初始化器

您可以指定0 ， nullptr要么NULL作为指针初始化程序的首选初始化程序，例如： const char *foo = nullptr 。

在成员初始化器中使用统一初始化

默认情况下，ReSharper在生成初始化程序时使用C ++ 03样式（例如，当您生成构造函数或应用将初始化程序插入现有构造函数的快速修复程序时）。您可以选择此复选框以在上述情况下启用统一初始化 。

在NSDMI中首选统一初始化

使用此选项，您可以配置ReSharper在应用其功能时如何生成非静态数据成员初始化（NSDMI）。例如，当您应用快速修复程序时 Alt+Enter在未初始化的field在以下代码中

类测试{int field; };
ReSharper将生成统一的初始化int field{0};启用该选项时，以及int field = 0;如果该选项被禁用。

重构
在“ Introduce Type”重构中插入类型别名而不是typedef

使用此复选框使Introduce Typedef重构使用类型别名代替typedef 。有关更多信息，请参见Microsoft Docs上的别名和typedef（C ++） 。

生成
在生成的函数声明中使用尾随返回类型

如果启用此选项，则各种代码生成功能将使用尾随返回类型创建函数声明，而不是使用经典的返回类型样式。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
