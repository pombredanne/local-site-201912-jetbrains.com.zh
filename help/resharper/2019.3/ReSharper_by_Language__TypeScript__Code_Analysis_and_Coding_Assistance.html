TypeScript中的代码检查和快速修复

ReSharper的代码分析的主要功能在TypeScript中也受支持。您可以在“ 代码分析”部分的相应主题中找到有关这些功能的详细信息。在本节的主要主题中，您还可以找到功能矩阵并检查TypeScript确切支持的功能。

在本主题中，您可以找到一些在TypeScript中使用代码分析功能的示例。

产品还允许您使用JSLint，ESLint和TSLint在编辑器中直接验证和修复JavaScript / TypeScript代码 。

代码检查

ReSharper的代码检查可检测TypeScript文件中的编译器错误和其他代码问题。默认情况下， 在设计时检查 TypeScript代码。如有必要，您还可以在特定范围内运行代码检查：

代码检查行为以及代码生成结果取决于TypeScript项目设置。在Visual Studio 2013及更高版本中，您可以在项目属性编辑器中配置此设置的前两个（右键单击项目，然后选择Properties ）。在Visual Studio 2012中，只能通过编辑项目文件.csproj来手动更改这些属性。

  • ECMAScript版本 TargetEsVersion
    如果版本低于5，ReSharper会检测到get和set作为错误。如果版本为5或更高，则ReSharper会将八进制文字检测为错误，并建议将其替换为十六进制或十进制文字。

  • 允许隐式“任何”类型 NotImplicitAny
    根据此属性，ReSharper要么检测隐式“ any”类型（例如var a; a = 5; ）是否为错误。

  • TypeScriptNoResolve
    如果此选项设置为“ true”，则ReSharper会将外部模块的使用检测为错误。

代码检查行为还取决于“ 代码编辑” | “ 代码编辑” |“配置”中配置的选项。打字稿| ReSharper选项的“代码样式”页面。有关更多信息，请参见TypeScript中的代码样式辅助 。

使ReSharper忽略特定代码

ReSharper允许您以不同的方式忽略特定的文件，文件夹和文件掩码。您可以在以下选项页面上配置忽略的文件：

  • 代码检查|从代码检查中排除文件，文件夹和掩码的设置 。

  • 代码检查|生成的代码可以部分禁用对生成的代码的代码检查 。

  • 环境|搜索和导航以指定“ 按名称搜索”命令应忽略的文件和文件夹。

  • 代码编辑|要添加的第三方代码 C ++，JavaScript，TypeScript，CSS，HTML和JSON文件，文件夹和通配符将被视为“跳过”或“库”。ReSharper将完全忽略“已跳过”的文件，并将“库”文件视为只读文件-为导航而建立索引，但不进行检查 ， 快速修复和重构 。
