代码语法样式：修饰符

类型和类型成员的C＃修饰符可以按任何顺序编写。但是，在整个代码中以类似的方式排列它们是一个好习惯，这可以提高代码的可读性。此外，您可能更喜欢使用或不使用private和internal显式修饰符。

使用ReSharper，您还可以自动为声明或构造函数中分配的且仅具有只读用法的字段添加 “只读”修饰符 。

ReSharper可以帮助您在现有代码中安排修饰符，并在使用时考虑您的偏好产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。

在现有代码中排列修饰符

默认情况下，ReSharper建议使用private和internal显式修饰符，并按特定顺序排列修饰符。如有必要，您可以灵活配置这些首选项。

在批量模式下强制使用所有修饰符首选项的另一种方法是代码清除 。您可以使用默认配置文件“ 完全清理”或“ 重新格式化并应用代码样式”之一运行代码清理 ，或者创建和运行仅针对特定任务的自定义配置文件 ，如下所述。

修饰符的首选项不适用于生成的代码。默认情况下，ReSharper提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ）。

通过代码清除将首选项应用于修饰符

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 转至代码清理设置页面： 代码编辑|代码清理 。

  3. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 选定的配置文件设置”部分中，选中“ 对类型 使用显式/隐式”修改器，对类型成员使用“显式/隐式”修改器以及“ 排序”修改器复选框。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  5. 选择您要布置修饰符的范围：

    • 将插入号设置在文件中的任何位置，以排列文件的修饰符。

    • 在解决方案资源管理器中选择一项或多项，以在这些节点及其子项下的文件中排列修饰符。

  6. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  7. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  8. 点击运行 。ReSharper会在选定范围内排列修改器。

如果您想在不打开“ 代码清除”对话框的情况下排列修饰符，则可以将创建的配置文件绑定到静默清除并通过按以下命令运行它Ctrl+Shift+Alt+F 。您还可以创建一个自定义清除配置文件 ，该配置文件将应用属性样式与其他代码样式任务结合在一起。

配置修饰符的首选项

与修改器相关的首选项是使用基于图层的设置机制保存的。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保留在VCS下，并自动与团队成员共享。

配置修饰符的首选项

  1. 去 代码编辑| C＃|的代码样式页面 ReSharper选项（ Alt+R,,O ）。

  2. 在“ 修饰符”类别中，指定修饰符的所需顺序并设置首选项，以明确使用private和internal修饰符。

  3. 通过右栏中的“ 使用选择器通知”，您可以设置代码检查的 严重性级别 ，以检测与首选项不同的代码。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

通过隐式使用修饰符，我们意味着没有类和类成员的修饰符。也就是说，如果非嵌套类没有修饰符，它将是internal默认;如果类成员或嵌套类没有修饰符，它将是private默认。
