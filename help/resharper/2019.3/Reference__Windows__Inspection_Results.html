检验结果窗口

ReSharper | Windows |视察
Ctrl+Alt+,
ReSharper_ShowInspectionWindow

该工具窗口显示代码分析和浏览的结果。不同的分析和探索工具在单独的选项卡上显示其结果。

代码检查结果

这些结果包括在特定范围内发现的代码问题-例如，在为项目或解决方案运行代码检查 ， 查找类似的代码问题或从文件中打开问题报告时发现的问题 。

此窗口中显示的所有代码问题均具有指示其严重性级别的图标：

  • -错误

  • - 警告

  • -建议

  • -提示

窗口标签

当检查结果在窗口中打开时，它们显示在新选项卡中，这些选项卡显示在左侧。如果窗口的宽度允许，先前打开的选项卡将移至右侧，否则它们将自行卸载以节省内存-但您始终可以通过单击选项卡右侧出现的箭头并单击所需的选项卡来还原它们：

为了防止标签页卸载，请点击图钉图标标签标题旁边。

工具栏控件

控制

名称

描述

刷新

更新窗口内容以反映代码中或外部所做的任何更改。

全部展开 / 全部折叠

展开/折叠当前选项卡中的所有节点。

导航到上一个/下一个项目，并在源代码中滚动。

隐藏或显示窗格，该窗格在使用列表指定的位置（在窗口的底部或右侧）显示所选项目的预览。

出口

点击这个按钮以文本格式导出当前在窗口中显示的数据，或使用下拉选择器以XML或HTML格式导出数据。出现的“ 导出数据”对话框将帮助您将数据保存到文件或将其复制到剪贴板。

筛选问题

单击此按钮以打开“ 过滤器问题”对话框，并为显示的代码问题自定义过滤 器

通过...分组

定义如何对代码问题进行分组。提供了几个选项：

  • 没有

  • 文件

  • 目录和文件

  • 项目结构

  • 发行类别

  • 发行类型

  • 发行类别和发行类型

  • 问题类别和文件

  • 问题类别和项目结构

  • 问题类型和文件

  • 发行类别，发行类型和文件

  • 问题类别和严重性

  • 问题类别，问题严重性和文件

  • 问题严重性

  • 问题严重性和文件

  • 问题严重性和问题类别

  • 问题严重性，问题类别和文件

此控件仅可用于代码问题 。

预选赛

定义应如何显示来电或去电。提供了几个选项：

  • 类

  • 命名空间和类

  • 项目，命名空间和类

此控件仅适用于传入或传出呼叫 。

语境

定义应如何显示值源或值目的地。提供了几个选项：

  • 没有

  • 命名空间

  • 命名空间和类型

  • 命名空间，类型和成员

  • 类型和成员

  • 类型

  • 项目

  • 项目和文件夹

  • 项目，文件夹和文件

  • 文件

此控件仅适用于价值来源和价值目的地 。
