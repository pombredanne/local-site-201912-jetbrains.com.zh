通过代码检查查找代码问题

ReSharper中的静态代码分析称为代码检查 ，它通过使用所有受支持的语言对您的代码进行2300多次单独的代码检查来执行。

代码问题可以通过以下几种方式找到：

  • ReSharper会在设计时自动发现并突出显示在编辑器中打开的所有文件的代码问题。

  • 可以通过解决方案范围的分析将设计时代码检查的范围扩展到整个解决方案 -在这种情况下，您将能够看到解决方案中的所有错误（以及可选的警告）以及打开的文件中的一些其他问题，例如未使用的公共成员。

  • 您可以在指定范围内找到代码问题，该问题可能与整个解决方案一样大。

  • 您还可以从命令行查找代码问题。

处理编辑器中突出显示的代码问题的最简单方法是将插入符号设置在突出显示的代码项上，然后按Alt+Enter或单击出现在代码问题左侧的操作指示器 。ReSharper会在其中列出针对大多数问题的一个或多个快速修复的操作列表 。

您可以使用结构搜索和替换模式扩展默认的代码检查集，并通过使用ReSharper扩展来获得更多用于特定目的的检查。

在这个部分：

  • 代码检查
  • 在设计时检测代码问题
  • 查找特定范围内的代码问题
  • 解决方案范围内的分析
  • 解决方案范围内的代码检查
  • 配置代码检查设置
  • 创建自定义代码检查和快速修复
  • 使用ReSpeller进行拼写检查
  • 价值和可空性分析
  • 馆藏访问分析
  • 字符串文字的代码分析和帮助器
  • 使用注释完善代码检查
    • 源代码中的注释
    • 外部注释
    • 合同注释

如有必要，您可以通过插入特定的代码块来禁用所有检查（检测编译器错误的检查除外） // ReSharper Disable All之前和// ReSharper Restore All之后，或通过添加以下属性以特定类型或成员添加： [SuppressMessage("ReSharper", "All")] 。
