单元测试协助

ReSharper的单元测试运行程序不允许运行两个或多个具有相同ID的测试。在实践中，这种情况很少发生，但是如果您确实有具有相同ID的测试，它们将在“单元测试”浏览器和“单元测试会话”窗口中显示为单个节点，并且仅运行其中一个。
请注意，测试框架以不同的方式识别测试。例如，MSTest使用测试方法FQN，而NUnit使用测试属性的属性作为测试ID。

ReSharper帮助您直接在Visual Studio中发现和运行或调试单元测试。支持以下单元测试框架：

  • NUnit 2.6.2 ...3.10.1。ReSharper的安装包括2个捆绑的NUnit测试运行程序版本（2.6.4和3.6.1）。您可以选择其中之一，也可以选择两者，也可以在以下位置指定NUnit的本地安装： 工具|单元测试|测试框架|的NUnit页面 ReSharper选项（ Alt+R,,O ）。

  • 所有支持的Visual Studio版本的 MSTest

  • 单位

  • 机。规范 （需要扩展 ）要了解有关使用扩展的更多信息，请参阅管理扩展 。

  • QUnit （有关更多信息，请参见JavaScript中的单元测试帮助 ）

  • 茉莉花 1.3。 -2.3。有关更多信息，请参见JavaScript中的单元测试协助。

  • Google C ++测试框架 （有关更多信息，请参阅C ++中的单元测试帮助 ）

  • 促进。测试框架（v。1.59及更高版本） （有关更多信息，请参阅C ++中的单元测试帮助 ）

使用ReSharper，您可以执行单个单元测试，测试类，文件，项目或解决方案中的所有测试。您也可以在测试会话中执行任意数量的测试。

可以使用其他JetBrains .NET产品扩展单元测试帮助：您可以使用dotTrace进行单元测试配置文件，并使用dotCover分析单元测试的代码覆盖率。这些产品也包含在ReSharper Ultimate中 。

在这个部分：

  • 在当前文档中
  • 在项目或解决方案中
  • 使用单元测试会话
  • 测试类别
  • 执行测试并分析结果
  • 使用dotCover分析代码覆盖率
  • 使用dotTrace进行配置文件单元测试
  • 使用dotMemory单元监视内存
