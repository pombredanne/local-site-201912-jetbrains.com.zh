价值和无效性分析模式

ReSharper可以分析不同模式下的值：

  • 乐观的
    默认情况下，值分析模式是乐观的。在这种模式下，ReSharper仅在显式检查了该值是否可为空或符号标记有符号的情况下才警告可能的“空”取消引用。 [CanBeNull]要么[ItemCanBeNull]属性。上面的示例演示了乐观模式。

  • 悲观
    在这种模式下，ReSharper会警告在所有未明确检查该值是否为可空性的情况下可能发生的“空”取消引用，除非用符号标记该符号。 [NotNull]要么[ItemNotNull]属性。

要更改值分析模式或禁用分析，请在以下位置配置“ 值分析模式”首选项 代码检查|的设置页面 ReSharper选项（ Alt+R,,O ）。
