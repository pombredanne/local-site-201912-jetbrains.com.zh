代码检查：带有默认值的冗余参数

您可以禁止执行此检查以忽略特定问题 ， 更改其严重性级别以使问题更小或更明显 ，或者将其完全禁用 。

此检查可以识别代码中调用具有可选参数的方法的位置，并使用与这些参数的默认值相同的值。

这是一个例子：

void Foo（int必需，[可选] bool optionalBool，int optionalInt = 10）{//做某事} void Test（）{Foo（10，false，10）; //警告：带有默认值的冗余参数}

上面的调用对两个可选参数使用与默认值相同的值， optionalBool和optionalInt ，这有点模棱两可：此调用是否应使用在的声明中指定的默认值 Foo()或者相反，是否应该独立于默认值使用调用方指定的值？

如果是前一种情况，则可以并且应该删除具有默认值的可选参数，因为默认值可以在声明中稍后更改。否则，您可以通过以下注释取消检查：
// ReSharper disable RedundantArgumentDefaultValue
