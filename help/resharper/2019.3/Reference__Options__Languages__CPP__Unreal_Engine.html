虚幻引擎

ReSharper |选项|代码编辑| C ++ |虚幻引擎

在此页面中，您可以调整一些首选项，这些首选项会影响处理C ++代码的性能。

项目

描述

索引虚幻引擎源文件

清除此复选框可跳过对虚幻引擎代码的索引。通常，如果您在开发游戏时发现性能下降，则需要这样做。

启用索引后，您将拥有许多功能。例如，使用引擎中的任何符号，ReSharper会自动添加缺少的#include或查找引擎类的用法 ，例如， TArray —在引擎的代码中，研究在那里如何使用它们。

如果禁用了索引编制，则ReSharper仍将为头文件名建立索引，这是非常快的，但是它将使您可以完成包括在内的代码，例如， #include 。包含标题后，其代码将被自动索引，因此您将完成代码完成并分析包含的标题中的符号。

索引第三方代码

如果要跳过索引引擎的ThirdParty文件夹下的源文件，请清除此复选框。

索引插件

如果要跳过引擎的Plugins文件夹下的源文件索引，请清除此复选框。

启用虚幻引擎源文件的后台索引

ReSharper默认情况下在后台将引擎代码编入索引，以便您可以在引擎代码完全处理之前就开始使用游戏逻辑代码。关闭后台索引编制将导致更快但更具侵入性的初始索引编制。
