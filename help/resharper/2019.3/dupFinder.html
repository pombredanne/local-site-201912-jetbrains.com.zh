dupFinder命令行工具

dupFinder是一个免费的命令行工具 ，它可以在C＃和Visual Basic .NET代码中查找重复项-不多也不少。但是作为一个JetBrains工具，dupFinder却以一种聪明的方式做到了。默认情况下，即使它们包含相同的变量，字段，方法，类型或文字，代码片段不仅会相同，而且在结构上相似时也会将它们视为重复。当然，您可以配置允许的相似度以及重复片段的最小相对大小。

JetBrains TeamCity内置了 dupFinder引擎，这可能是使用dupFinder的最简单，最有效的方法。

您也可以从命令行运行代码检查和覆盖率分析 。

运行重复分析

您可以使dupFinder在本地运行，也可以与您的自定义CI，版本控制，质量控制或任何其他服务器一起运行。

要运行dupFinder：

  1. 下载ReSharper命令行工具

  2. 将命令行工具包解压缩到任何目录中。
    解压缩之前，请确保已下载的.zip文件已被“取消阻止”：右键单击该文件，选择“ 属性” ，然后单击“ 取消阻止” 。否则，将导致.NET框架以部分信任的方式加载应用程序，这意味着它将无法正确加载或运行。

  3. 运行以下命令：

    dupFinder.exe -o =

的source上面定义了重复搜索中包含的文件。使用Visual Studio解决方案或项目文件，类似Ant的通配符或特定的源文件和文件夹名称。路径应该是绝对路径，也可以是相对于工作目录的路径。例如：

  • dupfinder.exe c:\MyProjects\MySolution.sln
  • dupfinder.exe "c:\MyProjects1\\**\*.cs"
  • dupfinder.exe **\*
  • dupfinder.exe FooFolder\src BarFolder\src

配置dupFinder选项

使用可选参数，您可以配置dupFinder分析来源的方式：

dupFinder.exe [选项]
要研究选项的完整列表，请运行dupFinder.exe --help 。以下是您可能感兴趣的一些选项：

  • --output-o —使您可以设置输出文件。

  • --exclude-e -允许从重复搜索中排除文件。该值是用分号分隔的通配符（例如， *Generated.cs ）。请注意，路径应为绝对路径或相对于工作目录的路径。

  • --exclude-by-comment -允许排除在开头的注释中具有匹配的子字符串的文件。多个值用分号分隔。

  • --exclude-code-regions —允许排除名称中具有匹配子字符串的代码区域。 （例如，“生成的代码”将排除包含“ Windows Form Designer生成的代码”的区域）。多个值用分号分隔。

  • --discard-fields ， --discard-literals ， --discard-local-vars ， --discard-types —通过这些选项，您可以指定如果相似的片段具有不同的变量，字段，方法，类型或文字，则是否将相似的片段视为重复片段。所有这些的默认值为“ false”。为了说明其工作方式，请考虑以下示例。有两个代码段，否则相同，一个包含myStatusBar.SetText("Logging In..."); ，其他包含myStatusBar.SetText("Not Logged In"); 。如果“ discard-literals”为“ false”（默认情况下为false），则这些片段被视为重复片段。要将此类项目从检测到的重复项列表中排除，您可以使用--discard-literals=true 。

  • --discard-cost -允许为重复片段的代码复杂度设置阈值。具有较低复杂度的片段将作为非重复片段丢弃。此选项的值以相对单位提供。它是使用内部算法计算得出的结果，该算法基本上会构建所分析代码的语法树（可以在文件，项目或解决方案上构建）。因此，该值与该树或其相应分支的大小成正比。它有点类似于“循环复杂性”。
    使用--discard-cost选项，您可以过滤掉没有语义重复的相等代码片段。例如，您经常在测试中可以包含以下语句： Assert.AreEqual(gold, result); 。如果“丢弃成本”值小于10，则类似的语句将显示为重复项，这显然无济于事。您需要对此值进行一些尝试，以在避免误报和丢失真实重复项之间找到平衡。对于不同的代码库，正确的值将有所不同。

  • --properties —使您可以覆盖MSBuild属性。您可以分别设置每个属性（ --properties:prop1=val1--properties:prop2=val2 ），或使用分号分隔多个属性--properties:prop1=val1;prop2=val2 。指定的属性将应用于所有分析的项目。当前，没有直接的方法仅将属性设置为特定项目。解决方法是在此项目中创建一个自定义属性，并将其分配给所需的属性，然后在dupFinder参数中使用该自定义属性。

  • --normalize-types —允许将类型名称规范化为输出中的最后一个子类型（默认值：False）。

  • --show-text —如果使用此参数，则检测到的重复片段将被嵌入到报告中。

  • --config-create和--config —这些选项使您可以将上述参数与配置文件一起传递。第一个选项将根据当前参数创建一个配置文件。第二个选项用于从该文件加载参数。

  • --debug (-d) —使用此选项可将dupFinder的执行详细信息添加到输出中。如果您在dupFinder上遇到问题，请与支持小组联系时使用这些详细信息。

  • --verbosity —默认情况下，dupFinder仅在日志中显示信息消息。使用此参数可以按以下级别更改写入日志的信息量（顺序从小到详细）： [OFF, FATAL, ERROR, WARN, INFO, VERBOSE, TRACE] 。
    例如，如果dupFinder出现问题，则可以联系ReSharper支持并与所有TRACE消息共享日志文件： --verbosity=TRACE 1> [path_to_log_file] 2>&1 。

这是使用可选参数运行dupFinder的示例：

dupfinder.exe --discard-literals = true --caches-home =“ C：\ Temp \ DFCache” --o =“ report.xml” --discard-cost = 50“ C：\ src \ MySolution.sln”

了解输出

dupFinder分析的输出是一个XML文件，该文件显示以下信息：

  • 的Statistics节点是所分析代码的概述，其中CodeBaseCost是目标源代码的相对大小， TotalFragmentsCost是应用过滤器（“ discard-cost”，“ discard-literals”等）之后用于分析的代码的相对大小，并且TotalDuplicatesCost是检测到的重复项的相对大小。

  • 的...Cost值是使用内部算法计算得出的结果，该算法基本上会构建所分析代码的语法树（可以在文件，项目或解决方案上构建）。因此，该值与该树或其相应分支的大小成正比。它有点类似于“循环复杂性”。

  • 的Duplicates节点包含Duplicate节点，这些节点又包含两个或多个Fragment表示实际代码重复的元素。

  • 每Duplicate节点有一个Cost属性-成本较高的重复项最为重要，因为它们可能会带来更大的问题。

  • 每Fragment元素包含文件名以及以两种替代方式显示的重复片段-作为符号中的文件偏移范围和行范围。如果--show-text选项启用了分析，然后Text具有重复代码的节点被添加到每个片段。
