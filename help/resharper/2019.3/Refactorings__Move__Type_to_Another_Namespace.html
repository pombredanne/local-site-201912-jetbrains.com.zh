将类型移动到另一个命名空间重构

ReSharper |重构|移动…
F6
ReSharper_Move

此重构可帮助您将一个或多个非嵌套类型移动到新名称空间或现有名称空间。命名空间声明将在原始文件中替换，所有用法都会相应更新。

如果需要移动嵌套类型，则可以首先将“ 移动类型”应用于“外部作用域重构” 。

将一个或多个类型移动到另一个名称空间

  1. 通过以下方式之一选择要移动的类型：

    • 在编辑器中，将插入号设置为类型的名称。

    • 在解决方案资源管理器中选择一种或几种类型和/或文件和/或文件夹。

    • 在“ 文件结构”窗口中选择一种或几种类型。

    • 在“类视图”中选择一种类型。

    • 在对象浏览器中选择一种类型。

    • 在类型依赖关系图中选择一种类型或文件夹。

  2. 请执行以下任一操作：

    • 按F6然后选择将类型移动到另一个命名空间

    • 按Ctrl+Shift+R然后选择将类型移动到另一个命名空间

    • 右键单击并选择“ 重构”。在上下文菜单中将“ 类型”移动到“另一个命名空间 ”。

    • 选择 ReSharper |重构|在主菜单中移动...。

    将打开将类型移动到另一个命名空间对话框。

  3. 开始输入目标名称空间的名称。如果要将类型移动到现有名称空间，请从显示所有与您输入匹配的名称空间的下拉列表中选择它。

  4. 要应用重构，请单击下一步 。

  5. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。
