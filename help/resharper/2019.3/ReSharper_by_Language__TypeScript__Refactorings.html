TypeScript的重构

除了重构之外，ReSharper还提供了 Context Actions可用的许多较小的本地代码转换 。

ReSharper为TypeScript提供以下重构：

  • 复制类型
  • 介绍领域
  • 介绍类型别名
  • 内联类型别名
  • 介绍变量
  • 为子字符串引入变量
  • 内联变量
  • 移动到另一个文件
  • 移到文件夹
  • 移至资源
  • 将类型移到另一个模块
  • 改名

执行重构

  1. 将插入符号设置为符号，选择要重构的代码片段，或在工具窗口中选择一个项目。

  2. 请执行以下任一操作：

    • 在主菜单中，选择“ ReSharper |重构 ，然后选择所需的重构。此菜单中可用的重构列表取决于当前上下文。如果ReSharper无法为上下文建议任何重构，则将禁用整个菜单。

    • 在编辑器的“ 文件结构”窗口或其他“ ReSharper”窗口中，右键单击要转换的项目，从上下文菜单中选择“ 重构 ”，然后选择所需的重构。

    • 在主菜单中，选择“ ReSharper |重构|重构此 ，或按Ctrl+Shift+R以显示适用的重构列表，然后选择其中之一。您也可以在选择的上下文菜单中选择“ 重构此 ”。

    • 使用分配给特定重构的默认键盘快捷方式 ，或将自定义快捷方式分配给您喜欢的重构命令。

  3. 如果所选的重构需要用户输入，则重构向导将打开。请注意，向导的对话框不是模态对话框，因此您可以在向导打开时编辑代码。

    要回滚重构操作，向导提供选项“启用撤消，打开所有具有更改的文件以进行编辑” 。如果选择此选项，ReSharper将在新的编辑器选项卡中打开所有修改的文件，并使您能够回滚重构。在这种情况下，您需要自己保存更改。如果未选择此选项，则ReSharper会自动保存已修改的文件，而无需打开它们。

  4. 如果重构操作将导致代码冲突（例如重复的名称，可见性冲突等），则在应用重构之前，向导将在最后一步显示冲突列表。对于某些冲突，向导还可以建议快速修复。有关更多信息，请参见解决重构中的冲突 。

Visual Studio提供了自己的“ 重构”菜单，可从主菜单和上下文菜单中使用。默认情况下，ReSharper隐藏本机Visual Studio菜单，将其替换为自己的菜单。要使Visual Studio菜单可访问，请在“ 环境” | “ 环境”中选择“ 隐藏覆盖的Visual Studio菜单项”复选框。 编辑器 ReSharper选项的“ Visual Studio功能”页面。

在编辑器中修改代码后，可以立即进行某些重构。有关更多信息，请参见就地重构。

复制类型

通过此重构，您可以创建所选类型的副本，并将其放入指定的模块中。

介绍领域

通过此重构，您可以基于所选表达式创建新字段，使用表达式或从构造函数对其进行初始化，并使用对新引入字段的引用替换当前类型中出现的表达式。

您还可以使用专用快捷方式调用此重构Ctrl+Alt+D 。

介绍类型别名

此重构可帮助您为类型组合创建类型别名，并用别名替换当前选择的组合或当前作用域中的所有类似组合。

要调用此重构，请选择可作为别名的类型交集或并集，文字或其他实体，然后按Ctrl+Shift+R或选择 ReSharper |重构|在菜单中重构… ，然后在“ 重构此”弹出窗口中选择“ 引入类型别名 ”。如果在当前上下文中出现了多个所选实体，则可以选择是替换当前实体还是替换所有实体。

内联类型别名

此重构可帮助您删除所选的类型别名，并用其定义替换其所有用法。

要调用此重构，请选择类型别名的声明或用法，然后按Ctrl+Shift+R或选择 ReSharper |重构|在菜单中重构… ，然后在“ 重构此”弹出窗口中选择“ 内联类型别名 ”。

介绍变量

通过此重构，您可以基于选定的表达式创建新的局部变量或常量，使用表达式对其进行初始化，最后使用对新引入的变量的引用替换该方法中所有出现的表达式。

您还可以使用专用快捷方式调用此重构Ctrl+Alt+V 。

内联变量

通过这种重构，您可以用其初始化程序替换代码中所有出现的变量。请注意，只有在变量值在初始化后保持不变的情况下，才应应用重构。

您还可以使用专用快捷方式调用此重构Ctrl+Alt+N 。

移动到另一个文件

此重构可帮助您将所选类型从当前文件移动到任何现有文件或新文件。如果要移至新文件，则会自动创建新文件。如果当前文件中没有其他类型，则可以将其删除。所有必需的导入都随类型一起移动。所有不再需要的导入都将从原始文件中删除。

移到文件夹

此重构可帮助您将一种或多种类型或文件移动到解决方案中任何位置的另一个项目或文件夹。如有必要，重构将为您创建新的目标文件夹。移动的类型所需的所有导入都被固定在新位置。所有不再需要的导入都将从原始文件中删除。

移至资源

在从Visual Studio模板（例如Apache Cordova）创建的TypeScript项目中，ReSharper允许将字符串文字移动到资源文件中 。ReSharper可以选择在所需范围内找到所有相同的字符串，并将其替换为资源使用情况。要执行此重构，您需要在项目中至少有一个资源文件（通常具有.resjson扩展名）。

您还可以使用专用快捷方式调用此重构F6 。

将类型移到另一个模块

此重构将模块声明包装在所选类型周围。

改名

TypeScript支持最耗时的重构之一。如果尝试手动修改符号名称，可能会导致很多问题。调用重命名重构时 （也可用于专用F2快捷方式），所有检查均由ReSharper完成。如果不存在冲突，则所有修改都可以顺利进行，或者获得可以手动解决的冲突列表，以确保仅进行必要和适当的更改。
