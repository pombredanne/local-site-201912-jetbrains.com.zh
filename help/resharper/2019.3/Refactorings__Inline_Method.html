内联方法重构

ReSharper |重构|内联|排队…
Ctrl+Alt+N
ReSharper_InlineVariable

此重构可帮助您用其主体替换静态，实例和扩展方法的用法，并有选择地删除原始方法声明。
您也可以将此重构应用于具有后备字段的属性。在这种情况下，读取用法被getter主体替换，写入用法被setter主体替换。

在下面的示例中，我们使用此重构来内联LogError方法：

重构后

抽象类Shape {public void Draw（）{try {// draw} catch（Exception e）{LogError（e）; }} private static void LogError（Exception e）{文件。WriteAllText（@“ c：\ Errors \ Exception.txt”，例如ToString（））; }}
抽象类Shape {public void Draw（）{try {// draw} catch（异常e）{文件。WriteAllText（@“ c：\ Errors \ Exception.txt”，例如ToString（））; }}}

反向功能可在“ 提取方法”重构中使用。

内联方法

  1. 将插入号放在编辑器中的声明或方法的用法上，或在“ 文件结构”窗口中选择它。

  2. 请执行以下任一操作：

    • 按Ctrl+Alt+N然后选择“ 内联方法”

    • 按Ctrl+Shift+R然后选择“ 内联方法”

    • 右键单击并选择“ 重构”。上下文菜单中的内联方法 。

    • 选择 ReSharper |重构|内联|内联…在主菜单中。

    内联方法对话框将打开。

  3. 如果在方法用法时调用了重构，则可以清除“ 内联所有用法”复选框以仅内联当前用法。

  4. （可选）清除“ 删除内联方法声明”复选框以保留原始方法声明。

  5. 要应用重构，请单击下一步 。

  6. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。
