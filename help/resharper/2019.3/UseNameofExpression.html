代码检查：使用“ nameof”表达式引用名称

您可以禁止执行此检查以忽略特定问题 ， 更改其严重性级别以使问题更小或更明显 ，或者将其完全禁用 。

此检查建议您应该使用C＃6.0功能。 默认情况下，ReSharper根据关联的编译器自动检测C＃版本。但是，您可以显式指定目标C＃版本-在解决方案资源管理器中右键单击项目，从上下文菜单中选择“ 编辑项目项属性” ，然后使用C＃语言级别选择器。

在.NET库，有很多的，旨在接受的类型，成员或变量的名称作为参数的方法。最常见的例子之一是ArgumentNullException(string paramName) 。如果将字符串文字用作此类方法的参数，则当重命名相应符号时文字拼写错误或未更改时，可能会出错。与任何有效符号都不对应的字符串文字不会被编译器捕获。

在C＃6.0中添加的nameof运算符解决了这一问题-它允许捕获范围内符号的字符串名称。

在下面的示例中，ReSharper建议替换字符串文字"order"在ArgumentNullException()与nameof(order) 。结果是一样的： nameof(order)返回字符串“ order”，但带有nameof您的代码变得不太容易出错。

快速修复后

public void OrderErrorHandler（object order）{if（order == null）throw new ArgumentNullException（“ order”）; / * ...* /}
公共无效OrderErrorHandler（对象顺序）{如果（order == null）抛出新的ArgumentNullException（nameof（order））; / * ...* /}
