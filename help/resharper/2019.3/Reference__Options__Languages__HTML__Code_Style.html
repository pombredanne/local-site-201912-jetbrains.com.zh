Code Style

ReSharper | Options | Code Editing | HTML | Code Style

Preferences configurable on this page are taken into account when ReSharper produces new code with code completion and code generation features, applies code templates and performs refactorings. They can also be applied to the existing code by using code cleanup with the corresponding settings.

Item

Description

Quotes

The preferences in this section define whether apostrophe (') or quote (") should be used as a quote symbol for attribute values, and whether to remove attribute quotes for attributes without spaces.

Other

The Normalize tag names checkbox defines whether to use lowercase letters for tags.

ReSharper's settings are saved in settings layers, which enable you to have different settings for different solutions and to share your settings with your team. To learn more, see Manage and Share ReSharper Settings.
