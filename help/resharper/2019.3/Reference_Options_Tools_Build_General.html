ReSharper 2019.3帮助

一般

ReSharper |选项|工具|建立一般

在ReSharper选项的此页面中，您可以配置ReSharper Build的首选项。

项目

描述

构建引擎
使用ReSharper构建

如果选中此复选框，ReSharper将使用以下命令优化构建过程 ReSharper Build ，它跟踪解决方案中的更改，并且仅重建修改后的项目和必要的依赖项。在这种情况下，将在所有构建命令上调用ReSharper Build，例如Build |。建立解决方案 Ctrl+Shift+B以及解决方案资源管理器中项目和解决方案文件夹上的相应上下文菜单命令。
如果未选中此复选框，则将使用本机Visual Studio生成管理。如果您的解决方案包括非MSBuild项目或具有自定义生成步骤的项目，则可能需要禁用此选项。

加载解决方案后运行构建

此自解释选项将独立于所选的构建管理首选项来应用。

构建参数
使用MSBuild.exe版本

使用此选择器，可以选择用于生成解决方案的MSBuild.exe版本。您可以选择一种自动检测到的MSBuild版本，也可以在非标准位置指定自定义MSBuild可执行文件的路径。

并行使用多达X个进程

使用此选择器，您可以选择用于构建解决方案的并行进程数。

在构建之前还原NuGet软件包

选中此复选框后，ReSharper将使用ReSharper的专有机制为当前解决方案中的所有项目还原NuGet软件包 ，该机制可在所有受支持的Visual Studio版本中使用。请注意，此选项要求在计算机上安装.NET Framework 4.5或更高版本。

将日志写入输出 选中此复选框后，在其下方选中的日志项目将显示在Visual Studio的“ 输出”窗口中 。
笔记：

  • 要在构建项目时显示“输出”窗口，请转到“ 工具” |“工具”。选项|项目和解决方案|常规并在构建开始时选择显示输出窗口 。

  • 如果您希望日志项根据其来源和种类而具有不同的颜色，请在此页面上方选择“ 绘制输出日志” 。

  • 您还可以配置如何从NuGet的日志项目在“ 工具” | “输出”中显示。 建立 ReSharper选项的NuGet页面。

写入日志文件

选中此复选框后，构建输出将保存到日志文件。使用此复选框下的控件来选择详细级别和保存日志文件的路径。如果所选的日志目录不存在，则以红色显示。但是，ReSharper首次登录到文件时将创建不存在的路径。
请注意，记录到文件并不取决于记录到“输出”窗口。

为跳过的项目调用构建前和构建后事件目标

当ReSharper在构建过程中使用启发式方法跳过某些项目时，此类项目将被完全跳过，包括所有其他构建逻辑。
使用此复选框始终对所有项目执行构建前和构建后事件 。
要检查和配置构建前和构建后事件，请在解决方案资源管理器中右键单击该项目，选择“ 属性” ，然后选择“ 构建事件” 。

为跳过的项目调用BeforeBuild和AfterBuild目标

当ReSharper在构建过程中使用启发式方法跳过某些项目时，此类项目将被完全跳过，包括所有其他构建逻辑。
使用此复选框始终执行BeforeBuild和AfterBuild MSBuild目标 。

介绍
状态栏：在状态栏中显示构建进度

如果选中此复选框，则在Visual Studio状态栏中显示生成进度。

任务栏：在Windows任务栏中显示生成进度

如果选中此复选框，则Windows任务栏上的Visual Studio图标将显示生成进度。

油漆输出日志

选中此复选框后，“输出”窗口中的条目将以不同的颜色显示，与它们的来源和种类相对应。
仅在选中“ 写入输出”复选框时，此选项才可用。

工具窗口：构建开始时，在工具窗口中打开构建进度视图

选中此复选框后，ReSharper会弹出“ 构建工具”窗口以可视化构建进度和状态。

构建结果视图：启用按名称空间，类型和成员对解决方案构建器结果分组的功能

使您可以在“ 查找结果”窗口中显示的构建输出中切换构建结果的分组。
您可能需要在大型解决方案上禁用此选项，以最大程度地减少构建完成之后和构建结果显示之前的时间。

构建完成后显示构建结果

该选择器使您可以选择构建消息的级别以显示构建结果 。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
