拉成员重构

ReSharper |重构|向上拉成员...
ReSharper_PullUp

这种重构有助于将类型成员和已实现的接口从派生类型向上继承层次结构移动到基本类型。例如，如果您有MyDerivedClass : MyBaseClass, IMyInterface ，此重构可以帮助您移动MyDerivedClass至MyBaseClass要么IMyInterface或他们的任何祖先。重构还可以移动整个实现IMyInterface至MyBaseClass或其任何祖先。

反向功能在“ 下推成员向下”重构中可用。

拉起成员

  1. 通过以下方式之一选择类型：

    • 在编辑器中，将插入号设置为类型的名称。

    • 在解决方案资源管理器中选择一种类型。

    • 在“ 文件结构”窗口中选择一种类型。

    • 在“类视图”中选择一种类型。

    • 在对象浏览器中选择一种类型。

    • 在类型依赖关系图中选择一种类型。

    或者，也可以在编辑器或工具窗口中选择一个成员。

  2. 请执行以下任一操作：

    • 按Ctrl+Shift+R然后选择上拉成员

    • 右键单击并选择“ 重构”。在上下文菜单中上拉成员 。

    • 选择 ReSharper |重构|在主菜单中向上拉成员… 。

    将打开“上拉成员”对话框。

  3. 从可用类型列表中选择目标基本类型。基本类型显示为反向层次结构，其中层次结构顶部的类型显示为大多数内部节点。

  4. 选择要移动的成员和/或界面。您还可以单击“ 所有公共”以快速选择所有公共成员，和/或“ 从属”以选择其他所选成员引用的成员，实现所选接口或覆盖基类中的成员。

  5. 对于任何成员，您都可以选择Make abstract （如果适用）以在目标基类中创建一个抽象成员，并将其实现保留在当前类中。

  6. 要应用重构，请单击下一步 。

  7. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。
