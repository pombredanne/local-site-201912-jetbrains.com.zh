内联参数重构

ReSharper |重构|内联|排队…
Ctrl+Alt+N
ReSharper_InlineVariable

通过此重构，您可以用方法调用中的参数值替换方法参数。如果有多个调用，则可以选择要从中获取参数的调用。

如果要内联的参数取决于其他变量和/或计算，则ReSharper可以将原始参数替换为其他参数，然后将计算移入目标方法内。

在以下示例中，我们将重构应用于action的参数PerformAction方法，以便将整个lambda（用作调用中的参数）移到方法主体中，并创建两个新参数以传递必要的值：

反向功能可在“ 引入参数”重构中使用。

内联参数

  1. 将插入号放在方法声明中的参数处或方法调用中的参数处。

  2. 请执行以下任一操作：

    • 按Ctrl+Alt+N然后选择“ 内联参数”

    • 按Ctrl+Shift+R然后选择“ 内联参数”

    • 右键单击并选择“ 重构”。上下文菜单中的内联参数 。

    • 选择 ReSharper |重构|内联|内联…在主菜单中。

    内联参数对话框将打开。

  3. 如果该方法有多种用法，请选择要内联其参数的用法，然后单击“ 下一步” 。

  4. 选择要内联的参数。

  5. 如果内联参数取决于其他变量，则ReSharper在“ 新参数”字段中建议调用者提供一个或多个变量。选择所需的参数。

  6. 检查新签名的预览，然后单击“ 下一步” 。

  7. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。
