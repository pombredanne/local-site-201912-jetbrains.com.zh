CamelHumps是一项功能，用于识别根据CamelCase组成的化合物名称的各个部分，其中每个部分都以大写字母开头，或者当化合物名称的各个部分之间用下划线分隔时。您只能输入这些部分的首字母，然后ReSharper会自动找到名称匹配的项目。

搜索命令中的驼峰

CamelHumps始终可以在所有“按名称搜索”命令中使用，即“随处搜索/转到类型” ，“ 转到文件” ，“ 转到文件成员 ”和“ 转到符号” 。键入名称部分的首字母并获取匹配项列表非常方便：

CamelHumps编辑协助动作

CamelHumps也可以用于“ 扩展/收缩选择”以及ReSharper在Visual Studio中覆盖的其他键入帮助命令：

  • 将选择范围扩展到下一个/上一个词 （ Ctrl+Shift+Right,Arrow / Ctrl+Shift+Left,Arrow ）

  • 转到下一个/上一个词 （ Ctrl+Left,Arrow / Ctrl+Right,Arrow ）

  • 从单词开头到插入符号删除文本 Ctrl+Backspace
  • 从插入符号删除单词末尾的文本 Ctrl+Delete

在键入帮助操作中启用CamelHumps

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 前往环境|编辑器 ReSharper选项的“ 编辑器行为”页面。

  3. 使用“ 使用CamelHumps”复选框，在键入辅助功能中切换CamelHumps支持。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

如果启用了CamelHumps，则“扩展/收缩选择”命令将考虑部分化合物名称。假设您将插入符号放置在复合名称的中间，并以大写字母开头的几部分组成：

与CamelHumps，按Ctrl+W第一次选择单词的当前部分：仅在第二次按下此按键后，ReSharper才选择整个单词：所有后续压机均照常进行处理。

启用“ 使用CamelHumps”选项后，Visual Studio的“下一个/上一个单词” Edit.WordNext / Edit.WordPrevious和“将选择范围扩展到下一个/上一个单词” Edit.WordNextExtend / Edit.WordPreviousExtend命令也以不同的方式工作。当您按Ctrl+Right/Left,Arrow要么Ctrl+Shift+Right/Left,Arrow插入符号可有/无选择地移动到下一个/上一个字符，以分隔当前单词中复合名称的各个部分，并且仅在单词的最后一部分之后，才移动到下一个/上一个单词。

还有一种方法可以使“下一个/上一个单词”和“将选择范围扩展到下一个/上一个单词”两种行为（尊重和忽略Camel大小写），即默认的Visual Studio快捷方式（ Ctrl+Right/Left,Arrow和Ctrl+Shift+Right/Left,Arrow ）可以忽略Camel Case，而某些自定义快捷方式则适用于Camel Case。
要配置它，请关闭ReSharper设置中的Use CamelHumps选项，然后在Visual Studio选项（“ 工具” |“选项” |“环境” |“键盘” ）中将自定义快捷方式分配给以下命令：

  • ReSharper_HumpNext –将插入符号移至下一个驼峰

  • ReSharper_HumpPrev –将插入符号移至上一个驼峰

  • ReSharper_HumpNextExtend –将选择范围扩展到下一个驼峰

  • ReSharper_HumpPrevExtend –将选择范围扩展到先前的驼峰

请注意，使用此设置（禁用“ 使用CamelHumps”选项），“ 扩展/收缩选择”命令将忽略CamelHumps。

CamelHumps代码完成

默认情况下，还对代码完成功能启用了CamelHumps支持，也就是说，您可以开始输入CamelCase符号的缩写（也用小写字母），并且完成弹出窗口将显示匹配的项目。如有必要，您可以配置此行为。

更改代码完成过滤与CamelHumps一起工作的方式

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择环境|智能感知左侧的完成行为 。

  2. 使用“ 匹配标识符中间”复选框可切换CamelHumps对完成功能的支持。

  3. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。
