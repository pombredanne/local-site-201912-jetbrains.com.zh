代码清理

ReSharper |选项|代码编辑|代码清理

ReSharper选项的此页面允许您管理和配置自定义清除配置文件 ，以及为静默代码清除分配清除配置文件。他们俩默认配置文件（ 完全清除 ， 重新格式化和应用代码样式以及重新格式化代码 ）也在此处显示。您不能修改，修改它们，但可以克隆它们以创建新的自定义配置文件。

您也可以使用命令行实用程序运行代码清除。

代码清除不适用于生成的代码。默认情况下，ReSharper提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ）。

工具栏控件

使用以下控件来配置选定的清理配置文件：

项目

描述

设为默认

单击此按钮以使用选定的配置文件进行静默清除 。用于静默清除的配置文件标有图标。

重复

单击此按钮创建所选配置文件的副本。如果您具有默认设置，则需要复制一个默认清除配置文件以创建自定义清除配置文件。 创建新配置文件后，它将以编辑模式打开，您可以在其中配置清除任务列表。

配置

单击此按钮为所选的自定义配置文件启用编辑模式。注意无法编辑默认配置文件（ 完全清除 ， 重新格式化和应用代码样式以及重新格式化代码 ）。

单击此按钮以重命名所选的自定义配置文件。注意无法重命名默认配置文件（“ 完全清除” ，“ 重新格式化和应用代码样式”和“ 重新格式化代码” ）。

完成了

单击此按钮退出编辑模式，然后返回到所选配置文件的预览。请注意，除非单击“ 保存”，否则将不会应用您的修改。

删除代码冗余

上 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ）中，在大多数语言的代码类别中，ReSharper会列出50多个与代码检查相关的冗余。其中有些需要特别注意，您不能以批处理模式进行处理。这样，在代码清理中，大约有30个冗余，只要您在清理配置文件中选择“ 删除代码冗余” ，它就可以自动删除 。
具体来说，代码清理：

  • 删除：

    • 冗余的空构造函数。

    • 枚举中的显式整数类型说明符。

    • true中的说明符for陈述。

    • 冗余类型转换。

    • 没有参数的属性中的多余括号。

    • 显式调用不带参数的基类构造函数。

    • 冗余布尔比较。

    • 来自集合初始值设定项的带括号的单个表达式。

    • 冗余的显式委托创建表达式。

    • 来自对象/集合初始化程序的空参数列表。

    • 多余的new关键字。

    • 多余的partial类型声明具有单个部分的类型声明上的修饰符。

    • 多余的unsafe上下文和陈述。

    • 多余的object.ToString()调用（例如，与字符串串联或作为a的参数string.Format()呼叫）。

    • 多余的string.ToCharArray()电话。

    • 参数类型的冗余规范。

    • 匿名方法中的签名，其中方法主体中未使用其参数。

    • 未引用的标签。

    • 冗余字段初始化程序。

    • 匿名类型属性中的冗余显式名称。

    • 冗余的可为空类型，其中值类型可隐式转换为可为空。

  • 转换：

    • 可空类型到短格式Nullable至T? ）。

    • 扩展方法调用的静态方法调用，其中扩展方法作为静态方法被调用。

    • 当可以从初始值设定项推断出数组类型时，以隐式类型化数组创建表达式。

  • 修正：

    • 通过类型的派生成员访问类型的静态成员的情况。
