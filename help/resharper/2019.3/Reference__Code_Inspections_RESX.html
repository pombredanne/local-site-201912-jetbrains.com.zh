资源文件中的代码检查

在资源文件中，ReSharper 2019.3提供了13 专有代码检查 ，您可以禁用或更改其严重性水平 。
这些代码检查可帮助您在设计时检测所有打开文件中的代码问题，此外，它们还使您可以查找特定范围内的代码问题 。

下面列出了可配置资源文件检查，按其类别分组。

潜在的代码质量问题（8次检查）

此类检查包括检测关键问题（代码异味）的检查，主要是错误或警告级别。此类别还包括确保本地化协助的检查 。

检查

 ID EditorConfig属性 默认严重性
无法解析符号 ResxNotResolved resharper_resx_not_resolved_highlighting 警告
资源名称重复 DuplicateResource resharper_duplicate_resource_highlighting 警告
元素可本地化 LocalizableElement resharper_localizable_element_highlighting 警告
格式字符串占位符不匹配 FormatStringPlaceholdersMismatch resharper_format_string_placeholders_mismatch_highlighting 警告
资源未在基本文化中声明 NotDeclaredInParentCulture resharper_not_declared_in_parent_culture_highlighting 警告
在特定文化中资源不会被覆盖 NotOverriddenInSpecificCulture resharper_not_overridden_in_specific_culture_highlighting 警告
资源以空值覆盖基本资源 OverriddenWithEmptyValue resharper_overridden_with_empty_value_highlighting 警告
资源值类型无效 InvalidValueType resharper_invalid_value_type_highlighting 警告

代码中的冗余（两次检查）

此类别中的代码检查查找冗余和无效代码，这些代码会影响代码的可读性和样式，并且可以安全地删除。 某些代码冗余无法自动修复，因此需要在用户输入的情况下以交互方式进行快速修复。但是，可以使用范围内的修复或代码清除来修复大多数冗余，而无需用户交互。

检查

 ID EditorConfig属性 默认严重性
冗余字符串类型 RedundantStringType resharper_redundant_string_type_highlighting 建议
资源被相同值覆盖 OverriddenWithSameValue resharper_overridden_with_same_value_highlighting 建议

拼写问题（3次检查）

这些检查可以检测各种情况下的错别字。

检查

 ID EditorConfig属性 默认严重性
错字评论 CommentTypo resharper_comment_typo_highlighting 建议
标记属性值中的错字 MarkupAttributeTypo resharper_markup_attribute_typo_highlighting 建议
标记文字中的错字 MarkupTextTypo resharper_markup_text_typo_highlighting 建议
