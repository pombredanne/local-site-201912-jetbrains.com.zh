浏览当前上下文

ReSharper提供了许多功能，可帮助您根据当前上下文（即根据打开的文档和您在其中的插入符号位置）导航代码。

按文件结构浏览文件

ReSharper | Windows |档案结构
Ctrl+F11
ReSharper_ShowCodeStructure

使用ReSharper，您可以使用“文件结构”窗口浏览当前文档的结构，这大大简化了大型文件的浏览。该窗口与编辑器同步：切换到另一个编辑器选项卡时，该窗口显示相应文件的结构。

  1. 按Ctrl+F11或选择 ReSharper | Windows |从主菜单中的文件结构 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  2. 当前文档的内容以树形结构显示，您可以浏览并浏览该文档。您可以双击任何符号，以使编辑器中的插入符号切换到相应位置。

  3. 如果代码的某些部分被#region块，区域也会出现在文件结构中。

    您可以直接从文件结构将符号包装到区域中。为此，请选择要包装的成员（ Ctrl单击多个选项的项目），然后单击“ 环绕区域” 在工具栏上。

    您可以通过单击十字图标删除任何区域在区域框架的右上角。

    要重命名区域，请双击其名称或按F2在它上面，然后键入新名称。

    ReSharper提供了更多用于管理区域的功能。要了解更多信息，请参阅“ 添加和删除#region块” 。

  4. 要使您在编辑器中更改插入符号位置的方式与“ 文件结构”窗口中突出显示的符号同步，请单击 在编辑器中跟踪插入符 在工具栏上。

  5. 要将您在“ 文件结构”窗口中选择的符号与编辑器中的插入符号位置同步，请单击 自动滚动到源 在工具栏上。

文件成员标有符号图标，您可以在“ 环境” |“配置”中配置它们。 ReSharper选项的常规页面。

在“文件结构”窗口中，您可以访问更多操作。要检查可用命令，可以右键单击任何符号。

  • 从符号导航
  • 查找符号的用法 Alt+F7
  • 生成类型成员 Alt+Insert

  • 执行重构 （ Ctrl+Shift+R ）

  • 拖放类型和类型成员以重新排列其顺序。您所做的更改将立即反映在文件中。

  • 如果打开的文件包含单元测试，则这些单元测试将用相应的图标标记，以显示其上次执行的状态。您也可以直接在“文件结构”窗口中运行和调试单元测试 。

要在“文件结构”窗口中搜索项目，只需开始输入名称或CamelHumps缩写。

导航到包含声明

ReSharper |导航|包含声明
Ctrl+[
ReSharper_GotoContainingDeclaration

使用单个快捷键，您可以在编辑器中从其体内的任何位置跳转到类型或函数的声明。

ReSharper提供了一项相关功能- 选择包含声明 （ Ctrl+Shift+[ ）。

  1. 将插入号设置为类型或函数体内的任何位置。

  2. 按Ctrl+[或选择 ReSharper |导航|从主菜单包含声明 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. ReSharper会将插入符号设置为类型或函数的声明。

  4. 如果在另一个类型中声明了该类型或函数，则可以重复第二步以导航到包含类型的声明。

在成员和标签之间导航

ReSharper |导航|下一个/上一个成员
Alt+Down / Alt+Up
ReSharper_GotoNextMethod / ReSharper_GotoPrevMethod

使用“ 下一个/上一个成员”命令，您可以在代码文件中的成员和嵌套类型之间或标记文件中的标记之间快速导航。

  1. 当焦点位于编辑器中时，按Alt+Down / Alt+Up或选择ReSharper |导航|主菜单中的下一个/上一个成员 。

  2. 插入符号将移至下一个/上一个成员/标签的声明。如果存在内部类型，则插入符号还将在每个内部类型的成员处停止。

在解决方案/组装资源管理器中找到当前文档

ReSharper |工具|在解决方案/装配体浏览器中找到
Shift+Alt+L
ReSharper_LocateInSolutionOrAssemblyExplorer

此便捷的导航功能使您可以在“解决方案资源管理器”中展开树，自动滚动并突出显示与当前文档相对应的项目。当您拥有庞大的解决方案或有限的屏幕区域（迫使您将“解决方案资源管理器”窗口缩小至极限）时，此功能特别有用。

请注意，如果当前解决方案中未包含在编辑器中打开的文件，则此命令将不起作用。

此命令还可以与反编译代码一起使用，从而使您可以在Assembly Explorer窗口中找到反编译的符号。在这种情况下，该命令对插入符号的位置敏感：如果光标位于特定符号上，则与该符号对应的项目在Assembly Explorer中突出显示；如果光标位于特定符号上，则该符号对应的项目会突出显示。如果光标在文档中的其他位置，则突出显示文档的根类型。

您也可以在使用该符号的源代码中直接在Assembly Explorer中找到一个库符号：插入符号在符号上时，按Ctrl+Shift+G或选择 ReSharper |导航|在主菜单中导航到… ，然后选择在装配体浏览器中定位 。

  1. 在编辑器中，切换到要在解决方案资源管理器中找到的文档。

  2. 按Shift+Alt+L或选择ReSharper |工具|在主菜单中的解决方案/装配体浏览器中找到 ，

  3. ReSharper将调出“解决方案资源管理器”窗口或“程序集资源管理器”窗口（取决于您的插入符号是源代码还是反编译代码），以集中精力并向您显示该项目。

浏览相邻文件和项目

ReSharper |导航|转到附近的文件
ReSharper_GotoFileNearby

使用此命令可在当前文件周围浏览项目结构。在打开的弹出窗口中，您可以查看当前文件所属的项目，文件夹和文件在同一目录级别，并且可以轻松地跳转到这些文件或创建新文件或文件夹。

您可以从主菜单调用此命令（ ReSharper |导航|转到附近的文件 ），但从“ 导航到”弹出窗口中选择它可能更方便Ctrl+Shift+G 。或者，您可以为该命令分配自定义快捷方式 ReSharper_GotoFileNearby 。

此命令是在解决方案资源管理器中查找当前文档的不错选择。 Shift+Alt+L 。在此命令打开的弹出窗口中，您可以查看当前文件的直接邻居以及在解决方案层次结构中导航任何项目。
如果您选择一个代码文件，它将在编辑器中打开；选择一个项目文件.csproj / .vbproj将在解决方案资源管理器中找到相应的项目。

浏览相关文件

ReSharper |导航|相关文件…
Ctrl+Shift+Alt+G
ReSharper_GotoRelatedFiles

该命令可以将您带到其内容与当前文件某种程度上相关的所有文件。

导航到相关文件对于Web应用程序可能非常有用：例如，它使您能够从Web表单跳转到其主页，用户控件，引用的图像或JavaScript和CSS文件。在ASP.NET MVC应用程序中，它简化了从视图到控制器以及向后的导航。

  1. 将插入号放置在文件中或在解决方案资源管理器中选择文件。

  2. 按Ctrl+Shift+Alt+G或选择 ReSharper |导航|相关文件…从主菜单。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. 如果存在一些相关文件，则会打开“ 选择文件”列表。从列表中选择一个文件，然后单击它以在编辑器中将其打开。
