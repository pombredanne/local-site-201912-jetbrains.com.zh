探索当前流程中的装配体

ReSharper | Windows |流程浏览器
ReSharper_ShowProcessExplorer

使用ReSharper的Process Explorer窗口 ，您可以浏览当前流程和已加载的程序集。这些过程与相应的装配一起显示，并且可以进行排序和分组。如有必要，您可以显示或隐藏本机进程。

在查看所有已加载程序集的情况下，您可以轻松找到要反编译和浏览的程序集。

打开加载的程序集

  1. 请执行以下任一操作：

    • 选择 ReSharper | Windows |主菜单中的Process Explorer 。

    • 在“ 程序集资源管理器”窗口的工具栏上，单击“ 从正在运行的进程中打开” 。

  2. 在“ 进程资源管理器”窗口中 ，执行下列操作之一：

    • 展开要浏览的过程，然后选择所需的装配。

    • 单击按窗口查找过程 然后将mose悬停在要探索其过程的窗口上：

      在工具提示中，您可以看到PID和CLR版本。并且该过程在“ 过程资源管理器”窗口中突出显示。

  3. 选择所需的装配后，单击“ 将装配添加到装配资源管理器”。 。程序集出现在“程序集资源管理器”窗口中，并且可以反编译。

如果您需要调试某些.NET进程，但是没有它的PDB，则可以直接从Process Explorer中生成丢失的PDB。为此， 右键单击任何.NET进程，然后选择使用ReSharper Decompiler加载符号 。 生成将立即开始到Visual Studio选项（“ 工具” |“选项” |“调试” |“符号” ）中指定的符号缓存目录。如果未指定目录，ReSharper将建议自动将％LOCALAPPDATA％\ Temp \ SymbolCache添加为缓存目录。

如果您想随时清除反编译缓存，请删除％LOCALAPPDATA％\ JetBrains \ Shared \ vAny \ DecompilerCache中的数据 。
