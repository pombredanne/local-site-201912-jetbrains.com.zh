该命令尝试获取库符号的原始源文件并在其中显示该符号。 在源代码和反编译源中的库符号上调用“ 导航至”时，该命令可用。 您可以从编辑器，“ 文件结构”窗口和其他工具窗口中调用此命令。

ReSharper从符号服务器下载所需的文件可能需要一些时间；在这种情况下，您将看到下载进度。

如果在“ 工具” | “ 工具”中选择了“ 默认Visual Studio导航”，则此命令不可用。 ReSharper选项的“ 外部源”页面。要使其可用，请在该组中选择任何其他选项。

要成功执行此命令，PDB信息应可用于目标程序集。ReSharper使用两种方法来搜索PDB：

  • 它在目标程序集所在的同一目录中本地搜索PDB。

  • 如果无法在此处找到PDB，则使用_NT_SYMBOL_PATH环境变量。此外，ReSharper使用Visual Studio选项来查找符号服务器。参见下文，如何在Visual Studio选项中指定符号服务器 。
    一些公司使用符号服务器（源服务器）发布部分源代码。例如，Microsoft使用此技术提供对.NET Framework源代码的访问。如果目标PDB的符号服务器可用，则此命令可以访问以这种方式存储的源。

为了使ReSharper知道可用的符号服务器，您需要在Visual Studio选项和/或_NT_SYMBOL_PATH环境变量。ReSharper将查找符号文件并从指定位置下载它们。

在Visual Studio选项中指定符号服务器的路径

  1. 选择工具|从菜单中选择选项 ，然后转到调试|符号页面。

  2. 在“ 符号”页面上，单击文件夹图标，然后在新的符号文件位置中键入源服务器的路径（例如http://domain/path/symbols ）。有关更多信息，请参见如何：使用符号服务器 。
    是否选中符号服务器旁边的复选框并不重要。ReSharper使用其自己的算法来处理符号服务器，并在Visual Studio选项中仅分析到这些服务器的路径。
