将抽象类转换为接口重构

ReSharper |重构|转换|接口的抽象类
ReSharper_Abstract2Interface

通过此重构，您可以将抽象类转换为接口。当您希望一个类从多个抽象类继承时，此功能很有用-然后您可以轻松地将其中一个抽象类转换为接口。

反向功能可通过将接口转换为抽象类重构来实现。

考虑下面的示例。应用重构后，将一个类转换为接口：

重构之前

重构后

公共抽象类CachedItem {公共抽象int ID {组; } public abstract void Register（）;公共抽象无效UnRegister（）; }
公共接口CachedItem {int ID {get;组; } void Register（）;无效的UnRegister（）; }

将抽象类转换为接口

  1. 通过以下方式之一选择抽象类：

    • 在编辑器中，将插入符号设置为抽象类的名称。

    • 在解决方案资源管理器中选择一个抽象类。

    • 在“ 文件结构”窗口中选择一个抽象类。

    • 在“类视图”中选择一个抽象类。

    • 在对象浏览器中选择一个抽象类。

    • 在类型依赖关系图中选择一个抽象类。

  2. 请执行以下任一操作：

    • 按Ctrl+Shift+R然后选择将抽象类转换为接口

    • 右键单击并选择“ 重构”。在上下文菜单中将Abstract Class转换为Interface 。

    • 选择 ReSharper |重构|转换|接口的抽象类…在主菜单中。

  3. 如果抽象类包含非抽象成员，则ReSharper会显示一个对话框，通知您将删除此类成员的主体和/或访问器。为避免这种情况，您需要将非抽象成员的逻辑手动移到子类或其他地方，然后再次调用重构。

  4. 如果未发现冲突，则将抽象类转换为具有相同名称的接口，并相应地更改抽象成员的修饰符。否则， 解决冲突 。

默认情况下，接口的命名方式要求接口名称以I 。因此，如果您没有更改设置，则接口名称将突出显示为名称不一致。只需按Alt+Enter并使用相应的快速修复来重命名接口。
