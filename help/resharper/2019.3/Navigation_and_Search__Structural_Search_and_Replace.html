有时，标准搜索方法（例如搜索与某些正则表达式匹配的特定符号或文本出现的用法）无法提供理想的结果。例如，您可能需要查找特定于项目的代码气味或查找需要重构的代码段。

ReSharper允许您查找并在必要时替换与特定模式匹配的代码块。这些块可能完全不同-它们可以包含不同的类型，表达式，语句等，但是在逻辑上它们是相似的。例如，您可以创建将匹配所有foreach循环与迭代的内容以及循环中的内容无关，或者您可以使此模式更加详细，以仅查找在特定类型上进行迭代的循环。模式可以根据需要复杂，也可以具有任何所需的详细程度。

代码分析部分的相应主题说明了使用结构模式作为自定义检查和快速修复

您创建的所有模式都存储在模式目录中，并用作具有自定义严重性级别的自定义代码检查 。这意味着与特定模式匹配的所有代码块都可以在编辑器中突出显示为提示，建议，警告或错误。还为模式提供了快速修复程序 -它们可以按照模式的“替换”部分中定义的方式转换匹配的代码块。

搜索模式中的占位符

模式通常包括：

  • 文本部分，必须仅包含目标语言中允许的标识符。在文本部分，您可以定义完全匹配的内容。请注意，空格，制表符和换行符将被忽略。

  • 占位符，允许匹配目标代码块的可变部分。占位符具有以下格式： $placeholder_name$ -其中placeholder_name是任意标识符。

每个占位符必须定义一次，并且可以在模式中多次使用。定义占位符时，需要定义其种类和可选约束。占位符有五种：

  • 参数占位符 -方法调用中的一个或多个参数。如有必要，可以指定应匹配的最小或最大数量的参数。

  • 表达式占位符 -一系列运算符和操作数。您可以选择指定此表达式返回的类型。

  • 标识符占位符 -任何符号标识符。您还可以指定用于匹配符号名称的正则表达式。

  • 语句占位符 -以分号或语句块结尾的单行语句。如有必要，可以指定应匹配的最小或最大语句数。

  • 类型占位符 -值类型或引用类型。默认情况下，这种占位符将匹配任何类型，但是您可以显式指定特定类型。

请注意，为模式的特定部分选择合适的占位符非常重要。如果您的模式与要匹配的代码块不匹配，则问题通常出在错误选择的占位符类型上。

创建搜索模式

要搜索或替换代码块，或创建自定义代码检查 ，您需要创建搜索模式。您可以将现有的代码块用作该模式的基础，也可以从头开始创建一个新的代码块。

创建搜索模式

  1. 在编辑器中，选择应该与模式匹配的代码块。此步骤是可选的，但它将帮助您更快地创建模式。

  2. 在主菜单中，选择“ ReSharper |查找|按模式搜索…或右键单击所选内容（如果有），然后在上下文菜单中选择按模式搜索 。“ 使用图案搜索”对话框将打开。

  3. 如果在第一步选择了某个代码块，则ReSharper会检测到该块中的所有符号名称，并将其替换为占位符 。如有必要，您可以编辑代码，添加和删除占位符以获得所需的模式：

    • ReSharper会检查该模式中使用的所有占位符是否在对话框的右侧都有正确的定义。如果存在没有定义的占位符，则将其突出显示为红色。

    • 要添加新的占位符，只需键入$$并且其定义将自动添加。
      始终检查自动添加的占位符定义的类型。有时无法明确定义种类，因此您可能需要手动更正。

    • 要更改占位符的类型或指定约束，请选择右侧的占位符定义，单击“ 编辑” ，然后编辑占位符定义。

    • 在模式主体中删除占位符用法后，其定义将自动删除。

    • 也可以手动添加和删除占位符定义。为此，请单击占位符定义列表上方的相应按钮。

  4. 根据您是否要匹配相似的代码块，选择或清除“ 匹配相似的结构”复选框。 ReSharper认为以下构造类似：

    • 用大括号括起来的单行语句和单行语句。

    • 在模式中以相反的顺序指定的二进制表达式。

    • 模式中指定的表达式以及具有不同操作数顺序的表达式。

    • 用括号括起来的表达式和没有用括号括起来的表达式。

    • 后缀和前缀增量运算符。

    • 具有不同访问修饰符的方法。

  5. ReSharper在模式主体中执行基本代码分析。如果未找到错误，则可以搜索匹配的代码块 ， 定义如何替换匹配的块 ，或者只需单击“ 保存”以将模式保存到模式目录中 。

指定替换模式

对于每种搜索模式，您都可以添加替换模式，这将使您可以快速替换与搜索模式匹配的代码块。

指定替换模式

  1. 创建新的搜索模式或打开以从模式目录中编辑模式 。

  2. 在“ 使用模式搜索”对话框中，单击右上角的“ 替换 ”。

  3. 在对话框下部显示的“ 替换图案”区域中，指定图案的替换部分。
    使用模式中定义的占位符 。替换匹配的代码块时，将根据占位符在替换部分中的位置复制由占位符匹配的每段代码。

  4. 使用替换区域下方的复选框来指定是否应用格式规则和/或在替换的代码块中用短名称替换完全限定的名称。

  5. ReSharper在模式的替换部分执行基本的代码分析。如果未发现错误，则可以替换匹配的代码块或将模式保存到模式目录中 。

管理模式目录

保存创建的图案时，ReSharper会将其放置在其图案目录中。要查看目录并研究现有模式，请转至“ 代码检查” | “ 代码检查”。 ReSharper选项的“ 自定义模式”页面，或仅单击“ 模式目录” 在“ 使用模式搜索”对话框中。

您可以在模式目录中执行以下操作：

  • 要预览图案，请在列表中选择它。您将在页面底部看到其主体和占位符的定义。

  • 要删除图案，请在列表中选择它，然后单击页面工具栏上的“ 删除 ”。

  • 要添加新图案，请单击工具栏上的添加图案 。ReSharper将打开“ 用图案搜索”对话框，您可以在其中指定图案详细信息 。

  • ReSharper的代码检查引擎中使用了所有模式，因此会自动检测到匹配的代码块。
    默认情况下，所有新模式在严重性级别选择器中均具有“不显示”，这意味着在特定范围内运行检查时，既不会在编辑器中突出显示匹配的代码块，也不会检测到匹配的代码块。如果要在代码检查中检测与搜索模式匹配的代码，请为该模式选择适当的严重性级别。

  • 要编辑现有图案，请双击它或选择它，然后在窗口工具栏上单击“ 编辑 ”。

    打开阵列进行编辑时，可以指定其搜索和替换零件的描述。搜索描述将用于显示目录中的模式，并作为在编辑器中突出显示的匹配块的描述。替换描述显示为快速修复的名称，建议用于突出显示的块。有关更多信息，请参见创建自定义代码检查和快速修复 。

    您可以为模式指定“抑制键”，该标识符可用于抑制ReSharper检查 ，该检查检测带有模式的注释。

用模式查找和替换代码

有三种方法可以用模式搜索和替换代码：

  • 您可以在调用“ 使用模式搜索”命令并创建新模式之后立即搜索或替换代码。在这种情况下，您只需要单击“ 搜索图案”对话框右下角的“ 查找 / 替换”按钮。

  • 您可以使用模式目录中的一种模式 。这样做，请单击“ 代码检查” | “工具栏”上的“ 立即搜索 ”。 ReSharper选项的“ 自定义模式”页面。

  • 可以在编辑器中突出显示与模式匹配的代码块。如果为该模式指定了替换逻辑，则可以使用quick-fix替换代码块。您所需要做的就是为模式指定一些严重性级别，而不是“不显示”。使用“ 代码检查” |“每个模式”右侧的相应选择器。 ReSharper选项的“ 自定义模式”页面。有关更多信息，请参见创建自定义代码检查和快速修复 。

如果运行前两种方法之一进行搜索，则解决方案中所有匹配的代码块都会在“ 查找结果”窗口中列出。如果结果不包含您要查找的内容，则可以快速修改模式并再次调用搜索过程。这样做，单击更改搜索模式 在窗口工具栏上。

如果运行前两种方法之一，则解决方案中的所有替换候选者将显示在“ 用图案替换”对话框中。在此对话框中，您可以研究候选人并将其分组。如果要在上下文中查看候选人，请单击“ 显示预览”。 在工具栏上。您也可以双击任何候选者以在编辑器中打开包含的文件。
如果显示的候选人是您要替换的候选人，请单击对话框底部的“ 替换 ”。

共享模式

您的搜索和替换模式可以保存在共享的设置层中，并可供所有团队使用。

如果在代码检查中创建模式| ReSharper选项的“ 自定义图案”页面，单击“ 保存到” ，然后在图案准备好后选择一个共享设置层。

要共享尚未共享的现有图案，请打开它进行编辑，然后在“ 编辑突出显示图案”对话框中单击“ 保存副本 ”。然后单击“ 保存到” ，然后选择一个共享设置层。
