使用注释完善代码检查

代码检查和许多其他ReSharper功能在很大程度上依赖于了解语言结构的行为来检测问题，提出可能的改进并以其他方式帮助您。

但是，这种分析无法检测所有内容。例如，如果某个方法设计为永不返回null并且其客户是经过相应设计的，如果有人更改了退货方法，则没有结构分析会发现可能的问题null 。

在这种情况以及很多其他情况下，ReSharper的JetBrains.Annotations有很大的帮助。通过使用在此框架中声明的属性，您可以使ReSharper根据需要分析代码。例如：

[NotNull]公共对象Foo（）{返回null； //警告：可能为“空”分配}

这是最简单的示例，还有其他有用的属性和更复杂的用例。您可以在参考资料中找到这些属性的完整列表。

在大多数情况下，代码注释属性启用特定的代码检查 ，例如：

  • StringFormatMethodAttribute帮助检测字符串格式化方法的滥用 。

  • CanBeNullAttribute和NotNullAttribute与可能标记为“值不能为空”属性的实体的“空”分配相关联。有关更多信息，请参见值和可空性分析 。

  • CannotApplyEqualityOperatorAttribute绑定到代码检查：与“ ==”类型进行比较，并用“ CannotApplyEqualityOperatorAttribute”标记 。

  • ContractAnnotationAttribute可用于为您的职能定义合同并打开相应的检查。例如，您可以使用[ContractAnnotation("input:null => false")]通知消费者该功能bool Foo(object input)总是回来false当参数s是null 。

此外，代码注释可在带注释的项目上启用更多快速修复 ， 代码完成建议和代码生成功能 。创建和使用源模板还需要注释。

ReSharper允许您以两种方式注释代码符号：

  • 您可以在源代码中注释符号，如上面的示例所示。在这种情况下，您需要参考JetBrains.Annotations项目中的名称空间。有关更多信息，请参见源代码中的注释 。

  • 即使您无权访问源，也可以在已编译的库代码中注释符号。有关更多信息，请参见外部注释 。

使用外部注释对标准库（例如.NET Framework类库，NUnit Framework）进行注释的工作已经完成，并且ReSharper安装中已包含这些库的外部注释。因此，使用这些库时，您可以受益于属性驱动的检查，快速修复和其他功能。
