单元测试会话窗口

ReSharper |单元测试|单元测试会议
Ctrl+Alt+R

ReSharper的单元测试运行程序不允许运行两个或多个具有相同ID的测试。在实践中，这种情况很少发生，但是如果您确实有具有相同ID的测试，它们将在“单元测试”浏览器和“单元测试会话”窗口中显示为单个节点，并且仅运行其中一个。
请注意，测试框架以不同的方式识别测试。例如，MSTest使用测试方法FQN，而NUnit使用测试属性的属性作为测试ID。

ReSharper在“ 单元测试会话”窗口中运行单元测试。该窗口在出现以下情况时自动出现您可以从当前文档或项目或解决方案中 运行测试 。有关更多信息，请参见使用单元测试会话 。

窗户是旨在帮助您同时且彼此独立地运行任意数量的单元测试会话。会话可以由测试的任意组合组成。在调试模式下，一次只能运行一个会话。

单元测试树显示属于会话的测试的结构，您可以对其进行过滤以仅显示通过，失败或忽略的单元测试。您可以通过双击导航到任何单元测试的代码。

状态栏和进度栏

工具栏下的进度栏和状态栏显示当前进度。

在状态栏上，您可以查看会话中的测试总数以及处于不同状态的测试数：

默认情况下，显示所有状态下的测试，但是您可以单击相应的图标以按测试状态过滤测试。你也可以Ctrl单击几个图标以显示几种不同状态的测试。

选件

您可以为每个测试会话分别配置这组首选项。

  • 构建策略 -指定在执行测试之前是否需要构建。提供几个选项：

    • 自动 （默认选中）—执行测试之前，ReSharper将重建与当前测试会话相关并且自上次执行以来已发生更改的所有项目。

    • 自动（整个解决方案） —执行测试之前，ReSharper将重建您解决方案中自上次执行以来发生更改的所有项目。

    • 从不构建 -ReSharper在执行测试之前不会构建任何东西。如果选择此选项，则必须确保所有相关项目均已构建。

  • 锁定会话 — 锁定当前测试会话，以防止自动向该会话添加新测试。

  • 显示时间 -允许您显示或隐藏测试会话中每个项目的执行时间。

  • 平台 -允许您明确指定测试运行器的平台体系结构，或选择“ 自动”以使用与项目设置相对应的平台体系结构。

  • 框架 -允许您明确指定测试运行程序的.NET框架（CLR）版本，或选择“ 自动”以使用.NET Framework 4.0。

  • 分组依据 -允许按不同类别对窗口中的项目进行分组。
