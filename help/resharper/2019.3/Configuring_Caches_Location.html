配置缓存

为了确保可接受的速度，ReSharper会缓存项目模型，该模型在分析解决方案时会获得。每种解决方案的缓存数据都单独存储。因此，当您首次打开新解决方案时，分析可能会花费一些时间，但随后的所有次您打开解决方案时，ReSharper都只会读取缓存的数据，这通常会很快发生。
根据您的系统配置，您可能更喜欢使用不同的位置来存储缓存文件。

缓存位置

指定缓存的位置

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择环境|一般在左边。

  2. 使用“ 将解决方案缓存保存到”来选择缓存文件的位置：

    • 用户本地设置文件夹以将它们存储在以下目录中： ％LOCALAPPDATA％\ JetBrains \ Transient

    • 系统TEMP文件夹将它们存储在以下目录中： ％TEMP％\ ReSharperCache

    • 解决方案文件夹 ，将它们存储在当前解决方案的根文件夹中

    • 自定义文件夹 ，为ReSharper缓存文件选择自定义位置。

  3. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  4. 重新打开您的解决方案以使更改生效。

如果您的解决方案处于版本控制之下，或者您计划将其放在该版本中，则最好选择System TEMP文件夹或User local settings文件夹 。它使您可以将缓存从解决方案文件夹中排除，并将其与源代码分开保存，以使它们不会被错误地提交到您的存储库中。

请注意，可以使用某些实用程序清除系统临时文件夹，如果将它们存储在其中，则可能会删除所有缓存。

清理缓存

缓存中断会影响ReSharper的行为。例如，ReSharper可以停止解析符号，或者某些导航命令不可用。如果您注意到这种奇怪的行为，则在大多数情况下清除当前解决方案的缓存可能会有所帮助。

清除当前解决方案的缓存

  1. 在Visual Studio中打开缓存中断的解决方案。

  2. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择环境|一般在左边。

  3. 单击清除缓存 。请注意，仅在当前选定的缓存位置中清除缓存 。

  4. 重新打开您的解决方案以使更改生效。

如果特定解决方案的打开时间超过30天，则ReSharper还会自动清除解决方案缓存。
