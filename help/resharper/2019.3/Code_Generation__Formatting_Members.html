生成格式化成员

ReSharper |编辑生成代码|格式化成员
Alt+Insert | 格式化成员
ReSharper_GenerateFormattingMembers

.NET中的任何类型都可以实现ToString()方法，该方法返回该类型对象的字符串表示形式。要为我们的类型返回有意义的字符串，我们通常需要覆盖ToString()方法。

ReSharper允许您使用“ 生成格式化成员”命令自动执行此例程。
根据目标C＃版本 ，ReSharper使用插值字符串或String.Format() 。请注意，您始终可以在具有上下文操作的操作之间进行转换Alt+Enter 。

在以下示例中，此命令用于生成ToString()基于的方法_radius和_center领域。

您可以在上配置通用代码生成首选项 代码编辑|的会员生成页面 ReSharper选项（ Alt+R,,O ）。

生成用于格式化成员的替代

  1. 在编辑器中，将插入号设置为类型名称，或者在要插入用于设置成员格式的替代的行的类型内。如果插入符号在类型名称上，则生成的代码将添加到类型声明的开头。

  2. 按Alt+Insert或选择 ReSharper |编辑从主菜单生成代码… 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. 在“ 生成”弹出窗口中，选择“ 格式化成员” 。

  4. 在出现的“ 生成”对话框中，选择要在广告系列中使用的字段和/或自动属性ToString()覆盖。

    如果您的类型中没有字段/属性，或者您未选择任何字段/属性，ReSharper会根据您的设置抛出新的NotImplementedException() ，返回默认值或将无法编译的代码放入生成的方法的主体中。您可以在“ 代码编辑” |“配置”中配置设置。 ReSharper选项的“ 成员生成”页面。

    （可选）在对话框中使用以下控件：

    • ToString已经存在 -如果实现ToString()已经存在，您可以选择是否：

      • 如果已经存在，请替换该方法。

      • 将新生成的方法与现有方法并排放置。

      • 完全跳过生成新方法。

    • 使用'nameof'-如果目标C＃版本高于6.0，则会出现。如果选中此复选框，则ReSharper将使用nameof(symbol_name)定义符号的字符串表示形式，否则，它将自动计算字符串表示形式，例如，一个名为_radius将呈现为"Radius" 。

  5. 单击完成以完成向导。
    您也可以在“ 代码编辑” |“选项”上单击“ 选项”以查看或修改常见的代码生成首选项。 ReSharper选项的“ 成员生成”页面。

您还可以通过在“ 生成”菜单中选择“ 覆盖成员 ”来生成 覆盖 ，但是在这种情况下，覆盖将返回base.ToString() 。

要将此代码生成命令绑定到其自己的快捷方式，请转到“ 工具” |“其他”。选项|环境|键盘并为ReSharper_GenerateFormattingMembers命令。
