JavaScript中的单元测试协助

ReSharper可帮助您直接在Visual Studio中发现和运行QUnit和Jasmine框架的单元测试。

ReSharper不支持JavaScript / TypeScript测试的调试。

使用ReSharper，您可以执行单个单元测试，所有测试都在文件，项目或解决方案中。您也可以在测试会话中执行任意数量的测试。

您可以通过清除下面的“ 单元测试”复选框来禁用单元测试辅助功能。 环境|的产品和功能页面 ReSharper选项（ Alt+R,,O ）。

ReSharper支持嵌套引用，因此您不必直接从具有测试的文件中引用单元测试框架文件。

配置JavaScript单元测试首选项

如果您使用Jasmine框架，请确保在以下位置选择框架版本 工具|单元测试|测试框架|的JavaScript测试页 ReSharper选项（ Alt+R,,O ）。

默认情况下，在开始测试时，ReSharper将启动默认的Web浏览器以显示单元测试框架的输出。它运行测试并将信息报告给ReSharper。如有必要，可以在“ 工具” | “ 工具”上更改浏览器。 单元测试|测试框架| ReSharper选项的“ JavaScript测试”页面。您也可以选择使用PhantomJS库运行测试，而无需启动浏览器。为此，选择PhantomJS并指定PhantomJS可执行文件的路径，并在必要时更改默认命令行参数。

您可以为测试指定自定义HTML工具。该线束与Chutzpah测试运行器兼容，可以使用特殊的Chutzpah占位符 。

在当前文档中运行测试

当前文档中有几种运行单元测试的方法。您可以使用操作指示器，主菜单或快捷方式：

  • 要运行一个测试或一个测试类中的所有测试，请单击其旁边的动作指示器或在测试/测试套件中设置插入符号，然后按Alt+Enter 。在操作列表中 ，选择“运行以测试”或“全部运行”以测试套件。

  • 或者，您可以使用 运行单元测试 Ctrl+T,R命令，这些命令也可以在主菜单（ ReSharper |单元测试 ）和上下文菜单中找到。这些命令的作用方式取决于插入符的位置或在编辑器中的选择：

    • 要在测试套件中运行单个测试或所有测试，请在测试/测试套件名称或编辑器中其声明内的任意位置设置插入符号。

    • 要运行多个测试，请在编辑器中选择所需的测试。

    • 要运行当前文件中的所有测试，请选择全部或在测试套件之外设置插入记号。

无论选择哪种方式运行测试，都将在“ 单元测试会话”窗口中看到执行进度，结果和输出。 如果存在开放的单元测试会话 ，则将已执行的测试添加到该会话中。如果没有测试会话或现有会话被锁定 ，则会创建一个新的测试会话。

如有必要，您始终可以通过按来重新运行上次执行的测试Ctrl+T,T或选择 ReSharper |单元测试|在菜单中重复上一个运行 。

发现解决方案中的单元测试

ReSharper将“ 单元测试资源管理器”窗口添加到Visual Studio（ ReSharper |单元测试|单元测试，或ReSharper | Windows |单元测试 ，或Ctrl+Alt+T ）。 使用此窗口，您可以浏览和运行整个解决方案中所有受支持框架的单元测试。请注意，来自项目的单元测试仅在构建项目后出现在窗口中。当前打开的文件中的测试会自动更新，创建后的新文件会立即显示在单元测试浏览器中。

单元测试浏览器允许您执行以下操作：

  • 探索解决方案中的测试：在列表或树视图中浏览所有单元测试，搜索测试并按子字符串过滤，按项目，名称空间重新组合单元测试，等等。

  • 通过在视图中双击导航到任何测试或测试套件的源代码。

  • 运行选定的测试。

  • 从所选测试和测试套件创建单元测试会话 ，和/或将所选项目添加到当前测试会话。

  • 将所有测试从解决方案导出到文本，XML或HTML文件。

在项目或解决方案中运行单元测试

您可以从单元测试资源管理器，解决方案资源管理器或类视图运行测试。单元测试资源管理器使您可以仅查看测试和测试套件，而使用其他窗口则需要知道哪些项目，文件和类包含测试。

  • 要从Unit Test Explorer执行测试，请选择所需的测试，然后点击 运行单元测试 Ctrl+T,R在工具栏上。

  • 要从解决方案资源管理器或类视图运行测试，请选择一个或多个包含测试的项目（套件，文件，文件夹，项目），然后使用 运行单元测试 Ctrl+T,R命令，这些命令也可以在主菜单（ ReSharper |单元测试 ）和上下文菜单中找到。

  • 要在解决方案中运行所有测试，请选择 ReSharper |单元测试|从主菜单中的“ 从解决方案运行所有测试”或按Ctrl+T,L 。

无论选择哪种方式运行测试，都将在“ 单元测试会话”窗口中看到执行进度，结果和输出。 如果存在开放的单元测试会话 ，则将已执行的测试添加到该会话中。如果没有测试会话或现有会话被锁定 ，则会创建一个新的测试会话。

如有必要，您始终可以通过按来重新运行上次执行的测试Ctrl+T,T或选择 ReSharper |单元测试|在菜单中重复上一个运行 。

使用单元测试会话

ReSharper允许您将针对应用程序特定部分的单元测试分组为单元测试会话。单元测试会话可以包含来自不同项目的测试。您可以有多个测试会话，并根据需要分别运行。单个测试/测试套件可以包含在多个不同的测试会话中。

有关与单元测试会话有关的详细说明，请参阅《 使用单元测试会话》 。

执行过程

ReSharper提供了几种执行单元测试的方法。无论选择哪种方式，执行进度，测试结果和输出都将显示在“单元测试会话”窗口中，而覆盖结果将显示在“ 单元测试覆盖率”窗口中。

在单元测试会话中运行测试时，执行进度将显示在“单元测试会话”窗口和“ 停止执行 ”工具栏下的状态栏中。 按钮变为活动状态。您可以单击此按钮来中止测试的执行。您可以同时运行多个单元测试会话。

如有必要，您可以启用“ 跟踪运行测试” 工具栏上的选项。如果已打开，则测试树中的选择将自动切换到当前正在运行的测试，并且输出面板在执行期间始终显示当前测试的输出。

您还可以在运行测试时启用自动滚动输出 在测试运行时跟踪其输出的选项。

分析执行结果和输出

执行结束后，结果将在“单元测试会话”窗口中可视化。

输出窗格（您可以使用“ 显示输出”将其放置在右侧或底部工具栏上的按钮）显示所选测试的输出。如果测试失败，ReSharper还将添加有关失败的简短信息和/或显示异常的堆栈跟踪。您可以使用输出窗格中的可单击链接直接导航到故障所涉及的类型和方法。如果输出显示指向不属于解决方案的文件的链接，则可以单击此链接在与相应文件类型关联的外部应用程序中打开该文件，或者Ctrl单击以在Visual Studio中将其打开。

默认情况下，ReSharper根据输出区域的当前宽度在输出中包装长行。如有必要，可以通过清除“ 在单元测试会话中将长行包裹”复选框取消选择不包裹长行。 工具| ReSharper选项的“ 单元测试”页面。
