代码检查：转换为lambda表达式

您可以禁止执行此检查以忽略特定问题 ， 更改其严重性级别以使问题更小或更明显 ，或者将其完全禁用 。

您可以使用lambda表达式来代替将匿名方法传递给委托人或使用lambda语句 。这仅适用于包含单个语句的方法。

lambda表达式和匿名方法都可以用于创建匿名函数，但是lambda表达式为此提供了较短的语法。使用这两种表示法之间有一些细微的区别-使用下面“ 另请参阅”部分中的链接了解详细信息。

如果所得的lambda表达式在同一行上包含不同的语句，则可以通过按以下命令在插入符所在的语句上设置断点F9 （而不是单击左边缘）。

在以下示例中，ReSharper建议为Lambda表达式分配一个sum委托，而不是使用更长的匿名方法：

次优代码

快速修复后

类MyClass {公共静态int MyMethod（）{ 函数总和=委托（int x，int y）{返回x + y; };返回sum（10，20）; }}
class MyClass {public static int MyMethod（）{Func sum =（x，y）=> x + y;返回sum（10，20）; }}

在以下示例中，ReSharper建议将语句lambda转换为lambda表达式：

次优代码

快速修复后

class MyClass1 {public static int MyMethod（）{Func sum =（x，y）=> {return x + y; };返回sum（10，20）; }}
class MyClass1 {public static int MyMethod（）{Func sum =（x，y）=> x + y;返回sum（10，20）; }}

逆向转换可与上下文动作一起使用 Alt+Enter 。
