转到衍生符号

ReSharper |导航|衍生符号
Ctrl+Alt+B
ReSharper_GotoInheritors

此命令获取当前符号的所有派生符号的列表，并让您直接沿继承层次结构导航至其中一个。有一个类似的命令Go to Implementation ，它仅列出实现，而在继承链中没有中间步骤。

除了导航之外，您还可以直观地浏览符号的完整继承层次。

您可以从编辑器，“ 文件结构”窗口和其他工具窗口中调用此命令。如有必要，可以使用“ 转到基本符号”命令沿相反方向导航。

导航到派生符号

  1. 将插入号放置在编辑器中的符号上，或在工具窗口中选择该符号。

  2. 按Ctrl+Alt+B或选择 ReSharper |导航|从主菜单派生的符号 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. 如果该符号只有一个衍生符号，则ReSharper会直接导航到该符号。
    如有必要，您可以更改默认行为，以便即使单个符号也显示在列表中。为此，请清除“ 环境” |“ 如果只有一个... ，则转到基类/继承者”复选框。 ReSharper选项的“ 搜索和导航”页面。

  4. 如果该符号具有多个派生符号，它们将在下拉列表中列出。 该列表可能包含粗体和常规字体的项目。粗体表示项目具有直接继承，而常规字体用于间接继承。

    您可以执行以下操作之一：

    • 单击所需的项目以在编辑器中将其打开。

    • 通过按选择所需的项目Up和Down然后按Enter在编辑器中打开它。

    • 开始输入以过滤结果。请注意，您可以在此处使用CamelHumps 。如有必要，请按Esc一次清除过滤。

    • 要在“ 查找结果”窗口中查看和分析匹配项的列表， 单击在查找结果中显示 ， 按Shift+Enter要么+在数字键盘上。

    • 要在类型依赖关系图上显示匹配的项目，请按Alt+Num+或单击在图表上显示 。

此命令也可以在“ 导航到”快捷方式中找到Ctrl+Shift+G 。

当类型或成员在当前解决方案中具有继承者时，您可以通过出现特殊图标（例如， ）显示在声明的左侧。您可以单击图标以展开派生符号列表并进行导航。

为了实时检测成员的继承者并显示相应的装订线标记，ReSharper必须执行一些计算。大型继承层次结构与弱硬件的组合可能导致性能下降。在这种情况下，您可以清除“ 显示已实施和已覆盖成员的装订线导航标记”复选框 环境|编辑器的编辑器外观页面 ReSharper选项可避免复杂的计算，从而提高性能。
另一方面，在类型上继承者的即时计算在性能上要便宜得多，因此不被禁用。
