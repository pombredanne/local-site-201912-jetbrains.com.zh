提取超类重构

ReSharper |重构|提取物|提取超类…
ReSharper_ExtractSuperclass

通过此重构，您可以将选定类中的某些成员提取到新的基类中。原始类将扩展创建的基类。
如果当前类型已经实现任何接口，则也可以将这些接口提取到新的基类中。
如果要创建抽象类，请标记至少一个提取的方法abstract。

您还可以使用“ 提取接口”重构从当前类中提取接口 。

提取基类

  1. 通过以下方式之一选择班级：

    • 在编辑器中，将插入号设置为类的名称。

    • 在解决方案资源管理器中选择一个类。

    • 在“ 文件结构”窗口中选择一个类。

    • 在“班级视图”中选择一个班级。

    • 在对象浏览器中选择一个类。

    • 在类型依赖关系图中选择一个类。

  2. 请执行以下任一操作：

    • 按Ctrl+Shift+R然后选择“ 提取超类”

    • 右键单击并选择“ 重构”。在上下文菜单中提取超类 。

    • 选择 ReSharper |重构|提取物|在主菜单中提取Superclass… 。

    将打开“ 提取超类”对话框。

  3. 在新文件或与原始类型相同的文件中，指定基类的名称以及应放置的名称。

  4. 选择要转移到新基类的成员。成员列表可以包含：

    • 当前类型的成员

    • 所选类型当前实现的接口

  5. 要制作/保留单个成员的摘要，请选中其旁边的“摘要”列中的复选框。如果至少有一个成员标记为abstract，则基类也将被创建为abstract。

  6. 要快速选择所有公共成员，请点击所有公共 。

  7. 如果选择引用其他成员的成员，则提取的类中将有未定义的符号。单击“ 从属”以通过选择所有依赖于当前选定成员的成员来解决冲突。

  8. 要应用重构，请单击下一步 。

  9. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。
