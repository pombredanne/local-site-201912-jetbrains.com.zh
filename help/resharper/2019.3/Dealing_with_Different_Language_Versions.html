处理不同的语言版本

随着编程语言的发展，采用新的语言功能来改进代码是很自然的。另一方面，某些因素可能会阻止您升级到最新的语言版本。

ReSharper知道不同的语言版本。它分析代码并根据当前语言版本应用其自身的功能。将自动检测每种语言的语言版本，但是您可以如下所述手动设置某些语言的版本。

C＃

ReSharper 2019.3完全支持高达C＃8.0的所有C＃版本。 默认情况下，ReSharper根据关联的编译器自动检测C＃版本。但是，您可以显式指定目标C＃版本-在解决方案资源管理器中右键单击项目，从上下文菜单中选择“ 编辑项目项属性” ，然后使用C＃语言级别选择器。

VB.NET

ReSharper 2019.3完全支持所有VB.NET版本（最高VB 14和部分VB 15）。 默认情况下，ReSharper根据关联的编译器自动检测VB.NET版本。但是，您可以显式指定目标VB.NET版本-右键单击解决方案资源管理器中的项目，从上下文菜单中选择“ 编辑项目项属性” ，然后使用VB.NET语言级别选择器。

打字稿

ReSharper 2019.3完全支持所有2.0版以下的TypeScript版本。ReSharper会通过以下方式自动检测TypeScript版本： Visual Studio项目文件中的属性。但是，您可以使用“ 代码编辑” | “ 代码编辑”上的TypeScript语言级别选择器显式指定目标TypeScript版本。打字稿| ReSharper选项的“ 检查”页面。

如果自动检测到TypeScript版本（默认情况下为默认），并且您的解决方案中有多个项目具有不同的TypeScript版本，则ReSharper将为整个解决方案使用最高版本。

的JavaScript

ReSharper完全支持JavaScript 直到ECMAScript 2016，包括实验性功能，例如异步/等待，指数运算符以及对象文字/解构中的休息/传播。 jQuery和JSX语法也受支持。

默认情况下， 代码检查和其他ReSharper功能会根据ECMAScript 5标准对JavaScript代码进行分析，该标准得到了普遍支持。如果在项目中使用更高级的JavaScript代码，则可以在“ 代码编辑” |“更改”中更改目标ECMAScript级别。 JavaScript | ReSharper选项的“ 检查”页面。

C ++

ReSharper 2019.3支持C，C ++ 03，C ++ / CLI和部分C ++ 11，C ++ 14，C ++ 17，C ++ 20。 ReSharper基于平台工具集（项目属性中的General | Platform Toolset ）和/或/ std开关自动检测C ++版本。

ReSharper C ++提供了C ++支持， ReSharper C ++是一种专用产品，可以单独安装，也可以与ReSharper或ReSharper Ultimate并排安装。

的CSS

ReSharper支持CSS直至4.0版本。实际上，CSS版本远不如其他Web浏览器支持的CSS功能集重要。因此，ReSharper允许您针对特定Web浏览器的特定版本调整其CSS代码检查 。您可以在“ 代码编辑” |“配置”中对其进行配置。 CSS |检查选项页面。
