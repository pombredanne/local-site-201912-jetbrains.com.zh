产生平等成员

ReSharper |编辑生成代码|平等成员
Alt+Insert | 平等成员
ReSharper_GenerateEqualityMembers

执行相等方法（即Equals()和GetHashCode() ）以及相等运算符（即==和!= ）中的Object类保证引用相等。在您创建的类型中（最终从Object因此默认情况下实现引用相等），您可能希望为此类型的对象实现值相等，并将哈希码用作唯一的对象标识符以进行哈希处理。在这种情况下，您需要覆盖类型的相等方法和运算符。

ReSharper允许您使用“ 生成相等成员”命令自动执行这些例程。

在下面的示例中，此命令用于生成Equals()和GetHashCode()基于的方法_radius和_center领域。

您可以在上配置通用代码生成首选项 代码编辑|的会员生成页面 ReSharper选项（ Alt+R,,O ）。

为相等成员生成替代

  1. 在编辑器中，将插入号设置为类型名称，或者在要插入相等成员替代的行的类型内。如果插入符号在类型名称上，则生成的代码将添加到类型声明的开头。

  2. 按Alt+Insert或选择 ReSharper |编辑从主菜单生成代码… 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. 在“ 生成”弹出窗口中，选择“ 平等成员” 。

  4. 在出现的“ 生成”对话框中，选择要比较的字段Equals()方法，并将其包含在GetHashCode()方法。

    如果您的类型中没有字段/属性，或者您未选择任何字段/属性，ReSharper会根据您的设置抛出新的NotImplementedException() ，返回默认值，或将无法编译的代码放入生成的方法的主体中。您可以在“ 代码编辑” |“配置”中配置设置。 ReSharper选项的“ 成员生成”页面。

    （可选）在对话框中使用以下控件：

    • 字段可以为null-如果您的类型中包含任何可为空的字段或属性，则显示该字段。默认情况下，此复选框处于选中状态，这意味着ReSharper将为所选字段生成空检查。如果不需要空检查，则可以清除此复选框。

    • 字符串比较 -如果您的类型中有任何字符串字段，则会出现。ReSharper将生成string.Compare(string, string, StringComparison)对于选定的字符串，选择器使您可以选择为StringComparison参数生成的值。

    • 如果相应的替代已经存在，则GetHashCode已经存在并且Equals已经存在，并让您选择是否：

      • 如果已经存在，请替换该方法。

      • 将新生成的方法与现有方法并排放置。

      • 完全跳过生成新方法。

    • 重载相等运算符 -生成自定义相等和不相等运算符（即>==和!= ）的当前类型。

    • 实现IEquatable 接口实现特定于类型的接口 Equals()方法。

    • 比较类型检查 -允许您选择Equals()方法将比较对象类型：

      • 类型与此完全相同

      • 输入与所有者类型完全相同的类型

      • 类型是所有者类型或子类型

  5. 单击完成以完成向导。
    您也可以在“ 代码编辑” |“选项”上单击“ 选项”以查看或修改常见的代码生成首选项。 ReSharper选项的“ 成员生成”页面。

您还可以通过在“ 生成”菜单中选择“ 覆盖成员 ”来生成两个覆盖，但是在这种情况下，它们都将返回基本方法。

作为在类型内生成相等成员的一种替代方法，可以为您的类型生成相等比较器类 。

要将此代码生成命令绑定到其自己的快捷方式，请转到“ 工具” |“其他”。选项|环境|键盘并为ReSharper_GenerateEqualityMembers命令。
