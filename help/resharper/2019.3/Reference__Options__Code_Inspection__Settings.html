突出显示上下文出口

默认情况下启用此选项，它告诉ReSharper突出显示控制流可以退出当前上下文的所有位置。例如，对于一个方法，它将突出显示该方法的返回类型，所有return ， throw关键字，等等，当您将插入符号设置为这些标识符之一时

对于循环，它将另外突出显示loop关键字以及所有break此循环中的语句。
请注意，如果方法在编辑器中不完全可见，则可以在方法名称上调用“ 导航到功能退出”命令以触发另一种突出显示，当插入符号离开方法名称时，该突出显示不会消失。

启用解决方案范围的分析

启用解决方案范围的分析 （包括解决方案范围的代码检查 ），默认情况下将其禁用。

请注意，在大型解决方案中，解决方案范围的分析可能会导致性能下降。但是，有多种方法可以提高解决方案范围分析的性能 。

包括警告

在解决方案范围内的分析中启用警告。

在解决方案范围的分析中启用警告时，其工作方式如下：只要解决方案中存在错误，就只会显示错误；如果解决方案中存在错误，则仅显示错误。未解决的警告仅在修复了最后一个错误后才会出现。禁用警告后，一旦纠正了最后一个错误，状态栏指示器就会变为绿色。

关闭解决方案范围的分析后，显示未使用的非私有类型成员

启用此选项后，即使禁用了解决方案范围的分析 ，ReSharper也会检测所有未使用的非私有成员。此选项对中小型解决方案没有明显的性能影响。
但是，如果在大型解决方案中或在处理非常大的源文件时遇到性能问题，则可能要禁用它。

使用弹出窗口显示“导入名称空间”操作 如果选择此选项，则在文件中检测到一种或多种未导入的类型时，将显示一个弹出窗口，提示在C＃和VB.NET中导入名称空间 ： 如果未选中该复选框，则相应的操作将在以下位置的操作列表中可用Alt+Enter 。

此选项不影响C ++。要切换C ++的导入弹出窗口，请在上使用相应的选项 代码编辑| C ++ |的检查页面 ReSharper选项。

价值分析模式

使用值分析 ，ReSharper找出哪些实体可以保留空值，并使用空解除引用突出显示可能的错误。您可以选择以下模式之一。

  • 乐观：当显式标记为[CanBeNull]属性或检查为null时-ReSharper假定只有显式标记为CanBeNull或ItemCanBeNull属性或显式检查为null的实体可以为null。

  • 悲观主义：当实体不具有显式[NotNull]属性时-ReSharper假定没有显式NotNull或ItemNotNull属性的所有可为空的实体都可以为null。

忽略的项目

ReSharper将不会为被忽略的项目建立索引，这意味着这些项目中将没有ReSharper功能，例如代码分析 ， 重构 ， 代码生成 。

该列表的主要目的是通过不对包含第三方代码或冻结代码（不会更改）的项目进行索引和分析，从而提高性能 。

尽管没有分析被忽略项目中的源文件，但ReSharper仍将处理这些项目输出中的编译代码（这在性能上非常便宜）。这意味着，在构建了被忽略的项目之后，ReSharper将正确解析这些项目中符号的用法，您将能够在这些项目中进行导航和搜索 。

因此，如果将一些项目添加到忽略列表后，您遇到诸如“无法解析符号”之类的错误，则需要构建被忽略的项目，ReSharper将能够从编译的工件中解析那些符号。

要将项目添加到忽略列表，请使用项目文件名（例如ProjectOne.csproj）或示例文件 （例如Project * .csproj）来匹配ProjectOne.csproj ， ProjectTwo.csproj等。

请注意， 从命令行运行代码检查时， 将不会忽略此处列出的项目。

要跳过的元素

ReSharper允许您配置应从代码检查中排除的文件，文件掩码和文件夹的列表。设计时代码检查和特定范围内的代码检查都将忽略被排除的项目，但是ReSharper仍对它们进行索引，因此您可以导航到或重构从代码检查中排除的符号。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
