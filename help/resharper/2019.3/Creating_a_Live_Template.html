创建实时/环绕声模板

实时/环绕声模板可用于在现有文件中生成代码，并且（如果存在$SELECTION$模板主体中的参数 ），用于包含模板代码的周围代码片段 ，例如try...catch声明。
每个活动模板都由一个快捷方式 （一个简短的字符串，例如foreach ）标识，您可以在编辑器中直接键入以调用该模板。

您的新实时/周围模板可以方便其他开发人员使用。根据需要，您可以在不同的设置层（包括团队共享层）中创建新的代码模板。如有必要，您可以稍后将其复制到团队共享的设置层或将其导出到文件中来 共享 。研究ReSharper 存储模板的方式可能也很有用。

实时/环绕声模板可用于多种用途，例如用于创建类，DTO或用于单元测试的框架。

实时/周围模板的主体由纯文本和parameters组成，纯文本将原样显示。想法是将这些结合起来以在调用模板时最大化效率。

创建新的实时/周围模板

  1. 在主菜单中，选择 ReSharper |工具|模板资源管理器… 。

  2. 在出现的“ 模板资源管理器”窗口中，单击“ 实时模板”或“ 环绕模板”选项卡。

  3. （可选）如果要在特定的设置图层中创建模板，请在“ 图层”列表中选择所需的图层 。要了解更多信息，请参阅模板存储 。

  4. 点击新模板 。在模板编辑器中打开新创建的模板 。

  5. 在“ 快捷方式”字段中指定模板快捷方式 。从模板创建代码时，它用于快速调用模板 ;它也出现在完成列表中 。在“ 描述”字段中，输入模板的可选描述。

    ReSharper不会将每个代码模板保存在单独的文件中。因此， 快捷方式标识符不必是唯一的。但是，请尝试提供可理解的标识符，以免在应用自定义代码模板时造成混淆。

  6. 要更改模板的默认范围 ，请单击“ 可用性”超链接。在出现的对话框中，使用复选框定义可以在何处应用新模板。您可以扩展某些范围以指定详细信息（例如，文件掩码或语言版本）：

  8. 使用“在...中使用”选择器，指定模板是否可用于生成新代码，围绕现有代码或同时使用这两者。如果您要将创建的模板用于周围的代码，请添加$SELECTION$参数，代表用户在调用模板之前选择的文本。

    • 如果指定了一些助记符数字，则按时，模板将在“ 环绕...”下可用Alt+Enter在选择上。

    • 按下时，指定的助记符数字将出现在模板旁边Ctrl+Alt+J （或选择 ReSharper |编辑在菜单中环绕模板... ）。然后，您可以按相应的键快速应用此模板。

  10. 配置模板格式选项：

    • 选择重新格式化复选框以使 ReSharper会根据您的格式首选项设置自动重新格式化扩展的代码片段。

    • 选择“ 缩短合格的引用”复选框以进行 ReSharper会自动插入名称空间导入指令或删除应用模板后变得多余的限定符。如果未选中此复选框，则ReSharper将不会插入任何名称空间导入指令。

  11. 要保存模板，请选择“ 文件” | “模板” 。在Visual Studio菜单上保存所选项目或按Ctrl+S 。

创建新模板后，您可能需要按类别组织模板 。
