代码检查

ReSharper提供了所有受支持语言的2500多种代码检查。这些检查用于在设计时检测和突出显示所有打开文件中的代码问题 ，和/或查找特定范围内的所有代码问题，这些问题可能与整个解决方案一样大。

要了解ReSharper提供的代码检查类型，请查看不同语言的ReSharper代码检查的完整列表 。

处理编辑器中突出显示的代码问题的最简单方法是将插入符号设置在突出显示的代码项上，然后按Alt+Enter或单击出现在代码问题左侧的操作指示器 。ReSharper会在其中列出针对大多数问题的一个或多个快速修复的操作列表 。

代码检查可以分为以下几类：

  • ReSharper自己的代码检查包括：

    • 固定严重性级别为“错误”的检查 。这些检查可以检测到编译器错误，无法禁用或配置它们。

    • 具有可配置的严重性级别的检查，可检测其余代码问题（例如，编译器警告，运行时和逻辑错误，代码气味，冗余，改进建议等）。您可以在以下位置找到这些可配置的检查 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ）。可以通过多种方式配置这些检查-您可以使用注释和属性抑制检测到的问题，禁用它们或在设置或.editorconfig文件中更改其严重性级别。

  • 自定义代码检查定义为结构搜索和替换模式 ，也具有可配置的严重性级别。

通过ReSharper扩展，您可以针对特定目的进行更多检查。

代码检查的严重性级别

每个ReSharper代码检查以及结构搜索和替换模式都具有以下严重性级别之一：

您可以在“ 代码检查索引”中找到每个可配置检查的默认严重性级别和其他属性-只需选择一个语言页面，然后使用浏览器搜索来查找所需检查的详细信息。

错误

默认情况下具有“错误”严重性级别的代码检查针对的是代码问题，这些问题会阻止代码编译或导致运行时错误。这些检查大多数都是不可配置的，即您不能禁用它们或更改其严重性级别。

在设计时检查中，ReSharper用红色显示未解决的符号：

并用红色的下划线突出显示错误的陈述或其中的一部分：如果当前文件中至少有一个错误，则状态指示器显示错误 图标，并在标记栏上为每个错误显示红色标记。

如果启用了解决方案范围的分析，则在严重性级别为“错误”的检查中发现的代码问题将显示在“ 解决方案中的错误/警告”窗口中 。

警告

此严重性级别对应于编译器警告以及其他不会阻止您的代码进行编译，但仍然可能表示严重的代码效率低下的问题。例如，ReSharper会通知您有关冗余类型强制转换或名称空间导入指令，错误的格式字符串，已声明但从未使用过的局部变量或私有字段，未使用的私有方法等的信息。

在设计时检查中，ReSharper会显示带有灰色文本的冗余符号：

并用蓝色的下划线突出显示语句或语句的一部分：如果当前文件中至少有一个警告，则状态指示器显示警告 图标，并在标记栏上为每个警告显示黄色标记。

如果启用了带有警告的解决方案范围的分析 ，则警告也将显示在“ 解决方案中的错误/警告”窗口中

建议

具有此严重级别的代码问题提供了对代码结构的深刻见解，将您的注意力吸引到不一定是不好或错误的事情上，但可能有助于您了解。

例如，您可能想知道“可以使用基本类型声明参数”，这意味着您的方法仅使用参数基本类型的成员。这可能表明存在其他错误，例如未能考虑实际参数类型的细节。基于此知识，您可能需要更改参数的类型-如果是设计使然，则可以更改。

在设计时检查中，ReSharper用绿色的卷曲下划线突出显示建议：

并在标记栏上为每个警告添加绿色标记。

暗示

这是最低的严重性级别。具有这种严重性的代码问题会引起您对特定代码细节的关注和/或建议一种改进方法。

在设计时检查中，ReSharper通过在相应符号的前两个字母上添加绿色虚线来突出显示提示：

与错误，警告和建议不同， 在编辑器中的代码问题之间导航时不会考虑提示，并且不会在标记栏中显示提示。

请注意，将尖号设置为突出显示的提示时，不会显示任何弹出窗口，相应的消息仅显示在状态栏中。

如有必要，您可以禁用任何可配置的代码检查。

在Visual Studio项目设置中，您可以选择将警告视为错误。ReSharper知道此选项并相应地突出显示警告：如果启用此设置，则与编译器警告相对应的问题将突出显示为错误。此设置在项目属性配置： 项目| [项目名称]属性|将警告视为错误 ，如果选择全部或仅将其应用于指定警告，则可以将其应用于所有警告。

可以更改大多数代码检查的严重性级别（检测到编译器错误的严重性级别除外），也就是说，您可以根据需要为每个代码检查定义严重性级别。

当您检查特定范围内的代码时 ，ReSharper会在检测到的问题中添加相应的图标，并允许在“ 检查结果”窗口中按严重性级别对问题进行排序。

代码检查的类别

ReSharper按几个类别将可配置的代码检查分组。这些类别大致定义了检查的目的以及它们检测到的各种代码问题。类别用于将代码检查分组 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ），并将在特定范围内发现并显示在“ 检验结果”窗口中的代码问题分组。

  • 潜在的代码质量问题
    此类检查包括检测关键问题（代码异味）的检查，主要是错误或警告级别。此类别还包括确保本地化协助的检查 。

  • 通用惯例和代码改进
    此类别将检查归类为主要影响代码可读性的中等严重性问题。

  • 代码中的冗余
    此类别中的代码检查查找冗余和无效代码，这些代码会影响代码的可读性和样式，并且可以安全地删除。 某些代码冗余无法自动修复，因此需要在用户输入的情况下以交互方式进行快速修复。但是，可以使用范围内的修复或代码清除来修复大多数冗余，而无需用户交互。

  • 语言使用机会
    此类别包括代码检查，大部分带有建议严重性级别，这些通知会在可以使用更高级的语言构造时通知您。这些检查将检测过时的语言版本的语法，并建议使用更多现代语言版本的功能。对于大多数受支持的语言， 可以自动检测语言版本或手动设置 。

  • 代码通知
    此类别将代码检查的严重性级别较小。

  • 代码风格
    此类别中的检查可检测到违反代码语法样式的行为。与大多数其他代码检查相比，这些检查可以检测与代码发布相同的代码构造，也可以不检测，具体取决于在代码上配置的相应代码样式规则。 代码编辑| [语言] |的代码样式页面 ReSharper选项（ Alt+R,,O ）。您还可以使用代码清除修复这些检查检测到的问题。

  • 约束违规
    此类别包括代码检查，大多数情况下具有警告严重性级别，该代码检查可检测与符号属性相关的违规行为，包括ReSharper的代码注释和其他类似问题。

  • 符号声明中的冗余
    此类别包括代码检查，大多数情况下具有警告严重性级别，可检测空的和未使用的符号声明。

  • 编译器警告
    此类检查将在编译之前检测到编译器警告。

  • 拼写问题
    这些检查可以检测各种情况下的错别字。

  • 不可运行的测试
    这些检查可以检测到无法执行的测试。

  • NUnit
    这些检查可以检测与NUnit测试有关的代码问题。

  • Xunit
    这些检查可以检测与xUnit.Net测试相关的代码问题。

  • 格式化
    此类别中的检查可检测代码编码格式问题。

  • lang整齐的支票
    此类检查由Clang-Tidy提供， Clang-Tidy是与ReSharper C ++ 集成的功能强大的开源代码分析工具。

  • 铛
    此类别中的检查对应于与ReSharper C ++ 集成的 Clang编译器警告。

  • lang静态分析仪检查
    此类别中的检查是来自与ReSharper C ++ 集成的 Clang静态分析器的诊断。
    默认情况下，所有静态分析器检查均处于禁用状态，因为启用它们会大大降低Clang-Tidy的速度。

  • 虚幻引擎
    此类别中的检查特定于Unreal Engine项目的C ++代码。

可配置代码检查的标识符

每个可配置的代码检查都有两个唯一的标识符，您可以使用它们来进行配置。让我们以“ 可能的系统”为例。以NullReferenceException'检查为例：

  • 检验ID用于取消带有评论的检验-

    //一次禁用ReSharper可能的NullReferenceException
    对于一个问题或
    // ReSharper禁用PossibleNullReferenceException
    直到文件末尾，
    或带有属性-
    [SuppressMessage（“ ReSharper”，“ PossibleNullReferenceException”）]
    。

  • Inspection EditorConfig属性可用于从.editorconfig文件配置检查。例如，您可以使用以下行将检查的严重性级别更改为“ 错误 ”：

    resharper_possible_null_reference_exception_highlighting =错误

您可以在“ 代码检查索引”中找到每个可配置检查的标识符-只需选择一个语言页面，然后使用浏览器搜索来查找所需检查的详细信息。
