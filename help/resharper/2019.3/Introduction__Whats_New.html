ReSharper 2019.3帮助

ReSharper的新功能

ReSharper / ReSharper C ++ 2019.3

  • 改进的C＃8支持，包括： notnull约束，目标类型switch表达式，早期支持default interface方法，新提示“本地函数可以是静态的”和新的快速修复“添加缺少的武器”。

  • 输入名称提示 lambda的参数， var包括模式匹配和元组解构，LINQ语句内的范围变量以及方法调用链中一行的末尾。

  • Xamarin支持修复： x:DataType和Class属性支持。

  • 现在， 查找依赖代码可用于NuGet引用，并可在整个解决方案中运行。在查找结果中也有新的过滤器用于相关代码。

  • ReSharper C ++ 2019.3的新功能：

    • 最初支持C ++ 20概念。

    • 整洁的集成改进：

      • 捆绑的clang-tidy二进制文件通过许多新检查已更新到版本9。此外，它不应再阻止对源文件的修改。

      • 现在可以从检查菜单禁用叮当警告。

      • 使用特定的clang-tidy配置文件的新设置（而不是将配置指定为字符串的设置）

    • 将格式选项导出到.clang格式的文件。

    • 多亏了代码突出显示引擎的一些改进，现在重新突出显示的速度更快了（更改函数的主体之后，将仅对主体本身进行重新分析）。

    • 新的快速修复程序有助于将变量添加到lambda捕获列表。

    • 虚幻引擎4支持改进：

      • UE智能指针（ TSharedPtr ， TSharedRef ， TWeakPtr ， TUniquePtr ）的支持方式与std::shared_ptr / unique_ptr （例如，它们受Parameter Info ， 代码分析和postfix完成的支持 ）。

      • 的新的后缀模板 UCLASS扩展到适当的工厂函数调用。

      • 新USTRUCT / UCLASS / UENUM / UENUMCLASS实时模板可用。

      • 在虚幻引擎4编辑器中执行“刷新Visual Studio项目”操作后，不再有Visual Studio挂起。

    • 重命名了几个实时模板以匹配相应的关键字（例如， cls至class ， str至struct ）。

    • 转到文件成员中的几个更新：现在立即显示对话框，而不是等待文件被完全分析，如果文件同时包含函数的声明和定义，则该函数仅在对话框中显示一次。

    • 现在可以从括号括起的初始化程序列表的开头括号导航到聚合类。

  • ReSharper C ++ 2019.2的新功能：

    • 改进的虚幻引擎4支持：

      • 重命名和更改签名重构可正确处理UE4 RPC功能。
