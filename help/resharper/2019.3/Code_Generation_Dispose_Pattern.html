生成处置模式

ReSharper |编辑生成代码|处置模式
Alt+Insert | 处置模式
ReSharper_GenerateDisposePattern

ReSharper可帮助您自动生成以下内容的各种实现IDisposable根据您班级拥有的资源的种类：

  • 如果您的班级仅拥有托管IDisposable资源，ReSharper将帮助您生成一个简单的实现public void Dispose()没有参数的地方Dispose()在选定的一次性构件上被调用。

  • 如果您的班级仅拥有非托管资源， ReSharper将另外生成析构函数和一个ReleaseUnmanagedResources您可以在其中编写清除代码的方法。

  • 如果您的课程拥有或可能（通过继承）拥有托管和非托管资源， ReSharper将另外创建一个Dispose(bool disposing)继承者可以重写的方法。

要配置一次性图案的实现方式，请使用“ 生成”对话框中的选项 。

生成处置模式

  1. 在编辑器中，将插入记号设置为类型名称，或在要插入处置模式的行的类型内。如果插入符号在类型名称上，则生成的代码将添加到类型声明的开头。

  2. 按Alt+Insert或选择 ReSharper |编辑从主菜单生成代码… 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. 在“ 生成”弹出窗口中，选择“ 处理图案” 。

  4. 在出现的“ 生成”对话框中，您将看到一个私有属性列表和可分配给IDisposable 。选择这些类型成员中的一些或全部，ReSharper将在当前类型中生成将执行委托给所选类型成员的包装器。

    （可选）在对话框中使用以下控件：

    • 字段可以为null-如果您的类型中包含任何可为空的字段或属性，则显示该字段。默认情况下，此复选框处于选中状态，这意味着ReSharper将为所选字段生成空检查。如果不需要空检查，则可以清除此复选框。

    • 我的资源不受管理 -如果选中此复选框， ReSharper将另外生成析构函数和一个ReleaseUnmanagedResources您可以在其中编写清除代码的方法。

    • 我打算从该类继承 -如果选中此复选框， ReSharper将另外创建一个Dispose(bool disposing)继承者可以重写的方法。

    • 处置已存在 -如果实施Dispose()已经存在，您可以选择是否：

      • 如果已经存在，请替换该方法。

      • 将新生成的方法与现有方法并排放置。

      • 完全跳过生成新方法。

    • 析构函数已存在 -如果您选择了“ 我拥有非托管资源”选项并且该类已经具有析构函数，则将出现。使用此选择器选择是否：

      • 如果已经存在，请替换该方法。

      • 将新生成的方法与现有方法并排放置。

      • 完全跳过生成新方法。

  5. 单击完成以完成向导。
    您也可以在“ 代码编辑” |“选项”上单击“ 选项”以查看或修改常见的代码生成首选项。 ReSharper选项的“ 成员生成”页面。

如果出现以下情况，此操作也可以作为快速解决方案 IDisposable未实现。

要将此代码生成命令绑定到其自己的快捷方式，请转到“ 工具” |“其他”。选项|环境|键盘并为ReSharper_GenerateDisposePattern命令。
