单元测试资源管理器窗口

ReSharper | Windows |单元测试
ReSharper |单元测试|单元测试
Ctrl+Alt+T

ReSharper的单元测试运行程序不允许运行两个或多个具有相同ID的测试。在实践中，这种情况很少发生，但是如果您确实有具有相同ID的测试，它们将在“单元测试”浏览器和“单元测试会话”窗口中显示为单个节点，并且仅运行其中一个。
请注意，测试框架以不同的方式识别测试。例如，MSTest使用测试方法FQN，而NUnit使用测试属性的属性作为测试ID。

使用此窗口，您可以浏览和运行或调试整个解决方案中所有受支持框架的单元测试。请注意，来自项目的单元测试仅在构建项目后出现在窗口中。当前打开的文件中的测试会自动更新，创建后的新文件会立即显示在单元测试浏览器中。有关更多信息，请参见在项目或解决方案中使用单元测试 。

要搜索测试，请在焦点进入此窗口后立即开始输入测试名称或其CamelHumps缩写。

更新窗口内容以反映代码中或外部所做的任何更改。

运行单元测试 Ctrl+T,R

运行选定的测试。要选择多个文本，请在按住它们的同时单击它们。 Ctrl键。

您可以使用集成的下拉选择器对所选测试执行其他操作：

封面单元测试 Ctrl+T,H 通过单击此按钮，可以开始覆盖所选测试的代码 。

如果JetBrains dotCover与Visual Studio集成在一起，则显示此按钮。

在dotMemory单元下运行单元测试 Ctrl+T,M

在启用dotMemory单元的情况下运行选定的测试测试。使用此按钮可以运行使用dotMemory Unit框架检查代码的内存问题测试。

配置文件单元测试 通过单击此按钮，可以分析所选的测试 。

如果JetBrains dotTrace与Visual Studio集成在一起，则显示此按钮。

运行单元测试直到失败

对不稳定的测试使用此操作。一旦单击它，选定的测试将开始循环运行，直到其中一个失败。您可以通过单击“ 停止执行”来中止此循环 。

调试单元测试 Ctrl+T,D

开始调试当前会话中的选定测试。

封面单元测试 Ctrl+T,H 通过单击此按钮，可以开始覆盖所选测试的代码 。

如果JetBrains dotCover与Visual Studio集成在一起，则显示此按钮。

配置文件单元测试 通过单击此按钮，可以分析所选的测试 。

如果JetBrains dotTrace与Visual Studio集成在一起，则显示此按钮。

将测试追加到会话 Ctrl+T,A

单击此按钮可将所选测试添加到测试会话中 。ReSharper将建议选择现有会话之一或创建一个新会话。

创建新会话 Ctrl+T,N

使用选定的测试创建一个新的测试会话 ，然后在“ 单元测试会话”窗口中将其打开 。

全部展开 / 全部折叠

展开/折叠当前选项卡中的所有节点。

上一页 / 下一页
Shift+F8 / F8

导航到上一个/下一个项目，并在源代码中滚动。

出口

点击这个按钮以文本格式导出当前在窗口中显示的数据，或使用下拉选择器以XML或HTML格式导出数据。出现的“ 导出数据”对话框将帮助您将数据保存到文件或将其复制到剪贴板。

通过...分组

允许按不同类别对窗口中的项目进行分组。
