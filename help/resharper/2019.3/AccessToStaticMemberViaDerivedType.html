代码检查：通过派生类型访问类型的静态成员

您可以禁止执行此检查以忽略特定问题 ， 更改其严重性级别以使问题更小或更明显 ，或者将其完全禁用 。

在通过派生自其类型的类型访问该类型的静态成员的代码中会出现此错误。例如：

A类{public static int N; } B类：A {}静态void Main（string [] args）{BN = 3; //警告在这里}

上面的代码导致错误地认为该字段N属于类型B ，而事实并非如此。此外，如果类型A和B引入不同的程序集，然后Main()方法会引入不必要的依赖关系B的组装，否则可以避免。

调用工厂方法可能会导致更加混乱的情况。如果您定义一个变量var ，您得到的类型可能与您期望的不一样。例如

var request = HttpWebRequest.Create（“ http://someplace.com”）;

上面的代码表明Create()操作是类型的对象HttpWebRequest而实际上并非如此-结果是类型WebRequest最好编写如下：

var request = WebRequest.Create（“ http://someplace.com”）;

同样，C＃编译器使用声明类型创建对方法的调用，因此在IL中，实际上它将被调用为基本类型。如果您使用new修改器和对派生类型使用相同的签名，除非重新编译，否则代码行为不会改变。
