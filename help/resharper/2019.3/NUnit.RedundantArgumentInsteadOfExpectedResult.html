代码检查：NUnit。冗余参数，而不是ExpectedResult

您可以禁止执行此检查以忽略特定问题 ， 更改其严重性级别以使问题更小或更明显 ，或者将其完全禁用 。

NUnit的TestCase属性的参数可用于提供测试参数的值和预期的测试结果。必须始终使用命名参数指定预期结果ExpectedResult 。

如果测试方法返回一个值，并且ReSharper找到一个带有数据值而不是数据值的参数ExpectedResult ，它将发出警告并提供快速修复程序，将该值转换为预期结果。

快速修复后

[TestCase（100）] //警告：该参数是否表示ExpectedResult？public async Task Test（）{//做一些返回等待的任务。FromResult（100）; }
[TestCase（ExpectedResult = 100）]公共异步任务Test（）{//做一些返回任务的任务。FromResult（100）; }
