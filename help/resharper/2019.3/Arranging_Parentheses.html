代码语法样式：可选括号

在许多情况下，表达式中的括号是可选的。有时，它们可以帮助您弄清操作的优先顺序-例如if ((a & b) | c) ，但是有时，它们根本无济于事，只是累加了死代码- var d = a + (b + c); 。ReSharper将帮助您定义可选括号对您有帮助的地方，然后清除多余的括号或在必要时自动添加缺失的括号。

ReSharper可以帮助您在现有代码中安排可选的括号，并在使用时考虑您的偏好产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。

强制执行可选括号的首选项

默认情况下，ReSharper突出显示不能清楚说明操作优先级的括号，并有助于将其删除：

同样，它会通知您有关非显而易见的操作优先级，并建议添加可选的括号以使代码更具可读性：

在批量模式下对可选括号强制执行首选项的另一种方法是代码清除 。您可以使用默认配置文件“ 完全清理”或“ 重新格式化并应用代码样式”之一运行代码清理 ，或者创建和运行仅针对特定任务的自定义配置文件 ，如下所述。

可选括号的首选项不适用于生成的代码。默认情况下，ReSharper提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ）。

通过代码清除将首选项应用于可选括号

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 转至代码清理设置页面： 代码编辑|代码清理 。

  3. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 所选配置文件设置”部分中，选中“ 删除冗余括号”和“ 添加括号以显式指定操作优先级”复选框。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  5. 选择您要安排可选括号的范围：

    • 将插入号设置在文件中的任何位置，以将可选的括号排列到文件中。

    • 在解决方案资源管理器中选择一项或多项，以在这些节点及其子项下的文件中排列可选的括号。

  6. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  7. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  8. 点击运行 。ReSharper会在所选范围内排列可选的括号。

如果要在不打开“ 代码清除”对话框的情况下排列可选的括号，则可以将创建的配置文件绑定到静默清除并按以下键运行Ctrl+Shift+Alt+F 。您还可以创建一个自定义清除配置文件 ，该配置文件将应用可选的括号样式与其他代码样式任务结合在一起。

配置可选括号的首选项

您的可选括号首选项使用基于图层的设置机制保存。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

为可选括号配置首选项

  1. 去 代码编辑| C＃|的代码样式页面 ReSharper选项（ Alt+R,,O ）。

  2. 在“ 括号”类别中，指定是否删除多余的括号以及应该将哪些操作视为没有明显的优先级。

  3. 通过右栏中的“ 使用选择器通知”，您可以设置代码检查的 严重性级别 ，以检测与首选项不同的代码。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。
