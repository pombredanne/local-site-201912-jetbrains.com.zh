代码清理对话框

ReSharper |编辑清理代码
Ctrl+Alt+F
ReSharper_CleanupCode

“ 代码清理”对话框可帮助您使用可用的代码清理配置文件来清理选定范围中的代码，以及创建和编辑自定义清理配置文件 。

请注意，如果您在此对话框中创建或修改清除配置文件，则将使用智能保存逻辑来保存您的更改。如果需要将修改保存到共享设置层 ，请在“ 代码编辑” |“修改”中进行修改。 ReSharper选项的“代码清理”页面，单击“ 保存到” ，然后选择所需的设置层。

使用以下控件来运行和配置选定的清理配置文件：

项目

描述

工具栏按钮
设为默认

单击此按钮以使用选定的配置文件进行静默清除 。用于静默清除的配置文件标有图标。

重复

单击此按钮创建所选配置文件的副本。如果您具有默认设置，则需要复制一个默认清除配置文件以创建自定义清除配置文件。 创建新配置文件后，它将以编辑模式打开，您可以在其中配置清除任务列表。

配置

单击此按钮启用所选定制配置文件的编辑模式。注意无法编辑默认配置文件（ 完全清除 ， 重新格式化和应用代码样式以及重新格式化代码 ）。

改名

单击此按钮以重命名所选的自定义配置文件。注意无法重命名默认配置文件（“ 完全清除” ，“ 重新格式化和应用代码样式”和“ 重新格式化代码” ）。

去掉

单击此按钮删除选定的自定义配置文件。注意无法删除默认配置文件（ 完全清除 ， 重新格式化和应用代码样式以及重新格式化代码 ）。

完成了

单击此按钮退出编辑模式，然后返回到所选配置文件的预览。请注意，除非单击“ 保存”，否则将不会应用您的修改。

命令按钮
跑

单击此按钮可以在打开对话框之前，从所选清理配置文件中的选定范围中执行所有任务。请注意，范围可以从编辑器中的选择到整个解决方案。

保存

如果您创建或编辑了自定义清除配置文件，则会显示此按钮，并允许您保存修改。 请注意，如果您在此对话框中创建或修改清除配置文件，则将使用智能保存逻辑来保存您的更改。如果需要将修改保存到共享设置层 ，请在“ 代码编辑” |“修改”中进行修改。 ReSharper选项的“代码清理”页面，单击“ 保存到” ，然后选择所需的设置层。

取消/放弃

单击此按钮可关闭对话框，而不运行清理任务或保存修改。
