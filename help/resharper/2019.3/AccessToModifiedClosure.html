首先，让我们确保您了解什么是闭包 。简而言之，C＃中的闭包是lambda表达式或匿名方法，该方法从外部作用域捕获一些变量。这是最简单的示例：

//一个自包含的lambda。没有关闭。动作printOne =（）=> {控制台。WriteLine（“ one”）; }; //闭包-从外部作用域捕获变量的lambda。字符串myStr =“ one”;操作打印=（）=> {控制台。WriteLine（myStr）; };

在上面的示例中， print将捕获变量myStr （而不是它的值 ），并且只会获得的值 myStr当您调用print() 。

在更复杂的情况下，当在变化的上下文中定义闭包时，它的行为可能不符合预期。

这是在循环内定义上述闭包的示例：

var myActions = new List （）; var myStrings = new List （）{“一个”，“两个”，“三个”};对于（var i = 0; i {控制台。WriteLine（myStrings [i]）; }; myActions。添加（打印）; } myActions [0]（）;

令人惊讶的是，这段代码产生了一个ArgumentOutOfRangeException当我们调用myActions[0](); 。这里发生以下情况：而不是执行Console.WriteLine(myStrings[0]); ，这看起来很直观，此调用尝试执行Console.WriteLine(myStrings[i]);而且因为i涵盖了整个for周期，其值不等于0 ，甚至没有2 （这是最后一次满足条件的时间）。作为最后的结果++操作值变成3在条件变为假之前，我们退出循环。如myStrings只有3个元素， myStrings[3]导致ArgumentOutOfRangeException 。
