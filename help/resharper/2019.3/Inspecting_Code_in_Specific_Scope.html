查找特定范围内的代码问题

当您在编辑器中进行编码或查看特定文件时， 设计时代码检查将为您提供很大的帮助。但是，ReSharper还允许您在整个解决方案或更狭窄的范围内运行静态代码分析，并在友好的视图中检查结果。如果需要，您甚至可以共享检查结果。

您也可以从命令行检查特定范围内的代码，而无需启动Visual Studio和ReSharper。

开始代码检查

在解决方案，项目或自定义范围中检查代码

  1. 决定是否要在报告中包括解决方案范围内的检查结果 （例如，未使用的公共成员）。如果要包括这些结果，请启用解决方案广泛分析 。请注意，启用解决方案范围的分析后，即使需要检查单个项目，ReSharper也必须分析整个解决方案，因此请做好准备，以免花费更多时间。

  2. 请执行以下任一操作：

    • 在主菜单中，选择 ReSharper |检查解决方案中的代码问题或 ReSharper |检查当前项目中的代码问题 。

    • 在“ 解决方案资源管理器”中 ，选择要检查的项目（文件，文件夹，项目，解决方案文件夹），右键单击所选内容，然后在上下文菜单中选择“ 查找代码问题 ”。

    • 在“ 体系结构视图”中 ，选择一项或多项，右键单击所选内容，然后在上下文菜单中选择“ 查找代码问题 ”。

  3. “ 检查结果”窗口将打开，显示检测到的代码问题。

分析检查结果

在“ 检查结果”窗口中，您可以查看发现的代码问题的列表，并通过双击以下条目在编辑器中找到它们：

每次新的代码检查运行都会在窗口中打开一个新选项卡 ，因此将保留以前运行的结果，直到您关闭其选项卡为止。

默认情况下，问题按文件分组。如有必要，您可以使用“ 分组依据：”选择器按类型， 严重性等对问题进行分组 。

要检查所选代码问题的上下文，请使用“ 预览”窗格，该窗格在代码文件中显示与该问题相关的一行或多行。使用表演预览 选择器以打开窗口底部或右侧的窗格。

筛选检查结果

检查代码时，ReSharper会应用所有严重性级别的代码检查。但是，默认情况下会过滤掉“提示”严重性检查中检测到的问题。如有必要，您可以自定义代码问题的筛选。

请注意，所选的筛选会影响当前选项卡中的代码问题筛选方式以及下次运行代码检查时的问题筛选方式。

过滤检查结果中的问题

  • 如果要隐藏某种类型的问题或隐藏除此类型的问题以外的所有其他问题，请右键单击代码问题，然后分别选择“ 隐藏所有[问题类型]问题”或“仅显示[问题类型]问题” 。

  • 要进行更精确的过滤，请点击过滤问题 在工具栏上，然后使用“ 筛选问题”对话框来指定应显示的问题类型。当严重性级别为“提示”的问题被过滤掉时，此对话框中的“ 默认值”按钮将恢复默认过滤。

分享检查结果

在大型项目中， 代码检查结果可以多种方式使用。例如，要将发现的问题分配给不同的开发人员：您可以复制或导出代码问题或一组问题，然后将副本提交给问题跟踪器。

检查结果窗口和解决方案窗口中的错误/警告都允许将代码问题导出到文本，HTML或XML文件中。

将检查结果导出到文件中

  1. 在“ 检查结果”窗口中打开代码检查结果 ，或启用解决方案范围的分析，并在“解决方案中的错误/警告”窗口中打开错误列表。

  2. 点击导出 在工具栏上以文本格式导出当前在窗口中显示的数据，或使用下拉选择器以XML或HTML格式导出数据。出现的“ 导出数据”对话框将帮助您将数据保存到文件或将其复制到剪贴板。

  3. 将当前问题导出到XML报表的另一种方法是选择ReSharper |。检查在主菜单中保存问题报告 ，然后指定文件名和位置。

将检查结果保存为XML格式有几个优点：首先，此格式与InspectCode命令行工具和JetBrains TeamCity的输出格式兼容，其次，您以后可以在“ 检查结果”窗口中加载和研究它。无论用于生成XML报告的工具是什么，您始终可以使用ReSharper打开并检查它。

研究保存在XML报告中的检查结果

  1. 在主菜单中，选择“ ReSharper |检查加载已保存的问题报告 。

  2. 选择一个XML报告文件，然后单击“ 打开” 。

  3. 该报告将在“ 检验结果”窗口的新选项卡中打开 。如果当前解决方案与XML报告的解决方案相对应，则可以通过双击检查结果项来导航到问题。

除导出外，您还可以将“ 检查结果”窗口和“解决方案”窗口中的“ 错误/警告”中显示的单个条目或条目的自定义选择复制到剪贴板。复制的问题信息包括项目和解决方案的名称，文件名和行号以及问题的描述。例如：

Solution MySolution.sln
Project MyProject
MyProject\About.aspx.cs:2 Using directive is not required by the code and can be safely removed

要将发现的问题的信息复制到剪贴板

  1. 选择一个或多个代码问题，或在“ 检查结果”窗口或“解决方案”窗口中的错误/警告中选择问题组。

  2. 右键单击所选内容，然后在上下文菜单中选择“ 复制 ”。

  3. 将复制的问题粘贴到适当的位置。
