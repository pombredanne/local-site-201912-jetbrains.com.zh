从堆栈跟踪导航到异常

ReSharper |工具|浏览堆栈跟踪
Ctrl+Shift+E
ReSharper_ExploreStackTrace

当您收到外部堆栈跟踪（例如，来自错误报告）时，可以在“ 堆栈跟踪资源管理器”窗口中将其打开，并导航到相应异常起源的代码。在堆栈跟踪中，文件，类型和方法显示为超链接。您可以单击它们以在编辑器中显示相应的项目。

要返回以前打开的堆栈跟踪（在关闭解决方案之前一直可用），请选择ReSharper | BackSharper。 Windows |菜单中的堆栈跟踪 。

导航到导致异常的代码

  1. 将异常堆栈跟踪复制到剪贴板。

  2. 默认情况下，如果您复制包含当前打开的解决方案中的符号的堆栈跟踪，然后将焦点切换到Visual Studio，则ReSharper会自动将其加载到“ 堆栈跟踪资源管理器”窗口并将其带到视图中。如果您不喜欢此行为，请清除“ 工具” | “ 自动打开外部堆栈跟踪”复选框。 ReSharper选项的“ 堆栈跟踪资源管理器”页面。如果禁用了自动打开，请按照以下说明使用快捷方式或菜单命令。

  3. 在主菜单中，选择“ ReSharper |工具|浏览堆栈跟踪 ，或按Ctrl+Shift+E 。将打开“ 堆栈跟踪资源管理器”窗口 ，在新选项卡中显示剪贴板中的异常堆栈跟踪。

  4. 在“ 堆栈跟踪浏览器”窗口中，单击突出显示的项目以在编辑器中显示相应的代码。

  5. 如果您希望将长行包裹在堆栈跟踪中，请使用相应的工具栏按钮或在“ 工具” |“换行”中指定换行的默认首选项ReSharper选项的“ 堆栈跟踪资源管理器”页面。
