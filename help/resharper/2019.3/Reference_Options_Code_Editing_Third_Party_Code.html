第三方代码

ReSharper |选项|代码编辑|第三方代码

在此页面上，您可以配置ReSharper忽略的C ++，JavaScript，TypeScript，CSS，HTML和JSON文件，文件夹和文件掩码的列表。

请注意，在指定文件和文件夹时，您只能添加位于解决方案文件夹下的项目。要在解决方案文件夹之外指定代码，请使用文件掩码。

面具以开头**\并与绝对文件路径匹配。如果要匹配文件夹中的所有文件， \**面罩的末端也将需要。因此，例如， C：\ Foo \ Bar中的文件将被匹配Bar\**要么Foo\Bar\**要么C:\Foo\Bar\** （尽管较短的掩码也可能与不需要的文件夹匹配）。

检查当前文件是否与掩码匹配的最简单方法是在其中键入内容，并确保在此文件中禁用了设计时代码检查 。

您还可以在以下选项页面上以其他方式忽略这些语言和其他语言的代码：

  • 代码检查|从代码检查中排除文件，文件夹和掩码的设置 。

  • 代码检查|生成的代码可以部分禁用对生成的代码的代码检查 。

  • 环境|搜索和导航以指定“ 按名称搜索”命令应忽略的文件和文件夹。

跳过的代码

ReSharper完全忽略了本节中指定的代码-根本不会将其编入索引，这可能是您想要的第三方依赖项。当项目中有ReSharper通常会识别为C ++，JavaScript，TypeScript或JSON的文件时，这可能很有用-可能是生成的缩小文件或已压缩的脚本，并且您不希望内容出现在代码完成或导航和搜索结果 。

图书馆代码

ReSharper将本节中指定的代码视为非用户代码。内容仍然被索引，并出现在代码完成和导航中（例如，您可以使用Go to Symbol在这些文件中找到符号），但是ReSharper不会对此代码进行代码检查 ，也不会尝试快速修改此代码。 -修复和重构 。本部分对于包含第三方代码（例如jquery.js ）很有用，您无需维护自己，但需要完成建议，并且需要能够导航到那里检查实现细节。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
