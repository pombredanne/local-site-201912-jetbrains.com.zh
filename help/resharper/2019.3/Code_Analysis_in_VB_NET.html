代码检查

ReSharper的静态代码分析可以检测VB.NET代码中150多种不同的错误和问题。

通过对当前文档或在任何指定范围内进行代码检查来执行分析。

要查看VB.NET可用检查的列表，请打开 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ），然后展开VB.NET节点。

解决方案范围内的分析

ReSharper不仅分析当前文件中的错误，还考虑了文件之间的依赖性来检查整个解决方案。它在“解决方案中的错误”窗口中显示分析结果。有关更多信息，请参见解决方案范围分析 。

快速修复的例子

删除多余的“导入”

如果没有使用来自特定名称空间的任何符号，则对应的Imports指令被认为是多余的。

ReSharper提供了快速修复程序，可从文件中删除所有此类指令。

导入类型

如果您使用的符号来自未导入的名称空间，则ReSharper建议导入相应的名称空间并提供必要的快速修复。

添加“异步”修饰符

异步操作比同步编程具有一些优势，因此ReSharper与时俱进，并完全支持异步编程的语言功能。

的GetQuotesAsync函数包含await运算符，但该函数未定义为异步函数。ReSharper会检测到这种不匹配，并使用添加“异步”修饰符快速修复提示您改进代码。应用快速修复后，将缺少的修饰符添加到函数声明中。

变更类型

如果方法的参数类型与相应的方法参数的类型不匹配，ReSharper建议更改参数的类型并提供必要的快速修复。

从构造函数参数初始化自动属性

如果您具有构造函数参数，并且想要使用参数的值初始化现有的自动属性，请使用此快速修复。

从用法创建方法

如果尚不存在对方法的调用，则ReSharper将提供创建此方法所需的快速修复。
