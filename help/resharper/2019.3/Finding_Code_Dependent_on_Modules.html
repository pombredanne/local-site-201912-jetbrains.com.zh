查找依赖于引用模块的代码

ReSharper |查找|查找相关代码
ReSharper_FindDependentCode

您可以快速找到在特定的被引用模块（被引用的项目，程序集或已安装的NuGet包 ）中使用符号的方式和位置。在大型解决方案中，这将帮助您检测架构问题并减少耦合。

要获得解决方案中项目关系的大规模视图，请构建项目依赖关系图 。

在搜索结果中，您将看到代码中模块类型或类型成员出现的所有位置。

如果您搜索依赖于引用模块的代码，则ReSharper会在引用该模块的项目中查找该模块符号的用法。如果您搜索依赖于项目的代码，则ReSharper会在整个解决方案中查找项目符号的用法。

要查看当前项目中所有引用程序集的所有用法，请分析项目引用 。

查找依赖于模块的代码

  1. 在解决方案资源管理器的项目的“ 引用”节点中，选择一个项目或程序集引用或NuGet包。

  2. 请执行以下任一操作：

    • 在主菜单中，选择“ ReSharper |查找|查找相关代码 。

    • 右键单击所选项目，然后在上下文菜单中选择“ 查找相关代码” 。

    • 选择“ 查找相关代码”（所有项目）以从整个解决方案的模块中查找符号的用法。

  3. 搜索结果将显示在“ 查找结果”窗口中 ，您可以在其中进一步分析结果 。
