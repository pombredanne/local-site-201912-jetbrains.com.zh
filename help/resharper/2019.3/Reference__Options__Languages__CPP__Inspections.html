在此页面上，您可以调整C ++中代码检查的工作方式。

项目

描述

一般
启用C ++ / CLI支持

默认情况下，ReSharper将以C ++ / CLI代码运行。如果您发现那里存在一些问题，则可以禁用此功能。

显示弹出窗口以进行导入操作

如果您使用标准库或解决方案中其他位置定义的符号，则ReSharper可以帮助您添加缺少的符号#include指令自动：

如果未选中该复选框，则相应的操作将在以下位置的操作列表中可用Alt+Enter 。

在解决方案外部的文件中启用检查，快速修复和上下文操作

默认情况下，所有解决方案文件中都提供代码检查 ， 快速修复和上下文操作 。如有必要，可以使用此复选框在解决方案引用的外部文件中启用这些功能。

颜色标识符

此选项可扩展突出显示C ++语法。 您可以突出显示20种标识符类型。您可以随时在Visual Studio选项（ 工具|选项|环境|字体和颜色 ）中更改颜色和字体。

替换Visual Studio工具提示

此选项使您可以将Visual Studio符号工具提示替换为ReSharper工具提示，当您将鼠标悬停在编辑器中的代码符号（快速信息）上时出现。

默认情况下启用此选项，因为ReSharper工具提示提供了更好的语法突出显示和符号文档：

显示装订线图标

默认情况下，ReSharper添加了动作指示器（装订线图标） ，可快速访问导航和其他动作。
在某些情况下，这些指示符可能太多，您可能要禁用它们：

最常出现的动作指示器是导航到声明/定义 ，因此您可以仅禁用此操作或隐藏所有操作指示器。

模板代码中检查的严重性
将解决错误视为模板代码中的警告

此选项使您可以降低C ++模板中错误的严重性。默认情况下，启用此选项。您可能要启用它有两个原因：

  • Visual Studio不会编译未使用的C ++模板。因此，如果未启用此选项，则可能会在完全可编译的代码中看到错误。

  • Visual Studio编译器的工作方式与C ++标准规定的并不完全相同 。因此，此选项使您不能突出显示Visual Studio编译器可能忽略的错误。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
