路径映射

在开发Web应用程序时，有时会将其部署到目标服务器上。但是，目标环境中文件的路径可能与本地环境中的路径不同，这使得分析路径很困难。为了使生活更轻松，ReSharper突出显示了未知的路径并提出了一些快速修复方法：

单击“ 设置路径映射”以打开“ 路径映射”对话框，您可以在其中创建和更改路径映射规则。单击“ 忽略”或“ 替代”以指定应采用的规则类型。根据选择，键入应忽略的路径或应替代的路径以及替代的路径。

您可以根据需要创建任意数量的规则并随时进行管理。

在Web项目中管理路径映射规则

  1. 在解决方案资源管理器中右键单击您的Web项目，然后在上下文菜单中选择“ 编辑项目项属性 ”。

  2. 在打开的“ 项目项属性”弹出窗口中，单击“ 路径映射” 。

  3. ReSharper将显示“ 路径映射”对话框，您将在其中看到项目中定义的所有现有路径映射规则。使用相应的控件来创建，修改和删除路径映射规则。
