在编译代码中导航和搜索

解决方案不仅限于项目的源代码，还包含对解决方案中使用的库的引用。可以访问并有机会从那些库中导航到符号，就像它们是解决方案的一部分一样，这非常方便。ReSharper提供了一个内置的反编译器和许多功能，可帮助您查看和浏览已编译的代码。

来自已编译程序集的符号的代码表示形式取决于 设置 ，PDB信息的可用性以及所应用的导航命令 。

有两种针对库符号的导航命令：

  • 明确指定在何处以及如何表示库符号的命令：

    • 导航到对象浏览器
    • 导航到装配体浏览器
    • 导航到反编译源
    • 导航到元数据视图
    • 导航到符号文件中的源

  • 当目标符号不在解决方案的源代码中时，这些命令可能隐式地导致编译代码。例如，

    • 前往宣言
    • 转到符号类型
    • 转到基本符号

对于第二种命令，您可以指定ReSharper如何表示库符号。首次执行这种导航时，您会看到一个对话框，提示您选择默认的导航操作：

您可以在以下位置配置默认导航 工具|的“外部来源”页面 ReSharper选项（ Alt+R,,O ）。

您还可以使用Assembly Explorer窗口浏览解决方案之外的任何程序集的源代码。有关更多信息，请参见探索和反汇编程序集 。

如果您想随时清除反编译缓存，请删除％LOCALAPPDATA％\ JetBrains \ Shared \ vAny \ DecompilerCache中的数据 。
