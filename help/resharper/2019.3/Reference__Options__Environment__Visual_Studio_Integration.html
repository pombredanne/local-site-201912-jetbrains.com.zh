Visual Studio功能

ReSharper |选项|环境|编辑器Visual Studio功能

ReSharper选项的此页面允许您配置与Visual Studio功能的集成。

项目

描述

命令和菜单
覆盖Visual Studio重构

从“ 编辑”菜单和上下文菜单中隐藏Visual Studio重构命令。此外，当您使用快捷方式调用Visual Studio重构时，将使用相应的ReSharper 重构 。
请注意，独立于此选项， Visual Studio方案中的某些键盘快捷键会覆盖本机Visual Studio重构快捷键。

隐藏覆盖的Visual Studio菜单项

隐藏被ReSharper功能覆盖的Visual Studio本机菜单项，例如Visual Studio的Organize Usings菜单。

Visual Studio代码分析
禁止Visual Studio弯曲

此选项在2015年之前的Visual Studio版本中不可用。
如果选择该选项，则不会在编辑器中显示Visual Studio的错误突出显示，而仅显示ReSharper的突出显示。

不显示Visual Studio灯泡

此选项在2015年之前的Visual Studio版本中不可用。
如果选择它，则不会单独显示Visual Studio的“快速操作”（灯泡和螺丝刀图标） ，而是集成到ReSharper的操作列表中 。

将Visual Studio灯泡动作合并到ReSharper灯泡中 如果选择了此选项，ReSharper将为每个Visual Studio的快速操作执行以下操作：

  • 如果存在类似的ReSharper的快速修复或上下文操作 ，则不会显示Visual Studio的快速操作。

  • 如果没有类似的ReSharper动作，则Visual Studio的动作在ReSharper 动作列表中可用。您可以通过特殊图标识别Visual Studio的快速操作 。

工具提示
替换Visual Studio工具提示

此选项使您可以将Visual Studio符号工具提示替换为ReSharper工具提示，当您将鼠标悬停在编辑器中的代码符号（快速信息）上时出现。

如果您希望查看本机Visual Studio工具提示或使用Productivity Power Tools扩展，则可能要禁用此选项。

Visual Studio和ReSharper工具提示基本上提供了有关您将光标悬停在其上的符号的类似信息。在不同的Visual Studio版本中，Visual Studio工具提示有所不同。

如果禁用ReSharper 语法突出显示 （通过清除ReSharper选项的“代码检查|设置”页面上的“ 颜色标识符”复选框），还将禁用ReSharper工具提示。

此选项不影响C ++。要替换C ++的Visual Studio工具提示，请使用此页面上的相应选项。

C ++工具提示
替换C ++的Visual Studio工具提示

该选项使您可以替换C ++代码中的Visual Studio符号工具提示，类似于上面的常规工具提示选项。默认情况下启用此选项，因为C ++中的ReSharper工具提示提供了更好的语法突出显示和符号文档：

调试器数据提示
用ReSharper DataTips替换Visual Studio DataTips

ReSharper可以选择用自己的DataTips替换Visual Studio DataTips，它们有很多区别：

  • 您可以在当前的暂停执行上下文中搜索特定值。为此，只需在焦点位于DataTip中时开始键入即可。请注意，搜索将仅在扩展节点中找到值。

  • 不是在删除鼠标指针时，而是在DataTip之外的任何位置单击时，DataTip消失。

  • 如果一个类型都没有意义ToString()覆盖也不用[DebuggerDisplay]属性，那么在调试器视图中这种类型的对象的表示可能完全没有信息。在ReSharper DataTips中，您可以即时自定义此类对象的表示形式。为此，请在“数据提示”中展开类型，右键单击所需的数据成员，然后选择“ 突出显示属性” ：

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
