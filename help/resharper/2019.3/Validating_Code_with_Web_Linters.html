使用Web Linter验证代码

ReSharper与Web代码质量工具（也称为linters）（JSLint，ESLint和TSLint）集成在一起，以检测并帮助修复代码中的更多问题。

Linters被设计为命令行工具，因此通常每次您想要验证代码时，都需要转到控制台，运行命令行，然后浏览其文本输出。
使用ReSharper则有所不同。在ReSharper设置中安装并启用了linter时，linter代码规则将与ReSharper自己的代码检查一起使用，并且在设计时会在编辑器中突出显示linters检测到的所有代码问题。

当您在特定范围内或从命令行分析代码时，产品将不会运行linter分析。

在编辑器中启用Web Linter验证

  1. 安装Node.js解释器。最简单的方法是为Visual Studio安装Node.js工具 。

  2. 在大多数情况下，ReSharper会自动找到Node.js解释器。如果安装在不常见的位置，请在 工具|的Node.js页面 ReSharper选项（ Alt+R,,O ）

  3. 在您的项目中安装所需的linter软件包。

  4. 在“ 工具” | “ 工具”下打开所需的短绒选项页面。 Web Linters并启用该Linter。

  5. 如果您的项目中有node_modules文件夹，则ReSharper将自动找到linter软件包。但是，您也可以指定一个自定义的linter文件夹。

  6. 如果使用ESLint或TSLint，则ReSharper将从linter配置文件中获取所有规则和设置。默认情况下，会自动找到这些文件，但您也可以指定自定义配置文件。

    如果启用了自动搜索，并且lint不起作用，请确保配置文件位于当前目录和/或其父目录或用户主目录中：

    • 对于ESLint： .eslintrc。*文件或eslintConfig package.json中的字段。

    • 对于TSLint： tslint.json ， tslint.yaml或tslint.yml 。

  7. 您还可以传递任何其他用空格分隔的命令行选项（请参阅JSLint ， ESLint和TSLint的命令行选项参考）。
    请注意，某些选项会影响lint的执行方式，因此无法应用。例如， --fix将被忽略，因为ReSharper提供了自己的方法来应用linter修复程序。

  8. 如果使用TSLint，则默认情况下，ReSharper将在TypeScript和JavaScript代码中启用linter。要仅在TypeScript代码中使用它，请清除相应的复选框。

设置linter支持后，您将看到编辑器根据相应规则的严重性级别（错误级别）突出显示了linter检测到的代码问题：

您还将在解决方案范围的错误/警告监视器中看到linters检测到的所有错误/警告。

如果短绒提供了规则的修补程序，则您可以直接从Alt+Enter菜单：

您还可以选择检验：[名称] |配置检查严重性| [严重程度] Alt+Enter菜单-这将在linter配置文件中为相应规则写入新的严重性级别（错误级别）。
