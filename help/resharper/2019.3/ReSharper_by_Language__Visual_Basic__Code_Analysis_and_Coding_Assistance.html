代码完成

代码完成功能通过提供一组根据周围环境要完成的项目来帮助您更快地编写代码。有关更多信息，请参见代码完成（IntelliSense） 。

Visual Basic中提供了三种代码完成功能：

自动和基本完成

基本完成

基本完成建议名称空间，类型，方法，字段，属性等。

枚举成员也包括在基本完成列表中。如果您使用匿名类型，则ReSharper会将其属性考虑在内，并将其与其他可用项一起显示在完成列表中。

智能完成

智能完成功能会根据当前上下文中预期的类型来过滤代码完成列表中的项目。看下面的例子，只有那些具有string类型显示在列表中。

导入符号完成

导入符号完成为在当前上下文中通过导入指令不可见的类型和扩展方法提供了完成项。

上下文动作示例

ReSharper提供了一组针对Visual Basic .NET代码的上下文操作 。您可以在“ 代码编辑”中找到这些动作的完整列表。 Visual Basic .NET | ReSharper选项的上下文操作页面。如有必要，您还可以使用此页面禁用某些操作。

一旦上下文动作可用于当前插入符号位置，ReSharper就会显示相应的动作指示符 在插入符号的左侧。但是，有时ReSharper为当前的插入位置提供了一些上下文相关的功能。在这种情况下，将显示与优先级最高的动作对应的动作指示器，并且仅当您通过单击动作指示器或按来展开动作列表时，所有其他动作才会出现Alt+Enter上下文操作的优先级最低，因此，它们通常出现在操作列表的底部。

您可以轻松地将自动属性转换为具有后备字段的属性。将插入符号放在自动属性的名称上，然后按Alt+Enter 。ReSharper建议采取适当的上下文操作。

对于函数的每个参数，都有一个上下文操作，它将创建一个没有该参数的函数来调用原始函数。

ReSharper自动检测所有派生类，并提示您确定应在哪里实现基类成员并生成代码。

重新排列代码

Visual Basic中也提供了重新排列代码元素 。例如，您可以在类型内上下移动成员声明。

您可以将语句移出代码块或移出代码块。例如，移动If陈述出来For Each循环，按Ctrl+Shift+Alt+Left 。

您可以重新排序方法调用或方法声明的参数。例如，移动FileId右边的参数，按Ctrl+Shift+Alt+Right 。
