命名方式

从2018.3版本开始，ReSharper可以从现有代码中自动学习C＃符号命名规则 。如果要手动配置命名规则，请清除“ 使用自动检测到的规则”复选框 代码编辑| C＃|的命名样式页面 ReSharper选项（ Alt+R,,O ）。

ReSharper |选项|代码编辑| C＃|命名方式

ReSharper选项的此页面定义C＃的符号命名规则 。

ReSharper可帮助您为代码中的符号定义，控制和应用命名样式。命名样式是作为一组规则实现的，每个规则都针对具有一组约束的特定标识符。每个规则可以具有一个或多个关联的样式，这些样式定义后缀，前缀，复合词的大写形式，等等。

ReSharper时会考虑这些规则产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。ReSharper还可以帮助您检测和修复违反命名规则的情况。如有必要，可以配置或禁用自动检查命名规则。

启用“不一致的命名”检查

此复选框切换C＃，VB.NET，JavaScript，TypeScript和XAML中的不一致命名检查。

使用自动检测的值

如果选中此复选框，ReSharper将执行以下操作：

  1. 根据新打开的解决方案中的符号命名来检测命名规则。

  2. 在检测到的样式不同于您当前设置的情况下，更改规则的首选项。

  3. 将修改后的命名规则保存到“个人解决方案”设置层，以便这些设置仅影响当前解决方案。

  4. 用黄色突出显示自动设置的命名规则：

编辑

单击此按钮，或双击列表条目，以使用“ 编辑规则设置”对话框编辑选定的命名规则。有关编辑命名规则的详细信息，请参阅“ 修改默认命名规则” 。

高级设置

单击此按钮可以使用“ 高级命名设置”对话框创建或管理自定义命名规则。有关详细信息，请参见创建自定义命名规则 。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
