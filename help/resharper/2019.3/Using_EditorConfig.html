什么是EditorConfig，ReSharper如何扩展它？

EditorConfig是一种配置文件约定，用于在使用相同代码的团队成员之间以及他们可能使用的不同编辑器和IDE之间定义和维护一致的代码样式。样式保存在名为.editorconfig的类似INI的文件中，其中节名称是文件掩码，节内的属性定义了与该掩码匹配的文件的代码样式。

要在Windows资源管理器中创建.editorconfig文件，请创建一个名为.editorconfig的文件。 ，它将自动重命名为.editorconfig 。

如EditorConfig惯例所建议，ReSharper将应用当前文件目录及其所有父目录中名为.editorconfig的文件中定义的代码样式，直到到达根文件路径或找到带有以下内容的EditorConfig文件为止： root=true 。例如，在.editorconfig文件中指定的文件掩码*Test.cs也考虑在内。

ReSharper理解标准的EditorConfig属性 （最常用的.NET编码约定EditorConfig属性） ，并提供一组自定义EditorConfig属性 ，这些属性允许对格式，语法和代码检查规则进行更细化的配置-实际上，每种代码样式您可以在ReSharper选项对话框中配置的首选项具有自己的EditorConfig属性。这意味着您可以在EditorConfig文件中维护代码样式和检查规则的完整配置 。下面是ReSharper支持的EditorConfig属性的示例：

将代码样式设置导出到EditorConfig

如果要通过EditorConfig共享代码样式设置，则可能需要将ReSharper中已配置的样式导出到.editorconfig文件。

要将代码样式设置导出到EditorConfig

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择代码编辑|左侧的常规格式化程序样式 。

  2. 在“ 代码样式配置文件”部分中，单击“ 将当前样式写入.editorconfig” 。这将打开.editorconfig对话框。

  3. 默认情况下，ReSharper将设置写入当前解决方案的根目录中的.editorconfig文件。如有必要，您可以使用对话框顶部的路径选择器更改目标文件。

  4. 在“ 节（文件掩码）”字段中，指定通配符模式以用于具有已生成属性的节。请注意，如果选择导出标准EditorConfig属性，则会根据语言约定将它们放置在其他部分中。

  5. 要进行更详细的配置，可以展开“ 显示其他选项”，然后选择以下选项：

    • 导出具有默认值的设置
      默认情况下，ReSharper仅保存已更改设置的属性。因此，当ReSharper或Rider读取生成的.editorconfig时，将仅应用这些设置并覆盖其他设置。所有其他设置将采用IDE中指定的值。
      如果启用此选项，则无论是否更改，ReSharper都会将所有设置保存为当前状态。这样，您将获得更严格的.editorconfig ，当被ReSharper或Rider读取时，它将覆盖所有代码样式和格式设置。

    • 删除具有不明确值的现有属性
      一些EditorConfig属性可能对应于多个ReSharper特定的属性，从而可以进行更精确的配置。因此，在某些情况下，更一般的属性与某些特定于ReSharper的相关属性一致，并且与其他属性相矛盾，这会使该更一般的属性变得模棱两可。
      由于更详细的属性具有更高的优先级，因此这种情况不成问题，并且无需删除具有歧义值的属性。
      可能要删除具有不明确值的属性的唯一情况是ReSharper和Rider将仅读取项目EditorConfig样式，并且要从.editorconfig文件中删除冗余。

    • 导出标准.editorconfig属性 -是否保存标准.editorconfig属性如果其他编辑器和IDE读取了生成的EditorConfig，则这些属性也将适用。

    • 导出跨编辑器特定于语言的属性 -是否保存.NET-coding-convention属性 。如果生成的EditorConfig被其他编辑器和IDE读取，则这些属性也将适用。

    • 导出JetBrains Rider / ReSharper特定的代码样式属性 -是否保存产品特定的代码格式样式 ， 代码语法样式 。

    • 导出JetBrains Rider / ReSharper特定的检查严重性 -是否保存严重性级别以用于ReSharper的可配置代码检查。

  6. 如果指定位置中存在现有的.editorconfig文件，则该位置上存在的属性值与ReSharper即将生成的属性值之间可能会发生冲突。这些属性将显示为红色，并且如果继续进行导出，则将解决冲突，而使用新生成的冲突。

  7. 点击导出 。ReSharper将在指定位置创建或更新.editorconfig文件。

使用EditorConfig禁用ReSharper格式化程序

在C＃，C ++，JavaScript，TypeScript，HTML和Protobuf中，您可以使用具有以下功能的EditorConfig掩码来禁用ReSharper格式化程序： disable_formatter=true 。

标准EditorConfig属性支持说明

ReSharper将应用以下标准EditorConfig属性：

  • indent_size
  • indent_style
  • tab_width
  • max_line_length
  • insert_final_newline

请注意，在ReSharper的自定义EditorConfig属性中 ，有些属性将覆盖特定语言的标准属性。例如，您可以通过覆盖C＃和JavaScript中不同的intent大小indent_size具有csharp_indent_size和js_indent_size属性的属性。
