在此选项页面上，您可以调整JavaScript中代码检查的工作方式。

项目

描述

JavaScript语言级别

允许选择用于分析JavaScript代码的ECMAScript版本。

在JS文件中启用JSX语法

如果希望ReSharper处理.js文件中的JSX语法 ，请选择此选项。

检查代码时分析属性的上下文

默认情况下，此复选框处于禁用状态，可进行更深入的属性分析。禁用后，仅将未知属性检测为错误。启用后，也会检测到重新分配的属性。例如：

var a = {prop1：“ property”}; var b = {prop2：“ property”}; a.prop2 //仅在启用该选项的情况下才会检测到此错误

启用该选项后，可能会导致性能显着下降。

如果周围有多余的括号，请忽略条件警告中的分配

如果此赋值用括号括起来，则此首选项使您可以忽略“条件语句中对变量的赋值” 代码检查 。
也就是说，ReSharper将不会在以下JavaScript代码中检测到问题：

var a = 10;如果（（a = 5））{doSomething（）; }
此外，如果作业没有用括号括起来，ReSharper将建议添加它们：

启用语义功能着色

启用ReSharper的语法突出显示后 ，此复选框允许您在语义上为包含函数（绿色）的变量（与其他变量（深紫色）相对）着色。
此选项可能会对性能产生负面影响，因此，如果您遇到任何性能问题，建议将其关闭。

解决与网站根文件夹有关的JavaScript参考注释

在具有专用网站根文件夹的Web项目中，例如wwwroot ，您可以在此文件夹中直接编辑JavaScript代码（在这种情况下，应选中此复选框），或者在其外部，则应清除此复选框。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
