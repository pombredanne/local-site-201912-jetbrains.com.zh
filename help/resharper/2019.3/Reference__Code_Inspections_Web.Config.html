Web.config中的代码检查

在Web.config中，ReSharper 2019.3提供两种代码检查：12种检查，以检测错误的语法，例如语法错误，符号解析失败，编译器错误等（您无法配置或禁用这些检查中的任何一种），以及13 专有代码检查 ，您可以禁用或更改其严重性水平 。
这些代码检查可帮助您在设计时检测所有打开文件中的代码问题，此外，它们还使您可以查找特定范围内的代码问题 。

下面列出了可配置的Web.config检查，并按其类别分组。

代码通知（2次检查）

此类别将代码检查的严重性级别较小。

检查

 ID EditorConfig属性 默认严重性
忽略的路径 Web.IgnoredPath resharper_web_ignored_path_highlighting 残障人士
映射路径 Web.MappedPath resharper_web_mapped_path_highlighting 暗示

潜在的代码质量问题（5次检查）

此类检查包括检测关键问题（代码异味）的检查，主要是错误或警告级别。此类别还包括确保本地化协助的检查 。

检查

 ID EditorConfig属性 默认严重性
无法解析模块 WebConfig.ModuleNotResolved resharper_web_config_module_not_resolved_highlighting 错误
无法解析符号 WebConfig.TypeNotResolved resharper_web_config_type_not_resolved_highlighting 错误
需要模块资格 WebConfig.ModuleQualificationResolve resharper_web_config_module_qualification_resolve_highlighting 警告
在网站上找不到参考路径 WebConfig.WebConfigPathWarning resharper_web_config_web_config_path_warning_highlighting 警告
错误的模块 WebConfig.WrongModule resharper_web_config_wrong_module_highlighting 错误

代码中的冗余（6次检查）

此类别中的代码检查查找冗余和无效代码，这些代码会影响代码的可读性和样式，并且可以安全地删除。 某些代码冗余无法自动修复，因此需要在用户输入的情况下以交互方式进行快速修复。但是，可以使用范围内的修复或代码清除来修复大多数冗余，而无需用户交互。

检查

 ID EditorConfig属性 默认严重性
冗余添加元素 WebConfig.UnusedAddTag resharper_web_config_unused_add_tag_highlighting 警告
冗余添加名称空间元素 WebConfig.RedundantAddNamespaceTag resharper_web_config_redundant_add_namespace_tag_highlighting 警告
由于'configSource'属性而导致的冗余元素或属性 WebConfig.UnusedElementDueToConfigSourceAttribute resharper_web_config_unused_element_due_to_config_source_attribute_highlighting 警告
冗余位置元素 WebConfig.RedundantLocationTag resharper_web_config_redundant_location_tag_highlighting 警告
多余的删除或清除元素 WebConfig.UnusedRemoveOrClearTag resharper_web_config_unused_remove_or_clear_tag_highlighting 警告
冗余标签前缀声明 WebConfig.TagPrefixRedundand resharper_web_config_tag_prefix_redundand_highlighting 警告
