将类型移动到另一个文件重构

ReSharper |重构|移动…
F6
ReSharper_Move

此重构可帮助您将所选类型从当前文件移动到任何现有文件或新文件。如果要移至新文件，则会自动创建新文件。如果当前文件中没有其他类型，则可以将其删除。所有必需的导入都随类型一起移动。所有不再需要的导入都将从原始文件中删除。

如果在嵌套类型上调用此重构，则ReSharper将使父类成为部分类，为该部分类的新部分创建一个新文件，然后将所选类型移到那里。如有必要，然后可以使用“ 将类型移到外部作用域”重构 ，使该类型不再嵌套，并删除父类的冗余部分。

将类型移动到另一个文件

  1. 通过以下方式之一选择类型：

    • 在编辑器中，将插入号设置为类型的名称。

    • 在解决方案资源管理器中选择一种类型。

    • 在“ 文件结构”窗口中选择一种类型。

    • 在“类视图”中选择一种类型。

    • 在对象浏览器中选择一种类型。

    • 在类型依赖关系图中选择一种类型。

  2. 请执行以下任一操作：

    • 按F6然后选择移至另一个文件

    • 按Ctrl+Shift+R然后选择移至另一个文件

    • 右键单击并选择“ 重构”。在上下文菜单中移动到另一个文件 。

    • 选择 ReSharper |重构|在主菜单中移动...。

    将打开“ 移至另一个文件”对话框。

  3. 在新文件名字段中输入文件名 。您可以为现有文件或新文件指定名称。在后一种情况下，新文件将在当前项目或文件夹中创建。

  4. 要应用重构，请单击下一步 。

  5. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。

撤消该操作时，请注意，将从新创建的文件中删除内容，但是不会删除文件本身。
