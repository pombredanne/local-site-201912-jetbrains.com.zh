使用dotCover分析代码覆盖率

ReSharper的单元测试运行器不允许运行两个或多个具有相同ID的测试。在实践中，这种情况很少发生，但是如果您确实有具有相同ID的测试，它们将在“单元测试”浏览器和“单元测试会话”窗口中显示为单个节点，并且仅运行其中一个。
请注意，测试框架以不同的方式识别测试。例如，MSTest使用测试方法FQN，而NUnit使用测试属性的属性作为测试ID。

如果您拥有ReSharper Ultimate许可证，则可以使用JetBrains dotCover分析单元测试的覆盖范围（即单元测试覆盖解决方案的代码的程度）。

启用dotCover后， 封面单元测试 Ctrl+T,H命令与 运行单元测试 Ctrl+T,R / 调试单元测试 Ctrl+T,D命令。您可以使用此命令从编辑器 ， 解决方案资源管理器或单元测试会话运行单元测试的覆盖率分析。

您可以通过取消选中dotCover复选框来禁用dotCover 环境|的产品和功能页面 ReSharper选项（ Alt+R,,O ）。

要了解有关分析覆盖率结果的更多信息，请参阅dotCover帮助：处理覆盖率结果 。
