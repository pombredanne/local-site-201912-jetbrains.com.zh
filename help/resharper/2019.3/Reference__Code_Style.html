ReSharper时会考虑在此页面上可配置的首选项产生具有代码完成和代码生成功能的新代码，应用代码模板并执行重构 。通过使用带有相应设置的代码清除 ，它们也可以应用于现有代码。

带有“ 通知方式”选择器的首选项具有相应的代码检查，这些检查会通知您检查范围内的代码样式的这一方面是否与首选样式不同。使用选择器，您可以配置检查的严重性级别 。

完成时应用代码样式

接受代码完成建议时，请使用此复选框来应用配置的语法样式。

声明中的“ var”用法 本节中的首选项定义隐式类型的局部变量 （也称为var关键字）。 您可以为不同类型设置使用'var'或显式类型的不同首选项：

  • 对于内置类型 -适用于C＃内置类型 。

  • 对于简单类型 -适用于不具有通用参数的类型。

  • 其他 -适用于泛型类型和解构声明。

对于这些首选项中的每一个，您都可以选择使用'var'，显式类型或'var'（当“ explain”时使用） ，这意味着“ var”应仅用于初始化为创建对象，数组和文字或显式类型转换的变量。有关更多信息，请参见代码语法样式：隐式/显式键入（“ var”关键字） 。
对于解构变量，建议使用单独的声明

默认情况下，ReSharper将建议多个的联合表示法var例如，在解构声明中 var (x, y) = GetTuple(); 。您可以选择对解构变量更喜欢单独的声明，以选择单独的符号，例如(var x, var y) = GetTuple(); 。

使用'var'关键字丢弃

默认情况下，ReSharper建议使用独立_例如，用于丢弃 (_, _) = (0, 1); 。您可以选择使用“ var”关键字来丢弃，以始终使用var在适当的地方使用丢弃，从而确保与可能命名的范围中的变量没有冲突_ 。

参考资格和“使用”指令

本节中的首选项定义了名称空间导入的样式 ：

  • 首选完全限定的引用 —如果要对导入的类型使用完全限定的名称而不是名称空间导入指令，请选中此复选框。

  • 将“ using”指令添加到最深的作用域 -如果选中此复选框，则在使用导入类型的名称空间中添加名称空间导入指令；否则，导入指令将添加到文件顶部。

  • 在对“使用”指令进行排序时，请首先放置“ System。*”和“ Windows。*”命名空间。如果选中此复选框，则“使用”指令将像using System.Linq;排序时将被放置在其他using指令之上。否则，这些指令将与其他指令按字母顺序排序。

  • 在嵌套范围内首选使用名称的完全限定名称 -如果选中此复选框，则在嵌套类型和名称空间中首选使用导入类型的完全限定名称。

  • 允许使用别名指令 -选中此复选框将允许在名称空间导入指令中使用别名，而不是完全限定的名称空间名称。

  • 允许使用'global ::'前缀 -如果选中此复选框，则global::前缀未删除，例如global::System.String 。有关更多信息，请参见如何：使用全局命名空间别名 。

可以在“ 代码编辑” |“配置”中配置与名称空间导入有关的许多其他选项。 C＃| ReSharper选项的“ 命名空间导入”页面。
