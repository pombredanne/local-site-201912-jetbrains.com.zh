代码检查：合同注释定义中的问题

您可以禁止执行此检查以忽略特定问题 ， 更改其严重性级别以使问题更小或更明显 ，或者将其完全禁用 。

此检查将验证合同注释的语法。

合同注释的语法

使用以下语法为合同注释指定输入输出依存关系：

[ContractAnnotation（“ [paramName：] [input] => output [; [paramName：] [input] => output]”，[forceFullStates：true]）]

的input可：

  • null / notnull用于参考类型参数

  • true / false用于布尔参数。

的output可

  • null / notnull / canbenull用于引用类型的返回值

  • true / false用于布尔类型的返回值

  • halt | stop | void | nothing （可互换）以指示该函数未正常返回。也就是说，它会引发异常或中止程序执行。

可选的布尔值forceFullStates参数（默认情况下为false）允许您强制悲观模式进行可空性分析。也就是说，如果合同条件未定义方法的返回值，则ReSharper将假定它可能为null。

备注：

  • 你可以省略paramName如果只有一个参数（请参见上面的示例）

  • 两者都可以省略paramName和input如果没有参数：

    [ContractAnnotation（“ =>暂停”）] public void TerminationMethod（）
    或者如果函数在输入上独立地具有相同的输出：
    [ContractAnnotation（“ =>暂停”）]公共无效TerminationMethod（对象数据，布尔标志）

  • 您可以为同一参数添加多个条件：

    [ContractAnnotation（“ input：null => null； input：notnull => notnull”）]公共对象Transform（对象输入，布尔标志）

  • 您可以逆转条件，即input => output等于output <= input ：

    [ContractAnnotation（“ null <= param：null”）]公共字符串GetName（字符串姓氏）

  • 您也可以为“ out”参数指定期望值。如果要为相同的输入条件同时指定返回值和'out'参数，请使用逗号：

    [ContractAnnotation（“ s：null => false，result：null”）]公共布尔TryParse（字符串s，输出对象结果）
