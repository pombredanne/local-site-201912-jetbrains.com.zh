在设计时检测代码问题

一旦在编辑器中打开代码文件，ReSharper便开始分析代码文件，直到您将其完全编辑为止。根据其严重性级别，所有检测到的代码问题均在编辑器中突出显示。问题图也显示在编辑器窗口右侧的标记栏上，您可以在其中立即查看文件的状态，然后单击标记以导航到特定的代码问题 。

产品还允许您使用JSLint，ESLint和TSLint在编辑器中直接验证和修复JavaScript / TypeScript代码 。

在这个相当简单的示例中，我们可以看到ReSharper代码检查的以下功能：

  1. 状态指示器 ，可帮助您立即查看当前文件是否有错误或警告。

  2. 针对非导入类型出现的修复弹出窗口。按下就足够了Alt+Enter或单击此弹出窗口，ReSharper将为文件中的所有类型添加缺少的指令。有关更多信息，请参见导入缺少的命名空间 。

  4. 中等优先级的代码问题（在这种情况下，是与符号名称不符的符号名称警告）以蓝色卷曲下划线突出显示。

  5. 与建议问题（3）对应的标记显示在标记栏上 。

  7. 如果ReSharper在此位置有任何建议，则该操作指示器将显示在插入符号位置的左侧。

  8. 高优先级的代码问题（在这种情况下，与未解决的符号和不正确的返回类型有关的错误）用红色文本和红色卷曲下划线突出显示。

  9. 与警告问题（4）相对应的标记显示在标记栏上。

  10. 操作列表 ，通过按可以打开Alt+Enter或单击操作指示器（7），包含在插入符号处针对该问题的快速修复列表。

  11. 状态栏中会出现插入符号问题的简短描述。您也可以通过将鼠标悬停在突出显示的代码或标记栏上的问题标记上来查看代码问题的描述（5,6,9）

  12. 如果启用了解决方案范围的分析 ，则ReSharper允许您查看更多代码问题。在此示例中，它检测到未使用的公共成员（3），并通知您解决方案其他文件中的错误。您可以单击解决方案范围的分析图标来浏览检测到的问题。

切换设计时检查

默认情况下，以所有受支持的语言启用设计时代码检查。但是，您可以在任何地方禁用它，也可以根据需要针对特定文件将其关闭。有关配置代码检查的更多信息，请参阅“ 配置代码检查设置” 。

如果要从代码检查中排除当前文件，请按Ctrl+Shift+Alt+8然后ReSharper将此文件添加到排除文件列表中。

如果您打开了未通过代码检查的文件，则可以通过相应的图标来识别它 状态指示灯的 。要快速切换当前文件的代码分析，请按Ctrl+Shift+Alt+8 。

    在字符串文字中突出显示特殊字符

浏览代码问题

如果启用了设计时代码检查，则可以轻松地在当前文件中检测到的所有问题（具有提示严重性级别的问题除外）之间导航。

导航到当前文件中的下一个/上一个代码问题

  • 按F12转到下一个代码问题，或者Shift+F12转到上一个代码问题。

  • 在主菜单中，选择 ReSharper |检查下一期文件或 ReSharper |检查文件中的上一期 ..

  • 使用编辑器窗口右侧的标记栏 ：单击标记会将插入号转到相应的问题；单击标记栏顶部的状态指示器会将插入号转到文件中的下一个问题。

  • 右键单击状态指示器，然后选择转到下一个/上一个错误/警告/建议 -这些命令将为您提供帮助在最高严重性级别的代码问题之间导航。因此，如果存在错误，它们只会在错误之间导航，一旦所有错误都被修复，它们就会在警告之间导航，依此类推。

在代码问题之间导航的另一种方法是在所需范围内运行代码检查 ，然后在专用工具窗口中研究问题。

如果您仅对当前文件中的错误感兴趣，则可以在它们之间跳转，从而跳过严重性级别较低的问题。此外，如果启用“ 解决方案范围分析” ，则可以在整个解决方案中的错误之间跳转。请注意ReSharper |中的导航命令名称。如果启用了解决方案范围的分析，请检查菜单更改。具体而言，“ 下一个错误/警告”将更改为“解决方案中的下一个错误/警告” ，“ 上一个错误/警告”将变为“ 上一个错误/警告”在解决方案中 。

导航到下一个/上一个错误

  • 按Alt+F12转到下一个错误，或者Shift+Alt+F12转到上一个错误。

  • 在主菜单中，选择 ReSharper |检查下一个错误/警告或 ReSharper |检查先前的错误/警告 。

  • 如果启用了“ 解决方案范围内的分析”并且有一些错误，则状态栏的右上角将显示错误数量。您可以单击此数字以转到解决方案中的下一个错误。

如果您对某些代码问题不感兴趣，请按Alt+Enter在您处理问题时，选择检查[检查名称] 。然后，您可以选择取消针对此问题的相应代码检查，或完全禁用此检查 。

检查选项菜单

除了建议的修复程序之外，对于每个可配置的代码检查以及自定义检查 ，ReSharper均显示检查[检查名称] 操作列表中的子菜单，包含以下项目：

  • 禁用一次评论
  • 禁用并恢复评论
  • 配置检查严重性
  • 查找所有此类问题

  • 对于最具争议性的问题， ReSharper为何建议这样做？ 该项目将打开代码检查索引页面，其中包含相应检查的详细说明。

查找类似的问题

您不仅可以使用快速修复来解决突出显示的问题，还可以在整个解决方案或更小范围内找到并调查所有类似的问题（通过相同的代码检查检测到的所有问题）。

除了寻找类似的问题之外，即在所需范围内应用单个代码检查 ，也可以在此范围内应用ReSharper的整个检查集 。

寻找类似问题

  1. 将尖号设置为编辑器中突出显示的问题。

  2. 按Alt+Enter或单击插入符号左侧的动作指示器以打开动作列表 。

  3. 要在当前文件中找到类似的问题，请选择检验[检验名称] | |。在[filename] file中找到类似的问题 。

  4. 要在当前项目或解决方案中找到类似的问题，请展开子菜单并选择所需的范围：

  5. 如有必要，您可以在解决方案中的任何项目或解决方案文件夹中搜索类似的问题。为此，请在子菜单中选择“在自定义范围中查找相似的问题 ”。
    在打开的对话框中，输入所需项目或解决方案文件夹的名称，然后单击“ 确定” 。

  6. 找到的所有问题将显示在“ 检查结果”窗口中 。

与Visual Studio代码检查控件集成

从2015版开始，Visual Studio附带了自己的代码分析引擎（Roslyn），并提供了自己的灯泡功能以执行快速动作，包括重构和修复错误。

Visual Studio的快速操作通常执行与ReSharper的快速修复 ， 上下文操作或重构相同的修复程序。因此，对于某些错误，Visual Studio和ReSharper可能有两个类似的建议。在下图中，您可以看到两个灯泡都建议删除多余的灯泡using指令：

为避免这种情况，ReSharper在“ 环境” | “ 环境”中提供了“不显示Visual Studio灯泡”选项。 编辑器 ReSharper选项的“ Visual Studio功能”页面。此选项默认情况下处于启用状态，因此除非禁用它，否则不会看到重复的灯泡。

如果抑制了Visual Studio灯泡，则可以通过选中将Visual Studio灯泡动作合并到ReSharper灯泡复选框来将Visual Studio动作集成到ReSharper的动作列表中 。 如果选择了此选项，ReSharper将为每个Visual Studio的快速操作执行以下操作：

  • 如果存在类似的ReSharper的快速修复或上下文操作 ，则不会显示Visual Studio的快速操作。

  • 如果没有类似的ReSharper动作，则Visual Studio的动作在ReSharper 动作列表中可用。您可以通过特殊图标识别Visual Studio的快速操作 。

如有必要，还可以在编辑器中禁用Visual Studio的错误突出显示，以便仅突出显示ReSharper发现的代码问题。为此，请在“ 环境” | “ 环境”中选择“ 抑制Visual Studio的花体 ” 。编辑器 ReSharper选项的“ Visual Studio功能”页面。
