创建自定义代码检查和快速修复

如果您在代码中发现问题或不良做法，但是ReSharper未检测到它，则可以创建自定义代码检查。此外，您可以指定应替换有问题的代码的代码，即自定义的快速修复 。

如果正确创建了自定义检查，它将能够在您的代码库中找到所有类似的问题，并在编辑器中突出显示有问题的代码。

要创建自定义检查，您可以利用“ 结构搜索和替换”机制。

为了说明创建自定义检查，让我们假设我们不喜欢以下代码：

弦线; var tr = new MyReader（）;尝试{行= tr。读（）; }最后{tr。Dispose（）; }
..并希望将其替换为：
弦线; using（var tr = new MyReader（））{line = tr。读（）; }

使用快速修复创建自定义代码检查

  1. 在编辑器中，选择应视为问题的一段代码。

  2. 右键单击所选内容，然后选择“使用模式搜索” 。

  3. ReSharper将显示“ 使用模式搜索”对话框，其中已将选定的代码预先准备为搜索模式。某些标识符，参数，类型等将自动替换为占位符：

  4. 要使检查模式更通用，请编辑可识别的占位符 ：，并用更多的占位符替换其他代码项。例如，我们可以在try在我们的示例中，使用了一个与任意数量的语句匹配的占位符。

  5. 要指定问题的快速修复，请单击对话框右上角的“ 替换 ”。
    在“ 替换模式”区域中，指定用于替换有问题的代码的模式。对变量代码项使用相同的占位符。
    这是我们的示例的精确模式和快速修复的外观：

  6. 单击“ 保存” ，然后单击“完成”。使用智能保存逻辑将自定义检查保存在模式目录中。
    请注意，有问题的代码段尚未在编辑器中突出显示。为了使其工作，我们需要做更多的事情。

  7. 在主菜单中，选择ReSharper |选项，然后转到代码检查|自定义模式页面。

  8. 在模式列表中，找到与您刚创建的自定义检查相对应的模式。默认情况下，其严重性级别设置为“不显示”，这意味着ReSharper将忽略与您的自定义检查相对应的问题。
    要启用检查，请选择其他任何严重级别，例如“建议”。

  9. 最后，我们可以指定将在自定义检查中显示的描述以及相应的快速修复。

    为此，请双击该图案，然后在打开的“ 编辑突出显示图案”对话框中，在相应的字段中指定描述：

    您可以为模式指定“抑制键”，该标识符可用于抑制ReSharper检查 ，该检查检测带有模式的注释。

  10. 保存对样式的更改，将其关闭，然后在ReSharper选项的“ 自定义样式”页面上再次单击“ 保存” 。

现在，问题已被识别并在编辑器中突出显示。将鼠标悬停在其上时，将显示指定的描述：

用于自定义检查的快速修复程序的外观和工作方式与内置的快速修复程序完全相同。

如果您需要在更大范围内查找自定义模式所描述的问题，则可以选择以下几种方法：

  • 在“ 代码检查” |“检查 ”中选择检查模式。 ReSharper选项的“ 自定义模式”页面，然后单击页面工具栏上的“ 立即搜索” 。

  • 在所需的范围内运行代码检查，然后与其余检测到的代码问题一起分析自定义问题。

  • 按下时，使用检查选项中显示的“ 查找类似问题”操作Alt+Enter在编辑器中突出显示的代码上。
