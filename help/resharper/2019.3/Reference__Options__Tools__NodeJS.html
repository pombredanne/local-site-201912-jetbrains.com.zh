ReSharper 2019.3 Help

Node.JS

ReSharper | Options | Tools | Node.js

On this page of ReSharper options you can specify location of the Node.js interpreter, which is required to validate and fix JavaSript/TypeScript code with Web linters (JSLint, ESLint, and TSLint).
You can configure specific linters on the corresponding options pages under Tools | Web Linters.

Item

Description

Path to Node.js interpreter

If Node.js is installed on your machine and used in your project, ReSharper will find it automatically in most cases. If it is not found, you can specify its location manually. In any case, ReSharper will show whether the interpreter is found or not.

ReSharper's settings are saved in settings layers, which enable you to have different settings for different solutions and to share your settings with your team. To learn more, see Manage and Share ReSharper Settings.
