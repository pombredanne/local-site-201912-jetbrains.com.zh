进程浏览器窗口

ReSharper | Windows |流程浏览器
ReSharper_ShowProcessExplorer

“进程资源管理器”窗口为您提供当前正在运行的进程的列表，并允许浏览其模块并反编译其中的.NET程序集。

与Windows Task Manager（或Sysinternals Process Explorer）不同，Process Explorer窗口不实时跟踪进程，而是显示在首次打开窗口或调用Refresh操作时所捕获的进程和模块的快照。即使过程已经终止，它也允许浏览过程信息。

将装配体添加到装配体浏览器

将在“流程资源管理器”树中选择的程序集添加到“程序集资源管理器”窗口 。
如果选择一个流程，则属于该流程的所有程序集都将添加到“程序集资源管理器”中。仅添加从磁盘文件加载的.NET程序集，而忽略动态程序集和本机模块。

显示流程层次

控制进程树是否反映进程之间的父子关系。
如果关闭此模式，则所有进程均显示在平面列表中。
如果启用此模式，则子进程将显示在“ 子进程”节点下其父进程内部。

按CLR分组

控制进程树是否反映CLR层次结构。
如果关闭此模式，则托管模块和本机模块将在其父流程节点下的平面列表中显示。
如果启用此模式，则将每个进程的托管程序集按其CLR版本和应用程序域分组，并且本机模块（如果启用了“ 显示本机模块”模式）将显示在单独的“本机模块”节点下。此模式在Windows Vista或更高版本上可用，并且需要管理权限才能充分发挥作用。

显示本地流程

控制“进程资源管理器”是否显示本机Windows进程。
默认情况下，Process Explorer仅显示托管进程。如果启用此模式，则将同时显示托管进程和本机进程。本机进程显示为灰色，无法将其添加到Assembly Explorer。

显示本地模块

控制Process Explorer是否显示本机模块。
默认情况下，Process Explorer仅显示托管程序集。如果启用此模式，则树中将同时显示托管程序集和本机模块。本机模块显示为灰色，无法将其添加到Assembly Explorer。

通过窗口查找过程

帮助通过其窗口查找进程。
单击此按钮后，您可以将鼠标指针悬停在桌面的窗口上并标识相关的进程。在这种模式下，工具提示会显示在每个窗口上，带有PID和CLR版本，并且该过程在Process Explorer树中突出显示。单击任何鼠标按钮或按任意键后，此模式将关闭。

刷新

更新显示的正在运行的进程的快照。
