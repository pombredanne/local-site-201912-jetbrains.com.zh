代码检查：加入局部变量声明和赋值

您可以禁止执行此检查以忽略特定问题 ， 更改其严重性级别以使问题更小或更明显 ，或者将其完全禁用 。

如果声明局部变量并在以后不加任何条件的情况下对其进行初始化，则ReSharper建议在首次初始化变量的地方加入声明和赋值。这消除了不必要的行并提高了代码的可读性。

这是此检查建议的快速修复示例：

次优代码

快速修复后

int Bar（）{int myInt; //做点myInt = 3; //做其他事情返回myInt-1; }
int Bar（）{//做点什么var myInt = 3; //做其他事情返回myInt-1; }

逆向转换可与上下文动作一起使用 Alt+Enter 。

当ReSharper加入声明和变量赋值时，它将使用var或根据您的喜好明确的类型。
