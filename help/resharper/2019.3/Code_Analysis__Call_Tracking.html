呼叫跟踪使您可以查看和浏览代码中的呼叫链。由于对事件，接口和闭包的支持，ReSharper呼叫跟踪是对Visual Studio本机呼叫层次结构的重大改进。

在层次结构视图中，ReSharper使用图标来区分不同种类的符号。您可以在“ 符号图标方案”参考页上了解图标的含义。如有必要，您可以在“ 环境” | “ 环境” | “使用源代码符号图标”主题单选按钮之间切换图标集。 ReSharper选项的常规页面。

在“ 检查结果”窗口中双击一个节点，以导航到编辑器中的相应调用。

调查去电

  1. 将插入号放在方法，事件，属性或构造函数的名称处。

  2. 在主菜单中，选择“ ReSharper |检查拨出电话或按Ctrl+Shift+Alt+A然后在“ 检查此”列表中选择“ 去电” 。

  3. 在打开的“ 检查结果”窗口中，可以展开成员节点以在其上运行代码分析并显示当前成员调用的所有成员。您还可以展开每个子节点。

调查来电

  1. 将插入号放在方法，事件，属性或构造函数的名称处。

  2. 在主菜单中，选择“ ReSharper |检查来电或按Ctrl+Shift+Alt+A然后在“ 检查此”列表中选择“ 来电” 。

  3. 在打开的“ 检查结果”窗口中，您可以展开成员节点以运行代码分析并显示调用当前成员的所有成员。您还可以展开每个子节点。

当您在呼叫跟踪结果中收到大量呼叫时，可以隐藏特定的呼叫或结果树的整个分支，以更清楚地了解要查找的内容。
请注意，当您在“检查结果”窗口中的选项卡之间切换时，隐藏的条目将被重置。

隐藏结果树中的条目

  1. 右键单击要隐藏的条目，然后在上下文菜单中选择“ 忽略 ”。

  2. 条目变为灰色，但仍可见且可搜索。同时显示忽略的用法 按钮出现在工具栏上。

  3. 要隐藏或显示忽略的条目，请切换“ 显示忽略的用法” 按钮。

  4. 要重置忽略的条目，请打开“ 显示忽略的用法” 按钮，右键单击灰色项目，然后在上下文菜单中选择“ 撤消忽略 ”。
