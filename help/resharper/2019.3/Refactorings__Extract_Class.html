提取类重构

ReSharper |重构|提取物|提取类…
ReSharper_ExtractClass

通过此重构，您可以将现有类的成员提取到新类中。例如，如果您需要将负责多个任务的单个类替换为每个具有单个职责的多个类，则此重构会很有用。

ReSharper将自动创建一个参考字段，该字段将提供对新创建的类的访问。如有必要，您还可以将提取的成员保留在原始类中作为副本，或者将实现委派给新类中的相同成员。

内联类重构重构提供反向功能。

提取课程

  1. 通过以下方式之一选择班级：

    • 在编辑器中，将插入号设置为类的名称。

    • 在解决方案资源管理器中选择一个类。

    • 在“ 文件结构”窗口中选择一个类。

    • 在“班级视图”中选择一个班级。

    • 在对象浏览器中选择一个类。

    • 在类型依赖关系图中选择一个类。

  2. 请执行以下任一操作：

    • 按Ctrl+Shift+R然后选择提取类

    • 右键单击并选择“ 重构”。在上下文菜单中提取类 。

    • 选择 ReSharper |重构|提取物|在主菜单中提取类… 。

    提取类对话框将打开。

  3. 在新文件中或与原始类型相同的文件中，指定类的名称及其放置位置。

  4. 选择要提取到班级中的成员。ReSharper建议提取所有使用选定成员的成员。您可以通过绿色的“ 提取”标签识别建议的成员。您不必接受此建议，但如果愿意，可以单击“ 提取所有建议的成员”以选择所有成员 。

  5. 选择成员后，它将在Source类成员列中获得选择器。如果希望该成员在当前类中直接可用，请单击此选择器，然后为方法和属性选择“ 创建委派包装 ”或为属性和字段选择“ 创建副本 ”。

  6. 如果您提取的是非公开成员，而您未提取的其他成员正在使用该非公开成员，则ReSharper警告您，如果提取的成员保留为非公开成员，则这些用法将不起作用。您可以通过单击提取所有建议的成员来选择从属成员，或者单击警告图标并选择建议的修复程序之一：

  7. 要应用重构，请单击下一步 。

  8. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。
