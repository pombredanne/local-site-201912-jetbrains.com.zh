管理和共享ReSharper设置

ReSharper使用分层设置机制来保存您的首选项，它具有以下优点：

  • 您可以在不同的解决方案中进行不同的设置。

  • 您可以在不同的位置保留首选项的不同子集，并在处理特定解决方案时将它们组合在一起。例如，您可以将符号图标的个人首选项和代码格式的团队首选项保留在不同的设置文件中，并自动组合这些首选项。

  • 您可以通过在VCS下保留特定的设置子集来同步团队中的首选项。 ReSharper提供了将团队首选项保留在单独文件中的默认选项，该文件保存在解决方案文件夹中，可以轻松添加到您的VCS中。如果其他团队成员使用ReSharper，则只要他们从VCS中提取更改，就会自动应用此文件中的设置-无需重新启动Visual Studio或重新加载解决方案。

  • 由于任何设置子集都可以保存在文件中并动态更新，因此您可以通过许多其他方式共享ReSharper首选项。例如，您可以将个人首选项保留在Dropbox中，并在其他计算机上使用它们。

  • 来自不同设置文件的首选项用作图层，因此，如果同一设置在不同设置文件中具有不同的值，则使用“上层”层的值。这使您可以覆盖在特定设置文件中定义的设置，而不用更改此文件中的设置。

    假设您更喜欢使用BSD样式括号，并在个人设置中保留此格式设置首选项。但是，然后您开始研究一种特定的解决方案，其中约定使用GNU样式括号-ReSharper允许您更改相应的格式设置，并将其保存在解决方案特定的设置层中，以便它覆盖您的个人设置并在您正在研究此解决方案。当您打开另一个解决方案时，您的个人喜好将再次应用。

默认设置和设置层

ReSharper开箱即用，带有默认的一组首选项，这些首选项基于.NET世界中的约定和最佳实践。这些默认设置在产品中进行了硬编码，如有必要，您始终可以重置为默认设置。如果更改任何设置，则更改将保存在设置层中，并且ReSharper会将其应用，覆盖相应的默认设置。最初，ReSharper建议使用三层，您可以在其中保存您的首选项： 此计算机 ， 解决方案团队共享的 解决方案和解决方案个人的 。

还可以为每个项目分别配置一些首选项。

“此计算机”层

此设置层指定用于您的个人全局设置，并适用于本地计算机上的所有解决方案。
相应的设置文件另存为： ％AppData％\ JetBrains \ Shared \ vAny \ GlobalSettingsStorage。点设置

“解决方案团队共享”层

此层用于执行您的团队对当前解决方案的首选项的常用设置，例如命名样式 ， 格式规则等。此层中的设置将覆盖此计算机层中的设置。
相应的设置文件.sln。DotSettings保存在解决方案文件夹中。
一旦将此文件添加到您的VCS中并且团队成员获得了该文件，他们将自动应用此文件中的设置，而无需重新加载解决方案。

“个人解决方案”层

该层允许您覆盖团队共享设置而不更改它们。如果您只需要在此解决方案中应用某些设置并且独立于团队，则可能会发现这很有用。
相应的设置文件.sln。DotSettings.user将保存在解决方案文件夹中。您不应将其添加到VCS。

保存并覆盖设置

每次在“ 选项”对话框中更改某些设置时，对话框底部都会显示一条消息，其中包含未保存的设置数。

要应用更改，您应该使用以下按钮之一保存它们：

保存（智能保存）

这将应用“智能保存”逻辑。
使用它来保存您的修改，并保持团队共享和/或自定义图层（如果有）完整。换句话说，如果您只想保存更改而不必担心设置图层，请使用“ 保存” 。
保存会将您的更改保存到“此计算机”设置层，然后-如果在“解决方案团队共享”和/或自定义层中还有其他修改后的设置值，则更改也将保存到“解决方案个人”覆盖其他图层中的值。

考虑下图中的示例。假设您更改了三个设置，并使用“绿色”而不是“蓝色”值；这些更改已保存在“此计算机”层中。然后，您的团队决定在团队共享解决方案中设置A和C为“黄色”，因此，这些值由团队负责人保存在“解决方案团队共享”层中，ReSharper会在您撤消时立即应用它们从您的VCS更改。最终，您意识到对设置A的“绿色”值感到更加自在，然后再次更改并保存了它，这一次它也保存在“解决方案个人”层中，使您可以应用所需的值无需更改团队共享层：

保存到（保存到特定图层）

您将需要使用这些命令将修改后的设置保存到“解决方案团队共享”层或自定义层。如果出于某些目的需要修改，还可以将其保存到其他任何层。

不幸的是，在当前的实现中，无法看到定义了在“ 选项”对话框中看到的设置的层值。

将设置保存到特定图层的唯一陷阱是，如果在同一层设置中用其他值定义了相同的设置，则该值在层堆叠中的其他层上的任何地方都不会应用。下图显示了我们如何尝试在“此计算机”层中将设置C更改为“红色”，以及为何无法更改结果设置。

您还可以保存每个项目的任何ReSharper首选项，有关更多信息，请参见JetBrains .NET工具博客：每个项目设置或如何为我的测试项目使用不同的命名样式

管理设置层

默认情况下，您可以使用“共享解决方案团队”层按解决方案共享ReSharper设置-唯一要做的就是将* .dotSettings文件置于版本控制下。
如果要以其他方式共享设置，则需要学习如何创建新的设置图层，以及如何导入和导出它们。为此，ReSharper提供了“设置图层”对话框 ，该对话框可在Visual Studio菜单（ ReSharper |“管理选项” ）和“ 选项”对话框（单击“ 管理”按钮）中找到。

自定义设置层将添加到现有默认层下。例如，如果要将自定义层的设置应用于所有解决方案，则将其添加到“此计算机”层下；否则，将其添加到解决方案特定的层之一下，具体取决于您是否希望它覆盖团队共享设置。

一旦添加了自定义设置图层，它就会出现在“ 选项”对话框的“ 保存到”选择器中，您可以使用该设置将修改后的设置保存到自定义图层。您还可以通过编辑此层来修改此层中的设置 。

添加自定义设置层

  1. 在“ 设置图层”对话框中，选择要在其下添加图层的默认图层之一。

  2. 请执行以下任一操作：

    • 点击添加图层 ，然后单击创建设置文件以创建一个新的空设置层，或单击打开设置文件以从您的文件系统或任何URI中包括一个现有的设置层。

    • 右键单击并在“ 添加层”选择器中选择相应的命令之一。

  3. 根据所选命令，为新的设置层文件指定名称和位置，或者使用来选择现有的设置层文件。DotSettings扩展名。

  4. 根据添加的图层中的设置，将根据其在图层堆栈中的位置来应用，这可以在“ 设置图层”对话框中看到，即，如果在上层中定义了某些设置，则在生成的结果中将使用上层的值设置。如有必要，可以使用在选择图层时可用的“上移”和“ 下移”按钮重新排列组中的自定义设置层。

除了使用“ 选项”对话框中的“ 保存到”按钮将修改后的设置保存到特定图层之外，还可以打开特定的设置图层进行编辑。

    • 右键单击并在上下文菜单中选择“ 编辑 ”。

    • 双击所需的图层。

  4. 根据需要在图层中编辑设置，然后单击“ 保存”以保存更改。

您可以将特定的设置子集从任何设置层导出到新的设置文件。然后，可以在ReSharper的另一个实例上使用此文件来应用这些设置。

从特定的设置层导出设置

  1. 在“ 设置层”对话框中，选择一个应从中导出设置的设置层。

  2. 请执行以下任一操作：

    • 在工具栏上，单击“ 导入/导出自” ，然后点击导出到文件 。

    • 右键单击该图层，然后在上下文菜单中选择“ 导出到文件 ”。

  3. 在打开的“ 导出到文件”对话框中，指定要导出的设置组，然后单击“ 确定” ，然后指定设置层文件的名称和位置。

作为添加自定义设置层的替代方法，您可以将设置的子集从该自定义层复制到现有默认层或自定义层之一。

从设置图层文件导入设置

  1. 在“ 设置层”对话框中，选择要在其中导入设置的设置层。

  2. 请执行以下任一操作：

    • 在工具栏上，单击“ 导入/导出设置” 。根据要导入的文件的位置，单击从文件 导入或从URL导入 。

    • 右键单击该层。从上下文菜单中单击导入 ，然后单击从文件导入或导入从URL。

  3. 指定要导入文件的位置。请注意，如果您选择从URL导入，则文件的路径应以UNC格式指定。

  4. 在打开的“ 从文件导入”对话框中，指定要导入的设置组，然后单击“ 确定” 。

ReSharper还允许您将设置的子集从一个已安装的设置层复制到另一层。

将设置从一层复制到另一层

  1. 在“ 设置层”对话框中，选择要从中复制设置的设置层。

  2. 请执行以下任一操作：

    • 在工具栏上，单击“ 将设置复制到”。 ，然后选择目标设置层。

    • 右键单击该图层，在上下文菜单中选择“ 将设置复制到 ”，然后选择目标设置图层。

  3. 在打开的“ 复制到”对话框中，指定要复制的设置组，然后单击“ 确定” 。

如有必要，您可以清除保存在特定图层中的所有设置。为此，右键单击该图层，然后在上下文菜单中单击“ 重置 ”。

此外，您可以重置所有预定义层中的所有设置，并卸载所有自定义设置层，从而将所有设置恢复为其默认值。为此，请单击“ 重置所有设置” 在工具栏上。

您也可以通过清除旁边的复选框来临时禁用自定义设置层，或者右键单击该自定义设置层并在上下文菜单中选择“ 删除” ，以卸载特定的自定义设置层。

请注意，当您重置所有设置或在对话框中删除设置图层时，该图层文件不会从磁盘上删除，因此您以后可以根据需要挂载该文件。

将设置升级到较新版本

有时，较新版本的ReSharper可能会修改设置的存储格式。在这种情况下，受影响的设置文件会在新版本的第一次启动时自动进行升级-ReSharper只是将具有修改格式的设置添加到设置文件中。它还添加了一个条目，该条目指出已升级内容以防止进一步升级。例如：

真正

但是，即使升级了设置文件，它也可以与较旧的ReSharper版本一起使用（例如，如果您与未升级的队友共享此文件），因为既不更改也不删除不赞成使用的设置。除其他外，这意味着您可以安全地将自动升级的.dotSettings文件中的更改提交到VCS。

此处需要特别注意的是，更新后返回旧版本，然后使用该旧版本更改某些设置，然后返回到新版本-ReSharper将不会第二次升级设置。因此，您更改的某些设置可能以较旧的格式保存，而较新的版本则不会读取。

因此，如果您（或您的团队）同时使用两个不同版本的ReSharper，则可能要检查在一个版本中修改的首选项是否在另一版本中正确读取，如果不正确，则只需在另一个版本中再次进行相同的修改即可。版。
