Code Inspection: Parameter type can be IEnumerable<T> (non-private accessibility)

You can suppress this inspection to ignore specific issues, change its severity level to make the issues less or more noticeable, or disable it altogether.

This is a solution-wide code inspection. It only works when the solution-wide analysis is enabled.

If you have a method whose parameter is an array, a List, or some other type that implements IEnumerable<T>, ReSharper offers an option to change the parameter type to IEnumerable<T> â€” provided that the method itself only iterates the collection and does not access any non-IEnumerable members.

The benefit from this change is that the method becomes agnostic with respect to the type of the collection you pass to it.
