搜索和导航

ReSharper |选项|环境|搜索和导航

此页面的ReSharper选项可让您调整ReSharper与导航和搜索有关的功能。

您也可以在“ 工具” |“ 编译代码”中配置与“ 导航和搜索”相关的功能。 ReSharper选项的“ 外部源”页面。

项目

描述

一般
启用“到处搜索”

如果选中此复选框，则“在任何地方搜索/转到类型”命令将搜索类型，符号和文件；否则，将不显示任何内容。如果清除，则仅搜索类型。

整合Go to Text到处搜索

默认情况下，文本导航已集成到“随处搜索”中 Ctrl+N — ReSharper将在所有匹配类型，符号和文件之后，在结果列表的末尾显示与查询字符串匹配的文本出现。您可以通过清除此复选框来禁用此行为。
如果禁用此选项，则可以通过按导航到文本 Ctrl+N三次。

记住上一次搜索

ReSharper可以记住您在“随处搜索/转到类型” ， “符号” ， “文件 ”和“文本”操作中用于查找内容的最后输入。选择此选项后，ReSharper还将在编辑器中使用您当前的选择作为初始搜索查询。例如，您可以在字符串中选择文件名，然后按Ctrl+Shift+N在您的解决方案中搜索与此名称匹配的文件。

记住扩展的搜索首选项

选择此选项可记住搜索命令选项，例如在“ 转到符号”中 包括库类型 ，在“ 转到文件 成员”中包括基本成员以及在“ 转到文本”中 区分大小写 。

转到用法：如果只有一个结果，请浏览而不显示列表

如果选中此复选框，并且通过“ 转到使用情况”功能找到了单个用法，则指针将立即转移到该用法。

转到基础/继承者：如果只有一个结果，请导航而不显示列表

如果选中此复选框，并且通过“ 转到基本符号”或“ 转到派生符号”找到了单个事件，则指针将立即转移到该事件。

编辑器中丰富的鼠标导航

如果选中此复选框，则可以通过以下方式导航到符号声明 Ctrl在编辑器中单击其任何用法，然后通过以下方式导航到符号的实现Ctrl+Alt单击符号。

主菜单命令和Ctrl+B / Ctrl+Shift+Alt+B无论此设置如何，快捷方式均保持可用。

从任何地方在预览选项卡中打开新文件

如果选中此复选框，则每次导航或搜索命令打开一个新文件时，它都会在“ 预览”选项卡中打开。

智能声明声明行为

使用此选择器选择转到声明命令的方式Ctrl+B插入符号已经在声明上时，行为会发生。您有以下选择：

  • 显示“导航至”菜单 -显示“ 导航至”菜单，在上下文中上下文中提供了所有导航目标。此菜单也始终可用Ctrl+Shift+G 。

  • 显示符号的用法弹出窗口 -列出已声明符号的所有用法。您也可以始终使用符号用法命令获取用法列表。 Ctrl+Alt+F7 。

  • 重复使用 -启用“转到声明”命令的使用意识行为 。

  • 不执行任何操作 -只能在符号用法上调用转到声明以转到其声明。

查找通用类型用法行为 该选择器使您可以选择“ 查找用法”功能如何搜索通用类型用法。

搜索通用类型的用法时，例如IFoo ，您实际上可能希望查找的所有用法IFoo<>或确切通用替代的用法IFoo只要。

查找结果窗口
默认情况下展开搜索结果树

如果选中此复选框，则每当ReSharper搜索功能（例如“ 查找用法”或“ 结构搜索和替换” ）得出可以显示为树结构的结果时，该树的所有分支都会在“ 查找结果”窗口中展开 。

在单行上合并事件

如果启用此复选框，则在“ 查找结果”窗口的搜索结果列表中，同一行上的多个搜索匹配项将作为单个结果显示。

打开所选项目的预览标签

如果选中此复选框，则在“ 查找结果”窗口中双击结果时，将在“ 预览”选项卡中打开文件。

要跳过的元素

在此部分中，您可以指定“ 按名称搜索”命令将忽略的文件，文件夹或文件掩码的列表，例如：

  • ?匹配一个字符

  • *匹配一个或多个字符

  • >**递归匹配文件和文件夹

源浏览URI
上游URI

如果您使用JetBrains Upsource进行代码检查和存储库浏览，则可以在此字段中为您的Upsource安装指定URI。这将允许您使用“ 复制全限定名称”功能快速复制当前文件的“上游”链接。
如果您在Upsource中的项目与Visual Studio解决方案名称相对应，则URI应为https://[yourUpsorceServer]/ 。如果Upsource项目名称不同，则URI应为https://[yourUpsorceServer]/[UpsourceProjectName]/ 。

源浏览URI

如果您使用基于Roslyn的源浏览器进行代码存储库浏览，则可以在此字段中为源浏览器安装指定URI。这将允许您使用“ 复制全限定名称”功能快速复制当前文件的源浏览器链接。

ReSharper的设置保存在设置层中，使您可以为不同的解决方案使用不同的设置，并与团队共享您的设置。要了解更多信息，请参阅管理和共享ReSharper设置 。
