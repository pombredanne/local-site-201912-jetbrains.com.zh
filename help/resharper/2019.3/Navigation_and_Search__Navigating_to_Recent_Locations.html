ReSharper 2019.3帮助

浏览最近的位置和更改

ReSharper会跟踪您的操作和导航历史记录，并提供一些命令，使您可以导航到编辑或刚刚研究过代码的地方。

对于每个打开的解决方案，已查看/已编辑文件的列表将分别存储，并在关闭解决方案时将其保存。这意味着您可以在打开解决方案后立即调用此命令，以获取先前工作会话中使用过的文件的列表。

您可以按以下键从查看/编辑的文件列表中删除任何项目： Delete选择此项目时。

导航到最近查看的文件

ReSharper |导航|最近的文件…
Ctrl+E
ReSharper_GotoRecentFiles

此命令可帮助您快速查找在当前解决方案中最近打开的文件。

如果您要查找的文件不在列表中，则可以开始键入其名称来查找它-初始列表将缩小范围以仅显示匹配的文件，同时，解决方案中的其他匹配文件将出现在列表中。清单。

导航到最近编辑的文件

ReSharper |导航|最近编辑…
Ctrl+Shift+Alt+Backspace
ReSharper_GotoRecentEdits

此命令类似于转到最近的文件 ，但是它允许您导航到在当前解决方案中最近修改的符号或文件。

以相同方法进行的最近编辑显示为单个条目，显示最新编辑。使用不同方法进行的最近编辑始终显示为单独的条目。

跳回到上一个编辑

ReSharper |导航|上一个编辑
Ctrl+Shift+Backspace
ReSharper_GotoLastEditLocation

此命令将您的插入符带到您的解决方案中进行了一些更改的位置。您可以多次调用此命令，以进一步回到编辑历史记录中。

浏览导航历史记录

ReSharper |导航|向后/向前导航
ReSharper_NavigateBackward / ReSharper_NavigateForward

使用这些命令，您可以快速浏览ReSharper导航结果的目标。每当您使用“ 导航”和“搜索”命令（当然，“ 向后导航”和“ 向前 导航”除外）时，都会记录导航历史记录，您可以在其中进行前进和后退。

这些命令不同于Visual Studio的向后导航和向后 导航命令。要了解有关后者的更多信息，请参见MSDN Blogs。Zain Naboulsi。向后导航和向前导航

在默认的键盘快捷方式中，这些命令没有快捷方式。但是，您可以根据需要分配自定义快捷方式 。这些命令的别名是ReSharper_NavigateBackward和ReSharper_NavigateForward 。
