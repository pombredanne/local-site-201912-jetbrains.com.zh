将类型移入匹配文件

ReSharper |重构|将类型移到匹配文件中…
ReSharper_MoveIntoMatchingFiles

此命令是一个批量修复程序，可帮助您组织类型和/或委托，以便每个类型/委托定义都位于具有匹配名称的单个文件中，也就是说，如果在单个文件中声明了多个类型/代理，则新建为每个声明创建文件；如果文件名与它包含的单个类型/委托声明不匹配，则文件被重命名。

所有必需的导入都添加到新文件中；所有不再需要的导入都将从原始文件中删除。

将类型移到匹配文件中

  1. 在解决方案资源管理器中，选择要在其中将类型组织到匹配文件中的节点。

  2. 请执行以下任一操作：

    • 按Ctrl+Shift+R然后选择将类型移动到匹配文件中

    • 右键单击并选择“ 重构”。在上下文菜单中将类型移动到匹配文件中。

    • 选择 ReSharper |重构|在主菜单中将类型移动到匹配文件中。

    将打开“ 将类型移动到匹配文件”对话框。

  3. 选择应移入匹配文件的声明类型。

  4. 要应用重构，请单击下一步 。

  5. 如果没有冲突，则立即执行重构。如果存在与新创建或重命名的文件冲突的文件，则向导将显示一个存在冲突的页面。您可以双击冲突以导航至相应的声明并进行修复。
