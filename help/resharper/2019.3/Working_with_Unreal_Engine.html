使用UE4项目

当您打开一个虚幻引擎项目时，ReSharper将自动调整其所有功能和动作，以便它们考虑到虚幻引擎的不同方面和约定。例如，自动导入和代码分析将尊重.generation.h文件的包含。

ReSharper的大多数UE功能都是在后台进行的，但本主题中仅介绍了其中的一些功能，这些功能提供了一些操作和配置。

在你开始前

如果您尚未这样做，请确保您的Visual Studio副本已设置为可与虚幻引擎项目一起使用-这是虚幻引擎文档中的详细说明 。

至少我们建议您禁用Visual Studio花体，因为它们经常在虚幻引擎项目中产生误报。选项| C ++ |进阶|代码分析并将禁用花粉或禁用C ++代码分析经验设置为True 。

索引虚幻引擎代码

您可以在打开UE4项目几秒钟后开始输入或浏览它，因为默认情况下，解析非引擎代码后，引擎代码（构成项目的主要部分）的索引在后台执行。

您可以配置引擎代码的索引方式 代码编辑| C ++ |的虚幻引擎页面 ReSharper选项（ Alt+R,,O ）：如果您希望在启动之前为所有引擎代码建立索引，则可以禁用后台索引编制，或者，如果您注意到任何性能下降，则可以通过清除“ 索引虚幻引擎”源文件来完全禁用索引编制。

启用索引功能后，您将拥有许多功能。例如，使用引擎中的任何符号，ReSharper会自动添加缺少的#include或查找引擎类的用法 ，例如， TArray —在引擎的代码中，研究在那里如何使用它们。

如果禁用了索引编制，则ReSharper仍将为头文件名建立索引，这是非常快的，但是它将使您可以完成包括在内的代码，例如， #include 。包含标题后，其代码将被自动索引，因此您将完成代码完成并分析包含的标题中的符号。

虚幻引擎命名约定

在开发UE4项目时，您可能正在使用Unreal Engine编码标准 。ReSharper会检测UE4项目，并在这些项目中生成和分析代码时自动使用UE4命名约定规则。

与默认规则集命名，您可以配置在 代码编辑| C ++ |的命名样式页面 ReSharper选项（ Alt+R,,O ），虚幻引擎的命名约定是不可配置的。ReSharper使用特殊的代码检查来检测是否违反了虚幻引擎编码标准的命名规则 。这些检查会抑制UE4项目中的当前命名规则集。如果要更改严重性级别或禁用这些检查，可以在C ++ | Windows Server 2003中找到它们。虚幻引擎开启 代码检查|的检查严重性页面 ReSharper选项（ Alt+R,,O ）。您还可以看到相应的警告，并且可以在“ 代码编辑” |“禁用”下禁用UE4命名检查。 C ++ |打开UE4解决方案时的命名样式 ：

UE4反射说明符

ReSharper支持虚幻引擎反射宏，例如UCLASS ， USTRUCT ， UENUM等等，用于模拟C ++语言中缺少的反射。这些宏将得到解决，即使未构建虚幻引擎项目，您也将获得有关它们的完成建议 。

快速文档 （ Ctrl+Q ）和代码完成中的摘要工具提示也可用于反射宏。

UE4远程过程调用支持

分析代码时，ReSharper通过以下方式识别UE4远程过程调用（RPC） ： Client ， Server和NetMulticast中的说明符UFUNCTION声明声明。

ReSharper检查UE4 RPC是否具有相应的必需项_Validate和_Implementation功能，如果未找到其中至少之一，则会突出显示RPC。你可以按Alt+Enter突出显示以生成缺少的实现存根。

在上面的示例中， _Validate函数丢失，ReSharper将为您生成以下存根：

bool AShooterCharacter :: ServerEquipWeapon_Validate（AShooterWeapon * NewWeapon）{}

从RPC中，您可以跳到他们_Validate和_Implementation导航到的功能Ctrl+Shift+G弹出窗口，或者从Alt+Enter菜单，如上面的屏幕截图所示。

它也适用于BlueprintNativeEvent ：传入相应的说明符时UFUNCTION ，ReSharper可以生成一个_Implementation功能，以及从声明导航到它。

您可以在UE4 RPC函数调用上使用“ 重命名和更改签名”重构-相关_Validate和_Implementation功能将相应更新。
