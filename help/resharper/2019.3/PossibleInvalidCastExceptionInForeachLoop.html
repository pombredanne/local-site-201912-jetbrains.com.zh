代码检查：可能的“系统。foreach循环中的InvalidCastException”

您可以禁止执行此检查以忽略特定问题 ， 更改其严重性级别以使问题更小或更明显 ，或者将其完全禁用 。

根据设计， foreach允许将隐藏的类型转换为派生类型。一方面，这使它易于使用，但另一方面，这可能导致运行时异常。

ReSharper希望您安全起见，因此通知可能System.InvalidCastException如果它检测到不安全的转换，例如下面的示例。

public void Foo（IEnumerable list）{//可能的'System。从BaseType转换为DeeachedType foreach（列表中的DerivedType项）时发生InvalidCastException'{//做某事}}

ReSharper在此建议的快速修复可帮助您使演员表明确。它仍然不安全，但不再隐藏：

公共无效Foo（IEnumerable 列表）{foreach（列表中的基本类型）{变量=（DerivedType）基本类型； // 做点什么 } }

如果要进行强制类型转换，建议将显式类型引用替换为var 。

如果您的收藏品实施IEnumerable而您使用的成员DerivedType不存在于BaseType ，您可能需要在迭代之前过滤您的集合。请注意，这将更改语义：

public void Foo（IEnumerable list）{foreach（列表中的DerivedType项。OfType （））{//仅对DerivedType的对象做某事}}
