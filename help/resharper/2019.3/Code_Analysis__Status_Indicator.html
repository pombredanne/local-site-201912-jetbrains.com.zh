标记栏和状态指示器

标记栏和状态指示器可让您观察当前文件中设计时代码检查的结果，并在代码问题之间导航。此外，标记栏还会显示其他ReSharper项目的特殊标记，例如当您突出显示当前文件中的用法时 。

标记栏未显示，状态指示器显示暂停图标如果：

  • 标记栏属性设置为“不显示在环境|编辑器 ReSharper选项的“ 编辑器外观”页面。

  • 设计时代码检查已禁用 。

  • 编辑器显示反编译的代码 。

标记栏

标记栏 （也称为“错误条带”）是集成到编辑器滚动栏中或显示在滚动栏右侧的侧边栏。状态栏可以将以下项目表示为单独的标记：

  • ReSharper 设计时检查发现的代码问题。

  • 结构搜索和替换结果
  • 待办事项
  • 文件中符号的用法

标记具有不同的颜色，表示相应项目的种类。颜色可能会因各种配色方案而异，但是您可以分别为每种项目配置标记栏颜色 。

条形上标记的位置指示文件中相应项目的相对位置。您可以通过将鼠标指针放在标记上来获得每个项目的弹出提示，还可以通过单击它们直接导航到相关的代码行。

在处理文件时，通常会在标记栏中看到许多标记。当您完成代码时，许多这些错误，警告和建议最终都会得到解决。如果您觉得自己的代码已完成，但仍然有任何错误，那么在尝试编译项目之前，探索并解决发现的错误可能会节省时间。

配置标记栏的外观

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择环境|编辑器编辑器外观在左侧。

  2. 使用标记栏下的单选按钮选择是否显示标记栏，以及是否应将其集成到滚动栏中（从Visual Studio 2013开始）。

  3. 如果标记条已集成到滚动条中，则还可以设置标记的宽度。

配置标记的颜色

  1. 选择工具|从菜单中选择选项 ，然后转到环境|字体和颜色页面。

  2. 在显示项目列表中，找到以ReSharper开头并以...结尾的项目。错误条 带上的 标记 ，例如： ReSharper代码分析错误条带上的错误标记 。

  3. 使用项目前景选择器为选定的标记选择颜色。

状态指示灯

状态指示器 -标记栏顶部的小图标-显示文件中代码检查的状态或指示用法突出显示模式 。将鼠标指针悬停在其上时，指示器将显示当前文件中的错误和警告总数，或者（如果处于使用情况突出显示模式）该文件中的使用次数。

您可以单击状态指示器在文件中的代码问题或用法之间跳转。

状态指示器的上下文菜单

您也可以右键单击状态指示器以从其上下文菜单中使用以下命令和选项：

转到下一个/上一个错误/警告/消息

如果当前文件中存在代码问题，则会显示这些命令。他们让你在最高严重性级别的代码问题之间导航。因此，如果存在错误，它们只会在错误之间导航，一旦所有错误都被修复，它们就会在警告之间导航，依此类推。

转到下一个/上一个用法

当您突出显示文件中的用法并允许您在找到的用法之间跳转时，将显示这些命令。

增强的语法着色

此选项使您可以启用或禁用ReSharper 语法突出显示方案。
如果被选中，语言标识符颜色突出显示在Visual Studio中的选项定义： 工具|选项|环境|字体和颜色 。
“ 显示项目”列表中提供了ReSharper提供的语法标识符列表，每个名称均以ReSharper前缀开头。
该选项与“ 代码检查” | “ 代码检查”中的“ 颜色标识符”选项同步。 ReSharper选项的“设置”页面。

显示导入/包含弹出窗口

如果选择此选项，则在文件中检测到一种或多种未导入的类型时，将显示一个弹出窗口，提示在C＃和VB.NET中导入名称空间 ：

如果未选中该复选框，则相应的操作将在以下位置的操作列表中可用Alt+Enter 。
该选项与“ 代码检查” | “使用弹出窗口显示'导入名称空间'操作 ”选项同步。 ReSharper选项的“设置”页面。

显示镶嵌提示
Ctrl+Shift+Alt+5

切换当前解决方案中的镶嵌提示 。

启用C整洁分析
Ctrl+Shift+Alt+9

切换当前文档的集成Clang-Tidy分析 。

抑制VS灯泡

此选项在2015年之前的Visual Studio版本中不可用。
如果选择它，则不会单独显示Visual Studio的“快速操作”（灯泡和螺丝刀图标） ，而是集成到ReSharper的操作列表中 。
该选项与“ 环境” | “不显示Visual Studio灯泡”选项同步。编辑器 ReSharper选项的“ Visual Studio功能”页面。

抑制VS曲线

此选项在2015年之前的Visual Studio版本中不可用。
如果选择该选项，则不会在编辑器中显示Visual Studio的错误突出显示，而仅显示ReSharper的突出显示。
该选项与“ 环境” | “ 环境”中的“ 抑制Visual Studio文字”选项同步。编辑器 ReSharper选项的“ Visual Studio功能”页面。

暂停/继续分析
Ctrl+Shift+Alt+8

此选项使您可以从代码检查中排除当前文件，或者如果先前已将其排除，则可以包括它。

状态指示器图标的含义

状态指示器可以具有不同的图标，其含义如下：

只要看到红色标记，就意味着该文件包含错误，这些错误会阻止项目编译或在运行时导致异常，除非在项目属性中启用了将警告视为错误 。

黄色标记表示存在一些警告，但没有错误会阻止项目编译。

绿色标记表示您的代码不包含错误或警告 ，但仍然可能存在严重性级别较低的代码问题（ Recommendation或Hint ）。

该文件从代码检查中排除 。

当您突出显示文件中的用法时 ，标记会显示与用法相对应的蓝色标记，状态指示灯会显示相应的图标，直到您按Esc 。在此模式下，您可以单击状态指示器在找到的用法之间跳转。

在Visual Studio项目设置中，您可以选择将警告视为错误。ReSharper知道此选项并相应地突出显示警告：如果启用此设置，则与编译器警告相对应的问题将突出显示为错误。此设置在项目属性配置： 项目| [项目名称]属性|将警告视为错误 ，如果选择全部或仅将其应用于指定警告，则可以将其应用于所有警告。
