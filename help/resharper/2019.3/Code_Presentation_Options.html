编译代码的演示选项

取决于在设置 ，PDB信息的可用性以及所应用的导航命令上 ，ReSharper可以以不同的方式显示已编译的代码：

  • 元数据视图 -此视图仅显示类型的签名及其所有成员的签名。您还可以通过从编辑器或“ 程序集资源管理器”窗口中应用“ 导航到元数据视图”命令来显示此视图。

    除了使用元数据视图外，您还可以查看文件结构

  • 反编译的代码 -此视图显示与您打开或导航到的类型相对应的反编译的C＃代码。如果没有调试信息可用于该类型，或者您使用“ 导航到反编译的源代码”命令故意导航到该视图，则默认使用此视图。

  • 符号文件中的代码 -此视图显示C＃或VB.NET中符号文件中的原始源代码。ReSharper使用调试信息从本地源或符号服务器中检索符号文件。

要配置反编译代码的显示方式，请使用以下复选框 工具|的“外部来源”页面 ReSharper选项（ Alt+R,,O ）：

可用时使用符号文件中的源

选中此复选框可使用调试信息（PDB文件）查找适当的源文件，并使用它们显示和导航代码。

如果选中此复选框，并且可通过调试信息获得源文件，则ReSharper将显示并导航到原始源代码。


如果清除此复选框或源文件不可用，则ReSharper会根据“ 反编译方法”选项反编译代码或显示元数据视图。

您始终可以强制显示反编译或源代码（如果可以通过PDB文件或从符号服务器获得）。为此，在Assembly Explorer窗口中右键单击一个项目，然后选择Decompiled Sources或Symbol Files中的Sources 。

允许从远程位置下载

选择此复选框以启用从Symbol Server或其他远程位置下载源。如果符号文件在本地和远程均可用，则ReSharper将显示本地符号文件。

您的计算机应连接到Internet，此选项才能起作用。如果使用代理服务器，请检查是否在以下位置正确配置了相应的选项 环境|的“ Web代理设置”页面 ReSharper选项（ Alt+R,,O ）。

反编译方法

选中此复选框可将方法主体反编译为C＃代码。如果未选中此复选框，将显示带有方法签名的元数据视图 。
如果选择了“ 在可用时使用符号文件中的源”，并且所需的符号文件可用，则此选项无效。

您可以同时在不同的选项卡中打开同一文档的不同视图。

要查看打开的文档的摘要和演示模式，请右键单击其选项卡标题，然后在上下文菜单中选择“ 外部源信息 ”。在出现的对话框中，您可以了解文档所属的程序集的名称和位置，以及外部源提供程序-元数据查看器，反编译器或符号服务器。

如果您想随时清除反编译缓存，请删除％LOCALAPPDATA％\ JetBrains \ Shared \ vAny \ DecompilerCache中的数据 。
