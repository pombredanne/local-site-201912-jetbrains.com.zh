查找外部符号的用法

ReSharper |查找|参考代号
ReSharper_FindReferencedCode

ReSharper可帮助您在一定范围内查找任何传出的引用，范围可以像函数一样狭窄，也可以像项目一样大。根据搜索范围，ReSharper将向您显示该范围之外定义的所有符号的用法。这些可能是解决方案中的符号，也可能是当前项目中引用的已编译程序集中的符号。

当您要将类或方法移动到另一个位置时，此命令会派上用场。您将能够使用外发引用列表来估计此举所需的其他更改。

要从项目中引用的特定模块中查找符号的用法，请使用“ 查找依赖于模块的代码”命令。

考虑一下ExternalSymbols下课。如果在构造函数上运行命令，则搜索结果将是该方法的一种用法Divide ，因为它是在构造函数之外定义的。如果您在该类上运行该命令，则搜索结果将是System.ArgumentOutOfRangeException ，因为其余部分在类内部定义。

class ExternalSymbols //类外部的符号：系统。ArgumentOutOfRangeException {ExternalSymbols（）//构造函数外部的符号：MyProject。外部符号。除法{var x = Divide（10,2）; } private int Divide（int numerator，int denominator）{如果（分母！= 0）返回分子/分母；抛出新的ArgumentOutOfRangeException（“ denominator”）; }}

如果在项目上运行命令，搜索结果将包括引用的程序集的符号用法以及解决方案中其他项目的符号用法（如果有）。

有两种方法可以搜索外部符号：如果范围是项目，项目文件夹或文件，请从解决方案资源管理器中开始搜索；如果范围是类或函数，请从编辑器开始搜索。

查找当前范围之外定义的符号的用法

  1. 请执行以下任一操作：

    • 在解决方案资源管理器中选择一个项目，一个项目文件夹或一个文件。

    • 在编辑器中，将插入符号设置为函数或类。

  2. 在主菜单中，选择 ReSharper |查找|参考代码 。
    如果要从解决方案资源管理器开始搜索，则还可以右键单击所选项目，然后在上下文菜单中选择“在作用域外部查找符号” 。
    如果要从编辑器开始搜索，也可以按Ctrl+Shift+G然后在“ 导航到”菜单中选择“ 参考代码” 。

  3. 搜索结果将显示在“ 查找结果”窗口中 ，您可以在其中进一步分析结果 。
