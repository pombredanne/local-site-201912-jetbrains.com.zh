探索项目依赖图

ReSharper |建筑|显示项目依赖图
ReSharper_ShowProjectDependencyGraph

使用ReSharper，您可以在解决方案中以可视化方式显示解决方案体系结构来探索项目依赖性。您随时可以获取解决方案的项目依赖项快照并进行研究。请注意，当您更改项目之间的依赖关系时，项目依赖关系快照不会更新-要刷新依赖关系视图，您必须创建另一个快照。

通过架构视图，可以直观地浏览项目依赖项快照， 并将解决方案的当前状态与之前拍摄的快照进行比较 。

右键单击图上的项目，以使用与解决方案资源管理器中解决方案节点的上下文菜单中可用的相同的ReSharper 项目级命令 （包括代码检查 ， 导航 ， 重构和单元测试 ）。

获取项目依赖项快照

项目相关性快照是解决方案中所有项目以及它们之间所有相关性的记录，在特定的时间获取。使用ReSharper，您可以拍摄并保存此类快照。以后，您可以加载已保存的快照并将其与解决方案的当前状态进行比较。架构快照中提供了依赖项快照的直观表示。

对于大型解决方案，构建依赖关系图和计算代码度量可能需要一些时间。但是，分析是异步执行的，没有构建解决方案。因此，您可以在构建图时继续使用代码。

获取项目依赖项快照

  1. 在解决方案资源管理器中 ，选择一个或多个项目，一个或多个解决方案文件夹或解决方案节点。

  2. 右键单击所选内容，然后选择“ 显示项目依赖关系图”或选择 ReSharper |建筑|在主菜单中显示项目依赖关系图 。

  3. “ 架构视图”将在新的文档选项卡中打开，其中显示了您最初选择的项目的依赖性。

  4. 最初选择的项目也会在Architecture View的左侧检查，该视图列出了所有解决方案项目。您可以检查其他项目：它们将出现在图中，并且将添加相应的链接。

  5. 如有必要，您可以使用工具栏中的分组依据选择器来更改项目分组：

    • 无 -所有项目均按分组显示。

    • 解决方案文件夹 -项目按解决方案文件夹分组。

    • 文件结构 -项目和解决方案文件夹按文件系统结构分组。

了解项目之间的联系

当您获得依赖项快照时，ReSharper会分两个步骤进行创建：

  1. 根据项目引用构建依赖关系图。即使在大型解决方案中，此步骤也可以很快完成。

  2. 进行耦合分析以计算参考项目中的使用次数 。此步骤可能需要几分钟，并且会在后台运行。分析的进度由“ 体系结构视图”顶部的状态栏反映。

    默认情况下启用的耦合分析状态对应于打开/关闭 “ 打开耦合分析”状态 按钮。 耦合分析是一个资源密集型过程，可能会影响大型解决方案的性能。如果在使用图表时遇到任何滞后，可以使用此按钮禁用耦合分析，根据需要调整图表，然后再次启用分析。分析算法首先计算显示项目的耦合，然后处理从视图中隐藏的项目。

图表中项目之间的链接可以通过以下方式显示：

线型

含义

黑色实线

耦合分析正在进行中或已禁用，因此该参考的使用次数未知。

特定宽度的蓝色实线

对于此参考，耦合分析已经结束，并且参考项目中至少存在一种用法。线宽表示在参考项目中使用参考项目的类型的次数。如果将鼠标悬停在参考上，此数字将显示在工具提示中。

该参考的耦合分析已结束，该参考未使用。您可以右键单击它，然后选择将其删除。

这两个项目通过当前在视图中隐藏的其他引用项目链接。

绿色实线

红色实线

显示和隐藏项目

您可以使用左侧对应项目的复选框在图中显示或隐藏特定解决方案项目（项目，文件夹，解决方案文件夹）。还有一些方法可以直接从图表区域显示/隐藏项目：

  • 要选择图中的某些项目并隐藏其他项目， Ctrl单击要选择的项目，然后右键单击该选择，然后选择“ 选择” |“选择”。仅显示选定的模块 。

  • 要选择两个链接的项目，请右键单击它们之间的链接，然后选择“ 选择” |“选择”。仅显示选定的模块 。

  • 要隐藏项目，可以在图表中选择它，然后按Delete 。

  • 专注于特定项目的另一种方法是显示依赖或引用的项目并隐藏所有其他项目。为此，选择一个或多个项目（项目或分组项目），右键单击所选内容，然后选择“ 显示此/这些以及从属/引用” 。请注意，这些命令仅隐藏不相关的项，并且不添加任何当前隐藏的引用/依赖节点。

  • 要显示特定项目引用的或依赖于特定项目的所有项目，即使这些项目当前已从视图中隐藏，也请选择一个或多个项目，然后选择“ 选择” |“选择”。添加引用/相关模块 。

如有必要，您可以通过单击撤消来逐步撤消对图表（显示/隐藏的项目）中所做的更改。 或按Ctrl+Z 。要撤消撤消命令，请单击“ 重做”。 或按Ctrl+Y 。

平移和缩放图表

要平移和缩放图表，请使用图表区域左上角的控件。或者，您可以使用滚动条平移该图，或在图项目之外的图区域中单击任意位置，然后在手形光标上拖动显示。您也可以使用鼠标滚轮进行缩放。

重新布置图中的项目

图表布局是自动计算的，但是您可以根据需要更改它：可以移动所有项目。要移动项目，请先单击它以进行选择，然后在十字光标的同时单击并拖动它显示。

在图上对项目进行分组

使用“ 分组依据”选择器，可以按解决方案文件夹或文件系统结构对项目进行分组。分组在左侧和图表中均适用。

如有必要，可以通过以下方式之一折叠和展开图中的分组项：

  • 单击分组项目左上角的向下箭头。折叠后，该符号将变为向上箭头，您可以单击它以展开该项目。

  • 在图中选择一个分组项目，然后按Ctrl+Left,Arrow和Ctrl+Right,Arrow 。

  • 单击折叠图 和展开图 在工具栏上折叠或展开所有分组项目。

分析项目参考

要了解有关用法的更多信息，请选择相应的链接，然后按Alt+F7或右键单击并选择显示用法 。ReSharper将在“ 查找结果”窗口中显示用法。

如果针对特定参考的耦合分析已经结束，并且ReSharper将其突出显示为未使用（带有虚线），则无需从图上切换以删除未使用的参考-只需选择它并按Ctrl+Shift+R或右键单击并选择Refactor This 。然后按Alt+Delete或单击“ 安全删除程序集参考” 。

检测可能的架构问题

如果您尝试遵循解决方案体系结构中的最佳实践，则您的应用程序可能具有分层的结构 ，并具有相互通信的单独层。

显示可能的路径 按钮使您不仅可以根据分组，还可以根据层之间的通信来找出应用程序层的设计是否正确。简而言之，当该按钮处于打开状态，您可以查看两个特定项目之间是否具有由当前从视图中隐藏的其他引用项目组成的路径。这些路径以绿色虚线显示。仅当存在隐藏的中间项目以链接当前显示的任何两个项目时，才启用该按钮。

将图导出到图像

在探索依赖项的任何时候，都可以将当前视图导出到图像。为此，请点击导出到PNG 并指定保存图像文件的位置。

将保存的快照与当前解决方案状态进行比较

如果架构视图中显示的依赖项快照与当前解决方案状态不同，则显示差异按钮变为可用。

如果要记录解决方案的当前状态，可以将快照保存到文件中。为此，请单击“ 保存体系结构图” 在工具栏上，然后选择保存快照文件的位置。

您可以稍后选择加载和研究已保存的快照 ReSharper |建筑|在主菜单中与已保存...进行比较 。如果使用与当前快照相同的解决方案来保存快照，您将能够看到解决方案的当前状态与快照中保存的状态之间的差异。
