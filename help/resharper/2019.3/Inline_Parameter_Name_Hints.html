参数名称提示

作为在方法调用中编写命名参数的一种替代方法，您可以使用ReSharper的参数名称提示 - 在方法调用的相应参数旁边显示参数名称的编辑器装饰。

考虑以下方法调用：

AddCheckbox（true，true，“ AutoFill”，null）;

阅读此代码时，您必须停止研究方法签名，以了解此处发生的情况。 （您可以使用参数信息工具提示或导航到方法声明来完成此操作 ）。使用参数名称提示，相同的调用将变得更加清晰：

您可以将鼠标悬停在嵌入提示上，以查看相应符号的XML文档（如果可用），或者Ctrl单击以跳转到声明。

参数名称提示的想法是自动显示意图不明确的参数的参数名称，但同时不要在提示参数目的很明显的提示中（当然，如果其中有命名参数）使编辑器混乱电话。为了实现这一目标，ReSharper应用了许多可配置的启发式方法 。

使用ReSharper，您还可以配置代码样式首选项 ，以建议在代码中使用命名参数。

您可以配置方法模式列表，其参数名称提示将始终隐藏，而与其他首选项无关。

模式与完全限定的方法名称，参数计数和参数名称匹配。例如：

  • System.Math.*匹配所有以FQN开头的方法System.Math

  • System.Math.*(*, *)匹配来自的所有方法System.Math具有两个参数的名称空间

  • (*info)匹配所有以参数名称结尾的单参数方法info

  • (key, value)将所有方法与参数匹配key和value

  • Put(key, value)匹配所有Put方法key和value
