代码检查：进入已关闭的封盖

您可以禁止执行此检查以忽略特定问题 ， 更改其严重性级别以使问题更小或更明显 ，或者将其完全禁用 。

首先，让我们确保您了解什么是闭包 。简而言之，C＃中的闭包是lambda表达式或匿名方法，该方法从外部作用域捕获一些变量。这是最简单的示例：

//一个自包含的lambda。没有关闭。动作printOne =（）=> {控制台。WriteLine（“ one”）; }; //闭包-从外部作用域捕获变量的lambda。字符串myStr =“ one”;操作打印=（）=> {控制台。WriteLine（myStr）; };

在上面的示例中， print将捕获变量myStr （而不是它的值 ），并且只会获得的值 myStr当您调用print() 。

在更复杂的情况下，当在变化的上下文中定义闭包时，它的行为可能不符合预期。

可能发生的情况之一是在using声明：

void Foo（字符串fileName，字符串文本）{使用（var writer = new StreamWriter（fileName））{ExecuteDelayed（（）=> {writer。写（文字）; }）; }} private static void ExecuteDelayed（Action action）{//执行动作}

在上面的代码中，ReSharper发出访问以下内容的已处置关闭警告： writer.Write(text); 。原因是ExecuteDelayed()可以在执行之后执行lambda writer的范围被处置，这将导致运行时异常。

如果ExecuteDelayed()完成堆栈上lambda的处理，您可以标记action InstantHandle属性的参数：

私有静态无效ExecuteDelayed（[InstantHandle] Action action）{//执行动作}

这将告诉ReSharper的代码分析引擎， ExecuteDelayed()在这种情况下是安全的，因此不会发出警告。
