ReSharper 2019.2帮助

TypeScript中的代码检查

在TypeScript中，ReSharper 2019.2提供了两种代码检查：141种检查错误的语法检查，例如语法损坏，无法解析的符号，编译器错误等（您不能配置或禁用任何这些检查），以及51 专有代码检查 ，您可以禁用或更改其严重性水平 。
这些代码检查可帮助您在设计时检测所有打开文件中的代码问题，此外，它们还使您可以查找特定范围内的代码问题 。

下面列出了可配置的TypeScript检查，按其类别分组。

代码样式（5次检查）

此类别中的检查可检测到违反代码语法样式的行为。与大多数其他代码检查相比，这些检查可以检测与代码发布相同的代码构造，也可以不检测，具体取决于在代码上配置的相应代码样式规则。 代码编辑| [语言] |的代码样式页面 ReSharper选项（ Alt+R,,O ）。您还可以使用代码清除修复这些检查检测到的问题。

语言使用机会（9次检查）

此类别包括代码检查，大部分带有建议严重性级别，这些通知会在可以使用更高级的语言构造时通知您。这些检查将检测过时的语言版本的语法，并建议使用更多现代语言版本的功能。对于大多数受支持的语言， 可以自动检测语言版本或手动设置 。

代码中的冗余（9次检查）

此类别中的代码检查查找冗余和无效代码，这些代码会影响代码的可读性和样式，并且可以安全地删除。 某些代码冗余无法自动修复，因此需要在用户输入的情况下以交互方式进行快速修复。但是，可以使用范围内的修复或代码清除来修复大多数冗余，而无需用户交互。
