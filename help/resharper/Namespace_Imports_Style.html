代码语法样式：命名空间导入

在每个源代码文件中，您应该具有正确的名称空间导入集（ using C＃中的指令）。如果缺少导入，则您的代码将无法编译，如果有未使用的导入，则随着时间的推移，它们可能会使您的代码难以维护。此外，导入命名空间的方法也不同，例如，您可以添加using指令或使用完全限定的类型名称，可以放置using文件顶部或名称空间中的指令，等等。

ReSharper提供了许多功能来帮助您根据喜好控制和自动更新导入名称空间的各个方面。

配置名称空间导入首选项

您的名称空间导入首选项使用基于层的设置机制保存。除其他外，此机制使您可以针对不同的解决方案维护不同的首选项，并将这些首选项保持在VCS下，并自动与团队成员共享。

配置名称空间导入首选项

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O ，然后选择代码编辑| C＃|命名空间在左侧导入 。

  2. 在此选项页面上，您可以指定不应删除的名称空间和/或应始终导入的名称空间。

  3. 打开代码编辑| C＃| ReSharper选项的“代码样式”页面。

  4. 使用“ 引用限定”和“使用”指令组中的首选项来微调名称空间导入样式：

    • 首选完全限定的引用 —如果要对导入的类型使用完全限定的名称而不是名称空间导入指令，请选中此复选框。

    • 将“ using”指令添加到最深的作用域 -如果选中此复选框，则在使用导入类型的名称空间中添加名称空间导入指令；否则，导入指令将添加到文件顶部。

    • 在对“使用”指令进行排序时，请首先放置“ System。*”和“ Windows。*”命名空间。如果选中此复选框，则“使用”指令将像using System.Linq;排序时将被放置在其他using指令之上。否则，这些指令将与其他指令按字母顺序排序。

    • 在嵌套范围内首选使用名称的完全限定名称 -如果选中此复选框，则在嵌套类型和名称空间中首选使用导入类型的完全限定名称。

    • 允许使用别名指令 -选中此复选框将允许在名称空间导入指令中使用别名，而不是完全限定的名称空间名称。

    • 允许使用'global ::'前缀 -如果选中此复选框，则global::前缀未删除，例如global::System.String 。有关更多信息，请参见如何：使用全局命名空间别名 。

  5. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

导入缺少的名称空间

ReSharper帮助您检测缺少名称空间导入的符号，并自动添加正确的导入。未解决的符号会立即在编辑器中突出显示 ，并建议相应的修复程序：

如果文件中有几个未解析的符号，则此修复程序将为所有符号添加导入。根据名称空间导入的已配置首选项添加导入 。有关更多信息，请参见导入缺少的命名空间 。

删除多余的名称空间导入并优化现有的名称空间导入

如果删除命名空间符号的用法，则任何导入的命名空间在代码中都将变得多余。如果没有ReSharper，则必须滚动到每个文件的顶部，找到未使用的名称空间导入指令，然后将其删除。ReSharper允许您自动从文件，项目或解决方案中删除未使用的名称空间导入指令，从而使工作变得更加轻松。除了删除多余的名称空间导入，ReSharper可以根据您的名称空间导入首选项以多种方式对它们进行优化。

与名称空间导入相关的首选项不会应用于生成的代码。默认情况下，ReSharper提供通常由Visual Studio生成的生成代码（例如Windows Form Designer生成的代码）的文件掩码和区域列表。如有必要，您可以在上编辑生成的项目列表 代码检查|的“生成的代码”页面 ReSharper选项（ Alt+R,,O ）。

编辑器会突出显示所有冗余的命名空间导入，并建议相应的快速修复 ，包括作用域内的修复 ：

通过应用建议的修复程序，您可以在所需范围内立即删除冗余名称空间。

在批量模式下删除冗余名称空间导入的另一种方法是代码清除 。您可以可以使用默认的默认值：完全清理配置文件运行代码清理 ，或者创建和运行仅针对您的特定任务的自定义配置文件 ，如下所述。

与快速修复相反，代码清除不仅可以删除未使用的导入，还可以根据您的喜好优化现有的导入。例如，使用代码清除，可以通过添加相应的名称空间导入，用纯类型名称替换完全限定的类型名称。

使用代码清理优化名称空间导入

  1. 选择ReSharper |在主菜单或按选项 Alt+R,,O 。

  2. 转至代码清理设置页面： 代码编辑|代码清理 。

  3. 如创建新的自定义清理配置文件部分中所述创建新的配置文件 。在新配置文件的“ 选定的配置文件”设置中：

    • 选择优化“使用”指令 。

    • （可选）包含未使用的using区域中的指令，选择在区域中包含“使用”指令，然后在“ 区域名称”字段中指定区域名称 。

  4. 请点击单击保存以应用修改，并让ReSharper选择将修改保存到何处，或使用“ 保存到”列表将修改保存到特定的设置层。有关更多信息，请参阅管理和共享转发器设置 。

  5. 选择您要优化名称空间导入的范围：

    • 将插入符号设置在文件中的任何位置，以优化名称空间导入到文件中。

    • 在解决方案资源管理器中选择一项或多项，以优化这些节点及其子项下文件中的命名空间导入。

  6. 请执行以下任一操作：

    • 按Ctrl+Alt+F或选择 ReSharper |编辑从主菜单清除代码 。

    • 右键单击文本编辑器中的任意位置，或右键单击所选内容，然后在上下文菜单中选择“ 清理代码” 。

  7. 在打开的“代码清除”对话框中，选择新创建的配置文件。

  8. 点击运行 。ReSharper将在选定范围内优化名称空间导入。

如果要在不打开“ 代码清理”对话框的情况下优化名称空间导入，则可以将创建的配置文件绑定到静默清理并通过按以下命令运行它Ctrl+Shift+Alt+F 。您还可以创建一个自定义清除配置文件 ，该配置文件将优化名称空间导入与其他代码样式任务结合在一起。
