代码检查：自动属性可以设为只读（非私有可访问性）

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

从C＃6.0开始，您可以定义仅获取自动属性，该属性（类似于readonly字段 ）只能通过构造函数或初始化程序进行初始化。ReSharper检测并帮助您删除多余的set从构造函数/初始化器初始化且没有写用法的自动属性上的访问器。

此检查建议您应该使用C＃6.0功能。 默认情况下，ReSharper根据关联的编译器自动检测C＃版本。但是，您可以显式指定目标C＃版本-在解决方案资源管理器中右键单击项目，从上下文菜单中选择“ 编辑项目项属性” ，然后使用C＃语言级别选择器。

在下面的示例中，将使用不可变的类，并且一旦Name在构造函数中检查属性的可空性，可以安全地使用它，而无需进行进一步的空检查。但是，私有设置者不保证以后私有成员不会更改该属性。因此，最好将此属性设为只读，以防止进行任何修改。

逆向转换可与上下文动作一起使用 Alt+Enter 。

公共类Person {公共字符串名称{get;组; } //可以将自动属性设为只读public Person（string name）{Name = name ??抛出新的ArgumentNullException（nameof（name））; }公开替代字符串ToString（）{返回$“ NAME：{Name。ToUpper（）}“;}}

为了使解决方案范围的检查有效，您需要以下一项或两项：

  • 启用了简化的全局使用检查— 在关闭解决方案范围的分析时，显示未使用的非私有类型成员 代码检查|的设置页面 ReSharper选项（ Alt+R,,O ）。

  • 该解决方案范围的分析已启用- 启用解决方案范围的分析 代码检查|的设置页面 ReSharper选项（ Alt+R,,O ）。

请注意，即使符号在您的解决方案中没有直接使用且ReSharper会警告您，在某些情况下，符号可能会间接使用（例如，通过反射），也可能只是设计为公共API。在所有这些情况下，您都希望抑制符号的使用检查，并且有几种方法可以这样做：

  • 推荐的方法是用代码注释属性修饰隐式使用的符号。为此，有两个属性： [UsedImplicitly]和[PublicAPI] ，它们在功能上相似，但是可以让您和您的队友了解如何实际使用该符号。

  • 您还可以使用任何自定义属性禁止使用情况检查检查。为此，请使用[MeansImplicitUse]属性标记该属性的定义。

  • 最后，您可以像抑制其他代码检查一样，通过抑制注释或抑制属性来抑制特定的使用检查。
