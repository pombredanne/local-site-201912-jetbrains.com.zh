更改签名重构

ReSharper |重构|更改签名…
Ctrl+F6
ReSharper_ChangeSignature

此重构结合了可以对方法，构造函数，属性和索引器的签名进行的几种不同的修改。除了更改声明中的签名外，ReSharper还可以查找和更新当前解决方案中所有用法，基本符号，实现以及修改后的符号的替代。

如果只需要更改方法名称，请使用Rename refactoring 。

ReSharper允许您使用Transform Parameters refactoring以其他方式（摆脱掉参数，将参数包装在元组或新类中，等等）更改方法签名。

用命令调用重构

更改功能的签名

  1. 将插入号放在编辑器中的声明或方法，属性，构造函数或索引器的声明处，或在“ 文件结构”窗口中选择它。

  2. 请执行以下任一操作：

    • 按Ctrl+F6 。

    • 按Ctrl+Shift+R然后选择更改签名

    • 右键单击并选择“ 重构”。在上下文菜单中更改签名 。

    • 选择 ReSharper |重构|在主菜单中更改签名...。

    将打开“ 更改签名”对话框。

  3. 在名称字段中输入符号的新名称。如有必要，请在“ 返回类型”字段中更改方法的返回类型 。

  4. 在“ 参数”区域中，编辑现有参数的类型，名称，修饰符和默认值。如有必要，请使用“ 添加”和“ 删除”按钮来创建或删除参数。单击上移和下移以对参数重新排序。

  5. 如果您不想更改函数的用法，ReSharper可以保留现有的声明，并在新的声明中调用它，从而使您可以保留现有的用法。为此，请在“ 呼叫”选择器中选择“ 通过重载方式委托” （ 有关详细信息，请参见下文 ）。

  6. 在预览字段中检查新签名。

  7. 要应用重构，请单击下一步 。

  8. 如果添加了参数，ReSharper建议使用几种方法来修复该函数的调用：您可以选择使调用的代码不可编译，对所有调用使用“空”或特定值，或者使用调用图为以下函数选择值每个特定的呼叫（ 请参阅下文以了解详细信息 ）。

  9. 如果未发现冲突，则ReSharper会立即执行重构。否则，它会提示您解决冲突 。

如果您将某些参数的类型或返回类型更改为当前名称空间中不可用的类型，或者更改为可以模棱两可地解析的类型，则ReSharper将建议为该类型指定全名。

更改签名而不更新呼叫

如果在重构向导的“ 调用”选择器中选择“ 通过重载通过委托” ，则ReSharper保留现有声明，并在新声明中调用它，从而使您不会更改现有用法。

请注意，如果您从继承层次结构修改函数，则此选项不可用。

就地执行重构

您可以通过在编辑器中修改函数的声明来更改函数的签名，然后应用快速修复程序来调用解决方案范围的重构。

要重新排序参数，请在函数声明或用法时将插入符号设置为参数，然后按Ctrl+Shift+Alt然后使用左右箭头更改参数的位置。有关更多信息，请参见重新排列代码元素 。

例如，如果您对方法中的参数重新排序，则方法签名周围会出现一个灰色边框，通知您重构可用。你可以按Alt+Enter在操作列表中找到重构：

应用快速修复后，将出现一个对话框，显示您对方法签名的更改：

您可以单击“ 下一步”在整个解决方案中应用更改。

在任何函数调用中添加一个新参数时，也可以应用“更改签名”重构。在这种情况下，ReSharper检测到错误的呼叫，将其突出显示并建议相应的快速修复：

此快速修复将在整个解决方案范围内调用重构并更新该函数的声明及其所有用法。如有必要，ReSharper将显示一个调用图以分别为每个特定调用选择值

您还可以在其他地方应用重构。

使用呼叫图更新呼叫（“推/拉参数”工具）

如果要更改函数签名以添加新参数，则ReSharper提供了几种更新函数调用的方法。除了对所有呼叫使用“空”或恒定值外，您还可以使用呼叫的可视化表示来更新每个单独的呼叫。

如果使用向导执行重构，请选择向导最后一页上的“使用调用树解析”，然后单击“ 下一步” 。如果您从更新的函数调用的快速修复中执行重构，则带有已修改函数的所有调用的工具窗口将自动打开：

在此视图中，您可以单击单击检查受影响的呼叫并在编辑器中找到它们。对于每个呼叫，选择获取新添加的参数的值的所需方式，或者选择“ 用户编辑”以手动编辑呼叫。
