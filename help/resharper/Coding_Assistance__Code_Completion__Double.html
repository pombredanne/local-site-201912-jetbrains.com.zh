ReSharper 2019.2帮助

双重完成

所有ReSharper的完成快捷方式都可以连续按几次。在这种情况下，ReSharper会向完成列表添加更多建议。

新建议以绿色垂直线标记，因此您可以轻松发现它们。如有必要，您可以通过清除选中突出显示新项目复选框来关闭此助手。 环境|智能感知的“完成外观”页面 ReSharper选项（ Alt+R,,O ）。

双重基本完成

当您调用基本完成时 Ctrl+Space两次，ReSharper向完成列表添加了许多新建议。

当前符号及其基本类型的非公共成员将添加到完成列表中。

请注意，ReSharper仍会突出显示公共成员，建议使用它，因为私有成员将生成不可编译的代码。但是，如果您决定使用私人会员，则将其公开很容易：只需按Alt+Enter选择突出显示的错误，然后选择将方法设为public quick-fix 。

如果检查对象与派生类型的兼容性，则ReSharper会将此派生类型的成员以及继承层次结构中的所有中间类型添加到完成列表中。如果选择其中之一，则ReSharper将自动将变量转换为以下类型：

您还可以投射sender事件处理程序中的参数传递给所需的父控件类型并访问其成员。

双重导入完成

调用导入符号完成时 Ctrl+Alt+Space两次，ReSharper将非引用模块的公共类型和扩展方法添加到完成列表。ReSharper在解决方案中的其他项目以及解决方案的其他项目中引用的所有已编译程序集中寻找公共类型和扩展方法。未引用的模块以红色显示：

如果在双重导入完成列表中看不到解决方案中的特定符号，则可能是由于以下原因之一：
（1）该符号不是公开的。
（2）引用声明了符号的项目将导致循环依赖。

如果您从非引用模块中选择一项，则ReSharper会自动添加缺少的引用。默认情况下，ReSharper为每个新引用显示一个确认对话框。如有必要，可以通过清除“ 环境” |“ 添加参考”之前的“ 显示确认对话框”来禁用它。智能感知 ReSharper选项的“ 完成行为”页面。

双重智能完成

调用智能完成时 Ctrl+Shift+Space两次，ReSharper添加了具有预期数据类型返回值的调用链。

你可以按Ctrl+Shift+Space多次使用更复杂的呼叫链扩展列表。您按下的次数越多，通话链就会越长。

默认情况下，ReSharper仅考虑实例方法及其返回值。如有必要，您还可以在建议列表中包括扩展方法。为此，请选择“ 环境” |“ 在双智能完成中显示扩展方法”。 智能感知 ReSharper选项的“ 完成行为”页面。
请注意，在大型解决方案上，这可能会减慢双重完成算法。
