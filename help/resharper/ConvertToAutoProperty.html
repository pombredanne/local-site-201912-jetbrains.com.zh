代码检查：将属性转换为自动属性

您可以配置此检查：

  • 禁用它以忽略所有相关问题
  • 在代码中的任何位置都禁止显示它以忽略特定问题
  • 更改其严重性级别，以使检测到的问题更少或更明显

如果get和set属性的访问者除了向后备字段返回值或设置值外什么也不做，ReSharper建议将该属性转换为自动实现的属性 （也称为自动属性），删除后备字段并将其所有用法替换为新创建的自动属性。

这是此检查建议的快速修复示例：

次优代码

快速修复后

类Shape {private Color bgColor; public Color BackgroundColor {get {return bgColor; }设置{bgColor = value; }} public Shape（Color background）{bgColor = background; }}
class Shape {public Color BackgroundColor {get;组; } public Shape（Color background）{BackgroundColor = background; }}

逆向转换可与上下文动作一起使用 Alt+Enter 。

请注意，此快速修复将调用Convert Property to Auto-Property refactoring ，它将替换整个解决方案范围内的后备字段的所有用法。在某些情况下，例如，如果在其他上下文中使用了属性名称，则可能需要解决出现的冲突 。
