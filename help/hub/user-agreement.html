Hub 2019.1帮助

用户协议

在集线器的“ 用户协议”页面上，您可以存储呈现给登录到集线器的用户的用户协议。这使您可以收集和跟踪个人对信息通知的接受，例如一组条款和条件或隐私政策。使用此功能可以通知用户您的策略，以处理在欧盟成员国开展业务的通用数据保护法规（GDPR）要求中概述的存储在中心中的个人数据。

启用用户协议

此功能的目的是为您提供一种收集和跟踪个人接受您的协议或政策以处理存储在Hub中的个人信息的方法。启用用户协议时：

  • 将向所有登录Hub的用户显示该协议。显示协议文本，如“ 用户协议”页面上的预览窗格中所示。

  • 启用协议后，首次登录时，会向用户提供接受或拒绝协议的选项。如果协议超出对话框中的可见空间量，则在用户滚动到协议的最后一行之前，不会启用接受协议的选项。

  • 接受该协议的用户被授予访问集线器的权限。版本号和接受日期记录在接受协议的每个用户的个人资料中。

  • 如果用户拒绝该协议，则该用户将从集线器中注销。要访问该服务，用户必须再次登录并接受协议。

要启用用户协议：

需要权限：低级管理员写

  1. 在“ 管理”菜单的“ 服务器设置”部分中，选择“ 用户协议” 。

  2. 在输入字段中输入或粘贴您的协议文本。

    • 您可以使用Markdown语法设置协议文本的格式 。

    • 右侧的预览窗格将显示文本，如将在协议对话框中显示的那样。

    • 请注意，协议对话框仅显示协议文本和用于接受或拒绝协议的按钮。

  3. 准备发布协议时，请单击“ 保存”按钮。

    • 用户协议将另存为新的次要版本。

  4. 如果要跟踪使用REST API访问YouTrack的用户帐户的接受程度，请启用“ 需要REST API访问”选项。

    可以独立启用和禁用要求接受协议才能使用REST API访问YouTrack的选项，而不会影响协议的版本。

  5. 单击启用按钮。

    • 用户协议已启用。

    • 协议将显示所有登录到集线器安装的用户。

    • 用户必须接受协议才能访问服务。

    • 接受日期和当前协议的版本号存储在接受协议的用户的用户配置文件中。

更新用户协议

如果您对用户协议进行的次要更新不会严重影响您先前从用户那里收集的个人信息方面的做法，则可以编辑协议文本并发布新的次要版本。不要求接受先前次要版本的用户接受新版本。仅向未接受该协议的先前发布版本的用户显示该协议的更新版本。

要发布该协议的新的次要版本：

需要权限：低级管理员写

  1. 从管理菜单中打开用户协议页面。

  2. 在输入字段中编辑协议文本。

  3. 单击保存按钮。

    • 您的用户协议将另存为新的次要版本。

    • 接受日期和更新后的协议的版本号存储在接受协议的用户的用户配置文件中。

如果您更改了处理个人信息的策略以包括以前版本中未披露的活动，则可以更新协议文本并使当前版本无效。所有用户，包括接受旧版协议的用户，均被要求接受更新的协议以使用集线器服务。

要发布该协议的新主要版本：

需要权限：低级管理员写

  1. 从管理菜单中打开用户协议页面。

  2. 在输入字段中编辑协议文本。

  3. 选择使当前协议无效 。

  4. 单击保存按钮。

  5. 确认无效当前协议中的操作？ 对话。

    • 您的用户协议将另存为新的主要版本。

    • 该协议的先前版本无效。

    • 要求所有用户在下次尝试登录集线器时接受更新的协议。

    • 接受日期和更新后的协议的版本号存储在接受协议的用户的用户配置文件中。

跟踪用户协议的更改

为了证明您在事件发生时正在按照数据处理准则进行工作，您需要跟踪用户协议的每个版本及其发布日期。虽然Hub提供了用于按用户显示协议并跟踪其接受程度的界面，但它并不打算用作版本跟踪工具。相反，Hub为您提供了下载协议文本的选项。这使您可以记录此信息的发布以及在中央存储库中跟踪其他站点的协议。

标题的“ 下载”菜单中提供以下选项：

选项

描述

。文本

将协议的当前版本下载为文本文件。该文件包含以Markdown语法格式设置的协议文本，如“ 用户协议”页面上的输入字段所示。

.html

以原始HTML下载协议的当前版本。

您还可以在Hub中跟踪已应用于用户协议的所有更改。

要跟踪对用户协议的更改：

需要权限：低级管理员读取

  1. 从管理菜单中打开用户协议页面。

  2. 单击标题中的“应用于用户协议的更改”链接。

    • 集线器将打开“ 审核事件”页面。

    • 该页面被过滤以仅显示与用户协议相关的事件。

  3. 要浏览对协议的更改，请从列表中选择一个事件，然后在“ 详细信息”侧栏中查看更改。侧边栏显示Author ， Timestamp和应用于协议的一组更改 。

  4. 要导出已应用于协议的完整更改集，请选择所有审核事件，然后单击“ 下载为.json”按钮。

    • 一个events.json文件下载到您的本地目录。

    • 该文件包含已以JSON格式应用于用户协议的所有更改的完整列表。

跟踪用户接受度

证明您在数据处理准则范围内工作的另一种方法是能够跟踪用户对协议的接受程度。在集线器中，此信息显示在每个用户帐户的配置文件中。

集线器配置文件显示最新版本的集线器中记录了以下信息：

  • 如果用户尚未接受协议，则用户协议显示当前正在接受当前版本。

  • 如果用户已接受协议的当前版本，则用户协议将显示接受的版本号以及用户接受协议的日期和时间。

  • 如果用户接受了已失效的协议版本，则用户协议将显示先前接受的版本的版本号以及日期和时间。该字段还显示正在接受当前版本。
