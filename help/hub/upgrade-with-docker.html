使用Docker升级

此页面描述了使用Hub docker映像升级现有Hub服务的方案。

升级Docker安装

  1. 现有集线器安装正在运行时， 创建集线器数据库的备份 。

  2. 停止正在运行的Hub docker容器 。

  3. 从Docker Hub存储库中提取最新Hub版本的图像：

    docker pull jetbrains /集线器：

    哪里是集线器版本的完整版本号。请参阅可用版本的完整列表。

  4. 执行以下命令以运行从存储库中提取的具有新Hub版本的容器。该命令的所有其他属性（卷和端口映射）必须与先前版本的集线器映像所使用的属性相同：

    docker run -it --name \ -v ：/ opt / hub / data \ -v ：/ opt / hub / conf \ -v ：/ opt / hub / logs \ -v ：/ opt / hub / backups \ -p ：8080 \ jetbrains / hub：

    • 在控制台中，显示基于Web的配置向导的URL。

  5. 在浏览器中打开URL。显示当前数据库中的设置：

  6. 查看设置，然后单击“ 升级”按钮。

集线器服务从更新的集线器版本开始。

升级非Docker安装

  1. 现有集线器安装正在运行时， 创建集线器数据库的备份 。

  2. 停止现有的中心服务 。

  3. 从Docker Hub存储库中提取最新Hub版本的图像：

    docker pull jetbrains /集线器：

    集线器版本的完整版本号在哪里。

  4. 在主机上创建和配置特定于集线器的目录。

  5. 将Hub数据库的备份文件复制到backups您在上一步中创建的目录。

  6. 执行以下命令以运行具有集线器服务器的容器，并映射集线器数据量和端口：

    docker run -it --name \ -v ：/ opt / hub / data \ -v ：/ opt / hub / conf \ -v ：/ opt / hub / logs \ -v ：/ opt / hub / backups \ -p ：8080 \ jetbrains / hub：

    与命令一起使用的属性：

    • -it —一个标志，它将集线器容器的输入和输出（包括启动日志）附加到终端窗口。请注意，当终端连接到容器输出时，按Ctrl + C会导致容器关闭。使用`Ctrl + PQ`可以将终端从容器输出中分离出来。有关更多详细信息，请参阅官方docker文档 。

    • --name hub-server-instance —容器的任意名称。

    • -v :/opt/hub/NNN —将主机上特定于集线器的“ NNN”目录绑定到各自/opt/hub/NNN容器内的目录。

    • -p :8080 —定义端口映射的参数。指示主机监听端口并将所有流量传播到docker容器内的端口8080。

    结果，启动集线器服务{0.0.0.0:8080}在Docker容器中，并且端口8080映射到您指定的端口。这样，可以从对主机具有网络访问权限的任何计算机上访问服务。例如，如果主机的FQDN为host.mydomain.com并且是7777，则可以在以下位置使用该服务http://host.mydomain.com:7777

  7. 集线器服务从上述地址处的基于Web的配置向导开始（在我们的示例中，它是http://host.mydomain.com:7777 ）。在浏览器中打开URL以访问向导。

  8. 在配置向导中 ，单击“ 升级” 。

  9. 在“ 选择升级源”页面上，单击“ 选择”按钮，然后选择备份文件作为升级源。

    单击下一步 。

  10. 在“ 确认设置”页面上，查看系统设置。

  11. 完成后，点击升级 。

    • 集线器启动其组件。

      设置完成之前，请勿在浏览器中关闭页面。当集线器服务器准备就绪时，您将被重定向到登录页面。

  12. 输入集线器管理员帐户的凭据，然后单击“ 登录”按钮。

    • 集线器将打开“ 用户”页面。

    您的集线器安装已升级并可以使用。
