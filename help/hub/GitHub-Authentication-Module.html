GitHub身份验证模块

该身份验证模块使用户可以使用其GitHub凭据登录到Hub。

启用GitHub身份验证

要允许具有现有GitHub帐户的用户登录到Hub，请启用GitHub身份验证模块。

此过程分为三个步骤：

  1. 在Hub中生成一个回调URL 。当您为GitHub创建身份验证模块时，Hub会生成用于此服务的回调URL。该URL标识每个GitHub登录请求的源。

  2. 在GitHub中生成客户端ID和密钥 。从Hub发送到GitHub的每个登录请求都包含一个唯一的标识符。您存储在身份验证模块中的ID和密码会告诉GitHub每个登录请求均已授权。

  3. 在Hub中启用身份验证模块 。生成Hub用来向GitHub进行身份验证的信息后，将值复制到Hub并启用该模块。

在此过程中，您将在Hub和GitHub中生成值。
每个步骤的标题告诉您要遵循的应用程序菜单。

在集线器中生成回调URL

首先，打开YouTrack并为GitHub帐户创建身份验证模块。创建身份验证模块时，YouTrack会生成用于授权服务的回调URL。

在Hub中生成回调URL：

需要权限：低级管理员写

  1. 在管理菜单的访问管理部分，选择身份验证模块 。

  2. 从“ 新建模块”下拉列表中，选择GitHub 。

    • 将打开“ 新建GitHub身份验证模块”对话框。

  3. 输入新的身份验证模块的名称和服务器URL ，然后单击创建按钮。

    • Auth Modules页面显示新GitHub身份验证模块的设置。

    • Hub会生成一个回调URL，供您在GitHub中使用。

  4. 复制授权回调URL 。如果浏览器支持该功能，请使用“ 复制”按钮将URI复制到剪贴板。

  5. 单击链接以访问GitHub中的“ 注册新的OAuth应用程序”页面。

    此设置要求您将值从GitHub复制到Hub上此页面上的输入字段中。

    为了简化设置，请在新的浏览器选项卡或窗口中打开此链接。

如果您的Hub实例的基本URL发生更改，请确保在GitHub设置中更新回调URL。例如，将数据迁移到另一个集线器服务或更改代理设置之后。

在GitHub中生成客户端ID和密钥

  1. 登录到您的GitHub帐户。

  2. 指定用于注册新OAuth应用程序的常规参数： 应用程序名称 ， 主页URL和可选的应用程序描述 。

  3. 在“ 授权”回调URL中 ，粘贴您从“集线器”中“ 身份验证模块”页面复制的回调URL。

  4. 单击注册应用程序按钮。

    • GitHub会生成设置Hub模块所需的凭据，并将其显示在页面顶部。

在集线器中启用身份验证模块

  1. 从GitHub复制客户端ID，并将其粘贴到Hub的“ 客户端ID”输入字段中。

  2. 从GitHub复制客户端密钥，并将其粘贴到Hub的“ 客户端密钥”输入字段中。

  3. 配置身份验证模块的可选设置。有关更多信息，请参阅设置 。

  4. 单击启用模块按钮。

    • GitHub身份验证模块已启用。

    • 按钮图像设置中存储的图标将添加到登录对话框窗口中。用户可以单击此图标以使用其GitHub帐户进行身份验证。

设定值

领域

描述

类型

显示在集线器中启用了第三方身份验证的应用程序或服务的名称。

名称

存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。该名称还显示在登录表单上的第三方服务图标的工具提示中。

按钮图片

显示用于用户单击以使用GitHub帐户登录到Hub的按钮的图像。

授权回调网址

显示用于在GitHub中注册到Hub的连接的Authorization回调URL。

客户编号

存储GitHub用于验证登录请求的标识符。当您为应用程序配置授权设置并输入回调URL时，便会在GitHub中生成此值。

客户机密

存储用于验证客户端ID的机密或密码。您将在GitHub中生成此值以及客户端ID。

服务器网址

当用户使用GitHub帐户登录时，显示集线器向其发送登录请求的服务器的URL。

范围

显示访问请求的范围。
此字段下方显示的信息可帮助您配置身份验证模块。

允许的组织

将此身份验证模块的登录名限制为属于GitHub中特定组织的用户。输入以逗号分隔的组织列表。如果为空，则具有GitHub帐户的任何用户都可以使用此身份验证模块登录。

其他设置

以下选项位于页面底部。使用这些设置可以管理集线器帐户的创建和组成员身份，并减少空闲连接消耗的处理资源的损失。

选项

描述

用户创建

为使用存储在连接的授权服务中的帐户登录的未注册用户启用集线器帐户的创建。集线器使用电子邮件地址来确定用户是否具有现有帐户。

自动加入群组

当用户使用存储在连接的授权服务中的帐户登录时，将用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。

连接超时

设置等待建立与授权服务的连接的时间。默认设置为5000毫秒（5秒）。

读取超时

设置等待从授权服务读取和检索用户配置文件数据的时间。默认设置为5000毫秒（5秒）。

审计

链接到中心中的“ 审核事件”页面。在这里，您可以查看已应用于此身份验证模块的更改列表。
