Azure AD 2.0身份验证模块

Azure AD 2.0身份验证模块是一个预配置的OAuth 2.0身份验证模块，该模块允许用户使用其Azure AD 2.0凭据登录集线器和所有连接的服务。

自定义租户设置

当您遵循此设置时，登录名仅限于其帐户存储在Azure AD实例中的用户。如果要接受来自任何Azure Active Directory（AD）租户的登录请求，请按照有关公共租户的说明进行操作。

为自定义租户创建Azure AD 2.0模块时，在创建auth模块时，将提供自定义租户的目录ID。这会自动将您的Azure租户ID添加到auth模块使用的授权和令牌终结点中。

获取自定义租户的目录ID：

  1. 访问Microsoft Azure门户并创建目录。

  2. 在边栏中，选择“ Azure Active Directory” 。

  3. 在Azure Active Directory菜单的“ 管理”部分中，选择“ 属性” 。

  4. 复制存储为目录ID的值 。

有了自定义租户的目录ID之后，就可以在Hub中创建Azure AD 2.0身份验证模块。

若要创建Azure AD 2.0模块，请执行以下操作：

需要权限：低级管理员写

  1. 在管理菜单的访问管理部分，选择身份验证模块 。

  2. 从“ 新模块”下拉列表中，选择“ Azure AD 2.0” 。

    • 将打开“ 新建Azure AD 2.0模块”对话框。

  3. 将目录ID从Microsoft Azure粘贴到“ 租户”输入字段中。

  4. 单击对话框中的创建按钮。

    • “身份验证模块”页面显示新的Azure AD 2.0身份验证模块的设置。

    • 集线器将生成一个重定向URI，供您在授权服务中使用。

  5. 如果浏览器支持该功能，请使用“ 复制”按钮将重定向URI复制到剪贴板。

下一步是在Microsoft Azure中注册中心的授权重定向URI。这使您可以访问启用Azure AD 2.0身份验证模块所需的客户端ID和密码。

从您的Microsoft Azure门户开始，然后根据指示切换回中心。

若要获得与Azure AD 2.0一起使用的客户端ID和密钥，请执行以下操作：

  1. 在Azure Active Directory菜单的“ 管理”部分中，选择“ 应用程序注册” 。

  2. 单击标题中的“ 新应用程序注册”按钮。

    • 在名称字段中输入应用程序的名称 。

    • 对于“ 应用程序类型” ，保留“ Web应用程序/ API”选择。

    • 对于登录URL ，输入集线器安装的基本URL。

    • 单击创建按钮。

  3. 复制应用程序ID并将其粘贴到集线器身份验证模块的“ 客户端ID”输入字段中。

  4. 单击工具栏中的“设置”按钮，然后在“设置 ”边栏选项卡的“ API访问”部分中选择“ 密钥 ” 。

  5. 在“ 密钥”刀片中生成密码：

    • 在描述字段中，输入一个任意值。

    • 从“ 过期”下拉列表中选择一个持续时间。

    • 单击标题中的“ 保存”按钮。

  6. 复制密钥值，关闭“ 密钥”刀片，然后切换回集线器。

  7. 将密钥粘贴到集线器身份验证模块的“ 客户端机密”输入字段中，然后单击“ 保存”按钮。

  8. 从集线器身份验证模块复制重定向URI ，然后切换回Microsoft Azure门户。

  9. 在“ 设置”边栏的“ 常规”部分中，选择“ 回复URL” 。

  10. 删除集线器安装的基本URL。

  11. 将来自集线器的重定向URI粘贴到“ 回复URL”刀片中的输入字段中，然后单击标题中的“ 保存”按钮。

  12. 切换回集线器并配置身份验证模块的可选设置。有关更多信息，请参见其他设置 。

  13. 单击标题中的“ 启用模块”按钮。

    • 启用了Azure AD 2.0身份验证模块。

    • 按钮图像设置中存储的图标将添加到登录对话框窗口中。用户可以单击此图标以其Microsoft帐户登录到Hub。

普通租户设置

将终结点用于普通租户的Azure AD 2.0模块遵循OAuth 2.0身份验证模块的标准设置。此安装程序接受来自任何Azure Active Directory（AD）租户的登录请求，包括非组织成员的用户。如果要将登录限制为在公司的Azure AD实例中注册的用户，请按照自定义租户的说明进行操作。

第一步是创建一个auth模块，并在Hub中生成一个重定向URI。

要在Hub中生成重定向URI，请执行以下操作：

需要权限：低级管理员写

  1. 在管理菜单的访问管理部分，选择身份验证模块 。

  2. 从“ 新建模块”下拉列表中，选择“ Azure AD 2.0” 。

    • 将打开“ 新建Azure AD 2.0模块”对话框。

  3. 单击对话框中的创建按钮。

    • “身份验证模块”页面显示新的Azure AD 2.0身份验证模块的设置。

    • 集线器将生成一个重定向URI，供您在授权服务中使用。

  4. 如果浏览器支持该功能，请使用“ 复制”按钮将重定向URI复制到剪贴板。

更改集线器安装的基本URL时，请确保在授权服务中更新“重定向URI”。例如，将数据迁移到另一个集线器服务或更改代理设置之后。

下一步是在Microsoft应用程序注册门户中注册中心的授权重定向URI。这使您可以访问启用Azure AD 2.0身份验证模块所需的客户端ID和密码。

若要获得与Azure AD 2.0一起使用的客户端ID和密钥，请执行以下操作：

  1. 访问Microsoft应用程序控制台 。您还可以从当前版本的Microsoft Azure门户访问此页面（在Azure Active Directory菜单的“ 管理”部分中选择“ 应用程序注册” ，然后单击“ Microsoft应用程序控制台”链接）。

  2. 在“ 我的应用程序”页面上，单击“ 添加应用程序”按钮。

  3. 输入名称，然后单击创建按钮。

  4. 在“ 平台”部分中，单击“ 添加平台”按钮，然后选择“ Web”选项。

  5. 将来自集线器的重定向URI粘贴到“ 重定向URI”输入字段中，然后单击“ 保存”按钮。

  6. 将存储为应用程序ID的值用于集线器中的客户端ID设置。

  7. 使用“ 生成新密码”选项为集线器中的“ 客户端机密”设置生成一个值。

现在您已经具有连接授权服务所需的值，您可以启用Azure AD 2.0身份验证模块。

  1. 从Microsoft应用程序注册门户复制应用程序ID ，然后将其粘贴到集线器的“ 客户端ID”输入字段中。

  2. 单击Microsoft应用程序注册门户的“ 应用程序 密码 ”部分中的“ 生成新密码”按钮，复制该值，然后将其粘贴到集线器中的“ 客户端密码”输入字段中。

  3. 配置身份验证模块的可选设置。有关更多信息，请参见其他设置 。

  4. 单击保存按钮以应用设置。

  5. 单击启用模块按钮。

    • 启用了Azure AD 2.0身份验证模块。

    • 按钮图像设置中存储的图标将添加到登录对话框窗口中。用户可以单击此图标以其Microsoft帐户登录到Hub。

设定值

设置页面的第一部分显示身份验证模块的常规设置。在这里，您还可以找到用于在授权服务中注册集线器的重定向URI，以及存储在授权服务中生成的客户端ID和客户端密钥的输入字段。

设置

描述

类型

显示为集线器中的第三方身份验证启用的授权服务的类型。

名称

存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。

按钮图片

显示用于用户单击以其连接的授权服务中的帐户登录集线器的按钮的图像。您可以上传JPG，GIF或PNG文件。图像会自动调整为48 x 48像素。

重定向URI

显示用于在授权服务中注册到集线器的连接的授权重定向URI。

客户编号

存储授权服务用来验证登录请求的标识符。当您配置Web应用程序的授权设置并输入授权的重定向URI时，会在授权服务中生成此值。

客户机密

存储用于验证客户端ID的机密或密码。您在授权服务中生成此值以及客户端ID。

授权服务端点

页面此部分中的设置存储Azure AD 2.0使用的OAuth 2.0终结点。

对于预配置的OAuth 2.0模块，将自动设置所选授权服务使用的值。

设置

描述

授权书

存储集线器用来通过用户代理重定向从资源所有者获取授权的端点。

代币

存储集线器用来交换访问令牌的授权授予的端点。

用户数据

存储用于定位已认证用户的概要文件数据的端点。

电子邮件

用于查找经过身份验证的用户的电子邮件地址的端点仅在电子邮件地址未存储在用户配置文件中时使用

默认电子邮件验证状态

确定当身份验证服务未返回电子邮件地址的验证状态时，应为集线器中的电子邮件地址设置哪个状态。

场图

Azure AD 2.0返回用户配置文件响应对象时，来自指定字段路径的值将复制到集线器中的用户配置文件。使用以下设置来定义终结点，该终结点可以找到已认证用户的配置文件数据，并将存储在授权服务中的字段映射到集线器中的用户帐户。

对于Azure AD 2.0模块，将自动设置值。

使用以斜杠（/）分隔的一系列路径段来指定嵌套对象内部字段的路径。

其他设置使您可以定义请求范围，并选择如何通过服务进行身份验证。

领域

描述

用户身份

映射到存储值的字段，该值将复制到Hub中的User ID属性。

电子邮件

映射到存储要复制到集线器配置文件中“ 电子邮件”字段的值的字段。

验证邮件

映射到存储要复制到Hub中已验证电子邮件属性的值的字段。

全名

映射到存储要复制到集线器配置文件中的全名字段的值的字段。

头像

映射到存储图像的字段，以用作集线器配置文件中的头像 。

图片网址格式

为ID引用的头像生成图像URL。使用占位符引用存储化身的字段。

范围

设置访问请求的范围。输入范围列表，以空格分隔。

认证方式

确定如何将凭据传递到授权服务。

其他设置

以下选项位于页面底部。使用这些设置可以管理集线器帐户的创建和组成员身份，并减少空闲连接消耗的处理资源的损失。

选项

描述

用户创建

为使用存储在连接的授权服务中的帐户登录的未注册用户启用集线器帐户的创建。集线器使用电子邮件地址来确定用户是否具有现有帐户。

自动加入群组

当用户使用存储在连接的授权服务中的帐户登录时，将用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。

连接超时

设置等待建立与授权服务的连接的时间。默认设置为5000毫秒（5秒）。

读取超时

设置等待从授权服务读取和检索用户配置文件数据的时间。默认设置为5000毫秒（5秒）。

审计

链接到中心中的“ 审核事件”页面。在这里，您可以查看已应用于此身份验证模块的更改列表。
