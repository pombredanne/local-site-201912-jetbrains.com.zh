获取角色的所有权限

请求

GET /roles/{role id}/permissions?fields=string&$skip=int&$top=int&query=string&orderBy=string

参数

描述

fields

可选的 。仅返回每个权限的指定字段子集。使用字段语法定义子集。

$skip

可选的 。可让您设置一些要返回的许可权，然后再返回第一个。

$top

可选的 。使您可以设置要返回的最大权限数。

query

可选的 。用于指定搜索查询以过滤返回的结果。有关支持的逻辑操作语法的定义，请参见查询语法。对于权限，支持以下字段和元组：

EntityType ： PermissionEntityType —给定entityType的权限

id ： value —具有给定ID的权限

is ： global-所有全局权限

key ： PermissionKey —具有给定密钥的权限

名称 ：permissionName -与给定名称权限

operation ： permissionOperation —给定操作的权限

角色 ： 角色 —给定角色的权限

service ： service —给定服务提供的权限

orderBy

可选的 。返回按指定字段排序的权限列表。有关详细信息，请参见排序语法 。权限支持按以下字段排序： EntityType ， 键 ， 名称 ， 操作

响应

回应码

含义

200 OK

成功的请求。

400错误的要求

至少一个请求参数无效。例如，缺少传递的JSON对象中的必填字段。有关详细信息，请检查响应中的错误消息。

403禁止

请求者无权访问所请求的资源。

找不到404

找不到请求的资源。

500内部服务器错误

由于服务器错误，无法处理请求。有关详细信息，请检查响应中的错误消息。

反应体

{ "skip": int, "top": int, "total": int, "permissions": [permission, ...] }
