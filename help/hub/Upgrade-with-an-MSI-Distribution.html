Hub 2019.1帮助

通过MSI发行版升级

您可以使用MSI分发来升级任何类型的现有集线器安装。

以下软件与Hub MSI发行版捆绑在一起：

  • Jetty Servlet引擎和HTTP服务器

  • Java SE运行时环境8 x64

MSI发行版不包括JRE x86。如果要在x86环境中安装Hub，则需要下载并安装与您的操作系统兼容的Java Runtime Environment。

MSI安装程序需要.NET Framework 2.0或更高版本。

先决条件

升级之前，请执行以下先决任务：

  • 创建当前数据库的备份 。集线器不提供前向数据库兼容性。在升级过程中，数据库将迁移为与最新版本兼容的格式。这意味着您无法还原到以前的版本，而无法继续使用升级期间处理的数据库。

  • 将当前数据库的备份复制到安全位置，以防万一升级遇到问题并需要回滚到以前的版本。

  • 如果要从1.0.663之前的Hub版本的MSI安装升级，请检查您当前的位置data和backups目录。默认情况下，不会在安装目录中创建这些文件夹。但是，在安装过程中可以在基于Web的配置向导中更改默认位置。

    Hub低于1.0.663的版本中内置的卸载程序将删除整个安装目录。如果你的data和backups文件夹已移至安装目录，卸载程序将其删除，升级失败。为避免此问题：

    1. 找到当前集线器安装的安装目录。MSI安装的默认安装目录是C:\Hub 。在任何情况下，当前安装目录都将存储为物业%programdata%\JetBrains\Hub\conf\internal\launcher.service.settings.xml文件。

    2. 如果安装目录包含data和backups子目录，创建当前安装的备份并将备份移动到安装目录之外的位置。

    3. 遵循配置向导中的“ 升级”路径，卸载当前的集线器安装并安装最新版本，选择当前备份作为升级源。

升级MSI安装

如果以前使用MSI发行版安装了Hub服务，则可以使用MSI发行版升级到最新版本的Hub。

Windows Installer软件包自动处理以下所有升级过程：

  • 停止当前正在运行的服务。

  • 卸载以前的版本。

  • 安装新版本。

  • 重新启动服务。

要升级现有的MSI安装：

  1. 从JetBrains网站下载MSI发行版。

  2. 打开Windows Installer程序包（MSI文件）。

    • 将打开“ JetBrains集线器设置向导” 。

  3. 请按照安装向导中的说明进行操作。

    • 设置向导完成后，基于Web的配置向导将在默认浏览器中打开。

      如果主机上未安装Web浏览器，请从设置向导复制URL，然后在具有Web浏览器的工作站上打开页面。

  4. 单击完成按钮以关闭安装向导。

  5. 切换到Web浏览器，然后在基于Web的配置向导中继续进行升级安装。配置向导为您提供以下选项以继续升级：

    • 使用先前安装中检测到的设置进行升级。

    • 选择其他升级源。当您在其他位置的中心数据库的另一个备份或想将另一个中心安装的位置用作升级源时，请使用此选项。

      如果选择其他升级源，请验证服务用户帐户JetBrainsHub对向导中显示的所有目录具有完全访问权限。此帐户由安装程序创建以运行Windows服务。如果继续操作而未授予对JetBrainsHub帐户的访问权限，则会遇到错误。

    • 丢弃检测到的设置并执行全新安装。

    选择要用于升级的设置后，请验证基本系统设置：

    设置

    描述

    基本网址

    最终用户访问集线器的URL。

    应用监听端口

    安装后，集线器服务器侦听的端口。

    申请监听地址

    安装后，集线器侦听的IP地址。

    数据目录位置

    集线器存储内容数据的目录。

    备份位置

    集线器存储数据库备份的目录。

    日志位置

    集线器存储日志文件的目录。

    临时位置

    集线器存储特定于产品的临时文件的目录。

    要更改任何这些文件类型的位置，请单击“ 更改”按钮，然后选择一个新位置。

  6. 完成后，点击升级 。

    • 该配置将应用于您的集线器服务器。

    设置完成之前，请勿在浏览器中关闭页面。当集线器服务器准备就绪时，您将被重定向到登录页面。

  7. 输入管理员帐户的凭据，然后单击“ 登录”按钮。

    • 集线器将打开“ 用户”页面。

    而已。您的Hub实例已升级并可以使用。

使用MSI发行版升级ZIP安装

您可以升级通过ZIP发行版安装的现有安装，并同时将安装类型更改为MSI。

升级之前，请创建当前数据库的备份 。

此过程中的所有命令均适用于Linux和macOS。如果要在Windows服务器上升级集线器，请更换.sh与.bat并使用反斜杠。

要使用MSI发行版升级ZIP安装，请执行以下操作：

  1. 下载MSI发行版 。

  2. 停止集线器服务。有关详细信息，请参阅启动和停止Hub MSI 。

  3. 使用操作系统工具来验证由Hub启动的所有Java进程都不再运行。如果仍有与Hub相关的Java进程仍在运行，请终止未完成的进程树。

  4. 打开Windows Installer程序包（MSI文件）。

    • 将打开“ JetBrains集线器设置向导” 。

  5. 如果安装向导从先前安装的版本中检测到数据和配置文件，请选择推荐的选项以执行全新安装。

    当安装了较早版本的MSI发行版本并随后在服务器上将其卸载时，安装向导将显示此选项。

    设置向导完成后，基于Web的配置向导将在默认浏览器中打开。

    如果主机上未安装Web浏览器，请在另一个工作站上的Web浏览器中打开安装向导提供的URL。

  6. 在配置向导中 ，单击“ 升级” 。

  7. 在“ 选择升级源”页面上，单击“ 选择”按钮，然后选择一个升级源。

    • 选择以前的ZIP安装位置作为升级源。
      要么

    • 选择数据库备份副本以ZIP文件或解压缩备份的存储位置。

  8. 验证服务用户帐户JetBrainsHub对向导中显示的所有目录具有完全访问权限。此帐户由安装程序创建以运行Windows服务。如果继续操作而未授予对JetBrainsHub帐户的访问权限，则会遇到错误。

  9. 完成后，单击“ 下一步” 。

  10. 按照“ MSI安装”中所述，从升级过程的第6步继续。

将Docker安装升级到MSI发行版

您可以升级在Docker容器中运行的现有安装，然后同时将安装类型更改为MSI。

要将Docker映像升级到MSI发行版，请执行以下操作：

  1. 现有集线器安装正在运行时， 创建集线器数据库的备份 。

  2. 从JetBrains网站下载MSI发行版。

  3. 停止Hub docker容器 。

  4. 打开Windows Installer程序包（MSI文件）。

    • 将打开“ JetBrains集线器设置向导” 。

  5. 设置向导完成后，基于Web的配置向导将在默认浏览器中打开。

    如果主机上未安装Web浏览器，请在另一个工作站上的Web浏览器中打开安装向导提供的URL。

  6. 在配置向导中 ，单击“ 升级” 。

  7. 在“ 选择升级源”页面上，单击“ 选择”按钮，然后选择您创建的备份文件作为升级源。

  8. 验证服务用户帐户JetBrainsHub对向导中显示的所有目录具有完全访问权限。此帐户由安装程序创建以运行Windows服务。如果继续操作而未授予对JetBrainsHub帐户的访问权限，则会遇到错误。

  9. 完成后，单击“ 下一步” 。

  10. 按照“ MSI安装”中所述，从升级过程的第6步继续。

故障排除

如果遇到问题并且安装失败，请查看是否符合以下条件之一。

条件

升级到Hub 2018.3失败。集线器服务未启动。您现有安装的数据库未升级到最新版本。安装程序会回滚到以前安装的版本。

原因

不确定。

解

  1. 运行启用了详细日志记录的MSI安装程序。在“ 命令提示符”窗口中，输入以下命令：

    msiexec /i hub-xxxxx.msi /l*v install.txt

    安装程序将详细的日志数据写入一个名为install.txt并将其保存在当前目录中。

  2. 查看内容install.txt日志文件。故障报告以字符串开头Return value 3 。

  3. 如果日志文件指示服务启动失败，请检查集线器日志文件中是否有错误。集线器日志文件的默认位置是%programdata%\JetBrains\Hub\logs

  4. 如果您可以自己确定并解决问题，请重复升级过程。否则，请在Hub项目中创建一个问题 。

条件

MSI安装程序已成功将安装升级到新版本，但是配置向导无法打开或报告错误。集线器服务无法启动。

原因

  • 指定的端口被另一服务占用。

  • 您没有访问权限来更新安装目录中的文件和文件夹。

解

  1. 检查日志文件，特别是service.log ，以了解错误。日志文件的默认位置是%programdata%\JetBrains\Hub\logs

  2. 如果您可以识别并解决问题（例如，更改端口号或获得对安装目录的访问权限），请按照以下步骤重复升级过程。

    • 执行新版本的全新安装。

    • 在“ 升级源”页面上，从以前的版本中选择数据库的备份副本。

    否则，请在Hub项目中创建一个问题 。

条件

升级失败。集线器已安装。显示先前安装的产品代码。先前的安装无法自动升级。

原因

集线器先前已安装了另一个管理员帐户。

解

  1. 登录到先前版本的Hub并创建数据库的备份。

  2. 要求执行安装的管理员卸载以前的版本，或者重置管理员帐户凭据并自己卸载以前的版本。

  3. 全新安装Hub 2018.3。选择选项以删除先前的配置。

  4. 在配置向导中，选择先前版本的备份作为升级源。

报告升级安装问题

如果继续遇到升级安装问题：

  1. 在Hub项目中创建问题 。

  2. 附加以下文件，以帮助我们调查和解决问题：

    • 压缩内容logs目录并附加ZIP存档。该目录的默认位置是%programdata%\JetBrains\Hub\logs 。

      如果此目录的位置在“配置向导”中已更改，则可以在logs-dir的属性%programdata%\JetBrains\Hub\conf\internal\bundle.properties文件。

    • 压缩内容conf目录并附加ZIP存档。该目录的默认位置是%programdata%\JetBrains\Hub\conf 。

    • 附上install.txt文件。

我们的客户支持工程师将调查问题并与您一起解决问题。
