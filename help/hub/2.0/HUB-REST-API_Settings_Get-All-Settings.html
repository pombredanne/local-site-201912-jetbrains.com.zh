Get All Settings

Request

GET /settings?fields=string&$skip=int&$top=int&query=string
Parameter Description
fields Optional query parameter. Specifies a subset of the fields to be returned. Use Fields Syntax to define the subset.
$skip Optional query parameter. Number of settings to skip before returning the first entry in the page.
$top Optional query parameter. Maximum number of settings to return in the page.
query

Filter query. See Query Syntax for the definition of supported logical operations syntax. Following fields and tuples are supported for settings:

id: value — Settings by given id

type: license — Get license settings

type: public — Get public settings

type: internal — Get internal settings

type: smtp — Get smtp settings

type: jabber — Get jabber settings

type: backup — Get jabber settings

Response

Response Code Meaning
200 OK Successful request.
400 Bad Request Some of the parameters of the request are invalid (for example, required fields in the passed JSON object are missing). The response contains message describing the error.
403 Forbidden Requester has no access to the request.
404 Not Found Requested resource wasn't found.
500 Internal Server Error Failed to process request because of the server error. The response contains message describing the error.

Response body

page

{ "skip": int, /* Requested number of settings to skip before returning the first entry in the page */ "top": int, /* Requested maximum number of settings to return in the page */ "total": int, /* Total number of settings. Returned only if the total field is included as a value of the fields parameter in the query */ "settings": [settings, ...] }
