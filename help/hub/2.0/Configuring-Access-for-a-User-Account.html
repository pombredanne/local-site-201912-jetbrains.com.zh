Configure Access for a User

On this page:

  • View Access Permissions
  • Grant a Role to a User Account
  • Grant a Permission to a User Account

In Hub, access permissions for a user account are granted with roles.

There are distinctions between the permissions that allow users to perform specific actions in Hub and permissions that allow operations in a connected service. Permissions are usually granted within a project, and provide access to specific resources that are connected to the project. For example, if you have a project in Hub that uses a YouTrack project and a TeamCity project as resources, you have YouTrack-specific permissions that apply only to operations in the YouTrack project and TeamCity-specific permissions that apply to the TeamCity project. In this case, users can have different View Project and Update Project permissions for each connected service.

You can grant permissions to a user account individually. Users also inherit the permissions that are granted to any group to which they belong as a member.

View Access Permissions

The access permissions for a user account are displayed on the Access tab of the account profile. Here, you can view which roles and permissions are assigned to the account.

To view the access rights for a user account:

  1. Open the Access tab of the account profile.
  2. Click one of the following buttons to see the access rights that are available to the user:
    • Access rights by project — displays a list of the projects for which the user account has access.
    • Roles — displays a list of roles that are granted to the user account. Select the role to see the project in which the role is granted and how the role is applied. For example, the user is a member of the New Users group and inherits the role that is assigned to this group.
    • Permissions — displays a list of permissions that are granted to the user account. Select a permission to view the project in which the permission is granted.

Grant a Role to a User Account

If you do not want to assign a role to all members of a group, you can assign the role to a user account directly.

To assign an individual role to a user:

  1. Click the Users link in the main navigation.
  2. Use the search box to find the desired user account.
  3. Select a user from the list.
  4. Select the Access tab.
  5. Click the Grant Access button.
  6. In the Grant Access dialog, choose a role to assign to the user.
  7. Select one or more projects where you want to apply the role.
  8. Select an option that determines how the role is assigned (for example, by creating a group and adding the current account to it, or by granting the role to the project).
  9. Click the Grant Access button.

Grant a Permission to a User Account

If you do not want to grant a permission to all members of a group, you can grant the permission to a user account directly.

To grant a permission to an account:

  1. Click the Users link in the main navigation.
  2. Use the search box to find the desired user account.
  3. Select a user from the list.
  4. Select the Access tab.
  5. Click the Grant Access button.
  6. In the Grant Access dialog, click the Permission button.
  7. Select a permission to grant to the account from the drop-down list. To narrow down the list of permissions, start typing in the input field.
  8. Select a project where you want to apply the permission.
  9. Select an option that determines how the permission is assigned (for example, by creating a group and adding the current account to it, or by granting the user a role in the project).
  10. Click the Grant Access button.
