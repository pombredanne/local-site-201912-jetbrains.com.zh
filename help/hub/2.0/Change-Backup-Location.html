Change Backup Location

If you want to store backups of your Hub database in a new location, you can change the backup location. The following options are available:

  • Change the Backup Location Setting
  • Set the Backup Location from the Command Line

Change the Backup Location Setting

To change the backup location:

  1. From the More Settings drop-down menu, select Backup.
  2. Enter the path to the new backup location in the Backup Location input field.
  3. Click the Save button.

Set the Backup Location from the Command Line

You can use the following procedure to set the backup location for both ZIP and MSI distributions from the command line. To execute these commands in an MSI distribution, open the Command Prompt window as an administrator.

To change the backup location:

The commands in this procedure are for Linux and OS X. For a Windows installation, replace .sh with .bat and use the backslash.

  1. Stop the Hub service. For specific instructions, see Stop and Start the Hub Service.
  2. Enter the following configure command to set the location of the new data directory:
    <hub_home>bin/hub.sh configure --backups-dir=<new Hub backup directory>
    • For a ZIP installation, the <hub_home> directory is the location where the ZIP distribution was unpacked during installation.
    • For an MSI installation, the <hub_home> directory is the installation directory.
  3. Start the Hub service. For specific instructions, see Stop and Start the Hub Service.
