Parameters of Hub as the Identity Provider

To configure Hub as the Identity Provider for your service, you need to know certain parameters of your Hub server. You can find all the required parameters in the More Settings > SAML 2.0 > Settings tab.

ParameterDescription
Sign In URLIdentity Provider URL
Sign Out URLThis is the URL to which the Service Provider sends the sign out request. Also, to this URL the Service Provider redirects your users after they sign out.
Metadata URLThe SAML 2.0 metadata file contains information about Hub as the Identity Provider.
Entity IDThe entityID is the unique identifier of Hub as the Identity Provider.
Default Name ID DefaultNameIDFormat configuration parameter of a SAML federation: The format of the default nameID returned by Hub when a service provider requests an unspecified name identifier format.

In SAML, a service provider indicates the name identifier format it wants to receive from Hub. If the SP uses unspecified format for the nameID (urn:oasis:names:tc:SAML:1.1:nameidformat:unspecified) in the sign in request to Hub, then Hub returns the nameID in the format defined by this parameter.

For the description of the standard name identifier formats, refer to the Section 8.3 of the Assertions and Protocols for the OASIS SAML2.0 standard.

SSL KeySSL Key to encrypt interaction between Hub and service providers. When the SSL Key is selected, the SHA-256, SHA-1 and MD-5 Fingerprints of the certificate packed into the key store are displayed.
AuditReview a list of changes related to the SAML 2.0 management.
