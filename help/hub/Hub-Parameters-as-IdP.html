SAML2.0 Identity Provider Parameters

To configure Hub as the Identity Provider for your service, you need to know certain parameters of your Hub server. You can find all the required parameters on the More Settings > SAML 2.0 > Settings tab.

Parameter

Description

Sign in URL

Identity Provider URL

Sign out URL

This is the URL to which the Service Provider sends the sign out request. Also, to this URL the Service Provider redirects your users after they sign out.

Metadata URL

The SAML 2.0 metadata file contains information about Hub as the Identity Provider.

IdP entity ID

The entityID is the unique identifier of Hub as the Identity Provider.

Default name ID

DefaultNameIDFormat configuration parameter of a SAML federation: The format of the default nameID returned by Hub when a service provider requests an unspecified name identifier format.

In SAML, a service provider indicates the name identifier format it wants to receive from Hub. If the SP uses unspecified format for the nameID (urn:oasis:names:tc:SAML:1.1:nameidformat:unspecified) in the sign in request to Hub, then Hub returns the nameID in the format defined by this parameter.

For the description of the standard name identifier formats, refer to the Section 8.3 of the Assertions and Protocols for the OASIS SAML2.0 standard.

SSL key

The SSL key that encrypts the connection between Hub and service providers. When an SSL key is selected, the SHA-256, SHA-1 and MD-5 Fingerprints of the certificate packed into the keystore are displayed.
