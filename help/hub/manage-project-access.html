管理项目访问

在“ 编辑项目”页面的“ 访问”选项卡上，您可以查看哪些角色被授予了项目中的用户和组。这包括分配给项目团队成员的角色，以及直接分配给其他用户和组的角色。使用此信息来确定如何授予特定用户对项目的访问权限，并查看他们有权执行的操作。

此选项卡上有两个视图。默认视图显示分配给用户或组，按照用户或组中列出角色的列表。

  • 第一部分显示直接分配给用户和组的角色。

  • 第二部分显示了作为项目团队成员分配给用户和组的角色。

  • 最后一部分显示在全局项目中分配给用户和组的角色。这些分配授予所有项目中每个角色定义的访问权限。

您还可以选择按角色查看列表。

该信息分为“ 用户”或“组”查看时显示的相同部分。

在两个视图中都可以执行以下操作：

  • 使用搜索框过滤列表。您可以使用一系列过滤条件，包括用户，组，角色和权限。输入格式的搜索条件field: value 。

    例如，如果要查看哪些用户有权更新项目设置，请输入permission: (Update Project)并应用过滤器。

  • 单击详细信息按钮以显示或隐藏侧栏。边栏显示分配给角色的一组权限。权限按服务分组。

授予用户访问当前项目的权限：

需要权限：读取用户基本，读取角色，更新项目，读取组

  1. 单击授予角色按钮。

    • “ 授予角色”对话框打开。

  2. 从“ 角色”下拉列表中，选择要在项目中授予的角色。

  3. 在“ 向用户或组输入”字段中，选择一个或多个用户或组。

  4. 单击授予角色按钮。

    • 根据项目中所选角色的定义，授予所选用户和组访问权限。

“ 访问”选项卡显示了授予用户访问项目的所有可能方式。当您想撤消访问权限时，您使用的方法取决于授予访问权限的方式。

撤销直接授予用户或组的访问权限：

需要权限：读取用户基本，读取角色，更新项目，读取组

  1. 从列表中选择一个或多个用户和/或组。

  2. 单击撤消角色按钮。

  3. 在确认对话框中确认操作。

    • 该角色将从选定的用户和组中撤消。

撤消项目团队成员的访问权限：

需要权限：读取用户基本，读取角色，更新项目，读取组

  1. 从列表中选择一个或多个用户和/或组。

  2. 单击从团队中删除按钮。

  3. 在确认对话框中确认操作。

    • 选定的用户和/或组将从项目团队中删除。

如果要撤消具有Global项目角色的用户和组的访问权限，则可以选择以下几种方法：

  • 打开用户或组的配置文件页面，然后从“ 角色”选项卡撤消访问权限。

  • 打开全局项目，然后从“ 访问”选项卡上的用户或组撤消访问。
