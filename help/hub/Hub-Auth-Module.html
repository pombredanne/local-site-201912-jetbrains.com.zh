集线器身份验证模块

集线器身份验证模块是用于管理用户帐户凭据的默认授权方案。该模块内置在集线器安装中，并且默认情况下启用。

通过此身份验证模块的设置，您可以控制用户如何管理其集线器凭据。

通用设置

安装集线器时，将自动配置设置。

设置

描述

类型

显示在集线器中启用身份验证的服务类型。内置授权意味着该模块是集线器的一部分，不会单独安装。

名称

存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。

密码恢复

启用用户重置忘记密码的选项。启用后，集线器会发送一封电子邮件，其中包含一个链接，可将其密码恢复给要求重设密码的用户。

密码强度

存储应用于集线器帐户的密码策略设置。有关密码强度和安全性的更多信息，请参阅“ 设置密码策略” 。

注册

为在Hub中注册自己的用户创建帐户。如果禁用此选项，则只有具有现有集线器帐户的用户才能登录。
启用后，将显示可用于管理在Hub中注册其自己帐户的用户的其他设置。

有关这些设置的详细说明，请参见注册设置 。

审计

链接到中心中的“ 审核事件”页面。在这里，您可以查看已应用于此身份验证模块的更改列表。

注册设置

启用“ 注册”选项后，可以使用其他设置来管理在集线器中注册自己的帐户的用户。

设置

描述

自动加入群组

当他们在Hub中注册自己的帐户时，自动将新用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。不要使用此选项将用户添加到授予不必要的敏感数据访问权限的组。

验证码

在注册页面上启用reCAPTCHA。此功能可保护您的安装不受注册机器人的攻击。

启用此选项后，您需要在Google reCAPTCHA密钥管理控制台中注册中心。存储站点密钥和秘密密钥的输入字段位于“ 身份验证模块”>“通用设置”页面上。要了解如何配置reCAPTCHA进行自我注册，请参阅CAPTCHA设置 。

动作

标头中提供以下操作：

行动

描述

需要更改密码

注册一个请求，以重置所有使用其集线器凭据登录的用户的密码。有关更多信息，请参阅要求中心登录需要新密码 。

禁用模块

禁用集线器身份验证模块。禁用内置授权服务时，用户必须使用来自第三方服务的凭据登录。

测试登录

用于输入登录名和密码以测试集线器身份验证服务。

删除

从集线器中删除身份验证模块。仅在配置其他允许用户登录集线器安装的身份验证模块时使用。
