匿名用户数据

用户数据匿名化已得到实施，以支持《通用数据保护条例》（GDPR）中针对居住在欧盟的数据主体的擦除权或“被遗忘权”。如果用户要求您删除其个人数据，请使用此操作来代替删除用户帐户。

匿名用户帐户时，与该帐户关联的个人数据将以无法识别的方式进行清理。全名，电子邮件地址，登录名，登录详细信息，配置文件更改历史记录和IP地址将根据当前的匿名方案替换。Hub中的匿名方案有两个选项。

选项

描述

昏迷

通过将个人数据替换为SHA-256哈希来假名化个人数据。虽然假名化数据不再以人类可读的状态存储，但仍可以将哈希值与原始值链接在一起。

随机化

个人数据将替换为随机值。这种匿名化不可逆转地破坏了识别数据主体的任何方式。随机数据无法恢复为可读状态。

选择匿名方案的设置位于“ 系统设置”页面上。有关更多信息，请参阅系统设置 。

为用户匿名化个人数据：

需要权限：更新用户

  1. 在管理菜单的访问管理部分，选择用户 。

  2. 在列表中单击要匿名的用户名。

    • 用户个人资料页面打开。

  3. 单击标题中的“ 匿名化”按钮。

    • 将打开“ 匿名用户”对话框。

  4. 在输入字段中输入要匿名的用户登录名。

  5. 单击匿名化按钮。

    • 根据“ 系统设置”页面上的“ 匿名化方案”设置，对用户的个人数据进行匿名化 。

    • 匿名配置文件以只读状态显示，无法更改。

    • 匿名用户无法再使用此帐户登录集线器或任何连接的服务。

匿名不会将用户帐户从集线器中删除。该帐户保留在系统中。分配给匿名用户的所有许可证仍将授予该帐户。最后，您可能要禁止用户帐户并释放这些许可证分配。有关说明，请参阅“ 禁止用户帐户” 。
