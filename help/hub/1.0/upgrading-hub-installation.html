This page covers upgrade procedures for Hub.

Upgrading MSI Installation

If you run Hub as the Windows Service installed by Hub MSI distribution, then to upgrade to a newer build you just need to start the MSI installer of the new Hub build. The installer will handle all upgrade procedures automatically: It will stop the currently running service, uninstall previous version, install the newer build and start the service.

Upgrading ZIP Installation

If you have installed Hub via ZIP distribution, then proceed as follows:

All commands listed in this section are Linux/Mac OS commands. If you are working on a Windows server, simply replace .sh to .bat.

  1. Stop Hub: <hub_home>/bin/hub.sh stop
  2. Use your operating system’s tools to make sure that all Java processes launched by Hub are no longer running. If there are Hub-related Java processes still running, kill the outstanding process trees.
  3. If your <hub_home> directory is not build-specific, rename <hub_home> toHub.old.
  4. Unpack the zip archive with a new Hub build.
  5. Copy data and conf directories from from Hub.old to<hub_home>.

    data and conf folders contain hidden files. Use recursive copying cp -r in Linux or Mac OS X to avoid data loss.

    Before copying data and conf folders to the new installation, the existing data and conf folder in that installation should be deleted to prevent data corruption.

  6. Start Hub:
    <hub_home>/bin/hub.sh start

Upgrading ZIP Installation with MSI

If you wish not only to upgrade the ZIP installation, but change the installation type to MSI at the same time, then you need to perform the procedure described below.

All commands listed in this section are Linux/Mac OS commands. If you’re working on a Windows server, simply replace .sh to.bat.

To upgrade previous ZIP installation with the MSI installer

  1. Stop currently running Hub server: <hub_home>/bin/hub.sh stop
  2. Back up the conf and data directories: Copy these directories to a separate location.
  3. Make sure that %programdata%\JetBrains\Hub directory is empty. Some data may have been left from the previous installations.

  4. If, while configuring your current ZIP installation, you changed the default database location so the database is not located in the <Hub Installation Directory>, then proceed to the next step.

    If the current database is located in the <Hub Installation Directory>, then put the content of the backed up data folder to the %programdata%\JetBrains\Hub\data.

  5. Copy the content of the backed up conf folder to %programdata%\JetBrains\Hub\conf.
  6. Install the new version using MSI distribution.
