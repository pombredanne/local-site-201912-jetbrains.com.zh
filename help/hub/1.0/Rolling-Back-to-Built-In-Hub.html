Migrating Data to External HubHub User Guide

Rolling Back to Built-In Hub

It is important to remember that once you migrated the data, and your application started to use the standalone Hub, you will be able to rollback to the in-application user management only by restoring from the pre-migration backup! Any change made in the standalone Hub will be lost.

Roll back to the built-in Hub service

  1. Restore the service that needs the rollback from a backup.
    • For YouTrack, refer to the Restoring database from the backup page.
    • For Upsource, refer to the Backing up and restoring data page.
  2. Delete the service from the external Hub:
    1. In the external Hub, open More Settings > Services page.
    2. Select the service in the list, and click the trash button in the toolbar.

When a service is deleted from Hub, all its resources and permissions are deleted as well.
