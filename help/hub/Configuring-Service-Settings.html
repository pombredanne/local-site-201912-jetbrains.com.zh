服务设定

服务的设置页面提供了几个选项，您可以使用这些选项来管理服务。要访问此页面，请从“ 管理”菜单的“ 服务器设置”部分中选择“ 服务 ” ，然后在列表中单击服务的名称。

标头中显示以下控件：

控制

描述

信任/不信任

切换服务的信任状态。

删除服务

从中心删除服务。

设定值

设置选项卡显示所选服务的设置和参数。

以下设置可用于每种服务：

设置

描述

名称

设置服务的名称。使用此设置可以将该服务与同一应用程序的其他服务区分开。

ID

显示唯一的服务ID。当服务连接到集线器时，会为其分配一个唯一的service_id 。如果要使用REST API连接到服务，则需要了解service_id授权和访问服务。

键

显示唯一的服务密钥。

图片

设置在集线器中为服务显示的图像。

秘密

打开“ 更改服务密钥”对话框。有关更多信息，请参见更改服务密钥 。

起始网址

显示服务的绝对基本URL。单击“ 打开URL”按钮导航到服务主URL。

用户URI模式

显示用于识别服务中用户帐户的模式。

组URI模式

显示用于识别服务中的组的URI模式。

基本网址

显示绝对URL列表，这些绝对URL除了用于解析重定向URI的Home URL之外，还用于

重定向URI

显示用于服务验证的已注册重定向URI。有关集线器如何验证重定向URI的更多信息，请参阅重定向URI验证 。

不受信任的重定向URI

显示已用于请求访问服务的不可信端点的列表。提供以下选项：

选项

描述

接受访问

将端点添加到已注册的重定向URI列表中。该地址用于验证对服务的访问。

忽视

从不受信任的重定向URI列表中删除端点。该地址不允许用于验证对服务的访问。

访问

明确授予“ 读取服务”权限给所选组的成员。这些组的成员可以从标题中的“ 服务”菜单访问服务提供的应用程序。有关更多信息，请参见将服务添加到服务菜单 。

审计

链接到中心中的“ 审核事件”页面。在此处，您可以查看已应用于此服务或此服务已应用的更改的列表。

页面的“ 应用程序”部分显示应用程序的名称，版本号，供应商和发布日期（如果有）。这些属性是从连接的服务导入的，无法更改。

资源资源

资源选项卡显示所选服务中可用资源的列表。资源类型因服务而异。例如，YouTrack服务显示YouTrack中的项目列表。集线器服务显示用户组列表。

每个服务的“ 详细信息”侧栏中显示以下信息：

柱

描述

键

分配给中心中资源的唯一ID。

类型

资源的实体类型。

起始网址

资源的URL。单击链接以在连接的服务中打开资源。

项目

使用此资源的Hub项目的名称。单击链接以在“ 项目”页面中打开项目。

权限

权限选项卡显示所选服务中可用的权限列表。

每个服务的权限列表中显示以下信息：

柱

描述

键

分配给权限的唯一ID。

名称

权限的名称和简短描述。

类型

权限授予访问权限的实体类型。

运作方式

权限授予执行权限的操作类型。

默认角色

“ 默认角色”选项卡显示了在所选服务中定义的默认角色的列表。该页面上列出了每个默认角色，并列出了当前分配给该角色的权限。

可以使用以下控件：

控制

描述

重置为默认权限

将所有默认角色重置为服务定义的默认权限。

重启

将单个默认角色重置为服务定义的默认权限。此按钮显示在每个默认角色的标题中。

执照

“ 许可证”选项卡显示所选服务的许可证属性。

在这里，您可以管理被授予每用户许可证的用户列表。有关更多信息，请参阅管理服务许可证 。
