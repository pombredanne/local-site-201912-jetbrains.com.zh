Hub 2019.1帮助

获取项目的所有资源

请求

GET /projects/{project id}/resources?fields=string&$skip=int&$top=int&query=string&orderBy=string

参数

描述

fields

可选的 。仅返回每个资源的字段的指定子集。使用字段语法定义子集。

$skip

可选的 。让您设置一些要返回第一个资源之前要跳过的资源。

$top

可选的 。使您可以设置要返回的最大资源数。

query

可选的 。用于指定搜索查询以过滤返回的结果。有关支持的逻辑操作语法的定义，请参见查询语法。对于资源，支持以下字段和元组：

有 ： 类型 -资源与某种类型

has ： homeUrl —带有一些homeURL的资源

homeUrl ： 值 —具有给定家庭URL的资源

id ： value —具有给定ID的资源

key ： resourceKey —具有给定密钥的资源

name ： resourceName-具有给定名称的资源

project ： project —给定项目中的资源

服务 ： 服务 —给定服务提供的资源

type ： resourceType —给定类型的资源

orderBy

可选的 。返回按指定字段排序的资源列表。有关详细信息，请参见排序语法 。资源支持按以下字段排序： 键 ， 名称 ， 项目 ， 服务 ， 类型

响应

回应码

含义

200 OK

成功的请求。

400错误的要求

至少一个请求参数无效。例如，缺少传递的JSON对象中的必填字段。有关详细信息，请检查响应中的错误消息。

403禁止

请求者无权访问所请求的资源。

找不到404

找不到请求的资源。

500内部服务器错误

由于服务器错误，无法处理请求。有关详细信息，请检查响应中的错误消息。

反应体

{ "skip": int, "top": int, "total": int, "resources": [resource, ...] }
