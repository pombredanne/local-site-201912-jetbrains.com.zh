更改服务秘密

就像用户帐户的密码一样，您可能希望定期更改服务密码以维护安全性。如果连接到集线器的服务受到威胁，您还应该更改服务密码。

此页面上描述的过程仅适用于外部集线器安装以及与Upsource安装捆绑在一起的内置集线器服务。在使用内置集线器服务或YouTrack InCloud实例的YouTrack Standalone安装中，请勿尝试更改这些设置。

在集线器中，您可以生成新的服务密钥，也可以手动设置服务密钥。如果服务密码保存在客户端，则还必须在连接的服务中更改服务密码。

先决条件

您必须有权访问连接服务中的相关属性文件。如果您不更改连接服务中的服务机密，则该服务将无法连接到集线器。

更改服务秘密：

需要权限：更新服务

  1. 在外部集线器服务中，从“ 管理”菜单的“ 服务器设置”部分中选择“ 服务 ” 。

  2. 从列表中选择服务。

  3. 在“ 设置”选项卡上，单击“ 机密”设置旁边的“ 更改”按钮。

    • 将打开“ 更改服务密钥”对话框。

  4. 在“ 更改服务机密”对话框中，您可以：

    • 自动生成新机密：

      • 复制新机密。

      • 点击更改机密按钮

    • 关键在手动将新的秘密：

      • 选择“ 键入”选项卡。

      • 输入并确认新密码。记录为新机密设置的值。您需要在服务的属性文件中更新此值。

      • 单击更改密码按钮。

  5. 将服务机密属性设置为已连接服务中的新服务机密。

    • 更改TeamCity中的服务机密
    • 更改Upsource中的服务机密
    • 更改YouTrack中的服务机密

如果服务秘密保存在客户端，则需要设置服务秘密属性以匹配集线器中的新服务秘密。

要更改TeamCity中的服务机密，请执行以下操作：

  1. 在TeamCity主目录中，打开/conf/hub-config.properties文件。

    • 对于Windows安装， directory是安装目录。

    • 对于Linux或macOS安装， 目录是安装过程中解压缩归档文件的位置。

  2. 将新的服务秘密粘贴或写入到hub.service.secret属性。删除行末的所有空格。

  3. 保存并关闭属性文件。

  4. 重新启动服务。

要更改Upsource中的服务机密，请执行以下操作：

  1. 在上游主目录中，打开/conf/internal/hub.properties文件。的 directory是安装过程中解压缩ZIP分发包的位置。

  2. 将新的服务秘密粘贴或写入到upsource-frontend.service.secret属性。删除行末的所有空格。

  3. 保存并关闭属性文件。

  4. 重新启动服务。

要在YouTrack中更改服务秘密：

  1. 在YouTrack主目录中，打开/conf/internal/hub.properties文件。

    • 对于ZIP安装， directory是安装过程中解压缩ZIP分发包的位置。

    • 对于MSI安装， 目录是%programdata%\JetBrains\YouTrack 。

  2. 将新的服务秘密粘贴或写入到youtrack.service.secret属性。删除行末的所有空格。

  3. 保存并关闭属性文件。

  4. 重新启动服务。
