还原MSI安装

如果安装集线器时遇到问题，则可能需要还原到最新的工作版本。需要还原安装的常见方案包括：

  • 您无法成功升级安装，需要还原以前安装的版本。

  • 安装的数据库中断，您需要从数据库的备份副本中还原当前版本。

在这两种情况下，您都可以使用Hub数据库的有效备份来还原安装。如果备份已过期，则可以尝试使用配置和数据文件的有效副本来还原安装。仅当您具有有效备份或数据库副本未损坏时，才可以进行还原。

您应该定期测试此过程，以确保在发生故障或攻击后可以使用备份文件来还原安装。

要成功地从备份还原数据库，备份文件的版本应与将数据库还原到的Hub的版本相同或更早。

先决条件

在开始之前，请确保您具有要还原的数据库的备份副本。请注意备份副本的位置，因此请确保在此过程中选择正确的文件。

如果您没有有效的备份文件，或者您的备份文件已过期，请按照说明进行操作，以还原没有数据库备份的当前版本 。

  • 如果升级失败后回滚到以前的安装版本，请使用在安装之前创建的备份。如果您忽略此先决条件，请在设置为与先前安装版本相对应的“ 备份”位置的目录中找到已归档的备份文件。

  • 如果尝试还原当前安装，请在设置为与当前安装版本相对应的“ 备份”位置的目录中找到已存档的备份文件。

从数据库的备份副本还原安装

要将数据库还原到当前或先前版本，需要重新安装Hub并将备份用作安装期间的升级源 。

还原到YouTrack的先前版本时，请使用与要还原的产品版本相对应的备份。

要从备份还原数据库：

需要权限：低级管理员写

  1. 从Windows控制面板卸载集线器。

  2. 确保集线器服务用户（ JetBrainsHub ，默认情况下）对您保存数据库备份文件的目录具有完全访问权限。

  3. 安装与数据库备份版本相同或更高的集线器版本。有关详细说明，请参阅《 MSI安装》 。

    请注意：

    • 要更有效地排除错误，请在启用详细日志记录的情况下运行MSI安装程序。在“ 命令提示符”窗口中，输入以下命令：

      msiexec / i hub- 。 .msi / l * v install.txt

      使用安装程序包中的和。

    • 在安装过程中可能会找到以前安装的配置文件。在这种情况下，请使用默认选择的（推荐）安装选项。

  4. 在配置向导中 ，选择升级选项。

  5. 在“ 选择升级源”页面上，选择备份作为升级源 ，然后单击“ 下一步”按钮。

  6. 确认或更改集线器存储产品数据的位置。完成后，单击“ 下一步”按钮。

    • 集线器从备份文件中的数据开始。

在没有数据库备份的情况下还原当前版本

在某些情况下，您将无法从备份还原安装。

  • 您的备份文件丢失或已过期。

  • 服务器上的备份文件已损坏，无法使用。

例如，当服务器崩溃后能够恢复数据，并且在YouTrack中记录的信息不在最新的备份文件中时，您可能还希望遵循此过程。

您可以尝试使用配置和数据文件的有效副本来还原安装的当前版本。仅当您的配置和数据文件未损坏时，此还原过程才能成功。仅当拥有从未用于启动产品更高版本的文件副本时，才可以还原到Hub的早期版本。

要还原数据库而不进行备份：

需要权限：低级管理员写

  1. 找到您的最新副本data和conf目录。在安装过程中创建的目录的默认位置是%programdata%\JetBrains\Hub\ 。
    请注意，管理员可能已更改了集线器存储这些文件的位置。数据库当前位置的路径存储在data-dir中的参数%programdata%\JetBrains\Hub\conf\internal\bundle.properties文件。您也可以使用这些目录的副本，这些副本存储在其他位置。

  2. 复制conf目录和hub子目录data夹。将它们粘贴到一个空目录中。

    • 如果数据库存储在默认位置，则可以从下图所示的位置复制这些文件，并按右图所示组织新目录的内容。

    • 如果您的数据库存储在其他位置，请参考bundle.properties文件以找到数据文件并重新创建结构，如右图所示。

  3. 将新目录的内容压缩到存档文件中。您可以使用任何文件名。

  4. 确保集线器服务用户（ JetBrainsHub ，默认情况下）对此目录具有完全访问权限。

  5. 从Windows控制面板卸载集线器。

  6. 安装与数据库版本相同或更高的集线器版本。

  7. 在配置向导中 ，选择升级选项。

  8. 在“ 选择升级源”页面上，选择包含您的目录的目录。 data和conf目录作为升级源 ，然后单击下一步按钮。

  9. 确认或更改集线器存储产品数据的位置。完成后，单击“ 下一步”按钮。

    • 集线器从配置和数据文件中的数据开始。
