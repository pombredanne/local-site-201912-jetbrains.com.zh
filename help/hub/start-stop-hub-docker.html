启动和停止Docker容器

本页描述如何停止和启动集线器服务器。

在以下情况下，您需要停止集线器服务器：

  • 您要更改中心数据库的位置。

  • 您要修改集线器服务器的集线器属性或JVM选项。

  • 您要升级集线器安装。

运行Hub docker容器

要运行集线器容器，请执行以下命令：

docker run -it --name hub-server-instance \ -v ：/ opt / hub / data \ -v ：/ opt / hub / conf \ -v ：/ opt / hub / logs \ -v ：/ opt / hub /备用\ -p ：8080 \ jetbrains / hub：

命令属性：

  • -it —一个标志，它将集线器容器的输入和输出（包括启动日志）附加到终端窗口。请注意，当终端连接到容器输出时，按Ctrl + C会导致容器关闭。使用`Ctrl + PQ`可以将终端从容器输出中分离出来。有关更多详细信息，请参阅官方docker文档 。

  • --name hub-server-instance —容器的任意名称。

  • -v :/opt/hub/NNN —将主机上特定于集线器的“ NNN”目录绑定到各自/opt/hub/NNN容器内的目录。

  • -p :8080 —定义端口映射的参数。指示主机监听端口并将所有流量传播到docker容器内的端口8080。

停止Hub Docker容器

要正常停止集线器服务，请运行以下命令：

docker exec 停止

要正常关机，您也可以使用标准docker kill命令：

码头工人杀--signal = SIGTERM
`

我们不建议使用标准命令` docker stop `。默认情况下，此命令发送` SIGTERM向Docker容器内的Hub进程发出信号，然后等待10秒。如果集线器在该时间段内未完成关闭，则` SIGKILL `被发送到内核，并且Hub进程被杀死。这可能会导致数据损坏。为避免此结果，请在使用此命令时指定适当的超时值。例如，以下命令为中心服务关闭留出了足够的时间：

码头工人停止-t 60

将集线器容器作为服务运行

Docker团队建议使用跨平台的内置重启策略来将容器作为服务运行。为此， 将您的docker服务配置为在系统启动时启动 ，只需添加参数--restart unless-stopped到docker run启动Hub的命令。

但是，当涉及到多个服务（包括集线器）的顺序启动时，重新启动策略方法将不适用。您可以改用流程管理器 。

这是一个如何在Linux的帮助下将Hub容器作为服务运行的示例systemd 。

在具有systemd的Linux上将集线器容器作为服务运行

  1. 创建服务描述符文件/etc/systemd/system/docker.hub.service ：

    [Unit]描述= Hub Service After = docker.service需求= docker.service [Service] TimeoutStartSec = 0重新启动=总是ExecStartPre =-/ usr / bin / docker exec％n stop ExecStartPre =-// usr / bin / docker rm％ n ExecStartPre = / usr / bin / docker pull jetbrains / hub： ExecStart = / usr / bin / docker run --rm --name％n \ -v：/ opt / hub / data \ -v：/ opt / hub / conf \ -v：/ opt / hub / logs \ -v：/ opt / hub / backups \ -p：8080 \ jetbrains / hub： [安装] WantedBy = default.target

  2. 使用以下命令启用在系统启动时启动服务：

    systemctl启用docker.hub

您还可以随时分别通过以下命令手动停止和启动服务：

sudo服务docker.hub停止sudo服务docker.hub开始
