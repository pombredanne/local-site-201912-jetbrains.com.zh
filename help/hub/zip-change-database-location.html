更改ZIP的数据库位置

集线器存储数据的默认目录包含以下子目录：

名称

描述

hub

包含中心服务的数据库

internal

可能包含其他内部产品数据

将现有数据库复制到新位置时，请确保新位置使用此处显示的相同文件夹结构。

您可以使用以下过程更改ZIP分发的数据库位置。

此页面上的所有命令均适用于Linux和macOS。对于Windows安装，请更换.sh与.bat并使用反斜杠。

要更改数据库位置：

  1. 备份数据库。以防万一。

  2. 停止集线器服务。有关特定说明，请参阅“ 启动和停止Hub ZIP” 。

  3. 将现有数据库移到新数据库目录。

  4. 输入以下内容configure命令来设置新数据目录的位置：
    bin/hub.sh configure --data-dir=

    对于ZIP安装， directory是安装过程中解压缩ZIP分发包的位置。

  5. 启动集线器服务。有关特定说明，请参阅“ 启动和停止Hub ZIP” 。

其他目录

如果将数据库移动到另一台服务器，则可能还需要更改集线器存储日志和临时文件的位置。

要更改这些文件的位置，请遵循描述的更改数据库位置的相同步骤，并在脚本文件中执行以下命令：

目录

命令

后备

 configure --backups-dir=

日志

 configure --logs-dir=

温度

 configure --temp-dir=
