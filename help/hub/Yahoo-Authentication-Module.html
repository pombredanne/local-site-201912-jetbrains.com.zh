雅虎！验证模块

该身份验证模块使用户可以使用Yahoo!登录到Hub。帐户。

启用Yahoo!认证方式

允许用户使用其Yahoo!登录Hub。帐户，启用Yahoo!身份验证模块。

启用Yahoo!验证：

需要权限：低级管理员写

  1. 在管理菜单的访问管理部分，选择身份验证模块 。

  2. 从“ 添加模块”下拉列表中，选择Yahoo（OpenID） 。

    • 雅虎！身份验证模块会自动创建，启用和配置。

    • 验证模块页面显示Yahoo!的设置。身份验证模块。有关此页面上设置的更多信息，请参阅设置 。

    • 按钮图像设置中存储的图标将添加到登录对话框窗口中。用户可以单击此图标以对其Yahoo!进行身份验证。帐户。

设定值

将模块添加到集线器后，将自动配置设置。您不必更改这些设置。配置选项以定义Hub如何使用Yahoo!处理新用户帐户的选项。凭据，请参阅其他设置 。

领域

描述

类型

显示在集线器中启用了第三方身份验证的应用程序或服务的名称。

名称

存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。该名称还显示在登录表单上的第三方服务图标的工具提示中。

按钮图片

显示用于用户单击以其连接的授权服务中的帐户登录集线器的按钮的图像。您可以上传JPG，GIF或PNG文件。图像会自动调整为48 x 48像素。

服务器网址

将用于验证登录请求的URL存储在Hub中。

架构图

其他设置

当用户使用Yahoo!登录到Hub时。帐户首次使用，Hub会检查是否存在具有相同电子邮件地址的帐户。如果存在具有相同电子邮件地址的帐户，则Yahoo!凭据将添加到现有帐户。

使用以下设置来配置集线器如何管理用户帐户。这些设置还可以减少空闲连接消耗的处理资源的损失。

选项

描述

用户创建

为使用存储在连接的授权服务中的帐户登录的未注册用户启用集线器帐户的创建。集线器使用电子邮件地址来确定用户是否具有现有帐户。

自动加入群组

当用户使用存储在连接的授权服务中的帐户登录时，将用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。

连接超时

设置等待建立与授权服务的连接的时间。默认设置为5000毫秒（5秒）。

读取超时

设置等待从授权服务读取和检索用户配置文件数据的时间。默认设置为5000毫秒（5秒）。

审计

链接到中心中的“ 审核事件”页面。在这里，您可以查看已应用于此身份验证模块的更改列表。
