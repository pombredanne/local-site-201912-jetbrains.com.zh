降价语法

在Hub周围的不同位置，您可以格式化文本块。此格式是使用Markdown标记语法应用的。集线器中的以下功能支持Markdown：

特征

描述

快速笔记小部件

这些小部件使用Markdown格式化文本。这些小部件可以放在仪表板和项目概述页面上。

项目概况

项目概述页面上显示的项目描述以Markdown格式设置。

用户协议

呈现给需要接受信息通知才能访问集线器的用户的协议文本采用Markdown格式。

YouTrack中的Markdown实施遵循带有扩展名的CommonMark规范 。这些扩展支持格式规范中未包括的格式选项，例如删除线文本，表格和自动链接。

要查看这些格式化选项中的任何一个，请将代码示例块粘贴到接受Markdown的输入字段中。

字符格式

您可以使用以下Markdown语法设置内联文本的格式。

样式

句法

强大

用两个星号包围文本（ ** ）或两个下划线字符（ __ ）。

重点

带有单星号的环绕文本（ * ）或下划线字符（ _ ）。

删除线

用两个波浪号环绕文本（ ~~ ）。

码

用单反引号将文本引起来（ ` ）。

组合方式

用两个下划线包围单个下划线字符或两个波浪号。用三个星号包围文本，以突出显示重点。

You should read this, it's **very important**. You _might_ want to read this. ~~Never mind~~ - it wasn't _that_ important. Let's try a few `combinations`: **This text is strong, ~~this text is strong with strikethrough~~, and _this text is formatted with strong emphasis_** ***This text is formatted with strong emphasis too.***

标题

要应用标题样式，请从一到六个数字符号开始# 。在数字符号后跟一个空格字符，然后输入标题文本。的数量#标志确定航向。

集线器还支持标题级别1和2的替代语法：

  • 对于标题级别1，输入一个或多个=下一行中的字符。

  • 对于标题级别2，输入一个或多个-下一行中的字符。

# Heading 1 ## Heading 2 ### Heading 3 #### Heading 4 ##### Heading 5 ###### Heading 6 Heading 1 - Alternative Syntax ======== Heading 2 - Alternative Syntax --------

段落和换行符

连续的文本行属于同一段落。使用以下准则将内容组织成段落并输入换行符。

  • 要开始新段落，请在文本行之间留空行。

  • 要在段落中开始新行，请在文本行末尾输入两个尾随空格。

Even though this text is written on two separate lines, it is parsed as a single paragraph. This paragraph is separated from the previous paragraph by a blank line.

专题休息

用水平线在内容中创建部分。使用以下任何一种方法来添加一条水平线：

  • 三个下划线（ ___ ）

  • 三个减号（ --- ）

  • 三个星号（ *** ）

The underscores on the next line create a thematic break below this paragraph. ___ The minus signs below must be separated from this paragraph by a blank line. If not, they are parsed as a level 2 heading. ---- Three or more asterisks also create a thematic break. ****

块引号

使用块引号可以特别注意来自另一个来源的引号。您可以将字符格式应用于带引号的块中的内联文本。

要将文本设置为引号块，请以一个或多个开头>字符。在这些字符后加一个空格，然后输入引号。的数量>符号确定引用块内嵌套的级别。

如果您的报价跨越多个段落，则每个空白行必须以>字符。这样可以确保将整个引号块组合在一起。

> Use quote blocks to emulate reply text. > This line is part of the same quote. This line is not formatted and does not belong to the quote block. > This block spans multiple paragraphs. > > The second paragraph is grouped with the previous paragraph in the same quote block. > Character formatting is _also_ supported inside the **quote block**. > Quote blocks can also be nested. >> When you start a new line with additional > characters, >>> it simulates a threaded conversation.

缩进代码块

您可以使用等宽字体设置文本块的格式，以使其更易于识别和读取为代码。

要在Markdown中格式化代码块，请将代码块的每一行缩进至少四个空格。缩进的代码块不能中断段落，因此您必须在紧随其后的缩进的代码块的段落之间插入至少一个空行。输入处理如下：

  • 从代码块的每一行删除了一个缩进级别（四个空格）。

  • 代码块的内容是文字文本，不解析为Markdown。

  • 任何前导空格少于四个的非空白行将结束代码块并开始一个新段落。

Start an indented code block following a paragraph with a blank line and at least four spaces of indentation: This is a code block. Blank lines between indented lines do not end the code block. Here is some HTML:
© 2009—2017 JetBrains · All rights reserved
This line of text is not indented. It ends the code block and starts a new paragraph.

围栏代码块

与缩进代码块不同，受围代码块具有一个信息字符串，可让您指定语法突出显示所使用的语言。特定于语言的突出显示使代码更易于阅读。

多种语言都支持语法突出显示。YouTrack自动检测并突出显示C，C ++，C＃，Java，JavaScript，Perl，Python，Ruby和SH中的代码。要突出显示其他语言的代码，请在信息字符串（带有开头代码围栏的行）中设置语言。支持以下语言：阿波罗（AGC / AEA汇编语言），基本语言，clj（Clojure）css，dart，erlang，hs（Haskell），kt（Kotlin），lisp，llvm，lua，matlab，ml，mumps，n （Nemerle），pascal，proto，scala，sql，tcl，tex，vb，vhdl，wiki，xq和yaml。

  • 要创建跨越多行代码的受保护代码块，请在三个或更多反引号（ ``` ）或波浪号（ ~~~ ）。

  • 用相同的字符打开和关闭块。

  • 使用相同数量的字符来打开和关闭代码围栏。

Set multiple lines of code in fenced code blocks. ``` action: function(ctx) { workflow.check(!ctx.issue.isChanged('votes'), workflow.i18n('Voting for a resolved issue is not allowed.')); }, ``` The following code block uses syntax highlighting for Haskell: ```hs -- Point-free style fib :: Integer -> Integer fib = (fibs !!) where fibs = 0 : scanl (+) 1 fibs -- Explicit fib :: Integer -> Integer fib n = fibs !! n where fibs = 0 : scanl (+) 1 fibs ```

清单

使用以下语法创建列表：

  • 要创建无序列表，请以破折号（ - ），星号（ * ）或加号（ + ）。

  • 要创建有序列表，请以数字和句点（ 1. ）。递增后续数字以格式化有序列表中的每个项目。

  • 要将无序列表嵌套在无序列表或有序列表中，请将行缩进两个空格。不支持嵌套有序列表。

Things I need to do today: 1. Fix usability problem 2. Clean up the page * Make the headings bigger 2. Push my changes 3. Create code review * Describe my changes * Assign reviewers * Ask for feedback

桌子

表格是为内容添加结构的绝佳工具。使用以下语法创建表：

  • 要创建列，请使用竖线（ | ）。外栏是可选的。

  • 用三个或更多破折号将标题行与表格的其余部分分开（ --- ）。

请注意，列不必在原始Markdown中完美对齐。您还可以将字符格式添加到表中的文本。

Kitchen Cleanup Rotation | Month | Assignee | Backup | | -------- | -------- | ------ | | January | Dave | Steve | | February | Gregg | Karen | | March | Diane | Jorge | Here's the same text with character formatting. + The text in the first column is flush right. + The text in the second column is centered. + The Markdown is stripped down to the minimum syntax that is required to render the table. Month | Assignee | Backup ---:|:---:| --- **January** | Dave | _Steve_ **February** | Gregg | _Karen_ **March** | Diane | _Jorge_

所有单元格都是左对齐的。不支持将文本右对齐或居中对齐的语法。

链接

有几种方法可以用Markdown插入超链接。

样式

格式

排队

用括号括住链接文本[ ]后跟括号中的URL ( ) 。

内嵌工具提示

使用内联格式，并在URL后面的引号中添加工具提示。

参考

将网址替换为任意的，不区分大小写的引用，并用方括号括起来[ ] 。将参考文本和URL添加到文档中的其他位置。要定义引用，请输入相同的标签名称，并用括号括起来，然后加冒号，再输入URL。这些引用未显示在渲染的Markdown中。当您有多个链接指向同一目标时，请使用引用样式链接。

集线器还扩展了标准语法，以包括自动链接 。

[inline link](https://www.jetbrains.com) [inline link with tooltip](https://www.jetbrains.com "JetBrains: Development Tools for Professionals and Teams") [reference link][1] [1]: https://www.jetbrains.com

URL和尖括号中的URL将自动转换为超链接。例如：

  • https://www.jetbrains.com

  • jetbrains.com

自动连结

自动链接是在尖括号（）中设置的绝对URI和电子邮件地址（ < > ）。它们被解析为链接，以URL或电子邮件地址作为链接标签。不同于链接 ，让你指定链接文本和工具提示，这句法简单地转换URL或电子邮件地址转换为可点击的链接。

集线器支持URL的扩展语法。即使没有尖括号，任何被解析为URL的字符串都将转换为可单击的链接。未在尖括号内设置的电子邮件地址将显示为文本。

图片

图像的语法类似于链接的语法。要插入嵌入式图像：

  • 以感叹号（ ! ）。

  • 将替代文本用方括号（ [ ] ）。

  • 在括号中设置图片网址和工具提示（ ( ) ）。

您也可以为图像使用参考样式。要插入图像参考：

  • 以感叹号（ ! ）。

  • 将替代文本用方括号（ [ ] ）。

  • 将图像参考设置在括号中（ [ ] ）。

  • 使用以下格式在其他位置定义图像参考[tag]: image URL "tooltip" 。

反斜杠转义符

当您拥有要显示为书面形式的被解析为Markdown的字符时，可以使用反斜杠（ \ ）。

  • 非标记字符之前的反斜杠显示为反斜杠字符。

  • 转义字符被视为常规字符。它们在Markdown语法中的通常含义将被忽略。

  • 反斜杠转义不适用于受限制的代码块，内联代码范围或自动链接。

Here are a few examples of backslash escapes: \*not emphasis* \`not an inline code span` 1\. not an ordered list \* not an unordered list \# not a heading \This is not a backslash escape - the escaped character is not a markup character.
