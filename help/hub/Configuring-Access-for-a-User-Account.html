管理用户访问

使用“ 角色”选项卡，您可以查看和管理单个用户帐户可用的权限。根据在一个或多个项目中分配的角色，向用户授予权限。

允许用户在集线器中执行特定操作的权限与允许在连接的服务中进行操作的权限之间存在区别。权限通常在项目内授予，并提供对连接到项目的特定资源的访问。例如，如果您在Hub中有一个项目，该项目使用YouTrack项目和TeamCity项目作为资源，则您具有特定于YouTrack的权限（仅适用于YouTrack项目中的操作）和特定于TeamCity的权限（适用于TeamCity项目）。在这种情况下，用户可以对每个连接的服务具有不同的“ 查看项目”和“ 更新项目”权限。

您还可以通过将用户直接添加到项目团队中来授予用户访问权限。有关更多信息，请参见管理项目团队 。

查看访问权限

用户帐户的访问权限显示在帐户配置文件的“ 角色”选项卡上。在这里，您可以查看将哪些角色和权限分配给该帐户。

此选项卡上有两个视图。默认视图显示Project列出的分配给用户的角色列表。

  • 第一部分显示直接分配给用户的角色。

  • 第二部分显示了作为项目团队成员分配给用户的角色。

  • 其余部分显示了作为组成员分配给用户的角色。对于分配给每个启用了自动加入选项的组的角色，有专门的部分。

您还可以选择按角色查看列表。

该信息分为与Project相同的部分。

在两个视图中都可以执行以下操作：

  • 使用搜索框过滤列表。您可以使用一系列过滤条件，包括用户，组，角色和权限。输入格式的搜索条件field: value 。

    例如，如果要查看用户有权更新的项目，请输入permission: (Update Project)并应用过滤器。

  • 单击详细信息按钮以显示或隐藏侧栏。边栏显示分配给角色的一组权限。权限按服务分组。

要查看用户帐户的访问权限：

需要权限：读取用户基础，读取角色，读取项目完整，读取组

  1. 打开帐户资料的“ 角色”标签。

  2. 使用搜索框按角色，项目甚至特定权限过滤列表，以查找帐户是否具有对资源或操作的必需访问权限。

向用户授予角色

如果您不想将角色分配给组中的所有成员，则可以直接将角色分配给用户。

要将角色直接分配给用户：

需要权限：读取角色，更新用户，更新项目

  1. 在管理菜单的访问管理部分，选择用户 。

  2. 使用搜索框找到所需的用户帐户。

  3. 从列表中选择一个用户。

  4. 选择角色选项卡。

  5. 单击授予角色按钮。

  6. 在“ 授予角色”对话框中，选择要分配给用户的角色。

  7. 选择一个或多个要在其中应用角色的项目。

  8. 单击授予按钮。

撤消用户角色

如果用户不再需要角色授予的访问权限，则可以撤消角色分配。此过程仅适用于直接分配给用户的角色。

要撤消用户的角色：

需要权限：读取角色，更新用户，更新项目

  1. 在管理菜单的访问管理部分，选择用户 。

  2. 使用搜索框找到所需的用户帐户。

  3. 从列表中选择一个用户。

  4. 选择角色选项卡。

  5. 找到您要从所选用户帐户中删除的角色分配。使用搜索框按名称，项目或权限查找角色。

  6. 在角色分配下，选择在其中授予角色的项目。

  7. 单击撤消角色按钮。

  8. 在确认对话框中确认操作。

    • 在所选项目中，该角色已从用户吊销。

    • 用户将丢失分配给项目中角色的权限。

对于用户作为组或项目团队的成员继承的角色，您可以通过其他方式撤消访问。

  • 如果将角色授予用户作为组成员，则可以通过从组中删除用户来撤消角色分配。在列表中选择角色，然后单击“ 从组中删除”按钮。

  • 如果将角色授予用户作为项目团队的成员，则可以通过从组或项目团队中删除来取消角色分配。在列表中选择角色，然后单击从团队中删除按钮。
