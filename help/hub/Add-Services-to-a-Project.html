在服务中创建资源

从项目概述页面，您可以在连接到集线器的服务中创建资源。根据用户在Hub项目中可用的权限，授予用户对资源的访问权限。

要在连接的服务中创建资源：

需要权限：读取服务，更新项目

  1. 打开项目概述页面。

  2. 在边栏的“ 服务”部分中，单击“ 在服务控件中创建资源” 。

    • 列表中显示了连接到集线器的所有服务。

  3. 从列表中选择一项服务。

    • 页面上显示特定于连接服务中资源配置的其他设置。

  4. 编辑新资源的配置。有关更多信息，请参见：

    • 问题跟踪器设置
    • 代码审查设置
    • 储存库设定

  5. 保存配置设置。

    • 资源已附加到项目。

    • 到资源的链接被添加到项目概述页面的“ 服务”部分。

    • 根据用户在Hub项目中可用的权限，授予用户对资源的访问权限。

问题跟踪器设置

使用以下设置在连接的问题跟踪服务中配置项目：

设置

描述

名称

在问题跟踪服务中输入项目的名称。默认情况下，输入Hub项目的名称。在连接的YouTrack服务中，项目名称必须唯一。

ID

显示在问题跟踪服务中分配给项目的ID。项目的ID用作问题ID的前缀。项目ID在所连接的YouTrack服务中必须是唯一的。

描述

在问题跟踪服务中输入项目的可选说明。

项目类型

选择您要在问题跟踪服务中创建的项目的类型。提供以下选项：

  • 默认 -使用默认问题字段和工作流创建一个标准的问题跟踪项目。

    要了解更多信息，请阅读YouTrack文档 。

  • Scrum —创建为Scrum配置的问题跟踪项目。自动创建两个不同的敏捷板：一个用于开发，另一个用于项目管理。

    要了解更多信息，请阅读YouTrack文档 。

  • 看板 —创建为看板配置的问题跟踪项目。系统会自动创建一个敏捷板。

    要了解更多信息，请阅读YouTrack文档 。

代码审查设置

使用以下设置可在连接的代码检查服务（上游）中配置项目：

设置

描述

名称

在代码检查服务中输入项目的名称。默认情况下，输入Hub项目的名称。

整合编号

在代码检查服务中输入分配给项目的ID。默认情况下，输入项目的ID。

评论ID模式

设置模式以分配给所连接服务中的代码审阅。默认情况下，设置以下模式： [Integration ID]-CR-{} 。的CR代表“代码复查”，花括号代表一个连续数值的占位符。
