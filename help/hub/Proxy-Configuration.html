反向代理配置

您可以将集线器设置为在反向代理服务器后面工作。

用于启用此设置的过程包括以下步骤：

  1. 将集线器配置为指向代理服务器的基本URL。

  2. 配置代理服务器的标头 。该页面包含有关Apache ， IIS和NGINX服务器的指南。

先决条件

为了从连接到集线器安装的服务中获得最佳性能，代理服务器应支持HTTP / 2。对于大多数服务器，HTTP / 2协议仅适用于HTTPS连接，这意味着您需要使用SSL / TLS证书保护连接的安全。

您还需要确保反向代理服务器使用的OpenSSL库为1.0.2版或更高版本。在早期版本的OpenSSL中，用户使用旧版HTTP / 1.1规范进行连接，而该规范缺乏HTTP / 2连接的性能优势。

本页上的说明假定满足这些要求。使用以下列表确定支持此设置所需的服务器版本：

服务器

最低版本

阿帕奇

2.4.17

IIS

NGINX

1.11.7

HTTP严格传输安全

我们也强烈建议您将反向代理服务器设置为使用HTTP严格传输安全性（HSTS）。这可以保护您的安装免受可能的中间人攻击，在中间人攻击中，可以利用从站点到HTTPS版本的重定向将用户定向到恶意站点，而不是原始页面的安全版本。

本页上提供的代理服务器标头配置说明包括支持HSTS的指令。

更改集线器服务器的基本URL

更改ZIP或MSI安装的基本URL

您可以使用以下过程来更改ZIP和MSI分发的基本URL。要在MSI分发中执行这些命令，请以管理员身份打开“ 命令提示符”窗口。始终执行configure命令代表运行集线器服务的OS用户。此命令创建配置文件和文件夹。集线器服务用户应具有足够的权限来访问这些文件和文件夹。

更改集线器服务器的基本URL：

  1. 停止集线器服务。有关特定说明，请参阅停止和启动集线器服务 。

  2. 在命令行界面中，将目录更改为/bin 。

    • 对于ZIP安装， directory是安装过程中解压缩ZIP分发包的位置。

    • 对于MSI安装， 目录是MSI分发的安装位置。默认安装目录为C:\Hub

  3. 输入以下命令：

    hub.sh configure --listen-port 1111 --base-url https://hub.mydomain.com:XXXX

    • 1111是集线器服务器侦听的端口号。

    • https://hub.mydomain.com是您的代理服务器的地址。

    • XXXX是您的代理服务器侦听的端口号。

  4. 在代理服务器中配置标头。请遵循适用于您的Apache ， IIS或NGINX的准则 。

  5. 启动集线器服务。有关特定说明，请参阅停止和启动集线器服务 。

更改Docker安装的基本URL

如果您需要更改正在运行的Hub实例的基本URL ，请使用configure用命令--base-url论点。

更改集线器服务的基本URL

  1. 停止集线器： docker exec stop

  2. 执行以下命令：

    docker运行--rm -it \ -v ：/ opt / hub / conf \ jetbrains / hub： \ configure --base-url = https：//hub.mydomain.com：XXXX

    • https://hub.mydomain.com是您的代理服务器的地址。

    • XXXX是您的代理服务器侦听的端口号。

  3. 启动中心： docker start

配置代理服务器头

要配置代理服务器的标头，请遵循特定于服务器的准则。下面提供了Apache HTTP Server，IIS和NGINX的配置指南。

Apache HTTP服务器配置

要将Apache HTTP Server用作反向代理，您需要运行a2enmod脚本并将指令添加到.conf文件在您的服务器上。

需要Apache httpd 2.4.17或更高版本。

要将Apache HTTP Server设置为反向代理，请执行以下操作：

  1. 使用以下a2enmod脚本来启用headers ， rewrite ， proxy_http ， ssl和http2模块：

    $ a2enmod标头$ a2enmod重写$ a2enmod proxy_http $ a2enmod ssl $ a2enmod http2

  2. 将以下指令添加到VirtualHost相关部分.conf文件：

    协议h2 http / 1.1 RequestHeader设置X-Forwarded-Proto“ https” DefaultType无RewriteEngine on AllowEncodedSlashes RewriteCond％{QUERY_STRING} transport = polling RewriteRule /(.*)$ http://127.0.0.1:1111/$1 [P] ProxyRequests关闭ProxyPass / http://127.0.0.1:1111/ ProxyPassReverse / http://127.0.0.1:1111/ SSLEngine on SSLCertificateFile SSLCertificateKeyFile

  3. 设置以下变量以匹配您的集线器配置：

    • 更换1111以及集线器服务器侦听的实际端口号。

    • 设置值服务器的SSL / TLS证书的位置。

    • 设置值服务器证书的PEM编码私钥文件的位置。

  4. 将其他HSTS标头添加到HTTPS VirtualHost指令。最大年龄以秒为单位。

    ＃保证1年的HTTPS（包括子域）标头始终设置为严格传输安全性“ max-age = 31536000; includeSubDomains”

    请注意，此标头仅在HTTPS VirtualHost上有效。

  5. 添加服务器端重定向以在首次访问站点时升级非HTTPS连接。将此添加到非HTTP VirtualHost。

    [...]ServerName myHubService.com重定向永久/ https://myHubService.com/

    更换myHubService.com与您的集线器服务器的实际域名。

IIS服务器配置

要将IIS服务器用作反向代理，您需要使用应用程序请求路由（ARR）扩展。

需要IIS 10.0版本1709或更高版本。

要将IIS服务器设置为反向代理，请执行以下操作：

  1. 从Microsoft网站下载并安装应用程序请求路由（ARR）扩展。

  2. 在IIS管理器中 ，连接到IIS服务器-在这种情况下， localhost 。

  3. 在“ 连接”窗格中突出显示服务器。

  4. 双击URL Rewrite 。

  5. 单击右窗格中的查看服务器变量 。

  6. 将以下服务器变量添加到列表中：

    HTTP_X_FORWARDED_HOST HTTP_X_FORWARDED_SCHEMA HTTP_X_FORWARDED_PROTO

  7. 在“ 连接”窗格中突出显示服务器。

  8. 双击“ 应用程序请求路由缓存” 。

  9. 单击“ 操作”窗格中“ 代理”标题下的“ 服务器代理设置” 。

  10. 选择启用代理复选框。

  11. 取消选择“ 在响应标头中反向重写主机”复选框，然后单击“ 应用” 。

  12. 在“ 连接”窗格中的“ 站点”下，选择“ 默认网站” 。

  13. 双击“ URL重写”功能，然后在“ 操作”窗格中单击“ 添加规则” 。

  14. 添加具有服务器名称的反向代理规则： localhost:1111 （替换为集线器服务的实际位置和端口）。

  15. 打开规则，检查重写URL，然后添加以下服务器变量：

    • 设置HTTP_X_FORWARDED_HOST可变为{HTTP_HOST} 。

    • 设置HTTP_X_FORWARDED_SCHEMA可变为https 。

    • 设置HTTP_X_FORWARDED_PROTO至https 。

  16. 确保启用了匿名身份验证：

    • 在“ 连接”窗格的“ 站点”部分中，选择“ 默认网站” 。

    • 双击Authentication ，选择Anonymous ，然后在右窗格中单击Enable 。

  17. 将新的SSL绑定添加到“ 默认网站” 。

    • SSL绑定侦听的地址（主机URL）应与集线器基本URL相匹配。

    • 您选择的证书应与服务器DNS地址相对应。

  18. 为默认网站配置HTTP严格传输安全性（HSTS）设置

    • 设置enabled的属性至true 。

    • 指定一个值max-age属性。例如， 31536000 （一年，以秒为单位）。

    • 设置值includeSubDomains和redirectHttpToHttps至true也一样

    有关特定说明，请参阅ISS配置参考 。

NGINX服务器配置

将NGINX服务器配置为反向代理的基本要求包括以下步骤：

  • 设置server_name指令的变量。

  • 添加proxy_set_header和proxy_pass指令。

需要NGINX版本1.11.7或更高版本。

配置NGINX反向代理头：

  1. 打开您的NGINX服务器的配置文件。默认情况下，配置文件名为nginx.conf 。默认目录为/usr/local/nginx/conf ， /etc/nginx ， 要么/usr/local/etc/nginx 。

    您可以通过输入以下内容找到配置文件的确切位置nginx -V在命令行界面中。

  2. 更新server指令并添加proxy_set_header和proxy_pass配置文件中的指令。使用以下示例作为指导：

    服务器{监听443 SSL http2; ssl_certificate ; ssl_certificate_key ; server_name localhost; add_header严格传输安全性max-age = 31536000;位置/ {proxy_set_header X-Forwarded-Host $ http_host; proxy_set_header X-Forwarded-For $ proxy_add_x_forwarded_for; proxy_set_header X-Forwarded-Proto $ scheme; proxy_http_version 1.1; proxy_pass http：//hubmachine.domain.local：1111; }}

  3. 设置以下变量以匹配您的集线器配置：

    • listen 443是您在--base-url参数。

    • proxy_pass http://hubmachine.domain.local:1111是您在集线器服务器上使用您指定的端口的路径-–listen-port命令。

  4. 保存并关闭您的配置文件。

  5. 通过使用以下命令停止和启动服务器，检查NGINX配置中的错误：

    sudo nginx -s停止sudo nginx

    或者，您可以使用以下命令重新加载配置：

    sudo nginx -s重新加载

如果使用Java密钥库，则需要将其转换为PKCS12格式，以将NGINX服务器用作SSL终止代理。以下过程显示了如何使用keytool和openssl转换密钥库。

要将Java密钥库转换为PKCS12格式，请执行以下操作：

  1. 使用keytool转换当前.jks文件转换为PKCS12密钥存储格式.p12 ：

    keytool -importkeystore -srckeystore oldkeystore.jks -destkeystore newkeystore.p12 -deststoretype PKCS12输入目标密钥库密码：[从oldkeystore.jks输入私钥密码，它将是newkeystore.p12的密码]重新输入新密码：[与上面相同]输入源密钥库密码：[输入oldkeystore.jks的密码] ...输入[从oldkeystore.jks输入私钥密码]的密钥密码...

    您将需要输入“目标密钥库密码”。如果你的.jks密钥库包含带有密码的私钥，那么“目标密钥库密码”应等于私钥的密码。

  2. 列出新的密钥库文件的内容：

    keytool -deststoretype PKCS12 -keystore newkeystore.p12 -list输入密钥库密码：[输入步骤1提供的newkeystore.p12的密码] ...

  3. 从中提取.pem文件（证书） .p12密钥库：

    openssl pkcs12 -nokeys -in newkeystore.p12 -out certfile.pem输入导入密码：[输入步骤1中提供的newkeystore.p12的密码] ...

  4. 从以下位置提取未加密的密钥文件.p12密钥库：

    openssl pkcs12 -nocerts -nodes -in newkeystore.p12 -out keyfile.key输入导入密码：[输入步骤1中提供的newkeystore.p12的密码] ...

故障排除

如果在反向代理后面运行服务时遇到集线器问题，请查看是否符合以下任何条件。

条件

用户会定期退出集线器或其他连接的服务。

原因

集线器使用隐藏的嵌入式框架刷新身份验证令牌。如果您已为X-Frame-Options标头DENY对于您的反向代理服务器，用户的令牌到期时将注销。

解

所有包含来自Hub的静态内容的响应都将为X-Frame-Options标头SAMEORIGIN 。要保留此值，请将反向代理服务器配置为使用X-Frame-Options: SAMEORIGIN指令，用于来自Hub的所有静态内容。有关更多信息，请参见X-Frame-Options 。
