Hub 2019.1帮助

获取所有用户详细信息

请求

GET /userdetails?fields=string&$skip=int&$top=int&query=string

参数

描述

fields

可选的 。仅返回每个UserDetail的字段的指定子集。使用字段语法定义子集。

$skip

可选的 。使您可以设置许多要返回的UserDetails，然后再返回第一个。

$top

可选的 。使您可以设置要返回的UserDetails的最大数量。

query

可选的 。用于指定搜索查询以过滤返回的结果。有关支持的逻辑操作语法的定义，请参见查询语法。对于UserDetails，支持以下字段和元组：

authName ： userSamlName — SAML提供的具有给定名称的用户详细信息

authLogin ： 值 —使用给定的JetBrains帐户登录名的用户详细信息

authLogin ： userGithubLogin —给定GitHub登录名的用户详细信息

fullName ： value —在GitHub中具有给定名称的用户详细信息

authLogin ： userJiraLogin —在Jira中使用给定登录名的用户详细信息

authName ： userJiraName —在Jira中具有给定名称的用户详细信息

authLogin ： userLdapLogin —具有给定LDAP用户ID的用户详细信息

authName ： value —在LDAP中具有给定名称的用户详细信息

authName ： userOpenIDName —在OpenID中具有给定名称的用户详细信息

has ： avatar —拥有头像

openID ： 值 —具有给定OpenID标识符的用户详细信息

authLogin ： value —给定集线器登录名的用户详细信息

authMethod ： authModule —与给定auth模块关联的用户详细信息

email ： email —给定电子邮件的用户详细信息

具有 ： 电子邮件 -带有电子邮件的用户详细信息

has ： jabber — jabber的用户详细信息

has ： originService —从某些服务导入的用户详细信息

has ： authLogin —登录详细信息

id ： value —具有给定ID的用户详细信息

jabber ： jabber —使用给定的jabber地址的用户详细信息

originService ： service —从给定服务导入的用户详细信息

type ： authModuleType —给定类型的auth模块的用户详细信息

user ： user —给定用户的用户详细信息

响应

回应码

含义

200 OK

成功的请求。

400错误的要求

至少一个请求参数无效。例如，缺少传递的JSON对象中的必填字段。有关详细信息，请检查响应中的错误消息。

403禁止

请求者无权访问所请求的资源。

找不到404

找不到请求的资源。

500内部服务器错误

由于服务器错误，无法处理请求。有关详细信息，请检查响应中的错误消息。

反应体

{ "skip": int, "top": int, "total": int, "userdetails": [details, ...] }
