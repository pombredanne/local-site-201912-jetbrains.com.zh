Active Directory身份验证模块

Active Directory身份验证模块使用户可以使用存储在目录服务中的凭据登录集线器和所有连接的服务。此身份验证模块已为Microsoft Active Directory预配置。您可以将模块配置为使用标准LDAP方案或基于SSL的LDAPS。

Active Directory身份验证模块不会从目录服务中导入所有用户帐户。集线器仅在未注册的用户首次登录集线器或连接的服务时创建用户帐户。

启用Active Directory身份验证后，集线器将为每次登录尝试检查目录服务。从目录服务中删除的用户无法登录集线器。

先决条件

如果要通过SSL连接到目录服务，请在启用身份验证模块之前为Active Directory服务导入受信任的SSL证书。如果SSL证书和根CA证书之间有任何中间证书，则需要上载包含完整证书链的文件。

Active Directory身份验证模块的设置中不支持导入受信任的SSL证书的选项。相反，您需要访问“ 受信任的SSL证书”页面并将其导入。有关更多信息，请参见Trusted SSL Certificates 。

启用Active Directory身份验证

要允许存储在Microsoft Active Directory中的用户登录到集线器，请启用Active Directory身份验证模块。

要启用Active Directory身份验证：

需要权限：低级管理员写

  1. 在管理菜单的访问管理部分，选择身份验证模块 。

  2. 从添加模块下拉列表中，选择Active Directory 。

    • 将打开“ 新模块”对话框。

  3. 在“ 新建模块”对话框中，输入以下设置的值：

    设置

    描述

    名称

    输入身份验证模块的名称。

    服务器

    输入目录服务的服务器地址。对于通过SSL的连接，请将地址的协议部分更改为ldaps 。

    港口

    输入用于与目录服务通信的端口号。

    • 标准LDAP的默认端口是389 。

    • LDAPS的默认端口是636 。

    SSL协议

    选择此选项可通过SSL连接到目录服务。在建立安全连接之前，您需要为LDAPS服务器导入受信任的SSL证书。有关说明，请参阅可信SSL证书 。

    搜索库

    输入定义用于存储用户帐户的顶级LDAP DN的域组件。例如，如果您的公司使用域mycompany.com ，输入顶级LDAP DN dc=mycompany,dc=com 。
    在此字段中输入的值将添加到LDAP URL中，并且不能包含不安全的字符。
    如果您使用组织单位来管理用户，请为每个组织创建单独的身份验证模块。在搜索库中包括组织单位，以为每个模块创建唯一的LDAP URL。LDAP认证模块不支持LDAP树中的递归搜索。

  4. 单击创建按钮。

    • Active Directory身份验证模块已启用。模块的当前状态显示在标题中模块名称的旁边。

    • “身份验证模块”页面显示Active Directory身份验证模块的设置。该模块已预先配置了标准设置，这些设置基于您在“ 添加模块”对话框中提供的信息。有关此页面上的设置的其他信息，请参阅“ 设置”部分。

测试与Active Directory服务的连接

若要验证Active Directory身份验证模块已连接到您的Active Directory服务，请测试连接。

要测试连接：

  1. 单击测试登录按钮。

  2. 在“ 测试身份验证”对话框中，输入存储在您的Active Directory服务中的用户的凭据：

    • 在登录字段中，输入domainusername 。

    • 在密码字段中，输入password 。

  3. 单击测试登录按钮。

    • 集线器在Active Directory服务中搜索指定的用户帐户。如果找到用户，则显示成功通知。如果出现错误，请检查用户凭据和服务器URL。

设定值

使用以下设置微调到目录服务的连接。

领域

描述

类型

显示为集线器中的第三方身份验证启用的目录服务的类型。

名称

存储认证模块的名称。使用此设置可以将该模块与“身份验证模块”列表中的其他身份验证模块区分开。该名称还显示在登录表单上受支持的身份验证提供程序的列表中。

服务器网址

将目录服务的LDAP URL存储在Hub中，该目录服务用于对登录请求进行身份验证。

LDAP URL使用以下格式ldap://host:port/DN 。输入存储用户帐户的目录的完整专有名称（DN）。

SSL密钥

选择一个SSL密钥以加密集线器和目录服务之间的连接。有关在Hub中管理SSL密钥的更多信息，请参见SSL密钥 。

绑定账户

确定哪个帐户用于LDAP绑定请求。有关更多信息，请参见绑定帐户选项 。

绑定DN

存储用于与目录服务绑定的值。有关更多信息，请参见绑定帐户选项 。

过滤

存储一个表达式，该表达式在LDAP服务中查找特定用户的记录。表达式中的替换变量将替换为在登录页面上作为用户名或电子邮件输入的值。

LDAP引荐

确定LDAP认证模块是忽略还是跟踪来自服务的请求以在LDAP目录中定位其他信息。

  • 如果目录服务将映射到集线器配置文件的属性存储在不同的分区中，请启用“ 跟随”选项。

  • 如果用户身份验证不需要遵循重定向，请启用“ 忽略”选项以提高性能。

绑定帐户选项

您可以配置模块以通过以下两种方式之一对Active Directory服务执行绑定请求。所使用的方法取决于为“ 绑定帐户”设置选择的选项。

用于“ 绑定DN”设置的值取决于您为“ 绑定帐户”设置选择的选项。使用以下准则来设置“ 绑定DN”设置的值：

选项

描述

绑定DN设置指南

固定

使用固定帐户绑定到LDAP服务，并代表绑定用户搜索要认证的用户。使用此选项，您可以设置LDAP身份验证模块，并且仍然使用不属于专有名称（DN）的登录名，例如电子邮件地址或令牌。此方法通常也称为搜索+绑定或两步身份验证 。

若要使用此方法，您需要在目录服务器上具有一个特殊帐户，该帐户有权在目录服务中查找其他用户帐户。

输入要用于LDAP绑定请求的用户帐户的完整DN。该帐户必须具有在目录服务中查找其他用户帐户的权限。

使用更改密码控件将此帐户的密码存储在集线器中。绑定用户的密码存储为纯文本值的加盐哈希。

动态

从登录名导出用户DN，并尝试直接以用户身份绑定到LDAP服务。此方法通常也称为直接绑定 。

使用查询与目录服务绑定。该查询查找要认证的用户的专有名称。用表达式引用用户名。该表达式将替换变量映射到将用户名存储在目录服务中的属性。您选择的属性确定在过滤字符串中使用哪个查询。

在登录页面上作为用户名输入的值在替换替换变量之前会被修剪。如果用户指定域，则将其丢弃。例如，用户名的值为WORKGROUP\smith修剪到smith 。要指定域，请输入域名作为静态值。例如， WORKGROUP\%u 。

属性映射

当集线器在LDAP服务中找到与过滤器匹配的记录时，它将从为每个字段指定的LDAP属性中获取值，并将其复制到集线器中的用户配置文件中。使用以下设置来定义筛选条件，并将目录服务中存储的属性映射到集线器中的用户帐户。

领域

描述

登录

必填项 。映射到LDAP属性，该属性存储要复制到中心配置文件中“ 登录”字段的值。对于Active Directory，默认值为sAMAccountName 。

除非严格要求，否则我们建议您不要更改默认值。如果您确实更改了默认值，则该字段必须指定LDAP记录的属性，该属性唯一地标识用户帐户。另外，如果更改了默认值，则在首次尝试使用LDAP凭据登录时，可能会将新的凭据记录添加到用户配置文件的“ 登录”部分。

全名

映射到LDAP属性，该属性存储值以复制到集线器配置文件中的“ 全名”字段。

电子邮件

映射到LDAP属性，该属性存储要复制到中心配置文件中“ 电子邮件”字段的值。

贾伯

映射到LDAP属性，该属性存储要复制到中心配置文件中的Jabber字段的值。

VCS用户名

映射到LDAP属性，该属性存储值以复制到集线器配置文件中的VCS用户名字段。

其他设置

以下选项位于页面底部。使用这些设置来管理中心帐户的创建，组成员身份和连接选项。

选项

描述

用户创建

为使用存储在连接的目录服务中的帐户登录的未注册用户创建中心帐户。集线器使用电子邮件地址来确定用户是否具有现有帐户。

所有LDAP身份验证模块都必须允许用户创建。如果拒绝用户创建，则向未注册的用户显示错误。

自动加入群组

当用户使用存储在连接的目录服务中的帐户登录时，将用户添加到组中。您可以选择一个或多个组。自动加入组的新用户将继承分配给该组的所有权限。
我们建议您将用户添加到至少一个组中。否则，仅授予新用户当前分配给“ 所有用户”组的权限。

连接超时

设置等待建立与授权服务的连接的时间。默认设置为5000毫秒（5秒）。

读取超时

设置等待从授权服务读取和检索用户配置文件数据的时间。默认设置为5000毫秒（5秒）。

审计

链接到中心中的“ 审核事件”页面。在这里，您可以查看已应用于此身份验证模块的更改列表。

组映射

在“ 组映射”选项卡上，可以将Active Directory服务中的现有组映射到集线器中的组。

如果你想在Active Directory服务中心组映射组，你需要指定组的属性，在设置该验证模块的属性映射部分存储LDAP组成员。

配置组映射后，当用户使用目录服务中管理的帐户登录时，集线器将检查Active Directory组成员身份。集线器对映射到集线器组的每个LDAP组执行以下操作：

  • 属于映射的Active Directory组成员而不是映射的Hub组成员的用户将添加到Hub中的组中。

  • 不是映射的Active Directory组成员的用户，而是映射的Hub组的成员的用户将从Hub的组中删除。

仅当用户使用Active Directory身份验证模块登录时，才在集线器中应用对Active Directory服务中的组成员身份所做的更改。

您可以将多个Active Directory组映射到Hub中的单个目标组。您不能将Active Directory组映射到多个集线器组。

要将Active Directory组映射到Hub中的组：

需要权限：低级管理员写

  1. 打开您的Active Directory身份验证模块。

  2. 选择“ 组映射”选项卡。

  3. 单击添加映射按钮。

    • 将打开“ 添加映射”对话框。

  4. 在LDAP组名称字段中输入Active Directory组的名称 。

    • 如果“ 绑定帐户”选项设置为“ 固定” ，则auth模块将使用目录服务中的绑定帐户查找组。可用组显示在LDAP组名称列表中。

    • 如果将绑定帐户选项设置为动态 ，则目录服务中的组列表对于绑定帐户不可用。要成功映射组，您需要输入与目录服务中显示的完全相同的组名称。

  5. 从集线器组列表中选择一个组。

  6. 单击添加按钮。

    • 映射将添加到列表中。
