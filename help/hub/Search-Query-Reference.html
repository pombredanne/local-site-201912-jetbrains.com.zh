有关集线器搜索语法的信息，请参阅“ 搜索查询语法” 。

经营者

使用以下运算符可在搜索查询中创建属性的逻辑组合。

从一组搜索查询结果中排除一个子集。

要查找没有与自己的帐户相关联的Google登录名并且直接将任何角色授予其帐户的用户，请输入：

在查询中，AND表达式始终按组处理，并且具有比OR表达式更高的优先级。

要在“死亡之星”项目中找到所有具有“低级管理员读取”访问权限的用户，或者是“ star-team”组的成员，或者具有登录名“ admin”，请在“用户”页面上输入：

布尔搜索

的has和is关键字用作布尔搜索词。在搜索查询中使用时，它们返回包含指定属性值的所有实体。使用not运算子（ - ）在指定属性之前查找具有空值的实体。

例如，要查找具有任何资源的所有受信任服务，请打开“ 服务”页面，然后输入：

有：资源是：受信任

您可以结合使用的属性has和is关键字取决于您搜索的实体的类型。有关可用属性的列表，请参见与特定实体类型相关的部分。

日期和期间值

几个搜索属性引用存储为日期的值。您可以将日期搜索为单个值，也可以使用一系列值来定义时间段。

以以下格式指定日期： YYYY-MM-DD要么YYYY-MM要么MM-DD 。您还可以指定24小时格式的时间： HH:MM:SS要么HH:MM 。要指定日期和时间，请使用以下格式： YYYY-MM-DD}T{HH:MM:SS 。例如，要查找在2017年5月26日12:00之后创建的所有帐户，请在“ 用户”页面上输入

之后：{2017-05-26T12：00}

依编号搜寻

集线器中的每个实体都有其唯一的标识符。您可以在日志，事件，URL和API请求和响应中找到实体ID。

您只能在列出相同类型实体的页面上的搜索查询中使用实体ID。即：在“ 服务”页面上通过用户帐户的ID搜索不会返回任何结果。但是，如果您使用相同的ID来过滤Users列表，则可以直接导航到具有该ID的用户的配置文件。

文字搜寻

Hub对基于文本的搜索提供了有限的支持。使用基于文本的搜索时，您无需指定要搜索的属性。而是直接输入值或部分值。

在大多数页面上，Hub返回名称以指定字符串开头的实体的列表。

对于列出用户的页面，集线器支持以下行为：

  • 输入文本字符串时，查询将返回其全名，登录名或电子邮件以指定字符串开头的所有用户。

  • 当您指定全部或部分电子邮件域时，查询将返回其电子邮件以指定字符串开头的所有用户。

您可以将基于文本的搜索字符串与使用属性和值对的任何其他搜索词结合使用。
