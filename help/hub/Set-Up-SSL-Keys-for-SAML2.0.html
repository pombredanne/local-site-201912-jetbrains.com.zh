设置SAML 2.0的SSL密钥

要将集线器用作带有SAML的身份提供程序，必须加密集线器和服务提供程序之间的连接。您需要生成SSL密钥和证书，将它们打包成PKCS12格式的文件，然后将其上传到集线器。

有几种工具可让您创建PKCS12格式的SSL密钥和证书。我们描述了如何使用OpenSSL工具箱创建SSL密钥库。

要使用OpenSSL创建SSL密钥库，请执行以下操作：

  1. 生成具有密码保护的新2048位RSA密钥：

    openssl genrsa -des3 -out Hub_SAML.key 2048

  2. 为生成的密钥生成证书请求：

    openssl req-新-key Hub_SAML.key -out Hub_SAML.csr

  3. 生成证书：

    openssl x509 -req -days 365 -in Hub_SAML.csr -signkey Hub_SAML.key -out Hub_SAML.crt

  4. 将密钥和证书打包在PKCS12文件中：

    openssl pkcs12-导出-out Hub_SAML.p12 -inkey Hub_SAML.key -in Hub_SAML.crt -certfile Hub_SAML.crt

    • 您有一个PKCS12密钥库（ Hub_SAML.p12在示例中），可以上传到集线器了。

请保留生成的SSL密钥和证书。某些服务提供商需要其实际内容才能进行SAML配置。

要将PKCS12密钥库导入到集线器：

需要权限：低级管理员写

  1. 在管理菜单的服务器设置部分中，选择SSL密钥 。

  2. 单击导入密钥库按钮。

  3. 选择创建的.p12密钥文件，提供密码，然后将密钥库上传到Hub。

  4. 在管理菜单的访问管理部分，选择SAML 2.0。

  5. 在“ 设置”选项卡上的“ SSL密钥”下拉列表中选择密钥。

集线器使用选定的SSL密钥加密与SAML服务提供商的连接。集线器还提取并显示打包到所选密钥库中的证书的SHA-256，SHA-1和MD-5 指纹 。

服务提供商可能需要这些指纹中的任何一个才能在其端启用和配置SAML2.0。
