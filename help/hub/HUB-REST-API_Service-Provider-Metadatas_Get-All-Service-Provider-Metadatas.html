Hub 2019.1帮助

获取所有服务提供商元数据

请求

GET /saml2/serviceproviders?fields=string&$skip=int&$top=int&query=string&orderBy=string

参数

描述

fields

可选的 。仅返回每个ServiceProviderMetadata字段的指定子集。使用字段语法定义子集。

$skip

可选的 。使您可以设置许多ServiceProviderMetadata，以便在返回第一个之前跳过。

$top

可选的 。使您可以设置要返回的ServiceProviderMetadatas的最大数量。

query

可选的 。用于指定搜索查询以过滤返回的结果。有关支持的逻辑操作语法的定义，请参见查询语法。对于ServiceProviderMetadatas，支持以下字段和元组：

name ： serviceProviderName —具有给定名称的服务提供商

samlEntityID ： serviceProviderEntityId —具有给定实体ID的服务提供商

support： 注销响应 — 支持 注销响应的所有服务提供商

url ： assertionConsumerUrl —具有断言使用者URL的服务提供者

orderBy

可选的 。返回按指定字段排序的ServiceProviderMetadatas列表。有关详细信息，请参见排序语法 。ServiceProviderMetadatas支持按以下字段排序： entityId ， name

响应

回应码

含义

200 OK

成功的请求。

400错误的要求

至少一个请求参数无效。例如，缺少传递的JSON对象中的必填字段。有关详细信息，请检查响应中的错误消息。

403禁止

请求者无权访问所请求的资源。

找不到404

找不到请求的资源。

500内部服务器错误

由于服务器错误，无法处理请求。有关详细信息，请检查响应中的错误消息。

反应体

{ "skip": int, "top": int, "total": int, "items": [serviceProviderMetadata, ...] }
