恢复Docker安装

如果安装集线器时遇到问题，则可能需要还原到最新的工作版本。需要还原安装的常见方案包括：

  • 您无法成功升级安装，需要还原以前安装的版本。

  • 安装的数据库中断，您需要从数据库的备份副本中还原当前版本。

在这两种情况下，您都可以使用Hub数据库的有效备份来还原安装。如果备份已过期，则可以尝试使用配置和数据文件的有效副本来还原安装。仅当您具有有效备份或数据库副本未损坏时，才可以进行还原。

您应该定期测试此过程，以确保在发生故障或攻击后可以使用备份文件来还原安装。

先决条件

要成功地从备份还原数据库，备份文件的版本应与将数据库还原到的Hub的版本相同或更早。

在开始之前，请确保您具有要还原的数据库的备份副本。请注意备份副本的位置，因此请确保在此过程中选择正确的文件。

  • 如果升级失败后回滚到以前的安装版本，请使用在安装之前创建的备份。如果忽略此先决条件，请找到与先前安装的版本相对应的备份文件。

  • 如果尝试还原当前安装，请在设置为与当前安装版本相对应的“ 备份”位置的目录中找到已存档的备份文件。

从数据库的备份副本还原安装

要将数据库还原到当前或先前版本，需要重新安装Hub并将备份用作安装期间的升级源 。

还原到Hub的早期版本时，请使用与要还原的产品版本相对应的备份。

要从备份还原数据库：

需要权限：低级管理员写

  1. 停止集线器服务。有关详细信息，请参阅启动和停止Docker容器 。

  2. 清洁内部物品data和conf目录。您需要在主机上清除这两个特定于 Hub的目录的内容。有关更多详细信息，请参阅安装说明的“ 创建和配置目录”一章。

  3. 确保要还原的备份文件位于backups 主机上的 Hub目录。用户还必须可以访问备份文件13001:13001在容器内运行集线器服务。例如，在Linux主机上，可以使用以下命令来确保访问权限：

    chmod 750 弦13001：13001

    运行docker映像时，需要将此目录映射到容器内的相应目录-/opt/hub/backups

  4. 执行以下命令以使用集线器服务器运行容器：

    docker run -it --name \ -v ：/ opt / hub / data \ -v ：/ opt / hub / conf \ -v ：/ opt / hub / logs \ -v ：/ opt / hub / backups \ -p ：8080 \ jetbrains / hub：

    有关运行Hub容器和命令参数的更多详细信息，请参阅Run Hub Docker Container 。

    • 集线器映像将启动基于Web的配置向导。在浏览器中，打开控制台输出中显示的“配置向导”的URL。

  5. 在配置向导中 ，选择升级选项。

  6. 在“ 选择升级源”页面上，选择备份作为升级源 ，然后单击“ 下一步”按钮。

  7. 查看集线器服务器设置，然后单击“ 升级”按钮。

    • 集线器服务从备份文件中的数据开始。
