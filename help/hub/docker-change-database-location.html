更改Docker的数据库位置

集线器存储数据的默认目录包含以下子目录：

名称

描述

hub

包含中心服务的数据库

internal

可能包含其他内部产品数据

将现有数据库复制到新位置时，请确保新位置使用此处显示的相同文件夹结构。

由于集线器服务在docker容器内运行，因此您只能更改数据库在主机上的位置。在这种情况下，您需要将数据库移动到新位置，然后将新路径指定为docker run命令。

  1. 备份数据库。以防万一。

  2. 停止Hub docker容器。有关特定说明，请参阅Stop Hub docker container 。

  3. 为数据库创建一个新目录。

  4. 移动当前数据库目录的内容（ data ）到新目录。

  5. 如果您在macOS上运行Hub docker，请跳过此步骤。

    提供用户13001:13001在容器内运行集线器服务，并具有对新数据库目录的访问权限。有关更多详细信息，请参见“ 创建和配置目录”页面。
  6. 使用指向主机上数据库的新路径运行docker容器：
    docker run -it --name \ -v ：/ opt / hub / data \ -v ：/ opt / hub / conf \ -v ：/ opt / hub / logs \ -v ：/ opt / hub / backups \ -p ：8080 \ jetbrains / hub：

    • 集线器服务从容器内的数据目录开始（ /opt/hub/data ）映射到主机上的新数据库位置。

用相同的方法可以更改conf ， logs ， backups和temp目录。
