获取所有身份验证模块

请求

GET /authmodules?fields=string&$skip=int&$top=int&query=string&orderBy=string

参数

描述

fields

可选的 。仅返回每个AuthModule的指定字段子集。使用字段语法定义子集。

$skip

可选的 。使您可以设置多个AuthModule，以便在返回第一个之前跳过。

$top

可选的 。可让您设置要返回的最大AuthModules数。

query

可选的 。用于指定搜索查询以过滤返回的结果。有关支持的逻辑操作语法的定义，请参见查询语法。对于AuthModule，支持以下字段和元组：

有 ： 用户 -有关联的用户身份验证提供者

id ： value —具有给定ID的Auth模块

is ： 禁用 -所有禁用的身份验证提供程序

name ： authModuleName —具有给定名称的Auth模块

serverUrl ： authModuleServerURL —使用给定服务器URL的Auth模块

type ： authModuleType —给定类型的Auth模块

user ： user —提供给定用户的用户详细信息的所有身份验证模块

orderBy

可选的 。返回按指定字段排序的AuthModules列表。有关详细信息，请参见排序语法 。AuthModules支持按以下字段排序： name ， ordinal

响应

回应码

含义

200 OK

成功的请求。

400错误的要求

至少一个请求参数无效。例如，缺少传递的JSON对象中的必填字段。有关详细信息，请检查响应中的错误消息。

403禁止

请求者无权访问所请求的资源。

找不到404

找不到请求的资源。

500内部服务器错误

由于服务器错误，无法处理请求。有关详细信息，请检查响应中的错误消息。

反应体

{ "skip": int, "top": int, "total": int, "authmodules": [authmodule, ...] }
