SMTP

在“ SMTP设置”页面上，配置集线器和SMTP服务器之间的连接。这样可以通过电子邮件在Hub中进行通知。

如果SMTP连接被禁用或配置不正确，您的用户将不会收到来自集线器的任何消息 。其中包括邀请注册帐户，联系验证电子邮件和密码恢复说明！

要启用电子邮件通知：

需要权限：低级管理员写

  1. 单击配置SMTP服务器按钮

  2. 输入以下设置的值：

    设置

    描述

    协议

    选择您的SMTP服务器使用的协议。

    主办

    输入您的SMTP服务器的服务器地址。

    港口

    输入SMTP服务器的连接端口。

    电子邮件“发件人”地址

    输入电子邮件地址，用作电子邮件通知的“发件人”地址。“发件人”地址包含两个部分。第一部分是发件人名称。第二部分是电子邮件地址本身。要指定两个组件的值，请使用以下格式"sender name" 。
    在许多电子邮件客户端中，仅显示发件人名称。其他人则丢弃发件人姓名，仅显示电子邮件地址。如果您未指定发件人名称的值，则电子邮件地址也会在电子邮件客户端中显示为发件人名称。

    用户名

    输入用于访问SMTP服务器的帐户的用户名。

    如果您要连接到Gmail，则可能需要遵守其他安全要求。有关详细信息，请参阅“连接到Gmail的特殊要求” 。

    密码

    单击“ 更改密码”按钮，然后在“ 更改密码”对话框中输入用于访问SMTP服务器的帐户的密码 。单击更改按钮以保存密码并关闭对话框。

    SSL密钥

    选择一个SSL密钥以加密集线器和SMTP服务器之间的连接。有关详细信息，请参见SSL密钥 。

  3. 完成后，保存更改，然后单击“ 启用通知”按钮。

  4. 测试连接。

    • 单击发送测试消息按钮。

    • 输入测试消息应发送到的电子邮件地址。

    • 单击发送按钮。

连接到Gmail的特殊要求

Google帐户的设置限制了对使用安全性较低的登录技术的第三方应用程序的访问权限。根据其定义，这适用于使用用户名和密码进行身份验证的连接，这些连接容易被劫持。

设置要求基于管理安全性较低的应用程序的当前设置。这些要求适用于您用来向Gmail服务器进行身份验证的Google帐户。

如果您要连接自己管理的Google帐户，则可以使用以下选项：

  • 在您的Google帐户中生成一个应用专用密码（应用密码），并将其用于身份验证。离开启用对您的谷歌不够安全的应用访问帐户关闭的选项。

    应用密码只能用于启用了两步验证的帐户。

  • 为您的Google帐户启用“ 不太安全的应用访问权限” ，并使用您的用户名和密码进行连接（不建议使用）。

如果您使用由您的组织管理的Google帐户作为G Suite的一部分，请验证您的域当前启用了以下哪些设置。

设置

要求

禁止所有用户访问安全性较低的应用

用于身份验证的Google帐户必须启用两步验证。要使用此帐户连接到Gmail服务器，请在您的Google帐户中生成一个应用程序专用密码（应用密码）。

  • 对于用户名，输入您的Google帐户的用户名。

  • 对于密码 ，输入您在Google中生成的应用专用密码。

允许用户管理对不太安全的应用程序的访问

您也许可以使用您的Google用户名和密码连接到Gmail服务器。此设置仅在以下情况下可用：

  • 您已启用该选项，以允许访问用于连接到Gmail服务的Google帐户中安全性较低的应用程序。

  • 您尚未为此Google帐户启用两步验证。

如果您为此Google帐户启用了两步验证，则需要生成并使用应用密码。在集线器中配置SMTP设置，就像已禁用对安全性较低的应用程序的访问一样。
