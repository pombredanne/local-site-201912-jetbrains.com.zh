Hub 2019.1帮助

部分要求

向Hub REST API发出请求时，可能会提取所有Hub对象的完整图形。通常，您只是不需要这么多的数据。

默认情况下，Hub API仅返回所请求对象的简单字段（例如，字符串，整数，布尔值）以及由其ID，URL等表示的对象字段。要自定义要在响应中获取的对象字段的子集，请使用类似Google API的部分请求 。

让我们来看一个示例中的部分请求如何在Hub API中工作。

让我们请求一个用户帐户的参数：

GET https://hub.example.com/rest/users/794faf72-ba29-447e-8984-3442207cbebb

由于请求不包含自定义响应的参数，因此Hub会在响应正文中返回所有参数：

{“ id”：“ 794faf72-ba29-447e-8984-3442207cbebb”，“ url”：“ https://hub.example.com/rest/users/794faf72-ba29-447e-8984-3442207cbebb”，“名称” ：“ admin”，“禁止”：false，“头像”：{“ pictureUrl”：“ http://www.gravatar.com/avatar/cc90309659114c3979c9b469f44252e7.jpg&?d=monsterid&s=60”}，“角色”：[ {“ id”：“ 12f8c35a-ef03-4304-869d-690315c690e8”，“ role”：{“ id”：“ 2d2ae9db-365f-486e-bac8-021af86f1444”，“ url”：“ https：//hub.example .com / rest / roles / 2d2ae9db-365f-486e-bac8-021af86f1444“，”名称“：”系统管理员“}，”项目“：{” id“：” 0“，” url“：” https：// hub.example.com/rest/projects/0“}}]]，” contacts“：[{” type“：” emailJSON“，” verified“：false，” email“：” admin@jetbrains.com“}]， “ groups”：[{“ id”：“ f4495122-993f-4005-b6be-7823f3bcac2d”，“ url”：“ https://hub.example.com/rest/usergroups/f4495122-993f-4005-b6be-7823f3bcac2d “}]，” details“：[{” type“：” emailuserdetailsJSON“，” id“：” 06cfc5b5-4cce-4aa7-8652-fb9ac1c920ff“}]}

为了自定义返回的子集，我们添加了fields请求的参数：

GET http://hub.example.com/rest/users/794faf72-ba29-447e-8984-3442207cbebb?fields=name,groups/name,contacts(verified,email,jabber）

在这种情况下，响应主体仅包含用户帐户的指定字段：

{“ name”：“ admin”，“ contacts”：[{“ type”：“ emailJSON”，“ verified”：false，“ email”：“ admin@jetbrains.com”}]，“ groups”：[{名称”：“新用户”}]}
