单步执行程序

步进是控制程序逐步执行的过程。

IntelliJ IDEA提供了一组步进操作，具体取决于您的策略使用（例如，您是否需要直接进入下一行或在此途中输入调用的方法）。

步进按钮位于“ 调试”工具窗口工具栏上。

越过

即使当前突出显示的行中包含方法调用，也将单步执行到当前代码行并带您到下一行。方法的实现将被跳过，您将直接移至调用者方法的下一行。

  • 点击“ 跳过”按钮或按F8 。

在该示例中，将执行第5行。如果您跨步，调试器将直接移至第6行，而无需跳入doCount（）方法。

如果跳过的方法中有断点，调试器将在它们处停止。如果需要跳过途中的任何断点，请使用“ 强制跨过” 。

步入

逐步进入该方法以显示其中发生的情况。如果不确定方法是否返回正确的结果，请使用此选项。

  • 单击单步执行按钮或按F7 。

在该示例中，将执行第5行。如果您进入，调试器将跳转到count(int to)该方法使您可以详细检查其结果是如何产生的。

如果线路上有多个方法调用，则IntelliJ IDEA会询问您要输入哪种方法。此功能称为“ 智能步进” 。

您可以将Smart Step Into配置为在线路上有多个方法调用时自动使用。或者，仅当您明确地调用时，才可以调用它。要配置此功能，请转到“设置/首选项” |“设置”。构建，执行，部署|调试器步进并设置始终执行所需的智能步入选项。

一些方法（例如标准Java类的方法，例如System ）会被Step跳过，因为您通常不需要调试它们。此列表可以在“ 构建，执行，部署” |“发布”中进行调整。 调试器步进页面的 设置/首选项对话框Ctrl+Alt+S 。

明智地进入

当一行上有多个方法调用，并且您想具体说明要输入哪种方法时， 智能步入很有用。此功能使您可以选择感兴趣的方法调用。

  1. 从主菜单中，选择运行|智能进入或按Shift+F7 。

  2. 单击方法或使用箭头键选择它，然后按Enter / F7 。

每次在线上有多个方法调用时，都可以配置要使用的智能步进而不是常规步进 。这是在“设置/首选项” |构建，执行，部署|调试器踩 。

走出去

退出当前方法，然后转到调用者方法。

  • 点击“ 跳出”按钮或按Shift+F8 。

在此示例中，单步执行会跳过循环的所有迭代，并直接转到main方法（调用方）。

运行到光标

继续执行直到插入符号的位置。

  1. 将插入号放在您要程序暂停的行上。

  2. 单击运行到光标按钮或按Alt+F9 。

另外，您也可以单击运行游标 。为此，请单击装订线中的行号。

您可以配置是否要运行“游标”以单击“设置/首选项” |“行”中的行号。 构建，执行，部署|调试器 。

在该示例中， “运行到游标”将继续执行并在第7行停止，就好像有一个断点一样。如果在count方法，程序将在那里暂停。

如果需要跳过途中的任何断点，请使用“ 强制运行至光标” 。

强制进入

即使常规Step Into 跳过了此方法，也要执行该方法中的步骤 。

  • 单击“ 强制进入”按钮或按Shift+Alt+F7 。

在该示例中，执行将在即将调用的行之前暂停System.out.println() 。

借助Force Step Into ，我们可以直接进入System.out.println()方法，而常规的“ 进入”则将我们带到第20行。

强制运行到光标

继续执行直到插入符号的位置。途中的所有断点都将被忽略。

  1. 将插入号放在您要程序暂停的行上。

  2. 从主菜单中，选择运行|强制运行至光标或按Ctrl+Alt+F9 。

在该示例中，“ 强制运行到光标”将继续执行并在第7行停止，就好像有一个断点一样。内部的断点count不会有任何效果。

强制越过

即使当前突出显示的行中包含方法调用，也将单步执行到当前代码行并带您到下一行。如果调用的方法中有断点，则将其忽略。

  • 从主菜单中，选择运行|强制跨步或按Shift+Alt+F8 。

在此示例中，即使在程序中存在断点，单步执行仍会将您带到第6行的打印语句。 count方法，否则将在循环的所有迭代中暂停应用程序。

落架

允许您撤消最后一帧并恢复堆栈中的前一帧。例如，如果您错误地走得太远，或者想重新进入错过关键点的功能，这可能会很有用。

请注意，此选项仅影响局部变量，不会恢复程序的整体状态，因为它不会还原静态变量和实例变量的值。这可能会导致程序流程更改。

  • 单击“ 落帧”按钮 。

在示例中，放下框架会使您返回到调用方方法，就像count从未执行过。

没有受影响的静态或实例变量，但是控制台输出保持不变。

提高步进速度

调试器功能会占用资源，并可能影响步进性能。如果性能不能令人满意，请按照本章中提供的建议进行优化。

  • 使用开销功能来确定性能下降的原因。

  • 如果您的项目不需要以下功能，请禁用它们或将其最小化：

    • 方法断点和字段观察点

    • 显示方法返回值选项

    • “ 集合类 ”的“ 备用视图”选项（ “设置/首选项” |“构建”，“执行”，“部署” |“调试器” |“数据视图” |“ Java” ）

    • 'toString'对象视图 （ 设置/首选项|构建，执行，部署|调试器|数据视图| Java ）

  • 简化断点和观察点的条件，尤其是经常遇到的断点 。

  • 在调试会话期间，切换到包含较少元素的视图。
