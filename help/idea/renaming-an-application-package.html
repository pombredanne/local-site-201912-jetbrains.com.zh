重命名应用程序包（应用程序ID）

您可能需要将应用程序构建为多个版本，这意味着将生成多个Android应用程序包（ .apk文件）。如果这些文件具有相同的名称，则用户将无法同时将它们部署在同一设备上。为避免这种情况，您可以让IntelliJ IDEA从相同的源代码生成几个具有不同名称（ 应用程序ID ）的.apk文件。

应用程序包名称（ application ID ）在package的属性manifest元素（有关详细信息，请参见http://developer.android.com/guide/topics/manifest/manifest-element.html#package ）。此名称遵循Java命名约定，并且默认情况下与为应用程序实现的类所属的包的名称相同。

应用程序的名称（ android:name的属性application元素，请参见http://developer.android.com/guide/topics/manifest/application-element.html#nm ）和活动名称（ android:name的属性activity元素，请参见http://developer.android.com/guide/topics/manifest/activity-element.html#nm ），默认情况下，是相对于应用程序ID以及相对于应用程序的父Java包指定的实现类。但是，重命名应用程序ID不会导致重命名应用程序类的父Java包。

您可以通过重命名重构来更改应用程序ID，也可以在构建时自动更改。

通过重命名重构来更改应用程序ID

的价值package属性更改为新指定的值。但是，因为这不会导致重命名应用程序类的父Java包，所以应用程序和活动的相对名称将替换为其完全限定的名称。

  1. 打开AndroidManifest.xml文件。

  2. 将光标放在package的属性manifest元素，然后选择重构|从上下文菜单中重命名 。

  3. 在打开的“ 重命名”对话框中，指定新的程序包名称，然后单击“ 确定” 。

在构建时重命名应用程序ID

  1. 通过选择文件|打开项目结构对话框。从主菜单的项目结构 。

  2. 在左窗格中选择“ 模块 ”。在中央窗格中，展开相关模块的节点，然后单击Android 。

  3. 在右窗格中，切换到“ 打包”选项卡，然后选择“ 重命名清单包”选项。
