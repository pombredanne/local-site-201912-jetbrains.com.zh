使用Android UI设计器添加和编辑布局组件

布局定义活动或应用程序小部件 （片段）的用户界面。布局在XML 资源定义文件中声明。有关如何创建资源文件夹和资源定义文件的说明，请参见创建资源 。

使用IntelliJ IDEA，您可以无需手动编辑布局定义文件即可构建应用程序的设计，并检查如何在各种目标环境中呈现应用程序设计，而无需在任何物理或虚拟设备上运行应用程序。

或者，可以手动编辑布局定义文件（可能使用IntelliJ IDEA提供的特定于Android的重构），并预览立即反映在专用“ 预览工具”窗口中的更改 ，您可以在其中调整布局以适应各种平台和设备。要切换到手动模式，请单击“ 文本”选项卡，或从“ 设计”窗格的上下文菜单中选择“ 转到声明” ，或单击“ 预览窗口工具栏中的图标 。

在可视模式下设计布局是在Android UI设计器工具窗口的“ 设计”窗格中执行的。窗格位于UI设计器的中心部分（假定使用默认的工具窗口布局）。当您打开布局定义文件进行编辑时，默认情况下该窗格显示在编辑器选项卡中。如果您要手动编辑布局定义文件，然后通过单击“ 设计”选项卡切换到可视模式，则窗格将在该选项卡中打开，其中已编辑的布局定义文件被打开，

该窗格显示了一个矩形画布，该画布与当前布局定义文件和“ 组件树”视图同步，因此对画布的任何更改都将在此反映出来。如果IntelliJ IDEA检测到代码中有任何差异，则会显示警告。

请注意，可以在设计模式下以与手动编辑布局定义文件时相同的方式使用意向操作和快速修复。界面动作图标或快速修复图标在画布上，“ 组件树”或“ 属性”窗格中显示。

要构建Android应用程序的设计，请执行以下基本操作：

  • 从面板添加预定义的组件
  • 添加用户定义的组件以及Android SDK中的组件
  • 安排组件
  • 指定组件属性
  • 将组件转换为保留通用属性的其他类型
  • 预览版面

从面板添加预定义的组件

请执行以下任一操作：

  • 在“ 面板”窗格中选择所需的元素，然后将其拖放到“ 设计”窗格中的画布上。

  • 单击“ 调色板”窗格中的所需元素，然后单击画布上的一个区域。

  • 单击“组件面板”窗格中的必需元素，然后单击要在其中放置此元素的组件树 。

以任一种方式添加的每个组件也将添加到“ 组件树”中，并在布局定义文件中声明。

添加用户定义的组件以及Android SDK中的组件

  1. 展开“ 面板”窗格中的“ 自定义”区域。

  2. 请执行以下任一操作：

    • 要嵌入布局，请单击“ 包含”图标。在打开的“ 资源”对话框中，单击“ 项目”选项卡以在布局定义中进行搜索，或者单击“ 系统”选项卡以在SDK中进行搜索。

    • 要添加组件的组合，请单击“ 片段” ，然后选择要嵌入到打开的“ 资源”对话框中的片段。

    • 要添加用户定义的视图 ，请单击CustomView并选择要在打开的对话框中插入的视图。

安排组件

画布与Component Tree同步，因此您可以通过在任一窗格的任一组件中移动它们来安排组件。

在画布上或“ 组件树”中选择所需的组件，然后将其拖动到正确的位置，或者使用上下文菜单将其复制并粘贴。

指定组件属性

您可以在画布上直接设置必需组件属性的值，或切换到“ 属性”窗格以进行更深入的配置。

  • 要为画布中的基本属性分配值，请双击有问题的组件，然后在出现的弹出窗口中指定值。

  • 要对组件属性进行深入配置，请在画布或“ 组件树”中选择组件 ，切换到“ 属性”窗格，然后指定所选属性的值。

    单击“ 属性”窗格中的右列，以开始编辑属性。您可以点击出现在选择或投影或系统资源的右侧。

    • 默认情况下，该窗格仅显示一组标准属性，而最常用的属性以粗体显示 。要使窗格显示根据规范为所选组件定义的所有属性，请单击“ 显示专家属性”按钮在工具栏上。

    • 要从Android参考中查看所选属性的简要文档，请单击显示文档按钮在工具栏上或按Ctrl+Q 。

    • 具有更新值的属性以蓝色突出显示。要放弃所做的更改并返回默认值，请选择属性，然后单击“ 恢复默认值”按钮。 在工具栏上。

将组件转换为保留通用属性的其他类型

在某些情况下，您可能需要将已经完全配置的组件转换为另一种类型的组件。使用IntelliJ IDEA，您可以做到这一点而不会丢失指定的属性：这两种类型共有的所有属性都将保留在新组件中。此操作称为“ 变形” 。

  1. 在画布上或“ 组件树”中选择要转换的组件，然后在所选内容的上下文菜单上选择“ 变形 ”。

  2. IntelliJ IDEA显示兼容组件类型的列表，即所选组件可以转换成的类型。选择目标类型并配置原始组件中未配置的属性 。
