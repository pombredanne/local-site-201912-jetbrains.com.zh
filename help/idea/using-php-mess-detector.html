PHP混乱检测器

仅当安装并启用PHP插件时，以下内容才有效。

除了内置的编码帮助外，IntelliJ IDEA还通过与PHP Mess Detector工具集成来检查源代码，该工具检测与代码大小，不一致，未使用的代码，违反命名约定，不良设计等相关的潜在问题 。 。

要使用IntelliJ IDEA而不是命令行使用PHP Mess Detector，您需要在IntelliJ IDEA中注册它并将其配置为IntelliJ IDEA 代码检查 。在IntelliJ IDEA中安装并启用该工具后，该工具将在任何打开的PHP文件中可用，并且不需要其他步骤即可启动它。文件中的每次更新都会激活即时代码检查，因此很容易消除发现的问题。

PHP Mess Detector即时报告的错误和警告将显示为弹出消息。当该工具以批处理模式运行时，错误和警告将显示在“ 检查结果”工具窗口中。每个消息都有phpmd前缀以将其与IntelliJ IDEA内部检查区分开。

您可以应用预定义的规则，也可以定义自己的自定义规则集 。

在你开始前

确保已启用PHP插件。 的该插件未与IntelliJ IDEA捆绑在一起，但可以如管理插件中所述从JetBrains插件存储库安装。

安装和配置PHP Mess Detector

PHP Mess Detector脚本可用作本地脚本，与PHP解释器关联的脚本，或者声明为项目依赖项并通过Composer安装的脚本，这是推荐的推荐方法。

代码质量工具脚本包含对系统范围的PHP引擎的调用。为了能够使用它们，请确保将包含PHP引擎可执行文件的目录添加到系统中path 。

使用Composer安装PHP Mess Detector

在你开始前， 确保Composer已安装在计算机上并按照Composer Dependency Manager中所述在当前项目中初始化。

当您在Composer中安装PHP Mess Detector时，IntelliJ IDEA会自动下载必要的脚本并将其注册到IDE中。

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择phpmd / phpmd软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

手动配置PHP Mess Detector

您可以使用手动下载的本地PHP代码质量工具脚本或与PHP解释程序关联的脚本。可以有许多本地和远程PHP解释器，在Settings / Preferences对话框的PHP页面上指定的一个解释器被认为是Project Default 。在配置远程PHP解释器或配置本地PHP解释器中了解有关配置PHP解释器的更多信息。

选择要使用的PHP Mess Detector脚本

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ 混乱检测器”区域。在配置列表中， 选择PHP Mess Detector脚本：

    • 要使用与特定的远程PHP解释器关联的脚本，请选择该解释器的名称。

    • 要使用本地脚本，请选择Local 。在这种情况下，无论项目中使用哪种PHP解释器（本地或远程），都将执行本地PHP Mess Detector。请注意，PHP Mess Detector只能有一个本地配置，因为IntelliJ IDEA运行的脚本（对于Windows为phpmd.bat或对于Linux和macOS为phpmd ）包含了指向PHP引擎的路径。

    • 要使用与默认项目解释器关联的脚本，即在“设置/首选项”对话框的PHP页面上选择的脚本 ，请选择“默认项目解释器” 。

配置本地PHP Mess Detector脚本

  1. 下载并安装PHP Mess Detector脚本。

    要检查PHP Mess Detector的安装，请切换到安装目录并运行以下命令：

    phpmd --version

    如果该工具可用，您将收到以下格式的消息：

    PHPMD版本

    要根据您自己的自定义编码标准检查代码，请创建它 。将规则和指向它们的ruleset.xml文件存储在规则集根目录中。

  2. 在IntelliJ IDEA中注册本地PHP Mess Detector脚本：

    • 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

    • 在打开的“ 质量工具”页面上，展开“ 混乱检测器”区域，然后单击 配置列表旁边。

    • 在打开的PHP Mess Detector对话框中，在PHP Mess Detector路径字段中指定phpmd.bat或phpmd PHP Mess Detector可执行文件的位置。手动输入路径或单击然后在打开的对话框中选择相关文件夹。

      要检查phpmd.bat或phpmd的指定路径是否可以确保IntelliJ IDEA和PHP Mess Detector之间的交互，即可以从IntelliJ IDEA启动该工具，并且IntelliJ IDEA会从中接收到问题报告，请单击Validate按钮。此验证等同于运行phpmd --version命令。如果验证成功通过，则IntelliJ IDEA将显示有关检测到的PHP Mess Detector版本的信息。

配置与PHP解释程序关联的PHP Mess Detector脚本

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ 混乱检测器”区域，然后单击 配置列表旁边。将打开“ PHP Mess Detector”对话框，在左侧窗格中显示所有已配置的PHP Mess Detector脚本的列表，其中之一的类型为Local ，其他名称则与与脚本相关联的PHP解释器命名。请点击在工具栏上。

  3. 在打开的“ 通过远程解释器检查PHP混乱”对话框中，选择远程PHP解释器以使用关联的脚本。如果列表不包含相关的解释器，请单击并在CLI解释器对话框中配置远程解释器 ，如配置远程PHP解释器中所述 。

    当您单击OK时 ，IntelliJ IDEA将带您返回到PHP Mess Detector对话框，在该对话框中将新的PHP Mess Detector配置添加到列表中，并且右侧窗格显示了所选的远程PHP解释器，以及与之关联的PHP Mess Detector的路径。以及高级PHP Mess Detector选项。

配置高级PHP Mess Detector选项

IntelliJ IDEA允许您指定高级PHP Mess Detector选项，从而根据计算机的配置和所使用的规则集来微调PHP Mess Detector的进程行为。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ 混乱检测器”区域。在“ 每个文件的最大邮件数”字段中，设置文件要报告的邮件总数的上限。执行检查后，所有超过此限制的消息将被拒绝，并且IntelliJ IDEA将在编辑器选项卡顶部或“ 检查结果工具”窗口中显示警告消息。

  3. 在“ 工具进程超时”字段中，指定您希望IntelliJ IDEA等待PHP Mess Detector的结果的时间，然后终止进程以防止CPU和内存使用过多。

将PHP Mess Detector配置为IntelliJ IDEA检查

配置PHP Mess Detector检查

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在编辑器下单击检查 。

  2. 在打开的“ 检查”页面上，展开PHP |。质量工具节点，然后选中PHP Mess Detector验证旁边的复选框。

  3. 在右侧窗格中，使用“ 选项”区域中的控件来配置PHP Mess Detector工具：

    • 从“ 严重性”列表中，选择“ PHP混乱检测器”检查的严重性等级 。选择的值确定IntelliJ IDEA对检测到的差异的严重程度，并显示在检查结果中 。

    • 从“ 范围”列表中，选择将检查应用程序限制到的范围 。

    • 指定要应用的规则。

      • 要使用预定义规则 ，请在“ 选项”区域中，选中要执行的验证旁边的复选框。

      • 要使用自定义规则集：

        1. 创建并保存一个或几个规则集文件 。有效的规则集文件是.xml文件，其中包含根元素与name属性。有关自定义规则集的更多详细信息，请参见http://phpmd.org/documentation/creating-a-ruleset.html 。

        2. 在“ 自定义规则集”区域中，单击然后在打开的对话框中选择相关的规则定义文件。单击“ 确定”时 ，新项目将添加到“ 自定义规则集”列表中，“ 名称”字段显示从属性检索到的规则集名称name在标签，“ 文件”字段显示所选规则集文件的位置。

如果未在其中指定规则集scripts在composer.json的部分中，IntelliJ IDEA将另外检查项目根目录以找到具有phpmd.xml默认名称的规则集。如果文件存在，它将被自动选择为检查的“ 自定义”规则集。

与团队共享自定义编码风格

  1. 将您的编码标准的根目录放在项目根目录下。

  2. 将PHP Mess Detector配置为IntelliJ IDEA检查 。

  3. 指定您的编码标准 。

  4. 确保在“ 检查”页面顶部选择了项目配置文件 。此类配置文件保存在特定项目的.idea目录中（例如， $ PROJECT_DIR / .idea / inspectionProfiles ）。有关详细信息，请参见配置配置文件 。

  5. 在“ 设置/首选项”对话框的“ 版本控制”页面上，确保将.idea目录置于版本控制下 。

以批处理模式运行PHP Mess Detector

  1. 通过选择Code | 运行检查 。从主菜单检查代码 。指定检查范围和配置文件。

  2. 在“ 检查结果工具”窗口中查看检查结果。PHP Mess Detector报告的错误和警告以开头phpmd区别于IntelliJ IDEA内部检查。

从PHP Mess Detector检查中排除文件

当等待“混乱检测器”响应超过“ 混乱检测器”对话框中“ 工具进程超时”字段中指定的限制时，IntelliJ IDEA建议将文件添加到忽略列表 。

管理被忽略的文件

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开“ 混乱检测器”区域，然后单击“ 显示忽略的文件”链接。

    • 要添加文件，请单击并在打开的对话框中找到所需的文件。

    • 要从列表中删除文件并让Mess Detector再次处理它，请选择该文件并单击 。

    • 要从列表中删除所有文件，请单击“ 清理列表”按钮 。
