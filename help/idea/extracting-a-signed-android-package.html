使用向导生成签名发布APK

要在物理设备上部署和运行Android应用程序，您需要对应用程序进行数字签名 。使用IntelliJ IDEA，您可以在提取包时使用现有的发行密钥对Android应用程序包（ .apk文件）进行签名。IntelliJ IDEA还集成了发行密钥生成工具，可以在打包过程中调用该工具。生成的密钥保存在密钥库二进制文件中。

您可以根据需要拥有任意数量的密钥库文件和密钥，并使用现有密钥，或者在现有密钥库中创建新的密钥库，甚至创建新的密钥库。

如果使用“ 生成签名的APK向导” ，则IntelliJ IDEA会在提取时对包进行签名 。

在此页：

  • 使用向导提取并签名Android应用程序包
  • 生成新的发行密钥

使用向导提取并签名Android应用程序包

  1. 从主菜单中，选择Build |生成签名APK 。生成签名的APK向导启动。

  2. 在第一页上 ，指定要使用的发布密钥以及包含它的密钥库文件。请执行以下任一操作：
    • 要使用现有密钥库文件中的密钥对软件包进行签名：

      1. 在“ 密钥存储路径”文本框中指定文件位置。手动输入路径或单击“ 选择现有”按钮，然后在打开的对话框中选择文件。在“ 密钥库密码”文本框中，键入所选密钥库的密码。

      2. 指定密钥别名，然后输入密码以访问密钥。

    • 要在现有密钥库中生成新密钥：

      1. 在“ 密钥存储路径”文本框中指定文件位置。手动输入路径或单击“ 选择现有”按钮，然后在打开的对话框中选择文件。在“ 密钥库密码”文本框中，键入所选密钥库的密码。

      2. 单击“ 创建新按钮”，并通过在打开的“ 新密钥存储”对话框中填写数据来配置要生成的发行密钥 。

    • 要使用新密钥生成新的密钥库文件：

      1. 单击创建新按钮。在打开的“ 新密钥库”对话框中，在“ 密钥库路径”文本框中指定要生成的文件的位置。手动输入路径或单击浏览 ，然后选择父文件夹并指定文件名。

      2. 指定并确认密码以访问密钥库。

      3. 通过在“ 密钥”区域中填写数据来配置新的发行密钥 。

    单击下一步 。

  3. 下一步 ，在“ 目的地APK路径”字段中，指定将保存.apk的文件夹。手动输入路径，或单击浏览 然后在打开的对话框中选择目标文件夹。

  4. 要使IntelliJ IDEA在打包过程中混淆应用程序，请选中“ 运行ProGuard”复选框，然后指定proguard.txt配置文件的位置。 该文件是在项目创建时生成的，并存储在项目根目录中。IntelliJ IDEA在“ 配置文件路径”文本框中建议此默认位置。接受建议或通过单击浏览指定自定义配置文件然后在打开的对话框中选择所需的文件。

  5. 单击完成以生成并签名包。IntelliJ IDEA会通知您操作成功完成。

    • 要打开生成的.apk文件所在的文件夹，请单击“ 打开文件位置”按钮。

    • 要完成向导，请单击“ 关闭” 。

生成新的发行密钥

  1. 通过执行以下操作之一打开“ 新密钥存储”对话框：

    • 在“ 生成签名的APK”向导的第一页上，单击“ 创建新按钮”。

    • 创建Android应用程序工件定义时 ，请在Android选项卡中单击创建新按钮。

  2. 指定密钥库位置，然后输入密码进行访问。

    如果要向现有的密钥库中添加新密钥，并且已经在向导或“ 工件”选项卡中选择了它，则这些字段已填写。

  3. 在别名字段中为新密钥分配别名。该密钥将使用此别名进行引用。

  4. 在“ 密码”字段中为密钥输入密码，然后进行确认。

  5. 根据您的应用程序的预期寿命来指定密钥的有效期。

  6. 提供以下个人信息，然后单击下一步 ：
    • 名字和姓氏
    • 组织单元
    • 组织
    • 城市或地区
    • 州或省
    • 国家代码
