显示

  • 声明的类型 ：显示持有对象引用的变量的类型。

  • 合成字段 ：显示合成字段（例如，内部类的编译器生成的字段）。

  • $ val字段作为局部变量 ：显示匿名类中使用的外部变量，就好像它们是局部变量一样。需要合成场 。

  • 完全合格的名称 ：显示完全实际和声明类型合格的名称。

  • 对象ID ：提供特定实例的ID，例如， java.util.ArrayList@798 。要显示ID，需要启用“ 显示类型”选项。

  • 静态字段 ：在“ 变量”选项卡和工具提示中显示静态字段。

  • 静态最终字段 ：在“ 变量”选项卡和工具提示中显示静态最终常量。需要静态字段 。

显示图元的十六进制值

为添加十六进制格式byte ， short ， int ， long和char类型。启用后，将显示十进制和十六进制格式。

隐藏数组和集合中的空元素

不包括null显示实现的数组和类的内容时的引用Collection （不适用于Map ）。

为集合类启用替代视图

选择此选项以显示实现的类的内容Collection ， Map ， 要么List以更方便的格式显示（例如，将每个地图条目显示为键值对）。

启用toString（）对象视图

允许您配置哪些类使用结果toString()作为它们的显示值。通过完全限定的类名指定时，此选项也适用于这些类的后代。您还可以使用正则表达式语法定义类模式，例如：

  • *.Test ：所有类都称为Test不论包装

  • javax.swing.* ：中的所有代码javax.swing包及其所有子包

  • ?est.Tes? ：所有以开头的类Tes然后是包装中的任何字母，以任何字母开头，然后是est

  • *.T* ：所有以开头的类T在所有包装中

Java Type Renderers选项卡

在此选项卡上，您可以为调试器“框架”视图中的数据表示创建和自定义渲染方案。

如果当前未定义渲染方案，请单击 。

项目

描述

渲染器名称

渲染器的名称。此名称用于管理渲染器，并不影响实际数据的显示方式。

将渲染器应用于类型（完全限定名称）的对象

此渲染器适用的对象类型。输入课程的全限定名或单击浏览 然后从列表中选择所需的类型。

如果该类使用泛型，请为它们使用原始类型。由于类型擦除，钻石算子内部的类型在渲染器中不起作用。

渲染节点时

此选项确定对象本身的显示方式：

  • 显示类型和对象ID ：控制是否显示类名称和实例ID。

  • 使用默认渲染器 ：显示没有该渲染器时将显示的对象。如果只想自定义对象内容的显示方式，这将很有用。

  • 使用以下表达式 ：允许您使用某些表达式的返回值来显示对象。所有方法调用和成员变量访问均相对于呈现的实例。例如， this.getName()将使用的结果getName()用于显示对象的吸气剂。

    “ 按需”选项禁用表达式的自动求值，并且仅在您明确要求时才计算结果。

扩展节点时

此选项确定对象展开时如何显示其内容。

  • 使用默认渲染器 ：显示节点子代，就像没有该渲染器时将显示它们一样。如果只想自定义对象本身的显示方式，这将很有用。

  • 使用以下表达式 ：让您使用某个表达式的返回值来显示节点的内容，例如， this.toArray() 。该表达式必须返回原始或引用类型的数组。返回数组的元素用于表示节点子级。

    测试节点是否可以扩展（可选） ：添加自动检查代表对象的节点是否可以扩展。该检查使用指定的布尔表达式，例如!this.isEmpty() 。在实例上调用该表达式，如果它的计算结果为true ，调试器使节点可扩展。

  • 使用表达式列表 ：允许您指定多个表达式，每个表达式都提供自己的对象子代表示形式。“ 按需”选项禁用它们的自动评估，并且仅在选择特定表达式时才计算结果。

追加默认子项

控制对象的字段（在未指定渲染器时通常显示的字段）是否应与表达式结果一起出现。该选项仅在“ 使用表达式列表”中可用。
