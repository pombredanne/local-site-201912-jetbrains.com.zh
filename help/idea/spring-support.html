配置Spring Facet

要在项目中使用Spring，您需要Spring构面，该构面包含用于配置特定于Spring的设置的库和UI元素。

IntelliJ IDEA会自动检测代码中的Spring配置。如果项目需要任何调整，IDE会通知您，并建议采取必要的措施。因此，通常不需要手动配置。

但是，如果IntelliJ IDEA无法自动检测配置文件，则可以手动添加构面：

将Spring支持手动添加到现有模块

  1. 按Ctrl+Shift+Alt+S打开项目结构对话框。

  2. 从左侧列表中，选择模块 。

  3. 选择必要的模块，然后单击按钮在中间部分。

  4. 从列表中选择Spring 。

  5. 添加构面时，您可能需要设置一个库。在这种情况下，请单击窗口底部警告消息旁边的修复 。

    如果您已经有一个Spring库，则可以按原样使用它，或使用计算机上的JAR文件创建一个新库。在这种情况下，请选择“ 使用库”选项。

    如果您没有资料库，请选择下载 。

如果正确配置了Spring构面，您将看到标记有每个组件，Bean等的Spring装订线图标。要配置装订线图标，请按Ctrl+Alt+S并转到编辑器|一般|天沟图标 。

创建一个应用程序上下文

Spring应用程序上下文是在IntelliJ IDEA中对配置文件进行分组的一种方式。创建上下文时，您可以让IntelliJ IDEA了解此上下文中包含的配置文件之间的关系。

您可以根据需要创建任意数量的应用程序上下文。任何配置文件都可以包含在多个上下文中。

  1. 按Ctrl+Shift+Alt+S ，然后选择Facets 。

  2. 从列表中选择Spring facet，然后单击右侧部分中的按钮。

  3. 在“ 新应用程序上下文”对话框中，输入名称，然后选择要包含在上下文中的文件。

在某些情况下，您无需配置上下文。例如，Spring MVC Web应用程序具有关于其配置的严格规则。IntelliJ IDEA中的Spring支持可以推断出它们并为您创建自动检测的应用程序上下文，以及设置Web构面。

配置父上下文

IntelliJ IDEA允许您配置上下文之间的父子关系。父上下文中的Bean对子上下文中的Bean可见，反之亦然。因此，子上下文中的bean可以使用父上下文中的配置。

例如，Spring MVC应用程序通常具有两个上下文。一个上下文属于Web层bean，另一个上下文用于服务和存储库。在这种情况下，Web层上下文将是子上下文，因为您需要将服务注入到控制器中，而不是其他方式。

要配置父上下文，请使用“ 新建应用程序上下文”对话框。

对于包含在两个或多个应用程序上下文中的文件，“ 多上下文”面板显示在编辑器的顶部。例如，如果要使用其他配置运行应用程序，并更改突出显示，则可以使用此面板选择其他活动上下文。

要禁用面板，请单击按钮，然后清除显示多个上下文面板复选框。

IntelliJ IDEA可以自动配置父上下文。例如，如果IDE检测到Spring Cloud上下文，它将使其成为Spring Boot的父应用程序上下文。

更改活动资料

Spring允许您将特定的上下文或bean映射到不同的配置文件，例如test或production 。这样，您可以在不同的环境中激活不同的配置文件。

如果您在项目中定义了至少一个配置文件，则IntelliJ IDEA将在编辑器顶部显示一个特殊面板。您可以使用此面板查看当前配置文件名称并更改活动的配置文件：

您也可以在“弹簧”工具窗口中更改活动轮廓。右键单击一个组件，然后选择Change Active Spring Profiles 。

  1. 单击面板上的更改配置文件 。

  2. 选择您要将配置文件映射到的组件。这可以是整个项目，当前模块或上下文。

  3. 选择要将此组件映射到的配置文件。

如果要隐藏面板，请单击在上面。在下一个对话框中，清除“ 显示配置文件面板”复选框。
