编译

选择应使用哪些可用的Flex编译器：
  • 内置编译器外壳。 使用Flex SDK编译器API的IntelliJ IDEA编译器外壳。

    该编译器外壳可以执行增量编译。作为多线程外壳，它能够同时并行运行多个编译。

  • Mxmlc / compc。 Flex SDK中提供了mxmlc / compc编译器。

    该编译器无法增量编译。但是，它可以同时运行多个独立的编译过程，从而大大提高了编译性能。

无论使用哪种编译器选项，IntelliJ IDEA都会跟踪自上次编译以来未发生任何更改的模块。因此，不会编译最新的SWF和SWC文件。

首选ActionScript Compiler 2.0进行纯ActionScript构建配置

如果选中此复选框，则ActionScript Compiler 2.0（ASC 2.0）用于纯ActionScript构建配置 。

多达线程或进程的并行编译

指定要同时运行的最大编译线程数（对于内置编译器外壳）或进程（对于mxml / compc编译器）。

编译器堆大小

指定启动编译器的进程可用的最大堆大小。

默认的512 Mb适用于大多数用途。但是，作为单进程编译器，内置编译器外壳可能需要更大的内存才能用于大型项目。

虚拟机选项

如有必要，键入IntelliJ IDEA启动编译器时要传递给VM的字符串。

如果需要更多输入空间，请单击字段旁边的，以访问“ Flex Compiler VM选项”对话框，其中文本输入区域较大。

指定JVM选项时，请遵循以下规则：

  • 使用空格分隔各个选项，例如， -client -ea -Xmx1024m 。

  • 如果选项包含空格，则将空格或包含空格的参数括在双引号中，例如， some" "arg要么"some arg" 。

  • 如果选项包含双引号（作为参数的一部分），请使用反斜杠对双引号进行转义，例如， -Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果您定义一个变量MY_ENV_VAR ，您可以将其传递给foo属性如下：

    -Dfoo = $ {MY_ENV_VAR}
