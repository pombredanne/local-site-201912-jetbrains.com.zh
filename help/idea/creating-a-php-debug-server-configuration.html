创建PHP调试服务器配置

仅当安装并启用PHP插件时，以下内容才有效。

PHP调试服务器配置一词表示可以确保 HTTP访问，用于调试引擎与本地和远程Web服务器进行交互，并在IntelliJ IDEA项目中设置服务器上的文件及其本地副本之间的对应关系。使用调试类型为PHP Web Page或PHP Remote Debug的运行/调试配置以及在零配置调试会话期间，将使用调试服务器配置中的设置。

您可以从头开始手动创建调试服务器配置，也可以从服务器访问（部署）配置中导入一些设置（有关详细信息，请参阅配置与Web服务器的同步 ）。

手动定义调试服务器配置

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| PHP |服务器 。

    或者，单击在“ 运行/调试配置：PHP网页”对话框中“ 服务器”列表旁边。

    无论哪种情况，“ 服务器”页面都会打开，左侧窗格显示所有可用的调试服务器配置。请点击在工具栏上，然后在右侧窗格中指定调试服务器设置。

  2. 指定调试服务器配置名称。

  3. 指定运行应用程序的主机以及访问它的端口。

  4. 从“ 调试器”列表中，选择要使用的调试引擎 。

  5. 指定IntelliJ IDEA如何在服务器上的文件及其本地副本之间建立对应关系。基于这些映射，IntelliJ IDEA将打开当前已处理文件的本地副本。

    PHP Debug Server配置中的路径映射看起来与服务器访问 （ 部署配置）中的路径映射非常相似。不幸的是，它们无法重用，因为部署配置使用相对路径，而PHP Debug Servers配置则依赖绝对路径。

    • 选择使用路径映射复选框如果在远程Web服务器上工作，即Web服务器在物理远程主机上，或者Web服务器已安装在计算机上，但您的项目在Web服务器文档根目录之外。如果使用符号链接，则也请选中此复选框。

      分别使用服务器上的路径和文件/目录字段将绝对路径映射到服务器上的文件和文件夹，并将其绝对路径映射到本地文件系统中的项目文件。

      • 文件/目录：此只读字段显示当前项目的文件和文件夹。选择要用作本地副本的文件或文件夹。

      • 服务器上的路径：在此字段中，指定目标服务器上与所选本地文件或文件夹对应的文件或文件夹的绝对路径。手动键入路径或从列表中选择它。

    • 清除使用路径映射复选框如果您在Web服务器上正确工作，则项目根目录位于服务器文档根目录下。在这种情况下，Web服务器上文件的绝对路径和项目中相应文件的绝对路径是相同的。

    如果您未指定任何路径映射并开始调试不在服务器文档根目录下的应用程序，则IntelliJ IDEA将显示错误消息：

    单击以设置路径映射链接将打开“ 解决路径映射问题”对话框，您可以在其中定义路径映射：

    当您单击“ 确定”并离开对话框时，“ 服务器”页面上的“ 使用路径映射”复选框将自动被选中。

  6. 选中共享复选框在团队中共享调试服务器配置。主机/端口设置和路径映射存储在.idea / php.xml文件中，所有团队成员均可通过版本控制系统使用。请注意，映射仅对项目内部的目录共享。

从服务器访问（部署）配置中导入设置

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| PHP |服务器 。

    或者，单击在“ 运行/调试配置：PHP网页”对话框中“ 服务器”列表旁边。

    无论哪种情况，“ 服务器”页面都会打开，左侧窗格显示所有可用的调试服务器配置。请点击在工具栏上。从部署配置导入对话框对话框打开。

  2. 在“ 部署”列表中， 选择服务器访问配置（部署配置）以从中复制服务器访问设置。该列表包含所有可用的部署配置。要创建新配置，请单击并在打开的“ 部署：连接选项卡”对话框中指定新设置。

  3. 根据从“ 部署”列表中选择的服务器访问配置的类型，指定以下选项之一：

    • 对于FTP ， SFTP或FTPS服务器访问配置，请指定服务器部署根目录的绝对路径。

      该路径将作为前缀添加到Deployment：Connection选项卡上Root Path字段的路径中。

      如果不确定此绝对路径，则可以打开“ 远程主机”工具窗口，选择所需的部署配置，将插入符号放置在根文件夹中，然后从上下文菜单中选择“ 复制路径 ”，有关详细信息，请参阅访问Web服务器上的文件。 。或者，请与您的主机提供商联系。

    • 对于本地文件夹或安装的文件夹 ，输入“ 创建PHP调试服务器配置”对话框的“ 文件夹”字段中指定的服务器根目录的绝对路径。

    • 对于Inplace Server配置，不需要映射，因为在这种情况下，本地路径和远程路径相同。

  4. 预览区显示从所选服务器访问配置（部署配置）检索到的主机/端口和路径映射。

    选择要使用的部署配置时，“ 服务器上的绝对路径”字段显示映射到所选配置中项目文件和文件夹的相对路径，即相对于部署根目录的文件和文件夹的路径。当您指定部署根目录（FTP / SFTP / FTPS的服务器根目录或已安装的文件夹）的绝对路径时，该字段的内容将自动更新，最后该字段显示服务器上的绝对路径。
