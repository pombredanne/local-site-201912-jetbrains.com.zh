JavaFX

JavaFX是用于开发可在各种设备上运行的桌面应用程序的软件平台。

IntelliJ IDEA中的JavaFX支持包括在JavaFX特定的源文件（包括.fxml和JavaFX .css文件）中进行代码完成，搜索，导航和重构，与JavaFX Scene Builder集成，JavaFX应用程序打包功能，等等。

请注意，从JDK 11开始，JavaFX 不再是JDK的一部分 。因此，如果您使用Java 11及更高版本，则除了JDK外，还需要下载开源JavaFX SDK 。

如果您使用Java 10及更早版本，则可以立即创建一个新项目 。

下载JavaFX SDK

  1. 下载适合您的操作系统的JavaFX SDK软件包。

  2. 解压缩档案并将文件夹放置在有意义的位置，例如： / Users / jetbrains / Desktop / javafx-sdk-12 。

创建一个新项目

创建新的JavaFX项目时，IntelliJ IDEA会生成一个完全配置的示例应用程序。

  1. 启动IntelliJ IDEA。

    如果“ 欢迎”屏幕打开，则单击“ 创建新项目” 。

    否则，从主菜单中选择File |新品|项目 。

  2. 在“ 新建项目”向导中，从左侧列表中选择JavaFX 。

  3. 从Project SDK列表中，选择要在项目中使用的JDK。

    如果列表为空，请单击“ 新建”并指定Java主目录的路径（例如/Library/Java/JavaVirtualMachines/jdk-12.0.1.jdk ）。

    如果您的计算机上没有Java，请参阅下载JDK软件包以获取有关如何下载和安装Java的说明。

  4. 在向导的下一步中，为项目命名，例如： JavaFX 。

  5. 如有必要，更改默认项目位置，然后单击完成 。

如果在“ 新建项目”向导中找不到JavaFX，请确保在“设置/首选项” |“启用”中启用了捆绑的JavaFX插件。插件 。

添加JavaFX库

如果您使用Java 11及更高版本，则需要执行这些操作。如果使用Java 10及更早版本，请运行与项目一起创建的示例应用程序 ，以确保一切正常。

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S或点击在工具栏上。

  2. 打开库部分，单击 ，然后选择Java 。

  3. 在JavaFX SDK软件包中指定lib文件夹的路径，例如： / Users / jetbrains / Desktop / javafx-sdk-12 / lib 。

  4. 在“ 选择模块”对话框中，选择必要的模块，然后单击“ 确定” 。

  5. 应用更改并关闭“ 项目结构”对话框。

添加虚拟机选项

如果您使用Java 11及更高版本，则需要执行这些操作。如果您使用Java 10及更早版本，则可以运行与项目一起创建的示例应用程序 ，以确保一切正常。

  1. 从主菜单中，选择运行|编辑配置 。

  2. 选择应用|主从左侧列表。

  3. 在“ VM选项”字段中，指定：--module-path %PATH_TO_FX% --add-modules javafx.controls,javafx.fxml

    代替%PATH_TO_FX% ，请指定JavaFX SDK lib目录的路径，例如： / Users / jetbrains / Desktop / javafx-sdk-12 / lib 。

  4. 应用更改并关闭对话框。

运行应用程序

  1. 从主菜单中，选择运行|运行“主要” Shift+F10 。

    IDE开始编译您的代码。

  2. 编译完成后，将出现应用程序窗口。

    这意味着项目配置正确，并且一切正常。该窗口目前为空，因为我们尚未添加任何元素。

有关如何使用Maven或Gradle创建模块化和非模块化项目的信息，请参阅JavaFX入门 。

疑难排解

  • Error: JavaFX runtime components are missing, and are required to run this application

    Java启动器检查主类是否扩展javafx.application.Application 。

    如果使用Java 11及更高版本，请确保在运行配置中指定VM选项 。

  • Error occurred during initialization of boot layer java.lang.module.FindException: Module javafx.base not found

    运行配置中VM选项字段中JavaFX库的路径可能不正确。确保路径有效，然后重试。

  • Process finished with exit code -1073740791 (0xC0000409)

    该问题是由NVIDEA驱动程序中的错误引起的。将驱动程序更新到最新版本。有关更多信息，请参考社区论坛 。
