开始使用Spring Boot

Spring Initializr是一个向导，可让您在创建项目或模块时选择必要的配置。例如，您可以选择一个构建工具，并添加Spring Boot启动器和依赖项。

创建一个Spring Boot项目

  1. 从主菜单中，选择文件|新品|项目 ，然后选择Spring Initializr 。

  2. 从Project SDK列表中，选择要在此Spring Boot项目中使用的JDK版本。如果未安装必需的版本，请参阅“ Java开发工具包（JDK）”一节以了解如何下载和安装该版本。

  3. 输入您要使用的Initializr服务的URL，或保留默认值。单击下一步 。

  4. 配置项目元数据：选择一种语言（Java，Kotlin或Groovy），构建工具（Maven或Gradle），并指定工件ID和版本。

    如果要在不同于项目JDK版本的Java版本上构建项目，则可以在此处选择它。

    单击下一步 。

  5. 选择启动器和依赖项。如果选择需要其他插件的技术，则在创建项目后，IDE会通知您有关该插件的信息，并建议您安装或启用它们。

结果， Spring Initializr将使用定义的构建文件pom.xml或build.gradle ，源文件，测试文件和空的application.properties文件创建一个项目。

如果您先前配置了高级代码格式设置，则它们将应用于预定义文件。如果您希望Spring Initializr用默认的代码格式设置创建项目，请打开“ 设置/首选项”对话框，然后转到“ 语言和框架|春季| Spring Boot并禁用Reformat Code选项。

配置自定义配置文件

Spring Initializr创建一个默认配置文件，该文件可能并不总是足以进行开发。如果您不想使用默认配置文件，或者要在其他环境中运行代码，则可以使用自定义配置文件。

为此，您必须让IntelliJ IDEA知道哪些文件是您项目中的配置文件。这将提供相关的突出显示和编码帮助。

  1. 从主菜单中，选择文件|项目结构方面 。

  2. 请点击 （ 自定义Spring Boot ）在工具栏中。

  3. 如果要使用自定义配置文件而不是默认配置文件，请在搜索框中键入新的自定义配置文件的名称。

    如果要使用多个配置文件，请单击按钮，然后从项目树中选择文件。

  4. 单击确定，然后应用更改。

    如果正确配置了自定义配置文件，则该文件将标有图标。

一些自定义配置文件是自动配置的。例如，名称与当前命名模式匹配的特定于配置文件的配置文件将被添加到上下文中。

监视Spring Boot端点

Spring Boot具有内置功能，可让您通过调用不同的端点（例如运行状况或bean详细信息）来获取关键指标并在生产环境中监视应用程序的状态。

在IntelliJ IDEA中，您可以在“ 端点”选项卡上查看端点。启动应用程序后，此选项卡将显示在“服务”工具窗口或“运行”工具窗口中。

使用Endpoints选项卡可以导航到方法和bean。双击必要的项目（或按Enter ）[ 映射或Beans选项卡上，打开在编辑器中所选择的方法或豆。

在“运行”工具窗口中访问端点

  • 启动应用程序后，“ 运行工具”窗口将自动打开。

    如果您关闭了工具窗口并想重新打开，请按Alt+4 ，或前往[ 查看|工具窗口|运行 。

在“调试工具”窗口中访问端点

  • 开始调试应用程序后，将打开“ 调试工具”窗口。

    如果您关闭了工具窗口并想重新打开，请按Alt+5 ，或前往[ 查看|工具窗口|调试 。

在“服务”工具窗口中访问端点

  1. 从主菜单中，选择运行|编辑配置 ，然后在左侧列表中单击模板 。

  2. 在“ 服务”中可用的配置部分下，单击按钮，然后选择Spring Boot 。

  3. 在“ 运行/调试配置”对话框中，打开必要的Spring Boot配置，然后选中“ 启用JMX代理”复选框。

此外，请确保添加org.springframework.boot.spring-boot-starter-actuator在运行应用程序之前，先依赖模块。

对于映射端点，添加org.springframework.boot.spring-boot-starter-web入门也是如此。

要显示或隐藏工具窗口，请转到“ 查看” |“隐藏” 。工具窗口 ，然后单击服务 。

运行应用程序时， 图标显示在bean声明的装订线中，表明在上下文中找到了bean。您可以使用此图标查看依赖和注入的Bean。

如果未出现该图标，请确保在“运行/调试配置”对话框中选中了“ 启用JMX代理”复选框。

在“ Beans”选项卡上时，可以使用装订线中的图标来更改豆类信息的显示方式。例如，您只能查看您的bean并禁用显示库bean ，您可以启用和禁用显示文档 ， 等等。每个图标都有一个标识它的工具提示。

使用devtools配置应用程序更新策略

随着spring-boot-devtools模块，您的应用程序将在类路径上的文件每次更改时重新启动。但是，如果您希望仅在特定时刻重新启动应用程序，则可以设置触发文件。在这种情况下，仅在您修改触发文件后，您的应用程序才会重新启动。

IntelliJ IDEA提供了更多选项来更新正在运行的应用程序。启用devtools之后， 更新应用程序 选项Ctrl+F10可以在“服务”工具窗口的左框架，“运行/调试”工具窗口以及主菜单（ “运行” |“更新应用程序” ）中使用。根据需要，您可以配置单击此选项后IDE的功能。

您可以使用不同的更新策略创建多个运行/调试配置，并在它们之间进行切换。

  1. 从主菜单中，选择运行|编辑配置 。

  2. 选择必要的Spring Boot配置，或创建一个新的（单击然后选择 Spring Boot ）。

  3. 指定必须应用此配置的主类。

  4. 从“更新”操作列表中，选择一个策略。您只能更新资源，更新类和资源（构建应用程序），更新触发器文件（重新启动）或执行类热交换，如果失败，则IntelliJ IDEA将更新触发器文件（重新启动）。

从“ 停用框架”列表中，您还可以选择IDE切换到另一个应用程序后将执行的操作。IntelliJ IDEA可以更新资源或构建您的应用程序。

对于更新触发器文件以及策略失败时的热插拔和更新触发器文件 ，IDE在您启动应用程序时会设置一个触发器文件。
