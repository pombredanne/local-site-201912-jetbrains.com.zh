运行检查

IntelliJ IDEA分析在编辑器中打开的文件中的代码，并在您键入时突出显示异常代码。

要快速查看即时分析的结果，请查看编辑器右上角的图标。如果检测到错误，您将看到 。的图标代表警告；的图标表示检测到错别字。如果一切正确，您将看到 。

滚动条中的色带还标记了检测到的代码问题，并可以帮助您快速访问相应的代码字符串而无需滚动文件。将鼠标悬停在条纹上的标记上可以在工具提示中查看检测到的问题。单击一个标记跳到相应的代码字符串。

您可以在文件中的编辑器中从一个突出显示的字符串导航到另一个字符串，方法是按F2 （转到下一个问题）和Shift+F2 （返回上一个）。

对于大多数代码问题，IntelliJ IDEA提供了快速修复程序，您可以通过按来查看和应用这些修复程序。 Alt+Enter 。

手动运行检查

有些检查需要全局代码分析，这就是为什么在编辑器中将其禁用。这些检查在“设置/首选项” |“首选项”中列出。 编辑器检查 。请点击 ，然后选择仅显示批处理模式检查 。

如果要获取所有检测到的异常代码字符串的完整报告，请显式运行检查。

运行所有检查

  1. 从主菜单中，选择“ 分析|检查代码 。

  2. 选择您要分析的文件范围。您也可以点击并配置一个新的范围。

  3. 选择要应用的检验配置文件 。要配置新的配置文件，请单击在检验配置文件区域中。

  4. 单击确定开始分析。

进行一次检查

如果要跟踪特定问题，运行一次检查很有用。如果在文件中找到警告，则可以检查整个项目或文件的必要范围，以确保代码库中不再存在此类警告。

  1. 从主菜单中，选择“ 分析|按名称运行检查 Ctrl+Shift+Alt+I 。

  2. 在弹出窗口中输入检查名称。使用CamelHumps将驼峰大小写单词和空白与单词的首字母匹配。建议列表将显示与您的搜索请求匹配的检查。

    如果不确定您选择的检查是否正确，则可以查看其说明。为此，请在弹出窗口中选择一个检查，然后按Ctrl+Q 。

  3. 双击必要的检查以打开其设置。

  4. 在打开的对话框中，选择要分析的文件范围。

  5. “ 文件掩码”选项可帮助您缩小将要检查的文件数量。

    选择该复选框，并指定与要分析的文件名匹配的字符和通配符模式。使用逗号分隔多个文件掩码。

  6. 某些检查可能会包含其他选项，提示您进行配置。

    这些设置将仅应用于此运行，并且不会影响当前配置文件中此检查的配置。

IDE将在专用工具窗口中显示检查结果。您可以在那里检查并修复检测到的问题。

离线运行检查

除了从IDE运行代码检查之外，您还可以从命令行启动检查，而无需实际运行IntelliJ IDEA。检查结果将存储在XML文件中。有关更多信息，请参阅从命令行启动代码检查 。

更改范围的顺序

默认情况下，所有启用的代码检查都会分析项目中的所有文件。根据您的需要，您可以在多个具有不同设置的文件范围中运行相同的检查。

如果两个或多个范围中包含一个文件，并且在这些范围中启用检查，则IntelliJ IDEA将根据它们在范围列表中的顺序对其进行处理-最高的范围将具有最高优先级，因此，首先分析。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择编辑器|检查 。

  2. 从列表中选择任何检查。

  3. 从“ 所有合并范围”列表中，选择“ 编辑合并范围顺序” 。

  4. 选择必要的范围，并使用和在列表中上下移动。

  5. 如果需要，创建一个新的范围。为此，请单击 （ 编辑范围 ），指定范围设置，然后选择要包括在其中的文件和文件夹。
