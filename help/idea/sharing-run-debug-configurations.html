共享运行/调试配置

如果您在团队中工作，则可能需要共享运行/调试配置，以便队友可以完全按照您的方式运行该应用程序，或者使他们能够远程连接到您正在运行的流程。

为此，IntelliJ IDEA提供了一种从项目文件中提取运行/调试配置并通过VCS共享它们的机制。当您要将配置作为文件发送给其他人时，也可以使用相同的机制。由于有时运行/调试配置会变得复杂，因此节省了大量时间，而手动创建它们既乏味又容易出错。

旧版（ .ipr基于项目的项目）不支持共享单独的运行/调试配置。对于旧项目，您只能通过添加以下内容一次共享所有配置.ipr文件到VCS。

  1. 转到要共享的运行/调试配置的属性（ “运行” |“编辑配置” ），然后启用“ 通过VCS共享”选项。这将从您的workspace.xml文件中提取运行/调试配置，并将其放在.idea / runConfigurations目录中。

  2. 如果要使用VCS共享运行/调试配置，请将.idea / runConfigurations中的相应文件添加到VCS。

启用“ 通过VCS共享”选项不会为您提交任何内容到VCS。为了使运行/调试配置进入共享存储库，您必须像其他版本控制的文件一样签入它们。
