PHP运行时选项卡

该区域列出了可用的PHP存根 ，这些存根是语法上正确的正常PHP文件，其中包含带注释的函数，方法和类签名，常量定义等。PHP存根被添加到IntelliJ IDEA的内部知识中，以增强对所有标准PHP库组件以及常见扩展的编码帮助。

  • 使用每个项目旁边的复选框启用/禁用相应的存根。

  • 要加载与当前配置的CLI解释器的已加载扩展集匹配的存根集，请单击“ 使用解释器同步扩展”按钮。有关配置解释器的详细信息，请参见“ 配置本地PHP解释器”和“ 配置远程PHP解释器” 。

在“ 项目工具窗口”中 ，当前加载的存根显示在“ 外部库”节点下。

PHP存根是开源的，您可以为它们的发展做出贡献。有关详细信息，请参见博客文章 。

高级设置区域

如有必要，您可以加载一组供IntelliJ IDEA使用的自定义PHP存根。请点击在默认存根路径字段旁边，并在打开的对话框中提供存根文件夹的位置。这样，您可以加载其他存根，也可以覆盖捆绑的存根。

PHP存根可作为Composer软件包提供 ，因此可以声明为第三方软件包的依赖项并进行安装。在这种情况下，它们存储在项目内的vendor / jetbrains / phpstorm-stubs文件夹下。

为防止Composer软件包存根与PhpStorm的捆绑存根之间发生冲突，请执行以下操作之一：

  • 要在项目中使用Composer程序包存根，请在“ 默认存根路径”字段中显式提供其位置的路径 。

  • 要使用IntelliJ IDEA捆绑的PHP存根，请确保清除了“ 默认存根路径”字段，并且项目中排除了vendor / jetbrains / phpstorm-stubs文件夹。

    • 如果在“ 语言和框架”中启用了“ 将软件包添加为库”选项PHP |作曲家页面的 设置/首选项对话框Ctrl+Alt+S ，IntelliJ IDEA会自动将vendor / jetbrains / phpstorm-stubs文件夹标记为已排除 。默认情况下，该选项处于启用状态。

    • 如果禁用“ 将软件包添加为库”，则可以手动排除存根软件包文件夹 。为此，在“ 项目”视图中，右键单击vendor / jetbrains / phpstorm-stubs文件夹，然后选择“将目录标记为|”。从上下文菜单中排除 。

有关在IntelliJ IDEA中使用Composer的详细信息，请参阅Composer Dependency Manager 。

调用树分析深度

使用此列表可以设置所需的异常分析深度级别。默认情况下，选择1 ，在这种情况下IntelliJ IDEA报告抛出异常的父方法以及直接调用该方法的方法的未处理异常。选择较大的值可让您更深入地了解呼叫层次结构。

如果选择0 ，则异常分析仅限于父方法。

大于1的值可能会对IDE的性能产生负面影响。

使用常量参数跳过呼叫

选中此复选框可使IntelliJ IDEA在异常分析期间跳过实例创建表达式中的某些方法调用：

  • 仅将字符串或常量作为参数的方法调用被跳过：

    $ date =新的DateTime（'2019-01-01'）;

  • 具有变量，整数，数组等的方法调用（不跳过参数）：

    $ date =新的DateTime（$ myDate）;

使用此部分定义自定义文件夹$_SERVER['DOCUMENT_ROOT']表达式应解析为，这会更改“ 未解析的包含”检查的行为。

在使用文件路径作为参数的函数/方法的代码完成中 ，将使用指定的路径（例如， require()要么include() ）。如果路径留为空白，代码完成将建议相对于项目根目录的路径。
