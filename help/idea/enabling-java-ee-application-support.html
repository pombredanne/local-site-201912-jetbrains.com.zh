IntelliJ IDEA 2019.3帮助

启用Java EE应用程序支持

本主题讨论打开JavaEE Application选项时可用的功能。

先决条件

要使JavaEE Application选项和相关功能可用：

  • 必须启用Java EE：EJB，JPA，Servlets 插件 。（此插件与IDE捆绑在一起，并默认启用。）

功能概述

当您打开JavaEE Application选项时，IntelliJ IDEA：

  • 创建META-INF / application.xml ，这是企业应用程序存档部署描述符 。

  • 创建一个Java EE应用程序构面，使您可以指定application.xml和应用程序特定于服务器的部署描述符的位置（例如glassfish-application.xml ， jboss-app.xml ）。

  • 创建爆炸的EAR 工件配置。

  • 在“ 项目结构”对话框中提供各种快速修复，例如，用于将application.xml与EAR工件的结构同步。

  • 使JavaEE：App工具窗口可用。

如果在创建项目或模块时打开JavaEE Application选项并指定应用程序服务器，则IntelliJ IDEA还将为该服务器创建运行/调试配置 。

打开JavaEE应用程序选项

您可以打开JavaEE Application选项：

  • 创建项目或模块时（ 文件|新建|项目或文件|新建|模块 ）。在“ 新建项目”或“ 新建模块”向导的第一页上，选择“ Java Enterprise” ，然后选中“ 其他库和框架”下的“ JavaEE应用程序”复选框。

  • 对于现有模块。在项目工具窗口（ 视图|工具窗口|项目 ）中，右键单击模块文件夹，然后选择添加框架支持 。然后在打开的对话框中选择JavaEE Application复选框。

管理部署描述符

您可以在“ 项目结构”对话框中管理application.xml和特定于服务器的部署描述符文件：

  1. 打开“ 项目结构”对话框（例如， Ctrl+Shift+Alt+S ）。

  2. 在最左侧的窗格中，选择Modules或Facets 。

  3. 在右侧窗格中，选择javaEEApplication 。

  4. 在对话框右侧打开的页面上 ，形成应用程序的部署描述符列表。

管理应用程序工件

要将应用程序部署到服务器，您需要一个应用程序工件 。对于Java EE应用程序，IntelliJ IDEA提供以下构件格式：

  • JavaEE应用程序：爆炸。 这是一个解压缩的企业应用程序归档文件（ EAR ），该目录结构已准备好部署到应用程序服务器上。

  • JavaEE应用程序：存档。 显然，这是EAR文件。

要管理工件配置，请使用“ 项目结构”对话框（“ 文件” |“项目结构” |“工件” ）。

另请参阅使用工件 。
