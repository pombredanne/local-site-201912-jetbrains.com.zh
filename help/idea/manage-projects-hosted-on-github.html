管理托管在GitHub上的项目

检出项目（克隆）

您可以直接从IntelliJ IDEA克隆要贡献的存储库，并基于该存储库创建一个新项目。

  1. 从主菜单中，选择VCS |。从版本控制中检出|吉特

  2. 在“ 克隆存储库”对话框中，指定要克隆的存储库的URL。您可以从与您的帐户以及该帐户所属的组织相关联的所有GitHub项目的列表中选择一个存储库。

  3. 在目录字段中，输入将在其中创建本地Git存储库的文件夹的路径。

  4. 单击克隆 。如果要基于这些来源创建项目，请在确认对话框中单击“ 是 ”。IntelliJ IDEA将自动将Git根映射设置为项目根目录。

在GitHub上共享项目

您可以为正在本地开发的项目添加远程GitHub存储库，以便其他人可以查看它或对此做出贡献。

  1. 打开您要共享的项目。

  2. 从主菜单中，选择VCS |。导入版本控制|在GitHub上共享项目 。

    如果您已在IntelliJ IDEA中注册了GitHub帐户 ，则将使用这些凭据建立连接。

    如果您尚未在IntelliJ IDEA中注册帐户，则会打开“ 登录GitHub”对话框。指定您的访问令牌或使用您的登录名和密码请求一个新的令牌。

  3. 建立与GitHub的连接后，“ 在GitHub上共享项目”对话框打开。指定新的存储库名称，远程名称，然后输入项目的描述。

    如果您不想允许其他GitHub用户公开访问您的存储库，则可以选择“ 私有”选项（请注意，免费帐户无法使用此选项）。

  4. 单击共享以启动新的存储库，并将项目源上载到该存储库。

跳转到文件的GitHub版本

您可以从IntelliJ IDEA跳转到文件的GitHub版本。IntelliJ IDEA会检测到当前处于活动状态的分支以及文件的最新版本，并在Web浏览器中的相应提交中打开所选文件的GitHub副本。如果要从编辑器打开GitHub文件版本，该文件还将自动滚动到当前行。

  • 选择在编辑器中或在项目视图中的文件，然后从上下文菜单中或从GitHub上选择打开 Ctrl+Shift+A弹出。该文件的远程版本将在浏览器中打开。

    如果文件指向多个远程，则系统将提示您选择适当的存储库。
