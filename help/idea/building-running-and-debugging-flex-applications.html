运行/调试配置类型

以下运行/调试配置类型可用于Flash模块：

  • 通过Flash App配置，您可以编译然后运行或调试Flash（ActionScript和Flex）应用程序。您可以在运行或调试之前创建必要的配置。使用源代码时，您还可以直接在编辑器中开始运行或调试应用程序（请参阅从编辑器内部运行或调试应用程序 ）。

  • FlexUnit配置使您可以编译然后运行或调试FlexUnit测试。请参阅测试ActionScript和Flex应用程序 。

  • Flash远程调试配置使您可以调试已经编译过的应用程序，并在必要时对其进行打包，并准备在本地或远程计算机或移动设备上运行这些应用程序。请参阅使用Flash Remote Debug配置 。

为了能够调试您的应用程序：

  • 应用程序必须准备好进行调试，即包含必要的调试信息。

  • 对于以Web为目标的应用程序，必须为Web浏览器安装Flash Player的调试器版本或Flash Player插件的调试器版本。通常，该软件包含在Flex SDK中。

    您也可以从Adobe Flash Player的“下载”页面单独下载相应的软件。

从编辑器中运行或调试应用程序

在编辑器中使用类源代码时，如果合适，可以使用当前类作为主要应用程序类来运行或开始调试应用程序 。为此，请右键单击编辑器区域中的某个位置以打开上下文菜单，然后选择：

  • 运行“ ” Ctrl+Shift+F10运行该应用程序。

  • 调试“ ”以开始调试应用程序。

如果已经存在适合该任务的Flash App运行/调试配置 ，则将选择并使用此配置。否则，将创建新的运行/调试配置，并将其另存为临时运行/调试配置 。有关详细信息，请参阅IntelliJ IDEA如何选择或创建特定于类的运行/调试配置 。

除了“运行和调试”之外 ，根据情况还可能提供以下相关命令：

  • 创建“ ”。 如果找不到合适的运行/调试配置，则可以使用此命令来创建新的运行/调试配置并将其配置为最新。（将打开“ 创建运行/调试配置”对话框。）

  • 保存“ ”。 如果相应的运行/调试配置可用作临时配置，则可以使用此命令保存配置，从而使其成为永久配置。

  • 选择“ ”。 如果要使相应的临时或永久运行/调试配置为最新，请使用此命令。

上述所有功能也可以在“ 项目”或“ 收藏夹”工具窗口中作为上下文菜单命令来访问。

IntelliJ IDEA如何选择或创建特定于类的运行/调试配置

如前所述，当您在编辑器中运行或调试应用程序时 ，IntelliJ IDEA首先尝试使用当前正在使用的类作为主类来查找现有的运行/调试配置。

如果找不到这样的配置，则会创建一个新的运行/调试配置。

如果找到具有合适主类的多个配置，则IntelliJ IDEA会根据以下条件对配置进行优先级排序，并选择优先级最高的配置：

  1. 运行/调试配置基于活动的构建配置，构建配置的输出类型为Application，运行/调试配置中不覆盖主类。

  2. 运行/调试配置基于输出类型为Application的构建配置，运行/调试配置中不会覆盖主类。

  3. 运行/调试配置基于活动的构建配置，运行/调试配置中覆盖了主类。

  4. 具有适当（重写）主类的任何运行/调试配置。

创建新的运行/调试配置时，IntelliJ IDEA会尝试查找具有合适主类的构建配置。如果找到，则新的运行/调试配置将基于这种构建配置。否则，将使用活动的构建配置，并且新的运行/调试配置中将覆盖主类。

使用Flash远程调试配置

  1. 选择并启动必要的Flash Remote Debug配置要么Shift+F9 。

    结果， 调试工具窗口打开；调试器正在等待应用程序连接。

  2. 现在，要连接到调试器，请执行以下一项操作：

    • 在本地计算机上启动Flash或AIR应用程序；应用程序将自动连接到调试器。

    • 如果应用程序已经在本地或远程计算机上的Web浏览器或Flash播放器中运行，请右键单击浏览器中相应的页面或播放器中的应用程序，然后从上下文菜单中选择“ 调试器 ”。

    • 在移动设备上启动应用程序。如果该设备能够与您的计算机通信，则该应用程序将自动连接到调试器。

隐藏或显示[SWF]和[UnloadSWF]调试器消息

您的Flash Player调试器输出可能包含许多[SWF]和[UnloadSWF]消息（“ 调试工具”窗口的“ 控制台”选项卡）。在某些情况下，您不想看到它们。

您可以隐藏或显示[SWF]和[UnloadSWF]通过使用工具栏上的按钮。

请注意，更改此切换的状态不会更改当前控制台的内容。也就是说，如果打开过滤器，输出中已经存在的消息将不会被隐藏。只显示新消息。
