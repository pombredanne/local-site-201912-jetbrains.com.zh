将项目文件保留在

选中此复选框可以指定导入后项目文件的位置。例如，当您导入一个项目并想要将iml文件和.idea目录文件保留在特定位置而不是默认位置时。默认情况下，IntelliJ IDEA将项目的文件放在pom.xml旁边。

在外部存储生成的项目文件

默认情况下，未选中此选项，并且生成的.iml和库文件存储在.idea目录中。

在以下情况下很有用：

  • 由于.idea目录存储项目级设置，因此可以通过VCS共享IDE特定的模块设置 。

  • 正确打开同时包含常规IntelliJ IDEA模块和Maven模块的项目。

  • 打开项目时可以更快地访问它，因为IntelliJ IDEA首先读取.iml文件，然后开始导入过程。

当您在Maven项目中进行更改时， .iml文件也会被更改，因此请不要忘记将它们与其他项目的更改一起推送到VCS下。

如果要将生成的iml和库文件从外部存储在idea.system.path中 ，请选择此选项。

自动导入Maven项目

如果希望IntelliJ IDEA每次更改pom.xml时自动执行重新导入，请选中此复选框。

自动检测编译器

选择此选项后，IntelliJ IDEA将使用自动检测到的编译器，该编译器在“设置/首选项” |“ 使用编译器”列表中指定。 构建，执行，部署|编译器Java编译器 。如果要手动指定编译器，请取消选择此选项。

为聚合器项目创建IntelliJ IDEA模块（带有“ pom”包装）

如果选中此复选框，则将为聚合项目的pom.xml文件中包含的每个模块创建IntelliJ IDEA Maven模块 ，前提是其包装设置为“ pom”。

为多模块Maven项目创建模块组

如果选中此复选框，则IntelliJ IDEA将通过聚合的Maven项目创建模块组，并将嵌套模块包含在该组中。

在重新导入时保留源文件夹和测试文件夹

如果选中此复选框，则每次导入时都将保留所有源文件夹和测试文件夹。

如果清除此复选框，则每次导入时都将删除所有先前配置的源文件夹和测试文件夹。

默认情况下，此复选框的设置如下：

  • 对于新项目 ：清除该复选框。

  • 对于已经导入的项目 ：选中该复选框。

排除构建目录PROJECT_ROOT / target

选择此复选框可从项目中排除构建目录。如果您想加快项目的导入过程，这可能会很有用。如果取消选中此复选框，则每次导入项目时，IntelliJ IDEA都会索引构建目录中的文件，这可能需要更多时间。

使用Maven输出目录

如果未选中此复选框，则将在常规IntelliJ IDEA的输出目录USER_HOME \ IdeaProjects \ \ classes \ Production \中创建该版本。如果选中此复选框，则将在Maven的输出目录中生成内部版本，并重复使用IntelliJ IDEA的编译结果。但是，IntelliJ IDEA本身不会重用Maven构建结果，而是从头开始执行编译。

生成的源文件夹

重新导入项目时，请指定源根目录。

您可以选择以下选项之一：

  • 自动检测这是默认选项。选择此选项时，IntelliJ IDEA会自动检测生成的源的位置。IntelliJ IDEA还检测将哪个目录标记为源根目录。但是，IntelliJ IDEA仅在目标/生成源和目标/生成源/ *目录中搜索生成的源 。

  • target / generated-sources此选项使您可以手动将目录标记为源根目录。

  • “目标/生成源”的子目录此选项使您可以手动将子目录标记为源根。

  • 不检测此选项可让您跳过检测过程。

用于文件夹更新的阶段

选择要用于文件夹更新的Maven阶段。如果您调整插件以便在某个阶段加载其他源，这可能会很有用。

自动下载

选择相应的复选框以自动下载有关打开Maven项目的源（ Sources ）和文档注释（ Documentation ）。

依赖类型

使用此字段可以指定在重新导入项目时要包括的依赖项类型。

进口商的VM选项

使用此字段可以指定VM选项。默认选项是-Xmx512m 。
指定选项时，请遵循以下规则：

  • 使用空格分隔各个选项，例如， -client -ea -Xmx1024m 。

  • 如果选项包含空格，则将空格或包含空格的参数括在双引号中，例如， some" "arg要么"some arg" 。

  • 如果选项包含双引号（例如，作为参数的一部分），请使用反斜杠将双引号转义，例如， -Dmy.prop=\"quoted_value\" 。

进口商JDK

使用此列表可以指定导入 maven项目时要使用的JDK。

您可以选择以下选项之一：

  • 内部JRE-这是使用JRE安装目录的默认选项。

  • JDK版本-此选项使用项目的JDK。

  • 使用JAVA_HOME-此选项使用用户的环境变量设置中指定的值。

导入Flexmojos项目时生成Flex编译器配置文件

如果选中此复选框，则在导入Flexmojos项目时，将自动生成Flex编译器配置文件。

自动生成配置文件是一个非常耗时的过程，尤其是对于大型项目。此外，出于独立于IntelliJ IDEA的原因，有时可能无法自动生成配置文件，或者可能导致错误的结果。

在所有这些情况下，您可能需要关闭此选项并使用以下命令从命令行生成Flex编译器配置文件：

mvn compile -DconfigurationReport=true

为了使此选项可用，在IntelliJ IDEA中启用了Flash / Flex支持 。 的默认情况下，插件已激活。如果禁用了插件 ，请按照“ 管理插件”中所述在“ 插件”页面上启用它。
