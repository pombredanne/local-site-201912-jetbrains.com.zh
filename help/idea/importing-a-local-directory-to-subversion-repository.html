将本地目录导入到Subversion存储库

如果您具有访问权限，则可以将整个目录导入Subversion存储库。这有助于将整个项目置于版本控制下。

即使未为您的项目启用Subversion集成，也始终可以导入到资源库。

要将目录导入Subversion存储库，请执行以下操作：

  1. 从主菜单中，选择VCS |。导入版本控制|导入Subversion 。

  2. 在打开的“ 导入到Subversion”对话框中，选择目标Subversion存储库。如果所需的目标存储库位置不在列表中，请单击按钮添加它（有关更多详细信息，请参阅配置Subversion信息库位置 ）。点击导入 。

  3. 在打开的“ 选择路径”对话框中，指定要导入的目录，然后单击“ 确定” 。

  4. 在打开的“ SVN导入选项”对话框中，选中“ 导入到”和“ 从路径导入”，并指定以下选项：
    • 深度 使用此列表可以指定递归到Subversion子目录的范围。可用的选项有：

      • 工作副本 ：选择此选项可从尚未检出的存储库子树中获取文件/目录。

      • 空 ：选择此选项以仅涉及当前文件。

      • files ：选择此选项可以包含当前文件夹中的文件。

      • 即时 ：选择此选项可包含当前文件的直接子级。

      • 无限 ：选择此选项以启用完全递归。

    • 包括被忽略的资源 ：选择此选项可将已添加到Subversion忽略列表中且不受版本控制的文件包括到导入中。

  5. （可选）输入提交消息，或从“ 最近的消息”下拉列表中选择一个，然后单击“ 确定” 。
