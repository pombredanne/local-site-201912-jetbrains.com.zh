从XML模式生成Java代码

介绍

本主题描述如何获取XML Schema的Java表示形式，其中涉及将XML Schema的元素映射到Java类的成员。使用IntelliJ IDEA，可以使用以下数据绑定器之一完成此转换：

  • JAXB生成类并将它们分组在Java包中。一个包由一个Java类名和一个ObjectFactory类。后者是一个工厂，用于返回绑定的Java类的实例。

  • XmlBeans将XML Schema转换为Java类，对其进行编译，然后将其放置在指定的输出jar文件中。

使用JAXB

使用JAXB从XML模式生成Java类

  1. 在活动的编辑器选项卡中，打开所需的Schema .xsd文件或包含所需Schema的XML文档。然后选择工具| JAXB |在主菜单上使用JAXB从XML Schema生成Java代码 。

  2. 在打开的使用JAXB从Xml架构生成Java对话框中，配置生成过程：

    • 在“ 模式/ DTD / WSDL路径”下拉列表中，指定用作代码生成基础的文件。默认情况下，该字段显示当前文件的完整路径。接受此建议或单击浏览 然后在打开的“ 选择用于JAXB生成的XML模式文件 ”中选择所需的文件。

    • 从“ 输出路径”下拉列表中，选择模块源目录，以将生成的Java类放入其中。

    • 在“ 包前缀”下拉列表中，指定要包含生成的存根的包。

    • 使用复选框，配置其他选项，例如生成注释，设置只读状态，下载和安装其他库。

使用XmlBeans

使用XmlBeans从XML架构生成和编译Java类

  1. 在活动的编辑器选项卡中，打开所需的Schema .xsd文件或包含所需Schema的XML文档。然后选择工具| XmlBeans |在主菜单上使用XmlBeans从XML模式生成Java代码 。

  2. 在打开的“ 使用XmlBeans从XML模式生成Java代码”对话框中，配置生成过程：

    • 在“ 架构路径”下拉列表中，指定用作代码生成基础的文件。默认情况下，该字段显示当前文件的完整路径。接受此建议或单击浏览 然后在打开的“ 选择XML模式/ WSDL生成文件”对话框中选择所需的文件。

    • 在“ 输出路径”下拉列表中，指定jar文件的名称，以将生成和编译的Java代码放入其中。默认情况下，IntelliJ IDEA建议创建一个新文件类型 。要覆盖现有文件，请单击“ 浏览”。 并在打开的“ 选择XML模式/ Wsdl文件生成”对话框中选择所需的文件。

    • 要自动下载并安装缺少的库，请选择“ 添加必要的库以生成生成的代码并进行工作”复选框。
