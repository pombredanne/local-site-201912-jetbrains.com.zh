创建Android应用程序组件

Android应用程序可能包含以下组件：

  • AIDL ：用于进程间通信的Android接口定义语言（AIDL）接口。

  • 活动 ：实现一个窗口，您可以在其中放置UI与用户进行交互。

  • Android Auto ：可让您扩展应用程序以用于车辆。您可以添加媒体服务或消息服务活动。

  • 文件夹：根据您为此组件选择的活动创建源根。

  • 片段 ：代表行为或行为在用户界面中的一部分。

  • Google：可让您为Google地图和AdMob Ads活动创建活动。

  • 应用程序 ：一个Android程序包，即一个.apk存档，其中包含一个Android应用程序和安装程序的内容。

  • 服务 ：表示应用程序希望执行操作而不与用户交互，或者为其他应用程序提供功能的愿望。

  • 其他：让您将以下组件添加到您的应用程序：
    • Android清单文件
    • 广播接收器
    • 内容提供商
    • 白日梦

  • UI组件 ：可让您向应用程序添加自定义视图。

  • Wear ：可让您扩展应用程序以用于Android Wear。

  • 小部件 ：您可以为应用程序添加不同类型的小部件。

  • XML ：可让您为Android布局和值添加不同类型的.xml文件。

所有Android应用程序组件均以相同的方式创建。创建新组件时，将生成实现该组件的类，并在AndroidManifest.xml文件中自动声明该组件。

在此页：

  • 创建一个Android组件
  • 在活动或片段及其相关的布局定义文件之间导航
  • 从组件导航到AndroidManifest.xml文件中的声明

创建一个Android组件

  1. 在“项目”视图中，右键单击存储应用程序类的目标folding-and-expanding-custom-blocks.htmlkage，然后从上下文菜单中选择“ 新建” |“创建”。 “组件名称”以及特定的服务，文件或活动 。

  2. 在打开的对话框中，指定必要的信息，然后单击完成 。

请注意，当您创建活动，服务或广播接收器时，它们会自动注册在androidmanifest.xml文件中。

在活动或片段及其相关的布局定义文件之间导航

您可以从活动或片段的源代码跳转到代表其内容视图的布局定义文件，反之亦然，从布局定义到源代码。

  • 要从组件跳转到相关的布局定义，请打开组件的源代码，然后执行以下操作之一：

    • 单击装订线区域中的图标，然后在“ 转到相关符号”列表中选择布局定义文件。

    • 在主菜单上，选择“ 导航”。相关符号 。

  • 要从布局定义跳至相应组件的源代码，请在文本模式下打开布局定义文件，然后执行以下操作之一：

    • 点击天沟区域中的图标。

    • 在主菜单上，选择“ 导航”。相关符号 。

从组件导航到AndroidManifest.xml文件中的声明

创建组件后，便会立即在AndroidManifest.xml文件中对其进行声明。对于activity ， fragment ， service和broadcast接收器类型的组件，可以直接从组件源代码跳转到此声明。

  1. 打开组件的源代码，然后单击装订线区域中的图标。

  2. 如果组件还具有相关的布局定义文件，请在“ 转到相关符号”列表中选择AndroidManifest.xml 。
