上下文和依赖注入（CDI）

IntelliJ IDEA中的上下文和依赖注入 （CDI）支持基于Java EE：上下文和依赖注入插件。该插件与IDE捆绑在一起，并且默认情况下启用。

您可以在创建项目或模块时启用CDI支持。您也可以为现有模块添加CDI支持。

模块中的CDI由模块依赖项中的相应CDI库表示。您可以直接从IDE中下载所有必需的库文件。如果需要，也可以在需要时更改使用的CDI版本。

确保已启用CDI插件

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择插件|已安装 。

  2. 在列表中找到Java EE：上下文和依赖注入插件，并确保选中它旁边的复选框。

创建项目或模块时启用CDI支持

  1. 要创建新项目，请在“欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件” |“ 新建” 。 新品|项目 。

    要创建一个新模块，请打开必要的项目，然后选择File | File。新品|模块 。

  2. 在打开的向导中，选择左侧的Java Enterprise 。

  3. 在向导的右侧，指定要使用的JDK，选择要支持的Java EE版本，然后指定要使用的应用程序服务器。

  4. 从“ 其他库和框架”列表中，选择“ CDI：上下文和依赖注入”复选框。

  5. 配置库。您可以使用现有的库，创建和使用新的库，下载库文件或将库的设置推迟到以后。

    单击下一步 。

  6. 在向导的下一页上，指定新项目或模块的名称和位置。点击完成 。

如果您的计算机上没有JDK，请参阅Java开发工具包（JDK）部分以获取有关如何下载和安装它的更多信息。

为现有模块添加CDI支持

  1. 在“项目”工具窗口中，右键单击所需的模块，然后选择“ 添加框架支持” 。

    要打开工具窗口，请从主菜单中选择“ 视图” | “工具”。工具窗口|项目 。

  2. 从左侧的列表中，选择“ CDI：上下文和依赖注入”复选框。

  3. 配置库。您可以使用现有的库，创建和使用新的库，下载库文件或将库的设置推迟到以后。

  4. 应用更改并关闭对话框。

结果，您指定的CDI库将添加到模块依赖项列表中。

更改CDI版本

  1. 点击工具栏上的图标（或按Ctrl+Shift+Alt+S ）以打开“ 项目结构”对话框。

  2. 选择左侧的模块 ，然后打开“ 依赖关系”选项卡。

  3. 选择必要的CDI库，然后单击 。

  4. 在“ 配置项目库”对话框中，单击“ 更改版本” 。

  5. 打开“ 版本”列表，然后选择一个新的库版本。应用更改并关闭对话框。
