通过在外部配置属性来自定义构建执行

仅当安装并启用PHP插件时，以下内容才有效。

Phing构建文件通常包含属性元素 ，这些属性元素可以在构建文件本身中手动配置，也可以在Phing构建工具窗口中的外部配置。启动构建执行时，将从外部配置的属性通过命令行传递给Phing，因此每次都会对其进行动态重新计算。外部配置的属性始终会覆盖构建文件中手动设置的属性。

在这个部分：

  • 访问Phing Build工具窗口
  • 在外部配置属性

访问Phing Build工具窗口

  1. 在编辑器中打开Phing构建文件，或在“ 项目”工具窗口中选择它，然后从所选内容的上下文菜单中选择“ 添加为Phing构建文件 ”。

  2. 选择查看|工具窗口|从主菜单开始构建 。通过编辑器或“项目”工具窗口中的Phing构建文件的上下文菜单打开工具窗口后，即可对其进行访问。

在外部配置属性

  1. 打开“ Phing Build”工具窗口。

  2. 在列表中选择所需的构建文件，然后单击“ 设置”按钮在工具栏上。

  3. 在打开的“ Phing设置”对话框中，切换到“ 属性”选项卡，然后配置要通过命令行传递给Phing的一个或多个列表。

    1. 要将新属性添加到列表中，请单击“ 添加”按钮。

    2. 在属性字段中，输入属性的名称。

    3. 在“ 值”字段中，指定要传递的属性值。请执行以下任一操作：

      • 手动输入值。

      • 若要使用IntelliJ IDEA宏 ，请单击“ 插入宏”按钮并在打开的“宏”对话框中配置相关宏定义的列表。要添加宏，请在列表中将其选中，然后点击插入 。

  4. 属性列表准备好后，在“ Phing设置”对话框中单击“ 确定 ” 。
