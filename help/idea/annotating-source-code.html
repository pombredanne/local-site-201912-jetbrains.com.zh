注解

Java注释是一些元数据，可提供有关它们所使用的代码的信息，并可以指示IDE如何处理该代码。

在Java中，有一组内置注释。最重要的是，IntelliJ IDEA提供了注释的集合，这些注释可以作为单独的项目使用 。您可以将它们添加到类路径并在您的代码中使用。

将org.jetbrains.annotations添加到Gradle或Maven项目

  • 要将带注释的库添加到Gradle项目中，请添加compile 'org.jetbrains:annotations:16.0.2'对build.gradle文件的依赖。

  • 对于Maven项目，添加org.jetbrains:annotations:16.0.2对pom.xml的依赖。

将org.jetbrains.annotations添加到其他项目

  1. 打开项目结构对话框Ctrl+Shift+Alt+S ，然后选择库 。

  2. 请点击 ，然后选择从Maven 。

  3. 在搜索字段中，输入org.jetbrains:annotations:16.0.2如果您使用的是JDK 1.8或更高版本。

    对于JDK 1.5、1.6或1.7，键入org.jetbrains:annotations-java5:16.0.2 。

  4. 单击确定 。

在编辑器中添加org.jetbrains.annotations

您还可以使用意图操作启用注释。

  1. 在编辑器中，输入注释，例如， @NotNull然后按Alt+Enter ：

  2. 从列表中，选择将“注释”添加到classpath 。

    IDE将提示您从Maven下载带有注释的库。

JetBrains注释

有关JetBrains注释的更多信息，请参阅GitHub上的JetBrains注释页面。

  • 的@Nls注解表示带注释的代码元素是需要本地化的字符串。

  • 的@NonNls注解表示带注释的代码元素是用户不可见的字符串，它不需要本地化，并且不包含需要本地化的字符串。当您使用注释元素时@NonNls ，本地化工具将跳过此元素和其中的字符串。

  • 的@PropertyKey注释指示方法参数接受必须是特定资源束中的有效属性键的参数。当不是捆绑包中的属性键的字符串文字作为参数传递时，IntelliJ IDEA会将其突出显示为错误。注释还用于提供作为参数传递的字符串文字的补全。

  • 的@TestOnly注解表示只能从测试代码中调用方法或构造函数。

  • @Contract批注使您可以指定方法必须遵循的一组规则（合同）。如果违反合同，则IntelliJ IDEA将报告问题。

  • @Nullable批注指示可以为空的变量，参数或返回值。

  • @NotNull批注指示不能为空的变量，参数或返回值。
