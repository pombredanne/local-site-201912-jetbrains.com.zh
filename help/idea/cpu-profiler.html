IntelliJ IDEA 2019.3帮助

分析工具

分析工具对于大多数时间都在运行哪些方法很有用。他们可以帮助您找到最昂贵的方法并准确了解它们的行为。

IntelliJ IDEA与以下分析工具集成：

  • Async Profiler ：用于Linux和macOS的CPU和内存分析工具。

  • Java Flight Recorder ：Oracle提供的CPU工具，可在Linux，macOS和Windows上使用。

异步探查器

Async Profiler在Linux和macOS上可用。

异步事件探查器监视应用程序的JVM级别参数，以更好地了解应用程序的执行方式以及内存和CPU资源的分配方式。此数据可以帮助您查找和解决性能问题。

您可以为使用Gradle和本机IntelliJ IDEA构建工具构建的项目运行事件探查器。如果使用Maven，则可以使用本机IntelliJ IDEA运行程序运行该应用程序以获取分析数据。

分析完成后，探查器会在报告中可视化输出数据。

在Linux上开始使用探查器之前，请安装Perf工具并调整内核选项。在macOS上，分析器可以直接使用。

在Linux上安装Perf并调整内核选项

如果您使用的是Linux，请确保完成以下步骤：

  1. 为您的特定内核版本安装Perf工具。

    使用uname -r找出确切的版本，然后安装相应的linux-tools软件包。例如：

    $ uname -r 4.15.0-36-generic $ sudo apt-get install linux-tools-4.15.0-36-generic

  2. 调整内核选项

    • perf_event_paranoid-控制非root用户对性能事件数据的使用。

      将该值设置为小于2，以允许探查器在没有root特权的情况下收集性能信息：

      sudo sh -c'回显1> / proc / sys / kernel / perf_event_paranoid'

    • kptr_restrict-设置对公开内核地址的限制。

      要正确解析内核符号， 请将其值设置为0，以禁用kptr_restrict提供的保护：

      sudo sh -c'回显0> / proc / sys / kernel / kptr_restrict'

    默认情况下，这些更改仅影响您当前的OS会话。要在整个系统重新引导期间保留设置，请运行：

    sudo sh -c'echo kernel.perf_event_paranoid = 1 >> /etc/sysctl.d/99-perf.conf'sudo sh -c'echo kernel.kptr_restrict = 0 >> /etc/sysctl.d/99-perf。 conf'sudo sh -c'sysctl --system'

探查器配置

IntelliJ IDEA具有两个预定义的Async Profiler配置：CPU Profiler和内存分配Profiler，您可以在Settings / Preferences |构建，执行，部署| Java Profiler 。

调整了这些配置以提供最准确的结果，这就是为什么它们不需要任何修改的原因。但是，如果您仍然需要更改设置，请参阅GitHub上的Async Profiler了解更多信息。

Java飞行记录器

Java Flight Recorder （JFR）是一种监视工具，可在应用程序执行期间收集有关Java虚拟机中特定时间实例中事件（数据片段）的信息。

为Oracle JDK 8或更高版本启用JFR

Java Flight Recorder可以在版本8开始的Oracle JDK构建上运行，前提是您已启用商业功能，并且可以在版本11开始的任何JDK构建上运行。

此标志解锁Oracle Java SE Advanced或Oracle Java SE Suite产品的商业功能。启用它们之前，请仔细阅读Java平台产品的Oracle二进制代码许可协议 。

要启用商业功能：

  1. 从主菜单中，选择运行|编辑配置，然后从左侧列表中选择要使用JFR分析的运行配置。

  2. 在“ 配置”选项卡上，展开“ 环境”部分。

  3. 在“ VM选项”字段中，添加以下行： -XX:+UnlockCommercialFeatures 。

  4. 应用更改并关闭对话框。

创建一个新的JFR配置

有两种预安装的配置： Default和Profile 。默认配置的开销很低（大约1％），这就是为什么它可以很好地用于连续分析。Profile配置的开销约为2％，可用于更详细的应用程序分析。

这些配置涵盖了所有用例的大部分。但是，您可以通过Java Mission Control创建和上传自己的设置。

  1. 运行您可以在JDK_HOME / bin / jmc中找到的Java Mission Control或从builds.shipilev.net下载。

  2. 从Java Mission Control主菜单中，选择Window |飞行记录模板管理器。

  3. 单击右侧面板上的导入文件 ，然后选择.jfc格式的预安装配置之一：

    • 对于Java 10及更早版本： JDK_HOME / jre / lib / jfr /

    • 对于Java 11和更高版本： JDK_HOME / Contents / Home / lib / jfr /

  4. 选择编辑|高级 ，更改必要的设置，然后单击确定 。

  5. 在“ 飞行记录模板管理器”对话框中，单击“ 导出文件” 。

  6. 为新设置文件指定另一个名称，并在必要时更改其位置。点击保存 。

有关如何配置自定义设置的更多信息，请参阅“ 使用模板控制记录数据” 。

Windows上的默认JDK_HOME路径： C：\ Program Files \ Java ，macOS上的默认路径： / Library / Java / JavaVirtualMachines 。

将自定义配置加载到IDE

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择构建，执行，部署| Java Profiler 。

  2. 选择您要向其加载自定义设置的Java Flight Recorder分析配置，或通过单击以下命令创建新配置 。

  3. 在“ 分析设置”区域中，选择“ 自定义设置文件”选项，然后单击“确定”。 。

  4. 指定自定义设置文件的路径，然后单击“ 打开” 。

  5. 应用更改并关闭“ 设置/首选项”对话框。

开始分析

选择配置文件配置

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择构建，执行，部署| Java Profiler 。

  2. 选择一种预定义的CPU或内存配置文件配置，或者通过单击创建一个新的配置 。

  3. 应用更改并关闭对话框。

运行分析

为了运行Async Profiler或Java Flight Recorder，请按照下列步骤操作：

  1. 在主菜单中，选择运行或单击在工具栏上。

  2. 选择“运行带有配置文件配置名称”的“运行配置名称” 。

分析数据准备就绪后，您将看到一个确认弹出窗口，并且Profiler工具按钮将出现。单击此按钮以打开Profiler工具窗口（如果不使用工具按钮，则转到View | Tool Windows | Profiler ）。

解释结果

在“探查器”工具窗口中，收集的数据显示在三个选项卡上：“ 火焰图” ，“ 调用树 方法列表 ”和“ 事件” 。左侧列出了应用程序线程。通过单击每个线程，您可以进入更多详细信息。

导出分析结果

  1. 在“探查器”工具窗口（ 视图|工具窗口|探查器 ）的左框架上，单击 。

  2. 在打开的对话框中，命名文件，指定要在其中保存文件的文件夹，然后单击“ 保存” 。

导入分析结果

您可以将分析数据导入到IDE中，以使用分析工具进行分析。确保您导入的文件是由Async Profiler创建的或具有.jfr格式。

  1. 从主菜单中，选择运行|导入探查器结果|从文件 。

  2. 选择要导入的文件，然后单击“ 打开” 。

火焰图

此选项卡在性能配置文件中随时显示调用堆栈的状态。这是调用树的直观表示：矩形代表调用堆栈的帧，按宽度排序。

每个块代表堆栈中的一个功能（堆栈框架）。每个块的宽度对应于所用方法的CPU时间（或分配大小（在分配概要分析的情况下））。在Y轴上，从下往上有一个堆栈深度。X轴显示了堆栈配置文件，从最消耗资源的功能到最不消耗功能的功能排序。

阅读火焰图时，请注意最宽的方块。这些块是配置文件中最常显示的功能。您可以从底部开始向上移动，按照从父方法到子方法的代码流进行操作，也可以使用相反的方向探索显示直接在CPU上运行的功能的顶部块。

呼叫树

“ 调用树”选项卡表示有关在概要分析期间采样的程序的调用堆栈的信息：

  • 方法名称

  • 总采样时间的百分比（可以切换到父级调用视图）

  • 总样本数

  • 过滤的呼叫数

百分比后面的可选数字表示已过滤的呼叫序列。单击以展开此序列。可以在设置中配置过滤规则。

要配置和过滤“呼叫树”视图，请使用“ 演示设置”按钮（ ）。

过滤通话树结果

“呼叫树”选项卡允许您折叠当前不感兴趣的帧。例如，您可以隐藏来自特定框架的数据。

  1. 配置过滤器。 在里面 设置/首选项对话框Ctrl+Alt+S ，选择构建，执行，部署| Java Profiler |过滤器 。

  2. 配置要隐藏在输出中的包或特定类。

  3. 应用更改并关闭对话框。

  4. 要隐藏报告中的框架，请单击位于框架名称之前的蓝色超链接数字。

    要快速禁用过滤，请单击然后单击“ 筛选呼叫”

方法清单

方法列表收集配置文件数据中的所有方法，并按累计采样时间对它们进行排序。此列表中的每个项目都有一个Back Trace树和一个Merge Callees树。

返回跟踪显示了呼叫者的层次结构。使用此视图可跟踪哪些方法调用了所选方法。合并被调用者尝试汇总由所选方法调用的所有方法。

Java Flight Recorder事件

Java Flight Recorder收集有关事件的数据。事件在特定时间点在JVM或Java应用程序中发生。每个都有一个名称，一个时间戳和一个可选的有效负载。
