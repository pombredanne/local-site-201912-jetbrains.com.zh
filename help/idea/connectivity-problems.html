无法连接到数据库

步骤1。检查您的网络设置

数据库可以在本地，服务器或云中工作。对于服务器和云数据库，您需要网络连接。要验证连接是否可用，请使用ping和telnet命令。

使用ping命令，可以确保从源计算机可以访问目标计算机。打开命令行并键入以下命令： ping -a ，在哪里-a是一个命令选项，用于将地址解析为主机名（如果可能）。如果将主机名与ping命令一起使用，则主机名将解析为IP地址。例如， ping -a example.com决心PING example.com (93.184.216.34) 。

ping -a

使用telnet命令，您可以测试与远程计算机的连接并发出命令。如果您将端口指定为telnet命令，您可以在给定端口上测试到远程主机的连接性。如果连接成功，您将看到以下消息： Connected to 。

远程登录

出于安全原因，DBMS通常会删除所有telnet连接。telnet命令允许您检查端口是否已打开以进行通信。

第2步。检查您的连接属性

每个数据库（MySQL，PostgreSQL，Oracle或任何其他供应商）都有自己的连接设置。大多数数据库包括连接设置：

  • 主机 ：存储数据库的计算机或其他设备的主机名。它可以是IP地址127.0.0.1或域名localhost 。

  • 数据库 ：要连接到的数据库的名称。您可以在数据库服务器的设置中找到数据库名称，也可以询问数据库管理员。在某些情况下，可以在数据库命令行中运行查询以查看所有可用数据库的名称。例如，在MySQL中，您可以运行SHOW DATABASES; 。

  • 用户 ：具有足够特权以执行数据库操作的用户名。在数据库命令行中运行查询以查看所有可用数据库的名称。例如，在MySQL中，您可以运行SHOW GRANTS; 。

  • 密码 ：用户的密码。

  • 端口 ：标识主机之间连接点的数字。主机使用端口号来确定必须建立到哪个应用程序，服务或进程的连接。不同的数据库供应商为其数据库使用不同的端口。以下列表是默认端口号的列表。

第三步检查驱动程序版本

使用JDBC驱动程序，您可以与IntelliJ IDEA中的数据库管理系统（DBMS）进行交互。每个DBMS需要其自己的JDBC驱动程序。确保驱动程序版本和DBMS版本相互兼容。

从IntelliJ IDEA，您可以下载所有受支持供应商的驱动程序。您可以在“ 驱动程序”列表中查看受支持的供应商的完整列表。 Ctrl+Alt+S 。或者，您可以将自己的驱动程序添加到现有供应商，或为不在“ 驱动程序”列表中的供应商创建新的驱动程序条目。

下载驱动程序并选择驱动程序版本

要从JetBrains FTP服务器下载驱动程序 ，请从“ 驱动程序”列表中选择一个供应商，然后单击“ 下载版本”。 驱动程序文件窗格中的链接。

要更改驱动程序版本，请单击ver。 驱动程序文件窗格中的链接，然后选择所需的驱动程序版本。

使用JDBC驱动程序创建与数据库的连接

如果在数据源列表中找不到数据库供应商的名称，请下载用于数据库管理系统（DBMS）的JDBC驱动程序，然后在IntelliJ IDEA中创建连接。使用JDBC驱动程序，您可以连接到DBMS并开始工作。

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 在“ 数据源和驱动程序”对话框中，单击“ 添加”图标（ ），然后选择驱动程序和数据源 。

  3. 单击用户驱动程序链接。

  4. 在驱动程序文件窗格中，单击添加图标，然后选择自定义JAR 。

  5. 导航到JDBC驱动程序的JAR文件，选择它，然后单击OK 。

  6. 在“ 类”字段中，指定要用于驱动程序的值。

  7. 点击应用 。

  8. 返回到创建的数据源连接。

  9. 指定数据库连接详细信息。或者，将JDBC URL粘贴在URL字段中。

    要删除密码，请右键单击“ 密码”字段，然后选择“ 设置为空” 。

  10. 要确保成功连接到数据源，请单击“ 测试连接” 。

将JDBC驱动程序添加到现有连接

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 在“ 数据源和驱动程序”对话框中，选择要在其中更改驱动程序的数据源，然后单击数据源设置中的“ 驱动程序”链接。

  3. 点击提供的驱动程序条目，然后点击删除 （ ）。

    要还原更改，请单击窗口右上方的“ 重置” 。或者，点击添加按钮（ ），然后选择提供的驱动程序| |

  4. 在驱动程序文件窗格中，单击添加图标（ ），然后选择“ 自定义JAR” 。

  5. 导航到JDBC驱动程序的JAR文件，选择它，然后单击OK 。

  6. 在“ 类”字段中，指定要用于驱动程序的值。

  7. 点击应用 。

第四步。检查是否需要使用SSH或SSL连接

为了使与数据库的连接更加安全，某些服务需要使用SSH或SSL。对于某些服务，您需要应用某些JDBC选项以使用SSL或SSH隧道（例如，当您尝试连接到Heroku Postgres时 ）。

使用SSL连接到数据库

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要在其中更改连接设置的数据源配置文件。

  3. 单击“ SSH / SSL”选项卡，然后选择“ 使用SSL”复选框。

  4. 在“ CA文件”字段中，导航到CA证书文件（例如， ca.pem ）。

  5. 在“ 客户端证书文件”字段中，导航到客户端证书文件（例如， client-cert.pem ）。

  6. 在“ 客户端密钥文件”字段中，导航到客户端密钥文件（例如， client-key.pem ）。

  7. 要确保成功连接到数据源，请单击“ 测试连接” 。

建议使用PEM证书。

使用自签名证书，在某些情况下使用由受信任的根实体发布的证书，当您使用最新的JDBC驱动程序版本时，可能会遇到错误。如果您的Java密钥库不接受证书链，则SSL连接可能会失败。作为临时解决方案，请尝试降级JDBC驱动程序（例如，对于MySQL连接器，您需要切换到5.1.40版本。）

使用SSH连接到数据库

  1. 在“ 数据库”工具窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标 。

  2. 选择要在其中更改连接设置的数据源配置文件。

  3. 单击“ SSH / SSL”选项卡，然后选择“ 使用SSH隧道”复选框。

  4. 在“ 代理主机” ，“ 代理用户” ，“ 代理密码 ”和“ 端口”字段中，指定连接详细信息。

  5. 从“身份验证类型”列表中，选择一种身份验证方法：

    • 密码 ：使用密码访问主机。指定密码，然后选择“ 保存密码”复选框以将密码保存在IntelliJ IDEA中。

    • 密钥对（OpenSSH或PuTTY） ：对密钥对使用SSH身份验证 。 要应用此身份验证方法，您必须在客户端计算机上具有私钥，而在远程服务器上必须具有公钥。IntelliJ IDEA支持使用OpenSSH实用程序生成的私钥。

      指定存储私钥的文件的路径，然后在相应的字段中键入密码（如果有）。要使IntelliJ IDEA记住密码，请选中“ 保存密码”复选框。

    • OpenSSH配置和身份验证代理 ：使用由凭证帮助程序（例如Windows上的Pageant或Mac和Linux上的ssh-agent ）管理的SSH密钥。

  6. 要确保成功连接到数据源，请单击“ 测试连接” 。

使用PuTTY创建SSH隧道（Windows）

  1. 下载并运行最新版本的PuTTY SSH和Telnet客户端（从https://www.putty.org/下载客户端）。

  2. 在“ PuTTY配置”对话框中，导航到“ 连接” |“连接”。 SSH |验证 。

  3. 在用于认证的私钥文件字段中，指定私钥文件的路径，然后单击打开 。

  4. 在命令行窗口中，指定用于SSH隧道的用户名，然后按Enter 。不要关闭命令行窗口。

  5. 在“数据库”窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标在工具栏上。

  6. 选择要在其中更改连接设置的数据源配置文件。

  7. 单击“ SSH / SSL”选项卡，然后选择“ 使用SSH隧道”复选框。

  8. 从“ 身份 验证类型”列表中，选择“ OpenSSH配置和身份验证代理” 。

  9. 在“ 代理主机” ，“ 代理用户 ”和“ 端口”字段中，指定连接详细信息。

  10. 要确保成功连接到数据源，请单击“ 测试连接” 。

使用Pageant创建SSH隧道（Windows）

Pageant是用于PuTTY，PSCP，PSFTP和Plink的SSH身份验证代理。Pageant会存储您的私钥，只要它正在运行，它就会为PuTTY或其他工具（如IntelliJ IDEA）提供解锁的私钥。您可以在Windows任务栏中找到Pageant图标。

  1. 下载Pageant的最新版本（从https://www.putty.org/下载客户端）。

  2. 在Windows任务栏中，右键单击Pageant图标，然后选择Add Key 。

  3. 在“ 选择私钥文件”对话框中，导航到私钥文件（PPK文件），然后单击“ 打开” 。

  4. （可选）输入私钥密码，然后按Enter 。

  5. 在“数据库”窗口（“ 视图” |“工具”窗口|“数据库” ）中，单击“ 数据源属性”图标在工具栏上。

  6. 选择要在其中更改连接设置的数据源配置文件。

  7. 单击“ SSH / SSL”选项卡，然后选择“ 使用SSH隧道”复选框。

  8. 从“ 身份 验证类型”列表中，选择“ OpenSSH配置和身份验证代理” 。

  9. 在“ 代理主机” ，“ 代理用户 ”和“ 端口”字段中，指定连接详细信息。

  10. 要确保成功连接到数据源，请单击“ 测试连接” 。
