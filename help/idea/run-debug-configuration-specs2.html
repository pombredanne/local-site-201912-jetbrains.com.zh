运行/调试配置：Specs2

IntelliJ IDEA允许您为Specs2指定运行/调试配置。

该对话框包含以下选项卡：

  • 配置选项卡
  • 代码覆盖率选项卡
  • 日志标签

配置选项卡

项目

描述

测试种类

从此下拉列表中，选择测试范围，并根据您的选择填写字段。
您可以从以下选项中选择：

  • 测试名称-选择此选项可运行指定的Scala测试。测试名称出现在“ 测试名称”字段中。

  • 全部包含在软件包中-选择此选项可运行指定软件包中的所有Scala测试。
    填写以下字段：

    • 测试软件包-指定要测试的软件包的名称。

    • 搜索测试-使用此下拉列表选择搜索范围。

  • 类-选择此选项可运行类中的所有测试。
    在“ 测试类”字段中指定要启动的类的标准名称。输入班级名称或单击然后在打开的对话框中选择所需的类。

虚拟机参数

如有必要，请指定要传递给VM的字符串。

指定JVM选项时，请遵循以下规则：

  • 使用空格分隔各个选项，例如， -client -ea -Xmx1024m 。

  • 如果选项包含空格，则将空格或包含空格的参数括在双引号中，例如， some" "arg要么"some arg" 。

  • 如果选项包含双引号（作为参数的一部分），请使用反斜杠对双引号进行转义，例如， -Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果您定义一个变量MY_ENV_VAR ，您可以将其传递给foo属性如下：

    -Dfoo = $ {MY_ENV_VAR}

如果空间不足，可以单击然后在打开的对话框中输入字符串。

环境变量

请点击打开“ 环境变量”对话框，您可以在其中创建变量并指定其值。

测试选项

使用此字段可以指定其他测试选项。

如果空间不足，可以单击然后在打开的对话框中输入字符串。

工作目录

指定运行测试时将用作当前目录的目录。它将充当所有相对输入和输出路径的根目录。默认情况下，项目文件所在的目录用作工作目录。输入目录名称，或单击然后在打开的对话框中选择所需的目录。

使用模块的类路径和SDK

从此下拉列表中，选择其类路径将用于运行应用程序的模块。

将信息消息打印到控制台

如果要将信息消息打印到Scala控制台，请选中此复选框。

代码覆盖率选项卡

使用此选项卡可以配置代码覆盖率监视选项。

项目

描述

选择报道亚军

选择所需的代码覆盖率运行器。

默认情况下，IntelliJ IDEA使用其自己的覆盖引擎和“采样”模式。您也可以选择JaCoCo或Emma来计算覆盖率。

采样

选择此选项以最小的速度测量代码覆盖率。

追踪

选择此选项以收集准确的分支机构覆盖范围。此模式仅适用于IntelliJ IDEA代码覆盖率运行器。

跟踪每个测试范围

选择此复选框以检测一个测试覆盖的线以及所有覆盖该线的测试。如果选中此复选框， 在coverage统计信息弹出窗口的工具栏上变为可用。

该选项仅适用于测试运行/调试配置的代码覆盖率度量的跟踪模式 。

请参阅“ 配置代码覆盖率选项 ”部分。

记录代码覆盖率数据的包和类

请点击和按钮指定要测量的类别和包装。您也可以从列表中删除类和包，方法是在列表中选择它们，然后单击按钮。

在测试文件夹中启用覆盖率。

如果选中此复选框，则标记为test的文件夹包含在代码覆盖率分析中。

日志标签

使用此选项卡可以指定在控制台或运行或调试工具窗口的专用选项卡上显示在运行或调试时生成的日志文件。

项目

描述

活跃

选择此列中的复选框可使日志条目显示在“ 运行”工具窗口或“ 调试 ”工具窗口的相应选项卡中。

日志文件输入

此列中的只读字段列出了要显示的日志文件。该列表可以包含：

  • 特定文件的完整路径。

  • 定义要显示文件范围的Ant模式 。

  • 别名，用于替代完整路径或模式。这些别名也显示在选项卡的标题中，其中显示了相应的日志文件。

    如果日志输入模式定义了多个文件，则选项卡页眉将显示文件名，而不是日志条目别名。

跳过内容

选择此复选框可跳过所选日志的先前内容。

将控制台输出保存到文件

选择此复选框可将控制台输出保存到指定位置。手动输入路径，或单击浏览按钮，然后在打开的对话框中指向所需位置。

在将消息打印到标准输出流时显示控制台

选中此复选框可激活输出控制台，并在关联的进程写入Standard.out时将其调出。

在将消息打印到标准错误流时显示控制台

选中此复选框可激活输出控制台，并在关联的进程写入Standard.err时将其调出。

单击此按钮以打开“ 编辑日志文件别名”对话框 ，您可以在其中选择新的日志条目并为其指定别名。

单击此按钮可在“ 编辑日志文件别名”对话框中编辑所选日志文件条目的属性。

单击此按钮从列表中删除所选的日志条目。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

如果使用基于目录的项目格式 ，则运行/调试配置的设置存储在.idea \ runConfigurations文件夹（如果共享）中的单独的.xml文件中，或者存储在.idea \ workspace中。 xml文件，否则。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以进行共享配置，否则将存储在.iws文件中。

允许并行运行

如果选中此复选框，则可以并行启动运行配置多次，而不必重新运行它。每个运行程序都将在“ 运行工具窗口”或“ 调试工具窗口 ”的自己的选项卡中启动 。

禁用后，每次启动运行/调试配置的新实例时，IntelliJ IDEA都会检查同一运行/调试配置的其他实例是否存在，并显示确认对话框。如果在确认对话框中单击“ 确定” ，则将停止运行器的第一个实例，而将其替换。当启动两个相同类型的运行/调试配置实例时，这会占用过多的CPU和内存资源。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在IntelliJ IDEA中定义它，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 构建 ：选择以编译指定的模块。将执行“ 构建模块”命令 。

    如果在编译过程中发生错误，则IntelliJ IDEA不会尝试启动运行/调试配置。

  • 生成项目 ：选择以编译整个项目。将执行“ 生成项目”命令 。

    如果在编译过程中发生错误，则IntelliJ IDEA不会尝试启动运行/调试配置。

  • Build，无错误检查 ：与Build选项相同，但是IntelliJ IDEA不管编译结果如何，都将尝试启动运行/调试配置。

  • 构建工件 ：选择此选项，以建立一个神器或假象。在打开的对话框中，选择应构建的工件。

  • 启动Web浏览器 ：选择此选项以启动浏览器。在打开的对话框中，选择浏览器的类型并提供起始URL。另外，指定是否要使用JavaScript调试器启动浏览器。

  • 运行Ant目标 ：选择此选项可运行Ant目标 。在打开的对话框中，选择要运行的目标。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行Maven目标 ：选择此选项可运行Maven目标 。在打开的对话框中，选择要运行的目标。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
