摩卡咖啡

Mocha是一个JavaScript测试框架，对于执行异步测试场景特别有用。您可以从IntelliJ IDEA外部运行Mocha测试，检查以树形视图排列的测试结果，并从那里轻松导航到测试源。在测试旁边，在编辑器中，IntelliJ IDEA显示了测试状态，并带有可快速运行或调试的选项。

在你开始前

  1. 下载并安装Node.js。

  2. 确保在“ 插件”页面上启用了NodeJS捆绑插件，有关详细信息，请参阅管理插件 。

安装摩卡

  • 在嵌入式终端 （ Alt+F12 ），键入以下命令之一：

    • npm install mocha用于项目中的本地安装。

    • npm install -g mocha用于全局安装。

    • npm install --save-dev mocha将Mocha安装为开发依赖项 。

    从Mocha官方网站上的入门中了解更多信息 。

您还可以按照npm和Yarn中的 说明 ，将Mocha软件包安装在Node.js和NPM页面上 。

运行测试

使用IntelliJ IDEA，您可以直接在编辑器中快速运行单个Mocha测试，也可以创建运行/调试配置来执行部分或全部测试。

从编辑器运行单个测试

  • 请点击要么在左侧装订线中，然后选择“运行” 从列表中。

    借助测试状态图标，您还可以在编辑器中查看测试是通过还是失败在排水沟中。

创建Mocha运行配置

  1. 打开“ 运行/调试配置”对话框（主菜单上的“运行” |“编辑配置 ”），单击在左侧窗格中，然后从列表中选择Mocha 。将打开“ 运行/调试配置：Mocha”对话框。

  2. 指定要使用的节点解释器以及其位置mocha包。

  3. 指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  4. （可选）配置对相关源文件中的更改自动重新运行测试。为此，请添加--watch 额外的摩卡选项字段中标记。

  5. 指定要运行的测试。这可以是特定的测试或套件，整个测试文件或包含测试文件的文件夹。您还可以定义模式以仅运行匹配文件中的测试。

  6. 选择要运行的测试中使用的接口 。

通过运行配置运行测试

  1. 从主工具栏上的列表中选择Mocha运行/调试配置，然后单击在列表的右边。

  2. 测试服务器将自动启动，而无需您采取任何步骤。在“ 运行工具”窗口中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

重新运行失败的测试

  • 在“ 测试运行器”选项卡中 ，单击“ 在工具栏上。IntelliJ IDEA将执行在上一个会话期间失败的所有测试。

  • 要重新运行特定的失败测试，请选择“运行”。 在其上下文菜单上。

有关详细信息，请参见重新运行测试 。

导航

使用IntelliJ IDEA，您可以在文件和相关测试文件之间跳转，也可以从“ 测试运行器”选项卡中的测试结果跳转到测试。

测试文件应遵循流行的命名约定，例如，具有.test。 ， .spec。或_spec。后缀，并且应位于源文件旁边或测试文件夹中。

  • 在编辑器中打开文件，然后选择“ 转到|测试或转到|从上下文菜单中测试受试者 ，或者只需按Ctrl+Shift+T 。

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源代码 ”。测试文件将在编辑器中打开，并将光标置于测试定义上。

  • 对于失败的测试，IntelliJ IDEA从堆栈跟踪将您带到测试的失败行。如果确切的行不在堆栈跟踪中，则将转到测试定义。

调试测试

使用IntelliJ IDEA，您可以直接从编辑器快速开始调试单个Mocha测试，或创建运行/调试配置来调试部分或全部测试。

从编辑器开始调试单个测试

  • 请点击要么在左侧装订线中，然后选择“ 调试” 从列表中。

通过运行/调试配置启动测试调试

  1. 如上所述创建Mocha运行/调试配置。

  2. 从主工具栏上的列表中选择Mocha运行/调试配置，然后单击在列表的右边。

  3. 在打开的“ 调试工具”窗口中，照常进行： 逐步执行测试 ， 停止并继续执行测试，在挂起时检查测试 ，依此类推。

监控代码覆盖率

使用IntelliJ IDEA， 您还可以监视Mocha测试覆盖了多少代码。IntelliJ IDEA在专用工具窗口中显示此统计信息，并在编辑器中直观地标记覆盖线和未覆盖线。要监视覆盖范围，您需要安装nyc （ 伊斯坦布尔的命令行界面）。

安装nyc

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install --save-dev nyc

您也可以安装1.1.0或更高版本的Istanbul本身。这些版本支持ES6和TypeScript。

运行覆盖测试

  1. 如上所述创建Mocha运行/调试配置。

  2. 从主工具栏上的列表中选择Mocha运行/调试配置，然后单击在列表的右边。
    或者，在编辑器的覆盖范围内快速运行特定套件或测试：单击要么在左侧装订线中，从列表中选择“ 以覆盖率运行” 。

  3. 在“ 覆盖率”工具窗口中监视代码覆盖率。该报告显示测试覆盖了多少文件以及其中覆盖行的百分比。从报告中，您可以跳至文件，查看覆盖了哪些行-标记为绿色-以及未覆盖哪些行-标记为红色：
