Node.js

Node.js是一个轻量级的运行时环境，用于在浏览器外部（例如，在服务器上或在命令行中）执行JavaScript。IntelliJ IDEA与Node.js集成，可在配置，编辑，运行，调试，测试，分析和维护应用程序方面提供帮助。

如果只需要Node.js作为应用程序的本地运行时或管理npm软件包，运行JavaScript linter，构建工具，测试框架等的本地运行时，则只需安装Node.js即可 。如果遵循标准安装过程，则大多数情况下IntelliJ IDEA会检测到Node.js本身。

即使计算机上没有Node.js，也可以在“ 创建新项目”对话框的“ 新建项目向导 ”中创建新的Node.js应用程序时进行安装，请参阅下面的“ 创建新的Node.js应用程序” 。

如果要在多个Node.js安装之间切换，则必须将它们配置为本地Node.js解释器 。在大多数情况下，IntelliJ IDEA会检测Node.js安装并将其自动配置为解释器，然后将其添加到列表中，您可以在其中选择相关的安装。

要远程运行Node.js应用程序，请将其配置为远程解释器。有关详细信息，请参见带有Docker的Node.js。

在Node.js版本之间切换

使用IntelliJ IDEA，您可以安装多个Node.js，并在同一项目上进行切换。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | Node.js和NPM 。

  2. 在打开的Node.js和NPM页面上，从“ 节点解释器”列表中选择所需的Node.js安装。

    如果您遵循标准安装过程，则在大多数情况下，所需的Node.js安装在列表中。如果缺少安装，请单击并将其手动配置为本地解释器 。

使用系统Node.js版本

使用IntelliJ IDEA，您可以将默认系统节点别名设置为项目的Node.js版本。之后，所有需要Node.js的工具都会在所有新的运行/调试配置中自动使用此版本。特别是，这意味着如果您安装新的Node.js版本并将其设置为系统中的默认节点别名，则不必更新每个工具的设置。

使用nvm时，此功能特别有用。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | Node.js和NPM 。

  2. 从“ 节点解释器”列表中，选择“ 节点” 。

  3. 在适用的情况下，例如在您的运行/调试配置或特定工具的设置中，指定此新的Node.js解释器。

配置本地Node.js解释器

您可能需要手动将Node.js安装配置为解释器，例如，如果Node.js安装在非默认位置，则IntelliJ IDEA不会自动检测到它。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | Node.js和NPM 。

  2. 在打开的Node.js和NPM页面上，单击 节点解释器列表旁边。

  3. 在打开的Node.js解释器对话框中，其中列出了所有当前配置的解释器，单击在工具栏上。在打开的对话框中，从上下文菜单中选择“ 添加本地” ，然后选择Node.js的安装，然后单击“ 确定” 。您将返回到“ Node.js解释器”对话框，其中“ 节点解释器”只读字段显示新解释器的路径。

  4. 在“ 包管理器”字段中， 选择当前项目的程序包管理器（npm或Yarn）。

    有关详细信息，请参见为项目配置程序包管理器 。

当单击OK时 ，返回到Node.js和NPM页面，其中Node解释器字段显示新的解释器。

在Linux的Windows子系统上使用Node.js

IntelliJ IDEA允许您在Windows Subsystem for Linux上使用Node.js运行和调试Node.js应用程序。您可以选择WSL上的Node.js作为当前项目的默认解释器，也可以在Node.js运行/调试配置中配置和使用此节点版本。

将WSL上的Node.js配置为默认项目节点解释器

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | Node.js和NPM 。

  2. 请点击在打开的Node.js Interpreters对话框中的Node Interpreter字段旁边，单击 ，然后从列表中选择“ 添加WSL ”。

  3. 在打开的“ 添加WSL节点解释器”对话框中，选择您正在使用的Linux发行版，并指定Node.js的路径。

创建一个Node.js应用程序

如果还没有应用程序，则可以从模板生成具有特定于Node.js的结构的IntelliJ IDEA项目，或者创建一个空的IntelliJ IDEA项目并在其中配置Node.js，如下面的从现有的Node.js应用程序开始所述。 。

创建一个新的Node.js应用程序

  1. 选择文件|新品|从主菜单中的项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 新建项目”对话框中，在左侧窗格中选择Node.js和NPM 。

  3. 在右侧窗格中，选择Node.js Express App并单击Next 。

  4. 在向导的第二页上， 指定项目文件夹，Node.js解释器，程序包管理器（ npm或Yarn ，有关详细信息，请参见npm和Yarn ）。

    对于Node.js Express应用，请指定express-generator的版本，然后选择要使用的模板语言和样式表语言。

  5. 单击“完成”时，IntelliJ IDEA将下载必要的依赖项并为它们以及Node.js核心API启用代码完成功能，有关详细信息，请参阅配置node_modules库和配置Node.js Core库 。

    对于Node.js Express ，IntelliJ IDEA使用默认设置创建Node.js类型的运行/调试配置，并生成特定于Node.js Express的基本目录结构 。

    对于Node.js ，IntelliJ IDEA仅运行npm init命令生成package.json文件。

创建一个空的IntelliJ IDEA项目

  1. 选择文件|新品|从主菜单中的项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 新建项目”对话框中，在左侧窗格中选择“ 静态Web ”。

  3. 在右侧窗格中，再次选择“ 静态Web” ，然后单击“ 下一步” 。

  4. 在向导的第二页上，指定项目文件夹和名称，然后单击完成 。

从现有的Node.js应用程序开始

如果要继续开发现有的Node.js应用程序，请在IntelliJ IDEA中将其打开，在其中配置Node.js，然后下载所需的依赖项。

打开计算机上已经存在的应用程序源

  • 单击“ 欢迎”屏幕上的“ 打开 ”，或选择“ 文件” |“共享”。从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中签出应用程序源

  1. 在“ 欢迎”屏幕上单击“从版本控制获取”，或选择“ VCS |从主菜单的版本控制中获取 。

  2. 在调用的对话框中，从列表中选择您的版本控制系统，然后指定存储库以签出应用程序源。

在项目中配置Node.js

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | Node.js和NPM 。

  2. 在“ 节点解释器”字段中， 指定当前项目的默认Node.js解释器。IntelliJ IDEA每次您选择Project创建运行/调试配置或配置依赖于Node.js的工具（例如Prettier或ESLint）时， Node Interpreter列表中的别名。

    从列表中选择配置的解释器，或单击然后在打开的对话框中配置一个新对话框，如配置本地Node.js解释器中所述 。如果选择node ，则使用系统Node.js版本 。

  3. 选中“ Node.js的编码帮助”复选框将Node.js Core模块源配置为JavaScript库并将其与您的项目关联。结果，IntelliJ IDEA为以下人员提供了代码完成，参考解析，验证和调试功能： fs ， path ， http ，以及其他已编译为Node.js二进制文件的Node.js部分。

    配置完成后，IntelliJ IDEA 显示有关当前配置版本的信息。

  4. 如果仅在项目的某些部分中需要Node.js API的代码完成，则可以使用“ 管理范围”链接进行配置。在打开的“ 用法”对话框中，单击相关目录，然后从列表中为每个目录选择配置的Node.js Core库。从配置库的范围中了解更多信息。

要下载项目依赖项，请执行以下任一操作：

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install

  • 从项目根目录中package.json文件的上下文菜单中选择Run'npm install' 。
