播放框架支持概述

请注意，IntelliJ IDEA版本11支持Play框架版本1.x。IntelliJ IDEA的更高版本支持Play 2.x版本。此外，对于IntelliJ IDEA 14.0版，Play 2.x框架与Scala插件合并。

IntelliJ IDEA中的Play框架支持包括：

  • 可以在IDE中直接使用play命令行实用程序（Play控制台）的功能。

  • 编码协助：

    • 在Application.java方法和相应模板之间的导航。

    • 通过相应的Application.java方法的变量更新模板。

    • 关键字，标签，变量，参数和函数的代码完成 。

    • 错误和语法突出显示。

    • 代码格式化和折叠 。

  • 先进的源代码搜索和替换功能 。

  • 结构视图 。

为Play应用程序开发做准备

为Play应用程序开发做准备：

  1. 下载并安装IntelliJ IDEA支持的所需Play框架版本。请参阅安装说明 。

  2. 创建一个新的Play应用程序 play new或获得一个现有的（您将在IntelliJ IDEA中使用的一个）。

  3. 准备要在IntelliJ IDEA中打开它的应用程序 play idea要么play idealize 。

  4. 确保在IntelliJ IDEA中正确指定了Play框架设置 。

  5. 在IntelliJ IDEA中打开应用程序 。

  6. （可选）将项目转换为基于目录的格式 。

创建一个Play应用程序

要创建新的Play应用，请运行play new IntelliJ IDEA外部的命令行外壳中的命令：

  1. 打开命令行外壳。

  2. 切换到要在其中创建Play应用程序的目录。

  3. 运行以下命令：
    play new

    哪里

    是将在当前目录中创建的新Play应用程序目录的名称（例如helloworld ）。

  4. 当要求输入应用程序名称时，键入名称并按Enter 。

结果，在指定目录中创建了具有指定名称的Play应用程序。

准备在IntelliJ IDEA中打开Play应用程序

要准备在IntelliJ IDEA中打开Play应用程序，请运行play idea或者play idealize Play应用程序根目录中的命令：

  1. 打开命令行外壳。

  2. 切换到您的Play应用程序的根目录。

  3. 运行以下命令：
    play idea

    （或者，您可以使用play idealize命令。）

结果，所有必需的IntelliJ IDEA配置文件都在当前目录中创建。这些包括.ipr现在可以在IntelliJ IDEA中打开的项目文件。

指定Play框架设置

在IntelliJ IDEA中，Play框架设置是在“ 播放配置”页面上的“设置 首选项”对话框中指定的。

  1. 打开“ 设置 偏好 设置 ”对话框（例如Ctrl+Alt+S ）。

  2. 在对话框的左侧窗格中，选择“ 播放配置” 。

  3. 在对话框右侧打开的“ 播放配置”页面上：

    • 在主页字段中，指定Play框架安装目录。

    • 在工作目录字段中，指定Play框架工作目录。这是运行play命令行实用程序命令的目录。

    • 根据您的首选项，打开或关闭在控制台上显示运行选项。

  4. 在“ 设置 首选项”对话框中，单击“ 确定 ” 。

在IntelliJ IDEA中打开Play应用程序

要在IntelliJ IDEA中打开Play应用程序，您应该打开.ipr 您先前生成的项目文件：

  1. 选择文件|打开 。

  2. 在“ 打开文件或项目”对话框中，转到您的Play应用程序根文件夹，选择.ipr文件，然后单击确定 。

在打开的项目中，您的Play应用程序由IntelliJ IDEA模块表示。

如有必要，将您的项目转换为基于目录的格式 。

为Play应用程序创建IntelliJ IDEA项目的另一种方法

在IntelliJ IDEA中开始使用Play应用程序最快，最方便的方法是运行play idea命令 ，然后在IntelliJ IDEA中打开生成的.ipr文件 。

或者，您可以使用File | File为Play应用程序源创建一个新项目。 新品|从现有来源进行项目 ，然后将必要的Play框架资产（ \ framework \ lib和\ framework \ play- .jar ）添加到生成的IntelliJ IDEA模块的依赖项中：

  1. 选择文件|新品|现有的项目 。

  2. 在打开的对话框中 ，选择您的Play应用程序根目录。

  3. 在“ 导入项目”向导的第一页上，选择“ 根据现有资源创建项目” ，然后单击“ 下一步” 。

  4. 在向导的下一页的“ 项目位置”字段中，指定Play应用程序根目录的路径。其余设置并不那么重要。单击下一步 。

  5. 请按照向导的说明进行操作。（通常，您要做的就是在接受默认设置的每个页面上单击“ 下一步 ”。）

    创建项目后，添加必要的模块依赖项：

  6. 打开项目结构对话框（例如Ctrl+Shift+Alt+S ）。

  7. 在对话框的左侧窗格中，选择模块 。

  8. 在对话框右侧显示的“ 模块”页面上 ，选择“ 依赖关系”选项卡 ，然后单击“确定”。 并选择Jars或目录 。

  9. 在“ 附加文件或目录”对话框中，转到Play框架安装目录，选择目录framework \ lib和文件framework \ play- .jar ，然后单击OK 。

  10. 在“ 项目结构”对话框中单击“ 确定 ”。

在IntelliJ IDEA中访问play命令行实用程序

如果当前在IntelliJ IDEA中打开了Play应用程序，则可以访问play命令行实用程序（Play控制台）并直接从IDE运行它：

  1. 选择工具|玩Playframework 。

  2. 在“ 播放配置”对话框中，指定“播放框架”设置，然后单击“ 确定” 。 （如果在Play框架设置中关闭了“在控制台上显示运行”选项，则不会显示此对话框。）

结果，在“ 运行”工具窗口中启动了play命令行实用程序。

将播放模块添加到IntelliJ IDEA模块

应将Play应用程序使用的Play模块作为模块内容的根添加到相应的IntelliJ IDEA模块中。例如，如果您的应用程序使用（或将要使用） 安全播放模块：

  1. 打开项目结构对话框（例如Ctrl+Shift+Alt+S ）。

  2. 在对话框的左侧窗格中，选择模块 。

  3. 在对话框右侧显示的“ 模块”页面上 ，选择“ 源”选项卡 ，然后单击“ 添加内容根” 。

  4. 在“ 选择内容根目录”对话框中，转到Play框架安装文件夹，选择目录modules \ secure ，然后单击“ 确定” 。（如果所需的Play模块未包含在Play框架发行版中，则应先下载该模块以使其在本地可用。）

  5. 在“ 项目结构”对话框中单击“ 确定 ”。

运行Play应用

要运行您的播放应用程序，您可以使用play run命令：

  1. 启动播放命令行实用程序 （“ 工具” |“使用Playframework播放” ）。

  2. 在“ 运行工具”窗口中， play ，类型run然后按Enter 。

  3. 打开Web浏览器，然后转到http：// localhost：9000以查看应用程序主页。

或者，您可以创建IntelliJ IDEA 运行/调试配置 ，并使用该配置运行您的应用程序。

要创建用于运行Play应用程序的运行配置，请执行以下操作：

  1. 打开“ 运行/调试配置”对话框（例如， “运行” |“编辑配置” ）。

  2. 请点击Alt+Insert然后选择“ 应用程序” 。

  3. 在以下字段中，指定：

    • 主班。 类型play.server.Server

    • VM选项。 类型-Dapplication.path="."

    • 工作目录。 指定您的Play应用程序根目录。

  4. 如有必要，更改运行配置名称，然后单击确定 。

现在，要执行此运行配置，您可以例如使用“运行” |“运行”。运行 。（有关更多信息，请参见运行应用程序 。）

运行Play应用程序测试

要为您的Play应用程序运行测试，您可以使用play test命令：

  1. 启动播放命令行实用程序 （“ 工具” |“使用Playframework播放” ）。

  2. 在“ 运行工具”窗口中， play ，类型test然后按Enter 。

  3. 打开Web浏览器，然后转到http：// localhost：9000 / @ tests以运行测试。

为了能够使用IntelliJ IDEA运行/调试配置来运行测试，请执行以下操作：

  • 将\ modules \ testrunner \ lib \ play-testrunner.jar添加到模块的依赖项中。

  • 修改运行配置以运行应用程序 ：在VM options字段中，之后-Dapplication.path="." ，键入空格，然后键入-Dplay.id=test

    因此，最后， VM options字段将包含：

    -Dapplication.path="." -Dplay.id=test

    （如果要为测试使用单独的运行配置，则可以创建现有运行配置的副本然后相应地修改该副本。）

调试Play应用程序：过程概述

  1. 创建用于调试的运行/调试配置 。这应该是一种远程类型的配置。此配置中指定的端口应对应于Java调试器端口（默认为8000 ）。

  2. 在代码中设置一个或多个断点 。请参阅断点 。

  3. 如果要使用IntelliJ IDEA运行配置启动应用程序，请修改相应的运行配置 ，或创建一个新的配置 。

  4. 使用以下命令运行应用程序play run命令（没有任何选项和参数）或通过执行相应的运行配置。

  5. 启动用于调试的运行/调试配置。

  6. 在Web浏览器中，转到http：// localhost：9000，然后使用该应用程序尝试到达断点。

  7. 当达到断点时，切换到IntelliJ IDEA并检查可疑代码片段。

  8. 除非您在代码中本地化了问题，否则请继续进行调试。

另请参见调试 。

创建用于调试的运行/调试配置

  1. 打开“ 运行/调试配置”对话框（例如， “运行” |“编辑配置” ）。

  2. 请点击Alt+Insert然后选择远程 。

  3. 在端口字段中，指定8000 。（这是Java调试器将连接到的端口。默认情况下，Play框架使用端口8000。 ）

  4. 如有必要，更改运行/调试配置名称，然后单击“ 确定” 。

修改运行配置以运行应用程序

如果您使用IntelliJ IDEA 运行配置来运行Play应用程序 ，则必须修改此配置，以便Java调试器可以连接到正在运行的应用程序。通过添加相应的JVM选项来完成此操作。可以从用于调试的运行/调试配置中复制必要的选项。

  1. 打开“ 运行/调试配置”对话框（例如， “运行” |“编辑配置” ）。

  2. 在对话框的左侧窗格中，选择用于调试的运行/调试配置。

  3. 请点击包含JVM命令行参数的上部字段的右侧。结果，字段内容被复制到剪贴板。

  4. 选择用于运行应用程序的运行配置，然后将选项粘贴到“ VM options”字段中。该字段最终应包含-Dapplication.path="." ，然后是空格，然后是您刚刚粘贴的选项。这是一个例子：
    -Dapplication.path="." -agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=8000

    （如果您使用相同的运行配置来运行应用程序和测试 ，则可能需要删除-Dplay.id=test ）

  5. 单击确定 。
