解决财产冲突

当IntelliJ IDEA检测到本地文件或文件夹的属性与其服务器版本之间的差异时，与服务器同步期间将报告属性冲突 。IntelliJ IDEA不会尝试自动解决属性冲突，而是在“ 更新 与属性冲突”节点的“ 更新信息”选项卡下显示冲突的文件和文件夹。 版本控制窗口Alt+9 。您必须手动解决属性冲突，然后告诉IntelliJ IDEA将相应的文件和文件夹视为无冲突。

解决财产冲突

  1. 打开 版本控制窗口Alt+9并切换到“ 存储库”选项卡。

  2. 在“ 更改的文件”窗格中，选择有冲突的文件。

  3. 从选择的上下文菜单中选择“ 属性差异 ”，或单击在工具栏上。

  4. 在Subversion属性差异查看器中探索差异 ：

  5. 更新属性，以便解决冲突。

将文件标记为已解决

  1. 在“ 更新信息”标签中 版本控制窗口Alt+9 ，在“ 与属性合并”节点下选择固定文件。如您所见，该文件仍然以红色显示为冲突。

  2. 从选择的上下文菜单中，选择Subversion ，然后选择标记为已解决 。

  3. 对话框关闭后， 的“本地更改”标签 版本控制窗口Alt+9将受影响的文件显示为已更新，并且可用于检入服务器。

  4. 签入已解析的文件。
