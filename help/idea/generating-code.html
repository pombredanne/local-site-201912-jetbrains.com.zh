产生程式码

IntelliJ IDEA提供了多种生成通用代码结构和循环元素的方法，可帮助您提高生产率。这些可以是预定义模板，也可以是自定义模板，它们会根据上下文，各种包装程序和字符自动配对而不同地应用。

此外，IntelliJ IDEA还提供代码完成和Emmet支持。

在主菜单中，选择代码|生成 Alt+Insert打开带有您可以生成的可用结构的弹出菜单。

生成构造函数

IntelliJ IDEA可以生成一个构造函数，该构造函数使用相应参数的值初始化特定的类字段。

为一个类生成一个构造函数

  1. 在代码菜单上，单击生成 Alt+Insert 。

  2. 在“ 生成”弹出窗口中，单击“ 构造函数” 。

  3. 如果该类包含字段，请选择要由构造函数初始化的字段，然后单击OK 。

以下代码片段显示了为类生成构造函数的结果：

公共类MyClass {int aInteger; double bDouble; public MyClass（int myAIntegerParam，double myBDoubleParam）{aInteger = myAIntegerParam; bDouble = myBDoubleParam; }}

生成委托方法

IntelliJ IDEA可以生成将行为委托给您的类的字段或方法的方法。这种方法可以在不直接暴露该字段或方法的情况下访问字段或方法的数据。

生成类的委托方法

  1. 在代码菜单上，单击生成 Alt+Insert 。

  2. 在“ 生成”弹出窗口中，单击“ 代理方法” 。

  3. 选择目标字段或方法，然后单击确定 。

  4. 选择要委派的所需方法，然后单击“ 确定” 。

以下代码片段显示了委托的结果get(i)的方法Calendar另一个班级内的班级：

日历日历； public int get（int i）{return calendar.get（i）; }

生成equals（）和hashCode（）方法

公共布尔等于（Object obj）

前一个方法返回true ，如果作为参数传递给它的对象等于在其上调用此方法的对象。

public int hashCode（）

前一个方法返回调用此方法的对象的整数哈希码值。

为类生成equals（）和hashCode（）

  1. 在代码菜单上，单击生成 Alt+Insert 。

  2. 在“ 生成”弹出窗口中，单击equals（）和hashCode（） 。

  3. 如果要接受子类并在代码生成过程中使用getter，请选择复选框。您也可以从“ 模板”列表中选择一个速度模板以生成代码或创建自定义模板 。单击下一步 。

  4. 选择用于确定相等性的字段，然后单击下一步 。

  5. 选择用于计算哈希码值的字段。您只能从上一步（用于确定相等性）中选择的字段中选择。单击下一步 。

  6. 选择包含非空值的字段。此可选步骤有助于生成的代码避免检查null，从而提高性能。点击完成 。

如果覆盖equals()和hashCode()类中已经存在方法，系统将提示您是否要在生成新方法之前将其删除。

生成获取器和设置器

IntelliJ IDEA可以为您的类中的字段生成访问器和更改器方法（ getter和setter ）。根据JavaBeans API的要求，生成的方法只有一个参数。

getter和setter方法名称由IntelliJ IDEA根据您的代码生成命名首选项生成 。

生成类的获取器和设置器：

  1. 在代码菜单上，单击生成 Alt+Insert 。

  2. 在“ 生成”弹出窗口中，单击以下选项之一：

    • Getter生成访问器方法以获取类字段的当前值。

    • Setter生成用于设置类字段值的增变方法。

    • 使用Getter和Setter生成访问器和更改器方法。

  3. 选择要为其生成getter或setter的字段，然后单击“ 确定” 。

    您可以通过单击添加自定义getter或setter方法 并访问“ 获取/设置模板”对话框。如果字段不在列表中，则已经为其定义了相应的getter和setter方法。

以下代码片段显示了为具有一个字段的类生成getter和setter方法的结果var ：

公共类MyClass {int字段； public int getField（）{返回字段； } public void setField（int field）{this.field = field; }}

生成toString（）

IntelliJ IDEA可以为标准生成替代toString()方法中定义Java.lang.Object 。生成的方法返回类的名称以及具有当前字段值的类字段名称的列表。

覆盖类的toString（）方法：

  1. 在代码菜单上，单击生成 Alt+Insert 。

  2. 在“ 生成”弹出窗口中，单击toString（） 。

  3. 配置以下内容：

    • 选择要用于生成toString()方法。默认情况下，所有可用字段均处于选中状态。单击全选以添加toString()仅返回类名称的方法。

    • 选择所需的生成方式toString() 模板列表中的方法。

    • 如有必要，请选择“ 插入@替代”复选框。有关更多信息，请参见Generate toString（）对话框 。

    • 单击设置按钮以打开toString（）生成设置对话框，您可以在其中调整行为并添加自定义模板 。有关更多信息，请参见生成toString（）设置对话框 。

  4. 单击确定 。

如果toString()方法已在该类中定义，默认情况下，系统会提示您是否要删除此方法，然后再继续操作。您可以使用“ 生成toString（）设置”对话框中的“ 当方法已存在时 ”选项组来更改行为，以自动替换现有方法或生成复制方法。

以下代码检查与toString()方法：

  • 类不覆盖'toString（）'方法，可用于标识其中toString()方法未定义。该检查使用排除设置来忽略带有不应被转储的字段的类。另一个设置是使用与它们的类名匹配的正则表达式排除某些类。默认情况下，它用于排除任何Exception类。

  • 'toString（）'方法中未使用的字段可用于标识未转储到toString()方法。例如，如果您将新字段添加到课程中，但忘记将其添加到toString()方法。更改此检查的严重性，以将错误显示为警告。这将突出显示编辑器中所有未使用的字段，并在滚动条上以黄色标记指示其位置。
