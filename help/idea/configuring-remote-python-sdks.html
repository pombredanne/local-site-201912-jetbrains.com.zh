配置远程Python解释器

以下内容仅在安装并启用Python插件后才有效。

IntelliJ IDEA提供与在远程主机上运行的Python解释器的完全集成。

在开始使用远程解释器之前，请确保已安装并启用了SSH Remote Run ， Vagrant和Docker Integration插件。如果未激活插件 ，请按照“ 管理插件 ”中的说明在“设置/首选项”对话框的“ 插件”页面上启用它们。

根据远程解释器的类型，请遵循以下过程之一：

配置SSH解释器

  1. 导航到文件|项目结构 Ctrl+Shift+Alt+S 。

  2. 在“ 项目结构”对话框中，在“ 平台设置”部分下选择“ SDK ” ，然后单击“确定”。 ，然后从弹出菜单中选择Python SDK 。

  3. 在“ 添加Python解释器”对话框的左侧窗格中，单击“ SSH解释器” 。

  4. 在右侧窗格中，选择新建服务器配置 ，然后指定服务器信息（主机，端口和用户名）。

    或者，您可以选择“ 现有服务器配置”，然后从列表中选择任何可用的部署配置。如果需要，请单击查看所选部署配置的“ 连接设置” ，“ 映射 ”和“ 排除的路径 ”。单击“ 下一步”继续配置解释器。

  5. 在下一个对话框窗口中，提供身份验证详细信息以连接到目标服务器。

    选择密码或密钥对（OpenSSL或PuTTY），然后输入密码或密码。

    单击下一步继续进行最后的配置步骤。

  6. 在下一个对话框窗口中，验证所需Python解释器的路径。您可以接受默认值，也可以指定其他默认值。您必须配置本地项目和服务器之间的路径映射。为此，请单击在“ 同步文件夹”字段旁边，输入本地项目文件夹的路径和远程服务器上的文件夹的路径。

    您还可以选择最低的复选框，以启用将本地更改自动上传到远程服务器的功能。

配置无业游民

  1. 确保在您的计算机上创建了Vagrant实例并正确初始化。

  2. 导航到文件|项目结构 Ctrl+Shift+Alt+S 。

  3. 在“ 项目结构”对话框中，在“ 平台设置”部分下选择“ SDK ” ，然后单击“确定”。 ，然后从弹出菜单中选择Python SDK 。

  4. 在“ 添加Python解释器”对话框的左侧窗格中，单击Vagrant ：

  5. 点击浏览按钮在“ Vagrant实例文件夹 ”字段旁边，并指定所需的Vagrant实例文件夹。

    结果是显示到Vagrant主机URL的链接。

  6. Python解释器路径字段显示所需Python可执行文件的路径。您可以接受默认值，也可以指定其他默认值。

  7. 单击确定 。配置的远程解释器将添加到列表中。

配置WSL

  1. 点击屏幕左下角的Windows按钮并开始输入System Information 。为确保您的系统与WSL兼容，请将Windows升级到最新的可用版本。

  2. 如WSL安装指南中所述，安装Windows Linux子系统并初始化Linux发行版。

  3. 导航到文件|项目结构 Ctrl+Shift+Alt+S 。

  4. 在“ 项目结构”对话框中，在“ 平台设置”部分下选择“ SDK ” ，然后单击“确定”。 ，然后从弹出菜单中选择Python SDK 。

  5. 在对话框的左侧窗格中，单击WSL 。

  6. 选择Linux发行版，然后在所选Linux发行版中指定python可执行文件的路径。

将Docker配置为远程解释器

  1. 确保已在计算机上设置Docker 。

  2. 导航到文件|项目结构 Ctrl+Shift+Alt+S 。

  3. 在“ 项目结构”对话框中，在“ 平台设置”部分下选择“ SDK ” ，然后单击“确定”。 ，然后从弹出菜单中选择Python SDK 。

  4. 在打开的对话框中，选择Docker选项，从下拉列表中选择Docker服务器（如果缺少该服务器，请单击New ... ），然后指定映像名称。

    Python解释器路径应具有默认值，例如， python ：

  5. 单击确定以完成任务。

将Docker Compose配置为远程解释器

  1. 确保已在计算机上设置并运行Docker Compose 。

  2. 导航到文件|项目结构 Ctrl+Shift+Alt+S 。

  3. 在“ 项目结构”对话框中，在“ 平台设置”部分下选择“ SDK ” ，然后单击“确定”。 ，然后从弹出菜单中选择Python SDK 。

  4. 在打开的对话框中，选择Docker Compose选项，从下拉列表中选择Docker服务器Docker Compose服务（此处web ），配置文件（此处docker-compose.yml ）和图片名称（此处python ）。

    接下来，等待IntelliJ IDEA启动Docker-Compose配置以进行扫描和编制索引：

  5. 单击确定以完成任务。

对于任何已配置的Python解释器，您可以：

  • 安装和卸载解释器路径
  • 安装和卸载解释程序包
