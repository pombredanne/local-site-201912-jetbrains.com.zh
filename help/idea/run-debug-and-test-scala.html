IntelliJ IDEA 2019.3帮助

运行，调试和测试Scala

IntelliJ IDEA使您可以像通常使用IntelliJ IDEA中的任何其他应用程序一样运行，调试和测试Scala应用程序。IntelliJ IDEA还允许您在覆盖范围内运行Scala代码并配置代码覆盖率设置。

要在执行运行或调试过程时调整配置设置，请使用“ 运行/调试配置”对话框。要访问该对话框，请选择“运行” |“运行”。从主菜单编辑配置 。

运行Scala应用程序

您可以通过IntelliJ IDEA运行Scala代码，使用sbt shell或使用Scala工作表进行快速代码评估。

通过Intellij IDEA运行Scala应用程序

  1. 与通常在IntelliJ IDEA中创建或导入任何其他项目一样，创建或导入Scala项目。

  2. 在编辑器中打开您的应用程序。

  3. 按Shift+F10执行应用程序。或者，在编辑器的左装订线中，单击图标，然后选择“运行名称” 。

您也可以点击工具栏上的图标运行您的应用程序。

使用sbt shell运行Scala应用程序

您可以使用属于任何sbt项目一部分的sbt shell运行应用程序。

  1. 打开您的sbt项目。

  2. 如果您想将构建和导入委托给sbt，请在sbt项目工具窗口中，单击图标打开sbt设置。
    在sbt settings页面上，选择Use sbt shell for build and import（必需的sbt 0.13.5+）选项，然后单击OK 。

    （当您创建或导入sbt项目时，此选项也可用。）
    如果您不想将构建和导入委托给sbt，您仍然可以在sbt shell中进行操作（ ）并直接从中运行sbt命令 ，包括运行应用程序。

  3. 在“ sbt项目”工具窗口中，单击“ sbt任务”节点。

  4. 在打开的列表中，选择将运行主要方法的运行任务。

    执行结果显示在sbt shell工具窗口中。

调试Scala代码

IntelliJ IDEA允许您使用IntelliJ IDEA调试器或sbt Shell调试代码。

使用Intellij IDEA调试Scala代码

  1. 在编辑器中打开您的Scala应用程序。

  2. 在左侧装订线中，为要调试的代码行设置断点。有关断点的更多信息，请参见“ 断点”主题。

  3. 如果需要，可以访问“ 运行/调试配置”对话框（“运行” |“编辑配置”）并调整设置，但是通常默认设置足以成功启动并完成调试会话。

  4. 按Shift+F9 。或者，在主工具栏上，单击图标以启动调试过程。

  5. 在“ 调试工具”窗口中评估结果。

    有关如何使用“ 调试工具”窗口中的选项的信息，请参阅“ 调试工具窗口参考” 。

使用sbt shell调试Scala代码

  1. 打开您的sbt项目。

  2. 在编辑器中打开您的应用程序。

  3. 在编辑器的左装订线中，为要调试的代码行设置断点。

    您无法调试实际.sbt文件中定义的代码，但是可以调试可从build.sbt调用的Scala文件中的代码。

  4. 在sbt项目工具窗口中，单击图标，以启动sbt shell。

  5. 在sbt shell工具窗口中，单击图标以连接到调试器服务器并开始调试会话。
    IntelliJ IDEA还为调试会话创建运行/调试配置 。

    如果需要，您可以编辑运行/调试配置设置，但是默认设置应足以成功调试代码。

  6. 运行程序。

    在“ 调试工具”窗口中评估结果。

测试Scala应用程序

IntelliJ IDEA允许您使用ScalaTest和Specs2测试您的Scala应用程序。您还可以设置和运行测试范围。

使用Scala测试测试Scala应用程序

  1. 打开您的项目。
    • 如果您有sbt项目，请打开build.sbt文件并为ScalaTest指定以下依赖项：
      libraryDependencies + =“ org.scalactic” %%“ scalactic”％“ 3.0.1”

      libraryDependencies + =“ org.scalatest” %%“ scalatest”％“ 3.0.1”％“ test”

      请点击 sbt项目工具窗口中的图标刷新您的项目，或使用sbt设置中指定的自动导入选项在您每次对build.sbt进行更改时自动刷新您的项目。

    • 如果您有一个常规的Scala项目，请使用“ 项目结构”对话框来配置测试库。

  2. 在编辑器中打开一个要为其创建测试的类，然后将光标放在包含类声明的行中。

  3. 按Ctrl+Shift+T然后选择创建新测试 。

  4. 在打开的对话框中，指定测试设置，然后单击“ 确定” 。

  5. 在编辑器中打开测试，按Ctrl+Shift+F10或右键单击测试类，然后从上下文菜单中选择Run'test name' 。

  6. IntelliJ IDEA自动为测试创建运行/调试配置，但是如果要编辑配置中的设置，请单击“运行” |“调试”。在主菜单上编辑配置 。

  7. 在右侧的“运行/调试配置”对话框中，指定测试套件的设置，然后单击“ 确定” 。
    该配置具有标准选项，您可以在“ 执行测试”部分中找到更多详细信息。但是，您也可以指定以下与Scala相关的选项：

    • 您可以选择“ 使用sbt shell”复选框以通过sbt shell运行测试，并选择“ 将sbt与UI一起使用”以与平台测试运行器相同的格式显示测试结果。

    • 您可以选择“ 测试种类”选项以指定要运行的测试种类 。例如，您可以选择正则表达式并设置类和测试模式。

  8. 在主工具栏上，单击图标以运行测试。

  9. 在“ 运行”工具窗口中评估结果。

使用Specs2测试一个Scala应用程序

使用Specs2测试Scala应用程序的过程与使用ScalaTest测试Scala应用程序部分中描述的过程相同，除了以下选项：

  • 您需要为sbt项目指定以下依赖项：
    libraryDependencies ++ = Seq（“ org.specs2” %%“ specs2-core”％“ 3.9.5”％“ test”）

    对于常规的Scala项目，请使用“ 项目结构”对话框来配置测试库。

  • 不可用以与平台测试运行器相同的格式显示测试结果的“ 将UI与sbt一起使用”选项。

Scala中的测试范围

IntelliJ IDEA允许您使用ScalaTest或Specs2测试范围。

您可以在范围内运行测试，也可以在Scala项目中测试整个范围。

  1. 创建您的代码。检查以下示例：

  2. 在编辑器中，根据您的测试范围，您可以执行以下操作：

    • 如果打开范围内测试之一的上下文菜单，则可以仅为该范围内的指定测试创建运行配置。

有覆盖范围地运行Scala测试

IntelliJ IDEA使您可以在代码覆盖范围内运行测试套件。

  1. 打开您的项目。

  2. 在编辑器中打开有问题的测试。

  3. 在左侧装订线中，单击图标，然后选择“使用覆盖率运行'名称'”选项。
    IntelliJ IDEA运行测试并显示带有代码覆盖率信息的Coverage工具窗口。

    如果需要，您还可以调整默认代码覆盖率设置或代码覆盖率行为。

    如果您已经运行了测试套件，则IntelliJ IDEA会自动创建运行/调试配置。您可以打开创建的运行配置并调整设置。
    如果要为测试套件创建新的运行配置，请执行以下操作：

    1. 从主菜单中，选择运行|编辑配置 。

    2. 在“ 运行/调试配置”对话框中，单击图标从左侧的选项。

    3. 从打开的列表中，选择所需的配置。

    4. 在右侧的选项中，单击“ 代码覆盖率”选项卡。

    5. 调整默认设置，然后单击“ 确定” 。
      有关更多信息，请参阅配置覆盖率测量

    您还可以调整代码覆盖率行为。

    1. 按Ctrl+Alt+S打开“ 设置/首选项”页面。

    2. 从左侧的选项中，选择“ 构建”，“执行”，“部署” |“部署”。覆盖范围 。

    3. 在右侧的选项中，调整设置，然后单击“ 确定” 。

或者，您可以选择运行|从主菜单运行Coverage ，以覆盖运行代码。
