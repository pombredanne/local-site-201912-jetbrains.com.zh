使用AspectJ编译器（ajc）

默认情况下，IntelliJ IDEA使用javac编译器。要使用AspectJ编译器ajc （而不是javac或与javac结合使用），应更改相应的IDE设置 。

在项目级别指定的ajc设置可以在各个模块级别进行微调。与模块关联的AspectJ构面用于此目的。

请注意， ajc未与IntelliJ IDEA捆绑在一起，应单独下载。

ajc是AspectJ发行版的一部分，您可以从AspectJ网站下载。

优化编译性能：将ajc与javac结合使用

IntelliJ IDEA使您可以将ajc与javac结合使用，而无需在IDE设置中切换编译器。

首先，您应该选择ajc作为项目编译器（“ Java编译器”页面上的“ 使用编译器”字段 ）。

如果您还希望使用javac ，请打开委托给Javac选项 。如果启用此选项，则没有方面的模块将使用javac编译 （通常更快），而包含方面的模块将使用ajc编译。（如果关闭此选项，则将ajc用于项目中的所有模块。）

您可以在各个模块级别微调编译器ajc和javac之间的任务分配。对于仅包含以下形式的方面的模块@Aspect -带注释的Java类（在Java文件中），可以指定ajc仅用于后编译编织。如果这样做，将使用javac编译所有源文件，然后将ajc应用于已编译的类文件以进行编织。结果，整个过程（编译+编织）将花费更少的时间。

如果“ 委托Javac”选项处于启用状态，则通过打开与模块关联的AspectJ构面中的相应选项，可以启用ajc的编译后编织模式。

请注意，您不应为包含代码样式方面（在.aj文件中定义的方面）的模块打开此选项。

控制AJC AspectPath

您可以控制AJC aspectpath选项分别针对每个模块，请参阅在模块级别微调ajc的使用 。

模块aspectpath可能是由于将适当配置的Maven项目导入IntelliJ IDEA而自动设置的。

选择ajc作为项目编译器并指定其设置

  1. 在“ 设置/首选项”对话框中Ctrl+Alt+S ，选择构建，执行，部署|编译器Java编译器 。

  2. 从“ 使用编译器”列表中，选择Ajc 。

  3. 如有必要，请指定字节码版本。大致来说，这些是最低目标JVM版本。

  4. 在“ Ajc编译器的路径”字段中指定编译器的路径 。

    您可以在字段中键入路径，或单击然后在相应的对话框中选择所需的文件。

    （所需的文件是\ lib中的 aspectjtools.jar 。）

  5. 如果需要，指定命令行选项要传递给在命令行参数字段中的编译器。

    您可以在字段中直接输入参数，或单击打开文本输入区域较大的“ 命令行参数”对话框。

  6. 如果要使用javac编译不包含任何方面的模块，请选中“ 委托给Javac”复选框。

    请注意，可以在单个模块级别上微调ajc和javac之间的任务分配。

  7. 应用更改并关闭对话框。

在模块级别微调AJC的使用

为了能够在模块级别微调ajc的使用，您应该向相应的一个或多个模块添加AspectJ构面。

完成此操作后，可以指定ajc仅用于后编译编织。您还可以指定aspectpath用于模块。

  1. 将AspectJ构面添加到感兴趣的模块。有关相应的说明，请参阅添加框架和技术 。

  2. 从主菜单中，选择文件|项目结构刻面| AspectJ 。

  3. 如果只想使用ajc来编织已编译的类文件（在这种情况下，源代码使用javac进行编译 ），则在Compiler下 ，选择Post-compile weave mode复选框。

    如果模块包含.aj文件，则不要选中此复选框。

  4. 形成aspectpath对于模块，使用选项Alt+Insert ， Alt+Up / Alt+Down和Alt+Delete 。

  5. 应用更改并关闭对话框。
