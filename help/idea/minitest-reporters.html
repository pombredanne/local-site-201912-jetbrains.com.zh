总览

IntelliJ IDEA可帮助您使用MiniTest测试框架编写测试。MiniTest随Ruby 1.9.x捆绑在一起，但我们建议使用minitest作为最新版本。

先决条件

在开始之前，请确保：

  • minitest已下载并安装在您的计算机上。

    IntelliJ IDEA支持高于3.1.0的迷你测试版本

  • minitest-reporters 版本0.5.0或更高版本已下载并安装在您的计算机上。与IntelliJ IDEA测试运行器集成时需要此gem。

    对于MINITEST> = 5.0.0，不需要MINITEST-记者宝石。

  • 对于Windows，将下载并安装win32Console 版本“ 1.3.0” 。

  • 这些宝石应该正确地连接到您的项目。例如，如果使用Bundler来管理gem，则应将所需的gem添加到项目的Gemfile中。

    例如， Gemfile可以包含以下代码：

    组：测试是否RUBY_PLATFORM =〜/（win32 | w32）/ gem“ win32console”，'1.3.0'end gem“ minitest” gem'minitest-reporters'，'> = 0.5.0'gem'cucumber-rails'结束
  • 取决于版本minitest-reporters gem，将以下代码片段之一添加到测试中（如果使用的是Rails应用程序，则最好使用test / test-helper.rb文件来完成）：

    • 0.8.0版及更高版本

      需要'minitest / reporters'MiniTest :: Reporters.use！

    • 旧版本
      如果ENV [“ RM_INFO”] ||要求'minitest / reporters'MiniTest :: Unit.runner = MiniTest :: SuiteRunner.new ENV [“ TEAMCITY_VERSION”] MiniTest :: Unit.runner.reporters << MiniTest :: Reporters :: RubyMineReporter.new elsif ENV ['TM_PID'] MiniTest :: Unit.runner.reporters << MiniTest :: Reporters :: RubyMateReporter。新的其他MiniTest :: Unit.runner.reporters << MiniTest :: Reporters :: ProgressReporter.new结束

      上面的代码启用了特定的结果报告程序，该报告程序取决于启动测试的环境：IntelliJ IDEA，TeamCity CI服务器或TextMate文本编辑器/终端。当您在使用不同的IDE或文本编辑器的混合团队中工作时，此方法很有用。

      在运行测试之前，在测试初始化时执行此代码很重要。

您可以使用上下文菜单命令“ 跳转到测试”直接从测试树跳转到测试主体。

命名

最小测试文件的名称必须匹配以下模式，这一点很重要：

* _test`.rb test_` *`.rb

测试用例类应具有后缀Test ：

FooTest <验收测试

此限制与最小测试无关 ，而是由IntelliJ IDEA的代码洞察力强加的。
