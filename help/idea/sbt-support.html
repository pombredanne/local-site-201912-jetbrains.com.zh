SBT

导入sbt项目

  1. 单击导入项目或在欢迎屏幕上打开 。

  2. 在打开的对话框中，选择一个包含sbt项目或build.sbt的目录 。单击确定 。

  3. 请遵循“ 导入项目”向导中建议的步骤。
    您可以使用建议的默认设置，因为它们足以成功导入您的项目。
    我们建议您在使用代码生成或其他功能来修改sbt中的生成过程时，启用“ 使用sbt shell进行生成和导入（需要sbt 0.13.5+）”选项。如果您的sbt项目不在IntelliJ IDEA项目根目录中，建议您跳过此选项。
    您也可以为项目中的模块分组选择适当的选项。

要创建新项目，请启动“ 新建项目”向导，并按照向导中建议的步骤操作，例如：

  1. 选择Scala和sbt 。

  2. 指定项目的名称，位置，JDK以及sbt和Scala版本。（将自动获取sbt和Scala版本。）

确保sbt和Scala版本的兼容性

通常，您在团队中共享您的项目，并且需要使用特定版本的sbt。

您可以在项目的build.properties文件中覆盖sbt版本。

当您尝试导入包含旧版本sbt的sbt项目时，可能会收到错误消息。我们建议您升级到与Scala 2.12版兼容的sbt 1.0版及更高版本（需要Java 8）。

  1. 创建或导入您的sbt项目。

  2. 在“ 项目”工具窗口的源根目录中，找到build.properties文件，然后在编辑器中将其打开。

  3. 在编辑器中，显式指定要在项目中使用的sbt版本。
    sbt.version = xxx

    请注意，较新的sbt版本（1.0 +）如果不存在，将自动创建build.properties文件。

  4. 重新导入您的项目。（点击在sbt项目工具窗口中。）

管理SBT项目

sbt项目结构

创建或导入sbt项目时，IntelliJ IDEA会生成以下sbt结构：

  • sbt项目（正确的构建），它定义一个项目并包含build.sbt文件， src和目标目录，模块；与常规项目有关的任何事情。

  • 在项目子目录中定义的sbt build项目。它包含作为构建定义一部分的其他代码。

  • sbt项目工具窗口包含您可以执行的sbt任务，命令和设置。

当您处理sbt项目时，您可以使用build.sbt文件对项目进行主要更改，因为IntelliJ IDEA会将sbt配置视为唯一的事实来源。

有关sbt项目结构的更多信息，请参见相关的sbt文档 。

将库添加到sbt项目

您可以通过build.sbt文件添加sbt依赖项 ，也可以使用import .scala文件中的语句。

  1. 在编辑器中打开一个.scala文件。

  2. 指定要导入的库。

  3. 将光标放在未解决的包装上，然后按Alt+Enter 。

  4. 从可用的意图操作列表中，选择添加sbt依赖项 。

  5. 按照打开的向导中建议的步骤操作，然后单击“ 完成” 。

  6. IntelliJ IDEA下载工件，将依赖项添加到build.sbt文件和sbt项目工具窗口。

  7. IntelliJ IDEA一旦检测到build.sbt中的更改，就会出现一条通知，提示您刷新项目。刷新您的项目。（点击在sbt项目工具窗口中。）
    或者，使用位于sbt设置中的自动导入选项。

使用sbt shell

sbt shell嵌入在sbt项目中，并且在项目开始时可用。您可以使用sbt Shell执行sbt命令和任务，运行和调试项目。

  • 要启动sbt shell，请按Ctrl+Shift+S （对于Windows）或⌘+⇧+S （对于Mac OS X）。或者，单击在屏幕底部的工具栏上。

  • 要将sbt shell用于构建和导入过程，请选择sbt设置中的使用sbt shell进行构建和导入（需要sbt 0.13.5+）选项，并执行使用sbt shell运行Scala应用程序部分中所述的步骤。请注意，sbt版本为0.13.16。推荐+ / 1.0.3。+。

  • 要使用SBT外壳进行调试，请参阅与SBT壳调试节。

  • 要从sbt shell运行测试：

    1. 打开运行/调试配置（ “运行” |“编辑配置” ）。

    2. 创建测试配置，然后从可用设置中选择use sbt选项。

运行sbt任务

  • 您可以通过从sbt项目工具窗口的sbt Tasks目录中选择所需的任务来运行sbt任务。

  • 您可以在sbt shell中手动输入任务（支持代码完成），然后直接在其中运行。

  • 您可以为任务创建运行配置。例如，您可以创建一个自定义任务，该任务不属于sbt项目工具窗口中的任务列表。

    1. 打开Shift+Alt+F10运行配置。

    2. 指定运行配置设置，然后单击“ 确定” 。如果需要，可以在运行配置之前添加其他配置或要执行的任务。点击 启动前部分中的图标，然后从打开的列表中选择需要执行的操作。

  • 当您在无干扰的模式下工作时（没有工具栏和工具窗口），可以从“ 运行任何内容”窗口中运行sbt任务或命令。按Ctrl两次打开它，键入您的命令，然后按Enter 。

    您可以查看“ 母带化快捷方式”部分以了解更多信息。

IntelliJ IDEA在sbt shell窗口中显示结果。

您可以像执行sbt任务一样执行sbt 命令和设置 。

使用sbt设置

要访问sbt设置，请单击在sbt项目工具窗口中。
您可以将sbt设置用于以下重要操作：

  • 如果您希望sbt每次对build.sbt进行更改时都自动刷新项目，请选择Use auto-import 。

  • 要将运行中的构建委托给sbt，请选择使用sbt shell进行构建和导入 。

  • 要通过sbt shell调试代码，请选择启用调试sbt shell调试选项以启用调试按钮在sbt shell工具窗口中。要启动调试会话，只需单击此按钮。有关调试的更多信息，请参见使用sbt进行调试 。

  • 要更改项目中的.ivy缓存位置或设置其他sbt属性 ，请使用VM参数字段。

要检查最常见的sbt问题和解决方法，请参阅sbt故障排除部分。
