运行/调试配置：Flash App

使用此对话框可以创建或编辑Flash App 运行/调试配置 ，使您可以运行和调试ActionScript和Flex应用程序。

请注意，此运行/调试配置类型不适用于输出为RLM或库的构建配置 。

另请注意，这些设置取决于在关联的构建配置（Web，桌面（AIR）或移动版（AIR Mobile））中指定的目标平台。

特定于Web的应用程序的设置

项目

描述

推出什么

选择以下选项之一：

  • 建立输出。 使用此选项可通过关联的HTML包装器运行生成的SWF文件。

  • URL或本地文件。 使用此选项可在Web浏览器中打开指定的URL，或运行指定的本地.swf文件（直接或使用相应的.html包装器）。

    在字段中输入所需的URL。通常，这将类似于http（s）：// ： / 。

    如果是本地文件，则可以使用然后在打开的对话框中选择所需的.swf或.html文件。

与启动

指定用于运行应用程序的程序。您可以使用：

  • 系统默认应用程序，即与操作系统中的目标文件类型（HTML或SWF）关联的程序。通常，这是一个Web浏览器。对于SWF文件，这也可能是独立的Flash播放器。

  • Web浏览器。

  • Flash播放器。

要选择所需的程序，请单击在启动方式字段的右侧，然后在启动方式对话框中指定程序：

  • 要选择系统默认应用程序，只需单击系统默认应用程序 。

  • 要选择浏览器，请单击浏览器，然后从列表中选择所需的浏览器。

    此外，您可以访问“ Web浏览器”对话框来调整Web浏览器设置。要打开此对话框，请使用在列表旁边。

  • 要选择Flash Player，请单击Flash Player ，然后单击“确定”。 Flash Player字段旁边。然后，在打开的对话框中指定所需Flash播放器的位置。

    • 新实例 （仅在macOS上可用）。如果要启动Flash Player的新实例，请选中此复选框。否则，如果Flash Player已经在运行，则将打开正在运行的实例的新窗口。

使用SDK中的调试器

指定包含应使用的调试器的Flex SDK。（如果使用的是Flex SDK 3进行编译，建议使用Flex SDK 4进行调试。）

如果在IntelliJ IDEA中已经定义了相应的SDK，请从列表中选择SDK。否则，请点击Shift+Enter并在打开的“ 配置SDK”对话框中添加必要的SDK的定义。

将SWF文件放置在本地信任的沙箱中

如果选择了Build output选项 ：如果要将应用程序SWF文件注册为受信任的，请选中此复选框。

受信任的SWF文件可以与任何其他SWF文件进行交互。他们可以从任何地方（远程或本地）加载数据。

从技术上讲，将受信任的SWF文件分配给本地受信任的沙箱 。

AIR调试启动器选项

如有必要，请指定AIR Debug Launcher选项。

使用与指定程序参数相同的规则。

程序参数

指定要传递给应用程序的参数。

  • 使用空格分隔各个参数。

  • 如果参数包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg要么"some arg" 。

  • 如果参数包含双引号（例如，作为参数的一部分），请使用反斜杠对双引号进行转义，例如， -Dmy.prop=\"quoted_value\" 。

特定于以移动设备为目标的应用程序（AIR Mobile）的设置

项目

描述

运行

指定您是要使用仿真器还是真正的Android或iOS设备来运行或调试应用程序：
  • 仿真器。 选择此选项以使用内置仿真器。从列表中选择所需的目标设备。列表右侧显示的数字是所选目标设备的屏幕特征，仅供参考。

    • 屏幕是应用程序可用的屏幕尺寸。

    • 全屏是设备的全屏尺寸。

    • ppi （每英寸像素）是屏幕像素密度。

    如果列表中没有必要的设备，则可以选择其他并在相应的字段中指定所需的屏幕参数。

  • Android设备。 选择此选项可以使用Android移动设备运行或调试您的应用程序。

    • 为了能够使用Android设备来运行或调试您的应用程序，必须在关联的构建配置（“ Android”选项卡上的“ 启用”选项 ）中为Android打包应用程序。

    • 通常，使用Android设备运行或调试应用程序时，假定已在目标设备上安装了应用程序包。为使此操作成功，必须使用USB将移动设备连接到计算机。

  • iOS模拟器。 （此选项仅在Mac计算机上可用。）选择此选项可以使用iOS设备模拟器运行或调试您的应用程序。在“ SDK”字段中，指定要使用的Apple iOS SDK的路径。（模拟器包含在Apple iOS SDK中。）

    您可以点击Shift+Enter然后在打开的对话框中选择SDK安装文件夹。

    为了能够使用iOS模拟器运行或调试您的应用程序，必须在关联的构建配置（“ iOS”选项卡上的“ 启用”选项 ）中为iOS打包应用程序。

  • iOS设备。 选择此选项可以使用iOS移动设备运行或调试应用程序。如有必要，请启用快速应用程序打包。（如果启用了快速打包，则将解释ActionScript字节码，而不将其翻译为机器代码。结果，打包的执行速度更快，但代码执行速度较慢。）

    • 为了能够使用iOS设备运行或调试您的应用程序，必须在关联的构建配置（“ iOS”选项卡上的“ 启用”选项 ）中为iOS打包应用程序。

    • 通常，使用iOS设备运行或调试应用程序时，假定已在目标设备上安装了应用程序包。为使此操作成功，必须使用USB将移动设备连接到计算机。

    • 如果您使用的是AIR SDK 3.4或更高版本，则该应用程序包将自动安装在目标设备上。对于早期版本的SDK，您必须手动启动安装。

通过设备调试

对于Android或iOS设备：指定在安装和启动应用程序后该设备将如何与您的计算机通信：
  • 网络。 如果设备要通过网络与计算机通信，请选择此选项。

    在设备上安装应用程序包需要USB连接。

  • USB。 如果设备将使用USB连接到计算机，请选择此选项。如有必要，请更改IntelliJ IDEA建议的端口。

ADL选项（仿真器）

对于仿真器：如有必要，请指定AIR Debug Launcher （ADL）选项。

使用与指定程序参数相同的规则。

应用描述符（仿真器）

对于仿真器：指定要使用的应用程序描述符。可用选项是指关联的构建配置中与描述符相关的设置。

  • 为Android设置表示在Android选项卡上的相应设置。

  • 针对iOS的设置是指“ iOS”标签上的设置。

根据构建配置中的设置，可能会出现以下情况：

  • 设置为Android ： 。 禁止为Android生成描述符。如果要启用相应的选项，请使用Android选项卡上的已启用复选框。

  • 如Android所设定：已产生。 将使用自动生成的描述符。如果要改为使用基于模板的描述符，请选择“ 自定义模板”选项并指定模板。

  • 设置为Android： .xml。 相应的自定义模板将用于生成描述符。的在这种情况下，将是将使用的模板文件的名称。

对于iOS选项，也可能有相应的情况。

启动前选项

指定在开始运行/调试配置之前应执行的任务。指定的任务按照它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert 单击此图标可将任务添加到列表中。选择要添加的任务：

  • 运行外部工具。 选择此选项可运行IntelliJ IDEA外部的应用程序。在打开的对话框中，选择应运行的一个或多个应用程序。如果尚未在IntelliJ IDEA中定义必要的应用程序，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 使。 选择此选项可以编译项目或模块。在制作模块命令将被执行，如果在运行/调试配置中指定一个特定的模块，所述生成项目命令否则。

    如果在编译过程中发生错误，则IntelliJ IDEA不会尝试启动运行/调试配置。

  • 做，没有错误检查。 与Make选项相同，但是IntelliJ IDEA不管编译结果如何，都将尝试启动运行/调试配置。

  • 建立工件。 选择此选项有一个神器内置或假象。在打开的对话框中，选择应构建的工件。

    另请参阅使用工件 。

  • 运行其他配置。 选择此选项可以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

    仅当您在当前项目中已经具有至少一个运行/调试配置时，此选项才可用。

  • 运行Ant目标。 选择此选项可以运行Ant目标。在打开的对话框中，选择要运行的目标。有关更多信息，请参见Ant 。

  • 生成CoffeeScript源映射。 选择此选项可生成CoffeeScript源的源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

    有关更多信息，请参见CoffeeScript 。

  • 运行Maven目标。 选择此选项可以运行Maven目标。在打开的对话框中，选择要运行的目标。

    有关更多信息，请参见Maven 。
