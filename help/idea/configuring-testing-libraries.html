配置测试库

JUnit和TestNG的库随IntelliJ IDEA一起提供，但是默认情况下不包含在项目或模块的类路径中。因此，在创建测试类时，不会解析对TestCase类或测试注释的引用。

要将必需的库添加到类路径，可以使用向模块添加依赖项的一般过程。相应的库位于以下目录中：

  • JUnit库（ hamcrest-core-1.3.jar和junit-4.12.jar ）： \ lib 。

  • TestNG库testng-jdk15.jar ： \ plugins \ testng \ lib 。

IntelliJ IDEA可以将必要的库自动添加到类路径。为类创建测试或为测试编写代码时，相应的功能可用。

为类创建测试时，将测试库添加到类路径中

  1. 在编辑器中，将光标放在包含类声明的行中。

  2. 按Alt+Enter查看可用的意图动作 。

  3. 选择创建测试 。

  4. 在“ 创建测试”对话框中，在通知您未找到相应库的文本右侧，单击“ 修复” 。

编写测试代码时，将测试库添加到类路径中

  1. 在测试类的源代码中，将光标放在未解析的引用中TestCase或注释。

  2. 按Alt+Enter查看可用的意图动作 。

  3. 选择添加到类路径 。
