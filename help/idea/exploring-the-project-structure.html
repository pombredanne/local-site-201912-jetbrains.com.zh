探索项目结构

Android项目主要是Java项目，因此它包含您在Java项目中找到的许多标准文件夹。让我们了解更多。

在本教程中：

  • 打开项目视图
  • 浏览与代码相关的文件夹

打开项目视图

浏览与代码相关的文件夹

一个Android项目包含以下关键文件夹和文件：

  • .idea文件夹：包含一些子文件夹和各种XML文件，其中大多数包含内部IntelliJ IDEA信息和常规设置。通常，不需要编辑此文件夹的内容。

  • src文件夹：包含组成应用程序的所有源文件（活动，帮助程序类，等等）。您可以在src文件夹下构建子文件夹的任何层次结构，以更好地反映应用程序的结构和复杂性。

  • res文件夹：包含所有项目资源，例如可绘制资源，布局等（有关更多详细信息，请参见浏览资源文件夹 ）。

  • libs文件夹：包含要从应用程序的源文件中引用的所有类库（ .jar文件）。您可以将.jar文件从磁盘拖动到此文件夹中。

浏览资源文件夹

res文件夹包含您的应用程序使用的所有外部资源，例如图像资产，布局文件，字符串，菜单等。大多数资源（图像除外）均通过.xml文件表示。res文件夹通常包含以下子文件夹：

  • drawable ：包含您在应用程序中引用的所有图像。实际上，有四个不同的可绘制文件夹设计用于包含具有不同屏幕密度（表示为dpi-每英寸点数）的设备的不同分辨率的图像：

    • hdpi（高密度，240）

    • ldpi（低密度120）

    • mdpi（中密度，160）

    • xhdpi（超高密度，320）

    为了支持不同的设备和多个屏幕，需要设计不同分辨率的可绘制资源（有关更多详细信息，请参阅支持多个屏幕 ）。

  • layout ：包含用于定义活动或应用程序小部件的用户界面的布局文件.xml 。您可以手动编辑布局定义文件，也可以通过集成的图形设计器（有关更多详细信息，请参阅《 Android应用程序的设计布局》 ）进行编辑。

  • values ：包含声明字符串，图形样式和颜色的.xml文件。通常，存储在这些文件中的数据以名称/值对的形式表示。

  • menu ：包含应用程序要使用的菜单的定义。在创建新项目时不会生成菜单文件夹，并且仅在创建第一个菜单后才会出现在项目结构中。

探索AndroidManifest.xml文件

每个Android应用程序的根目录中都必须具有AndroidManifest文件。此文件包含有关Android操作系统处理的应用程序的常规信息。此信息对于运行应用程序至关重要。

除其他外， AndroidManifest.xml文件声明了程序包名称（用作应用程序的唯一标识符），以及运行该应用程序的设备所需的最低版本的Android SDK。它还在代码中声明了操作系统启动应用程序的入口点，以及应用程序所需的权限。有关AndroidManifest文件的更多详细信息，请参见App Manifest 。
