仅当启用“ FTP / SFTP连接”插件时，此选项才可用。 的默认情况下，插件已激活。如果禁用了插件 ，请按照“ 管理插件”中所述在“ 插件”页面上启用它。

选择此选项可使用服务器访问配置来配置对远程主机上的Node.js解释器的访问 。仅当您具有至少一个SFTP类型的服务器访问配置时 ，此选项才可用，请参阅创建远程服务器配置 。

在“ 部署配置”列表中， 选择您希望IntelliJ IDEA连接到目标主机的SFTP类型的服务器访问配置 。如果在所选配置中指定的设置确保成功连接，则IntelliJ IDEA在“ 部署主机URL”字段中将目标主机的URL地址显示为链接。

要使用解释器配置，您需要以下路径映射 ： 设置项目文件夹，服务器上用于将项目文件复制到的文件夹和URL地址之间的对应关系，以访问服务器上复制的数据。默认情况下，IntelliJ IDEA从所选的服务器访问（部署）配置中检索路径映射。如果配置不包含路径映射，则IntelliJ IDEA将显示相应的错误消息。

SSH凭证

选择此选项可配置通过SSH凭据访问远程主机上的Node.js解释器。在对话框的字段中，指定以下内容：

  • 主机：在此字段中，输入安装解释程序的主机的名称。

  • 端口：在此字段中，键入远程主机上的SSH服务器侦听的端口。默认端口号是22。

  • 用户名：在字段中，输入用于在SSH服务器上注册的用户名。

  • 身份验证类型：从此列表中选择身份验证方法。

    • 密码 ：使用密码访问主机。指定密码，然后选择“ 保存密码”复选框以将密码保存在IntelliJ IDEA中。

    • 密钥对（OpenSSH或PuTTY） ：对密钥对使用SSH身份验证 。 要应用此身份验证方法，您必须在客户端计算机上具有私钥，而在远程服务器上必须具有公钥。IntelliJ IDEA支持使用OpenSSH实用程序生成的私钥。

      指定存储私钥的文件的路径，然后在相应的字段中键入密码（如果有）。要使IntelliJ IDEA记住密码，请选中“ 保存密码”复选框。

    • OpenSSH配置和身份验证代理 ：使用由凭证帮助程序（例如Windows上的Pageant或Mac和Linux上的ssh-agent ）管理的SSH密钥。

要使用解释器配置，您需要以下路径映射 ： 设置项目文件夹，服务器上用于将项目文件复制到的文件夹和URL地址之间的对应关系，以访问服务器上复制的数据。IntelliJ IDEA首先尝试通过处理所有可用的应用程序级配置来检索路径映射本身。如果IntelliJ IDEA找到与上述主机具有相同主机的配置，则在“ 主机”字段中，来自这些配置的映射将自动合并。如果找不到与此主机的配置，则IntelliJ IDEA将显示一条错误消息，通知您未配置路径映射。

要解决此问题，请按照“ 创建远程服务器配置 ”部分“ 映射”中的说明 ，在“ 构建，执行，部署”节点下打开“ 部署”页面，选择服务器访问配置，切换到“ 映射”选项卡，然后将本地文件夹映射到服务器上的文件夹。 服务器上的本地文件夹到文件夹以及访问它们的URL地址 。

码头工人

仅当按照管理插件中所述安装并启用Node.js ， Node.js远程解释器和Docker集成插件时，此选项才可用。

选择此选项以使用在Docker容器中运行的Node.js解释器。

  1. 在服务器字段中， 指定要使用的Docker配置 ，有关详细信息，请参阅配置Docker守护程序连接设置 。从列表中选择配置或单击并在打开的Docker页面上创建新配置。

  2. 在图片名称字段中， 指定要使用的基本Docker映像。从列表中选择以前下载的图像或您的自定义图像之一，或手动键入图像名称，例如， node:argon要么mhart/alpine-node 。当您稍后启动运行配置时，Docker将在您的机器上搜索指定的映像。如果搜索失败，将从Docker Registry页面上的Docker Official Images存储库中下载映像 。

  3. Node.js解释器路径字段显示指定图像中默认Node.js解释器的位置。

  4. 当您单击OK时 ，IntelliJ IDEA将关闭Configure Node.js Remote Interpreter对话框，并带您到Node.js Interpreters对话框，在该对话框中将新的解释器配置添加到列表中。单击“ 确定”返回到运行配置。

Docker撰写

仅当按照管理插件中所述安装并启用Node.js ， Node.js远程解释器和Docker集成插件时，此选项才可用。

选择此选项以使用Docker Compose文件docker-compose.yml中定义的Node.js解释器配置。请注意，此文件必须具有node要么npm在里面command字段，例如， command: node ./src/app.js 。有关更多详细信息，请访问Docker官方网站 。

  1. 在服务器字段中， 指定要使用的Docker配置 ，有关详细信息，请参阅配置Docker守护程序连接设置 。从列表中选择配置或单击并在打开的Docker页面上创建新配置。

  2. 在“ 配置文件”字段中，指定定义应用程序服务的docker-compose.yml文件 。

  3. 从服务列表中，选择要运行的服务。

  4. （可选）在“ 环境变量”字段中，定义环境变量。有关详细信息，请参见使用环境变量 。

Node.js解释器路径

在这个领域里， 根据所选远程开发环境的配置，指定Node.js可执行文件的位置。

  • 对于远程主机，默认情况下，IntelliJ IDEA建议使用/ usr / bin / node文件夹。

  • 对于Vagrant实例，默认情况下，IntelliJ IDEA建议使用/ usr / bin / node文件夹。

  • 对于Docker容器，默认情况下，IntelliJ IDEA建议使用节点文件夹。

当您单击OK时 ，IntelliJ IDEA会检查Node.js可执行文件是否实际存储在指定的文件夹中。

  • 如果找不到Node.js可执行文件，则IntelliJ IDEA将显示一条错误消息，询问您是否继续搜索还是保存解释器配置。

  • 如果找到了Node.js可执行文件，则返回“ Node.js解释器”对话框，其中显示安装文件夹和检测到的Node.js解释器版本。
