准备开发适用于PHP应用程序的Google App

所描述的功能由Google App Engine支持的PHP插件处理。从2018.1。版本开始，该插件不再维护，并且与IntelliJ IDEA不兼容。

IntelliJ IDEA负责创建特定的目录结构和设置。IntelliJ IDEA可以为特定于PHP的项目存根生成Google App Engine，也可以在现有项目中启用对PHP的Google App Engine支持。

在你开始前

  1. 下载并安装2.7版Python 。

  2. 下载并安装适用于PHP的Google App Engine 。

  3. 如“ 管理插件”中所述，在“ 插件”页面上安装Google App Engine支持的PHP插件。 的该插件未与IntelliJ IDEA捆绑在一起，但可以如管理插件中所述从JetBrains插件存储库安装。

  4. 在http://www.google.com上创建一个Google帐户。

  5. 在Google App Engine的“ 创建应用程序”页面上创建应用程序 ，并记住其应用程序ID，您将必须在创建项目期间指定它。

为PHP应用程序创建Google App Engine的项目存根

  1. 选择文件|新品|从主菜单中的项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 新建项目”对话框中，在左侧窗格中选择“ PHP ”。

  3. 在右侧窗格中，选择“ App Engine项目” ，然后单击“ 下一步” 。

  4. 在向导的第二页上，指定以下内容：

    1. 项目的名称和要在其中创建文件夹。

    2. 要使用的模板版本。

    3. 在“ 应用程序ID”字段中， 输入您在“ 创建应用程序”页面上指定的应用程序标识符。

    4. 在SDK目录字段中， 指定安装PHP的Google App Engine SDK的文件夹的路径。

    5. Python可执行文件的位置。在“ Python可执行文件”字段中手动键入路径，或单击“ 浏览” 然后在打开的对话框中选择Python可执行文件。

    准备好后，单击完成 。

IntelliJ IDEA使用main.php文件和app.yaml配置文件创建一个项目。的app.yaml文件包含用于将所生成的设置运行时 （应该是php55），则API_VERSION ，所述的applicationID ，所述应用程序的版本 （默认为1），则线程与元件true值，以发送多个并行请求，以及脚本处理程序的 url和脚本 （值为main.php ）。

激活现有项目中的Google App Engine支持

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，单击“ 语言和框架”下的“ PHP ”，然后单击“ PHP的Google App Engine” 。

  2. 在打开的Google App Engine for PHP页面上，选中启用PHP支持Google App Engine复选框并指定以下内容：

    1. 用于PHP SDK的Google App Engine存储的文件夹。

    2. Python 2.7可执行文件的路径。

    3. 在“ App Engine帐户设置”区域中， 选择对开发服务器进行身份验证的方式，可用选项包括：

      • 通过OAuth2使用无密码登录：选择此选项以使用OAuth 2.0协议 。要保存通过Google Developers Console获得的令牌 ，请清除不保存令牌复选框。

      • 使用电子邮件和密码登录：选择此选项以使用您的Gmail地址和密码。
