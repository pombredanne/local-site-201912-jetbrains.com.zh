Filtering Out and Removing Unshelved Changes

Most shelved changes are applied only once. By default, upon unshelving a change is no longer shown on the list of available shelved changes. This helps you keep your list of available shelved changes more easier to use. However you can return an applied shelf to the list it at any time, restore and unshelve it again as many times as necessary.

Shelves that are not likely to be applied in the future anymore, can be permanently removed from the list.

On this page:

  • Removing a single change
  • Returning unshelved changes to the list
  • Hiding unshelved changes
  • Removing unshelved changes permanently

Removing a single change

You can permanently delete any change no matter whether it has been already unshelved or not.

  • To remove a shelved change permanently, select it in the list and choose Delete on the context menu of the selection.

Returning unshelved changes to the list

  • In the Shelf tab of the Version Control tool window, choose Show Already Unshelved on the context menu.

Hiding unshelved changes

  • In the Shelf tab of the Version Control tool window, choose Hide Already Unshelved on the context menu.

Removing unshelved changes permanently

  1. In the Shelf tab of the Version Control tool window, choose Show Already Unshelved on the context menu.
  2. Select the desired file or changelist.
  3. On the context menu of the selection, choose Delete All Already Unshelved.
