Configuring Compiler Settings

This section describes how to access the compiler settings, modify the list of recognized resources, exclude certain paths from compilation, select the desired compiler, and configure annotation processing.

For instructions on how to configure compiler output on the project and module levels, refer to Configuring Project Compiler Output and Configuring Module Compiler Output.

To configure compiler settings

  1. Open the Settings dialog box.
  2. On the Compiler page, under Build, Execution, Deployment, configure common settings, for example:
    • In the Resource patterns field, update the regular expression that describes the extensions of the files to be recognized as resources. Separate individual patterns with semicolons (;). Refer to the list of wildcard characters, and examples.
    • If necessary, choose to perform compilation as an external process.
  3. On the Excludes page, specify the files and folders that must be excluded from compilation. Click , and choose the desired directory from the Select Path dialog box.

    Note the following:

    • If an excluded path is a dependency of the source code being compiled, this path will be included in compilation and processed by the compiler as required.
    • If a file contains errors and fails to compile, but it is not important for the current project state, or if you want to skip some files and not include them in the output directory, you can exclude such files from compilation.
  4. Select a specific compiler. For example, on the Java Compiler page, select the desired compiler from the Use compiler drop-down list and specfify the compiler-specific settings.
  5. In the Annotation Processors page, configure the annotation processing parameters.
  6. Apply the changes and close the dialog.
  7. Define the paths, providing the proper order of module roots and libraries, as described in the Configuring Module Dependencies section.

Note that the compiler in IntelliJ IDEA works sequentially, that is, in accordance with the order of module dependencies, see Order of dependencies.
