运行/调试配置：Ruby远程调试

以下仅在安装并启用Ruby插件时有效。

使用此对话框可以定义用于Ruby脚本的远程调试的配置。

在这个部分：

  • 先决条件
  • 信息和设置
  • 共同
  • 工具列
  • 发射前

先决条件

在开始使用Ruby之前，请确保已安装并启用 Ruby插件。

另外，请确保满足以下先决条件：

  • Ruby SDK已下载并安装在您的计算机上。

  • 所需的框架SDK已下载并安装在您的计算机上。

有关详细信息，请参阅其各自的下载和安装页面：

  • 红宝石
  • Ruby on Rails

信息和设置

项目

描述

名称

在此字段中，指定当前运行/调试配置的名称。

服务器命令

在服务器端使用以下命令来启用远程调试：

rdebug-ide --host 0.0.0.0 --port --dispatcher-port -- $COMMAND$

此只读字段显示应在服务器端输入以启动调试服务器的命令。这里的端口号和调度程序端口号取自“ 端口”和“ 本地端口”字段。 $COMMAND$对应于要调试的脚本的名称。

远程主机

指定将进行远程调试的主机的URL。

远端埠

在服务器端指定端口号。请注意，在此字段中输入的数字会自动在rdebug命令。键入端口号时，建议的命令字符串将相应更改。

远程根文件夹

在服务器端指定要调试脚本所在的根目录。该字段定义到本地根文件夹的映射。

本地端口

指定本地端口号。请注意，在此字段中输入的数字会自动在rdebug命令。当您键入本地端口号时，建议的命令字符串也会相应更改。

本地根文件夹

指定包含相关脚本的本地根目录。输入路径，或单击并在“选择路径”对话框中找到所需的根。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

如果使用基于目录的项目格式 ，则运行/调试配置的设置存储在.idea \ runConfigurations文件夹（如果共享）中的单独的.xml文件中，或者存储在.idea \ workspace中。 xml文件，否则。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以进行共享配置，否则将存储在.iws文件中。

工具列

运行/调试配置的树形视图具有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

项目

捷径

描述

Alt+Insert

创建运行/调试配置。

Alt+Delete

删除所选的运行/调试配置。请注意，您无法删除默认配置。

Ctrl+D

创建所选运行/调试配置的副本。请注意，您将创建默认配置的副本。

仅当您选择临时配置时，才会显示该按钮。单击此按钮将临时配置保存为永久配置。

查看和编辑所选运行/调试配置的模板（即默认设置）。模板显示在“ 模板”节点下，并用于新创建的配置。

/

Alt+Up /Alt+Down

在列表中上下移动选定的运行/调试配置。

列表中的配置顺序定义了顺序，当您选择运行/调试配置时，配置以这些顺序出现。

运行/调试配置的默认模板始终按字母顺序排序。

移至新文件夹/创建新文件夹。您可以通过将运行/调试配置放入文件夹来对它们进行分组。

要创建文件夹，请在类别中选择配置，然后单击 ，然后指定文件夹名称。如果仅关注某个类别，则会创建一个空文件夹。

然后，要将配置移动到文件夹中，在文件夹之间或从文件夹中移出，请使用拖动或和纽扣。

要删除分组，请选择一个文件夹，然后单击 。

单击此按钮以按字母顺序对配置进行排序。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在IntelliJ IDEA中定义它，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 构建 ：选择以编译指定的模块。将执行“ 构建模块”命令 。

    如果在编译过程中发生错误，则IntelliJ IDEA不会尝试启动运行/调试配置。

  • 生成项目 ：选择以编译整个项目。将执行“ 生成项目”命令 。

    如果在编译过程中发生错误，则IntelliJ IDEA不会尝试启动运行/调试配置。

  • Build，无错误检查 ：与Build选项相同，但是IntelliJ IDEA不管编译结果如何，都将尝试启动运行/调试配置。

  • 构建工件 ：选择此选项，以建立一个神器或假象。在打开的对话框中，选择应构建的工件。

  • 启动Web浏览器 ：选择此选项以启动浏览器。在打开的对话框中，选择浏览器的类型并提供起始URL。另外，指定是否要使用JavaScript调试器启动浏览器。

  • 运行Ant目标 ：选择此选项可运行Ant目标 。在打开的对话框中，选择要运行的目标。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行Maven目标 ：选择此选项可运行Maven目标 。在打开的对话框中，选择要运行的目标。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 启动React Native Bundler ：选择此选项以在运行或调试会话中自动运行捆绑程序。默认情况下，这是通过react-native start 。如果您的应用程序使用Expo ，则需要通过以下方式运行开发服务器start npm任务。为此，请单击 ，然后在“ 配置React Native”对话框中，选择npm脚本 ， 然后从列表中选择start 。

  • 编译TypeScript ：选择运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都会反映在生成的JavaScript文件中。 在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防万一检测到任何错误：

    • 如果选中“检查错误”复选框，则编译器将显示所有错误，并且运行配置将不会启动。

    • 如果清除“ 检查错误”复选框，则编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 将文件上传到远程主机 ：选择此选项可以根据默认服务器访问配置将应用程序文件自动上传到服务器 。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

  • 运行Rake任务 ：添加要在运行或调试之前执行的Rake任务 。要选择耙任务，请单击浏览按钮 ，然后从可用任务列表中选择所需的任务。

    请注意，此处提供代码完成功能。

  • 运行JRuby编译器 ：选择此选项可使用指定的目标路径，编译器进程堆大小和命令行参数（如果有）执行JRuby编译器。

Alt+Delete

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
