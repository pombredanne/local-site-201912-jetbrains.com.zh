IntelliJ IDEA 2019.3帮助

为REST开发做准备

在IntelliJ IDEA级别，RESTful Web服务开发支持基于Java EE：RESTful Web服务（JAX-RS） 插件 。该插件与IntelliJ IDEA捆绑在一起，并且默认情况下启用。

在模块级别，可以在创建项目，将新模块添加到现有项目以及现有模块时启用服务开发支持。

  • 确保已启用RESTful Web服务插件
  • 创建项目时启用REST支持
  • 在将模块添加到项目时启用REST支持
  • 为现有模块启用REST支持

确保已启用RESTful Web服务插件

即使默认情况下启用了CDI支持插件，也总是值得在开始使用CDI之前确保仍启用该插件。

  1. 打开设置/首选项对话框Ctrl+Alt+S 。

  2. 在对话框的左侧，选择插件 。

  3. 在对话框的右侧，在“ 插件”页面上 ，键入restful在搜索框中。结果，只有名称和说明包含的插件restful显示在插件列表中。

  4. 如果未选中Java EE：RESTful Web服务（JAX-RS）右侧的复选框，请选择它。

  5. 在“ 设置”对话框中，单击“ 确定 ” 。

  6. 如果建议，请重新启动IntelliJ IDEA。

创建项目时启用REST支持

  1. 如果IntelliJ IDEA中当前没有打开任何项目，请在“ 欢迎”屏幕上单击“ 创建新项目 ”。否则，选择文件|新品|项目 。

    结果，“ 新建项目”向导打开。

  2. 在向导的第一页的左侧窗格中，选择Java Enterprise 。在页面的右侧，指定要使用的JDK并选择要支持的Java EE版本。

  3. 在“ 其他库和框架”下 ，选中“ Restful Web Service”复选框。

  4. 您将需要一个实现JAX-RS API的库。您可以选择使用现有的库，创建和使用新的库，下载库文件（如果计算机上尚不可用）或将库的设置推迟到以后。

    使用图书馆

    选择要使用的库（如果在IntelliJ IDEA中已经定义了相应的库）。

    创建

    如果您的计算机上已经有相应的jar文件，则可以将这些文件放在库中。单击创建，然后在打开的对话框中选择所需的文件。

    （可选）单击“ 配置”以编辑选定的库。

    下载

    选择此选项以下载必要的库文件。

    （可选）单击“ 配置”以编辑库设置和内容。

    稍后设置图书馆

    选择此选项可以将库的设置推迟到以后。

    单击下一步 。

  5. 指定项目和模块的名称和位置设置。

    点击完成 。

在将模块添加到项目时启用REST支持

  1. 打开要向其中添加模块的项目，然后选择“ 文件” |“添加”。新品|模块 。

    结果，“ 新建模块”向导将打开。

  2. 在向导的第一页的左侧窗格中，选择Java Enterprise 。在页面的右侧，选择要支持的Java EE版本。

  3. 在“ 其他库和框架”下 ，选中“ Restful Web Service”复选框。

为现有模块启用REST支持

  1. 打开项目工具窗口（例如， 视图|工具窗口|项目 ）。

  2. 右键单击感兴趣的模块，然后选择添加框架支持 。

  3. 在打开的“ 添加框架支持”对话框的左侧窗格中，选中“ RESTful Web服务”复选框。

  4. 您将需要一个实现JAX-RS API的库。您可以选择使用现有的库，创建和使用新的库，下载库文件（如果计算机上尚不可用）或将库的设置推迟到以后。
