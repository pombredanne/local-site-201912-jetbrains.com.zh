IntelliJ IDEA 2019.3帮助

角度的

Angular是用于构建跨平台应用程序的流行框架。IntelliJ IDEA提供了对Angular的支持，并在开发过程的每个步骤中为您提供帮助-从创建新的Angular应用程序，处理组件到调试和测试。

在你开始前

  1. 下载并安装Node.js。

  2. 确保在“ 插件”页面上启用了Angular和AngularJS捆绑插件，有关详细信息，请参阅管理插件 。

创建一个新的Angular应用程序

推荐开始构建新的Angular应用程序的方法是Angular CLI ，IntelliJ IDEA使用npx下载并运行IntelliJ IDEA。如此一来，您的应用程序就可以使用TypeScript和Webpack配置进行引导。

当然，您可以自己下载Angular CLI或创建一个空的IntelliJ IDEA项目并在其中安装Angular。

  1. 选择文件|新品|从主菜单中的项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 新建项目”对话框中，在左侧窗格中选择“ 静态Web ”。

  3. 在右侧窗格中，选择AngularCLI ，然后单击下一步 。

  4. 在向导的第二页上，指定项目名称和用于创建项目的文件夹。 在“ 节点解释器”字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择添加以配置新的解释器。

    从Angular CLI列表中，选择npx --package @ angular / cli ng 。

    另外，对于npm 5.1及更早版本，请安装@angular/cli通过运行打包自己npm install -g @angular/cli在航站楼 Alt+F12 。创建应用程序时，选择@angular/cli包已存储。

    （可选 ）在“ 其他参数”字段中，指定要传递给Angular CLI的其他 ng新选项 。此字段中提供代码完成：在您开始输入选项名称或按时Ctrl+Space ，IntelliJ IDEA向您显示可用的选项及其说明。

  5. 单击“完成”时，IntelliJ IDEA会使用所有必需的配置文件生成特定于Angular的项目，并下载所有新必需的依赖项。

创建一个空的IntelliJ IDEA项目

  1. 选择文件|新品|从主菜单中的项目 ，或单击欢迎屏幕上的新建项目按钮。

  2. 在“ 新建项目”对话框中，在左侧窗格中选择“ 静态Web ”。

  3. 在右侧窗格中，选择“ 静态Web” ，然后单击“ 下一步” 。

  4. 在向导的第二页上，指定项目名称和用于创建项目的文件夹。

  5. 当您单击完成时，IntelliJ IDEA将创建并打开一个空项目。

在空项目中安装Angular

  1. 打开将在其中使用Angular的空项目。

  2. 在嵌入式终端 （ Alt+F12 ），输入：

    npm install @angular/core

    这将安装核心Angular软件包以及框架的关键运行时部分。

    您可能还需要安装Angular的其他软件包，请参阅软件包列表 。

从现有的Angular应用程序开始

要继续开发现有的Angular应用程序，请在IntelliJ IDEA中将其打开并下载所需的依赖项。

打开计算机上已经存在的应用程序源

  • 单击“ 欢迎”屏幕上的“ 打开 ”，或选择“ 文件” |“共享”。从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中签出应用程序源

  1. 在“ 欢迎”屏幕上单击“从版本控制获取”，或选择“ VCS |从主菜单的版本控制中获取 。

  2. 在调用的对话框中，从列表中选择您的版本控制系统，然后指定存储库以签出应用程序源。

下载依赖项

  • 在弹出窗口中单击运行“ npm install” ：

推荐的启动新Angular应用程序的方法是Angular CLI 。仅在这种情况下，您的应用程序才可以使用TypeScript和Webpack配置进行引导。

运行和调试Angular应用程序

此外，对于使用Angular CLI创建的应用程序，IntelliJ IDEA会使用默认设置生成两个运行/调试配置：

  • 一个npm配置 ，默认名称为Angular CLI Server 。此配置运行ng serve启动开发服务器并以开发模式启动应用程序的命令。

  • 默认名称为Angular Application的 JavaScript Debug配置。此配置启动调试会话。

运行Angular应用程序

  1. 对于Angular CLI应用程序，从工具栏上的列表中选择Angular CLI Server运行配置，然后单击在列表旁边。

    或者，运行npm start在航站楼 Alt+F12或双击start npm工具窗口 （ 视图|工具窗口| npm ）中的任务。

  2. 等待应用程序被编译并且Webpack开发服务器准备就绪。在http：// localhost：4200 /上打开浏览器以查看该应用程序。

开发服务器运行时，更改任何源文件并保存更新后，应用程序将自动重新加载。

调试Angular应用程序

  1. 在代码中设置断点。

  2. 如上所述 ，以开发模式启动应用程序，然后等待应用程序被编译并且Webpack开发服务器准备就绪。

  3. 要查看该应用程序，请在“ 运行工具”窗口中显示的URL上打开浏览器，默认情况下为http：// localhost：4200 / 。

    如果您正在使用其他端口，或者您的应用不是使用Angular CLI创建的，请复制应用实际运行的URL地址，稍后将在调试配置中指定此URL。

  4. 如果您使用Angular CLI创建了应用程序，并且该应用程序在默认端口上运行4200 ，IntelliJ IDEA会生成默认的Angular Application配置。

    否则，创建一个新的JavaScript调试配置：去跑|编辑配置 ，单击 ，然后从列表中选择JavaScript Debug 。在“ 运行/调试配置：JavaScript调试”对话框中，将保存的URL粘贴到“ URL”字段中并保存配置。

  5. 从配置列表中，选择自动生成的Angular Application配置或新创建的自定义配置，然后通过单击启动调试会话。 在列表旁边。

  6. 遇到第一个断点时，切换到“ 调试工具”窗口，然后照常进行： 逐步执行程序 ， 停止并继续执行程序， 在挂起时对其进行检查 ，浏览调用堆栈和变量，设置监视，评估变量， 查看实际HTML DOM等等。

导航到Angular应用程序

在Angular项目上工作时，您必须在不同的组件文件（例如TypeScript，模板和样式文件）之间跳转。使用IntelliJ IDEA，可以使用导航栏 ， 项目视图或“ 转到文件”操作来执行此操作。 Ctrl+Shift+N 。

您也可以下载Angular CLI QuickSwitch插件并将其安装在计算机上 。

还有一种方法是“ 相关符号”弹出窗口。

  1. 在Angular组件文件中，按Ctrl+Alt+Home 。IntelliJ IDEA显示一个带有相关文件列表的“相关符号”弹出窗口。在TypeScript组件文件中，弹出窗口还列出了导入到该文件中的所有符号。

  2. 要打开文件，请选择它并按Enter 。或者，使用与每种文件类型关联的数字：

    1. 具有组件类的TypeScript文件

    2. 模板

    3. 测验

    4. 款式

使用ng add将新功能添加到Angular应用程序

在使用Angular CLI 6或更高版本的项目中，可以使用Angular Dependency操作来添加新的库。此操作运行ng add命令，用于安装依赖项并使用特殊的安装脚本更新应用程序。请注意，并非所有库都支持使用ng add 。

  1. 选择文件|从主菜单新建或按Alt+Insert在“ 项目”工具窗口中，然后选择Angular Dependency 。

  2. 从列表中，选择要添加的库。该列表显示了可以肯定安装的库ng add 。要安装不在列表中的软件包，请滚动至其末尾并双击“ 安装以上未列出的软件包”链接，然后在打开的对话框中指定软件包名称。

    可以安装的软件包列表ng add在GitHub上可用 。

如果您通过package.json手动管理依赖项，则IntelliJ IDEA仍会识别支持ng add 。将此类软件包添加到package.json时 ，IntelliJ IDEA建议使用ng add 。

如果已经安装了这种依赖关系，您可能仍想使用ng add 。在package.json文件中，选择包的名称，然后按Alt+Enter ，然后单击ng add重新安装 。

使用Angular Schematics生成代码

在Angular CLI 6项目中，IntelliJ IDEA可以使用库中定义的两个原理图生成代码，例如@angular/material以及在Angular CLI本身中定义的内容。对于早期版本，IntelliJ IDEA仅生成Angular CLI中定义的组件，服务和其他蓝图。

  1. 选择文件|从主菜单新建或按Alt+Insert在“ 项目”工具窗口中，然后选择“ 角度示意图” 。

  2. 从列表中选择相关的原理图。

  3. 在打开的对话框中，指定要生成的原理图的名称以及其他选项（如果需要）。IntelliJ IDEA显示了原理图的描述，并提供了代码完成和可用选项的描述。
