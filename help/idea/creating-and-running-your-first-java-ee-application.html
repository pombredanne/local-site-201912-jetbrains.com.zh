开发Java EE应用程序

仅Ultimate Edition支持此功能。

本教程说明了Java EE应用程序开发工作流程。

我们将要开发的应用程序将是最小的。这将是一个JSP页面的Java Web应用程序。但是，此处显示的IntelliJ IDEA功能适用于任何复杂程度的Java EE应用程序。

在你开始前

确保您的计算机上安装了以下软件：

  • IntelliJ IDEA ULTIMATE版。

  • GlassFish Server版本3.0.1或更高版本。下载GlassFish 。（您可以使用任何其他支持Java EE的应用程序服务器。这里仅以GlassFish为例。）

  • 网络浏览器。

建立专案

  1. 单击“欢迎”屏幕上的“ 创建新项目 ”，或选择“ 文件” |“ 新建” 。 新品|项目 。将打开“ 新建项目”向导。

  2. 在左侧窗格中，选择Java Enterprise 。

  3. 指定要使用的JDK （“ Project SDK”字段）：从列表中选择一个，单击“ 新建” ，然后选择JDK安装文件夹，或单击“ 下载JDK” 。

  4. 指定您的应用程序服务器。（我们将使用GlassFish Server。）

    如果IntelliJ IDEA中尚未定义GlassFish，请单击“ 应用程序服务器”字段右侧的“ 新建” ，然后选择“ Glassfish服务器” 。

    在“ Glassfish服务器”对话框中，指定“ GlassFish Server”安装目录。

  5. 在“ 其他库和框架”下 ，选中“ Web应用程序”复选框。

    单击下一步 。

  6. 指定新项目的名称（例如JavaEEHelloWorld ）。

    单击“ 完成”，然后在IntelliJ IDEA创建项目时等待。

探索项目结构

创建项目后，您将在“ 项目”工具窗口中看到类似的内容。

  • JavaEEHelloWorld是一个模块文件夹（在这种情况下与项目文件夹一致）。.idea文件夹和文件JavaEEHelloWorld.iml分别包含项目和模块的配置数据。文件夹src用于您的Java源代码。Web文件夹用于您的应用程序的Web部分。目前，此文件夹包含部署描述符WEB-INF / web.xml和文件index.jsp ，该文件打算用作应用程序的起始页。

  • 外部库包括用于处理GlassFish的JDK和JAR文件。

开发源代码

我们的应用程序将是单个JSP页面应用程序。它的唯一功能是输出文本Hello，World!。

  1. 打开index.jsp进行编辑：在“ 项目”工具窗口中选择文件，然后按F4 。

  2. 之间和类型Hello, World!

    此步骤的代码已准备就绪。

运行应用程序

在工作区的右上方，单击 。

IntelliJ IDEA编译您的源代码并构建应用程序工件 。

之后，将打开“ 运行工具”窗口。IntelliJ IDEA启动服务器并将工件部署到该服务器上。

最后，您的默认Web浏览器启动，您会看到应用程序输出Hello, World!那里。

修改代码并观察更改

  1. 在index.jsp中 ，更改Hello, World!至Hello! 。

  2. 在“ 运行工具”窗口中，单击“更新”。 。

  3. 在“ 更新”对话框中，选择“ 更新资源” ，然后单击“ 确定” 。（有关更多信息，请参阅应用程序更新选项 。）

  4. 切换到Web浏览器并重新加载页面以查看更改。

另请参阅更新应用程序服务器上的应用程序 。

探索运行配置

创建项目时，我们将GlassFish指定为应用程序服务器。结果，IntelliJ IDEA为GlassFish创建了运行配置。

当我们执行运行命令时 ，我们开始了运行配置。现在让我们看一下运行配置，看看它的设置如何映射到我们刚刚观察到的事件上。

  1. 单击运行配置选择器，然后选择“ 编辑配置” 。

    将打开“ 运行/调试配置”对话框，并显示GlassFish运行配置的设置。

    启动前任务列表（在对话框的下部）指定在执行运行配置之前应编译应用程序代码并构建相应的工件。

  2. 选择“ 启动/连接”选项卡，以查看如何在运行，调试和代码覆盖模式下启动服务器。

  3. 选择“ 部署”选项卡以查看在服务器启动后部署了哪些工件。

  4. 返回到服务器选项卡。

    Open浏览器下的设置指定启动后（即，在启动服务器并将工件部署到服务器之后），默认的Web浏览器应启动并转到指定的URL http：// localhost：8080 / JavaEEHelloWorld_war_exploded 。

    “更新”操作右侧的设置指定点击在“ 运行工具”窗口中，应显示“ 更新”对话框，并且默认情况下应使用“ 更新资源”选项。（ 最后使用的更新选项成为默认选项 ）。

  5. 单击确定 。

探索工件配置

在创建项目时，我们表示将要开发一个Web应用程序。结果，除其他外，IntelliJ IDEA创建了用于构建Web应用程序工件的配置。让我们看一下这个配置。

  1. 打开项目结构对话框： 文件|项目结构或Ctrl+Shift+Alt+S 。

  2. 在“ 项目设置”下 ，选择“ 工件” 。

    可用的工件配置显示在右窗格中的下方和 。（目前只有一种配置。）

    工件设置显示在对话框的右侧。

    类型。 工件类型为“ Web应用程序：爆炸”。这是一个解压缩的Web应用程序存档（WAR），它是一种目录结构，可以部署到Web服务器上。

    输出目录。 构建后的工件将放置在/ out / artifacts / JavaEEHelloWorld_war_exploded中 。

    输出布局。 工件结构显示在“ 输出布局”选项卡的左侧窗格中。

    的对应于输出目录。其他元素具有以下含义：

    • “ JavaEEHelloWorld”的编译输出表示其源位于src目录中的已编译Java类。它们被放置在输出目录中的WEB-INF / classs中。

    • “ Web”构面资源表示Web目录的内容。

将应用程序打包到WAR文件中

当您对应用程序感到满意时，您可能需要将其放置在WAR （Web应用程序存档）中。为此，您应该创建适当的工件配置，然后构建工件：

  1. 请点击 ，指向“ Web应用程序：存档”，然后选择“ JavaEEHelloWorld：战争爆发” 。

    创建一个新的工件配置，其设置显示在对话框的右侧。

  2. 为您的存档创建清单文件：单击创建清单并同意IntelliJ IDEA web / META-INF / MANIFEST.MF所建议的位置。

  3. 在“ 项目结构”对话框中单击“ 确定 ”。

  4. 选择构建|建立工件 。

  5. 在Build Artifact弹出窗口中，指向JavaEEHelloWorld：war并选择Build 。

    现在，如果您查看out / artifacts / JavaEEHelloWorld_war文件夹，您将在此处看到档案。

要将Java EE应用程序打包到EAR中 ，您应该：

  2. 配置EAR 构件 。

  3. 构建该工件。

正如我们将要看到的，作为其Java EE应用程序支持的一部分，IntelliJ IDEA可以为您执行大多数这些任务：

  1. 在“ 项目”工具窗口中，右键单击您的模块文件夹，然后选择添加框架支持 。

  2. 在打开的对话框中，选择JavaEE Application复选框，然后单击OK 。

    请注意在模块文件夹中创建的描述符文件META-INF / application.xml 。

  3. 在编辑器中打开文件F4 。

    目前，该文件几乎为空。

  4. 现在让我们看一下工件配置。

    注意，出现了一种新配置，一个用于分解的EAR工件的配置。

    当前，工件中仅包含JavaEE Application方面资源META-INF / application.xml 。
