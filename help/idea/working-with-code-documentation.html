添加Javadoc注释

对于文档注释，IntelliJ IDEA提供了默认情况下启用的完成功能。输入后/**在声明之前按Enter ，IDE会自动为您完成文档注释。对于方法注释，新的注释存根包含必需的标签（ @param每个方法参数的标签@return ， 要么@throws ）。

文档注释也可以在JavaScript，Python，Ruby和PHP中获得 。

如果要禁用完成功能，请在“ 设置/首选项”对话框中Ctrl+Alt+S ，转到编辑器|一般| Smart Keys ，然后清除“ 插入文档注释存根”复选框。

使用“修复文档”注释操作添加注释

  1. 将插入符号放在类，方法，函数或字段中，然后按Ctrl+Shift+A 。

  2. 类型fix doc comment然后按Enter 。

IntelliJ IDEA使用相应的标签添加缺少的文档存根。

如果方法签名已更改，则IntelliJ IDEA会突出显示与该方法签名不匹配的标签，并建议快速修复。

对于文档注释中的标签，IntelliJ IDEA还提供了代码完成功能。按Ctrl+Space之后@标签，然后从建议列表中选择所需的标签。如果某个标签具有多个值，则可以从建议列表中选择必要的值。

生成Javadoc参考

IntelliJ IDEA提供了一个实用程序，使您可以为项目生成Javadoc参考。

  1. 在主菜单中，选择工具|生成JavaDoc 。

  2. 在打开的对话框中，选择一个范围-您要为其生成引用的一组文件或目录，并设置要放置生成的文档的输出目录。

    输出目录是必填字段：只要它为空，就不能生成JavaDoc文件 。

  3. 使用滑块定义将包含在生成的文档中的成员的可见性级别。选择以下选项之一：

    • 私有的 ：包括参考的所有类和成员。

    • 包 ：包括除私人类之外的所有类和成员。

    • 受保护的 ：包括公共和受保护的班级和成员。

    • 公开 ：仅包括公开课程和成员。

  4. 您可以指定语言环境（例如en_US.UTF-8 ），命令行参数和最大堆大小。

  5. 单击确定以生成参考。

“指定生成JavaDoc范围”对话框

工具|生成JavaDoc对话框调用JavaDoc实用程序。对话框的控件对应于此实用程序的选项和标签。

项目

描述

生成JavaDoc范围

使用此区域可以指定应为其生成JavaDoc的文件，文件夹和程序包的子集。

该范围可以是整个项目，未提交的文件（启用VCS时），当前文件，自定义范围等。

包括测试源

包括文档注释以测试生成的JavaDoc。

将JDK和库源包含在-sourcepath

如果选中此复选框，则JDK和库源的路径将传递到JavaDoc实用程序。有关详细信息，请参阅文档 。

包括指向JDK文档的链接

如果选中此复选框，则JDK中对类和包的引用将变成链接，这与使用JavaDoc utiity的-link选项相对应。

仅当在SDK设置的“ 文档路径”选项卡中指定了指向联机文档的链接时，才会启用此复选框。

有关详细信息，请参考JavaDoc文档。

输出目录

指定将存储生成的文档的目录的标准路径。手动输入路径或单击并在对话框中选择位置。指定的值传递给-d JavaDoc实用程序的参数。如果系统中不存在指定的目录，则将提示您创建它。

请注意，除非指定了输出目录，否则将禁用“ 确定”按钮。

滑杆

指定要包含在生成的文档中的成员的可见性级别：

  • 私人 ：选择此级别可以包含所有班级和成员。级别对应于-private JavaDoc参数。

  • 包 ：选择此级别以包含除私人类之外的所有类和成员。级别对应于-package JavaDoc参数。

  • 受保护的 ：选择此级别仅包含公共和受保护的类和成员。级别对应于-protected JavaDoc参数。

  • 公开 ：选择此级别仅包含公开课程和成员。级别对应于-public JavaDoc参数。

生成层次树

生成类层次结构。如果清除此复选框，则-notree参数传递给JavaDoc。

生成导航栏

生成导航栏。如果清除此复选框，则-nonavbar参数传递给JavaDoc。

产生索引

生成文档索引。如果清除此复选框，则-noindex参数传递给JavaDoc。

每个字母分别索引

为每个字母生成一个单独的索引文件。如果清除此复选框，则-splitindex参数传递给JavaDoc。

仅当选中“ 生成索引”复选框时，此复选框才可用。

@使用

记录类和包的使用。选中后，该复选框对应于-use JavaDoc参数。

@作者

包括@author段落。选中后，该复选框对应于-author JavaDoc参数。

@版

包括@version段落。选中后，该复选框对应于-version JavaDoc参数。

@已弃用

包括@deprecated信息。清除复选框后， -nodeprecated参数传递给JavaDoc。

不推荐使用的清单

生成不推荐使用的列表。清除复选框后， -nodeprecatedlist参数传递给JavaDoc。

仅当选中@deprecated复选框时，此复选框才可用。

语言环境

键入所需的语言环境。

其他命令行参数

输入要传递给JavaDoc的其他参数。使用命令行语法。

最大堆大小（Mb）

键入Java VM用于运行JavaDoc的最大堆大小（以Mb为单位）。

在浏览器中打开生成的文档

在浏览器中自动打开生成的JavaDoc。
