在调试模式下生成APK

除了生成要在物理设备上部署的签名软件包或要在仿真器上运行的未签名软件包之外，还可以在调试模式下提取并签名应用程序软件包。此签名足以测试和调试应用程序，但不允许发布它们。只有通过配置构件才能在调试模式下对应用程序包进行签名。

在调试模式下 ，您可以使用默认证书或自定义证书对APK进行签名。

如果您决定使用默认证书 ， IntelliJ IDEA使用调试密钥库或由Android SDK工具生成的具有预定义名称和密码的密钥在调试模式下对提取的程序包进行签名：

  • 密钥库名称： debug.keystore
  • 密钥库密码： android
  • 密钥别名： androiddebugkey
  • 密钥密码： android

  • CN （ 通用名称 ）： CN=Android Debug,O=Android,C=US

这些是IntelliJ IDEA中的默认设置。这意味着，如果您没有手动配置工件，而是在“ 运行/调试配置：Android应用程序”对话框中选择“ 部署默认APK”选项，则IntelliJ IDEA将使用证书中的预定义值生成.apk 。

如果您使用自定义证书 ， IntelliJ IDEA使用调试密钥库或您指定的密钥在调试模式下对提取的软件包进行签名。您可以生成新证书，也可以重复使用现有证书。后一种方法很有用，例如，如果您有多个应用程序，并且希望它们都使用相同的证书签名，那么它们可以存储在设备的同一文件夹中。

在调试模式下对软件包进行签名

  1. 选择文件|从主菜单中选择项目结构 ，然后单击左窗格中的Artifacts 。

  2. 点击新建按钮然后从上下文菜单中选择“ Android应用程序 ”。

  3. 在弹出菜单中，通过选择以下选项之一来定义工件内容：

    • 要创建一个空的布局定义，请选择Empty 。

    • 要将模块数据包括在工件中，请选择来自模块

    新工件的常规设置将显示在右侧的“ 工件布局”窗格中。

  4. 指定工件的常规设置 。在输出目录字段中，指定目标包.apk文件的位置。

  5. 通过以下步骤完成工件定义：

    • 配置构件结构 。

    • 将资源添加到工件。

    • 安排工件中包含的元素 。

    • 如有必要，在“ 预处理”和“ 后处理”选项卡中指定在构建工件之前和之后要执行的其他活动。

  6. 切换到“ Android”选项卡，然后从“ 类型”列表中指定要使用的证书：

    • 选择使用默认证书签名的调试，以使IntelliJ IDEA使用调试密钥库或由Android SDK工具生成并具有预定义名称和密码的密钥。

    • 选择“ 使用自定义证书签名的调试”，以使用您选择的证书对软件包进行签名。通过执行以下操作之一来指定要使用的密钥和包含它的密钥库文件：

      • 要使用现有密钥库文件中的密钥对软件包进行签名：

        1. 在“ 密钥存储路径”文本框中指定文件位置。手动输入路径或单击“ 选择现有”按钮，然后在打开的对话框中选择文件。在“ 密钥库密码”文本框中，键入所选密钥库的密码。

        2. 指定密钥别名，然后输入密码以访问密钥。

      • 要在现有密钥库中生成新密钥：

        1. 在“ 密钥存储路径”文本框中指定文件位置。手动输入路径或单击“ 选择现有”按钮，然后在打开的对话框中选择文件。在“ 密钥库密码”文本框中，键入所选密钥库的密码。

        2. 单击“ 创建新按钮”，并通过在打开的“ 新密钥存储”对话框中填写数据来配置要生成的发行密钥 。

      • 要使用新密钥生成新的密钥库文件：

        1. 单击创建新按钮。在打开的“ 新密钥库”对话框中，在“ 密钥库路径”文本框中指定要生成的文件的位置。手动输入路径或单击浏览 ，然后选择父文件夹并指定文件名。

        2. 指定并确认密码以访问密钥库。

        3. 通过在“ 密钥”区域中填写数据来配置新的发行密钥 。

    单击下一步 。

  7. 要使IntelliJ IDEA在打包过程中混淆应用程序 ，请选中“ 运行ProGuard”复选框，然后指定proguard.txt配置文件的位置。 该文件是在项目创建时生成的，并存储在项目根目录中。IntelliJ IDEA在“ 配置文件路径”文本框中建议此默认位置。接受建议或通过单击浏览指定自定义配置文件然后在打开的对话框中选择所需的文件。
