启用Web应用程序支持

本主题讨论打开Web应用程序选项时可用的功能。

  • 先决条件
  • 功能概述
  • 打开Web应用程序选项
  • 管理部署描述符，Web资源目录和Java Web源根
  • 管理应用程序工件

先决条件

对于“ Web应用程序”选项和相关功能可用：

  • 您应该使用IntelliJ IDEA的ULTIMATE版本。（相应的功能在Community Edition中不可用。）

  • 必须启用Java EE：EJB，JPA，Servlets 插件 。（此插件与IDE捆绑在一起，并默认启用。）

功能概述

当您打开“ Web应用程序”选项时，IntelliJ IDEA：

  • 使用index.jsp创建一个Web资源目录网站 ，该网站旨在用作您的应用程序的起始页，还可以创建一个Web应用程序部署描述符 WEB-INF / web.xml 。

  • 创建一个Web构面，使您可以管理部署描述符，Web资源目录和Java Web 源根目录。

  • 创建爆炸的WAR 工件配置。

  • 使Web工具窗口可用。

如果在创建项目或模块时打开“ Web应用程序”选项并指定应用程序服务器，则IntelliJ IDEA还将为该服务器创建运行/调试配置 。

打开Web应用程序选项

您可以打开“ Web应用程序”选项：

  • 创建项目或模块时（ 文件|新建|项目或文件|新建|模块 ）。在“ 新建项目”或“ 新建模块”向导的第一页上，选择“ Java Enterprise” ，然后选中“ 其他库和框架”下的“ Web应用程序”复选框。

  • 对于现有模块。在项目工具窗口（ 视图|工具窗口|项目 ）中，右键单击模块文件夹，然后选择添加框架支持 。然后在打开的对话框中选择“ Web应用程序”复选框。

管理部署描述符，Web资源目录和Java Web源根

您可以在“ 项目结构”对话框中管理Web应用程序部署描述符 ，Web资源目录和Java Web 源根目录：

  1. 打开项目结构对话框（例如Ctrl+Shift+Alt+S ）。

  2. 在最左侧的窗格中，选择Modules或Facets 。

  3. 在右侧窗格中，选择Web或Web（ ） 。

  4. 在对话框右侧打开的页面上：

    部署描述符。 形成Web应用程序的部署描述符列表。

    Web资源目录。 指定包含Web应用程序资源（例如网页，图像等）的目录。

    源根。 选择包含您的Web应用程序Java类（Servlet，过滤器，托管Bean等）的源根。

    有关更多信息，请参见Web facet页面 。

管理应用程序工件

要将应用程序部署到服务器，您需要一个应用程序工件 。对于Java Web应用程序，IntelliJ IDEA提供以下构件格式：

  • Web应用程序：爆炸。 这是一个解压缩的Web应用程序归档文件（ WAR ），它是一种目录结构，可以部署到Web服务器上。

  • Web应用程序：存档。 显然，这是一个WAR文件。

要管理工件配置，请使用“ 项目结构”对话框（“ 文件” |“项目结构” |“工件” ）。

另请参阅使用工件 。
