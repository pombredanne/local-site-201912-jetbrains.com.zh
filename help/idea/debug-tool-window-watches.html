调试工具窗口。手表

在“ 监视”窗格中，您可以在当前堆栈框架的上下文中评估任意数量的变量或表达式。值随着应用程序中的每个步骤更新，并且在每次挂起应用程序时都可见。

通过“ 变量”窗格的上下文菜单中的“ 评估表达式”命令可以一次查看一个表达式，而“ 监视”窗格则显示从一个调试会话到另一个调试会话持续存在的多个表达式，直到您将其删除为止。

您可以在此窗格，“ 变量”窗格以及编辑器中创建监视。

监视表达式始终在“ 框架”窗格中当前检查的堆栈框架的上下文中评估。如果无法评估表达式，则显示带有问号的表达式。

打开“监视”窗格

默认情况下，“ 监视”窗格是隐藏的，并且监视在“ 变量”窗格中显示 。

  • 要单独显示“ 手表”窗格并在其中查看配置的手表，请释放 变量窗格工具栏上的切换按钮。默认情况下，按钮被按下。

  • 要隐藏“ 监视”窗格并在“ 变量”窗格中查看监视，请按 监视窗格的工具栏上的切换按钮。

查看为

使用此命令选择值的布局。对于数字值，可以在十进制和十六进制格式之间切换。这些对象通常由它们的类名和实例标识符表示，但是您可以选择以字符串格式显示它们。

  • 自动 ：适用于可能具有不同布局的项目。通过这种布局，IntelliJ IDEA会在所有可用的渲染器中进行搜索，以搜索当前项目的第一个合适的布局。如果未找到，则应用默认布局。

  • 十六进制 ：此布局可用于数字变量。如果选中，则该变量以十六进制格式显示。

  • 原始 ：此布局可用于原始类型变量。它显示适合原始类型的值。

  • toString（） ：此布局可用于所有引用类型，其中toString()除数组外被覆盖，并显示节点的toString()树中的值。

  • 数组 ：可用于数组。

  • 对象 ：默认布局。适用于所有非基本类型节点。

  • 地图 ：显示为地图。

  • 收藏 ：显示为收藏。

  • 用户定义的渲染器 ：可用于用户在“ 调试器” | “ 调试器”中创建的渲染器。在“自定义数据视图”对话框中键入“渲染器”或。菜单中显示了相应的渲染器名称。
