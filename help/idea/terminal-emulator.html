终端仿真器

IntelliJ IDEA包括一个嵌入式终端仿真器，用于从IDE内部使用命令行外壳。在IDE内部拥有命令行可以让您执行诸如运行Git命令之类的操作，而无需在需要执行快速命令行任务时切换窗口和集中精力。

最初，终端仿真器使用默认的系统外壳程序运行，但是它支持许多其他外壳程序，例如Windows PowerShell，命令提示符cmd.exe ， sh ， bash ， zsh ， csh ， 等等。

如果您从命令中获得大量输出并想要查找某个字符串，请按Ctrl+F搜索输出。

打开终端工具窗口

请执行以下任一操作：

  • 按Alt+F12 。

  • 选择查看|工具窗口|从主菜单的终端 。

  • 单击终端工具窗口按钮 。

  • 将鼠标指针悬停在在IDE的左下角，然后从菜单中选择“ 终端 ”。

默认情况下，终端仿真器在当前目录设置为当前项目的根目录的情况下运行。您可以在以下位置更改默认的开始目录 工具| IntelliJ IDEA设置的终端页面Ctrl+Alt+S 。如果右键单击任何文件（例如，在“ 项目”工具窗口或任何打开的选项卡中），然后选择“ 在终端中打开” ，这将在该文件的目录中运行终端。

开始新的会话

  • 请点击在工具栏上。在单独的选项卡中打开一个新会话。

当您关闭项目或IntelliJ IDEA时，将保存会话，即，选项卡名称，当前工作目录，甚至将保留Shell历史记录。

要关闭活动会话，请单击在终端工具栏上，或右键单击当前会话选项卡，然后从上下文菜单中选择“ 关闭选项卡 ”。

在标签之间切换

  • 按Alt+Right和Alt+Left切换活动标签。

或者，您可以按Alt+Down查看所有终端标签的列表。

重命名标签

  • 右键单击选项卡，然后从上下文菜单中选择“ 重命名会话 ”。

浏览输入命令的历史记录

  • 使用Up和Down键。

配置终端仿真器

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到工具|码头 。

  2. 指定要与嵌入式终端仿真器一起使用的外壳程序，更改开始目录，并在其他设置中定义环境变量。

    如果您要使用备用Shell，可以配置以下示例路径：

    • PowerShell的：PowerShell的

    • Cmder： “ cmd” / k“”％CMDER_ROOT％\ vendor \ init.bat“” （注意CMDER_ROOT必须设置环境变量）

    • Cygwin： “ C：\ cygwin \ bin \ bash.exe” --login -i

    • Zsh： / bin / zsh

    • Bash： / bin / bash（或Windows的bash：bash.exe）

  3. （可选）配置嵌入式终端仿真器从IDE设置继承的设置。

    • 在“ 键盘映射”页面上，您可以配置副本Ctrl+C并粘贴Ctrl+V捷径。

    • 在编辑器上一般|外观页面，您可以配置插入符号闪烁。终端中不会继承“ 使用块插入符 ”设置：其插入符始终是阻塞的。

    • 您可以在终端上通过以下pgaes配置视觉演示：

      • 编辑器配色方案|控制台字体 ：行距和控制台字体。

      • 编辑器配色方案|控制台颜色 ：控制台颜色。

      • 编辑器配色方案|常规 ：选择前景色和背景色。
