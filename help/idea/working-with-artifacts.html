什么是人工制品？

工件是项目资产的组合，将它们组合在一起以测试，部署或分发软件解决方案或其一部分。示例包括编译后的Java类或打包在Java归档中的Java应用程序，作为目录结构的Web应用程序或Web应用程序归档等的集合。

工件可以是包含以下结构元素的归档文件或目录结构：

  • 一个或多个模块的编译输出

  • 模块依赖项中包含的库

  • 资源集合（网页，图像，描述符文件等）

  • 其他文物

  • 单个文件，目录和档案。

使用工件配置

伪影是根据伪影配置生成的。工件配置在“ 项目结构”对话框（ 文件|项目结构|工件 ）中进行管理。

配置工件的关键部分是在“ 输出布局”选项卡上指定工件的结构和内容。

建筑文物

你可以开始建立一个神器自己： 建设|建立工件 。

您还可以通过执行运行/调试配置来构建工件：

在相应的运行/调试配置中，将“ 构建工件”任务添加到“ 启动前任务”列表中。当您执行运行/调试配置（ Run | Run或Run | Debug ）时，将自动构建工件。

当运行/调试配置以某种方式使用相应的工件时（例如，启动打包在JAR工件中的应用程序或将WAR或EAR工件部署到应用程序服务器），将Build工件任务包含在运行/调试配置中就很有意义。

默认情况下，构建后的工件会放入out / artifacts / 夹。

构建选项（“构建”，“重建”等）

在构建工件（ Build | Build Artifacts ）时，您具有以下选项：

  • 建立。 首次使用时，将构建整个工件。下次每次使用此选项时，只会构建自上次构建以来所做更改所影响的工件的一部分，并将其添加到输出文件夹中。

  • 重建。 仿佛第一次构建整个工件。从技术上讲，这是Clean，然后是Build 。

  • 清洁。 删除工件输出目录的所有内容。

  • 编辑。 编辑工件配置 。

运行JAR工件

要运行打包在Java归档（JAR）中的Java应用程序，IntelliJ IDEA提供了JAR Application运行配置。要创建这样的运行配置：

  1. 打开“ 运行/调试配置”对话框（例如， “运行” |“编辑配置” ）。

  2. 请点击 ，然后选择JAR Application 。

将工件部署到应用程序服务器和云平台

许多工件格式（例如WAR，爆炸的WAR，EAR，爆炸的EAR）都适合于部署到应用程序服务器和云平台。这是您部署此类工件的方式：

  1. 在服务器或云运行/调试配置中，指定要部署的工件。（使用“ 部署”选项卡或字段。）

  2. 执行运行/调试配置或使用Deploy命令（ ）在“ 应用服务器” ， “运行或调试”工具窗口中。
