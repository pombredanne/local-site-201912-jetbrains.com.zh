测试框架

档案|设置|语言和框架| PHP | Windows和Linux的测试框架
IntelliJ IDEA |首选项|语言和框架| PHP | macOS的测试框架

该页面仅在启用PHP插件时可用。 的该插件未与IntelliJ IDEA捆绑在一起，但可以如管理插件中所述从JetBrains插件存储库安装。

使用此页面可将特定于PHP的测试框架与IntelliJ IDEA集成到当前项目中。随着IntelliJ IDEA的，你可以运行和调试PHPUnit的 ， 贝哈特 ， Codeception和PHPSpec测试。

该页面包含两个窗格：

  • 中央窗格显示了针对不同解释器的测试框架的现有配置。

  • 右侧窗格的内容取决于测试框架和所选解释器的类型。

在项目中配置测试框架

  1. 选择如何使用框架。请点击然后从列表中选择相关的配置类型：

    您可以配置任何测试框架以与本地或远程PHP解释器一起使用。

  2. 对于远程配置，请选择已配置的PHP解释器之一：

  3. 在右侧窗格中，选择从何处获取测试框架。

    • 对于Behat，PHPSpec和Codeception，输入框架可执行文件的路径。

    • 对于PHPUnit，指定您使用的框架安装类型，可用的选项是composer autoloader autoload.php或phpunit.phar归档文件或PEAR 。

  4. 检查所选的远程PHP解释器和路径映射。

  5. （可选）指定配置文件。对于PHPUnit，您还可以指定要使用的引导文件。

PHPUnit

在此窗格中，配置要与PHP解释器一起使用的PHPUnit安装。

PHPUnit库

在此区域中，指定PHPUnit安装的类型。可用选项如下：

使用Composer自动加载器

选择此选项以运行由Composer Dependency Manager安装的PHPUnit。从供应商文件夹中的autoload.php文件检索并加载该软件包。在“ 脚本路径”字段中指定autoload.php的位置。

phpunit.phar的路径

选择此选项可从phar存档中运行PHPUnit。

  • 如果您的项目中已经有一个phpunit.phar存档，请在phpunit.phar的路径字段中指定其位置。手动输入路径或单击然后在打开的对话框中选择文件。

  • 如果您的计算机上还没有phar存档，请单击Download phpunit.phar ...链接以让IntelliJ IDEA自动下载它。

    无论哪种情况，IntelliJ IDEA都会在测试执行之前加载存档。

CLI解释器

仅针对PHPUnit by Remote Interpreter配置显示该区域，并显示以下内容：

  • 与PHPUnit配合使用的远程PHP CLI解释器 。 “ CLI解释器”字段为只读。请点击在“ 解释器”对话框中更新所选的解释器 。

  • 的本地源与Vagrant实例，Docker容器或远程主机中的源之间的路径映射 。

  • 的用于从映像启动容器的Docker容器设置。这些设置可能包括卷配置，暴露的端口，网络等。

在大多数情况下，IntelliJ IDEA会检测路径映射和容器设置，并自动填写所有字段。或者，单击字段旁边，然后手动指定设置。有关详细信息，请参见配置远程PHP解释器 。

测试亚军

在这方面指定用于启动和执行方案的配置XML文件。
默认情况下，PHPUnit在项目根文件夹或config文件夹中查找phpunit.xml配置文件。您可以指定一个自定义配置文件。

您也可以输入引导文件的路径在启动测试之前始终执行PHP脚本。在字段中，指定脚本的位置。手动输入路径或单击然后在打开的对话框中选择所需的文件夹。

默认配置文件

选中此复选框指定您自己的XML配置文件。此文件以后将在所有PHPUnit运行/调试配置中用作默认文件。 请注意，在编辑PHPUnit运行/调试配置时，您还可以提供备用配置文件。

在该字段中，指定要使用的配置文件的位置。手动输入路径或单击然后在打开的对话框中选择文件。

清除复选框使PHPUnit使用项目根文件夹或config文件夹中的phpunit.xml配置文件。如果找不到这样的文件，则测试执行失败，因此显式指定配置文件可能更可靠。

默认引导文件

选中此复选框在启动测试之前始终执行PHP脚本。在字段中，指定脚本的位置。手动输入路径或单击然后在打开的对话框中选择所需的文件夹。 请注意，在编辑PHPUnit运行/调试配置时，您还可以提供备用引导文件。

PHPSpec库

在这方面指定要使用的PHPSpec安装。

PHPSpec可执行文件的路径

在此字段中，指定phpspec的位置。PHPSpec不一定必须安装在当前项目的根目录下。

如果没有为本地解释器指定PHPSpec的路径，则IntelliJ IDEA不提供对PHPSpec的完全支持，例如，它不显示代码完成建议，也不解析引用。

前缀（'spec_prefix'）

此只读字段显示规范的名称空间前缀。IntelliJ IDEA检测spec_prefix从默认配置文件字段中指定的配置文件中。默认值为spec 。有关详细信息，请参见PHPSpec配置：PSR-4和PHPSpec配置：规范和源位置 。

该字段仅针对本地配置显示。

默认配置文件
