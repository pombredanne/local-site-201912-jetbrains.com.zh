将文件另存为模板对话框

IntelliJ IDEA可帮助您从现有文件创建文件模板 。

项目

描述

名称

在此处指定新模板将在“ 文件和代码模板”设置的“ 文件”选项卡中显示的名称。默认情况下，使用当前文件的名称。

延期

在此指定此新模板要创建的文件的扩展名。默认情况下，使用当前文件的扩展名。

范本文字

编辑模板内容。您可以使用：

  • 纯文本。

  • #parse用于模板的指令包括 。

  • 自定义变量。变量名可以通过以下方式直接在模板中定义： #set指令或在文件创建期间。

    请注意，IntelliJ IDEA不会提示输入使用以下命令定义的Velocity变量的值#set 。

  • 变量将被扩展为相应的值${}格式。

    IntelliJ IDEA为PHP包含模板提供了一组附加变量，这些模板用于定义可重用的代码段 ，例如文件头和PHPDoc注释 ）。

    以下其他预定义变量可在PHP包含模板中使用 ：

    变量

    描述

    ${CARET}

    完成变量编辑后的插入位置。仅在文件创建期间生成并插入PHPDoc注释时应用。通过代码创建PHPDoc注释时|产生| PHPDoc块 ，提供了多种功能或方法选择。在这种情况下，可以为多个类，函数，方法或字段创建文档注释，因此${CARET}变量将被忽略。

    ${CLASS_NAME}

    定义为其生成PHPDoc注释的字段或方法的类的名称

    ${NAME}

    将为其生成PHPDoc注释的类，字段或函数（方法）的名称

    ${NAMESPACE}

    类或字段名称空间的全限定名称（不带斜杠）

    ${PARAM_DOC}

    参数的文档注释。评估为一组@param type name线。如果为其生成注释的函数不包含任何参数，则求值为空字符串。

    ${STATIC}

    被替换为static如果生成PHPDoc注释的函数（方法）或字段是静态的，则为关键字。否则计算为空字符串。

    ${THROWS_DOC}

    文档注释中的例外。评估为一组@throws type线。如果为其生成注释的函数没有引发任何异常，则求值为空字符串。

    ${TYPE_HINT}

    要为其生成PHPDoc注释的函数（方法）的返回值。如果通过函数（方法）的静态分析无法检测到返回类型，则求值为void 。

    治疗美元符号

    • 您可以防止对待美元字符$在模板变量中作为前缀。如果您需要美元字符（ $按原样插入，请使用${DS}文件模板变量。应用模板后，此变量的值将为纯美元字符$ 。

      例子：

      • 使用一些版本控制关键字（例如$Revision$ ， $Date$ ，依此类推）在您的默认类模板中， ${DS}代替美元前缀$ 。

      • 模板代码${DS}this将呈现为$this 。

根据样式重新格式化

选中此复选框，以使IntelliJ IDEA根据“ 代码样式”页面上定义的样式重新生成存根文件。

此选项仅在“ 文件”选项卡中可用。

启用实时模板

选中此复选框可在文件模板中使用实时模板。为此，必须将实时模板片段放入Velocity转义语法中。

例如：

#[[ $MY_VARIABLE$ $END$ ]]#

因此，可以指定光标位置。注意，这里需要使用实时模板变量 ！

此只读字段提供有关模板，模板的预定义变量及其工作方式的信息。
