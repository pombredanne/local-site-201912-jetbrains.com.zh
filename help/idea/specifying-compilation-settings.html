编译输出位置

对于您的源代码，测试源代码以及每个模块，都有单独的编译输出文件夹。

默认情况下，编译结果输出到：

  • 来源： / out / production /
  • 测试： / out / test /

在项目级别，您可以更改输出路径的/ out部分。如果这样做（例如，指定一些而不是/ out ），但不要在模块级别重新定义路径，编译结果将转到/ production / 和/ test / 。

在模块级别，您可以为模块源和测试分别指定任何所需的编译输出位置。

指定编译输出文件夹

编译输出文件夹在“ 项目结构”对话框（“ 文件” |“项目结构” ）中指定。

项目默认文件夹。 选择项目 。在“ 项目编译器”输出字段中，指定相应的路径。

模块输出文件夹。 选择模块 ，选择所需的模块，然后选择路径 。所需的控件在上部的“ 编译器输出”下 。

配置编译器设置

您可以修改已识别资源的列表，从编译中排除某些路径，选择所需的编译器，配置注释处理等。

  1. 在“ 设置/首选项”对话框中Ctrl+Alt+S ，选择生成，执行，部署 。

  2. 例如，在“ 编译器”页面上 ，您可能想要修改正则表达式，该正则表达式描述了将被识别为资源的文件扩展名（“ 资源模式”字段）。使用分号;分离出各个模式。请参阅通配符和示例列表 。

  3. 在“ 排除”页面上 ，指定不应包含在编译中的文件和文件夹。使用将项目添加到列表中。

    注意：

    • 如果排除的路径是要编译的源代码的依存关系，则此路径将包含在编译中，并由编译器根据需要进行处理。

    • 如果文件包含错误且无法编译，但是对于当前项目状态而言并不重要，或者如果要跳过某些文件而不将其包括在输出目录中，则可以从编译中排除此类文件。

  4. 在“ Java编译器”页面上 ，检查所使用的编译器是否是所需的。如有必要，请选择其他编译器。

    如果您对与IntelliJ IDEA捆绑在一起的Eclipse编译器的版本不满意，则可以使用“ 设置” | “ 设置”中的“ 路径到ECJ批处理编译器工具”选项。构建，执行，部署|编译器Java编译器，并指定特定的Eclipse编译器版本。

  5. 在“ 注释处理器”页面上 ， 配置注释处理参数 。

  6. 应用更改并关闭对话框。
