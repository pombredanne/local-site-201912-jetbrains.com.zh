IntelliJ IDEA 2019.3帮助

WordPress特定的编码协助

IntelliJ IDEA为开发WordPress提供了完整的编码帮助，包括WordPress感知代码完成，搜索钩子注册功能和指定为钩子注册参数的功能，钩子注册与钩子调用之间的导航，回调以及根据以下方式配置编码样式的可能性WordPress代码样式，从IntelliJ IDEA编辑器中查看WordPress官方文档，等等。

将WordPress添加到项目

为了利用IntelliJ IDEA提供的编码帮助，您正在使用的WordPress安装应在项目中配置为外部库 。结果， WordPress核心文件将参与索引编制，这是解析参考并提供代码完成，导航，搜索等的基础。

如果wp-content文件夹在WordPress安装之外，则需要将其添加为包含路径或内容根目录 。

将WordPress安装配置为项目包含路径

要将WordPress作为外部库添加到项目中，您需要将WordPress添加到包含路径列表中。 在配置包含路径中了解更多信息。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP 。

  2. 在打开的PHP页面上，将WordPress安装文件夹的路径添加到“ 包含路径”列表中：单击按钮，然后在打开的对话框中指定安装文件夹的路径。

将wp-content文件夹添加到项目中

如果wp-content文件夹位于WordPress安装之外，则除了WordPress核心文件之外，您需要将其单独添加到项目中。这可以通过两种方式完成：

  • 要使wp-content文件夹参与索引编制而不将其置于项目版本控制之下，请将其添加到包含路径列表中：

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP 。

    2. 在打开的PHP页面上，将wp-content文件夹的路径添加到“ 包含路径”列表中：单击按钮，然后在打开的对话框中指定文件夹的路径。

  • 要使wp-content文件夹参与索引并将其置于项目版本控制下，请将其添加为内容根目录 ：

    1. 打开项目结构对话框（例如Ctrl+Shift+Alt+S ）。

    2. 在左侧窗格中，选择Modules 。

    3. 在右侧窗格中，选择所需的模块。

    4. 在对话框的右侧，选择“ 源”选项卡。

    5. 单击添加内容根 。

    6. 在打开的对话框中 ，找到wp-content目录，然后单击“ 确定” 。

    在“ 内容根目录”中了解有关配置内容根目录的更多信息。有关将文件和文件夹添加到版本控制的信息，请参阅“ 启用版本控制” 。

配置WordPress代码样式

在IntelliJ IDEA中，您可以根据WordPress编码标准使用WordPress代码样式 。在配置代码样式中了解更多信息。

激活WordPress集成后，IntelliJ IDEA将显示一个弹出窗口，供您设置WordPress代码样式。单击弹出窗口中的“ 设置它”链接，然后在“代码样式”上设置代码样式。打开的PHP页面。

要手动设置WordPress代码样式：

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|代码样式| PHP 。

    2. 在代码样式上。打开PHP页面 ，单击Set from ...链接。

    3. 从上下文菜单中，选择“ 预定义样式” ，然后选择“ WordPress” 。

根据Wordpress编码标准检查代码

借助IntelliJ IDEA，您可以结合使用PHP_CodeSniffer工具和WordPress Coding Standards来检测编码标准问题，该工具为PHP_CodeSniffer提供了一组特定于Wordpress的标准。这将确保您的代码干净，一致并且没有一些常见错误。

首先，请使用安装和配置PHP_CodeSniffer中描述的任何技术来安装PHP_CodeSniffer 。可能最简单的方法是使用Composer安装它。

要同时安装两个软件包，请将它们添加到require-dev composer.json部分，然后按照Composer中所述运行Install 。

安装PHP_CodeSniffer

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择squizlabs / php_codesniffer软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

接下来，安装WordPress编码标准 ，它将为PHP_CodeSniffer提供特定于Wordpress的标准。

安装WordPress编码标准

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择wp-coding-standards / wpcs软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

将安装WordPress编码标准软件包，并自动为PHP_CodeSniffer验证检查选择相应的Wordpress标准。如有必要，您可以在“ 编辑器” | “ 编辑器”中进一步自定义检查。 检查页面的 设置/首选项对话框Ctrl+Alt+S 。有关详细信息，请参见将PHP_CodeSniffer配置为IntelliJ IDEA检查 。

挂钩支持

IntelliJ IDEA索引在WordPress核心和随附的插件中声明的所有挂钩。这是使用钩子时进行编码辅助的基础。

完成动作和过滤器功能的参数

挂钩名称在动作和过滤器功能的标准参数的代码完成中可用add_action()和add_filter() 。

调用参数完成

  1. 开始输入add_action()要么add_filter()功能。

  2. 按Ctrl+Space然后从列表中选择相关参数。

从挂钩注册导航到挂钩调用

您可以从挂钩注册中导航（ add_action()要么add_filter()函数）到钩子调用。

  • 点击挂钩注册旁边的装订线区域中的图标。

    IntelliJ IDEA打开在其中调用该挂钩的文件，并将光标置于调用方法上，在当前示例中为do_action 。

挂钩注册中的回调

您可以导航到作为钩子注册的第二个参数指定的函数或方法的声明（ add_action()要么add_filter()功能）。

  • 随着Ctrl按下键，将光标悬停在所需参数上。IntelliJ IDEA显示弹出信息消息，其中包含指定为此参数的函数或方法的定义：

    参数变成一个链接。当您单击此链接时，IntelliJ IDEA将打开在其中声明了函数或方法的文件，并将光标和声明放在位置。

导航到挂钩调用

在“在文件中搜索目标”和“ 源代码导航”中了解有关搜索和导航的更多信息。

搜索钩子调用并导航到它

  • 请执行以下任一操作：

    使用导航到符号功能：

    1. 选择导航|主菜单中的符号 。

    2. 在出现的输入符号名称弹出窗口中， 开始在搜索字段中输入挂钩名称。如有必要，请选中“ 包括非项目项”复选框。键入时，搜索字段下方的列表内容会更改。

    3. 单击列表中的相关钩子。IntelliJ IDEA在调用文件的位置打开文件，并将光标置于调用位置。

    使用“ 随处搜索”功能：

    1. 按两次Shift 。

    2. 开始在搜索字段中输入挂钩名称。如有必要，请选中“ 包括非项目项”复选框。键入时，搜索字段下方的列表内容会更改。

    3. 单击列表中的相关钩子。IntelliJ IDEA在调用文件的位置打开文件，并将光标置于调用位置。

搜索钩子注册功能

您可以搜索钩子注册函数的出现，以及搜索被指定为钩子注册的第二个参数的函数或方法的出现。

在搜索用法中了解更多信息

列出钩子注册功能的用法

  1. 将插入号放置在所需的挂钩注册功能的名称上，然后按Alt+F7或选择编辑|查找|从主菜单中找到用法 。

  2. 从显示挂钩或功能名称方面可用选项的弹出列表中，选择相关选项。“ 查找”工具窗口打开，并带有一个新的选项卡，其中显示了所有检测到的所选功能。

  3. 探索列出的事件。要导航到相关的页面，请单击它。IntelliJ IDEA使用选定的功能或方法打开文件，然后将光标置于该文件或位置。

列出指定为挂钩注册的第二个参数的函数或方法的出现

  1. 将插入号置于感兴趣的参数处，然后按Alt+F7或选择编辑|查找|从主菜单中找到用法 。“ 查找”工具窗口打开，并带有一个新的选项卡，其中显示所有检测到的功能或方法，这些功能或方法指定为在其上调用搜索的参数。

  2. 探索列出的事件。要导航到相关的页面，请单击它。IntelliJ IDEA使用选定的功能或方法打开文件，然后将光标置于该文件或位置。

从IntelliJ IDEA查看官方WordPress文档

您可以从IntelliJ IDEA编辑器中直接在http://wordpress.org/上查看官方的WordPress文档 。

  1. 选择您感兴趣的文本。

  2. 从选择的上下文菜单中，选择“在WordPress.org上搜索” 。

    IntelliJ IDEA在默认浏览器中打开包含相关文档的页面。
