使用库项目共享Android源代码和资源

IntelliJ IDEA支持包含共享Android源代码和资源的Android 库项目 。其他Android应用程序项目可以引用一个库项目，并在构建时将其编译后的源包含在其.apk文件中。

在IntelliJ IDEA中，通过单独的库模块支持库项目。

要启用共享Android源代码和资源，请执行以下一项操作：

  • 创建一个新的库模块。

  • 将包含要共享的源的“应用程序”模块转换为“库”模块。

在本主题中：

  • 创建一个库模块
  • 将应用程序模块转换为库模块
  • 在另一个项目中使用库模块
  • 将库模块的AndroidManifest.xml中的数据添加到整个应用程序的AndroidManifest.xml中
  • 将库模块的.dex文件包括到整个应用程序的.apk中，而无需重新构建（预删除）

创建一个库模块

  1. 请执行以下任一操作：
    • 从头开始创建项目
    • 将模块添加到现有项目

  2. 在向导的第一页上，在左窗格中选择“ Android” ，在右窗格中选择“ 库模块 ”。

  3. 完成向导。

将应用程序模块转换为库模块

您可以通过更新Android构面将Application模块转换为Library模块。

  1. 通过选择文件|打开项目结构对话框。从主菜单的项目结构 。

  2. 在左窗格中选择“ 模块 ”。在中央窗格中，展开要变成“库”模块的模块的节点，然后单击“ Android” 。

  3. 在右侧窗格中，选中“ Android facet”页面顶部的“ 库模块”复选框。

在另一个项目中使用库模块

要在另一个项目中使用Library模块，您需要将该模块导入其中。

  1. 从主菜单中，选择文件|新品|现有资料模块 。

  2. 在打开的对话框中 ，浏览到要导入的iml模块文件，然后单击“ 确定” 。模块节点将添加到树视图中。

  3. 将导入的库模块上的依赖项添加到将使用其数据的模块中：

    1. 打开非库模块的设置：导航到文件|项目结构 ，在左窗格中选择模块 ，并在中央窗格中选择您的非图书馆模块。

    2. 在右窗格中，切换到“ 依赖关系”选项卡。点击添加按钮在右侧的工具栏中，然后从上下文菜单中选择“ 模块依赖关系 ”。

    3. 在打开的“选择模块”对话框中，从列表中选择导入的库模块，然后单击“ 确定” 。

将库模块的AndroidManifest.xml中的数据添加到整个应用程序的AndroidManifest.xml中

要将库模块成功集成到另一个应用程序中，必须在应用程序AndroidManifest.xml文件中声明其组件。清单文件包含运行应用程序所需的信息（有关更多信息，请参见App Manifest ）。您可以手动在库模块上添加此信息，也可以从库模块的AndroidManifest.xml中提取该信息，然后将其自动添加到应用程序的AndroidManifest.xml中。

第二种方法称为合并清单 。

要将库模块的清单与应用程序清单自动合并：

  1. 通过选择文件|打开项目结构对话框。从主菜单的项目结构 。

  2. 在左窗格中选择“ 模块 ”。在中央窗格中，展开非库模块，然后单击其节点下的Android facet。

  3. 在右窗格中的“ Android facet”页面上 ，切换到“ 打包”选项卡，然后选择“ 启用清单合并”选项。

将库模块的.dex文件包括到整个应用程序的.apk中，而无需重新构建（预删除）

在应用程序打包期间，库模块的类文件将转换为.dex文件。此操作称为dexing 。最后，从库模块输出的.dex文件包含在最终应用程序.apk中 （从Building and Running中了解有关构建过程的更多信息）。

通常，库模块的内容保持不变。在这种情况下，您可以拥有它们dexed仅一次，因此输出的.dex文件包含在.apk中 。这种方法称为“ 预去除” 。

默认情况下，IntelliJ IDEA预删除库模式依赖关系以及外部jars自上次构建以来尚未更新的文件。您可以更改这些设置，以便始终清除所有类文件。

  1. 通过选择文件|打开项目结构对话框。从主菜单的项目结构 。

  2. 在左窗格中选择“ 模块 ”。在中央窗格中，展开非库模块，然后单击其节点下的Android facet。

  3. 在右窗格的Android facet页面上 ，切换到Packaging标签，然后选择Pre-dex外部jar和Android库依赖项选项。
