测试驱动开发

无论您是想在编写生产代码之前编写测试，还是想在之后创建测试，IntelliJ IDEA都可以轻松地创建和运行单元测试。在本教程中，我们将展示如何使用IntelliJ IDEA首先编写测试（ Test Driven Development或TDD ）。

先决条件

本教程假定满足以下先决条件：

  • 您已经在IntelliJ IDEA中创建了Java项目 。

  • 您有用于生产和测试代码的文件夹，可以手动创建 ，也可以从创建项目时创建 。

  • 通过添加测试库或在Gradle或Maven中添加依赖项，可以在类路径上具有JUnit 4。

  • 您的源和测试根目录已正确配置-源根目录将显示为蓝色文件夹，测试文件夹将具有绿色背景。

  • 您已经在项目中创建了所需的包。

创建您的第一个测试

鉴于我们先编写测试而不必向我们提供要测试的类，因此，我们将通过项目面板创建第一个测试。

  1. 右键单击要在其中创建测试的软件包，然后选择“ 新建” |“新建”。 Java类 。

  2. 输入测试名称-假设我们正在使用JUnit进行测试，则该名称可能是[Something] Test，例如MoodAnalyserTest 。

  3. 将光标放在班级大括号内的某个位置，按Alt+Insert 。

  4. 选择测试方法|菜单中的JUnit 4 。这将使用默认模板创建一个测试方法。填写测试名称，按Enter键，光标将最终出现在方法主体中。

    您可以更改默认的测试方法模板 -例如，如果您希望将方法名称的开头更改为test至should 。

编写测试体

为不存在的类和方法编写测试代码似乎违反直觉，但是IntelliJ IDEA使这一过程变得简单而又使编译器满意。IntelliJ IDEA可以为您创建类和方法（如果尚不存在）。

  1. 编写测试描述您要达到的目标，然后按Alt+Enter在任何不存在的类上，然后选择“创建类'[ClassName]'”。这将为您提供使编译器满意的最低限度实现。

    与往常一样，您可以使用IntelliJ IDEA的重构工具来创建变量来存储结果，如果在类路径上有正确的库，则IntelliJ IDEA将为您导入最合适的类。

  2. 继续编写测试主体，包括不存在的所需方法的名称，再次可以使用Alt+Enter并选择“创建方法'[methodName]'”，以使IntelliJ IDEA创建一个简单的骨架方法。

运行测试

遵循TDD方法时，通常会经历一个Red-Green-Refactor循环。您将运行一个测试，看到它失败（变成红色），实施最简单的代码以使测试通过（变成绿色），然后重构代码，以使测试保持绿色并且代码足够干净。

此循环的第一步是运行测试，然后查看失败。

鉴于我们已经使用IntelliJ IDEA的功能来创建要测试的方法的最简单的空实现，因此我们不希望测试通过。

  • 在测试内部，按Ctrl+Shift+F10运行此单独的测试。

    结果将显示在运行对话框中 。测试名称旁边将有一个图标-红色表示异常，黄色表示断言。对于任何一种失败类型，也会显示一条消息，指出发生了什么问题。

实施守则

下一步是使测试通过，这意味着实施最简单的方法。

  1. 您可以使用常用方法导航至要测试的代码-单击方法名称，然后按Ctrl+Alt+B当光标在方法名称上时，或按Ctrl+Shift+T在测试代码和生产代码之间切换。

  2. 更改方法以通过测试。通常，使用TDD时，最简单的方法可能是对您的期望值进行硬编码。稍后我们将看到在此过程上进行迭代将如何导致更实际的生产代码。

  3. 使用以下命令重新运行测试Shift+F10重新运行上一个测试。

  4. 请参阅测试通过-测试方法旁边的图标应变为绿色。如果没有，请进行必要的更改，直到测试通过。

重复

开发代码是一个反复的过程。当遵循TDD风格的方法时，情况更是如此。为了排除更复杂的行为，我们为其他情况添加了测试。

  1. 在您的测试课中，使用Alt+Insert再次创建新的测试方法。

  2. 选择另一个测试用例，它指定要测试的方法的更多要求。请记住，可以使用IntelliJ IDEA的功能来创建类和方法，以使编译器满意。

  3. 运行第二个测试用例，显示它由于正确的原因而失败。

  4. 更改要测试的方法中的代码以使此测试通过。

  5. 通过重新运行两个测试Ctrl+Shift+F10在测试类中，而不是在单个方法中，看到这两个测试现在都通过了。如果任何一个测试失败，请对代码进行必要的更改以确保测试通过。

摘要

以测试优先方式编写第一个测试需要进行少量设置-创建测试类，创建测试方法，然后为最终将成为生产代码的代码创建空实现。IntelliJ IDEA可自动执行许多初始设置。

当您遍历该过程，创建测试并进行必要的更改以使这些测试通过时，您将针对所需的功能建立一套全面的测试套件，并满足这些要求的最简单的解决方案。
