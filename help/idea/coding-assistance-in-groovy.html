在编辑器中使用Groovy代码

在编辑器中使用Groovy代码时，IntelliJ IDEA为您提供各种代码完成，重构，检查和Groovy意向操作，可帮助您更快，更轻松地编写代码。

代码完成

通过按访问基本完成时 Ctrl+Space ，您将获得有关变量，类型，方法，表达式的基本建议，对于参数名称，您将获得类型建议等。两次调用“ 基本完成”时 ，它会向您显示更多结果。

智能完成功能了解预期的类型和数据流，并提供与上下文相关的选项。要调用“ 智能完成”功能 ，请按Ctrl+Shift+Space 。当您两次调用智能完成功能时 ，它会为您显示更多结果，包括连锁店。

要覆盖插入符处的标识符，而不仅仅是插入建议，请按Tab 。如果您要编辑部分标识符（例如文件名），这将很有帮助。

要让IntelliJ IDEA为您完成声明，请按Ctrl+Shift+Enter 。语句完成将自动添加缺少的括号，方括号，花括号和必要的格式。

Postfix Completion功能使您可以根据在点后键入的后缀将已键入的表达式转换为另一个表达式。

有关更多详细信息，请参阅自动完成代码 。

重构基础

IntelliJ IDEA提供了一套全面的自动代码重构功能，如果正确使用它们，可以大大提高生产率。首先，在应用重构之前不要打扰任何选择。IntelliJ IDEA足够聪明，可以找出您要重构的语句，并且仅询问是否有几种选择是可能的。

要撤消上一次重构，请将焦点切换到“ 项目工具”窗口 ，然后按Ctrl+Z 。

常规检查

检查是内置的静态代码分析工具，可以帮助您发现可能的错误，查找无效代码，检测性能问题并改善整体代码结构。

要检查和配置可用的Groovy检查， 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|检查 。

大多数检查不仅告诉您问题出在哪里，而且还提供快速修复以立即解决问题。按Alt+Enter选择快速修复。

编辑器使您可以通过键盘快捷键在突出显示的问题之间快速导航。按F2转到下一个问题，并且Shift+F2转到上一个。

当您对整个项目执行代码分析时，可以进行太复杂而无法即时运行的检查。您可以通过以下两种方式之一进行：通过选择分析|从主菜单检查代码 ，或选择“ 分析” |“ 检查” 。按名称运行检查以按其名称运行检查。

请注意，虽然检查可以为存在潜在问题的代码提供快速修复，但是这样做可以帮助您将自动更改应用于正确的代码。要获得适用于插入符号的意图列表，请按Alt+Enter 。

有关更多详细信息，请参阅代码检查 。

时髦意图

要阅读任何Groovy意图操作的简短描述，请查看其代码示例，并禁用不需要的代码示例，请访问Groovy意图设置。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|用心 。

  2. 在选项列表中，单击“ Groovy”节点，找到所需的意图并进行调整。

  3. 或者，在编辑器中，当您按Alt+Enter要从上下文菜单中选择意图动作，请按其旁边的箭头键以选择一个选项来编辑其设置或禁用意图。

Groovy中的字符串

带着表演的意图 Alt+Enter您可以为字符串选择不同的用途，例如将单行更改为多行，添加串联，将文字字符串转换为GString等。

  • 转换为多行 ：将单个长行转换为多行。您可以在不添加任何串联的情况下分解行。这对于复制和粘贴代码很有帮助。

  • 转换为单行文字 ：将多行转换回单行。在这种情况下，转义序列/n被添加。您可以继续并选择“ / n上的分隔字符串”操作以拆分添加连接的行。

  • 转换为字符串和转换为GString ：将GString转换为常规String，反之亦然。

在Groovy中声明显式类型

在标准声明中，例如为参数创建字段或将局部变量引入表达式的结果中，IntelliJ IDEA允许您为所选变量声明显式类型。添加显式声明使大型项目中的代码更具可读性和可持续性。

  • 在编辑器中，按Alt+Enter在所选变量上，然后从意图列表中选择声明显式类型 。要删除它，请使用“ 删除显式类型”操作。

向参数添加显式类型

您可以在没有明确定义参数类型的Groovy方法的方法声明中为所有参数添加明确类型。

  1. 在编辑器中，将插入符号放置在所需的参数上，然后按Alt+Enter 。

  2. 从上下文菜单中，选择“ 将显式类型添加到参数” 。

转换为@CompileStatic

由于默认情况下Groovy代码是动态编译的，因此IntelliJ IDEA允许您使用@CompileStatic使用此注释时，该注释将切换到静态编译。

  1. 在编辑器中，您可以自动添加@CompileStatic批注。从上下文菜单中，选择“ 重构|转换为@CompileStatic 。

    IntelliJ IDEA对代码进行了以下调整：

    • 的@CompileDynamic注释已添加到依赖于动态语言功能并且无法自动转换的方法中。

    • 需要时添加显式类型强制。

    • 当可以从字段初始化程序中推断出字段时，将向字段添加显式类型。

    • 一个显式的返回类型被添加到方法中。

  2. 如果您添加了@CompileStatic手动注释并想修复相关的红色代码，请调用“ 转换为@CompileStatic”意图操作。

将匿名类更改为动态实例

您可以将匿名类转换为闭包以提高可读性。

  1. 在编辑器中，选择要转换的类，然后按Alt+Enter 。

  2. 从选项列表中，选择“ 更改为动态实例化” 。
