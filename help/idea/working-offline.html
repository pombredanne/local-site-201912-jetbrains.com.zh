IntelliJ IDEA 2019.3帮助

离线工作

IntelliJ IDEA中的许多功能都需要访问Internet。如果您离线工作（例如，在隔离的环境中），则应牢记某些方面。

如果您没有Internet权限来查看联机帮助，则可以使用IntelliJ IDEA帮助插件，该插件通过内置的Web服务器提供帮助页面以供离线使用。

更新

默认情况下，IntelliJ IDEA配置为自动检查更新，并在有新版本可用时通知您。更新通常是基于修补程序的 ：它们适用于现有安装，仅要求您重新启动IDE。但是，有时补丁更新不可用，并且必须安装新版本的IntelliJ IDEA。

如果IntelliJ IDEA在本地网络外部没有HTTP访问权限，则将无法检查更新和应用补丁。在这种情况下，您必须下载IDE的新版本并按照独立安装中所述手动安装它们。

如果没有Internet访问，则无法使用Toolbox App和snaps安装IntelliJ IDEA。

有关更多信息，请参见更新IntelliJ IDEA 。

外挂程式

通常，插件是从JetBrains插件存储库安装的。但是，您可以在本地网络中设置自定义插件存储库 ，并将IntelliJ IDEA配置为使用它来安装和更新插件。

或者，您可以从disk下载并手动安装插件 。

许可证激活

您最多可以评估IntelliJ IDEA 30天。之后，您需要购买并注册许可证。

如果IntelliJ IDEA在本地网络外部没有HTTP访问权限，则将无法使用JetBrains帐户登录。但是，您可以生成一个离线激活码，该激活码在您的订阅期内有效。

如果您的组织至少拥有50个JetBrains产品的有效订阅或许可证，则可以使用浮动许可证服务器激活公司网络中的IntelliJ IDEA实例。请记住，许可证服务器本身需要Internet访问才能连接到JetBrains帐户。

有关更多信息，请参见注册IntelliJ IDEA 。

代码检查

一些代码检查会验证外部资源。例如， 不存在的Web资源检查会突出显示无效链接。如果您无法访问互联网，这些检查将无法进行，无效链接也不会突出显示。

有关更多信息，请参阅代码检查 。

外部文件

外部文档会在Web浏览器中打开必要的信息，以便您可以导航到相关符号并同时保留信息以备将来参考。在为项目配置外部文档后，您还可以在快速文档弹出窗口中查看它。

脱机访问外部文档

如果您离线工作，则可以在本地查看外部文档。

  1. 下载所需版本的文档包。该文档包通常以ZIP压缩文件的形式分发，下载后需要解压缩。

    例如，您可以下载官方Java SE Development Kit 8文档并解压缩。

  2. 在项目结构对话框中Ctrl+Shift+Alt+S ，选择SDK 。

  3. 如果已配置了多个JDK，则选择必需的JDK版本，然后打开右侧的Documentation Path选项卡。

  4. 点击图标，并使用下载的文档包指定目录（例如C：\ Users \ john.doe \ Desktop \ docs \ api ）。

  5. 应用更改并关闭对话框。

有关更多信息，请参阅外部文档 。

版本控制系统

您的源代码很可能在某种版本控制系统（VCS）下。如果远程存储库不在您的本地网络中，并且无法访问Internet，则IntelliJ IDEA将无法与VCS通信。例如，如果您使用的是Git，则可以提交更改，但不能将其推送到远程存储库或从中获取更新。

有关VCS集成的更多信息，请参见版本控制 。

任务和问题跟踪器

您可以与问题跟踪器建立连接，以处理直接从IntelliJ IDEA分配给您的任务和错误。例如，您可以连接到YouTrack ， Jira ， GitHub等。

如果问题跟踪服务器不在您的本地网络中，并且没有Internet访问，则IntelliJ IDEA将无法同步您的问题。在这种情况下，您将只能处理自己创建的本地任务。

有关更多信息，请参见任务和上下文 。

Maven依赖

默认情况下，Maven连接到远程存储库，并在每次启动时检查更新。解决Maven依赖项可能需要下载新的工件。如果希望Maven仅使用本地可用的那些资源，则可以切换到脱机模式。

将Maven切换到离线模式

  • 在Maven工具窗口中，单击 。

这将附加--offline IntelliJ IDEA运行的所有Maven命令的选项。它还将报告在本地存储库中找不到的所有项目。

Gradle依赖

默认情况下，Gradle连接到远程存储库并在每次启动时检查更新。解决Gradle依赖性可能需要下载新的工件。如果希望Gradle仅使用本地可用的那些资源，则可以切换到离线模式。

将Gradle切换为离线模式

  • 在Gradle工具窗口中，单击 。

这将附加--offline IntelliJ IDEA运行的所有Gradle命令的选项。它还将报告在本地存储库中找不到的所有项目。

使用情况统计

首次运行IntelliJ IDEA时，系统会提示您是否发送有关所使用的功能和插件，硬件和软件配置，文件类型，每个项目的文件数等的匿名数据。这不包括任何个人或敏感数据，例如部分源代码或文件名。这些信息是根据JetBrains隐私政策收集的，用于帮助改善产品和整体体验。

即使您启用匿名使用情况统计信息，如果您的本地网络外部没有HTTP访问，也不会发送该统计信息。另外，如果您先同意，然后再改变主意，则可以完全禁用此功能。

禁用发送使用情况统计信息

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择外观和行为|系统设置|数据共享 。

  2. 清除发送使用情况统计信息复选框。
