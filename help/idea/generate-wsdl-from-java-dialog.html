从Java对话框生成WSDL

工具|网络服务|从Java代码生成WSDL

使用该对话框来配置Web服务WSDL生成，并选择要作为Web服务操作公开和部署的方法。对话框的内容取决于Web服务类型。

Web服务URL

指定将在其上提供Web服务的URL地址。

所有

操作方法

指定要将所选类的哪些方法部署为Web服务操作。

  • 公开方法 ：显示所选类中所有方法的列表。

  • 添加到部署 ：选中要作为操作部署的方法旁边的复选框。

如果无法选择方法，则工具提示将说明原因。

阿帕奇轴心

Web服务命名空间

指定Web服务名称空间前缀。

阿帕奇轴心

SOAP动作

选择的值soapAction的属性领域。可用的选项有：

  • 默认值 ：设置soapAction根据操作的元数据（通常为“”）。

  • 操作方式 ：设置soapAction是该操作的名称。

  • 无 ：设置soapAction是“”。

阿帕奇轴心

装订风格

指定要生成的WSDL文档的样式。可用的选项有：

  • RPC ：生成一个rpc WSDL（默认情况下选中）。

  • 文档 ：生成文档 WSDL。

  • 包装 ：使用包装方法生成WSDL。如果选择此选项，则会在“ 使用绑定中的项目”列表中自动选择“ 文字” 。

阿帕奇轴心

在绑定中使用项目

选择应如何使用生成的WSDL文档。可用的选项有：

  • 文字 ：通过XML Schema定义请求的XML表示形式。

  • 编码 ：在生成的WSDL中指定SOAP编码。

阿帕奇轴心

类型映射版本

使用默认的序列化程序，指定用于将Java类映射到XML限定名称的默认类型映射注册表。可用的选项有：

  • 1.1表示符合SOAP 1.1 JAX-RPC。

  • 1.2表示SOAP 1.1编码。
