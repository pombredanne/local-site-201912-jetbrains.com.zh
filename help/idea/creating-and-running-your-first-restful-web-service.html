在GlassFish应用程序服务器上创建和运行您的第一个RESTful Web服务

本教程说明了开发简单的RESTful Web服务并将其部署到GlassFish应用程序服务器。

  • 在你开始前
  • 在IntelliJ IDEA中配置GlassFish服务器
  • 配置JDK
  • 建立专案
  • 探索项目结构
  • 开发源代码
  • 检查生成的工件配置
  • 探索并完成运行配置
  • 运行应用程序

在你开始前

  1. 安装Java SE Development Kit（JDK）版本1.8或更高版本，请参阅下载Oracle JDK 。

  2. 下载3.0.1版或更高版本的GlassFish应用服务器，请参阅下载GlassFish 。

  3. 确保您的计算机上可以使用网络浏览器。

在IntelliJ IDEA中配置GlassFish服务器

  1. 通过按打开设置/首选项对话框 Ctrl+Alt+S或选择文件|设置 Windows和Linux或IntelliJ IDEA的| macOS的首选项 ，以及单击“ 构建，执行，部署”下的“ 应用服务器 ”。

  2. 在打开的“ 应用程序服务器”页面上，单击在中央窗格上方，然后从列表中选择“ GlassFish Server ”。

  3. 在右侧窗格中的“ GlassFish主页”字段中指定GlassFish Server安装文件夹。手动输入路径或单击然后在打开的对话框中选择安装文件夹。IntelliJ IDEA检测到应用程序服务器的版本，并按如下所示自动填写“ 名称”字段： GlassFish 。在我们的示例中是GlassFish 4.1.1 。

    其他字段将自动填写或为可选字段，因此只需单击确定 。

配置JDK

  1. 按Ctrl+Shift+Alt+S或选择文件|从主菜单的项目结构 。

  2. 在打开的“ 项目结构”对话框中，在“ 平台设置”下选择“ SDK ” 。

  3. 在打开的“ SDK”页面上，单击在中央窗格上方，然后选择JDK 。

  4. 在右侧窗格中，指定要使用的Java SE Development Kit（JDK）的安装文件夹。手动输入路径或单击然后在打开的对话框中选择安装文件夹。IntelliJ IDEA检测到JDK的版本，并自动在“ 名称”字段中输入它。在我们的示例中为1.8 。

    其他选项卡中的所有必填字段将自动填写，因此只需点击确定 。

建立专案

  1. 在“欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件” |“ 新建” 。 新品|在主菜单上项目 。

    将打开“ 新建项目”向导。

  2. 在向导的第一页上：

    1. 在左侧窗格中，选择Java Enterprise 。

    2. 从Project SDK列表中，选择要使用的JDK。在我们的示例中，它是1.8 。

    3. 从“ 应用程序服务器”列表中，选择“ GlassFish 4.1.1” 。

    4. 从JavaEE版本下拉列表中，选择JavaEE 7 。

    5. 在“ 其他库和框架”区域中，选中“ Web应用程序”和“ RESTful Web服务”复选框。

    6. 在“ 其他库和框架”列表下方的区域中选择“ 下载”选项。仅在选中“ Web应用程序”和“ RESTful Web服务”复选框后，才会显示该区域。

    7. 单击下一步 。

  3. 在向导的第二页上，指定新项目的名称，在我们的示例中为Rest_glassfish_hello_world 。

    单击“ 完成”，然后在IntelliJ IDEA创建项目时等待。

探索项目结构

创建项目后，您将在“ 项目”工具窗口中看到类似的内容：

  • rest_glassfish_hello_world是一个模块文件夹（在这种情况下与项目文件夹一致）。

  • .idea文件夹和文件Rest_glassfish_hello_world.iml分别包含项目和模块的配置数据。

  • 文件夹src用于您的Java源代码。

  • Web文件夹用于您的应用程序的Web部分。目前，该文件夹包含部署描述符WEB-INF \ web.xml 。

  • 外部库包括用于GlassFish Server的 JDK和JAR文件。

    探索并完成运行配置

    在IntelliJ IDEA中，根据专用运行配置启动任何应用程序。在项目创建期间，我们已将GlassFish Server指定为运行我们的应用程序的应用程序服务器。基于此选择和代码中的注释，IntelliJ IDEA创建了运行配置并填充了几乎所有必填字段。

    1. 从主菜单中，选择运行|编辑配置 。

    2. 在打开的“ 编辑配置”对话框中，展开“ GlassFish服务器”节点，然后单击“ GlassFish 4.1.1” 。右侧窗格显示自动生成的运行配置的设置。

      • “ 应用程序服务器”字段显示GlassFish 4.1.1 ，这是在项目创建过程中选择的GlassFish Server的安装。“ 名称”字段还显示GlassFish 4.1.1 ，IntelliJ IDEA已根据指定的应用程序服务器自动命名了生成的配置。

      • 在“ 打开浏览器”区域中，选中“ 启动后”复选框，因此将自动打开包含应用程序输出的页面。

        在下面的字段中，我们需要指定要打开的页面的URL地址。在我们的示例中，它是http：// localhost：8080 / rest_glassfish_hello_world_war_exploded / helloworld 。

      • 要在启动运行配置时自动分解rest_glassfish_hello_world：war，必须标记工件以进行部署。如果您已成功完成项目创建步骤，则将工件标记为自动部署。如果不是，则IntelliJ IDEA将显示警告“ 没有标记为部署的工件”和“ 修复”按钮。

        当您单击Fix时 ，IntelliJ IDEA将打开Deployment选项卡，在该选项卡中将rest_glassfish_hello_world：war爆炸的内容添加到服务器启动中的Deploy列表中。

      所有其他字段将自动填写或为可选，因此只需单击“ 确定”以保存运行配置。

    运行应用程序

    请点击在工具栏上。之后：

    1. IntelliJ IDEA编译您的源代码并构建应用程序工件。

    2. 将打开“ 运行工具”窗口 。IntelliJ IDEA启动服务器并在其上部署工件。

    3. 最后，您的默认Web浏览器启动，您会看到应用程序输出Hello World那里。
