硬编码字符串文字

如果您的源代码包含硬编码的字符串文字，则可以启用国际化代码检查以突出显示它们。然后，您可以将这些字符串提取到属性文件中进行本地化，或者如果不打算将其本地化，则可以忽略它们。

启用突出显示硬编码的字符串文字

  1. 打开设置/首选项对话框Ctrl+Alt+S ，展开编辑器 ，然后单击检查 。

  2. 选择所需的概要文件，然后在Java下找到节点国际化问题 。

  3. 启用“ 硬编码字符串检查”以在编辑器中突出显示硬编码字符串文字。

  4. 应用更改并关闭对话框。

提取硬编码的字符串文字

IntelliJ IDEA提供了一种特殊的动作， 即对硬编码的字符串文字进行提取以将字符串文字提取到属性文件中。您可以使用java.util访问资源束。ResourceBundle类或自定义资源包类 。

使用java.util提取字符串文字。资源束

  1. 在您的源代码中，以以下形式指定将用于存储提取的文字的资源包：

    私有静态ResourceBundle = ResourceBundle.getBundle（“ ”）;

    例如：

    私有静态ResourceBundle myBundle = ResourceBundle.getBundle（“ awesomeBundle”）;

  2. 单击突出显示的字符串，然后按Alt+Enter ，Editing_HTML_Files.xmlEditing_HTML_Files.xml并选择i18nize硬编码字符串文字 。

  3. 在“ I18nize硬编码字符串文字”对话框中，指定目标属性文件，属性的键和值以及资源束表达式。

    如果ResourceBundle字段已在源代码中声明，默认情况下IntelliJ IDEA建议使用其名称。如果尚未在源代码中声明此字段，则仍可以立即在对话框中定义所需的表达式。为此，请输入有效的ResourceBundle在资源包表达式字段中输入。

    指定资源束表达式时，可以使用基本代码完成 。输入班级名称并按Ctrl+Space在句点之后选择一种方法。

  4. 单击确定 。带有硬编码字符串文字的行将被替换。

    • 如果资源包已在源代码中声明：

      System.out.println（myBundle.getString（“ hello.world”））;

    • 如果在对话框中定义了资源束：

      System.out.println（ResourceBundle.getBundle（“ awesomeBundle”）。getString（“ hello.world”）））;

使用自定义资源包类提取字符串文字

  1. 确保将IntelliJ IDEA安装下的redist / annotations.jar存档添加到模块依赖项中。

  2. 在您的项目中创建一个新的Java类，并粘贴以下代码：

    导入org.jetbrains.annotations。PropertyKey;导入org.jetbrains.annotations。NonNls;导入java.util。ResourceBundle;导入java.text。MessageFormat;公共类I18nSupport {@NonNls私有静态最终ResourceBundle捆绑包= ResourceBundle.getBundle（“ com.intellij。FontChooser“）；公共静态字符串i18n_str（@PropertyKey（resourceBundle =” com.intellij。FontChooser“）String key，Object ... params）{字符串值= bundle.getString（key）;如果（params.length> 0）返回MessageFormat.format（value，params）;返回值;}}

  3. 在包含硬编码文字的类中，单击突出显示的字符串，然后按Alt+Enter ，然后在意图操作列表中选择i18nize硬编码字符串文字 。I18nize Hardcoded String对话框打开，没有有效的资源包表达式。

  4. 单击编辑i18n模板 。在“ 文件模板”对话框中，更改“ 国际化表达式”以指向您的自定义资源包类的方法。

    单击“ 确定”保存更新的模板，然后关闭对话框。

  5. 在“ I18nize硬编码字符串”对话框的“ 预览”部分中，验证建议的替代，然后单击“ 确定” 。源代码将相应更改：

    System.out.println（I18nSupport.i18n_str（“ hello.world”））;

忽略硬编码的字符串文字

如果要忽略硬编码的字符串文字，请使用“ 不需要国际化注释”。

  1. 按Alt+Enter显示字符串文字的意图动作。

  2. 从操作列表中选择“ 作为@NonNls注释” 。

  3. 指定您要存储annotations.xml文件的位置。
