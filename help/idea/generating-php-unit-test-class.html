生成PHP测试

仅当安装并启用PHP插件时，以下内容才有效。

您可以根据要测试的PHP类自动生成测试类存根。IntelliJ IDEA为PHPUnit ， Codeception和PHPSpec测试框架提供了此功能。如果使用的是Behat框架，则必须手动创建测试。

Test类是一个PHP类，其名称源自生产类名称。例如，如果要测试的类是MyClass.php ，则IntelliJ IDEA会自动将对应的PHPUnit测试类的名称组成Test.php 。

IntelliJ IDEA可以针对单独文件中定义的类以及单个PHP文件中定义的类生成测试。在后一种情况下，IntelliJ IDEA将为每个生成的测试类创建一个单独的文件。

  1. 请执行以下任一操作：

    • 从主菜单中，选择文件|新的 。然后，选择PHP Test |测试或PHP测试|从弹出菜单进行测试 。

    • 在项目工具窗口中，按Alt+Insert或右键单击要测试的PHP类，然后选择“ 新建| PHP测试|测试或PHP测试|从选择上下文菜单中进行测试 。

    • 在要测试的PHP类的编辑器中，将插入号放置在类的定义内。然后，选择转到|从上下文菜单进行测试或按Ctrl+Shift+T并从弹出菜单中选择创建新测试 。这样，您可以为一个PHP文件中的多个类之间定义的PHP类生成测试。

      要为特定方法创建测试，请将插入号放置在方法声明中。所选方法将在“ 创建新PHP测试”对话框的“ 生成测试方法”区域中自动选择。

    将打开“ 创建新的PHP测试”对话框：

  2. 在“ 创建新的PHP测试”对话框中，指定以下数据：

    1. 将基于其生成测试类的测试文件模板 。IntelliJ IDEA提供了内置文件模板，用于使用以下受支持的测试框架生成测试类： PHPUnit ， Codeception和PHPSpec 。

    2. 测试类的名称 。IntelliJ IDEA根据所选测试框架的命名约定自动从生产类名称中组成名称。例如，如果要测试的类是MyClass.php ，则IntelliJ IDEA会自动将PHPUnit测试类的名称组成Test.php 。

    3. 将根据生产类的包含目录和名称空间，配置的测试源root及其psr-4包前缀或相应测试框架的配置文件中提供的值自动建议测试类的文件夹。要使用路径补全，请按Ctrl+Space然后从列表中选择路径。

    4. 测试类所属的名称空间 。IntelliJ IDEA可以根据配置的PSR包前缀和特定于框架的配置文件中指定的值自动完成名称空间 。要使用名称空间补全，请按Ctrl+Space并从列表中选择相关的名称空间。

    5. 在“ 生成测试方法用于”区域中，选中要为其生成测试方法存根的生产类方法旁边的复选框。要包括从父类继承的方法，请选中显示继承的方法复选框。

      IntelliJ IDEA将自动组合测试方法的名称为test 。您可以在“ 文件和代码模板设置”页面的“代码”选项卡上自定义用于生成测试方法存根的代码模板 。

测试准备就绪后，可以通过选择导航|返回到生产类。去测试主题 。有关详细信息，请参阅在测试和测试主题之间导航 。
