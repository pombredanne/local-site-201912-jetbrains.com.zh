在Tomcat服务器容器内部署Java Web应用程序

您可以使用Docker运行Tomcat服务器并部署Java Web应用程序。本教程介绍如何创建简单的Java Web应用程序，构建可部署的Web应用程序资源（WAR）文件，然后将其部署在作为Docker容器运行的Tomcat服务器中。

在开始之前，请确保已正确配置Docker集成 。

创建一个Java Web应用程序

  1. 打开“ 文件”菜单，指向“ 新建” ，然后单击“ 项目” 。

  2. 单击Java ，然后选择Web应用程序 ，然后单击下一步 。

  3. 指定名称DockerJavaWebApp并单击Finish 。

  4. 在“ 项目”工具窗口中，右键单击src ，指向“ 新建” ，然后单击“ Servlet” 。

  5. 指定名称MyServlet并单击OK 。

构建一个WAR工件

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S然后点击工件 。

  2. 点击添加 ，指向Web应用程序：存档 ，然后单击For'DockerJavaWebApp：war exploded' 。如有必要，创建清单文件。

  3. 打开“ 生成”菜单，然后单击“ 生成工件” 。选择以构建DockerJavaWebApp：war工件。您应该看到工件out / artifacts / DockerJavaWebApp_war / DockerJavaWebApp_war.war 。

在Tomcat服务器中部署应用程序

  1. 在Docker工具窗口中，右键单击Images节点，然后单击Pull image 。

  2. 在“ 拉取映像”对话框中，选择默认的Docker Hub注册表，并指定Tomcat服务器映像存储库名称和标签。 tomcat和latest 。

  3. 单击确定，然后等待直到图像被拉出。

  4. 在Docker工具窗口中，右键单击Tomcat服务器映像，然后单击创建容器 。

  5. 在“ 创建容器”弹出窗口中，单击“ 创建” 。

  6. 在“ 创建Docker配置”对话框中，执行以下操作：

    • 指定容器的名称： TomcatContainer

    • 将容器端口8080绑定到主机IP 127.0.0.1和端口8080

    • 将WAR工件输出目录[PROJECT_PATH] / out / artifacts / DockerJavaWebApp_war /映射到Tomcat服务器部署目录/ usr / local / tomcat / webapps / 。

    • 单击运行以启动容器。

  7. 容器启动后，在Web浏览器中打开以下地址： http : //127.0.0.1 : 8080/DockerJavaWebApp_war/ 。
