在Wildfly服务器容器内部署Java Web应用程序

您可以使用Docker运行Wildfly服务器并在其上部署Java Web应用程序。本教程描述了如何创建一个简单的Java Web应用程序，构建一个可部署的Web应用程序资源（WAR）文件，然后将其部署在作为Docker容器运行的Wildfly服务器中。

在开始之前，请确保已正确配置Docker集成 。

创建一个Java Web应用程序

  1. 打开“ 文件”菜单，指向“ 新建” ，然后单击“ 项目” 。

  2. 单击Java ，然后选择Web应用程序 ，然后单击下一步 。

  3. 指定名称SimpleJavaWebApp并单击Finish 。

  4. 打开web / index.jsp文件，并将以下HTML代码粘贴为servlet的入口点：

    <％@页面contentType =“ text / html; charset = UTF-8” language =“ java”％> 简单的Java Web App演示

    您好，我是Java网络应用程序！

    我正在从WAR工件运行。

  5. 在“ 项目”工具窗口中选择顶级节点，打开“ 文件”菜单，指向“ 新建” ，然后单击“ 目录” 。

  6. 输入目录名称docker-dir ，然后单击OK 。

    该目录将用于Dockerfile和WAR工件。

构建一个WAR工件

  1. 打开项目设置Ctrl+Shift+Alt+S然后点击工件 。

  2. 点击添加 ，指向Web应用程序：存档 ，然后单击对于'SimpleJavaWebApp：war exploded' 。如有必要，创建清单文件。

  3. 将工件的输出目录更改为[PROJECT_PATH] / docker-dir / SimpleJavaWebApp_war 。

  4. 打开“ 生成”菜单，然后单击“ 生成工件” 。选择以构建SimpleJavaWebApp：war工件。您应该在指定的输出目录中看到工件。

从Dockerfile运行Wildfly服务器容器

  1. 在“ 项目”工具窗口中，右键单击docker-dir目录，指向“ 新建” ，然后单击“ 文件” 。

  2. 输入文件名Dockerfile并单击“ 确定” 。

  3. 将以下代码粘贴到Dockerfile中：

    从jboss / wildfly复制COPY SimpleJavaWebApp_war / SimpleJavaWebApp_war.war / opt / jboss / wildfly / standalone / deployments /

    这会将容器的基本映像设置为jboss / wildfly ，并将WAR工件复制到Wildfly服务器的默认部署目录。

  4. 点击装订线旁边的装订线中的绿色箭头FROM命令，然后单击在Docker上运行 。

    这将使用默认设置启动运行配置。该应用程序将部署在容器中运行的Wildfly服务器中，但是您将无法从主机访问该应用程序，因为容器中的端口8080未发布到主机。

  5. 点击装订线旁边的装订线中的绿色箭头FROM命令，然后单击编辑'docker-dir / Dockerfile' 。

  6. 在“ 编辑运行配置”对话框中，为方便起见，您可以选择指定容器的名称（否则，Docker使用默认名称）。

    设置以下端口绑定：

    • 货柜码头： 8080

    • 主机IP： 127.0.0.1

    • 主机端口： 18080

    • 通讯协定： tcp

    每次启动此运行配置时，请在启动前部分中添加任务以构建SimpleJavaWebApp_war工件。

  7. 单击运行以使用新设置重新创建容器。

  8. 容器启动后，在Web浏览器中打开以下地址： http : //127.0.0.1 : 18080/SimpleJavaWebApp_war/ 。
