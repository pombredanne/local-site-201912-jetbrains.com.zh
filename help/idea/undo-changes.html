取消更改

还原未提交的更改

在提交更改之前，您始终可以撤消在本地所做的更改：

  1. 打开 版本控制窗口Alt+9并切换到“ 本地更改”标签。

  2. 在活动的更改列表中，选择一个或多个要还原的文件，然后从上下文菜单中选择“ 回滚 ”，或者按Ctrl+Alt+Z 。自上次提交以来，对选定文件所做的所有更改都将被放弃，并且它们将从 的“本地更改”标签 版本控制窗口Alt+9 。

撤消上一次提交

IntelliJ IDEA允许您撤消当前分支中的最后一次提交（例如HEAD）。

如果将提交推送到受保护的分支，则该撤消不能撤消，即不允许强制使用--push的分支（配置受保护的分支） 在里面 设置/首选项对话框Ctrl+Alt+S在版本控制下| Git ）。

  1. 打开 版本控制窗口Alt+9并切换到“ 存储库”选项卡。

  2. 选择当前分支中的最后一个提交，然后从上下文菜单中选择“ 撤消提交 ”。

  3. 在打开的对话框中，选择一个更改列表，将要放弃的更改将在其中移动。您可以从“ 名称”列表中选择一个现有的变更列表，也可以指定新变更列表的名称（默认情况下使用提交消息）。

  4. 如果要使用即将放弃活动更改列表的更改来制作更改列表，请选择“ 设置活动选项”。

  5. 如果您希望IntelliJ IDEA记住您的上下文并在此更改列表变为活动状态时在编辑器中重新加载当前打开的文件，请选择“ 跟踪上下文”选项。

还原推送的提交

如果发现已经推送的特定提交中有错误，则可以还原该提交。此操作将导致一个新的提交，该提交将撤销您要撤消的提交的效果。因此，由于原始提交保持不变，因此项目历史得以保留。

  1. 在“ 日志”视图中找到要还原的提交，右键单击它，然后从上下文菜单中选择“ 还原提交 ”。从文件“ 历史记录”视图中的提交的上下文菜单中也可以使用此选项。“ 提交更改”对话框将打开，并带有自动生成的提交消息。

  2. 如果选定的提交包含多个文件，而您只需要还原其中一些文件，则取消选择您不想触摸的文件。

  3. 单击“ 提交”以提交一个更改集，该更改集将对此特定提交中的选定文件还原更改。

将分支重置为特定提交

如果您在一组最近的提交中发现错误，并且想要重做该部分，则可以将存储库回滚到特定状态。这可以通过将当前分支HEAD重置为指定的提交来完成（如果您不希望在历史记录中反映撤消操作，则可以选择重置索引和工作树）。

  1. 打开 版本控制窗口Alt+9并切换到“ 存储库”选项卡。

  2. 选择要将HEAD移至的提交，然后从上下文菜单中选择“将当前分支重置为此处 ”。

  3. 在打开的Git Reset对话框中，选择更新工作树和索引的方式，然后单击Reset ：

    • 软 ：将暂存从选定提交之后对提交所做的所有更改（这意味着它们将被移至 的“本地更改”标签 版本控制窗口Alt+9这样您就可以对其进行审核，并在以后提交（如有必要）。

    • 混合 ：将保留选定提交后所做的更改，但不会暂存进行提交。

    • 困难 ：在选定的提交之后进行的所有更改都将被丢弃（暂存和提交）。

    • 保留 ：在选定提交之后所做的提交更改将被丢弃，但本地更改将保持不变。

获取文件的先前版本

如果您需要还原单个文件而不是丢弃包含对多个文件所做更改的整个提交，则可以返回该文件的特定版本：

  1. 在任何视图中选择所需的文件（在项目工具窗口工具窗口中的编辑器中，在 的“本地更改”标签 版本控制窗口Alt+9 ， 等等）。

  2. 选择Git |从VCS主菜单或所选内容的上下文菜单显示历史记录 。历史记录选项卡已添加到 “版本控制”窗口显示所选文件的历史记录，并允许您查看和比较其修订版本。

  3. 确定要回滚的修订后，在列表中选择它，然后从上下文菜单中选择“ 获取 ”。
