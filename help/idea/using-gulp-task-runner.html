IntelliJ IDEA 2019.3帮助

古尔普

IntelliJ IDEA与Gulp.js Task Runner集成。 IntelliJ IDEA解析Gulpfile.js文件，识别任务定义，在树视图中显示任务，让您在树中的任务和Gulpfile.js文件中的定义之间导航，并支持运行和调试任务。

通过启动Gulp.js运行配置 ，可以从专用的Gulp工具窗口中的任务树中运行Gulp.js任务，也可以从Gulpfile.js文件中运行Gulp.js任务，也可以将其作为另一个运行配置中的启动前任务来运行。IntelliJ IDEA在“ 运行工具”窗口中显示执行任务的结果。 工具窗口显示Grunt输出，报告发生的错误，列出未找到的软件包或插件，等等。最后执行的任务的名称显示在工具窗口的标题栏上。

在你开始前

下载并安装Node.js。

安装Gulp.js

要在IntelliJ IDEA项目中使用Gulp，您需要两个软件包：

  • 全局安装的gulp-cli软件包（Gulp命令行界面），用于执行Gulp命令。

  • 一饮而尽包安装为发展依赖建项目任务树，并在编辑Gulpfile.js文件提供编码的援助。从Gulp.js官方网站上了解有关Gulpfile.js的更多信息。

全局安装gulp-cli

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install -global gulp-cli

在项目中安装Gulp.js

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install gulp --save-dev

您还可以按照npm和Yarn中的说明将软件包安装在Node.js和NPM页面上 。

从任务树运行Gulp.js任务

IntelliJ IDEA使您可以直接从Gulp工具窗口中的任务树中轻松快速地运行Gulp.js任务。IntelliJ IDEA自动创建一个临时运行配置，您可以在以后保存并在需要时使用。

Gulp.js一开始就开始构建任务树 通过在“ 项目”工具窗口中的Gulpfile.js或在编辑器中打开的Gulpfile.js的上下文菜单中选择“ 显示Gulp任务”来调用Gulp.js。 该树是根据在其上调用Gulp.js的Gulpfile.js文件构建的。如果项目中有多个Gulpfile.js文件，则可以为每个文件构建一个单独的任务树并运行任务，而无需放弃以前构建的任务树。每个树显示在单独的节点下。

从技术上讲，IntelliJ IDEA 会根据默认的Gulp.js运行配置调用Gulp.js并处理Gulpfile.js 。这是静默完成的，不需要您采取任何步骤。

打开Gulp工具窗口

当您在当前IntelliJ IDEA会话中首次构建任务树时， Gulp工具窗口尚未打开。

  • 在“ 项目”工具窗口中选择所需的Gulpfile.js文件，或在编辑器中将其打开，然后从上下文菜单中选择“ 显示Gulp任务 ”。

默认情况下，IntelliJ IDEA无法在Gulpfile.js中识别ES6，并且无法构建任务树。要解决此问题，请更新默认的Gulp.js运行配置 。

从ES6 Gulpfile.js构建任务树

  1. 从主菜单中，选择运行|编辑配置 。

  2. 在“ 模板”节点下，单击Gulp.js。

  3. 在打开的“运行/调试配置：Gulp.js”对话框中，键入--harmony在“ 节点选项”字段中，然后单击“ 确定” 。

从Gulp工具窗口构建任务树

  • 在“ Gulp工具”窗口中，单击在工具栏上，然后从列表中选择所需的Gulpfile.js文件。默认情况下，IntelliJ IDEA在项目的根目录中显示Gulpfile.js文件。

  • 如果还有另一个Gulpfile.js文件，请单击“选择Gulpfile.js” ，然后在打开的对话框中选择所需的Gulpfile.js文件。IntelliJ IDEA在其标题上添加一个具有所选Gulpfile.js文件路径的新节点，并在新节点下构建任务树。

重建一棵树

  • 切换到所需的节点，然后单击在工具栏上。

按名称对树中的任务进行排序

  • 请点击在工具栏上，从菜单中选择“ 排序依据 ”，然后选择“ 名称” 。
    默认情况下，一棵树以在Gulpfile.js中定义的顺序显示这些任务（选项Definition order ）。

运行任务

  • 双击任务。

  • 在树中选择任务，然后按Enter或选择运行从上下文菜单中。

运行默认任务

  • 在树中选择根节点，然后从上下文菜单中选择“运行默认值 ”。

运行几个任务

  • 使用多选模式：按住Shift （用于相邻项目）或Ctrl （对于非相邻项目）键并选择所需的任务，然后从所选内容的上下文菜单中选择“运行”或“ 调试 ”。

导航到任务的定义

  • 在树中选择所需的任务，然后从所选内容的上下文菜单中选择“ 跳转到源 ”。

从Gulpfile.js运行任务

  1. 将光标置于要运行的任务的定义上，然后选择“运行” 从上下文菜单中。IntelliJ IDEA使用所选任务的名称创建并启动临时运行配置。

  2. 要保存自动创建的临时运行配置，请将光标置于为其创建任务的定义上，然后选择“ 保存”。 从选择的上下文菜单中。

根据运行配置运行和调试任务

除了使用IntelliJ IDEA自动创建的临时运行配置之外，您还可以创建和启动自己的Gulp.js运行配置。

创建一个Gulp.js运行配置

  1. 从主菜单中，选择运行|编辑配置 。

  2. 请点击在工具栏上， 然后从列表中选择Gulp.js。将打开“ 运行/调试配置：Gulp.js”对话框。

  3. 指定运行配置的名称，任务运行（使用空格作为分隔符），其中这些任务定义Gulpfile.js文件的位置和路径，目前该项目的根目录下安装吞气包。

  4. 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

    （可选）指定Node.js的环境变量和用于执行任务的参数。使用格式-- ， 例如： --env development 。从Gulp官方网站了解更多信息。

运行任务

  • 从主工具栏上的列表中选择新创建的运行配置，然后单击在列表旁边。输出显示在“ 运行”工具窗口中 。

调试任务

  1. 创建Gulp.js运行/调试配置如上所述 。

  2. 在编辑器中打开Gulpfile.js文件，并在必要时在其中设置断点。

  3. 要启动调试会话， 从主工具栏上的列表中选择所需的调试配置，然后单击列表旁边或选择运行|除错从主菜单。

  4. 在打开的“ 调试工具”窗口中，分析挂起的任务执行，单步执行任务等，如检查挂起的程序和单步 执行程序中 所述 。

将Gulp任务作为启动前任务运行

  1. 通过选择运行|打开运行/调试配置对话框对话框。在主菜单上， 编辑配置 ，然后从列表中选择所需的配置，或者通过单击来重新创建它并选择相关的运行配置类型。

  2. 在打开的对话框中，单击在启动前区域中，然后从列表中选择运行Gulp任务 。

  3. 在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

自动运行Gulp.js任务

如果您有定期运行的某些任务，则可以将相应的运行配置添加到启动任务列表中。这些任务将在项目启动时自动执行。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，单击“ 工具”下的“ 启动任务 ”。

  2. 在打开的“ 启动任务”页面上，单击在工具栏上。

  3. 从列表中选择所需的Gulp.js运行配置。配置已添加到列表中。

    如果项目中没有适用的配置，请单击然后选择编辑配置 。然后在打开的“运行/调试配置：Gulp.js”页面上定义具有所需设置的配置 。保存新配置后，它将自动添加到启动任务列表中。
