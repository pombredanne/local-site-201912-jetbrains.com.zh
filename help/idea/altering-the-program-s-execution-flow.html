改变程序的执行流程

在调试应用程序时，通常遵循程序的正常流程。但是，在某些情况下，您需要偏离它。为了重现某些条件或测试程序如何处理问题（例如，处理null值或处理异常）。

同样，当您需要跳过程序中与当前正在检查的问题无关的特定部分时，这很方便。

返回上一个堆栈框架

IntelliJ IDEA使您可以回退到程序执行流程中的上一个堆栈帧。例如，如果您错误地走了太远，或者想重新输入错过关键点的方法，这可能会很有用。

  • 单击“ 落帧”图标在“ 调试工具”窗口的工具栏上。

请注意，不会恢复对全局状态所做的更改，仅重置局部变量。

使用断点表达式

为了更改程序的流程，您可以使用非挂起的断点，在命中时对表达式进行求值。例如，当您要在调试过程中自动修改变量时，此功能很有用。

公共类AlteringFlow {私有静态boolean readyToExit = false;私有无效printHello（）{System.out.println（“ Hello”）; //更多的代码在这里readyToExit = true; } public static void main（String [] args）{AlteringFlow af = new AlteringFlow（）; af.printHello（）;如果（！readyToExit）main（null）; //第n1行}}

在示例中，重置readyToExit至false将使printHello反复执行该方法，直到我们禁用将其重置的断点为止。

同样，可以使用断点表达式来添加一些您只想在调试期间使用的逻辑。

公共类AlteringFlow {私有静态布尔addDebugLogic;公共静态无效main（String [] args）{System.out.println（“ Hello”）;如果（addDebugLogic）new Debug（）。doSomething（）; }}类Debug {void doSomething（）{//代码在此处}}

  1. 在要修改程序状态的行上添加一个断点 。

  2. 转到断点属性 。

  3. 在新断点的属性中，清除“ 挂起”复选框，然后在“ 评估和日志”字段中输入将修改程序状态的表达式。

从当前方法强制返回

您可以在方法到达之前退出return语句并使其返回任意值。当问题与方法的返回值有关，而不与方法的产生方式有关时，这很有用。强制返回可帮助您测试程序如何处理返回值，而不必重现导致这些值的条件。

使用Force return时 ，将忽略当前执行点之后的指令。

  1. 确保在“ 框架”选项卡上选择了当前方法，然后右键单击选项卡中的任何位置，然后选择“ 强制返回” 。

  2. 如果该方法返回一个值（其返回类型不是void ），请指定该值或将对其进行计算的表达式。

    表达式是在局部上下文中求值的，可以使用已经声明的任何局部变量。返回值必须符合方法的返回类型。

  3. 如果执行点当前在try ， catch ， 要么finally块中有代码行finally尚未执行的操作，请选择是否要跳过它们。

引发异常

IntelliJ IDEA使您可以从当前执行的方法中引发异常或错误。当您想测试程序中如何处理特定类型的异常而不必重现原因或修改代码时，此功能很有用。

  1. 确保在“ 框架”选项卡上选择了当前方法，然后右键单击选项卡中的任何位置，然后选择“ 抛出异常” 。

  2. 创建例外（可以是任何Throwable包含Error并检查该方法未处理的异常）。请勿使用throw关键词。

重新加载修改后的类

有时，当您对代码进行细微更改时，您希望立即查看它们在正在运行的应用程序中的行为，而无需关闭该过程。HotSwap机制使您可以重新加载在调试会话期间更改的类，而不必重新启动整个应用程序。

重新加载单个文件

  • 右键单击已修改文件的编辑器选项卡，然后选择“ 编译并重新加载文件” 。

您还可以使用“ 设置/首选项” |“ 编译后重新加载类”选项，配置重新编译后的类的自动重新加载。 构建，执行，部署|调试器热插拔 。可以启用/禁用自动重新加载，也可以对其进行配置，以便调试器在每种特定情况下询问您是否重新加载文件。

重新加载所有文件

  • 从主菜单中，选择运行|重新加载更改的类 。

如果在“设置” /“首选项” |“首选项”中启用了“ 在重新加载类之前生成项目”选项，则自动进行重新编译。构建，执行，部署|调试器热插拔 。如果禁用此选项，则需要在重新加载之前重新编译文件（“ 构建” |“重新编译”）。 Ctrl+Shift+F9 ）。

热插拔限制

由于VM的设计，HotSwap具有以下限制：

  • 仅在修改方法主体时才可用。不支持更改签名。

  • 不支持添加和删除类成员

  • 如果修改后的方法已经在调用堆栈中，则只有在程序退出修改后的方法之后，更改才会生效。在那一刻之前，方法主体保持不变，并且框架标记为过时 。

如果要消除标准VM所施加的限制，则可以使用Dynamic Code Evolution VM，它不受限制地支持在运行时重新加载类。
