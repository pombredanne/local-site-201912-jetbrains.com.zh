节点解释器

在这个领域里， 指定当前项目的默认Node.js解释器。IntelliJ IDEA每次您选择Project创建运行/调试配置或配置依赖于Node.js的工具（例如Prettier或ESLint）时， Node Interpreter列表中的别名。

从列表中选择配置的解释器，或单击然后在打开的对话框中配置一个新对话框，如配置本地Node.js解释器中所述 。如果选择node ，则使用系统Node.js版本 。

在这里，您只能选择或配置本地Node.js解释器，即计算机上安装的Node.js或Windows Subsystem for Linux上的Node.js ，请参阅配置本地Node.js解释器和使用Node.js。有关详细信息，请参见Windows Subsystem for Linux 。

可在“ 运行/调试配置：Node.js”中访问“ 配置Node.js远程解释器”对话框中配置远程解释器 。有关详细信息，请参见使用Docker的 Node.js，通过SFTP的 Node.js，通过SSH的 Node.js以及通过Vagrant的Node.js以获得详细信息。

Node.js的编码帮助

选中此复选框将Node.js Core模块源配置为JavaScript库并将其与您的项目关联。结果，IntelliJ IDEA为以下人员提供了代码完成，参考解析，验证和调试功能： fs ， path ， http ，以及其他已编译为Node.js二进制文件的Node.js部分。

配置完成后，IntelliJ IDEA 显示有关当前配置版本的信息。

如果仅在项目的某些部分中需要Node.js API的代码完成，则可以使用“ 管理范围”链接进行配置。在打开的“ 用法”对话框中，单击相关目录，然后从列表中为每个目录选择配置的Node.js Core库。从配置库的范围中了解更多信息。

包装经理

在这个领域里， 选择当前项目的程序包管理器（npm或Yarn）。

IntelliJ IDEA使用npm和yarn这些管理器的当前系统路径的别名。要使用自定义安装，请单击“ 选择” ，然后选择相关软件包管理器的安装文件夹，有关详细信息，请参阅为项目配置软件包管理器 。

默认情况下，IntelliJ IDEA建议使用npm。但是，如果打开带有yarn.lock文件的项目，并且计算机上安装了Yarn，则IntelliJ IDEA会自动将此项目的程序包管理器更改为Yarn。您还可以将Yarn设置为默认的IntelliJ IDEA软件包管理器 。

配套

软件包区域显示计算机上当前安装的所有NPM相关软件包的列表。

  • 软件包：此只读字段显示软件包的名称，与在命令行模式下安装软件包时应引用的名称完全相同。

  • 版本：此只读字段显示计算机上安装的软件包的版本。

  • 最新：此只读字段显示软件包的最新发行版本。如果软件包不是最新的，则标记为 。
