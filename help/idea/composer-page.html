该页面仅在启用PHP和命令行工具支持插件时可用。 这些插件未与IntelliJ IDEA捆绑在一起，但可以如管理插件中所述从JetBrains插件存储库安装。启用后，这些插件将在IDE级别可用，也就是说，您可以在所有IntelliJ IDEA项目中使用它们。

在此页面上，通过专用用户界面启用Composer Dependency Manager命令的执行，并为IntelliJ IDEA项目指定默认的composer.json 。

描述

composer.json的路径

在此字段中，指定要用作default的composer.json的位置。从工具|调用所有Composer命令。将根据该配置文件中的设置执行主菜单中的Composer 。

一个IntelliJ IDEA项目中可以有多个composer.json文件。对于每个composer.json ，从编辑器或项目视图中的上下文菜单中调用操作，或使用编辑器面板顶部的快捷链接来调用操作。

将软件包添加为库

使用此复选框可以配置Composer程序包的打开状态。为了防止供应商/ * / *下的软件包被编辑，请保持选中复选框（这是默认设置）。

如果要在vendor / * / *下编辑Composer软件包，请清除该复选框。

将IDE设置与composer.json同步

选中此复选框可以根据composer.json的配置自动检测PHP语言级别并配置项目Source和Test根目录。

IntelliJ IDEA知道PSR-0 / PSR-4源根及其名称空间前缀。设置源根基于autoload部分和“测试”根目录基于autoload-dev部分。IntelliJ IDEA还基于以下内容检测PHP语言级别： php设置在require部分。

选中此复选框可以根据composer.json的配置自动检测PHP语言级别并配置项目Source和Test根目录。

由于composer.json包含有关项目配置的最新信息，因此这种自动同步可确保Source和Test文件夹与项目结构完全匹配，并自动设置正确的PHP语言级别。

选中此复选框可以根据composer.json的配置自动检测PHP语言级别并配置项目Source和Test根目录。

要了解有关PSR和自动加载的更多信息，请参考Composer官方网站 。有关同步设置的示例和详细信息，请参阅PhpStorm博客文章 。

启用与composer.json的设置同步后，无法在PHP页面上手动设置PHP语言级别的 设置/首选项对话框Ctrl+Alt+S 。

执行

在这方面选择如何执行Composer命令：

  • 要通过全局安装的composer可执行文件运行命令，请选择“ composer”可执行文件并在字段中指定其位置。

  • 要通过本地composer.phar存档运行命令，请选择composer.phar 。然后，在字段中指定现有composer.phar文件的位置，或选中从getcomposer.org下载composer.phar复选框以下载文件的新实例。 composer.phar文件将保存在“ 位置”字段中指定的项目根文件夹下。

    然后，从PHP解释器列表中选择已配置的本地PHP解释器之一 。有关详细信息，请参考配置本地PHP解释器 。

  • 要通过远程解释器运行命令，请选择远程解释器 。

    • 从CLI解释器列表中，选择要使用的远程解释器，或单击并按照配置远程PHP解释器中的说明配置一个新的。

      在大多数情况下，IntelliJ IDEA会自动检测路径映射。如有必要，可以单击在“ 路径映射”字段中，然后在打开的“ 编辑项目路径映射”对话框中手动指定它们。

    • 在Composer可执行文件字段中，提供Composer可执行文件的名称。您可以点击 ，然后在打开的“ 在服务器上选择路径”对话框中找到可执行文件。

    如果您依赖Docker，则可以使用composer Official Docker镜像，该镜像已在IntelliJ IDEA中预先配置为远程解释器。该图像不适用于生产用途。如有必要，您可以按照“管理映像”中的说明在IntelliJ IDEA中创建和管理自己的Docker 映像 。
