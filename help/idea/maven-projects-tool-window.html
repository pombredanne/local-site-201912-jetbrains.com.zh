Maven工具窗口

查看|工具窗口|马文

当您打开Maven项目时，将自动启用Maven工具窗口。

使用Maven工具窗口可查看可用的Maven项目，下载源代码和Javadocs，并执行构建生命周期的各个阶段。工具窗口显示每个Maven项目的节点，以及Lifecycle ， Plugins子节点。如果pom.xml文件中至少有一个包含配置文件定义，则“ 配置文件”节点将添加到工具窗口。如果将依赖项添加到项目中，则Maven工具窗口还将显示“ 依赖项”节点。

项目

描述

单击此按钮将所有Maven项目与IntelliJ IDEA项目同步。请参阅“ Maven集成”对话框的“ 导入”选项卡 。

单击此按钮以启动Maven目标，以为源目录和测试目录生成源和资源，并读取生成的目录结构。根据这种生成的结果，将IntelliJ IDEA文件夹正确标记为源或测试根目录。

请参阅导入设置 。

您可以在“ Maven集成”对话框的“ 导入”页面上设置源和文档的自动下载。

单击此按钮添加一个Maven项目。在打开的对话框中选择所需的pom.xml文件。

单击此按钮以执行构建生命周期或插件目标的选定阶段。如果选择了多个目标，则将按照与树中相同的顺序执行它们。请注意，默认情况下此按钮是禁用的，要激活它，您需要选择一个构建阶段或一个插件目标来运行。

单击此按钮以使用“ 运行任何东西”窗口执行一个Maven目标。

单击此按钮切换离线模式 。

单击此按钮可以打开Maven选项“ 跳过测试模式” ，并忽略正在运行的单元测试。

单击此按钮可以在UML框架中显示当前Maven项目的依赖关系。

单击此按钮以折叠所选Maven项目下的所有节点。

单击此按钮可在“ Maven设置”对话框中配置当前Maven项目的设置。

单击此按钮以显示显示选项的菜单：

  • 组模块 ：选择此选项可按目录对节点进行分组。

  • 显示忽略的项目 ：选择此选项dislplay的忽略项目的工具窗口。

    在Maven工具窗口中，被忽略的项目显示为灰色。

  • 仅显示基本阶段 ：选择此选项时，IntelliJ IDEA仅显示基本构建阶段；仅显示基本阶段。否则，将显示阶段的完整列表。

  • Always Show ArtifactId ：选择此选项以显示在Maven项目的pom.xml中指定的artifactId。

  • 显示版本 ：选择此选项时，IntelliJ IDEA将显示pom.xml中指定的Maven项目的版本。

  • 显示工具栏 ：选择此选项以显示Maven项目的工具栏。

  • 查看模式

  • 组选项卡 ：如果工具窗口中有多个视图，则取消选择此选项可在单独的选项卡上查看视图。

  • 移至 ：选择此选项可将Maven项目工具窗口移至顶部，左侧或右侧。

  • 调整大小 ：选择此选项可调整Maven项目工具窗口的大小。

生命周期阶段的上下文菜单命令

命令

描述

创建[阶段]

选择此命令可为生命周期的选定阶段创建运行/调试配置 。

运行[阶段]

选择此命令可以使用特定于阶段的运行/调试配置来运行生命周期的选定阶段。

调试[阶段]

选择此命令可以使用特定于阶段的运行/调试配置来调试生命周期的选定阶段 。

生成前执行

使用此命令为所选目标设置触发器 。使用此触发器时，在运行项目的构建（ Build | Build Project ）之前，将执行一个目标。“ 生成之前”注释出现在节点名称旁边。

生成后执行

使用此命令为所选目标设置触发器 。使用此触发器时，将在运行项目的构建（ Build | Build Project ）后执行目标。“ 生成后”注释出现在节点名称旁边。

重建前执行

使用此命令为所选目标设置触发器 。使用此触发器时，将在运行重建操作（ Build | Rebuild Project ）之前执行目标。“ 重建之前”注释出现在节点名称旁边。

重建后执行

使用此命令为所选目标设置触发器 。使用此触发器时，在运行重建操作（ Build | Rebuild Project ）后将执行目标。“ 重建后”注释出现在节点名称旁边。

在运行/调试之前执行

选择此命令以指定运行/调试配置 ，在此之前应执行生命周期的选定阶段。运行前注释出现在节点名称旁边。

分配快捷方式

选择此命令可以将选定的阶段与键盘快捷键相关联 。带有快捷方式的注释出现在节点名称旁边。

Maven项目的上下文菜单命令

命令

描述

重新导入

选择此选项以重新导入您的Maven项目。调用此操作后，IntelliJ IDEA会在Maven工具窗口中解析项目结构。

忽略项目/忽略项目

选择“ 忽略项目”命令以忽略构建中的项目 。或屏蔽项目在构建包括以前被忽视的项目。

在这种情况下，IntelliJ IDEA将忽略的Maven项目和模块保留在Maven工具窗口中（它们变灰），但停止将其导入（模块，内容根，目标等）到项目中。但是，IntelliJ IDEA会将被忽略的项目与当前项目同步。例如，如果您需要跳过不相关的子项目，这可能会有所帮助。

编辑忽略的文件

如果您使用Maven设置中的“ 路径模式”字段输入要从项目构建中排除的文件，则此选项可用。选择此选项，然后在“ 忽略的文件”对话框中添加，更改或删除模式。

取消链接Maven项目

选择此选项可从Maven工具窗口以及“ 项目”工具窗口 分离Maven项目及其模块。

创建/打开“ settings.xml”
创建/打开“ profiles.xml”

选择此选项以创建settings.xml或profile.xml ，或者如果已创建该文件，则将其打开。

下载源或/和文档

选择以下选项之一以下载缺少的源或/和Javadocs。

显示有效的POM

选择此选项可生成有效的POM作为该版本的XML，并包含活动配置文件和超级POM。有效的POM显示以下信息：

  • 默认项目源文件夹结构

  • 输出目录

  • 需要插件

  • 仓库

  • Maven在执行所需目标时将使用的报告目录

显示依赖项/显示依赖项弹出窗口

选择以下选项之一以在UML框架中生成当前Maven项目的依赖关系图

跳转到源

选择此选项可在编辑器中打开所选Maven项目的pom.xml文件。
