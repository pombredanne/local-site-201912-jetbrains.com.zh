准备开发Web服务

要在IntelliJ IDEA中开发Web服务，必须对相关模块进行相关配置，并提供所有必需的库和servlet引用。本节介绍满足这些要求的最常用方法。

确保已启用Web和JavaEE：WebServices（JAX-WS）捆绑插件。 这些插件默认情况下处于激活状态。如果禁用了插件 ，请按照“ 管理插件”中所述在“ 插件”页面上启用它们。

通过专用方面启用Web服务开发支持

如果要开发GlassFish / JAXWS2类型的Web服务，则可以使用此方法。X RI / Netro 1。X / JWSDP2.0或Apache Axis 。

Web服务构面只能作为Web构面的子级添加。请注意，模块中仅允许一个Web服务构面。

您可以使用专用构面创建新模块，也可以将构面添加到现有模块中。无论哪种情况，IntelliJ IDEA都会下载预定义的WS引擎实现版本。

要创建具有Web服务构面的模块：

  1. 请执行以下任一操作：

    • 如果要创建新项目，请：在“ 欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件” |“ 新建” 。 新品|项目 。

      结果，“ 新建项目”向导打开。

    • 如果要向现有项目添加模块，请执行以下操作：打开要向其中添加模块的项目，然后选择“ 文件” |“添加”。新品|模块 。

      结果，“ 新建模块”向导将打开。

  2. 在向导的第一页的左侧窗格中，选择Java 。在页面的右侧，指定要使用的JDK 。

  3. 在“ 其他库和框架”下 ，选中“ Web应用程序”复选框。

    从“ 版本”列表中选择要支持的Servlet规范的版本 。

    如果要创建部署描述符web.xml文件，请选中“ 创建web.xml”复选框。

  4. 选择WebServices复选框。

  5. 从版本列表中选择所需的WS引擎实现。

  6. 您将需要一个实现所选WS引擎的库。您可以选择使用现有的库，创建和使用新的库，下载库文件（如果计算机上尚不可用）或将库的设置推迟到以后。

    使用图书馆

    选择要使用的库（如果在IntelliJ IDEA中已经定义了相应的库）。

    创建

    如果您的计算机上已经有相应的jar文件，则可以将这些文件放在库中。单击创建，然后在打开的对话框中选择所需的文件。

    （可选）单击“ 配置”以编辑选定的库。

    下载

    选择此选项以下载必要的库文件。

    （可选）单击“ 配置”以编辑库设置和内容。

    稍后设置图书馆

    选择此选项可以将库的设置推迟到以后。

    单击下一步 。

  7. 指定名称和位置设置。

    点击完成 。

要将Web服务构面添加到现有模块中，请执行以下操作：

    将必要的库添加到现有模块

    如果先前已经下载了必需的库，则可以使用此方法。

    1. 打开模块设置。

    2. 选择所需的模块后，单击“ 依赖关系”选项卡。

    3. 要启用Web开发，请单击“ 添加”按钮，然后从上下文菜单中选择“ 单项模块库 ”。在打开的对话框中 ，选择javaee.jar库，然后单击“ 确定” 。

    4. 再次单击“ 添加”按钮，然后从上下文菜单中选择“ 单项模块库 ”。在打开的对话框中 ，选择javaee.jar库和所需的特定于Web服务的库。

      • javaee.jar库的位置是在IntelliJ IDEA安装期间定义的。

      • Web服务特定库的位置是在其下载期间定义的。

    5. 准备好后，单击“ 确定” 。

    启用额外的WS引擎

    除了大多数常见的WS引擎外，还可以通过专用方面来支持该引擎，借助IntelliJ IDEA，您还可以使用以下引擎：

    • Apache Axis2

    • XFire 1。X / CXF

    • JBossWS
    • WebSphere 6。X

    此外，您可以使用任何所需的WS引擎实现版本，而不必局限于预定义的版本。

    要使用定制的WS引擎：

    1. 下载所需的WS引擎实现。

    2. 在里面 设置/首选项对话框Ctrl+Alt+S ，单击工具|在左窗格中的Web服务 ，并指定到外部Web服务引擎的路径，服务器名称和端口等（有关详细信息，请参阅Web服务 ）。
