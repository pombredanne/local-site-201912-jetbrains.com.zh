调试第一个Java应用程序

您已经创建并运行 Java应用程序。想象一下，您发现它的功能与预期的不同。例如，它返回错误的值或崩溃并带有异常。似乎您的代码中有错误，是时候调试它了。

什么是调试？

广泛来说，调试是检测和纠正程序错误的过程。

有不同类型的错误，您将要处理。其中一些很容易捕获，例如语法错误，因为它们由编译器处理。另一个简单的情况是，通过查看堆栈跟踪可以快速识别错误，这可以帮助您确定错误发生的位置。

但是，有些错误可能非常棘手，并且需要很长时间才能找到并修复。例如，在程序的早期发生的细微逻辑错误可能直到很晚才显现出来，有时将其整理出来确实是一个挑战。

这是调试器有用的地方。调试器是功能强大的工具，通过深入了解程序的内部操作，您可以更快地发现错误。通过暂停执行并通过彻底检查变量以及如何逐行更改变量来分析程序的状态，可以实现此目的。在调试时，您完全可以控制一切。在本手册中，我们介绍了一种基本的调试方案以帮助您入门。

该程序应该计算作为命令行参数传递的所有值的平均值。

它可以编译和运行而不会出现问题，但是结果却不是人们所期望的。例如，当我们通过1 2 3作为输入，结果是6.0 。

首先，您需要考虑可疑错误可能来自何处。

我们可以假设问题不在打印语句中。最有可能的意外结果来自我们findAverage方法。为了找到原因，让我们在运行时检查其行为。

设置断点

断点指示将暂停该程序的代码行，以供您检查其状态。

  • 单击装订线findAverage方法被调用。

在调试模式下运行程序

现在让我们以调试模式启动程序。

由于我们将传递用于运行和调试程序的参数，因此请确保运行/调试配置中具有这些参数。

  1. 从主菜单中，选择运行|编辑配置 。

  2. 在“ 程序参数”字段中输入参数。

  3. 点击旁边的运行按钮main方法或其包含的类。从菜单中选择Debug 。

分析程序状态

调试器会话开始后，程序将正常运行，直到遇到断点为止。发生这种情况时，程序暂停的行将突出显示，并显示“ 调试工具”窗口。

高亮显示的行尚未执行。该程序现在等待您的进一步指示。挂起状态使您可以检查保存程序状态的变量。

逐步执行程序

既然我们对“ 调试”工具窗口感到满意，现在该进入 findAverage方法并找出其中发生了什么。

  1. 要进入方法，请单击“ 进入”按钮或按F7 。

  2. 让我们继续前进，看看局部变量如何result声明以及如何在循环的每次迭代中更改它。

    现在变量s包含值"3" 。它将被转换为整数并添加到result ，其当前值为3.0 。到目前为止没有错误。总和计算正确。

  3. 还有两个步骤将我们带到return语句，我们看到了遗漏的位置。我们忘记了将总和除以值的数量。这是错误返回方法的原因。

  4. 让我们更正错误。

    return result/input.length;

停止并重新运行调试器会话

为了检查程序是否正常运行，让我们停止调试器会话并重新运行程序。

  1. 点击Stop按钮或按Ctrl+F2 。

  2. 点击旁边的运行按钮main方法或其包含的类。从菜单中选择运行 。

  3. 验证程序现在可以正常运行。
