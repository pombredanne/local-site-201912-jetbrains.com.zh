解决常见的Scala问题

检查以下有关最常见的scala / sbt问题的答案，这些问题可以帮助您解决在处理项目时可能遇到的编译器，性能或代码突出显示问题。

为什么我的代码未正确突出显示？

如果类型识别突出显示突出显示正确的代码，请尝试在本地禁用突出显示。

类型意识突出显示默认情况下在项目中启用。要禁用它，请按Ctrl+Shift+Alt+E 。如果禁用赫克托 ，则类型识别突出显示也将被禁用。使用相同的快捷方式启用突出显示。

  1. 突出显示代码，然后在主菜单上选择代码|。环绕 。

  2. 从列表中选择/ * _ * /.../ * _ * /选项。


    该错误不再突出显示。

使用“生成项目”命令时，为什么会出现断言错误？

将Scala版本更改为2.12及更高版本。我们还建议您为值类中的私有方法使用唯一的名称。

为什么我无法编译我的Scala项目？

在大多数情况下，您的项目JDK与Scala或sbt版本不兼容，请尝试更改Scala Compile Server SDK。

  1. 按Ctrl+Alt+S打开设置/首选项对话框。

  2. 从左侧的选项中，选择“ 构建”，“执行”，“部署” |“部署”。编译器Scala | Scala编译器服务器 。

  3. 在JVM SDK字段中，指定适当的SDK。

    我们建议您切换到Java 8 SDK，因为Scala 2.12版和sbt 1.10版及更高版本与较旧的Java SDK版本不兼容。

编译Scala项目时如何解决OutOfMemory异常？

修理Out Of Memory Scala编译器出现问题，请在“ 文件” |“文件” | “ Scala编译器服务器设置”的“ JVM最大堆大小”字段中增加堆大小。 设置/首选项|语言和框架 。

如何将基于IDEA的Scala项目迁移到sbt项目？

您需要将build.sbt文件添加到项目中，在build.sbt中描述整个项目，然后重新导入该项目。
我们建议您从一开始就选择基于sbt的项目。

IntelliJ IDEA是否支持sbt的集成测试模型？

在构建sbt项目时，请记住，IDEA项目模型实际上与sbt具有不同的配置概念，因此仅完全支持编译和测试。对于其他情况，我们建议您从集成的sbt shell运行构建。

我该如何解决sbt项目中的OutOfMemory异常？

如果您的sbt进程没有足够的内存，请在sbt设置中增加最大堆大小（单击位于sbt项目工具窗口中）。
