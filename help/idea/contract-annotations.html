@合同

的@Contract批注用于定义方法必须满足的合同。这样，IDE可以在调用已注释方法的方法中发现问题。您不仅可以使用此注释来注释自己的代码，还可以将其用于其他现有库。

的@Contract注释具有两个属性- value和pure 。的value属性包含描述参数与返回值之间因果关系的子句。

的pure该属性适用于不更改其对象状态而仅返回新值的方法。如果未使用其返回值，则除非方法调用引发异常（不将异常视为副作用），否则删除其调用不会影响程序状态或更改语义。

如果方法本身不会产生副作用，则不应将其标记为纯方法，但可以将其用于建立另一个线程中的事件之间的事前发生关系，以便在另一个线程中执行的更改可能在当前可见调用此方法后的线程。另一方面，某些同步方法可以标记为纯方法，因为此处的同步目的是保持集合的内部完整性，而不是等待另一个线程中的事件。允许不影响重要程序语义的“不可见”副作用（例如日志记录）。

合同是一组描述输入和输出的条款。它们与->符号： "A -> B" 。这形成了合同，这意味着当您向方法提供A时，您将始终获得B。合同中的条款必须与; （分号）符号。例如：

@Contract("_, null -> null")

方法返回null如果第二个参数是null 。

@Contract("_, null -> null; _, !null -> !null")

方法返回null如果第二个参数是null ，否则不为null。

@Contract("true -> fail")

典型的assertFalse()抛出异常的方法true被传递给它。

@Contract("_ -> this")

该方法始终返回其限定符（例如StringBuilder.append ）。

@Contract("null -> fail; _ -> param1")

如果第一个参数为null，则该方法将引发异常，否则它将返回第一个参数（例如， Objects.requireNonNull ）。

@Contract("!null, _ -> param1; null, !null -> param2; null, null -> fail")

该方法返回第一个非null参数，或者如果两个参数均为null，则抛出异常（例如， Objects.requireNonNullElse在Java 9中）。

定义合同

  1. 按Alt+Enter在方法上，然后选择添加方法合同或编辑方法合同 。

  2. 配置合同并应用更改。

句法

的@Contract批注值具有以下语法：

contract ::= (clause ‘;’)* clause
clause ::= args ‘->’ effect
args ::= ((arg ‘,’)* arg )?
arg ::= value-constraint
value-constraint ::= ‘_’ | ‘null’ | ‘!null’ | ‘false’ | ‘true’
effect ::= ‘_’ | ‘null’ | ‘!null’ | ‘false’ | ‘true’ | ‘fail’ | ‘new’ | ‘this’ | ‘param’ number
number ::= [1-9] [0-9]*

约束是：

_

任何值

null

空值

!null

静态证明值不为空

true

真实布尔值

false

错误的布尔值

fail

如果参数满足参数约束，则该方法将引发异常

new

每次执行该方法时，它都会返回一个非空的新对象，该对象不同于方法执行之前堆中存在的其他对象。如果该方法是纯方法，则新对象不会存储在任何字段或数组中，如果不使用方法返回值，则它将丢失。

this

该方法返回非空this参考

param1 (param2, param3, and so on)

该方法返回其第一个（第二个，第三个等）参数
