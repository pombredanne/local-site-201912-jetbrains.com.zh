运行/调试配置：Mocha

运行编辑配置| |摩卡咖啡

该对话框仅在“ 插件”页面上启用NodeJS捆绑插件时可用，有关详细信息，请参阅管理插件 。

在开始之前，请确保您的计算机上装有Node.js。

在此对话框中，使用Mocha测试框架创建用于运行和调试JavaScript单元测试的配置。

在你开始前

  1. 下载并安装Node.js。

  2. 如管理插件中所述，在“ 插件”页面上安装并启用NodeJS存储库插件。

  3. 安装摩卡测试框架中描述摩卡 。

Mocha特定的配置设置

项目

描述

Node.js解释器

在这个领域里， 指定要使用的Node.js解释器。选择配置的解释器之一或单击并如配置本地Node.js解释器中所述配置一个新的。如果您选择Project作为别名，IntelliJ IDEA将自动使用Node.js和NPM页面上“ 节点解释器”字段中的项目默认解释器 。

节点选项

在此字段中键入要传递给Node.js可执行文件的特定于Node.js的命令行选项。有关详细信息，请参见节点参数 。

工作目录

在此字段中，指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

环境变量

 在此字段中，为Node.js可执行文件指定环境变量 （如果适用）。 点击浏览 在字段右侧，并在“ 环境变量”对话框中配置变量列表，该对话框将打开：

  • 要定义新变量，请单击并指定变量名称和值。

  • 要放弃变量定义，请在列表中选择它，然后单击 。

  • 准备好后，单击确定 。

变量的定义显示在“ 环境变量”只读字段中，以分号作为分隔符。可接受的变量是：

  • NODE_PATH ： 一种: -前缀为模块搜索路径的目录的列表。

  • NODE_MODULE_CONTEXTS ：设置为1可以在各自的全局上下文中加载模块。

  • NODE_DISABLE_COLORS ：设置为1以禁用REPL中的颜色。

摩卡套餐

在此字段中，指定Mocha安装目录/ npm / node_modules / mocha 。如果您通过Node Package Manager定期安装Mocha ，则IntelliJ IDEA会检测Mocha的安装目录。或者，手动键入可执行文件的路径，或单击“ 浏览”。 并在打开的对话框中选择位置。

用户界面

从此列表中，选择编写测试文件夹中的测试所依据的接口 。IntelliJ IDEA将仅识别符合所选接口的测试。如果在测试执行期间IntelliJ IDEA针对另一个接口的测试运行，则测试会话将被取消并显示错误。这意味着必须根据相同的接口编写指定测试文件夹中的所有测试，并且必须从列表中选择该接口 。

额外的摩卡咖啡选项

在此字段中，指定其他Mocha命令行选项 。例如，添加一个--watch标志以打开类似autotest的Runner 。

结果，当前运行配置中的任何测试都会在更改相关源代码时自动重新启动，而无需单击“ 重新运行”按钮 。

测验

在该区域中，指定要执行的测试。可用的选项有：

  • 全部在目录中：选择此选项可从存储在文件夹中的文件中运行所有测试，并在“ 测试目录”字段中指定此文件夹的路径。要在指定目录下的子文件夹中查找测试，请选中“ 包括子目录”复选框。

  • 文件模式：选择此选项可在名称与某个掩码匹配的所有文件中查找测试，并在“ 测试文件模式”字段中指定掩码。

  • 测试文件：选择此选项可仅运行一个文件中的测试 ，然后在“ 测试文件”字段中指定此文件的路径。

  • 套件：选择此选项可从测试文件运行单个套件，并在“ 测试文件”字段中指定此文件的路径。单击套件名称字段，然后配置要运行的套件列表。要将套件添加到列表中，请单击并输入所需套件的名称。要删除套件，请在列表中选择它，然后单击

  • 测试：选择此选项可运行测试文件中的各个测试，并在“ 测试文件”字段中指定此文件的路径。单击测试名称字段，然后配置要运行的测试列表。要将测试添加到列表中，请单击并输入所需测试的名称。要删除测试，请在列表中选择它，然后单击

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

如果使用基于目录的项目格式 ，则运行/调试配置的设置存储在.idea \ runConfigurations文件夹（如果共享）中的单独的.xml文件中，或者存储在.idea \ workspace中。 xml文件，否则。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以进行共享配置，否则将存储在.iws文件中。

项目

捷径

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在IntelliJ IDEA中定义它，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 构建工件 ：选择此选项，以建立一个神器或假象。在打开的对话框中，选择应构建的工件。

  • 启动Web浏览器 ：选择此选项以启动浏览器。在打开的对话框中，选择浏览器的类型并提供起始URL。另外，指定是否要使用JavaScript调试器启动浏览器。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行Maven目标 ：选择此选项可运行Maven目标 。在打开的对话框中，选择要运行的目标。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
