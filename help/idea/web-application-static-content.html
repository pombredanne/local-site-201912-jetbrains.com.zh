配置静态内容资源

配置静态Web资源涉及：

  1. 指定服务器上静态Web内容的位置 。

    部署后，实现用户界面的所有静态内容资源应位于Web应用程序根目录下。默认情况下，IntelliJ IDEA将目标Web应用程序根目录映射到启用Web开发支持后创建的Web文件夹。默认的应用程序主页映射到index.jsp存根文件，该文件也会自动创建。

    您可以通过两种方式在模块中安排静态内容资源：

    • 将所需的资源和目录存储在\ web文件夹下。部署后，这些资源的整个文件夹层次结构将被复制到应用程序根目录下的服务器。

    • 将所需的资源本地存储在任何可能的位置，然后将它们映射到服务器上的文件夹 。

  2. 在要部署的工件中包括静态Web内容 。

配置静态Web内容资源

  1. 打开项目结构对话框（例如Ctrl+Shift+Alt+S ）。

  2. 单击左侧窗格上的模块 ，然后展开所需的模块。

  3. 单击Web构面。对话框的右侧窗格显示所有可用的设置。

  4. 在“ Web资源目录”区域中，管理具有所需资源的本地文件夹与将资源部署到的服务器上的目标目录之间的映射列表。\ web文件夹和应用程序根目录之间的默认映射已在列表中：

    • 要配置映射，请单击“ 新建”按钮。在打开的“ Web资源目录路径”对话框中，指定所需的本地文件夹和相对于应用程序根文件夹的目标位置。默认情况下，IntelliJ IDEA在部署后将\ web文件夹映射到应用程序的根文件夹。例如，如果您输入正斜杠/ ，Web资源目录中的文件将被复制到部署根目录中。

    • 要编辑映射，请在列表中选择它，单击“ 编辑”按钮，然后在打开的“ Web资源目录路径”对话框中更新映射。

    • 要丢弃映射，请在列表中选择并单击“ 删除”按钮。
