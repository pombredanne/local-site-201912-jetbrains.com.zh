除了常见的Web服务支持之外 ，IntelliJ IDEA还提供以下工具来开发RESTful Web服务：

  • 代码检查和快速修复

代码检查和快速修复

IntelliJ IDEA支持代码检查，并在以下情况下建议快速修复 ：

问题

默认快速修复

方法注释与方法返回类型之间的不一致： @GET带注释的方法返回空值。

更改为字符串。

资源方法错误。

去掉@Path注解。

参数类型错误。

验证带注释的参数@QueryParam要么@PathParam注释。

@DefaultValue问题。

@GET public String get（@DefaultValue（“ 33.5”）@QueryParam（“ str”）int str）{返回“你好”； }
DefaultValue被标记为红色，并带有“无法转换为int”的描述。

休息@Path和@PathParam注解检查。

其余参考文献解决了问题。

参考文献@PathParam批注解析为的模板@Path注解

此方法应只有一个HTTP方法指示符。

例：

@GET @POST public String get（）{返回“你好”； }
@POST标记为红色。

WADL配置错误。

代码编辑器中的HTTP客户端

IntelliJ IDEA代码编辑器中的专用HTTP客户端提供以下用于处理HTTP请求的功能：

  • 编写和执行请求

  • 显示服务器响应和已执行的请求历史记录

  • 从中完成 HTTP请求部分@Path和@Produces Java代码中指定的注释
