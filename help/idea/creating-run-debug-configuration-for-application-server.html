使用服务器运行/调试配置

仅Ultimate Edition支持此功能。

要在应用程序服务器上运行或调试Java EE或Web应用程序，您需要一个应用程序服务器运行/调试配置 。

IntelliJ IDEA可以自动创建一种这样的配置。如果在创建项目或模块时指定要使用的应用程序服务器，则会发生这种情况（请参阅在创建项目或模块时定义服务器 ）。

如果需要，您可以创建更多服务器运行/调试配置。

本地和远程运行配置

服务器运行/调试配置可以是本地的也可以是远程的。

本地配置适用于您计算机上安装的服务器。此类配置包括定义如何启动相应服务器的设置。因此，执行本地配置时，除其他外，IntelliJ IDEA将启动服务器。

远程配置不会启动服务器。通常，它们用于在不同（远程）计算机上运行的服务器。如果您不希望运行配置启动（或停止）服务器，它们也可以用于本地（在计算机上）安装的服务器。

当执行远程配置时，IntelliJ IDEA将连接到服务器，以便能够部署应用程序工件或执行其调试。那时，相应的服务器必须已经在运行。

总而言之，本地和远程配置的主要区别不是服务器的物理安装位置（尽管这也很重要），而是服务器是否通过相应的运行配置启动（或停止）。

启动服务器运行配置时会发生什么

启动服务器运行/调试配置时，通常，IntelliJ IDEA将执行以下操作：

  1. 执行启动前任务。默认情况下，这些是Make and Build Artifacts ：IntelliJ IDEA编译项目并构建应用程序工件。Build Artifacts任务最初并不存在于运行配置中，而是在您指定要部署到服务器上的工件后立即自动添加。

  2. 启动服务器（用于本地配置）或连接到服务器（用于远程配置）。（这时，将打开“ 运行”或“ 调试”工具窗口，以便您可以监视和控制该过程。）

  3. 将指定的工件（和/或项目外部的可部署组件）部署到服务器。

  4. 如果指定，则启动Web浏览器并打开指定的URL（通常对应于应用程序的起始页）。

创建服务器运行/调试配置

  1. 打开“ 运行/调试配置”对话框（例如， “运行” |“编辑配置” ）。

  2. 请点击Alt+Insert ，选择感兴趣的服务器（例如Tomcat Server ），如果可用，选择Local或Remote 。（请参阅本地和远程运行配置 。）

  3. 在对话框的右侧，指定运行/调试配置设置，然后单击“ 确定” 。（有关可用设置的信息，请参阅“ 运行/调试配置”对话框中相应的服务器特定主题）。

另请参阅创建和编辑运行/调试配置 。

启动服务器运行/调试配置

可以像其他任何运行配置一样，以通常的方式启动应用程序服务器的运行/调试配置。请参阅运行应用程序和启动调试器会话 。

或者，您可以使用“ 服务”工具窗口。
