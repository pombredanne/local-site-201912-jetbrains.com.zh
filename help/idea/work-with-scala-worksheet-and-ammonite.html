使用Scala工作表和Ammonite

您可以使用Scala工作表或Ammonite快速评估并运行Scala代码。

由于在IntelliJ IDEA中，Scala工作表和Ammonite文件具有相同的文件扩展名.sc ，因此您需要让IntelliJ IDEA知道如何在项目中处理此类文件。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架|斯卡拉（Scala） 。

  2. 在“ 工作表”选项卡上的“ 将.sc文件视为”字段中，默认情况下选中“ 始终工作表”选项。

    选择“ 始终使用亚ite石”以获得亚support石支持。

    您也可以在测试源中选择Ammonite，否则可以选择Worksheet选项以仅将测试文件视为Ammonite。

  3. 保存您的更改。

在设置中，您还可以设置输出截止限制，指定要以哪种模式运行工作表或Ammonite脚本，或者配置希望IntelliJ IDEA处理.sc和草稿文件的方式。

如果选中“ 使用“ eclipse兼容性”模式复选框，则对象内部的所有语句将包含在工作表的输出中。

创建一个.sc文件

  1. 右键单击您的项目，然后选择新建 | Scala工作表 。我们建议您在src目录中创建一个.sc文件，以避免代码突出显示问题。

  2. 在New Scala Worksheet窗口中，输入文件名，然后单击OK 。

    结果，将打开一个扩展名为.sc的文件。根据文件配置 ，它被视为Scala工作表或Ammonite。

使用Scala工作表运行Scala代码

  1. 创建一个Scala工作表文件并在编辑器中打开它。

  2. 添加您的代码。

    除了诸如代码完成和字符串格式之类的标准操作外，IntelliJ IDEA还支持以下操作：

    • 评估Scala对象

    • 折叠输出而不影响左侧的代码，仅扩展与特定语句匹配的输出块。

  3. 请点击看结果。

使用当前工作表设置

  1. 在编辑器中打开一个Scala工作表文件。请点击在工作表编辑器的工具栏上打开“ 工作表设置”对话框。

  2. 在“ 工作表设置”对话框中，为整个项目配置当前工作表设置或默认工作表设置，这将影响其他工作表。

  3. 使用以下选项进行配置：

    • 交互模式 ：选中此复选框可自动显示结果。

    • 生成项目 ：清除此复选框以禁用对项目更改的自动检查，从而提高评估性能。

    • 运行类型 ：从此列表中选择适当的评估模式。例如，选择REPL类型以快速评估Scala表达式。添加新表达式时，它们将以增量方式执行。

    • 使用模块的类路径 ：由于工作表是一个文件，您可以在其他项目之间共享该文件，并且应保留在创建它的位置，因此此字段已禁用。您可以在默认设置中更改其位置，但是它将应用于整个项目，并且会影响其他工作表。

      如果需要更改类路径，建议您创建并使用暂存文件 。

    • 编译器配置文件 ：在此字段中，为您的工作表选择一个编译器配置文件。从列表中选择一个或单击访问Scala编译器设置以获得更多选项。

      如果通过编译器设置页面添加或更改配置文件，并且您有sbt项目 ，则所做的更改将在下一次项目导入时被放弃。

在IntelliJ IDEA中配置Ammonite支持

  • 根据您的项目，执行以下操作：

    • 如果您有sbt项目 ，则将以下代码添加到build.sbt文件中：

      libraryDependencies + = {val版本= scalaBinaryVersion.value匹配{case“ 2.10” =>“ 1.0.3” case _⇒“ 1.6.5”}“ com.lihaoyi”％“ ammonite”％版本％“ test”交叉CrossVersion。测试中的sourceGenerators + = Def.task {val file =（测试中的sourceManaged）.value /“ amm.scala” IO.write（file，“”“对象amm扩展了App {炸药。Main.main（args）}“”“）Seq（file）} .taskValue
      刷新项目以导入更改。

    • 如果您有一个常规的Scala项目，请下载Ammonite并指定其可执行文件的位置。例如，您可以修改$ path变量或在Amm可执行文件字段的默认Ammonite运行/调试配置中指定路径。

使用Ammonite运行Scala脚本

  1. 创建一个Ammonite文件 ，然后在编辑器中将其打开。

  2. 添加您的代码。

    除了标准的代码完成和导入操作外，IntelliJ IDEA还支持以下特定于Ammonite的操作：

    • 特定于Ammonite的命令的代码完成

    • 从其他Ammonite文件引用代码并为其完成代码

  3. 在左侧装订线中，单击然后选择“运行脚本”以在“ 运行工具”窗口中查看结果。

    或者，您可以创建并运行Ammonite运行/调试配置 。
