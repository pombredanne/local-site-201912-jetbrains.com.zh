IntelliJ IDEA 2019.3帮助

将IntelliJ IDEA项目导出到Eclipse

在开始导出项目之前，请确保已启用 Eclipse Integration插件。

当前，从IntelliJ IDEA到Eclipse的迁移受到某些限制：

  • 具有多个内容根目录的IntelliJ IDEA模块无法迁移。

  • Eclipse中的外部源不会迁移。

  • 在导出过程中仅自动创建Java模块。但是，任何模块都可以手动转换为Eclipse兼容格式。

  • 默认工作空间JRE不会与项目SDK相互转换。

将项目导出到Eclipse

当您将IntelliJ IDEA项目导出到Eclipse时， 将为包含内容根目录的模块目录中的每个模块文件.iml创建Eclipse项目文件.project和.classpath 。

  1. 从主菜单中，选择文件|将项目导出到Eclipse 。

    “ 导出到Eclipse”对话框将显示尚未转换并切换为使用Eclipse格式的模块（具有IntelliJ IDEA模块格式.iml的模块）的列表。

  2. 选择要导出的模块。

  3. 如有必要，请选择建议的选项：

    • 将选定的模块切换为Eclipse兼容格式 ：将在“模块”中选择的模块转换为导出为Eclipse兼容格式。

    • 导出非模块库 ：IntelliJ IDEA为Eclipse * .userlibraries创建一个用户库配置文件，其中包含项目中使用的所有外部库的定义。

    • 生成的.userlibraries的路径：生成的* .userlibraries文件的路径。选择“ 导出非模块库”选项时，此字段可用。

  4. 单击确定 。

    该项目将导出到同一目录。“项目”工具窗口显示新创建的Eclipse文件。

将模块转换为Eclipse兼容格式

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S或点击在工具栏上。

  2. 选择要转换的模块，然后切换到“ 依赖关系”选项卡。

  3. 从Dependencies存储格式列表中，选择Eclipse（.classpath） 。

  4. 应用更改并关闭对话框。
