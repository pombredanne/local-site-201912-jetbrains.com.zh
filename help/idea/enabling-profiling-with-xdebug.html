使用Xdebug启用性能分析

仅当安装并启用PHP插件时，以下内容才有效。

Xdebug分析器已包含在Xdebug工具中。因此，您首先需要下载，安装和启用Xdebug本身，然后再在其中启用分析功能。

要使用Xdebug进行性能分析，请执行以下常规步骤：

  • 配置Xdebug
  • 启用Xdebug分析器
  • 配置从浏览器切换探查器的方式
  • 指定用于存储累积分析数据的位置

配置Xdebug

  1. 下载并安装 Xdebug工具。

  2. 将Xdebug与PHP引擎集成 。

  3. 将Xdebug与IntelliJ IDEA集成 。

启用Xdebug分析器

  1. 在编辑器中打开活动的php.ini文件：

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在“ 语言和框架”下单击“ PHP ” 。

    2. 在打开的PHP页面上，单击 CLI解释器字段旁边。

    3. 在打开的“ CLI解释器”对话框中，“ 配置文件”只读字段显示活动php.ini文件的路径。单击“ 在编辑器中打开” 。

  2. 请执行以下任一操作：

    • 要永久启用探查器，请将xdebug.profiler_enable指令设置为1：

      xdebug.profiler_enable = 1;

    • 要使用浏览器从浏览器触发分析器XDEBUG_PROFILE Cookie或GET / POST参数，请将xdebug.profiler_enable指令设置为0，并将xdebug.profiler_enable_trigger指令设置为1：

      xdebug.profiler_enable = 0; xdebug.profiler_enable_trigger = 1;

配置从浏览器切换探查器的方式

要指定XDEBUG_PROFILE cookie或GET / POST参数，请执行以下任一操作：

  • 手动指定值 。

  • 生成书签以切换调试器。 这些书签将显示在浏览器的工具栏上。它们提供了对调试器cookie的控制，通过它们您将激活和停用调试器。

    1. 通过执行以下操作之一（取决于浏览器类型），在浏览器中启用“ 书签”工具栏 ：

      • 在Firefox中 ，选择“ 查看|工具栏书签工具栏 。

      • 在Chrome中 ，选择书签|显示书签栏 。

    2. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |调试 。

    3. 在打开的“ 调试”页面上，单击“ 使用调试器”书签，以从您喜欢的浏览器链接启动调试器 。

    4. 在打开的Zend Debugger＆Xdebug书签页面上，检查调试引擎设置，然后单击Generate 。生成列出的与调试相关的动作的书签。

    5. 将生成的链接拖动到浏览器中的书签工具栏。

指定用于存储累积分析数据的位置

  1. 在编辑器中打开活动的php.ini文件：

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在“ 语言和框架”下单击“ PHP ” 。

    2. 在打开的PHP页面上，单击 CLI解释器字段旁边。

    3. 在打开的“ CLI解释器”对话框中，“ 配置文件”只读字段显示活动php.ini文件的路径。单击“ 在编辑器中打开” 。

  2. 通过指定xdebug.profiler_output_dir指令来定义累积分析快照的位置。

    xdebug.profiler_output_dir = ""

  3. 通过的值指定用于存储快照的文件名。 xdebug .profiler_output_name指示。默认名称为cachegrind.out。％p ，其中%p是名称格式说明符。接受默认名称或定义符合以下标准的自定义名称：

    1. 名称应始终为cachegrind.out 。

    2. 使用支持的格式说明符 。
