PHP_CodeSniffer

在本部分中，选择要使用的PHP_CodeSniffer脚本。

项目

描述

组态

 选择PHP_CodeSniffer脚本：

  • 要使用与特定的远程PHP解释器关联的脚本，请选择该解释器的名称。

  • 要使用本地脚本，请选择Local 。在这种情况下，无论项目中使用哪种PHP解释器（本地或远程），都将执行本地PHP_CodeSniffer。请注意，PHP_CodeSniffer只能有一个本地配置，因为IntelliJ IDEA运行的脚本（对于Windows为phpcs.bat或对于Linux和macOS为phpcs ）包含一个PHP引擎路径的脚本。

  • 要使用与默认项目解释器关联的脚本，即在“设置/首选项”对话框的PHP页面上选择的脚本 ，请选择“默认项目解释器” 。

忽略的文件

单击以打开“ PHP_CodeSniffer忽略的文件”对话框，其中列出了PHP_CodeSniffer跳过的文件。当等待来自PHP_CodeSniffer的响应超出“ 工具进程超时”字段中指定的限制时，IntelliJ IDEA建议在检查期间将新文件添加到列表中。这样做是为了防止减慢处理速度。对于每个文件，IntelliJ IDEA都会显示其名称和位置。

  • 要添加文件，请单击并在打开的对话框中找到所需的文件。

  • 要从列表中删除文件并让PHP_CodeSniffer再次对其进行处理，请选择该文件，然后单击“ 删除文件”按钮 。

  • 要从列表中删除所有文件，请单击 。

PHP_CodeSniffer对话框

单击时将打开对话框 PHP_CodeSniffer页上的“ 配置”列表旁边。

使用此对话框可以配置本地PHP_CodeSniffer脚本或与远程PHP解释器关联的脚本，以及配置PHP_CodeSniffer的行为。有关详细信息，请参见PHP_CodeSniffer 。

描述

PHP_CodeSniffer（phpcs）路径

在这个领域中，指定PHP_CodeSniffer脚本的位置：对于Windows或Linux和MacOS phpcs phpcs.bat。

如果脚本与PHP解释器关联，则IntelliJ IDEA会检测到该脚本的路径并自动填写该字段，但是您可以根据需要进行编辑。手动输入路径或单击然后在打开的对话框中选择路径。

要检查phpcs.bat或phpcs的指定路径是否可以确保IntelliJ IDEA和PHP_CodeSniffer之间的交互，即可以从IntelliJ IDEA启动该工具，并且IntelliJ IDEA会从中接收到问题报告，请单击Validate按钮。此验证等同于运行phpcs --version命令。如果验证成功通过，则IntelliJ IDEA将显示有关检测到的PHP_CodeSniffer版本的信息。

代码质量工具脚本包含对系统范围的PHP引擎的调用。为了能够使用它们，请确保将包含PHP引擎可执行文件的目录添加到系统中path 。

口译员

该字段显示选择的PHP解释器以使用PHP_CodeSniffer。

每个文件的最大邮件数

在此字段中，为文件报告的消息总数设置上限。超过此限制的所有消息将被拒绝。IntelliJ IDEA将在代码中显示以下警告： Too many PHP_CodeSniffer messages并建议将文件添加到“ 忽略的文件”列表中。

工具处理超时

在此字段中，指定您希望IntelliJ IDEA等待PHP_CodeSniffer的结果的时间。如果超过了超时，该进程将被终止以防止过多的CPU和内存使用。这使您可以根据计算机的配置和所使用的规则集来微调PHP_CodeSniffer进程的行为。

PHP代码美化和修复程序设置

在此区域中，在phpcbf的路径字段中提供PHP代码美化工具和修复程序工具的路径 。手动输入路径或单击然后在打开的对话框中选择路径。
