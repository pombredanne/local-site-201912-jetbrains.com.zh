创建一个包

  1. 在“ 项目”工具窗口中，选择您的源根文件夹（例如src ）或要在其中创建新软件包的软件包。选择文件|新品|打包 ，或按Alt+Insert ，然后选择Package 。

    或者，右键单击相应的文件夹或软件包，然后选择“ 新建|从上下文菜单打包 。

  2. 在打开的“ 新软件包”对话框中，指定软件包名称，然后单击“ 确定” 。

    请注意，如果使用点，则可以一次创建多个包.分隔软件包名称。例如，如果您键入myPackage.mySubpackage并且这些软件包目前都不存在，这两个软件包myPackage和mySubpackage将被创建。

    另请注意，在创建新的ActionScript类或接口或MXML组件时，可以创建新的程序包。请参阅创建ActionScript类或接口和创建MXML组件 。

创建一个ActionScript类或接口

  1. 在“ 项目”工具窗口中，选择源根文件夹（例如src ）或要在其中创建新ActionScript类或接口的包。选择文件|新品| ActionScript类 ，或按Alt+Insert然后选择ActionScript Class 。

    或者，右键单击相应的文件夹或软件包，然后选择“ 新建|从上下文菜单中选择ActionScript类 。

  2. 在打开的“ 新建ActionScript类”对话框中，指定类的名称，包，要使用的文件模板 ，然后单击创建 。如果使用带有超类的模板 ，则还可以指定一个超类和/或该类应实现的一个或多个接口。

    请注意，如果您指定了尚不存在的程序包，则会创建相应的程序包。

另请参见在编辑器中启动ActionScript类的创建 。

创建一个MXML组件

  1. 在“ 项目”工具窗口中，选择源根文件夹（例如src ）或要在其中创建新MXML组件的包。选择文件|新品| MXML组件 ，或按Alt+Insert然后选择MXML Component 。

    或者，右键单击相应的文件夹或软件包，然后选择“ 新建|从上下文菜单中选择ActionScript类 。

  2. 在打开的“ 新建MXML组件”对话框中，指定组件的名称，包，要使用的文件模板 ，父组件，然后单击“ 创建” 。

    请注意，如果您指定了尚不存在的程序包，则会创建相应的程序包。

在编辑器中启动ActionScript类的创建

使用类的源代码时，可以直接在编辑器中开始创建其子类。同样，您可以启动创建实现接口的类。为此，IntelliJ IDEA提供了分别称为“创建子类”和“实现接口”的意图动作 。

这是使用创建子类意图操作的示例。（以类似的方式访问“实施接口”操作。）

  1. 在编辑器中，将光标放在包含类声明的行中。

  2. 按Alt+Enter然后选择创建子类 。

  3. 在打开的“ 新建ActionScript类”对话框中，指定新类的设置，然后点击创建 。

用于ActionScript和Flex的预定义文件模板变量

对于ActionScript和Flex文件模板， 预定义模板变量的列表更广泛。此外，还可以使用以下预定义的变量：

  • ${Superclass} -ActionScript类的超类或MXML组件的父组件。

  • ${SuperInterfaces} -ActionScript类实现的接口的列表。

使用SWF元数据标记控制HTML包装器属性

您可以在主应用程序类中使用SWF元数据标记来设置标题，背景色和和 HTML包装器中的属性。

例如，如果ActionScript类包含

打包myPackage {导入flash.display。雪碧; [SWF（pageTitle =“ hello”，backgroundColor =“＃ccddee”，width =“ 400”，height =“ 200”）]公共类MyClass扩展了Sprite {...}}

相应的属性将在HTML包装器中设置。

如果相应的文件包含以下内容，则SWF元数据标签将以类似的方式适用于MXML组件：

[SWF（pageTitle =“ hello”，backgroundColor =“＃ccddee”，width =“ 400”，height =“ 200”）] ...
