移动和复制重构

复制重构使您可以在其他程序包中创建类的副本。它还允许您在其他目录或包中创建文件，目录或包的副本。

执行复制重构

  1. 选择要重构的标识符（例如，编辑器中的类或“ 项目”工具窗口中的文件）。

  2. 选择重构|复制 F5

  3. 在“ 复制”窗口中，指定副本的名称和位置，然后单击“ 确定” 。

您可以通过上下文菜单访问所有重构操作。

要撤消重构，请按Ctrl+Z 。

通过移动重构，您可以将包和类在项目的源根之间移动，将类成员移动到其他类，将内部类移动到更高层次。
例如，如果在其他类中使用的移动次数超过其自己的类，则可以对方法或字段执行移动重构。

执行移动重构

  1. 选择要重构的标识符。

  2. 选择重构|移动 （或按F6 ）。

  3. 在打开的对话框中，根据您为重构选择的项目，指定适当的选项，然后单击“ 重构” （对于程序包，单击“ 确定 ”）。您还可以单击“ 预览” （如果有）以预览可能的更改。

  4. 指定目标目录的路径，文件名和程序包名称。

对软件包执行移动重构时，您需要指定将软件包及其内容（移动到另一个软件包，源根目录或另一个目录）的位置。

移动重构也可以从UML类图中获得 。

将静态方法移到另一个类

  1. 在编辑器中打开您的班级，将尖号放在要移动的静态方法上，然后按F6 。

  2. 在“ 移动成员”对话框中，指定以下选项：

    • 在要移动到另一个类的成员（仅静态）字段中，选中要移动到另一个类的方法旁边的复选框。该列表显示了当前类中检测到的所有静态方法。

    • 在“ 将成员移动到类”字段中，键入您要将方法列表中选定的成员移动到的类的完全限定名称。您可以点击图标以选择或搜索现有课程。

  3. 在实际重构之前，单击“ 重构”以继续进行操作或单击“ 预览”以检查结果。

将实例方法移到另一个类

如果实例（非静态）方法在项目中具有类型参数，则可以将其移动到其他类。在任何其他情况下，您都需要使用此方法static第一。

  1. 在编辑器中，将插入符号放置在实例方法上，然后按F6 （ 重构|移动 ）。

  2. 在打开的“ 移动实例方法”对话框中，指定以下内容：

    • 在“ 选择实例表达式”列表中，选择要将实例方法移至的目标类。潜在移动目标的列表包括当前类中方法参数的类和字段的类。

    • 在“ 可见性”区域中，为目标方法选择首选的可见性修改器。

    • 在参数名称中将“参数引用”替换为类字段中，键入所需的参数名称。执行重构时，将为要移动的方法添加参数，它将替换当前类的所有参数引用。

  3. 在实际重构之前，单击“ 重构”以继续进行操作或单击“ 预览”以检查结果。
