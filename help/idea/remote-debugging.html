远程调试

以下内容仅在安装并启用Python插件后才有效。

IntelliJ IDEA提供了两种远程调试方法：

  • 使用远程解释器
  • 使用Python调试服务器

使用远程解释器进行远程调试

如果使用远程Python解释器执行远程调试，则所有操作都在单个SSH连接中完成，并且不需要端口号。

要使用远程解释器进行远程调试，请遵循以下常规步骤

  1. 确保已配置远程解释器。

  2. 使用常规的运行/调试配置启动调试过程 。

PyPi中提供了Python调试器，因此可以安装该调试器以使用pip进行远程调试。

调试在另一台计算机上运行的进程时，可以pip install pydevd而不是从IntelliJ IDEA安装目录下的debug-eggs目录中复制档案。

使用Python调试服务器进行远程调试

对于远程调试，IntelliJ IDEA提供了带有pydev目录的存档。这些存档文件取决于Python版本：

  • pycharm-debug.egg（适用于Python 2或更高版本）

  • 适用于Python 3的pycharm-debug-py3k.egg

该归档文件位于以下位置，具体取决于平台：

  • Windows：下。用户主目录的IntelliJIDEAXXXX.X \ config \ plugins \ python目录。

  • * NIX：下。用户主目录的IntelliJIDEAXXXX.X / config / plugins / python目录。

  • macOS：在/ Users / jetbrains / Library / Application Support / IntelliJIDEAXXXX.X / python /目录下。

远程调试过程涉及以下常规步骤：

  • 配置远程调试服务器 。

  • 准备调试 。

  • 启动调试服务器 。

  • 从外部启动脚本 。

配置远程调试服务器

  1. 如创建运行/调试配置一节中所述，打开“ 运行/调试配置”对话框，然后选择“ Python远程调试”配置类型。

  2. 指定本地主机名和调试服务器将运行的端口号。如果您未指定任何值，则IntelliJ IDEA将随机提供端口号。

  3. 如果要调试驻留在远程计算机上的脚本，请指定该远程脚本的源根目录以及本地源目录以保持映射。

准备进行远程调试

  1. 在IntelliJ IDEA中，打开本地脚本进行编辑，然后添加以下命令（您可以从远程调试配置对话框中复制它）：

    pydevd.settrace(, port=)

    另外，将相应的import语句添加到脚本中：

    import pydevd

  2. 将本地脚本复制到要在其中调试的远程位置。

  3. 包括pycharm-debug.egg档案。您可以通过多种方式进行操作，例如：

    • 将存档添加到PYTHONPATH 。

    • 将存档附加到sys.path 。

    • 只需将pydev从存档复制到远程脚本所在的目录。

启动调试服务器

如果该过程没有停止，但是您仍想停止跟踪并与Python Remote Debug Server断开连接，请在要调试的远程脚本的末尾添加以下函数：

pydevd.stoptrace（）

此功能将停止使用以下命令启动的远程调试过程pydevd.settrace() 。因此，Python远程调试服务器将进入等待新连接的状态。
