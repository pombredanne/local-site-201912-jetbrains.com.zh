使用PHP网页调试配置进行调试

仅当安装并启用PHP插件时，以下内容才有效。

在这种调试模式下，IntelliJ IDEA完全控制调试过程：它根据PHP Web Page调试配置启动应用程序，打开浏览器并激活调试引擎。

PHP Web页面调试配置告诉IntelliJ IDEA URL地址，以访问应用程序的起始页面，浏览器以打开该起始页面以及要使用的调试服务器配置 。

您还可以指定希望IntelliJ IDEA在调试期间忽略的脚本请求。 当您的应用程序包含使用AJAX的脚本时，此方法很有用。假设您有一个menu-ajax-script.php文件 ，它可以“重新加载”网页的一部分。该脚本可以正常工作，因此您无需对其进行调试。但是，在调试会话期间仍然需要此脚本。要忽略此脚本的传入连接，请将menu-ajax-script.php脚本添加到跳过的路径列表中。

准备调试引擎

在开始调试之前，请确保已正确安装和配置了调试引擎。IntelliJ IDEA支持使用两种最受欢迎的工具进行调试 ： Xdebug和Zend Debugger 。这些工具相互阻塞，因此无法同时使用。为避免此问题，您需要按照配置Xdebug和配置Zend Debugger中的描述更新php.ini文件中的相应部分。

要在编辑器中打开活动的php.ini文件：

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在“ 语言和框架”下单击“ PHP ” 。

  2. 在打开的PHP页面上，单击 CLI解释器字段旁边。

  3. 在打开的“ CLI解释器”对话框中，“ 配置文件”只读字段显示活动php.ini文件的路径。单击“ 在编辑器中打开” 。

设置断点

断点是用于在调试会话期间触发操作的源代码标记。 通常，设置断点的目的是暂停程序执行以允许您检查程序数据。但是，IntelliJ IDEA可以将断点用作各种不同操作的触发器。可以在调试过程中随时设置断点。断点不会直接影响源文件，但是断点及其设置将与IntelliJ IDEA项目一起保存，因此您可以在调试会话中重复使用它们。

  1. 将插入号放在源代码的所需行。

    可以在PHP上下文中的php ， html和其他类型的文件中设置断点。行断点只能在可执行行上设置，而不能在注释，声明或空行上设置。

  2. 请执行以下任一操作：

    • 在要切换断点的行上单击装订线区域。

    • 从主菜单中，选择运行|切换行断点 。

    • 按Ctrl+F8 。

创建类型为PHP Web Page的调试配置

  1. 通过执行以下操作之一打开“ 运行/调试配置”对话框：

    • 在主菜单上，选择“运行”。编辑配置 。

    • 按Shift+Alt+F10 ，然后按0显示“ 编辑配置”对话框或从弹出窗口中选择配置，然后按F4 。

  2. 请点击在工具栏上或按Insert 。从列表中，选择“ PHP Web Page”配置类型。将打开“ PHP网页”对话框。

  3. 指定配置名称。

  4. 从服务器列表中选择适用的调试服务器配置，或单击浏览 。并在“ 服务器”对话框中定义调试服务器配置，如创建PHP调试服务器配置中所述打开。

  5. 在“ 开始URL”字段中，键入实现应用程序开始页面的文件的服务器路径。指定相对于服务器配置根目录的路径（ 服务器配置根目录是通过服务器配置可访问的本地或远程服务器上文件树中的最高文件夹。对于就地服务器，它是项目根。）。下面的只读字段显示了应用程序起始页面的URL地址。URL地址是在您键入时动态组成的。

  6. 指定浏览器以打开应用程序。从浏览器列表中选择配置的浏览器，或单击浏览 并在打开的“ Web浏览器”对话框中指定另一个浏览 器 。

启动调试会话并检查挂起的程序

  1. 要开始调试，请单击“ 调试”按钮在工具栏上。

  2. 调试器暂停到达第一个断点后，立即通过分析frame检查应用程序。框架与活动的方法或函数调用相对应，并存储被调用的方法或函数的局部变量，其自变量以及实现表达式求值的代码上下文。所有当前活动的框架都显示在“ 调试”工具窗口的“ 框架”窗格中，您可以在它们之间进行切换并分析“ 变量和监视”窗格中存储在其中的信息。有关更多详细信息，请参阅检查暂停的程序一节。

  3. 继续运行该程序，并在再次挂起后检查其框架。

    • 要手动控制程序执行，请使用“ 运行”菜单或工具栏按钮下的命令逐步执行代码：单步执行 F7 ， 走出 Shift+F8 ， 跨步 F8 ， 和别的。有关更多详细信息，请参阅逐步完成程序 。

    • 要使程序自动运行到下一个断点，请选择“运行” |“继续”以恢复会话。恢复程序或按F9 。

指定脚本以跳过请求

当您的应用程序包含使用AJAX的脚本时，此方法很有用。假设您有一个menu-ajax-script.php文件 ，它可以“重新加载”网页的一部分。该脚本可以正常工作，因此您无需对其进行调试。但是，在调试会话期间仍然需要此脚本。要忽略此脚本的传入连接，请将menu-ajax-script.php脚本添加到跳过的路径列表中。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| PHP |调试|跳过的路径 。

  2. 在打开的“ 跳过的路径”页面上，如果IntelliJ IDEA收到传入的脚本和文件夹的“忽略列表”，则不要调用这些脚本和文件夹。

    • 要将新条目添加到列表中，请单击或按Alt+Insert 。然后点击然后在打开的对话框中选择要跳过连接的文件或文件夹。

    • 要从列表中删除条目，请选择它并单击或按Alt+Delete 。现在，脚本将在收到请求后执行。

  3. 要使IntelliJ IDEA每次收到要跳过的脚本的请求时通知您，请选中“ 关于跳过的路径通知”复选框。
