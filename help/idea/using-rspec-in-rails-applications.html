在Rails应用程序中使用RSpec

以下仅在安装并启用Ruby插件时有效。

在Rails应用程序中使用RSpec需要将rspec-rails gem添加到您的项目中。这样做的依赖rspec gem将自动添加到项目中。添加rspec-rails gem时，相应的生成器rspec:install出现在可用发电机列表中。

IntelliJ IDEA支持rspec-rails gem 2.12.0及更高版本。

在Rails应用程序中启用RSpec支持后，IntelliJ IDEA将提供：

  • 生成RSpec测试基础结构。

  • RSpec特定的代码完成：

  • Zeus服务器的用法 。

  • RSpec运行/调试配置

根据您特定的Rails版本，可以使用不同的工作流程。如果您使用的是Rails 3.0及更高版本，建议使用捆绑程序并在Gemfile中指定rspec-rails gem。因此，IntelliJ IDEA建议以下工作流程：

要使用rspec，请遵循以下常规步骤

  1. 确保已安装必需的宝石。

  2. 在Gemfile中，添加以下行：

    gem 'rspec-rails'

  3. 生成RSpec测试基础结构。为此，请按Alt+Insert ，在弹出菜单上选择“运行Rails Generator ”，开始输入rspec生成器名称，然后选择rspec:install从列表中：

    在“ 设置RSpec支持”对话框中，指定生成器选项。执行相应的脚本，并将其输出显示在“运行工具”窗口的控制台选项卡中。

  4. 生成测试模板的方式与其他Rails应用程序元素相同。

  5. 运行或调试您的规范。

    如果使用捆绑程序，请确保在RSpec运行/调试配置中 ，选中了在捆绑程序上下文中运行脚本复选框。
    请注意，您可以执行文件夹中的所有测试，特定的测试脚本或脚本中的单个示例。

    • 要执行文件夹中的所有测试，请在“项目”工具窗口中右键单击包含规格的文件夹，然后选择“运行：所有规格” 从上下文菜单中。

    • 要执行测试脚本，请在“项目”工具窗口中右键单击规范，或在编辑器中打开该规范，然后在单个示例之外的某个地方右键单击。然后选择运行 。从上下文菜单中。

    • 要执行单个示例，请在编辑器中打开所需的规范，右键单击要执行的示例，然后选择“运行”。 。
