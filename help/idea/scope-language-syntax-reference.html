范围语言语法参考

范围语言用于指定参与各种分析的项目范围 。

类集

  • 单个类由类名定义，即com.intellij.openapi.MyClass

  • 包中所有类的集合（不递归到子包中）由点后的星号定义，例如：com.intellij.openapi.*

  • 包中所有子类（包括子包的内容）的集合由双点后的星号定义，例如com.intellij.openapi..*

文件集

  • 单个文件由文件名定义，即MyDir/MyFile.txt

  • 目录中所有文件的集合（不递归到子目录）由斜杠后的星号定义，例如： file:src/main/myDir/*

  • 目录中所有文件的集合（包括子目录的内容）由双斜杠后的星号定义，例如file:src/main/myDir//*

修饰符

位置修饰符

位置修饰符可帮助您指定所需文件的位置：

  • src: –用于源文件

  • lib: –图书馆课

  • test: –用于测试代码

例如， src:com.intellij.openapi.* patten将作用域中所有类放在源根目录下com.intellij.openapi包，不包括子包。默认位置是模块根目录。

模块修饰符

模块修饰符通过指定相关模块的名称来帮助您缩小范围：

  • src[module name]:
  • lib[module name]:
  • test[module name]:

例如， src[MyModule]:com.intellij.openapi.*模式将范围内与源文件夹相关的所有类放置在范围内MyModule中的模块com.intellij.openapi包，不包括子包。

逻辑运算符

范围语言允许您使用常见的逻辑运算符：

&&代表AND ||或！不

除此之外，括号可用于将逻辑运算符组合在一起。例如，以下范围

(||)&&

暗含或或或与。

另一个例子

file[*web*]:src/main/java//*
表示名称包含的所有模块的作用域web ，并且所有文件都以递归方式位于目录src / main / java中 。

从现有范围创建新范围

您可以从几个现有范围中编译一个新范围。在这种情况下，您可以使用引用现有范围$$MyScope 。

例如， $Scope1||$Scope2模式将范围内的所有文件置于范围内Scope1和Scope2 。

例子

  • file[MyMod]:src/main/java/com/example/my_package//* -在项目中包含模块“ MyMod”中位于指定目录和所有子目录中的所有文件。

  • src[MyMod]:com.example.my_package..* -将所有类递归地包含在模块源目录中的包中。

  • lib:com.company..*||com.company..* -将项目和库中的所有类递归地包含在包中。

  • test:com.company.* -将所有测试类包括在包中，但不包括在子包中。

  • [MyMod]:com.company.util.* -在指定模块的软件包中包含所有类和测试类。

  • file:*.js||file:*.coffee -包括所有JavaScript和CoffeeScript文件。

  • file:*js&&!file:*.min.* -包括除通过minification生成的文件以外的所有JavaScript文件，这些文件由指示min延期。
