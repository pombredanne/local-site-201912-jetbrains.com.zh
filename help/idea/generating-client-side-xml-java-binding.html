生成客户端XML-Java绑定

要开发从客户端到目标Web服务的格式正确且有效的请求，您需要了解Web服务的可用方法，其使用的数据类型，服务的接口，可接受的请求格式，产生的回应等所有这些数据都显示在目标Web服务的WSDL描述符中。

IntelliJ IDEA可以根据所需的WSDL描述符生成必要的客户端XML-Java绑定，从而在开发客户端请求时为您提供有效的编码帮助。您只需要指定WSDL描述符的URL地址，IntelliJ IDEA将检索必要的数据并生成Java类。

在“从Wsdl或Wadl生成Java代码”对话框中配置Java代码生成，该对话框主要在启用Web服务客户端开发支持时打开。

配置客户端XML-Java绑定的生成

  1. 通过执行以下操作之一，打开“从Wsdl或Wadl生成Java代码”对话框：

    • 创建一个模块，并在其中支持 Web服务客户端。

    • 在开发过程中的任何时候，都可以在“ 项目”视图中选择所需的客户端模块，然后选择“ WebServices” |“ WebServices”。从上下文菜单中的Wsdl或Wadl生成Java代码 。

  2. 在“ Web Service WSDL URL”字段中，指定所需的Web Service WSDL描述符的URL地址。

    如果“ 状态只读”字段通知您有关WSDL URL连接异常的信息，请确保目标Web服务正在运行并且其WSDL描述符的URL地址正确。

  3. 从“ 输出路径”下拉列表中，选择目录以放置生成的文件。

  4. 在“ 包前缀”下拉列表中，指定用于放置已编译Java类的包。

  5. 对于Apache Axis Web服务的客户端，使用以下字段为代码生成过程指定其他配置选项：
    • 类型映射版本
    • 生成测试用例
    • 生成架构数组的类
    • 生成未引用的元素
    • 支持包装文件/文字样式
