运行/调试配置：Spring Boot

此页面描述控件。有关如何在IntelliJ IDEA中使用Spring Boot的更多信息，请参阅Spring Boot 。

配置选项卡

主班

指定将要执行的类的标准名称（传递给JRE）。

环境

虚拟机选项

如有必要，请指定在服务器启动时传递给服务器JVM的命令行选项。

  • 使用空格分隔选项，例如， -client -ea -Xmx1024m 。

  • 如果选项包含空格，则将空格或参数本身用双引号引起来，例如， some" "arg要么"some arg" 。

  • 如果选项在参数中包含双引号，请使用反斜杠对双引号进行转义，例如， -Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果您定义一个变量MY_ENV_VAR ，您可以将其传递给foo属性：

    -Dfoo = $ {MY_ENV_VAR}

程序参数

以您在命令行中使用的格式指定要传递给程序的参数。使用与指定VM选项相同的规则。

工作目录

如有必要，请指定正在运行的应用程序将使用的目录。该目录是所有相对输入和输出路径的起点。默认情况下指定项目目录。

环境变量

单击浏览创建变量并指定其值。

使用模块的类路径

选择其类路径将用于运行应用程序的模块。

在“已提供”范围内包含依赖项

启用此选项可将具有Provided范围的依赖项添加到运行时类路径。

杰瑞

默认情况下，来自模块依赖项的最新JDK用于运行应用程序。您可以指定备用的JDK或JRE。

缩短命令行

如果类路径太长或您有许多VM参数超出OS命令行长度限制，请选择一种可用于缩短命令行的方法：

  • none ：IntelliJ IDEA不会缩短长类路径。如果命令行超出了操作系统限制，则IntelliJ IDEA将无法运行您的应用程序，并显示一条消息，提示您指定缩短方法。

  • JAR清单 ：IntelliJ IDEA将通过临时classpath.jar传递长类路径 。原始类路径在清单文件中定义为class-path classpath.jar中的属性。请注意，如果使用此方法将完整的命令行缩短了，您将能够预览整个命令行，而不仅仅是临时classpath.jar的classpath 。

  • classpath.file ：IntelliJ IDEA会将长类路径写入文本文件。

  • 用户本地默认值 ：将为IntelliJ IDEA版本10.3之前创建的项目自动设置此旧选项。IntelliJ IDEA将根据ide / workspace.xml和idea.config.path / options / options.xml文件中设置的属性来配置此设置。

如果您选择共享您的运行/调试配置，则此设置是共享的，因此，无论其操作系统如何，都将对您的团队成员应用相同的方法。

春季靴

启用调试输出

通过添加以下命令启用日志记录调试输出-Ddebug VM选项。

隐藏横幅

通过添加来隐藏Spring Boot启动标语-Dspring.main.banner-mode=OFF VM选项（对于Spring Boot 1.4.x和更早版本，该选项为-Dspring.main.banner=false ）。

启用启动优化

选择此选项可以加快应用程序的启动。的-XX:TieredStopAtLevel=1和-noverify VM选项将被添加。

启用JMX代理

使用“ 启用JMX代理”选项可能会减慢应用程序的启动速度。

通过添加以下VM选项来启用显示应用程序端点数据的JMX代理（需要Spring Boot 1.3.0或更高版本）：

  • -Dcom.sun.management.jmxremote
  • -Dspring.jmx.enabled=true
  • -DSpring.liveBeansView.mbeanDomain
  • -Dspring.application.admin.enabled=true

对于具有FAT文件系统的Windows用户，默认情况下不提供Spring Boot端点功能。要启用它，添加-XX:+PerfBypassFileSystemCheck到VM options ，并确保％TMP％\ hsperfdata_username具有完全文件夹访问权限。

IntelliJ IDEA使用本地JMX连接器来检索Spring Boot执行器端点的数据。但是，如果Spring Boot应用程序和IntelliJ IDEA JVM具有不同的位数，并且您正在使用OpenJ9 JDK，则不可能获得本地JMX连接器地址。在这种情况下，请将以下行添加到VM选项 ：

  • -Dcom.sun.management.jmxremote.port={some_port}
  • -Dcom.sun.management.jmxremote.authenticate=false
  • -Dcom.sun.management.jmxremote.ssl=false

如Oracle官方文档中所述，此配置是不安全的。任何知道（或猜测）您的端口号和主机名的远程用户都将能够监视和控制您的Java应用程序和平台。

关于“更新”操作

指定单击“ 更新应用程序”后，IDE将执行的操作：

  • 更新资源 -编译所有修改过的和依赖的资源文件。

  • 更新类和资源 -编译所有已修改和相关的文件。

  • 更新触发器文件 -更新触发器文件。通过添加以下内容来设置触发器文件名-Dspring VM选项。

  • 热交换类和更新触发器文件（如果失败） -（仅在调试模式下可用）编译所有修改过的文件和从属文件，热交换类，如果交换失败的方式，则更新触发器文件。通过添加以下内容来指定触发器文件名
    -Dspring.devtools.restart.trigger-file=.restartTriggerFile VM选项。

停用框架

指定切换到另一个应用程序（IntelliJ IDEA除外）后，IDE将执行的操作：

  • 更新资源 -编译所有修改过的和依赖的资源文件。

  • 更新类和资源 -编译所有已修改和相关的文件。

活动资料

指定要在此运行/调试配置中使用的活动Spring配置文件 。

覆盖参数

使用此表来指定要覆盖的Spring Boot配置参数。此类参数通常在配置文件中定义。将它们列出在运行/调试配置中，使您可以通过修改运行/调试配置（而不是配置文件）来轻松切换参数，或者具有多个具有不同参数值的运行/调试配置。

代码覆盖率选项卡

使用此选项卡可以配置代码覆盖率监视选项。

选择报道亚军

选择所需的代码覆盖率运行器。

默认情况下，IntelliJ IDEA使用其自己的覆盖引擎和“采样”模式。您也可以选择JaCoCo或Emma来计算覆盖率。

采样

选择此选项以最小的速度测量代码覆盖率。

追踪

选择此选项以收集准确的分支机构覆盖范围。此模式仅适用于IntelliJ IDEA代码覆盖率运行器。

跟踪每个测试范围

选择此复选框以检测一个测试覆盖的线以及所有覆盖该线的测试。

记录代码覆盖率数据的包和类

请点击和按钮指定要测量的类别和包装。您也可以从列表中删除类和包，方法是在列表中选择它们并单击 。

在测试文件夹中启用覆盖率

如果选中此复选框，则标记为测试的文件夹包含在代码覆盖率分析中。

日志标签

使用此选项卡可以指定在运行或调试工具窗口的控制台中必须显示在运行或调试时生成的日志文件。

日志文件显示在控制台中

在此表中，指定将在“运行/调试”工具窗口中显示的一个文件或一组文件。

将控制台输出保存到文件

将控制台输出保存到指定位置。

在将消息打印到标准输出流时显示控制台

启用输出控制台，并在关联的进程正在写入Standard.out的情况下将其转发。

在将消息打印到标准错误流时显示控制台

启用输出控制台，并在关联的进程正在写入Standard.err的情况下将其转发。

启动前选项

使用此部分可以指定在开始运行/调试会话之前必须执行的任务。您指定的任务将按照它们在列表中出现的顺序执行。

显示此页面

此选项显示在开始运行/调试会话之前的运行/调试设置。

激活工具窗口

运行会话后，选中该复选框以打开“运行/调试”工具窗口。
