IntelliJ IDEA 2019.3帮助

马文

IntelliJ IDEA支持与Maven的全功能集成，可帮助您自动化构建过程。您可以轻松地创建一个新的Maven项目，打开并同步一个现有的项目，向任何现有的IntelliJ IDEA项目添加Maven支持 ，配置和管理多模块项目。

如果要使用未与IntelliJ IDEA捆绑在一起的自定义Maven版本，则需要在本地安装它 。

创建一个新的Maven项目

  1. 如果IntelliJ IDEA中当前没有打开任何项目，请在“欢迎”屏幕上单击“ 创建新项目 ”。否则，选择文件|新品|从主菜单进行项目 。

  2. 从左侧的选项中选择Maven 。

  3. 如果要使用预定义的项目模板，请指定项目的SDK（JDK）或使用默认的SDK和原型 （通过单击Add Archetype配置您自己的原型）。

    单击下一步 。

  4. 在向导的下一页上，指定添加到pom.xml文件中的以下Maven坐标 ：

    • GroupId-新项目的软件包。

    • ArtifactId-您的项目的名称。

    • 版本 -新项目的版本。默认情况下，此字段是自动指定的。

    有关Maven坐标的更多信息，请参见Maven命名约定 。

    单击下一步 。
  5. 如果要使用Maven原型创建项目，则IntelliJ IDEA将显示可用于设置Maven主目录和Maven存储库的Maven设置 。另外，您可以检查原型属性。

    单击下一步 。

  6. 指定名称和位置设置。

    点击完成 。

打开一个现有的Maven项目

  1. 从主菜单中，选择文件|打开 。

  2. 在打开的对话框中，选择要打开的项目的pom.xml文件。

    单击确定 。

  3. 在打开的对话框中，单击“ 以项目形式打开” 。

IntelliJ IDEA在IDE中打开并同步Maven项目。如果在打开项目时需要调整导入选项，请参考Maven设置 。

将新的Maven模块添加到现有项目

您可以将Maven模块添加到已经在其中工作的项目中。

  1. 在“ 项目”工具窗口中，右键单击项目文件夹，然后选择“ 新建” |“新建”。模块 。或者，从主菜单中选择File |。新品|模块打开新模块向导。

  2. 如果使用主菜单添加模块，则添加模块的过程与创建新的Maven项目相同 。

    如果要通过右键单击根文件夹来添加子模块，则添加新模块的过程会更短。您需要在“ 名称”字段中指定模块的名称 。其余信息将自动添加，您可以使用默认设置，也可以根据自己的喜好进行更改。

配置多模块Maven项目

您可以在IntelliJ IDEA中创建一个多模块Maven项目 。多模块项目由具有几个子模块的父POM文件定义。

  1. 创建一个Maven父项目 。IntelliJ IDEA创建一个包含src文件夹的标准Maven布局。

  2. 在“ 项目”工具窗口中，删除src文件夹，因为仅在极少数情况下才需要它。对于您的常规项目，父POM不需要src文件夹。

  3. 在“ 项目”工具窗口中，右键单击您的项目（或从主菜单中，单击“ 文件” ），然后选择“ 新建” |“新建”。添加子项目的模块 。

  4. 按照有关如何添加模块的说明，在“ 新建模块”向导中，指定必要的信息，然后单击“ 完成” 。

    src文件夹是自动创建的，您可以打开POM并添加所需的包装。IntelliJ IDEA将模块添加到父项目。IntelliJ IDEA还将子项目的名称和描述添加到父POM。

    在多模块项目中，父POM需要具有一个pom打包。

    最后但并非最不重要的一点是，IntelliJ IDEA将父POM的描述添加到子项目的POM中。

    您可以点击在左侧装订线中，以从子项目中快速打开父POM。
  5. 您还可以将依赖项添加到将由子项目继承的父POM。

  6. 打开Maven工具窗口，以查看父项目POM中所做的所有更改都反映在子项目中。

访问Maven设置

使用Maven设置可配置选项，例如Maven版本，本地存储库，脱机模式等。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 构建，执行，部署” | Maven 。

    请点击在工具栏上的Maven工具窗口中，以访问Maven设置。

  2. 在“ Maven设置”页面上，配置可用选项，然后单击“ 确定”保存更改。

安装Maven自定义版本

  1. 在计算机上下载所需的Maven版本。

  2. 从主菜单中选择文件|设置/首选项|生成，执行，部署|生成工具| Maven 。

  3. 在“ Maven设置”页面上的“ Maven主目录”字段中，指定Maven定制版本安装的位置。

  4. 单击确定 。

在Maven项目中更改JDK版本

您可以在几个地方更改JDK版本，这不仅会影响您当前的项目，还会影响整个应用程序。

在项目结构中更改JDK版本

在“ 项目结构”对话框中更改JDK版本只会影响当前项目。

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S 。

  2. 在打开的对话框中，在Project SDK中 ，指定JDK版本，然后单击“ 确定”以保存更改。

更改Maven运行程序的JDK版本

当IntelliJ IDEA运行Maven目标时，它将使用为Maven运行器指定的JDK版本。默认情况下，IntelliJ IDEA使用项目的JDK 。

更改Maven运行程序的JDK仅会影响当前项目。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 构建，执行，部署” | Maven |赛跑者 。

  2. 在打开的页面的JRE字段中，选择JDK版本。

更改Maven导入器的JDK版本

更改Maven导入程序的JDK版本将影响整个应用程序，因为它是Maven全局设置的一部分。如果要使用与项目中用于同步或解析依赖项的JDK版本相同的JDK版本 ，请更改导入程序的JDK版本。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 构建，执行，部署” | Maven |正在导入 。

  2. 在打开的页面上，在“ JDK for importer”字段中，选择与在“ 项目结构”中使用的相同的JDK版本，然后单击“ 确定”以保存更改。
