测试Node.js

以下内容仅在安装并启用Node.js插件时有效。

借助IntelliJ IDEA，您可以使用许多框架来测试Node.js应用程序。 自2018年5月起，不推荐使用Nodeunit。有关详细信息，请参见Nodeunit官方网站。

在你开始前

  1. 确保在“ 插件”页面上启用了NodeJS捆绑插件，有关详细信息，请参阅管理插件 。

  2. 下载并安装测试框架。

为Node.js应用程序创建和运行单元测试

  1. 编写单元测试 。

  2. 将存储单元测试的文件夹标记为测试源文件夹，请参阅内容根目录 。

  3. 创建类型为Nodeunit 的运行配置 。

  4. 在“ 运行工具”窗口中启动单元测试并监视测试结果 。

创建节点单元测试

  1. 创建与src文件夹相同级别的文件夹测试

  2. 填充测试文件夹。对于每个生产文件，创建一个单独的测试文件。

  3. 将存储单元测试的文件夹标记为测试源文件夹，请参阅内容根目录 。

创建Nodeunit运行配置

  1. 通过选择运行|打开运行/调试配置对话框。从主菜单编辑配置 。

  2. 点击添加按钮在工具栏上，然后选择Nodeunit配置类型。

  3. 在打开的对话框中，指定以下内容：

    1. 标识配置的名称。

    2. 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

    3. 工作目录。这可以是项目根文件夹，也可以是测试文件夹的父目录。

    4. 要运行的测试范围。

      • 要使IntelliJ IDEA运行文件夹中的所有测试文件，请从“ 运行”列表中选择目录中的“ 所有JavaScript测试文件 ” 。在目录字段中，提供相对于工作目录的测试文件夹的路径。

      • 要执行特定的测试，请从“ 运行”列表中选择JavaScript测试文件 。在JavaScript测试文件字段中，提供相对于工作目录的文件路径。

  4. 应用更改并关闭对话框。
