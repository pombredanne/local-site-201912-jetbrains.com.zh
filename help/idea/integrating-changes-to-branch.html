将更改集成到分支

IntelliJ IDEA允许您将更改集成到所选分支中，并将此类集成的结果提交到存储库。请执行下列操作：

  1. 在里面 版本控制窗口Alt+9 ，切换到“ 存储库”选项卡 。

  2. 右键单击要集成的变更列表，然后选择“ Subversion | Subversion”。从上下文菜单集成到分支 ：

    如果要集成单独的文件而不是整个更改列表，请在“ 更改的文件”窗格中选择它们，然后选择“ Subversion | Subversion”。从上下文菜单集成到分支 。

    如果您在服务器上，并在本地工作副本使用SVN 1.5或更高版本两种，选择变更列表相关的变更列表/文件（S）或更改的文件窗格，然后单击在工具栏上。

    将打开“ 集成到分支”对话框，显示源分支和目标分支的URL地址以及可用的本地工作副本列表。

  3. 从“ 集成到工作副本”列表中，选择本地工作副本的路径，选定的更改列表将被集成到其中。

    要向列表添加路径，请点击按钮。

    要从列表中删除路径，请单击按钮。

    确保指定的工作副本目录在Subversion版本控制下！

  4. 要预览合并结果，请启用--dry-run的开关svn命令，选择“ 尝试合并，但不进行更改”复选框。

    如果未选中此复选框，则源将以静默方式合并。

  5. 单击确定 。 “ 提交更改”对话框打开。

  6. 查看摘要，指定必要的选项，然后运行提交。
