基本

为了支持ActionScript和Flex ，IntelliJ IDEA提供了：

  • Flash / Flex支持插件 。该插件与IDE捆绑在一起，必须启用。

  • 专用模块类型（闪存）。

  • 构建各种目标平台（Web，桌面和移动）和输出类型（SWF和SWC）的配置。

  • 专用的运行/调试配置类型（Flash App，FlexUnit和Flash Remote Debug）。

  • ActionScript Profiler和Flash UI Designer插件。这些插件可从JetBrains存储库下载。有关更多信息，请参见在Flash和Flex应用程序中对CPU进行性能分析 。另请参阅管理插件 。

ActionScript和Flex支持

ActionScript和Flex支持包括：

  • 代码完成 ，包括语句的完成 Ctrl+Shift+Enter和智能类型完成 Ctrl+Shift+Space 。

  • 错误和语法突出显示。

  • ActionScript和Flex代码重构 ：

    • 更改方法签名 。

    • 委托方法。

    • 提取接口 。

    • 提取方法 。

    • 提取超类 。

    • 内联 。

    • 介绍常量 。

    • 介绍领域 。

    • 提取参数 。

    • 介绍变量 。

    • 将类或接口移动到包中 。

    • 将内部上移到上层，以将在包外部声明的类，函数，变量，常量和名称空间移动到包中。

    • 移动/复制文件。

    • 移动静态成员 。

    • 向上拉成员，向下拉成员 ， 通过重构，您可以通过将类成员移动到子类或子接口来清理类层次结构。然后将成员仅重定位到直接子类/接口中。这样可以提高内部类的一致性。 。

    • 重命名文件，函数，变量，参数，属性或标签（直接或通过引用）。

    • 安全删除文件。

  • ActionScript和Flex代码检查和快速修复 。

  • 用于创建各种应用程序元素的意图动作 。

  • 代码格式化和折叠 。

  • 高级搜索和导航以及结构视图 。

  • 带有装订线图标的增强导航 。

  • 从CSS属性和选择器导航到ActionScript中的声明Ctrl+B 。

  • 可以使用各种编译器外壳和编译选项来构建ActionScript和Flex应用程序。

  • 支持断点和特定的运行/调试配置 ，可直接从IntelliJ IDEA调试ActionScript和Flex应用程序。

  • 使用ActionScript和Flex感知调试器 ，您可以逐步执行应用程序，评估表达式，检查相关信息并查找运行时错误。

  • 快速Javadoc Ctrl+Q对于AsDoc。

  • BlazeDS支持。

  • 在应用程序开发周期的所有阶段都提供AIR应用程序开发支持。还支持为移动设备开发AIR应用程序。有关基本操作方法信息，请参阅IntelliJ IDEA 10.5中的新增内容：开发适用于Android的移动AIR应用程序 。

  • 创建纯ActionScript应用程序的可能性。

  • ActionScript动态模板 ： 文件|设置|实时模板| ActionScript和JavaScript组。

  • 能够包装和解压缩ActionScript和MXML中的代码结构Ctrl+Alt+T和Ctrl+Shift+Delete 。

  • 类型层次结构Ctrl+H ，方法层次结构Ctrl+Shift+H和呼叫层次结构Ctrl+Alt+H用于Flex源（ .mxml和.as文件）。

  • 轻松导入在Adobe Flash Builder中创建的Flex项目 。

  • 导入Flexmojos项目。请参阅相关导入设置的描述。

  • ActionScript和Flex UML类图 。这些功能包括以结构化可视形式查看 ActionScript源文件中的更改的功能。

FlexUnit支持

IntelliJ IDEA支持FlexUnit的 0.9和4版本， FlexUnit是Flex和ActionScript应用程序和库的单元测试框架。

FlexUnit支持包括：

  • 专用的FlexUnit运行/调试配置，以运行单个测试方法，测试套件，特定测试类中的所有方法或给定程序包中的所有测试类。

  • 能够在运行和调试模式下执行测试。

  • 通过FlexUnit 4测试运行器支持Flunit测试。

  • 支持FlexUnit的代码检查（默认情况下处于关闭状态）。
