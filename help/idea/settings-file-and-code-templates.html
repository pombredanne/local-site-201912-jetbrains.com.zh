文件和代码模板

档案|设置|编辑器 Windows和Linux的文件和代码模板
IntelliJ IDEA |首选项|编辑器 macOS的文件和代码模板
Ctrl+Alt+S

IntelliJ IDEA使用文件模板创建文件。使用此页面可以查看，编辑和创建此类模板。

不同的模板组位于不同的选项卡上 。

选择模板后，其内容和说明将显示在页面的右侧。

每个项目与默认方案

项目

描述

方案

选择文件和代码模板设置是属于整个工作区还是当前项目：

  • 文件和代码模板为全局时，选择默认方案。

  • 如果要使用可共享的特定于项目的文件和代码模板，则选择“ 项目方案”。

标签

标签

描述

档案

此选项卡显示可用的文件模板。

您可以编辑现有模板，也可以创建新模板。

粗体显示的模板无法删除；其名称和扩展名无法编辑。

包括

此选项卡显示可包含在文件模板中的可重用片段的模板。

您可以编辑现有模板，也可以创建新模板。

码

此选项卡显示内置的片段，这些片段是IntelliJ IDEA在各种典型情况下可以生成的代码片段的模板，例如，用于生成实现的或重写的方法主体。

您可以编辑现有的代码段，但不能创建新的代码段。

其他

此选项卡显示与各种应用程序服务器和框架相关的模板组。

您可以编辑现有模板，但不能创建新模板。

工具提示

描述

建立范本

单击此按钮可在当前打开的选项卡中创建一个新模板。此选项仅在“ 文件”和“ 包含”选项卡中可用。

新模板的位置由“ 架构”列表定义。

删除模板

单击此按钮删除所选模板。此选项仅适用于“ 文件”和“ 包含”选项卡中的某些模板。

复制模板

单击此按钮创建所选模板的副本。此选项仅在“ 文件”和“ 包含”选项卡中可用。

重置为默认

单击此按钮可将所选模板还原为其原始状态。此选项仅适用于已修改的模板（此类模板以蓝色突出显示）。

不适用

重启

当您开始编辑模板时，此链接显示在页面的右上角。单击此链接会将所有未保存的更改重置为任何选项卡中的任何模板。

模板设置和内容

项目

描述

名称

创建新模板时，将显示此字段。指定新模板的名称。

延期

在此字段中，指定扩展名。创建此类型的新文件时，IntelliJ IDEA将应用此模板。

范本文字

编辑模板内容。您可以使用：

  • 纯文本。

  • #parse用于模板的指令包括 。

  • 自定义变量。变量名可以通过以下方式直接在模板中定义： #set指令或在文件创建期间。

    请注意，IntelliJ IDEA不会提示输入使用以下命令定义的Velocity变量的值#set 。

  • 变量将被扩展为相应的值${}格式。

    以下预定义变量可以在文件模板中使用 ：

    变量

    描述

    ${DATE}

    当前系统日期

    ${DAY}

    该月的当前日期

    ${DS}

    美元符号$ 。此变量用于转义美元字符，因此不会将其视为模板变量的前缀。

    ${FILE_NAME}

    新PHP文件的名称（如果启用了PHP插件 ）

    ${HOUR}

    当前时间

    ${MINUTE}

    当前分钟

    ${MONTH}

    这个月

    ${MONTH_NAME_FULL}

    当月的全名（一月，二月等）

    ${MONTH_NAME_SHORT}

    当前月份名称的前三个字母（一月，二月，依此类推）

    ${NAME}

    新实体的名称（文件，类，接口等）

    ${ORGANIZATION_NAME}

    在项目设置中指定的组织名称（ Ctrl+Shift+Alt+S ）

    ${PACKAGE_NAME}

    创建新类或接口文件的目标包的名称

    ${PRODUCT_NAME}

    IDE的名称（例如，IntelliJ IDEA）

    ${PROJECT_NAME}

    当前项目名称

    ${TIME}

    当前系统时间

    ${USER}

    当前用户的登录名

    ${YEAR}

    今年

    IntelliJ IDEA为PHP包含模板提供了一组附加变量，这些模板用于定义可重用的代码段 ，例如文件头和PHPDoc注释 ）。

    以下其他预定义变量可在PHP包含模板中使用 ：

    变量

    描述

    ${CARET}

    完成变量编辑后的插入位置。仅在文件创建期间生成并插入PHPDoc注释时应用。通过代码创建PHPDoc注释时|产生| PHPDoc块 ，提供了多种功能或方法选择。在这种情况下，可以为多个类，函数，方法或字段创建文档注释，因此${CARET}变量将被忽略。

    ${CLASS_NAME}

    定义为其生成PHPDoc注释的字段或方法的类的名称

    ${NAME}

    将为其生成PHPDoc注释的类，字段或函数（方法）的名称

    ${NAMESPACE}

    类或字段名称空间的全限定名称（不带斜杠）

    ${PARAM_DOC}

    参数的文档注释。评估为一组@param type name线。如果为其生成注释的函数不包含任何参数，则求值为空字符串。

    ${STATIC}

    被替换为static如果生成PHPDoc注释的函数（方法）或字段是静态的，则为关键字。否则计算为空字符串。

    ${THROWS_DOC}

    文档注释中的例外。评估为一组@throws type线。如果为其生成注释的函数没有引发任何异常，则求值为空字符串。

    ${TYPE_HINT}

    要为其生成PHPDoc注释的函数（方法）的返回值。如果通过函数（方法）的静态分析无法检测到返回类型，则求值为void 。

    治疗美元符号

    • 您可以防止对待美元字符$在模板变量中作为前缀。如果您需要美元字符（ $按原样插入，请使用${DS}文件模板变量。应用模板后，此变量的值将为纯美元字符$ 。

      例子：

      • 使用一些版本控制关键字（例如$Revision$ ， $Date$ ，依此类推）在您的默认类模板中， ${DS}代替美元前缀$ 。

      • 模板代码${DS}this将呈现为$this 。

根据样式重新格式化

选中此复选框，以使IntelliJ IDEA根据“ 代码样式”页面上定义的样式重新生成存根文件。

此选项仅在“ 文件”选项卡中可用。

启用实时模板

选中此复选框可在文件模板中使用实时模板。为此，必须将实时模板片段放入Velocity转义语法中。

例如：

#[[ $MY_VARIABLE$ $END$ ]]#

因此，可以指定光标位置。注意，这里需要使用实时模板变量 ！

描述

此只读字段提供有关模板，模板的预定义变量及其工作方式的信息。

此字段在自定义模板中不可用。
