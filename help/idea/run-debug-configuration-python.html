指定脚本参数时，请遵循以下规则：

  • 使用空格分隔各个脚本参数。

  • 包含空格的脚本参数应使用双引号分隔，例如， some" "param要么"some param" 。

  • 如果脚本参数包含双引号，请使用反斜杠对双引号进行转义，例如：

    -s"main.snap_source_dirs=[\"pcomponents/src/main/python\"]" -s"http.cc_port=8189" -s"backdoor.port=9189" -s"main.metadata={\"location\": \"B\", \"language\": \"python\", \"platform\": \"unix\"}"

在输出控制台中模拟终端

使您可以在带有模拟终端模式的输出控制台中运行脚本或模块。此模式对于无法使用标准输出控制台执行的任务很有用，例如，当脚本执行插入符号返回操作时（ print(i, flush=True , end='\r' ）。
请注意，在输出控制台中模拟终端不同于运行终端，后者是用于运行系统外壳命令的单独工具窗口。

Docker容器设置

仅当为项目选择基于Docker的远程解释器时，才会显示此字段。

请点击打开对话框并指定以下设置：

  • 禁用网络 ：选中此复选框可禁用网络。这对应于--net="none" ，这意味着在容器内部，外部网络资源不可用。

  • 网络模式 ：对应于该选项的其他值--net 。

    • bridge是默认值。将为网桥的网络上的容器分配一个IP地址，流量将通过该网桥路由到容器。
      默认情况下，容器可以通过其IP地址进行通信。要通过名称进行通信，必须将它们链接起来。

    • host ：使用容器内主机的网络堆栈。

    • container: ：使用通过其名称或id指定的另一个容器的网络堆栈。

    有关详细信息，请参阅网络设置文档。

  • 链接 ：使用此部分将要创建的容器与其他容器链接。这适用于Network mode = bridge并对应于--link选项。

  • 发布所有端口 ：将所有容器端口公开给主机。这对应于选项--publish-all 。

  • 端口绑定 ： 指定端口绑定列表。类似于使用-p与选项docker run 。

  • 额外主机 ：这对应于--add-host选项。有关详细信息，请参阅管理/ etc / hosts页面。

  • 卷绑定 ：使用此字段可以指定特殊文件夹（ 卷）和运行Docker守护程序的计算机的文件夹之间的绑定。这对应于-v选项。
    有关详细信息，请参见管理容器中的数据。

  • 环境变量 ：使用此字段可以指定环境变量及其值的列表。这对应于-e选项。有关详细信息，请参见页面ENV（环境变量） 。

请点击扩展表格。请点击 ， ， 要么组成列表。

说到设置与某些选项的对应关系（ --net ， --link等），请注意，这些选项来自Docker命令行参数 。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

如果使用基于目录的项目格式 ，则运行/调试配置的设置存储在.idea \ runConfigurations文件夹（如果共享）中的单独的.xml文件中，或者存储在.idea \ workspace中。 xml文件，否则。

如果使用基于文件的格式 ，则设置将存储在.ipr文件中以进行共享配置，否则将存储在.iws文件中。

允许并行运行

如果选中此复选框，则可以并行启动运行配置多次，而不必重新运行它。每个运行程序都将在“ 运行工具窗口”或“ 调试工具窗口 ”的自己的选项卡中启动 。

禁用后，每次启动运行/调试配置的新实例时，IntelliJ IDEA都会检查同一运行/调试配置的其他实例是否存在，并显示确认对话框。如果在确认对话框中单击“ 确定” ，则将停止运行器的第一个实例，而将其替换。当启动两个相同类型的运行/调试配置实例时，这会占用过多的CPU和内存资源。
