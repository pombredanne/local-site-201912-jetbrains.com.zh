启用EJB支持

创建新项目或模块时，可以启用EJB支持。您还可以为现有模块添加EJB支持。

  • 在创建项目或模块时启用EJB支持
  • 为现有模块启用EJB支持

在创建项目或模块时启用EJB支持

  1. 请执行以下任一操作：

    • 如果要创建新项目，请：在“ 欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件” |“ 新建” 。 新品|项目 。

      结果，“ 新建项目”向导打开。

    • 如果要向现有项目中添加模块，请执行以下操作：打开要向其中添加模块的项目，然后选择“ 文件” |“添加” 。新品|模块 。

      结果，“ 新建模块”向导将打开。

  2. 在向导的第一页的左侧窗格中，选择Java Enterprise 。在页面的右侧，指定要使用的JDK并选择要支持的Java EE版本。

  3. 在其他库和框架下 ，选择EJB：Enterprise Java Beans复选框。

  4. 从“ 版本”列表中选择要支持的EJB版本。

    如果要创建EJB部署描述符文件ejb-jar.xml ，请选中“ 创建ejb-jar.xml”复选框。

    选择所需的库选项，并在必要时指定关联的设置。您可以选择：

    • 下载并使用EJB库。

      为此，请在“ 库”下选择“ 下载” 。

      现在，要查看或修改关联的选项，请单击“ 配置” ，然后在打开的“ 下载选项”对话框中：

      • 指定库名称。

      • 选择库级别 （全局，项目或模块）。

      • 在“要下载的文件”下 ，选择要下载的文件。

      • 在“ 将下载的文件复制到”下 ，指定目标文件夹的路径。如果要更改默认路径，请单击并在打开的对话框中指定文件夹位置。

    • 使用EJB库IntelliJ IDEA已经知道。

      为此，请单击“ 使用库” ，然后从列表中选择所需的库。

      如有必要，请配置库设置（例如，更改其名称）。这是在“ 编辑库”对话框中完成的，您可以通过单击“ 配置”打开该对话框。

    • 使用计算机上可用的适当JAR文件创建新库。

      为此，单击使用库 ，然后单击创建 。在打开的对话框中选择所需的JAR文件。（对于多项选择，请保留Ctrl键）。

      如有必要，请配置新库（例如，更改其名称或级别 ）。为此，请单击“ 配置” ，然后在“ 创建库”对话框中指定所需的设置。

    • 将库的设置推迟到以后。在这种情况下，请选择稍后设置库 。

    单击下一步 。

  5. 指定名称和位置设置。

    点击完成 。

为现有模块启用EJB支持

  1. 打开项目工具窗口 （例如， 视图|工具窗口|项目 ）。

  2. 右键单击该模块，然后选择添加框架支持 。

  3. 在“ 添加框架支持”对话框的左侧窗格中，选择“ EJB：Enterprise Java Beans”复选框。

    从“ 版本”列表中选择要支持的EJB版本。

    如果要创建EJB部署描述符文件ejb-jar.xml ，请选中“ 创建ejb-jar.xml”复选框。

    选择所需的库选项，并在必要时指定关联的设置。您可以选择：

    • 下载并使用EJB库。

      为此，请在“ 库”下选择“ 下载” 。

      现在，要查看或修改关联的选项，请单击“ 配置” ，然后在打开的“ 下载选项”对话框中：

      • 指定库名称。

      • 选择库级别 （全局，项目或模块）。

      • 在“要下载的文件”下 ，选择要下载的文件。

      • 在“ 将下载的文件复制到”下 ，指定目标文件夹的路径。如果要更改默认路径，请单击并在打开的对话框中指定文件夹位置。

    • 使用EJB库IntelliJ IDEA已经知道。

      为此，请单击“ 使用库” ，然后从列表中选择所需的库。

      如有必要，请配置库设置（例如，更改其名称）。这是在“ 编辑库”对话框中完成的，您可以通过单击“ 配置”打开该对话框。

    • 使用计算机上可用的适当JAR文件创建新库。

      为此，单击使用库 ，然后单击创建 。在打开的对话框中选择所需的JAR文件。（对于多项选择，请保留Ctrl键）。

      如有必要，请配置新库（例如，更改其名称或级别 ）。为此，请单击“ 配置” ，然后在“ 创建库”对话框中指定所需的设置。

    • 将库的设置推迟到以后。在这种情况下，请选择稍后设置库 。

  4. 在“ 添加框架支持”对话框中单击“ 确定 ”。
