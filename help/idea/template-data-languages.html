模板语言：Velocity和FreeMarker

IntelliJ IDEA使您可以使用Velocity和FreeMarker模板语言（VTL和FTL）开发模板。使用相应的插件提供了编码帮助和语法高亮显示，默认情况下启用了它们。

默认情况下支持以下文件类型：

VTL

.ft ， .vm ， .vsl

FTL

.ftl ， .ftlh ， .ftlx

要为编写模板的静态部分的语言（称为模板数据语言 ）启用编码帮助，请执行以下操作之一：

将模板数据语言与项目中的文件和文件夹相关联

  1. 打开设置/首选项对话框 Ctrl+Alt+S ，单击“ 语言和框架” ，然后单击“ 模板数据语言” 。

  2. 单击项目或相应目录或文件右边的模板数据语言单元格，然后选择语言。

  3. 单击确定以应用更改。

可以使用将模板数据语言更改为上下文菜单命令直接在编辑器中为各个模板文件分配模板数据语言。

为相应的文件类型添加扩展名模式

  1. 打开设置/首选项对话框 Ctrl+Alt+S ，单击“ 编辑器” ，然后单击“ 文件类型” 。

  2. 在“ 识别的文件类型”下 ，选择“ FreeMarker模板”或“ 速度模板” 。

  3. 在“已注册模式”下 ，单击 。

  4. 在“ 添加通配符”对话框中，指定文件扩展名模式，选择一种语言，然后单击“ 确定” 。

  5. 单击确定以应用更改。

未解决的引用可以使用意图动作来修复。您可以选择在同一文件中添加评论，也可以创建带有评论的单独文件。有关更多信息，请参见特殊注释 。

在这种情况下，注释用于提供有关参考的缺失信息。

在后一种情况下，具有默认名称的文件velocity_implicit.vm要么freemarker_implicit.ftl被建造。该文件以以下注释开头：

#* @implicitly included *#

注释文件中提供了用于定义引用类型的代码完成。

如果重命名文件或将其移动到源根目录中的其他位置，引用定义将不会丢失。

特别评论

IntelliJ IDEA为使用模板语言提供以下特殊注释：

FreeMarker注释使用以下任意一种来分隔<#--和-->要么[#--和--#] 。

评论

描述

@implicitly included

任何以该注释开头的FreeMarker文件都包含在同一模块或其依赖项中的所有文件中。使用这样的专用注释文件来定义特定于IDE的注释，以便它们不会污染您的模板文件。

@ftlvariable

定义一个变量。例如，以下注释声明了一个名为foo类型的String ，仅在指定文件中可见：

<#-- @ftlvariable name="foo" type="java.lang.String" file="path/to/file" -->
正确解决String在这种情况下，该模块应连接JDK。支持原始和通用类型。的file参数是可选的。

@ftlroot

定义相对于的路径import和include指令已解决。相对于当前FTL文件包中的任何目录指定路径。还支持JAR内部的路径，例如：

<#-- @ftlroot "path/to.jar!/path/inside/jar" -->

速度注释使用分隔#*和*# 。

评论

描述

@implicitly included

以该注释开头的任何Velocity文件都包含在同一模块中的所有文件中。使用这样的专用注释文件来定义特定于IDE的注释，以便它们不会污染您的模板文件。

@vtlvariable

定义一个变量。例如，以下注释声明了一个名为foo类型的String ，仅在指定文件中可见：

＃* @vtlvariable name =“ foo” type =“ java.lang。字符串” file =“ path / to / file” *＃
正确解决String在这种情况下，该模块应连接JDK。支持原始和通用类型。的file参数是可选的。

@vtlmacrolibrary

使用相对于带有此注释的文件的模板宏定义文件。例如，以下注释使宏从lib.vm可访问代码洞察file.vm ：

＃* @vtlmacrolibrary path =“ lib.vm” file =“ file.vm” *＃

@velocityproperties

定义带有该注释的Velocity.properties文件和相对于该文件的运行时根目录。例如：

＃* @velocityproperties path =” path / to / velocity.properties” runtime_root =” path / to / runtime / root / dir” *＃
