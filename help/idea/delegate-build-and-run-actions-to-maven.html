IntelliJ IDEA 2019.3帮助

Maven项目

IntelliJ IDEA使您可以管理Maven项目。您可以链接，忽略项目，同步Maven和IntelliJ IDEA项目中的更改以及配置生成和运行操作。

导航到POM

  1. 在Maven工具窗口中，右键单击一个链接的项目。

  2. 从上下文菜单中，选择“ 跳转到源” F4 。

    IntelliJ IDEA导航到相应的Maven配置文件，然后在编辑器中打开相关的POM。

忽略Maven项目

您可以使用“ 忽略项目”选项停用Maven项目。在这种情况下，IntelliJ IDEA将忽略的Maven项目和子项目保留在Maven工具窗口中，但停止将其导入（模块，内容根，目标等）到项目中。但是，IntelliJ IDEA会将被忽略的项目与当前项目同步。如果您需要跳过不相关的子项目，这可能会有所帮助。

  1. 在Maven工具窗口中，右键单击要忽略的项目。

  2. 从上下文菜单中，选择“ 忽略项目” 。

  3. 如果要从“ 项目”工具窗口中删除项目，请在打开的窗口中单击“ 是” 。

如果要激活Maven项目或子项目，请从上下文菜单中选择“ 忽略项目 ”。

重新导入一个Maven项目

如果在“ Maven设置”对话框中选择“ 自动导入Maven项目”选项，则每次对pom.xml文件进行更改时，IntelliJ IDEA都会自动重新导入项目。

但是，如果要控制项目的导入，则可以手动触发操作。

  1. 在Maven工具窗口中，右键单击一个链接的项目。

  2. 从上下文菜单中，选择“ 重新导入” 。

    调用此操作后，IntelliJ IDEA会在Maven工具窗口中解析项目结构。

    IntelliJ IDEA无法重新导入项目的一部分，而是重新导入整个项目，包括子项目和依赖项。

    如果您通过“ 项目结构”对话框配置依赖项（请单击 ，从主菜单），该依赖项将仅出现在IntelliJ IDEA 项目工具窗口中，而不出现在Maven工具窗口中。请注意，下一次重新导入项目时，IntelliJ IDEA将删除添加的依赖项，因为IntelliJ IDEA将Maven配置视为单个事实来源。

要重新导入所有Maven项目，请单击在工具栏上的Maven工具窗口中。

链接和取消链接Maven项目

一个IntelliJ IDEA项目中可以有多个Maven项目。如果将部分代码保留在不同的项目中，或者需要处理一些旧项目，这可能会有所帮助。您可以在IntelliJ IDEA中链接此类项目并同时进行管理。您也可以从Maven结构中快速删除此类项目。

链接一个Maven项目

  1. 打开Maven工具窗口。

  2. 在Maven工具窗口中，单击附加一个Maven项目。

  3. 在打开的对话框中，选择所需的pom.xml文件，然后单击OK 。

    该项目已链接。Maven工具窗口显示工具栏和Maven实体的树状视图。

取消链接Maven项目

当您取消链接Maven项目时，IntelliJ IDEA会删除所有相关项目和内容根，从Maven工具窗口和Project工具窗口中都删除Maven项目，并停止其同步。如果需要从当前IntelliJ IDEA项目中完全删除以前链接的Maven项目，这可能会有所帮助。

  1. 在Maven工具窗口中，右键单击一个链接的项目。

  2. 从上下文菜单中，选择“ 取消链接Maven项目” （ Delete ）。或者，您可以选择链接的项目，然后单击在工具窗口的工具栏上。

  3. 单击确定 。

将构建和运行操作委托给Maven

默认情况下，IntelliJ IDEA使用本机IntelliJ IDEA构建器来构建Maven项目。如果您使用的是纯Java或Kotlin项目，则可能会有所帮助，因为IntelliJ IDEA支持增量构建 ，从而显着加快了构建过程。但是，如果您有一个配置来动态更改编译，或者您的构建生成具有自定义布局的工件，那么Maven对于构建过程将是更可取的。

用Maven建立一个项目

  1. 请点击在Maven工具窗口中。或者，从主菜单中选择File | File。设置/首选项|生成，执行，部署|生成工具| Maven 。

  2. 单击Maven，然后从列表中选择Runner 。

  3. 在Runner页面上，选择Delegate IDE build / run action to maven 。

  4. 单击确定 。

  5. 从主菜单中，选择Build |建设项目 Ctrl+F9 。IntelliJ IDEA调用适当的Maven目标。

  6. 请点击在状态栏上查看，以在“ 构建”工具窗口中查看同步结果。

使用Maven运行和调试

选择对Maven的Delegate IDE构建/运行操作时 ，IntelliJ IDEA使用Maven运行并调试代码。HotSwap也将被触发，并且在调试过程中将重新加载类。

您可以像在其他任何项目中一样使用常规的运行和调试操作。

  1. 根据要使用的操作，从主菜单中选择“运行” |“运行”。运行或运行|调试 。

  2. 如果要调试代码，请在“ 运行”工具窗口或“ 调试 ”工具窗口中检查结果。例如，当您运行main方法时Ctrl+Shift+F10在您的Java项目中，IntelliJ IDEA使用Maven运行该类。
