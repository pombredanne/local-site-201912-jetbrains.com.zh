制表符和缩进

项目

描述

使用制表符

  • 如果选中此复选框，则使用制表符：

    • 在按Tab键

    • 缩进

    • 用于代码重新格式化

  • 清除复选框后，IntelliJ IDEA将使用空格而不是选项卡。

智能标签

  • 如果选中此复选框，则由代码块嵌套定义的缩进部分由制表符和（如有必要）空格组成，而由对齐方式定义的缩进部分仅由空格组成。

  • 如果清除此复选框，则仅使用选项卡。这意味着将将一组适合指定制表符大小的空格自动替换为一个制表符，这可能会导致中断精细对齐。

如果选中了“ 使用选项卡字符”复选框，则“ 智能选项卡”复选框可用。

标签尺寸

在此字段中，指定选项卡中包含的空格数。

缩进

在此字段中，为每个缩进级别指定要插入的空格数（如果选中了“ 使用制表符”复选框，则为制表符 ）。

缩进空行

如果选中此复选框，则IntelliJ IDEA将使空行缩进，就好像它们包含一些代码一样。
如果清除此复选框，则IntelliJ IDEA将删除制表符和空格。

标签缩进

在此字段中，指定要在label语句之前的下一行插入的空格数（如果选中了“ 使用制表符”复选框，则为制表符 ）。

绝对标签缩进

如果选中此复选框，则标签缩进将计为绝对空格数。否则，标签缩进相对于先前的缩进级别进行计数。

空间

使用此选项卡可以指定代码中需要空格的位置。要使IntelliJ IDEA自动在某个位置插入空格，请在列表中选中该位置旁边的复选框。结果显示在“ 预览”窗格中。

包装和大括号

在此选项卡中，定制代码样式选项，IntelliJ IDEA将在重新格式化源代码时应用这些样式选项。左侧窗格包含异常列表（ 重新格式化时保留 ），以及各种代码结构（列表，语句，操作，注释等）的放置和对齐选项。右侧窗格显示预览。

对齐优先于缩进选项。

右边距（列）

使用“ 硬包装在”字段可以指定元素右侧所需的页边空白空间。如果选择默认选项，则使用全局设置中右边距的值。

包装打字

使用打字设置环绕到指定编辑的文本是怎样安装在指定的硬包装的 。您可以选择以下选项之一：

  • 默认值 ：在这种情况下，IntelliJ IDEA使用全局设置中指定的“ 键入时自动换行”选项。

  • 是 ：在这种情况下，IntelliJ IDEA使用“ 右边界”字段中指定的值。

  • 否 ：在这种情况下，此选项已关闭，并且一行可以超出右边界中指定的值。

视觉指南

使用“ 视觉参考线”字段来指定多个右边距。您可以保留默认值，也可以输入空白。如果要指定多个边距，请输入用逗号分隔的数字。

重新格式化时保持

使用复选框来配置IntelliJ IDEA在重新格式化源代码时将产生的异常。例如，默认情况下，“ 换行符”复选框处于选中状态。如果您的代码包含比标准约定短的行，则可以在重新格式化源代码之前通过禁用换行符复选框来转换它们。

包装选项

包装样式适用于在左侧窗格中指定的各种代码构造（例如，方法调用参数或赋值语句）。

包装方式

从此列表中选择所需的包装样式：

  • 不包裹 ：选择此选项时，不应用特殊的包裹样式。

    选择此选项后，嵌套的对齐方式和花括号设置将被忽略。

  • 如果很长则自动换行：选择此选项可使行超出正确的边距并用适当的缩进包裹。

  • 总是换行 ：选择此选项可使列表中的所有元素都换行 ，以便每行有一个具有适当缩进的元素。

  • 如果时间长，则切短 ：选择此选项可使列表中的元素超出右边距的范围，从而使每行有一个带有适当缩进的元素。

大括号放置样式

使用此列表可以在类声明 ， 方法声明和其他类型的声明中指定左括号的位置。可用的选项有：

  • 行尾 ：选择此选项可将开括号放在声明行的末端。

  • 如果被换行，则换行 ：选择此选项可将开括号放在多行声明行之后的行首。

  • 下一行 ：选择此选项可将开头括号放在声明行之后的行开头。

  • 转移下一行 ：选择此选项可在将声明行转移到相应的缩进高度后，将开括号放在该行。

  • 每次移动下一行 ：选择此选项可在将声明行移动到相应的缩进级别之后将开括号放在该行，并将下一行也移动到下一个缩进级别。

    进口货

    下表列出了优化导入时要执行的操作。

    项目

    描述

    一般

    在此区域中，配置常规导入选项。

    选项：

    • 使用单类导入 ：选中此复选框可使IntelliJ IDEA在代码生成或导入优化期间仅从包中导入特定的类。否则，将插入一个导入整个程序包的语句。

    • 使用完全限定的类名称 ：选中此复选框，以使IntelliJ IDEA使用要在代码生成或导入优化过程中导入的类的完全限定名称。否则，将插入普通的import语句。

    • 插入内部类的导入 ：选中此复选框可使IntelliJ IDEA为代码中引用的内部类创建导入。

    • 在Javadoc中使用完全限定的名称 ：选中此复选框可使IntelliJ IDEA在Javadoc中使用完全限定的类名称。否则，将导入一个类。

    • 要使用带'*'的导入的类计数 ：在此文本字段中，指定要从单个包中导入的类的数量，直到所有导入单个类的语句都替换为导入整个包的语句。

    • 名称计数要使用带'*'的静态导入 ：在此字段中，指定要从单个类导入的成员数，直到所有导入单个成员的语句都替换为导入整个类的语句。

    JSP导入布局

    在此区域中，配置应如何在代码中组织JSP导入语句。引入的更改将显示在下面的“ 预览”窗格中。

    选项：

    • 首选用逗号分隔的导入列表 ：选择此选项可导入以逗号分隔的列表组织的语句。

    • 首选每页指令一个导入语句 ：选择此选项以每行创建一个导入语句。

    使用“ *”导入的软件包

    在此区域中，配置要始终完全导入的包和类的列表。

    选项：

    • 静态 ：如果要为所选类声明静态导入，请选中此复选框。

    • 包 ：在此列的文本字段中，指定始终完全导入的包和类。

      请注意，IntelliJ IDEA会自动添加.*完成输入后，在包名称的末尾输入。

    • 使用子包：选中此复选框可完全导入所选包的所有子包。

    • 添加软件包 ：单击此按钮可将新条目添加到软件包和类列表中。

    • 添加空白 ：单击此按钮可以在包和类列表中添加一个空的分隔符。

    • 删除 ：单击此按钮可以从列表中删除所选的软件包或类。

    导入布局

    在此区域中，配置如何在代码中组织导入语句。您可以将某些类设置为首先放置，最后放置或一个接一个放置。导入的类将按照其包进行分组，并在包中按字母顺序排序。

    选项：

    • 分别布局静态导入 ：如果选中此复选框，则所有静态导入将保留在单独的部分中。否则，将根据指定的布局规则对所有import语句进行排序。

    • 静态 ：如果要为所选软件包声明静态导入，请选中此复选框。

    • 包 ：在此列的文本字段中，指定要导入的包。

    • 使用“子包” ：选中此复选框，以使IntelliJ IDEA将布局规则应用于所选包的所有子包。

    • 添加软件包 ：单击此按钮可将新条目添加到软件包列表中。

    • 添加空白 ：单击此按钮可在所选条目后插入空白行，这表示应在相应的导入语句之间插入空白行。

    • 上移/下移 ：单击这些按钮可在列表中上下移动包或空白行，从而定义导入语句的顺序。

    • 删除 ：单击此按钮可以从列表中删除选定的软件包。
