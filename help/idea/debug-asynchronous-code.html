调试异步代码

调试异步代码是一个挑战，因为执行在不同的帧之间跳转，并且使跟踪代码变得更加困难。为了帮助您从代码中的特定点进行回顾，IntelliJ IDEA提供了捕获点 。

捕获点是代码中的一个位置，调试器将捕获堆栈跟踪，以便稍后在代码中到达特定点（插入点）并想知道如何到达时使用。捕获点由方法名称（和包含的类）和键表达式指定。当调试器停止时，它将开始使用插入点 （这是另一种方法和表达式）来匹配堆栈帧。如果找到匹配项，则调试器将评估表达式，并且如果该值具有一些相关的堆栈信息，它将使用捕获的堆栈替换其余的调用堆栈。这可以帮助您了解相关捕获点正在发生的事情，以及如何到达当前点。

默认情况下，启用异步堆栈跟踪（ “设置/首选项” |“构建”，“执行”，“部署” |“调试器” |“异步堆栈跟踪” ）。内置了最常见的捕获点，因此无需进行配置。

使用异步注释

如果需要默认配置中未包含的捕获点，则可以使用可以在代码中引用的注释。

为您的工件向JetBrains Maven Java注释存储库 添加依赖项 ，并使用@Async.Schedule和@Async.Execute代码中的注释。

当点带有注释时@Async.Schedule达到后，当前堆栈将保存在存储映射中。当点带有注释时@Async.Execute达到后，调试器将尝试在存储映射中找到相关的堆栈。

您可以注释：

  • 方法： this值用作键

  • 参数：参数值用作键

定义自定义注释

如果由于某种原因您不想将JetBrains Maven存储库添加为项目的依赖项，则可以定义自己的注释并使用它们来代替默认注释。

  1. 为捕获点和插入点创建自己的注释（可以使用Async.java作为参考）。

  2. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到Build |执行|部署|调试器异步堆栈跟踪 ，然后单击配置注释 。

  3. 在“ 异步注释配置”对话框中，单击将您的自定义注释添加到“ 异步计划”注释和“ 异步执行”注释中 。

手动配置捕获点

如果由于某种原因您不能使用注释，或者需要能够捕获局部变量，则可以手动配置捕获点。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S转到构建，执行，部署|调试器异步堆栈跟踪 。

  2. 请点击添加新的捕获点，并输入与捕获点和插入点有关的信息。

    例如， javax.swing.SwingUtilities.invokeLater与doRun键将捕获所有的调用invokeLater方法并将它们与Runnable实例参数。 java.awt.event.InvocationEvent.dispatch与runnable插入键将插入捕获的信息invokeLater到哪里Runnable被执行。

  3. （可选）如果还希望捕获局部变量（基元和字符串值以及调用堆栈），请选择“ 捕获局部变量”选项。请注意，这可能会减慢调试过程，并且如果启用了Instrumentation Agent，则此选项不可用。

在远程JVM中查看异步堆栈跟踪

如果要调试远程进程（例如在Docker容器中管理的远程进程），则仍可以使用JVM Instrumentation Agent显示异步堆栈跟踪，就好像它是从IDE启动的一样。

要远程使用代理，请执行以下操作：

  • 将/lib/rt/debugger-agent.jar复制到远程计算机上的任何位置

  • 加-javaagent:到远程JVM选项
