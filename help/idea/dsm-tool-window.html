查看|工具窗口|帝斯曼

  • 执行DSM分析后，此工具窗口将变为可用。

  • 确保已安装并启用DSM Analysis插件。请参阅“设置”对话框的“ 插件”页面。

在这里，您可以看到典型的矩阵视图。

行标题代表程序结构。现在一切都折叠了，仅显示模块。展开时，标头类似于树，使您可以扩展模块并深入研究程序包。 *-节点将包内的类分组。列标题与相应的行标题相同。因此，为了节省空间，未显示它们。相反，在行标题上使用了不同的视觉辅助工具。

如果选择一行，矩阵将如下所示。

在这里您可以学习以下内容：

  1. 选中的行和相应的列将突出显示以可视化行依赖性。

  2. 单元格中的省略号表示maven-core模块具有许多（超过99个）依赖项maven-project模块。

  3. 栏示出所选择的行的相关性。

  4. 该行显示了对所选行的依赖关系。

  5. 这意味着maven-project模块有16个依赖项maven-settings模块。

  6. 各种阴影对应于依赖项的数量。

  • 颜色注释有助于一目了然地可视化行依赖性。

  • maven-core依赖于取决于maven-project 。

  • maven-project依赖于取决于maven-profile 。

  • 对角线上的破折号对应于未显示的自相关项。

您可以选择任何单元以浏览其中指示的依赖关系。

选择单元格＃1。这些颜色注释表示maven-project 有 16依赖性maven-settings 。对称单元格（单元格2）显示了另一个方向的依赖关系-在这种情况下为零。

有一个简单的记忆规则-所有的依赖关系总是从绿流入黄河。

DSM视图不是按字母顺序对行进行排序，而是以一种特殊的方式对依赖项进行排序：将最常用的类移到底部。在结构良好的项目中，这会在矩阵的左下半部分创建一个三角形。

周期数

相互依赖性显示为红色。这意味着插件和可用性软件包都相互依赖。

模式

有两种类型的视觉模式。垂直线表示聚合器 。

水平线出现在最低级别或实用程序功能中。
