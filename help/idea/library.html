图书馆

库是模块可以依赖的已编译代码的集合。在IntelliJ IDEA中，可以在三个级别上定义库： 全局 （可用于许多项目）， 项目 （可用于项目中的所有模块）和模块 （可用于一个模块）。

Java库可以包含类文件，包含类文件的存档和目录，以及包含Java本机库.dll ， .so或.jnilib的目录 。

定义一个库

定义库并将其添加到模块依赖项之后，IDE将在您编写代码时向您提供其内容。IntelliJ IDEA还将使用库中的代码来构建和部署您的应用程序。

您也可以从位于项目内容根目录内的JAR文件创建新库。在“ 项目”工具窗口中选择这些文件，然后从上下文菜单中选择“ 添加为库 ”。

定义一个全局库

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S 。

  2. 在“ 平台设置”下 ，选择“ 全局库” 。

  3. 请点击并选择要如何添加新库：您可以从计算机上的文件中添加Java和Kotlin库，或者从Maven下载库。

定义项目库

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S 。

  2. 在“ 项目设置”下 ，选择“ 库” 。

  3. 请点击并选择要如何添加新库：您可以从计算机上的文件中添加Java和Kotlin库，或者从Maven下载库。

定义模块库

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S 。

  2. 在项目设置下 ，选择模块|。依赖关系 。

  3. 点击按钮，然后选择库 。

  4. 单击对话框底部的“ 新建库” ，然后选择要如何添加新库：您可以从计算机上的文件中添加Java和Kotlin库，也可以从Maven下载库。

从Maven下载库

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S然后单击库 。

  2. 请点击然后选择From Maven 。

  3. 在下一个对话框中，指定库工件（例如， org.jetbrains:annotations:16.0.2 ）。如果您不知道其确切名称，请输入关键字并单击 。

    您还可以指定另一个库位置，然后选择是否要下载传递依赖项，源文件，JavaDoc文件或注释。

IntelliJ IDEA将从Maven或Nexus公共存储库下载该库。您还可以配置自定义远程存储库。

配置自定义远程存储库

您可以查看远程存储库的完整列表，并在设置中添加自定义存储库。请注意，即使您不将Maven用作项目的构建工具，IntelliJ IDEA也可以从Maven加载库。

  1. 在“ 设置/首选项”对话框中Ctrl+Alt+S ，转到“ 构建，执行，部署” |远程Jar存储库 。

  2. 单击相应对话框部分中的添加 ，然后指定存储库URL。

将库添加到模块依赖项

全局库和项目库在将它们添加到模块依赖项之前不可用：

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S ，然后选择模块 。

  2. 打开“ 依赖关系”选项卡，然后单击 。

  3. 选择要添加的库，然后单击对话框底部的“ 添加所选内容”。

包括特定的传递依赖项

如果只想包含特定的传递依赖项，则可以使用库属性编辑器。

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S ，然后转到模块|依赖关系 。

  2. 选择必要的Maven库，然后单击 。

  3. 在下一个对话框中，单击“ 编辑” ，然后单击“ 包括传递依赖项”选项旁边的“ 配置” 。

  4. 选择要包含在库中的依赖项，然后单击“ 确定” 。

排除图书馆项目

IntelliJ IDEA允许您暂时排除库项目以提高IDE性能。您可以排除文件夹，档案（例如JAR）和档案中的文件夹。

排除软件包中的类将不会显示在代码完成建议列表中，对此类的引用将在编辑器中显示为“未解决”等。但是，在编译或运行代码时，无论该库中是否有排除的项目，库仍将作为一个整体使用。

从项目或全局库中排除项目

  1. 按Ctrl+Shift+Alt+S打开“ 项目结构”对话框，然后单击“ 库” 。

  2. 请点击 ，然后选择要排除的库项目。

从模块库中排除项目

  1. 按Ctrl+Shift+Alt+S打开项目结构对话框，然后转到模块|依赖关系 。

  2. 请点击 ，然后选择要排除的库项目。

排除的项目将标有图标。要将库项目恢复为原始状态，请删除排除的项目。

更改库级别

在IntelliJ IDEA中，您可以将项目或模块库移到更高的级别。如果要扩展其使用范围，这将很有帮助。例如，如果要在项目或整个IDE中使用模块库。要将库移到更高级别，请在“ 项目结构”对话框中右键单击该库，然后选择“ 移至项目库”或“ 移至全局库” 。

您还可以在较低级别上创建库的副本。例如，如果要向项目库中添加更多类，但只想在一个模块中使用它们。

右键单击所需的全局库，然后选择复制到项目库 。然后指定是否还需要库文件的副本以及应在何处创建该副本。对于模块依赖性中包含的全局或项目库，要开始在模块级别创建副本，请右键单击所需的库，然后选择“ 复制到模块库” 。
