在Zeus服务器上执行测试

以下仅在安装并启用Ruby插件时有效。

先决条件

在开始使用Ruby之前，请确保已安装并启用 Ruby插件。

另外，请确保满足以下先决条件：

  • Ruby SDK已下载并安装在您的计算机上。

  • 所需的框架SDK已下载并安装在您的计算机上。

有关详细信息，请参阅其各自的下载和安装页面：

  • 红宝石
  • Ruby on Rails

基本

运行测试服有时会产生开销，因为每次执行测试服时，都应重新加载整个环境。您可以使用Zeus服务器避免这种情况，该服务器仅加载一次环境。

当Zeus服务器在后台运行时，您可以选择使用Zeus服务器或在本地执行任何测试脚本。

使用Zeus服务器运行测试脚本

  1. 确保使用IntelliJ IDEA启动Zeus服务器，并且该服务器在后台运行。

  2. 运行测试脚本或其示例之一。请注意，在相应的运行配置中会自动选择选项Zeus 。如果要在本地运行此测试，则必须选择选项None 。

宙斯下的调试测试

  1. 确保在要调试的测试脚本中设置了断点。

  2. 在主工具栏上，单击运行/调试配置选择器，然后选择编辑配置 。

  3. 在“ 运行/调试配置”对话框中 ，为Zeus服务器创建运行/调试配置。有关该过程的详细说明，请参考“ 创建和编辑运行/调试配置 ”部分。

  4. 在调试模式下启动Zeus。为此，请选择Zeus运行/调试配置，然后单击 ，或按Shift+F9 。

  5. 在Zeus服务器以调试模式运行的情况下，运行所需的测试脚本。

请注意，如果您调试测试脚本，它将在没有Zeus服务器的情况下执行。

技巧和窍门

  • IntelliJ IDEA为Zeus服务器创建一个临时运行/调试配置。稍后，您可以根据需要更改此运行/调试配置，将其另存为永久文件，然后使用其启动服务器。

  • 如果Zeus和Spork DRb服务器同时运行，则Zeus将获得优先权。

  • 如果要在Zeus服务器下运行RSpec测试，请确保使用Zeus版本0.13.4.pre2或更高版本。
