使用Dockerfile调试Java应用程序

您可以使用IntelliJ IDEA调试在Docker容器中运行的Java应用程序。本教程描述了如何创建一个Dockerfile，以便在具有OpenJDK 8的容器中运行一个简单的Java控制台应用程序，然后通过在源代码中设置断点并使用远程调试配置来对其进行调试。

创建一个示例Java控制台应用程序

本教程的示例应用程序模拟控制台实用程序来更改密码。它要求输入登录名和当前密码。验证凭据后，它要求输入两次新密码。如果密码不匹配，它将再次要求新密码，直到它们匹配为止，然后更改密码。在本教程中，未实现用于验证和更改密码的实际方法。

  1. 使用以下代码创建一个Java类：

  2. 将应用程序构建到项目内部的JAR文件中： out / artifacts / pass / pass.jar

创建一个Dockerfile和一个运行配置以运行该应用程序

  1. 使用以下代码创建一个Dockerfile：

    来自openjdk：8 COPY。 / tmp WORKDIR / tmp CMD [“ java”，“-jar”，“ pass.jar”]

  2. 请点击在装订线中，选择“ 新建运行配置” 。

  3. 在“ 编辑运行配置”对话框中，将上下文文件夹更改为out / artifacts / pass / （构建JAR工件的位置），为容器添加一个名称（例如pass ），并指定运行应用程序的命令： java -jar pass.jar

  4. 单击运行以检查该应用程序是否正常运行。容器启动后，在Docker工具窗口中将其选中，然后打开Attached Console选项卡。您应该看到以下提示： Enter your login: 。

创建远程调试配置

调试器使用远程调试配置附加到应用程序。

  1. 在主菜单上，选择运行|。编辑配置 。

  2. 请点击然后选择远程 。

  3. 在启动前部分中，单击并选择在调试之前启动Docker 。

  4. 选择运行您的应用程序的Docker配置。确保“ 配置Docker”对话框中的端口和远程调试配置中的端口匹配。单击确定 。

  5. 在“ 运行/调试配置”对话框中单击“ 确定 ” 。

设置断点并调试您的应用程序

此时，您应该具备以下条件：

  • 您的应用程序的源代码（ Java_pass_from_console类文件）

  • 内置的JAR文件位于out / artifacts / pass / pass.jar中

  • 使用Docker运行配置运行应用程序的Dockerfile

  • 远程调试配置，该配置首先启动Docker运行配置，然后将其附加到已启动容器中的应用程序

在开始调试配置之前，您需要在源代码中设置断点并重建JAR工件。
