用PHPSpec测试

使用IntelliJ IDEA，您可以通过使用PHPSpec工具集运行规范来练习行为驱动的开发。

在你开始前

  1. 确保已安装并启用PHP和PHPSpec插件。 这些插件未与IntelliJ IDEA捆绑在一起，但可以如管理插件中所述从JetBrains插件存储库安装。启用后，这些插件将在IDE级别可用，也就是说，您可以在所有IntelliJ IDEA项目中使用它们。

  2. 确保PHP解释器中的IntelliJ IDEA配置的PHP页面 ，如在配置本地PHP解释器和配置远程PHP解释器 。

使用Composer安装PHPSpec

在你开始前， 确保Composer已安装在计算机上并按照Composer Dependency Manager中所述在当前项目中初始化。

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择phpspec / phpspec软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

从PHPSpec官方网站了解有关PHPSpec安装的更多信息。

将PHPSpec与IntelliJ IDEA项目集成

如果使用本地PHP解释器 ，则IntelliJ IDEA会自动执行初始PHPSpec配置。对于远程PHP解释器 ，需要手动配置PHPSpec。

自动配置PHPSpec

  1. 将phpspec.yml或phpspec.yml.dist配置文件存储在项目根目录下。

  2. 使用Composer安装PHPSpec 。

IntelliJ IDEA将在“ 测试框架”页面上创建本地框架配置以及PHPSpec运行/调试配置。

手动配置PHPSpec

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，展开Languages and Frameworks节点，然后选择PHP下的Test Frameworks 。

    在打开的“ 测试框架”页面上，单击在中央窗格中，然后从列表中选择配置类型：

    • 在本地配置中，使用默认项目PHP解释器 ，有关详细信息，请参见默认项目CLI解释器 。

    • 要将PHPSpec与远程PHP解释器一起使用，请在打开的对话框中选择一种配置：

  2. 在“ PHPSpec可执行文件的路径”字段中，指定phpspec的位置。PHPSpec不一定必须安装在当前项目的根目录下。 请点击 PHPSpec目录路径或phar文件字段旁边 。IntelliJ IDEA检测到PHPSpec的版本并将其显示在字段下方。

    如果未为本地解释器指定到PHPSpec的路径，则IntelliJ IDEA不提供对PHPSpec的完全支持，例如，它不显示代码完成建议，也不解析引用。

  3. 在“ 测试运行器”区域中， 指定配置YML文件以用于启动和执行规范。

    默认情况下，PHPSpec在项目根文件夹中查找phpspec.yml或phpspec.yml.dist配置文件。您可以指定一个自定义配置文件。

    • 清除默认配置文件复选框使PHPSpec使用项目根文件夹中的phpspec.yml或phpspec.yml.dist配置文件。如果找不到这样的文件，则测试执行失败，因此显式指定配置文件可能更可靠。

    • 选择默认配置文件复选框指定您自己的YML配置文件。此文件以后将在所有PHPSpec运行/调试配置中用作默认文件。

      在该字段中，指定要使用的配置文件的位置。手动输入路径或单击然后在打开的对话框中选择文件。

    在本地配置中， Prefix（'spec_prefix'）：只读字段显示规范的名称空间前缀。IntelliJ IDEA检测spec_prefix从默认配置文件字段中指定的配置文件中。默认值为spec 。有关详细信息，请参见PHPSpec配置：PSR-4和PHPSpec配置：规范和源位置 。

为类生成PHPSpec测试

  1. 通过执行以下任一操作，打开“ 创建新的PHP测试”对话框：

    • 在主菜单中，选择“ 文件” |“文件”。新的 。然后，选择PHP Test |上下文菜单中的PHPSpec Spec 。

    • 在项目工具窗口中，按Alt+Insert或右键单击要测试的PHP类，然后选择“ 新建| PHP测试| PHPSpec规范 。

    • 在要测试的PHP类的编辑器中，将插入号放置在类的定义内。然后，选择转到|从上下文菜单进行测试或按Ctrl+Shift+T并从弹出菜单中选择创建新测试 。这样，您可以为一个PHP文件中的多个类之间定义的PHP类生成测试。

      要为特定方法创建测试，请将插入号放置在方法声明中。所选方法将在“ 创建新PHP测试”对话框的“ 生成测试方法”区域中自动选择。

  2. 将打开“ 创建新的PHP测试”对话框。

    提供生成的测试的参数：

    • 测试文件模板，即IntelliJ IDEA将基于其生成测试类的模板。确保在“ 测试文件模板”列表中选择了PHPSpec 。

    • 测试类的名称。IntelliJ IDEA从生产类名称中自动将名称组成为Spec.php 。

    • 根据生产类的包含目录和名称空间，配置的测试源root及其psr-4包前缀或Windows Server 2003的名称，将自动建议测试类文件的文件夹。 src_path在phpspec.yml配置文件中指定的值。请注意，要使自动文件夹建议正常工作，测试套件和生产类名称空间必须匹配。

      要指定其他文件夹，请单击在目录字段旁边，然后选择相关文件夹。

    • 测试类将属于的名称空间，将根据生产类的包含目录和名称空间，配置的测试源root及其psr-4包前缀或名称自动建议该名称空间。 spec_prefix和namespace在phpspec.yml配置文件中指定的值。

    • 生产类方法为其生成测试方法存根。在“ 生成测试方法用于”区域中，选中所需生产类方法旁边的复选框。要包括从父类继承的方法，请选中显示继承的方法复选框。

      IntelliJ IDEA将自动组合测试方法的名称为test 。您可以在“ 文件和代码模板设置”页面的“代码”选项卡上自定义用于生成测试方法存根的代码模板 。

测试准备就绪后，可以通过选择导航|返回到生产类。去测试主题 。有关详细信息，请参阅在测试和测试主题之间导航 。

如果要依靠PHPSpec配置文件为测试类提供包含的文件夹和名称空间，请确保按照在将PHPSpec与IntelliJ IDEA项目集成中所述在“ 测试框架”页面上选择了它。

运行和调试PHPSpec测试

有关编写PHPSpec规范的信息 ，请参见PHPSpec文档 。要运行或调试测试，请执行以下一项操作：

运行或调试PHPSpec测试

  • 在“项目”工具窗口中，选择要从中运行测试的文件或文件夹，然后从所选内容的上下文菜单中选择“运行”或“ 调试” 。

    IntelliJ IDEA生成默认的运行配置，并使用它启动运行/调试测试会话。

保存自动生成的默认配置

  • 测试会话结束后，选择保存从文件或文件夹的上下文菜单中。

通过先前保存的运行/调试配置运行或调试测试

  • 从工具栏上的列表中选择所需的PHPSpec配置，然后单击要么 。

创建自定义运行/调试配置

  1. 在“项目”工具窗口中，选择要运行测试的文件或文件夹，然后从上下文菜单中选择“ 创建运行配置 ”。或者，选择运行|从主菜单中编辑配置 ，然后单击然后从列表中选择PHPSpec 。

  2. 在打开的“运行/调试配置：PHPSpec”对话框中，指定要运行的方案，选择要使用的PHP解释器 ，并通过指定要传递给PHP可执行文件的选项和参数来自定义其行为。

监控测试结果

IntelliJ IDEA在“ 运行工具”窗口的“ 测试运行器”选项卡中显示测试执行结果。

该选项卡分为2个主要区域：

  • 左侧区域使您可以深入浏览所有单元测试，以查看成功和失败的单元测试。您可以过滤测试，导出结果，并使用上下文菜单命令运行特定的测试或导航至源代码。

  • 右侧区域显示原始PHPSpec输出。
