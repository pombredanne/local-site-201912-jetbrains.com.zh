新的Android组件对话框

工程工具窗口|具有应用程序类的包的上下文菜单|新增-Android组件

在此对话框中，指定实现新的Android组件的类，组件类型以及要显示给用户的标题。

项目

描述

名称

在此文本框中，键入实现组件的类的名称。

类

在此下拉列表中，指定组件类型。可用的选项有：

  • AIDL ：用于进程间通信的Android接口定义语言（AIDL）接口。

  • 活动 ：实现一个窗口，您可以在其中放置UI与用户进行交互。

  • Android Auto ：可让您扩展应用程序以用于车辆。您可以添加媒体服务或消息服务活动。

  • 文件夹：根据您为此组件选择的活动创建源根。

  • 片段 ：代表行为或行为在用户界面中的一部分。

  • Google：可让您为Google地图和AdMob Ads活动创建活动。

  • 应用程序 ：一个Android程序包，即一个.apk存档，其中包含一个Android应用程序和安装程序的内容。

  • 服务 ：表示应用程序希望执行操作而不与用户交互，或者为其他应用程序提供功能的愿望。

  • 其他：让您将以下组件添加到您的应用程序：
    • Android清单文件
    • 广播接收器
    • 内容提供商
    • 白日梦

  • UI组件 ：可让您向应用程序添加自定义视图。

  • Wear ：可让您扩展应用程序以用于Android Wear。

  • 小部件 ：您可以为应用程序添加不同类型的小部件。

  • XML ：可让您为Android布局和值添加不同类型的.xml文件。

标签

在此文本框中，键入将显示给用户的标题。

  • 如果未指定标题，将显示分配给整个应用程序的标签。

  • 没有为远程接口组件指定标签。

标记为启动活动

选中此复选框可在应用程序启动时默认显示活动类型的新组件。

仅在“ 种类”下拉列表中指定“ 活动”组件类型时，此复选框才可用。

创建布局文件

  • 选中此复选框可使IntelliJ IDEA生成相关布局定义文件的存根，即新活动或片段的内容视图。

  • 如果清除该复选框，则必须手动创建新活动或片段的内容视图。

该复选框仅对活动或片段组件类型可用。
