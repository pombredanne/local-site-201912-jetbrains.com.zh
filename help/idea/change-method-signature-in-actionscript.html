在ActionScript中更改方法签名

在ActionScript中，可以使用“ 更改方法签名”重构来：

  • 更改方法名称和返回类型。

  • 添加新参数并删除现有参数。请注意，您还可以使用专用的“ 提取参数”重构来添加参数。

  • 重新排序参数。

  • 更改参数名称和类型。

  • 通过方法调用层次结构传播新参数。

初始化程序，默认值和新参数的传播

对于添加到函数的每个新参数，您可以指定：

  • 用于初始化参数的值（或表达式）（IntelliJ IDEA中的Initializer字段）。

  • 默认值（或表达式）（“ 默认值”字段）。

您还可以将已引入的参数传播到调用要更改其签名的函数的函数。

重构结果取决于您是否指定这些值并使用传播。

传播。 新参数可以传播到调用您要更改其签名的函数的任何函数。在这种情况下，通常，调用函数的签名会相应更改。但是，这些更改还取决于初始化程序和为新参数设置的默认值的组合。

初始化程序。 在Initializer字段中指定的值将作为默认参数值添加到函数定义中。这使相应的参数成为可选参数。（请参阅Flex / ActionScript文档中有关必需和可选参数的讨论。）

如果未指定新参数的默认值 （在“ 默认值”字段中），则无论是否使用传播，函数调用和调用函数的签名均不会更改。

如果同时指定了初始化程序和默认值，则重构结果取决于是否使用传播：

  • 如果不使用传播，则初始化器值不会影响函数调用和调用函数的签名。

  • 如果使用传播，则初始化程序值将作为相应参数的默认值添加到调用函数的定义中（与更改其签名的函数中的方法相同）。

默认值。 通常，这是要添加到函数调用中的值。

如果新参数未传播到调用函数，则此类函数内的函数调用也将使用此值。

如果使用传播，则此值对于调用函数内的函数调用无关紧要。

更改方法签名

  1. 在编辑器中，将光标置于要更改其签名的方法的名称内。

  2. 请执行以下任一操作：

    • 按Ctrl+F6 。

    • 选择重构|在主菜单中更改签名 。

    • 选择重构|从上下文菜单中更改签名 。

  3. 在“ 更改签名”对话框中 ，对方法签名进行必要的更改，并指定还需要进行哪些其他相关更改。

    您可以：

    • 通过编辑“ 返回类型”字段的内容来更改方法的返回类型 。

      在此字段以及包含函数参数的表的某些字段中，都可以使用代码完成功能。

    • 更改方法名称。为此，请在“ 名称”字段中编辑文本。

    • 使用参数表及其右边的按钮来管理方法参数：
      • 要添加新参数，请单击Alt+Insert并在相应字段中指定新参数的属性。

        添加参数时，您可能需要将这些参数传播到调用当前方法的方法。

      • 要删除参数，请单击相应行中的任何单元格，然后单击Alt+Delete 。

      • 要重新排序参数，请使用Alt+Up和Alt+Down 。例如，如果您想将某个参数设为列表中的第一个参数，则可以单击该参数对应的行中的任何单元格，然后单击所需的次数。

      • 要更改参数的名称或类型，请在相应的表格单元格中进行必要的编辑。

    • 沿调用当前方法的方法层次结构传播新的方法参数（如果有）。

      （可能有一些方法调用要更改其签名的方法。这些方法又可以被其他方法调用，依此类推。您可以通过调用方法的层次结构传播对当前方法的参数所做的更改，还可以指定哪些调用方法应该受到影响，哪些不应该受到影响。）

      传播新参数：

      1. 请点击Alt+G 。

      2. 在“ 选择方法以传播新参数”对话框的左侧窗格中，展开必要的节点，然后选中要将新参数传播到的方法旁边的复选框。

        为了帮助您选择必要的方法，对话框右侧部分分别显示了调用方法和被调用方法的代码（分别在“调用方方法”和“被调用方 方法”窗格中）。

        当您在左侧窗格中的方法之间切换时，右侧部分中的代码将相应更改。

      3. 单击确定 。

  4. 要立即执行重构，请点击重构 。

    要查看预期的更改并在实际执行重构之前进行必要的调整，请单击“ 预览” 。
