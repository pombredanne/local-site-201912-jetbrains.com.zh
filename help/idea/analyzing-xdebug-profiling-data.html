分析Xdebug分析数据

仅当安装并启用PHP插件时，以下内容才有效。

启用与Xdebug探查器的集成后，IntelliJ IDEA将提供探查器快照的直观表示。IntelliJ IDEA将打开一个单独的编辑器选项卡，该选项卡具有四个视图，这些视图根据不同的条件显示数据。

要收集和分析配置文件数据，请执行以下常规步骤：

  • 启动Xdebug调试会话
  • 检索分析器累积的数据
  • 检查分析数据

启动Xdebug调试会话

请执行以下任一操作：

  • 要开始调试整个应用程序，请创建类型为PHP Web Page的调试配置，然后通过单击启动调试。 。

    有关详细信息，请参见使用PHP网页调试配置进行调试 。

  • 要调试特定的PHP HTTP请求，请定义类型为PHP HTTP Request的调试配置，然后通过单击启动调试。 。

    有关详细信息，请参见调试PHP HTTP请求 。

  • 要启动零配置调试会话：

    1. 切换“ 开始侦听PHP调试连接”按钮在IntelliJ IDEA工具栏上将其更改为 。之后，IntelliJ IDEA开始侦听当前项目中使用的调试引擎的端口。调试端口在“ 语言和框架” | “ IntelliJ IDEA”级别上设置。 PHP |调试页面的 设置/首选项对话框Ctrl+Alt+S 。

    2. 在浏览器中打开应用程序的起始页 ，选择“ 启动调试器”书签以从浏览器激活调试引擎，重新加载当前页面（应用程序的起始页），然后返回IntelliJ IDEA。

检索分析器累积的数据

  1. 在主菜单中，选择“ 工具” |“工具”。分析Xdebug Profiler快照 。

  2. 在打开的“ 选择Xdebug Profiler快照”对话框中，选择存储概要分析数据的文件夹和文件 。

    IntelliJ IDEA在单独的编辑器选项卡中显示收集的性能分析数据，其中包含选定的探查器输出文件的名称。

检查分析数据

当您请求累积的分析数据时，IntelliJ IDEA在单独的编辑器选项卡中打开其可视化演示文稿。该选项卡以选定的探查器输出文件命名，并包含多个视图。在视图之间切换以基于各种分析标准来分析概要分析数据。

  • 在“ 执行统计信息”视图中，检查有关每个调用函数的执行指标的摘要信息。

  • 在“ 调用树”视图中，浏览所有被调用函数的执行路径。

  • 要浏览特定功能的执行路径，请在“ 调用树”视图中选择该功能，然后在“ 被调用者”视图中查看其被调用者 。

  • 要探索可能导致调用特定功能的所有路径，请在“ 调用树”视图中选择该功能，然后在“ 调用者”视图中检查其可能的调用者 。
