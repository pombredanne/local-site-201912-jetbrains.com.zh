从命令行格式化文件

命令行源代码格式化程序是IntelliJ IDEA中的一项特殊功能，可让您格式化项目外部的任意文件。

为了能够格式化文件，请确保已安装并启用了支持所需文件类型的相应插件。

  • 在命令行中运行format使用以下语法命令：
    格式[-h] [-r | -R] [-s |-设置settingsPath] [-m | -mask masks] path1 [path2 ...]

    该命令将启动IntelliJ IDEA，该格式将格式化指定的文件，然后退出。

命令选项如下：

选项

描述

format

位于/ bin目录中的format.bat （在macOS和Linux上为format.sh ）脚本的完整路径。

-h

（可选）显示帮助消息并退出。

-r|-R

（可选）扫描在中指定的目录path1,path2...递归地

-s|-settings settingsPath （可选）根据代码中提供的代码样式设置文件设置代码格式settingsPath 。您可以使用以下之一：

  • 具有导出的代码样式设置的文件： 在里面 设置/首选项对话框Ctrl+Alt+S ，打开编辑器|单击代码样式页，单击 ，然后选择导出 。

  • 项目目录中存储的.idea / codeStyleSettings.xml文件（适用于IntelliJ IDEA 2017.2和更早版本）。

  • 存储在项目目录中的.idea / codeStyles / codeStyleConfig.xml文件（适用于IntelliJ IDEA 2017.3及更高版本）。

如果省略该参数，则将使用默认代码样式设置。

-m|-mask masks

（可选）用逗号分隔的文件掩码列表，用于定义要处理的文件。的* （任何字符串）和? （任何单个字符）通配符受支持。

pathN

要处理的文件或目录的路径。

例子

  1. 使用默认的代码样式设置来格式化C：\ Data \ src目录中的所有文件，包括所有子目录：

    \ bin \ format.bat -r C：\ Data \ src

  2. 使用C：\ Data \ settings.xml中的代码样式设置，以非递归方式格式化C：\ Data \ src目录中的所有.xml和.html文件：

    \ bin \ format.bat -s C：\ Data \ settings.xml -m * .xml，*。html C：\ Data \ src

  1. 使用默认的代码样式设置来格式化〜/ MyProject / src目录中的所有文件，包括所有子目录：

    /bin/format.sh -r〜/ MyProject / src

  2. 使用〜/ Data / settings.xml中的代码样式设置，以非递归方式格式化〜/ MyProject / src目录中的所有.xml和.html文件：

    /bin/format.sh -s〜/ Data / settings.xml -m * .xml，*。html〜/ MyProject / src
