配置覆盖率测量

配置代码覆盖行为

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择构建，执行，部署|覆盖范围 。

  2. 定义如何处理收集的覆盖率数据：

    • 在将覆盖率应用于编辑器之前显示选项 ：每次运行具有代码覆盖率的新运行配置时，都会显示“ 代码覆盖率”对话框。

    • 不要应用收集的coverage ：丢弃新的代码coverage结果。

    • 用新套件替换活动套件 ：丢弃活动套件，并在每次启动具有代码覆盖范围的新运行配置时使用新套件。

    • 添加到活动套件中 ：每次启动具有代码覆盖率的新运行配置时，将新的代码覆盖套件添加到活动套件中 。

    • 忽略隐式构造函数 ：从coverage统计信息中排除隐式声明的默认构造函数。显式声明的默认构造函数将保留在其中。

    • 忽略实用程序类的空私有构造函数 ：从覆盖范围统计信息中排除所有其他方法均为静态的类中的空私有构造函数。

  3. 选择“ 激活险种视图”复选框以自动打开险种工具窗口。

配置代码覆盖率选项

  1. 从主菜单中，选择运行|编辑配置 ，添加必要的运行/调试配置，然后切换到代码覆盖率选项卡。

  2. 在“ 代码覆盖率”选项卡上，从“ 选择覆盖率运行器”列表运行器中选择一个代码覆盖率： EMMA ， JaCoCo或IntelliJ IDEA 。

  3. （对于IntelliJ IDEA运行程序）选择要使用的模式： 采样或跟踪 。

    • 采样模式可使线路覆盖率的降低幅度可以忽略不计。

    • 跟踪模式可准确收集分支覆盖范围，并具有跟踪测试，查看覆盖范围统计信息以及获取每个覆盖行上的其他信息的能力。

    • 跟踪每个测试覆盖率选项允许跟踪每个测试用例产生的单个代码覆盖率。如果您想确切地知道特定测试涵盖了哪些代码，请启用此选项。您还将能够看到与每个代码段最相关的测试。

  4. 要包括在覆盖率数据中的 包和类以及要从覆盖率数据区域中排除的包和类，可以缩小代码的覆盖范围。

    例如，您可以在覆盖率测量中包括或排除特定的类和程序包。

    请点击 （ 添加类别 ）或 （ 添加软件包 ），然后选择必要的项目。

  5. 要收集测试的代码覆盖率统计信息，请选中“ 在测试文件夹中启用覆盖率”复选框。

更改覆盖范围突出显示的颜色

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到编辑器|配色方案|一般

  2. 在组件列表中，展开Line Coverage节点，然后选择Coverage的类型：例如Full ， Partial或Uncovered 。

  3. 单击“ 前景”字段以打开“ 选择颜色”对话框。

  4. 选择一种颜色，然后单击“ 应用” 。
