在编译器上在“ Java编译器”页面上，可以选择要使用的Java编译器并指定关联的选项。

编译器和字节码版本

项目

描述

使用编译器

选择要使用的编译器：

  • Javac 。这可能是IntelliJ IDEA发行版中包含的编译器，也可能是项目JDK之一的编译器。

  • Eclipse（也称为Java或ECJ Eclipse编译器）。IntelliJ IDEA与Eclipse编译器捆绑在一起。

  • Groovy-Eclipse 。该编译器使您可以使用Eclipse编译器执行Groovy和Java代码的联合编译。

  • Ajc （AspectJ编译器）。

    AspectJ编译器未包含在IntelliJ IDEA发行版中，应单独下载。有关更多信息，请参见AspectJ和使用AspectJ编译器（ajc） 。

使用“ --release选项”进行交叉编译（Java 9和更高版本）

默认情况下，此选项处于选中状态。当需要交叉编译时，IntelliJ IDEA会从项目设置中推断出并自动应用--release Java 9的编译器选项。

项目字节码版本

选择要生成的字节码的版本。（大致来说，这是最低目标JVM版本。）

如果未指定特定版本，则由编译器定义字节码版本。

要为特定模块指定不同版本，请使用“每个模块字节码版本”区域中的控件。

每个模块的字节码版本

如有必要，请为各个模块指定目标字节码版本（例如，如果它们应与项目设置的字节码版本不同）。

请点击然后在打开的对话框中选择感兴趣的模块。然后，对于每个模块，单击相应的“ 目标字节码版本”单元格，然后从列表中选择版本。

使用从列表中删除选定的一个或多个模块。

Javac和Eclipse选项

项目

描述

尽可能使用来自模块目标JDK的编译器

对于Javac编译器：

启用此选项并且与模块关联的JDK版本与构建过程JDK的版本不同时 ，将使用模块JDK的编译器。例外是模块JDK的版本早于1.6。在这种情况下，构建过程JDK中的编译器将在交叉编译模式下与模块JDK的类一起使用。

禁用该选项后，所有模块都使用同一编译器编译，而编译器来自构建过程JDK。必要时，使用交叉编译模式。

要开始构建过程，将使用最新的可用JDK。该JDK是从模块中使用的所有JDK，默认项目JDK以及与IntelliJ IDEA捆绑在一起的JDK中选择的。

重要！ 编译器的选择不会影响源代码语言级别，也不会影响字节码目标级别和链接。也就是说，无论使用哪个编译器，字节码都将与与模块关联的JDK链接，并且所得到的代码级别正是您的项目设置中指定的代码级别。

生成调试信息

如果选中此复选框，则编译器将生成在调试器中运行已编译类所需的信息。

报告不推荐使用的功能

如果选中此复选框，则编译器将显示有关在编译过程中遇到的不赞成使用的方法，类或字段的警告。（相应的警告显示在编译器输出窗口中。）

不产生警告

如果选中此复选框，则编译器将忽略有关可疑使用语言构造的警告。

继续进行错误

对于Eclipse编译器：如果选中此复选框，则即使发生编译错误，编译器也会继续进行编译。

ECJ批处理编译器工具的路径（留空以使用捆绑包）

对于Eclipse编译器：如果需要指定特定版本的Eclipse编译器，请使用此选项。

附加命令行参数

指定在启动时传递给编译器的命令行参数和选项。有关可用选项，请参考编译器文档。

如果需要更多输入空间，请单击打开“ 其他命令行参数”对话框，其中文本输入区域较大。

指定参数和选项时，请遵循以下规则：

  • 使用空格分隔各个参数和选项，例如， -client -ea -Xmx1024m 。

  • 如果参数或选项包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg要么"some arg" 。

  • 如果参数或选项包含双引号（例如，作为参数的一部分），请使用反斜杠对双引号进行转义，例如， -Dmy.prop=\"quoted_value\" 。

AJC选项

项目

描述

Ajc编译器的路径

指定ajc的路径（位于\ lib中的文件Aspectjtools.jar ）。

在字段中输入路径，或单击然后在打开的对话框中选择所需的文件。

测试

单击此按钮以检查路径和命令行参数是否正确。

如果一切正常，将显示编译器版本。否则，将显示错误消息。（使用指定的路径和参数，IntelliJ IDEA尝试使用其他路径启动编译器。 -version参数。）

命令行参数

如有必要，请指定要传递给编译器的命令行选项 。

您可以在字段中直接输入参数，或单击打开文本输入区域较大的“ 命令行参数”对话框。

指定JVM选项时，请遵循以下规则：

  • 使用空格分隔各个选项，例如， -client -ea -Xmx1024m 。

  • 如果选项包含空格，则将空格或包含空格的参数括在双引号中，例如， some" "arg要么"some arg" 。

  • 如果选项包含双引号（作为参数的一部分），请使用反斜杠对双引号进行转义，例如， -Dmy.prop=\"quoted_value\" 。

  • 您可以将环境变量值传递给自定义Java属性。例如，如果您定义一个变量MY_ENV_VAR ，您可以将其传递给foo属性如下：

    -Dfoo = $ {MY_ENV_VAR}

注意：当ajc用于编译后编织时，将忽略指定的参数。

另请参阅优化编译性能：将ajc与javac结合使用 ， 控制ajc方面路径，并在模块级别微调ajc的使用 。

生成调试信息

如果选中此复选框，则编译器将生成在调试器中运行已编译类所需的信息。

委托给Javac

如果关闭此选项，则在所有情况下均使用ajc 。

如果启用此选项，则将javac用作ajc的补充或替代。例如， javac将用于编译不包含任何方面的模块。结果，编译可能变得更快。

另请参见优化编译性能：将ajc与javac结合 使用，并在模块级别上微调ajc的使用 。
