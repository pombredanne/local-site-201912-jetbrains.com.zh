PHP_CodeSniffer

仅当安装并启用PHP插件时，以下内容才有效。

IntelliJ IDEA通过与PHP_CodeSniffer工具集成来提供代码样式检查，该工具可验证您的代码与所选编码标准的一致性。您可以指定一种预定义的编码标准，也可以将自己先前定义的编码标准与默认PHP_CodeSniffer的Standards目录之外的根目录一起使用。此外，您可以与团队共享您的自定义编码样式。

要使用来自IntelliJ IDEA的PHP_CodeSniffer而不是命令行，您需要在IntelliJ IDEA中注册它并将其配置为IntelliJ IDEA 代码检查 。在IntelliJ IDEA中安装并启用该工具后，该工具将在任何打开的PHP文件中可用，并且不需要其他步骤即可启动它。文件中的每次更新都会激活即时代码检查，因此很容易消除发现的问题。

由PHP_CodeSniffer即时报告的错误和警告将显示为弹出消息。当该工具以批处理模式运行时，错误和警告将显示在“ 检查结果”工具窗口中。每个消息都有phpcs前缀以将其与IntelliJ IDEA内部检查区分开。IntelliJ IDEA还与PHP Code Beautifier and Fixer工具集成在一起，可让您修复许多检测到的问题。

在你开始前

确保已启用PHP插件。 的该插件未与IntelliJ IDEA捆绑在一起，但可以如管理插件中所述从JetBrains插件存储库安装。

安装和配置PHP_CodeSniffer

PHP_CodeSniffer脚本可用作本地脚本，与PHP解释器关联的脚本或声明为项目依赖项并通过Composer安装的脚本，这是推荐的推荐方法。

IntelliJ IDEA支持PHP_CodeSniffer版本1.5.0和更高版本。

代码质量工具脚本包含对系统范围的PHP引擎的调用。为了能够使用它们，请确保将包含PHP引擎可执行文件的目录添加到系统中path 。

使用Composer安装PHP_CodeSniffer

在你开始前， 确保Composer已安装在计算机上并按照Composer Dependency Manager中所述在当前项目中初始化。

在Composer中安装PHP_CodeSniffer时，IntelliJ IDEA会自动下载必要的脚本，在IDE中注册它们，并有选择地启用和配置相应的代码检查 。

  1. 从composer.json上下文菜单中，选择Composer |。管理依赖项 。或者选择工具|作曲家|从主菜单管理依赖关系 。

  2. 在打开的“ 管理Composer依赖关系”对话框中，从“ 可用软件包”列表中选择squizlabs / php_codesniffer软件包，可能使用搜索字段。

    从“ 要安装的版本”列表中选择相关版本。

  3. 如有必要，请展开“ 设置”隐藏区域并指定高级安装选项。在命令行参数字段中， 键入其他命令行参数。

  4. 点击安装 。

手动配置PHP_CodeSniffer

您可以使用手动下载的本地PHP代码质量工具脚本或与PHP解释程序关联的脚本。可以有许多本地和远程PHP解释器，在Settings / Preferences对话框的PHP页面上指定的一个解释器被认为是Project Default 。在配置远程PHP解释器或配置本地PHP解释器中了解有关配置PHP解释器的更多信息。

选择一个PHP_CodeSniffer脚本以使用

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开PHP_CodeSniffer区域。在配置列表中， 选择PHP_CodeSniffer脚本：

    • 要使用与特定的远程PHP解释器关联的脚本，请选择该解释器的名称。

    • 要使用本地脚本，请选择Local 。在这种情况下，无论项目中使用哪种PHP解释器（本地或远程），都将执行本地PHP_CodeSniffer。请注意，PHP_CodeSniffer只能有一个本地配置，因为IntelliJ IDEA运行的脚本（对于Windows为phpcs.bat或对于Linux和macOS为phpcs ）包含一个PHP引擎路径的脚本。

    • 要使用与默认项目解释器关联的脚本，即在“设置/首选项”对话框的PHP页面上选择的脚本 ，请选择“默认项目解释器” 。

配置本地PHP_CodeSniffer脚本

  1. 下载并安装PHP_CodeSniffer脚本。

    要检查PHP_CodeSniffer安装，请切换到安装目录并运行以下命令：

    phpcs --version

    如果该工具可用，您将收到以下格式的消息：

    Squiz的 PHP_CodeSniffer版本（稳定）。（http://www.squiz.net）

    要根据您自己的自定义编码标准检查代码，请创建它 。将规则和指向它们的ruleset.xml文件存储在编码标准根目录中。

  2. 在IntelliJ IDEA中注册本地PHP_CodeSniffer脚本：

    • 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

    • 在打开的“ 质量工具”页面上，展开PHP_CodeSniffer区域，然后单击 配置列表旁边。

    • 在打开的PHP_CodeSniffer对话框中，在PHP_CodeSniffer路径字段中指定phpcs.bat或phpcs PHP_CodeSniffer可执行文件的位置。手动输入路径或单击然后在打开的对话框中选择相关文件夹。

      要检查phpcs.bat或phpcs的指定路径是否可以确保IntelliJ IDEA和PHP_CodeSniffer之间的交互，即可以从IntelliJ IDEA启动该工具，并且IntelliJ IDEA会从中接收到问题报告，请单击Validate按钮。此验证等同于运行phpcs --version命令。如果验证成功通过，则IntelliJ IDEA将显示有关检测到的PHP_CodeSniffer版本的信息。

配置与PHP解释器关联的PHP_CodeSniffer脚本

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开PHP_CodeSniffer区域，然后单击 配置列表旁边。将打开“ PHP_CodeSniffer”对话框，在左侧窗格中显示所有已配置的PHP_CodeSniffer脚本的列表，其中一个是“ 本地 ”类型，其他则以与脚本关联的PHP解释器命名。请点击在工具栏上。

  3. 在打开的PHP_CodeSniffer by Remote Interpreter对话框中，选择要使用关联脚本的远程PHP解释器。如果列表不包含相关的解释器，请单击并在CLI解释器对话框中配置远程解释器 ，如配置远程PHP解释器中所述 。

    当您单击OK时 ，IntelliJ IDEA将带您回到PHP_CodeSniffer对话框，在该对话框中将新的PHP_CodeSniffer配置添加到列表中，并且右侧窗格显示了所选的远程PHP解释器，与之关联的PHP_CodeSniffer的路径以及高级的PHP_CodeSniffer选项。

配置高级PHP_CodeSniffer选项

IntelliJ IDEA允许您指定高级PHP_CodeSniffer选项，从而根据计算机的配置和所使用的规则集来微调PHP_CodeSniffer进程的行为。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开PHP_CodeSniffer区域。在“ 每个文件的最大邮件数”字段中，设置文件要报告的邮件总数的上限。执行检查后，所有超过此限制的消息将被拒绝，并且IntelliJ IDEA将在编辑器选项卡顶部或“ 检查结果工具”窗口中显示警告消息。

  3. 在“ 工具进程超时”字段中，指定您希望IntelliJ IDEA等待PHP_CodeSniffer的结果的时间，然后终止进程以防止CPU和内存使用过多。

  4. 如有必要，请在“ PHP代码美化工具和修复程序设置”区域的“ phpcbf的路径”字段中提供“ PHP代码美化工具和修复程序”工具的路径 。这将使您自动修复PHP_CodeSniffer检测到的许多错误。如果您将PHP_CodeSniffer与Composer一起安装 ， PHP Code Beautifier和Fixer将被检测并自动设置。

将PHP_CodeSniffer配置为IntelliJ IDEA检查

手动配置PHP_CodeSniffer检查

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在编辑器下单击检查 。

  2. 在打开的“ 检查”页面上，展开PHP |。质量工具节点，然后选中PHP_CodeSniffer验证旁边的复选框。

  3. 在页面的右侧窗格上，使用“ 选项”区域中的控件配置PHP_CodeSniffer工具：

    1. 从“ 严重性”列表中，选择PHP_CodeSniffer检查的严重性等级 。选择的值确定IntelliJ IDEA对检测到的差异的严重程度，并显示在检查结果中 。

    2. 从“ 范围”列表中，选择将检查应用程序限制到的范围 。

    3. 在“ 检查带扩展名的文件”字段中，提供用逗号分隔的文件扩展名列表，这些文件扩展名应由PHP_CodeSniffer检查。

    4. 要使PHP_CodeSniffer除报告错误外还报告警告，请选中将警告显示为...复选框，然后从列表中选择严重性等级 。要仅报告错误并禁止报告警告，请清除“将警告显示为...”复选框。

    5. 如果您依赖于自定义的第三方编码标准（例如Coder或Joomla编码标准 ），则需要在使用前将其与PHP_CodeSniffer集成。为此，请选择“ 安装标准路径”复选框，然后单击“确定”。 ，然后在打开的对话框中选择自定义标准安装目录。

    6. 选中“ 显示嗅探名称”复选框，以在编辑器中显示相应的嗅探名称，或者在检测摘要中显示检测结果 。

    7. 指定要应用的编码标准。

      • 要使用预定义的编码标准之一 ，请选择“ 编码标准”列表，并指定编码样式以对照您的代码。该列表包含安装在php_codesniffer主目录结构内的所有编码标准。

      • 要根据您自己先前定义的编码标准检查代码，请选择“ 自定义” 。请点击然后在打开的“ 自定义编码标准”对话框中，在“ 规则集的路径”字段中为您自己的编码标准指定ruleset.xml文件的路径 。手动输入路径或单击然后在打开的对话框中选择相关文件夹。

使用Composer配置PHP_CodeSniffer检查

您可以在composer.json的脚本部分内包含有关默认和自定义PHP_CodeSniffer规则集的信息。当您安装或更新项目依赖项时，将检测到指定的规则集，并且将自动启用PHP_CodeSniffer验证检查。

如果未在其中指定规则集scripts在composer.json的部分中，IntelliJ IDEA将另外检查项目根目录以找到具有phpcs.xml默认名称的规则集。如果文件存在，它将被自动选择为检查的“ 自定义”规则集。

使用Composer配置PHP_CodeSniffer检查

  • 在里面scripts在composer.json部分，添加phpcs将PHP_CodeSniffer启动命令放入叶子元素之一。

    提供--standard参数表示使用的编码标准。例如，添加以下记录会将编码标准设置为PSR2 ：

    “脚本”：{“ phpcs”：“ phpcs --standard = PSR2”}

    添加以下记录会将编码标准设置为Custom ，将规则集的路径设置为/phpcs.xml ：

    “脚本”：{“ phpcs”：“ phpcs --standard = phpcs.xml”}

此外，您还可以在以下位置提供自定义的非PSR标准依赖项： require-dev composer.json部分以自动检测它。当前，支持以下标准：

  • 教义
  • Drupal编码器
  • Joomla
  • Magento
  • MediaWiki
  • PHP兼容性
  • Symfony
  • WordPress的
  • i

修复PHP_CodeSniffer检测到的问题

IntelliJ IDEA与PHP Code Beautifier and Fixer工具集成在一起，可让您修复许多检测到的问题。如果您将PHP_CodeSniffer与Composer一起安装 ， PHP Code Beautifier和Fixer将被检测并自动设置。否则，您需要手动设置 。

通过使用快速修复来解决问题

检测到问题后，IntelliJ IDEA会根据PHP_CodeSniffer 检查设置突出显示该问题。

  1. 将插入符号放在检测到的问题上，然后单击 ，或按Alt+Enter 。

  2. 选择PHP Code Beautifier and Fixer：从弹出菜单中修复整个文件 。请注意，这将解决当前文件中检测到的所有问题。

您还可以在批处理模式下检测问题，或通过代码清除自动解决问题。

使用代码清理自动修复问题

  1. 从主菜单中，选择“ 分析|代码清理 。

  2. 在打开的“ 指定代码清除范围”对话框中，选择要将检验配置文件应用到的范围。

  3. 从列表中选择检验配置文件，或单击在打开的“代码清除检查”对话框中配置新的配置文件。您也可以点击进行检查，将应用哪些修复程序，并确保启用了PHP_CodeSniffer验证检查。

  4. 单击“ 确定”启动代码清除。

以批处理模式运行PHP_CodeSniffer

  1. 通过选择Code | 运行检查 。从主菜单检查代码 。指定检查范围和配置文件。

  2. 在“ 检查结果工具”窗口中查看检查结果。PHP_CodeSniffer报告的错误和警告以开头phpcs区别于IntelliJ IDEA内部检查。

从PHP_CodeSniffer检查中排除文件

当等待PHP_CodeSniffer响应超过在PHP_CodeSniffer对话框的“ 工具进程超时”字段中指定的限制时，IntelliJ IDEA建议将文件添加到忽略列表 。

管理被忽略的文件

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP |质量工具 。

  2. 在打开的“ 质量工具”页面上，展开PHP_CodeSniffer区域，然后单击“ 显示忽略的文件”链接。

    • 要添加文件，请单击并在打开的对话框中找到所需的文件。

    • 要从列表中删除文件并让PHP_CodeSniffer再次对其进行处理，请选择该文件并单击 。

    • 要从列表中删除所有文件，请单击“ 清理列表”按钮 。
