Java 9模块系统入门

Java平台模块系统（JSR 376）或Project Jigsaw有望成为JDK 9版本的一部分。JSR描述的系统目标是：

  • 可靠的配置 ，用一种使程序组件相互声明显式依赖关系的方法来替换易碎的，容易出错的类路径机制

  • 强封装 ，允许组件声明其他组件可以访问的公共类型，而其他组件则不能访问。

下面讨论的功能基于IntelliJ IDEA和JDK 9的早期访问版本，并且将来可能会更改

IntelliJ IDEA已经有了一个项目模块的概念。每个IntelliJ IDEA模块都构建自己的类路径。
随着新Java平台模块系统的引入，IntelliJ IDEA模块必须通过支持Java平台的模块路径（如果使用它而不是类路径）来扩展其功能。
在本教程中，我们将探讨IntelliJ IDEA在哪里协助创建和使用Java Platform模块，以及这些模块如何与IntelliJ IDEA模块一起使用。
我们将使用OpenJDK快速入门指南中完整给出的“问候世界”示例。

创建一个模块

在IntelliJ IDEA中创建模块后，我们可以通过在模块源目录下创建module-info.java文件将其定义为Java Platform模块。
每个IntelliJ IDEA模块最多只能有一个Java平台模块。
我们可以为模块创建一个新的module-info.java ，方法是选择要在其中创建源代码的目录，然后使用菜单选项New | module-info.java

创建module-info.java声明文件时，IntelliJ IDEA将选择IntelliJ IDEA模块的名称作为Java Platform模块的默认名称。这可以更改，不需要匹配。

使用模块

与所有Java文件类型一样，IntelliJ IDEA可以帮助我们对module-info.java内容进行自动完成和有效性检查。

需要在IntelliJ IDEA和Java Platform（Jigsaw）模块中定义模块的依赖项。
IntelliJ IDEA帮助我们保持它们同步。
要定义项目模块之间的依赖关系，我们可以编写一个requires在module-info.java中进行声明，然后IntelliJ IDEA会建议我们也将其添加为模块中的依赖项。
这也适用于库依赖项，但前提是已经将库jar文件声明为项目依赖项。

从另一个方向来看，我们可以编写我们的Java代码。IntelliJ IDEA将建议添加另一个模块作为对我们当前模块的依赖，然后还将建议我们添加一个requires声明。

使用模块运行

运行类时使用Java平台模块中声明的信息Ctrl+Shift+F10在IntelliJ IDEA中。
这意味着IntelliJ IDEA将使用模块路径而不是类 路径运行JVM。
这将强制我们从模块系统获得强大的封装，并且此运行将重现我们可能遇到的任何依赖关系问题。
