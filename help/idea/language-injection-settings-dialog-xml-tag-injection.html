项目

描述

名称

注射的名称。

语言

要注入的语言。

  • ID。 语言ID或名称。

  • 字首。 在相应的字符串值之前要添加的字符序列。

  • 后缀。 在相应的字符串值之后要添加的字符序列。

前缀和后缀是可选的。

XML标签

在此区域中，定义XML标记，该标记指示应将该标记中包含的文本视为所选语言。

  • 本地名称 ：指定没有名称空间前缀的标签名称。使用正则表达式指定多个标签名称name1|name2 ，不区分大小写的名称（ (?i)tagname匹配tagname和TagName ），依此类推。

    确保不要输入任何空格字符，因为它们对于比赛很重要。

  • 命名空间 ：指定XML标签的命名空间URI。该字段是可选的。

子标签

选择此复选框以递归地包括所有子标签。

高级

指定其他设置以对注射过程进行更细粒度的控制。

  • 值模式 -键入一个正则表达式，该表达式确定将XML语言注入的XML文本值的一部分。通过使用模式的第一个捕获组作为注入目标，您可以将过程配置为仅将语言注入到与特定模式匹配的值或与模式匹配的多个部分中。

    例子：

    [$#]\{(.*?)\}匹配JSP / JSF表达式语言使用的模式。

    ^javascript:(.*)符合javascript可以在超链接-hrefs中使用以执行JavaScript代码的协议。

      如果启用此选项，则所有相应的片段都将合并在一起以形成一个单元或“文件”。

  • XPath条件 ：指定XPath表达式以更精确地解决注入目标。计算表达式的上下文是周围的XML标记。

    可以使用XPath 扩展功能 ，例如， lower-case() 。此外，还有三个附加功能可用于确定当前文件的名称，扩展名和文件类型： file-name() ， file-ext()和file-type() 。或者，可以通过标准代码完成来检索可用功能的列表。

    为了使该字段处于活动状态，必须启用XPathView + XSLT支持插件 。

    出于性能原因，建议您使这些表达式尽可能简单。特别是导致整个文档被扫描的表达式，例如//foo/bar可能会导致大文件的性能问题。
