安装，卸载和升级Python软件包

以下内容仅在安装并启用Python插件后才有效。

IntelliJ IDEA提供了用于安装，卸载和升级特定Python SDK的Python软件包的方法。默认情况下，IntelliJ IDEA使用pip管理项目包。对于Conda环境，您可以使用conda软件包管理器 。

要管理项目解释器的Python包，请导航至“ 文件” |“文件”。项目结构 ，在“ 平台设置”部分下选择SDK ，然后从可用SDK列表中选择目标Python SDK。切换到“ 软件包”选项卡以预览可用软件包的列表。

如果您选择具有配置的Conda环境的Python SDK，则“ 使用Conda程序包管理器”切换将显示在“ 程序包”选项卡工具栏中。

使用此切换按钮可以从Conda环境存储库管理软件包。默认情况下，此切换开关处于关闭状态，并且您只能管理所选Python解释器可用的软件包。

安装软件包

  1. 点击安装 。

  2. 在打开的对话框中，预览可用软件包的列表。

    要指定自定义存储库（包括devpi或PyPi） ，请点击管理存储库 。
    在打开的“ 管理存储库”对话框中，单击添加本地存储库的URL，例如http：// localhost：3141 / root / pypi / + simple / ，然后单击OK 。在“ 可用软件包”对话框中，单击“ 重新加载软件包列表。

  3. 在搜索字段中输入要安装的软件包的名称。列表缩小以仅显示匹配的软件包。

  4. 如果需要，请选择以下复选框：

    • 指定版本 ：如果选中此复选框，则可以从可用版本列表中选择所需的版本。默认情况下，采用最新版本。

    • 选项 ：如果选中此复选框，则可以在文本字段中键入选项。

    • 安装到用户的站点软件包目录注意：如果未选中此复选框（默认情况下），则这些软件包将安装到当前的解释器软件包目录中。如果选中此复选框，则软件包将安装到指定目录中。此选项不适用于Conda环境。

  5. 选择目标软件包，然后单击“ 安装软件包” 。

IntelliJ IDEA提供了一个快速修复程序，可以自动安装您要导入的软件包：如果在关键字之后import ，您输入了计算机上当前不可用的软件包的名称，一个快速修复建议建议忽略未解决的引用，或者下载并安装缺少的软件包：

卸载软件包

  1. 在“ 软件包”选项卡中，选择要删除的软件包。

  2. 点击卸载 （ ）。所选软件包将从磁盘中删除。

IntelliJ IDEA通过显示当前安装的软件包版本号（列版本 ）和最新可用版本（列最新版本 ）的数量，智能地跟踪软件包的状态并识别过时的版本。当检测到较新版本的软件包时，IntelliJ IDEA会使用箭头符号对其进行标记。 并建议对其进行升级。

默认情况下，“ 最新版本”列仅显示软件包的稳定版本。如果要将最新可用版本的范围扩展到任何预发行版本（例如Beta或候选发行版 ），请点击显示早期版本 。

升级套件

  1. 在“ 软件包”选项卡中，选择要升级的软件包。

  2. 点击升级 （ ）。请注意，单击“ 应用 ”不会导致软件包升级。此按钮仅利用所有执行的软件包更改。

    选定的软件包将升级到最新的可用版本。

您可以一次升级多个软件包。按住Cmd （macOS）或Ctrl （在Unix或Windows上），单击鼠标左键以在软件包列表中选择多个项目，然后单击“ 升级” 。

如果您习惯于从命令行安装软件包，则可以使用Python控制台继续进行工作流程。
