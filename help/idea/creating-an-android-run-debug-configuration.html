创建一个Android运行/调试配置

  1. 要开始创建Android运行配置，请选择运行|运行。从主菜单编辑配置 。或者，单击Shift+Alt+F10然后从列表中选择“ 编辑配置 ”。单击添加新配置按钮在工具栏上，然后从列表中选择Android应用程序 。在打开的“运行/调试配置：Android应用程序”页面上，指定配置名称，然后选择将应用此配置的模块。
  2. 指定将在目标设备上部署的.apk文件，并指定将在应用程序启动时启动的活动。

    • 要运行或调试整个应用程序， 从安装选项区域的部署下拉列表中选择默认APK，从启动选项区域的启动下拉列表中选择 默认活动 。IntelliJ IDEA将上传.apk 根据上面“ 模块”下拉列表中指定的模块构建。.apk是自动构建的，不需要您进行任何初步的工件配置。

    • 要运行或调试自定义.apk ，以后将其嵌入到应用程序中， 从“ 安装选项”区域的“ 部署”下拉列表中选择“ 自定义工件”选项，然后选择要从中构建.apk的工件。在这种情况下，您必须在创建运行/调试配置之前手动定义工件（有关详细信息，请参阅通过工件生成签名发布APK和使用工件 ）。然后从“ 启动 选项”区域的“ 启动”下拉列表中选择“ 指定的活动”选项，并从所选工件.apk中指定启动活动。 手动输入活动名称或单击浏览 然后在打开的“ 选择活动类”对话框中选择它。

      可用活动的列表由模块的选择确定。

    • 如果要为已经运行的应用程序启动调试会话，请在“ 软件包”区域中选择“不部署任何东西 ”以禁止将数据上传到设备，然后在“ 活动”区域中选择“不启动活动 ”。 使用这些设置执行运行配置与单击“ 将调试器附加到Android进程”按钮相同在工具栏上。

  3. 在“ 部署目标选项”区域中，指定将在其中启动应用程序的设备。
    • 要指定虚拟设备，请选择“ 仿真器”选项，然后从“ 首选Android虚拟设备”下拉列表中选择一个虚拟设备。

      如果没有可用的虚拟设备，请单击“ 浏览”。 以启动Android虚拟设备（AVD）管理器并配置新的模拟器（有关详细信息，请参阅管理虚拟设备 ）。

    • 如果要手动选择目标设备，请选择“ 显示选择器”对话框选项。每次启动运行/调试会话并应用此配置时，IntelliJ IDEA都会显示“ 选择设备”对话框。

    • 要使IntelliJ IDEA在应用程序启动时检测到插入的USB设备，请选择USB设备选项。

    如果要在稍后插入的物理设备上运行应用程序，则选择“ 显示选择器对话框”或“ USB设备”选项可能会有所帮助，因此，无法预见可用设备的集合。
