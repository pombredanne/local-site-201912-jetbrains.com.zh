创建CMP Bean字段

CMP字段用于提供实体Bean的持久性。创建后，CMP字段将显示在部署描述符中，并将其访问器方法添加到实体Bean类中。为此，IntelliJ IDEA在编辑器中提供了装订线图标，可帮助您从部署描述符中的CMP字段声明跳转到Bean类中的访问器方法，反之亦然：

IntelliJ IDEA使您可以通过多种方式创建CMP字段：

  • 通过直接编辑源代码的方式 。在这种情况下，IntelliJ IDEA提供了编码帮助，并且在EJB工具窗口中创建了相应的节点。

  • 使用实体bean的上下文菜单 。

  • 使用bean编辑器 。

通过编辑源代码创建CMP字段

  1. 打开部署描述符文件ejb-jar.xml进行编辑。

  2. 在里面entity相应实体bean的“部分”，键入CMP字段的标记，然后指定其名称。

  3. 打开所需的实体bean类的源代码以进行编辑。代码检查可检测到缺少的访问器方法：

  4. 按Alt+Enter揭示建议的快速修复列表：

  5. 在bean实现类中创建getter和setter方法。

  6. 如果要使新的CMP字段成为主键 ，请将该条目添加到部署描述符中。

创建CMP字段

  1. 在EJB工具窗口中，右键单击所需的实体bean。

  2. 在上下文菜单上，单击“ 新建” ，然后在子菜单上选择“ CMP”字段 。

    或者， 打开所需的bean进行编辑 ，然后在“ CMP字段”部分中单击 。

  3. 在“ 创建CMP字段”对话框中，指定以下参数：

    • 字段名称，可选说明和类型。您可以从下拉列表中选择所需的类型，或单击省略号按钮，然后从“ 选择CMP字段类”对话框中选择类型。

    • 新的CMP字段是否将为主键。如果要将此字段设为主键，请选中“ 主键”复选框。

    • 是否在本地和远程接口中生成访问器方法。如果选中相应的复选框，则除了bean类之外，还将在本地和远程接口中创建getter和setter方法。

  4. 单击确定 。
