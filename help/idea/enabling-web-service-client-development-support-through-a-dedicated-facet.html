通过专用方面启用Web Service客户端开发支持

如果要为以下类型之一的Web服务开发客户端，则可以使用此方法：

  • GlassFish / JAXWS2。X RI / Netro 1。X / JWSDP2.0 。

  • Apache轴 。

  • RESTful Web服务 。

IntelliJ IDEA创建相关的Web服务客户端模块结构，并通过专用的Web服务客户端界面自动识别所有必需的库。此外，IntelliJ IDEA可以找到以前下载的所需库，也可以建议将它们下载到您选择的位置。

Web服务客户端构面可以直接添加到模块中，而无需父Web构面。

要通过专用方面启用Web Service客户端开发支持，请执行以下任一操作

无论哪种情况，IntelliJ IDEA都会下载预定义的WS引擎实现版本。

  • 使用专用的Web服务客户端构面创建Java模块 。除了设置相关的模块结构并自动提供所有必要的库之外，IntelliJ IDEA还将创建一个示例包，其中包含HelloWorldClient类，您可以将其用作示例或在开发应用程序时根据需要填充。

  • 将专用构面添加到现有模块。

为Web服务客户端应用程序创建模块

  1. 请执行以下任一操作：

    • 如果要创建新项目，请：在“ 欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件” |“ 新建” 。 新品|项目 。

      结果，“ 新建项目”向导打开。

    • 如果要向现有项目中添加模块，请执行以下操作：打开要向其中添加模块的项目，然后选择“ 文件” |“添加” 。新品|模块 。

      结果，“ 新建模块”向导将打开。

  2. 在向导的第一页的左侧窗格中，选择Java 。在页面的右侧，指定要使用的JDK 。

  3. 在“ 其他库和框架”下 ，选中“ WebServices客户端”复选框。

  4. 从版本列表中选择所需的WS引擎实现。

  5. 您将需要一个实现所选WS引擎的库。您可以选择使用现有的库，创建和使用新的库，下载库文件（如果计算机上尚不可用）或将库的设置推迟到以后。

    使用图书馆

    选择要使用的库（如果在IntelliJ IDEA中已经定义了相应的库）。

    创建

    如果您的计算机上已经有相应的jar文件，则可以将这些文件放在库中。单击创建，然后在打开的对话框中选择所需的文件。

    （可选）单击“ 配置”以编辑选定的库。

    下载

    选择此选项以下载必要的库文件。

    （可选）单击“ 配置”以编辑库设置和内容。

    稍后设置图书馆

    选择此选项可以将库的设置推迟到以后。

    单击下一步 。

  6. 指定名称和位置设置。

    点击完成 。

要将专用构面添加到现有模块中

  1. 打开项目工具窗口（例如， 视图|工具窗口|项目 ）。

  2. 右键单击感兴趣的模块，然后选择添加框架支持 。

  3. 在打开的“ 添加框架支持”对话框的左侧窗格中，选中“ WebServices客户端”复选框。

  4. 如果要生成示例客户端代码，请选择相应的复选框。

  5. 从版本列表中选择所需的WS引擎实现。

  6. 您将需要一个实现所选WS引擎的库。您可以选择使用现有的库，创建和使用新的库，下载库文件（如果计算机上尚不可用）或将库的设置推迟到以后。

    使用图书馆

    选择要使用的库（如果在IntelliJ IDEA中已经定义了相应的库）。

    创建

    如果您的计算机上已经有相应的jar文件，则可以将这些文件放在库中。单击创建，然后在打开的对话框中选择所需的文件。

    （可选）单击“ 配置”以编辑选定的库。

    下载

    选择此选项以下载必要的库文件。

    （可选）单击“ 配置”以编辑库设置和内容。

    稍后设置图书馆

    选择此选项可以将库的设置推迟到以后。

  7. 在“ 添加框架支持”对话框中单击“ 确定 ”。
