使用Docker Compose运行和调试Spring Boot应用程序

您可以使用IntelliJ IDEA在Docker Compose下运行和调试在多个Docker容器中运行的Spring Boot应用程序。本教程描述了如何在容器内运行两个Docker Compose服务：一个简单的Spring Boot应用程序和一个MySQL数据库。该应用程序可以接收将条目添加到数据库的GET请求。本教程还描述了如何使用远程调试配置来设置断点和调试应用程序。

克隆样本项目

该应用程序的源代码托管在GitHub上 。

  1. 在主菜单中，选择VCS |。从版本控制中检出|吉特

  2. 指定URL： https : //github.com/IdeaUJetBrains/SpringBootDockerDemoDebug

    如有必要，测试连接，然后单击“ 克隆” 。

  3. 同意打开克隆的项目。

使用Docker Compose运行应用程序

  1. 打开docker-compose-debug.yml文件。

  2. 请点击在排水沟中。

    这将创建一个Docker Compose运行配置，该配置在一个容器中启动应用程序作为应用程序服务，在另一个容器中启动数据库的数据库服务作为db服务。如果成功，您应该在部署日志中看到类似于以下内容：

    创建数据库...创建springbootdockerdemodebug2_app_1 ...'Compose：docker-compose-debug.yml'已成功部署。

    您可以通过http：// localhost：18080访问该应用程序。例如，您可以尝试执行以下GET请求，该请求应添加一个新entitybus数据库条目并列出所有可用条目： http : //127.0.0.1 : 18080/entitybus/post

    您可以通过jdbc：mysql：//0.0.0.0：13306 / DOCKERDB连接到数据库。例如，您可以使用“ 数据库”工具窗口将数据库添加为数据源，然后检查entitybus表。使用以下连接设置：

    • 主机： 0.0.0.0或localhost

    • 港口：13306

    • 用户：root

    • 密码：root

    • 数据库：DOCKERDB

创建远程调试配置

  1. 打开docker-compose-debug.yml文件。

  2. 请点击在排水沟中。

  3. 在“ 使用模块类路径”列表中选择模块。

  4. 在启动前列表中双击Docker Compose运行配置。如果不在列表中，请单击并选择在调试之前启动Docker 。

  5. 选择Docker Compose运行配置以在启动远程调试之前启动，并确保已将其配置为连接到应用程序服务。还要检查“ 自定义命令”字段：该字段应包含-agentlib选项和来自command docker-compose-debug.yml文件中的字段：

    java -agentlib：jdwp = transport = dt_socket，server = y，suspend = y，address = 5005 -Djava.security.egd = file：/ dev /./ urandom -jar /project/target/demo-0.0.1-SNAPSHOT 。罐

    如果将默认端口分配给其他端口，请更改Java调试器要连接的容器端口，并确保远程调试配置中的端口相同。

如果应用程序已经在运行，请不要运行调试配置。应用设置，然后单击“ 取消” 。

启动调试配置

  1. 如果应用程序已经在运行，请删除容器。为此，右键单击Docker Compose 应用程序和数据库服务下的容器，然后单击删除 。

  2. 打开docker-compose-debug.yml文件。

  3. 请点击在装订线中启动调试配置。

应用程序启动并运行后，将打开“ 调试工具”窗口。

设置断点并调试应用程序

  1. 打开src / main / java / entity / Entitybus.java文件，并在setEid()方法。

  2. 执行以下GET请求： http : //127.0.0.1 : 18080/entitybus/ post

    该应用程序将在断点处停止，您可以在“ 调试”工具窗口中检查当前变量值和框架。

  3. 请点击直到它执行请求，然后您才能获得返回的值。
