命令行工具支持：WP-CLI

档案|设置|工具| Windows和Linux的命令行工具支持
IntelliJ IDEA |首选项|工具|适用于macOS的命令行工具支持

单击时将打开对话框然后在“ 命令行工具”对话框中选择WP-CLI 。使用该对话框在IntelliJ IDEA中配置WP-CLI支持。

在此对话框中，通过指定启动WordPress的方式，以命令行模式启用WordPress内容管理系统 。

该工具仅适用于本地PHP解释器 。

项目

描述

通过PHAR安装

 选择此选项可通过PHP脚本启动WordPress ，或让IntelliJ IDEA在wp-cli.phar存档中检测并启动启动器。

  • PHP解释器 ：从“执行”区域的列表中选择已配置的本地PHP解释器之一。

  • phar的路径 ：在此字段中，指定wp-cli.phar存档的位置。

可执行文件（通过Composer等安装）

选择此选项可通过可执行文件启动WordPress ，当您使用软件包管理工具（例如Composer）安装WordPress时，该文件可用。

wp的路径

在这个领域里， 指定wp.bat或wp可执行文件的位置。如果您使用的是Composer，则默认位置为\ vendor \ wp \ cli \ bin \ wp或\ vendor \ wp \ cli \ bin \ wp.bat 。 手动输入路径或单击然后在打开的对话框中选择所需的位置。
