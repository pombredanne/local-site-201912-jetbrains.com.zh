创建单元测试

在Android中，单元测试基于JUnit ，而JUnit的平面使用足以测试仅基于Java代码的功能。

但是，要测试特定于Android的功能，您需要在JUnit之上构建一堆包装器类。IntelliJ IDEA简化了围绕Android测试项目构建的大多数任务。

1。确保您的代码是可测试的

单元测试要求以这样一种方式来组成源代码，以便可以通过模拟轻松地消除模块之间的依赖关系。另外，单元测试要求功能之间必须相互隔离。

照原样， HelloDroid上课不容易测试。在进行单元测试之前，让我们首先应用快速重构。

2。创建一个测试模块

现在让我们创建一个新的测试模块并进行设置HelloDroid作为测试模块。这样可以确保测试模块对包含要测试的源代码的模块保持引用。

  1. 从主菜单中，选择文件|新品|模块以启动“ 新建模块”向导 。

  2. 在向导的第一页上，在左侧窗格中选择“ Android ”，然后在右侧选择“ Test Module ”：

  3. 在第二页上，指定新的模块名称，例如， Tests 。保留其他字段不变。的HelloDroid模块将自动指定为测试模块，因为这是项目中唯一的模块。

新节点将追加到名为Tests的项目。该模块具有自己的清单文件和src目录。清单文件链接到android.test库以构建测试类。

新创建的模块有一个名为HelloDroidTest在src文件夹中。您可以通过添加更多Java类来添加更多测试文件，如下所示：

5，创建测试的运行/调试配置

为了运行测试，您需要创建专用的运行/调试配置。设置测试模块时，将自动为您创建默认配置。

要编辑其设置，请在主菜单中选择“运行” |“设置”。编辑配置，然后在左窗格中的“ Android 测试”下选择“ 测试 ”：

您可以选择运行模块中的所有测试，或将测试限制为给定类中的方法。

6。运行测试

要运行测试，请确保在编辑器右上角的下拉列表中选择了适当的运行/调试配置，然后单击“ 运行”按钮在它的旁边：

测试结果显示在自动激活的“运行”工具窗口的“ 测试运行器”选项卡中。如果测试成功完成，编辑器右上角将出现一个绿色的正方形图标。如果有警告，则图标为黄色，如果测试失败，则为红色。您可以单击该图标以获取更多详细信息。

您可以通过单击导出测试结果图标将测试报告导出为多种格式。 在“ 测试”选项卡工具栏中 。
