介绍

IntelliJ IDEA中的每个Flash模块都包含一个或多个构建配置。构建配置定义了如何将模块源文件转换为目标输出类型（SWF或SWC），然后打包。

创建模块时，将创建一个构建配置。如果需要，您可以添加更多的构建配置。

构建配置类型

构建配置类型由以下定义：

  • 目标平台（即，将在其中使用开发的内容的环境）：Web（用于Flash播放器/以Web浏览器为目标的内容），Desktop（用于Adobe AIR为目标的内容）或Mobile（用于内容的Mobile）适用于Android和iOS移动设备）。

  • 构建配置是使用Flex框架还是纯ActionScript。

  • 输出类型：应用程序（可运行的应用程序，SWF文件），库（SWC文件）或运行时加载的模块（动态加载的模块 ，SWF文件）。

构建配置的主要选项

定义构建配置类型后，可以指定以下主要选项：

  • 对于应用程序和运行时加载的模块： main class 。

  • 输出文件名。

  • 输出文件夹。

  • 对于Web应用程序：具有HTML包装器模板的文件夹。

  • 对于Web和桌面应用程序： 模块和运行时样式表 。

构建配置依赖项（构建路径）

通常，可以通过以下方式定义每个构建配置的构建路径：

  • Flex SDK。根据构建配置类型自动选择必需的SDK SWC。此外，您可以通过选择以下内容来管理SWC集：

    • 对于Web目标平台：Flash Player版本。如果SDK包含多个播放器版本，则可以选择应使用哪个相应的SWC。

    • 对于基于Flex框架的构建配置：Flex 4组件集。您可以指定仅使用Spark或MX或两个组件集。

      对于Flex组件，您可以指定其框架链接类型 （合并到代码，RLS或外部）。

  • 生成库和运行时加载的模块的其他构建配置。

  • 第三方库，包括SWC和原始ActionScript。

编译器选项

每个构建配置都与一组编译器选项相关联。

IntelliJ IDEA在IDE，项目和模块级别提供了默认的编译器选项集。

较低级别的默认值可以从较高级别继承或重新定义。在各个构建配置级别上，类似地，编译器选项可以从模块级别的默认值继承。或者，可以指定特定于构建配置的值。

IntelliJ IDEA提供了一个方便的界面，用于在各个级别上编辑默认值。还可以恢复已更改值的默认值。

活动构建配置

构建配置之一已设置为对相应模块处于活动状态。

活动的构建配置为在编辑器中突出显示源代码提供了基础。因此，当您更改活动配置时，模块中突出显示的代码也会更改。

管理构建配置及其设置

要管理内部配置及其设置，请按照以下步骤操作

  1. 打开项目结构对话框（例如Ctrl+Shift+Alt+S ）。

  2. 在“ 项目设置”下 ，选择“ 模块” 。

  3. 在下面的区域 ，展开必要的Flash模块节点，然后选择任何构建配置。现在你可以：

    • 在对话框右侧显示的“ 构建配置”页面上查看和编辑所选构建配置的设置。

    • 创建所选构建配置的副本。为此，请单击或在上下文菜单中选择“ 复制 ”。在打开的对话框中指定用于复制构建配置的设置。请注意，根据设置的不同，构建配置的副本的类型可能与原始副本的类型不同。请参阅更改构建配置类型时可能的更改 。

    • 在项目中找到所选构建配置的用法。为此，请单击 ， 按Alt+F7或在上下文菜单中选择“ 查找用法 ”。

    • 删除所选的构建配置。为此，请单击 ， 按Delete或在上下文菜单中选择“ 删除 ”。

    • 创建一个新的构建配置。要做到这一点：

      1. 请点击 ， 按Alt+Insert或在上下文菜单中选择“ 新建 ”。

      2. 选择Flash构建配置 。

      3. 在打开的“ 添加构建配置”对话框中，指定名称并选择新构建配置的主要选项，然后单击“ 确定” 。

      4. 如有必要，在“ 构建配置”页面上编辑构建配置设置。

    • 更改构建配置类型 。为此，请在“ 常规”选项卡上（显示构建配置类型的区域的右侧）单击“ 更改 ”，然后在打开的对话框中指定构建配置属性。请参阅更改构建配置类型时可能的更改 。

  4. 在“ 项目结构”对话框中单击“ 确定 ”。

另请参见要使用快捷方式打开内部配置设置，请按照以下步骤操作 。

更改构建配置类型时可能的更改

在更改构建配置类型时 （通过使用“更改类型”命令或在创建构建配置的副本时），如果构建配置输出类型已更改（例如，从“应用程序”更改为“库”），则可能会发生以下更改：

  • 输出文件的扩展名可能会更改（从.swf更改为.swc ，反之亦然）。

  • 如果输出类型已更改为“库”，则将删除对运行时加载的模块（如果存在）的依赖关系。

  • 如果对具有更改后的类型的构建配置的依赖关系变得不合适，则将删除此类依赖关系。

创建构建配置的副本时，此外还会发生以下更改：

  • 输出文件名更改。

  • 如果适用，则软件包文件名将更改。
