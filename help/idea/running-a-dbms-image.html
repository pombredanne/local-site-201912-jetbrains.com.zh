在Docker容器中运行数据库

您可以使用Docker运行数据库容器并测试您的应用程序如何与之交互。本教程描述了如何使用PostgeSQL服务器运行Docker容器并使用IntelliJ IDEA连接到它。

在开始之前，请确保已正确配置Docker集成 。

提取PostgreSQL服务器映像

  1. 在Docker工具窗口中，右键单击Images节点，然后单击Pull image 。

  2. 在“ 拉取映像”对话框中，选择默认的Docker Hub注册表并指定PostgreSQL服务器映像存储库名称和标签。 postgres和latest 。

  3. 单击确定，然后等待直到图像被拉出。

从PostgreSQL服务器映像运行容器

  1. 在Docker工具窗口中，右键单击PostgreSQL服务器映像，然后单击Create container 。

  2. 在“ 创建容器”弹出窗口中，单击“ 创建” 。

  3. 在“ 创建Docker配置”对话框中，指定容器的名称，选择“ 全部”以将所有公开的容器端口发布到主机接口，然后单击“运行” 。

连接到PostgreSQL服务器

Docker会自动将容器中的默认PostgreSQL服务器端口5432映射到临时端口范围内的主机端口（通常从32768到61000）。本教程假定使用端口32768。要检查实际映射是什么，请在Docker工具窗口中右键单击PostgreSQL容器，单击Inspect ，然后找到HostPort输出中的字段。

  1. 打开数据库工具窗口（“ 视图” |“工具窗口” |“数据库” ）。

  2. 点击新建 ，指向数据源 ，然后单击PostgreSQL 。

  3. 将主机名设置为localhost ，转至32768 （或映射到的任何主机端口），默认数据库和用户postgres 。默认情况下不需要密码。

  4. 如有必要，请下载驱动程序并测试连接。

  5. 单击确定以将PostgreSQL服务器添加为数据源

有关在IntelliJ IDEA中使用数据库源的更多信息，请参见数据库工具和SQL 。
