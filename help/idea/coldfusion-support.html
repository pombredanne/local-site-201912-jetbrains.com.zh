IntelliJ IDEA 2019.3帮助

冷融合

IntelliJ IDEA为ColdFusion文件提供了广泛的编辑支持，并提供了将应用程序部署到ColdFusion服务器的功能。

IntelliJ IDEA通过捆绑的插件实现ColdFusion功能，可以通过清除Windows上的CFML支持复选框来完全禁用它。 IntelliJ IDEA设置的“ 插件”页面Ctrl+Alt+S 。

ColdFusion服务器应下载并安装在您的计算机上。

ColdFusion支持

ColdFusion文件标记有图标。

ColdFusion支持包括：

  1. 编码协助：

    • 通过“ createObject”函数创建的标签，属性，属性值，当前作用域的函数和变量，函数参数，Java类和组件的函数的代码完成，对于“ createObject”中组件的名称和Java类的名称，对于继承的方法组件。

    • 错误和语法突出显示。

    • 代码格式化和折叠 。

  2. 浏览源代码的多种方法，其中包括：

    • 可以打开具有指定名称的文件或组件 。

    • 用结构视图导航 。

    • 导航|宣言 Ctrl+B 。

  3. 高级功能可搜索源代码 。

  4. 代码生成

    • 在文件创建过程中基于文件模板生成代码存根。

    • 使用实时模板插入，扩展和生成代码块。

    • 通过意图动作创建各种应用程序元素。

    • 能够创建行和块注释 Ctrl+/ / Ctrl+Shift+/ 。

  5. 查看方法参数信息。

  6. 运行/调试 ColdFusion的配置 。

配置部署到ColdFusion服务器

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| ColdFusion 。

  2. 在打开的ColdFusion页面上，指定带有应用程序源的本地文件夹和服务器上的路径之间的映射。
    • 要添加新的映射，请单击Alt+Insert并在目录路径字段中指定带有应用程序源的本地文件夹。手动输入路径或单击然后在打开的对话框中选择文件夹。

      在“ 逻辑路径”字段中，键入要将本地文件夹的内容部署到的服务器的URL地址。

    • 要从列表中删除映射，请选择该映射，然后单击Alt+Delete 。
