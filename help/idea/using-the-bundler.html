使用捆绑器

以下仅在安装并启用Ruby插件时有效。

IntelliJ IDEA支持与Bundler集成，并使您能够在IDE内部处理应用程序的gem依赖关系。使用Bundler之前，请确保满足以下先决条件：

  • 您的项目具有Gemfile。如果没有，您可以通过选择“ 工具” |“创建”来创建它。捆绑器|初始化命令。

  • Bundler gem已安装到项目SDK 。如果没有，请选择工具|捆绑器|安装Bundler来执行此操作。您还可以将Bundler宝石与Gemfile中的其他宝石一起安装

将gem安装到本地解释器

您可以按如下所示将Gemfile中定义的gem安装到本地项目解释器中：

  1. 请执行以下任一操作：

    • 按Ctrl两次。在调用的弹出窗口中，开始键入bundler ，选择bundle install然后按Enter 。

    • 点击工具|捆绑器|安装 。在此菜单中，您还可以使用其他Bundler命令，例如Update ， Clean等。

    • 使用IDE提供的Missing gem 检查安装缺少的gem。例如，打开Gemfile，将插入符号插入项目SDK中缺少的所有突出显示的gem，然后按Alt+Enter ：

      单击安装以安装缺少的gem。

将gem安装到项目

IntelliJ IDEA使您不仅可以处理安装到SDK上的gem，还可以处理项目本身上安装的gem。为此，请在运行bundle install命令： --path vendor/bundle

在这种情况下，Bundler将gems安装到vendor / bundle目录中，并将.bundle / config文件添加到项目的根目录中。IntelliJ IDEA处理此配置文件以检测gem是否已安装在vendor / bundle中 。

将gem安装到远程解释器

如果使用的是远程Ruby解释器 ，则IntelliJ IDEA将使用放置在远程计算机上的Gemfile来安装gem。这需要配置本地和远程项目的文件之间的映射。为此，请单击“ 编辑路径映射”。 Ruby SDK and Gems页面中的按钮：

在“ 编辑项目路径映射”对话框中，指定本地和远程项目根路径：

配置映射后，请使用“ 将宝石安装到本地解释器”指令来安装宝石 。

请注意，当您使用远程解释器时，IntelliJ IDEA会将gem从远程计算机下载到本地缓存，以便能够使用代码洞察功能。在这种情况下，您可能会遇到在IDE外部的远程计算机上安装了新的gem的情况。要更新存储在本地缓存中的一组宝石，请单击“ 同步宝石” Ruby SDK和Gems页面中的按钮。
