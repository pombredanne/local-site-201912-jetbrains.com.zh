IntelliJ IDEA 2019.3帮助

Java SE

以下基本操作方法旨在帮助您开始在IntelliJ IDEA中使用Java。

将源文件移动到子文件夹中

您不应将源（.java）文件保留在项目根目录中。如果当前是这种情况，请在项目根目录中创建一个子目录，然后将所有源文件移动到该子目录中。

为.java文件启用编码帮助

将带有.java文件的文件夹标记为包含源代码后， Java编码辅助功能就会自动打开。在“ 项目”工具窗口（“ 视图” |“工具窗口” |“项目” ）中，右键单击该文件夹，指向“ 标记目录为”，然后选择“ 源根目录” 。

使Java API可被您的代码访问

用Java编写时，通常，您将重用（即引用） Java API类。这些类在JDK中可用。因此，要使您的代码可访问Java API，您应该下载然后指定JDK：

  1. 在编辑器中打开.java文件：在“ 项目”工具窗口中选择文件，然后按F4 。

    如果您尚未指定JDK，则在编辑区域上方将显示一条消息，消息未定义 。

  2. 单击设置SDK 。

  3. 在打开的对话框中，选择您的JDK并单击确定 ，或单击配置 ，单击 ，选择JDK ，然后选择安装了所需JDK的目录。

为包创建文件夹结构或指定包前缀

如果您的软件包还没有合适的文件夹结构（假设您的软件包名称为com.example.mypackage ），您有两个选择可导致大致相同的结果：

  • 在源目录中创建相应的文件夹结构 （例如/ com / example / mypackage ）：

    在编辑器的package语句中（例如package com.example.mypackage; ），将光标放在包名称中，按Alt+Enter并选择移至包'com.example.mypackage' 。

  • 在没有实际创建文件夹结构的情况下为源文件夹分配一个包前缀 ：

    1. 打开“ 项目结构”对话框（例如Ctrl+Shift+Alt+S ），选择模块，然后选择您的模块。

    2. 在“ 源”选项卡的右侧，在源文件夹旁边，单击 。

    3. 在打开的对话框中，指定前缀（例如com.example.mypackage ）。

使JAR中的类可被您的应用访问

假设您有一个JAR ，其中包含要在应用程序中重用的类。在这种情况下，您应该将JAR添加到模块的依赖项中。结果，JAR类变得可以在您的代码中进行引用，并且在您构建应用时将其包含在您的应用中。

  1. 打开“ 项目结构”对话框（例如Ctrl+Shift+Alt+S ），选择模块，然后选择您的模块。

  2. 选择依赖项选项卡，单击并选择JAR或目录 。

  3. 在打开的对话框中，选择您的JAR文件。

编译.java文件

如果您按照本页前面的说明进行了所有操作，则编译一个.java文件或多个文件都不会成为问题。只需选择下列选项之一： 建设|构建项目或构建|构建模块'module_name' 。

如果编译有问题，则IntelliJ IDEA会通知您有关原因，并提供解决问题的提示。

运行你的应用

如果编译成功 ，则可以运行您的应用程序：

  1. 在编辑器中使用main（）方法打开该类。

  2. 在main（）方法旁边的左空白处，单击运行标记（绿色箭头），然后选择Run'.main（）' 。

将您的应用打包为JAR

要将应用程序打包为JAR，应配置JAR工件，然后对其进行构建：

  1. 打开项目结构对话框Ctrl+Shift+Alt+S ，然后选择Artifacts 。

  2. 请点击 ，选择JAR，然后选择来自具有依赖项的模块 。

  3. 在打开的对话框中，使用main（）方法选择类（“ 主类”字段）。

  4. 通过在“ 项目结构”对话框中单击“ 确定”来保存工件设置。

  5. 选择构建|构建构件，选择您的构件，然后选择构建 。（如果只有一个工件，您可以按Enter弹出“ 构建工件”菜单时。）
