从Java代码生成WSDL文档

Web服务的可用功能，访问它们的端口，可接受的请求格式，生成的响应的格式等都反映在Web服务WSDL描述符中 ，该描述符通常在Web服务部署期间在服务器上生成。Web服务客户端开发的主要部分是实现对服务请求的生成，并根据WSDL描述符设置来解析来自服务的响应。

假设您已经开发了Web服务，并且希望在部署Web服务本身之前（即，在服务器上生成WSDL描述符之前）开始其客户端开发。使用IntelliJ IDEA，可以在部署之前生成它。

从Java代码创建WSDL描述符

  1. 在编辑器中选择所需的类名称。

  2. 选择工具|网络服务|从主菜单上的Java代码生成WSDL或选择Web服务| Web服务。从上下文菜单中的Java代码生成WSDL 。

  3. 在打开的“ 从Java生成WSDL”对话框中，指定以下内容：

    • Web服务的名称和URL地址。

    • 访问Web服务的公共操作时使用的协议和编码样式。

    • 类型信息，包括名称，操作，参数和构成Web服务接口的数据。

    准备就绪后，单击确定 。
