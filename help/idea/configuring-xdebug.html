配置Xdebug

仅当安装并启用PHP插件时，以下内容才有效。

下载Xdebug

  • 下载与您的PHP版本兼容的Xdebug扩展 ，并按照安装指南中的说明进行安装 。

    如果使用的是AMP软件包 ，则可能已经安装了Xdebug扩展名。请参阅包装的特定说明。

将Xdebug与PHP解释器集成

  1. 在编辑器中打开活动的php.ini文件：

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在“ 语言和框架”下单击“ PHP ” 。

    2. 在打开的PHP页面上，单击 CLI解释器字段旁边。

    3. 在打开的“ CLI解释器”对话框中，“ 配置文件”只读字段显示活动php.ini文件的路径。单击“ 在编辑器中打开” 。

  2. 要禁用阻止Xdebug的Zend Debugger和Zend Optimizer工具，请删除或注释掉php.ini文件中的以下几行：

    zend_extension = zend_extension =

  3. 要启用Xdebug，请找到或创建[xdebug] php.ini文件中的部分，并进行如下更新：

    [xdebug] zend_extension =“ ” xdebug.remote_enable = 1 xdebug.remote_port =“ ”（默认端口是9000）

    在PHP 5.3和更高版本中，您只需要使用zend_extension ，不是zend_extension_ts ， zend_extension_debug ， 要么extension 。

    要通过Xdebug代理启用多用户调试 ，请找到xdebug.idekey设置并为其指定一个值。此值将用于在Xdebug代理服务器上注册IDE。

  4. 保存并关闭php.ini文件。

  5. 通过执行以下任一操作来验证Xdebug的安装：

    • 在命令行中，运行以下命令：

      php --version

      输出应在已安装的扩展中列出Xdebug ：

    • 创建一个包含以下代码的php文件：

      在浏览器中打开文件。的phpinfo输出应包含Xdebug部分：

在IntelliJ IDEA中配置Xdebug

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择语言和框架| PHP 。

  2. 检查与所选PHP解释器关联的Xdebug安装：

    1. 在“ PHP”页面上，从“ CLI解释器”列表中选择相关的PHP安装，然后单击“确定”。 旁边的领域。该列表显示了IntelliJ IDEA中可用的所有PHP安装，请参阅“ 配置本地PHP解释器”和“ 配置远程PHP解释器” 。

    2. 打开的“ CLI解释器”对话框显示以下内容：

      • 所选PHP安装的版本。

      • 与所选PHP安装（Xdebug或Zend Debugger）关联的调试引擎的名称和版本。如果未配置调试器，则IntelliJ IDEA将显示Debugger: Not installed 。

    或者，打开Xdebug检查器 ，粘贴phpinfo() ，然后单击分析我的phpinfo（）输出 。在“ 验证调试引擎的配置”中了解有关检查Xdebug安装的更多信息。

  3. 定义Xdebug行为。单击“ PHP”节点下的“ 调试 ”。在打开的“ 调试”页面上，在Xdebug区域中指定以下设置：

    • 在“ 调试端口”字段中，指定该工具将与IntelliJ IDEA通信的端口。

      该端口号必须与php.ini文件中指定的端口号完全相同：
      xdebug.remote_port =
      默认情况下，Xdebug侦听端口9000 。

    • 要使IntelliJ IDEA通过“ 调试端口”字段中指定的端口接受来自Xdebug引擎的任何传入连接，请选中“ 可以接受外部连接”复选框。

    • 选择未指定路径映射时在第一行强制中断复选框使调试器到达并立即停止，并在服务器页面上打开未映射到项目中任何文件的文件。调试器在此文件和调试工具窗口的第一行停止。变量显示以下错误消息： 在服务器上找不到文件的本地副本以及链接单击以设置映射 。单击链接以打开“ 解决路径映射问题”对话框，并将问题文件映射到其本地副本。

      清除此复选框后，调试器在到达并打开未映射的文件时不会停止，文件仅被处理，并且不显示任何错误消息。

    • 如果脚本不在项目中，请在第一行选择“ 强制中断”复选框使调试器在到达并在当前项目外打开文件后立即在第一行停止。清除此复选框后，调试器在打开当前项目之外的文件时继续。

  4. 在“ 外部连接”区域中， 指定您希望IntelliJ IDEA如何处理从主机以及通过未注册为部署服务器配置的端口接收的连接。

    • 忽略通过未注册的服务器配置进行的外部连接 ：选中此复选框可使IntelliJ IDEA忽略从主机以及未注册为部署服务器配置的端口收到的连接。选中此复选框后，IntelliJ IDEA不会尝试自动创建部署服务器配置。

    • 在PHP脚本的第一行中断 ：选中此复选框在调试器和IntelliJ IDEA之间建立连接后立即停止它（而不是自动运行直到到达第一个断点）。或者，打开运行|从主菜单在PHP脚本选项的第一行中断 。

    • 最高同时连接：使用此旋转框限制可以同时处理的外部连接的数量。

默认情况下，IntelliJ IDEA仅侦听传入的IPv4连接。要启用IPv6支持，您需要在IntelliJ IDEA JVM选项中进行调整：

  1. 选择帮助|在主菜单中编辑“自定义VM选项 ”。

  2. 在打开的.vmoptions文件中，删除-Djava.net.preferIPv4Stack=true线。

  3. 重新启动IntelliJ IDEA。

配置Xdebug以按需模式使用

IntelliJ IDEA 2016.2和更高版本支持按需模式，您可以在其中禁用全局PHP安装的Xdebug，并仅在调试命令行脚本或需要代码覆盖率报告时按需自动启用Xdebug。这使您的命令行脚本（包括Composer和单元测试）运行得更快。

  1. 为命令行脚本禁用Xdebug：

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在“ 语言和框架”下选择“ PHP ” 。

    2. 从PHP可执行文件列表中，选择相关的PHP解释器，然后单击在它的旁边。在打开的“ CLI解释器”对话框中，单击配置文件旁边的“ 在编辑器中打开”链接： 文件。关闭所有对话框，然后切换到打开php.ini文件的选项卡。

    3. 在php.ini文件中，找到[xdebug]部分，并在其中添加以下注释行;在介词中：

      ; [xdebug]; zend_extension =“ ”

    4. 打开“ CLI解释器”对话框，然后单击 PHP可执行文件字段旁边。IntelliJ IDEA通知您未安装调试器：

  2. 要使IntelliJ IDEA在必要时激活Xdebug，请在“ 其他”区域的“ 调试器扩展”字段中指定其路径。手动输入路径或单击然后在打开的对话框中选择位置。

配置Xdebug以在即时模式下使用

IntelliJ IDEA支持在即时（JIT）模式下使用Xdebug，因此它不会一直附加到您的代码上，而是仅在发生错误或引发异常时才连接到IntelliJ IDEA。Xdebug操作模式通过xdebug.remote_mode设置切换，默认情况下设置为req 。该模式可用于调试命令行脚本和Web服务器调试。

根据要调试命令行脚本还是使用Web服务器，请使用以下方案之一。

命令行脚本

对于调试命令行脚本，请指定自定义-dxdebug.remote_mode=jit指令作为附加配置选项 ：

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，导航到语言和框架| PHP 。

  2. 从PHP可执行文件列表中，选择相关的PHP解释器，然后单击在它的旁边。

  3. 在打开的“ CLI解释器”对话框中，单击 附加区域中的配置选项字段旁边。

  4. 在打开的“ 配置选项”对话框中，单击添加一个新条目，然后键入-dxdebug.remote_mode在配置指令字段中， jit在值字段中。

    单击“ 确定”后 ，将返回“ CLI解释器”对话框，其中显示“ 配置选项”字段-dxdebug.remote_mode=jit 。

Web服务器调试

从主菜单中，选择运行| Web服务器调试验证 。

在打开的“ Web服务器调试验证”对话框中，选择要在其上验证调试器的Web服务器。

  • 选择本地Web服务器或共享文件夹 检查与本地Web服务器关联的调试器。

    • 创建验证脚本的路径 ：在此字段中，指定将在其中创建验证脚本的服务器文档根目录下的文件夹的绝对路径。对于Inplace类型的Web服务器，该文件夹位于项目根目录下。

      该文件夹必须可通过http访问。

    • 验证脚本的URL ：在此字段中，输入将在其中创建验证脚本的文件夹的URL地址。如果项目根目录已映射到可通过http访问的文件夹，则可以指定项目根目录或其下的任何其他文件夹。

  • 选择远程Web服务器 检查与远程服务器关联的调试器。

    • 创建验证脚本的路径 ：在此字段中，指定将在其中创建验证脚本的服务器文档根目录下的文件夹的绝对路径。该文件夹必须可通过http访问。

    • 部署服务器 ：在此字段中，指定本地服务器或远程服务器类型的服务器访问配置以访问目标环境。有关详细信息，请参阅配置与Web服务器的同步 。

      从列表中选择配置，或单击浏览 在部署对话框中 。

单击验证 让IntelliJ IDEA创建一个验证脚本，将其部署到目标远程环境中并在其中运行。

配置在Docker容器中运行的Xdebug

要配置在Docker容器中运行的Xdebug，请在Dockerfile中提供特定于Xdebug的参数，例如：

＃设置调试器RUN echo“ xdebug.remote_enable = 1” >> /etc/php/7.0/apache2/php.ini RUN echo“ xdebug.remote_host = your_host_name_or_ip” >> /etc/php/7.0/apache2/php.ini

在此示例中，我们将修改/etc/php/7.0/apache2/php.ini，以提供Xdebug扩展的路径，以及remote_enable和remote_host Xdebug参数。

请注意xdebug.remote_host值应替换为运行PhpStorm的计算机的IP地址，该地址可从Docker容器访问。如果您使用Windows的 Docker或Mac的Docker ，则可以设置xdebug.remote_host至host.docker.internal ，它会自动解析为主机的内部地址，使您可以轻松地从容器连接到主机。

配置在Vagrant实例上运行的Xdebug

要配置在Vagrant实例上运行的Xdebug，请连接到Vagrant计算机，并在php.ini文件中提供Xdebug特定的参数：

[xdebug] zend_extension =“ ” xdebug.remote_enable = 1 xdebug.remote_host = 10.0.2.2 xdebug.remote_port = 9000

请注意xdebug.remote_host值是10.0.2.2 。这是默认Vagrant设置中使用的网关，该网关允许从实例连接到运行IntelliJ IDEA的主机。
