查找涵盖范围的未使用代码

IntelliJ IDEA使您可以在客户端应用程序中找到未使用的JavaScript，TypeScript和CSS代码。当您以特殊的代码覆盖率模式运行应用程序时，IntelliJ IDEA会创建一个报告，显示每个文件和文件夹中使用了多少代码。多亏了源地图 ，该报告显示了源文件的覆盖范围，但没有显示实际在浏览器中运行的编译代码的覆盖范围。

以代码覆盖率模式运行应用程序

  1. 创建类型为JavaScript Debug的运行/调试配置：

    选择运行|从主菜单编辑配置 ， 点击在工具栏上，然后从列表中选择JavaScript Debug 。 在打开的“运行/调试配置：JavaScript调试”对话框中，指定应用程序运行所在的URL地址。可以从浏览器的地址栏中复制此URL。

    由于JavaScript调试配置仅适用于Chrome，因此代码覆盖率模式也仅在此浏览器中可用。

  2. 在工具栏的“ 选择运行/调试配置”列表中选择新创建的配置，然后单击在列表旁边。

    在运行配置中指定的URL地址在浏览器中打开。

  3. 要了解页面加载期间执行了什么代码，只需加载应用程序，然后通过单击停止它在工具栏上， ，或在“ 运行工具”窗口中。如果您需要应用程序某些特定功能的覆盖率报告，请在浏览器中触发这些功能，然后单击停止应用程序。

  4. 在“ 覆盖率”工具窗口中查看报告。“ 项目”工具窗口显示有关文件和文件夹覆盖范围的信息。在编辑器中，装订线在已执行的行旁边显示绿色标记，在未执行的行旁边显示红色标记。您还可以将鼠标悬停在行标记上，查看每行代码被击中了多少次。
