处理项目

如何在IntelliJ IDEA中打开NetBeans项目？

使用文件|新品|从现有源进行项目，然后选择NetBeans项目目录。

打开“ 导入项目”向导后，选择“ 从现有来源创建项目”选项，然后按照向导的说明进行操作。

IntelliJ IDEA会将必要的定义文件（ .idea目录）添加到您的项目目录中。NetBeans .nbproject目录和build.xml将保持不变，并且您可以将IntelliJ IDEA与NetBeans一起使用。

在导入期间，IntelliJ IDEA将修复缺少的库，为不同的Web框架添加构面，并创建运行配置。

如果将Maven与NetBeans一起使用，并且要将Maven项目导入到IntelliJ IDEA中，请选择“ 文件” |“文件”。打开 ，然后选择项目的pom.xml 。您仍然需要配置运行配置，但是，所有项目依赖关系都将得到解决。

项目和模块之间有什么区别？

IntelliJ IDEA为整个代码库创建一个项目，并为其每个单独的组件创建一个模块。因此，IntelliJ IDEA模块更像一个NetBeans项目。

IntelliJ IDEA中是否存在基于目录的项目格式？

是的，有一个.idea目录，用于存储项目定义XML文件。有关更多信息，请参见项目 。

如何为我的项目更改JDK？

  1. 打开项目结构对话框（ 文件|项目结构或Ctrl+Shift+Alt+S ）。

  2. 在“ 平台设置”下 ，选择“ SDK” 。

  3. 请点击 ，选择JDK并指定JDK安装目录。

  4. 点击应用 。

  5. 在“ 项目设置”下 ，选择“ 项目” 。

  6. 在Project SDK下 ，从列表中选择JDK。

  7. 单击确定 。

有关更多信息，请参见SDK 。

如何将库添加到我的项目中？

  1. 打开项目结构对话框（ 文件|项目结构或Ctrl+Shift+Alt+S ）。

  2. 在“ 项目设置”下 ，选择“ 库” 。

  3. 请点击 ，选择Java并指定库位置。

  4. 选择将在其中使用该库的模块。

  5. 单击确定 。

有关更多信息，请参见使用库 。

如何为我的项目配置Web框架？

在IntelliJ IDEA Ultimate中（社区版中没有相应的功能）：

  1. 打开“ 项目”工具窗口（例如“ 视图” |“工具窗口” |“项目” ）。

  2. 右键单击所需的模块，然后选择添加框架支持 。（在模块级别启用了框架支持。）

  3. 在打开的“ 添加框架支持”对话框中，选择要支持的框架，指定关联的设置，然后单击“ 确定” 。

有关更多信息，请参见添加框架和技术 ，例如，为现有模块启用JSF支持 。

运行按钮被禁用。如何运行我的应用程序？

“运行”按钮被禁用，因为您的项目中没有运行配置 。

如果您有一个带有main()方法，在编辑器中打开相应的文件，右键单击编辑区域，然后选择Run'.main（）' 。结果，将自动创建必要的运行配置，然后执行。

您可以自己创建运行配置：在主菜单中，选择“运行” |“运行”。编辑配置| 。请点击添加新配置并选择要如何运行应用程序。

如何为我的项目生成Ant构建脚本？

选择构建|生成Ant Build 。有关更多信息，请参见生成Ant构建文件 。

如何同时在IntelliJ IDEA中打开多个项目？

使用IntelliJ IDEA可以同时处理多个项目。为此，您只需要打开一个项目，而另一个项目已经打开，然后选择添加到当前打开的项目 。

如何关闭项目？

选择文件|关闭项目 。您也可以使用File |退出以关闭所有打开的项目并退出IntelliJ IDEA。

“选项”对话框在哪里？

在IntelliJ IDEA中，“ 设置”对话框用于类似目的。要打开此对话框，请按Ctrl+Alt+S 。

还有“ 项目结构”对话框（ Ctrl+Shift+Alt+S ），让您管理JDK，库，模块依赖项等。

有关更多信息，请参见设置/首选项对话框和项目结构对话框 。

我如何开始VCS集成？

IntelliJ IDEA支持最流行的版本控制系统，包括Git，Subversion，Mercurial，Perforce等。可以在“ 设置/首选项”对话框的“ 版本控制”页面上配置项目的VCS集成。有关详细信息，请参见版本控制 。

使用代码编辑器

我可以在IntelliJ IDEA中使用NetBeans键绑定吗？

是的你可以。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，在“ 外观和行为”下选择“ 键盘映射 ”。

  2. 在对话框的右侧，在Keymaps旁边，从列表中选择NetBeans 6.5 。

IntelliJ IDEA中的代码完成如何工作？

键入一个或两个字母后，将自动显示代码完成建议列表。要缩小此列表的范围，请使用：

  • Ctrl+Space 。该列表将简化为关键字以及当前上下文中可用的类，方法和字段的名称。请注意，按时列表会更改Ctrl+Space第二次或第三次。

  • Ctrl+Shift+Space 。仅显示适合当前上下文的类型。

有关更多信息，请参见代码完成 。

IntelliJ IDEA中的本地历史记录与NetBeans中的历史记录是否有所不同？

通常，IntelliJ IDEA中的本地历史记录会更详细。您对目录，文件，类，方法或字段或代码块的任何处理都会反映在本地历史记录中。本地历史记录还包括VCS操作。

有关更多信息，请参阅本地历史记录 。

IntelliJ IDEA中是否有任何特殊的代码分析功能？

IntelliJ IDEA可以分析依赖关系，数据流和堆栈跟踪，查找重复项并评估代码质量。只需查看“ 分析”菜单中的选项即可。

我可以在IntelliJ IDEA中启用“标记出现”吗？

您可以。IntelliJ IDEA中的相应选项称为在插入符号处突出显示元素的用法 。默认情况下启用此选项。

以防万一：

  1. 打开设置对话框Ctrl+Alt+S 。

  2. 在“ 编辑器”类别中，选择“ 常规” 。

  3. 在对话框的右侧，在“插入 符号移动突出显示”下，选中“ 插入符号突出显示用法”复选框。

  4. 单击确定 。

我可以在IntelliJ IDEA中启用“保存时编译”吗？

您可以。

要在每次保存（或自动保存）时启用自动编译， 请在“ 设置”对话框中启用“自动创建项目”选项：

  1. 打开设置对话框Ctrl+Alt+S 。

  2. 在“ 构建，执行，部署”类别中，选择“ 编译器” 。

  3. 在对话框的右侧，选中自动生成项目复选框。

  4. 单击确定 。

请注意，默认情况下，IntelliJ IDEA会自动保存更改的文件，因此您无需使用Ctrl+S与其他IDE一样频繁。

我可以在IntelliJ IDEA中启用“保存时部署”吗？

IntelliJ IDEA设置中没有此类选项，但是，通过在相应的运行配置中选择适当的应用程序更新选项，可以获得类似的结果。

有关更多信息，请参见更新应用程序服务器上的应用程序 。

（相应的功能仅在IntelliJ IDEA Ultimate中可用。社区版不提供与应用程序服务器的集成。）

使用插件

我可以在IntelliJ IDEA中使用NetBeans插件吗？

不幸的是没有。但是，“开箱即用”的IntelliJ IDEA提供了许多作为NetBeans插件实现的功能。此外，还有很多IntelliJ IDEA插件，因此您始终可以找到功能与您最喜欢的NetBeans插件类似的IntelliJ IDEA插件。

如何找到所需的插件？

与插件相关的所有功能都在“ 设置”对话框的“ 插件”页面上Ctrl+Alt+S 。您可以查找，下载，安装和更新插件，以及启用和禁用它们。

有关更多信息，请参阅插件和管理插件 。

如何安装计算机上可用的插件？

  1. 打开设置对话框Ctrl+Alt+S 。

  2. 在左侧窗格中，选择插件 。

  3. 在“ 插件”页面的下部，单击从磁盘安装插件 。

  4. 在打开的对话框中，选择插件文件（通常是JAR或ZIP）。

  5. 单击确定 。

  6. 如果询问，请重新启动IntelliJ IDEA。

我想为IntelliJ IDEA编写一个插件。有什么指示吗？

是的，看看：

  • IntelliJ Platform SDK开发人员指南 。

  • IntelliJ IDEA插件页面上的插件开发人员信息

  • 开放式API和插件开发

是否可以使用IntelliJ IDEA构建NetBeans RCP应用程序？

有可能，但是您将无法获得与NetBeans一样的支持（向导，菜单操作等）。看看使用IntelliJ IDEA进行NetBeans平台开发 。

配置PHP开发环境

开始之前需要什么配置？

启动后，无需任何配置即可使用许多IntelliJ IDEA功能。尽管如此，要充分利用运行PHP应用程序的优势，您需要配置PHP解释器和服务器。

如果计划在本地启动该应用程序，则需要在IntelliJ IDEA中安装并注册一个PHP引擎 ，以及安装，配置并与IntelliJ IDEA集成的Web服务器。您可以单独安装这些组件，也可以使用AMP软件包 。有关初始环境配置的更多详细信息，请参阅《 配置PHP开发环境》 。

如果要直接在远程主机上运行和调试应用程序，则唯一需要的是在IntelliJ IDEA中注册对此主机的访问权限以启用同步。

如何开始部署到远程主机？

如果您已从远程主机中签出项目，则已经配置了部署服务器。否则，您需要在“设置/首选项”对话框的“ 部署”页面上对其进行配置（可以是FTP / SFTP / FTPS服务器或已安装/本地文件夹）。“ 远程主机工具”窗口位于IntelliJ IDEA窗口的右侧，可以方便地浏览远程服务器并执行各种操作。

有关详细信息，请参见部署应用程序 。

如何开始调试？

IntelliJ IDEA支持Xdebug和Zend Debugger的调试和性能分析。可以使用零配置调试工作流程 ，这意味着要开始调试，您只需要：

  • 单击“ 开始侦听PHP调试连接” 在IDE的工具栏上。

  • 通过单击行旁边的编辑器装订线，在代码中放置断点 。

  • 使用插件或浏览器书签在浏览器中开始调试。
