IntelliJ IDEA 2019.3帮助

摇篮

IntelliJ IDEA支持与Gradle的全功能集成，可帮助您自动化构建过程。您可以轻松地创建一个新的Gradle项目，打开并同步一个现有的项目，同时处理多个链接的项目并进行管理。

创建一个新的Gradle项目

  1. 启动“ 新建项目”向导 。如果IntelliJ IDEA当前未打开任何项目，请在欢迎屏幕上单击“ 创建新项目 ”。否则，选择文件|新品|从主菜单进行项目 。

  2. 从左侧的选项中选择Gradle 。

  3. 指定项目SDK和其他框架或库（IntelliJ IDEA将适当的插件添加到build.gradle文件中）。单击下一步 。

    若要查看IntelliJ IDEA如何选择已安装的Gradle JVM版本，请参阅Gradle JVM选择部分。

  4. 在向导的下一页上，指定类似于Maven坐标的字段。如果您决定将项目部署到Maven存储库，则这些设置可能会有所帮助。您指定的字段将添加到build.gradle文件中。

    • GroupId- groupId新项目。如果您计划在本地部署项目，则可以忽略此字段。

    • ArtifactId- artifactId将其添加为新项目的名称。

    • 版本 - version新项目。默认情况下，此字段是自动指定的。

    有关Maven坐标的更多信息，请参见Maven命名约定 。

    如果指定了父级Gradle项目，则可以从父级继承指定的字段（单击Inherit ）。
    单击下一步 。

  5. 指定名称和位置设置。点击完成 。

打开一个现有的Gradle项目

如果您在项目中启用了离线模式 ，则打开或重新导入项目可能会失败。要解决此问题，请禁用离线模式并重新导入您的项目。

  1. 如果当前在IntelliJ IDEA中没有打开任何项目，请在欢迎屏幕上单击“ 打开 ”。否则，选择文件|从主菜单打开 。

  2. 在打开的对话框中，选择一个包含Gradle项目描述build.gradle的文件 。单击确定 。

  3. 在打开的对话框中，单击“ 以项目形式打开” 。IntelliJ IDEA在IDE中打开并同步项目。

如果需要调整Gradle设置选项，请参阅Gradle设置 。

若要查看IntelliJ IDEA如何选择已安装的Gradle JVM版本，请参阅Gradle JVM选择部分。

将Gradle项目链接到IntelliJ IDEA项目

您可以在一个IntelliJ IDEA项目中包含多个Gradle项目。如果将部分代码保留在不同的项目中，需要处理一些旧项目，使用Gradle组合构建或使用微服务 ，这可能会有所帮助。您可以在IntelliJ IDEA中链接此类项目并同时进行管理。

当您打开 Gradle项目时，会自动建立项目的链接并启用Gradle工具窗口。

如果IntelliJ IDEA项目未链接到Gradle项目，则禁用Gradle工具窗口。在这种情况下，IntelliJ IDEA将显示一条带有链接的消息，该链接可让您快速重新导入Gradle项目并启用Gradle工具窗口。如果Gradle工具窗口处于活动状态，则您至少链接了一个Gradle项目。

  1. 打开Gradle工具窗口。

  2. 在Gradle工具窗口中，单击附加一个Gradle项目。

  3. 在打开的对话框中，选择所需的build.gradle文件，然后单击OK 。

  4. 在“ 从Gradle导入模块”窗口中，为要尝试链接的Gradle项目指定选项，然后单击“ 确定” 。

    该项目已链接。Gradle工具窗口显示工具栏和Gradle实体的树状视图。

如果需要链接回先前未链接的项目 ，请在“ 项目”工具窗口中，右键单击添加的build.gradle；如果它是Gradle Kotlin模块，则右键单击build.gradle.kts文件，然后选择Import Gradle Project 。

向现有项目添加新的Gradle模块

您可以将Gradle模块添加到已经在其中工作的项目中。

  1. 在项目中，从主菜单中选择File |。新品|模块打开新模块向导。

  2. 如果现有项目不是Gradle项目，则添加模块的过程与创建新的Gradle项目相同 。如果现有项目是Gradle项目，则添加新模块的过程会更短。您需要在ArtifactId字段中指定模块的名称。其余信息将自动添加，您可以使用默认设置，也可以根据自己的喜好进行更改。另外，请注意，默认情况下，“ 作为模块添加到”字段显示要向其添加模块的项目的名称。您可以点击如果您有其他链接的Gradle项目，请选择其他名称。

访问Gradle设置

使用Gradle设置可以为每个链接的Gradle项目，Gradle版本，项目更改的导入等配置构建和运行操作。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 构建，执行，部署” |摇篮 。

    请点击在工具栏上的Gradle工具窗口中，以访问Gradle设置。

  2. 在“ Gradle设置”页面上，配置可用选项，然后单击“ 确定”保存更改。

为项目配置Gradle版本

IntelliJ IDEA允许您使用不同的选项来为Gradle项目配置Gradle版本。您可以使用默认的Gradle包装器，将Gradle包装器用作任务或配置本地Gradle分发。

  1. 选择在Gradle工具窗口中快速访问Gradle设置页面。

  2. 在“ 使用Gradle来自”列表中，选择以下选项之一：
    • 'gradle-wrapper.properties'文件 ：这是建议的默认选项，使用Gradle包装器 。

      在这种情况下，您可以将Gradle版本的更新委派给Gradle并自动下载该版本的Gradle。此选项还允许您使用精确的Gradle版本进行构建。Gradle版本保存在项目的gradle目录中的gradle-wrapper.properties文件中，可帮助您消除任何Gradle版本问题。

    • Gradle构建脚本中的'wrapper'任务 ：选择此选项可根据wrapper任务配置。如果您希望控制在项目中使用哪个Gradle版本，可能会很方便。

      如果您使用了默认的Gradle包装器选项，然后切换到了Gradle wrapper任务配置，您在任务中所做的更改将在项目导入期间自动更新。

    • 指定位置 ：如果要手动下载并使用特定的Gradle版本，请选择此选项。指定您的Gradle安装位置和JVM，当您导入指定的Gradle项目并执行其任务时，IntelliJ IDEA将在其下运行Gradle。

    单击确定保存更改。

为Gradle项目添加VM选项

您可以使用gradle.properties文件为Gradle项目指定VM选项。

  1. 创建或打开您的Gradle项目。

  2. 在“ 项目”工具窗口中，右键单击项目，然后从上下文菜单中选择“ 新建” |“新建”。文件 。

  3. 在“ 新建文件”对话框中，输入gradle.properties作为文件名，然后单击“ 确定” 。

  4. 在编辑器中打开创建的文件，然后添加所需的VM选项。

有关更多信息，请参阅Gradle文档 。
