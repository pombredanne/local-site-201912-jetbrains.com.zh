项目反应堆

Project Reactor是一个基于Reactive Streams的开源框架，用于开发非阻塞应用程序。IntelliJ IDEA为Reactive应用程序提供全面支持，包括代码完成，检查，快速修复和专用调试模式。

添加Reactor支持

Reactor适用于Kotlin ， Java 8及更高版本，以及Android SDK 26 （Android O）及更高版本。

有几种向项目中添加Reactor支持的选项：您可以将必要的依赖项添加到Maven或Gradle项目的构建文件中，或者可以手动下载Reactor库。对于Spring Boot应用程序，IntelliJ IDEA允许您在创建新项目时添加所需的依赖项。

将Reactor添加到Maven项目

对于Maven项目，建议您将材料清单（BOM）与核心一起导入，因为这可确保Reactor组件协同工作。

  1. 在里面pom.xml文件，添加以下依赖项以导入BOM表：

    io.projectreactor 反应器物料清单--SR1 pom 进口

  2. 添加以下依赖项以导入Reactor：

    io.projectreactor 反应堆堆芯

将Reactor添加到Gradle（5.0和更高版本）

对于Gradle项目，建议您将材料清单（BOM）与核心一起导入，因为这可确保Reactor组件协同工作。

  • 在里面build.gradle ，添加以下依赖项：

    依赖项{//导入BOM实现平台（'io.projectreactor：reactor-bom：Dysprosium-SR1'）//添加没有版本号实现的依赖项'io.projectreactor：reactor-core'}

将Reactor添加到Gradle（版本4.x和更早版本）

对于Gradle项目，建议您将材料清单（BOM）与核心一起导入，因为这可确保Reactor组件协同工作。

以前的Gradle版本不支持BOM，但是，您可以使用Spring gradle-dependency-management插件将其导入到项目中。

  1. 通过将以下代码添加到Gradle插件门户获取插件build.gradle ：
    插件{id“ io.spring.dependency-management”版本“ 1.0.6。发布” }

    使用最新的可用版本；我们使用的是1.0.6。以RELEASE版本为例。

  2. 通过添加以下代码导入BOM表：

    依赖管理{导入{mavenBom“ io.projectreactor：reactor-bom：Dysprosium-SR1”}}

  3. 添加Reactor支持：

    依赖项{编译'io.projectreactor：reactor-core'}

将Reactor库添加到项目中

如果使用本机IDE构建器构建项目，则可以将Reactor支持添加为库。

  1. 从主菜单中，选择文件|项目结构 Ctrl+Shift+Alt+S或点击在工具栏上。

  2. 选择项目设置|库并单击 |从Maven 。

  3. 在打开的对话框中，指定库工件io.projectreactor:reactor-core:jar:3.3.0.RELEASE然后单击确定

    使用最新的可用版本；我们使用的是3.3.0。以RELEASE版本为例。

反应堆调试模式

IntelliJ IDEA使您可以在程序暂停后更方便地查看框架堆栈。

如果您具有通过使用启用的调试模式Hooks.onOperatorDebug() ， ReactorDebugAgent ， 或者checkpoint()操作员，此视图在“ 调试”工具窗口的“ 框架”选项卡上显示失败操作的回溯。

此模式可以帮助您更好地检查框架并更容易理解为什么将特定参数传递给Reactive应用程序中的方法。

默认情况下启用此功能。如果要禁用它， 在里面 设置/首选项对话框Ctrl+Alt+S ，选择语言和框架| Reactor并取消选中“ 启用堆栈框架自定义”复选框。

有关如何在IntelliJ IDEA中调试应用程序的更多信息，请参阅启动调试器会话并检查挂起的程序 。

高级反应堆检查

IDE提供了一个选项，用于调整不适当的线程阻塞方法调用检查，该检查报告报告在不应阻塞线程的代码片段中检测到的线程阻塞方法调用。使用此选项，IDE可以通过处理以下内容本地了解操作员将在哪个线程上执行subscribeOn和publishOn操作员。

默认情况下启用此功能。如果要禁用它， 在里面 设置/首选项对话框Ctrl+Alt+S ，选择语言和框架|反应器并取消选择使用高级分析来检测非阻塞作用域选项。

在Reactor中完成Postfix

IntelliJ IDEA为使用Reactor的项目提供后缀代码完成。后缀补全可以根据您键入的内容将已经键入的表达式转换为其他表达式。

在具有Reactor支持的项目中，IDE可以使用合适的表达式来包装表达式reactor.core.publisher.Flux工厂还是reactor.core.publisher.Mono厂。

有关postfix完成的更多信息，请参阅Postfix代码完成 。
