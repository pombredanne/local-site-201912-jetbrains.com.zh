IntelliJ IDEA 2019.3帮助

重构Android XML布局文件

除了常见的重构 ，IntelliJ IDEA还为Android布局定义XML文件提供了许多特定于Android的重构。这些重构中的大多数都可以在两种设计模式下使用 ：

  • 手动模式，直接在打开目标布局定义文件的编辑器选项卡中，并在“ 结构”工具窗口中 。

  • 在Android UI设计器中 ，从“ 设计”窗格或“ 组件树”中 。

通过在打开布局定义文件的编辑器中的“ 设计”和“ 文本”选项卡之间切换，可以在这些模式之间切换。

在本主题中：

  • 提取样式
  • 内联样式
  • 提取版面
  • 内联布局重构

提取样式

Android样式有助于将应用程序设计与应用程序功能区分开，就像层叠样式表（CSS）一样 。

通过提取样式重构，您可以从布局XML标签创建样式。IntelliJ IDEA在/res/values/styles.xml文件中创建样式声明，并将标记属性转换为在item元素。好处是现在您不再需要解析整个资源定义文件来更新标签属性来编辑布局。您需要做的就是更新样式定义，然后在引用样式的任何地方自动应用更改。

应用提取样式重构

  1. 请执行以下任一操作：

    • 在可视模式下 ，选择“ 设计”窗格中的画布上或“ 组件树”中的定义中使用了源XML标签的组件 。

    • 在手动模式下 ，将光标放置在源XML标记内的任何位置。

  2. 从主菜单中，选择“ 重构|重构或按Ctrl+Shift+Alt+T 。在出现的弹出窗口中，从“ 提取”组中选择“ 样式 ”。或者，选择Refactor |提取物|在所选内容的上下文菜单上提取样式 。

  3. 在打开的“ 提取Android样式”对话框中，指定要创建的样式的名称，然后选择要包含为样式属性的标签属性。

  4. 要使IntelliJ IDEA通过使用对新创建的样式的引用替换标记属性来自动更新布局定义，请选中提取样式后重新启动“在可能时使用样式”重构复选框。

内联样式

此重构与“ 提取样式”重构相反，并且导致将一种样式中定义的所有属性添加到应用此样式的一个或所有组件中。当您需要将父样式与其继承者合并时，也会使用此重构。

您可以从样式定义或应用样式的组件调用内联样式重构。在第一种情况下，IntelliJ IDEA删除样式定义，并将相应的属性添加到使用样式的所有组件。从组件调用重构时，可以将其应用于具有这种样式的所有组件，也可以仅应用于当前样式。

应用内联样式重构

请执行以下任一操作：

  • 要从样式定义中调用重构：

    1. 打开样式定义文件，将光标置于要内联的样式的名称上，然后选择“ 重构” | “样式” 。从上下文菜单内联或按Ctrl+Alt+N 。

    2. 打开的“ 内联Android样式”对话框仅提供一个选项（ 内联所有引用并删除样式 ），该选项默认情况下处于选中状态。结果，IntelliJ IDEA将样式的属性转换为XML标记，将它们添加到使用该样式的所有组件的定义中，并删除样式定义。

      • 要立即应用更改，请点击重构 。

      • 要打开“ 查找工具”窗口并在应用之前预览更改 ，请单击“ 预览” 。

  • 要从组件定义调用重构：
    1. 选择具有相关样式的组件：

      • 在可视模式下 ，在“ 设计”窗格或“ 组件树”中选择具有此样式的组件 。然后选择重构|从主菜单或所选内容的上下文菜单中选择“ 内联样式” 。

      • 在手动模式下 ，在“ 结构”视图中选择具有此样式的组件，然后选择“ 重构|分解”。从主菜单或所选内容的上下文菜单中选择“ 内联样式” 。

    2. 在打开的“ 嵌入式Android样式”对话框中，配置重构：

      • 要使IntelliJ IDEA将样式的属性转换为XML标记，将它们添加到使用样式的所有组件的定义中，然后删除样式定义，选择“ 内联所有引用并删除样式” 。

      • 要仅将重构应用于当前组件，请选择“ 内联此用法”并保留样式 。

    3. 指定您要如何应用更改。

      • 要立即应用更改，请点击重构 。

      • 要打开“ 查找工具”窗口并在应用之前预览更改 ，请单击“ 预览” 。

提取版面

IntelliJ IDEA支持重复使用Android布局 ：IntelliJ IDEA将现有布局定义的一部分移至单独的布局定义文件，并通过自动插入在原始布局定义中对其进行引用标签。在IntelliJ IDEA中，此操作称为提取布局重构 。

提取布局

  1. 选择要提取为单独布局的片段。请执行以下任一操作：

    • 在手动模式下 ，选择代码片段以将光标提取或定位在片段的根元素中。

    • 在可视模式下 ，选择要在设计中提取的一个或一组组件 。窗格或在组件树中 。

  2. 从主菜单中，选择“ 重构|重构或按Ctrl+Shift+Alt+T 。在出现的列表中，从“ 提取”组中选择“ 布局 ”。或者，选择Refactor |提取物|从选择的上下文菜单中提取布局 。

  3. 在打开的“ 提取Android布局”对话框中，指定将存储提取的布局定义的文件的名称及其父文件夹的名称。您不需要预先创建文件和文件夹，IntelliJ IDEA会自动创建它们。

  4. 创建布局的配置限定符列表。此列表确定了必须提供的一组备用资源 ，以使应用程序能够在特定的目标设备配置中运行。

    • 要将限定词添加到集合中，请在“ 可用限定词”列表中将其选中，然后单击按钮，然后指定限定符的值。

    • 要从集合中删除限定词，请在“ 选择限定词”列表中将其选中 ，然后单击按钮。

    有关限定符，其含义，可接受的值和格式的详细信息，请参阅替代资源的配置限定符规范 。

内联布局重构

此重构与“ 提取布局”重构相反，并且会导致嵌入通过标签并删除标签。

内联布局重构是从布局定义文件中的标记。您可以让IntelliJ IDEA将其应用于所有参考并删除布局定义文件，或者仅将其应用于当前参考。

内联布局

  1. 打开XML定义文件，从该文件中引用有问题的布局。

  2. 导航到标记，该标记引用了相关的布局，并将光标定位在布局名称上。

  3. 请执行以下任一操作：

    • 选择重构|从主菜单内联 。

    • 选择重构|从主菜单中重构它 ，然后在打开的列表中选择“ 内联 ”，

    • 按Ctrl+Alt+N 。

    • 选择重构|从上下文菜单内联 。

  4. 在打开的“ 嵌入式Android布局”对话框中，配置重构：

    • 要使IntelliJ IDEA将布局嵌入到所有通过其引用的文件中标记并删除引用的布局定义文件，选择“ 内联所有引用并删除文件” 。

    • 要仅将重构应用于当前引用，请选择“ 内联此用法”并保留文件 。

  5. 指定要如何应用更改：

    • 要立即应用更改，请点击重构 。

    • 要打开“ 查找工具”窗口并在应用之前预览更改 ，请单击“ 预览” 。
