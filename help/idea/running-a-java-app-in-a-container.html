在Docker容器中运行Java应用程序

您可以使用Docker与特定的运行时环境一起在容器中运行Java应用程序。本教程描述了如何创建一个Dockerfile，以使用OpenJDK 8在容器中运行一个简单的Java应用程序。

该示例应用程序包含一个Main.java文件，该文件将打印Hello World!到控制台并退出。编译输出位于/ out / production / DockerJavaApp下的项目目录中。

在容器中运行Java应用程序

  1. 在项目工具窗口中，右键单击项目名称，指向新建 ，然后单击文件 。

  2. 在“ 新建文件”对话框中，键入Dockerfile ，然后单击“ 确定” 。

  3. 在新的Dockerfile中键入以下内容：

    来自openjdk：8 COPY ./out/production/DockerJavaApp/ / tmp WORKDIR / tmp ENTRYPOINT [“ java”，“ Main”]
  4. 单击在Docker装订线上运行图标（ ），然后点击“在Docker上运行” 。

    IntelliJ IDEA创建一个Docker运行配置，该配置从Dockerfile构建一个映像，然后基于该映像运行一个容器。输出目录Main.class的内容被复制到容器的/ tmp目录中。然后它运行java Main / tmp目录中的命令。结果，您应该看到Hello World!打印到容器日志。

例如，您可以与其他人共享该映像，以演示应用程序的确切运行方式，而无需安装必要的运行时（仅需要Docker）。

要共享构建的映像，您需要配置一个Docker注册表 ，您有权在其中推送映像。例如，您可以在Docker Hub上创建一个帐户，并使用公共或私有存储库共享映像，或者设置自己的Docker注册表 。

将您的Java应用程序作为Docker映像共享

  1. 在Docker工具窗口中，找到从Dockerfile构建的映像。默认情况下，它由唯一的图像ID指定，因为未提供图像标签。您可以编辑相应的运行配置并指定所需的图像标签。要找出映像ID，请从Dockerfile中选择运行的容器，然后打开“ 属性”选项卡。

  2. 右键单击具有所需ID的图像，然后在上下文菜单中单击“ 推送图像 ”。

  3. 在“ 推送图像”对话框中，选择注册表，指定图像的存储库名称和标记，然后单击“ 确定” 。

将映像推送到注册表后，有权访问该映像的任何人都可以将其拉出并从映像运行容器。
