在DRb服务器上执行测试

以下仅在安装并启用Ruby插件时有效。

基本

运行测试服有时会产生开销，因为每次执行测试服时，都应重新加载整个环境。您可以使用Spork DRb服务器避免这种情况，该服务器仅加载一次环境。

在后台运行Spork DRb服务器的情况下，您可以选择使用Spork DRb服务器或在本地执行任何测试脚本。

先决条件

  • 在启动Spork DRb服务器之前，请确保在您的应用程序中使用了spork gem和相应的测试gem（ rspec-rails ， 黄瓜 ， cumulage-rails等）。

  • 将spork gem添加到Gemfile中。

在开始使用之前，请确保插件已安装并启用 。

启动Spork DRb服务器

  1. 在主菜单中，选择“ 工具” |“工具”。运行Spork DRb服务器...。

  2. 在“ Spork DRb启动选项”对话框中，选择所需的测试框架，然后单击“运行” 。

    如果您要在项目中第一次使用spork ，请选中“ 执行引导程序”复选框以启动操作。因此，Spork将修补测试脚本。

Spork DRb服务器在“运行工具”窗口的单独选项卡中启动。

使用Spork DRb服务器运行测试脚本

  1. 确保使用IntelliJ IDEA启动了Spork DRb服务器，并且该服务器在后台运行。

  2. 运行测试脚本或其示例之一。请注意，将在相应的运行配置中自动选择选项Spork DRb 。如果要在本地运行此测试，则必须选择选项None 。

在Spork DRb下进行调试测试

  1. 确保在要调试的测试脚本中设置了断点。

  2. 在主工具栏上，单击运行/调试配置选择器，然后选择编辑配置 。

  3. 在“ 运行/调试配置”对话框中 ，为Spork DRb服务器创建运行/调试配置。有关该过程的详细说明，请参考“ 创建和编辑运行/调试配置 ”部分。

  4. 在调试模式下启动Spork DRb。为此，请选择Spork DRb运行/调试配置，单击 ，或按Shift+F9 。

  5. 在Spork DRb服务器以调试模式运行的情况下，运行所需的测试脚本。

请注意，如果您调试测试脚本，它将在没有Spork DRb服务器的情况下执行。

管理Spork DRb服务器

请参阅“ 运行工具”窗口的描述。特别是，请使用以下按钮：

  • -在不关闭“运行工具”窗口中其选项卡的情况下，停止Spork DRb服务器。

  • -关闭Spork DRb服务器选项卡。

  • -在同一标签中重新运行Spork DRb服务器。

技巧和窍门

  • IntelliJ IDEA为Spork DRb服务器创建一个临时运行/调试配置。稍后，您可以根据需要更改此运行/调试配置，将其另存为永久文件，然后使用其启动服务器。

  • 如果您使用的是TeamCity ，请通过将以下语句添加到Spork.preload ：

    需要'teamcity / spec / runner / formatter / teamcity / formatter'

  • Spork DRb服务器不适用于Ruby 2.0。

  • 如果Zeus和Spork DRb服务器同时运行，则Zeus将获得优先权。
