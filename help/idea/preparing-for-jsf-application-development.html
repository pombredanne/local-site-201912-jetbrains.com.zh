为JSF应用程序开发做准备

仅Ultimate Edition支持此功能。

为了准备JSF应用程序开发，您应该：

  • 确保已启用Java EE：Java Server Faces 插件 。该插件与IDE捆绑在一起，并且默认情况下启用。但是，您可能由于某些原因已将其禁用。

  • 创建启用了JSF支持的项目或模块，或者在现有模块中启用JSF支持。

在此页：

  • 确保已启用Java Server Faces插件
  • 创建项目或模块时启用JSF支持
  • 为现有模块启用JSF支持

确保已启用Java Server Faces插件

即使默认情况下启用了CDI支持插件，也总是值得在开始使用CDI之前确保仍启用该插件。

  1. 打开设置/首选项对话框Ctrl+Alt+S 。

  2. 在对话框的左侧，选择插件 。

  3. 在对话框的右侧，在“ 插件”页面上 ，键入faces在搜索框中。结果，只有名称和说明包含的插件faces显示在插件列表中。

  4. 如果未选中“ Java EE：Java Server Faces ”右边的复选框，请选择它。

  5. 在“ 设置”对话框中，单击“ 确定 ” 。

  6. 如果建议，请重新启动IntelliJ IDEA。

创建项目或模块时启用JSF支持

  1. 请执行以下任一操作：

    • 如果要创建新项目，请：在“ 欢迎”屏幕上单击“ 创建新项目 ”，或选择“ 文件” |“ 新建” 。 新品|项目 。

      结果，“ 新建项目”向导打开。

    • 如果要向现有项目中添加模块，请执行以下操作：打开要向其中添加模块的项目，然后选择“ 文件” |“添加” 。新品|模块 。

      结果，“ 新建模块”向导将打开。

  2. 在向导的第一页的左侧窗格中，选择Java Enterprise 。在页面的右侧，指定要使用的JDK并选择要支持的Java EE版本。

  3. 如果在此步骤中准备好指定要将应用程序部署到的应用程序服务器（例如，测试应用程序行为），请执行此操作。这将为您的模块自动生成相应的特定于服务器的运行/调试配置 。否则，为了能够运行您的应用程序，您将必须自己创建运行/调试配置。

    从列表中选择服务器，或单击“ 新建”，然后选择所需的服务器。然后，指定服务器设置：

    • 对于本地安装的服务器，请指定服务器安装目录的路径。（点击在相应对话框中选择目录。）

    • 对于托管服务器（Cloud Foundry或CloudBees），请指定您的用户帐户详细信息。

  4. 在“ 其他库和框架”下 ，选中“ Web应用程序”复选框。

    从“ 版本”列表中选择要支持的Servlet规范的版本 。

    如果要创建部署描述符web.xml文件，请选中“ 创建web.xml”复选框。

  5. 选择“ JSF”复选框。

    如果要创建配置文件faces-config.xml ，请选中“ 创建faces-config.xml”复选框。

    选择所需的库选项，并在必要时指定关联的设置。您可以选择：

    • 下载并使用JSF实现库（Mojarra）。

      为此，请在“ 库”下选择“ 下载” 。

      现在，要查看或修改关联的选项，请单击“ 配置” ，然后在打开的“ 下载选项”对话框中：

      • 选择库版本。

      • 指定库名称。

      • 选择库级别 （全局，项目或模块）。

      • 在“要下载的文件”下 ，选择要下载的文件。

      • 在“ 将下载的文件复制到”下 ，指定目标文件夹的路径。如果要更改默认路径，请单击并在打开的对话框中指定文件夹位置。

    • 使用JSF库IntelliJ IDEA已经知道。

      为此，请单击“ 使用库” ，然后从列表中选择所需的库。

      如有必要，请配置库设置（例如，更改其名称）。这是在“ 编辑库”对话框中完成的，您可以通过单击“ 配置”打开该对话框。

    • 使用计算机上可用的适当JAR文件创建新库。

      为此，单击使用库 ，然后单击创建 。在打开的对话框中选择所需的JAR文件。（对于多项选择，请保留Ctrl键）。

      如有必要，请配置新库（例如，更改其名称或级别 ）。为此，请单击“ 配置” ，然后在“ 创建库”对话框中指定所需的设置。

    • 将库的设置推迟到以后。在这种情况下，请选择稍后设置库 。

  6. 如果要使用一个或多个JSF组件库（例如PrimeFaces，RichFaces等），请选中相应的一个或多个复选框，并指定关联的选项。该过程类似于JSF实现库的过程。

    单击下一步 。

  7. 指定名称和位置设置。

    点击完成 。

结果，您的新模块将包含：

  • Web和WEB-INF目录。

  • Web目录中的文件index.xhtml 。稍作修改，您就可以将该文件用作应用程序的起始页。

  • 在WEB-INF目录中，如果已指定：

    • web.xml ，Web应用程序部署描述符。

    • faces-config.xml ，JSF配置文件。

  • 如果指定，则模块依赖性中包含的一个或多个JSF库。

  • 您模块的工件规范。

  • 如果指定了服务器，则为在该服务器的上下文中运行应用程序的运行/调试配置 。

为现有模块启用JSF支持

  1. 打开项目工具窗口 （例如， 视图|工具窗口|项目 ）。

  2. 右键单击该模块，然后选择添加框架支持 。

  3. 在“ 添加框架支持”对话框的左侧窗格中，选中“ Web应用程序”复选框。

    在对话框的右侧，从“ 版本”列表中选择要支持的Servlet规范的版本 。

    如果要创建部署描述符web.xml文件，请选中“ 创建web.xml”复选框。

  4. 选择“ JSF”复选框。

    如果要创建配置文件faces-config.xml ，请选中“ 创建faces-config.xml”复选框。

    选择所需的库选项，并在必要时指定关联的设置。您可以选择：

    • 下载并使用JSF实现库（Mojarra）。

      为此，请在“ 库”下选择“ 下载” 。

      现在，要查看或修改关联的选项，请单击“ 配置” ，然后在打开的“ 下载选项”对话框中：

      • 选择库版本。

      • 指定库名称。

      • 选择库级别 （全局，项目或模块）。

      • 在“要下载的文件”下 ，选择要下载的文件。

      • 在“ 将下载的文件复制到”下 ，指定目标文件夹的路径。如果要更改默认路径，请单击并在打开的对话框中指定文件夹位置。

    • 使用JSF库IntelliJ IDEA已经知道。

      为此，请单击“ 使用库” ，然后从列表中选择所需的库。

      如有必要，请配置库设置（例如，更改其名称）。这是在“ 编辑库”对话框中完成的，您可以通过单击“ 配置”打开该对话框。

    • 使用计算机上可用的适当JAR文件创建新库。

      为此，单击使用库 ，然后单击创建 。在打开的对话框中选择所需的JAR文件。（对于多项选择，请保留Ctrl键）。

      如有必要，请配置新库（例如，更改其名称或级别 ）。为此，请单击“ 配置” ，然后在“ 创建库”对话框中指定所需的设置。

    • 将库的设置推迟到以后。在这种情况下，请选择稍后设置库 。

  5. 如果要使用一个或多个JSF组件库（例如PrimeFaces，RichFaces等），请选中相应的一个或多个复选框，并指定关联的选项。该过程类似于JSF实现库的过程。

  6. 在“ 添加框架支持”对话框中单击“ 确定 ”。
