配置Pipenv环境

Pipenv是一种工具，提供了为Python项目创建虚拟环境所需的所有必要手段。在您安装或卸载软件包时，它会通过Pipfile文件自动管理项目软件包。

Pipenv还会生成Pipfile.lock文件，该文件用于生成确定性的构建并创建工作环境的快照。当项目要求和软件包版本很重要时，这对于安全敏感型部署可能特别有用。有关pipenv更多详细信息，请参阅项目文档docs.pipenv.org 。

要将Pipenv与PyCharm结合使用，您需要执行几个准备步骤。

安装Pipenv

  1. 运行以下命令以确保系统中已安装pip：

    $ pip --version

    您应该期望收到指示pip版本的系统响应。如果未发现任何点，请按照《 安装说明》中的说明进行安装 。另外，您可以从http://python.org下载并安装Python。

  2. 安装pipenv通过运行以下命令：

    $ pip install --user pipenv

    安装完成后，您将看到以下消息：

  3. 为了方便起见，您可以将用户库的二进制目录添加到您的PATH环境变量。如果您跳过此过程，PyCharm将在添加pipenv环境时提示您指定pipenv可执行文件的路径。

    1. 运行以下命令：

      $ py -m site --user-site

      示例输出可以是：

      C：\ Users \ jetbrains \ AppData \ Roaming \ Python \ Python37 \ site-packages

    2. 更换site-packages与Scripts在此路径中接收要添加到的字符串PATH变量，例如：

      $ setx PATH "%PATH%;C:\Users\jetbrains\AppData\Roaming\Python\Python37\Scripts"

    1. 运行以下命令以查找用户库的二进制目录：

      $ python -m site --user-base

      输出示例可以是

      /Users/jetbrains/.local（masOS ）或/home/jetbrains/.local（Linux ）

    2. 将bin添加到此路径以接收用于添加到〜/ .bashrc文件的字符串，例如：

      $ export PATH="$PATH:/Users/jetbrains/.local/bin"

    3. 运行以下命令以使更改生效：

      $ source ~/.bashrc

    4. 确保已在bash_profile中启用了bashrc 。

您可以随时在项目设置中更改pipenv可执行文件的指定路径。 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），导航至工具| Python Integrated Tools ，然后在“ 可执行文件路径”字段中输入目标路径。

准备步骤完成后，您可以使用pipenv为新项目或现有项目创建虚拟环境。

为新的Python项目设置pipenv

  1. 按照创建纯Python项目中所述开始创建一个新的Python项目 。

  2. 在“ 新建项目”对话框中，单击要展开“ 项目解释器”节点，请选择“使用新建环境” ，然后从可用虚拟环境列表中选择Pipenv 。

  3. 如果您已将用户库的二进制目录添加到您的PATH环境变量，您无需设置任何其他选项：pipenv可执行文件的路径将被自动检测。

  4. 单击确定以完成任务。

  5. 如果您尚未修改PATH变量，PyCharm显示一条错误消息： 未找到Pipenv可执行文件 。按照pipenv安装过程中的描述找到正确的可执行路径，然后在Pipenv可执行字段中输入目标字符串，例如： C：\ Users \ jetbrains \ AppData \ Roaming \ Python \ Python37 \ Scripts \ pipenv.exe （Windows）或/Users/jetbrains/.local/bin/pipenv（macOS ）。

    单击确定以保存更改并完成任务。

您还可以为任何先前创建的项目设置pipenv。

为现有的Python项目配置pipenv

  1. 通过以下两种方式打开“ 添加Python解释器”对话框：

    • 当您在编辑器中时 ，最方便的方法是在状态栏中使用Python Interpreter 小部件 。单击小部件，然后选择添加解释器...

    • 通过按打开设置/首选项对话框 Ctrl+Alt+S或选择文件|设置 Windows和Linux或PyCharm | macOS的首选项 。

      在“ 设置/首选项”对话框中Ctrl+Alt+S ，选择项目|项目口译员 。点击图标，然后选择添加 。

  2. 在此对话框的左侧窗格中，单击Pipenv Environment 。

  3. 如果您已将用户库的二进制目录添加到您的PATH环境变量，您无需设置任何其他选项：pipenv可执行文件的路径将被自动检测。

    如果您看到“ 找不到Pipenv可执行文件”错误消息，请按照pipenv安装过程查找可执行文件路径，并将其添加到Pipenv可执行文件字段。

  4. 单击确定以完成任务。

完成所有步骤后，将为您的项目设置新的pipenv环境，并安装Pipfile中列出的软件包。

如果您打开的项目中添加了Pipfile文件，但未配置任何解释器，则PyCharm会为您提供使用Pipenv环境的功能。

如果选择此选项，PyCharm会自动为您设置pipenv。或者，您可以单击“ 配置Python解释器”以遵循标准工作流程。

同样，当您第一次在PyCharm中打开带有Pipfile文件的项目时，例如，通过从版本控制中检出该项目，将会自动配置Pipenv虚拟环境。

将Pipenv虚拟环境设置为项目解释器后，将从Pipfile中定义的源中添加所有可用软件包。这些软件包是通过pipenv而非pip在软件包列表中安装，删除和更新的。

对于任何已配置的Python解释器，您可以：

  • 安装和卸载解释器路径
  • 安装和卸载解释程序包
