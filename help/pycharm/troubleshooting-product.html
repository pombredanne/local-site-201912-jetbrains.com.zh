项目设定

在“首选项/设置” |“设置” |“可用”中，项目解释器被标记为不支持。 项目|项目口译员 。

标记解释器的Python版本已过时，PyCharm不支持

您可以从可用解释器列表中删除不支持的解释器。有关受支持的Python版本的更多信息，请参见Python支持 。

代码编辑

PyCharm编辑器限制代码编辑。

您已经安装了IdeaVim插件，从而启用了vim编辑模式。

在工具菜单上取消选择Vim Emulator 。有关更多详细信息，请参见在PyCharm中使用Vim编辑器仿真 。

Python代码未在编辑器中突出显示。

PY文件与文本文件格式相关联。

代码补全

代码完成操作不可用。

省电模式已启用。

从主菜单中，选择“ 文件”，然后清除“ 省电模式”复选框。

您的文件不驻留在内容根目录中，因此它没有获得代码完成所需的类定义和资源。

重组您的源文件。

包含要在完成建议列表中显示的类和函数的文件是纯文本文件。

重新考虑并修改（如果需要）源文件的格式。

调试

调试Gevent代码时，Python Debugger挂起。

Gevent兼容模式未启用。

在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），导航至“ 构建，执行，部署| Python调试器，然后选择Gevent兼容复选框。

您的PyQt代码中报告了一些导入错误。

PyQt已安装在解释器上，但未导入应用程序代码中。

在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），导航至“ 构建，执行，部署| Python Debugger ，然后清除中的PyQt兼容复选框。默认情况下启用此模式。

调试过程很慢。

调试器不仅会在进程异常终止时停止，还会在抛出的每个异常上终止（即使已捕获并没有导致进程终止）。

清除“ 断点”对话框中的“ 抬高”复选框（ “运行” |“查看断点” ）。

在您首次尝试附加到本地进程时，会出现以下错误消息： ptrace: Operation not permitted 。

这是Ubuntu的已知问题 。

确保kernel.yama.ptrace_scope内核参数设置为0。您可以使用以下命令临时更改其值sudo sysctl kernel.yama.ptrace_scope=0或通过添加永久设置kernel.yama.ptrace_scope = 0到/etc/sysctl.d/10-ptrace.conf文件。

Cython加速

显示以下错误消息：

Python.h: no such file or directory 。

您缺少Python开发人员的标头文件和静态库。

使用包管理器为系统范围内的Python开发人员安装头文件和静态库。

例如，您可以使用以下命令：

  • 对于Python 2.x：

    须藤apt-get install python-dev

  • 对于Python 3.5：

    须藤apt-get install python3-dev

  • 对于Python 3.6：

    须藤apt-get install python3.6-dev

显示以下错误消息：

Command 'gcc' failed with exit status 1 。

您缺少C编译器。

安装C编译器以便为调试器生成Cython扩展。

您已经单击了“ 安装”链接
在Cython加速扩展通知中
但是PyCharm反复提示安装。

您没有写PyCharm使用的目录的权限。

检查并修改您的权限。

打包

软件包安装失败。

pip不适用于特定的Python解释器，或者不满足任何安装要求。

  • 如果pip不可用，请为所需的Python解释器安装pip （请参阅相应的故障排除技巧 ）

  • 请查阅产品文档以获取特定的软件包，以确保可以将其安装在操作系统中以及目标Python解释器中。另外，检查是否应满足任何其他系统要求。

文件观察者
您可能会在“ 首选项/设置” |“设置”中注意到以下消息。 工具| “文件监视程序”窗口。

Unknown Scope error

File Watcher使用此项目中未定义的范围。

双击监视程序并选择一个可用的作用域或创建一个新的作用域。

Not found error

该项目使用已删除的全局File Watcher。

使用“ 删除”按钮从列表中删除观察者或对其进行编辑以创建一个具有相同名称的新的全局监视程序。

部署方式

使用SSH配置时的错误消息：

Connection to failed: SSH: invalid privatekey: [. SSH: invalid privatekey:

PyCharm不支持OpenSSH密钥的RFC 4716格式。

  • 将私钥添加到ssh-agent使用ssh-add命令（在ssh.com上查看更多详细信息）

  • 在“ 添加Python解释器”对话框中重复配置步骤。

  • 将私钥添加到ssh-agent使用ssh-add命令（在ssh.com上查看更多详细信息）

  • 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择生成，执行，部署|部署 ，然后选择SFTP部署配置。

    在“ 身份验证”列表中选择“ OpenSSH Config和身份验证代理 ”。

为特定的Python解释器安装的软件包版本已过时。

运行以下命令以安装所需的软件包：

\python.exe -mpip install ''

例如，
C:\Python27\python.exe -mpip install 'Flask'

一旦执行，这些命令将安装指定软件包的最新版本。

要安装特定版本的软件包，请使用以下表达式：

  • '==' –安装软件包的特定版本

  • '>=' –安装的软件包大于指定版本或更高版本

  • '~=' –安装与软件包的指定版本兼容的版本

例如，以下命令将安装与1.0.2版兼容的Flask：
C:\Python27\python.exe -mpip install 'Flask~=1.0.2'
