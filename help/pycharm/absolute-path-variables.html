路径变量

路径变量是占位符，代表占链接到项目但存储在项目外部的资源的绝对路径。如果您在团队中工作，则队友计算机上的这些绝对路径可能会有所不同。使用路径变量，您可以灵活地共享代码，以便当路径变量根据每台特定计算机上的配置接受值时，可以正确解析对链接资源的所有引用。

在PyCharm中，有一些预定义的变量：

  • $USER_HOME$ —代表您的主目录。

  • $PROJECT_DIR$ —代表存储项目的目录。

创建一个新的路径变量

例如，您有一个Python脚本，该脚本处理处理存储在系统中的reports.csv文件中的某些数据。您创建了运行/调试配置来运行此脚本，并希望通过VCS与您的队友共享此配置。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到外观和行为|路径变量 ，然后单击按钮。

  2. 输入新变量的名称（例如， DATA_PATH ）及其指向磁盘上数据文件的目标目录的值。

  3. 通过您的版本控制系统共享运行/调试配置。

  4. 检查.idea / runConfiguration / .xml文件：

    在您的队友从VCS更新项目后，他们将更改DATA_PATH变量值，使其指向其计算机上的数据目录。

忽略路径变量

当您打开一个项目时，PyCharm会检查是否存在任何未解析的路径变量。如果IDE检测到任何东西，它将要求您为其定义值。如果由于某种原因您不想这样做（例如，如果您不打算将文件或目录与未解析的路径变量一起使用），则可以将它们添加到忽略的变量列表中。

例如，如果在运行/调试配置中传递给JVM的程序参数的格式与内部变量的格式相同，则也可以使用忽略的变量列表。 ($SOME_STRING$)路径变量。在这种情况下，可以将此参数添加到忽略的变量列表中，以避免混淆。输入SOME_STRING 路径变量对话框中的忽略变量字段。
