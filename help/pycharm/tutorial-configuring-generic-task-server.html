配置通用任务服务器

PyCharm支持开箱即用地与许多任务跟踪器集成。但是，如果您使用PyCharm还不支持的跟踪器，您仍然可以将其集成在一起，以配置所谓的通用服务器。

本教程描述了如何：

  • 作为通用服务器连接到JIRA Cloud

  • 获取分配给您的问题列表

  • 对于每个问题，获取其问题的ID，标题，描述，创建和更新的日期和时间

在开始配置与跟踪器的连接之前，请注意PyCharm：

  • 仅支持REST API服务。

  • 支持基本HTTP身份验证或向服务器发送初步请求。

  • 支持GET和POST请求。

  • 服务器响应中不支持分页。

指定服务器URL和凭据

  1. 在“ 设置/首选项”对话框中Ctrl+Alt+S ，转到工具|任务服务器 。

  2. 请点击然后选择通用 。

  3. 在“ 常规”选项卡上，指定任务跟踪器的URL，连接凭据，然后选中“ 使用HTTP身份验证”复选框。的{serverUrl}是一个变量，代表您在“ 常规”选项卡上指定的URL。

    您可以在密码字段中输入API令牌值，而不是密码。

配置服务器设置

  1. 切换到“ 服务器配置”选项卡。请注意，因为您正在使用HTTP身份验证，所以登录URL字段将被禁用。

  2. 在“ 任务列表URL”中 ，输入用于从服务器获取问题的URL。您可以使用变量或输入完整的URL：

    {serverUrl} / rest / api / 2 / search或https://serverurl.atlassian.net/rest/api/2/search

    您可以在登录URL ， 任务列表URL和单个任务URL字段中使用代码完成功能。

  3. 添加?jql={JQL_Query}任务列表URL的表达式： {serverUrl} / rest / api / 2 / search？jql = {JQL_Query} 。

  4. 点击窗口底部的管理模板变量以配置JQL_Query变量。

  5. 请点击在右上角。

  6. 在新字段中，指定变量名称JQL_Query ，并增加其价值assignee = currentUser() and resolution = Unresolved 。
    这将使您获得分配给您的未解决的问题。

  7. 单击确定 。

配置响应类型并指定选择器

  1. 在“ 服务器配置”对话框中，选择JSON响应类型。

  2. 在表中指定选择器以获取问题的ID和标题，并获得其描述。您还可以了解创建和更新问题的日期和时间：

    • 任务： $.issues

    • ID：key

    • 摘要：fields.summary

    • 描述：fields.description

    • 更新：fields.updated

    • 创建：fields.created

  3. 单击测试以确保正确配置了所有参数。

从服务器上传问题

  1. 单击任务组合，然后选择打开任务 。PyCharm将从服务器加载与您的配置匹配的所有问题。

  2. 从列表中选择必要的问题。

  3. 按Ctrl+Q打开问题描述并确保获得所有必需的详细信息。
