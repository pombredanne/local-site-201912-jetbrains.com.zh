使用React Native，您可以使用JavaScript和React开发适用于iOS和Android的本地移动应用程序。它是由Facebook创建的，用于Instagram，Airbnb等著名应用，现在也用于JetBrains自己的YouTrack移动应用。从React Native官方网站上了解更多信息。

PyCharm可帮助您创建，编辑，皮棉，运行，调试和维护您的React Native应用程序。PyCharm还为React和Flow符号提供代码完成。

在开始之前，请确保您的计算机上装有Node.js。

创建一个新的React Native应用程序

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 新建项目”对话框。

  2. 在左侧窗格中，选择React Native 。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 在“ 节点解释器”字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择添加以配置新的解释器。

    3. 从React Native列表中选择npx --package react-native-cli react-native 。

      另外，对于npm 5.1及更早版本，请安装react-native-cli通过运行打包自己npm install -g react-native-cli在航站楼 Alt+F12 。创建应用程序时，选择react-native-cli包已存储。

  4. 当您单击Create时 ，PyCharm会使用所有必需的配置文件生成特定于React Native的项目，下载依赖项，并使用默认设置创建React Native类型的运行/调试配置。

从现有的React Native应用程序开始

要继续开发现有的React Native应用程序，请在PyCharm中将其打开并下载所需的依赖项。

打开计算机上已经存在的应用程序源

  • 单击“ 欢迎”屏幕上的“ 打开 ”，或选择“ 文件” |“共享”。从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中签出应用程序源

  1. 单击“ 欢迎”屏幕上的“ 从版本控制中签出”，或选择“ VCS |从主菜单中的“ 版本控制”中签出 。

  2. 从列表中选择您的版本控制系统。

  3. 在打开的特定于VCS的对话框中，键入您的凭据和存储库以从中检出应用程序源。

  • 在弹出窗口中单击运行“ npm install” ：

编码协助

PyCharm使用JavaScript代码为React API和JSX提供代码完成。代码完成适用于React方法，React特定的属性，HTML标记和组件名称， React事件 ，组件属性等。有关更多信息，请参见React：完成代码 。

运行和调试React Native应用程序

您可以在物理设备或模拟器上运行和调试应用程序。在启动应用程序之前，请确保模拟器已在运行，或者，如果您使用的是物理设备，则该模拟器已连接到计算机。

PyCharm使运行和调试React Native应用程序变得非常灵活。例如，如果您是第一次启动应用程序，则可以选择运行React Native捆绑器，构建应用程序并在模拟器上打开它-所有这些都是运行或调试会话的一部分。如果捆绑程序已经在运行，则也可以跳过启动它；如果自上次运行以来未对其本机代码进行任何更改，则可以拒绝构建该应用程序。

创建一个React Native运行/调试配置

  1. 在主菜单上，选择“运行”。编辑配置 ，单击然后从列表中选择React Native 。

  2. 选择是否要让PyCharm为您构建并启动应用程序：

    • 如果是第一次启动应用程序，或者自上次运行以来已更新其本机代码，请选中“ 构建并启动”复选框。

    • 如果自上次构建以来未对应用程序的本机代码进行任何更改，请清除此复选框。当您开始调试时，PyCharm等待您在模拟器中打开您的应用程序，并按照React Native官方网站上的说明启用了远程调试。

    • 如果您的应用程序使用Expo ，请清除该复选框，因为此捆绑程序将自行处理。

  3. 如果选择了“ 构建并启动”复选框，请选择目标平台，Android或iOS。根据您的选择，PyCharm将使用react-native run-ios或搭配react-native run-android 。

    （可选） 输入要传递给React Native的参数，例如，通过‑‑simulator旗： ‑‑simulator="iPhone 4s" 。

  4. 在Bundler主机字段中，指定运行React Native捆绑程序的主机，默认值为localhost 。

    如果使用的是Expo，则可能需要更改默认的捆绑程序主机，以确保与调试器的成功连接。此连接可能会失败，因为默认情况下，PyCharm使用localhost启动调试，而Expo希望使用127.0.0.1或外部IP地址，具体取决于在Metro捆绑器配置的“连接”字段中选择的内容。

  5. 在“ Bundler端口”字段中，指定运行React Native捆绑程序的端口，默认情况下选择8081 ，从React Native官方网站了解更多信息。

  6. 选择要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

    指定react-native-cli的路径和应用程序的工作目录 。（可选）键入以下环境变量react-native run-android要么react-native run-ios 。

  7. 默认情况下，当您调用运行/调试配置时，PyCharm会自动启动React Native捆绑程序。如果您已经从PyCharm外部（例如，从命令行）启动了捆绑程序，则可以重新使用它而无需停止并重新启动。在启动前区域中选择捆绑器，然后单击 。

准备Android设备

每次开始使用应用程序时，您都需要准备一个Android设备。

  • 启动Android虚拟设备或通过USB启用调试，然后通过USB 连接到物理Android设备 。从React Native官方网站上了解更多信息。

准备iOS模拟器

您只需要安装一次iOS模拟器，PyCharm会自动通过以下方式启动它react-native run-ios 。

  • 在嵌入式终端 （ Alt+F12 ），输入：

    \npm install --global ios-sim
