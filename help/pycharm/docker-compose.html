以下仅在安装并启用Docker Integration插件时有效。

先决条件

确保满足以下先决条件：

  • 如Docker Compose页面所述，已安装Docker Compose 。

  • 您的网络连接稳定。

    要使用Docker进行操作，您需要在计算机上使用busybox映像。确保您具有稳定的Internet连接，以便PyCharm可以下载并运行busybox:latest 。成功配置Docker后，您可以脱机。

  • 在开始使用Docker Compose之前，请确保已启用Docker Integration插件。该插件与PyCharm捆绑在一起，并且默认情况下处于激活状态。如果未激活插件，请在“ 插件”页面上启用它 设置/首选项对话框（ Ctrl+Alt+S ），如“ 管理插件”中所述 。

    如果您正在使用Windows的Docker，请在Docker设置的`` 常规''部分中启用tcp：// localhost：2375上没有TLS选项的Expose守护程序 。

请注意，PyCharm支持高达3.x的docker-compose配置版本。

重要的提示

支持在Docker Compose配置文件中声明的选定服务作为Python解释器。基于此解释器的Python运行配置的运行方式类似于docker-compose up命令，并将项目源映射到选定的服务容器中。

该功能适用于Linux和本地安装的Docker和macOS，在Windows平台上运行的PyCharm中不会显示。在macOS上，Docker Compose配置文件必须位于默认共享文件夹/ Users /中 。

使用Docker Compose

在“ 设置/首选项”对话框中（ Ctrl+Alt+S ）框，展开节点Build，Execution，Deployment ，然后在Docker页面中 ，添加所需的Docker，然后指定Docker Compose可执行文件 。

结果，“ Docker Compose”选项出现在“ Configure Remote Python Interpreter”对话框中。

PyCharm可让您指定多个docker-compose配置文件，并将它们与docker-compose做。
