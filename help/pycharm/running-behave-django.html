启用对Behavior-django的支持

仅专业版支持此功能。该版本是商业版本，它提供了一组出色的功能：有关更多详细信息，请参见比较矩阵 。

使用PyCharm，您可以在Django应用程序中启用行为-Django集成，从而受益于Django的行为驱动开发（BDD）。

要启用行为Django集成：

  1. 在您的Django项目中 ，安装以下Python软件包 ：
    • behave
    • behave-django

  2. 修改项目的结构以满足行为的结构要求。在Django应用程序目录中，为测试场景文件创建features目录，并为场景的Python实现创建steps子目录。创建一个新的Gherkin功能文件features / Auth.feature和一个新的Python步骤文件features / steps / Auth.py。

  3. 现在，使用Gherkin功能测试语言将测试场景添加到Auth.feature中 ：

    功能：身份验证方案：未经身份验证的用户无法访问该页面假设我访问页面时未通过身份验证，则状态代码为302情形：已身份验证的用户可以访问该页面鉴于我获得身份验证时，我访问该页面时，状态代码为200

  4. 在features / steps / Auth.py Python文件中实现Auth.feature场景中使用的步骤 ：

    从行为导入*从django.contrib.auth.models导入用户use_step_matcher（“ re”）@given（“我未通过身份验证”）def step_impl（上下文）：通过@when（“我访问页面”）def step_impl（ context）：context.response = context.test.client.get（“ /”）@then（“状态代码为（？P \ d +）“）def step_impl（上下文，状态）：代码= context.response.status_code断言代码== int（状态），” {0}！= {1}“。格式（代码，状态）@given（“我已通过身份验证”））def step_impl（上下文）：user = User.objects.create_superuser（” jane“，” jane@example.org“，” 123“ ）context.test.client.force_login（user）

  5. 修改views.py文件和urls.py文件，以添加测试样本所需的代码：

    views.py

    从django.contrib.auth.decorators导入django.http所需的login_required。import HttpResponse＃在此处创建视图。@login_required def show_user（request）：返回HttpResponse（“ OK”）

    urls.py

    从django.urls从Django_Behave导入路径导入视图urlpatterns = [path（''，views.show_user），]

  6. 现在，执行关键的调整：添加'behave_django'到INSTALLED_APPS settings.py部分。

    完成此操作后，您的应用程序就可以进行测试了。您所需要做的就是为您的方案运行测试配置。

  7. 在编辑器窗口中打开Auth.feature文件。将光标放在与Scenario语句，然后右键单击它，然后选择“运行” 。预览测试运行结果。

    从测试报告中，您可以了解到该测试是由manage.py运行的。您还可以查看方案中已通过，失败或跳过了多少步骤。
