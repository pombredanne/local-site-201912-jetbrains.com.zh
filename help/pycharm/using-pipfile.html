使用pipfile管理依赖项

Pipfile是Pipenv虚拟环境用于管理项目依赖项的专用文件。该文件对于使用Pipenv是必不可少的。当为新项目或现有项目创建Pipenv环境时，会自动生成Pipfile 。同样，当您第一次在PyCharm中打开带有Pipfile文件的项目时，Pipenv虚拟环境将自动配置。

考虑一项从头开始创建依赖项列表的任务。

在Pipfile中记录依赖项以管理项目包

  1. 当PyCharm为新的pipenv虚拟环境创建Pipfile时 ，该文件如下所示：

    的python_version参数是您在创建新的pipenv环境时指定的基本解释器的版本。的packages部分是您可以列出项目所需软件包的地方。

  2. 请注意，PyCharm建议您安装特定于Pipfile的插件。单击相应的链接以安装Tom的“最小语言”（ TOML ）的插件。您必须重新启动PyCharm才能启用插件。重新启动后，您将看到确认确认Pipfile格式的消息：

  3. 通过修改packages部分。

    [packages] django =“ *”

  4. 每当您修改Pipfile文件时，PyCharm都会建议执行以下操作之一：

    • pipenv lock —将新要求记录到Pipfile.lock文件中。

    • pipenv update —将新要求记录到Pipfile.lock文件中，并将缺少的依赖项安装在项目解释器上。

    这一步非常重要，因为PyCharm根据Pipfile.lock文件中记录的信息来管理项目依赖项 。因此，任何添加到Pipfile但未锁定的要求都将被忽略。

    让我们选择pipenv update命令以安装Django软件包。

  5. 确保Django在已安装软件包的列表中。 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），展开专案节点，单击Project Interpreter ，然后检查软件包列表。

  6. 现在探索相反的工作流程。在项目解释器对话框中，单击添加Flask包。请注意，pipenv使用标准过程限制配置索引。所有可用的软件包都是从Pipfile中指定的源中添加的 。这些软件包是通过pipenv而不是pip进行安装，删除和更新的。

  7. 一旦添加了Flask软件包，请关闭“ 可用软件包”对话框并查看Pipfile 。的[packages]现在，该部分如下所示：

    [包装] django =“ *”烧瓶=“ *”

PyCharm跟踪是否不满足Pipfile中列出的任何要求，并建议您应用受影响的依赖项。
