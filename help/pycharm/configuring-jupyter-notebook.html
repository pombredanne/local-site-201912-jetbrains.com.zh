管理Jupyter笔记本服务器

仅专业版支持此功能。该版本是商业版本，它提供了一组出色的功能：有关更多详细信息，请参见比较矩阵 。

在PyCharm中，您可以使用以下命令执行代码单元：

  • 托管服务器 – Jupyter服务器，由PyCharm自动为当前项目启动。当您关闭PyCharm时，它将终止。

  • 配置的服务器 –您通过指定其URL和令牌连接到的任何Jupyter服务器。

启动Jupyter服务器

要运行Jupyter服务器，只需执行任何代码单元即可 。当您启动单元执行时，PyCharm使用任何可用端口（默认为8888端口）在本地主机上启动Jupyter服务器。您可以切换到Jupyter服务器工具窗口以预览服务器的配置详细信息。

启动服务器后， 它将在Jupyter工具栏的Jupyter服务器小部件中显示为受管服务器。您还可以在内核列表中看到自动创建的服务器内核。该内核基于PyCharm项目解释器 。

停止Jupyter服务器

要停止任何正在运行的服务器，请切换到Jupyter服务器工具窗口，然后单击图标，或在Jupyter快速列表中选择相应的命令。在“ 服务器日志”窗口中预览状态。

关闭服务器后，当前会话将终止。下次使用时启动服务器图标或Jupyter快速列表 ，以前所有会话和所有笔记本的执行结果都将丢失。

重新启动内核

您可能希望刷新计算而不关闭整个服务器，也不影响其他笔记本电脑。要重新启动当前运行的内核，请单击在Jupyter笔记本工具栏上 ，从菜单中选择“ 重新启动内核”命令，然后在“ 服务器日志”窗口中预览状态

您可以在特定的Jupyter服务器上执行笔记本代码单元。

配置Jupyter服务器

启动任何受管服务器时，默认情况下，它使用当前的项目解释器和自动选择的端口。但是，您可以选择PyCharm实例中可用的任何其他解释器并指定备用端口。如果知道服务器的URL和令牌，也可以连接到任何已配置的服务器。

  1. 要打开服务器设置，请单击“ Jupyter服务器”窗口小部件，然后选择“ 配置Jupyter服务器” 。

  2. 配置服务器选项：

    • 要自定义默认的Jupyter服务器，请在“ Jupyter服务器”对话框中，选择“ 受管服务器”，然后从Python解释器列表中选择任何本地Python解释器。

    • 要更改自动检测的端口，请在“ 端口”字段中键入其编号。

    • 要连接到任何正在运行的Jupyter服务器，请在“ Jupyter服务器”对话框中，选择“ 配置的服务器”并指定服务器的路径，包括URL和令牌。

  3. 完成配置后，点击Apply和OK 。然后，要显式切换到配置的服务器，请单击Jupyter Server小部件，然后选择Switch to current Jupyter Server 。

  4. 执行任何代码单元以运行受管服务器或连接到正在运行的已配置服务器。

  5. 如果看到以下警告，则新配置的服务器的内核与您当前的项目解释器不匹配。例如，它可能具有一组不同的已安装软件包。

    您可以通过单击相应的链接将项目解释器注册为已配置服务器上的内核。之后，您可以在已配置服务器的内核列表中看到它。
