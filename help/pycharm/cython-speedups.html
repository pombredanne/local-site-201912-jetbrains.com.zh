安装Python扩展

PyCharm的调试器是用Python编写的。Python是一种解释性语言，这意味着在代码运行时必须将指令翻译为机器代码。

这会浪费时间 ，因此不如编译后的代码快。幸运的是，Python通过Cython支持编译的C-扩展。PyCharm附带了C版本的调试器代码，您可以对其进行编译以更快地在PyCharm中进行调试。

如果您的代码库很大，则使用此加速扩展是合理的。

获得更多加速所需的操作取决于您的平台：

* NIX

当用户启动调试器会话时，将显示通知弹出窗口。用户只需要单击此处的“ 安装”链接：

macOS和Windows

您什么都不需要做。我们捆绑了预编译的扩展程序，因此您已经在使用它们。

显示以下错误消息：

Python.h: no such file or directory 。

您缺少Python开发人员的标头文件和静态库。

使用包管理器为系统范围内的Python开发人员安装头文件和静态库。

例如，您可以使用以下命令：

  • 对于Python 2.x：

    须藤apt-get install python-dev

  • 对于Python 3.5：

    须藤apt-get install python3-dev

  • 对于Python 3.6：

    须藤apt-get install python3.6-dev

显示以下错误消息：

Command 'gcc' failed with exit status 1 。

您缺少C编译器。

安装C编译器以便为调试器生成Cython扩展。

在没有PyCharm UI的情况下使用Cython加速器

如果要手动编译Cython加速器，则他/她不应使用PyCharm UI。

在Ubuntu上，使用以下命令：

/ usr / bin / python3 / /helpers/pydev/setup_cython.py build_ext --inplace

请注意，您的路径应根据您的特定本地系统进行调整。
