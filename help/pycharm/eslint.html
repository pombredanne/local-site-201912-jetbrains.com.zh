PyCharm 2019.2帮助

ESLint

仅专业版支持此功能。该版本是商业版本，它提供了一组出色的功能：有关更多详细信息，请参见比较矩阵 。

PyCharm与ESLint集成这带来了广泛的棉绒规则，也可以通过插件扩展。键入时，PyCharm会在编辑器中立即显示ESLint报告的警告和错误。借助ESLint，您还可以使用JavaScript标准样式 。

要查看问题的描述，请将鼠标悬停在突出显示的代码上。

默认情况下，PyCharm会根据ESLint配置中的严重性级别来标记检测到的问题。请参阅配置ESLint突出显示以了解如何覆盖这些设置。

在开始之前，请确保您的计算机上装有Node.js。

安装ESLint

  1. 在嵌入式终端 （ Alt+F12 ），键入以下命令之一：

    • npm install eslint -g用于全局安装。

    • npm install eslint --save-dev将ESLint安装为开发依赖项。

  2. （可选）安装其他插件，例如eslint-plugin-react，以启动React应用程序。

在PyCharm中激活和配置ESLint

默认， PyCharm使用项目node_modules文件夹中的ESLint包和存储当前文件的文件夹中的.eslint。*配置文件。如果在当前文件夹中找不到.eslint。* ，PyCharm将在其父文件夹中寻找一个项目，直到项目根目录。

如果您有多个package.json文件，其中列出了ESLint作为依赖项，则PyCharm会为每个package.json启动一个单独的进程，并处理其下面的所有内容。这使您可以将特定的ESLint版本或特定的插件集应用于monorepo或具有多个ESLint配置的项目中的每个路径。

在所有新的PyCharm项目中，此行为都是默认行为。要在先前创建的项目中启用它，请转至语言和框架| JavaScript | ESLint 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），然后选择自动ESLint配置选项。

您还可以手动配置ESLint以使用自定义ESLint软件包和配置。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到语言和框架| JavaScript | ESLint 。

  2. 选择“ 手动配置”选项以使用自定义ESLint软件包和配置。

  3. 在“ 节点解释器”字段中，指定Node.js的路径。 如果您遵循标准安装过程，PyCharm会检测路径并填写字段本身。

  4. 在“ ESLint软件包”字段中， 指定eslint或标准包装的位置。

  5. 选择要使用的配置。

    • 通过自动搜索 ，PyCharm会查找.eslintrc文件或尝试检测在以下位置定义的配置eslintConfig在package.json中 。PyCharm首先在要插入文件的文件夹中寻找.eslintrc或package.json ，然后在其父文件夹中寻找，直到项目根。

    • 选择配置文件以使用自定义文件，然后在路径字段中指定文件位置。

    从ESLint官方网站了解有关配置ESLint的更多信息。

  6. 可选地：

    • 在“ 额外的ESLint选项”字段中，指定运行ESLint的其他命令行选项，使用空格作为分隔符。

      从ESLint官方网站了解有关ESLint CLI选项的更多信息。

    • 在其他规则目录字段中， 使用其他代码验证规则指定文件的位置。这些规则将在.eslintrc或上述指定的自定义配置文件中的规则之后应用，并因此将覆盖它们。

为ESLint配置突出显示

默认情况下，PyCharm根据来自ESLint配置的严重性级别标记检测到的错误和警告。例如，错误用红色波浪线突出显示，而警告则用黄色背景标记。有关详细信息，请参见代码检查和更改检查严重性 。

在ESLint配置中更改规则的严重性级别

  • 在.eslintrc或以下eslintConfig在package.json中 ，找到要编辑的规则并将其ID设置为1warn或者2error 。

    从ESLint官方网站了解更多信息。

您可以覆盖ESLint配置中的严重性，以便PyCharm忽略它们，并以错误，警告或自定义颜色的形式显示linter报告的所有内容。

忽略配置中的严重性级别

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择编辑器|检查 。将打开“ 检查”页面 。

  2. 在中央窗格中，转到JavaScript |代码质量工具| ESLint 。

  3. 在右侧窗格中，清除“ 使用配置文件中的规则严重性”复选框，然后选择要使用的严重性级别，而不是默认级别。

从ESLint导入代码样式

您可以将一些ESLint代码样式规则导入PyCharm JavaScript代码样式设置 。这使得PyCharm在自动完成，生成或重构代码或添加导入语句时，可以为项目使用更准确的代码样式选项。当您使用“重新格式化”操作时，PyCharm将不再从ESLint角度中断正确格式化的代码。

PyCharm可以理解所有正式格式的ESLint配置： .eslintrc JSON文件， package.json文件以及eslintConfig字段以及JavaScript和YAML配置文件。

  • 首次打开项目时，PyCharm会自动从项目ESLint配置中导入代码样式。

  • 如果您的ESLint配置已更新（手动或从版本控制中），请在编辑器中将其打开，然后从上下文菜单中选择“ 应用ESLint代码样式规则” 。

    或者，只是回答是 从ESLint“应用代码风格？文件顶部的“”问题。

    应用规则的列表显示在“ 事件日志”工具窗口中：

使用JavaScript标准样式

您可以将JavaScript标准样式设置为应用程序的默认JavaScript代码样式，以便在键入代码或重新设置格式时应用其主要规则。由于Standard是基于ESLint的，因此您也可以通过PyCharm ESLint集成来使用Standard。

要安装JavaScript Standard

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install standard --save-dev

    从JavaScript标准样式官方网站上了解更多信息。

通过ESLint启用Standard的linting

如果您在哪里打开一个项目standard在项目的package.json文件中列出时，PyCharm会自动启用Standard的linting。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到语言和框架| JavaScript |代码质量工具| ESLint 。

  2. 在打开的ESLint页面上，选择“ 手动ESLint配置”并指定安装位置。 standard ESLint软件包字段中的软件包 。

将JavaScript标准样式设置为默认样式

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |代码样式| JavaScript ）。

  2. 在代码样式上。将打开一个JavaScript页面，单击从中设置 ，然后选择预定义样式|。 JavaScript标准样式 。该样式将替换您当前的方案。

使用ESLint整理TypeScript代码

PyCharm会在以下情况下突出显示ESLint在.ts和.tsx文件中报告的错误@typescript-eslint/parser在项目ESLint配置中设置为解析器。从typescript-eslint repo中的自述文件中了解更多信息。

在新项目中将ESLint用于TypeScript

  1. 在嵌入式终端 （ Alt+F12 ），输入：

    npm install @typescript-eslint/parser @typescript-eslint/eslint-plugin --save-dev

    如果您全局安装了ESLint，请安装@typescript-eslint/eslint-plugin全球范围内。从@ typescript-eslint / eslint-plugin官方文档中了解有关安装和版本兼容性的更多信息。

  2. 在.eslintrc配置文件中或在eslintConfig在package.json中 ，添加：

    {“ parser”：“ @ typescript-eslint / parser”，“ plugins”：[“ @ typescript-eslint”]，“ extends”：[“ plugin：@ typescript-eslint / recommended”]}

使用ESLint禁止输入TypeScript代码

  • 如果您已经在使用@typescript-eslint/parser但您不想使用ESLint检查TypeScript代码，请将.ts或.tsx添加到.eslintignore文件中。

ESLint 4.0

如果使用的是早期版本的ESLint，则必须安装babel-eslint ， typescript-eslint-parser ， 要么eslint-plugin-typescript因为ESLint 4.0和更早版本不支持作用域软件包。

使用babel-eslint

  1. 在嵌入式终端 （ Alt+F12 ），输入：

    npm install eslint babel-eslint --save-dev

    从babel-eslint官方文档中了解有关安装和版本兼容性的更多信息。

  2. 在.eslintrc配置文件中或在eslintConfig在package.json中 ，添加：

    {“ parser”：“ babel-eslint”}

使用typescript-eslint-parser

  1. 在嵌入式终端 （ Alt+F12 ），输入：

    npm install typescript-eslint-parser --save-dev

    从typescript-eslint-parser官方文档中了解更多信息。

  2. 在.eslintrc配置文件中或在eslintConfig在package.json中 ，添加：

    {“ parser”：“ typescript-eslint-parser”}

使用eslint-plugin-typescript

  1. 在嵌入式终端 （ Alt+F12 ），输入：

    npm install typescript-eslint-parser eslint-plugin-typescript --save-dev

    如果您全局安装了ESLint，则也要全局安装eslint-plugin-typescript。从eslint-plugin-typescript官方文档中了解更多信息。

  2. 在.eslintrc配置文件中或在eslintConfig在package.json中 ，添加：

    {“ parser”：“ typescript-eslint-parser”，“ plugins”：[“ eslint-plugin-typescript”]}
