一般用法

此功能在PyCharm的教育版中不可用。

该插件的用法简单明了，但非常灵活。您可以将提供的注释集添加到项目中并开始使用它们，将插件配置为使用自定义注释集，或者简单地使用UI配置使PyCharm了解例如String参数。 Pattern.compile()应视为正则表达式。

使用注释

IntelliLang使用三个基本注释： @Language ， @Pattern和@Subst 。

  • @Language负责语言注入功能。

  • @Pattern用于根据特定的正则表达式模式验证字符串。

  • @Subst用于将非编译时间常数表达式替换为固定值。这使您可以验证模式并构建注入语言的前缀/后缀（请参见下文），即使对于在运行时已知包含某些类型值的非恒定表达式也是如此。

IntelliLang随附的注释位于文件批注 .jar中，该文件位于％product_home％/ plugins / IntelliLang / lib中 。

@语言

的@Language批注可用于注释字符串字段，局部变量，方法参数和返回字符串的方法。这将导致分配给字段/变量，作为参数传递或用作方法的返回值的字符串文字被解释为指定的语言。

此外，还有语言不匹配检查，当分配了字段/变量或方法返回的值时，该检查将检查预期语言与实际语言之间的冲突。

该插件支持直接和间接注释，即您可以像这样直接使用注释：

@Language（“ JavaScript”）字符串代码=“ var x = 1 + 2”;

或像这样注释另一个注释类：

@Language（“ XPath”）public @interface XPath {}

然后可以简单地用于将元素注释为@XPath 。

很容易获得language-id必须作为注释的value属性提供：IntelliLang通过常规的代码完成操作提供可用语言的列表。只需从ctrl-space弹出窗口中选择适当的语言：

@图案

的@Pattern批注负责标记必须符合某个正则表达式的字符串，并且可以与@Language注解。这意味着可以创建派生的注释，例如@Number要求String包含一个或多个数字的注释：

@Pattern（“ \\ d +”）public @interface编号{}

实际上，预定义的注释已经包含两个这样的派生注释：第一个， @PrintFormat ，匹配由java.util.Formatter还有一个@Identifier ，描述有效的Java标识符。这些都可以使用，而无需指定任何其他模式。

@Subst

的@Subst批注用于替换不是编译时常量的引用，这使插件可以基于替换值与运行时预期值兼容的假设进行模式验证 。如果该值与期望的模式不匹配，则插件将抱怨。

它还有助于为语言注入功能构建有效的前缀/后缀上下文（请参阅下一节）。考虑以下示例：

@Subst（“ Tahoma”）最终字符串字体= new JLabel（）。getFont（）。getName（）; @Language（“ HTML”）字符串消息=““ + ... +”“;

不替换变量的值font具有价值Tahoma （实际上，此处只是一个字符），因此插入的片段在语法上是不正确的，从而导致错误， 该术语应在“ font: ”指令。

提供上下文：前缀和后缀

注释元素时，可以提供在解析语言片段时应该在前面添加/添加的前缀和后缀。这可以用来提供上下文信息，即，如果JavaScript注入的前缀是“ var someContextVariable;”，则PyCharm将知道声明了someContextVariable变量，并且不会警告使用该变量时未声明它。

除了提供前缀和后缀的手动功能之外，IntelliLang还可以从以下上下文中动态确定这些值：

@Language（value = JavaScript，prefix =“ function doSomethingElse（a）{}”）字符串代码=“ function doSomething（）{\ n” +“ var x = 1; \ n” +“ doSomethingElse（x）; \ n “ +”}“;

在此示例中，JavaScript语言将被注入到三个String文字中的每一个中，并且每个人的前缀和后缀都将根据之前和之后的表达式进行计算，以便所解析的结果文本是有效的JavaScript语法：

  • 不会显示“ missing'}'”错误：右括号是第一个文字后缀的一部分。

  • 变量x用于doSomethingElse(a);将被声明：其声明是第二个文字的前缀的一部分。

  • 功能doSomethingElse()也将众所周知：它是在静态提供的前缀中定义的。

重要笔记

  • PyCharm语言注入API存在一些问题，这些问题对注入的语言片段的前缀/后缀施加了某些限制。例如，不允许该语言的标记跨越元素的前缀/后缀。例如，如果前缀以空格字符结尾并且片段以空格开头，则可能发生这种情况。

    该插件通过修剪前缀/后缀并插入一个空格字符作为分隔符来处理这种特殊情况。但是，如果空格字符不是令牌分隔符（例如，适用于JavaScript字符串文字），则此方法不起作用。

    此类案件无法自动处理，PyCharm核心将产生断言。

  • 即使动态前缀/后缀计算为语言片段提供了适当的上下文，但某些情况仍可能无法按预期进行。最值得注意的是重构（重命名）和导航功能。
