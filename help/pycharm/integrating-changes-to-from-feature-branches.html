集成到/来自功能分支的更改

功能分支旨在用于特定功能。它通常由从主干下载的数据组成，并在完成功能部件上的工作后重新集成到主干中。您可以应用所有更改或选择更改的子集。PyCharm使用合并的更改创建一个更改列表，并将其提交。

要集成分支中的更改，请执行以下操作：

  1. 打开“ 版本控制”工具窗口，然后切换到“ Subversion工作副本信息”选项卡 。

  2. 单击“ 合并自”链接，然后从弹出菜单中选择更改源。可用的选项有：

    • trunk ：选择此选项可将更改从主干合并到当前分支。

    • 分支 ：选择此选项可将特定分支的更改应用于当前分支。从“ 分支”弹出窗口中选择源分支。

    • 标签 ：选择此选项可将特定分支的更改应用于当前分支。从“ 标签”弹出窗口中选择源分支。

    要编辑分支列表，请选择“ 配置分支”选项，然后在打开的“ 配置子版本分支”对话框中更新分支列表。

  3. 在合并中打开对话框，指定要应用的更改范围。

    • 要应用所有更改，请单击“ 全部合并”按钮。

    • 要应用更改的子集，请单击“ 选择要合并的修订”按钮。在显示的修订列表中，通过选中所需修订旁边的复选框来指定要应用更改的修订。要应用所选更改，请单击“ 合并所选”按钮。

      要应用所有更改而不管选择如何，请单击“ 全部合并”按钮。

  4. 在“ 冲突”对话框中，查看在合并过程中出现问题的文件列表，并使用以下按钮解决问题 ：

    • 接受您 -单击此按钮可使PyCharm强制您进行更改。

    • 接受他们的 -单击此按钮可使PyCharm覆盖您的更改。

    • 合并 -单击此按钮以打开合并工具并解决那里的冲突 。
