附加到流程

使用PyCharm可以在运行从您的操作系统或使用PyCharm终端启动的Python脚本时附加到Python本地进程，但不能在调试模式下进行。请注意，您无法连接到远程进程。

Ubuntu用户的先决条件

如果您在Ubuntu（或可能在其他Linux发行版）上使用PyCharm，则在第一次尝试附加到进程时可能会出现错误消息ptrace：不允许操作 。在这种情况下，请执行以下任一操作：

  • 要暂时禁用此限制，请输入命令：

    回声0 |须藤tee / proc / sys / kernel / yama / ptrace_scope

  • 要永久禁用此限制，请打开/etc/sysctl.d/10-ptrace.conf文件进行编辑并更改该行kernel.yama.ptrace_scope = 1至kernel.yama.ptrace_scope = 0 。
    要立即应用更改，请输入以上命令echo 0 | sudo tee /proc/sys/kernel/yama/ptrace_scope 。或者，运行sudo service procps restart或重新启动系统。

您可以在此处找到更多详细信息。

附加到流程

  1. 从主菜单中，选择运行|附加到处理或按Ctrl+Alt+F5 。

  2. 从可用的本地进程列表中，选择所需的进程。

    通过键入名称或PID的前几个字母来简化搜索。如果有多个调试器可用于选定的进程，则可以在弹出菜单中选择它们。

    您可能只想在“ 附加到进程”列表中显示特定进程。将所需的模式添加到Python Debugger设置（ Settings / Preferences | Build，Execution，Deployment | Python Debugger ）中的相应字段。例如，如果您只想附加与此字符串匹配的进程，请输入“ Scientific”：

  3. 进行调试的方式与您在PyCharm中通常执行的方式相同（ 设置断点 ， 逐步执行 ， 暂停和继续执行过程， 评估表达式等）

    请注意，如果附加的进程不提供调试信息，则断点将不可用，并且将在反汇编视图中执行单步执行。

  4. 完成后，分离该过程：选择Run |停止或单击停止进程按钮 调试工具窗口中的 。
