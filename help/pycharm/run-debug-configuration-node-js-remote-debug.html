运行/调试配置：附加到Node.js / Chrome

仅专业版支持此功能。该版本是商业版本，它提供了一组出色的功能：有关更多详细信息，请参见比较矩阵 。

以下内容仅在安装并启用Node.js插件时有效。

运行编辑配置| |附加到Node.js / Chrome

在此对话框中，创建用于调试已经运行的Node.js应用程序的配置。这种方法使您无需重新启动Node.js服务器即可重新启动调试会话。

还可以使用此配置来调试NW.js应用程序的后台程序和其他主要进程，请参阅“运行/调试配置：NW.js” 。

在你开始前

  1. 下载并安装Node.js。

  2. 如管理插件中所述，在“ 插件”页面上安装Node.js插件。

特定附加到Node.js / Chrome配置设置

项目

描述

主办

在此字段中，指定运行应用程序的主机。

港口

在此字段中，指定传递给的端口--inspect要么--debug启动要连接的Node.js进程时。从控制运行中的应用程序的“运行”工具窗口中的信息消息中复制此端口号。

附加到

在此区域中，选择要使用的调试协议：

  • Chrome或Node.js> 6.3以--inspect开头：选择此选项以使用Chrome调试协议 。

  • Node.js <8以--debug开头：选择此选项以使用V8调试协议 （也称为Legacy Protocol ）。

自动重新连接

选中此复选框，以使PyCharm内置调试器可以在更改应用程序代码后重新附加到正在运行的Node.js进程。 这使您可以调试使用nodemon实用程序的 Node.js应用程序，该实用程序会在代码更新时自动重新加载Node.js进程。有关详细信息，请参见调试使用nodemon的应用程序 。

本地文件的远程URL

在该区域中，指定项目文件夹的远程路径。如果正在运行的应用程序的根文件夹与您的PyCharm项目根文件夹的名称不同，这将很有帮助。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

工具列

运行/调试配置的树形视图具有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

查看和编辑所选运行/调试配置的模板（即默认设置）。模板显示在“ 模板”节点下，并用于新创建的配置。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在PyCharm中定义，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 启动Web浏览器 ：选择此选项以启动浏览器。在打开的对话框中，选择浏览器的类型并提供起始URL。另外，指定是否要使用JavaScript调试器启动浏览器。

  • 运行文件监视程序 ：选择此选项可使PyCharm应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

Alt+Delete

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
