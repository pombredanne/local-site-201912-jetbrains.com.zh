调试

开始调试会话

  1. 要调试CoffeeScript ， TypeScript和Dart代码，您需要为其生成源映射 。这将设置原始代码和生成的JavaScript代码中各行之间的对应关系。如果未生成任何源映射，则将无法正确识别和处理您的断点。

  2. 为要调试的应用程序定义运行/调试配置 。

  3. 创建一个断点 。

  4. 点击调试纽扣 。从启动调试器会话中了解更多信息。

根据需要暂停或恢复调试会话。

在调试器会话期间， 逐步执行断点 ， 评估表达式 ，实时更改值以检查挂起的程序并设置监视 。

启动调试会话后， 标记“ 调试工具”窗口的图标切换到指示调试过程处于活动状态。

除错

为项目配置运行配置后 ，可以按以下方式以调试模式启动它： Shift+F9 。

在“ 调试”工具窗口中，您可以看到框架和线程及其状态，变量和监视的列表。选择框架时，将看到与所选框架相对应的变量。

断点

自定义断点设置

要自定义断点设置，请按Ctrl+Shift+F8 。要查看项目中的所有断点（带有其他设置），请单击“ 更多”或按相同的快捷方式Ctrl+Shift+F8再次。

不要暂停代码执行

使用动作断点可以在代码的特定行评估变量，而无需暂停代码执行。要创建动作断点，请在按住的同时单击装订线Shift 。

创建一个临时断点

要创建仅停止一次的断点，请在按住的同时单击左侧装订线Shift+Alt 。有关更多信息，请参阅断点部分。

禁用断点

要禁用断点，请在按住的同时单击断点Alt 。有关详细信息，请参见“ 断点 ”部分。

调试器会话

明智地进入

有时，您会停留在某个行上并想进入一个特定的方法，而不是第一个将被调用的方法。在这种情况下使用智能步骤到按Shift+F7选择一种特定的方法。

有关详细信息，请参阅“ 选择方法进入”一节。

运行到光标

  1. 创建一个断点。

  2. 运行调试会话。要运行调试会话，请单击“ 运行应用程序”图标（ ）在装订线区域，然后选择调试 。

要停止在光标位置执行代码而不添加其他断点，请单击“ 运行到光标”图标（ ）或按Alt+F9 。或者，您可以单击装订线区域中的行号。

图标在“调试工具”窗口的工具栏参考中进行了介绍。

评估表达

在调试模式下，你可以计算表达式按Alt+F8 。

“ 评估”对话框支持代码完成。

若要查看各种函数调用的行为在调试会话期间如何变化，请设置自定义监视。您可以在添加，编辑和删除手表中阅读有关手表的更多信息。

有关详细信息，请参阅“ 评估表达式 ”部分。

配置调试器设置

要更改调试器设置，请单击“ 文件” |“更改” 。设置 。在“ 设置”菜单中，单击“ 构建，执行，部署” |“部署”。调试器 。
