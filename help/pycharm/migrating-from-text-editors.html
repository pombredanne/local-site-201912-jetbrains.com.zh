从文本编辑器迁移

本指南适用于使用文本编辑器（例如Vim，Emacs或Sublime Text）并且需要有关切换到PyCharm的信息的Python开发人员。它包括对IDE概念的简短讨论，然后探讨了特定的差异。

什么是IDE？

Python开发人员长期以来使用各种工具来编写其代码。尽管这些内容模糊不清，但这些工具可分为两大阵营：文本编辑器和集成开发环境（IDE）。为了简化起见，文本编辑器专注于单个文件的编辑，从而提供了非常简洁的体验。

顾名思义，IDE具有更广阔的视野。他们希望查看您的整个项目以及与编码有关的所有活动，并将这些活动统一到一个一致且功能强大的UI中。虽然这两个部分重叠，但总体范围是主要区别。

因为像PyCharm这样的IDE会在您的项目，依赖项以及平台本身中查看所有代码，所以它可以在编码活动中提供很多帮助。这种分析和协助显示了许多强大的功能：

  • 自动完成功能可通过完成输入来加快准确的编码速度

  • 快速修复可以发现常见错误并提供针对特定语言的更正

  • 代码意图建议对常见的Python模式进行优化和改进

  • 代码重构消除了常用Python重构的麻烦

  • 模板自动执行重复任务

  • 代码导航分析所有代码的结构和语义以及所使用的代码，以提供丰富的方法在软件中移动

尽管文本编辑器提供了其中的一些功能，但PyCharm不仅限于面向字符串的操作，而且还解决了代码和语言的语义问题，从而提供了智能帮助。PyCharm可以理解Python，也可以理解您的代码，因此它可以跳转到定义符号的位置或使用类的位置，或在整个代码库中重构方法及其用法。实际上，这些功能超出了Python：PyCharm从IntelliJ家族的弟兄那里继承了对Web开发和许多其他领域的一流IDE支持。

除了这些编码辅助功能之外，PyCharm还可以将相关操作绑定到同一环境中：版本控制，数据库管理，配置文件，测试运行，远程环境等。这些功能都是以集成方式提供的：在查看差异时进行更改可以自动触发重新运行测试。

最终，这种力量伴随着以产品为中心的思维定势：一切正常。尽管其他工具列出了很多可能的功能，但您可以自行组装和维护汇总。对于喜欢修补的人来说很好。使用PyCharm作为您的IDE，这些功能每年都可立即使用并得到专业支持。

PyCharm界面

由于诸如PyCharm之类的IDE具有如此广泛的范围，因此与简单的文本编辑器相比，用户界面（UI）需要更多的方向。例如，这是PyCharm中的一个典型项目：

  1. 左侧的项目视图显示您的项目文件。

  2. 右侧的编辑器 ，您实际在其中编写代码的地方。它具有用于在打开的文件之间轻松导航的选项卡。

  3. 编辑器上方的导航栏还使您可以快速运行和调试应用程序以及执行基本的VCS操作 。

  4. 左边的装订线 ，即编辑器旁边的垂直条纹，显示了您的断点，并提供了一种方便的方法来浏览代码层次结构，例如转到定义/声明。它还显示行号和每行VCS历史记录。

  5. 右侧装订线，位于编辑器的右侧。PyCharm会持续监控您的代码质量，并始终在正确的装订线中显示其代码检查的结果：错误，警告等。右上角的指示器显示整个文件的代码检查的总体状态。

  6. 工具窗口是连接到工作区底部和侧面的专用窗口，可用于访问典型任务，例如项目管理，源代码搜索和导航，与版本控制系统的集成等。

  7. 状态栏指示项目和整个IDE的状态，并显示各种警告和信息消息，例如文件编码，行分隔符，检查配置文件等。它还提供对项目解释器设置的快速访问。

还是太多？如果您要从文本编辑器过渡并希望获得斯巴达式外观，PyCharm提供了几种解决方案：

  • Windows和工具栏可以隐藏直到需要

  • 免干扰模式和其他模式消除了大部分视觉效果，重点关注通过键盘快捷键和动作搜索完成交互的代码

专案

如前所述，PyCharm不仅将文件视为文件，而且将项目视为完整的软件解决方案。不仅是项目随附的软件，而且还有相关任务。

该项目不仅包含这些工件。在一个项目中，您可能会以特定顺序在选项卡中打开3个文件。该信息将保存在项目的元数据中，位于项目根文件夹的.idea子目录中。因此，下次打开项目时，将还原为该配置。这不仅仅适用于配置：PyCharm PyCharm自动保存您在文件中所做的更改。保存是由各种事件触发的，例如编译，运行，调试，执行版本控制操作，关闭文件或项目或退出IDE。大多数实际事件是预定义的，无法配置，但是您可以确保所做的更改不会丢失。 在编辑时，由本地历史记录作为后盾，可以将代码恢复为以前的状态。PyCharm在其IDE中具有大量功能，它是项目配置的一部分：Python设置，运行代码的方案，数据库的连接信息等。

您的第一个项目

如果您当前使用文本编辑器，那么进入PyCharm的第一步将询问您有关创建项目的信息。您有很多选择和机会。让我们讨论其中的一些。

首先，您可能是从空白列表开始，或者是从磁盘上的现有目录开始，或者是从尚未签出的远程存储库开始。PyCharm 具有特定的功能可在所有这些情况下提供帮助 。例如，在选择项目的起点时能够浏览GitHub帐户的存储库真是太好了。

接下来，您可能要在项目中使用一个或多个Python解释器。PyCharm支持本地安装的Python 2和3，以及制作或重新使用虚拟环境。它还支持基于Anaconda的解释器，远程解释器，Vagrant和Docker。您可以在启动PyCharm的“新建项目”向导之前进行设置，或者在创建项目时指定（甚至创建）这些。同样，您可以在创建项目后改变主意。

借助PyCharm专业版，我们可以轻松开始使用流行类型的项目。例如，创建Django项目将遵循Django最佳做法生成目录结构，并在特殊的工具窗口中添加“运行配置”以启动Django服务器。

Emacs和emacsIDEA

Emacs和vim是针对核心开发人员的两种流行的文本编辑器。PyCharm提供了一个与许多Emacs键序列匹配的键映射 。

尽管PyCharm没有适用于IdeaVim的 Emacs插件，但社区中有一个名为emacsIDEA的活动插件，该插件提供了Emacs和Emacs扩展（例如AceJump）中的一些有用功能。

您可以将Emacs配置为PyCharm的外部工具 ，并将其用于编辑文件。本教程对此过程进行了描述。

定制和扩展

PyCharm的IntelliJ基础非常广泛（在其上构建了许多IDE）并且非常成熟，这还带来了另一个好处：PyCharm可以高度自定义。例如，虽然默认情况下带有浅色主题，但它还提供了一个可选的深色主题“ Darcula”，它与编辑人员的现代趋势相匹配。除了调整血红素之外，还可以自定义许多外观选项：字体大小，配色方案，工具栏设置，气球行为等。

“外观”只是自定义选项的开始。我们讨论了切换到其他捆绑的keymap的问题 。但是您可以通过多种方式扩展和自定义按键映射，将不同的击键映射到不同的动作。除此之外，PyCharm的行为可以扩展：安装插件，用于操作的自定义“模板”，更改代码意图等。

多个游标

这是其他工具的开发人员所保证的一项强大功能：多个游标。Sublime Text率先提出了这一概念，IntelliJ在其平台上的所有IDE中添加了该概念。使用PyCharm的multicursor ，可以使用多个插入符，并且将IDE动作应用于每个插入符。这在PyCharm 多项选择截屏中详细显示。

标签页和拆分窗口

成熟的文本编辑器（例如Emacs和vim）以将屏幕划分为多个区域的丰富方式而闻名，从而可以一次看到多个文件。这些工具不仅仅具有基础知识：您可以以丰富的方式在这些区域之间移动，垂直或水平分割，重新分割，然后通过键盘进行所有这些操作。

PyCharm还具有一套成熟的功能来显示多个文件。最基本的说，您可以将多个文件打开到选项卡中，这些选项卡可以手动重新组织或按字母顺序排序。可以将选项卡固定或分离到单独的窗口中。

除了选项卡之外，您还可以将编辑器窗口划分为多个独立的窗格，以一次显示多个文件/选项卡。拆分可以垂直或水平进行。您可以通过将选项卡移至相反的组来重新排列当前打开的文件。

最后，上述所有操作都可以以键盘为中心的方式完成，从而无需使用鼠标即可轻松处理多个文件和窗口。

运行代码

文本编辑器提供了用于处理Python代码运行的工具，而PyCharm通过一种称为运行配置的工具将这一工作进一步扩展了。这些使您可以定义与运行Python（或其他语言，例如JavaScript）代码相关的参数。然后，当您运行该运行配置时，将应用所有关联的设置。

此外，PyCharm在专门的工具窗口中运行代码，该窗口提供了许多用于运行代码的功能：停止，重新运行等。输出显示在迷你控制台中，错误显示带行的追溯，可以单击这些行以跳至该文件的行号。这个工具窗口，以及运行和重新运行代码，都可以通过键盘来驱动。

这些运行配置还适用于特殊类型的运行：带有特定窗口的测试运行器配置，用于显示测试输出，使用可视调试器的调试配置，测试覆盖率运行配置，探查器运行配置，甚至是并发图配置。所有这些都为专业的Python开发人员的工作流程提供了真正的价值。

最后，PyCharm Professional为其支持的框架添加了新的运行配置类型。例如，Pyramid运行配置了解Pyramid配置文件和Pyramid独有的功能。

生产力提示

从命令行运行

传统上，PyCharm与其他应用程序一样（从桌面）启动，但是您也可以从命令行使用PyCharm。这些可以在初始安装期间进行设置。例如，在Linux和macOS上，可以使用/ usr / local / bin / charm在PyCharm中打开文件。您还可以运行检查并使用PyCharm的差异工具。

打开一个文件

您无需创建项目即可打开某些特定文件。PyCharm的“ 打开”菜单（或命令行“魅力”程序）可以在现有项目的窗口或新窗口中打开一个文件。

临时文件

如果你需要的代码或内容片段临时编辑，并且不希望将其保存在一个项目甚至一个文件，PyCharm提供了一个名为设施划痕 。您可以创建暂存文件（与一种语言相关联，因此具有突出显示的语法，代码完成，可运行等）或暂存缓冲区（一小段文本）。

这些划痕具有许多功能：您可以列出它们（最多五个），关闭和删除它们，将它们与一种语言相关联并重新组织它们。划痕保存在项目的特殊区域中。

内置提示

首次安装PyCharm时，每次启动时它都会为您提供一个友好的提示。它还使您可以关闭这些提示。如果以后要查看更多提示 ，请选择“ 帮助” |“帮助”。主菜单上的每日提示 。

安装插件

如到目前为止多次提到的，PyCharm和IntelliJ具有庞大的IDE插件生态系统。PyCharm本身拥有将近500个插件，涵盖许多不同的类别。

实际上，PyCharm IDE本身是由插件组成的。例如，这就是为什么PyCharm从WebStorm继承许多Web功能的原因：它们通过插件共享相同的核心功能。

通过PyCharm的“设置/首选项”对话框，可以浏览，安装，更新和删除插件 。另外，还会检查插件以及PyCharm本身是否有更新。如果插件已更新，将要求您下载并重新启动PyCharm。

多种语言开发

尽管文本编辑器与Python一起支持多种语言，但PyCharm在Python之外带来的独特功能。例如，PyCharm是一流的Web开发IDE。它不只是语法突出显示和自动完成的HTML，CSS和JavaScript。它提供了全部功能：语义感知的代码检查，重构，调试，启动实时服务器，将SASS文件转换为CSS，浏览npm运行脚本等等。

这不仅适用于Web开发，还包括：IPython Notebook仿真，reStucturedText等。PyCharm如何做到这一点？如上所述，它与其他基于IntelliJ的IDE兄弟共享JetBrains插件。

小地图

其他工具，例如Sublime Text和Visual Studio（通过附加组件），提供了在大型文档中滚动的可视模式。这样，普通滚动条将替换为文档的图形缩略图以及其中的当前滚动位置。

PyCharm用户还可以使用CodeGlance JetBrains插件来获得此插件，该插件的编写符合所选主题。
