创建和应用实时模板（代码段）

本教程是关于什么的？

PyCharm附带了很多实时模板...有几个针对Python的预定义实时模板。本教程旨在指导您为Python类声明创建活动模板，并使用该活动模板。

Python编程超出了本教程的范围。有关详细信息，请参阅Python文档。

活动模板的基本知识和用法在这里也没有讨论。您可以在“实时模板”部分中找到有关实时模板的类型，缩写，变量和存储的所有必要信息。

先决条件

确保您使用的是PyCharm 2.7或更高版本。如果您仍然没有PyCharm，请从此页面下载。要安装PyCharm，请根据您的平台按照说明进行操作。

本教程是使用PyCharm Professional 2016.1版创建的。

  • 请点击再次。这次，选择实时模板 。新模板将出现在拥有焦点的组中，即user 。

2。定义模板缩写和上下文

因此，让我们输入模板的缩写 。在我们的示例中，我们将输入单词class在“ 缩写 ”字段中，然后输入描述（这是可选的，但是...）。

接下来，让我们选择新模板将应用的上下文。到目前为止，您已经看到上下文尚未定义-因此单击链接Define ，然后选择上下文（在我们的示例中是Python）。将来，当上下文已定义时，链接将更改为Change 。

扩展键为Tab ，默认情况下指定。

3。定义模板文字

在模板文本字段中输入以下代码：

class $ class $（$ object $）：“”“ $ cls_doc $”“”“ def __init __（self，$ args $）：”“” $ class $的构造函数“”“ $ END $

包含在模板文本中的部分$符号是模板变量 。您可以轻松地从整个模板文本中分辨出它们，因为它们在颜色上非常突出：

这些模板变量仍然是无效的，所以让我们先定义它们...除一个以外的所有内容（请稍等一下以了解原因）。

关于$ END $的旁注

您已经注意到模板变量$END$不建议编辑。这是因为模板变量$END$是预定义的 ，因此不可编辑。在模板扩展并填写所有必填字段之后，它始终代表光标的位置。因此，在本例中，光标将停留在新类声明的末尾。

使用自己的实时模板

首先，创建一个Python文件（ Alt+Insert -Python文件），并将其命名为sample.py 。

新的Python文件将打开以进行编辑。接下来，让我们在其中创建一个类声明。为此，请输入模板缩写class 。

正如预期的那样，该缩写扩展为存根Python类。红线标记下一个入口点：键入类名时（变量class ），它将在此特定位置输入。

注意，顺便说一句class已经多次使用-在类声明中，在类构造函数的注释中。在这种情况下， class已自动填写。

输入班级名称后，按Enter ，然后看到红线（框架）移动到下一个字段。因此，您必须填写所有必填字段，然后按Enter在最后。
