运行SSH终端

仅专业版支持此功能。该版本是商业版本，它提供了一组出色的功能：有关更多详细信息，请参见比较矩阵 。

您可以直接从PyCharm启动SSH会话 。通过在专用SSH终端上运行命令，您可以通过SSH隧道访问远程Web服务器或默认的远程解释器上的数据，主要是上传和下载文件。

准备在SSH终端中工作

  1. 确保已启用“ SSH远程运行”捆绑插件。如果插件被禁用，请按照“ 管理插件”中所述激活它。

  2. 确保SSH服务器在目标环境中可用：远程Web服务器或默认的远程解释器。

  3. 在目标环境中的SSH服务器上注册一个帐户，并根据服务器策略生成一对SSH密钥或密码。

    PyCharm不支持OpenSSH密钥的RFC 4716格式。请参阅解决方法 。

  4. 指定目标环境并指定设置以与其建立连接：

    1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 工具| SSH终端 。

    2. 在“ 连接设置”区域中，指定目标环境 ：

      • 默认远程解释器：选择此选项可使SSH终端中的命令在运行默认远程解释器的同一主机上执行。

      • 当前的Vagrant：选择此选项可使SSH终端中的命令在当前运行的Vagrant虚拟机上执行 。

      • 部署服务器：选择此选项可通过服务器访问配置之一访问在本地或远程Web服务器上执行的SSH终端中的命令。从列表中，选择用于指定目标环境的服务器访问配置，以及用于建立与目标环境的连接的设置。

        • 每次运行时选择服务器：如果选择此选项，则每次选择“ 工具” |“服务器”时，都必须从弹出窗口中选择所需的服务器访问配置。从主菜单启动SSH会话 。

        • 如果所需的服务器访问配置未出现在列表中，请单击“ 配置服务器 ”链接，然后在“ 部署”页面中定义一个。有关详细信息，请参阅“ 配置与远程主机的同步”部分。

    3. 从“ 默认编码”列表中，选择要在SSH终端中使用的所需编码。

启动SSH终端

  1. 在主菜单中，选择“ 工具” |“工具”。启动SSH会话 。或者，调用帮助|寻找行动 Ctrl+Shift+A对话框中，搜索start ssh .. ，然后选择Start SSH Session 。

  2. 根据连接设置，在“ 工具” | “ 工具”中定义。 SSH终端页面 设置/首选项对话框（ Ctrl+Alt+S ），则可能出现以下几种类型的行为：

    • 如果选择了默认远程解释器选项，则SSH终端将提供对运行默认远程解释器的同一主机的访问。

    • 如果选择了“ 当前Vagrant”选项，则SSH终端将提供对当前正在运行的Vagrant虚拟机的访问。有关详细信息，请参见Vagrant 。

    • 如果已选择选项“ 部署服务器” ，则SSH终端将提供对本地或远程Web服务器上数据的控制，该服务器可通过从列表中选择的服务器访问配置进行访问 。有关详细信息，请参阅配置与Web服务器的同步 。

    • 如果选择了“每次运行时选择服务器 ”选项，PyCharm将显示一个列表，从中选择所需的服务器访问配置 。
