运行/调试配置：Python

首次运行应用程序时，PyCharm会自动创建临时的“运行/调试”配置 。您可以修改它以指定或更改默认参数，并将其另存为永久的“运行/调试”配置。

先决条件

在开始之前，请确保已安装Python和相关框架。

有关详细信息，请参阅其各自的下载和安装页面：

  • 蟒蛇
  • Django的

使用此对话框可为Python脚本创建运行/调试配置。

配置选项卡

项目

描述

脚本路径/模块名称

单击列表以选择要运行的目标的类型。然后，在相应的字段中，指定Python脚本的路径或要执行的模块名称。

参量

在此字段中，指定要传递给Python脚本的参数。

指定脚本参数时，请遵循以下规则：

  • 使用空格分隔各个脚本参数。

  • 包含空格的脚本参数应使用双引号分隔，例如， some" "param要么"some param" 。

  • 如果脚本参数包含双引号，请使用反斜杠对双引号进行转义，例如：

    -s"main.snap_source_dirs=[\"pcomponents/src/main/python\"]" -s"http.cc_port=8189" -s"backdoor.port=9189" -s"main.metadata={\"location\": \"B\", \"language\": \"python\", \"platform\": \"unix\"}"
    。

环境

环境变量

该字段显示环境变量列表。如果列表包含多个变量，则用分号分隔。
默认情况下，该字段包含变量PYTHONUNBUFFERED设置为1。要填写列表，请单击浏览按钮，或按Shift+Enter并在“ 环境变量”对话框中指定所需的环境变量集。
要创建一个新变量，请单击 ，然后输入所需的名称和值。 您可能想要使用存储为文本文件中的一系列记录的变量填充列表，例如：

变量1 =值1变量2 =值2
只需复制文本文件中的变量列表，然后点击粘贴 （ ）在环境变量对话框中。变量将添加到表中。单击确定以完成任务。您可以随时在“ 环境变量”对话框中选择所有变量，然后单击“ 复制”。 ，然后将其粘贴到文本文件中。

Python解释器

从列表中选择一个预配置的Python解释器 。

当PyCharm停止支持任何过时的Python版本时，相应的项目解释器将标记为不支持。

口译员选项

在此字段中，指定要传递给解释器的字符串。如有必要，请单击 ，然后在编辑器中键入字符串。

工作目录

指定正在运行的任务要使用的目录。

  • 通过键盘快捷键创建默认的运行/调试配置时Ctrl+Shift+F10 ，或者通过从脚本的上下文菜单中选择“运行” ，工作目录就是包含可执行脚本的目录。此目录可能与项目目录不同。

  • 如果将此字段留空，则将使用PyCharm安装的bin目录。

将内容根添加到PYTHONPATH

将源根添加到PYTHONPATH

执行

使用Python控制台运行

允许使用Python控制台运行脚本或模块。

从重定向输入

允许将数据从文本文件重定向到标准输入。如果脚本需要一些输入并且您想自动提交值而不是在“ 运行”控制台中键入它们，请使用此选项。要启用重定向，请选中复选框并指定目标文本文件的路径。

Docker容器设置

仅当为项目选择基于Docker的远程解释器时，才会显示此字段。

请点击打开对话框并指定以下设置：

  • 禁用网络 ：选中此复选框可禁用网络。这对应于--net="none" ，这意味着在容器内部，外部网络资源不可用。

  • 网络模式 ：对应于该选项的其他值--net 。

    • bridge是默认值。将为网桥的网络上的容器分配一个IP地址，流量将通过该网桥路由到容器。
      默认情况下，容器可以通过其IP地址进行通信。要通过名称进行通信，必须将它们链接起来。

    • host ：使用容器内主机的网络堆栈。

    • container: ：使用通过其名称或id指定的另一个容器的网络堆栈。

    有关详细信息，请参阅网络设置文档。

  • 链接 ：使用此部分将要创建的容器与其他容器链接。这适用于Network mode = bridge并对应于--link选项。

  • 发布所有端口 ：将所有容器端口公开给主机。这对应于选项--publish-all 。

  • 端口绑定 ： 指定端口绑定列表。类似于使用-p与选项docker run 。

  • 额外主机 ：这对应于--add-host选项。有关详细信息，请参阅管理/ etc / hosts页面。

  • 卷绑定 ：使用此字段可以指定特殊文件夹（ 卷）和运行Docker守护程序的计算机的文件夹之间的绑定。这对应于-v选项。
    有关详细信息，请参见管理容器中的数据。

  • 环境变量 ：使用此字段可以指定环境变量及其值的列表。这对应于-e选项。有关详细信息，请参见页面ENV（环境变量） 。

请点击扩展表格。请点击 ， ， 要么组成列表。

说到设置与某些选项的对应关系（ --net ， --link等），请注意，这些选项来自Docker命令行参数 。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

允许并行运行

如果选中此复选框，则可以并行启动运行配置多次，而不必重新运行它。每个运行程序都将在“ 运行工具窗口”或“ 调试工具窗口 ”的自己的选项卡中启动 。

禁用后，每次启动运行/调试配置的新实例时，PyCharm都会检查同一运行/调试配置的其他实例是否存在，并显示确认对话框。如果在确认对话框中单击“ 确定” ，则将停止运行器的第一个实例，而将其替换。当启动两个相同类型的运行/调试配置实例时，这会占用过多的CPU和内存资源。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
