更新您的脚本

本部分包含准备进行远程调试所需的重要信息。

将PyCharm安装中的pydevd-pycharm.egg添加到Python路径或执行。

例：pip install pydevd-pycharm~=191.3490

请参阅使用PyCharm进行远程调试中的过程描述。

添加以下命令以连接到调试服务器

复制下面的命令行语句，然后将其粘贴到本地脚本中。

import pydevd_pycharm

pydevd_pycharm.settrace(, port=) ，在哪里

  • 从此调试配置的本地主机名字段获取。

  • 是从此调试配置的“ 端口”字段获取的数字，或者，如果未指定，则是写入控制台的一些随机数。

请注意，此命令的参数取决于此页面中指定的设置。默认设置的命令是：

pydevd_pycharm.settrace('localhost', port=$SERVER_PORT,stdoutToServer=True,stderrToServer=True)

对应于主机名'localhost'，端口号0，选中的复选框将输出重定向到控制台并在connect之后挂起 。

本地主机名

指定运行IDE的计算机的IP地址，通过该IP地址可以访问远程调试服务器。该主机名将自动替换为命令行。默认情况下，使用本地主机 。

港口

指定端口号，它将自动替换为命令行。如果使用默认的帖子编号（0），则每次启动此调试配置时，PyCharm都会在命令行中替换任意编号；如果您指定任何其他值，它将被永久使用。

路径映射

使用此字段可在本地路径和远程路径之间创建映射。点击浏览按钮会导致打开“ 编辑路径映射”对话框，您可以在其中添加新的路径映射并删除所选的路径映射。本地路径是项目的绝对路径，例如/ Users / jetbrains / PyCharmProjects / MyDeployment 。远程路径是相对于SFTP服务器根路径的，例如/ tmp / pycharm_project_923 。

将输出重定向到控制台

如果选中此复选框，则输出流和错误流将被重定向到PyCharm控制台，并且命令行将与stdoutToServer=True, stderrToServer=True

连接后挂起

如果选中此复选框，则调试器将在连接到IDE之后立即挂起，在settrace呼叫。
如果未选中此复选框，则调试器仅在达到断点或单击后才会挂起 ，并且命令行补充了suspend=False

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在PyCharm中定义，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 启动Web浏览器 ：选择此选项以启动浏览器。在打开的对话框中，选择浏览器的类型并提供起始URL。另外，指定是否要使用JavaScript调试器启动浏览器。

  • 运行文件监视程序 ：选择此选项可使PyCharm应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 启动React Native Bundler ：选择此选项以在运行或调试会话中自动运行捆绑程序。默认情况下，这是通过react-native start 。如果您的应用程序使用Expo ，则需要通过以下方式运行开发服务器start npm任务。为此，请单击 ，然后在“ 配置React Native”对话框中，选择npm脚本 ， 然后从列表中选择start 。

  • 编译TypeScript ：选择运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都会反映在生成的JavaScript文件中。 在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防万一检测到任何错误：

    • 如果选中“检查错误”复选框，则编译器将显示所有错误，并且运行配置将不会启动。

    • 如果清除“ 检查错误”复选框，则编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
