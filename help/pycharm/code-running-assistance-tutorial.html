运行/调试配置-这是什么？

您希望在PyCharm中运行或调试的每个脚本或测试都需要一个特殊的配置文件，该配置文件指定脚本名称，工作目录以及运行或调试所需的其他重要数据。PyCharm带有许多此类预定义的配置文件或运行/调试配置 ，这些配置文件可提供模式，您可以针对这些配置文件创建任意数量的自己的运行/调试配置。

每次单击“运行”或“调试”按钮（或在上下文菜单上选择“运行”或“调试”命令）时，实际上是在运行或调试模式下启动当前的运行/调试配置。

如果您看第一张图片，您会发现组合框中根本没有运行/调试配置。在出现的第二张图像上，用绿色圆圈标记。这意味着在上下文菜单中选择“运行'求解器”后 ，PyCharm会自动创建求解器运行/调试配置。现在，由于此运行/调试配置以绿色圆圈标记，因此它是当前的。

这是什么意思？

在“ 模板”节点下，您仅看到存根或样式。如果创建新的运行/调试配置 ，则会根据所选模式来创建它。如果您在“ 模板”节点下更改了任何内容，则您创建的相应运行/调试配置将有所不同。

例如，您想要更改Python解释器以使用远程或其他本地解释器。确定，在Python页面中选择所需的解释器-然后，任何新创建的Python类型的运行/调试配置都将使用此解释器。

在Python节点下，您将看到唯一的运行/调试配置Solver 。它属于Python类型，是针对模式Python创建的。它以正常不透明度的图标表示-对应于永久性的运行/调试配置（请记住，它已变为永久性，因为您已经保存了它-但是，任何专门创建的运行/调试配置也将变为永久性）。例如，为相同的Solver脚本创建Python类型的新运行/调试配置，并将其称为“ Solver1”。

如果在现有的运行/调试配置之一中进行任何更改，则仅会影响此特定的运行/调试配置。

调试运行

首先，为什么我们需要调试？假设您遇到了运行时错误。如何找出其起源？这是需要调试的地方。

使用PyCharm，您可以在不离开IDE的情况下调试应用程序。您唯一需要做的就是将断点放置在所需的位置。让我们详细研究一下。

断点-这是什么？

断点是源代码的一行，到达该行时，PyCharm将在其中暂停。PyCharm可以识别几种类型的断点 ，每种断点都有自己的图标表示。

在这里，我们将使用Python行断点。

设置断点

这绝对是该过程中最简单的部分。只需在要浏览的行上单击左侧装订线-即可找到断点：

注意，每个断点在整行上也用红色条纹表示。此颜色对应于尚未达到的断点。稍后，我们将看到断点处的行如何更改其颜色。

顺便说一句，删除断点同样容易-只需再次单击左侧装订线即可。

将鼠标指针悬停在断点上。PyCharm显示了具有最基本断点信息的工具提示-行号和脚本地址。但是，如果要更改断点设置，则必须右键单击一个断点。尝试自己更改断点设置，然后查看断点图标如何更改。
