创建就地服务器配置

仅专业版支持此功能。该版本是商业版本，它提供了一组出色的功能：有关更多详细信息，请参见比较矩阵 。

就地服务器是这样一种服务器，其文档根目录是项目根目录的父目录，无论是直接目录还是非直接目录。换句话说，Web服务器正在您的计算机上运行，您的项目位于其文档根目录下（例如，在/ htdocs文件夹中），并且您可以直接在服务器上进行开发。

要在此设置中配置对服务器的访问，只需指定服务器文档root的URL地址，指定项目根文件夹，然后指定要访问的URL地址。

指定服务器配置的名称，类型和可见性

  1. 通过执行以下操作之一打开“ 部署”页面：

    • 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），选择部署下建立，执行，部署 。

    • 选择工具|部署|从主菜单进行配置 。

  2. 在列出所有现有服务器配置的左侧窗格中，单击“ 添加”。 并从列表中选择就地 。

  3. 在打开的“ 创建新服务器”对话框中，键入要创建的服务器的名称，然后单击“ 确定” 。关闭“ 创建新服务器”对话框，然后返回到“ 部署”节点的“ 连接”选项卡。

  4. 单击用作默认值 在以下情况下，让PyCharm静默应用当前配置：

    • 自动上传更改的文件 。

    • 手动上传和下载文件，而无需选择目标主机。

    • 将本地文件和文件夹与其远程版本进行比较。

    • 编辑远程主机上的单个文件 。

  5. 仅在此项目中使用可见复选框配置服务器访问配置的可见性。

    • 选中复选框以将配置的使用限制为当前项目。这样的配置不能在当前项目外部重用。它不会出现在其他项目的可用配置列表中。

      服务器记录与项目一起存储在.idea目录中，从而允许通过VCS在团队成员之间共享它们。

      在左侧窗格的服务器访问配置列表中，仅在当前项目中可见的配置标记有图标。

    • 清除复选框后，该配置在所有PyCharm项目中均可见。它的设置（包括SFTP服务器的SSH凭据）可以在多个项目中重复使用。

指定服务器文档根目录的URL地址

  • 在“ Web服务器URL”字段中，键入与Web服务器配置文件中定义的Web服务器的文档根目录关联的URL地址。该URL地址将是构建应用程序URL地址的起点。

    HTTP和HTTPS协议均受支持。

例如，默认文档根目录设置为htdocs文件夹，用于访问其中数据的默认URL地址为http：// localhost 。如果更改了默认端口80 ，则必须明确指定端口： http：// localhost： 。

指定项目根文件夹和访问它的URL地址

  1. 切换到“ 映射”选项卡。

  2. 在本地路径字段中，指定项目根文件夹的完整路径。手动输入路径，或单击然后在打开的对话框中选择文件夹。

  3. 在“ Web路径”字段中，键入相对于服务器配置文件中指定的服务器文档根目录的项目根文件夹的路径。
