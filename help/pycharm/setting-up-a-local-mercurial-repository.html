设置本地Mercurial存储库

尽管Mercurial在安排数据和使用存储库方面提供了很高的灵活性，但是以下情形最常用于设置本地Mercurial存储库：

  • 克隆现有的远程存储库，并使用下载的数据创建一个新项目。

  • 创建本地存储库 ，如有必要，您可以稍后将其推送到远程位置。

克隆远程Mercurial存储库

  1. 在主菜单上，选择VCS |。从版本控制中检出|水星 。将打开“ 克隆Mercurial存储库”对话框。

  2. 在“ Mercurial存储库URL”文本框中，键入要克隆的远程存储库的URL。

  3. 单击“ Mercurial存储库URL”文本框旁边的“ 测试存储库”按钮，以检查是否可以成功建立到远程存储库的连接。

  4. 在“ 父目录”文本框中，指定要让PyCharm为本地Mercurial存储库创建文件夹的目录。使用浏览 按钮（如有必要）。

  5. 在目录名称文本框中，指定将存储库克隆到的新文件夹的名称。单击克隆 。

  6. 通过接受PyCharm显示的相应建议，基于克隆的数据创建一个新项目。

创建本地Mercurial存储库

  1. 打开要存储在存储库中的项目。

  2. 在主菜单上，选择VCS |。导入版本控制|创建Mercurial存储库 。将打开“ 创建Mercurial存储库”对话框。

  3. 指定新存储库的位置。
    • 要在项目根目录中创建存储库 ，请选择“ 为整个项目创建存储库”选项。PyCharm将在项目根文件夹中创建.hg目录。

      默认情况下选中此选项。

    • 要在另一个位置创建新的存储库，请选择“ 选择在何处创建存储库”选项，然后在下面的文本框中指定存储库位置的路径。手动输入路径或单击浏览 然后在打开的“ 选择hg init的目录”对话框中选择相关文件夹。

      Mercurial不支持外部路径。因此，如果选择另一个目录，请注意该目录必须包含项目根目录所在的树。

      如果您选择一个已经受Mercurial控制的目录，PyCharm将打开“目录位于hg”对话框，您可以在其中选择在指定位置创建一个存储库或保留在父存储库中。

  4. 将所需的文件置于Mercurial版本控制下。这些文件显示在“ 版本控制”工具窗口的“ 默认”节点下。

    请注意，如果在“ 版本控制”对话框中将Mercurial指定为目录的版本控制系统，PyCharm将建议将每个新文件放在该目录下的Mercurial控制之下。
