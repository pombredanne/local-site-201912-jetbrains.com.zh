PyCharm与Vagrant集成在一起，可帮助您创建由Vagrantfile配置文件定义的可复制开发环境。您可以创建和删除虚拟框 ， 初始化Vagrant框 ，通过从Vagrant配置文件中读取设置来配置远程Python解释器，以及执行其他Vagrant命令而无需离开IDE。执行vagrant up命令（多个Vagrant配置）时，您还可以在几个虚拟框中选择。

在PyCharm的上下文中，以下术语用于表示特定于Vagrant的概念：

  • 在当前文档中的PyCharm用户界面或流浪碱箱 A 流浪盒是一个盒子中的本机流浪术语。它表示一个纯图像，即一个框架，在此基础上可以自定义，配置和部署特定环境到您的计算机上。

  • 实例是虚拟机 。换句话说，它是个性化，配置和颠沛流离的底盒的基础上，并按照Vagrantfile部署你的机器上一个特定的环境。在其他类似的产品和文档中， 实例可以称为虚拟机 。

  • 实例文件夹是初始化后相关Vagrantfile存储在其中的文件夹，PyCharm将在该文件夹中寻找它。默认情况下，它是项目的根文件夹。

准备与Vagrant合作

在开始之前，请确保完成以下步骤和检查：

  1. 确保在计算机上安装了Vagrant和Oracle的VirtualBox ，并将其可执行文件的父文件夹添加到系统中PATH变量。如果您遵循标准安装过程，则会自动添加这些文件夹。

  2. 确保将以下可执行文件的父文件夹添加到系统PATH变量中：

    • vagrant.bat或Vagrant安装中的vagrant 。这应该由Vagrant安装程序自动完成。

    • Oracle的VirtualBox安装中的VBoxManage.exe或VBoxManage 。

在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），验证或修改Vagrant设置：

  1. 单击工具下的Vagrant 。将打开“ Vagrant”页面 。

  2. 指定Vagrant可执行文件。由于可执行文件的父文件夹已被添加到系统PATH变量中，因此仅键入可执行文件的名称。

  3. 在“ 实例文件夹”字段中，指定初始化和存储Vagrantfile的目录的标准路径。 Vagrantfile是一个配置文件，用于定义所需的实例 （ 虚拟机 ）。该文件包含虚拟IP地址，端口映射和要分配的内存大小。该文件可以指定要共享的文件夹以及应该安装的第三方软件。根据Vagrantfile，您的实例 （ 虚拟机 ）已配置，根据相关的Vagrant基本框进行了配置，并已部署在您的计算机上。Vagrantfile是通过以下方式创建的vagrant init命令。

    通过创建实例 （ 虚拟机 ）的调用时。 vagrant up命令或通过工具|无业游民向上菜单选项中，PyCharm在“ 实例文件夹”字段中指定的目录中查找Vagrantfile 。有关更多信息，请参见http://docs.vagrantup.com/v2/vagrantfile/ 。

    您可以在任何目录中创建Vagrantfile并将其指定为实例文件夹 。如果该字段为空，PyCharm会将项目根目录视为实例文件夹，并在其中查找Vagrantfile 。

  4. 在“ 无家可归的盒子”区域中，配置PyCharm 中可用的预定义“ 无家可归”基本盒子的列表。 每个项目都显示一个Vagrant基本框 ，Vagrant在该框上配置和启动其实例 （ 虚拟机 ）。该列表的条目与命令的输出相对应vagrant box list 。

    • 要下载新的基本框 ，请单击“ 添加”按钮 。在打开的对话框中，指定访问基本框的URL地址和在PyCharm中引用该基本框的名称。默认情况下，PyCharm会向lucid32框建议URL。

      该命令对应于vagrant box add 。结果，指定的基本框被下载到您的计算机上。

    • 要删除基本框 ，请在列表中选择它，然后单击“ 删除”按钮 。基本框和嵌套文件将从磁盘上物理删除。该命令对应于vagrant box remove 。

  5. PyCharm可以识别Vagrantfile语法。要更改默认的配色方案或代码样式语法 ，请切换到“设置/首选项” |“设置”。编辑器 ，然后在“ 配色方案”和“代码样式”对话框中调整相应的设置。

初始化Vagrantfile

Vagrantfile是一个配置文件，用于定义所需的实例 （ 虚拟机 ）。该文件包含虚拟IP地址，端口映射和要分配的内存大小。该文件可以指定要共享的文件夹以及应该安装的第三方软件。根据Vagrantfile，您的实例 （ 虚拟机 ）已配置，根据相关的Vagrant基本框进行了配置，并已部署在您的计算机上。Vagrantfile是通过以下方式创建的vagrant init命令。

您可以在任何文件夹中初始化Vagrantfile ，只需记住应在“设置 首选项”对话框的“ Vagrant”页面上将此文件夹指定为实例文件夹 。否则，PyCharm在实例 （ 虚拟机 ）创建期间将无法找到相关的Vagrantfile 。

要初始化Vagrantfile ，请执行以下一项操作：

初始化Vagrantfile

  1. 要在项目根目录中创建Vagrantfile ，请选择“ 工具” |“工具”。无业游民在主菜单上启动Project Root ，然后从打开的列表中选择目标项目根。输出init命令显示在“ 运行工具”窗口中。

  2. 要在特定文件夹中创建Vagrantfile ， 打开嵌入式终端 （ Alt+F12 ），然后键入以下命令：

    cd vagrant init

创建Vagrantfile之后，您可以在编辑器中对其进行预览：它包含可用配置选项的详细列表。如果需要，请更改或添加配置设置。请注意，已为Vagrantfile启用代码完成功能。

一旦Vagrantfile初始化成功完成，您就可以导入基础框 ，根据Vagrantfile进行配置和部署，从而创建您自己的实例 （ 虚拟机 ）。

创建和启动实例（虚拟机）

创建实例 （ 虚拟机 ）意味着根据位于实例文件夹中的Vagrantfile导入和设置基础框 。

创建一个实例，请按照下列步骤操作：

  1. 请执行以下任一操作：

    • 要在项目根目录中创建实例，请转至工具|无业游民上 。

    • 要在特定文件夹中创建实例， 打开嵌入式终端 （ Alt+F12 ），然后键入以下命令：

      cd vagrant up

  2. 从建议列表中选择所需的虚拟机配置：

停止，挂起，恢复，重新加载和销毁实例（虚拟机）

重新加载实例

  • 如果您对Vagrantfile进行了一些更改，并希望根据它们进行更新，请选择“ 工具” |“虚拟机” 。无业游民在主菜单上重新加载或在嵌入式终端中运行以下命令：

    vagrant reload
    有关详细信息，请参见http://docs.vagrantup.com/v2/cli/reload.html 。

挂起实例

  • 要暂时在正在运行的虚拟机（ 来宾机 ）上停止操作系统并保存当前环境的确切状态，以便可以从此时开始完全恢复该环境，请选择“ 工具” |“虚拟机” 。无业游民挂在主菜单上或在嵌入式终端中运行以下命令：

    vagrant suspend
    有关详细信息，请参见http://docs.vagrantup.com/v2/cli/suspend.html 。

恢复实例

  • 要在虚拟机（ 来宾计算机 ）上恢复以前挂起的操作系统，并使其从挂起时刻保存的状态运行，请选择“ 工具” |“其他”。无业游民在主菜单上继续操作 ，或在嵌入式终端中运行以下命令：

    vagrant resume
    有关详细信息，请参见http://docs.vagrantup.com/v2/cli/resume.html 。

关闭实例

  • 要关闭虚拟机（ 来宾计算机 ）上的操作系统，请选择“ 工具” |“其他”。无业游民暂停主菜单或在嵌入式终端中运行以下命令：

    vagrant halt
    有关详细信息，请参见http://docs.vagrantup.com/v2/cli/halt.html 。

  • 要关闭虚拟机（ 来宾机 ）上的操作系统，请停止虚拟机本身，并删除在创建过程中为其配置的资源vagrant up ，选择工具|无业游民在主菜单上销毁或在嵌入式终端中运行以下命令：

    vagrant destroy
    有关详细信息，请参见http://docs.vagrantup.com/v2/cli/destroy.html 。
