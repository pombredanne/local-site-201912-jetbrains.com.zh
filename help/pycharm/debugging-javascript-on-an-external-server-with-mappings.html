什么是远程Web服务器？

在PyCharm中， 文档根目录在当前项目之外的任何服务器都称为remote 。该服务器实际上可能在物理上远程主机上或您的计算机上运行。

例如，如果您的项目位于C：/ PyCharmProjects / MyProject中 ，并且Web服务器文档根目录为C：/ XAMPP / htdocs ，则对于PyCharm而言，此Web服务器是remote 。

如何在服务器上的应用程序源与其PyCharm项目中的本地副本同步？

要在远程 Web服务器上调试应用程序，您需要在PyCharm项目中拥有其源代码的副本。要同步本地和远程源，请按照创建远程服务器配置和配置与Web服务器的同步中所述创建 部署配置 。

在远程服务器上调试应用程序

  1. 根据需要在JavaScript代码中设置断点 。

  2. 创建类型为JavaScript Debug的调试配置：

    从主菜单中，选择运行|编辑配置 ，然后在“ 编辑配置”对话框中， 点击在工具栏上，然后从列表中选择JavaScript Debug 。

  3. 在打开的“运行/调试配置：JavaScript调试”对话框中，指定应用程序运行所在的URL地址。根据服务器访问配置 ，此URL地址应该是Web服务器根URL和HTML文件相对于Web服务器文档根目录的路径的串联，请参阅配置与Web服务器同步 。

    单击确定以保存配置设置。

  4. 在工具栏上的“ 选择运行/调试配置”列表中选择新创建的配置，然后单击“ 调试”。 。运行配置中指定的HTML文件将在所选浏览器中打开，并显示“ 调试工具”窗口 。

  5. 在“ 调试工具”窗口中，照常进行操作： 单步执行程序 ， 停止并继续执行程序， 在挂起时检查它 ， 查看实际的HTML DOM等

默认情况下，调试会话会在带有自定义Chrome用户数据的新窗口中启动。要以熟悉的外观打开新的Chrome实例，请在PyCharm中将Chrome配置为以用户数据开头，有关详细信息，请参见使用默认的Chrome用户数据启动调试会话 。

配置映射

映射设置Web服务器上的文件与其本地副本之间的对应关系。您需要映射：

  • 在远程 Web服务器上部署和运行应用程序时。

  • 当您定义了除项目根目录之外的多个资源根目录文件夹。

在大多数情况下，PyCharm通过重用部署配置中的映射来自动设置路径映射。如果您的应用程序结构复杂，则需要其他手动配置。

要配置映射：

  1. 如调试应用程序中所述，创建JavaScript Debug类型的调试配置。

  2. 在“ 本地文件的远程URL”区域中，根据当前使用的部署配置将文件和文件夹映射到服务器上文件和文件夹的URL地址，请参阅将本地文件夹映射到服务器上的文件夹和URL地址以进行访问 。
