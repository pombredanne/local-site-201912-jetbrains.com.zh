RubyMine 2019.3帮助

配置Ruby解释器

RubyMine使您可以为当前项目或新创建的项目选择所需的Ruby解释器。IDE自动检测安装在本地计算机上并由版本管理器维护的解释器 。此外，您可以添加其他本地或远程解释器（Docker，Vagrant，SSH等）。

支持以下解释器：

  • Ruby MRI（1.8.7版和更高版本）

  • 红宝石

  • Rubinius（无需调试 ）

  • TruffleRuby（无需调试 ）

选择Ruby解释器

要为当前项目配置Ruby SDK，请执行以下步骤：

  1. 调用设置/首选项对话框Ctrl+Alt+S并转到Ruby SDK和Gems页面。

  2. （可选）如果RubyMine打开了多个项目 ，请选择所需的一个以定义Ruby解释器。

  3. 选择所需的Ruby解释器。您还可以为RVM和rbenv 版本管理器选择gemset。

  4. 对于选定的Ruby解释器/ gemset，您可以在右侧看到已安装的gem。在“ 使用Bundler安装宝石”中了解有关Ruby宝石支持的更多信息。

如果使用RVM或rbenv来管理本地Ruby SDK，则可以通过以下方式快速设置所需的解释器：

  1. 按Ctrl两次。

  2. 在调用的弹出窗口中，开始输入rvm use要么rbenv shell ，选择所需的口译员，然后按Enter 。

  3. （可选）如有必要，请在弹出窗口中单击“ 回滚”按钮，以通知已更改的SDK。

添加本地Ruby解释器

RubyMine自动检测使用版本管理器安装的本地解释器 。要手动添加所需的本地解释器，请执行以下步骤：

  1. 在Ruby SDK和Gems页面中，点击按钮，然后选择新建本地… ：

  2. 提供Ruby可执行文件的路径（例如，对于使用Homebrew安装到macOS的Ruby，为/ usr / local / bin / ruby ）。RubyMine将显示选定的解释器以及自动检测到的解释器。

  3. 要从列表中删除解释器，请选择它，然后单击按钮。
