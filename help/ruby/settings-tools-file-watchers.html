启用File Watchers插件后，该页面可用。有关配置文件监视程序的更多信息，请参见文件监视程序 。 的插件未与RubyMine捆绑在一起，但可以如管理插件中所述从JetBrains插件存储库安装。

File Watcher是RubyMine系统，它跟踪对文件的更改并运行第三方独立应用程序。RubyMine为许多此类标准流行的第三方工具（ 编译器 ，压缩器，修饰词等）提供了预定义的File Watcher模板。您还可以配置自定义File Watcher以运行任何其他第三方工具。

文件监视程序具有两个专用的代码检查 ：

  • 适用于预定义File Watcher的每个文件都会运行File watcher可用检查。如果项目没有配置相关的File Watcher，则RubyMine建议添加一个。

  • 正在运行的File Watcher调用File Watcher 问题检查，并突出显示特定于它的错误。

该页面由可用文件监视程序列表和管理此列表的工具栏组成。

对于每个文件监视程序， 您可以决定是否仅在当前项目（从“ 级别”列表中选择“ 项目 ”）或所有项目（选择“ 全局” ）中可用。

要激活File Watcher ，请选中它旁边的复选框。 启用文件监视程序后，一旦选定类型的文件以及选定范围内的文件被更改或保存，它将自动启动，请参阅配置高级选项 。 如果在运行文件监视程序时发生错误，则会自动禁用文件监视程序。要恢复状态，请手动启用File Watcher。

和捷径

描述

加
Alt+Insert

单击此按钮打开“ 选择模板”弹出窗口，然后选择“ 文件监视程序”的相关类型。之后，RubyMine打开“ 新建监视程序”对话框，根据当前项目的设置来自定义预定义的文件监视程序 。

编辑
Enter

单击此按钮可在“ 编辑监视程序”对话框中更新所选文件监视程序的设置。此更新仅适用于当前项目File Watcher ，不会影响预定义的RubyMine级别的模板。

去掉
Alt+Delete

单击此按钮删除选定的File Watcher 。File Watcher不再应用于当前项目中的文件。请注意，此操作不会影响相应的预定义模板，该模板在RubyMine级别仍然可用。

 

上Ctrl+Alt+Up
下Ctrl+Alt+Down

使用这些按钮可以更改列表中File Watcher的顺序。如果启用了多个功能，这将确定启动File Watchers的顺序。

复制

使用此按钮创建所选文件查看器的副本。

进口

单击此按钮导入现有的文件监视程序，并将其添加到可用文件监视程序列表中。

出口

单击此按钮可将所选观察者导出到位于用户主目录下的watchers.xml文件。
