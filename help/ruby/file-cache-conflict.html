文件缓存冲突

外部进程更改了文件，并在RubyMine中打开并未保存文件，这导致文件的两个版本相互冲突。使用以下选项解决此冲突。

项目

描述

加载FS更改

单击此按钮以加载在RubyMine之外生成的文件版本，并覆盖您的本地更改。

保持记忆变化

单击此按钮以保留由RubyMine生成并存储在缓存中的版本。

显示差异

单击此按钮以调用差异查看器 ，该差异查看器在左侧显示文件系统中的版本，在右侧显示RubyMine版本。您可以根据需要合并差异，然后将所需的版本加载到RubyMine。默认情况下，保存缓存版本。
