RubyMine 2019.3帮助

调试

RubyMine提供了用于调试源代码的全套工具：

  • Ruby中的断点（包括ERB和HAML文件）。

  • JavaScript中的断点。

  • 多个同时调试会话。

  • 可定制的断点属性：条件，通过次数等。

  • 调试器用户界面中的框架 ， 变量和监视视图。

  • 表达式的运行时评估 。

本节介绍了各种类型的应用程序通用的过程。

先决条件

RubyMine需要安装在项目SDK中的debase和ruby-debug-ide gem才能调试Ruby 2.0+项目。如果未安装这些gem，则RubyMine会在您首次运行调试时建议这样做。

请注意，较早的Ruby版本（例如Ruby 1.9）或JRuby可能需要其他调试工具，并提供有限的调试功能。

开始调试会话

  1. 为要调试的应用程序定义运行/调试配置 。

  2. 创建断点 。

  3. 点击调试纽扣 。从启动调试器会话中了解更多信息。

根据需要暂停或恢复会话。

在调试会话期间，您可以评估表达式 ，动态更改值， 检查挂起的程序并设置watch 。

断点

自定义断点设置

要自定义断点设置，请按Ctrl+Shift+F8 。要查看项目中的所有断点（带有其他设置），请单击“ 更多”或按相同的快捷方式Ctrl+Shift+F8再次。

不要暂停代码执行

使用动作断点可以在代码的特定行评估变量，而无需暂停代码执行。要创建动作断点，请在按住的同时单击装订线Shift 。

创建一个临时断点

要创建仅停止一次的断点，请在按住的同时单击左侧装订线Shift+Alt 。有关更多信息，请参考“ 断点 ”部分。

禁用断点

要禁用断点，请在按住的同时单击断点Alt 。有关详细信息，请参见“ 断点 ”部分。

调试器会话

运行到光标

  1. 创建一个断点。

  2. 运行调试会话。要运行调试会话，请单击“ 运行应用程序”图标（ ）在装订线区域，然后选择调试 。

要停止在光标位置执行代码而不添加其他断点，请单击“ 运行到光标”图标（ ）或按Alt+F9 。或者，您可以单击装订线区域中的行号。

图标在“调试工具”窗口的工具栏参考中进行了介绍。

评估表达

在调试模式下，你可以计算表达式按Alt+F8 。

“ 评估”对话框支持代码完成。

若要查看各种函数调用的行为在调试会话期间如何变化，请设置自定义监视。您可以在“ 使用手表”中阅读有关手表的更多信息。

有关详细信息，请参阅评估表达式部分。

配置调试器设置

要更改调试器设置，请单击“ 文件” |“更改” 。设置 。在“ 设置”菜单中，单击“ 构建，执行，部署” |“部署”。调试器 。

远程调试

关于RubyMine调试，您绝对应该知道的最后一件事是远程调试。远程调试意味着将调试器附加到已经在您的主机或任何其他主机上的特定端口上运行的进程。这样，您可以将调试器附加到独立运行的应用程序服务器上。

要创建远程配置， 请转到编辑配置，然后添加远程运行配置 。在运行此配置之前，请确保指定正确的主机和端口。
