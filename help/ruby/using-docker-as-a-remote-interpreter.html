使用Docker作为远程解释器

RubyMine提供了各种功能来处理在您的计算机上运行的Docker。您可以在Docker主题中了解更多信息。

本教程描述了如何将Docker用作示例Rails应用程序的远程解释器。我们将在单个Docker容器中设置并运行该应用程序。

安装Docker并打开一个示例应用程序

在本教程中，我们将结合使用Mac和MacOS，并安装RubyMine。此外，必须满足以下先决条件才能完成所有步骤：

  • 确认Docker已安装并正在运行。

  • 确保已启用 Docker插件。

  • 在RubyMine中打开以下Rails应用程序：

    https://github.com/JetBrains/sample_rails_app.git

构建一个Docker镜像

首先，我们需要为我们的应用程序构建一个Docker映像。为此，请打开Dockerfile 。按Ctrl+Shift+N ，输入Dockerfile并按Enter 。在这里，您可以查看有关基于ruby图像对我们的应用进行docker化的说明。

执行以下步骤：

  1. 请点击在装订线中，然后从菜单中选择“ 新建运行配置 ”。

  2. 在“ 编辑运行配置”对话框中，指定以下设置：

    • 在图像标签字段中指定图像名称和标签。

    • 禁用“ 运行内置映像”选项，因为我们将在将其分配为远程解释器之后再运行。

    • 在命令预览中预览生成的Docker命令。

    单击运行以开始构建映像。

  3. RubyMine在Docker工具窗口中显示构建映像的过程。

    等待图像建立并显示在“ 图像”组中。

将Docker配置为远程解释器

现在，我们可以使用构建的映像为我们的应用程序配置远程解释器，因为该映像已安装了Ruby解释器和必需的gem。要将Docker配置为远程解释，请执行以下操作：

  1. 打开设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| Ruby SDK和Gems页面。

  2. 请点击并在下拉菜单中选择新建遥控器...。

  3. 在调用的对话框中，选择Docker并指定以下选项：

    • 服务器 -此选项指定用于运行容器的Docker服务器。

    • 图像名称 -选择在上一章中创建的图像。

    • Ruby或版本管理器路径 -保留默认的ruby值以自动检测到Ruby解释器的路径。您也可以手动指定解释器或版本管理器可执行文件的路径。

    单击确定 。

  4. 在Ruby SDK and Gems页面中选择添加的SDK，然后单击OK 。

  5. 等待RubyMine完成索引过程并创建辅助Docker映像和容器。

运行应用程序

现在，我们可以迁移数据库并运行我们的Rails应用程序：

  1. 迁移数据库。要在RubyMine中执行此操作，请按Ctrl两次并输入db：migrate 。选择rake db:migrate在下拉菜单中，然后按Enter 。将默认设置保留在调用的“ 执行db：migrate”对话框中，然后单击“ 确定” 。

    如果看不到rake db:migrate在下拉列表中，尝试重新加载Rake任务： 重新加载Rake任务 。

  2. 启动应用程序。按Ctrl两次，然后开始输入开发 。从列表中选择Development：sample_rails_app_docker配置，然后按Enter 。RubyMine将在运行工具窗口中显示准备运行应用程序的过程。

  3. 打开浏览器，输入0.0.0.0:3000地址，然后按Enter查看我们的工作应用程序。
