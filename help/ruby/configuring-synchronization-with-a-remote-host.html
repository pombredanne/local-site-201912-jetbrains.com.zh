配置与Web服务器的同步

RubyMine区分就地服务器， 本地服务器和远程服务器，但是这些术语在RubyMine上下文中的含义与它们的通用含义略有不同：

  • 就地服务器是这样的服务器，其文档根目录是项目根目录的父目录，无论是直接目录还是非直接目录。换句话说，Web服务器正在您的计算机上运行，您的项目位于其文档根目录下（例如，在/ htdocs文件夹中），并且您可以直接在服务器上进行开发。

  • 本地服务器是在本地或已安装文件夹中运行的服务器，其文档根目录 不是项目根目录的父目录。您进行开发，然后将源复制到服务器。

  • 远程服务器是在另一台计算机（远程主机）上运行的服务器。可通过FTP / SFTP / FTPS协议访问服务器上的文件。

在你开始前

通过FTP / SFTP Connectivity捆绑插件提供与服务器同步，上载，下载和管理文件的功能，该插件默认情况下处于启用状态。如果插件被禁用，请在“ 设置”对话框的“ 插件”页面中将其激活。有关详细信息，请参阅管理插件 。

服务器访问配置

RubyMine通过服务器访问配置来控制与Web服务器的交互。无论您是在远程主机上还是在计算机上，无论何时使用服务器，都需要定义服务器访问配置。

服务器访问配置定义以下内容：

  • 服务器类型（ 就地 ， 本地或远程 ）。

  • 运行服务器的计算机（主机）。对于就地和本地服务器，RubyMine假定它是项目所在的当前计算机。

  • 服务器访问配置根目录 ：服务器层次结构中可以通过服务器配置访问的最高文件夹。

  • 访问服务器配置根目录的URL地址。

  • 传输数据所通过的协议和连接参数。

  • 本地文件夹（即项目根目录下的文件夹），服务器上的目标文件夹（本地或远程）以及用于访问服务器上数据的URL地址之间的对应关系。这种对应称为映射 。

    例如，您的项目存储在C：/ Projects / My_Project /下，并且包含文件夹C：/ Projects / My_Project / My_Folder ，而本地服务器的文档根目录位于C：/ xampp / htdocs下 。您将整个项目树上传到C：/ xampp / htdocs / My_Project 。用RubyMine术语来说， C：/ Projects / My_Project / My_Folder文件夹称为本地 ，而C：/ xampp / htdocs / My_Project / My_Folder文件夹称为remote 。

您可以根据需要定义任意数量的配置，从而可以在上传/下载设置之间灵活切换。

如果需要将代码部署到多个服务器，则可以创建一个服务器组，并避免分别部署到每个服务器。

您可以将服务器配置设置为默认配置，以便在以下情况下让RubyMine静默选择它：

  • 自动上传更改的文件 。

  • 手动上传和下载文件，而无需选择目标主机。

  • 将本地文件和文件夹与其远程版本进行比较。

  • 编辑远程主机上的单个文件 。
