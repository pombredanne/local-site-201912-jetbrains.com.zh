RubyMine中的Rails / Spring支持

Spring是一个Rails应用程序预加载器，它通过使应用程序在后台运行来加快开发速度。这意味着您在进行更改时不需要重新启动服务器。

在RubyMine中，Spring可用于运行Rails生成器 ， tests和Rake任务。确保将spring gem添加到Gemfile并安装到项目的SDK。

请注意，默认情况下，不使用Spring即可运行Test :: Unit / Shoulda / Minitest测试。要了解如何启用Spring来运行测试，请参阅使用Spring来运行测试 。

使用Spring进行测试

RubyMine允许您指定是否使用Spring预加载器针对指定的运行/调试配置运行测试。在下面的示例中，我们将演示如何在sample_rails_app项目的test目录中运行所有Minitest测试。为此，您可以使用以下工作流程：

  1. 右键单击项目视图中的测试目录，然后选择“运行'测试中的所有测试：...' 。执行测试后，将创建相应的运行/调试配置（ test中的所有测试：sample_rails_app ）。

  2. 在导航栏中，展开运行/调试配置的列表，然后选择编辑配置 。

  3. 在调用的对话框中，确保在左侧列表中选择“ 测试中的所有测试：sample_rails_app配置”。然后，将“ 使用预加载服务器”选项设置为Spring 。

    单击确定 。现在将使用Spring预加载器通过此运行/调试配置运行测试。

使用Spring进行调试

RubyMine允许您使用Spring 调试 Rails应用程序。这可能需要更新系统或项目Spring配置文件，以将调试器加载到Spring 派生的每个进程中。当您开始调试时，RubyMine可能会显示以下选项对话框：

  • 系统配置（〜/ .spring.rb）

    单击以更新系统Spring config（ 〜/ .spring.rb文件）并开始调试。

  • 项目配置（config / spring.rb）

    单击以更新项目Spring config（ config / spring.rb文件）并开始调试。

  • 不要更改，禁用Spring

    单击以禁用用于调试应用程序的Spring。

禁用弹簧

如果需要，可以为当前项目禁用Spring。去做这个：

  1. 从主菜单中，选择帮助|寻找行动 Ctrl+Shift+A 。

  2. 开始输入Spring并禁用Spring pre-loader选项。

停止Spring服务器

当您在RubyMine中启动Rails生成器，测试或Rake任务时，它将使用自定义SPRING_TMP_PATH 环境变量来启动Spring服务器的单独实例。该变量的值等于{temp_dir} / RMSpring 。这意味着您需要在Spring服务器命令之前添加SPRING_TMP_PATH={temp_dir}/RMSpring 。例如，如果您在macOS上遇到Spring的一些问题并需要将其停止，请运行以下命令：

SPRING_TMP_PATH = $ {TMPDIR} / RM弹簧仓/弹簧停止
