响应处理示例

在本主题中，我们将研究几个HTTP响应处理示例。要自己尝试示例，请浏览auth-requests和test-responses 请求集合 。

检查响应标头，正文和内容类型

在此示例中，我们将创建一些测试以验证以下内容：

  • 该请求已成功执行，即响应状态为200 。

  • 标头在响应正文中接收。

  • 响应的内容类型为application / json 。

要创建测试，我们将test 客户端对象的方法。在测试中，我们可以通过调用assert的方法client对象并引用响应对象的各种属性以对其进行验证。

###检查响应状态，标头和内容类型GET https://httpbin.org/get> {％client.test（“请求成功执行”，function（）{client.assert（response.status === 200，“响应状态不是200”）;}）; client.test（“ Headers选项存在”，function（）{client.assert（response.body.hasOwnProperty（“ headers”），“在响应中找不到'headers'选项”）;}）; client.test（“响应内容类型为json”，function（）{var type = response.contentType.mimeType; client.assert（type ===“ application / json”，“预期为'application / json'，但收到了' “ +类型+”'“）;}）; ％}

使用环境变量

在此示例中，我们将从接收到的响应中捕获一个值到一个环境变量中，然后将其用于后续请求中。

第一个请求涉及一个响应处理程序脚本，该脚本将身份验证令牌从接收到的响应主体保存到auth_token client.global下的变量 。为此，我们使用client.global.set(VariableName, VariableValue)施工。

POST https://httpbin.org/post内容类型：application / json {“ token”：“ my-secret-token”} //保存变量> {％client.global.set（“ auth_token”，响应。 body.json.token）; ％}

由于httpbin POST实现 ，我们使用response.body.json.token检索令牌。根据您与之交互的实际Web服务，响应结构可能会有所不同，并且您可能需要使用其他结构，例如， response.body.token 。

执行请求后， auth_token可以从两个后续请求中访问变量{{auth_token}}和响应处理程序脚本client.global.get("auth_token")施工。

//访问变量GET https://httpbin.org/headers授权：Bearer {{auth_token}}

要从响应标头获取值，请使用valueOf 标头对象的方法。如果收到多个具有相同名称的标头（这是Set-Cookie标头的常见情况），请使用valuesOf方法代替。这将返回所有响应标头值的数组。

POST https://httpbin.org/cookies //从第一个Set-Cookie标头保存一个cookie> {％client.global.set（“ my_cookie”，response.headers.valuesOf（“ Set-Cookie”）[0] ）; ％}
