运行/调试配置：NPM

运行编辑配置| | NPM

在此对话框中，创建用于在本地运行npm和Yarn脚本的配置。在当前上下文中，“本地”表示RubyMine本身会启动计算机上安装的Node.js ，从而启动脚本执行。

在你开始前

  1. 下载并安装Node.js。请注意，还安装了npm，因此，如果要使用它，请按照初步步骤操作。

  2. 要使用纱线，请按照纱线官方网站上的说明进行安装。

配置选项卡

项目

描述

package.json

在此字段中，指定要从中运行脚本的package.json文件。从下拉列表中选择文件，该列表显示当前项目中检测到的所有package.json文件，或单击然后在打开的对话框中选择所需的package.json 。

命令

从此列表中，默认选择要执行的npm CLI命令 run-script被选中。要了解更多信息，请参见npm文档的CLI命令部分。

剧本

从此下拉列表中，选择将应用所选命令的脚本。该列表包含在scripts package.json文件中的属性。

争论

在此字段中，指定用于执行脚本的命令行参数。从npm官方网站了解更多信息。

节点解释器

在这个领域里， 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

节点选项

在此文本框中，键入要传递到Node.js可执行文件的特定于Node.js的命令行选项。可接受的选项是：

  • 使用--require coffee-script/register在运行期间将CoffeeScript文件即时编译为JavaScript。

    这种模式要求3软件包的一部分register.js文件应该位于项目内部。因此，您需要在本地的Node.js页面上安装coffee-script软件包，如npm和Yarn中所述 。

  • 使用--inspect要么--inspect-brk您将Node.js v7用于Chrome调试协议支持时的参数。否则，默认情况下，调试过程将使用V8调试协议 。

包装经理

在这个领域里。 指定要使用的包管理器。如果您选择Project别名，RubyMine将使用Node.js页面中的默认项目包管理器。您还可以选择相关的软件包别名（ npm或yarn ）或为软件包管理器的自定义安装指定显式路径。

环境变量

 在此字段中，为Node.js可执行文件指定环境变量 （如果适用）。 点击浏览 在字段右侧，并在“ 环境变量”对话框中配置变量列表，该对话框将打开：

  • 要定义新变量，请单击并指定变量名称和值。

  • 要放弃变量定义，请在列表中选择它，然后单击 。

  • 准备好后，单击确定 。

变量的定义显示在“ 环境变量”只读字段中，以分号作为分隔符。可接受的变量是：

  • NODE_PATH ： 一种: -前缀为模块搜索路径的目录的列表。

  • NODE_MODULE_CONTEXTS ：设置为1可以在各自的全局上下文中加载模块。

  • NODE_DISABLE_COLORS ：设置为1以禁用REPL中的颜色。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

允许并行运行

如果选中此复选框，则可以并行启动运行配置多次，而不必重新运行它。每个运行程序都将在“ 运行工具窗口”或“ 调试工具窗口 ”的自己的选项卡中启动 。

禁用后，每次启动运行/调试配置的新实例时，RubyMine都会检查同一运行/调试配置的其他实例是否存在，并显示确认对话框。如果在确认对话框中单击“ 确定” ，则将停止运行器的第一个实例，而将其替换。当启动两个相同类型的运行/调试配置实例时，这会占用过多的CPU和内存资源。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在RubyMine中定义，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 运行文件监视程序 ：选择此选项可使RubyMine应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

Alt+Delete

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
