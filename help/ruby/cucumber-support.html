RubyMine 2019.3帮助

黄瓜

Cucumber是用于行为驱动的Ruby开发的测试框架。黄瓜方案是使用Gherkin语法编写的，并存储在.feature文件中。每个场景都有多个步骤，这些步骤链接到代表Ruby块的步骤定义。

RubyMine与Cucumber集成，并允许您运行测试，创建步骤定义，在功能和步骤定义之间导航等等。在本主题中，我们将介绍主要的IDE功能，这些功能可帮助您使用Cucumber进行Ruby开发。要了解RubyMine如何帮助您使用Cucumber进行JavaScript开发，请参阅Cucumber.js 。

从以下主题学习黄瓜基础知识：

  • 开始使用
  • Ruby工具

先决条件

在RubyMine中使用Cucumber之前，请确保：

  • Gherkin存储库插件已安装并启用。有关更多信息，请参阅管理插件 。

  • 根据项目类型（Ruby或Rails），将“ cucumber”或“ cucumber-rails” gem添加到Gemfile中并安装到项目的SDK中。从“ 使用捆绑程序安装宝石”主题中学习如何使用RubyMine 安装宝石 。

在Rails应用程序中启用Cucumber支持

  1. 将“ cucumber-rails” gem添加到test组的Gemfile组并安装它。

  2. 在主菜单中，选择工具|运行Rails Generator Ctrl+Alt+G并开始输入Cucumber：install 。从列表中选择相应的命令，然后按N/A 。

  3. 在Setup Cucumber Support对话框中，选择Cucumber的测试框架，指定所需的选项，然后单击OK 。

    RubyMine将运行cucumber:install生成器，可在Rails项目中设置Cucumber并在features目录中生成必要的文件。运行此生成器后，您还将获得一个名为Rake的新任务。 cucumber可用于在RubyMine中运行Cucumber测试 。

创建功能文件

  1. 在“ 项目”工具窗口中，右键单击应在其中创建特征文件的目录。

  2. 从此目录的上下文菜单中，选择“ 新建” |“新建”。文件 Alt+Insert ，并创建一个扩展名为.feature的文件。

  3. 在创建的功能文件中，键入您的方案。例如，我们将使用Cucumber入门指南中的代码片段：

    功能：现在是星期五吗？每个人都想知道何时是星期五方案：星期天不是星期五今天是星期天当我问是否是星期五时，然后应该告诉我“不”

    由于没有步骤定义，因此这些步骤将突出显示为未解决。请参阅下一章以了解如何创建步骤定义。

创建步骤定义

在编辑功能文件时，RubyMine会检测并突出显示缺少定义的步骤。您可以通过以下方式生成缺少的步骤定义：

  1. 将插入符号放在没有定义的步骤中，按Alt+Enter ，然后选择创建步骤定义或创建所有步骤定义 。

  2. 选择是使用现有步骤定义文件还是创建一个新的步骤定义文件。

    按Enter 。

  3. （可选）如果选择创建新文件 ，请指定新文件的名称和用于编写定义的语言。

    单击确定 。

  4. 在打开的Ruby文件中，编写步骤定义代码。

运行测试

RubyMine允许您运行单个Cucumber方案或指定文件夹或文件中的所有方案。您还可以使用Rake任务运行方案。

从文件夹运行多个测试

  • 在“ 项目”视图中，右键单击所需的文件夹，然后选择“在...中运行所有功能”。' 。

    要运行特定的测试子集，例如，仅匹配特定名称或标记有特定标记的方案，请自定义Cucumber运行/调试配置 。

从编辑器运行测试

要从编辑器运行测试，请执行以下任一操作：

  • 打开.feature文件，然后单击所需功能或方案旁边的左侧装订线中的“运行”按钮。然后，从上下文菜单中选择所需的运行命令。

  • 将插入符号置于所需的功能或场景中，按Alt+Enter ，选择所需的运行动作，然后按Enter 。

使用Rake任务运行测试

  • 按Ctrl两次调用“ 运行任何东西”弹出窗口。开始键入所需的任务名称，从列表中选择它，然后按Enter 。

执行测试后，您可以在“ 运行工具”窗口中分析结果 。

黄瓜运行/调试配置

使用上下文菜单或从编辑器运行Cucumber测试时，RubyMine会自动创建一个相应的Cucumber 临时配置 ，可以将其保存。如有必要，可以从预定义的模板手动创建Cucumber运行/调试配置。

要自定义和运行创建的配置，请执行以下操作：

  1. 按Ctrl+Shift+A然后开始输入编辑配置 。选择编辑配置 ，然后按Enter 。

  2. 在打开的“运行/调试配置”对话框中，选择所需的配置，然后指定其设置。

    例如，您可以指定以下设置：

    描述

    模式

    功能文件夹/功能文件

    指定包含所需功能的文件夹/文件的完整路径。

    元素名称过滤器

    仅允许您运行与特定名称匹配的方案。这可以是功能，方案，方案大纲或示例块的名称的子字符串。在Cucumber帮助中了解更多信息。

    标签过滤器

    指定标签以运行功能的子集。进一步了解Tag逻辑 。

    跑步者选项

    指定运行黄瓜测试所需的选项。例如， -r features参数在运行测试之前从功能文件夹中加载文件。

    黄瓜宝石

    使用自定义黄瓜赛跑者脚本

    如果要使用备用Cucumber运行器脚本，请启用此选项。您可以在文本字段中输入CucumberRunner脚本的完整路径，或单击 ，然后在打开的对话框中选择所需的运行程序脚本。

    单击确定保存运行/调试配置。

  3. 要运行保存的配置，请按Ctrl两次，开始输入配置名称，从列表中选择它，然后按Enter 。

方案概述和示例

的Scenario Outline可用于以不同的值组合多次运行相同的方案。这些值存储在Examples表。您可以从黄瓜帮助中了解更多信息。

RubyMine允许您将方案转换为轮廓并产生缺失Examples表。

将方案转换为轮廓

  1. 将插入符置于所需的场景，然后按Alt+Enter 。

  2. 选择将方案转换为轮廓并按Enter 。

    RubyMine将改变Scenario至Scenario Outline并添加Examples表。

在方案大纲中创建示例

RubyMine提供检查以检测场景大纲中缺少的示例。要创建缺少的示例，请执行以下操作：

  1. 将插入符放在突出显示的方案轮廓上，然后按Alt+Enter 。

  2. 选择创建示例部分 ，然后按Enter 。

    RubyMine将创建Examples带有标题行的表。

导航

  • 要从.feature文件中的步骤跳至步骤定义，请按住Ctrl ，将鼠标悬停在该步骤上，然后在该步骤变成链接时单击它。

配置语法突出显示

您可以通过以下方式配置可识别黄瓜的语法突出显示：

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|配色方案|黄瓜

  2. 选择配色方案，接受从默认设置继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。
