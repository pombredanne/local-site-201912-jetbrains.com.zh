RubyMine代码编辑器中的HTTP客户端

测试Web服务时，可以直接在RubyMine 代码编辑器中创建，编辑和执行HTTP请求。

HTTP请求存储在.http和.rest文件中，并标有图标。

对HTTP文件的支持包括以下功能：

  • 代码突出显示

  • 主机，方法类型和标题字段的代码完成

  • 请求，其部分和响应处理程序脚本的代码折叠

  • 内联文档，用于请求标头字段和doc标签

  • 查看 HTTP请求的结构

  • 请求消息正文中的语言注入

  • 实时模板

在开始之前，请在“ HTTP代理”页面上配置代理设置 。 的 设置/首选项对话框Ctrl+Alt+S如有必要。

通过捆绑的HTTP客户端插件支持测试Web服务 ，默认情况下，该插件在“设置/首选项” |“默认设置”中启用。插件 。

创建HTTP请求文件

您可以从临时文件或HTTP请求类型的物理文件处理HTTP请求 。每个文件可以包含多个请求，并且您可以根据需要创建任意数量的文件。

临时文件可用于在开发过程中测试HTTP请求。临时文件未存储在项目中，因此RubyMine可以修改它们并添加有关请求的其他信息。当从头文件执行HTTP请求时，指向响应输出文件的链接将添加到请求下方和请求历史记录文件顶部。

创建一个HTTP请求暂存文件

  • 按Ctrl+Shift+Alt+Insert ，然后选择HTTP Request 。

物理文件可用于记录，测试和验证HTTP请求。物理文件存储在您的项目中，RubyMine不会修改它们。从物理文件执行HTTP请求时，不会修改此文件。带有响应输出文件链接的有关已执行请求的信息将添加到请求历史记录文件的顶部。

使用变量

编写HTTP请求时，可以使用variables来参数化其元素。变量可以保存请求的主机，端口和路径的值，查询参数或值，标头值或在请求主体内部或外部文件中使用的任意值。

要在请求中提供变量，请将其括在双花括号中，如下所示： {{variable}} 。变量名称只能包含字母，数字，下划线符号_或连字符- 。

变量的值可以是以下任意值：

  • 通过环境变量为每个环境显式提供

  • 通过以下方式动态生成$uuid ， $timestamp和$randomInt预定义的动态变量

  • 通过以下方式在响应处理程序脚本中以编程方式定义client.global.set方法。

动态变量

每次您运行请求时，动态变量都会生成一个值：

  • $uuid ：生成通用的唯一标识符（UUID-v4）

  • $timestamp ：生成当前的UNIX时间戳

  • $randomInt ：生成介于0和1000之间的随机整数。

例如：

GET http：// localhost / api / get？id = {{$ uuid}}

环境变量

环境变量使您可以在项目内部存储一组环境定义。例如，您可以创建并使用{{host}}变量，而不是在请求中显式提供主机名。执行请求时，可以在定义的环境之间进行选择，然后将其发送到特定主机：

在查看请求的结构并在浏览器中打开请求或为其创建运行/调试配置时，所选环境将用作默认环境。

定义环境变量

环境变量在环境文件中定义。

  1. 在项目内部，创建以下文件：

    • rest-client.env.json （或http-client.env.json ）是一个包含公共变量的常规文件，旨在与您的项目一起分发。

    • rest-client.private.env.json （或http-client.private.env.json ）是一个私有文件，可能包含密码，令牌，证书和其他敏感信息。默认情况下，此文件被添加到VCS忽略文件列表中。在http-client.private.env.json文件中指定的变量的值将覆盖环境文件中的值。

  2. 用所需的变量填充创建的文件。

    以下示例http-client.env.json环境文件定义了两个环境： development和production 。附加的http-client.private.env.json文件包含敏感授权数据。

使用响应处理程序脚本

使用响应处理程序脚本，您可以以编程方式对收到的HTTP响应做出反应。通过使用这些脚本，您可以自动处理接收到的数据，并根据您指定的条件对其进行验证。响应处理程序脚本作为HTTP请求文件中请求的一部分提供，并在收到响应后立即执行。要查看响应处理示例，请打开auth-requests或test-responses 请求集合 。

您可以就地或通过引用外部文件将响应处理程序脚本插入请求中。

将脚本插入请求中

  • 要就地插入脚本，请在脚本之前加上>并将其封装在{% %} ：

    GET host / api / test> {％//响应处理程序脚本...％}

  • 要从外部文件插入脚本，请在脚本前面加上> ：

    GET host / api / test> scripts / my-script.js

响应处理程序脚本使用JavaScript ECMAScript 5.1编写，编码帮助和文档由捆绑包处理HTTP Response Handler图书馆。对于就地脚本，此功能会自动启用。对于外部脚本，您需要手动启用它。

为响应处理程序脚本启用JavaScript编码帮助

  1. 在编辑器中打开脚本文件。

  2. 在上下文菜单中，选择“ 使用JavaScript库”。 HTTP响应处理程序 。

的HTTP Response Handler库公开了两个用于构成响应处理程序脚本的对象：

  • 的client对象存储会话元数据，可以在脚本内部对其进行修改。的client状态一直保持到您关闭RubyMine。保存的每个变量client.global如variable_name后续的HTTP请求可以访问{{variable_name}} 。

  • response包含有关已接收响应的信息：其内容类型，状态，响应正文等。

要在编辑器中打开“ HTTP响应处理程序”库，请将插入号放在库对象上，然后按Ctrl+B 。

响应处理程序脚本可以包含测试，该测试使您可以将HTTP客户端用作测试框架。要创建测试，请调用client.test(testName, function)方法。在测试中，您可以通过调用client.assert(condition, message)方法，例如：

GET https://httpbin.org/status/200> {％client.test（“请求成功执行”，function（）{client.assert（response.status === 200，“响应状态不是200”）； }）; ％}

    • 在编辑器中，选择创建从您要为其创建运行/调试配置的请求的上下文菜单中。

查看来自Web服务的响应

当您执行HTTP请求时，RubyMine会自动将响应保存到.idea / httpRequests /目录下的单独文件中。您可以查看50个最近存储的响应，并使用请求历史记录导航到相应的文件。如果该请求是从头文件执行的，则其响应输出的链接也会添加到原始请求的下方：

通过响应收到的cookie将自动保存到.idea / httpRequests /目录下的专用http-client.cookies文件中。可以保存的Cookie数量限制为300个 。如果未到到期日期 ，则在对URL的每个后续请求中都会自动包含cookie的名称和值 ，该URL与为cookie指定的域和路径相匹配。

如有必要，可以通过在请求之前添加带有@ no-cookie-jar标记的注释行来防止将接收到的cookie保存到cookie罐中。

查看请求历史记录

RubyMine自动将最近执行的50个请求保存到http-requests-log.http文件中，该文件存储在项目级别的.idea / httpRequests /目录下。使用请求历史记录，您可以快速导航到特定的响应并再次发出任何请求 。如果从请求历史记录再次发出请求，则其执行信息和到响应输出的链接将添加到请求历史记录文件的顶部。
