创建Rails应用程序元素

RubyMine帮助您使用所有必需元素的存根来填充Rails应用程序：控制器，视图，模型等。您可以通过几种方式启动Rails生成器：

  • 按Ctrl两次，然后开始键入所需的命令或其部分（例如， rails g controller ）。然后，从建议列表中选择此命令，然后按Enter 。

  • 使用工具|运行Rails Generator菜单命令。

  • 转到文件|新 Alt+Insert然后选择Run Rails Generator 。

在本主题中，我们将向您展示有关如何创建Rails应用程序元素的几个示例。

创建控制器和动作

要生成控制器及其可选的动作，请执行以下操作：

  1. 按Ctrl两次并开始输入rails g controller 。选择rails g controller然后按Enter 。

  2. 在调用的“ 添加新控制器”对话框中，指定控制器名称。（可选）添加用空格分隔的操作名称（例如new create ）。单击确定 。

    其他选项允许您提供其他参数。例如，您可以添加--no-test-framework参数。

  3. RubyMine生成所需的文件（控制器，视图和其他文件），并在“运行”工具窗口中显示输出。

    您可以单击任何文件以在编辑器中快速打开它。

从动作创建视图

您可以直接在编辑器中为指定的控制器操作创建视图。去做这个：

  1. 在编辑器中打开控制器文件。

  2. 找到所需的操作，然后单击图标位于编辑器的左装订线中。

  3. 如果存在与该方法关联的视图，则在编辑器中将其打开。如果一个视图不存在，RubyMine建议创建一个新视图。

    指定视图文件的名称，然后单击“ 确定” 。请注意，您可以创建ERB或Haml视图。

建立模型

要创建模型，请执行以下步骤：

  1. 按Ctrl两次并开始输入rails g model 。选择rails g model然后按Enter 。

  2. 在调用的“ 添加新模型”对话框中，指定模型名称，属性及其类型（例如， Article title：string text：text ）。单击确定 。

    按Ctrl+Space完成属性类型。

  3. RubyMine生成所需的文件（模型和迁移文件），并在“运行”工具窗口中显示输出。

    您可以单击任何文件以在编辑器中快速打开它。

要查看所创建模型的依赖关系图，请按Ctrl+Shift+Alt+U ，在调用的弹出窗口中选择Rails Model Dependency Diagram ，然后按Enter 。在Show model diagram中了解更多信息。
