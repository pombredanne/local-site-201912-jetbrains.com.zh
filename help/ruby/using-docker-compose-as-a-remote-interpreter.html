使用Docker Compose作为远程解释器

RubyMine提供了各种功能来处理在您的计算机上运行的Docker。您可以在Docker主题中了解更多信息。

本教程描述了如何将Docker Compose用作示例Rails应用程序的远程解释器。该应用程序由数据库后端和Web前端组成。我们将配置该应用程序以使用两个单独的服务运行它，然后尝试调试功能。

安装Docker并打开一个示例应用程序

在本教程中，我们将结合使用Mac和MacOS，并安装RubyMine。此外，必须满足以下先决条件才能完成所有步骤：

  • 确认Docker已安装并正在运行。

  • 确保已启用 Docker插件。

  • 在RubyMine中打开以下Rails应用程序：

    https://github.com/JetBrains/sample_rails_app.git

使用Compose构建和运行应用程序

在本章中，我们将在两个单独的服务中用完应用程序：用于前端的Web和用于数据库的db 。为此，我们需要在database.yml文件中重新配置数据库设置。然后，我们可以运行docker-compose up 。执行以下步骤：

  1. 转到config / database.yml文件并注释Ctrl+/以下代码：

    开发：<<：*默认适配器：sqlite3数据库：db / development.sqlite3

  2. 取消评论Ctrl+/使用Postgres数据库为开发环境进行配置的部分：

    开发：<<：*默认适配器：postgresql编码：unicode主机：db用户名：postgres密码：数据库：sample_rails_app_db

  3. 打开docker-compose.yml文件。请注意，以下命令用于Web服务以使其保持运行：

    web：＃命令：tail -f / dev / null

    我们需要Web服务运行，因为RubyMine使用docker-compose exec内部使用命令将Docker Compose添加为远程解释器 。

  4. 请点击在装订线中等待Docker Compose拉取/构建映像并启动容器。

    或者，您可以运行docker-compose up RubyMine终端中的命令。

将Compose配置为远程解释器

  1. 打开设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| Ruby SDK和Gems页面。

  2. 请点击 ，然后选择新建遥控器...。

  3. 在调用的对话框中，选择Docker Compose并指定以下选项：

    • 服务器 -此选项指定用于运行容器的Docker服务器。

    • 配置文件 -离开docker-compose.yml文件。

    • 服务 -选择Web服务。

    • Ruby或版本管理器路径 -保留默认的ruby值以自动检测到Ruby解释器的路径。您也可以手动指定解释器或版本管理器可执行文件的路径。

  4. 在Ruby SDK and Gems页面中选择添加的SDK，然后单击OK 。

  5. 等待RubyMine完成索引过程并创建辅助Docker映像和容器。

创建数据库并运行迁移

在调试Rails应用程序之前，我们需要创建数据库并运行迁移。

  1. 按Ctrl两次并输入db：create 。选择rake db:create在下拉菜单中，然后按Enter 。将默认设置保留在调用的“ 执行db：create”对话框中，然后单击“ 确定” 。

    如果看不到rake db:create在下拉列表中，尝试重新加载Rake任务： 重新加载Rake任务 。

  2. 最后，要迁移数据库，请按Ctrl两次，键入db：migrate ，选择db:migrate在下拉菜单中，然后按Enter 。在调用的对话框中单击确定 。

安装调试gem并设置断点

要使用Docker Compose调试应用程序，我们需要将调试工具安装到我们的远程解释器中：

  1. 打开Gemfile并取消注释Ctrl+/以下代码行：

    gem'debase'gem'ruby-debug-ide'

  2. 将插入符号放在这些宝石中的任何一个上，然后按Alt+Enter 。在弹出窗口中，选择Run'docker-compose build' ，按Enter然后等待RubyMine安装这些gem。

  3. 打开users_controller.rb文件，并在创建新用户所在行旁边的create方法内设置一个断点。

撰写运行/调试配置设置

在运行调试会话之前，让我们检查与Docker Compose相关的启动设置：

  1. 按Ctrl+Shift+A然后开始输入编辑配置 。选择编辑配置 ，然后按Enter 。

  2. 在调用的“运行/调试配置”对话框中，选择“ Rails”节点下的“ 开发”配置。

    在docker-compose组中，您可以看到以下用于运行/调试Rails应用程序的选项：

    • docker-compose exec -RubyMine在已经运行的容器中运行命令。

    • docker-compose up -RubyMine使用其他设置启动用作远程解释器的服务（例如，公开调试器所需的其他端口）。这是开始调试会话的推荐方法。

    • docker-compose run -RubyMine在新容器中运行命令。

    保留默认的docker-compose并单击OK 。

开始调试

现在我们准备调试该应用程序：

  1. 请点击Shift+F9开始调试。在调用的对话框中，选择Patch project config 。

  2. 打开浏览器并指定应用程序地址0.0.0.0:3000 。

  3. 单击立即注册！ 按钮。在注册页面上，输入所需的用户参数，然后点击创建我的帐户 。

  4. 调试器在断点处暂停其会话，并使您能够检查应用程序状态。

    例如，您可以检查在“ 注册”页面上指定的用户参数。
