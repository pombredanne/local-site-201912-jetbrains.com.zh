配置JavaScript调试器

通过JavaScript Debugger插件支持在RubyMine中调试JavaScript。 的默认情况下，插件已激活。如果禁用了插件 ，请按照“ 管理插件”中所述在“ 插件”页面上启用它。

Google Chrome和Chrome系列的其他浏览器均支持JavaScript代码的调试。

为了确保成功调试，只需指定内置的Web服务器端口并接受RubyMine建议的其他调试器选项的默认设置即可。

在你开始前

  • 确保在“ 插件”页面上启用了JavaScript调试程序捆绑的插件 ，有关详细信息，请参阅管理插件 。

设置内置的Web服务器端口

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 构建，执行，部署” |调试器 。

  2. 在“ 内置服务器”区域中，指定运行内置Web服务器的端口。默认情况下，此端口设置为默认的RubyMine端口63342 ，RubyMine通过该端口接受来自服务的连接。您可以将端口号设置为以1024或更高版本开头的任何其他值。

    如果您正在使用JetBrains Chrome扩展程序进行调试，它也使用此端口连接到运行页面，请参阅使用JetBrains Chrome扩展程序 进行调试和使用默认的Chrome用户数据启动调试会话 。

可选地

  • 通过分别清除“ 可以接受外部连接”或“ 允许未签名的请求”复选框，禁止从其他计算机或从RubyMine 外部调用内置服务器上的文件。

  • 选择删除断点的方法，默认设置为单击 。

  • 在调试器上。在“ 调试器”节点下的“ 数据视图”页面上，配置高级调试器选项：启用或禁用“ 内联调试” ，指定何时要查看带有对象值和表达式评估结果的工具提示，等等。

使用默认的Chrome用户数据启动调试会话

您可能会注意到，调试会话会在一个新窗口中开始，其中包含自定义的Chrome用户数据，而不是默认的用户数据 。结果，该窗口看起来异常，例如，您的书签，浏览器历史记录和扩展名丢失，这完全破坏了您的开发经验。发生这种情况是因为RubyMine使用Chrome调试协议，并使用--remote-debugging-port选项。但是，如果Chrome已经启动，则无法为任何具有相同用户数据的新的或现有的Chrome实例打开调试端口。因此，当使用Chrome调试协议时，RubyMine总是在带有自定义用户数据的新窗口中启动调试会话。

摘要

  • 要在同一个Chrome实例中启动调试会话，请像以前一样使用JetBrains Chrome扩展程序 。

  • 要以熟悉的外观打开新的Chrome实例，请在RubyMine中配置Chrome以从用户数据开始。在这种情况下，在开始调试会话之前，请始终确保Chrome尚未与用户数据一起运行。否则，RubyMine仍会使用您的用户数据启动另一个Chrome实例，但无法为其打开调试端口。结果，RubyMine调试器无法连接到新的Chrome实例中的应用程序，并且调试会话无法启动。

在RubyMine中配置Chrome以从您的Chrome用户数据开始

  1. 将Chrome用户数据保存在计算机上的任何位置。

  2. 在里面 设置/首选项对话框Ctrl+Alt+S ，单击“ 工具”下的“ Web浏览器 ”。将打开“ Web浏览器”页面。

  3. 要创建新的Chrome配置，请点击 。一个新项目出现在列表中。在路径字段中，指定Chrome安装文件夹的路径。

  4. 选择新的配置，然后单击 。将打开“ Chrome设置”对话框。

  5. 选择使用自定义用户数据目录复选框并指定路径到您的用户数据目录中的RubyMine设置。

  6. 按照网络浏览器中的说明将您的Chrome浏览器配置标记为默认 ，并且在创建运行/调试配置时不要忘记从浏览器列表中选择默认 。

    另外，请始终从“ 浏览器”列表中选择此Chrome浏览器配置。
