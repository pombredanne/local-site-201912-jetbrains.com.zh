HTML，CSS和JavaScript的实时编辑

使用实时编辑功能，您对HTML，CSS或JavaScript代码所做的更改将立即显示在浏览器中，而无需重新加载页面。Live Edit仅在调试会话期间可用。

Live Edit适用于包含或生成HTML，CSS或JavaScript的其他文件类型。您还可以在调试Node.js应用程序时使用Live Edit 。有关详细信息，请参见Node.js应用程序中的Live Edit 。

默认情况下，仅对HTML和CSS文件启用Live Edit 。

在你开始前

  • 如管理插件中所述，在“ 插件”页面上安装并启用LiveEdit存储库插件。

在JavaScript中激活和配置Live Edit

  1. 如管理插件中所述，在“ 插件”页面上安装LiveEdit插件。

  2. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 构建，执行，部署” |调试器实时编辑 。

  3. 在打开的“ 实时编辑”页面上，选中“ 更改时在Chrome中更新应用程序”复选框。这样可以即时预览HTML和CSS。

  4. 要在JavaScript中启用实时编辑 ，请选择JavaScript，HTML和CSS选项。

  5. 指定在编辑器中更改代码与在浏览器中显示此更改之间的时间延迟：接受默认值300 ms或使用相应字段旁边的旋转框指定自定义值。

  6. 要在可编译为JavaScript，HTML或CSS的代码中启用实时编辑（例如，在TypeScript，Pug或SCSS中），请选中“ 跟踪编译为JavaScript，HTML或CSS的文件中的更改”复选框。

    对此类文件所做的所有更改都会自动保存以触发编译，然后出现在浏览器中。

    要在可编译为JavaScript，HTML或CSS的代码中启用实时编辑（例如，在TypeScript，Pug或SCSS中），请选中“ 跟踪编译为JavaScript，HTML或CSS的文件中的更改”复选框。

    请注意，与关闭文件，项目或RubyMine或其他事件触发的常规自动保存相反， 更新后不久将保存更改的文件。

  7. 要配置突出显示，请选中“ 插入符号更改中的浏览器中突出显示当前元素”复选框。否则，在调试会话期间，您必须保持Shift然后在编辑器中单击该元素，以在浏览器中突出显示该元素。

  8. 选择热交换失败时重新启动复选框如果没有这些更改无法应用更改，则允许RubyMine自动重新加载页面。
