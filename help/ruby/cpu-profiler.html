分析应用程序

RubyMine提供了使用rbspy采样分析器来分析运行中的Ruby / Rails进程的功能 。探查器收集有关您的应用程序执行的信息。您可以使用附加的事件探查器运行Ruby / Rails应用程序，也可以将事件探查器附加到已经运行的进程。停止性能分析会话后，您可以分析每个方法/块花费了多少时间并确定性能问题。

在本主题中，我们将向您展示如何运行/停止分析会话以及分析结果。

先决条件

在开始性能分析会话之前，请使用以下指示信息为您的平台安装rbspy分析器： 安装rbspy 。之后，您将能够使用来自1.9.1及更高版本的本地Ruby MRI解释器来分析应用程序。如果不支持您的版本，请在rbspy Github存储库中创建一个问题： https : //github.com/rbspy/rbspy/issues 。

配置配置文件设置

在运行分析会话之前，您可以调整探查器设置。打开“ 设置/首选项”对话框，然后转到“ 构建，执行，部署” |“设置”。 Ruby Profiler 。您可以在此处指定以下选项：

  • 名称 -指定分析器名称。

  • RbSpy可执行文件 -指定路径rbspy可执行文件。

  • 配置文件速率 -设置每秒记录多少个样本。

  • 配置文件子流程 -指定是否为配置文件的流程配置子流程。

在此对话框中，您可以使用按钮添加具有特定设置的其他配置文件。的按钮删除选定的配置文件。

运行分析会话

RubyMine提供了两种主要的运行分析会话的方式：

  • 使用附带的探查器运行Ruby / Rails应用程序

  • 将探查器附加到已运行的进程

以下过程显示了如何启动性能分析会话。

使用事件探查器运行Ruby脚本

  • 右键单击编辑器或“项目”视图中的脚本 ，然后选择“运行带有'RbSpy profiler'的脚本名称 ” 。

使用事件探查器运行测试

请执行以下任一操作：

  • 要分析特定的测试，请点击按钮，然后选择“使用'RbSpy profiler'运行' 测试名称 '” 。

  • 要在特定文件中分析测试，请在编辑器或“项目”视图中右键单击脚本，然后选择“使用'RbSpy profiler'运行' 测试名称 '” 。

  • 要在特定文件夹中分析测试，请在“项目”视图中右键单击该文件夹，然后选择“使用'RbSpy profiler'运行” 。

使用事件探查器运行Ruby / Rails配置

RubyMine允许您在附加了探查器的情况下运行任何特定于Ruby / Rails的配置。例如，这可能是Ruby，Rails，Rake或RSpec配置。要使用事件探查器运行所需的配置，请执行以下步骤：

  1. 在主工具栏上选择所需的运行配置。

  2. 点击单击所选配置旁边的按钮，然后选择“运行带有'RbSpy Profiler'的配置名称 ” 。

将探查器附加到正在运行的进程

要将分析器附加到已经在您的计算机上运行的Ruby / Rails进程中，请执行以下操作：

  1. 从主菜单中，选择运行|将Profiler附加到Process 。

  2. 从弹出窗口中选择所需的过程，然后按Enter 。

停止分析会话

停止应用程序后，分析会话将自动停止。要手动停止分析会话，请执行以下操作：

  1. 打开事件探查器工具窗口。

  2. 点击按钮。

分析会话完成后，您可以分析分析结果。

分析分析结果

此脚本的性能分析数据将显示在三个选项卡上- 火焰图 ， 调用树和方法列表 。在左侧，您可以按指定的线程过滤数据。

火焰图

此选项卡随时显示调用堆栈的状态。每个框架代表堆栈中的一个方法/块（堆栈框架）。在Y轴上，从下往上有一个堆栈深度。X轴显示从最耗时的方法/块到最少耗费的方法/块分类的堆栈。

请注意，项目框架和宝石框架具有不同的颜色。

阅读火焰图时，请注意最宽的框架。您可以按照从父框架到子框架的代码流程，从底部开始并向上移动。

要在火焰图表中导航，请执行以下操作：

  • 要放大/缩小图表，请使用Ctrl+mouse wheel要么 / 工具栏上的按钮。您可以使用按钮重置为初始图表宽度。

  • 使用鼠标或箭头键滚动图表。

  • 通过双击导航到特定方法。在这种情况下，该图表显示了从以灰色着色的父方法开始的调用堆栈。

您还可以通过以下方式分析火焰图表中的数据：

  • 将鼠标悬停在任何项目上可以查看详细信息。

  • 右键单击该项目，然后选择“ 跳转到源代码”以在编辑器中打开相应的代码。

  • 右键单击该项目，然后在“方法列表”中选择“ 关注方法”以找到“ 方法列表”选项卡中的相应方法。

呼叫树

“ 调用树”选项卡显示程序调用树，其中包含每种方法在总分析时间内的百分比。

要配置和过滤“呼叫树”视图，请使用和纽扣。

方法清单

“ 方法列表”选项卡显示按样品数排序的方法列表 。

对于列表中的每种方法，探查器都会在嵌套选项卡上提供更多信息。Back Traces选项卡显示了调用所选方法的位置。“ 合并的被呼叫者”选项卡显示从所选方法开始的呼叫跟踪。

导出分析结果

  3. 在打开的对话框中，命名文件并指定要在其中保存文件的文件夹。点击保存 。

要从文件导入性能分析结果，请选择“运行” |“运行”。导入探查器结果|从文件从主菜单。例如，如果要使用rbspy分析生产服务器上记录的性能分析结果，此功能将非常有用。
