运行/调试配置：Capistrano

使用此对话框可以为Capistrano任务定义运行/调试配置。

该对话框具有以下选项卡：

  • 配置选项卡
  • 捆绑器标签
  • 代码覆盖率选项卡
  • 钉枪标签
  • 日志标签

配置选项卡

项目

描述

名称

指定当前运行/调试配置的名称。

任务名称

指定要执行的Capistrano任务的名称。

任务参数

指定要传递给Capistrano任务的参数列表。参数应以空格分隔。

阶段

选择所需的阶段。可以在验证应用程序后显示的config / deploy.rb文件中配置阶段。

如果您的项目仅使用一个阶段，则默认阶段对应于production 。有关详细信息，请参阅2.x多级扩展 。

打开调用/执行跟踪，启用完整回溯--trace

选择此复选框以打开标准Capistrano --trace选项。

工作目录

指定正在运行的任务要使用的当前目录。默认情况下，项目目录用作工作目录。

环境变量

将环境变量列表指定为名称/值对，以分号分隔。或者，单击省略号按钮以创建变量并在“ 环境变量”对话框中指定其值。

Ruby参数

指定要传递给Ruby解释器的参数。

将Classpath属性添加到Nailgun设置中。

Ruby SDK

指定所需的Ruby解释器。您可以选择选择项目的默认Ruby SDK，也可以从已配置的Ruby SDK列表中选择其他项目。

日志标签

使用此选项卡可以指定在控制台或运行或调试工具窗口的专用选项卡上显示在运行或调试时生成的日志文件。

项目

描述

活跃

选择此列中的复选框可使日志条目显示在“ 运行”工具窗口或“ 调试 ”工具窗口的相应选项卡中。

日志文件输入

此列中的只读字段列出了要显示的日志文件。该列表可以包含：

  • 特定文件的完整路径。

  • 别名，用于替代完整路径或模式。这些别名也显示在选项卡的标题中，其中显示了相应的日志文件。

    如果日志输入模式定义了多个文件，则选项卡页眉将显示文件名，而不是日志条目别名。

跳过内容

选择此复选框可跳过所选日志的先前内容。

将控制台输出保存到文件

选择此复选框可将控制台输出保存到指定位置。手动输入路径，或单击浏览按钮，然后在打开的对话框中指向所需位置。

在将消息打印到标准输出流时显示控制台

选中此复选框可激活输出控制台，并在关联的进程写入Standard.out时将其调出。

在将消息打印到标准错误流时显示控制台

选中此复选框可激活输出控制台，并在关联的进程写入Standard.err时将其调出。

单击此按钮以打开“ 编辑日志文件别名”对话框 ，您可以在其中选择新的日志条目并为其指定别名。

单击此按钮可在“ 编辑日志文件别名”对话框中编辑所选日志文件条目的属性。

单击此按钮从列表中删除所选的日志条目。

单击此按钮以编辑选择日志文件条目。该按钮仅在选择条目后可用。

捆绑器标签

项目

描述

在包的上下文中运行脚本

如果选中此复选框，则脚本将按照Gemfile中的指定执行。

代码覆盖率选项卡

使用此选项卡可以配置代码覆盖率监视选项。

项目

描述

单击此按钮定义代码覆盖率分析的范围。在打开的“ 添加模式”对话框中，键入以逗号分隔的Ruby正则表达式列表，并指定匹配的文件应包含在代码覆盖率分析中还是应从代码覆盖率分析中排除。

定义要包含在代码覆盖率分析中的文件的模式用+标记；被排除的对象用-标记。

每个模式都可以启用或禁用。为此，请选中或清除图案旁边的复选框。默认情况下，启用所有新创建的模式。

单击此按钮可以从列表中删除选定的图案。

单击此按钮可以更改选定的代码覆盖率模式。

不要使用优化的C运行时

选择此复选框以启用该选项--no-rcovrt 。请谨慎使用此选项，因为它会大大降低性能。

在测试文件夹中启用覆盖率。

如果选中此复选框，则标记为test的文件夹包含在代码覆盖率分析中。

钉枪标签

项目

描述

运行Nailgun服务器的新实例，或使用已启动的实例

该复选框仅适用于用作项目解释器的 JRuby。

启动运行/调试配置（选中此复选框）后，RubyMine会分析正在运行的进程，并根据正在运行的Nailgun服务器的存在情况执行以下操作之一：

  • 如果没有正在运行的Nailgun服务器，或者非默认端口上有Nailgun服务器，或者具有其他gemset，则RubyMine建议指定所需的端口号。

  • 如果Nailgun服务器在具有所需gemset的默认端口上运行，则RubyMine不执行任何操作。

  • 如果Nailgun服务器在具有所需gemset的其他端口上运行，则RubyMine建议指定所需的端口号。

  • 如果Nailgun服务器在具有其他gemset的默认端口上运行，则RubyMine会删除-ng论点。

如果未选中此复选框，那么将以常规方式启动脚本，而不使用Nailgun。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

工具列

运行/调试配置的树形视图具有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

项目

捷径

描述

Alt+Insert

创建运行/调试配置。

Alt+Delete

删除所选的运行/调试配置。请注意，您无法删除默认配置。

Ctrl+D

创建所选运行/调试配置的副本。请注意，您将创建默认配置的副本。

仅当您选择临时配置时，才会显示该按钮。单击此按钮将临时配置保存为永久配置。

查看和编辑所选运行/调试配置的模板（即默认设置）。模板显示在“ 模板”节点下，并用于新创建的配置。

运行/调试配置的默认模板始终按字母顺序排序。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

Alt+Insert

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在RubyMine中定义，请添加其定义。有关更多信息，请参阅外部工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 运行文件监视程序 ：选择此选项可使RubyMine应用所有当前活动的文件监视程序 。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 启动React Native Bundler ：选择此选项以在运行或调试会话中自动运行捆绑程序。默认情况下，这是通过react-native start 。如果您的应用程序使用Expo ，则需要通过以下方式运行开发服务器start npm任务。为此，请单击 ，然后在“ 配置React Native”对话框中，选择npm脚本 ， 然后从列表中选择start 。

  • 编译TypeScript ：选择运行内置的TypeScript编译器，从而确保您对TypeScript代码所做的所有更改都会反映在生成的JavaScript文件中。 在打开的“ TypeScript编译设置”对话框中，选中或清除“ 检查错误”复选框以配置编译器的行为，以防万一检测到任何错误：

    • 如果选中“检查错误”复选框，则编译器将显示所有错误，并且运行配置将不会启动。

    • 如果清除“ 检查错误”复选框，则编译器将显示所有检测到的错误，但仍将启动运行配置。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

  • 将文件上传到远程主机 ：选择此选项可以根据默认服务器访问配置将应用程序文件自动上传到服务器 。

  • 运行远程外部工具 ：添加远程SSH外部工具 。

  • 运行Rake任务 ：添加要在运行或调试之前执行的Rake任务 。要选择耙任务，请单击浏览按钮 ，然后从可用任务列表中选择所需的任务。

    请注意，此处提供代码完成功能。

  • 运行JRuby编译器 ：选择此选项可使用指定的目标路径，编译器进程堆大小和命令行参数（如果有）执行JRuby编译器。

Alt+Delete

单击此图标从列表中删除所选任务。

Enter

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

Alt+Up /Alt+Down

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
