配置SDK宝石集

RubyMine为RVM和rbenv 版本管理器提供了使用gemset的功能。

要查看可用于已安装解释器的宝石集，请打开“ 设置/首选项”对话框Ctrl+Alt+S并转到语言和框架| Ruby SDK和Gems页面。在此页面上，gemset显示为解释器子代：

对于RVM，您可以选择某个gemset并将项目gem安装到该gemset。另一方面，rbenv版本管理器提供了为项目选择多个gemset的功能：

在这种情况下，会将gem安装到第一个启用的gemset。有关如何安装宝石的说明，请参见将宝石安装到本地解释器 。

在项目创建时创建RVM gemset

您可以在项目创建时创建RVM宝石集。为此，请执行以下步骤：

  1. 按照创建并运行第一个Ruby项目中所述开始创建一个新项目 。

  2. 在“ 新建项目”页面上，单击“ 使用gemset'project_name'”复选框旁边的“ 创建”链接：

  3. 创建宝石集之后，请确保已启用“使用宝石集'project_name'”复选框。

    该项目将使用创建的宝石集作为其宝石。

请注意，如果要创建新的Rails应用程序，则需要将Rails gem安装到创建的gemset中。

为现有项目创建RVM宝石集

在“ 设置/首选项”对话框中创建RVM宝石集Ctrl+Alt+S ，请转到语言和框架| Ruby SDK and Gems页面并执行以下步骤：

  1. 点击按钮：

  2. 在弹出窗口中，选择New RVM gemset… ：

  3. 在打开的对话框中，选择将为其创建gemset的Ruby SDK 。然后，指定gemset名称，然后单击OK ：

  4. Ruby SDK和Gems页面将把gemset显示为所选SDK的子代：

    请注意，星号被追加到宝石集名称之后。这意味着尚未创建gemset。要创建它，请在“ 设置/首选项”对话框中单击“ 应用”或“ 确定 ” 。

使用.rbenv-gemset创建Rbenv宝石集

RubyMine使您可以使用rbenv-gemsets插件处理rbenv宝石集 。在这种情况下， .rbenv-gemsets文件管理项目gemset。RubyMine会将Ruby SDK和Gems页面中启用的gemset与.rbenv-gemsets中指定的同步 。例如，如果您取消选中Ruby SDK和Gems中的gemset，则相应的记录将从.rbenv-gemsets文件中删除。

此外，您可以直接在编辑器中创建和禁用gemsets：

请注意，使用远程解释器时不支持rbenv gemsets。

  1. 打开项目的.rbenv-gemsets文件。

  2. 在单独的行上键入所需的宝石集名称。

  3. 使用创建宝石集“名称” 快速修复程序来创建宝石集：

    此gemset将显示为当前SDK的子代：

  4. 如果您从.rbenv-gemsets中删除此gemset，请单击“ 更新库” ：

    对于当前的SDK，该宝石集将被禁用。
