编辑外部工具的设置。

项目

描述

名称

在“ 工具”菜单和上下文菜单中以命令名称显示的工具名称。

组

工具所属的组。工具组对应于“ 工具”菜单和上下文菜单中的子菜单。

从列表中选择一个现有组。

要创建新组，只需在列表中键入其名称。

描述

工具说明（可选）。

工具设定

程序

要运行的可执行文件的路径。使用要选择文件，请插入宏以打开“宏”对话框以选择一个宏。

（宏在运行时已解决，您可以指定上下文信息，例如当前选择的文件，项目源路径等）

争论

传递给程序的参数的方式与您在命令行上指定的方式相同。使用“ 插入宏”打开“宏”对话框以选择一个宏。

指定参数时，请遵循以下规则：

  • 请注意 ，外部工具提供文件的绝对路径，而不是项目目录相对的文件。

  • 使用空格分隔各个参数。

  • 如果参数包含空格，请将空格或包含空格的参数括在双引号中，例如， some" "arg要么"some arg" 。

  • 如果参数包含双引号（例如，作为参数的一部分），请使用反斜杠对双引号进行转义，例如， -Dmy.prop=\"quoted_value\" 。

工作目录

程序当前工作目录的路径。使用要选择目录，请插入宏以打开“宏”对话框以选择一个宏。

高级选项

执行后同步文件

当工具完成执行时，使RubyMine意识到文件系统中的更改。

打开控制台以输出工具

打开控制台以查看工具输出，例如错误消息等。

使控制台在stdout中的消息上处于活动状态

使输出控制台处于活动状态，并在发生相应事件时将其调高。

使控制台在stderr中的消息上处于活动状态

与前面的选项相同，但stderr。

输出滤波器

输出过滤器用于将工具输出中的绝对文件路径和行号转换为超链接。您将能够使用这些链接在编辑器中打开相应的文件。您可以指定一个或多个过滤器。每行都是一个正则表达式。您可以使用以下宏： $FILE_PATH$ ， $LINE$和$COLUMN$ ， 例如，

$ FILE_PATH $：$ LINE $：$ COLUMN $:。*

连接设定

在本节中，指定％destination_environment％通过SSH终端进行访问，并指定应从何处获取连接设置：

  • 当前的Vagrant：选择此选项可使SSH终端中的命令在当前运行的Vagrant虚拟机上执行 。

  • 部署服务器：选择此选项可通过服务器访问配置之一访问在本地或远程Web服务器上执行的SSH终端中的命令。从列表中，选择用于指定目标环境的服务器访问配置，以及用于建立与目标环境的连接的设置。

    • 每次运行时选择服务器：如果选择此选项，则每次选择“ 工具” |“服务器”时，都必须从弹出窗口中选择所需的服务器访问配置。从主菜单启动SSH会话 。

    • 如果所需的服务器访问配置未出现在列表中，请单击“ 配置服务器 ”链接，然后在“ 部署”页面中定义一个。有关详细信息，请参阅“ 配置与远程主机的同步”部分。
