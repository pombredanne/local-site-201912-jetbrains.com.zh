RubyMine 2019.3帮助

流星

Meteor是一个全栈框架，可让您在客户端和服务器端使用JavaScript。RubyMine与Meteor集成在一起，因此您可以在IDE内部使用它。 RubyMine中的Meteor支持包括：

  • 通过检测.meteor文件夹并从项目中排除.meteor / local文件夹，自动识别Meteor项目。有关详细信息，请参见隐藏排除的文件 。

  • 将预定义的Meteor库自动附加到项目。这样可以突出显示语法，解析引用和代码完成。

  • 通过if和each指令完成的通过车把支持空格键。RubyMine可以识别空格键模板，但副作用是在Meteor项目中用 。RubyMine可以在声明中提供JavaScript源代码和模板之间的导航Ctrl+B 。

  • 一种专用的复杂Meteor运行/调试配置，用于在一个调试会话中调试客户端和服务器端代码，请参阅调试Meteor应用程序 。

在你开始前

如“ 管理插件”中所述，在“ 插件”页面上安装并启用Meteor和Handlebars / Mustache存储库插件 。

安装流星

安装过程取决于您使用的操作系统。从Meteor官方网站了解更多信息。

  • 在Meteor官方网站上下载LaunchMeteor.exe安装程序。

确保您使用的是macOS 10.6或更高版本。

  • 在嵌入式终端 （ Alt+F12 ），输入：

    $ curl https://install.meteor.com | /bin/sh

  • 在嵌入式终端 （ Alt+F12 ），输入：

    $ curl https://install.meteor.com | /bin/sh

创建一个新的Meteor应用程序

如果还没有应用程序，则可以从Meteor样板模板生成具有Meteor特定结构的RubyMine项目。或者，创建一个空的RubyMine项目并在其中配置Meteor支持，如下面的从现有Meteor应用程序开始中所述。

从样板模板创建Meteor项目

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 新建项目”对话框。

  2. 在左侧窗格中，选择Meteor App 。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 指定Meteor可执行文件的位置（请参阅安装Meteor ）。

    3. 在模板列表中， 选择要生成的样本。要生成基本的项目结构，请选择“ 默认”选项。

    4. 在文件名字段中， 输入将相互生成的.js ， .html和.css文件的名称。仅当从“ 模板”下拉列表中选择“ 默认样品类型”时，此字段才可用。

创建一个空的RubyMine项目

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 新建项目”对话框。

  2. 在左侧窗格中，选择空项目 。在右侧窗格中，指定应用程序文件夹，然后单击创建 。

从现有的Meteor应用程序开始

如果要继续开发现有的Meteor应用程序，请在RubyMine中打开它，在其中配置Meteor，然后按照下面的“ 下载Meteor依赖项”中的说明下载所需的依赖项。

打开计算机上已经存在的应用程序源

  • 单击“ 欢迎”屏幕上的“ 打开 ”，或选择“ 文件” |“共享”。从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中签出应用程序源

  1. 在“ 欢迎”屏幕上单击“从版本控制获取”，或选择“ VCS |从主菜单的版本控制中获取 。

  2. 在调用的对话框中，从列表中选择您的版本控制系统，然后指定存储库以签出应用程序源。

在现有项目中配置Meteor支持

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | JavaScript |流星 。将打开“ 流星”页面。

  2. 指定流星可执行文件的路径。如果您遵循标准的安装过程，RubyMine将自动检测该文件。

  3. 要使.meteor / local文件夹及其内容包含在索引中，请清除“打开项目时自动排除“ .meteor / local”目录复选框。有关详细信息，请参见下面的隐藏排除的文件 。

  4. 确保选中“ 自动将Meteor包作为外部库导入”复选框。

    • 选中此复选框后，RubyMine会自动从流星/包文件中导入外部包。结果，RubyMine提供了全方位的编码帮助：解析对Meteor内置函数的引用，例如， check(true)以及第三方软件包的功能，提供正确的语法和错误突出显示，支持使用源映射进行调试，等等。

    • 清除此复选框后，RubyMine不会自动从流星/包文件中导入外部包。结果，没有提供编码辅助。要改善这种情况，请在编辑器中打开“ 流星/包裹”文件，然后单击“将包裹作为库导入”链接或运行meteor --update命令。

  5. 确保RubyMine已将Meteor库附加到项目。

    1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到“ 语言和框架” | JavaScript |图书馆 。

    2. 在JavaScript上。将打开“ 库”页面，确保已选中“ 库”列表中“流星”项目库旁边的复选框。

导入流星包

除了预定义的流星库基本流星特定的编码，确保援助，你可以下载额外的软件包在.meteor /本地/包文件中定义的。

下载其他流星包

  1. 在编辑器中打开.meteor / local / packages文件。

  2. 单击屏幕右上角的“ 导入流星包”链接。

  3. 在打开的对话框中，根据要在项目中开发的应用程序的类型，指定要下载的软件包。

    • 客户
    • 服务器

    • Cordova ：选择此选项可导入支持iOS和Android的Meteor应用程序开发的软件包，有关详细信息，请参阅Meteor Cordova Phonegap Integration 。

RubyMine自动将用于存储已构建应用程序的.meteor / local文件夹标记为已排除，但仍将其显示在项目树中。

标记为已排除的文件夹和文件不会在代码完成中使用。

隐藏.meteor / local文件夹

  • 点击按钮在“ 项目”工具窗口的工具栏上单击，然后删除“ 显示排除的文件”选项旁边的对勾。

运行流星应用程序

RubyMine根据Meteor类型的运行配置运行Meteor应用程序。如果您是从样板模板创建应用程序的，那么RubyMine会为您生成一个Meteor运行配置。

创建流星运行配置

  1. 在主菜单上，选择“运行”。编辑配置 ，单击然后从列表中选择“ 流星 ”。

  2. 在“ 配置”选项卡中，根据安装情况指定Meteor可执行文件的路径（请参阅安装Meteor ）。

  3. 指定用于存储要运行的应用程序文件的文件夹。这个文件夹的根目录必须有一个.meteor子文件夹，因此RubyMine会将您的应用程序识别为Meteor项目 。

    默认情况下，工作目录是项目根文件夹。

从技术上讲，RubyMine为服务器端代码和客户端代码创建单独的运行配置，但是您可以在一个专用的Meteor运行配置中指定所有设置。

可选地

  1. 在“ 程序参数”字段中， 指定在启动时传递给可执行文件的命令行其他参数（如果适用）。例如，这些可以是--dev ， --test ， 要么--prod指示运行应用程序的环境（ 开发 ， 测试或生产环境），以便在启动时加载不同的资源。

  2. 默认情况下，RubyMine在“ 运行”工具窗口中显示应用程序输出。要查看客户端代码执行的结果，请在“ 浏览器” /“实时编辑”选项卡中，选择“ 启动后”复选框，然后从列表中选择要打开的浏览器。在下面的字段中，指定用于打开应用程序的URL地址。默认值为http：// localhost：3000 。

运行流星应用程序

  1. 从主工具栏上的列表中选择所需的运行配置，然后转到“运行” |“运行”。跑从主菜单或单击 。

  2. 如果已将浏览器配置为如上所述在应用程序启动时打开，请在“ 运行”工具窗口或浏览器中查看应用程序输出。

调试Meteor应用程序

使用RubyMine，您可以在一个调试会话中调试Meteor JavaScript代码的客户端和服务器端 。调试会话仅通过专用的Meteor运行配置启动。

从技术上讲，可以将实现不同应用程序的多个Meteor项目合并到一个RubyMine项目中。要独立运行和调试这些应用程序，请使用相关的工作目录为每个应用程序创建一个单独的运行配置。为避免端口冲突，这些运行配置应使用不同的端口。在“ 程序参数”字段中，以格式为每个运行配置指定一个单独的端口--port= 。

调试Meteor应用程序

  1. 必要时在代码中设置断点 。

  2. 如上所述创建Meteor运行/调试配置。在“ 浏览/实时编辑”选项卡中，选中“ 启动后”复选框，从列表中选择“ Chrome ”，然后选择“ 带有JavaScript调试器”复选框。

  3. 要启动调试会话， 从主工具栏上的列表中选择所需的调试配置，然后单击列表旁边或选择运行|除错从主菜单。

    将打开“ 调试工具”窗口，其中显示两个选项卡：一个用于调试标记为的服务器端代码另一个用于调试标有的客户端代码 。

  4. 探索已暂停的程序并逐步执行该程序 。

  5. （可选） 按如下所述即时预览对应用程序的更改。

在浏览器中预览更改

在调试会话期间，您可以即时预览对HTML，CSS或JavaScript代码的更改。您编辑的页面的实时内容显示在Elements 调试工具窗口的选项卡。更新策略取决于您正在编辑的应用程序的哪一部分。

要预览对客户端代码的更改，请执行以下任一操作：

  • 切换到JavaScript 标签并点击在工具栏上。

  • 通过选中“ 流星”页面上的“ 启用流星热代码推送”复选框，配置更新的自动上传。从Meteor官方网站了解更多信息。

要预览对服务器端代码的更改，请执行以下操作之一：

  • 切换到 标签并点击在工具栏上。

  • 如HTML，CSS和JavaScript中的实时编辑中所述，使用实时编辑功能配置自动上传。建议您在“ 实时编辑”页面上选中“ 如果hotswap失败 ，请重新启动”复选框，然后，如果没有更改，则RubyMine将尝试重新加载页面。

如果自动上传仍然失败，请通过单击重新启动应用程序在工具栏上。
