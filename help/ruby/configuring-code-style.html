RubyMine 2019.3帮助

配置代码样式

如果公司中存在某些编码准则，则在创建源代码时必须遵循这些准则。RubyMine帮助您维护所需的代码样式。

代码样式是在项目级别和IDE级别 （全局）定义的。

  • 在项目级别，设置在“ 项目”方案下分组，该方案是预定义的并以粗体标记。项目样式方案仅适用于当前项目。

    您可以使用“ 复制到IDE”命令将Project方案复制到IDE级别。

  • 在IDE级别，将设置分为预定义的“ 默认”方案（以粗体标记）和用户通过“ 复制”命令创建的任何其他方案（标记为纯文本）。当用户不想在项目中保留代码样式设置并共享它们时，将使用全局设置。

    您可以使用“ 复制到项目”命令将IDE方案复制到当前项目。

配置语言的代码样式

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|代码样式，然后打开您的编程语言页面。

  2. 选择一种代码样式方案，以用作所选语言的自定义编码样式的基础。

  3. 浏览所选语言页面的选项卡，并为其配置代码样式首选项。

从其他语言复制代码样式设置

对于大多数受支持的语言，您可以从其他语言或框架复制代码样式设置。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|代码样式，然后打开您的编程语言页面。

  2. 单击右上角的“ 设置自 ”。

    仅显示这些语言的链接，这些链接适用于以其他语言为基础定义设置的情况。

  3. 从出现的列表中，选择要从中复制代码样式的语言。

创建代码样式设置的副本

您可以定义与预定义样式不同的代码样式。这些代码样式方案存储在XML文件中，位于用户主目录下的config / codestyles文件夹中。您可以使用创建的副本来修改代码样式和导出。

如果选择Project以外的其他代码样式方案，则将为项目保存此代码样式。因此，您可以为每个项目分配全局（IDE）代码样式。

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|代码样式 。

  2. 从“ 方案”列表中选择所需的方案，然后单击 。

  3. 选择以下选项之一：

    • 复制到IDE-选择此选项可将所选方案存储在全局级别。

      RubyMine将指定名称的新代码样式保存在RubyMine主目录下的config / codestyles / .xml文件中。

    • 复制到项目 -选择此选项可将所选方案存储在项目级别。

      选定的代码样式保存在文件codeStyleSettings.xml中的.idea目录中。

    • 复制 -选择此选项可复制所选方案并将其存储在同一级别。

  4. 在“ 方案”字段中，输入新方案的名称，然后按Enter保存更改。

管理代码样式方案

  1. 在“ 代码样式”页面中 ，选择所需的方案，然后单击 。

  2. 选择以下选项之一：

    • 重命名 -选择此选项可以更改所选方案的名称。

    • 导出 -选择此选项可将您的代码样式设置导出到所需位置。

    • 导入 -选择此选项可导入RubyMine XML代码样式设置，JSCS配置文件或Eclipse XML Profile。

  3. 在“ 方案”字段中，输入新方案的名称，然后按Enter保存更改。

使用EditorConfig在目录级别管理代码样式

要使用EditorConfig， 确保在“ 插件”页面上启用了EditorConfig捆绑插件，有关详细信息，请参阅管理插件 。

RubyMine允许您使用EditorConfig支持（默认启用）管理每个文件集的所有代码样式设置在里面 设置/首选项对话框Ctrl+Alt+S 。您需要做的就是将.editorconfig文件放置在根目录中，该目录包含要定义其代码样式的文件。您可以根据需要在项目中拥有多个.editorconfig文件，因此可以为不同的模块指定不同的样式。

.editorconfig文件中的所有选项都将应用到它所在的目录以及当前项目代码样式之上的所有子目录。如果未定义任何内容.editorconfig ，它取自项目设置。

.editorconfig文件中的所有选项均分为以下类别：

  • 标准选项，例如indent_size ， indent_style ， 等等。这些选项没有任何特定于域的前缀。

  • 具有以下功能的通用IntelliJ选项ij_前缀，适用于所有语言：

    • ij_visual_guides
    • ij_formatter_off_tag
    • ij_formatter_on_tag
    • ij_formatter_tags_enabled
    • ij_wrap_on_typing
    • ij_continuation_indent_size
    • ij_smart_tabs

  • 许多（但不是全部）语言支持的常见IntelliJ选项。他们从ij_any前缀，例如ij_any_brace_style 。

  • IntelliJ语言特定的选项从ij__前缀在哪里是语言域ID（通常是小写的语言名称），例如， ij_java_blank_lines_after_imports 。

可以将相同的选项定义为通用选项和特定于语言的选项，例如， ij_<...>_brace_style 。特定于语言的选项比通用或通用选项具有更高的优先级。

添加一个.editorconfig文件

  1. 在“ 项目”视图中，右键单击包含要定义其代码样式的文件的源目录，然后选择“ 新建” |“新建”。从上下文菜单中选择EditorConfig 。

  2. 选择要定义的属性，以便RubyMine为它们创建存根，或将所有复选框都留空以手动添加所需的属性。

  3. 要预览对代码样式设置的更改将如何影响实际的源文件，请单击在.editorconfig文件的装订线中，选择受其影响的源文件。预览将在右侧打开。

    您可以在预览窗格中进行更改，以尝试测试配置更改的反映方式，而不必担心对源代码进行不必要的更改：关闭.editorconfig文件时，所有这些更改都将被丢弃。
