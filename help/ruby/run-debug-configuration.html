运行/调试配置

RubyMine通过使用运行/调试配置 ，以通用的方式提供了运行和调试Ruby和Rails应用程序，运行测试以及部署代码的功能。运行/调试配置提供了默认的启动参数集，可让您快速运行所需的命令。您可以根据需要自定义这些参数，例如：

  • 对于Rails运行/调试配置，您可以指定服务器类型（例如Puma或Passenger），IP地址和端口，环境（默认为开发 ）等等。

  • 对于Ruby运行/调试配置，您可以定义命令行参数，传递所需的环境变量，等等。

  • 对于RSpec ，您可以选择一个包含所需测试的文件夹并提供自定义运行器选项。

让我们来看看一般如何使用运行/调试配置-从创建临时配置到通过VCS共享配置：

  1. 当您运行Ruby脚本 ， Rails服务器 ， 测试或其他命令时，RubyMine会自动创建相应的临时配置 。您可以编辑并保存此配置。

  2. 对于任何项目，您都可以从预定义的模板中手动创建运行/调试配置 。

  3. 如有必要，您可以自定义用于创建运行配置的模板 。

  4. 要同时启动多个运行/调试配置，请使用复合配置 。

  5. 创建文件夹来组织您的配置。

  6. 通过VCS与团队成员共享您的配置。
