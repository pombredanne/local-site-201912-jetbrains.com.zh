文件编码

档案|设置|编辑器 Windows和Linux的文件编码
RubyMine |首选项|编辑器 macOS的文件编码
Ctrl+Alt+S

使用此设置页面可为当前项目或项目中的任何文件或目录全局配置编码选项 。

如果未定义文件或目录编码，则使用项目编码。如果无法采用项目编码（例如，尚未创建项目时），则RubyMine使用全局编码。

文件或目录编码优先于项目编码 ，而项目编码则优先于全局编码 。

项目

描述

全局编码

如果未指定其他编码选项，则选择默认情况下要使用的编码。

例如，当您指定默认项目的设置或从版本控制存储中签出源时，将应用所选的编码。

选择系统默认值以使用操作系统的默认编码。

项目编码

在下面的“ 默认编码”字段中选择未为其指定编码的文件夹中要使用的默认编码 。

选择系统默认值以使用操作系统的默认编码。

路径

此列显示文件或目录的路径。

默认编码

此列显示Path中指定的文件或目录的编码。

如果编码是在文件中定义的，并且无法更改，则显示为灰色。

如果编码是可配置的，请从选项列表中选择它。

嵌入文件中的编码信息会覆盖所选的信息；嵌套文件或目录的编码信息会覆盖外部目录或整个项目的编码信息。

属性文件的默认编码

指定项目中属性文件的编码。

透明的本机到ASCII转换

选择此选项可将本国字符（非ISO 8859-1 ）存储为转义序列，并显示在属性文件中。

如果清除此复选框，则RubyMine不显示国家字符。

创建UTF-8文件

使用此选择器选择RubyMine如何创建UTF-8文件：

  • 具有字节顺序标记（BOM） ，

  • 没有BOM，

  • 或仅适用于Windows的BOM。

如果您在Windows以外的其他操作系统上工作，并且希望为UTF-8文件编码提供更大的灵活性，则此选项可能会有所帮助。

请记住， 带有BOM表的UTF-8选项可能会导致与BOM表不兼容的软件或需要由解释程序运行的脚本引起问题。
