记录代码

YARD和RDoc是在多个库中用于记录代码的最流行的文档生成工具。RubyMine允许您使用快速文档查找来查看以YARD或RDoc语法编写的文档。此外，RubyMine提供了扩展功能以与YARD标签一起使用：

  • 创建缺少的YARD标签

  • 检查YARD标签的有效性并进行修复

  • 利用YARD标记更好地了解代码（例如，确定对象类型，查看方法的参数信息等等）

添加@param标签

  1. 将插入号放在方法名称上，然后按Alt+Enter 。

  2. 在调用的弹出窗口中，选择“ 添加@param标记”操作。

  3. RubyMine将在方法上方添加相应的注释，并建议您指定每个参数值的类型。开始键入Integer ，然后按Ctrl+Space查看建议。选择Integer ，按Enter确认您的选择，然后按Enter再一次指定第二个参数的类型。

为@param标签配置语法格式

默认情况下，RubyMine在参数类型之后生成参数名称（例如， @param [Integer] val1 ）。要更改此行为并在参数名称之前设置参数名称（ @param val1 [Integer] ）， 请执行下列操作：

  1. 打开“设置/首选项”对话框（ Ctrl+Alt+S ）。

  2. 打开编辑器|检查页面，然后在Ruby组中选择Add @param标记检查。

  3. 使用“ 标记语法格式”选项指定所需的顺序。

删除不正确的YARD标签

当您编辑YARD标签时，RubyMine会检查是否存在任何错误的标签。例如，这可以是重复的标记，或者标记在代码中没有相应的参数。要删除不正确的YARD标签：

  1. 将插入符号放在包含所需标签的突出显示的行上，然后按Alt+Enter 。

  2. 在调用的弹出窗口中，选择“ 删除标签”操作。

YARD提供代码洞察力

RubyMine利用YARD类型注释来实现各种代码洞察功能，例如，确定对象类型，在代码完成中使用获得的对象类型，查看方法的参数信息等等。让我们看几个例子。

除了YARD标签之外，RubyMine还可以处理自定义@type标签以接收有关对象类型的信息。

返回/参数类型不匹配

RubyMine可以检查方法的实际返回值和参数类型是否与使用@return和@param注释。例如，如果方法的返回类型与@return类型，编辑器将向您显示警告。

您可以使用YARD返回类型匹配和YARD参数类型匹配 检查来管理这些警告。
