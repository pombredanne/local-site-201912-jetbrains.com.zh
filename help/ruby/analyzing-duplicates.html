RubyMine 2019.3帮助

分析重复项

RubyMine通过重复代码片段 检查，可以帮助您在特定的文件集中找到重复的代码块。如果您通过编写或粘贴代码来创建副本，则RubyMine会立即突出显示该副本并建议快速修复。

要一次查看所选文件集中的所有重复项，请按名称运行检查 。

该检查是开箱即用的，并且具有许多设置 ，您可以更改这些设置以调整其行为。

配置检查

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，转到编辑器|检查 。

  2. 确保在“ 配置文件”列表中选择了必要的检查配置文件 。

  3. 在检查对话框的搜索字段中，键入duplicated code fragment在列表中找到检查。

    单击检查后，其设置将显示在右侧。

    • 严重性 ：从列表中选择一个严重性级别或创建一个新的严重性级别 。

    • 范围 ：选择要在其中进行检查的文件的范围（要在其上运行检查的代码）。

    • 显示重复项来自 ：选择将从中显示重复项的文件集。

      您可以限制分析以仅查看来自同一模块或同一文件的重复项，以避免出现假阳性结果。

    • 跳过重复比以下简单 ：配置代码片段的大小，将其与所选文件集中的其余代码进行比较（以单位为单位）。

      默认值是准确性和找到的重复项数量之间的折衷。增加值以获得更准确的结果。减小值以查找更多重复项。

      单位的确切值可以计算如下： units = 2 * number of statements + number of expressions 。

    • 选择此检查将要使用的语言。

  4. 应用更改并关闭对话框。

搜索重复项时忽略名称和值

您可以为每种文件类型配置要匿名的构造。这意味着您可以定义是否应将使用不同名称或值的两个相同实体视为重复项。

  1. 单击检查描述中的链接，或打开 设置/首选项对话框Ctrl+Alt+S并转到编辑器|重复项 。

  2. 选择要应用分析的文件类型，然后选择要匿名化的结构旁边的复选框。

    例如，如果启用了变量或标识符名称以及功能或字段名称 ，则以下两个功能将被视为重复的：

    def乘法（a，b）a * b端def乘法器（c，d）c * d端

  3. 应用更改并关闭对话框。

该对话框还允许您从分析和索引编制中完全排除一种语言。如果在此页面上禁用语言，则该语言将在检查设置中隐藏。

手动搜索重复项（不建议使用）

此选项已被弃用，不久将被删除。

  1. 在主菜单中，选择代码|找到重复项 。

  2. 在“ 指定代码重复分析范围”对话框中，指定分析范围：整个项目，当前文件，未提交的文件（对于受版本控制的项目）或某些自定义范围。此外，您也可以将测试源包括在分析中。

  3. 在“ 代码重复分析设置”对话框中，选择要分析的语言。

    对于每种语言，检查选项以定义分析首选项。例如，您可以选择请求将代码片段视为重复的相同匹配项，或者指定一个特定的限制，在该限制之下，代码构造不被视为重复（例如，避免报告每个if在源代码中构建）。

  4. 在“ 重复工具”窗口中 ，浏览分析结果。

    • 在工具窗口的左窗格中查看重复项列表。

    • 在右窗格中查看找到的重复项之间的差异。使用箭头按钮将选定的副本放置在差异查看器的部分之一中，并比较代码片段。

    • 使用跳转到源或显示源上下文菜单命令在编辑器中导航到重复项。
