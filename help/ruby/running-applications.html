运行Rails应用程序

RubyMine允许您使用配置的一组启动参数（ 运行/调试配置）来运行或调试Rails应用程序。例如，您可以为Rails运行/调试配置指定以下参数：服务器类型（例如Puma或Passenger），IP地址和端口，环境（默认为开发 ）等等。请注意，对于Rails项目，RubyMine会使用相应的Rails环境自动创建开发和生产运行/调试配置。

在本主题中，我们将向您展示如何：

  • 在没有运行配置的情况下首次运行Rails服务器

  • 使用运行配置运行Rails服务器

  • 配置其他参数以运行Rails服务器

在不配置的情况下运行Rails服务器

要在没有任何配置的情况下首次运行Rails服务器，请执行以下操作：

  1. 按Ctrl两次。

  2. 输入rails server在调用的弹出窗口中输入命令，然后按Enter 。

  3. 等到RubyMine启动Rails服务器。运行工具窗口显示应用程序的输出。

运行Rails服务器后，RubyMine会自动创建一个特殊的配置文件-临时运行/调试配置 。您可以自定义此配置的设置 ，例如，服务器类型，IP地址和端口，环境等。然后，您可以保存自定义配置，以在将来快速运行此配置 。

使用运行/调试配置运行Rails服务器

您可以通过以下方式之一使用现有的运行/调试配置来运行Rails服务器：

  • 按Ctrl两次调用“ 运行任何东西”弹出窗口。开始输入所需的配置名称，从列表中选择它，然后按Enter 。

  • 点击工具栏上的按钮可运行当前选定的运行/调试配置。

    您还可以使用“ 服务”工具窗口在专用工具窗口中一次管理多个运行/调试配置。

  • 选择运行|跑 Shift+F10从主菜单输入命令，然后选择所需的配置。

配置Rails服务器参数

首次运行Rails服务器时 ，RubyMine会自动创建一个相应的Rails 临时配置 ，可以将其保存。如有必要，可以从预定义的模板手动创建Rails运行/调试配置。

要自定义运行/调试配置，请执行以下操作：

  1. 打开“ 运行/调试配置”对话框：

    • 选择运行|从主菜单编辑配置 。

    • 在导航栏可见（“ 视图” |“外观” |“导航栏” ）下，从运行/调试配置选择器中选择“ 编辑配置 ”。

    • 按Shift+Alt+F10 ，然后按0或从弹出窗口中选择配置，然后按F4 。

  2. 在打开的“运行/调试配置”对话框中，在“ Rails”组中选择所需的配置，然后指定其设置。

    例如，您可以在“ 配置”选项卡中指定以下设置：

    选项

    描述

    服务器

    指定用于运行应用程序的Web服务器。您可能需要在项目SDK中安装相应的gem（例如，Passenger服务器的passenger gem）。

    IP地址

    指定可访问Rails服务器的IP地址。

    港口

    指定用于运行Rails服务器的端口。

    服务器参数

    输入可选的服务器参数。您可以运行rails server --help 终端仿真器中的命令以了解有关可用参数的更多信息。

    环境

    选择所需的Rails环境以运行服务器。

    虚拟应用

    仅对可安装Rails的引擎项目启用此选项。在此指定虚拟目录的绝对路径。运行引擎需要此路径。

    运行浏览器

    启用在默认浏览器中自动打开应用程序的功能。

    自动启动JavaScript调试器

    启用此选项以启动JavaScript调试器。请注意，JavaScript调试可用于Firefox和Chrome 。

    环境变量

    将环境变量列表指定为名称/值对，以分号分隔。或者，单击省略号按钮以创建变量并在“ 环境变量”对话框中指定其值。

    Ruby参数

    指定要传递给Ruby解释器的命令行参数 。

    Ruby SDK

    指定用于运行服务器的Ruby解释器 。您可以选择项目默认的Ruby SDK，也可以从已配置的Ruby SDK列表中选择另一个。

重新运行一个应用程序

要重新运行启动的应用程序，请执行以下操作：

  1. 在“ 运行工具”窗口中，选择打开所需应用程序的选项卡。

  2. 在“ 运行工具”窗口的工具栏中，单击纽扣Ctrl+F5 。
