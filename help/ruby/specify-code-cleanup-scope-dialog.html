指定代码清除范围对话框

代码代码清理

指定代码清除的范围。代码清除意味着找到可能有问题的代码片段，并立即自动对其进行修复。

项目

描述

整个项目

如果要对整个项目执行代码清除，请选择此选项。

未提交的文件

此选项仅适用于受版本控制的项目。

如果只想对尚未提交到版本控制系统的文件执行代码清除，请选择此选项，然后从列表中选择一个更改列表。

文件

选择此选项可以对在编辑器中打开或在“ 项目”工具窗口中选择的文件执行代码清除。

模组

选择此选项可以对“项目”工具窗口中当前选择的项目执行代码清除。

仅在一个RubyMine窗口中打开多个项目时，此选项才可用。

目录

选择此选项可以对“ 项目工具”窗口中当前选定的目录执行代码清除。

选定的文件

选择此选项可以对在“ 项目”工具窗口中选择的文件执行代码清除。

自订范围

选择此选项以指定自定义范围。从列表中选择一个预定义的范围，或单击并在打开的“ 范围”对话框中定义范围。有关如何定义范围的说明，请参阅“ 范围语言语法参考” 。

包括测试源

选择此选项可将代码清除应用于测试源。

检验资料

选择要使用的检验配置文件。从列表中选择一个预定义的配置文件，或单击并在打开的“ 检查”对话框中配置配置文件。

您可以打开“ 检查”对话框来检查在运行代码清除时哪些修订将应用于所选范围。
