RubyMine 2019.3帮助

入门

RubyMine是一个集成开发环境（IDE） ，可帮助您在Ruby / Rails项目开发的各个方面（从编写和调试代码到测试和部署完整的应用程序）提高工作效率。RubyMine可用于包括macOS，Windows和Linux在内的不同平台。

在本教程中，我们将使用为Ruby on Rails教程创建的示例应用程序的分支向您展示RubyMine的主要功能。在开始本教程之前，请执行以下操作：

  • 安装Git 。

  • 为您的平台安装Ruby发行版 。

  • 安装Node.js和Yarn 。

  • 安装并设置 RubyMine。

我们将使用安装在macOS上的RubyMine执行所有步骤。

打开一个项目

首先，我们需要克隆包含示例应用程序的存储库：

  1. 运行RubyMine并在“ 欢迎”屏幕上单击“从版本控制获取 ” 。

  2. 在“ 从版本控制获取”对话框中，执行以下操作：

    • 确保在“ 版本控制”字段中选择了Git 。

    • 将以下地址插入URL字段：

      https://github.com/JetBrains/sample_rails_app.git

    单击克隆按钮。RubyMine将显示一个进度条，指示克隆过程。

  3. 克隆存储库后，将提示您打开包含项目的目录。单击是 。

  4. RubyMine打开目录并开始索引过程。您可以在状态栏中看到进度。

    RubyMine为您的项目建立索引，以分析其源代码并收集有关可用文件，类和方法定义等的信息。这是代码洞察功能（例如代码完成和导航）所必需的。

    当您首次打开项目，安装依赖项，向项目中添加新文件，从版本控制系统获取更新或切换分支时，将发生索引编制。

选择Ruby解释器并安装gem依赖项

打开项目后，有必要选择所需的Ruby解释器并安装在项目的Gemfile中指定的依赖项 ：

  1. 按Ctrl+Alt+S要打开“ 设置/首选项”对话框，请转到“ 语言和框架” | Ruby SDK and Gems页面，然后选择所需的解释器。

    在本教程中，我们使用RVM管理Ruby解释器。您还可以使用由其他版本管理器管理的解释器，或使用操作系统的程序包管理系统安装的解释器 。

    单击确定 。

  2. 现在，让我们安装Gemfile中指定的gem。RubyMine允许您使用Bundler来管理宝石。要安装Bundler，请按Ctrl两次，然后输入以下命令： gem install bundler 。

    按Enter并等待，直到安装了Bundler gem。

  3. 现在，您可以在RubyMine中使用Bundler命令。要安装宝石，请按Ctrl再次两次，然后开始键入bundle install 。然后，从列表中选择bundle install命令，然后按Enter 。

  4. 在“ 捆绑安装”对话框中，单击“ 安装” 。

  5. 等到RubyMine安装所有gem。

    现在我们可以在编辑器中尝试一些功能。

安装JavaScript依赖项

要使用Yarn安装项目的package.json文件中提到的JavaScript依赖项，请执行以下操作：

  1. 按Ctrl再次输入两次并开始输入yarn install 。然后，从列表中选择纱线安装命令，然后按Enter 。

  2. 等到RubyMine安装所有gem。

运行迁移

在运行 Rails应用程序之前，我们需要迁移数据库。为此，请在RubyMine中执行以下步骤：

  1. 按Ctrl两次并输入db：migrate 。选择rake db:migrate在下拉菜单中，然后按Enter 。

  2. 将默认设置保留在调用的“ 执行db：migrate”对话框中，然后单击“ 确定” 。

    这将在db文件夹中创建development.sqlite3数据库。

  3. 跑rake db:migrate如第一步中所述，再重复一次。这次，在Execute'db：migrate'对话框中设置Environment选项以进行测试 ，然后单击OK 。创建的test.sqlite3数据库将用于运行测试 。

浏览项目

RubyMine提供了丰富的导航功能，可以浏览任何规模的项目。您可以在文件之间导航，转到声明，搜索任何类型的实体，等等。

专案检视

项目视图（ Alt+1 IDE左侧的）显示项目结构。您可以使用它来打开项目中的任何文件，创建新文件，等等。

前往申报

转到声明允许您从任何符号用法导航到符号的声明。要查看此功能的实际效果，请按Ctrl+Shift+N ，开始输入users_controller ，选择users_controller.rb文件并单击Enter 。

在打开的app / controllers / users_controller.rb文件中，将插入符号放在User类旁边，然后按Ctrl+B 。

您将跳到user.rb文件中的类声明。

请注意，您不仅可以跳到项目实体，还可以跳到外部库（在我们的例子中是宝石）中的定义。例如，保持Ctrl按下，将鼠标指针悬停在has_many方法上。当方法变成超链接时，单击它而不释放键。

RubyMine将在ActiveRecord Rails模块中打开方法定义。

查找用法

让我们演示查找用法功能。在app / controllers / users_controller.rb文件中，向下滚动到edit操作，将插入号放在它旁边，然后按Alt+F7 。在“ 查找”窗口中，您可以浏览使用此操作的位置。

在Rails实体之间导航

在使用特定的Rails实体（例如控制器）时，您可以导航到相关的测试，视图，模型和帮助器。将插入号放在编辑方法旁边，按Ctrl+Alt+Home ，选择查看并按Enter 。RubyMine将打开包含相应视图的edit.html.erb文件。

您可以在视图中使用相同的快捷方式，并使用编辑器装订线中的图标可以快速转到相应的操作。

随处搜寻

RubyMine的下一个功能允许您搜索文件，类，符号或选项，然后跳到所需的实体。

让我们尝试在UsersController中找到destroy动作。按Shift两次，然后开始输入destroy 。下拉列表销毁了Symbols组中的所有控制器。从UsersController中选择销毁操作，然后按Enter 。

将打开users_controller.rb文件，并将插入符号放置在destroy动作的定义上。

编辑代码

RubyMine在编辑器中提供了多种代码编辑功能，这些功能使您可以加快开发过程。这些包括代码完成，重构，代码检查等。

代码完成

RubyMine可以帮助您完成类，方法，关键字等的名称。当您调用代码完成功能时，RubyMine会分析上下文并建议适用于当前插入符号位置的选择。

例如，打开users_controller.rb文件，然后转到UsersController类中声明的索引方法。在方法中键入以下代码...

@user =用户

...，然后键入点。因为User类是从ApplicationRecord模块继承的，所以编辑器将显示所有继承的成员。

在此之后，开始输入第一过滤列表，找到查询模块相应的成员，然后按Enter 。

您还可以通过按手动调用代码完成Ctrl+Space 。

意向

意图可以帮助您快速应用各种代码更改：转换语句以获得更好的代码样式，将字符串添加到语言环境字典，使用语言注入，等等。

要查看实际意图，请打开user.rb文件，然后向下滚动到User.digest方法，该方法使用多行三元运算符（ ？ ：） 。根据Ruby样式指南 ，最好用if / then / else / end块替换此类运算符。为此，将插入符放在此三元表达式（例如，ActiveModel旁边）上，然后按Alt+Enter 。按Enter将三元运算符转换为if / then / else / end块。

请注意，您可以检查代码并使用检查来发现可能的问题。

重构代码

重构是修改源代码以使其更易于维护和扩展，但又不改变其行为的过程。让我们看一下RubyMine中可用的一些重构功能。

重命名重构使您可以重命名类，方法，文件，变量和参数，并在代码中对它们的所有引用进行相应地更正。打开users.rb文件，然后向下滚动到before_save ActiveRecord回调中提出的downcase_email方法。将插入号放在此方法旁边，然后按Ctrl+Shift+I查看其定义。

请点击Esc然后按Ctrl+Shift+Alt+T 。在调用的弹出窗口中选择“ 重命名 ”，这将建议进行各种重构。

在“ 重命名”对话框中，指定一个新的方法名称（在本例中为小写字母 _email ），然后单击“ 重构” 。

“ 重构预览”窗口将显示对重命名方法的所有引用。

单击“ 执行重构”以在所有位置重命名该方法。

提取变量

提取变量重构将所选表达式的结果放入变量中。它声明一个新变量，并将该表达式用作初始化程序。原始表达式将替换为新变量。

打开user.rb文件，然后转到feed方法。在这种方法中，我们可以将SQL子查询提取到变量中。为此，选择一个相应的子字符串，然后按Ctrl+Alt+V 。然后，指定变量名称并按Enter完成提取。

重新格式化代码

RubyMine允许您重新格式化源代码，以满足您的代码风格的要求。

让我们重新格式化user.rb文件的代码。打开此文件，然后按Ctrl+Alt+L 。

RubyMine将重新格式化整个文件，并显示许多更改的行。

分析代码

在这一部分中，我们将执行静态代码分析并检测问题。

RubyMine支持多种检查类型，并且允许在IDE中显示RuboCop违规行为。默认情况下，RubyMine中启用了RuboCop检查，并且要求将RuboCop gem安装在项目的SDK中。如果未安装此gem，RubyMine将建议您这样做。

让我们打开Gemfile，其中包含应用程序使用的gem列表。将鼠标指针悬停在针对bcrypt gem显示的警告上。

RubyMine将显示Rubocop消息，通知您有关按字母顺序订购宝石的必要性（请参阅OrderedGems ）。

将插入符号放在bcrypt宝石旁边，然后按Alt+Enter 。编辑器将建议您修复与不正确的宝石排序有关的所有问题。按Enter去做这个。

您还可以检查整个项目，并在单个报告中显示所有警告。为此，请选择“代码|在主菜单中检查代码 。在调用的对话框中，您可以指定所需的检查范围。

保留整个项目选项，然后单击确定 。检查结果窗口将显示整个项目的警告。

您可以浏览此报告并修复或禁止显示特定警告。

运行测试

运行一个应用程序

要运行我们的Rails应用程序，请按Ctrl两次，然后开始输入开发 。从列表中选择“ 开发：sample_rails_app”配置，然后按Enter 。

调试应用程序

RubyMine的主要功能之一是调试支持。调试器提供了多种方法来检查正在运行的应用程序的状态。您可以单步执行代码并检查变量值，对变量进行监视以查看值何时更改，等等。

我们的示例应用程序使用puma Web服务器，其默认工作程序超时等于60秒。出于调试目的，将此超时设置为一个较大的值可能很有用。为此，请打开config / puma.rb文件并指定worker_timeout选项。

设置断点并开始调试

首先，打开users_controller.rb文件。在创建新用户的行旁边的create方法中设置一个断点。

要开始调试，请按Ctrl两次并开始输入sample_rails_app 。从列表中选择“ 开发：sample_rails_app”配置，按住Shift键（对话框标题将更改为Debug ），然后按Enter 。

如果尚未安装调试所需的debase和ruby-debug-ide gem，RubyMine建议安装它们。

安装gems之后，“ 调试工具”窗口将显示应用程序输出。

打开本地计算机上的浏览器，然后指定应用程序地址0.0.0.0:3000 。

单击立即注册！ 按钮。在注册页面上，输入您的凭据，然后点击创建我的帐户 。

程序到达断点时将停止。

检查变量

现在，您可以检查应用程序状态和变量值。

“ 框架”窗格显示应用程序线程和相应的调用堆栈。在我们的例子中， create方法在线程30中被调用。变量窗格使您可以检查当前上下文中可用的变量。

使用按钮以显示或隐藏来自外部库的方法调用。

让我们将user_params变量添加到列表中。点击按钮在“ 变量”窗格中，然后开始键入user_params 。在调用的下拉列表中选择user_params变量，然后按Enter 。

然后，点击按钮，然后以相同方式展开@parameters变量。您可以在“ 注册”表单中看到指定的用户凭据。

一旦断点被击中，我们就可以逐步执行代码。

越过

单步执行将继续到当前范围的下一行（例如，转到下一行），而不会在途中进行任何方法调用。

在下面的屏幕上，您可以看到尚未创建User对象，并且@user变量尚未初始化（等于nil ）。

按F8或点击 调试窗口工具栏中的“确定”按钮。调试器将转到下一行-if语句。在编辑器和“ 变量”窗格中，您可以看到@user变量已初始化。

使用按钮以检查@user属性。

步入

进入将导致调试器进入当前行中的方法调用或块，并逐步执行它们。如果有多个方法调用或块，则可以选择所需的目标。

点击按钮恢复程序执行。再次转到浏览器，并在“ 注册”表单中创建另一个用户。当脚本到达创建用户所在的行时，它将停止。

按F7或点击按钮。编辑器将重点放在user_params方法上。您可以使用箭头键或Tab选择所需的方法（在下面的示例中为new或user_params ）。选择user_params并按Enter 。程序执行将跳转到user_params方法定义。

如果按F7下次，调试器会建议您在参数之间进行选择，并从StrongParameters模块中要求方法。

在控制台中调试

调试器具有“ 控制台”选项卡，通过它可以使用类似于IRB的控制台与已调试的应用程序进行交互。

开始在控制台中输入user_params ，选择相应的变量并按Enter 。

控制台窗口将显示变量的值。
