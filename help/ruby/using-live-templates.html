实时模板

实时模板 （或代码片段 ）使您可以将常用的构造插入代码中。这些可以是条件，块，循环等。例如，以下动画显示了如何快速插入if-else通过使用预定义来阻止ife缩写。

RubyMine为各种语言和框架提供了许多预定义的模板，包括Ruby，Rails，RSpec，JavaScript等。如有必要，您可以自定义预定义模板或添加新模板。

实时模板有三种类型：

  • 简单模板仅包含预定义的代码。扩展简单模板时，文本将自动插入源代码中，以代替缩写。

  • 参数化的模板包含允许用户输入的变量。扩展参数化模板时，可以手动替换变量或提供表达式以自动计算它们。

  • 环绕模板用用户指定的文本包装所选代码块。

要查看RubyMine中可用的实时模板，请打开“设置/首选项” Ctrl+Alt+S并转到编辑器|实时模板页面。

展开实时模板

扩展简单的参数化模板

要展开实时模板，请键入相应的模板缩写，然后按Tab 。让我们来看看如何插入if-else使用预定义块ife模板。

  1. 在编辑器中，键入ife然后按Tab 。

  2. 指定所需条件，然后按Tab移至下一条语句。

  3. 如果条件为true然后按Tab移动else条款。

  4. 如果条件为false然后按Tab再次完成。

  5. （可选）在完成之前，您可以按Shift+Tab移至上一个子句并进行更改。

扩展环绕声模板

环绕模板对于将带有标签的HTML代码包装起来很有用。执行以下步骤以执行此操作：

  1. 选择所需的代码片段，然后按Ctrl+Alt+J 。

  2. 从调用的弹出窗口中，选择带有模板的环绕声，然后按Enter 。

  3. 输入所需标签，然后按Enter 。

您还可以通过按以下方式扩展环绕声模板Ctrl+Alt+T 。

创建简单的模板

让我们添加一个简单的模板，其中包含没有任何模板变量的代码。例如，我们打开了以下编辑器TODO评论：

＃TODO＃描述：请输入描述＃作者：jetbrains＃日期：01/01/2020

要将此代码另存为模板，请执行以下操作：

从代码片段创建一个简单的模板

  1. 在编辑器中选择代码。

  2. 在主菜单中，选择工具|另存为实时模板 。

  3. 在调用的对话框中，指定以下设置。

    • 缩写 ：指定用于在编辑器中扩展模板的模板缩写。在这里我们用todo 。

    • 描述 ：包含将在完成弹出窗口中显示的模板描述。

    • 模板文字 ：包含要插入代码的模板正文。

    • 适用于 ：指定可以在其中扩展创建的代码段的语言或代码段。单击更改以指定所需的上下文。

    单击确定 。

  4. 要尝试使用创建的模板，请输入todo在编辑器中，然后按Tab 。

您也可以在“设置/首选项”中创建模板（ Ctrl+Alt+S ）在编辑器上|实时模板页面，或复制并重复使用现有模板。在“ 实时模板设置”中了解更多信息。

添加模板变量

实时模板使您可以添加可用于不同目的的变量，例如：

  • 提供有关您需要在当前插入符位置指定的内容的提示

  • 提供要插入的静态默认值

  • 提供一个表达式以计算要插入的值

让我们修改创建一个简单的现场模板前面章节，并添加提示，使得输入更加方便。

添加模板变量的默认值

  1. 打开设置/首选项，然后转到编辑器|实时模板页面。

  2. 选择todo模板在创建之前的章节。

  3. 添加$DESCRIPTION$ ， $AUTHOR$和$DATE$ 模板文本中的变量，如下所示。

    ＃TODO＃说明：$ DESCRIPTION $＃作者：$ AUTHOR $＃日期：$ DATE $

  4. 单击编辑变量按钮。在调用的“ 编辑模板变量”对话框中，在“ 默认值”列中的引号中提供默认值。

    单击确定在此对话框，然后单击设置/首 OK。

    要了解如何使用表达式提供变量值（“ 表达式”列），请参阅使用表达式计算变量 。

  5. 要尝试使用自定义模板，请输入todo在编辑器中，然后按Tab 。使用Tab在变量之间跳转。

    如有必要，请按Shift+Tab移至上一个变量。

使用表达式计算变量

RubyMine允许您使用表达式来计算活动模板变量，并自动插入计算出的值。让我们修改我们的自定义todo在上一过程中创建的模板。

  1. 在编辑器中|实时模板页面，选择todo模板，然后点击编辑变量按钮。

  2. 在调用的“ 编辑模板变量”对话框中，指定设置，如下所示。

    • DESCRIPTION ：保留上一过程中指定的静态默认值 。

    • AUTHOR ：展开旁边的表达式 AUTHOR然后选择user列表中的功能。此函数返回当前用户的名称。

    • DATE ：展开旁边的表达式 DATE然后选择date从列表中。此函数以指定格式返回当前系统日期。

    • 对彼此而言AUTHOR和DATE变量，请启用“ 如果定义则跳过”选项。在这种情况下，RubyMine不会建议您在扩展模板时指定这些变量。

    单击确定在此对话框，然后单击设置/首 OK。

    从变量中使用的预定义函数中了解有关表达式中使用的预定义函数的更多信息。

  3. 在编辑器中，键入todo ，然后按Tab扩展模板。

    正如你所看到的，RubyMine的自动填写作者和日期使用所得到的值user和date功能。因此，您只需要提供描述。

预定义模板变量

RubyMine支持以下无法修改的预定义实时模板变量：

  • $END$代码段完成后指示插入符的位置，您不能再按Tab跳到下一个变量。

  • $SELECTION$在环绕模板中使用，表示要包装的代码片段。在“ 环绕声模板”中了解更多信息。

环绕声模板

环绕声模板使用预定义的代码包装代码块。的$SELECTION$ 预定义变量用于表示要包装的代码片段。让我们来看看如何创建类似于预定义的自定义环绕声模板ife模板。

创建环绕声模板

  1. 打开设置/首选项，然后转到编辑器|实时模板页面。

  2. 选择Ruby组，然后单击按钮。

  3. 指定设置，如下所示。

    • 缩写 ：指定模板的缩写（ ife_sur在我们的情况下）。

    • 描述 ：包含将在“ 选择模板”弹出窗口中显示的模板描述。

    • 模板文本 ：通过以下方式指定模板主体：

      如果$ SELECTION $ $ STATEMENT_ONE $否则$ STATEMENT_TWO $结尾$ END $

      请注意，此代码段包含$SELECTION$展开模板时将用所选代码替换的变量。

    • 定义 ：在此对话框中单击确定 ，然后单击确定在设置/首 。

  4. 要展开创建的环绕声模板，请在编辑器中选择所需的代码片段，然后按Ctrl+Alt+J 。选择所需的环绕声模板，然后按Enter 。

    所选代码将作为条件插入。使用Tab在变量之间跳转。完成后，插入符号将被放置在生成的代码段旁边的行上，作为$END$变量位于模板文本中 。

后缀代码完成与实时模板相似。它转换当前表达式而不选择它。例如，您可以输入.if在表达式之后调用相应的后缀完成并用if声明。

共享实时模板

RubyMine将自定义实时模板组的定义以及添加到预定义模板组的模板存储在自动生成的XML配置文件中。

  • 对于自定义组，该文件包含该组包含的所有模板的定义。

  • 对于已修改的预定义组，该文件仅包含已添加（或已修改）实时模板的定义。

实时模板组配置文件存储在IDE配置目录的模板目录中。通过复制模板目录中的相关文件，您可以在团队成员和多个RubyMine安装之间共享实时模板。此外，您可以在基于IntelliJ平台的所有IDE之间共享实时模板。

如果您的设置是通过“ IDE设置同步”插件同步的 ，则模板存储在IDE配置目录下的jba_config / templates下。

不要将实时模板配置文件复制到项目目录中。

导出实时模板配置

  1. 选择文件|从菜单中导出设置 。

  2. 在“ 导出设置”对话框中，确保已选中“ 实时模板”复选框，并指定将保存导出的设置的存档的路径和名称。

  3. 单击“ 确定”以基于实时模板配置文件生成文件。您可以与团队成员共享此文件，也可以将其导入另一个RubyMine安装中。

导入实时模板配置

  1. 选择文件|从菜单导入设置 。

  2. 使用导出的实时模板配置指定存档的路径。

  3. 在导入设置对话框中，选择实时模板复选框，然后单击确定 。

  4. 重新启动RubyMine之后，您将在上看到导入的实时模板 编辑器| RubyMine设置的“ 实时模板”页面Ctrl+Alt+S 。
