RubyMine 2019.3 Help

Starting the debugger session

RubyMine allows you to debug Ruby scripts, Rails applications, and tests. Starting a debugger session is very similar to running your code. The debugger is attached behind the scenes, so you don't have to configure anything specific to start a debugger session. The topics below describe how to run:

  • Ruby scripts

  • Rails applications

  • Rake tasks

  • Tests

RubyMine allows you to debug code without previous configuring or by using the configured set of startup parameters - run/debug configurations. For instance, you can specify the following parameters for the Rails run/debug configuration: a server type (such as Puma or Passenger), IP address and port, environment (development by default), and so on.

Before debugging

  1. Set breakpoints in the source code.

  2. If necessary, create or modify the corresponding Run/Debug configuration.

  3. Configure the debugger options on the Settings/Preferences | Build, Execution, Deployment | Debugger page.

Start debugging

Debug a Ruby script

To debug a specified Ruby script (for example, script.rb), do one of the following:

  1. Press Ctrl twice to invoke the Run Anything popup and type the ruby script.rb command. Then, hold down the Shift key (the dialog title will be changed to Debug) and pressEnter.

    To debug scratch files or scripts outside the project root, hold down the Alt and Shift keys before running the command (in this case, the dialog title is changed to Run in Context with Debug).

  2. Right-click a script in the editor or project tool window, and select Debug 'script'.

Debug an application using a specific run/debug configuration

  1. Press Ctrl twice to invoke the Run Anything popup. Start typing the required configuration name and select it from the list. Then, hold down the Shift key (the dialog title will be changed to Debug) and pressEnter.

    Note that you can also use Run Anything to debug Ruby/Rails without previous configuring.

  2. Click the button on the toolbar to run the currently selected run/debug configuration.

    You can also use the Services tool window to manage multiple run/debug configurations at once in a dedicated tool window.

  3. Select the Run | Debug Shift+F9 command from the main menu and select the required configuration.

Debug tests

  • Learn how to start a debugging session for tests from the Running and debugging tests topic.

Pause/Resume a debugger session

When the debugger session is running, you can pause/resume it as required using the buttons on the toolbar of the Debug tool window:

  1. To pause a debugger session, click .

  2. To resume a debugger session, click F9.

Terminate a debugger session

  • Click on the toolbar of the Debug tool window. Alternatively, press Ctrl+F2 and select the process to terminate (if there are two or more of them).
