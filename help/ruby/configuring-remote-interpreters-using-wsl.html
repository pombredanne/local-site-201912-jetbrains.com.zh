使用WSL配置远程解释器

本文档介绍如何使用WSL使用Windows上安装的Ruby解释器。支持以下Linux发行版：

  • 的Ubuntu
  • openSUSE
  • SUSE Linux Enterprise
  • Debian GNU / Linux
  • 卡利Linux

执行以下步骤来配置解释器：

  1. 打开设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| Ruby SDK和Gems页面。

  2. 点击并在下拉菜单中选择“ 新建遥控器... ”：

  3. 在调用的对话框中，选择WSL ：

    选择所需的Linux发行版 。

    在Ruby或版本管理器path中 ，指定Ruby解释器或版本管理器可执行文件的路径。单击确定 。

  4. （可选）如果在上一个对话框中指定了版本管理器可执行文件的路径，则RubyMine建议您选择用于运行远程应用程序的Ruby解释器：

  5. 在Ruby SDK and Gems页面中选择添加的SDK：

  6. （可选）如果要使用添加的SDK调试远程进程 ，请指定本地和远程项目的文件之间的映射。为此，请单击“ 编辑路径映射”。 按钮。在“ 编辑项目路径映射”对话框中，指定本地和远程项目根路径：

自定义WSL发行版

RubyMine允许您使用在WSL上运行的自定义Linux发行版 。这可以通过在检测到WSL之后自动编辑RubyMine创建的wsl.distributions.xml配置文件来完成。

添加自定义发行版

  1. 打开％\ HOMEPATH％\。RubyMine2019.3 \ config \ options \ wsl.distributions.xml文件。

  2. 添加descriptor元素并提供设置以访问您的自定义发行版。请注意id值应该是唯一的。例如：

    DEBIAN_CUSTOM Debian debian-custom.exe Debian GNU / Linux-自定义

    您可以指定executable-path有两种方式：

    • 指定自定义发行版的可执行文件名称。在这种情况下，RubyMine将在％LOCALAPPDATA％\ Microsoft \ WindowsApps中找到一个自定义发行版。

    • 指定自定义分发可执行文件的绝对路径。

  3. 如上所述，重新启动RubyMine并将您的自定义发行版添加为远程解释器。
