自定义上传/下载

除了确保在各种项目–服务器设置中成功上载和下载的强制设置之外，您还可以选择其他选项来自定义与服务器的交互。其中大多数选项适用于所有服务器访问配置类型。对于FTP / FTPS / SFTP服务器配置，可以指定其他协议特定的选项。

设置常用的上传/下载选项

  1. 通过执行以下操作之一打开“ 部署”页面：

    • 在里面 设置/首选项对话框Ctrl+Alt+S ，选择部署| 生成，执行，部署下的选项 。

    • 选择工具|部署|从主菜单进行配置 。

  2. 指定其他设置：

    • 要使RubyMine在上载/下载过程中跳过特定文件或整个文件夹，请在按名称排除项目字段中，指定定义这些文件和文件夹名称的模式。

      使用分号;作为分隔符，星号*匹配零个或多个字符和问号?匹配单个字符。

      排除是递归应用的。这意味着，如果匹配的文件夹包含子文件夹，则这些子文件夹的内容也不会部署。

      有关从上传/下载中排除文件和文件夹的更多详细信息，请参阅从上传/下载中排除文件和文件夹 。

    • 通过选中或清除相应的复选框，指定上载/下载过程的详细信息。

为FTP / SFTP / FTPS服务器指定其他协议特定的自定义选项

  1. 通过执行以下操作之一打开“ 部署”页面：

    • 在里面 设置/首选项对话框Ctrl+Alt+S选择部署下建立，执行，部署 。

    • 选择工具|部署|从主菜单进行配置 。

  2. 展开“ 高级”区域并指定其他上载设置：

    • 要将客户端设置为被动模式 ，请选中“ 被动模式”复选框。在这种模式下，您计算机上的客户端连接到服务器以通知处于被动模式，接收要侦听的端口号，并使用接收到的端口通过端口建立数据连接。当您的计算机位于防火墙后面时，此模式很有用。

    • 有隐藏文件，并在显示目录（用点开头的名称。） 服务器浏览器工具窗口 ，选择显示和处理文件隐藏 。

    • 在“ 使用列表”命令区域中选择“ 与旧版本的列出子代兼容”。 以确保子文件命名与FTP服务器兼容 。

      如果远程FTP服务器报告以下错误，则此选项很有用：

      无效的后代文件名
      选择此选项可能会减慢与服务器的同步。

    • 在“ 使用列表”命令区域中选择“ 代替MLSD ” 使用标准LIST用于列出的命令，而不是MLSD命令。这样可以避免出现问题，例如，如果FTP服务器支持，则上传过程中会失败，并带有无效的后代文件名异常MLSD并返回cdir 。

    • 使用“ 检索文件时间戳”列表指定MDTM FTP命令调用策略以检索给定文件在远程主机上的最后修改时间。

      可用的选项有：

      • 始终 -选择此选项可为“ 远程主机”工具窗口中显示的每个文件调用MDTM。

      • 复制时 -选择此选项可在以下情况下调用MDTM：

        • 当清除“ 选项”对话框中的“ 覆盖最新文件”复选框时，检查文件是否最新。

        • 在下载过程中保留文件的实际时间戳。

      • 从不 -选择此选项可禁止调用MDTM。

    • 选择并发连接限制复选框让RubyMine限制同时支持的连接数，并在字段中指定允许的最大连接数。

    • 在“ 为客户端-服务器通信编码”字段中， 指定与服务器使用的编码匹配的编码。如果不确定默认值是否支持UTF-8编码，请接受默认值。

    • 在“ 发送保持活动消息的每个字段”中， 指定您希望RubyMine多久发送一次命令到服务器以重置超时，从而保留连接。

    • 在“ 保持活动”命令列表中， 选择要发送到服务器的命令以重置超时，从而保留连接。

    • 在某些SFTP服务器上，可能会启用SSH标语 。每次建立连接时，可能会显示带有信息消息的弹出窗口，要继续操作，您需要单击OK 。

      要取消显示信息弹出框，请选中“ 忽略信息消息”复选框。
