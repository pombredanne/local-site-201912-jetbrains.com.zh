搜索模板，过滤器和脚本约束

当您为结构化搜索构建模式时，您基本上是在编写脚本。为了简化脚本编写过程，RubyMine为您提供了一系列预定义的搜索模板 ，您可以将它们用作搜索模式的原型。

有关如何访问现有搜索和替换模板的列表，请参见结构性搜索和替换部分。

有效的搜索或替换模板代表以下受支持的语言构造之一：

  • 表达式，例如"John" + " " + "Doe"

  • 语句或语句序列，例如document.getElementById("demo").innerHTML = "Hello Dolly.";

  • 例如，类指示符class Engine implements IEngine

  • 行或块注释，例如/** Created in RubyMine */ 。

在搜索模板中，可以使用以下简化：

  • 方法主体可以省略。

  • 在模板和约束字段中使用短类名（而不是标准名称）。

  • 使用class $Class$作为模板也会导致查找匿名类。

  • 注释和文档注释的模板应包含具有正确注释和JSDoc语法的变量和构造。

每个搜索或替换模板均包含变量$variable_name$您可以向其添加条件（过滤器）以缩小搜索结果的范围。过滤器取决于搜索模板中的变量。
