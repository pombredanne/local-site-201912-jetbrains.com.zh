方案

在这方面选择代码样式方案并根据需要进行更改。每次RubyMine生成，重构或重新格式化代码时，都会自动应用代码样式方案设置。

代码样式是在项目级别和IDE级别 （全局）定义的。

  • 在项目级别，设置在“ 项目”方案下分组，该方案是预定义的并以粗体标记。项目样式方案仅适用于当前项目。

    您可以使用“ 复制到IDE”命令将Project方案复制到IDE级别。

  • 在IDE级别，将设置分为预定义的“ 默认”方案（以粗体标记）和用户通过“ 复制”命令创建的任何其他方案（标记为纯文本）。当用户不想在项目中保留代码样式设置并共享它们时，将使用全局设置。

    您可以使用“ 复制到项目”命令将IDE方案复制到当前项目。

行分隔符（用于新文件）

使用此列表可以指定在RubyMine创建的文件中使用哪个行分隔符 。可用的选项有：

  • 取决于系统 -选择此选项以使用默认选择。

  • Unix和macOS（\ n） -选择此选项以使用Unix和macOS行分隔符。

  • Windows（\ r \ n） -选择此选项以使用Windows行分隔符。

  • 经典Mac（\ r） -选择此选项以使用经典Mac行分隔符。

格式化程序控件

在此区域中，指定标记以限制要从重新格式化中排除的代码片段。在源代码中，格式标记写在行注释中 。

项目

描述

在评论中启用格式标记

  • 如果选中此复选框，则行格式注释之间带有格式标记的代码片段将不会重新格式化，并会保留原始格式。选中此复选框后，以下字段将变为可用，并且您可以指定将字符串视为格式标记。

  • 如果清除该复选框，则格式标记将被忽略，带有标记的行注释之间的代码将重新格式化。

标记物

格式化关闭：

在此字段中，指定将指示您要从重新格式化中排除的代码片段的开头的字符串。用@介词中的符号或保留预定义的值@formatter:off 。

格式化程序：

在此字段中，指定将指示您要从重新格式化中排除的代码片段结尾的字符串。用@介词中的符号或保留预定义的值@formatter:on 。

常用表达

选择此复选框以使用正则表达式，而不是显式指定格式标记。RubyMine使用指定的正则表达式而不是确切的字符串来匹配格式化程序的开/关标记。
