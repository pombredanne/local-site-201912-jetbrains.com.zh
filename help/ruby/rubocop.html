RubyMine 2019.3帮助

RuboCop

RubyMine支持与RuboCop集成，并使您能够在IDE内部（例如，在代码编辑器中）修复其违规行为。

您还可以检查整个项目，并在单个报告中显示所有RuboCop警告。

如有必要，您可以启用标准包装器并将其用于分析项目源。

安装RuboCop /标准宝石

默认情况下，RubyMine中启用了RuboCop检查，并且要求将“ rubocop” gem安装在项目的SDK中 。如果未安装该gem，RubyMine将建议您这样做。

如果要使用Standard来分析项目，请将'standard'gem添加到Gemfile中并进行安装 。

启用/禁用RuboCop和标准检查

要启用或禁用RuboCop和标准检查，请执行以下操作：

  1. 打开设置/首选项对话框Ctrl+Alt+S 。

  2. 转到编辑器|检查页面并启用/禁用RuboCop检查。

  3. 如有必要，启用“ 标准”选项以使用“标准” gem。

在编辑器中修复RuboCop违规行为

启用RuboCop检查后，RubyMine会在代码编辑器中突出显示其违规行为。要查看并修复警告说明，请执行以下操作：

  1. 将插入符号放在突出显示的区域，然后按Ctrl+F1 。

  2. 要解决该问题，请按Alt+Enter 。您可以修正文件中所有建议的违法行为，也可以仅应用集体违法行为或警察部门的特定修正案。

运行Rubocop检查

RubyMine使您可以通过运行相应的检查来查找整个项目的RuboCop违规行为。为此，请按照以下步骤操作：

  1. 从主菜单转到代码|按名称进行检查 Ctrl+Shift+Alt+I 。

  2. 在弹出窗口中，找到RuboCop并按Enter 。

  3. 在“ 运行'RuboCop'”对话框中，指定要分析的文件范围。如有必要，启用Use'standard'gem 。单击确定 。

  4. 等到RubyMine分析您的项目。在“ 检查结果”工具窗口中，您可以浏览RuboCop攻击并使用“ 修复所有自动更正的RuboCop攻击”按钮修复自动更正的攻击 。

    您还可以浏览类文件，并由犯罪或警察部门应用特定的修复程序。

.rubocop.yml

RubyMine考虑了.rubocop.yml文件中指定的所有设置。例如，如果您禁用了对指定警察部门的某些检查，则编辑器将不会显示相应的违法行为。而且，这些违法行为不会在RuboCop的检查报告中显示。

一些RuboCop检查取决于Ruby解释器版本。确保为RuboCop配置的目标Ruby版本与为项目指定的Ruby解释器相同。

如何：使用自定义选项为打开的文件运行RuboCop

有时有必要使用自定义选项为当前打开的文件运行RuboCop。例如，当禁用 RuboCop检查时，或者您要为特定的RuboCop操作分配快捷方式时，此功能很有用。

在此示例中，我们将展示如何使用RuboCop作为外部工具来更正当前打开的文件中的布局违规。

创建一个自定义的RuboCop操作

  1. 在里面 设置/首选项对话框Ctrl+Alt+S ，选择工具|外部工具 。

  2. 请点击并指定以下设置：

    • 名称 ：将在RubyMine界面（“ 工具”菜单和上下文菜单）中显示的菜单命令。

    • 组 ：命令所属的组的名称。您可以选择一个现有组或键入一个新组的名称。

    • 程序 ：RuboCop可执行文件的名称。

    • 参数 ：传递给可执行文件的参数，就像您在命令行上指定的一样。在这里我们用-x RuboCop选项仅对当前文件运行布局警察。

    • 工作目录 ：执行工具的当前工作目录的路径。

  3. 单击确定以添加该工具，然后应用更改。

  4. （可选）为创建的动作分配快捷方式。 在里面 设置/首选项对话框Ctrl+Alt+S选择键映射 ，找到RuboCop节点下的修复动作布局 ，并指定一个快捷方式。

运行自定义的RuboCop操作

  • 要运行创建的RuboCop操作，请转到“ 工具”主菜单或使用文件的上下文菜单。
