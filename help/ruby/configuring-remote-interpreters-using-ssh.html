使用SSH配置远程解释器

RubyMine允许您使用SSH访问来配置远程Ruby解释器，并与部署在远程服务器上的应用程序一起使用。这对于运行或调试远程应用程序，执行测试等很有用。在配置远程解释器之前，请确保满足远程服务器的以下先决条件：

  • 启用SSH访问。

  • Ruby解释器已安装。

  • 远程项目源与本地项目源同步（例如，通过使用Capistrano或远程服务器配置 ）。

现在，您可以按照以下步骤使用SSH配置远程解释器：

  1. 打开设置/首选项对话框Ctrl+Alt+S ，请转到语言和框架| Ruby SDK和Gems页面。

  2. 点击并在下拉菜单中选择“ 新建遥控器... ”：

  3. 在调用的对话框中，选择SSH凭据，然后根据身份验证类型指定所需的参数：

    • 主机/端口 ：用于连接到远程计算机的主机名/ IP地址和端口。

    • 用户名 ：用于在远程计算机上进行身份验证的用户名。

    • 身份验证类型 ：在此字段中，选择“ 密码”值以使用密码对用户进行身份验证。

    • 密码 ：指定用于访问远程计算机的密码。

    • Ruby或版本管理器路径 ：在这里，您应该指定Ruby解释器或版本管理器可执行文件的路径。

    • 主机/端口 ：用于连接到远程计算机的IP地址和端口。

    • 用户名 ：用于在远程计算机上进行身份验证的用户名。

    • 身份验证类型 ：在此字段中，选择密钥对 。

    • 私钥文件 ：指定存储在本地计算机上的私钥文件的路径。

    • 密码短语 （可选）：指定用于指定私钥的密码短语。

    • Ruby或版本管理器路径 ：在这里，您应该指定Ruby解释器或版本管理器可执行文件的路径。

    • 主机/端口 ：用于连接到远程计算机的IP地址和端口。

    • 用户名 ：用于在远程计算机上进行身份验证的用户名。

    • 身份验证类型 ：在此字段中，选择OpenSSH配置和身份验证代理以使用在本地计算机上运行的ssh-agent。

    • Ruby或版本管理器路径 ：在这里，您应该指定Ruby解释器或版本管理器可执行文件的路径。

    指定所有设置后，单击确定 。

  4. （可选）如果在上一个对话框中指定了版本管理器可执行文件的路径，则RubyMine建议您选择用于运行远程应用程序的Ruby解释器。

  5. 在Ruby SDK and Gems页面中选择添加的SDK。

  6. 指定本地和远程项目的文件之间的映射。为此，请单击“ 编辑路径映射”。 按钮。在“ 编辑项目路径映射”对话框中，指定本地和远程项目根路径。
