文件编码

不同类型的文件使用不同的方式来定义编码。AppCode根据文件内容识别文件的编码。编码会影响AppCode读取或写入文件的方式。如果文件已被修改但尚未保存，则任何编码更改都会影响文件的写入。

您可以配置和更改目录或文件（包括属性文件）的编码。

AppCode使您可以在编辑器中工作时更改编码。如果需要检查编码详细信息并配置更多选项，请使用“ 编辑器” | “ 编辑器”。文件编码页面在“ 首选项”对话框中（ ⌘, ）。

在没有内置编码信息的情况下为目录或文件配置编码

  1. 在“ 首选项”对话框中（ ⌘, ），点击“ 编辑器”下的“ 文件编码 ” 。将打开“ 文件编码”页面 。

  2. “ 路径”列显示文件或目录的路径。编码列显示编码。单击指定的编码进行更改。您也可以添加新路径，删除或编辑现有路径。

更改包含显式编码的文件的编码

  1. 在编辑器中打开所需的文件。

  2. 更改显式编码信息。使用错误突出显示识别错误的编码，然后按⌃Space显示可用编码的列表：

更改不包含显式编码的单个文件的编码

  1. 打开所需的文件进行编辑。

  2. 从主菜单中，选择文件|文件编码或单击状态栏上的文件编码。

  3. 从弹出窗口中选择所需的编码。

  4. 如果要么将在所选编码旁边显示，这表示此编码可能会更改文件内容。在这种情况下，AppCode将打开一个对话框，您可以在其中确定要处理的文件：选择“ 重新加载”以从磁盘中将文件加载到编辑器中，并将编码更改仅应用于编辑器，或者选择“ 转换”以使用编码覆盖文件你的选择。
