Node.js

Node.js是一个轻量级的运行时环境，用于在服务器端执行JavaScript。AppCode与Node.js集成在一起，可在配置，编辑，运行，调试，测试，分析和维护应用程序方面提供帮助。

如果只需要Node.js作为应用程序的本地运行时或管理npm软件包，运行JavaScript linter，构建工具，测试框架等的本地运行时，则只需安装Node.js即可 。如果遵循标准安装过程，则大多数情况下AppCode会检测到Node.js本身。

即使您的计算机上没有Node.js，也可以在“ 创建新项目”对话框中 创建新的Node.js应用程序时进行安装，请参阅下面的创建新的Node.js应用程序 。

如果要在多个Node.js安装之间进行切换，则需要将Node.js配置为本地解释器。

要远程运行Node.js应用程序，请将其配置为远程解释器。有关详细信息，请参见使用Docker的 Node.js，通过SFTP的 Node.js，通过SSH的 Node.js以及通过Vagrant的Node.js以获得详细信息。

要配置Node.js解释器， 如管理插件中所述，在“ 插件”页面上安装NodeJS插件。

配置本地Node.js解释器

  1. 在“ 首选项”对话框中（ ⌘, ），请转到语言和框架| Node.js和NPM 。

  2. 在打开的Node.js和NPM页面上，单击 节点解释器列表旁边。

  3. 在打开的Node.js解释器对话框中，其中列出了所有当前配置的解释器，单击在工具栏上。在打开的对话框中，从上下文菜单中选择“ 添加本地” ，然后选择Node.js的本地安装，然后单击“ 确定” 。您将返回“ Node.js解释器”对话框，其中“ 节点解释器”只读字段显示所选解释器的路径。

  4. 在“ 包管理器”字段中， 选择要在当前项目中使用的程序包管理器（npm或Yarn），有关详细信息，请参阅为项目配置程序包管理器 。

当单击OK时 ，返回到Node.js和NPM页面，其中Node解释器字段显示新的解释器。

使用系统Node.js版本

使用AppCode，可以将默认系统节点别名设置为项目的Node.js版本。这意味着，如果您安装新的节点版本并将其设置为系统中的默认版本，则在Node.js解释器字段中指定了该系统别名的AppCode中的所有工具和运行配置都将使用此较新版本。

使用nvm时，此功能特别有用。

  1. 在“ 首选项”对话框中（ ⌘, ），请转到语言和框架| Node.js和NPM 。

  2. 从“ 节点解释器”列表中，选择“ 节点” 。

  3. 在适用的情况下，例如在您的运行/调试配置或特定工具的设置中，指定此新的Node.js解释器。

在Linux的Windows子系统上使用Node.js

AppCode允许您在Windows Subsystem for Linux上使用Node.js运行和调试Node.js应用程序。您可以选择WSL上的Node.js作为当前项目的默认解释器，也可以在Node.js运行/调试配置中配置和使用此节点版本。

将WSL上的Node.js配置为默认项目节点解释器

  1. 在“ 首选项”对话框中（ ⌘, ），请转到语言和框架| Node.js和NPM 。

  2. 请点击在打开的Node.js Interpreters对话框中的Node Interpreter字段旁边，单击 ，然后从列表中选择“ 添加WSL ”。

  3. 在打开的“ 添加WSL节点解释器”对话框中，选择您正在使用的Linux发行版，并指定Node.js的路径。

创建一个Node.js应用程序

如果还没有应用程序，则可以从模板生成具有特定于Node.js的结构的AppCode项目，或者创建一个空的AppCode项目并在其中配置Node.js，如下面的从现有的Node.js应用程序开始所述。

创建一个新的Node.js应用程序

  1. 选择文件|从主菜单中新建项目或单击欢迎屏幕上创建新项目 。将打开“ 新建项目”对话框。

  2. 在左侧窗格中，选择Node.js来创建基本的Node.js应用程序，或者选择Node.js Express App来创建Express应用程序 。

  3. 在右侧窗格中， 指定项目文件夹，Node.js解释器，程序包管理器（ npm或Yarn ，有关详细信息，请参见npm和Yarn ）。

    如果您的计算机上只有一个Node.js，并且遵循标准安装过程，则AppCode会自动检测到Node.js。否则，请从列表中选择相关的解释器，请参阅上面的配置本地Node.js解释器 。

    如果没有安装Node.js，请选择Download Node.js。

    对于Node.js Express应用，请指定express-generator的版本，然后选择要使用的模板语言和样式表语言。

  4. 单击“ 创建”时 ，AppCode将下载必要的依赖项并为它们以及Node.js核心API启用代码完成功能，有关详细信息，请参阅配置node_modules库和配置Node.js Core库 。

    对于Node.js Express ，AppCode使用默认设置创建类型为Node.js的运行/调试配置，并生成特定于Node.js Express的基本目录结构 。

    对于Node.js ，AppCode仅运行npm init命令生成package.json文件。

创建一个空的AppCode项目

  1. 选择文件|从主菜单中新建项目或单击欢迎屏幕上创建新项目 。将打开“ 新建项目”对话框。

  2. 在左侧窗格中，选择空项目 。在右侧窗格中，指定应用程序文件夹，然后单击创建 。

从现有的Node.js应用程序开始

如果要继续开发现有的Node.js应用程序，请在AppCode中打开它，在其中配置Node.js，然后下载所需的依赖项。

打开计算机上已经存在的应用程序源

  • 单击“ 欢迎”屏幕上的“ 打开 ”，或选择“ 文件” |“共享”。从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中签出应用程序源

  1. 单击“ 欢迎”屏幕上的“ 从版本控制中签出”，或选择“ VCS |从主菜单中的“ 版本控制”中签出 。

  2. 从列表中选择您的版本控制系统。

  3. 在打开的特定于VCS的对话框中，键入您的凭据和存储库以从中检出应用程序源。

要下载项目依赖项，请执行以下任一操作：

  • 在嵌入式终端 （ ⌥F12 ），输入：

    npm install

  • 从项目根目录中package.json文件的上下文菜单中选择Run'npm install' 。

在项目中配置Node.js

  1. 在“ 首选项”对话框中（ ⌘, ），请转到语言和框架| Node.js和NPM 。

  2. 在“ 节点解释器”字段中， 指定当前项目的默认Node.js解释器。每次您选择Project创建运行/调试配置或配置依赖于Node.js的工具（例如Prettier或Yeoman）时， Node Interpreter列表中的别名。

    从列表中或单击时打开的对话框中选择解释器 。如果选择node ，则使用系统Node.js版本 。
