改名

通过重命名重构，您可以在自动更正的代码中使用所有对符号和文件的引用来全局重命名符号和文件。

由于仅会影响有限的范围，因此可以轻松地内联重命名局部变量或私有方法。重命名类或公共方法可能会影响许多文件。在这种情况下，建议您在重构之前先预览可能的更改。

重命名符号

  1. 选择要重命名的项目。

    • 要在编辑器中选择一个符号，请将插入符号放置在要重命名的符号名称上。

    • 要在“ 结构”视图中选择一个符号，请在“ 结构”工具窗口中单击所需的符号⌘7 。

  2. 选择重构|从主菜单或上下文菜单中重命名或按⇧F6 。

  3. Swift的后续行为取决于“首选项” | “ 启用”中的 “ 启用就地模式”复选框。编辑器一般

    • 如果选中此复选框，则可以在编辑器中突出显示的行中键入新名称。但是，如果按⇧F6 AppCode将再次显示带有更多选项的“ 重命名”对话框。

    • 如果未选中此复选框，则“ 重命名”对话框将立即打开。

    对于Objective-C，建议的名称显示在符号的正下方。您可以接受建议，也可以输入新名称。

    在“ 重命名”对话框中选择“在注释和字符串中搜索”，也可以在注释和字符串中重命名符号。在就地模式下，AppCode建议您在发现符号出现后，将重构扩展到字符串和注释。

  4. 预览并应用更改 。

重命名文件

  1. 在“ 项目”工具窗口中选择所需的文件。

  2. 选择重构|在主菜单或上下文菜单上重命名或按⇧F6 。

  3. 在打开的“ 重命名”对话框中，指定新文件名。选中在注释和字符串中搜索复选框，以使AppCode将更改应用于注释和字符串。

  4. 按预览以观察“ 查找工具”窗口中的可能更改。按重构继续。

    AppCode查找所有出现的文件名并分别更改它们。
