配置版本控制选项

配置常规VCS设置

常规版本控制设置适用于与AppCode集成的所有版本控制系统。常规设置在“首选项”对话框的“ 版本控制”页面上指定，并且包括定义需要确认的操作，后台操作，忽略的文件，问题导航和历史记录深度。

配置常规版本控制设置

  1. 在“ 首选项”对话框中（ ⌘, ），点击版本控制 。将打开“ 版本控制”页面 。

  2. 指定需要确认与版本控制相关的操作。

  3. 指定应在后台执行哪些操作。

  4. 配置历史记录缓存处理 。

  5. 定义问题导航规则，以从签入注释切换到错误跟踪系统中的相应问题。

指定哪些动作需要确认

您可以定义某些与版本控制相关的活动仅应在您身边确认后才能执行。需要确认的活动在“ 确认设置”页面中指定。

  1. 在“ 首选项”对话框中（ ⌘, ），在“ 版本控制”节点下，选择“ 确认”页面。

  2. 在“ 文件创建/删除”区域中，定义应如何在AppCode中创建或删除的文件添加到版本控制系统或从版本控制系统中删除。可用的选项有：

    • 在添加到版本控制之前显示选项

    • 静默添加

    • 不加

  3. 在“ 调用这些命令的显示选项对话框”区域中，指定要让AppCode显示“ 选项”对话框的命令。可用的选项有：

    • 查看

    • 状态

    • 获取最新版本

    • 更新资料

    • 撤消签出

  4. 通过选中或清除相应的复选框，配置用于更改列表的其他设置：

    • 指定文件的只读状态是否需要显式取消。

    • 指定是否需要关于将文件提交到存储库的有意义的注释。

    • 指定是否将未提交的更改移到另一个更改列表。

    • 指定在提交操作失败时是否以及如何创建更改列表。

指定要在后台运行的操作

您可以启用某些与版本控制相关的活动的后台执行。这些活动在“ 后台设置”页面中指定。

  1. 以下在“ 首选项”对话框中（ ⌘, ），在“ 版本控制”节点下，选择“ 后台”页面。

  2. 通过选中相应的复选框来启用必要操作的后台执行。可以为以下操作设置后台执行：

    • 更新资料

    • 承诺

    • 查看

    • 编辑/结帐

    • 添加和删除

    • 还原

    • 历史记录缓存处理

    • 检测“服务器上已更改”冲突

配置历史记录缓存处理

  1. 在“ 首选项”对话框中（ ⌘, ），在“ 版本控制”节点下，选择“ 后台”页面。

  2. 设置缓存范围。根据您使用的版本控制系统，执行以下操作之一：

    • 指定要存储在缓存中的最大更改列表数。

    • 指定将更改列表存储在缓存中的最大天数。

  3. 指定您是否希望版本控制系统以及多长时间（以分钟为单位）刷新缓存。

配置VCS特定的设置

某些设置适用于分配给整个项目或其目录的版本控制系统。其他与所选版本控制系统有关。使用“ 首选项”对话框的“ VCS”节点下的各个VCS页面可以定义特定于VCS的设置。

  1. 在“ 首选项”对话框中（ ⌘, ），单击“ 版本控制” ，然后单击您的VCS页面。

  2. 根据需要设置选项。有关详细信息，请参阅版本控制设置中特定于VCS的页面。
