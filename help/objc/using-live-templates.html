实时模板

通过使用实时模板 ，您可以将常用结构插入代码中。例如，循环，条件，各种声明或打印语句。

要展开代码段，请键入相应的模板缩写，然后按⇥ 。继续按⇥从模板中的一个变量跳转到下一个变量 。按⇧⇥移至上一个变量。

如果您不知道必要的缩写，请按⌘J打开当前上下文中可用的模板列表。

实时模板的类型

区分以下类型的实时模板：

  • 简单模板仅包含固定的纯文本。扩展简单模板时，文本将自动插入源代码中，以代替缩写。

  • 参数化的模板包含允许用户输入的变量 。展开参数化模板时，变量将由输入字段替换，以供用户手动指定，或者由AppCode自动计算。

  • 环绕模板用用户指定的文本包装所选代码块。例如， T扩展为一对标签，您可以为其指定名称。您也可以选择一个代码块，然后按⌥⌘J打开“ 选择模板”弹出窗口，然后选择T模板，以使用一对标签包装选择内容。

配置实时模板

要配置实时模板，请打开 编辑器| AppCode首选项的“ 实时模板”页面⌘, 。在“ 实时模板”页面上，您可以查看所有可用的实时模板，对其进行编辑并创建新模板 。

模板根据可以使用它们的上下文进行分组（通常是通过相应的语言）。要将模板移至另一个组，请右键单击该模板，选择Move ，然后选择所需的组名。

如果为活动模板设置了方言，则该活动模板仅在该方言中有效。

每个活动模板均由包含字母数字字符，点和连字符的缩写定义。缩写在一个组中必须唯一，但是相同的缩写可以在不同的组中使用，并根据相应组的上下文扩展到不同的构造。修改后的默认模板的缩写以蓝色字体显示在列表中。

将修改后的模板还原为默认设置

  • 在“ 首选项”对话框的“ 实时模板”页面上，右键单击要还原的模板，然后单击“还原默认值” 。

共享实时模板

AppCode将自定义实时模板组的定义以及添加到预定义模板组的模板存储在自动生成的XML配置文件中。

  • 对于自定义组，该文件包含该组包含的所有模板的定义。

  • 对于已修改的预定义组，该文件仅包含已添加（或已修改）实时模板的定义。

实时模板组配置文件存储在IDE配置目录 〜/ Library / Preferences / AppCode2019.2的templates文件夹中。

如果您的设置是通过“ IDE设置同步”插件同步的 ，则模板存储在IDE配置目录下的jba_config / templates下。

不要将实时模板配置文件复制到项目目录中。

要共享实时模板，您可以手动将必要的配置文件复制并粘贴到模板文件夹中。或者，您可以使用AppCode功能共享IDE设置，包括实时模板（ IDE设置同步插件， 设置存储库或将设置导出/导入为存档）。在“ 共享IDE设置”部分中了解更多信息。
