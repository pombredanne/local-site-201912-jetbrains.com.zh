配置配置文件

检查配置文件保留有关已启用检查，它们分析的文件范围以及严重性设置的信息。在AppCode中，有两种类型的配置文件。

  • IDE中存储的配置文件保存在AppCode 配置目录中 ，并且可用于任何项目。您可以使用“ 复制到项目”选项将IDE配置文件复制到当前项目。

  • 项目中存储的配置文件保存在.idea目录中，并且在一个项目中可用。您可以使用“ 复制到IDE”选项将项目配置文件复制到IDE级别。

AppCode带有两个您可以自定义或创建新类型的默认配置文件，每种类型（ Default和Project Default ）。您还可以共享，导入和导出检查配置文件。

建立一个新的个人资料

要创建新的配置文件，请复制默认配置文件之一并更改其设置。

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |检查 。

  2. 选择您要复制的配置文件，单击 ，然后选择复制 。

  3. 更改配置文件名称，然后按⏎ 。

恢复默认配置文件配置

您可以还原默认检查设置：默认启用的检查及其严重性级别。

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |检查 。

  2. 从列表中选择必要的配置文件，然后单击 ，然后选择恢复默认值 。

在计算机之间同步配置文件

项目中存储的配置文件与其他项目配置文件一起保存，并通过VCS共享。这就是为什么在您配置了新的项目配置文件并与团队中的其他成员共享之后，在他们从VCS签出项目后，该配置文件将自动传播到他们的项目中的原因。

如果要共享存储在IDE中的配置文件，则可以使用与AppCode捆绑在一起的Settings Repository插件。

在这种情况下，您必须将此配置文件配置为所有新创建项目的默认检查配置文件（使其全局）。该插件会将这种全局配置提升为连接到设置存储库的所有计算机。

共享存储在IDE中的配置文件

  1. 确保已配置设置存储库 。

  2. 从主菜单中，选择文件|其他设置|新建项目的设置 ，然后单击编辑器。检查 。

  3. 在“ 个人档案”列表中，选择要共享的个人档案，应用更改并关闭对话框。

  4. 从主菜单中，选择文件|设置存储库 ，然后单击覆盖远程以将新配置发送到存储库。

  5. 在连接到资源库的另一台计算机上，转到“ 文件” |“其他”。设置存储库，然后选择覆盖本地以下载并应用配置。

导出和导入配置文件

要共享配置文件，您也可以将其导出，然后将其导入另一台计算机。

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |检查 。

  2. 请点击然后选择导出 。

  3. 要导入配置文件，请单击然后选择导入配置文件 。

  4. 指定配置文件的路径，然后单击“ 打开” 。
