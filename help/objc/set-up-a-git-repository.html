设置一个Git仓库

从远程主机（克隆）检出项目

AppCode允许您检出（用Git术语clone ）一个现有存储库，并根据您下载的数据创建一个新项目。

  1. 从主菜单中，选择VCS |。从版本控制中检出| Git ，或者，如果当前未打开任何项目，请从“版本控制” |“选择签出”。在欢迎屏幕上Git 。

  2. 在“ 克隆存储库”对话框中，指定要克隆的远程存储库的URL（可以单击“ 测试”以确保可以建立到远程的连接）。

  3. 在目录字段中，指定将创建本地Git存储库的文件夹的路径，远程存储库将被克隆到该文件夹中。

  4. 单击克隆 。如果要基于已克隆的源创建AppCode项目，请在确认对话框中单击“ 是 ”。Git根映射将自动设置为项目根目录。

    如果您的项目包含子模块 ，它们也将被克隆并自动注册为项目根目录。

将现有项目置于Git版本控制下

除了克隆远程存储库之外 ，您还可以基于现有项目的源创建本地存储库。

将整个项目导入单个Git存储库

  1. 打开要放在Git下的项目。

  2. 从主菜单中，选择VCS |。启用版本控制集成 。

  3. 在打开的对话框中，从列表中选择Git ，然后单击确定 。

将不同的项目模块导入不同的Git存储库

  1. 打开要放在Git下的项目。

  2. 从主菜单中，选择VCS |。导入版本控制|创建Git存储库 。

  3. 在打开的对话框中，指定将在其中创建新Git存储库的目录。

Git不支持外部路径，因此，如果您选择项目根目录之外的目录，请确保要在其中创建存储库的文件夹也包含项目根目录。

将文件添加到本地存储库

为项目初始化Git存储库后 ，需要向其中添加项目数据。

打开版本控制工具窗口⌘9并切换到“ 本地更改”标签。

  • 通过按将任何文件放在版本控制下的“无版本文件” 更改列表中⌥⌘A或从上下文菜单中选择“ 添加到VCS ”。您可以添加整个更改列表，也可以选择单独的文件。

    您还可以从“ 项目”工具窗口将文件添加到本地Git存储库。选择要添加的文件，然后按⌥⌘A或选择Git |从上下文菜单中添加 。

如果您为项目启用了Git集成，AppCode建议在Git版本控制下添加每个新创建的文件（您可以在“ 设置 首选项”对话框中更改此行为⌘,在版本控制下|确认 ）。如果希望某些文件始终保持未版本控制，则可以将Git配置为忽略它们 。

从版本控制中排除文件（忽略）

有时您可能需要保留某些文件的版本。这些可以是VCS管理文件，实用程序，备份副本等。您可以通过AppCode忽略文件，IDE不会建议将它们添加到Git中，而是将其突出显示为被忽略。

Git允许您在VCS根目录及其子目录的.gitignore文件中列出忽略的文件模式。如果VCS根目录中没有.gitignore文件，则可以在“项目”窗口中的任何位置单击鼠标右键，然后选择“ 新建” |“新建”。在“ 新建文件”对话框中输入文件并键入.gitignore 。

请注意，将文件添加到Git版本控制后，忽略该文件将无效。您需要先将其从Git存储库中删除。

将文件添加到.gitignore

  1. 检查您的VCS根目录，以检查其中是否存在.gitignore文件。

    要创建.gitignore文件，可以在“项目”窗口中的任何位置上单击鼠标右键，然后选择“ 新建” |“新建”。在“ 新建文件”对话框中输入文件并键入.gitignore 。

  2. 在“ 项目”视图中找到要忽略的未版本化文件或文件夹，右键单击它并选择“ Git |添加到.gitignore 。

    也可以从版本控制工具窗口的“ 本地更改”视图中的文件的上下文菜单中执行此操作。

如果您需要按某种模式或某种类型的文件排除文件，则可以编辑.gitignore直接归档。请参阅.gitignore模式格式

检查项目状态

AppCode使您可以检查本地工作副本与项目存储库版本相比的状态。它使用特定的颜色让您查看哪些文件已被修改，哪些新文件已添加到VCS，以及哪些文件未被Git跟踪。

打开版本控制工具窗口⌘9并切换到“ 本地更改”标签：

  • 默认更改列表显示自上次与远程存储库同步以来已修改的所有文件（以蓝色突出显示），以及已添加到VCS但尚未提交的所有新文件（以绿色突出显示）。

  • “ 未版本控制的文件 ” 更改列表显示所有已添加到项目中但未被Git跟踪的文件。

如果合并期间存在尚未解决的冲突 ，则“ 合并冲突”节点将显示在相应的更改列表中，并带有一个链接来解决这些冲突 。

有关变更列表的更多信息，请参阅将变更分组为不同的变更列表 。

添加一个远程仓库

为了能够在您的Git项目上进行协作，您需要配置要从中获取数据并在需要共享工作时将其推送到的远程存储库。

如果您已从GitHub 克隆了一个远程Git存储库 ，例如该远程库是自动配置的，则在您要与之同步时（即执行拉或推操作时）不必指定它。Git给您从其克隆的遥控器的默认名称是origin 。

但是，如果您基于本地资源创建了一个Git存储库 ，则需要添加一个远程存储库，以使其他贡献者能够将其更改推送到该存储库，并能够共享您的工作结果。

定义一个遥控器

  1. 在任何Git托管（例如Bitbucket或GitHub）上创建一个空的存储库。

    您可以在GitHub上创建存储库而无需离开AppCode：请参阅在GitHub上共享项目 。

  2. 准备好通过选择VCS | Push提交时调用Push对话框。 Git |从主菜单，或按推 ⇧⌘K 。

  3. 如果您到目前为止尚未添加任何远程，则将显示“ 定义远程”链接，而不是远程名称。单击它添加一个遥控器。

  4. 在打开的对话框中，指定远程名称和将在其上托管的URL，然后单击“ 确定” 。

在某些情况下，您还需要添加另一个远程存储库。例如，如果您克隆了一个没有写访问权限的存储库，并且打算将更改推送到原始项目的自己的分支中，则这可能很有用。另一个常见的情况是，您已经克隆了自己的存储库，该存储库是其他人的项目分支，并且您需要与原始项目同步并从中获取更改。

添加第二个遥控器

  1. 从主菜单中，选择VCS |。 Git |遥控器 。Git Remotes对话框将打开。

  2. 点击添加 工具栏上的按钮或按⌘N 。

  3. 在打开的对话框中，指定远程名称和URL，然后单击“ 确定” 。


要编辑一个遥控器（例如，要更改您克隆的原始项目的名称），请在“ Git遥控器”对话框中选择它，然后单击“ 编辑”。 工具栏上的按钮，或按⏎ 。

要删除不再有效的存储库，请在Git Remotes对话框中选择它，然后单击删除。 工具栏上的按钮，或按⌘⌦ 。
