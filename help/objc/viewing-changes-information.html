查看变更

本主题说明如何跟踪您和您的团队成员对源代码所做的更改。

回顾项目历史

AppCode允许您查看对与指定过滤器匹配的项目源所做的所有更改。

对于分布式版本控制系统（例如Git和Mercurial），您可以在“ 版本控制”工具窗口的“ 日志”选项卡中查看项目历史记录 （请参阅调查更改） 。

对于集中的版本控制系统，例如Subversion，Perforce和ClearCase，项目历史记录可在“ 版本控制”工具窗口的“ 存储库”选项卡中找到。

在编辑器中跟踪对文件的更改

修改受版本控制的文件时，所有更改都会在编辑器中突出显示，并带有更改标记 ，这些更改标记显示在修改行旁边的左装订线中，并显示自上次与存储库同步以来引入的更改类型。将修改后的文件提交到存储库后，更改标记消失。

您对文本所做的更改使用颜色编码：

  • 行已添加。

  • 线变了。

要为线路状态自定义默认颜色， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |配色方案| VCS 。

删除行时，以下标记将出现在左装订线中： 。

您可以使用专用工具栏管理更改。要调用它，请将鼠标光标悬停在更改标记上，然后单击它。显示工具栏以及显示修改后行的先前内容的框架：

工具栏中的操作使您可以导航到下一个或上一个更改，回滚更改，查看当前版本与存储库版本之间的差异，将修改后的行的先前版本复制到剪贴板，或打开突出显示代码中的差异。

如果您要关闭突出显示的更改， 在“ 首选项”对话框中（ ⌘, ）选择编辑器|左侧的“ 常规 ”，然后取消选中“ 高亮显示装订线”选项。

将本地更改与存储库版本进行比较

除了导航通过在编辑器文件中的本地更改，您可以查看这些变化相对于文件的基础版本。

您可以通过以下方式之一查看更改：

  • 在“ 本地更改”视图中预览更改：选择一个文件，然后单击“ 预览差异”按钮在工具栏上。

  • 探索以下内容的更改：选择一个文件，然后按⌘D或点击在工具栏上。

左窗格显示受影响的代码，与基本版本中的代码相同，右页面显示更改后的受影响的代码。

使用工具栏按钮和控件在更改之间导航，并配置“ 更改详细信息”窗格或“ 差异查看器”的外观：

查看器类型

使用此列表选择查看器类型。并排查看器有两个面板。统一查看器只有一个面板。

两种类型的查看器都使您能够

  • 编辑代码。请注意， 只能在默认查看器的右侧部分更改文本，或者如果使用统一查看器，则可以在下部（“之后”）行（即文件的本地版本）更改文本。

  • 执行“ 应用/添加/还原”操作。

空格

使用此列表定义差异查看器应如何对待空白。

  • 不要忽略 ：空格很重要，所有差异都突出显示。默认情况下选中此选项。

  • 修剪空格 ： ("\t", " ") ，如果它们出现在行的结尾和开头。

    • 如果只有两行在尾随空白处不同，则认为这些行相等。

    • 如果两行不同，则在按单词模式下不会突出显示这些尾随空格。

  • 忽略空格 ：无论空格在源代码中的位置如何，空格都不重要。

  • 忽略空格和空行 ：将忽略以下实体：

    • 所有空格（如“忽略空格”选项中所述）

    • 所有添加或删除的仅由空格组成的行

    • 所有包含分割线或连接线的更改，但不更改非空白部分。
      例如，改变a b c至a \n b c在此模式下不突出显示。

高亮模式

选择突出显示差异粒度的方式。

可用的选项有：

  • 突出显示单词 ：修改后的单词突出显示

  • 高亮线 ：修改后的线高亮显示

  • 突出显示拆分更改 ：如果选择此选项，则将较大的更改拆分为较小的“原子”更改。
    例如， A \n B与A X \n B X将被视为两次更改，而不是一次更改。

  • 突出显示符号 ：修改后的符号将突出显示

  • 不突出显示 ：如果选择了此选项，则根本不会突出显示差异。此选项用于经过重大修改的文件，其中突出显示仅会带来其他困难。

收起未变的片段

单击此按钮可以折叠两个文件中所有未更改的片段。不可折叠的不变行数可以在“差异和合并”设置页面中配置。

同步滚动

单击此按钮以同时滚动两个差异窗格。如果释放此按钮，则每个窗格都可以独立滚动。

设定值

单击此按钮以调用可用设置列表。选择或清除这些选项可显示或隐藏空格，行号和缩进参考线，以使用或禁用软包装，以及设置突出显示级别。

这些命令也可从差异查看器装订线的上下文菜单中获得。

在外部工具中显示差异

单击此按钮可以调用在“外部差异工具设置”页面中指定的外部差异查看器。

仅在“ 外部差异 工具”设置页面中启用了“ 使用外部差异工具”选项时，此按钮才会出现在工具栏上。

您可以通过突出显示文件名的颜色来检查任何界面元素（例如，编辑器或各种工具窗口）中文件的状态。

您可以在“ 颜色和字体设置”页面中自定义文件状态的默认颜色。

使用注释

什么是VCS注释？

注释是一种文件表示形式，显示每行代码的详细信息。特别是，对于每一行，您可以看到此行的来源版本，提交此行的人员的用户ID以及提交日期。带注释的视图可帮助您找出谁在何时何地进行操作，并追溯所做的更改。

代码注释行可用于ClearCase，Mercurial，Git，Perforce和Subversion。

“ 注释”命令可从“版本控制”菜单的特定于VCS的节点，“编辑器”左侧装订线的上下文菜单，文件上下文菜单以及文件历史记录视图中获得。

启用注释

  • 在编辑器或“ 差异查看器”中的左装订线上单击鼠标右键，然后从上下文菜单中选择“ 注释 ”。

    您可以为“ 注释”命令分配自定义快捷方式： 在“ 首选项”对话框中（ ⌘, ），选择左侧的Keymap ，然后转到“ 版本控制| Git |注释 。

配置注释中显示的信息量

您可以选择要在注释视图中查看的信息量。

  • 右键单击注释装订线，选择“ 查看”，然后选择要查看的信息类型，包括此更改的发起版本，日期，不同格式的作者姓名以及提交编号。

    您还可以在颜色下设置突出显示。

配置注释选项

  • 右键单击注释装订线，然后从上下文菜单中选择“ 选项 ”：

    • 忽略空格 ：空格将被忽略（git blame -w ）。这意味着注释将指向先前的有意义的提交。

    • 检测文件内的移动：当提交在同一文件内移动或复制行时，此类更改将被忽略（git blame -M ）。这意味着注释将指向先前的有意义的提交。

    • 检测跨文件移动：当一次提交移动或从同一提交中修改的其他文件复制行时，此类更改将被忽略（git blame -C ）。这意味着注释将指向先前的有意义的提交。

注释以前的修订

AppCode使您不仅可以注释当前文件修订版，还可以注释以前的修订版。注释装订线的上下文菜单中提供以下选项：

  • 批注修订 ：如果要在提交特定更改后检查文件的外观，此选项很有用。为此，请右键单击此更改，然后从上下文菜单中选择“ 注释修订 ”。

  • 注释以前的修订版 ：如果您发现自己在特定行中的最后更改无意义的情况下（例如，如果所有更改都是代码格式的），则此选项很有用。在这种情况下，您可以检查文件的先前版本。为此，请右键单击更改，然后从上下文菜单中选择“ 注释以前的版本 ”。

您也可以从文件历史记录视图中注释特定文件。在“历史记录”选项卡中，选择要查看的文件版本，右键单击相应的行，然后从上下文菜单中选择“ 注释 ”。

查看版本之间的差异

要查看文件的带注释的版本与其先前版本之间的区别，请将插入号放置在注释上，右键单击它，然后从上下文菜单中选择“ 显示差异”。AppCode将打开文件差异查看器 ：

您也可以调用VCS Operations弹出窗口⌃V然后选择带注释的线|显示差异 。

导航到日志

如果使用Git进行版本控制，还可以从“注释”视图跳转到“ 版本控制”工具窗口的“ 日志”选项卡中的相应提交。

为此，将插入号放置在注释上，右键单击它，然后从上下文菜单中选择“在Git日志中选择”。您还可以使用“ 复制修订号”命令在日志中找到修订。

对于托管在https://github.com/上的项目，还可以使用“ 在GitHub上打开”命令将您带到相应的提交。
