使用JetBrains Chrome扩展程序进行调试

您不再需要安装和使用适用于Chrome的JetBrains IDE支持扩展来在AppCode中调试应用程序。但是，如果要在同一Chrome实例中启动调试会话，则仍然可以使用它。默认情况下，AppCode将使用自定义的Chrome用户数据打开一个新窗口，请参阅使用默认的Chrome用户数据启动调试会话 。

在使用JetBrains Chrome扩展程序进行调试的过程中，可以使用HTML，CSS和JavaScript进行实时编辑 。您还可以查看DOM树和实际页面的源代码。

安装JetBrains Chrome扩展程序

  1. 在Chrome网上应用店中打开JetBrains IDE支持页面。

  2. 点击添加到Chrome按钮 ，然后在打开的“ 确认新扩展名”对话框中单击“ 添加” 。“ 添加到Chrome浏览器”按钮更改为“ 添加到Chrome浏览器” 。

使用JetBrains Chrome扩展程序启用实时编辑和调试

  1. 在“ 首选项”对话框中（ ⌘, ），在“ 生成，执行，部署”下单击“ 调试器 ”，然后单击“ 实时编辑” 。将打开“ 实时编辑”页面。

  2. 选中“ 使用JetBrains IDE支持扩展进行调试”和“实时编辑”复选框。

激活，停用和卸载JetBrains Chrome扩展程序

  • chrome：// extensions页面提供了对JetBrains Chrome扩展程序的控制。 要打开页面，只需键入chrome://extensions在Chrome地址栏中。 或者，点击自定义和控制Google Chrome （ ），从上下文菜单中选择设置 ，然后在打开的chrome：// settings页面上单击扩展 。

  • 要禁用扩展，请清除“已启用”复选框。复选框名称更改为Enable 。

  • 要激活扩展，请选择启用复选框。

  • 要卸载扩展程序，请点击从Chrome删除按钮 。

在Chrome扩展程序选项中指定实际的AppCode端口

在调试会话期间， Chrome扩展程序会侦听从中调用扩展程序的JetBrains IDE的端口。每个包含AppCode的IDE都有其自己的默认端口，该端口可在其上启动。

如果由于某种原因默认的AppCode端口已经繁忙，则AppCode会找到最近的可用端口并在其上启动。这导致冲突：AppCode在“新”端口上运行，而Chrome扩展程序仍在侦听以前启动的产品的端口。

当您使用Live Edit启动调试会话时，冲突就会显示出来：扩展名无法通过默认端口连接，AppCode等待扩展名的连接，并显示以下消息及其实际运行的端口号（例如， 当前端口） 63342 ）：

  1. 右键点击并在上下文菜单上选择选项 。将打开一个带有Chrome扩展程序选项的网页，其中显示了连接到AppCode的参数。

  2. 在“ IDE连接”区域中，在“ 端口”微调框中指定实际的AppCode端口。

覆盖默认的CORS设置

假设您正在使用Live Edit调试的页面请求的资源出于安全原因而受到保护，以防止通过CORS设置进行访问。您可以通过更改Chrome扩展程序选项来启用对受保护资源的访问。

  1. 右键点击并在上下文菜单上选择选项 。将打开一个带有Chrome扩展程序选项的网页，其中显示了连接到AppCode的参数。

  2. 在“ 强制CORS”文本框中，键入用于定义要访问的URL地址的模式，例如：http: //youtrack.jetbrains.com/rest/* 。
