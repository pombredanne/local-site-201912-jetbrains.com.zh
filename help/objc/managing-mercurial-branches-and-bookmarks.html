管理水银分行和书签

使用AppCode，您可以同时使用命名分支和轻量分支（书签） 。AppCode提供了用于在分支和书签之间创建，合并和切换的界面，请参阅在工作目录之间切换 。您也可以在终端仿真器中运行命令。

在此页：

  • 打开分支列表
  • 创建一个命名分支
  • 创建一个书签
  • 关闭分支
  • 合并命名的分支和书签
  • 将命名分支或书签与另一个命名分支或书签合并
  • 将命名分支或书签与变更集合并

有关在分支和书签之间切换的信息，请参见在工作目录之间切换 。

打开分支列表

大多数带有分支和书签的操作都是从“ 分支”列表中调用的。

该列表显示了项目根目录下的所有Mercurial存储库，所有已命名分支以及其中的所有书签。当前存储库和当前书签均带有刻度。当前命名分支的名称显示在状态栏上的专用hg区域中：

要打开“ 分支”列表，请执行以下任一操作：

  • 在状态栏上，单击专用hg区域中当前命名分支的名称。

  • 从主菜单中，选择VCS |。水银|分支机构 。

  • 从编辑器或版本控制工具窗口的上下文菜单中，选择“ Mercurial” |“ 版本控制” 。分支机构 。

创建一个命名分支

新分支立即变为活动状态，其名称显示在hg区域的状态栏上。

  1. 在“ 分支”列表中，单击“ 新建分支” 。

  2. 在打开的“ 创建新分支”对话框中，指定新分支的名称。

创建一个书签

  1. 在“ 分支”列表中，单击“ 新建书签” 。

  2. 在打开的中，指定要创建的书签的名称。

  3. 指定是否要立即切换到新书签。

    • 要激活新书签并由此启用跟踪和更新书签标识的轻量级分支，请清除“不活动”复选框。新书签将立即变为活动状态，并且其名称在“ 分支”列表中用勾号标记。

    • 要创建一个非活动书签，即保留在当前的轻量级分支（书签）或命名分支中，并在以后切换到新书签，请选中“ 非活动”复选框。

关闭分支

根据Mercurial工作流程 ，当您完成功能开发并且不希望有任何进一步的更改时，请关闭相应的分支。在活动分支之间，在“ 日志”视图等中 ，不会显示关闭的分支。要关闭分支，请执行以下操作：

  1. 在“ 分支”弹出窗口中，单击“ 关闭分支” 。将显示“ 提交更改”对话框 。

  2. 点击提交并关闭 。所有更改将被提交，并且当前分支将被关闭。

请注意，如果您在“ 存储库”部分中列出了多个存储库 ，则相应的菜单选项将切换为“ 关闭分支” ，并且close操作将应用于所有这些。

合并命名的分支和书签

您可以将命名分支或书签与另一个命名分支，另一个书签或由标签或修订号标识的特定变更集合并。

  • 将命名分支或书签与另一个命名分支或书签合并意味着合并其头部 。

    与命名分支和书签的合并可以通过菜单项VCS |调用。水银|合并 ，从“ 分支”列表中打开特定于Mercurial的“ 合并”对话框。

  • 将命名分支或书签与变更集合并意味着将分支头与指定的变更集合并。变更集可以通过修订号或标签来标识，请参见https://www.mercurial-scm.org/wiki/Tag 。

    只能通过VCS |调用命名分支或具有特定变更集的书签。水银|合并 。

有关合并操作本身的定义和特定于Mercurial的详细信息，请参阅https://www.mercurial-scm.org/wiki/Merge 。

默认情况下， Mercurial要求在合并之前，当前工作目录应该是干净的 ，也就是说，它不应包含任何未提交的更改。否则，合并操作将失败，并且AppCode显示相应的错误消息。该消息还建议您通过运行以下命令来清理当前工作目录hg merge -C放弃未提交的更改。

如果您当前的工作副本不干净，则可以按照搁置和取消搁置中的说明提交更改或搁置它们。

将命名分支或书签与另一个命名分支或书签合并

将命名分支或书签与另一个命名分支或书签合并意味着合并其头部 。

  1. 确保您当前的工作目录为clean ，即它不包含任何未提交的更改。提交或搁置更改（如果有）。

  2. 通过执行以下任一操作来调用合并：

    • 在“ 分支”列表中，单击要与之合并的分支或书签的名称，然后从列表中选择“ 合并 ”：

    • 选择VCS |水银|合并主菜单或Mercurial |从编辑器的上下文菜单合并 。

      在打开的“ 合并”对话框中：

      1. 从“ 存储库”列表中选择目标存储库，其中显示了当前项目根目录下所有可用的Mercurial存储库。

      2. 选择“ 分支”或“ 书签”选项，然后选择与当前工作目录合并的命名分支或书签。

  3. 解决冲突。一旦发生冲突，“ 冲突”对话框就会打开，并显示冲突文件列表。使用对话框的控件来解决问题：

    • 要保留当前工作目录的版本，请点击接受您的 。

    • 要保留您要合并的分支的版本，请点击接受他们的 。

    • 要手动解决冲突，请按照解决冲突中所述单击合并并使用冲突解决工具 。

    如果在合并过程中没有冲突，操作将以静默方式通过，并且合并日志将显示在“ 版本控制”工具窗口中。

将命名分支或书签与变更集合并

将命名分支或书签与变更集合并意味着将分支头与指定的变更集合并。变更集可以通过修订号或标签来标识，请参见https://www.mercurial-scm.org/wiki/Tag 。

  1. 确保您当前的工作目录为clean ，即它不包含任何未提交的更改。提交或搁置更改（如果有）。

  2. 选择VCS |水银|合并主菜单或Mercurial |从编辑器的上下文菜单合并 。

  3. 在打开的“ 合并”对话框中：

    1. 从“ 存储库”列表中选择目标存储库，其中显示了当前项目根目录下所有可用的Mercurial存储库。

    2. 选择“ 标记”或“ 修订”选项，然后选择标记或指定哈希或修订号以与当前工作目录合并。要复制哈希，请打开“ 版本控制”工具窗口的“ 日志”选项卡，选择相关的分支和修订，然后从所选内容的上下文菜单中选择“ 复制哈希 ”。

  4. 解决冲突。一旦发生冲突，“ 冲突”对话框就会打开，并显示冲突文件列表。使用对话框的控件来解决问题：

    • 要保留当前工作目录的版本，请点击接受您的 。

    • 要保留您要合并的分支的版本，请点击接受他们的 。

    • 要手动解决冲突，请按照解决冲突中所述单击合并并使用冲突解决工具 。

    如果在合并过程中没有冲突，操作将以静默方式通过，并且合并日志将显示在“ 版本控制”工具窗口中。
