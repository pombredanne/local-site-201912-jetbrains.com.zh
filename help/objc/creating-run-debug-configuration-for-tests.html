创建测试的运行/调试配置

您可以使用运行/调试配置来运行测试，方式类似于运行普通应用程序。

除了创建常规测试配置外 ，AppCode还支持为特定目标创建测试配置 ，例如，为文件，类或目录中的所有测试，或为任意选择的测试类和方法创建测试配置 。

测试运行配置与Xcode的方案同步。它们中的所有更改都会自动传播。

创建测试配置

  1. 从主菜单中，选择运行|编辑配置 。

  2. 请点击在工具栏上，然后选择可用的测试框架之一：Boost。测试，捕获，Google测试或XCTest。

    要为基于XCTest的框架（例如Quick和Kiwi）创建配置，请选择XCTest模板。

  3. 在打开的对话框中，您可以更改以下配置：

    • 在“ 名称”字段中，更改运行/调试配置的名称。默认名称包括所选的类/方法和测试目标。

    • 选择测试种类。

      根据框架，可以使用以下测试类型：

      • XCTest：类/方法或模式。

      • 促进。测试和Google测试：套件/测试或模式。

      • 捕获：标签/测试或模式。

    • 如果需要选择特定的测试类或方法，请在“ 类”字段中输入测试类名称。AppCode将在您键入时建议您使用可用的类。选择一个类后，可以开始在“ 方法”字段中键入此类的方法名称，然后选择建议的变体之一。

      如果未指定任何方法/类，则AppCode将运行所有可用的类/方法。

    • 在“ 目标”字段中，已预先选择了测试目标。如有必要，请进行更改。

    • 在配置字段中，可以选择配置类型-调试或发布。

    • 如有必要，请指定工作目录， 并在相应的字段中添加程序参数和环境变量 。

    • 在运行之前部分，您可以定义在运行之前是要启动任何外部工具还是要运行其他运行/调试配置。

为特定文件创建测试配置

  1. 在“ 项目”工具窗口中，右键单击包含测试类的所需文件，然后选择“ 创建所有” 。

  2. 在打开的“运行/调试配置”对话框中，指定要在测试执行之前执行的配置参数和活动。应用更改并关闭对话框。

为特定的类或方法创建测试配置

  1. 在编辑器中，打开包含测试类的文件。

  2. 右键单击类或方法的名称，然后选择Create'All in ' 。

  3. 在打开的“运行/调试配置”对话框中，指定要在测试执行之前执行的配置参数和活动。应用更改并关闭对话框。
