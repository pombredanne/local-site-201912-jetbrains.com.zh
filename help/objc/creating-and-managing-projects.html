管理你的项目

项目是代表完整软件解决方案的组织单位。它用作编码辅助，批量重构，编码样式一致性等的基础。

AppCode使用Xcode项目格式 ，该格式允许您在AppCode中打开现有Xcode项目并创建与Xcode兼容的新项目。所有Xcode项目模板在AppCode中都可用。

创建一个新项目

  1. 如果当前在AppCode中没有打开任何项目，请在“ 欢迎”屏幕上单击“ 新建项目 ”。否则，选择文件|从主菜单中新建项目 。

  2. 在打开的对话框中，选择目标和项目模板，然后单击下一步 ：

  3. 在下一页上，填写以下字段：

    • 产品名称 ：您的项目的名称，也将是您的应用程序的名称。

    • 组织名称 ：您的公司名称或您的名称。

    • Organization Identifier ：标识您公司的字符串，通常为反向DNS格式。

    在捆绑包标识符字段中，您可以看到一个自动生成的字符串，它将在操作系统中标识您的应用程序。

    从语言列表中，选择编程语言。

    如果要将测试目标添加到项目中，请选择“ 包括单元测试”和“ 包括UI测试” 。如果要使用Core Data框架，请选择Use Code Data 。

    点击完成 。

公开项目

AppCode使您可以在不同的窗口中同时打开多个项目。默认情况下，每次打开一个项目同时打开另一个项目时，AppCode都会提示您选择在同一窗口还是在新窗口中打开项目。您可以选择“ 记住，不要再询问”复选框，以将所需的行为应用于所有项目。如有必要，您可以稍后更改此行为 。

打开一个项目

  1. 单击“ 欢迎”屏幕上的“ 打开 ”，或单击“ 文件” |“ 打开” 。从主菜单打开 。

  2. 导航到所需的.xcodeproj目录。

  3. 在出现的对话框中，选择包含要打开的项目的目录。

  4. 指定是要在新窗口中打开项目，还是要关闭当前项目并重用现有窗口。

配置项目开始

  1. 在“ 首选项”对话框中（ ⌘, ），转到外观和行为|系统设置|项目开幕 。

  2. 选择以下选项之一：

    • 在新窗口中打开项目 ：每次打开新项目时，打开一个新的AppCode窗口。

    • 在同一窗口中打开项目 ：留在同一AppCode窗口中。

    • 确认窗口以打开项目 ：保留默认行为，并显示一个对话框以选择如何打开每个新项目。

重新打开最近的项目

AppCode保留最近项目的历史记录列表，您可以从中选择所需的项目。

启动AppCode时，默认情况下会重新打开最新的项目（除非禁用此选项在“ 首选项”对话框中（ ⌘, ）。

重新打开最近的项目

  • 选择文件|从主菜单中打开“最近的” ，然后从列表中选择所需的项目。

  • 在“ 欢迎”屏幕上 ，在左侧窗格中单击感兴趣的项目，在该窗格中会显示您最近的项目的列表。开始在“欢迎”屏幕中键入内容以过滤最近的项目列表。

从最近的项目列表中删除一个项目

  • 选择文件|打开最近|从主菜单中管理项目 ，然后从列表中选择所需的项目。

  • 单击要删除的项目的关闭图标。

    所选项目将从最近的项目列表中删除，并且不会出现在“ 欢迎”屏幕上 。

要仅从“ 欢迎屏幕”上的列表中删除项目，请右键单击目标项目，然后选择“从欢迎屏幕中删除所选项目” 。

结束项目

关闭唯一打开的项目时，将显示“ 欢迎”屏幕 。如果有多个项目，则每个项目都用其窗口关闭。

关闭项目

  • 从主菜单中，选择文件|关闭项目 。

项目文件

AppCode中的项目以基于目录的格式表示 。项目目录标有图标。

该项目目录包含.idea目录，其中包含以下文件：

  • 描述项目结构的.iml文件。

  • 包含您的工作空间首选项的workspace.xml文件。

  • 许多xml文件。每个xml文件负责自己的一组设置，这些设置可以通过其名称来识别： projectCodeStyle.xml ， encodings.xml ， vcs.xml等等。

    因此，例如，添加新的运行/调试配置和更改编码将影响两个不同的xml文件。当项目设置存储在版本控制系统中并由不同团队成员修改时，这有助于避免合并冲突。

.idea目录中的所有设置文件都应置于版本控制下 ，而workspace.xml除外，该文件存储您的本地首选项。VCS应该将workspace.xml文件标记为已忽略 。

.idea目录在“ 项目 ”工具窗口的“ 项目”视图中不可见⌘1 。
