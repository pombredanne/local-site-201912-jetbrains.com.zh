Create and save temporary run/debug configurations

A temporary configuration is created every time you run test classes or methods from the context menu or using the gutter icons or . The new configuration appears greyed out in the run/debug Configuration selector when the run or debug session is over.

By default, there can be created only five temporary configurations per project, so when you create the sixth configuration, the one created first gets removed, and so on. You can change this limit in the Temporary configurations limit field in Edit Configurations | Templates.

Temporary configurations cannot be viewed and edited in XCode. You can save a temporary configuration to make it available among other XCode schemes.

Save a temporary run/debug configuration

  • In the Run/Debug Configuration selector, choose Save <configuration name>.

  • In the Edit Configurations dialog, select the temporary configuration in the list, and click .

  • From the context menu of the editor or Project tool window, choose Save <configuration name>.
