运行/调试配置：复合

运行编辑配置| |复合

使用此对话框可以创建包含多个运行/调试配置的配置，您可以立即启动它们。例如，这在您要启动各种自动化测试时很有用。每个配置将在“ 运行”或“ 调试”工具窗口的单独选项卡中提供。

要顺序启动配置，请将第一个配置添加为启动前任务。例如，您可以将Node.js运行/调试配置添加到JavaScript调试配置中。

请点击选择要包含在“复合”配置中的现有配置中的哪些，并填写以下字段：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 ⌃⌥R 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。
