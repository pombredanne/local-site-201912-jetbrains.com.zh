重新格式化和重新排列代码

AppCode使您可以根据在代码样式设置中指定的要求重新格式化代码。

要访问设置， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |代码样式 。

您可以重新格式化一部分代码，整个文件，文件组，目录和模块。您也可以从重新格式化中排除部分代码或某些文件。

重新格式化文件中的代码片段

  1. 在编辑器中，选择要重新格式化的代码片段。

  2. 在主菜单中，选择代码|重新格式化 ⌥⌘L 。

    如果您不选择代码片段，AppCode将重新格式化整个文件。

重新格式化文件

  1. 在“ 项目”工具窗口中，右键单击文件，然后选择“ 重新设置代码格式” 。

    继续按⇧并选择其他文件以进行组重新格式化。

  2. 如果需要，在打开的对话框中，选择以下重新格式化选项：

    • 优化导入 ：如果要删除未使用的导入，添加丢失的导入或组织导入语句，请选择此选项。

      有关更多信息，请参阅优化导入部分。

    • 重新排列条目 ：如果您需要根据代码样式设置中指定的排列规则重新排列代码，请选择此选项。

      您可以为JavaScript，TypeScript，HTML，XML，CSS和SCSS设置排列规则。

    • 清理代码 ：选择此选项可运行代码清理检查。

    单击确定 。

重新格式化目录中的代码

  1. 在项目视图中，右键单击目录，然后从上下文菜单中选择“ 重新设置代码格式”或按⌥⌘L 。

  2. 在打开的对话框中，指定必要的选项，然后单击“ 确定” 。

重新格式化行缩进

您可以根据指定的设置重新格式化行缩进。

  1. 在编辑器中，选择必要的代码片段，然后按⌃⌥I 。

  2. 如果您需要调整缩进设置， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |代码样式 。

  3. 在适当的语言页面上的“ 选项卡和缩进”选项卡上，指定适当的缩进选项，然后单击“ 确定” 。

在某些情况下，“ 文件” | “ 缩进检测”部分中的“ 检测并使用现有文件缩进进行编辑 ”选项。首选项|编辑器代码样式可以覆盖您的设置。在这种情况下，AppCode将显示通知。

从重新格式化中排除代码或文件

您可以从重新格式化中排除一组文件或部分代码。

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |代码样式 。

  2. 在“ 格式化程序控件”选项卡上，选中“ 在注释中启用格式化程序标记”复选框。范围区域变为活动状态

  3. 在“ 范围”区域中，单击添加一个范围 ，您可以在其中指定要从重新格式化中排除的文件。

如果您尝试重新格式化排除文件的格式，AppCode将显示一个弹出窗口，通知您该文件的格式已禁用。如果需要，请单击弹出窗口中的链接以打开“ 代码样式设置”页面并更改排除范围。

在编辑器中排除重新格式化的代码片段

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |代码样式，然后在“ 格式器控件”选项卡上选择“ 在注释中启用格式器标记”复选框。

  2. 在编辑器中，要排除的区域的开头，创建行注释⌘/和类型//@formatter:off ，在该区域的末尾，再次创建行注释并输入//@formatter:on 。

保留现有格式

您可以选择重新设置代码格式时将被忽略的格式设置规则。例如，您可以调整IDE以将简单的方法和函数放在一行中，而通常在代码重新格式化后它们会扩展为多行。

  1. 转到首选项|编辑器代码样式 ，选择您的编程语言，然后打开包装和括号选项卡。

  2. 在重新格式化时保留部分中，选择要忽略的格式设置规则，然后取消选择应应用的格式设置规则。

  3. 重新格式化您的代码 （ ⌥⌘L ）。

AppCode将根据当前样式设置重新格式化您的代码，并保持所选规则的现有格式。

重新排列代码项

您可以根据在“首选项” |“设置”中的“ 排列”选项卡上设置的排列规则来重新排列代码。编辑器代码样式 。

您可以为JavaScript，TypeScript，HTML，XML，CSS和SCSS设置排列规则。

例如，如果您需要按字母顺序对代码条目进行排序，请选择适当的匹配规则条目，并将“ 订单”字段设置为“按名称排序” 。

  • 在编辑器中，选择要重新排列的代码条目，然后从主菜单中选择Code |重新排列代码 。
