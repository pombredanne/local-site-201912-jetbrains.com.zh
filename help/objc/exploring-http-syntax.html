使用编辑器语法探索HTTP请求

要在AppCode代码编辑器中编写HTTP请求，请使用以下常规语法：

方法Request-URI HTTP版本的标头字段：标头值Request-Body

编辑器格式的HTTP请求引入了其他功能，如以下示例所示。有关使用HTTP请求的详细信息，请参阅AppCode代码编辑器中的HTTP客户端 。

在HTTP请求中使用注释

  • 在请求中，以开头的任何一行//要么#使其成为注释行。

    //基本请求GET http://example.com/a/

对GET请求使用简短格式

  • 对于GET请求，您可以省略请求方法，而仅指定URI。

    //基本请求http://example.com/a/

在一个文件中编写多个请求

  1. 输入以下命令，标记请求结束###它下面的分隔符。

    //基本请求http://example.com/a/ ###

  2. 在分隔符下方撰写另一个请求。

    //基本请求http://example.com/a/ ### //使用GET方法的第二个请求http://example.com:8080/api/html/get?id=123&value=content

将长请求分成几行

  • 缩进除第一行外的所有查询字符串行。

    //使用带有缩进GET的换行符http://example.com:8080 / api / html / get？id = 123＆value = content

通过身份验证访问Web服务

  • 根据您访问的Web服务，提供基本或摘要 Authorization标头。

    //基本身份验证GET http://example.com授权：基本用户名密码### //摘要身份验证GET http://example.com授权：摘要用户名密码

与其他HTTP请求的元素类似， username和password可以通过环境变量进行参数化。

提供请求消息正文

在请求内部，在请求正文之前添加一个空行，然后执行以下操作之一：

  • 在适当的位置键入请求正文：

    //请求正文在适当的位置提供POST http://example.com:8080/api/html/post HTTP / 1.1内容类型：application / json Cookie：key = first-value {“ key”：“ value” ，“列表”：[1、2、3]}

    如果将Content-Type标头字段值设置为AppCode支持的一种语言 ，则相应的语言片段将自动注入到HTTP请求消息主体中。如果未指定Content-Type，则可以手动插入语言片段。有关更多信息，请参见语言注入 。

    要从文件中读取请求正文，请键入<符号，后跟文件路径。

    //从文件POST中读取请求正文POST http://example.com:8080/api/html/post内容类型：application / json <./input.json

使用多部分/表单数据内容类型

  • 将请求的Content-Type设置为multipart / form-data 。要将文件作为multipart / form-data消息的一部分发送，请包括filename Content-Disposition标头中的参数。

    POST http://example.com/api/upload HTTP / 1.1 Content-Type：multipart / form-data; boundary = boundary --boundary Content-Disposition：表单数据； name =“ first”; filename =“ input.txt” //'input.txt'文件将被上传<./input.txt --boundary Content-Disposition：form-data; name =“ second”; filename =“ input-second.txt” //将创建一个带有'Text'内容的临时'input-second.txt'文件并上传Text --boundary Content-Disposition：form-data; name =“第三”; //'input.txt'文件的内容将以纯文本格式发送。<./input.txt --boundary--

要加快创建multipart / form-data请求的速度，请使用mptr live模板 。

启用或禁用以下重定向

根据您正在访问的Web服务，您可能希望HTTP请求遵循重定向或不遵循重定向。当遵循重定向时，将返回重定向的页面响应；否则，返回实际的重定向响应头（例如301或302 ）。

  • 在提出要求之前，请在@no-redirect标签。

    // @ no-redirect example.com/status/301

启用或禁用将请求保存到请求历史记录

如有必要，可以防止将请求保存到请求历史记录中 。如果请求包含一些敏感数据，并且您不想对其进行记录，这将很有帮助。

  • 在提出要求之前，请在@no-log标签。

    // @ no-log GET example.com/api

启用或禁用将收到的cookie保存到cookie jar

如有必要，可以防止将收到的cookie保存到cookie jar中 。这样，您将避免手动从http-client.cookies文件中删除不需要的cookie。

  • 在提出要求之前，请在@no-cookie-jar标签。

    // @ no-cookie-jar GET example.com/api
