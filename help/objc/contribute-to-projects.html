参与项目

如果您想为托管在GitHub上的项目做出贡献，并且不能直接推送到该项目，请遵循以下工作流程：

  • 创建您想要贡献的项目的分支 。

  • 克隆此派生文件以创建本地存储库。

  • 对原始项目的副本进行更改，然后提交并推送 。

  • 当你准备共享你的工作成果， 重订你的主分支的原始项目当前HEAD叉。这是必要的，以确保您的更改不会与创建派生后推送的新提交冲突。

  • 创建一个拉取请求，以建议您对原始项目的更改。

创建请求请求

拉取请求用于开源项目或某些公司工作流中，以管理来自贡献者的更改并在合并此类更改之前启动代码审查。

通过创建拉取请求，您可以将已推送到GitHub存储库分支的更改告知其他人，以便原始存储库的维护者可以查看您的更改，进行讨论并将它们集成到基础分支中。

  1. 从主菜单中，选择VCS |。 Git |创建请求请求 。 “ 创建请求请求”对话框打开。

  2. 在“ 基叉”下 ，指定要将请求发送到的项目。从由AppCode填充的列表中选择一个存储库，或单击Select Other Fork 。

  3. 在“ 基础分支”下 ，指定将要应用更改的目标项目中的分支。单击显示差异以查看将包含在拉取请求中的提交列表。要查看提交的详细信息，请选择它，然后切换到“ 版本控制”工具窗口的“ 日志”选项卡，您可以在其中查看所选提交中包含的文件列表，查看差异等。

  4. 在标题字段中指定拉取请求的名称 ，并有选择地提供对通过请求进行的更改的描述。

管理传入的拉取请求

如果您是项目维护者，并且为项目配置了GitHub远程服务器，则可以直接从AppCode查看传入的请求：从主菜单中选择VCS |。 Git |查看拉取请求 。

使用GitHub Pull Requests工具窗口执行以下操作：

  • 按州，作者，受让人和日期过滤请求 ：

  • 跳转到GitHub上的拉取请求 ：选择一个拉取请求，然后从上下文菜单中选择“ 在GitHub上打开” 。

  • 查看建议的更改和基本修订之间的差异 ：在右窗格中选择一个文件，然后按⌘D或点击 。

  • 根据传入的更改创建本地分支 ：选择拉取请求，然后从上下文菜单中选择“ 创建新本地分支 ”。
