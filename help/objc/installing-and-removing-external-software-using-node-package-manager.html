npm和纱线

AppCode与npm和Yarn软件包管理器集成在一起，因此您可以从IDE内部安装，定位，升级和删除可重用代码的软件包。Node.js和NPM页面提供了用于管理程序包的专用UI。当然，您也可以从内置Terminal中的命令行执行此操作。

AppCode还允许您运行和调试npm脚本 。AppCode解析package.json文件，识别脚本的定义，在树视图中显示脚本，并让您在树中的脚本和package.json文件中的定义之间导航。

AppCode使用Yarn工作区检测项目，并为工作区的package.json文件中列出但位于根node_modules文件夹中的所有依赖项建立索引。

在你开始前

  1. 下载并安装Node.js。请注意，还安装了npm，因此，如果要使用它，则已完成准备工作。

  2. 要使用纱线，请按照纱线官方网站上的说明进行安装。

选择一个项目包管理器

使用AppCode，您可以选择在项目中使用npm还是Yarn。默认情况下，AppCode建议使用npm。但是，如果您打开带有yarn.lock文件的项目，并且计算机上安装了Yarn，则AppCode会自动将此项目的程序包管理器更改为Yarn。您还可以将Yarn设置为默认的AppCode程序包管理器 。

  1. 在“ 首选项”对话框中（ ⌘, ），请转到语言和框架| Node.js和NPM 。将打开“ Node.js和NPM”页面。

  2. 在包管理器字段中，为您的项目指定包管理器。

    • 选择npm或yarn以使用系统默认安装。

    • 要使用软件包管理器的自定义安装，请单击“ 选择” ，然后在打开的对话框中选择软件包管理器的相关路径。

    选择以下代码后，AppCode将自动使用所选的包管理器Project “ 运行/调试配置：NPM”对话框中“ 程序包管理器”列表中的别名。每当您调用运行“ npm install” / “ yarn install”命令或运行“ npm / Yarn”脚本时，AppCode还将使用所选软件包管理器的路径，有关详细信息，请参阅运行和调试脚本 。

AppCode使用npm和yarn分配给这些管理器的当前系统路径的别名。如果选择其他Node.js版本，AppCode还将更改npm别名路径，以指向与所选Node.js版本捆绑在一起的npm版本。

将Yarn设置为默认的AppCode程序包管理器

  1. 在“新项目的设置”对话框（“ 文件” |“新项目的设置...” ）中，单击“ 语言和框架”下的Node.js和NPM 。

  2. 在打开的Node.js和NPM页面上，选择yarn从包管理器列表中。之后，每次创建新项目时，AppCode都会建议Yarn作为默认值。

使用内置终端安装软件包

如您所知，npm可以全局安装软件包，也可以作为项目依赖项或开发依赖项安装软件包，请从npm官方网站了解更多信息。使用Yarn，您还可以全局安装软件包或作为项目依赖项安装，有关详细信息，请参见Yarn官方网站。

安装软件包

  • 在嵌入式终端 （ ⌥F12 ），键入以下命令之一：

    • npm install --global 要么yarn global add 用于全局安装。

    • npm install --save ， npm install --save-dev ， 要么yarn add --dev将软件包安装为项目依赖项或开发依赖项。

从package.json文件安装所有依赖项

  • 在嵌入式终端 （ ⌥F12 ），键入以下命令之一：

    • npm install

    • yarn install

    结果，您从当前文件夹中获得了package.json中列出的所有依赖项。从Editing package.json了解更多信息。

在Node.js和NPM页面上安装和升级软件包

AppCode在Node.js和NPM页面上显示所有当前安装的软件包。要打开页面，请在“ 设置/首选项”对话框中⌘, ，然后在“ 语言和框架”下单击“ Node.js和NPM ”。全局安装的软件包在顶部列出。对于每个软件包，AppCode都会显示其当前安装的版本和最新的可用版本。

安装软件包

  1. 请点击在打开的“ 可用软件包”对话框中，选择所需的软件包。

  2. 默认情况下，将安装最新的软件包版本。要安装另一个版本 ，请选中“ 指定版本”复选框，然后从列表中选择所需的版本。

  3. 默认安装是本地 。要使用其他安装类型，请选择“ 选项”复选框，然后键入-g用于全局安装或--save / --save-dev将软件包安装为依赖项/开发依赖项。从npm和Yarn官方网站上了解更多信息。

编辑package.json

AppCode可帮助您处理package.json文件中的项目依赖项，从而提供广泛的编码帮助：

  • 程序包名称的代码完成。

  • 有关最新可用软件包版本的信息。

  • 有关可以通过安装的版本范围的信息npm install @要么yarn add @ 。`注意，正在运行npm install要么yarn install将安装该范围内的最新可用版本。

    按⌘然后将鼠标悬停在该版本上即可查看工具提示中的信息。有关语义版本控制的详细信息，请参见npm官方文档 。

  • 先前软件包版本的代码完成。当您按⌃Space或开始输入与最新版本不同的版本，AppCode将显示建议列表以及该软件包的所有先前版本。

  • 快速查找包装文件。

运行和调试脚本

您可以根据专用运行配置，从编辑器中的package.json文件，专用npm工具窗口中的脚本树中的脚本树启动npm或Yarn脚本，也可以自动将其作为启动或启动 前任务。

脚本执行的结果显示在“ 运行工具”窗口中 。 工具窗口显示npm或Yarn脚本输出，报告发生的错误，列出未找到的软件包或插件，等等。最后执行的脚本的名称显示在工具窗口的标题栏上。

在脚本调试会话期间，AppCode将打开“ 调试工具”窗口 ，您可以在其中 单步执行程序 ， 停止并继续执行程序， 在挂起时检查它 ， 查看实际的HTML DOM等

启用调试脚本

  • 添加$NODE_DEBUG_OPTION到package.json文件中的定义，例如：

    {“名称”：“应用程序名称”，“版本”：“ 0.0.1”，“脚本”：{“主”：“节点$ NODE_DEBUG_OPTION ./app-compiled.js”}}

从package.json运行和调试脚本

AppCode使您可以从package.json文件中快速启动单个脚本。要运行或调试几个脚本，请使用运行配置或npm工具窗口。

  • 在编辑器中打开package.json文件，单击在脚本旁边的装订线中，然后选择“运行”
