创建测试

AppCode提供了用于基于XCTest框架创建单元和UI测试的模板。在AppCode中创建新项目时，可以选择是否要添加单元测试和UI测试。在这种情况下，AppCode将创建包含带有初始内容的测试类的测试目标。但是，您可以在创建项目后随时添加测试目标和测试类 。

要为测试类中的方法快速添加存根代码 ，请使用“ 生成”菜单⌘N 。

添加测试目标

  1. 通过按打开项目设置 ⌘;或选择文件|从主菜单中的项目设置 。

  2. 请点击然后从打开的对话框中选择一个测试目标。

    选择“ iOS UI测试包”以创建UI测试的目标，或者选择“ iOS单元测试包” （用于单元测试）。单击下一步 。

  3. 在下一页上，如有必要，请更改以下字段中的值：

    • 产品名称 ：新测试目标的名称。

    • 组织名称 ：您的公司名称或您的名称。

    • 语言 ：目标中测试类的编程语言。

    • 项目 ：将在其中创建新目标的项目。

    • 要测试的目标：将要测试的目标。

新的测试目标将添加到项目中。该目标包含一个带有存根代码的测试类，其中包含您可以填写的几种测试方法。

添加测试课程

  1. 在“ 项目”工具窗口中，右键单击要向其添加测试类的组，然后选择“ 新建” |“新建”。 Xcode模板中的文件 。

  2. 从左窗格中选择源文件类型，从右窗格中选择UI测试用例类或单元测试用例类 。单击下一步 。

  3. 在下一页上，如有必要，请更改以下字段中的值：

    • 文件名 ：新测试类的名称。

    • 位置 ：文件系统中将在其中创建新文件的目录。

    • 子类 ：所创建类的父类。默认情况下，XCTestCase。

    • 语言 ：班级将使用的编程语言。

    • 组 ：该类所属的AppCode项目中的组。

    • Target ：将在其中添加类的目标列表。

生成测试代码

要生成测试方法的代码 ，请将插入号放在测试类代码中，按⌘N ，然后从打开的菜单中选择所需的操作。

根据您使用的测试框架，可以生成不同的代码模板：

  • XCTest和Google Test ： SetUp ， TearDown和Test方法：对于Google Test，AppCode还为您提供了多种TEST巨集：
    • TEST()
    • TEST_F()
    • TEST_P()
    • TYPED_TEST()
    • TYPED_TEST_P()

    您可以生成一个TEST()宏，并指定套件名称后，AppCode会自动将此宏更改为正确的宏。
