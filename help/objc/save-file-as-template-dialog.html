AppCode 2019.2帮助

将文件另存为模板对话框

AppCode可帮助您从现有文件创建文件模板 。

项目

描述

名称

在此处指定新模板将在“ 文件和代码模板”设置的“ 文件”选项卡中显示的名称。默认情况下，使用当前文件的名称。

延期

在此指定此新模板要创建的文件的扩展名。默认情况下，使用当前文件的扩展名。

范本文字

编辑模板内容。您可以使用：

  • 纯文本。

  • #parse用于模板的指令包括 。

  • 自定义变量。变量名可以通过以下方式直接在模板中定义： #set指令或在文件创建期间。

    请注意，AppCode不会提示输入使用定义的Velocity变量的值#set 。

  • 变量将被扩展为相应的值${}格式。

    以下预定义变量可以在文件模板中使用 ：

    变量

    描述

    ${DATE}

    当前系统日期

    ${DAY}

    该月的当前日期

    ${DS}

    美元符号$ 。此变量用于转义美元字符，因此不会将其视为模板变量的前缀。

    ${FILE_NAME}

    新的Objective-C或C ++文件的名称

    ${HOUR}

    当前时间

    ${MINUTE}

    当前分钟

    ${MONTH}

    这个月

    ${MONTH_NAME_FULL}

    当月的全名（一月，二月等）

    ${MONTH_NAME_SHORT}

    当前月份名称的前三个字母（一月，二月，依此类推）

    ${NAME}

    新实体的名称（文件，类，接口等）

    ${ORGANIZATION_NAME}

    在项目设置中指定的组织名称（ ⌘; ）

    ${PRODUCT_NAME}

    IDE的名称（例如，AppCode）

    ${PROJECT_NAME}

    当前项目名称

    ${TIME}

    当前系统时间

    ${USER}

    当前用户的登录名

    ${YEAR}

    今年

    治疗美元符号

    • 您可以防止对待美元字符$在模板变量中作为前缀。如果您需要美元字符（ $按原样插入，请使用${DS}文件模板变量。应用模板后，此变量的值将为纯美元字符$ 。

      例子：

      • 使用一些版本控制关键字（例如$Revision$ ， $Date$ ，依此类推）在您的默认类模板中， ${DS}代替美元前缀$ 。

      • 模板代码${DS}this将呈现为$this 。

根据样式重新格式化

选中此复选框，以使AppCode根据“ 代码样式”页面上定义的样式重新生成存根文件。

此选项仅在“ 文件”选项卡中可用。

启用实时模板

选中此复选框可在文件模板中使用实时模板。为此，必须将实时模板片段放入Velocity转义语法中。

例如：

#[[ $MY_VARIABLE$ $END$ ]]#

因此，可以指定光标位置。注意，这里需要使用实时模板变量 ！

此只读字段提供有关模板，模板的预定义变量及其工作方式的信息。
