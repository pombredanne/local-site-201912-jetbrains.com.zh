代码完成

本节介绍了各种上下文感知的代码完成技术，这些技术可帮助您加快编码过程。

基本完成

基本代码完成功能可帮助您完成可见性范围内的类，方法和关键字的名称。当您调用代码完成功能时，AppCode会分析上下文并建议从当前插入符号位置可以到达的选择（建议还包括Live模板 ）。

非英语键盘布局也可以使用基本完成功能。

代码完成功能可用于自定义文件类型 。但是，AppCode无法识别此类文件的结构，并且会建议选项，无论它们在当前上下文中是否合适。

调用基本完成

  1. 开始输入名称。默认情况下，AppCode在您键入时自动显示代码完成弹出窗口。或者，按⌃Space或选择代码|完成|从主菜单基本 。

  2. 要缩小建议列表的范围，请键入期望名称或关键字的任何部分（甚至接受中间位置的字符），或在点分隔符后调用代码完成功能。AppCode显示的建议在任何位置都包含键入的字符。

    如果使用CamelCase或snake_case名称，则只能输入首字母，AppCode将显示适当的建议。

  3. 接受适当的建议⏎ 。您还可以执行以下操作：

    • 双击列表项以将其插入插入符号的左侧。

    • 按⇥替换插入符号右侧的字符。

    • 使用⇧⌘⏎使当前的代码构造在语法上正确（平衡括号，添加缺少的花括号和分号，等等）。

智能完成

智能代码完成功能会过滤建议列表，并仅显示适用于当前上下文的类型。例如，它可以建议预期的类的构造函数或工厂方法。

调用智能完成

  1. 开始输入。

  2. 按⌃⇧Space或选择代码|完成|从主菜单中选择SmartType 。

  3. 如有必要，请按⌃⇧Space再来一次。AppCode将查找可以通过链式方法调用检索的预期类型的值。

您可以使用语句完成来创建语法正确的代码构造。它插入必要的语法元素（括号，花括号和分号），并使您处于可以开始键入下一条语句的位置。

嬉皮完成

嬉皮补全是一种补全引擎，它可以在可见范围内分析您的文本并根据当前上下文生成建议。它可以帮助您从任何当前打开的文件中补全单词。

将插入符号处的字符串扩展为现有单词

  1. 键入初始字符串，然后执行以下操作之一：

    • 按⌥/或选择代码|完成|循环展开单词可在插入符号前搜索匹配的单词。

    • 按⌥⇧/或选择代码|完成|循环扩展单词（向后）可在插入符号之后和其他打开的文件中搜索匹配的单词。

    出现第一个建议值，并且原型在源代码中突出显示。

  2. 接受建议，或按住⌥键并保持按下/直到找到所需的单词。

后缀代码完成

后缀代码完成功能可帮助您减少编写代码时向后插入符号的跳跃。您可以根据在点后键入的后缀，表达式的类型及其上下文，将已经键入的表达式转换为其他表达式。

Postfix完成选项显示为基本完成建议列表的一部分。要查看适用于当前上下文的后缀完成的完整列表，请按⌘J 。

Postfix完成可用于TypeScript，JavaScript和SQL。

  • 要查看可用的后缀模板的完整列表， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|后缀完成 。

    从JavaScript Postfix Templates了解更多信息。

  • 选择⇥ ， Space ， 要么⏎扩展后缀模板。

用后缀转换语句

  • 键入一个表达式，然后在一个点后键入一个后缀，例如，.if:

    函数m（arg）{arg.if}

    初始表达式用if声明：

    函数m（arg）{if（arg）{}}

您可以编辑预定义的后缀模板，例如，将长键替换为较短的后缀模板，或扩展适用的表达式类型列表。

创建自定义的后缀模板

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|后缀完成 。

  2. 请点击在工具栏上。

  3. 指定Key ，它是将调用模板的符号的组合，选择新模板将适用的表达式类型，并以以下格式键入目标表达式： $EXPR$ ， 例如， $EXPR$ =!null 。

标签和属性的完成

AppCode自动完成许多文件类型的标记和属性的名称和值：

  • HTML / XHTML，包括CSS类和JSX内部HTML标记的 完成 。

  • XML / XSL ，包括名称空间的完成。

  • JSON，有关详细信息，请参见编辑package.json 。

标记和属性名称的完成取决于与文件关联的DTD或Schema 。如果没有架构关联，AppCode将使用文件内容（标记和属性名称及其值）来完成输入。

完整的标签名称

  1. 按<并开始输入标签名称。AppCode显示适用于当前上下文的标记名称列表。

    使用ArrowUp和ArrowDown按钮滚动列表。

  2. 按⏎接受列表中的选择。如果您的文件与架构或DTD关联，则AppCode会根据其自动插入必填属性。

故障排除

如果无法完成代码，则可能是由于以下原因之一：

  • 省电模式已打开（ 文件|省电模式 ）。开启它可以消除背景操作，包括错误突出显示，即时检查和代码完成，从而将笔记本电脑的功耗降至最低。

  • 未为您的项目配置SDK。

  • 您的文件不驻留在内容根目录中，因此它没有获得代码完成所需的类定义和资源。

  • 包含要在完成建议列表中显示的类和函数的文件被标记为纯文本文件。

  • 包含要显示在完成建议列表中的功能的外部库不会作为依赖项或全局库添加 。
