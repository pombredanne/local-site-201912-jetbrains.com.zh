家

选中此复选框后，按↖ ，插入号位于当前行的第一个非空格字符处。紧迫↖随后将插入符号从“ 智能家居”位置移至第一列，然后再移回。

结束（在空白行上）

选中此复选框后，按↘在空行中，插入符号与缩进位置一起定位，AppCode认为该缩进在当前代码点中是合理的（缩进基于当前的代码样式设置 ）。

插入对括号

选中此复选框，AppCode会分别为每个键入的开口圆括号或方括号自动添加一个圆括号或方括号。

插入对报价

选中此复选框可让AppCode自动为每个键入的开头单引号或双引号分别添加一个结束单引号或双引号。

输入“}”时重新格式化块

如果选中此复选框，则在键入右花括号时，如果此代码块的格式与所选代码样式不匹配，则将自动重新格式化所包含的代码块的格式。

用'CamelHumps'词

选中此复选框可使AppCode识别CamelHump名称中的单独单词。名称中的单词应以大写字母或下划线开头。此选项会影响某些编辑器操作，例如：

  • 插入符移动 ⌥→ / ⌥←

  • 插入符选择移动 （ ⌥⇧→ / ⌥⇧← ）

  • 在插入符号处选择单词 ⌥↑

  • 删除到Word的开始/结束 （ ⌥⌫和⌥⌦分别）

  • 双击 （如果在“ 编辑器” |“常规设置”页面上启用了使用双击选择时的“荣誉” CamelHumps”单词设置 ）。

输入引号或花括号时的环绕声选择

如果选中此复选框，则在键入引号，双引号或大括号时所选的文本将被这些字符包围。如果未选中此复选框，则键入的引号，双引号或大括号将替换所选内容。

双重添加多个插入符⌥带方向键

如果选中此复选框，则：

  • 紧迫⌥加上向上/向下箭头键会导致创建多个插入符。

  • 紧迫⌥加上左/右箭头键或Home / End可以创建一个选择。

有关更多信息，请参见“ Multicursor”部分。

用制表符跳到右括号/引号之外

如果选中此复选框，请按⇥在方括号/引号内键入时，将插入符号移动到右方括号/引号之外。如果未选中此复选框，请按⇥将插入Tab字符。

请注意，这仅适用于初始输入：在后续编辑过程中，按⇥括号/引号内将插入Tab字符。

输入

使用此区域可以定义要通过按调用的动作⏎ 。
  • 智能缩进 -选中此复选框可让AppCode添加新行并将插入号放置在其中，并且该缩进表明AppCode在当前代码点中认为是合理的（缩进基于当前的代码样式设置）。

    如果清除该复选框，请按⏎在空白行中，AppCode添加新行并将插入号放置在当前的非空格字符列处。

  • 插入对'}' -选中此复选框可使AppCode自动定位右括号}在适当的时候⏎在空行中按。在这种情况下，AppCode向后搜索最近的未封闭的开括号{并将最后一个放在相应的缩进级别。

  • 插入文档注释存根 -此复选框定义了按下时的行为⏎在打开文档标签之后。
    • 如果选中此复选框，则AppCode会生成文档注释存根。

      对于功能注释，此存根包含必需的标签（ @param签名中声明的每个参数的标签，以及@return ）。有关详细信息，请参考创建JSDoc注释 。

    • 如果未选中此复选框，则仅生成结束标记。

退格键

使用此下拉列表可以定义要按下的操作⌫键。可用的选项有：

  • 禁用 -按下⌫一次将插入符返回一个位置。

  • 到最近的缩进位置
  • 适当缩进

在粘贴时重新格式化

使用此下拉列表可以指定如何放置粘贴的代码块。可用的选项有：

  • 无 -粘贴的代码以纯文本形式插入插入符号的位置，没有任何重新格式化或缩进。

  • 缩进块 -根据当前的“代码样式设置” ，粘贴的代码块位于适当的缩进级别，但是其内部结构未更改。

  • 每行缩进 -根据当前的代码样式设置 ，将粘贴的代码块的每行放置在适当的缩进级别。

  • 重新格式化块 -根据当前的代码样式设置，重新格式化粘贴的代码块。

此功能适用于包含尾随换行符的行。

XML / HTML

在此区域中，定义在编辑XML或HTML代码时自动调用的动作。

  • 在标记完成时插入结束标记 ：选中此复选框，以使AppCode在输入相应的开始标记时自动插入结束XML或HTML标记。

  • 在标记完成时插入所需的属性 ：选中此复选框可使AppCode显示带有类型化标记的所有必需属性的模板。

  • 在标记完成时插入所需的子标记 ：选中此复选框可使AppCode显示带有所有必需子标记的模板。

  • 标签完成时开始属性 ：选中此复选框可使AppCode显示带有已键入标签的第一个必需属性的模板。

  • 在键入'='时为属性值添加引号 ：选中此复选框可使AppCode自动为您当前键入的属性的值添加引号。

  • 输入“'”时自动关闭标签：选中此复选框可在输入后自动添加关闭标签

  • 同时编辑 ：

    • 选中此复选框并编辑开始标签后，相应的结束标签将相应地自动更改。

    • 如果清除此复选框，则编辑开始标签不会影响保持不变的结束标签。结果，开始标签和结束标签不匹配，并且整个结构被加下划线标为错误。

    此复选框在以下情况下控制AppCode的行为：

    • HTML文件

    • JavaScript代码中的HTML注入

    • 带有模板的HTML 把手/胡子模板

    • 扩展名为.hbs的车把 /胡子模板文件

    • XML，XHTML文件

    • DTD文件

    • JSX文件

JSON格式

该区域中的设置控制是否按照JSON语法在JSON上下文中自动添加逗号，引号和空格。默认情况下，所有设置都是启用的。
