从实例文档生成XML模式

在Web内容文件上运行结构验证检查需要XSD（XML架构定义） 。AppCode可以扫描任何XML文件中的现有元素和属性，并为其生成模式。

生成基于XML文档的模式

  1. 在活动的编辑器选项卡中打开所需的XML文档，然后选择“ 工具” |“工具”。 XML操作|从主菜单上的XML文件生成XSD架构 。从实例文档生成架构对话框将打开。

  2. 在“ 实例文档路径”文本框中，指定用作架构生成基础的文件的位置。默认情况下，该字段显示当前文件的完整路径。接受此建议或单击浏览 然后在打开的对话框中选择所需的文件。

  3. 在“ 结果模式文件名”文本框中，指定用于放置生成的模式的输出文件的名称。

    AppCode建议使用扩展名.xsd的源XML文档的名称。如果键入其他名称，请确保扩展名正确。

  4. 指定生成的架构的位置。默认情况下，生成的Schema文件将与源XML实例文档放置在同一目录中。要指定其他位置，请单击浏览 然后在打开的对话框中选择所需的路径。

  5. 从“ 设计类型”下拉列表中，选择声明元素和复杂类型的方式。

  6. 从“ 检测简单内容类型”下拉列表中，选择要用于叶文本的类型。

  7. 在“ 检测枚举数限制”文本框中，键入引起架构枚举出现的出现次数。

若要取消架构枚举，请指定0。
