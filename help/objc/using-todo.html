TODO评论

有时，您需要标记部分代码以供将来参考：优化和改进，可能的更改，要讨论的问题等。AppCode允许您添加特殊类型的注释，这些注释在编辑器中突出显示 ，被索引并在TODO工具窗口中列出。这样，您和您的队友就可以跟踪需要注意的问题。

默认情况下，AppCode可以识别两种模式： TODO和FIXME无论大小写这些模式可在任何受支持文件类型的行注释和块注释内部使用。您可以根据需要修改默认模式或添加自己的模式 。

添加待办事项

  • 将插入符号放置在要创建TODO注释的位置，例如，通过按⌘/ ，然后输入TODO要么FIXME ，然后键入您的笔记。

  • 您还可以创建一个实时模板，以添加带有一些预定义内容的TODO注释。请参阅第41页的创建实时模板中的示例。

添加多行TODO注释

  • 要创建多行TODO项目，请缩进第一行之后的注释行（至少带有一个选项卡或两个空格）。如果没有缩进，则将行视为常规注释行。

  • 要禁用多行待办事项， 在“ 偏好设置”对话框中（ ⌘, ），转到“ 编辑器” | TODO并清除以下行中的将缩进文本视为同一TODO复选框的一部分。

查看所有待办事项注释的列表

  • 要查看TODO项目，打开TODO工具窗口： 查看|工具窗口|待办事项或⌘6 。

  • 要跳至源代码中的TODO注释，请在TODO工具窗口中单击相应的TODO项。若要禁用此行为，请右键单击TODO工具窗口标题栏，指向“ 查看选项” ，然后清除“ 自动滚动到源代码”选项（在这种情况下，您需要双击TODO项以跳到相关注释）。

添加自定义模式并过滤待办事项

您可以添加自己的模式并过滤列表以仅显示与某些模式匹配的TODO项目。例如，您可以选择在代码中使用OPTIMIZE在“ 待办事项”工具窗口中查看它们时，请更改样式并忽略所有其他类型的待办事项。

  1. 在“ 首选项”对话框中（ ⌘, ），选择编辑器|待办事项 。

  2. 使用正则表达式指定自定义模式。

    例如，为OPTIMIZE项目，单击在“ TODO”对话框的“ 模式”部分中，键入以下正则表达式：

    \ boptimize \ b。*

    然后单击确定以保存新模式。

  3. 添加过滤器以将TODO模式分组，并分别在TODO工具窗口中查看相应的TODO项目。

    例如，添加Optimization使用新模式进行过滤，请点击在“ 待办事项”对话框的“ 过滤器”部分中，指定Optimization作为其名称，然后选择要包含在此过滤器中的新模式。

    然后单击确定以保存新的筛选器。

  4. 单击“ 确定”以在“ TODO设置”对话框中应用更改。

  5. 要应用新的过滤器，请在TODO工具窗口中单击并选择优化过滤器。

您为模式选择的图标显示在TODO工具窗口中，以更好地区分各种TODO项目。通过启用模式的区分大小写复选框，可以强制模式仅与指定的大小写匹配。
