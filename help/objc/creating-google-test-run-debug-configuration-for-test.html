Google测试支持

Google Test和Google Mock是一对强大的单元测试工具：该框架具有可移植性，它包含一组致命的和非致命的断言，提供用于创建装置和测试组的工具，提供信息性消息并将结果导出到XML。可能唯一的缺点是需要在项目中构建gtest / gmock才能使用它。

生成Google Test的菜单

使用“ 生成”菜单可以节省编写测试代码的时间：在您拥有的文件中gtest包括在内，按⌘N查看代码生成选项列表。

从灯具调用时，此菜单还包括SetUp方法和TearDown方法 ：

对于夹具测试，代码生成转换TEST()宏放入适当的TEST_F() ， TEST_P() ， TYPED_TEST() ， 要么TYPED_TEST_P() （请参阅类型化测试 ）。

Google Test运行/调试配置

尽管Google测试提供了main()条目，并且您可以将测试作为常规应用程序运行，我们建议您使用专用的Google Test运行/调试配置。它包括与测试相关的设置，使您可以从内置的测试运行器中受益，如果您以常规程序运行测试，则无法使用该功能。

AppCode会自动为与之链接的每个CMake目标创建一个Google Test配置gtest要么gmock 。

  1. 要创建Google测试配置，请转至运行|编辑配置 ，单击并从模板列表中选择Google测试 。

  2. 指定要包含在配置中的测试或套件，或者提供用于过滤测试名称的模式。字段中提供了自动完成功能，可帮助您快速填写它们：

    设置通配符以指定测试模式，例如：

    在配置设置的其他字段中，可以设置环境变量和命令行选项。例如，使用“ 程序参数”字段来传递--gtest_repeat标记并多次运行Google测试：

    输出将是：

    重复所有测试（迭代1）...重复所有测试（迭代2）...重复所有测试（迭代3）...

  3. 保存配置，即可运行 或调试 。

您可以修改Google Test模板本身 ，而不用编辑单个配置。在这种情况下，您指定的设置将作为默认值应用于此类型的所有新配置。

装订线图标还显示测试结果（如果可用）：成功或失败 。
