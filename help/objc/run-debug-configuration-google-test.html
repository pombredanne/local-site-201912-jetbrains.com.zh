运行/调试配置：Google测试

使用此页面可以配置Google Test框架的运行/调试选项。

在这里，您可以找到所有运行/调试配置通用的选项说明。

项目

描述

测试种类

选择套件/测试以运行特定套件及其测试。选择模式以运行不同套件和测试的组合。

套房

测试

如果选择套件/测试，则会显示这些字段。在这里，您需要指定一个套件及其要运行的测试。

图案

如果选择图案，则显示此字段。在这里，您需要指定要运行的套件和测试的组合。

程序参数

在此字段中，以将在命令行中使用的格式键入要传递给程序的参数列表 。如有必要，请单击按钮并输入所需的参数。

工作目录

指定用于运行应用程序的工作目录。该目录是所有相对输入和输出路径的起点。默认情况下，该字段包含项目文件所在的目录。要指定另一个目录，请单击并选择目录。

展开列表以查看可用的路径变量 ，您可以将其用作工作目录的路径。在这种情况下，您不需要在“ 环境变量”字段中指定任何其他环境变量。

路径变量的列表可能会因启用的插件而异。

环境变量

创建环境变量并指定其值。

缩短命令行

如果类路径太长，或者您有许多VM参数超出OS命令行长度限制，请选择一种可用于缩短命令行的方法：

  • none ：AppCode不会缩短长类路径。如果命令行超出了操作系统限制，则AppCode将无法运行您的应用程序，并显示一条消息，提示您指定缩短方法。

  • JAR清单 ：AppCode将通过临时classpath.jar传递长类路径 。原始类路径在清单文件中定义为class-path classpath.jar中的属性。如果使用此方法将完整的命令行缩短了，您将可以预览完整的命令行，而不仅仅是临时classpath.jar的classpath 。

  • classpath文件 ：AppCode将长的类路径写入文本文件。

  • 用户本地默认设置 ：为在AppCode版本10.7.3之前创建的项目自动设置此旧选项。AppCode将根据ide / workspace.xml和idea.config.path / options / options.xml文件中设置的属性来配置此设置。

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 ⌃⌥R 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.idea \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.idea \ workspace.xml中 。

允许并行运行

如果选中此复选框，则可以并行启动运行配置多次，而不必重新运行它。每个运行程序都将在“ 运行工具窗口”或“ 调试工具窗口 ”的自己的选项卡中启动 。

禁用后，每次启动运行/调试配置的新实例时，AppCode都会检查同一运行/调试配置的其他实例是否存在，并显示确认对话框。如果在确认对话框中单击“ 确定” ，则将停止运行器的第一个实例，而将其替换。当启动两个相同类型的运行/调试配置实例时，这会占用过多的CPU和内存资源。

工具列

运行/调试配置的树形视图具有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

发射前

在此区域中，您可以指定在启动选定的运行/调试配置之前必须执行的任务。任务以它们在列表中出现的顺序执行。

项目

捷径

描述

⌘N

单击此图标以添加以下可用任务之一：

  • 运行外部工具 ：选择运行外部应用程序。在打开的对话框中，选择一个或多个要运行的应用程序。如果尚未在AppCode中定义它，请添加其定义。有关更多信息，请参见管理第三方工具和外部工具 。

  • 运行另一个配置 ：选择以执行另一个运行/调试配置。在打开的对话框中，选择要运行的配置。

  • 运行Grunt任务 ：选择此选项可运行Grunt任务。

    在打开的“ Grunt任务”对话框中，指定定义了所需任务的Gruntfile.js ，选择要执行的任务，然后指定要传递给Grunt工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及grunt-cli包的路径。

  • 运行Gulp任务 ：选择此选项可运行Gulp任务。

    在打开的Gulp任务对话框中，指定在其中定义了所需任务的Gulpfile.js ，选择要执行的任务，然后指定要传递给Gulp工具的参数。

    指定Node.js解释器的位置，传递给它的参数以及gulp包的路径。

  • 运行npm脚本 ：选择此选项以执行npm脚本。

    在打开的NPM脚本对话框中，指定npm运行/调试配置设置 。

  • 生成CoffeeScript源映射 ：选择此选项可为CoffeeScript源生成源映射。在打开的对话框中，指定CoffeeScript源文件的位置。

⌘⌦

单击此图标从列表中删除所选任务。

⏎

单击此图标以编辑所选任务。在打开的对话框中进行必要的更改。

/

⌃↑ /⌃↓

单击这些图标可将所选任务在列表中上移或下移一行。（任务按照它们在列表中显示的顺序执行。）

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口⌘4要么⌘5 。
