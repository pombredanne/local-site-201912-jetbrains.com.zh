合并分支对话框

VCS | Git |合并变更

使用此对话框可以指定用于合并本地Git存储库中的分支的参数。

项目

描述

Git根

从此下拉列表中，选择要在其中合并分支的本地存储库的路径。

当前分支

此只读字段显示当前在所选本地存储库中检出的分支名称。这是目标分支，来自选定源分支的更改将应用到它。

该字段的内容取决于“ Git Root”下拉列表中的选择。

合并分支

使用此列表框可以指定源分支，更改将从这些源分支应用于目标分支。

该列表仅显示那些包含适用提交的分支。适用的提交是在分支与目标分支分离之后进行的提交。

战略

从此下拉列表中，选择合并策略 。可用的选项有：
  • 默认

  • 解决 -如果需要解析两个HEAD，则选择此选项，其中一个是当前分支，另一个HEAD是在“ 要合并的分支”列表中选择的分支 。选择此选项时，将应用三向合并算法。

  • 递归 -用于将当前分支与一个分支合并的默认合并策略。如果您需要通过应用三向合并算法来解析两个HEAD，并且有多个以上的共同祖先可用于三向合并，请选择此选项。

  • 章鱼 -用于将当前分支与多个分支合并的默认合并策略。

    不会执行需要手动解决冲突的合并。

  • 我们的 -如果您需要解析多个HEAD，请选择此选项。合并的结果始终是当前分支的HEAD。

  • 子树 -修改后的递归策略。

在“ 要合并的分支”列表框中选择两个或多个源分支时，仅“ 章鱼”和“ 我们的”选项可用。

没有提交

如果需要检查，请选中此复选框，必要时在提交结果之前调整合并结果。合并已执行，但不会自动提交，就好像失败了。

没有快进

选择此复选框以生成合并提交，即使合并被解析为快进 。

壁球提交

选择此复选框可在当前分支的顶部创建一个提交，而不是合并一个或多个其他分支。工作树和索引状态的产生就像发生了真正的合并一样，但是未执行提交并且HEAD未被移动。

添加日志信息

选中此复选框，除分支名称外，还将使AppCode填充一条日志消息，其中包含来自正在合并的实际提交的单行描述。

提交讯息

在此文本框中，提供有关提交的描述。

仅当未选中“ 不提交”复选框时，文本框才可用。

合并

单击此按钮可根据定义的设置启动合并本地存储库中的指定分支。
