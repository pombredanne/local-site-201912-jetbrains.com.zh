使用连接到Docker守护程序

Mac的Docker ：如果您正在使用macOS的Docker，则推荐使用此连接选项。

Unix套接字 ：这是Linux的推荐连接选项。

如果出现权限被拒绝错误，请确保当前用户已添加到计算机上的docker组。

Docker Machine ：如果您正在使用Windows或macOS的Docker Toolbox，则建议使用此选项连接到Docker API。如果未显示“ 连接成功”消息，请检查Docker |上的Docker Machine可执行文件设置。 工具页面。

TCP套接字 ：如果您正在使用Windows的Docker，这是通常的连接选项。该选项还适用于Linux，适用于macOS的Docker和Docker Toolbox。

  • 引擎API URL ：取决于Docker版本和操作系统：

    • 适用于Windows的Docker： tcp：// localhost：2375

      在Docker设置的`` 常规''部分中，打开tcp：// localhost：2375上没有TLS选项的Expose守护程序 。

    • 适用于macOS或Linux的Docker： unix：///var/run/docker.sock

    • 适用于Windows或macOS的Docker Toolbox： https ://192.168.99.100: 2376

  • 证书文件夹 ： 证书文件夹的路径。取决于您的Docker版本和操作系统：

    • 适用于Windows，macOS或Linux的Docker：将此字段保留为空。

    • Windows版Docker工具箱： ％HOMEPATH％\。docker \ machine \ machines \ default

    • 适用于macOS的Docker Toolbox：通常为〜/ .docker /或其子目录。

    由于“ 证书文件夹”字段指定了带有证书的任何文件夹，因此该字段对应于DOCKER_CERT_PATH环境变量。有关更多信息，请参阅Docker文档中的客户端模式 。

路径映射

对于Windows和macOS：指定可以在主机卷和容器卷之间共享的文件夹的映射。

使用编辑现有映射，或创建一个新的。

  • 本地路径 ：要用于卷绑定的本地文件夹的路径。

  • 虚拟机路径 ：Docker虚拟机文件系统中的相应目录路径。
