节点解释器

在这个领域里， 指定当前项目的默认Node.js解释器。每次您选择Project创建运行/调试配置或配置依赖于Node.js的工具（例如Prettier或Yeoman）时， Node Interpreter列表中的别名。

从列表中或单击时打开的对话框中选择解释器 。如果选择node ，则使用系统Node.js版本 。

版

此只读字段显示Node.js的当前版本。

配套

软件包区域显示计算机上当前安装的所有NPM相关软件包的列表。

  • 软件包：此只读字段显示软件包的名称，与在命令行模式下安装软件包时应引用的名称完全相同。

  • 版本：此只读字段显示计算机上安装的软件包的版本。

  • 最新：此只读字段显示软件包的最新发行版本。如果软件包不是最新的，则标记为 。

  • 请点击安装新软件包。在打开的“ 可用软件包”对话框中，选择相关的软件包。要全局安装软件包，请选择选项复选框，然后键入-g在选项字段中。全局安装使该软件包在AppCode级别可用，因此可以在任何AppCode项目中使用。准备好后，单击“ 安装软件包” 。

  • 请点击删除选定的软件包。

  • 请点击将所选软件包的当前版本替换为最新发行版本。仅当所选项目不是最新时才启用该按钮。
