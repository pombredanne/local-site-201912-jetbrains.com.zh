代码样式。CoffeeScript

使用此部分为CoffeeScript文件配置格式选项。更改这些设置时，更改结果将显示在“ 预览”窗格中。

制表符和缩进

使用制表符

  • 如果选中该复选框，则制表符将用于缩进和代码重新格式化。

  • 如果清除该复选框，则AppCode将使用空格而不是制表符。

智能标签

压痕由两部分组成。一部分来自嵌套代码块，另一部分由对齐确定。

  • 如果选中此复选框，则嵌套产生的零件将同时包含制表符和空格（如果需要），而由对齐方式定义的零件仅包含空格。

  • 如果清除此复选框，则仅使用选项卡。这意味着在重新格式化一组适合指定制表符大小的空格后，将自动用制表符替换，这可能会导致中断精细对齐。

如果选中了“ 使用选项卡字符”复选框，则“ 智能选项卡”复选框可用。

标签尺寸

在此字段中，指定选项卡中可以容纳的空格数。

缩进

在此字段中，为每个缩进级别指定要插入的空格数（如果选中了“ 使用制表符”复选框，则为制表符 ）。

连续缩进

在此字段中，在表达式，方法声明和方法调用中，指定要在数组的元素之间插入的空格数（如果选中了“ 使用制表符”复选框，则为制表符 ）。

缩进空行

如果选中此复选框，AppCode将在空白行上保留缩进，就好像它们包含一些代码一样。如果清除此复选框，则AppCode删除制表符和空白行。

缩进链接方法

在函数声明中，链中的第二个方法和其他方法显示在单独的行上。

  • 选中此复选框后，链中的第二个方法和其他方法将与第一个调用对齐。

  • 清除复选框后，链中的第二个方法和其他方法将与调用它们的对象对齐。

空间

使用此标签可以指定您希望AppCode在何处自动插入空格。选中相关位置的描述旁边的复选框，然后在“ 预览”窗格中检查结果。

包装和大括号

在此标签中，自定义AppCode在重新格式化源代码时将应用于各种代码构造的异常，括号放置和对齐选项。在“ 预览”窗格中检查结果。

对齐优先于缩进选项。

硬包装

在此字段中，指定元素右侧所需的空格数。如果您接受默认选项，则使用全局设置中的值。

包装打字

在此字段中，指定如何将编辑的文本适合指定的“ 硬包装”字段。

  • 默认值 ：选择此选项可使用全局设置中的“ 键入时自动换行”值。

  • 是 ：选择此选项可以使用“ 右边距”字段中的值。

  • 否 ：如果选择此选项，则一行可能会超出右边距中指定的值。

视觉指南

在此字段中，指定多个右边距。您可以保留默认值，也可以输入空白。如果要指定多个边距，请输入用逗号分隔的数字。

重新格式化时保持

使用复选框配置AppCode在重新格式化源代码时将产生的异常。例如，默认情况下，“ 换行符”复选框处于选中状态。如果您的代码包含比标准约定短的行，则可以通过在重新格式化之前禁用“ 换行符”复选框来转换它们。

包装选项

包装样式适用于在左侧窗格中指定的各种代码构造（例如，方法调用参数或赋值语句）。

不要包裹

选择此选项时，不应用特殊的换行样式，嵌套的对齐方式和花括号设置将被忽略。

如果长条包装

选择此选项可使用适当的缩进来包装超出右边距的行。

总是包装

选择此选项可将所有元素包装在列表中，以便每行有一个带有适当缩进的元素。

砍多久

选择此选项可将元素包装在超出右边距的列表中，以便每行有一个具有适当缩进的元素。

对齐选项

多行时对齐

如果选中此复选框，则代码构造将从下一行的同一列开始。否则，代码构造的位置由当前的缩进级别确定。

在下一行

换行时，选中此复选框可将指定的一个或多个字符移动到下一行。

新行上的“ else”

使用此复选框将相应的语句或字符移动到下一行。

之后的新行

选中此复选框可将指定字符后的代码移动到新行。

特别待遇

如果选中此复选框， else if语句位于同一行。

除此以外， else if语句移动到相应缩进级别的下一行。

变量声明

选择以下选项之一为等号配置对齐方式：

  • 不对齐 ：等号不对齐。

  • 多行时对齐 ：等号在多行中签名var语句通过插入其他空格来对齐。

  • 分组时对齐 ：等号签到多个var语句通过插入其他空格来对齐。

ES6导入/导出

对齐'from'子句：选中此复选框后，AppCode对齐import和export ECMAScript 6代码中的语句自动使您的代码更易于阅读和维护。在“ 预览”窗格中比较带有对齐和不带对齐的代码片段的外观。

启用此选项后，AppCode将即时调整新代码。现有import和export通过按以下命令重新格式化代码后，语句将对齐⌥⌘L ，有关详细信息，请参见“ 重新格式化源代码”部分。

大括号放置选项

大括号放置样式

使用此列表可以在类声明 ， 方法声明 ， 函数声明和其他类型的声明中指定左括号的位置。可用的选项有：

  • 行尾 ：选择此选项可将开括号放在声明行的末端。

  • 如果被换行，则换行 ：选择此选项可将开括号放在多行声明行之后的行首。

  • 下一行 ：选择此选项可将左括号放在声明行之后的行首。

  • 下一行移位 ：选择此选项可在将声明行移位到相应的缩进级别后的行处放置开括号。

  • 每行都移位的下一行 ：选择此选项可在将声明行移动到相应的缩进级别之后的行处放置开括号，并将下一行也移到下一个缩进级别。

大括号

从此列表中选择括号的引入方法if ， for ， while和do () while陈述。可用的选项有：

  • 不强制 ：选择此选项可自动禁止插入括号。

  • 多行时：如果语句占用多行，请选择此选项以自动插入大括号。请注意，AppCode分析整个语句中的行数，而不仅分析其条件。

    例如，如果代码是

    int foo（int Days）{if（（（Days％10）== 3）返回1;否则返回0； }

    AppCode将自动插入花括号：

    int foo（int Days）{if（（Days％10）== 3）{返回1; } else {返回0； }}

  • 始终 ：选中此复选框后，AppCode始终自动插入括号。

空行

使用此选项卡定义重新格式化后希望AppCode保留并插入代码中的空白行的位置和数量。结果显示在“ 预览”窗格中。

此选项卡上的设置不会影响第一个项目之前和最后一个项目之后的空白行数。

保留最大空白行

在此区域中，指定重新格式化后要保留的额外空白行数。
