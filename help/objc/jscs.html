JSCS

AppCode使您可以使用JSCS linter验证JavaScript代码。AppCode还可以检测错误并为JSCS配置文件提供代码完成。

在开始之前，请确保您的计算机上装有Node.js。

全局安装JSCS

  • 在嵌入式终端 （ ⌥F12 ），输入：

    npm install jscs -g

启用JSCS并在AppCode中配置其行为

  1. 在“ 首选项”对话框中（ ⌘, ），请转到语言和框架| JavaScript |代码质量工具| JSCS 。在打开的JSCS页面上，选中Enable复选框。之后，页面上的其他控件将变为可用。

  2. 在“ 节点解释器”字段中，指定Node.js的路径。 如果您遵循标准安装过程，则AppCode会检测路径并填写字段本身。

  3. 在JSCS包字段中，指定jscs软件包安装在当前项目中。

  4. 指定要使用的配置。

    默认情况下，AppCode首先查找jscsConfig当前项目的package.json文件中的属性。如果找不到此类属性，AppCode将查找.jscsrc或.jscs.json配置文件。AppCode从存储要检查文件的文件夹开始搜索，然后在父文件夹中搜索，依此类推，直到到达项目根目录。因此，您必须定义配置以应用为jscsConfig package.json文件或.jscsrc或.jscs.json配置文件中的属性 ，或自定义JSON配置文件中的属性。

    您还可以独立或与配置文件结合使用一组预定义的规则。在后一种情况下，配置文件中的规则将覆盖预定义的规则。

    • 要让AppCode寻找一个jscsConfig package.json文件或.jscsrc或.jscs.json文件中的“属性”属性，请选择“ 搜索配置”选项。

    • 要使用自定义文件，请选择“ 配置文件”选项，然后在“ 路径”字段中指定文件的位置。从列表中选择路径，或手动输入，或单击然后从打开的对话框中选择相关文件。

    • 要应用预定义的集或规则，请从“ 代码样式预设”列表中选择所需的集。

  5. 如有必要，从“ 代码样式预设”列表中， 选择与您使用的代码样式关联的一组预定义规则。
