AppCode 2019.2帮助

自动导入

当您开始使用尚未导入的类或尚未包含的库中的方法时，AppCode会建议自动添加相应的import语句，甚至在您完成代码时也添加导入。

导入丢失的头文件

  1. 键入对未导入的类或函数的引用。

  2. 按⌥⏎ 。如果有多个选择，请从列表中选择所需的选项。

自动添加导入语句（在TypeScript中）

在TypeScript文件中，如果只有一个选项可供选择，则AppCode可以自动添加导入语句。

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|自动导入| TypeScript / JavaScript 。

  2. 选中即时导入明确复选框，然后应用更改。

当您粘贴包含对尚未导入的符号的引用的代码块时，AppCode将自动添加缺少的导入。

禁用导入弹出窗口

  1. 在“ 首选项”对话框中（ ⌘, ），点击编辑器|一般|自动导入 。

  2. 清除所用编程语言的“ 显示导入弹出窗口”复选框。使用TypeScript时，请为此清除“ 使用导入弹出窗口”复选框。

禁用弹出窗口时，未解析的引用标记为红色。要查看建议列表，请将光标放在参考上，然后按⌥⏎ 。

优化进口

优化导入功能可帮助您删除未使用的导入，并一次组织当前文件或目录中所有文件中的导入语句。

优化所有进口

  1. 在“项目”工具窗口中选择一个文件或目录。

  2. 请执行以下任一操作：

    • 在主菜单中，选择代码|优化进口 （或按⌃⌥O ）。

    • 从上下文菜单中，选择“ 优化导入” 。

  3. （如果选择了目录）选择是要优化目录中所有文件的导入，还是仅优化本地修改的文件（如果您的项目受版本控制），然后单击运行 。

在单个文件中优化导入

  1. 将插入符号放在导入语句中，然后按⌥⏎ （或使用意图动作图标）。

  2. 选择优化导入 。

自动优化修改文件中的导入

如果您的项目受版本控制，则可以指示AppCode在将修改的文件提交到VCS之前优化导入。

  1. 从主菜单中，选择VCS |。提交 （或按⌘K ）。

  2. 在“ 提交更改”对话框的“ 提交 之前”区域中，选中“ 优化导入”复选框。
