AppCode 2019.2帮助

使用源代码

使用代码时，AppCode可确保您的工作轻松自如。它提供了各种快捷方式和功能来帮助您添加，选择，复制，移动，编辑，折叠和保存代码。

有关在编辑器内部进行导航的信息，请参阅《 编辑器基础知识》 。

寻找行动

  • 如果您不记得要使用的操作的快捷方式，请按⇧⌘A以名称查找任何动作。

您可以使用相同的对话框来查找类，文件或符号。有关更多信息，请参阅处处搜索 。

切换文件的只读属性

如果文件是只读文件，则该文件带有关闭的锁定图标标记在状态栏 ，其编辑器标签或“ 项目工具窗口”中 。如果文件可写，则用打开锁图标标记在状态栏中。

  1. 在编辑器中打开文件，或在“项目”工具窗口中选择它。

  2. 请执行以下任一操作：

    • 从主菜单中，选择文件|将文件设为只读或文件|使文件可写 。

    • 单击状态栏中的锁定图标。

如果版本控制系统设置了只读状态，则建议您使用AppCode版本控制集成功能。有关更多信息，请参见版本控制 。

选择代码构造

  • 在编辑器中，将插入符号放置在要选择的项目上，然后按⌥↑ / ⌥↓扩展或缩小您的选择。

    例如，在纯文本文件中，选择从整个单词开始，然后扩展到句子，段落等。

  • 如果只需要突出显示支架，则将插入标记立即放在块关闭支架/支架之后或块打开支架/支架之前。

根据大小写选择代码

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|智能钥匙 。

  2. 选中使用“ CamelHumps”单词复选框。

如果要在根据大写字母进行选择时使用双击，请确保在“ 编辑器” | “ 编辑器”中选中“ 荣誉CamelHumps单词...”复选框。一般页面 首选项对话框（ ⌘, ）。

选择事件

您可以在文件中快速找到所选元素的所有实例，进行多项选择，或选择下一个或上一个实例。

  • 要选择一个文件中的多个匹配项，请对代码进行初始选择，然后按⌃G 。按相同的键查找下一个匹配项。

    或者，保持⇧⌥按下，双击或将您要添加到选择中的文本拖到上方。

  • 要选择文件中所有出现的内容，请按⌃⌘G 。

  • 要将插入符号移至下一个或上一个出现的位置，请按⌘G要么⇧⌘G分别。

复制并粘贴代码

您可以使用标准的快捷方式进行复制⌘C并粘贴⌘V任何选定的代码片段。如果未选择任何内容，则AppCode会自动复制插入符号所在的整行。

默认情况下，当您在编辑器中粘贴任何内容时，AppCode会执行“智能”粘贴，例如，在注释中粘贴多行会自动将适当的标记添加到您要粘贴的行中。如果只需要粘贴纯文本，请按⌥⇧⌘V 。

  • 将插入符号放在线条或符号上，然后按⌥⇧⌘C / ⌘V 。

  • AppCode跟踪您复制到剪贴板的所有内容。要从历史记录中粘贴，请按⇧⌘V 。在打开的对话框中，选择您的输入，然后点击粘贴 。

  • 您可以配置剪贴板堆栈的深度。

    在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般在“ 限制”部分的“ 剪贴板中要保留的最大内容数”字段中，指定条目数。当超过指定的数字时，最早的条目将从列表中删除。

代码行

AppCode提供了一些有用的快捷方式来处理代码行。

如果您需要撤消或重做更改，请按⌘Z / ⇧⌘Z分别。

  • 要在当前行之后添加一行，请按⇧⏎ 。AppCode将插入符号移动到下一行。

  • 要在当前行之前添加一行，请按⌥⌘⏎ 。AppCode将插入符号移到上一行。

  • 要复制一行，请按⌘D 。

  • 要删除行，请将插入号放在所需的行上，然后按⌘⌫ 。

  • 要连接线，请将插入号放置在要连接其他线的线处，然后按⌃⇧J 。持续按下键，直到所有需要的元素都被加入。

    您还可以连接字符串文字，字段或变量声明以及语句。请注意，AppCode检查代码样式设置，并消除了多余的空格和多余字符。

  • 要将字符串文字分为两部分，请按⏎ 。

  • 要注释一行代码，请将插入符号放在适当的行，然后按⌘/ 。

  • 要向上或向下移动一行，请按⌥⇧↑要么⌥⇧↓分别。

  • 要删除一行代码，请按⌘⌫ 。

  • 要将XML或HTML标签属性向左或向右移动（交换），请在其上插入插入符号，或选择它，然后按⌥⇧⌘←左或⌥⇧⌘→为了正确。

代码声明

移动语句

  • 在编辑器中，将插入符号放在所需的语句处，然后按⇧⌘↑向上移动语句或⇧⌘↓将声明下移。AppCode移动选定的语句以执行语法检查。

如果在当前上下文中不允许移动语句，则将禁用操作。

完成当前声明

  • 在编辑器中，按⇧⌘⏎ 。AppCode会在结构，切片和其他组合文字中自动插入所需的尾随逗号。插入符号移动到可以开始键入下一条语句的位置。

解开或删除语句

  1. 将插入号放置在要删除或展开的表达式上。

  2. 按⇧⌘⌦ 。

    AppCode显示一个弹出窗口，其中包含当前上下文中所有可用的操作。为了更容易地区分要提取的语句和要删除的语句，AppCode使用不同的背景色。

  3. 选择一个动作并按⏎ 。

代码片段

  • 通过在编辑器中拖动来移动和复制代码片段。

    • 要移动代码片段，请选择它并将其拖到目标位置。

    • 复制代码选择，保持⌘按下，将其拖动到目标位置。

      复制操作可能在macOS中不可用，因为它可能与全局OS快捷方式冲突 。

    默认情况下启用拖动功能。要禁用它， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |常规，然后清除“ 鼠标”部分中的“ 在编辑器中启用拖放功能”复选框。

  • 要在所选代码段的大小写之间切换，请按⇧⌘U 。

    请注意，当您将切换大小写操作应用于CamelCase名称格式时，AppCode会将名称转换为小写。

  • 要注释或取消注释代码片段，请选择它，然后按⌥⌘/ 。

    要配置注释行为的设置，请使用“语言” |“首选项” |“特定语言”下“ 代码生成”选项卡上“ 注释代码”部分中的选项。编辑器代码样式 。

多个插入符

您可以同时编辑多行。例如，您可以在多行中添加注释标记，也可以删除单词中多次出现的某些字符。

  • 要添加或删除多个插入符，请按住并单击插入符的下一个位置。

  • 要删除所有添加的插入符号，请按⎋ 。

  • 您可以在列中选择多个代码。要启用或禁用列选择模式，请按⇧⌘8或使用编辑器的上下文菜单。

    当您要同时编辑多行代码时，此模式很有用。将插入符号放置在所需的符号上，然后将其插入或拖移，或者选择一个代码块，然后按指定的快捷方式即可获得相同的结果。

    作为启用列选择模式的替代方法，请按住⌥ ，然后拖动插入标记以进行多项选择。

    您可以按时更改插入符号移动的行为⌥+Arrows 。默认行为是将插入符号移动到当前单词的末尾。要更改此设置，请打开设置⌘,并导航到编辑器|一般在“ 插入符号移动”部分，从列表中选择适当的行为。

默认情况下，禁用列选择模式。启用后，该模式仅影响当前编辑器，如果关闭并重新打开该编辑器，则该模式将被禁用。

内联提示

使用内联提示使您的代码易于阅读和理解。AppCode可以显示有关变量类型，引发的错误和参数名称的提示。

类型提示

类型提示仅适用于Swift。

类型提示显示没有显式类型注释的已声明变量的类型。

要在打开的文件中显示类型提示，请确保编辑器处于焦点位置，然后选择“ 查看” |“查看”。活动编辑器|从主菜单显示“类型提示 ”。

要在所有文件中显示类型提示， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|外观，然后选择显示Swift类型提示复选框。此设置将应用于所有项目中的所有Swift文件。

如果要向变量声明中添加显式类型，请将插入符号放在变量上，按⌥⏎ ，然后选择添加显式类型 意图操作 ：

如果该意图操作不可用，请在“首选项” |“首选项”中启用它。 编辑器意向|斯威夫特 。

错误提示

错误提示仅适用于Swift。

错误提示出现在函数/方法声明和调用的旁边，并显示函数/方法可能引发的所有错误。

要在打开的文件中显示错误提示，请确保编辑器处于焦点位置，然后选择“ 查看” |“查看”。活动编辑器|从主菜单显示“类型提示 ”。

要在所有文件中显示错误提示， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|外观，然后选择显示Swift类型提示复选框。此设置将应用于所有项目中的所有Swift文件。

如果您调用的方法具有多个实现/覆盖，则AppCode将显示这些实现/覆盖中可能抛出的所有已知错误的提示。此外，在某些实现中，抛出的错误在编译时可能是未知的。在这种情况下， …符号出现在错误提示中。

错误名称旁边的向下箭头表示错误不是在方法的主体中而是在方法的替代或实现中引发。

参数提示

参数提示可用于JavaScript，TypeScript，Ruby和SQL。

参数提示显示方法和函数中参数的名称。默认情况下，这些提示仅对文字或函数表达式的值启用，而对命名对象不显示。

要隐藏参数提示， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|外观并清除显示参数名称提示复选框。

要配置参数提示， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|外观 。单击“ 显示参数名称提示”复选框旁边的“ 配置” （默认情况下，该复选框处于选中状态）。在打开的对话框中，配置参数设置，然后单击“ 确定” 。

配置提示外观

您可以更改错误的方式，并在Swift中键入提示。

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |配色方案|斯威夫特 。

  2. 查找内联提示|错误提示或内联提示|在列表中键入提示 。调整要更改的参数，例如前景色和背景色，效果等。在预览中，您可以看到提示的外观。

代码折叠

折叠的代码片段显示为阴影椭圆（ ）。如果折叠的代码片段包含错误，则AppCode会以红色突出显示该片段。

要配置默认的代码折叠行为， 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|代码折叠 。

展开或折叠代码元素

  • 要折叠或展开代码片段，请按⌘- / ⌘+ 。AppCode折叠或展开当前代码片段，例如，一个方法。

  • 要折叠或展开所有代码片段，请按⇧⌘- / ⇧⌘+ 。

    AppCode折叠或展开所选内容中的所有片段，或者，如果未选择任何内容，则折叠或展开当前文件中的所有片段，例如，文件中的所有方法。

  • 要递归折叠或扩展代码，请按⌥⌘- / ⌥⌘+ 。AppCode折叠或扩展当前片段以及该片段内的所有从属区域。

  • 要折叠或展开当前文件中的文档注释，请在主菜单中选择代码|折叠展开文档注释/折叠文档注释 。

  • 要折叠或展开自定义代码选择，请按⌘. 。

    您可以折叠或展开代码中任何手动选择的区域。

折叠或展开嵌套片段

  • 要扩展当前片段和所有嵌套片段，请按⌘⌨ *, 1 。您可以将当前片段扩展到指定的嵌套级别（从1到5）。

  • 要展开文件中所有折叠的片段，请按⌥⌘⌨ *, 1 。您可以将折叠后的片段扩展到指定的嵌套级别（从1到5）。

将环绕声与动作配合使用

您可以使用“ 环绕声”操作折叠或展开代码。

  1. 在编辑器中，选择一个代码片段，然后按⌥⌘T 。

  2. 在弹出菜单中，选择注释或区域...结束区域注释 。

  3. 如果需要，可以像往常一样命名自定义区域并折叠或展开创建的区域⌘- / ⌘+ 。

  4. 按⌥⌘.导航到创建的自定义区域。

禁用代码折叠轮廓

您可以禁用出现在左侧装订线的代码折叠轮廓。

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|代码折叠 。

  2. 清除“ 显示代码折叠轮廓”复选框。

自动保存

AppCode会自动保存您在文件中所做的更改。保存是由各种事件触发的，例如编译，运行，调试，执行版本控制操作，关闭文件或项目或退出IDE。大多数实际事件是预定义的，无法配置，但是您可以确保所做的更改不会丢失。

要强制保存所有文件，请打开“ 文件”菜单，然后单击“ 全部保存” ，或按⌘S 。

配置自动保存行为

  1. 在“ 首选项”对话框中（ ⌘, ），转到外观和行为|系统设置 。

  2. 在“ 同步”下 ，配置以下选项：

    • 在停用框架时保存文件 （当您切换到其他应用程序时）

    • 如果应用程序闲置了N秒，则自动保存文件

用未保存的更改标记文件

  1. 在“ 首选项”对话框中（ ⌘, ），转到“ 编辑器” |一般|编辑器标签 。

  2. 选中用星号标记已修改的标签复选框。

还原变更

  • 对于最近的更改，请打开“ 编辑”菜单，然后单击“ 撤消” ，或按⌘Z 。

  • 有关过去更改的更详细视图，请使用“ 本地历史记录” 。

  • 为了最稳健地跟踪所有更改，请使用AppCode支持的版本控制系统 。
