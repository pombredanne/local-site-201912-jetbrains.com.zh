执行

我们首先介绍如何调试Java应用程序。如果用户拥有具有main方法的类，则应使用Java应用程序运行配置来运行/调试此类程序。

创建运行配置实例

可以分别使用main方法或MPS项目为类创建Java应用程序或MPS实例运行配置。转到运行->编辑配置菜单，然后按下带有“ +”的按钮，如下图所示：

出现一个菜单，从中选择Java应用程序，然后将创建一个新的Java应用程序配置：

如果选择Java Application ，则可以指定要运行的Java类，以及一些可选的配置参数：

应为每个运行配置指定一个名称，并应指定一个主节点，即具有main方法的类。VM和程序参数也可以在配置中指定。查看“运行配置”一章，了解有关运行配置的更多信息。

调试语言定义

如果要调试MPS语言定义代码，请选择MPS实例 。MPS将使用使用您的语言的项目（也可能是当前项目）启动MPS 的新实例，并且您将在原始MPS实例中设置断点并进行调试。

在“ 调试配置”对话框中，您需要指示要在新的MPS实例中打开哪个MPS项目-通过选中“ 打开当前项目”复选框来选择当前的MPS 项目 ，或在下面的字段中指定的任何项目。一旦新的MPS实例启动，您也可以将其保留为空，并从菜单中创建/打开项目。

调试配置

要调试运行配置，请从配置菜单中选择它，然后按“ 调试”按钮。调试器启动，并在下面显示“ 调试器”工具窗口 。

工具中有两个选项卡：一个用于控制台视图，另一个用于调试器视图。在控制台中，显示了应用程序的输出。

断点

下一节将介绍断点的用法。

设置断点

可以在语句，字段或异常上设置断点。要设置或删除断点，请在编辑器中的某个节点上按Ctrl-F8或单击该节点附近的左边界。断点在左边距标记为红色，编辑器内部为粉红色线，并在断点节点周围带有红色框。从断点对话框创建异常断点。

启动程序时，调试器无法停止的断点会特别突出显示。

当调试器在断点处停止时，当前断点行标记为蓝色，并且该断点的实际节点周围用黑框装饰。

如果程序在其上停止的节点的单元格位于表内部，则表单元格将突出显示而不是一行。

通过“断点”对话框查看断点。

可以通过“断点”对话框查看项目中设置的所有断点。

Java断点功能包括：

  • 现场观察点；

  • 异常断点；

  • 暂停Java断点策略；

  • 相关的断点数据（如引发的异常或更改的字段值）显示在变量树中。

在断点处检查程序的状态

在断点时，可以使用“调试器”选项卡检查程序的状态。共有三个面板：

  • “框架”面板，其中包含使用组合框选择的线程堆栈框架的列表；

  • “变量”树，显示在所选堆栈框架中可见的可观察对象（变量，参数，字段和静态字段）；

  • “手表”面板，其中包含手表及其值的列表。

在Java调试器中，可从变量树的上下文菜单中使用“复制值”操作。

运行

控制程序的执行

  • 要跳过，请使用运行->跳过或F8。

  • 要退出某个方法，请使用运行->退出或Shift-F8。

  • 要进入方法调用，请使用运行->单步执行或F7。

  • 要恢复程序执行，请使用“恢复”按钮或运行->恢复或F9。

  • 要手动暂停程序，请使用暂停按钮或运行->暂停。手动暂停（即不在断点处）时，有关变量的信息不可用。

在“调试器”窗口中有一个工具栏，可从其中执行步进操作。

表达方式

表达评估

MPS Java调试器允许用户在调试期间使用程序堆栈中的信息来评估表达式。之所以称为低级评估，是因为只允许用户使用生成的代码中的纯Java变量/字段，而不能使用高级源代码中的实体。

要激活评估模式，应在断点处停止程序。按Alt-F8，将出现一个对话框。
在对话框中，有一个MPS编辑器，其中带有语句列表。可能在此处编写了一些代码，这些代码使用了堆栈框架中的变量和字段。要评估此代码，请按评估按钮。评估值将显示在下面的树形视图中。

要从编辑器评估一段代码，请选择它并按Alt + F8，然后代码将被复制到评估窗口。

手表

监视API和Java调试器的低级监视已实现。 “低级”意味着用户可以使用堆栈上可用的变量来编写表达式。要编辑手表，必须指定一个所谓的“上下文”（使用的变量，静态上下文类型和此类型）。如果当前有可用的堆栈框架，则将自动填充上下文。

可以在“调试”工具窗口的“监视”树中查看监视。可以通过上下文菜单或工具栏按钮来创建，编辑和删除手表。
