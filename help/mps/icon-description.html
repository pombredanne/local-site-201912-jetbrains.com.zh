图标说明

图标描述语言有助于描述和实例化各种MPS元素的图标：概念，操作等。

语言有两个目标：
1。提供用于快速原型制作的工具（例如，为概念制作新图标）
2。使图标成为可扩展的语言构造

第一印象

只要在MPS语言定义语言中需要使用图标，就可以输入所需图标的文字说明，而不用指向现有的.png文件。

 

jetbrains.mps.lang.resources包含两个构造：

 

  • icon {}将图像表示为javax.swing的实例。图标类。

  • iconResource {}返回jetbrains.mps.smodel.runtime的实例。IconResource类。

创建图标原型

在描述图标时，您可以从“ 创建图标”意图中获得帮助，该意图提供了一种自动方式来创建图标的文本描述，从而快速对其进行原型制作。

调用意图将导致创建简单的图标定义。

此定义描述了一个圆形图标，其中带有字母“ X”。重新生成后，生成的图标将生效并显示在UI中。

语言说明

图标说明由图层组成，每个图层可以是以下任意一个：

  • 原始的图形形状

  • 从文件加载的自定义图像

  • 一个人物

然后将这些层组合成单个图像以表示图标。可以使用以下图标描述：

  • 在语言定义语言的不同位置指定图标-在概念，动作等需要图标的地方

  • 在MPS UI中应该返回一个Icon的方法中

扩展语言

该语言可以扩展。要添加新的图标类型，您需要创建一个新概念并使其实现Icon界面。Icon界面代表所需的图标，在制作过程中将转换为.png文件。

生成模型后，将从输出模型中收集所有图标 ，并调用它们的generate（）方法。它们创建与对应图标描述的图像相对应的.png文件。当请求图标资源时（例如，使用icon {}语法），使用相应模块的类加载器加载由Icon.getResourceId（）方法引用的资源，并将其转换为Java Icon对象。

icon {}与iconResource {}

资源语言中有两种构造来加载资源。 icon {}将图像加载为javax.swing的实例。Icon类，而iconResource {}返回jetbrains.mps.smodel.runtime的实例。IconResource类。第二个用于MPS核心方面，不应依赖于javax.swing包。所有与UI相关的代码都使用icon {} 。
