具有语言模式的样本项目

MPS与示例项目捆绑在一起。这里讨论的许多模式已经在languagePatterns示例项目中实现。您可以在MPS中将其打开 （在MPS欢迎屏幕中单击“ 打开示例项目 ”），并检查涉及示例项目的本章模式的实际实现。


在语言定义和沙箱解决方案中，这些模式都被组织成虚拟包，以帮助您识别形成各个模式的元素。

初步提示

命名概念

许多概念需要提供一个字符串属性来保存节点的名称。由于概念的名称具有某些特殊的性质，需要由MPS以特殊的方式识别和处理（例如，在代码完成菜单以及项目视图，节点浏览器，调试器工具窗口等中标记节点） ，建议从INamedConcept概念接口继承属性。因此，您的许多概念以及几乎所有根概念都应声明为实现 INamedConcept 。

有效标识符

如果要扩展BaseLanguage且概念名称直接转换为Java标识符，则应考虑从IValidIdentifier概念接口继承了name属性，该接口扩展了INamedConcept 。它将通过约束确保名称符合有效Java标识符的条件。

唯一名称

通常，您要确保具有相同概念的某些节点组的名称唯一。您可以使用约束或非类型系统规则。非类型系统规则使您可以选择自定义错误消息，甚至可以为用户提供快速修复以自动解决问题。在“计算器”教程的“ 计算器”中 ，一个示例唯一性检查InputField的名称如下所示：

检查规则check_InputField {适用于concept = InputField，因为inputField覆盖了false do {如果（inputField.parent：Calculator.inputField.any（{〜it => it.name：eq：inputField.name && it：ne：inputField;}） ）{错误“名称重复” + inputField.name-> inputField; }}}

用约束指定的相同约束将如下所示：

概念约束InputField {可以是子项， 可以是父项， 可以是祖先属性{name}获取： 设置： 有效：（propertyValue，node）-> boolean {node.parent：Calculator.inputField.where（{〜it => it。名称：eq：propertyValue;}）。size <= 1; } < >默认范围}

请注意，由于以下几个原因，约束在这里可能不是最理想的：

  • 您不能指定快速修复

  • 您不能自定义错误消息

  • 非类型系统规则仅在重复名称下划线表示错误，而约束则防止将无效值插入模型并以红色字体显示无效值，这使错误看起来更加严重

没有看到类型系统错误

类型系统在后台运行，有时可能会很慢，无法将其结果传递到编辑器中。点击F5以刷新编辑器中的所有错误和警告消息。另外，请确保已关闭“ 省电模式”设置。在省电模式下 ，类型系统仅在用户明确要求下运行（F5）。

常用表达

默认情况下，属性可以是以下三种类型之一： 整数 ， 布尔值和字符串 。如果要使用更多自定义的数据类型，则必须定义一个数据类型并通过正则表达式限制允许的值。例如， BaseLanguage以此方式定义浮点类型。打开FloatingPointConstant概念（Control / Cmd + N），然后查看属性值 -它具有_FPNumber_String的类型。导航到_FPNumber_String （Control / Cmd + B）的定义，以查看受限数据类型的示例：

安全的操作人员和操作

BaseLanguage及其核心扩展为Java中需要付出更多努力才能实现的常见操作提供了一些方便的快捷方式-例如，在比较之前检查null ，区分空列表和null列表，调用equals而不是“ ==”等。最常用的列表：

基本模式

容器-组件

在languagePatterns示例中说明为容器组件虚拟包。

容器（在我们的例子中为FruitPlate）包含多种元素（苹果，橙子）的常见情况。抽象概念（Fruit）用作容器中的占位符，然后由语言用户显式提供具体的子概念（Apples，Oranges）。这些子概念可能具有完全不同的视觉外观。


另请参见无缝替换模式，以了解如何允许用户毫不费力地将Apple切换为Oranges。当前的实现要求用户首先选择整个节点（Apple或Orange），然后才通过代码完成提供替代方案。

定制展示

在languagePatterns示例中说明为自定义表示虚拟包。

容器中保存的组件是从ComponentUsages引用的。做基准，但是，应与组件的名字中含有含集装箱的名字联系在一起 -无论是在完成菜单，并在编辑器中。


此外，作用域规则应确保对任何组件的引用都只能存在，并且对完成菜单进行过滤，以便不提供已经引用的组件 。

声明-参考

在languagePatterns示例中说明为声明引用虚拟包。

声明和参考的典型模式-活动中的歌手可以和他们的表演一起编入议程。提供不同类型的表演 。

作用域规则确保：

  • 仅将当前活动中列出的歌手添加到其议程中

  • 每位歌手在组合表演中只能被列出一次

  • 一个方便的意图（ IntroduceSinger ）可用于根据演奏中键入的字符串来创建歌手（通常称为“根据使用创建”或“引入变量”重构）。

流利的编辑

在languagePatterns示例中显示为流畅编辑虚拟包。

一个创建类似文本的编辑体验并实现许多编辑器食谱（ Editor cookbook ）中提到的建议的示例。它实现了一种用于指定绘图命令（线，矩形）的简单语言：

  • 按下Enter键后会插入一个空行，由于为子集合指定了默认节点工厂，因此空行可以放置在任何地方

  • 由于实现了IDontSubstituteByDefault概念界面，因此空行不会在完成菜单中显示为选项

  • 在空行上键入将插入所需的项目（线或矩形）

  • 当代码块的主体为空时，光标位于第一行（标题旁边），则可以在第一行开始编辑

  • 如果一个节点被另一个替换， NodeFactories将把值传播到新节点

  • 由于左侧的转换，可以在绘制命令的左侧指定一个可选的“线型”子级

  • 包装器允许在空行上键入所需的线条样式，并且会自动创建一个中间的“ IncompleteCommand ”

  • IncompleteCommand期望键入直线或矩形，并将立即替换为所需的绘制命令

  • 绘制命令定义别名和编辑器，以便它们以相同的词开头

  • 具体的子概念可以重用draw命令的抽象超级概念中的编辑器

  • 在命令的左侧键入将提供允许的前缀

  • 删除前缀（实线，虚线）将仅正确删除前缀

覆盖编辑器组件

在languagePatterns示例中说明为override-editor-component虚拟包。

使用在子概念（ Truck ）中被覆盖的编辑器组件的示例。Car中的编辑器使用也在Car中定义的编辑器组件CarProperties 。一个子概念（ 卡车 ）覆盖与TruckProperties编辑器组件包含它自己的属性的CarProperties编辑器组件。Car中的编辑器将使用Trucks编辑器组件的Trucks变体和Cars的Car变体。

无缝替代

在languagePatterns示例中说明为无缝替代虚拟包。

在不同的相关子概念之间无缝切换（替换）的示例。一个请求包含一个“描述”，可以是字符串 ， 简单形式或复杂形式 。完成菜单使用户可以选择请求的描述类型。如果用户仅键入文本，则将在“ PickTheRightDescriptionType ”替代操作中自动选择基于“字符串”的描述。每个描述概念的第一个编辑器单元格对替换都很敏感（通过单元格的“菜单”属性设置），因此提供了在带有完成菜单的描述类型之间进行切换的选项。“ 转换器 ”节点工厂包含保留部分描述信息并将其传播到新实例化的描述概念的代码。


层次范围

如果要定义某种引用，则通常需要限制引用的范围，以便它们只能指向某些位置的节点或与引用“相距”的节点。此外，定义可能会隐藏其他具有相同名称的引用，而这些引用的名称距离引用更远。例如，以Java的局部变量隐藏参数声明或字段为例。在MPS中，当定义引用时，可以在语言定义的“ 约束”方面提供引用的范围。对于分层作用域，您可以在引用中使用继承的作用域类型，然后让您的代码容器（即类，方法，块语句等）实现ScopeProvider概念接口。在其getScope（）方法中，您实现了为想要指向它们的引用检索某种候选目标的逻辑。有关详细的操作方法，请查阅“ 范围”文档 。
