正则表达式语言

正则表达式是被广泛使用的最早的DSL之一。如今，几乎所有现代编程语言都以一种或另一种方式支持正则表达式。MPS也不例外。MPS中的正则表达式支持通过基本语言扩展来实现。

我们还建议您查阅《 正则表达式食谱》 ，以更全面地介绍该语言。

定义正则表达式。

Regexp语言允许您创建java.util.regex的实例。模式类使用特殊的模式表达式： / regexp / 。在生成的代码中，MPS为每个定义的模式表达式在最外面的类中创建一个最终的静态字段 ，因此该模式在应用程序运行时仅被编译一次。

模式模式= / [az] + /

有三个选项，您可以在正则表达式的斜杠后添加。

/一世

不区分大小写的匹配

/秒

将字符串视为单行，点字符类将包括换行符

/米

多行模式：^和$字符匹配字符串中的任何行（而不是字符串的开头或结尾）

可以通过在编辑器中键入或删除字符或通过检查器来打开/关闭选项。Inspector中提供了生成的正则表达式预览。

重用定义

要为整个项目中的常用模式重用正则表达式，请创建一个单独的根：

型号->新建-> jetbrains.mp.baseLanguage.regexp->正则表达式

每个可重复使用的正则表达式都应该有一个名称和一个可选的描述。

regexp标识符{//无描述（标识符：[az AZ _] [az AZ _ 0-9] +）}

模式匹配运算符

如果字符串与指定的模式匹配，则=〜运算符返回true。

“字符串或变量” =〜/ regexp /

捕获文字

表达式中括号的可选使用将创建捕获组。为了能够在以后引用该组，最佳做法是给它起一个名字。

/ ^（名称：[az AZ _] [az AZ _ 0-9] +）/

例子

if（“ any string” =〜/ ^＃define（identifier：[az AZ _] [az AZ _ 0-9] +）/）{process（identifier）; }

如果模式与字符串匹配，则将匹配的值捕获到标识符中，并可以在if块中进行访问。

不要忘记查看正则表达式食谱 ，以更全面地介绍该语言。
