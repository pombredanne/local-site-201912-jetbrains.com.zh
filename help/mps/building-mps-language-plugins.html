构建MPS语言插件

将语言作为MPS插件分发与IntelliJ IDEA的打包语言非常相似。您遵循构建IntelliJ IDEA语言插件中描述的相同说明，但有两个重要区别：

  1. 如所附的屏幕截图所示，您的构建脚本应取决于mps 。


    此外，取决于mps，您的语言可以利用整个MPS基础结构和整个语言集，而mpsPlugin则将 IDEA插件限制为仅IntelliJ IDEA中可用的功能子集。

  2. 从命令行运行Ant构建脚本时，您可以设置一个artifacts.mps属性（而不是artifacts.mpsPlugin ），并使其指向MPS安装主目录。


    注意：由于在上面的示例构建脚本中，我们将 mps 的工件位置设置 为 $ mps_home ，我们可以改为设置-Dmps_home（仅当构建机器上的MPS位置与构建脚本中设置的路径不同时才需要）。

然后可以将生成的插件导入MPS。您可以手动或通过MPS插件管理器安装插件。如果手动安装，则必须将下载的插件zip文件解压缩到MPS插件文件夹中。

  • Windows和Linux：通常为USER_HOME / MPS_version / config / plugins或MPS_HOME / plugins

  • MacOS： 〜/ Library / Application Support / MPS_version /或MPS_HOME / plugins

重新启动后，导入的语言将在MPS中可用。
