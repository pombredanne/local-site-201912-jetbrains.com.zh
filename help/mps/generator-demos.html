发电机教程

欢迎使用更新的生成器教程，它将指导您完成在MPS中定义和扩展语言生成器的过程。
本教程由七个部分组成，它们的复杂性逐渐增加。所有演示都共享相同的基础故事-将XML代码转换为Java Swing组件 。每个演示都基于前一个演示，并演示了更高级的（或有时只是不同的）功能，实践和方法。

所有教程演示都与MPS发行版捆绑在一起，作为一个示例项目，名为tutorial-demos 。如果您想查看现有代码或跳过某些演示，这将对您特别有帮助。

尽管所有演示都在逻辑上相关，但是出于教育目的，我们将在每个演示中创建一个新的生成器。由于生成器并不存在于真空中，而是始终属于一种特定的语言，因此我们将为每个演示创建一种新的语言来包装生成器。这些语言（演示6和7除外）没有引入自己的新概念 （即语法）。相反，它们为现有概念赋予了新的语义 。

例如，名称为'button'的xml元素不再只是一个xml元素，而是代表GUI组件button的'虚拟概念'。

请注意，这不是进行面向语言的编程的典型方法。语言很少是纯粹的“虚拟”语言，无需添加或更改任何语法元素即可定义语义。为我们的演示选择这种方法可以使我们仅专注于我们的主要主题-生成器。

如何使用本文件

最好的方法是在计算机上安装JetBrains MPS ，设置一个新的空演示项目，然后执行演示中描述的所有步骤。这样，您将在发电机开发及其他方面获得丰富的经验。

第二种选择是阅读文档，或者浏览MPS发行版中包含的Generator演示项目。生成器演示项目包括所有演示语言和生成器，以及我们将在这些演示中创建的测试模型。
生成器演示项目位于： {mps} / samples / generator_demo

设置演示项目

如果您采用更具教育性的方式并计划从头开始构建代码，则在开始单个练习之前，需要做一些小的设置工作。

新项目

  • 选择“ 文件”->“新建项目”

  • 创建一个新的空项目

  • 输入项目名称：“ generator_demo”


在本演示中，我们将为xml元素定义新的语义，创建包含一些xml的测试模型，并从该xml生成Java代码。XML语言的语法已经在SampleXML示例语言中为您定义，该示例语言还与MPS发行版捆绑在一起作为示例项目。SampleXML项目位于：

$ HOME / MPSSamples3.0 / samples / sampleXML

您显然可以通过在MPS中打开SampleXML项目来单独尝试。但是，在演示中，我们将仅重用jetbrains.mps.sampleXML语言。首先，让我们将jetbrains.mps.sampleXML语言添加到“ generator_demo”项目中。

  • 右键单击左侧“ 项目视图”面板中的generator_demo节点，然后选择“ 项目路径” 。Alt + Enter键快捷键将具有相同的效果。

  • 点击+按钮添加一个新模块

  • 选择$ HOME / MPSSamples.3.0 / sampleXML / languages / sampleXML / jetbrains.mps.sampleXML.mpl文件

现在可以轻松访问'jetbrains.mps.sampleXML'语言。

右键单击sampleXML语言节点，然后选择“ Rebuild language ... ”，以便可以使用该语言。

新的演示解决方案

  • 在项目的弹出菜单中选择“ 新解决方案”

  • 输入解决方案名称：“ test_models”

新的测试模型

  • 选择“ 新模型”解决方案的弹出菜单

  • 输入模型名称：'generator_demo.test1'

  • 在模型属性对话框的“ 使用的语言”部分中添加“ jetbrains.mps.sampleXML”语言

现在，您应该拥有一个带有单个模型的解决方案，并准备使用刚刚导入的simpleXML语言编写一些xml。

在'text1'模型中，创建新的Document节点，并将其命名为'Button'。

  • 在模型“ test1”中创建一个文档“按钮”

  • 在此文档中添加根元素“按钮”

  • 以相同的方式创建包含元素“标签”的文档“标签”

如果您在屏幕上看到的内容与上述屏幕截图中的类似XML的编辑器有很大不同，则您可能尚未重建sampleXML语言。现在就做。

测试生成

  • 在模型“ test1”的弹出菜单中选择“ 生成文本 ”

文本生成的结果将在MPS编辑器中显示为新选项卡，并且与输入大致相同。

现在，我们已经准备好继续为那些原本毫无意义的xml概念定义语义！
