类型扩展方法

jetbrains.mps.extensionMethods语言提供了一种使用新定义或重写的方法扩展任何有效MPS类型的方法，类似于Java静态方法。

“这是一个具有令人惊讶的行为的普通字符串！“ .myNewMethod（）”开发愉快！“ .isAPleasureMessage（）int num = ...num.isPrime（）

静态方法永远不会成为扩展类的内部组成部分，而必须始终指定“ extended”对象作为扩展方法的参数之一进行操作，而使用扩展方法 ，新方法将直接添加到扩展方法列表中目标类型上可用的操作。

因此，假设我们要向字符串类型添加一个反向方法，而不是旧的“静态方法”方法：

公共静态字符串reverse（string target）{//反转目标}

我们将通过New-> jmbaseLanguage.extensionMethods / type extension创建新的扩展方法 ，定义新方法并将其绑定到字符串类：

字符串{的公共扩展方法MySampleMethods {（ > public string reverse（）{//反向字符串，通过'this'引用它）}}

可以使用完全相同的机制来覆盖现有方法。当需要调用原始方法时，只需在此方法上调用它即可：

字符串{的公共扩展方法MySampleMethods {{ >公开字符串trim（）{this.trim（）+“ trimmed” //调用原始trim（）实现}}

由于MPS可以很好地通过扩展方法机制在视觉上区分原始方法和重写方法，因此您可以从下拉列表中选择正确的方法 ，这不会出错。


显然，该机制也可以用于在您自己的域对象上实现正交概念：

公共扩展方法my_type {< > public int foo（）{return this.bar（）; }}

使用上面的声明，可以在类型my_type上写一个操作：

my_type var = ...; var.foo（）;

根节点

使用方法扩展类型有两种同样好的方法。类型扩展允许您在一个位置将方法添加到单个类型，而简单扩展方法容器在需要一个位置来实现多种不同类型的正交概念时派上用场。
