开放式API-从代码访问模型

可通过Open API以编程方式方便地访问语言存储库，项目模块，语言和模型。 Open API使您可以对模型进行受控访问，还可以为某些方面（例如持久性）提供自己的实现。

将分别讨论这两种用法类型。

注意：本文档旨在提供有关Open API哲学的一般性高级概述，并为您提供有用的指向API的入门链接。有关如何使用它的技术细节，请咨询 。

 

有关使用API的动手代码示例，请查看使用模型和模块依赖性

使用Open API

该API位于org.jetbrains.mps.openapi包下，并分为几个子包：

  • 事件 -包含用于更改模型的事件类

  • 语言 -提供一组接口来访问已编译的语言并检查其结构

  • 模型 -为您提供检查和修改模型（AST）的方法

  • 模块 -抽象存储库和模块，以逻辑方式组织模型

  • 持久性 -拥有扩展和定制模型持久性机制所必需的接口

  • 仓库 -容纳监听者特定仓库的事件

  • util-包含实用程序类，例如ProgressMonitor，以将持久的操作挂接到UI进度指示器

API识别这些逻辑元素，其中上面的元素包含列表中下面的元素：

  • 资料库

  • 模组

  • 模型

  • 节点

  • 属性，参考

Open API还可以识别与上述元素正交的元结构。元结构包含以下关键元素：

  • 语言

  • 概念，枚举

  • 成员（例如属性，链接和枚举文字）

每个节点都有一个关联的概念 。概念属于语言 。语言可以保留指向其来源模块的指针，从而为语言用户提供一种详细研究语言的方式。

该API使您能够浏览整个存储库并研究其模块，其模型以及这些模型所基于的节点。此外，API能够搜索元素的用途或通过其名称查找任何元素，而无论其在存储库中的位置如何。您还可以修改所有这些元素，保存更改或从持久性存储中重新加载它们。API将检测内存及其持久存储中对模型的冲突修改。

使用命令

Open API还提供了更改模型的方法。修改需要作为传递到存储库进行处理的命令来执行。典型的模型更改/编辑动作可以撤消/重做，而对模块结构进行重大更改的动作则不能。更改分为三种：

  1. 模型和节点可以通过不可撤销的动作进行更改

  2. 模块，它们的属性和依赖性可以通过存储库命令来执行

  3. 对项目的重大更改，例如VCS更新或完整的项目重新加载，都需要执行外部更新操作 -在这种情况下，不会触发节点级通知，仅触发模型更换或模块更改通知。

根据命令的类型，这些命令将在开始更改模型之前自动分配所有必要的读取或写入权限。更改通知会发送到节点，模型，模块或存储库级别的注册侦听器。

并发访问

Open API设计用于并发访问，只要调用代码正确使用了提供的同步机制，Open API即可正确处理通过Open API同时访问模型的多个线程。Open API将拒绝所有不正确同步的请求，从而保留模型的完整性。

具体来说，在开始执行操作之前，您需要获取读或写操作，否则将从代码中引发异常。

使用SRepository 。 getRepositoryAccess（）. applyChanges（）可以将您的更改应用于整个存储库SRepository 。 getModelAccess（）. runXXXAction（）来运行读/写操作和SRepository 。 getModelAccess（）. executeCommand（）来运行命令。命令自动获得所有写权限，因此始终获得对存储库的独占访问权限。两种方法都提供了一个异步变体，可以在EDT中异步运行提供的动作。

自定义持久性

默认情况下，MPS以XML或二进制格式将模型存储为平面文件。为了允许您自定义模型的持久化方式，Open API提供了几种选择。

备用文件格式

更改存储在平面文件中的模型数据的格式是自定义模型持久性的最简单方法。您只需使用您选择的文件扩展名（通过PersistenceFacade ）注册自己的ModelFactory ，只要发现该文件扩展名，MPS就会使用该工厂实例化您的自定义SModel实现。您还需要提供并注册自己的SModelIdFactory和SNodeIdFactory实现 。

替代存储

如果您喜欢冒险，例如想要从数据库或其他非文件存储中加载模型，则需要另外提供一个ModelRootFactory ，它可以创建自定义ModelRoot实例。然后，这些模型根将处理您选择的存储的所有详细信息，以便加载/保存模型。通常，您可能还需要捆绑UI，以允许用户配置数据源详细信息，例如数据库位置，用户名或其他。

MPS附带的xmlPersistence示例项目显示了一个实现定制持久性的重要示例。有关如何在MPS和IDEA中为MPS模型提供自己的持久性的实用详细信息，请参阅《 Custom Persistence Cookbook》的描述。

自定义查找用法和导航参与者

提供FindUsagesParticipant的自定义实现将使您可以使用自定义持久性在模型中优化FindUsages 。同样， NavigationParticipant的自定义实现将有机会优化“ 转到根/类/符号”操作。通过让PersistenceFacade提供自定义参与者，您可以选择直接访问持久性存储，从而加快搜索和导航的速度，而不是让默认的查找用法和导航实现将所有模型加载到内存中并以标准方式进行处理。
