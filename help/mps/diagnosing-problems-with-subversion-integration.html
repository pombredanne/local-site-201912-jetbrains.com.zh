使用Subversion集成诊断问题

如果Subversion集成发生任何问题，请随时与我们的支持联系 。为了便于检测，定位和解决问题，请提供有关Subversion与MPS集成的详细信息。本主题列出了必要的信息，并说明了如何检索它。

诊断Subversion问题通常需要以下数据：

  • MPS版本和内部版本号，

  • 使用的操作系统。

在此页：

  • 常规VCS设置
  • 颠覆设置
  • 本地工作副本格式
  • 所使用分支的父文件夹
  • 启用svnkit日志记录

常规VCS设置

在“设置/首选项”对话框的“ 版本控制”页面上指定了应用于项目的常规版本控制设置。 通过按打开设置/首选项对话框 Ctrl+Alt+S或选择文件|设置 Windows和Linux或MPS | macOS的首选项 ，然后单击版本控制 。

颠覆设置

特定于Subversion的设置是在Subversion页面上的Settings / Preferences对话框的Version Control节点下配置的。 通过按打开设置/首选项对话框 Ctrl+Alt+S或选择文件|设置 Windows和Linux或MPS | macOS的首选项 ，以及单击“ 版本控制”下的“ Subversion ”。

本地工作副本格式

工作副本格式是创建工作副本所遵循的Subversion格式。要查看工作副本格式，请选择“ 查看” |“复制”。工具窗口|主菜单上的版本控制 。在打开的“ 版本控制”工具窗口中，切换到“ Subversion工作副本信息”选项卡。

所使用分支的父文件夹

要查看分支的配置，请在“ 版本控制”工具窗口的“ Subversion工作副本信息”选项卡中，单击“ 配置分支”链接。

启用svnkit日志记录

如果保存在MPS日志中的数据不足以解决问题，并且问题与通信协议或身份验证有关，请启用svnkit日志记录 ，重现问题，然后附加idea.log 。

要启用svnkit日志记录，请添加-Djavasvn.log=true取决于所使用的操作系统，将其更改为以下文件之一：

  • Windows ： MPS.exe.vmoptions

  • Linux ： MPS.vmoptions

  • macOS ： http : //stackoverflow.com/a/13581526/72788
