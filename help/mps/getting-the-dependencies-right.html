动机

模块和模型通常通过各种类型的依存关系网络互连。假设您已经了解了MPS项目结构页面上所描述的模块和模型的基本原理和分类，那么现在我们可以更深入地学习所有细节。

在MPS中正确获得依赖关系是缺乏经验的用户以及经验丰富的退伍军人感到沮丧的常见原因。此页面旨在一劳永逸地解决问题。您应该能够找到按各种模块和依赖项类型分类为各节的所有相关信息。

所有编程语言都有“导入”的概念。在Java中，您可以获得软件包和“ import”语句。在Ruby或Python中，您具有“模块”和“要求”或“导入”语句。在MPS中，我们提供了一种类似的机制，用于以跨语言通用的方式打包代码和表达依赖关系-将代码打包到模型中，并且这些模型可以表达相互依赖关系 。

另外，由于MPS是一种多语言开发环境，因此模型可以指定在其中启用的语言（即语法）。这与使用Java，Ruby或其他语言编写和指定要使用的语言的代码不同。

有用的键盘快捷键

无论何时将其放置在左侧“ 项目工具”窗口中的模型或节点上 ，或者在编辑器中进行编辑时，都可以使用键盘调用快速操作，从而将依赖项或使用的语言添加到当前模型及其包含的解决方案中。

  • Control + L-添加使用的语言

  • Control + M-添加依赖项

  • Control / Cmd + R-添加一个包含给定名称根概念的依赖项

  • Control / Cmd + Shift + A-弹出一般的动作选择对话框，您可以在其中选择适用于当前上下文的所需动作

解

解决方案表示以一种或多种语言编写的程序。它们通常有两个作用：

  1. 沙盒解决方案 -这些解决方案包含最终用户代码。IDE不会以任何特殊方式处理代码。

  2. 运行时 解决方案 -这些解决方案包含其他模块（解决方案，语言或生成器）所依赖的代码。该代码可以包含MPS模型以及Java类，源文件或jar文件。每当在外部对其进行编译或更改时，IDE都会重新加载这些类。

  3. 插件解决方案 的S -这些解决方案以某种方式扩展IDE的功能。他们可以提供新的菜单项，添加侧面工具面板窗口，从“项目设置”对话框定义自定义首选项屏幕，等等。同样，只要更改，MPS就会继续重新加载类。此外，IDE功能将相应更新。

我们将从适用于所有解决方案的属性开始，然后介绍运行时和插件解决方案的细节。

共同

物产

  • 名称 -解决方案的名称

  • 文件路径 -模块文件的路径

  • 生成器输出路径 -指向应在其中放置生成源的文件夹

  • 左侧面板 -包含模型根 ，每个模型根可容纳一个或多个模型 。

  • 右侧面板 -在左侧面板中当前选择的模型根目录下显示目录结构。文件夹和jar文件可被选择和标记/未标记为当前模型根的模型 。

模型根类型

解决方案包含模型根，而模型根又包含模型。每个模型根通常都指向一个文件夹，并且所包含的模型位于该文件夹的一个或多个子文件夹中。根据所包含模型的类型，模型根有不同的种类：

  • 默认值 -包含MPS模型的标准MPS模型根类型

  • java_classes-一组包含Java类文件的目录或jar文件

  • javasource_stubs-一组包含Java源代码的目录或jar文件

    当作为模型包含在项目中时，目录或jar文件中的Java类将成为MPS模型池的一等公民，并将可用于导入这些存根模型的其他模型的直接引用。在MPS中包括类和jar的第二个选项是使用Java选项卡并将它们定义为库。在这种情况下，将加载类，但不能直接从MPS代码中引用这些类。这对于存根模型所需的库很有用。

依存关系

解决方案的依存关系是其他解决方案和语言，从该解决方案中可以看到其模型。

然后， Export标志指定是否将依赖项作为依赖项可传递地添加到所有依赖当前解决方案的模块。例如，模块A与出口依赖于B 上 ，并且，c取决于A，那么C取决于B.

使用的语言

解决方案的模型可以使用的语言和devkit列在使用的语言中。在模型级别指定了使用的语言 ，并且模块上的“ 使用的语言”选项卡仅显示其所有模型的使用的语言的集合。

爪哇

这是不同解决方案的最大不同之处 。

Java选项卡包含几个选项：

  • 解决方案类型 -MPS对不同类型的解决方案的处理略有不同，并且可以访问不同的MPS内部

    • 无 -默认，用于用户代码，不需要任何特殊的类加载策略-用于沙盒解决方案

    • 其他 -由其他语言和解决方案所利用的典型可重用代码库使用-用于运行时解决方案

    • 核心插件 -由与MPS IDE内核相关联的代码使用，并且需要相应地管理其类加载-用于插件解决方案

    • 与MPS编辑器绑定的代码使用的编辑器插件 ，需要与其他编辑器同步管理其类加载-用于仅增强编辑器的插件解决方案

  • 在MPS中进行编译 -指示是否应使用Java编译器直接在MPS中编译生成的工件，以及生成过程的一部分

  • 源路径 -应该使项目中的其他Java代码可以使用的Java源

  • 库 -一种或多种解决方案模型中Java代码在运行时所需的Java类和jars

刻面

  • Idea Plugin-检查解决方案是否与IDE功能挂钩

  • Java-检查解决方案是否以某种方式依赖Java。在大多数情况下，请保持选中状态。

  • 测试 -检查解决方案是否包含测试模型

解决方案模型

解决方案包含一个或多个模型。模型可以相互嵌套并形成层次结构，就像Java包可以这样。属性对话框隐藏了一些可以调整的配置选项：

依存关系

可以在此处列出来自当前或导入模块的模型，以便可以在此模型的代码中访问其元素。

使用的语言

此模型使用的语言必须在此处列出。

高级

“高级”选项卡上列出了一些其他选项：

  • 不生成 -从代码生成中排除此模型，可能是因为无法有意义地生成它

  • 文件路径 -模型文件的位置

  • 语言从事代 -所需的适当产生的模型的列表的语言，如果语言不直接或间接地与任何使用的语言相关联的，因此发电机失败自动寻找这些语言

虚拟包装

模型中的节点可以在逻辑上组织成虚拟包的层次结构。使用节点的上下文弹出菜单中的“ 设置虚拟包”选项，并指定一个名称，可能用点符号分隔嵌套的虚拟文件夹名称。

将外部Java类和jar添加到项目-运行时解决方案

运行时解决方案表示MPS中的可重用代码库。它们可能包含保存MPS代码的模型以及引用外部Java源代码，类或jar文件的模型。要在项目中正确包含外部Java代码，您需要执行一些步骤：

  1. 创建一个新的解决方案

  2. 在“解决方案属性”对话框（ Alt + Enter ）中，指定Java代码，例如：

    1. Common选项卡 -单击添加模型根 ，选择类或JAR，为Java源代码，并导航到你的lib文件夹中选择javasource_stubs javaclasses。

    2. 选择属性对话框右侧面板中列出的文件夹或jar，然后单击蓝色的“模型”按钮。

    3. 同样，在Java选项卡上，将所有jar或类根文件夹添加到窗口的库部分，否则使用库类的解决方案将无法编译。使用java_sourcestubs时 ，请将源添加到Java选项卡窗口的“ 源路径”部分。

  3. 解决方案中应出现一个名为存根的新文件夹。

  4. 现在，将解决方案导入另一个模块（解决方案，语言，生成器）后，这些类将在该模块的模型中可用

  5. 想要使用运行时解决方案的语言将需要在其模块属性的“ 运行时”选项卡的“ 运行时解决方案”部分中引用它。

语言

语言代表一种语言定义，由几种模型组成，每种模型都代表语言的不同方面。语言还包含一个生成器模块。语言的属性对话框在许多方面类似于解决方案之一 。下面我们仅提及差异：

共同

语言通常具有指向目录的单个模型根，目录中包含各个方面的所有模型。

依存关系

语言的依存关系是其他解决方案和语言，从该解决方案中可以看到其模型。然后， Export标志指定是否应将依赖项作为依赖项可传递地添加到所有依赖当前语言的模块。

对语言的依赖性为您提供了范围选项：

  • 默认值 -仅使其他语言/解决方案的模型可用于参考（汇总）
    结构方面通过整合（即在子角色中使用外来概念）体现了它们所合并的语言，因此无需将此类语言显式导入到使用聚合语言的模型中或“扩展”语言模块。

  • 扩展 -允许语言定义概念以从那里的语言扩展概念

  • 生成目标 -指定将当前语言生成为另一种语言，从而产生生成器排序约束，即必须仅在当前语言完成生成后生成另一种语言

使用的语言

这与解决方案相同。

运行

  • 运行时解决方案 -列出了该语言所需的可重用代码的解决方案。有关如何创建这样的解决方案的详细信息，请参见上面的“ 向项目中添加外部Java类和jar-运行时解决方案 ”部分。

  • 附件模型 -列出语言所需的附件模型。这些附件模型中包含的节点在使用该语言的Java类路径和任何模型的依赖项上都是隐式可用的。

爪哇

除了缺少两个不适用于语言的选项之外，这与解决方案相同。

刻面

这与解决方案相同。

在使用一种语言的运行时解决方案时，需要在“ 依赖关系”选项卡和“ 运行 时”选项卡的“ 运行时解决方案”中同时设置依赖关系 。

语言模型/方面

依赖项/使用的语言/高级

这些设置与任何其他型号的设置相同，并且具有相同的含义，如“ 解决方案”部分所述。

发电机

生成器模块设置与其他模块类型的设置非常相似：

共同

这与语言相同。

依存关系

这与解决方案相同。另外，生成器模块可能依赖于其他生成器模块并指定范围 ：

  • 默认值 -仅使其他语言/解决方案的模型可供参考

  • 扩展 -电流发生器将能够扩展扩展发电机的发电机元件

  • 设计 -仅需要从该生成器的优先级规则中引用目标生成器

使用的语言

这与语言相同。

发电机优先

此选项卡允许定义生成器的优先级规则 ，以便在生成过程中正确排序生成器。此外，可以通过对话框底部的复选框配置三个选项：

  • 生成模板 -指示是否应将生成器模板生成并编译到Java中，或者在生成过程中由生成器来解释它们

  • 反射查询 -指示是否通过Java反射调用生成的查询。（有关详细信息，请查看生成器文档）

  • IOperationContext参数 -指示生成器是否使用传递到查询中的operationContext参数。以后将删除该参数，并且发电机应逐渐停止使用它。

爪哇

这与语言相同。

刻面

这与语言相同。

发电机型号

这与解决方案相同。
