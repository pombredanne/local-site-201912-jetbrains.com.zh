运行/调试配置：JUnit测试

JUnit运行/调试配置定义了应如何运行基于JUnit测试框架的单元测试。

这种运行/调试配置允许将存储在MPS模型中的JUnit测试与JUnit一起运行。

项目

描述

测试范围

单击一个单选按钮以选择要运行的测试的粒度-整个项目，特定模块，模型，类或方法。

项目/模块/模型/
类/方法

指定保存测试运行的MPS项目/模块/模型/类/方法

在同一过程中执行

选中此复选框以指示是否应在同一MPS实例中运行测试。
选中此复选框可在后台启动新的不可见（无头）MPS流程以运行测试。
在同一过程中运行测试通常要快得多。

重用缓存

指示是否应重用或删除在先前运行期间缓存的生成的工件。在运行测试时重用无头MPS实例的旧缓存可以节省大量时间来设置MPS的测试实例。可以在运行配置对话框中设置和取消设置此选项。

将缓存保存在

指定用于保存缓存的目录。默认情况下，MPS选择临时目录。因此，选中“ 重用缓存 ”复选框后，MPS会将其缓存保存在指定的文件夹中，并尽可能重用它们。如果未选中此复选框，则每次运行都会清除目录。

虚拟机参数

向JVM提供命令行参数

程序参数

提供该类的命令行参数

工作目录

用显式目录覆盖默认工作目录

使用替代的JRE

明确指定用于运行类的JRE发行版

共同

编辑运行配置（而不是运行配置模板）时，可以指定以下选项：

项目

描述

名称

在此字段中，指定运行/调试配置的名称。该名称将帮助您在以后选择编辑它或从“ 运行”弹出窗口中调用它时识别所创建的配置。 Shift+Alt+F10 。

通过VCS共享

选中此复选框可将运行/调试配置置于版本控制下，并使其可用于其他团队成员。

共享的运行/调试配置保存在.mps \ runConfigurations文件夹下的单独xml文件中，而本地运行/调试配置保存在.mps \ workspace.xml中 。

允许并行运行

如果选中此复选框，则可以并行启动运行配置多次，而不必重新运行它。每个运行程序都将在“ 运行工具窗口”或“ 调试工具窗口 ”的自己的选项卡中启动 。

禁用后，每次启动运行/调试配置的新实例时，MPS都会检查同一运行/调试配置的其他实例是否存在，并显示确认对话框。如果在确认对话框中单击“ 确定” ，则将停止运行器的第一个实例，而将其替换。当启动两个相同类型的运行/调试配置实例时，这会占用过多的CPU和内存资源。

工具列

运行/调试配置的树形视图具有一个工具栏，可帮助您管理项目中可用的配置以及调整默认配置模板。

显示此页面

在实际开始运行/调试配置之前，选中此复选框以显示运行/调试配置设置。

激活工具窗口

默认情况下，此复选框处于选中状态，并且在启动运行/调试配置时会打开“ 运行”或“ 调试”工具窗口。

否则，如果清除该复选框，则不会显示工具窗口。但是，在运行配置时，您可以通过按自己打开相应的工具窗口Alt+4要么Alt+5 。
