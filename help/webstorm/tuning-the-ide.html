WebStorm 2019.3帮助

进阶设定

除了可用的标准选项在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），WebStorm使您能够执行基础平台和Java运行时的低级配置。

如果不确定您要做什么，这可能会导致意外问题，并使WebStorm安装无法运行。强烈建议您与JetBrains支持联系，以获取有关可帮助您解决任何问题的选项和值的说明。

配置JVM选项

WebStorm在Java虚拟机（JVM）上运行，该Java虚拟机具有控制其性能的各种选项。在以下文件中指定了用于运行WebStorm的默认选项：

\ bin \ webstorm64.exe.vmoptions （用于默认的64位JVM）

\ bin \ webstorm.exe.vmoptions （用于可选的32位JVM）

WebStorm.app/Contents/bin/webstorm.vmoptions

/bin/webstorm64.vmoptions （用于默认的64位JVM）

/bin/webstorm.vmoptions （用于可选的32位JVM）

不要更改默认文件中的JVM选项，因为在更新WebStorm时将替换它。此外，在macOS情况下，编辑此文件会违反应用程序签名。

要配置JVM选项：

  • 在“ 帮助”菜单上，单击“ 编辑自定义VM选项” 。

  • 在“欢迎”屏幕上，单击“ 配置” ，然后单击“ 编辑自定义VM选项” 。

  • 如果无法启动WebStorm，则可以将带有JVM选项的默认文件手动复制到WebStorm配置目录。

WebStorm在配置目录中使用JVM选项创建文件的副本，并在新的编辑器选项卡中将其打开。您在此文件中更改的任何值都将覆盖原始默认文件中的值。

如果您没有对WebStorm配置目录的写访问权，则可以添加WEBIDE_VM_OPTIONS环境变量，以使用JVM选项指定文件的位置。该文件中的值将覆盖原始默认文件和位于WebStorm配置目录中的副本中的相应值。

如果使用的是Toolbox App，它将管理安装和配置目录，并允许您为每个IDE实例配置JVM选项。打开工具箱应用程序，单击所需实例的螺母图标，然后选择设置 。

常用选项

在大多数情况下，JVM选项的默认值应该是最佳的。以下是最常修改的内容：

选项

描述

-Xmx

限制JVM可以为运行WebStorm分配的最大内存堆大小。默认值取决于平台。如果遇到速度下降的情况，可能需要增加此值，例如，要将其设置为2048 MB，请将此选项更改为-Xmx2048m 。

-Xms

指定JVM分配的用于运行WebStorm的初始内存。默认值取决于平台。通常，它设置为最大允许内存（ -Xmx ）的一半，例如， -Xms1024m 。

-XX:NewRatio

指定堆的年轻代和老代大小之间的比率。在大多数情况下，建议使用2到4之间的比率。这样会将年轻一代的大小相应地设置为老一代的1/2到1/4，这在您经常处理一个项目并且一次只处理几个文件时非常有用。但是，如果您不断打开新文件并在多个项目之间切换，则可能需要增加年轻一代。在这种情况下，请尝试设置-XX:NewRatio=1 ，这会使年轻一代的年龄与老一代一样大，从而使对象在年轻一代中的保留时间更长。

有关可用的JVM选项的更多信息，请参见java Windows或macOS / Linux的参考 。

配置平台属性

WebStorm使您可以自定义各种特定于平台的属性，例如用户安装的插件的路径和支持的最大文件大小。在以下文件中指定了用于运行WebStorm的默认属性：

\ bin \ idea.properties

WebStorm.app/Contents/bin/idea.properties

/bin/idea.properties

不要更改默认文件中的平台属性，因为在更新WebStorm时会替换它。此外，在macOS情况下，编辑此文件会违反应用程序签名。

要配置平台属性：

  • 从“ 帮助”菜单中，选择“ 编辑自定义属性” 。

  • 在“欢迎”屏幕上，单击“ 配置” ，然后选择“ 编辑自定义属性” 。

  • 如果无法启动WebStorm，则可以在WebStorm配置目录中手动创建一个空的idea.properties文件。

WebStorm在配置目录中创建一个空的idea.properties文件，并在新的编辑器选项卡中将其打开。您添加到此文件的任何属性都将覆盖原始默认文件中的相应属性。

如果您没有对WebStorm配置目录的写访问权，则可以添加WEBIDE_PROPERTIES环境变量以指定idea.properties文件的位置。该文件中的属性将覆盖原始默认文件和WebStorm配置目录中的属性。

共同特性

用户解决特定问题通常会更改以下属性：

  • 可能需要移动默认IDE目录的位置，例如，如果用户配置文件驱动器空间不足或位于慢速磁盘上，或者主目录已加密（使IDE变慢）或位于网络驱动器上，如果要创建可移植的安装或从主目录备份中排除高速缓存等。

    您可以使用具有任何属性名称的宏来插入其值，例如use ${user.home} （标准Java系统属性）以指定相对于用户主目录的路径。

    属性

    通往

    idea.config.path配置目录
    idea.system.path系统目录
    idea.plugins.path插件目录
    idea.log.path日志目录

  • 可能影响性能的限制：

    属性

    描述

    idea.max.content.load.filesize

    WebStorm可以打开的最大文件大小（以千字节为单位）。使用大文件可能会影响编辑器性能并增加内存消耗。默认值为20000 。

    idea.max.intellisense.filesize

    WebStorm为其提供编码帮助的最大文件大小（以千字节为单位）。大文件的编码辅助可能会影响编辑器性能并增加内存消耗。默认值为2500 。

    idea.cycle.buffer

    控制台循环缓冲区的最大大小（以千字节为单位）。如果控制台输出大小超过此值，则会删除最早的行。要禁用循环缓冲区，请设置idea.cycle.buffer.size=disabled 。

    idea.max.vcs.loaded.size.kb

    比较更改时，WebStorm加载以显示过去文件内容的最大大小（以千字节为单位）。默认值为20480 。

WebStorm提供了许多其他属性，这些属性定义了与环境的交互（窗口管理器，启动器，文件系统等）。它们中的大多数都类似于隐藏设置（在某种意义上说它们没有明显暴露），在某些情况下可能需要启用或禁用它们。强烈建议仅在联系JetBrains支持寻求帮助后才更改这些属性。

默认的IDE目录

默认情况下，WebStorm将用户特定的文件（配置，缓存，插件，日志等）存储在用户的主目录中。但是，如有必要，您可以更改存储这些文件的位置。

配置目录

WebStorm配置目录包含带有个人设置的XML文件，例如按键图，配色方案等。它也是用户定义的VM选项和平台属性文件的默认位置。

句法
％HOMEPATH％\。 \ config
例
C：\ Users \ JohnS \。WebStorm2019.3 \ config
句法
〜/图书馆/首选项/
例
〜/图书馆/首选项/WebStorm2019.3
句法
〜/。 /配置
例
〜/。WebStorm2019.3 /配置

您可以使用idea.config.path属性更改WebStorm配置目录的位置。

通过将文件从配置目录复制到另一个WebStorm安装上的相应文件夹，可以共享您的个人IDE设置。确保WebStorm不在运行，以避免在关闭IDE时擦除复制的文件。下表列出了WebStorm配置目录中的子文件夹以及其中包含的相应设置。

如果使用设置存储库插件同步设置 ，则这些子文件夹位于配置目录中的settingsRepository下。

如果您的设置是通过IDE Settings Sync插件同步的 ，则这些子文件夹位于配置目录中的jba_config下。
