WebStorm 2019.3帮助

JSTestDriver

WebStorm支持与JSTestDriver测试框架的集成。

在你开始前

  • 如管理插件中所述，在“ 插件”页面上安装并启用JSTestDriver存储库插件。

    通过该插件，WebStorm提供了JSTestDriver服务器和断言框架。在测试创建期间，插件会检测到单元测试框架并建议“ 添加支持 意图”操作 。

在项目中配置测试框架

  1. 下载您选择的框架并将其配置为WebStorm JavaScript库 。

  2. 请执行以下任一操作：

    • 将项目文件夹添加到库范围 。

    • 使用添加支持意图操作，在测试创建过程中即时启用框架支持。

    • 要使用茉莉花，添加jasmine-jstd-adapter到配置文件。

      打开jsTestDriver.conf并在其中键入以下代码：

      load: lib/jasmine/jasmine.js lib/jasmine-jstd-adapter/JasmineAdapter.js

手动创建测试运行器配置文件

测试运行程序配置文件定义测试和生产文件以加载加载顺序。WebStorm会将扩展名为.jstd或.conf的任何文件视为测试运行程序配置文件。

创建配置文件

  1. 在项目树中，选择生产和测试文件夹的父文件夹，然后选择“ 新建” |“测试”。上下文菜单中的文件 。

  2. 在打开的“ 新文件”对话框中，输入扩展名为.jstd或.conf的配置文件的名称。

  3. 在编辑器中打开新文件，并指定当前文件夹的完整路径以及要相对于其加载的文件的路径。在文件名模式中使用通配符。

  4. 使用YAML完成配置文件，请参阅测试运行器配置文件的描述 。

运行测试

使用WebStorm，您可以直接从编辑器快速运行单个JSTestDriver测试，也可以创建运行/调试配置来执行部分或全部测试。

从编辑器运行单个测试

  • 请点击要么在左侧装订线中，然后选择“运行” 从列表中。

  • 借助测试状态图标，您还可以在编辑器中查看测试是通过还是失败在排水沟中。

仅当测试中使用的测试框架与项目相关联时，箭头才会出现，因此WebStorm可以识别测试。

创建JSTestDriver运行配置

  1. 打开“ 运行/调试配置”对话框（主菜单上的“运行” |“编辑配置 ”），单击在左侧窗格中，然后从列表中选择JSTestDriver 。将打开“ 运行/调试配置：JSTestDriver”对话框。

    或者， 在“ 项目”工具窗口中选择一个测试文件，然后选择“ 创建” 从上下文菜单中。

  2. 指定要运行的测试，配置文件的路径以及在测试执行之前要执行的活动。

启动WebStorm默认的JSTestDriver测试服务器

  1. 确保项目中至少有一个配置文件。

  2. 打开JSTestDriver Server工具窗口（“ 视图” |“工具”窗口| JSTestDriver Server ），然后单击在工具栏上。

要使用在另一台计算机上运行或监听自定义端口的测试服务器，请根据服务器特定的说明启动它，并在“ 运行/调试配置：JSTestDriver”对话框的“ 服务器”区域中指定其URL地址。

完成单元测试后停止服务器

  • 请点击在工具栏上。

捕获浏览器

  1. 如果JSTestDriver Server尚未运行，请启动它，然后切换到JSTestDriver Server工具窗口。

  2. 要使用JSTestDriver的远程控制台启动本地浏览器，请执行以下任一操作：

    • 单击表示您选择的浏览器的图标。

    • 如果已经打开浏览器，则使用URL只读字段复制“ 捕获浏览器”的内容， 并将URL粘贴到地址栏中。

    在任何一种情况下，指示所选浏览器的图标将变为活动状态。

  3. 切换到JSTestDriver服务器工具窗口，然后单击表示您刚刚打开的浏览器的图标。WebStorm将显示一条消息，通知您已准备好执行测试。

通过运行配置运行测试

  1. 从主工具栏上的列表中选择JSTestDriver运行/调试配置，然后单击在列表的右边。

  2. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

导航

使用WebStorm，可以在文件和相关测试文件之间跳转，也可以从“ 测试运行器”选项卡中的测试结果跳转到测试。

  • 在编辑器中打开文件，然后选择“ 转到|测试或转到|从上下文菜单中测试受试者 ，或者只需按Ctrl+Shift+T 。

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源代码 ”。测试文件将在编辑器中打开，并将光标置于测试定义上。

调试测试

JSTestDriver测试的调试会话仅通过运行/调试配置启动。

开始调试测试

  1. 创建JSTestDriver运行/调试配置如上所述 。

  2. 从主工具栏上的列表中选择JSTestDriver运行/调试配置，然后单击在列表的右边。

  3. 在打开的“ 调试工具”窗口中，照常进行： 逐步执行测试 ， 停止并继续执行测试，在挂起时检查测试 ，依此类推。

监控代码覆盖率

借助WebStorm， 您还可以监视Karma测试涵盖了多少代码。WebStorm在专用工具窗口中显示此统计信息，并在编辑器中直观地标记覆盖和未覆盖的线。

启动覆盖测试

  1. 创建JSTestDriver运行/调试配置如上所述 。

  2. 启动JSTestDriver服务器，并捕获浏览器以运行测试。

  3. 从主工具栏上的列表中选择JSTestDriver运行/调试配置，然后单击在列表的右边。

  4. 在“ 覆盖率”工具窗口中监视代码覆盖率。

从覆盖率分析中排除文件

  1. 创建JSTestDriver运行/调试配置如上所述 。

  2. 在“ 覆盖率”选项卡中，指定要排除的文件的路径。
