创建新项目对话框

档案|新项目

欢迎屏幕|建立新专案

使用此对话框可以创建空项目或生成特定于框架的项目存根以开发应用程序。

对话框的左侧窗格显示了可以为其生成存根的项目类型的列表。可用的项目类型集取决于已安装和已激活的插件。右侧窗格的内容取决于所选项目类型。位置字段对于所有项目类型都是通用的。

项目类型

描述

空项目

选择此选项仅获取不带任何内容的项目文件夹。在位置字段中指定项目文件夹的路径，然后单击创建 。

HTML5样板

选择此选项可设置项目结构并根据HTML5 Boilerplate模板生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 从版本列表中，选择要使用的模板版本，然后单击创建 。

Web入门套件

选择此选项可以创建一个项目，设置其结构，并根据Web Starter Kit的要求生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 从版本列表中，选择要使用的模板版本，然后单击创建 。

React App

选择此选项可使用全局安装的create-react-app软件包生成项目，有关详细信息，请参见React 。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 在“ 节点解释器”字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择添加以配置新的解释器。

    如果您的计算机上没有Node.js，请从列表中选择Download Node.js ，WebStorm会为您下载它。

  3. 从create-react-app列表中，选择npx create-react-app 。

    另外，对于npm 5.1及更早版本，请安装create-react-app通过运行打包自己npm install --g create-react-app在航站楼 Alt+F12 。创建应用程序时，选择create-react-app包已存储。

  4. （可选）在“ 脚本版本”字段中， 指定在项目生成期间要使用的自定义包，而不是react-scripts 。这可以是从react-scripts分叉的软件包之一，例如， react-awesome-scripts ， custom-react-scripts ， react-scripts-ts等。

引导程序

选择此选项可创建一个项目，设置其结构，并根据Bootstrap模板生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 从版本列表中，选择要使用的模板版本，然后单击创建 。

基础

选择此选项可设置项目结构，并根据Foundation框架模板生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 从版本列表中，选择要使用的模板版本，然后单击创建 。

AngularJS

选择此选项可设置项目结构并根据AngularJS框架模板生成一些源。

在右侧窗格中， 指定将存储与项目相关的文件的文件夹的路径。

角度CLI

选择此选项可获取一个存根，稍后您可以使用Angular CLI命令行界面自动生成特定的结构，例如Classs ， Components ， Routes ， Pipes ， Services等。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 在“ 节点解释器”字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择添加以配置新的解释器。

    如果您的计算机上没有Node.js，请从列表中选择Download Node.js ，WebStorm会为您下载它。

  3. 从Angular CLI列表中，选择npx --package @ angular / cli ng 。

    另外，对于npm 5.1及更早版本，请安装@angular/cli通过运行打包自己npm install -g @angular/cli在航站楼 Alt+F12 。创建应用程序时，选择@angular/cli包已存储。

反应本机

选择此选项可获取用于开发React Native应用程序的存根。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 在“ 节点解释器”字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择添加以配置新的解释器。

    如果您的计算机上没有Node.js，请从列表中选择Download Node.js ，WebStorm会为您下载它。

  3. 从React Native列表中选择npx --package react-native-cli react-native 。

    另外，对于npm 5.1及更早版本，请安装react-native-cli通过运行打包自己npm install -g react-native-cli在航站楼 Alt+F12 。创建应用程序时，选择react-native-cli包已存储。

Node.js

选择此选项以启动一个简单的Node.js应用程序。WebStorm只需运行npm init该命令添加package.json文件并启用Node.js核心API的代码完成。

在右侧窗格中， 指定项目文件夹，Node.js解释器，程序包管理器（ npm或Yarn ，有关详细信息，请参见npm和Yarn ）。

Node.js Express应用

选择此选项可设置项目结构并根据Express框架生成一些项目源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. Node解释器，请参阅配置本地Node.js解释器 。

  3. 要使用-npm或Yarn的程序包管理器，请参见npm和Yarn 。

  4. 要使用的Express模板引擎 。从模板引擎列表中，选择以下选项之一：
    • 灰尘
    • EJS
    • 车把
    • Hogan.js

    • 帕格 （ 帕格（玉） ）

    • 枝条
    • 瓦什

    • 静态HTML

  5. 要使用的CSS引擎。从CSS引擎列表中，选择以下选项之一：
    • 纯CSS
    • 触控笔
    • 减

    • 指南针 。

    • 萨斯

科尔多瓦应用

选择此选项可设置项目结构，并根据PhoneGap ， Apache Cordova和Ionic框架生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 指定可执行文件phonegap.cmd或cordova.cmd或ionic.cmd的位置 （请参阅安装PhoneGap / Cordova / Ionic ）。

约曼

选择此选项可获取使用Yeoman工具生成特定于框架的项目存根的界面。

右侧窗格显示了以前安装的所有Yeoman生成器 。从列表中选择所需的生成器，单击“ 下一步” ，然后按照Yeoman中所述进行操作。

流星应用

选择此选项可设置项目结构并基于Meteor框架生成一些源。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 指定Meteor可执行文件的位置（请参阅安装Meteor ）。

  3. 在模板列表中， 选择要生成的样本。要生成基本的项目结构，请选择“ 默认”选项。

  4. 在文件名字段中， 输入将相互生成的.js ， .html和.css文件的名称。仅当从“ 模板”下拉列表中选择“ 默认样品类型”时，此字段才可用。

Vue.js

选择此选项以使用Vue CLI创建Vue.js应用程序。

在右侧窗格中：

  1. 指定将存储与项目相关的文件的文件夹的路径。

  2. 在“ 节点解释器”字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择添加以配置新的解释器。

    如果您的计算机上没有Node.js，请从列表中选择Download Node.js ，WebStorm会为您下载它。

  3. 从Vue CLI列表中，选择npx --package @ vue / cli vue 。

    另外，对于npm 5.1及更早版本，请安装@vue/cli通过运行打包自己npm install --g @vue/cli在航站楼 Alt+F12 。创建应用程序时，选择@vue/cli包已存储。

  4. 要使用babel和ESLint引导您的应用程序，请选中使用默认项目设置复选框。

镖

选择此选项可设置项目结构并为Dart应用程序生成一些源。
