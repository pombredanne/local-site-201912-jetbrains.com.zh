使用运行/调试配置

要在WebStorm中运行或调试代码，可以使用多种运行/调试配置。每个运行/调试配置代表一组命名的运行/调试启动属性。使用WebStorm执行运行，调试或测试操作时，始终使用其参数根据现有配置之一启动进程。

WebStorm带有许多用于各种运行，调试和测试问题的运行/调试配置类型。您可以创建自己的特定类型的运行/调试配置。

每种运行/调试配置类型都有其自己的默认设置。每当创建相应类型的新运行/调试配置时，它都会基于这些默认设置。

临时运行/调试配置

每次选择运行时，都会自动创建一个临时运行/调试配置或调试没有永久配置的商品。

将临时配置另存为永久配置

  • 在“ 运行/调试配置”对话框中选择临时配置，然后单击在工具栏上。

  • 或者，在运行/调试配置选择器中选择临时配置，然后单击保存配置 。

临时配置用半透明图标标记，并且与永久配置的管理方式相同。
默认情况下，每个项目允许五个临时运行/调试配置，因此当您创建第六个配置时，将删除第一个创建的配置，依此类推。要更改此限制，请使用“ 运行/调试配置”对话框的“ 模板”页面中的“ 临时配置限制”字段。

永久运行/调试配置

为特定的类或方法显式创建一个永久的运行/调试配置。如果没有项目的永久配置，则在选择“运行”时，WebStorm会自动为其创建临时配置。 或调试从此类或方法的上下文菜单中。

运行/调试配置模板

具有默认运行/调试配置设置的模板在“ 模板”节点下的“ 运行/调试配置”对话框中列出。它们表示创建新的运行/调试配置时使用的设置。

您可以为特定配置类型设置默认设置，该设置将适用于以后创建的该类型的任何运行/调试配置。更改默认值不会影响现有的运行/调试配置。

在“运行/调试配置的更改模板”中介绍了按类型缺省配置设置的编辑过程。创建和编辑运行/调试配置中描述了创建或编辑自定义运行/调试配置的过程。
