WebStorm 2019.3帮助

笑话

Jest是专门针对客户端JavaScript应用程序和React应用程序的测试平台。从Jest官方网站了解有关该平台的更多信息。

您可以直接在WebStorm中使用Jest运行和调试测试。您可以在树形视图中查看测试结果，并从那里轻松导航到测试源。测试状态显示在编辑器中测试的旁边，并带有可以快速运行或调试的选项。

在开始之前，请确保您的计算机上装有Node.js。

安装笑话

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install --save-dev jest

    从Jest官方网站上的入门中了解更多信息 。

您还可以按照npm和Yarn中的描述，在Node.js和NPM页面上安装jest软件包。

运行测试

使用WebStorm，您可以直接在编辑器中快速运行单个Jest测试，也可以创建运行/调试配置来执行部分或全部测试。

从编辑器运行单个测试

  • 请点击要么在装订线中，然后选择“运行” 从列表中。

    借助测试状态图标，您还可以在编辑器中直接查看测试是否通过或失败在排水沟中。

创建Jest运行配置

  1. 打开“ 运行/调试配置”对话框（主菜单上的“运行” |“编辑配置 ”），单击在左侧窗格中，然后从列表中选择Jest 。将打开“ 运行/调试配置：Jest”对话框。

    或者， 在“ 项目”工具窗口中选择一个测试文件，然后选择“ 创建” 从上下文菜单中。

  2. 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

  3. 指定位置 jest ， react-scripts ， react-script-ts ， react-super-scripts或react-awesome-scripts软件包。

  4. 指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  5. （可选）指定要使用的jest.config文件：从列表中选择相关文件，或单击并在打开的对话框中选择它，或仅在字段中键入路径。如果该字段为空，则WebStorm将查找带有以下内容的package.json文件： jest键。搜索是在文件系统中从工作目录向上进行的。如果找不到合适的package.json文件，那么将即时生成Jest默认配置 。

  6. （可选）配置对相关源文件中的更改自动重新运行测试。为此，请添加--watch 笑话选项字段中的标记。

通过运行配置运行测试

  1. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右边。

  2. 测试服务器将自动启动，而无需您采取任何步骤。在“ 运行工具”窗口中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

重新运行失败的测试

  • 在“ 测试运行器”选项卡中 ，单击“ 在工具栏上。WebStorm将执行在上一个会话期间失败的所有测试。

  • 要重新运行特定的失败测试，请选择“运行”。 在其上下文菜单上。

有关详细信息，请参见重新运行测试 。

导航

使用WebStorm，可以在文件和相关测试文件之间跳转，也可以从“ 测试运行器”选项卡中的测试结果跳转到测试。

测试文件应遵循流行的命名约定，例如，具有.test。 ， .spec。或_spec。后缀，并且应位于源文件旁边或测试文件夹中。

  • 在编辑器中打开文件，然后选择“ 转到|测试或转到|从上下文菜单中测试受试者 ，或者只需按Ctrl+Shift+T 。

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源代码 ”。测试文件将在编辑器中打开，并将光标置于测试定义上。

  • 对于失败的测试，WebStorm从堆栈跟踪将您带到测试的失败行。如果确切的行不在堆栈跟踪中，则将转到测试定义。

快照测试

WebStorm与Jest的集成支持快照测试等强大功能。如果快照与渲染的应用程序不匹配，则测试将失败。这表明您代码中的某些更改导致了此不匹配，或者快照已过时并且需要更新。您可以在WebStorm内置的Difference Viewer中查看导致此不匹配的原因。

为失败的测试更新快照

  • 使用“ 运行工具”窗口的“ 测试运行器”选项卡中的“ 单击更新快照”链接：

从测试文件导航到相关的快照文件

  • 请点击在测试旁边的排水沟中。

  • 光标在toMatchSnapshot() ， 按Ctrl+Alt+Home然后从列表中选择所需的快照。

将实际输出与预期快照进行比较

  • 使用“ 测试运行器”选项卡右侧窗格中的“ 单击以查看差异”链接。

调试测试

使用WebStorm，您可以直接从编辑器快速开始调试单个Jest测试，也可以创建运行/调试配置来调试部分或全部测试。

从编辑器开始调试单个测试

  • 请点击要么在装订线中，然后选择“ 调试” 从列表中。

通过运行/调试配置启动测试调试

  1. 如上所述创建Jest运行/调试配置。

  2. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右边。

  3. 在打开的“ 调试工具”窗口中，照常进行： 逐步执行测试 ， 停止并继续执行测试，在挂起时检查测试 ，依此类推。

监控代码覆盖率

借助WebStorm， 您还可以监视Jest测试覆盖了多少代码。WebStorm在专用工具窗口中显示此统计信息，并在编辑器中直观地标记覆盖和未覆盖的线。

运行覆盖测试

  1. 如上所述创建Jest运行/调试配置。

  2. 从主工具栏上的列表中选择Jest运行/调试配置，然后单击在列表的右边。
    或者，在编辑器的覆盖范围内快速运行特定套件或测试：单击要么在装订线中，从列表中选择“ 以覆盖率运行” 。

  3. 在“ 覆盖率”工具窗口中监视代码覆盖率。该报告显示测试覆盖了多少文件以及其中覆盖行的百分比。从报告中，您可以跳至文件，查看覆盖了哪些行-标记为绿色-以及未覆盖哪些行-标记为红色：
