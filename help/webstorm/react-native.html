反应本机

使用React Native，您可以使用JavaScript和React开发适用于iOS和Android的本地移动应用程序。它是由Facebook创建的，用于Instagram，Airbnb等著名应用，现在也用于JetBrains自己的YouTrack移动应用。从React Native官方网站上了解更多信息。

WebStorm可帮助您创建，编辑，运行，调试，维护React Native应用程序。WebStorm还为React和Flow符号提供代码完成。

在开始之前，请确保您的计算机上装有Node.js。

创建一个新的React Native应用程序

在WebStorm中创建React Native应用程序的推荐方法是使用专用的项目生成器，例如React Native CLI 。

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择React Native 。

  3. 在右侧窗格中：

    1. 指定将存储与项目相关的文件的文件夹的路径。

    2. 在“ 节点解释器”字段中，指定要使用的Node.js解释器。从列表中选择已配置的解释器，或选择添加以配置新的解释器。

    3. 从React Native列表中选择npx --package react-native-cli react-native 。

      另外，对于npm 5.1及更早版本，请安装react-native-cli通过运行打包自己npm install -g react-native-cli在航站楼 Alt+F12 。创建应用程序时，选择react-native-cli包已存储。

  4. 当您单击Create时 ，WebStorm会使用所有必需的配置文件生成特定于React Native的项目，下载依赖项，并使用默认设置创建React Native类型的运行/调试配置。

  5. 安装其他工具以开始使用React Native，例如iOS模拟器 。这些工具的列表取决于您的操作系统和要将应用程序定位到的移动平台。有关详细的安装说明，请参见《 React Native入门指南》 。

运行和调试React Native应用程序

您可以在物理设备或模拟器上运行和调试应用程序。在启动应用程序之前，请确保模拟器已在运行，或者，如果您使用的是物理设备，则该模拟器已连接到计算机。

WebStorm使运行和调试React Native应用程序变得非常灵活。例如，如果您是第一次启动应用程序，则可以选择运行React Native捆绑器，构建应用程序并在模拟器上打开它-所有这些都是运行或调试会话的一部分。如果捆绑程序已经在运行，则也可以跳过启动它；如果自上次运行以来未对其本机代码进行任何更改，则可以拒绝构建该应用程序。

创建一个React Native运行/调试配置

  1. 在主菜单上，选择“运行”。编辑配置 ，单击然后从列表中选择React Native 。将打开“ 运行/调试配置：React Native” 。

  2. 选择是否要WebStorm为您构建并启动该应用程序：

    • 如果是第一次启动应用程序，或者自上次运行以来已更新其本机代码，请选中“ 构建并启动”复选框。

    • 如果自上次构建以来未对应用程序的本机代码进行任何更改，请清除此复选框。当您开始调试时，WebStorm会按照React Native官方网站上的说明，在启用了远程调试的情况下，在模拟器中打开您的应用程序。

    • 如果您的应用程序使用Expo ，请清除该复选框，因为此捆绑程序将自行处理。

  3. 如果选择了“ 构建并启动”复选框，请选择目标平台，Android或iOS。根据您的选择，WebStorm将使用以下命令运行捆绑程序react-native run-ios或搭配react-native run-android 。

    （可选） 输入要传递给React Native的参数，例如，通过‑‑simulator旗： ‑‑simulator="iPhone 4s" 。

  4. 在Bundler主机字段中，指定运行React Native捆绑程序的主机，默认值为localhost 。

    如果使用的是Expo，则可能需要更改默认的捆绑程序主机，以确保与调试器的成功连接。此连接可能会失败，因为默认情况下，WebStorm使用localhost启动调试，而Expo希望使用127.0.0.1或外部IP地址，具体取决于在Metro捆绑器配置的“连接”字段中选择的内容。有关详细信息，请参见调试使用Expo的React Native应用程序 。

  5. 在“ Bundler端口”字段中，指定运行React Native捆绑程序的端口，默认情况下选择8081 ，从React Native官方网站了解更多信息。

  6. 选择要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

    指定react-native-cli的路径和应用程序的工作目录 。（可选）键入以下环境变量react-native run-android要么react-native run-ios 。

  7. 默认情况下，WebStorm在调用运行/调试配置时自动启动React Native捆绑程序。如果您已经从WebStorm外部（例如，从命令行）启动了捆绑程序，则可以重新使用它而无需停止并重新启动。在启动前区域中选择捆绑器，然后单击 。

准备Android设备

每次开始使用应用程序时，您都需要准备一个Android设备。

  • 启动Android虚拟设备或通过USB启用调试，然后通过USB 连接到物理Android设备 。从React Native官方网站上了解更多信息。

准备iOS模拟器

您只需安装一次iOS模拟器，然后WebStorm便会自动启动它react-native run-ios 。

  • 在嵌入式终端 （ Alt+F12 ），输入：

    npm install --global ios-sim

调试应用程序

使用WebStorm，您可以调试使用本机React Native捆绑器和使用Expo的应用程序。

在调试React Native应用程序时，WebStorm依赖于Chrome运行时，该运行时由React Native本身使用。您也可以将DevTools与WebStorm一起使用。当您启动调试会话时，WebStorm会启动一个新的Chrome实例并将其附加到该实例。如果您不想看到任何新的Chrome窗口，请使用Chrome无头模式 。

调试使用本机捆绑程序的React Native应用程序

  1. 根据需要在代码中设置断点。

  2. 如上所述创建新的React Native运行/调试配置。如果捆绑程序已经在WebStorm外部运行，请在“启动前”区域中选择它，然后单击 。

  3. 要启动您的应用程序，请从工具栏上的“ 选择运行/调试配置”列表中选择新创建的React Native配置，然后单击在它的旁边。WebStorm打开“ 运行工具”窗口，并在新的React Native选项卡中运行捆绑程序。

  4. 构建完成且应用程序出现在模拟器中后， 打开“应用程序内开发人员菜单”，然后选择“ 远程调试JS” 。从React官方网站了解更多信息。

    内置的WebStorm调试器连接到模拟器。

  5. 当遇到第一个断点时，继续进行调试会话- 逐步执行这些断点 ，在帧之间切换，即时更改值， 检查已暂停的程序 ， 评估表达式并设置监视 。

调试使用Expo的React Native应用程序

  1. 根据需要在代码中设置断点。

  2. 如上所述创建新的React Native运行/调试配置，并如下更改默认设置：

    1. 清除“ 生成并启动”复选框，因为此捆绑程序负责处理流程本身。

    2. 在Bundler主机字段中，更改默认值localhost设置为127.0.0.1或外部IP地址，具体取决于在Metro捆绑器配置的“连接”字段中选择的内容。

    3. 在“ Bundler端口”字段中，将默认的8081设置更改为19001 。

    4. 在启动前区域中，选择默认情况下选中的启动React Native Bundler任务，然后单击 。然后，在Configure React Native对话框中，选择npm script ，然后选择start或另一个启动应用程序的任务。

  3. 选择新创建的配置，然后单击 。

  4. 在手机或模拟器上打开Expo客户端应用程序，选择当前应用程序，并启用远程调试，请从Expo官方网站了解更多信息。

  5. 遇到第一个断点时，请像调试使用本机捆绑程序的应用程序一样进行。

在WebStorm中配置Chrome Headless

MacOS和Linux上的Chrome版本59及更高版本以及Windows上的版本60及更高版本均支持Headless模式。

  1. 请点击 调试用浏览器字段旁边。“ Web浏览器”对话框打开。

  2. 选择Chrome ，点击 ，然后将复制的配置重命名为例如Chrome Headless 。

  3. 选择新的浏览器配置，单击 ，然后输入--headless对于macOS和Linux或--headless --disable-gpu Windows的“ Chrome设置”对话框中。

  4. 在运行/调试配置中，从用于调试的浏览器列表中选择新的Chrome Headless配置。
