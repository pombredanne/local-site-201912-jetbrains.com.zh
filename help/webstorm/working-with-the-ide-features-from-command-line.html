从命令行使用WebStorm功能

使用WebStorm，您可以从命令行打开，格式化，检查，比较和合并文件。

启用命令行启动器

  • 将WebStorm bin文件夹的路径添加到Path环境变量（例如， C：\ Program Files \ JetBrains \ WebStorm \ bin ）。

在macOS和Linux上，创建命令行启动器以将WebStorm与您的Shell集成。确保创建的启动器脚本在外壳程序的搜索路径内，以便可以在系统范围内调用它。

  1. 在主菜单中，选择工具|创建命令行启动器 。

  2. 在打开的“ 创建启动器脚本”对话框中，键入启动器的名称及其路径，或接受建议的默认值webstorm 。

  3. 在WebStorm之外，将启动脚本的路径和名称添加到您的path 。

在Linux上，将WebStorm 作为快照程序包安装会自动创建名为webstorm 。工具|因此，“创建命令行启动器”命令不可用。

启动WebStorm

  • 在命令提示符下，键入命令并按Enter 。 是特定于平台的启动器。

webstorm.exe
网络风暴

对于macOS和Linux，默认名称webstorm可以在创建期间进行自定义，请参阅为macOS和Linux启用命令行启动器 。

打开文件，文件夹和项目

  • 在命令提示符下，键入：

    路径

  • 对于文件，可以选择使用以下格式之一指定放置光标的行：

    [--line ]路径路径：

  • 如果文件或文件夹属于现有项目，则将打开该项目。

  • 如果该文件不属于任何项目，则将自动为此文件创建一个临时项目。

  • 如果该文件夹不属于任何项目，则将在此文件夹内创建一个新项目。

运行代码检查

除了从主菜单或“ 项目工具窗口”的上下文菜单中运行代码检查之外，还可以从命令行启动检查器，而无需实际运行WebStorm。

这样，您可以在开发过程中执行常规代码检查，这对于大型项目尤其重要。检查结果以XML格式存储，您可以随时下载和查看这些结果。

从代码检查中了解更多信息。

从命令行启动代码检查

  1. 退出WebStorm，否则退出inspect失败。

  2. 在命令提示符下，键入：

    检查项目路径检查路径输出路径[-vX] [-d子目录路径]
    特定于平台的启动器在哪里。

查看离线检查的结果

  1. 打开要对其运行检查的项目。

  2. 在主菜单中，选择代码|查看离线检查结果 。

  3. 在打开的“ 选择路径”对话框中，单击存储带有检查结果的XML文件的目录。

  4. 单击确定 。检查结果显示在“ 检查结果工具”窗口的“ 脱机视图”选项卡中。

命令行选项

选项

描述

project-path

包含要检查的项目的目录的完整路径。

inspection-path

要检查项目的检查配置文件的完整路径。配置文件存储为一组.xml文件。概要文件有两种：项目概要文件和IDE概要文件，它们保存在不同的位置。

默认情况下，项目检查配置文件存储在.idea / inspectionProfiles节点下的项目目录中。

IDE配置文件位于以下目录中：

  • Windows： ％HOMEPATH％\。WebStorm2019.3 \ config \ inspection

  • macOS： 〜/ Library / Preferences / WebStorm2019.3 / inspection

  • Linux： 〜/。WebStorm2019.3 / config / inspection

output-path

现有目录的完整路径，该目录将存储带有检查报告的XML文件。

-vX

（可选）输出的详细级别，其中X是0为了安静1 –嘈杂，和2 –额外的噪音。

-d subdirectory-path

（可选）要检查的子目录的完整路径。

比较文件

  • 在命令提示符下，键入：

    diff path1 path2

    特定于平台的启动器在哪里， path1和path2是要比较的文件的完整路径。WebStorm在“ 差异查看器”中打开文件 。

    从比较文件，文件夹和文本源中了解更多信息。

例子

句法
webstorm.exe差异path1 path2
例
webstorm.exe差异C：\ MyProject \ Readme.md C：\ MyProject \ Readme.md.bak
句法
webstorm diff path1 path2
例
webstorm diff〜/ MyProject / Readme.md〜/ MyProject / Readme.md.bak

合并档案

  1. 在命令提示符下，键入：

    合并path1 path2 path3输出

    特定于平台的启动器在哪里， path1和path2是要合并的文件的完整路径， path3是可选的，它指向文件偏离的基本版本，并且output是要存储合并结果的文件的路径。

    如果您跳过path3 ，WebStorm对待output作为基本修订。

  2. 在打开的“ 合并”对话框中，浏览更改，并按照解决冲突中所述逐一接受或放弃。

例子

句法
webstorm.exe合并path1 path2 path3输出
例
webstorm.exe合并C：\ MyProject \ Readme.md.bak C：\ MyProject \ Readme.md C：\ Archive \ Readme.md C：\ NewProject \ Readme.md
句法
webstorm合并path1 path2 path3输出
例
webstorm合并〜/ MyProject / Readme.md.bak〜/ MyProject / Readme.md〜/ Archive / Readme.md〜/ NewProject / Readme.md
