结构搜索和替换

常规的搜索过程没有考虑源代码的语法和语义。即使您使用正则表达式，WebStorm仍会将您的代码视为正则文本。结构搜索和替换 （ SSR ）操作使您可以考虑代码结构来搜索代码中的特定代码模式或语法结构。

WebStorm会根据您创建的搜索模板和您应用的条件来查找并替换源代码的片段。

从结构上使用“搜索目标”

  1. 从主菜单中，选择编辑|查找|搜索结构上开结构搜索对话框。

    在“ 结构搜索”对话框中，可以快速切换到“ 结构替换”对话框。请点击 ，然后选择切换到替换 。

  2. 在“ 结构搜索”对话框中，执行下列操作之一：

    • 从头开始创建自己的模板 。

      在编辑器区域中，输入代码模式（ $variable$代表您的代码），请点击 ，然后选择保存模板以将其保存以备将来使用。

      WebStorm将创建的模板添加到“ 现有模板”对话框中的现有模板列表（“ 用户定义”节点）中。

    • 使用现有模板之一作为原型 。

      请点击 ，选择“ 现有模板”以打开预定义模板的列表，然后选择适当的模板。

      在现有模板列表中，

      单击“ 确定”返回到“ 结构搜索”对话框，以使用所选模板。

      WebStorm立即在编辑器中突出显示找到的代码实例。

  3. “ 结构搜索”对话框显示所选模板以及该模板中使用的过滤器的值。您可以编辑现有过滤器或添加新条件 ，例如，添加一些正则表达式或脚本约束 。将插入号放在代码变量处，然后使用过滤器区域管理过滤器。

  4. 在过滤器区域中，单击添加新条件。例如，如果需要添加正则表达式，请选择Text 。您还可以根据变量添加其他条件 。

  5. 指定以下选项以进一步缩小搜索范围：

    • 文件类型 -使用列表选择哪些文件类型应作为搜索的一部分。在我们的例子中，它是JavaScript 。

    • 范围 -指定搜索位置：在项目，模块，目录或自定义范围内。

    • 搜索目标 -您可以选择完全匹配以匹配确切的模板或其他可用的代码元素（在我们的示例中为）。

  6. 点击查找 。

WebStorm在“ 查找”工具窗口中显示结果。

从结构上查找和替换源代码

  1. 在主菜单中，选择“ 编辑” |“选择” 。查找|结构搜索 ，或编辑|查找|结构上更换 。

  2. 在打开的对话框中，定义搜索模板 。

  3. 如果是结构替换，请指定替换模式，并根据需要定义变量约束。

  4. 指定搜索和替换选项，尤其是要匹配的出现次数以及要分析的文件类型。

  5. 指定执行结构搜索和替换的范围。为此，请单击“ 作用域”列表中的向下箭头，然后选择一个预定义的作用域，或者单击“省略号”按钮，然后在“ 作用域”对话框中配置所需的作用域 。

  6. 点击查找 。检测到的事件显示在“ 查找工具”窗口中 。

    请注意，在进行替换的情况下，您可以在搜索结果中选择所需的匹配项，然后单击“ 预览替换”按钮。相应的事件在源代码中突出显示。

共享搜索模板

您可以通过导出或导入与他人共享搜索模板。

  1. 在“ 结构搜索”对话框（“ 编辑” |“查找” |“结构搜索” ）中， 创建一个新的搜索模板或使用现有的模板 。

  2. 要导出模板，请单击 ，然后选择将模板导出到剪贴板 。WebStorm将模板的XML表示添加到剪贴板（按Ctrl+Shift+V查看剪贴板的内容）。您可以在聊天，电子邮件或论坛中与其他开发人员共享此表示。

    要导入模板，请复制（ Ctrl+C ）从任何地方（电子邮件，聊天或论坛）共享的XML代码，然后在“ 结构搜索”对话框中，单击然后选择从剪贴板导入模板 。WebStorm采用XML代码表示形式，并将其转换为包含变量和范围（如果存在）的模板。
