重构代码

重构意味着在不更改应用程序行为的情况下更新源代码。重构可帮助您保持代码的牢固， 干燥和易于维护。WebStorm提供了各种各样的代码重构 ，可以自动跟踪和纠正受影响的代码引用。

重构源代码：主要步骤

尽管目标和工作流程有所不同，但所有WebStorm重构都需要一些强制性步骤。以下是重构代码时将执行的这些主要步骤的概述。

选择要重构的项目

  • 在“ 项目”工具窗口中，选择要移动，复制或重命名的文件或文件夹。

  • 在编辑器中，选择要重构的表达式或符号。

调用重构

  • 从主菜单或所选内容的上下文菜单中，选择“ 重构” |“选择” 。 。

  • 按专用快捷键（请参阅下面的可用重构表 ）。

  • 打开“ 重构此列表”（选择“ 重构” |“重构此”或按Ctrl+Shift+Alt+T ），然后选择所需的重构。

预览，调整和应用建议的更改

对于某些重构，WebStorm提供了一个选项，可以在应用更改之前预览更改。要预览潜在的更改并进行必要的调整，请在特定于重构的对话框中单击“ 预览 ”。WebStorm在“ 查找”工具窗口中打开一个专用选项卡，并显示将要进行的更改。

如有必要，可以在重构期间排除某些更改。重构后，如果预计会发生任何冲突，则WebStorm会显示一个对话框，其中简要介绍所遇到的问题。

从重构中排除条目

  • 在列表中选择它，然后按Delete 。

为了避免潜在的冲突

  • 通过单击继续按钮忽略冲突。结果，将执行重构，但是，这可能导致错误的结果。

  • 通过单击“ 在视图中显示”按钮预览冲突。WebStorm在“ 查找”工具窗口的“ 冲突”选项卡上显示所有冲突的条目，使您可以导航到有问题的代码行并进行必要的修复。

  • 取消重构并返回到编辑器。

应用更改

  • 对建议的结果感到满意后，点击执行重构 。

复制，重命名和移动文件和文件夹

通过这些重构，您可以复制具有相同名称或新名称的文件和文件夹，将它们存储在相同的父文件夹或新文件夹中，以及将文件和文件夹移动到其他位置。

文件和文件夹的重构在所有类型的应用程序中都可用。

复制文件或文件夹

  1. 在“ 项目”工具窗口中选择要复制的文件或文件夹，然后按F5或选择“ 重构|从上下文菜单或主菜单中复制 。

    或者，在“ 项目”工具窗口中选择要复制的文件或文件夹，然后将其拖到新文件夹中。 Ctrl按键。

  2. 在打开的对话框中，指定文件或文件夹的新名称以及新的父文件夹（如果适用）。

重命名文件或文件夹

  1. 在“ 项目”工具窗口中，选择要重命名的文件或文件夹，然后选择“ 重构” |“确定”。从上下文菜单中重命名或按Shift+F6 。

  2. 在打开的对话框中，键入文件或文件夹的新名称。

  3. 要更新注释，字符串和文本（对于文件）中对文件或文件夹的引用，请选中相应的复选框。

  4. 单击“ 重构”，然后在“ 查找”工具窗口的专用选项卡中检查建议的更改，如上面的“ 预览，调整和应用建议的更改”中所述。对预期结果满意时，请单击“重构” 。

移动文件或文件夹

  1. 在“ 项目”工具窗口中，选择要移动的文件或文件夹，然后选择“ 重构”。从上下文菜单中移动或按F6 。

  2. 在打开的对话框中，指定新的父文件夹。选择一个现有文件夹或键入要创建的新文件夹的名称。

  3. 选中“ 搜索引用”复选框以将引用更新为移动的文件或文件夹。

安全删除文件（安全删除）

与简单删除文件不同（ Delete或从上下文菜单中删除 ），此重构可确保您不删除源代码中引用的文件。WebStorm查找要删除的文件的用法，并在“ 检测到的用法”对话框中显示它们。

将安全删除应用于文件

  1. 在“ 项目”工具窗口中，选择要删除的文件，然后选择“ 重构|分解”。从上下文菜单中安全删除或按Alt+Delete 。

  2. 在打开的“ 安全删除”对话框中，选中相应的复选框，以在注释和字符串中查找文件的用法，并在文本中查找文件的出现。

  3. 如果检测到任何用法，WebStorm会在“ 检测到的用法”对话框中列出它们。

    • 要删除文件而不查看发现的用法，请单击仍然删除 。

    • 要停止重构，请单击取消 。

    • 要检查潜在的冲突，请单击查看用法 。WebStorm将打开“ 查找”工具窗口的专用选项卡，其中列出了用法。按照上面的“ 预览，调整和应用建议的更改”中所述的方法，探索它们，分析潜在的冲突并更新代码。
