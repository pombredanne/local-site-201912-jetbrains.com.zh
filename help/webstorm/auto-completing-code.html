自动完成代码和路径

代码完成是WebStorm编辑器的基本功能之一。为了向您显示完成建议，WebStorm将使用源代码浏览项目文件，并将其中定义的类，方法，函数和变量添加到特殊的内部索引中。来自JSDoc注释， TypeScript类型定义等的其他信息也可以显着提高完成度。

完成也适用于第三方代码中的符号。在大多数情况下，您所需要做的就是将所需文件添加到项目中，有关详细信息，请参阅配置JavaScript库 。

本节介绍了各种上下文感知的代码完成技术，这些技术可帮助您加快编码过程。

基本完成

基本代码完成功能可帮助您完成可见性范围内的类，方法，字段和关键字的名称。当您调用代码完成功能时，WebStorm会分析上下文并建议从当前插入符号位置可以到达的选择（建议还包括Live模板 ）。

非英语键盘布局也可以使用基本完成功能。

调用基本完成

  1. 开始输入名称。默认情况下，WebStorm会在您键入时自动显示代码完成弹出窗口。

    或者，按Ctrl+Space或选择代码|完成|从主菜单基本 。

  2. 要获取更多建议，请按Ctrl+Space第二次（或按Ctrl+Alt+Space ）。

  3. 要缩小建议列表的范围，请键入期望名称或关键字的任何部分（甚至接受中间位置的字符），或在点分隔符后调用代码完成功能。WebStorm显示的建议在任何位置都包含键入的字符。

    对于CamelCase或snake_case名称，WebStorm仅自动识别并匹配首字母。

通过以下方式之一接受列表中的建议

  • 按Enter或双击相关列表项以将其插入插入符号的左侧。

  • 按Tab替换插入符号右侧的字符。

  • 使用Ctrl+Shift+Enter使当前的代码构造在语法上正确（平衡括号，添加缺少的花括号和分号，等等）。

  • 使用特定的语言和上下文相关的键来插入选定的完成建议，有关详细信息，请参阅配置完成设置 。

查看所选建议的参考

  • 为了快速查找文档，请在建议列表中选择一个项目，然后按Ctrl+Q 。WebStorm在“ 文档”弹出窗口中为您显示了快速文档。

    或者，如配置完成设置中所述，启用自动文档查找。

  • 要在带有更多控件的工具窗口中查看文档，请固定“ 文档”弹出窗口。 要在“ 文档”弹出窗口和“ 文档工具”窗口之间切换，请按Ctrl+Q按顺序。

  • 要查看所选建议的定义 ，请按Ctrl+Shift+I 。

查看代码层次结构

  • 在建议列表中选择一个条目，然后按以下快捷方式之一：

    • Ctrl+H查看类型层次结构

    • Ctrl+Alt+H查看视图调用层次结构。

    • Ctrl+Shift+H查看方法层次结构。

    有关详细信息，请参见构建层次结构 。

有关新变量和参数名称的建议

除了完成对现有交易品种的调用之外， WebStorm会在声明过程中为新变量和参数建议名称。这些建议基于项目，使用的库以及标准API中定义的类，类型和接口的名称。

要关闭此功能，请打开“ 设置/首选项”对话框Ctrl+Alt+S ，转到编辑器|一般|代码完成 ，然后清除“ JavaScript”部分中的“ 建议变量和参数名称”复选框。

JavaScript中TypeScript语言服务的完成

您可以使用TypeScript语言服务的建议来增强JavaScript文件中的基本代码完成率。从验证TypeScript中了解更多信息。

  1. 打开jsconfig.json或tsconfig.json文件并添加'allowJS' : true对此。

  2. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到语言和框架| TypeScript 。

  3. 在打开的“ TypeScript”页面上，确保选中“ TypeScript语言服务”复选框。

声明完成

您可以使用语句完成功能来创建语法正确的代码构造Ctrl+Shift+Enter 。它插入必要的语法元素（括号，花括号和分号），并使您处于可以开始键入下一条语句的位置。

  • 要完成方法声明，请开始输入function然后按Ctrl+Shift+Enter 。WebStorm创建方法的整个结构，并将光标置于方法主体内。

  • 要完成代码构造，请开始输入它，例如，键入if ，然后按Ctrl+Shift+Enter 。WebStorm会自动添加所需的标点符号，并将光标置于下一个编辑点。

嬉皮完成

嬉皮补全是一种补全引擎，它可以在可见范围内分析您的文本并根据当前上下文生成建议。它可以帮助您从任何当前打开的文件中补全单词。

将插入符号处的字符串扩展为现有单词

  1. 键入初始字符串，然后执行以下操作之一：

    • 按Alt+/或选择代码|完成|循环展开单词可在插入符号前搜索匹配的单词。

    • 按Shift+Alt+/或选择代码|完成|循环扩展单词（向后）可在插入符号之后和其他打开的文件中搜索匹配的单词。

    出现第一个建议值，并且原型在源代码中突出显示。

  2. 接受建议，或按住Alt键并保持按下/直到找到所需的单词。

后缀代码完成

后缀代码完成功能可帮助您减少编写代码时向后插入符号的跳跃。您可以根据在点后键入的后缀，表达式的类型及其上下文，将已经键入的表达式转换为其他表达式。

WebStorm附带了一组预定义的后缀模板，使您可以为JavaScript和TypeScript定义自己的自定义模板。 可以复制，更新和删除自定义模板。对于预定义的模板，您只能更改其后缀，例如，将长键替换为较短的后缀。

JavaScript模板也可以在TypeScript文件中使用。

  • 要查看可用的后缀模板的完整列表， 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |一般|后缀完成 。

    从JavaScript Postfix Templates了解更多信息。

  • 选择Tab ， Space ， 要么Enter扩展后缀模板。

完成替代

JavaScript和TypeScript中提供了用于覆盖的方法主体的生成。

当您想从父类或接口重写方法并从完成建议列表中选择此方法时，WebStorm会自动添加参数，生成一个super()调用，并在可能的情况下添加类型信息。

关闭替代的方法体的生成

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到语言和框架| JavaScript 。

  2. 在打开的JavaScript页上，清除“ 展开方法主体以完成覆盖”复选框。

标签，属性和属性的完成

WebStorm自动完成许多文件类型的标记和属性的名称和值：

  • HTML / XHTML，包括CSS类和JSX内部HTML标记的 完成 。

  • XML / XSL ，包括名称空间的完成。

  • JSON，有关详细信息，请参见编辑package.json 。

标记和属性名称的完成取决于与文件关联的DTD或Schema 。如果没有架构关联，WebStorm将使用文件内容（标记和属性名称及其值）来完成输入。

  1. 按<并开始输入标签名称。WebStorm显示适用于当前上下文的标记名称列表。使用Up和Down按钮滚动列表。

  2. 按Enter接受列表中的选择。如果您的文件与模式或DTD关联，则WebStorm会根据其自动插入必填属性。

配置代码完成设置

要自定义默认的完成行为， 打开“ 设置/首选项”对话框（ Ctrl+Alt+S ），转到“ 编辑器” |一般|代码完成 ，然后在“ 代码完成”页面上更新默认完成设置。要在JavaScript上下文中调整完成，请使用JavaScript区域中的控件。

键入时自动显示建议列表

  • 选择键入时显示建议复选框。如果取消选中该复选框，则必须通过按显式调用代码完成Ctrl+Space基本完成或Ctrl+Shift+Space智能完成。

    您也可以选择只有一种选择时自动插入建议：在中为自动插入单个建议下选择完成类型。

默认情况下，选中“ 键入时显示建议”复选框。

以字母顺序显示建议

  • 选择按字母顺序排序建议复选框。默认情况下，该复选框处于清除状态，并且按相关性对建议进行排序。

  • 替代使用要么在建议列表的右下角，可以切换这些模式。请注意，对于仅包含少量条目的列表，不会显示这些图标。

将有关案例的建议包括在列表中

  • 选中“ 区分大小写”复选框，然后选择是否只对首字母或所有字母都区分大小写。

打开自动文档查找

  • 选择显示文档弹出复选框。从基本完成：文档查找中了解更多信息。

使用特定键插入建议

  • 通过按空格，点或其他上下文相关的键复选框，选择“ 插入所选建议” 。这些键取决于调用完成功能的语言和上下文。

虽然此设置可以帮助您节省时间，但将其打开可能会导致偶然插入项目。

完成路径

这种完成方式可以加快文件和文件夹的选择速度。例如，当您指定外部工具或版本控制引擎的安装文件夹或配置Node.js解释器时，通常会调用用于此目的的对话框 。

文件和文件夹路径的完成在macOS上不可用。

  1. 通过在文件菜单上选择打开文件或通过单击浏览打开选择路径对话框 在需要路径的字段旁边。

  2. 确保“ 路径”字段可见。如果该字段是隐藏的，则单击“ 显示路径”链接。

  3. 开始输入路径。WebStorm会提供一个列表，您可以在其中快速找到所需的目录。键入时，建议列表将缩小以仅显示匹配的路径。您也可以按Ctrl+Space显示建议列表。

  4. 从列表中选择相关建议。

故障排除

如果无法完成代码，则可能是由于以下原因之一：

  • 省电模式已打开（ 文件|省电模式 ）。开启它可以消除背景操作，包括错误突出显示，即时检查和代码完成，从而将笔记本电脑的功耗降至最低。

  • 您的文件不驻留在内容根目录中，因此它没有获得代码完成所需的类定义和资源。

  • 带有要在代码完成中使用的符号的文件被标记为纯文本文件 。

  • 带有要在代码完成中使用的符号的第三方文件未配置为外部JavaScript库 。
