CoffeeScript

借助WebStorm，您可以使用可编译为JavaScript的CoffeeScript 。WebStorm识别* .coffee文件并将其标记为 。您可以直接在CoffeeScript代码中设置断点，然后WebStorm会使用编译期间生成的源映射来识别断点。

WebStorm通过建议关键字，标签，变量，参数和函数的完成来帮助您编写CoffeeScript代码。您可以从符号跳到其声明Ctrl+B或实施Ctrl+Alt+B并找到其名称的任何符号Ctrl+Shift+Alt+N 。

在你开始前

  1. 确保您的计算机上装有Node.js。

  2. 确保在“ 插件”页面上启用了CoffeScript和File Watchers捆绑的插件 ，有关详细信息，请参阅管理插件 。

将CoffeeScript编译为JavaScript

WebStorm与coffee-script编译器集成在一起，后者将CoffeeScript代码转换为JavaScript。该工具还会创建源映射 ，该源映射设置CoffeeScript代码中的行与生成的JavaScript代码中的行之间的对应关系，以便WebStorm调试器识别CoffeeScript代码中的断点并正确处理它们。 要即时编译代码，需要将编译器配置为WebStorm File Watcher 。

您也可以从命令行使用编译器，或者将其配置为第三方工具，有关详细信息，请参阅配置第三方工具 。

安装CoffeeScript编译器

  • 在嵌入式终端 （ Alt+F12 ），键入以下命令之一：

    • npm install --global coffeescript用于全局安装。

    • npm install --save-dev coffeescript将CoffeeScript安装为开发依赖项 。

      从CoffeeScript官方网站了解更多信息。

您还可以按照npm和Yarn中的说明，在node.js 和NPM页面上安装coffeescript软件包。

创建CoffeeScript文件监视程序

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 工具|文件监视程序 。

  2. 请点击或按Alt+Insert然后从列表中选择CoffeeScript预定义模板。您的代码将转换为JavaScript并随生成的源映射一起提供 。

  3. 在“ 程序”字段中，指定coffee.cmd文件的路径。手动输入路径或单击然后在打开的对话框中选择文件位置。

  4. 按照文件监视程序中所述进行操作。

编译代码

当您打开文件时，WebStorm会检查当前项目中是否有适用的文件监视程序。如果已配置但禁用了这种文件监视程序，则WebStorm会显示一个弹出窗口，通知您有关已配置的文件监视程序并建议启用它。

如果在当前项目中配置并启用了适用的文件监视程序，则WebStorm会在“ 新建监视程序”对话框中指定的事件后自动启动它。

  • 如果选中了“ 自动保存编辑的文件以触发监视程序”复选框，则在对源代码进行任何更改后都会立即调用文件监视程序。

  • 如果清除了自动保存编辑的文件以触发监视程序复选框，则文件监视程序会在保存时启动（ File | Save All ， Ctrl+S ）或将焦点从WebStorm移开（框架关闭时）。

编译器将生成的输出存储在单独的文件中。该文件具有源CoffeeScript文件的名称，扩展名.js或.js.map（取决于编译器类型）。在“ 新建监视程序”对话框的“ 刷新的输出路径”字段中定义了生成文件的位置。基于此设置，WebStorm会检测到编译器输出。但是，在“ 项目”工具窗口中，它们显示在源Coffee文件下，该文件现在显示为节点。

在不运行编译器的情况下预览编译结果

WebStorm可以在不实际运行编译器的情况下对CoffeeScript代码执行静态分析，并在专用的只读查看器中显示预测的编译输出。

  1. 在编辑器中打开所需的CoffeeScript文件，然后右键单击编辑器背景。

  2. 从上下文菜单中，选择“ 预览编译的CoffeeScript文件” 。预览在专用的只读查看器中打开：左侧窗格显示原始CoffeeScript源代码，右侧窗格显示编译器在运行时将生成的JavaScript代码。

运行CoffeeScript

在WebStorm中有两种运行CoffeeScript的方法：

  • 手动编译CoffeeScript代码，然后像运行Node.js应用程序一样运行输出的JavaScript代码。

  • 通过Node.js运行配置运行原始的CoffeeScript代码，并让WebStorm即时对其进行编译。

手动编译CoffeeScript并运行生成的JavaScript代码

  1. 将CoffeeScript代码编译为Javascript 。

  2. 开始使用以下必需设置创建Node.js运行配置 ：

    1. 要使用的Node.js引擎。默认情况下，该字段显示在Node.js配置期间在Node.js页面上指定的解释器的路径。

    2. 在工作目录字段中，指定要从运行的起始CoffeeScript文件引用的文件的位置，例如， include 。如果此文件未引用任何其他文件，则将该字段保留为空。

    3. 在“ Node App JS文件的路径”字段中，指定在编译过程中从原始CoffeeScript文件生成的JavaScript文件的完整路径。

  3. 保存配置，然后单击在工具栏上。

  4. 在运行Node.js应用程序时继续进行。

在运行过程中即时编译CoffeeScript

  1. 这种模式要求3软件包的一部分register.js文件应该位于项目内部。因此，您需要在本地的Node.js页面上安装coffee-script软件包，如npm和Yarn中所述 。

  2. 在编辑器中打开开始的CoffeeScript文件，或在“ 项目”工具窗口中选择，然后选择“ 创建” 从上下文菜单中或者，按照创建Node.js运行/调试配置中所述开始创建Node.js运行配置 。在打开的“运行/调试配置：Node.js”对话框中，指定以下必需设置：

    1. 要使用的Node解释器。选择相关的解释器配置或创建一个新的解释器配置，请参阅默认情况下，该字段显示在Node.js配置期间在Node.js页面上指定的解释器的路径。

      对于Linux和macOS，Node.js会从CoffeeScript编译器可执行文件的路径中覆盖此设置。

    2. 在“ 节点参数”字段中，键入--require coffee-script/register 。

    3. 在工作目录字段中， 指定应用程序的工作目录 。 默认情况下，该字段显示项目根文件夹 。

    4. 在“ JavaScript文件”字段中，指定要运行的CoffeeScript文件的完整路径。

    请注意，如果直接从所需的CoffeeScript文件中创建运行配置，则所有必填字段都会自动填写。

  3. 保存配置，然后单击在工具栏上。

  4. 在运行Node.js应用程序时继续进行。

调试CoffeeScript

要在WebStorm中调试CoffeeScript，除了JavaScript代码外，还需要生成源映射 。源映射设置CoffeeScript代码中的行与生成的JavaScript代码中的行之间的对应关系，否则将无法正确识别和处理您的断点。通过使用CoffeeScript类型的File Watcher手动编译CoffeeScript代码来生成JavaScript和源映射。之后，您可以调试输出的JavaScript代码，就像它是Node.js应用程序一样。

仅在本地模式下才支持调试CoffeeScript。这意味着WebStorm本身根据运行配置启动Node.js引擎和目标应用程序，并获得对会话的完全控制。

有关调试Node.js应用程序的更多详细信息，请参阅“ 运行和调试Node.js” 。

调试CoffeeScript代码

  1. 必要时在CoffeeScript代码中设置断点 。

  2. 使用CoffeeScript类型的File Watcher 将CoffeeScript代码编译为Javascript 。

  3. 开始使用以下必需设置创建Node.js运行配置 ：

    1. 要使用的Node.js引擎。默认情况下，该字段显示在Node.js配置期间在Node.js页面上指定的解释器的路径。

    2. 在工作目录字段中，指定要从运行的起始CoffeeScript文件引用的文件的位置，例如， include 。如果此文件未引用任何其他文件，则将该字段保留为空。

    3. 在“ Node App JS文件的路径”字段中，指定在编译过程中从原始CoffeeScript文件生成的JavaScript文件的完整路径。

  4. 保存配置，然后单击在工具栏上。

  5. 与在计算机上与Node.js应用程序一起启动调试器时一样进行 。

配置语法突出显示

您可以根据自己的喜好和习惯配置可识别CoffeeScript的语法突出显示。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |配色方案| CoffeeScript 。

  2. 选择配色方案，接受从默认设置继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。
