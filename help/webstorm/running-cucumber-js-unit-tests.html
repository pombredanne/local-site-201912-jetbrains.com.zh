Cucumber.js

Cucumber.js是用于行为驱动的JavaScript开发的测试框架。Cucumber.js测试以人类可读的Gherkin语言编写，并存储在具有功能扩展名的功能文件中。WebStorm与Cucumber.js集成并识别用Gherkin编写的功能，因此您可以直接在IDE中运行Cucumber.js测试。

在你开始前

  1. 确保您的计算机上装有Node.js。

  2. 确保在“ 插件”页面上启用了Cucumber.js和Gherkin捆绑的插件 ，有关详细信息，请参阅管理插件 。

安装Cucumber.js

  • 在嵌入式终端 （ Alt+F12 ），键入以下命令之一：

    • npm install cucumber用于项目中的本地安装。

    • npm install -g cucumber用于全局安装。

    • npm install --save-dev cucumber将Cucumber.js安装为开发依赖项 。

    另请参见Cucumber.js官方网站上的Cucumber.js演示 。

您还可以按照npm和Yarn中的描述，在Node.js和NPM页面上安装黄瓜包。

创建测试定义

使用WebStorm，可以在JavaScript和TypeScript中编写步骤定义。WebStorm会检测并突出显示缺少定义的步骤，并提出生成它们的快速修复方法。

  1. 将插入号放置在没有定义的步骤处，按Alt+Enter ，然后选择创建步骤定义或创建所有步骤定义 。

    或者，将鼠标悬停在该步骤上，并使用弹出窗口中的链接。

  2. 在打开的对话框中，指定新文件的名称以及用于写入定义的语言。

    如果此类文件已经存在，则WebStorm会显示一个弹出窗口，您必须在其中选择一个文件来向其中添加定义，或选择创建一个新文件。

运行测试

Cucumber.js测试仅通过运行/调试配置启动。

创建一个Cucumber.js运行配置

  1. 打开“ 运行/调试配置”对话框（主菜单上的“运行” |“编辑配置 ”），单击在左侧窗格中，然后从列表中选择Cucumber.js 。将打开“ 运行/调试配置：Cucumber.js”对话框。

    要在单个测试文件中为所有测试创建运行/调试配置，请选择“运行”。 从其上下文菜单中。

    要为一个方案创建运行/调试配置，请选择“ 创建方案”： 从编辑器的上下文菜单中。

  2. 在功能文件或目录字段中，指定要运行的测试。 如果要运行大量功能，请键入特定.feature文件或文件夹的路径。

  3. 指定要使用的Node.js解释器。这可以是本地Node.js解释器，也可以是Windows Subsystem for Linux上的Node.js。

  4. 在“ 黄瓜包装”字段中， 指定黄瓜包装袋所在文件夹的路径。

  5. 指定应用程序的工作目录。默认情况下， 工作目录字段显示项目根文件夹。要更改此预定义设置，请指定所需文件夹的路径，或从列表中选择以前使用的文件夹。

  6. 可选地：

    • 在“ 名称过滤器”字段中，键入要运行的特定方案的名称，而不是功能文件或目录中的所有方案。

    • 指定要传递给可执行文件的命令行参数，例如-r--require LIBRARY|DIR ， -t--tags TAG_EXPRESSION ， 要么--coffee 。有关详细信息，请参见通过cucumber-js --help命令。

    如果您为一个方案创建运行/调试配置（ 创建方案： ），WebStorm会在“ 名称过滤器”字段中自动填写方案的名称 。

通过运行配置运行测试

  1. 从主工具栏上的列表中选择Cucumber.js运行/调试配置，然后单击在列表的右边。

  2. 测试服务器将自动启动，而无需您采取任何步骤。在“ 运行工具”窗口中查看和分析来自测试服务器的消息。

  3. 在“ 运行”工具窗口的“ 测试运行器”选项卡中监视测试执行 。

从测试文件运行和调试测试

  • 要运行或调试单个测试，请在编辑器中打开测试文件，将插入号置于要运行或调试的场景下，然后选择“运行场景”： 或调试方案： 从上下文菜单中。

  • 要从单个测试文件运行或调试所有测试，请在编辑器中打开测试文件，或在“ 项目”工具窗口中选择它，然后选择“运行”。 或调试从上下文菜单中。

无论哪种情况，WebStorm都会创建运行/调试配置，您可以在以后保存和使用该配置。

运行和调试用TypeScript编写的测试

  1. 如上所述创建Cucumber.js运行/调试配置。

  2. 在启动前区域中，单击 ，从列表中选择“ 编译TypeScript ”，然后指定要使用的tsconfig.json文件。WebStorm将在运行Cucumber.js之前使用此tsconfig.json运行TypeScript编译器。

    或者，将TypeScript编译设置为另一个构建步骤。

  3. 要调试用TypeScript编写的测试，请打开tsconfig.json并设置"sourceMap": true启用源地图。

导航

使用WebStorm，可以在文件和相关测试文件之间跳转，也可以从“ 测试运行器”选项卡中的测试结果跳转到测试。

  • 在编辑器中打开文件，然后选择“ 转到|测试或转到|从上下文菜单中测试受试者 ，或者只需按Ctrl+Shift+T 。

  • 要从.feature文件中的步骤跳至步骤定义，请按住Ctrl ，将鼠标悬停在该步骤上，然后在该步骤变成链接时单击它。

  • 要从测试结果跳转到测试，请在“ 测试运行器”选项卡中选择测试名称，然后从上下文菜单中选择“ 跳转到源代码 ”。测试文件将在编辑器中打开，并将光标置于测试定义上。

配置语法突出显示

您可以根据自己的喜好和习惯配置支持Cucumber.js的语法突出显示。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |配色方案|黄瓜

  2. 选择配色方案，接受从默认设置继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。
