“语言注入设置”对话框：通用JavaScript

档案|设置|编辑器语言注入-适用于Windows和Linux的通用JavaScript
WebStorm |首选项|编辑器语言注入-适用于macOS的通用JavaScript
Ctrl+Alt+S

单击时将打开对话框并从上下文菜单中选择通用JS ，或选择一个条目并单击 。

WebStorm带有一组预定义的注射配置，这些配置足以确保高生产率和舒适的环境。因此， 强烈建议您使用预定义的进样配置，并避免创建新的进样配置。

项目

描述

名称

注射的名称。

语言

要注入的语言。

  • ID。 语言ID或名称。

  • 字首。 在相应的字符串值之前要添加的字符序列。

  • 后缀。 在相应的字符串值之后要添加的字符序列。

前缀和后缀是可选的。

位置模式

在此字段中，输入规则，这些规则定义了您希望WebStorm将文字识别为注入的上下文。

这些规则是从“ 程序结构接口模式”构建的 ，实际上是内部WebStorm语言方法的链式调用。程序结构界面显示了WebStorm处理文件时的结构。

这些规则是WebStorm的内部规则，强烈建议您使用预定义的注入配置，并避免创建新的注入配置。

高级

在此区域中，指定其他设置以缩小可应用注入的上下文，从而实现对注入过程的更精细控制。

  • 值模式 -在此字段中，输入一个正则表达式，以确定将语言注入其中的上下文。通过使用模式的第一个捕获组作为注入目标，您可以将过程配置为仅将语言注入到与特定模式匹配的值或与模式匹配的多个部分中。例如， ^javascript:(.*)符合javascript可以在超链接-hrefs中使用以执行JavaScript代码的协议。

  • 单一档案 - 如果关闭该选项，则与值模式匹配的片段将被分别视为不同的“文件”，例如，从片段编辑器的角度来看。

    如果启用此选项，则所有对应的片段都将合并在一起以形成一个单元或“文件”。

    给定价值模式

    xxx (.+) yyy (.+) zzz

    和片段

    xxx select * yyy from family zzz ，

    select *和from family如果关闭该选项，则被视为两个独立的片段（或“文件”）。如果该选项打开， select * from family被视为一个单位或“文件”。
