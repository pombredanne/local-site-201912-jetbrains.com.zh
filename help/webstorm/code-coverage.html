代码覆盖率

借助WebStorm， 您还可以监视测试覆盖了多少代码。WebStorm在专用工具窗口中显示此统计信息，并在编辑器中直观地标记覆盖和未覆盖的线。

  • Coverage工具窗口工具窗口显示每个文件夹包含的类和行的百分比以及每个文件包含的行的百分比。

  • 在编辑器中打开文件时，在仿真过程中执行的行标记为绿色，而未涉及的行标记为红色。

覆盖范围测量的结果保存在覆盖范围套件中 。您也可以将它们与任何现有套件合并。在这种情况下，如果至少有一个测试运行覆盖了某条线，则该行被视为已覆盖。

测量代码覆盖率可用于Mocha测试和Karma测试 。

进行覆盖测试

  • 创建Mocha或Karma运行配置，从工具栏上的列表中选择它，然后单击在列表旁边。

  • 或者，使用编辑器中的测试图标快速运行特定的套件或具有覆盖率的测试：

配置代码覆盖行为

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 构建，执行，部署|覆盖范围 。

  2. 定义如何处理收集的覆盖率数据：

    • 要在每次启动具有代码覆盖率的新运行配置时显示“ 代码覆盖率”对话框，请在将覆盖率应用于编辑器之前选择“ 显示选项” 。

    • 要舍弃新的代码覆盖率结果，请选择不应用收集的覆盖率 。

    • 要丢弃活动套件并在每次启动具有代码覆盖范围的新运行配置时使用新套件 ，请选择“ 用新套件替换活动套件” 。

    • 要在每次启动具有代码覆盖率的新运行配置时将新的代码覆盖率套件附加到活动套件中，请选择添加到活动套件 。

  3. 当应用程序或测试以coverage运行时，定义Coverage工具窗口的行为：

    • 要自动打开Coverage工具窗口，请选中“ 激活Coverage视图”复选框。

    • 要手动打开Coverage工具窗口，请清除“ 激活Coverage视图”复选框。
