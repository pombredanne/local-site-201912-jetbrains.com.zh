TODO评论

在处理应用程序时，您可能需要标记代码的某些部分以供将来参考：要优化和改进的区域，可能的更改，要讨论的问题等等。在WebStorm中，您可以添加特殊类型的注释，这些注释在编辑器中突出显示并在TODO工具窗口中列出。

使用默认模式

默认情况下，WebStorm识别两种模式-单词TODO和FIXME无论是大写还是小写这些模式可以在任何受支持的文件类型的注释内使用。

创建待办事项注释

  1. 将插入符号放置在要创建TODO项的位置并添加注释，例如，通过按Ctrl+/ ，然后输入TODO要么FIXME ，然后键入您的笔记。

  2. 在“ 待办事项”工具窗口中查看待办事项列表（ Alt+6 ）。工具窗口由四个选项卡组成，其中WebStorm显示整个项目，活动编辑器选项卡中的文件，所选作用域或活动变更列表的TODO项目。

    • 默认情况下，当您选择一个项目时，您会自动跳到源代码中的相应注释。要禁用此自动导航，请右键单击工具窗口标题栏，单击“ 查看选项” ，然后删除“ 单击导航”旁边的对勾。

    • 在禁用“ 单击导航”的情况下 ，双击所需的TODO项以跳至源代码中的相应注释。

在JavaScript和TypeScript文件中，WebStorm默认情况下还支持多行TODO注释。用通常的方式开始TODO注释TODO并在下一行添加额外的缩进以继续TODO注释。如果删除缩进，WebStorm会将其视为普通注释。

禁止多行TODO注释

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），点击编辑器下的TODO 。

  2. 在打开的“ TODO”页面上，清除“ 将以下行中的缩进文本视为同一TODO”复选框的一部分。

使用自定义模式

您可以修改默认模式或添加自己的模式，并过滤列表以仅显示与某些模式匹配的TODO项目。例如，您可能想用OPTIMIZE模式并忽略所有其他类型的TODO项目。

创建一个自定义模式

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），点击编辑器下的TODO 。将打开“ 待办事项”页面 。

  2. 为模式添加OPTIMIZE项目，单击在“ 模式”区域中，键入以下模式： \boptimize\b.*

  3. 要使用新模式添加过滤器，请点击在“ 过滤器”区域中，指定新过滤器的名称，然后选择要包含在此过滤器中的新模式。

应用自定义过滤器

  • 在“ TODO工具”窗口中，单击并选择必要的过滤器。
