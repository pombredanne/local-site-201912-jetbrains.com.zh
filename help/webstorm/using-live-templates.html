实时模板

通过使用实时模板 ，您可以将常用结构插入代码中。例如，循环，条件，各种声明或打印语句。

要展开代码段，请键入相应的模板缩写，然后按Tab 。继续按Tab从模板中的一个变量跳转到下一个变量 。按Shift+Tab移至上一个变量。

实时模板的类型

区分以下类型的实时模板：

  • 简单模板仅包含固定的纯文本。扩展简单模板时，文本将自动插入源代码中，以代替缩写。例如， flow插入一个@flow注解， vbase扩展为Vue单个文件组件的存根，依此类推。

  • 参数化的模板包含允许用户输入的变量 。扩展参数化模板时，变量将由输入字段替换，以供用户手动指定，或者由WebStorm自动计算。例如， importitems扩展为import语句，您可以先在其中指定路径，然后再指定导入的成员作为参数。

  • 环绕模板用用户指定的文本包装所选代码块。例如， T扩展为一对标签，您可以为其指定名称。您也可以选择一个代码块，然后按Ctrl+Alt+J打开“ 选择模板”弹出窗口，然后选择T模板，以使用一对标签包装选择内容。

JavaScript Postfix模板是实时模板，但是会为现有代码创建构造。例如，您可以输入.if在表达式之后调用相应的后缀完成并用if声明。

配置实时模板

要配置实时模板，请打开 编辑器| WebStorm设置的“ 实时模板”页面Ctrl+Alt+S 。在“ 实时模板”页面上，您可以查看所有可用的实时模板，对其进行编辑并创建新模板 。

为了使查找和编辑模板更加容易，将它们分为几组。要将模板移至另一个组，请右键单击该模板，选择Move ，然后选择所需的组名。

如果为活动模板设置了方言，则该活动模板仅在该方言中有效。

每个活动模板均由包含字母数字字符，点和连字符的缩写定义。缩写在一个组中必须唯一，但是相同的缩写可以在不同的组中使用，并根据相应组的上下文扩展到不同的构造。修改后的默认模板的缩写以蓝色字体显示在列表中。

将修改后的模板还原为默认设置

  • 在“ 设置/首选项”对话框的“ 实时模板”页面上，右键单击要还原的模板，然后单击“还原默认值” 。
