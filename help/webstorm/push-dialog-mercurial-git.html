推送对话框（Mercurial，Git）

该对话框可用于以下版本控制系统：

  • 吉特

  • 水银

该对话框包含两个窗格（“ 存储库”窗格和“ 提交详细信息”窗格）和“ 推送控件”区域。

存储库窗格

左窗格显示Git和/或Mercurial存储库的列表（以及哪个本地分支/活动书签将被推送到哪个远程分支），以及在每个存储库中执行的提交的列表。

  • 将鼠标悬停在提交上：显示工具提示，显示提交编号，日期和时间，作者以及提交消息。如果提交的作者与当前用户不同，则此提交带有星号。

  • 选中要推送到的每个存储库旁边的复选框。

    如果您有一个不同步控制存储库的多根项目，则默认情况下仅选择当前存储库（或在“ 项目视图”中选择多个存储库）。有关如何启用/禁用同步存储库控制的详细信息，请参阅以下资源：

    • 对于Git ： 版本控制设置：Git

    • 对于Mercurial ： 版本控制设置：Mercurial

  • 要修改要推送到的目标分支（以蓝色突出显示），请单击它。标签变成一个文本字段，您可以在其中指定目标分支。您还可以通过选择要修改的分支并按来切换到编辑模式。 F2 。

  • 您也可以以与远程分支相同的方式编辑远程存储库（如果有多个存储库）。请注意，如果未指定任何远程，则将显示“ 定义远程”链接，而不是远程名称。单击它添加一个遥控器。

  • 如果存储库中没有远程服务器，则将显示“ 定义远程”链接。单击此链接，然后在打开的对话框中指定远程名称和URL。

提交详细信息窗格

右窗格显示了所选提交中包含的文件。如果在左窗格中选择多个分支，则将显示所有相应的提交。

该区域的工具栏提供以下选项：

项目

工具提示和快捷方式

描述

显示差异

Ctrl+D

单击此按钮以打开“ 文件差异查看器”对话框，该对话框显示所选文件的提交版本与其先前版本之间的差异。

编辑来源

F4

单击此按钮可在编辑器中打开所选文件。

按目录分组

Ctrl+P

单击此按钮可以在平面视图和目录视图之间切换。

全部折叠/全部展开

Ctrl+NumPad - /Ctrl+NumPad Plus

单击这些按钮以折叠/展开目录树中的所有节点。如果选择了平面视图，则这些按钮不可用。

推动控制

此区域中的控件使您可以选择以下推送选项：

项目

描述

推送标签

仅当您使用Git时，此选项才可用。

默认情况下，当您执行push操作时，不会将标签发送到远程存储库。如果要在提交中推送标签，请选择此选项。

  • 如果要推送所有标签，包括不属于您要推送的所选分支的标签，请选择全部 （等同于push --tags ）。

  • 如果您只想推送属于您要推送的所选分支的标签，则选择“ 当前分支”。 push --follow-tags自Git 1.8.3起可用）。

导出活动书签

仅当您使用Mercurial时，此选项才可用。

默认情况下，当您执行push操作，书签不会发送到远程存储库。如果您要在提交时推送活动书签，请选择此选项。

推

单击此按钮，然后从下拉菜单中选择要执行的操作： push要么push --force 。

有关如何使用push --force命令以及可能有用的地方，请参考：

  • 对于Git：将更改推送到远程存储库

  • 对于Mercurial： 使用强制推送
