Creating a Command Line Tool

The command line tools that are available from WebStorm are listed in the Command Line Tool Support page of the Settings dialog box. Integration with command line tools is provided at the WebStorm level, so once configured, a tool can be used in all your WebStorm projects. Just activate and de-activate when necessary depending on the needs of each specific project.

You can update the command definitions of a tool and have the .xml definition file validated right in the WebStorm editor.

In this section:

  • Creating and configuring a command line tool
  • Activating a configured command line tool

To create and configure a command line tool

  1. Open the WebStorm settings, and then click the Command Line Tool Support.
  2. In the Command Line Tool Support page, click the Add button. In the Create Custom Tool dialog box that opens, click Yes
  3. In the Tool Settings dialog box that opens, specify the following:
    1. Tool name and location.
    2. The alias to use in command calls instead of the full path to the tool.
    3. Brief explanation of the tool functionality.
  4. When you click OK, WebStorm brings you to the Command Line Tool Support page, where the new tool is added to the list of available tools.
  5. From the Console encoding drop-down list, choose the character set to show the tool's output in the Command Line Tools Console Tool Window.
  6. Select the newly created tool and click the Open definition in editor button. In the tool definition .xml file that opens in the editor, define the tool commands.

To activate a configured command line tool

  1. Open the WebStorm settings, and then click the Command Line Tool Support.
  2. Select the Enable check box next to to the tool name.
