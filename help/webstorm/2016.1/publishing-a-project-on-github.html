Publishing a Project on GitHub

You can publish your project sources in a GitHub Repository right from WebStorm.

Make sure that you have registered your GitHub account credentials in WebStorm.

To publish your project sources on GitHub

  1. On the main menu, choose VCS | Import into Version Control | Share Project on GitHub.
    • If you have registered login and password, WebStorm establishes connection with GitHub using these credentials.
    • If you have not registered your GitHub credentials in WebStorm, the Login to GitGub dialog box. Specify your GitHub login and password or create an account there.
  2. Upon establishing connection, the Share Project on GitHub dialog box opens. Specify the name of the repository to store your project sources in. By default, WebStorm suggests the name of the current project. Provide a brief description of the project functionality.

    Click the Share button. WebStorm initiates creation of the new repository on the GitHub and uploads the project sources to it.
