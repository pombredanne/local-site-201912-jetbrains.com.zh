WebStorm 2019.3帮助

产生程式码

WebStorm提供了多种生成通用代码结构和重复元素的方法，可帮助您提高生产率。这些可以是预定义模板，也可以是自定义模板，它们会根据上下文，各种包装程序和字符自动配对而不同地应用。

此外，WebStorm还提供代码完成和Emmet支持。

在主菜单中，选择代码|生成 Alt+Insert打开带有您可以生成的可用结构的弹出菜单。

使用实时模板创建代码构造

WebStorm为许多常见的代码构造提供了许多预定义的实时模板 。您还可以定义自定义模板，以涵盖特定于您的工作流程的用例。

您可以通过插入和扩展实时模板以及使用环绕模板包装代码片段来创建代码构造。

插入实时模板

  1. 将插入符号放置在您希望模板扩展的位置。

  2. 输入模板缩写，然后按调用键（ Tab默认）。

    或者，要使用代码完成功能，请按Ctrl+J或从上下文菜单中选择“ 插入实时模板” ，然后从建议列表中选择所需的模板。要查看所选建议的快速文档 ，请按Ctrl+Q 。

  3. 如果所选模板需要用户输入，则第一个输入字段会用红色框标记。在此框中键入您的值，然后按Enter要么Tab完成输入并传递到下一个输入字段。在完成最后一个输入字段后，插入符号将移至构造的末尾，编辑器将返回常规操作模式。

    有关详细信息，请参见实时模板：参数化模板 。

用实时模板包围代码片段

  1. 在编辑器中，选择要包装的代码，然后按Ctrl+Alt+J或从上下文菜单或选择中选择“ 实时模板环绕” 。

  2. 在建议列表中，选择所需的模板。

查看可用的实时模板列表

  • 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），点击编辑器下的实时模板 。打开的实时模板页面显示按语言分组的所有已配置实时模板。有关更多信息，请参见创建实时模板 。

展开和删除语句

WebStorm使您可以快速从封装语句中解开或提取表达式。如果... else ， for ， while和do ... while控件结构以及XML和HTML标记， 则此动作可用于JavaScript。

  1. 将插入符号放在要提取或解包的表达式上。

  2. 选择代码|从主菜单解包/删除或按Ctrl+Shift+Delete 。WebStorm显示一个弹出窗口，其中包含当前上下文中可用的所有操作。要提取的语句显示在蓝色背景上，要删除的语句显示在灰色背景上。

  3. 单击所需的操作或使用向上和向下箭头键选择它，然后按Enter 。

完成配对字符

WebStorm可以自动添加右括号和其他成对的元素。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），单击“ 编辑器”下的“ 常规 ” ，然后单击“ 智能键” 。将打开“ 智能键”页面 。

  2. 按下时自动添加右括号Enter ，在输入区域中选择插入对}复选框。

  3. 通过选择XML / HTML区域中的相关复选框，配置自动插入结束标记。

缩进级别在Code Style中定义。

生成构造函数

WebStorm可以生成一个构造函数，该构造函数使用相应参数的值来初始化特定字段。

类Person {私有只读名称：字符串private只读lastName：字符串}
类Person {私有只读名称：字符串private只读lastName：字符串构造函数（名称：字符串，lastName：字符串）{this.name =名称； this.lastName = lastName; }}

  1. 从课程的上下文菜单中选择“ 生成” ，或按Alt+Insert 。

  2. 在“ 生成”弹出窗口中，单击“ 构造函数” 。

  3. 如果该类包含字段，请选择要由构造函数初始化的字段，然后单击OK 。

生成getter和setter

WebStorm可以为类中的字段生成访问器和更改器方法（ getter和setter ）。WebStorm会根据您的代码生成命名首选项生成getter和setter的名称，有关详细信息，请参见JavaScript代码样式：代码生成和TypeScript代码样式：代码生成 。

类Person {私有名称：字符串private lastName：字符串}
class Person {get name（）：string {return this._name; } get lastName（）：string {return this._lastName; }私人_name：字符串私人_lastName：字符串}
类Person {设置名称（值：字符串）{this._name = value; } set lastName（value：string）{this._lastName = value; }私人_name：字符串私人_lastName：字符串}

  1. 从课程的上下文菜单中选择“ 生成” ，或按Alt+Insert 。

  2. 在“ 生成”弹出窗口中，单击以下选项之一：

    • Getter生成访问器方法以获取类字段的当前值。

    • Setter生成用于设置类字段值的增变方法。

    • 使用Getter和Setter生成访问器和更改器方法。

  3. 在打开的对话框中，选择要为其生成getter或setter的字段，然后单击“ 确定” 。

接口或抽象类的实现方法

如果一个类被声明为实现接口或抽象类，则它必须实现父类或基本接口的方法。WebStorm为实现的方法创建存根。

  1. 从类的上下文菜单中选择“ 生成 ” Alt+Insert并从弹出菜单中选择“ 实施成员” ，或按Ctrl+I 。

  2. 在打开的对话框中，选择要实现的方法，然后单击确定 。

超类的重写方法

您可以通过从预定义的模板生成必要的代码来覆盖父类的任何方法。WebStorm创建一个存根，该存根包含对超类的方法的调用，您只需要在方法的主体中提供一些有意义的源代码即可。

  1. 从类的上下文菜单中选择“ 生成 ” Alt+Insert然后从弹出菜单中选择“ 替代方法” ，或按Ctrl+O 。

  2. 在打开的对话框中，选择要覆盖的方法。方法列表不包括已经被覆盖的方法，或者不能从当前子类访问的方法。

  3. 单击“ 确定”，并提供方法主体的源代码。使用在左装订线中标记覆盖方法的图标，以查看基类的名称，并打开覆盖方法的声明 。

您也可以使用代码完成功能，并从完成建议列表中选择父方法。WebStorm自动添加参数，生成一个super()调用，并在可能的情况下显示类型信息。从自动完成代码和路径中了解更多信息。

关闭替代的方法体的生成

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），请转到语言和框架| JavaScript 。

  2. 在打开的JavaScript页上，清除“ 展开方法主体以完成覆盖”复选框。
