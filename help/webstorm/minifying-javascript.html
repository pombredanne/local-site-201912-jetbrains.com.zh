WebStorm 2019.3帮助

缩小JavaScript

WebStorm与Closure Compiler ， YUI Compressor和UglifyJS集成在一起，可让您即时压缩JavaScript应用程序源。

缩小或压缩意味着在不更改源代码功能的情况下删除所有不必要的字符（例如空格，换行和注释）。 在开发和调试期间，这些字符使代码更易于阅读。在生产阶段，它们只会增加要传输的代码的大小。

在你开始前

  1. 确保您的计算机上装有Node.js。

  2. 确保在“ 插件”页面上启用了File Watchers捆绑的插件 ，有关详细信息，请参阅管理插件 。

  3. 如果要使用YUI Compressor或Closure Compiler，请下载并安装Java Runtime Environment（JRE） ：

    • 适用于YUI Compressor的JRE 1.4或更高版本。

    • 用于Closure编译器的JRE版本7。

安装缩小工具

  • 要安装UglifyJS，请打开内置的WebStorm 终端 Alt+F12和类型npm install uglify-js要么npm install uglify-js@ 。从UglifyJS官方网站了解更多信息。

  • 要安装YUI Compressor，请按照YUI Compressor官方网站上的说明进行操作。

  • 要安装Closure Compiler，请按照Closure Compiler官方网站上的说明进行操作。

压缩代码

要动态压缩代码，您需要将缩小工具配置为WebStorm File Watcher 。

创建文件监视程序

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 工具|文件监视程序 。

  2. 在打开的“ 文件监视程序”页面上，单击然后从列表中选择相关的File Watcher模板。

  3. 在打开的“ 新建监视程序”对话框中，在“ 程序”字段中指定相关可执行文件或jar存档的路径：

    • compiler.jar用于Closure编译器。

    • yuicompressor-.jar用于YUI Compressor JS。

    • uglifyjs.cmd用于UglifyJS。

  4. （可选） 自定义File Watcher的行为 。

  5. 确保已选中“文件监视程序”旁边的复选框，该复选框指示已启用“文件监视程序”。

运行压缩机

启用缩小文件监视程序时，一旦更改或保存了文件监视程序范围内的JavaScript文件，缩小就会自动开始。

WebStorm使用生成的输出创建一个单独的文件。该文件具有源JavaScript文件的名称以及扩展名min.js。在“ 新建监视程序”对话框的“ 刷新的输出路径”字段中定义了生成文件的位置。但是，在项目树中 ，默认情况下，它显示在源JavaScript文件下，该文件现在显示为节点。要更改默认演示文稿，请在“ 项目”工具窗口中配置文件嵌套 。
