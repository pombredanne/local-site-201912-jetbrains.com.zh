WebStorm 2019.3帮助

镖

使用WebStorm，您可以开发，运行和调试Dart Web和命令行应用程序。WebStorm可帮助您提供代码完成，错误和语法突出显示，代码检查和快速修复，搜索和导航，重构等更多功能。WebStorm还与pub工具和Dart Analysis Server集成。

在你开始前

如管理插件中所述，在“ 插件”页面上安装并启用Dart存储库插件。

下载Dart工具

下载并安装Dart SDK 。

Dart SDK包含用于开发命令行和Web Dart应用程序的所有工具。它包含Dart虚拟机 ， Dart核心库以及所有命令行工具，包括编译器（ dart2js和dartdevc ）， dartfmt代码格式化程序和pub工具。

从Dart官方网站了解有关Dart开发工具的更多信息。

创建一个新的Dart应用程序

如果还没有应用程序，则可以从Stagehand模板生成具有Dart特定结构的WebStorm项目。或者，创建一个空的WebStorm项目并在其中配置Dart支持，如下面的从现有Dart应用程序开始所述。

从Stagehand模板创建Dart项目

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择Dart 。

  3. 在右侧窗格中， 指定将存储与项目相关的文件的文件夹的路径。

  4. 指定Dart SDK的路径。WebStorm检测并显示Dart版本 。

    为每个项目分别配置了Dart SDK。这意味着您可以在WebStorm中拥有多个Dart SDK，并在一个项目与另一个项目之间切换。

  5. 要在项目中创建示例应用程序，请选中“ 生成示例内容”复选框，然后从下面的列表中选择相关的Stagehand模板。如果清除该复选框，则WebStorm会创建一个空项目。

  6. 点击创建 。 WebStorm设置项目结构并根据所选的Stagehand模板生成一些源。

创建一个空的WebStorm项目

  1. 选择文件|新品|项目从主菜单或单击欢迎屏幕上创建新项目 。将打开“ 创建新项目”对话框 。

  2. 在左侧窗格中，选择空项目 。在右侧窗格中，指定应用程序文件夹，然后单击创建 。

从现有的Dart应用程序开始

如果要继续开发现有的Dart应用程序，请在WebStorm中打开它，在其中配置Dart，然后按照下面的“ 管理Dart依赖项”中的说明下载所需的依赖项。

打开计算机上已经存在的应用程序源

单击“ 欢迎”屏幕上的“ 打开 ”，或选择“ 文件” |“共享”。从主菜单打开 。在打开的对话框中，选择存储源的文件夹。

从版本控制中检出应用程序源

  1. 在“ 欢迎”屏幕上单击“从版本控制获取”，或选择“ VCS |从主菜单的版本控制中获取 。

  2. 在调用的对话框中，从列表中选择您的版本控制系统，然后指定存储库以签出应用程序源。

在现有项目中配置Dart支持

  1. 在“设置/首选项”对话框中Ctrl+Alt+S ，在“ 语言和框架”下选择“ Dart ”。Dart页面打开。

  2. 选择项目的Enable Dart支持复选框。

  3. 在Dart SDK路径字段中， 指定下载的Dart SDK的位置。手动输入路径或单击然后在打开的对话框中选择路径。如果WebStorm正确识别Dart SDK，则其修订号将显示在“ 版本”只读字段中。

在一个WebStorm项目中使用多个Dart项目（程序包）

要将Dart项目（程序包）附加到现有WebStorm项目，您需要将其根文件夹添加为内容根。

将Dart项目（程序包）添加到现有的WebStorm项目

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到目录 。“ 目录”页面打开。

  2. 在带有现有内容根目录列表的右侧窗格中，单击添加内容根 。然后在打开的对话框中，选择要附加的Dart项目（程序包）的根文件夹。

管理Dart依赖项

WebStorm与pub工具集成，可让您直接在编辑器中运行其操作。一旦打开pubspec.yaml文件，WebStorm就会在其编辑器选项卡的顶部显示一个窗格。使用此窗格上的链接调用pub get ， pub upgrade和pub缓存修复操作。

或者，在编辑器中打开pubspec.yaml或在“ 项目”工具窗口中选择它，然后从所选内容的上下文菜单中调用pub操作。

您也可以在终端仿真器的命令行模式下使用该工具。

pub工具将下载的软件包保存在缓存中，并在pubspec.yaml文件旁边创建一个.packages文件和一个pubspec.lock文件。

WebStorm在消息工具窗口中记录发布命令的执行情况，您还可以通过单击以下命令来重新运行最后执行的命令在工具栏上。

建立Dart应用程式

  1. 在编辑器中打开pubspec.yaml文件，或切换到打开该文件的选项卡。

  2. 在选项卡顶部的窗格上，单击Build 。这将调用webdev构建操作。

  3. 在打开的对话框中，指定要从中获取源代码的输入文件夹以及要在其中存储编译后的应用程序的输出文件夹。

或者，从pubspec.yaml的上下文菜单中选择webdev build 。

对于Dart SDK 2之前的版本，将调用操作发布构建 。

运行和调试Dart命令行应用程序

使用WebStorm，可以运行和调试Dart命令行应用程序。WebStorm支持两种调试模式：

  • 本地调试 ：在此模式下，您的应用程序是从WebStorm启动的，并在计算机上本地运行。要运行或调试它，请使用Dart命令行应用程序配置。

  • 调试远程应用程序 ：在这种模式下，您的应用程序在远程环境中运行，例如在Docker容器中。要对其进行调试，请使用Dart远程调试配置。

运行Dart命令行应用程序

  1. 打开Dart文件以从中启动应用程序，或在“ 项目工具”窗口中选择它。该文件必须包含一个main()方法。

  2. 从选择的上下文菜单中，选择Run' ' 。WebStorm会生成Dart命令行应用程序类型的运行/调试配置，并使用它启动您的应用程序。

在本地调试Dart命令行应用程序

  1. 在Dart代码中配置和设置断点 。

  2. 打开Dart文件以从中启动应用程序，或在“ 项目工具”窗口中选择它。该文件必须包含一个main()方法。

  3. 从选择的上下文菜单中，选择Debug' ' 。WebStorm会生成Dart命令行应用程序类型的运行/调试配置，并使用它启动调试会话。

  4. 在打开的“ 调试工具”窗口中， 逐步执行程序 ， 停止并继续执行程序， 在挂起时对其进行检查 ，等等。

创建Dart远程调试运行/调试配置

  1. 在主菜单上，选择“运行”。编辑配置 ，单击并从列表中选择Dart远程调试 。将打开“ 运行/调试配置：Dart远程调试” 。

  2. 在主机字段中， 指定运行Dart虚拟机的计算机的地址，默认值为localhost 。

  3. 指定调试器将通过该端口连接到远程应用程序的端口，默认值为5858 。 启动远程VM只读时，在“ 使用命令行参数”字段中显示指定的端口。 请注意，必须使用这些参数完全启动远程应用程序。

  4. 在“ 搜索来源”列表中， 如果您的WebStorm项目包含配置为内容根目录的多个Dart项目，请选择Dart项目进行调试。

启动远程调试会话

  1. 使用Dart远程调试运行配置中远程Dart VM字段的命令行参数中的VM选项来启动远程Dart应用程序，例如， --enable-vm-service:5858 --pause_isolates_on_start 。该应用程序启动，由于--pause_isolates_on_start参数，并等待调试器连接。

  2. 在“ 选择运行/调试配置”列表中选择新创建的Dart远程调试配置，然后单击 。

  3. 在打开的“ 调试工具”窗口中， 逐步执行程序 ， 停止并继续执行程序， 在挂起时对其进行检查 ，等等。

运行和调试Dart Web应用程序

您可以在任何浏览器中运行Dart Web应用程序，而只有Chrome支持调试。要运行Dart Web应用程序，请在浏览器中打开应用程序的主HTML文件。Dart Web应用程序的调试是通过JavaScript Debug类型的运行配置启动的。

WebStorm与webdev服务工具集成在一起， 可在必要时将Dart代码编译为JavaScript。对于Dart SDK 2之前的版本，使用发布服务工具。

在浏览器中打开Dart Web应用程序时，它通常以内置服务器URL开头，例如http：// localhost：63342 / project-name / web / index.html 。但是，内置服务器不用于为应用程序提供服务。相反，WebStorm会在Dart对话框中指定的端口上自动启动Webdev服务器，并将浏览器页面重定向到Webdev服务器URL（例如http：// localhost：53322 / index .html ）。

默认的webdev服务器端口为53322 。如果该端口已经繁忙，则WebStorm使用最接近的空闲端口。

Webdev服务器的工作记录在专用的Dart Webdev工具窗口中。在当前WebStorm会话期间首次开始运行或调试Dart Web应用程序时，将打开工具窗口。您可以通过以下方式停止该工具在工具栏上。当您再次开始运行或调试时，webdev服务器将自动重新启动。

建议使用JavaScript调试配置来调试Dart Web应用程序。但是，当Webdev服务在启用Dart DevTools的情况下启动时，WebStorm提供了替代工作流程。在这种情况下，您需要Dart Web运行/调试配置。有关详细信息，请参见运行和调试Dart Web应用程序的替代方法 。

运行Dart Web应用程序

  1. 使用Dart引用打开HTML文件，或在“ 项目工具”窗口中选择它。

  2. 从编辑器或选择的上下文菜单中，选择“ 在浏览器中打开”，然后在列表中选择所需的浏览器。

或者，在编辑器中打开HTML文件，然后按Alt+F2 ，然后从列表中选择一个浏览器。

创建JavaScript调试运行/调试配置

  1. 打开引用Dart的HTML文件，或在“ 项目工具”窗口中选择文件。

  2. 从上下文菜单中，选择创建' ' 。将打开“ 运行/调试配置：JavaScript调试”对话框 。

  3. 从Chrome系列中指定浏览器以调试应用程序。

  4. 的URL现场已经显示格式为http应用程序的URL地址：//本地主机：/ / 。在调试会话期间，浏览器将从该URL重定向到webdev服务器URL。

    确保此URL地址中的端口与“ 调试器”页面上的内置服务器端口相同。

开始调试

  1. 在Dart代码中配置和设置断点 。

  2. 启动调试会话：从工具栏上的“ 编辑配置”列表中选择创建的运行配置，然后单击“确定”。 。WebStorm在选定的浏览器中打开指定的URL。

测试Dart应用程序

WebStorm支持运行和调试使用dart测试包编写的Dart测试。您可以在任何目标平台上运行测试，仅VM测试支持调试。

您可以运行和调试单个测试，测试组以及整个文件和文件夹中的测试。WebStorm使用默认设置创建运行/调试配置，然后启动测试。您以后可以保存此配置以供进一步使用。

运行或调试单个测试

打开测试文件，在编辑器中，右键单击的调用test（）方法 ，然后从上下文菜单中选择Run“”或调试'。

运行或调试组

打开测试文件，在编辑器中，右键单击的通话组（）方法 ，然后从上下文菜单中选择Run“”或调试'。

从文件运行或调试Dart测试

在项目工具窗口 ，然后从上下文菜单与测试文件来运行，然后选择Run“”或调试'。

从文件夹运行或调试Dart测试

在项目工具窗口中，选择与测试“或Debug'从上下文菜单中的文件夹运行并选择运行”。

如果选择了文件夹，则测试运行程序仅在名称格式为* _test.dart的文件中查找测试。

保存自动生成的默认配置

测试会话结束后，选择保存从测试，测试组，测试文件或文件夹的上下文菜单中。

使用先前保存的运行/调试配置运行或调试测试

从工具栏上的列表中选择所需的Dart Test配置，然后单击要么 。

配置语法突出显示

您可以根据自己的喜好和习惯配置Dart-aware语法突出显示。

  1. 在“ 设置/首选项”对话框中（ Ctrl+Alt+S ），转到“ 编辑器” |配色方案|飞镖

  2. 选择配色方案，接受从默认设置继承的突出显示设置，或按照配置颜色和字体中的说明自定义它们。
