从堆栈跟踪导航到异常

工具|浏览堆栈跟踪
Ctrl+Shift+E

当您收到外部堆栈跟踪（例如，来自错误报告）时，可以在“ 堆栈跟踪资源管理器”窗口中将其打开，并导航到相应异常起源的代码。在堆栈跟踪中，文件，类型和方法显示为超链接。您可以单击它们以在编辑器中显示相应的项目。

要返回以前打开的堆栈跟踪（在关闭解决方案之前一直可用），请选择ReSharper | BackSharper。 Windows |菜单中的堆栈跟踪 。

导航到导致异常的代码

  1. 将异常堆栈跟踪复制到剪贴板。

  2. 在主菜单中，选择“ 工具” |“工具”。浏览堆栈跟踪 ，或按Ctrl+Shift+E 。将打开“ 堆栈跟踪资源管理器”窗口 ，在新选项卡中显示剪贴板中的异常堆栈跟踪。

  3. 在“ 堆栈跟踪浏览器”窗口中，单击突出显示的项目以在编辑器中显示相应的代码。

  4. 如果您希望将长行包裹在堆栈跟踪中，请使用相应的工具栏按钮

此功能的灵感来自JetBrains ReSharper （Microsoft Visual Studio的开发人员生产力工具）。
