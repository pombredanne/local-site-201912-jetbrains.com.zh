从NuGet包中打开装配体

dotPeek提供了多种方法来浏览其中包含的NuGet程序包和程序集：

从磁盘

您可以使用“ 文件” |“磁盘”打开任何NuGet软件包。打开命令或单击在Assembly Explorer窗口的工具栏上，然后选择一个或几个.nupkg文件。

从软件包配置文件

您可以使用“ 文件” | “ 文件” |“加载”在Visual Studio项目中的packages.config文件中列出的包 。打开NuGetpackages.config命令。

从包源

您可以使用任何本地或远程软件包源从NuGet软件包加载程序集。

从NuGet包源加载程序集

  1. 请点击在Assembly Explorer窗口的工具栏上，或选择File |从主菜单中的NuGet打开 。这将打开“ 从NuGet打开”对话框。

  2. 指定软件包来源。这可以是磁盘路径或URL，dotPeek将在其中查找NuGet软件包。

  3. 指定软件包过滤器。 dotPeek将使用此字段显示名称中仅包含指定子字符串的软件包。

  4. 使用复选框定义是否要查找最新/全部和发行/预发行软件包版本。

  5. 如果指定的程序包源包含与指定的过滤器匹配的任何程序包，您将在列表中看到这些程序包。

  6. 选择所需的软件包，然后单击“ 打开”以将软件包及其包含的所有部件加载到“部件浏览器”窗口中 。（可选）选择“ 加载依赖项”以加载所有依赖的程序包和程序集。

从本地包缓存

如果要从最近使用的任何NuGet软件包中浏览程序集，dotPeek将帮助您从NuGet全局缓存文件夹％USERPROFILE％\。nuget \ packages \中快速查找并打开它们。

从本地NuGet程序包缓存加载程序集

  1. 请点击在Assembly Explorer窗口的工具栏上，或选择File |从主菜单中的NuGet程序包缓存中打开 。这将打开“ 从NuGet程序包缓存中打开”对话框。

  2. 该对话框列出了计算机上所有NuGet缓存位置的软件包。使用对话框中的搜索字段查找所需的软件包。

  3. 选择所需的包装（您可以Ctrl单击以选择多个程序包），然后单击确定以将程序包及其中的所有程序集加载到“程序集资源管理器”窗口中 。

如果您想随时清除反编译缓存，则可以删除％LOCALAPPDATA％\ JetBrains \ Shared \ vAny \ DecompilerCache中的数据 。
