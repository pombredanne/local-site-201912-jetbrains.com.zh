转到实施

导航|转到实施
Ctrl+Shift+Alt+B

通过此命令，您可以从基本类型或成员跳到其任何最终实现，而绕过继承链中的中间步骤。换句话说，它的工作方式类似于“ 转到派生符号”，但不显示抽象类和接口。

除了导航之外，您还可以直观地浏览符号的完整继承层次。

您可以从代码查看器，“ 文件结构”窗口和其他工具窗口调用此命令。如有必要，可以使用“ 转到基本符号”命令沿相反方向导航。

导航到类型或类型成员的实现

  1. 将插入号放置在代码查看器中的符号上，或在工具窗口中选择该符号。

  2. 选择导航|在主菜单中转到实施 ，然后按Ctrl+Shift+Alt+B ，或单击符号保持Ctrl+Alt键。

  3. 如果该符号只有一个实现，则dotPeek将直接导航到它。

  4. 如果该符号具有多个实现，则将在下拉列表中列出它们。 该列表可能包含粗体和常规字体的项目。粗体表示项目具有直接继承，而常规字体用于间接继承。

    您可以执行以下操作之一：

    • 单击所需的项目以在编辑器中将其打开。

    • 通过按选择所需的项目Up和Down然后按Enter在编辑器中打开它。

    • 开始输入以过滤结果。请注意，您可以在此处使用CamelHumps 。如有必要，请按Esc一次清除过滤。

    • 要在“ 查找结果”窗口中查看和分析匹配项的列表， 单击在查找结果中显示 ， 按Shift+Enter要么+在数字键盘上。

此命令在“ 导航到”快捷方式中也可用。

dotPeek根据dotPeek选项的Decompiler页面的Navigation部分中定义的设置导航到目标符号。有关更多信息，请参见浏览编译代码 。
