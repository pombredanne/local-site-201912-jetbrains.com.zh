dotPeek 2019.2帮助

导航和搜索

dotPeek使您可以使用声明，成员，方法调用，继承层次结构等来查找源代码。

导航和搜索范围包括当前在“ 程序集资源管理器”窗口中加载的所有程序集以及从这些程序集引用的所有程序集。请参阅浏览编译的代码以了解如何在编译的程序集中执行导航。

您可以从代码查看器中的任何位置，从“ 文件结构”窗口 ，“ 查找结果”窗口以及其他dotPeek工具窗口开始导航。

导航和搜索命令可在带有键盘快捷键的“ 导航”菜单中以及代码查看器和其他窗口的上下文菜单中使用。

所有导航和搜索功能均受JetBrains ReSharper （为Microsoft Visual Studio开发的生产力工具）的启发和借鉴。

根据您的任务，不同组的dotPeek导航和搜索功能可能会帮助您：

  • 如果需要按名称在解决方案或引用的库中查找任何内容，请研究“ 按名称搜索”部分中的命令。

  • 如果需要转到与当前代码符号相关的任何项目，请研究“ 从符号导航”部分中的命令。

  • dotPeek还提供了通用的“ 导航至”快捷方式，该快捷方式将当前插入符号位置中的所有可用导航选项分组。要了解有关此快捷方式的更多信息，请参见导航到部分。

  • 如果要在当前文件，项目，解决方案或引用的库中查找特定符号的用法，请研究“ 查找用法”部分中的命令。

  • 根据编辑器中当前插入符号的位置，dotPeek可以建议许多相关的导航选项。要了解有关这些选项的更多信息，请参见浏览当前上下文 。

  • 如果您需要在当前文档中查找任何文本，请使用“ 快速查找”窗口 。

  • 转到Line命令，该命令将您带到当前文档中的特定行。

    有关更多信息，请参见转到基本符号和转到派生符号 。

  • 代表当前文件结构的文件结构窗口保留导航功能。有关更多信息，请参见浏览文件结构 。

  • 堆栈跟踪资源管理器窗口 ，使您能够从堆栈跟踪导航到源代码中的错误位置，从而查明故障。有关更多信息，请参见从堆栈跟踪导航到异常 。
