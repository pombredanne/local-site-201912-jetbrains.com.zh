Go to Type of Symbol

Navigate | Type of Symbol
Ctrl+Shift+T

While the Go to Declaration command navigates you to the place where a symbol is declared, the Type of Symbol command navigates to the declaration of the type of the currently selected symbol. For example, if you have a variable Foo myVar = new Foo(); and invoke this command on a usage of myVar, it will bring you to the declaration of the class Foo. If you invoke this command on a function, it navigates to the function's return type.

For variables of generic types, this command allows you to navigate to the generic type declaration as well as to declarations of all generic parameters. For example, suppose you have a type class MyGeneric<X, Y> and a variable of this type var myGeneric = new MyGeneric<Author, Book>();. Invoking this command on any usage of myGeneric, you will be able to choose the navigation destination between MyGeneric, Author, and Book classes.

You can invoke this command from the code viewer, from the File Structure window and other tool windows.

To navigate to the type of a symbol

  1. Place the caret at a symbol in the code viewer or select the symbol in a tool window.

  2. Press Ctrl+Shift+T or choose Navigate | Type of Symbol %menuPath% from the main menu. Alternatively, you can press Ctrl+Shift+A, start typing the command name in the popup, and then choose it there.

  3. If the command returns a single result, dotPeek navigates directly there. In case of several declarations (for example, in a partial class), they will be listed in a drop-down where you can do one of the following:

    • Click the desired item to open it in the editor.

    • Select the desired item by pressing Up and Down and then press Enter to open it in the editor.

    • To view and analyze the list of matched items in the Find Results window, click Show in Find Results , press Shift+Enter or + on the numeric keypad.

This command is also available in the Navigate To shortcut.

dotPeek navigates to target symbols according to settings defined in the Navigation section on the Decompiler page of dotPeek options. For more information, see Navigate Compiled Code.
