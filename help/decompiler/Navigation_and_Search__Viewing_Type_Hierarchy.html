探索类型层次

检查层次结构
Ctrl+Alt+H

使用dotPeek，您可以研究类型的继承层次结构。您可以同时看到基本类型和所选类型的继承者，并通过单击鼠标导航到它们中的任何一个。对于层次结构中的任何节点，您都可以在预览窗格中查看所有或仅多态成员。

调查等级

  1. 在代码查看器中找到所需的类型，或在Assembly Explorer中将其展开。

  2. 按Ctrl+Alt+H或选择 ReSharper |检查层次结构 检查|主菜单中的层次结构 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  3. 检查添加到“ 层次结构”窗口的专用选项卡中的类型层次结构 。

树中的某些符号以粗体显示。这意味着它们直接从要检查的类型继承。

层次结构可能非常复杂，很难找到一个特定的符号，尤其是当您对该符号的名称和在层次结构中的位置有疑问时。在这种情况下，您可以开始输入符号名称或其CamelHumps缩写的一部分，而dotPeek可以缩小窗口中元素的范围：

如果要查看层次结构中哪些模块包含符号，请在“ 分组依据”列表中选择“ 包含模块层次结构 ”。

如果关闭了“ 层次结构”窗口，则可以随时选择“ Windows | Windows | Windows”来查看和检查以前生成的层次结构。主菜单中的层次结构 。
但是，此命令不会生成任何新的层次结构，并且如果由于当前解决方案已打开而尚未生成任何层次结构，则该窗口可能为空。

您还可以通过单击在图上显示 ，来可视化类型依赖图上的当前层次结构在工具栏上。

为中间节点重建层次结构

  1. 在层次结构树视图中，选择所需的节点。

  2. 单击基于此 在工具栏上，或在所选内容的上下文菜单中选择相应的命令。

预览层次结构中选定类型的成员

  1. 在层次结构中选择一种类型。

  2. 单击显示成员 在工具栏上。成员预览窗格显示在窗口底部：

  3. 您可以通过单击“ 显示成员”旁边的向下箭头将预览窗格移至窗口右侧。 在工具栏上，选择权 。

  4. 您也可以预览类型的所有成员，也可以仅预览多态的成员。要在这两种模式之间切换，请使用“ 所有实例成员” 并且只有多态成员 预览窗格顶部的按钮。

此功能的灵感来自JetBrains ReSharper （Microsoft Visual Studio的开发人员生产力工具）。
