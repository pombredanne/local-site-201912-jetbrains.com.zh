装配体浏览器窗口

Windows |装配浏览器
Ctrl+Alt+L

装配浏览器 允许打开多个程序集并保留打开的程序集和文件夹的列表 。您可以遍历和浏览所有打开的装配。扩展程序集节点将列出程序集内的名称空间，可以将其进一步扩展为类型和类型成员以及程序集引用。您可以双击任何类型以对其进行反编译或从符号服务器获取源（取决于您的设置 ）。
另外，Assembly Explorer包含代表基本类型的节点和当前类型的继承者。

要在Assembly Explorer中过滤项目，请使用顶部的搜索框。请注意，您可以在此处使用CamelHumps和元数据令牌 。

dotPeek在Assembly Explorer以及Code Viewer和其他工具窗口中还提供了许多ReSharper启发的导航功能 。还请注意，Assembly Explorer使用与Visual Studio中用于标识成员相同的图标集。

项目

描述

打开

从项目文件，库文件或ZIP存档中打开装配。有关更多信息，请参见打开和关闭装配体 。

浏览文件夹

允许选择要递归浏览的文件夹并加载在此文件夹中找到的所有部件。有关更多信息，请参见在文件夹中查找程序集 。

由于dotPeek会递归处理所选文件夹，因此请确保不要添加可能包含太多程序集的文件夹（例如Program Files ）。在dotPeek中加载所有程序集可能需要很长时间。

从GAC开放

允许从全局程序集缓存 （GAC）中打开一个或多个程序集。有关更多信息，请参见从全局程序集缓存中打开程序集 。

从NuGet包缓存中打开

帮助您从本地NuGet包缓存中查找并打开所需的NuGet包。

从NuGet打开

帮助您从指定的NuGet包源 （nuget.org或自定义）中查找并打开所需的NuGet包。

从运行过程中打开

打开“ 进程资源管理器”窗口以浏览当前正在运行的进程的程序集 。

打开装配清单

打开使用“ 保存装配体列表”命令保存的装配体列表 。有关更多信息，请参见打开程序集列表。

保存装配体列表

保存当前在装配浏览器中打开的装配列表。有关更多信息，请参见保存当前部件列表。

程序集列表另存为本地资源的链接，因此可能无法在其他计算机上正确打开。

清除装配体浏览器

从程序集浏览器中删除所有程序集。

导出到项目

将选定的程序集导出到Visual Studio项目。有关更多信息，请参见将装配输出到项目 。

生成PDB

为选定的程序集生成符号文件（PDB和源文件）。有关更多信息，请参见生成PDB文件 。
