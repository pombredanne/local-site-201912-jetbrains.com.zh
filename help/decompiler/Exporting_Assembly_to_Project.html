将装配导出到项目

探索程序集的最终步骤是将其导出到Visual Studio项目中，以便您可以浏览源代码，构建和调试它。

与程序集中使用的原始编程语言无关，导出的项目使用C＃创建。

要将程序集导出到Visual Studio项目

  1. 在“ 程序集资源管理器”窗口中选择一个程序集，然后单击“ 导出到项目” 在工具栏上，或在上下文菜单中选择“ 导出到项目 ”。

  2. 在出现的“ 导出到项目”对话框中，指定导出项目的目标文件夹。在此文件夹中，向导将为您的项目创建一个子文件夹。

    如果[DestinationFolder] / [ProjectSubfolder]不为空，则dotPeek会检测到它并建议覆盖现有文件，在导出之前清理文件夹或选择另一个文件夹。

  3. 如有必要，更改项目名称。

  4. 如果要为项目创建解决方案，请勾选创建* .sln文件 。
    Visual Studio对创建的解决方案文件的兼容性取决于程序集中使用的.NET Framework版本。例如，如果程序集使用.NET Framework 4.5，则将创建Visual Studio 2012解决方案。

  5. 勾选创建* .pdb文件以创建项目的调试信息文件。

  6. 如果要将反编译的项目添加到Assembly Explorer窗口并研究项目结构，文件等，请勾选将项目添加到Assembly Explorer 。

  7. 如果要立即打开项目，请在Visual Studio中打勾“ 打开项目” 。

  8. 单击导出以完成。您可以在“ 项目/ PDB生成状态”窗口中查看导出程序集的进度。
