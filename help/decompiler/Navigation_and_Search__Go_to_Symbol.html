转到符号

导航|转到符号...
Ctrl+Shift+Alt+N

此命令允许按名称搜索其中的任何代码符号当前在“ 程序集资源管理器”窗口中加载的所有程序集以及从这些程序集引用的所有程序集。

请注意，此命令在全局范围内起作用，也就是说，您无需将重点放在代码查看器上即可调用它。

导航到符号

  1. 按Ctrl+Shift+Alt+N或选择 导航|从主菜单转到Symbol ... ％menuPath％ 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  2. 在出现的输入符号名称弹出窗口中，开始输入项目名称。键入时，查找列表将变窄，并建议与输入的子字符串匹配的名称。

  3. 请执行以下任一操作：

    • 单击所需的项目以在编辑器中将其打开。

    • 通过按选择所需的项目Up和Down然后按Enter在编辑器中打开它。

    • 要在“ 查找结果”窗口中查看和分析匹配项的列表， 单击在查找结果中显示 ， 按Shift+Enter要么+在数字键盘上。

dotPeek根据dotPeek选项的Decompiler页面的Navigation部分中定义的设置导航到目标符号。有关更多信息，请参见浏览编译代码 。

此处支持不区分大小写的CamelHumps ：例如，您可以键入“ ch”而不是“ ColorHelper”。如果要导航到特定类型的符号，则可以指定该类型的标准名称和成员名称的一部分，并用空格或点将它们分开。

您也可以在指定名称时使用通配符：“ *”（星号）表示零个或多个字符； '+'（加号），'：'（分号），'。' （点）和'\'（反斜杠）将类型的全限定名分开。

每当收集项目以匹配输入的字符串花费大量时间时，dotPeek就会显示一个沙漏图标在搜索字符串的右边。在dotPeek完全加载解决方案之前执行搜索时，预计会有很大的延迟。

此功能的灵感来自JetBrains ReSharper （Microsoft Visual Studio的开发人员生产力工具）。
