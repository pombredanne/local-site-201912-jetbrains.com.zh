随处搜寻/输入

导航|搜索无处不在/键入...
Ctrl+N

此命令结合了两个不同的功能：

  • 首先出现的Search Everywhere允许您导航到以下目标：类型，符号，文件，源文件和文本文件中的字符串文字，最近的文件， 元数据标记和最近查看的方法。建议列表将在您调用此功能后立即出现，并且最初包括您的近期文件和导航的项目。

  • 转到类型 ，这是您第二次按Ctrl+N或从菜单调用命令，使您可以导航到任何类型。

使用这些功能，您可以在当前在“ 程序集资源管理器”窗口中加载的所有程序集以及从这些程序集引用的所有程序集。

请注意，此命令在全局范围内起作用，也就是说，您无需将重点放在代码查看器上即可调用它。

在各处搜索或查找类型

  1. 按Ctrl+N或选择 导航|在任何位置搜索/转到主菜单中的Type… ％menuPath％ 。或者，您可以按Ctrl+Shift+A ，开始在弹出窗口中键入命令名称，然后在此处选择它。

  2. 在出现的“ 搜索各处”列表中，开始输入项目名称。键入时，项目列表会变窄，并建议与输入的子字符串匹配的名称。

  3. 要进一步缩小项目列表，您可以输入/应用过滤器。例如，您可以输入/me仅显示事件。请注意，您可以在搜索查询之前和之后输入过滤器。

  4. 要切换到“ 转到类型”功能，请单击菜单命令或再次按快捷键。将显示“ 输入类型名称”列表，您可以在其中指定要查找的类型。

  5. 请执行以下任一操作：

    • 单击所需的项目以在编辑器中将其打开。

    • 通过按选择所需的项目Up和Down然后按Enter在编辑器中打开它。

    • 要在“ 查找结果”窗口中查看和分析匹配项的列表， 单击在查找结果中显示 ， 按Shift+Enter要么+在数字键盘上。

dotPeek根据dotPeek选项的Decompiler页面的Navigation部分中定义的设置导航到目标符号。有关更多信息，请参见浏览编译代码 。

此处支持不区分大小写的CamelHumps ：例如，您可以键入“ ch”而不是“ ColorHelper”。如果要按全限定名导航到类型，则可以键入名称的一部分，并用空格或点将它们分开。

您也可以在指定名称时使用通配符：“ *”（星号）表示零个或多个字符； '+'（加号），'：'（分号），'。' （点）和'\'（反斜杠）将类型的完全限定名称分开。

每当收集项目以匹配输入的字符串花费大量时间时，dotPeek就会显示一个沙漏图标在搜索字符串的右边。在dotPeek完全加载解决方案之前执行搜索时，预计会有很大的延迟。

此功能的灵感来自JetBrains ReSharper （Microsoft Visual Studio的开发人员生产力工具）。
