dotPeek 2019.2帮助

dotPeek的新增功能

dotPeek 2019.2

  • IL Viewer中带有关键字描述的工具提示

  • 现在转到文件成员还会在当前文档中搜索文本匹配项。

  • Assembly Explorer窗口中的 Win32资源子树

  • 元数据树改进：非托管（win32）资源目录和条目（类型，语言）的描述，节标题的描述。

dotPeek 2019.1

  • 元数据树改进：

    • 从文档令牌节点（便携式pdb子树）导航到源。

    • 从可移植pdb子树根节点导航到pdb内容.xml 。

  • 导航到nuspec （在Assembly Explorer窗口中双击一个nuget节点）

  • “程序集资源管理器”窗口中 NuGet程序包的“依赖项”节点。

dotPeek 2018.3

  • 局部函数反编译（C＃7.0）

  • 模式匹配反编译：case子句中的const模式，类型模式，类型模式（支持谓词和丢弃）（C＃7.0）

  • 反编译System.Enum ， System.Delegate和非托管约束（C＃7.3）

  • 反编译fixed字段索引而无需固定（C＃7.3）

  • 将符号信息复制到剪贴板

dotPeek 2018.2

  • 现在，“ 到处搜索”可以成功解析从dotTrace或Visual Studio的“调用堆栈”窗口复制和粘贴的函数签名，以进行进一步搜索。

  • 改进了对接口，枚举以及没有方法实现的任何类型的导航。

dotPeek 2018.1

  • 对以下C＃构造的反编译支持：

    • 空条件（ ?.要么?[] ）运算子

    • 等待中catch / finally块

    • 索引器（字典）初始化器

    • 异步Main

  • 元数据tee改进：

    • 反编译器不支持的文件的元数据子树（标题/目录）。

    • 标头中改进了演示和新的归档描述。

    • 元数据表，元数据表行，堆项的绝对偏移量（这些偏移量显示在“属性”窗口中）。

  • “属性”窗口中的部件的文件描述（版本信息）。

dotPeek 2017.3

  • 从本地NuGet程序包缓存加载程序集。

  • 在可读的XML文件中浏览程序集PDB的内容。

  • Windows PDB中的Source Link支持（以前仅适用于Portable PDB）。

dotPeek 2017.2

  • 如果任何导航和搜索命令都会在一个方法中导致多个匹配项，dotPeek现在将所有这些显示在搜索结果中。

  • 随处搜寻的结果Ctrl+N现在在字符串中包括搜索查询的出现。

  • 转到字符串的改进Ctrl+Alt+T ：搜索属性，更好的表示形式并直接导航到长行和多行字符串中的搜索查询。

  • 元数据树中可移植PDB的更多功能。

  • 有能力检索程序集的原始源代码，该源代码在可移植PDB中使用SourceLink引用。

  • 正确反编译nameof() 。

  • 对IL代码的显示和导航的改进和修复。

dotPeek 2017.1

  • 能够浏览便携式PDB元数据 。

  • 便携式PDB生成（ Symbol Server ， 生成PDB ）。

  • 支持嵌入到便携式PDB中的源 。

  • 随处搜索 。

dotPeek 2016.3

  • 一个新的Metadata 程序集资源管理器中的“节点”有助于浏览程序集元数据 。

dotPeek 2016.2

  • IL Viewer的改进：当您将尖号设置为代码符号或指令时，此项目的所有用法都会突出显示；代码中的循环可以通过相应的缩进和注释来区分。

  • 装配依赖图 。

dotPeek 2016.1

  • dotPeek会自动突出显示插入符号下方符号的用法。

  • 扩展方法在反编译代码中作为实例方法呈现。

  • 从任何包源加载NuGet包 。

dotPeek 10.0

  • 与反编译的C＃代码并排查看中间语言（IL） 。

  • 查找用法异步工作，使您可以在反编译器在后台执行耗时的用法搜索时继续使用它。

  • dotPeek支持与Visual Studio相同的颜色主题集，并且能够将颜色方案首选项与Visual Studio设置同步。

  • 转到声明命令的使用意识行为 。

dotPeek 1.4

  • 能够反编译Roslyn构建的程序集

  • 能够反编译以下C＃6.0功能：异常过滤器，带有初始化程序的自动属性

  • 能够在Assembly Explorer中显示元数据标记 ，并在反编译代码中显示为注释。

  • 能够导航到元数据令牌 。

  • 现在， 转到字符串命令使您可以在常量字符串字段和字符串参数的默认值中找到字符串文字。

dotPeek 1.3

  • 类型依赖图

  • 导航到“暴露API”命令，该命令使您可以从一种类型导航到当前加载的程序集中的所有位置，您可以在其中获取该类型的实例。

  • 转到字符串命令。

  • 转到“动作”命令，该命令可帮助您快速找到并应用dotPeek的任何动作。

  • 可以在“选项”对话框中搜索首选项

dotPeek 1.2

  • 能够为使用.NET程序集编译的.NET程序集生成PDB文件 。 /debug选项。

  • 使用dotPeek作为符号服务器

  • “进程资源管理器”窗口为您提供当前正在运行的进程的列表，并允许浏览其模块并反编译其中的.NET程序集。

  • 项目/ PDB生成状态窗口

  • 在Assembly Explorer窗口 ， Hierarchy窗口和Find Results窗口中搜索/过滤。

dotPeek 1.1

  • 反编译器改进：异步方法和lambda，系统。Linq。表达式，类似字段的事件

  • 能够将程序集导出到Visual Studio项目 。

  • 随处搜寻/输入
  • 书签
