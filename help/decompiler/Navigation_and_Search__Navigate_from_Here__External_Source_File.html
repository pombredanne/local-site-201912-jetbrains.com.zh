导航到符号文件中的源

导航|导航到|符号文件的来源
Ctrl+Shift+G | 符号文件的来源

该命令尝试获取库符号的原始源文件并在其中显示该符号。 在源代码和反编译源中的库符号上调用“ 导航至”时，该命令可用。 您可以从代码查看器，“ 文件结构”窗口和其他工具窗口调用此命令。

dotPeek从符号服务器下载所需文件可能需要一些时间。在这种情况下，您将看到下载进度。

要成功执行此命令，PDB信息应可用于目标程序集。 dotPeek使用两种方法来搜索PDB：

  • 它在目标程序集所在的同一目录中本地搜索PDB。

  • 如果无法在此处找到PDB，则使用_NT_SYMBOL_PATH环境变量。
    一些公司使用符号服务器（源服务器）发布部分源代码。例如，Microsoft使用此技术提供对.NET Framework源代码的访问。如果目标PDB的符号服务器可用，则此命令可以访问以这种方式存储的源。

如果PDB不可用，请使用“ 导航到元数据视图”命令来仅分析从元数据重构的符号的签名；否则，请执行以下操作：或“ 导航到反编译源”命令来分析符号的反编译主体。

为了使dotPeek知道可用的符号服务器，您需要在_NT_SYMBOL_PATH环境变量。 dotPeek将查找符号文件并从指定位置下载它们。

此功能的灵感来自JetBrains ReSharper （Microsoft Visual Studio的开发人员生产力工具）。
