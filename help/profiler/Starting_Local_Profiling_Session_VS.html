dotTrace 2019.2帮助

开始本地分析会话

Visual Studio中集成的dotTrace的主要使用场景是对当前开发的应用程序进行性能分析。为此，在此模式下，dotTrace提供了一些其他配置功能。更具体地说，它使您不仅可以分析特定的项目，还可以分析特定的“运行配置”。

*每个特定的静态方法都在特殊的JetLauncher流程下运行。

运行配置使您能够为启动项目，任意可执行文件甚至任何静态方法*创建许多具有不同运行参数的预定义配置。例如，您可以创建许多配置，具体取决于CPU体系结构和CLR版本。这些配置可用于快速运行或调试项目，而无需每次都设置特定的运行参数。由于dotTrace共享ReSharper平台，因此能够分析任何运行配置。

默认情况下，您的解决方案只有一个运行配置-启动项目的配置。

在这个部分：

  • 开始分析启动项目
  • 开始分析运行配置
  • 快速开始运行配置分析
  • 快速开始分析特定方法
  • 开始分析应用程序
  • 开始分析已经运行的进程

*您还可以通过Performance Profiler窗口运行启动项目的性能分析，该窗口可以通过ReSharper |简介|打开Performance Profiler窗口。

开始分析启动项目

  1. 请执行以下任一操作*：

    • 如果尚未创建任何运行配置，请在Visual Studio菜单中，选择“ ReSharper |简介|运行启动配置性能分析...。

    • 如果您有多个（默认）运行配置，请在Visual Studio菜单中，选择“ ReSharper |工具|运行配置...，然后选择[project_name]（VS启动）配置。然后选择ReSharper |简介|运行启动配置性能分析...。

    这将打开“ 概要文件启动项目”窗口。

  2. 在“ 配置文件启动项目”窗口中，按照配置配置文件会话中所述配置配置文件设置。

  3. 点击运行 。

开始分析特定的运行配置

  1. 为您的项目，可执行文件或特定的静态方法创建运行配置。为此，在Visual Studio菜单中，选择“ ReSharper |工具|运行配置...|添加 ，选择相应的项目。这将打开带有运行配置选项的窗口。有关它们的更多详细信息，请参阅ReSharper文档 。

  2. 使用Visual Studio菜单选择创建的运行配置。工具|运行配置... | [run_config_name] |配置|选择 。

  3. 在Visual Studio菜单中，选择“ ReSharper”。简介|运行启动配置性能分析...。这将打开“ 概要文件启动项目”窗口。

  4. 在Profile Startup Project窗口中，按照Profiler Options中所述配置配置设置。

  5. 点击运行 。

您也可以使用Performance Profiler窗口快速开始对特定运行配置进行性能分析。

快速开始对特定运行配置进行性能分析

  1. 为您的项目，可执行文件或特定的静态方法创建运行配置。为此，在Visual Studio菜单中，选择“ ReSharper |工具|运行配置...|添加 ，选择相应的项目。这将打开带有运行配置选项的窗口。
    有关它们的更多详细信息，请参阅ReSharper文档 。

  2. 使用Visual Studio菜单ReSharper | Performance Manager打开窗口。简介|打开Performance Profiler 。

  3. 在Performance Profiler窗口中， 列表中，选择分析类型。

  4. 在“ 性能分析器”窗口的运行配置列表中，选择创建的配置。

  5. 请点击 。

请注意，可以使用特定静态方法的操作列表来开始对静态方法的运行配置进行性能分析。

快速开始分析静态方法

  1. 通过将光标放在方法上并按来打开方法的动作列表Alt+Enter 。

  2. 在列表中，选择“ 调试方法”。配置文件（[profiling_type]）方法 。

    请注意，此会话将使用“ 性能分析器”窗口中当前选择的分析类型。

  3. 在Profile Startup Project窗口中，按照Profiler Options中所述配置配置设置。

  4. 点击运行 。

如果要从Visual Studio（而不是特定的运行配置）分析任意应用程序，则具有这种功能。在这种情况下，启动性能分析会话看起来与dotTrace Standalone中的几乎相同。

开始分析应用程序

  1. 在Visual Studio菜单中，选择“ ReSharper”。简介|运行应用程序性能分析...。这将打开“ 配置文件应用程序”窗口。

  2. 在“ 配置文件应用程序”窗口的左侧面板中，选择要配置文件的应用程序类型。

  3. 在右侧面板中，按照配置性能分析会话中所述配置性能分析设置。

  4. 点击运行 。

开始对已经运行的进程进行概要分析

  1. 在Visual Studio菜单中，选择“ ReSharper”。简介|运行应用程序性能分析...。这将打开“ 配置文件应用程序”窗口。

  2. 在“ 附加到进程”中的“ 概要文件应用程序”窗口的左面板中，选择要概要分析的.NET进程。仅显示在.NET Framework 4.0或更高版本上运行的进程，并且可以附加这些进程。
    如果缺少您要查找的进程，请单击“ 刷新”以更新进程列表，和/或单击“ 显示所有进程”以授予dotTrace管理权限（这将显示系统中正在运行的更多进程）。

    无法附加到通用Windows平台（UWP）应用程序。

  3. 在右侧面板中，按照配置性能分析会话中所述配置性能分析设置。

    请注意，将探查器附加到进程时，不支持跟踪和逐行分析类型。

  4. 点击运行 。
