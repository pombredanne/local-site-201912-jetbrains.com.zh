.NET内存分配

在“ 事件”筛选器中选择“ .NET内存分配”后 ，时间轴查看器将向您显示应用程序如何将内存分配给托管堆。例如，“ 呼叫树”显示特定呼叫已分配了多少内存（以MB为单位）。

选择.NET内存分配事件，以分析过多的内存流量和垃圾回收。后者是UI冻结的主要原因之一，因为可以在GC期间挂起主线程。

请注意，分配点事件仅在分配的内存大小总计超过100 KB后才会引发。例如，一个线程在性能分析期间分配了五个50 KB的内存块。在这种情况下，dotTrace将仅检测100 KB的两个分配。

当您选择.NET内存分配主题时，时间线查看器会将辅助.NET内存分配：堆过滤器添加到“ 过滤器”窗口中。

.NET内存分配：堆

.NET内存分配：堆子过滤器用于显示大小对象堆之间分配的内存分配。有关分配给LOH的信息，您可以评估应用程序如何使用大数据集合。这对于性能分析可能很重要，因为LOH不会自动压缩，并且会随着时间的推移而变得支离破碎。

作为过滤器， .NET内存分配：堆可用于过滤出发生在分配给特定堆的点事件。例如，如果要确定在LOH中分配内存的方法，请在“ 事件”过滤器中选择“ .NET内存分配” ，然后在“ .NET内存分配：堆”中选择“ 大对象 堆” 。这些方法将显示在“ 方法和子系统”中的“ 调用堆栈”窗口中。

堆名称： 小对象堆或大对象堆 。

堆中分配的对象的大小。

堆中分配的对象大小相对于分配的总内存的百分比。

例如，您的应用程序有两个线程：线程1和线程2。在配置过程中，线程＃1为SOH分配了8 MB，为LOH分配了1 MB。线程＃2为LOH分配了1 MB。这两个线程的事件过滤器如下所示：

若要应用.NET内存分配：堆过滤器

在过滤器中选择所需的堆。

选择堆后，其他过滤器将仅显示分配给所选堆的点事件。例如，“ 调用堆栈”窗口中的“ 顶部方法 ”将显示在堆中分配对象的方法列表。

.NET内存分配：类型

.NET内存分配：类型子过滤器用于显示某种类型的对象之间分配的内存。此信息可以帮助您确定在选定时间范围内分配的对象的类型。

作为筛选器， .NET内存分配：类型可用于筛选出分配了某种类型的对象的点事件。例如，如果要确定分配字符串的方法，请在“ 事件”过滤器中选择“ .NET内存分配” ，然后System.String在.NET内存分配中：类型 。这些方法将显示在“ 方法和子系统”中的“ 调用堆栈”窗口中。

输入名称。

对象的大小。

某种类型的对象的大小相对于已分配的总内存的百分比。
