未发布的本机分配

为了能够分析本机内存分配，您应该在高级分析会话选项中选择收集本机分配 。

在“ 事件”过滤器中选择了“ 未发布的本机分配”时 ，“时间线查看器”将显示您的应用程序如何将内存分配给本机（非托管）堆。例如，“ 呼叫树”显示特定呼叫已分配了多少内存（以MB为单位）。

选择“ 未发布的本机分配”事件以分析与本机内存有关的所有问题：潜在的内存泄漏，托管代码使用的非托管组件的问题等等。

请注意，与仅在分配的内存大小总计超过100 KB时才引发的托管内存分配事件不同，无论分配的内存大小如何，每次发生分配时都会引发“本机”内存分配事件。

重要！当前，dotTrace仅显示在获取快照时尚未释放的本机对象的分配。因此，您不能使用此筛选器来分析过多的本机内存分配（本机内存流量）。
