Tasks

The Tasks event indicates time intervals where tasks started via the Task API (e.g., Task.StartNew or Task.ContinueWith) are executed. Use the Tasks filter to analyze the code behind tasks.

Note that if you use the async/await syntactic sugar to run tasks, the applied Tasks filter will leave only the code behind the Task execution node (to be more correct, time intervals when this code was executed). The async method itself including the continuation code is out of the filter's scope. For more information about how to analyze asynchronous code, refer to Analyzing Async Calls.
