Profile Unity应用程序

本节中的说明仅适用于Windows。如果要在macOS或Linux上分析Unity应用程序，则应使用JetBrains Rider中内置的dotTrace。

概要分析独立的Unity应用程序（Unity 2018.3及更高版本）

  1. 将dotTrace作为独立的应用程序运行 。dotTrace 主页窗口将打开。

  2. 在左侧面板中，选择Profile Profile App 。

  3. 在Profile Application的中间面板中，选择Unity Application 。

  4. 在右侧面板的Unity Application中 ：

    • 在“ 应用程序”中 ，指定要分析的应用程序可执行文件的路径。

    • （可选）如果您的应用程序需要命令行参数，请选择Advanced并指定Arguments 。

    • （可选）如果您的应用程序创建了多个进程，并且您想对特定进程进行概要分析，请选择高级和概要文件子进程 。如独立应用程序部分中所述添加流程过滤器。

    有关独立应用程序选项的更多详细信息，请参阅“ 独立应用程序”部分。

  5. 在“ 探查器选项”中 ，指定分析类型。当前，您只能使用时间轴概要分析类型（确定最慢的方法以及方法调用的时间分布方式）。
    有关探查器选项的更多详细信息，请参阅探查器选项部分。

  6. （可选）如果要使用API（而不是Controller窗口）控制分析会话（进行快照，结束会话等），请选择Advanced（高级）并使用Profiler API 。有关如何使用API控制分析的更多详细信息，请参阅通过API控制分析会话 。

  7. 点击运行 。

  8. 收集分析数据（快照）。有关如何控制性能分析会话的更多详细信息，请参阅“控制性能分析会话” （如果您手动控制会话）或“ 通过API控制性能分析会话” （如果您使用API控制性能分析）。

  9. 收集数据后，关闭配置文件的应用程序或使用“ 控制器”窗口的“ 分离”按钮分离配置文件 。如果使用API控制配置文件，则必须由您的代码执行分离。

  10. 在时间轴查看器中分析收集的快照。
