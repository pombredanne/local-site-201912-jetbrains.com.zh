dotTrace 2019.2帮助

配置源文件的路径

您可以明确指定源文件的路径，以确保dotTrace在正确的位置查找文件。 dotTrace可以在本地磁盘或网络共享上查找源。因此，您可以指定本地路径或UNC路径。

添加新的文件夹替换

  1. 在菜单栏上，选择“ 查看” |“选择”。选项 。将打开“ 选项”对话框。

  2. 在左窗格中选择“ 文件夹替换”节点。

  3. 在“ 文件夹替换”窗格中，单击“ 添加”以创建新的替换。将打开“ 编辑文件夹替换”对话框。

  4. 输入源文件夹的路径和源文件实际所在的目标文件夹的路径。单击确定 。

  5. 新的替代出现在表中。单击确定保存更改。

配置文件夹替换

  1. 在文件夹替换列表中选择一个项目。

  2. 请执行以下任一操作：

    • 单击编辑以更改文件夹替换。

    • 单击删除以从列表中删除选定的替换。

    • 单击上移或下移以更改替换顺序。

  3. 单击确定保存更改。

如果要忽略此替换，请在文件夹替换列表中取消选择一个项目。

指定源服务器的路径

  1. 在菜单栏上，选择“ 查看” |“选择”。选项 。将打开“ 选项”对话框。

  2. 在左窗格中选择“ 符号服务器集成”节点。

  3. 在“ 符号服务器集成”窗格中，单击选项之一以启用从特定服务器之一下载PDB和源文件：
    • Microsoft参考源服务器
    • 自定义源服务器

    对于最后一个选项，请键入源服务器地址，其中dotTrace应在其中查找PDB和源文件。

    否则，请单击“不使用符号服务器” 。

  4. 单击确定以应用更改。

或者，您可以指定到的路径.pdb或直接从Source View中获取源文件。

单击“ 浏览PDB”以在计算机上手动找到所需的文件。
