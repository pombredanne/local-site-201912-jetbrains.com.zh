Basic Operations with Snapshots

While you profile an application, dotTrace collects performance or concurrency data and writes it correspondingly to a *.dtp or *.dtt file. You may also supply the collected data with some contextual information: snapshot annotations, comments to functions, and so on. These contextual data is stored in a file with the *.dtp.DotSettings extension. A profiling snapshot is essentially a combination of these files.

Snapshots are saved to a default location which you may change anytime.

In case you want to copy a snapshot to another computer, you can save it as a single archive.
