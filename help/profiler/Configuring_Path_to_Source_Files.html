Configuring Path to Source Files

You can specify the path to source files explicitly to be sure that Source View looks for the files in the right place. As Source View can look for sources on local disks or network shares, you can specify a local path or UNC path.

To add a new folder substitution

  1. Select the menu item View | Options. The Options dialog opens.

  2. Select the Folder substitutions node in the left pane.

  3. In the Folder substitutions pane, click Add to create a new substitution. The Edit Folder Substitution dialog opens.

  4. Enter a path to the source folder and a path to the target folder where source files are actually located. Click OK.

  5. New substitution appears in the table. Click Save to save changes.

To configure folder substitutions

  1. Select an item in the list of folder substitutions.

  2. Do one of the following:

    • Click Edit to change the folder substitution.

    • Click Remove to remove the selected substitution from the list.

    • Click Move Up or Move Down to change the order of substitutions.

  3. Click Save to save changes.

Deselect an item in the list of folder substitutions if you want to omit this substitution.

To specify the path to a source server

  1. Select the menu item View | Options. The Options dialog opens.

  2. Select the Symbol server integration node in the left pane.

  3. In the Symbol server integration pane, click one of the options to enable downloading PDB and source files from one of the particular servers:
    • Microsoft Reference Source Server
    • Custom source server

    For the last option, type source server address where dotTrace should look for PDB and source files.

    Otherwise, click Don't use symbol server.

  4. Click Save to apply the changes.

Alternatively, you can specify the path to .pdb or source files right from Source View.

Click Browse for PDB to find the desired file on your computer manually.
