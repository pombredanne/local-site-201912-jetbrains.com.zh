编辑符号搜索策略

dotTrace主页窗口|新会议| [应用程序类型] | [逐行分析类型] |编辑符号搜索策略

要开始逐行分析应用程序，dotTrace需要存储在PDB文件中的符号信息。使用“ 编辑符号搜索策略”对话框，您可以显式指定文件的路径或指定文件应位于的位置。

项目

描述

搜索路径

输入PDB文件的路径。使用分号分隔符号路径。

查询注册表以查找符号搜索路径

选中此复选框以浏览注册表，并尝试查找PDB文件的路径。

访问符号服务器 选中此复选框以允许dotTrace访问符号服务器，查找PDB文件并下载它们。

dotTrace从_NT_SYMBOL_PATH和_NT_ALT_SYMBOL_PATH变量中指定的符号服务器中获取调试信息。

搜索在Debug文件夹中指定的路径

选择此复选框以在已分析应用程序的“调试”目录中查找PDB文件。

在应用程序启动文件夹中搜索PDB

选中此复选框可在配置文件应用程序的可执行文件所在的目录中查找PDB文件。
