将dotTrace与Typemock隔离器集成

dotTrace与Typemock隔离器集成，因此您不仅可以测试代码，还可以调查性能问题。

在开始之前，请确保已安装适当版本的工具：

  • JetBrains dotTrace 4.5.1或更高版本

  • Typem隔离器6.2.5

将dotTrace与Typemock隔离器集成

  1. 启动Typemock。Configuration.exe 。“ Typemock隔离器配置”对话框打开。

  2. 选中“ 仅显示可用的探查器”复选框以缩小概要分析和覆盖率工具的列表。

  3. 查找并单击适当版本的dotTrace。

  4. 单击“使用Typemock隔离器链接”，然后关闭对话框。

透明整合概念

透明的集成机制允许在dotTrace事件探查器内核之前运行任何第3方内核。第三方核心负责加载和配置dotTrace探查器核心。确保在进行概要分析之前已部署并注册dotTrace探查器核心。

为确保dotTrace和3rd party实用程序一起正常工作，请进行一些其他检查。

至少应存在以下注册表项之一：

  • HKEY_CURRENT_USER \ SOFTWARE \ JetBrains \ Profiler \ TransparentIntegration
  • HKEY_LOCAL_MACHINE \ SOFTWARE \ JetBrains \ Profiler \ TransparentIntegration（32位或64位）

并且应包含一个或多个值：

  • SamplingPerformanceProfiler = <3rd party CLSID>
  • TracingPerformanceProfiler = <3rd party CLSID>
  • TracingInjectPerformanceProfiler = <3rd party CLSID>
这些值均应为REG_SZ类型，并且不能为空。

这些位置之一应包含JETBRAINS_PROFILER_CORE_GUID ，以便第3方核心可以加载dotTrace事件探查器核心。

  • HKEY_CURRENT_USER \ SOFTWARE \ JetBrains \ Profiler \ Pid_XXX ，其中XXX是十进制进程ID

  • HKEY_LOCAL_MACHINE \ SOFTWARE \ JetBrains \ Profiler \ Pid_XXX ，其中XXX是十进制进程ID

  • 环境变量

  • HKEY_LOCAL_MACHINE \ SOFTWARE \ JetBrains \ Profiler \ Global
第三方实用程序遍历这些位置以搜索以下位置的顺序JETBRAINS_PROFILER_CORE_GUID非常重要。如果不遵循该顺序，则集成可能无法正常工作。
