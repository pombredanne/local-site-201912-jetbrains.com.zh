dotTrace 2019.2帮助

开始本地分析会话

dotTrace的起点是主窗口。在这里，您可以启动新的性能分析会话或打开最近会话中收集的快照。

开始分析应用程序

  1. 在左侧面板中选择“ 配置文件本地应用 ”。

  2. 在“ 配置文件应用程序”的中央面板中，选择要配置文件的应用程序类型。

  3. 在右侧面板中，按照配置性能分析会话中所述配置性能分析设置。

  4. 点击运行 。

开始对已经运行的应用程序进行性能分析

  1. 在左侧面板中选择“ 附加到正在运行的应用程序 ”。

  2. 请执行以下任一操作：

    • 在进程列表的中央面板中，选择要分析的.NET进程。
      如果缺少您要查找的进程，请单击“ 刷新”以更新进程列表，和/或单击“ 显示所有进程”以授予dotTrace管理权限（这将显示系统中正在运行的更多进程）。

    • 拖动附件图标到您要附加的应用程序上。

    1。仅显示在.NET Framework 4.0（或更高版本）或.NET Core 3.0 Preview 7（或更高版本）上运行的进程，并且可以附加这些进程。
    2。无法附加到通用Windows平台（UWP）应用程序。

  3. 在右侧面板中，按照配置性能分析会话中所述配置性能分析设置。请注意，将探查器附加到进程时，不支持跟踪和逐行分析类型。

  4. 点击运行 。
