WCF个人资料服务

概要分析WCF服务库

  1. 将dotTrace作为独立的应用程序运行 。dotTrace 主页窗口将打开。

  2. 在左侧面板中，选择Profile Profile App 。

  3. 在Profile Application的中间面板中，选择WCF服务 。

  4. 在右侧窗格中的WCF服务库中 ：

    • 在“ 库路径”中 ，指定配置文件服务库的完整路径。

    • 在“ 配置路径”中 ，指定库配置文件的完整路径。

    有关WCF服务选项的更多详细信息，请参阅WCF服务部分。

  5. 在“ 探查器选项”中 ，指定分析类型。通常，您应该在采样 （评估性能并确定应用程序中最慢的方法）和时间轴 （确定最慢的方法以及如何及时分配方法调用）之间进行选择。
    有关探查器选项的更多详细信息，请参阅探查器选项部分。

  6. （可选）如果要使用API（而不是Controller窗口）控制概要分析会话（进行快照，结束会话等），请选择Advanced（高级）并使用Profiler API 。有关如何使用API控制分析的更多详细信息，请参阅通过API控制分析会话 。

  7. 点击运行 。

  8. 收集分析数据（快照）。有关如何控制性能分析会话的更多详细信息，请参阅“控制性能分析会话” （如果您手动控制会话）或“ 通过API控制性能分析会话” （如果您使用API控制性能分析）。

  9. 收集数据后，关闭配置文件的应用程序或使用“ 控制器”窗口的“ 分离”按钮分离配置文件 。如果使用API控制配置文件，则必须由代码执行分离。

  10. 在Performance Viewer或Timeline Viewer中分析收集的快照（取决于您选择的配置文件类型）。
