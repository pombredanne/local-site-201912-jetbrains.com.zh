dotTrace 2019.2帮助

什么是新的

dotTrace 2019.2

  • Rider中的时间轴过滤器。
    Rider中的dotTrace变得更加成熟：现在，它获得了全套的时间轴过滤器，从而更易于分析就位的时间轴分析快照。

  • .NET Core 3.0完全支持。
    dotTrace现在可以附加到.NET Core 3.0 Preview 7进程。

  • 改进了对Visual Studio中ASP.NET Core应用程序的支持：

    • 正确支持不同的托管模式，

    • 启动概要分析会话时，会考虑到浏览器设置和“ 启用SSL” ，

    • 支持ASP.NET Core 3.0 Preview 7，

    • 和其他改进...

  • 分析API可以脱离概要分析过程。

  • 不再支持Silverlight和WebDev应用程序。

dotTrace 2019.1

  • Mono和Mono Unity应用程序支持：

    • 独立dotTrace：仅Windows。

    • dotTrace已集成到JetBrains Rider中：Windows，macOS和Linux。

  • 命令行探查器的改进：

    • 的start和attach命令现在接受高级配置参数，例如时间测量类型等。

    • 的Reporter.exe工具接受--save-signature允许您在最终报告中区分重载方法的参数。

  • 新的分析API 。我们已经对概要分析API进行了重大更新：

    • 我们摆脱了状态机。现在，无论控制器状态如何，您都可以调用任何API方法。

    • 我们减少了可用方法的数量，从而简化了API的使用。

    • 现在，我们以NuGet包的形式分发API。

dotTrace 2018.3

  • JetBrains骑士集成 。您可以配置和运行概要分析会话，获取快照并在内置查看器中对其进行分析。请注意，当前查看器无法在时间轴上显示事件的分布。这就是为什么将时间线快照作为简单采样快照打开的原因。

  • 改进的.NET内存分配过滤器 。时间轴查看器获得了一个新的子过滤器，使您可以分析分配的内存如何在某种类型的对象之间分配。

  • 重做的IIS Express配置文件设置 。现在，配置IIS Express上托管的Web应用程序的默认方法是为dotTrace提供applicationhost.config文件。

dotTrace 2018.2

  • 改进的性能和稳定性。

  • 包括用于.NET进程的筛选器 。在配置任意.NET进程的配置文件时，您可以设置一个包含过滤器：dotTrace将仅附加到与该过滤器匹配的进程。

dotTrace 2018.1

  • 改进的性能。dotTrace的此版本主要集中在提高稳定性和性能上。

dotTrace 2017.3

  • 支持异步调用 。异步代码的缺点是很难对其进行概要分析并分析其性能。 dotTrace 2017.3大大简化了异步代码的分析。它标记了所有async 呼叫树中的呼叫节点并将相应的分组await该节点下的时间和延续代码。这意味着您可以在一个地方快速找到异步调用的所有“部分”，而不必在不同的调用堆栈中进行搜索。

  • 时间轴查看器中的性能预测 。现在，您可以在时间轴查看器中“预测”应用程序性能。简单地说，从调用树中排除特定方法，dotTrace将重新计算整个快照，就好像没有这样的方法一样。

  • 方法和子系统中的回溯（时间轴查看器） 。现在，您可以直接在“ 方法和子系统”中快速查看特定方法的回溯， 而无需切换到“ 调用树” 。

  • 将用户程序集标记为系统程序集 。您可以将特定的用户程序集标记为系统用户程序集。之后，应用于系统模块中方法的所有规则也将应用于该用户程序集中的方法 ：例如，在“ 方法和子系统”以及系统调用折叠中计算方法时间的方式。

dotTrace 2017.2

  • 调用树中的导航路径 。在调用树上导航时，始终很难理解您是如何进入特定功能的。使用dotTrace 2017.2解决了该问题：“调用树”视图在左侧装订线中显示了所有转换。

  • 从命令行进行时间轴分析 。命令行探查器最终支持时间轴分析类型。

  • 在单元测试中分析子进程。
    现在，您可以通过一个简单的复选框在单元测试中启用子进程的性能分析。不再需要复杂的配置文件配置！

  • 清除缓存文件 。现在，您可以使用dotTrace Home首选项中的按钮强行删除快照缓存文件。这将导致现有快照的加载速度变慢，但会释放一些磁盘空间。

dotTrace 2017.1

  • Visual Studio 2017支持。

  • 通过拖放进行附加 。现在，您可以使用拖放将事件探查器附加到正在运行的应用程序。只需将一个特殊图标放到要分析的应用程序窗口上。

  • 从dotTrace Home打开快照文件 。现在，您可以直接从dotTrace主页窗口打开.dtp和.dtt快照文件。

dotTrace 2016.3

  • Visual Studio 2017 RC初始支持。

  • 时间线查看器中的子系统 。时间轴查看器具有Performance Performance Viewer的最大功能之一：子系统。子系统的机制非常简单：在大多数情况下，每个子系统仅将在特定名称空间或程序集中进行的调用分组。当您需要快速评估特定调用子树中的时间在各个组件之间的分配方式时，此功能非常有用：用户和系统代码，WPF，LINQ，集合，字符串等。

  • 分析本地内存分配 。本机内存分配事件过滤器使您可以查看进行分配的方法，并分析与本机内存有关的所有问题。

  • 分析应用程序的调试输出 。调试输出过滤器对于高级调试非常有用：选择时，时间轴将仅向您显示应用程序写入调试输出的事件，例如使用Debug.Write系统方法。

  • 事件窗口 。“ 事件”工具窗口仅向您显示事件日志-概要分析的应用程序中发生的事件的列表：文件操作，JIT，SQL查询等。

  • .NET Core应用程序的性能分析 。如果您的计算机上安装了.NET Core工具，则可以分析.NET Core应用程序而无需任何其他操作。

  • 呼叫堆栈窗口的多项改进 。

dotTrace 2016.2

  • 在时间轴查看器中折叠递归调用 。通过将递归调用折叠到一个节点中，可以简化对深度递归调用堆栈的分析。

dotTrace 2016.1

  • Visual Studio中的线程图。
    Visual Studio中集成的dotTrace Viewer与独立视图具有完全相同的“线程”图，因此，现在您可以使用它来选择线程，时间范围，当然，还可以查看线程活动和过滤的时间间隔。

  • 线程图中的智能工具提示 。将鼠标指针悬停在“线程”图中的任何线程上，工具提示将向您显示有关该线程在特定时间点正在执行的操作的详细信息：执行了什么方法，线程使用了哪个文件（如果是文件I /， O）等数据。

  • 将整个普通列表复制到Performance Viewer中的剪贴板 。现在，您可以将纯列表的全部内容复制到剪贴板。

  • Reporter.exe工具生成的报告中的已分析过程数据 。现在，用于从快照获取性能数据的Reporter.exe工具（例如，用于性能分析自动化和CI）将附加数据放入其XML报告中。如果您在启用状态下生成报告/add_process_info选项，该报告将包含已分析的流程的可执行文件的路径。在对具有多个子进程的应用程序进行性能分析时，此信息非常有用。
