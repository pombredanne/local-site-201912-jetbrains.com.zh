UI冻结

对于Mono和Unity应用程序，UI冻结不会显示，因为Mono框架不提供此类事件。

UI Freeze事件指示应用程序无法响应用户输入的时间间隔。更具体地说，这些时间间隔是窗口消息的泵送时间不超过200毫秒，或者特定消息的处理时间超过200毫秒。

由于以下原因，UI线程可能无法处理用户输入事件：

  • UI线程上过多的计算工作。

  • 阻止垃圾收集（也阻止UI线程）。

  • UI线程缺少空闲的处理器时间。

  • 文件I / O操作过多。

通常，Windows应用程序使用单个线程来处理用户界面。默认情况下，它是应用程序主线程。这就是为什么在应用UI Freeze筛选器之后，仅在UI线程上选择了相应的时间间隔的原因。

为了简化UI冻结分析，“ 流程概述”图以单独的条形显示冻结间隔。

选择“ UI冻结间隔”过滤器以过滤出UI冻结时间间隔并确定冻结的来源。例如，要检查UI冻结是否是阻止垃圾收集的结果，请在“ 线程”图中选择“主线程”，然后在“ 间隔过滤器”中选择“ UI冻结” 。然后检查“ 事件|事件”中显示的时间。垃圾收集过滤器。如果它很大，那么减少内存流量是有道理的，因为它肯定会影响UI响应能力。
