配置文件运行过程

剖析已经运行的进程

  1. 将dotTrace作为独立的应用程序运行 。dotTrace 主页窗口将打开。

  2. 在左侧面板中，选择“ 附加到正在运行的应用程序” 。

  3. 在中间面板中，选择要分析的.NET进程。仅显示在.NET Framework 4.0或更高版本上运行的进程，并且可以附加这些进程。

    如果缺少您要查找的进程，请单击“ 刷新”以更新进程列表，和/或单击“ 显示所有进程”以授予dotTrace管理权限（这将显示系统中正在运行的更多进程）。

  4. 在右侧面板的Profiler Options中 ，指定分析类型。将事件探查器附加到流程时，只能在采样 （评估性能并确定应用中最慢的方法）和时间轴 （确定最慢的方法以及如何及时分配方法）之间进行选择。
    有关探查器选项的更多详细信息，请参阅探查器选项部分。

  5. （可选）如果要使用API（而不是Controller窗口）控制概要分析会话（进行快照，结束会话等），请选择Advanced（高级）并使用Profiler API 。有关如何使用API控制分析的更多详细信息，请参阅通过API控制分析会话 。

  6. 点击运行 。

  7. 收集分析数据（快照）。有关如何控制性能分析会话的更多详细信息，请参阅“控制性能分析会话” （如果您手动控制会话）或“ 通过API控制性能分析会话” （如果您使用API控制性能分析）。

  8. 收集数据后，请使用“ 控制器”窗口的“ 分离”按钮从进程中分离探查器。如果使用API控制配置文件，则代码必须提供分离功能。

  9. 在Performance Viewer或Timeline Viewer中分析收集的快照（取决于您选择的配置文件类型）。
