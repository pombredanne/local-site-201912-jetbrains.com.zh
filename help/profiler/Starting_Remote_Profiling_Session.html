开始远程分析会话

您不仅可以在本地计算机上配置.NET应用程序，还可以在远程计算机上配置文件。此外，为了进行远程配置，您不必在远程计算机上安装dotTrace。但是，远程分析需要在远程计算机上执行许多准备步骤。

dotTrace使用基于HTTP的Web服务进行远程配置。这样可以避免大多数身份验证问题和防火墙问题。

在远程计算机上启动性能分析会话

  1. 在远程计算机上，执行以下操作：

    • 将RemoteAgent.zip归档文件从本地计算机上的dotTrace安装目录（默认为C：\ Users \ [用户名] \ AppData \ Local \ JetBrains \ Installations \ dotTrace [N] ） 复制到远程计算机上的任何目录。

    • 在远程计算机上解压缩存档。

    • 从远程计算机上的控制台启动Remote Agent。

    • 确保防火墙未阻止与远程代理的入站连接（默认情况下，它使用端口9100）。

  2. 在本地计算机上，将dotTrace作为独立的应用程序启动。

  3. 在左侧面板中选择“ Profile Remote ”。

  4. 在中央面板的“ 远程代理URL”中指定远程代理服务的地址 。
    默认情况下，URL为net.tcp：// [remote_computer_IP_address]：9100 / RemoteAgent 。但是，您可以使用远程代理的命令行选项更改服务的默认端口和虚拟目录。运行“ RemoteAgent.exe / help ”以获取更多详细信息。

    建立与远程服务的连接后，其URL将保存在“ 最近的连接”列表中，以便快速访问。

  5. 单击“ 连接”以建立与远程主机的连接。

  6. 在中央面板中，执行下列操作之一：

    • 如果要分析的应用程序，选择配置文件应用程序的应用类型。

    • 如果要分析已经运行的进程，请在“ 附加到进程”中选择一个特定的进程 。

  7. 在右侧面板中，按照配置性能分析会话中所述配置性能分析设置。

  8. 点击运行 。

dotTrace考虑了特权。因此，它仅适用于根据当前用户帐户的权限可以达到的那些进程。
如果缺少您要查找的进程，请以管理员身份重新启动Remote Agent。这将使您看到系统中正在运行的更多进程。
