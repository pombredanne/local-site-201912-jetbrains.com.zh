dotTrace 2019.2帮助

预览源代码

dotTrace会自动找到所选功能的基础源代码。要使源代码在dotTrace中可见，请使其知道在哪里.pdb找到项目的文件，或指定源代码的路径 。

请注意，如果您选择逐行分析类型 ，则“ 源代码视图”将显示有关特定代码行被调用多少次的更多信息。在Source View和Line-by-Line Profiling中了解更多信息。

查看源代码

在菜单栏上，选择“ 查看” |“选择”。源代码视图 。

如果在.NET Framework 2.0下配置了在.NET Framework 1.1下创建的应用程序，则无法查看源代码。

如果您需要对代码进行低级调试，或者出于其他原因检查编译器发出的代码，则可以在Source View中轻松地进行调试。

查看IL代码

  1. 选择一个功能。

  2. 开源视图 。

  3. 在“ 源代码视图 ”中选中“ 显示IL代码”复选框。

IL代码显示在每个代码语句下。

有时源代码现在暂时无法在此处使用，但是dotTrace可以使用内置的反编译器来重新创建接近原始代码的源代码。

查看反编译的代码

  1. 选择一个功能。

  2. 开源视图 。

  3. 在源代码视图中，单击反编译源 。

  4. 为了启用代码反编译，请执行以下操作之一：

    • 选中我接受许可协议的条款，允许反编译复选框，然后单击应用 。

    • 在菜单栏上，选择“ 查看” |“选择”。选项 。将打开“ 选项”对话框。在左窗格中选择“ 反编译器”节点。选中右侧窗格中的我接受许可协议的条款，允许反编译复选框，然后单击保存以应用更改。

  5. 立即显示相应的反编译代码。
