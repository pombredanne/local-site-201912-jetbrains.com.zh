剖析IIS Express中托管的Web应用程序

  1. 关闭所有正在运行的IIS Express实例。

  2. 将dotTrace作为独立的应用程序运行 。dotTrace 主页窗口将打开。

  3. 在左侧面板中，选择Profile Profile App 。

  4. 在Profile Application的中间面板中，选择IIS Express 。

  5. 在右侧面板中的IIS Express应用程序中 ：

    • 如果要配置的Web应用在默认的IIS Express配置文件中进行了描述applicationhost.config ，选择“ 配置文件” ，然后在“ 站点”中选择要配置的网站。

      请注意，配置文件的位置取决于您使用的IDE：

      • JetBrains骑士： （solutionDir）\。idea \ config \ applicationhost.config

      • Microsoft Visual Studio 2015和更高版本： （solutionDir）\。vs \ config \ applicationhost.config

      • Microsoft Visual Studio 2013和更早版本： ％USERPROFILE％Documents \ IISExpress \ config \ applicationhost.config

      如果您不想使用配置文件，则可以选择“ 应用程序路径”，并指定包含配置文件应用程序的目录的完整路径。

    • 如果要自动打开应用程序URL，则在分析开始后，选择“ 打开URL”并指定应用程序URL。使用浏览器中的参数可以选择必须在其中打开URL的浏览器。

    • （可选）如果选择“ 高级” ，则可以指定自定义IIS Express和运行时设置：

      • 在“ 服务器”中 ，选择一个IIS Express实例，

      • 在运行时版本 ：.NET Framework版本中，

      • 在Port（端口） ：HTTP端口中，如果希望dotTrace自动选择端口，则选择Use random port（使用随机端口）。

  6. 在“ 探查器选项”中 ，指定分析类型。通常，您应该在采样 （评估性能并确定应用程序中最慢的方法）和时间轴 （确定最慢的方法以及如何及时分配方法调用）之间进行选择。
    有关探查器选项的更多详细信息，请参阅探查器选项部分。

  7. （可选）如果要使用API（而不是Controller窗口）控制概要分析会话（进行快照，结束会话等），请选择Advanced（高级）并使用Profiler API 。有关如何使用API控制分析的更多详细信息，请参阅通过API控制分析会话 。

  8. 点击运行 。

  9. 收集分析数据（快照）。有关如何控制性能分析会话的更多详细信息，请参阅“控制性能分析会话” （如果您手动控制会话）或“ 通过API控制性能分析会话” （如果您使用API控制性能分析）。

  10. 收集数据后，关闭配置文件的应用程序或使用“ 控制器”窗口的“ 分离”按钮分离配置文件 。如果使用API控制配置文件，则必须由代码执行分离。

  11. 在Performance Viewer或Timeline Viewer中分析收集的快照（取决于您选择的配置文件类型）。
