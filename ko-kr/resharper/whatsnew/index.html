ReSharper의 새로운 기능

ReSharper Ultimate 2019.2에서는 C# 8.0에 대한 지원이 향상되고, Navigation, Find Usages 및 디버거 데이터에 대한 팁이 일부 업데이트되었습니다. 또한, 시작 시간이 개선되고 VS 2019의 “Per-Monitor DPI Awareness” 모드에 대한 초기 지원이 제공됩니다. C++ 프로젝트의 인덱싱이 더 빨라지고, 더 많은 C++ 20 기능이 지원됩니다.

향상된 C# 지원

ReSharper의 향상된 지원을 통해 C# 8을 최대한 활용하세요.

  • 색인 생성 및 범위 지정이 가능한 유형 등 색인 및 범위의 최근 변경 사항이 지원됩니다.
  • 컨텍스트 액션 Add range check가 이제 System.Index/System.Range 기준 인덱싱을 지원하고 정확한 검사를 수행합니다.
  • 관리되지 않은 제네릭 구조체를 지원합니다.
  • 두 개의 새로운 검사, ‘Convert switch statement to switch expression’ 및 ‘Convert chain of ternary expressions to switch expression’과 이와 연관된 ‘Convert to switch expression’ 빠른 수정.

여러분의 요청에 따라 컨텍스트 액션 Convert concatenation to interpolation을 범위에 적용할 수 있도록 변경했습니다.

빨라진 시작 시간

저희는 Rider의 기능을 빌려서 Visual Studio COM API 대신 MSBuild에 기반한 프로젝트 모델을 사용하기 시작했습니다. 이를 통해 솔루션의 로딩 시간을 최적화하고, UI 스레드를 잠그는 COM API의 사용을 최소화할 수 있습니다.

또한 성능 가이드 페이지가 업데이트되었습니다. 새로운 Windows Defender 섹션 및 제안이 적용된 후 정확히 무슨 작업이 수행되는지에 관하여 설명하는 모든 성능 팁에 대한 설명이 추가되었습니다.

탐색 및 사용 위치 검색

이번 릴리스에서 Find Usages(사용 위치 검색)는 Solution Wide Error Analysis(솔루션 전체 오류 분석)의 데이터를 사용해 결과를 수집합니다. 솔루션에서 SWEA를 활성화하면 Find Usages가 훨씬 더 빠르게 결과를 표시합니다.

도움이 될 만한 작은 개선 사항들이 탐색에 추가되었습니다.

  • Go to File Member 대화 상자에서 직접 파일을 검색할 수 있습니다. 결과 목록에는 일치하는 모든 멤버의 이름 및 파일 안에서 검색 문자열이 나타나는 모든 장소가 표시됩니다.
  • 이제 Go To Text 결과 창에서 주석 문자열 안의 발생 위치를 검색하고 싶지 않을 경우 해당 주석 줄을 필터링할 수 있습니다.

텍스트로 이동에서 대상 파일 확장

Go to Text(텍스트로 이동)의 기반이 되는 엔진은 솔루션에서 말 그대로 모든 텍스트 파일만 고려합니다.

이전에 Go to Text는 ReSharper에서 색인이 생성된 파일만 대상으로 삼았습니다. ReSharper는 코드가 포함되어 있거나 코드 작성에 도움이 되는 파일만 색인을 생성합니다. 그 밖의 파일에서 전체 색인을 생성하는 일은 대부분의 경우 효율적이지 않기 때문입니다. 이에 따라 '알 수 없는' 파일 확장자를 가진 파일은 모두 검색 범위에서 완전히 제외되어 코드 파일이 아닌 경우 검색할 수 없었습니다.

이제 엔진이 한 단계 더 발전되어, 색인 생성된 파일의 결과가 표시되는 동안 색인 생성되지 않은 텍스트 파일을 백그라운드에서 검색하기 시작합니다. 이는 *proj, .yml, .csv 등의 다양한 확장자를 가진 파일에 일부 데이터를 저장한 경우 매우 유용합니다.

SDK 프로젝트의 참조 최적화

또한 이제 ReSharper는 SDK 프로젝트에서 사용되지 않은 패키지 참조를 검사합니다. 프로젝트의 컨텍스트 메뉴에서 'Optimize References(참조 최적화)'를 사용하여 이러한 참조를 분석하고 최적화해 보세요. 주어진 참조를 사용하는 코드가 프로젝트에 하나도 없을 경우, 해당 참조는 사용되지 않은 것으로 표시됩니다. 사용되지 않은 패키지는 간단히 제거해 프로젝트의 종속 요소 트리를 깔끔하게 유지할 수 있습니다.

자세한 내용은 EditorConfig 코드 스타일 내보내기, SDK 프로젝트의 참조 최적화 및 ReSharper 2019.1의 다양한 업데이트를 참조하세요.

추출 메소드 리팩토링 개선

Extract Method(메소드 추출) 리팩토링이 이제 훨씬 개발자 친화적으로 바뀌었습니다. 삽입된 매개변수의 자동 생성된 이름이 정확히 원하는 것이 아닌 경우, 간단히 리팩토링 대화상자에서 바로 이름을 편집할 수 있으며 삽입된 메소드 이름, 반환 유형 및 가시성도 함께 수정할 수 있습니다.

코드 에디터의 힌트 기능

이제 매개변수 이름의 힌트가 더 많은 코드 위치에서 작동합니다. 우선, 속성의 사용 위치에서 힌트를 찾으면 해당 속성의 매개변수 이름이 표시됩니다. 다음으로, 특별한 힌트가 StringFormatMethod 속성, 즉 Console.WriteLine()으로 표시된 메소드에 나타나며 해당 위치에 삽입될 값을 가진 매개변수의 색인을 보여줍니다. 단, 'StringFormatMethod' 힌트는 기본적으로 사용 중지되어 있으며 옵션에서 사용 설정할 수 있습니다.

이 외에도, 메소드 선언의 매개변수에 대한 코드 주석 힌트가 새로 추가되었습니다. 이 힌트는 ReSharper 코드 분석 기능에서 산출되고 다음과 같은 메소드에 지정된 매개변수의 코드 주석을 보여줍니다.

  • 상위 클래스에 있는 재정의된 베이스 메소드
  • 상위 인터페이스에서 선언된 구현 메소드

ReSharper는 Rider의 디버그 기능 중 하나를 Visual Studio에 도입하여 이제 지역 변수, 현재 줄의 표현식, 함수 반환값이 에디터에 표시됩니다. 이에 따라 디버그 세션 중 코드 에디터를 벗어나지 않고도 즉각적인 피드백을 확인할 수 있습니다.

또한 ReSharper에는 모든 값을 검색할 수 있는 자체적인 디버거인 DataTips가 함께 제공됩니다. 코드 입력을 시작하는 즉시 바로 그 자리에서 검색 결과를 얻을 수 있습니다.

열에서 코드 정렬

코드 정렬과 관련해 요청된 내용 중 여러 인기 있는 기능들이 구현되었습니다. 같은 피연사자를 정렬하고, 내어쓰기를 구현하고, ternary 연산자를 사용하는 코드를 더욱 보기 좋게 작성하기 위한 요청이 있었으며, 코드를 더욱 읽기 쉽게 만들 수 있는 그 밖의 여러 지원 요청이 있었습니다.

일부 개발자들은 클래스의 속성을 열에 맞추는 것을 선호합니다. 예를 들어, { get; set; }을 같은 열에서 시작합니다. 또 어떤 개발자들은 열에 할당을 정렬하고 싶어 합니다. 이제 ReSharper 2017.3은 이러한 모든 옵션을 지원합니다!

중단점/추적점을 효율적으로 생성 및 탐색

ReSharper 2017.3에는 결과 찾기 창의 검색 결과로부터 중단점 또는 추적점을 바로 생성할 수 있는 기능이 도입되었습니다. 이 기능을 사용하면 디버그하려는 모든 코드 엔티티의 사용 위치에 빠르게 중단점을 생성할 수 있습니다.

또한 새로운 릴리스에는 솔루션에서 사용 가능한 모든 중단점/추적점을 보여주는 새로운 중단점 메뉴가 추가되었습니다. 이 메뉴를 통해 직접 항목을 검색/삭제하거나 별도의 도구 창에서 목록을 열 수 있습니다.

새로운 R# 중단점 도구 창은 중단점/추적점과 함께 다양한 작업을 지원합니다.

  • 다양한 방식으로 그룹 지정(파일/폴더/프로젝트/유형/멤버)
  • 단일 항목 또는 항목 그룹을 제거/비활성화/활성화
  • 항목 검색

사용 위치 검색/호출 추적 업데이트

결과 찾기 도구 창은 멀티 라인 발생에 더욱 다양한 이점을 선사합니다. 즉, 결과 트리에서 컨텍스트를 단일 라인으로 나타내도록 표현이 변경되었습니다. 검색 결과 목록에서 여러 줄로 서식 지정된 사용 위치를 쉽게 확인할 수 있습니다.

업데이트된 호출 추적 도구 창은 이미 검토한 호출을 제거함으로써 초점을 유지할 수 있게 도와줍니다.

ToDo 항목의 외부 리소스로 연결

할일 패턴에 URL 필드가 추가되었습니다: 할일 항목에서 Ctrl클릭을 사용하거나 AltEnter를 누르면 코드에서 외부 리소스로 이동할 링크가 만들어집니다. 이렇게 하면 할일 주석을 간략하고 직관적으로 유지할 수 있으며, 전체 URL이 필요하지 않고 발행 ID만 나타납니다.

할일 항목의 링크는 피크 미리보기 기능을 지원하여 에디터에서 연결된 문제 인라인을 표시합니다. 또한 AltEnter 메뉴를 통해 할일 탐색기에서 해당 할일 항목을 쉽게 찾을 수 있습니다.

마지막으로, 할일 탐색기의 새로운 필터를 통해 여러 패턴을 선택할 수 있습니다.

향상된 C# 7.0 지원

각각의 릴리스에서 C# 7.0에 대한 ReSharper 지원이 더욱 향상되었습니다. 이번 릴리스 사이클에서는 ReSharper가 패턴 일치를 처리하는 방식을 업그레이드하였습니다.

  • 이제 switch 후위 템플릿을 비-밀폐형 참조 유형의 변수에서 사용할 수 있습니다.
  • 새로운 컨텍스트 작업인 유형 패턴 생성은 선택한 유형에 대한 switch를 쉽게 생성할 수 있게 도와줍니다.
  • switch로 전환 빠른 수정을 완전히 재작업하여 패턴을 사용해 여러 개의 if 문을 단일 switch 문으로 변환할 수 있습니다.

이제 ReSharper가 out 변수에서 var(또는 코드 스타일에 따라 명시적 유형 사용) 사용을 제안할 수 있습니다. 또한 out 매개변수와 함께 메소드 호출에만 사용되는 변수 대신 out 변수를 도입하기 위한 검사와 빠른 수정을 추가하였습니다.

C# 7.1 이해

Visual Studio 2017 15.3에는 C# 7.1이 도입되었으며, ReSharper는 빠르게 해당 언어 기능을 파악할 수 있습니다. 예를 들어, C# 7.1은 default(T) 식 대신 사용할 수 있는 default 리터럴을 추가해 사용 위치 컨텍스트를 기반으로 타겟 유형을 추론합니다. ReSharper는 default 리터럴 구문을 인식하여 default(T)이 사용될 때 검사를 제공하고, 중복 유형 사양을 제거하기 위한 빠른 수정을 제안합니다.

또한 ReSharper는 이니셜라이저에서 추론할 수 있을 때 중복 tuple 값 구성요소를 표시하는 검사 기능을 갖춘 tuple 투사 이니셜라이저를 지원합니다.

마지막으로, ReSharper는 비동기 메인 및 패턴 일치(일반)를 이해합니다.

새로운 IEnumerable 검사

새로운 시나리오를 수용할 수 있도록 ReSharper의 IEnumerable 검사 세트가 확장되었습니다.

  • 이제 가능한 다중 에뮬레이션 코드 검사가 IOrderedEnumerable 및 ParallelQuery에서 파생된 유형을 고려할 수 있습니다.
  • 새로운 코드 검사는 IQueryable에서 IEnumerable까지 계획되지 않은 변환을 감지합니다.

다양해진 코드 검사 및 컨텍스트 액션

C# 7.1에서는 tuple 값의 요소 이름이 추론되며, ReSharper가 중복 값 tuple 구성요소 이름을 식별할 수 있게 도와줍니다.

매개변수에서 자동 속성 도입 빠른 수정 덕분에 이미 비공개 세터 추가 또는 공개 이변 속성 설정과 같은 방대한 옵션 세트와 함께 get-only 자동 속성을 도입할 수 있었습니다. 이제 매개변수로부터 자동 속성 초기화 컨텍스트 작업이 빠른 수정에 맞게 조율되어 똑같이 광범위한 옵션 세트를 제공합니다.

Null 검사 기본 설정

null 값을 확인할 수 있는 여러 가지 방법이 있습니다. 그러면 코드를 생성할 때 ReSharper에 도입된 null을 확인하는 방법이 마음에 들지 않는 경우 어떻게 할까요?

ReSharper 2017.2에서는 새로운 옵션 페이지인 Null 검사를 추가하였습니다. 여기서 빠른 수정, 컨텍스트 작업 및 코드 생성 작업에서 사용되는 null 검사 패턴의 우선순위를 구성할 수 있습니다. 뿐만 아니라 자체적인 사용자 지정 검사를 만들 수도 있습니다! null의 매개변수 확인 컨텍스트 작업을 사용 중인 경우, 새로운 하위 메뉴 항목인 null 검사 패턴 구성을 살펴보십시오. 이 메뉴를 통해서 ReSharper의 옵션 대화상자를 살펴보지 않고도 이 새로운 페이지에 액세스할 수 있습니다.

탐색 및 검색 개선 사항

이제 모든 항목으로 이동 및 기타 탐색 작업이 임의 순서로 타겟 쿼리의 단어를 포함하는 요소를 매칭시킬 수 있습니다. 예를 들어, exactMatching 검색이 isMatchingExactly와 일치합니다.

반면, 정확한 검색은 검색 엔진에서 기대하는 방식으로 작동합니다. "컬렉션"을 조회하면 컬렉션이 반환되지만, FilteredCollection, IterateCollection() 등은 반환되지 않습니다. 그러나 정확한 검색을 사용해 합성물 이름을 필터링하는 동안 계속해서 * 및 ? 와일드카드를 사용해 필요한 수만큼의 변형을 정확히 검색할 수 있습니다.

다른 노드에서 이제 텍스트로 이동이 모든 항목으로 이동에 통합됩니다(CtrlT). 검색어에 해당하는 코드가 없을 경우, ReSharper가 대신 텍스트를 찾기 시작합니다.

또한 이제 도트 구분 기호를 사용해 확장 메소드를 검색할 수 있습니다: Class.ExtensionMethod.

마지막으로, ReSharper 2017.2는 결과 찾기 창에서 검색 결과를 탐색할 수 있는 새로운 바로가기를 제공합니다. 숫자 키패드에서 + 대신 ShiftEnter를 누르십시오.

  • ReSharper에서는 실제로 전체 특성 이름을 입력하지 않고도 NotNull 및 CanBeNull 특성을 사용해 인수를 빠르게 표기할 수 있습니다. 유형 이름 다음에 ! 또는 ?를 입력하면 매개변수 또는 멤버 선언의 이름이 각각 NotNull 또는 CanBeNull 주석을 삽입합니다. 이름이 이미 NotNull로 표기된 경우, !를 입력하면 메소드 바디의 시작 부분에 null 검사가 자동으로 추가됩니다.

탐색 및 검색

결과 찾기 창에는 사용 위치 종류별로 검색 결과를 그룹화할 수 있는 새로운 옵션과 특정 사용 위치의 발생을 선택하기 위한 새로운 필터, 정확한 일치 검색을 위한 따옴표 사용 등이 제공됩니다.

ReSharper의 검색 및 탐색 페이지에는 어느 위치에서든 미리보기 탭에서 파일을 열 수 있는 새로운 옵션이 추가되었습니다. 이 옵션은 기본적으로 해제되어 있습니다.

모든 항목으로 이동 및 텍스트로 이동 팝업은 검색 결과의 고급 필터링을 위해 인라인 명령을 지원합니다.

Angular 2 템플릿 구문

ReSharper는 Angular 2 템플릿 구문의 다음 요소를 이해합니다: 템플릿 식, 템플릿 문, NgFor 명령 및 템플릿 변수. Angular @Component 데코레이터의 template: 속성을 비롯하여 @Component의 templateUrl: 속성으로 참조되는 순수 HTML에 대한 지원이 구현되었습니다. 상대적 경로는 templateUrl에서만 지원됩니다. HTML 페이지의 경우, 기본적으로 Angular 마크업이 해제되어 있으며, ReSharper 옵션에서 활성화할 수 있습니다(코드 편집 | HTML | 에디터).

코드 완성은 Angular 특성과 구성요소 태그를 비롯해 [attr., [style. 및 [class.에 대해 작동합니다. 구성요소 특성에는 선언으로 이동 (F12) 명령에 대한 지원이 제공됩니다. 이름 바꾸기 리팩토링은 구성요소 태그 및 [class.foo]의 'foo'에 대해 작동합니다.

Angular 1은 아직 지원되지 않습니다. ReSharper 옵션에서 HTML 페이지에서 Angular 마크업 지원이 '없음'으로 설정된 경우, 코드 완성 및 라이브 템플릿에 대해 AngularJS 플러그인을 사용할 수 있습니다.

TypeScript 및 JavaScript 지원

  • TypeScript 2.1에 대한 전체 지원: 매핑 및 인덱싱 유형, 객체 rest 및 spread 속성, 리터럴 유형의 논리 업데이트, 구성 상속, 미입력 가져오기, 유형 정규화 결합, 부분적 주석이 추가된 시그너처, 암묵적 및 암묵적 어레이에 대한 제어 흐름 분석.
  • TypeScript 2.2에 대한 최초 지원: 'object' 유형, JSX spread 구문 및 시그너처/tuples/교차/매핑 유형으로부터 파생(확장/구현)
  • TypeScript ES6 스타일 가져오기에서 미사용 가져오기에 대한 검사 및 빠른 수정
  • JavaScript 및 TypeScript의 Postfix 템플릿
  • TypeScript 및 JavaScript에서 코드 완성 항목의 관련성 개선
  • 코드 생성 메뉴에 TypeScript 클래스에 대한 속성 또는 읽기 전용 속성을 생성하기 위한 새로운 옵션이 제공됩니다. 이제 가능한 경우 재정의 멤버 생성 명령이 'super'를 호출합니다.
  • TypeScript에서 구현 멤버로 이동
  • TypeScript 및 JavaScript에서 함수 종료로 이동
  • TypeScript의 이름 바꾸기 리팩토링은 로컬 기호에 대해 더 빠르게 작동하고 TypeScript에 대한 동적 검색을 비활성화할 수 있는 새로운 옵션을 제공합니다.

코드 분석

ReSharper의 파일 상태 표시기(에러 스트라이프 상단의 작은 아이콘)가 새로운 컨텍스트 메뉴를 제공합니다. 아이콘을 마우스 오른쪽 버튼으로 클릭하면 ReSharper 코드 분석을 비롯해 식별자 강조 표시 및 Visual Studio 코드 분석 간을 빠르게 전환할 수 있습니다(전구 및 구불구불한 선). 현재 파일에 코드 문제가 있을 경우, 가장 높은 심각도 레벨의 문제를 탐색하기 위한 명령도 제공됩니다.

대용량 파일의 경우, ReSharper가 코드 분석을 자동으로 일시 중지합니다. 이 경우, 상태 표시기에서 회색 일시 중지 아이콘을 볼 수 있습니다.

ReSharper C++ 업데이트 Ultimate

ReSharper C++ 2017.1은 Visual Studio 2017 RTM 지원을 완성하고, 폴더 열기 모드 및 CMake 프로젝트와의 연동을 인식합니다. 변경 내용에는 성능 개선, 확장된 postfix 테믚ㄹ릿 세트 및 코드 정리 작업, 제어 흐름 검사에 대한 업데이트가 포함됩니다.

ReSharper C++ 2017.1에 대해 자세히 알아보십시오.

명령줄 프로파일러 Ultimate

dotMemory 2017.1에는 명령줄에서 프로파일링을 할 수 있는 dotMemory.exe 도구가 포함됩니다. 이 도구는 메모리 스냅샷 수집 프로세스를 자동화해야 할 경우 매우 유용합니다. 예를 들어, 프로파일링을 연속 통합 빌드의 일부로 설정하려는 경우가 해당됩니다.
