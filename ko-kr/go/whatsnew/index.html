GoLand의 새로운 기능

GoLand 2019.3에서는 CPU 소비가 감소하고 성능이 빨라졌으며 Go 모듈 지원이 강화되고 새로운 빠른 수정 모음이 추가되고 리팩토링 기능이 향상되었습니다. 또 Evaluate Expression(표현식 평가)을 통해 포함된 필드의 메소드를 호출하고 Mozilla rr을 사용하여 원격으로 애플리케이션을 디버그할 수 있습니다.

모든 새 프로젝트에 대해 Global File Watcher(전역 파일 와처)를 활성화하고 MongoDB로 작업하는 등 다양한 기능을 이용해 보세요.

성능

올 여름 GoLand 2019.2를 출시한 이후, JetBrains는 GoLand의 성능을 높이고 CPU 소비를 줄이기 위해 열심히 작업해 왔습니다. 주요 성능 개선 사항 중 하나는 IDE가 백그라운드에서 모든 것을 준비하고 시작 화면을 표시하고 프로젝트의 색인을 생성하는 데 걸리는 시작 시간을 줄이는 것입니다.

결과는 작업 중인 프로젝트의 크기와 사용 중인 기기의 연식, RAM, 디스크 공간에 따라 달라질 수 있습니다. 이 개선 작업은 앞으로도 계속 진행될 예정이므로 향후 릴리스에서 더 빠른 시작 시간을 기대하실 수 있습니다.

Go 모듈 개선 사항

GoLand 2019.3은 go.mod 파일에 구문 강조 및 탐색 기능을 제공합니다. 특정 종속 요소의 가져오기 경로에서 macOS의 경우 ⌘+B를 누르고 Windows/Linux의 경우 Ctrl+B를 누르면 Project View(프로젝트 뷰)에서 해당 소스로 이동할 수 있습니다.

또한 Project View에서 Go 모듈은 그저 마지막 세그먼트가 아닌 전체 가져오기 경로와 함께 표시됩니다. 또한 각 Go 모듈의 버전은 가져오기 경로보다 어두운 글꼴 색상으로 표시되므로 쉽게 구분할 수 있습니다.

디버거 업데이트

포함된 필드의 메소드 호출

디버거에서 Evaluate Expression(표현식 평가)을 사용하면 임베디드된 필드의 메소드를 바로 호출할 수 있습니다.

Mozilla rr로 원격 디버그

Linux 및 Mozilla rr에서 실행되는 Go로 작성된 많은 프로덕션 서비스는 재현하기 어려운 버그를 조사하는 데 도움이 될 수 있습니다. Mozilla rr 디버그 기능은 GoLand 2018.3부터 지원되었지만 Linux가 실행되는 개발자 시스템에서만 작동했습니다. 이제 Linux 시스템에서 디버그 서버를 시작하고 macOS 및 Windows에서 원격으로 추적을 디버그할 수 있습니다. 이 작업을 수행하려면 다음 단계를 따르세요.

  • 원격 시스템에서 rr 추적을 수집합니다.
  • 다음과 같이 원격 시스템에서 디버거를 시작합니다. dlv --headless --api-version 2 -l localhost:2345 replay /path/to/trace/dir /path/to/binary
  • Go Remote 구성을 통해 원격 시스템에 연결합니다.

단, Mozilla rr은 Linux 시스템에서만 사용 가능합니다.

새로운 빠른 수정 기능

사용되지 않는 매개변수 처리

사용되지 않는 매개변수에서 해당 매개변수의 이름과 타입을 모두 삭제할 수 있는 경우 GoLand는 두 가지를 모두 강조 표시하고, 타입이 메소드 사양을 구현하는 데 사용되는 경우에는 이름만 강조 표시합니다.

후자의 경우, 선택 가능한 두 가지 빠른 수정 옵션으로 Delete parameter names(매개변수 이름 삭제)와 Rename to_(다음으로 이름 변경)가 제공됩니다. 함수 유형으로 사용되는 함수에도 동일한 동작이 제공됩니다.

또한 사용되지 않는 모든 매개변수를 한 번에 삭제할 수 있습니다. Alt+Enter를 누르고 Delete all unused parameters(사용되지 않는 모든 매개변수 삭제) 빠른 수정을 사용하세요.

누락된 메소드 또는 함수 생성

GoLand는 함수 타입에 사용되는 함수 또는 메소드가 누락된 경우 이를 생성하기 위한 빠른 수정을 제공합니다. 간단히 Alt+Enter를 누르고 Create method/function(메소드/함수 생성)을 선택하기만 하면 됩니다.

Enhanced handling of package aliases

GoLand now suggests importing packages based on their used aliases via Alt+Enter in any file in your project.

리팩토링

시그니처 변경

이제 Change Signature(시그너처 변경)에서 Parameters(매개변수) 및 Result parameters(결과 매개변수) 탭에 매개변수 유형이 항상 표시됩니다. 새로운 Reuse types(타입 재사용) 체크박스를 선택하면 Signature Preview(시그니처 미리보기) 및 결과에서 결과 타입을 축소할 수 있습니다.

또한 매개변수 정의를 단일 선언으로 병합하는 Reuse signature types(시그니처 타입 재사용)와 Alt+Enter를 눌러 각 매개변수에 유형을 추가하는 Expand signature types(시그니처 타입 확장)라는 두 가지 인텐션 액션이 새로 추가되었습니다.

변수 추출 및 이름 변경

Extract Variable(변수 추출) 및 Rename(이름 변경)에서 더 알맞은 이름을 제안합니다. 예를 들어 유형 오류 표현식에는 err을 제안하고 context.Context에는 ctx를 제안합니다.

향상된 코드 검사 동작

이제 GoLand의 코드 검사가 검사 대상에 더 초점을 맞춥니다. Alt+Enter를 눌러 컨텍스트 메뉴를 호출할 때 특정 코드에 가장 적절한 빠른 수정 및 인텐션 액션이 표시되도록 코드 에디터에서 탐지된 문제가 강조 표시되는 범위를 줄였습니다.

코드 편집

향상된 rune 리터럴 렉서

rune 리터럴 렉서가 여러 면에서 개선되었습니다.

  • GoLand가 더 명확하고 간결한 오류 메시지를 통해 표현식이 잘못된 이유를 설명합니다.
  • GoLand가 편집 후 유효한 rune 리터럴을 오류로 강조 표시하지 않습니다.
  • 유효하지 않은 rune 리터럴이 더 안정적으로 파싱됩니다.

재할당된 부분 강조 표시

Go에서는 짧은 var 선언의 왼쪽에 기존 변수를 사용할 수 있습니다. 따라서 동일해 보이는 2개의 구문이 서로 다른 의미를 가지는 경우가 있습니다. 이를 쉽게 구분할 수 있도록 재할당된 부분에 대한 구문 강조 표시를 추가했습니다.

복합 리터럴에서 코드 접기

GoLand 2019.3은 복합 리터럴에서 코드 접기를 제공합니다. 코드 조각을 접거나 확장하려면 코드 접기 토글을 사용하거나 Windows에서 Ctrl+NumPad Minus/Ctrl+NumPad Plus를 누르거나 macOS에서 ⌘+NumPad Minus/⌘+NumPad Plus를 누르세요. 기본 코드 접기 동작을 구성하려면 Settings(설정)/Preferences(기본 설정) | Editor(에디터) | General(일반) | Code Folding(코드 접기)로 이동하세요.

향상된 라이브 템플릿

for 및 forr 라이브 템플릿에서 이제 적절한 변수 이름을 제안합니다. 예를 들어 색인에는 i 및 index와 같은 이름이 제안되고 값에는 s, string 및 line이 제안됩니다.

라이브 템플릿 meth에서는 리시버 이름에 대한 코드 제안을 제공합니다.

탐색

선택 영역에서만 검색

Find(검색)를 사용할 때 선택 영역에서만 검색할 수 있습니다(macOS에서는 ⌘+F, Windows/Linux에서는 Ctrl+F). 간단히 에디터에서 필요한 코드를 선택하고 Find를 호출하면 IDE가 선택한 영역에서만 검색합니다. 전체 파일 내 검색으로 간편하게 전환하려면 ⌘+F/Ctrl+F를 두 번 누르거나 표시줄에서 In Selection(선택 영역 내) 옵션을 전환하세요.

파일 마스크 옵션으로 '경로에서 검색' 사용

파일 마스크를 사용해 Find in Path(경로에서 검색) 대화상자에서 검색하면 GoLand가 그에 따라 검색 결과에서 파일 확장자를 표시합니다. 이렇게 하면 이름은 같지만 파일 확장자가 다른 파일을 더 쉽게 구별할 수 있습니다.

모든 새 프로젝트에 전역 파일 와처 사용

GoLand 2018.2에는 Project(프로젝트) 및 Global(전역) 옵션을 사용하여 다른 프로젝트 간에 파일 와처 구성을 공유할 수 있는 옵션인 Global File Watcher(전역 파일 와처)가 추가되었습니다.

그러나 이 와처를 사용하려면 새 프로젝트마다 활성화해야 했습니다. 이제 생성하거나 열려 있는 모든 새 프로젝트에서 기본적으로 이 와처를 사용하도록 설정할 수 있습니다.

File(파일) 메뉴에서 Other Settings(기타 설정) | Settings for new projects(새 프로젝트 설정)를 선택하고 Tools(도구) | File Watchers(파일 와처)로 이동해 새로운 전역 와처를 추가한 다음, 그 옆에 있는 체크박스를 선택하여 모든 새 프로젝트에서 활성화하세요.

튜토리얼

프로젝트를 완료하고 마감일을 앞두고 있는 상황에서 사용하는 제품에 대한 새로운 기능 페이지, 릴리스 노트, 블로그 게시물을 읽을 시간이 부족한 경우가 종종 있습니다. 그래서 JetBrains에서는 시간이 있을 때 GoLand 기능을 배울 수 있는 Tutorials(튜토리얼)을 추가했습니다.

Tutorials는 프로젝트 목록 상단의 시작 화면에서 찾을 수 있습니다. IDE를 새 버전으로 업그레이드하면 새 릴리스의 기능을 다루는 레슨이 생깁니다. 또는 모든 프로젝트에서 해당 도구 창을 통해 Tutorials에 액세스할 수 있습니다.

기타 변경 내용

계층 구조 도구 창

Hierarchy(계층 구조) 도구 창에서 피호출자의 메소드 계층 구조를 분석할 수 있는 새로운 옵션이 생겼습니다. 선택한 메소드의 호출자와 피호출자 사이를 전환할 수 있는 2개의 전용 버튼이 추가되었으며 메뉴에서 특수 버튼을 클릭하면 트리 내 요소를 알파벳 순으로 정렬할 수 있습니다. Tool Windows(도구 창) 목록에서 Hierarchy 도구 창을 보려면 먼저 계층 구조를 빌드한 다음, Navigate(탐색) | Call Hierarchy or Navigate(호출 계층 구조 또는 탐색) | Type Hierarchy(타입 계층 구조)로 이동하세요.

테마 및 단축키 플러그인을 빠르게 전환

GoLand 2019.3에서는 IDE를 다시 시작하지 않아도 테마 플러그인과 단축키 플러그인을 설치 및 제거할 수 있습니다.

에디터로 초점을 돌려주는 사용자 지정 단축키

에디터로 초점을 돌리기 위해 ESC를 사용하는 대신 사용자 지정 단축키를 설정할 수 있습니다. Preferences(기본 설정)/Settings(설정) | Keymap(단축키)에서 Focus Editor(에디터에 초점 두기) 액션을 찾은 다음, 오른쪽 마우스 버튼을 클릭한 후 드롭다운에서 Add Keyboard Shortcut(키보드 단축키 추가)을 선택합니다.

스크롤바의 가시성 개선

스크롤바가 더 눈에 띄도록 설정하는 옵션이 추가되었습니다. 이 옵션을 사용하려면 Preferences(기본 설정)/Settings(설정) | Appearance & Behavior(꾸미기 및 동작) | Appearance(꾸미기)에서 Use contrast scrollbars(명암대비 스크롤바 사용) 체크박스를 선택하세요.

버전 관리 개선 사항

재설계된 복제 대화상자

Clone(복제) 대화상자는 VCS | Get from Version control(버전 관리에서 가져오기)에 있습니다. 이제 이 대화상자에서 로그인할 수 있으며, 이미 로그인한 경우 계정 또는 조직별로 그룹화된 모든 저장소 목록이 IDE에서 즉시 미리보기로 표시됩니다.

병합 충돌에 대한 더 많은 정보 제공

이제 병합, 리베이스 또는 체리픽 작업 중에 발생한 충돌을 더 간편하게 조사할 수 있습니다. Show Details(세부 정보 표시) 링크를 클릭하면 결과 코드 상태의 원인이 된 커밋 목록이 표시되고, 변경 소스에 대한 자세한 정보는 Merge(병합) 대화상자에서 확인할 수 있습니다.

데이터베이스 업데이트

MongoDB 지원

GoLand 2019.3에서는 MongoDB에 대한 기본 지원이 제공되어 이제 다음을 수행할 수 있습니다.

  • 데이터베이스 탐색기에서 컬렉션 및 필드를 확인할 수 있습니다.
  • 컬렉션을 열거나 쿼리 결과를 확인할 수 있습니다. 열을 기준으로 정렬하거나 값을 필터링할 수도 있습니다.
  • 쿼리를 실행하고 결과를 얻을 수 있습니다. 구문을 선택하고 Ctrl/⌘+Enter를 누르거나 도구 모음의 Play(실행) 단추를 사용하여 실행하세요.

Search for connection error

이제 연결 오류를 쉽게 검색할 수 있습니다. 연결이 실패하면 IDE에서 전용 버튼을 간단히 클릭하여 Google 검색 엔진을 통한 오류 검색 결과로 이동할 수 있습니다.

웹 개발

CDN상의 CSS 라이브러리에 코드 완성 지원

Bootstrap이나 다른 CSS 라이브러리를 사용하고 해당 라이브러리가 HTML 파일 내에서 CDN에 연결된 경우 이제 클래스 소스를 프로젝트에 추가하지 않아도 해당 라이브러리의 클래스 이름에 코드 완성 기능을 사용할 수 있습니다.

새 심볼의 이름 제안

JavaScript 또는 TypeScript에서 새 변수 및 매개변수 이름을 추가하면 프로젝트, 표준 라이브러리, 사용 중인 종속 요소에 정의된 클래스, 인터페이스, 유형 별칭의 이름을 기반으로 이름이 제안됩니다.

새로운 변수, 함수, 클래스, 인터페이스의 경우 동일한 범위에서 이미 사용되었지만 아직 정의되지 않은 이름에 대한 제안도 제공됩니다.

2019.2(7월 25일)

함수 호출 지원

프로그램 실행 중 어떤 지점에서든 표현식 내부에서 발생하고 있는 상황을 상세히 검사할 수 있습니다.

함수 호출 지원과 관련된 버그나 문제를 발견하신 경우 JetBrains의 이슈 트래커로 알려 주시기 바랍니다.

기본 및 Darcula 색 구성표 개선

GoLand 2019.2는 기본 및 Darcula 색 구성표에서 시맨틱 코드 강조 표시에 다양성을 추가합니다.

이를 통해 코드를 더 잘 이해하고 로직을 더 빨리 파악할 수 있습니다. 패키지 이름과 변수, 내장된 코드 구성과 사용자의 코드 구성, 타입 변환과 함수 호출을 간편하게 구별하세요. 또한, 메소드의 수신자 및 함수/함수 호출도 이제 각자의 색상을 갖습니다.

언제나처럼 Settings/Preferences | Editor | Color Scheme에서 취향에 맞게 모든 항목을 변경하세요.

정수 변수가 디버거에서 기본적으로 표시되는 형식(2진수, 10진수 또는 16진수 형식)을 선택할 수 있습니다. Settings/Preferences | Build, Execution, Deployment | Debugger | Data Views | Go에서 형식을 설정하세요.

Go 1.13 기능에 대한 지원

GoLand 2019.2는 숫자 리터럴 및 부호 있는 정수를 시프트 횟수로 지원합니다. 이 변경 사항은 저희가 Go 2를 향해 진행하고 있는 단계와 관련이 있습니다. 자세한 내용은 공식 Go 블로그를 참조하세요.

GoLand의 코드 강조 표시 기능은 실수가 있을 경우 사용자에게 알려 주고 콘솔에 정확하게 출력을 표시합니다.

새로운 Go 1.13 기능 및 GoLand 2019.2가 지원하는 방식을 자세히 알아보려면 이 문서를 읽어 보세요.

사용자 지정 접미어 완성 템플릿

GoLand 2019.2에서는 자신만의 사용자 지정 접미어 완성 템플릿을 생성할 수 있습니다.

사용자 지정 템플릿을 생성하는 방법에 대한 자세한 내용은 이 문서를 참조하세요.

또한, 미리 정의된 새로운 접미어 완성 템플릿 — .aappend를 이용할 수 있습니다. 이 템플릿은 내장된 append 함수 호출을 사용해 표현식을 추가하고, 그 결과를 이 표현식에 할당합니다.

시그니처 변경에 따른 빠른 수정

Change Signature 리팩토링을 기반으로 한 수많은 빠른 수정이 추가되었습니다. 예를 들어, Alt+Enter를 누르기만 하면 자동으로 매개변수를 유추하여 함수 호출에 추가하거나, 시그니처에 누락된 return 매개변수를 추가할 수 있습니다.

return이 포함된 메소드 추출 리팩토링

Extract Method(메소드 추출) 리팩토링이 이제 return 문이 있는 코드를 처리합니다. 코드가 언제나 return을 통해 끝날 경우, 있는 그대로 추출됩니다. 하지만, 코드에 return이 없는 실행 경로가 포함되어 있을 경우 추출된 메소드는 추가적인 boolean 플래그를 return 값으로 갖게 됩니다. 이 플래그는 호출 측에서 조기에 return을 수행하는 데 사용됩니다.

코드 완성 개선

코드 완성은 이제 리시버가 없더라도 리시버 멤버에 대한 제안을 제공합니다. 기본적으로 리시버 이름으로 this 또는 self와 같은 이름은 권장되지 않으므로 메소드는 서로 다른 리시버 이름을 갖게 됩니다. 그러나 이 코드 완성 기능을 사용하면 그 차이를 무시할 수 있습니다.

더 똑똑해진 Go 모듈 및 dep 통합

GoLand 2019.2에서는 프로젝트가 Go 모듈 또는 dep를 사용하는 것으로 확인될 경우 Go 모듈/dep 통합이 자동으로 활성화됩니다. 이전에는 GOPATH 인덱싱이 비활성화되고 Go 모듈/dep 통합 활성화에 대한 알림만 표시되었습니다. 이 알림은 놓치거나 무시하기가 쉬워서 프로젝트가 절반만 구성된 상태가 되었기에 저희는 이것으로는 충분하지 않다고 판단했습니다.

이 기능이 자신에게 적합하지 않을 경우 Settings/Preferences | Go | Go Modules (vgo)에서 언제든 수동으로 통합을 비활성화할 수 있습니다.

코드에서 발생할 수 있는 성능 문제에 대해 더 잘 이해하려면 각 프로파일러 유형별로 이용할 수 있는 측정 기준을 사용해 결과를 분류합니다. Flame Chart, Call Tree 및 Method List 탭을 사용하거나, 해당 소스 코드로 이동하여 문제를 검사합니다.

스마트 스텝인투

새로운 디버거 액션인 Smart Step Into를 사용하면 일련의 호출과 중첩 메소드 호출을 더 쉽게 디버깅할 수 있습니다. 표현식 안의 임의의 호출로 스텝인투하고 관심 없는 메소드는 자동으로 스텝오버할 수 있습니다.

예를 들어, fmt.Printf("My name is %s, I am %d!", me.FullName(), me.Age())와 같은 코드를 디버그할 때 구체적으로 Age()로 스텝인투할 수 있습니다.

Smart Step Into(스마트 스텝인투)를 사용하려면 Shift+F7을 누르거나 Run(실행) 메뉴에서 Smart Step Into를 선택하세요.

자세한 내용은 GoLand에서의 디버그 – 2019.1 개선 사항을 참조하세요.

새로운 리팩토링

인터페이스 추출

인터페이스 추출(Extract Interface) 리팩토링을 사용하면 선택한 유형에 따라 새로운 인터페이스를 빠르게 생성할 수 있습니다. 실행하려면 Refactor This(다음을 리팩토링)를 호출한 후, Windows/Linux에서 Ctrl+Alt+Shift+T를 누르거나 macOS에서 ^T를 누르고 팝업 메뉴에서 Extract Interface를 선택하세요.

이 리팩토링 대화상자에는 새로운 인터페이스로 이전하도록 제안된 멤버 목록이 표시됩니다. 인터페이스의 이름을 지정하고 해당 인터페이스를 배치할 파일, 디렉터리 또는 패키지를 선택할 수 있습니다. 해당 유형이 이미 인터페이스를 구현하고 있는 경우, 목록에서 자동으로 제외됩니다.

자세한 내용은 GoLand에서의 리팩토링 — 인터페이스 추출을 참조하세요.

더 강력해진 이름 변경

Rename(이름 변경) 리팩토링은 인터페이스에 선언된 메소드의 이름 변경도 지원합니다. 이러한 메소드의 이름을 변경할 때 GoLand는 해당 구현의 이름도 변경하도록 제안합니다.

Find(검색) 도구 창의 Refactoring Preview(리팩토링 미리보기) 창에서 현재 리팩토링이 영향을 미치는 항목을 유형별, 프로젝트 내 위치별로 그룹화된 사용 위치와 함께 확인할 수 있습니다.

Nil 여부 분석기

Go는 제로 값을 활용할 수 있는 다양한 방법을 제공하지만, 기회가 많을수록 자칫 함정에 빠질 수 있습니다.

새로운 Nilness Analyzer(Nil 여부 분석기)는 다양한 경우에서 변수에 nil 값이 있는지 즉시 확인할 수 있도록 고안된 코드 검사 모음으로, 잠재적인 오류나 문제를 알려줍니다.

자세한 내용은 Nilness Analyzer 검사를 참조하세요.

스레드 대신 고루틴

이제 디버그하는 동안 줄 번호를 클릭하여 Run to Cursor(커서 위치까지 실행) 액션을 사용할 수 있습니다. 줄을 마우스로 가리켜서 강조 표시한 후 줄 번호를 클릭해서 Run to Cursor를 실행하세요.

사용할 수 있는 액션 목록 및 자세한 설명은 Settings(설정)/Preferences(기본 설정) | Editor(에디터) | Intentions(인텐션)에서 Go의 Declaration(선언) 섹션을 참조하세요.

또한, GoLand가 내보내지 않은 심볼에 대한 참조를 확인하고, 오류일 경우 알려줍니다.

이제 GoLand에서 Google App Engine 애플리케이션을 간편하게 로컬에서 실행 및 디버그할 수 있습니다.

새로운 App Engine 프로젝트 템플릿도 추가되었습니다.

코어 덤프 지원

표준 디버그 도구가 코드 문제를 이해하는 데 도움이 되지 않는 경우가 있습니다. 이러한 경우 코어 덤프가 필요합니다.

GoLand에서는 Run(실행) | Open Core Dump(코어 덤프 열기)를 선택하기만 하면 타사 도구 없이도 코어 덤프를 더욱 쉽게 탐색할 수 있습니다.

충돌 덤프를 자동으로 생성하려면 간단히 Run(실행) 구성 설정의 Environment(환경) 필드에 GOTRACEBACK=crash를 입력하세요.

코어 덤프는 기본적으로 Linux 시스템에서만 사용 가능합니다. 그러나 메모리 덤프는 macOS, Windows, Linux 어디에서든지 분석할 수 있습니다.

Testify 지원

이제 에디터에서 바로 스위트 및 메소드를 일반 테스트 함수로 실행할 수 있습니다.

Comparison Failure(실패 비교) 창은 실패한 어설션에 대한 예상 값과 실제 값 사이의 차이점을 살펴보는 데 도움이 됩니다.

관련 변경 내용에서는 Generate(생성) 팝업 메뉴를 통해 Empty test file(테스트 파일 비우기), Test for function(함수 테스트), Tests for file(파일 테스트), Tests for package(패키지 테스트)를 사용할 수 있습니다.

코드 검사

처리되지 않은 오류

Unhandled Error(처리되지 않은 오류) 코드 검사는 오류가 검사되지 않을 때마다 해당 오류를 반환하는 함수 또는 메소드에 대해 경고합니다.

도달할 수 없는 코드

Unreachable Code(도달할 수 없는 코드) 검사는 전혀 실행되지 않는 코드 부분을 탐지합니다.

형식 문자열 인수 추가

Add format string argument(형식 문자열 인수 추가) 인텐션 액션은 제공된 모든 표현식에 대해 자리표시자를 팝업을 통해 생성합니다. 자리표시자를 형식 문자열에 삽입하고 표현식을 자리표시자의 인수로서 전달합니다.

또한 자리표시자를 해당 인수로 바꾸는 형식 문자열에서 코드 접기를 사용할 수 있습니다.

생성 인텐션 액션

생성자 생성

Generate Constructor(생성자 생성) 인텐션 액션은 구조체 유형의 값을 생성하기 위한 함수를 생성합니다.

Generate(생성) 팝업 메뉴를 통해 Generate Constructor, Generate getter/setter(게터/세터 생성), Generate getter and setter(게터 및 세터 생성)를 사용할 수 있습니다.

게터 및 세터 생성

이제 Generate getter/setter(게터/세터 생성) 또는 Generate getter and setter(게터 및 세터 생성) 인텐션 액션으로 상용구 코드를 작성하고 포인터/비 포인터 리시버 유형 및 이름을 사용자 지정할 수 있습니다.

코드 완성

GoLand 2018.3은 함수에 대해 메소드와 같은 코드 완성 기능을 지원합니다. T 유형의 값 t가 있을 때 t.Foo를 입력하면 T 유형의 값을 첫 번째 인수로 받아들이는 함수를 살펴볼 수 있습니다.

사용 가능한 모든 코드 완성 옵션을 보려면 Basic completion(기본 완성)(Ctrl + Space + Space)를 두 번 실행하세요.

코드 에디터

현재 구문 완성

Complete Current Statement(현재 구문 완성)는 이제 필요한 후행 쉼표를 복합 리터럴(예: struct, slice 등)에 자동으로 삽입하고 캐럿을 다음 구문의 위치로 옮깁니다. 간단히 macOS에서는 Cmd + Shift + Enter를, Windows/Linux에서는 Ctrl + Shift + Enter를 누르세요.

쉐도우 처리되거나 쉐도우 처리하는 메소드 찾기

새로운 여백 아이콘을 사용하면 포함된 유형의 쉐도우 처리되거나 쉐도우 처리하는 메소드를 더 쉽게 찾을 수 있습니다. 한 번 클릭하거나 macOS에서 Cmd+U 또는 Windows/Linux에서 Ctrl+U를 눌러 앞뒤로 이동할 수도 있습니다.

이 파일 모음의 서식 지정 금지

GoLand 2018.3 이후 버전에서는 Reformat code(코드 서식 다시 지정), Optimize imports, Rearrange(재배치) 코드 액션에서 특정 파일 모음을 제외할 수 있습니다.

Go 템플릿(html/template)

Go 템플릿(html/template)으로 작업하면서 코드 완성, 사용 위치 검색, Rename(이름 변경) 리팩토링, 향상된 코드 서식 지정을 활용해 보세요.

디버거 업데이트

디버거는 Evaluate Expression(표현식 평가) 대화상자 및 Watchers(감시자) 창에서 코드 완성, 검사, 빠른 수정 기능을 제공합니다.

Mozilla rr 지원

Linux에서 실행되는 디버거에 Mozilla rr 지원이 추가되었습니다. Mozilla rr은 실패가 확인될 때까지 프로그램 실행을 기록하므로 디버거에서 필요한 횟수만큼 실패한 실행을 재생할 수 있습니다.

Mozilla rr을 설치하는 즉시 모든 것이 한 번의 클릭으로 작동합니다. 중단점을 삽입하고 에디터의 Run/Debug(실행/디버그) 여백 아이콘에서 Record and Debug(기록 및 디버그) 옵션을 선택하세요. 이 액션을 사용하면 Rewind(되감기) 버튼이 있는 Run(실행) 창의 Debugger(디버거) 탭이 열리고 해당 버튼을 누르면 이전 중단점에 도달할 때까지 프로그램이 뒤로 실행됩니다.

다이어그램 지원

  • 데이터베이스와 SQL의 테이블 및 관계

자세한 내용은 다이어그램 지원에 관한 블로그 게시물을 참조하세요.

경로 대화상자에서 검색 및 바꾸기

JavaScript 및 TypeScript

JavaScript에서 자동 import 문 추가

이제 GoLand는 프로젝트에 정의된 심볼뿐만 아니라 프로젝트의 종속 요소에 있는 심볼에도 import 문을 자동으로 추가합니다.

JavaScript에서 매개변수 힌트 지원

아시다시피 Go, TypeScript, SQL에는 매개변수 힌트가 제공되어 왔습니다. 이제 이 기능이 JavaScript에도 제공됩니다.

향상된 'null' 및 'undefined' 검사 기능

Object is null or undefined(null이거나 정의되지 않은 객체) 검사는 undefined 또는 null인 값에 대해 메소드가 호출되거나 해당 값이 함수에 전달되거나 해당 속성이 사용되어 TypeError가 발생한 경우 이를 알려줍니다.

package.json 완성 기능

GoLand는 package.json 종속 요소의 이전 버전뿐만 아니라 최신 버전에 대한 제안을 제공합니다.

Cassandra 데이터베이스 지원

DataGrip팀의 훌륭한 동료들 덕분에 GoLand는 이제 Cassandra 데이터베이스를 지원할 수 있게 되었습니다.

그 외 다양한 변경 사항

  • Attach(연결) 액션을 Open Project(프로젝트 열기) 팝업을 통해 이용할 수 있습니다. 이 액션을 사용하면 이미 열려 있는 프로젝트에 디렉터리를 연결할 수 있습니다. File(파일) | Open(열기)을 클릭하여 연결하려는 프로젝트를 선택하세요.
  • 파일과 프로젝트를 시작 화면으로 끌어서 열 수 있습니다.
  • Activity Monitor(활동 모니터링)은 하위 시스템 및 플러그인 사용하고 있는 CPU를 표시합니다(Help(도움말) | Activity Monitor(활동 모니터링)).
  • Settings(설정) | Plugins(플러그인)에 있는 Plugins 페이지가 모든 기능 및 UI를 비롯해 완전히 새로 작업되었습니다.
  • 색인 생성 속도가 크게 빨라졌습니다.
