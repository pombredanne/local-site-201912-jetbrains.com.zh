Python开发人员调查2018年结果

在2018年秋天，Python软件基金会与JetBrains一起第二次进行了年度官方Python开发者调查。与之前的调查非常相似，我们着手确定最新趋势并深入了解Python开发世界在2018年的样子。今年，来自150多个国家/地区的2万多名开发人员参加了此次活动，以帮助我们绘制出一个准确，最新的Python社区景观。

查看2017年Python开发人员调查结果 。

Python作为主要语言与次要语言

在我们的调查中，有84％的Python用户使用Python作为主要语言，而对于16％的用户来说，这是他们的辅助语言。在2017年，我们的分布略有不同：79％的人表示他们将Python作为主要语言，而21％的人将其作为第二语言。

Python作为主要语言的使用率比2017年Python软件基金会进行的先前调查的79％上升了5个百分点。

值得注意的是，数据分析比Web开发更受欢迎，从2017年的50％增长到2018年的58％。机器学习也增长了7个百分点。与Web开发相比，此类开发的增长速度更快，Web开发仅比上一年增长了2个百分点。

像2017年一样，Web开发是唯一一个将Python作为主要语言与补充语言之间的差距很大的类别（分别为56％和36％）。对于其他类型的开发，差异要小得多。

在2018年，我们有更多的受访者指定他们参与DevOps（与2017年相比增加了8％）。对于使用Python作为辅助语言的Python用户，DevOps已经超越了Web开发。

我们问：“您使用哪个版本的Python最多？”。Python 3是有力的领导者，占84％，Python 2被用作主要的解释器，只有16％。与2017年的75％相比，Python 3的普及率实现了巨大飞跃。

Python 3的使用继续快速增长。根据2017年的最新研究，使用Python 3的占75％，而使用Python 2的占25％。Python 2的使用正在下降，因为它不再积极开发，没有获得新功能，并且其维护将在2020年停止 。

另一个有趣的发现是，只有82％的主要从事Web开发的人使用Python 3，而对于那些从事数据科学的人来说，这一比例高达90％。

一个可能的解释是，某些Web开发人员在过渡到Python 3时仍然需要维护许多遗留代码。另一方面，许多数据分析师和机器学习专家最近才加入Python生态系统，并开始使用最新的Python 3。

Python安装和升级

71％的受访者从python.org或与操作系统提供的软件包管理器（如APT和Homebrew）一起安装Python。这与2017年的结果非常相似。有趣的是，Anaconda与2017年相比增长了7个百分点。这可能是证据，表明数据科学的普及度比其他类型的开发更快。

您是否使用以下任何工具来隔离Python环境（如果有）？（多个答案）

长期以来，在开发或部署阶段隔离Python环境一直是最佳实践。毫不奇怪，pipenv和较低级别的virtualenv一起是创建和管理新Python环境的两个最常用的工具。不过，令人惊讶的是，仍有21％的Python用户尚未接受这种做法。

本节重点介绍Python开发人员使用的各种Python框架，库和技术。

Web框架（多个答案）

毫不奇怪，最受欢迎的数据科学框架和库是：NumPy，Pandas，Matplotlib和SciPy。特定于机器学习的库（例如SciKit-Learn，TensorFlow，Keras等）也非常受欢迎，这与快速增长的机器学习作为Python开发的主要类型的发现紧密相关。

令人惊讶的是，几乎三分之二的受访者选择Linux作为他们的开发环境操作系统。请注意，对于这个问题，我们允许有多个答案。我们在这里没有得出主要的操作系统流行性结论。

测试框架

领先的单元测试框架是pytest，然后是unittest。其他单元测试框架远不那么受欢迎。令人惊讶的是，有35％的Python用户没有使用任何测试框架，并且大概没有测试他们的代码。在“创建隔离的Python环境的工具”部分中，我们确定了大约五分之一的Python用户不使用Python隔离，这是另一种最佳做法。

资料库

大多数人都在使用免费或开放源数据库，例如PostgreSQL，MySQL或SQLite。鉴于大量的Python用户从事某种形式的机器学习或数据工程，非关系数据库（如MongoDB和Redis）也非常受欢迎。

ORM

两种最受欢迎的ORM是SQLAlchemy和Django ORM，它们与两种领先的Web开发框架（Flask和Django）的流行程度相匹配。

您使用以下哪个大数据工具？

机器学习工程师更可能使用大数据工具，这就是为什么76％的受访者没有选择。Spark是Hadoop和Kafka之后的领导者。

您经常使用哪种持续集成（CI）系统？（多个问题）

大约一半的Python用户不使用任何CI解决方案。在Python世界中，三种最流行的CI解决方案是Jenkins，Gitlab CI和Travis。

配置管理

大多数Python用户不使用配置管理工具。在那些人中，明确的领导者是Ansible。

PyCharm是最流行的Python开发工具，在PyCharm专业版和社区版中占35％的份额。有趣的是，VS Code从2017年的7％增长到2018年的16％，成为Python开发中第二受欢迎的编辑器。很有可能是由于VS Code的快速增长，许多其他编辑者的用户份额却减少了。

Web开发人员与数据科学家的编辑偏好略有不同。他们比数据科学家更喜欢PyCharm，VS Code，Vim和Sublime文本，而许多数据科学家更喜欢Jupyter Notebook作为主要工具。

Python开发工具和功能

我们已经确定了用于Python开发的工具和功能的相对普及：版本控制，代码自动完成，代码重构，编写单元测试以及对Python项目使用虚拟环境都占据了头等位置。

其他流行的工具和功能包括SQL数据库，调试和代码插入。NoSQL数据库，Python分析器和代码覆盖率工具是使用最少的工具。

尽管类型提示是一种可选技术，但它似乎越来越流行。 59％的受访者表示，他们在使用Python开发时经常或不时使用类型提示。类型提示的使用高于代码覆盖率和分析器工具的使用，现在与CI解决方案在Python开发中的使用持平。

鉴于2018年我们的受访者人数较少，而开发人员的经验更丰富，因此可以得出结论，人们在专业团队中工作的时间越长，他们拥有的经验越多，他们使用的工具和专业技术就越多。

就业与工作

就业状况

超过一半的Python用户全职工作，其中19％是学生，而只有13％是自雇或自由职业者。与2017年相比，我们在2018年的学生人数明显减少，而被聘用的人才更多。

工作角色

受访者可以选择多个职位，因此总数超过100％。大约四分之三的人认为是开发人员，几乎五分之一的人认为是数据分析师，架构师或团队负责人。在“其他”中，收集的总数占12％，其中最主要的写入答案包括数据科学家，DevOps，研究员和老师。

重要要点

Python 3的采用率正在快速增长，目前已经达到84％，而只有16％的Python用户将Python 2用作主要的解释器。与2017年的75％相比，Python 3的普及率实现了巨大飞跃。

在Python用户中，数据分析比Web开发更受欢迎，从2017年的50％增长到2018年的58％。

使用Python作为主要语言的所有开发人员中，有一半也使用JavaScript。Python也经常与HTML / CSS，Bash / Shell，SQL，C / C ++和Java一起使用。

Flask和Django是Web开发人员中最受欢迎的框架。它们拥有相等的份额（每个份额约45％），远远落后于其他Python Web框架。

NumPy，Pandas，Matplotlib和SciPy是最受欢迎的数据科学框架和库。特定于机器学习的库，例如SciKit-Learn，TensorFlow，Keras等也很受欢迎。

AWS是Python开发人员最受欢迎的云平台，其次是Google Cloud Platform，Heroku，DigitalOcean和Microsoft Azure。

在2018年，参与DevOps的受访者数量明显增加（与2017年相比增加了8个百分点）。对于使用Python作为辅助语言的Python用户，DevOps已经超越了Web开发。

PyCharm的两个版本是最流行的Python开发工具。有趣的是，VS Code已从2017年的7％扩大到2018年的16％，使其成为Python开发中第二受欢迎的编辑器。其他流行的Python编辑器包括Vim，Sublime和Jupyter Notebook。

令人惊讶的是，几乎三分之二的Python开发人员选择Linux作为其开发环境OS。
