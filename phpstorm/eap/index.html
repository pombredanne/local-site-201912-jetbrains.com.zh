PHP 7.4支持

PHP 7.4现在已发布，PhpStorm 2019.3完全支持其所有新功能。IDE还提供检查以突出显示不赞成使用的内容以及用于升级代码库的快速修复。

类型属性

我们在2019.2版本中增加了对类型化属性的支持。在v2019.3中，快速修复程序可以处理所有极端情况，并且您可以通过按Alt-Enter并为该字段选择添加声明的类型来快速轻松地添加类型声明。PhpStorm将根据构造函数中的PHPDoc，默认值或参数类型声明自动检测类型。

箭头功能

PhpStorm完全支持PHP 7.4中的单行lambda函数的全新短语法。IDE为外部作用域提供了变量的补全功能，单击一下，PhpStorm将带您进入声明。

还提供了一种快速修复程序，可将单行lambda函数从旧语法转换为新的短语法。只需在匿名函数上按Alt-Enter ，然后选择将闭包转换为箭头函数即可 。

数值文字分隔符

在PHP 7.4中，您可以为大量数字添加下划线分隔符，以直观地设置格式并提高代码的可读性。

如果看到较长的数字，请将插入符号放在其上，按Alt + Enter ，然后选择添加数字分隔符 。这将在每十进制数字的第三个数字或十六进制，二进制和八进制的每个第四个数字之后添加下划线“ _ ”。

嵌套三元运算符

PHP 7.4弃用了嵌套的左关联性? :运算符。这意味着要么根本不使用嵌套的三元数，要么（如果必须）使用括号显式声明计算顺序。

PhpStorm 2019.3将突出显示不赞成使用的表达式，并将为您提供快速修复程序以阐明行为。

弃用

这个PHP版本不仅增加了新功能，而且还淘汰了许多旧功能。

PhpStorm 2019.3进行了相应的检查，并将突出显示需要注意的代码段。如果可能，您可以选择自动修复不赞成使用的行为。

协变量返回和协变量参数

PHP 7.4修复了多种不一致，包括一种影响类型系统的不一致。现在，当您使用从父类继承的方法时，现在可以为它定义一个更具体的返回类型，而不是具体的参数类型。

PSR-12

新的PSR-12代码样式标准通过一些新的PHP功能的附加规则和新准则扩展了以前的PSR-2。PhpStorm 2019.3具有预定义的PSR-12代码样式，IDE会建议您切换到它。您也可以通过“ 首选项” | “ 设置为...”操作手动切换到该菜单。 编辑器代码样式| PHP 。

PhpStorm还建议启用一组PSR-12代码样式检查，以检查某些违反规则的情况。

您可以通过按Cmd + Alt + L或在提交之前运行它来在编辑器中重新格式化代码。您还可以使用“代码”>“代码清理”来重新格式化整个项目或文件夹。

WSL

适用于Linux的Windows子系统是一个兼容性层，用于在Windows 10上本地运行Linux二进制可执行文件。PhpStorm现在支持在WSL环境中进行开发，即在Windows计算机上启动IDE但在开发中将WSL / Linux作为目标时。您可以在WSL中指定一个远程PHP解释器，并将其用于运行脚本，测试和Composer命令以及进行调试。

为PHP测试运行配置指定解释器

过去，您只能通过默认的项目PHP解释器执行测试运行配置，这可能会带来不便。

这已不再是这种情况。在PhpStorm 2019.3中，您可以选择已配置的任何解释器，并使用它运行测试： Docker ， Vagrant ，通过SSH ， WSL或其他任何方式–命名，选择就是您自己！

编辑器工具栏

每当您打开.http文件时，您都将获得一个工具栏，该工具栏使您可以快速访问所有常见操作：

  • 添加请求–从模板创建请求。
  • 从cURL转换–通过将cURL粘贴到编辑器中来创建请求。
  • 打开日志–查看已发送请求的列表并打开/比较响应。
  • 示例–探索方便的HTTP请求的集合。

动态变量

我们添加了三个动态变量，您可以在请求中使用它们：

  • $ uuid –生成一个新的UUID-v4。
  • $ timestamp –当前的Unix时间戳。
  • $ randomInt – 0到1000之间的随机整数。

使用大括号将这些变量插入需要的位置。

全部运行

您可以在一个.http文件中写入多个请求。如果您需要根据前一个请求的结果执行请求链，这将非常方便。

之前，您必须逐个运行每个请求，现在在PhpStorm 2019.3中，您只需单击即可运行所有请求！

折叠要求

如果要发送的请求的正文较长，例如，多部分表单数据或可能有很多标题，则可能需要折叠它以减少编辑器中的干扰。

通过单击装订线中的箭头图标，可以折叠整个请求，仅保留第一行或正文。

如果您要处理旧版代码库并希望清理内容，那么一个好的开始方法是删除无效代码。现在，PhpStorm将检测冗余代码，并突出显示可能从未使用过的类，类成员和函数。未使用的声明检查会考虑代码的动态用法，例如通过魔术方法。要检查整个项目的报告，请选择代码->检查代码… 。

重构，意图和快速修复

弦乐杂耍工具箱

我们添加了多个处理字符串的新意图。现在，您可以将字符串串联转换为插值，将串联或插值转换为sprintf调用，将串联的字符串复制到剪贴板，并执行许多其他操作。在字符串上按Alt + Enter查看可用的操作。

删除未使用的变量

PhpStorm 2019.1引入了一个新的快速修复程序，可帮助您摆脱出现在分配或闭包使用列表中的未使用变量。快速修复足够聪明，可以检测到赋值表达式可能有副作用的情况。

转到类重构

使用新的“移至类”重构，您可以将函数或常量传递到它们在逻辑上所属的类，然后PhpStorm会为您查找和修改所有用法。通过在“ 结构”工具窗口中选择多个功能，可以同时移动它们。

方法可能是“静态”检查

如果一个类的方法不使用任何实例引用，则PhpStorm会检测到它并提供一个方便的快速修复方法：在该方法上按Alt + Enter ，并将其设为静态。所有用法将自动更新。

码头工人

重用容器

对于基于Docker Compose的解释器，您现在可以选择docker-compose run或docker-compose exec来执行容器。如果您有一个沉重的容器并且不想在每次测试运行时都重新启动，则可以通过选择docker-compose exec选项来重用它 。或者，您可以将docker-compose run用于轻量级容器或不在守护程序模式下工作的容器（即，在它们启动后立即停止）。

的PHP

改进的自动补全

借助特殊文件.phpstorm.meta.php ，PhpStorm现在可以建议参数并更好地返回值 。这是为了解决以下情况：您希望看到一些建议的常量而不是整数或字符串之类的简单类型。或者，如果您希望某些函数返回某个常数。您还可以通过提供自己的.phpstorm.meta.php文件来改善PhpStorm中针对您的库或项目的建议。

返工进口

我们重新设计了与导入和使用FQN的名称空间有关的检查和意图操作。它们背后的主要思想是尽可能避免使用限定词。现在，PhpStorm将允许您简单地删除一个多余的限定符，或将其替换为相应的“ use”导入语句。同样，当您将一些代码粘贴到文件中时，PhpStorm会要求重用现有的别名。

代码清理

在PhpStorm 2019.1中，代码清理工具具有特定于PHP的意图：它可以通过添加'use'语句或从中删除不必要的部分来优化完整类名的出现 。它还可以使用PHP CS Fixer或PHP_CodeSniffer的phpcbf 自动修复代码样式问题。在VCS提交之前执行清理，或随时通过“代码”->“代码清理...”手动触发它。

网络技术

像往常一样， PhpStorm中提供了WebStorm的所有功能和改进，可以直接使用 ，也可以在资源库中使用免费的插件。

更新了文档和浏览器兼容性

CSS属性以及HTML标记和属性的文档（ F1 ）现在显示MDN 的最新描述和有关浏览器支持的信息，以及指向完整MDN文章的链接。另外，要检查目标浏览器版本是否支持您使用的所有CSS属性，您可以在首选项中启用新的浏览器兼容性检查。

改进了对Vue应用程序中TypeScript的支持

PhpStorm现在利用TypeScript语言服务以及它自己对.vue文件中TypeScript代码的TypeScript支持。这意味着您现在将获得更准确的类型检查和类型信息，能够使用服务提供的快速修复，并在TypeScript工具窗口中查看当前文件中的所有TypeScript错误。

Angular应用程序的新检查

对于Angular应用程序，PhpStorm添加了17个新检查，可帮助您在键入时检测到应用程序中特定于Angular的错误，并提出适当的快速修复方法。这些检查都可以在TypeScript和模板文件中工作，它们将检查您对绑定，指令，组件和许多其他东西的使用。

新的调试器控制台

在JavaScript和Node.js调试工具窗口中访问新的，经过改进的交互式调试器控制台！现在，它使用树视图显示对象，并且支持使用CSS设置日志消息的样式并使用console.group（）和console.groupEnd（）将它们分组。您还可以过滤掉任何类型的日志消息。

将Promise函数转换为异步/等待

您可以自动将使用.then（）和.catch（）调用返回Promise的函数更改为使用async / await语法的async函数。只需在函数名称上按Alt-Enter ，然后选择“转换为异步函数”即可。这不仅在TypeScript文件中，而且在JavaScript和Flow中都是可能的。

集成开发环境

最近的位置弹出

在这个新的导航弹出窗口中，您会发现最近访问过的带有上下文的代码点-前两行和后两行。在此弹出窗口中，所有位置均按时间顺序排列，最后访问的位置在顶部。要调出新的“最近的位置”弹出窗口，请按Cmd-Shift-E / Ctrl + Shift + E。输入任何文本以在上下文中搜索。

VCS改进

对于部分Git提交，新的“全部取消选中”复选框使您可以立即取消选中所有代码块，然后选择要提交的代码块。此外，现在可以在Git日志中直接使用修正，压扁和摘樱桃动作。

自定义UI主题

如果默认的白色和Darcula主题不足以满足您的需求，请尝试“深紫色”或创建自定义主题 ！现在可以配置IDE的任何元素的颜色，从图标到单选按钮再到箭头。根据您的喜好微调所有内容，并将其另存为新主题插件！

数据库工具

由于我们的DataGrip团队的同事，PhpStorm中的数据库工具有了一些不错的改进：

  • 支持新数据库：Greenplum，Vertica和Apache Hive
  • 代码完成支持CREATE和DROP的组合语句
  • 在MySQL和MariaDB中支持DEFINER属性
  • 在MariaDB中支持Oracle模式
  • 您现在可以为项目设置默认文件夹

其他改进：
