如何获取基于IntelliJ的IDE中支持的新RDBMS

人们经常要求我们将对他们最喜欢的数据库系统的支持添加到我们的IDE中。来自跟踪器的一些功能请求包括Vertica ， Snowflake ， Informix ， Apache Hive和Teradata等等。

尽管我们不提供构建此类插件的API，但我们愿意在这些系统的供应商的帮助下自行构建支持。

为此，我们需要以下内容：

SQL语法语法

在BNF注释文件中提供。语法工具包 ， ANTLR或Bison会更好。

内置功能的文档

以HTML格式提供，每个内置函数都有单独的HTML。例如： dbms-help.zip!/functions/convert.html ，显示为convert(..)函数：

<html>
<head><title>convert</title></head><body>
CONVERT() provides a way to convert data between different character sets. The syntax is:
<pre><code>
CONVERT(expr USING transcoding_name)
</code></pre>
</body></html>

数据库目录说明

一些具有其属性的对象。类似于BNF的表示法示例：

Root ::= Schema *
Schema ::= Table * | View * | ..
Table ::= TableColumn * | Index * | Key * | ForeignKey * | Check * | Trigger * | ..
TableColumn ::= Name | DataType | Default | NotNull | AutoInc | ..
View ::= SourceText | ViewColumn * | ..
ViewColumn ::= Name | DataType | ..

目录自省查询

—对检索主要属性（例如“ RetrieveViewsQuery” ， “ RetrieveTablesQuery”等）的每个架构子对象的查询。

—查询子对象和更详细的信息，例如“ RetrieveColumnsQuery” ， “ RetrieveSources”等。

返回的数据列的名称应与数据模型中的属性名称匹配。

这些查询用于在两种情况下检索数据库的结构： 完全自省和增量自省。

—完全自省模式：为了获得更好的用户体验，DataGrip首先检索架构，然后检索表和视图，然后检索更详细的信息。因此，完整的自省要分几个步骤执行，并且在每个步骤上都会更新UI。这就是为什么查询需要针对多遍执行进行优化的原因。

—增量自省模式：如果可以跟踪模式或任何其他对象的上次更新时间或事务ID，请告诉我们。增量自省逻辑用于立即动态更新架构。例如，当您在查询控制台中创建表时，该表应出现在数据库树中而不刷新整个架构。

某些数据库根据用户权限提供不同的目录，并且它们的性能可能会有所不同。请告知应该使用哪些来确保最佳性能。

JDBC驱动程序

直接链接到官方JDBC驱动程序以从IDE下载。我们也可以在我们这边托管驱动程序，但是需要您的许可才能这样做。

测试实例和数据

—泊坞窗映像（首选）和/或云实例。

—具有大量交叉引用的复杂模式DDL转储。

—我们将用来测试解析器的SQL / DDL / DML示例。如果您在内部使用了一些样本/摘要进行测试，请与我们分享这些样本/摘要，我们将不胜感激。
