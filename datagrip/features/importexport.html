导入/导出选项

汇入CSV

从文件导入数据

有一个专用的UI，用于将DSV（CSV和TSV）文件导入数据库。单击要导入数据的架构，然后从上下文菜单中选择“ 从文件导入... ”。然后选择存储数据的CSV文件。

您将看到“导入”对话框窗口。左侧面板用于格式说明：如果第一行是标题（单独的格式选项可用），请选择定界符，并指定文件中是否带引号。

在右侧，您会看到描述要创建的表的框架和结果数据预览。按Delete键从结果中删除一列。如果要从现有表导入数据，只需使用此特定表的上下文菜单选择“ 从文件导入”…

如果文件中有错误怎么办？提供了“将错误记录写入文件”选项。导入过程不会中断，但是所有错误的行都会记录在该文件中。

将CSV粘贴到数据编辑器

从Excel表粘贴数据。通常，您需要具有以DSV格式粘贴数据的功能。在DataGrip中，您可以定义所需的任何格式，也可以让IDE自动检测格式： 齿轮图标→粘贴格式 。

CSV编辑器

DataGrip可以将CSV文件编辑为表格。单击上下文菜单中的“ 按表编辑” 。

然后选择定界符，指定表是否具有标题，何时引用值等。可以使用即时表预览。

汇出资料

什么地方

表

表格可以导出到：
— 文件 。表的上下文菜单→将数据转储到文件。
— 另一个现有表。 表的上下文菜单 → 将表复制到 （或仅在表上按F5键 ）→选择现有表。
— 任何数据库供应商的任何数据源中的新表。 表的上下文菜单 → 复制表到 （或表上的F5 ）→选择目标架构。或者只是将其拖放到此处！此gif说明了如何将PostgreSQL表复制到SQLite数据源。

SQL查询

结果可以将SQL查询导出到文件 。语句上的上下文菜单 → 执行到文件 →选择格式。如果查询速度很慢，则很有用；如果导出结果集，查询将再次运行。在这种情况下，您只能运行一次。

结果集

结果集可以导出到：
— 文件 。转储按钮 → 归档 。
— 剪贴板 。转储按钮 → 至剪贴板 。
— 另一个现有表。 导出按钮 →选择现有表格。
— 任何数据库供应商的任何数据源中的新表。 导出按钮 →选择目标架构。

选拔

可以将数据编辑器中的选择复制到剪贴板：可以使用上下文菜单 → 复制或Ctrl + C以通常的方式进行选择 。

汇出格式

每次导出到文件或剪贴板时，都需要选择一种格式。通常，它是CSV，但在许多情况下，您需要JSON，XML甚至DML语句。从下拉列表中选择适当的数据提取器 。或创建自己的导出格式。

SQL查询

任何表或结果集都可以显示为一批INSERT语句。为此，请从下拉列表中选择“ SQL插入 ”。在某些情况下，它可能比编写INSERT + SELECT查询以将数据添加到另一个表更快。这一代知道自动增量ID 。

它也可以也可以呈现为一个批次的UPDATE语句，它可以帮助你修改数据。

CSV，TSV

有两个预建格式的列表 ： 逗号分隔值和制表符分隔值 。您可以基于DSV创建任何自定义格式。例如，它可以是Confluence Wiki Markup 。

HTML / XML

有一个您无法更改的预制提取器： HTML table 。还有脚本提取器： HTML-Groovy.html.groovy ， XML-Groovy.xml.groovy等。您可以修改这些提取器。

JSON格式

JSON-Groovy.json.groovy提取程序将表或结果集导出为JSON。

自订格式

对于更复杂的情况，请考虑使用脚本提取器。其中一些已经可用，例如CSV-Groovy.csv.groovy ， HTML-Groove.html.groovy等。在提取器菜单中选择“ 转到脚本目录 ”以导航到存储它们的文件夹。

这些脚本是用Groovy编写的，但也可以用JavaScript编写，通常位于Scratches和Consoles / Extensions / Database Tools和SQL / data / extractors中 。修改现有提取器或在此处添加自己的提取器。

要了解有关创建提取器的更多信息，请阅读本教程。

转储/还原工具

我们已经将DataGrip与mysqldump和pg_dump集成在一起。要转储对象，请使用“ 与...一起转储”上下文菜单选项。还可以从上下文菜单访问MySQL和PostgreSQL的还原工具。对于PostgreSQL ，可以使用pg_dump或psql来完成还原操作：在“ 还原”对话框中可以选择一种方法。
