代码见解

DataGrip使您的SQL编码尽可能简单有效。它可以完成关键字和标识符，生成SQL，检测SQL代码中可能存在的错误，并提供即时修复它们的最佳选择。

意向动作

一旦DataGrip找到了一种更改代码的方法，您可以通过按Alt + Enter来应用它。您将看到带有Intention Actions的弹出窗口，其中涵盖了从警告到优化建议的各种情况。一些最有用的功能包括：

  • 从INSERT语句创建表定义
  • 合格/不合格标识符
  • 引用/取消引用标识符
  • 展开通配符
  • 介绍子查询的别名
  • 将丢失的对象添加到数据库
  • 将许多AND运算符转换为一个IN运算符

代码完成

在DataGrip中，代码完成非常快。在键入时，您会得到一系列建议：我们的主要目标是在这里实现零延迟。完成是自动的，但是您可以随时按Ctrl + Space来调用它。它也是上下文感知的，这意味着它仅尝试建议您在任何特定时刻可能需要的对象和关键字。

了解有关代码完成的更多信息。

检查：错误

DataGrip可以检测SQL代码中可能存在的错误，并提供最佳的解决方案来即时修复它们。如果存在以下情况，IDE会通知您：

  • 未解决的对象：查询中的无效对象
  • 查询中的歧义名称
  • 集运算符中使用的查询中的列数不匹配
  • INSERT语句中的值数量不匹配
  • 您使用的游标未打开

检查：警告

有时，您的代码可以工作，但包含不应该存在的内容。在以下情况下，DataGrip会警告您：

  • 过程或函数中未使用的参数
  • 未使用的子查询项
  • 无法访问的代码
  • WHERE子句中的常量条件
  • 您将要运行的不安全语句
  • 字符串可能被截断

展开通配符

由于在许多情况下不建议使用通配符，因此DataGrip可以对其进行扩展以为您列出表的所有列。在发生冲突的情况下，标识符将是合格的。

参数信息

DataGrip还可以为您提供有关函数和过程的参数名称和类型的信息。要查看此工具提示，请按Ctrl + P。

INSERT语句的列名工具提示

为了帮助您插入有效值，IDE会提示相应列的名称和类型。通过按Ctrl + P调用它。

INSERT语句的列名提示

尝试启用设置→编辑器→常规→外观→显示参数名称提示 ：DataGrip将提示您有关INSERT语句中列名称的信息。
