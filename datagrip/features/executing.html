运行查询

查询控制台

根据我们的经验，查询控制台是编写常规日常SQL的最佳位置。每个数据源都有其自己的默认控制台。要打开它，请转到上下文菜单，然后单击“ 打开控制台” 。或使用快捷键F4 。

您可以在此处编写SQL，然后运行它并获得结果！它是如此简单。

如果出于任何原因要为数据源创建另一个控制台，也可以这样做： 上下文菜单→新建→控制台

模式切换器

创建多个控制台以同时管理大量查询。每个控制台都有一个数据库/模式切换器。如果您使用PostgreSQL，请在此处定义搜索路径。

执行选择

选择一些代码，然后仅执行该选择。请注意，所选查询按原样运行：DataGrip只是将此字符串发送到数据库并获取结果。当IDE错误地检测到查询中的错误时，此功能很有用。

执行选项

DataGrip提供了用于执行查询的智能选项。当光标位于查询中时，可以使用几个可能的选项来自定义执行。定义其预期的行为：执行最小的语句，最大的语句，要求您选择要执行的操作或执行整个脚本。

 

此外，还有执行动作。更改行为并分配快捷方式：它们是完全独立的。例如，您可以创建一个将执行整个脚本的操作，而不必从脚本中选择确切的查询。

就是这样。

唯读模式

可以在数据源属性中打开“ 只读” 。实际上，它同时打开两种只读模式： 相应的jdbc驱动程序中的只读模式和IDE级别的只读模式。

Jdbc驱动程序 只读模式将拒绝在MySQL，PostgreSQL，AWS Redshift，H2和Derby中运行修改查询。在其他DBMS中，它什么也不做。

这就是为什么我们有一个IDE级别的只读模式。它与jdbc级之一一起被激活。因此，在只读模式下，每个DDL或DML查询都将带有下划线。如果执行它，将会出现警告。按该栏上执行 ，如果你在你在做什么完全有信心这是可能的。

现在，我们还索引了函数和过程的源代码，并在后台构建了一个调用树。这意味着，如果一个函数执行另一个执行DDL或DML语句的函数（重复n次），您还将看到一条警告。

交易控制

定义在使用特定数据源时要使用哪种事务控制。该选项在数据源属性中可用。在自动模式下，您不需要提交任何执行commit语句的事务，而在手动模式下，您显然需要提交任何事务。

评估表达

这将帮助您查看数据，而不必将查询写入控制台。在其他基于IntelliJ的IDE中，对对象按Ctrl + Alt + F8可以快速评估。在DataGrip中，在查询的表上调用它以查看该表的数据。 列名上的Ctrl + Alt + F8将在预期结果集中显示该列的值。如果您对查询（或子查询）的关键字调用相同的快速评估，则弹出窗口将为您显示结果。请注意， Alt + Click也会对此起作用。

运行查询的历史记录

每个控制台的工具栏上都有一个按钮： 。按下它可以查看针对该数据源运行的所有查询的历史记录。速度搜索也可以！

另外，不要忘记每个文件的本地历史记录 。

完整的SQL日志

从字面上看，来自IDE的所有查询都记录在一个文本文件中。要打开此文件，请转到“ 帮助” |“帮助”。显示SQL日志 。

运行存储过程

DataGrip可以生成用于运行存储过程的代码。输入所需参数的值，然后单击“ 确定” 。

请注意，打开源代码时，可以通过单击工具栏中的“运行”按钮来运行该过程。或者只是从数据库树中过程的上下文菜单中使用执行… 。

不安全的查询

如果您使用不带WHERE子句的DELETE或UPDATE语句，DataGrip将显示警告提示。

解释计划

语句的执行计划直观地表示数据库为了返回查询所需的数据而执行的操作。它可以帮助您优化查询。

 

该计划可以图形方式显示为表格或图表。

用户参数

从另一个应用程序获得SQL查询时，可以对其进行参数化。DataGrip支持运行此类查询。

可在设置/首选项→数据库→用户参数中使用用于编辑参数模式的UI。正则表达式突出显示，您可以选择方言，这些模式有效。

结构图

对于每个控制台或SQL文件，您都有一个弹出窗口，其中所有语句均作为结构提交。您可以仅从此“ 结构”弹出窗口 （ Ctrl + F12 ）运行查询，当您需要在大脚本中运行多个查询时，这将非常有用。

查询结果

结果集

在DataGrip中，结果集是可修改的，因此您可以更改执行查询后返回的数据。它与数据编辑器相似，具有相同的可用选项，包括自动提交，添加/删除行，文本搜索和数据导航。

比较结果

可以在Diff Viewer中比较两个结果集。DataGrip将突出显示差异，并允许您通过Tolerance参数管理比较标准。例如，如果您希望两行的数据仅在同一列中不同，则认为两行相等，请在“ 公差”字段中输入相应的值。如果需要，可以从比较中排除任何列。

点击比较按钮在结果的工具栏上，然后选择要与之比较的另一个结果集。

结果选项卡可以命名

方便的结果功能：标签命名！只需在查询前使用注释即可。

如果仅希望某些注释成为选项卡名称，请使用设置中的将文本视为标题后的字段来指定前缀词。然后，仅该单词之后的单词将用作标题。

页面大小易于更改

要定义要从数据库中检索的行数，可以从结果集的工具栏中进行操作。
