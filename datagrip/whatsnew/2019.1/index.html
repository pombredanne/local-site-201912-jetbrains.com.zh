DataGrip 2019.1的新增功能

新的DBMS支持

在此发行周期中，我们集中精力支持多个新数据库。

Greenplum是一个基于PostgreSQL的分析数据库。它目前由Pivotal和开源社区开发。

Vertica是面向列的存储，旨在处理大量数据。

Apache Hive是建立在Apache Hadoop之上的数据仓库，用于提供数据查询和分析。

Snowflake是基于云的数据仓库。在2019.1中，DataGrip仅支持SQL方言，稍后将提供自省支持。

连接对话框

一般

我们已经重新排列了此处的字段，以使连接过程更加人性化。

连接类型字段是以前的URL类型 。它确定了此对话框的整个UI，这就是为什么它与驱动程序选择器一起位于顶部的原因。

“ 数据库”字段位于凭据之后，因为有时需要凭据才能提供数据库列表。该列表应通过Ctrl + Space调用

现在可以通过下拉菜单实现密码保存。它具有四个选项： 从不 ， 直到重新启动 ， 对于会话和永远 。对于Session ，可能是唯一需要说明的选项：它将保存您的密码，直到您与数据源断开连接。

另外，现在可以通过用户界面设置空密码；使用上下文菜单可以做到这一点。

现在， 测试连接结果是嵌入式的，可以通过专用的小按钮复制到剪贴板。还添加了故障排除页面的链接。

从2019.1开始，如果没有下载的驱动程序，则不会禁用`` 测试连接''按钮。当您单击Test Connection时 ，DataGrip将为您提供下载JDBC驱动程序。

选件

我们已经从“ 常规”标签中将“ 只读” ，“ 自动同步 ”和“ 交易”控制设置移到了此处。

新引入的设置 ：
– 每N秒运行一次keep-alive查询，将帮助您保持连接状态。您可以在驱动程序设置中为不支持的数据库定义自定义查询。
– N秒后自动断开连接 。很不言自明。
– 启动脚本是一个SQL查询，它将在每次建立连接时运行。请记住，如果未选中“ 单一连接”模式 ，则每个新控制台都会创建一个新连接。

重命名设置 ：
– 如果过时的DDL是旧时打开时的通知的新名称， 则发出警告 。

模式

对象过滤器已从“ 选项”选项卡转移到此处。

资料来源清单

一个小的增强： DDL数据源现在在连接对话框内的数据源列表中有其自己的组。

搜索和导航

最近的位置

有一个新的“ 最近的位置”弹出窗口，其中列出了您最近访问/更改的代码位置。它在位置前后以几行代码的形式显示为代码片段。

此弹出窗口的灵感来自“ 最近的文件” ，但具有按访问的代码位置进行搜索的功能。当您记住代码的内容但不记得代码的位置时，这应该派上用场。特别是当您使用多个查询控制台时：它们通常具有相似的名称！

按Delete / Backspace键从弹出窗口中删除位置。

该弹出窗口的默认快捷键是Ctrl + Shift + E。

如果您习惯按Ctrl + Shift + E来调出最近更改的文件 ，请改用双Ctrl + E。

寻找路径

DataGrip可以搜索对象的源代码。仅当您选择“ 所有位置”范围时，此选项才可用。我们已经稍微清理了该用户界面，并删除了“ 模块”和“ 项目”选项卡。

一个新的范围， 附加目录 ，可用。如果要在“ 文件”工具窗口中搜索文件，请使用它。

转到弹出窗口

现在，所有导航操作都可以从此处开始。

例如，如果要搜索表，请在此处查看可以打开的内容：
表格的DDL ： Ctrl + B。
表的数据 ： F4 （ Enter对表执行相同的操作） 。
修改表对话框： Ctrl + F6 。
在…中找到……弹出窗口： Alt + F1 （使用它在数据库树视图中查看对象） 。
快速文档弹出窗口： Ctrl + Q。
SQL生成器 ： Ctrl + Alt + G。

还有更多与您在此处看到的对象有关的动作！

另外，不要忘记“ 数据库” |“ 导航”中的导航设置 。 一般随时创建自己的流程，因为DataGrip现在可能涵盖了所有可能的用例。

代码见解

完成合并的项目

我们在完成列表中为CREATE和DROP语句添加了一些组合选项。例如，当您开始输入CREATE ，可以选择CREATE TABLE 。

另一个好处是您可以使用缩写！

新检查

应该使用 GOTO语句和Named参数的 用法默认情况下，检查是关闭的，因为只有某些开发人员希望报告这些情况。

GOTO语句检查的用法有三种选择：
-报告所有GOTO声明；
-报告使用GOTO语句退出循环的情况；
-报告向后的GOTO语句；

应该使用命名参数 。一些开发人员更喜欢这样做，因为它使代码更具可读性。

如果在过程或函数中未打开使用的游标， DataGrip也会警告您。

处理文件

现在， “设置/首选项” |“ 默认目录”选项中有一个。外观与行为|系统设置 。它将定义默认情况下将在何处创建新项目。

控制台文件的“ 另存为...”操作：
1。提供默认的项目目录。
2。保留最后选择的文件夹。

“ 文件”工具窗口中的文件夹提供了一个新的操作“ 分离目录” 。从上下文菜单中使用它或分配一个快捷方式。

您可以从上下文菜单中格式化文件而无需打开它们。

数据库树视图

我们改进了IBM DB2的数据库自省功能 。现在，您应该在数据库树中看到以下新对象：触发器，方法，模块，数据类型，变量，条件，序列，检查约束，角色和全局临时表。

现在，树将单独保存上下文 ，因此您始终会在顶部看到数据源名称。

我们还为一些尚不支持的数据库添加了图标 。以前，如果您有许多不受支持的数据库的数据源，那么它们都可能在“数据库”工具窗口中造成混乱。我们希望现在看起来更好。

您可以使用所有其他新的自定义图标 。选择驱动程序设置的“ 选项”选项卡上的图标。

定义者支持

我们在MySQL和MariaDB中添加了对DEFINER属性的支持。

首先，您可以选择SQL生成器是否应在生成的代码中添加DEFINER属性。

其次，您可以在迁移期间忽略DEFINER属性。
