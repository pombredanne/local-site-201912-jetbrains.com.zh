轻松启动

开始使用CLion之前，请查看此《 快速入门指南》 。

配置工具链

为CLion中的不同项目配置和使用不同的工具链，甚至在一个项目内部为不同的配置进行配置和使用。

您可以更改CMake可执行文件，Make，C和C ++编译器和调试器。请注意，您可以使用捆绑版本或自定义版本的CMake和调试器可执行文件。

在Linux和macOS上，您可以使用GCC或Clang作为编译器，在Windows上–您可以在MinGW ， MinGW-w64 ， Cygwin和MSVC之间进行选择（请参见下文）。

使用远程工具链在远程主机上构建，运行和调试。

Windows Linux子系统

在Windows上，使用WSL / WSL2工具链。在Windows上工作时编译，运行和调试Linux二进制文件。在Valgrind Memcheck下测试目标，以检测可能的错误和泄漏。

Microsoft Visual C ++编译器

CLion支持并自动检测Visual Studio 2013、2015、2017和2019中的Microsoft Visual C ++编译器。

明确打开Microsoft Visual C ++工具链的实验性调试器是可用的：调用“维护”对话框（ Shift + Ctrl + Alt + / ）|实验功能，然后选择cidr.debugger.lldb.windows对其进行控制。由JetBrains团队在LLDB之上实现的调试器，并捆绑到CLion中。要启用对本地可视化工具的捆绑支持，请使用“设置” |“设置”。构建，执行，部署|调试器数据视图|为LLDB启用NatVis渲染器。

项目模型

CLion中的所有工作都是在项目上下文中完成的，该项目是编码帮助，批量重构，编码样式一致性和其他智能功能的基础。

CLion支持：

  • CMake的
  • 新的Gradle C ++插件，
  • 编译数据库项目。

对于这三个方面，CLion都提供了完整的代码见解。

对于CMake，您可以启用各种CMake生成器（Ninja，Xcode，Visual Studio等）。

开始一个新项目

要启动一个全新的项目，请使用File | New Project...生成示例项目。对于C和C ++，CLion将创建CMake项目并使用必要的信息预填充根CMakeLists.txt。

打开一个现有项目

要从CLion中的现有项目开始，请选择以下选项之一：

  • 对于CMake项目：找到根CMakeLists.txt文件，CMakeCache.txt文件或CMake生成目录（后两个选项仅适用于Makefiles生成器）。
  • 对于Gradle C ++：从项目源打开build.gradle文件。
  • 对于编译数据库：使用compile_commands.json文件（请参阅此方法列表以获取项目的编译数据库）。
  • 对于其他项目模型：在CLion中打开包含C / C ++源代码的单个文件或文件夹，请注意，在这种情况下，代码洞察力将受到限制。
  • 或者，从现有源导入C / C ++项目，CLion将为您创建CMakeLists.txt。

新增档案

可以使用“新建”菜单将新文件添加到项目中。在项目树视图中，按Alt + Insert或在上下文菜单中选择“新建”。

  • 单击“ C ++类”以生成带有标题和类存根的.cpp文件源文件。
  • 单击“ C / C ++源文件”以生成一个简单的源文件，还可以选择包含相关的标头。
  • 单击C / C ++头文件以生成一个简单的头文件。
