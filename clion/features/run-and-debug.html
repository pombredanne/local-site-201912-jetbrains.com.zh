运行和调试

构型

对于项目中的每个目标（对于CMake和Gradle项目），CLion都可以生成可以构建和运行的配置，并允许您在单独的UI对话框中将外部工具配置为在构建阶段之前/之后运行。

配置模板

使用配置模板创建用于单元测试，远程调试或常规应用程序运行的配置。

运行配置

您可以为您选择的任何配置更改可执行文件，甚至可以通过将该值更改为“未选择”来使配置不可运行。

通过按Ctrl + F9来构建项目，然后使用Shift + F10运行当前选择的配置。

调试配置

要进行调试，请按Shift + F9 。

为了帮助您在调试过程中检查代码的状态，CLion为您提供了许多有用的快捷方式，例如“跳入/跳入（ F8 / F7 ）”，“跳出（ Shift + F8 ）”或“运行到光标”（ Alt + F9 ）。

自定义构建和运行

独立于构建系统的构建目标使您可以添加自定义目标并配置用于构建/清理目标的任何工具。

要从CLion运行和调试自定义目标，请使用“自定义构建应用程序”模板创建相应的“运行/调试”配置。

重新编译一个文件

要在使用CMake或编译数据库项目时加快在CLion中的开发周期，请使用Recompile（ Shift + Ctrl + F9 ）。它为选定的文件重新运行编译，而不会触发整个项目的重新编译。

调试/发布构建类型

使用CMake时，请在工具栏上的运行配置切换器或“运行”（ Shift + Alt + F10 ）/调试（ Shift + Alt + F9 ）配置切换器弹出窗口之间切换CMake配置文件（和内置类型）。

输入输出

CLion使用PTY作为I / O单元，这意味着即使在Windows上，也可以期待与在终端中运行项目类似的行为。

远程开发

从Windows，Linux或macOS客户端计算机到Linux远程主机进行远程开发，仅适用于基于CMake的项目。当前设置假定源代码位于本地计算机上，CLion会将其同步到远程计算机。

要开始进行远程开发，请在“设置/首选项” |“设置”中创建一个远程工具链。构建，执行，部署|工具链。提供远程主机地址和凭据，然后在一个或多个CMake配置文件中使用工具链。重新加载CMake之后，您就可以开始在CLion中远程处理项目了。

除错

CLion与所有平台上的GDB后端以及macOS和Linux上的LLDB集成。

附加到本地流程

CLion允许您通过使用进程或进程名称附加到进程上来调试进程，这些进程可以在同一台计算机上运行但不能从IDE启动。受益于CLion的内置调试器UI，用于Linux / Windows上的GDB和macOS / Linux上的LLDB的本地进程。

远程GDB调试

在可执行文件在gdbserver下的一台计算机上运行后，您可以从另一台计算机上的CLion与GDB连接到该文件，并使用CLion调试器UI的所有优点检查代码。

使用特殊的GDB远程调试或远程GDB服务器配置模板为远程连接提供设置。

断点

CLion包含一个集成的调试器，可帮助您检查代码的执行情况。您可以在CLion中的几种断点类型中进行选择（所有断点都可以在单独的对话框中查看（可通过Ctrl + Shift + F8进行访问 ））：

  • 行断点是调试代码的最简单方法。通过单击行的左侧装订线，只需用鼠标设置一个断点。
  • 当特定功能开始执行时， 符号断点可帮助您停止程序执行。利用代码完成功能，按名称查找符号，以转到适当的功能。
  • 异常断点有助于捕获任何抛出的异常，而不会造成任何麻烦。

手表与评价

所有可用变量（包括STL容器内容）的描述都将放置在“调试”工具窗口的“变量”选项卡中。您可以在那里观察甚至更改任何值，而无需中断当前的调试会话：

“ GDB / LLDB”选项卡提供对GDB / LLDB控制台的访问，而“框架”和“监视”视图可帮助更详细地调查任何问题。另外，选择观看符号时，您可以受益于自动补全。

另外，您可以在调试会话期间评估任何表达式，只需按Alt + F8即可 。

内联变量视图

使用此功能，您可以直接在编辑器中查看当前变量值（变量声明旁边）。无需在调试工具窗口中切换到变量选项卡！

拆卸视图

当源不可用时，进入反汇编代码。阅读在编辑器中正确突出显示的程序集，并逐步解决该问题以进行研究。

要进入反汇编代码，请使用“ 强制步入” （ Shift + Alt + F7 ）。通常的单步执行 （ F7 ）将仅跳过所有没有源的帧。

支持GDB和LLDB。

记忆体检视

内存视图在调试模式下将内存转储到指针后面。要调用它，请在调试器的变量视图中选择一个指针变量，然后按Ctrl + Enter 。

在步进过程中，如果打开了“内存视图”，则CLion将突出显示所选内存块中发生的内存更改。

从“内存视图”导航到内存中的特定地址。使用代表地址的十六进制值，指针变量，或调用您感兴趣的任何变量的地址（ & ）。

代码完成将帮助您使用已知符号。

用于Microsoft Visual C ++工具链的实验性调试器

明确打开Microsoft Visual C ++工具链的实验性调试器是可用的：调用“维护”对话框（ Shift + Ctrl + Alt + / ）|实验功能，然后选择cidr.debugger.lldb.windows对其进行控制。

由JetBrains团队在LLDB之上实现的调试器，并捆绑到CLion中。要启用对本地可视化工具的捆绑支持，请使用“设置” |“设置”。构建，执行，部署|调试器数据视图|为LLDB启用NatVis渲染器。
