爪哇

Java 12

IntelliJ IDEA 2018.3初步支持Java 12！现在在IDE中预览Raw String Literals （ JEP 326 ），并利用大量有用的检查和快速修复。例如，将多行字符串文字转换为原始字符串文字，反之亦然，检测冗余的前后引号（`）等。学到更多

在此处阅读法律声明

请注意，Java 12尚未发布。您必须接受beta Java规范的法律声明中的条款才能启用对Java 12的支持。在Java社区流程（JCP）下开发的早期草案规范的实现仅可用于测试和评估目的，并且与JCP的任何规范都不兼容。

检测更多重复

即使在更复杂的情况下，例如变量名不同，IntelliJ IDEA现在也可以快速发现重复项。IDE将在diff查看器中显示重复项，因此您可以将它们并排比较。

Java Stream API改进

IDE会检测到在随后的min调用之前进行不必要的排序调用的情况。还提供了删除冗余排序呼叫的快速修复程序。

数据流增强

我们创建了一个新的基于数据流的检查，称为Condition由更多condition覆盖 。它可以检测到第二个条件中没有第一个条件的情况。IDE提供了一个快速修复程序来消除这种冗余情况。

抑制检查的冗余用法

IntelliJ IDEA现在可以检测@SuppressWarning注释或// noinspection行注释或/ ** noinspection * / JavaDoc注释的冗余用法。当禁止检查不再解决关联的方法体，类或语句中的任何警告时，IDE不仅会向您发出警告，而且还将提供快速修复程序来删除这些不必要的注释和注释。

编辑

多行TODO注释

IntelliJ IDEA 2018.3现在支持多行TODO注释 ，现在在编辑器中突出显示了第一行和所有以后的TODO注释行，并在TODO工具窗口中列出了这些行。只需从多行TODO注释的第二行开始添加缩进，IDE就会将其与普通注释区分开。

禁用特定文件和文件夹的代码格式

我们使锁定某些文件免受格式化，导入优化和代码重新排列的影响成为可能，实际上是从涉及格式化的任何操作中明确或隐含地锁定了文件。只需创建一个作用域并向其中添加任何不能格式化的文件！

新缩进状态栏

IntelliJ IDEA在新的缩进状态栏中显示当前文件中的缩进大小。如果文件的缩进与项目的当前设置不匹配，则会在缩进视图中添加一个星号。现在，您可以在状态栏弹出窗口中选择控制当前缩进的操作。

EditorConfig支持方面的改进

IDE现在为EditorConfig文件提供语法突出显示和代码完成功能。您还可以使用新的缩进状态栏弹出窗口轻松导航至正在使用的EditorConfig文件。

版本控制

GitHub拉取请求

IntelliJ IDEA初步支持GitHub Pull Requests。从全新的GitHub Pull Requests工具窗口预览GitHub存储库中的所有Pull Request 。使用diff预览更改，或通过上下文菜单从“拉取请求”创建本地分支，然后将其合并为常规分支。

“直到这里的历史”可以显示全部历史

以前，“ 到此为止的历史记录”操作仅显示文件到选定提交为止的历史记录，但是现在，通过切换到工具栏上的“ 显示所有分支” ，它可以显示文件后来发生的情况。如果以后删除该文件，这可能特别有用，因为要找出这种情况并非易事。

VCS日志中的高级导航

您现在可以使用前进和后退导航操作，而你是在VCS登录您导航后，从提交哈希在VCS日志选项卡中的提交，或者您使用转到散列/分支/标签动作之后。使用左右箭头键导航到子提交或父提交。

忽略空格

现在，IDE可以在合并时忽略空格更改。在“ 合并修订的版本”对话框中，有一个新的“ 忽略”下拉菜单，其中包含在合并时隐藏或修剪空白更改的选项。

将文件从一个Git分支复制到另一个

IntelliJ IDEA 2018.3允许您通过方便的新``从分支获取''操作轻松地将文件从一个Git分支复制到另一个Git分支，该操作可从`` 比较分支''对话框的``文件''选项卡中获得。

提交对话框中的分支名称

IDE在“ 提交”对话框中立即显示您要将文件提交到的分支名称。我们已经为Git，Mercurial和SVN实现了此功能。

在“文件历史记录”选项卡中预览差异

现在，可以在“ 版本控制”工具窗口的“ 文件历史记录”选项卡中使用Diff预览 。

Git子模块支持

IntelliJ IDEA 2018.3现在支持Git子模块！克隆或更新项目，提交更改，查看差异并解决冲突。

搜索和替换

新搜索无处不在

IntelliJ IDEA 2018.3引入了一种更好的新方法来浏览项目和IDE。请热烈欢迎您重新设计“到处搜索” ！实际上，它不仅包含了现在所有位置的导航对话框，而且还不仅仅是现在搜索所有位置： 随处搜索 ， 查找操作 ， 转到类 ， 转到文件 ， 转到symbol 。请注意，所有这些导航对话框仍然可以像以前一样通过其自己的键盘快捷方式进行访问。

在“在路径中查找”中搜索多行片段

现在，可以在“在路径中查找” /“在路径中替换”对话框中搜索/替换多行片段。您不再需要使用正则表达式来执行多行搜索。

新的“结构搜索和替换”对话框

IntelliJ IDEA 2018.3提供了更新和简化的`` 结构搜索和替换''对话框。现在，自动补全功能可在搜索字段中的预定义模板中工作。您可以在搜索图标下的下拉菜单中查看搜索历史。合并范围配置现在也变得更加容易和灵活。复杂的“ 编辑变量”表单已替换为过滤器面板，您可以在更新的对话框中找到该面板。

插件首选项页面

我们已经更新了“首选项” （ “设置” ）中的“ 插件”页面 ，以使管理，安装，卸载和更新插件更加容易。现在，您不仅可以按名称搜索插件，还可以按标签搜索插件。它变得更好，因为您现在还可以按下载，名称，评级，功能或更新对结果进行排序。

运行任何东西

运行任何东西

IntelliJ IDEA 2018.3带有新的集中式方式，可通过新的`` 运行任何内容''操作（Double Ctrl）在IDE中运行命令。使用新操作启动运行配置，包括Maven和Gradle，运行Gradle任务或运行终端命令。

调试任何东西

只需按Shift键，即可将“ 运行任何内容”对话框切换到“ 调试任何内容”模式。

科特林

与IDE捆绑在一起的Kotlin插件已更新至v1.3。

Kotlin 1.3支持

就在最近，Kotlin对语言，工具和库发布了重要版本。IntelliJ IDEA可以帮助您将Kotlin项目迁移到Kotlin 1.3，并以过时的代码执行所有必需的更改，以使其与库的最新状态兼容。

多平台项目支持方面的增强

在Kotlin 1.3中，对多平台项目的模型进行了完全重新设计，以提高表达性和灵活性，并使共享通用代码更加容易。IntelliJ IDEA提供了一组涵盖最常见用例的项目示例。

新的Kotlin检查和快速修复

自IntelliJ IDEA 2018.2发布以来，Kotlin插件进行了30多项新检查，快速修复和意图，可帮助您更有效地编写代码。

春季及春季靴

Spring Boot 2.1支持

IntelliJ IDEA 2018.3完全支持最近发布的Spring Boot 2.1。如果已指定，则使用默认@DataSizeUnit验证使用@DataSize注释的配置值。

Spring Initializr的改进

现在，当您使用Spring Initializr创建项目时，IDE将建议安装或启用适当的插件，以确保存在对所有选定技术的支持。

对Kotlin的更好的JPA和Spring Data支持

现在，IDE可以自动将JPQL插入查询字符串，从而提供实体名称和参数的完成。您可以在Kotlin中编写Spring Data接口，并且IntelliJ IDEA将了解您使用的实体。IDE为方法名称提供了智能完成，并为参数提供了快速修复。

辅助功能

高对比度主题

我们致力于为所有用户提供出色的体验。借助IntelliJ IDEA 2018.3，我们推出了一些重要的可访问性改进，例如全新的高对比度主题。

更好地支持屏幕阅读器

我们改进了对屏幕阅读器的支持，使他们能够读取行号，VCS批注，调试器和装订线图标。

显示影片

JVM调试器

附加到Java进程

附加到没有调试代理启动的Java进程。附加到进程后，您将能够查看当前线程的状态和变量值，并使用内存视图。

如果要将调试器附加到本地进程，可以使用方便的新的“ 附加调试器”操作，该操作在“ 运行工具”窗口中可用

自动重新启动侦听远程连接

现在，在调试远程进程时，IDE可以自动重新启动配置。只需启用新的自动重新启动选项即可进行远程调试配置。

远程JVM中的异步堆栈跟踪

IntelliJ IDEA 2018.3现在在远程JVM中支持异步堆栈跟踪。要开始远程使用代理：

  • 将/lib/rt/debugger-agent.jar复制到远程计算机。
  • 将-javaagent：debugger-agent.jar添加到远程JVM选项。

现在，您可以在“ 异步注释配置”对话框中定义自己的异步注释 。

删除所有断点的操作

IntelliJ IDEA 2018.3附带了方便的新操作，这些操作将删除项目中的所有断点或文件中的所有断点。您可以在首选项（设置）|设置中为这些快捷方式分配自己的快捷方式。 按键图|调试器动作 。

马文

现在，可以将所有构建和运行操作委派给Maven。转到首选项（设置）|构建，执行，部署|生成工具| Maven | Runner并选择新的Delegate IDE构建/运行操作到maven选项。

终奌站

对嵌入式终端进行了改进，因此当您关闭项目或IntelliJ IDEA时，现在IntelliJ IDEA 2018.3将保存终端选项卡名称，当前工作目录和外壳历史记录。

我们还重新设计了终端选项卡，使其看起来与其他工具窗口选项卡相似。

运行配置

Java运行配置中的宏

IntelliJ IDEA现在允许您在“应用程序”和“ npm”运行配置中使用宏。您可以在“ 程序参数”字段中输入预定义的宏。

能够使用文本文件作为输入

如果要使用文本或日志文件作为输入，只需在“ 运行/调试配置”对话框中选择“ 重定向输入自”选项，然后指定包含输入数据的文件的路径。

默认情况下，运行配置仅作为单个实例运行

默认情况下，运行配置现在仅作为单个实例运行。但是，如果要并行运行同一运行配置的多个实例，则可以在“ 运行/调试配置”对话框中启用“ 允许并行 运行”选项。

JavaScript和TypeScript

用JavaScript自动导入

IntelliJ IDEA现在可以在JavaScript文件中自动添加来自项目依赖项的符号导入。只要程序包中有TypeScript定义文件，或者程序包包含以ES模块编写的源代码，此方法就起作用。

改进的Angular支持

Angular模板中的变量，管道，异步管道和模板引用变量的代码完成和转到定义现在更加准确。

支持Node.js工作线程

现在，您可以在IntelliJ IDEA中调试Node.js工作者 –只需确保您使用的是Node.js 10.12或更高版本以及--experimental-worker标志。IDE为工作线程的API提供代码完成。

ESLint和TSLint具有更大的灵活性

您可以从linter的配置文件中覆盖严重性级别，并以错误或警告的形式查看来自linter的所有问题。

对于TSLint错误，请按Alt-Enter，然后按右箭头键。现在，您可以选择为此规则添加特殊的TSLint禁用注释。

Kubernetes

支持Helm资源文件

现在，IDE可以正确解析Helm资源模板文件，并为您提供众所周知的编辑支持：代码完成，重命名重构以及检查和快速修复。

在Helm资源文件中导航

通过IDE，您可以从值的用法导航到图表的values.yaml文件中的声明。您可以使用装订线图标在标签定义及其选择器之间导航。此外，values.yaml文件中还有新的装订线图标，可用于导航到覆盖/覆盖值。

Helm模板结果预览

现在，IDE可以在diff中预览Helm模板渲染的结果。只需选择新的Kubernetes |上下文菜单中的“ 舵模板”操作。

头盔依赖更新

新的Helm依赖关系更新操作可用于下载外部tgz依赖关系（或更新现有的依赖关系）并将其显示在项目树中。

数据库工具

Cassandra数据库

在此版本中，我们增加了对NoSQL数据库Cassandra的支持 。

SQL代码完成方面的改进

现在，代码完成功能适用于：GROUP BY中的非聚合字段，所有列都列在SELECT，MERGE和INSERT INTO表变量中，存储过程的命名参数，SUM（）和AVG（）中的数字字段，FILTER（在哪里）子句以及SQLite中的字段类型。

介绍表别名

现在，您可以使用Introduce表别名操作直接在表上创建别名，该别名将自动替换表名的用法。

单连接方式

每个新的控制台都意味着一个新的连接。但是在IntelliJ IDEA 2018.3中，可以仅将一个连接用于数据源，然后让所有控制台都使用该单个连接。您可以查看数据库树中的任何临时对象。另外，可以在不同的控制台中使用同一事务。要打开它，请转到数据源属性|选项|单连接模式 。
