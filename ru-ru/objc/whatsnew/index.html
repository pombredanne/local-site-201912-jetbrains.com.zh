2019.2 (24 июля)

Swift

В AppCode 2019.2 появилась начальная поддержка Swift 5.1, включая обертки свойств (property wrappers), статические индексы и многое другое.

Сворачивание замыканий

Сворачивание кода теперь доступно для замыканий в Swift, что делает код более читаемым.

Подсветка ключевых слов

Ключевые слова Swift подсвечиваются моментально, даже если индексация и кэширование еще не завершены.

Объединение строк

Объявление и инициализацию переменной теперь легко объединить с помощью действия Join Lines (⌃⇧J).

Действие Create from usage

Создавайте элементы перечисления Swift с помощью действия Create from usage (⌥⏎) из места, в котором они используются.

Find Usages (поиск использований)

Поиск использований стал еще быстрее благодаря удобной сортировке в окне Find Usages.

Отладка

Просмотр кода на дизассемблере

При отсутствии файлов с исходным кодом используйте действие Force Step Into (⌥⇧F7) для просмотра дизассемблированного кода (или просто выберите нужный фрейм в окне Debug ).

Автодополнение кода

В консоли отладчика теперь доступно автодополнение команд LLDB (⇥).

Покрытие кода

Запускайте модульные тесты с анализом покрытия, просто нажав кнопку Run With Coverage.

Результаты анализа покрытия отображаются в левом поле редактора.

Нажмите на цветовой индикатор покрытия, чтобы увидеть количество вызовов данного участка кода или скрыть покрытие.

Посмотреть статистику покрытия кода можно на панели Project или в специальном окне.

Сборка, запуск, отладка и тестирование во время индексации

Сборку, запуск, отладку и тестирование приложения можно выполнять, не дожидаясь завершения индексации и кэширования.

Интеграция

Поддержка скриптов командной оболочки

Для скриптов командной оболочки теперь доступна помощь при написании кода.

Плагин TextMate

Базовая подсветка синтаксиса и автодополнение кода теперь доступны для более чем 20 языков благодаря поддержке грамматик TextMate.

Окно коммита

Благодаря усилиям команды IntelliJ Platform выполнять коммит изменений теперь можно прямо из вкладки Local Changes. Чтобы активировать эту функциональность, установите флажок Commit from the Local Changes without showing a dialog в разделе Preferences | Version Control | Commit Dialog.

2019.1 (10 апреля)

Swift

AppCode 2019.1 поддерживает все нововведения Swift 5, включая атрибут @dynamicCallable, возможность записывать строки, в которых кавычки и обратные слэши используются как обычные символы, identity keypath и многое другое.

Переход к объявлению

При использовании инициализаторов, чтобы перейти к объявлению класса с помощью Go To Declaration (⌘B или ⌘Click), поместите курсор перед круглыми скобками. А чтобы перейти к объявлению инициализатора — внутри скобок.

Многострочные литералы

Преобразовывайте однострочные литералы в многострочные простым нажатием ⏎.

Перемещение выражений

Для перемещения циклов, условий, методов, классов, функций и других выражений вверх или вниз по коду используйте действие Move Statement Up / Down (⇧⌘↑/↓).

Rename

Рефакторинг Rename теперь корректно работает для переопределенных методов и методов родительского класса.

Анализ кода

Все исходники, сгенерированные в папке DerivedSources (модели CoreData и Intents), теперь обрабатываются корректно. Улучшена работа инспекций SourceKit, что позволило избавится от многих ложных ошибок и предупреждений.

Objective-C/C/C++

Правила именования

В AppCode появилась возможность настройки правил именования для Objective-C/C/C++. Выбранные параметры будут использоваться при автодополнении, генерации кода, рефакторингах и быстрых исправлениях. Выберите одну из стандартных схем или настройте свою в меню Preferences | Editor | Code Style | C/C++/Objective-C | Naming Convention.

Запуск и отладка

Присоединение к процессам на устройстве

AppCode теперь умеет присоединяться к процессам, запущенным на устройстве (Run | Attach to Process).

Настройки языка и региона приложения

На вкладке Options в окне Edit configurations… можно выбрать язык и регион приложения.

Выполнение программы до строки с курсором

Действие Run to Cursor можно вызывать, кликнув по номеру нужной строки в режиме отладки.

Отключение загрузки значений переменных

Чтобы ускорить пошаговое выполнение программы, теперь можно запретить пересчет переменных при отладке с помощью действия Mute Variables в окне Debug.

Сборка

Мы исправили ряд проблем интеграции с Xcode — инкрементальные сборки стали быстрее.

Темы интерфейса

Благодаря разработчикам платформы IntelliJ темы интерфейса для AppCode теперь можно создавать самостоятельно. Скачайте понравившуюся тему из репозитория плагинов или сделайте свою!

Диалог Recent Locations

В новом диалоге Recent Locations (⇧⌘E) можно просмотреть места в коде, которые вы недавно посещали. Для просмотра недавно измененного кода нажмите это же сочетание клавиш дважды.

Система контроля версий

Визуальное сравнение файлов

Теперь можно менять местами сравниваемые файлы при использовании действий Compare with…, Compare with the clipboard и Open Blank Diff.

Частичные коммиты

Добален флажок для включения в частичный коммит и исключения из коммита всех строк кода.

Fixup и Squash

Быстро создавать коммиты с сообщениями fixup! и squash! теперь можно прямо из контекстного меню Git Log.

2018.3 (28 ноября)

Swift

В версии 2018.3 появилась поддержка новых возможностей Swift 4.2, включая обработку будущих кейсов перечисления и новые диагностические директивы компилятора.

Действия Generate

В AppCode появились новые действия генерации (⌘N), которые помогут создать инициализаторы, description и debugDescription, hashValue и оператор сравнения.

Навигация к определению типа

Чтобы перейти к определению типа, просто нажмите ⇧⌘B.

Подсветка точек выхода

AppCode теперь подсвечивает точки выхода при установке курсора на ключевых словах break,continue, try, throw или return.

Форматирование

Исправлено формирование отступов для многострочных строковых литералов.

Добавлены настройки форматирования кода внутри препроцессорных директив.

Поддержка Swift Package Manager в CLion

Разрабатывайте серверные проекты в CLion с помощью плагина Swift — теперь проекты Swift Package Manager поддерживаются напрямую на macOS и Linux.

Objective-C/C/C++

Change Signature

Рефакторинг Change Signature теперь корректно обрабатывает значения аргументов, используемые по умолчанию в объявлениях методов/функций.

Быстрый просмотр документации

Макроподстановки в окне Quick Documentation (F1) форматируются и подсвечиваются корректно.

Многострочные TODO-комментарии

В AppCode появилась поддержка многострочных TODO-комментариев: достаточно добавить отступ для каждой строки кроме первой. Для быстрого просмотра текста TODO-комментария нажмите ⌘6.

Проекты с несколькими таргетами

Подсветка синтаксиса и автодополнение теперь работают корректно для одноименных классов из разных таргетов.

CocoaPods

Зависимости CocoaPods были исключены из проверок, рефакторингов и форматирования — так вы сможете работать быстрее.

Смешанный код

Исправлены проблемы при переименовании конструкций со swift_name в Objective-C и перечислений в Swift и Objective-C.

Производительность

Благодаря исправленным проблемам производительности IDE больше не будет зависать при вызове диалогов, связанных с запуском тестов (например, окна Run Configurations с тестовыми конфигурациями), при запуске наборов тестов и при навигации к тестовому коду.

Запуск и отладка

Расширения iOS

AppCode теперь умеет отлаживать расширения iOS так же, как это делает Xcode.

Консоль LLDB для Swift

Теперь в консоли LLDB для Swift работает автодополнение.

Плагин Features Trainer

AppCode предлагает множество полезных действий и удобных сочетаний клавиш, но для знакомства с ними требуется время. Чтобы вам было проще освоить все самое необходимое, мы разработали обучающий плагин IDE Features Trainer, который вы найдете в меню Preferences | Plugins | Marketplace | Featured.

IDE

Search Everywhere (поиск повсюду)

В обновленном окне Search Everywhere (двойное нажатие ⇧) собрано несколько действий:

  • Search Everywhere (поиск повсюду)
  • Go to Class/File/Symbol (переход к классу/файлу/символу)
  • Find Action (поиск команды)
Для каждого действия предусмотрена отдельная вкладка, переключаться между ними можно с помощью ⇥.

Run Anything

Run Anything (Double⌃) — это новый универсальный диалог, из которого можно запустить приложение в обычном режиме или из-под отладчика, а также открыть любой проект.

Чтобы запустить конфигурацию, просто начните вводить ее имя. Чтобы начать отладку, удерживайте клавишу ⇧ для переключения в режим Debug. Чтобы открыть проект, введите open и выберите нужный проект из списка.

Мониторинг активности

Эта экспериментальная функция показывает, сколько ресурсов ЦП потребляют различные подсистемы и плагины (как встроенные, так и установленные дополнительно). Чтобы получить доступ к функции, перейдите в меню Help | Activity Monitor. Если вы увидите, что какая-то подсистема потребляет необычно много ресурсов ЦП, обратитесь в нашу службу поддержки или сообщите об этом в баг-трекере.

Новый интерфейс плагинов

Мы полностью переработали меню настроек плагинов (Preferences | Plugins). С новым интерфейсом гораздо проще управлять плагинами и поддерживать установленные плагины в актуальном состоянии.

Список избранных плагинов и Marketplace

В разделе Featured вы найдете список наиболее полезных плагинов для AppCode. Плагины можно отсортировать по рейтингу или количеству скачиваний. Если вы разрабатываете плагины, познакомьтесь с нашей новой платформой для авторов плагинов.

Интеграция с системами контроля версий

Подмодули Git

AppCode теперь поддерживает Git-репозитории с подмодулями: такие репозитории клонируются рекурсивно. Для папок подмодулей доступен просмотр Diff, история и разрешение конфликтов. Кроме того, подмодули корректно обновляются при обновлении проекта.

Пулреквесты GitHub

Теперь можно просматривать все пулреквесты GitHub прямо в IDE в специальном окне GitHub Pull Requests (VCS | Git | View Pull Requests).

Специальные возможности

Чтобы сделать наши инструменты доступными для самых разных разработчиков, мы добавили несколько новых опций в раздел Appearance в настройках IDE (Settings | Appearance & Behavior | Appearance). Там вы найдете новую высококонтрастную тему интерфейса, расширенную поддержку инструментов чтения с экрана (теперь они зачитывают номера строк, аннотации VCS и значения иконок на полях редактора), а также настройки цветов IDE для людей с нарушенным цветовосприятием.
