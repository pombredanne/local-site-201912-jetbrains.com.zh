Больше файлов для Go to Text

Движок, лежащий в основе Go to Text, учитывает абсолютно все текстовые файлы, входящие в решение.

Ранее Go to Text учитывал только те файлы, которые уже были проиндексированы в ReSharper. ReSharper индексирует только файлы, содержащие код, или файлы, помогающие писать код, поскольку полная индексация прочих файлов, как правило, оказывается неоправданно ресурсоемкой. Поэтому все файлы с «неизвестными» расширениями полностью выпадали из поиска, т. е. искать можно было только в файлах кода.

Теперь движок прокачан! Во время отображения результатов из проиндексированных файлов движок в фоновом режиме ищет в неиндексированных текстовых файлах. Это очень удобно, если какие-либо данные хранятся в файлах с разными расширениями, например .*proj, .yml и .csv.

Оптимизация ссылок для проектов SDK

ReSharper теперь также проверяет проекты SDK на неиспользуемые ссылки на пакеты. Команда контекстного меню проекта Optimize References позволяет проанализировать и оптимизировать эти ссылки. Если проект не содержит кода, использующего данную ссылку, она отображается в числе неиспользуемых. Неиспользуемые пакеты можно легко удалить, чтобы дерево зависимостей проекта не захламлялось!

Автоматическое определение отступов и улучшения, касающиеся стиля кода

Уровни параметров ReSharper облегчают соблюдение стиля кода в команде, в которой большинство участников используют ReSharper. Теперь добавились параметры табуляции и отступов для всех поддерживаемых языков.

При переформатировании части кода в файле или при автоматическом форматировании кода при редактировании или вставке ReSharper может вычислить и применить необходимый отступ в соответствии с уже существующими отступами в этом файле. Этой возможностью удобно пользоваться, когда требуется добавить код в существующие проекты, отступы в которых настроены не так, как в текущих параметрах.

Контекстная настройка правил форматирования работает для С#, C++, JavaScript и TypeScript. Нажмите AltEnter на выделенном блоке кода, который требуется переформатировать, затем выберите Format Selection | Configure в списке действий, чтобы увидеть все правила форматирования, воздействующие на данный код.
